### Title
`SwapAllowlistExtension` gates the router address instead of the real user, allowing any caller to bypass per-user swap restrictions via `MetricOmmSimpleRouter` — (`metric-periphery/contracts/extensions/SwapAllowlistExtension.sol`)

---

### Summary

`SwapAllowlistExtension.beforeSwap` receives `sender` = `msg.sender` of `MetricOmmPool.swap`. When a swap is routed through `MetricOmmSimpleRouter`, that value is the **router contract address**, not the originating EOA. If the pool admin allowlists the router (the natural action for a periphery-enabled pool), every user — including those not individually allowlisted — can bypass the per-user gate by routing through the router.

---

### Finding Description

**Call chain:**

```
EOA → MetricOmmSimpleRouter.exactInputSingle(params)
        router calls: IMetricOmmPoolActions(params.pool).swap(
                          params.recipient, params.zeroForOne, ..., params.extensionData
                      )
        // msg.sender inside pool.swap == router address

MetricOmmPool.swap:
    _beforeSwap(msg.sender, ...)   // msg.sender == router
        → ExtensionCalling._beforeSwap(sender=router, ...)
            → SwapAllowlistExtension.beforeSwap(sender=router, ...)
                checks: allowedSwapper[msg.sender/*pool*/][sender/*router*/]
```

`MetricOmmPool.swap` passes `msg.sender` as the `sender` argument to `_beforeSwap`: [1](#0-0) 

`ExtensionCalling._beforeSwap` forwards that value unchanged as the first argument to the extension: [2](#0-1) 

`SwapAllowlistExtension.beforeSwap` then checks `allowedSwapper[pool][sender]`, where `sender` is the router: [3](#0-2) 

`MetricOmmSimpleRouter.exactInputSingle` calls `pool.swap` directly with no mechanism to forward the originating EOA: [4](#0-3) 

The same pattern applies to `exactInput`, `exactOutputSingle`, and `exactOutput`. [5](#0-4) 

---

### Impact Explanation

A pool configured with `SwapAllowlistExtension` is intended to restrict swaps to a curated set of addresses (e.g., KYC'd users, institutional counterparties, or protocol-controlled addresses). When the pool admin allowlists the router — the expected action for any pool that supports periphery routing — the allowlist becomes completely ineffective: **any EOA can call `router.exactInputSingle(...)` and the extension approves the swap because it sees `sender = router`**, which is allowlisted. The disallowed user receives output tokens and the pool's curation policy is silently voided. This is a direct policy bypass with fund-impacting consequences on curated pools.

---

### Likelihood Explanation

The trigger requires only two conditions, both of which are the natural production state:

1. A pool is deployed with `SwapAllowlistExtension` in its extension set.
2. The pool admin calls `setAllowedToSwap(pool, router, true)` — the obvious step to make the pool usable through the official periphery.

No privileged escalation, no malicious setup, and no non-standard tokens are required. Any unprivileged EOA can execute the bypass in a single transaction.

---

### Recommendation

The extension must resolve the originating EOA, not the immediate caller. Two approaches:

1. **Pass the original caller through the router.** Add a `msgSender()` getter to `MetricOmmSimpleRouter` (backed by transient storage, set at entry and cleared after the pool call) and have the extension read it when `sender` is a known router. This mirrors the Kyber/Uniswap V4 recommendation in the external report.

2. **Prohibit router allowlisting.** Document that allowlisting any public intermediary contract (router, adder, multicall) defeats the per-user policy, and enforce this in the extension's `setAllowedToSwap` by rejecting known periphery addresses. This is weaker because it relies on admin awareness.

The cleanest fix is approach 1: the router stores `msg.sender` in transient storage before calling the pool, exposes it via `msgSender()`, and the extension calls `IMetricOmmSimpleRouter(sender).msgSender()` when `sender` is a registered router to obtain the real user.

---

### Proof of Concept

```solidity
// Setup:
// 1. Deploy pool with SwapAllowlistExtension
// 2. Admin allowlists the router (natural production step)
extension.setAllowedToSwap(pool, address(router), true);

// 3. Alice is NOT individually allowlisted
assertFalse(extension.isAllowedToSwap(pool, alice));

// Attack: Alice routes through the router
vm.prank(alice);
router.exactInputSingle(
    IMetricOmmSimpleRouter.ExactInputSingleParams({
        pool: pool,
        recipient: alice,
        zeroForOne: true,
        amountIn: 1000,
        amountOutMinimum: 0,
        priceLimitX64: 0,
        deadline: block.timestamp,
        tokenIn: token0,
        extensionData: ""
    })
);
// Extension sees sender = address(router), which is allowlisted → swap succeeds.
// Alice, a disallowed user, has successfully swapped on a restricted pool.
```

  function exactInput(ExactInputParams calldata params) external payable returns (uint256 amountOut) {
    _checkDeadline(params.deadline);
    _validatePath(params.tokens, params.pools, params.extensionDatas);

    uint256 last = params.pools.length - 1;
    int128 amount = MetricOmmSwapInputs.asAmountSpecifiedIn(params.amountIn);

    for (uint256 i = 0; i <= last; i++) {
      address pool = params.pools[i];
      bool zeroForOne = MetricOmmSwapPath.resolveZeroForOneBitmap(params.zeroForOneBitMap, i);

      _setNextCallbackContext(pool, CALLBACK_MODE_JUST_PAY, i == 0 ? msg.sender : address(this), params.tokens[i]);
      (int128 amount0Delta, int128 amount1Delta) = IMetricOmmPoolActions(pool)
        .swap(
          i == last ? params.recipient : address(this),
          zeroForOne,
          amount,
          MetricOmmSwapPath.openLimit(zeroForOne),
          "",
          params.extensionDatas[i]
        );

      int128 amountInActual = MetricOmmSwapResults.extractAmountIn(zeroForOne, amount0Delta, amount1Delta);
      if (amountInActual < amount) revert InvalidInputAmountAtHop(uint8(i), amountInActual, amount);

      amount = MetricOmmSwapResults.extractAmountOut(zeroForOne, amount0Delta, amount1Delta);
    }

    if (amount <= 0) revert InvalidSwapDeltas();
    amountOut = MetricOmmSwapInputs.int128ToUint128(amount);
    if (amountOut < params.amountOutMinimum) revert InsufficientOutput(amountOut, params.amountOutMinimum);

    _clearExpectedCallbackPool();
  }
