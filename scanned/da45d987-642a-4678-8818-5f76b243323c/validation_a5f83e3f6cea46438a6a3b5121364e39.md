### Title
Single-Step `transferProviderOwnership` Permanently Bricks Updater Management, Enabling Swap DoS — (`smart-contracts-poc/contracts/PriceProviderFactory.sol`)

---

### Summary

`PriceProviderFactory.transferProviderOwnership` transfers the `providerOwner` role in a single atomic step with only a zero-address guard. If the new owner address is inaccessible (typo, wrong contract, burned key), all updater-management functions (`grantUpdater`, `revokeUpdater`) are permanently locked. When existing updaters eventually stop posting prices, the `PriceProvider` begins returning stale data, causing every `MetricOmmPool` swap that relies on it to revert, making the pool's swap flow permanently unusable without an emergency price-provider rotation.

---

### Finding Description

`PriceProviderFactory` is permissionlessly deployable: any caller can invoke `createPriceProvider`, which sets `providerOwner[provider] = msg.sender`. [1](#0-0) 

The ownership transfer function is:

```solidity
function transferProviderOwnership(address provider, address newOwner)
    external override onlyProviderOwner(provider)
{
    require(_providers.contains(provider), ProviderNotTracked());
    require(newOwner != address(0));
    address previousOwner = providerOwner[provider];

    providerOwner[provider] = newOwner;
    _providersByCreator[previousOwner].remove(provider);
    _providersByCreator[newOwner].add(provider);

    emit ProviderOwnershipTransferred(provider, previousOwner, newOwner);
}
``` [2](#0-1) 

The only validation is `newOwner != address(0)`. There is no pending-owner pattern, no acceptance step, and no cancellation path. Once `providerOwner[provider]` is overwritten with an inaccessible address, the two gating functions become permanently unreachable: [3](#0-2) 

The pool's swap path calls `_getBidAndAskPriceX64()`, which calls `IPriceProvider(activePriceProvider).getBidAndAskPrice()`. If the provider's `maxTimeDelta` is exceeded (because no updater can be granted to replace a defunct one), the call reverts and the entire swap reverts. [4](#0-3) 

By contrast, `MetricOmmPoolFactory` already applies the two-step pattern for its own ownership (via `Ownable2Step`) and for pool-admin transfers (`proposePoolAdminTransfer` / `acceptPoolAdmin`): [5](#0-4) [6](#0-5) 

`PriceProviderFactory` applies no equivalent protection.

---

### Impact Explanation

- **Updater management permanently bricked.** With the owner inaccessible, no new updater can be granted and no compromised updater can be revoked.
- **Swap DoS.** Once all active updaters for the provider stop posting (key rotation, operational failure, or deliberate exit), the provider's staleness check causes every swap on every pool bound to that provider to revert.
- **No in-contract recovery.** The factory's `ADMIN_ROLE` can only `addProvider`/`removeProvider` from the registry; it cannot reassign `providerOwner`. Recovery requires the pool admin to propose and execute a price-provider rotation through `MetricOmmPoolFactory.proposePoolPriceProvider` + `executePoolPriceProviderUpdate`, subject to the pool's configured timelock — during which swaps remain broken. [7](#0-6) 

---

### Likelihood Explanation

- `createPriceProvider` is permissionless; any EOA or contract can become a `providerOwner`.
- A single-character address typo during `transferProviderOwnership` is sufficient to trigger the condition.
- The external bug report (Clober/Spearbit) documents exactly this class of error occurring in production.
- No existing guard in `PriceProviderFactory` prevents it.

---

### Recommendation

Replace the single-step transfer with a two-step pattern mirroring `MetricOmmPoolFactory`'s pool-admin transfer:

1. Add `mapping(address => address) public pendingProviderOwner`.
2. `proposeProviderOwnershipTransfer(address provider, address newOwner)` — callable by current owner; stores `pendingProviderOwner[provider] = newOwner`.
3. `acceptProviderOwnership(address provider)` — callable only by `pendingProviderOwner[provider]`; overwrites `providerOwner` and clears the pending slot.
4. `cancelProviderOwnershipTransfer(address provider)` — callable by current owner to abort.

This mirrors the already-correct pattern at: [8](#0-7) 

---

### Proof of Concept

```
1. Alice calls PriceProviderFactory.createPriceProvider(...) → providerOwner[P] = Alice.
2. Alice calls transferProviderOwnership(P, 0xDEAD...BEEF)  // typo; address is inaccessible.
   → providerOwner[P] = 0xDEAD...BEEF  (irreversible).
3. The sole updater for P stops posting prices (key lost / service shutdown).
4. Any call to MetricOmmPool.swap() on a pool bound to P:
     _getBidAndAskPriceX64()
       → IPriceProvider(P).getBidAndAskPrice()
       → staleness check fails → revert PriceProviderFailed(...)
5. Swaps on the affected pool are permanently broken until the pool admin
   completes a full price-provider rotation through the factory timelock.
```

    external
    override
    nonReentrant
    onlyPoolAdmin(pool)
  {
    PoolImmutables memory p = IMetricOmmPool(pool).getImmutables();
    uint256 timelock = priceProviderTimelock[pool];
    if (p.immutablePriceProvider != address(0)) revert PriceProviderImmutable();
    _validatePriceProvider(p.token0, p.token1, newPriceProvider);

    address mutableProvider = PoolStateLibrary._slot3(pool);
    address current = mutableProvider != address(0) ? mutableProvider : p.immutablePriceProvider;
    uint256 executeAfter = block.timestamp + timelock;
    pendingPriceProvider[pool] = newPriceProvider;
    pendingPriceProviderExecuteAfter[pool] = executeAfter;
    emit PoolPriceProviderChangeProposed(pool, current, newPriceProvider, executeAfter);
  }

  /// @inheritdoc IMetricOmmPoolFactoryPoolAdmin
  function executePoolPriceProviderUpdate(address pool) external override nonReentrant onlyPoolAdmin(pool) {
    address pending = pendingPriceProvider[pool];
    if (pending == address(0)) revert NoPriceProviderChangeProposed();
    uint256 execAfter = pendingPriceProviderExecuteAfter[pool];
    // forge-lint: disable-next-line(block-timestamp) -- timelock enforcement legitimately relies on `block.timestamp`.
    if (block.timestamp < execAfter) revert PriceProviderTimelockNotElapsed(execAfter, block.timestamp);
    PoolImmutables memory p = IMetricOmmPool(pool).getImmutables();
    if (p.immutablePriceProvider != address(0)) revert PriceProviderImmutable();
    _validatePriceProvider(p.token0, p.token1, pending);
    IMetricOmmPoolFactoryActions(pool).setPriceProvider(pending);
    delete pendingPriceProvider[pool];
    delete pendingPriceProviderExecuteAfter[pool];
    emit PoolPriceProviderUpdated(pool, pending);
  }

  function proposePoolAdminTransfer(address pool, address newAdmin) external override nonReentrant onlyPoolAdmin(pool) {
    if (newAdmin == address(0)) revert InvalidAdmin();
    if (newAdmin == poolAdmin[pool]) revert InvalidAdmin();
    pendingPoolAdmin[pool] = newAdmin;
    emit PoolAdminTransferProposed(pool, poolAdmin[pool], newAdmin);
  }

  /// @inheritdoc IMetricOmmPoolFactoryPoolAdmin
  function acceptPoolAdmin(address pool) external override nonReentrant {
    address pending = pendingPoolAdmin[pool];
    if (pending == address(0)) revert NoPendingPoolAdminTransfer();
    if (msg.sender != pending) revert NotPendingPoolAdmin(pool, msg.sender, pending);
    address previousAdmin = poolAdmin[pool];
    poolAdmin[pool] = pending;
    delete pendingPoolAdmin[pool];
    emit PoolAdminTransferred(pool, previousAdmin, pending);
  }

  /// @inheritdoc IMetricOmmPoolFactoryPoolAdmin
  function cancelPoolAdminTransfer(address pool) external override nonReentrant onlyPoolAdmin(pool) {
    address pending = pendingPoolAdmin[pool];
    if (pending == address(0)) revert NoPendingPoolAdminTransfer();
    delete pendingPoolAdmin[pool];
    emit PoolAdminTransferCancelled(pool, pending);
  }
