### Title
`SwapAllowlistExtension` Checks the Router Address Instead of the Actual User, Allowing Any User to Bypass the Swap Allowlist via `MetricOmmSimpleRouter` â€” (`metric-periphery/contracts/extensions/SwapAllowlistExtension.sol`)

---

### Summary

`SwapAllowlistExtension.beforeSwap` gates swaps by checking the `sender` parameter, which is `msg.sender` of `pool.swap()`. When `MetricOmmSimpleRouter` is the caller of `pool.swap()`, `sender` equals the router address, not the actual end-user. If the pool admin adds the router to the allowlist (the only way to let any user swap through the router), every userâ€”including those not individually allowlistedâ€”can bypass the per-user curation policy.

---

### Finding Description

`MetricOmmPool.swap()` passes `msg.sender` as the `sender` argument to `_beforeSwap`, which forwards it to every configured extension: [1](#0-0) 

`SwapAllowlistExtension.beforeSwap` then checks whether that `sender` is on the per-pool allowlist: [2](#0-1) 

`MetricOmmSimpleRouter.exactInputSingle` (and all other router swap paths) calls `pool.swap()` directly, making the router itself `msg.sender` of that call: [3](#0-2) 

The same pattern holds for `exactInput` (all hops), `exactOutputSingle`, and `exactOutput` (all recursive hops): [4](#0-3) 

Consequently, the extension always evaluates `allowedSwapper[pool][router]`, never `allowedSwapper[pool][actualUser]`. The pool admin faces an impossible choice:

- **Router NOT allowlisted**: no user can swap through the router, even individually allowlisted ones.
- **Router IS allowlisted**: every user can swap through the router, even those not individually allowlisted.

The actual user's address is never forwarded to the extension; there is no mechanism to recover it inside `beforeSwap`.

The `DepositAllowlistExtension` does not share this flaw because it gates by the `owner` parameter (the position owner explicitly passed to `addLiquidity`), which the `MetricOmmPoolLiquidityAdder` correctly sets to the intended beneficiary: [5](#0-4) 

---

### Impact Explanation

A pool configured with `SwapAllowlistExtension` to restrict trading to KYC'd, institutional, or otherwise curated addresses is completely open to any user who routes through `MetricOmmSimpleRouter`. The allowlist provides no protection on the standard periphery path. Any non-allowlisted address can execute live swaps against the pool, receiving real token output and paying real token input, defeating the pool admin's curation policy. This is a direct, unprivileged bypass of an access-control guard with fund-flow consequences (unauthorized trades execute at oracle prices against LP capital).

---

### Likelihood Explanation

The `MetricOmmSimpleRouter` is the standard, documented periphery for swapping. Pool admins who deploy a `SwapAllowlistExtension` and want any user to swap through the router must add the router to the allowlistâ€”the natural and expected configuration. Once the router is allowlisted, the bypass is trivially reachable by any address with no special setup. The trigger requires only a standard `exactInputSingle` or `exactInput` call.

---

### Recommendation

The `beforeSwap` extension hook must receive the original end-user's address to enforce per-user policies correctly. Two approaches:

1. **Pass the payer/originator through `extensionData`**: The router encodes `msg.sender` (the actual user) into `extensionData` before calling `pool.swap()`. The `SwapAllowlistExtension` decodes and checks that address instead of `sender`.

2. **Check `recipient` as a proxy**: For single-hop swaps where the user is also the recipient, the extension could check `recipient`. This is fragile for multi-hop paths where intermediate recipients are the router itself.

The cleanest fix is option 1: the router appends the originating user address to `extensionData`, and the `SwapAllowlistExtension` reads it from there, verifying it against the allowlist instead of the `sender` (router) address.

---

### Proof of Concept

```solidity
// Pool is deployed with SwapAllowlistExtension as beforeSwap extension.
// Pool admin allowlists alice and the router (so alice can use the router).
extension.setAllowedToSwap(address(pool), alice, true);
extension.setAllowedToSwap(address(pool), address(router), true);

// charlie is NOT on the allowlist.
// charlie calls the router directly:
vm.prank(charlie);
router.exactInputSingle(
    IMetricOmmSimpleRouter.ExactInputSingleParams({
        pool: address(pool),
        tokenIn: address(token1),
        recipient: charlie,
        deadline: block.timestamp + 1,
        amountIn: 1000,
        amountOutMinimum: 0,
        zeroForOne: false,
        priceLimitX64: 0,
        extensionData: ""
    })
);
// Succeeds: extension sees sender = router (allowlisted), never checks charlie.
// charlie receives token0 output despite not being on the allowlist.
```
