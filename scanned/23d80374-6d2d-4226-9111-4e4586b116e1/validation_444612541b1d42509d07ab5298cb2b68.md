### Title
SwapAllowlistExtension Checks Router Address Instead of Actual User, Allowing Any User to Bypass the Swap Allowlist — (`metric-periphery/contracts/extensions/SwapAllowlistExtension.sol`)

### Summary

`SwapAllowlistExtension.beforeSwap()` gates swaps by checking the `sender` argument, which is the `msg.sender` of `pool.swap()`. When users route through `MetricOmmSimpleRouter`, `sender` equals the router's address, not the end user. A pool admin who allowlists the router (required for any router-mediated swap to work for legitimate users) inadvertently opens the pool to every user, because the extension cannot distinguish which end user is behind the router call.

### Finding Description

`SwapAllowlistExtension.beforeSwap()` performs its identity check as follows:

```solidity
// SwapAllowlistExtension.sol L37
if (!allowAllSwappers[msg.sender] && !allowedSwapper[msg.sender][sender]) {
    revert IMetricOmmPoolActions.NotAllowedToSwap();
}
```

`msg.sender` here is the pool (the contract calling the extension), and `sender` is the value the pool forwarded — which is the `msg.sender` of `pool.swap()`.

In `MetricOmmPool.swap()`, the pool passes its own `msg.sender` as `sender`:

```solidity
// MetricOmmPool.sol L230-240
_beforeSwap(
    msg.sender,   // ← whoever called pool.swap()
    recipient,
    ...
);
```

When a user calls `router.exactInputSingle()`, the router calls `pool.swap()` with itself as `msg.sender`:

```solidity
// MetricOmmSimpleRouter.sol L72-80
(int128 amount0Delta, int128 amount1Delta) = IMetricOmmPoolActions(params.pool)
    .swap(
        params.recipient,
        params.zeroForOne,
        MetricOmmSwapInputs.asAmountSpecifiedIn(params.amountIn),
        priceLimitX64,
        "",
        params.extensionData
    );
```

So the extension receives `sender = address(router)`, not the end user's address. The allowlist lookup becomes `allowedSwapper[pool][router]`.

**Bypass path:**

1. Pool admin deploys a pool with `SwapAllowlistExtension` and allowlists `user1`, `user2` (trusted traders).
2. To let `user1`/`user2` use the router, the admin must also allowlist `address(router)`.
3. Once the router is allowlisted, any address — including `user3` who is not on the allowlist — can call `router.exactInputSingle()` and the extension will pass, because it only sees `sender = router`.

The pool admin has no way to selectively allow specific users to use the router while blocking others; it is all-or-nothing.

### Impact Explanation

A curated pool's swap allowlist is completely bypassed for any user who routes through `MetricOmmSimpleRouter`. Unauthorized traders can execute swaps on pools that were intended to be restricted to specific counterparties. This breaks the core curation invariant and exposes LP funds to trades from actors the pool admin explicitly excluded. The impact is direct loss of LP value if the allowlist was protecting against adversarial or uninformed order flow.

### Likelihood Explanation

Any pool that (a) uses `SwapAllowlistExtension` and (b) wants to support router-mediated swaps for its allowlisted users is vulnerable. The pool admin must allowlist the router to enable router usage, and doing so silently removes all per-user enforcement. This is a routine configuration that any operator of a curated pool would reach.

### Recommendation

The extension should receive and check the **original end-user address**, not the immediate caller of `pool.swap()`. One approach: pass the end user's address through `extensionData` and verify it in the extension (with the router signing or forwarding it). A cleaner approach: add a dedicated `swapper` field to the `beforeSwap` hook arguments that the pool populates from a trusted source (e.g., a transient-storage context set by the router before calling the pool), so the extension can always gate the true economic actor regardless of intermediary.

Alternatively, document clearly that allowlisting the router grants access to all users, and provide a separate router that forwards the original `msg.sender` as a verified parameter.

### Proof of Concept

```
Pool admin:
  allowedSwapper[pool][user1] = true
  allowedSwapper[pool][router] = true   // needed for user1 to use router

Attacker (user3, not allowlisted):
  router.exactInputSingle({
      pool: pool,
      tokenIn: token0,
      ...
  })
  // pool.swap() is called with msg.sender = router
  // extension checks allowedSwapper[pool][router] → true
  // swap executes despite user3 not being allowlisted
