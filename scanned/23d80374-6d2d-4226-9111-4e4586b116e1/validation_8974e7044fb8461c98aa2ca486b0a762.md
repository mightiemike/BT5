**Fee collection** (`collectFees`): `notionalFeeToken0Scaled` and `notionalFeeToken1Scaled` are correctly zeroed after transfer — no analog. [1](#0-0) 

**LP share removal** (`removeLiquidity`): `positionBinShares[posKey]` and `binTotalShares[binIdx]` are decremented before tokens are transferred — no analog. [2](#0-1) 

**`PriceVelocityGuardExtension.beforeSwap`**: State (`lastMidPriceX64`, `lastUpdateBlock`) is written before the velocity check. If the check reverts, the write is rolled back atomically — no persistent state-not-reset issue. [3](#0-2) 

**`OracleValueStopLossExtension._checkAndUpdateWatermarks`**: Watermarks are written only after both breach checks pass; a revert rolls back all writes — no state-not-reset issue. [4](#0-3) 

**`SwapAllowlistExtension.beforeSwap`**: Checks `sender` (the immediate `msg.sender` of the pool's `swap` call). When the router mediates a swap, `sender` is the router, not the end user. If the router is allowlisted, all users bypass the allowlist — but this requires the pool admin to explicitly allowlist the router, which is a privileged configuration
