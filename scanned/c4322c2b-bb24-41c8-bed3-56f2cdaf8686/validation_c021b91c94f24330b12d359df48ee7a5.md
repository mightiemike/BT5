### Title
`SwapAllowlistExtension` Swap Allowlist Bypassed via `MetricOmmSimpleRouter` — Wrong-Actor Binding on `sender` - (File: `metric-periphery/contracts/extensions/SwapAllowlistExtension.sol`)

---

### Summary

`SwapAllowlistExtension.beforeSwap` gates swaps by checking the `sender` argument, which the pool sets to `msg.sender` — the **direct caller of `pool.swap()`**. When a user swaps through `MetricOmmSimpleRouter`, the router is the direct caller, so `sender` is the router's address, not the user's. If the pool admin allowlists the router (required for router-based swaps to function), every user — including those not individually allowlisted — bypasses the swap allowlist entirely.

---

### Finding Description

**Call chain:**

```
User → MetricOmmSimpleRouter.exactInputSingle()
         → pool.swap(recipient, ..., extensionData)   // msg.sender = router
              → _beforeSwap(msg.sender=router, ...)
                   → SwapAllowlistExtension.beforeSwap(sender=router, ...)
                        → checks allowedSwapper[pool][router]  ← router, not user
```

In `MetricOmmPool.swap`, the pool passes `msg.sender` as `sender` to the extension dispatcher:

```solidity
_beforeSwap(
  msg.sender,   // ← router address when called via router
  recipient,
  ...
);
``` [1](#0-0) 

`SwapAllowlistExtension.beforeSwap` then checks `allowedSwapper[msg.sender][sender]`, where `msg.sender` is the pool and `sender` is the router:

```solidity
function beforeSwap(address sender, ...) external view override returns (bytes4) {
  if (!allowAllSwappers[msg.sender] && !allowedSwapper[msg.sender][sender]) {
    revert IMetricOmmPoolActions.NotAllowedToSwap();
  }
  ...
}
``` [2](#0-1) 

In `MetricOmmSimpleRouter.exactInputSingle`, the router calls `pool.swap()` directly — there is no mechanism to forward the original user's address as `sender`:

```solidity
(int128 amount0Delta, int128 amount1Delta) = IMetricOmmPoolActions(params.pool)
  .swap(params.recipient, params.zeroForOne, ..., params.extensionData);
``` [3](#0-2) 

The same wrong-actor binding applies to `exactInput`, `exactOutputSingle`, and `exactOutput`. [4](#0-3) 

---

### Impact Explanation

A pool admin who deploys a curated pool (e.g., KYC-gated, institutional-only, or compliance-restricted) and attaches `SwapAllowlistExtension` intends to restrict swaps to individually approved addresses. To also support the standard periphery UX, the admin must allowlist the router address. Once the router is allowlisted, **any user** — regardless of individual allowlist status — can call `router.exactInputSingle()` and the extension check passes because it sees `sender = router`, which is allowlisted. The curation policy is completely nullified. Unauthorized users can trade on the pool, extract value, and violate any compliance or economic invariant the allowlist was meant to enforce.

---

### Likelihood Explanation

High. `MetricOmmSimpleRouter` is the standard, documented periphery swap interface. Any pool operator who:
1. Deploys a pool with `SwapAllowlistExtension` to restrict access, **and**
2. Allowlists the router so that approved users can use the standard UX

is immediately vulnerable. This is the expected operational pattern. The bypass requires no special privileges, no flash loans, and no unusual token behavior — any EOA can call the router.

---

### Recommendation

The `sender` argument passed to `beforeSwap` must represent the **economically responsible actor** (the user), not the intermediary router. Two approaches:

1. **Extension-side:** `SwapAllowlistExtension` should check both `sender` (the direct pool caller) and, if `sender` is a known trusted router, fall back to checking the router's stored payer context. This requires a protocol-level convention for routers to expose the originating user.

2. **Pool/router-side:** The pool's `swap()` interface should accept an explicit `swapper` address (separate from `msg.sender`) that the router populates with `msg.sender` before calling the pool. The extension then checks this explicit `swapper` field. This is the cleanest fix and mirrors how `addLiquidity` separates `sender` from `owner`.

Until fixed, pool admins should not rely on `SwapAllowlistExtension` for access control on pools that are also intended to be reachable via `MetricOmmSimpleRouter`.

---

### Proof of Concept

```solidity
// Setup: pool with SwapAllowlistExtension; only `allowedUser` is allowlisted.
// Router is allowlisted so allowedUser can use it.
extension.setAllowedToSwap(address(pool), address(router), true);
extension.setAllowedToSwap(address(pool), allowedUser, true);
// bannedUser is NOT allowlisted.

// bannedUser bypasses the allowlist via the router:
vm.prank(bannedUser);
token0.approve(address(router), type(uint256).max);

vm.prank(bannedUser);
uint256 amountOut = router.exactInputSingle(
    IMetricOmmSimpleRouter.ExactInputSingleParams({
        pool: address(pool),
        tokenIn: address(token0),
        tokenOut: address(token1),
        zeroForOne: true,
        amountIn: 1_000,
        amountOutMinimum: 0,
        recipient: bannedUser,
        deadline: block.timestamp + 1,
        priceLimitX64: 0,
        extensionData: ""
    })
);
// ✓ Swap succeeds — bannedUser traded on a pool they should not have access to.
// The extension checked allowedSwapper[pool][router] = true, not allowedSwapper[pool][bannedUser].
assertGt(amountOut, 0);
```

  function exactInput(ExactInputParams calldata params) external payable returns (uint256 amountOut) {
    _checkDeadline(params.deadline);
    _validatePath(params.tokens, params.pools, params.extensionDatas);

    uint256 last = params.pools.length - 1;
    int128 amount = MetricOmmSwapInputs.asAmountSpecifiedIn(params.amountIn);

    for (uint256 i = 0; i <= last; i++) {
      address pool = params.pools[i];
      bool zeroForOne = MetricOmmSwapPath.resolveZeroForOneBitmap(params.zeroForOneBitMap, i);

      _setNextCallbackContext(pool, CALLBACK_MODE_JUST_PAY, i == 0 ? msg.sender : address(this), params.tokens[i]);
      (int128 amount0Delta, int128 amount1Delta) = IMetricOmmPoolActions(pool)
        .swap(
          i == last ? params.recipient : address(this),
          zeroForOne,
          amount,
          MetricOmmSwapPath.openLimit(zeroForOne),
          "",
          params.extensionDatas[i]
        );

      int128 amountInActual = MetricOmmSwapResults.extractAmountIn(zeroForOne, amount0Delta, amount1Delta);
      if (amountInActual < amount) revert InvalidInputAmountAtHop(uint8(i), amountInActual, amount);

      amount = MetricOmmSwapResults.extractAmountOut(zeroForOne, amount0Delta, amount1Delta);
    }

    if (amount <= 0) revert InvalidSwapDeltas();
    amountOut = MetricOmmSwapInputs.int128ToUint128(amount);
    if (amountOut < params.amountOutMinimum) revert InsufficientOutput(amountOut, params.amountOutMinimum);

    _clearExpectedCallbackPool();
  }
