### Title
SwapAllowlistExtension Checks Router Address Instead of Actual User, Allowing Allowlist Bypass - (File: metric-periphery/contracts/extensions/SwapAllowlistExtension.sol)

---

### Summary

`SwapAllowlistExtension.beforeSwap()` gates swaps by checking the `sender` argument passed by the pool, which is `msg.sender` of the `pool.swap()` call. When a user routes through `MetricOmmSimpleRouter`, the router is `msg.sender` to the pool, so the allowlist checks the **router address** rather than the actual end-user. If the pool admin allowlists the router (the natural step to enable router-based swaps on a curated pool), every user — including those not individually allowlisted — can bypass the per-user gate.

---

### Finding Description

**Call chain:**

```
User (not allowlisted)
  → MetricOmmSimpleRouter.exactInputSingle()
      → pool.swap(recipient, ...)          // msg.sender = router
          → _beforeSwap(msg.sender, ...)   // sender = router address
              → SwapAllowlistExtension.beforeSwap(router, ...)
                  → allowedSwapper[pool][router]  ← checked, NOT the user
```

`MetricOmmPool.swap()` passes `msg.sender` as the `sender` argument to `_beforeSwap`: [1](#0-0) 

`ExtensionCalling._beforeSwap()` forwards that value unchanged to every configured extension: [2](#0-1) 

`SwapAllowlistExtension.beforeSwap()` then checks `allowedSwapper[msg.sender][sender]`, where `msg.sender` is the pool and `sender` is whoever called `pool.swap()`: [3](#0-2) 

When the router is the caller of `pool.swap()`, `sender` = router address. The extension never sees the original user's address.

The router calls `pool.swap()` directly with no forwarding of the original `msg.sender`: [4](#0-3) 

---

### Impact Explanation

A pool admin deploying a curated pool (e.g., KYC-only, institutional-only) attaches `SwapAllowlistExtension` and allowlists specific user addresses. To let those users trade conveniently, the admin also allowlists the router address. At that point, **any** address — including those explicitly excluded — can call `MetricOmmSimpleRouter.exactInputSingle()` and pass the allowlist check, because the extension only sees the router. The curation invariant is completely broken: the pool accepts swaps from arbitrary users, exposing LP capital to trades the pool was designed to reject.

---

### Likelihood Explanation

The scenario requires the pool admin to allowlist the router, which is the natural operational step for any curated pool that intends to support the standard periphery. The bypass is then reachable by any unprivileged user with no special setup. The `exactInputSingle`, `exactInput`, `exactOutputSingle`, and `exactOutput` paths all exhibit the same behavior. [5](#0-4) 

---

### Recommendation

The pool must forward the original end-user identity to the extension rather than its own `msg.sender`. Two options:

1. **Router passes the originating user as a separate argument** (e.g., in `callbackData` or a dedicated field), and the pool propagates it to extensions as a distinct `originator` parameter.
2. **`SwapAllowlistExtension` reads the originating user from transient storage** set by the router at entry, analogous to how the router already stores the payer in transient context for the swap callback.

The `DepositAllowlistExtension` does not share this flaw because it ignores `sender` and gates on `owner` instead: [6](#0-5) 

---

### Proof of Concept

```
Setup:
  pool configured with SwapAllowlistExtension
  allowedSwapper[pool][alice] = true          // alice is KYC'd
  allowedSwapper[pool][router] = true         // admin enables router for alice
  allowedSwapper[pool][attacker] = false      // attacker is NOT allowlisted

Attack:
  attacker calls router.exactInputSingle({pool: pool, ...})
    → router calls pool.swap(recipient, ...)
        → pool calls _beforeSwap(router, ...)
            → SwapAllowlistExtension checks allowedSwapper[pool][router] == true
            → PASSES — attacker's swap executes on the curated pool
```

The attacker receives output tokens from a pool that was designed to exclude them, with no privileged access required beyond calling the public router.
