### Title
SwapAllowlistExtension Checks Router Address Instead of Originating EOA, Allowing Full Allowlist Bypass via MetricOmmSimpleRouter - (File: metric-periphery/contracts/extensions/SwapAllowlistExtension.sol)

---

### Summary

`SwapAllowlistExtension.beforeSwap` gates swaps by checking the `sender` argument passed by the pool. When a user swaps through `MetricOmmSimpleRouter`, the pool's `msg.sender` is the router contract, not the originating EOA. The extension therefore checks whether the **router** is allowlisted, not the actual user. Any pool admin who allowlists the router to enable router-based swaps for legitimate users simultaneously opens the gate to every unprivileged address on the network.

---

### Finding Description

`MetricOmmPool.swap` passes `msg.sender` as the `sender` argument to `_beforeSwap`, which forwards it verbatim to every configured extension:

```solidity
// MetricOmmPool.sol line 230-240
_beforeSwap(
    msg.sender,   // ← always the direct caller of pool.swap()
    recipient,
    ...
);
```

`SwapAllowlistExtension.beforeSwap` then checks that `sender` is allowlisted for the calling pool (`msg.sender` inside the extension = the pool):

```solidity
// SwapAllowlistExtension.sol line 37
if (!allowAllSwappers[msg.sender] && !allowedSwapper[msg.sender][sender]) {
    revert IMetricOmmPoolActions.NotAllowedToSwap();
}
```

When a user calls `MetricOmmSimpleRouter.exactInputSingle` or `exactInput`, the router is the entity that calls `pool.swap(...)`:

```solidity
// MetricOmmSimpleRouter.sol line 72-80
(int128 amount0Delta, int128 amount1Delta) = IMetricOmmPoolActions(params.pool)
    .swap(
        params.recipient,
        params.zeroForOne,
        MetricOmmSwapInputs.asAmountSpecifiedIn(params.amountIn),
        priceLimitX64,
        "",
        params.extensionData
    );
```

So `sender` arriving at the extension is `address(router)`, not the originating EOA. The extension has no visibility into who initiated the transaction.

**The bypass path:** A pool admin who wants allowlisted EOAs to be able to use the router must add `allowedSwapper[pool][router] = true`. Once that entry exists, every address on the network can call `router.exactInputSingle(pool, ...)` and the extension passes, because it only sees the router address.

The same structural problem applies to `exactInput` (multi-hop), `exactOutputSingle`, and `exactOutput`.

---

### Impact Explanation

A curated pool protected by `SwapAllowlistExtension` is intended to restrict swaps to a specific set of counterparties. Once the router is allowlisted (the only way to let legitimate users trade through the standard periphery), the restriction is nullified for all callers. Any unprivileged address can execute swaps at oracle-anchored prices against LP capital that was deposited under the assumption that only vetted counterparties would trade. This constitutes unauthorized extraction of LP value and a complete failure of the pool's curation policy.

---

### Likelihood Explanation

The `MetricOmmSimpleRouter` is the standard, documented swap entry point for end users. A pool admin who deploys a curated pool and wants allowlisted users to access it through the normal UX will inevitably allowlist the router. The bypass requires no special knowledge, no privileged access, and no unusual token behavior — only a standard router call.

---

### Recommendation

The extension must recover the originating EOA rather than trusting the `sender` argument, which reflects only the immediate caller of `pool.swap`. Two sound approaches:

1. **Require direct pool interaction for allowlisted pools.** Document that pools using `SwapAllowlistExtension` must not be accessed through the router; the router should not be allowlisted. Allowlisted EOAs call `pool.swap` directly.

2. **Thread the originating caller through the router.** Add an authenticated "original sender" field to the extension payload (`extensionData`) that the router populates with `msg.sender` before calling the pool. The extension decodes and checks this field instead of the `sender` argument. This requires the extension to trust the router's self-reported identity, which is safe only if the router is itself allowlisted and the payload is authenticated (e.g., the extension verifies `msg.sender` is the router before trusting the decoded EOA).

---

### Proof of Concept

```
Setup:
  pool  = MetricOmmPool with SwapAllowlistExtension (beforeSwap order set)
  alice = allowlisted EOA: allowedSwapper[pool][alice] = true
  bob   = non-allowlisted EOA

Step 1 (admin, legitimate):
  extension.setAllowedToSwap(pool, address(router), true)
  // Admin adds router so alice can use the standard UX

Step 2 (bob, attacker):
  router.exactInputSingle(ExactInputSingleParams({
      pool:      address(pool),
      tokenIn:   token0,
      ...
      recipient: bob,
      ...
  }))
  // pool.swap is called with msg.sender = router
  // extension checks allowedSwapper[pool][router] → true
  // bob's swap executes despite not being on the allowlist
```

The extension passes because `sender = address(router)` is allowlisted, not because `bob` is allowlisted. Bob receives oracle-priced output from LP capital that was never meant to serve him. [1](#0-0) [2](#0-1) [3](#0-2) [4](#0-3) [5](#0-4)
