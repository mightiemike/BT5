### Title
SwapAllowlistExtension Gates the Router Address Instead of the End User, Allowing Any Caller to Bypass a Curated Pool's Swap Allowlist via the Public Router - (File: `metric-periphery/contracts/extensions/SwapAllowlistExtension.sol`)

---

### Summary

`SwapAllowlistExtension.beforeSwap` checks the `sender` argument passed by the pool, which is `msg.sender` of the `pool.swap()` call. When a user routes through `MetricOmmSimpleRouter`, the router is `msg.sender` to the pool, so the extension checks whether the **router** is allowlisted — not the actual end user. If the pool admin allowlists the router address (the natural configuration to enable router-based trading), the swap allowlist is completely bypassed for every user, including those the pool was designed to exclude.

---

### Finding Description

`SwapAllowlistExtension.beforeSwap` enforces its check as:

```solidity
// SwapAllowlistExtension.sol L37-39
if (!allowAllSwappers[msg.sender] && !allowedSwapper[msg.sender][sender]) {
    revert IMetricOmmPoolActions.NotAllowedToSwap();
}
```

`msg.sender` here is the pool (correct), and `sender` is the first argument the pool passes when it calls the extension hook.

In `MetricOmmPool.swap`, the pool dispatches the before-swap hook as `_beforeSwap(msg.sender, ...)`, where `msg.sender` is whoever called `pool.swap()`. When a user calls `router.exactInputSingle(...)`, the router calls `pool.swap(...)` directly:

```solidity
// MetricOmmSimpleRouter.sol L72-80
(int128 amount0Delta, int128 amount1Delta) = IMetricOmmPoolActions(params.pool)
  .swap(
    params.recipient,
    params.zeroForOne,
    MetricOmmSwapInputs.asAmountSpecifiedIn(params.amountIn),
    priceLimitX64,
    "",
    params.extensionData
  );
```

So `msg.sender` to the pool is the **router contract**, and `sender` forwarded to the extension is the **router address** — not the end user (`msg.sender` of the router call). The extension therefore checks `allowedSwapper[pool][router]`, not `allowedSwapper[pool][endUser]`.

The same actor-mismatch applies to `exactOutputSingle`, `exactInput`, and `exactOutput`.

---

### Impact Explanation

**Critical/High — allowlist bypass enabling unauthorized swaps on curated pools.**

A pool admin who deploys a `SwapAllowlistExtension`-gated pool and allowlists the router address (to enable router-mediated trading for approved users) inadvertently opens the pool to **all** callers. Any address — including those explicitly not in the allowlist — can call `router.exactInputSingle(...)` and execute swaps against the curated pool. The pool's curation policy is entirely nullified. Funds flow out of the pool to unauthorized counterparties, and the pool's LP providers suffer trades they never consented to allow.

---

### Likelihood Explanation

**High.** The `MetricOmmSimpleRouter` is the primary user-facing swap entrypoint. Pool admins who want to allow router-based swaps for their approved users must allowlist the router address — there is no other mechanism. This is the expected operational configuration, and it is the exact configuration that triggers the bypass. Any non-allowlisted user who discovers the router path can exploit it immediately without any privileged access or special setup.

---

### Recommendation

The extension must recover the true end user rather than accepting the router-supplied `sender` argument at face value. Two sound approaches:

1. **Pass `tx.origin` as a secondary check** (weak, susceptible to contract callers).
2. **Require the router to forward the original caller in `extensionData`**, and have the extension decode and verify it — but this is forgeable unless the pool enforces it.
3. **Preferred:** The pool should expose a `msg.sender`-authenticated swap entry that records the original caller in transient storage, and the extension should read that transient slot rather than the `sender` argument. Alternatively, the allowlist should gate on the **callback payer** (the address that actually settles tokens), which the router stores in transient storage and which cannot be spoofed.

At minimum, document that allowlisting the router address opens the pool to all callers, and provide a router variant that forwards the original caller identity in a verifiable way.

---

### Proof of Concept

1. Pool admin deploys a pool with `SwapAllowlistExtension` configured as the `beforeSwap` hook.
2. Admin calls `swapExtension.setAllowedToSwap(pool, address(router), true)` to enable router-based swaps for approved users.
3. Attacker (not in the allowlist) calls:
   ```solidity
   router.exactInputSingle(ExactInputSingleParams({
       pool: curatedPool,
       tokenIn: token0,
       tokenOut: token1,
       zeroForOne: true,
       amountIn: 1_000_000,
       amountOutMinimum: 0,
       recipient: attacker,
       deadline: block.timestamp + 1,
       priceLimitX64: 0,
       extensionData: ""
   }));
   ```
4. The router calls `pool.swap(attacker, true, ...)`. The pool calls `_beforeSwap(address(router), ...)`. The extension checks `allowedSwapper[pool][router]` → `true`. The swap executes. The attacker receives output tokens from a pool they were never authorized to trade against.

The allowlist invariant — that only explicitly approved addresses may swap — is broken for every pool that allowlists the router. [1](#0-0) [2](#0-1) [3](#0-2)
