### Title
SwapAllowlistExtension Checks Immediate Caller (Router) Instead of Original User, Allowing Full Allowlist Bypass via Public Router - (File: metric-periphery/contracts/extensions/SwapAllowlistExtension.sol)

---

### Summary

`SwapAllowlistExtension.beforeSwap` gates swaps by checking `allowedSwapper[msg.sender][sender]`, where `msg.sender` is the pool and `sender` is the first argument forwarded by the pool — which is `msg.sender` of the pool's own `swap()` call. When a user routes through `MetricOmmSimpleRouter`, the pool's `msg.sender` is the **router contract**, not the original user. If the pool admin allowlists the router (the natural action to support the standard periphery path), every user on the network can bypass the allowlist by routing through the public router.

---

### Finding Description

**Actor binding in `SwapAllowlistExtension`:** [1](#0-0) 

The hook receives `sender` as its first argument and checks `allowedSwapper[msg.sender][sender]`. `msg.sender` here is the pool (correct), and `sender` is whatever the pool passed as the first argument to the extension call.

**What the pool passes as `sender`:** [2](#0-1) 

The pool passes `msg.sender` — the immediate caller of `pool.swap()` — as `sender` to `_beforeSwap`.

**What the router passes to the pool:** [3](#0-2) 

The router calls `pool.swap(...)` directly. It does not forward the original user's address. Therefore `msg.sender` inside `pool.swap()` is the **router contract address**, not the end user.

**The full call chain:**

```
User → MetricOmmSimpleRouter.exactInputSingle()
         → pool.swap()          [msg.sender = router]
           → _beforeSwap(sender = router, ...)
             → SwapAllowlistExtension.beforeSwap(sender = router, ...)
               → checks allowedSwapper[pool][router]
```

If the pool admin allowlists the router address (to let their allowlisted users trade via the standard periphery), the check `allowedSwapper[pool][router]` returns `true` for **every caller** of the router, because the router is a shared public contract with no user-level access control. [4](#0-3) 

The `ExtensionCalling._beforeSwap` dispatcher faithfully forwards `sender` (= pool's `msg.sender` = router) to every configured extension. There is no mechanism to recover the original EOA.

---

### Impact Explanation

A pool deploying `SwapAllowlistExtension` is a curated pool: LPs deposit expecting to trade only against a specific set of counterparties (e.g., KYC'd users, institutional market makers, or whitelisted protocols). If the router is allowlisted — which is the natural configuration for any pool that wants to support the standard periphery UX — the allowlist is completely nullified. Any unprivileged user can call `MetricOmmSimpleRouter.exactInputSingle` targeting the restricted pool and execute swaps at oracle prices. LPs suffer direct loss because they are forced to trade with unauthorized counterparties they explicitly excluded.

---

### Likelihood Explanation

The pool admin faces an impossible choice with the current design:

- **Do not allowlist the router** → allowlisted users cannot use the standard periphery; they must call the pool directly, which requires custom tooling.
- **Allowlist the router** → the allowlist is bypassed for all users.

A pool admin who wants to support the standard UX will naturally allowlist the router, triggering the bypass. This is a predictable operational decision, making the likelihood medium-to-high for any curated pool that intends to support router-based swaps.

---

### Recommendation

The extension must check the **economically relevant actor** — the original user — not the immediate caller. Two approaches:

1. **Pass the original user through the router**: The router should forward `msg.sender` (the original user) as an explicit `sender` argument to `pool.swap()`, and the pool interface should accept a separate `sender` parameter distinct from `msg.sender`. This is a core interface change.

2. **Check `msg.sender` of the pool call against a router allowlist, then verify the user inside the extension**: The extension could require the router to encode the original user in `extensionData` and verify a signature or trusted-forwarder pattern. This is complex and error-prone.

3. **Simplest safe fix**: Document that the `SwapAllowlistExtension` only supports direct pool calls (not router-mediated swaps), and revert if `msg.sender` (the pool's caller) is a known router. However, this is fragile.

The root fix is to thread the original user's address through the call stack so extensions can gate on the correct actor.

---

### Proof of Concept

```
Setup:
  - Deploy pool with SwapAllowlistExtension configured.
  - Pool admin calls setAllowedToSwap(pool, router, true)
    (to let allowlisted users trade via the standard periphery).
  - Pool admin does NOT call setAllowedToSwap(pool, attacker, true).

Attack:
  - attacker calls MetricOmmSimpleRouter.exactInputSingle({
        pool: restrictedPool,
        zeroForOne: true,
        amountIn: X,
        ...
    })
  - Router calls pool.swap(...) with msg.sender = router.
  - Pool calls _beforeSwap(sender = router, ...).
  - SwapAllowlistExtension checks allowedSwapper[pool][router] → true.
  - Swap executes. Attacker receives tokens at oracle price.
  - Allowlist is bypassed; LPs traded with an unauthorized counterparty.
