### Title
SwapAllowlistExtension Checks Router Address Instead of Actual Swapper, Allowing Any User to Bypass the Curated Pool Allowlist via MetricOmmSimpleRouter - (File: metric-periphery/contracts/extensions/SwapAllowlistExtension.sol)

---

### Summary

`SwapAllowlistExtension.beforeSwap` gates swaps by checking the `sender` argument, which is `msg.sender` of the pool's `swap` call. When a user routes through `MetricOmmSimpleRouter`, `msg.sender` to the pool is the **router contract**, not the actual end-user. The extension therefore checks `allowedSwapper[pool][router]` instead of `allowedSwapper[pool][actualUser]`. A pool admin who allowlists the router to enable router-mediated swaps for permitted users inadvertently opens the gate to every user on the internet.

---

### Finding Description

**Actor binding in `SwapAllowlistExtension`:**

`SwapAllowlistExtension.beforeSwap` receives `sender` as its first argument and checks:

```solidity
if (!allowAllSwappers[msg.sender] && !allowedSwapper[msg.sender][sender]) {
    revert IMetricOmmPoolActions.NotAllowedToSwap();
}
```

Here `msg.sender` is the pool (the extension caller) and `sender` is whoever called `pool.swap`. [1](#0-0) 

**How the pool passes `sender`:**

`MetricOmmPool.swap` passes `msg.sender` as the `sender` argument to `_beforeSwap`:

```solidity
_beforeSwap(msg.sender, recipient, zeroForOne, ...);
``` [2](#0-1) 

**How the router calls the pool:**

`MetricOmmSimpleRouter.exactInputSingle` calls `pool.swap(params.recipient, ...)` directly — making the router itself `msg.sender` to the pool:

```solidity
(int128 amount0Delta, int128 amount1Delta) = IMetricOmmPoolActions(params.pool)
    .swap(params.recipient, params.zeroForOne, ..., params.extensionData);
``` [3](#0-2) 

**The broken invariant:**

The full call chain is:

```
User → router.exactInputSingle(params)
     → pool.swap(recipient=user, ...) [msg.sender = router]
     → _beforeSwap(sender=router, recipient=user, ...)
     → extension.beforeSwap(sender=router, ...)
     → checks allowedSwapper[pool][router]   ← wrong actor
```

The extension was designed to gate individual users. But when the router is the intermediary, the extension sees the router's address as `sender`. The pool admin faces an impossible choice:

1. **Do not allowlist the router** → even legitimately permitted users cannot swap through the router.
2. **Allowlist the router** → every user on the internet can bypass the allowlist by routing through `MetricOmmSimpleRouter`.

There is no configuration that achieves the intended behavior: "allow specific users to swap via the router." [4](#0-3) 

---

### Impact Explanation

A curated pool deploying `SwapAllowlistExtension` to restrict trading to a known set of counterparties (e.g., KYC'd addresses, institutional partners, or whitelisted market makers) loses that protection entirely for any user who routes through `MetricOmmSimpleRouter`. Since the router is the primary user-facing swap interface, the allowlist is effectively nullified for the most common access path. Any disallowed user can execute swaps against the pool's LP assets, directly contradicting the pool's curation policy and potentially causing direct loss of LP funds if the pool was designed to trade only with trusted counterparties.

---

### Likelihood Explanation

The `MetricOmmSimpleRouter` is the standard supported periphery path for swaps. Any user who discovers a curated pool and attempts to swap through the router will succeed if the router is allowlisted. The router address is public and fixed, so no special knowledge or privilege is required. The trigger is a normal `exactInputSingle` or `exactInput` call from any EOA.

---

### Recommendation

The `beforeSwap` hook must gate the **actual end-user**, not the intermediary. Two approaches:

1. **Pass the real user through `extensionData`**: Require the router to encode the original `msg.sender` into `extensionData`, and have the extension decode and verify it. This requires the extension to trust the router's encoding, which introduces its own trust assumptions.

2. **Check `recipient` instead of `sender`**: For swap allowlists, the economically relevant actor is the recipient of the output tokens. The extension could check `allowedSwapper[pool][recipient]` instead of `allowedSwapper[pool][sender]`. The pool already passes `recipient` as the second argument to `beforeSwap`.

3. **Require direct pool calls only**: Document that `SwapAllowlistExtension` is incompatible with router-mediated swaps and enforce this by checking `sender != knownRouter` or by requiring `sender == recipient`.

The most robust fix is option 2 — check `recipient` — since the recipient is the address that economically benefits from the swap and cannot be spoofed by an intermediary:

```solidity
function beforeSwap(address, address recipient, bool, int128, uint128, uint256, uint128, uint128, bytes calldata)
    external view override returns (bytes4) {
    if (!allowAllSwappers[msg.sender] && !allowedSwapper[msg.sender][recipient]) {
        revert IMetricOmmPoolActions.NotAllowedToSwap();
    }
    return IMetricOmmExtensions.beforeSwap.selector;
}
```

### Proof of Concept

**Setup:**
- Deploy a pool with `SwapAllowlistExtension` configured.
- Pool admin allowlists `alice` as a permitted swapper: `extension.setAllowedToSwap(pool, alice, true)`.
- Pool admin also allowlists the router so that `alice` can use it: `extension.setAllowedToSwap(pool, router, true)`.

**Attack:**
- `bob` (not allowlisted) calls `router.exactInputSingle({pool: pool, recipient: bob, ...})`.
- Router calls `pool.swap(recipient=bob, ...)` with `msg.sender = router`.
- Extension checks `allowedSwapper[pool][router]` → `true` → swap proceeds.
- `bob` successfully swaps against the curated pool despite never being allowlisted.

**Verification that the check is on the wrong actor:**

```solidity
// Extension sees sender = router, not bob
function beforeSwap(address sender, ...) external view {
    // sender == address(router), not address(bob)
    // allowedSwapper[pool][router] == true  ← passes
    // allowedSwapper[pool][bob]    == false ← would have blocked
}

contract SwapAllowlistExtension is BaseMetricExtension, ISwapAllowlistExtension {
  mapping(address pool => mapping(address swapper => bool)) public allowedSwapper;
  mapping(address pool => bool) public allowAllSwappers;

  constructor(address factory_) BaseMetricExtension(factory_) {}

  function setAllowedToSwap(address pool_, address swapper, bool allowed) external onlyPoolAdmin(pool_) {
    allowedSwapper[pool_][swapper] = allowed;
    emit AllowedToSwapSet(pool_, swapper, allowed);
  }

  function setAllowAllSwappers(address pool_, bool allowed) external onlyPoolAdmin(pool_) {
    allowAllSwappers[pool_] = allowed;
    emit AllowAllSwappersSet(pool_, allowed);
  }

  function isAllowedToSwap(address pool_, address swapper) external view returns (bool) {
    return allowAllSwappers[pool_] || allowedSwapper[pool_][swapper];
  }

}
