### Title
`SwapAllowlistExtension` Checks Router Address Instead of End-User — Allowlist Fully Bypassed via `MetricOmmSimpleRouter` - (File: metric-periphery/contracts/extensions/SwapAllowlistExtension.sol)

---

### Summary

`SwapAllowlistExtension.beforeSwap` receives `sender` from the pool, which is `msg.sender` of the `pool.swap()` call. When a user routes through `MetricOmmSimpleRouter`, the pool's `msg.sender` is the router contract, not the end user. The allowlist lookup therefore keys on the router address, not the actual trader. If the router is allowlisted to support router-mediated swaps, every user — including those the admin explicitly excluded — can bypass the curated allowlist by routing through the public router.

---

### Finding Description

`SwapAllowlistExtension.beforeSwap` enforces:

```solidity
// metric-periphery/contracts/extensions/SwapAllowlistExtension.sol
function beforeSwap(address sender, address, bool, int128, uint128, uint256, uint128, uint128, bytes calldata)
    external view override returns (bytes4)
{
    if (!allowAllSwappers[msg.sender] && !allowedSwapper[msg.sender][sender]) {
        revert IMetricOmmPoolActions.NotAllowedToSwap();
    }
    return IMetricOmmExtensions.beforeSwap.selector;
}
``` [1](#0-0) 

`msg.sender` here is the pool (the extension caller). `sender` is the first argument forwarded by the pool — it is the `msg.sender` of the `pool.swap()` call. When `MetricOmmSimpleRouter` calls `pool.swap(...)`, the pool's `msg.sender` is the router, so `sender = router`. The allowlist lookup becomes:

```
allowedSwapper[pool][router]   // ← router identity, not the end user
``` [2](#0-1) 

The pool's own documentation confirms the operator-separation design: `msg.sender` pays/initiates but need not equal the economic actor the admin intended to gate. [3](#0-2) 

---

### Impact Explanation

Two fund-impacting outcomes follow from this misbinding:

**Scenario A — Allowlist bypass (High):** The pool admin allowlists the router so that allowlisted users can reach the pool through the supported periphery path. Because `allowedSwapper[pool][router] = true`, every call that arrives through the router passes the check regardless of who the end user is. Non-allowlisted users — those the admin explicitly excluded — can swap freely by routing through `MetricOmmSimpleRouter`. The curated pool's entire access-control policy is nullified.

**Scenario B — Allowlisted users locked out (Medium):** The admin allowlists individual user addresses but does not allowlist the router. Allowlisted users who call through the router are rejected (`sender = router`, not in the allowlist). They can only interact by calling `pool.swap()` directly, which may not be the intended UX and breaks the supported periphery path for legitimate users.

Scenario A is the primary contest-relevant impact: a curated pool's swap allowlist is completely bypassed, allowing unauthorized traders to execute swaps and extract value from a pool that was designed to be restricted. [4](#0-3) 

---

### Likelihood Explanation

- `MetricOmmSimpleRouter` is a public, permissionless periphery contract — any user can call it.
- The router is the standard, documented entry point for swaps; users are expected to route through it.
- No special setup is required: the attacker simply calls the router's `exact*` functions targeting the curated pool.
- The admin enabling router-mediated swaps (Scenario A) is the natural operational step to make the pool usable through the periphery, making the bypass a near-certain consequence of normal deployment. [1](#0-0) 

---

### Recommendation

The extension must resolve the actual end-user identity rather than accepting the immediate pool caller. Two sound approaches:

1. **Pass end-user identity explicitly:** Have the pool forward the original `tx.origin` or a user-supplied identity field through `extensionData`, and have the extension decode and check that value. This requires a protocol-level convention.

2. **Check `sender` against a router registry:** The extension can maintain a mapping of trusted routers. When `sender` is a known router, decode the real user from `extensionData` (which the router must populate). When `sender` is not a router, treat `sender` as the user directly.

3. **Gate at the router level:** Require the router to verify the caller is allowlisted before forwarding to the pool, and remove the extension-level check. This centralizes enforcement but removes the composable guard.

The simplest correct fix is option 2: the extension distinguishes between direct callers and router-mediated calls and resolves the true economic actor in both cases.

### Proof of Concept

```
Setup:
  - Deploy pool with SwapAllowlistExtension configured
  - Admin calls setAllowedToSwap(pool, router, true)
    (necessary so allowlisted users can use the router)
  - Admin does NOT call setAllowedToSwap(pool, attacker, true)

Attack:
  1. attacker calls MetricOmmSimpleRouter.exactInput(...)
     targeting the curated pool
  2. Router calls pool.swap(recipient=attacker, ...)
     → pool's msg.sender = router
  3. Pool calls extension.beforeSwap(sender=router, ...)
  4. Extension checks: allowedSwapper[pool][router] == true  ✓
  5. Swap executes — attacker bypasses the allowlist entirely

Result:
  - attacker swaps on a pool they were explicitly excluded from
  - Curated pool's access policy is nullified
``` [2](#0-1) [5](#0-4)

contract SwapAllowlistExtension is BaseMetricExtension, ISwapAllowlistExtension {
  mapping(address pool => mapping(address swapper => bool)) public allowedSwapper;
  mapping(address pool => bool) public allowAllSwappers;

  constructor(address factory_) BaseMetricExtension(factory_) {}

  function setAllowedToSwap(address pool_, address swapper, bool allowed) external onlyPoolAdmin(pool_) {
    allowedSwapper[pool_][swapper] = allowed;
    emit AllowedToSwapSet(pool_, swapper, allowed);
  }

  function setAllowAllSwappers(address pool_, bool allowed) external onlyPoolAdmin(pool_) {
    allowAllSwappers[pool_] = allowed;
    emit AllowAllSwappersSet(pool_, allowed);
  }

  function isAllowedToSwap(address pool_, address swapper) external view returns (bool) {
