### Title
`SwapAllowlistExtension` Checks Router Address Instead of Actual User, Enabling Allowlist Bypass via `MetricOmmSimpleRouter` - (File: `metric-periphery/contracts/extensions/SwapAllowlistExtension.sol`)

---

### Summary

`SwapAllowlistExtension.beforeSwap` gates swaps by checking the `sender` argument passed by the pool, which is `msg.sender` of the pool's `swap` call. When a user routes through `MetricOmmSimpleRouter`, the pool's `msg.sender` is the **router contract**, not the actual user. The extension therefore checks whether the router is allowlisted rather than whether the real user is allowlisted. If the pool admin allowlists the router (a natural operational choice to enable the standard periphery path), any unprivileged user can bypass the per-user allowlist by routing through the router.

---

### Finding Description

**Call chain — direct swap (correct):**

```
user → pool.swap(...)
         msg.sender = user
         _beforeSwap(sender=user, ...)
         extension checks allowedSwapper[pool][user]  ✓
```

**Call chain — router-mediated swap (broken):**

```
user → router.exactInputSingle(...)
         router → pool.swap(...)
                    msg.sender = router
                    _beforeSwap(sender=router, ...)
                    extension checks allowedSwapper[pool][router]  ✗
```

In `MetricOmmPool.swap`, the pool passes its own `msg.sender` as the `sender` argument to `_beforeSwap`: [1](#0-0) 

`ExtensionCalling._beforeSwap` forwards that value unchanged to every configured extension: [2](#0-1) 

`SwapAllowlistExtension.beforeSwap` then checks `allowedSwapper[msg.sender][sender]`, where `msg.sender` is the pool and `sender` is the first argument — the router address when the user entered through the router: [3](#0-2) 

`MetricOmmSimpleRouter.exactInputSingle` calls `pool.swap` directly, making the router the pool's `msg.sender`. The actual user (`msg.sender` of the router call) is stored only in the transient callback context for payment settlement and is never forwarded to the pool or to any extension: [4](#0-3) 

The same wrong-actor binding applies to `exactInput`, `exactOutputSingle`, and `exactOutput` — all router entry points call `pool.swap` with `msg.sender = router`. [5](#0-4) 

---

### Impact Explanation

**Bypass path (High — direct loss of curation policy):** A pool admin who allowlists the router address (to let users reach the pool through the standard periphery path) inadvertently opens the allowlist to every user on the network. Any address can call `router.exactInputSingle` and the extension will pass because `allowedSwapper[pool][router] == true`. The curated pool's per-user gate is completely defeated; non-KYC'd or otherwise excluded users can trade freely.

**Broken functionality path (Medium — unusable core flow):** If the admin does not allowlist the router, every legitimately allowlisted user who tries to swap through the router is blocked (`NotAllowedToSwap`). The standard periphery path is rendered unusable for all allowlisted users, breaking the core swap flow for the pool's intended participants.

Both outcomes are direct consequences of the same root cause: the extension is keyed on the wrong actor.

---

### Likelihood Explanation

The `MetricOmmSimpleRouter` is the primary user-facing entry point for swaps. Pool admins who configure a `SwapAllowlistExtension` will naturally need to decide whether to allowlist the router. Allowlisting it is the only way to let their allowlisted users reach the pool through the router, making the bypass scenario a predictable operational outcome. The attack requires no special privilege — any address can call the public router.

---

### Recommendation

The pool must forward the original user identity to extensions. Two complementary fixes:

1. **Pass the original initiator through the router.** Add an `initiator` field to the swap call (analogous to the Connext fix of adding `initiator` to `InvariantTransactionData`) so the pool can forward the real user to extensions instead of `msg.sender`.

2. **Short-term mitigation.** Until the pool interface is updated, the `SwapAllowlistExtension` should not be used with pools that are also reachable through the router, or the router must not be allowlisted.

---

### Proof of Concept

```solidity
// Setup
SwapAllowlistExtension ext = new SwapAllowlistExtension(factory);
// Admin allowlists the router so that allowlisted users can reach the pool
ext.setAllowedToSwap(pool, address(router), true);
// Alice is NOT individually allowlisted
// assert(!ext.isAllowedToSwap(pool, alice));

// Attack: Alice bypasses the allowlist by going through the router
vm.prank(alice);
router.exactInputSingle(ExactInputSingleParams({
    pool:            pool,
    recipient:       alice,
    tokenIn:         token0,
    zeroForOne:      true,
    amountIn:        1e18,
    amountOutMinimum: 0,
    priceLimitX64:   0,
    deadline:        block.timestamp,
    extensionData:   ""
}));
// ✓ swap succeeds — extension saw sender=router, which is allowlisted
// Alice traded on a curated pool she was never authorized to access
```

The extension receives `sender = address(router)`, finds `allowedSwapper[pool][router] == true`, and returns the success selector. Alice's swap executes in full despite never being individually allowlisted. [6](#0-5) [7](#0-6) [8](#0-7)

    _checkDeadline(params.deadline);
    _validatePath(params.tokens, params.pools, params.extensionDatas);

    uint256 last = params.pools.length - 1;
    int128 amount = MetricOmmSwapInputs.asAmountSpecifiedIn(params.amountIn);

    for (uint256 i = 0; i <= last; i++) {
      address pool = params.pools[i];
      bool zeroForOne = MetricOmmSwapPath.resolveZeroForOneBitmap(params.zeroForOneBitMap, i);

      _setNextCallbackContext(pool, CALLBACK_MODE_JUST_PAY, i == 0 ? msg.sender : address(this), params.tokens[i]);
      (int128 amount0Delta, int128 amount1Delta) = IMetricOmmPoolActions(pool)
        .swap(
          i == last ? params.recipient : address(this),
          zeroForOne,
          amount,
          MetricOmmSwapPath.openLimit(zeroForOne),
          "",
          params.extensionDatas[i]
        );

      int128 amountInActual = MetricOmmSwapResults.extractAmountIn(zeroForOne, amount0Delta, amount1Delta);
      if (amountInActual < amount) revert InvalidInputAmountAtHop(uint8(i), amountInActual, amount);

      amount = MetricOmmSwapResults.extractAmountOut(zeroForOne, amount0Delta, amount1Delta);
    }

    if (amount <= 0) revert InvalidSwapDeltas();
    amountOut = MetricOmmSwapInputs.int128ToUint128(amount);
    if (amountOut < params.amountOutMinimum) revert InsufficientOutput(amountOut, params.amountOutMinimum);

    _clearExpectedCallbackPool();
  }
