### Title
SwapAllowlistExtension gates the router address instead of the actual user, enabling allowlist bypass via MetricOmmSimpleRouter — (`metric-periphery/contracts/extensions/SwapAllowlistExtension.sol`)

---

### Summary

`SwapAllowlistExtension.beforeSwap` checks the direct caller of `pool.swap()` as the gated identity. When users route through `MetricOmmSimpleRouter`, the router becomes that direct caller. If the pool admin allowlists the router (the only way to enable router-mediated swaps on an allowlisted pool), every user — including non-allowlisted ones — can bypass the individual swapper gate.

---

### Finding Description

`MetricOmmPool.swap` passes `msg.sender` as the `sender` argument to `_beforeSwap`: [1](#0-0) 

`ExtensionCalling._beforeSwap` forwards that value verbatim to the extension: [2](#0-1) 

`SwapAllowlistExtension.beforeSwap` then checks `allowedSwapper[msg.sender][sender]`, where `msg.sender` is the pool and `sender` is whoever called `pool.swap()`: [3](#0-2) 

When a user calls `MetricOmmSimpleRouter.exactInputSingle`, the router is the direct caller of `pool.swap()`: [4](#0-3) 

So `sender` received by the extension is the **router address**, not the end user. This creates a binary, all-or-nothing situation:

| Router allowlisted? | Effect |
|---|---|
| **No** | All router-mediated swaps blocked, even for individually allowlisted users — core swap flow broken |
| **Yes** | Every user can bypass the individual allowlist by routing through the router |

The extension has no mechanism to recover the original user's identity from `extensionData` or any other channel; the router does not forward it.

The broken invariant is: *only addresses in `allowedSwapper[pool]` may swap on the pool*. When the router is allowlisted, this invariant is violated for every non-allowlisted user who routes through it.

---

### Impact Explanation

A pool operator who deploys `SwapAllowlistExtension` to restrict swaps to a known set of addresses (KYC, private LP pool, rate-limited access) and then allowlists the router to enable standard UX inadvertently opens the pool to all users. Any non-allowlisted address can call `MetricOmmSimpleRouter.exactInputSingle` and execute a swap that the allowlist was supposed to block. This is a direct bypass of a configured access-control guard with fund-impacting consequences (unauthorized swaps drain pool reserves at oracle price).

---

### Likelihood Explanation

The trigger is a standard, unprivileged public call to the router. The prerequisite — the pool admin allowlisting the router — is a natural and expected configuration step for any pool that wants to support the protocol's own periphery. The pool admin has no reason to suspect that allowlisting the router opens the gate to all users, because the extension's interface implies per-address control. Likelihood is **medium**: the misconfiguration is easy to make and the exploit path is trivially reachable by any user.

---

### Recommendation

The extension must gate the **economically relevant actor**, not the intermediary. Two viable approaches:

1. **Pass the real user through `extensionData`**: The router encodes `msg.sender` into `extensionData`; the extension decodes and checks it. This requires a coordinated change to the router and the extension.

2. **Check `sender` only for direct pool calls; require `extensionData` attestation for router calls**: The extension detects whether `sender` is a known router and, if so, requires a signed or encoded real-user identity in `extensionData`.

The simplest correct fix is for `MetricOmmSimpleRouter` to encode the original `msg.sender` into `extensionData` and for `SwapAllowlistExtension` to decode and check that value when `sender` is a recognized router address.

---

### Proof of Concept

```
1. Pool P is deployed with SwapAllowlistExtension E configured.
2. Pool admin calls E.setAllowedToSwap(P, router, true)
   — necessary to allow any router-mediated swap.
3. Alice (not in allowedSwapper[P]) calls:
     router.exactInputSingle({pool: P, recipient: Alice, ...})
4. Router executes: P.swap(Alice, zeroForOne, amount, limit, "", extensionData)
   — msg.sender of this call is the router.
5. Pool calls E.beforeSwap(sender=router, ...)
6. Extension evaluates: allowedSwapper[P][router] == true  ✓
7. Swap proceeds. Alice receives output tokens from a pool she was not authorized to access.
```

Alice's address is never checked. The allowlist is fully bypassed.
