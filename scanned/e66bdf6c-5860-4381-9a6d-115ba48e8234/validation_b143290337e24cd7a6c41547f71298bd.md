### Title
`SwapAllowlistExtension` Allowlist Bypassed for All Router-Mediated Swaps — (`metric-periphery/contracts/extensions/SwapAllowlistExtension.sol`)

---

### Summary

`SwapAllowlistExtension.beforeSwap` gates swaps by checking the `sender` argument passed by the pool. Because `MetricOmmPool.swap` always passes `msg.sender` (the immediate caller) as `sender`, any user who routes through `MetricOmmSimpleRouter` causes the extension to check the **router's address** instead of the **user's address**. If the router is allowlisted (the only way to let legitimate users swap through it), the gate is open to every address on the network.

---

### Finding Description

`MetricOmmPool.swap` passes `msg.sender` as the `sender` argument to `_beforeSwap`: [1](#0-0) 

`ExtensionCalling._beforeSwap` forwards that value unchanged to every configured extension: [2](#0-1) 

`SwapAllowlistExtension.beforeSwap` then checks `allowedSwapper[msg.sender][sender]`, where `msg.sender` is the pool and `sender` is whoever called `pool.swap`: [3](#0-2) 

When a user calls `MetricOmmSimpleRouter.exactInputSingle`, the router calls `pool.swap(...)` directly: [4](#0-3) 

The call chain is:

```
User → Router.exactInputSingle()
         └─ Pool.swap()   [msg.sender = Router]
               └─ Extension.beforeSwap(sender = Router, ...)
                     └─ checks allowedSwapper[pool][Router]
                                                  ^^^^^^
                                          NOT the user's address
```

The pool admin intends to allowlist specific user addresses. But the extension sees only the router's address. Two outcomes:

| Router allowlisted? | Effect |
|---|---|
| No | Legitimate allowlisted users **cannot** swap through the router (DoS) |
| Yes | **Every** address on the network can bypass the allowlist via the router |

The second case is the critical one. A pool admin who allowlists the router to enable normal UX inadvertently opens the gate to all callers. The same bypass applies to `exactInput`, `exactOutputSingle`, and `exactOutput`.

---

### Impact Explanation

**Severity: Medium**

A pool configured with `SwapAllowlistExtension` (e.g., for KYC/compliance, institutional-only access, or rate-limited liquidity) can be freely accessed by any address through the standard periphery router. The allowlist guard — the only on-chain enforcement of the access policy — is completely ineffective for router-mediated swaps. Unauthorized users can execute swaps against restricted liquidity, violating the pool admin's intended access control and potentially draining LP assets at prices the pool was not designed to offer to arbitrary counterparties.

---

### Likelihood Explanation

**Likelihood: High**

`MetricOmmSimpleRouter` is the standard, documented user-facing entry point. Any user who follows normal UX (or reads the periphery interface) will route through it. The bypass requires no special knowledge, no privileged role, and no unusual token behavior — just calling the public router.

---

### Recommendation

The extension must verify the **economic actor** (the end user), not the immediate `msg.sender` of `pool.swap`. Two options:

1. **Pool-side fix**: Have `MetricOmmPool.swap` accept an explicit `swapper` parameter (separate from `msg.sender`) and pass it as `sender` to extensions. The router would forward `msg.sender` (the user) in this field.

2. **Extension-side fix**: `SwapAllowlistExtension.beforeSwap` should also check `allowedSwapper[msg.sender][msg.sender_of_the_router]`. This is not directly available, so option 1 is cleaner.

The simplest safe interim fix: document that pools using `SwapAllowlistExtension` **must not** allowlist the router address, and must require users to call `pool.swap` directly. However, this breaks normal UX and is not a real fix.

---

### Proof of Concept

```
Setup:
  - Deploy pool with SwapAllowlistExtension
  - Pool admin allowlists only `alice` via setAllowedToSwap(pool, alice, true)
  - Pool admin also allowlists the router (required for alice to use the router)

Attack:
  - bob (not allowlisted) calls MetricOmmSimpleRouter.exactInputSingle(pool, ...)
  - Router calls pool.swap(...) → msg.sender = router
  - Extension checks allowedSwapper[pool][router] → true (router is allowlisted)
  - Bob's swap succeeds despite not being on the allowlist

Result:
  - The allowlist is bypassed; bob swaps freely in a restricted pool
  - If the pool admin does NOT allowlist the router, alice cannot use the router either
    (the extension checks allowedSwapper[pool][router] → false → revert)
``` [5](#0-4) [6](#0-5) [7](#0-6)
