The vulnerability is real. Here is the complete trace:

**Call chain:**
1. Attacker EOA → `MetricOmmSimpleRouter.exactInputSingle()` (`msg.sender` = attacker)
2. Router → `MetricOmmPool.swap(recipient, ...)` (`msg.sender` to pool = **router address**)
3. Pool → `_beforeSwap(msg.sender, ...)` — passes **router address** as `sender`
4. Extension → `SwapAllowlistExtension.beforeSwap(sender=router, ...)` — checks `allowedSwapper[msg.sender][sender]` = `allowedSwapper[pool][router]`

The check never sees the original EOA.

---

### Title
Router Address Substitution Bypasses Per-User Swap Allowlist in `SwapAllowlistExtension` — (`metric-periphery/contracts/extensions/SwapAllowlistExtension.sol`)

### Summary
`SwapAllowlistExtension.beforeSwap` checks the `sender` argument against the allowlist, but `sender` is `msg.sender` of `MetricOmmPool.swap()` — which is the router contract, not the originating EOA. Any user who routes through an allowlisted router bypasses the per-user swap allowlist entirely.

### Finding Description
`MetricOmmPool.swap()` passes `msg.sender` as the `sender` argument to `_beforeSwap`: [1](#0-0) 

`ExtensionCalling._beforeSwap` forwards that value unchanged as the first argument to `IMetricOmmExtensions.beforeSwap`: [2](#0-1) 

`SwapAllowlistExtension.beforeSwap` then checks `allowedSwapper[msg.sender][sender]`, where `msg.sender` is the pool and `sender` is the router: [3](#0-2) 

When `MetricOmmSimpleRouter.exactInputSingle` calls `pool.swap(...)`, the router is `msg.sender` to the pool: [4](#0-3) 

So the allowlist check resolves to `allowedSwapper[pool][router]`. If the router is allowlisted, every user of that router passes the check regardless of their own allowlist status.

### Impact Explanation
Pools using `SwapAllowlistExtension` to restrict trading to a curated set of counterparties (e.g., KYC'd addresses, institutional partners) are fully bypassed. Any EOA can route through the allowlisted router and execute swaps, draining LP assets at will. This breaks the core invariant of the extension ("Gates `swap` by swapper address, per pool") and constitutes direct loss of LP principal above Sherlock thresholds.

### Likelihood Explanation
The `MetricOmmSimpleRouter` is the canonical swap interface for the protocol. Pool admins who allowlist the router (a natural and expected configuration) unknowingly open the pool to all users. No special privileges or unusual conditions are required — any EOA can exploit this immediately.

### Recommendation
`SwapAllowlistExtension.beforeSwap` should check the `recipient` parameter or require the pool to pass the original transaction originator. The cleanest fix is to have the pool pass `tx.origin` or require routers to forward the end-user address via `extensionData`. Alternatively, the extension should document that it only gates the direct caller of `pool.swap()` and is incompatible with router-mediated flows, and the router should be explicitly excluded from allowlisting.
