### Title
SwapAllowlistExtension Checks Router Address Instead of Original Swapper, Enabling Complete Allowlist Bypass — (File: metric-periphery/contracts/extensions/SwapAllowlistExtension.sol)

---

### Summary

`SwapAllowlistExtension.beforeSwap` gates swaps by checking the `sender` argument, which the pool sets to `msg.sender` of the `pool.swap()` call. When a user routes through `MetricOmmSimpleRouter`, `msg.sender` to the pool is the **router address**, not the original user. The extension therefore checks whether the router is allowlisted, not the actual trader. If the pool admin allowlists the router to support periphery-mediated swaps, every unprivileged user can bypass the allowlist entirely by routing through the router.

---

### Finding Description

`MetricOmmPool.swap` passes its own `msg.sender` as the `sender` argument to `_beforeSwap`: [1](#0-0) 

`ExtensionCalling._beforeSwap` forwards that value unchanged to every configured extension: [2](#0-1) 

`SwapAllowlistExtension.beforeSwap` then checks `allowedSwapper[msg.sender][sender]`, where `msg.sender` is the pool and `sender` is whoever called `pool.swap()`: [3](#0-2) 

`MetricOmmSimpleRouter.exactInputSingle` calls `pool.swap(...)` directly, making the router the `msg.sender` seen by the pool: [4](#0-3) 

The same pattern applies to `exactInput`, `exactOutputSingle`, and `exactOutput`: [5](#0-4) [6](#0-5) 

The router never passes the original `msg.sender` to the pool; the pool has no mechanism to receive it. Consequently, the extension always evaluates the router's address against the allowlist, not the actual trader's address.

This produces two mutually exclusive failure modes:

| Router allowlist state | Effect |
|---|---|
| Router **not** allowlisted | Every allowlisted user who calls through the router is blocked — broken core functionality |
| Router **allowlisted** (admin enables router support) | Every unprivileged user bypasses the allowlist by routing through the router — complete policy bypass |

There is no configuration that simultaneously allows specific users to use the router while blocking others.

---

### Impact Explanation

**Medium — broken core extension functionality with direct LP exposure.**

A pool deploying `SwapAllowlistExtension` to create a curated trading environment (e.g., KYC-gated, institutional-only, or low-toxicity flow) cannot enforce that policy for router-mediated swaps. If the admin allowlists the router to support the standard periphery path, any unprivileged user can trade on the pool. Unrestricted toxic or arbitrage flow against a pool designed for curated flow causes direct LP value leakage through adverse selection — a fund-impacting consequence above the Medium threshold.

---

### Likelihood Explanation

**Medium.** `MetricOmmSimpleRouter` is the canonical periphery entry point. Pool admins who deploy a swap-allowlisted pool and want their allowlisted users to access the standard router will allowlist the router address. This is the natural, expected configuration and is reachable by any unprivileged user with a single public call.

---

### Recommendation

The router must propagate the original caller's identity to the pool so the extension can gate the economically relevant actor. Two viable approaches:

1. **Extension-data convention**: The router encodes `msg.sender` into `extensionData` for each hop, and `SwapAllowlistExtension` decodes and checks that field instead of (or in addition to) `sender`.
2. **Pool-level originator field**: Add an optional `originator` parameter to `pool.swap()` that the router populates with `msg.sender`; the pool forwards it to extensions alongside `sender`.

Either approach must be enforced consistently across all router entry points (`exactInputSingle`, `exactInput`, `exactOutputSingle`, `exactOutput`) and the extension must validate that the encoded originator matches the actual payer context to prevent spoofing.

---

### Proof of Concept

```
1. Deploy pool with SwapAllowlistExtension configured.
2. Pool admin calls setAllowedToSwap(pool, alice, true)   // alice is the intended gated user
3. Pool admin calls setAllowedToSwap(pool, router, true)  // admin enables router support
4. Bob (not allowlisted) calls:
       router.exactInputSingle({pool: pool, ...})
5. Router calls pool.swap(recipient, ...) with msg.sender = router
6. Pool calls _beforeSwap(sender=router, ...)
7. SwapAllowlistExtension checks allowedSwapper[pool][router] → true
8. Bob's swap executes — allowlist completely bypassed
```
