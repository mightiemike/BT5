### Title
SwapAllowlistExtension Gates the Router Address Instead of the Actual User, Allowing Any User to Bypass the Swap Allowlist via the Router — (`metric-periphery/contracts/extensions/SwapAllowlistExtension.sol`)

---

### Summary

`SwapAllowlistExtension.beforeSwap` checks the `sender` argument, which the pool sets to `msg.sender` of `pool.swap()`. When a user routes through `MetricOmmSimpleRouter`, the pool's `msg.sender` is the **router contract**, not the actual user. A pool admin who allowlists the router so that legitimate users can trade through it simultaneously opens the gate to every unprivileged address, completely defeating the allowlist.

---

### Finding Description

`MetricOmmPool.swap` passes `msg.sender` as the `sender` argument to every extension hook: [1](#0-0) 

`SwapAllowlistExtension.beforeSwap` then checks that `sender` against the per-pool allowlist: [2](#0-1) 

Every public entry point in `MetricOmmSimpleRouter` — `exactInputSingle`, `exactInput`, `exactOutputSingle`, and `exactOutput` — calls `pool.swap()` directly from the router contract: [3](#0-2) [4](#0-3) 

Because the router is always the immediate caller of `pool.swap()`, the `sender` seen by `beforeSwap` is always the **router address**, never the end user. The allowlist therefore evaluates `allowedSwapper[pool][router]`, not `allowedSwapper[pool][user]`.

This creates an irresolvable dilemma for any pool admin who wants to:
1. Restrict swaps to a specific set of users, **and**
2. Allow those users to trade through the public router.

To satisfy (2) the admin must execute `setAllowedToSwap(pool, router, true)`. Once the router is allowlisted, condition (1) is void: any address can call `router.exactInputSingle()` and the extension will pass because `allowedSwapper[pool][router] == true`.

The only alternative — not allowlisting the router — means even legitimately allowlisted users cannot trade through the router, breaking the intended user flow.

This is the direct EVM analog of the Solana bug in the external report: just as `list_state` (the wrong account) was used as the payer instead of the user-provided account, here the **router** (the wrong identity) is checked by the guard instead of the actual economic actor.

---

### Impact Explanation

Any pool that deploys `SwapAllowlistExtension` to gate access to a curated set of traders is rendered ineffective the moment the router is allowlisted. An unprivileged, non-allowlisted address can execute a full swap against the pool — receiving output tokens and draining pool liquidity — simply by calling the public router. This constitutes a broken core pool functionality and an admin-boundary break where an unprivileged path bypasses a configured access control.

---

### Likelihood Explanation

- The router is a public, permissionless contract.
- No special role, token balance, or prior interaction is required.
- The bypass is a single standard router call (`exactInputSingle`).
- Any pool that uses `SwapAllowlistExtension` for access control and also needs router compatibility is affected by construction.

---

### Recommendation

The pool should pass the **original initiating user** as `sender` to extension hooks, not the immediate `msg.sender`. Two concrete approaches:

1. **Router-side**: Store the original `msg.sender` in transient storage alongside the payer (already done for payment) and expose it via a `getOriginalSender()` view. The pool reads this during the swap and passes it as `sender` to extensions when the immediate caller is a known periphery contract.

2. **Extension-side**: Change `SwapAllowlistExtension.beforeSwap` to accept an explicit `originalSender` field from `extensionData`, and require the router to encode the real user address there. The extension then validates that field rather than the `sender` argument.

Option 1 is preferable because it does not require callers to self-report their identity.

---

### Proof of Concept

```
Setup:
  pool configured with SwapAllowlistExtension
  allowedSwapper[pool][alice]  = true   // intended: only alice may swap
  allowedSwapper[pool][router] = true   // needed: so alice can use the router

Attack:
  vm.prank(eve);                        // eve is NOT allowlisted
  router.exactInputSingle(ExactInputSingleParams({
      pool:          address(pool),
      tokenIn:       token0,
      tokenOut:      token1,
      zeroForOne:    true,
      amountIn:      1_000e18,
      amountOutMinimum: 0,
      recipient:     eve,
      deadline:      block.timestamp + 1,
      priceLimitX64: 0,
      extensionData: ""
  }));
  // pool.swap() is called with msg.sender = router
  // beforeSwap receives sender = router
  // allowedSwapper[pool][router] == true → guard passes
  // eve receives token1 output; allowlist is bypassed
