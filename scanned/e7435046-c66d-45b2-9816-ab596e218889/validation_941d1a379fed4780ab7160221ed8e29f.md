### Title
`SwapAllowlistExtension` checks the router address as the swapper instead of the actual user, allowing any unprivileged caller to bypass the per-pool allowlist via `MetricOmmSimpleRouter` — (`metric-periphery/contracts/extensions/SwapAllowlistExtension.sol`)

---

### Summary

`SwapAllowlistExtension.beforeSwap` gates swaps by checking the `sender` argument, which the pool sets to `msg.sender` of the `pool.swap(...)` call. When a user routes through `MetricOmmSimpleRouter`, `msg.sender` of the pool call is the **router contract**, not the actual user. If the pool admin allowlists the router (the natural step to let allowlisted users reach the pool through the supported periphery), every unprivileged user can bypass the allowlist by routing through the same contract.

---

### Finding Description

`MetricOmmPool.swap` passes `msg.sender` as the `sender` argument to `_beforeSwap`: [1](#0-0) 

`ExtensionCalling._beforeSwap` forwards that value unchanged as the first positional argument to every configured extension: [2](#0-1) 

`SwapAllowlistExtension.beforeSwap` then checks `allowedSwapper[msg.sender][sender]`, where `msg.sender` is the pool and `sender` is the value forwarded above: [3](#0-2) 

When a user calls `MetricOmmSimpleRouter.exactInputSingle(...)`, the router calls `pool.swap(params.recipient, ...)` with `msg.sender = router`: [4](#0-3) 

The extension therefore evaluates `allowedSwapper[pool][router]`, not `allowedSwapper[pool][actualUser]`. The pool admin must allowlist the router to let any allowlisted user reach the pool through the supported periphery path. Once the router is allowlisted, the check passes for **every** caller of the router, regardless of whether that caller is on the allowlist.

The `DepositAllowlistExtension` does not share this flaw: it checks `owner` (the address that receives LP shares), which the liquidity adder sets to the actual user, so the identity checked matches the economically relevant actor. [5](#0-4) 

---

### Impact Explanation

A pool deploying `SwapAllowlistExtension` intends to restrict trading to a curated set of addresses (e.g., KYC'd market makers, whitelisted counterparties). Once the router is allowlisted — the only way to let allowlisted users use the supported periphery — the gate is open to every unprivileged user. Those users can extract value from LP positions that were priced and sized under the assumption of a restricted, known counterparty set. This is a direct loss of LP principal and a complete failure of the core protection the extension was deployed to provide.

---

### Likelihood Explanation

The pool admin must allowlist the router for the bypass to be reachable. Allowlisting the router is the expected, non-malicious configuration step: without it, even allowlisted users cannot use the protocol's own periphery contract to trade on the pool. Any pool that deploys `SwapAllowlistExtension` and also wants its allowlisted users to reach it through `MetricOmmSimpleRouter` will be in the vulnerable state. The attacker's action — calling a public router function — requires no privilege.

---

### Recommendation

The extension must check the identity of the **economic actor**, not the immediate caller of `pool.swap`. Two viable approaches:

1. **Extension-data attestation**: require the router to encode the originating user's address in `extensionData` and verify it in `beforeSwap`. The extension should reject calls where `extensionData` is absent or the attested address is not allowlisted.
2. **Sender-only direct-call enforcement**: document that `SwapAllowlistExtension` is incompatible with router-mediated flows and revert in `beforeSwap` when `sender` is a known router address, forcing allowlisted users to call the pool directly.

Option 1 is preferable because it preserves router usability for allowlisted users.

---

### Proof of Concept

```
Setup
─────
1. Pool admin deploys MetricOmmPool with SwapAllowlistExtension.
2. Pool admin calls setAllowedToSwap(pool, alice, true)   // alice is the intended user
3. Pool admin calls setAllowedToSwap(pool, router, true)  // required so alice can use the router

Attack
──────
4. bob (not allowlisted) calls:
       MetricOmmSimpleRouter.exactInputSingle({
           pool:      pool,
           recipient: bob,
           ...
       })

5. Router executes: pool.swap(bob, zeroForOne, amount, limit, "", extensionData)
   → msg.sender seen by pool = router

6. Pool calls _beforeSwap(sender=router, recipient=bob, ...)

7. SwapAllowlistExtension.beforeSwap checks:
       allowAllSwappers[pool]          → false
       allowedSwapper[pool][router]    → true   ← router is allowlisted
   → no revert

8. Swap settles in full for bob despite bob never being allowlisted.
   The allowlist invariant is broken; bob extracts value from LP positions
   sized for a curated counterparty set.
``` [3](#0-2) [6](#0-5) [7](#0-6)
