### Title
`SwapAllowlistExtension` Checks Router Address Instead of Actual User, Allowing Any User to Bypass the Per-Pool Swap Allowlist — (`File: metric-periphery/contracts/extensions/SwapAllowlistExtension.sol`)

---

### Summary

`SwapAllowlistExtension.beforeSwap` gates swaps by checking the `sender` argument, which is `msg.sender` of `MetricOmmPool.swap()`. When a user routes through `MetricOmmSimpleRouter`, `msg.sender` to the pool is the router contract, not the actual user. The extension therefore checks whether the **router** is allowlisted, not the individual user. Any user can bypass a per-user swap allowlist on a curated pool by routing through the public router.

---

### Finding Description

`MetricOmmPool.swap()` passes `msg.sender` as the `sender` argument to `_beforeSwap`, which forwards it verbatim to every configured extension: [1](#0-0) 

`ExtensionCalling._beforeSwap` encodes and dispatches that same `sender` to each extension in order: [2](#0-1) 

`SwapAllowlistExtension.beforeSwap` then enforces the allowlist using `msg.sender` (the pool) as the pool key and the received `sender` argument as the identity to check: [3](#0-2) 

When a user calls `MetricOmmSimpleRouter`, the router calls `pool.swap()`, making `msg.sender` to the pool the **router address**. The extension therefore evaluates `allowedSwapper[pool][router]` — not `allowedSwapper[pool][user]`. The actual user's identity is never checked.

The pool admin faces an impossible choice:
- **Do not allowlist the router** → all router-mediated swaps revert with `NotAllowedToSwap`, even for individually allowlisted users (DoS on the supported periphery path).
- **Allowlist the router** → every user on the network can bypass the per-user allowlist by routing through the public router.

The `SwapAllowlistExtension` is explicitly documented and deployed as a production guard for curated pools: [4](#0-3) 

---

### Impact Explanation

A pool configured with `SwapAllowlistExtension` is a curated pool — only specific addresses are intended to trade. Any unprivileged user can bypass this restriction entirely by calling `MetricOmmSimpleRouter` instead of `pool.swap()` directly. This is a direct allowlist-bypass with fund-impacting consequences: unauthorized users can drain liquidity from restricted pools (e.g., institutional, KYC-gated, or partner-only pools) at oracle-derived prices, causing direct loss of LP principal. The bypass requires no special privilege and is reachable through the standard supported periphery path.

---

### Likelihood Explanation

- `MetricOmmSimpleRouter` is a public, permissionless contract — any user can call it.
- The bypass requires only a single transaction through the router; no setup, no flash loan, no admin cooperation.
- Pool admins who configure `SwapAllowlistExtension` and also allowlist the router (the natural operational choice to support the periphery) unknowingly open the pool to all users.
- The `SwapAllowlistExtension` is a production extension explicitly listed in the periphery: [5](#0-4) 

---

### Recommendation

The extension must check the **economically relevant actor** — the end user — not the intermediary router. Two sound approaches:

1. **Pass the real user through `extensionData`**: The router encodes the original `msg.sender` into `extensionData`; the extension decodes and checks it. This requires a trusted router convention.
2. **Check `sender` only when `msg.sender` (pool) is the direct caller**: If `sender == tx.origin` is not acceptable, the cleanest fix is to have the pool pass a separate `originator` field that the router populates, and have the extension check that field.

The `DepositAllowlistExtension` has the same structural risk for the `owner`/`sender` split on the liquidity-adder path and should be audited with the same lens.

---

### Proof of Concept

```
Setup:
  pool P configured with SwapAllowlistExtension E
  pool admin allowlists the router R: allowedSwapper[P][R] = true
  user Alice (address A) is NOT in the allowlist: allowedSwapper[P][A] = false

Attack:
  Alice calls MetricOmmSimpleRouter.exactInput(...)
    → router calls P.swap(recipient=Alice, ...)
    → pool calls E.beforeSwap(sender=router, ...)
    → extension checks: allowedSwapper[P][router] == true  ✓
    → swap executes; Alice receives tokens

Result:
  Alice, an unauthorized user, successfully swaps on a curated pool.
  The per-user allowlist guard is completely bypassed.
  LP funds are exposed to any public user through the supported router path.
``` [3](#0-2) [1](#0-0)

    external
    view
    override

    }
