### Title
`SwapAllowlistExtension.beforeSwap` gates the router address instead of the end-user, allowing any unprivileged user to bypass the per-user swap allowlist via `MetricOmmSimpleRouter` — (`metric-periphery/contracts/extensions/SwapAllowlistExtension.sol`)

---

### Summary

`SwapAllowlistExtension.beforeSwap` checks `allowedSwapper[pool][sender]` where `sender` is `msg.sender` of `MetricOmmPool.swap`. When a user routes through `MetricOmmSimpleRouter`, `sender` becomes the router's address, not the original user's address. If the pool admin allowlists the router (the only way to permit router-mediated swaps for legitimate users), every unprivileged user can bypass the per-user allowlist by routing through the router.

---

### Finding Description

`MetricOmmPool.swap` passes `msg.sender` as the `sender` argument to `_beforeSwap`, which forwards it verbatim to the extension: [1](#0-0) 

`SwapAllowlistExtension.beforeSwap` then checks the allowlist keyed on that `sender`: [2](#0-1) 

When a user calls `MetricOmmSimpleRouter`, the router calls `pool.swap(...)` and becomes `msg.sender` inside the pool. The extension therefore checks `allowedSwapper[pool][router]`, not `allowedSwapper[pool][original_user]`.

This creates an irresolvable dilemma for any pool admin who deploys a swap allowlist:

| Router allowlisted? | Effect |
|---|---|
| No | Allowlisted users cannot use the router at all — broken UX |
| Yes | **Any** user can bypass the per-user allowlist by routing through the router |

The `DepositAllowlistExtension` does not share this flaw because it gates on `owner` (the position owner explicitly passed by the caller), not on `sender`: [3](#0-2) 

The swap extension has no equivalent owner-level parameter to fall back on.

---

### Impact Explanation

A pool protected by `SwapAllowlistExtension` is intended to restrict swaps to a curated set of addresses (e.g., KYC'd counterparties, institutional traders). Once the pool admin allowlists the router — a necessary step for any legitimate user who relies on the standard periphery UX — the guard is silently nullified. Any unprivileged address can call `MetricOmmSimpleRouter` and execute swaps on the restricted pool. Depending on pool configuration, this enables unauthorized price-taking, front-running of allowlisted traders, and extraction of LP value at oracle-anchored prices the pool was not intended to expose to the public.

---

### Likelihood Explanation

The trigger is a valid, expected pool-admin action: allowlisting the router so that legitimate users can access the pool through the standard periphery. Any user who observes that the router is allowlisted (on-chain readable via `allowedSwapper`) can immediately exploit the bypass without any privileged access. The bypass is reachable on every swap through `MetricOmmSimpleRouter` on an affected pool.

---

### Recommendation

The extension must gate on the economically relevant actor — the original end-user — not the intermediate router. Two sound approaches:

1. **Pass the original caller through `extensionData`**: The router encodes `msg.sender` into `extensionData`; the extension decodes and checks it. This requires a trusted router convention.
2. **Check `recipient` instead of `sender` for swap allowlisting**: If the pool's design equates the recipient with the beneficiary, gating on `recipient` avoids the router indirection. Verify this matches the intended semantics.
3. **Document that the router must never be allowlisted** and provide a direct-call path for allowlisted users — but this breaks the standard UX and is fragile.

---

### Proof of Concept

```
1. Deploy pool with SwapAllowlistExtension.
2. Pool admin: setAllowedToSwap(pool, alice, true)
   → Alice is the only intended swapper.
3. Pool admin: setAllowedToSwap(pool, router, true)
   → Necessary so Alice can use MetricOmmSimpleRouter.
4. Bob (not allowlisted) calls:
       router.exactInputSingle({ pool: pool, ... })
5. Router calls pool.swap(recipient=bob, ...) with msg.sender = router.
6. Extension evaluates: allowedSwapper[pool][router] == true  ✓
7. Bob's swap executes on the restricted pool — allowlist bypassed.
``` [4](#0-3) [5](#0-4)
