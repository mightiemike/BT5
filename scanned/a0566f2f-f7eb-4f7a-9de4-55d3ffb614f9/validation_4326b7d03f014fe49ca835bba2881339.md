### Title
SwapAllowlistExtension Checks Router Address Instead of End-User, Enabling Allowlist Bypass or Breaking Router Swaps for Allowlisted Users - (File: metric-periphery/contracts/extensions/SwapAllowlistExtension.sol)

---

### Summary

`SwapAllowlistExtension.beforeSwap` gates swaps by checking the `sender` argument passed by the pool. When `MetricOmmSimpleRouter` mediates a swap, the pool receives the router as `msg.sender` and forwards the router address as `sender` to the extension. The extension therefore checks `allowedSwapper[pool][router]` instead of `allowedSwapper[pool][end_user]`, making per-user allowlisting either bypassable or non-functional for router-mediated swaps.

---

### Finding Description

`MetricOmmPool.swap` passes `msg.sender` as the `sender` argument to `_beforeSwap`: [1](#0-0) 

`ExtensionCalling._beforeSwap` forwards that value unchanged to every configured extension: [2](#0-1) 

`SwapAllowlistExtension.beforeSwap` then checks `allowedSwapper[msg.sender][sender]`, where `msg.sender` is the pool and `sender` is whatever the pool forwarded: [3](#0-2) 

When `MetricOmmSimpleRouter` calls `pool.swap(...)`, the pool's `msg.sender` is the router. The router address is therefore what the extension sees as `sender`. The extension has no visibility into the original EOA that called the router.

This produces two mutually exclusive failure modes for any pool that uses `SwapAllowlistExtension`:

**Mode A – Allowlist bypass:** The pool admin allowlists the router address so that router-mediated swaps work. Because the check is `allowedSwapper[pool][router]`, every user who calls the router passes the check regardless of whether they are individually allowlisted. Any unprivileged user can trade on a curated pool by routing through `MetricOmmSimpleRouter`.

**Mode B – Broken router functionality:** The pool admin allowlists only individual user addresses and does not allowlist the router. Allowlisted users who call `exactInputSingle` or `exactInput` through the router have their swaps revert with `NotAllowedToSwap` because the router address is not in the allowlist. Core swap functionality is broken for every allowlisted user who relies on the router. [4](#0-3) 

---

### Impact Explanation

**Mode A** is a direct allowlist bypass. An unprivileged user gains access to a curated pool that was explicitly configured to exclude them. Depending on pool configuration, this can result in unauthorized extraction of LP value, fee manipulation, or violation of regulatory/compliance constraints the pool admin intended to enforce. This meets the "admin-boundary break" and "broken core pool functionality" impact gates.

**Mode B** renders the router unusable for every allowlisted user on any pool that uses `SwapAllowlistExtension` without also allowlisting the router. This is broken core swap functionality.

---

### Likelihood Explanation

`SwapAllowlistExtension` is a first-party periphery extension explicitly documented as a reference implementation for access-controlled pools. Any pool that deploys it and expects per-user gating will encounter one of the two failure modes the moment a user attempts a router-mediated swap. The router is the primary user-facing swap surface, so the collision is inevitable in normal operation. No special privileges or malicious setup are required — a standard `exactInputSingle` call through `MetricOmmSimpleRouter` is sufficient to trigger either mode.

---

### Recommendation

The pool's `swap` function should accept an explicit `sender` parameter (the originating user) rather than deriving it from `msg.sender`. The router would pass `msg.sender` (the EOA) as that parameter, and the pool would forward it to extensions. This is the same pattern used for `recipient`. Alternatively, `SwapAllowlistExtension` could require the router to embed the real user address in `extensionData` and verify it, but this is fragile and requires router cooperation. The cleanest fix is a first-class `sender` parameter on `pool.swap`.

---

### Proof of Concept

1. Deploy a pool with `SwapAllowlistExtension` configured in `BEFORE_SWAP_ORDER`.
2. Pool admin calls `setAllowedToSwap(pool, alice, true)` — Alice is the only allowlisted user.
3. Pool admin also calls `setAllowedToSwap(pool, router, true)` so that router-mediated swaps are possible.
4. Bob (not allowlisted) calls `MetricOmmSimpleRouter.exactInputSingle(pool, ...)`.
5. The router calls `pool.swap(recipient=bob, ...)`. The pool's `msg.sender` is the router.
6. `_beforeSwap(router, bob, ...)` is called; the extension checks `allowedSwapper[pool][router] == true` → passes.
7. Bob's swap executes successfully on a pool he was explicitly excluded from.

Conversely, if step 3 is omitted (router not allowlisted):

4. Alice calls `MetricOmmSimpleRouter.exactInputSingle(pool, ...)`.
5. The router calls `pool.swap(...)`. Pool's `msg.sender` is the router.
6. Extension checks `allowedSwapper[pool][router] == false` → reverts `NotAllowedToSwap`.
7. Alice cannot use the router despite being individually allowlisted. [3](#0-2) [2](#0-1) [1](#0-0)

contract SwapAllowlistExtension is BaseMetricExtension, ISwapAllowlistExtension {
  mapping(address pool => mapping(address swapper => bool)) public allowedSwapper;
  mapping(address pool => bool) public allowAllSwappers;

  constructor(address factory_) BaseMetricExtension(factory_) {}

  function setAllowedToSwap(address pool_, address swapper, bool allowed) external onlyPoolAdmin(pool_) {
    allowedSwapper[pool_][swapper] = allowed;
    emit AllowedToSwapSet(pool_, swapper, allowed);
  }

  function setAllowAllSwappers(address pool_, bool allowed) external onlyPoolAdmin(pool_) {
    allowAllSwappers[pool_] = allowed;
    emit AllowAllSwappersSet(pool_, allowed);
  }

  function isAllowedToSwap(address pool_, address swapper) external view returns (bool) {
    return allowAllSwappers[pool_] || allowedSwapper[pool_][swapper];
  }

  function beforeSwap(address sender, address, bool, int128, uint128, uint256, uint128, uint128, bytes calldata)
    external
    view
    override
    returns (bytes4)
  {
    if (!allowAllSwappers[msg.sender] && !allowedSwapper[msg.sender][sender]) {
      revert IMetricOmmPoolActions.NotAllowedToSwap();
    }
    return IMetricOmmExtensions.beforeSwap.selector;
  }
