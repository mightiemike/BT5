### Title
`SwapAllowlistExtension` checks the router's address instead of the end user's address, allowing any user to bypass the per-user swap allowlist via `MetricOmmSimpleRouter` — (`metric-periphery/contracts/extensions/SwapAllowlistExtension.sol`)

### Summary

`SwapAllowlistExtension.beforeSwap` receives `sender` from the pool, which is always `msg.sender` of `pool.swap()`. When a user routes through `MetricOmmSimpleRouter`, `msg.sender` of `pool.swap()` is the **router contract**, not the end user. If the pool admin allowlists the router (a natural step to let their users access multi-hop routing), every unprivileged user can bypass the per-user swap restriction by routing through the public router.

### Finding Description

The pool's `swap()` function passes `msg.sender` as `sender` to `_beforeSwap`: [1](#0-0) 

`ExtensionCalling._beforeSwap` forwards that value unchanged to every configured extension: [2](#0-1) 

`SwapAllowlistExtension.beforeSwap` then gates on `allowedSwapper[msg.sender][sender]`, where `msg.sender` is the pool and `sender` is whoever called `pool.swap()`: [3](#0-2) 

When a user calls `MetricOmmSimpleRouter.exactInputSingle()`, the router calls `pool.swap()` directly: [4](#0-3) 

At that point `msg.sender` inside `pool.swap()` is the **router**, so `sender` forwarded to the extension is the router address — not the end user. The extension therefore checks `allowedSwapper[pool][router]`, not `allowedSwapper[pool][endUser]`.

The `DepositAllowlistExtension` does **not** share this flaw: it gates on the `owner` argument (the position owner), which the pool passes explicitly and which the liquidity adder preserves correctly: [5](#0-4) 

### Impact Explanation

A pool admin who wants to restrict swaps to a curated set of users (e.g., KYC'd addresses) deploys a pool with `SwapAllowlistExtension`. To let those users access multi-hop routing, the admin allowlists the router address. From that moment, **any** unprivileged user can call `MetricOmmSimpleRouter.exactInputSingle()` and swap against the restricted pool, because the extension sees `sender = router` (allowlisted) rather than the actual end user (not allowlisted). The pool's LP assets are exposed to unrestricted trading, defeating the entire access-control purpose of the extension.

### Likelihood Explanation

The trigger is fully unprivileged: any external account can call `MetricOmmSimpleRouter.exactInputSingle()`. The precondition — the router being allowlisted — is the natural operational step a pool admin takes when they want their allowlisted users to use the public router. No malicious setup is required; the bypass is a direct consequence of normal, expected admin behaviour.

### Recommendation

The `SwapAllowlistExtension` should gate on the **end user** rather than the direct caller of `pool.swap()`. Two complementary fixes:

1. **Pool-side**: Pass an additional "originating user" field through the extension interface (analogous to how `addLiquidity` separates `msg.sender` from `owner`), so extensions can always see the economic actor.

2. **Extension-side (short-term)**: Until the interface is extended, `SwapAllowlistExtension` should also check the `recipient` argument (the second parameter of `beforeSwap`) when `sender` is a known router, or the admin documentation must explicitly warn that allowlisting the router opens the pool to all users.

### Proof of Concept

```
Setup:
  pool configured with SwapAllowlistExtension
  admin calls setAllowedToSwap(pool, router, true)   // to let KYC'd users use the router
  alice (not in allowlist) wants to swap

Attack:
  alice calls MetricOmmSimpleRouter.exactInputSingle({pool, ...})
    → router calls pool.swap(recipient=alice, ...)
    → pool calls _beforeSwap(sender=router, ...)
    → SwapAllowlistExtension.beforeSwap checks allowedSwapper[pool][router] == true
    → no revert
    → alice's swap executes against the restricted pool

Result:
  alice, who was never allowlisted, successfully swaps in a pool
  that was intended to be restricted to specific users only.
``` [6](#0-5) [7](#0-6) [8](#0-7)
