### Title
SwapAllowlistExtension Gates Router Address Instead of Actual User, Enabling Full Allowlist Bypass via MetricOmmSimpleRouter — (`metric-periphery/contracts/extensions/SwapAllowlistExtension.sol`)

---

### Summary

`SwapAllowlistExtension.beforeSwap` checks the `sender` argument forwarded by the pool, which equals `msg.sender` at the pool call site. When a swap is routed through `MetricOmmSimpleRouter`, the pool's `msg.sender` is the router contract, not the end user. The extension therefore checks `allowedSwapper[pool][router]` instead of `allowedSwapper[pool][actualUser]`. This is the direct analog of M-03: a non-unique intermediary identity (the router address) is used in place of the unique per-user identity the guard was designed to enforce.

---

### Finding Description

`MetricOmmPool.swap()` passes `msg.sender` as the `sender` argument to `_beforeSwap`, which forwards it verbatim to every configured extension. [1](#0-0) 

`SwapAllowlistExtension.beforeSwap` receives that value as its first parameter and checks it against the per-pool allowlist:

```solidity
function beforeSwap(address sender, address, bool, int128, uint128, uint256, uint128, uint128, bytes calldata)
    external view override returns (bytes4)
{
    if (!allowAllSwappers[msg.sender] && !allowedSwapper[msg.sender][sender]) {
        revert IMetricOmmPoolActions.NotAllowedToSwap();
    }
    return IMetricOmmExtensions.beforeSwap.selector;
}
``` [2](#0-1) 

When a user calls `pool.swap()` directly, `sender` = user → `allowedSwapper[pool][user]` is checked correctly.

When a user calls `MetricOmmSimpleRouter.exactInput*()` → router calls `pool.swap()`, `sender` = router → `allowedSwapper[pool][router]` is checked instead.

The allowlist is keyed per-pool and per-swapper: [3](#0-2) 

This creates an irreconcilable design conflict for any pool that uses `SwapAllowlistExtension` and also expects users to route through `MetricOmmSimpleRouter`:

- **If the pool admin does NOT allowlist the router**: allowlisted users cannot swap through the router (their swaps revert with `NotAllowedToSwap` because `sender` = router is not on the list).
- **If the pool admin DOES allowlist the router** (the only way to let allowlisted users use the router): `allowedSwapper[pool][router] = true`, so the check passes for *any* caller who routes through the router — the per-user gate is completely bypassed.

The second branch is the fund-impacting path. A pool admin who wants to allow their allowlisted LPs or partners to use the public router must add the router to the allowlist. Once the router is allowlisted, any unprivileged user can call `MetricOmmSimpleRouter` and swap on the restricted pool, defeating the entire access-control purpose.

---

### Impact Explanation

**Direct loss / broken core functionality.** A pool deploying `SwapAllowlistExtension` to restrict trading to specific counterparties (e.g., a private OTC pool, a KYC-gated pool, or a pool with a stop-loss extension that relies on the allowlist to limit who can trigger it) can be fully opened to arbitrary public swappers the moment the router is allowlisted. Unauthorized swappers can drain liquidity at oracle-quoted prices, extract value from the pool's LP positions, or trigger downstream extension state (e.g., stop-loss watermarks) in ways the pool admin did not intend.

---

### Likelihood Explanation

The `MetricOmmSimpleRouter` is the primary user-facing swap entry point documented in the periphery. Any pool admin who wants allowlisted users to be able to use the standard router UI will naturally add the router to the allowlist. The bypass is then immediately available to all public users with no further preconditions. The trigger is a routine, expected admin action, not an exotic configuration. [4](#0-3) 

---

### Recommendation

The extension must receive and check the **original end-user address**, not the intermediary's address. Two complementary fixes:

1. **Router-level**: `MetricOmmSimpleRouter` should accept an explicit `swapper` parameter (the actual user) and pass it as `extensionData` or via a dedicated field so extensions can verify the real initiator.
2. **Extension-level**: `SwapAllowlistExtension.beforeSwap` should decode the real user from `extensionData` when `sender` is a known router, or the pool interface should propagate `tx.origin`-equivalent identity through a trusted forwarder pattern.

The cleanest fix mirrors the M-03 recommendation: gate on the unique, non-spoofable identity (the end user's address) rather than the intermediary address that any public user can route through.

---

### Proof of Concept

1. Pool admin deploys a pool with `SwapAllowlistExtension` configured.
2. Admin calls `setAllowedToSwap(pool, alice, true)` — only Alice should be able to swap.
3. Admin calls `setAllowedToSwap(pool, router, true)` — necessary so Alice can use the standard router UI.
4. Bob (not allowlisted) calls `MetricOmmSimpleRouter.exactInputSingle(pool, ...)`.
5. Router calls `pool.swap(recipient=Bob, ...)` — pool's `msg.sender` = router.
6. Pool calls `SwapAllowlistExtension.beforeSwap(sender=router, ...)`.
7. Extension checks `allowedSwapper[pool][router]` → `true` → swap proceeds.
8. Bob successfully swaps on the restricted pool, bypassing the per-user allowlist. [5](#0-4)
