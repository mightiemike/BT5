### Title
`SwapAllowlistExtension` Checks Router Address Instead of Actual Swapper, Allowing Allowlist Bypass via `MetricOmmSimpleRouter` - (File: metric-periphery/contracts/extensions/SwapAllowlistExtension.sol)

---

### Summary

`SwapAllowlistExtension.beforeSwap` gates swaps by checking the `sender` argument passed by the pool. When a swap is routed through `MetricOmmSimpleRouter`, the pool's `msg.sender` is the **router contract**, not the originating user. The extension therefore checks whether the router is allowlisted, not whether the actual user is allowlisted. Any user can bypass a curated pool's swap allowlist by routing through the supported periphery router.

---

### Finding Description

The call chain for a router-mediated swap is:

```
User → MetricOmmSimpleRouter.exactInputSingle()
     → IMetricOmmPoolActions(pool).swap(recipient, ...)   // msg.sender = router
     → MetricOmmPool.swap() calls _beforeSwap(msg.sender=router, ...)
     → ExtensionCalling._beforeSwap passes sender=router to extension
     → SwapAllowlistExtension.beforeSwap(sender=router, ...)
```

In `MetricOmmPool.swap`, the pool passes its own `msg.sender` (the router) as the `sender` argument to `_beforeSwap`: [1](#0-0) 

`ExtensionCalling._beforeSwap` forwards this value unchanged to every configured extension: [2](#0-1) 

`SwapAllowlistExtension.beforeSwap` then checks `allowedSwapper[msg.sender][sender]`, where `msg.sender` is the pool and `sender` is the router: [3](#0-2) 

The router never forwards the originating user's address to the pool — it simply calls `pool.swap()` directly, making itself the `msg.sender`: [4](#0-3) 

This creates two mutually exclusive broken states for a pool admin who deploys a `SwapAllowlistExtension`:

1. **Router not allowlisted**: All router-mediated swaps revert with `NotAllowedToSwap`, even for users who are individually allowlisted. The standard periphery path is completely unusable for the pool.
2. **Router allowlisted** (to restore router functionality): `allowedSwapper[pool][router] = true` means every call through the router passes the check regardless of who the originating user is. Any non-allowlisted user bypasses the gate by calling `exactInputSingle` / `exactInput` / `exactOutput` / `exactOutputSingle`.

The same structural mismatch applies to `exactOutput` and multi-hop `exactInput`/`exactOutput` paths: [5](#0-4) [6](#0-5) 

---

### Impact Explanation

A pool configured with `SwapAllowlistExtension` and `allowAllSwappers = false` is intended to restrict trading to a curated set of addresses (e.g., KYC'd users, whitelisted market makers). Any non-allowlisted user can bypass this restriction entirely by calling any of the four router entry points. The allowlist provides no protection against router-mediated swaps. This constitutes a broken core pool functionality and an admin-boundary break: the pool admin's curation policy is silently voided by the standard supported periphery path.

---

### Likelihood Explanation

The `MetricOmmSimpleRouter` is the primary supported swap interface for end users. Any actor who is blocked by a direct `pool.swap()` call will naturally attempt the router path. No special privileges, flash loans, or multi-transaction sequences are required — a single `exactInputSingle` call suffices.

---

### Recommendation

The extension must gate on the **originating user**, not the intermediary. Two approaches:

1. **Pass the original caller through the pool**: Add an explicit `originator` field to the `beforeSwap` hook arguments (alongside `sender`) that the pool populates from a trusted source (e.g., a router that identifies itself and attests the real caller via `extensionData`).
2. **Decode originator from `extensionData`**: The router encodes `msg.sender` into `extensionData` before calling the pool; the extension decodes and verifies it. This requires the extension to trust only known router addresses.
3. **Check `recipient` instead of `sender`** (partial mitigation only): For single-hop swaps where the user is also the recipient, checking `recipient` would gate the correct actor, but this breaks for multi-hop paths where intermediate recipients are the router itself.

The cleanest fix is option 1: extend `IMetricOmmExtensions.beforeSwap` to carry a verified `originator` address that the pool sets to `tx.origin` or that the router attests via a signed field in `extensionData`.

---

### Proof of Concept

```solidity
// Pool admin sets up a curated pool
SwapAllowlistExtension ext = new SwapAllowlistExtension(factory);
// allowlist only alice
ext.setAllowedToSwap(pool, alice, true);
// router is NOT allowlisted

// Alice can swap directly:
pool.swap(alice, true, 1000, type(uint128).max, "", ""); // passes: allowedSwapper[pool][alice] = true

// Bob (not allowlisted) tries directly — correctly blocked:
pool.swap(bob, true, 1000, type(uint128).max, "", ""); // reverts: NotAllowedToSwap

// Bob routes through MetricOmmSimpleRouter:
// pool.swap is called with msg.sender = router
// extension checks allowedSwapper[pool][router]
// If admin allowlisted router to enable alice's router swaps → bob passes too
router.exactInputSingle(ExactInputSingleParams({
    pool: pool,
    recipient: bob,
    zeroForOne: true,
    amountIn: 1000,
    ...
})); // PASSES — allowlist bypassed
```
