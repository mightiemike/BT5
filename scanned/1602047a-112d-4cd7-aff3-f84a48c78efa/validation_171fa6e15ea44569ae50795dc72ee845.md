### Title
Swap Allowlist Bypassed via Router: Any EOA Can Swap Through an Allowlisted Router — (`metric-periphery/contracts/extensions/SwapAllowlistExtension.sol`)

### Summary
`SwapAllowlistExtension.beforeSwap` checks the `sender` argument passed by the pool, which is `msg.sender` of the `pool.swap` call — the router contract — not the original transaction initiator. Any EOA can bypass a per-pool swap allowlist by routing through `MetricOmmSimpleRouter` if the router address is allowlisted.

### Finding Description
`MetricOmmPool.swap` passes `msg.sender` as the `sender` argument to `_beforeSwap`: [1](#0-0) 

`ExtensionCalling._beforeSwap` forwards that value unchanged to the extension: [2](#0-1) 

`SwapAllowlistExtension.beforeSwap` then checks `allowedSwapper[msg.sender][sender]`, where `msg.sender` is the pool and `sender` is whoever called `pool.swap` — the router, not the EOA: [3](#0-2) 

`MetricOmmSimpleRouter.exactInputSingle` calls `pool.swap` directly without forwarding the original `msg.sender` to the pool as the swapper identity: [4](#0-3) 

The router's address is what the pool and extension see as `sender`. If `allowedSwapper[pool][router] = true`, every call through the router passes the allowlist check regardless of who the actual EOA caller is.

### Impact Explanation
Any non-allowlisted address can swap against a pool that is supposed to be restricted to a specific set of swappers. The allowlist invariant — "only allowlisted addresses may swap" — is completely broken for any pool that allowlists the public router. Unauthorized swappers can drain LP token balances by executing swaps the pool admin intended to restrict.

### Likelihood Explanation
The `MetricOmmSimpleRouter` is the standard public swap entrypoint. Pool admins who configure `SwapAllowlistExtension` and add the router to the allowlist (a natural and expected configuration) unknowingly open the pool to all users. The attack requires no special privileges, no flash loans, and no unusual token behavior — just calling the public router.

### Recommendation
`SwapAllowlistExtension` must gate the **original transaction initiator**, not the immediate `pool.swap` caller. Two options:

1. **Pass `tx.origin` as an additional parameter** — fragile and generally discouraged.
2. **Require the router to forward the original caller's identity** — `MetricOmmSimpleRouter` should pass the original `msg.sender` as an explicit `sender` argument to `pool.swap`, and the pool should expose a separate `swapOnBehalf(address sender, ...)` entry point that only trusted periphery contracts may call, with the extension checking that forwarded identity.

The cleaner fix is option 2: add a privileged `swapFor(address realSender, ...)` function on the pool (callable only by factory-registered routers), and have `SwapAllowlistExtension` check `realSender` rather than the pool's `msg.sender`.
