### Title
SwapAllowlistExtension Checks Router Address Instead of Actual User, Allowing Full Allowlist Bypass via MetricOmmSimpleRouter — (`metric-periphery/contracts/extensions/SwapAllowlistExtension.sol`)

---

### Summary

`SwapAllowlistExtension.beforeSwap` gates swaps by checking the `sender` argument, which the pool sets to its own `msg.sender`. When a user routes through `MetricOmmSimpleRouter`, the pool's `msg.sender` is the **router contract**, not the actual user. If the pool admin allowlists the router address (a natural operational step to enable router-mediated swaps), every user — including those explicitly excluded from the allowlist — can bypass the gate by routing through the router.

---

### Finding Description

**Pool → Extension call chain**

`MetricOmmPool.swap` passes `msg.sender` as the `sender` argument to `_beforeSwap`: [1](#0-0) 

`ExtensionCalling._beforeSwap` forwards that value unchanged to every configured extension: [2](#0-1) 

**The allowlist check**

`SwapAllowlistExtension.beforeSwap` checks `allowedSwapper[msg.sender][sender]`, where `msg.sender` is the pool and `sender` is whatever called `pool.swap()`: [3](#0-2) 

**The router never forwards the real user**

`MetricOmmSimpleRouter.exactInputSingle` calls `pool.swap(...)` directly — the pool sees `msg.sender = router`, not the EOA: [4](#0-3) 

The same is true for `exactInput`, `exactOutputSingle`, and `exactOutput`. [5](#0-4) 

**Result**

| Scenario | `sender` seen by extension | Allowlist check |
|---|---|---|
| User calls `pool.swap()` directly | `user_address` | Correct — gates the real actor |
| User calls `router.exactInputSingle()` | `router_address` | Wrong — gates the router, not the user |

A pool admin who wants to allow router-mediated swaps must call `setAllowedToSwap(pool, router, true)`. The moment they do, the check becomes `allowedSwapper[pool][router] == true` for **every** caller of the router, including addresses that were explicitly excluded from the allowlist. There is no way to allowlist specific users while also permitting them to use the router, because the extension cannot observe the real initiator. [6](#0-5) 

---

### Impact Explanation

A curated pool deploying `SwapAllowlistExtension` to restrict trading to a known set of counterparties loses that protection entirely once the router is allowlisted. Any unprivileged address can execute swaps at the oracle-anchored bid/ask, draining LP-owned token reserves at will. Because the pool is oracle-priced, a disallowed actor can repeatedly extract value from bins at the live mid-price, causing direct loss of LP principal. This satisfies the "broken core pool functionality causing loss of funds" and "admin-boundary break bypassed by an unprivileged path" impact criteria.

---

### Likelihood Explanation

The `MetricOmmSimpleRouter` is the canonical user-facing entry point documented in the periphery README. Any pool admin who wants end-users to be able to swap through the router must allowlist it. This is a routine operational step, not an exotic configuration. The bypass is therefore reachable on any production pool that uses `SwapAllowlistExtension` together with the router.

---

### Recommendation

The extension must gate the **economically relevant actor**, not the intermediary. Two options:

1. **Require the real user to self-identify in `extensionData`**: The router already forwards `params.extensionData` to the pool unchanged. The extension can decode a user-supplied address from that payload and check it. The pool admin documents that router users must include their address in the extension payload. (Note: this relies on the router not stripping or replacing the payload, which the current code does not do.)

2. **Document incompatibility**: Clearly state in the extension NatDoc that `SwapAllowlistExtension` cannot enforce per-user restrictions when the pool is accessed through any intermediary contract. Pool admins who need per-user gating must require direct `pool.swap()` calls and must **not** allowlist the router.

A minimal code-level fix for option 1:

```solidity
function beforeSwap(address sender, address, bool, int128, uint128, uint256, uint128, uint128, bytes calldata extensionData)
    external view override returns (bytes4)
{
    address actor = sender;
    // If sender is a known router, decode the real user from extensionData
    if (extensionData.length >= 32) {
        address claimed = abi.decode(extensionData, (address));
        if (claimed != address(0)) actor = claimed;
    }
    if (!allowAllSwappers[msg.sender] && !allowedSwapper[msg.sender][actor]) {
        revert IMetricOmmPoolActions.NotAllowedToSwap();
    }
    return IMetricOmmExtensions.beforeSwap.selector;
}
```

### Proof of Concept

```
1. Deploy pool with SwapAllowlistExtension; allowAllSwappers = false.
2. Pool admin calls setAllowedToSwap(pool, alice, true).
   // alice is the only allowed swapper.
3. Pool admin calls setAllowedToSwap(pool, router, true).
   // Necessary so alice can use the router.
4. Bob (not allowlisted) calls router.exactInputSingle({pool: pool, ...}).
   // router calls pool.swap(); pool sees msg.sender = router.
   // Extension checks allowedSwapper[pool][router] == true → passes.
   // Bob's swap executes despite being excluded from the allowlist.
5. Bob drains token reserves at oracle mid-price, causing LP losses.
```
