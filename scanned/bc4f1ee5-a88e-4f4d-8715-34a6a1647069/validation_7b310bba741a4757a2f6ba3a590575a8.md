### Title
`SwapAllowlistExtension` Checks Router Identity Instead of Actual Swapper, Enabling Full Allowlist Bypass via `MetricOmmSimpleRouter` — (`metric-periphery/contracts/extensions/SwapAllowlistExtension.sol`)

---

### Summary

`SwapAllowlistExtension.beforeSwap` gates swaps by checking the `sender` argument, which the pool sets to `msg.sender` of the `pool.swap()` call. When a user routes through `MetricOmmSimpleRouter`, `msg.sender` at the pool is the **router**, not the user. If the pool admin allowlists the router (a natural action to enable router-based swaps for permitted users), every unprivileged address can bypass the per-user allowlist by routing through the router.

---

### Finding Description

`MetricOmmPool.swap` passes `msg.sender` as the `sender` argument to `_beforeSwap`: [1](#0-0) 

`_beforeSwap` forwards that value unchanged to the extension: [2](#0-1) 

`SwapAllowlistExtension.beforeSwap` then checks whether `sender` (the pool's `msg.sender`) is on the allowlist: [3](#0-2) 

`MetricOmmSimpleRouter.exactInputSingle` calls `pool.swap()` directly with no forwarding of the original caller's identity: [4](#0-3) 

The router is `msg.sender` at the pool boundary, so the extension sees `sender = router`. The actual user who called the router is invisible to the extension.

The same identity-loss occurs in `exactInput`, `exactOutputSingle`, and `exactOutput`: [5](#0-4) 

---

### Impact Explanation

A pool admin who wants to allow only specific addresses to swap will:
1. Deploy the pool with `SwapAllowlistExtension`.
2. Allowlist specific users: `setAllowedToSwap(pool, user1, true)`.
3. Allowlist the router so that permitted users can use it: `setAllowedToSwap(pool, router, true)`.

Step 3 is a reasonable, non-malicious action. Its unintended consequence is that **any address** can now call `router.exactInputSingle({pool: pool, ...})`. The extension sees `sender = router`, which is allowlisted, and the swap proceeds regardless of who the actual caller is. The per-user allowlist is completely defeated.

The admin has no way to selectively enable router-based swaps for only allowlisted users: either the router is not allowlisted (blocking all router paths, including for permitted users) or it is allowlisted (opening the pool to everyone). The invariant that only approved addresses may swap is broken.

This matches the "admin-boundary break: factory/oracle role checks are bypassed by an unprivileged path" impact class.

---

### Likelihood Explanation

The trigger is the pool admin allowlisting the router — a routine operational step that any operator would take to make the pool usable through the standard periphery. No malicious admin intent is required. Once the router is allowlisted, any unprivileged address can exploit the bypass immediately and repeatedly.

---

### Recommendation

The extension must gate the **original user**, not the intermediary. Two sound approaches:

1. **Pass original caller through `extensionData`**: The router encodes `msg.sender` into `extensionData`; the extension decodes and verifies it. This requires a trusted router convention.
2. **Check `sender` only for direct pool calls; require a signed proof for router calls**: The extension inspects `extensionData` for a pool-admin-signed allowlist credential when `sender` is a known router.

The simplest correct fix is to have the router always encode the originating user address in `extensionData` and have `SwapAllowlistExtension` read that field when `sender` is not itself on the per-address allowlist.

---

### Proof of Concept

```solidity
// Setup: pool with SwapAllowlistExtension; only `permittedUser` is allowlisted.
// Admin also allowlists the router so permittedUser can use it.
extension.setAllowedToSwap(address(pool), address(router), true);
extension.setAllowedToSwap(address(pool), permittedUser, true);

// Attacker is NOT allowlisted.
// Direct swap reverts:
vm.prank(attacker);
vm.expectRevert(IMetricOmmPoolActions.NotAllowedToSwap.selector);
pool.swap(attacker, false, 1000, type(uint128).max, "", "");

// Router-mediated swap succeeds — allowlist bypassed:
vm.prank(attacker);
// router.exactInputSingle calls pool.swap with msg.sender = router (allowlisted)
router.exactInputSingle(
    IMetricOmmSimpleRouter.ExactInputSingleParams({
        pool: address(pool),
        recipient: attacker,
        tokenIn: address(token1),
        zeroForOne: false,
        amountIn: 1000,
        amountOutMinimum: 0,
        priceLimitX64: type(uint128).max,
        deadline: block.timestamp,
        extensionData: ""
    })
);
// Attacker receives token0 — allowlist was not enforced.
```
