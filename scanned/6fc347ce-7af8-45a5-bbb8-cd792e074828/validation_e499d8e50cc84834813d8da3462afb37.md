### Title
`SwapAllowlistExtension.beforeSwap` Checks Router Address as `sender` Instead of Actual User, Allowing Any User to Bypass the Per-Swapper Allowlist — (`metric-periphery/contracts/extensions/SwapAllowlistExtension.sol`)

---

### Summary

`SwapAllowlistExtension` is designed to gate swaps by individual swapper address. However, its `beforeSwap` hook checks `sender`, which is `msg.sender` of the pool's `swap()` call. When users route through `MetricOmmSimpleRouter`, `sender` is the **router address**, not the actual user. If the pool admin allowlists the router (the only way to make router-mediated swaps work at all), every user — including those not individually allowlisted — can bypass the per-swapper restriction.

---

### Finding Description

`SwapAllowlistExtension.beforeSwap` receives `sender` as its first argument and checks it against `allowedSwapper[msg.sender][sender]`, where `msg.sender` is the pool: [1](#0-0) 

`sender` is populated by the pool's `ExtensionCalling._beforeSwap`, which passes `msg.sender` of the pool's own `swap()` call: [2](#0-1) 

When `MetricOmmSimpleRouter` executes a swap, it calls `pool.swap(recipient, ...)` directly. The pool's `msg.sender` at that point is the **router**, not the end user. The end user's address appears only as `recipient` (the second parameter), which `SwapAllowlistExtension.beforeSwap` silently discards (`address,`). [1](#0-0) 

This creates two mutually exclusive failure modes:

1. **Router not allowlisted**: Individually allowlisted users cannot swap through the router at all — `allowedSwapper[pool][router]` is false, so every router-mediated swap reverts with `NotAllowedToSwap`. Core swap functionality is broken for all allowlisted users.

2. **Router allowlisted** (the only fix): `allowAllSwappers` is false but `allowedSwapper[pool][router] = true`. Now **every** user — including those never individually allowlisted — can swap through the router, completely defeating the per-swapper gate.

The admin setter name `setAllowedToSwap(pool_, address swapper, bool allowed)` and the mapping name `allowedSwapper` confirm the design intent is per-user gating, not per-router gating: [3](#0-2) 

---

### Impact Explanation

An unprivileged user who is **not** on the allowlist can bypass the pool admin's swap restriction by routing through `MetricOmmSimpleRouter` once the router is allowlisted. This is an admin-boundary break: the pool admin's intended per-user access control is rendered ineffective by a valid, unprivileged path (the standard router). Any user can drain pool liquidity or execute swaps in a pool that was supposed to be restricted (e.g., KYC-gated, whitelist-only LP pools).

---

### Likelihood Explanation

- The `SwapAllowlistExtension` is a production periphery contract intended for deployment.
- Any pool that uses `SwapAllowlistExtension` with `allowAllSwappers = false` and also wants users to use the router **must** allowlist the router — triggering the bypass automatically.
- No special privileges or malicious setup are required; any user with access to the router can exploit this.

---

### Recommendation

Change `beforeSwap` to check the actual end-user. Since the router passes the user as `recipient`, the extension should check `recipient` instead of (or in addition to) `sender`:

```solidity
function beforeSwap(
    address sender,
    address recipient,   // ← use this
    bool, int128, uint128, uint256, uint128, uint128, bytes calldata
) external view override returns (bytes4) {
    address actor = (sender == address(router)) ? recipient : sender;
    if (!allowAllSwappers[msg.sender] && !allowedSwapper[msg.sender][actor]) {
        revert IMetricOmmPoolActions.NotAllowedToSwap();
    }
    return IMetricOmmExtensions.beforeSwap.selector;
}
```

Alternatively, require that the pool's registered router always forwards the real caller (e.g., via a trusted forwarder pattern), and validate `sender` is a known router before falling back to `recipient`.

---

### Proof of Concept

1. Pool admin deploys a pool with `SwapAllowlistExtension`, `allowAllSwappers[pool] = false`.
2. Admin calls `setAllowedToSwap(pool, alice, true)` — only Alice is supposed to swap.
3. Admin calls `setAllowedToSwap(pool, router, true)` — necessary for Alice to use the router.
4. Bob (not allowlisted) calls `MetricOmmSimpleRouter.exactInputSingle(pool, ...)`.
5. Router calls `pool.swap(bob, ...)` → pool calls `extension.beforeSwap(router, bob, ...)`.
6. Extension checks `allowedSwapper[pool][router]` → `true` → swap proceeds.
7. Bob successfully swaps despite never being allowlisted, bypassing the admin-configured restriction. [1](#0-0) [2](#0-1)

```text
contract SwapAllowlistExtension is BaseMetricExtension, ISwapAllowlistExtension {
  mapping(address pool => mapping(address swapper => bool)) public allowedSwapper;
  mapping(address pool => bool) public allowAllSwappers;

  constructor(address factory_) BaseMetricExtension(factory_) {}

  function setAllowedToSwap(address pool_, address swapper, bool allowed) external onlyPoolAdmin(pool_) {
    allowedSwapper[pool_][swapper] = allowed;
    emit AllowedToSwapSet(pool_, swapper, allowed);
  }

  function setAllowAllSwappers(address pool_, bool allowed) external onlyPoolAdmin(pool_) {
    allowAllSwappers[pool_] = allowed;
    emit AllowAllSwappersSet(pool_, allowed);
  }

  function isAllowedToSwap(address pool_, address swapper) external view returns (bool) {
    return allowAllSwappers[pool_] || allowedSwapper[pool_][swapper];
  }
```
