### Title
`SwapAllowlistExtension.beforeSwap` Gates the Router Address Instead of the Actual Swapper — (`metric-periphery/contracts/extensions/SwapAllowlistExtension.sol`)

---

### Summary

`SwapAllowlistExtension` is designed to restrict swaps to a per-pool allowlist of swapper addresses. However, the `beforeSwap` hook checks `sender`, which is the **direct caller of `pool.swap()`**, not the economic actor initiating the trade. When a user routes through `MetricOmmSimpleRouter`, the router becomes `msg.sender` of `pool.swap()`, so the extension checks the router's address against the allowlist — not the user's address. If the router is allowlisted (a natural admin action to enable router-based swaps), every user bypasses the per-user restriction entirely.

---

### Finding Description

`MetricOmmPool.swap()` passes `msg.sender` as the `sender` argument to `_beforeSwap`: [1](#0-0) 

`ExtensionCalling._beforeSwap` forwards that value unchanged as the first argument to the extension: [2](#0-1) 

`SwapAllowlistExtension.beforeSwap` then checks `allowedSwapper[msg.sender][sender]`, where `msg.sender` is the pool and `sender` is the direct caller of `pool.swap()`: [3](#0-2) 

`MetricOmmSimpleRouter.exactInputSingle` (and `exactInput`, `exactOutput`) calls `pool.swap()` directly, making the router the `msg.sender` seen by the pool: [4](#0-3) 

The result: the allowlist check resolves to `allowedSwapper[pool][router]`, not `allowedSwapper[pool][user]`. A pool admin who allowlists the router address (to permit router-based swaps) inadvertently opens the pool to **all** users, because any user routing through `MetricOmmSimpleRouter` is seen as the same allowlisted entity.

---

### Impact Explanation

A pool configured with `SwapAllowlistExtension` to restrict trading to a curated set of addresses (e.g., KYC-verified counterparties, whitelisted market makers, or LP-protection policies) is fully bypassed by any user who routes through the official `MetricOmmSimpleRouter`. The allowlist provides no per-user protection when the router is allowlisted. Unauthorized users can execute swaps against restricted pools, draining LP value to counterparties the pool admin explicitly intended to exclude.

---

### Likelihood Explanation

The `MetricOmmSimpleRouter` is the canonical periphery entry point. A pool admin deploying `SwapAllowlistExtension` to restrict individual users would naturally also need to allowlist the router so that permitted users can access the pool through the standard interface. This is the expected operational pattern, making the bypass condition a near-certain outcome of normal configuration. No special privileges or malicious setup are required — any user can call `router.exactInputSingle(...)` targeting the restricted pool.

---

### Recommendation

The `beforeSwap` hook should check the **actual economic actor**, not the intermediary caller. Two options:

1. **Check `recipient` instead of `sender`** — the recipient is the address receiving value from the swap, which is closer to the economic actor. However, this is also settable by the router.

2. **Pass the originating user through `extensionData`** — require the router to embed the originating user in `extensionData` and have the extension decode and verify it. This requires a coordinated router + extension design.

3. **Check both `sender` and `recipient`** — require that at least one of the two is allowlisted, which prevents the router-as-proxy bypass while still allowing direct swaps.

The cleanest fix is to redesign the extension interface so that the pool passes the **payer** (the address that will fund the swap callback) as a distinct argument, separate from the intermediary caller.

---

### Proof of Concept

1. Deploy a pool with `SwapAllowlistExtension` configured.
2. Pool admin calls `setAllowedToSwap(pool, router, true)` to allow router-based swaps for permitted users.
3. Pool admin calls `setAllowedToSwap(pool, alice, true)` to allowlist Alice.
4. Bob (not allowlisted) calls `router.exactInputSingle(...)` targeting the restricted pool.
5. Inside `pool.swap()`, `msg.sender` = router. The extension checks `allowedSwapper[pool][router]` → `true`. Bob's swap succeeds.
6. Bob has bypassed the per-user allowlist entirely by routing through the official periphery.

The check that should have blocked Bob — `allowedSwapper[pool][bob]` — is never evaluated. [5](#0-4)
