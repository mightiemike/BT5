### Title
SwapAllowlistExtension Checks Router Address Instead of Real Swapper, Allowing Any User to Bypass the Swap Allowlist — (File: metric-periphery/contracts/extensions/SwapAllowlistExtension.sol)

### Summary
When a swap is routed through `MetricOmmSimpleRouter`, the `sender` argument delivered to `SwapAllowlistExtension.beforeSwap` is the router's address, not the originating user's address. Because the allowlist is keyed on `sender`, any user can bypass a pool's swap allowlist by routing through the public router rather than calling `pool.swap()` directly.

### Finding Description
`MetricOmmPool.swap()` passes `msg.sender` verbatim as the `sender` argument to `_beforeSwap`: [1](#0-0) 

`ExtensionCalling._beforeSwap` then forwards that same `sender` value unchanged to every configured extension: [2](#0-1) 

The `IMetricOmmExtensions.beforeSwap` interface exposes both `sender` and `recipient` to the extension: [3](#0-2) 

When a user calls `MetricOmmSimpleRouter.exactInput*` or `exactOutput*`, the router itself calls `pool.swap()`, so `msg.sender` inside the pool is the router's address. The `sender` that reaches `SwapAllowlistExtension.beforeSwap` is therefore the router address — not the originating user. The allowlist lookup is performed against the router, not the real trader.

Two outcomes are both broken:
1. **Router not allowlisted** — every router-mediated swap reverts, making the public routing path completely unusable for any user on an allowlisted pool.
2. **Router allowlisted** (the only way to allow router-mediated swaps) — every user, including those explicitly excluded from the allowlist, can bypass the restriction by routing through the public `MetricOmmSimpleRouter`.

The research target in the repository explicitly identifies this path: [4](#0-3) 

### Impact Explanation
A pool operator deploys a pool with `SwapAllowlistExtension` to restrict trading to a curated set of counterparties (e.g., KYC'd addresses, institutional partners). Any non-allowlisted user routes their swap through the public `MetricOmmSimpleRouter`. The allowlist check passes because the router address — not the user — is evaluated. The unauthorized user receives pool output tokens; LP assets are consumed by a counterparty the pool was explicitly designed to exclude. This is a direct loss of LP-controlled assets to an unauthorized actor, satisfying the Sherlock Medium/High threshold for broken core pool functionality.

### Likelihood Explanation
`MetricOmmSimpleRouter` is the canonical public entry point for multi-hop and single-hop swaps. Any user aware of the allowlist restriction has an immediate, zero-cost bypass: call the router instead of the pool directly. No privileged access, no special setup, and no non-standard token behavior is required. The bypass is reachable by any EOA or contract on every allowlisted pool that also permits router-mediated swaps.

### Recommendation
`SwapAllowlistExtension.beforeSwap` must gate the economically relevant actor. Two sound approaches:

1. **Check `recipient` as a proxy for the real user** — only valid if the pool's design guarantees the recipient is always the originating user, which the router does not enforce.
2. **Require the router to forward the originating user** — add an `originatingUser` field to `extensionData` that the router populates with `msg.sender` before calling the pool, and have the extension decode and check that field. This requires a coordinated change to the router and the extension.
3. **Require direct pool interaction for allowlisted pools** — document and enforce that pools using `SwapAllowlistExtension` must not allowlist the router address, and the router must revert when calling allowlisted pools (requires a pool-level flag readable by the router).

The cleanest fix is option 2: the router encodes the real caller into `extensionData`, and the extension decodes and checks that address against the allowlist.

### Proof of Concept
```
Setup:
  - Deploy pool with SwapAllowlistExtension configured in BEFORE_SWAP_ORDER.
  - allowedSwapper[pool][allowedUser] = true
  - allowedSwapper[pool][attackerUser] = false
  - allowedSwapper[pool][routerAddress] = true  ← required for any router swap to work

Attack:
  1. attackerUser calls MetricOmmSimpleRouter.exactInputSingle(tokenIn, tokenOut, pool, ...)
  2. Router calls pool.swap(recipient=attackerUser, ...) → msg.sender = routerAddress
  3. Pool calls _beforeSwap(sender=routerAddress, recipient=attackerUser, ...)
  4. SwapAllowlistExtension.beforeSwap checks allowedSwapper[pool][routerAddress] → true → passes
  5. Swap executes; attackerUser receives output tokens from the LP-restricted pool.

Expected: revert NotAllowedToSwap()
Actual:   swap succeeds; attacker bypasses the allowlist entirely.
