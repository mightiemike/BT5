### Title
SwapAllowlistExtension Checks Router Address Instead of Actual Swapper, Allowing Full Allowlist Bypass — (`metric-periphery/contracts/extensions/SwapAllowlistExtension.sol`)

### Summary

`SwapAllowlistExtension.beforeSwap` gates swaps by checking the `sender` argument passed by the pool. When a user swaps through `MetricOmmSimpleRouter`, the pool's `msg.sender` is the router contract, so the pool forwards the router's address as `sender` to the extension. The allowlist check therefore evaluates `allowedSwapper[pool][router]` rather than `allowedSwapper[pool][actualUser]`. Any unprivileged user can bypass a curated pool's swap allowlist by routing through the public router.

### Finding Description

`SwapAllowlistExtension.beforeSwap` receives `sender` as its first argument and checks:

```solidity
if (!allowAllSwappers[msg.sender] && !allowedSwapper[msg.sender][sender]) {
    revert IMetricOmmPoolActions.NotAllowedToSwap();
}
```

`msg.sender` here is the pool (correct — the pool calls the extension). `sender` is whatever address the pool passes as the first argument to the hook. When a user calls `MetricOmmSimpleRouter.exactInput*()`, the router calls `pool.swap()`; the pool's `msg.sender` is the router, so the pool forwards the router's address as `sender` to the extension. The allowlist lookup becomes `allowedSwapper[pool][router]`, not `allowedSwapper[pool][actualUser]`.

This is the direct analog to the Sense Finance bug: just as `WstETHAdapter` bypassed `safeApprove()` because it inherited from `BaseAdapter` instead of `CropAdapter`, here the router-mediated swap path bypasses the identity check the allowlist was designed to enforce. The guard is configured and reachable, but the actor it evaluates is the wrong one.

Two concrete failure modes follow:

1. **Bypass (primary impact):** If the pool admin adds the router to the allowlist (a natural operational step to allow "normal" router usage), every unprivileged user can bypass the curated allowlist by routing through `MetricOmmSimpleRouter`.
2. **Lockout:** If the router is not on the allowlist, every allowed user who routes through the router is incorrectly blocked, breaking core swap functionality for the intended audience.

### Impact Explanation

A curated pool's swap allowlist is entirely ineffective for router-mediated swaps. Any user — including those explicitly excluded by the pool admin — can trade on a restricted pool by calling `MetricOmmSimpleRouter` instead of the pool directly. This constitutes a broken core pool functionality and an admin-boundary break: the pool admin's curation policy is bypassed by an unprivileged path (the public router).

### Likelihood Explanation

`MetricOmmSimpleRouter` is the primary user-facing entrypoint for swaps. Any user aware of the router can trivially exploit this. No special privileges, flash loans, or multi-step setup are required — a single router call suffices.

### Recommendation

The pool should pass the original transaction initiator (the end-user's address) as `sender` to extension hooks, not `msg.sender` (the router). One approach: require callers to supply a `sender` override in the swap parameters, validated against `msg.sender` or a trusted forwarder registry. Alternatively, `SwapAllowlistExtension` should also check `allowedSwapper[pool][msg.sender_of_pool_call]` by reading it from a trusted source, or the router should be explicitly excluded from acting as a transparent forwarder for allowlisted pools.

### Proof of Concept

1. Pool admin deploys a pool with `SwapAllowlistExtension` configured as a `beforeSwap` hook.
2. Pool admin calls `setAllowedToSwap(pool, alice, true)` — only Alice is allowed.
3. Bob (not on the allowlist) calls `MetricOmmSimpleRouter.exactInputSingle(pool, ...)`.
4. The router calls `pool.swap(...)`. Inside the pool, `msg.sender == router`.
5. The pool calls `extension.beforeSwap(router, ...)`.
6. The extension checks `allowedSwapper[pool][router]`. If the router was added to the allowlist (or `allowAllSwappers[pool] == true`), the check passes and Bob's swap executes — bypassing the allowlist entirely.
7. Bob receives tokens from a pool he was explicitly excluded from trading on. [1](#0-0) [2](#0-1) [3](#0-2)
