### Title
SwapAllowlistExtension Gates on Router Address Instead of End User, Enabling Full Allowlist Bypass - (`metric-periphery/contracts/extensions/SwapAllowlistExtension.sol`)

### Summary

`SwapAllowlistExtension.beforeSwap` checks the `sender` argument, which is `msg.sender` of the pool's `swap` call. When users route through `MetricOmmSimpleRouter`, the router becomes `sender`. A pool admin who allowlists the router to enable router-mediated swaps inadvertently opens the allowlist to every user on the network.

### Finding Description

`MetricOmmPool.swap` passes `msg.sender` as the `sender` argument to `_beforeSwap`: [1](#0-0) 

`ExtensionCalling._beforeSwap` forwards that value unchanged to every configured extension: [2](#0-1) 

`SwapAllowlistExtension.beforeSwap` then gates on that `sender` value: [3](#0-2) 

`MetricOmmSimpleRouter.exactInputSingle` (and every other router entry point) calls `pool.swap(...)` directly, making the router itself `msg.sender` to the pool: [4](#0-3) 

The same pattern holds for `exactInput` (all hops) and `exactOutput` (all recursive hops): [5](#0-4) 

**Consequence:** The extension never sees the real end user. It sees the router address. A pool admin who wants to support router-mediated swaps must call `setAllowedToSwap(pool, router, true)`. Once the router is allowlisted, the check `allowedSwapper[pool][router]` returns `true` for every caller regardless of who they are, because the router is a shared, permissionless contract. Every user on the network can now swap on a pool that was supposed to be curated.

The inverse failure also exists: if the admin does **not** allowlist the router, legitimately allowlisted users are silently blocked from using the router even though they are individually permitted, because the extension sees `sender = router` and rejects it.

### Impact Explanation

A pool configured with `SwapAllowlistExtension` and the router allowlisted has its entire access-control policy nullified. Any address can call `router.exactInputSingle` or `router.exactInput` and trade on the pool. This is a direct policy bypass on curated pools. Depending on the pool's purpose (KYC-gated, institutional-only, rate-limited), the consequence ranges from regulatory violation to direct LP losses from unrestricted arbitrage or sandwich attacks by actors the pool was designed to exclude.

### Likelihood Explanation

The `MetricOmmSimpleRouter` is the canonical, documented periphery entry point for swaps. Pool admins who deploy a curated pool and want their allowlisted users to be able to use the standard router will naturally add the router to the allowlist. The misconfiguration is the expected, intuitive action. No privileged attacker role is required; any EOA can exploit it once the router is allowlisted.

### Recommendation

The extension must gate on the actual end user, not the immediate pool caller. Two viable approaches:

1. **Extension-data attestation**: Require the router to embed the original `msg.sender` in `extensionData` for each hop, and have `SwapAllowlistExtension.beforeSwap` decode and check that address instead of `sender`. The extension should reject calls where `sender` is a known router but no valid user attestation is present.

2. **Separate router allowlist slot**: Add a second mapping `allowedRouter` and, when `sender` is an allowlisted router, require the extension to receive and verify the real user address from `extensionData`.

Either approach must be applied consistently across `exactInputSingle`, `exactInput`, and `exactOutput` (including recursive callback hops).

### Proof of Concept

```
1. Deploy pool with SwapAllowlistExtension in BEFORE_SWAP_ORDER.
2. Pool admin calls setAllowedToSwap(pool, alice, true)       // alice is the intended user
3. Pool admin calls setAllowedToSwap(pool, router, true)      // to let alice use the router
4. Attacker (bob, not allowlisted) calls:
       router.exactInputSingle({pool: pool, recipient: bob, ...})
5. Router calls pool.swap(bob, ...) with msg.sender = router
6. Extension receives sender = router
7. allowedSwapper[pool][router] == true  →  check passes
8. Bob's swap executes on the curated pool despite never being allowlisted.
``` [3](#0-2) [6](#0-5) [1](#0-0)
