### Title
`SwapAllowlistExtension` Checks Wrong Actor (`sender` = Router) Instead of Actual User, Enabling Allowlist Bypass Through `MetricOmmSimpleRouter` — (`metric-periphery/contracts/extensions/SwapAllowlistExtension.sol`)

---

### Summary

`SwapAllowlistExtension.beforeSwap` gates swaps by checking `sender`, which is `msg.sender` of the pool call. When a user routes through `MetricOmmSimpleRouter`, `sender` is the router address, not the actual user. If the pool admin allowlists the router to enable router-based trading, the per-user allowlist is completely bypassed: every user, including explicitly non-allowlisted ones, can swap freely through the router.

---

### Finding Description

`MetricOmmPool.swap()` passes `msg.sender` as the `sender` argument to `_beforeSwap`, which forwards it to every configured extension:

```solidity
// MetricOmmPool.sol – swap()
_beforeSwap(
    msg.sender,   // ← sender = immediate caller (router when routed)
    recipient,
    ...
);
```

`ExtensionCalling._beforeSwap` encodes this value and calls each extension in order:

```solidity
// ExtensionCalling.sol
abi.encodeCall(IMetricOmmExtensions.beforeSwap,
    (sender, recipient, zeroForOne, ...))
```

`SwapAllowlistExtension.beforeSwap` then checks only `sender` (the first parameter) against its per-pool allowlist, ignoring `recipient` entirely:

```solidity
// SwapAllowlistExtension.sol
function beforeSwap(address sender, address, bool, int128, uint128, uint256, uint128, uint128, bytes calldata)
    external view override returns (bytes4)
{
    if (!allowAllSwappers[msg.sender] && !allowedSwapper[msg.sender][sender]) {
        revert IMetricOmmPoolActions.NotAllowedToSwap();
    }
    return IMetricOmmExtensions.beforeSwap.selector;
}
```

`MetricOmmSimpleRouter` is a supported periphery path that calls `pool.swap()` directly, making itself `msg.sender` to the pool. The pool's swap callback is issued back to the router:

```solidity
// MetricOmmPool.swap()
IMetricOmmSwapCallback(msg.sender).metricOmmSwapCallback(amount0Delta, amount1Delta, callbackData);
```

**Bypass path:**

1. Pool admin configures `SwapAllowlistExtension` and allowlists User A (trusted) but not User B (untrusted).
2. Pool admin also allowlists the router address so that User A can trade through the official UI/router.
3. User B calls `MetricOmmSimpleRouter.swap(...)`.
4. Router calls `pool.swap(...)` → `msg.sender` in pool = router.
5. Extension checks `allowedSwapper[pool][router]` = `true` → passes.
6. User B's swap executes successfully, bypassing the per-user restriction.

The pool admin cannot simultaneously:
- Allow router-based trading (requires allowlisting the router), and
- Enforce per-user allowlist restrictions (requires checking the actual user, not the router).

These two goals are mutually exclusive under the current design. The impact gate explicitly identifies this class: *"Allowlist path: deposit/swap allowlist checks must cover the exact actor/action intended and cannot be bypassed through router."*

---

### Impact Explanation

Any user explicitly excluded from a curated pool's swap allowlist can bypass the restriction by routing through `MetricOmmSimpleRouter`. This breaks the core curation invariant of the pool: the pool admin's intent to restrict trading to specific addresses is nullified. Depending on the pool's purpose (e.g., compliance-gated, institutional-only, or KYC-restricted pools), this allows unauthorized parties to trade, potentially draining liquidity or manipulating pool state in ways the admin intended to prevent.

**Severity: Medium** — Broken core pool functionality (allowlist policy bypass) via a supported public periphery path; no direct arithmetic loss per swap, but the configured protection is rendered ineffective.

---

### Likelihood Explanation

**Medium.** The bypass requires the pool admin to allowlist the router. This is a natural and expected configuration for any curated pool that wants to support the official periphery UI. Pool admins who configure both per-user restrictions and router support will unknowingly expose the bypass. The `SwapAllowlistExtension` NatSpec ("Gates `swap` by swapper address") does not warn that router intermediaries collapse all users into a single identity.

---

### Recommendation

`SwapAllowlistExtension.beforeSwap` should check the actual economic actor, not the immediate caller. Two options:

1. **Check `recipient` instead of `sender`**: The recipient is the address that receives tokens from the swap. When the router is used, the router typically sets `recipient` to the actual user. This is a partial fix (recipient could still be a contract).

2. **Decode actual user from `extensionData`**: Require the router to embed the originating user address in `extensionData`, and have the extension decode and check it. This requires a coordinated change to the router and extension.

3. **Document the limitation explicitly**: If the design intent is that `sender` = immediate caller, document that curated pools using `SwapAllowlistExtension` must not allowlist any intermediary contract (including the official router), and that users must call the pool directly.

The cleanest fix matching the invariant ("the same allowlist policy regardless of which supported public entrypoint reaches it") is option 2.

---

### Proof of Concept

```solidity
// Assume pool is deployed with SwapAllowlistExtension configured.
// Pool admin allowlists userA and the router, but NOT userB.

swapExtension.setAllowedToSwap(address(pool), address(router), true);  // router allowlisted
swapExtension.setAllowedToSwap(address(pool), userA, true);             // userA allowlisted
// userB is NOT allowlisted

// Direct call by userB → correctly reverts
vm.prank(userB);
vm.expectRevert(IMetricOmmPoolActions.NotAllowedToSwap.selector);
pool.swap(userB, false, int128(1000), type(uint128).max, "", "");

// Router call by userB → bypasses allowlist, succeeds
vm.prank(userB);
// router.swap() internally calls pool.swap() with msg.sender = router
// extension checks allowedSwapper[pool][router] = true → passes
router.swap(address(pool), false, int128(1000), type(uint128).max, userB, "");
// swap executes successfully — userB bypassed the allowlist
```
