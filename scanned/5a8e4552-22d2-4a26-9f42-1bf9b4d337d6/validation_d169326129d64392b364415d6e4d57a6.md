### Title
SwapAllowlistExtension Checks Immediate Pool Caller (Router) Instead of End User, Enabling Full Allowlist Bypass When Router Is Whitelisted - (`metric-periphery/contracts/extensions/SwapAllowlistExtension.sol`)

---

### Summary

`SwapAllowlistExtension.beforeSwap` gates swaps by checking the `sender` parameter, which the pool sets to `msg.sender` of the `pool.swap()` call. When `MetricOmmSimpleRouter` intermediates the swap, `sender` is the router address, not the end user. If a pool admin adds the router to the allowlist (the only way to let allowed users trade through the router), the check passes for **every** caller, completely defeating the allowlist.

---

### Finding Description

`MetricOmmPool.swap` passes its own `msg.sender` as the `sender` argument to `_beforeSwap`: [1](#0-0) 

`ExtensionCalling._beforeSwap` forwards that value unchanged to the extension: [2](#0-1) 

`SwapAllowlistExtension.beforeSwap` then checks whether that `sender` is on the per-pool allowlist: [3](#0-2) 

`MetricOmmSimpleRouter.exactInputSingle` (and every other router entry point) calls `pool.swap(...)` directly, making the router the `msg.sender` to the pool: [4](#0-3) 

The router never forwards the original user's address as `sender`; it only passes `recipient` (the output destination). The extension therefore sees `sender = router` on every router-mediated swap.

A pool admin who wants their allowlisted users to trade through the router must add the router address to the allowlist via `setAllowedToSwap(pool, router, true)`. Once the router is allowlisted, the check `allowedSwapper[pool][router]` is `true` for every call, so **any** address that calls `router.exactInputSingle / exactInput / exactOutputSingle / exactOutput` passes the guard regardless of whether that address is individually permitted. [5](#0-4) 

The admin faces an impossible choice: either block all router usage for their allowed users, or open the pool to every user on the network.

---

### Impact Explanation

A curated pool using `SwapAllowlistExtension` is intended to restrict trading to a known set of counterparties (e.g., KYC'd addresses, institutional partners). Once the router is allowlisted, any unprivileged address can execute swaps against the pool's liquidity at oracle prices, draining LP value through adverse selection or simply trading in a pool that was never meant to be public. This is a direct loss of LP principal and a complete failure of the pool's curation invariant.

---

### Likelihood Explanation

The `MetricOmmSimpleRouter` is the primary supported periphery entry point. Pool admins who deploy curated pools will naturally want their allowed users to benefit from multi-hop routing, slippage protection, and deadline enforcement that the router provides. Adding the router to the allowlist is the only mechanism available to them, making this a predictable operational step. No attacker capability beyond calling a public router function is required once the router is listed.

---

### Recommendation

The extension must gate on the **end user**, not the intermediate caller. Two complementary fixes:

1. **Pass the original user through the router.** Add an `originator` field to the `extensionData` bytes that the router populates with `msg.sender`. The extension can decode and check this field when `sender` is a known router address.

2. **Check `sender` and fall back to a decoded originator.** In `beforeSwap`, if `sender` is a registered router, extract the real user from `extensionData` and apply the allowlist check against that address instead.

Alternatively, document explicitly that adding any router to the allowlist opens the pool to all users, and provide a separate `RouterAllowlistExtension` that checks the decoded originator from `extensionData`.

---

### Proof of Concept

**Setup:**
- Pool deployed with `SwapAllowlistExtension` configured as `beforeSwap` extension.
- Pool admin calls `swapExtension.setAllowedToSwap(pool, alice, true)` — only Alice is meant to trade.
- Pool admin calls `swapExtension.setAllowedToSwap(pool, address(router), true)` — to let Alice use the router.

**Attack (Bob, not on allowlist):**
```solidity
// Bob calls the router directly
router.exactInputSingle(ExactInputSingleParams({
    pool:            address(pool),
    recipient:       bob,
    zeroForOne:      true,
    amountIn:        1_000e18,
    amountOutMinimum: 0,
    priceLimitX64:   0,
    deadline:        block.timestamp,
    tokenIn:         token0,
    extensionData:   ""
}));
```

**Trace:**
1. `router.exactInputSingle` → `pool.swap(bob, true, 1000e18, 0, "", "")` with `msg.sender = router`.
2. Pool calls `_beforeSwap(router, bob, ...)`.
3. Extension checks `allowedSwapper[pool][router]` → `true` → passes.
4. Swap executes at oracle price; Bob receives token1 output.

Bob successfully trades in a pool he was never permitted to access. The allowlist is fully bypassed. [3](#0-2) [6](#0-5) [1](#0-0)
