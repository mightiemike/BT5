### Title
Router-Mediated Swap Passes Router Address as `sender` to `SwapAllowlistExtension::beforeSwap`, Enabling Allowlist Bypass — (`metric-periphery/contracts/extensions/SwapAllowlistExtension.sol`)

---

### Summary

`SwapAllowlistExtension::beforeSwap` gates swaps by checking `allowedSwapper[pool][sender]`, where `sender` is the immediate caller of `MetricOmmPool::swap`. When a user swaps through `MetricOmmSimpleRouter`, the pool's `msg.sender` is the **router**, not the original user. The hook therefore checks whether the **router** is allowlisted, not the user. If the router is allowlisted for a pool (a natural admin action to enable router-mediated swaps), any unprivileged user can bypass the per-user allowlist by routing through it.

---

### Finding Description

**Call chain — direct swap:**

```
User → MetricOmmPool::swap()
         _beforeSwap(msg.sender=User, ...)
           SwapAllowlistExtension::beforeSwap(sender=User)
             allowedSwapper[pool][User]  ← correct check
```

**Call chain — router swap:**

```
User → MetricOmmSimpleRouter::exactInputSingle()
         IMetricOmmPoolActions(pool).swap(recipient, ...)
           MetricOmmPool::swap()  [msg.sender = Router]
             _beforeSwap(msg.sender=Router, ...)
               SwapAllowlistExtension::beforeSwap(sender=Router)
                 allowedSwapper[pool][Router]  ← checks Router, not User
```

In `MetricOmmPool::swap`, the pool passes `msg.sender` as the `sender` argument to `_beforeSwap`: [1](#0-0) 

`ExtensionCalling::_beforeSwap` forwards that value verbatim to the extension: [2](#0-1) 

`SwapAllowlistExtension::beforeSwap` then checks `allowedSwapper[msg.sender][sender]`, where `msg.sender` is the pool and `sender` is whatever was passed — the router when called via the router: [3](#0-2) 

The router calls `pool.swap()` directly with no mechanism to forward the original user's identity: [4](#0-3) 

---

### Impact Explanation

A pool admin who wants to allow router-mediated swaps for their allowlisted users has only one option: allowlist the router address. Once the router is allowlisted, **every user** — including those explicitly not in the allowlist — can swap through the router and pass the check. The per-user allowlist is completely defeated. The invariant "a curated pool must enforce the same allowlist policy regardless of which supported public entrypoint reaches it" is broken.

---

### Likelihood Explanation

Any pool that:
1. Configures `SwapAllowlistExtension` as a `beforeSwap` hook, and
2. Allowlists the router (a natural and expected admin action to support the official periphery)

is immediately vulnerable. This is not a misconfiguration edge case — it is the only way to make the allowlist work with the router at all, and it silently opens the gate to all users.

---

### Recommendation

The extension must receive the **original user's identity**, not the immediate caller. Options:

- Pass `tx.origin` as an additional parameter (fragile, breaks contract-to-contract flows).
- Require the router to forward the original sender explicitly in `extensionData`, and have the extension decode and verify it (requires router cooperation and extension-side validation).
- Have the router store the original sender in transient storage and expose a view function the extension can call back to retrieve it (similar to how the router already uses transient storage for callback context).

The cleanest fix is the third option: the router writes `msg.sender` into transient storage before calling `pool.swap()`, and `SwapAllowlistExtension::beforeSwap` reads it back via a known router interface when `msg.sender` (the pool's caller) is a recognized router.

---

### Proof of Concept

1. Deploy a pool with `SwapAllowlistExtension` as `beforeSwap` hook.
2. Pool admin calls `setAllowedToSwap(pool, alice, true)` — only Alice is allowed.
3. Pool admin calls `setAllowedToSwap(pool, router, true)` — to enable router-mediated swaps for Alice.
4. Bob (not allowlisted) calls `MetricOmmSimpleRouter::exactInputSingle(...)` targeting the pool.
5. The pool calls `beforeSwap(sender=router, ...)`.
6. The check `allowedSwapper[pool][router]` returns `true`.
7. Bob's swap succeeds — the allowlist is bypassed.

Assert: `allowedSwapper[pool][bob] == false` yet Bob's swap completes without revert.
