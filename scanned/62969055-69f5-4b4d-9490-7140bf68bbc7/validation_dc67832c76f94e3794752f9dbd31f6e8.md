### Title
`SwapAllowlistExtension.beforeSwap` checks the router's address instead of the end user's address, allowing any caller to bypass the swap allowlist on curated pools via `MetricOmmSimpleRouter` — (`metric-periphery/contracts/extensions/SwapAllowlistExtension.sol`)

---

### Summary

`SwapAllowlistExtension.beforeSwap` gates swaps by checking the `sender` argument, which the pool sets to `msg.sender` of `pool.swap()`. When a user routes through `MetricOmmSimpleRouter`, the pool's `msg.sender` is the router contract, not the end user. The extension therefore checks whether the **router** is allowlisted, not whether the **end user** is allowlisted. If the pool admin allowlists the router (the only way to permit router-mediated swaps on a curated pool), the allowlist is completely open to any caller who routes through the router.

---

### Finding Description

**Call chain producing the wrong actor:**

`MetricOmmPool.swap()` captures `msg.sender` and forwards it as `sender` to `_beforeSwap()`:

```solidity
// MetricOmmPool.sol:230-240
_beforeSwap(
    msg.sender,   // ← direct caller of pool.swap(), i.e. the router
    recipient,
    ...
);
```

`ExtensionCalling._beforeSwap()` passes that value unchanged to every configured extension:

```solidity
// ExtensionCalling.sol:162-175
abi.encodeCall(
    IMetricOmmExtensions.beforeSwap,
    (sender, recipient, ...)   // sender == router address
)
```

`SwapAllowlistExtension.beforeSwap()` then checks that value against the per-pool allowlist:

```solidity
// SwapAllowlistExtension.sol:37
if (!allowAllSwappers[msg.sender] && !allowedSwapper[msg.sender][sender]) {
    revert IMetricOmmPoolActions.NotAllowedToSwap();
}
```

Here `msg.sender` is the pool and `sender` is whoever called `pool.swap()`. When the user enters through `MetricOmmSimpleRouter.exactInputSingle()`:

```solidity
// MetricOmmSimpleRouter.sol:72-80
IMetricOmmPoolActions(params.pool).swap(
    params.recipient,
    params.zeroForOne,
    ...
    params.extensionData
);
```

the pool sees `msg.sender = address(router)`, so the extension evaluates `allowedSwapper[pool][router]`, not `allowedSwapper[pool][user]`.

**The two broken outcomes:**

| Pool admin configuration | Result |
|---|---|
| Router **not** allowlisted | Allowlisted users cannot swap through the router; the curated pool is unusable via the supported periphery path |
| Router **allowlisted** (to fix the above) | Every address on the network can bypass the allowlist by calling `router.exactInputSingle()` |

There is no configuration that simultaneously permits router-mediated swaps for allowlisted users and blocks non-allowlisted users.

The same wrong-actor binding applies to `exactInput`, `exactOutputSingle`, and `exactOutput` — all four router entry points call `pool.swap()` directly, making the router the `sender` in every case.

---

### Impact Explanation

A pool admin who deploys a curated pool with `SwapAllowlistExtension` and allowlists the router (the natural step to support the official periphery) inadvertently opens the pool to all callers. Any non-allowlisted address can execute swaps against the pool's oracle-anchored liquidity by routing through `MetricOmmSimpleRouter`, receiving tokens at the pool's bid/ask price. The allowlist guard — the sole access-control mechanism on the swap path — provides no protection. LP funds are exposed to unrestricted toxic flow that the pool admin explicitly intended to block.

---

### Likelihood Explanation

The `MetricOmmSimpleRouter` is the primary user-facing swap interface. Any pool admin who wants their allowlisted users to be able to use the standard router must allowlist the router address. This is the expected operational path, not an edge case. The bypass is therefore reachable by any unprivileged caller on every curated pool that has been configured to support router-mediated swaps.

---

### Recommendation

The extension must check the economically relevant actor — the end user — not the intermediary contract. Two viable approaches:

1. **Pass the original initiator through the router.** The router already stores `msg.sender` in transient storage as the payer (`_setNextCallbackContext(..., msg.sender, ...)`). The pool could expose a mechanism for the router to attest the original caller, or the extension could read a router-provided attestation from `extensionData`.

2. **Check `recipient` instead of `sender` for router flows.** The `recipient` parameter is the address that receives output tokens and is set by the end user. For direct swaps `recipient` equals the user; for router swaps the user controls `recipient`. This is imperfect (recipient ≠ payer) but closer to the intended gate.

3. **Require direct pool calls for allowlisted pools.** Document that pools using `SwapAllowlistExtension` must not allowlist the router, and that allowlisted users must call `pool.swap()` directly. This is operationally restrictive but avoids the wrong-actor binding entirely.

The cleanest fix is option 1: have the router encode the original `msg.sender` into `extensionData` and have the extension decode and verify it when present.

### Proof of Concept

```solidity
// SPDX-License-Identifier: MIT
pragma solidity ^0.8.35;

// Setup:
//   1. Deploy pool with SwapAllowlistExtension configured on beforeSwap.
//   2. Pool admin allowlists the router: setAllowedToSwap(pool, address(router), true)
//   3. Pool admin does NOT allowlist attacker: allowedSwapper[pool][attacker] == false
//
// Attack:
//   attacker (not allowlisted) calls router.exactInputSingle(pool, ...)
//   → router calls pool.swap(recipient, ...)
//   → pool calls extension.beforeSwap(sender=router, ...)
//   → extension checks allowedSwapper[pool][router] == true  ✓
//   → swap executes; attacker receives tokens
//
// Expected: revert NotAllowedToSwap()
// Actual:   swap succeeds

function test_allowlistBypassViaRouter() public {
    // attacker is not allowlisted
    assertFalse(swapExtension.isAllowedToSwap(address(pool), attacker));

    // pool admin allowlists the router so legitimate users can use it
    vm.prank(poolAdmin);
    swapExtension.setAllowedToSwap(address(pool), address(router), true);

    // attacker routes through the router — allowlist check passes on router address
    vm.prank(attacker);
    router.exactInputSingle(
        IMetricOmmSimpleRouter.ExactInputSingleParams({
            pool: address(pool),
            recipient: attacker,
            tokenIn: address(token0),
            zeroForOne: true,
            amountIn: 1000,
            amountOutMinimum: 0,
            priceLimitX64: type(uint128).max,
            deadline: block.timestamp,
            extensionData: ""
        })
    );
    // swap succeeds — attacker bypassed the allowlist
}
