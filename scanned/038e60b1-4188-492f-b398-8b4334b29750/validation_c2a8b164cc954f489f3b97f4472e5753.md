### Title
`AnchoredPriceProvider` Uses L1-Only Staleness Check on L2 Deployments, Causing Swap Failures - (`smart-contracts-poc/contracts/AnchoredPriceProvider.sol`)

---

### Summary

`AnchoredPriceProvider` is described as "the one standard provider for public pools" and is deployed on all supported chains including Base (L2). However, its `_isStale` function uses an L1-only check that unconditionally rejects any oracle `refTime` ahead of `block.timestamp`. On L2 networks, sequencer clock skew routinely causes oracle `refTime` to be slightly ahead of `block.timestamp`, making the staleness check incorrectly classify valid, fresh prices as stale. This causes `getBidAndAskPrice()` to revert with `FeedStalled()`, breaking all swaps through any pool using this provider.

---

### Finding Description

The protocol explicitly separates L1 and L2 staleness logic. `PriceProvider.sol` (L1) and `PriceProviderL2.sol` (L2) are distinct contracts. `PriceProviderL2` adds a `FUTURE_TOLERANCE` immutable and a four-argument `_isStale` that tolerates oracle `refTime` up to `FUTURE_TOLERANCE` seconds ahead of `block.timestamp`: [1](#0-0) 

`AnchoredPriceProvider`, however, has no L2 variant. Its `_isStale` is explicitly annotated `(L1)` and hard-rejects any future `refTime`: [2](#0-1) 

The three-argument `_isStale` at line 228 (`if (refTime > nowTs) return true`) is the broken guard. When the oracle publishes a price with `refTime` one or two seconds ahead of the L2 sequencer's `block.timestamp` (normal clock skew on Base), `_readLeg` returns `ok = false`: [3](#0-2) 

`_getBidAndAskPrice` then returns the `(0, type(uint128).max)` sentinel: [4](#0-3) 

`getBidAndAskPrice()` reverts with `FeedStalled()`: [5](#0-4) 

`MetricOmmPool._getBidAndAskPriceX64()` catches and re-reverts with `PriceProviderFailed`, blocking `swap()`: [6](#0-5) 

There is no `AnchoredPriceProviderL2.sol` in the codebase. The protocol's supported chains include Base (L2): [7](#0-6) 

---

### Impact Explanation

Every swap through a pool whose price provider is an `AnchoredPriceProvider` fails during any period of sequencer clock skew on Base. This is broken core pool functionality — the swap flow is completely unusable. LPs cannot exit via swap-based routes, and traders cannot execute. This matches the allowed impact gate: "Broken core pool functionality causing loss of funds or unusable withdraw/swap/liquidity flows."

---

### Likelihood Explanation

Base (L2) is an explicitly listed supported chain. Sequencer clock skew causing oracle `refTime > block.timestamp` is a documented, recurring condition on optimistic rollups — it is precisely why `PriceProviderL2` was written with `FUTURE_TOLERANCE`. Any oracle update published during a skew window triggers the failure. No privileged action or malicious setup is required; any ordinary swap call suffices.

---

### Recommendation

Create an `AnchoredPriceProviderL2` variant that mirrors `PriceProviderL2`'s four-argument `_isStale` with a `FUTURE_TOLERANCE` immutable, replacing the hard `if (refTime > nowTs) return true` guard with:

```solidity
if (refTime > nowTs) {
    return (refTime - nowTs) > FUTURE_TOLERANCE;
}
```

Alternatively, add a `futureTolerance` immutable directly to `AnchoredPriceProvider` and select the appropriate check at construction time based on the target chain, consistent with how `PriceProviderL2` already handles this.

---

### Proof of Concept

1. Deploy `AnchoredPriceProvider` on Base with a Pyth Lazer or Chainlink Data Streams feed.
2. The oracle publishes a price update with `refTime = block.timestamp + 2` (two-second sequencer clock skew, within normal Base operating range).
3. Call `pool.swap(...)`.
4. `_getBidAndAskPriceX64` calls `provider.getBidAndAskPrice()`.
5. Inside `_readLeg`, `_isStale(refTime=T+2, nowTs=T, MAX_REF_STALENESS)` hits `if (refTime > nowTs) return true` → stale.
6. `_readLeg` returns `ok = false`; `_getBidAndAskPrice` returns `(0, type(uint128).max)`.
7. `getBidAndAskPrice()` reverts `FeedStalled()`.
8. `swap()` reverts `PriceProviderFailed(...)`.
9. All swaps are blocked for the duration of the clock skew window.

```text
        uint256 refTime,
        uint256 nowTs,
        uint256 maxDelta,
        uint256 futureTol

        if (refTime > nowTs) {
            // refTime in the future: tolerate only within futureTol
            return (refTime - nowTs) > futureTol;
        }

        // refTime in the past or equal: check age

    }
```

```text
    function _readLeg(bytes32 feedId)
        internal returns (uint256 mid, uint256 spreadBps, uint256 refTime, bool ok)
    {
        (mid, spreadBps, , refTime) = IPricedOracle(address(offchainOracle)).price(feedId, msg.sender);

        // Stale reference → not ok. Clamping to a stale anchor is the one false-safety case.
        if (_isStale(refTime, block.timestamp, MAX_REF_STALENESS)) return (mid, spreadBps, refTime, false);

        // Basic validity — mid positive, spreadBps not the stalled/off-hours marker (the Chainlink oracle
        // writes spreadBps = ORACLE_BPS when an RWA market is closed).
        if (mid == 0 || spreadBps >= ORACLE_BPS) return (mid, spreadBps, refTime, false);

        // Per-leg price guard.
        (uint128 guardMin, uint128 guardMax) = offchainOracle.priceGuard(feedId);
        guardMax = guardMax == 0 ? type(uint128).max : guardMax;
        if (mid < guardMin || mid > guardMax) return (mid, spreadBps, refTime, false);

        ok = true;
    }
```
