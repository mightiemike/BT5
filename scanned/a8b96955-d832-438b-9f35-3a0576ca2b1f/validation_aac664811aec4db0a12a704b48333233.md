### Title
Stranded Native ETH on Router Consumed by Subsequent WETH Swap Payments — (`metric-periphery/contracts/base/PeripheryPayments.sol`)

---

### Summary

`PeripheryPayments.pay()` uses `address(this).balance` unconditionally when settling a WETH payment. It does not distinguish between ETH sent in the **current** transaction (`msg.value`) and ETH left over from a **prior** transaction. Any ETH stranded on the router from a previous call is silently consumed as part of the next user's WETH payment, causing the stranded-ETH owner to lose those funds.

---

### Finding Description

`PeripheryPayments.pay()` contains the following branch for WETH payments: [1](#0-0) 

When `0 < nativeBalance < value`, the router:
1. Wraps `nativeBalance` ETH into WETH and forwards it to the pool.
2. Pulls only `value - nativeBalance` WETH from the actual payer.

The pool receives the full `value` in WETH, but the payer only contributed `value - nativeBalance`. The difference was taken from whatever ETH happened to be sitting on the router — regardless of who deposited it.

**How ETH becomes stranded:**

`receive()` blocks plain ETH transfers from non-WETH addresses: [2](#0-1) 

However, `exactInputSingle` (and all other router entry points) are `payable`: [3](#0-2) 

ETH sent as `msg.value` with a payable call bypasses `receive()` entirely and is held by the contract. If a user sends `msg.value = 150` for a swap that only needs `100` ETH worth of WETH, the remaining `50` ETH stays on the router after the callback. Without an explicit `refundETH()` call, it is permanently stranded.

**Exploitation path:**

1. **Victim (User A):** calls `exactInputSingle` with `tokenIn = WETH`, `amountIn = 100`, `msg.value = 150`. The callback wraps and forwards 100 ETH; 50 ETH remains on the router. User A does not call `refundETH()`.
2. **Beneficiary (User B):** calls `exactInputSingle` with `tokenIn = WETH`, `amountIn = 100`, `msg.value = 0`. In the callback, `pay(WETH, userB, pool, 100)` sees `nativeBalance = 50`. It wraps the 50 stranded ETH, sends it to the pool, then pulls only 50 WETH from User B. User B's swap is fully settled; User A's 50 ETH is gone. [4](#0-3) 

---

### Impact Explanation

Direct loss of user principal. The stranded-ETH owner (User A) loses ETH they sent to the router. The loss is bounded by the stranded amount but is permanent and requires no privileged access to trigger. The beneficiary (User B) receives a subsidized swap at User A's expense.

---

### Likelihood Explanation

ETH stranding is a realistic user error in Uniswap-style routers: users frequently send `msg.value` with WETH swaps (to avoid a separate WETH-wrap step) and omit `refundETH()` from their multicall. Any subsequent WETH swap by any other user will silently drain the stranded balance. An attacker can also monitor the mempool or router balance and time their WETH swap to capture stranded ETH opportunistically.

---

### Recommendation

Track only the ETH that arrived in the **current** call. One approach: snapshot `address(this).balance - msg.value` at entry and treat only `msg.value` as spendable native ETH in `pay()`. Alternatively, pass the spendable ETH amount explicitly through the callback context rather than reading the live contract balance.

---

### Proof of Concept

```solidity
// Foundry integration test sketch
function test_strandedEthConsumedByWethSwap() public {
    // 1. User A swaps with excess msg.value, forgets refundETH()
    vm.prank(userA);
    router.exactInputSingle{value: 150e18}(
        ExactInputSingleParams({tokenIn: WETH, amountIn: 100e18, ...})
    );
    // Router now holds 50e18 stranded ETH

    assertEq(address(router).balance, 50e18);

    // 2. User B swaps WETH with no msg.value
    uint256 wethBefore = IERC20(WETH).balanceOf(userB);
    vm.prank(userB);
    router.exactInputSingle{value: 0}(
        ExactInputSingleParams({tokenIn: WETH, amountIn: 100e18, ...})
    );

    // User B only spent 50 WETH (not 100), stranded ETH covered the rest
    assertEq(wethBefore - IERC20(WETH).balanceOf(userB), 50e18);

    // Router balance is now 0 — User A's 50 ETH is gone
    assertEq(address(router).balance, 0);
}
```
