The vulnerability is real. Here is the complete trace:

**Execution path:**

1. `attacker` calls `router.exactInputSingle(...)` — `msg.sender` in the router = `attacker`
2. Router calls `pool.swap(recipient, ...)` — `msg.sender` in the pool = `router`
3. Pool's `swap` calls `_beforeSwap(msg.sender, ...)` → `_beforeSwap(router, ...)` [1](#0-0) 
4. `_beforeSwap` encodes `sender = router` and calls `extension.beforeSwap(sender=router, ...)` [2](#0-1) 
5. In `SwapAllowlistExtension.beforeSwap`: `msg.sender = pool`, `sender = router` → check is `allowedSwapper[pool][router]` — passes if router is allowlisted [3](#0-2) 

The `sender` parameter in `beforeSwap` is always the **immediate caller of `pool.swap`** (the router), never the original transaction initiator. The extension has no access to the original `msg.sender` chain.

### Title
SwapAllowlistExtension Checks Router Address Instead of Originating User, Allowing Any Attacker to Bypass Per-User Swap Allowlist via Router — (`metric-periphery/contracts/extensions/SwapAllowlistExtension.sol`)

### Summary
`SwapAllowlistExtension.beforeSwap` checks `allowedSwapper[pool][sender]` where `sender` is the address that called `pool.swap`. When the router is used, `sender` is always the router address, not the originating user. If the router is allowlisted, every user — including explicitly blocked attackers — can bypass the per-user allowlist by routing through `MetricOmmSimpleRouter`.

### Finding Description
`MetricOmmPool.swap` passes `msg.sender` as the `sender` argument to `_beforeSwap`:

```solidity
// MetricOmmPool.sol:230-240
_beforeSwap(
  msg.sender,   // <-- this is the router, not the original user
  recipient,
  ...
);
```

`SwapAllowlistExtension.beforeSwap` then checks:

```solidity
// SwapAllowlistExtension.sol:37
if (!allowAllSwappers[msg.sender] && !allowedSwapper[msg.sender][sender]) {
```

Here `msg.sender` is the pool and `sender` is the router. The check resolves to `allowedSwapper[pool][router]`. The original transaction initiator (the attacker) is never checked.

A pool admin who wants to allow router-based swaps while restricting direct swaps — or who allowlists the router as a trusted intermediary — inadvertently grants swap access to every user of that router, including explicitly non-allowlisted addresses.

### Impact Explanation
Any address not in `allowedSwapper` can execute swaps on a pool that has `SwapAllowlistExtension` configured, as long as the router is allowlisted. The attacker receives output tokens from the pool. Pool curation (the core purpose of the extension) is completely broken. This is a direct loss of the pool admin's access-control guarantee and enables unauthorized fund flows from the pool to the attacker.

**Severity: High** — unauthorized swap executes, attacker receives output tokens, pool curation invariant is broken for all non-allowlisted users.

### Likelihood Explanation
The router is the standard, documented entry point for swaps. A pool admin who wants to allow router-based swaps will naturally allowlist the router address. The extension provides no mechanism to distinguish the originating user when a router intermediary is used, so this bypass is triggered by normal, expected usage.

### Recommendation
`SwapAllowlistExtension.beforeSwap` should check the `recipient` (the address receiving output tokens) or require the pool to pass the originating user through `extensionData`. Alternatively, the extension should explicitly document that allowlisting any router grants swap access to all users of that router, and provide a separate `allowedRouter` mapping that is distinct from per-user allowlisting. The cleanest fix is to have the router pass the originating `msg.sender` in `extensionData` and have the extension decode and check it.

### Proof of Concept
```solidity
// Foundry integration test
function test_attackerBypassesAllowlistViaRouter() public {
    // Setup: pool with SwapAllowlistExtension
    // Admin allowlists the router, NOT the attacker
    swapExtension.setAllowedToSwap(address(pool), address(router), true);
    // attacker is NOT allowlisted

    // Attacker calls through router
    vm.prank(attacker);
    uint256 amountOut = router.exactInputSingle(
        IMetricOmmSimpleRouter.ExactInputSingleParams({
            pool: address(pool),
            tokenIn: address(token0),
            tokenOut: address(token1),
            zeroForOne: true,
            amountIn: 1000,
            amountOutMinimum: 0,
            recipient: attacker,
            deadline: block.timestamp + 1,
            priceLimitX64: 0,
            extensionData: ""
        })
    );
    // Swap succeeds despite attacker not being in allowedSwapper
    assertGt(amountOut, 0);
}
```
