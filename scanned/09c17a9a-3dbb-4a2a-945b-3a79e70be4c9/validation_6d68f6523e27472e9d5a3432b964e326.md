### Title
SwapAllowlistExtension Gates Router Identity Instead of End-User Identity, Allowing Any User to Bypass the Swap Allowlist - (File: metric-periphery/contracts/extensions/SwapAllowlistExtension.sol)

### Summary

`SwapAllowlistExtension.beforeSwap` checks the `sender` argument, which is the immediate caller of `pool.swap()`. When swaps are routed through `MetricOmmSimpleRouter`, `sender` is the router address, not the end user. A pool admin who adds the router to the allowlist (to let their approved users access the router) inadvertently opens the gate to every user, fully bypassing the allowlist guard.

### Finding Description

`MetricOmmPool.swap` passes `msg.sender` as the `sender` argument to `_beforeSwap`: [1](#0-0) 

`ExtensionCalling._beforeSwap` forwards that value unchanged to every configured extension: [2](#0-1) 

`SwapAllowlistExtension.beforeSwap` then checks `allowedSwapper[msg.sender][sender]`, where `msg.sender` is the pool and `sender` is whoever called `pool.swap()`: [3](#0-2) 

`MetricOmmSimpleRouter.exactInputSingle` (and every other router entry point) calls `pool.swap()` directly, making the router the `msg.sender` inside the pool: [4](#0-3) 

This creates an irreconcilable dilemma for the pool admin:

| Admin action | Effect |
|---|---|
| Allowlist individual users only (not the router) | Allowlisted users **cannot** use the router; their router calls revert |
| Allowlist the router (to enable router access for approved users) | **Every** user can bypass the allowlist via the router |

There is no configuration that simultaneously allows approved users to use the router and blocks unapproved users from doing the same. The extension's stated purpose — gating `swap` by swapper address — is structurally unachievable for router-mediated paths.

### Impact Explanation

A pool configured with `SwapAllowlistExtension` is typically a restricted pool (e.g., a private market-maker pool with tighter spreads or favorable oracle pricing). Once the pool admin adds the router to the allowlist (a natural step to let approved users access the standard periphery), any unprivileged user can call `MetricOmmSimpleRouter.exactInputSingle` and trade at the restricted pool's rates. LPs in that pool bear the cost of trades they intended to restrict, resulting in direct loss of LP value through unfavorable swap execution against a pool priced for a controlled counterparty set.

### Likelihood Explanation

The `MetricOmmSimpleRouter` is the standard user-facing entry point documented in the periphery. A pool admin who wants approved users to use the router will naturally add the router to the allowlist. The mistake is non-obvious: the admin believes they are enabling the router for their approved list, not for the entire public. The trigger requires no privileged escalation — any user with a valid ERC-20 balance can call the public router.

### Recommendation

The `beforeSwap` hook should receive and check the **original end-user identity**, not the immediate `pool.swap()` caller. Two concrete options:

1. **Pass the original sender through `extensionData`**: The router encodes `msg.sender` into `extensionData`; the extension decodes and checks it. This requires a convention between router and extension.
2. **Check both sender and a decoded originator**: The extension checks `allowedSwapper[pool][sender]` OR `allowedSwapper[pool][decodedOriginator]`, where `decodedOriginator` is extracted from `extensionData` when present.

Additionally, document clearly that allowlisting the router address grants access to all router users, so pool admins are not misled about the scope of that entry.

### Proof of Concept

```
Setup:
  - Deploy pool with SwapAllowlistExtension.
  - Pool admin calls setAllowedToSwap(pool, alice, true)   // alice is approved
  - Pool admin calls setAllowedToSwap(pool, router, true)  // to let alice use the router

Attack (Bob, not in allowlist):
  1. Bob calls MetricOmmSimpleRouter.exactInputSingle({pool: pool, ...})
  2. Router calls pool.swap(recipient, ...) → msg.sender inside pool = router
  3. _beforeSwap passes sender = router to SwapAllowlistExtension
  4. Extension checks allowedSwapper[pool][router] → true → no revert
  5. Bob's swap executes at the restricted pool's rates

Result: Bob, an unapproved user, swaps successfully in a pool the admin intended to restrict.
```
