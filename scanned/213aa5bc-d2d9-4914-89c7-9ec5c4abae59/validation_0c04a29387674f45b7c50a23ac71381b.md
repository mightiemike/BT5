### Title
SwapAllowlistExtension Gates the Router Address Instead of the Actual Swapper, Allowing Any User to Bypass the Allowlist — (`metric-periphery/contracts/extensions/SwapAllowlistExtension.sol`)

---

### Summary

`SwapAllowlistExtension.beforeSwap` checks the `sender` argument, which the pool sets to `msg.sender` of the `pool.swap()` call. When a user routes through `MetricOmmSimpleRouter`, `sender` is the router's address, not the actual user's address. If the pool admin allowlists the router (a necessary step to let legitimate users trade via the router), every unpermissioned user can bypass the allowlist by routing through the same public contract.

---

### Finding Description

`MetricOmmPool.swap` passes `msg.sender` as the `sender` argument to `_beforeSwap`: [1](#0-0) 

`ExtensionCalling._beforeSwap` forwards that value verbatim to every configured extension: [2](#0-1) 

`SwapAllowlistExtension.beforeSwap` then gates on `allowedSwapper[msg.sender][sender]`, where `msg.sender` is the pool and `sender` is whoever called `pool.swap()`: [3](#0-2) 

When a user calls `MetricOmmSimpleRouter.exactInput*`, the router becomes the direct caller of `pool.swap()`, so `sender = router`. The extension therefore checks `allowedSwapper[pool][router]`, not `allowedSwapper[pool][actualUser]`.

A pool admin who wants their allowlisted users to be able to trade via the router must add the router to the allowlist. Once the router is allowlisted, the check `allowedSwapper[pool][router] == true` passes for **every** caller of the router, regardless of whether that caller is individually permitted. The per-user allowlist is completely bypassed for all router-mediated swaps.

The `allowAllSwappers` short-circuit does not help: if it is `false` (the restrictive mode the allowlist is designed for), the bypass still works as long as the router address itself is in `allowedSwapper`. [4](#0-3) 

---

### Impact Explanation

A pool configured with `SwapAllowlistExtension` is a gated pool — only approved counterparties are supposed to trade against its liquidity. Once the router is allowlisted (a routine operational step), any address can execute swaps against the pool's oracle-anchored bins. An unpermissioned user can drain value from the pool at the oracle mid-price, causing direct loss of LP principal. This breaks the core security invariant of the allowlist extension and constitutes a Medium-to-High loss of user principal.

---

### Likelihood Explanation

The trigger requires the router to be in the allowlist, which is a normal operational action any pool admin would take to let their legitimate users trade via the standard periphery. The bypass itself requires no privilege — any EOA or contract can call `MetricOmmSimpleRouter`. The combination of a routine admin action and a zero-privilege attacker path makes this reliably reachable.

---

### Recommendation

The extension must resolve the actual end-user identity rather than the immediate caller. Two sound approaches:

1. **Pass the original user through the router.** Have `MetricOmmSimpleRouter` encode the real `msg.sender` in `extensionData` and have `SwapAllowlistExtension.beforeSwap` decode and gate on that value (with the pool verifying the router is the trusted forwarder).
2. **Gate on `recipient` instead of `sender`.** If the pool's design guarantees that the economic beneficiary of a swap is always `recipient`, gate on that field. This is simpler but must be verified against the pool's settlement logic.

Either way, the extension must check the identity that the pool admin actually intends to gate, not the intermediate contract that happens to call `pool.swap()`.

---

### Proof of Concept

```
Setup:
  pool = MetricOmmPool with SwapAllowlistExtension, allowAllSwappers[pool] = false
  allowedSwapper[pool][alice] = true          // alice is a legitimate user
  allowedSwapper[pool][router] = true         // admin adds router so alice can use it
  bob is NOT in allowedSwapper

Attack:
  bob calls MetricOmmSimpleRouter.exactInputSingle({pool: pool, ...})
    → router calls pool.swap(recipient=bob, ...)
    → pool calls extension.beforeSwap(sender=router, ...)
    → check: allowedSwapper[pool][router] == true  ✓
    → swap executes, bob receives token1 output
    → LP funds reduced without bob being an approved swapper
```

Bob successfully swaps against a pool he is not allowlisted for, extracting value from LPs who expected only approved counterparties.
