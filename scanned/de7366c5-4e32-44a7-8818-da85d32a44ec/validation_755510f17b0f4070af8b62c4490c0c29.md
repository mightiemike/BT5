## Analysis

**Call chain when using the router:**

```
User → MetricOmmSimpleRouter.exactInputSingle()
         → pool.swap(recipient, ..., extensionData)   [msg.sender = router]
              → _beforeSwap(msg.sender=router, ...)
                   → SwapAllowlistExtension.beforeSwap(sender=router, ...)
                        → allowedSwapper[pool][router]  ← checks ROUTER, not user
```

**Pause claim**: The `whenNotPaused` modifier fires before `_beforeSwap` is ever reached. A paused pool reverts immediately at the modifier. There is no pause bypass.

**Sender identity claim**: This is real. The pool passes `msg.sender` to `_beforeSwap`, and when the router calls `pool.swap()`, `msg.sender` is the router address. The extension then checks `allowedSwapper[pool][router]` — not the original user.

Two concrete broken outcomes:

1. **Allowlisted user cannot use the router**: Alice is allowlisted, router is not → Alice's router swap reverts. She must call `pool.swap()` directly, breaking the standard UX.
2. **Router allowlisting opens the gate to everyone**: Pool admin allowlists the router to enable router-mediated swaps → any user can bypass the per-user allowlist by routing through `MetricOmmSimpleRouter`.

Neither outcome requires trusted-owner assumptions or malicious pool setup. The router is a factory-validated public contract; any user can call it.

### Title
Router-mediated swaps corrupt swapper identity in `SwapAllowlistExtension::beforeSwap`, breaking per-user allowlist enforcement — (`metric-periphery/contracts/extensions/SwapAllowlistExtension.sol`)

### Summary

`SwapAllowlistExtension.beforeSwap` gates swaps by checking `allowedSwapper[pool][sender]`, where `sender` is the address that called `pool.swap()`. When `MetricOmmSimpleRouter` mediates the swap, `sender` is the router's address, not the original user's address. This makes the allowlist check meaningless for router-mediated swaps.

### Finding Description

`MetricOmmPool.swap` passes `msg.sender` as the first argument to `_beforeSwap`: [1](#0-0) 

`ExtensionCalling._beforeSwap` forwards that value unchanged as `sender` to every configured extension: [2](#0-1) 

`SwapAllowlistExtension.beforeSwap` then checks `allowedSwapper[msg.sender][sender]`, where `msg.sender` is the pool and `sender` is whoever called `pool.swap()`: [3](#0-2) 

`MetricOmmSimpleRouter.exactInputSingle` calls `pool.swap()` directly, making the router the `msg.sender` to the pool: [4](#0-3) 

The same applies to `exactInput` (all hops), `exactOutputSingle`, and `exactOutput`: [5](#0-4) 

There is no mechanism in the router to forward the original `msg.sender` as the `sender` argument to the pool. The router stores the original payer only in transient storage for the payment callback, not for the extension hook. [6](#0-5) 

### Impact Explanation

**Broken core functionality (Medium)**:

- **Scenario A — allowlisted user locked out of router**: Pool admin allowlists `alice` but not the router. Alice calls `exactInputSingle` → hook sees `sender = router` → `allowedSwapper[pool][router] = false` → revert. Alice must bypass the standard router and call `pool.swap()` directly.
- **Scenario B — router allowlisting opens the gate to all**: Pool admin allowlists the router address to enable router-mediated swaps. Now any user can call `exactInputSingle` and the hook passes, defeating the per-user allowlist entirely.

Both scenarios corrupt the invariant that the allowlist gates the actual economic actor performing the swap.

### Likelihood Explanation

Any pool that deploys `SwapAllowlistExtension` and expects users to interact via `MetricOmmSimpleRouter` is affected. The router is a public, factory-validated contract. No privileged access or special setup is required by the attacker.

### Recommendation

Pass the original caller's identity through the extension data or add a dedicated `originalSender` field to the hook signature. Alternatively, `SwapAllowlistExtension` should also check `allowedSwapper[pool][msg.sender_of_pool]` against a router registry, or the router should be extended to embed the original sender in `extensionData` in a tamper-proof way (e.g., signed or verified by the pool).

### Proof of Concept

```solidity
// Pool configured with SwapAllowlistExtension
// Pool admin allowlists alice, NOT the router
extension.setAllowedToSwap(pool, alice, true);

// Alice tries to swap via router — REVERTS
// hook sees sender = address(router), not alice
vm.prank(alice);
router.exactInputSingle(ExactInputSingleParams({
    pool: pool,
    recipient: alice,
    tokenIn: token0,
    zeroForOne: true,
    amountIn: 1e18,
    amountOutMinimum: 0,
    priceLimitX64: 0,
    deadline: block.timestamp,
    extensionData: ""
}));
// → NotAllowedToSwap()

// Scenario B: admin allowlists the router to "fix" this
extension.setAllowedToSwap(pool, address(router), true);

// Now ANY user (bob, who is NOT allowlisted) can swap via router
vm.prank(bob);
router.exactInputSingle(...); // succeeds — allowlist bypassed
```
