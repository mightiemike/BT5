Looking at the `SwapAllowlistExtension`, the `ExtensionCalling` plumbing, and the `MetricOmmSimpleRouter`, I can trace a concrete wrong-actor binding vulnerability.

---

### Title
SwapAllowlistExtension Checks Router Address Instead of Actual User, Allowing Allowlist Bypass via MetricOmmSimpleRouter — (`metric-periphery/contracts/extensions/SwapAllowlistExtension.sol`)

### Summary

`SwapAllowlistExtension.beforeSwap` gates swaps by checking the `sender` parameter, which is the `msg.sender` of the pool's `swap()` call. When a user routes through `MetricOmmSimpleRouter`, `sender` is the **router address**, not the actual user. A pool admin cannot simultaneously allow legitimate allowlisted users to use the router and block non-allowlisted users: allowlisting the router opens the gate to everyone; not allowlisting it silently breaks the router for all users including allowlisted ones.

### Finding Description

`ExtensionCalling._beforeSwap` passes `msg.sender` (the immediate caller of `pool.swap()`) as the `sender` argument to every registered extension: [1](#0-0) 

`SwapAllowlistExtension.beforeSwap` then checks that `sender` against the per-pool allowlist: [2](#0-1) 

`MetricOmmSimpleRouter.exactInputSingle` calls `pool.swap()` directly, making the router the `msg.sender` seen by the pool and therefore the `sender` seen by the extension: [3](#0-2) 

The same pattern holds for `exactInput`, `exactOutputSingle`, and `exactOutput`. [4](#0-3) 

The allowlist check therefore resolves to `allowedSwapper[pool][router]`, not `allowedSwapper[pool][actual_user]`. The pool admin faces an impossible choice:

| Admin action | Effect |
|---|---|
| Do **not** allowlist the router | All router users (including allowlisted ones) are blocked — broken swap flow |
| Allowlist the router | Every address on-chain can bypass the allowlist through the router |

### Impact Explanation

A pool configured with `SwapAllowlistExtension` to restrict trading to KYC'd or otherwise curated addresses loses that restriction entirely the moment the admin allowlists the router to restore normal UX. Any unprivileged address can call `MetricOmmSimpleRouter.exactInputSingle` and trade on the curated pool. This is a direct curation-policy bypass with fund-impacting consequences: the pool's pricing or liquidity may be calibrated for a restricted counterparty set, and unrestricted access allows value extraction by unintended parties.

### Likelihood Explanation

The `MetricOmmSimpleRouter` is the canonical periphery entry point documented and expected to be used by end users. A pool admin who deploys a `SwapAllowlistExtension` and then discovers that allowlisted users cannot use the router will naturally allowlist the router — triggering the bypass. The trigger is a routine, well-motivated admin action, not an exotic configuration.

### Recommendation

The extension must gate on the **actual user**, not the intermediary caller. Two viable approaches:

1. **Pass the real user in `extensionData`**: The router encodes `msg.sender` into `extensionData`; the extension decodes and checks it. The extension must also verify that `msg.sender` (the pool's caller) is a trusted router registered with the factory before trusting the decoded address.

2. **Add a `swapper` field to the pool's `swap()` signature**: The pool accepts an explicit `swapper` address (distinct from `recipient`) and passes it to extensions. The router sets `swapper = msg.sender`. This is a breaking interface change but is the cleanest fix.

Until fixed, pool admins should be warned that `SwapAllowlistExtension` only gates **direct** pool callers and provides no protection when the canonical router is in use.

### Proof of Concept

```
1. Pool P is deployed with SwapAllowlistExtension E in its beforeSwap slot.
2. Admin allowlists Alice (0xAlice) on pool P: allowedSwapper[P][0xAlice] = true.
3. Admin also allowlists the router (0xRouter) so Alice can use it:
       allowedSwapper[P][0xRouter] = true.
4. Mallory (0xMallory, NOT allowlisted) calls:
       MetricOmmSimpleRouter.exactInputSingle({pool: P, recipient: 0xMallory, ...})
5. Router calls pool.swap(0xMallory, ...) → msg.sender to pool = 0xRouter.
6. Pool calls E.beforeSwap(sender=0xRouter, ...).
7. Extension checks allowedSwapper[P][0xRouter] == true → passes.
8. Mallory's swap executes on the curated pool despite never being allowlisted.
``` [5](#0-4) [6](#0-5)
