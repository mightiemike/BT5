### Title
`SwapAllowlistExtension` gates the router address instead of the end user, allowing any user to bypass the per-user swap allowlist via `MetricOmmSimpleRouter` — (`metric-periphery/contracts/extensions/SwapAllowlistExtension.sol`)

---

### Summary

`SwapAllowlistExtension.beforeSwap` checks the `sender` argument, which is the direct caller of `pool.swap()`. When a user routes through `MetricOmmSimpleRouter`, the pool sees `msg.sender = router`, so the extension checks whether the **router** is allowlisted, not the end user. If the pool admin allowlists the router to let legitimate users access it, every unpermissioned user can bypass the swap allowlist by routing through the same public router contract.

---

### Finding Description

`SwapAllowlistExtension.beforeSwap` enforces the per-user gate as follows:

```solidity
// SwapAllowlistExtension.sol L31-41
function beforeSwap(address sender, address, bool, int128, uint128, uint256, uint128, uint128, bytes calldata)
    external view override returns (bytes4)
{
    if (!allowAllSwappers[msg.sender] && !allowedSwapper[msg.sender][sender]) {
        revert IMetricOmmPoolActions.NotAllowedToSwap();
    }
    return IMetricOmmExtensions.beforeSwap.selector;
}
```

`msg.sender` here is the pool (the extension is called by the pool), and `sender` is the first argument forwarded by `ExtensionCalling._beforeSwap`, which is `msg.sender` of the original `pool.swap()` call:

```solidity
// MetricOmmPool.sol L230-240
_beforeSwap(
    msg.sender,   // ← whoever called pool.swap()
    recipient,
    ...
);
```

When a user calls `MetricOmmSimpleRouter.exactInputSingle()`, the router calls `pool.swap()` with itself as `msg.sender`:

```solidity
// MetricOmmSimpleRouter.sol L71-80
_setNextCallbackContext(params.pool, CALLBACK_MODE_JUST_PAY, msg.sender, params.tokenIn);
(int128 amount0Delta, int128 amount1Delta) = IMetricOmmPoolActions(params.pool)
    .swap(
        params.recipient,
        params.zeroForOne,
        ...
        params.extensionData
    );
```

The pool therefore calls `_beforeSwap(msg.sender = router, ...)`, and the extension evaluates `allowedSwapper[pool][router]` — not `allowedSwapper[pool][end_user]`.

This creates an irresolvable dilemma for the pool admin:

| Admin action | Consequence |
|---|---|
| Does **not** allowlist the router | Allowlisted users cannot use the router at all |
| **Does** allowlist the router | Every non-allowlisted user can bypass the gate via the router |

The `DepositAllowlistExtension` does **not** share this flaw because it checks `owner` (the position owner explicitly passed to `pool.addLiquidity`), which is correctly preserved through the liquidity adder path.

---

### Impact Explanation

Any pool that deploys `SwapAllowlistExtension` to restrict trading to a curated set of addresses has its access control completely nullified for any user who routes through `MetricOmmSimpleRouter`. The router is a public, permissionless contract. A disallowed user can execute swaps against the pool, draining LP value or trading against oracle prices that the pool admin intended to expose only to vetted counterparties. This is a direct loss of LP principal and a broken core pool invariant.

---

### Likelihood Explanation

The `MetricOmmSimpleRouter` is the canonical periphery entry point for swaps. Any pool operator who configures `SwapAllowlistExtension` and also expects users to use the router (the normal UX path) will either inadvertently open the gate to all users or block legitimate users from using the router. No special attacker setup is required beyond calling a public function.

---

### Recommendation

The extension must check the **economically relevant actor** — the end user — not the intermediary. Two complementary fixes:

1. **Pass the original initiator through the router.** The router already stores `msg.sender` as the payer in transient storage (`_setNextCallbackContext(..., msg.sender, ...)`). The pool could forward this as an additional field, or the extension could accept an ABI-encoded `extensionData` payload containing the real initiator address, verified against the router's transient payer slot.

2. **Check `sender` only when `sender` is not a known router; otherwise decode the real user from `extensionData`.** The extension's `beforeSwap` should require that callers going through the router include a signed or router-attested identity in `extensionData`, and gate on that identity instead of the router address.

The simplest safe fix is to require that `extensionData` always carries the real user address when `sender` is a router, and have the router populate that field, so the extension can verify `allowedSwapper[pool][realUser]`.

---

### Proof of Concept

```solidity
// Pool is configured with SwapAllowlistExtension.
// Admin allowlists userA and the router (so userA can use the router).
swapExtension.setAllowedToSwap(address(pool), address(router), true);
swapExtension.setAllowedToSwap(address(pool), userA, true);
// userB is NOT allowlisted.

// userB calls the router directly — extension checks allowedSwapper[pool][router] == true → passes.
vm.prank(userB);
router.exactInputSingle(
    IMetricOmmSimpleRouter.ExactInputSingleParams({
        pool: address(pool),
        tokenIn: address(token0),
        recipient: userB,
        deadline: block.timestamp + 1,
        amountIn: 1000,
        amountOutMinimum: 0,
        zeroForOne: true,
        priceLimitX64: 0,
        extensionData: ""
    })
);
// userB successfully swaps despite not being on the allowlist.
