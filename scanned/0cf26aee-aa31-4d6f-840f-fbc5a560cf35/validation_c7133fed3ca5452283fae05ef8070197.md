### Title
`allowPushers` Signature Replay Nullifies Pusher Revocation — (`smart-contracts-poc/contracts/oracles/compressed/CompressedOracle.sol`)

### Summary

`CompressedOracleV1.allowPushers` accepts a pusher's EIP-191 consent signature and writes `namespaceRemapping[pusher] = creator`. The function's own NatSpec acknowledges the deadline is the only guard against post-revocation re-establishment. However, no used-signature registry or per-pusher nonce exists, so the creator can replay the identical signature any number of times before the deadline expires. A pusher who calls `revokePusher()` has their revocation silently overwritten the next time the creator replays the original signature, making revocation ineffective for the entire deadline window.

### Finding Description

`allowPushers` validates the pusher's consent signature and then unconditionally overwrites `namespaceRemapping[pusher]`:

```solidity
// CompressedOracle.sol L192-L211
function allowPushers(uint256 deadline, address[] calldata pushers, bytes[] memory signatures) external {
    _ensureDeadline(deadline);
    ...
    bytes32 hash = MessageHashUtils.toEthSignedMessageHash(
        keccak256(abi.encode(block.chainid, address(this), deadline, pusher, msg.sender))
    );
    require(pusher == ECDSA.recover(hash, signatures[i]));
    namespaceRemapping[pusher] = msg.sender;   // ← no guard against replay
    ...
}
``` [1](#0-0) 

`revokePusher` clears the mapping:

```solidity
// CompressedOracle.sol L238-L243
function revokePusher() external {
    address creator = namespaceRemapping[msg.sender];
    if (creator == address(0) || creator == msg.sender) revert NoSelfRemapping();
    namespaceRemapping[msg.sender] = address(0);
    emit PusherRevoked(msg.sender, creator);
}
``` [2](#0-1) 

Because `allowPushers` has no record of consumed signatures, the creator can immediately call `allowPushers` again with the same `(deadline, pusher, sig)` tuple, restoring `namespaceRemapping[pusher] = creator`. The pusher's revocation is overwritten in the same block. The creator can repeat this indefinitely until `block.timestamp > deadline`.

The NatSpec comment on line 189-191 explicitly states the deadline is the mechanism that prevents post-revocation re-establishment:

> *"The deadline is likewise required: the signed consent carries no timestamp of its own, so an undated signature could re-establish a delegation AFTER the pusher revoked it."* [3](#0-2) 

The deadline prevents replay only after it expires. Within the deadline window — which can be set to any future time the creator chooses — the signature is replayable without limit. The intended protection is therefore incomplete.

`_ensureDeadline` only checks `block.timestamp <= deadline`: [4](#0-3) 

No nonce, no used-signature mapping, and no per-pusher revocation flag exists anywhere in the contract.

### Impact Explanation

The `fallback` push path resolves the namespace at call time:

```solidity
// CompressedOracle.sol L315-L316
address creator = namespaceRemapping[msg.sender];
if (creator == address(0)) creator = msg.sender;
``` [5](#0-4) 

After the creator replays the signature, every subsequent push by the pusher lands in the **creator's** namespace, not the pusher's own. The pusher's own feedId (keyed by `pusher` address) receives no new data and goes stale. Any pool or price provider reading the pusher's own feedId will see a timestamp-zero or stale price, which every consumer rejects — effectively halting price delivery for that feed. Meanwhile the creator's namespace continues to receive fresh data the pusher explicitly tried to stop providing.

This maps to two allowed impacts:
- **Bad-price execution / stale quote**: pools relying on the pusher's own feedId receive stale data after the pusher stops pushing to avoid contributing to the creator's namespace.
- **Admin-boundary break**: the pusher's explicit on-chain revocation is bypassed by an unprivileged replay of a previously consumed signature.

### Likelihood Explanation

The creator holds the signature off-chain from the original delegation flow. Replaying it requires a single transaction with no additional privileges. The pusher has no on-chain mechanism to invalidate the signature before the deadline expires. Likelihood is **medium**: it requires the creator to act adversarially, but the action is trivially cheap and fully permissionless.

### Recommendation

Track consumed signatures with a `mapping(bytes32 => bool) private _usedSignatures` and mark each signature hash as used after the first successful `allowPushers` call:

```solidity
bytes32 sigHash = keccak256(signatures[i]);
require(!_usedSignatures[sigHash], SignatureAlreadyUsed());
_usedSignatures[sigHash] = true;
```

Alternatively, include a per-pusher nonce in the signed message and increment it on each successful delegation, so any previously issued signature is automatically invalidated.

### Proof of Concept

```solidity
// 1. Pusher signs consent for creator with deadline = now + 30 days
uint256 deadline = block.timestamp + 30 days;
bytes memory sig = pusherSign(deadline, pusherAddr, creatorAddr);

// 2. Creator establishes delegation
vm.prank(creatorAddr);
oracle.allowPushers(deadline, toArray(pusherAddr), toArray(sig));
assertEq(oracle.namespaceRemapping(pusherAddr), creatorAddr); // delegated

// 3. Pusher revokes
vm.prank(pusherAddr);
oracle.revokePusher();
assertEq(oracle.namespaceRemapping(pusherAddr), address(0)); // revoked

// 4. Creator replays the SAME signature — no revert, revocation overwritten
vm.prank(creatorAddr);
oracle.allowPushers(deadline, toArray(pusherAddr), toArray(sig));
assertEq(oracle.namespaceRemapping(pusherAddr), creatorAddr); // delegation restored

// 5. Pusher's subsequent fallback pushes land in creator's namespace again,
//    pusher's own feedId goes stale, pools reading it receive timestamp-0 data.
```

    function allowPushers(uint256 deadline, address[] calldata pushers, bytes[] memory signatures) external {
        _ensureDeadline(deadline);

        uint256 l = pushers.length;
        require(l == signatures.length);
        for (uint256 i; i < l; i++) {
            address pusher = pushers[i];

            if (pusher == msg.sender) {
                revert NoSelfRemapping();
            }

            bytes32 hash = MessageHashUtils.toEthSignedMessageHash(
                keccak256(abi.encode(block.chainid, address(this), deadline, pusher, msg.sender))
            );
            require(pusher == ECDSA.recover(hash, signatures[i]));

            namespaceRemapping[pusher] = msg.sender;
            emit PusherAuthorized(pusher, msg.sender);
        }
