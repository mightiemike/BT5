### Title
`SwapAllowlistExtension` gates the router address instead of the end user, allowing any user to bypass the swap allowlist via `MetricOmmSimpleRouter` — (`File: metric-periphery/contracts/extensions/SwapAllowlistExtension.sol`)

---

### Summary

`SwapAllowlistExtension.beforeSwap` checks the `sender` argument, which the pool sets to `msg.sender` of the `swap` call. When a user routes through `MetricOmmSimpleRouter`, `msg.sender` to the pool is the **router contract**, not the end user. A pool admin who allowlists the router to enable router-mediated swaps for legitimate users simultaneously grants every non-allowlisted user the ability to bypass the guard by routing through the same public router.

---

### Finding Description

**Invariant broken**: The swap allowlist must gate the economically relevant actor — the end user initiating the trade — regardless of which supported public entrypoint reaches the pool.

**Root cause — wrong actor identity checked:**

`SwapAllowlistExtension.beforeSwap` receives `sender` as its first argument and checks it against the per-pool allowlist: [1](#0-0) 

The pool populates that `sender` argument with `msg.sender` of the `swap` call: [2](#0-1) 

When a user calls `MetricOmmSimpleRouter.exactInputSingle`, the router calls `pool.swap(...)` directly: [3](#0-2) 

So `msg.sender` to the pool is `address(router)`, not the end user. The extension therefore checks `allowedSwapper[pool][router]`, not `allowedSwapper[pool][endUser]`.

The same substitution occurs in `exactInput` (all hops), `exactOutputSingle`, and `exactOutput` (all recursive hops): [4](#0-3) 

**Contrast with `DepositAllowlistExtension`**, which correctly ignores the `sender` (first arg) and checks `owner` (second arg) — the economically relevant depositor: [5](#0-4) 

---

### Impact Explanation

A pool admin who deploys a curated pool with `SwapAllowlistExtension` intends to restrict swaps to a specific set of addresses (e.g., KYC'd counterparties). To allow those allowlisted users to also use the router (the standard periphery path), the admin must add `allowedSwapper[pool][router] = true`. The moment the router is allowlisted, **every** address — including non-allowlisted users — can bypass the guard by calling `router.exactInputSingle(...)`. The pool's `beforeSwap` hook sees `sender = router`, which passes the check, and the swap executes.

Impact: direct policy bypass on curated pools; non-allowlisted users can trade against LP positions that were intended to be restricted, causing LP value leakage and breaking the pool's curation invariant.

---

### Likelihood Explanation

The router is the standard, documented periphery entry point. Any pool admin who wants allowlisted users to be able to use the router (rather than calling the pool directly) will add the router to the allowlist. This is a natural operational step. Once done, the bypass is trivially reachable by any public user with no special privileges.

---

### Recommendation

The extension must check the **end user**, not the immediate pool caller. Two sound approaches:

1. **Pass the original `msg.sender` through the router as a separate field in `extensionData`** and have the extension decode and verify it (requires router cooperation and a matching extension implementation).

2. **Check `recipient` instead of `sender`** if the pool's design equates the recipient with the authorized swapper — but this only works for single-hop exact-input flows.

3. **Most robust**: change `SwapAllowlistExtension.beforeSwap` to check the `recipient` argument (the address that receives output tokens) rather than `sender`, or redesign the hook interface to carry the originating EOA explicitly, similar to how `DepositAllowlistExtension` gates `owner` rather than `sender`.

---

### Proof of Concept

```
Setup:
  - Deploy pool with SwapAllowlistExtension
  - Pool admin: setAllowedToSwap(pool, alice, true)       // alice is KYC'd
  - Pool admin: setAllowedToSwap(pool, router, true)      // enable router for alice

Attack:
  - bob (not allowlisted) calls router.exactInputSingle({pool: pool, ...})
  - router calls pool.swap(recipient=bob, ...)
  - pool calls extension.beforeSwap(sender=router, ...)
  - extension checks: allowedSwapper[pool][router] == true  → PASSES
  - bob's swap executes against the curated LP pool

Result:
  - bob bypasses the swap allowlist entirely
  - curated pool's access control is nullified for all router-mediated swaps
``` [6](#0-5) [7](#0-6)
