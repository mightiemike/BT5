### Title
Router-Mediated Swaps Bypass `SwapAllowlistExtension` Allowlist — (`metric-periphery/contracts/extensions/SwapAllowlistExtension.sol`)

---

### Summary

`SwapAllowlistExtension.beforeSwap` gates swaps by checking the `sender` argument, which the pool sets to `msg.sender` of `pool.swap()`. When a user routes through `MetricOmmSimpleRouter`, the pool's `msg.sender` is the **router contract**, not the original user. If the pool admin allowlists the router (required for any allowlisted user to use the router), every public user can bypass the allowlist by routing through the router.

---

### Finding Description

**Call path — direct swap:**

```
User → MetricOmmPool.swap()
         msg.sender = User
         _beforeSwap(sender=User, ...)
         SwapAllowlistExtension.beforeSwap(sender=User)
         check: allowedSwapper[pool][User]   ✓ or ✗
```

**Call path — router swap:**

```
User → MetricOmmSimpleRouter.exactInputSingle()
         → IMetricOmmPoolActions(pool).swap(...)
              msg.sender = Router
              _beforeSwap(sender=Router, ...)
              SwapAllowlistExtension.beforeSwap(sender=Router)
              check: allowedSwapper[pool][Router]   ← always passes if router is allowlisted
```

In `MetricOmmPool.swap()`, the pool passes `msg.sender` as the `sender` argument to `_beforeSwap`: [1](#0-0) 

`ExtensionCalling._beforeSwap` forwards that `sender` verbatim to the extension: [2](#0-1) 

`SwapAllowlistExtension.beforeSwap` then checks `allowedSwapper[msg.sender][sender]`, where `msg.sender` is the pool and `sender` is whoever called `pool.swap()`: [3](#0-2) 

`MetricOmmSimpleRouter.exactInputSingle` calls `pool.swap()` directly, making the router the pool's `msg.sender`: [4](#0-3) 

The same pattern applies to `exactInput`, `exactOutputSingle`, and `exactOutput`. [5](#0-4) 

**The dilemma the pool admin faces:**

| Router allowlisted? | Allowlisted user via router | Non-allowlisted user via router |
|---|---|---|
| No | ✗ Blocked (router not in list) | ✗ Blocked |
| Yes | ✓ Passes | ✓ **Passes — bypass** |

If the admin allowlists the router to enable router-mediated swaps for curated users, the check degenerates to "is the router allowlisted?" — which is true for every caller. Any public user can then bypass the allowlist by routing through `MetricOmmSimpleRouter`.

---

### Impact Explanation

A curated pool using `SwapAllowlistExtension` to restrict trading to specific counterparties (e.g., KYC'd addresses, institutional partners) loses that restriction entirely for any user who routes through the supported periphery router. Unauthorized users can trade on pools that were designed to exclude them, breaking the core curation guarantee and potentially causing regulatory, financial, or protocol-integrity harm.

---

### Likelihood Explanation

Any pool that:
1. Deploys with `SwapAllowlistExtension` (a supported, documented extension), **and**
2. Wants allowlisted users to be able to use the public router

is vulnerable. This is the expected production configuration for curated pools that integrate with the periphery. The bypass requires only a standard `exactInputSingle` call — no special privileges, no flash loans, no multi-transaction setup.

---

### Recommendation

The extension must verify the **original transaction initiator** (`tx.origin`) or the pool must propagate a trusted "real sender" through the call. The cleanest fix is to have the router pass the original user's address in `extensionData`, and have `SwapAllowlistExtension.beforeSwap` decode and check that address instead of (or in addition to) `sender`. Alternatively, the pool could expose `tx.origin` as a separate hook argument, though `tx.origin` has its own risks. A third option is a dedicated router allowlist that maps `(pool, router) → true` while separately checking the user via a signed permit in `extensionData`.

---

### Proof of Concept

```solidity
// Pool admin setup:
// allowedSwapper[pool][alice] = true
// allowedSwapper[pool][router] = true  ← required for Alice to use the router

// Attacker (Eve, not allowlisted):
router.exactInputSingle(ExactInputSingleParams({
    pool: curatedPool,
    recipient: eve,
    tokenIn: token0,
    amountIn: 1e18,
    amountOutMinimum: 0,
    zeroForOne: true,
    priceLimitX64: 0,
    deadline: block.timestamp,
    extensionData: ""
}));
// pool.swap() is called with msg.sender = router
// beforeSwap receives sender = router
// allowedSwapper[pool][router] == true  → check passes
// Eve's swap executes successfully despite not being allowlisted
```
