### Title
`SwapAllowlistExtension` Checks Router Address Instead of Actual Swapper, Allowing Any User to Bypass the Swap Allowlist — (`metric-periphery/contracts/extensions/SwapAllowlistExtension.sol`)

---

### Summary

`SwapAllowlistExtension.beforeSwap` gates swaps by checking the `sender` argument passed by the pool. When a user routes through `MetricOmmSimpleRouter`, the pool's `msg.sender` is the router contract, so the extension checks whether the **router** is allowlisted rather than the actual user. A pool admin who allowlists the router to enable router-mediated swaps for permitted users inadvertently opens the pool to every user on the router.

---

### Finding Description

`MetricOmmPool.swap` passes `msg.sender` as the `sender` argument to `_beforeSwap`: [1](#0-0) 

`ExtensionCalling._beforeSwap` forwards that value unchanged to every configured extension: [2](#0-1) 

`SwapAllowlistExtension.beforeSwap` then checks `allowedSwapper[msg.sender][sender]`, where `msg.sender` is the pool and `sender` is whatever the pool received as its own `msg.sender`: [3](#0-2) 

`MetricOmmSimpleRouter.exactInputSingle` calls `pool.swap()` directly, making the router the pool's `msg.sender`: [4](#0-3) 

The same pattern applies to `exactInput`, `exactOutputSingle`, and `exactOutput`. [5](#0-4) 

Result: when any user calls the router, the extension sees `sender = router_address` and checks `allowedSwapper[pool][router]`. If the pool admin has allowlisted the router (the natural step to enable router-mediated swaps for permitted users), the check passes for **every** caller of the router, regardless of whether that caller is on the allowlist.

The existing test suite only exercises direct-pool swaps through `TestCaller` and never tests the router path against an allowlisted pool, so this bypass is untested: [6](#0-5) 

---

### Impact Explanation

A curated pool (e.g., institutional, KYC-gated, or whitelist-only market) deploys with `SwapAllowlistExtension`. The pool admin allowlists the router so that permitted users can swap through the standard periphery. From that moment, any unpermitted user can call `MetricOmmSimpleRouter.exactInputSingle` (or any other router entry point) and execute swaps at oracle-derived prices against the pool's LP reserves. LP principal is directly at risk because the oracle-anchored pool will fill any swap up to the configured price limit; there is no internal price discovery to limit the damage. The allowlist — the sole access-control mechanism for the pool — is completely neutralized.

---

### Likelihood Explanation

The router is the standard, documented periphery entry point. Pool admins who want to allow permitted users to use the router must allowlist it. The bypass requires no special knowledge: any user who observes that the router is allowlisted (readable on-chain via `allowedSwapper`) can immediately exploit it. The trigger is a single public call to `MetricOmmSimpleRouter`.

---

### Recommendation

The extension must gate the **economic actor**, not the intermediary. Two sound approaches:

1. **Pass the original caller through the router.** Add a `payer`/`originator` field to the swap call or use a trusted-forwarder pattern so the pool can supply the real user address to the extension.

2. **Check `sender` against the router's own caller.** The router already stores the real payer in transient storage (`_getPayer()`). The extension could read a standardized "originator" slot from the calling contract, but this requires a protocol-level convention.

The simplest safe fix is to have the router pass the real user address as an explicit `originator` parameter that the pool forwards to extensions, replacing the raw `msg.sender` binding for allowlist purposes.

---

### Proof of Concept

```
Setup:
  pool configured with SwapAllowlistExtension
  pool admin calls swapExtension.setAllowedToSwap(pool, router, true)
    (to enable router-mediated swaps for permitted users)
  pool admin calls swapExtension.setAllowedToSwap(pool, alice, true)
    (alice is the only permitted user)

Attack (executed by bob, who is NOT on the allowlist):
  bob calls MetricOmmSimpleRouter.exactInputSingle({
      pool: pool,
      recipient: bob,
      zeroForOne: true,
      amountIn: X,
      ...
  })

Trace:
  router.exactInputSingle()
    → pool.swap(recipient=bob, ...) [msg.sender = router]
      → _beforeSwap(sender=router, ...)
        → SwapAllowlistExtension.beforeSwap(sender=router, ...)
          → allowedSwapper[pool][router] == true  ✓  (passes)
      → swap executes, bob receives tokens from LP reserves
```

`allowedSwapper[pool][router]` is `true` because the admin set it to enable router access for alice. Bob's identity is never checked. The allowlist invariant is broken. [7](#0-6) [8](#0-7)
