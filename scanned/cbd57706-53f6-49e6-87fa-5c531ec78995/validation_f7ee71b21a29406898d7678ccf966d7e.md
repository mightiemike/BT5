### Title
`SwapAllowlistExtension` checks the router address instead of the end-user, allowing any caller to bypass a curated pool's swap allowlist via `MetricOmmSimpleRouter` — (File: `metric-periphery/contracts/extensions/SwapAllowlistExtension.sol`)

---

### Summary

`SwapAllowlistExtension.beforeSwap` gates swaps by checking the `sender` argument passed by the pool. Because `MetricOmmPool.swap` passes `msg.sender` as `sender`, and `MetricOmmSimpleRouter` is `msg.sender` when it calls the pool, the extension checks the router's address against the allowlist instead of the actual end-user's address. This makes the allowlist either permanently broken (allowlisted users cannot swap through the router) or trivially bypassable (if the router is allowlisted, every user can trade on a curated pool).

---

### Finding Description

**Step 1 — Pool passes `msg.sender` as `sender` to the extension.**

`MetricOmmPool.swap` calls `_beforeSwap` with `msg.sender` as the first argument: [1](#0-0) 

`ExtensionCalling._beforeSwap` forwards that value verbatim as the `sender` parameter in the ABI-encoded call to every configured extension: [2](#0-1) 

**Step 2 — `SwapAllowlistExtension` keys the allowlist on that `sender`.**

```solidity
// SwapAllowlistExtension.sol line 37
if (!allowAllSwappers[msg.sender] && !allowedSwapper[msg.sender][sender]) {
    revert IMetricOmmPoolActions.NotAllowedToSwap();
}
```

`msg.sender` here is the pool; `sender` is whoever called `pool.swap()`. [3](#0-2) 

**Step 3 — `MetricOmmSimpleRouter` is `msg.sender` to the pool.**

Every router entry-point (`exactInputSingle`, `exactInput`, `exactOutputSingle`, `exactOutput`) calls `IMetricOmmPoolActions(pool).swap(...)` directly, making the router contract the `msg.sender` the pool sees: [4](#0-3) 

**Result — the extension checks `allowedSwapper[pool][router]`, not `allowedSwapper[pool][user]`.**

The pool admin configures the allowlist with individual user addresses (e.g., `setAllowedToSwap(pool, alice, true)`). When Alice calls `router.exactInputSingle(...)`, the extension receives `sender = address(router)`, looks up `allowedSwapper[pool][router]` (false), and reverts `NotAllowedToSwap`. Alice is blocked even though she is explicitly allowlisted.

The only way to unblock router-mediated swaps is to allowlist the router address itself. But `allowedSwapper[pool][router] = true` grants every user on the planet the ability to swap on the curated pool by routing through the router, defeating the entire purpose of the allowlist. [5](#0-4) 

**Contrast with `DepositAllowlistExtension`** — the deposit extension checks the explicit `owner` parameter (the position owner), not `sender`, so it correctly identifies the economically relevant actor regardless of who the `msg.sender` is: [6](#0-5) 

---

### Impact Explanation

A pool configured with `SwapAllowlistExtension` as a `beforeSwap` hook cannot simultaneously (a) allow its allowlisted users to trade through the standard periphery router and (b) block non-allowlisted users from doing the same. The two goals are mutually exclusive:

- **Allowlist not including the router** → allowlisted users are blocked from the router; core swap functionality is broken for the intended audience.
- **Allowlist including the router** → every unprivileged user bypasses the curation gate and trades on a pool that should be restricted, causing direct loss of the pool's intended access-control invariant and potential fund-impacting trades by unauthorized parties.

---

### Likelihood Explanation

`MetricOmmSimpleRouter` is the primary public swap interface documented and deployed for end-users. Any pool that activates `SwapAllowlistExtension` and expects users to interact through the router will immediately encounter this failure. No special privileges, flash loans, or unusual token behavior are required — a standard `exactInputSingle` call is sufficient.

---

### Recommendation

Pass the original end-user identity through the swap path so the extension can check it. Two approaches:

1. **Preferred — add a `payer` / `originator` field to the swap call or extension payload**: The router encodes `msg.sender` (the real user) into `extensionData` or a dedicated parameter, and the extension reads it from there after verifying the pool is the caller.

2. **Simpler — check `sender` only when the caller is not a known router**: The extension reads the pool's factory to determine whether `sender` is a registered periphery contract, and if so, decodes the real user from `extensionData`.

Either way, the extension must be able to distinguish the intermediary router from the economic actor it is supposed to gate.

---

### Proof of Concept

```
Setup:
  pool configured with SwapAllowlistExtension as beforeSwap hook
  pool admin calls: setAllowedToSwap(pool, alice, true)
  pool admin does NOT allowlist the router

Attack path A — allowlisted user is blocked:
  alice calls router.exactInputSingle({pool: pool, ...})
  router calls pool.swap(recipient, ...) → msg.sender to pool = router
  pool calls _beforeSwap(sender=router, ...)
  extension checks allowedSwapper[pool][router] → false
  → revert NotAllowedToSwap   ← alice cannot use the router

Attack path B — bypass (pool admin allowlists router to fix A):
  pool admin calls: setAllowedToSwap(pool, router, true)
  eve (not allowlisted) calls router.exactInputSingle({pool: pool, ...})
  router calls pool.swap(...) → msg.sender = router
  extension checks allowedSwapper[pool][router] → true
  → swap succeeds   ← eve bypasses the allowlist entirely
``` [3](#0-2) [1](#0-0) [7](#0-6)

contract SwapAllowlistExtension is BaseMetricExtension, ISwapAllowlistExtension {
  mapping(address pool => mapping(address swapper => bool)) public allowedSwapper;
  mapping(address pool => bool) public allowAllSwappers;

  constructor(address factory_) BaseMetricExtension(factory_) {}

  function setAllowedToSwap(address pool_, address swapper, bool allowed) external onlyPoolAdmin(pool_) {
    allowedSwapper[pool_][swapper] = allowed;
    emit AllowedToSwapSet(pool_, swapper, allowed);
  }

  function setAllowAllSwappers(address pool_, bool allowed) external onlyPoolAdmin(pool_) {
    allowAllSwappers[pool_] = allowed;
    emit AllowAllSwappersSet(pool_, allowed);
  }

  function isAllowedToSwap(address pool_, address swapper) external view returns (bool) {
    return allowAllSwappers[pool_] || allowedSwapper[pool_][swapper];
  }

  function beforeSwap(address sender, address, bool, int128, uint128, uint256, uint128, uint128, bytes calldata)
    external
    view
    override
    returns (bytes4)
  {
    if (!allowAllSwappers[msg.sender] && !allowedSwapper[msg.sender][sender]) {
      revert IMetricOmmPoolActions.NotAllowedToSwap();
    }
    return IMetricOmmExtensions.beforeSwap.selector;
  }
}
