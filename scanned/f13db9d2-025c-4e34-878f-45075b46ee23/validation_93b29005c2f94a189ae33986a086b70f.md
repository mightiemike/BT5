### Title
Pool Admin Bypasses `maxAdminSpreadFeeE6` Cap via Uncapped Per-Bin Fees — (`metric-core/contracts/MetricOmmPoolFactory.sol`, `metric-core/contracts/MetricOmmPool.sol`)

---

### Summary

`setPoolBinAdditionalFees` applies no upper-bound check on `addFeeBuyE6` / `addFeeSellE6`, allowing a pool admin to set per-bin spread fees up to `type(uint16).max` (65 535, ≈ 6.5535 % in E6 terms) and bypass the factory-owner-configured `maxAdminSpreadFeeE6` cap that is enforced on the global spread-fee path.

---

### Finding Description

The factory enforces `maxAdminSpreadFeeE6` only inside `setPoolAdminFees`:

```solidity
// MetricOmmPoolFactory.sol:414-415
if (newAdminSpreadFeeE6 > maxAdminSpreadFeeE6) revert AdminFeeTooHigh();
if (newAdminNotionalFeeE8 > maxAdminNotionalFeeE8) revert AdminFeeTooHigh();
``` [1](#0-0) 

The sibling pool-admin entrypoint `setPoolBinAdditionalFees` passes values straight through with **no cap check**:

```solidity
// MetricOmmPoolFactory.sol:450-457
function setPoolBinAdditionalFees(address pool, int8 bin, uint16 addFeeBuyE6, uint16 addFeeSellE6)
    external override nonReentrant onlyPoolAdmin(pool)
{
    IMetricOmmPoolFactoryActions(pool).setBinAdditionalFees(bin, addFeeBuyE6, addFeeSellE6);
}
``` [2](#0-1) 

The pool-side handler also performs no cap check — only a bin-index range check:

```solidity
// MetricOmmPool.sol:464-474
function setBinAdditionalFees(int8 bin, uint16 addFeeBuyE6, uint16 addFeeSellE6)
    external onlyFactory nonReentrant(PoolActions.SET_BIN_ADDITIONAL_FEES)
{
    if (bin < LOWEST_BIN || bin > HIGHEST_BIN) revert InvalidBinIndex(bin);
    BinState storage s = _binStates[bin];
    s.addFeeBuyE6 = addFeeBuyE6;
    s.addFeeSellE6 = addFeeSellE6;
    ...
}
``` [3](#0-2) 

These stored values are then added directly to the effective swap fee in every swap path:

```solidity
// MetricOmmPool.sol:999 (buy-token0 path)
params.baseFeeX64 + Math.mulDiv(binState.addFeeBuyE6, ONE_X64, 1e6),
``` [4](#0-3) 

```solidity
// MetricOmmPool.sol:1088 (sell-token0 path)
params.baseFeeX64 + Math.mulDiv(binState.addFeeSellE6, ONE_X64, 1e6),
``` [5](#0-4) 

The hard cap is `HARD_MAX_SPREAD_FEE_E6 = 200_000` (20 %). `maxAdminSpreadFeeE6` starts there but the factory owner can lower it. Regardless of what `maxAdminSpreadFeeE6` is set to, a pool admin can set `addFeeBuyE6 = type(uint16).max` (65 535 ≈ 6.5535 %) on any bin, completely bypassing the cap. [6](#0-5) 

---

### Impact Explanation

Traders swapping through a bin with `addFeeBuyE6 = 65535` pay ≈ 6.5535 % additional spread fee on top of the base spread, regardless of the factory-owner-configured `maxAdminSpreadFeeE6`. This is a direct loss of trader principal — the excess fee is extracted by the pool admin beyond the cap the factory owner intended to enforce. The `getSellAndBuyPrices` view also reflects the inflated fee, so any price-limit protection a trader sets based on that view is consistent with the inflated price, meaning the swap succeeds and the trader loses funds. [7](#0-6) 

---

### Likelihood Explanation

Any pool admin can call `setPoolBinAdditionalFees` at any time with no preconditions beyond holding the `poolAdmin[pool]` role. The call is a single transaction. The factory owner's cap (`maxAdminSpreadFeeE6`) provides no protection because it is not checked on this path. [2](#0-1) 

---

### Recommendation

Add a cap check in `setPoolBinAdditionalFees` before forwarding to the pool:

```solidity
function setPoolBinAdditionalFees(address pool, int8 bin, uint16 addFeeBuyE6, uint16 addFeeSellE6)
    external override nonReentrant onlyPoolAdmin(pool)
{
    if (addFeeBuyE6 > maxAdminSpreadFeeE6) revert AdminFeeTooHigh();
    if (addFeeSellE6 > maxAdminSpreadFeeE6) revert AdminFeeTooHigh();
    IMetricOmmPoolFactoryActions(pool).setBinAdditionalFees(bin, addFeeBuyE6, addFeeSellE6);
}
```

Alternatively, enforce the cap at the pool level inside `setBinAdditionalFees`, though the factory is the correct policy layer per the design. [2](#0-1)

### Proof of Concept

```solidity
// Foundry test sketch
function test_binFeeBypassesAdminCap() public {
    // Factory owner lowers the admin spread cap to 1% (10_000 in E6)
    factory.setFeeCaps(200_000, 10_000, 1_000_000, 1_000_000);

    // Pool admin sets per-bin fee to uint16.max (65535 ≈ 6.5535%)
    vm.prank(admin);
    factory.setPoolBinAdditionalFees(pool, 0, type(uint16).max, type(uint16).max);

    // Verify stored value exceeds maxAdminSpreadFeeE6
    (,,, uint16 buyFee, uint16 sellFee) = PoolStateLibrary._binState(pool, 0);
    assertEq(buyFee, type(uint16).max);  // 65535 >> 10000 cap
    assertGt(uint256(buyFee), factory.maxAdminSpreadFeeE6());

    // Perform a swap and assert effective fee exceeds the cap
    // ... swap call, measure amountIn vs amountOut vs oracle mid-price ...
}
