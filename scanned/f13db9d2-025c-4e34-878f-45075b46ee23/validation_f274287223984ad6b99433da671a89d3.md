### Title
`SwapAllowlistExtension` gates the router address instead of the actual user, enabling full allowlist bypass via the router ŌĆö (`metric-periphery/contracts/extensions/SwapAllowlistExtension.sol`)

---

### Summary

`SwapAllowlistExtension.beforeSwap` checks the `sender` argument, which the pool sets to `msg.sender` of the `pool.swap()` call. When any swap is routed through `MetricOmmSimpleRouter`, `msg.sender` of `pool.swap()` is the **router contract**, not the end user. If the pool admin adds the router address to the allowlist (the natural action to enable router-based swaps for curated pools), every user ŌĆö including those explicitly not allowlisted ŌĆö can bypass the per-user gate by routing through the router.

---

### Finding Description

**Actor binding in the extension:** [1](#0-0) 

`beforeSwap` receives `sender` as its first argument and checks `allowedSwapper[msg.sender][sender]`, where `msg.sender` is the pool and `sender` is whoever called `pool.swap()`.

**What the pool passes as `sender`:** [2](#0-1) 

The pool always passes `msg.sender` ŌĆö the direct caller of `pool.swap()` ŌĆö as `sender`.

**What the router passes to the pool:** [3](#0-2) 

The router calls `pool.swap(params.recipient, ...)` directly. The pool's `msg.sender` is therefore the **router contract address**, not the end user. The end user's address is only stored in the router's transient callback context (`_setNextCallbackContext(..., msg.sender, ...)`), which is never visible to the extension.

The same pattern holds for `exactInput` multi-hop: [4](#0-3) 

And for `exactOutput` recursive callbacks: [5](#0-4) 

In every router path, `sender` seen by the extension is the router, not the user.

---

### Impact Explanation

A pool admin who deploys a curated pool with `SwapAllowlistExtension` and wants allowlisted users to be able to use the standard router has only one option: add the router address to `allowedSwapper[pool]`. Once the router is allowlisted, **every** address ŌĆö including those explicitly excluded from the per-user allowlist ŌĆö can bypass the gate by calling any router entry point (`exactInputSingle`, `exactInput`, `exactOutputSingle`, `exactOutput`). The allowlist is rendered completely ineffective for all router-mediated swaps. This breaks the core curation invariant of the extension: that only approved addresses may trade in the pool.

---

### Likelihood Explanation

The trigger is a reasonable, expected admin action. The `SwapAllowlistExtension` is the supported periphery mechanism for curated pools. Any admin who wants allowlisted users to access the standard router will add the router to the allowlist. The bypass is then immediately available to any unprivileged user with no further preconditions. The admin has no way to simultaneously (a) allow allowlisted users to use the router and (b) block non-allowlisted users from using the router, because the extension has no visibility into the actual payer.

---

### Recommendation

The `beforeSwap` hook should gate on the **economic actor** (the payer), not the intermediary. Two options:

1. **Pass the payer through `extensionData`**: The router encodes the real user address in `extensionData`; the extension reads and verifies it. This requires router cooperation and is trust-dependent.

2. **Check `recipient` instead of `sender`**: For single-hop swaps the recipient is often the real user, but this breaks for multi-hop where intermediate recipients are the router itself.

3. **Preferred ŌĆö separate the allowlist key**: Change `allowedSwapper[pool][sender]` to `allowedSwapper[pool][payer]` and require the router to forward the payer identity in a standardized field of `extensionData`, validated against a registry of trusted routers. This preserves the operator pattern while correctly attributing the economic action.

---

### Proof of Concept

```
Setup:
  pool configured with SwapAllowlistExtension
  admin allowlists: userA (KYC'd)
  admin also adds router to allowlist (to let userA use the router)

Attack:
  userB (not KYC'd) calls:
    router.exactInputSingle({pool: pool, ..., extensionData: ""})

  Execution trace:
    router.exactInputSingle()          [msg.sender = userB]
      pool.swap(recipient, ...)        [msg.sender = router]
        _beforeSwap(sender=router, ...)
          SwapAllowlistExtension.beforeSwap(sender=router, ...)
            allowedSwapper[pool][router] == true  ŌåÉ passes
    swap executes for userB
```
