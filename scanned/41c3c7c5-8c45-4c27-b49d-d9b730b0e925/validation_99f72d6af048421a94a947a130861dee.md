### Title
Single `MAX_REF_STALENESS` applied to both feeds in synthetic mode allows stale quote-feed prices to reach pool swaps — (`AnchoredPriceProvider.sol`)

---

### Summary

`AnchoredPriceProvider` supports a synthetic ratio mode (`quoteFeedId != 0`) where the pool price is computed as `price(baseFeedId) / price(quoteFeedId)`. Both feeds are validated for staleness using the same single immutable `MAX_REF_STALENESS`. However, `AnchoredProviderFactory.createAnchoredProvider()` validates `maxRefStaleness` only against the envelope for `baseFeedId`'s class — the `quoteFeedId` has no envelope-bound staleness check. When the quote feed has a shorter heartbeat than `MAX_REF_STALENESS`, a stale quote-feed price passes the staleness guard and reaches the synthetic ratio computation, producing a bad bid/ask that the pool uses for swap execution.

---

### Finding Description

`AnchoredPriceProvider._readLeg()` applies the same `MAX_REF_STALENESS` to every feed it reads: [1](#0-0) 

Both the base and quote legs are dispatched through `_readLeg` in `_getBidAndAskPrice()`: [2](#0-1) 

`MAX_REF_STALENESS` is a single immutable set at construction: [3](#0-2) 

In `AnchoredProviderFactory.createAnchoredProvider()`, the staleness envelope is looked up using only `baseFeedId`'s class — `quoteFeedId` is never consulted: [4](#0-3) 

The factory's `setEnvelope` / `setFeedClass` machinery is keyed on `feedId`, so a separate, tighter envelope for the quote feed exists in the system — it is simply never queried during provider creation: [5](#0-4) 

**Concrete scenario**: A provider is created for BTC/ETH = BTC/USD ÷ ETH/USD. The BTC/USD class envelope allows `stalenessMax = 3600 s` (1 hour). The creator sets `maxRefStaleness = 3600`. ETH/USD has a 60-second heartbeat. When ETH/USD goes 61 seconds without an update, `_readLeg(quoteFeedId)` evaluates `(nowTs − refTime) > 3600` → false, so the stale ETH/USD price is accepted. The synthetic ratio BTC/ETH is computed from a stale denominator and the resulting bid/ask is wrong.

---

### Impact Explanation

Bad-price execution: the pool's bid/ask is derived from a stale quote-feed price. Swappers trade at an incorrect synthetic ratio — they can receive more output than the oracle curve permits (loss to the pool/LPs) or be forced to pay more input than warranted (loss to the trader). This is a direct fund-impacting consequence of stale oracle data reaching swap settlement.

---

### Likelihood Explanation

`createAnchoredProvider` is permissionless within the envelope bounds. Any creator can deploy a synthetic provider with `quoteFeedId` set to a feed whose heartbeat is shorter than the base feed's `stalenessMax`. The factory provides no guard against this combination. Pyth and Chainlink publish feeds with widely varying update frequencies (seconds to hours), making the mismatch easy to encounter in practice.

---

### Recommendation

In `AnchoredProviderFactory.createAnchoredProvider()`, when `quoteFeedId != 0`, look up `feedClass[quoteFeedId]` and validate `maxRefStaleness` against that envelope's `stalenessMax` as well — taking the stricter (smaller) of the two upper bounds:

```solidity
if (quoteFeedId != bytes32(0)) {
    bytes32 quoteClassId = feedClass[quoteFeedId];
    if (quoteClassId == bytes32(0)) quoteClassId = DEFAULT_CLASS;
    Envelope storage qEnv = envelopes[quoteClassId];
    if (!qEnv.exists) revert EnvelopeNotFound(quoteClassId);
    if (maxRefStaleness > qEnv.stalenessMax) revert ParamsOutOfEnvelope();
}
```

Alternatively, store a separate `MAX_QUOTE_REF_STALENESS` immutable in `AnchoredPriceProvider` and apply it in `_readLeg` when reading the quote leg.

---

### Proof of Concept

1. Admin calls `setFeedClass(BTC_USD_FEED, BTC_CLASS)` and `setEnvelope(BTC_CLASS, {stalenessMin: 60, stalenessMax: 3600, ...})`.
2. Admin calls `setFeedClass(ETH_USD_FEED, ETH_CLASS)` and `setEnvelope(ETH_CLASS, {stalenessMin: 1, stalenessMax: 60, ...})`.
3. Creator calls `createAnchoredProvider(oracle, BTC_USD_FEED, ETH_USD_FEED, ..., maxRefStaleness=3600, ...)`. Factory validates `3600 <= envelopes[BTC_CLASS].stalenessMax` → passes. `ETH_CLASS` envelope is never checked.
4. The deployed `AnchoredPriceProvider` has `MAX_REF_STALENESS = 3600`.
5. ETH/USD oracle is not updated for 61 seconds. `refTime` for ETH/USD is now 61 seconds old.
6. Pool calls `getBidAndAskPrice()`. `_readLeg(ETH_USD_FEED)` checks `(nowTs − refTime) > 3600` → `61 > 3600` → false. Stale ETH/USD price accepted.
7. Synthetic ratio `BTC/ETH = BTC_price / stale_ETH_price` is computed and returned as the pool's bid/ask.
8. Swaps execute at the incorrect price, causing fund loss to LPs or traders.

```text
    function _getBidAndAskPrice() internal returns (uint128, uint128) {
        (uint256 mid, uint256 spreadBps, , bool ok) = _readLeg(baseFeedId);
        if (!ok) return (0, type(uint128).max);

        bytes32 _quote = quoteFeedId;
        if (_quote != bytes32(0)) {
            (uint256 mid2, uint256 spreadBps2, , bool ok2) = _readLeg(_quote);
            if (!ok2 || mid2 == 0) return (0, type(uint128).max);
            // Synthetic ratio (8-decimal): mid1 / mid2. Relative uncertainties of a ratio add.
            mid = Math.mulDiv(mid, ORACLE_DECIMALS, mid2);
            spreadBps += spreadBps2;
        }

        return _computeBidAsk(mid, spreadBps);
    }
```

```text
    function _readLeg(bytes32 feedId)
        internal returns (uint256 mid, uint256 spreadBps, uint256 refTime, bool ok)
    {
        (mid, spreadBps, , refTime) = IPricedOracle(address(offchainOracle)).price(feedId, msg.sender);

        // Stale reference → not ok. Clamping to a stale anchor is the one false-safety case.
        if (_isStale(refTime, block.timestamp, MAX_REF_STALENESS)) return (mid, spreadBps, refTime, false);

        // Basic validity — mid positive, spreadBps not the stalled/off-hours marker (the Chainlink oracle
        // writes spreadBps = ORACLE_BPS when an RWA market is closed).
        if (mid == 0 || spreadBps >= ORACLE_BPS) return (mid, spreadBps, refTime, false);

        // Per-leg price guard.
        (uint128 guardMin, uint128 guardMax) = offchainOracle.priceGuard(feedId);
        guardMax = guardMax == 0 ? type(uint128).max : guardMax;
        if (mid < guardMin || mid > guardMax) return (mid, spreadBps, refTime, false);

        ok = true;
    }
```
