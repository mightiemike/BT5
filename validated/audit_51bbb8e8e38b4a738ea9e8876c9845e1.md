Audit Report

## Title
`uint32` Truncation in `_afterTimelock` Allows Pool Admin to Bypass Stop-Loss Timelock Immediately — (`metric-periphery/contracts/extensions/OracleValueStopLossExtension.sol`)

## Summary

`_afterTimelock` computes `uint32(block.timestamp + oracleStopLossConfig[pool_].timelock)`. When a pool admin sets a `timelock` value large enough that the sum exceeds `type(uint32).max`, the result silently wraps to a small integer already in the past. Every subsequent `_requireElapsed` check then passes immediately, letting the admin apply any parameter change (drawdown, decay, watermarks) with zero delay — defeating the entire LP-protection mechanism.

## Finding Description

`_afterTimelock` at L297–299 performs the addition in `uint256` and then truncates to `uint32`:

```solidity
function _afterTimelock(address pool_) private view returns (uint32) {
    return uint32(block.timestamp + oracleStopLossConfig[pool_].timelock);
}
``` [1](#0-0) 

The elapsed guard at L301–303 compares `block.timestamp` (uint256) against the stored `uint32 executeAfter`. If `executeAfter` wrapped to any value ≤ current `block.timestamp`, the revert is never triggered:

```solidity
function _requireElapsed(uint32 executeAfter) private view {
    if (block.timestamp < executeAfter) revert OracleStopLossTimelockNotElapsed(executeAfter, block.timestamp);
}
``` [2](#0-1) 

Every propose function stores the result of `_afterTimelock`: [3](#0-2) [4](#0-3) [5](#0-4) 

`_validateDrawdown` and `_validateDecay` exist, but there is no `_validateTimelock` — neither in `initialize` nor in `proposeOracleStopLossTimelock`: [6](#0-5) [7](#0-6) 

The `timelock` field is `uint32` in `PoolStopLossConfig`, so any value up to `2^32 − 1` is syntactically valid: [8](#0-7) 

## Impact Explanation

With the timelock bypassed, the pool admin can atomically (same block as a swap):

- Set `drawdownE6 = 0` — disabling stop-loss entirely (`_afterSwapOracleStopLoss` returns early at L217 when `drawdown == 0`), allowing unlimited value drain per swap.
- Set `decayPerSecondE8` to maximum — rapidly eroding all high-watermarks so the guard never triggers.
- Set per-bin `highWatermarks` to 0 — making the guard trigger on every swap, freezing the pool.

LPs receive no advance warning and have no opportunity to exit. This is a direct admin-boundary break: pool admin exceeds the timelock constraint that is the sole LP-protection mechanism. [9](#0-8) 

## Likelihood Explanation

The pool admin is semi-trusted; "pool admin bypasses timelocks" is an explicitly allowed impact in the contest scope. The attack requires only two sequential admin transactions and no external conditions. `block.timestamp` on all live EVM chains is currently ~1.75 × 10⁹, so any `timelock` value above `type(uint32).max − block.timestamp` (≈ 2.54 × 10⁹ seconds, ~80 years) triggers the overflow — a value that is syntactically valid as `uint32`. The precondition is that the current timelock is 0 (a valid initial configuration set by the factory at pool creation), or the admin has already waited out the existing timelock to reset it to 0. [1](#0-0) 

## Recommendation

Add a `_validateTimelock` function and call it in both `initialize` and `proposeOracleStopLossTimelock`. Either validate that `block.timestamp + newTimelock` does not overflow `uint32`, or store and compare `executeAfter` as `uint256` throughout:

```solidity
// Option A: validate on propose/initialize
function _validateTimelock(uint32 newTimelock) private view {
    uint256 ts = block.timestamp + newTimelock;
    require(ts <= type(uint32).max, "timelock overflow");
}

// Option B: store executeAfter as uint256 in PoolStopLossSchedule and PendingHighWatermarks
```

Additionally, cap `newTimelock` to a protocol-defined ceiling (e.g., 365 days) to prevent unreasonably long timelocks. [3](#0-2) 

## Proof of Concept

Assume `block.timestamp = 1_753_142_400` (July 2026), initial timelock = 0.

**Step 1 — Install overflow-inducing timelock (executes immediately since current timelock = 0):**

```solidity
uint32 X = uint32(type(uint32).max - 1_753_142_400 + 101); // = 2_541_824_996

vm.prank(poolAdmin);
ext.proposeOracleStopLossTimelock(pool, X);
// executeAfter = uint32(1_753_142_400 + 0) = 1_753_142_400 (== block.timestamp, already elapsed)

vm.prank(poolAdmin);
ext.executeOracleStopLossTimelock(pool);
// oracleStopLossConfig[pool].timelock = X
```

**Step 2 — Propose drawdown = 0:**

```solidity
vm.prank(poolAdmin);
ext.proposeOracleStopLossDrawdown(pool, 0);
// executeAfter = uint32(1_753_142_400 + 2_541_824_996)
//              = uint32(4_294_967_396) = uint32(4_294_967_396 % 2^32) = 100
```

**Step 3 — Execute immediately (no delay elapsed):**

```solidity
vm.prank(poolAdmin);
ext.executeOracleStopLossDrawdown(pool);
// _requireElapsed(100): block.timestamp (1.75B) < 100 → false → passes
// drawdownE6 = 0 — stop-loss disabled with zero timelock delay
``` [10](#0-9) [11](#0-10)
