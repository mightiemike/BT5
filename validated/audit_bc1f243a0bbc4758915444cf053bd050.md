### Title
`run_mainchain_gc` Has No Caller Access Control — Any Unprivileged Account Can Aggressively Prune the Mainchain - (File: `contract/src/lib.rs`)

### Summary

`run_mainchain_gc` is a public NEAR contract function that permanently deletes mainchain block headers from on-chain storage. It carries only a `#[pause]` attribute, which gates access exclusively when the contract is paused. When the contract is running normally (unpaused), any unprivileged NEAR account can call it with an arbitrary `batch_size`, triggering irreversible deletion of mainchain headers. This breaks SPV proof verification for pruned blocks and disables chain reorganization near the pruned region.

### Finding Description

`run_mainchain_gc` is declared as a plain `pub fn` with only `#[pause(except(roles(Role::UnrestrictedRunGC)))]`: [1](#0-0) 

The `#[pause]` attribute from `near-plugins` only restricts callers when the contract is in a paused state. When the contract is unpaused (its normal operating mode), the attribute is a no-op with respect to caller identity — any NEAR account can invoke the function. There is no `#[trusted_relayer]`, no `acl_require_role`, and no `predecessor_account_id` check.

The function body permanently removes entries from `mainchain_height_to_header` and `headers_pool`, then advances `mainchain_initial_blockhash`: [2](#0-1) 

Compare this to `submit_blocks`, which is correctly guarded by both `#[pause]` and `#[trusted_relayer]`: [3](#0-2) 

`run_mainchain_gc` is also called internally from `submit_blocks` at line 181, but that internal call path does not prevent the function from being called externally by anyone.

**Exploit path:**
1. Attacker (any NEAR account, no role required) calls `run_mainchain_gc` with `batch_size = u64::MAX` while the contract is unpaused.
2. The GC logic computes `total_amount_to_remove = amount_of_headers_we_store - gc_threshold` and removes up to `min(total_amount_to_remove, batch_size)` blocks — bounded only by how many blocks exceed the threshold.
3. Deleted entries are gone from NEAR storage permanently; there is no undo path.
4. The attacker can repeat this call every time new blocks are submitted (pushing the stored count above `gc_threshold` again), continuously pruning the oldest available mainchain blocks.

### Impact Explanation

**SPV proof verification is permanently broken for pruned blocks.** `verify_transaction_inclusion` and `verify_transaction_inclusion_v2` look up the target block in `mainchain_header_to_height`: [4](#0-3) 

Once a block is pruned, its entry is removed from `mainchain_header_to_height`, causing this lookup to panic with `"block does not belong to the current main chain"`. Any downstream contract or user relying on SPV proofs for transactions in pruned blocks receives a permanent denial of service.

**Chain reorganization is permanently broken near the pruned region.** The project's own documentation states: *"If mainchain blocks near the fork point have been garbage collected, reorg will fail — the contract panics with `PrevBlockNotFound` when it cannot walk the chain back to the common ancestor."* [5](#0-4) 

An attacker who prunes aggressively can ensure that any pending fork cannot be resolved, permanently locking the contract into a state where the canonical chain cannot be updated via reorg.

### Likelihood Explanation

Exploitation requires no privileged keys, no staked tokens, no special role, and no prior setup. Any NEAR account can call `run_mainchain_gc` at any time the contract is unpaused. The call costs only standard NEAR gas fees. The attack is trivially repeatable and the damage is irreversible.

### Recommendation

Restrict `run_mainchain_gc` to callers holding a designated role (e.g., `Role::DAO` or a new `Role::GCManager`), mirroring the pattern already used for `submit_blocks`. The `#[pause]` attribute should be supplemented with an explicit role check:

```rust
#[pause(except(roles(Role::UnrestrictedRunGC)))]
pub fn run_mainchain_gc(&mut self, batch_size: u64) {
    // Add at the top:
    require!(
        self.acl_has_role(Role::GCManager, &env::predecessor_account_id())
            || self.acl_has_role(Role::DAO, &env::predecessor_account_id()),
        "Caller is not authorized to run GC"
    );
    // ...
}
```

Alternatively, make `run_mainchain_gc` a private/internal function and expose GC only through `submit_blocks`, which is already properly guarded by `#[trusted_relayer]`.

```rust
    /// Public call to run GC on a mainchain.
    /// `batch_size` is how many block headers should be removed in the execution
    ///
    /// # Panics
    /// If initial blockheader or tip blockheader are not in a header pool
    #[pause(except(roles(Role::UnrestrictedRunGC)))]
    pub fn run_mainchain_gc(&mut self, batch_size: u64) {
        let initial_blockheader = self
            .headers_pool
            .get(&self.mainchain_initial_blockhash)
            .unwrap_or_else(|| env::panic_str(ERR_KEY_NOT_EXIST));

        let tip_blockheader = self
            .headers_pool
            .get(&self.mainchain_tip_blockhash)
            .unwrap_or_else(|| env::panic_str(ERR_KEY_NOT_EXIST));

        let amount_of_headers_we_store =
            tip_blockheader.block_height - initial_blockheader.block_height + 1;

        if amount_of_headers_we_store > self.gc_threshold {
            let total_amount_to_remove = amount_of_headers_we_store - self.gc_threshold;
            let selected_amount_to_remove = std::cmp::min(total_amount_to_remove, batch_size);

            let start_removal_height = initial_blockheader.block_height;
            let end_removal_height = initial_blockheader.block_height + selected_amount_to_remove;
            env::log_str(&format!(
                "Num of blocks to remove {selected_amount_to_remove}"
            ));

            for height in start_removal_height..end_removal_height {
                let blockhash = &self
                    .mainchain_height_to_header
                    .get(&height)
                    .unwrap_or_else(|| env::panic_str(ERR_KEY_NOT_EXIST));

                self.remove_block_header(blockhash);
                self.mainchain_height_to_header.remove(&height);
            }

            self.mainchain_initial_blockhash = self
                .mainchain_height_to_header
                .get(&end_removal_height)
                .unwrap_or_else(|| env::panic_str(ERR_KEY_NOT_EXIST));
        }
    }
```
