### Title
Integer Overflow in `op_unknown` Cost Multiplier Produces Undercharged Execution Cost - (`File: src/more_ops.rs`)

### Summary

`op_unknown` in `src/more_ops.rs` computes a final cost by multiplying a base cost (bounded by `max_cost`) by `cost_multiplier + 1`, where `cost_multiplier` is an attacker-controlled `u64` derived from the opcode bytes. In Rust release mode, this multiplication silently wraps on overflow, producing a cost value that is far smaller than the true cost — potentially zero — allowing the operation to pass the `u32::MAX` guard and return `Ok` with a drastically undercharged cost.

---

### Finding Description

`op_unknown` handles unknown opcodes in lenient mode. The opcode byte layout encodes a `cost_multiplier` (up to `u32::MAX` = 4,294,967,295) and a `cost_function` (0–3). The function computes a base cost from the arguments, verifies it against `max_cost`, then scales it:

```rust
// src/more_ops.rs lines 260-266
check_cost(cost, max_cost)?;
cost *= cost_multiplier + 1;
if cost > u32::MAX as u64 {
    Err(EvalErr::Invalid(o))?
} else {
    Ok(Reduction(cost as Cost, allocator.nil()))
}
```

`Cost` is `u64` (`src/cost.rs` line 3). `cost_multiplier` is extracted via `u32_from_u8` (max value `u32::MAX`), cast to `u64`, so `cost_multiplier + 1` reaches at most `2^32 = 4,294,967,296`. After `check_cost` passes, `cost` is bounded by `max_cost` (e.g., Chia's block limit of 11,000,000,000 ≈ `2^33.4`). The multiplication `cost *= cost_multiplier + 1` is **not** guarded by `checked_mul` or `saturating_mul`. In Rust release mode, overflow wraps silently (two's complement).

**Concrete trigger:**
- Set `cost_function = 1` (ARITH-like) and `cost_multiplier = u32::MAX` (opcode prefix bytes `0xff 0xff 0xff 0xff`).
- Pass ~13,421,771 zero-length atom arguments plus 159 total bytes so that the base cost equals exactly `2^32 = 4,294,967,296` (within the 11-billion `max_cost` limit).
- `check_cost(2^32, max_cost)` passes.
- `cost *= 2^32` → `2^32 * 2^32 = 2^64 ≡ 0 (mod 2^64)` — wraps to **0**.
- `0 > u32::MAX` is false → returns `Ok(Reduction(0, nil))` with **zero cost**.

The `assert!(cost > 0)` at line 258 fires before the multiplication and does not prevent the post-multiplication value from being 0.

---

### Impact Explanation

The CLVM cost model is the sole resource-bounding mechanism for smart coin execution on Chia. Returning a cost of 0 (or any wrapped-down value ≤ `u32::MAX`) for an operation that should cost billions means:

- An attacker can embed many such unknown ops in a CLVM program and the cumulative reported cost stays near zero, allowing programs that should exceed the block cost limit to pass validation.
- This breaks consensus: nodes running debug builds (which panic on overflow) would reject the program, while release builds accept it — a direct consensus divergence.
- Any validator or mempool that enforces the cost limit based on the returned `Reduction` cost would be bypassed.

---

### Likelihood Explanation

The trigger is fully attacker-controlled via the CLVM program bytes passed to `run_program` / `run_serialized_chia_program`. No privileged access, admin action, or external condition is required. The opcode bytes and argument list are both attacker-supplied. The only constraint is that `max_cost` must exceed `2^32` (≈ 4.3 billion), which is satisfied by Chia's standard block cost limit of 11 billion. The argument count needed (~13 million zero-length atoms) is within the allocator's `MAX_NUM_ATOMS = 62,500,000` limit.

---

### Recommendation

Replace the unchecked multiplication with a checked or saturating variant, and treat overflow as an invalid opcode:

```rust
// Replace:
cost *= cost_multiplier + 1;

// With:
cost = cost
    .checked_mul(cost_multiplier + 1)
    .ok_or(EvalErr::Invalid(o))?;
```

This ensures that any overflow is caught and returns an error rather than silently wrapping to a small value.

### Proof of Concept

**Opcode bytes:** `[0xff, 0xff, 0xff, 0xff, 0x40]`
- Prefix `[0xff, 0xff, 0xff, 0xff]` → `cost_multiplier = u32::MAX = 4,294,967,295` via `u32_from_u8`
- Last byte `0x40` → `cost_function = (0x40 & 0b11000000) >> 6 = 1` (ARITH-like)

**Arguments:** 13,421,771 empty atoms + one atom of 159 bytes

**Cost computation (cost_function = 1):**
```
cost = 99 + 320 × 13,421,771 + 3 × 159
     = 99 + 4,294,966,720 + 477
     = 4,294,967,296
     = 2^32
```

**After `check_cost(2^32, 11_000_000_000)` passes:**
```
cost *= cost_multiplier + 1
     = 2^32 * 2^32
     = 2^64
     ≡ 0  (mod 2^64, Rust release-mode wrapping)
```

**Guard check:** `0 > u32::MAX` → false → `Ok(Reduction(0, nil))` returned.
