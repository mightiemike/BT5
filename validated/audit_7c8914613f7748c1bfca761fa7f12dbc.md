## Title
Sierra Version Downgrade in DeclareTransactionV3 Bypasses `execute_max_sierra_gas` Cap — (`crates/starknet_api/src/executable_transaction.rs`, `crates/apollo_gateway/src/stateless_transaction_validator.rs`, `crates/apollo_transaction_converter/src/transaction_converter.rs`)

## Summary

An unprivileged attacker can submit a `DeclareTransactionV3` whose `sierra_program[0..3]` encodes a version below `min_sierra_version_for_sierra_gas` (e.g., `[1, 5, 0]`). The gateway's Sierra version gate accepts versions in `[1.1.0, 1.9.0]`, so 1.5.0 passes. The class manager compiles the class and stores `ContractClass::V1((casm, SierraVersion(1.5.0)))`. When the contract is later invoked, `CompiledClassV1::tracked_resource()` returns `TrackedResource::CairoSteps` (because `1.5.0 < min_sierra_version_for_sierra_gas = 1.7.0`), and the VM is initialized with `infinite_gas_for_vm_mode()` instead of the capped `execute_max_sierra_gas = 1_000_000_000`.

---

## Finding Description

**Step 1 — Gateway stateless validation passes.**

`validate_sierra_version()` checks that the version extracted from `sierra_program[0..3]` lies in `[min_sierra_version, max_sierra_version]`. The production config sets `min_sierra_version = 1.1.0` and `max_sierra_version = 1.9.0`. [1](#0-0) [2](#0-1) 

Version 1.5.0 satisfies `1.1.0 ≤ 1.5.0 ≤ 1.9.0`, so the check passes.

**Step 2 — Class manager compiles and stores the attacker-controlled version.**

`SierraCompiler::compile()` calls `class.get_sierra_version()` which calls `SierraVersion::extract_from_program()` — reading `sierra_program[0..3]` verbatim. The resulting `SierraVersion(1.5.0)` is stored in `ContractClass::V1((casm, sierra_version))`. [3](#0-2) [4](#0-3) 

**Step 3 — `compiled_class_hash` check does not cover the Sierra version.**

`convert_rpc_tx_to_internal()` validates `tx.compiled_class_hash == executable_class_hash_v2`, where `executable_class_hash_v2 = casm.hash(&HashVersion::V2)`. This hash covers only the CASM bytecode, not the `SierraVersion` stored alongside it. [5](#0-4) [6](#0-5) 

The attacker can compute the correct CASM hash for their Sierra program and supply it in the transaction. The check passes.

**Step 4 — `ClassInfo.sierra_version` is set to 1.5.0 at execution time.**

`convert_internal_rpc_tx_to_executable_tx()` re-reads the stored Sierra and calls `SierraVersion::extract_from_program(&sierra.sierra_program)` to populate `ClassInfo.sierra_version`. [7](#0-6) 

**Step 5 — `tracked_resource()` returns `CairoSteps`.**

`CompiledClassV1::tracked_resource()` compares `self.sierra_version` against `min_sierra_version_for_sierra_gas`. In VC 0.14.1, `min_sierra_version_for_sierra_gas = "1.7.0"`. Since `1.5.0 < 1.7.0`, the result is `TrackedResource::CairoSteps`. [8](#0-7) [9](#0-8) 

**Step 6 — VM is initialized with `infinite_gas_for_vm_mode()` instead of `execute_max_sierra_gas`.**

When `tracked_resource == CairoSteps`, the entry point's initial gas is set to `infinite_gas_for_vm_mode()` (a sentinel value representing no gas cap), bypassing the `execute_max_sierra_gas = 1_000_000_000` limit that applies in `SierraGas` mode. [10](#0-9) 

The OS flow test `test_resources_type` explicitly demonstrates this mechanism — overriding `sierra_program[0..3]` to `[1, 5, 0]` on a modern contract and confirming `TrackedResource::CairoSteps` and `gas_consumed == 0`: [11](#0-10) 

---

## Impact Explanation

A contract declared with a manipulated Sierra version executes with no per-transaction Sierra gas cap. The block bouncer still limits total steps per block, but the per-transaction `execute_max_sierra_gas` boundary (1 billion gas units in VC 0.14.1) is completely bypassed. This constitutes incorrect gas/resource accounting with economic impact: a single transaction can consume far more L2 gas than the protocol intends to allow, degrading block throughput and fairness, and potentially enabling griefing attacks against the sequencer.

---

## Likelihood Explanation

The attack requires only a valid account and fee payment. The Sierra version range accepted by the gateway (`1.1.0`–`1.9.0`) deliberately overlaps with the pre-`SierraGas` era. Any attacker who writes a Cairo contract using only libfuncs available in Sierra ≤ 1.6.x can declare it with version 1.5.0 and exploit this path. No privileged access is required.

---

## Recommendation

1. **Raise `min_sierra_version` in the gateway** to at least `min_sierra_version_for_sierra_gas` (1.7.0), so the gateway rejects any Sierra program that would execute in `CairoSteps` mode.
2. **Alternatively**, enforce at the class manager or transaction converter that `ClassInfo.sierra_version >= min_sierra_version_for_sierra_gas` for newly declared classes, returning an error otherwise.
3. **Do not rely solely on `compiled_class_hash`** to bind the Sierra version — the CASM hash is independent of the version metadata stored alongside it.

---

## Proof of Concept

```rust
// 1. Take any valid Sierra program that uses only ≤1.5.0 libfuncs.
// 2. Set sierra_program[0..3] = [Felt::from(1), Felt::from(5), Felt::from(0)].
// 3. Compute compiled_class_hash = CasmContractClass::from_contract_class(...).hash(V2).
// 4. Submit DeclareTransactionV3 { compiled_class_hash, contract_class: modified_sierra, ... }.
// Gateway accepts (1.5.0 ∈ [1.1.0, 1.9.0]).
// Class manager stores ContractClass::V1((casm, SierraVersion(1.5.0))).
// On invocation: tracked_resource() == CairoSteps, initial_gas == infinite_gas_for_vm_mode().
// execute_max_sierra_gas cap is NOT applied.
```

The existing test at `crates/starknet_os_flow_tests/src/tests.rs:2490–2526` confirms the mechanism end-to-end (bypassing the gateway, but demonstrating the execution-layer effect).

            })

        Ok(Self::new(version_components[0], version_components[1], version_components[2]))

    /// Returns whether this contract should run using Cairo steps or Sierra gas.
    pub fn tracked_resource(
        &self,
        min_sierra_version: &SierraVersion,
        last_tracked_resource: Option<&TrackedResource>,
    ) -> TrackedResource {
        let contract_tracked_resource = match self {
            Self::V0(_) => TrackedResource::CairoSteps,
            Self::V1(contract_class) => contract_class.tracked_resource(min_sierra_version),
            #[cfg(feature = "cairo_native")]
            Self::V1Native(contract_class) => {
                contract_class.casm().tracked_resource(min_sierra_version)
            }
        };
        match last_tracked_resource {
            // Once we ran with CairoSteps, we will continue to run using it for all nested calls.
            Some(TrackedResource::CairoSteps) => TrackedResource::CairoSteps,
            Some(TrackedResource::SierraGas) | None => contract_tracked_resource,
        }
    }

    /// Returns the tracked resource for calling this contract from within a context.
    pub fn get_current_tracked_resource(
        &self,
        context: &EntryPointExecutionContext,
    ) -> TrackedResource {
        self.tracked_resource(
            &context.versioned_constants().min_sierra_version_for_sierra_gas,
            context.tracked_resource_stack.last(),
        )
    }
