Audit Report

## Title
Off-by-one in `_validateDrawdown` allows `drawdownE6 = 1e6`, zeroing `floorMultiplier` and permanently disabling stop-loss — (`metric-periphery/contracts/extensions/OracleValueStopLossExtension.sol`)

## Summary

`_validateDrawdown` uses a strict `>` comparison, so `drawdownE6 = 1e6` (= `E6`) passes validation without reverting. When stored and applied, `floorMultiplier = E6 - drawdown = 0`, reducing the breach condition to `metric < 0` — impossible for `uint256` — so `OracleStopLossTriggered` is never emitted and the stop-loss is silently disabled for all future swaps. A pool admin exploiting this exceeds the intended drawdown cap, removing the sole on-chain guard protecting LP principal.

## Finding Description

**Validation off-by-one (L305–307):**

```solidity
function _validateDrawdown(uint256 drawdownE6) private pure {
    if (drawdownE6 > E6) revert OracleStopLossDrawdownTooLarge(drawdownE6);
}
``` [1](#0-0) 

`drawdownE6 == 1e6` satisfies `drawdownE6 > E6 → false`, so no revert. The value is accepted and stored via `executeOracleStopLossDrawdown`.

**Floor multiplier collapses to zero (L234):**

```solidity
uint256 floorMultiplier = E6 - drawdown;   // 1e6 - 1e6 = 0
``` [2](#0-1) 

**Breach check becomes `metric < 0` — always false for `uint256` (L333–335):**

```solidity
if (metric >= hwm) return (metric, false);
breached = metric < (hwm * floorMultiplier) / E6;  // metric < 0 → always false
return (hwm, breached);
``` [3](#0-2) 

**The zero-drawdown early-return guard (L217) does not fire** because `drawdown = 1e6 ≠ 0`, so the loop runs but never triggers a revert: [4](#0-3) 

**Exploit path:**
1. Pool admin calls `proposeOracleStopLossDrawdown(pool, 1e6)` — passes `_validateDrawdown` (L103–110).
2. Waits `timelock` seconds (zero if pool initialized with `timelock = 0` — no minimum is enforced).
3. Calls `executeOracleStopLossDrawdown(pool)` — writes `drawdownE6 = 1e6` to storage (L112–120).
4. All subsequent swaps execute `_afterSwapOracleStopLoss` with `floorMultiplier = 0`; no breach is ever detected regardless of value drained per bin. [5](#0-4) 

The `onlyPoolAdmin` modifier resolves the admin from the factory registry — this is the pool creator, a semi-trusted role distinct from the factory owner: [6](#0-5) 

## Impact Explanation

`OracleValueStopLossExtension` is the sole on-chain guard preventing per-bin value drain. With `floorMultiplier = 0`, the guard is a no-op: any swap that moves value out of a bin — including a fully draining swap — completes without reverting. LPs who deposited under the assumption that the stop-loss extension protects their principal lose that protection entirely, with no on-chain recourse after the timelock elapses. This constitutes a direct loss of LP principal protection and broken core pool functionality, meeting the Critical/High threshold under the allowed impact gate (admin-boundary break: pool admin exceeds the drawdown cap).

## Likelihood Explanation

- A pool admin who wishes to rug LPs has a direct, one-step path: propose `drawdownE6 = 1e6`, wait the timelock, execute.
- If the pool was initialized with `timelock = 0` (permitted by the initializer — no minimum timelock is enforced), the disable is atomic within a single block.
- Even with a non-zero timelock, LPs who are not actively monitoring `OracleStopLossDrawdownProposed` events have no automated protection.
- The pool admin role is assigned at pool creation and is a semi-trusted but externally controlled address, making this a realistic attack vector.

## Recommendation

Change the strict inequality to `>=` in `_validateDrawdown`:

```solidity
function _validateDrawdown(uint256 drawdownE6) private pure {
    if (drawdownE6 >= E6) revert OracleStopLossDrawdownTooLarge(drawdownE6);
}
```

This ensures `floorMultiplier` is always at least 1, keeping the breach check meaningful. Apply the same fix symmetrically to `_validateDecay` (L309–311), since `decayPerSecondE8 = E8` causes `_decayed` to return 0 immediately (via the `factor >= E8` branch), which similarly collapses watermarks. [7](#0-6) 

## Proof of Concept

```solidity
// Foundry test sketch
function test_drawdown_1e6_disables_stop_loss() public {
    // pool admin proposes 100% drawdown — passes _validateDrawdown (> not >=)
    vm.prank(poolAdmin);
    ext.proposeOracleStopLossDrawdown(pool, 1e6);

    // skip timelock (or 0 if initialized with timelock=0)
    skip(oracleStopLossConfig.timelock);

    vm.prank(poolAdmin);
    ext.executeOracleStopLossDrawdown(pool);

    // assert config stored
    assertEq(ext.oracleStopLossConfig(pool).drawdownE6, 1e6);

    // perform a fully draining swap — expect NO revert
    // OracleStopLossTriggered is never emitted because floorMultiplier=0
    // makes breached = metric < 0, always false for uint256
    pool.swap(/* drain params */);
    // assertion: swap succeeded, stop-loss never fired
}
```

  function executeOracleStopLossDrawdown(address pool_) external onlyPoolAdmin(pool_) {
    PoolStopLossSchedule storage sched = _initializedSchedule(pool_);
    if (sched.pendingDrawdownExecuteAfter == 0) revert OracleStopLossNoPendingDrawdown(pool_);
    _requireElapsed(sched.pendingDrawdownExecuteAfter);
    uint32 drawdown = sched.pendingDrawdownE6;
    oracleStopLossConfig[pool_].drawdownE6 = drawdown;
    (sched.pendingDrawdownE6, sched.pendingDrawdownExecuteAfter) = (0, 0);
    emit OracleStopLossDrawdownSet(pool_, drawdown);
  }
