### Title
Compile-time Static EVM `CHAIN_ID` in Wallet Contract Enables Cross-Chain Replay of ETH-Implicit Account Transactions - (File: `runtime/near-wallet-contract/implementation/wallet-contract/src/internal.rs`)

### Summary

The NEAR Wallet Contract, which backs every ETH-implicit account on NEAR, validates incoming EVM-style transactions against a compile-time constant `CHAIN_ID` baked into the WASM binary. Because this value is frozen at build time and never read from the live chain context, any NEAR chain fork that inherits the same binary will accept transactions signed for the original chain, enabling full cross-chain replay of every ETH-implicit account action.

### Finding Description

`runtime/near-wallet-contract/implementation/wallet-contract/src/internal.rs` declares the EVM chain ID as a compile-time constant:

```rust
pub const CHAIN_ID: u64 = std::include!("CHAIN_ID");
``` [1](#0-0) 

The `CHAIN_ID` file contains the literal integer `397` for mainnet: [2](#0-1) 

This constant is written into the WASM binary at build time by `build.rs`, which compiles three separate binaries — one per network — each with its chain ID frozen in: [3](#0-2) 

The compiled binaries are then embedded directly into the nearcore binary: [4](#0-3) 

At runtime, `validate_tx_relayer_data` enforces the chain ID check against this frozen constant:

```rust
if tx.chain_id != Some(CHAIN_ID) {
    return Err(Error::Relayer(RelayerError::InvalidChainId));
}
``` [5](#0-4) 

The `wallet_contract_magic_bytes` dispatcher selects which binary to serve based on the NEAR string chain ID, falling back to the localnet binary (chain ID `399`) for every chain that is not `"mainnet"` or `"testnet"`: [6](#0-5) 

This creates two concrete replay surfaces:

1. **Fork of mainnet/testnet**: If NEAR mainnet forks (analogous to ETH/ETHPoW), both chains inherit the same wallet contract WASM with `CHAIN_ID = 397`. The nonce stored in each ETH-implicit account's contract state is identical at the fork height. An attacker who observes a signed EVM transaction on chain A (nonce = N, chain_id = 397) can submit it verbatim on chain B before the account's nonce advances there.

2. **Any two non-mainnet/non-testnet chains**: All custom, private, or forked NEAR chains that do not carry the string `"mainnet"` or `"testnet"` as their genesis chain ID receive the localnet binary with `CHAIN_ID = 399`. Two such chains share an identical domain, so a transaction signed for one is valid on the other.

The nonce mechanism (`WalletContract.nonce`) only prevents replay *within* a single chain. It provides no cross-chain protection because both chains start from the same nonce state at the fork point.

### Impact Explanation

Every ETH-implicit account on NEAR is backed by the wallet contract. The `rlp_execute` entry point accepts an RLP-encoded EVM transaction and, after passing `validate_tx_relayer_data`, executes the encoded NEAR action (Transfer, FunctionCall, AddKey, DeleteKey). A replayed transaction can:

- Drain the account's NEAR balance via a `Transfer` action.
- Add an attacker-controlled access key via `AddKey`, giving permanent access to the account.
- Remove the owner's access key via `DeleteKey`. [7](#0-6) 

### Likelihood Explanation

The mainnet fork scenario requires a chain split, which is a low-probability but non-zero event (cf. ETH/ETHPoW). The custom-chain scenario (two non-mainnet/non-testnet NEAR chains sharing `CHAIN_ID = 399`) is immediately reachable by anyone running a private NEAR network or a fork with a different genesis chain ID string. The attack requires no privileged role: any party who observed a signed EVM transaction on one chain can replay it on the other.

### Recommendation

Replace the compile-time constant with a runtime read of the chain ID from the NEAR host environment. The `chain_id` host function is already exposed to contracts: [8](#0-7) 

Inside the wallet contract, replace:

```rust
pub const CHAIN_ID: u64 = std::include!("CHAIN_ID");
```

with a runtime call to `env::chain_id()` (or the equivalent NEAR SDK host binding), and derive the expected EVM chain ID from the live NEAR chain ID string at the time of each `rlp_execute` call. This eliminates the static domain and ensures that a transaction signed for one chain is always rejected on any other chain, regardless of fork history.

If gas efficiency is a concern, cache the resolved EVM chain ID in contract storage on first call and revalidate it on each invocation (analogous to the EIP-712 recommendation of caching and recomputing on chain ID change).

### Proof of Concept

1. Deploy a NEAR node with genesis `chain_id = "myfork"` (not `"mainnet"` or `"testnet"`). The runtime selects the localnet wallet contract binary (`CHAIN_ID = 399`).
2. Deploy a second independent NEAR node with a different genesis but also `chain_id = "otherfork"`. It also receives the localnet binary (`CHAIN_ID = 399`).
3. Create an ETH-implicit account on chain 1. Sign an EVM transaction (nonce=0, chain_id=399) transferring funds.
4. Submit the identical RLP bytes to the same ETH-implicit account address on chain 2. `validate_tx_relayer_data` passes (`tx.chain_id == Some(399) == CHAIN_ID`), nonce matches (both chains start at 0), and the transfer executes — draining the victim's funds on chain 2 without their knowledge or consent. [9](#0-8)

```rust
fn validate_tx_relayer_data<'a>(
    tx: &NormalizedEthTransaction,
    target: &'a AccountId,
    context: &ExecutionContext,
    expected_nonce: u64,
) -> Result<TargetKind<'a>, Error> {
    if tx.address.raw() != context.current_address {
        return Err(Error::Relayer(RelayerError::InvalidSender));
    }

    if tx.chain_id != Some(CHAIN_ID) {
        return Err(Error::Relayer(RelayerError::InvalidChainId));
    }

    let to = tx.to.ok_or(Error::User(UserError::EvmDeployDisallowed))?.raw();

    let target_kind = parse_target(target, context.current_address);

    // valid targets satisfy `to == target` or `to == hash(target)`
    let is_valid_target = match target_kind {
        TargetKind::CurrentAccount if to == context.current_address => {
            target == &context.current_account_id
        }
        TargetKind::EthImplicit(address) if to == address => {
            target.as_str()
                == format!("0x{}{}", hex::encode(address), context.current_account_suffix())
        }
        _ => to == account_id_to_address(target),
    };

    if !is_valid_target {
        return Err(Error::Relayer(RelayerError::InvalidTarget));
    }

    let nonce = if tx.nonce <= U64_MAX {
        tx.nonce.low_u64()
    } else {
        return Err(Error::Relayer(RelayerError::InvalidNonce));
    };
    if nonce != expected_nonce {
        return Err(Error::Relayer(RelayerError::InvalidNonce));
    }

    // Relayers must attach at least as much gas as the user requested.
    let gas_limit = if tx.gas_limit < U64_MAX { tx.gas_limit.as_u64() } else { u64::MAX };
    if env::prepaid_gas().as_gas() < gas_limit.saturating_mul(GAS_MULTIPLIER) {
        return Err(Error::Relayer(RelayerError::InsufficientGas));
    }

    Ok(target_kind)
}

    pub fn rlp_execute(
        &mut self,
        target: AccountId,
        tx_bytes_b64: String,
    ) -> PromiseOrValue<ExecuteResponse> {
        // To ensure user actions are executed in the desired order,
        // having multiple transactions in flight at the same time is
        // not allowed.
        if self.has_in_flight_tx {
            return PromiseOrValue::Value(ExecuteResponse {
                success: false,
                success_value: None,
                error: Some(
                    "Error: transaction already in progress, please try again later.".into(),
                ),
            });
        }
        let current_account_id = env::current_account_id();
        let predecessor_account_id = env::predecessor_account_id();
        let result = inner_rlp_execute(
            current_account_id.clone(),
            predecessor_account_id,
            target,
            tx_bytes_b64,
            &mut self.nonce,
        );

        match result {
            Ok(promise) => {
                self.has_in_flight_tx = true;
                PromiseOrValue::Promise(promise)
            }
            Err(Error::Relayer(_)) if env::signer_account_id() == current_account_id => {
                let promise = create_ban_relayer_promise(current_account_id);
                self.has_in_flight_tx = true;
                PromiseOrValue::Promise(promise)
            }
            Err(e) => PromiseOrValue::Value(e.into()),
        }
    }
