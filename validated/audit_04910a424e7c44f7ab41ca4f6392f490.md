Let me look at the block hash verification and signature validation in the sync path to understand what guards exist.

Looking at the full data flow from protobuf deserialization through p2p sync storage to the consensus orchestrator's `try_sync`, I need to check every guard in the chain.

**Step 1 – Protobuf conversion (no bounds check)** [1](#0-0) 

`l2_gas_consumed` is a plain `uint64` field cast directly with `.into()` — `u64::MAX` becomes `GasAmount(u64::MAX)` with no rejection. `next_l2_gas_price` is a `Uint128` converted via `u128::from(...)` — `{high:u64::MAX, low:u64::MAX}` becomes `GasPrice(u128::MAX)` with no rejection. Both values pass `TryFrom` and are stored in `BlockHeaderWithoutHash`.

**Step 2 – P2P sync client validation (no hash/signature check)** [2](#0-1) 

`parse_data_for_block` performs exactly two checks: block number ordering and `signatures.len() == ALLOWED_SIGNATURES_LENGTH`. There is **no** cryptographic verification of the block hash, no check that the `block_hash` field matches the computed hash of the header fields, and no ECDSA/Stark signature validation over the block hash. The header is then written directly to storage. [3](#0-2) 

**Step 3 – `try_sync` consumes `next_l2_gas_price` without a guard** [4](#0-3) 

`try_sync` reads the stored `SyncBlock` from the state sync service (backed by the same storage the p2p sync client writes to) and sets:

```rust
self.l2_gas_price = max(
    sync_block.block_header_without_hash.next_l2_gas_price,
    VersionedConstants::latest_constants().min_gas_price,

```

`max(u128::MAX, min_gas_price)` = `u128::MAX`. No guard exists.

**Step 4 – `calculate_next_base_gas_price` cannot recover quickly** [5](#0-4) 

With `price = u128::MAX` and `gas_used ≤ gas_target`, the price decreases by `u128::MAX / gas_price_max_change_denominator` per block. With `denominator = 333`, recovery to a normal price (~1 Gwei) takes approximately **22,000 blocks (~16 hours at 2.6 s/block)**.

The gateway's `validate_tx_l2_gas_price_within_threshold` computes `threshold = (min_gas_price_percentage/100) * u128::MAX`. Every transaction whose `max_price_per_unit` is below this threshold is rejected, effectively halting the sequencer's transaction intake for ~16 hours.

### Title
Malicious P2P Sync Peer Injects `next_l2_gas_price = u128::MAX` via Unverified Protobuf Header, Corrupting Fee Market for ~16 Hours — (`crates/apollo_protobuf/src/converters/header.rs`)

### Summary
The `TryFrom<protobuf::SignedBlockHeader>` conversion accepts any `next_l2_gas_price` value without bounds checking. The p2p sync client stores headers without verifying the block hash or signatures. The consensus orchestrator's `try_sync` reads `next_l2_gas_price` directly from the stored header and uses it as the starting gas price for the next block. An unprivileged sync peer can set `next_l2_gas_price = u128::MAX`, causing the sequencer to reject all transactions for approximately 16 hours.

### Finding Description
Three independent missing guards combine to form the attack:

1. **No bounds check in protobuf conversion.** `l2_gas_consumed` is a `uint64` field cast with `.into()` and `next_l2_gas_price` is a `Uint128` converted with `u128::from(...)`. Neither has a range check. [1](#0-0) 

2. **No block-hash or signature verification in the p2p sync client.** `parse_data_for_block` only checks block number ordering and signature vector length. It does not verify that `block_hash` matches the computed hash of the header fields, and does not cryptographically verify the signature over the block hash. The header is written directly to storage. [7](#0-6) 

3. **`try_sync` uses `next_l2_gas_price` from the synced header without validation.** The consensus orchestrator reads the stored block and sets `self.l2_gas_price = max(next_l2_gas_price, min_gas_price)`. With `next_l2_gas_price = u128::MAX`, this sets the fee market starting price to `u128::MAX`. [8](#0-7) 

The proto definition confirms `l2_gas_consumed` is a plain `uint64` and `next_l2_gas_price` is a `Uint128` with no protocol-level range constraint. [9](#0-8) 

### Impact Explanation
With `self.l2_gas_price = GasPrice(u128::MAX)`, `calculate_next_base_gas_price` decreases the price by `u128::MAX / 333 ≈ 1.02×10³⁶` per block. Starting from `u128::MAX ≈ 3.4×10³⁸`, approximately 22,000 blocks (~16 hours) are needed to return to a normal price. During this window, the gateway's `validate_tx_l2_gas_price_within_threshold` rejects every transaction whose `max_price_per_unit` is below `(min_gas_price_percentage/100) × u128::MAX`, which is every real transaction. This is incorrect fee/gas accounting with direct economic impact: the sequencer cannot include any user transactions. [10](#0-9) 

### Likelihood Explanation
The attack requires only that the victim node enters sync mode (falls behind consensus) while connected to a malicious p2p peer. This is a realistic scenario during network partitions, restarts, or deliberate latency attacks. No validator or operator privileges are needed — any node on the p2p network can respond to header queries.

### Recommendation
1. **Verify block hash in the p2p sync client.** Before storing a `SignedBlockHeader`, compute the block hash from the header fields and compare it to the `block_hash` field. Reject headers where they do not match.
2. **Verify the signature.** Call `verify_block_signature` (already implemented in `starknet_api::block`) against the known sequencer public key before storing. [11](#0-10) 
3. **Add a range check in `try_sync`.** Clamp `next_l2_gas_price` to a protocol-defined maximum (e.g., `max_block_size × max_gas_price_per_unit`) before assigning it to `self.l2_gas_price`.
4. **Add a range check in the protobuf converter.** Reject `next_l2_gas_price` values that exceed a protocol-defined ceiling.

```rust
pub fn calculate_next_base_gas_price(
    price: GasPrice,
    gas_used: GasAmount,
    gas_target: GasAmount,
    min_gas_price: GasPrice,
) -> GasPrice {
    let versioned_constants = VersionedConstants::latest_constants();
    assert!(
        gas_target < versioned_constants.max_block_size,
        "Gas target must be lower than max block size."
    );
    assert!(gas_target.0 > 0, "Gas target must be greater than zero.");
    assert!(
        versioned_constants.gas_price_max_change_denominator > 0,
        "Denominator constant must be greater than zero."
    );

    // If the current price is below the minimum, apply a gradual adjustment and return early.
    // This allows the price to increase by at most 1/MIN_GAS_PRICE_INCREASE_DENOMINATOR per block.
    if price < min_gas_price {
        let max_increase = price.0 / MIN_GAS_PRICE_INCREASE_DENOMINATOR;
        let adjusted = price.0 + max_increase;
        // Cap at min_gas_price to avoid overshooting
        let adjusted_price = adjusted.min(min_gas_price.0);
        info!(
            "Fee Market: Price {} below minimum gas price {}, adjusted price: {} )",
            price.0, min_gas_price.0, adjusted_price
        );
        return GasPrice(adjusted_price);
    }

    // Use U256 to avoid overflow, as multiplying a u128 by a u64 remains within U256 bounds.
    let gas_delta = U256::from(gas_used.0.abs_diff(gas_target.0));
    let gas_target_u256 = U256::from(gas_target.0);
    let price_u256 = U256::from(price.0);

    // Calculate price change by multiplying first, then dividing. This avoids the precision loss
    // that occurs when dividing before multiplying.
    let denominator =
        gas_target_u256 * U256::from(versioned_constants.gas_price_max_change_denominator);
    let price_change = (price_u256 * gas_delta) / denominator;

    let adjusted_price_u256 =
        if gas_used > gas_target { price_u256 + price_change } else { price_u256 - price_change };

    // Sanity check: ensure direction of change is correct
    assert!(
        gas_used > gas_target && adjusted_price_u256 >= price_u256
            || gas_used <= gas_target && adjusted_price_u256 <= price_u256
    );

    // Price should not realistically exceed u128::MAX, bound to avoid theoretical overflow.
    let adjusted_price = u128::try_from(adjusted_price_u256).unwrap_or(u128::MAX);
    GasPrice(max(adjusted_price, min_gas_price.0))
```
