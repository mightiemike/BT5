### Title
User Funds Locked Indefinitely When Bonus Pool Is Empty During `TierUndelegate` — (`x/tieredrewards/keeper/msg_server.go`, `x/tieredrewards/keeper/claim_rewards.go`)

### Summary

`TierUndelegate` unconditionally calls `claimRewards`, which calls `processEventsAndClaimBonus`, which calls `sufficientBonusPoolBalance`. If the bonus pool (`RewardsPoolName` module account) has insufficient balance to cover accrued bonus rewards, `ErrInsufficientBonusPool` is returned and propagates all the way up to `TierUndelegate` with no special handling. The transaction reverts, and the user's staked tokens remain locked in the position with no alternative exit path.

### Finding Description

The call chain is:

**`TierUndelegate`** → **`claimRewards`** → **`processEventsAndClaimBonus`** → **`sufficientBonusPoolBalance`**

In `msg_server.go`, `TierUndelegate` calls `claimRewards` and propagates any error directly:

```go
pos, _, _, err = ms.claimRewards(ctx, pos)
if err != nil {
    return nil, err   // no special handling for ErrInsufficientBonusPool
}
``` [1](#0-0) 

Inside `processEventsAndClaimBonus`, once `totalBonus > 0`, the pool balance is checked:

```go
if err := k.sufficientBonusPoolBalance(ctx, bonusCoins); err != nil {
    return nil, err
}
``` [2](#0-1) 

`sufficientBonusPoolBalance` returns `ErrInsufficientBonusPool` when the pool balance is below the owed bonus:

```go
if !poolBalance.IsAllGTE(bonus) {
    return errorsmod.Wrapf(types.ErrInsufficientBonusPool, ...)
}
``` [3](#0-2) 

There is no `errors.Is(err, types.ErrInsufficientBonusPool)` guard anywhere in `TierUndelegate` that would allow it to skip the bonus payout and proceed with the undelegation. The same flaw applies to `TierRedelegate` and `ExitTierWithDelegation`, which also call `claimRewards` unconditionally. [4](#0-3) [5](#0-4) 

### Impact Explanation

A user who has accrued any non-zero bonus rewards cannot call `TierUndelegate`, `TierRedelegate`, or `ExitTierWithDelegation` while the bonus pool is empty. Their staked tokens are locked in the position indefinitely. The bonus pool can be drained by normal reward payouts over time (or by a large number of positions claiming simultaneously), and there is no governance-independent mechanism for a user to exit without bonus being paid. This is a direct, concrete fund-locking impact.

### Likelihood Explanation

The bonus pool is a finite module account (`RewardsPoolName`). As positions accrue rewards over time, the pool balance decreases. If the pool is not replenished (e.g., by governance or inflation routing), it will eventually be insufficient. Any user whose position has accrued even 1 unit of bonus at that point is locked. This is a realistic operational condition, not a theoretical one.

### Recommendation

In `TierUndelegate` (and similarly in `TierRedelegate` and `ExitTierWithDelegation`), add explicit handling for `ErrInsufficientBonusPool`: if the pool is insufficient, skip the bonus payout (forfeiting or deferring it) and allow the undelegation to proceed. Alternatively, split `claimRewards` into separate base and bonus steps so that the bonus step can be made non-blocking for exit operations.

### Proof of Concept

1. Create a position with a non-zero `BonusApy` tier and delegate tokens.
2. Advance block time so that `processEventsAndClaimBonus` computes `totalBonus > 0`.
3. Drain the `RewardsPoolName` module account to zero (e.g., via other positions claiming rewards).
4. Call `MsgTierUndelegate` for the position.
5. Observe: the transaction fails with `ErrInsufficientBonusPool` and the delegation remains active — the user cannot exit.
6. Assert: the user's tokens are locked until the pool is externally refilled.

The exact state delta: `RewardsPoolName` balance = 0, `pos.LastBonusAccrual` is in the past with `bonded=true`, so `totalBonus > 0`. `TierUndelegate` returns error, no state change occurs, delegation persists. [6](#0-5) [7](#0-6)

func (k Keeper) claimRewards(ctx context.Context, pos types.PositionState) (types.PositionState, sdk.Coins, sdk.Coins, error) {
	if !pos.IsDelegated() {
		return pos, sdk.NewCoins(), sdk.NewCoins(), nil
	}

	base, err := k.claimBaseRewards(ctx, pos)
	if err != nil {
		return types.PositionState{}, nil, nil, err
	}

	bonus, err := k.processEventsAndClaimBonus(ctx, &pos)
	if err != nil {
		return types.PositionState{}, nil, nil, err
	}

	return pos, base, bonus, nil
}
