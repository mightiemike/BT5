### Title
v8 Upgrade Migration Halts When Vesting-Owned Position Delegates to a Non-Bonded Validator — (File: x/tieredrewards/migrations/v2/migrate.go)

---

### Summary

The v8 upgrade migration unconditionally calls `ForceFullExitWithDelegation` for every vesting-owned tiered-rewards position. That function internally calls `transferDelegationFromPosition`, which hard-rejects any non-bonded validator with `ErrValidatorNotBonded`. If even one vesting-owned position is delegated to a jailed or tombstoned validator at upgrade height, the error propagates through the entire migration call chain with no recovery path, causing the upgrade handler to fail and the chain to halt.

---

### Finding Description

`Migrate` in `x/tieredrewards/migrations/v2/migrate.go` runs two sequential steps: [1](#0-0) 

The second step, `exitVestedAccountsPositions`, walks every position, collects those whose owner is a `VestingAccount`, then calls `ForceFullExitWithDelegation` for each — with no error isolation between positions: [2](#0-1) 

`ForceFullExitWithDelegation` (in `x/tieredrewards/keeper/force_exit.go`) delegates the actual delegation transfer to `transferDelegationFromPosition`: [3](#0-2) 

`transferDelegationFromPosition` (in `x/tieredrewards/keeper/transfer_delegation.go`) performs a hard bonded-status check before proceeding: [4](#0-3) 

`ErrValidatorNotBonded` is a terminal error — it is not caught or skipped anywhere in the migration path. It propagates back through `ForceFullExitWithDelegation` → `exitVestedAccountsPositions` → `Migrate` → `Migrate1to2`: [5](#0-4) 

In Cosmos SDK, an upgrade handler that returns a non-nil error causes the node to panic and halt at the upgrade height.

---

### Impact Explanation

**Corrupted invariant**: The chain upgrade itself. If the migration returns an error, the upgrade handler panics, the chain halts at the upgrade block, and no further blocks are produced. All vesting-owned positions remain in the pre-v8 state (with stale `DelegatorAddress` fields already backfilled by the first migration step), leaving module state partially migrated and the chain non-functional.

---

### Likelihood Explanation

Validators are routinely jailed for downtime (missing too many blocks) or double-signing. A vesting account holder who created a tiered-rewards position on a validator that happens to be jailed at the exact upgrade height triggers this path. No attacker action is required — ordinary validator downtime is sufficient. The probability is low per individual position but non-zero across the full set of vesting-owned positions on a live network.

---

### Recommendation

In `exitVestedAccountsPositions`, isolate per-position errors. At minimum, log and skip positions whose validator is not bonded rather than aborting the entire migration:

```go
if err := pk.ForceFullExitWithDelegation(ctx, posID); err != nil {
    if errors.Is(err, types.ErrValidatorNotBonded) {
        sdkCtx.Logger().Error("v8 migration: skipping force-exit for non-bonded validator",
            "position_id", posID, "error", err)
        continue
    }
    return fmt.Errorf("force-exit position %d: %w", posID, err)
}
```

A more complete fix would add a separate code path in `ForceFullExitWithDelegation` for non-bonded validators (e.g., queue an unbonding entry rather than an instant `Unbond`+`Delegate`), so the delegation is still returned to the owner without requiring the validator to be bonded.

---

### Proof of Concept

1. A vesting account `V` holds a tiered-rewards position `P` delegated to validator `VAL`.
2. Before the v8 upgrade height, `VAL` is jailed (e.g., due to downtime); its status transitions to `Unbonding`.
3. The upgrade handler runs `Migrate1to2` → `Migrate` → `exitVestedAccountsPositions`.
4. `exitVestedAccountsPositions` identifies `P` as vesting-owned and calls `ForceFullExitWithDelegation(ctx, P.Id)`.
5. `ForceFullExitWithDelegation` calls `transferDelegationFromPosition(ctx, posState, valAddr, positionAmount)`.
6. `transferDelegationFromPosition` fetches `VAL`, finds `!validator.IsBonded()` → returns `types.ErrValidatorNotBonded`.
7. The error propagates: `force_exit.go:62` → `migrate.go:99` → `migrate.go:37` → `migrations.go:18` → upgrade handler.
8. The upgrade handler returns a non-nil error; the node panics and halts at the upgrade block. [6](#0-5) [7](#0-6) [8](#0-7)

func exitVestedAccountsPositions(
	ctx context.Context,
	positions collections.Map[uint64, types.Position],
	ak AccountKeeper,
	pk PositionForceExiter,
) error {
	sdkCtx := sdk.UnwrapSDKContext(ctx)

	var toExit []uint64
	if err := positions.Walk(ctx, nil, func(posID uint64, pos types.Position) (bool, error) {
		ownerAddr, err := sdk.AccAddressFromBech32(pos.Owner)
		if err != nil {
			return false, fmt.Errorf("parse owner address for position %d: %w", posID, err)
		}
		acc := ak.GetAccount(ctx, ownerAddr)
		if acc == nil {
			return false, fmt.Errorf("owner account not found for position %d: %s", posID, ownerAddr.String())
		}
		if _, ok := acc.(sdkvesting.VestingAccount); ok {
			toExit = append(toExit, posID)
		}
		return false, nil
	}); err != nil {
		return fmt.Errorf("walk positions: %w", err)
	}

	for _, posID := range toExit {
		sdkCtx.Logger().Info("v8 migration: force-exit vesting-owned position", "position_id", posID)
		if err := pk.ForceFullExitWithDelegation(ctx, posID); err != nil {
			return fmt.Errorf("force-exit position %d: %w", posID, err)
		}
	}

	sdkCtx.Logger().Info("v8 migration: vesting-owned positions exited", "count", len(toExit))
	return nil
}

		"owner", posState.Owner,

func (k Keeper) transferDelegationFromPosition(ctx context.Context, pos types.PositionState, valAddr sdk.ValAddress, amount math.Int) (math.LegacyDec, math.LegacyDec, math.Int, error) {
	owner, err := sdk.AccAddressFromBech32(pos.Owner)
	if err != nil {
		return math.LegacyDec{}, math.LegacyDec{}, math.Int{}, errorsmod.Wrap(sdkerrors.ErrInvalidAddress, "invalid owner address")
	}

	posDelAddr, err := sdk.AccAddressFromBech32(pos.DelegatorAddress)
	if err != nil {
		return math.LegacyDec{}, math.LegacyDec{}, math.Int{}, errorsmod.Wrap(sdkerrors.ErrInvalidAddress, "invalid delegator address")
	}

	// Defensive
	if !pos.IsDelegated() {
		return math.LegacyDec{}, math.LegacyDec{}, math.Int{}, errorsmod.Wrapf(types.ErrPositionNotDelegated, "position %d is not delegated", pos.Id)
	}

	// Defensive
	isRedelegating, err := k.isRedelegating(ctx, pos.DelegatorAddress)
	if err != nil {
		return math.LegacyDec{}, math.LegacyDec{}, math.Int{}, err
	}
	if isRedelegating {
		return math.LegacyDec{}, math.LegacyDec{}, math.Int{}, errorsmod.Wrapf(types.ErrActiveRedelegation, "position %d has an active redelegation", pos.Id)
	}

	validator, err := k.stakingKeeper.GetValidator(ctx, valAddr)
	if errors.Is(err, stakingtypes.ErrNoValidatorFound) {
		return math.LegacyDec{}, math.LegacyDec{}, math.Int{}, types.ErrTransferDelegationDestNotFound
	} else if err != nil {
		return math.LegacyDec{}, math.LegacyDec{}, math.Int{}, err
	}

	if !validator.IsBonded() {
		return math.LegacyDec{}, math.LegacyDec{}, math.Int{}, types.ErrValidatorNotBonded
	}
