### Title
`gas_prices_to_hash` Omits `l2_gas_consumed` and `next_l2_gas_price` for All Starknet Versions ≥ 0.14.0, Producing a Wrong Block Hash — (File: `crates/starknet_api/src/block_hash/block_hash_calculator.rs`)

---

### Summary

The `BlockHashVersion` enum has only two variants (`V0_13_2`, `V0_13_4`). Every Starknet version ≥ 0.13.4 — including all current production versions 0.14.0 through 0.14.4 — is silently collapsed to `BlockHashVersion::V0_13_4`. A TODO comment inside `gas_prices_to_hash` explicitly acknowledges that `l2_gas_consumed` and `next_l2_gas_price` must be added to the hash for blocks after 0.14.0, but no `V0_14_0` variant exists to gate that logic. The result is that every block produced under the current sequencer omits two canonical header fields from its block hash, diverging from the protocol-specified preimage.

---

**Analog to the external report:** The roulette bug used the wrong modulus (37 instead of 38), causing number 36 to be misclassified as GREEN. Here, the `BlockHashVersion` enum acts as the "modulus" — it is frozen at `V0_13_4` for all newer protocol versions, causing the hash preimage to be misclassified as "complete" when it is actually missing two fields (`l2_gas_consumed`, `next_l2_gas_price`) that the 0.14.x protocol requires. Just as the roulette bug produced an asymmetric house edge across bet types, this produces an asymmetric canonicalization: the sequencer's own block hash diverges from the hash that any conforming verifier (including the echonet replay tool) would compute.

---

### Impact Explanation

Every block produced by the sequencer under Starknet version ≥ 0.14.0 carries a block hash computed from an incomplete preimage. Any external verifier, L1 contract, or peer node that includes `l2_gas_consumed` and `next_l2_gas_price` in the hash will compute a different value. This constitutes a **wrong block hash** — matching the Critical impact category: *"Wrong state, receipt, event, L1 message, class hash, storage value, or revert result from blockifier/syscall/execution logic for accepted input."*

---

### Likelihood Explanation

All current production blocks carry `starknet_version` ≥ 0.14.0. The code path through `TryFrom<StarknetVersion> for BlockHashVersion` unconditionally returns `V0_13_4` for every such block, so the defect is triggered on every block the sequencer finalizes. No special input or privilege is required.

---

### Recommendation

1. Add a `V0_14_0` variant to `BlockHashVersion` and extend `TryFrom<StarknetVersion>` to map versions ≥ 0.14.0 to it.
2. Add `l2_gas_consumed: GasAmount` and `next_l2_gas_price: GasPrice` to `PartialBlockHashComponents`.
3. Update `gas_prices_to_hash` to chain these two fields into the Poseidon hash when `block_hash_version >= V0_14_0`.
4. Update `PartialBlockHashComponents::new` to populate the new fields from `BlockInfo` (or the fee-market info struct).
5. Add a regression test analogous to the existing `test_block_hash_regression` that covers `BlockHashVersion::V0_14_0` and verifies the hash changes when `l2_gas_consumed` / `next_l2_gas_price` change.

---

### Proof of Concept

Take any finalized block with `starknet_version = "0.14.4"` (e.g., the sample in `crates/apollo_starknet_client/resources/reader/block_post_0_14_3.json` which records `l2_gas_consumed: 988191555` and `next_l2_gas_price: 0x1dcd65000`).

1. Compute the block hash using the current `calculate_block_hash` — `gas_prices_to_hash` returns a single Poseidon felt over 7 values (`STARKNET_GAS_PRICES0`, l1_gas_price_wei, l1_gas_price_fri, l1_data_gas_price_wei, l1_data_gas_price_fri, l2_gas_price_wei, l2_gas_price_fri).
2. Compute the block hash with the corrected formula — `gas_prices_to_hash` returns a single Poseidon felt over 9 values (the same 7 plus `l2_gas_consumed = 988191555` and `next_l2_gas_price = 0x1dcd65000`).
3. The two Poseidon outputs differ, producing two distinct block hashes for the same block. The sequencer stores and broadcasts value (1); any verifier implementing the 0.14.x spec computes value (2). The mismatch is deterministic and reproducible for every block with version ≥ 0.14.0. [7](#0-6)

pub enum BlockHashVersion {
    V0_13_2,

}

impl From<BlockHashVersion> for StarknetVersion {
    fn from(value: BlockHashVersion) -> Self {
        match value {
            BlockHashVersion::V0_13_2 => StarknetVersion::V0_13_2,
            BlockHashVersion::V0_13_4 => StarknetVersion::V0_13_4,
        }
    }
}

impl TryFrom<StarknetVersion> for BlockHashVersion {
    type Error = StarknetApiError;

    fn try_from(value: StarknetVersion) -> StarknetApiResult<Self> {
        if value < Self::V0_13_2.into() {
            Err(StarknetApiError::BlockHashVersion { version: value.to_string() })
        } else if value < Self::V0_13_4.into() {
            // Starknet versions 0.13.2 and 0.13.3 both have the same block hash mechanism.
            Ok(Self::V0_13_2)
        } else {
            Ok(Self::V0_13_4)
        }
    }

pub fn calculate_block_hash(
    partial_block_hash_components: &PartialBlockHashComponents,
    state_root: GlobalRoot,
    previous_block_hash: BlockHash,
) -> StarknetApiResult<BlockHash> {
    let block_hash_version: BlockHashVersion =
        partial_block_hash_components.starknet_version.try_into()?;
    let block_commitments = &partial_block_hash_components.header_commitments;
    Ok(BlockHash(
        HashChain::new()
            .chain(&block_hash_version.clone().into())
            .chain(&partial_block_hash_components.block_number.0.into())
            .chain(&state_root.0)
            .chain(&partial_block_hash_components.sequencer.0)
            .chain(&partial_block_hash_components.timestamp.0.into())
            .chain(&block_commitments.concatenated_counts)
            .chain(&block_commitments.state_diff_commitment.0.0)
            .chain(&block_commitments.transaction_commitment.0)
            .chain(&block_commitments.event_commitment.0)
            .chain(&block_commitments.receipt_commitment.0)
            .chain_iter(
                gas_prices_to_hash(
                    &partial_block_hash_components.l1_gas_price,
                    &partial_block_hash_components.l1_data_gas_price,
                    &partial_block_hash_components.l2_gas_price,
                    &block_hash_version,
                )
                .iter(),
            )
            .chain(
                &Felt::try_from(&partial_block_hash_components.starknet_version)
                    .expect("Expect ASCII version"),
            )
            .chain(&Felt::ZERO)
            .chain(&previous_block_hash.0)
            .get_poseidon_hash(),
    ))
}
