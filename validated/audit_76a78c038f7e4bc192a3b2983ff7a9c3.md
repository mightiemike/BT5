### Title
Deprecated `verify_transaction_inclusion` Accepts Attacker-Controlled Internal Merkle Node as Valid Transaction Proof — (`contract/src/lib.rs`)

### Summary
The public method `verify_transaction_inclusion` remains callable by any unprivileged NEAR account despite being marked deprecated. It accepts a caller-supplied `tx_id` that can be an internal Merkle tree node rather than a real transaction hash, causing the function to return `true` for a transaction that does not exist on Bitcoin. Downstream NEAR dApps that gate value-bearing actions on this result can be deceived.

### Finding Description
`verify_transaction_inclusion` is exposed as a `#[pause]`-gated but otherwise unrestricted public method. It accepts `ProofArgs` containing a caller-controlled `tx_id`, `tx_index`, and `merkle_proof`, then delegates directly to `compute_root_from_merkle_proof`: [1](#0-0) 

`compute_root_from_merkle_proof` treats `tx_id` as a leaf hash and walks the proof upward using `tx_index` to determine left/right ordering at each level: [2](#0-1) 

Because the function only checks that the computed root equals the stored `merkle_root`, any 32-byte value that is a genuine internal node of the Merkle tree satisfies the check. The code's own warning acknowledges this: [3](#0-2) 

The `#[deprecated]` attribute in Rust emits a compiler warning only; it does not restrict callers. The function carries no role guard beyond `#[pause]`, so any NEAR account can invoke it directly. [4](#0-3) 

### Impact Explanation
A NEAR dApp that calls `verify_transaction_inclusion` to confirm a Bitcoin payment before releasing funds or minting tokens receives a `true` result for a fabricated `tx_id`. The attacker never needs to broadcast any Bitcoin transaction; they only need to identify an internal Merkle node from a real block already accepted by the light client. The corrupted invariant is: **the function's `true` return value no longer guarantees that a transaction with the supplied hash exists on Bitcoin**.

### Likelihood Explanation
The entry path requires no privilege: any NEAR account can call the method. The required inputs (a real block hash accepted by the contract, the block's Merkle tree structure, and an internal node hash) are all publicly available from Bitcoin's blockchain. The attack is deterministic and requires no brute force.

### Recommendation
Remove `verify_transaction_inclusion` from the public ABI entirely, or add an explicit `env::panic_str` body that unconditionally aborts, preventing any caller from reaching the vulnerable logic. All callers must migrate to `verify_transaction_inclusion_v2`, which enforces a same-length coinbase proof at position 0, making the internal-node substitution infeasible without a hash collision. [5](#0-4) 

### Proof of Concept

Given a real Bitcoin block already stored in the contract with four transactions `T0, T1, T2, T3`:

```
Level 0 (leaves): T0,  T1,  T2,  T3
Level 1:          H01 = hash(T0‖T1),  H23 = hash(T2‖T3)
Level 2 (root):   R   = hash(H01‖H23)
```

The attacker calls `verify_transaction_inclusion` with:

```json
{
  "tx_id":            "<H01>",
  "tx_block_blockhash": "<real accepted block hash>",
  "tx_index":         0,
  "merkle_proof":     ["<H23>"],
  "confirmations":    1
}
```

`compute_root_from_merkle_proof(H01, 0, [H23])` computes `hash(H01 ‖ H23) = R`, which equals `header.block_header.merkle_root`. The function returns `true`. No transaction with hash `H01` exists on Bitcoin; `H01` is an internal node. The only guard that would have caught this — the non-empty proof check — is satisfied because `merkle_proof` has one element. [6](#0-5) [7](#0-6)

pub fn compute_root_from_merkle_proof(
    transaction_hash: H256,
    transaction_position: usize,
    merkle_proof: &Vec<H256>,
) -> H256 {
    let mut current_hash = transaction_hash;
    let mut current_position = transaction_position;

    for proof_hash in merkle_proof {
        if current_position % 2 == 0 {
            current_hash = compute_hash(&current_hash, proof_hash);
        } else {
            current_hash = compute_hash(proof_hash, &current_hash);
        }
        current_position /= 2;
    }

    current_hash
}
