### Title
Duplicate Bonus Reward Payment via Repeated Position ID in `ClaimTierRewards` - (File: x/tieredrewards/keeper/msg_server.go)

### Summary

`ClaimTierRewards` loads all positions from state before processing any of them, and never deduplicates `msg.PositionIds`. If the same position ID appears more than once in a single `MsgClaimTierRewards` transaction, the bonus reward computation runs twice against the same stale in-memory snapshot, causing the `RewardsPoolName` module account to pay out bonus rewards twice for a single accrual period.

### Finding Description

`ClaimTierRewards` in `msg_server.go` first collects all requested positions into a slice, then hands the slice to `claimRewardsAndUpdatesPositions`: [1](#0-0) 

```go
positions := make([]types.PositionState, 0, len(msg.PositionIds))
for _, posId := range msg.PositionIds {
    pos, err := ms.getPositionState(ctx, posId)
    ...
    positions = append(positions, pos)
}
totalBase, totalBonus, err := ms.claimRewardsAndUpdatesPositions(ctx, positions)
```

If `msg.PositionIds` contains the same ID twice, `positions` holds two independent copies of the same on-chain state, both with the same `LastBonusAccrual` and `LastEventSeq` values.

`claimRewardsAndUpdatesPositions` then iterates over the slice: [2](#0-1) 

For each element it calls `processEventsAndClaimBonus`, which:
1. Reads validator events since `pos.LastEventSeq` from the store.
2. Computes the bonus for every bonded segment up to `blockTime`.
3. Calls `bankKeeper.SendCoinsFromModuleToAccount` to transfer bonus from `RewardsPoolName` to the owner.
4. Updates `pos.LastBonusAccrual` and `pos.LastEventSeq` **in the local copy only**.
5. Persists the updated position via `setPosition`. [3](#0-2) 

Because the second element of `positions` is a separate copy loaded before the first iteration ran, its `LastBonusAccrual` and `LastEventSeq` are still the pre-claim values. The second call to `processEventsAndClaimBonus` therefore re-computes the full bonus for the same time window and issues a second `SendCoinsFromModuleToAccount` transfer. The final `setPosition` call for the second element overwrites the store with the same checkpoint that was already written by the first element, leaving no trace of the double payment.

The base-reward path (`claimBaseRewards` → `distributionKeeper.WithdrawDelegationRewards`) is **not** duplicated because the distribution module tracks what has already been withdrawn. The bonus path has no equivalent guard; the only protection is the `LastBonusAccrual` checkpoint, which is only updated in the local copy. [4](#0-3) 

### Impact Explanation

An attacker who owns any valid tiered-rewards position can drain the `RewardsPoolName` module account by submitting a single `MsgClaimTierRewards` transaction with the same position ID repeated N times. Each repetition causes a full bonus payout for the same accrual period. The corrupted value is the balance of the `RewardsPoolName` module account (staking bond-denom tokens held as bonus reserves), which is reduced by N × (legitimate bonus amount) instead of 1 × (legitimate bonus amount). [5](#0-4) 

### Likelihood Explanation

The entry path is a standard, unprivileged Cosmos SDK transaction (`MsgClaimTierRewards`) reachable by any account that holds a tiered-rewards position. No special role, leaked key, or social engineering is required. The only prerequisite is owning at least one position with non-zero accrued bonus, which is the normal operating state for any participant in the tiered-rewards system.

### Recommendation

Deduplicate `msg.PositionIds` before loading positions, or check for duplicate IDs inside the validation loop and return an error if any ID appears more than once:

```go
seen := make(map[uint64]struct{}, len(msg.PositionIds))
for _, posId := range msg.PositionIds {
    if _, dup := seen[posId]; dup {
        return nil, errorsmod.Wrapf(sdkerrors.ErrInvalidRequest,
            "duplicate position id %d", posId)
    }
    seen[posId] = struct{}{}
    ...
}
```

Alternatively, re-read each position from the store inside `claimRewardsAndUpdatesPositions` immediately before processing it (rather than using the pre-loaded snapshot), so that the second occurrence of the same ID sees the already-updated checkpoint.

### Proof of Concept

1. Alice creates a tiered-rewards position; its ID is `42`.
2. After some blocks, bonus accrues (e.g., 100 tokens owed).
3. Alice broadcasts `MsgClaimTierRewards{Owner: alice, PositionIds: [42, 42, 42, ..., 42]}` (N repetitions).
4. `ClaimTierRewards` loads position 42 from state N times, producing N identical `PositionState` copies in the `positions` slice.
5. `claimRewardsAndUpdatesPositions` iterates N times; each iteration finds `LastBonusAccrual` at the pre-claim value and pays 100 tokens.
6. Alice receives N × 100 tokens from `RewardsPoolName` instead of 100 tokens.
7. The final `setPosition` call writes the same checkpoint as the first, so the store appears consistent and no error is returned. [6](#0-5) [7](#0-6)

func (ms msgServer) ClaimTierRewards(ctx context.Context, msg *types.MsgClaimTierRewards) (*types.MsgClaimTierRewardsResponse, error) {
	if err := msg.Validate(); err != nil {
		return nil, err
	}

	positions := make([]types.PositionState, 0, len(msg.PositionIds))
	for _, posId := range msg.PositionIds {
		pos, err := ms.getPositionState(ctx, posId)
		if err != nil {
			return nil, err
		}

		if err := ms.validateClaimRewards(pos.Position, msg.Owner); err != nil {
			return nil, err
		}

		positions = append(positions, pos)
	}

	totalBase, totalBonus, err := ms.claimRewardsAndUpdatesPositions(ctx, positions)
	if err != nil {
		return nil, err
	}

	sdkCtx := sdk.UnwrapSDKContext(ctx)
	if err := sdkCtx.EventManager().EmitTypedEvent(&types.EventTierRewardsClaimed{
		Owner:        msg.Owner,
		PositionIds:  msg.PositionIds,
		BaseRewards:  totalBase,
		BonusRewards: totalBonus,
	}); err != nil {
		return nil, err
	}

	return &types.MsgClaimTierRewardsResponse{
		BaseRewards:  totalBase,
		BonusRewards: totalBonus,
		PositionIds:  msg.PositionIds,
	}, nil
}

func (k Keeper) claimRewardsAndUpdatesPositions(ctx context.Context, positions []types.PositionState) (sdk.Coins, sdk.Coins, error) {
	totalBase := sdk.NewCoins()
	totalBonus := sdk.NewCoins()

	for i := range positions {
		pos := &positions[i]

		if !pos.IsDelegated() {
			continue
		}

		base, err := k.claimBaseRewards(ctx, *pos)
		if err != nil {
			return nil, nil, err
		}
		totalBase = totalBase.Add(base...)

		bonus, err := k.processEventsAndClaimBonus(ctx, pos)
		if err != nil {
			return nil, nil, err
		}
		totalBonus = totalBonus.Add(bonus...)

		if err := k.setPosition(ctx, pos.Position, nil); err != nil {
			return nil, nil, err
		}
	}

	return totalBase, totalBonus, nil
}

func (k Keeper) processEventsAndClaimBonus(ctx context.Context, pos *types.PositionState) (sdk.Coins, error) {
	// Rewards should have been claimed before undelegation
	if !pos.IsDelegated() {
		return sdk.NewCoins(), nil
	}

	valAddr, err := sdk.ValAddressFromBech32(pos.Delegation.ValidatorAddress)
	if err != nil {
		return nil, err
	}

	events, err := k.getValidatorEventsSince(ctx, valAddr, pos.LastEventSeq)
	if err != nil {
		return nil, err
	}

	sdkCtx := sdk.UnwrapSDKContext(ctx)
	blockTime := sdkCtx.BlockTime()

	totalBonus := math.ZeroInt()
	// Use the persisted bonded state from the last replay, not a hardcoded default.
	// This prevents overpaying bonus for unbonded gaps between claims.
	bonded := pos.LastKnownBonded
	segmentStart := pos.LastBonusAccrual

	tier, err := k.getTier(ctx, pos.TierId)
	if err != nil {
		return nil, err
	}

	for _, entry := range events {
		evt := entry.Event

		if bonded {
			// Compute bonus for the bonded segment [segmentStart, eventTime]
			// using the snapshot rate at the event.
			bonus := k.computeSegmentBonus(*pos, tier, segmentStart, evt.Timestamp, evt.TokensPerShare)
			totalBonus = totalBonus.Add(bonus)
		}

		// Update bonded state based on event type.
		switch evt.EventType {
		case types.ValidatorEventType_VALIDATOR_EVENT_TYPE_UNBOND:
			bonded = false
		case types.ValidatorEventType_VALIDATOR_EVENT_TYPE_BOND:
			bonded = true
		case types.ValidatorEventType_VALIDATOR_EVENT_TYPE_SLASH:
			// Slash doesn't change bonded state.
		}

		segmentStart = evt.Timestamp
		pos.UpdateLastEventSeq(entry.Seq)

		// Decrement reference count.
		if err := k.decrementEventRefCount(ctx, valAddr, entry.Seq); err != nil {
			return nil, err
		}
	}

	val, err := k.stakingKeeper.GetValidator(ctx, valAddr)
	if err != nil {
		return nil, err
	}
	// Defensive: validator bond status check
	if bonded && val.IsBonded() {
		currentRate, err := k.getTokensPerShare(ctx, valAddr)
		if err != nil {
			return nil, err
		}
		bonus := k.computeSegmentBonus(*pos, tier, segmentStart, blockTime, currentRate)
		totalBonus = totalBonus.Add(bonus)
	}

	applyBonusAccrualCheckpoint(&pos.Position, blockTime)
	// Persist the bonded state so the next replay starts correctly.
	pos.UpdateLastKnownBonded(bonded)

	if totalBonus.IsZero() {
		return sdk.NewCoins(), nil
	}

	bondDenom, err := k.stakingKeeper.BondDenom(ctx)
	if err != nil {
		return nil, err
	}

	bonusCoins := sdk.NewCoins(sdk.NewCoin(bondDenom, totalBonus))

	if err := k.sufficientBonusPoolBalance(ctx, bonusCoins); err != nil {
		return nil, err
	}

	ownerAddr, err := sdk.AccAddressFromBech32(pos.Owner)
	if err != nil {
		return nil, errorsmod.Wrap(sdkerrors.ErrInvalidAddress, "invalid owner address")
	}

	if err := k.bankKeeper.SendCoinsFromModuleToAccount(ctx, types.RewardsPoolName, ownerAddr, bonusCoins); err != nil {
		return nil, err
	}

	if err := sdkCtx.EventManager().EmitTypedEvent(&types.EventBonusRewardsClaimed{
		PositionId: pos.Id,
		Owner:      pos.Owner,
		Rewards:    bonusCoins,
	}); err != nil {
		return nil, err
	}

	return bonusCoins, nil
