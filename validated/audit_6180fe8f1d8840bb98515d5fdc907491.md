### Title
Spread Health Bonus Inflated Due to `getCoreRisk` Always Returning Long Weight — (`core/contracts/BaseEngine.sol`)

---

### Summary

`getCoreRisk` hardcodes `1` as the `amount` argument to `_getWeightX18`, so it always returns the **long weight** regardless of whether the actual position is short. This corrupts the `existingWeight` term in `Clearinghouse.getHealth`'s spread-adjustment formula, causing the spread health bonus to be systematically over-stated for any spread pair that has a short leg. The result is that health is overestimated, allowing over-withdrawal of collateral and blocking legitimate liquidations — both paths to bad debt.

---

### Finding Description

**Root cause — `getCoreRisk` always returns long weight** [1](#0-0) 

```solidity
function getCoreRisk(...) external returns (IProductEngine.CoreRisk memory) {
    RiskHelper.Risk memory risk = _risk(productId);
    (int128 amount, ) = _getBalance(productId, subaccount);
    return IProductEngine.CoreRisk(
        amount,
        risk.priceX18,
        RiskHelper._getWeightX18(risk, 1, healthType)  // ← always 1, never `amount`
    );
}
```

`_getWeightX18` branches on the sign of its second argument: [2](#0-1) 

Because `1 >= 0`, it always returns `longWeightInitialX18` / `longWeightMaintenanceX18`. The actual `amount` (which may be negative for a short position) is never passed, so the short weight is never returned.

**Where the wrong value is consumed — `Clearinghouse.getHealth`** [3](#0-2) 

The spread-adjustment formula is:

```
health += basisAmount
    .mul(spotCoreRisk.price + perpCoreRisk.price)
    .mul(spreadWeight - existingWeight);
```

`existingWeight` is meant to represent the average weight **already applied** to the two legs in the individual `getHealthContribution` calls: [4](#0-3) 

```solidity
int128 existingWeight = (spotCoreRisk.longWeight + perpCoreRisk.longWeight) / 2;
```

Because `getCoreRisk` always returns the long weight, `existingWeight` is always `(longWeightSpot + longWeightPerp) / 2`.

But `getHealthContribution` applies the **actual** weight — short weight for a short leg: [5](#0-4) 

The risk constraints enforce `shortWeight >= 1e9` and `longWeight <= 1e9` (in 1e9 scale): [6](#0-5) 

So for any short leg, `shortWeight > longWeight`, meaning the buggy `existingWeight` is **always lower** than the true weight already applied. This makes `spreadWeight - existingWeight` larger than it should be, inflating the spread bonus.

---

### Impact Explanation

**Concrete numeric example** (longWeight = 0.8, shortWeight = 1.2, price = 10, position = short 100 spot + long 100 perp):

| | Correct | Buggy |
|---|---|---|
| Individual health | −120 + 80 + USDC | same |
| `existingWeight` | (1.2 + 0.8)/2 = 1.0 | (0.8 + 0.8)/2 = 0.8 |
| `spreadWeight` | 0.96 | 0.96 |
| Spread adjustment | 100 × 20 × (0.96 − 1.0) = **−80** | 100 × 20 × (0.96 − 0.8) = **+320** |
| **Health delta** | | **+400 phantom health units** |

The same overestimation applies to the more common long-spot / short-perp direction (the short perp leg's short weight is ignored).

**Consequences:**
1. **Over-withdrawal**: `withdrawCollateral` calls `getHealth >= 0` after deducting the withdrawal amount. Phantom health allows the attacker to drain collateral beyond what their position can cover.
2. **Liquidation bypass**: `isUnderMaintenance` returns false when it should return true, preventing timely liquidation and allowing losses to compound into bad debt. [7](#0-6) 

---

### Likelihood Explanation

- Any user who holds a registered spread pair (spot + perp in the `spreads` bitmap) with one short leg triggers the overestimation automatically — no special privilege required.
- Short spot positions (borrowing) are explicitly supported: the `shortWeightMaintenance >= 10^9` constraint and the `weight == 2*ONE → -INF` guard confirm that assets with `shortWeight < 2` allow borrowing.
- The path is reachable through normal deposit → trade → withdraw or deposit → trade → avoid-liquidation flows.

---

### Recommendation

Pass the actual `amount` to `_getWeightX18` in `getCoreRisk`, so the returned `longWeight` field reflects the weight that `getHealthContribution` actually applies to that position:

```solidity
// Before (always long weight):
RiskHelper._getWeightX18(risk, 1, healthType)

// After (correct weight for the actual position direction):
RiskHelper._getWeightX18(risk, amount, healthType)
```

The `CoreRisk.longWeight` field name is misleading; rename it to `weight` or `appliedWeight` to match its corrected semantics.

---

### Proof of Concept

1. Deploy with a spread pair `(spotId=1, perpId=2)`, `longWeightInitial = 0.8e18`, `shortWeightInitial = 1.2e18`, oracle price = 10e18.
2. Attacker deposits 1000 USDC.
3. Attacker borrows 100 units of spot asset (spot balance = −100) and opens long 100 perp.
4. Call `Clearinghouse.getHealth(attacker, INITIAL)`:
   - Individual contributions: spot = −100 × 1.2 × 10 = −1200, perp = 100 × 0.8 × 10 = 800, USDC = 1000 → subtotal = 600.
   - Buggy spread adjustment: basisAmount = 100, existingWeight = 0.8, spreadWeight = 0.96 → +320.
   - Reported health = **920**.
   - Correct health = 600 + (−80) = **520**.
5. Attacker calls `withdrawCollateral` for 920 USDC — succeeds (health check passes with phantom 400 units).
6. Attacker has extracted 920 USDC against a position that can only support 520 USDC of withdrawal, leaving a 400-unit deficit that becomes bad debt if the basis moves adversely.

```text
    function _getWeightX18(
        Risk memory risk,
        int128 amount,
        IProductEngine.HealthType healthType
    ) internal pure returns (int128) {
        if (healthType == IProductEngine.HealthType.PNL) {
            return ONE;
        }

        int128 weight;
        if (amount >= 0) {
            weight = healthType == IProductEngine.HealthType.INITIAL
                ? risk.longWeightInitialX18
                : risk.longWeightMaintenanceX18;
        } else {
            weight = healthType == IProductEngine.HealthType.INITIAL
                ? risk.shortWeightInitialX18
                : risk.shortWeightMaintenanceX18;
        }

        return weight;
    }
```
