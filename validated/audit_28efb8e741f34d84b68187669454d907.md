### Title
Migration Halt via Active Redelegation on Vesting-Owned Position — (`x/tieredrewards/keeper/transfer_delegation.go`)

---

### Summary

`transferDelegationFromPosition` contains a hard guard that returns `ErrActiveRedelegation` when the position's delegator address has an active redelegation entry. The v8 migration's `ForceFullExitWithDelegation` calls `transferDelegationFromPosition` without bypassing this guard. A vesting account owner who called `MsgTierRedelegate` on their position before the upgrade (no vesting check exists on that path) can leave an active redelegation entry on the position's delegator address, causing `Migrate1to2` to return an error and halting the chain upgrade.

---

### Finding Description

**Step 1 — Vesting account can call `MsgTierRedelegate`.**

`validateRedelegatePosition` in `msg_validate.go` does **not** call `validateNonVestingAccount`. The vesting-account filter only lives in `validateNewPosition` (called by `LockTier`/`CommitDelegationToTier`). A vesting account owner with an existing position can freely call `MsgTierRedelegate`. [1](#0-0) 

**Step 2 — `MsgTierRedelegate` creates a redelegation entry on the position's delegator address.**

`TierRedelegate` calls `ms.redelegate(ctx, delAddr, srcValAddr, dstValAddr, pos.Delegation.Shares)` where `delAddr` is `pos.DelegatorAddress` (the per-position module-derived address). This writes a staking redelegation entry keyed to that address with a completion time equal to `blockTime + unbondingTime` (typically 21 days). [2](#0-1) 

**Step 3 — `isRedelegating` checks that same address.**

`isRedelegating` calls `k.stakingKeeper.GetRedelegations(ctx, delAddr, 1)` on `pos.DelegatorAddress`. If the redelegation entry has not yet matured, it returns `true`. [3](#0-2) 

**Step 4 — `transferDelegationFromPosition` returns a hard error.**

The guard at lines 117–123 is unconditional: if `isRedelegating` is true, the function returns `ErrActiveRedelegation` with no bypass path. [4](#0-3) 

**Step 5 — `ForceFullExitWithDelegation` propagates the error.**

The call site in `ForceFullExitWithDelegation` wraps and returns the error immediately. [5](#0-4) 

**Step 6 — `exitVestedAccountsPositions` propagates the error, halting migration.**

The loop in `exitVestedAccountsPositions` returns on the first error, so a single redelegating vesting-owned position aborts the entire migration. [6](#0-5) 

**Step 7 — `Migrate1to2` returns the error to the upgrade handler.** [7](#0-6) 

In Cosmos SDK, an upgrade handler that returns an error causes the node to panic, halting the chain.

---

### Impact Explanation

A single vesting account owner who calls `MsgTierRedelegate` within the staking unbonding window (default 21 days) before the v8 upgrade block causes `Migrate1to2` to return `ErrActiveRedelegation`. The upgrade handler panics, the chain halts at the upgrade height, and no further blocks are produced until the binary is patched and the chain is restarted. All vesting-owned positions remain open, continuing to accrue rewards and retain delegation control post-upgrade.

---

### Likelihood Explanation

The preconditions are reachable by any user who held a vesting account with a tier position before v8 (explicitly the scenario the migration was designed to handle). `MsgTierRedelegate` is a standard, publicly accessible transaction with no vesting-account guard. The attacker only needs to submit one `MsgTierRedelegate` transaction within 21 days of the announced upgrade height — a trivially achievable timing constraint given that upgrade heights are announced in advance.

---

### Recommendation

In `ForceFullExitWithDelegation` (or in `transferDelegationFromPosition` when called from the migration path), skip the `isRedelegating` guard. Because the redelegation entry is on the position's own delegator address (not the owner's address), the slashing-escape concern that motivated the guard does not apply here. The position delegator is a module-controlled address; there is no user-controlled slashing-escape vector. The guard should either be removed entirely from `transferDelegationFromPosition` for the force-exit path, or `ForceFullExitWithDelegation` should wait for redelegation completion by checking and handling the active-redelegation case (e.g., by unbonding the destination delegation directly instead of using `transferDelegationFromPosition`).

func (s *KeeperSuite) TestMigrate1to2_HaltsOnActiveRedelegation() {
    s.setupTier(1)
    vals, bondDenom := s.getStakingData()
    val := vals[0]
    valAddr := sdk.MustValAddressFromBech32(val.GetOperator())
    s.setValidatorCommission(valAddr, sdkmath.LegacyZeroDec())

    amount := sdkmath.NewInt(sdk.DefaultPowerReduction.Int64())
    // Create vesting owner with a LockTier position (pre-v8 state)
    vestingOwner := s.newVestingOwnerWithBalance(bondDenom, amount, amount.MulRaw(2))
    _ = s.createLockTierPositionV1(vestingOwner, valAddr, amount)

    // Redelegate the position to a second validator (no vesting check on this path)
    dstValAddr, _ := s.createSecondValidator()
    msgServer := keeper.NewMsgServerImpl(s.keeper)
    pos, _ := s.keeper.GetPositionState(s.ctx, /* posID */ 1)
    _, err := msgServer.TierRedelegate(s.ctx, &types.MsgTierRedelegate{
        Owner:        vestingOwner.String(),
        PositionId:   pos.Id,
        DstValidator: dstValAddr.String(),
    })
    s.Require().NoError(err)

    // Confirm active redelegation on the position's delegator address
    isRedel, _ := s.keeper.IsRedelegating(s.ctx, pos.DelegatorAddress)
    s.Require().True(isRedel)

    // Run migration — must fail with ErrActiveRedelegation
    migrator := keeper.NewMigrator(s.keeper)
    err = migrator.Migrate1to2(s.ctx)
    s.Require().ErrorIs(err, types.ErrActiveRedelegation)
}

func (k Keeper) validateRedelegatePosition(ctx context.Context, pos types.PositionState, owner, dstValidator string) error {
	if !pos.IsOwner(owner) {
		return types.ErrNotPositionOwner
	}

	if !pos.IsDelegated() {
		return types.ErrPositionNotDelegated
	}

	if pos.Delegation.ValidatorAddress == dstValidator {
		return types.ErrRedelegationToSameValidator
	}

	sdkCtx := sdk.UnwrapSDKContext(ctx)
	if pos.HasTriggeredExit() && pos.CompletedExitLockDuration(sdkCtx.BlockTime()) {
		return types.ErrExitLockDurationElapsed
	}

	tier, err := k.getTier(ctx, pos.TierId)
	if err != nil {
		return err
	}

	if tier.IsCloseOnly() {
		return types.ErrTierIsCloseOnly
	}

	isRedelegating, err := k.isRedelegating(ctx, pos.DelegatorAddress)
	if err != nil {
		return err
	}
	if isRedelegating {
		return errorsmod.Wrapf(types.ErrActiveRedelegation, "position %d has an active redelegation", pos.Id)
	}

	return nil
}

func (k Keeper) isRedelegating(ctx context.Context, delegatorAddress string) (bool, error) {
	delAddr, err := sdk.AccAddressFromBech32(delegatorAddress)
	if err != nil {
		return false, errorsmod.Wrap(sdkerrors.ErrInvalidAddress, "invalid delegator address")
	}
	reds, err := k.stakingKeeper.GetRedelegations(ctx, delAddr, 1)
	if err != nil {
		return false, err
	}
	return len(reds) > 0, nil
}
