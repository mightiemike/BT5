### Title
`SwapAllowlistExtension.beforeSwap` checks `sender` (the router address) instead of the end-user, allowing any user to bypass the swap allowlist by routing through `MetricOmmSimpleRouter` — (`File: metric-periphery/contracts/extensions/SwapAllowlistExtension.sol`)

---

### Summary

`SwapAllowlistExtension` is a production extension that gates `swap` access on curated pools. Its `beforeSwap` hook checks the `sender` argument, which the pool sets to `msg.sender` of the `swap` call. When a user routes through `MetricOmmSimpleRouter`, `msg.sender` to the pool is the **router contract address**, not the end-user. If the pool admin allowlists the router (the natural step to support periphery-based swaps), every user — including non-allowlisted ones — can bypass the allowlist by routing through the router.

---

### Finding Description

**Root cause — wrong actor checked in `SwapAllowlistExtension.beforeSwap`:**

`SwapAllowlistExtension.beforeSwap` receives `sender` as its first argument and checks it against the per-pool allowlist:

```solidity
// metric-periphery/contracts/extensions/SwapAllowlistExtension.sol
function beforeSwap(address sender, address, bool, int128, uint128, uint256, uint128, uint128, bytes calldata)
    external view override returns (bytes4) {
    if (!allowAllSwappers[msg.sender] && !allowedSwapper[msg.sender][sender]) {
        revert IMetricOmmPoolActions.NotAllowedToSwap();
    }
    return IMetricOmmExtensions.beforeSwap.selector;
}
``` [1](#0-0) 

The pool passes `msg.sender` of the `swap` call as `sender` to `_beforeSwap`:

```solidity
// metric-core/contracts/MetricOmmPool.sol
_beforeSwap(
    msg.sender,   // ← this is the router address when routing through MetricOmmSimpleRouter
    recipient, ...
);
``` [2](#0-1) 

**The router calls `pool.swap` directly, making itself `msg.sender`:**

```solidity
// metric-periphery/contracts/MetricOmmSimpleRouter.sol
(int128 amount0Delta, int128 amount1Delta) = IMetricOmmPoolActions(params.pool)
    .swap(
        params.recipient,
        params.zeroForOne,
        MetricOmmSwapInputs.asAmountSpecifiedIn(params.amountIn),
        priceLimitX64,
        "",
        params.extensionData
    );
``` [3](#0-2) 

The actual end-user (`msg.sender` of the router call) is never forwarded to the pool or the extension. The extension only ever sees the router's address as `sender`.

**Contrast with `DepositAllowlistExtension` — which correctly checks `owner`:**

`DepositAllowlistExtension.beforeAddLiquidity` ignores `sender` (which would be the `LiquidityAdder` contract address) and checks `owner` — the position owner explicitly passed as a parameter:

```solidity
function beforeAddLiquidity(address, address owner, uint80, LiquidityDelta calldata, bytes calldata)
    external view override returns (bytes4) {
    if (!allowAllDepositors[msg.sender] && !allowedDepositor[msg.sender][owner]) {
        revert IMetricOmmPoolActions.NotAllowedToDeposit();
    }
``` [4](#0-3) 

This works because `addLiquidity` has an explicit `owner` parameter that carries the end-user's identity through the periphery. `swap` has no equivalent — only `msg.sender` (the router) is available to the pool.

**The bypass scenario:**

| Step | Actor | `sender` seen by extension | Allowlist result |
|---|---|---|---|
| Pool admin allowlists user A and the router | — | — | — |
| User A calls pool directly | user A | user A | ✓ allowed |
| User A calls via router | router | router | ✓ allowed (router is allowlisted) |
| User C (not allowlisted) calls pool directly | user C | user C | ✗ blocked |
| **User C calls via router** | **router** | **router** | **✓ allowed — BYPASS** |

Once the router is in the allowlist (required for any allowlisted user to use the standard periphery), every user can bypass the allowlist by routing through `MetricOmmSimpleRouter`.

---

### Impact Explanation

Any user can swap on a curated pool protected by `SwapAllowlistExtension` by routing through `MetricOmmSimpleRouter`, provided the router address is in the allowlist. This completely defeats the purpose of the allowlist — KYC gates, institutional-only pools, or any access-controlled trading venue built on this extension are rendered ineffective. The bypass is a direct, fund-impacting action: unauthorized users execute real swaps, moving tokens out of the pool at oracle-derived prices.

---

### Likelihood Explanation

The likelihood is high. `MetricOmmSimpleRouter` is the primary user-facing swap interface for the protocol. Any pool admin who deploys a curated pool and wants allowlisted users to be able to use the standard periphery must add the router to the allowlist. This is the obvious and expected operational step. Once done, the bypass is trivially reachable by any user with no special privileges, no malicious setup, and no non-standard tokens.

---

### Recommendation

The `SwapAllowlistExtension` must gate the economically relevant actor — the end-user — not the immediate caller of `pool.swap`. Two approaches:

1. **Pass the end-user through `extensionData`:** The router encodes `msg.sender` into `extensionData`; the extension decodes and checks it. This requires coordination between the router and the extension but preserves the current pool interface.

2. **Add a `swapper` parameter to `swap`:** Analogous to how `addLiquidity` has an explicit `owner` parameter, `swap` could accept an explicit `swapper` address that the pool passes to `_beforeSwap` instead of `msg.sender`. The router would pass `msg.sender` (the end-user) as this parameter.

Option 2 is the cleaner fix and mirrors the design that makes `DepositAllowlistExtension` correct.

---

### Proof of Concept

```solidity
// Setup: pool with SwapAllowlistExtension, router allowlisted, user C not allowlisted
SwapAllowlistExtension ext = new SwapAllowlistExtension(factory);
// Pool admin allowlists the router so allowlisted users can use periphery
ext.setAllowedToSwap(address(pool), address(router), true);
// User A is allowlisted directly
ext.setAllowedToSwap(address(pool), userA, true);
// User C is NOT allowlisted

// Direct call by user C — correctly blocked
vm.prank(userC);
vm.expectRevert(IMetricOmmPoolActions.NotAllowedToSwap.selector);
pool.swap(userC, false, 1000, type(uint128).max, "", "");

// Router call by user C — BYPASSES the allowlist
// sender seen by extension = router address = allowlisted
vm.prank(userC);
router.exactInputSingle(ExactInputSingleParams({
    pool: address(pool),
    recipient: userC,
    zeroForOne: false,
    amountIn: 1000,
    amountOutMinimum: 0,
    priceLimitX64: type(uint128).max,
    deadline: block.timestamp,
    extensionData: ""
}));
// Swap succeeds — user C traded on a pool they were not authorized to access
