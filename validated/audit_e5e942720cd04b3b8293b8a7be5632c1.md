### Title
Gateway Admission Checks Only the L2 Gas Price Minimum Threshold, Skipping L1 Gas and L1 Data Gas Price Floors for `AllResources` Transactions - (`File: crates/apollo_gateway/src/stateful_transaction_validator.rs`)

---

### Summary

The gateway's stateful (and stateless) resource-bounds validators enforce a minimum gas-price floor only against `l2_gas.max_price_per_unit`. For V3 transactions that carry `ValidResourceBounds::AllResources`, the analogous floors for `l1_gas.max_price_per_unit` and `l1_data_gas.max_price_per_unit` are never checked. An unprivileged sender can therefore submit a transaction whose L1 or L1-data gas price is zero (or arbitrarily below the current block price), have it accepted by the gateway and admitted to the mempool, and only have it rejected later—inside the blockifier's pre-validation—after sequencer resources have already been consumed.

---

### Finding Description

`StatefulTransactionValidator::validate_resource_bounds` reads the previous block's L2 gas price and delegates to `validate_tx_l2_gas_price_within_threshold`:

```rust
// crates/apollo_gateway/src/stateful_transaction_validator.rs  L223-243
async fn validate_resource_bounds(&self, executable_tx: &ExecutableTransaction)
    -> StatefulTransactionValidatorResult<()>
{
    if self.config.validate_resource_bounds {
        let previous_block_l2_gas_price = self
            .gateway_fixed_block_state_reader
            .get_block_info().await?
            .gas_prices.strk_gas_prices.l2_gas_price;
        self.validate_tx_l2_gas_price_within_threshold(
            executable_tx.resource_bounds(),
            previous_block_l2_gas_price,
        )?;
    }
    Ok(())
}
```

Inside that function, only `l2_gas.max_price_per_unit` is tested; the developer even left a self-documenting TODO:

```rust
// L358-390
// TODO(Arni): Consider running this validation for all gas prices.
fn validate_tx_l2_gas_price_within_threshold(
    &self,
    tx_resource_bounds: ValidResourceBounds,
    previous_block_l2_gas_price: NonzeroGasPrice,
) -> StatefulTransactionValidatorResult<()> {
    match tx_resource_bounds {
        ValidResourceBounds::AllResources(tx_resource_bounds) => {
            let tx_l2_gas_price = tx_resource_bounds.l2_gas.max_price_per_unit;
            // ... threshold check on l2_gas only ...
            if tx_l2_gas_price.0 < threshold { return Err(...); }
        }
        ValidResourceBounds::L1Gas(_) => { /* No validation required for legacy */ }
    }
    Ok(())
}
```

`l1_gas.max_price_per_unit` and `l1_data_gas.max_price_per_unit` are never compared against any floor.

The stateless validator in `StatelessTransactionValidator::validate_resource_bounds` has the same gap:

```rust
// crates/apollo_gateway/src/stateless_transaction_validator.rs  L71-75
if resource_bounds.l2_gas.max_price_per_unit.0 < self.config.min_gas_price {
    return Err(StatelessTransactionValidatorError::MaxGasPriceTooLow { ... });
}
```

Again, only `l2_gas` is checked; `l1_gas` and `l1_data_gas` prices are ignored.

By contrast, the blockifier's pre-validation (`account_transaction.rs` lines 398–425) correctly iterates all three resources and rejects any transaction whose `max_price_per_unit` is below the actual block gas price:

```rust
ValidResourceBounds::AllResources(AllResourceBounds {
    l1_gas: l1_gas_resource_bounds,
    l2_gas: l2_gas_resource_bounds,
    l1_data_gas: l1_data_gas_resource_bounds,
}) => {
    vec![
        (L1Gas,     l1_gas_resource_bounds,     ..., l1_gas_price),
        (L1DataGas, l1_data_gas_resource_bounds, ..., l1_data_gas_price),
        (L2Gas,     l2_gas_resource_bounds,      ..., l2_gas_price),
    ]
}
```

The gateway and the blockifier therefore apply different invariants to the same transaction object: the gateway admits it, the blockifier rejects it.

---

### Impact Explanation

A transaction with `AllResources` bounds where `l1_gas.max_price_per_unit = 0` and/or `l1_data_gas.max_price_per_unit = 0` (but `l2_gas.max_price_per_unit` meets the threshold) will:

1. Pass both the stateless and stateful gateway checks.
2. Be inserted into the mempool.
3. Fail inside the blockifier's `check_resource_bounds` with `TransactionFeeError::InsufficientResourceBounds` at sequencing time.

This matches the allowed impact: **"High. Mempool/gateway/RPC admission accepts invalid transactions … before sequencing."**

The sequencer wastes CPU and I/O resources processing, storing, and later discarding these transactions. At scale, an attacker can flood the mempool with zero-L1-gas-price transactions that all pass gateway admission, degrading throughput for legitimate users.

---

### Likelihood Explanation

- The trigger is fully unprivileged: any user can craft a V3 `InvokeTransaction` with `AllResources` bounds and set `l1_gas.max_price_per_unit = 0`.
- The stateless validator's `ZeroResourceBounds` guard only fires when `max_possible_fee(Tip::ZERO) == 0`, which requires **all three** resource amounts or prices to be zero simultaneously. A transaction with non-zero `l2_gas` bounds but zero `l1_gas` price passes that guard.
- The TODO comment in the production source confirms the gap is known but unresolved.

---

### Recommendation

Extend `validate_tx_l2_gas_price_within_threshold` (or replace it with a new `validate_all_gas_prices_within_threshold`) to apply the same percentage-of-previous-block threshold to `l1_gas.max_price_per_unit` and `l1_data_gas.max_price_per_unit` when the transaction carries `ValidResourceBounds::AllResources`. The stateless validator's `min_gas_price` check should be extended symmetrically.

```rust
ValidResourceBounds::AllResources(tx_resource_bounds) => {
    for (price, label) in [
        (tx_resource_bounds.l2_gas.max_price_per_unit,      "L2"),
        (tx_resource_bounds.l1_gas.max_price_per_unit,      "L1"),
        (tx_resource_bounds.l1_data_gas.max_price_per_unit, "L1 data"),
    ] {
        if price.0 < threshold {
            return Err(/* GAS_PRICE_TOO_LOW for {label} */);
        }
    }
}
```

---

### Proof of Concept

1. Construct a V3 Invoke transaction with:
   ```
   resource_bounds = AllResources {
       l1_gas:      { max_amount: 1, max_price_per_unit: 0 },   // zero price
       l2_gas:      { max_amount: 1, max_price_per_unit: <threshold> }, // meets check
       l1_data_gas: { max_amount: 1, max_price_per_unit: 0 },   // zero price
   }
   ```
2. Submit to the gateway. `validate_resource_bounds` in both the stateless and stateful validators passes because only `l2_gas.max_price_per_unit` is compared against the floor.
3. The transaction is admitted to the mempool.
4. When the batcher pulls the transaction and the blockifier runs `check_resource_bounds`, it finds `l1_gas.max_price_per_unit (0) < actual_l1_gas_price` and returns `InsufficientResourceBounds { resource: L1Gas, … }`, reverting the transaction after sequencer resources have been consumed.

    fn validate_resource_bounds(
        &self,
        tx: &RpcTransaction,
    ) -> StatelessTransactionValidatorResult<()> {
        if !self.config.validate_resource_bounds {
            return Ok(());
        }

        let resource_bounds = *tx.resource_bounds();
        // The resource bounds should be positive even without the tip.
        if ValidResourceBounds::AllResources(resource_bounds).max_possible_fee(Tip::ZERO) == Fee(0)
        {
            return Err(StatelessTransactionValidatorError::ZeroResourceBounds { resource_bounds });
        }

        if resource_bounds.l2_gas.max_price_per_unit.0 < self.config.min_gas_price {
            return Err(StatelessTransactionValidatorError::MaxGasPriceTooLow {
                gas_price: resource_bounds.l2_gas.max_price_per_unit,
                min_gas_price: self.config.min_gas_price,
            });

        if let RpcTransaction::Declare(_) = tx {
        } else if resource_bounds.l2_gas.max_amount.0 > self.config.max_l2_gas_amount {
            return Err(StatelessTransactionValidatorError::MaxGasAmountTooHigh {
                gas_amount: resource_bounds.l2_gas.max_amount,
                max_gas_amount: self.config.max_l2_gas_amount,
            });
        }

        Ok(())
