Audit Report

## Title
Router-Mediated Swaps Bypass `SwapAllowlistExtension` Per-User Gate — (`metric-periphery/contracts/extensions/SwapAllowlistExtension.sol`)

## Summary

`SwapAllowlistExtension.beforeSwap` gates on the `sender` argument, which the pool sets to `msg.sender` of the `pool.swap()` call. When a user routes through `MetricOmmSimpleRouter`, the pool's `msg.sender` is the router contract, not the end user. If the router is allowlisted (a natural configuration for pools that want to support router-mediated swaps), any address can bypass the per-user allowlist gate with a single `exactInputSingle` call.

## Finding Description

`MetricOmmPool.swap` calls `_beforeSwap(msg.sender, ...)`, passing the immediate caller as `sender`. `ExtensionCalling._beforeSwap` encodes that value verbatim and dispatches it to each configured extension. `SwapAllowlistExtension.beforeSwap` then evaluates `allowedSwapper[msg.sender][sender]`, where `msg.sender` is the pool and `sender` is whoever called `pool.swap()`.

When `MetricOmmSimpleRouter.exactInputSingle` is used, the router calls `IMetricOmmPoolActions(params.pool).swap(params.recipient, ...)` directly. The pool therefore sees `msg.sender = Router`, and the extension checks `allowedSwapper[pool][Router]`. The real end-user identity (`msg.sender` of `exactInputSingle`) is stored only in transient storage (`_getPayer()`) and is never surfaced to the extension.

A pool admin who allowlists the router so that approved users can swap via the router inadvertently opens the gate to every caller: `allowedSwapper[pool][Router] = true` causes `beforeSwap` to pass for any address that routes through the router, regardless of whether that address is individually approved.

The `allowAllSwappers` escape hatch is irrelevant here — it is a separate flag and does not mitigate the identity mismatch. There is no configuration that simultaneously permits router-mediated swaps for allowlisted users and blocks non-allowlisted users.

## Impact Explanation

The `SwapAllowlistExtension`'s core invariant — "only approved addresses may swap in this pool" — is broken for any pool that configures the extension and also allowlists the router. Any unprivileged address can execute swaps in a pool intended to be restricted to a curated set (KYC'd users, institutional counterparties, etc.) by routing through `MetricOmmSimpleRouter`. This constitutes a broken core pool functionality (the allowlist extension) and an admin-boundary break where the pool admin's access control is bypassed by an unprivileged path, matching the contest's "Allowlist path" smart audit pivot explicitly.

## Likelihood Explanation

`MetricOmmSimpleRouter` is the primary public swap entrypoint. Any pool deploying `SwapAllowlistExtension` with the intent of restricting access while still supporting the official router will naturally allowlist the router address. No privileged access, oracle manipulation, or multi-transaction setup is required — a single `exactInputSingle` call from any address suffices. The precondition (router allowlisted) is the expected operational state for such pools.

## Recommendation

The extension must resolve the true end-user identity rather than trusting the `sender` argument when the caller is a known intermediary:

1. **Pass the real payer through `extensionData`**: The router stores the real payer in transient storage (`_getPayer()`). Encode the real user address in `extensionData`; the extension verifies it by trusting only the factory-registered router address as the caller.
2. **Gate on `recipient` instead of `sender`**: For allowlist purposes, check `recipient` (the address receiving tokens) rather than `sender`. This is semantically different but may match the pool designer's intent for many use cases.
3. **Document incompatibility**: If neither fix is applied, document that pools using `SwapAllowlistExtension` must not allowlist the router and must require direct `pool.swap()` calls only.

## Proof of Concept

```
Setup:
  pool configured with SwapAllowlistExtension
  allowedSwapper[pool][Alice] = true      // Alice is approved
  allowedSwapper[pool][Router] = true     // Router allowlisted so Alice can use it

Attack (single transaction):
  Bob (not allowlisted) calls:
    MetricOmmSimpleRouter.exactInputSingle({pool: pool, recipient: Bob, ...})

  Router calls: IMetricOmmPoolActions(pool).swap(Bob, ...)
  Pool calls:   _beforeSwap(msg.sender=Router, ...)
  Extension checks: allowedSwapper[pool][Router] → true
  Result: Bob's swap succeeds despite not being on the allowlist
```

Foundry test: deploy pool with `SwapAllowlistExtension`, set `allowedSwapper[pool][router] = true`, call `exactInputSingle` from an address not in `allowedSwapper`, assert the call succeeds.
