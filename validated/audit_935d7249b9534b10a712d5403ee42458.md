### Title
Duplicate Position IDs in `MsgClaimTierRewards` Enable Multiple Bonus Reward Claims — (`x/tieredrewards/keeper/msg_server.go`)

### Summary
`ClaimTierRewards` loads all requested positions into a local in-memory slice before processing any of them. Because there is no deduplication guard at either the loading or the processing stage, an attacker who submits the same position ID multiple times in a single `MsgClaimTierRewards` transaction can cause `processEventsAndClaimBonus` to pay out bonus rewards for the same validator events more than once, draining the `RewardsPoolName` module account.

---

### Finding Description

`ClaimTierRewards` in `x/tieredrewards/keeper/msg_server.go` iterates over `msg.PositionIds`, loads each position from the store into a local `[]types.PositionState` slice, and then hands the entire slice to `claimRewardsAndUpdatesPositions`:

```go
// msg_server.go – ClaimTierRewards
positions := make([]types.PositionState, 0, len(msg.PositionIds))
for _, posId := range msg.PositionIds {
    pos, err := ms.getPositionState(ctx, posId)
    ...
    if err := ms.validateClaimRewards(pos.Position, msg.Owner); err != nil {
        return nil, err
    }
    positions = append(positions, pos)   // ← snapshot, not a live store reference
}
totalBase, totalBonus, err := ms.claimRewardsAndUpdatesPositions(ctx, positions)
``` [1](#0-0) 

`claimRewardsAndUpdatesPositions` then processes each element of the slice independently:

```go
// claim_rewards.go
for i := range positions {
    pos := &positions[i]          // pointer into the LOCAL slice, not the store
    ...
    bonus, err := k.processEventsAndClaimBonus(ctx, pos)
    ...
    if err := k.setPosition(ctx, pos.Position, nil); err != nil { ... }
}
``` [2](#0-1) 

`processEventsAndClaimBonus` reads `pos.LastEventSeq` from the **local copy** to query validator events from the store, computes the bonus, and sends coins from the module account to the owner:

```go
events, err := k.getValidatorEventsSince(ctx, valAddr, pos.LastEventSeq)
...
if err := k.bankKeeper.SendCoinsFromModuleToAccount(ctx, types.RewardsPoolName, ownerAddr, bonusCoins); err != nil {
``` [3](#0-2) 

It also decrements the per-event reference count for each processed event:

```go
if err := k.decrementEventRefCount(ctx, valAddr, entry.Seq); err != nil {
    return nil, err
}
``` [4](#0-3) 

**The critical ordering flaw:** all positions are snapshotted from the store *before* any processing begins. When the same position ID appears twice in `msg.PositionIds`, both slice entries carry the same stale `LastEventSeq`. The first processing iteration pays out the bonus and decrements event reference counts. The second iteration reads the same stale `LastEventSeq` from its own local copy and calls `getValidatorEventsSince` against the store again. If the reference count for those events is still > 0 (because other positions on the same validator have not yet processed them), the events are still present in the store and the full bonus is paid out a second time.

---

### Impact Explanation

The corrupted value is the `RewardsPoolName` module account balance. An attacker who owns position P delegated to a validator that also has at least one other tiered-rewards position (making the per-event reference count ≥ 2) can submit:

```
MsgClaimTierRewards { owner: attacker, position_ids: [P, P, P, …] }
```

Each duplicate entry independently computes and transfers the full bonus for all unprocessed validator events. With N duplicates and a reference count of R ≥ N, the attacker receives up to N× the legitimate bonus. As a side effect, the reference counts are decremented N times per event; if N ≥ R, the events are deleted before the other legitimate positions can claim their own bonus, stealing those rewards as well.

---

### Likelihood Explanation

Any user who holds a tiered-rewards position can craft this transaction with no special privileges. The only environmental requirement is that at least one other position exists on the same validator — a condition that is virtually always satisfied on a live network with multiple participants. The attack is a single standard Cosmos SDK transaction and requires no external coordination.

---

### Recommendation

1. **Deduplicate `msg.PositionIds` in `ClaimTierRewards`** before loading positions, or reject the message if any ID appears more than once. This is the minimal fix.
2. **Re-read position state from the store inside `claimRewardsAndUpdatesPositions`** rather than operating on a pre-loaded snapshot, so that the updated `LastEventSeq` written by `setPosition` is visible to any subsequent iteration over the same position.
3. Alternatively, track a per-position "claim-in-progress" flag (analogous to a reentrancy guard) that is set before processing and cleared after `setPosition`, causing any duplicate entry to be skipped or rejected.

---

### Proof of Concept

**Setup:** Two users each hold one tiered-rewards position (IDs 1 and 2) delegated to the same validator. Two validator events (E1, E2) exist with reference count 2.

**Attack transaction:** Attacker (owner of position 1) submits:
```
MsgClaimTierRewards { owner: attacker, position_ids: [1, 1] }
```

**Execution trace inside `claimRewardsAndUpdatesPositions`:**

| Step | Slice entry | `pos.LastEventSeq` (local) | Events returned by store | Bonus paid | Ref count after |
|------|-------------|---------------------------|--------------------------|------------|-----------------|
| 1 | positions[0] (pos 1) | 0 | [E1, E2] | B coins | E1→1, E2→1 |
| 2 | positions[1] (pos 1, duplicate) | 0 (stale snapshot) | [E1, E2] (still present, ref > 0) | B coins again | E1→0 (deleted), E2→0 (deleted) |

**Result:** Attacker receives 2B bonus coins instead of B. Position 2's owner can no longer claim their bonus because E1 and E2 have been deleted. [5](#0-4) [2](#0-1) [6](#0-5)

func (ms msgServer) ClaimTierRewards(ctx context.Context, msg *types.MsgClaimTierRewards) (*types.MsgClaimTierRewardsResponse, error) {
	if err := msg.Validate(); err != nil {
		return nil, err
	}

	positions := make([]types.PositionState, 0, len(msg.PositionIds))
	for _, posId := range msg.PositionIds {
		pos, err := ms.getPositionState(ctx, posId)
		if err != nil {
			return nil, err
		}

		if err := ms.validateClaimRewards(pos.Position, msg.Owner); err != nil {
			return nil, err
		}

		positions = append(positions, pos)
	}

	totalBase, totalBonus, err := ms.claimRewardsAndUpdatesPositions(ctx, positions)
	if err != nil {
		return nil, err
	}

	sdkCtx := sdk.UnwrapSDKContext(ctx)
	if err := sdkCtx.EventManager().EmitTypedEvent(&types.EventTierRewardsClaimed{
		Owner:        msg.Owner,
		PositionIds:  msg.PositionIds,
		BaseRewards:  totalBase,
		BonusRewards: totalBonus,
	}); err != nil {
		return nil, err
	}

	return &types.MsgClaimTierRewardsResponse{
		BaseRewards:  totalBase,
		BonusRewards: totalBonus,
		PositionIds:  msg.PositionIds,
	}, nil
}

func (k Keeper) claimRewardsAndUpdatesPositions(ctx context.Context, positions []types.PositionState) (sdk.Coins, sdk.Coins, error) {
	totalBase := sdk.NewCoins()
	totalBonus := sdk.NewCoins()

	for i := range positions {
		pos := &positions[i]

		if !pos.IsDelegated() {
			continue
		}

		base, err := k.claimBaseRewards(ctx, *pos)
		if err != nil {
			return nil, nil, err
		}
		totalBase = totalBase.Add(base...)

		bonus, err := k.processEventsAndClaimBonus(ctx, pos)
		if err != nil {
			return nil, nil, err
		}
		totalBonus = totalBonus.Add(bonus...)

		if err := k.setPosition(ctx, pos.Position, nil); err != nil {
			return nil, nil, err
		}
	}

	return totalBase, totalBonus, nil

func (k Keeper) processEventsAndClaimBonus(ctx context.Context, pos *types.PositionState) (sdk.Coins, error) {
	// Rewards should have been claimed before undelegation
	if !pos.IsDelegated() {
		return sdk.NewCoins(), nil
	}

	valAddr, err := sdk.ValAddressFromBech32(pos.Delegation.ValidatorAddress)
	if err != nil {
		return nil, err
	}

	events, err := k.getValidatorEventsSince(ctx, valAddr, pos.LastEventSeq)
	if err != nil {
		return nil, err
	}

	sdkCtx := sdk.UnwrapSDKContext(ctx)
	blockTime := sdkCtx.BlockTime()

	totalBonus := math.ZeroInt()
	// Use the persisted bonded state from the last replay, not a hardcoded default.
	// This prevents overpaying bonus for unbonded gaps between claims.
	bonded := pos.LastKnownBonded
	segmentStart := pos.LastBonusAccrual

	tier, err := k.getTier(ctx, pos.TierId)
	if err != nil {
		return nil, err
	}

	for _, entry := range events {
		evt := entry.Event

		if bonded {
			// Compute bonus for the bonded segment [segmentStart, eventTime]
			// using the snapshot rate at the event.
			bonus := k.computeSegmentBonus(*pos, tier, segmentStart, evt.Timestamp, evt.TokensPerShare)
			totalBonus = totalBonus.Add(bonus)
		}

		// Update bonded state based on event type.
		switch evt.EventType {
		case types.ValidatorEventType_VALIDATOR_EVENT_TYPE_UNBOND:
			bonded = false
		case types.ValidatorEventType_VALIDATOR_EVENT_TYPE_BOND:
			bonded = true
		case types.ValidatorEventType_VALIDATOR_EVENT_TYPE_SLASH:
			// Slash doesn't change bonded state.
		}

		segmentStart = evt.Timestamp
		pos.UpdateLastEventSeq(entry.Seq)

		// Decrement reference count.
		if err := k.decrementEventRefCount(ctx, valAddr, entry.Seq); err != nil {
			return nil, err
		}
	}

	val, err := k.stakingKeeper.GetValidator(ctx, valAddr)
	if err != nil {
		return nil, err
	}
	// Defensive: validator bond status check
	if bonded && val.IsBonded() {
		currentRate, err := k.getTokensPerShare(ctx, valAddr)
		if err != nil {
			return nil, err
		}
		bonus := k.computeSegmentBonus(*pos, tier, segmentStart, blockTime, currentRate)
		totalBonus = totalBonus.Add(bonus)
	}

	applyBonusAccrualCheckpoint(&pos.Position, blockTime)
	// Persist the bonded state so the next replay starts correctly.
	pos.UpdateLastKnownBonded(bonded)

	if totalBonus.IsZero() {
		return sdk.NewCoins(), nil
	}

	bondDenom, err := k.stakingKeeper.BondDenom(ctx)
	if err != nil {
		return nil, err
	}

	bonusCoins := sdk.NewCoins(sdk.NewCoin(bondDenom, totalBonus))

	if err := k.sufficientBonusPoolBalance(ctx, bonusCoins); err != nil {
		return nil, err
	}

	ownerAddr, err := sdk.AccAddressFromBech32(pos.Owner)
	if err != nil {
		return nil, errorsmod.Wrap(sdkerrors.ErrInvalidAddress, "invalid owner address")
	}

	if err := k.bankKeeper.SendCoinsFromModuleToAccount(ctx, types.RewardsPoolName, ownerAddr, bonusCoins); err != nil {
		return nil, err
	}

	if err := sdkCtx.EventManager().EmitTypedEvent(&types.EventBonusRewardsClaimed{
		PositionId: pos.Id,
		Owner:      pos.Owner,
		Rewards:    bonusCoins,
	}); err != nil {
		return nil, err
	}

	return bonusCoins, nil
