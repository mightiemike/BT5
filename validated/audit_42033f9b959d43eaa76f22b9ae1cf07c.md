### Title
Duplicate Position IDs in `MsgClaimTierRewards` Enable Double-Spend of Bonus Rewards Pool — (`x/tieredrewards/keeper/msg_server.go`)

---

### Summary

`ClaimTierRewards` loads all requested positions into an in-memory slice before processing any of them. Because there is no deduplication check on `msg.PositionIds`, an attacker can include the same position ID multiple times. Each duplicate entry carries the stale pre-claim checkpoint values (`LastEventSeq`, `LastBonusAccrual`), causing `processEventsAndClaimBonus` to re-process the same validator events and pay the same bonus amount from the `RewardsPoolName` module account on every iteration.

---

### Finding Description

`ClaimTierRewards` in `msg_server.go` first collects all positions into a slice, then calls `claimRewardsAndUpdatesPositions`:

```go
// msg_server.go lines 434–448
positions := make([]types.PositionState, 0, len(msg.PositionIds))
for _, posId := range msg.PositionIds {
    pos, err := ms.getPositionState(ctx, posId)
    ...
    if err := ms.validateClaimRewards(pos.Position, msg.Owner); err != nil {
        return nil, err
    }
    positions = append(positions, pos)
}
totalBase, totalBonus, err := ms.claimRewardsAndUpdatesPositions(ctx, positions)
``` [1](#0-0) 

`validateClaimRewards` only checks ownership — it performs no deduplication:

```go
// msg_validate.go lines 175–181
func (k Keeper) validateClaimRewards(pos types.Position, owner string) error {
    if !pos.IsOwner(owner) {
        return types.ErrNotPositionOwner
    }
    return nil
}
``` [2](#0-1) 

`claimRewardsAndUpdatesPositions` then iterates the in-memory slice. For each entry it calls `processEventsAndClaimBonus`, which reads `pos.LastEventSeq` from the **in-memory** (stale) copy to determine which validator events to replay, then immediately pays bonus coins from the module account:

```go
// claim_rewards.go lines 153–239 (key lines)
events, err := k.getValidatorEventsSince(ctx, valAddr, pos.LastEventSeq)
...
if err := k.bankKeeper.SendCoinsFromModuleToAccount(
    ctx, types.RewardsPoolName, ownerAddr, bonusCoins); err != nil {
    return nil, err
}
``` [3](#0-2) [4](#0-3) 

After the first iteration, `setPosition` writes the updated `LastEventSeq` to the KV store. However, the second (duplicate) entry in the slice was loaded **before** that write and still holds the original stale `LastEventSeq`. When `processEventsAndClaimBonus` runs on the duplicate, it re-queries the same events from the KV store (using the stale sequence number) and pays the same bonus a second time. [5](#0-4) 

The base-rewards path (`claimBaseRewards` → `distributionKeeper.WithdrawDelegationRewards`) is **not** vulnerable to doubling because the distribution module tracks the outstanding reward independently; the second call returns zero. Only the bonus pool is affected. [6](#0-5) 

---

### Impact Explanation

An attacker who owns any delegated tier position with accrued bonus rewards can drain the `RewardsPoolName` module account by submitting a single `MsgClaimTierRewards` transaction with the same position ID repeated N times. Each repetition re-pays the same bonus amount. The only natural limit is `sufficientBonusPoolBalance`, which causes the transaction to fail once the pool is exhausted — but by that point the attacker has already received N × (legitimate bonus) tokens.

The corrupted invariant is: **the `RewardsPoolName` module account balance decreases by more than the legitimately accrued bonus for the position**. [7](#0-6) 

---

### Likelihood Explanation

The entry path is a standard, unprivileged Cosmos SDK transaction (`MsgClaimTierRewards`) signed by any account that holds a tier position. No special role, governance approval, or IBC relayer is required. The attacker only needs to have locked tokens in any tier to obtain a position ID, then submit the message with that ID repeated in `PositionIds`. This is trivially constructable via CLI or any SDK client. [8](#0-7) 

---

### Recommendation

Add a deduplication check in `ClaimTierRewards` before building the `positions` slice, rejecting any message that contains the same position ID more than once:

```go
seen := make(map[uint64]struct{}, len(msg.PositionIds))
for _, posId := range msg.PositionIds {
    if _, dup := seen[posId]; dup {
        return nil, errorsmod.Wrapf(sdkerrors.ErrInvalidRequest,
            "duplicate position id %d in claim request", posId)
    }
    seen[posId] = struct{}{}
    ...
}
```

Alternatively, enforce the same invariant inside `claimRewardsAndUpdatesPositions` by re-reading each position from the KV store immediately before processing it, rather than relying on the pre-loaded in-memory snapshot. [1](#0-0) 

---

### Proof of Concept

1. Attacker locks tokens in any tier → receives `positionId = 42`.
2. Bonus rewards accrue over several blocks.
3. Attacker submits:
   ```
   MsgClaimTierRewards{
       Owner:       attacker,
       PositionIds: [42, 42, 42, 42, 42],  // same ID × 5
   }
   ```
4. `ClaimTierRewards` loads position 42 five times into `positions[0..4]`, all with the same stale `LastEventSeq = S`.
5. Iteration 0: `processEventsAndClaimBonus` replays events since seq S, pays bonus B, writes `LastEventSeq = S'` to KV store.
6. Iteration 1: `processEventsAndClaimBonus` reads `positions[1].LastEventSeq = S` (stale), replays the same events, pays bonus B again.
7. Iterations 2–4: same — bonus B paid each time.
8. Total bonus received: `5 × B` instead of `1 × B`. The `RewardsPoolName` account is debited `5 × B`. [9](#0-8) [8](#0-7)

func (ms msgServer) ClaimTierRewards(ctx context.Context, msg *types.MsgClaimTierRewards) (*types.MsgClaimTierRewardsResponse, error) {
	if err := msg.Validate(); err != nil {
		return nil, err
	}

	positions := make([]types.PositionState, 0, len(msg.PositionIds))
	for _, posId := range msg.PositionIds {
		pos, err := ms.getPositionState(ctx, posId)
		if err != nil {
			return nil, err
		}

		if err := ms.validateClaimRewards(pos.Position, msg.Owner); err != nil {
			return nil, err
		}

		positions = append(positions, pos)
	}

	totalBase, totalBonus, err := ms.claimRewardsAndUpdatesPositions(ctx, positions)
	if err != nil {
		return nil, err
	}

	sdkCtx := sdk.UnwrapSDKContext(ctx)
	if err := sdkCtx.EventManager().EmitTypedEvent(&types.EventTierRewardsClaimed{
		Owner:        msg.Owner,
		PositionIds:  msg.PositionIds,
		BaseRewards:  totalBase,
		BonusRewards: totalBonus,
	}); err != nil {
		return nil, err
	}

	return &types.MsgClaimTierRewardsResponse{
		BaseRewards:  totalBase,
		BonusRewards: totalBonus,
		PositionIds:  msg.PositionIds,
	}, nil
}

func (k Keeper) claimBaseRewards(ctx context.Context, pos types.PositionState) (sdk.Coins, error) {
	if !pos.IsDelegated() {
		return sdk.NewCoins(), nil
	}
	valAddr, err := sdk.ValAddressFromBech32(pos.Delegation.ValidatorAddress)
	if err != nil {
		return nil, err
	}
	posDelAddr, err := sdk.AccAddressFromBech32(pos.DelegatorAddress)
	if err != nil {
		return nil, errorsmod.Wrap(sdkerrors.ErrInvalidAddress, "invalid delegator address")
	}
	rewards, err := k.distributionKeeper.WithdrawDelegationRewards(ctx, posDelAddr, valAddr)
	if err != nil {
		return nil, err
	}
	if rewards.IsZero() {
		return rewards, nil
	}
	sdkCtx := sdk.UnwrapSDKContext(ctx)
	if err := sdkCtx.EventManager().EmitTypedEvent(&types.EventBaseRewardsClaimed{
		PositionId: pos.Id,
		Owner:      pos.Owner,
		Rewards:    rewards,
	}); err != nil {
		return nil, err
	}
	return rewards, nil
}

func (k Keeper) claimRewardsAndUpdatesPositions(ctx context.Context, positions []types.PositionState) (sdk.Coins, sdk.Coins, error) {
	totalBase := sdk.NewCoins()
	totalBonus := sdk.NewCoins()

	for i := range positions {
		pos := &positions[i]

		if !pos.IsDelegated() {
			continue
		}

		base, err := k.claimBaseRewards(ctx, *pos)
		if err != nil {
			return nil, nil, err
		}
		totalBase = totalBase.Add(base...)

		bonus, err := k.processEventsAndClaimBonus(ctx, pos)
		if err != nil {
			return nil, nil, err
		}
		totalBonus = totalBonus.Add(bonus...)

		if err := k.setPosition(ctx, pos.Position, nil); err != nil {
			return nil, nil, err
		}
	}

	return totalBase, totalBonus, nil
}

func (k Keeper) processEventsAndClaimBonus(ctx context.Context, pos *types.PositionState) (sdk.Coins, error) {
	// Rewards should have been claimed before undelegation
	if !pos.IsDelegated() {
		return sdk.NewCoins(), nil
	}

	valAddr, err := sdk.ValAddressFromBech32(pos.Delegation.ValidatorAddress)
	if err != nil {
		return nil, err
	}

	events, err := k.getValidatorEventsSince(ctx, valAddr, pos.LastEventSeq)
	if err != nil {
		return nil, err
	}

	sdkCtx := sdk.UnwrapSDKContext(ctx)
	blockTime := sdkCtx.BlockTime()

	totalBonus := math.ZeroInt()
	// Use the persisted bonded state from the last replay, not a hardcoded default.
	// This prevents overpaying bonus for unbonded gaps between claims.
	bonded := pos.LastKnownBonded
	segmentStart := pos.LastBonusAccrual

	tier, err := k.getTier(ctx, pos.TierId)
	if err != nil {
		return nil, err
	}

	for _, entry := range events {
		evt := entry.Event

		if bonded {
			// Compute bonus for the bonded segment [segmentStart, eventTime]
			// using the snapshot rate at the event.
			bonus := k.computeSegmentBonus(*pos, tier, segmentStart, evt.Timestamp, evt.TokensPerShare)
			totalBonus = totalBonus.Add(bonus)
		}

		// Update bonded state based on event type.
		switch evt.EventType {
		case types.ValidatorEventType_VALIDATOR_EVENT_TYPE_UNBOND:
			bonded = false
		case types.ValidatorEventType_VALIDATOR_EVENT_TYPE_BOND:
			bonded = true
		case types.ValidatorEventType_VALIDATOR_EVENT_TYPE_SLASH:
			// Slash doesn't change bonded state.
		}

		segmentStart = evt.Timestamp
		pos.UpdateLastEventSeq(entry.Seq)

		// Decrement reference count.
		if err := k.decrementEventRefCount(ctx, valAddr, entry.Seq); err != nil {
			return nil, err
		}
	}

	val, err := k.stakingKeeper.GetValidator(ctx, valAddr)
	if err != nil {
		return nil, err
	}
	// Defensive: validator bond status check
	if bonded && val.IsBonded() {
		currentRate, err := k.getTokensPerShare(ctx, valAddr)
		if err != nil {
			return nil, err
		}
		bonus := k.computeSegmentBonus(*pos, tier, segmentStart, blockTime, currentRate)
		totalBonus = totalBonus.Add(bonus)
	}

	applyBonusAccrualCheckpoint(&pos.Position, blockTime)
	// Persist the bonded state so the next replay starts correctly.
	pos.UpdateLastKnownBonded(bonded)

	if totalBonus.IsZero() {
		return sdk.NewCoins(), nil
	}

	bondDenom, err := k.stakingKeeper.BondDenom(ctx)
	if err != nil {
		return nil, err
	}

	bonusCoins := sdk.NewCoins(sdk.NewCoin(bondDenom, totalBonus))

	if err := k.sufficientBonusPoolBalance(ctx, bonusCoins); err != nil {
		return nil, err
	}

	ownerAddr, err := sdk.AccAddressFromBech32(pos.Owner)
	if err != nil {
		return nil, errorsmod.Wrap(sdkerrors.ErrInvalidAddress, "invalid owner address")
	}

	if err := k.bankKeeper.SendCoinsFromModuleToAccount(ctx, types.RewardsPoolName, ownerAddr, bonusCoins); err != nil {
		return nil, err
	}

	if err := sdkCtx.EventManager().EmitTypedEvent(&types.EventBonusRewardsClaimed{
		PositionId: pos.Id,
		Owner:      pos.Owner,
		Rewards:    bonusCoins,
	}); err != nil {
		return nil, err
	}

	return bonusCoins, nil
}
