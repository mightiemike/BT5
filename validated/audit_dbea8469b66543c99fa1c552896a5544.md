### Title
Unprivileged Caller Can Trigger Aggressive Garbage Collection, Permanently Destroying Mainchain Block Headers - (File: `contract/src/lib.rs`)

### Summary
`run_mainchain_gc` is a public, state-mutating entry point that carries no caller identity check. Any unprivileged NEAR account can invoke it with an arbitrarily large `batch_size`, immediately purging all mainchain block headers beyond `gc_threshold` in a single transaction. This permanently destroys headers that downstream contracts rely on for SPV proof verification and that the contract itself requires for chain reorganization.

### Finding Description
`run_mainchain_gc` is decorated only with `#[pause(except(roles(Role::UnrestrictedRunGC)))]`: [1](#0-0) 

The `#[pause]` macro exclusively checks whether the contract is paused. When the contract is **not** paused — its normal production state — the macro imposes zero restriction on who the caller is. No role check, no `#[private]`, no `require!(env::predecessor_account_id() == ...)` guard exists for the unpaused path.

The function then unconditionally deletes mainchain block headers and advances `mainchain_initial_blockhash`: [2](#0-1) 

An attacker supplies `batch_size = u64::MAX`. The inner `std::cmp::min(total_amount_to_remove, batch_size)` resolves to `total_amount_to_remove`, so the entire purgeable window is deleted in one call: [3](#0-2) 

The design intent is that GC runs automatically inside `submit_blocks` with `batch_size = num_of_headers` (a small, bounded value per batch): [4](#0-3) 

The manual `run_mainchain_gc` entry point is meant as an operator tool, but the absence of a caller guard makes it freely accessible to any account.

### Impact Explanation
Two concrete, irreversible state corruptions result:

1. **SPV proof forgery / denial**: `verify_transaction_inclusion` and `verify_transaction_inclusion_v2` look up the block header by hash from `headers_pool`. Once a header is GC'd, any call referencing that block panics with `"cannot find requested transaction block"`. Downstream contracts that gate asset releases on a `true` SPV result are permanently blocked for any transaction confirmed in a GC'd block.

2. **Reorg failure**: The contract's own documentation states: *"If mainchain blocks near the fork point have been garbage collected, reorg will fail — the contract panics with `PrevBlockNotFound`"*: [5](#0-4) 

An attacker who GCs aggressively immediately before a legitimate relayer submits a fork chain causes `submit_blocks` to panic, halting chain tracking.

### Likelihood Explanation
The entry point is unconditionally reachable by any NEAR account with no deposit, no staking, and no role. The call is a single NEAR transaction. The attacker needs only to know the contract address (public on-chain). The damage is immediate and irreversible within the same block.

### Recommendation
Add a role guard to `run_mainchain_gc` that mirrors the pattern already used for `submit_blocks`. The existing `Role::UnrestrictedRunGC` role is already defined but is only used in the pause-bypass path. It should instead gate the function entirely:

```rust
#[access_control_any(roles(Role::DAO, Role::UnrestrictedRunGC))]
pub fn run_mainchain_gc(&mut self, batch_size: u64) { ... }
```

Alternatively, restrict it with `#[private]` if only the contract itself should trigger it, or add an explicit `require!(env::predecessor_account_id() == authorized_gc_account, ...)` check.

### Proof of Concept
1. Deploy the contract normally (not paused).
2. Submit enough blocks so that `mainchain_size > gc_threshold`.
3. From any unprivileged account (no roles granted), call:
   ```
   near call <contract> run_mainchain_gc '{"batch_size": 18446744073709551615}' --accountId attacker.near
   ```
4. The call succeeds. All purgeable mainchain headers are deleted and `mainchain_initial_blockhash` is advanced.
5. Any subsequent call to `verify_transaction_inclusion` for a block that was just GC'd panics with `"cannot find requested transaction block"`, and any `submit_blocks` call that triggers a reorg through the now-missing ancestor panics with `"PrevBlockNotFound"`. [6](#0-5)

```rust
    #[pause(except(roles(Role::UnrestrictedRunGC)))]
    pub fn run_mainchain_gc(&mut self, batch_size: u64) {
        let initial_blockheader = self
            .headers_pool
            .get(&self.mainchain_initial_blockhash)
            .unwrap_or_else(|| env::panic_str(ERR_KEY_NOT_EXIST));

        let tip_blockheader = self
            .headers_pool
            .get(&self.mainchain_tip_blockhash)
            .unwrap_or_else(|| env::panic_str(ERR_KEY_NOT_EXIST));

        let amount_of_headers_we_store =
            tip_blockheader.block_height - initial_blockheader.block_height + 1;

        if amount_of_headers_we_store > self.gc_threshold {
            let total_amount_to_remove = amount_of_headers_we_store - self.gc_threshold;
            let selected_amount_to_remove = std::cmp::min(total_amount_to_remove, batch_size);

            let start_removal_height = initial_blockheader.block_height;
            let end_removal_height = initial_blockheader.block_height + selected_amount_to_remove;
            env::log_str(&format!(
                "Num of blocks to remove {selected_amount_to_remove}"
            ));

            for height in start_removal_height..end_removal_height {
                let blockhash = &self
                    .mainchain_height_to_header
                    .get(&height)
                    .unwrap_or_else(|| env::panic_str(ERR_KEY_NOT_EXIST));

                self.remove_block_header(blockhash);
                self.mainchain_height_to_header.remove(&height);
            }

            self.mainchain_initial_blockhash = self
                .mainchain_height_to_header
                .get(&end_removal_height)
                .unwrap_or_else(|| env::panic_str(ERR_KEY_NOT_EXIST));
        }
    }
```
