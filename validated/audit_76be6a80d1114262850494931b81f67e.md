### Title
`mintNlp` / `burnNlp` Have No User-Controlled Minimum Return Check and `oraclePriceX18` Is Excluded from the User's EIP-712 Signature — (`core/contracts/Clearinghouse.sol`)

---

### Summary

`Clearinghouse.mintNlp` computes the NLP tokens a user receives as `nlpAmount = quoteAmount / oraclePriceX18`. The `oraclePriceX18` is supplied by the sequencer inside `SignedMintNlp` and is **not covered by the user's EIP-712 signature**. The user's signed struct (`MintNlp`) only commits to `sender`, `quoteAmount`, and `nonce`. There is no on-chain minimum NLP amount check. The same structural flaw applies symmetrically to `burnNlp`, where `quoteAmount = nlpAmount * oraclePriceX18` is the return the user receives.

---

### Finding Description

The EIP-712 type string for `MintNlp` is:

```
"MintNlp(bytes32 sender,uint128 quoteAmount,uint64 nonce)"
``` [1](#0-0) 

The `computeDigest` function for `MintNlp` hashes only the three fields of the inner `MintNlp` struct:

```solidity
digest = keccak256(abi.encode(
    keccak256(bytes(MINT_NLP_SIGNATURE)),
    signedTx.tx.sender,
    signedTx.tx.quoteAmount,
    signedTx.tx.nonce
));
``` [2](#0-1) 

The `oraclePriceX18` field lives in the outer `SignedMintNlp` wrapper, which is assembled and submitted by the sequencer:

```solidity
struct SignedMintNlp {
    MintNlp tx;
    bytes signature;
    int128 oraclePriceX18;       // ← NOT signed by user
    int128[] nlpPoolRebalanceX18;
}
``` [3](#0-2) 

In `Clearinghouse.mintNlp`, the NLP tokens minted to the user are computed entirely from this unsigned price:

```solidity
int128 nlpAmount = quoteAmount.div(oraclePriceX18);
``` [4](#0-3) 

There is no check of the form `require(nlpAmount >= txn.minNlpAmount)`. The user has no on-chain mechanism to bound the conversion rate they accept.

The same flaw applies to `burnNlp`. The user signs only `sender`, `nlpAmount`, `nonce`:

```
"BurnNlp(bytes32 sender,uint128 nlpAmount,uint64 nonce)"
``` [5](#0-4) 

The quote returned is `quoteAmount = nlpAmount * oraclePriceX18`, again driven entirely by the sequencer-supplied price with no user-specified floor:

```solidity
int128 quoteAmount = nlpAmount.mul(oraclePriceX18);
``` [6](#0-5) 

---

### Impact Explanation

**For `mintNlp`:** A user who signs a `MintNlp` for `quoteAmount = 10,000 USDC` expecting to receive NLP at a price of `1.00` (i.e., 10,000 NLP) will receive only 5,000 NLP if the sequencer submits `oraclePriceX18 = 2.00`. The full `quoteAmount` is still debited from the user's subaccount. The user suffers a 50% dilution of their expected LP share with no recourse.

**For `burnNlp`:** A user burning `nlpAmount = 10,000 NLP` expecting `10,000 USDC` back receives only `5,000 USDC` (minus fee) if the sequencer submits a deflated `oraclePriceX18 = 0.50`. The NLP is fully burned regardless.

In both cases the corrupted state delta is: `spotEngine` balance of `NLP_PRODUCT_ID` and `QUOTE_PRODUCT_ID` for `txn.sender` are updated at a price the user never agreed to. [7](#0-6) [8](#0-7) 

---

### Likelihood Explanation

The trigger does not require a malicious sequencer. It fires whenever the NLP oracle price moves between the moment the user signs the transaction and the moment the sequencer includes it in a batch — a routine occurrence in any volatile market. The sequencer has no on-chain constraint on which `oraclePriceX18` value it supplies; the contract accepts any positive value. Because the user's signature is valid for any price the sequencer chooses to attach, there is no cryptographic protection. The entry path is the normal `MintNlp` / `BurnNlp` user flow through `EndpointTx`:

```solidity
clearinghouse.mintNlp(
    signedTx.tx,
    signedTx.oraclePriceX18,   // sequencer-chosen, not user-signed
    nlpPools,
    signedTx.nlpPoolRebalanceX18
);
``` [9](#0-8) 

---

### Recommendation

1. Add a `minNlpAmount` field to the `MintNlp` struct and include it in the EIP-712 digest. In `Clearinghouse.mintNlp`, assert `nlpAmount >= txn.minNlpAmount`.
2. Add a `minQuoteAmount` field to the `BurnNlp` struct and include it in the EIP-712 digest. In `Clearinghouse.burnNlp`, assert `quoteAmount >= txn.minQuoteAmount`.
3. Alternatively, include `oraclePriceX18` itself in the signed digest so the user commits to an exact price, and the sequencer cannot substitute a different one.

---

### Proof of Concept

1. User signs `MintNlp { sender: alice, quoteAmount: 10_000e18, nonce: 5 }` when NLP price is `1.00`.
2. NLP price rises to `2.00` before the sequencer batches the transaction.
3. Sequencer wraps the signed tx in `SignedMintNlp { tx: <above>, oraclePriceX18: 2e18, ... }`.
4. `validateSignedTx` passes — the digest covers only `sender`, `quoteAmount`, `nonce`; `oraclePriceX18` is not checked.
5. `mintNlp` executes: `nlpAmount = 10_000e18 / 2e18 = 5_000e18`. Alice receives 5,000 NLP instead of 10,000.
6. Alice's `QUOTE_PRODUCT_ID` balance is debited the full `10,000 USDC`. She has no on-chain protection. [10](#0-9) [11](#0-10)
