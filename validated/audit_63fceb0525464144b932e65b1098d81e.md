Looking at the `DepositAllowlistExtension`, `SwapAllowlistExtension`, `MetricOmmPoolLiquidityAdder`, and `MetricOmmPool` together, I can identify a concrete analog.

---

### Title
`DepositAllowlistExtension` gates on `owner` instead of `sender`, letting any unpermissioned caller bypass the deposit guard — (`metric-periphery/contracts/extensions/DepositAllowlistExtension.sol`)

---

### Summary
`DepositAllowlistExtension.beforeAddLiquidity` silently drops the `sender` argument and checks only `owner`. Because `pool.addLiquidity` accepts a caller-supplied `owner`, any unpermissioned address can bypass the deposit restriction by naming an allowlisted address as `owner`. The caller's tokens enter the pool; the allowlisted address receives the LP position.

---

function beforeAddLiquidity(address, address owner, uint80, LiquidityDelta calldata, bytes calldata)
    external view override returns (bytes4)
{
    if (!allowAllDepositors[msg.sender] && !allowedDepositor[msg.sender][owner]) {
        revert IMetricOmmPoolActions.NotAllowedToDeposit();
    }
    return IMetricOmmExtensions.beforeAddLiquidity.selector;
}
