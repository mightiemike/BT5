Audit Report

## Title
`SwapAllowlistExtension` checks the router address instead of the originating swapper, enabling complete allowlist bypass via `MetricOmmSimpleRouter` â€” (`metric-periphery/contracts/extensions/SwapAllowlistExtension.sol`)

## Summary
`SwapAllowlistExtension.beforeSwap` gates swap access by checking `allowedSwapper[pool][sender]`, where `sender` is the pool's `msg.sender` â€” the direct caller of `pool.swap()`. When swaps are routed through `MetricOmmSimpleRouter`, the pool's `msg.sender` is the router contract, not the originating user. Any pool admin who allowlists the router to support standard periphery UX inadvertently grants every unpermitted user the ability to bypass the allowlist by routing through the same public router.

## Finding Description

**Root cause â€” pool forwards `msg.sender` as `sender`:**

`MetricOmmPool.swap()` passes `msg.sender` as the first argument to `_beforeSwap`: [1](#0-0) 

`_beforeSwap` encodes that value as the `sender` argument of the hook call, which is then received by `SwapAllowlistExtension.beforeSwap`.

**Extension checks the wrong address:**

`SwapAllowlistExtension.beforeSwap` evaluates `allowedSwapper[msg.sender][sender]`, where `msg.sender` is the pool and `sender` is whoever called `pool.swap()`: [2](#0-1) 

**Router substitutes itself as the caller:**

`MetricOmmSimpleRouter.exactInputSingle` calls `pool.swap()` directly. The pool therefore sees the router as `msg.sender`, not the originating EOA: [3](#0-2) 

The same pattern applies to `exactInput`, `exactOutputSingle`, and `exactOutput`: [4](#0-3) 

**The dilemma for pool admins:**

`setAllowedToSwap` stores the allowlist entry keyed by the address that will appear as `sender` in the hook: [5](#0-4) 

A pool admin who wants allowlisted users to use the router must call `setAllowedToSwap(pool, router, true)`. Once set, `allowedSwapper[pool][router] == true` for every caller who routes through the router â€” including every address that was never individually permitted.

**Contrast with `DepositAllowlistExtension`:**

`DepositAllowlistExtension.beforeAddLiquidity` checks the `owner` parameter, which the liquidity adder passes explicitly and which the pool does not replace with `msg.sender`. This design correctly identifies the actual LP, not the intermediary: [6](#0-5) 

The swap hook has no equivalent "originating user" parameter â€” `sender` is structurally bound to the pool's `msg.sender`.

## Impact Explanation

A curated pool relying on `SwapAllowlistExtension` to restrict trading to specific counterparties loses that protection entirely for any user who routes through `MetricOmmSimpleRouter`. An unauthorized user can execute swaps at the pool's oracle-anchored bid/ask prices, draining LP-owned token balances at rates the pool admin never intended to offer. Because the pool's pricing is oracle-driven and not self-correcting, repeated unauthorized swaps in one direction can exhaust one side of the pool's liquidity, causing direct loss of LP principal. This constitutes a **High** severity direct loss of user principal.

## Likelihood Explanation

`MetricOmmSimpleRouter` is a public, permissionless contract. Any user can call it. The only precondition is that the pool admin has allowlisted the router â€” a natural and expected configuration step for any pool that wants to support the standard periphery UX. The bypass requires no special privileges, no flash loans, and no multi-step setup. It is trivially repeatable.

## Recommendation

The extension must gate the **originating user**, not the intermediary. Two viable approaches:

1. **Extension-data forwarding (preferred):** Require the router to encode the originating user's address in `extensionData` and have the extension decode and check that address. The extension should revert if the field is absent or zero. The router already accepts per-hop `extensionData` and forwards it unchanged to the pool, which forwards it to the extension.
2. **Direct-pool-only enforcement:** Document that `SwapAllowlistExtension` is incompatible with router-mediated swaps and add an explicit check that `sender == tx.origin` (with the usual caveats about smart-contract callers), or revert if `sender` is a known router address.

## Proof of Concept

1. Pool admin deploys a pool with `SwapAllowlistExtension` as a `beforeSwap` hook.
2. Pool admin allowlists Alice: `setAllowedToSwap(pool, alice, true)`.
3. Pool admin allowlists the router so Alice can use the standard UX: `setAllowedToSwap(pool, router, true)`.
4. Bob (never allowlisted) calls `router.exactInputSingle({pool: pool, recipient: bob, ...})`.
5. The router calls `pool.swap(bob, ...)` â€” pool's `msg.sender` is the router.
6. The pool calls `extension.beforeSwap(router, bob, ...)` â€” extension's `sender` is the router.
7. Extension evaluates `allowedSwapper[pool][router] == true` â†’ passes without revert.
8. Bob's swap executes at the oracle price, draining pool liquidity the admin intended to reserve for Alice only.
