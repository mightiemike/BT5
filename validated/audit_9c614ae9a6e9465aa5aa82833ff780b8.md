Audit Report

## Title
`SwapAllowlistExtension` checks the router address as `sender` instead of the end user, allowing any caller to bypass the per-pool swap allowlist via `MetricOmmSimpleRouter` — (`metric-periphery/contracts/extensions/SwapAllowlistExtension.sol`)

## Summary
`SwapAllowlistExtension.beforeSwap` checks `allowedSwapper[pool][sender]` where `sender` is the direct caller of `MetricOmmPool.swap`. When a user routes through `MetricOmmSimpleRouter`, the pool's `msg.sender` is the router contract, not the end user. If the router is allowlisted (required for any allowlisted user to use the router), every non-allowlisted address can bypass the gate by calling `exactInputSingle`, `exactInput`, `exactOutputSingle`, or `exactOutput` through the public router.

## Finding Description

`MetricOmmPool.swap` passes its own `msg.sender` as `sender` to `_beforeSwap`:

```solidity
// MetricOmmPool.sol L230-240
_beforeSwap(
    msg.sender,   // ← direct caller of pool.swap()
    recipient,
    ...
);
```

`ExtensionCalling._beforeSwap` forwards this value unchanged to the extension:

```solidity
// ExtensionCalling.sol L160-176
_callExtensionsInOrder(
    BEFORE_SWAP_ORDER,
    abi.encodeCall(IMetricOmmExtensions.beforeSwap, (sender, ...))
);
```

`SwapAllowlistExtension.beforeSwap` then checks it against the per-pool allowlist:

```solidity
// SwapAllowlistExtension.sol L37-38
if (!allowAllSwappers[msg.sender] && !allowedSwapper[msg.sender][sender]) {
    revert IMetricOmmPoolActions.NotAllowedToSwap();
}
```

`MetricOmmSimpleRouter.exactInputSingle` calls `pool.swap` directly, making the router the pool's `msg.sender`:

```solidity
// MetricOmmSimpleRouter.sol L71-80
_setNextCallbackContext(params.pool, CALLBACK_MODE_JUST_PAY, msg.sender, params.tokenIn);
(int128 amount0Delta, int128 amount1Delta) = IMetricOmmPoolActions(params.pool)
    .swap(params.recipient, params.zeroForOne, ...);
```

The pool therefore passes `router` as `sender` to the extension. The extension evaluates `allowedSwapper[pool][router]`, not `allowedSwapper[pool][actual_user]`. The same pattern applies to `exactInput` (L103-112), `exactOutputSingle` (L135-137), and `exactOutput` (L165-181).

For any allowlisted user to use the router, the pool admin must call `setAllowedToSwap(pool, router, true)`. Once the router is allowlisted, the check `allowedSwapper[pool][router] == true` passes for every caller of the router, regardless of whether the actual end user is on the allowlist.

## Impact Explanation

A pool admin who deploys `SwapAllowlistExtension` to restrict swaps to a curated set of addresses (KYC-verified wallets, whitelisted market makers, protocol-owned contracts) faces a binary failure:

1. **Router allowlisted**: every non-allowlisted address can call `exactInputSingle` through the public router and bypass the gate entirely. The allowlist is nullified — any attacker can swap on the restricted pool, bypassing compliance controls or draining LP assets at oracle-quoted prices.
2. **Router not allowlisted**: allowlisted users cannot use the router at all, breaking the intended UX.

This is broken core pool functionality (the allowlist guard) causing potential direct loss of LP principal and a complete bypass of an admin-configured access control. Meets the "broken core pool functionality causing loss of funds" and "admin-boundary break bypassed by an unprivileged path" impact criteria.

## Likelihood Explanation

`MetricOmmSimpleRouter` is the primary public entry point for swaps. No special privileges, flash loans, or multi-block setup are required. A single `exactInputSingle` call from any EOA suffices. The bypass is reachable by any unprivileged address on any pool that has `SwapAllowlistExtension` configured in `BEFORE_SWAP_ORDER` with the router allowlisted. The condition (router must be allowlisted) is a necessary operational requirement, not a mitigating one.

## Recommendation

The extension must check the original end user, not the intermediary router. Options:

1. **Router-attested `extensionData`**: The router encodes `msg.sender` (the actual user) into `extensionData` with a trusted signature or a dedicated router-only encoding scheme. The extension decodes and verifies it, trusting only known router addresses.
2. **Dedicated `payer` field in pool swap context**: Add a `payer` field to the pool's swap call that the router sets to `msg.sender` (the actual user), and have the extension read `payer` instead of `sender`.
3. **Documentation minimum**: At minimum, document that `SwapAllowlistExtension` only gates direct pool callers and is incompatible with the public router.

## Proof of Concept

1. Pool admin deploys a pool with `SwapAllowlistExtension` in `BEFORE_SWAP_ORDER`.
2. Pool admin calls `setAllowedToSwap(pool, router, true)` so allowlisted users can use the router.
3. Attacker (address not in `allowedSwapper`) calls `MetricOmmSimpleRouter.exactInputSingle({pool: restrictedPool, ...})`.
4. Router calls `pool.swap(recipient, ...)` — pool's `msg.sender` = router.
5. Pool calls `_beforeSwap(sender=router, ...)` → `ExtensionCalling._callExtensionsInOrder` → `SwapAllowlistExtension.beforeSwap(sender=router, ...)`.
6. Extension evaluates `allowedSwapper[pool][router]` → `true` → swap proceeds.
7. Attacker receives output tokens from the restricted pool without being on the allowlist.

Foundry test skeleton:
```solidity
function test_swapAllowlistBypassViaRouter() public {
    // Setup: pool with SwapAllowlistExtension, router allowlisted
    extension.setAllowedToSwap(pool, address(router), true);
    // Attacker is NOT allowlisted
    assertFalse(extension.isAllowedToSwap(pool, attacker));
    // Attacker calls router — should revert but does not
    vm.prank(attacker);
    router.exactInputSingle(ExactInputSingleParams({pool: pool, ...}));
    // Assert attacker received output tokens
}
