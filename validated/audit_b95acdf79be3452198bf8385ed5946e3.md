Audit Report

## Title
`SwapAllowlistExtension.beforeSwap()` checks router address instead of original user, allowing any user to bypass per-user swap allowlist via `MetricOmmSimpleRouter` — (`metric-periphery/contracts/extensions/SwapAllowlistExtension.sol`)

## Summary

`SwapAllowlistExtension.beforeSwap()` gates swaps by checking the `sender` argument, which equals `msg.sender` at the pool level — the direct caller of `pool.swap()`. When `MetricOmmSimpleRouter` mediates a swap, it is the direct caller, so `sender` = router address. If the pool admin allowlists the router (the natural step to let allowlisted users trade via the router), every user — including those explicitly excluded from the per-user allowlist — can bypass the guard by routing through the router. This constitutes an admin-boundary break where an unprivileged path voids the pool admin's configured access-control boundary.

## Finding Description

**Root cause — wrong identity is checked**

`SwapAllowlistExtension.beforeSwap()` performs:

```solidity
if (!allowAllSwappers[msg.sender] && !allowedSwapper[msg.sender][sender]) {
    revert IMetricOmmPoolActions.NotAllowedToSwap();
}
``` [1](#0-0) 

Here `msg.sender` is the pool (the extension is called by the pool) and `sender` is whoever called `pool.swap()`. The check is therefore `allowedSwapper[pool][direct_caller_of_pool_swap]`.

**How the pool sets `sender`**

`MetricOmmPool.swap()` passes `msg.sender` (its own direct caller) as the `sender` argument to `_beforeSwap()`:

```solidity
_beforeSwap(
    msg.sender,   // ← direct caller of pool.swap(), not the original user
    recipient,
    ...
``` [2](#0-1) 

**How the router loses the original user**

`MetricOmmSimpleRouter.exactInputSingle()` stores the original user in transient storage for the payment callback, then calls `pool.swap()` directly without forwarding the original user's address to the pool or any extension:

```solidity
_setNextCallbackContext(params.pool, CALLBACK_MODE_JUST_PAY, msg.sender, params.tokenIn);
IMetricOmmPoolActions(params.pool).swap(
    params.recipient, params.zeroForOne, ..., params.extensionData
);
``` [3](#0-2) 

When the pool receives this call, `msg.sender` = router address, so `sender` passed to the extension = router address. The extension evaluates `allowedSwapper[pool][router]`. If the pool admin has allowlisted the router, this check passes for **every** user who routes through the router, regardless of whether that user is individually allowlisted or explicitly excluded.

The allowlist admin setter confirms the per-pool, per-swapper design intent:

```solidity
function setAllowedToSwap(address pool_, address swapper, bool allowed) external onlyPoolAdmin(pool_) {
    allowedSwapper[pool_][swapper] = allowed;
``` [4](#0-3) 

The design intent is per-user gating. The implementation collapses all router users into a single identity (the router address).

The same bypass applies to `exactOutputSingle()` and `exactInput()` / `exactOutput()` multi-hop paths, all of which call `pool.swap()` with `msg.sender` = router. [5](#0-4) 

## Impact Explanation

Any user excluded from the per-user allowlist can bypass the `SwapAllowlistExtension` guard by calling `MetricOmmSimpleRouter.exactInputSingle()` (or `exactInput` / `exactOutput` / `exactOutputSingle`) on a pool where the router address is allowlisted. The pool admin's configured access-control boundary is silently voided. Pools designed to restrict swaps to specific counterparties (e.g., institutional market makers, KYC-verified addresses, or protocol-controlled accounts) will accept swaps from arbitrary addresses. This is an admin-boundary break: an unprivileged path bypasses the pool admin's access-control enforcement, matching the allowed impact gate for this contest.

## Likelihood Explanation

Likelihood is **medium**. The trigger condition — the router being allowlisted — is the natural and expected configuration for any pool that wants allowlisted users to be able to trade via the standard periphery router. A pool admin who allowlists Alice and also allowlists the router to let Alice use the router will not realize they have simultaneously opened the pool to all router users. The bypass requires no special privilege, no flash loan, and no contract deployment; any EOA can execute it by calling the production `MetricOmmSimpleRouter`.

## Recommendation

The extension must verify the **original user's identity**, not the intermediary's. Two viable approaches:

1. **Pass original user in `extensionData`**: Have the router encode `msg.sender` into `extensionData` before calling `pool.swap()`, and have the extension decode and verify it. This requires a convention between router and extension.
2. **Router registry with transient-storage fallback**: The extension could detect that `sender` is a known router and require the extension data to carry a signed or transient-storage-backed original user address.
3. **Short-term mitigation**: Document that allowlisting the router is equivalent to `setAllowAllSwappers(pool, true)` and that per-user restrictions cannot be enforced for router-mediated swaps.

## Proof of Concept

```
Setup:
  pool   = MetricOmmPool with SwapAllowlistExtension in BEFORE_SWAP_ORDER
  alice  = allowlisted swapper  (setAllowedToSwap(pool, alice, true))
  bob    = NOT allowlisted
  router = MetricOmmSimpleRouter

Step 1 — admin allowlists router so Alice can use it:
  setAllowedToSwap(pool, router, true)

Step 2 — Bob calls the router (not the pool directly):
  router.exactInputSingle({
      pool:      pool,
      recipient: bob,
      zeroForOne: true,
      amountIn:  X,
      ...
  })

Step 3 — router calls pool.swap(); msg.sender in pool = router:
  pool.swap(bob, true, X, limit, "", extensionData)
  → _beforeSwap(sender=router, ...)

Step 4 — extension evaluates:
  allowedSwapper[pool][router]  →  TRUE   ← router is allowlisted
  → no revert, swap proceeds

Result: Bob swaps successfully on a pool that was intended to block him.
Direct pool.swap() from Bob's EOA would have reverted:
  allowedSwapper[pool][bob]  →  FALSE  → NotAllowedToSwap ✓
```

The bypass is reachable by any unprivileged address through the production `MetricOmmSimpleRouter` without any special setup beyond the pool admin performing the natural configuration of allowlisting the router.
