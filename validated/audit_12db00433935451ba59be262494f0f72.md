### Title
SwapAllowlistExtension checks the router address instead of the end user, allowing any user to bypass the swap allowlist via MetricOmmSimpleRouter - (File: metric-periphery/contracts/extensions/SwapAllowlistExtension.sol)

### Summary

`SwapAllowlistExtension.beforeSwap` receives `sender` from the pool, which is `msg.sender` of `pool.swap()`. When `MetricOmmSimpleRouter` intermediates the call, `sender` is the router address, not the end user. The allowlist check is therefore applied to the router rather than the actual swapper, making the guard trivially bypassable.

### Finding Description

`SwapAllowlistExtension` is designed to gate individual swappers per pool: [1](#0-0) 

```solidity
function beforeSwap(address sender, address, bool, int128, uint128, uint256, uint128, uint128, bytes calldata)
    external view override returns (bytes4)
{
    if (!allowAllSwappers[msg.sender] && !allowedSwapper[msg.sender][sender]) {
        revert IMetricOmmPoolActions.NotAllowedToSwap();
    }
    return IMetricOmmExtensions.beforeSwap.selector;
}
```

Here `msg.sender` is the pool (correct pool key), and `sender` is the first argument forwarded by the pool from its own `msg.sender` — i.e., whoever called `pool.swap()`. When `MetricOmmSimpleRouter` calls `pool.swap()`, the pool's `msg.sender` is the router contract, so `sender = router address`.

The allowlist lookup therefore becomes:

```
allowedSwapper[pool][router]   // checked
allowedSwapper[pool][endUser]  // never checked
```

The `DepositAllowlistExtension` avoids this problem because it checks `owner`, which is an explicit parameter passed through the call stack and represents the actual position owner regardless of who the intermediary caller is: [2](#0-1) 

`SwapAllowlistExtension` has no equivalent explicit end-user parameter — it relies solely on `sender`, which collapses to the router address in the router path.

### Impact Explanation

Two fund-impacting failure modes arise:

1. **Allowlist bypass (unauthorized swaps):** If the pool admin allowlists the router (a natural operational step to let users swap through the standard interface), every user — including those explicitly denied — can swap freely through the router. The allowlist guard is completely defeated for the router path.

2. **Allowlist DoS (authorized users locked out):** If the pool admin does not allowlist the router, all individually allowlisted users are unable to swap through `MetricOmmSimpleRouter` even though they hold explicit permission. They are forced to call `pool.swap()` directly, which may not be practical or may not be the intended UX.

Both outcomes break the core invariant that the allowlist controls which addresses may trade in a restricted pool, matching the "broken core pool functionality" and "admin-boundary break" impact categories.

### Likelihood Explanation

- No special preconditions: any user can call `MetricOmmSimpleRouter` without privilege.
- The router is the standard, documented swap entry point for the protocol.
- Pool admins who configure a `SwapAllowlistExtension` will naturally expect it to gate end users, not the router contract.
- The bypass requires only that the router is allowlisted (or that the admin has set `allowAllSwappers = true` for the router), which is the expected operational state.

### Recommendation

Pass the actual end user through the extension data or add a dedicated `swapper` parameter to the `beforeSwap` hook that the pool populates from a user-supplied field (analogous to how `owner` is explicit in `beforeAddLiquidity`). Alternatively, `SwapAllowlistExtension` should check the `recipient` argument (the second positional parameter, currently unnamed/ignored) if that reliably represents the end user, or the router should encode `msg.sender` into `extensionData` and the extension should decode it.

### Proof of Concept

1. Pool admin deploys a pool with `SwapAllowlistExtension` as `extension1` and `beforeSwap` order set.
2. Admin calls `setAllowedToSwap(pool, router, true)` to allow the router (standard operational step).
3. Admin calls `setAllowedToSwap(pool, alice, false)` (or simply never allowlists Alice).
4. Alice calls `MetricOmmSimpleRouter.exactInputSingle(...)` targeting the pool.
5. Router calls `pool.swap(...)` — pool's `msg.sender` = router.
6. Pool calls `extension.beforeSwap(router, ...)`.
7. Extension evaluates `allowedSwapper[pool][router] == true` → passes.
8. Alice's swap executes despite being explicitly denied. [3](#0-2) [4](#0-3)
