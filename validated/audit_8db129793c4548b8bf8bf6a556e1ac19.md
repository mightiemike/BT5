### Title
Negative `canSettle` in `settlePnl` Improves Loser's `vQuoteBalance` and Inflates `availableSettle` Pool — (`core/contracts/PerpEngine.sol`)

### Summary

`settlePnl` applies `canSettle` unconditionally without guarding against negative values. When a subaccount holds a losing perp position (`positionPnl < 0`), `canSettle` becomes negative, which **increases** `balance.vQuoteBalance` and **increases** `state.availableSettle`, creating value from nothing.

---

### Finding Description

`getSettlementState` computes:

```solidity
availableSettle = MathHelper.min(
    calculatePositionPnl(balance, productId),
    state.availableSettle
);
``` [1](#0-0) 

`calculatePositionPnl` is:

```solidity
positionPnl = priceX18.mul(balance.amount) + balance.vQuoteBalance;
``` [2](#0-1) 

For a losing position (`positionPnl < 0`) with a non-negative `state.availableSettle`, `MathHelper.min` returns the negative `positionPnl`. Back in `settlePnl`, this negative `canSettle` is applied without any positivity guard:

```solidity
state.availableSettle -= canSettle;   // subtracts a negative → INCREASES pool
balance.vQuoteBalance -= canSettle;   // subtracts a negative → INCREASES user vQuote
totalSettled += canSettle;            // returns a negative value to caller
``` [3](#0-2) 

There is no `if (canSettle <= 0) continue;` guard anywhere in the loop.

---

### Impact Explanation

| Effect | Direction | Consequence |
|---|---|---|
| `balance.vQuoteBalance` | Increases by `\|positionPnl\|` | User's perp loss is erased |
| `state.availableSettle` | Increases by `\|positionPnl\|` | Phantom quote added to settlement pool |
| `totalSettled` returned | Negative | Clearinghouse credits user's spot balance |

A subaccount with a deeply negative perp PnL can have its `vQuoteBalance` artificially raised, making it appear healthier than it is. This can suppress a legitimate liquidation trigger and allow the account to continue trading while insolvent. The `availableSettle` pool inflation also means other accounts can over-settle against phantom funds.

---

### Likelihood Explanation

`settlePnl` is gated by `_assertInternal()`, which restricts callers to the endpoint, clearinghouse, and offchain exchange. [4](#0-3) 

The attacker cannot call `settlePnl` directly. However, the clearinghouse calls it during normal settlement flows (trades, liquidations, withdrawals). Any time the clearinghouse invokes `settlePnl` for a subaccount that currently holds a losing position, the bug fires automatically — no special attacker action is required beyond holding a losing perp position when settlement is triggered.

---

### Recommendation

Add a positivity guard before applying settlement:

```solidity
// In settlePnl, inside the while loop:
if (canSettle <= 0) {
    productIds >>= 32;
    continue;
}
```

Or equivalently clamp at zero:

```solidity
canSettle = MathHelper.max(canSettle, 0);
``` [5](#0-4)

### Proof of Concept

**Setup:**
- Product ID `2` (even, perp), `priceX18 = 1e18` (price = 1.0)
- Subaccount opens a long: `balance.amount = 100e18`, `balance.vQuoteBalance = -110e18` (bought at 1.10, now worth 1.00 → PnL = -10e18)
- `state.availableSettle = 5e18` (some real quote deposited)

**Trace through `settlePnl`:**

1. `calculatePositionPnl` → `1e18 * 100e18 / 1e18 + (-110e18)` = `-10e18`
2. `canSettle = min(-10e18, 5e18)` = `-10e18`
3. `state.availableSettle -= (-10e18)` → `5e18 + 10e18 = 15e18` (**inflated**)
4. `balance.vQuoteBalance -= (-10e18)` → `-110e18 + 10e18 = -100e18` (**loss erased**)
5. `totalSettled = -10e18` returned to clearinghouse

The user's perp loss shrinks from `-10e18` to `0` (break-even), and the settlement pool grows by `10e18` of phantom value. The clearinghouse receives `-10e18` as `totalSettled`, which — depending on its handling — may additionally credit the user's spot balance. [6](#0-5)

    function settlePnl(bytes32 subaccount, uint256 productIds)
        external
        returns (int128)
    {
        _assertInternal();
        int128 totalSettled = 0;

        while (productIds != 0) {
            uint32 productId = uint32(productIds & ((1 << 32) - 1));
            // otherwise it means the product is a spot.
            if (productId % 2 == 0) {
                (
                    int128 canSettle,
                    State memory state,
                    Balance memory balance
                ) = getSettlementState(productId, subaccount);

                state.availableSettle -= canSettle;
                balance.vQuoteBalance -= canSettle;

                totalSettled += canSettle;

                _setState(productId, state);
                _setBalanceAndUpdateBitmap(productId, subaccount, balance);
            }
            productIds >>= 32;
        }
        return totalSettled;
    }
```
