### Title
`ValidResourceBounds` Canonicalization Collapse in Protobuf Round-Trip Produces Hash Mismatch and Consensus Rejection of Valid Transactions - (`crates/apollo_protobuf/src/converters/transaction.rs`)

---

### Summary

The protobuf deserializer for `ValidResourceBounds` silently collapses `AllResources { l1_gas: X, l2_gas: 0, l1_data_gas: 0 }` into `L1Gas(X)`. Because the transaction hash function treats these two variants differently (the `AllResources` path includes an extra `L1_DATA_GAS` felt in the Poseidon preimage), a V3 invoke transaction whose resource bounds happen to have zero L2 and L1-data gas is accepted by the gateway with one hash, but the consensus protobuf round-trip fails entirely, causing the transaction to be dropped before sequencing.

---

### Finding Description

**Canonicalization collapse in `TryFrom<protobuf::ResourceBounds> for ValidResourceBounds`** [1](#0-0) 

When `l1_data_gas` is absent from the wire (old 0.13.2 peers) or present but zero, the deserializer applies:

```rust
Ok(if l1_data_gas.is_zero() && l2_gas.is_zero() {
    ValidResourceBounds::L1Gas(l1_gas)   // ← collapses AllResources to L1Gas
} else {
    ValidResourceBounds::AllResources(...)
})
```

**Hash function treats the two variants differently**

`get_tip_resource_bounds_hash` includes the `L1_DATA_GAS` felt only for `AllResources`: [2](#0-1) 

So `AllResources { l1_gas: X, l2_gas: 0, l1_data_gas: 0 }` hashes as `[tip, L1_GAS, L2_GAS(0), L1_DATA_GAS(0)]`, while `L1Gas(X)` hashes as `[tip, L1_GAS, L2_GAS(0)]` — a different Poseidon digest.

**Gateway always computes the hash with `AllResources`**

`RpcInvokeTransactionV3` carries `resource_bounds: AllResourceBounds` (always `AllResources`). The gateway converter wraps it unconditionally: [3](#0-2) 

The hash is computed from `InternalRpcTransactionWithoutTxHash` which also always uses `AllResources`: [4](#0-3) 

**Consensus protobuf round-trip breaks for zero-L2/L1-data transactions**

When the proposer serializes the `ConsensusTransaction` to protobuf, `From<RpcInvokeTransactionV3> for protobuf::InvokeV3WithProof` converts through `InvokeTransactionV3` (preserving `AllResources`): [5](#0-4) 

On the receiving validator, `TryFrom<protobuf::InvokeV3WithProof> for RpcInvokeTransactionV3` deserializes the `InvokeTransactionV3` (which collapses to `L1Gas`), then attempts `snapi_invoke.try_into()` back to `RpcInvokeTransactionV3`: [6](#0-5) 

The comment on line 129 explicitly acknowledges: *"This conversion can fail only if the resource_bounds are not AllResources."* The `L1Gas` variant triggers `DEPRECATED_RESOURCE_BOUNDS_ERROR` and the transaction is rejected.

---

### Impact Explanation

**High — valid transactions are rejected before sequencing.**

A V3 invoke transaction with `AllResourceBounds { l1_gas: {amount: N, price: P}, l2_gas: 0, l1_data_gas: 0 }` (non-zero L1 fee, zero L2/L1-data) passes gateway stateless and stateful validation, is assigned a canonical hash, and enters the mempool. When the proposer broadcasts the block proposal over consensus P2P, every validator node fails to deserialize the transaction (`DEPRECATED_RESOURCE_BOUNDS_ERROR`) and rejects it. The transaction is silently dropped from the proposed block, causing liveness failure for that transaction class.

---

### Likelihood Explanation

Any user or automated system that submits a V3 invoke transaction specifying only L1 gas (the common pre-0.13.3 pattern) but using the new `AllResourceBounds` wire format with zero L2 and L1-data fields triggers this path. The gateway accepts it; consensus drops it. The condition `l2_gas.is_zero() && l1_data_gas.is_zero()` is the normal state for L1-only fee transactions.

---

### Recommendation

Remove the collapsing heuristic from `TryFrom<protobuf::ResourceBounds> for ValidResourceBounds`. When `l1_data_gas` is explicitly present on the wire (even as zero), always reconstruct `AllResources`. Only fall back to `L1Gas` when `l1_data_gas` is truly absent (the `None` case for legacy 0.13.2 peers):

```rust
let l1_data_gas_opt = value.l1_data_gas;
Ok(match l1_data_gas_opt {
    None => ValidResourceBounds::L1Gas(l1_gas),   // legacy wire: no data-gas field
    Some(raw) => {
        let l1_data_gas: ResourceBounds = raw.try_into()?;
        ValidResourceBounds::AllResources(AllResourceBounds { l1_gas, l2_gas, l1_data_gas })
    }
})
```

This preserves the `AllResources` variant whenever the sender explicitly encoded it, keeping the hash preimage consistent across the gateway → mempool → consensus → execution pipeline.

---

### Proof of Concept

1. Submit a V3 invoke transaction to the gateway with:
   ```
   resource_bounds = AllResourceBounds {
       l1_gas:      { max_amount: 1000, max_price_per_unit: 1 },
       l2_gas:      { max_amount: 0,    max_price_per_unit: 0 },
       l1_data_gas: { max_amount: 0,    max_price_per_unit: 0 },
   }
   ```
2. Gateway computes `tx_hash` using `get_invoke_transaction_v3_hash` → `get_tip_resource_bounds_hash` with `AllResources` → hash includes `L1_DATA_GAS(0)` felt. Transaction accepted, `tx_hash = H_all`.
3. Proposer calls `convert_internal_consensus_tx_to_consensus_tx` → `RpcInvokeTransactionV3` → `protobuf::InvokeV3WithProof` with `resource_bounds = { l1_gas: 1000/1, l2_gas: 0/0, l1_data_gas: 0/0 }`.
4. Validator deserializes: `TryFrom<protobuf::ResourceBounds> for ValidResourceBounds` sees `l2_gas.is_zero() && l1_data_gas.is_zero()` → returns `L1Gas(l1_gas)`.
5. `TryFrom<InvokeTransactionV3> for RpcInvokeTransactionV3` fails → `DEPRECATED_RESOURCE_BOUNDS_ERROR`.
6. Transaction is rejected by all validators; block proposal either fails or the transaction is silently omitted, despite having a valid gateway-assigned hash `H_all`.

    type Error = ProtobufConversionError;
    fn try_from(value: protobuf::ResourceBounds) -> Result<Self, Self::Error> {

pub fn get_tip_resource_bounds_hash(
    resource_bounds: &ValidResourceBounds,
    tip: &Tip,
) -> Result<Felt, StarknetApiError> {
    let l1_resource_bounds = resource_bounds.get_l1_bounds();
    let l2_resource_bounds = resource_bounds.get_l2_bounds();

    // L1 and L2 gas bounds always exist.
    // Old V3 txs always have L2 gas bounds of zero, but they exist.
    let mut resource_felts = vec![
        get_concat_resource(&l1_resource_bounds, L1_GAS)?,
        get_concat_resource(&l2_resource_bounds, L2_GAS)?,
    ];

    // For new V3 txs, need to also hash the data gas bounds.
    resource_felts.extend(match resource_bounds {
        ValidResourceBounds::L1Gas(_) => vec![],
        ValidResourceBounds::AllResources(all_resources) => {
            vec![get_concat_resource(&all_resources.l1_data_gas, L1_DATA_GAS)?]
        }
    });

    Ok(HashChain::new().chain(&tip.0.into()).chain_iter(resource_felts.iter()).get_poseidon_hash())
}

impl TryFrom<protobuf::InvokeV3WithProof> for RpcInvokeTransactionV3 {
    type Error = ProtobufConversionError;
    fn try_from(mut value: protobuf::InvokeV3WithProof) -> Result<Self, Self::Error> {
        // Extract proof first, since `starknet_api::transaction::InvokeTransactionV3` does not
        // carry a `proof` field.
        let proof = Proof::from(std::mem::take(&mut value.proof));

        let snapi_invoke: InvokeTransactionV3 = value
            .invoke
            .ok_or(ProtobufConversionError::MissingField {
                field_description: "InvokeV3WithProof::invoke",
            })?
            .try_into()?;

        // This conversion can fail only if the resource_bounds are not AllResources.
        Ok(Self { proof, ..snapi_invoke.try_into().map_err(|_| DEPRECATED_RESOURCE_BOUNDS_ERROR)? })
    }
