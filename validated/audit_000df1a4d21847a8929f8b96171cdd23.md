### Title
Stale `_cached_sha256_treehash` Trusted Without Recomputation in `Program.tree_hash()` — (`wheel/python/clvm_rs/program.py`, `wheel/python/clvm_rs/tree_hash.py`)

---

### Summary

`Program.tree_hash()` and `Treehasher.sha256_treehash()` unconditionally trust a pre-existing `_cached_sha256_treehash` attribute on any `CLVMStorage`-compatible Python object without recomputing it. `Program.wrap()` propagates this attribute from the wrapped object into the new `Program` instance. An attacker-controlled Python object carrying a forged `_cached_sha256_treehash` value will cause the entire puzzle-hash pipeline to return an incorrect hash — directly analogous to `exchangeRateStored()` returning a stale cached rate instead of accruing interest first.

---

### Finding Description

**Root cause — `Program.wrap()` blindly copies the cached hash:** [1](#0-0) 

```python
o._cached_serialization = getattr(v, "_cached_serialization", None)
o._cached_sha256_treehash = getattr(v, "_cached_sha256_treehash", None)
```

No verification is performed. Whatever value `v._cached_sha256_treehash` holds is accepted as authoritative.

**Root cause — `Program.tree_hash()` short-circuits on the cached value:** [2](#0-1) 

```python
def tree_hash(self) -> bytes:
    if self._cached_sha256_treehash is None:
        self._cached_sha256_treehash = sha256_treehash(self._unwrapped)
    return self._cached_sha256_treehash
```

If `_cached_sha256_treehash` is non-`None` (copied from the wrapped object), `sha256_treehash` is **never called**. The stored value is returned verbatim.

**Root cause — `Treehasher.sha256_treehash()` trusts the attribute on every sub-node:** [3](#0-2) 

```python
r = getattr(obj, "_cached_sha256_treehash", None)
if r is not None:
    self.cache_hits += 1
    hash_stack.append(r)
    return
```

This applies to every node in the traversal, not just the root. A forged hash on any sub-node poisons the hash of every ancestor.

**Parallel to the Blueberry bug:**

| Blueberry | clvm_rs |
|---|---|
| `exchangeRateStored()` returns cached rate | `Program.tree_hash()` returns `_cached_sha256_treehash` |
| No interest accrual before use | No recomputation before use |
| Wrong risk → wrong liquidation | Wrong hash → wrong puzzle identity |

---

### Impact Explanation

`Program.tree_hash()` is used in:

- **`Program.__eq__`** — equality comparison is entirely hash-based: [4](#0-3) 
  A forged hash makes two structurally different programs compare as equal, or two identical programs compare as unequal.

- **`Program.curry_hash()`** — computes puzzle hashes for curried programs: [5](#0-4) 
  A wrong puzzle hash causes a wallet to derive an incorrect coin address, potentially locking funds permanently.

- **`Program.run_with_cost()`** — serializes the program via `bytes(self)` which also trusts `_cached_serialization`: [6](#0-5) 
  A forged `_cached_serialization` causes a different program to be executed than the one the caller believes they are running, which is a consensus-divergence class impact.

---

### Likelihood Explanation

The prompt explicitly lists **Python objects** as a valid attacker-controlled entry path. The attack requires constructing a `CLVMStorage`-compatible Python object with a forged `_cached_sha256_treehash` and passing it to any API that calls `Program.wrap()` or `Program.to()`. This is realistic in:

1. Any library that accepts user-supplied `CLVMStorage` objects for program composition.
2. Any deserialization path that creates intermediate Python objects before wrapping them in `Program`.
3. Any test harness or tooling that mixes Python-constructed and Rust-backed nodes in the same tree.

The `LazyNode` (Rust-backed) path is safe because `LazyNode` has no `_cached_sha256_treehash` Python attribute, so `getattr` returns `None` and the hash is computed fresh. The vulnerability is confined to the Python-object path.

---

### Recommendation

1. **In `Program.wrap()`**: Do not copy `_cached_sha256_treehash` from the wrapped object. Leave it as `None` and let `tree_hash()` compute it on first access from the actual node content.

2. **In `Treehasher.sha256_treehash()`**: Either remove the `_cached_sha256_treehash` shortcut entirely, or restrict it to objects whose type is known-safe (e.g., `isinstance(obj, Program)` where the cache is set only by this same function).

3. **In `Program.__bytes__()`**: Apply the same fix to `_cached_serialization` — do not propagate it from the wrapped object in `wrap()`.

### Proof of Concept

```python
from clvm_rs.clvm_storage import CLVMStorage
from clvm_rs.program import Program

# Attacker-controlled CLVMStorage object with forged tree hash
class ForgedNode:
    atom = b"real_data"
    pair = None
    _cached_sha256_treehash = b"\x00" * 32  # forged: all zeros

forged = ForgedNode()

# Program.wrap() copies _cached_sha256_treehash without verification
p = Program.wrap(forged)

# Program.tree_hash() returns the forged hash without recomputing
assert p.tree_hash() == b"\x00" * 32  # wrong: should be sha256(b"\x01" + b"real_data")

# Equality comparison is now broken
real = Program.new_atom(b"real_data")
# real.tree_hash() != b"\x00"*32, so p != real even though they represent the same atom
assert p != real  # wrong: they should be equal

# Puzzle hash derived from p is wrong — funds sent to this address are unspendable
wrong_puzzle_hash = p.tree_hash()
```

The forged hash propagates through `Program.wrap()` at [1](#0-0)  and is returned unchecked by `Program.tree_hash()` at [7](#0-6) , exactly mirroring how `exchangeRateStored()` returns a stale cached rate without accruing interest.
