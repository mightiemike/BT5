Audit Report

## Title
SwapAllowlistExtension Checks Router Address Instead of Original Caller, Allowing Any User to Bypass a Curated Pool's Swap Allowlist — (`metric-periphery/contracts/extensions/SwapAllowlistExtension.sol`)

## Summary

`SwapAllowlistExtension.beforeSwap` receives `sender` as the direct caller of `pool.swap`, which is the router address when users route through `MetricOmmSimpleRouter`. Because the router is the natural allowlisted entry point for permitted users, any non-allowlisted user can bypass the allowlist entirely by calling any of the router's public `exact*` functions. The extension never sees the original user's address.

## Finding Description

**Root cause — three confirmed code facts:**

1. `MetricOmmPool.swap` passes its own `msg.sender` as the `sender` argument to `_beforeSwap`: [1](#0-0) 

2. `SwapAllowlistExtension.beforeSwap` checks `allowedSwapper[msg.sender][sender]` — `msg.sender` here is the pool, `sender` is whoever called `pool.swap`: [2](#0-1) 

3. `MetricOmmSimpleRouter.exactInputSingle` calls `pool.swap` directly with no user-identity forwarding — the pool sees `msg.sender = router`: [3](#0-2) 

**Exploit flow:**

```
attacker → MetricOmmSimpleRouter.exactInputSingle(pool, ...)
              → pool.swap(recipient, ...) [msg.sender = router]
              → _beforeSwap(sender=router, ...)
              → SwapAllowlistExtension.beforeSwap(sender=router, ...)
                   checks: allowedSwapper[pool][router] == true  ← passes
                   (never checks attacker's address)
```

The router stores `msg.sender` only in transient storage for the payment callback (`_setNextCallbackContext(..., msg.sender, ...)`), but this value is never forwarded to the pool or the extension. There is no mechanism in the current code to propagate the original caller's identity to `beforeSwap`.

## Impact Explanation

A pool admin deploying `SwapAllowlistExtension` to restrict swaps to specific counterparties (KYC'd addresses, whitelisted market makers, protocol bots) loses that restriction entirely for any user routing through the public router. Non-allowlisted users can execute real swaps, move pool reserves, and extract output tokens at oracle-derived prices. This is a direct, silent bypass of the curation policy enabling unauthorized fund flows out of the pool.

## Likelihood Explanation

- `MetricOmmSimpleRouter` is the standard user-facing entry point; pool admins will naturally allowlist it so that permitted users can trade through the normal UI/SDK path.
- No special privileges, flash loans, or multi-step setup are required — a single `exactInputSingle` call from any EOA suffices.
- The bypass is silent: the extension sees `allowedSwapper[pool][router] == true` and emits no anomalous event.
- All four router entry points (`exactInputSingle`, `exactInput`, `exactOutputSingle`, `exactOutput`) are affected.

## Recommendation

The extension must check the original user, not the intermediary. The cleanest fix is to have the router encode the original `msg.sender` in `extensionData` and have `SwapAllowlistExtension.beforeSwap` decode and verify it when `sender` matches a registered trusted router address. Alternatively, maintain a registry of trusted routers in the extension; when `sender` is a trusted router, decode the real caller from `extensionData` and apply the allowlist check against that address instead.

## Proof of Concept

```solidity
// Setup: pool with SwapAllowlistExtension; only allowedUser is allowlisted.
// Router is allowlisted so that allowedUser can trade through it.
extension.setAllowedToSwap(pool, router, true);
extension.setAllowedToSwap(pool, allowedUser, true);
// attacker is NOT allowlisted

// Direct swap from attacker → reverts correctly
vm.prank(attacker);
pool.swap(attacker, true, 1000, 0, "", ""); // reverts NotAllowedToSwap ✓

// Router swap from attacker → bypasses allowlist
vm.prank(attacker);
router.exactInputSingle(ExactInputSingleParams({
    pool: pool, tokenIn: token0, tokenOut: token1,
    zeroForOne: true, amountIn: 1000, amountOutMinimum: 0,
    recipient: attacker, deadline: block.timestamp + 1,
    priceLimitX64: 0, extensionData: ""
}));
// pool.swap called with msg.sender=router
// extension checks allowedSwapper[pool][router] == true → passes
// attacker receives token1 output despite not being allowlisted ✗
```
