Audit Report

## Title
`SwapAllowlistExtension` checks router address instead of actual user for router-mediated swaps â€” (`metric-periphery/contracts/extensions/SwapAllowlistExtension.sol`)

## Summary
`SwapAllowlistExtension.beforeSwap` gates on the `sender` argument, which is `msg.sender` of the pool's `swap()` call. When swaps are routed through `MetricOmmSimpleRouter`, `msg.sender` to the pool is the router contract, not the actual user. This causes the extension to check `allowedSwapper[pool][router]` instead of `allowedSwapper[pool][actual_user]`, breaking per-user access control for all router-mediated swaps.

## Finding Description
In `MetricOmmPool.swap()`, `msg.sender` is passed as `sender` to `_beforeSwap`: [1](#0-0) 

`ExtensionCalling._beforeSwap` forwards `sender` unchanged to the extension: [2](#0-1) 

`SwapAllowlistExtension.beforeSwap` then gates on `sender` (the router): [3](#0-2) 

In `MetricOmmSimpleRouter.exactInputSingle`, the actual user (`msg.sender`) is stored only in transient callback context for token payment; the pool's `swap()` call receives no user identity: [4](#0-3) 

This creates two failure modes:
1. **Allowlisted users blocked**: If Alice is allowlisted but the router is not, Alice's router swap reverts because `allowedSwapper[pool][router]` is false.
2. **Complete bypass**: If the admin allowlists the router (to let vetted users use the periphery), any non-allowlisted user can call `router.exactInputSingle(...)` and pass the check, since `allowedSwapper[pool][router] = true` is satisfied for all callers.

By contrast, `DepositAllowlistExtension.beforeAddLiquidity` correctly gates on `owner` (the actual position beneficiary), not `sender`: [5](#0-4) 

## Impact Explanation
A pool admin who configures `SwapAllowlistExtension` to restrict trading to a vetted set of counterparties loses that guarantee entirely when the router is allowlisted. Any unprivileged user can call `router.exactInputSingle(...)` and bypass the per-user allowlist, trading on a pool explicitly restricted to approved counterparties. This constitutes broken core pool functionality (access control nullification) and directly undermines the pool's curation guarantee â€” a concrete, fund-impacting impact since non-vetted counterparties can now drain liquidity from a restricted pool.

## Likelihood Explanation
The trigger requires the pool admin to allowlist the router â€” a natural and expected action when vetted users need to use the standard periphery. The admin has no reason to suspect this opens the pool to everyone, especially since the analogous `DepositAllowlistExtension` handles the operator pattern correctly. The inconsistency makes the mistake easy to overlook. Once the router is allowlisted, the bypass is unconditional and repeatable by any unprivileged user with no special capability required.

## Recommendation
The `beforeSwap` hook receives `sender` (the router) and `recipient` (the output address), neither of which reliably identifies the actual user. The correct fix is to pass the original caller's address through `extensionData` and have the extension decode it, or add a dedicated `swapper` field to the `beforeSwap` signature that the router populates explicitly. As a minimal mitigation, document that `SwapAllowlistExtension` is incompatible with router-mediated swaps and must only be used with direct pool calls.

## Proof of Concept
```solidity
// Setup: pool with SwapAllowlistExtension; only alice is allowlisted.
swapExt.setAllowedToSwap(address(pool), alice, true);

// Admin allowlists the router so alice can use the periphery:
swapExt.setAllowedToSwap(address(pool), address(router), true);

// Bob (not allowlisted) swaps through the router:
vm.prank(bob);
router.exactInputSingle(IMetricOmmSimpleRouter.ExactInputSingleParams({
    pool: address(pool),
    tokenIn: address(token0),
    tokenOut: address(token1),
    zeroForOne: true,
    amountIn: 1000,
    amountOutMinimum: 0,
    recipient: bob,
    deadline: block.timestamp + 1,
    priceLimitX64: 0,
    extensionData: ""
}));
// Succeeds: pool calls _beforeSwap(msg.sender=router, ...),
// extension finds allowedSwapper[pool][router] = true,
// Bob's swap executes despite never being allowlisted.
```
