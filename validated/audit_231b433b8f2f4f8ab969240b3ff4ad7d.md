### Title
Protobuf `ValidResourceBounds` Deserialization Silently Downgrades `AllResources` to `L1Gas`, Causing P2P Peers to Reject Valid Transactions - (File: `crates/apollo_protobuf/src/converters/transaction.rs`)

### Summary

When a V3 transaction carries `AllResourceBounds` with `l2_gas = 0` and `l1_data_gas = 0`, the gateway accepts it and computes its hash using the `AllResources` variant (which includes `l1_data_gas` in the hash preimage). When that transaction is serialized to protobuf and deserialized by a P2P peer, the peer's `TryFrom<protobuf::ResourceBounds> for ValidResourceBounds` silently downgrades the bounds to `ValidResourceBounds::L1Gas`. The subsequent `TryFrom<InvokeTransactionV3> for RpcInvokeTransactionV3` then hard-rejects the transaction with `DEPRECATED_RESOURCE_BOUNDS_ERROR`, so the transaction is never admitted to the peer's mempool.

### Finding Description

**Step 1 – Gateway accepts the transaction.**

The gateway's stateless validator accepts `RpcInvokeTransactionV3` with `AllResourceBounds { l1_gas: X, l2_gas: 0, l1_data_gas: 0 }`. The test suite explicitly covers this case: [1](#0-0) 

**Step 2 – Hash is computed with the `AllResources` variant.**

`InternalRpcInvokeTransactionV3` always stores `AllResourceBounds` and always exposes it as `ValidResourceBounds::AllResources`: [2](#0-1) 

`get_tip_resource_bounds_hash` includes `l1_data_gas` in the hash preimage **only** for `AllResources`: [3](#0-2) 

So the stored `tx_hash` covers three resource-bound felts (L1, L2, L1-data).

**Step 3 – Protobuf serialization preserves all three gas fields.**

`From<ValidResourceBounds> for protobuf::ResourceBounds` always emits all three fields, even when they are zero: [4](#0-3) 

**Step 4 – Protobuf deserialization silently downgrades to `L1Gas`.**

`TryFrom<protobuf::ResourceBounds> for ValidResourceBounds` returns `ValidResourceBounds::L1Gas` whenever `l2_gas.is_zero() && l1_data_gas.is_zero()`: [5](#0-4) 

**Step 5 – Conversion to `RpcInvokeTransactionV3` hard-fails.**

`TryFrom<InvokeTransactionV3> for RpcInvokeTransactionV3` rejects any variant that is not `AllResources`: [6](#0-5) 

The P2P receive path maps this failure to `DEPRECATED_RESOURCE_BOUNDS_ERROR` and drops the transaction: [7](#0-6) 

### Impact Explanation

A valid transaction accepted by the originating gateway node is permanently rejected by every P2P peer that receives it. The transaction can never propagate beyond the first hop, so it cannot be sequenced by any node that did not receive it directly from the user. This matches:

> **High. Mempool/gateway/RPC admission accepts invalid transactions or rejects valid transactions before sequencing.**

### Likelihood Explanation

Any user who submits a V3 invoke with only L1 gas set (l2 = 0, l1_data = 0) — a configuration the gateway explicitly validates as correct — triggers this path. The gateway's own test suite demonstrates this is a supported and expected input. No special privileges or malicious intent are required.

### Recommendation

In `TryFrom<protobuf::ResourceBounds> for ValidResourceBounds`, remove the heuristic that downgrades to `L1Gas` when l2 and l1_data are zero. Instead, always produce `AllResources` when all three fields are present in the protobuf message (regardless of their values), and reserve `L1Gas` only for legacy messages that genuinely lack the `l1_data_gas` field (i.e., `value.l1_data_gas.is_none()`):

```rust
Ok(if value.l1_data_gas.is_none() && l2_gas.is_zero() {
    ValidResourceBounds::L1Gas(l1_gas)
} else {
    ValidResourceBounds::AllResources(AllResourceBounds { l1_gas, l2_gas, l1_data_gas })
})
```

This preserves backward compatibility with pre-0.13.3 peers (which never send `l1_data_gas`) while correctly round-tripping modern `AllResources` transactions whose gas fields happen to be zero.

### Proof of Concept

1. Construct an `RpcInvokeTransactionV3` with `AllResourceBounds { l1_gas: ResourceBounds { max_amount: 1, max_price_per_unit: 1 }, l2_gas: ResourceBounds::default(), l1_data_gas: ResourceBounds::default() }`.
2. Submit to the gateway — it is accepted (as confirmed by the existing test at line 70–82 of `stateless_transaction_validator_test.rs`).
3. Serialize the resulting `RpcTransaction` to `protobuf::MempoolTransaction` via `From<RpcTransaction> for protobuf::MempoolTransaction`.
4. Deserialize on the receiving side via `TryFrom<protobuf::MempoolTransaction> for RpcTransaction`.
5. Observe that step 4 returns `Err(DEPRECATED_RESOURCE_BOUNDS_ERROR)` — the transaction is dropped.

The divergence is rooted in the `is_zero()` check at `crates/apollo_protobuf/src/converters/transaction.rs:431` selecting `L1Gas` for a transaction whose canonical hash was computed under `AllResources`. [8](#0-7)

impl TryFrom<protobuf::ResourceBounds> for ValidResourceBounds {
    type Error = ProtobufConversionError;
    fn try_from(value: protobuf::ResourceBounds) -> Result<Self, Self::Error> {
        let Some(l1_gas) = value.l1_gas else {
            return Err(missing("ResourceBounds::l1_gas"));
        };
        let Some(l2_gas) = value.l2_gas else {
            return Err(missing("ResourceBounds::l2_gas"));
        };
        // TODO(Shahak): Assert data gas is not none once we remove support for 0.13.2.
        let l1_data_gas = value.l1_data_gas.unwrap_or_default();
        let l1_gas: ResourceBounds = l1_gas.try_into()?;
        let l2_gas: ResourceBounds = l2_gas.try_into()?;
        let l1_data_gas: ResourceBounds = l1_data_gas.try_into()?;
        Ok(if l1_data_gas.is_zero() && l2_gas.is_zero() {
            ValidResourceBounds::L1Gas(l1_gas)
        } else {
            ValidResourceBounds::AllResources(AllResourceBounds { l1_gas, l2_gas, l1_data_gas })
        })
    }

impl TryFrom<protobuf::InvokeV3WithProof> for RpcInvokeTransactionV3 {
    type Error = ProtobufConversionError;
    fn try_from(mut value: protobuf::InvokeV3WithProof) -> Result<Self, Self::Error> {
        // Extract proof first, since `starknet_api::transaction::InvokeTransactionV3` does not
        // carry a `proof` field.
        let proof = Proof::from(std::mem::take(&mut value.proof));

        let snapi_invoke: InvokeTransactionV3 = value
            .invoke
            .ok_or(ProtobufConversionError::MissingField {
                field_description: "InvokeV3WithProof::invoke",
            })?
            .try_into()?;

        // This conversion can fail only if the resource_bounds are not AllResources.
        Ok(Self { proof, ..snapi_invoke.try_into().map_err(|_| DEPRECATED_RESOURCE_BOUNDS_ERROR)? })
    }
