Audit Report

## Title
SwapAllowlistExtension Checks Router Address Instead of End User, Allowing Full Allowlist Bypass via MetricOmmSimpleRouter - (File: metric-periphery/contracts/extensions/SwapAllowlistExtension.sol)

## Summary

`SwapAllowlistExtension.beforeSwap` receives the `sender` argument forwarded from `MetricOmmPool.swap`, which is `msg.sender` of the pool call — the router contract, not the end user. When a pool admin allowlists the router to support normal user flows, every unprivileged address on-chain can bypass the allowlist by routing through `MetricOmmSimpleRouter`, completely defeating the access-control invariant the extension is designed to enforce.

## Finding Description

`MetricOmmPool.swap` captures `msg.sender` and passes it as `sender` to `_beforeSwap`: [1](#0-0) 

`ExtensionCalling._beforeSwap` encodes that value verbatim into the extension call: [2](#0-1) 

`SwapAllowlistExtension.beforeSwap` then checks `allowedSwapper[msg.sender][sender]`, where `msg.sender` is the pool and `sender` is whoever called the pool: [3](#0-2) 

When a user calls `MetricOmmSimpleRouter.exactInputSingle`, the router calls `pool.swap(...)` directly: [4](#0-3) 

At that point, `msg.sender` inside the pool is the **router address**. The extension receives `sender = router` and checks `allowedSwapper[pool][router]`. The end user's address is never seen by the extension. There is no mechanism in the extension interface, the pool, or the router to propagate the originating user address through to the hook.

This creates an irresolvable dilemma: if the router is not allowlisted, legitimate allowlisted users cannot use the router; if the router is allowlisted, every address on-chain can bypass the allowlist via the router.

## Impact Explanation

A pool configured with `SwapAllowlistExtension` to restrict swaps to specific addresses (e.g., KYC-verified counterparties, institutional market makers, or whitelisted bots) is fully bypassed the moment the pool admin allowlists the router to support normal user flows. Any unprivileged address can call `exactInputSingle`, `exactInput`, or `exactOutputSingle` and execute swaps against the restricted pool. This constitutes broken core pool access-control functionality with direct LP asset exposure to the full public swap surface the admin intended to gate, satisfying the "Admin-boundary break bypassed by an unprivileged path" and "Broken core pool functionality causing loss of funds" impact criteria.

## Likelihood Explanation

The router is the canonical periphery swap interface; pool admins will routinely allowlist it to enable standard user flows. The bypass requires no special privileges, no flash loans, and no unusual token behavior — a plain `exactInputSingle` call suffices. Any deployment combining `SwapAllowlistExtension` with router support hits this automatically.

## Recommendation

Pass the originating user through the extension call rather than the immediate pool caller. Options include: (1) add an `originSender` field to the extension interface populated from a trusted router that forwards the real payer address (e.g., via `extensionData` with router attestation verified by the extension); (2) have `SwapAllowlistExtension` explicitly revert when called by a known router address, documenting incompatibility; or (3) perform the allowlist check in the router before the pool call, with the pool verifying the router's attestation via signed `extensionData`.

## Proof of Concept

1. Deploy a pool with `SwapAllowlistExtension` attached as a before-swap hook.
2. Pool admin calls `setAllowedToSwap(pool, alice, true)` — only Alice is intended to swap.
3. Pool admin calls `setAllowedToSwap(pool, router, true)` — router allowlisted so Alice can use it.
4. Bob (not allowlisted) calls `MetricOmmSimpleRouter.exactInputSingle({pool: pool, ...})`.
5. The router calls `pool.swap(recipient, ...)` with `msg.sender = router` inside the pool.
6. `_beforeSwap` forwards `sender = router` to `SwapAllowlistExtension.beforeSwap`.
7. The extension checks `allowedSwapper[pool][router] == true` → passes.
8. Bob's swap executes against the restricted pool with no revert, bypassing the allowlist entirely.
