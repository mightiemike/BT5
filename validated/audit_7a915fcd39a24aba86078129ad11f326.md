Audit Report

## Title
Router address substitutes for actual swapper in `SwapAllowlistExtension::beforeSwap`, enabling allowlist bypass — (`metric-periphery/contracts/extensions/SwapAllowlistExtension.sol`)

## Summary
`MetricOmmPool.swap` passes `msg.sender` (the immediate caller) as the `sender` argument to `_beforeSwap` and onward to every configured extension. When `MetricOmmSimpleRouter` intermediates the call, `sender` received by `SwapAllowlistExtension.beforeSwap` is the router address, not the end user. A pool admin who allowlists the router to permit router-mediated swaps inadvertently opens the gate to every user, fully voiding per-user access control.

## Finding Description
`MetricOmmPool.swap` is decorated with `whenNotPaused` and passes `msg.sender` directly as the first argument to `_beforeSwap`: [1](#0-0) 

`ExtensionCalling._beforeSwap` forwards that value unchanged as `sender` to every extension in `BEFORE_SWAP_ORDER`. `SwapAllowlistExtension.beforeSwap` then gates on `allowedSwapper[msg.sender][sender]`, where `msg.sender` is the pool and `sender` is whoever called `pool.swap()`: [2](#0-1) 

`MetricOmmSimpleRouter.exactInputSingle` calls `pool.swap()` directly with no mechanism to forward the original end-user address: [3](#0-2) 

The result is that `sender` inside the hook is always the router address when the router is used. The admin has no way to simultaneously (a) allow allowlisted users to use the router and (b) block non-allowlisted users from doing the same, because the router collapses all user identities into one address. Allowlisting the router is equivalent to setting `allowAllSwappers = true` for all router users.

The paused-pool angle is a dead end: `swap` carries `whenNotPaused` so it reverts before `_beforeSwap` is ever reached when the pool is paused: [4](#0-3) 

`removeLiquidity` has no `whenNotPaused` guard and calls `_beforeRemoveLiquidity`/`_afterRemoveLiquidity`, never `_beforeSwap`: [5](#0-4) 

There is no code path that reaches `SwapAllowlistExtension::beforeSwap` through `removeLiquidity` under any pause state.

## Impact Explanation
Broken core access-control functionality. The allowlist is the sole mechanism for restricting who may swap on a gated pool. Router mediation silently voids per-user restrictions, allowing any unprivileged user to trade on a pool that was intended to be restricted. This constitutes broken core pool functionality causing an unusable swap access-control flow, qualifying under the allowed impact gate.

## Likelihood Explanation
Medium. Requires a pool configured with `SwapAllowlistExtension` and the router allowlisted. Both conditions are expected in normal production use of the periphery — a pool admin who wants to permit router-mediated swaps for approved users must allowlist the router, which is the exact configuration that triggers the bypass.

## Recommendation
Pass the original end-user address through the router (e.g., via `extensionData` or a dedicated field in the swap call) and have the extension read it from there, or document that `MetricOmmSimpleRouter` is incompatible with `SwapAllowlistExtension` and enforce this incompatibility at the factory level during pool configuration.

## Proof of Concept
1. Deploy a pool with `SwapAllowlistExtension`; call `setAllowedToSwap(pool, router, true)`, leave all other addresses blocked.
2. Call `router.exactInputSingle(...)` as any non-allowlisted EOA.
3. The router calls `pool.swap()`; the pool passes `msg.sender = router` as `sender` to `_beforeSwap`; the hook finds `allowedSwapper[pool][router] = true` and returns success — the swap executes despite the caller never being individually approved.
4. Repeat with any number of non-allowlisted EOAs to confirm the bypass is universal.
