Audit Report

## Title
Router-Mediated Swaps Pass Router Address as `sender` to `SwapAllowlistExtension`, Breaking Per-User Allowlist Gate — (`metric-periphery/contracts/extensions/SwapAllowlistExtension.sol`)

## Summary

`MetricOmmPool.swap()` forwards `msg.sender` as the `sender` argument to `_beforeSwap()`. When a user swaps through `MetricOmmSimpleRouter`, `msg.sender` at the pool is the router contract address, not the originating user. `SwapAllowlistExtension.beforeSwap()` checks `allowedSwapper[pool][sender]` against the router address, not the user, making it impossible to enforce per-user allowlist restrictions for router-mediated swaps.

## Finding Description

**Step 1 — Pool passes `msg.sender` as `sender` to the hook:**

`MetricOmmPool.swap()` always passes `msg.sender` as the first argument to `_beforeSwap()`: [1](#0-0) 

**Step 2 — Router calls the pool directly; original user identity is lost:**

`exactInputSingle` calls `pool.swap()` directly. At the pool, `msg.sender` is the router contract, not the end user. The original `msg.sender` is only stored in transient storage for the payment callback, never forwarded to the pool's `sender` parameter: [2](#0-1) 

The same applies to `exactOutputSingle` and all multi-hop paths (`exactInput`, `exactOutput`): [3](#0-2) 

**Step 3 — The hook checks the router address, not the user:**

`SwapAllowlistExtension.beforeSwap` receives `sender` = router address and checks `allowedSwapper[msg.sender][sender]` (where `msg.sender` = pool, `sender` = router): [4](#0-3) 

The allowlist is keyed per individual address by the pool admin: [5](#0-4) 

**The dilemma is irreconcilable:**

| Router allowlist state | Effect |
|---|---|
| Router NOT allowlisted | All router swaps revert with `NotAllowedToSwap()`, even for individually allowlisted users |
| Router IS allowlisted | Every user bypasses the per-user allowlist by routing through the router |

## Impact Explanation

A pool deployer who configures `SwapAllowlistExtension` to restrict swaps to a curated set of addresses (e.g., KYC'd counterparties, institutional traders, or whitelisted market makers) cannot enforce that restriction for router-mediated swaps. Any unprivileged user can call `MetricOmmSimpleRouter.exactInputSingle()` on a restricted pool and bypass the allowlist entirely if the router is allowlisted, or cause all router-path swaps to fail for legitimate allowlisted users if it is not. This constitutes broken access-control core functionality — the `SwapAllowlistExtension`'s stated purpose of gating swaps by swapper address is unenforceable through the canonical router interface.

## Likelihood Explanation

`MetricOmmSimpleRouter` is the canonical public swap interface. Any pool using `SwapAllowlistExtension` and expecting users to interact via the router is affected. The bypass requires no special privileges — any user can call `exactInputSingle`. The condition is met whenever a pool admin enables the extension without also allowlisting the router (broken functionality) or does allowlist the router (bypass).

## Recommendation

The pool should propagate the original user's identity to the hook. The preferred fix is for the router to encode the original `msg.sender` into `extensionData`, and for `SwapAllowlistExtension.beforeSwap` to decode and check that value instead of the `sender` argument. Alternatively, the pool's `swap` interface could accept an explicit `swapper` parameter distinct from `msg.sender` (the callback payer), analogous to how `addLiquidity` separates payer from owner.

## Proof of Concept

```
1. Pool admin deploys pool with SwapAllowlistExtension configured.
2. Pool admin calls setAllowedToSwap(pool, alice, true).
   → allowedSwapper[pool][alice] = true
3. Pool admin calls setAllowedToSwap(pool, router, true)
   (required for any router swap to succeed).
   → allowedSwapper[pool][router] = true
4. Bob (not allowlisted) calls MetricOmmSimpleRouter.exactInputSingle(pool, ...).
   → Router calls pool.swap(...); at the pool, msg.sender = router.
   → _beforeSwap(sender=router, ...) is called.
   → beforeSwap checks allowedSwapper[pool][router] == true → PASSES.
5. Bob successfully swaps on a pool restricted to Alice only.
```
