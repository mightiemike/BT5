### Title
Missing Access Control on `run_mainchain_gc` Allows Any Caller to Permanently Delete Mainchain Headers - (File: contract/src/lib.rs)

---

### Summary

`run_mainchain_gc` is a public, state-mutating NEAR contract function that permanently removes block headers from the light client's mainchain index. It carries only a `#[pause]` decorator, which gates execution when the contract is paused but imposes no role restriction on unprivileged callers when the contract is live. Any NEAR account can call it with an arbitrarily large `batch_size`, immediately purging all headers that exceed `gc_threshold` and permanently advancing `mainchain_initial_blockhash`.

---

### Finding Description

`submit_blocks` â€” the intended and only legitimate trigger for GC â€” is correctly gated behind `#[trusted_relayer]`, which enforces that only staked, role-bearing relayers may submit headers. [1](#0-0) 

`run_mainchain_gc`, however, is exposed as a standalone public entry point with no equivalent role check: [2](#0-1) 

The `#[pause(except(roles(Role::UnrestrictedRunGC)))]` attribute only controls whether the function is callable while the contract is paused. When the contract is unpaused â€” its normal production state â€” the attribute is a no-op with respect to caller identity. No `acl_role_check`, no `#[access_control_any]`, and no `require!(env::predecessor_account_id() == ...)` guard exists.

Inside the function, the attacker-controlled `batch_size` is clamped only by the number of headers currently eligible for removal (`amount_of_headers_we_store - gc_threshold`): [3](#0-2) 

Passing `u64::MAX` as `batch_size` causes the function to remove every header beyond `gc_threshold` in a single call. For each removed height, `remove_block_header` deletes the entry from both `mainchain_header_to_height` and `headers_pool`, and `mainchain_height_to_header` is also cleared. Finally, `mainchain_initial_blockhash` is permanently advanced to the new oldest block. [4](#0-3) 

---

### Impact Explanation

Deleted headers are gone from all three storage maps. `verify_transaction_inclusion` and `verify_transaction_inclusion_v2` both look up the target block in `mainchain_header_to_height` and panic with `"block does not belong to the current main chain"` if the entry is absent: [5](#0-4) 

Any downstream NEAR contract or bridge that relies on the light client to verify Bitcoin transaction inclusion for blocks that were purged will receive a permanent, unrecoverable failure. The deleted headers cannot be re-inserted through the normal `submit_blocks` path because the relayer only submits new chain tips; re-promoting old fork headers to mainchain requires a chain reorg, which is not possible for already-finalized Bitcoin history. The canonical mapping (`height â†’ hash`, `hash â†’ height`) is irreversibly corrupted for the purged range.

---

### Likelihood Explanation

The entry point is a standard NEAR cross-contract call requiring no deposit, no special key, and no staking. Any NEAR account can construct and submit the transaction. The contract is deployed on a public network, so the attack surface is fully open. The only precondition is that `amount_of_headers_we_store > gc_threshold`, which is the normal steady-state of a live light client that has been running for more than `gc_threshold` blocks.

---

### Recommendation

Add a role-based access control guard to `run_mainchain_gc` consistent with the pattern already used for `submit_blocks`. Define a dedicated role (e.g., `Role::GCManager`) or reuse `Role::DAO`/`Role::RelayerManager`, and apply `#[access_control_any(roles(Role::GCManager, Role::DAO))]` (or the equivalent `near_plugins` attribute) above the function signature, in addition to the existing `#[pause]` decorator. The internal call from `submit_blocks` will continue to work because it is a self-call within the same contract execution context.

---

### Proof of Concept

1. Deploy the contract and initialize it with a genesis block and enough subsequent blocks so that `amount_of_headers_we_store > gc_threshold` (e.g., `gc_threshold = 52704` and the chain has advanced past that).
2. From any unprivileged NEAR account (no role granted), call:
   ```
   run_mainchain_gc(batch_size: 18446744073709551615)
   ```
3. Observe that all headers from `mainchain_initial_blockhash` up to `tip_height - gc_threshold` are deleted from `headers_pool`, `mainchain_height_to_header`, and `mainchain_header_to_height`, and that `mainchain_initial_blockhash` is advanced.
4. Attempt `verify_transaction_inclusion` for any block in the purged range â€” the call panics with `"block does not belong to the current main chain"`, confirming permanent proof-verification failure for those blocks. [6](#0-5)

    #[pause(except(roles(Role::UnrestrictedRunGC)))]
    pub fn run_mainchain_gc(&mut self, batch_size: u64) {
        let initial_blockheader = self
            .headers_pool
            .get(&self.mainchain_initial_blockhash)
            .unwrap_or_else(|| env::panic_str(ERR_KEY_NOT_EXIST));

        let tip_blockheader = self
            .headers_pool
            .get(&self.mainchain_tip_blockhash)
            .unwrap_or_else(|| env::panic_str(ERR_KEY_NOT_EXIST));

        let amount_of_headers_we_store =
            tip_blockheader.block_height - initial_blockheader.block_height + 1;

        if amount_of_headers_we_store > self.gc_threshold {
            let total_amount_to_remove = amount_of_headers_we_store - self.gc_threshold;
            let selected_amount_to_remove = std::cmp::min(total_amount_to_remove, batch_size);

            let start_removal_height = initial_blockheader.block_height;
            let end_removal_height = initial_blockheader.block_height + selected_amount_to_remove;
            env::log_str(&format!(
                "Num of blocks to remove {selected_amount_to_remove}"
            ));

            for height in start_removal_height..end_removal_height {
                let blockhash = &self
                    .mainchain_height_to_header
                    .get(&height)
                    .unwrap_or_else(|| env::panic_str(ERR_KEY_NOT_EXIST));

                self.remove_block_header(blockhash);
                self.mainchain_height_to_header.remove(&height);
            }

            self.mainchain_initial_blockhash = self
                .mainchain_height_to_header
                .get(&end_removal_height)
                .unwrap_or_else(|| env::panic_str(ERR_KEY_NOT_EXIST));
        }
    }
