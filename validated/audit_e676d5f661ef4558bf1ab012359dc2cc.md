### Title
Unconditional `LastKnownBonded = true` After Redelegation to Unbonded Validator Enables Unauthorized Bonus Reward Extraction - (`x/tieredrewards/keeper/msg_server.go`)

---

### Summary

`MsgTierRedelegate` unconditionally sets `LastKnownBonded = true` on the position after redelegation, regardless of the destination validator's actual bonded state. Because `validateRedelegatePosition` imposes no check that the destination validator is bonded, and the Cosmos SDK staking module permits redelegation to unbonded validators, an attacker can redelegate to a currently-unbonded validator, receive an incorrect `LastKnownBonded = true` checkpoint, and then claim bonus rewards for the entire unbonded gap once the validator re-bonds — draining the shared rewards pool for a period during which no bonus should have accrued.

---

### Finding Description

In `TierRedelegate`, after the staking redelegation completes, the position's bonus checkpoint is updated:

```go
latestSeq, err := ms.getValidatorEventLatestSeq(ctx, dstValAddr)
...
pos.UpdateBonusCheckpoints(latestSeq, sdkCtx.BlockTime(), true)  // ← hardcoded true
``` [1](#0-0) 

The third argument `true` is the `lastKnownBonded` flag. It is hardcoded to `true` unconditionally, regardless of whether the destination validator is currently bonded or unbonded.

`validateRedelegatePosition` performs no check on the destination validator's bonded state:

```go
func (k Keeper) validateRedelegatePosition(ctx context.Context, pos types.PositionState, owner, dstValidator string) error {
    // checks: owner, delegated, same-validator, exit-elapsed, close-only, active-redelegation
    // NO check: destination validator must be bonded
    ...
}
``` [2](#0-1) 

At claim time, `processEventsAndClaimBonus` uses `pos.LastKnownBonded` as the starting bonded state for the first segment:

```go
bonded := pos.LastKnownBonded
segmentStart := pos.LastBonusAccrual
...
for _, entry := range events {
    if bonded {
        bonus := k.computeSegmentBonus(*pos, tier, segmentStart, evt.Timestamp, evt.TokensPerShare)
        totalBonus = totalBonus.Add(bonus)
    }
    ...
}
``` [3](#0-2) 

Because `LastEventSeq` is set to `latestSeq` (the seq of the destination validator's most recent event, which is the UNBOND event), the UNBOND event itself is excluded from replay (`StartExclusive(startSeq)`). The next event processed is the subsequent BOND event. The segment `[redelegation_time, bond_time]` is evaluated with `bonded = true` — but during this entire interval the validator was actually unbonded. [4](#0-3) 

The bonus formula `shares × tokensPerShare × bonusApy × durationSeconds / SecondsPerYear` is applied to this segment, paying out from the shared rewards pool for a period when no bonus should have accrued. [5](#0-4) 

---

### Impact Explanation

The shared `RewardsPoolName` module account is drained by bonus payments that correspond to a period when the position's validator was unbonded. The invariant stated in the ADR — *"Bonus rewards are earned only while delegated to a bonded validator"* — is violated. The corrupted value is the rewards pool balance: it is reduced by `shares × tokensPerShare × bonusApy × unbondedDuration / SecondsPerYear` per exploiting position, a quantity that can be significant for large positions or long unbonded gaps. [6](#0-5) 

---

### Likelihood Explanation

The attack requires only a standard `MsgTierRedelegate` transaction — no privileged keys, no governance access. The attacker needs:
1. An existing tier position (requires `amount >= tier.MinLockAmount`, e.g. 1,000,000 basecro).
2. A destination validator that is currently unbonded (jailed validators are common on live networks) and will eventually re-bond.

Both conditions are routinely satisfied on a live network. The Cosmos SDK `BeginRedelegation` does not require the destination validator to be bonded, so the staking layer will not reject the transaction. The attacker recovers their locked tokens on exit, making the net cost only the opportunity cost of the lock duration.

---

### Recommendation

In `TierRedelegate`, after obtaining the destination validator, read its actual bonded state and pass it to `UpdateBonusCheckpoints` instead of hardcoding `true`:

```go
dstVal, err := ms.stakingKeeper.GetValidator(ctx, dstValAddr)
if err != nil {
    return nil, err
}
pos.UpdateBonusCheckpoints(latestSeq, sdkCtx.BlockTime(), dstVal.IsBonded())
```

Additionally, `validateRedelegatePosition` should reject redelegation to a non-bonded destination validator to prevent positions from being parked on validators that cannot earn rewards:

```go
dstVal, err := k.stakingKeeper.GetValidator(ctx, dstValAddr)
if err != nil || !dstVal.IsBonded() {
    return types.ErrValidatorNotBonded
}
``` [1](#0-0) [2](#0-1) 

---

### Proof of Concept

1. Alice locks `MinLockAmount` basecro into tier 1 on bonded validator V1. Position is created with `LastKnownBonded = true`.
2. Validator V2 is jailed and transitions to unbonded. Its `ValidatorEventSeq` is now at seq=5 (the UNBOND event).
3. Alice submits `MsgTierRedelegate` targeting V2.
   - `validateRedelegatePosition` passes — no bonded-state check on destination.
   - Staking module accepts the redelegation (Cosmos SDK does not require destination to be bonded).
   - `latestSeq = 5` (the UNBOND event seq for V2).
   - `pos.UpdateBonusCheckpoints(5, blockTime, true)` → `LastKnownBonded = true`, `LastEventSeq = 5`.
4. V2 is unjailed 30 days later. `AfterValidatorBonded` fires, recording BOND event at seq=6.
5. Alice submits `MsgClaimTierRewards`.
   - `processEventsAndClaimBonus` starts with `bonded = true`.
   - Events since seq=5: only seq=6 (BOND).
   - Segment `[redelegation_time, bond_time]` (30 days): `bonded = true` → bonus computed and paid from pool.
   - Correct behavior: zero bonus for this segment (validator was unbonded the entire time).
6. Alice receives 30 days of bonus rewards she was not entitled to, draining the shared rewards pool. [7](#0-6) [8](#0-7)

func (k Keeper) validateRedelegatePosition(ctx context.Context, pos types.PositionState, owner, dstValidator string) error {
	if !pos.IsOwner(owner) {
		return types.ErrNotPositionOwner
	}

	if !pos.IsDelegated() {
		return types.ErrPositionNotDelegated
	}

	if pos.Delegation.ValidatorAddress == dstValidator {
		return types.ErrRedelegationToSameValidator
	}

	sdkCtx := sdk.UnwrapSDKContext(ctx)
	if pos.HasTriggeredExit() && pos.CompletedExitLockDuration(sdkCtx.BlockTime()) {
		return types.ErrExitLockDurationElapsed
	}

	tier, err := k.getTier(ctx, pos.TierId)
	if err != nil {
		return err
	}

	if tier.IsCloseOnly() {
		return types.ErrTierIsCloseOnly
	}

	isRedelegating, err := k.isRedelegating(ctx, pos.DelegatorAddress)
	if err != nil {
		return err
	}
	if isRedelegating {
		return errorsmod.Wrapf(types.ErrActiveRedelegation, "position %d has an active redelegation", pos.Id)
	}

	return nil
}

func (k Keeper) getValidatorEventsSince(ctx context.Context, valAddr sdk.ValAddress, startSeq uint64) ([]EventEntry, error) {
	// Range from (valAddr, startSeq+1) to end of valAddr prefix.
	rng := collections.NewPrefixedPairRange[sdk.ValAddress, uint64](valAddr).
		StartExclusive(startSeq)

	var entries []EventEntry
	err := k.ValidatorEvents.Walk(ctx, rng, func(key collections.Pair[sdk.ValAddress, uint64], event types.ValidatorEvent) (bool, error) {
		entries = append(entries, EventEntry{Seq: key.K2(), Event: event})
		return false, nil
	})
	if err != nil {
		return nil, err
	}
	return entries, nil
}

```go
func (k Keeper) computeSegmentBonus(pos types.PositionState, tier types.Tier, segmentStart, segmentEnd time.Time, tokensPerShare math.LegacyDec) math.Int {
	if !pos.ExitUnlockAt.IsZero() && segmentEnd.After(pos.ExitUnlockAt) {
		segmentEnd = pos.ExitUnlockAt
	}

	if !segmentEnd.After(segmentStart) {
		return math.ZeroInt()
	}

	durationSeconds := int64(segmentEnd.Sub(segmentStart) / time.Second)
	if durationSeconds <= 0 {
		return math.ZeroInt()
	}

	tokens := pos.Delegation.Shares.Mul(tokensPerShare)

	return tokens.
		Mul(tier.BonusApy).
		MulInt64(durationSeconds).
		QuoInt64(types.SecondsPerYear).
		TruncateInt()
}
