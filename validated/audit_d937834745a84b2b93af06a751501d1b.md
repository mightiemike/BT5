### Title
`burnNlp()` Permits Quote Collateral Extraction Under Initial-Health Threshold, Bypassing Protocol Risk Controls — (File: `core/contracts/Clearinghouse.sol`)

---

### Summary

The `burnNlp()` function in `Clearinghouse.sol` enforces only a **MAINTENANCE** health check after burning NLP tokens and crediting quote to the caller. Every other collateral-reducing path in the protocol (`withdrawCollateral()`, `transferQuote()`, `mintNlp()`) enforces an **INITIAL** health check. This asymmetry lets a user whose initial health is already negative — but whose maintenance health is still non-negative — burn unlocked NLP tokens and receive quote, effectively extracting collateral from the protocol in a state where the protocol's own risk model forbids it.

---

### Finding Description

`withdrawCollateral()` enforces:

```solidity
IProductEngine.HealthType healthType = sender == X_ACCOUNT
    ? IProductEngine.HealthType.PNL
    : IProductEngine.HealthType.INITIAL;

require(getHealth(sender, healthType) >= 0, ERR_SUBACCT_HEALTH);
``` [1](#0-0) 

`mintNlp()` also enforces INITIAL health after minting:

```solidity
require(
    getHealth(txn.sender, IProductEngine.HealthType.INITIAL) >= 0,
    ERR_SUBACCT_HEALTH
);
``` [2](#0-1) 

`burnNlp()`, however, only enforces MAINTENANCE health:

```solidity
// Burning NLP can decrease health if the burn fee exceeds the health improvement
// from the withdrawal. This check prevents malicious actors from deliberately
// creating unhealthy subaccounts through NLP burns.
require(
    getHealth(txn.sender, IProductEngine.HealthType.MAINTENANCE) >= 0,
    ERR_SUBACCT_HEALTH
);
``` [3](#0-2) 

Before that check, `burnNlp()` unconditionally credits quote to the caller:

```solidity
if (quoteAmount > 0) {
    spotEngine.updateBalance(QUOTE_PRODUCT_ID, txn.sender, quoteAmount);
    _applyNlpRebalance(spotEngine, nlpPools, nlpPoolRebalanceX18);
}
``` [4](#0-3) 

The INITIAL health threshold is always stricter than MAINTENANCE. A subaccount can be in the band `MAINTENANCE >= 0 > INITIAL` — meaning it is over-leveraged at the initial level (no new positions or withdrawals allowed) but not yet liquidatable. In this band, `withdrawCollateral()` reverts, but `burnNlp()` succeeds, because the post-burn MAINTENANCE check passes. The net effect is identical to a collateral withdrawal: the caller's quote balance increases and the NLP pool's quote balance decreases.

---

### Impact Explanation

A user holding unlocked NLP tokens whose subaccount is in the `MAINTENANCE >= 0 > INITIAL` health band can:

1. Burn NLP tokens and receive quote.
2. Increase their quote balance while their initial health is already negative.
3. Effectively withdraw collateral in a state where `withdrawCollateral()` would revert.

This breaks the protocol's core invariant that no collateral-reducing action is permitted when initial health is negative. The extracted quote remains in the subaccount (improving maintenance health just enough to pass the post-burn check), but the NLP pool's backing is permanently reduced. If the subaccount is subsequently liquidated, the protocol absorbs a larger shortfall than it would have had the burn been blocked.

---

### Likelihood Explanation

The condition is reachable by any user who:
- Holds unlocked NLP tokens (past the `NLP_LOCK_PERIOD`), and
- Has opened leveraged positions that have moved against them enough to push initial health below zero while maintenance health remains non-negative.

This is a normal market scenario (e.g., a position that has lost ~initial-margin but not yet ~maintenance-margin). The user submits a signed `BurnNlp` transaction; the sequencer processes it via `submitTransactionsChecked()` → `processTransaction()` → `clearinghouse.burnNlp()`. The contract itself does not block it. No privileged access is required beyond holding unlocked NLP.

---

### Recommendation

Replace the MAINTENANCE health check in `burnNlp()` with an INITIAL health check, consistent with every other collateral-reducing path:

```solidity
// Before (vulnerable):
require(
    getHealth(txn.sender, IProductEngine.HealthType.MAINTENANCE) >= 0,
    ERR_SUBACCT_HEALTH
);

// After (fixed):
require(
    getHealth(txn.sender, IProductEngine.HealthType.INITIAL) >= 0,
    ERR_SUBACCT_HEALTH
);
``` [3](#0-2) 

---

### Proof of Concept

**Setup:**
- User mints 1,000 NLP at price 1.0 → receives 1,000 NLP, pays 1,000 USDC quote.
- User opens a leveraged perp long. Position moves against user.
- After price move: `getHealth(user, INITIAL) = -50`, `getHealth(user, MAINTENANCE) = +30`.

**Exploit:**
1. `withdrawCollateral()` → reverts (`ERR_SUBACCT_HEALTH`, INITIAL < 0). ✓ blocked.
2. User submits `BurnNlp` for 100 NLP at oracle price 1.0 → `quoteAmount ≈ 99.9` (after burn fee).
3. `burnNlp()` credits +99.9 USDC to user's quote balance.
4. Post-burn MAINTENANCE health check: `getHealth(user, MAINTENANCE) = +30 - small_delta >= 0` → passes.
5. User has extracted ~99.9 USDC of collateral despite having negative initial health.

**State delta corrupted:** `QUOTE_PRODUCT_ID` balance of `txn.sender` increases; NLP pool quote decreases; initial health remains negative. The protocol's invariant — *no collateral extraction when initial health < 0* — is violated.
