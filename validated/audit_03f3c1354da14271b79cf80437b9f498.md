### Title
Silent Bonus Forfeit with Permanent Checkpoint Advance on Partial Redelegation Slash with Insufficient Pool — (`x/tieredrewards/keeper/slash.go`)

### Summary

In `slashRedelegationPosition`, when `processEventsAndClaimBonus` returns `ErrInsufficientBonusPool`, the error is deliberately swallowed to avoid a chain halt. However, by the time the error is returned, `processEventsAndClaimBonus` has already (1) decremented event reference counts in the KV store, (2) advanced `pos.LastBonusAccrual` to `blockTime`, (3) advanced `pos.LastEventSeq` past all pending events, and (4) updated `pos.LastKnownBonded`. For a **partial** slash, `setPositionWithState` then persists these advanced checkpoints. The bonus coins are never paid. The accrual window is permanently consumed — the user cannot retry.

### Finding Description

`processEventsAndClaimBonus` performs all state mutations before the pool balance check: [1](#0-0) 

The event reference count decrements happen inside the loop at line 196, before the pool check at line 230. The in-memory position checkpoints (`LastBonusAccrual`, `LastEventSeq`, `LastKnownBonded`) are all updated before the pool check at lines 215–217: [2](#0-1) 

In `slashRedelegationPosition`, when `ErrInsufficientBonusPool` is returned, the error is swallowed: [3](#0-2) 

For a **partial** slash, the function then calls `setPositionWithState` with the already-mutated `pos` (advanced checkpoints): [4](#0-3) 

The result: `pos.LastBonusAccrual` is persisted at `blockTime`, all pending events are consumed (ref counts decremented, events potentially GC'd), but the bonus coins are never transferred to the owner. The accrual window is permanently lost — there is no retry path.

This is structurally identical to the keep-tecdsa bug: a conditional return path (pool insufficient) leaves accounting state advanced while the corresponding value transfer is skipped.

### Impact Explanation

A position owner permanently loses all accrued bonus rewards that were computed at the time of the partial redelegation slash. The exact corrupted values are:

- `pos.LastBonusAccrual` — advanced to `blockTime`, consuming the entire accrual window
- `pos.LastEventSeq` — advanced past all pending events
- `pos.LastKnownBonded` — updated
- Event `ReferenceCount` entries — decremented in KV store (events may be garbage-collected)

None of these are rolled back. The bonus coins (`bonusCoins = shares × tokensPerShare × bonusApy × duration / SecondsPerYear`) are permanently forfeited from the owner's perspective, while the rewards pool retains them.

### Likelihood Explanation

The trigger requires two independently reachable conditions:

1. A tier position is redelegated (`MsgTierRedelegate`) and the destination validator is subsequently slashed — a normal staking event.
2. The `RewardsPoolName` module account has insufficient balance to cover the computed bonus at slash time.

The pool can be empty or near-empty at any time: it is funded externally and drained by every `MsgClaimTierRewards` call and every `BeginBlocker` top-up. An attacker who monitors pool balance and times a validator slash (or simply waits for a natural slash event when the pool is low) can trigger this. No privileged access is required.

### Recommendation

Move all KV-store mutations (event reference count decrements) and in-memory checkpoint updates to occur **after** the pool balance check succeeds. Alternatively, perform the pool balance check at the start of `processEventsAndClaimBonus` before any state is mutated, or use a cache-wrapped context that is only committed on success. The user-driven paths already handle this correctly by failing atomically; the `BeforeRedelegationSlashed` path must either match that atomicity or explicitly reset the position's checkpoints to their pre-call values when the pool is insufficient.

### Proof of Concept

1. Governance sets `TargetBaseRewardsRate` high enough to drain the pool quickly, or the pool is simply not replenished.
2. Alice calls `MsgLockTier` to create a tier position on validator V, then `MsgTierRedelegate` to validator V2. A `RedelegationMappings` entry is created.
3. Time passes; Alice accrues bonus rewards.
4. The rewards pool is drained to zero (by other claimants or BeginBlocker top-ups).
5. V2 is slashed (downtime or double-sign). The staking module calls `BeforeRedelegationSlashed` → `slashRedelegationPosition`.
6. `processEventsAndClaimBonus` walks all pending events, decrements their ref counts, advances `pos.LastBonusAccrual` to `blockTime`, then hits `sufficientBonusPoolBalance` → returns `ErrInsufficientBonusPool`.
7. `slashRedelegationPosition` swallows the error and calls `setPositionWithState` with the mutated `pos`.
8. Alice's `LastBonusAccrual` is now `blockTime`; all events are consumed. When Alice later calls `MsgClaimTierRewards` after the pool is replenished, `processEventsAndClaimBonus` finds no pending events and computes zero bonus for the already-consumed window. The bonus is permanently lost. [5](#0-4) [6](#0-5)

func (k Keeper) processEventsAndClaimBonus(ctx context.Context, pos *types.PositionState) (sdk.Coins, error) {
	// Rewards should have been claimed before undelegation
	if !pos.IsDelegated() {
		return sdk.NewCoins(), nil
	}

	valAddr, err := sdk.ValAddressFromBech32(pos.Delegation.ValidatorAddress)
	if err != nil {
		return nil, err
	}

	events, err := k.getValidatorEventsSince(ctx, valAddr, pos.LastEventSeq)
	if err != nil {
		return nil, err
	}

	sdkCtx := sdk.UnwrapSDKContext(ctx)
	blockTime := sdkCtx.BlockTime()

	totalBonus := math.ZeroInt()
	// Use the persisted bonded state from the last replay, not a hardcoded default.
	// This prevents overpaying bonus for unbonded gaps between claims.
	bonded := pos.LastKnownBonded
	segmentStart := pos.LastBonusAccrual

	tier, err := k.getTier(ctx, pos.TierId)
	if err != nil {
		return nil, err
	}

	for _, entry := range events {
		evt := entry.Event

		if bonded {
			// Compute bonus for the bonded segment [segmentStart, eventTime]
			// using the snapshot rate at the event.
			bonus := k.computeSegmentBonus(*pos, tier, segmentStart, evt.Timestamp, evt.TokensPerShare)
			totalBonus = totalBonus.Add(bonus)
		}

		// Update bonded state based on event type.
		switch evt.EventType {
		case types.ValidatorEventType_VALIDATOR_EVENT_TYPE_UNBOND:
			bonded = false
		case types.ValidatorEventType_VALIDATOR_EVENT_TYPE_BOND:
			bonded = true
		case types.ValidatorEventType_VALIDATOR_EVENT_TYPE_SLASH:
			// Slash doesn't change bonded state.
		}

		segmentStart = evt.Timestamp
		pos.UpdateLastEventSeq(entry.Seq)

		// Decrement reference count.
		if err := k.decrementEventRefCount(ctx, valAddr, entry.Seq); err != nil {
			return nil, err
		}
	}

	val, err := k.stakingKeeper.GetValidator(ctx, valAddr)
	if err != nil {
		return nil, err
	}
	// Defensive: validator bond status check
	if bonded && val.IsBonded() {
		currentRate, err := k.getTokensPerShare(ctx, valAddr)
		if err != nil {
			return nil, err
		}
		bonus := k.computeSegmentBonus(*pos, tier, segmentStart, blockTime, currentRate)
		totalBonus = totalBonus.Add(bonus)
	}

	applyBonusAccrualCheckpoint(&pos.Position, blockTime)
	// Persist the bonded state so the next replay starts correctly.
	pos.UpdateLastKnownBonded(bonded)

	if totalBonus.IsZero() {
		return sdk.NewCoins(), nil
	}

	bondDenom, err := k.stakingKeeper.BondDenom(ctx)
	if err != nil {
		return nil, err
	}

	bonusCoins := sdk.NewCoins(sdk.NewCoin(bondDenom, totalBonus))

	if err := k.sufficientBonusPoolBalance(ctx, bonusCoins); err != nil {
		return nil, err
	}

	ownerAddr, err := sdk.AccAddressFromBech32(pos.Owner)
	if err != nil {
		return nil, errorsmod.Wrap(sdkerrors.ErrInvalidAddress, "invalid owner address")
	}

	if err := k.bankKeeper.SendCoinsFromModuleToAccount(ctx, types.RewardsPoolName, ownerAddr, bonusCoins); err != nil {
		return nil, err
	}

	if err := sdkCtx.EventManager().EmitTypedEvent(&types.EventBonusRewardsClaimed{
		PositionId: pos.Id,
		Owner:      pos.Owner,
		Rewards:    bonusCoins,
	}); err != nil {
		return nil, err
	}

	return bonusCoins, nil
}

func (k Keeper) slashRedelegationPosition(ctx context.Context, unbondingId uint64, sharesToUnbond math.LegacyDec) error {
	positionId, err := k.getRedelegationMapping(ctx, unbondingId)
	if errors.Is(err, collections.ErrNotFound) {
		return nil
	}
	if err != nil {
		return err
	}

	pos, err := k.getPositionState(ctx, positionId)
	if errors.Is(err, types.ErrPositionNotFound) {
		k.logger(ctx).Error("position not found during redelegation slash",
			"position_id", positionId,
			"unbonding_id", unbondingId,
			"error", err.Error(),
		)
		return nil
	}
	if err != nil {
		return err
	}

	if !pos.IsDelegated() {
		// Defensive
		k.logger(ctx).Error("delegation missing during BeforeRedelegationSlashed",
			"position_id", positionId,
			"unbonding_id", unbondingId,
			"shares_to_unbond", sharesToUnbond.String(),
		)
		return nil
	}

	dstValStr := pos.Delegation.ValidatorAddress

	// Settle bonus against PRE-slash shares.
	if _, err := k.processEventsAndClaimBonus(ctx, &pos); err != nil {
		// Deliberately forgo bonus rewards if pool is insufficient to prevent chain halt.
		if errors.Is(err, types.ErrInsufficientBonusPool) {
			k.logger(ctx).Error("insufficient bonus pool during redelegation slash",
				"position_id", pos.Id,
				"error", err.Error(),
			)
		} else {
			return err
		}
	}

	fullSlash := sharesToUnbond.GTE(pos.Delegation.Shares)

	if fullSlash {
		pos.Delegation = nil
		pos.ClearBonusCheckpoints()
		return k.setPositionWithState(ctx, pos, &ValidatorTransition{PreviousAddress: dstValStr})
	}
	// In-memory only: the persisted Position carries no share count, and the
	// live delegation will reflect the post-Unbond shares on the next read.
	// Update the local copy so any follow-up logic in this call sees consistent state.
	pos.Delegation.Shares = pos.Delegation.Shares.Sub(sharesToUnbond)
	return k.setPositionWithState(ctx, pos, nil)
