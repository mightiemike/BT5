### Title
Protobuf `ValidResourceBounds` Round-Trip Silently Downgrades `AllResources` to `L1Gas`, Producing a Different Transaction Hash - (`crates/apollo_protobuf/src/converters/transaction.rs`)

### Summary

The `TryFrom<protobuf::ResourceBounds> for ValidResourceBounds` conversion in `transaction.rs` silently collapses `AllResources(l1_gas=X, l2_gas=0, l1_data_gas=0)` into `L1Gas(l1_gas=X)`. Because `get_tip_resource_bounds_hash` hashes a **different number of elements** for the two variants, the transaction hash computed before serialization diverges from the hash recomputed after deserialization. This is the direct sequencer analog of the `killGauge` invariant break: an aggregate representation is updated (the variant is downgraded) while the individual component value (`l1_data_gas`) is silently dropped from the hash preimage, producing an inconsistent result downstream.

---

### Finding Description

**Step 1 – The lossy protobuf conversion.**

`TryFrom<protobuf::ResourceBounds> for ValidResourceBounds` in `transaction.rs` applies the following classification rule:

```rust
Ok(if l1_data_gas.is_zero() && l2_gas.is_zero() {
    ValidResourceBounds::L1Gas(l1_gas)          // ← variant downgraded
} else {
    ValidResourceBounds::AllResources(AllResourceBounds { l1_gas, l2_gas, l1_data_gas })
})
``` [1](#0-0) 

The symmetric serializer (`From<ValidResourceBounds> for protobuf::ResourceBounds`) emits **identical bytes** for both `L1Gas(X)` and `AllResources(X, 0, 0)`: both produce `{l1_gas: X, l2_gas: 0, l1_data_gas: 0}`. [2](#0-1) 

The round-trip is therefore lossy: `AllResources(X, 0, 0)` → protobuf → `L1Gas(X)`.

---

**Step 2 – The hash preimage diverges.**

`get_tip_resource_bounds_hash` builds the poseidon preimage differently for the two variants:

```rust
// L1 and L2 gas bounds always exist.
let mut resource_felts = vec![
    get_concat_resource(&l1_resource_bounds, L1_GAS)?,
    get_concat_resource(&l2_resource_bounds, L2_GAS)?,
];

// For new V3 txs, need to also hash the data gas bounds.
resource_felts.extend(match resource_bounds {
    ValidResourceBounds::L1Gas(_) => vec![],                          // ← 3 elements total
    ValidResourceBounds::AllResources(all_resources) => {
        vec![get_concat_resource(&all_resources.l1_data_gas, L1_DATA_GAS)?]  // ← 4 elements total
    }
});
``` [3](#0-2) 

- `L1Gas(X)` → `poseidon_hash_many([tip, l1_gas_packed, l2_gas_packed=0])` — **3 elements**
- `AllResources(X, 0, 0)` → `poseidon_hash_many([tip, l1_gas_packed, l2_gas_packed=0, l1_data_gas_packed=0])` — **4 elements**

`poseidon_hash_many(3 elems) ≠ poseidon_hash_many(4 elems)` even when the 4th element is zero, because the Poseidon sponge absorbs the length implicitly through the number of rounds.

---

**Step 3 – The gateway accepts the triggering transaction.**

`RpcTransaction` always carries `AllResourceBounds` (never `ValidResourceBounds`), so the gateway computes the hash via the `AllResources` path. The stateless validator only rejects a transaction if:

1. `max_possible_fee(Tip::ZERO) == 0` — passes when `l1_gas.max_amount > 0` and `l1_gas.max_price_per_unit > 0`.
2. `l2_gas.max_price_per_unit < min_gas_price` — passes when `min_gas_price = 0` (the default in many configurations). [4](#0-3) 

A transaction with `AllResources(l1_gas={amount=1, price=1}, l2_gas=default=0, l1_data_gas=default=0)` therefore passes gateway admission and is included in a block with hash **H₁** (4-element preimage).

---

**Step 4 – Block-sync node recomputes a different hash.**

When a peer syncs the block via the protobuf sync protocol, it deserializes the transaction through `TryFrom<protobuf::ResourceBounds> for ValidResourceBounds`, obtaining `L1Gas(l1_gas={amount=1, price=1})`. Any subsequent hash recomputation (e.g., `validate_transaction_hash`) produces **H₂** (3-element preimage). H₁ ≠ H₂. [5](#0-4) 

The syncing node either rejects the block (chain split / sync stall) or stores the wrong hash, causing RPC responses to return an authoritative-looking but incorrect `transaction_hash`.

---

### Impact Explanation

**Transaction conversion / hash logic binds the wrong hash (High).** A user-submitted V3 transaction with zero `l2_gas` and `l1_data_gas` bounds is accepted by the gateway with hash H₁, but any node that reconstructs the transaction from protobuf recomputes hash H₂ ≠ H₁. This breaks hash-based transaction lookup, receipt matching, and block-hash integrity for any block containing such a transaction.

impl TryFrom<protobuf::ResourceBounds> for ValidResourceBounds {
    type Error = ProtobufConversionError;
    fn try_from(value: protobuf::ResourceBounds) -> Result<Self, Self::Error> {
        let Some(l1_gas) = value.l1_gas else {
            return Err(missing("ResourceBounds::l1_gas"));
        };
        let Some(l2_gas) = value.l2_gas else {
            return Err(missing("ResourceBounds::l2_gas"));
        };
        // TODO(Shahak): Assert data gas is not none once we remove support for 0.13.2.
        let l1_data_gas = value.l1_data_gas.unwrap_or_default();
        let l1_gas: ResourceBounds = l1_gas.try_into()?;
        let l2_gas: ResourceBounds = l2_gas.try_into()?;
        let l1_data_gas: ResourceBounds = l1_data_gas.try_into()?;
        Ok(if l1_data_gas.is_zero() && l2_gas.is_zero() {
            ValidResourceBounds::L1Gas(l1_gas)
        } else {
            ValidResourceBounds::AllResources(AllResourceBounds { l1_gas, l2_gas, l1_data_gas })
        })
    }

pub fn get_tip_resource_bounds_hash(
    resource_bounds: &ValidResourceBounds,
    tip: &Tip,
) -> Result<Felt, StarknetApiError> {
    let l1_resource_bounds = resource_bounds.get_l1_bounds();
    let l2_resource_bounds = resource_bounds.get_l2_bounds();

    // L1 and L2 gas bounds always exist.
    // Old V3 txs always have L2 gas bounds of zero, but they exist.
    let mut resource_felts = vec![
        get_concat_resource(&l1_resource_bounds, L1_GAS)?,
        get_concat_resource(&l2_resource_bounds, L2_GAS)?,
    ];

    // For new V3 txs, need to also hash the data gas bounds.
    resource_felts.extend(match resource_bounds {
        ValidResourceBounds::L1Gas(_) => vec![],
        ValidResourceBounds::AllResources(all_resources) => {
            vec![get_concat_resource(&all_resources.l1_data_gas, L1_DATA_GAS)?]
        }
    });

    Ok(HashChain::new().chain(&tip.0.into()).chain_iter(resource_felts.iter()).get_poseidon_hash())
}
