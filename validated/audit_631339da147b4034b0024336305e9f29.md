### Title
Protobuf `ValidResourceBounds` Deserialization Silently Downgrades `AllResources` to `L1Gas`, Producing a Different Transaction Hash - (`crates/apollo_protobuf/src/converters/transaction.rs`)

### Summary

The `TryFrom<protobuf::ResourceBounds> for ValidResourceBounds` implementation uses a zero-value heuristic to classify the deserialized type. When a transaction carrying `ValidResourceBounds::AllResources(l1_gas=X, l2_gas=0, l1_data_gas=0)` is serialized to protobuf and then deserialized, it silently becomes `ValidResourceBounds::L1Gas(l1_gas=X)`. Because `get_tip_resource_bounds_hash` hashes a different number of fields for each variant, the two representations produce **different Poseidon hashes** for the same logical transaction. Any node that receives this transaction over P2P block sync will compute a transaction hash that does not match the hash committed by the proposing node.

### Finding Description

**Serialization path (send side):**

`From<ValidResourceBounds> for protobuf::ResourceBounds` in `crates/apollo_protobuf/src/converters/transaction.rs` serializes `AllResources` with zero `l2_gas` and `l1_data_gas` as three present-but-zero fields:

```rust
ValidResourceBounds::AllResources(AllResourceBounds { l1_gas, l2_gas, l1_data_gas }) =>
    protobuf::ResourceBounds {
        l1_gas: Some(l1_gas.into()),
        l2_gas: Some(l2_gas.into()),        // zeros
        l1_data_gas: Some(l1_data_gas.into()), // zeros
    },
``` [1](#0-0) 

**Deserialization path (receive side):**

`TryFrom<protobuf::ResourceBounds> for ValidResourceBounds` applies a zero-value heuristic:

```rust
Ok(if l1_data_gas.is_zero() && l2_gas.is_zero() {
    ValidResourceBounds::L1Gas(l1_gas)          // ← type changed
} else {
    ValidResourceBounds::AllResources(...)
})
``` [2](#0-1) 

The round-trip is **not type-preserving**: `AllResources(X, 0, 0)` → protobuf → `L1Gas(X)`.

**Hash divergence:**

`get_tip_resource_bounds_hash` hashes a **different number of elements** depending on the variant:

- `L1Gas`: hashes `[tip, L1_GAS_packed, L2_GAS_packed(zeros)]` — **3 elements**
- `AllResources`: hashes `[tip, L1_GAS_packed, L2_GAS_packed(zeros), L1_DATA_GAS_packed(zeros)]` — **4 elements**

```rust
resource_felts.extend(match resource_bounds {
    ValidResourceBounds::L1Gas(_) => vec![],                          // no L1_DATA_GAS
    ValidResourceBounds::AllResources(all_resources) => {
        vec![get_concat_resource(&all_resources.l1_data_gas, L1_DATA_GAS)?]  // adds L1_DATA_GAS
    }
});
Ok(HashChain::new().chain(&tip.0.into()).chain_iter(resource_felts.iter()).get_poseidon_hash())
``` [3](#0-2) 

Because Poseidon is sensitive to the number of inputs, `hash([tip, L1, L2])` ≠ `hash([tip, L1, L2, L1_DATA])` even when `L2 = L1_DATA = 0`.

**Trigger path:**

A transaction with `AllResources(l1_gas=X, l2_gas=0, l1_data_gas=0)` can enter the system via the feeder gateway (central sync). The JSON deserializer `TryFrom<DeprecatedResourceBoundsMapping>` produces `AllResources` whenever the `L1_DATA_GAS` key is present in the JSON, regardless of its value:

```rust
match resource_bounds_mapping.0.get(&Resource::L1DataGas) {
    Some(data_bounds) => Ok(Self::AllResources(AllResourceBounds {
        l1_gas: *l1_bounds,
        l1_data_gas: *data_bounds,   // zero is accepted
        l2_gas: *l2_bounds,
    })),
``` [4](#0-3) 

The feeder gateway block data confirms that `L1_DATA_GAS` keys with zero values appear in real blocks: [5](#0-4) 

Once such a transaction is stored as `AllResources` (hash H1), it is serialized to protobuf for P2P block sync. The receiving node deserializes it as `L1Gas` and computes hash H2 ≠ H1.

The `InternalRpcTransactionWithoutTxHash::calculate_transaction_hash` and `InvokeTransactionV3::calculate_transaction_hash` both delegate to `get_invoke_transaction_v3_hash`, which calls `get_tip_resource_bounds_hash`: [6](#0-5) [7](#0-6) 

### Impact Explanation

A syncing node that receives a block over P2P containing an `AllResources(l1_gas=X, l2_gas=0, l1_data_gas=0)` transaction will deserialize it as `L1Gas(X)` and compute a transaction hash H2 that differs from the proposer's committed hash H1. This causes:

1. **Wrong transaction hash stored** in the syncing node's DB — the receipt, event, and state queries for that transaction will return results keyed under H2 instead of H1.
2. **Wrong block hash** — the block hash computation incorporates transaction hashes; a divergent transaction hash propagates to a divergent block hash.
3. **Permanent sync divergence** — the syncing node's view of the chain state permanently diverges from the canonical chain.

This matches the allowed impact: *"Wrong state, receipt, event, L1 message, class hash, storage value, or revert result from blockifier/syscall/execution logic for accepted input"* and *"Transaction conversion or signature/hash logic binds the wrong signer, hash, type, or executable payload."*

### Likelihood Explanation

The trigger requires a committed block to contain a V3 transaction where `L1_DATA_GAS` is present in the JSON but zero, and `L2_GAS` is also zero. The feeder gateway JSON deserializer produces `AllResources` whenever the `L1_DATA_GAS` key is present, so any such transaction from the feeder gateway enters the system as `AllResources`. The P2P block sync path then silently downgrades it. No privileged access is required; the condition arises from normal protocol operation.

### Recommendation

Replace the zero-value heuristic in `TryFrom<protobuf::ResourceBounds> for ValidResourceBounds` with an explicit discriminant field in the protobuf schema, or preserve the original variant by always deserializing to `AllResources` when all three fields are present (even if zero), matching the behavior of `TryFrom<DeprecatedResourceBoundsMapping>` when the `L1_DATA_GAS` key is present:

```rust
// Current (lossy):
Ok(if l1_data_gas.is_zero() && l2_gas.is_zero() {
    ValidResourceBounds::L1Gas(l1_gas)
} else {
    ValidResourceBounds::AllResources(...)
})

// Fixed: always AllResources when l1_data_gas field was present
Ok(ValidResourceBounds::AllResources(AllResourceBounds { l1_gas, l2_gas, l1_data_gas }))
// (L1Gas is only produced when l1_data_gas was absent/None in the protobuf message)
```

Alternatively, add a boolean `is_all_resources` discriminant to `protobuf::ResourceBounds` so the variant is encoded explicitly and the round-trip is lossless.

### Proof of Concept

```
1. Obtain a block from the feeder gateway containing a V3 invoke transaction
   where the JSON resource_bounds has:
     "L1_GAS":      { "max_amount": "0x1000", "max_price_per_unit": "0x1" }
     "L2_GAS":      { "max_amount": "0x0",    "max_price_per_unit": "0x0" }
     "L1_DATA_GAS": { "max_amount": "0x0",    "max_price_per_unit": "0x0" }

2. Deserialize via TryFrom<DeprecatedResourceBoundsMapping>:
   → ValidResourceBounds::AllResources(l1_gas=X, l2_gas=0, l1_data_gas=0)

3. Compute hash H1 = get_tip_resource_bounds_hash(AllResources(...), tip):
   → Poseidon([tip, pack(L1_GAS,X), pack(L2_GAS,0), pack(L1_DATA_GAS,0)])  // 4 elements

4. Serialize to protobuf via From<ValidResourceBounds> for protobuf::ResourceBounds:
   → { l1_gas: X, l2_gas: 0, l1_data_gas: 0 }  (all three fields present)

5. Deserialize via TryFrom<protobuf::ResourceBounds> for ValidResourceBounds:
   → l1_data_gas.is_zero() && l2_gas.is_zero() == true
   → ValidResourceBounds::L1Gas(l1_gas=X)        // type changed!

6. Compute hash H2 = get_tip_resource_bounds_hash(L1Gas(X), tip):
   → Poseidon([tip, pack(L1_GAS,X), pack(L2_GAS,0)])  // 3 elements

7. Assert H1 ≠ H2  ← hash mismatch; syncing node stores wrong transaction hash
```

impl TryFrom<protobuf::ResourceBounds> for ValidResourceBounds {
    type Error = ProtobufConversionError;
    fn try_from(value: protobuf::ResourceBounds) -> Result<Self, Self::Error> {
        let Some(l1_gas) = value.l1_gas else {
            return Err(missing("ResourceBounds::l1_gas"));
        };
        let Some(l2_gas) = value.l2_gas else {
            return Err(missing("ResourceBounds::l2_gas"));
        };
        // TODO(Shahak): Assert data gas is not none once we remove support for 0.13.2.
        let l1_data_gas = value.l1_data_gas.unwrap_or_default();
        let l1_gas: ResourceBounds = l1_gas.try_into()?;
        let l2_gas: ResourceBounds = l2_gas.try_into()?;
        let l1_data_gas: ResourceBounds = l1_data_gas.try_into()?;
        Ok(if l1_data_gas.is_zero() && l2_gas.is_zero() {
            ValidResourceBounds::L1Gas(l1_gas)
        } else {
            ValidResourceBounds::AllResources(AllResourceBounds { l1_gas, l2_gas, l1_data_gas })
        })
    }
}

pub fn get_tip_resource_bounds_hash(
    resource_bounds: &ValidResourceBounds,
    tip: &Tip,
) -> Result<Felt, StarknetApiError> {
    let l1_resource_bounds = resource_bounds.get_l1_bounds();
    let l2_resource_bounds = resource_bounds.get_l2_bounds();

    // L1 and L2 gas bounds always exist.
    // Old V3 txs always have L2 gas bounds of zero, but they exist.
    let mut resource_felts = vec![
        get_concat_resource(&l1_resource_bounds, L1_GAS)?,
        get_concat_resource(&l2_resource_bounds, L2_GAS)?,
    ];

    // For new V3 txs, need to also hash the data gas bounds.
    resource_felts.extend(match resource_bounds {
        ValidResourceBounds::L1Gas(_) => vec![],
        ValidResourceBounds::AllResources(all_resources) => {
            vec![get_concat_resource(&all_resources.l1_data_gas, L1_DATA_GAS)?]
        }
    });

    Ok(HashChain::new().chain(&tip.0.into()).chain_iter(resource_felts.iter()).get_poseidon_hash())
}

pub(crate) fn get_invoke_transaction_v3_hash<T: InvokeTransactionV3Trait>(
    transaction: &T,
    chain_id: &ChainId,
    transaction_version: &TransactionVersion,
) -> Result<TransactionHash, StarknetApiError> {
    let tip_resource_bounds_hash =
        get_tip_resource_bounds_hash(&transaction.resource_bounds(), transaction.tip())?;
    let paymaster_data_hash =
        HashChain::new().chain_iter(transaction.paymaster_data().0.iter()).get_poseidon_hash();
    let data_availability_mode = concat_data_availability_mode(
        transaction.nonce_data_availability_mode(),
        transaction.fee_data_availability_mode(),
    );
    let account_deployment_data_hash = HashChain::new()
        .chain_iter(transaction.account_deployment_data().0.iter())
        .get_poseidon_hash();
    let calldata_hash =
        HashChain::new().chain_iter(transaction.calldata().0.iter()).get_poseidon_hash();
    let mut hash_chain = HashChain::new()
        .chain(&INVOKE)
        .chain(&transaction_version.0)
        .chain(transaction.sender_address().0.key())
        .chain(&tip_resource_bounds_hash)
        .chain(&paymaster_data_hash)
        .chain(&Felt::try_from(chain_id)?)
        .chain(&transaction.nonce().0)
        .chain(&data_availability_mode)
        .chain(&account_deployment_data_hash)
        .chain(&calldata_hash);
    if !transaction.proof_facts().0.is_empty() {
        let proof_facts_hash =
            HashChain::new().chain_iter(transaction.proof_facts().0.iter()).get_poseidon_hash();
        hash_chain = hash_chain.chain(&proof_facts_hash);
    }
    Ok(TransactionHash(hash_chain.get_poseidon_hash()))
}

    type Error = StarknetApiError;
    fn try_from(
        resource_bounds_mapping: DeprecatedResourceBoundsMapping,

        if let (Some(l1_bounds), Some(l2_bounds)) = (
            resource_bounds_mapping.0.get(&Resource::L1Gas),
            resource_bounds_mapping.0.get(&Resource::L2Gas),

            match resource_bounds_mapping.0.get(&Resource::L1DataGas) {

                             {resource_bounds_mapping:?}",
                        )))
                    }
                }
            }
        } else {
            Err(StarknetApiError::InvalidResourceMappingInitializer(format!(
                "{resource_bounds_mapping:?}",
            )))
        }
    }
