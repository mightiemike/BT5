### Title
Silent Bonus Reward Forfeit with Permanent Checkpoint Advancement on Insufficient Pool During Redelegation Slash - (File: x/tieredrewards/keeper/slash.go)

### Summary

In `x/tieredrewards/keeper/slash.go`, the function `slashRedelegationPosition` deliberately swallows `ErrInsufficientBonusPool` from `processEventsAndClaimBonus`. However, `processEventsAndClaimBonus` advances the position's bonus checkpoints (`LastBonusAccrual`, `LastEventSeq`, `LastKnownBonded`) and decrements validator event reference counts **before** the pool balance check. When the error is swallowed and the position is subsequently persisted via `setPositionWithState`, the advanced checkpoints are written to state permanently — with no bonus ever paid. The accrued bonus is irrecoverably lost.

### Finding Description

`processEventsAndClaimBonus` walks pending validator events, computes `totalBonus`, and updates the in-memory position checkpoints before checking pool sufficiency: [1](#0-0) 

Specifically, `pos.UpdateLastEventSeq(entry.Seq)` is called inside the event loop, `decrementEventRefCount` writes to the store, and `applyBonusAccrualCheckpoint` / `UpdateLastKnownBonded` update the in-memory `pos` — all before `sufficientBonusPoolBalance` is checked: [2](#0-1) 

When `sufficientBonusPoolBalance` returns `ErrInsufficientBonusPool`, the error propagates back to `slashRedelegationPosition`, which explicitly swallows it: [3](#0-2) 

Execution then continues and calls `setPositionWithState` with the already-mutated `pos` (advanced checkpoints): [4](#0-3) 

Because no `CacheContext` wraps the `processEventsAndClaimBonus` call, the event reference count decrements written to the store inside the loop are also not rolled back. Events may be garbage-collected even though the bonus they represent was never paid.

### Impact Explanation

The position owner permanently loses all bonus rewards accrued since their last checkpoint. After `setPositionWithState` persists the advanced `LastBonusAccrual`, `LastEventSeq`, and `LastKnownBonded`, there is no mechanism to re-claim the forfeited segment — the module's double-claim prevention (`LastEventSeq` prevents event replay, `LastBonusAccrual` prevents segment replay) now treats the forfeited period as already settled. The corrupted value is the owner's `RewardsPool` entitlement for the accrual window `[old LastBonusAccrual, blockTime]`.

### Likelihood Explanation

The trigger requires two concurrent conditions: (1) the `RewardsPoolName` module account balance is below the position's accrued bonus at slash time, and (2) a redelegation slash fires via `BeforeRedelegationSlashed`. The pool can be depleted naturally over time (the `BeginBlocker` drains it for base-reward top-ups, as confirmed by `TestBeginBlocker_InsufficientPool`). A validator slash on a redelegating position is a normal protocol event. No privileged access is required; the position owner is a passive victim. [5](#0-4) 

### Recommendation

Wrap the `processEventsAndClaimBonus` call in `slashRedelegationPosition` with a `CacheContext` so that all store mutations (event ref-count decrements, in-memory checkpoint updates) are rolled back atomically when `ErrInsufficientBonusPool` is returned. Only commit the cache if the call succeeds. This preserves the chain-halt-avoidance intent while preventing permanent loss of the user's accrued bonus:

```go
cacheCtx, commit := sdk.UnwrapSDKContext(ctx).CacheContext()
if _, err := k.processEventsAndClaimBonus(cacheCtx, &pos); err != nil {
    if errors.Is(err, types.ErrInsufficientBonusPool) {
        k.logger(ctx).Error("insufficient bonus pool during redelegation slash", ...)
        // Do NOT advance checkpoints; user can retry on next claim.
    } else {
        return err
    }
} else {
    commit()
}
```

### Proof of Concept

1. A position owner calls `MsgTierRedelegate`, creating a redelegation entry tracked in `RedelegationMappings`.
2. The `RewardsPool` is drained to zero (e.g., by `BeginBlocker` top-ups over many blocks).
3. The destination validator is slashed; staking calls `SlashRedelegation` → `BeforeRedelegationSlashed` → `slashRedelegationPosition`.
4. `processEventsAndClaimBonus` walks events, advances `LastBonusAccrual` to `blockTime`, decrements event ref counts, then hits `sufficientBonusPoolBalance` → returns `ErrInsufficientBonusPool`.
5. `slashRedelegationPosition` swallows the error and calls `setPositionWithState` with the mutated `pos`.
6. The position is now persisted with `LastBonusAccrual = blockTime` and no bonus paid.
7. The owner later calls `MsgClaimTierRewards`; `processEventsAndClaimBonus` computes zero bonus for the already-advanced window. The accrued rewards are permanently gone. [6](#0-5) [7](#0-6) [8](#0-7)

func (k Keeper) slashRedelegationPosition(ctx context.Context, unbondingId uint64, sharesToUnbond math.LegacyDec) error {
	positionId, err := k.getRedelegationMapping(ctx, unbondingId)
	if errors.Is(err, collections.ErrNotFound) {
		return nil
	}
	if err != nil {
		return err
	}

	pos, err := k.getPositionState(ctx, positionId)
	if errors.Is(err, types.ErrPositionNotFound) {
		k.logger(ctx).Error("position not found during redelegation slash",
			"position_id", positionId,
			"unbonding_id", unbondingId,
			"error", err.Error(),
		)
		return nil
	}
	if err != nil {
		return err
	}

	if !pos.IsDelegated() {
		// Defensive
		k.logger(ctx).Error("delegation missing during BeforeRedelegationSlashed",
			"position_id", positionId,
			"unbonding_id", unbondingId,
			"shares_to_unbond", sharesToUnbond.String(),
		)
		return nil
	}

	dstValStr := pos.Delegation.ValidatorAddress

	// Settle bonus against PRE-slash shares.
	if _, err := k.processEventsAndClaimBonus(ctx, &pos); err != nil {
		// Deliberately forgo bonus rewards if pool is insufficient to prevent chain halt.
		if errors.Is(err, types.ErrInsufficientBonusPool) {
			k.logger(ctx).Error("insufficient bonus pool during redelegation slash",
				"position_id", pos.Id,
				"error", err.Error(),
			)
		} else {
			return err
		}
	}

	fullSlash := sharesToUnbond.GTE(pos.Delegation.Shares)

	if fullSlash {
		pos.Delegation = nil
		pos.ClearBonusCheckpoints()
		return k.setPositionWithState(ctx, pos, &ValidatorTransition{PreviousAddress: dstValStr})
	}
	// In-memory only: the persisted Position carries no share count, and the
	// live delegation will reflect the post-Unbond shares on the next read.
	// Update the local copy so any follow-up logic in this call sees consistent state.
	pos.Delegation.Shares = pos.Delegation.Shares.Sub(sharesToUnbond)
	return k.setPositionWithState(ctx, pos, nil)
