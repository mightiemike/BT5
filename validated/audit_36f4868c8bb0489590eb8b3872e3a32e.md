### Title
`ValidResourceBounds` Variant Collapse in Protobuf Deserialization Produces Wrong Transaction Hash for Synced Blocks - (`crates/apollo_protobuf/src/converters/transaction.rs`)

### Summary

The protobuf deserializer for `ValidResourceBounds` silently collapses `AllResources` into `L1Gas` whenever `l2_gas` and `l1_data_gas` are both zero. Because `get_tip_resource_bounds_hash` includes the `l1_data_gas` packed felt in the hash only for `AllResources`, the same transaction produces two different hashes depending on whether it is read from local storage (preserves `AllResources`) or reconstructed from a P2P protobuf sync message (produces `L1Gas`). A syncing node that recomputes or verifies the transaction hash from the deserialized body will compute a wrong value, breaking hash canonicalization across the network.

### Finding Description

`get_tip_resource_bounds_hash` in `crates/starknet_api/src/transaction_hash.rs` is variant-sensitive:

```rust
resource_felts.extend(match resource_bounds {
    ValidResourceBounds::L1Gas(_) => vec![],                                    // 2 entries total
    ValidResourceBounds::AllResources(all_resources) => {
        vec![get_concat_resource(&all_resources.l1_data_gas, L1_DATA_GAS)?]     // 3 entries total
    }
});
```

For `L1Gas` the poseidon hash covers `[tip, l1_gas_packed, l2_gas_packed]`. For `AllResources` it covers `[tip, l1_gas_packed, l2_gas_packed, l1_data_gas_packed]`. Even when `l2_gas` and `l1_data_gas` are numerically zero, the `AllResources` path appends a zero-packed felt (`0x4c315f444154415f47415300000000000000000000000000000000000000`), producing a strictly different poseidon output.

The protobuf deserializer in `crates/apollo_protobuf/src/converters/transaction.rs` destroys the variant:

```rust
Ok(if l1_data_gas.is_zero() && l2_gas.is_zero() {
    ValidResourceBounds::L1Gas(l1_gas)          // variant collapsed
} else {
    ValidResourceBounds::AllResources(...)
})
```

The originating node always submits through the gateway, which converts `RpcInvokeTransactionV3` (whose `resource_bounds` field is typed `AllResourceBounds`) into `InternalRpcInvokeTransactionV3`, and the `InvokeTransactionV3Trait` implementation for that type unconditionally returns `ValidResourceBounds::AllResources(self.resource_bounds)`. The hash `H_A` is therefore computed with the 3-entry path.

When a peer syncs the block via P2P, the transaction body travels as `protobuf::InvokeV3`. `TryFrom<protobuf::InvokeV3> for InvokeTransactionV3` calls `ValidResourceBounds::try_from(protobuf::ResourceBounds)`, which applies the zero-check and produces `ValidResourceBounds::L1Gas`. Any subsequent call to `calculate_transaction_hash` on the deserialized body computes `H_B` via the 2-entry path. `H_A ≠ H_B`.

### Impact Explanation

A syncing node that recomputes or validates the transaction hash from the deserialized `InvokeTransactionV3` body (e.g., `validate_transaction_hash`, block-hash verification, RPC `starknet_getTransactionByHash` hash cross-check, or re-execution) will obtain a wrong hash. This is a **Critical** wrong-state / wrong-hash result from blockifier/execution logic for accepted input, and a **High** RPC execution / tracing path returning an authoritative-looking wrong value.

### Likelihood Explanation

Any Invoke V3 transaction submitted with `AllResources` where both `l2_gas.max_amount == 0 && l2_gas.max_price_per_unit == 0` and `l1_data_gas.max_amount == 0 && l1_data_gas.max_price_per_unit == 0` triggers the collapse. The `AllResourceBounds::new_unlimited_gas_no_fee_enforcement()` helper sets `l1_gas` and `l1_data_gas` to `ResourceBounds { max_amount: 0, max_price_per_unit: 1 }` (non-zero price), so those are safe. However, any user-submitted transaction with zero L2 and L1DataGas bounds (a valid pre-0.13.3-style transaction submitted through the new `AllResources` path) hits this path. The condition is user-controllable and requires no privilege.

### Recommendation

1. **Fix the protobuf deserializer** to preserve the variant explicitly. Add a discriminant field to `protobuf::ResourceBounds` (e.g., a boolean `is_all_resources`) or always deserialize as `AllResources` when all three fields are present, regardless of zero values:

```rust
// In TryFrom<protobuf::ResourceBounds> for ValidResourceBounds
// Remove the zero-check; always produce AllResources when l1_data_gas is present.
Ok(if value.l1_data_gas.is_none() {
    ValidResourceBounds::L1Gas(l1_gas)
} else {
    ValidResourceBounds::AllResources(AllResourceBounds { l1_gas, l2_gas, l1_data_gas })
})
```

2. **Add a round-trip hash test** that serializes an `AllResources` transaction with zero L2/L1DataGas bounds to protobuf, deserializes it, and asserts the computed hash equals the original.

### Proof of Concept

```
1. Construct an InvokeTransactionV3 with:
     resource_bounds = AllResources { l1_gas: X, l2_gas: {0,0}, l1_data_gas: {0,0} }
   Hash H_A = get_invoke_transaction_v3_hash(tx, chain_id, version)
   → get_tip_resource_bounds_hash uses AllResources path → 3 entries → poseidon(tip, l1, l2, l1_data)

2. Serialize to protobuf::ResourceBounds:
     { l1_gas: X, l2_gas: {0,0}, l1_data_gas: {0,0} }

3. Deserialize via TryFrom<protobuf::ResourceBounds>:
     l1_data_gas.is_zero() && l2_gas.is_zero() == true
     → ValidResourceBounds::L1Gas(X)

4. Hash H_B = get_invoke_transaction_v3_hash(deserialized_tx, chain_id, version)
   → get_tip_resource_bounds_hash uses L1Gas path → 2 entries → poseidon(tip, l1, l2)

5. Assert H_A != H_B  ← canonicalization failure
```

pub fn get_tip_resource_bounds_hash(
    resource_bounds: &ValidResourceBounds,
    tip: &Tip,
) -> Result<Felt, StarknetApiError> {
    let l1_resource_bounds = resource_bounds.get_l1_bounds();
    let l2_resource_bounds = resource_bounds.get_l2_bounds();

    // L1 and L2 gas bounds always exist.
    // Old V3 txs always have L2 gas bounds of zero, but they exist.
    let mut resource_felts = vec![
        get_concat_resource(&l1_resource_bounds, L1_GAS)?,
        get_concat_resource(&l2_resource_bounds, L2_GAS)?,
    ];

    // For new V3 txs, need to also hash the data gas bounds.
    resource_felts.extend(match resource_bounds {
        ValidResourceBounds::L1Gas(_) => vec![],
        ValidResourceBounds::AllResources(all_resources) => {
            vec![get_concat_resource(&all_resources.l1_data_gas, L1_DATA_GAS)?]
        }
    });

    Ok(HashChain::new().chain(&tip.0.into()).chain_iter(resource_felts.iter()).get_poseidon_hash())
}

impl TryFrom<protobuf::ResourceBounds> for ValidResourceBounds {
    type Error = ProtobufConversionError;
    fn try_from(value: protobuf::ResourceBounds) -> Result<Self, Self::Error> {
        let Some(l1_gas) = value.l1_gas else {
            return Err(missing("ResourceBounds::l1_gas"));
        };
        let Some(l2_gas) = value.l2_gas else {
            return Err(missing("ResourceBounds::l2_gas"));
        };
        // TODO(Shahak): Assert data gas is not none once we remove support for 0.13.2.
        let l1_data_gas = value.l1_data_gas.unwrap_or_default();
        let l1_gas: ResourceBounds = l1_gas.try_into()?;
        let l2_gas: ResourceBounds = l2_gas.try_into()?;
        let l1_data_gas: ResourceBounds = l1_data_gas.try_into()?;
        Ok(if l1_data_gas.is_zero() && l2_gas.is_zero() {
            ValidResourceBounds::L1Gas(l1_gas)
        } else {
            ValidResourceBounds::AllResources(AllResourceBounds { l1_gas, l2_gas, l1_data_gas })
        })
    }
}
