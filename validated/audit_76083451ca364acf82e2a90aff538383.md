### Title
SwapAllowlistExtension Bypassed via Router: Any User Swaps on Restricted Pools - (`metric-periphery/contracts/extensions/SwapAllowlistExtension.sol`)

### Summary

`SwapAllowlistExtension.beforeSwap` checks the `sender` argument, which the pool sets to `msg.sender` of `pool.swap()`. When users route through `MetricOmmSimpleRouter`, `msg.sender` inside `pool.swap()` is the **router address**, not the original user. A pool admin who allowlists the router (the natural action to let legitimate users use the standard interface) simultaneously opens the gate to every unprivileged user, fully bypassing the access-control guard.

---

### Finding Description

`MetricOmmPool.swap()` passes `msg.sender` as the `sender` argument to `_beforeSwap`: [1](#0-0) 

`ExtensionCalling._beforeSwap` forwards that value unchanged to every registered extension: [2](#0-1) 

`SwapAllowlistExtension.beforeSwap` then gates on that `sender`: [3](#0-2) 

`MetricOmmSimpleRouter.exactInputSingle` calls `pool.swap()` directly, making the router the `msg.sender` the pool sees. The original user's address is stored only in the transient callback context for payment, and is never forwarded to the pool or the extension: [4](#0-3) 

The same pattern applies to `exactInput`, `exactOutputSingle`, and `exactOutput`.

**Bypass path:**

1. Pool admin deploys a pool with `SwapAllowlistExtension` and allowlists only addresses `A` and `B` (e.g., KYC'd counterparties).
2. To let `A` and `B` use the standard router interface, the admin calls `setAllowedToSwap(pool, router, true)`.
3. Attacker `C` (not allowlisted) calls `router.exactInputSingle(...)`.
4. Router calls `pool.swap(...)` → pool passes `msg.sender = router` as `sender` to `beforeSwap`.
5. Extension evaluates `allowedSwapper[pool][router] == true` → passes.
6. Attacker `C` swaps successfully on a pool that should have blocked them.

If the admin does **not** allowlist the router, the allowlist works but legitimate users `A` and `B` cannot use the router either — the extension forces direct `pool.swap()` calls, breaking the standard periphery interface.

---

### Impact Explanation

Any unprivileged user can swap on a pool whose admin intended to restrict access to a specific set of addresses. This breaks the core access-control invariant of `SwapAllowlistExtension`. Depending on pool design (e.g., institutional pools, pools with asymmetric pricing for specific counterparties), unauthorized swaps can drain LP value or extract favorable oracle-priced trades that were reserved for trusted parties. This is a direct loss of LP principal and broken core pool functionality.

---

### Likelihood Explanation

The trigger is a natural, expected admin action: allowlisting the router so that legitimate users can use the standard periphery. No malicious setup is required. Any pool that uses `SwapAllowlistExtension` and wants router compatibility is vulnerable. The attacker needs only to call the public `exactInputSingle` function on the router — no special privileges, no flash loans, no callbacks.

---

### Recommendation

Pass the **original user's address** through the swap call so extensions can gate on the true initiator. Two concrete approaches:

1. **Add an `initiator` field to `pool.swap()`** — the router passes `msg.sender` explicitly, and `_beforeSwap` forwards it as a separate argument alongside `sender`.
2. **Check `sender` in the extension against a router registry** — if `sender` is a known router, read the initiator from a standardized field in `extensionData` that the router is required to populate.

Until fixed, pool admins must not allowlist the router on allowlisted pools; users on such pools must call `pool.swap()` directly.

---

### Proof of Concept

```
Setup:
  pool = MetricOmmPool with SwapAllowlistExtension (BEFORE_SWAP_ORDER set)
  allowedSwapper[pool][alice]  = true   // legitimate user
  allowedSwapper[pool][router] = true   // admin adds router so alice can use it
  allowedSwapper[pool][attacker] = false

Attack:
  attacker calls:
    router.exactInputSingle(ExactInputSingleParams{
        pool: pool,
        tokenIn: token0,
        zeroForOne: true,
        amountIn: 1_000e18,
        amountOutMinimum: 0,
        ...
    })

  Execution trace:
    router.exactInputSingle()
      → pool.swap(recipient, zeroForOne, amount, limit, "", extensionData)
          msg.sender inside pool = router
        → _beforeSwap(sender=router, ...)
          → SwapAllowlistExtension.beforeSwap(sender=router, ...)
              allowedSwapper[pool][router] == true  ← passes
        → swap executes, attacker receives token1
        → router.metricOmmSwapCallback() pulls token0 from attacker

Result: attacker swaps on a pool they are not allowlisted for.
