### Title
`AllResourceBounds` with zero `l2_gas`/`l1_data_gas` is silently downcast to `ValidResourceBounds::L1Gas` during protobuf deserialization, producing a wrong hash and causing consensus block rejection - (`File: crates/apollo_protobuf/src/converters/transaction.rs`)

---

### Summary

The protobuf deserializer for `ResourceBounds` converts any `AllResources` payload whose `l2_gas` and `l1_data_gas` are both zero into `ValidResourceBounds::L1Gas`. This is the exact same "skip-when-zero" pattern as the external bug: a conditional on a zero value causes a downstream state/type update to be omitted. The consequence is twofold: (1) `RpcInvokeTransactionV3::try_from` rejects the deserialized transaction with `DEPRECATED_RESOURCE_BOUNDS_ERROR`, causing every consensus validator to reject a proposer's block that contains such a transaction; and (2) if the hash were recomputed after the round-trip, it would differ from the hash computed at gateway ingestion because `get_tip_resource_bounds_hash` includes `l1_data_gas` only for `AllResources`, not for `L1Gas`.

---

### Finding Description

**Step 1 – Gateway accepts the transaction.**

`RpcInvokeTransactionV3` carries `resource_bounds: AllResourceBounds`. A user may legitimately submit a transaction with `AllResourceBounds { l1_gas: X, l2_gas: 0, l1_data_gas: 0 }`. The gateway's stateless validator explicitly allows this (test case `valid_l1_gas` passes with `l2_gas` and `l1_data_gas` defaulting to zero). [1](#0-0) 

**Step 2 – Hash is computed as `AllResources`.**

`InternalRpcInvokeTransactionV3::resource_bounds()` always returns `ValidResourceBounds::AllResources(self.resource_bounds)`. `get_tip_resource_bounds_hash` therefore includes the `l1_data_gas` packed felt in the poseidon input: [2](#0-1) [3](#0-2) 

**Step 3 – Serialisation to protobuf preserves `AllResources`.**

`From<ValidResourceBounds> for protobuf::ResourceBounds` always emits all three fields when the variant is `AllResources`, so the wire bytes carry `l2_gas = 0` and `l1_data_gas = 0` explicitly. [4](#0-3) 

**Step 4 – Deserialisation silently downcasts to `L1Gas`.**

`TryFrom<protobuf::ResourceBounds> for ValidResourceBounds` checks whether both `l2_gas` and `l1_data_gas` are zero and, if so, returns `ValidResourceBounds::L1Gas` instead of `ValidResourceBounds::AllResources`:

```rust
Ok(if l1_data_gas.is_zero() && l2_gas.is_zero() {
    ValidResourceBounds::L1Gas(l1_gas)          // ← wrong for new AllResources txs
} else {
    ValidResourceBounds::AllResources(AllResourceBounds { l1_gas, l2_gas, l1_data_gas })
})
``` [5](#0-4) 

This logic was introduced to handle legacy 0.13.2 transactions that omit `l1_data_gas` entirely (`unwrap_or_default`), but it also fires for new transactions that explicitly set both fields to zero. [6](#0-5) 

**Step 5 – `RpcInvokeTransactionV3::try_from` fails.**

`TryFrom<protobuf::InvokeV3WithProof> for RpcInvokeTransactionV3` calls `snapi_invoke.try_into()` which is `RpcInvokeTransactionV3::try_from(InvokeTransactionV3)`. That conversion rejects any variant other than `AllResources`:

```rust
resource_bounds: match value.resource_bounds {
    ValidResourceBounds::AllResources(bounds) => bounds,
    _ => return Err(StarknetApiError::OutOfRange { string: "resource_bounds".to_string() }),
},
``` [7](#0-6) 

The error is mapped to `DEPRECATED_RESOURCE_BOUNDS_ERROR` and propagated through `TryFrom<protobuf::ConsensusTransaction> for ConsensusTransaction` via `?`: [8](#0-7) [9](#0-8) 

**Step 6 – Hash domain divergence (secondary).**

Even if the conversion did not fail, the hash computed at gateway time (with `AllResources`, 4-element poseidon input including `l1_data_gas`) differs from the hash that would be computed after the round-trip (with `L1Gas`, 3-element poseidon input): [10](#0-9) 

---

### Impact Explanation

A proposer that includes a transaction with `AllResourceBounds { l1_gas: X, l2_gas: 0, l1_data_gas: 0 }` will have its entire consensus block rejected by every validator, because the block's protobuf deserialisation fails with `DEPRECATED_RESOURCE_BOUNDS_ERROR`. This is a liveness attack: any unprivileged user can submit such a transaction to the gateway (which accepts it), wait for it to be included in a proposed block, and cause that block to be rejected. The proposer's round is wasted and consensus progress stalls until the transaction is evicted or the proposer skips it.

The secondary hash-domain divergence means that even if the deserialization were fixed without also fixing the hash, the transaction hash stored in the mempool would not match the hash recomputed by validators, producing a wrong receipt/event binding.

---

### Likelihood Explanation

The gateway explicitly accepts `AllResourceBounds` with zero `l2_gas` and `l1_data_gas` (the `valid_l1_gas` test case confirms this). No special privilege is required. Any user who submits a standard invoke transaction and sets only `l1_gas` (leaving the other two bounds at their default zero) triggers the bug.

---

### Recommendation

Distinguish between a truly absent `l1_data_gas` field (old 0.13.2 wire format, where `value.l1_data_gas` is `None`) and an explicitly present but zero field (new format). Only downcast to `L1Gas` in the former case:

```rust
// Before (buggy):
let l1_data_gas = value.l1_data_gas.unwrap_or_default();
...
Ok(if l1_data_gas.is_zero() && l2_gas.is_zero() {
    ValidResourceBounds::L1Gas(l1_gas)
} else { ... })

// After (fixed):
match value.l1_data_gas {
    None if l2_gas.is_zero() => Ok(ValidResourceBounds::L1Gas(l1_gas)),
    _ => {
        let l1_data_gas: ResourceBounds = value.l1_data_gas.unwrap_or_default().try_into()?;
        Ok(ValidResourceBounds::AllResources(AllResourceBounds { l1_gas, l2_gas, l1_data_gas }))
    }
}
``` [5](#0-4) 

---

### Proof of Concept

1. Submit to the gateway:
```json
{
  "type": "INVOKE",
  "version": "0x3",
  "sender_address": "0x<valid_account>",
  "calldata": ["0x1"],
  "nonce": "0x0",
  "resource_bounds": {
    "l1_gas":      { "max_amount": "0x100", "max_price_per_unit": "0x1" },
    "l2_gas":      { "max_amount": "0x0",   "max_price_per_unit": "0x0" },
    "l1_data_gas": { "max_amount": "0x0",   "max_price_per_unit": "0x0" }
  },
  "tip": "0x0",
  "paymaster_data": [],
  "account_deployment_data": [],
  "nonce_data_availability_mode": "L1",
  "fee_data_availability_mode": "L1",
  "signature": ["0x<valid_sig>"]
}
```

2. Gateway accepts the transaction (stateless validation passes; `AllResourceBounds` with zero `l2_gas`/`l1_data_gas` is valid).

3. Transaction enters the mempool with hash `H` computed over `AllResources` (4-element tip-resource-bounds poseidon).

4. Batcher includes the transaction in a proposed block. Proposer serialises the block to protobuf: `InvokeV3 { resource_bounds: { l1_gas: X, l2_gas: 0, l1_data_gas: 0 } }`.

5. Every validator calls `TryFrom<protobuf::ConsensusTransaction> for ConsensusTransaction`:
   - `ValidResourceBounds::try_from(ResourceBounds { l1_gas: X, l2_gas: 0, l1_data_gas: 0 })` → `ValidResourceBounds::L1Gas(X)` ← wrong branch
   - `RpcInvokeTransactionV3::try_from(InvokeTransactionV3 { resource_bounds: L1Gas(X), ... })` → `Err(OutOfRange)` → `DEPRECATED_RESOURCE_BOUNDS_ERROR`
   - Block deserialization returns `Err`; block is rejected.

6. The proposer's round is wasted. Repeat with any subsequent proposer that picks up the same transaction.

pub fn get_tip_resource_bounds_hash(
    resource_bounds: &ValidResourceBounds,
    tip: &Tip,
) -> Result<Felt, StarknetApiError> {
    let l1_resource_bounds = resource_bounds.get_l1_bounds();
    let l2_resource_bounds = resource_bounds.get_l2_bounds();

    // L1 and L2 gas bounds always exist.
    // Old V3 txs always have L2 gas bounds of zero, but they exist.
    let mut resource_felts = vec![
        get_concat_resource(&l1_resource_bounds, L1_GAS)?,
        get_concat_resource(&l2_resource_bounds, L2_GAS)?,
    ];

    // For new V3 txs, need to also hash the data gas bounds.
    resource_felts.extend(match resource_bounds {
        ValidResourceBounds::L1Gas(_) => vec![],
        ValidResourceBounds::AllResources(all_resources) => {
            vec![get_concat_resource(&all_resources.l1_data_gas, L1_DATA_GAS)?]
        }
    });

    Ok(HashChain::new().chain(&tip.0.into()).chain_iter(resource_felts.iter()).get_poseidon_hash())
}

pub(crate) fn get_invoke_transaction_v3_hash<T: InvokeTransactionV3Trait>(
    transaction: &T,
    chain_id: &ChainId,
    transaction_version: &TransactionVersion,
) -> Result<TransactionHash, StarknetApiError> {
    let tip_resource_bounds_hash =
        get_tip_resource_bounds_hash(&transaction.resource_bounds(), transaction.tip())?;
    let paymaster_data_hash =
        HashChain::new().chain_iter(transaction.paymaster_data().0.iter()).get_poseidon_hash();
    let data_availability_mode = concat_data_availability_mode(
        transaction.nonce_data_availability_mode(),
        transaction.fee_data_availability_mode(),
    );
    let account_deployment_data_hash = HashChain::new()
        .chain_iter(transaction.account_deployment_data().0.iter())
        .get_poseidon_hash();
    let calldata_hash =
        HashChain::new().chain_iter(transaction.calldata().0.iter()).get_poseidon_hash();
    let mut hash_chain = HashChain::new()
        .chain(&INVOKE)
        .chain(&transaction_version.0)
        .chain(transaction.sender_address().0.key())
        .chain(&tip_resource_bounds_hash)
        .chain(&paymaster_data_hash)
        .chain(&Felt::try_from(chain_id)?)
        .chain(&transaction.nonce().0)
        .chain(&data_availability_mode)
        .chain(&account_deployment_data_hash)
        .chain(&calldata_hash);
    if !transaction.proof_facts().0.is_empty() {
        let proof_facts_hash =
            HashChain::new().chain_iter(transaction.proof_facts().0.iter()).get_poseidon_hash();
        hash_chain = hash_chain.chain(&proof_facts_hash);
    }
    Ok(TransactionHash(hash_chain.get_poseidon_hash()))
}

impl TryFrom<protobuf::ResourceBounds> for ValidResourceBounds {
    type Error = ProtobufConversionError;
    fn try_from(value: protobuf::ResourceBounds) -> Result<Self, Self::Error> {
        let Some(l1_gas) = value.l1_gas else {
            return Err(missing("ResourceBounds::l1_gas"));
        };
        let Some(l2_gas) = value.l2_gas else {
            return Err(missing("ResourceBounds::l2_gas"));
        };
        // TODO(Shahak): Assert data gas is not none once we remove support for 0.13.2.
        let l1_data_gas = value.l1_data_gas.unwrap_or_default();
        let l1_gas: ResourceBounds = l1_gas.try_into()?;
        let l2_gas: ResourceBounds = l2_gas.try_into()?;
        let l1_data_gas: ResourceBounds = l1_data_gas.try_into()?;
        Ok(if l1_data_gas.is_zero() && l2_gas.is_zero() {
            ValidResourceBounds::L1Gas(l1_gas)
        } else {
            ValidResourceBounds::AllResources(AllResourceBounds { l1_gas, l2_gas, l1_data_gas })
        })
    }
}

impl TryFrom<protobuf::InvokeV3WithProof> for RpcInvokeTransactionV3 {
    type Error = ProtobufConversionError;
    fn try_from(mut value: protobuf::InvokeV3WithProof) -> Result<Self, Self::Error> {
        // Extract proof first, since `starknet_api::transaction::InvokeTransactionV3` does not
        // carry a `proof` field.
        let proof = Proof::from(std::mem::take(&mut value.proof));

        let snapi_invoke: InvokeTransactionV3 = value
            .invoke
            .ok_or(ProtobufConversionError::MissingField {
                field_description: "InvokeV3WithProof::invoke",
            })?
            .try_into()?;

        // This conversion can fail only if the resource_bounds are not AllResources.
        Ok(Self { proof, ..snapi_invoke.try_into().map_err(|_| DEPRECATED_RESOURCE_BOUNDS_ERROR)? })
    }
}
