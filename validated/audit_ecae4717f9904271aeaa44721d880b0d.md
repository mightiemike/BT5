### Title
ML-DSA-65 Gas-Key Host-Function Fee Uses Wire Length Instead of Trie-ID Length, Breaking Fee-Parity Invariant - (File: `runtime/near-vm-runner/src/logic/logic.rs`)

### Summary

The gas-key host-function paths (`promise_batch_action_add_gas_key_with_full_access`, `promise_batch_action_add_gas_key_with_function_call`, `promise_batch_action_transfer_to_gas_key`) pass the raw contract-supplied `public_key_len` (the borsh wire length) to `gas_key_add_key_exec_fee` and `gas_key_transfer_exec_fee`. The transaction path for the same operations correctly uses `public_key.trie_id_len()`. For ed25519 and secp256k1 these two values are identical, so the divergence was invisible before ML-DSA-65. For ML-DSA-65 they differ by exactly 1920 bytes (wire = 1953, trie-id = 33), causing the exec-fee charged via host function to be computed against the wrong key-length domain, breaking the fee-parity invariant that the `test_gas_key_fee_parity` test asserts.

### Finding Description

`gas_key_add_key_exec_fee` computes the per-nonce exec fee as:

```
nonce_key_len = access_key_key_len(account_id_len, public_key_len) + sizeof(NonceIndex)
per_byte = gas_key_byte_exec * (nonce_key_len + NONCE_VALUE_LEN) * num_nonces
```

The transaction path in `runtime/runtime/src/config.rs` calls this with `public_key.trie_id_len()`:

```rust
// config.rs line 392 — correct
let nonce_fee = gas_key_add_key_exec_fee(
    fees,
    account_id.len(),
    public_key.trie_id_len(),   // 33 for ML-DSA-65
    gas_key_info.num_nonces,
);
```

The host-function path in `logic.rs` calls it with the raw `public_key_len` from guest memory:

```rust
// logic.rs lines 3155-3160 — diverges for ML-DSA-65
let exec_fee = gas_key_add_key_exec_fee(
    &self.fees_config,
    receiver_id.len(),
    public_key_len as usize,    // 1953 for ML-DSA-65 (borsh wire length)
    num_nonces,
);
```

The same pattern appears in `promise_batch_action_transfer_to_gas_key` (lines 3092–3096) and in the wasmtime runner mirror at `wasmtime_runner/logic.rs` lines 3395–3400 and 3323–3325.

For ML-DSA-65 the exact divergent values are:
- Wire/borsh length: `1 + 1952 = 1953` bytes
- Trie-id length (`trie_id_len()`): `1 + 32 = 33` bytes (SHA3-256 hash form)
- Divergence: **1920 bytes per nonce key**

The design document explicitly states: *"Every storage-stake and trie-byte-priced fee path was updated to call `trie_id_len()`"* — but the host-function paths were not updated.

The `test_gas_key_fee_parity` test asserts that adding a gas key via transaction and via host function produces identical `gas_burnt` and `tokens_burnt`, but it only exercises ED25519 keys, leaving the ML-DSA-65 divergence undetected.

### Impact Explanation

Any contract calling `promise_batch_action_add_gas_key_with_full_access` or `promise_batch_action_add_gas_key_with_function_call` with an ML-DSA-65 public key is charged exec gas computed against a 1953-byte key length instead of the actual 33-byte trie-id. The overcharge per nonce is proportional to 1920 × `gas_key_byte` exec fee. For the maximum `num_nonces = 65535`, the total overcharge is `1920 × 65535 × gas_key_byte_exec_fee`, which can exceed the per-transaction gas limit, making ML-DSA-65 gas keys with many nonces effectively uncreatable via host functions even when the actual trie work is well within budget. The fee-parity invariant between the transaction path and the host-function path is broken for every ML-DSA-65 gas-key operation.

### Likelihood Explanation

Both `GasKeys` and `PostQuantumSignatures` stabilized at protocol version 85 (production). Any contract deployed on a v85+ network can call the affected host functions with an ML-DSA-65 public key. No privileged role is required; any unprivileged contract author controls `public_key_len`. The divergence is deterministic and reproducible on every invocation.

### Recommendation

Replace `public_key_len as usize` with `public_key.trie_id_len()` in all three host-function call sites after the key has been decoded from guest memory:

```rust
// After: let public_key = self.get_public_key(public_key_ptr, public_key_len)?;
let exec_fee = gas_key_add_key_exec_fee(
    &self.fees_config,
    receiver_id.len(),
    public_key.decode()?.trie_id_len(),  // use trie-id length, not wire length
    num_nonces,
);
```

Apply the same fix to `gas_key_transfer_exec_fee` call sites. Extend `test_gas_key_fee_parity` to cover ML-DSA-65 keys to prevent regression.

pub fn gas_key_add_key_exec_fee(
    cfg: &RuntimeFeesConfig,
    account_id_len: usize,
    public_key_len: usize,
    num_nonces: NonceIndex,
) -> GasKeyAddFee {
    let num_nonces = num_nonces as u64;
    let base =
        cfg.fee(ActionCosts::gas_key_nonce_write_base).exec_fee().checked_mul(num_nonces).unwrap();
    let nonce_key_len =
        access_key_key_len(account_id_len, public_key_len) + std::mem::size_of::<NonceIndex>();
    let per_byte = cfg
        .fee(ActionCosts::gas_key_byte)
        .exec_fee()
        .checked_mul((nonce_key_len + AccessKey::NONCE_VALUE_LEN) as u64)
        .unwrap()
        .checked_mul(num_nonces)
        .unwrap();
    GasKeyAddFee { base, per_byte }

### 5. Storage usage and fee plumbing

The storage-stake calculation
(`runtime/runtime/src/access_keys.rs::access_key_storage_usage`) and the
gas-key fee helpers (`gas_key_*_fee` in `runtime/runtime/src/config.rs`) use
`PublicKey::trie_id_len()` rather than `PublicKey::len()`:

- `len()` reports the borsh-encoded length (33 / 65 / 1953 across the
  three `PublicKey` variants).
- `trie_id_len()` reports the on-trie length (33 / 65 / **33**).

The two diverge only for `PublicKey::MLDSA65`. Every storage-stake and
trie-byte-priced fee path was updated to call `trie_id_len()`.

fn test_gas_key_fee_parity(mode: GasKeyKind) {
    let mut setup = setup_host_function_test();
    let account = setup.account.clone();

    let num_nonces: NonceIndex = 4;
    let gas_key_a_signer: Signer =
        InMemorySigner::from_seed(account.clone(), KeyType::ED25519, "gas_key_tx").into();
    let gas_key_b_signer: Signer =
        InMemorySigner::from_seed(account.clone(), KeyType::ED25519, "gas_key_host_fn").into();

    let public_key_b_base64 = near_primitives_core::serialize::to_base64(
        &borsh::to_vec(&gas_key_b_signer.public_key()).unwrap(),
    );

    // Add gas key A via transaction, B via host function
    let add_a_outcome = setup.run_actions(vec![Action::AddKey(Box::new(AddKeyAction {
        public_key: gas_key_a_signer.public_key(),
        access_key: mode.access_key(num_nonces, &account),
    }))]);
    let add_b_outcome = setup.run_call_promise(serde_json::json!([
        {"batch_create": {"account_id": account.as_str()}, "id": 0},
        mode.add_action_json(num_nonces, &account, &public_key_b_base64),
    ]));
    assert_eq!(add_a_outcome.gas_burnt, add_b_outcome.gas_burnt);
    assert_eq!(add_a_outcome.tokens_burnt, add_b_outcome.tokens_burnt);
