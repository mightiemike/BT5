### Title
Router-Mediated Swaps Pass Router Address as `sender` to `SwapAllowlistExtension::beforeSwap`, Breaking Per-User Allowlist Enforcement - (File: metric-periphery/contracts/extensions/SwapAllowlistExtension.sol)

---

### Summary

`SwapAllowlistExtension::beforeSwap` gates swaps by checking `allowedSwapper[pool][sender]`, where `sender` is the immediate caller of `pool.swap`. When a user routes through `MetricOmmSimpleRouter`, the pool sees the **router** as `msg.sender`, so the hook checks whether the **router** is allowlisted — not the actual user. This makes the per-user allowlist either completely bypassable (if the router is allowlisted) or completely unusable via the router (if it is not).

---

### Finding Description

**Call chain — direct swap:**

```
user → pool.swap(...)
  msg.sender in pool = user
  _beforeSwap(sender=user, ...)
  extension.beforeSwap(sender=user) with msg.sender=pool
  check: allowedSwapper[pool][user]   ← correct
```

**Call chain — router-mediated swap:**

```
user → router.exactInputSingle(params)
  router → pool.swap(...)
    msg.sender in pool = router
    _beforeSwap(sender=router, ...)
    extension.beforeSwap(sender=router) with msg.sender=pool
    check: allowedSwapper[pool][router]  ← wrong actor
```

In `MetricOmmPool::swap`, the pool passes `msg.sender` as the `sender` argument to `_beforeSwap`: [1](#0-0) 

`ExtensionCalling._beforeSwap` forwards that `sender` verbatim to the extension: [2](#0-1) 

`SwapAllowlistExtension::beforeSwap` then checks `allowedSwapper[msg.sender][sender]`, where `msg.sender` is the pool and `sender` is whoever called `pool.swap`: [3](#0-2) 

When the router calls `pool.swap`, the router's address is what lands in `sender`: [4](#0-3) 

This produces two mutually exclusive failure modes:

1. **Router not allowlisted**: Allowlisted users cannot swap through the router at all — the hook reverts with `NotAllowedToSwap`. Core router functionality is broken for any allowlisted pool.
2. **Router allowlisted** (the only way to enable router swaps): `allowAllSwappers[pool]` is set, or `allowedSwapper[pool][router] = true`. Now every user — including those the admin explicitly did not allowlist — can bypass the per-user gate by routing through the router.

**Pause aspect**: The `swap` function carries `whenNotPaused` which fires before `_beforeSwap` is ever reached, so there is no paused-flow regression through the extension. The pause claim in the question is a red herring; the real defect is the identity mismatch. [5](#0-4) 

---

### Impact Explanation

The `SwapAllowlistExtension` is a core access-control primitive. Its stated purpose is to gate `swap` by swapper address, per pool. Because the router is an intermediate contract, the hook can never see the real user — it always sees the router. Pool admins who deploy this extension to restrict swap access to a curated set of addresses get either a broken router integration or a fully bypassed allowlist. This constitutes broken core functionality.

---

### Likelihood Explanation

Any pool that uses `SwapAllowlistExtension` and also expects users to interact via `MetricOmmSimpleRouter` hits this immediately. The router is the standard periphery entry point, so the collision is not an edge case — it is the default usage pattern.

---

### Recommendation

The extension must receive the **original user** identity, not the immediate caller. Two options:

1. **Pass `tx.origin` as an additional parameter** in the hook ABI (not recommended — `tx.origin` is fragile and breaks account-abstraction).
2. **Require router-mediated swaps to embed the real payer in `extensionData`** and have the extension decode and verify it, with the router signing or committing to that identity.
3. **Preferred**: Add a `realSender` field to the hook call data that the pool populates from a trusted periphery registry, or require the router to be a known, trusted contract that forwards the original `msg.sender` through a dedicated field in `extensionData`, which the extension then reads and validates.

---

### Proof of Concept

```solidity
// Setup: pool admin allowlists alice directly, not the router
extension.setAllowedToSwap(pool, alice, true);
// allowedSwapper[pool][alice] = true
// allowedSwapper[pool][router] = false

// Alice tries to swap through the router
// router calls pool.swap(...) → msg.sender in pool = router
// _beforeSwap(sender=router, ...)
// extension checks allowedSwapper[pool][router] → false → revert NotAllowedToSwap
// Alice is blocked from using the router despite being allowlisted.

// Fix: admin must allowlist the router instead
extension.setAllowedToSwap(pool, router, true);
// Now bob (not allowlisted) calls router.exactInputSingle(...)
// router calls pool.swap(...) → msg.sender in pool = router
// extension checks allowedSwapper[pool][router] → true → PASSES
// Bob bypasses the allowlist entirely.
```
