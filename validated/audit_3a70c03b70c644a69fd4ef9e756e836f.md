### Title
Migration `ForceFullExitWithDelegation` Fails on Vesting-Owned Positions with Active Redelegation, Causing Chain Halt During v8 Upgrade — (`x/tieredrewards/keeper/force_exit.go`, `x/tieredrewards/migrations/v2/migrate.go`)

### Summary

The v8 upgrade migration (`Migrate1to2`) force-exits all vesting-account-owned tier positions via `ForceFullExitWithDelegation`. However, `ForceFullExitWithDelegation` calls `transferDelegationFromPosition`, which contains a hard guard that returns `ErrActiveRedelegation` if the position's delegator address has any live staking redelegation. There is no bypass for the migration context. If any vesting-owned position had an in-flight `TierRedelegate` at upgrade time, the migration returns an error, the upgrade handler returns an error, and the chain halts.

### Finding Description

**Preconditions (all achievable on v7.2.0):**
1. A vesting account creates a tier position — allowed pre-v8 since `validateNewPosition` had no vesting check.
2. The vesting account calls `MsgTierRedelegate` on that position — `validateRedelegatePosition` has no vesting-account check, so this succeeds and creates a staking redelegation entry with a non-zero `unbondingID`.
3. The redelegation's unbonding period has not elapsed at upgrade time (typical unbonding period is 21 days).

**Failure path:**

```
Migrate1to2
  └─ v2.Migrate
       └─ exitVestedAccountsPositions
            └─ ForceFullExitWithDelegation(posID)
                 └─ transferDelegationFromPosition
                      └─ isRedelegating(pos.DelegatorAddress)
                           └─ stakingKeeper.GetRedelegations(...) → [entry] → true
                      → return ErrActiveRedelegation  ← hard error, no bypass
                 → return error
            → return error  ← no error recovery, no skip
       → return error
  → return error
RunMigrations → error → upgrade handler → error → chain HALT
```

The `isRedelegating` check at [8](#0-7)  is correct for normal user-facing operations (preventing exits during redelegation), but it is not conditioned on whether the caller is the migration. `ForceFullExitWithDelegation` has no alternative code path to handle this case. [5](#0-4) 

`exitVestedAccountsPositions` does not catch or skip errors — any failure in `ForceFullExitWithDelegation` aborts the entire migration. [6](#0-5) 

### Impact Explanation

The upgrade handler returns an error from `RunMigrations`. In Cosmos SDK, this causes a panic in `BeginBlock` at the upgrade height, halting the chain. The chain cannot advance past the upgrade block until a patched binary is deployed via an emergency governance proposal. This is a **chain halt**, which is more severe than the "stuck position" framing in the question — the entire network stops, not just one user's funds.

The position itself is not "permanently stuck" in isolation — the chain halt is the primary impact. After a patched binary is deployed, the migration would need to handle the redelegating case (e.g., by waiting for redelegation completion or using a direct staking unbond bypass).

### Likelihood Explanation

This requires a vesting account to have created a tier position on v7.2.0 and called `TierRedelegate` within the 21-day unbonding window before the v8 upgrade. The v8 upgrade integration test (`setup_pre_v8_upgrade`) creates vesting-owned positions but does not test the redelegating case, so this scenario is not covered. [9](#0-8)  The likelihood depends on whether any real vesting-owned positions existed on mainnet with active redelegations at upgrade time.

### Recommendation

`ForceFullExitWithDelegation` should handle the active-redelegation case. Options:
- **Skip and log**: If `isRedelegating` is true, log a warning and skip the position (leaving it for the owner to exit manually after the redelegation completes). This avoids the chain halt but leaves the position alive post-upgrade.
- **Wait for completion**: Not feasible in a synchronous migration.
- **Direct staking bypass**: Call `stakingKeeper.Unbond` directly on the position's delegator address, bypassing the `isRedelegating` guard. This is safe because the redelegation entry is on the *source* validator, not the current delegation.

The migration test `TestMigrate1to2_ExitsVestedOwnerPositions` should be extended to cover the case where a vesting-owned position has an active redelegation. [10](#0-9)

func (s *KeeperSuite) TestMigrate1to2_FailsOnVestingOwnerWithActiveRedelegate() {
    s.setupTier(1)
    vals, bondDenom := s.getStakingData()
    val := vals[0]
    valAddr := sdk.MustValAddressFromBech32(val.GetOperator())
    dstValAddr, _ := s.createSecondValidator()

    amount := sdkmath.NewInt(sdk.DefaultPowerReduction.Int64())
    vestingOwner := s.newVestingOwnerWithBalance(bondDenom, amount, amount.MulRaw(2))

    // Create position on v7.2.0 (no vesting check pre-v8)
    pos := s.createLockTierPositionV1(vestingOwner, valAddr, amount)

    // Call TierRedelegate — no vesting check in validateRedelegatePosition
    msgServer := keeper.NewMsgServerImpl(s.keeper)
    _, err := msgServer.TierRedelegate(s.ctx, &types.MsgTierRedelegate{
        Owner:        vestingOwner.String(),
        PositionId:   pos.Id,
        DstValidator: dstValAddr.String(),
    })
    s.Require().NoError(err)

    // Verify redelegation is active
    isRed, err := s.keeper.IsRedelegating(s.ctx, pos.DelegatorAddress)
    s.Require().NoError(err)
    s.Require().True(isRed)

    // v8 migration fails — chain halt
    migrator := keeper.NewMigrator(s.keeper)
    err = migrator.Migrate1to2(s.ctx)
    s.Require().Error(err) // ErrActiveRedelegation propagates → upgrade handler fails
}

func (k Keeper) validateRedelegatePosition(ctx context.Context, pos types.PositionState, owner, dstValidator string) error {
	if !pos.IsOwner(owner) {
		return types.ErrNotPositionOwner
	}

	if !pos.IsDelegated() {
		return types.ErrPositionNotDelegated
	}

	if pos.Delegation.ValidatorAddress == dstValidator {
		return types.ErrRedelegationToSameValidator
	}

	sdkCtx := sdk.UnwrapSDKContext(ctx)
	if pos.HasTriggeredExit() && pos.CompletedExitLockDuration(sdkCtx.BlockTime()) {
		return types.ErrExitLockDurationElapsed
	}

	tier, err := k.getTier(ctx, pos.TierId)
	if err != nil {
		return err
	}

	if tier.IsCloseOnly() {
		return types.ErrTierIsCloseOnly
	}

	isRedelegating, err := k.isRedelegating(ctx, pos.DelegatorAddress)
	if err != nil {
		return err
	}
	if isRedelegating {
		return errorsmod.Wrapf(types.ErrActiveRedelegation, "position %d has an active redelegation", pos.Id)
	}

	return nil
}

func (s *KeeperSuite) TestMigrate1to2_ExitsVestedOwnerPositions() {
	s.setupTier(1)
	vals, bondDenom := s.getStakingData()
	val := vals[0]
	valAddr := sdk.MustValAddressFromBech32(val.GetOperator())
	s.setValidatorCommission(valAddr, sdkmath.LegacyZeroDec())

	amount := sdkmath.NewInt(sdk.DefaultPowerReduction.Int64())

	// Regular (non-vesting) owner with a tier position; must survive.
	regularOwner := sdk.AccAddress(secp256k1.GenPrivKey().PubKey().Address())
	regularAcc := s.app.AccountKeeper.NewAccountWithAddress(s.ctx, regularOwner)
	s.app.AccountKeeper.SetAccount(s.ctx, regularAcc)
	s.Require().NoError(banktestutil.FundAccount(
		s.ctx, s.app.BankKeeper, regularOwner, sdk.NewCoins(sdk.NewCoin(bondDenom, amount)),
	))
	regularPos := s.createLockTierPositionV1(regularOwner, valAddr, amount)

	// Vesting owner with two tier positions.
	// Both must be deleted by the migration.
	vestingOwner := s.newVestingOwnerWithBalance(bondDenom, amount, amount.MulRaw(3))
	commitPos := s.createCommitPositionV1(vestingOwner, val, valAddr, amount)
	lockPos := s.createLockTierPositionV1(vestingOwner, valAddr, amount)

	s.advanceForRewards(valAddr, bondDenom)

	migrator := keeper.NewMigrator(s.keeper)
	s.Require().NoError(migrator.Migrate1to2(s.ctx))

	// Both vesting-owned positions deleted.
	_, err := s.keeper.Positions.Get(s.ctx, commitPos.Id)
	s.Require().Error(err, "commit-origin vesting position must be deleted")
	_, err = s.keeper.Positions.Get(s.ctx, lockPos.Id)
	s.Require().Error(err, "lock-origin vesting position must be deleted")

	// Regular position survives, with DelegatorAddress equal to the legacy
	// derivation.
	survived, err := s.keeper.Positions.Get(s.ctx, regularPos.Id)
	s.Require().NoError(err, "regular position must survive")
	s.Require().Equal(regularOwner.String(), survived.Owner)
	s.Require().Equal(migration.LegacyDelegatorAddress(regularPos.Id), survived.DelegatorAddress)
}
