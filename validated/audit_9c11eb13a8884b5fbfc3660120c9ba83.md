### Title
`ValidResourceBounds` Variant Collapse in Protobuf Round-Trip Produces Divergent Transaction Hash Preimage — (`crates/apollo_protobuf/src/converters/transaction.rs`)

### Summary

The protobuf deserializer for `ValidResourceBounds` silently collapses an `AllResources` transaction whose `l2_gas` and `l1_data_gas` are both zero into the `L1Gas` variant. Because `get_tip_resource_bounds_hash` hashes a different number of resource elements depending on the variant (2 for `L1Gas`, 3 for `AllResources`), the same on-chain transaction produces two distinct hash preimages depending on which path computed it. A transaction admitted by the gateway with `AllResources(l1_gas=X, l2_gas=0, l1_data_gas=0)` receives hash H₁; after a protobuf round-trip the same body is reconstructed as `L1Gas(l1_gas=X)` and produces hash H₂ ≠ H₁. Any component that recomputes the hash from the deserialized body (block-sync hash validation, RPC tracing, fee estimation) will diverge from the canonical value.

### Finding Description

**Step 1 – The hash preimage is variant-sensitive.**

`get_tip_resource_bounds_hash` in `crates/starknet_api/src/transaction_hash.rs` builds the resource-bounds element of the Poseidon hash chain:

```
L1Gas variant  → [tip, L1_GAS_concat, L2_GAS_concat]          (2 resource felts)
AllResources   → [tip, L1_GAS_concat, L2_GAS_concat, L1_DATA_GAS_concat]  (3 resource felts)
``` [1](#0-0) 

**Step 2 – The gateway always admits `AllResources`.**

`RpcInvokeTransactionV3.resource_bounds` is typed `AllResourceBounds` (never `L1Gas`). The gateway's `validate_resource_bounds` only requires that `max_possible_fee(Tip::ZERO) > 0`, which is satisfied by any transaction with non-zero `l1_gas` even when `l2_gas = 0` and `l1_data_gas = 0`. [2](#0-1) [3](#0-2) 

The gateway converter wraps the `AllResourceBounds` in `ValidResourceBounds::AllResources` when computing the canonical hash: [4](#0-3) 

So the stored `tx_hash` is always computed with the 3-element preimage.

**Step 3 – The protobuf deserializer collapses the variant.**

`TryFrom<protobuf::ResourceBounds> for ValidResourceBounds` in the P2P/block-sync converter applies the following decision:

```rust
Ok(if l1_data_gas.is_zero() && l2_gas.is_zero() {
    ValidResourceBounds::L1Gas(l1_gas)
} else {
    ValidResourceBounds::AllResources(AllResourceBounds { l1_gas, l2_gas, l1_data_gas })
})
``` [5](#0-4) 

A transaction with `l2_gas = 0` and `l1_data_gas = 0` that was originally `AllResources` is silently reconstructed as `L1Gas`. The comment on line 426 even acknowledges the fragility: `// TODO(Shahak): Assert data gas is not none once we remove support for 0.13.2.`

**Step 4 – The reconstructed body produces a different hash.**

After deserialization, any call to `InvokeTransactionV3::calculate_transaction_hash` on the reconstructed body invokes `get_invoke_transaction_v3_hash`, which calls `get_tip_resource_bounds_hash` with the now-`L1Gas` variant. The resulting hash H₂ omits the `L1_DATA_GAS` felt and therefore H₂ ≠ H₁. [6](#0-5) 

**Step 5 – The divergence is observable in production paths.**

`validate_transaction_hash` is called during block sync to verify that the hash in the block header matches the computed hash of each transaction body. For the affected transaction class, this check will always fail on any node that received the block via P2P, causing the block to be rejected as invalid even though it was produced correctly by the sequencer. [7](#0-6) 

### Impact Explanation

Any V3 invoke/declare/deploy-account transaction with `l2_gas = 0` and `l1_data_gas = 0` (a valid, gateway-admitted configuration) will have its hash recomputed incorrectly after a protobuf round-trip. This causes:

- **Block-sync rejection**: nodes receiving the block via P2P recompute H₂ ≠ H₁ and reject the block as having an invalid transaction hash — a valid block is treated as invalid.
- **RPC tracing/simulation divergence**: any RPC endpoint that reconstructs the transaction from storage via the protobuf path and recomputes the hash will return an authoritative-looking wrong value.

This matches the allowed impact: *"High. Mempool/gateway/RPC admission accepts invalid transactions or rejects valid transactions before sequencing"* and *"High. RPC execution, fee estimation, tracing, simulation, or pending view returns an authoritative-looking wrong value."*

### Likelihood Explanation

The trigger requires only that a user submit a V3 transaction with non-zero `l1_gas` and zero `l2_gas`/`l1_data_gas`. This is a normal, gateway-accepted configuration (pre-0.13.3 style bounds expressed in the new format). No privileged access, no malformed bytes, and no attacker cooperation is required. Any such transaction included in a block will cause every syncing peer to fail hash validation for that block.

### Recommendation

Preserve the original `ValidResourceBounds` variant across the protobuf boundary. The serializer already encodes the distinction: when the variant is `L1Gas`, `l1_data_gas` is set to a zero `ResourceLimits` struct (not `None`), while `AllResources` sets it to the actual value. The deserializer should use the presence/absence of the `l1_data_gas` field (or an explicit tag) to reconstruct the correct variant rather than inferring it from whether the values are zero.

Concretely, change the deserializer to:

```rust
Ok(if value.l1_data_gas.is_none() {
    ValidResourceBounds::L1Gas(l1_gas)
} else {
    ValidResourceBounds::AllResources(AllResourceBounds { l1_gas, l2_gas, l1_data_gas })
})
```

and ensure the serializer sets `l1_data_gas = None` (not a zero struct) for `L1Gas` transactions.

Alternatively, add an explicit variant tag to the protobuf `ResourceBounds` message.

### Proof of Concept

```
1. Submit via RPC:
   RpcInvokeTransactionV3 {
       resource_bounds: AllResourceBounds {
           l1_gas:      ResourceBounds { max_amount: 1000, max_price_per_unit: 1 },
           l2_gas:      ResourceBounds { max_amount: 0,    max_price_per_unit: 0 },
           l1_data_gas: ResourceBounds { max_amount: 0,    max_price_per_unit: 0 },
       },
       ...
   }

2. Gateway computes hash H₁ using AllResources (3-element preimage):
   hash_chain = [INVOKE, version, sender, tip_resource_hash(AllResources), ...]
   where tip_resource_hash includes L1_DATA_GAS felt.

3. Transaction is included in block B. Block header records H₁.

4. Syncing peer receives block B via P2P protobuf.
   Deserializer sees l1_data_gas.is_zero() && l2_gas.is_zero()
   → reconstructs ValidResourceBounds::L1Gas(l1_gas).

5. Peer recomputes hash H₂ using L1Gas (2-element preimage):
   tip_resource_hash omits L1_DATA_GAS felt → H₂ ≠ H₁.

6. validate_transaction_hash(tx, block_number, chain_id, H₁, options)
   computes H₂, checks H₁ ∈ {H₂} → false → block B rejected.

pub fn get_tip_resource_bounds_hash(
    resource_bounds: &ValidResourceBounds,
    tip: &Tip,
) -> Result<Felt, StarknetApiError> {
    let l1_resource_bounds = resource_bounds.get_l1_bounds();
    let l2_resource_bounds = resource_bounds.get_l2_bounds();

    // L1 and L2 gas bounds always exist.
    // Old V3 txs always have L2 gas bounds of zero, but they exist.
    let mut resource_felts = vec![
        get_concat_resource(&l1_resource_bounds, L1_GAS)?,
        get_concat_resource(&l2_resource_bounds, L2_GAS)?,
    ];

    // For new V3 txs, need to also hash the data gas bounds.
    resource_felts.extend(match resource_bounds {
        ValidResourceBounds::L1Gas(_) => vec![],
        ValidResourceBounds::AllResources(all_resources) => {
            vec![get_concat_resource(&all_resources.l1_data_gas, L1_DATA_GAS)?]
        }
    });

    Ok(HashChain::new().chain(&tip.0.into()).chain_iter(resource_felts.iter()).get_poseidon_hash())
}

pub(crate) fn get_invoke_transaction_v3_hash<T: InvokeTransactionV3Trait>(
    transaction: &T,
    chain_id: &ChainId,
    transaction_version: &TransactionVersion,
) -> Result<TransactionHash, StarknetApiError> {
    let tip_resource_bounds_hash =
        get_tip_resource_bounds_hash(&transaction.resource_bounds(), transaction.tip())?;
    let paymaster_data_hash =
        HashChain::new().chain_iter(transaction.paymaster_data().0.iter()).get_poseidon_hash();
    let data_availability_mode = concat_data_availability_mode(
        transaction.nonce_data_availability_mode(),
        transaction.fee_data_availability_mode(),
    );
    let account_deployment_data_hash = HashChain::new()
        .chain_iter(transaction.account_deployment_data().0.iter())
        .get_poseidon_hash();
    let calldata_hash =
        HashChain::new().chain_iter(transaction.calldata().0.iter()).get_poseidon_hash();
    let mut hash_chain = HashChain::new()
        .chain(&INVOKE)
        .chain(&transaction_version.0)
        .chain(transaction.sender_address().0.key())
        .chain(&tip_resource_bounds_hash)
        .chain(&paymaster_data_hash)
        .chain(&Felt::try_from(chain_id)?)
        .chain(&transaction.nonce().0)
        .chain(&data_availability_mode)
        .chain(&account_deployment_data_hash)
        .chain(&calldata_hash);
    if !transaction.proof_facts().0.is_empty() {
        let proof_facts_hash =
            HashChain::new().chain_iter(transaction.proof_facts().0.iter()).get_poseidon_hash();
        hash_chain = hash_chain.chain(&proof_facts_hash);
    }
    Ok(TransactionHash(hash_chain.get_poseidon_hash()))

impl TryFrom<protobuf::ResourceBounds> for ValidResourceBounds {
    type Error = ProtobufConversionError;
    fn try_from(value: protobuf::ResourceBounds) -> Result<Self, Self::Error> {
        let Some(l1_gas) = value.l1_gas else {
            return Err(missing("ResourceBounds::l1_gas"));
        };
        let Some(l2_gas) = value.l2_gas else {
            return Err(missing("ResourceBounds::l2_gas"));
        };
        // TODO(Shahak): Assert data gas is not none once we remove support for 0.13.2.
        let l1_data_gas = value.l1_data_gas.unwrap_or_default();
        let l1_gas: ResourceBounds = l1_gas.try_into()?;
        let l2_gas: ResourceBounds = l2_gas.try_into()?;
        let l1_data_gas: ResourceBounds = l1_data_gas.try_into()?;
        Ok(if l1_data_gas.is_zero() && l2_gas.is_zero() {
            ValidResourceBounds::L1Gas(l1_gas)
        } else {
            ValidResourceBounds::AllResources(AllResourceBounds { l1_gas, l2_gas, l1_data_gas })
        })
    }
