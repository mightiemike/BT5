### Title
`SignedDelegateAction` Nonce Unconditionally Consumed by Any Mempool Observer, Griefing Relayer — (File: `runtime/runtime/src/actions.rs`)

---

### Summary

NEAR's meta-transaction system (`DelegateAction`, NEP-366) is structurally identical to EIP-2612 `permit()`: a user signs an authorization off-chain, hands it to a relayer, and the relayer submits it on-chain. Because `SignedDelegateAction` contains no binding to the outer transaction's signer, any unprivileged node that observes the relayer's pending transaction in the mempool can extract the `SignedDelegateAction` verbatim, wrap it in their own outer transaction, and submit it first. `validate_delegate_action_key` then unconditionally advances the sender's access-key nonce, causing the legitimate relayer's transaction to fail with `DelegateActionInvalidNonce` and burning the relayer's gas with no compensation.

---

### Finding Description

**Root cause — no relayer binding in the signed payload.**

`DelegateAction` is signed by the user (Alice) over a payload that covers `sender_id`, `receiver_id`, `actions`, `nonce`, `max_block_height`, and `public_key`: [1](#0-0) 

The outer transaction's signer (the relayer) is **not** part of this payload. The `SignedDelegateAction` struct carries only the delegate action and Alice's signature: [2](#0-1) 

**Execution path — nonce advanced unconditionally.**

When the outer transaction is processed, `apply_delegate_action` verifies Alice's signature and then calls `validate_delegate_action_key`: [3](#0-2) 

`validate_delegate_action_key` checks `delegate_nonce > current_nonce`: [4](#0-3) 

If the check passes, the nonce is **unconditionally written back** to state — regardless of who submitted the outer transaction: [5](#0-4) 

**Attack path.**

1. Alice signs a `DelegateAction` and sends it off-chain to a relayer.
2. The relayer wraps it in `SignedTransaction { signer: relayer, receiver: alice, actions: [Delegate(signed_delegate_action)] }` and broadcasts it.
3. An attacker observes the mempool, extracts `signed_delegate_action` verbatim, and wraps it in `SignedTransaction { signer: attacker, receiver: alice, actions: [Delegate(signed_delegate_action)] }`.
4. If the attacker's transaction is included first, `validate_delegate_action_key` advances Alice's nonce to `delegate_nonce`.
5. The relayer's transaction is then processed; the same nonce check now fails (`delegate_nonce <= current_nonce`), returning `DelegateActionInvalidNonce`.
6. The relayer's gas is burned; the relayer receives no compensation.

This is the exact structural analog of the EIP-2612 permit front-run: a signed authorization detached from the submitter's identity can be replayed by any mempool observer to consume the one-time nonce before the intended submitter.

---

### Impact Explanation

- **Relayer financial loss**: The relayer's outer transaction is charged gas even when it fails. Gas is non-refundable on failure.
- **Relayer DoS**: An attacker can automate mempool monitoring and systematically front-run every relayer transaction, draining the relayer's NEAR balance and making the relayer service economically unviable.
- **User disruption**: If the relayer service is disrupted, users who rely on gasless meta-transactions lose that capability. Alice must sign a new `DelegateAction` with a fresh nonce for each front-run attempt.
- **No attacker profit**: The attacker pays gas for their own transaction. This is a pure griefing attack.

---

### Likelihood Explanation

- Any node with mempool access can perform this attack — no validator role, no special privilege, no cryptographic capability required.
- NEAR transactions are broadcast to all nodes before inclusion, so the `SignedDelegateAction` is universally observable.
- The attack is trivially automatable: watch for `Action::Delegate` or `Action::DelegateV2` in pending transactions, extract the inner signed action, resubmit under a different outer signer.
- The only attacker cost is gas per front-run, which can be kept minimal.

---

### Recommendation

Bind the `SignedDelegateAction` to the intended relayer by including a `relayer_id: Option<AccountId>` field in `DelegateAction` (and `DelegateActionV2`) that is part of the signed payload. During `validate_delegate_action_key`, verify that `action_receipt.signer_id == relayer_id` when `relayer_id` is `Some`. A `None` value preserves the current open-relayer semantics for backward compatibility. This mirrors the standard mitigation for EIP-2612 permit front-running: bind the authorization to the specific caller that is expected to consume it.

pub struct SignedDelegateAction {
    pub delegate_action: DelegateAction,
    pub signature: Signature,
}

impl SignedDelegateAction {
    pub fn verify(&self) -> bool {
        let delegate_action = &self.delegate_action;
        let hash = delegate_action.get_nep461_hash();
        let public_key = &delegate_action.public_key;

        self.signature.verify(hash.as_ref(), public_key)
    }

    pub fn sign(singer: &Signer, delegate_action: DelegateAction) -> Self {
        let signature = singer.sign(delegate_action.get_nep461_hash().as_bytes());
        Self { delegate_action, signature }
    }
}

pub(crate) fn apply_delegate_action(
    state_update: &mut TrieUpdate,
    apply_state: &ApplyState,
    action_receipt: &VersionedActionReceipt,
    sender_id: &AccountId,
    signed_delegate_action: VersionedSignedDelegateActionRef<'_>,
    result: &mut ActionResult,
) -> Result<(), RuntimeError> {
    if !signed_delegate_action.verify() {
        result.result = Err(ActionErrorKind::DelegateActionInvalidSignature.into());
        return Ok(());
    }
    let delegate_action = signed_delegate_action.delegate_action();
    if apply_state.block_height > delegate_action.max_block_height() {
        result.result = Err(ActionErrorKind::DelegateActionExpired.into());
        return Ok(());
    }
    if delegate_action.sender_id().as_str() != sender_id.as_str() {
        result.result = Err(ActionErrorKind::DelegateActionSenderDoesNotMatchTxReceiver {
            sender_id: delegate_action.sender_id().clone(),
            receiver_id: sender_id.clone(),
        }
        .into());
        return Ok(());
    }

    validate_delegate_action_key(state_update, apply_state, delegate_action, result)?;
    if result.result.is_err() {
        // Validation failed. Need to return Ok() because this is not a runtime error.
        // "result.result" will be return to the User as the action execution result.
        return Ok(());
    }
