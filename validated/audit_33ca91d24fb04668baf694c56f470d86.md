Audit Report

## Title
SwapAllowlistExtension Bypass via MetricOmmSimpleRouter: Router Address Replaces End-User in Allowlist Check — (File: metric-periphery/contracts/extensions/SwapAllowlistExtension.sol)

## Summary
`SwapAllowlistExtension.beforeSwap` gates swaps by checking the `sender` argument, which resolves to `msg.sender` of the `pool.swap()` call — the router, not the end user. When a pool admin allowlists the router to enable router-mediated swaps, every unprivileged user can bypass the allowlist by routing through `MetricOmmSimpleRouter`. The pool admin's intent to restrict which addresses can deplete LP reserves is silently nullified.

## Finding Description
`MetricOmmPool.swap()` passes `msg.sender` as the first argument to `_beforeSwap`: [1](#0-0) 

`SwapAllowlistExtension.beforeSwap` receives that value as `sender` and checks `allowedSwapper[msg.sender][sender]`, where `msg.sender` is the pool and `sender` is whoever called `pool.swap()`: [2](#0-1) 

`MetricOmmSimpleRouter.exactInputSingle` calls `pool.swap()` directly, making the router the `msg.sender` of that call — the end user (`msg.sender` of `exactInputSingle`) is never forwarded to the pool or extension: [3](#0-2) 

The same pattern applies to `exactInput` (intermediate hops use `address(this)` as payer): [4](#0-3) 

And to `exactOutputSingle`: [5](#0-4) 

For router-mediated swaps to work at all on a restricted pool, the pool admin must call `setAllowedToSwap(pool, address(router), true)`. Once the router is allowlisted, `allowedSwapper[pool][router]` is `true`, and the check passes for every caller of the router regardless of whether that caller is individually allowlisted. [6](#0-5) 

No existing guard recovers the originating user identity. The `extensionData` field is passed through from the router to the pool and then to the extension, but the extension does not decode it for identity purposes — it relies solely on the `sender` parameter.

## Impact Explanation
Any address not on the allowlist can trade against restricted LP funds by calling `MetricOmmSimpleRouter.exactInputSingle`, `exactInput`, or `exactOutputSingle` targeting the restricted pool. The pool's liquidity is exposed to unauthorized counterparties. This is a direct loss-of-access-control over LP principal matching the allowlist path impact gate: swap allowlist checks must cover the exact actor intended and cannot be bypassed through the router.

## Likelihood Explanation
The trigger is unprivileged: any user can call the public router. The only precondition is that the pool admin has allowlisted the router address, which is the necessary and natural production step to support router-mediated swaps on a restricted pool. A pool admin who deploys `SwapAllowlistExtension` and also wants users to use the router will inevitably create this condition. No malicious setup is required; the bypass is a structural consequence of the design.

## Recommendation
The extension must gate the actual end user, not the intermediary. Two viable approaches:

1. **Forward the originating user through `extensionData`**: the router encodes `msg.sender` (the end user) into `extensionData` before calling `pool.swap()`, and the extension decodes and checks that address when `sender` is a known router. This requires a trusted encoding convention between the router and extension.

2. **Router registry in the extension**: the extension maintains a set of trusted routers; when `sender` is a router, it requires the actual user address to be present in `extensionData` and checks `allowedSwapper[pool][decodedUser]` instead of `allowedSwapper[pool][router]`.

Either approach must ensure the extension checks the economic actor, not the syntactic caller of `pool.swap()`.

## Proof of Concept
```
Setup:
  pool admin deploys pool with SwapAllowlistExtension
  pool admin calls swapExtension.setAllowedToSwap(pool, address(router), true)
    → router is allowlisted so that router-mediated swaps work
  pool admin does NOT call setAllowedToSwap(pool, alice, true)

Attack:
  alice calls router.exactInputSingle({pool: restrictedPool, ...})
  router calls pool.swap(recipient, ...) with msg.sender = router
  pool calls _beforeSwap(sender=router, ...)
  extension checks allowedSwapper[pool][router] → true
  swap executes; alice receives output tokens from restricted LP funds

Expected: revert NotAllowedToSwap (alice is not allowlisted)
Actual:   swap succeeds (router is allowlisted, alice is invisible to the check)
```
