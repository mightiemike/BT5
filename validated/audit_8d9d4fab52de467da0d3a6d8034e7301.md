Audit Report

## Title
`SwapAllowlistExtension.beforeSwap` checks the router address instead of the originating user, allowing any caller to bypass the swap allowlist via `MetricOmmSimpleRouter` — (`metric-periphery/contracts/extensions/SwapAllowlistExtension.sol`)

## Summary

`MetricOmmPool.swap` passes `msg.sender` as the `sender` argument to `_beforeSwap`, and `SwapAllowlistExtension.beforeSwap` gates on `allowedSwapper[pool][sender]`. When a user routes through `MetricOmmSimpleRouter`, the router is `msg.sender` of `pool.swap()`, so the extension evaluates `allowedSwapper[pool][router]` rather than `allowedSwapper[pool][actual_user]`. A pool admin who allowlists the router — the only way to let any allowlisted user use the router — inadvertently opens the pool to every caller of the router, defeating the curated-pool invariant.

## Finding Description

`MetricOmmPool.swap` passes `msg.sender` as the first argument to `_beforeSwap`: [1](#0-0) 

`SwapAllowlistExtension.beforeSwap` receives that value as `sender` and gates on `allowedSwapper[msg.sender][sender]`, where `msg.sender` is the pool and `sender` is whoever called `pool.swap()`: [2](#0-1) 

`MetricOmmSimpleRouter.exactInputSingle` calls `pool.swap()` directly, making the router contract the `msg.sender` seen by the pool: [3](#0-2) 

The router never encodes the originating user's address into `extensionData` or any other field visible to the extension. The extension therefore evaluates `allowedSwapper[pool][router_address]`, and the original user's identity is never consulted. Once the router address is allowlisted (the only way to permit allowlisted users to use the router), the guard passes for every caller who routes through the router.

## Impact Explanation

Any non-allowlisted user can swap on a curated pool protected by `SwapAllowlistExtension` by routing through `MetricOmmSimpleRouter`. The pool admin's curation guarantee — that only explicitly approved addresses may trade — is silently nullified. Unauthorized users can execute swaps at oracle-derived prices, draining LP funds at will. This is a direct loss of LP principal and a broken core pool invariant (allowlist enforcement), meeting the contest's Critical/High threshold for direct loss of user principal and broken core pool functionality. [4](#0-3) 

## Likelihood Explanation

The pool admin must allowlist the router address for the bypass to be reachable. However, this is the natural and expected action: without allowlisting the router, no allowlisted user can use the router either, making the router entirely unusable on curated pools. Once the router is allowlisted (a single admin transaction), the bypass is trivially reachable by any unprivileged user with no further preconditions, no special knowledge, and no privileged access. The attack is repeatable indefinitely. [5](#0-4) 

## Recommendation

The extension must gate on the **original user**, not the intermediary router. The cleanest production fix is to have `MetricOmmSimpleRouter` encode `msg.sender` (the real user) into `extensionData` using a trusted, authenticated format, and have `SwapAllowlistExtension` decode and verify it when the caller is a known trusted router. Alternatively, the extension can maintain a registry of trusted routers; when `sender` is a trusted router, it decodes the real user from `extensionData`; otherwise it uses `sender` directly. The simplest safe fix with no router compatibility is to document that router-mediated swaps on allowlisted pools are unsupported and require allowlisted users to call `pool.swap()` directly, removing the need to allowlist the router address. [6](#0-5) 

## Proof of Concept

```
Setup:
  pool = MetricOmmPool with SwapAllowlistExtension (beforeSwap order = extension 1)
  alice = allowlisted user  (allowedSwapper[pool][alice] = true)
  bob   = non-allowlisted user
  admin calls setAllowedToSwap(pool, router, true)
    (necessary so alice can use the router at all)

Attack (bob):
  1. bob calls MetricOmmSimpleRouter.exactInputSingle({pool: pool, ...})
  2. Router calls pool.swap(recipient, ...) — msg.sender inside pool = router address
  3. pool._beforeSwap(sender=router, ...) → SwapAllowlistExtension.beforeSwap(sender=router, ...)
  4. Check: allowedSwapper[pool][router] == true → PASSES
  5. Swap executes; bob receives output tokens at oracle price
  6. LP funds are drained by an actor the pool admin never intended to allow

Expected: revert NotAllowedToSwap()
Actual:   swap succeeds
``` [7](#0-6)
