### Title
`SplitKeyDenom` Misidentifies Regular `"ibc"` Denom Keys as IBC Denom Keys, Permanently Corrupting the Owner Index — (`x/nft/types/keys.go`, `x/nft/types/validation.go`)

---

### Summary

An unprivileged attacker can issue a denom with ID `"ibc"` (which passes all validation), mint one NFT under it, and permanently corrupt the NFT owner KV-store index. After that, every call to `GetOwner` for any holder of an `"ibc"`-denom NFT returns an error, and `GetOwners` returns an error for **all owners on the chain**, breaking owner enumeration globally.

---

### Finding Description

**Root cause — validation gap:**

`MsgIssueDenom.ValidateBasic()` calls `ValidateDenomID`, not `ValidateDenomIDWithIBC`. [1](#0-0) 

`ValidateDenomID` only checks length (3–64), starts-with-alpha, and all-alphanumeric. The string `"ibc"` satisfies all three conditions. [2](#0-1) 

`ValidateDenomIDWithIBC` — which would reject `"ibc"` because it doesn't start with `"ibc/"` and is not 68 chars — is **never called** in the `IssueDenom` or `MintNFT` message paths. [3](#0-2) 

**Root cause — key parsing collision:**

`KeyOwner` for denom `"ibc"` and tokenID `"abc"` produces the byte sequence:

```
0x02 + "/" + <address> + "/" + "ibc" + "/" + "abc"
``` [4](#0-3) 

When `SplitKeyOwner` parses this key, it strips the `PrefixOwners+delimiter` prefix and calls `bytes.SplitN(remaining, "/", 2)`, yielding `keys[1] = []byte("ibc/abc")`. [5](#0-4) 

`SplitKeyDenom` then receives `[]byte("ibc/abc")` and checks:

```go
if bytes.HasPrefix(key, []byte(IBCPrefix)) {   // "ibc/" — TRUE for "ibc/abc"
    rest := key[len(IBCPrefix):]               // rest = "abc"
    idx := bytes.Index(rest, delimiter)        // idx = -1 (no "/" in "abc")
    if idx < 0 || idx != IBCDenomLen-len(IBCPrefix) {  // -1 < 0 → TRUE
        return denomID, tokenID, errors.New("wrong KeyDenom")
    }
``` [6](#0-5) 

The error is returned unconditionally for every NFT minted under the `"ibc"` denom.

**Propagation to `GetOwner` and `GetOwners`:**

`GetOwner` iterates all owner keys for the given address+denom prefix and calls `SplitKeyOwner` on each. On the first `"ibc"`-denom key it hits the error and returns `types.Owner{}, err`. [7](#0-6) 

`GetOwners` iterates **all** owner keys in the store. The moment it encounters any key belonging to the `"ibc"` denom it returns `types.Owners{}, err`, breaking owner enumeration for every address on the chain. [8](#0-7) 

---

### Impact Explanation

| Affected surface | Effect |
|---|---|
| `GetOwner(ctx, A, "ibc")` | Returns error for every holder of an `"ibc"`-denom NFT |
| `GetOwners(ctx)` | Returns error for **all** owners chain-wide after one `"ibc"` NFT is minted |
| Genesis export | Broken if it calls `GetOwners` |
| Any IBC or downstream module relying on owner enumeration | Broken |

The NFT data itself (stored under `KeyNFT`) is unaffected, so individual `GetNFT`/`TransferNFT`/`BurnNFT` operations that go through `IsOwner → GetNFT` still work. However, the owner index is permanently corrupted: the malicious key cannot be removed without burning the NFT, and even after burning, any future query that scans the full owner prefix before the delete completes will error.

---

### Likelihood Explanation

The attack requires only two standard, permissionless transactions available to any account:

1. `MsgIssueDenom{Id: "ibc", Name: "anyname", ...}` — no special permission needed, `"ibc"` passes `ValidateDenomID`.
2. `MsgMintNFT{DenomId: "ibc", Id: "abc", ...}` — denom creator can mint.

No governance, no operator access, no leaked keys. The precondition (no existing `"ibc"` denom) is trivially satisfied on any fresh chain or any chain that has not yet seen this denom registered.

---

### Recommendation

1. **Block `"ibc"` as a plain denom ID.** In `ValidateDenomID`, add an explicit rejection of any string that, when used as a denom, would produce a key segment matching `IBCPrefix`:
   ```go
   if strings.HasPrefix(denomID, "ibc") {
       return sdkerrors.Wrapf(ErrInvalidDenom, "denom id cannot start with 'ibc'")
   }
   ```
2. **Or fix `SplitKeyDenom` to require the full IBC format.** Only treat a key as an IBC denom if it has the prefix `"ibc/"` **and** the hash segment is exactly 64 hex chars before the next `/`. The current check `idx < 0` already catches the malformed case, but the key should never have been written in the first place.
3. **Use `ValidateDenomIDWithIBC` in `MsgIssueDenom.ValidateBasic`** to ensure the `"ibc/"` prefix is reserved exclusively for IBC-issued denoms.

---

### Proof of Concept

```go
// keeper integration test (no mocks, standard SDK test context)
func TestIBCDenomCorruption(t *testing.T) {
    ctx, keeper := setupKeeper(t)
    creator, _ := sdk.AccAddressFromBech32("cosmos1...")

    // Step 1: issue denom "ibc" — passes ValidateDenomID
    err := keeper.IssueDenom(ctx, "ibc", "ibcname", "", "", creator)
    require.NoError(t, err)

    // Step 2: mint one NFT — setOwner writes key "0x02/address/ibc/tok1"
    err = keeper.MintNFT(ctx, "ibc", "tok1", "", "", "", creator, creator)
    require.NoError(t, err)

    // Step 3: GetOwner returns error — SplitKeyDenom("ibc/tok1") fails
    _, err = keeper.GetOwner(ctx, creator, "ibc")
    require.Error(t, err) // "wrong KeyDenom"

    // Step 4: GetOwners returns error for ALL owners chain-wide
    _, err = keeper.GetOwners(ctx)
    require.Error(t, err) // "wrong KeyDenom"
}
```

func SplitKeyDenom(key []byte) (denomID, tokenID string, err error) {
	if bytes.HasPrefix(key, []byte(IBCPrefix)) {
		// IBC denom: ibc/hash/tokenID (take remainder after second delimiter as tokenID)
		rest := key[len(IBCPrefix):]
		idx := bytes.Index(rest, delimiter)
		if idx < 0 || idx != IBCDenomLen-len(IBCPrefix) {
			return denomID, tokenID, errors.New("wrong KeyDenom")
		}
		denomID = IBCPrefix + string(rest[:idx])
		tokenID = string(rest[idx+len(delimiter):])
	} else {
		// Standard denom: denom/tokenID (take remainder after first delimiter as tokenID)
		idx := bytes.Index(key, delimiter)
		if idx < 0 {
			return denomID, tokenID, errors.New("wrong KeyDenom")
		}
		denomID = string(key[:idx])
		tokenID = string(key[idx+len(delimiter):])
	}

	return denomID, tokenID, nil
}
```

func KeyOwner(address sdk.AccAddress, denomID, tokenID string) []byte {
	key := append(PrefixOwners, delimiter...)

		key = append(key, delimiter...)
	}

		key = append(key, []byte(denomID)...)
		key = append(key, delimiter...)
	}

	if address != nil && len(denomID) > 0 && len(tokenID) > 0 {
		key = append(key, []byte(tokenID)...)
	}
	return key
}
