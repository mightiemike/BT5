Audit Report

## Title
SwapAllowlistExtension Gates the Router Address Instead of the Actual User, Allowing Any User to Bypass the Swap Allowlist via MetricOmmSimpleRouter - (File: metric-periphery/contracts/extensions/SwapAllowlistExtension.sol)

## Summary

`SwapAllowlistExtension.beforeSwap` checks the `sender` argument against the per-pool allowlist, but `sender` is `msg.sender` of the `pool.swap` call — the router's address when users route through `MetricOmmSimpleRouter`. If the pool admin allowlists the router to support router-mediated swaps for allowlisted users, every user — including those not individually allowlisted — can bypass the per-user restriction by routing through the router. The allowlist invariant ("only approved addresses may swap") is silently violated for all router-mediated trades.

## Finding Description

The call chain is confirmed in production code:

1. `MetricOmmSimpleRouter.exactInputSingle` calls `pool.swap(params.recipient, ...)` with the router as `msg.sender`, passing `params.extensionData` verbatim without encoding the originating user's address. [1](#0-0) 

2. `MetricOmmPool.swap` passes `msg.sender` (= router) as the `sender` argument to `_beforeSwap`. [2](#0-1) 

3. `ExtensionCalling._beforeSwap` forwards `sender` (= router) verbatim to the extension via `abi.encodeCall`. [3](#0-2) 

4. `SwapAllowlistExtension.beforeSwap` evaluates `allowedSwapper[msg.sender][sender]`, which resolves to `allowedSwapper[pool][router]` — not `allowedSwapper[pool][user]`. [4](#0-3) 

The pool admin faces an impossible choice: not allowlisting the router prevents allowlisted users from using the standard periphery path at all; allowlisting the router (the only way to support it) opens the gate to every user regardless of individual allowlist status. No existing guard checks the originating user's address anywhere in the call chain.

## Impact Explanation

This is broken core pool functionality. `SwapAllowlistExtension` is the production mechanism for restricting which addresses may trade on a curated pool. Once the router is allowlisted (required for normal periphery usage), any non-allowlisted user can execute a full swap and receive output tokens by calling `MetricOmmSimpleRouter.exactInputSingle` or `exactInput`. The pool settles the swap normally; the allowlist guard silently passes because it sees the router. The per-user curation policy is completely nullified for all router-mediated trades, constituting broken core pool functionality with direct loss of curation control over who may trade.

## Likelihood Explanation

High. The router is the standard, documented user-facing entry point. No special knowledge, privilege, or on-chain setup is required beyond what any user already has. Any user who observes that a direct pool call is blocked can immediately retry through the router. The bypass is a single function call and is repeatable indefinitely.

## Recommendation

The extension must check the economic actor (the end user), not the technical caller (the router). The cleanest fix is for the router to encode `msg.sender` into `extensionData` before forwarding to the pool (e.g., `abi.encode(msg.sender)` appended or prepended), and for `SwapAllowlistExtension.beforeSwap` to decode and check that address when `sender` is a known router. Alternatively, a registry of trusted routers can be maintained in the extension, and when `sender` is a known router, the extension decodes the originating user from `extensionData` for the allowlist check.

## Proof of Concept

```solidity
// Setup: pool with SwapAllowlistExtension, only alice is allowlisted
swapExtension.setAllowedToSwap(address(pool), alice, true);

// Admin must also allowlist the router to support standard periphery usage
swapExtension.setAllowedToSwap(address(pool), address(router), true);

// Bob (not allowlisted) bypasses the allowlist via the router
vm.prank(bob);
router.exactInputSingle(
    IMetricOmmSimpleRouter.ExactInputSingleParams({
        pool: address(pool),
        tokenIn: address(token0),
        recipient: bob,
        amountIn: 1000,
        amountOutMinimum: 0,
        zeroForOne: true,
        priceLimitX64: 0,
        deadline: block.timestamp + 1,
        extensionData: ""
    })
);
// Bob's swap succeeds: extension checked allowedSwapper[pool][router] = true
// Bob received token1 despite not being on the allowlist
assertGt(token1.balanceOf(bob), 0);
```
