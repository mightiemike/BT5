### Title
Unprivileged Caller Can Trigger Bulk Mainchain GC, Permanently Deleting Block Headers — (`contract/src/lib.rs`)

### Summary

`submit_blocks` is gated by `#[trusted_relayer]`, requiring the caller to be a registered, staked relayer. The public `run_mainchain_gc` method, which performs the same irreversible state mutations (deleting headers from `headers_pool`, `mainchain_height_to_header`, `mainchain_header_to_height`, and advancing `mainchain_initial_blockhash`), carries **no caller authorization check at all**. Any unprivileged NEAR account can call it directly with an arbitrary `batch_size`, triggering bulk deletion of all GC-eligible block headers in a single transaction.

---

### Finding Description

`submit_blocks` is decorated with both `#[pause]` and `#[trusted_relayer]`: [1](#0-0) 

Internally it calls `self.run_mainchain_gc(num_of_headers)`, where `num_of_headers` is bounded by the batch submitted in that call.

`run_mainchain_gc` is also exposed as a standalone public `&mut self` method: [2](#0-1) 

Its only guard is `#[pause(except(roles(Role::UnrestrictedRunGC)))]`, which blocks the call only when the contract is paused. There is **no `#[trusted_relayer]` check**. When the contract is live (unpaused), any NEAR account — with no role, no stake, no registration — can call `run_mainchain_gc` directly.

The function removes up to `batch_size` headers from the oldest end of the mainchain: [3](#0-2) 

Passing `batch_size = u64::MAX` causes it to remove every header currently eligible for GC (all blocks beyond `gc_threshold`) in a single call, then advance `mainchain_initial_blockhash` to the new oldest block. These deletions are permanent — there is no recovery path.

---

### Impact Explanation

- `mainchain_height_to_header`, `mainchain_header_to_height`, and `headers_pool` entries for all GC-eligible blocks are permanently deleted.
- `mainchain_initial_blockhash` is advanced in one jump rather than gradually.
- Any subsequent call to `verify_transaction_inclusion` or `verify_transaction_inclusion_v2` for a block that was deleted will panic with `"block does not belong to the current main chain"` — permanently, since the headers cannot be re-inserted.
- Downstream bridges, atomic-swap protocols, or any consumer that relies on SPV proof verification for already-confirmed transactions will have those proofs permanently invalidated, potentially locking funds irreversibly.

---

### Likelihood Explanation

The entry point is a public NEAR contract method callable by any account with no preconditions. The attacker needs only to know the contract address and call `run_mainchain_gc` with a large `batch_size`. No privileged keys, staking, or special knowledge is required. The attack is a single transaction.

---

### Recommendation

Add `#[trusted_relayer]` (or an equivalent role-based access check) to `run_mainchain_gc`, consistent with how `submit_blocks` is protected:

```rust
#[pause(except(roles(Role::UnrestrictedRunGC)))]
#[trusted_relayer]   // add this
pub fn run_mainchain_gc(&mut self, batch_size: u64) { … }
```

Alternatively, restrict the method to callers holding a designated role (e.g., `Role::UnrestrictedRunGC` or a new `GCManager` role) so that unprivileged accounts cannot trigger state-mutating GC directly.

---

### Proof of Concept

1. Deploy the contract normally (any chain feature flag).
2. Let the relayer submit enough blocks so that `amount_of_headers_we_store > gc_threshold` (i.e., at least one block is GC-eligible).
3. From any unprivileged NEAR account (no role granted), call:
   ```
   run_mainchain_gc(18446744073709551615)   // u64::MAX
   ```
4. Observe that all GC-eligible headers are deleted from `headers_pool` and the height maps, and `mainchain_initial_blockhash` is advanced.
5. Call `verify_transaction_inclusion` for any of the deleted blocks — the call panics with `"block does not belong to the current main chain"`, confirming permanent proof invalidation.

The root cause is the asymmetry between `submit_blocks` (line 168–169, `#[trusted_relayer]` enforced) and `run_mainchain_gc` (line 376–377, no caller authorization), directly analogous to the reported pattern where `ClosePosition` lacked the ownership check that sibling instructions correctly enforced.     /// Public call to run GC on a mainchain.
```rust
    /// `batch_size` is how many block headers should be removed in the execution
    ///
    /// # Panics
    /// If initial blockheader or tip blockheader are not in a header pool
    #[pause(except(roles(Role::UnrestrictedRunGC)))]
    pub fn run_mainchain_gc(&mut self, batch_size: u64) {
        let initial_blockheader = self
            .headers_pool
            .get(&self.mainchain_initial_blockhash)
            .unwrap_or_else(|| env::panic_str(ERR_KEY_NOT_EXIST));

        let tip_blockheader = self
            .headers_pool
            .get(&self.mainchain_tip_blockhash)
            .unwrap_or_else(|| env::panic_str(ERR_KEY_NOT_EXIST));

        let amount_of_headers_we_store =
            tip_blockheader.block_height - initial_blockheader.block_height + 1;

        if amount_of_headers_we_store > self.gc_threshold {
            let total_amount_to_remove = amount_of_headers_we_store - self.gc_threshold;
            let selected_amount_to_remove = std::cmp::min(total_amount_to_remove, batch_size);

            let start_removal_height = initial_blockheader.block_height;
            let end_removal_height = initial_blockheader.block_height + selected_amount_to_remove;
            env::log_str(&format!(
                "Num of blocks to remove {selected_amount_to_remove}"
            ));

            for height in start_removal_height..end_removal_height {
                let blockhash = &self
                    .mainchain_height_to_header
                    .get(&height)
                    .unwrap_or_else(|| env::panic_str(ERR_KEY_NOT_EXIST));

                self.remove_block_header(blockhash);
                self.mainchain_height_to_header.remove(&height);
            }

            self.mainchain_initial_blockhash = self
                .mainchain_height_to_header
                .get(&end_removal_height)
                .unwrap_or_else(|| env::panic_str(ERR_KEY_NOT_EXIST));
        }
    }
```
