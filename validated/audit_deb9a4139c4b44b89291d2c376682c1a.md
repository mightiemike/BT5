Audit Report

## Title
Deposit Allowlist Gates on Caller-Supplied `owner` Instead of Actual Token Provider `sender`, Allowing Full Allowlist Bypass — (`metric-periphery/contracts/extensions/DepositAllowlistExtension.sol`)

## Summary
`DepositAllowlistExtension.beforeAddLiquidity` discards the `sender` argument (the actual token provider, i.e., `msg.sender` of the pool call) and checks only `owner` (the LP recipient, a free caller-supplied parameter). Because any caller can pass an allowlisted address as `owner`, any unlisted address can inject tokens into a curated pool while LP shares are minted to that allowlisted address, completely nullifying the allowlist.

## Finding Description
`MetricOmmPool.addLiquidity` accepts a caller-supplied `owner` and passes `(msg.sender, owner)` to `_beforeAddLiquidity` (L191). `ExtensionCalling._beforeAddLiquidity` faithfully encodes both as `(sender, owner)` in the hook calldata (L95–98). However, `DepositAllowlistExtension.beforeAddLiquidity` declares the first parameter (`sender`) as unnamed and ignores it, then evaluates `allowedDepositor[msg.sender][owner]` (L32–38) — checking the LP recipient rather than the token provider. The token pull occurs via `IMetricOmmModifyLiquidityCallback(msg.sender).metricOmmModifyLiquidityCallback(...)` (L147–148 of `LiquidityLib`), so the attacker's contract supplies the tokens. LP shares are stored under `_positionBinKey(owner, salt, binIdx)` (L72), crediting the allowlisted address. The allowlisted address can subsequently call `removeLiquidity` (which requires `msg.sender == owner`, L206 of `MetricOmmPool`) to withdraw the deposited tokens. No existing guard checks the actual token provider against the allowlist.

## Impact Explanation
The deposit allowlist is the sole mechanism for pool curation. Bypassing it allows any address to inject liquidity into a pool restricted to vetted depositors, breaking the core pool invariant that only allowlisted addresses can provide liquidity. This constitutes broken core pool functionality under the contest rules. The attacker's tokens enter the pool and LP shares are minted to an allowlisted address that did not consent to the deposit; that address can then drain those tokens via `removeLiquidity`.

## Likelihood Explanation
The bypass requires a single public call to `pool.addLiquidity` with any known allowlisted address as `owner`. No privileged access, flash loan, special token behavior, or off-chain data is needed. Any observer of on-chain allowlist state (via `allowedDepositor` public mapping) can execute this immediately and repeatably.

## Recommendation
Gate on `sender` (the actual token provider) rather than `owner` (the LP recipient):

```solidity
// Fix:
function beforeAddLiquidity(address sender, address, uint80, LiquidityDelta calldata, bytes calldata)
    external view override returns (bytes4)
{
    if (!allowAllDepositors[msg.sender] && !allowedDepositor[msg.sender][sender]) {
        revert IMetricOmmPoolActions.NotAllowedToDeposit();
    }
    return IMetricOmmExtensions.beforeAddLiquidity.selector;
}
```

## Proof of Concept
```solidity
function test_allowlistBypass_ownerVsSender() public {
    address allowlisted = makeAddr("allowlisted");
    address attacker    = makeAddr("attacker");

    vm.prank(admin);
    depositExtension.setAllowedToDeposit(address(pool), allowlisted, true);

    deal(address(token0), attacker, 1_000 ether);
    deal(address(token1), attacker, 1_000 ether);
    vm.startPrank(attacker);
    token0.approve(address(pool), type(uint256).max);
    token1.approve(address(pool), type(uint256).max);

    // Attacker passes allowlisted address as owner — extension checks owner, passes
    LiquidityDelta memory d = _deltaAbovePrice(4, 10_000);
    pool.addLiquidity(allowlisted, 1, d, callbackData, "");
    vm.stopPrank();

    // LP shares minted to allowlisted; attacker's tokens consumed
    uint256 shares = stateView.positionBinShares(address(pool), allowlisted, 1, int8(4));
    assertGt(shares, 0);
    assertLt(token0.balanceOf(attacker) + token1.balanceOf(attacker), 2_000 ether);
}
```
