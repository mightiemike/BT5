### Title
`SplitKeyDenom` IBC-prefix false-match on denom `"ibc"` corrupts owner-index enumeration ŌĆö (`x/nft/types/keys.go`)

---

### Summary

An unprivileged attacker can issue a denom with ID `"ibc"` and mint one NFT into it. Because `KeyOwner` encodes the owner key as `denomID + "/" + tokenID`, the stored suffix becomes `"ibc/<tokenID>"`. `SplitKeyDenom` then incorrectly enters its IBC-parsing branch (due to `bytes.HasPrefix(key, []byte("ibc/"))` matching), fails to find a second delimiter, and returns `"wrong KeyDenom"`. This causes `GetOwner` to return an error for any query on that denom, and causes `GetOwners` to abort with an error for **all** owners on the chain.

---

### Finding Description

**Entry point ŌĆö `MsgIssueDenom.ValidateBasic` calls `ValidateDenomID`, not `ValidateDenomIDWithIBC`:** [1](#0-0) 

`ValidateDenomID` only checks length (`[3, 64]`) and alphanumeric format: [2](#0-1) 

`"ibc"` is 3 chars, begins with alpha, is alphanumeric ŌĆö it passes. `ValidateDenomIDWithIBC` would also pass it because `"ibc"` does not start with `"ibc/"`: [3](#0-2) 

`MsgMintNFT.ValidateBasic` also calls only `ValidateDenomID`: [4](#0-3) 

**Key construction ŌĆö `KeyOwner` encodes as `denomID + "/" + tokenID`:** [5](#0-4) 

For `denomID="ibc"`, `tokenID="abc"`, the stored key suffix (after stripping `PrefixOwners + "/"` and the address) is `"ibc/abc"`.

**Parsing ŌĆö `SplitKeyOwner` passes that suffix to `SplitKeyDenom`:** [6](#0-5) 

**Root cause ŌĆö `SplitKeyDenom` false-matches the IBC branch:** [7](#0-6) 

`bytes.HasPrefix([]byte("ibc/abc"), []byte("ibc/"))` is `true`. The function enters the IBC branch, strips `"ibc/"`, gets `rest = "abc"`, calls `bytes.Index("abc", "/")` which returns `-1`, hits `idx < 0`, and returns `errors.New("wrong KeyDenom")`.

**Propagation ŌĆö `GetOwner` and `GetOwners` both propagate the error immediately:** [8](#0-7) [9](#0-8) 

`GetOwners` iterates **all** owner keys. Once it hits any key for denom `"ibc"`, it returns an error and aborts enumeration for every owner on the chain.

---

### Impact Explanation

- `GetOwner(ctx, addr, "ibc")` always returns an error after the mint.
- `GetOwners(ctx)` returns an error for the entire chain, breaking invariant checks, IBC NFT transfer accounting, and any module that iterates the owner store.
- The corruption is permanent: the key is written by `setOwner` and cannot be read back correctly by any existing code path.

---

### Likelihood Explanation

The attack requires only two standard, permissionless transactions (`MsgIssueDenom` + `MsgMintNFT`) from any account with enough gas. No governance, no special role, no key leak. The denom `"ibc"` is not reserved anywhere in the validation layer.

---

### Recommendation

Add an explicit rejection of `"ibc"` (and any string that, when used as a denom, would produce a key with the `"ibc/"` prefix) in `ValidateDenomID`. The simplest fix is:

```go
// In ValidateDenomID, after the length/alphanumeric checks:
if denomID == "ibc" {
    return sdkerrors.Wrapf(ErrInvalidDenom, "denom id 'ibc' is reserved")
}
```

Alternatively, fix `SplitKeyDenom` to require that the IBC branch only fires when the key length is exactly `IBCDenomLen` (68) before the tokenID delimiter, making the check length-gated rather than prefix-only.

---

### Proof of Concept

```go
// keeper unit test (no chain needed)
func TestIBCDenomPoisoning(t *testing.T) {
    // setup keeper with in-memory store ...
    err := k.IssueDenom(ctx, "ibc", "ibcname", "", "", issuer)
    require.NoError(t, err)

    err = k.MintNFT(ctx, "ibc", "abc", "", "", "", issuer, recipient)
    require.NoError(t, err)

    // GetOwner must not error
    owner, err := k.GetOwner(ctx, recipient, "ibc")
    require.NoError(t, err)                  // FAILS: returns "wrong KeyDenom"
    require.Len(t, owner.IDCollections, 1)

    // GetOwners must not error
    owners, err := k.GetOwners(ctx)
    require.NoError(t, err)                  // FAILS: returns "wrong KeyDenom"
    require.Len(t, owners, 1)
}
```

```go
func SplitKeyDenom(key []byte) (denomID, tokenID string, err error) {
	if bytes.HasPrefix(key, []byte(IBCPrefix)) {
		// IBC denom: ibc/hash/tokenID (take remainder after second delimiter as tokenID)
		rest := key[len(IBCPrefix):]
		idx := bytes.Index(rest, delimiter)
		if idx < 0 || idx != IBCDenomLen-len(IBCPrefix) {
			return denomID, tokenID, errors.New("wrong KeyDenom")
		}
		denomID = IBCPrefix + string(rest[:idx])
		tokenID = string(rest[idx+len(delimiter):])
	} else {
		// Standard denom: denom/tokenID (take remainder after first delimiter as tokenID)
		idx := bytes.Index(key, delimiter)
		if idx < 0 {
			return denomID, tokenID, errors.New("wrong KeyDenom")
		}
		denomID = string(key[:idx])
		tokenID = string(key[idx+len(delimiter):])
	}

	return denomID, tokenID, nil
}
```

```go
func KeyOwner(address sdk.AccAddress, denomID, tokenID string) []byte {
	key := append(PrefixOwners, delimiter...)
	if address != nil {
		key = append(key, []byte(address.String())...)
		key = append(key, delimiter...)
	}

	if address != nil && len(denomID) > 0 {
		key = append(key, []byte(denomID)...)
		key = append(key, delimiter...)
	}

	if address != nil && len(denomID) > 0 && len(tokenID) > 0 {
		key = append(key, []byte(tokenID)...)
	}
	return key
}
```
