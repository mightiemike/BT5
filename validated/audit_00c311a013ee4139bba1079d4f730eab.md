### Title
Deprecated `verify_transaction_inclusion` Accepts Internal-Node Hashes as Valid Transaction IDs, Enabling Fake SPV Proof Acceptance — (File: `contract/src/lib.rs`)

### Summary

`verify_transaction_inclusion` (the deprecated v1 function) skips the coinbase-proof depth check that `verify_transaction_inclusion_v2` performs. Any unprivileged NEAR caller can supply an internal Merkle-tree node hash as `tx_id` and a shortened proof path, causing the function to return `true` for a Bitcoin transaction that does not exist. Downstream contracts that gate fund releases on this result are directly exploitable.

### Finding Description

The contract exposes two SPV proof functions. The v2 function adds a coinbase-proof length guard to prevent the 64-byte internal-node forgery attack: [1](#0-0) 

The v1 function is still `pub`, still `#[pause]`-active by default, and still reachable by any NEAR account: [2](#0-1) 

The only validation performed is:

1. `tx_block_blockhash` must be in the mainchain (`mainchain_header_to_height`).
2. `merkle_proof` must be non-empty.
3. `compute_root_from_merkle_proof(tx_id, tx_index, merkle_proof) == header.block_header.merkle_root`. [3](#0-2) 

`compute_root_from_merkle_proof` is a pure positional hash-chain computation: [4](#0-3) 

It does not distinguish between a leaf-level transaction hash and an internal-node hash. If the attacker supplies an internal node `H` at depth `d` (position `p` in the tree at that level) and provides the `max_depth − d` sibling hashes from depth `d` up to the root, the computation produces the correct Merkle root and the function returns `true`.

The missing guard — present only in v2 — is: [5](#0-4) 

This length equality check (`merkle_proof.len() == coinbase_merkle_proof.len()`) ensures the claimed transaction is at the same tree depth as the coinbase (a real leaf), blocking any internal-node forgery. v1 has no equivalent check.

The `#[deprecated]` Rust attribute is a compile-time warning only; it imposes no on-chain access restriction. The function remains callable by any NEAR account.

### Impact Explanation

Any downstream NEAR contract that calls `verify_transaction_inclusion` to confirm a Bitcoin payment before releasing funds can be tricked into releasing those funds for a Bitcoin transaction that was never broadcast or confirmed. The attacker needs only a real mainchain block hash (publicly known) and the block's Merkle tree structure (also publicly known from any Bitcoin block explorer). No cryptographic computation is required. The corrupted value is the boolean proof result returned to the consuming contract.

### Likelihood Explanation

The attack is computationally trivial:

- Bitcoin Merkle trees are fully public.
- The attacker picks any internal node `H` at depth `d`, position `p` in any mainchain block.
- The sibling hashes at each level from `d` to the root are read directly from the block.
- No hash preimage search is needed; the attacker is not inverting SHA-256.
- `confirmations` can be set to `1` to avoid the confirmation-count guard.

Any attacker aware of the 64-byte Merkle forgery class can execute this immediately.

### Recommendation

Remove or gate `verify_transaction_inclusion` so it is no longer callable by unprivileged accounts. The simplest approach is to add a `#[private]` attribute or a role check, or to have it unconditionally panic with a message directing callers to v2. All integrators must migrate to `verify_transaction_inclusion_v2`.

### Proof of Concept

Assume a mainchain block at height H with Merkle root R. The block has 4 transactions: `[T0, T1, T2, T3]`. The internal node at depth 1, position 0 is `N = SHA256d(T0 || T1)`. The sibling at depth 1 is `S = SHA256d(T2 || T3)`. The Merkle root is `R = SHA256d(N || S)`.

Call `verify_transaction_inclusion` with:
- `tx_id = N` (internal node hash, not a real txid)
- `tx_block_blockhash = <any mainchain block hash with this structure>`
- `tx_index = 0`
- `merkle_proof = [S]` (one hop, not the full leaf-depth proof)
- `confirmations = 1`

`compute_root_from_merkle_proof(N, 0, [S])`:
- position 0 is even → `hash(N, S) = R` ✓

The function returns `true`. No such transaction `N` exists in the block. A downstream contract releasing funds on this result would be drained.

    pub fn verify_transaction_inclusion(&self, #[serializer(borsh)] args: ProofArgs) -> bool {
        require!(
            args.confirmations <= self.gc_threshold,
            "The required number of confirmations exceeds the number of blocks stored in memory"
        );

        let heaviest_block_header = self
            .headers_pool
            .get(&self.mainchain_tip_blockhash)
            .unwrap_or_else(|| env::panic_str(ERR_KEY_NOT_EXIST));
        let target_block_height = self
            .mainchain_header_to_height
            .get(&args.tx_block_blockhash)
            .unwrap_or_else(|| env::panic_str("block does not belong to the current main chain"));

        // Check requested confirmations. No need to compute proof if insufficient confirmations.
        require!(
            (heaviest_block_header.block_height).saturating_sub(target_block_height) + 1
                >= args.confirmations,
            "Not enough blocks confirmed"
        );

        let header = self
            .headers_pool
            .get(&args.tx_block_blockhash)
            .unwrap_or_else(|| env::panic_str("cannot find requested transaction block"));

        require!(!args.merkle_proof.is_empty(), "Merkle proof is empty");

        // compute merkle tree root and check if it matches block's original merkle tree root
        merkle_tools::compute_root_from_merkle_proof(
            args.tx_id,
            usize::try_from(args.tx_index).unwrap(),
            &args.merkle_proof,
        ) == header.block_header.merkle_root
    }

pub fn compute_root_from_merkle_proof(
    transaction_hash: H256,
    transaction_position: usize,
    merkle_proof: &Vec<H256>,
) -> H256 {
    let mut current_hash = transaction_hash;
    let mut current_position = transaction_position;

    for proof_hash in merkle_proof {
        if current_position % 2 == 0 {
            current_hash = compute_hash(&current_hash, proof_hash);
        } else {
            current_hash = compute_hash(proof_hash, &current_hash);
        }
        current_position /= 2;
    }

    current_hash
}
