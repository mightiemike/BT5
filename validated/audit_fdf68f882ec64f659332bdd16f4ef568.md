Audit Report

## Title
Router-Mediated Swaps Replace User Identity with Router Address in `SwapAllowlistExtension.beforeSwap` — (`metric-periphery/contracts/extensions/SwapAllowlistExtension.sol`)

## Summary
`SwapAllowlistExtension.beforeSwap` gates swaps by checking `allowedSwapper[pool][sender]`, where `sender` is the direct caller of `MetricOmmPool.swap()`. When a user routes through `MetricOmmSimpleRouter`, the pool receives `msg.sender = router`, so the allowlist checks the router's address rather than the actual user's address. This collapses per-user granularity to a binary router-level gate, allowing any unprivileged user to bypass the curated allowlist by calling the public router.

## Finding Description
`MetricOmmPool.swap()` passes `msg.sender` as the `sender` argument to `_beforeSwap`: [1](#0-0) 

`ExtensionCalling._beforeSwap` forwards that `sender` verbatim to the extension: [2](#0-1) 

`SwapAllowlistExtension.beforeSwap` checks `allowedSwapper[msg.sender][sender]` where `msg.sender` is the pool and `sender` is whoever called `pool.swap()`: [3](#0-2) 

`MetricOmmSimpleRouter.exactInputSingle` calls `pool.swap()` directly with no user identity forwarding — `msg.sender` inside the pool becomes the router address: [4](#0-3) 

The same applies to `exactInput`, `exactOutputSingle`, and `exactOutput`. Notably, `params.extensionData` is passed through unchanged from the caller — the router does not encode the original user into it, and the extension does not decode it. There is no existing guard that recovers the true end-user identity on the router path.

This creates an irreconcilable conflict: if the router is not allowlisted, all router-mediated swaps revert for every user including allowlisted ones. If the router is allowlisted (the natural operational step to let approved users access the periphery), the per-user allowlist is completely bypassed — any address can swap through the router regardless of whether it appears in `allowedSwapper`.

## Impact Explanation
Disallowed users can trade on pools restricted to a curated set of swappers. This breaks the core access-control guarantee of `SwapAllowlistExtension`: the allowlist policy configured by the pool admin does not apply when users go through the supported periphery path (`MetricOmmSimpleRouter`). This constitutes broken core pool functionality — the exact wrong value is `allowedSwapper[pool][router]` being evaluated instead of `allowedSwapper[pool][user]`, permitting unauthorized swaps on curated pools.

## Likelihood Explanation
Any pool using `SwapAllowlistExtension` that allowlists the router is vulnerable. Allowlisting the router is the expected operational pattern for a curated pool that also wants to support the standard router UX. The attacker requires no special privileges — calling the public `exactInputSingle` or `exactInput` on the router pointing at the curated pool is sufficient. The condition is repeatable for every swap attempt.

## Recommendation
The extension must verify the actual end-user identity, not the immediate caller. The most compatible fix is for the router to encode `msg.sender` into `extensionData` before calling `pool.swap()`, and for `SwapAllowlistExtension.beforeSwap` to decode and check it when the immediate `sender` is a known trusted router. Since the router is a factory-deployed periphery contract, it can be trusted to supply the correct originating user. Alternatively, document and enforce at the factory level that `SwapAllowlistExtension` is incompatible with router-mediated swaps.

## Proof of Concept
```
1. Pool admin deploys pool with SwapAllowlistExtension attached.
2. Pool admin calls setAllowedToSwap(pool, router, true)
   // intended to let approved users access the pool via the router
3. Attacker (address not in allowedSwapper) calls:
     MetricOmmSimpleRouter.exactInputSingle({pool: curated_pool, ...})
4. Router calls pool.swap(...) — msg.sender inside pool = router address.
5. _beforeSwap(sender=router, ...) → extension checks allowedSwapper[pool][router] = true → passes.
6. Attacker's swap executes on the curated pool despite not being allowlisted.
```
