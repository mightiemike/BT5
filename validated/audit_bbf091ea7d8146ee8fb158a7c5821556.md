Audit Report

## Title
`SwapAllowlistExtension.beforeSwap` gates the router address instead of the originating user, allowing any caller to bypass the curated-pool swap allowlist via `MetricOmmSimpleRouter` — (`metric-periphery/contracts/extensions/SwapAllowlistExtension.sol`)

## Summary
`SwapAllowlistExtension.beforeSwap` checks `allowedSwapper[msg.sender][sender]` where `sender` is the value forwarded from `MetricOmmPool.swap` — which is the pool's own `msg.sender`. When a user routes through `MetricOmmSimpleRouter`, the pool's `msg.sender` is the router contract, not the originating user. If the router is allowlisted (required for any legitimate user to use it), every address on the network can bypass the curated-pool allowlist by routing through the public router.

## Finding Description
`MetricOmmPool.swap` passes `msg.sender` as the `sender` argument to `_beforeSwap`: [1](#0-0) 

`ExtensionCalling._beforeSwap` forwards that value verbatim to every configured extension: [2](#0-1) 

`SwapAllowlistExtension.beforeSwap` then checks `allowedSwapper[msg.sender][sender]` — where `msg.sender` is the pool and `sender` is the router: [3](#0-2) 

All four router entry points (`exactInputSingle`, `exactInput`, `exactOutputSingle`, `exactOutput`) call `pool.swap(...)` directly, making the router the pool's `msg.sender`: [4](#0-3) [5](#0-4) 

The router never encodes the originating `msg.sender` into `extensionData` or any other field that the extension could use to recover the real user identity. The extension therefore checks whether the **router** is allowlisted, not whether the **user** is allowlisted.

`DepositAllowlistExtension` avoids this problem for deposits because it checks `owner` — the LP-share recipient — which is a pool-enforced field tied to the economic actor: [6](#0-5) 

No equivalent pool-enforced binding exists for swaps; the swap extension has no way to recover the real user from the arguments it receives.

## Impact Explanation
A pool admin deploys a curated pool with `SwapAllowlistExtension` to restrict swaps to trusted counterparties (e.g., KYC'd addresses). To allow those trusted users to use the public router, the admin must call `setAllowedToSwap(pool, router, true)`. Once the router is allowlisted, **any** address can call `MetricOmmSimpleRouter.exact*` and the extension will see `sender = router` (allowed) rather than the real user (not allowed). The allowlist is completely bypassed. Disallowed users can drain LP liquidity at oracle-derived prices, execute arbitrage the pool was designed to prevent, or interact with pools that were meant to be access-controlled. This constitutes a direct loss of LP principal and a broken core pool access-control invariant.

## Likelihood Explanation
The trigger requires only a standard public call to `MetricOmmSimpleRouter`. No privileged access, flash loan, or special token behavior is needed. Any user who observes that a pool uses `SwapAllowlistExtension` and that the router is allowlisted can exploit this immediately. The router is a supported, documented periphery contract, so the admin is expected to allowlist it for legitimate users.

## Recommendation
`SwapAllowlistExtension.beforeSwap` must gate on the originating user, not on `sender`. Two options:

1. **Pass the real user through `extensionData`**: The router encodes the originating `msg.sender` into `extensionData` before forwarding to the pool; the extension decodes and checks it. This requires the extension to maintain a registry of trusted routers whose `extensionData` it accepts.

2. **Require the pool to expose a pool-enforced identity field for swaps**: Analogous to how `addLiquidity` enforces `owner`, the swap path could enforce a `swapper` field that the pool binds to `msg.sender` of the direct caller, preventing router substitution.

## Proof of Concept
```
Setup:
  pool configured with SwapAllowlistExtension
  admin calls setAllowedToSwap(pool, router, true)   // required for legitimate users
  admin calls setAllowedToSwap(pool, alice, true)
  // bob is NOT allowlisted

Attack (Bob bypasses allowlist):
  1. Bob calls MetricOmmSimpleRouter.exactInputSingle({pool: pool, ...})
  2. Router calls pool.swap(recipient=bob, ...)
     → pool's msg.sender = router
  3. Pool calls _beforeSwap(sender=router, ...)
  4. ExtensionCalling calls SwapAllowlistExtension.beforeSwap(sender=router, ...)
  5. Extension checks allowedSwapper[pool][router] → true → PASSES
  6. Swap executes; Bob receives output tokens from the curated pool

Expected: revert NotAllowedToSwap (bob is not allowlisted)
Actual:   swap succeeds because router is allowlisted and sender=router
```

Foundry test: deploy pool with `SwapAllowlistExtension`, allowlist only the router and alice, call `exactInputSingle` as bob, assert no revert and bob receives output tokens.
