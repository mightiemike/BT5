### Title
Linked Signer Can Unilaterally Rotate Itself via `LinkSigner` in Fast Mode — (`core/contracts/EndpointTx.sol`)

---

### Summary

In `processTransactionImpl`, the `LinkSigner` transaction type is validated with `allowLinkedSigner = true`. This means the currently linked signer — not just the subaccount owner — can sign a `LinkSigner` transaction to replace the linked signer with any arbitrary address. A compromised linked signer can permanently hijack the subaccount by rotating the linked signer to an attacker-controlled address, locking out the subaccount owner from the fast path for the full 3-day slow-mode delay window.
