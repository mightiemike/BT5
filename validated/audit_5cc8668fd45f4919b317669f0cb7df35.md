### Title
Bonus Rewards Permanently Lost When `BeforeRedelegationSlashed` Fires With Insufficient Pool — (`x/tieredrewards/keeper/slash.go`)

### Summary

In `x/tieredrewards`, when a validator slash triggers `BeforeRedelegationSlashed` and the bonus pool is insufficient, `slashRedelegationPosition` silently swallows the `ErrInsufficientBonusPool` error and then persists the position with already-advanced reward checkpoints. Because `processEventsAndClaimBonus` advances `LastBonusAccrual`, `LastEventSeq`, and decrements event reference counts (potentially garbage-collecting events) **before** checking pool sufficiency, the accrued bonus is permanently unrecoverable — unlike every user-driven path, which fails atomically and allows retry.

### Finding Description

`processEventsAndClaimBonus` in `x/tieredrewards/keeper/claim_rewards.go` performs the following operations in order:

1. Iterates all pending `ValidatorEvent`s since `pos.LastEventSeq`, calling `pos.UpdateLastEventSeq(entry.Seq)` and `k.decrementEventRefCount(...)` (which may garbage-collect events from the store) for each entry.
2. Calls `applyBonusAccrualCheckpoint(&pos.Position, blockTime)` — advancing `LastBonusAccrual` to the current block time.
3. Calls `pos.UpdateLastKnownBonded(bonded)`.
4. **Only then** calls `k.sufficientBonusPoolBalance(ctx, bonusCoins)` — returning `ErrInsufficientBonusPool` if the pool cannot cover the computed bonus. [1](#0-0) [2](#0-1) 

The `decrementEventRefCount` writes are committed to the KV store inside the loop, before the error is returned: [3](#0-2) 

In `slashRedelegationPosition`, the `ErrInsufficientBonusPool` error is explicitly caught and swallowed to avoid a chain halt: [4](#0-3) 

Execution then continues and `setPositionWithState` is called with the in-memory `pos` whose checkpoints have already been advanced by `processEventsAndClaimBonus`: [5](#0-4) 

The result: the position is persisted with `LastBonusAccrual = blockTime` and `LastEventSeq` advanced past the relevant events, while the events themselves may have been garbage-collected. The bonus is never paid and can never be reclaimed.

This is structurally distinct from all user-driven paths (`MsgClaimTierRewards`, `MsgTierRedelegate`, `MsgTierUndelegate`, `MsgAddToTierPosition`, `MsgClearPosition`), which fail atomically on `ErrInsufficientBonusPool` — leaving checkpoints unchanged so the user can retry after the pool is replenished: [6](#0-5) 

### Impact Explanation

The corrupted value is the position owner's accrued bonus reward balance. The amount permanently lost equals `shares × tokensPerShare × bonusApy × duration / SecondsPerYear` for the entire accrual period since the position's last `LastBonusAccrual` checkpoint. This is a direct, unrecoverable loss of funds owed to the user — the bonus pool retains the tokens that should have been paid, but the position's checkpoints have moved past the accrual window and the events needed to recompute it may be gone. [7](#0-6) 

### Likelihood Explanation

The trigger requires two concurrent conditions: (1) a tier position is in its redelegation period (i.e., `MsgTierRedelegate` was called and the 21-day unbonding window has not elapsed), and (2) the bonus pool balance is insufficient to cover the accrued bonus at the moment the source validator is slashed. Both are realistic in normal protocol operation — the pool can be drained by the `BeginBlocker` top-up mechanism or simply be underfunded, and redelegating positions are a normal user action. A malicious validator operator cannot directly trigger this, but any double-sign or downtime slash on a validator with redelegating tier positions while the pool is low will silently forfeit those users' bonus rewards. [8](#0-7) 

### Recommendation

Move the `sufficientBonusPoolBalance` check (and the `SendCoinsFromModuleToAccount` call) to **before** any in-memory checkpoint mutations and before any `decrementEventRefCount` store writes. If the pool is insufficient, return the error without modifying `pos` or the event store, so the state remains consistent and the bonus can be claimed once the pool is replenished.

Alternatively, separate the "compute and advance checkpoints" phase from the "pay" phase: compute the bonus amount first (read-only pass), check pool sufficiency, and only if sufficient proceed to advance checkpoints, decrement reference counts, and transfer funds.

### Proof of Concept

1. User calls `MsgTierRedelegate` on position P from validator V1 to V2. A `RedelegationMappings[unbondingId] = P.Id` entry is created.
2. The bonus pool is drained (e.g., by `BeginBlocker` top-ups or simply not replenished).
3. V1 is slashed. The Cosmos SDK staking module calls `BeforeRedelegationSlashed(ctx, unbondingId, sharesToUnbond)`.
4. `slashRedelegationPosition` is invoked. `processEventsAndClaimBonus` runs: it advances `pos.LastEventSeq`, decrements event reference counts (GC-ing events), advances `pos.LastBonusAccrual` to `blockTime`, then hits `sufficientBonusPoolBalance` → returns `ErrInsufficientBonusPool`.
5. `slashRedelegationPosition` logs the error and continues. `setPositionWithState` persists `pos` with the advanced checkpoints.
6. The user later calls `MsgClaimTierRewards` for position P. `processEventsAndClaimBonus` finds no events since the new `LastEventSeq` and computes zero bonus for the segment `[LastBonusAccrual, now]` (which starts at the slash block time, not the original accrual start). The pre-slash bonus is permanently lost. [9](#0-8) [10](#0-9)

func (k Keeper) processEventsAndClaimBonus(ctx context.Context, pos *types.PositionState) (sdk.Coins, error) {
	// Rewards should have been claimed before undelegation
	if !pos.IsDelegated() {
		return sdk.NewCoins(), nil
	}

	valAddr, err := sdk.ValAddressFromBech32(pos.Delegation.ValidatorAddress)
	if err != nil {
		return nil, err
	}

	events, err := k.getValidatorEventsSince(ctx, valAddr, pos.LastEventSeq)
	if err != nil {
		return nil, err
	}

	sdkCtx := sdk.UnwrapSDKContext(ctx)
	blockTime := sdkCtx.BlockTime()

	totalBonus := math.ZeroInt()
	// Use the persisted bonded state from the last replay, not a hardcoded default.
	// This prevents overpaying bonus for unbonded gaps between claims.
	bonded := pos.LastKnownBonded
	segmentStart := pos.LastBonusAccrual

	tier, err := k.getTier(ctx, pos.TierId)
	if err != nil {
		return nil, err
	}

	for _, entry := range events {
		evt := entry.Event

		if bonded {
			// Compute bonus for the bonded segment [segmentStart, eventTime]
			// using the snapshot rate at the event.
			bonus := k.computeSegmentBonus(*pos, tier, segmentStart, evt.Timestamp, evt.TokensPerShare)
			totalBonus = totalBonus.Add(bonus)
		}

		// Update bonded state based on event type.
		switch evt.EventType {
		case types.ValidatorEventType_VALIDATOR_EVENT_TYPE_UNBOND:
			bonded = false
		case types.ValidatorEventType_VALIDATOR_EVENT_TYPE_BOND:
			bonded = true
		case types.ValidatorEventType_VALIDATOR_EVENT_TYPE_SLASH:
			// Slash doesn't change bonded state.
		}

		segmentStart = evt.Timestamp
		pos.UpdateLastEventSeq(entry.Seq)

		// Decrement reference count.
		if err := k.decrementEventRefCount(ctx, valAddr, entry.Seq); err != nil {
			return nil, err
		}
	}

	val, err := k.stakingKeeper.GetValidator(ctx, valAddr)
	if err != nil {
		return nil, err
	}
	// Defensive: validator bond status check
	if bonded && val.IsBonded() {
		currentRate, err := k.getTokensPerShare(ctx, valAddr)
		if err != nil {
			return nil, err
		}
		bonus := k.computeSegmentBonus(*pos, tier, segmentStart, blockTime, currentRate)
		totalBonus = totalBonus.Add(bonus)
	}

	applyBonusAccrualCheckpoint(&pos.Position, blockTime)
	// Persist the bonded state so the next replay starts correctly.
	pos.UpdateLastKnownBonded(bonded)

	if totalBonus.IsZero() {
		return sdk.NewCoins(), nil
	}

	bondDenom, err := k.stakingKeeper.BondDenom(ctx)
	if err != nil {
		return nil, err
	}

	bonusCoins := sdk.NewCoins(sdk.NewCoin(bondDenom, totalBonus))

	if err := k.sufficientBonusPoolBalance(ctx, bonusCoins); err != nil {
		return nil, err
	}

	ownerAddr, err := sdk.AccAddressFromBech32(pos.Owner)
	if err != nil {
		return nil, errorsmod.Wrap(sdkerrors.ErrInvalidAddress, "invalid owner address")
	}

	if err := k.bankKeeper.SendCoinsFromModuleToAccount(ctx, types.RewardsPoolName, ownerAddr, bonusCoins); err != nil {
		return nil, err
	}

	if err := sdkCtx.EventManager().EmitTypedEvent(&types.EventBonusRewardsClaimed{
		PositionId: pos.Id,
		Owner:      pos.Owner,
		Rewards:    bonusCoins,
	}); err != nil {
		return nil, err
	}

	return bonusCoins, nil
}

// decrementEventRefCount decrements the reference count of a validator event.
// If the reference count reaches zero, the event is deleted.
func (k Keeper) decrementEventRefCount(ctx context.Context, valAddr sdk.ValAddress, seq uint64) error {
	key := collections.Join(valAddr, seq)
	event, err := k.ValidatorEvents.Get(ctx, key)
	if errors.Is(err, collections.ErrNotFound) {
		return nil // already cleaned up
	}
	if err != nil {
		return err
	}

	if event.ReferenceCount <= 1 {
		return k.ValidatorEvents.Remove(ctx, key)
	}

	event.ReferenceCount--
	return k.ValidatorEvents.Set(ctx, key, event)
}

func (k Keeper) slashRedelegationPosition(ctx context.Context, unbondingId uint64, sharesToUnbond math.LegacyDec) error {
	positionId, err := k.getRedelegationMapping(ctx, unbondingId)
	if errors.Is(err, collections.ErrNotFound) {
		return nil
	}
	if err != nil {
		return err
	}

	pos, err := k.getPositionState(ctx, positionId)
	if errors.Is(err, types.ErrPositionNotFound) {
		k.logger(ctx).Error("position not found during redelegation slash",
			"position_id", positionId,
			"unbonding_id", unbondingId,
			"error", err.Error(),
		)
		return nil
	}
	if err != nil {
		return err
	}

	if !pos.IsDelegated() {
		// Defensive
		k.logger(ctx).Error("delegation missing during BeforeRedelegationSlashed",
			"position_id", positionId,
			"unbonding_id", unbondingId,
			"shares_to_unbond", sharesToUnbond.String(),
		)
		return nil
	}

	dstValStr := pos.Delegation.ValidatorAddress

	// Settle bonus against PRE-slash shares.
	if _, err := k.processEventsAndClaimBonus(ctx, &pos); err != nil {
		// Deliberately forgo bonus rewards if pool is insufficient to prevent chain halt.
		if errors.Is(err, types.ErrInsufficientBonusPool) {
			k.logger(ctx).Error("insufficient bonus pool during redelegation slash",
				"position_id", pos.Id,
				"error", err.Error(),
			)
		} else {
			return err
		}
	}

	fullSlash := sharesToUnbond.GTE(pos.Delegation.Shares)

	if fullSlash {
		pos.Delegation = nil
		pos.ClearBonusCheckpoints()
		return k.setPositionWithState(ctx, pos, &ValidatorTransition{PreviousAddress: dstValStr})
	}
	// In-memory only: the persisted Position carries no share count, and the
	// live delegation will reflect the post-Unbond shares on the next read.
	// Update the local copy so any follow-up logic in this call sees consistent state.
	pos.Delegation.Shares = pos.Delegation.Shares.Sub(sharesToUnbond)
	return k.setPositionWithState(ctx, pos, nil)
