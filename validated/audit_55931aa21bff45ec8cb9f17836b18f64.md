### Title
`ValidResourceBounds` Protobuf Deserialization Uses Value-Based Heuristic That Silently Changes Variant, Causing Valid V3 Transactions to Be Rejected at P2P/Consensus Receivers — (`crates/apollo_protobuf/src/converters/transaction.rs`)

---

### Summary

The protobuf deserializer for `ValidResourceBounds` uses a value-based heuristic (`l1_data_gas.is_zero() && l2_gas.is_zero()`) to decide between the `L1Gas` and `AllResources` variants. A valid V3 transaction with `AllResourceBounds { l1_gas: X, l2_gas: 0, l1_data_gas: 0 }` accepted by the gateway is serialized to protobuf and then deserialized at P2P/consensus receivers as `ValidResourceBounds::L1Gas(X)`. The subsequent `TryFrom<InvokeTransactionV3> for RpcInvokeTransactionV3` conversion rejects any non-`AllResources` variant, causing the transaction to be dropped at every receiving node. The same variant flip also produces a divergent transaction hash preimage (the `AllResources` path includes the `L1_DATA_GAS` term; the `L1Gas` path does not), so any hash recomputed at the receiver would not match the hash committed by the proposer.

---

### Finding Description

**Root cause — `TryFrom<protobuf::ResourceBounds> for ValidResourceBounds`:** [1](#0-0) 

```rust
Ok(if l1_data_gas.is_zero() && l2_gas.is_zero() {
    ValidResourceBounds::L1Gas(l1_gas)
} else {
    ValidResourceBounds::AllResources(AllResourceBounds { l1_gas, l2_gas, l1_data_gas })
})
```

The condition `l1_data_gas.is_zero() && l2_gas.is_zero()` is purely value-based. There is no type tag in the protobuf wire format that distinguishes `L1Gas` from `AllResources` with zero secondary bounds. The codebase itself acknowledges this in the consensus test: [2](#0-1) 

> "If all the fields of `AllResources` are 0 upon serialization, then the deserialized value will be interpreted as the `L1Gas` variant."

**Downstream rejection — `TryFrom<InvokeTransactionV3> for RpcInvokeTransactionV3`:** [3](#0-2) 

Any `ValidResourceBounds` that is not `AllResources` causes this conversion to return `Err(StarknetApiError::OutOfRange)`, which the protobuf converter maps to `DEPRECATED_RESOURCE_BOUNDS_ERROR`: [4](#0-3) 

**Divergent hash preimage — `get_tip_resource_bounds_hash`:** [5](#0-4) 

- `AllResources` path: poseidon over `[tip, L1_GAS_packed, L2_GAS_packed, L1_DATA_GAS_packed]` (4 elements)
- `L1Gas` path: poseidon over `[tip, L1_GAS_packed, L2_GAS_packed]` (3 elements)

Even when `l2_gas = 0` and `l1_data_gas = 0`, the two hashes differ because the `L1_DATA_GAS` packed felt (`concat(L1_DATA_GAS_name, 0, 0)`) is a non-zero felt that changes the poseidon output.

**Gateway acceptance path:**

The gateway's stateless validator accepts the transaction if `max_possible_fee > 0` (satisfied by non-zero `l1_gas`) and `l2_gas.max_price_per_unit >= min_gas_price` (satisfied when `min_gas_price = 0` or `validate_resource_bounds = false`): [6](#0-5) 

The hash is then computed with the `AllResources` variant and stored in `InternalRpcTransaction.tx_hash`: [7](#0-6) 

**Consensus propagation path:**

When the proposer converts the internal transaction back to a `ConsensusTransaction` for broadcast, it goes through `convert_internal_consensus_tx_to_consensus_tx` → `convert_internal_rpc_tx_to_rpc_tx` → protobuf serialization → protobuf deserialization at validators → `L1Gas` variant → rejection: [8](#0-7) 

---

### Impact Explanation

A valid V3 invoke/declare/deploy-account transaction with `l2_gas = 0` and `l1_data_gas = 0` (but non-zero `l1_gas`) is:

1. Accepted by the gateway and assigned a canonical hash computed over the `AllResources` preimage.
2. Propagated to mempool peers and consensus validators via protobuf.
3. Rejected at every receiving node because the deserialized `L1Gas` variant fails `TryFrom<InvokeTransactionV3> for RpcInvokeTransactionV3`.

In the consensus path, a proposer that includes such a transaction causes validators to reject the entire transaction batch, degrading liveness. This matches the allowed impact: **"High. Mempool/gateway/RPC admission accepts invalid transactions or rejects valid transactions before sequencing."**

---

### Likelihood Explanation

The trigger requires `min_gas_price = 0` (or `validate_resource_bounds = false`) in the stateless validator config so that a transaction with `l2_gas.max_price_per_unit = 0` passes gateway admission. This is a non-default but reachable configuration (e.g., test/staging environments, or a node operator who disables resource-bound validation). The protobuf round-trip ambiguity is unconditional once the transaction reaches the wire.

---

### Recommendation

Replace the value-based heuristic in `TryFrom<protobuf::ResourceBounds> for ValidResourceBounds` with an explicit type tag in the protobuf schema (e.g., a `oneof` or a boolean `is_all_resources` field), so that `AllResources` with zero secondary bounds is preserved across the serialization boundary and never silently collapsed to `L1Gas`.

As an immediate mitigation, add a gateway-level check that rejects any V3 transaction where `l2_gas` is entirely zero (both `max_amount` and `max_price_per_unit`), since such a transaction cannot be propagated over P2P without variant corruption.

---

### Proof of Concept

```
1. Submit to gateway (min_gas_price = 0):
   RpcInvokeTransactionV3 {
     resource_bounds: AllResourceBounds {
       l1_gas:      { max_amount: 1000, max_price_per_unit: 1 },
       l2_gas:      { max_amount: 0,    max_price_per_unit: 0 },
       l1_data_gas: { max_amount: 0,    max_price_per_unit: 0 },
     },
     ...
   }

2. Gateway computes hash H using AllResources preimage
   (includes L1_DATA_GAS packed felt in poseidon input).
   Transaction stored as InternalRpcTransaction { tx_hash: H }.

3. Gateway propagates via P2P / consensus:
   AllResourceBounds serialized → protobuf::ResourceBounds {
     l1_gas: Some(1000, 1), l2_gas: Some(0, 0), l1_data_gas: Some(0, 0)
   }

4. Receiver deserializes:
   l1_data_gas.is_zero() && l2_gas.is_zero() == true
   → ValidResourceBounds::L1Gas({ max_amount: 1000, max_price_per_unit: 1 })

5. TryFrom<InvokeTransactionV3> for RpcInvokeTransactionV3:
   resource_bounds is L1Gas, not AllResources → Err(OutOfRange)
   → DEPRECATED_RESOURCE_BOUNDS_ERROR → transaction rejected.

6. Hash recomputed at receiver (if attempted) would use L1Gas preimage
   (omits L1_DATA_GAS packed felt) → H' ≠ H.
```

impl TryFrom<protobuf::ResourceBounds> for ValidResourceBounds {
    type Error = ProtobufConversionError;
    fn try_from(value: protobuf::ResourceBounds) -> Result<Self, Self::Error> {
        let Some(l1_gas) = value.l1_gas else {
            return Err(missing("ResourceBounds::l1_gas"));
        };
        let Some(l2_gas) = value.l2_gas else {
            return Err(missing("ResourceBounds::l2_gas"));

pub fn get_tip_resource_bounds_hash(
    resource_bounds: &ValidResourceBounds,
    tip: &Tip,
) -> Result<Felt, StarknetApiError> {
    let l1_resource_bounds = resource_bounds.get_l1_bounds();
    let l2_resource_bounds = resource_bounds.get_l2_bounds();

    // L1 and L2 gas bounds always exist.
    // Old V3 txs always have L2 gas bounds of zero, but they exist.
    let mut resource_felts = vec![
        get_concat_resource(&l1_resource_bounds, L1_GAS)?,
        get_concat_resource(&l2_resource_bounds, L2_GAS)?,
    ];

    // For new V3 txs, need to also hash the data gas bounds.
    resource_felts.extend(match resource_bounds {
        ValidResourceBounds::L1Gas(_) => vec![],
        ValidResourceBounds::AllResources(all_resources) => {
            vec![get_concat_resource(&all_resources.l1_data_gas, L1_DATA_GAS)?]
        }
    });

    Ok(HashChain::new().chain(&tip.0.into()).chain_iter(resource_felts.iter()).get_poseidon_hash())
}

        let resource_bounds = *tx.resource_bounds();

        } else if resource_bounds.l2_gas.max_amount.0 > self.config.max_l2_gas_amount {
