### Title
Duplicate Position IDs in `MsgClaimTierRewards` Allow Repeated Bonus Reward Extraction - (`File: x/tieredrewards/keeper/msg_server.go`)

### Summary

`ClaimTierRewards` pre-loads all requested positions into an in-memory slice before processing them. If the same position ID appears more than once in `msg.PositionIds`, each duplicate entry carries the same stale checkpoint state (`LastEventSeq`, `LastBonusAccrual`). The inner loop in `claimRewardsAndUpdatesPositions` then re-computes and re-pays the same bonus rewards for every duplicate entry, draining the `RewardsPoolName` module account.

### Finding Description

`ClaimTierRewards` in `msg_server.go` first collects all positions into a slice, then delegates to `claimRewardsAndUpdatesPositions`:

```go
// msg_server.go lines 434-451
positions := make([]types.PositionState, 0, len(msg.PositionIds))
for _, posId := range msg.PositionIds {
    pos, err := ms.getPositionState(ctx, posId)   // loaded BEFORE any claim
    ...
    positions = append(positions, pos)
}
totalBase, totalBonus, err := ms.claimRewardsAndUpdatesPositions(ctx, positions)
``` [1](#0-0) 

`claimRewardsAndUpdatesPositions` then iterates over the pre-loaded slice:

```go
// claim_rewards.go lines 110-134
for i := range positions {
    pos := &positions[i]
    ...
    bonus, err := k.processEventsAndClaimBonus(ctx, pos)   // uses pos.LastEventSeq
    ...
    if err := k.setPosition(ctx, pos.Position, nil); err != nil { ... }
}
``` [2](#0-1) 

`processEventsAndClaimBonus` uses `pos.LastEventSeq` and `pos.LastBonusAccrual` to determine which validator events to replay and how much bonus to pay:

```go
// claim_rewards.go lines 153-165
events, err := k.getValidatorEventsSince(ctx, valAddr, pos.LastEventSeq)
...
bonded := pos.LastKnownBonded
segmentStart := pos.LastBonusAccrual
``` [3](#0-2) 

After processing the first occurrence of position X, `setPosition` persists the updated checkpoints to the store. However, the second occurrence of position X in the `positions` slice still holds the **original** (pre-claim) values of `LastEventSeq` and `LastBonusAccrual`, because it was loaded before any processing occurred. When the loop reaches the second entry, `processEventsAndClaimBonus` re-queries the same validator events (which are still present in the store as long as their reference count has not reached zero) and re-pays the same bonus amount via `bankKeeper.SendCoinsFromModuleToAccount`. [4](#0-3) 

The base rewards path (`claimBaseRewards` → `distributionKeeper.WithdrawDelegationRewards`) is naturally idempotent and returns zero on the second call, so only **bonus rewards** are double-paid. [5](#0-4) 

### Impact Explanation

An attacker who owns a position with accrued bonus rewards can include the same position ID N times in a single `MsgClaimTierRewards` message and receive N × (accrued bonus) tokens from the `RewardsPoolName` module account. Repeated exploitation across multiple transactions can fully drain the bonus pool, permanently denying bonus rewards to all other legitimate position holders.

The corrupted value is the `RewardsPoolName` module account balance (bond-denom coins) and the per-position `LastEventSeq` / `LastBonusAccrual` invariant that is supposed to prevent re-payment.

### Likelihood Explanation

The entry path is a standard, unprivileged Cosmos SDK transaction (`MsgClaimTierRewards`) signed by any position owner. No special role, leaked key, or social engineering is required. The attacker only needs one position with any accrued bonus. The attack is repeatable across blocks.

### Recommendation

Deduplicate `msg.PositionIds` in `MsgClaimTierRewards.Validate()` (in `x/tieredrewards/types/msgs.go`) and/or re-read position state from the store inside each iteration of `claimRewardsAndUpdatesPositions` rather than operating on a pre-loaded slice. A simple guard in the loop is sufficient:

```go
// Inside claimRewardsAndUpdatesPositions, reload from store each iteration:
freshPos, err := k.getPositionState(ctx, pos.Id)
if err != nil { return nil, nil, err }
pos = &freshPos
```

Alternatively, enforce uniqueness of position IDs at message validation time.

### Proof of Concept

1. Attacker owns position ID `42` with accrued bonus rewards of `R` tokens.
2. Attacker broadcasts `MsgClaimTierRewards{Owner: attacker, PositionIds: [42, 42, 42, ...]}` (N repetitions).
3. `ClaimTierRewards` loads position `42` N times into the `positions` slice — all N copies carry the same stale `LastEventSeq` and `LastBonusAccrual`.
4. `claimRewardsAndUpdatesPositions` iterates N times. Each iteration calls `processEventsAndClaimBonus` with the stale checkpoints, re-queries the same validator events (ref count still > 0 if other positions share the same validator), computes bonus `R`, and calls `bankKeeper.SendCoinsFromModuleToAccount(RewardsPoolName, attacker, R)`.
5. Attacker receives `N × R` tokens. The `RewardsPoolName` module account is drained by `N × R`. [6](#0-5) [7](#0-6) [8](#0-7)

func (ms msgServer) ClaimTierRewards(ctx context.Context, msg *types.MsgClaimTierRewards) (*types.MsgClaimTierRewardsResponse, error) {
	if err := msg.Validate(); err != nil {
		return nil, err
	}

	positions := make([]types.PositionState, 0, len(msg.PositionIds))
	for _, posId := range msg.PositionIds {
		pos, err := ms.getPositionState(ctx, posId)
		if err != nil {
			return nil, err
		}

		if err := ms.validateClaimRewards(pos.Position, msg.Owner); err != nil {
			return nil, err
		}

		positions = append(positions, pos)
	}

	totalBase, totalBonus, err := ms.claimRewardsAndUpdatesPositions(ctx, positions)
	if err != nil {
		return nil, err
	}

	sdkCtx := sdk.UnwrapSDKContext(ctx)
	if err := sdkCtx.EventManager().EmitTypedEvent(&types.EventTierRewardsClaimed{
		Owner:        msg.Owner,
		PositionIds:  msg.PositionIds,
		BaseRewards:  totalBase,
		BonusRewards: totalBonus,
	}); err != nil {
		return nil, err
	}

	return &types.MsgClaimTierRewardsResponse{
		BaseRewards:  totalBase,
		BonusRewards: totalBonus,
		PositionIds:  msg.PositionIds,
	}, nil
}

func (k Keeper) claimBaseRewards(ctx context.Context, pos types.PositionState) (sdk.Coins, error) {
	if !pos.IsDelegated() {
		return sdk.NewCoins(), nil
	}
	valAddr, err := sdk.ValAddressFromBech32(pos.Delegation.ValidatorAddress)
	if err != nil {
		return nil, err
	}
	posDelAddr, err := sdk.AccAddressFromBech32(pos.DelegatorAddress)
	if err != nil {
		return nil, errorsmod.Wrap(sdkerrors.ErrInvalidAddress, "invalid delegator address")
	}
	rewards, err := k.distributionKeeper.WithdrawDelegationRewards(ctx, posDelAddr, valAddr)
	if err != nil {
		return nil, err
	}
	if rewards.IsZero() {
		return rewards, nil
	}
	sdkCtx := sdk.UnwrapSDKContext(ctx)
	if err := sdkCtx.EventManager().EmitTypedEvent(&types.EventBaseRewardsClaimed{
		PositionId: pos.Id,
		Owner:      pos.Owner,
		Rewards:    rewards,
	}); err != nil {
		return nil, err
	}
	return rewards, nil
}

func (k Keeper) claimRewardsAndUpdatesPositions(ctx context.Context, positions []types.PositionState) (sdk.Coins, sdk.Coins, error) {
	totalBase := sdk.NewCoins()
	totalBonus := sdk.NewCoins()

	for i := range positions {
		pos := &positions[i]

		if !pos.IsDelegated() {
			continue
		}

		base, err := k.claimBaseRewards(ctx, *pos)
		if err != nil {
			return nil, nil, err
		}
		totalBase = totalBase.Add(base...)

		bonus, err := k.processEventsAndClaimBonus(ctx, pos)
		if err != nil {
			return nil, nil, err
		}
		totalBonus = totalBonus.Add(bonus...)

		if err := k.setPosition(ctx, pos.Position, nil); err != nil {
			return nil, nil, err
		}
	}

	return totalBase, totalBonus, nil
}

func (k Keeper) processEventsAndClaimBonus(ctx context.Context, pos *types.PositionState) (sdk.Coins, error) {
	// Rewards should have been claimed before undelegation
	if !pos.IsDelegated() {
		return sdk.NewCoins(), nil
	}

	valAddr, err := sdk.ValAddressFromBech32(pos.Delegation.ValidatorAddress)
	if err != nil {
		return nil, err
	}

	events, err := k.getValidatorEventsSince(ctx, valAddr, pos.LastEventSeq)
	if err != nil {
		return nil, err
	}

	sdkCtx := sdk.UnwrapSDKContext(ctx)
	blockTime := sdkCtx.BlockTime()

	totalBonus := math.ZeroInt()
	// Use the persisted bonded state from the last replay, not a hardcoded default.
	// This prevents overpaying bonus for unbonded gaps between claims.
	bonded := pos.LastKnownBonded
	segmentStart := pos.LastBonusAccrual

	tier, err := k.getTier(ctx, pos.TierId)
	if err != nil {
		return nil, err
	}

	for _, entry := range events {
		evt := entry.Event

		if bonded {
			// Compute bonus for the bonded segment [segmentStart, eventTime]
			// using the snapshot rate at the event.
			bonus := k.computeSegmentBonus(*pos, tier, segmentStart, evt.Timestamp, evt.TokensPerShare)
			totalBonus = totalBonus.Add(bonus)
		}

		// Update bonded state based on event type.
		switch evt.EventType {
		case types.ValidatorEventType_VALIDATOR_EVENT_TYPE_UNBOND:
			bonded = false
		case types.ValidatorEventType_VALIDATOR_EVENT_TYPE_BOND:
			bonded = true
		case types.ValidatorEventType_VALIDATOR_EVENT_TYPE_SLASH:
			// Slash doesn't change bonded state.
		}

		segmentStart = evt.Timestamp
		pos.UpdateLastEventSeq(entry.Seq)

		// Decrement reference count.
		if err := k.decrementEventRefCount(ctx, valAddr, entry.Seq); err != nil {
			return nil, err
		}
	}

	val, err := k.stakingKeeper.GetValidator(ctx, valAddr)
	if err != nil {
		return nil, err
	}
	// Defensive: validator bond status check
	if bonded && val.IsBonded() {
		currentRate, err := k.getTokensPerShare(ctx, valAddr)
		if err != nil {
			return nil, err
		}
		bonus := k.computeSegmentBonus(*pos, tier, segmentStart, blockTime, currentRate)
		totalBonus = totalBonus.Add(bonus)
	}

	applyBonusAccrualCheckpoint(&pos.Position, blockTime)
	// Persist the bonded state so the next replay starts correctly.
	pos.UpdateLastKnownBonded(bonded)

	if totalBonus.IsZero() {
		return sdk.NewCoins(), nil
	}

	bondDenom, err := k.stakingKeeper.BondDenom(ctx)
	if err != nil {
		return nil, err
	}

	bonusCoins := sdk.NewCoins(sdk.NewCoin(bondDenom, totalBonus))

	if err := k.sufficientBonusPoolBalance(ctx, bonusCoins); err != nil {
		return nil, err
	}

	ownerAddr, err := sdk.AccAddressFromBech32(pos.Owner)
	if err != nil {
		return nil, errorsmod.Wrap(sdkerrors.ErrInvalidAddress, "invalid owner address")
	}

	if err := k.bankKeeper.SendCoinsFromModuleToAccount(ctx, types.RewardsPoolName, ownerAddr, bonusCoins); err != nil {
		return nil, err
	}

	if err := sdkCtx.EventManager().EmitTypedEvent(&types.EventBonusRewardsClaimed{
		PositionId: pos.Id,
		Owner:      pos.Owner,
		Rewards:    bonusCoins,
	}); err != nil {
		return nil, err
	}

	return bonusCoins, nil
}
