### Title
Unvalidated `oraclePriceX18` in `SignedMintNlp`/`SignedBurnNlp` Allows Arbitrary NLP Price Manipulation via Slow-Mode — (`core/contracts/Verifier.sol`, `core/contracts/Clearinghouse.sol`)

---

### Summary

The EIP-712 digest for `MintNlp` and `BurnNlp` transactions does not commit to the `oraclePriceX18` field. Because this field is excluded from the signed message, any user who submits a slow-mode `MintNlp` or `BurnNlp` transaction can freely set an arbitrary NLP price. This allows an attacker to mint NLP tokens at a near-zero price (receiving far more NLP than entitled) or burn NLP tokens at an inflated price (extracting far more quote than entitled), directly corrupting NLP accounting and draining the pool at the expense of other NLP holders.

---

### Finding Description

`SignedMintNlp` and `SignedBurnNlp` each carry an `oraclePriceX18` field alongside the user's signature:

```solidity
struct SignedMintNlp {
    MintNlp tx;
    bytes signature;
    int128 oraclePriceX18;       // ← not signed
    int128[] nlpPoolRebalanceX18; // ← not signed
}
``` [1](#0-0) 

The EIP-712 type strings and digest construction in `Verifier.computeDigest` only commit to `{sender, quoteAmount, nonce}` for `MintNlp` and `{sender, nlpAmount, nonce}` for `BurnNlp`:

```solidity
string internal constant MINT_NLP_SIGNATURE =
    "MintNlp(bytes32 sender,uint128 quoteAmount,uint64 nonce)";
string internal constant BURN_NLP_SIGNATURE =
    "BurnNlp(bytes32 sender,uint128 nlpAmount,uint64 nonce)";
``` [2](#0-1) 

The digest for `MintNlp` is computed as:

```solidity
digest = keccak256(abi.encode(
    keccak256(bytes(MINT_NLP_SIGNATURE)),
    signedTx.tx.sender,
    signedTx.tx.quoteAmount,
    signedTx.tx.nonce
    // oraclePriceX18 is absent
));
``` [3](#0-2) 

And for `BurnNlp`:

```solidity
digest = keccak256(abi.encode(
    keccak256(bytes(BURN_NLP_SIGNATURE)),
    signedTx.tx.sender,
    signedTx.tx.nlpAmount,
    signedTx.tx.nonce
    // oraclePriceX18 is absent
));
``` [4](#0-3) 

The `oraclePriceX18` is then used directly in `Clearinghouse.mintNlp` to determine how many NLP tokens the user receives:

```solidity
int128 nlpAmount = quoteAmount.div(oraclePriceX18);
``` [5](#0-4) 

And in `Clearinghouse.burnNlp` to determine how much quote the user receives:

```solidity
int128 quoteAmount = nlpAmount.mul(oraclePriceX18);
``` [6](#0-5) 

The price is also written into the global `priceX18` mapping and into the spot engine, affecting all subsequent health checks:

```solidity
priceX18[NLP_PRODUCT_ID] = signedTx.oraclePriceX18;
clearinghouse.mintNlp(signedTx.tx, signedTx.oraclePriceX18, nlpPools, signedTx.nlpPoolRebalanceX18);
``` [7](#0-6) 

**Attacker-controlled entry path — slow-mode transactions:**

`MintNlp` and `BurnNlp` are not in the admin-only transaction type list, so any user can submit them as slow-mode transactions. The slow-mode path stores the raw transaction bytes (which include the attacker-controlled `oraclePriceX18`) and later executes them via `processTransactionImpl`:

```solidity
slowModeTxs[_slowModeConfig.txCount++] = IEndpoint.SlowModeTx({
    executableAt: uint64(block.timestamp) + SLOW_MODE_TX_DELAY,
    sender: sender,
    tx: transaction
});
``` [8](#0-7) 

When executed, `processTransactionImpl` decodes the stored bytes and uses `signedTx.oraclePriceX18` directly — the value the attacker embedded — without any on-chain validation against a trusted price source.

---

### Impact Explanation

**Impact: High**

- **Mint path:** Attacker sets `oraclePriceX18` to `1` (minimum positive value). `nlpAmount = quoteAmount / 1` yields an astronomically large NLP balance for a small `quoteAmount`. This dilutes all existing NLP holders proportionally.
- **Burn path:** Attacker sets `oraclePriceX18` to `INT128_MAX`. `quoteAmount = nlpAmount * INT128_MAX` extracts the entire quote reserve of the NLP pool for a tiny `nlpAmount`, draining the pool.
- The corrupted `oraclePriceX18` is also written to `priceX18[NLP_PRODUCT_ID]` and `spotEngine`, poisoning all subsequent health calculations that depend on the NLP price until the next legitimate `UpdatePrice` transaction.

---

### Likelihood Explanation

**Likelihood: High**

The slow-mode path is a censorship-resistance mechanism explicitly designed to be accessible to any user without sequencer cooperation. No special privilege, flash loan, or external dependency is required — the attacker only needs to craft a valid `MintNlp` or `BurnNlp` transaction with a manipulated `oraclePriceX18` field and submit it via `submitSlowModeTransaction`. The 3-day delay before execution does not prevent the attack; it only delays it.

---

### Recommendation

Include `oraclePriceX18` in the EIP-712 type string and digest for both `MintNlp` and `BurnNlp`:

```solidity
string internal constant MINT_NLP_SIGNATURE =
    "MintNlp(bytes32 sender,uint128 quoteAmount,uint64 nonce,int128 oraclePriceX18)";
string internal constant BURN_NLP_SIGNATURE =
    "BurnNlp(bytes32 sender,uint128 nlpAmount,uint64 nonce,int128 oraclePriceX18)";
```

And update the corresponding `MintNlp`/`BurnNlp` inner structs to include `oraclePriceX18` so the user explicitly commits to the price at signing time. This ensures the sequencer cannot substitute a different price, and a slow-mode attacker cannot embed an arbitrary price in the transaction bytes.

---

### Proof of Concept

```solidity
function test_nlpPriceManipulation() public {
    // Attacker has a small amount of quote (e.g., 1 USDC = 1e6)
    uint128 quoteAmount = 1e6;

    // Attacker crafts a MintNlp transaction with oraclePriceX18 = 1 (near-zero price)
    IEndpoint.MintNlp memory mintTx = IEndpoint.MintNlp({
        sender: attackerSubaccount,
        quoteAmount: quoteAmount,
        nonce: 0
    });

    // Sign only {sender, quoteAmount, nonce} — oraclePriceX18 is NOT in the digest
    bytes memory sig = signMintNlp(mintTx);

    IEndpoint.SignedMintNlp memory signedMint = IEndpoint.SignedMintNlp({
        tx: mintTx,
        signature: sig,
        oraclePriceX18: 1,          // ← attacker-controlled, not validated
        nlpPoolRebalanceX18: ...
    });

    // Submit via slow-mode — no sequencer needed
    endpoint.submitSlowModeTransaction(abi.encodePacked(
        uint8(IEndpoint.TransactionType.MintNlp),
        abi.encode(signedMint)
    ));

    // After SLOW_MODE_TX_DELAY, execute:
    endpoint.executeSlowModeTransaction();

    // Attacker received nlpAmount = 1e6 / 1 = 1e6 NLP tokens
    // instead of the fair ~1 NLP token at a real price of ~1e6 per NLP
    // All existing NLP holders are massively diluted
}
```
