### Title
Permissionless `creditDeposit()` Allows Anyone to Force Token Deposits, Blocking Owner's Direct Token Recovery — (File: `core/contracts/DirectDepositV1.sol`)

---

### Summary

The `creditDeposit()` function in `DirectDepositV1.sol` has no access control. Any unprivileged caller can invoke it to force all spot-registered tokens held by the DDA into the protocol's hardcoded `subaccount`, permanently draining the DDA's on-chain balance. This blocks the DDA owner's ability to directly recover tokens via the `withdraw()` function and forces a mandatory 3-day slow-mode withdrawal delay.

---

### Finding Description

`creditDeposit()` at line 83 carries no access-control modifier. It iterates over every spot product ID returned by `spotEngine.getProductIds()`, approves the endpoint for the DDA's full token balance, and calls `depositCollateralWithReferral()` to push all tokens into the hardcoded `subaccount` set at construction. [1](#0-0) 

The `withdraw()` function at line 103 is `onlyOwner` and transfers tokens directly from the DDA to the owner. [2](#0-1) 

Because `creditDeposit()` has no guard, any address can call it at any time. If an attacker calls it before (or races against) the owner's `withdraw()` call, all tokens are deposited into the protocol, leaving the DDA with a zero balance. The owner's `withdraw()` then transfers nothing.

The tokens are now inside the protocol's subaccount. Recovery requires submitting a slow-mode `WithdrawCollateral` transaction and waiting the hardcoded `SLOW_MODE_TX_DELAY` (3 days). [3](#0-2) 

The `depositCollateralWithReferral()` path in `Endpoint.sol` pulls tokens from `msg.sender` (the DDA) via `handleDepositTransfer` only after the DDA has already approved the endpoint, confirming the token transfer is irreversible once `creditDeposit()` executes. [4](#0-3) 

---

### Impact Explanation

- **DoS of `withdraw()`**: The owner cannot directly recover tokens from the DDA; they are forcibly deposited into the protocol.
- **Mandatory 3-day delay**: Recovery requires a slow-mode withdrawal with a hardcoded 3-day delay, disrupting any time-sensitive operational need.
- **Affects all spot-registered tokens**: Every token registered as a spot product that the DDA holds is subject to this forced deposit.

---

### Likelihood Explanation

**Medium.** The attacker needs no special privilege, no capital, and no signature. They can call `creditDeposit()` proactively at any time without needing to front-run a specific transaction. The attack is trivially executable by any EOA or contract. The attacker gains no direct financial benefit, but the operational disruption is concrete and repeatable.

---

### Recommendation

Add an `onlyOwner` modifier to `creditDeposit()` to restrict invocation to the contract owner:

```solidity
function creditDeposit() external onlyOwner {
``` [5](#0-4) 

---

### Proof of Concept

1. DDA holds 100 USDC (a registered spot product).
2. Owner submits `withdraw(USDC)` to recover the tokens.
3. Attacker front-runs (or proactively calls) `creditDeposit()`.
4. `creditDeposit()` approves the endpoint for 100 USDC and calls `depositCollateralWithReferral()`, which pulls all 100 USDC from the DDA into the protocol's hardcoded `subaccount`.
5. Owner's `withdraw(USDC)` executes against a zero DDA balance — transfers nothing.
6. Owner must submit a slow-mode `WithdrawCollateral` transaction and wait 3 days to recover the tokens. [1](#0-0) [2](#0-1)

    function creditDeposit() external {
        uint32[] memory productIds = spotEngine.getProductIds();
        for (uint256 i = 0; i < productIds.length; i++) {
            uint32 productId = productIds[i];
            address tokenAddr = spotEngine.getToken(productId);
            require(tokenAddr != address(0), "Invalid productId.");
            IIERC20Base token = IIERC20Base(tokenAddr);
            uint256 balance = token.balanceOf(address(this));
            if (balance != 0) {
                token.approve(address(endpoint), balance);
                endpoint.depositCollateralWithReferral(
                    subaccount,
                    productId,
                    uint128(balance),
                    "-1"
                );
            }
        }
    }
```
