### Title
Unprotected `initialize()` Front-Run Grants Attacker Full Owner Control of Airdrop Contract — (`core/contracts/Airdrop.sol`)

---

### Summary

`Airdrop.initialize()` is `external` with no caller restriction beyond the `initializer` guard. On a proxy deployment, if the initialization transaction is submitted separately from the proxy deployment, any observer can front-run it, become the contract owner, and inject attacker-controlled `token` and `sanctions` addresses. All subsequent `registerMerkleRoot` and `claim` logic then executes under attacker-controlled state.

---

### Finding Description

`_disableInitializers()` in the constructor only locks the *implementation* contract's initializer slot. [1](#0-0) 

The *proxy* contract's storage is independent; its `initialize()` remains callable by the first external caller: [2](#0-1) 

There is no `msg.sender` check, no factory-enforced caller, and no `onlyOwner` guard — because `__Ownable_init()` is called *inside* `initialize()`, making the first caller the owner.

Once the attacker is owner:

1. **`registerMerkleRoot` is `onlyOwner`** — attacker registers an arbitrary Merkle root for week 1 containing `(attacker, type(uint256).max)`. [3](#0-2) 

2. **`_verifyProof` calls `ISanctionsList(sanctions).isSanctioned(sender)`** — attacker controls `sanctions`, so it returns `false`. [4](#0-3) 

3. **`_claim` calls `SafeERC20.safeTransfer(IERC20(token), msg.sender, totalAmount)`** — attacker controls `token`, so the transfer succeeds for any amount. [5](#0-4) 

4. **No balance check exists anywhere** — the contract never verifies it holds sufficient tokens before transferring. If real tokens are deposited after the takeover, the attacker can drain them entirely by registering a legitimate-looking Merkle root with themselves as the sole recipient. [6](#0-5) 

---

### Impact Explanation

- **Unauthorized owner control**: The attacker becomes the permanent owner of the Airdrop proxy, satisfying the Critical scope definition directly.
- **Fund seizure**: Any real protocol tokens deposited to the contract after the takeover can be drained via a crafted Merkle root + `claim()` call.
- **State corruption**: `claimed[week][attacker]` is set to `type(uint256).max`, permanently poisoning the claimed mapping for that week/address pair.
- **Denial of service to legitimate users**: The legitimate deployer's `initialize()` call reverts (`already initialized`), forcing a full redeployment.

---

### Likelihood Explanation

Front-running `initialize()` on a proxy is a well-known, mechanically straightforward attack. It requires only mempool visibility and a higher gas bid. No special privileges, leaked keys, or social engineering are needed. The attack window exists whenever proxy deployment and initialization are separate transactions, which is the standard Hardhat/OpenZeppelin upgrade pattern.

---

### Recommendation

Restrict `initialize()` so only a trusted deployer or factory can call it. The standard mitigations are:

- **Deploy + initialize atomically** in a single transaction (e.g., via a factory or deployment script that calls `deployProxy` which bundles both steps).
- **Add a deployer check**: pass the intended owner as a constructor argument to the proxy admin or use `OpenZeppelin`'s `TransparentUpgradeableProxy` with the initializer encoded in the constructor call data, ensuring no separate initialization transaction exists in the mempool.
- **Use `initialOwner` parameter**: OpenZeppelin ≥ 5.x `OwnableUpgradeable` accepts an `initialOwner` argument; pass the intended owner explicitly and validate `msg.sender == expectedDeployer` before calling `__Ownable_init`.

### Proof of Concept

```solidity
// SPDX-License-Identifier: MIT
pragma solidity ^0.8.0;

// 1. Deploy malicious token: transfer() always returns true, no balance check
contract MaliciousToken {
    function transfer(address, uint256) external pure returns (bool) { return true; }
    function balanceOf(address) external pure returns (uint256) { return type(uint256).max; }
}

// 2. Deploy malicious sanctions: isSanctioned() always returns false
contract MaliciousSanctions {
    function isSanctioned(address) external pure returns (bool) { return false; }
}

// 3. Front-run initialize() on the Airdrop proxy
// airdrop.initialize(address(maliciousToken), address(maliciousSanctions));
// → attacker is now owner

// 4. Build leaf: keccak256(bytes.concat(keccak256(abi.encode(attacker, type(uint256).max))))
// Build a single-leaf Merkle tree; root == leaf (trivial proof)

// 5. registerMerkleRoot(1, root)  ← onlyOwner, attacker passes

// 6. claim([{week:1, totalAmount:type(uint256).max, proof:[]}])
// _verifyProof passes all checks (malicious sanctions, valid proof)
// safeTransfer on malicious token succeeds
// claimed[1][attacker] == type(uint256).max  ✓

// 7. If real tokens were deposited: repeat with real token address set at initialize,
//    or transferOwnership + upgrade to drain balance.
```

The `claimed[1][attacker] == type(uint256).max` assertion holds. With real tokens deposited post-takeover, the asset delta is the full contract balance transferred to the attacker.
