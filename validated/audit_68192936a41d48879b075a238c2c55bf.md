### Title
Unchecked `transferFrom` Return Value Enables Free usdcE Drain — (`core/contracts/ContractOwner.sol`)

---

### Summary

`ContractOwner.replaceUsdcEWithUsdc` calls `IERC20Base(usdc).transferFrom(...)` without checking its return value. If the USDC token returns `false` instead of reverting on a failed transfer, the function continues execution, withdrawing usdcE from the `DirectDepositV1` contract and sending it to the caller — without the caller having paid any USDC.

---

### Finding Description

In `replaceUsdcEWithUsdc`, the function is intended to let a caller swap USDC for usdcE held in a `DirectDepositV1` address. The sequence is:

1. Read `balance` of usdcE held in `directDepositV1`
2. Pull `balance` USDC from `msg.sender` into `directDepositV1` via `transferFrom`
3. Withdraw usdcE from `directDepositV1` to `ContractOwner`
4. Send usdcE to `msg.sender` via `safeTransfer`

Step 2 uses a raw, unchecked `transferFrom`:

```solidity
IERC20Base(usdc).transferFrom(msg.sender, directDepositV1, balance);
``` [1](#0-0) 

The return value is silently discarded. Steps 3 and 4 execute unconditionally regardless of whether the USDC pull succeeded.

The protocol already defines a `safeTransferFrom` wrapper in `ERC20Helper` that properly checks the return value:

```solidity
require(
    success && (data.length == 0 || abi.decode(data, (bool))),
    ERR_TRANSFER_FAILED
);
``` [2](#0-1) 

`ContractOwner` already imports and uses `ERC20Helper` via `using ERC20Helper for IERC20Base` — the `safeTransfer` on line 618 uses it correctly — but `transferFrom` on line 616 bypasses it entirely. [3](#0-2) 

---

### Impact Explanation

If the USDC token at `0x2D270e6886d130D724215A266106e6832161EAEd` (Ink chain) returns `false` on a failed transfer rather than reverting, an attacker can:

- Call `replaceUsdcEWithUsdc(subaccount)` with zero USDC allowance or balance
- The `transferFrom` silently fails (returns `false`)
- `DirectDepositV1(directDepositV1).withdraw(IIERC20Base(usdcE))` executes, pulling all usdcE from the DDA into `ContractOwner`
- `IERC20Base(usdcE).safeTransfer(msg.sender, balance)` sends that usdcE to the attacker

The attacker receives the full usdcE balance of the targeted `directDepositV1` address at zero cost. This is a direct, concrete asset loss for the subaccount owner whose DDA holds usdcE. [4](#0-3) 

---

### Likelihood Explanation

The function has no access control modifier — it is callable by any unprivileged address on chain 57073 (Ink). [5](#0-4) 

The only precondition is that a `directDepositV1` address for the target subaccount exists and holds a nonzero usdcE balance. The attacker does not need any special role, key, or governance access.

---

### Recommendation

Replace the raw `transferFrom` call with the protocol's existing `safeTransferFrom` helper:

```solidity
// Before (unsafe):
IERC20Base(usdc).transferFrom(msg.sender, directDepositV1, balance);

// After (safe):
IERC20Base(usdc).safeTransferFrom(msg.sender, directDepositV1, balance);
```

`safeTransferFrom` is already defined in `ERC20Helper` and the `using ERC20Helper for IERC20Base` directive is already active in `ContractOwner`, so no additional imports are needed. [6](#0-5) 

---

### Proof of Concept

1. Identify a subaccount whose `directDepositV1` holds a nonzero usdcE balance.
2. Call `ContractOwner.replaceUsdcEWithUsdc(subaccount)` from an EOA with zero USDC allowance.
3. If USDC returns `false` on the failed `transferFrom`, execution continues.
4. `DirectDepositV1.withdraw(usdcE)` transfers all usdcE to `ContractOwner`.
5. `safeTransfer(msg.sender, balance)` sends all usdcE to the attacker.
6. Attacker receives usdcE without paying USDC; the subaccount's DDA is drained.

    function replaceUsdcEWithUsdc(bytes32 subaccount) external {
        require(block.chainid == 57073, ERR_UNAUTHORIZED);
        address payable directDepositV1 = directDepositV1Address[subaccount];
        require(directDepositV1 != address(0), "no dda");
        address usdcE = 0xF1815bd50389c46847f0Bda824eC8da914045D14;
        address usdc = 0x2D270e6886d130D724215A266106e6832161EAEd;
        uint256 balance = IERC20Base(usdcE).balanceOf(directDepositV1);
        if (balance > 0) {
            IERC20Base(usdc).transferFrom(msg.sender, directDepositV1, balance);
            DirectDepositV1(directDepositV1).withdraw(IIERC20Base(usdcE));
            IERC20Base(usdcE).safeTransfer(msg.sender, balance);
        }
    }
```

**File:** core/contracts/libraries/ERC20Helper.sol (L23-42)
```text
    function safeTransferFrom(
        IERC20Base self,
        address from,
        address to,
        uint256 amount
    ) internal {
        (bool success, bytes memory data) = address(self).call(
            abi.encodeWithSelector(
                IERC20Base.transferFrom.selector,
                from,
                to,
                amount
            )
        );

        require(
            success && (data.length == 0 || abi.decode(data, (bool))),
            ERR_TRANSFER_FAILED
        );
    }
```
