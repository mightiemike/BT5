### Title
Duplicate Position IDs in `ClaimTierRewards` Enable Double Bonus Reward Withdrawal - (File: x/tieredrewards/keeper/msg_server.go)

### Summary
`MsgClaimTierRewards` accepts a caller-supplied list of position IDs with no deduplication. All positions are loaded from state before any are processed. When the same ID appears twice, both copies carry the same stale `LastEventSeq`, causing `processEventsAndClaimBonus` to replay the same validator-event segment twice and transfer bonus coins from the `RewardsPoolName` module account to the owner twice.

### Finding Description

`ClaimTierRewards` in `msg_server.go` collects positions in a pre-processing loop and then hands the slice to `claimRewardsAndUpdatesPositions`: [1](#0-0) 

```go
positions := make([]types.PositionState, 0, len(msg.PositionIds))
for _, posId := range msg.PositionIds {
    pos, err := ms.getPositionState(ctx, posId)   // loaded from KV store
    ...
    positions = append(positions, pos)             // no duplicate check
}
totalBase, totalBonus, err := ms.claimRewardsAndUpdatesPositions(ctx, positions)
```

`claimRewardsAndUpdatesPositions` then iterates the slice, calling `processEventsAndClaimBonus` and `setPosition` for each element: [2](#0-1) 

When the same position ID appears at index `i` and index `j` in the slice:

1. **Iteration `i`**: `processEventsAndClaimBonus` walks all validator events since `pos.LastEventSeq`, computes bonus, calls `bankKeeper.SendCoinsFromModuleToAccount` to pay the owner, advances `pos.LastEventSeq` in memory, and calls `setPosition` to persist the updated checkpoint.

2. **Iteration `j`**: the copy at index `j` was snapshotted before iteration `i` ran. It still holds the **old** `LastEventSeq`. `getValidatorEventsSince` is called again with that stale cursor. If the validator's events have not been fully garbage-collected (their reference count is > 1 because other positions on the same validator still hold references), the same event segment is returned, the same bonus is computed, and `bankKeeper.SendCoinsFromModuleToAccount` transfers the bonus a second time. [3](#0-2) 

`claimBaseRewards` is not doubled because `distributionKeeper.WithdrawDelegationRewards` returns zero on the second call (rewards already withdrawn). The bonus path has no such idempotency guard. [4](#0-3) 

### Impact Explanation

The `RewardsPoolName` module account is drained by the duplicated `SendCoinsFromModuleToAccount` call. The attacker receives `N × bonus` for a single position by submitting the same ID `N` times. This directly reduces the pool available to all other participants and constitutes unauthorized asset movement from a shared module account.

**Impact: High**

### Likelihood Explanation

The attack requires only a standard `MsgClaimTierRewards` transaction with a repeated position ID — no privileged role, no leaked key, no special chain state. The only prerequisite for the bonus double-payment is that at least one other position exists on the same validator (so the event reference count stays above zero after the first processing pass). In any production deployment with multiple stakers on a validator this condition is trivially satisfied.

**Likelihood: High**

### Recommendation

Deduplicate `msg.PositionIds` before loading positions, or re-read each position from the KV store inside `claimRewardsAndUpdatesPositions` rather than operating on a pre-loaded snapshot. A minimal fix is to reject the message if any ID appears more than once:

```go
seen := make(map[uint64]struct{}, len(msg.PositionIds))
for _, posId := range msg.PositionIds {
    if _, dup := seen[posId]; dup {
        return nil, errorsmod.Wrapf(sdkerrors.ErrInvalidRequest, "duplicate position id %d", posId)
    }
    seen[posId] = struct{}{}
    ...
}
```

### Proof of Concept

1. Attacker owns position `P` on validator `V`. Several other positions also delegate to `V`, so validator events for `V` have reference count > 1.
2. Attacker submits `MsgClaimTierRewards{Owner: attacker, PositionIds: [P, P]}`.
3. The pre-load loop at lines 434–446 appends two identical `PositionState` copies (both with `LastEventSeq = S`) to the slice.
4. First iteration: `processEventsAndClaimBonus` processes events `[S+1 … latest]`, pays bonus `B`, advances the in-memory copy's `LastEventSeq`, calls `setPosition` (KV store now has `LastEventSeq = latest`).
5. Second iteration: the stale copy still has `LastEventSeq = S`. `getValidatorEventsSince(V, S)` returns the same events (ref count still > 0). Bonus `B` is paid again via `SendCoinsFromModuleToAccount`.
6. Attacker receives `2B` bonus; `RewardsPoolName` is debited `2B`. [5](#0-4) [6](#0-5)

func (ms msgServer) ClaimTierRewards(ctx context.Context, msg *types.MsgClaimTierRewards) (*types.MsgClaimTierRewardsResponse, error) {
	if err := msg.Validate(); err != nil {
		return nil, err
	}

	positions := make([]types.PositionState, 0, len(msg.PositionIds))
	for _, posId := range msg.PositionIds {
		pos, err := ms.getPositionState(ctx, posId)
		if err != nil {
			return nil, err
		}

		if err := ms.validateClaimRewards(pos.Position, msg.Owner); err != nil {
			return nil, err
		}

		positions = append(positions, pos)
	}

	totalBase, totalBonus, err := ms.claimRewardsAndUpdatesPositions(ctx, positions)
	if err != nil {
		return nil, err
	}

	sdkCtx := sdk.UnwrapSDKContext(ctx)
	if err := sdkCtx.EventManager().EmitTypedEvent(&types.EventTierRewardsClaimed{
		Owner:        msg.Owner,
		PositionIds:  msg.PositionIds,
		BaseRewards:  totalBase,
		BonusRewards: totalBonus,
	}); err != nil {
		return nil, err
	}

	return &types.MsgClaimTierRewardsResponse{
		BaseRewards:  totalBase,
		BonusRewards: totalBonus,
		PositionIds:  msg.PositionIds,
	}, nil
}

func (k Keeper) claimRewardsAndUpdatesPositions(ctx context.Context, positions []types.PositionState) (sdk.Coins, sdk.Coins, error) {
	totalBase := sdk.NewCoins()
	totalBonus := sdk.NewCoins()

	for i := range positions {
		pos := &positions[i]

		if !pos.IsDelegated() {
			continue
		}

		base, err := k.claimBaseRewards(ctx, *pos)
		if err != nil {
			return nil, nil, err
		}
		totalBase = totalBase.Add(base...)

		bonus, err := k.processEventsAndClaimBonus(ctx, pos)
		if err != nil {
			return nil, nil, err
		}
		totalBonus = totalBonus.Add(bonus...)

		if err := k.setPosition(ctx, pos.Position, nil); err != nil {
			return nil, nil, err
		}
	}

	return totalBase, totalBonus, nil
}
