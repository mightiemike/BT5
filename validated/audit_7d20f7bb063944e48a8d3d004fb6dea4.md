### Title
Protobuf `ValidResourceBounds` Deserialization Silently Downgrades `AllResources` to `L1Gas`, Producing a Different Transaction Hash Preimage - (File: crates/apollo_protobuf/src/converters/transaction.rs)

### Summary

`get_tip_resource_bounds_hash` in `crates/starknet_api/src/transaction_hash.rs` produces structurally different Poseidon hash preimages for `ValidResourceBounds::L1Gas` (2 resource felts) versus `ValidResourceBounds::AllResources` (3 resource felts). The protobuf deserializer in `crates/apollo_protobuf/src/converters/transaction.rs` silently converts any `AllResources { l1_gas: X, l2_gas: 0, l1_data_gas: 0 }` back to `L1Gas(X)`. A V3 transaction submitted with those bounds is hashed at the gateway under the `AllResources` path (H1), but after a protobuf round-trip in the P2P block-sync path the same transaction is reconstructed as `L1Gas` and re-hashed to a different value (H2 ≠ H1), breaking hash canonicalization across the pipeline.

### Finding Description

**Hash preimage divergence.** `get_tip_resource_bounds_hash` branches on the enum variant:

```
L1Gas(_)        → resource_felts = [L1_GAS_packed, L2_GAS_packed(0)]          // 2 elements
AllResources(_) → resource_felts = [L1_GAS_packed, L2_GAS_packed, L1_DATA_GAS_packed] // 3 elements
```

The final Poseidon hash is `Poseidon(tip, ...resource_felts)`. Because Poseidon is length-sensitive, hashing 3 elements ≠ hashing 4 elements even when the extra element is zero. [1](#0-0) 

**Protobuf deserializer collapses the variant.** When `l1_data_gas` and `l2_gas` are both zero the deserializer silently selects `L1Gas`:

```rust
Ok(if l1_data_gas.is_zero() && l2_gas.is_zero() {
    ValidResourceBounds::L1Gas(l1_gas)          // ← drops the AllResources context
} else {
    ValidResourceBounds::AllResources(...)
})
``` [2](#0-1) 

**Gateway always hashes under `AllResources`.** `InternalRpcInvokeTransactionV3` stores `resource_bounds: AllResourceBounds` and its `InvokeTransactionV3Trait` impl unconditionally wraps it as `ValidResourceBounds::AllResources(self.resource_bounds)` before calling `get_invoke_transaction_v3_hash`. So H1 is always a 4-element Poseidon. [3](#0-2) [4](#0-3) 

**P2P block-sync path uses `Transaction` / `ValidResourceBounds`.** The `Transaction::Invoke(InvokeTransaction::V3(InvokeTransactionV3))` type carries `resource_bounds: ValidResourceBounds`. When a committed block is propagated over P2P and deserialized from protobuf via `TryFrom<protobuf::ResourceBounds> for ValidResourceBounds`, a transaction with zero l2_gas and l1_data_gas is reconstructed as `L1Gas`. Any subsequent call to `get_transaction_hash` or `validate_transaction_hash` on that object computes H2 ≠ H1. [5](#0-4) 

**Gateway accepts the triggering input.** `validate_resource_bounds` only requires `max_possible_fee(Tip::ZERO) != 0` and `l2_gas.max_price_per_unit >= min_gas_price`. When `min_gas_price = 0` (a valid configuration), a transaction with `AllResourceBounds { l1_gas: {amount>0, price>0}, l2_gas: 0, l1_data_gas: 0 }` passes all checks and is admitted. [6](#0-5) 

The two-path incompatibility is structurally identical to the external report: one component (the gateway / `InternalRpcInvokeTransactionV3`) always uses the "AllResources" (3-resource) hash path, while the other component (the P2P sync deserializer) silently selects the "L1Gas" (2-resource) hash path for the same transaction, making the two hash values irreconcilable.

### Impact Explanation

Any syncing node that calls `validate_transaction_hash` on a protobuf-deserialized `Transaction` with zero l2_gas and l1_data_gas will compute H2 ≠ H1 and reject the block as invalid. Alternatively, if the hash is taken from the wire without re-verification, the stored `Transaction` object carries the wrong variant and any later RPC or execution path that re-derives the hash (e.g., for receipt lookup, event indexing, or signature domain binding) will produce the wrong value. This matches the **"Transaction conversion or signature/hash logic binds the wrong signer, hash, type, or executable payload"** impact.

### Likelihood Explanation

The trigger requires `min_gas_price = 0` in the gateway config and a user who submits a V3 transaction with non-zero l1_gas but zero l2_gas and l1_data_gas. Both conditions are realistic: `min_gas_price` is operator-configurable and zero is a natural default in non-production or early-stage deployments; users who only intend to pay L1 gas naturally set the other bounds to zero.

### Recommendation

Fix the protobuf deserializer to never produce `ValidResourceBounds::L1Gas` for transactions that arrived as V3 (which always carry three resource-bound slots). The simplest fix is to always return `AllResources` in `TryFrom<protobuf::ResourceBounds> for ValidResourceBounds` when all three fields are present, reserving `L1Gas` only for legacy wire formats that genuinely omit l2_gas and l1_data_gas. Alternatively, `get_tip_resource_bounds_hash` should be made variant-agnostic by always including the L1_DATA_GAS element (even as zero) so that `L1Gas` and `AllResources{..zero..}` produce identical preimages.

pub fn get_tip_resource_bounds_hash(
    resource_bounds: &ValidResourceBounds,
    tip: &Tip,
) -> Result<Felt, StarknetApiError> {
    let l1_resource_bounds = resource_bounds.get_l1_bounds();
    let l2_resource_bounds = resource_bounds.get_l2_bounds();

    // L1 and L2 gas bounds always exist.
    // Old V3 txs always have L2 gas bounds of zero, but they exist.
    let mut resource_felts = vec![
        get_concat_resource(&l1_resource_bounds, L1_GAS)?,
        get_concat_resource(&l2_resource_bounds, L2_GAS)?,
    ];

    // For new V3 txs, need to also hash the data gas bounds.
    resource_felts.extend(match resource_bounds {
        ValidResourceBounds::L1Gas(_) => vec![],
        ValidResourceBounds::AllResources(all_resources) => {
            vec![get_concat_resource(&all_resources.l1_data_gas, L1_DATA_GAS)?]
        }
    });

    Ok(HashChain::new().chain(&tip.0.into()).chain_iter(resource_felts.iter()).get_poseidon_hash())
}

pub(crate) fn get_invoke_transaction_v3_hash<T: InvokeTransactionV3Trait>(
    transaction: &T,
    chain_id: &ChainId,
    transaction_version: &TransactionVersion,
) -> Result<TransactionHash, StarknetApiError> {
    let tip_resource_bounds_hash =
        get_tip_resource_bounds_hash(&transaction.resource_bounds(), transaction.tip())?;
    let paymaster_data_hash =
        HashChain::new().chain_iter(transaction.paymaster_data().0.iter()).get_poseidon_hash();
    let data_availability_mode = concat_data_availability_mode(
        transaction.nonce_data_availability_mode(),
        transaction.fee_data_availability_mode(),
    );
    let account_deployment_data_hash = HashChain::new()
        .chain_iter(transaction.account_deployment_data().0.iter())
        .get_poseidon_hash();
    let calldata_hash =
        HashChain::new().chain_iter(transaction.calldata().0.iter()).get_poseidon_hash();
    let mut hash_chain = HashChain::new()
        .chain(&INVOKE)
        .chain(&transaction_version.0)
        .chain(transaction.sender_address().0.key())
        .chain(&tip_resource_bounds_hash)
        .chain(&paymaster_data_hash)
        .chain(&Felt::try_from(chain_id)?)
        .chain(&transaction.nonce().0)
        .chain(&data_availability_mode)
        .chain(&account_deployment_data_hash)
        .chain(&calldata_hash);
    if !transaction.proof_facts().0.is_empty() {
        let proof_facts_hash =
            HashChain::new().chain_iter(transaction.proof_facts().0.iter()).get_poseidon_hash();
        hash_chain = hash_chain.chain(&proof_facts_hash);
    }
    Ok(TransactionHash(hash_chain.get_poseidon_hash()))
}

impl TryFrom<protobuf::ResourceBounds> for ValidResourceBounds {
    type Error = ProtobufConversionError;
    fn try_from(value: protobuf::ResourceBounds) -> Result<Self, Self::Error> {
        let Some(l1_gas) = value.l1_gas else {
            return Err(missing("ResourceBounds::l1_gas"));
        };
        let Some(l2_gas) = value.l2_gas else {
            return Err(missing("ResourceBounds::l2_gas"));
        };
        // TODO(Shahak): Assert data gas is not none once we remove support for 0.13.2.
        let l1_data_gas = value.l1_data_gas.unwrap_or_default();
        let l1_gas: ResourceBounds = l1_gas.try_into()?;
        let l2_gas: ResourceBounds = l2_gas.try_into()?;
        let l1_data_gas: ResourceBounds = l1_data_gas.try_into()?;
        Ok(if l1_data_gas.is_zero() && l2_gas.is_zero() {
            ValidResourceBounds::L1Gas(l1_gas)
        } else {
            ValidResourceBounds::AllResources(AllResourceBounds { l1_gas, l2_gas, l1_data_gas })
        })
    }
