### Title
Duplicate Position ID in `MsgClaimTierRewards` Enables Double-Claiming Bonus Rewards from the Rewards Pool - (File: `x/tieredrewards/keeper/msg_server.go`)

---

### Summary

The `ClaimTierRewards` message handler in `x/tieredrewards` loads all requested positions from state **before** processing any of them, and performs no deduplication on the caller-supplied `PositionIds` list. A position owner can include the same position ID multiple times in a single `MsgClaimTierRewards` transaction, causing the bonus reward computation to execute repeatedly against the same stale in-memory snapshot, draining the `RewardsPoolName` module account.

---

### Finding Description

`ClaimTierRewards` in `msg_server.go` first collects all positions into a slice, then passes the slice to `claimRewardsAndUpdatesPositions`:

```go
// x/tieredrewards/keeper/msg_server.go:434-448
positions := make([]types.PositionState, 0, len(msg.PositionIds))
for _, posId := range msg.PositionIds {
    pos, err := ms.getPositionState(ctx, posId)
    ...
    positions = append(positions, pos)
}
totalBase, totalBonus, err := ms.claimRewardsAndUpdatesPositions(ctx, positions)
``` [1](#0-0) 

If the same `posId` appears twice, two copies of the same position state (identical `LastEventSeq`, `LastBonusAccrual`) are appended to the slice. The only validation performed is `validateClaimRewards`, which only checks ownership:

```go
// x/tieredrewards/keeper/msg_validate.go:175-181
func (k Keeper) validateClaimRewards(pos types.Position, owner string) error {
    if !pos.IsOwner(owner) {
        return types.ErrNotPositionOwner
    }
    return nil
}
``` [2](#0-1) 

Inside `claimRewardsAndUpdatesPositions`, each element of the slice is processed independently:

```go
// x/tieredrewards/keeper/claim_rewards.go:110-132
for i := range positions {
    pos := &positions[i]
    ...
    bonus, err := k.processEventsAndClaimBonus(ctx, pos)
    ...
    if err := k.setPosition(ctx, pos.Position, nil); err != nil { ... }
}
``` [3](#0-2) 

`processEventsAndClaimBonus` computes the "current segment" bonus unconditionally from `pos.LastBonusAccrual` to `blockTime` and immediately transfers funds:

```go
// x/tieredrewards/keeper/claim_rewards.go:206-213
if bonded && val.IsBonded() {
    currentRate, err := k.getTokensPerShare(ctx, valAddr)
    bonus := k.computeSegmentBonus(*pos, tier, segmentStart, blockTime, currentRate)
    totalBonus = totalBonus.Add(bonus)
}
``` [4](#0-3) 

```go
// x/tieredrewards/keeper/claim_rewards.go:239-240
if err := k.bankKeeper.SendCoinsFromModuleToAccount(ctx, types.RewardsPoolName, ownerAddr, bonusCoins); err != nil {
    return nil, err
}
``` [5](#0-4) 

Because the second copy of the position was loaded from state **before** the first copy was processed and written back via `setPosition`, its `LastBonusAccrual` and `LastEventSeq` are still the original values. The current-segment bonus is therefore computed and paid a second time from the same starting checkpoint.

For historical events, `decrementEventRefCount` silently no-ops on already-deleted events:

```go
// x/tieredrewards/keeper/validator_events.go:88-89
if errors.Is(err, collections.ErrNotFound) {
    return nil // already cleaned up
}
``` [6](#0-5) 

This means the second iteration skips deleted historical events but still pays the current-segment bonus in full, resulting in a double (or N-fold) payment.

---

### Impact Explanation

The `RewardsPoolName` module account holds a finite supply of staking bond tokens designated as bonus rewards. A position owner can drain this pool by submitting a single `MsgClaimTierRewards` transaction with the same position ID repeated N times. Each repetition pays the full current-segment bonus. The only limiting factor is the pool's current balance (checked by `sufficientBonusPoolBalance`), which caps each individual `SendCoinsFromModuleToAccount` call but does not prevent the N-1 extra payments that occur before the pool is exhausted. Other legitimate stakers lose their entitled bonus rewards permanently.

---

### Likelihood Explanation

Any position owner can trigger this with a standard, unprivileged `MsgClaimTierRewards` transaction by simply repeating a position ID in the `PositionIds` field. No special role, leaked key, or social engineering is required. The transaction is valid and passes all existing validation checks.

---

### Recommendation

Add a deduplication check in `ClaimTierRewards` before loading positions. For example, use a `map[uint64]struct{}` to detect and reject duplicate IDs in `msg.PositionIds`:

```go
seen := make(map[uint64]struct{}, len(msg.PositionIds))
for _, posId := range msg.PositionIds {
    if _, dup := seen[posId]; dup {
        return nil, errorsmod.Wrapf(types.ErrInvalidPositionID, "duplicate position id: %d", posId)
    }
    seen[posId] = struct{}{}
    ...
}
```

Alternatively, enforce uniqueness at the `MsgClaimTierRewards` `Validate()` level so the check is applied before the message reaches the keeper.

---

### Proof of Concept

1. Alice owns position ID `42` with an active delegation to a bonded validator. The position has accrued bonus rewards worth `R` tokens since `LastBonusAccrual`.
2. Alice submits `MsgClaimTierRewards{ Owner: alice, PositionIds: [42, 42, 42] }`.
3. The handler loads position `42` three times from state (all three copies have the same `LastBonusAccrual`).
4. `claimRewardsAndUpdatesPositions` iterates:
   - **Iteration 1**: computes bonus `R`, calls `SendCoinsFromModuleToAccount(RewardsPoolName, alice, R)`, updates `pos.LastBonusAccrual` in the local copy, calls `setPosition` (writes updated state).
   - **Iteration 2**: uses the second copy (original `LastBonusAccrual`), computes bonus `R` again, calls `SendCoinsFromModuleToAccount(RewardsPoolName, alice, R)` again.
   - **Iteration 3**: same as iteration 2, pays `R` a third time.
5. Alice receives `3R` tokens instead of `R`, draining `2R` extra from the rewards pool. [7](#0-6) [8](#0-7)

func (ms msgServer) ClaimTierRewards(ctx context.Context, msg *types.MsgClaimTierRewards) (*types.MsgClaimTierRewardsResponse, error) {
	if err := msg.Validate(); err != nil {
		return nil, err
	}

	positions := make([]types.PositionState, 0, len(msg.PositionIds))
	for _, posId := range msg.PositionIds {
		pos, err := ms.getPositionState(ctx, posId)
		if err != nil {
			return nil, err
		}

		if err := ms.validateClaimRewards(pos.Position, msg.Owner); err != nil {
			return nil, err
		}

		positions = append(positions, pos)
	}

	totalBase, totalBonus, err := ms.claimRewardsAndUpdatesPositions(ctx, positions)
	if err != nil {
		return nil, err
	}

	sdkCtx := sdk.UnwrapSDKContext(ctx)
	if err := sdkCtx.EventManager().EmitTypedEvent(&types.EventTierRewardsClaimed{
		Owner:        msg.Owner,
		PositionIds:  msg.PositionIds,
		BaseRewards:  totalBase,
		BonusRewards: totalBonus,
	}); err != nil {
		return nil, err
	}

	return &types.MsgClaimTierRewardsResponse{
		BaseRewards:  totalBase,
		BonusRewards: totalBonus,
		PositionIds:  msg.PositionIds,
	}, nil
}

func (k Keeper) claimRewardsAndUpdatesPositions(ctx context.Context, positions []types.PositionState) (sdk.Coins, sdk.Coins, error) {
	totalBase := sdk.NewCoins()
	totalBonus := sdk.NewCoins()

	for i := range positions {
		pos := &positions[i]

		if !pos.IsDelegated() {
			continue
		}

		base, err := k.claimBaseRewards(ctx, *pos)
		if err != nil {
			return nil, nil, err
		}
		totalBase = totalBase.Add(base...)

		bonus, err := k.processEventsAndClaimBonus(ctx, pos)
		if err != nil {
			return nil, nil, err
		}
		totalBonus = totalBonus.Add(bonus...)

		if err := k.setPosition(ctx, pos.Position, nil); err != nil {
			return nil, nil, err
		}
	}

	return totalBase, totalBonus, nil
}
```
