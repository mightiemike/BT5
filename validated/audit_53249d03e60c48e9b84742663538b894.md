Audit Report

## Title
SwapAllowlistExtension gates the router address instead of the actual swapper, enabling full allowlist bypass via MetricOmmSimpleRouter when the router is allowlisted — (File: metric-periphery/contracts/extensions/SwapAllowlistExtension.sol)

## Summary
`SwapAllowlistExtension.beforeSwap` receives `sender` from the pool, which is the pool's `msg.sender` — the direct caller of `pool.swap()`. When swaps are routed through `MetricOmmSimpleRouter`, the pool's `msg.sender` is the router contract, not the end user. If a pool admin allowlists the router address to enable router-mediated swaps, every unprivileged caller of the router can bypass the per-user allowlist gate entirely.

## Finding Description
**Root cause — `sender` is the immediate caller, not the end user.**

`MetricOmmPool.swap()` passes `msg.sender` as the `sender` argument to `_beforeSwap`:

```solidity
// metric-core/contracts/MetricOmmPool.sol:230-240
_beforeSwap(
    msg.sender,   // ← direct caller of pool.swap()
    recipient,
    ...
);
```

`ExtensionCalling._beforeSwap` forwards that value unchanged to the extension:

```solidity
// metric-core/contracts/ExtensionCalling.sol:160-176
_callExtensionsInOrder(
    BEFORE_SWAP_ORDER,
    abi.encodeCall(IMetricOmmExtensions.beforeSwap, (sender, ...))
);
```

`SwapAllowlistExtension.beforeSwap` then checks that `sender` is allowlisted for the calling pool (`msg.sender` = pool):

```solidity
// metric-periphery/contracts/extensions/SwapAllowlistExtension.sol:37
if (!allowAllSwappers[msg.sender] && !allowedSwapper[msg.sender][sender]) {
    revert IMetricOmmPoolActions.NotAllowedToSwap();
}
```

When `MetricOmmSimpleRouter.exactInputSingle` (or any `exact*` variant) is used, the router calls `pool.swap()` directly:

```solidity
// metric-periphery/contracts/MetricOmmSimpleRouter.sol:72-80
(int128 amount0Delta, int128 amount1Delta) = IMetricOmmPoolActions(params.pool)
    .swap(params.recipient, params.zeroForOne, ...);
```

So `sender` arriving at the extension is the **router address**, not the end user. The extension has no visibility into who called the router.

**Exploit path:**
1. Pool admin deploys a pool with `SwapAllowlistExtension` to restrict swapping to approved addresses.
2. Pool admin calls `setAllowedToSwap(pool, routerAddress, true)` — a natural step to permit allowlisted users to access the pool via the router.
3. Any unprivileged EOA calls `MetricOmmSimpleRouter.exactInputSingle({pool: restrictedPool, ...})`.
4. The router calls `pool.swap()`; the pool passes `router` as `sender` to the extension.
5. The extension evaluates `allowedSwapper[pool][router] == true` → passes.
6. The unprivileged user's swap executes against the restricted pool, bypassing the allowlist entirely.

**Why existing guards are insufficient:** There is no mechanism in the router or the pool to forward the original `msg.sender` (the end user) to the extension. The extension has no way to distinguish a legitimate allowlisted user routing through the router from an arbitrary caller routing through the same router.

## Impact Explanation
Any unprivileged trader can swap against a pool that is intended to be restricted to approved addresses, by routing through `MetricOmmSimpleRouter`. This breaks the core access-control invariant of `SwapAllowlistExtension` — that only explicitly approved addresses may swap. Depending on pool configuration (e.g., private liquidity pools, institutional pools, compliance-gated pools), this allows unauthorized parties to extract value from or interact with pools they should be excluded from, constituting a broken core pool functionality / admin-boundary break with potential direct loss of user principal or protocol integrity.

## Likelihood Explanation
The precondition is that the pool admin allowlists the router address. This is a natural and expected administrative action: without it, no user can swap via the router on an allowlist-gated pool. Any admin who wants to support router-mediated swaps for their allowlisted users will take this step. Once the router is allowlisted, the bypass is trivially repeatable by any unprivileged caller with zero additional preconditions.

## Recommendation
The extension should check the actual end user, not the intermediate caller. Two viable approaches:

1. **Pass the original initiator through the router:** Have the router encode the original `msg.sender` in `extensionData` and have the extension decode and check it. This requires a coordinated change between the router and the extension.
2. **Check `recipient` instead of (or in addition to) `sender`:** If the pool's design guarantees that the recipient is the economically relevant party, gate on `recipient`. However, this changes the semantics of the allowlist.
3. **Disallow router allowlisting at the extension level:** The extension could maintain a registry of known router addresses and reject any `sender` that is a router, forcing direct-pool-only access for allowlisted pools.

The cleanest fix is option 1: the router passes `msg.sender` (the end user) in a standardized field of `extensionData`, and `SwapAllowlistExtension.beforeSwap` decodes and checks that field when `sender` is a known router.

## Proof of Concept
```solidity
// Foundry test sketch
function test_allowlistBypassViaRouter() public {
    // Setup: pool with SwapAllowlistExtension, router allowlisted
    swapAllowlist.setAllowedToSwap(address(pool), address(router), true);

    // Attacker: unprivileged EOA, NOT individually allowlisted
    address attacker = makeAddr("attacker");
    assertFalse(swapAllowlist.isAllowedToSwap(address(pool), attacker));

    // Attacker routes through the router — swap succeeds despite not being allowlisted
    vm.prank(attacker);
    router.exactInputSingle(
        IMetricOmmSimpleRouter.ExactInputSingleParams({
            pool: address(pool),
            tokenIn: token0,
            recipient: attacker,
            zeroForOne: true,
            amountIn: 1e18,
            amountOutMinimum: 0,
            priceLimitX64: 0,
            deadline: block.timestamp,
            extensionData: ""
        })
    );
    // Swap completes — allowlist bypassed
}
```
