### Title
Gas exec-fee domain invariant break: `promise_batch_action_transfer_to_gas_key` uses wire key length instead of trie key length for ML-DSA-65 keys — (File: `runtime/near-vm-runner/src/logic/logic.rs`)

---

### Summary

The `promise_batch_action_transfer_to_gas_key` host function computes the per-byte exec fee using the raw `public_key_len` argument supplied by the contract (the borsh wire length of the key). For an ML-DSA-65 key that is 1953 bytes on the wire, the actual trie key is only 33 bytes (tag + SHA3-256 hash). The transaction-path fee helper correctly calls `action.public_key.trie_id_len()` = 33. The host-function path passes `public_key_len` = 1953. This produces a ~59× exec-fee overcharge for any contract-emitted `TransferToGasKey` action that carries an ML-DSA-65 key, breaking the gas-fee domain invariant that the same action costs the same gas regardless of submission path.

---

### Finding Description

**The invariant.** The design document for ML-DSA-65 integration explicitly states:

> "Every storage-stake and trie-byte-priced fee path was updated to call `trie_id_len()`."
> "Callers that still use `len()` for trie-storage costing will misprice ML-DSA-65 keys by ~1900 bytes."

`PublicKey::trie_id_len()` returns 33 for `MLDSA65` (tag + 32-byte SHA3-256 hash), while `PublicKey::len()` returns 1953 (tag + 1952-byte raw key). [1](#0-0) 

**Transaction path — correct.** `runtime/runtime/src/config.rs` passes `action.public_key.trie_id_len()` to both `gas_key_transfer_send_fee` and `gas_key_transfer_exec_fee`:

```rust
TransferToGasKey(action) => {
    gas_key_transfer_exec_fee(fees, receiver_id.len(), action.public_key.trie_id_len())
        .total()
}
``` [2](#0-1) 

**Host-function path — broken.** `VMLogic::promise_batch_action_transfer_to_gas_key` passes the raw `public_key_len` argument (the borsh wire length as supplied by the contract) to the same fee helpers:

```rust
let send = gas_key_transfer_send_fee(&self.fees_config, sir, public_key_len as usize);
let exec = gas_key_transfer_exec_fee(
    &self.fees_config,
    receiver_id.len(),
    public_key_len as usize,   // ← wire length, not trie_id_len
);
``` [3](#0-2) 

The same pattern appears in the wasmtime shim: [4](#0-3) 

**Exact divergent value.** `gas_key_transfer_exec_fee` computes:

```rust
let trie_key_len = access_key_key_len(account_id_len, public_key_len);
let per_byte = cfg.fee(ActionCosts::gas_key_byte).exec_fee()
    .checked_mul((trie_key_len + estimated_value_len) as u64)
    .unwrap();
``` [5](#0-4) 

For an ML-DSA-65 key:
- Transaction path: `public_key_len = 33` → `trie_key_len = 2 + account_id_len + 33`
- Host-function path: `public_key_len = 1953` → `trie_key_len = 2 + account_id_len + 1953`

Overcharge = `gas_key_byte_exec_rate × 1920` per action. The same overcharge applies to `gas_key_transfer_send_fee` (also called with `public_key_len`). [6](#0-5) 

The same bug is present in `promise_batch_action_add_gas_key_with_full_access`, which passes `public_key_len` to `gas_key_add_key_exec_fee` while the transaction path passes `public_key.trie_id_len()`: [7](#0-6) 

---

### Impact Explanation

Any contract that calls `promise_batch_action_transfer_to_gas_key` with an ML-DSA-65 key is charged ~1920 × `gas_key_byte` more gas than the identical action submitted as a transaction. This breaks the gas-fee domain invariant that the same action costs the same gas regardless of submission path. A contract that budgets gas based on the transaction-path fee will abort with out-of-gas when the same action is emitted via host function. Because the overcharge is proportional to the key-size difference (1920 bytes), it is deterministic and reproducible. The gas-fee parity test `test_gas_key_fee_parity_full_access` only exercises ed25519 keys and would not catch this divergence. [8](#0-7) 

**Impact: High** — broken gas-fee domain invariant; contract-emitted ML-DSA-65 gas-key transfers are systematically mispriced relative to the transaction path, causing predictable out-of-gas failures in contracts that rely on fee parity.

---

### Likelihood Explanation

Reachable whenever both `ProtocolFeature::GasKeys` and `ProtocolFeature::PostQuantumSignatures` (stabilized at protocol version 85) are active. Any unprivileged user can deploy a contract that calls `promise_batch_action_transfer_to_gas_key` with an ML-DSA-65 public key. The contract author need not be malicious; the overcharge fires on any honest use of the host function with a PQ key.

---

### Recommendation

After decoding the public key in `promise_batch_action_transfer_to_gas_key` (and `promise_batch_action_add_gas_key_with_full_access`), derive the fee-sizing length from the decoded key's `trie_id_len()` rather than from the raw `public_key_len` argument:

```rust
let decoded_pk = public_key.decode()?;
let pk_fee_len = decoded_pk.trie_id_len();   // 33 for ML-DSA-65, wire len for ed25519/secp256k1
let send = gas_key_transfer_send_fee(&self.fees_config, sir, pk_fee_len);
let exec = gas_key_transfer_exec_fee(&self.fees_config, receiver_id.len(), pk_fee_len);
```

This mirrors the transaction-path convention established in `runtime/runtime/src/config.rs` and documented in the ML-DSA-65 design notes. [9](#0-8) 

---

### Proof of Concept

1. Enable both `GasKeys` and `PostQuantumSignatures` on a test network.
2. Add an ML-DSA-65 full-access key to account `alice`.
3. Deploy a contract on `alice` that calls `promise_batch_action_transfer_to_gas_key` targeting `alice`'s ML-DSA-65 gas key with a small deposit.
4. Submit the identical `TransferToGasKey` action as a direct transaction signed by `alice`.
5. Compare `gas_burnt` on the action receipt from step 3 vs. step 4.

**Expected (correct):** both paths burn the same gas.
**Actual (broken):** the host-function path burns approximately `gas_key_byte_exec_rate × 1920 + gas_key_byte_send_rate × 1920` more gas than the transaction path, because `public_key_len = 1953` is used instead of `trie_id_len() = 33`.

The existing parity test at `test-loop-tests/src/tests/gas_keys.rs::test_gas_key_fee_parity_full_access` would expose this divergence if extended to use `KeyType::MLDSA65` instead of `KeyType::ED25519`. [10](#0-9)

pub fn gas_key_transfer_exec_fee(
    cfg: &RuntimeFeesConfig,
    account_id_len: usize,
    public_key_len: usize,
) -> GasKeyTransferFee {
    let base = cfg.fee(ActionCosts::gas_key_transfer_base).exec_fee();
    let trie_key_len = access_key_key_len(account_id_len, public_key_len);
    let estimated_value_len = AccessKey::min_gas_key_borsh_len();
    let per_byte = cfg
        .fee(ActionCosts::gas_key_byte)
        .exec_fee()
        .checked_mul((trie_key_len + estimated_value_len) as u64)
        .unwrap();
    GasKeyTransferFee { base, per_byte }
}
```
