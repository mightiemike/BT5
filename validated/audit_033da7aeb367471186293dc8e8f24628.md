### Title
Unchecked `fee_proposal_fri` During Startup Window Poisons `fee_actual` Floor, Inflating Minimum L2 Gas Price - (File: crates/apollo_consensus_orchestrator/src/validate_proposal.rs)

---

### Summary

`is_proposal_init_valid` skips all bounds enforcement on the proposer-supplied `fee_proposal_fri` field whenever `fee_actual` is `None` — which is the case for every block during the first `fee_proposal_window_size` (10) blocks. A malicious proposer who controls a majority of those startup blocks can insert arbitrarily large `fee_proposal_fri` values that are accepted by every honest validator, permanently seeding the sliding-window median with inflated data. Once the window fills, `fee_actual` is computed from those poisoned values and becomes the hard floor for the minimum L2 gas price, causing all subsequent transactions to be priced against an inflated baseline and potentially causing valid transactions to be rejected.

---

### Finding Description

In `is_proposal_init_valid`, the `fee_proposal_fri` bounds check is guarded by a pattern-match that fires only when `proposal_init_validation.fee_actual` is `Some`:

```rust
// Validate fee_proposal is within the configured margin of fee_actual.
// During initiation (fee_actual is None, <window_size blocks), bounds are not enforced.
if let (Some(fee_actual), Some(fee_proposal)) =
    (proposal_init_validation.fee_actual, init_proposed.fee_proposal_fri)
{
    let (lower_bound, upper_bound) = fee_proposal_bounds(
        fee_actual,
        VersionedConstants::latest_constants().fee_proposal_margin_ppt,
    );
    if fee_proposal.0 < lower_bound || fee_proposal.0 > upper_bound {
        return Err(...);
    }
}
``` [1](#0-0) 

`fee_actual` is computed by `compute_fee_actual`, which returns `None` whenever `height < window_size` (i.e., for the first 10 blocks):

```rust
let Some(start) = height.0.checked_sub(window_size) else {
    warn!("Cannot compute fee_actual for height {height}: height is below window_size ({window_size})");
    return None;
};
``` [2](#0-1) 

`window_size` is `fee_proposal_window_size = 10` across all deployed versioned-constants files: [3](#0-2) 

During those 10 blocks, every `fee_proposal_fri` value — including `u128::MAX` — is accepted by honest validators without any check. The accepted value is then recorded into `fee_proposals_window`:

```rust
fn record_fee_proposal(&mut self, height: BlockNumber, fee_proposal_fri: Option<GasPrice>) {
    self.fee_proposals_window.insert(height, fee_proposal_fri);
}
``` [4](#0-3) 

After block 10, `compute_fee_actual` computes the **median** of the 10 recorded values. If a malicious proposer controlled ≥ 6 of those 10 blocks and set `fee_proposal_fri` to an extreme value, the median is inflated. That inflated `fee_actual` is then passed as the floor to `calculate_next_l2_gas_price_for_fin`:

```rust
let effective_min = match fee_actual {
    Some(fa) => GasPrice(max(config_min.0, fa.0)),
    None => config_min,
};
calculate_next_base_gas_price(current_l2_gas_price, l2_gas_used, gas_target, effective_min)
``` [5](#0-4) 

Because `effective_min` is a hard floor in the EIP-1559 adjustment, the L2 gas price can never fall below the poisoned `fee_actual`. The inflated values are also persisted in the block header (`fee_proposal_fri` field) and re-read by `initialize_fee_proposals_window` on node restart, making the effect durable across restarts: [6](#0-5) 

---

### Impact Explanation

Once `fee_actual` is inflated, `calculate_next_l2_gas_price_for_fin` enforces it as the minimum L2 gas price for every subsequent block. The gateway's stateful validator rejects transactions whose `max_price_per_unit` is below the current L2 gas price. An inflated floor therefore causes valid, correctly-priced transactions to be rejected at the mempool/gateway admission layer, and causes fee estimation (`starknet_estimateFee`) to return inflated values. This matches the **High** impact category: "Mempool/gateway/RPC admission accepts invalid transactions or rejects valid transactions before sequencing" and "RPC execution, fee estimation … returns an authoritative-looking wrong value."

---

### Likelihood Explanation

The attack requires a single entity to be the block proposer for ≥ 6 of the first 10 blocks. In a BFT system with standard 1/3 Byzantine tolerance, a single adversary controls at most ~3 proposer slots, which is insufficient to skew the median (the existing test `test_compute_fee_actual_lone_adversary_cannot_skew_median` confirms this). However, likelihood rises in:

- **New or permissioned networks** where a small set of validators controls the initial proposer rotation.
- **Stake-weighted proposer selection** where a large-stake validator is disproportionately selected during the first 10 blocks.
- **Coordinated multi-party attacks** where colluding validators together hold > 50% of proposer slots.

The window is only 10 blocks, so the attack surface is narrow but permanently consequential once exploited.

---

### Recommendation

Add a sanity-bound check on `fee_proposal_fri` that applies even when `fee_actual` is `None`. A reasonable bound is `[min_gas_price, some_configured_max_fee_proposal]`. For example:

```rust
if let Some(fee_proposal) = init_proposed.fee_proposal_fri {
    let min_bound = VersionedConstants::latest_constants().min_gas_price.0;
    let max_bound = /* configurable ceiling, e.g. 1000× min_gas_price */;
    if fee_proposal.0 < min_bound || fee_proposal.0 > max_bound {
        return Err(ValidateProposalError::InvalidProposalInit(
            init_proposed.clone(),
            proposal_init_validation.clone(),
            format!("fee_proposal_fri={} out of absolute bounds [{min_bound}, {max_bound}]",
                    fee_proposal.0),
        ));
    }
    // existing margin check against fee_actual when available
    if let Some(fee_actual) = proposal_init_validation.fee_actual { ... }
}
```

This mirrors the external report's recommendation to add a sanity check (`ltv <= 100`) independent of the reference value, so that even during the startup phase the field cannot carry an unbounded value into the sliding window.

---

### Proof of Concept

1. Deploy a fresh Starknet network (height 0). `fee_proposals_window` is empty; `fee_actual` is `None` for heights 0–9.
2. For each of heights 0–9, act as the block proposer and set `fee_proposal_fri = u128::MAX` in `ProposalInit`.
3. Honest validators call `is_proposal_init_valid`. Because `proposal_init_validation.fee_actual` is `None`, the `if let (Some(fee_actual), Some(fee_proposal))` guard does not fire. The proposal is accepted.
4. At height 10, `compute_fee_actual` is called with the window `{0: u128::MAX, 1: u128::MAX, …, 9: u128::MAX}`. The median is `u128::MAX`.
5. `calculate_next_l2_gas_price_for_fin` sets `effective_min = u128::MAX`. The L2 gas price is permanently pinned at `u128::MAX`.
6. Every subsequent transaction submitted to the gateway has `max_price_per_unit < u128::MAX` and is rejected by the stateful validator's resource-bounds check, effectively halting the network. [1](#0-0) [7](#0-6) [8](#0-7)

```rust
pub fn compute_fee_actual(
    fee_proposals_window: &BTreeMap<BlockNumber, Option<GasPrice>>,
    height: BlockNumber,
    window_size: u64,
) -> Option<GasPrice> {
    let Some(start) = height.0.checked_sub(window_size) else {
        warn!(
            "Cannot compute fee_actual for height {height}: height is below window_size \
             ({window_size})"
        );
        return None;
    };
    let window_size_usize = usize::try_from(window_size).expect("window_size fits in usize");
    let mut window = Vec::with_capacity(window_size_usize);
    for source_height in (start..height.0).map(BlockNumber) {
        match fee_proposals_window.get(&source_height) {
            Some(Some(price)) => window.push(*price),
            Some(None) | None => {
                warn!(
                    "Cannot compute fee_actual for height {height}: fee_proposals_window has no \
                     recorded fee_proposal for height {source_height}"
                );
                return None;
            }
        }
    }
    window.sort();
    let mid = window_size_usize / 2;
    let median = if window_size_usize.is_multiple_of(2) {
        // Even: average of the two middle values, rounded down.
        // Overflow-safe averaging: a + (b - a) / 2 (safe because sorted, so b >= a).
        GasPrice(window[mid - 1].0 + (window[mid].0 - window[mid - 1].0) / 2)
    } else {
        window[mid]
    };
    Some(median)
}

/// Compute the next L2 gas price (for the fin or for updating state). Respects override when set.
pub fn calculate_next_l2_gas_price_for_fin(
    current_l2_gas_price: GasPrice,
    height: BlockNumber,
    l2_gas_used: GasAmount,
    override_l2_gas_price_fri: Option<u128>,
    min_l2_gas_price_per_height: &[PricePerHeight],
    fee_actual: Option<GasPrice>,
) -> GasPrice {
    if let Some(override_value) = override_l2_gas_price_fri {
        info!(
            "L2 gas price ({}) is not updated, remains on override value of {override_value} fri",
            current_l2_gas_price.0
        );
        return GasPrice(override_value);
    }
    let gas_target = VersionedConstants::latest_constants().gas_target;
    let config_min = get_min_gas_price_for_height(height, min_l2_gas_price_per_height);
    let effective_min = match fee_actual {
        Some(fa) => GasPrice(max(config_min.0, fa.0)),
        None => config_min,
    };
    calculate_next_base_gas_price(current_l2_gas_price, l2_gas_used, gas_target, effective_min)
}
