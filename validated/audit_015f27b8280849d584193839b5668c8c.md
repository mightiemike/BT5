### Title
`LinkSigner` Does Not Check If the Linked Signer Is Sanctioned, Allowing Sanctioned Addresses to Sign Protocol Transactions — (`File: core/contracts/EndpointTx.sol`)

---

### Summary

The Nado protocol enforces a sanctions check (`requireUnsanctioned`) on depositors and slow-mode submitters, but this check is never applied to the `signer` address in `LinkSigner` transactions. A non-sanctioned user can link a sanctioned address as their subaccount's authorized signer. Once linked, the sanctioned address can sign orders, withdrawals, and other protocol transactions on behalf of the subaccount, directly bypassing the sanctions enforcement mechanism.

---

### Finding Description

The protocol's sanctions enforcement is applied in two places:

1. `depositCollateralWithReferral` in `Endpoint.sol` checks both `msg.sender` and the subaccount owner: [1](#0-0) 

2. `submitSlowModeTransactionImpl` in `EndpointTx.sol` checks the slow-mode submitter (`msg.sender`): [2](#0-1) 

However, neither the fast-path `LinkSigner` handler in `processTransactionImpl` nor the slow-path handler in `processSlowModeTransactionImpl` checks whether the `signer` address being linked is sanctioned.

**Fast path (`processTransactionImpl`):** [3](#0-2) 

**Slow path (`processSlowModeTransactionImpl`):** [4](#0-3) 

In both cases, `linkedSigners[txn.sender]` is set to the signer address with no sanctions validation on that address. [5](#0-4) 

The `linkedSigners` mapping is then consumed by `getLinkedSigner` and `getLinkedSignerOrNlpSigner`, which are used to authorize signatures for `MatchOrders`, `WithdrawCollateral`, `MintNlp`, `BurnNlp`, `TransferQuote`, and other operations: [6](#0-5) 

---

### Impact Explanation

A sanctioned address, once linked as a signer, can sign any transaction type that `allowLinkedSigner = true` is passed for — including `WithdrawCollateral`, `MatchOrders`, `MintNlp`, `BurnNlp`, and `TransferQuote`. This allows a sanctioned party to actively trade, withdraw funds, and move collateral through a proxy subaccount, directly defeating the protocol's sanctions enforcement. The invariant that sanctioned addresses cannot interact with the protocol is broken. [7](#0-6) 

---

### Likelihood Explanation

The slow-mode path is a permissionless, censorship-resistant entry point available to any user. A non-sanctioned user can submit a `LinkSigner` slow-mode transaction at any time, linking any sanctioned address. The `requireUnsanctioned` check on `msg.sender` passes because the submitter is not sanctioned. After the `SLOW_MODE_TX_DELAY`, the transaction executes and the sanctioned address is permanently linked. No privileged access is required. [8](#0-7) 

---

### Recommendation

Add a `requireUnsanctioned` check on the `signer` address in both `LinkSigner` handlers:

**In `processTransactionImpl`:**
```solidity
} else if (txType == IEndpoint.TransactionType.LinkSigner) {
    IEndpoint.SignedLinkSigner memory signedTx = abi.decode(
        transaction[1:],
        (IEndpoint.SignedLinkSigner)
    );
    validateSignedTx(...);
+   requireUnsanctioned(address(uint160(bytes20(signedTx.tx.signer))));
    linkedSigners[signedTx.tx.sender] = address(uint160(bytes20(signedTx.tx.signer)));
}
```

**In `processSlowModeTransactionImpl`:**
```solidity
} else if (txType == IEndpoint.TransactionType.LinkSigner) {
    ...
    validateSender(txn.sender, sender);
    requireSubaccount(txn.sender);
+   requireUnsanctioned(address(uint160(bytes20(txn.signer))));
    linkedSigners[txn.sender] = address(uint160(bytes20(txn.signer)));
}
```

---

### Proof of Concept

1. Sanctioned address `S` is on the `ISanctionsList`.
2. Non-sanctioned user `U` owns subaccount `sub = bytes32(abi.encodePacked(U, label))`.
3. `U` calls `submitSlowModeTransaction` with a `LinkSigner` payload: `{sender: sub, signer: bytes32(uint256(uint160(S))), nonce: 0}`.
4. `submitSlowModeTransactionImpl` runs: `requireUnsanctioned(U)` passes. The transaction is queued. [9](#0-8) 
5. After `SLOW_MODE_TX_DELAY`, anyone calls `executeSlowModeTransaction`.
6. `processSlowModeTransactionImpl` reaches the `LinkSigner` branch and sets `linkedSigners[sub] = S` with no sanctions check. [4](#0-3) 
7. `S` can now sign a `WithdrawCollateral` or `MatchOrders` transaction for `sub`. `getLinkedSigner(sub)` returns `S`, and `validateSignature` / `validateCompactSignature` accepts `S`'s signature as valid. [10](#0-9) 
8. The sanctioned address `S` has successfully interacted with the protocol, bypassing the sanctions enforcement.

```text
    function submitSlowModeTransactionImpl(bytes calldata transaction) public {
        IEndpoint.TransactionType txType = IEndpoint.TransactionType(
            uint8(transaction[0])
        );

        // special case for DepositCollateral because upon
        // slow mode submission we must take custody of the
        // actual funds

        address sender = msg.sender;

        if (txType == IEndpoint.TransactionType.DepositCollateral) {
            revert();
        } else if (txType == IEndpoint.TransactionType.DepositInsurance) {
            IEndpoint.DepositInsurance memory txn = abi.decode(
                transaction[1:],
                (IEndpoint.DepositInsurance)
            );
            require(
                txn.amount >= uint128(SLOW_MODE_FEE),
                ERR_DEPOSIT_TOO_SMALL
            );
            handleDepositTransfer(_getQuote(), sender, uint256(txn.amount));
        } else if (
            txType == IEndpoint.TransactionType.WithdrawInsurance ||
            txType == IEndpoint.TransactionType.DelistProduct ||
            txType == IEndpoint.TransactionType.DumpFees ||
            txType == IEndpoint.TransactionType.RebalanceXWithdraw ||
            txType == IEndpoint.TransactionType.UpdateTierFeeRates ||
            txType == IEndpoint.TransactionType.AddNlpPool ||
            txType == IEndpoint.TransactionType.UpdateNlpPool ||
            txType == IEndpoint.TransactionType.DeleteNlpPool ||
            txType == IEndpoint.TransactionType.ForceRebalanceNlpPool ||
            txType == IEndpoint.TransactionType.NlpProfitShare ||
            txType == IEndpoint.TransactionType.UpdateBuilder
        ) {
            require(sender == owner());
        } else {
            chargeSlowModeFee(_getQuote(), sender);
            slowModeFees += SLOW_MODE_FEE;
        }

        IEndpoint.SlowModeConfig memory _slowModeConfig = slowModeConfig;
        requireUnsanctioned(sender);
        slowModeTxs[_slowModeConfig.txCount++] = IEndpoint.SlowModeTx({
            executableAt: uint64(block.timestamp) + SLOW_MODE_TX_DELAY, // hardcoded to three days
            sender: sender,
            tx: transaction
        });
        // TODO: to save on costs we could potentially just emit something
        // for now, we can just create a separate loop in the engine that queries the remote
        // sequencer for slow mode transactions, and ignore the possibility of a reorgy attack
        slowModeConfig = _slowModeConfig;
    }
```
