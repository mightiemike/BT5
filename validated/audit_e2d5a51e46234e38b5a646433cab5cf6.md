### Title
`withdrawFeeX18` Bypassed via Slow-Mode `WithdrawCollateral` Path — (`core/contracts/EndpointTx.sol`)

---

### Summary

The sequencer-processed `WithdrawCollateral` path charges `withdrawFeeX18` per product. The slow-mode execution path for the same transaction type omits this fee entirely. Any user can exploit this asymmetry to withdraw collateral while paying only the flat `SLOW_MODE_FEE` ($1), bypassing the per-product `withdrawFeeX18`.

---

### Finding Description

`processTransactionImpl` (the sequencer path) handles `WithdrawCollateral` by explicitly calling `chargeFee` with the product's configured `withdrawFeeX18` before executing the withdrawal: [1](#0-0) 

`processSlowModeTransactionImpl` (the slow-mode path) handles the **identical** `WithdrawCollateral` transaction type but calls `clearinghouse.withdrawCollateral` directly — with **no** `chargeFee` call: [2](#0-1) 

When a user submits a `WithdrawCollateral` transaction via `submitSlowModeTransaction`, the only cost charged on-chain is the flat `SLOW_MODE_FEE` ($1), deducted at submission time via `chargeSlowModeFee`: [3](#0-2) 

After the 3-day delay, `executeSlowModeTransaction` is a permissionless `external` function — the user can call it themselves: [4](#0-3) 

The slow-mode delay constant confirms the 3-day window: [5](#0-4) 

---

### Impact Explanation

Every user who withdraws via slow mode pays `SLOW_MODE_FEE` ($1 flat) instead of `withdrawFeeX18` (a per-product fee that can be significantly larger). The `withdrawFeeX18` revenue that should accrue to `sequencerFee[productId]` is never collected. This is a direct, quantifiable loss of protocol fee revenue proportional to the withdrawal volume routed through slow mode.

---

### Likelihood Explanation

The attack requires no privileged access, no sequencer cooperation, and no special contract. Any user who knows the slow-mode path exists can exploit it. The 3-day delay is the only friction. For large withdrawals where `withdrawFeeX18` exceeds $1, the economic incentive is clear and the path is fully self-service.

---

### Recommendation

Add a `chargeFee` call for `withdrawFeeX18` inside the `WithdrawCollateral` branch of `processSlowModeTransactionImpl`, mirroring the sequencer path:

```solidity
} else if (txType == IEndpoint.TransactionType.WithdrawCollateral) {
    IEndpoint.WithdrawCollateral memory txn = abi.decode(
        transaction[1:],
        (IEndpoint.WithdrawCollateral)
    );
    validateSender(txn.sender, sender);
+   chargeFee(
+       txn.sender,
+       spotEngine.getConfig(txn.productId).withdrawFeeX18,
+       txn.productId
+   );
    clearinghouse.withdrawCollateral(
        txn.sender,
        txn.productId,
        txn.amount,
        address(0),
        nSubmissions
    );
```

---

### Proof of Concept

1. Alice holds collateral in product `P` where `withdrawFeeX18 = 5e18` ($5).
2. Alice calls `submitSlowModeTransaction` with a `WithdrawCollateral{sender: aliceSubaccount, productId: P, amount: X}` payload.
3. `submitSlowModeTransactionImpl` charges `SLOW_MODE_FEE` = $1 and enqueues the transaction.
4. After 3 days, Alice calls `executeSlowModeTransaction()` (no access control).
5. `processSlowModeTransactionImpl` decodes the `WithdrawCollateral` transaction, validates the sender, and calls `clearinghouse.withdrawCollateral` — **no `chargeFee` call is made**.
6. Alice receives her full withdrawal. Protocol collects $1 instead of $5. Alice saves $4 per withdrawal. [2](#0-1) [6](#0-5) [7](#0-6)
