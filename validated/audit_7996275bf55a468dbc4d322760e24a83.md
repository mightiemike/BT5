### Title
SwapAllowlistExtension Checks Router Address Instead of Actual User, Allowing Any User to Bypass the Swap Allowlist - (File: metric-periphery/contracts/extensions/SwapAllowlistExtension.sol)

### Summary

`SwapAllowlistExtension.beforeSwap` gates swaps by checking the `sender` argument, which the pool sets to `msg.sender` of the `pool.swap()` call. When a user routes through `MetricOmmSimpleRouter`, `msg.sender` to the pool is the **router contract**, not the actual user. The extension therefore checks whether the router is allowlisted, not whether the individual user is allowlisted. Any user who routes through the public router bypasses the per-user allowlist entirely.

### Finding Description

**Call chain:**

```
user → MetricOmmSimpleRouter.exactInputSingle()
         → pool.swap(recipient, zeroForOne, amount, limit, "", extensionData)
              [msg.sender to pool = router]
              → _beforeSwap(msg.sender=router, ...)
                   → SwapAllowlistExtension.beforeSwap(sender=router, ...)
                        checks: allowedSwapper[pool][router]
```

In `MetricOmmPool.swap`, the pool passes `msg.sender` (the router) as `sender` to `_beforeSwap`: [1](#0-0) 

`ExtensionCalling._beforeSwap` forwards that `sender` argument unchanged to the extension: [2](#0-1) 

`SwapAllowlistExtension.beforeSwap` then checks `allowedSwapper[msg.sender][sender]`, where `msg.sender` is the pool and `sender` is the router: [3](#0-2) 

The router passes no user-identity information to the pool; it simply calls `pool.swap()` directly: [4](#0-3) 

The allowlist is therefore keyed on the router address for all router-mediated swaps, making it impossible to distinguish between individual users.

### Impact Explanation

A pool admin who deploys a curated pool with `SwapAllowlistExtension` intends to restrict swaps to a specific set of addresses (e.g., KYC-verified users). Because the router is the primary user-facing interface, the admin must allowlist the router address to enable normal usage. Once the router is allowlisted, **every user** — including those explicitly not on the allowlist — can bypass the gate by calling `MetricOmmSimpleRouter.exactInputSingle`, `exactInput`, `exactOutputSingle`, or `exactOutput`. The allowlist provides zero protection for router-mediated swaps. This is a direct loss of curation control and allows unauthorized parties to trade against LP funds in a pool that was designed to be restricted.

### Likelihood Explanation

The router is the standard, documented entry point for swaps. Pool admins who want their allowlisted users to be able to swap at all must allowlist the router. The bypass requires no special privileges, no malicious setup, and no non-standard tokens — any public user can call the router. Likelihood is high.

### Recommendation

The `sender` argument passed to the extension must represent the **economic actor** (the end user), not the intermediary contract. Two complementary fixes:

1. **Router-side**: Have the router pass the original `msg.sender` (the user) as an explicit `sender` field in `extensionData`, and update `SwapAllowlistExtension` to decode and check that field when `msg.sender` to the extension is a known router.

2. **Extension-side (preferred)**: Redesign the hook interface so the pool passes both the immediate caller (`msg.sender`) and an optional `originSender` (e.g., from a trusted router's transient context). `SwapAllowlistExtension` should check `originSender` when the immediate caller is a registered router, and `sender` otherwise.

Until fixed, pool admins should not rely on `SwapAllowlistExtension` for pools where `MetricOmmSimpleRouter` is expected to be used.

### Proof of Concept

```
Setup:
  - Deploy pool with SwapAllowlistExtension
  - Pool admin calls setAllowedToSwap(pool, alice, true)   // alice is KYC'd
  - Pool admin calls setAllowedToSwap(pool, router, true)  // needed for router-mediated swaps
  - bob is NOT allowlisted

Attack:
  - bob calls MetricOmmSimpleRouter.exactInputSingle({pool: pool, ...})
  - Router calls pool.swap(...) with msg.sender = router
  - Pool calls _beforeSwap(sender=router, ...)
  - Extension checks allowedSwapper[pool][router] → true
  - Swap executes successfully for bob despite not being allowlisted

Result:
  - bob trades against LP funds in a pool designed to be restricted to alice only
  - The allowlist invariant is broken for all router-mediated swaps
```
