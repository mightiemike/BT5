### Title
Reward Checkpoints Advanced Without Bonus Payment in `BeforeRedelegationSlashed` Hook, Causing Permanent Bonus Loss - (File: x/tieredrewards/keeper/slash.go)

### Summary
In `slashRedelegationPosition`, when `processEventsAndClaimBonus` returns `ErrInsufficientBonusPool`, the error is silently swallowed and the position is persisted with fully-advanced reward checkpoints (`LastBonusAccrual`, `LastKnownBonded`, `LastEventSeq`) even though no bonus was paid. The accrued bonus is permanently forfeited — the user can never recover it.

### Finding Description
`processEventsAndClaimBonus` advances the position's reward checkpoints and decrements validator event reference counts **before** it checks whether the bonus pool has sufficient balance: [1](#0-0) 

```go
applyBonusAccrualCheckpoint(&pos.Position, blockTime)   // checkpoints advanced here
pos.UpdateLastKnownBonded(bonded)
```

Only after those in-memory mutations does the pool sufficiency check occur: [2](#0-1) 

```go
if err := k.sufficientBonusPoolBalance(ctx, bonusCoins); err != nil {
    return nil, err   // returns ErrInsufficientBonusPool
}
```

Additionally, inside the event-processing loop, `decrementEventRefCount` is called for every processed event **before** the pool check: [3](#0-2) 

When `ErrInsufficientBonusPool` is returned, `slashRedelegationPosition` catches it and deliberately continues: [4](#0-3) 

```go
if _, err := k.processEventsAndClaimBonus(ctx, &pos); err != nil {
    if errors.Is(err, types.ErrInsufficientBonusPool) {
        k.logger(ctx).Error("insufficient bonus pool during redelegation slash", ...)
    } else {
        return err
    }
}
```

It then persists the position — with the already-advanced checkpoints — via `setPositionWithState`: [5](#0-4) 

Because no cache context is used, the decremented event reference counts are also committed. The net result: `LastBonusAccrual` is moved forward to the current block time, `LastEventSeq` is advanced past all processed events, those events' reference counts are decremented (potentially garbage-collected), and the bonus is never transferred to the owner.

### Impact Explanation
The position owner permanently loses all bonus rewards that accrued between `LastBonusAccrual` and the slash block time. Because `LastBonusAccrual` is now set to the slash block time and `LastEventSeq` is advanced, a subsequent `MsgClaimTierRewards` will compute zero bonus for the already-consumed time window. The corrupted value is `pos.LastBonusAccrual` and the associated `ValidatorEvent` reference counts stored in `ValidatorEvents`.

### Likelihood Explanation
The bonus pool (`RewardsPoolName`) is funded by governance and drained by the `BeginBlocker` base-rewards top-up on every block. A pool that is empty or nearly empty is a realistic operational state (e.g., governance has not yet replenished it, or a large number of concurrent claims have drained it). Any redelegation slash that fires while the pool is short triggers the silent forfeiture. The slash itself is a normal protocol event (validator misbehaviour), not an attacker-controlled action, but the resulting fund loss is borne entirely by the position owner with no recourse.

### Recommendation
Move the pool sufficiency check **before** advancing the checkpoints and decrementing event reference counts. One approach: compute `totalBonus` first, verify pool sufficiency, and only then advance `LastBonusAccrual`, `LastKnownBonded`, `LastEventSeq`, and decrement reference counts. Alternatively, wrap the entire `processEventsAndClaimBonus` call in a cache context inside `slashRedelegationPosition` so that all state mutations (reference count decrements, in-memory checkpoint updates) are discarded atomically when the pool is insufficient, leaving the position's checkpoints unchanged for a future successful claim.

### Proof of Concept

1. A position owner redelegates from validator A to validator B via `MsgTierRedelegate`. A `RedelegationMappings` entry is created.
2. Time passes; bonus accrues on validator B.
3. The bonus pool is drained to zero (e.g., by other claims or by the `BeginBlocker` top-up exhausting it).
4. Validator B is slashed. The staking module fires `BeforeRedelegationSlashed`.
5. `slashRedelegationPosition` calls `processEventsAndClaimBonus`:
   - `applyBonusAccrualCheckpoint` sets `pos.LastBonusAccrual = blockTime` (in memory).
   - `decrementEventRefCount` is called for each processed event (committed to store).
   - `sufficientBonusPoolBalance` returns `ErrInsufficientBonusPool`.
6. `slashRedelegationPosition` swallows the error and calls `setPositionWithState(ctx, pos, nil)`, persisting `LastBonusAccrual = blockTime`.
7. The owner later calls `MsgClaimTierRewards`. `processEventsAndClaimBonus` starts from the new `LastBonusAccrual` and computes zero bonus for the already-consumed window. The accrued bonus is permanently lost. [6](#0-5) [7](#0-6)

```go
func (k Keeper) slashRedelegationPosition(ctx context.Context, unbondingId uint64, sharesToUnbond math.LegacyDec) error {
	positionId, err := k.getRedelegationMapping(ctx, unbondingId)
	if errors.Is(err, collections.ErrNotFound) {
		return nil
	}
	if err != nil {
		return err
	}

	pos, err := k.getPositionState(ctx, positionId)
	if errors.Is(err, types.ErrPositionNotFound) {
		k.logger(ctx).Error("position not found during redelegation slash",
			"position_id", positionId,
			"unbonding_id", unbondingId,
			"error", err.Error(),
		)
		return nil
	}
	if err != nil {
		return err
	}

	if !pos.IsDelegated() {
		// Defensive
		k.logger(ctx).Error("delegation missing during BeforeRedelegationSlashed",
			"position_id", positionId,
			"unbonding_id", unbondingId,
			"shares_to_unbond", sharesToUnbond.String(),
		)
		return nil
	}

	dstValStr := pos.Delegation.ValidatorAddress

	// Settle bonus against PRE-slash shares.
	if _, err := k.processEventsAndClaimBonus(ctx, &pos); err != nil {
		// Deliberately forgo bonus rewards if pool is insufficient to prevent chain halt.
		if errors.Is(err, types.ErrInsufficientBonusPool) {
			k.logger(ctx).Error("insufficient bonus pool during redelegation slash",
				"position_id", pos.Id,
				"error", err.Error(),
			)
		} else {
			return err
		}
	}

	fullSlash := sharesToUnbond.GTE(pos.Delegation.Shares)

	if fullSlash {
		pos.Delegation = nil
		pos.ClearBonusCheckpoints()
		return k.setPositionWithState(ctx, pos, &ValidatorTransition{PreviousAddress: dstValStr})
	}
	// In-memory only: the persisted Position carries no share count, and the
	// live delegation will reflect the post-Unbond shares on the next read.
	// Update the local copy so any follow-up logic in this call sees consistent state.
	pos.Delegation.Shares = pos.Delegation.Shares.Sub(sharesToUnbond)
	return k.setPositionWithState(ctx, pos, nil)
```
