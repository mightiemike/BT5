Audit Report

## Title
`SwapAllowlistExtension` gates the direct pool caller (`sender`) instead of the end user, enabling allowlist bypass via `MetricOmmSimpleRouter` — (`metric-periphery/contracts/extensions/SwapAllowlistExtension.sol`)

## Summary

`SwapAllowlistExtension.beforeSwap` checks `allowedSwapper[msg.sender][sender]` where `sender` is `msg.sender` of `pool.swap()` — the direct pool caller, not the originating user. When `MetricOmmSimpleRouter` mediates a swap, it becomes the direct pool caller, so the extension checks the router's allowlist status rather than the end user's. Any pool admin who allowlists the router to support router-mediated swaps for their permitted users inadvertently opens the allowlist to all users, since the router is a shared public contract.

## Finding Description

**Root cause — `MetricOmmPool.swap()` passes `msg.sender` as `sender`:**

In `MetricOmmPool.swap()`, `_beforeSwap` is called with `msg.sender` as the first argument: [1](#0-0) 

This value propagates unchanged through `ExtensionCalling._beforeSwap` into every configured extension's `beforeSwap` hook.

**`SwapAllowlistExtension.beforeSwap` checks the direct caller, not the end user:** [2](#0-1) 

Here `msg.sender` is the pool (the extension's caller) and `sender` is whoever called `pool.swap()`. The check is `allowedSwapper[pool][directCaller]`.

**`MetricOmmSimpleRouter.exactInputSingle()` calls `pool.swap()` directly, making itself the direct caller:** [3](#0-2) 

The router passes `params.extensionData` through unchanged — it does **not** encode the end user's (`msg.sender`) address into `extensionData`. So the extension receives `sender` = router address, with no visibility into the originating user.

**`setAllowedToSwap` gates by address only:** [4](#0-3) 

There is no mechanism to distinguish "router acting on behalf of Alice" from "router acting on behalf of Eve." Allowlisting the router is functionally equivalent to calling `setAllowAllSwappers(pool, true)`.

**Exploit path:**
1. Pool admin deploys a curated pool with `SwapAllowlistExtension` to restrict swaps to KYC'd users.
2. Admin calls `setAllowedToSwap(pool, alice, true)` and `setAllowedToSwap(pool, address(router), true)` so Alice can use the router.
3. Eve (not allowlisted) calls `router.exactInputSingle(...)` targeting the pool.
4. Router calls `pool.swap(...)` — pool passes `msg.sender` = router address to `_beforeSwap`.
5. Extension checks `allowedSwapper[pool][router]` → `true` → swap proceeds.
6. Eve successfully swaps despite never being allowlisted.

## Impact Explanation

High. The swap allowlist of any curated pool is completely bypassed by any user routing through `MetricOmmSimpleRouter`. Unauthorized users can trade on pools designed for specific participants (e.g., KYC'd users, institutional counterparties), violating the pool's intended access control. LP funds are directly at risk: unauthorized swappers can extract value from LPs at oracle-anchored prices without the pool admin's consent, constituting a direct loss of LP assets and a broken core pool access-control invariant.

## Likelihood Explanation

Medium. The bypass requires the pool admin to allowlist the router address, which is a natural and expected operational action for any curated pool that wants to support router-mediated swaps for its permitted users. The admin has no way to achieve both goals simultaneously with the current design, making this a predictable operational mistake. No existing mechanism warns the admin that allowlisting the router is equivalent to opening the allowlist entirely.

## Recommendation

Gate the allowlist on the end user's identity rather than the direct pool caller. The router should encode `msg.sender` into `extensionData` before calling the pool, and `SwapAllowlistExtension.beforeSwap` should decode and verify it:

```solidity
function beforeSwap(
    address sender,
    address,
    bool, int128, uint128, uint256, uint128, uint128,
    bytes calldata extensionData
) external view override returns (bytes4) {
    address swapper = (extensionData.length >= 32)
        ? abi.decode(extensionData, (address))
        : sender;
    if (!allowAllSwappers[msg.sender] && !allowedSwapper[msg.sender][swapper]) {
        revert IMetricOmmPoolActions.NotAllowedToSwap();
    }
    return IMetricOmmExtensions.beforeSwap.selector;
}
```

`MetricOmmSimpleRouter` would prepend `abi.encode(msg.sender)` to `params.extensionData` before calling `pool.swap`. Alternatively, document explicitly that allowlisting the router address is equivalent to opening the allowlist to all users.

## Proof of Concept

```solidity
function test_SwapAllowlistBypassViaRouter() public {
    // Pool admin allowlists Alice and the router
    swapExtension.setAllowedToSwap(address(pool), alice, true);
    swapExtension.setAllowedToSwap(address(pool), address(router), true);

    // Eve (not allowlisted) bypasses the allowlist via the router
    vm.prank(eve);
    router.exactInputSingle(IMetricOmmSimpleRouter.ExactInputSingleParams({
        pool:             address(pool),
        tokenIn:          address(token0),
        tokenOut:         address(token1),
        zeroForOne:       true,
        amountIn:         1_000,
        amountOutMinimum: 0,
        recipient:        eve,
        deadline:         block.timestamp + 1,
        priceLimitX64:    0,
        extensionData:    ""
    }));
    // Eve successfully swaps despite not being allowlisted.
    // Extension saw sender = router (allowlisted), not Eve (not allowlisted).
}
```
