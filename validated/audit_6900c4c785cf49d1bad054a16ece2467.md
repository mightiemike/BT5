Audit Report

## Title
`SwapAllowlistExtension` Allowlist Bypassed via `MetricOmmSimpleRouter` — Router Address Replaces Actual Swapper Identity in `beforeSwap` Hook — (`metric-periphery/contracts/extensions/SwapAllowlistExtension.sol`)

## Summary
`SwapAllowlistExtension.beforeSwap()` gates swaps by checking `allowedSwapper[pool][sender]`, where `sender` is the `msg.sender` of the pool's own `swap()` call. When any user routes through `MetricOmmSimpleRouter`, the pool's `msg.sender` is the router contract, not the actual user. Because the router is a single shared address, allowlisting it to permit legitimate users simultaneously opens the gate for every unprivileged caller, making the allowlist guard structurally ineffective.

## Finding Description
**Root cause:** `MetricOmmPool.swap()` passes its own `msg.sender` as the `sender` argument to `_beforeSwap()`: [1](#0-0) 

`ExtensionCalling._beforeSwap()` forwards this value verbatim to the extension via `abi.encodeCall`: [2](#0-1) 

`SwapAllowlistExtension.beforeSwap()` then checks `allowedSwapper[msg.sender][sender]`, where `msg.sender` is the pool (correct) and `sender` is whoever called `pool.swap()`: [3](#0-2) 

When a user calls `MetricOmmSimpleRouter.exactInputSingle()`, the router itself calls `IMetricOmmPoolActions(params.pool).swap(...)`, making the router the `msg.sender` inside the pool: [4](#0-3) 

The extension therefore checks `allowedSwapper[pool][router]`, not `allowedSwapper[pool][actualUser]`. There is no configuration that simultaneously permits allowlisted users to use the router and blocks non-allowlisted users:

| Router allowlisted? | Allowlisted user via router | Non-allowlisted user via router |
|---|---|---|
| No | Blocked (broken UX) | Blocked |
| Yes | Allowed | **Allowed — bypass** |

No existing guard in `SwapAllowlistExtension`, `ExtensionCalling`, or `MetricOmmPool` checks the actual human caller's identity when a router intermediary is involved.

## Impact Explanation
Any unprivileged user can bypass a `SwapAllowlistExtension`-gated pool by routing through `MetricOmmSimpleRouter`, provided the pool admin has allowlisted the router. This breaks the core access-control invariant of the extension: unauthorized counterparties can trade in a restricted pool, exposing LPs to adverse selection and toxic flow from non-KYC'd or non-institutional actors. This matches the "Admin-boundary break bypassed by an unprivileged path" and "Broken core pool functionality causing loss of funds" impact categories.

## Likelihood Explanation
`MetricOmmSimpleRouter` is the primary user-facing swap interface. Any pool admin who deploys a `SwapAllowlistExtension` and wants their allowlisted users to use the standard router will naturally allowlist the router address. Once the router is allowlisted, the bypass is reachable by any unprivileged caller with no special preconditions, no privileged access, and no special tokens required.

## Recommendation
The extension must gate on the actual human swapper, not the intermediary contract. Two viable approaches:

1. **Extension-data identity**: Require the router to encode the real user's address in `extensionData`; the extension decodes and checks it. The router already forwards `params.extensionData` unchanged, so this is additive and backward-compatible.
2. **Separate router allowlist**: Add a second mapping `allowedRouter[pool][router]` and, when `sender` is an allowlisted router, decode the real user from `extensionData` and check `allowedSwapper[pool][realUser]`.

Either approach closes the gap without changing the pool's core `swap()` interface.

## Proof of Concept
```
1. Deploy pool with SwapAllowlistExtension as beforeSwap hook.
2. Pool admin calls setAllowedToSwap(pool, alice, true)   // allowlist Alice
3. Pool admin calls setAllowedToSwap(pool, router, true)  // allowlist router so Alice can use it
4. Bob (non-allowlisted) calls:
       router.exactInputSingle({pool: pool, ...})
5. Router calls pool.swap(recipient, ...) — msg.sender = router
6. Pool calls _beforeSwap(sender=router, ...)
7. ExtensionCalling forwards sender=router to SwapAllowlistExtension.beforeSwap()
8. Extension checks allowedSwapper[pool][router] → true
9. Bob's swap executes successfully — allowlist bypassed.
```
