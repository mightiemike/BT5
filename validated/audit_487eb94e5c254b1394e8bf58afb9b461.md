Audit Report

## Title
`SwapAllowlistExtension.beforeSwap` Checks Router Address Instead of End-User Address, Enabling Allowlist Bypass — (`metric-periphery/contracts/extensions/SwapAllowlistExtension.sol`)

## Summary
`SwapAllowlistExtension.beforeSwap` gates swaps by checking `allowedSwapper[msg.sender][sender]`, where `sender` is `msg.sender` of `pool.swap()`. When `MetricOmmSimpleRouter` intermediates the swap, `sender` becomes the router's address. If the pool admin allowlists the router to support router-mediated swaps for approved users — a natural operational step — every non-allowlisted address can bypass the swap gate unconditionally by routing through the router, rendering the allowlist ineffective for all router-mediated flows.

## Finding Description
`MetricOmmPool.swap()` passes `msg.sender` as `sender` to `_beforeSwap`: [1](#0-0) 

`ExtensionCalling._beforeSwap` forwards `sender` unchanged to the extension: [2](#0-1) 

`SwapAllowlistExtension.beforeSwap` checks `allowedSwapper[msg.sender][sender]` — `msg.sender` is the pool, `sender` is whoever called `pool.swap()`: [3](#0-2) 

`MetricOmmSimpleRouter.exactInputSingle` calls `pool.swap()` directly, making the router `msg.sender` of that call (and thus `sender` in the extension): [4](#0-3) 

When a non-allowlisted user calls the router, the check becomes `allowedSwapper[pool][router]`, which passes if the router is allowlisted. The user's own address is never checked. The `DepositAllowlistExtension` avoids this flaw by checking `owner`, an explicit parameter that always identifies the economic actor regardless of who calls `addLiquidity`: [5](#0-4) 

`beforeSwap` has no analogous explicit end-user parameter — `sender` is structurally bound to the direct caller of `pool.swap()`.

## Impact Explanation
The swap allowlist is the sole access-control mechanism on the swap path for curated pools. Once the router is allowlisted (to support approved users using multi-hop routing), the allowlist is bypassed unconditionally for all router-mediated swaps. Non-allowlisted addresses can trade on pools explicitly designed to restrict counterparties, causing unauthorized value flows through curated pools and breaking the core access-control invariant the extension is designed to enforce. This constitutes broken core pool functionality and an admin-boundary break where an unprivileged path circumvents the pool admin's configured access control.

## Likelihood Explanation
The trigger requires the pool admin to allowlist the router — a natural and expected operational step once the admin wants approved users to benefit from multi-hop routing. Once the router is allowlisted, the bypass requires no special privilege: any EOA calls `MetricOmmSimpleRouter.exactInputSingle` (or `exactInput`/`exactOutput`/`exactOutputSingle`), which is a public, permissionless entry point. The bypass is unconditional and repeatable.

## Recommendation
The `beforeSwap` hook must gate the economically relevant actor, not the intermediate contract. Two options:

1. **Pass the originating user via `extensionData`**: Have `MetricOmmSimpleRouter` encode `msg.sender` into `extensionData` and have `SwapAllowlistExtension.beforeSwap` decode and check that address when `sender` is a known router.
2. **Trusted-router registry**: Maintain a registry of trusted routers in the extension; when `sender` is a trusted router, decode the real user from `extensionData`; otherwise check `sender` directly.

The checked identity must always be the address that economically initiates the swap, not the relay contract.

## Proof of Concept
```
Setup:
  pool configured with SwapAllowlistExtension
  allowedSwapper[pool][alice] = true      // alice is approved
  allowedSwapper[pool][router] = true     // admin enables router for alice's use
  allowedSwapper[pool][bob]   = false     // bob is NOT approved

Attack:
  bob calls MetricOmmSimpleRouter.exactInputSingle(pool, ...)
  → router calls pool.swap(recipient=bob, ...)          [MetricOmmSimpleRouter.sol L72-80]
  → pool calls _beforeSwap(sender=router, ...)          [MetricOmmPool.sol L230-240]
  → SwapAllowlistExtension checks allowedSwapper[pool][router] == true  ✓
  → swap executes for bob despite bob not being allowlisted
```
