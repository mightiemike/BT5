Audit Report

## Title
SwapAllowlistExtension Checks Router Address Instead of Originating User, Allowing Allowlist Bypass via MetricOmmSimpleRouter — (`metric-periphery/contracts/extensions/SwapAllowlistExtension.sol`)

## Summary

`SwapAllowlistExtension.beforeSwap` validates the `sender` argument forwarded by the pool, which is always `msg.sender` of the pool's `swap()` call. When a user routes through `MetricOmmSimpleRouter`, that `msg.sender` is the router contract address, not the end user. A pool admin who allowlists the router to enable their curated users to access it inadvertently opens the pool to every user, because the extension cannot distinguish individual callers behind the router.

## Finding Description

`MetricOmmPool.swap()` passes `msg.sender` verbatim as the `sender` argument to `_beforeSwap`:

```solidity
// MetricOmmPool.sol L230-240
_beforeSwap(
    msg.sender,   // always the direct caller of pool.swap()
    recipient,
    ...
    extensionData
);
```

`SwapAllowlistExtension.beforeSwap` then checks that exact address against the per-pool allowlist, where `msg.sender` is the pool and `sender` is the address the pool forwarded:

```solidity
// SwapAllowlistExtension.sol L37-38
if (!allowAllSwappers[msg.sender] && !allowedSwapper[msg.sender][sender]) {
    revert IMetricOmmPoolActions.NotAllowedToSwap();
}
```

When a user calls `MetricOmmSimpleRouter.exactInputSingle`, the router calls `pool.swap()` directly, making the router the `msg.sender` seen by the pool:

```solidity
// MetricOmmSimpleRouter.sol L71-80
_setNextCallbackContext(params.pool, CALLBACK_MODE_JUST_PAY, msg.sender, params.tokenIn);
(int128 amount0Delta, int128 amount1Delta) = IMetricOmmPoolActions(params.pool)
    .swap(params.recipient, params.zeroForOne, ..., "", params.extensionData);
```

Critically, the router stores the real `msg.sender` only in transient storage for the payment callback (`_setNextCallbackContext`), and passes `params.extensionData` (user-supplied, not augmented with the real caller) to the pool. The extension receives `sender = router_address` and has no access to the originating user.

A pool admin who wants allowlisted users to use the router must call `setAllowedToSwap(pool, address(router), true)`. Once the router is allowlisted, `allowedSwapper[pool][router] == true` passes the check for every caller behind the router, including addresses never individually allowlisted.

## Impact Explanation

A curated pool (KYC-gated, institutional, or restricted-access) that deploys `SwapAllowlistExtension` to enforce per-user swap permissions loses that enforcement entirely for any user routing through the public `MetricOmmSimpleRouter`. Non-allowlisted users can execute swaps at oracle-anchored prices on pools they were explicitly excluded from, directly breaking the access-control invariant the extension was deployed to enforce. This constitutes a broken core pool functionality causing potential loss of funds for LP positions in the curated pool, meeting the "Admin-boundary break" and "Broken core pool functionality" impact criteria.

## Likelihood Explanation

The bypass requires the pool admin to have allowlisted the router — a natural and expected operational step for any pool admin who wants their allowlisted users to access the standard periphery router. The admin has no way to express "allow the router only for specific end users" because the extension only sees the router address. The misconfiguration is therefore not a mistake but an unavoidable consequence of the current design whenever the router is used alongside a per-user allowlist. Any unprivileged user can exploit this by simply calling the router instead of the pool directly.

## Recommendation

The `SwapAllowlistExtension` should check the originating user rather than the direct pool caller. The cleanest fix: the router always appends its `msg.sender` to `extensionData` before calling the pool, and the extension decodes and checks that address when the direct caller (`sender`) is a known/trusted router. Alternatively, introduce a router-level allowlist check where `MetricOmmSimpleRouter` queries the pool's allowlist extension before forwarding the swap and reverts if the originating `msg.sender` is not allowlisted.

## Proof of Concept

```
Setup:
  - Deploy pool with SwapAllowlistExtension
  - Pool admin calls setAllowedToSwap(pool, alice, true)   // alice is KYC'd
  - Pool admin calls setAllowedToSwap(pool, router, true)  // so alice can use the router
  - bob is NOT allowlisted

Direct swap attempt by bob:
  bob → pool.swap(...)
  pool calls extension.beforeSwap(sender=bob, ...)
  allowedSwapper[pool][bob] == false  →  revert NotAllowedToSwap ✓

Router-mediated swap by bob:
  bob → router.exactInputSingle({pool, recipient=bob, ...})
  router → pool.swap(recipient=bob, ...)
  pool calls extension.beforeSwap(sender=router, ...)
  allowedSwapper[pool][router] == true  →  swap executes ✗

bob successfully swaps on a pool he was explicitly excluded from.
