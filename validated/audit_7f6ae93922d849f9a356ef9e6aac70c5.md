### Title
IBC NFT Transfer `OnAcknowledgementPacket` Default Case Silently Drops Nil-Response Acknowledgements, Causing Permanent NFT Loss — (`x/nft-transfer/keeper/relay.go`)

---

### Summary

The keeper's `OnAcknowledgementPacket` switch statement treats every non-`Acknowledgement_Error` response — including a nil `Response` field — as a successful transfer and returns `nil` without calling `refundPacketToken`. A malicious counterparty chain can commit an acknowledgement whose JSON body is `{}` (nil oneof), causing the source chain to silently discard the failed transfer and permanently lose the sender's NFT.

---

### Finding Description

In `x/nft-transfer/keeper/relay.go`, `OnAcknowledgementPacket` is:

```go
switch ack.Response.(type) {
case *channeltypes.Acknowledgement_Error:
    return k.refundPacketToken(ctx, packet, data)
default:
    // the acknowledgement succeeded on the receiving chain so nothing
    // needs to be executed and no error needs to be returned
    return nil
}
``` [1](#0-0) 

The `default` branch is intended only for `*channeltypes.Acknowledgement_Result`, but it also matches any other type, including a nil `Response` field. The caller in `ibc_module.go` unmarshals the raw acknowledgement bytes with no content validation:

```go
var ack channeltypes.Acknowledgement
if err := types.ModuleCdc.UnmarshalJSON(acknowledgement, &ack); err != nil {
    return ...
}
``` [2](#0-1) 

If the counterparty chain commits the bytes `{}` (a valid JSON object for the `Acknowledgement` proto message, but with no oneof variant set), `UnmarshalJSON` succeeds and `ack.Response` is nil. The switch then falls into `default`, returns `nil`, and `refundPacketToken` is never called. [3](#0-2) 

The IBC core layer verifies only the Merkle proof that the counterparty committed those bytes; it does not validate the semantic content of the acknowledgement. So the malformed acknowledgement passes all IBC core checks and reaches the application.

---

### Impact Explanation

When a user sends an NFT via `SendTransfer`, the NFT is either escrowed (source chain) or burned (sink chain) before the packet is sent: [4](#0-3) 

If the counterparty returns a nil-Response acknowledgement, `refundPacketToken` — which would unescrow or re-mint the NFT — is never called: [5](#0-4) 

The result is permanent, irrecoverable NFT loss: the NFT is gone from the sender's account, the receiver never received it, and no refund path is triggered.

---

### Likelihood Explanation

This requires a malicious (or compromised) counterparty chain that deliberately writes `{}` as the IBC acknowledgement. This is a realistic threat in the IBC ecosystem — chains are independent actors, and the IBC protocol is explicitly designed to be trustless. The attack is fully deterministic and requires no special privileges beyond operating the counterparty chain.

---

### Recommendation

Replace the `default` branch with an explicit `*channeltypes.Acknowledgement_Result` case and return an error for any other type (including nil):

```go
switch ack.Response.(type) {
case *channeltypes.Acknowledgement_Error:
    return k.refundPacketToken(ctx, packet, data)
case *channeltypes.Acknowledgement_Result:
    return nil
default:
    return sdkerrors.Wrapf(sdkerrors.ErrUnknownRequest,
        "invalid acknowledgement response type: %T", ack.Response)
}
```

Returning an error from `OnAcknowledgementPacket` causes the IBC core to abort the transaction, preserving the packet commitment so the sender can retry or wait for timeout and receive a refund via `OnTimeoutPacket`.

func (k Keeper) refundPacketToken(ctx sdk.Context, packet channeltypes.Packet, data types.NonFungibleTokenPacketData) error {
	sender, err := sdk.AccAddressFromBech32(data.Sender)
	if err != nil {
		return err
	}

	classTrace := types.ParseClassTrace(data.ClassId)
	voucherClassID := classTrace.IBCClassID()

	isAwayFromOrigin := types.IsAwayFromOrigin(packet.GetSourcePort(),
		packet.GetSourceChannel(), data.ClassId)

	escrowAddress := types.GetEscrowAddress(packet.GetSourcePort(), packet.GetSourceChannel())

	if isAwayFromOrigin {
		// unescrow tokens back to the sender
		for _, tokenID := range data.TokenIds {
			if err := k.nftKeeper.TransferOwner(ctx, voucherClassID, tokenID, escrowAddress, sender); err != nil {
				return err
			}
		}
	} else {
		// we are sink chain, mint voucher back to sender
		for i, tokenID := range data.TokenIds {
			if err := k.nftKeeper.MintNFT(ctx, voucherClassID, tokenID, "", data.TokenUris[i], "", escrowAddress, sender); err != nil {
				return err
			}
		}
	}

	return nil
