### Title
Stale Delayed Declare Inserted into Pool Without Nonce Guard Masks Real Nonce Gap, Corrupting Eviction Admission Decisions — (`crates/apollo_mempool/src/mempool.rs`)

---

### Summary

`add_ready_declares` in `crates/apollo_mempool/src/mempool.rs` promotes expired delayed-declare transactions into `tx_pool` via `add_tx_inner` with **no nonce validity check**. In a multi-sequencer scenario a delayed declare can become stale (its nonce superseded by a competing commit) while still sitting in the delay queue. Once the delay elapses the stale declare (nonce N < account\_nonce) is unconditionally inserted into the pool. The subsequent call to `update_accounts_with_gap` then evaluates `account_nonce < tx_pool.get_lowest_nonce()` as **FALSE** (because `get_lowest_nonce` returns N < account\_nonce), incorrectly clearing the gap flag for the account. The account's genuinely stuck transactions are no longer candidates for eviction, causing the mempool to reject valid transactions from other accounts when at capacity.

---

### Finding Description

**Invariant broken (analog of the external bug):** A lifecycle object (the delayed declare) continues to occupy pool state and influence admission decisions even after its underlying validity condition (nonce currency) has been invalidated by a competing commit — exactly as an out-of-money put option continues to lock collateral and consume funding after the underlying bond is redeemed.

**Root cause — `add_ready_declares` (line 619):**

```rust
fn add_ready_declares(&mut self) {
    let now = self.clock.now();
    while let Some((submission_time, _args)) = self.delayed_declares.front() {
        if now - self.config.static_config.declare_delay < *submission_time {
            break;
        }
        let (_submission_time, args) =
            self.delayed_declares.pop_front().expect("Delay declare should exist.");
        self.add_tx_inner(args);   // ← no nonce guard; stale declare enters tx_pool

    self.update_state_metrics();   // ← update_accounts_with_gap NOT called here

        // First remove old transactions from the pool.
        let mut account_nonce_updates = self.remove_expired_txs();
        if !self.is_fifo() {
            self.add_ready_declares();
        }

        let tx_reference = TransactionReference::new(&args.tx);
        self.add_tx_validations(tx_reference, &args.tx, args.account_state.nonce)
            .inspect_err(|err| self.log_add_tx_error(err, &args))?;

        MEMPOOL_TRANSACTIONS_RECEIVED.increment(
            1,
            &[(LABEL_NAME_TX_TYPE, InternalRpcTransactionLabelValue::from(&args.tx.tx).into())],
        );

        // May override a removed queued nonce with the received account nonce or the account's
        // state nonce.
        account_nonce_updates.insert(
            args.account_state.address,
            self.state.resolve_nonce(args.account_state.address, args.account_state.nonce),
        );

            matches!(&args.tx.tx, InternalRpcTransactionWithoutTxHash::Declare(_))
                && !self.is_fifo();
