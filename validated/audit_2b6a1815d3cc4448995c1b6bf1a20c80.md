### Title
`allowed_proof_versions` in `VersionedConstants` accepts `PROOF_VERSION_V0` while Cairo OS `check_proof_facts` enforces only `PROOF_VERSION_V1`, causing blockifier to accept transactions the OS will reject — (File: `crates/blockifier/resources/blockifier_versioned_constants_0_14_2.json`)

### Summary
The `allowed_proof_versions` list in the blockifier's `VersionedConstants` (e.g., `blockifier_versioned_constants_0_14_2.json`) contains only `PROOF_VERSION_V0` (`0x50524f4f4630`). The blockifier's Rust-side `validate_proof_facts` uses this list to gate acceptance. However, the Cairo OS `check_proof_facts` in `execution_constraints.cairo` hard-asserts `proof_header.proof_version = PROOF_VERSION_V1`, unconditionally rejecting V0. This version-boundary mismatch is the direct sequencer analog of the external bug: two "authorities" (blockifier pre-validation vs. OS execution) disagree on which proof version is valid, allowing a transaction to pass one gate and fail the other.

### Finding Description

**Blockifier side — `allowed_proof_versions` contains only V0:**

In `blockifier_versioned_constants_0_14_2.json`:
```json
"allowed_proof_versions": [
    "0x50524f4f4630"
]
```
`0x50524f4f4630` is `PROOF_VERSION_V0` ("PROOF0"). [1](#0-0) 

The Rust constant confirms this:
```rust
pub const PROOF_VERSION_V0: Felt = Felt::from_hex_unchecked("0x50524f4f4630");
pub const PROOF_VERSION_V1: Felt = Felt::from_hex_unchecked("0x50524f4f4631");
``` [2](#0-1) 

The code comment on `ProofVersion` explicitly acknowledges the divergence:
> "V0 is retained only so that historical blocks carrying V0 proof facts can be replayed (e.g. via reexecution). Whether V0 is accepted is gated per protocol version in the blockifier; **the proof verifier no longer supports it**." [3](#0-2) 

**Blockifier pre-validation — accepts V0:**

`validate_proof_facts` in `account_transaction.rs` checks `allowed_proof_versions`:
```rust
if !os_constants.allowed_proof_versions.contains(&snos_proof_facts.proof_version.as_felt()) {
    return Err(TransactionPreValidationError::InvalidProofFacts(...));
}
```
Because `allowed_proof_versions = [V0]`, a transaction carrying `proof_version = V0` passes this check. [4](#0-3) 

This pre-validation is called from `perform_pre_validation_stage`, which runs during block building: [5](#0-4) 

**Cairo OS side — only V1 is allowed:**

`check_proof_facts` in `execution_constraints.cairo` hard-asserts V1:
```cairo
// Proof version may be V0 (legacy) or V1 (current).
with_attr error_message("Unsupported proof version") {
    assert proof_header.proof_version = PROOF_VERSION_V1;
}
```
The comment acknowledges V0 as legacy but the assertion unconditionally rejects it. [6](#0-5) 

`check_proof_facts` is called from `transaction_impls.cairo` for every invoke transaction during OS execution: [7](#0-6) 

**The divergence path:**

1. User submits an invoke V3 transaction with `proof_facts[0] = PROOF_VERSION_V0`.
2. Gateway stateless validator has no proof-version check — passes.
3. Blockifier `validate_proof_facts` checks `allowed_proof_versions = [V0]` — **accepts**.
4. Transaction is included in a proposed block; blockifier produces an execution receipt.
5. OS `check_proof_facts` asserts `proof_version = PROOF_VERSION_V1` — **hard abort** with "Unsupported proof version".
6. The OS execution fails; the block cannot be proven; the sequencer's committed state diverges from what the prover can certify.

The blockifier does **not** run the OS Cairo code during block building — it only runs account contract entry points. Therefore the OS-level rejection is invisible to the blockifier, and the two components produce contradictory verdicts for the same accepted transaction.

### Impact Explanation

**Critical — Wrong revert/execution result from blockifier/execution logic for accepted input.**

The blockifier accepts and executes a transaction that the OS will unconditionally abort on. The blockifier emits a valid-looking receipt; the OS emits a hard failure. This produces a divergent execution result: the sequencer believes the transaction succeeded (or reverted cleanly), while the prover cannot produce a valid proof for the block. Any block containing such a transaction is unprovable, breaking L1 settlement.

### Likelihood Explanation

**Medium.** The trigger requires a transaction with `proof_facts[0] = PROOF_VERSION_V0` and a structurally valid (but V0-versioned) proof-facts payload. The `allow_client_side_proving` gateway flag must be enabled. An attacker who reads the versioned-constants JSON and the Cairo OS source can craft this transaction deterministically. No privileged access is required.

### Recommendation

Align the two authorities to a single canonical version:

1. **Update `allowed_proof_versions`** in all current and future versioned-constants JSON files to contain only `PROOF_VERSION_V1` (`0x50524f4f4631`). V0 should be present only in versioned-constants files that correspond to historical protocol versions where V0 was valid, and those files should never be used for new block production.
2. **Add a cross-check test** that reads every production versioned-constants file and asserts that `allowed_proof_versions` is a subset of the proof versions the Cairo OS constant `PROOF_VERSION_V1` accepts, preventing future drift.
3. Consider removing `ProofVersion::V0` from the live acceptance path entirely and restricting it to a reexecution-only feature flag.

// Represent the `PROOF_VERSION_V0` marker as a Felt ('PROOF0').
pub const PROOF_VERSION_V0: Felt = Felt::from_hex_unchecked("0x50524f4f4630");

// Represent the `PROOF_VERSION_V1` marker as a Felt ('PROOF1').
pub const PROOF_VERSION_V1: Felt = Felt::from_hex_unchecked("0x50524f4f4631");

/// Supported proof-facts version markers.
///
/// V0 is retained only so that historical blocks carrying V0 proof facts can be replayed (e.g. via
/// reexecution). Whether V0 is accepted is gated per protocol version in the blockifier; the proof
/// verifier no longer supports it.
#[cfg_attr(any(test, feature = "testing"), derive(EnumIter))]
#[derive(Clone, Copy, Debug, Eq, PartialEq)]
pub enum ProofVersion {
    V0,
    V1,
}

impl ProofVersion {
    /// Felt (Cairo short-string) representation written into proof facts.
    pub const fn as_felt(self) -> Felt {
        match self {
            ProofVersion::V0 => PROOF_VERSION_V0,
            ProofVersion::V1 => PROOF_VERSION_V1,
        }
    }

    /// Human-readable short-string label (matches the Cairo constant value).
    pub const fn as_str(self) -> &'static str {
        match self {
            ProofVersion::V0 => "PROOF0",
            ProofVersion::V1 => "PROOF1",
        }
    }
}

impl TryFrom<Felt> for ProofVersion {
    type Error = ();
    fn try_from(value: Felt) -> Result<Self, Self::Error> {
        if value == PROOF_VERSION_V0 {
            Ok(ProofVersion::V0)
        } else if value == PROOF_VERSION_V1 {
            Ok(ProofVersion::V1)
        } else {
            Err(())
        }
    }
}

    fn validate_proof_facts(
        &self,
        block_context: &BlockContext,
        state: &mut dyn State,
    ) -> TransactionPreValidationResult<()> {
        // Only Invoke V3 transactions can carry proof facts.
        let Transaction::Invoke(invoke_tx) = &self.tx else {
            return Ok(());
        };
        if invoke_tx.version() < TransactionVersion::THREE {
            return Ok(());
        }

        // Parse proof facts.
        let proof_facts = invoke_tx.proof_facts();
        let snos_proof_facts = match ProofFactsVariant::try_from(&proof_facts)
            .map_err(|e| TransactionPreValidationError::InvalidProofFacts(e.to_string()))?
        {
            ProofFactsVariant::Empty => return Ok(()),
            ProofFactsVariant::Snos(snos_proof_facts) => snos_proof_facts,
        };
        let os_constants = &block_context.versioned_constants.os_constants;

        if !os_constants.allowed_proof_versions.contains(&snos_proof_facts.proof_version.as_felt())
        {
            return Err(TransactionPreValidationError::InvalidProofFacts(format!(
                "Proof version {} is not allowed under this protocol version.",
                snos_proof_facts.proof_version
            )));
        }

        // Validate the program hash.
        let allowed = &os_constants.allowed_virtual_os_program_hashes;
        if !allowed.contains(&snos_proof_facts.program_hash) {
            return Err(TransactionPreValidationError::InvalidProofFacts(format!(
                "Virtual OS program hash {} is not allowed",
                snos_proof_facts.program_hash
            )));
        }

        // Validate the block hash and block number.
        let proof_block_hash = snos_proof_facts.block_hash.0;
        let proof_block_number = snos_proof_facts.block_number.0;
        Self::validate_proof_block_number(
            proof_block_number,
            block_context.block_info.block_number,
        )?;
        Self::validate_proof_block_hash(proof_block_hash, proof_block_number, os_constants, state)?;

        // Validate the config hash.
        let virtual_os_config_hash = block_context.virtual_os_config_hash();
        let proof_config_hash = snos_proof_facts.config_hash;
        if virtual_os_config_hash != proof_config_hash {
            return Err(TransactionPreValidationError::InvalidProofFacts(format!(
                "Virtual OS config hash mismatch. Computed virtual OS config hash: \
                 {virtual_os_config_hash}, expected virtual OS config hash: {proof_config_hash}."
            )));
        }

        Ok(())
    }

func check_proof_facts{range_check_ptr, contract_state_changes: DictAccess*}(
    proof_facts_size: felt,
    proof_facts: felt*,
    current_block_number: felt,
    virtual_os_config_hash: felt,
) {
    if (proof_facts_size == 0) {
        return ();
    }

    assert_le(ProofHeader.SIZE + VirtualOsOutputHeader.SIZE, proof_facts_size);

    // Validate the proof header.
    static_assert ProofHeader.SIZE == 3;
    let proof_header = cast(proof_facts, ProofHeader*);
    assert proof_header.proof_variant = VIRTUAL_SNOS;
    assert is_program_hash_allowed(proof_header.program_hash) = TRUE;
    // Proof version may be V0 (legacy) or V1 (current).
    with_attr error_message("Unsupported proof version") {
        assert proof_header.proof_version = PROOF_VERSION_V1;
    }

    // Validate the virtual OS output header.
    let os_output_header = cast(&proof_facts[ProofHeader.SIZE], VirtualOsOutputHeader*);

    with_attr error_message("Virtual OS output version is not supported") {
        assert os_output_header.output_version = VIRTUAL_OS_OUTPUT_VERSION;
    }

    // Validate that the proof facts block number is not too recent.
    // (This is a sanity check - the following non-zero check ensures that the block hash is
    // not trivial).
    assert_nn_le(
        os_output_header.base_block_number, current_block_number - STORED_BLOCK_HASH_BUFFER
    );
    // Not all block hashes are stored in the contract; Make sure the requested one is not trivial.
    assert_not_zero(os_output_header.base_block_hash);

    // validate that the proof facts block hash is the true hash of the proof facts block number.
    read_block_hash_from_storage(
        block_number=os_output_header.base_block_number,
        expected_block_hash=os_output_header.base_block_hash,
    );

    // validate that the proof facts config hash is the true hash of the OS config.
    assert os_output_header.starknet_os_config_hash = virtual_os_config_hash;

    return ();
}
