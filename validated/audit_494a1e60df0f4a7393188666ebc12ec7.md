Audit Report

## Title
SwapAllowlistExtension Checks Router Address Instead of Actual Swapper, Allowing Any User to Bypass the Curated-Pool Allowlist - (File: metric-periphery/contracts/extensions/SwapAllowlistExtension.sol)

## Summary
`SwapAllowlistExtension.beforeSwap` gates swaps by checking the `sender` argument it receives from the pool, which is always `msg.sender` of the `pool.swap` call — the router contract, not the end user. If the pool admin allowlists the router to enable normal periphery access, every unprivileged user can swap in the curated pool by calling `MetricOmmSimpleRouter`, defeating the entire per-user allowlist.

## Finding Description
In `SwapAllowlistExtension.beforeSwap`, the check at line 37 is:

```solidity
if (!allowAllSwappers[msg.sender] && !allowedSwapper[msg.sender][sender]) {
    revert IMetricOmmPoolActions.NotAllowedToSwap();
}
```

Here `msg.sender` is the pool (correct for pool-namespacing) and `sender` is the first argument passed by the pool when it calls the extension.

In `MetricOmmPool.swap` (lines 230–240), `_beforeSwap` is called with `msg.sender` as the first positional argument:

```solidity
_beforeSwap(
    msg.sender,   // becomes `sender` in the extension
    recipient,
    ...
);
```

When a user routes through `MetricOmmSimpleRouter.exactInputSingle` (lines 72–80), the router calls `pool.swap(...)` directly, making the pool's `msg.sender` the router contract address. The extension therefore receives `sender = router`, not `sender = user`.

The allowlist is keyed `allowedSwapper[pool][swapper]` where the admin intends `swapper` to be the end user. But the extension receives and checks the router's address. The admin must either: (1) allowlist the router — every user can swap through the router, bypassing per-user curation entirely; or (2) not allowlist the router — individually allowlisted users cannot use the standard periphery path at all. Neither outcome matches the intended invariant.

## Impact Explanation
A pool configured with `SwapAllowlistExtension` is a curated pool where only approved addresses may trade. If the pool admin allowlists the router (the expected operational step to enable normal periphery usage), any unprivileged user can call `MetricOmmSimpleRouter.exactInputSingle` and swap against the pool. The allowlist provides zero protection against router-routed swaps. This constitutes a direct bypass of curated-pool access control, allowing unauthorized users to execute swaps and receive output tokens from the pool they were never authorized to receive — a direct loss of LP-owed output tokens at oracle prices.

## Likelihood Explanation
`MetricOmmSimpleRouter` is the standard swap entry point. Any pool admin who deploys a `SwapAllowlistExtension` and allowlists the router to enable normal user access triggers the bypass automatically. No special knowledge or privileged access is required — any user who calls `exactInputSingle`, `exactInput`, or `exactOutputSingle` through the router is affected. The precondition (router allowlisted) is the natural operational setup.

## Recommendation
The extension must check the actual economic actor, not the immediate pool caller. The preferred fix is to have the router encode the original `msg.sender` in `extensionData`, and have the extension decode and verify it against the allowlist. This requires a coordinated change to the router and extension. Alternatively, redesign the hook to accept a trusted-forwarder pattern where the router attests the real user address.

## Proof of Concept
1. Pool admin deploys a pool with `SwapAllowlistExtension` configured as a `beforeSwap` hook.
2. Admin calls `setAllowedToSwap(pool, alice, true)` — only Alice is allowed.
3. Admin calls `setAllowedToSwap(pool, router, true)` — router is allowlisted so Alice can use the periphery.
4. Eve (not allowlisted) calls `MetricOmmSimpleRouter.exactInputSingle({pool: pool, ...})`.
5. The router calls `pool.swap(...)` with `msg.sender = router` (line 72–80 of `MetricOmmSimpleRouter.sol`).
6. The pool calls `extension.beforeSwap(router, ...)` (line 230–240 of `MetricOmmPool.sol`).
7. The extension checks `allowedSwapper[pool][router]` → `true` → swap proceeds (line 37 of `SwapAllowlistExtension.sol`).
8. Eve successfully swaps in the curated pool, receiving output tokens she was never authorized to receive. [1](#0-0) [2](#0-1) [3](#0-2)
