### Title
Division by Zero in `PerpEngine.socializeSubaccount()` When `openInterest == 0` Permanently Blocks Liquidation Finalization — (`core/contracts/PerpEngine.sol`)

---

### Summary

`PerpEngine.socializeSubaccount()` unconditionally divides by `state.openInterest` when a subaccount has residual negative `vQuoteBalance` that insurance cannot cover. If `openInterest` is zero at the time of finalization — a realistic condition in a low-liquidity or single-participant market — the division reverts with `"DBZ"`, permanently blocking the finalization of any insolvent subaccount in that perp market.

---

### Finding Description

In `PerpEngine.socializeSubaccount()`, when insurance is insufficient to cover a subaccount's negative `vQuoteBalance`, the protocol attempts to spread the loss across all open-interest holders:

```solidity
// core/contracts/PerpEngine.sol lines 164–170
if (balance.vQuoteBalance < 0) {
    // socialize across all other participants
    int128 fundingPerShare = -balance.vQuoteBalance.div(
        state.openInterest   // ← DIVISION BY ZERO if openInterest == 0
    );
    state.cumulativeFundingLongX18 += fundingPerShare;
    state.cumulativeFundingShortX18 -= fundingPerShare;
    balance.vQuoteBalance = 0;
}
``` [1](#0-0) 

There is no guard checking `state.openInterest != 0` before this division. `MathSD21x18.div()` enforces `require(y != 0, ERR_DIV_BY_ZERO)`, so the call reverts. [2](#0-1) 

**When can `openInterest == 0` while `balance.vQuoteBalance < 0`?**

Finalization (`txn.productId == type(uint32).max`) requires all perp positions to have `balance.amount == 0`:

```solidity
// core/contracts/ClearinghouseLiq.sol line 319
require(balance.amount == 0, ERR_NOT_FINALIZABLE_SUBACCOUNT);
``` [3](#0-2) 

A subaccount can have `amount == 0` (position closed) but still carry a negative `vQuoteBalance` from accumulated funding payments or realized losses. If no other participants hold open positions in that perp market, `state.openInterest == 0`. The division by zero then fires.

`socializeSubaccount` is called unconditionally during finalization:

```solidity
// core/contracts/ClearinghouseLiq.sol lines 386–389
v.insurance = perpEngine.socializeSubaccount(
    txn.liquidatee,
    v.insurance
);
``` [4](#0-3) 

---

### Impact Explanation

The revert permanently blocks `_finalizeSubaccount`, which means:

1. The insolvent subaccount can never be finalized — bad debt is never resolved.
2. The insurance fund cannot be properly applied or recovered.
3. The loss cannot be socialized across remaining participants.
4. The protocol is stuck with an unresolvable insolvent position in that market.

This is a solvency/accounting corruption impact, not merely a gas or performance issue.

---

### Likelihood Explanation

The condition is realistic in any perp market that has seen low participation or where the liquidatee was the last active trader. A subaccount can accumulate negative `vQuoteBalance` through funding payments even after closing its position (`amount == 0`). If no other subaccount holds an open position at the time of finalization, `openInterest == 0` and the revert is guaranteed. A liquidator (unprivileged caller) triggers this by submitting a `LiquidateSubaccount` transaction with `productId == type(uint32).max`.

---

### Recommendation

Add a zero-check on `state.openInterest` before the division. If `openInterest == 0`, there are no participants to socialize the loss against; the remaining negative balance should be absorbed by the insurance fund or written off:

```solidity
if (balance.vQuoteBalance < 0) {
    if (state.openInterest != 0) {
        int128 fundingPerShare = -balance.vQuoteBalance.div(
            state.openInterest
        );
        state.cumulativeFundingLongX18 += fundingPerShare;
        state.cumulativeFundingShortX18 -= fundingPerShare;
    }
    // If openInterest == 0, no one to socialize against; write off the balance
    balance.vQuoteBalance = 0;
}
```

---

### Proof of Concept

**Attack path:**

1. A perp market has a single active trader (Trader A) who opens a long position and accumulates negative `vQuoteBalance` via funding payments.
2. Trader A closes their position (`balance.amount = 0`), but `balance.vQuoteBalance` remains negative.
3. No other subaccount holds an open position → `state.openInterest == 0`.
4. Trader A's subaccount becomes unhealthy (negative `vQuoteBalance` reduces health below maintenance threshold).
5. A liquidator submits `LiquidateSubaccount` with `productId = type(uint32).max` (finalization mode).
6. `Clearinghouse.liquidateSubaccount` → delegatecall → `ClearinghouseLiq.liquidateSubaccountImpl` → `_finalizeSubaccount` → `perpEngine.socializeSubaccount`.
7. Inside `socializeSubaccount`, `balance.vQuoteBalance < 0` and insurance is insufficient → reaches `fundingPerShare = -balance.vQuoteBalance.div(state.openInterest)` → `state.openInterest == 0` → `MathSD21x18.div` reverts with `"DBZ"`.
8. The entire liquidation finalization reverts. The insolvent subaccount can never be finalized. [5](#0-4) [6](#0-5)

        uint32[] memory _productIds = getProductIds();

            uint32 productId = _productIds[i];
            (State memory state, Balance memory balance) = getStateAndBalance(
                productId,
                subaccount
            );
            if (balance.vQuoteBalance < 0) {
                int128 insuranceCover = MathHelper.min(
                    insurance,
                    -balance.vQuoteBalance

                insurance -= insuranceCover;

    function _finalizeSubaccount(
        IEndpoint.LiquidateSubaccount calldata txn,
        ISpotEngine spotEngine,
        IPerpEngine perpEngine
    ) internal returns (bool) {
        if (txn.productId != type(uint32).max) {
            return false;
        }
        // check whether the subaccount can be finalized:
        // - all perps positions have closed
        // - all spread positions have closed
        // - all spot assets have closed
        // - all positive pnls have been settled

        FinalizeVars memory v;

        v.spotIds = spotEngine.getProductIds();
        v.perpIds = perpEngine.getProductIds();

        require(v.spotIds[0] == QUOTE_PRODUCT_ID);

        // all spot assets (except USDC) must be closed out
        for (uint32 i = 1; i < v.spotIds.length; ++i) {
            uint32 spotId = v.spotIds[i];
            if (spotEngine.getRisk(spotId).longWeightInitialX18 == 0) {
                continue;
            }
            ISpotEngine.Balance memory balance = spotEngine.getBalance(
                spotId,
                txn.liquidatee
            );
            require(balance.amount <= 0, ERR_NOT_FINALIZABLE_SUBACCOUNT);
        }

        for (uint32 i = 0; i < v.perpIds.length; ++i) {
            uint32 perpId = v.perpIds[i];
            IPerpEngine.Balance memory balance = perpEngine.getBalance(
                perpId,
                txn.liquidatee
            );
            require(balance.amount == 0, ERR_NOT_FINALIZABLE_SUBACCOUNT);
        }

        // settle all positive pnl
        for (uint32 i = 0; i < v.perpIds.length; ++i) {
            uint32 perpId = v.perpIds[i];
            IPerpEngine.Balance memory balance = perpEngine.getBalance(
                perpId,
                txn.liquidatee
            );
            if (balance.vQuoteBalance > 0) {
                _settlePnlAgainstLiquidator(
                    txn,
                    perpId,
                    balance.vQuoteBalance,
                    spotEngine,
                    perpEngine
                );
            }
        }

        ISpotEngine.Balance memory quoteBalance = spotEngine.getBalance(
            QUOTE_PRODUCT_ID,
            txn.liquidatee
        );

        // settle all negative pnl until quote balance becomes 0
        for (uint32 i = 0; i < v.perpIds.length; ++i) {
            uint32 perpId = v.perpIds[i];
            IPerpEngine.Balance memory balance = perpEngine.getBalance(
                perpId,
                txn.liquidatee
            );
            if (balance.vQuoteBalance < 0 && quoteBalance.amount > 0) {
                int128 canSettle = MathHelper.max(
                    balance.vQuoteBalance,
                    -quoteBalance.amount
                );
                _settlePnlAgainstLiquidator(
                    txn,
                    perpId,
                    canSettle,
                    spotEngine,
                    perpEngine
                );
                quoteBalance.amount += canSettle;
            }
        }

        v.insurance = insurance;
        v.insurance -= lastLiquidationFees;
        v.canLiquidateMore = (quoteBalance.amount + v.insurance) > 0;

        if (v.canLiquidateMore) {
            for (uint32 i = 1; i < v.spotIds.length; ++i) {
                uint32 spotId = v.spotIds[i];
                ISpotEngine.Balance memory balance = spotEngine.getBalance(
                    spotId,
                    txn.liquidatee
                );
                if (spotEngine.getRisk(spotId).longWeightInitialX18 == 0) {
                    continue;
                }
                require(balance.amount == 0, ERR_NOT_FINALIZABLE_SUBACCOUNT);
            }
        }

        v.insurance = perpEngine.socializeSubaccount(
            txn.liquidatee,
            v.insurance
        );

        // we can assure that quoteBalance must be non positive, because if quoteBalance.amount > 0,
        // there must be 1) no negative pnl in perps, and 2) no liabilities in spot after above actions.
        // however, in this case the liquidatee must be healthy and cannot pass the health check at
        // the beginning.
        int128 insuranceCover = MathHelper.min(
            v.insurance,
            -quoteBalance.amount
        );
        if (insuranceCover > 0) {
            v.insurance -= insuranceCover;
            spotEngine.updateBalance(
                QUOTE_PRODUCT_ID,
                txn.liquidatee,
                insuranceCover
            );
        }
        if (v.insurance <= 0) {
            spotEngine.socializeSubaccount(txn.liquidatee);
        }
        v.insurance += lastLiquidationFees;
        insurance = v.insurance;
        return true;
    }
