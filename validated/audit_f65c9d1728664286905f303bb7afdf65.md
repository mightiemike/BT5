Audit Report

## Title
SwapAllowlistExtension Checks Router Address Instead of Actual User, Allowing Any User to Bypass a Per-User Swap Allowlist via the Router - (File: metric-periphery/contracts/extensions/SwapAllowlistExtension.sol)

## Summary
`SwapAllowlistExtension.beforeSwap` gates swaps by checking the `sender` argument, which is `msg.sender` of `MetricOmmPool.swap()`. When a user routes through `MetricOmmSimpleRouter`, `msg.sender` of `pool.swap()` is the router contract address, not the actual user. A pool admin who allowlists the router to enable router-based swaps for curated users simultaneously opens the pool to every user on the router, completely defeating the per-user allowlist.

## Finding Description
`MetricOmmPool.swap()` passes `msg.sender` as the `sender` argument to `_beforeSwap`: [1](#0-0) 

`SwapAllowlistExtension.beforeSwap` then checks that `sender` against the per-pool allowlist: [2](#0-1) 

`MetricOmmSimpleRouter.exactInputSingle` stores `msg.sender` in transient storage only for payment purposes (`_setNextCallbackContext`), but calls `pool.swap()` with no forwarding of the original caller's address to the pool: [3](#0-2) 

So `sender` delivered to the extension is the router address, not the user's address. The same pattern applies to `exactInput`, `exactOutputSingle`, and `exactOutput`.

For contrast, `DepositAllowlistExtension.beforeAddLiquidity` correctly ignores `sender` and checks `owner` (the economically relevant actor): [4](#0-3) 

**Exploit path:**
1. Pool admin deploys pool with `SwapAllowlistExtension`, allowlists `userA` and the router address (required for `userA` to use the router).
2. `userB` (never allowlisted) calls `MetricOmmSimpleRouter.exactInputSingle(pool, ...)`.
3. Router calls `pool.swap(...)` — `msg.sender` = router.
4. Pool calls `_beforeSwap(router, ...)` — `sender` = router.
5. `allowedSwapper[pool][router] == true` → check passes.
6. `userB`'s swap executes on the curated pool.

There is no way to simultaneously allow `userA` through the router while blocking `userB`, because the router is a single shared address.

## Impact Explanation
The `SwapAllowlistExtension` is a production extension explicitly designed to restrict which addresses may trade on a curated pool. Allowlisting the router — the only way to let any user use the standard UX path — makes the allowlist completely ineffective for all router-mediated swaps. Any user can trade on a pool intended to be restricted, potentially draining LP liquidity at oracle-quoted prices that the pool admin intended only for specific counterparties. This is a direct admin-boundary break triggered by an unprivileged path, meeting the High severity threshold.

## Likelihood Explanation
`MetricOmmSimpleRouter` is the primary user-facing swap entrypoint. Any pool admin who deploys a swap-allowlisted pool and wants their allowlisted users to use the router will naturally allowlist the router address. The bypass is immediately active for all users once the router is allowlisted. No special attacker setup is required — any EOA can call `exactInputSingle` on the router pointing at the curated pool.

## Recommendation
The extension must check the economically relevant actor — the end user — not the intermediary. The preferred fix is to pass the original initiator address through the router via `extensionData` or a dedicated transient slot (the router already uses transient storage for the payer context), and have the extension read it when `msg.sender` is a known router. Alternatively, redesign `beforeSwap` to receive the original initiator address as a separate argument from the pool, analogous to how `beforeAddLiquidity` correctly passes `owner` separately from `sender`.

## Proof of Concept
```
Setup:
  - Deploy pool with SwapAllowlistExtension
  - Pool admin calls setAllowedToSwap(pool, userA, true)
  - Pool admin calls setAllowedToSwap(pool, router, true)  ← required for userA to use the router

Attack:
  - userB (not allowlisted) calls MetricOmmSimpleRouter.exactInputSingle(pool, ...)
  - Router calls pool.swap(...) → msg.sender = router
  - beforeSwap receives sender = router
  - allowedSwapper[pool][router] == true → check passes
  - userB's swap executes on the curated pool
  - userB was never allowlisted; the per-user gate is bypassed
```
