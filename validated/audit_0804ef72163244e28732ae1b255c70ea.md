### Title
Deprecated `verify_transaction_inclusion` Remains a Live Public Entry Point Without Coinbase Proof Validation, Enabling 64-Byte Merkle Forgery — (File: `contract/src/lib.rs`)

### Summary
`verify_transaction_inclusion` is still a callable public contract method. It validates transaction inclusion by computing a Merkle root from a caller-supplied proof but performs **no coinbase Merkle proof check**. The coinbase check was the sole fix introduced in `verify_transaction_inclusion_v2` to close the 64-byte transaction Merkle forgery vulnerability. Because the deprecated function remains reachable, any unprivileged NEAR caller can bypass that fix entirely and obtain a `true` proof result for a non-existent transaction.

### Finding Description
`verify_transaction_inclusion_v2` was introduced specifically to require a coinbase Merkle proof of the same depth as the transaction proof, preventing an attacker from supplying an internal Merkle-tree node as `tx_id`. The fix is applied only inside `verify_transaction_inclusion_v2`; the older function is left intact and public.

The deprecated function:

```rust
#[deprecated(since = "0.5.0", note = "Use `verify_transaction_inclusion_v2` instead.")]
#[pause]
pub fn verify_transaction_inclusion(&self, #[serializer(borsh)] args: ProofArgs) -> bool {
    ...
    merkle_tools::compute_root_from_merkle_proof(
        args.tx_id,
        usize::try_from(args.tx_index).unwrap(),
        &args.merkle_proof,
    ) == header.block_header.merkle_root
}
``` [1](#0-0) 

It accepts any `tx_id` and any `merkle_proof` and returns `true` whenever the computed root matches the stored `merkle_root`. There is no check that `tx_id` is a leaf node rather than an internal node.

`verify_transaction_inclusion_v2` adds the missing component:

```rust
require!(
    merkle_tools::compute_root_from_merkle_proof(
        args.coinbase_tx_id.clone(),
        0usize,
        &args.coinbase_merkle_proof,
    ) == header.block_header.merkle_root,
    "Incorrect coinbase merkle proof"
);
``` [2](#0-1) 

This coinbase check is the **only** guard against the forgery. Because the deprecated function omits it, calling the deprecated function directly bypasses the guard entirely.

The `#[pause]` attribute allows a `PauseManager` to disable the function, but it is active by default and carries no `#[private]` restriction, so any NEAR account can invoke it. [3](#0-2) 

### Impact Explanation
Any downstream NEAR contract or off-chain application that calls `verify_transaction_inclusion` to gate a financial or state-changing action (e.g., releasing bridged funds, minting tokens, settling a bet) can be deceived into accepting a forged Bitcoin transaction inclusion proof. The corrupted value is the **proof result**: the function returns `true` for a `tx_id` that corresponds to an internal Merkle-tree node rather than a real transaction, violating the core security invariant of the light client.

### Likelihood Explanation
The 64-byte transaction Merkle forgery attack is publicly documented (https://www.bitmex.org/blog/64-Byte-Transactions), requires no special privileges, and needs only a valid Bitcoin block header already stored in the contract. An attacker selects any block with two or more transactions, treats the depth-1 internal node as `tx_id`, constructs the one-step proof to the root, and calls the deprecated function. No key material, role, or social engineering is required.

### Recommendation
Remove `verify_transaction_inclusion` from the public ABI entirely, or add `#[private]` to prevent external calls. If backward compatibility is required, inline the coinbase proof check into the deprecated function body so both entry points enforce the same invariant.

### Proof of Concept
1. Pick any Bitcoin block already accepted by the contract that has at least two transactions (e.g., the genesis-adjacent blocks stored during `init`).
2. Compute the depth-1 internal node: `internal = double_sha256(tx0 || tx1)`.
3. The Merkle proof for `internal` at position `0` with depth reduced by one step is the sibling subtree root — a single-element proof that hashes `internal` with its sibling to produce `merkle_root`.
4. Call `verify_transaction_inclusion` with `tx_id = internal`, `tx_index = 0`, `merkle_proof = [sibling]`, `tx_block_blockhash = <known block>`, `confirmations = 1`.
5. The function returns `true` despite no real transaction having that hash, because `compute_root_from_merkle_proof(internal, 0, [sibling]) == merkle_root`. [4](#0-3) [5](#0-4)

    #[pause]
    pub fn verify_transaction_inclusion(&self, #[serializer(borsh)] args: ProofArgs) -> bool {
        require!(
            args.confirmations <= self.gc_threshold,
            "The required number of confirmations exceeds the number of blocks stored in memory"
        );

        let heaviest_block_header = self
            .headers_pool
            .get(&self.mainchain_tip_blockhash)
            .unwrap_or_else(|| env::panic_str(ERR_KEY_NOT_EXIST));
        let target_block_height = self
            .mainchain_header_to_height
            .get(&args.tx_block_blockhash)
            .unwrap_or_else(|| env::panic_str("block does not belong to the current main chain"));

        // Check requested confirmations. No need to compute proof if insufficient confirmations.
        require!(
            (heaviest_block_header.block_height).saturating_sub(target_block_height) + 1
                >= args.confirmations,
            "Not enough blocks confirmed"
        );

        let header = self
            .headers_pool
            .get(&args.tx_block_blockhash)
            .unwrap_or_else(|| env::panic_str("cannot find requested transaction block"));

        require!(!args.merkle_proof.is_empty(), "Merkle proof is empty");

        // compute merkle tree root and check if it matches block's original merkle tree root
        merkle_tools::compute_root_from_merkle_proof(
            args.tx_id,
            usize::try_from(args.tx_index).unwrap(),
            &args.merkle_proof,
        ) == header.block_header.merkle_root
    }

pub fn compute_root_from_merkle_proof(
    transaction_hash: H256,
    transaction_position: usize,
    merkle_proof: &Vec<H256>,
) -> H256 {
    let mut current_hash = transaction_hash;
    let mut current_position = transaction_position;

    for proof_hash in merkle_proof {
        if current_position % 2 == 0 {
            current_hash = compute_hash(&current_hash, proof_hash);
        } else {
            current_hash = compute_hash(proof_hash, &current_hash);
        }
        current_position /= 2;
    }

    current_hash
}
