### Title
Wrong Weight Used for Short Leg in Spread Health Calculation — (`File: core/contracts/BaseEngine.sol`, `core/contracts/Clearinghouse.sol`, `core/contracts/libraries/RiskHelper.sol`)

---

### Summary

`BaseEngine.getCoreRisk` always passes the hardcoded value `1` (positive) to `RiskHelper._getWeightX18`, so `CoreRisk.longWeight` is always the long weight regardless of the actual position direction. This propagates into both `existingWeight` and `_getSpreadWeightX18` inside `Clearinghouse.getHealth`, causing the spread health adjustment to be computed with the wrong weight for the short leg of every spread position. The result is a systematic overstatement of health for all spread positions, which is a direct solvency/accounting corruption analog to the Beanstalk `beanUsdPrice` miscalculation.

---

### Finding Description

**Root cause — `BaseEngine.getCoreRisk` (line 190):**

```solidity
return
    IProductEngine.CoreRisk(
        amount,
        risk.priceX18,
        RiskHelper._getWeightX18(risk, 1, healthType)  // ← hardcoded 1, always long weight
    );
```

The actual position `amount` is stored in `CoreRisk.amount`, but the weight is computed with `1` (positive), so `CoreRisk.longWeight` is always the long weight (`longWeightInitialX18` or `longWeightMaintenanceX18`), never the short weight, regardless of whether the position is actually short.

Compare with `_calculateProductHealth` (line 167), which correctly passes the real `amount`:

```solidity
int128 weight = RiskHelper._getWeightX18(risk, amount, healthType);
```

**Bug propagation 1 — `existingWeight` in `Clearinghouse.getHealth` (lines 125–126):**

```solidity
int128 existingWeight = (spotCoreRisk.longWeight +
    perpCoreRisk.longWeight) / 2;
```

A spread always has one long leg and one short leg. The individual health contributions already applied in `getHealthContribution` used the correct direction-aware weights (long weight for long positions, short weight for short positions). But `existingWeight` uses `longWeight` for both legs, so it understates the average weight already applied when the short leg has `shortWeight > 1`.

**Bug propagation 2 — `RiskHelper._getSpreadWeightX18` (lines 66–70):**

```solidity
if (spotCoreRisk.amount > 0) {
    spreadWeight = ONE - (ONE - perpCoreRisk.longWeight) / 5;  // perp is SHORT, should use shortWeight
} else {
    spreadWeight = ONE - (ONE - spotCoreRisk.longWeight) / 5;  // spot is SHORT, should use shortWeight
}
```

When `spotCoreRisk.amount > 0` (long spot, short perp), the spread weight should be derived from the perp's short weight. Instead it uses `perpCoreRisk.longWeight`, which is always the long weight.

**Net effect on the health adjustment (lines 133–135):**

```solidity
health += basisAmount
    .mul(spotCoreRisk.price + perpCoreRisk.price)
    .mul(spreadWeight - existingWeight);
```

With typical parameters (longWeight = 0.9, shortWeight = 1.1) for a long-spot / short-perp spread:

| | Buggy | Correct |
|---|---|---|
| `existingWeight` | (0.9 + 0.9) / 2 = **0.90** | (0.9 + 1.1) / 2 = **1.00** |
| `spreadWeight` | 1 − (1 − 0.9)/5 = **0.98** | 1 − (1 − 1.1)/5 = **1.02** |
| `spreadWeight − existingWeight` | **0.08** | **0.02** |

The health bonus is **4× larger** than it should be.

---

### Impact Explanation

Every subaccount holding a spread position (long spot + short perp, or short spot + long perp) has its health systematically overstated. Concretely:

- A user can borrow more quote than the protocol's risk parameters permit, because the inflated health passes the `getHealth >= 0` check in `mintNlp` and other operations.
- A spread position that should be liquidatable (health < 0 under correct math) passes `isAboveInitial` / `isUnderInitial` checks, blocking liquidation and leaving the protocol holding undercollateralized exposure.
- The error scales linearly with `basisAmount * (spotPrice + perpPrice)`, so large spread positions produce large absolute health overstatements.

---

### Likelihood Explanation

Any unprivileged user can open a spread position (long spot + short perp) through normal trading. The `getHealth` function is called on every `mintNlp`, every liquidation eligibility check, and every health-gated action. The bug is always active for any subaccount with a non-zero spread. No special permissions, governance, or admin access are required.

---

### Recommendation

In `BaseEngine.getCoreRisk`, pass the actual `amount` instead of the hardcoded `1` so that `CoreRisk.longWeight` reflects the direction-appropriate weight:

```diff
- RiskHelper._getWeightX18(risk, 1, healthType)
+ RiskHelper._getWeightX18(risk, amount, healthType)
```

This ensures that `existingWeight` in `getHealth` and the weight used in `_getSpreadWeightX18` both reflect the actual short weight for the short leg of the spread, making `spreadWeight − existingWeight` correct.

---

### Proof of Concept

1. Alice opens a long spot position of 100 units at price 1.0 and a short perp position of 100 units at price 1.0 (a delta-neutral spread).
2. Risk parameters: `longWeightInitial = 0.9e18`, `shortWeightInitial = 1.1e18`.
3. `getCoreRisk` for the perp returns `longWeight = 0.9e18` (hardcoded `1` passed to `_getWeightX18`), even though the perp position is short.
4. In `getHealth`:
   - `existingWeight = (0.9 + 0.9) / 2 = 0.9` (correct would be `(0.9 + 1.1) / 2 = 1.0`)
   - `spreadWeight = 1 − (1 − 0.9)/5 = 0.98` (correct would be `1 − (1 − 1.1)/5 = 1.02`)
   - Health bonus = `100 * (1.0 + 1.0) * (0.98 − 0.90) = 16` (correct would be `100 * 2.0 * 0.02 = 4`)
5. Alice's health is overstated by 12 units of quote, allowing her to borrow 12 additional quote that the risk model does not actually support. [1](#0-0) [2](#0-1) [3](#0-2) [4](#0-3)

```text
    function _getSpreadWeightX18(
        IProductEngine.CoreRisk memory perpCoreRisk,
        IProductEngine.CoreRisk memory spotCoreRisk,
        IProductEngine.HealthType healthType
    ) internal pure returns (int128) {
        if (healthType == IProductEngine.HealthType.PNL) {
            return ONE;
        }
        int128 spreadWeight;
        if (spotCoreRisk.amount > 0) {
            spreadWeight = ONE - (ONE - perpCoreRisk.longWeight) / 5;
        } else {
            spreadWeight = ONE - (ONE - spotCoreRisk.longWeight) / 5;
        }
        int128 maxSpreadWeight;
        if (healthType == IProductEngine.HealthType.INITIAL) {
            maxSpreadWeight = ONE - ONE / 100; // 0.99
        } else {
            maxSpreadWeight = ONE - (ONE * 6) / 1000; // 0.994
        }
        if (spreadWeight > maxSpreadWeight) {
            spreadWeight = maxSpreadWeight;
        }
        return spreadWeight;
    }
```
