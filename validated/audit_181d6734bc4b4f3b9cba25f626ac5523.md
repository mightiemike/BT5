### Title
NFT Metadata Can Be Mutated by Denom Creator Immediately Before Transfer, Deceiving Buyers ŌĆö (File: `x/nft/keeper/keeper.go`)

---

### Summary

The `EditNFT` message handler allows the denom creator, while they still own an NFT, to silently mutate the token's `URI`, `Name`, and `Data` fields at any time before a `TransferNFT` is executed. A malicious seller who is also the denom creator can agree to sell an NFT at a price reflecting its current metadata, then call `MsgEditNFT` in the same block or just before the transfer to replace the URI with one pointing to worthless content. The buyer receives an NFT whose on-chain metadata no longer matches what was agreed upon.

---

### Finding Description

`EditNFT` in `x/nft/keeper/keeper.go` enforces two checks: the caller must be the current owner of the token (`IsOwner`) **and** the creator of the denom (`IsDenomCreator`). [1](#0-0) 

Because both conditions must hold simultaneously, only the denom creator who still holds the NFT can invoke this path. This is precisely the situation during a pending sale: the seller (denom creator) retains ownership until `MsgTransferNFT` is submitted. In that window ŌĆö which can be as short as a single block ŌĆö the seller can call `MsgEditNFT` to overwrite `nft.URI`, `nft.Name`, and `nft.Data`: [2](#0-1) 

The `MsgEditNFT` message is a standard, unprivileged Cosmos SDK transaction reachable by any account that created a denom: [3](#0-2) 

The `TransferOwner` function that executes the actual sale does **not** snapshot or validate the metadata at transfer time ŌĆö it only swaps the owner field: [4](#0-3) 

The `BaseNFT` struct stores `URI`, `Name`, and `Data` as mutable string fields with no immutability guarantee after minting: [5](#0-4) 

---

### Impact Explanation

The corrupted on-chain value is `BaseNFT.URI` (and optionally `BaseNFT.Name` / `BaseNFT.Data`). A buyer who inspects the NFT before agreeing to a price sees one URI; after the transfer they own an NFT whose URI has been silently replaced. Any off-chain value tied to the original URI (artwork, access credential, provenance record) is lost. The buyer suffers a direct financial loss proportional to the difference between the agreed price and the post-mutation value of the NFT.

---

### Likelihood Explanation

The attack requires no special privilege beyond having issued a denom ŌĆö any user can call `MsgIssueDenom` followed by `MsgMintNFT`. The manipulation window exists in every peer-to-peer NFT sale that is not atomic (i.e., any sale that is not a single transaction containing both the payment and the transfer). The `MsgEditNFT` transaction is cheap and indistinguishable from a legitimate metadata update. Likelihood is medium-high wherever NFTs from this module are traded on non-custodial or non-atomic venues.

---

### Recommendation

1. **Freeze metadata on transfer**: Once an NFT is transferred away from the denom creator, disallow further `EditNFT` calls on that token (or require the new owner's consent).
2. **Immutable URI option**: Add a boolean `immutable` flag to `BaseNFT` or `Denom` that, when set, prevents any `EditNFT` after minting.
3. **Atomic sale primitives**: Document that safe peer-to-peer sales must be performed atomically (e.g., via a single transaction or a custodial escrow contract) so that no `EditNFT` can be interleaved.
4. **Event monitoring**: Emit a distinct event when metadata is changed on a token that has previously been transferred, so indexers and marketplaces can flag suspicious activity.

---

### Proof of Concept

```
# 1. Attacker issues a denom and mints a high-value NFT
chaind tx nft issue myDenom --name="Rare Art" --from=attacker
chaind tx nft mint myDenom token1 --uri="ipfs://QmRareArt" --recipient=attacker --from=attacker

# 2. Victim inspects the NFT, sees URI "ipfs://QmRareArt", agrees to pay 1000 CRO

# 3. Attacker, still the owner, mutates the URI before submitting the transfer
chaind tx nft edit myDenom token1 --uri="ipfs://QmWorthless" --from=attacker

# 4. Attacker transfers the NFT to the victim (same block or next block)
chaind tx nft transfer myDenom token1 --from=attacker --recipient=victim

# 5. Victim now owns token1 with URI "ipfs://QmWorthless"
chaind query nft token myDenom token1
# ŌåÆ uri: "ipfs://QmWorthless"   ŌåÉ not what was agreed upon
```

func (k Keeper) EditNFT(
	ctx sdk.Context, denomID, tokenID, tokenNm,
	tokenURI, tokenData string, owner sdk.AccAddress,
) error {
	if !k.HasDenomID(ctx, denomID) {
		return sdkerrors.Wrapf(types.ErrInvalidDenom, "denom ID %s not exists", denomID)
	}

	nft, err := k.IsOwner(ctx, denomID, tokenID, owner)
	if err != nil {
		return err
	}

	_, err = k.IsDenomCreator(ctx, denomID, owner)
	if err != nil {
		return err
	}

	if types.Modified(tokenNm) {
		nft.Name = tokenNm
	}

	if types.Modified(tokenURI) {
		nft.URI = tokenURI
	}

	if types.Modified(tokenData) {
		nft.Data = tokenData
	}

	k.setNFT(ctx, denomID, nft)

	return nil

// TransferOwner transfers the ownership of the given NFT to the new owner
func (k Keeper) TransferOwner(
	ctx sdk.Context, denomID, tokenID string, srcOwner, dstOwner sdk.AccAddress,
) error {
	if !k.HasDenomID(ctx, denomID) {
		return sdkerrors.Wrapf(types.ErrInvalidDenom, "denom ID %s not exists", denomID)
	}

	nft, err := k.IsOwner(ctx, denomID, tokenID, srcOwner)
	if err != nil {
		return err
	}

	nft.Owner = dstOwner.String()

	k.setNFT(ctx, denomID, nft)
	k.swapOwner(ctx, denomID, tokenID, srcOwner, dstOwner)
	return nil
}

//nolint:dupl
func (m msgServer) EditNFT(goCtx context.Context, msg *types.MsgEditNFT) (*types.MsgEditNFTResponse, error) {
	sender, err := sdk.AccAddressFromBech32(msg.Sender)
	if err != nil {
		return nil, err
	}

	ctx := sdk.UnwrapSDKContext(goCtx)
	if err := m.Keeper.EditNFT(ctx, msg.DenomId, msg.Id,
		msg.Name,
		msg.URI,
		msg.Data,
		sender,
	); err != nil {
		return nil, err
	}

	ctx.EventManager().EmitEvents(sdk.Events{
		sdk.NewEvent(
			types.EventTypeEditNFT,
			sdk.NewAttribute(types.AttributeKeyTokenID, msg.Id),
			sdk.NewAttribute(types.AttributeKeyDenomID, msg.DenomId),
			sdk.NewAttribute(types.AttributeKeyTokenURI, msg.URI),
			sdk.NewAttribute(types.AttributeKeyOwner, msg.Sender),
		),
		sdk.NewEvent(
			sdk.EventTypeMessage,
			sdk.NewAttribute(sdk.AttributeKeyModule, types.AttributeValueCategory),
			sdk.NewAttribute(sdk.AttributeKeySender, msg.Sender),
		),
	})

	return &types.MsgEditNFTResponse{}, nil
}
