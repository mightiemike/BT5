### Title
Protobuf `ValidResourceBounds` deserialization silently downgrades `AllResources` to `L1Gas` when `l2_gas` and `l1_data_gas` are both zero, producing a divergent transaction hash preimage - (File: `crates/apollo_protobuf/src/converters/transaction.rs`)

### Summary

The protobuf deserializer for `ValidResourceBounds` silently changes the variant from `AllResources` to `L1Gas` whenever both `l2_gas` and `l1_data_gas` are zero. Because `get_tip_resource_bounds_hash` includes the `l1_data_gas` packed felt **only** for the `AllResources` variant, the hash preimage shrinks by one element after a protobuf round-trip. A transaction that was signed and committed under the `AllResources` hash will produce a different hash when a syncing node deserializes it, breaking hash-based validation.

### Finding Description

`InvokeTransactionV3` (the canonical `Transaction` type used in the sync/storage path) carries `resource_bounds: ValidResourceBounds`. When a V3 transaction is serialized to protobuf for P2P sync, `From<ValidResourceBounds> for protobuf::ResourceBounds` emits identical wire bytes for both `L1Gas(X)` and `AllResources { l1_gas: X, l2_gas: 0, l1_data_gas: 0 }`:

```rust
// crates/apollo_protobuf/src/converters/transaction.rs  lines 471-489
impl From<ValidResourceBounds> for protobuf::ResourceBounds {
    fn from(value: ValidResourceBounds) -> Self {
        match value {
            ValidResourceBounds::L1Gas(l1_gas) => protobuf::ResourceBounds {
                l1_gas: Some(l1_gas.into()),
                l2_gas: Some(value.get_l2_bounds().into()),   // → zero
                l1_data_gas: Some(ResourceBounds::default().into()), // → zero
            },
            ValidResourceBounds::AllResources(AllResourceBounds { l1_gas, l2_gas, l1_data_gas }) =>
                protobuf::ResourceBounds {
                    l1_gas: Some(l1_gas.into()),
                    l2_gas: Some(l2_gas.into()),          // zero if zero
                    l1_data_gas: Some(l1_data_gas.into()), // zero if zero
                },
        }
    }
}
```

On the receiving side, the deserializer applies a value-based heuristic to reconstruct the variant:

```rust
// crates/apollo_protobuf/src/converters/transaction.rs  lines 417-436
Ok(if l1_data_gas.is_zero() && l2_gas.is_zero() {
    ValidResourceBounds::L1Gas(l1_gas)   // ← variant silently changed
} else {
    ValidResourceBounds::AllResources(AllResourceBounds { l1_gas, l2_gas, l1_data_gas })
})
```

The hash function branches on this variant:

```rust
// crates/starknet_api/src/transaction_hash.rs  lines 203-208
resource_felts.extend(match resource_bounds {
    ValidResourceBounds::L1Gas(_) => vec![],                          // 2 elements total
    ValidResourceBounds::AllResources(all_resources) => {
        vec![get_concat_resource(&all_resources.l1_data_gas, L1_DATA_GAS)?]  // 3 elements total
    }
});
```

`AllResources { l1_gas: X, l2_gas: 0, l1_data_gas: 0 }` hashes as:

```
poseidon(tip, packed(L1_GAS, X), packed(L2_GAS, 0), packed(L1_DATA_GAS, 0))
```

After the round-trip the same wire bytes are decoded as `L1Gas(X)`, which hashes as:

```
poseidon(tip, packed(L1_GAS, X), packed(L2_GAS, 0))
```

`packed(L1_DATA_GAS, 0)` is **not** the zero felt — it encodes the non-zero `L1_DATA_GAS` resource-name constant in the upper 56 bits — so the two Poseidon outputs are distinct.

The gateway always computes and stores the hash under `AllResources` (it uses `AllResourceBounds` directly, never `ValidResourceBounds`). The user signs that hash. After protobuf sync the receiving node recomputes a different hash, so `validate_transaction_hash` fails for every such transaction.

### Impact Explanation

**High. Transaction conversion or signature/hash logic binds the wrong hash type or executable payload.**

A syncing node that deserializes a committed block containing a transaction with `AllResources { l2_gas: 0, l1_data_gas: 0 }` will recompute a hash that does not match the stored/transmitted hash. This causes the syncing node to reject the block or store a mismatched hash, breaking chain consistency and potentially causing a permanent sync failure for any block that contains such a transaction.

### Likelihood Explanation

**High.** The gateway's stateless validator accepts any `AllResourceBounds` where at least one resource bound is non-zero. A transaction with `l1_gas > 0`, `l2_gas = 0`, `l1_data_gas = 0` passes all gateway checks. The user signs the `AllResources` hash (3 resource bounds). The transaction is sequenced and committed. Any node that later syncs the block via protobuf will silently decode the resource bounds as `L1Gas` and compute a divergent hash. No privileged access is required; any unprivileged user can trigger this by submitting a normal invoke transaction with only `l1_gas` set.

### Recommendation

Replace the value-based variant reconstruction with an explicit wire-level discriminant. The simplest fix is to add a boolean or enum field to `protobuf::ResourceBounds` that records whether the transaction was signed under the two-resource (`L1Gas`) or three-resource (`AllResources`) scheme. Alternatively, always deserialize to `AllResources` when all three fields are present on the wire (even if zero), and only fall back to `L1Gas` when the `l1_data_gas` field is absent (the pre-0.13.3 backward-compatibility case):

```rust
// crates/apollo_protobuf/src/converters/transaction.rs
impl TryFrom<protobuf::ResourceBounds> for ValidResourceBounds {
    fn try_from(value: protobuf::ResourceBounds) -> Result<Self, Self::Error> {
        let l1_gas = value.l1_gas.ok_or(missing("ResourceBounds::l1_gas"))?.try_into()?;
        let l2_gas = value.l2_gas.ok_or(missing("ResourceBounds::l2_gas"))?.try_into()?;
        // Only use L1Gas variant when l1_data_gas is absent (pre-0.13.3 wire format).
        match value.l1_data_gas {
            None => Ok(ValidResourceBounds::L1Gas(l1_gas)),
            Some(l1_data_gas) => {
                let l1_data_gas: ResourceBounds = l1_data_gas.try_into()?;
                Ok(ValidResourceBounds::AllResources(AllResourceBounds { l1_gas, l2_gas, l1_data_gas }))
            }
        }
    }
}
```

The serializer for `L1Gas` must then omit `l1_data_gas` (set it to `None`) rather than emitting a zero value, so the discriminant is preserved across the round-trip.

### Proof of Concept

```rust
use starknet_api::transaction::fields::{
    AllResourceBounds, ResourceBounds, GasAmount, GasPrice, ValidResourceBounds,
};
use starknet_api::transaction::fields::Tip;
use starknet_api::transaction_hash::get_tip_resource_bounds_hash;
use apollo_protobuf::protobuf;
use std::convert::TryInto;

fn main() {
    // A V3 transaction with AllResources but zero l2_gas and l1_data_gas.
    let original = ValidResourceBounds::AllResources(AllResourceBounds {
        l1_gas:      ResourceBounds { max_amount: GasAmount(100), max_price_per_unit: GasPrice(1) },
        l2_gas:      ResourceBounds::default(),   // zero
        l1_data_gas: ResourceBounds::default(),   // zero
    });
    let tip = Tip(0);

    // Hash as signed by the user / computed by the gateway.
    let hash_before = get_tip_resource_bounds_hash(&original, &tip).unwrap();

    // Protobuf round-trip (sync path).
    let proto: protobuf::ResourceBounds = original.into();
    // proto.l2_gas = Some(zero), proto.l1_data_gas = Some(zero)

    let after: ValidResourceBounds = proto.try_into().unwrap();
    // after == ValidResourceBounds::L1Gas(...)  ← variant changed!

    let hash_after = get_tip_resource_bounds_hash(&after, &tip).unwrap();

    // Hashes diverge: AllResources includes packed(L1_DATA_GAS, 0) in the preimage.
    assert_ne!(hash_before, hash_after,
        "hash_before={hash_before:?}  hash_after={hash_after:?}");
}

```rust
pub fn get_tip_resource_bounds_hash(
    resource_bounds: &ValidResourceBounds,
    tip: &Tip,
) -> Result<Felt, StarknetApiError> {
    let l1_resource_bounds = resource_bounds.get_l1_bounds();
    let l2_resource_bounds = resource_bounds.get_l2_bounds();

    // L1 and L2 gas bounds always exist.
    // Old V3 txs always have L2 gas bounds of zero, but they exist.
    let mut resource_felts = vec![
        get_concat_resource(&l1_resource_bounds, L1_GAS)?,
        get_concat_resource(&l2_resource_bounds, L2_GAS)?,
    ];

    // For new V3 txs, need to also hash the data gas bounds.
    resource_felts.extend(match resource_bounds {
        ValidResourceBounds::L1Gas(_) => vec![],
        ValidResourceBounds::AllResources(all_resources) => {
            vec![get_concat_resource(&all_resources.l1_data_gas, L1_DATA_GAS)?]
        }
    });

    Ok(HashChain::new().chain(&tip.0.into()).chain_iter(resource_felts.iter()).get_poseidon_hash())
}
