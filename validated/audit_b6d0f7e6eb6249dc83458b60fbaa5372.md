### Title
Duplicate Position IDs in `MsgClaimTierRewards` Enable Double-Claiming of Bonus Rewards from the Rewards Pool — (`File: x/tieredrewards/keeper/msg_server.go`)

---

### Summary

`ClaimTierRewards` in `msg_server.go` loads position states into a slice without deduplicating `msg.PositionIds`. When the same position ID appears twice, `claimRewardsAndUpdatesPositions` processes a stale in-memory copy of the position on the second pass, recomputing and paying the same bonus segment a second time from the `RewardsPoolName` module account.

---

### Finding Description

`ClaimTierRewards` iterates over `msg.PositionIds`, loads each position from the store, validates ownership, and appends to a `positions` slice: [1](#0-0) 

There is no deduplication guard. If `msg.PositionIds = [1, 1]`, the slice becomes `[posState_1, posState_1]` — two independent in-memory copies of the same position, both reflecting the pre-claim state.

`claimRewardsAndUpdatesPositions` then iterates over this slice: [2](#0-1) 

On the **first** pass for position 1:
- `claimBaseRewards` calls `WithdrawDelegationRewards` — drains accrued base rewards (returns 0 on a second call, so base rewards are safe).
- `processEventsAndClaimBonus` reads `pos.LastBonusAccrual` (= `T0`) and `pos.LastEventSeq` (= `S0`) from the in-memory copy, walks validator events, computes the final open segment `[T0, blockTime]`, and pays bonus coins from `RewardsPoolName` to the owner.
- `applyBonusAccrualCheckpoint` advances `pos.LastBonusAccrual` to `blockTime` **in the in-memory copy only**.
- `setPosition` persists the updated position to the store. [3](#0-2) 

On the **second** pass for the duplicate entry:
- `pos` is `positions[1]` — the **stale** copy still holding `LastBonusAccrual = T0` and `LastEventSeq = S0`.
- `processEventsAndClaimBonus` recomputes the final segment `[T0, blockTime]` identically and issues a second `SendCoinsFromModuleToAccount` from `RewardsPoolName`. [4](#0-3) 

The second `setPosition` call overwrites the store with the same checkpoint values, so the on-chain position record looks correct — only the module account balance is silently drained.

---

### Impact Explanation

The `RewardsPoolName` module account balance is reduced by at least one extra bonus payment per duplicate position ID per `MsgClaimTierRewards` call. An owner with `N` duplicate entries in a single message receives `N × bonus` instead of `1 × bonus`. With a sufficiently large position and accrued bonus, repeated submissions drain the shared rewards pool, depriving all other participants of their entitled bonus rewards.

**Corrupted value**: `types.RewardsPoolName` module account coin balance.

---

### Likelihood Explanation

Any position owner can craft a `MsgClaimTierRewards` transaction with a repeated position ID. No privileged role, leaked key, or social engineering is required. The transaction is a standard Cosmos SDK message reachable via CLI or any signing client. The only practical constraint is that the rewards pool must hold sufficient balance to cover the duplicate payment (checked by `sufficientBonusPoolBalance`), which is satisfied whenever the pool is non-trivially funded.

---

### Recommendation

Add a deduplication check before appending to the `positions` slice in `ClaimTierRewards`:

```go
seen := make(map[uint64]struct{}, len(msg.PositionIds))
for _, posId := range msg.PositionIds {
    if _, dup := seen[posId]; dup {
        return nil, errorsmod.Wrapf(sdkerrors.ErrInvalidRequest, "duplicate position id: %d", posId)
    }
    seen[posId] = struct{}{}
    // ... existing load + validate logic
}
```

Alternatively, enforce uniqueness at the `msg.Validate()` level in `msg_validate.go` so the check is applied uniformly across all entry paths (direct tx, authz, feegrant).

---

### Proof of Concept

1. Owner creates a tiered position (ID = 42) and lets bonus accrue over several blocks.
2. Owner submits `MsgClaimTierRewards{Owner: owner, PositionIds: [42, 42]}`.
3. `ClaimTierRewards` loads position 42 twice, appending two stale copies to `positions`.
4. First loop iteration: `processEventsAndClaimBonus` computes segment `[LastBonusAccrual, blockTime]`, pays `X` coins from `RewardsPoolName` to owner, updates in-memory copy.
5. Second loop iteration: stale copy still has old `LastBonusAccrual`; `processEventsAndClaimBonus` recomputes the same segment, pays another `X` coins from `RewardsPoolName` to owner.
6. Owner receives `2X` bonus; `RewardsPoolName` balance decreases by `2X`; position checkpoint in store is identical to a legitimate single claim. [5](#0-4) [6](#0-5)

func (ms msgServer) ClaimTierRewards(ctx context.Context, msg *types.MsgClaimTierRewards) (*types.MsgClaimTierRewardsResponse, error) {
	if err := msg.Validate(); err != nil {
		return nil, err
	}

	positions := make([]types.PositionState, 0, len(msg.PositionIds))
	for _, posId := range msg.PositionIds {
		pos, err := ms.getPositionState(ctx, posId)
		if err != nil {
			return nil, err
		}

		if err := ms.validateClaimRewards(pos.Position, msg.Owner); err != nil {
			return nil, err
		}

		positions = append(positions, pos)
	}

	totalBase, totalBonus, err := ms.claimRewardsAndUpdatesPositions(ctx, positions)
	if err != nil {
		return nil, err
	}

	sdkCtx := sdk.UnwrapSDKContext(ctx)
	if err := sdkCtx.EventManager().EmitTypedEvent(&types.EventTierRewardsClaimed{
		Owner:        msg.Owner,
		PositionIds:  msg.PositionIds,
		BaseRewards:  totalBase,
		BonusRewards: totalBonus,
	}); err != nil {
		return nil, err
	}

	return &types.MsgClaimTierRewardsResponse{
		BaseRewards:  totalBase,
		BonusRewards: totalBonus,
		PositionIds:  msg.PositionIds,
	}, nil
}

func (k Keeper) claimRewardsAndUpdatesPositions(ctx context.Context, positions []types.PositionState) (sdk.Coins, sdk.Coins, error) {
	totalBase := sdk.NewCoins()
	totalBonus := sdk.NewCoins()

	for i := range positions {
		pos := &positions[i]

		if !pos.IsDelegated() {
			continue
		}

		base, err := k.claimBaseRewards(ctx, *pos)
		if err != nil {
			return nil, nil, err
		}
		totalBase = totalBase.Add(base...)

		bonus, err := k.processEventsAndClaimBonus(ctx, pos)
		if err != nil {
			return nil, nil, err
		}
		totalBonus = totalBonus.Add(bonus...)

		if err := k.setPosition(ctx, pos.Position, nil); err != nil {
			return nil, nil, err
		}
	}

	return totalBase, totalBonus, nil

func (k Keeper) processEventsAndClaimBonus(ctx context.Context, pos *types.PositionState) (sdk.Coins, error) {
	// Rewards should have been claimed before undelegation
	if !pos.IsDelegated() {
		return sdk.NewCoins(), nil
	}

	valAddr, err := sdk.ValAddressFromBech32(pos.Delegation.ValidatorAddress)
	if err != nil {
		return nil, err
	}

	events, err := k.getValidatorEventsSince(ctx, valAddr, pos.LastEventSeq)
	if err != nil {
		return nil, err
	}

	sdkCtx := sdk.UnwrapSDKContext(ctx)
	blockTime := sdkCtx.BlockTime()

	totalBonus := math.ZeroInt()
	// Use the persisted bonded state from the last replay, not a hardcoded default.
	// This prevents overpaying bonus for unbonded gaps between claims.
	bonded := pos.LastKnownBonded
	segmentStart := pos.LastBonusAccrual

	tier, err := k.getTier(ctx, pos.TierId)
	if err != nil {
		return nil, err
	}

	for _, entry := range events {
		evt := entry.Event

		if bonded {
			// Compute bonus for the bonded segment [segmentStart, eventTime]
			// using the snapshot rate at the event.
			bonus := k.computeSegmentBonus(*pos, tier, segmentStart, evt.Timestamp, evt.TokensPerShare)
			totalBonus = totalBonus.Add(bonus)
		}

		// Update bonded state based on event type.
		switch evt.EventType {
		case types.ValidatorEventType_VALIDATOR_EVENT_TYPE_UNBOND:
			bonded = false
		case types.ValidatorEventType_VALIDATOR_EVENT_TYPE_BOND:
			bonded = true
		case types.ValidatorEventType_VALIDATOR_EVENT_TYPE_SLASH:
			// Slash doesn't change bonded state.
		}

		segmentStart = evt.Timestamp
		pos.UpdateLastEventSeq(entry.Seq)

		// Decrement reference count.
		if err := k.decrementEventRefCount(ctx, valAddr, entry.Seq); err != nil {
			return nil, err
		}
	}

	val, err := k.stakingKeeper.GetValidator(ctx, valAddr)
	if err != nil {
		return nil, err
	}
	// Defensive: validator bond status check
	if bonded && val.IsBonded() {
		currentRate, err := k.getTokensPerShare(ctx, valAddr)
		if err != nil {
			return nil, err
		}
		bonus := k.computeSegmentBonus(*pos, tier, segmentStart, blockTime, currentRate)
		totalBonus = totalBonus.Add(bonus)
	}

	applyBonusAccrualCheckpoint(&pos.Position, blockTime)
	// Persist the bonded state so the next replay starts correctly.
	pos.UpdateLastKnownBonded(bonded)

	if totalBonus.IsZero() {
		return sdk.NewCoins(), nil
	}

	bondDenom, err := k.stakingKeeper.BondDenom(ctx)
	if err != nil {
		return nil, err
	}

	bonusCoins := sdk.NewCoins(sdk.NewCoin(bondDenom, totalBonus))

	if err := k.sufficientBonusPoolBalance(ctx, bonusCoins); err != nil {
		return nil, err
	}

	ownerAddr, err := sdk.AccAddressFromBech32(pos.Owner)
	if err != nil {
		return nil, errorsmod.Wrap(sdkerrors.ErrInvalidAddress, "invalid owner address")
	}

	if err := k.bankKeeper.SendCoinsFromModuleToAccount(ctx, types.RewardsPoolName, ownerAddr, bonusCoins); err != nil {
		return nil, err
	}

	if err := sdkCtx.EventManager().EmitTypedEvent(&types.EventBonusRewardsClaimed{
		PositionId: pos.Id,
		Owner:      pos.Owner,
		Rewards:    bonusCoins,
	}); err != nil {
		return nil, err
	}

	return bonusCoins, nil
}
