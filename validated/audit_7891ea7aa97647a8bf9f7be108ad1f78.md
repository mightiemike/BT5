### Title
`burnNlp` Lacks Minimum Quote Output Protection — (File: `core/contracts/Clearinghouse.sol`)

---

### Summary

The `burnNlp` function in `Clearinghouse.sol` calculates the quote amount a user receives by multiplying their NLP burn amount by a sequencer-supplied `oraclePriceX18` that is **not committed to in the user's EIP-712 signature**. The `BurnNlp` struct contains no `minQuoteAmount` field, so users have no on-chain protection against receiving less quote than expected when the oracle price moves between signing and execution. NLP tokens are burned and the nonce is consumed regardless of the quote amount actually received.

---

### Finding Description

The user signs a `BurnNlp` transaction containing only three fields:

```solidity
struct BurnNlp {
    bytes32 sender;
    uint128 nlpAmount;
    uint64 nonce;
}
``` [1](#0-0) 

The `oraclePriceX18` that determines the quote output lives in `SignedBurnNlp` **outside** the `tx` field the user signs:

```solidity
struct SignedBurnNlp {
    BurnNlp tx;
    bytes signature;
    int128 oraclePriceX18;      // NOT covered by user's signature
    int128[] nlpPoolRebalanceX18;
}
``` [2](#0-1) 

In `EndpointTx.processTransactionImpl`, `validateSignedTx` is called with the full `transaction` bytes, but `computeDigest` follows the EIP-712 pattern and hashes only the typed `BurnNlp tx` struct — not `oraclePriceX18`. The sequencer then passes `signedTx.oraclePriceX18` directly to `clearinghouse.burnNlp`:

```solidity
validateSignedTx(signedTx.tx.sender, signedTx.tx.nonce, transaction, signedTx.signature, true);
// ...
clearinghouse.burnNlp(signedTx.tx, signedTx.oraclePriceX18, nlpPools, signedTx.nlpPoolRebalanceX18);
``` [3](#0-2) 

Inside `burnNlp`, the quote output is computed entirely from the sequencer-provided price with no floor check:

```solidity
int128 quoteAmount = nlpAmount.mul(oraclePriceX18);
int128 burnFee = MathHelper.max(ONE, quoteAmount / 1000);
quoteAmount = MathHelper.max(0, quoteAmount - burnFee);
``` [4](#0-3) 

The NLP balance is then unconditionally debited and the quote credited at whatever `quoteAmount` results:

```solidity
spotEngine.updateBalance(NLP_PRODUCT_ID, txn.sender, -nlpAmount);
spotEngine.updateBalance(NLP_PRODUCT_ID, N_ACCOUNT, nlpAmount);
if (quoteAmount > 0) {
    spotEngine.updateBalance(QUOTE_PRODUCT_ID, txn.sender, quoteAmount);
``` [5](#0-4) 

There is no `require(quoteAmount >= minQuoteAmount)` guard anywhere in the execution path.

---

### Impact Explanation

A user who signs a `BurnNlp` for `nlpAmount` NLP tokens commits irrevocably to burning those tokens. If the NLP oracle price falls between the moment the user signs and the moment the sequencer includes the transaction in a batch, the user receives a `quoteAmount` materially below their expectation. The nonce is incremented and the NLP balance is permanently reduced — the user cannot re-submit the same signed transaction at a better price. This is a direct, unrecoverable asset loss: the user surrenders NLP tokens and receives fewer quote tokens than the price at signing time implied.

The same structural flaw exists symmetrically in `mintNlp`: the user commits to spending a fixed `quoteAmount` but the `nlpAmount` received is `quoteAmount.div(oraclePriceX18)`, which shrinks if the oracle price rises before execution. [6](#0-5) 

---

### Likelihood Explanation

The sequencer processes transactions in ordered batches. There is an inherent latency between when a user signs a `BurnNlp` transaction and when the sequencer includes it. During volatile market conditions, the NLP oracle price can move meaningfully within that window. No sequencer compromise is required — an honest sequencer using the current market price at execution time is sufficient to trigger the slippage. The user has no on-chain mechanism to bound the acceptable output.

---

### Recommendation

Add a `minQuoteAmount` field to the `BurnNlp` struct so the user's EIP-712 signature commits to a minimum acceptable output:

```solidity
struct BurnNlp {
    bytes32 sender;
    uint128 nlpAmount;
    uint128 minQuoteAmount;   // add this
    uint64 nonce;
}
```

In `burnNlp`, after computing `quoteAmount`, enforce:

```solidity
require(quoteAmount >= int128(txn.minQuoteAmount), "STH");
```

Apply the same pattern to `MintNlp` (`minNlpAmount`).

---

### Proof of Concept

1. NLP oracle price is `$1.00`. Alice signs `BurnNlp{sender=alice, nlpAmount=1000e18, nonce=7}`, expecting to receive approximately `1000 USDC` minus the burn fee.
2. Before the sequencer includes Alice's transaction, the NLP oracle price drops to `$0.60`.
3. The sequencer submits the batch with `oraclePriceX18 = 0.6e18` (the current honest market price).
4. `burnNlp` executes: `quoteAmount = 1000e18 * 0.6 = 600e18`. After the burn fee, Alice receives approximately `599.4 USDC`.
5. Alice's 1000 NLP tokens are burned and her nonce is incremented. She has no recourse — the signed transaction is spent and cannot be resubmitted.
6. Alice suffers an unprotected loss of ~400 USDC relative to her expectation at signing time, with no on-chain mechanism to have prevented it. [7](#0-6) [8](#0-7) [9](#0-8)

```text
    function burnNlp(
        IEndpoint.BurnNlp calldata txn,
        int128 oraclePriceX18,
        IEndpoint.NlpPool[] calldata nlpPools,
        int128[] calldata nlpPoolRebalanceX18
    ) external onlyEndpoint {
        require(!RiskHelper.isIsolatedSubaccount(txn.sender), ERR_UNAUTHORIZED);

        ISpotEngine spotEngine = _spotEngine();
        spotEngine.updatePrice(NLP_PRODUCT_ID, oraclePriceX18);

        require(txn.nlpAmount <= INT128_MAX, ERR_CONVERSION_OVERFLOW);
        int128 nlpAmount = int128(txn.nlpAmount);
        require(
            spotEngine.getNlpUnlockedBalance(txn.sender).amount >= nlpAmount,
            ERR_UNLOCKED_NLP_INSUFFICIENT
        );
        int128 quoteAmount = nlpAmount.mul(oraclePriceX18);
        int128 burnFee = MathHelper.max(ONE, quoteAmount / 1000);
        quoteAmount = MathHelper.max(0, quoteAmount - burnFee);

        _validateNlpRebalance(nlpPools, nlpPoolRebalanceX18, -quoteAmount);
        for (uint128 i = 0; i < nlpPoolRebalanceX18.length; i++) {
            require(nlpPoolRebalanceX18[i] <= 0, ERR_INVALID_NLP_REBALANCE);
        }

        spotEngine.updateBalance(NLP_PRODUCT_ID, txn.sender, -nlpAmount);
        spotEngine.updateBalance(NLP_PRODUCT_ID, N_ACCOUNT, nlpAmount);

        if (quoteAmount > 0) {
            spotEngine.updateBalance(QUOTE_PRODUCT_ID, txn.sender, quoteAmount);
            _applyNlpRebalance(spotEngine, nlpPools, nlpPoolRebalanceX18);
        }

        require(
            spotEngine.getBalance(NLP_PRODUCT_ID, txn.sender).amount >= 0,
            ERR_SUBACCT_HEALTH
        );
        // Burning NLP can decrease health if the burn fee exceeds the health improvement
        // from the withdrawal. This check prevents malicious actors from deliberately
        // creating unhealthy subaccounts through NLP burns.
        require(
            getHealth(txn.sender, IProductEngine.HealthType.MAINTENANCE) >= 0,
            ERR_SUBACCT_HEALTH
        );
    }
