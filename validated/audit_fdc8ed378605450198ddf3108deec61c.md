### Title
`deserialize_transaction_json_to_starknet_api_tx` strips `l1_data_gas` based solely on `max_amount == "0x0"`, silently misclassifying a valid `AllResources` V3 transaction as `L1Gas` and producing a wrong transaction hash - (File: `crates/starknet_api/src/serde_utils.rs`)

---

### Summary

`deserialize_transaction_json_to_starknet_api_tx` contains an "rpc_v8 fix" that removes the `l1_data_gas` field from a transaction's resource bounds when `l1_data_gas.max_amount == "0x0"` and `l2_gas.max_amount == "0x0"`. The condition does **not** check `l1_data_gas.max_price_per_unit`. A legitimate `AllResources` V3 transaction with `l1_data_gas = {max_amount: 0, max_price_per_unit: X}` (X ≠ 0) and `l2_gas = {max_amount: 0, max_price_per_unit: 0}` will have its `l1_data_gas` silently stripped. The downstream `DeprecatedResourceBoundsMapping::try_into()` then produces `ValidResourceBounds::L1Gas` instead of `ValidResourceBounds::AllResources`. Because `get_tip_resource_bounds_hash` hashes only 2 resource felts for `L1Gas` but 3 for `AllResources`, the recomputed transaction hash diverges from the on-chain hash, causing hash validation to fail and the wrong executable payload to be bound.

---

### Finding Description

**Root cause — `crates/starknet_api/src/serde_utils.rs` lines 187–199:**

```rust
// rpc_v8 fix (remove redundantly added L1DataGas)
let raw_resourcebounds = &raw_transaction["resource_bounds"];
if !raw_resourcebounds.is_null()
    && !raw_resourcebounds["l1_data_gas"].is_null()
    && raw_resourcebounds["l1_data_gas"]["max_amount"] == "0x0"   // ← only checks amount
    && !raw_resourcebounds["l2_gas"].is_null()
    && raw_resourcebounds["l2_gas"]["max_amount"] == "0x0"
{
    raw_transaction["resource_bounds"]
        .as_object_mut()
        .expect("should be map of resource bounds")
        .remove("l1_data_gas");                                    // ← unconditionally strips
}
``` [1](#0-0) 

The intent is to undo a spurious `l1_data_gas` field that RPC v8 added to old pre-0.13.3 `L1Gas` transactions. However, the guard only tests `l1_data_gas["max_amount"] == "0x0"` and `l2_gas["max_amount"] == "0x0"`. It never inspects `l1_data_gas["max_price_per_unit"]`.

**Downstream classification — `crates/starknet_api/src/transaction/fields.rs` lines 575–606:**

After `l1_data_gas` is stripped, `DeprecatedResourceBoundsMapping::try_into()` finds no `L1DataGas` key. Because `l2_gas.max_price_per_unit` is also zero (the only case where the strip does not immediately error), `l2_bounds.is_zero()` is `true`, and the function returns `ValidResourceBounds::L1Gas(l1_gas)` instead of `AllResources`. [2](#0-1) 

**Hash divergence — `crates/starknet_api/src/transaction_hash.rs` lines 188–211:**

`get_tip_resource_bounds_hash` hashes **2 felts** (L1, L2) for `ValidResourceBounds::L1Gas` and **3 felts** (L1, L2, L1DataGas) for `AllResources`. A transaction originally committed on-chain as `AllResources` with a non-zero `l1_data_gas.max_price_per_unit` will produce a different Poseidon hash after the strip, because the `l1_data_gas` packed felt is absent from the preimage. [3](#0-2) 

**Analogy to the seed bug:** Just as `stakeLength()` treats a zeroed-out array slot as if it were absent (returning a count that includes ghost entries), this fix treats a zero-`max_amount` `l1_data_gas` entry as if the entire field were absent — without verifying that `max_price_per_unit` is also zero — and silently removes it, producing a ghost-free but incorrect representation.

### Impact Explanation

Any component that calls `deserialize_transaction_json_to_starknet_api_tx` on a V3 transaction with `l1_data_gas = {max_amount: "0x0", max_price_per_unit: <non-zero>}` and `l2_gas = {max_amount: "0x0", max_price_per_unit: "0x0"}` will:

1. Produce `ValidResourceBounds::L1Gas` instead of `AllResources`.
2. Compute a transaction hash that differs from the on-chain hash.
3. Cause `validate_transaction_hash` to return `false` for a legitimately committed transaction.
4. Bind the wrong executable payload (2-resource-bound fee semantics instead of 3).

This affects the blockifier reexecution path (used for proving, tracing, and simulation), causing authoritative-looking wrong hash/receipt values to be returned. It matches the **High** impact: *"Transaction conversion or signature/hash logic binds the wrong signer, hash, type, or executable payload."*

---

### Likelihood Explanation

A V3 transaction with `l1_data_gas.max_amount = 0` and `l1_data_gas.max_price_per_unit ≠ 0` is structurally valid and passes gateway stateless validation (no rule forbids a non-zero price with a zero amount). Any user who submits such a transaction — intentionally or by using a wallet that sets a non-zero default price — creates a permanently committed on-chain transaction that will be misclassified whenever it passes through this deserializer. The trigger requires no privilege.

---

### Recommendation

Extend the strip condition to also verify that `l1_data_gas.max_price_per_unit` is zero before removing the field:

```rust
// rpc_v8 fix (remove redundantly added L1DataGas)
if !raw_resourcebounds.is_null()
    && !raw_resourcebounds["l1_data_gas"].is_null()
    && raw_resourcebounds["l1_data_gas"]["max_amount"] == "0x0"
    && raw_resourcebounds["l1_data_gas"]["max_price_per_unit"] == "0x0"  // ← add this
    && !raw_resourcebounds["l2_gas"].is_null()
    && raw_resourcebounds["l2_gas"]["max_amount"] == "0x0"
{
    ...remove("l1_data_gas");
}
```

This ensures only a fully-zeroed `l1_data_gas` (both `max_amount` and `max_price_per_unit` equal to `"0x0"`) is treated as the redundant RPC v8 addition, preserving the correct `AllResources` classification for any transaction that carries a non-zero price.

### Proof of Concept

Construct a V3 invoke transaction JSON as returned by an RPC v8 endpoint:

```json
{
  "type": "INVOKE",
  "version": "0x3",
  "resource_bounds": {
    "L1_GAS":      { "max_amount": "0x100", "max_price_per_unit": "0x1" },
    "L2_GAS":      { "max_amount": "0x0",   "max_price_per_unit": "0x0" },
    "L1_DATA_GAS": { "max_amount": "0x0",   "max_price_per_unit": "0xDEAD" }
  },
  ...
}
```

1. The on-chain hash was computed with `AllResources` including the `l1_data_gas` packed felt `pack(L1_DATA_GAS, 0, 0xDEAD)`.
2. `deserialize_transaction_json_to_starknet_api_tx` evaluates the strip condition: `l1_data_gas.max_amount == "0x0"` ✓, `l2_gas.max_amount == "0x0"` ✓ → strips `l1_data_gas`.
3. `DeprecatedResourceBoundsMapping::try_into()` sees no `L1DataGas`, `l2_bounds.is_zero()` = true → returns `ValidResourceBounds::L1Gas(l1_gas)`.
4. `get_tip_resource_bounds_hash` hashes only `[tip, pack(L1_GAS,...), pack(L2_GAS,...)]` — the `l1_data_gas` felt is absent.
5. `validate_transaction_hash(tx, block_number, chain_id, on_chain_hash, opts)` returns `false` — the recomputed hash does not match the committed hash. [1](#0-0) [4](#0-3) [5](#0-4)

impl TryFrom<DeprecatedResourceBoundsMapping> for ValidResourceBounds {
    type Error = StarknetApiError;
    fn try_from(
        resource_bounds_mapping: DeprecatedResourceBoundsMapping,
    ) -> Result<Self, Self::Error> {
        if let (Some(l1_bounds), Some(l2_bounds)) = (
            resource_bounds_mapping.0.get(&Resource::L1Gas),
            resource_bounds_mapping.0.get(&Resource::L2Gas),
        ) {
            match resource_bounds_mapping.0.get(&Resource::L1DataGas) {
                Some(data_bounds) => Ok(Self::AllResources(AllResourceBounds {
                    l1_gas: *l1_bounds,
                    l1_data_gas: *data_bounds,
                    l2_gas: *l2_bounds,
                })),
                None => {
                    if l2_bounds.is_zero() {
                        Ok(Self::L1Gas(*l1_bounds))

                        Err(StarknetApiError::InvalidResourceMappingInitializer(format!(

                             {resource_bounds_mapping:?}",
                        )))
                    }
                }
            }
        } else {
            Err(StarknetApiError::InvalidResourceMappingInitializer(format!(
                "{resource_bounds_mapping:?}",
            )))
        }
    }

```rust
pub fn get_tip_resource_bounds_hash(
    resource_bounds: &ValidResourceBounds,
    tip: &Tip,
) -> Result<Felt, StarknetApiError> {
    let l1_resource_bounds = resource_bounds.get_l1_bounds();
    let l2_resource_bounds = resource_bounds.get_l2_bounds();

    // L1 and L2 gas bounds always exist.
    // Old V3 txs always have L2 gas bounds of zero, but they exist.
    let mut resource_felts = vec![
        get_concat_resource(&l1_resource_bounds, L1_GAS)?,
        get_concat_resource(&l2_resource_bounds, L2_GAS)?,
    ];

    // For new V3 txs, need to also hash the data gas bounds.
    resource_felts.extend(match resource_bounds {
        ValidResourceBounds::L1Gas(_) => vec![],
        ValidResourceBounds::AllResources(all_resources) => {
            vec![get_concat_resource(&all_resources.l1_data_gas, L1_DATA_GAS)?]
        }
    });

    Ok(HashChain::new().chain(&tip.0.into()).chain_iter(resource_felts.iter()).get_poseidon_hash())
}
