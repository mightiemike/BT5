### Title
`ValidResourceBounds` Variant Collapse in Protobuf Conversion Produces a Different Transaction Hash Than the Signed Preimage — (`crates/apollo_protobuf/src/converters/transaction.rs`)

---

### Summary

The protobuf deserializer for `ValidResourceBounds` silently collapses `AllResources(l1_gas=X, l2_gas=0, l1_data_gas=0)` into `L1Gas(X)`. Because `get_tip_resource_bounds_hash` hashes a different number of fields for each variant, the two representations produce **different Poseidon digests** even when the economic commitment is identical. Any V3 transaction whose only non-zero resource bound is `l1_gas` will have its hash recomputed to a different value after a protobuf round-trip, breaking hash canonicalization in the state-sync path.

---

### Finding Description

**Step 1 – Variant-sensitive hash in `get_tip_resource_bounds_hash`**

`get_tip_resource_bounds_hash` in `crates/starknet_api/src/transaction_hash.rs` builds the Poseidon preimage differently depending on the `ValidResourceBounds` variant:

```
L1Gas(X)          → poseidon(tip, L1_GAS(X), L2_GAS(0))          // 3 elements
AllResources(X,0,0) → poseidon(tip, L1_GAS(X), L2_GAS(0), L1_DATA_GAS(0))  // 4 elements
``` [1](#0-0) 

The two hashes are **not equal** because the Poseidon chain over 3 elements ≠ the chain over 4 elements (even when the 4th element is zero).

**Step 2 – Protobuf deserializer collapses the variant**

`TryFrom<protobuf::ResourceBounds> for ValidResourceBounds` in `crates/apollo_protobuf/src/converters/transaction.rs`:

```rust
let l1_data_gas = value.l1_data_gas.unwrap_or_default();   // None → zero
...
Ok(if l1_data_gas.is_zero() && l2_gas.is_zero() {
    ValidResourceBounds::L1Gas(l1_gas)          // ← variant changed
} else {
    ValidResourceBounds::AllResources(...)
})
``` [2](#0-1) 

A message carrying `AllResources(X, 0, 0)` (all three fields present but l2/l1_data zero) is deserialized as `L1Gas(X)`. The comment `// TODO(Shahak): Assert data gas is not none once we remove support for 0.13.2.` confirms the branch is intentionally kept open.

**Step 3 – The serializer always emits `Some(default)` for `l1_data_gas`**

The forward direction (`From<ValidResourceBounds> for protobuf::ResourceBounds`) always emits `l1_data_gas: Some(ResourceBounds::default().into())` for the `L1Gas` arm, so the wire bytes for `L1Gas(X)` and `AllResources(X,0,0)` are **identical**. The round-trip is therefore:

```
AllResources(X, 0, 0)  →  protobuf  →  L1Gas(X)
``` [3](#0-2) 

**Step 4 – Reachable trigger: any V3 transaction with only `l1_gas` set**

The gateway's stateless validator accepts a transaction as long as `max_possible_fee(Tip::ZERO) > 0`. A transaction with `l1_gas = {max_amount: N, max_price: P}` and `l2_gas = l1_data_gas = {0, 0}` satisfies this check and is admitted. [4](#0-3) 

The `From<AllResourceBounds> for ValidResourceBounds` conversion always produces `AllResources`, so the transaction is stored in the DB with the `AllResources(X, 0, 0)` variant and hash H₁. [5](#0-4) 

**Step 5 – Hash divergence on sync**

When a peer node syncs the block via P2P, it deserializes the transaction through `TryFrom<protobuf::ResourceBounds> for ValidResourceBounds`, obtaining `L1Gas(X)`. It then recomputes the hash as H₂ ≠ H₁. `validate_transaction_hash` compares the recomputed hash against the block-committed hash H₁ and fails. [6](#0-5) 

---

### Impact Explanation

The protobuf conversion **binds the wrong hash** to a legitimately submitted transaction. Any syncing node that processes a block containing a V3 `DeployAccountTransactionV3` or `InvokeTransactionV3` whose `l2_gas` and `l1_data_gas` are both zero will compute a hash that diverges from the committed hash, causing the sync to fail or the stored hash to be wrong. This matches the **High** impact category: *"Transaction conversion or signature/hash logic binds the wrong signer, hash, type, or executable payload."*

---

### Likelihood Explanation

The trigger requires no privilege. Any user submitting a V3 transaction with only `l1_gas` non-zero (a common pattern for pre-0.13.3 style transactions) produces the divergent condition. The gateway accepts such transactions today.

---

### Recommendation

Two complementary fixes:

1. **Preserve the variant in the deserializer.** When all three fields are present in the protobuf message (even if zero), always produce `AllResources`. Only fall back to `L1Gas` when `l1_data_gas` is absent (`None`) *and* `l2_gas` is zero — i.e., enforce the TODO comment immediately:

```rust
let l1_data_gas = match value.l1_data_gas {
    Some(v) => {
        let rb: ResourceBounds = v.try_into()?;
        // AllResources path: l1_data_gas was explicitly provided
        return Ok(ValidResourceBounds::AllResources(
            AllResourceBounds { l1_gas, l2_gas, l1_data_gas: rb }
        ));
    }
    None => ResourceBounds::default(),  // legacy 0.13.2 path
};
// Only here: l1_data_gas was absent → legacy L1Gas variant
Ok(if l2_gas.is_zero() {
    ValidResourceBounds::L1Gas(l1_gas)
} else { ... })
```

2. **Canonicalize `get_tip_resource_bounds_hash`.** Make `AllResources(X, 0, 0)` and `L1Gas(X)` produce the same hash by always including the L1_DATA_GAS element (as zero) for the `L1Gas` variant, or by normalizing `AllResources(X, 0, 0)` to `L1Gas(X)` before hashing.

pub fn get_tip_resource_bounds_hash(
    resource_bounds: &ValidResourceBounds,
    tip: &Tip,
) -> Result<Felt, StarknetApiError> {
    let l1_resource_bounds = resource_bounds.get_l1_bounds();
    let l2_resource_bounds = resource_bounds.get_l2_bounds();

    // L1 and L2 gas bounds always exist.
    // Old V3 txs always have L2 gas bounds of zero, but they exist.
    let mut resource_felts = vec![
        get_concat_resource(&l1_resource_bounds, L1_GAS)?,
        get_concat_resource(&l2_resource_bounds, L2_GAS)?,
    ];

    // For new V3 txs, need to also hash the data gas bounds.
    resource_felts.extend(match resource_bounds {
        ValidResourceBounds::L1Gas(_) => vec![],
        ValidResourceBounds::AllResources(all_resources) => {
            vec![get_concat_resource(&all_resources.l1_data_gas, L1_DATA_GAS)?]
        }
    });

    Ok(HashChain::new().chain(&tip.0.into()).chain_iter(resource_felts.iter()).get_poseidon_hash())
}
