### Title
Missing Blocked-Address Guard in `MsgMintNFT` Allows Permanent NFT Lock in Module Account — (`x/nft/keeper/keeper.go`, `x/nft/types/msgs.go`)

---

### Summary

The NFT module's `MsgMintNFT` message handler has no check preventing the `recipient` field from being set to a blocked module account address (e.g., `fee_collector`, `bonded_tokens_pool`). Because module accounts cannot sign transactions, any NFT minted to one is permanently irrecoverable, and the denom's supply counter is permanently inflated.

---

### Finding Description

The full call chain is:

**1. `ValidateBasic` — only validates bech32 format** [1](#0-0) 

The only recipient check is `sdk.AccAddressFromBech32(msg.Recipient)`. Any valid bech32 address — including a module account — passes.

**2. `msgServer.MintNFT` — passes recipient directly to keeper** [2](#0-1) 

No blocked-address check is performed before calling `m.Keeper.MintNFT(...)`.

**3. `keeper.MintNFT` — only checks denom creator, then delegates to `MintNFTUnverified`** [3](#0-2) 

**4. `MintNFTUnverified` — unconditionally sets owner and increments supply** [4](#0-3) 

`setOwner` and `increaseSupply` are called with no validation of the owner address against any blocked-address list.

**5. The `Keeper` struct has no bank keeper or account keeper with blocked-address capability** [5](#0-4) 

A grep across the entire `x/nft/` tree for `BlockedAddr`, `blockedAddr`, or `IsBlockedAddr` returns zero matches — confirming no such guard exists anywhere in the module.

**6. Recovery is impossible**

`BurnNFT` requires the caller to be both the NFT owner and the denom creator: [6](#0-5) 

`TransferOwner` requires the caller to be the current owner: [7](#0-6) 

Since a module account cannot sign transactions, neither `BurnNFT` nor `TransferOwner` can ever be called with the module account as the authorizing signer. The NFT is permanently locked and the supply counter permanently inflated.

---

### Impact Explanation

- **Supply corruption**: `GetTotalSupply` for the denom returns a count that includes permanently inaccessible tokens, misrepresenting actual circulating supply.
- **Permanent asset lock**: The minted NFT is owned by a module account that can never sign a transaction, making it irrecoverable by any on-chain path.
- **Scarcity manipulation**: A denom creator can inflate apparent supply while those tokens are functionally burned, deceiving secondary-market participants about collection scarcity.

---

### Likelihood Explanation

- Issuing a denom is permissionless — any account can become a denom creator.
- Module account addresses are deterministic and publicly known (SHA-256 of the module name, truncated to 20 bytes).
- No special privileges, governance, or operator access are required beyond being the denom creator.

---

### Recommendation

Add a blocked-address check in `msgServer.MintNFT` (or in `keeper.MintNFT`) before writing state. The NFT module's `Keeper` should accept a `BlockedAddrs map[string]bool` (populated from `app.BlockedAddrs()`) and reject any recipient whose address appears in that map, mirroring the pattern used by the bank module's `SendCoins`.

---

### Proof of Concept

```go
// Pseudocode — directly exercisable as a keeper unit test
feeCollectorAddr := authtypes.NewModuleAddress(authtypes.FeeCollectorName)

// Step 1: issue denom as attacker
err := keeper.IssueDenom(ctx, "testdenom", "Test", "", "", attackerAddr)
require.NoError(t, err)

// Step 2: mint NFT to fee_collector — ValidateBasic passes, keeper passes
err = keeper.MintNFT(ctx, "testdenom", "token1", "", "", "", attackerAddr, feeCollectorAddr)
require.NoError(t, err) // succeeds — no blocked-address guard

// Step 3: supply is now 1, but NFT is permanently locked
supply := keeper.GetTotalSupply(ctx, "testdenom")
require.Equal(t, uint64(1), supply) // inflated

// Step 4: neither burn nor transfer is possible — module account cannot sign
err = keeper.BurnNFT(ctx, "testdenom", "token1", feeCollectorAddr)
require.Error(t, err) // would require module account to sign — impossible

err = keeper.TransferOwner(ctx, "testdenom", "token1", feeCollectorAddr, attackerAddr)
require.Error(t, err) // same — module account cannot be srcOwner signer
```

func (msg MsgMintNFT) ValidateBasic() error {
	if _, err := sdk.AccAddressFromBech32(msg.Sender); err != nil {

	if _, err := sdk.AccAddressFromBech32(msg.Recipient); err != nil {

		return err
	}
	if err := ValidateTokenURI(msg.URI); err != nil {
		return err
	}
	return ValidateTokenID(msg.Id)
}

		msg.URI,

func (k Keeper) MintNFTUnverified(ctx sdk.Context, denomID, tokenID, tokenNm, tokenURI, tokenData string, owner sdk.AccAddress) error {
	if !k.HasDenomID(ctx, denomID) {
		return sdkerrors.Wrapf(types.ErrInvalidDenom, "denom ID %s not exists", denomID)
	}

	if k.HasNFT(ctx, denomID, tokenID) {
		return sdkerrors.Wrapf(types.ErrNFTAlreadyExists, "NFT %s already exists in collection %s", tokenID, denomID)
	}

	k.setNFT(
		ctx, denomID,
		types.NewBaseNFT(
			tokenID,
			tokenNm,
			owner,
			tokenURI,
			tokenData,
		),
	)
	k.setOwner(ctx, denomID, tokenID, owner)
	k.increaseSupply(ctx, denomID)

	return nil

func (k Keeper) TransferOwner(
	ctx sdk.Context, denomID, tokenID string, srcOwner, dstOwner sdk.AccAddress,
) error {
	if !k.HasDenomID(ctx, denomID) {
		return sdkerrors.Wrapf(types.ErrInvalidDenom, "denom ID %s not exists", denomID)
	}

	nft, err := k.IsOwner(ctx, denomID, tokenID, srcOwner)
	if err != nil {
		return err
	}

	nft.Owner = dstOwner.String()

	k.setNFT(ctx, denomID, nft)
	k.swapOwner(ctx, denomID, tokenID, srcOwner, dstOwner)
	return nil
}

func (k Keeper) BurnNFT(ctx sdk.Context, denomID, tokenID string, owner sdk.AccAddress) error {
	if !k.HasDenomID(ctx, denomID) {
		return sdkerrors.Wrapf(types.ErrInvalidDenom, "denom ID %s not exists", denomID)
	}

	nft, err := k.IsOwner(ctx, denomID, tokenID, owner)
	if err != nil {
		return err
	}

	_, err = k.IsDenomCreator(ctx, denomID, owner)
	if err != nil {
		return err
	}

	k.deleteNFT(ctx, denomID, nft)
	k.deleteOwner(ctx, denomID, tokenID, owner)
	k.decreaseSupply(ctx, denomID)

	return nil
}
