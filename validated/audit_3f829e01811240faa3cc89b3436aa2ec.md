### Title
IBC NFT Refund Permanently Blocked by Wrong Escrow Address in `MintNFT` Creator Check — (File: `x/nft-transfer/keeper/packet.go`)

---

### Summary

In `refundPacketToken`, when the source chain is a **sink chain** (i.e., it previously received the NFT via IBC from another chain), the refund path calls `k.nftKeeper.MintNFT(...)` using the **outbound channel's escrow address** as the `sender`. However, `MintNFT` enforces `IsDenomCreator`, and the denom was originally created with the **inbound channel's escrow address** as creator. These two escrow addresses are always different on multi-hop paths, making the creator check permanently unsatisfiable and the refund permanently blocked.

---

### Finding Description

**`refundPacketToken`** (`x/nft-transfer/keeper/packet.go`, lines 21–52) handles IBC packet timeouts and acknowledgement failures. It computes:

```go
escrowAddress := types.GetEscrowAddress(packet.GetSourcePort(), packet.GetSourceChannel())
```

When `isAwayFromOrigin` is false (the source chain is a sink chain), it calls:

```go
k.nftKeeper.MintNFT(ctx, voucherClassID, tokenID, "", data.TokenUris[i], "", escrowAddress, sender)
``` [1](#0-0) 

`MintNFT` in `x/nft/keeper/keeper.go` enforces:

```go
_, err := k.IsDenomCreator(ctx, denomID, sender)
``` [2](#0-1) 

The denom `voucherClassID` was created in `processReceivedPacket` using the **destination** channel's escrow address:

```go
escrowAddress := types.GetEscrowAddress(packet.GetDestPort(), packet.GetDestChannel())
// ...
k.nftKeeper.IssueDenom(ctx, voucherClassID, voucherClassID, "", data.ClassUri, escrowAddress)
``` [3](#0-2) 

**The mismatch**: on a multi-hop path where chain A received an NFT from chain C (via `channel-0`) and then forwarded it to chain B (via `channel-1`):

| Step | Escrow address used |
|---|---|
| Denom creator (set when A received from C) | `GetEscrowAddress("nft", "channel-0")` |
| Refund sender (set when A's send to B times out) | `GetEscrowAddress("nft", "channel-1")` |

`GetEscrowAddress` is a deterministic SHA-256 hash of the port/channel pair: [4](#0-3) 

These two addresses are always distinct. `IsDenomCreator` will always return `ErrUnauthorized`, and the refund transaction will always fail.

Note the asymmetry: the **outbound burn** in `createOutgoingPacket` correctly uses `BurnNFTUnverified` (no creator check), but the **refund mint** uses `MintNFT` (with creator check): [5](#0-4) 

---

### Impact Explanation

When a multi-hop IBC NFT transfer times out (or the receiving chain sends a failure acknowledgement), the NFT is permanently lost on the source chain. The NFT was burned on chain A during `createOutgoingPacket` (via `BurnNFTUnverified`), but the corresponding re-mint in `refundPacketToken` always fails with `ErrUnauthorized`. The NFT's on-chain supply is permanently decremented with no recovery path for the owner.

The corrupted invariant is: **NFT supply on the sink chain is reduced without the owner receiving their token back**.

---

### Likelihood Explanation

Multi-hop IBC NFT transfers are a supported and documented use case of ICS-721. Any user who receives an IBC NFT on chain A (making A a sink chain) and then forwards it to chain B is exposed. A packet timeout can occur due to network congestion, relayer downtime, or an intentionally short timeout height. This is a realistic production scenario requiring no privileged access — any NFT owner can trigger it by submitting a standard `MsgTransfer` via the `nft-transfer` CLI.

---

### Recommendation

Replace `MintNFT` with `MintNFTUnverified` in the `else` branch of `refundPacketToken`, mirroring the `BurnNFTUnverified` used in `createOutgoingPacket`. The IBC module itself is the trusted authority for refund mints; the denom-creator check is not appropriate here.

```go
// else branch in refundPacketToken — was MintNFT, must be MintNFTUnverified
if err := k.nftKeeper.MintNFTUnverified(ctx, voucherClassID, tokenID, "", data.TokenUris[i], "", sender); err != nil {
    return err
}
``` [6](#0-5) 

---

### Proof of Concept

1. On chain A, user receives an IBC NFT from chain C via `channel-0`. `processReceivedPacket` creates denom `ibc/<hash>` with creator = `GetEscrowAddress("nft", "channel-0")`.
2. User on chain A submits `MsgTransfer` to forward the NFT to chain B via `channel-1`. `createOutgoingPacket` calls `BurnNFTUnverified` — succeeds, NFT is burned on A.
3. The IBC packet times out (relayer offline, timeout height reached, etc.).
4. IBC core calls `OnTimeoutPacket` → `refundPacketToken` on chain A.
5. `isAwayFromOrigin = IsAwayFromOrigin("nft", "channel-1", "nft/channel-0/originalClass")` → `false` (channel-1 is not a prefix of the class path).
6. `escrowAddress = GetEscrowAddress("nft", "channel-1")` ≠ denom creator `GetEscrowAddress("nft", "channel-0")`.
7. `MintNFT` calls `IsDenomCreator(ctx, "ibc/<hash>", GetEscrowAddress("nft","channel-1"))` → returns `ErrUnauthorized`.
8. `refundPacketToken` returns an error. The NFT is permanently lost; the owner's balance is not restored. [7](#0-6) [8](#0-7) [9](#0-8)

func (k Keeper) refundPacketToken(ctx sdk.Context, packet channeltypes.Packet, data types.NonFungibleTokenPacketData) error {
	sender, err := sdk.AccAddressFromBech32(data.Sender)
	if err != nil {
		return err
	}

	classTrace := types.ParseClassTrace(data.ClassId)
	voucherClassID := classTrace.IBCClassID()

	isAwayFromOrigin := types.IsAwayFromOrigin(packet.GetSourcePort(),
		packet.GetSourceChannel(), data.ClassId)

	escrowAddress := types.GetEscrowAddress(packet.GetSourcePort(), packet.GetSourceChannel())

	if isAwayFromOrigin {
		// unescrow tokens back to the sender
		for _, tokenID := range data.TokenIds {
			if err := k.nftKeeper.TransferOwner(ctx, voucherClassID, tokenID, escrowAddress, sender); err != nil {
				return err
			}
		}
	} else {
		// we are sink chain, mint voucher back to sender
		for i, tokenID := range data.TokenIds {
			if err := k.nftKeeper.MintNFT(ctx, voucherClassID, tokenID, "", data.TokenUris[i], "", escrowAddress, sender); err != nil {
				return err
			}
		}
	}

	return nil
}

func GetEscrowAddress(portID, channelID string) sdk.AccAddress {
	// a slash is used to create domain separation between port and channel identifiers to
	// prevent address collisions between escrow addresses created for different channels
	contents := fmt.Sprintf("%s/%s", portID, channelID)

	// ADR 028 AddressHash construction
	preImage := []byte(Version)
	preImage = append(preImage, 0)
	preImage = append(preImage, contents...)
	hash := sha256.Sum256(preImage)
	return hash[:20]
```

func (k Keeper) IsDenomCreator(ctx sdk.Context, denomID string, address sdk.AccAddress) (types.Denom, error) {
	denom, err := k.GetDenom(ctx, denomID)
	if err != nil {
		return types.Denom{}, err
	}

	creator, err := sdk.AccAddressFromBech32(denom.Creator)
	if err != nil {
		panic(err)
	}

	if !creator.Equals(address) {
		return types.Denom{}, sdkerrors.Wrapf(types.ErrUnauthorized, "%s is not the creator of %s", address, denomID)
	}

	return denom, nil
```
