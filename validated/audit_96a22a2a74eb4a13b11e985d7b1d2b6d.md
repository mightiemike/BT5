### Title
Off-by-One in AuxPoW Coinbase Chain-Root Position Guard Allows Invalid Block Acceptance — (File: `contract/src/dogecoin.rs`)

---

### Summary

The `check_aux` function enforces that the chain merkle root appears within the "first 20 bytes" of the parent coinbase script. Because `script_sig` is a hex string (2 chars per byte), 20 bytes = 40 hex characters. The guard uses `<= 40` instead of `< 40`, allowing the chain root to start at hex position 40 (byte 20 — one byte past the protocol boundary). An unprivileged relayer or proof submitter can supply a crafted AuxPoW block that passes this check on the light client while being rejected by the real Dogecoin network.

---

### Finding Description

In `check_aux`, when no merged-mining header is present in the coinbase, the code enforces the position constraint as:

```rust
require!(pos_chain_root <= 40, "Aux POW chain merkle root must start in the first 20 bytes of the parent coinbase");
```

`script_sig` is obtained via `.to_hex_string()`, so every raw byte occupies two characters. "First 20 bytes" therefore corresponds to hex positions 0–39 (inclusive). The correct guard is `pos_chain_root < 40`. The current `<= 40` admits position 40, which maps to raw byte offset 20 — one byte beyond the protocol limit.

The Dogecoin reference implementation (linked in the source comment at line 10) enforces this boundary on raw bytes; the Rust port converts to hex but fails to adjust the constant accordingly. [1](#0-0) 

---

### Impact Explanation

A Dogecoin AuxPoW block whose parent coinbase places the chain merkle root at hex offset 40 (raw byte 20) will pass `check_aux` and be stored in `headers_pool` as a valid mainchain or fork header. The real Dogecoin network rejects such a block. Downstream callers of `verify_transaction_inclusion` or `verify_transaction_inclusion_v2` can then obtain a `true` result for transactions inside a block that is invalid on the canonical chain — a corrupted proof result and a corrupted header-acceptance decision. [2](#0-1) [3](#0-2) 

---

### Likelihood Explanation

Medium-low. The attacker-controlled entry point is the public `submit_blocks` call, which is reachable by any NEAR account that satisfies the trusted-relayer check (or by the DAO/`UnrestrictedSubmitBlocks` bypass roles). The attacker must also supply a parent block whose PoW hash satisfies the declared target — a real mining cost. However, the coinbase script is freely composable by the miner, so placing the chain root at exactly byte offset 20 requires no additional work beyond normal block construction. On testnet or in any deployment where `skip_pow_verification = true`, the PoW barrier disappears entirely and the off-by-one becomes trivially exploitable. [4](#0-3) [5](#0-4) 

---

### Recommendation

Change the comparison operator from `<=` to `<`:

```rust
// Before (buggy):
require!(pos_chain_root <= 40, "...");

// After (correct):
require!(pos_chain_root < 40, "Aux POW chain merkle root must start in the first 20 bytes of the parent coinbase");
```

This aligns the guard with the Dogecoin reference implementation, which requires the chain root to begin strictly within the first 20 raw bytes (hex positions 0–39). [1](#0-0) 

---

### Proof of Concept

1. Construct a Dogecoin AuxPoW parent block whose coinbase `scriptSig` contains no merged-mining header (`fabe6d6d`) and whose chain merkle root hex string begins at character offset 40 (raw byte 20).
2. Call `submit_blocks` on the NEAR light-client contract with this block (AuxPoW variant).
3. `check_aux` evaluates `pos_chain_root <= 40` → `40 <= 40` → `true`; the `require!` passes.
4. The block is stored in `headers_pool` and promoted to the mainchain if its chainwork exceeds the current tip.
5. A subsequent call to `verify_transaction_inclusion_v2` for any transaction in this block returns `true`, even though the block is invalid on the real Dogecoin network.
6. The real Dogecoin network rejects the same block because its reference implementation enforces `pos < 40` (strictly). [6](#0-5) [7](#0-6)

    pub(crate) fn check_aux(&mut self, block_header: &Header, aux_data: &AuxData) {
        // The Dogecoin block must have the AuxPoW flag set (bit 8) when AuxPoW data is present.
        // https://github.com/dogecoin/dogecoin/blob/master/src/auxpow.h
        const BLOCK_VERSION_AUXPOW: i32 = 0x100;

        require!(
            aux_data.chain_merkle_proof.len() <= 30,
            "Aux POW chain merkle branch too long"
        );
        require!(
            block_header.version & BLOCK_VERSION_AUXPOW != 0,
            "Aux POW block does not have AuxPoW flag set in version"
        );

        let chain_id = self.get_config().aux_chain_id;

        require!(
            chain_id == block_header.get_chain_id(),
            format!(
                "block does not have our chain ID (got {}, expected {chain_id})",
                block_header.get_chain_id()
            )
        );

        require!(
            chain_id != aux_data.parent_block.get_chain_id(),
            "Aux POW parent has our chain ID"
        );

        let chain_root = merkle_tools::compute_root_from_merkle_proof(
            block_header.block_hash(),
            aux_data.chain_id,
            &aux_data.chain_merkle_proof,
        );

        let coinbase_tx = aux_data.get_coinbase_tx();
        let coinbase_tx_hash = coinbase_tx.compute_txid();

        require!(
            merkle_tools::compute_root_from_merkle_proof(
                H256::from(coinbase_tx_hash.to_raw_hash().to_byte_array()),
                0,
                &aux_data.merkle_proof,
            ) == aux_data.parent_block.merkle_root
        );

        let script_sig = coinbase_tx
            .input
            .first()
            .unwrap()
            .script_sig
            .to_hex_string();
        let pos_merged_mining_header = script_sig.find(MERGED_MINING_HEADER);
        let mut pos_chain_root = script_sig
            .find(&chain_root.to_string())
            .expect("Aux POW missing chain merkle root in parent coinbase");

        match pos_merged_mining_header {
            Some(pos_merged_mining_header) => {
                if script_sig[pos_merged_mining_header + MERGED_MINING_HEADER.len()..]
                    .contains(MERGED_MINING_HEADER)
                {
                    env::panic_str("Multiple merged mining headers in coinbase");
                }

                require!(
                    pos_merged_mining_header + MERGED_MINING_HEADER.len() == pos_chain_root,
                    "Merged mining header is not just before chain merkle root"
                );
            }
            None => {
                require!(pos_chain_root <= 40, "Aux POW chain merkle root must start in the first 20 bytes of the parent coinbase");
            }
        }

        pos_chain_root += chain_root.to_string().len();
        require!(
            script_sig.len() - pos_chain_root >= 16,
            "Aux POW missing chain merkle tree size and nonce in parent coinbase"
        );

        let bytes = hex::decode(&script_sig[pos_chain_root..pos_chain_root + 8]).unwrap();
        let n_size = u32::from_le_bytes([bytes[0], bytes[1], bytes[2], bytes[3]]);
        require!(
            n_size == (1u32 << aux_data.chain_merkle_proof.len()),
            "Aux POW merkle branch size does not match parent coinbase"
        );

        let bytes = hex::decode(&script_sig[pos_chain_root + 8..pos_chain_root + 16]).unwrap();
        let n_nonce = u32::from_le_bytes([bytes[0], bytes[1], bytes[2], bytes[3]]);

        let chain_id = block_header.get_chain_id();

        let expected_index =
            Self::get_expected_index(n_nonce, chain_id, aux_data.chain_merkle_proof.len());

        require!(
            u32::try_from(aux_data.chain_id).ok() == Some(expected_index),
            "Aux POW wrong index"
        );
        let pow_hash = aux_data.parent_block.block_hash_pow();
        require!(
            self.skip_pow_verification
                || U256::from_le_bytes(&pow_hash.0) <= target_from_bits(block_header.bits),
            format!("block should have correct pow")
        );
    }

    #[payable]
    #[pause]
    #[trusted_relayer]
    pub fn submit_blocks(
        &mut self,
        #[serializer(borsh)] headers: Vec<BlockHeader>,
    ) -> PromiseOrValue<()> {
        let amount = env::attached_deposit();
        let initial_storage = env::storage_usage();
        let num_of_headers = headers.len().try_into().unwrap();

        for header in headers {
            self.submit_block_header(header, self.skip_pow_verification);
        }

        self.run_mainchain_gc(num_of_headers);
        let diff_storage_usage = env::storage_usage().saturating_sub(initial_storage);
        let required_deposit = env::storage_byte_cost().saturating_mul(diff_storage_usage.into());

        require!(
            amount >= required_deposit,
            format!("Required deposit {}", required_deposit)
        );

        let refund = amount.saturating_sub(required_deposit);
        if refund > NearToken::from_near(0) {
            Promise::new(env::predecessor_account_id())
                .transfer(refund)
                .into()
        } else {
            PromiseOrValue::Value(())
        }
    }

    pub fn verify_transaction_inclusion_v2(&self, #[serializer(borsh)] args: ProofArgsV2) -> bool {
        require!(
            args.merkle_proof.len() == args.coinbase_merkle_proof.len(),
            "Coinbase merkle proof and transaction merkle proof should have the same length"
        );

        let header = self
            .headers_pool
            .get(&args.tx_block_blockhash)
            .unwrap_or_else(|| env::panic_str("cannot find requested transaction block"));

        require!(
            merkle_tools::compute_root_from_merkle_proof(
                args.coinbase_tx_id.clone(),
                0usize,
                &args.coinbase_merkle_proof,
            ) == header.block_header.merkle_root,
            "Incorrect coinbase merkle proof"
        );

        #[allow(deprecated)]
        self.verify_transaction_inclusion(args.into())
    }
