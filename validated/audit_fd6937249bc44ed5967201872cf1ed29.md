Audit Report

## Title
`SwapAllowlistExtension` bypass via `MetricOmmSimpleRouter`: `sender` passed to `_beforeSwap` is the router address, not the actual user - (File: metric-periphery/contracts/extensions/SwapAllowlistExtension.sol)

## Summary
`MetricOmmPool.swap` passes `msg.sender` as the `sender` argument to `_beforeSwap`, which forwards it verbatim to `SwapAllowlistExtension.beforeSwap`. When a user routes through `MetricOmmSimpleRouter`, `msg.sender` inside the pool is the router contract, not the originating user. The allowlist therefore gates the router's address rather than the actual swapper, allowing any unprivileged user to bypass a per-pool swap allowlist by routing through the public router.

## Finding Description

**Root cause — `sender` binding in `MetricOmmPool.swap`:**

```solidity
// metric-core/contracts/MetricOmmPool.sol:230-240
_beforeSwap(
    msg.sender,   // <-- always the direct caller of pool.swap()
    recipient,
    ...
);
```

When a user calls `MetricOmmSimpleRouter.exactInputSingle`, the router calls `pool.swap(...)` directly:

```solidity
// metric-periphery/contracts/MetricOmmSimpleRouter.sol:72-80
IMetricOmmPoolActions(params.pool).swap(
    params.recipient,
    params.zeroForOne,
    ...
);
```

So `msg.sender` inside `MetricOmmPool.swap` is the **router address**. `_beforeSwap` receives the router address as `sender` and passes it to `SwapAllowlistExtension.beforeSwap`:

```solidity
// metric-periphery/contracts/extensions/SwapAllowlistExtension.sol:37
if (!allowAllSwappers[msg.sender] && !allowedSwapper[msg.sender][sender]) {
    revert IMetricOmmPoolActions.NotAllowedToSwap();
}
```

Here `msg.sender` is the pool and `sender` is the router. The check passes if the **router** is allowlisted, regardless of who the actual user is. Any user can call the public router and inherit the router's allowlist status.

**Existing guards are insufficient:** The `onlyPool` modifier in `BaseMetricExtension` only verifies the pool is the caller of the extension — it does not recover the original user identity. There is no mechanism in the router or the pool to propagate the originating EOA/contract to the extension hook.

## Impact Explanation
A pool admin deploys a pool with `SwapAllowlistExtension` to restrict swapping to a set of approved addresses (e.g., KYC'd users, whitelisted protocols). To allow those approved users to also use the router, the admin must allowlist the router address. Once the router is allowlisted, **any** address — including unapproved users — can call `MetricOmmSimpleRouter.exactInputSingle/exactInput/exactOutputSingle/exactOutput` and successfully swap on the restricted pool. The allowlist is completely ineffective for router-mediated swaps. This is a broken core pool access-control mechanism causing unauthorized swap execution.

## Likelihood Explanation
The router is a public, permissionless contract. Any unprivileged user can call it. The only precondition is that the router address itself is allowlisted on the target pool, which is a necessary operational step for the router to be usable at all on allowlisted pools. The attack is trivially repeatable with zero special privileges.

## Recommendation
Pass the originating user identity through the swap path. One approach: add an `originator` parameter to `pool.swap` that the router populates with `msg.sender` (the actual user), and forward that as `sender` to `_beforeSwap`. Alternatively, the `SwapAllowlistExtension` should check the payer stored in the router's transient callback context rather than the `sender` argument, but this couples the extension to the router. The cleanest fix is to have the pool accept an explicit `sender` override only from trusted routers, or to have the router pass the real user address in `extensionData` and have the extension decode it — with the pool verifying the router's identity before trusting the override.

## Proof of Concept

```solidity
// Foundry test sketch
function test_swapAllowlistBypassViaRouter() public {
    // Setup: pool with SwapAllowlistExtension, only `allowedUser` is allowlisted
    // Router address is also allowlisted (required for router to work)
    extension.setAllowedToSwap(address(pool), address(router), true);
    // allowedUser is NOT set for attacker

    address attacker = makeAddr("attacker");
    // Fund attacker with token0
    deal(token0, attacker, 1e18);
    vm.startPrank(attacker);
    IERC20(token0).approve(address(router), type(uint256).max);

    // Attacker routes through the public router — bypasses allowlist
    router.exactInputSingle(ExactInputSingleParams({
        pool: address(pool),
        tokenIn: token0,
        recipient: attacker,
        amountIn: 1000,
        amountOutMinimum: 0,
        zeroForOne: true,
        priceLimitX64: 0,
        deadline: block.timestamp,
        extensionData: ""
    }));
    // Swap succeeds despite attacker not being in the allowlist
}
```
