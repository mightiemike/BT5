### Title
Bonus Rewards Permanently Lost When Pool Is Insufficient During Redelegation Slash - (`x/tieredrewards/keeper/slash.go`)

### Summary

In `x/tieredrewards`, when a validator slash fires against a redelegating position (`BeforeRedelegationSlashed`) and the bonus pool is empty or insufficient, the staker's accrued bonus rewards are **permanently and silently forfeited**. The position's accrual checkpoints (`LastBonusAccrual`, `LastEventSeq`, `LastKnownBonded`) are advanced and persisted before the pool balance is checked, so the rewards can never be reclaimed — even after the pool is replenished.

---

### Finding Description

`processEventsAndClaimBonus` in `claim_rewards.go` advances the position's checkpoints and decrements event reference counts **before** checking whether the pool has sufficient balance to pay the computed bonus: [1](#0-0) 

Inside the event loop, `pos.UpdateLastEventSeq(entry.Seq)` is called and `decrementEventRefCount` is called for every processed event — potentially garbage-collecting them when the reference count reaches zero. After the loop, `applyBonusAccrualCheckpoint` and `UpdateLastKnownBonded` advance the time-based checkpoint: [2](#0-1) 

Only **after** all of this mutation does the code check pool balance: [3](#0-2) 

When `ErrInsufficientBonusPool` is returned, the caller `slashRedelegationPosition` deliberately swallows it to avoid a chain halt: [4](#0-3) 

It then calls `setPositionWithState` with the already-mutated `pos`, persisting the advanced checkpoints: [5](#0-4) 

The result: the position's `LastEventSeq` and `LastBonusAccrual` are now past the accrual window, the validator events have had their reference counts decremented (and may be garbage-collected), and the bonus is gone with no recovery path.

---

### Impact Explanation

Stakers who hold tier positions on a validator that gets slashed during a redelegation, when the `rewards_pool` module account is empty or underfunded, permanently lose all bonus rewards that had accrued since their last claim. The corrupted value is the `bonus_rewards` balance owed from `RewardsPoolName` to the position owner. There is no message or governance action that can reclaim these rewards after the checkpoints have been advanced and events garbage-collected. [6](#0-5) 

---

### Likelihood Explanation

The `rewards_pool` module account is funded externally — by governance or by any account sending tokens to it. The `BeginBlocker` also continuously drains it to top up base rewards: [7](#0-6) 

The pool can therefore be empty or near-empty at any block. Validator slashes are normal protocol events (downtime, double-sign). A redelegation slash (`BeforeRedelegationSlashed`) fires whenever a validator is slashed while a position's redelegation entry is still in the unbonding queue (up to 21 days). The combination is realistic and requires no privileged access — any delegator/validator action that triggers a slash while the pool is drained hits this path. [8](#0-7) 

---

### Recommendation

Move the pool balance check **before** mutating any position state. Specifically, compute `totalBonus` first, check `sufficientBonusPoolBalance` before calling `pos.UpdateLastEventSeq`, `decrementEventRefCount`, `applyBonusAccrualCheckpoint`, or `UpdateLastKnownBonded`. Only advance checkpoints and decrement reference counts after confirming the payment will succeed.

Alternatively, in `slashRedelegationPosition`, when `ErrInsufficientBonusPool` is caught, **do not persist** the mutated position state — return early without calling `setPositionWithState`, so the position retains its pre-slash checkpoints and the user can retry the claim once the pool is replenished.

---

### Proof of Concept

1. Alice creates a tier position on validator V1 and redelegates to V2 via `MsgTierRedelegate`. A `RedelegationMapping` entry is created for the unbonding ID.
2. The `rewards_pool` is empty (drained by `BeginBlocker` top-ups or simply never funded).
3. V2 is slashed (downtime). The staking module fires `BeforeRedelegationSlashed` with Alice's unbonding ID.
4. `slashRedelegationPosition` is called → `processEventsAndClaimBonus` runs, advances `pos.LastEventSeq` and `pos.LastBonusAccrual` past the accrual window, decrements event reference counts (garbage-collecting events), then returns `ErrInsufficientBonusPool`.
5. The error is swallowed; `setPositionWithState` persists the advanced checkpoints.
6. The pool is later replenished. Alice calls `MsgClaimTierRewards`.
7. `processEventsAndClaimBonus` finds no events to replay (they were garbage-collected) and `LastBonusAccrual` is already at the slash block time — zero bonus is computed. Alice receives nothing for the entire accrual period. [9](#0-8) [10](#0-9)

func (k Keeper) processEventsAndClaimBonus(ctx context.Context, pos *types.PositionState) (sdk.Coins, error) {
	// Rewards should have been claimed before undelegation
	if !pos.IsDelegated() {
		return sdk.NewCoins(), nil
	}

	valAddr, err := sdk.ValAddressFromBech32(pos.Delegation.ValidatorAddress)
	if err != nil {
		return nil, err
	}

	events, err := k.getValidatorEventsSince(ctx, valAddr, pos.LastEventSeq)
	if err != nil {
		return nil, err
	}

	sdkCtx := sdk.UnwrapSDKContext(ctx)
	blockTime := sdkCtx.BlockTime()

	totalBonus := math.ZeroInt()
	// Use the persisted bonded state from the last replay, not a hardcoded default.
	// This prevents overpaying bonus for unbonded gaps between claims.
	bonded := pos.LastKnownBonded
	segmentStart := pos.LastBonusAccrual

	tier, err := k.getTier(ctx, pos.TierId)
	if err != nil {
		return nil, err
	}

	for _, entry := range events {
		evt := entry.Event

		if bonded {
			// Compute bonus for the bonded segment [segmentStart, eventTime]
			// using the snapshot rate at the event.
			bonus := k.computeSegmentBonus(*pos, tier, segmentStart, evt.Timestamp, evt.TokensPerShare)
			totalBonus = totalBonus.Add(bonus)
		}

		// Update bonded state based on event type.
		switch evt.EventType {
		case types.ValidatorEventType_VALIDATOR_EVENT_TYPE_UNBOND:
			bonded = false
		case types.ValidatorEventType_VALIDATOR_EVENT_TYPE_BOND:
			bonded = true
		case types.ValidatorEventType_VALIDATOR_EVENT_TYPE_SLASH:
			// Slash doesn't change bonded state.
		}

		segmentStart = evt.Timestamp
		pos.UpdateLastEventSeq(entry.Seq)

		// Decrement reference count.
		if err := k.decrementEventRefCount(ctx, valAddr, entry.Seq); err != nil {
			return nil, err
		}
	}

	val, err := k.stakingKeeper.GetValidator(ctx, valAddr)
	if err != nil {
		return nil, err
	}
	// Defensive: validator bond status check
	if bonded && val.IsBonded() {
		currentRate, err := k.getTokensPerShare(ctx, valAddr)
		if err != nil {
			return nil, err
		}
		bonus := k.computeSegmentBonus(*pos, tier, segmentStart, blockTime, currentRate)
		totalBonus = totalBonus.Add(bonus)
	}

	applyBonusAccrualCheckpoint(&pos.Position, blockTime)
	// Persist the bonded state so the next replay starts correctly.
	pos.UpdateLastKnownBonded(bonded)

	if totalBonus.IsZero() {
		return sdk.NewCoins(), nil
	}

	bondDenom, err := k.stakingKeeper.BondDenom(ctx)
	if err != nil {
		return nil, err
	}

	bonusCoins := sdk.NewCoins(sdk.NewCoin(bondDenom, totalBonus))

	if err := k.sufficientBonusPoolBalance(ctx, bonusCoins); err != nil {
		return nil, err
	}

	ownerAddr, err := sdk.AccAddressFromBech32(pos.Owner)
	if err != nil {
		return nil, errorsmod.Wrap(sdkerrors.ErrInvalidAddress, "invalid owner address")
	}

	if err := k.bankKeeper.SendCoinsFromModuleToAccount(ctx, types.RewardsPoolName, ownerAddr, bonusCoins); err != nil {
		return nil, err
	}

	if err := sdkCtx.EventManager().EmitTypedEvent(&types.EventBonusRewardsClaimed{
		PositionId: pos.Id,
		Owner:      pos.Owner,
		Rewards:    bonusCoins,
	}); err != nil {
		return nil, err
	}

	return bonusCoins, nil

func (k Keeper) slashRedelegationPosition(ctx context.Context, unbondingId uint64, sharesToUnbond math.LegacyDec) error {
	positionId, err := k.getRedelegationMapping(ctx, unbondingId)
	if errors.Is(err, collections.ErrNotFound) {
		return nil
	}
	if err != nil {
		return err
	}

	pos, err := k.getPositionState(ctx, positionId)
	if errors.Is(err, types.ErrPositionNotFound) {
		k.logger(ctx).Error("position not found during redelegation slash",
			"position_id", positionId,
			"unbonding_id", unbondingId,
			"error", err.Error(),
		)
		return nil
	}
	if err != nil {
		return err
	}

	if !pos.IsDelegated() {
		// Defensive
		k.logger(ctx).Error("delegation missing during BeforeRedelegationSlashed",
			"position_id", positionId,
			"unbonding_id", unbondingId,
			"shares_to_unbond", sharesToUnbond.String(),
		)
		return nil
	}

	dstValStr := pos.Delegation.ValidatorAddress

	// Settle bonus against PRE-slash shares.
	if _, err := k.processEventsAndClaimBonus(ctx, &pos); err != nil {
		// Deliberately forgo bonus rewards if pool is insufficient to prevent chain halt.
		if errors.Is(err, types.ErrInsufficientBonusPool) {
			k.logger(ctx).Error("insufficient bonus pool during redelegation slash",
				"position_id", pos.Id,
				"error", err.Error(),
			)
		} else {
			return err
		}
	}

	fullSlash := sharesToUnbond.GTE(pos.Delegation.Shares)

	if fullSlash {
		pos.Delegation = nil
		pos.ClearBonusCheckpoints()
		return k.setPositionWithState(ctx, pos, &ValidatorTransition{PreviousAddress: dstValStr})
	}
	// In-memory only: the persisted Position carries no share count, and the
	// live delegation will reflect the post-Unbond shares on the next read.
	// Update the local copy so any follow-up logic in this call sees consistent state.
	pos.Delegation.Shares = pos.Delegation.Shares.Sub(sharesToUnbond)
	return k.setPositionWithState(ctx, pos, nil)
