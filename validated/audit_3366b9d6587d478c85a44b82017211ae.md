### Title
Authz `GenericAuthorization` for `MsgTransferNFT` Allows Grantee to Steal NFTs by Setting Themselves as Recipient — (`x/nft/keeper/msg_server.go`, `x/nft/keeper/keeper.go`)

---

### Summary

The NFT module registers `MsgTransferNFT` as a standard `sdk.Msg` but provides no custom `Authorization` type restricting the `Recipient` field. A grantee holding a `GenericAuthorization` for `MsgTransferNFT` from owner A can craft a `MsgExec` with `Sender=A` and `Recipient=<grantee>`, pass the `IsOwner` check (because `srcOwner=A` is the real owner), and seize the NFT.

---

### Finding Description

**Step 1 — `GetSigners` returns `msg.Sender` (the granter)**

`MsgTransferNFT.GetSigners()` returns the `Sender` field, which is the granter's address A. [1](#0-0) 

The Cosmos SDK authz module uses `GetSigners()` to identify the granter and verify the grant exists. It does **not** constrain any other field of the inner message.

**Step 2 — `TransferNFT` msg server passes `msg.Sender` as `srcOwner` and `msg.Recipient` as `dstOwner` verbatim** [2](#0-1) 

`msg.Recipient` is fully attacker-controlled; there is no check that it differs from the grantee's address.

**Step 3 — `TransferOwner` only checks that `srcOwner` is the current owner** [3](#0-2) 

`IsOwner` verifies `srcOwner == nft.Owner`. Since `srcOwner = msg.Sender = A` (the real owner), the check passes unconditionally regardless of who `dstOwner` is. [4](#0-3) 

**Step 4 — No custom `Authorization` type exists for NFT transfers**

`codec.go` registers all NFT messages only as `sdk.Msg` implementations, with no custom `Authorization` that could restrict the recipient address or the set of transferable token IDs. [5](#0-4) 

`ValidateBasic` on `MsgTransferNFT` only validates that `Sender` and `Recipient` are valid bech32 addresses — no restriction on who the recipient may be. [6](#0-5) 

---

### Impact Explanation

An attacker B who obtains a `GenericAuthorization` grant for `MsgTransferNFT` from owner A (e.g., by posing as a marketplace or escrow service) can:

1. Submit `MsgExec{msgs: [MsgTransferNFT{DenomId: X, Id: Y, Sender: A, Recipient: B}]}` signed by B.
2. Authz verifies A granted B the right to execute `MsgTransferNFT` — passes.
3. `IsOwner(A)` passes because A is the real owner.
4. NFT ownership is permanently transferred to B.

This directly violates the NFT ownership invariant: the grantee can redirect the transfer destination to themselves, stealing any NFT covered by the grant.

---

### Likelihood Explanation

`GenericAuthorization` for `MsgTransferNFT` is a realistic grant in marketplace, escrow, or delegation scenarios. A user who intends to allow a third party to "transfer my NFT to a buyer" does not expect that third party to be able to designate themselves as the buyer. The exploit requires no privileged access beyond the grant itself, which is a normal on-chain operation.

---

### Recommendation

Implement a custom `NFTTransferAuthorization` type (analogous to Cosmos SDK's `SendAuthorization` for bank transfers) that:
- Restricts the allowed `Recipient` address(es), or
- Restricts the specific `DenomId`/`Id` pairs that may be transferred, or
- Both.

Register it via `RegisterImplementations` in `codec.go` so it can be used in place of `GenericAuthorization`.

---

### Proof of Concept

```
// 1. Owner A mints NFT (denomID="testdenom", tokenID="token1")
// 2. A submits MsgGrant granting B GenericAuthorization for
//    "/chainmain.nft.v1.MsgTransferNFT"
// 3. B submits:
MsgExec{
    Grantee: B,
    Msgs: [MsgTransferNFT{
        DenomId:   "testdenom",
        Id:        "token1",
        Sender:    A,       // granter — passes authz check
        Recipient: B,       // attacker — no restriction
    }],
}
// 4. Assert: NFT owner is now B, not A.
```

The authz module dispatches the inner message with `Sender=A` satisfying `IsOwner`, while `Recipient=B` is never validated against any authorization constraint.

func (msg MsgTransferNFT) ValidateBasic() error {
	if err := ValidateDenomID(msg.DenomId); err != nil {
		return err
	}

	if _, err := sdk.AccAddressFromBech32(msg.Sender); err != nil {
		return newsdkerrors.Wrapf(sdkerrors.ErrInvalidAddress, "invalid sender address (%s)", err)
	}

	if _, err := sdk.AccAddressFromBech32(msg.Recipient); err != nil {
		return newsdkerrors.Wrapf(sdkerrors.ErrInvalidAddress, "invalid recipient address (%s)", err)
	}
	return ValidateTokenID(msg.Id)
}

func (k Keeper) TransferOwner(
	ctx sdk.Context, denomID, tokenID string, srcOwner, dstOwner sdk.AccAddress,
) error {
	if !k.HasDenomID(ctx, denomID) {
		return sdkerrors.Wrapf(types.ErrInvalidDenom, "denom ID %s not exists", denomID)
	}

	nft, err := k.IsOwner(ctx, denomID, tokenID, srcOwner)
	if err != nil {
		return err
	}

	nft.Owner = dstOwner.String()

	k.setNFT(ctx, denomID, nft)
	k.swapOwner(ctx, denomID, tokenID, srcOwner, dstOwner)
	return nil
}

func (k Keeper) IsOwner(ctx sdk.Context, denomID, tokenID string, owner sdk.AccAddress) (types.BaseNFT, error) {
	nft, err := k.GetNFT(ctx, denomID, tokenID)
	if err != nil {
		return types.BaseNFT{}, err
	}

	if !owner.Equals(nft.GetOwner()) {
		return types.BaseNFT{}, sdkerrors.Wrapf(types.ErrUnauthorized, "%s is not the owner of %s/%s", owner.String(), denomID, tokenID)
	}

	return nft.(types.BaseNFT), nil
}
