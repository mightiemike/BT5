### Title
Adversary Can Permanently Block IBC NFT Transfers by Pre-Creating Denom with Attacker-Controlled Creator ‚Äî (`File: x/nft-transfer/keeper/packet.go`)

---

### Summary

`processReceivedPacket` in the `nft-transfer` keeper guards denom creation with a `HasDenomID` check but does **not** guard the subsequent `MintNFT` call. An unprivileged actor can pre-create the IBC voucher denom (with their own address as creator) and pre-mint an NFT with the target `tokenID`, causing any in-flight or future IBC NFT packet for that `(classID, tokenID)` pair to permanently fail with an error acknowledgement.

---

### Finding Description

In `processReceivedPacket`, when an NFT packet arrives moving away from origin, the code does:

```go
if !k.nftKeeper.HasDenomID(ctx, voucherClassID) {
    if err := k.nftKeeper.IssueDenom(ctx, voucherClassID, voucherClassID, "", data.ClassUri, escrowAddress); err != nil {
        return err
    }
}
// ...
for i, tokenID := range data.TokenIds {
    if err := k.nftKeeper.MintNFT(ctx, voucherClassID, tokenID, "", data.TokenUris[i], "", escrowAddress, receiver); err != nil {
        return err
    }
}
``` [1](#0-0) 

The `voucherClassID` is fully deterministic: `ibc/{sha256(destPort + "/" + destChannel + "/" + classID)}`. [2](#0-1) 

An attacker can:

1. Compute `voucherClassID` for any NFT class they wish to block.
2. Submit `MsgIssueDenom` with `id = voucherClassID`, registering themselves as the denom creator.
3. Submit `MsgMintNFT` with `denomId = voucherClassID` and `id = <target tokenID>` (observable from the pending IBC packet on the source chain), minting a collision token.

When the legitimate IBC packet is relayed and `OnRecvPacket` calls `processReceivedPacket`:

- `HasDenomID` returns `true` ‚Üí denom creation is skipped (the adversary's denom, with wrong creator, is silently accepted).
- `MintNFT` is called with `escrowAddress` as the minter and the colliding `tokenID` ‚Üí fails because the token already exists. [3](#0-2) 

The error propagates back through `OnRecvPacket` as an error acknowledgement: [4](#0-3) 

`OnAcknowledgementPacket` then calls `refundPacketToken`, returning the NFT to the sender on the source chain. [5](#0-4) 

The collision token remains in the denom on the destination chain. Every subsequent relay attempt for that `(voucherClassID, tokenID)` pair will hit the same `MintNFT` failure. The transfer is permanently blocked.

---

### Impact Explanation

- The targeted NFT can never be transferred to the destination chain via IBC for the colliding `tokenID`.
- If the adversary pre-creates the denom with their own address as creator (before any legitimate IBC transfer for that class), `MintNFT` will also fail due to creator mismatch, blocking **all** token IDs for that entire IBC class ‚Äî not just one.
- The corrupted value is the NFT ownership state on the destination chain: the IBC voucher denom is permanently owned by the adversary rather than the escrow address, and the escrow address can never mint into it.

---

### Likelihood Explanation

- `voucherClassID` is fully deterministic and publicly computable from on-chain data (port, channel, classID).
- The attacker only needs to submit two standard Cosmos SDK transactions (`MsgIssueDenom` + `MsgMintNFT`) before the IBC packet is relayed ‚Äî a window that spans the entire IBC packet timeout period (default 10 minutes per `DefaultRelativePacketTimeoutTimestamp`). [6](#0-5) 

- No special privileges are required.

---

### Recommendation

Mirror the same idempotency guard used for denom creation around `MintNFT`: before calling `MintNFT`, check `k.nftKeeper.HasNFT(ctx, voucherClassID, tokenID)` and skip (or verify ownership) if the token already exists. Additionally, after the `HasDenomID` guard, verify that the existing denom's creator matches `escrowAddress`; if it does not, return an error acknowledgement with a clear message rather than silently proceeding to a guaranteed `MintNFT` failure.

---

### Proof of Concept

1. Alice initiates `MsgTransfer` on chain A to send NFT `(classA, token1)` to chain B. The packet is in-flight.
2. Attacker computes `voucherClassID = ibc/sha256("nft/channel-0/classA")` on chain B.
3. Attacker submits `MsgIssueDenom{id: voucherClassID, name: voucherClassID, sender: attacker}` on chain B ‚Äî succeeds because no denom with that ID exists yet.
4. Attacker submits `MsgMintNFT{denomId: voucherClassID, id: "token1", sender: attacker, recipient: attacker}` on chain B ‚Äî succeeds because attacker is the denom creator.
5. Relayer submits the IBC packet on chain B. `processReceivedPacket` runs: `HasDenomID` returns `true` (skips denom creation), `MintNFT(voucherClassID, "token1", ..., escrowAddress, alice)` fails ‚Äî `token1` already exists.
6. Error ack is written. `OnAcknowledgementPacket` calls `refundPacketToken` on chain A, returning `token1` to Alice.
7. Alice retries ‚Äî step 5 repeats indefinitely. The transfer is permanently blocked. [7](#0-6) [8](#0-7) [9](#0-8)

// Hash returns the hex bytes of the SHA256 hash of the ClassTrace fields using the following formula:
//
// hash = sha256(tracePath + "/" + baseClassId)
func (ct ClassTrace) Hash() tmbytes.HexBytes {
	hash := sha256.Sum256([]byte(ct.GetFullClassPath()))
	return hash[:]
}

// IBCClassID a classID for an ICS721 non-fungible token in the format
// 'ibc/{hash(tracePath + BaseClassId)}'. If the trace is empty, it will return the base classID.
func (ct ClassTrace) IBCClassID() string {
	if ct.Path != "" {
		return fmt.Sprintf("%s/%s", ClassPrefix, ct.Hash())
	}
	return ct.BaseClassId
}

func (k Keeper) OnAcknowledgementPacket(ctx sdk.Context, channelVersion string, packet channeltypes.Packet, data types.NonFungibleTokenPacketData, ack channeltypes.Acknowledgement) error {
	switch ack.Response.(type) {
	case *channeltypes.Acknowledgement_Error:
		return k.refundPacketToken(ctx, packet, data)
	default:
		// the acknowledgement succeeded on the receiving chain so nothing
		// needs to be executed and no error needs to be returned
		return nil
	}
}

func (k Keeper) SetDenom(ctx sdk.Context, denom types.Denom) error {
	if k.HasDenomID(ctx, denom.Id) {
		return sdkerrors.Wrapf(types.ErrInvalidDenom, "denomID %s has already exists", denom.Id)
	}

	if k.HasDenomNm(ctx, denom.Name) {
		return sdkerrors.Wrapf(types.ErrInvalidDenom, "denomName %s has already exists", denom.Name)
	}

	store := ctx.KVStore(k.storeKey)
	bz := k.cdc.MustMarshal(&denom)
	store.Set(types.KeyDenomID(denom.Id), bz)
	store.Set(types.KeyDenomName(denom.Name), []byte(denom.Id))
	return nil
```
