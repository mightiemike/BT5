### Title
`WithdrawCollateral` via Slow Mode Path Bypasses Per-Product Withdrawal Fee — (`core/contracts/EndpointTx.sol`)

---

### Summary

The slow mode execution path for `WithdrawCollateral` in `processSlowModeTransactionImpl` omits the per-product withdrawal fee that is enforced in the sequencer path (`processTransactionImpl`). Any user can exploit this discrepancy to withdraw collateral without paying the withdrawal fee by routing their withdrawal through the slow mode path instead of the sequencer path.

---

### Finding Description

**Sequencer path** (`processTransactionImpl`, lines 413–436): `WithdrawCollateral` charges a per-product withdrawal fee before executing the withdrawal:

```solidity
chargeFee(
    signedTx.tx.sender,
    spotEngine.getConfig(signedTx.tx.productId).withdrawFeeX18,
    signedTx.tx.productId
);
clearinghouse.withdrawCollateral(
    signedTx.tx.sender,
    signedTx.tx.productId,
    signedTx.tx.amount,
    address(0),
    nSubmissions
);
``` [1](#0-0) 

**Slow mode path** (`processSlowModeTransactionImpl`, lines 217–229): `WithdrawCollateral` is processed with only a `validateSender` check — no withdrawal fee is charged:

```solidity
validateSender(txn.sender, sender);
clearinghouse.withdrawCollateral(
    txn.sender,
    txn.productId,
    txn.amount,
    address(0),
    nSubmissions
);
``` [2](#0-1) 

The only fee charged in the slow mode path is a flat `SLOW_MODE_FEE` collected at submission time in `submitSlowModeTransactionImpl` (line 370–371), which is structurally unrelated to the per-product `withdrawFeeX18`: [3](#0-2) 

The two paths are both reachable by any user. The sequencer path is triggered via `submitTransactionsChecked` → `processTransactionImpl`. The slow mode path is triggered via `submitSlowModeTransaction` → `submitSlowModeTransactionImpl` (queues the tx) → `executeSlowModeTransaction` / `_executeSlowModeTransaction` → `processSlowModeTransactionImpl`. [4](#0-3) [5](#0-4) 

The `WithdrawCollateral` struct carries a `nonce` field that is validated in the sequencer path via `validateSignedTx` but is entirely ignored in the slow mode path, confirming the two paths apply materially different restrictions to the same operation. [6](#0-5) 

---

### Impact Explanation

A user withdrawing a large amount of collateral can avoid the percentage-based `withdrawFeeX18` entirely by routing through the slow mode path. The flat `SLOW_MODE_FEE` paid at submission is independent of withdrawal size and will be negligible for large withdrawals. The protocol's fee revenue from withdrawals is directly reduced. Because `WithdrawCollateralV2` (the newer variant) is not handled in the slow mode path and would revert, the only available bypass is through the legacy `WithdrawCollateral` type, which remains fully functional in slow mode.

---

### Likelihood Explanation

High. The slow mode path is publicly accessible to any user via `submitSlowModeTransaction`. No privileged access is required. The user pays only the flat slow mode fee. For any withdrawal where `withdrawFeeX18 * amount > SLOW_MODE_FEE`, the slow mode path is strictly cheaper, giving every large withdrawer a direct economic incentive to exploit this path.

---

### Recommendation

Add the per-product withdrawal fee charge to the `WithdrawCollateral` branch inside `processSlowModeTransactionImpl`, mirroring the sequencer path:

```solidity
chargeFee(
    txn.sender,
    spotEngine.getConfig(txn.productId).withdrawFeeX18,
    txn.productId
);
```

This should be inserted before the `clearinghouse.withdrawCollateral(...)` call at line 223.

---

### Proof of Concept

1. User holds 1,000,000 USDC in their subaccount. The configured `withdrawFeeX18` for the USDC product is 0.1% (1,000 USDC). The flat `SLOW_MODE_FEE` is, say, 1 USDC.
2. User calls `submitSlowModeTransaction` with a `WithdrawCollateral` payload encoding `{sender: subaccount, productId: USDC_ID, amount: 1_000_000e18, nonce: <any>}`.
3. The slow mode fee of 1 USDC is deducted at submission. The transaction is queued.
4. After `SLOW_MODE_TX_DELAY`, anyone calls `executeSlowModeTransaction`. `processSlowModeTransactionImpl` runs: `validateSender` passes (msg.sender matches the address in the subaccount), and `clearinghouse.withdrawCollateral` executes with no fee deduction.
5. User receives 1,000,000 USDC, having paid 1 USDC instead of 1,000 USDC — a 999 USDC fee bypass per withdrawal.
