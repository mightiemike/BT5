### Title
Sequencer-Injected `oraclePriceX18` in `SignedMintNlp`/`SignedBurnNlp` Is Not Covered by User EIP-712 Signature — (File: `core/contracts/EndpointTx.sol`)

---

### Summary

The `MintNlp` and `BurnNlp` transaction types carry sequencer-supplied fields (`oraclePriceX18`, `nlpPoolRebalanceX18`) that sit **outside** the inner struct the user actually signs. The EIP-712 digest covers only the inner `MintNlp`/`BurnNlp` fields (`sender`, `quoteAmount`/`nlpAmount`, `nonce`). A malicious or compromised sequencer can therefore inject an arbitrary oracle price after the user has signed, silently altering the NLP exchange rate and causing the user to receive fewer NLP tokens (on mint) or less quote (on burn) than they agreed to.

---

### Finding Description

`IEndpoint.sol` defines two wrapper structs whose extra fields are sequencer-provided at submission time, not at signing time:

```solidity
struct SignedMintNlp {
    MintNlp tx;           // ← user signs only this inner struct
    bytes   signature;
    int128  oraclePriceX18;        // ← sequencer-injected, NOT signed
    int128[] nlpPoolRebalanceX18;  // ← sequencer-injected, NOT signed
}

struct SignedBurnNlp {
    BurnNlp tx;           // ← user signs only this inner struct
    bytes   signature;
    int128  oraclePriceX18;        // ← sequencer-injected, NOT signed
    int128[] nlpPoolRebalanceX18;  // ← sequencer-injected, NOT signed
}
``` [1](#0-0) 

The inner structs contain only `sender`, `quoteAmount`/`nlpAmount`, and `nonce` — no price field:

```solidity
struct MintNlp { bytes32 sender; uint128 quoteAmount; uint64 nonce; }
struct BurnNlp  { bytes32 sender; uint128 nlpAmount;  uint64 nonce; }
``` [1](#0-0) 

In `processTransactionImpl`, after signature validation, the sequencer-supplied price is written directly to protocol state and forwarded to the clearinghouse **without any bounds check**:

```solidity
priceX18[NLP_PRODUCT_ID] = signedTx.oraclePriceX18;
clearinghouse.mintNlp(
    signedTx.tx,
    signedTx.oraclePriceX18,   // ← unchecked sequencer value
    nlpPools,
    signedTx.nlpPoolRebalanceX18
);
``` [2](#0-1) 

The same pattern applies to `BurnNlp`: [3](#0-2) 

The EIP-712 digest computation follows the same pattern as `getDigest` in `OffchainExchange.sol`, which hashes only the inner `Order` struct fields and not any outer wrapper:

```solidity
string memory structType =
    "Order(bytes32 sender,int128 priceX18,int128 amount,uint64 expiration,uint64 nonce,uint128 appendix)";
bytes32 structHash = keccak256(abi.encode(keccak256(bytes(structType)), order.sender, ...));
``` [4](#0-3) 

Consistently, `validateSignedTx` passes the full transaction bytes to `computeDigest`, which delegates to `verifier.computeDigest`. The Verifier decodes and hashes only the inner typed struct (e.g., `MintNlp`), leaving `oraclePriceX18` and `nlpPoolRebalanceX18` outside the signed commitment. [5](#0-4) 

The contrast with `WithdrawCollateralV2` is instructive: that transaction type also has a sequencer-injected field (`feeX18`) outside the inner struct, but the protocol **does** bound it: `require(signedTx.feeX18 <= currentFeeX18)`. No equivalent bound exists for `oraclePriceX18`. [6](#0-5) 

---

### Impact Explanation

`oraclePriceX18` is the NLP token price used to compute how many NLP tokens a user receives for their `quoteAmount` (mint) or how much quote they receive for their `nlpAmount` (burn). An inflated price on mint means fewer NLP tokens issued; a deflated price on burn means less quote returned. The user's signed intent (`quoteAmount`/`nlpAmount`) is honored, but the exchange rate — the financially material parameter — is silently substituted. The corrupted state delta is: **user's NLP balance (mint) or quote balance (burn) is reduced below the fair-value amount**, with the difference effectively stolen or misallocated via `nlpPoolRebalanceX18`.

---

### Likelihood Explanation

The sequencer is a trusted but external off-chain component. The protocol explicitly acknowledges sequencer failure/censorship risk (the slow-mode mechanism exists precisely for this). A compromised sequencer operator, or an attacker who gains sequencer key access, can exploit this on every `MintNlp`/`BurnNlp` submission with no on-chain detection. The entry path is the standard `submitTransactions` → `processTransactionImpl` fast path, reachable by the sequencer for any user who submits a mint or burn request.

---

### Recommendation

- **Short term:** Include `oraclePriceX18` in the EIP-712 typed-data struct for `MintNlp` and `BurnNlp` so users explicitly commit to the price at signing time. Add an on-chain staleness/bounds check (e.g., compare against a trusted on-chain oracle or a protocol-configured price band) analogous to the `feeX18 <= currentFeeX18` guard in `WithdrawCollateralV2`.
- **Long term:** Derive `oraclePriceX18` entirely from a trusted on-chain oracle rather than accepting it as sequencer input, eliminating the trust dependency.

---

### Proof of Concept

1. Alice signs `MintNlp { sender: alice_subaccount, quoteAmount: 10_000e18, nonce: 7 }` — she expects to receive NLP tokens at the current fair price of, say, `1e18` (1:1).
2. The sequencer (compromised) wraps the signed inner struct into `SignedMintNlp { tx: ..., signature: alice_sig, oraclePriceX18: 2e18, nlpPoolRebalanceX18: [...] }` — doubling the stated price.
3. Sequencer calls `Endpoint.submitTransactions(...)`.
4. `processTransactionImpl` calls `validateSignedTx(signedTx.tx.sender, signedTx.tx.nonce, transaction, signedTx.signature, true)`.
5. The EIP-712 digest is computed over `MintNlp { sender, quoteAmount, nonce }` only — **signature validates successfully** despite the injected price.
6. `priceX18[NLP_PRODUCT_ID] = 2e18` is written to state.
7. `clearinghouse.mintNlp(signedTx.tx, 2e18, ...)` issues Alice ~50% of the NLP tokens she should have received for her `quoteAmount`.
8. Alice's quote balance is debited by `10_000e18`; her NLP balance is credited at the manipulated rate — a direct, quantifiable asset loss with no recourse.

                keccak256(bytes(structType)),

                order.amount,
                order.expiration,
                order.nonce,
                order.appendix
            )

        bytes32 domainSeparator = keccak256(

                _EIP712NameHash(),
                _EIP712VersionHash(),
                block.chainid,

            )
        );

        return ECDSAUpgradeable.toTypedDataHash(domainSeparator, structHash);
