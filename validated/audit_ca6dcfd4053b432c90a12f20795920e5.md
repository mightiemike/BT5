### Title
`SwapAllowlistExtension` Gates the Router Address Instead of the End User, Allowing Any Caller to Bypass the Swap Allowlist via `MetricOmmSimpleRouter` — (`metric-periphery/contracts/extensions/SwapAllowlistExtension.sol`)

---

### Summary

`SwapAllowlistExtension.beforeSwap` receives `sender` from the pool, which equals `msg.sender` of the `pool.swap()` call. When a user routes through `MetricOmmSimpleRouter`, `msg.sender` to the pool is the router contract, not the end user. The allowlist check therefore evaluates `allowedSwapper[pool][router]` instead of `allowedSwapper[pool][user]`. If the router is allowlisted (the only way to permit router-mediated swaps for legitimate users), the gate is fully open to every caller regardless of their allowlist status.

---

### Finding Description

`SwapAllowlistExtension.beforeSwap` enforces:

```solidity
if (!allowAllSwappers[msg.sender] && !allowedSwapper[msg.sender][sender]) {
    revert IMetricOmmPoolActions.NotAllowedToSwap();
}
``` [1](#0-0) 

Here `msg.sender` is the pool (correct), and `sender` is the value the pool forwarded from its own `msg.sender` — the direct caller of `pool.swap()`. When a user calls `MetricOmmSimpleRouter.exactInput*`, the router calls `pool.swap(...)`, so the pool's `msg.sender` is the router. The pool then passes `sender = router` into `_beforeSwap`:

```solidity
function _beforeSwap(
    address sender,   // ← pool's msg.sender, i.e. the router
    address recipient,
    ...
) internal {
    _callExtensionsInOrder(
        BEFORE_SWAP_ORDER,
        abi.encodeCall(IMetricOmmExtensions.beforeSwap, (sender, recipient, ...))
    );
}
``` [2](#0-1) 

The extension therefore checks `allowedSwapper[pool][router]`, not `allowedSwapper[pool][user]`.

**Two broken outcomes:**

| Router allowlist status | Effect |
|---|---|
| Router **is** allowlisted (to let legitimate users swap via router) | Every caller — including non-allowlisted users — passes the check. Allowlist is fully bypassed. |
| Router **is not** allowlisted | Allowlisted users cannot use the router at all. Core swap path is broken for them. |

The bypass path is entirely unprivileged: any EOA calls `MetricOmmSimpleRouter.exactInputSingle()` targeting a pool whose `SwapAllowlistExtension` has the router allowlisted.

---

### Impact Explanation

A pool deployer uses `SwapAllowlistExtension` to restrict swaps to KYC'd or otherwise vetted addresses. To allow those users to benefit from the router (slippage protection, multi-hop), the admin allowlists the router. At that point the allowlist provides zero protection: any address can call the router and execute swaps on the restricted pool. LP funds are exposed to unrestricted trading, defeating the entire purpose of the permissioned pool design. This is a direct broken-invariant on the admin-boundary gate and on core swap functionality.

---

### Likelihood Explanation

The `MetricOmmSimpleRouter` is the primary user-facing entry point documented in the protocol. Any pool that deploys `SwapAllowlistExtension` and also wants router support must allowlist the router, triggering the bypass automatically. The attacker needs no special role, no privileged setup, and no non-standard token — only a public router call.

---

### Recommendation

The pool should pass the **original end-user** as `sender` to extensions, not its own `msg.sender`. Two standard approaches:

1. **Explicit sender parameter**: Add `address sender` to `pool.swap(address sender, address recipient, ...)` so the router can forward `msg.sender` (the actual user). The pool uses this explicit value when calling `_beforeSwap`.
2. **Callback-derived identity**: Resolve the true sender inside the swap callback (similar to Uniswap v4's `hookData` pattern), where the router attests the real caller.

`SwapAllowlistExtension.beforeSwap` should then check the attested end-user address, not the intermediary.

---

### Proof of Concept

```
Setup:
  pool P with SwapAllowlistExtension E
  allowedSwapper[P][router] = true   // admin must do this to allow router swaps
  allowedSwapper[P][alice]  = true   // alice is KYC'd
  allowedSwapper[P][bob]    = false  // bob is NOT allowlisted

Attack:
  bob calls MetricOmmSimpleRouter.exactInputSingle({pool: P, ...})
  → router calls P.swap(recipient=bob, ...)
  → pool passes sender = address(router) to _beforeSwap
  → extension checks allowedSwapper[P][router] == true  ✓
  → swap executes — bob bypasses the allowlist entirely
``` [3](#0-2) [2](#0-1)
