### Title
`verify_transaction_inclusion_v2` Always Reverts When Deprecated `verify_transaction_inclusion` Is Independently Paused - (`contract/src/lib.rs`)

### Summary
`verify_transaction_inclusion_v2` delegates its final verification step to `verify_transaction_inclusion` via a direct internal Rust call. Both functions carry independent `#[pause]` guards from `near_plugins`. Because `verify_transaction_inclusion` is explicitly marked `#[deprecated]`, a `PauseManager` or `DAO` will naturally pause it to enforce migration to v2. Doing so silently breaks `verify_transaction_inclusion_v2` for every caller, permanently reverting all SPV proof requests until the pause is lifted.

### Finding Description

`verify_transaction_inclusion_v2` is the current, recommended SPV verification entry point. Its implementation validates the coinbase merkle proof and then delegates the full proof check to `verify_transaction_inclusion`: [1](#0-0) 

```rust
#[pause]
pub fn verify_transaction_inclusion_v2(&self, ...) -> bool {
    ...
    #[allow(deprecated)]
    self.verify_transaction_inclusion(args.into())   // line 368
}
```

`verify_transaction_inclusion` carries its own independent `#[pause]` guard: [2](#0-1) 

```rust
#[deprecated(since = "0.5.0", note = "Use `verify_transaction_inclusion_v2` instead.")]
#[pause]
pub fn verify_transaction_inclusion(&self, ...) -> bool { ... }
```

The `near_plugins` `Pausable` implementation assigns each `#[pause]`-decorated function its own pause key derived from the function name. This means `verify_transaction_inclusion` can be paused independently of `verify_transaction_inclusion_v2`.

When a `PauseManager` or `DAO` pauses `verify_transaction_inclusion` (the deprecated v1 endpoint), the pause guard inside that function fires on the internal Rust call at line 368, causing `verify_transaction_inclusion_v2` to panic and revert — even though `verify_transaction_inclusion_v2` itself is not paused.

### Impact Explanation

All callers of `verify_transaction_inclusion_v2` — the only non-deprecated SPV verification API — receive a permanent revert. No NEAR contract or user can verify Bitcoin transaction inclusion. This is the primary production use case of the light client. The deadlock persists until the `PauseManager` or `DAO` explicitly unpauses `verify_transaction_inclusion`, which may not be obvious since the intent was to retire v1, not to break v2.

**Impact: 5** — Complete loss of the contract's core SPV verification functionality.

### Likelihood Explanation

The function is explicitly annotated `#[deprecated(since = "0.5.0", note = "Use verify_transaction_inclusion_v2 instead.")]`. Pausing a deprecated endpoint to enforce migration is a standard, expected operational action for any `PauseManager` or `DAO` key holder. The trigger requires no adversarial actor — it is a routine administrative step that any authorized role would take. The hidden internal dependency is non-obvious from the public API surface.

**Likelihood: 4** — High; the deprecation annotation actively invites the triggering action.

### Recommendation

Remove the `#[pause]` attribute from `verify_transaction_inclusion` and replace the internal delegation in `verify_transaction_inclusion_v2` with a direct call to the shared verification logic (extracted into a private helper). This eliminates the hidden pause dependency while preserving independent pausability of the public endpoints.

Alternatively, if `verify_transaction_inclusion` must remain independently pausable, `verify_transaction_inclusion_v2` must not delegate to it. Instead, both should call a common private function that carries no pause guard.

### Proof of Concept

1. A `PauseManager` calls `pa_pause_feature("verify_transaction_inclusion")` to retire the deprecated v1 endpoint.
2. Any NEAR contract or user calls `verify_transaction_inclusion_v2` with a valid proof.
3. `verify_transaction_inclusion_v2` passes its own `#[pause]` check (v2 is not paused).
4. Execution reaches line 368: `self.verify_transaction_inclusion(args.into())`.
5. The `#[pause]` guard inside `verify_transaction_inclusion` fires because v1 is paused.
6. The contract panics with a "Paused" error; the call reverts.
7. All SPV verification is broken for all callers until v1 is explicitly unpaused — defeating the purpose of the deprecation. [2](#0-1) [1](#0-0)

    pub fn verify_transaction_inclusion_v2(&self, #[serializer(borsh)] args: ProofArgsV2) -> bool {

            args.merkle_proof.len() == args.coinbase_merkle_proof.len(),
            "Coinbase merkle proof and transaction merkle proof should have the same length"

            .get(&args.tx_block_blockhash)
            .unwrap_or_else(|| env::panic_str("cannot find requested transaction block"));

            merkle_tools::compute_root_from_merkle_proof(
                args.coinbase_tx_id.clone(),
                0usize,
                &args.coinbase_merkle_proof,
            ) == header.block_header.merkle_root,
            "Incorrect coinbase merkle proof"
        );

        #[allow(deprecated)]
