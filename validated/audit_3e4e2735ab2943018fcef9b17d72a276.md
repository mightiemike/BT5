### Title
Minimum Burn Fee Floor Causes 100% NLP Value Destruction for Sub-$1 Burns — (`core/contracts/Clearinghouse.sol`)

### Summary

`Clearinghouse.burnNlp` applies a minimum burn fee of `ONE = 1e18` (representing $1 in X18 fixed-point). When a user burns an `nlpAmount` whose X18 value is less than $1, the fee exceeds the gross quote, `quoteAmount` is clamped to zero, and the user's NLP is burned while they receive no quote in return.

### Finding Description

In `burnNlp`, the fee and payout are computed as:

```solidity
int128 quoteAmount = nlpAmount.mul(oraclePriceX18);          // X18 value of NLP burned
int128 burnFee     = MathHelper.max(ONE, quoteAmount / 1000); // floor = 1e18 = $1
quoteAmount        = MathHelper.max(0, quoteAmount - burnFee);// clamped to 0 if value < $1
``` [5](#0-4) 

When `nlpAmount * oraclePriceX18 / 1e18 < 1e18` (i.e., the NLP being burned is worth less than $1):

- `quoteAmount / 1000 < 1e15 < ONE`, so `burnFee = ONE = 1e18`
- `quoteAmount - burnFee < 0`, so `MathHelper.max(0, ...)` yields `quoteAmount = 0`

The NLP balance update is **unconditional**:

```solidity
spotEngine.updateBalance(NLP_PRODUCT_ID, txn.sender, -nlpAmount); // always burns
spotEngine.updateBalance(NLP_PRODUCT_ID, N_ACCOUNT,   nlpAmount); // N_ACCOUNT gains NLP
``` [6](#0-5) 

But the quote credit is gated:

```solidity
if (quoteAmount > 0) {
    spotEngine.updateBalance(QUOTE_PRODUCT_ID, txn.sender, quoteAmount);
    _applyNlpRebalance(spotEngine, nlpPools, nlpPoolRebalanceX18);
}
``` [7](#0-6) 

When `quoteAmount = 0`, the `if` block is skipped entirely. The user's NLP is gone; they receive nothing; `N_ACCOUNT` absorbs the NLP tokens. There is no minimum-burn-amount guard anywhere in the function. [8](#0-7) 

Additionally, a `HEALTHCHECK_FEE = 1e18` ($1) is charged by the Endpoint **before** `burnNlp` is even called:

```solidity
chargeFee(signedTx.tx.sender, HEALTHCHECK_FEE);
clearinghouse.burnNlp(...);
``` [9](#0-8) 

So the user also loses an additional $1 in quote from the healthcheck fee, compounding the loss.

### Impact Explanation

- The user's NLP tokens are burned (transferred to `N_ACCOUNT`) with zero quote returned.
- 100% of the NLP value (up to just under $1) is destroyed for the user.
- `N_ACCOUNT` gains NLP tokens without paying quote, representing a value transfer from the user to the protocol's internal account.
- The broken invariant: *a user burning NLP must receive quote proportional to NLP value minus a reasonable fee*. Here the fee consumes the entire value.
- Maximum loss per transaction is bounded to just under `ONE` ($1) in NLP value, plus the $1 `HEALTHCHECK_FEE` charged separately.

### Likelihood Explanation

The path is directly reachable: any user holding a small unlocked NLP balance (or choosing to burn a fractional amount) can trigger this. No special privileges are required. The user signs a `BurnNlp` transaction with a small `nlpAmount`; the sequencer processes it normally. The condition `nlpAmount * oraclePriceX18 / 1e18 < 1e18` is easily satisfied — for example, at an NLP price of $1, burning any amount less than 1 full NLP token (i.e., `nlpAmount < 1e18`) triggers it. [5](#0-4) 

### Recommendation

Add a minimum burn amount check before the fee calculation, reverting if the gross `quoteAmount` does not exceed the minimum fee:

```solidity
int128 quoteAmount = nlpAmount.mul(oraclePriceX18);
int128 burnFee = MathHelper.max(ONE, quoteAmount / 1000);
require(quoteAmount > burnFee, "ERR_NLP_BURN_AMOUNT_TOO_SMALL");
quoteAmount = quoteAmount - burnFee;
```

This ensures the transaction reverts rather than silently destroying the user's entire NLP value.

### Proof of Concept

```solidity

// nlpAmount = 5e17 (0.5 NLP = $0.50 in X18)

int128 oraclePrice  = 1e18;  // $1 per NLP

// Step 1: quoteAmount = 5e17 * 1e18 / 1e18 = 5e17  (< ONE = 1e18)
int128 quoteAmount  = nlpAmount.mul(oraclePrice);  // = 5e17

// Step 2: burnFee = max(1e18, 5e17/1000) = max(1e18, 5e14) = 1e18
int128 burnFee      = MathHelper.max(ONE, quoteAmount / 1000);  // = 1e18

// Step 3: quoteAmount = max(0, 5e17 - 1e18) = max(0, -5e17) = 0
quoteAmount         = MathHelper.max(0, quoteAmount - burnFee);  // = 0

// Result:
// - spotEngine.updateBalance(NLP_PRODUCT_ID, sender, -5e17)  ← NLP burned
// - spotEngine.updateBalance(NLP_PRODUCT_ID, N_ACCOUNT, 5e17) ← N_ACCOUNT gains
// - if (0 > 0) → false → user receives NO quote
// Assert: user received 0 quote for NLP worth $0.50 → 100% value destruction
``` [10](#0-9)
