### Title
Bonus Accrual Checkpoint Advanced Without Payment During Insufficient-Pool Redelegation Slash — (`x/tieredrewards/keeper/slash.go`)

### Summary

In `slashRedelegationPosition`, when `processEventsAndClaimBonus` returns `ErrInsufficientBonusPool`, the function deliberately continues and persists the position. However, `processEventsAndClaimBonus` has already (a) decremented validator-event reference counts in storage and (b) advanced `LastBonusAccrual` and `LastEventSeq` in the in-memory `pos` struct — all before the pool-balance check. The position is then saved with those advanced checkpoints even though zero bonus coins were transferred. The user's accrued bonus for the entire period `[original LastBonusAccrual, blockTime]` is permanently and irrecoverably lost.

### Finding Description

`processEventsAndClaimBonus` processes events in a loop, decrementing each event's reference count in storage and advancing `pos.LastEventSeq` in memory. After the loop it calls `applyBonusAccrualCheckpoint` (advancing `pos.LastBonusAccrual`) and `pos.UpdateLastKnownBonded`, then checks pool balance: [1](#0-0) 

The `decrementEventRefCount` calls inside the loop are unconditional storage writes committed to the live context — they are not rolled back when the function later returns an error. `applyBonusAccrualCheckpoint` and `UpdateLastKnownBonded` mutate the in-memory `pos` struct before the pool check at line 230: [2](#0-1) 

`slashRedelegationPosition` catches `ErrInsufficientBonusPool` and silently continues: [3](#0-2) 

It then calls `k.setPositionWithState(ctx, pos, nil)` (partial slash) or `k.setPositionWithState(ctx, pos, &ValidatorTransition{...})` (full slash with `ClearBonusCheckpoints`), persisting the already-mutated `pos`: [4](#0-3) 

For a **partial slash**, the position is saved with `LastBonusAccrual = blockTime` and `LastEventSeq` advanced, but no bonus coins were sent. The user can never reclaim the forfeited period because the checkpoint has moved forward and the events have been garbage-collected.

For a **full slash**, `ClearBonusCheckpoints()` zeroes the checkpoints, so the advanced state from `processEventsAndClaimBonus` is overwritten — but the event reference counts were still decremented in storage, and the bonus is still lost.

The hook entry point does not use a `CacheContext`: [5](#0-4) 

In contrast, all user-driven paths (e.g., `ClaimTierRewards`) rely on Cosmos SDK's `DeliverTx` CacheContext to roll back the entire transaction on error, so the checkpoint advancement is never persisted when the pool is insufficient. The slash hook path has no such rollback.

### Impact Explanation

A delegator who has a tier position on a redelegating validator loses all accrued bonus rewards for the period `[LastBonusAccrual, blockTime]` whenever the bonus pool is insufficient at slash time. The loss is permanent: the checkpoint is advanced, the validator events are reference-count-decremented (and potentially garbage-collected), and no retry is possible. The corrupted value is the `RewardsPoolName` module account balance relative to what was owed to the position owner — coins that should have been transferred are silently retained in the pool while the position's accounting records them as settled.

### Likelihood Explanation

The bonus pool is a finite, externally funded module account (`RewardsPoolName`). As positions accumulate and claim, the pool depletes. Validator slashes (jailing for downtime, double-sign evidence) are normal mainnet events. Any delegator who redelegates a tier position and whose source validator is subsequently slashed while the pool balance is below the position's accrued bonus will silently lose that bonus. No privileged access is required; the trigger is a standard `MsgBeginRedelegate` followed by a validator slash, both of which are normal Cosmos SDK transactions.

### Recommendation

Move `applyBonusAccrualCheckpoint` and `pos.UpdateLastKnownBonded` to after the successful `SendCoinsFromModuleToAccount` call, so checkpoints are only advanced when coins are actually transferred. Additionally, `slashRedelegationPosition` should use a `CacheContext` for the `processEventsAndClaimBonus` call so that event reference-count decrements are also rolled back on `ErrInsufficientBonusPool`, preserving the user's ability to claim later:

```go
cacheCtx, commit := sdk.UnwrapSDKContext(ctx).CacheContext()
if _, err := k.processEventsAndClaimBonus(cacheCtx, &pos); err != nil {
    if errors.Is(err, types.ErrInsufficientBonusPool) {
        // do NOT commit; checkpoints and ref-counts stay unchanged
        k.logger(ctx).Error(...)
        // proceed with slash without advancing bonus state
    } else {
        return err
    }
} else {
    commit()
}
```

### Proof of Concept

1. Alice locks CRO in a tier and redelegates to validator B via `MsgTierRedelegate`. A `RedelegationMapping` entry is created.
2. The bonus pool is drained to near-zero by other legitimate claims.
3. Validator A (source) is slashed. The SDK calls `BeforeRedelegationSlashed` → `slashRedelegationPosition`.
4. `processEventsAndClaimBonus` runs: it decrements event reference counts in storage, advances `pos.LastBonusAccrual` to `blockTime` in memory, then hits `sufficientBonusPoolBalance` and returns `ErrInsufficientBonusPool`.
5. `slashRedelegationPosition` catches the error, reduces `pos.Delegation.Shares`, and calls `setPositionWithState` — persisting `LastBonusAccrual = blockTime` and the advanced `LastEventSeq`.
6. Alice later calls `MsgClaimTierRewards`. `processEventsAndClaimBonus` starts from the new `LastBonusAccrual` (blockTime of the slash), skipping the entire pre-slash accrual period. Alice receives zero bonus for that period and cannot recover it. [6](#0-5) [1](#0-0) [7](#0-6)

```go
func (k Keeper) slashRedelegationPosition(ctx context.Context, unbondingId uint64, sharesToUnbond math.LegacyDec) error {
	positionId, err := k.getRedelegationMapping(ctx, unbondingId)
	if errors.Is(err, collections.ErrNotFound) {
		return nil
	}
	if err != nil {
		return err
	}

	pos, err := k.getPositionState(ctx, positionId)
	if errors.Is(err, types.ErrPositionNotFound) {
		k.logger(ctx).Error("position not found during redelegation slash",
			"position_id", positionId,
			"unbonding_id", unbondingId,
			"error", err.Error(),
		)
		return nil
	}
	if err != nil {
		return err
	}

	if !pos.IsDelegated() {
		// Defensive
		k.logger(ctx).Error("delegation missing during BeforeRedelegationSlashed",
			"position_id", positionId,
			"unbonding_id", unbondingId,
			"shares_to_unbond", sharesToUnbond.String(),
		)
		return nil
	}

	dstValStr := pos.Delegation.ValidatorAddress

	// Settle bonus against PRE-slash shares.
	if _, err := k.processEventsAndClaimBonus(ctx, &pos); err != nil {
		// Deliberately forgo bonus rewards if pool is insufficient to prevent chain halt.
		if errors.Is(err, types.ErrInsufficientBonusPool) {
			k.logger(ctx).Error("insufficient bonus pool during redelegation slash",
				"position_id", pos.Id,
				"error", err.Error(),
			)
		} else {
			return err
		}
	}

	fullSlash := sharesToUnbond.GTE(pos.Delegation.Shares)

	if fullSlash {
		pos.Delegation = nil
		pos.ClearBonusCheckpoints()
		return k.setPositionWithState(ctx, pos, &ValidatorTransition{PreviousAddress: dstValStr})
	}
	// In-memory only: the persisted Position carries no share count, and the
	// live delegation will reflect the post-Unbond shares on the next read.
	// Update the local copy so any follow-up logic in this call sees consistent state.
	pos.Delegation.Shares = pos.Delegation.Shares.Sub(sharesToUnbond)
	return k.setPositionWithState(ctx, pos, nil)
```

```go
func applyBonusAccrualCheckpoint(pos *types.Position, blockTime time.Time) {
	accrualEnd := blockTime
	if pos.CompletedExitLockDuration(blockTime) {
		accrualEnd = pos.ExitUnlockAt
	}
	pos.UpdateLastBonusAccrual(accrualEnd)
}
```
