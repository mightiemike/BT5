### Title
Any User Can Permanently Block IBC NFT Transfers for a Specific Class by Pre-Registering a Denom Name Collision — (`x/nft-transfer/keeper/packet.go`, `x/nft/keeper/denom.go`)

### Summary

`processReceivedPacket` in `x/nft-transfer` creates an NFT denom using the IBC voucher class ID as **both** the denom ID and the denom name. `SetDenom` enforces global uniqueness on both fields independently. Because `MsgIssueDenom.ValidateBasic()` only restricts the denom **ID** (alphanumeric, no `ibc/` prefix) but places no meaningful restriction on the denom **name**, any unprivileged user can pre-register a denom whose name equals the deterministic IBC voucher class ID for a target class/channel pair. When the first IBC NFT packet for that class arrives, denom creation fails with "denomName already exists", the packet is permanently unprocessable, and the IBC class is irreversibly broken on this chain.

### Finding Description

When an IBC NFT packet is received, `processReceivedPacket` computes a deterministic voucher class ID and attempts to create the denom if it does not yet exist: [1](#0-0) 

```go
if !k.nftKeeper.HasDenomID(ctx, voucherClassID) {
    if err := k.nftKeeper.IssueDenom(ctx, voucherClassID, voucherClassID, "", data.ClassUri, escrowAddress); err != nil {
        return err
    }
}
```

Both the denom **ID** and denom **name** are set to `voucherClassID` (e.g. `ibc/ABCDEF...`).

`SetDenom` enforces uniqueness on both fields independently: [2](#0-1) 

```go
func (k Keeper) SetDenom(ctx sdk.Context, denom types.Denom) error {
    if k.HasDenomID(ctx, denom.Id) {
        return sdkerrors.Wrapf(types.ErrInvalidDenom, "denomID %s has already exists", denom.Id)
    }
    if k.HasDenomNm(ctx, denom.Name) {
        return sdkerrors.Wrapf(types.ErrInvalidDenom, "denomName %s has already exists", denom.Name)
    }
    ...
}
```

`MsgIssueDenom.ValidateBasic()` calls `ValidateDenomID` on the ID field (which rejects `ibc/` prefix) but calls `ValidateDenomName` on the name field, which only rejects empty strings: [3](#0-2) [4](#0-3) 

```go
func ValidateDenomName(denomName string) error {
    denomName = strings.TrimSpace(denomName)
    if len(denomName) == 0 {
        return sdkerrors.Wrapf(ErrInvalidDenomName, "denom name(%s) can not be space", denomName)
    }
    return nil
}
```

An attacker can therefore submit a `MsgIssueDenom` with:
- `Id = "attackdenom"` (valid alphanumeric)
- `Name = "ibc/ABCDEF..."` (the target voucher class ID — accepted by `ValidateDenomName`)

The voucher class ID is fully deterministic and publicly computable: [5](#0-4) 

```go
// hash = sha256(tracePath + "/" + baseClassId)
func (ct ClassTrace) Hash() tmbytes.HexBytes {
    hash := sha256.Sum256([]byte(ct.GetFullClassPath()))
    return hash[:]
}
func (ct ClassTrace) IBCClassID() string {
    if ct.Path != "" {
        return fmt.Sprintf("%s/%s", ClassPrefix, ct.Hash())
    }
    return ct.BaseClassId
}
```

After the attacker's denom is registered, `HasDenomID(voucherClassID)` returns `false` (the attacker used a different ID), so the code proceeds to `IssueDenom`. `SetDenom` then finds `HasDenomNm(voucherClassID) == true` and returns an error. This error propagates out of `processReceivedPacket`, causing the IBC packet to fail. There is no `DeleteDenom` function in the module, so the name collision is permanent. [6](#0-5) 

### Impact Explanation

The targeted IBC NFT class can never be received on this chain. Every inbound packet for that class/channel pair will fail at denom creation, triggering an error acknowledgment. The source chain refunds the sender via `refundPacketToken`, so NFTs are not permanently lost, but the IBC class is irreversibly broken on the receiving chain. Any application or user relying on cross-chain NFT transfers for that class is permanently denied service with no on-chain recovery path.

### Likelihood Explanation

The attack requires only a standard `MsgIssueDenom` transaction — no special role, no leaked key, no governance action. The target voucher class ID is deterministic and computable from public information (destination port, channel ID, and source class ID). An attacker monitoring mempool or IBC channel announcements can front-run the first legitimate transfer of any NFT class. The cost is a single transaction fee.

### Recommendation

Restrict the denom name in `MsgIssueDenom.ValidateBasic()` to the same character set as the denom ID (i.e., apply `ValidateDenomID` to the name as well), or explicitly reject names that match the `ibc/` prefix pattern. Alternatively, decouple the IBC denom name from the IBC class ID so that name collisions with user-issued denoms cannot occur.

### Proof of Concept

1. Observe that `nft/channel-0/kitty` will be transferred via IBC. Compute `voucherClassID = "ibc/" + hex(sha256("nft/channel-0/kitty"))`.
2. Submit `MsgIssueDenom{Id: "attackdenom", Name: voucherClassID, Sender: attacker}`. This passes `ValidateDenomID("attackdenom")` and `ValidateDenomName(voucherClassID)` (non-empty), and is accepted on-chain.
3. A legitimate user sends the NFT from the source chain. The relayer submits the IBC packet.
4. `processReceivedPacket` calls `HasDenomID(voucherClassID)` → `false`, then `IssueDenom(voucherClassID, voucherClassID, ...)`.
5. `SetDenom` calls `HasDenomNm(voucherClassID)` → `true` (attacker's denom has this name). Returns `ErrInvalidDenom: "denomName ibc/... has already exists"`.
6. `processReceivedPacket` returns the error. The packet fails permanently. All future packets for `kitty` on `channel-0` fail identically.

```go
func (k Keeper) SetDenom(ctx sdk.Context, denom types.Denom) error {
	if k.HasDenomID(ctx, denom.Id) {
		return sdkerrors.Wrapf(types.ErrInvalidDenom, "denomID %s has already exists", denom.Id)
	}

	if k.HasDenomNm(ctx, denom.Name) {
		return sdkerrors.Wrapf(types.ErrInvalidDenom, "denomName %s has already exists", denom.Name)
	}

	store := ctx.KVStore(k.storeKey)
	bz := k.cdc.MustMarshal(&denom)
	store.Set(types.KeyDenomID(denom.Id), bz)
	store.Set(types.KeyDenomName(denom.Name), []byte(denom.Id))
	return nil
```

```go
// IBCClassID a classID for an ICS721 non-fungible token in the format
// 'ibc/{hash(tracePath + BaseClassId)}'. If the trace is empty, it will return the base classID.
```
