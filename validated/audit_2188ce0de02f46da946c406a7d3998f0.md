### Title
Unchecked `transferFrom` Return Value Enables usdcE Drain Without Providing USDC - (File: `core/contracts/ContractOwner.sol`)

---

### Summary

`ContractOwner.replaceUsdcEWithUsdc()` calls `IERC20Base(usdc).transferFrom(...)` directly without checking its return value, while the protocol already provides `ERC20Helper.safeTransferFrom()` for exactly this purpose. If the raw `transferFrom` silently returns `false`, execution continues and the caller receives usdcE from the `DirectDepositV1` contract without having deposited any USDC.

---

### Finding Description

`replaceUsdcEWithUsdc` is an **unpermissioned** external function (no `onlyOwner` or `onlyDeployer` modifier) callable by any address on chain ID 57073. Its logic is:

1. Read `usdcE` balance held by `directDepositV1`.
2. Call `IERC20Base(usdc).transferFrom(msg.sender, directDepositV1, balance)` — **raw call, return value discarded**.
3. Call `DirectDepositV1(directDepositV1).withdraw(usdcE)` — pulls usdcE into `ContractOwner`.
4. Call `IERC20Base(usdcE).safeTransfer(msg.sender, balance)` — sends usdcE to caller.

Step 2 uses the raw `IERC20Base.transferFrom` interface method whose return value is never checked: [1](#0-0) 

The protocol already defines `ERC20Helper.safeTransferFrom`, which wraps the call in a low-level `.call()` and asserts both `success` and the decoded boolean return value: [2](#0-1) 

`ERC20Helper` is imported and `using ERC20Helper for IERC20Base` is active in `ContractOwner`, so the safe wrapper is available but unused at this call site: [3](#0-2) 

---

### Impact Explanation

If `usdc.transferFrom` returns `false` (e.g., caller has insufficient allowance or balance, or the token is a non-reverting non-standard ERC20), the function does **not** revert. It proceeds to withdraw usdcE from `directDepositV1` into `ContractOwner` and then transfers that usdcE to the caller. The caller receives real usdcE tokens without having provided any USDC — a direct asset loss from the `directDepositV1` vault.

---

### Likelihood Explanation

The function is publicly callable with no access control. Any user who identifies a `directDepositV1` address holding usdcE can call `replaceUsdcEWithUsdc` with zero USDC allowance. If the deployed USDC token at `0x2D270e6886d130D724215A266106e6832161EAEd` on chain 57073 does not revert on a failed transfer (returning `false` instead), the drain succeeds unconditionally. Even for a reverting USDC, the missing check is a latent integration risk for any future token substitution following the same pattern.

---

### Recommendation

Replace the raw `transferFrom` call with the protocol's own `safeTransferFrom` wrapper:

```solidity
// Before (vulnerable)
IERC20Base(usdc).transferFrom(msg.sender, directDepositV1, balance);

// After (safe)
IERC20Base(usdc).safeTransferFrom(msg.sender, directDepositV1, balance);
```

`safeTransferFrom` in `ERC20Helper` uses a low-level `.call()` and requires `success && (data.length == 0 || abi.decode(data, (bool)))`, correctly handling both reverting and non-reverting non-standard tokens. [4](#0-3) 

---

### Proof of Concept

1. A `directDepositV1` for some `subaccount` exists and holds `N` usdcE tokens.
2. Attacker calls `replaceUsdcEWithUsdc(subaccount)` with **zero** USDC allowance granted to `ContractOwner`.
3. `IERC20Base(usdc).transferFrom(attacker, directDepositV1, N)` returns `false` (no revert).
4. Return value is ignored; execution continues.
5. `DirectDepositV1(directDepositV1).withdraw(usdcE)` moves `N` usdcE to `ContractOwner`.
6. `IERC20Base(usdcE).safeTransfer(attacker, N)` sends `N` usdcE to the attacker.
7. Attacker receives `N` usdcE; `directDepositV1` loses its entire usdcE balance; no USDC was deposited. [5](#0-4)

        require(block.chainid == 57073, ERR_UNAUTHORIZED);

        require(directDepositV1 != address(0), "no dda");

        if (balance > 0) {
            IERC20Base(usdc).transferFrom(msg.sender, directDepositV1, balance);
            DirectDepositV1(directDepositV1).withdraw(IIERC20Base(usdcE));
            IERC20Base(usdcE).safeTransfer(msg.sender, balance);
        }
    }

**File:** core/contracts/libraries/ERC20Helper.sol (L23-42)
```text
    function safeTransferFrom(
        IERC20Base self,
        address from,
        address to,
        uint256 amount
    ) internal {
        (bool success, bytes memory data) = address(self).call(
            abi.encodeWithSelector(
                IERC20Base.transferFrom.selector,
                from,
                to,
                amount
            )
        );

        require(
            success && (data.length == 0 || abi.decode(data, (bool))),
            ERR_TRANSFER_FAILED
        );
    }
```
