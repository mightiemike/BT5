### Title
Protobuf `ValidResourceBounds` Deserialization Heuristic Silently Flips `AllResources` to `L1Gas`, Producing a Different Transaction Hash on the Receiving Node - (File: `crates/apollo_protobuf/src/converters/transaction.rs`)

---

### Summary

The protobuf deserialization of `ValidResourceBounds` uses a zero-value heuristic to decide whether a transaction carries `L1Gas` or `AllResources` bounds. An `AllResources` V3 transaction whose `l2_gas` and `l1_data_gas` fields are both zero is silently re-typed to `L1Gas` on the receiving P2P node. Because `get_tip_resource_bounds_hash` produces a structurally different hash for the two variants (two vs. three packed resource felts), the receiving node computes a transaction hash that differs from the one committed by the proposer, causing block/transaction rejection and a reachable consensus split.

---

### Finding Description

**Serialization side** (`From<ValidResourceBounds> for protobuf::ResourceBounds`, line 471–489):

When `ValidResourceBounds::AllResources` is serialized, all three resource fields are written to the wire, including `l1_data_gas`. If the original transaction had `l2_gas = 0` and `l1_data_gas = 0`, the wire bytes are numerically identical to those of a `ValidResourceBounds::L1Gas` transaction. [1](#0-0) 

**Deserialization side** (`TryFrom<protobuf::ResourceBounds> for ValidResourceBounds`, line 431):

```rust
Ok(if l1_data_gas.is_zero() && l2_gas.is_zero() {
    ValidResourceBounds::L1Gas(l1_gas)          // ← wrong variant for AllResources tx
} else {
    ValidResourceBounds::AllResources(...)
})
``` [2](#0-1) 

The heuristic is lossy: the `AllResources` variant with zero `l2_gas`/`l1_data_gas` is indistinguishable from `L1Gas` in the protobuf wire format, so the receiving node always reconstructs the wrong variant.

**Hash divergence** (`get_tip_resource_bounds_hash`, lines 188–211):

The function branches on the variant tag to decide how many resource felts to include in the Poseidon hash:

- `L1Gas` → `[tip, L1_GAS_packed, L2_GAS_packed]` — **2 resource felts**
- `AllResources` → `[tip, L1_GAS_packed, L2_GAS_packed, L1_DATA_GAS_packed]` — **3 resource felts** [3](#0-2) 

Because the packed `L1_DATA_GAS` felt is omitted for `L1Gas`, the two hashes are structurally different even when `l1_data_gas = 0`.

**Concrete divergence path:**

1. User submits `RpcInvokeTransactionV3` with `AllResourceBounds { l1_gas: X, l2_gas: 0, l1_data_gas: 0 }`. The `RpcInvokeTransactionV3` struct mandates `AllResourceBounds`, so this is always an `AllResources` transaction at the RPC layer. [4](#0-3) 

2. `TransactionConverter::convert_rpc_tx_to_internal` converts it to `InternalRpcInvokeTransactionV3` (also `AllResourceBounds`), then calls `calculate_transaction_hash` which invokes `get_invoke_transaction_v3_hash` → `get_tip_resource_bounds_hash` with `ValidResourceBounds::AllResources(...)` → **hash H1** (3-felt variant). [5](#0-4) [6](#0-5) 

3. The transaction is stored and included in a block with hash H1.

4. During P2P propagation, `InvokeTransactionV3` (with `ValidResourceBounds::AllResources`) is serialized to `protobuf::InvokeV3`. The `l2_gas` and `l1_data_gas` wire fields are both zero.

5. The receiving node calls `InvokeTransactionV3::try_from(protobuf::InvokeV3)`, which calls `ValidResourceBounds::try_from(protobuf::ResourceBounds)`. Because `l1_data_gas.is_zero() && l2_gas.is_zero()`, the result is `ValidResourceBounds::L1Gas(l1_gas)`. [7](#0-6) 

6. The receiving node recomputes the hash using `get_tip_resource_bounds_hash` with `ValidResourceBounds::L1Gas(...)` → **hash H2** (2-felt variant). H1 ≠ H2.

---

### Impact Explanation

The receiving node holds a transaction object whose recomputed hash differs from the hash committed by the proposer. Any hash-equality check during block validation, state diff application, or receipt verification will fail. A single crafted `AllResources` transaction with zero `l2_gas`/`l1_data_gas` is sufficient to make every validator reject the proposer's block, causing a consensus liveness failure. The impact matches: **High — Transaction conversion or signature/hash logic binds the wrong signer, hash, type, or executable payload.**

---

### Likelihood Explanation

The `RpcInvokeTransactionV3` struct unconditionally uses `AllResourceBounds`, so any V3 invoke transaction submitted through the RPC is an `AllResources` transaction. Setting `l2_gas = 0` and `l1_data_gas = 0` while keeping `l1_gas > 0` passes all stateless and stateful gateway checks (the gateway only requires at least one non-zero bound). No privileged access is required; any unprivileged user can submit such a transaction.

---

### Recommendation

1. **Add a discriminant to the protobuf schema** — introduce a boolean or enum field (e.g., `is_all_resources`) so the receiving node can reconstruct the exact variant without relying on zero-value inference.
2. **Alternatively, always deserialize V3 transactions as `AllResources`** — since `L1Gas` is a pre-0.13.3 legacy variant, any transaction arriving over the V3 P2P path should unconditionally produce `AllResources`, eliminating the ambiguity.
3. **Add a round-trip invariant test** — assert that `ValidResourceBounds::AllResources(AllResourceBounds { l1_gas: X, l2_gas: 0, l1_data_gas: 0 })` survives a protobuf serialize→deserialize cycle with the same variant and the same computed hash.

---

### Proof of Concept

```
// Step 1: craft the transaction on the originating node
let bounds = ValidResourceBounds::AllResources(AllResourceBounds {
    l1_gas:      ResourceBounds { max_amount: GasAmount(1000), max_price_per_unit: GasPrice(1) },
    l2_gas:      ResourceBounds::default(),   // zero
    l1_data_gas: ResourceBounds::default(),   // zero
});
// get_tip_resource_bounds_hash produces:
//   poseidon(tip, pack(L1_GAS, 1000, 1), pack(L2_GAS, 0, 0), pack(L1_DATA_GAS, 0, 0))
//   → H1  (3-felt input)

// Step 2: serialize to protobuf (wire bytes: l2_gas=0, l1_data_gas=0)
let proto: protobuf::ResourceBounds = bounds.into();

// Step 3: deserialize on the receiving node
let reconstructed = ValidResourceBounds::try_from(proto).unwrap();
// reconstructed == ValidResourceBounds::L1Gas(ResourceBounds { max_amount: 1000, ... })
// because l1_data_gas.is_zero() && l2_gas.is_zero()

// Step 4: receiving node recomputes hash
// get_tip_resource_bounds_hash produces:
//   poseidon(tip, pack(L1_GAS, 1000, 1), pack(L2_GAS, 0, 0))
//   → H2  (2-felt input)

assert_ne!(H1, H2);  // ← hash mismatch; block rejected by receiving node
```

pub fn get_tip_resource_bounds_hash(
    resource_bounds: &ValidResourceBounds,
    tip: &Tip,
) -> Result<Felt, StarknetApiError> {
    let l1_resource_bounds = resource_bounds.get_l1_bounds();
    let l2_resource_bounds = resource_bounds.get_l2_bounds();

    // L1 and L2 gas bounds always exist.
    // Old V3 txs always have L2 gas bounds of zero, but they exist.
    let mut resource_felts = vec![
        get_concat_resource(&l1_resource_bounds, L1_GAS)?,
        get_concat_resource(&l2_resource_bounds, L2_GAS)?,
    ];

    // For new V3 txs, need to also hash the data gas bounds.
    resource_felts.extend(match resource_bounds {
        ValidResourceBounds::L1Gas(_) => vec![],
        ValidResourceBounds::AllResources(all_resources) => {
            vec![get_concat_resource(&all_resources.l1_data_gas, L1_DATA_GAS)?]
        }
    });

    Ok(HashChain::new().chain(&tip.0.into()).chain_iter(resource_felts.iter()).get_poseidon_hash())
}

pub(crate) fn get_invoke_transaction_v3_hash<T: InvokeTransactionV3Trait>(
    transaction: &T,
    chain_id: &ChainId,
    transaction_version: &TransactionVersion,
) -> Result<TransactionHash, StarknetApiError> {
    let tip_resource_bounds_hash =
        get_tip_resource_bounds_hash(&transaction.resource_bounds(), transaction.tip())?;
    let paymaster_data_hash =
        HashChain::new().chain_iter(transaction.paymaster_data().0.iter()).get_poseidon_hash();
    let data_availability_mode = concat_data_availability_mode(
        transaction.nonce_data_availability_mode(),
        transaction.fee_data_availability_mode(),
    );
    let account_deployment_data_hash = HashChain::new()
        .chain_iter(transaction.account_deployment_data().0.iter())
        .get_poseidon_hash();
    let calldata_hash =
        HashChain::new().chain_iter(transaction.calldata().0.iter()).get_poseidon_hash();
    let mut hash_chain = HashChain::new()
        .chain(&INVOKE)
        .chain(&transaction_version.0)
        .chain(transaction.sender_address().0.key())
        .chain(&tip_resource_bounds_hash)
        .chain(&paymaster_data_hash)
        .chain(&Felt::try_from(chain_id)?)
        .chain(&transaction.nonce().0)
        .chain(&data_availability_mode)
        .chain(&account_deployment_data_hash)
        .chain(&calldata_hash);
    if !transaction.proof_facts().0.is_empty() {
        let proof_facts_hash =
            HashChain::new().chain_iter(transaction.proof_facts().0.iter()).get_poseidon_hash();
        hash_chain = hash_chain.chain(&proof_facts_hash);
    }
    Ok(TransactionHash(hash_chain.get_poseidon_hash()))
