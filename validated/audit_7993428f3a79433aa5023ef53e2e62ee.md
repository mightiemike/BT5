### Title
`TierRedelegate` Hardcodes `LastKnownBonded=true` for Dst Validator Regardless of Its Bond Status, Enabling Unbacked Bonus Drain — (`x/tieredrewards/keeper/msg_server.go`)

### Summary
`TierRedelegate` unconditionally calls `pos.UpdateBonusCheckpoints(latestSeq, blockTime, true)` after redelegating to a destination validator, setting `LastKnownBonded=true` even when the destination validator is currently unbonded or unbonding. Because `validateRedelegatePosition` imposes no check on the destination validator's bond status, an attacker can redelegate to an unbonded validator, skip the UNBOND event via the `latestSeq` cursor, and cause `processEventsAndClaimBonus` to compute bonus for the entire unbonded gap when the validator later re-bonds.

### Finding Description

**Step 1 — Entry point: `TierRedelegate`** [1](#0-0) 

After `redelegate` succeeds, the code reads the latest event sequence for the destination validator and calls:

```go
pos.UpdateBonusCheckpoints(latestSeq, sdkCtx.BlockTime(), true)
``` [2](#0-1) 

The third argument `true` is hardcoded. `UpdateBonusCheckpoints` writes it directly into `pos.LastKnownBonded`: [3](#0-2) 

**Step 2 — No guard in `validateRedelegatePosition`** [4](#0-3) 

The function checks owner, delegated state, same-validator, exit lock, tier close-only, and active redelegation. There is no check that the destination validator is bonded.

**Step 3 — UNBOND event is skipped by the cursor**

`AfterValidatorBeginUnbonding` only appends an UNBOND event if there are already positions on the validator at unbonding time: [5](#0-4) 

If valB had no tiered positions when it unbonded, `getValidatorEventLatestSeq(valB)` returns 0. After redelegate, `LastEventSeq=0` and `LastKnownBonded=true`. If valB *did* have positions when it unbonded (UNBOND event at seq=N), `UpdateBonusCheckpoints(N, …, true)` sets `LastEventSeq=N`, skipping that UNBOND event entirely.

**Step 4 — Bonus computed for the unbonded gap** [6](#0-5) 

`processEventsAndClaimBonus` initialises `bonded = pos.LastKnownBonded` (= `true`). When the subsequent BOND event (seq=N+1) is processed inside the loop, the code computes:

```go
bonus := k.computeSegmentBonus(*pos, tier, segmentStart, evt.Timestamp, evt.TokensPerShare)
```

where `segmentStart = redelegate_time` and `evt.Timestamp = bond_event_time`. This pays bonus for the entire period `[redelegate_time, bond_event_time]` even though the validator was unbonded throughout.

**Step 5 — Defensive check does not help** [7](#0-6) 

The `if bonded && val.IsBonded()` guard only protects the **final open segment** (last event → current block). The bonus for the historical unbonded gap is already accumulated in `totalBonus` before this check is reached.

**Step 6 — Funds leave `RewardsPoolName`** [8](#0-7) 

The computed `bonusCoins` are sent from `types.RewardsPoolName` to the attacker's owner address.

### Impact Explanation
An attacker with a tiered position can redelegate to a validator that is currently unbonded or unbonding. On the validator's next re-bonding, `ClaimTierRewards` pays bonus for the entire unbonded interval as if the validator had been bonded throughout. The funds are drawn from `RewardsPoolName`, constituting a direct, unbacked economic loss to the protocol's reward pool. The magnitude scales with position size, tier `BonusApy`, and the length of the unbonded gap.

### Likelihood Explanation
The path is reachable via a standard `MsgTierRedelegate` transaction signed by any position owner. No governance, operator, or privileged access is required. Validators cycle through unbonding/bonding states in normal chain operation, making the precondition easy to satisfy.

### Recommendation
In `TierRedelegate`, replace the hardcoded `true` with the actual bond status of the destination validator at redelegate time:

```go
dstVal, err := ms.stakingKeeper.GetValidator(ctx, dstValAddr)
if err != nil {
    return nil, err
}
pos.UpdateBonusCheckpoints(latestSeq, sdkCtx.BlockTime(), dstVal.IsBonded())
```

Alternatively, reject `TierRedelegate` when the destination validator is not bonded in `validateRedelegatePosition`.

### Proof of Concept
1. Create a bonded tiered position on `valA`.
2. Transition `valB` to unbonded (no tiered positions on `valB` at this point, so no UNBOND event is recorded; `getValidatorEventLatestSeq(valB)` = 0).
3. Call `TierRedelegate` to `valB`. `UpdateBonusCheckpoints(0, T_redelegate, true)` is stored.
4. Advance block time by duration `D`.
5. Transition `valB` back to bonded. `AfterValidatorBonded` fires (position now exists on `valB`), appending BOND event at seq=1 with timestamp `T_bond`.
6. Call `ClaimTierRewards`. `processEventsAndClaimBonus` starts with `bonded=true`, `segmentStart=T_redelegate`. It processes the BOND event (seq=1) and computes bonus for `[T_redelegate, T_bond]` — the entire unbonded gap — then pays it from `RewardsPoolName`.
7. Assert: bonus paid > 0, but correct value is 0 (validator was unbonded throughout `[T_redelegate, T_bond]`).

func (k Keeper) validateRedelegatePosition(ctx context.Context, pos types.PositionState, owner, dstValidator string) error {
	if !pos.IsOwner(owner) {
		return types.ErrNotPositionOwner
	}

	if !pos.IsDelegated() {
		return types.ErrPositionNotDelegated
	}

	if pos.Delegation.ValidatorAddress == dstValidator {
		return types.ErrRedelegationToSameValidator
	}

	sdkCtx := sdk.UnwrapSDKContext(ctx)
	if pos.HasTriggeredExit() && pos.CompletedExitLockDuration(sdkCtx.BlockTime()) {
		return types.ErrExitLockDurationElapsed
	}

	tier, err := k.getTier(ctx, pos.TierId)
	if err != nil {
		return err
	}

	if tier.IsCloseOnly() {
		return types.ErrTierIsCloseOnly
	}

	isRedelegating, err := k.isRedelegating(ctx, pos.DelegatorAddress)
	if err != nil {
		return err
	}
	if isRedelegating {
		return errorsmod.Wrapf(types.ErrActiveRedelegation, "position %d has an active redelegation", pos.Id)
	}

	return nil
}
