### Title
`verify_transaction_inclusion` Remains Publicly Callable, Bypassing the Coinbase Merkle Proof Guard Added in v2 — (`contract/src/lib.rs`)

### Summary

`verify_transaction_inclusion_v2` introduced a coinbase Merkle proof check to close the 64-byte transaction Merkle forgery vulnerability. The original `verify_transaction_inclusion` (v1) is still a live public NEAR method and lacks this check. Any caller can invoke v1 directly to obtain a `true` result for a forged transaction inclusion proof, bypassing the security invariant that v2 enforces.

### Finding Description

`verify_transaction_inclusion_v2` enforces two guards absent from v1:

1. `merkle_proof.len() == coinbase_merkle_proof.len()` — structural length parity.
2. The coinbase Merkle proof must reconstruct the block's `merkle_root` with `tx_index = 0`. [1](#0-0) 

These checks prevent an attacker from supplying a 64-byte value as `tx_id` that is actually an internal Merkle tree node, which would otherwise hash correctly to the `merkle_root` without corresponding to any real transaction.

`verify_transaction_inclusion` (v1) performs none of these checks: [2](#0-1) 

It only verifies that `compute_root_from_merkle_proof(tx_id, tx_index, merkle_proof) == header.block_header.merkle_root`, which is exactly the condition the 64-byte forgery satisfies.

The `#[deprecated]` Rust attribute is a compile-time lint only. The function is decorated with `#[pause]` (not paused by default) and is `pub`, making it a fully reachable NEAR contract method: [3](#0-2) 

### Impact Explanation

An attacker can:
1. Identify a real Bitcoin block whose Merkle tree has an internal 64-byte node that, when treated as a leaf, hashes up to the correct `merkle_root`.
2. Call `verify_transaction_inclusion` with that crafted `tx_id` and a valid `tx_block_blockhash` already accepted by the light client.
3. Receive `true` — a forged proof of transaction inclusion.

Any downstream NEAR contract that calls `verify_transaction_inclusion` (v1) and acts on the boolean result (e.g., releasing tokens, confirming a cross-chain payment) is deceived into accepting a non-existent transaction. This is a direct violation of the system's core security guarantee: that only real, on-chain Bitcoin transactions can be proven included.

### Likelihood Explanation

There are no prerequisites. Any unprivileged NEAR account can call `verify_transaction_inclusion` at any time. The 64-byte forgery technique is well-documented (referenced in the contract's own NatDoc comment at line 271) and tooling to construct such proofs exists. The contract is live and the method is reachable.

### Recommendation

Remove the public exposure of `verify_transaction_inclusion`. Either:

- Change its visibility from `pub` to `pub(crate)` or remove the NEAR method export entirely, forcing all callers to use `verify_transaction_inclusion_v2`; or
- Add the same coinbase Merkle proof guard to v1 before the root comparison.

The simplest safe fix is to make v1 non-callable externally, since v2 already delegates to it internally via `self.verify_transaction_inclusion(args.into())`. [4](#0-3) 

### Proof of Concept

1. Deploy the contract with a known Bitcoin mainnet genesis block.
2. Submit a real block header (e.g., height 685452) via `submit_blocks`.
3. Construct a 64-byte value `fake_tx_id` that is an internal Merkle node of that block's transaction tree, such that `compute_root_from_merkle_proof(fake_tx_id, crafted_index, crafted_proof) == block.merkle_root`.
4. Call `verify_transaction_inclusion` (v1) with `tx_id = fake_tx_id`, `tx_block_blockhash = <real block hash>`, `confirmations = 1`, and the crafted proof.
5. The function returns `true` despite `fake_tx_id` not corresponding to any real transaction.
6. Call `verify_transaction_inclusion_v2` with the same inputs (plus any `coinbase_merkle_proof`) — the coinbase check causes it to `panic` with `"Incorrect coinbase merkle proof"`, demonstrating the guard is absent in v1. [5](#0-4)

        since = "0.5.0",

    #[pause]
    pub fn verify_transaction_inclusion(&self, #[serializer(borsh)] args: ProofArgs) -> bool {
        require!(
            args.confirmations <= self.gc_threshold,
            "The required number of confirmations exceeds the number of blocks stored in memory"
        );

        let heaviest_block_header = self
            .headers_pool
            .get(&self.mainchain_tip_blockhash)
            .unwrap_or_else(|| env::panic_str(ERR_KEY_NOT_EXIST));
        let target_block_height = self
            .mainchain_header_to_height
            .get(&args.tx_block_blockhash)
            .unwrap_or_else(|| env::panic_str("block does not belong to the current main chain"));

        // Check requested confirmations. No need to compute proof if insufficient confirmations.
        require!(
            (heaviest_block_header.block_height).saturating_sub(target_block_height) + 1
                >= args.confirmations,
            "Not enough blocks confirmed"
        );

        let header = self
            .headers_pool
            .get(&args.tx_block_blockhash)
            .unwrap_or_else(|| env::panic_str("cannot find requested transaction block"));

        require!(!args.merkle_proof.is_empty(), "Merkle proof is empty");

        // compute merkle tree root and check if it matches block's original merkle tree root
        merkle_tools::compute_root_from_merkle_proof(
            args.tx_id,
            usize::try_from(args.tx_index).unwrap(),
            &args.merkle_proof,
        ) == header.block_header.merkle_root
    }

    pub fn verify_transaction_inclusion_v2(&self, #[serializer(borsh)] args: ProofArgsV2) -> bool {
        require!(
            args.merkle_proof.len() == args.coinbase_merkle_proof.len(),
            "Coinbase merkle proof and transaction merkle proof should have the same length"
        );

        let header = self
            .headers_pool
            .get(&args.tx_block_blockhash)
            .unwrap_or_else(|| env::panic_str("cannot find requested transaction block"));

        require!(
            merkle_tools::compute_root_from_merkle_proof(
                args.coinbase_tx_id.clone(),
                0usize,
                &args.coinbase_merkle_proof,
            ) == header.block_header.merkle_root,
            "Incorrect coinbase merkle proof"
        );

        #[allow(deprecated)]
        self.verify_transaction_inclusion(args.into())
