Audit Report

## Title
`SwapAllowlistExtension` checks the router address instead of the end user, allowing any caller to bypass the per-user swap allowlist via `MetricOmmSimpleRouter` — (File: `metric-periphery/contracts/extensions/SwapAllowlistExtension.sol`)

## Summary
`SwapAllowlistExtension.beforeSwap` receives `sender` from the pool, which is always `msg.sender` of `pool.swap()`. When swaps are routed through `MetricOmmSimpleRouter`, `msg.sender` to the pool is the router contract address, not the originating user. A pool admin who allowlists the router address to enable periphery-mediated swaps for approved users inadvertently grants every public caller unrestricted access to the curated pool, completely defeating the allowlist.

## Finding Description
`MetricOmmPool.swap()` calls `_beforeSwap(msg.sender, ...)` at line 231, passing the direct caller as `sender`. `ExtensionCalling._beforeSwap()` encodes this `sender` value verbatim into the extension call at lines 160–176. `SwapAllowlistExtension.beforeSwap()` then evaluates `allowedSwapper[msg.sender][sender]` at line 37, where `msg.sender` is the pool and `sender` is whoever called `pool.swap()`.

When a user calls `MetricOmmSimpleRouter.exactInputSingle()` or `exactInput()`, the router calls `IMetricOmmPoolActions(pool).swap(...)` directly (lines 72–80 and 104–112), making the router contract `msg.sender` to the pool. The extension therefore checks `allowedSwapper[pool][routerAddress]`, not `allowedSwapper[pool][userAddress]`.

A pool admin who wants to support router-mediated swaps for their allowlisted users has no mechanism other than calling `setAllowedToSwap(pool, routerAddress, true)`. Once the router is allowlisted, the check `allowedSwapper[pool][routerAddress]` returns `true` for every caller regardless of who they are, because the extension has no way to recover the originating EOA from the `sender` argument. There is no secondary check, no `extensionData` inspection, and no fallback path that recovers the real user.

## Impact Explanation
Any unpermissioned user can execute swaps on a curated pool that relies on `SwapAllowlistExtension` to restrict trading to approved counterparties, provided the router is allowlisted. Because the pool is oracle-anchored with no internal price discovery, every unauthorized swap executes at the current fair-market price, extracting real value from LP positions. LP providers suffer direct loss of principal equal to the swap output, and the core invariant that a curated pool enforces the same allowlist policy regardless of which supported public entrypoint reaches it is broken. This constitutes a direct loss of LP assets above Sherlock thresholds and broken core pool functionality.

## Likelihood Explanation
Likelihood is High. `MetricOmmSimpleRouter` is the primary user-facing swap interface. A pool admin who intends to support router-mediated swaps for their allowlisted users must allowlist the router — there is no alternative mechanism in the extension. This is a natural, expected configuration step. Once taken, the bypass is immediately available to any public caller with zero additional preconditions: no privileged access, no special token, no admin cooperation beyond the one natural setup step. The attack is repeatable indefinitely.

## Recommendation
The extension must gate the originating user, not the intermediary contract. Two sound approaches:

1. **Pass the originating user through `extensionData`**: The router encodes `msg.sender` into `extensionData` before calling `pool.swap()`; the extension decodes and verifies it. This is acceptable because `MetricOmmSimpleRouter` is a protocol-controlled contract.

2. **Maintain a trusted-router registry in the extension**: When `sender` is a known router, extract the real user from `extensionData`; when `sender` is an EOA, check it directly. This preserves backward compatibility for direct pool callers.

Either fix must ensure the address checked against `allowedSwapper` is the address that economically initiates and benefits from the swap, not the settlement intermediary.
