Audit Report

## Title
`DepositAllowlistExtension` checks LP position `owner` instead of token-providing `sender`, allowing full deposit allowlist bypass — (`File: metric-periphery/contracts/extensions/DepositAllowlistExtension.sol`)

## Summary

`DepositAllowlistExtension.beforeAddLiquidity` silently discards its first parameter (`sender`, the actual token depositor) and gates on `owner` (the LP position recipient) instead. Because `owner` is a caller-controlled argument, any unprivileged address can bypass the allowlist in a single transaction by supplying any allowlisted address as `owner`. The pool accepts tokens from a non-allowlisted source, and the depositor permanently loses their principal to a position they cannot withdraw.

## Finding Description

`MetricOmmPool.addLiquidity` invokes `_beforeAddLiquidity(msg.sender, owner, ...)`, correctly passing the actual caller as `sender`: [1](#0-0) 

`ExtensionCalling._beforeAddLiquidity` encodes both arguments and forwards them to the extension, with `sender` as the first positional argument: [2](#0-1) 

`DepositAllowlistExtension.beforeAddLiquidity` declares the first parameter as an unnamed `address` (discarded) and checks only `owner`: [3](#0-2) 

The mapping name `allowedDepositor` and the NatDoc comment "Gates `addLiquidity` by depositor address" confirm the intended semantics are to restrict the token-providing caller, not the LP recipient: [4](#0-3) 

**Exploit path:**
1. Pool is configured with `DepositAllowlistExtension`; only `Alice` is in `allowedDepositor[pool]`.
2. `Bob` (not allowlisted) calls `pool.addLiquidity(owner = Alice, salt, deltas, callbackData, extensionData)`.
3. The extension evaluates `allowedDepositor[pool][Alice]` → `true` → no revert.
4. The pool executes the liquidity callback against `Bob` (`msg.sender`), pulling Bob's tokens.
5. LP shares are credited to `Alice`; Bob's tokens are inside the pool.

Since `removeLiquidity` enforces `msg.sender == owner` (line 206 of `MetricOmmPool.sol`), Bob cannot recover the position. Alice receives an unsolicited LP position she did not fund. [5](#0-4) 

## Impact Explanation

The deposit allowlist — an admin-configured access-control guard — is fully bypassed by any unprivileged address. The pool accepts tokens from non-allowlisted sources, breaking the core invariant the pool admin intended to enforce. The non-allowlisted depositor suffers direct, unrecoverable loss of principal: their tokens are transferred into the pool and the resulting LP position is credited to a third party they do not control. This constitutes broken core pool access-control with direct loss of user principal, meeting Critical/High severity thresholds under Sherlock contest rules.

## Likelihood Explanation

The bypass requires no special privilege, no flash loan, and no multi-step setup. Any EOA or contract can execute it in a single transaction. The only precondition is knowing at least one allowlisted address, which is trivially enumerable from the public `allowedDepositor` mapping on-chain. The attack is repeatable indefinitely.

## Recommendation

Name the first parameter `sender` and check it instead of `owner`:

```solidity
// Fixed:
function beforeAddLiquidity(address sender, address, uint80, LiquidityDelta calldata, bytes calldata)
    external view override returns (bytes4)
{
    if (!allowAllDepositors[msg.sender] && !allowedDepositor[msg.sender][sender]) {
        revert IMetricOmmPoolActions.NotAllowedToDeposit();
    }
    return IMetricOmmExtensions.beforeAddLiquidity.selector;
}
```

```solidity

pragma solidity ^0.8.35;

contract BypassDepositAllowlist {
    IMetricOmmPool pool;
    IERC20 token0;
    IERC20 token1;
    address alice; // allowlisted in allowedDepositor[pool]

    function exploit() external {
        // Bob (msg.sender) is NOT allowlisted.
        // Extension checks allowedDepositor[pool][alice] == true → passes.
        // Pool callback pulls tokens from Bob; Alice receives LP shares.
        LiquidityDelta memory delta = /* valid delta */;
        pool.addLiquidity(
            alice,   // owner — allowlisted, passes the guard
            0,       // salt
            delta,
            abi.encode(token0, token1),
            ""
        );
        // Bob has lost tokens; Alice holds the LP position; Bob cannot withdraw.
    }

    function metricOmmAddLiquidityCallback(uint256 amount0, uint256 amount1, bytes calldata) external {
        token0.transfer(msg.sender, amount0);
        token1.transfer(msg.sender, amount1);
    }
}
```
