### Title
Events Silently Dropped Across Protobuf Serialization Boundary in P2P Sync — (`File: crates/apollo_protobuf/src/converters/receipt.rs`)

### Summary

Every `TryFrom<protobuf::receipt::*>` implementation for all five transaction output types (`Invoke`, `Declare`, `Deploy`, `DeployAccount`, `L1Handler`) unconditionally sets `events = vec![]`. The corresponding `From<*TransactionOutput> for protobuf::receipt::*` serializers never write events into the protobuf message either. Because the P2P sync client assembles `BlockBody` from these deserialized outputs and writes them directly to storage, the events table is populated with zero entries for every block synced over P2P. Any subsequent `starknet_getTransactionReceipt` or `starknet_getEvents` RPC call against a P2P-synced node returns an authoritative-looking but completely empty events list.

### Finding Description

**Serialization boundary — drop on both sides.**

In `crates/apollo_protobuf/src/converters/receipt.rs`, every deserialization path hardcodes `events = vec![]`: [1](#0-0) [2](#0-1) [3](#0-2) [4](#0-3) [5](#0-4) 

The serialization direction (`From<*TransactionOutput> for protobuf::receipt::*`) never encodes events into the `Common` message either — `create_proto_receipt_common_from_txn_output_fields` only carries `actual_fee`, `messages_sent`, `execution_resources`, and `revert_reason`: [6](#0-5) 

**P2P sync client — no event protocol, no recovery.**

`P2pSyncClientChannels` accepts only `header_sender`, `state_diff_sender`, `transaction_sender`, and `class_sender` — there is no event sender: [7](#0-6) 

The transaction stream factory assembles a `BlockBody` directly from the deserialized `FullTransaction` objects (which carry the empty-event `transaction_output`) and writes it to storage: [8](#0-7) 

A developer comment on line 48 acknowledges the gap: `// TODO(Eitan): Add events protocol to BlockBody or split their write to storage`.

**Storage write — events derived solely from `tx_output.events()`.**

`append_body` calls `write_events`, which iterates `tx_output.events()` to populate the events index table. With `events = vec![]`, nothing is written: [9](#0-8) [10](#0-9) 

**RPC layer — reads from the now-empty storage.**

`get_transaction_receipt` reads the stored `TransactionOutput` and returns its `events` field directly. For P2P-synced blocks this is always `[]`: [11](#0-10) 

### Impact Explanation

Any node that syncs blocks via P2P (rather than central/feeder-gateway sync) will store zero events for every transaction in every synced block. All subsequent RPC calls — `starknet_getTransactionReceipt`, `starknet_getEvents` — return an authoritative-looking response with an empty `events` array. Clients reconstructing token transfer history, monitoring contract events, or verifying event commitments against the block header will observe a permanent, silent divergence from the canonical chain state.

This matches the allowed impact: **High — RPC returns an authoritative-looking wrong value.**

### Likelihood Explanation

Any node operator running P2P sync (the default distributed topology) is affected for every block received over the network. The bug is structural — it is present in every protobuf round-trip — and requires no special transaction type or attacker action to trigger. A normal user querying events on a P2P-synced node is sufficient.

### Recommendation

1. Add an `events` field to the protobuf `Receipt` `Common` message (or per-type messages) and populate it in both the `From` (serialization) and `TryFrom` (deserialization) directions.
2. Alternatively, implement the separate event protocol on the P2P sync client side (resolving the `TODO(Eitan)`) and reconstruct events from that channel before calling `append_body`.
3. Add a round-trip test asserting that `TransactionOutput` with non-empty events survives a protobuf encode/decode cycle with events intact.

### Proof of Concept

1. Start a node configured for P2P sync.
2. Let it sync any block containing transactions that emit events (e.g., any ERC-20 transfer).
3. Call `starknet_getTransactionReceipt` for any such transaction hash.
4. Observe `"events": []` in the response despite the transaction having emitted events on the canonical chain.
5. Call `starknet_getEvents` with a filter covering that block range.
6. Observe an empty result set.

The root cause is confirmed by the explicit `events = vec![]` literals at lines 79, 127, 174, 202, and 230 of `crates/apollo_protobuf/src/converters/receipt.rs`, combined with the absence of any event-recovery path in `crates/apollo_p2p_sync/src/client/transaction.rs`.

fn create_proto_receipt_common_from_txn_output_fields(
    actual_fee: Fee,
    messages_sent: Vec<MessageToL1>,
    execution_resources: ExecutionResources,
    execution_status: TransactionExecutionStatus,
) -> protobuf::receipt::Common {
    let actual_fee = Felt::from(actual_fee).into();
    let messages_sent = messages_sent.into_iter().map(protobuf::MessageToL1::from).collect();
    let execution_resources = execution_resources.into();
    let revert_reason =
        if let TransactionExecutionStatus::Reverted(reverted_status) = execution_status {
            Some(reverted_status.revert_reason)
        } else {
            None
        };
    protobuf::receipt::Common {
        actual_fee: Some(actual_fee),
        price_unit: 0,
        messages_sent,
        execution_resources: Some(execution_resources),
        revert_reason,
    }
}

pub(crate) struct TransactionStreamFactory;

impl BlockDataStreamBuilder<FullTransaction> for TransactionStreamFactory {
    // TODO(Eitan): Add events protocol to BlockBody or split their write to storage
    type Output = (BlockBody, BlockNumber);

    const TYPE_DESCRIPTION: &'static str = "transactions";
    const BLOCK_NUMBER_LIMIT: BlockNumberLimit = BlockNumberLimit::HeaderMarker;

    fn parse_data_for_block<'a>(
        transactions_response_manager: &'a mut ClientResponsesManager<DataOrFin<FullTransaction>>,
        block_number: BlockNumber,
        storage_reader: &'a StorageReader,
    ) -> BoxFuture<'a, Result<Option<Self::Output>, ParseDataError>> {
        async move {
            let mut block_body = BlockBody::default();
            let mut current_transaction_len = 0;
            let target_transaction_len = storage_reader
                .begin_ro_txn()?
                .get_block_header(block_number)?
                .expect("A header with number lower than the header marker is missing")
                .n_transactions;
            while current_transaction_len < target_transaction_len {
                let maybe_transaction = transactions_response_manager.next().await.ok_or(
                    ParseDataError::BadPeer(BadPeerError::SessionEndedWithoutFin {
                        type_description: Self::TYPE_DESCRIPTION,
                    }),
                )?;
                let Some(FullTransaction { transaction, transaction_output, transaction_hash }) =
                    maybe_transaction?.0
                else {
                    if current_transaction_len == 0 {
                        return Ok(None);
                    } else {
                        return Err(ParseDataError::BadPeer(BadPeerError::NotEnoughTransactions {
                            expected: target_transaction_len,
                            actual: current_transaction_len,
                            block_number: block_number.0,
                        }));
                    }
                };
                block_body.transactions.push(transaction);
                block_body.transaction_outputs.push(transaction_output);
                // TODO(eitan): Validate transaction hash from untrusted sources
                block_body.transaction_hashes.push(transaction_hash);
                current_transaction_len += 1;
            }
            Ok(Some((block_body, block_number)))
        }
        .boxed()
    }

fn write_events<'env>(
    tx_output: &TransactionOutput,
    txn: &DbTransaction<'env, RW>,
    events_table: &'env EventsTable<'env>,
    transaction_index: TransactionIndex,
) -> StorageResult<()> {
    let mut contract_addresses_set = HashSet::new();

    for event in tx_output.events().iter() {
        contract_addresses_set.insert(event.from_address);
    }

    for contract_address in contract_addresses_set {
        let key = (contract_address, transaction_index);
        // Here, we use the function assumption; the append will fail if an older transaction_index
        // is a table.
        events_table.append_greater_sub_key(txn, &key, &NoValue)?;
    }
    Ok(())
}
