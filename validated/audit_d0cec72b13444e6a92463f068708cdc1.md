### Title
Absolute Off-Chain NLP Oracle Price in `MintNlp`/`BurnNlp` Enables Stale-Price Value Extraction from the NLP Pool - (File: `core/contracts/Clearinghouse.sol`, `core/contracts/EndpointTx.sol`)

---

### Summary

The NLP token price (`oraclePriceX18`) is supplied as an **absolute value** by the off-chain sequencer in every `MintNlp` and `BurnNlp` transaction. This price is computed off-chain and embedded in the transaction without any on-chain staleness check. When the sequencer batches transactions that include both trades (which alter NLP pool value) and NLP mint/burn operations, the NLP price used in those operations is stale relative to the pool-value changes caused by trades processed earlier in the same batch. This allows users to mint NLP below the true current price or burn NLP above the true current price, extracting value from existing NLP holders.

---

### Finding Description

In `EndpointTx.sol`, both the `MintNlp` and `BurnNlp` handlers unconditionally accept an `oraclePriceX18` field that the sequencer computed off-chain and embedded as an absolute value in the signed transaction: [1](#0-0) [2](#0-1) 

This absolute price is forwarded directly to `Clearinghouse.mintNlp` / `Clearinghouse.burnNlp`, where it is used to compute the NLP token amount the user receives (mint) or the quote amount the user receives (burn): [3](#0-2) [4](#0-3) 

The NLP price represents `totalNlpPoolValue / totalNlpSupply`. This value changes continuously as trades are matched and PnL is settled across the NLP pool subaccounts. The sequencer computes this price off-chain at some point in time and embeds it as an absolute snapshot in the transaction. There is **no on-chain mechanism** that verifies the price is still accurate at the moment of execution.

When the sequencer constructs a batch that contains both trade-matching transactions (which change the NLP pool's quote balance and therefore its value) and `MintNlp`/`BurnNlp` transactions, the NLP price in the latter was computed before the former executed. The on-chain contract applies the stale price without correction.

The `manualAssert` mechanism checks that spot/perp states match the sequencer's off-chain view, but it does not assert that the NLP price used in any given `MintNlp`/`BurnNlp` was accurate at execution time: [5](#0-4) 

---

### Impact Explanation

**Impact: High**

- A user whose `MintNlp` is processed after trades that increased the NLP pool value receives `quoteAmount / P_stale` NLP tokens, where `P_stale < P_actual`. They immediately hold NLP worth more than they paid.
- A user whose `BurnNlp` is processed after trades that decreased the NLP pool value receives `nlpAmount × P_stale` quote, where `P_stale > P_actual`. They receive more quote than the NLP is currently worth.
- In both cases, the surplus is extracted from the NLP pool, diluting the value held by all other NLP holders.
- The discrepancy compounds: each stale-price mint/burn widens the gap between the sequencer's off-chain NLP price model and the on-chain reality, exactly mirroring the AUM drift described in the reference report.

---

### Likelihood Explanation

**Likelihood: High**

- The sequencer routinely batches trade-matching transactions alongside NLP mint/burn operations. Any batch that interleaves trades with NLP operations creates the condition.
- A user who observes (or can predict) that a batch will contain profitable trades for the NLP pool before their `MintNlp` can time their request to receive a below-market price.
- No special privilege is required: any user with a registered subaccount can submit `MintNlp`/`BurnNlp` requests.
- The attack is repeatable on every batch that contains both trade and NLP operations.

---

### Recommendation

Replace the absolute off-chain price with an **on-chain computed NLP price** derived from the current state of the NLP pool subaccounts at execution time. Concretely:

1. At `mintNlp`/`burnNlp` execution, compute `nlpPrice = totalNlpPoolQuoteValue / totalNlpSupply` directly from the `SpotEngine` state, rather than accepting a sequencer-supplied absolute value.
2. If an off-chain price hint is still needed for gas efficiency, add a **maximum deviation check**: require `|oraclePriceX18 - onChainComputedPrice| / onChainComputedPrice ≤ maxDeviationBps`, and revert if the supplied price is stale beyond the tolerance.
3. Alternatively, ensure the sequencer always recomputes the NLP price **after** all state-changing transactions in a batch have been applied, and never reuses a price computed before earlier batch entries execute.

---

### Proof of Concept

1. NLP pool has total value `V = 1,000,000 USDC`, total NLP supply `S = 1,000,000`, so `P = 1.00 USDC/NLP`.
2. Sequencer computes `oraclePriceX18 = 1.00` and prepares a batch:
   - `[MatchOrders(trade that nets +50,000 USDC profit to NLP pool), MintNlp(sender=attacker, quoteAmount=100,000, oraclePriceX18=1.00)]`
3. `MatchOrders` executes first; NLP pool value becomes `1,050,000 USDC`. True NLP price is now `1.05 USDC/NLP`.
4. `MintNlp` executes with stale `oraclePriceX18 = 1.00`:
   - `nlpAmount = 100,000 / 1.00 = 100,000 NLP` (attacker receives)
   - Attacker paid `100,000 USDC` for NLP now worth `100,000 × 1.05 = 105,000 USDC`.
5. Attacker calls `BurnNlp` at the updated price `1.05`, receiving `≈ 104,895 USDC` (after burn fee), netting `≈ 4,895 USDC` profit extracted from existing NLP holders.
6. The attack is repeatable on every batch containing profitable trades followed by `MintNlp`, or loss-making trades followed by `BurnNlp`. [6](#0-5) [7](#0-6) [8](#0-7)

    function mintNlp(
        IEndpoint.MintNlp calldata txn,
        int128 oraclePriceX18,
        IEndpoint.NlpPool[] calldata nlpPools,
        int128[] calldata nlpPoolRebalanceX18
    ) external onlyEndpoint {
        require(!RiskHelper.isIsolatedSubaccount(txn.sender), ERR_UNAUTHORIZED);

        ISpotEngine spotEngine = _spotEngine();
        spotEngine.updatePrice(NLP_PRODUCT_ID, oraclePriceX18);

        require(txn.quoteAmount <= INT128_MAX, ERR_CONVERSION_OVERFLOW);
        int128 quoteAmount = int128(txn.quoteAmount);
        int128 nlpAmount = quoteAmount.div(oraclePriceX18);

        _validateNlpRebalance(nlpPools, nlpPoolRebalanceX18, quoteAmount);
        for (uint128 i = 0; i < nlpPoolRebalanceX18.length; i++) {
            require(nlpPoolRebalanceX18[i] >= 0, ERR_INVALID_NLP_REBALANCE);
        }

        spotEngine.updateBalance(NLP_PRODUCT_ID, txn.sender, nlpAmount);
        spotEngine.updateBalance(NLP_PRODUCT_ID, N_ACCOUNT, -nlpAmount);

        spotEngine.updateBalance(QUOTE_PRODUCT_ID, txn.sender, -quoteAmount);
        _applyNlpRebalance(spotEngine, nlpPools, nlpPoolRebalanceX18);

        require(
            getHealth(txn.sender, IProductEngine.HealthType.INITIAL) >= 0,
            ERR_SUBACCT_HEALTH
        );
    }

    function burnNlp(
        IEndpoint.BurnNlp calldata txn,
        int128 oraclePriceX18,
        IEndpoint.NlpPool[] calldata nlpPools,
        int128[] calldata nlpPoolRebalanceX18
    ) external onlyEndpoint {
        require(!RiskHelper.isIsolatedSubaccount(txn.sender), ERR_UNAUTHORIZED);

        ISpotEngine spotEngine = _spotEngine();
        spotEngine.updatePrice(NLP_PRODUCT_ID, oraclePriceX18);

        require(txn.nlpAmount <= INT128_MAX, ERR_CONVERSION_OVERFLOW);
        int128 nlpAmount = int128(txn.nlpAmount);
        require(
            spotEngine.getNlpUnlockedBalance(txn.sender).amount >= nlpAmount,
            ERR_UNLOCKED_NLP_INSUFFICIENT
        );
        int128 quoteAmount = nlpAmount.mul(oraclePriceX18);
        int128 burnFee = MathHelper.max(ONE, quoteAmount / 1000);
        quoteAmount = MathHelper.max(0, quoteAmount - burnFee);

        _validateNlpRebalance(nlpPools, nlpPoolRebalanceX18, -quoteAmount);
        for (uint128 i = 0; i < nlpPoolRebalanceX18.length; i++) {
            require(nlpPoolRebalanceX18[i] <= 0, ERR_INVALID_NLP_REBALANCE);
        }

        spotEngine.updateBalance(NLP_PRODUCT_ID, txn.sender, -nlpAmount);
        spotEngine.updateBalance(NLP_PRODUCT_ID, N_ACCOUNT, nlpAmount);

        if (quoteAmount > 0) {
            spotEngine.updateBalance(QUOTE_PRODUCT_ID, txn.sender, quoteAmount);
            _applyNlpRebalance(spotEngine, nlpPools, nlpPoolRebalanceX18);
        }

        require(
            spotEngine.getBalance(NLP_PRODUCT_ID, txn.sender).amount >= 0,
            ERR_SUBACCT_HEALTH
        );
        // Burning NLP can decrease health if the burn fee exceeds the health improvement
        // from the withdrawal. This check prevents malicious actors from deliberately
        // creating unhealthy subaccounts through NLP burns.
        require(
            getHealth(txn.sender, IProductEngine.HealthType.MAINTENANCE) >= 0,
            ERR_SUBACCT_HEALTH
        );
    }
