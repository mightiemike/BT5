### Title
NFT Minted to Non-Creator Recipient Becomes Permanently Unburnable by Either Party — (`x/nft/keeper/keeper.go`)

### Summary

`keeper.BurnNFT` enforces a conjunctive guard requiring the caller to simultaneously be both the NFT owner (`IsOwner`) **and** the denom creator (`IsDenomCreator`). Because `MsgMintNFT` explicitly supports minting to a `Recipient` different from the `Sender` (denom creator), any NFT minted to a non-creator address enters a state where no single account can satisfy both conditions, making the NFT permanently unburnable without voluntary cooperation from the recipient.

---

### Finding Description

`keeper.BurnNFT` at lines 141–161 enforces two sequential guards on the same `owner` address:

```go
nft, err := k.IsOwner(ctx, denomID, tokenID, owner)   // must be NFT owner
if err != nil { return err }

_, err = k.IsDenomCreator(ctx, denomID, owner)         // must also be denom creator
if err != nil { return err }
``` [1](#0-0) 

`MsgMintNFT` in `msg_server.go` passes `sender` (the denom creator) as the authorization check and `recipient` as the NFT owner, which are explicitly allowed to differ:

```go
if err := m.Keeper.MintNFT(ctx, msg.DenomId, msg.Id,
    msg.Name, msg.URI, msg.Data,
    sender,     // denom creator check
    recipient,  // NFT owner stored on-chain
); err != nil { ... }
``` [2](#0-1) 

`keeper.MintNFT` confirms this split: it validates `sender` is the denom creator, then calls `MintNFTUnverified` with `owner = recipient`: [3](#0-2) 

**Resulting deadlock:**

| Actor | `IsOwner` | `IsDenomCreator` | Can burn? |
|---|---|---|---|
| accountB (recipient/owner) | ✓ | ✗ | No |
| accountA (denom creator) | ✗ | ✓ | No |

Neither party can satisfy both guards simultaneously. The only escape is accountB voluntarily calling `MsgTransferNFT` back to accountA, which accountA cannot compel.

The existing `TestBurnNFT` test acknowledges this behavior — it explicitly transfers the NFT back to the creator before burning — but does not cover the case where the creator mints directly to a non-creator recipient: [4](#0-3) 

---

### Impact Explanation

- The denom creator permanently loses administrative burn authority over any NFT minted to a non-creator address, unless the recipient cooperates.
- The NFT supply counter (`decreaseSupply`) can never be decremented for such tokens, breaking the collection's supply invariant.
- The denom creator cannot reclaim or destroy tokens they issued, undermining the expected authority model of the NFT module.

---

### Likelihood Explanation

This is reachable via a standard, fully supported production transaction path: `MsgMintNFT` with `Recipient != Sender`. The spec explicitly documents this field (`Recipient`: "The recipient of the new NFT"). Any denom creator who mints to a third-party address — the primary use case for an NFT marketplace or airdrop — triggers this condition. [5](#0-4) 

---

### Recommendation

Replace the conjunctive `IsOwner AND IsDenomCreator` requirement in `BurnNFT` with a disjunctive check: allow burn if the caller is **either** the NFT owner **or** the denom creator. Alternatively, restrict `MintNFT` so that `Recipient` must equal `Sender` (the denom creator), requiring an explicit transfer afterward. The `BurnNFTUnverified` function already demonstrates the correct pattern for owner-only burns (used for IBC): [6](#0-5) 

---

### Proof of Concept

```go
// 1. accountA issues denom
err := keeper.IssueDenom(ctx, denomID, "testdenom", "", "", accountA)
require.NoError(t, err)

// 2. accountA mints tokenX to accountB (non-creator recipient)
err = keeper.MintNFT(ctx, denomID, tokenID, "tok", "uri", "data", accountA, accountB)
require.NoError(t, err)

// 3. accountB (owner) cannot burn — not denom creator
err = keeper.BurnNFT(ctx, denomID, tokenID, accountB)
require.ErrorContains(t, err, "ErrUnauthorized") // fails IsDenomCreator

// 4. accountA (creator) cannot burn — not NFT owner
err = keeper.BurnNFT(ctx, denomID, tokenID, accountA)
require.ErrorContains(t, err, "ErrUnauthorized") // fails IsOwner

// 5. tokenX still exists; supply unchanged
require.True(t, keeper.HasNFT(ctx, denomID, tokenID))
require.Equal(t, uint64(1), keeper.GetTotalSupply(ctx, denomID))
```

// BurnNFTUnverified deletes a specified NFT without verifying if the owner is the creator of denom
// Needed for IBC transfer of NFT
func (k Keeper) BurnNFTUnverified(ctx sdk.Context, denomID, tokenID string, owner sdk.AccAddress) error {
	if !k.HasDenomID(ctx, denomID) {
		return sdkerrors.Wrapf(types.ErrInvalidDenom, "denom ID %s not exists", denomID)
	}

	nft, err := k.IsOwner(ctx, denomID, tokenID, owner)
	if err != nil {
		return err
	}

	k.deleteNFT(ctx, denomID, nft)
	k.deleteOwner(ctx, denomID, tokenID, owner)
	k.decreaseSupply(ctx, denomID)

	return nil
