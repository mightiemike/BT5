### Title
Sanctioned User's Airdrop Allocation Permanently Locked Due to Claim-Time Sanction Check Without Recovery Mechanism — (File: `core/contracts/Airdrop.sol`)

---

### Summary

`Airdrop._verifyProof()` evaluates `isSanctioned(sender)` at **claim time**. Once a Merkle root is registered, a user's allocation is committed on-chain. If the user is subsequently added to the sanctions list before claiming, their tokens are permanently locked — the contract has no recovery function.

---

### Finding Description

`Airdrop.registerMerkleRoot()` commits a Merkle root on-chain, fixing each included user's allocation as a protocol obligation. [1](#0-0) 

When a user later calls `claim()`, execution reaches `_verifyProof()`, which checks the **current** sanctions status: [2](#0-1) 

This check uses the live state of the external `ISanctionsList` contract at the moment of the call — not the state at the time the Merkle root was registered. If the user's address is added to the sanctions list in the window between Merkle root registration and their claim, the call reverts unconditionally.

The entire `Airdrop.sol` contract exposes no `withdraw`, `recoverTokens`, or equivalent owner-callable function. The only state-changing functions are `registerMerkleRoot` (owner) and `claim` (user). There is no path by which the protocol can retrieve tokens allocated to a now-sanctioned user. [3](#0-2) 

---

### Impact Explanation

Airdrop tokens allocated to users who become sanctioned after Merkle root registration are permanently locked in the `Airdrop` contract. Neither the user nor the protocol owner can recover them. The exact corrupted asset delta is the full `totalAmount` encoded in the Merkle leaf for each affected user, held indefinitely at the `Airdrop` contract address with no exit path. [4](#0-3) 

---

### Likelihood Explanation

Sanctions lists (e.g., Chainalysis OFAC) are updated continuously and independently of the Nado protocol. The protocol registers Merkle roots on a weekly cadence (`pastWeeks` counter). The window between root registration and a user's claim can span days to weeks. It is a realistic and non-negligible scenario that a user earns rewards in one epoch, the root is registered, and the user is subsequently sanctioned before claiming. No attacker action is required — the trigger is a routine external sanctions list update. [5](#0-4) 

---

### Recommendation

Add an owner-callable recovery function to `Airdrop.sol` that allows retrieval of tokens allocated to sanctioned users (e.g., by marking the allocation as forfeited and transferring the tokens to a protocol treasury). Alternatively, move the sanctions check out of `_verifyProof` and into a separate gate that does not permanently forfeit the allocation — for example, by redirecting the claim to a protocol-controlled address rather than reverting.

---

### Proof of Concept

1. Owner calls `registerMerkleRoot(5, root)` — the root encodes Alice's address with `totalAmount = 1000e18`.
2. Alice's address is added to the Chainalysis sanctions list (external action, no Nado admin involvement).
3. Alice calls `claim([{week: 5, totalAmount: 1000e18, proof: [...]}])`.
4. `_verifyProof` executes: `ISanctionsList(sanctions).isSanctioned(Alice)` returns `true` → reverts with `"address is sanctioned."`.
5. `claimed[5][Alice]` remains `0` (the allocation was never marked consumed).
6. Alice cannot retry (still sanctioned). The owner has no function to recover the 1000e18 tokens. They remain locked in the `Airdrop` contract indefinitely. [6](#0-5)

```text
    function _verifyProof(
        uint32 week,
        address sender,
        uint256 totalAmount,
        bytes32[] calldata proof
    ) internal {
        require(claimed[week][sender] == 0, "Already claimed.");
        require(
            merkleRoots[week] != bytes32(0),
            "Week hasn't been registered."
        );
        require(
            !ISanctionsList(sanctions).isSanctioned(sender),
            "address is sanctioned."
        );
        bytes32 leaf = keccak256(
            bytes.concat(keccak256(abi.encode(sender, totalAmount)))
        );
        bool isValidLeaf = MerkleProof.verify(proof, merkleRoots[week], leaf);
        require(isValidLeaf, "Invalid proof.");
        claimed[week][sender] = totalAmount;
    }
```

```text
        _verifyProof(week, msg.sender, totalAmount, proof);

    function claim(ClaimProof[] calldata claimProofs) external {
        for (uint32 i = 0; i < claimProofs.length; i++) {
            _claim(
                claimProofs[i].week,
                claimProofs[i].totalAmount,
                claimProofs[i].proof

        }
```
