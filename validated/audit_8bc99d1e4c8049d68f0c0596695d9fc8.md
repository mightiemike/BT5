Audit Report

## Title
`DepositAllowlistExtension.beforeAddLiquidity` checks `owner` instead of `sender`, allowing any unprivileged caller to bypass the depositor allowlist — (`metric-periphery/contracts/extensions/DepositAllowlistExtension.sol`)

## Summary
`DepositAllowlistExtension` is intended to gate which addresses may deposit capital into a pool. Its `beforeAddLiquidity` hook receives `sender` (the actual `msg.sender` of `addLiquidity`, i.e., the capital provider) as its first argument but silently discards it via an unnamed parameter, checking only `owner` (the LP share recipient). Because `owner` is a free caller-controlled parameter on `addLiquidity` with no access restriction, any non-allowlisted address can pass an allowlisted address as `owner`, satisfy the check, and inject their own capital into the pool.

## Finding Description
`MetricOmmPool.addLiquidity` accepts a free `owner` parameter and passes `msg.sender` as `sender` to the hook:

```solidity
// metric-core/contracts/MetricOmmPool.sol L191
_beforeAddLiquidity(msg.sender, owner, salt, deltas, extensionData);
``` [1](#0-0) 

`ExtensionCalling._beforeAddLiquidity` faithfully forwards both `sender` and `owner` to the extension:

```solidity
// metric-core/contracts/ExtensionCalling.sol L95-98
abi.encodeCall(IMetricOmmExtensions.beforeAddLiquidity, (sender, owner, salt, deltas, extensionData))
``` [2](#0-1) 

`DepositAllowlistExtension.beforeAddLiquidity` then silently drops `sender` (unnamed first parameter) and gates only on `owner`:

```solidity
// metric-periphery/contracts/extensions/DepositAllowlistExtension.sol L32-42
function beforeAddLiquidity(address, address owner, uint80, LiquidityDelta calldata, bytes calldata)
    external view override returns (bytes4)
{
    if (!allowAllDepositors[msg.sender] && !allowedDepositor[msg.sender][owner]) {
        revert IMetricOmmPoolActions.NotAllowedToDeposit();
    }
    return IMetricOmmExtensions.beforeAddLiquidity.selector;
}
``` [3](#0-2) 

An attacker who is not in `allowedDepositor[pool]` calls `pool.addLiquidity(owner=allowlistedAddr, ...)`. The check evaluates `allowedDepositor[pool][allowlistedAddr]` → `true`. The hook returns success. The pool then invokes `metricOmmModifyLiquidityCallback` on the attacker (the real `msg.sender`), pulling the attacker's tokens. LP shares are minted to `allowlistedAddr`. The allowlist gate is completely bypassed.

## Impact Explanation
The depositor allowlist — the sole mechanism for pool curation — is rendered ineffective. Any address can deposit capital into a restricted pool by nominating an allowlisted address as `owner`. Unauthorized capital enters the pool on every such call, breaking the admin-configured invariant that only approved depositors may provide liquidity. This constitutes a broken admin-boundary under the contest rules: an unprivileged path bypasses a pool admin access control.

## Likelihood Explanation
The attack requires only a direct call to `pool.addLiquidity` with a known allowlisted address as `owner`. No privileged access, no special token behavior, and no off-chain data are needed. Any allowlisted address is publicly discoverable via `AllowedToDepositSet` events or `allowedDepositor` view calls. The attacker bypasses the periphery `MetricOmmPoolLiquidityAdder` entirely and calls the pool directly. Likelihood is high.

## Recommendation
Check `sender` (the actual capital provider) instead of `owner` (the share recipient):

```solidity
function beforeAddLiquidity(address sender, address, uint80, LiquidityDelta calldata, bytes calldata)
    external view override returns (bytes4)
{
    if (!allowAllDepositors[msg.sender] && !allowedDepositor[msg.sender][sender]) {
        revert IMetricOmmPoolActions.NotAllowedToDeposit();
    }
    return IMetricOmmExtensions.beforeAddLiquidity.selector;
}
```

## Proof of Concept
```solidity
function test_allowlistBypass() public {
    // Setup: pool with DepositAllowlistExtension; only `allowlisted` is approved
    address allowlisted = makeAddr("allowlisted");
    extension.setAllowedToDeposit(address(pool), allowlisted, true);

    address attacker = makeAddr("attacker");
    // attacker is NOT in allowedDepositor

    deal(token0, attacker, 1_000e18);
    deal(token1, attacker, 1_000e18);
    vm.startPrank(attacker);
    IERC20(token0).approve(address(attackerCallback), type(uint256).max);
    IERC20(token1).approve(address(attackerCallback), type(uint256).max);

    // Attacker calls pool directly with owner=allowlisted
    // attackerCallback implements metricOmmModifyLiquidityCallback and pays from attacker
    pool.addLiquidity(allowlisted, salt, deltas, callbackData, "");
    vm.stopPrank();

    // allowlisted received LP shares; attacker's capital entered the pool without being allowlisted
    assertGt(positionBinShares(address(pool), allowlisted, salt, binIdx), 0);
    assertLt(IERC20(token0).balanceOf(attacker), 1_000e18);
    assertEq(positionBinShares(address(pool), attacker, salt, binIdx), 0);
}
```
