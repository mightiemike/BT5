### Title
Missing Per-Element TokenID Validation in IBC Packet Allows Empty-ID NFT Minting — (`x/nft-transfer/types/packet.go`, `x/nft-transfer/keeper/packet.go`, `x/nft/keeper/keeper.go`)

### Summary

`NonFungibleTokenPacketData.ValidateBasic` only checks that the `TokenIds` slice is non-empty and that its length matches `TokenUris`. It never validates that individual token IDs are non-blank. The downstream `processReceivedPacket` → `MintNFT` → `MintNFTUnverified` call chain also performs no per-element tokenID format check. A malicious IBC counterparty can therefore send a packet containing `TokenIds=["", "real"]` that passes all validation and causes the NFT keeper to store an NFT with `tokenID=""`.

---

### Finding Description

**`ValidateBasic` gap** — `x/nft-transfer/types/packet.go` lines 46–52:

```go
if len(nftpd.TokenIds) == 0 {
    return newsdkerrors.Wrap(ErrInvalidTokenID, "tokenId cannot be blank")
}
if len(nftpd.TokenIds) != len(nftpd.TokenUris) {
    return newsdkerrors.Wrap(ErrInvalidPacket, "tokenIds and tokenUris lengths do not match")
}
```

No loop iterates over individual elements to check `strings.TrimSpace(id) == ""`. A packet with `TokenIds=[""]` or `TokenIds=["", "real"]` passes both guards. [1](#0-0) 

**`OnRecvPacket` → `processReceivedPacket`** — `x/nft-transfer/keeper/relay.go` lines 104–110:

```go
if err := data.ValidateBasic(); err != nil {
    return err
}
return k.processReceivedPacket(ctx, packet, data)
```

After `ValidateBasic` passes, `processReceivedPacket` iterates directly over `data.TokenIds` with no further per-element check. [2](#0-1) 

**`processReceivedPacket` mints without tokenID validation** — `x/nft-transfer/keeper/packet.go` lines 181–184:

```go
for i, tokenID := range data.TokenIds {
    if err := k.nftKeeper.MintNFT(ctx, voucherClassID, tokenID, "", data.TokenUris[i], "", escrowAddress, receiver); err != nil {
        return err
    }
}
``` [3](#0-2) 

**`MintNFT` skips `ValidateTokenID`** — `x/nft/keeper/keeper.go` lines 72–82:

```go
func (k Keeper) MintNFT(...) error {
    _, err := k.IsDenomCreator(ctx, denomID, sender)
    if err != nil { return err }
    return k.MintNFTUnverified(ctx, denomID, tokenID, ...)
}
```

`IsDenomCreator` passes because the IBC path creates the denom with `escrowAddress` as creator and calls `MintNFT` with `escrowAddress` as sender. `ValidateTokenID` is never called. [4](#0-3) 

**`MintNFTUnverified` stores without format validation** — `x/nft/keeper/keeper.go` lines 44–69:

```go
func (k Keeper) MintNFTUnverified(...) error {
    if !k.HasDenomID(ctx, denomID) { ... }
    if k.HasNFT(ctx, denomID, tokenID) { ... }
    k.setNFT(ctx, denomID, types.NewBaseNFT(tokenID, ...))
    k.setOwner(ctx, denomID, tokenID, owner)
    k.increaseSupply(ctx, denomID)
    return nil
}
```

No call to `ValidateTokenID`. An empty string `""` has length 0, which is below `MinDenomLen = 3` and fails the alphanumeric regex — but `ValidateTokenID` is only invoked in `MsgMintNFT.ValidateBasic()` (the direct Cosmos SDK transaction path) and `ValidateGenesis`, never in the IBC receive path. [5](#0-4) 

**`ValidateTokenID` exists but is unreachable from IBC path** — `x/nft/types/validation.go` lines 66–74:

```go
func ValidateTokenID(tokenID string) error {
    if len(tokenID) < MinDenomLen || len(tokenID) > MaxDenomLen { ... }
    if !IsBeginWithAlpha(tokenID) || !IsAlphaNumeric(tokenID) { ... }
    return nil
}
``` [6](#0-5) 

---

### Impact Explanation

A malicious IBC counterparty sends a packet with `TokenIds=[""]` (or `["", "real"]`) and matching-length `TokenUris`. The receiving chain:

1. Passes `ValidateBasic` (slice non-empty, lengths match).
2. Creates the IBC voucher denom with `escrowAddress` as creator.
3. Calls `MintNFT(voucherClassID, "", ..., escrowAddress, receiver)`.
4. `IsDenomCreator` passes; `MintNFTUnverified` stores `BaseNFT{ID: ""}` in the KV store under `KeyNFT(voucherClassID, "")`.
5. `setOwner` records the receiver as owner of tokenID `""`.
6. Supply counter is incremented.

The result is an NFT with an empty ID stored in the keeper. The `GetNFTs` prefix iterator uses `KeyNFT(denom, "")` as its prefix scan key — the empty-ID NFT sits at exactly that boundary, potentially interfering with collection enumeration. The receiver holds an NFT that should never have been mintable under the chain's own rules (`ValidateTokenID` would have rejected it), constituting unauthorized voucher minting that bypasses the receiving chain's invariants.

---

### Likelihood Explanation

Any IBC counterparty that controls packet construction (e.g., a malicious or compromised chain, or a chain running custom relayer software) can craft this packet. No governance, key compromise, or privileged role is required — only the ability to send an IBC packet on an established channel. The path is fully reachable in production.

---

### Recommendation

Add a per-element tokenID validation loop inside `NonFungibleTokenPacketData.ValidateBasic`:

```go
for _, id := range nftpd.TokenIds {
    if strings.TrimSpace(id) == "" {
        return newsdkerrors.Wrap(ErrInvalidTokenID, "tokenId cannot be blank")
    }
}
```

Alternatively (or additionally), call `ValidateTokenID(tokenID)` inside `processReceivedPacket` before each `MintNFT` call, so the IBC receive path enforces the same tokenID format rules as the direct Cosmos SDK transaction path.

---

### Proof of Concept

```go
// Keeper unit test (no IBC stack needed)
func TestOnRecvPacketEmptyTokenID(t *testing.T) {
    // setup keeper, create IBC channel state
    packet := channeltypes.Packet{
        SourcePort: "nft-transfer", SourceChannel: "channel-0",
        DestPort:   "nft-transfer", DestChannel:   "channel-1",
    }
    data := types.NonFungibleTokenPacketData{
        ClassId:   "cryptoCat",
        ClassUri:  "uri",
        TokenIds:  []string{""},          // empty tokenID
        TokenUris: []string{""},
        Sender:    validBech32Addr,
        Receiver:  validBech32Addr,
    }
    // ValidateBasic must return error (currently it does not)
    err := data.ValidateBasic()
    require.Error(t, err, "empty tokenID should be rejected by ValidateBasic")
    // Without the fix, err == nil and OnRecvPacket proceeds to mint
}
```

// MintNFTUnverified mints an NFT without verifying if the owner is the creator of denom
// Needed during genesis initialization
func (k Keeper) MintNFTUnverified(ctx sdk.Context, denomID, tokenID, tokenNm, tokenURI, tokenData string, owner sdk.AccAddress) error {
	if !k.HasDenomID(ctx, denomID) {
		return sdkerrors.Wrapf(types.ErrInvalidDenom, "denom ID %s not exists", denomID)
	}

	if k.HasNFT(ctx, denomID, tokenID) {
		return sdkerrors.Wrapf(types.ErrNFTAlreadyExists, "NFT %s already exists in collection %s", tokenID, denomID)
	}

	k.setNFT(
		ctx, denomID,
		types.NewBaseNFT(
			tokenID,
			tokenNm,
			owner,
			tokenURI,
			tokenData,
		),
	)
	k.setOwner(ctx, denomID, tokenID, owner)
	k.increaseSupply(ctx, denomID)

	return nil
}
```

func ValidateTokenID(tokenID string) error {
	if len(tokenID) < MinDenomLen || len(tokenID) > MaxDenomLen {
		return sdkerrors.Wrapf(ErrInvalidTokenID, "the length of nft id(%s) only accepts value [%d, %d]", tokenID, MinDenomLen, MaxDenomLen)
	}
	if !IsBeginWithAlpha(tokenID) || !IsAlphaNumeric(tokenID) {
		return sdkerrors.Wrapf(ErrInvalidTokenID, "nft id(%s) only accepts lowercase alphanumeric characters, and begin with an english letter", tokenID)
	}
	return nil
}
