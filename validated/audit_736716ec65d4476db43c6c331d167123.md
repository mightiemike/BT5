### Title
SwapAllowlistExtension Gates the Router Address Instead of the End-User, Allowing Full Allowlist Bypass When the Router Is Allowlisted - (File: metric-periphery/contracts/extensions/SwapAllowlistExtension.sol)

### Summary

`SwapAllowlistExtension.beforeSwap` checks the `sender` argument, which the pool sets to `msg.sender` of the `swap()` call. When a user routes through `MetricOmmSimpleRouter`, `msg.sender` inside the pool is the **router contract**, not the end-user. If the pool admin allowlists the router address (the natural step to enable router-based swaps), every user on the network can bypass the allowlist by calling any router entry-point, because the extension sees only the router's address.

### Finding Description

**Call chain:**

1. User calls `MetricOmmSimpleRouter.exactInputSingle()` (or `exactInput`, `exactOutputSingle`, `exactOutput`).
2. The router calls `IMetricOmmPoolActions(pool).swap(recipient, ...)` — `msg.sender` inside the pool is now the **router**.
3. The pool calls `_beforeSwap(msg.sender, ...)`, forwarding the router's address as `sender`.
4. `SwapAllowlistExtension.beforeSwap` receives `sender = router` and checks `allowedSwapper[pool][router]`. [1](#0-0) [2](#0-1) [3](#0-2) 

The router never forwards the original caller's address to the pool; it simply calls `pool.swap()` directly, making itself `msg.sender`. [4](#0-3) 

**The broken invariant:** The allowlist is keyed on `allowedSwapper[pool][sender]` where `sender` is supposed to be the economically relevant actor (the user providing funds and receiving output). When the router is the intermediary, `sender` is the router — a single shared public contract — not the individual user.

**Trigger condition:** A pool admin who wants to allow router-based swaps for their allowlisted users will add the router address to `allowedSwapper[pool][router] = true`. This is the only way to make the router work with an allowlisted pool. Once done, the check `allowedSwapper[pool][router]` passes for **every** caller of the router, regardless of who they are. [5](#0-4) 

### Impact Explanation

Any user can swap on a pool that is supposed to be restricted to a specific set of allowlisted addresses (e.g., KYC-verified traders, institutional counterparties, or protocol-internal actors). The allowlist guard is completely nullified for all router-mediated swaps. LP providers who deposited into the restricted pool under the assumption that only vetted counterparties would trade against their liquidity are exposed to unrestricted public swap flow, which can drain their positions at oracle-derived prices they did not consent to trade against with arbitrary counterparties. This constitutes a broken core pool functionality and a direct loss of LP assets above Sherlock thresholds.

### Likelihood Explanation

The likelihood is high. The `MetricOmmSimpleRouter` is the primary user-facing entry point documented and promoted by the protocol. Any pool admin who configures a swap allowlist and also wants users to access the pool via the router will inevitably allowlist the router address — there is no other way to make the two features coexist. The bypass requires no special privileges, no flash loans, and no exotic token behavior: any EOA calls `exactInputSingle` on the router pointing at the restricted pool.

### Recommendation

The extension must gate the **end-user**, not the intermediary. Two complementary fixes:

1. **Pass the original payer through the router.** The router already stores the original `msg.sender` in transient storage as the payer (`_setNextCallbackContext(..., msg.sender, ...)`). Expose this as an additional argument to `pool.swap()`, or have the pool accept an explicit `swapper` override that the router populates. The pool then forwards the true user address as `sender` to the extension.

2. **Alternatively, check `recipient` instead of `sender` in the extension.** The `recipient` is the address that receives output tokens and is set by the end-user. However, this is also spoofable and is not the canonical identity for access control.

The cleanest fix is option 1: add a `swapper` field to the swap parameters that the router fills with `msg.sender` before calling the pool, and have the pool forward that field — not its own `msg.sender` — to extension hooks. [6](#0-5) 

### Proof of Concept

```
Setup:
  - Deploy pool with SwapAllowlistExtension
  - Pool admin calls setAllowedToSwap(pool, router, true)   // to enable router usage
  - Pool admin does NOT allowlist attacker address

Attack:
  - Attacker (non-allowlisted EOA) calls:
      router.exactInputSingle({
          pool: restrictedPool,
          recipient: attacker,
          zeroForOne: true,
          amountIn: X,
          ...
      })

Trace:
  router.exactInputSingle()
    → pool.swap(recipient, ...)          // msg.sender = router
      → _beforeSwap(router, ...)
        → extension.beforeSwap(router, ...)
          → allowedSwapper[pool][router] == true  ✓  (bypass succeeds)
      → swap executes, attacker receives output tokens
```

The attacker successfully swaps on a pool they are not allowlisted for, because the extension checked the router's address (allowlisted) rather than the attacker's address (not allowlisted). [7](#0-6) [8](#0-7) [4](#0-3)

contract SwapAllowlistExtension is BaseMetricExtension, ISwapAllowlistExtension {
  mapping(address pool => mapping(address swapper => bool)) public allowedSwapper;
  mapping(address pool => bool) public allowAllSwappers;

  constructor(address factory_) BaseMetricExtension(factory_) {}

  function setAllowedToSwap(address pool_, address swapper, bool allowed) external onlyPoolAdmin(pool_) {
    allowedSwapper[pool_][swapper] = allowed;
    emit AllowedToSwapSet(pool_, swapper, allowed);
  }

  function setAllowAllSwappers(address pool_, bool allowed) external onlyPoolAdmin(pool_) {
    allowAllSwappers[pool_] = allowed;
    emit AllowAllSwappersSet(pool_, allowed);
  }

  function isAllowedToSwap(address pool_, address swapper) external view returns (bool) {
    return allowAllSwappers[pool_] || allowedSwapper[pool_][swapper];
  }

  function beforeSwap(address sender, address, bool, int128, uint128, uint256, uint128, uint128, bytes calldata)
    external
    view
    override
    returns (bytes4)
  {
    if (!allowAllSwappers[msg.sender] && !allowedSwapper[msg.sender][sender]) {
      revert IMetricOmmPoolActions.NotAllowedToSwap();
    }
    return IMetricOmmExtensions.beforeSwap.selector;
  }
