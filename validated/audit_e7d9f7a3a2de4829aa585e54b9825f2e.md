Audit Report

## Title
SwapAllowlistExtension Checks Router Address Instead of Original User, Enabling Full Allowlist Bypass via MetricOmmSimpleRouter - (File: metric-periphery/contracts/extensions/SwapAllowlistExtension.sol)

## Summary
`SwapAllowlistExtension.beforeSwap` gates swaps by checking the `sender` parameter, which `MetricOmmPool.swap` sets to `msg.sender` — the immediate caller of the pool. When swaps are routed through `MetricOmmSimpleRouter`, `msg.sender` at the pool is the router contract, not the original user. A pool admin who allowlists the router address to permit router-mediated swaps inadvertently grants every address on the network the ability to bypass the allowlist entirely.

## Finding Description
`MetricOmmPool.swap` passes `msg.sender` as the `sender` argument to `_beforeSwap`: [1](#0-0) 

`SwapAllowlistExtension.beforeSwap` then evaluates `allowedSwapper[msg.sender][sender]`, where `msg.sender` is the pool and `sender` is whoever called `pool.swap`: [2](#0-1) 

`MetricOmmSimpleRouter.exactInputSingle` calls `pool.swap` directly, making the router the `msg.sender` at the pool: [3](#0-2) 

The same pattern applies to `exactInput`, `exactOutputSingle`, and `exactOutput`. When the router is allowlisted (the only way to permit allowlisted users to use the router), the check `allowedSwapper[pool][router] == true` passes for every call arriving via the router, regardless of who the original caller is. There is no mechanism in the extension to recover the original `msg.sender` from the router call. By contrast, `DepositAllowlistExtension.beforeAddLiquidity` correctly checks `owner` (the economic beneficiary) rather than `sender`, so the deposit path does not share this flaw: [4](#0-3) 

## Impact Explanation
A curated pool (KYC-gated, institutional, or RWA pool) deploying `SwapAllowlistExtension` loses all swap access control the moment the router is allowlisted. Any unprivileged address can execute swaps against the pool's liquidity by routing through `MetricOmmSimpleRouter`, draining LP value at oracle-anchored prices and bypassing the intended curation policy. This constitutes a direct loss of LP principal and a broken core pool invariant (swap access control), meeting the contest threshold for a High severity finding.

## Likelihood Explanation
The router is the primary user-facing entry point for swaps. Pool admins who want allowlisted users to be able to use the router — a completely reasonable and expected operational requirement — must allowlist the router address. There is no documented warning against this. The misconfiguration is a natural consequence of normal pool administration, not an exotic mistake, making exploitation highly likely once any curated pool is deployed with this extension.

## Recommendation
The extension must identify the original economic actor, not the intermediate caller. Two viable approaches:

1. **Extension-data convention**: Require the router to ABI-encode the original `msg.sender` into `extensionData` for every hop. The extension decodes and checks that address. The router already forwards `extensionData` per-hop unchanged, so this requires only a router-side encoding convention and an extension-side decoder.

2. **Separate router allowlist**: Maintain a second mapping `allowedRouter` and, when `sender` is a known router, decode the real user from `extensionData`. This preserves backward compatibility for direct pool calls.

Until fixed, pool admins should not allowlist the router address and should instruct allowlisted users to call `pool.swap` directly.

## Proof of Concept
```
Setup:
  pool configured with SwapAllowlistExtension
  allowedSwapper[pool][alice] = true          // alice is KYC-approved
  allowedSwapper[pool][router] = true         // admin adds router so alice can use it
  allowedSwapper[pool][attacker] = false      // attacker is NOT approved

Attack:
  attacker calls router.exactInputSingle({
      pool: pool,
      recipient: attacker,
      ...
  })

  router calls pool.swap(recipient=attacker, ...)
    pool.msg.sender = router
    _beforeSwap(sender=router, ...)
      SwapAllowlistExtension.beforeSwap(sender=router, ...)
        allowedSwapper[pool][router] == true  ✓  → no revert

  attacker receives output tokens from the curated pool
  allowlist completely bypassed
```
