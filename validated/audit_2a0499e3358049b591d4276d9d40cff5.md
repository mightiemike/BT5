### Title
Gateway L2 Gas Price Admission Uses Stale `l2_gas_price` Instead of `next_l2_gas_price`, Causing Divergent Accept/Reject Decisions - (File: `crates/apollo_gateway/src/stateful_transaction_validator.rs`)

---

### Summary

The gateway's stateful resource-bounds check validates a transaction's `max_price_per_unit` against the **previous block's execution gas price** (`block_header.l2_gas_price`). The blockifier, however, executes the transaction in the **next block** whose gas price is `block_header.next_l2_gas_price` — the EIP-1559-adjusted value computed from the previous block's usage. Because `GatewayFixedBlockSyncStateClient::get_block_info_from_sync_client` never reads `next_l2_gas_price` from the block header, the two components operate on different values for the same conceptual quantity. Transactions whose `max_price_per_unit` falls between the two values are admitted by the gateway but will fail at execution, and the inverse gap causes valid transactions to be rejected.

---

### Finding Description

**Step 1 — Gateway admission check (wrong reference price)**

`StatefulTransactionValidator::validate_resource_bounds` reads the previous block's gas price from `GatewayFixedBlockStateReader::get_block_info()`:

```rust
// TODO(Arni): getnext_l2_gas_price from the block header.
let previous_block_l2_gas_price = self
    .gateway_fixed_block_state_reader
    .get_block_info()
    .await?
    .gas_prices
    .strk_gas_prices
    .l2_gas_price;
self.validate_tx_l2_gas_price_within_threshold(
    executable_tx.resource_bounds(),
    previous_block_l2_gas_price,
)?;
```

The admission threshold is `min_gas_price_percentage * block_header.l2_gas_price / 100`. [1](#0-0) 

**Step 2 — `get_block_info` omits `next_l2_gas_price`**

`GatewayFixedBlockSyncStateClient::get_block_info_from_sync_client` constructs a `BlockInfo` from `block_header.l2_gas_price.price_in_fri` — the price used for execution *in the already-committed block*. The field `block_header.next_l2_gas_price` (which is stored separately in `BlockHeaderWithoutHash` and `StorageBlockHeader`) is never read and never placed into the returned `BlockInfo`. [2](#0-1) 

**Step 3 — Storage schema confirms the two fields are distinct**

`StorageBlockHeader` carries both `l2_gas_price: GasPricePerToken` (the price used for execution in block N) and `next_l2_gas_price: GasPrice` (the EIP-1559-adjusted price to be used in block N+1). They are separate fields with different semantics. [3](#0-2) 

**Step 4 — Execution uses `next_l2_gas_price`**

When the consensus orchestrator builds a proposal, `ProposalInit.l2_gas_price_fri` is set to `self.l2_gas_price`, which is the value computed by `calculate_next_l2_gas_price_for_fin` / `calculate_next_base_gas_price` — i.e., the EIP-1559 output that becomes `next_l2_gas_price` in the committed block header. The blockifier then enforces `tx.max_price_per_unit >= block_info.gas_prices.l2_gas_price` during pre-validation. [4](#0-3) [5](#0-4) 

**Step 5 — EIP-1559 price change magnitude**

`calculate_next_base_gas_price` uses `gas_price_max_change_denominator` from `VersionedConstants::latest_constants()`. The price can increase by up to `current_price / gas_price_max_change_denominator` per block. Under sustained full-block conditions the gap between `l2_gas_price` and `next_l2_gas_price` grows monotonically. [6](#0-5) 

---

### Impact Explanation

Any transaction whose `max_price_per_unit` satisfies:

```
(min_gas_price_percentage / 100) * l2_gas_price  ≤  max_price_per_unit  <  next_l2_gas_price
```

passes the gateway's admission check but is rejected by the blockifier's pre-validation with `ResourceBoundsError::MaxGasPriceTooLow`. The gateway has accepted an invalid transaction. Conversely, when `next_l2_gas_price < l2_gas_price` (price falling), transactions that would succeed at execution are rejected at the gateway.

This matches: **High — Mempool/gateway/RPC admission accepts invalid transactions or rejects valid transactions before sequencing.**

---

### Likelihood Explanation

The divergence is present in every block where the EIP-1559 mechanism adjusts the price (i.e., whenever block gas usage differs from `gas_target`). Under sustained high load the gap is proportional to `current_price / gas_price_max_change_denominator` per block. Any user who sets `max_price_per_unit` close to the current block's gas price — a natural choice when following the "previous block price" heuristic — will be affected. The TODO comment in the source code (`// TODO(Arni): getnext_l2_gas_price from the block header`) confirms the developers are aware of the wrong reference but the fix has not been applied.

---

### Recommendation

In `GatewayFixedBlockSyncStateClient::get_block_info_from_sync_client`, replace the use of `block_header.l2_gas_price.price_in_fri` for the `strk_gas_prices.l2_gas_price` field with `block_header.next_l2_gas_price` (converting it to `NonzeroGasPrice`). This makes the gateway's admission threshold match the price the blockifier will actually enforce during execution. [7](#0-6) 

---

### Proof of Concept

Assume:
- Block N committed with `l2_gas_price = 1000 fri`, `next_l2_gas_price = 1125 fri` (12.5% EIP-1559 increase due to full block).
- `min_gas_price_percentage = 100` (default: threshold = 100% of previous block price).

1. User submits a transaction with `max_price_per_unit = 1050 fri`.
2. Gateway calls `get_block_info()` → receives `l2_gas_price = 1000 fri`.
3. Threshold = `100% * 1000 = 1000`. Check: `1050 >= 1000` → **PASS**. Transaction admitted to mempool.
4. Batcher builds block N+1 with `ProposalInit.l2_gas_price_fri = 1125 fri` (from `next_l2_gas_price`).
5. Blockifier pre-validates: `max_price_per_unit (1050) < actual_block_price (1125)` → `MaxGasPriceTooLow` → **transaction reverts / is dropped**.

The gateway admitted a transaction that the blockifier rejects, violating the admission invariant.

```rust
pub fn calculate_next_base_gas_price(
    price: GasPrice,
    gas_used: GasAmount,
    gas_target: GasAmount,
    min_gas_price: GasPrice,
) -> GasPrice {
    let versioned_constants = VersionedConstants::latest_constants();
    assert!(
        gas_target < versioned_constants.max_block_size,
        "Gas target must be lower than max block size."
    );
    assert!(gas_target.0 > 0, "Gas target must be greater than zero.");
    assert!(
        versioned_constants.gas_price_max_change_denominator > 0,
        "Denominator constant must be greater than zero."
    );

    // If the current price is below the minimum, apply a gradual adjustment and return early.
    // This allows the price to increase by at most 1/MIN_GAS_PRICE_INCREASE_DENOMINATOR per block.
    if price < min_gas_price {
        let max_increase = price.0 / MIN_GAS_PRICE_INCREASE_DENOMINATOR;
        let adjusted = price.0 + max_increase;
        // Cap at min_gas_price to avoid overshooting
        let adjusted_price = adjusted.min(min_gas_price.0);
        info!(
            "Fee Market: Price {} below minimum gas price {}, adjusted price: {} )",
            price.0, min_gas_price.0, adjusted_price
        );
        return GasPrice(adjusted_price);
    }

    // Use U256 to avoid overflow, as multiplying a u128 by a u64 remains within U256 bounds.
    let gas_delta = U256::from(gas_used.0.abs_diff(gas_target.0));
    let gas_target_u256 = U256::from(gas_target.0);
    let price_u256 = U256::from(price.0);

    // Calculate price change by multiplying first, then dividing. This avoids the precision loss
    // that occurs when dividing before multiplying.
    let denominator =
        gas_target_u256 * U256::from(versioned_constants.gas_price_max_change_denominator);
    let price_change = (price_u256 * gas_delta) / denominator;

    let adjusted_price_u256 =
        if gas_used > gas_target { price_u256 + price_change } else { price_u256 - price_change };

    // Sanity check: ensure direction of change is correct
    assert!(
        gas_used > gas_target && adjusted_price_u256 >= price_u256
            || gas_used <= gas_target && adjusted_price_u256 <= price_u256
    );

    // Price should not realistically exceed u128::MAX, bound to avoid theoretical overflow.
    let adjusted_price = u128::try_from(adjusted_price_u256).unwrap_or(u128::MAX);
    GasPrice(max(adjusted_price, min_gas_price.0))
}
```
