### Title
v8 Migration Chain Halt: Vesting-Owned Position with Pending Redelegation Causes `Migrate()` to Return `ErrActiveRedelegation` — (`x/tieredrewards/migrations/v2/migrate.go`)

---

### Summary

A vesting account owner with a legacy tier position can call `MsgTierRedelegate` before the v8 upgrade. If the staking redelegation's 21-day completion window has not elapsed by upgrade height, `Migrate()` → `exitVestedAccountsPositions()` → `ForceFullExitWithDelegation()` → `transferDelegationFromPosition()` hits the `isRedelegating` guard and returns `ErrActiveRedelegation`. This error propagates unhandled all the way back to the upgrade handler, causing a chain halt.

---

### Finding Description

**Step 1 — Vesting account calls `MsgTierRedelegate` (pre-upgrade)**

`validateRedelegatePosition` has no vesting-account restriction: [1](#0-0) 

The only guards are: owner match, position delegated, not same validator, exit lock not elapsed, tier not CloseOnly, and position not already redelegating. A vesting account owner of a legacy position passes all of them.

`TierRedelegate` then calls `BeginRedelegation` on `pos.DelegatorAddress` (the position's dedicated sub-account): [2](#0-1) 

This creates a staking-layer redelegation entry keyed to `pos.DelegatorAddress` with a 21-day completion time.

**Step 2 — v8 `Migrate()` runs at upgrade height**

`Migrate` calls `exitVestedAccountsPositions`, which walks all positions, identifies vesting-owned ones, and calls `ForceFullExitWithDelegation` for each — with no error recovery: [3](#0-2) 

**Step 3 — `ForceFullExitWithDelegation` calls `transferDelegationFromPosition`** [4](#0-3) 

**Step 4 — `transferDelegationFromPosition` hits the `isRedelegating` guard** [5](#0-4) 

`isRedelegating` queries `GetRedelegations` on `pos.DelegatorAddress`: [6](#0-5) 

Because `BeginRedelegation` was called on that exact address in Step 1, `GetRedelegations` returns a non-empty list → `isRedelegating = true` → `ErrActiveRedelegation` is returned.

**Step 5 — Error propagates to chain halt**

`transferDelegationFromPosition` → `ForceFullExitWithDelegation` → `exitVestedAccountsPositions` → `Migrate` — every layer wraps and returns the error with no skip/continue logic: [7](#0-6) 

The upgrade handler receives a non-nil error from `Migrate`, causing the upgrade block to fail and the chain to halt.

---

### Impact Explanation

Any single vesting-owned position with a pending redelegation at upgrade height is sufficient to abort the entire migration. The upgrade block panics/errors, all validators halt at the same height, and the chain cannot advance. This is a **critical** availability impact: the chain is permanently halted until a coordinated emergency patch and re-upgrade.

---

### Likelihood Explanation

- Vesting accounts with legacy positions exist (the migration was written specifically to handle them).
- `MsgTierRedelegate` is a normal, unrestricted user operation for position owners — no governance or special privilege required.
- The staking unbonding period is 21 days; any redelegate within 21 days before the upgrade height triggers the halt.
- A single user, acting entirely within normal protocol rules, can trigger this unilaterally.

---

### Recommendation

In `ForceFullExitWithDelegation` (or in `exitVestedAccountsPositions`), handle the `isRedelegating` case explicitly instead of propagating it as a fatal error. Options:

1. **Skip and log**: If `isRedelegating` is true, log a warning and skip the force-exit for that position. The position will remain and can be exited after the redelegation completes.
2. **Wait for redelegation completion**: Before calling `transferDelegationFromPosition`, check `isRedelegating` and, if true, either skip or use a different exit path (e.g., `TierUndelegate` which does not require the redelegation to be complete).
3. **Remove the `isRedelegating` guard from `transferDelegationFromPosition` for the migration path**: The guard exists to prevent double-redelegation slashing exposure, but during a forced migration exit the concern is different. A separate migration-specific exit function without this guard could be used.

### Proof of Concept

```go
func (s *KeeperSuite) TestMigrate_VestingOwner_PendingRedelegate_CausesChainHalt() {
    // 1. Create a vesting-owned position (legacy, pre-vesting-restriction).
    s.setupTier(1)
    vals, bondDenom := s.getStakingData()
    val := vals[0]
    valAddr := sdk.MustValAddressFromBech32(val.GetOperator())
    lockedAmount := sdkmath.NewInt(sdk.DefaultPowerReduction.Int64())
    owner := s.newVestingOwnerWithBalance(bondDenom, lockedAmount, lockedAmount)
    pos := s.createLockPositionV1(owner, val, valAddr, lockedAmount)

    // 2. Vesting owner calls TierRedelegate — no restriction, succeeds.
    dstValAddr, _ := s.createSecondValidator()
    msgServer := keeper.NewMsgServerImpl(s.keeper)
    _, err := msgServer.TierRedelegate(s.ctx, &types.MsgTierRedelegate{
        Owner:        owner.String(),
        PositionId:   pos.Id,
        DstValidator: dstValAddr.String(),
    })
    s.Require().NoError(err)

    // 3. Confirm redelegation is active on pos.DelegatorAddress.
    isRedelegating, err := s.keeper.IsRedelegating(s.ctx, pos.DelegatorAddress)
    s.Require().NoError(err)
    s.Require().True(isRedelegating)

    // 4. Run v8 Migrate — must fail with ErrActiveRedelegation → chain halt.
    err = v2.Migrate(s.ctx, s.keeper.Positions, s.app.AccountKeeper, s.keeper)
    s.Require().Error(err)
    s.Require().ErrorIs(err, types.ErrActiveRedelegation)
}
```

func (k Keeper) validateRedelegatePosition(ctx context.Context, pos types.PositionState, owner, dstValidator string) error {
	if !pos.IsOwner(owner) {
		return types.ErrNotPositionOwner
	}

	if !pos.IsDelegated() {
		return types.ErrPositionNotDelegated
	}

	if pos.Delegation.ValidatorAddress == dstValidator {
		return types.ErrRedelegationToSameValidator
	}

	sdkCtx := sdk.UnwrapSDKContext(ctx)
	if pos.HasTriggeredExit() && pos.CompletedExitLockDuration(sdkCtx.BlockTime()) {
		return types.ErrExitLockDurationElapsed
	}

	tier, err := k.getTier(ctx, pos.TierId)
	if err != nil {
		return err
	}

	if tier.IsCloseOnly() {
		return types.ErrTierIsCloseOnly
	}

	isRedelegating, err := k.isRedelegating(ctx, pos.DelegatorAddress)
	if err != nil {
		return err
	}
	if isRedelegating {
		return errorsmod.Wrapf(types.ErrActiveRedelegation, "position %d has an active redelegation", pos.Id)
	}

	return nil
}

func (k Keeper) isRedelegating(ctx context.Context, delegatorAddress string) (bool, error) {
	delAddr, err := sdk.AccAddressFromBech32(delegatorAddress)
	if err != nil {
		return false, errorsmod.Wrap(sdkerrors.ErrInvalidAddress, "invalid delegator address")
	}
	reds, err := k.stakingKeeper.GetRedelegations(ctx, delAddr, 1)
	if err != nil {
		return false, err
	}
	return len(reds) > 0, nil
}
