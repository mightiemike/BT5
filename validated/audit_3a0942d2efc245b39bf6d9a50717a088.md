### Title
`read_pending_data` stale-fallback always constructs `DeprecatedPendingBlock`, silently zeroing `l1_data_gas_price` for starknet_version ≥ 0.13.1 — (`crates/apollo_rpc/src/v0_8/api/api_impl.rs`)

### Summary
When the cached pending block's `parent_block_hash` does not match the latest committed block hash, `read_pending_data` synthesizes a fallback `PendingData` using `PendingBlockOrDeprecated::Deprecated`. That variant's `l1_data_gas_price()` accessor unconditionally returns `GasPricePerToken::default()` (both fields zero). The latest header's real `l1_data_gas_price` is available at the call site but is never copied into the fallback. Any RPC consumer that reads the pending block during the normal inter-block window receives zero data-gas price.

### Finding Description
`read_pending_data` copies `l1_gas_price` (wei and fri) from the latest header into the fallback `DeprecatedPendingBlock` but omits `l1_data_gas_price`. [1](#0-0) 

`DeprecatedPendingBlock` has no `l1_data_gas_price` field; the `Deprecated` arm of `l1_data_gas_price()` is hardcoded to return zero with the comment "In older versions, data gas price was 0." [2](#0-1) 

`PendingBlock` (the `Current` variant) does carry `l1_data_gas_price` as a proper field, so the fix is to use that variant in the fallback for starknet_version ≥ 0.13.1. [3](#0-2) 

### Impact Explanation
Every RPC endpoint that calls `read_pending_data` and then reads `block.l1_data_gas_price()` — including `get_block` (which populates `PendingBlockHeader.l1_data_gas_price`) — returns zero for data-gas price during the fallback window. [4](#0-3) 

Fee estimation calls that use the pending block's gas prices will compute data-gas cost as zero, producing an underestimate. Transactions submitted with that estimate may be under-resourced or incorrectly priced. This matches the allowed impact: **"High. RPC execution, fee estimation, tracing, simulation, or pending view returns an authoritative-looking wrong value."**

### Likelihood Explanation
The fallback is triggered by the ordinary race between block-commit and pending-data-sync. It fires on every block boundary for the duration of the sync lag — no attacker action is needed. Any caller querying `pending` during that window observes the zero value.

### Recommendation
Replace the fallback `Deprecated` block with a `Current` (`PendingBlock`) block that copies `l1_data_gas_price` (and `l1_da_mode`, `l2_gas_price`) from the latest header, gated on the header's `starknet_version`. Alternatively, add `l1_data_gas_price` to the fallback construction and adjust `l1_data_gas_price()` on `Deprecated` to read it. [5](#0-4)
