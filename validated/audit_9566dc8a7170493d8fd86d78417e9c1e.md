### Title
Asymmetric Averaging in L1 Gas Price FRI Conversion: Mean Wei Price × Spot ETH/STRK Rate Produces Wrong Block Fee — (`crates/apollo_l1_gas_price/src/l1_gas_price_provider.rs`, `crates/apollo_l1_gas_price/src/exchange_rate_oracle.rs`, `crates/apollo_consensus_orchestrator/src/utils.rs`)

---

### Summary

The L1 gas price in FRI that is embedded in every block and used for fee charging is computed as `mean(L1_gas_wei) × spot(ETH/STRK_rate)`. The L1 gas price in wei is averaged over a configurable window of L1 blocks (`number_of_blocks_for_mean`), but the ETH/STRK conversion rate is a single spot oracle query for a quantized timestamp. When the ETH/STRK rate is volatile, the product of a time-averaged wei price and a spot conversion rate diverges from the true time-consistent FRI price, causing every transaction in the affected block to be charged the wrong L1 gas fee in FRI.

---

These two values are then multiplied together in `get_l1_prices_in_fri_and_wei_and_conversion_rate` to produce the FRI-denominated L1 gas price that is placed in the block:

```rust

    l1_gas_price_provider_client.get_rate(timestamp),          // spot
    l1_gas_price_provider_client.get_price_info(BlockTimestamp(timestamp)) // mean
);
...
let l1_gas_prices_fri_result =
    L1PricesInFri::convert_from_wei(&prices_in_wei, eth_to_fri_rate);
``` [3](#0-2) 

The resulting `l1_gas_price_fri` and `l1_data_gas_price_fri` are embedded in `ProposalInit` / `BlockInfo` and passed to the blockifier for execution-time fee charging:

```rust
// crates/apollo_consensus_orchestrator/src/utils.rs:333-337
strk_gas_prices: GasPriceVector {
    l1_gas_price: l1_gas_price_fri,
    l1_data_gas_price: l1_data_gas_price_fri,
    l2_gas_price: l2_gas_price_fri,
},
``` [4](#0-3) 

The invariant violation is:

```
l1_gas_price_fri = mean_over_N_blocks(eth_base_fee_wei) × spot_at_T(eth_strk_rate)
```

A time-consistent computation would require either both values to be averaged over the same window, or both to be spot values. The current hybrid produces a value that is neither the true mean FRI price nor the true spot FRI price.

---

### Impact Explanation

Every transaction in a block that pays L1 gas fees in FRI is charged using the incorrect `l1_gas_price_fri`. If the ETH/STRK rate spikes at proposal time (e.g., a sudden STRK price drop), the FRI-denominated L1 gas price is inflated relative to the true time-consistent value, and all users in that block overpay. If the rate drops at proposal time (STRK price spike), users underpay. Because both the proposer and the validator independently fetch the same spot rate for the same timestamp, the margin check in `validate_proposal` passes, and the wrong value is committed to the block without any rejection signal.

This matches the allowed impact: **Incorrect fee/L1 gas price effect with economic impact** — the wrong `l1_gas_price_fri` is committed to the block and used by the blockifier to charge actual fees.

---

### Likelihood Explanation

The ETH/STRK rate is volatile and can move significantly within a single `lag_interval_seconds` window (default 60 seconds). The L1 gas price averaging window spans `number_of_blocks_for_mean` L1 blocks (default 300), covering roughly 60 minutes of L1 history. A 10–20% ETH/STRK rate swing within a single quantized interval — common during market events — produces a proportional error in `l1_gas_price_fri` for every block proposed during that interval. No privileged access is required; the condition arises from normal oracle volatility.

---

### Recommendation

Apply the same temporal smoothing to the ETH/STRK conversion rate as is applied to the L1 gas price in wei. Concretely, maintain a ring buffer of ETH/STRK rate samples (analogous to `price_samples_by_block`) and return the mean over the same `number_of_blocks_for_mean` window from `get_rate`. Alternatively, document and enforce that both inputs must use the same granularity (both spot or both averaged), and add an assertion or type-level distinction to prevent future divergence.

---

### Proof of Concept

**Setup**: `number_of_blocks_for_mean = 300`, `lag_interval_seconds = 60`.

**Normal state** (blocks 1–300): ETH base fee = 10 gwei, ETH/STRK rate = 1000 STRK/ETH.
- `mean(eth_gas_wei)` = 10 gwei
- `spot(eth_strk_rate)` = 1000 STRK/ETH
- `l1_gas_price_fri` = 10 gwei × 1000 = 10,000 FRI/gas ✓

**Rate spike at block 301**: ETH base fee still 10 gwei (mean unchanged), but ETH/STRK rate spikes to 2000 STRK/ETH for the current quantized interval.
- `mean(eth_gas_wei)` = 10 gwei (unchanged — 300-block average)
- `spot(eth_strk_rate)` = 2000 STRK/ETH (spot spike)
- `l1_gas_price_fri` = 10 gwei × 2000 = **20,000 FRI/gas** (2× the correct value)

**True time-consistent FRI price** (if rate were also averaged over 300 blocks): ≈ 10,000 FRI/gas.

Every transaction in block 301 that consumes L1 gas is charged **double** the correct FRI amount. The proposer and validator both fetch the same spot rate for the same timestamp, so `within_margin` passes and the inflated price is committed to the block. [5](#0-4) [6](#0-5) [7](#0-6)

                timestamp: timestamp.0,
                lag: self.config.lag_margin_seconds.as_secs(),
            })?
            .timestamp;

        // Check if the prices are stale.
        if timestamp.0 > (*last_timestamp + self.config.max_time_gap_seconds) {
            return Err(L1GasPriceProviderError::StaleL1GasPricesError {
                current_timestamp: timestamp.0,
                last_valid_price_timestamp: *last_timestamp,
            });
        }

        // This index is for the last block in the mean (inclusive).
        let index_last_timestamp_rev = samples.iter().rev().position(|data| {
            data.timestamp <= timestamp.saturating_sub(&self.config.lag_margin_seconds.as_secs())

        // Could not find a block with the requested timestamp and lag.

    async fn fetch_rate(&self, timestamp: u64) -> Result<u128, ExchangeRateOracleClientError> {
        const NUMBER_OF_TIMESTAMPS_BACK: u64 = 1;
        let quantized_timestamp = (timestamp - self.config.lag_interval_seconds)
            .checked_div(self.config.lag_interval_seconds)
            .expect("lag_interval_seconds should be non-zero");

        let mut cache = self.cached_prices.lock().unwrap();

        if let Some(rate) = cache.get(&quantized_timestamp) {
            debug!("Cached conversion rate for timestamp {timestamp} is {rate}");
            return Ok(*rate);
        }

        // Check if there is a query already sent out for this timestamp, if not, start one.
        let mut queries = self.queries.lock().unwrap();
        let handle = queries
            .get_or_insert_mut(quantized_timestamp, || self.spawn_query(quantized_timestamp));
        // If the query is not finished, return an error.
        if !handle.is_finished() {
            debug!("Query not yet resolved: timestamp={timestamp}");
            // If the previous quantized timestamp is in the cache, use it.
            if let Some(rate) = cache.get(&(quantized_timestamp - NUMBER_OF_TIMESTAMPS_BACK)) {
                debug!(
                    "Query not yet resolved: timestamp={timestamp}, using previous rate {rate} \
                     from quantized timestamp={}",
                    (quantized_timestamp - NUMBER_OF_TIMESTAMPS_BACK)
                        * self.config.lag_interval_seconds
                );
                return Ok(*rate);
            }
            // If not, return a query not ready error.
            return Err(ExchangeRateOracleClientError::QueryNotReadyError(timestamp));
        }
        let result = handle.now_or_never().expect("Handle must be finished if we got here");
        let rate = match result {
            Ok(Ok(rate)) => rate,
            Ok(Err(e)) => {
                warn!("Query returned an error for timestamp {timestamp}: {e:?}");
                // Must remove failed query from the cache, to avoid re-polling it.
                queries.pop(&quantized_timestamp);
                return Err(e);
            }
            Err(e) => {
                warn!("Query failed to join handle for timestamp {timestamp}: {e:?}");
                self.metrics.error_count.increment(1);
                // Must remove failed query from the cache, to avoid re-polling it.
                queries.pop(&quantized_timestamp);
                return Err(ExchangeRateOracleClientError::JoinError(e.to_string()));
            }
        };

        // Make sure to cache the result.
        cache.put(quantized_timestamp, rate);
        // We don't need to come back to this query since we have the result in cache.
        queries.pop(&quantized_timestamp);
        debug!("Caching conversion rate for timestamp {timestamp}, with rate {rate}");
        Ok(rate)
    }

pub(crate) async fn get_l1_prices_in_fri_and_wei_and_conversion_rate(
    l1_gas_price_provider_client: Arc<dyn L1GasPriceProviderClient>,
    timestamp: u64,
    previous_proposal_init: Option<&PreviousProposalInitInfo>,
    gas_price_params: &GasPriceParams,
) -> (L1PricesInFri, L1PricesInWei, u128) {
    // One of these paths should fill the return values:
    // 1. Both L1 gas price and eth/strk rate are Ok, use those.
    // 2. Otherwise, use previous block info.
    // 3. If that isn't available either, use min gas prices and default eth/strk rate.

    // Get the eth to fri rate from the oracle, and the L1 gas price (in wei) from the provider.
    let (eth_to_fri_rate, price_info) = tokio::join!(
        l1_gas_price_provider_client.get_rate(timestamp),
        l1_gas_price_provider_client.get_price_info(BlockTimestamp(timestamp))
    );
    if price_info.is_err() {
        warn!("Failed to get l1 gas price from provider: {:?}", price_info);
        CONSENSUS_L1_GAS_PRICE_PROVIDER_ERROR.increment(1);
    }
    if eth_to_fri_rate.is_err() {
        warn!("Failed to get eth to fri rate from oracle: {:?}", eth_to_fri_rate);
    }
    if let (Ok(eth_to_fri_rate), Ok(mut price_info)) = (eth_to_fri_rate, price_info) {
        // Both L1 prices and rate are Ok, so we can use them.
        info!(
            "raw eth_to_fri_rate (from oracle): {eth_to_fri_rate}, raw l1 gas price wei (from \
             provider): {price_info:?}"

        apply_fee_transformations(&mut price_info, gas_price_params);
        let prices_in_wei = L1PricesInWei {
            l1_gas_price: price_info.base_fee_per_gas,
            l1_data_gas_price: price_info.blob_fee,
        };
        // Apply the eth/strk rate to get prices in fri.
        let l1_gas_prices_fri_result =
            L1PricesInFri::convert_from_wei(&prices_in_wei, eth_to_fri_rate);
        // If conversion fails, leave return_value=None to try backup methods.
        if let Ok(prices_in_fri) = l1_gas_prices_fri_result {
            return (prices_in_fri, prices_in_wei, eth_to_fri_rate);
        } else {
            warn!(
                "Failed to convert L1 gas prices to FRI: {:?}",
                l1_gas_prices_fri_result.clone().err()
            );
        }
    }
