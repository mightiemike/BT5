### Title
Attacker-Controlled `ClassId` Prefix Bypasses `IsAwayFromOrigin` Check, Triggering Unauthorized Unescrow — (`x/nft-transfer/keeper/packet.go`)

---

### Summary

The question's specific framing contains one factual error about which port/channel is checked, but the underlying vulnerability is **real and exploitable**. The code in `processReceivedPacket` checks `IsAwayFromOrigin` against the **source** port/channel (not the destination port/channel as the question states). A malicious counterparty chain can craft a `ClassId` that starts with their own **source** port/channel prefix, causing `IsAwayFromOrigin` to return `false`, which triggers the unescrow path and allows the attacker to drain NFTs legitimately held in the destination chain's escrow address.

---

### Finding Description

**Exact code path:**

In `processReceivedPacket`, the direction check is:

```go
isAwayFromOrigin := types.IsAwayFromOrigin(packet.GetSourcePort(), packet.GetSourceChannel(), data.ClassId)
``` [1](#0-0) 

`IsAwayFromOrigin` returns `false` (i.e., "moving back to origin") when `data.ClassId` starts with `sourcePort/sourceChannel/`:

```go
func IsAwayFromOrigin(sourcePort, sourceChannel, fullClassPath string) bool {
    prefixClassID := GetClassPrefix(sourcePort, sourceChannel)
    return !strings.HasPrefix(fullClassPath, prefixClassID)
}
``` [2](#0-1) 

When `isAwayFromOrigin == false`, the unescrow branch executes:

```go
unprefixedClassID := types.RemoveClassPrefix(packet.GetSourcePort(), packet.GetSourceChannel(), data.ClassId)
voucherClassID := types.ParseClassTrace(unprefixedClassID).IBCClassID()
for _, tokenID := range data.TokenIds {
    if err := k.nftKeeper.TransferOwner(ctx, voucherClassID, tokenID, escrowAddress, receiver); err != nil {
        return err
    }
}
``` [3](#0-2) 

The escrow address used is `GetEscrowAddress(packet.GetDestPort(), packet.GetDestChannel())` — the destination chain's escrow address. [4](#0-3) 

**The attack:**

The attacker controls the source chain. They know their own source port (e.g., `"nft"`) and source channel (e.g., `"channel-0"`). They craft a packet with:

```
ClassId = "nft/channel-0/victimClass"
```

On the receiving (destination) chain:
- `IsAwayFromOrigin("nft", "channel-0", "nft/channel-0/victimClass")` → `false`
- `unprefixedClassID = RemoveClassPrefix("nft", "channel-0", "nft/channel-0/victimClass")` = `"victimClass"`
- `voucherClassID = ParseClassTrace("victimClass").IBCClassID()` = `"victimClass"` (no path, so `IBCClassID()` returns the base class ID directly) [5](#0-4) 
- `TransferOwner(ctx, "victimClass", tokenID, escrowAddress, receiver)` is called

**Precondition:** A legitimate user on the destination chain previously sent an NFT with `classID = "victimClass"` to the attacker's chain via the same channel. This escrowed the NFT at `GetEscrowAddress(destPort, destChannel)`. The attacker now unescrows it to their own receiver address.

**No guard prevents this.** `ValidateBasic` on `NonFungibleTokenPacketData` only checks that `ClassId` is non-blank, that token/URI counts match, and that sender/receiver are valid bech32 addresses. There is no check that `ClassId` does not start with the source port/channel prefix: [6](#0-5) 

---

### Impact Explanation

An attacker operating a malicious IBC counterparty chain can steal any NFT that has been legitimately escrowed on the victim chain. For every NFT with `classID = X` that a user on chain A sent to the attacker's chain B via channel `(nft, channel-0)`, the attacker can send a crafted packet with `ClassId = "nft/channel-0/X"` and any `tokenID` that is currently held in the escrow address. The `TransferOwner` call moves the NFT from the escrow address to the attacker's receiver, with no ownership check beyond the escrow address holding the token.

---

### Likelihood Explanation

Any IBC-connected counterparty chain can exploit this. The attacker only needs to:
1. Have an established IBC channel to the victim chain.
2. Know the `classID` and `tokenID` of an NFT currently held in the victim chain's escrow address (observable on-chain).
3. Craft a packet with `ClassId = sourcePort/sourceChannel/victimClassID`.

No governance, privileged keys, or social engineering are required.

---

### Recommendation

Before entering the unescrow branch, validate that the `ClassId` in the packet data is consistent with a legitimate prior escrow. Specifically, verify that a class trace record exists for the unprefixed class, or that the NFT is actually owned by the escrow address before calling `TransferOwner`. The ICS-721 spec requires that the receiving chain only unescrow tokens it previously escrowed — this invariant must be enforced in code, not assumed from the counterparty.

A minimal fix: before calling `TransferOwner`, check that the NFT with `(voucherClassID, tokenID)` is actually owned by `escrowAddress`. If not, return an error acknowledgement rather than panicking or silently failing.

func RemoveClassPrefix(portID, channelID, classID string) string {
	classPrefix := GetClassPrefix(portID, channelID)
	return classID[len(classPrefix):]
}

// IsAwayFromOrigin determine if non-fungible token is moving away from
// the origin chain (the chain issued by the native nft).
// Note that fullClassPath refers to the full path of the unencoded classID.
// The longer the fullClassPath, the farther it is from the origin chain
func IsAwayFromOrigin(sourcePort, sourceChannel, fullClassPath string) bool {
	prefixClassID := GetClassPrefix(sourcePort, sourceChannel)
	return !strings.HasPrefix(fullClassPath, prefixClassID)
}

func (nftpd NonFungibleTokenPacketData) ValidateBasic() error {
	if strings.TrimSpace(nftpd.ClassId) == "" {
		return newsdkerrors.Wrap(ErrInvalidClassID, "classId cannot be blank")
	}

	if len(nftpd.TokenIds) == 0 {
		return newsdkerrors.Wrap(ErrInvalidTokenID, "tokenId cannot be blank")
	}

	if len(nftpd.TokenIds) != len(nftpd.TokenUris) {
		return newsdkerrors.Wrap(ErrInvalidPacket, "tokenIds and tokenUris lengths do not match")
	}

	if strings.TrimSpace(nftpd.Sender) == "" {
		return newsdkerrors.Wrap(sdkerrors.ErrInvalidAddress, "sender address cannot be blank")
	}

	// decode the sender address
	if _, err := sdk.AccAddressFromBech32(nftpd.Sender); err != nil {
		return newsdkerrors.Wrap(sdkerrors.ErrInvalidAddress, "invalid sender address")
	}

	if strings.TrimSpace(nftpd.Receiver) == "" {
		return newsdkerrors.Wrap(sdkerrors.ErrInvalidAddress, "receiver address cannot be blank")
	}

	// decode the receiver address
	if _, err := sdk.AccAddressFromBech32(nftpd.Receiver); err != nil {
		return newsdkerrors.Wrap(sdkerrors.ErrInvalidAddress, "invalid receiver address")
	}
	return nil
}
