Audit Report

## Title
`SwapAllowlistExtension.beforeSwap` gates on router address instead of originating user, enabling complete allowlist bypass — (`metric-periphery/contracts/extensions/SwapAllowlistExtension.sol`)

## Summary
`SwapAllowlistExtension.beforeSwap` receives `sender` from the pool's `_beforeSwap` hook, which is always `msg.sender` of the `pool.swap()` call. When a user routes through `MetricOmmSimpleRouter`, `msg.sender` to the pool is the router contract, so the allowlist check evaluates `allowedSwapper[pool][router]` rather than `allowedSwapper[pool][user]`. Any pool that allowlists the router to support router-mediated swaps inadvertently opens its curated gate to every address that calls through the router, regardless of individual allowlist status.

## Finding Description
**Root cause — wrong actor in the allowlist key:**

`MetricOmmPool.swap` passes `msg.sender` directly as `sender` to `_beforeSwap`: [1](#0-0) 

`ExtensionCalling._beforeSwap` forwards that `sender` verbatim to every configured extension: [2](#0-1) 

`SwapAllowlistExtension.beforeSwap` keys its check on `(msg.sender=pool, sender)`. When the call originates from the router, `sender` is the router address: [3](#0-2) 

**Exploit call chain:**
```
attacker → MetricOmmSimpleRouter.exactInputSingle(pool=curated_pool, ...)
              └─→ pool.swap(recipient, ...) [msg.sender = router]
                       └─→ _beforeSwap(sender=router, ...)
                                └─→ SwapAllowlistExtension.beforeSwap(sender=router, ...)
                                         └─→ allowedSwapper[pool][router] == true → PASSES
```

`MetricOmmSimpleRouter.exactInputSingle` calls `pool.swap` directly with no user identity forwarded: [4](#0-3) 

The same applies to `exactInput`, `exactOutputSingle`, and `exactOutput`: [5](#0-4) 

**Why existing guards fail:** The `allowAllSwappers` escape hatch and the per-swapper mapping both operate on `sender`, which is structurally the router address for any router-mediated call. There is no secondary check on the originating EOA or `recipient`. [6](#0-5) 

## Impact Explanation
A curated pool using `SwapAllowlistExtension` is intended to restrict trading to a vetted set of counterparties. Once the router is allowlisted (the only way to permit any router-mediated swap), every address — including those explicitly excluded — can execute swaps against the pool's liquidity. This constitutes a complete bypass of the pool's access-control invariant, allowing unauthorized actors to trade at oracle-anchored prices, extract LP value, and front-run allowlisted participants. This is a direct loss of the pool's intended access-control guarantee with immediate fund-impacting consequences for LPs.

## Likelihood Explanation
`MetricOmmSimpleRouter` is a first-party periphery contract. Any user aware of it can exploit this in a single transaction with no privileged access, special tokens, or multi-step setup. The only precondition — the router being allowlisted — is the normal operational requirement for any pool that supports routing. This is a low-barrier, unprivileged, single-transaction exploit reachable on every production pool using `SwapAllowlistExtension` with router support enabled.

## Recommendation
The extension must gate on the originating user, not the intermediary router. Two complementary approaches:

1. **Encode the real caller in `extensionData`**: The router encodes `msg.sender` into `extensionData` before calling `pool.swap`; the extension reads and verifies it. This is acceptable if the router is a known, immutable contract.
2. **Preferred — add a dedicated `user` field to the `beforeSwap` interface**: Separate `user` (economic beneficiary / originating caller) from `sender` (contract initiating the pool call). The allowlist gates on `user`. The router populates `user = msg.sender` before the pool call.

Until fixed, pools relying on `SwapAllowlistExtension` for access control must not allowlist the router, accepting that router-mediated swaps are unavailable for those pools.

    uint256 last = params.pools.length - 1;
    int128 amount = MetricOmmSwapInputs.asAmountSpecifiedIn(params.amountIn);

    for (uint256 i = 0; i <= last; i++) {
      address pool = params.pools[i];
      bool zeroForOne = MetricOmmSwapPath.resolveZeroForOneBitmap(params.zeroForOneBitMap, i);

      _setNextCallbackContext(pool, CALLBACK_MODE_JUST_PAY, i == 0 ? msg.sender : address(this), params.tokens[i]);
      (int128 amount0Delta, int128 amount1Delta) = IMetricOmmPoolActions(pool)
        .swap(
          i == last ? params.recipient : address(this),
          zeroForOne,
          amount,

          "",
          params.extensionDatas[i]
