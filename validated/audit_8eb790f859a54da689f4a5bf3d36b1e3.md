### Title
Cost-Limit Bypass in `op_bls_verify` via Wrong `check_cost` Argument — (`File: src/bls_ops.rs`)

### Summary

`op_bls_verify` accumulates cost per `(PublicKey, msg)` pair but passes `cost` instead of `max_cost` as the ceiling argument to `check_cost` inside the processing loop. Because `cost ≤ cost` is always true, the mid-loop guard never fires. An attacker-controlled CLVM program can supply an arbitrarily large flat list of `(G1, msg)` pairs and force the node to execute all BLS pairing operations before the caller ever sees the final cost — performing far more cryptographic work than the declared cost budget permits.

### Finding Description

`op_bls_verify` opens with a correct guard:

```rust
let mut cost = BLS_PAIRING_BASE_COST;
check_cost(cost, max_cost)?;   // correct
```

Inside the loop that consumes each `(pk, msg)` pair, cost is accumulated correctly, but the ceiling argument is wrong:

```rust
cost += BLS_PAIRING_COST_PER_ARG;
cost += msg.as_ref().len() as Cost * BLS_MAP_TO_G2_COST_PER_BYTE;
cost += DST_G2.len() as Cost * BLS_MAP_TO_G2_COST_PER_DST_BYTE;
check_cost(cost, cost)?;   // BUG: should be check_cost(cost, max_cost)?
``` [1](#0-0) 

`check_cost(cost, cost)` evaluates `cost ≤ cost`, which is unconditionally `true`. The loop therefore never terminates early regardless of how many pairs are supplied or how far `cost` has grown past `max_cost`.

Compare with the sibling operator `op_bls_pairing_identity`, which correctly passes `max_cost`:

```rust
cost += BLS_PAIRING_COST_PER_ARG;
check_cost(cost, max_cost)?;   // correct
``` [2](#0-1) 

### Impact Explanation

`BLS_PAIRING_COST_PER_ARG` is `1_200_000` cost units per pair. A standard Chia block cost limit is on the order of `11_000_000_000`. Without the mid-loop guard, an attacker can supply thousands of `(G1, msg)` pairs. Each pair triggers a full BLS12-381 hash-to-G2 and pairing operation — among the most expensive operations in the codebase. All of this work is completed before the `Reduction` is returned and the caller can observe that `cost > max_cost`. The node performs unbounded cryptographic work per submitted program, enabling a practical denial-of-service against full nodes and block validators.

### Likelihood Explanation

`op_bls_verify` is a production operator reachable by any attacker-controlled CLVM program submitted to the mempool or included in a block. No special privilege is required. The attacker only needs to craft a puzzle or solution that invokes `bls_verify` with a long flat argument list. The bug is silent — no error is raised, no log is emitted — so it is unlikely to be caught by casual testing.

### Recommendation

Change line 391 in `src/bls_ops.rs` from:

```rust
check_cost(cost, cost)?;
```

to:

```rust
check_cost(cost, max_cost)?;
```

This matches the pattern used in every other variadic BLS operator in the same file (`op_bls_pairing_identity`, `op_bls_g1_subtract`, `op_bls_g2_add`, `op_bls_g2_subtract`).

### Proof of Concept

Construct a CLVM program that calls `bls_verify` with a valid G2 signature followed by N `(G1, msg)` pairs where N is chosen so that `N × BLS_PAIRING_COST_PER_ARG` greatly exceeds `max_cost`:

```
(bls_verify <sig> <pk1> <msg1> <pk2> <msg2> ... <pkN> <msgN>)
```

With `max_cost = 11_000_000_000` and `BLS_PAIRING_COST_PER_ARG = 1_200_000`, supplying ~10 000 pairs pushes the true cost to ~12 billion, yet the loop never exits early. The node completes all ~10 000 BLS pairing operations, then returns a `Reduction` whose cost the caller rejects — after the damage is done. [3](#0-2)

```rust
pub fn op_bls_verify(
    a: &mut Allocator,
    input: NodePtr,
    max_cost: Cost,
    _flags: ClvmFlags,
) -> Response {
    let mut cost = BLS_PAIRING_BASE_COST;
    check_cost(cost, max_cost)?;

    let mut args = input;

    // the first argument is the signature
    let signature = a.g2(first(a, args)?)?;

    // followed by a variable number of (G1, msg)-pairs (as a flat list)
    args = rest(a, args)?;

    let mut items = Vec::<(PublicKey, Atom)>::new();
    while !nilp(a, args) {
        let pk = a.g1(first(a, args)?)?;
        args = rest(a, args)?;
        let msg = atom(a, first(a, args)?, "bls_verify message")?;
        args = rest(a, args)?;

        cost += BLS_PAIRING_COST_PER_ARG;
        cost += msg.as_ref().len() as Cost * BLS_MAP_TO_G2_COST_PER_BYTE;
        cost += DST_G2.len() as Cost * BLS_MAP_TO_G2_COST_PER_DST_BYTE;
        check_cost(cost, max_cost)?;

        items.push((pk, msg));
    }

    if !aggregate_verify(&signature, items) {
        Err(EvalErr::BLSVerifyFailed(input))?
    } else {
        Ok(Reduction(cost, a.nil()))
    }
}
