Audit Report

## Title
SwapAllowlistExtension Checks Router Address Instead of Actual Trader, Allowing Any User to Bypass the Swap Allowlist — (`metric-periphery/contracts/extensions/SwapAllowlistExtension.sol`)

## Summary

`MetricOmmPool.swap` passes `msg.sender` as the `sender` argument to `_beforeSwap`, which forwards it unchanged to `SwapAllowlistExtension.beforeSwap`. When swaps are routed through `MetricOmmSimpleRouter`, the pool's `msg.sender` is the router contract, so the extension evaluates `allowedSwapper[pool][router]` instead of `allowedSwapper[pool][user]`. Once the pool admin allowlists the router (required for any router-mediated swap to succeed), the per-user gate is completely defeated and any unprivileged user can trade on a curated pool.

## Finding Description

`MetricOmmPool.swap` fires `_beforeSwap` with `msg.sender` as the `sender` argument: [1](#0-0) 

`ExtensionCalling._beforeSwap` forwards that value unchanged to every configured extension: [2](#0-1) 

`SwapAllowlistExtension.beforeSwap` enforces the allowlist against that `sender`, where `msg.sender` is the pool and `sender` is whoever called `pool.swap()`: [3](#0-2) 

`MetricOmmSimpleRouter.exactInputSingle` and `exactInput` call `pool.swap()` directly, making the router the direct `msg.sender` to the pool: [4](#0-3) [5](#0-4) 

The `allowedSwapper` mapping is keyed by `(pool, swapper)`: [6](#0-5) 

Two failure modes result. First, if the admin does not allowlist the router, every router-mediated swap reverts — including those from legitimately allowlisted users. Second, once the admin allowlists the router to unblock allowlisted users, `allowedSwapper[pool][router] == true` for every caller, and any non-allowlisted user can bypass the gate by routing through `MetricOmmSimpleRouter`. There is no mechanism in the extension or pool to recover the originating user identity from the router.

## Impact Explanation

A pool deployer who configures `SwapAllowlistExtension` to restrict trading to a curated set of counterparties (e.g., KYC'd addresses, whitelisted market makers) loses that protection entirely for any user who routes through the public `MetricOmmSimpleRouter`. The attacker receives full access to pool liquidity at oracle-derived bid/ask prices with no special privileges required. LP providers are exposed to trades from actors the pool admin explicitly intended to exclude, constituting a direct admin-boundary bypass with fund-impacting consequences.

## Likelihood Explanation

`MetricOmmSimpleRouter` is the primary public entrypoint for end-users. Any non-allowlisted user who discovers the pool is curated can trivially route through the router with a single `exactInputSingle` or `exactInput` call. No special privileges, flash loans, or multi-transaction setup are required. The bypass is repeatable and unconditional once the router is allowlisted.

## Recommendation

The extension must check the economically relevant actor, not the intermediary. Two sound approaches:

1. **Forward the originating user inside `extensionData`**: Have the router encode `msg.sender` into `extensionData`, and have the extension decode and verify it — combined with a check that `msg.sender` (the pool) is a known pool so the payload cannot be spoofed by a direct caller.
2. **Check `tx.origin` in the extension**: Acceptable only if the router is the sole trusted intermediary and re-entrancy is already blocked by the pool's `nonReentrant` guard.
3. **Document incompatibility**: Require direct pool interaction for allowlisted pools and enforce this at the factory or extension configuration level.

## Proof of Concept

```
Setup:
  pool configured with SwapAllowlistExtension
  allowedSwapper[pool][alice] = true   // alice is the only allowed trader
  allowedSwapper[pool][router] = false // router not explicitly listed

Step 1 — admin enables router for alice to use it:
  allowedSwapper[pool][router] = true

Step 2 — bob (not allowlisted) calls:
  MetricOmmSimpleRouter.exactInputSingle(pool, zeroForOne=true, ...)

  Router calls: pool.swap(recipient=bob, ...)
  Pool calls:   _beforeSwap(sender=router, ...)
  Extension evaluates:
    allowAllSwappers[pool]       → false
    allowedSwapper[pool][router] → true
    → swap proceeds for bob      ← allowlist bypassed

The moment the admin enables the router for any allowlisted user,
the gate opens for all users routing through MetricOmmSimpleRouter.
```
