### Title
Division-by-Zero in `PerpEngine.socializeSubaccount` Permanently Blocks Finalization When `openInterest == 0` — (`core/contracts/PerpEngine.sol`)

---

### Summary

`PerpEngine.socializeSubaccount` unconditionally divides by `state.openInterest` when a product still has a negative `vQuoteBalance` after insurance coverage. `MathSD21x18.div` explicitly reverts with `"DBZ"` when the divisor is zero. If `state.openInterest == 0` at the time of finalization — a reachable state when all other participants have closed their positions — the entire `_finalizeSubaccount` call reverts, permanently trapping the bad debt in the protocol.

---

### Finding Description

In `PerpEngine.socializeSubaccount`, after attempting to cover a negative `vQuoteBalance` with insurance, the code falls through to the socialization branch:

```solidity
// core/contracts/PerpEngine.sol lines 164–168
if (balance.vQuoteBalance < 0) {
    // socialize across all other participants
    int128 fundingPerShare = -balance.vQuoteBalance.div(
        state.openInterest   // ← reverts if == 0
    );
``` [1](#0-0) 

`MathSD21x18.div` contains an explicit guard:

```solidity
// core/contracts/libraries/MathSD21x18.sol line 64
require(y != 0, ERR_DIV_BY_ZERO);
``` [2](#0-1) 

`_finalizeSubaccount` in `ClearinghouseLiq.sol` requires `balance.amount == 0` for every perp product before calling `perpEngine.socializeSubaccount`:

```solidity
// core/contracts/ClearinghouseLiq.sol lines 313–320
for (uint32 i = 0; i < v.perpIds.length; ++i) {
    uint32 perpId = v.perpIds[i];
    IPerpEngine.Balance memory balance = perpEngine.getBalance(perpId, txn.liquidatee);
    require(balance.amount == 0, ERR_NOT_FINALIZABLE_SUBACCOUNT);
}
``` [3](#0-2) 

Because `balance.amount == 0` is enforced, the liquidatee contributes **zero** to `state.openInterest`. `state.openInterest` therefore reflects only the aggregate of all *other* participants. If all other participants have closed their positions in that product, `state.openInterest` reaches zero while the liquidatee's `vQuoteBalance` can remain negative (accumulated funding losses from a previously held position). There is no guard anywhere in `socializeSubaccount` checking `state.openInterest != 0` before the division.

The call to `perpEngine.socializeSubaccount` is made unconditionally from `_finalizeSubaccount`:

```solidity
// core/contracts/ClearinghouseLiq.sol lines 386–389
v.insurance = perpEngine.socializeSubaccount(
    txn.liquidatee,
    v.insurance
);
``` [4](#0-3) 

Because the revert propagates all the way up through `liquidateSubaccountImpl`, the finalization transaction always fails under these conditions.

---

### Impact Explanation

- The subaccount can never be finalized: `_finalizeSubaccount` always reverts.
- The bad debt (negative `vQuoteBalance`) is permanently locked in the protocol's accounting.
- Insurance funds that *could* partially cover the deficit are never applied.
- The subaccount cannot be closed, leaving a zombie entry in the protocol state.
- This matches the Critical scope: **solvency/accounting failure in liquidation/settlement that leaves the protocol undercollateralized**.

---

### Likelihood Explanation

The preconditions are realistic and can arise organically:

1. A liquidatee accumulates a large negative `vQuoteBalance` through funding payments on a perp position.
2. Their position is liquidated (amount → 0) but `vQuoteBalance` remains negative.
3. The product is low-liquidity or being wound down; all remaining participants close their positions, driving `openInterest` to zero.
4. A liquidator calls `liquidateSubaccountImpl` with `productId = type(uint32).max` to finalize — the call reverts every time.

No privileged access, no governance capture, and no external dependency failure is required. The state is reachable through normal protocol operation.

---

### Recommendation

Add a zero-check for `state.openInterest` before the division in `PerpEngine.socializeSubaccount`. When `openInterest == 0`, there are no other participants to socialize the loss across; the only correct action is to zero out the balance and absorb the remaining deficit directly (e.g., record it as unrecoverable protocol loss or skip the socialization step):

```solidity
if (balance.vQuoteBalance < 0) {
    if (state.openInterest != 0) {
        int128 fundingPerShare = -balance.vQuoteBalance.div(state.openInterest);
        state.cumulativeFundingLongX18 += fundingPerShare;
        state.cumulativeFundingShortX18 -= fundingPerShare;
    }
    // If openInterest == 0, no one to socialize to; absorb the loss silently.
    balance.vQuoteBalance = 0;
}
``` [5](#0-4) 

---

### Proof of Concept

```solidity
// Hardhat unit test (pseudocode)
it("socializeSubaccount reverts when openInterest == 0", async () => {
    // 1. Set up a perp product with openInterest = 0
    await perpEngine.setStateForTest(productId, {
        cumulativeFundingLongX18: 0,
        cumulativeFundingShortX18: 0,
        availableSettle: 0,
        openInterest: 0,          // all other participants have closed
    });

    // 2. Give liquidatee a zero-amount position with negative vQuoteBalance
    await perpEngine.setBalanceForTest(productId, liquidatee, {
        amount: 0,
        vQuoteBalance: -1e18,     // 1 USDC bad debt
    });

    // 3. Call socializeSubaccount with zero insurance
    await expect(
        perpEngine.connect(clearinghouse).socializeSubaccount(liquidatee, 0)
    ).to.be.revertedWith("DBZ");   // MathSD21x18 ERR_DIV_BY_ZERO
});
```

The revert at `MathSD21x18.div` with `"DBZ"` is deterministic and reproducible on an unmodified local Hardhat fork whenever `state.openInterest == 0` and `balance.vQuoteBalance < 0` after insurance. [6](#0-5) [7](#0-6)

        external
        returns (int128)
    {

            uint32 productId = _productIds[i];
            (State memory state, Balance memory balance) = getStateAndBalance(
                productId,
                subaccount
            );
            if (balance.vQuoteBalance < 0) {
                int128 insuranceCover = MathHelper.min(
                    insurance,
                    -balance.vQuoteBalance
                );
                insurance -= insuranceCover;
                balance.vQuoteBalance += insuranceCover;
                state.availableSettle += insuranceCover;

                // actually socialize if still not enough

                        state.openInterest

                    balance.vQuoteBalance = 0;
                }
                _setState(productId, state);
                _setBalanceAndUpdateBitmap(productId, subaccount, balance);
            }
        }
        return insurance;
    }
