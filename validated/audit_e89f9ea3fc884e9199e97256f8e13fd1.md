Audit Report

## Title
`SwapAllowlistExtension.beforeSwap` checks router address instead of actual user, enabling allowlist bypass — (`metric-periphery/contracts/extensions/SwapAllowlistExtension.sol`)

## Summary
`SwapAllowlistExtension.beforeSwap` receives `sender` from the pool, which is `msg.sender` of the pool's `swap()` call — the router contract, not the end user. Any pool admin who allowlists the router (required for allowlisted users to use `MetricOmmSimpleRouter`) simultaneously grants every unpermissioned user the ability to bypass the swap allowlist by routing through the same router.

## Finding Description
The call chain is confirmed in production code:

1. `MetricOmmSimpleRouter.exactInputSingle` calls `pool.swap()` directly with no user identity forwarded in `extensionData`: [1](#0-0) 

2. `MetricOmmPool.swap` passes `msg.sender` (the router) as `sender` to `_beforeSwap`: [2](#0-1) 

3. `ExtensionCalling._beforeSwap` forwards that `sender` value unchanged to the extension: [3](#0-2) 

4. `SwapAllowlistExtension.beforeSwap` evaluates `allowedSwapper[msg.sender (pool)][sender (router)]`, never touching the real user's address: [4](#0-3) 

The pool admin faces an impossible choice: not allowlisting the router blocks all allowlisted users from using `MetricOmmSimpleRouter`; allowlisting the router opens the gate to every address. There is no configuration that simultaneously permits router-mediated swaps for allowlisted users and blocks them for non-allowlisted users.

`DepositAllowlistExtension` does not share this flaw because it checks `owner` (the position owner supplied by the caller), not `sender`: [5](#0-4) 

## Impact Explanation
A pool configured with `SwapAllowlistExtension` to restrict trading to a curated set (e.g., KYC'd counterparties, whitelisted market makers) can be bypassed by any unpermissioned user routing through `MetricOmmSimpleRouter`. The bypassing user executes swaps at oracle-derived prices against LP funds, extracting value the pool admin intended to restrict. This is a direct, repeatable loss of LP principal protection on restricted pools — matching the "broken core pool functionality causing loss of funds" and "admin-boundary break by an unprivileged path" allowed impacts.

## Likelihood Explanation
`MetricOmmSimpleRouter` is the standard documented periphery entry point. Any pool admin who wants allowlisted users to use the router must allowlist the router address. Once the router is allowlisted, the bypass requires only a single public call to `exactInputSingle` or `exactInput` with no special capital or knowledge beyond the swap amount. The condition is trivially met on any restricted pool where the router is allowlisted.

## Recommendation
The extension must resolve the actual end-user identity rather than the immediate `msg.sender` of the pool call. Two sound approaches:

1. **Forward the real initiator via `extensionData`.** Have `MetricOmmSimpleRouter` encode `msg.sender` (the real user) into `extensionData` using a protocol-level convention, and have `SwapAllowlistExtension.beforeSwap` decode and check that field when `sender` is a known router.

2. **Router registry + transient storage attestation.** Maintain a registry of trusted routers. When `sender` is a registered router, query the router's transient storage (via `_getPayer()`) for the real initiator and check that address against the allowlist instead of `sender`.

## Proof of Concept
1. Deploy pool P with `SwapAllowlistExtension` as `EXTENSION_1` on the `beforeSwap` order.
2. Pool admin calls `setAllowedToSwap(P, Alice, true)` and `setAllowedToSwap(P, router, true)` so Alice can use the router.
3. Bob (not allowlisted) calls `MetricOmmSimpleRouter.exactInputSingle({pool: P, ...})`.
4. Router calls `P.swap(recipient=Bob, ...)` with `msg.sender = router`.
5. Pool calls `_beforeSwap(sender=router, ...)`.
6. `SwapAllowlistExtension.beforeSwap` evaluates `allowedSwapper[P][router] == true` → passes.
7. Bob's swap executes at oracle price against LP funds.

**Expected:** revert `NotAllowedToSwap`. **Actual:** swap succeeds.
