### Title
Permanent Engine Assignment with No Removal Mechanism Prevents Protocol Recovery from Compromised Engine - (File: core/contracts/Clearinghouse.sol)

### Summary
`Clearinghouse.addEngine()` enforces a one-time-only assignment of a product engine to an engine type via a hard `require` guard. No `removeEngine()` or `replaceEngine()` function exists anywhere in the protocol. If a registered `SpotEngine` or `PerpEngine` is exploited or must be replaced, the protocol has no on-chain mechanism to recover.

### Finding Description
In `Clearinghouse.sol`, the `addEngine()` function contains the following guard:

```solidity
require(address(engineByType[engineType]) == address(0));
``` [1](#0-0) 

This check enforces that each `EngineType` slot (`SPOT` or `PERP`) can only ever be assigned once. Once written, the `engineByType` mapping entry is permanent. [2](#0-1) 

A search across the entire codebase confirms there is no `removeEngine`, `deleteEngine`, `updateEngine`, or `replaceEngine` function in `Clearinghouse.sol`, `ClearinghouseStorage.sol`, `ContractOwner.sol`, or `IClearinghouse.sol`. The `ContractOwner.addEngine()` wrapper simply delegates to `clearinghouse.addEngine()` with no additional override path. [3](#0-2) 

The `IClearinghouse` interface exposes `addEngine` but defines no removal counterpart. [4](#0-3) 

### Impact Explanation
`SpotEngine` and `PerpEngine` are the sole handlers for all user collateral balances, position accounting, health calculations, and liquidation logic. Every deposit, withdrawal, trade settlement, and liquidation flows through the engine registered in `engineByType`. If either engine is exploited — draining balances, corrupting `vQuoteBalance`, or manipulating health weights — the protocol owner has no on-chain path to swap in a patched engine. The compromised engine remains permanently authoritative. All user funds and open positions across the protocol are irreversibly exposed for the lifetime of the deployment. [5](#0-4) 

### Likelihood Explanation
The Nado protocol is deployed on Ink Chain and manages real user collateral across spot and perpetual products. Engine contracts are complex, upgradeable-adjacent logic targets. A vulnerability in `SpotEngine` or `PerpEngine` (e.g., in `updateBalance`, `assertUtilization`, or health contribution logic) is a realistic attack surface. The absence of a removal path means any such exploit becomes permanent and unrecoverable at the `Clearinghouse` layer, even if the engine itself is upgradeable via proxy.

### Recommendation
Add a privileged `removeEngine(IProductEngine.EngineType engineType)` function to `Clearinghouse.sol` that clears the `engineByType[engineType]` slot and removes the type from `supportedEngines`. Guard it with `onlyOwner`. To replace an engine, the owner would call `removeEngine` followed by `addEngine` with the patched address. Ensure that `productToEngine` entries pointing to the old engine are also cleared or remapped as part of the removal flow.

### Proof of Concept
1. Owner calls `ContractOwner.addEngine(spotEngineAddr, offchainExchange, EngineType.SPOT)`.
2. `Clearinghouse.addEngine` passes the `require(address(engineByType[SPOT]) == address(0))` check and permanently writes `engineByType[SPOT] = spotEngineAddr`.
3. A critical vulnerability is discovered in `SpotEngine` (e.g., `updateBalance` can be called to inflate a subaccount balance).
4. Owner attempts to call `addEngine` again with a patched engine address — transaction reverts because `engineByType[SPOT] != address(0)`.
5. No `removeEngine` function exists. The compromised `SpotEngine` remains the sole authority for all spot balance accounting indefinitely. [6](#0-5) [2](#0-1)

    function addEngine(
        address engine,
        address offchainExchange,
        IProductEngine.EngineType engineType
    ) external onlyOwner {
        require(address(engineByType[engineType]) == address(0));
        require(engine != address(0));
        IProductEngine productEngine = IProductEngine(engine);
        // Register
        supportedEngines.push(engineType);
        engineByType[engineType] = productEngine;

        // add quote to product mapping
        if (engineType == IProductEngine.EngineType.SPOT) {
            productToEngine[QUOTE_PRODUCT_ID] = productEngine;
        }

        // Initialize engine
        productEngine.initialize(
            address(this),
            offchainExchange,
            quote,
            getEndpoint(),
            owner()
        );
    }
```
