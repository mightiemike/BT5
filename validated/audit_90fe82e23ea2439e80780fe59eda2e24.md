### Title
Spread Health Adjustment Uses Wrong Weight Direction, Corrupting Collateral Accounting — (`core/contracts/Clearinghouse.sol`)

---

### Summary

`Clearinghouse.getHealth` computes a spread health adjustment by constructing an `existingWeight` from `getCoreRisk`, which always returns the **long** weight regardless of the actual position direction. For a long spread (long spot, short perp), the perp's `shortWeight` should be used, but `longWeight` is used instead. The resulting `existingWeight` is systematically wrong, causing the spread health credit to be overestimated. Any user holding a spread position can withdraw more collateral or open larger positions than the protocol intends, creating undercollateralized accounts.

---

### Finding Description

`getCoreRisk` in `BaseEngine.sol` always passes the literal `1` as the `amount` argument to `_getWeightX18`: [1](#0-0) 

```solidity
return IProductEngine.CoreRisk(
    amount,
    risk.priceX18,
    RiskHelper._getWeightX18(risk, 1, healthType)   // ← always longWeight
);
```

`_getWeightX18` branches on the sign of `amount`: [2](#0-1) 

Because `1 > 0`, it always returns `longWeightInitialX18` / `longWeightMaintenanceX18`, never the short-side weight.

`Clearinghouse.getHealth` then builds `existingWeight` from the two `longWeight` values returned by `getCoreRisk`: [3](#0-2) 

```solidity
int128 existingWeight = (spotCoreRisk.longWeight +
    perpCoreRisk.longWeight) / 2;
...
health += basisAmount
    .mul(spotCoreRisk.price + perpCoreRisk.price)
    .mul(spreadWeight - existingWeight);
```

For a **long spread** (`basisAmount > 0`: long spot, short perp):

| Component | Weight actually used in `_calculateProductHealth` | Weight used in `existingWeight` |
|---|---|---|
| Spot (long) | `spotLongWeight` | `spotLongWeight` ✓ |
| Perp (short) | `perpShortWeight` | `perpLongWeight` ✗ |

Because `perpLongWeight < perpShortWeight` (long positions receive a discount, short positions carry a penalty), `existingWeight` is **underestimated**. Consequently `spreadWeight − existingWeight` is **overestimated**, and the spread health credit added to `health` is larger than the protocol intends.

The same asymmetry applies to a **short spread** (short spot, long perp), where `spotLongWeight` is used instead of `spotShortWeight`.

---

### Impact Explanation

The spread health credit is inflated for every subaccount holding a spread position. A user can:

1. Open a spread position (long spot + short perp of the same underlying).
2. Call `withdrawCollateral`, `transferQuote`, `mintNlp`, or `burnNlp` — all of which gate on `getHealth`.
3. Receive a health value that is higher than the true risk-adjusted value.
4. Withdraw more collateral or open additional positions beyond what the protocol's risk parameters allow.

The resulting undercollateralization cannot be detected by the liquidation system (which uses the same `getHealth` path), so bad debt accumulates silently until positions move against the user. [4](#0-3) [5](#0-4) 

---

### Likelihood Explanation

Any user who holds a spread position (a common strategy in a perp DEX) is affected on every health check. The entry path requires no special privileges: a user deposits collateral, opens a spot long and a matching perp short through the normal `matchOrders` flow, and then calls `withdrawCollateral`. The bug is triggered unconditionally whenever `getHealth` iterates over the `spreads` bitmask. [6](#0-5) 

---

### Recommendation

`getCoreRisk` must pass the **actual position amount** (not the constant `1`) to `_getWeightX18` so that the returned weight reflects the real direction of the position:

```solidity
// BaseEngine.sol – getCoreRisk
return IProductEngine.CoreRisk(
    amount,
    risk.priceX18,
    RiskHelper._getWeightX18(risk, amount, healthType)  // use actual amount
);
```

Alternatively, expose both `longWeight` and `shortWeight` in `CoreRisk` and select the correct one in `Clearinghouse.getHealth` based on the sign of `basisAmount` for each leg.

---

```text
    function _getWeightX18(
        Risk memory risk,
        int128 amount,
        IProductEngine.HealthType healthType
    ) internal pure returns (int128) {
        if (healthType == IProductEngine.HealthType.PNL) {
            return ONE;
        }

        int128 weight;
        if (amount >= 0) {
            weight = healthType == IProductEngine.HealthType.INITIAL
                ? risk.longWeightInitialX18
                : risk.longWeightMaintenanceX18;
        } else {
            weight = healthType == IProductEngine.HealthType.INITIAL
                ? risk.shortWeightInitialX18
                : risk.shortWeightMaintenanceX18;
        }

        return weight;
    }
```
