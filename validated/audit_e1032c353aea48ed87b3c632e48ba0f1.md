### Title
Unchecked `fee_proposal_fri` During Startup Window Enables Permanent Gas Price Corruption and Fee Calculation Overflow — (`crates/apollo_consensus_orchestrator/src/validate_proposal.rs`)

### Summary

The `fee_proposal_fri` bounds check in `is_proposal_init_valid` is gated on `fee_actual` being `Some`. During the first `window_size` blocks (startup), `fee_actual` is `None` and the bounds check is entirely skipped. A malicious proposer controlling a majority of startup proposer slots can inject `fee_proposal_fri = u128::MAX` into the sliding window. Once the window fills, `compute_fee_actual` returns `u128::MAX` as the median. This value becomes the `effective_min` floor in `calculate_next_l2_gas_price_for_fin`, which passes `min_gas_price = u128::MAX` to `calculate_next_base_gas_price`. As the gas price ramps toward `u128::MAX`, the unchecked addition `price.0 + max_increase` overflows `u128` (wraps in release mode), and any transaction consuming moderate L2 gas causes `GasVector::cost()` to panic on overflow — permanently disrupting block execution.

### Finding Description

**Root cause — bounds check skipped during startup:**

In `is_proposal_init_valid`, the `fee_proposal_fri` value is validated in two steps. The version gate (step 1) is always enforced. The bounds check (step 2) is only applied when `fee_actual` is `Some`:

```rust
// Validate fee_proposal is within the configured margin of fee_actual.
// During initiation (fee_actual is None, <window_size blocks), bounds are not enforced.
if let (Some(fee_actual), Some(fee_proposal)) =
    (proposal_init_validation.fee_actual, init_proposed.fee_proposal_fri)
{
    let (lower_bound, upper_bound) = fee_proposal_bounds(
        fee_actual,
        VersionedConstants::latest_constants().fee_proposal_margin_ppt,
    );
    if fee_proposal.0 < lower_bound || fee_proposal.0 > upper_bound {
        return Err(...);
    }
}
``` [1](#0-0) 

`fee_actual` is `None` for the first `window_size` blocks because `compute_fee_actual` requires a full window:

```rust
let Some(start) = height.0.checked_sub(window_size) else {
    warn!("Cannot compute fee_actual for height {height}: height is below window_size ({window_size})");
    return None;
};
``` [2](#0-1) 

During this startup window, any value of `fee_proposal_fri` — including `u128::MAX` — passes validation.

**Downstream propagation — `fee_actual` becomes the gas price floor:**

After the window fills, `compute_fee_actual` returns the median of the injected values. If a majority of startup blocks carry `u128::MAX`, the median is `u128::MAX`. This value is then used as the `effective_min` floor:

```rust
let effective_min = match fee_actual {
    Some(fa) => GasPrice(max(config_min.0, fa.0)),
    None => config_min,
};
calculate_next_base_gas_price(current_l2_gas_price, l2_gas_used, gas_target, effective_min)
``` [3](#0-2) 

**Integer overflow in `calculate_next_base_gas_price`:**

With `min_gas_price = u128::MAX`, the gas price is always below the minimum, so the gradual-increase branch is always taken:

```rust
if price < min_gas_price {
    let max_increase = price.0 / MIN_GAS_PRICE_INCREASE_DENOMINATOR;
    let adjusted = price.0 + max_increase;   // ← plain u128 add, no overflow check
    let adjusted_price = adjusted.min(min_gas_price.0);
    return GasPrice(adjusted_price);
}
``` [4](#0-3) 

When `price` approaches `u128::MAX - 1`, `price.0 + price.0/333` overflows `u128`. In Rust release mode this wraps, producing a value near `10^36`. The comment "Cap at min_gas_price to avoid overshooting" is defeated: `adjusted.min(u128::MAX)` returns the wrapped small value, not `u128::MAX`. The gas price then oscillates at values around `10^36` FRI.

**Panic in `GasVector::cost()`:**

With `l2_gas_price ≈ 10^36`, any transaction consuming more than `u128::MAX / 10^36 ≈ 333` L2 gas units causes `GasVector::cost()` to panic:

```rust
let cost = gas.checked_mul(price.get()).unwrap_or_else(|| {
    panic!(
        "{resource} cost overflowed: multiplication of gas amount ({gas}) by price \
         per unit ({price}) resulted in overflow."
    )
});
``` [5](#0-4) 

A typical transaction uses far more than 333 L2 gas units, so this panic is triggered for virtually all transactions.

**`fee_proposal_fri` is not in the block hash:**

The `BlockHeaderWithoutHash` struct carries a TODO noting that `fee_proposal_fri` is not yet included in the block hash:

```rust

``` [6](#0-5) 

This means the injected extreme value does not affect the block hash, making the attack undetectable from the hash alone.
