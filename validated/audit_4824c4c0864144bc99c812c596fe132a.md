### Title
IBC-origin NFTs loaded via genesis are permanently locked — `MsgTransferNFT`/`MsgBurnNFT`/`MsgEditNFT` all call `ValidateDenomID` instead of `ValidateDenomIDWithIBC` - (`x/nft/types/msgs.go`, `x/nft/types/genesis.go`)

### Summary

`ValidateGenesis` accepts denom IDs of the form `ibc/<64-hex-chars>` via `ValidateDenomIDWithIBC`, but every mutation message (`MsgTransferNFT`, `MsgBurnNFT`, `MsgEditNFT`) calls the stricter `ValidateDenomID` in `ValidateBasic`, which unconditionally rejects any string containing `/` or exceeding 64 characters. An NFT stored under such a denom can never be transferred, burned, or edited by its owner.

### Finding Description

**Genesis acceptance path:**

`ValidateGenesis` calls `ValidateDenomIDWithIBC`, which has an explicit IBC fast-path: [1](#0-0) [2](#0-1) 

`ibc/<64-hex-chars>` is 68 characters long and passes this check cleanly.

**Mutation rejection path:**

All three mutation messages call the strict `ValidateDenomID`: [3](#0-2) [4](#0-3) [5](#0-4) 

`ValidateDenomID` rejects `ibc/HASH` on two independent grounds:

1. **Length**: `IBCDenomLen = 68 > MaxDenomLen = 64` — fails the length check immediately.
2. **Character set**: `IsAlphaNumeric` is `^[a-z0-9]+$` — the `/` in `ibc/` is not alphanumeric. [6](#0-5) 

### Impact Explanation

Any NFT whose denom was loaded into genesis (or migrated) under an `ibc/<hash>` denom ID is permanently immovable at the `ValidateBasic` layer. The owner cannot:
- Transfer it (`MsgTransferNFT.ValidateBasic` → `ErrInvalidDenom`)
- Burn it (`MsgBurnNFT.ValidateBasic` → `ErrInvalidDenom`)
- Edit it (`MsgEditNFT.ValidateBasic` → `ErrInvalidDenom`)

There is no alternative authorized exit path in the production code. The NFT is locked in the owner's account with no recourse short of a governance upgrade.

### Likelihood Explanation

The `x/nft-transfer` module is the intended production path for IBC NFTs. When an IBC NFT packet is received and the class/denom does not exist locally, the nft-transfer module mints it under an `ibc/<hash>` denom. If the chain is ever exported and re-imported via genesis (a standard upgrade/migration procedure), `ValidateGenesis` accepts those denoms, `InitGenesis` stores them, and the lock is permanent from that point forward. This is a realistic, reachable production scenario.

### Recommendation

Replace `ValidateDenomID` with `ValidateDenomIDWithIBC` in the `ValidateBasic` methods of `MsgTransferNFT`, `MsgEditNFT`, and `MsgBurnNFT` in `x/nft/types/msgs.go`. The IBC-aware validator already exists and is used correctly in `ValidateGenesis` and the CLI query commands — it simply needs to be applied consistently to all mutation messages.

### Proof of Concept

```
1. Export chain state containing an NFT with DenomId = "ibc/0123456789ABCDEF...{64 hex chars}"
2. Re-import via InitGenesis — ValidateGenesis calls ValidateDenomIDWithIBC → passes
3. NFT is stored in keeper under denom "ibc/0123456789ABCDEF..."
4. Owner constructs MsgTransferNFT{DenomId: "ibc/0123456789ABCDEF...", ...}
5. ValidateBasic calls ValidateDenomID("ibc/0123456789ABCDEF...")
   → len("ibc/"+64chars) = 68 > MaxDenomLen(64) → ErrInvalidDenom
6. Transaction is rejected at mempool/ante; NFT is permanently locked.
```

The same rejection occurs for `MsgBurnNFT` and `MsgEditNFT` via the identical `ValidateDenomID` call in their respective `ValidateBasic` implementations.

```go
func ValidateDenomID(denomID string) error {
	if len(denomID) < MinDenomLen || len(denomID) > MaxDenomLen {
		return sdkerrors.Wrapf(ErrInvalidDenom, "the length of denom(%s) only accepts value [%d, %d]", denomID, MinDenomLen, MaxDenomLen)
	}
	if !IsBeginWithAlpha(denomID) || !IsAlphaNumeric(denomID) {
		return sdkerrors.Wrapf(ErrInvalidDenom, "the denom(%s) only accepts lowercase alphanumeric characters, and begin with an english letter", denomID)
	}
	return nil
}
```

```go
func ValidateDenomIDWithIBC(denomID string) error {
	if strings.HasPrefix(denomID, IBCPrefix) {
		if len(denomID) != IBCDenomLen {
			return sdkerrors.Wrapf(ErrInvalidDenom, "the length of ibc denom(%s) only accepts value [%d]", denomID, IBCDenomLen)
		}
		if _, err := hex.DecodeString(denomID[len(IBCPrefix):]); err != nil {
			return sdkerrors.Wrapf(ErrInvalidDenom, "the hash of ibc denom(%s) must be valid hex", denomID)
		}

		return nil
	}

	return ValidateDenomID(denomID)
```
