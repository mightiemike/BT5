### Title
Trailing Bytes Silently Accepted by `node_from_bytes` / `node_from_bytes_backrefs` — Non-Canonical Serialization and Consensus Divergence - (File: `src/serde/de.rs`, `src/serde/de_br.rs`)

---

### Summary

`node_from_bytes` and `node_from_bytes_backrefs` parse exactly one CLVM S-expression from a caller-supplied `&[u8]` slice but never verify that the cursor reached the end of the slice. Any trailing bytes after the first valid CLVM object are silently discarded and the call returns `Ok`. This creates non-canonical serialization (multiple distinct byte strings decode to the same program) and opens a consensus-divergence path if any other implementation or validation layer rejects trailing bytes.

---

### Finding Description

`node_from_bytes` wraps a `Cursor` over the full input slice and delegates to `node_from_stream`, which stops as soon as one complete S-expression has been read:

```rust
// src/serde/de.rs  lines 44-46
pub fn node_from_bytes(allocator: &mut Allocator, b: &[u8]) -> Result<NodePtr> {
    let mut buffer = Cursor::new(b);
    node_from_stream(allocator, &mut buffer)   // cursor position never checked afterward
}
```

`node_from_stream` exits its loop the moment the `ops` stack is empty — i.e., after one root node is fully parsed — without asserting `buffer.position() == b.len() as u64`:

```rust
// src/serde/de.rs  lines 16-42
pub fn node_from_stream(allocator: &mut Allocator, f: &mut Cursor<&[u8]>) -> Result<NodePtr> {
    let mut values: Vec<NodePtr> = Vec::new();
    let mut ops = vec![ParseOp::SExp];
    let mut b = [0; 1];
    while let Some(op) = ops.pop() { /* ... */ }
    Ok(values.pop().unwrap())   // returns without checking remaining bytes
}
```

The same structural omission exists in `node_from_bytes_backrefs`:

```rust
// src/serde/de_br.rs  lines 114-117
pub fn node_from_bytes_backrefs(allocator: &mut Allocator, b: &[u8]) -> Result<NodePtr> {
    let mut buffer = Cursor::new(b);
    node_from_stream_backrefs(allocator, &mut buffer, |_node| {})
    // cursor position never checked
}
```

The Python-facing `deser_auto` entry point in `wheel/src/api.rs` calls `node_from_bytes_backrefs` directly, so the same gap is exposed to every Python caller:

```rust
// wheel/src/api.rs  lines 153-155
node_from_bytes_backrefs(&mut a, blob).map_err(eval_to_py)?
```

Concrete trigger: supplying `[0x80, 0xAB]` to `node_from_bytes` parses `0x80` as NIL, returns `Ok(NIL)`, and silently discards `0xAB`. Any byte string of the form `<valid_clvm> || <arbitrary_suffix>` is accepted without error.

The `deserialize_2026` path has the same property: `deserialize_2026_body_from_stream` reads the structured header and instruction stream but does not assert the reader is exhausted before returning.

---

### Impact Explanation

**Non-canonical serialization.** For every valid CLVM program `P` with canonical encoding `B`, the infinite family `B || suffix` (for any non-empty `suffix`) is also accepted and produces the same `NodePtr`. This breaks the one-to-one correspondence between byte strings and programs that callers and protocol layers rely on.

**Consensus divergence.** If any other CLVM implementation (e.g., the Python `clvm` reference library, or a future strict-mode validator) rejects inputs with trailing bytes, the two implementations will disagree on whether a given serialized puzzle or generator is well-formed. A block containing a puzzle serialized as `canonical_bytes || garbage` would be accepted by `clvm_rs`-based full nodes and rejected by strict-mode nodes, causing a chain split.

**Caller confusion / silent data loss.** A caller that passes a bounded slice expecting full-slice validation (e.g., a network-message parser that embeds a CLVM program followed by a signature) will receive `Ok` even when the slice contains extra bytes, masking framing errors and potentially allowing malformed inputs to propagate.

---

### Likelihood Explanation

The `node_from_bytes` and `node_from_bytes_backrefs` functions are the primary deserialization entry points used throughout the codebase and exposed via the Python wheel. Any network path that delivers attacker-controlled bytes to these functions — block generators, puzzle reveals, solution bytes — is a reachable trigger. An attacker who can craft a serialized CLVM object with appended garbage bytes can exercise this path without any special privilege.

---

### Recommendation

After `node_from_stream` (and `node_from_stream_backrefs`) return, assert that the cursor has consumed the entire input:

```rust
pub fn node_from_bytes(allocator: &mut Allocator, b: &[u8]) -> Result<NodePtr> {
    let mut buffer = Cursor::new(b);
    let node = node_from_stream(allocator, &mut buffer)?;
    if buffer.position() != b.len() as u64 {
        return Err(EvalErr::SerializationError);
    }
    Ok(node)
}
```

Apply the same fix to `node_from_bytes_backrefs`. For `deserialize_2026_body_from_stream`, after the instruction loop, verify the reader is exhausted (e.g., attempt a one-byte read and confirm `UnexpectedEof`). Document explicitly whether stream-mode callers are expected to bound the reader themselves.

pub fn node_from_stream(allocator: &mut Allocator, f: &mut Cursor<&[u8]>) -> Result<NodePtr> {
    let mut values: Vec<NodePtr> = Vec::new();
    let mut ops = vec![ParseOp::SExp];

    let mut b = [0; 1];
    while let Some(op) = ops.pop() {
        match op {
            ParseOp::SExp => {
                f.read_exact(&mut b)?;
                if b[0] == CONS_BOX_MARKER {
                    ops.push(ParseOp::Cons);
                    ops.push(ParseOp::SExp);
                    ops.push(ParseOp::SExp);
                } else {
                    values.push(parse_atom(allocator, b[0], f)?);
                }

                // cons

                let v1 = values.pop();
                values.push(allocator.new_pair(v1.unwrap(), v2.unwrap())?);
            }
        }
    }
    Ok(values.pop().unwrap())
}

pub fn deserialize_2026_body_from_stream<R: Read>(
    allocator: &mut Allocator,
    reader: &mut R,
    max_atom_len: usize,
    strict: bool,
) -> Result<NodePtr> {
    let mut atoms: Vec<NodePtr> = Vec::new();
    let group_count = checked_usize(read_varint(reader, strict)?)?;
    let mut buf: Vec<u8> = Vec::new();

    for _ in 0..group_count {
        let length_val = read_varint(reader, strict)?;
        let (length, count) = if length_val < 0 {
            if length_val == i64::MIN {
                return Err(EvalErr::SerializationError);
            }
            (
                checked_bounded_usize(-length_val, max_atom_len)?,
                checked_usize(read_varint(reader, strict)?)?,
            )
        } else {
            (checked_bounded_usize(length_val, max_atom_len)?, 1)
        };
        if length == 0 || count == 0 {
            return Err(EvalErr::SerializationError);
        }
        buf.resize(length, 0);
        for _ in 0..count {
            reader
                .read_exact(&mut buf)
                .map_err(|_| EvalErr::SerializationError)?;
            atoms.push(allocator.new_atom(&buf)?);
        }
    }

    let instruction_count = checked_usize(read_varint(reader, strict)?)?;
    if instruction_count == 0 {
        return Err(EvalErr::SerializationError);
    }

    let nil = allocator.nil();
    let mut pairs: Vec<NodePtr> = Vec::new();
    let mut stack: Vec<NodePtr> = Vec::with_capacity(64);

    for _ in 0..instruction_count {
        let inst = read_varint(reader, strict)?;
        match inst {
            0 => stack.push(nil),
            1 => {
                if stack.len() < 2 {
                    return Err(EvalErr::SerializationError);
                }
                let right = stack.pop().unwrap();
                let left = stack.pop().unwrap();
                let pair = allocator.new_pair(left, right)?;
                pairs.push(pair);
                stack.push(pair);
            }
            -1 => {
                if stack.len() < 2 {
                    return Err(EvalErr::SerializationError);
                }
                let left = stack.pop().unwrap();
                let right = stack.pop().unwrap();
                let pair = allocator.new_pair(left, right)?;
                pairs.push(pair);
                stack.push(pair);
            }
            n if n >= 2 => {
                let ai = (n - 2) as usize;
                stack.push(*atoms.get(ai).ok_or(EvalErr::SerializationError)?);
            }
            n => {
                let pi = n
                    .checked_neg()
                    .and_then(|x| x.checked_sub(2))
                    .ok_or(EvalErr::SerializationError)? as usize;
                stack.push(*pairs.get(pi).ok_or(EvalErr::SerializationError)?);
            }
        }
    }

    if stack.len() != 1 {
        return Err(EvalErr::SerializationError);
    }
    Ok(stack[0])
}
