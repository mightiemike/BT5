### Title
Lossy `ValidResourceBounds` Protobuf Deserialization Produces Wrong Transaction Hash for `AllResources` Transactions with Zero L2/Data Gas - (`crates/apollo_protobuf/src/converters/transaction.rs`)

### Summary

The `TryFrom<protobuf::ResourceBounds> for ValidResourceBounds` conversion in the block-sync protobuf layer uses a zero-value heuristic to collapse `AllResources` into `L1Gas`. Because `get_tip_resource_bounds_hash` produces structurally different hash preimages for these two variants—even when the numeric gas values are identical—any `InvokeTransactionV3` submitted with `AllResourceBounds { l2_gas: 0, l1_data_gas: 0 }` will have its transaction hash recomputed to a different value after a protobuf round-trip, breaking hash validation on syncing nodes.

### Finding Description

**Conversion boundary — `transaction.rs`:**

```rust
// crates/apollo_protobuf/src/converters/transaction.rs
Ok(if l1_data_gas.is_zero() && l2_gas.is_zero() {
    ValidResourceBounds::L1Gas(l1_gas)          // ← drops AllResources variant
} else {
    ValidResourceBounds::AllResources(AllResourceBounds { l1_gas, l2_gas, l1_data_gas })
})
``` [1](#0-0) 

When a sequencer-produced `InvokeTransactionV3` carries `ValidResourceBounds::AllResources` with `l2_gas = 0` and `l1_data_gas = 0`, the serializer emits all three protobuf fields as zero. The deserializer then silently downgrades the variant to `ValidResourceBounds::L1Gas`, discarding the original type information.

**Hash preimage divergence — `transaction_hash.rs`:**

`get_tip_resource_bounds_hash` branches on the variant:

```rust
// crates/starknet_api/src/transaction_hash.rs
resource_felts.extend(match resource_bounds {
    ValidResourceBounds::L1Gas(_) => vec![],                          // 2 elements
    ValidResourceBounds::AllResources(all_resources) => {
        vec![get_concat_resource(&all_resources.l1_data_gas, L1_DATA_GAS)?]  // 3 elements
    }
});
``` [2](#0-1) 

`AllResources` always appends a third element `concat(L1_DATA_GAS, 0)` to the poseidon hash input, even when the value is zero. `L1Gas` does not. The resulting `tip_resource_bounds_hash` values are therefore structurally different, and the full transaction hash diverges.

**Origination path — `InternalRpcInvokeTransactionV3`:**

The internal representation always stores `AllResourceBounds` (not `ValidResourceBounds`), and its `InvokeTransactionV3Trait::resource_bounds()` always returns `ValidResourceBounds::AllResources(...)`:

```rust
impl InvokeTransactionV3Trait for InternalRpcInvokeTransactionV3 {
    fn resource_bounds(&self) -> ValidResourceBounds {
        ValidResourceBounds::AllResources(self.resource_bounds)
    }
``` [3](#0-2) 

So the sequencer always computes H1 (AllResources path, 3 elements). After block-sync protobuf deserialization, the same transaction is represented as `ValidResourceBounds::L1Gas`, and `get_invoke_transaction_v3_hash` computes H2 (L1Gas path, 2 elements). H1 ≠ H2. [4](#0-3) 

### Impact Explanation

A syncing node that recomputes transaction hashes from deserialized `Transaction` objects (via `validate_transaction_hash`) will obtain H2 ≠ H1 for every invoke V3 transaction with `AllResourceBounds { l2_gas: 0, l1_data_gas: 0 }`. This causes:

- **Wrong transaction hash committed to storage and served by RPC** — the RPC returns H2 for a transaction whose canonical hash is H1, an authoritative-looking wrong value.
- **Transaction commitment mismatch** — the Merkle root over transaction hashes computed by the syncing node will differ from the one in the block header, causing valid blocks to be rejected.

This matches: *"RPC execution, fee estimation, tracing, simulation, or pending view returns an authoritative-looking wrong value"* and *"Wrong state, receipt, event, L1 message, class hash, storage value, or revert result."*

### Likelihood Explanation

Any unprivileged user can submit an `RpcInvokeTransactionV3` with `AllResourceBounds { l1_gas: X, l2_gas: 0, l1_data_gas: 0 }`. The gateway's stateless validator accepts this (non-zero l1_gas satisfies the resource-bounds check). The sequencer includes it in a block. Every syncing peer that processes that block via p2p block sync will hit the divergence. No special privileges or malformed bytes are required.

### Recommendation

Remove the zero-value heuristic from `TryFrom<protobuf::ResourceBounds> for ValidResourceBounds`. Either:

1. **Always return `AllResources`** when all three protobuf fields are present (regardless of whether l2_gas and l1_data_gas are zero), preserving the original variant; or
2. **Add an explicit discriminant field** to the protobuf `ResourceBounds` message to encode which `ValidResourceBounds` variant was serialized.

The serializer in `From<ValidResourceBounds> for protobuf::ResourceBounds` already emits all three fields for `AllResources`; the deserializer must not silently collapse them. [5](#0-4) 

### Proof of Concept

**Step 1.** Submit an invoke V3 transaction with `AllResourceBounds { l1_gas: ResourceBounds { max_amount: 100, max_price_per_unit: 1 }, l2_gas: ResourceBounds::default(), l1_data_gas: ResourceBounds::default() }`.

**Step 2.** The sequencer's `InternalRpcInvokeTransactionV3::resource_bounds()` returns `ValidResourceBounds::AllResources(...)`. `get_tip_resource_bounds_hash` computes:

```
H_fee_1 = poseidon(tip, concat(L1_GAS, 100, 1), concat(L2_GAS, 0, 0), concat(L1_DATA_GAS, 0, 0))
```

The full transaction hash H1 is committed to the block.

**Step 3.** The block is propagated via p2p block sync. The protobuf payload carries `l1_gas={100,1}`, `l2_gas={0,0}`, `l1_data_gas={0,0}`.

**Step 4.** `TryFrom<protobuf::ResourceBounds> for ValidResourceBounds` sees `l1_data_gas.is_zero() && l2_gas.is_zero()` → returns `ValidResourceBounds::L1Gas(ResourceBounds { max_amount: 100, max_price_per_unit: 1 })`.

**Step 5.** The syncing node calls `get_tip_resource_bounds_hash` with `L1Gas`:

```
H_fee_2 = poseidon(tip, concat(L1_GAS, 100, 1), concat(L2_GAS, 0, 0))
```

`H_fee_2 ≠ H_fee_1` because the third element `concat(L1_DATA_GAS, 0, 0)` is absent.

**Step 6.** The full transaction hash H2 ≠ H1. `validate_transaction_hash` returns `false`. The syncing node rejects the block or stores the wrong hash. [6](#0-5) [1](#0-0)

impl TryFrom<protobuf::ResourceBounds> for ValidResourceBounds {
    type Error = ProtobufConversionError;
    fn try_from(value: protobuf::ResourceBounds) -> Result<Self, Self::Error> {
        let Some(l1_gas) = value.l1_gas else {
            return Err(missing("ResourceBounds::l1_gas"));
        };
        let Some(l2_gas) = value.l2_gas else {
            return Err(missing("ResourceBounds::l2_gas"));
        };
        // TODO(Shahak): Assert data gas is not none once we remove support for 0.13.2.

        let l2_gas: ResourceBounds = l2_gas.try_into()?;
        let l1_data_gas: ResourceBounds = l1_data_gas.try_into()?;

            ValidResourceBounds::L1Gas(l1_gas)

pub fn get_tip_resource_bounds_hash(
    resource_bounds: &ValidResourceBounds,
    tip: &Tip,
) -> Result<Felt, StarknetApiError> {
    let l1_resource_bounds = resource_bounds.get_l1_bounds();
    let l2_resource_bounds = resource_bounds.get_l2_bounds();

    // L1 and L2 gas bounds always exist.
    // Old V3 txs always have L2 gas bounds of zero, but they exist.
    let mut resource_felts = vec![
        get_concat_resource(&l1_resource_bounds, L1_GAS)?,
        get_concat_resource(&l2_resource_bounds, L2_GAS)?,
    ];

    // For new V3 txs, need to also hash the data gas bounds.
    resource_felts.extend(match resource_bounds {
        ValidResourceBounds::L1Gas(_) => vec![],
        ValidResourceBounds::AllResources(all_resources) => {
            vec![get_concat_resource(&all_resources.l1_data_gas, L1_DATA_GAS)?]
        }
    });

    Ok(HashChain::new().chain(&tip.0.into()).chain_iter(resource_felts.iter()).get_poseidon_hash())
}

pub(crate) fn get_invoke_transaction_v3_hash<T: InvokeTransactionV3Trait>(
    transaction: &T,
    chain_id: &ChainId,
    transaction_version: &TransactionVersion,
) -> Result<TransactionHash, StarknetApiError> {
    let tip_resource_bounds_hash =
        get_tip_resource_bounds_hash(&transaction.resource_bounds(), transaction.tip())?;
    let paymaster_data_hash =
        HashChain::new().chain_iter(transaction.paymaster_data().0.iter()).get_poseidon_hash();
    let data_availability_mode = concat_data_availability_mode(
        transaction.nonce_data_availability_mode(),
        transaction.fee_data_availability_mode(),
    );
    let account_deployment_data_hash = HashChain::new()
        .chain_iter(transaction.account_deployment_data().0.iter())
        .get_poseidon_hash();
    let calldata_hash =
        HashChain::new().chain_iter(transaction.calldata().0.iter()).get_poseidon_hash();
    let mut hash_chain = HashChain::new()
        .chain(&INVOKE)
        .chain(&transaction_version.0)
        .chain(transaction.sender_address().0.key())
        .chain(&tip_resource_bounds_hash)
        .chain(&paymaster_data_hash)
        .chain(&Felt::try_from(chain_id)?)
        .chain(&transaction.nonce().0)
        .chain(&data_availability_mode)
        .chain(&account_deployment_data_hash)
        .chain(&calldata_hash);
    if !transaction.proof_facts().0.is_empty() {
        let proof_facts_hash =
            HashChain::new().chain_iter(transaction.proof_facts().0.iter()).get_poseidon_hash();
        hash_chain = hash_chain.chain(&proof_facts_hash);
    }
    Ok(TransactionHash(hash_chain.get_poseidon_hash()))
