### Title
`SwapAllowlistExtension` Gates the Router Address Instead of the Originating User, Allowing Non-Allowlisted Users to Bypass the Swap Guard via `MetricOmmSimpleRouter` — (`metric-periphery/contracts/extensions/SwapAllowlistExtension.sol`)

### Summary

The `SwapAllowlistExtension.beforeSwap` hook enforces a per-pool swap allowlist keyed on `(pool, sender)`. When a swap is routed through `MetricOmmSimpleRouter`, the `sender` argument delivered to the extension is the router's own address rather than the originating user. Because the router is a public, permissionless contract, any user can route through it. If the router address is on the allowlist, every non-allowlisted user bypasses the guard; if the router is not on the allowlist, every allowlisted user is silently blocked from using the router. Either outcome breaks the invariant the pool admin intended to enforce.

### Finding Description

`MetricOmmPool.swap` accepts `sender` as an explicit parameter and forwards it verbatim to every configured extension via `ExtensionCalling._beforeSwap`. The `SwapAllowlistExtension` then performs:

```solidity
allowedSwapper[pool][sender]   // or allowAll[pool]
``` [1](#0-0) 

When `MetricOmmSimpleRouter.exactInputSingle` (or any `exact*` entry point) calls `pool.swap(...)`, it passes `address(this)` — the router — as `sender`, because the router is the contract that will handle the `swapCallback` and settle token transfers. The extension therefore checks `allowedSwapper[pool][router]`, not `allowedSwapper[pool][originalUser]`. [2](#0-1) 

This is structurally identical to the M-06 pattern: just as `MIMOProxyFactory.deployFor()` could be called directly to circumvent the registry, `MetricOmmSimpleRouter` acts as an unguarded alternate path that the allowlist extension was never configured to gate. [3](#0-2) 

### Impact Explanation

**Bypass path (router allowlisted):** A pool admin allowlists the router address (a natural operational choice so that normal users can trade). Any non-allowlisted address can then call `MetricOmmSimpleRouter.exactInputSingle` and the extension passes, because `allowedSwapper[pool][router] == true`. The curated pool's swap restriction is completely nullified for every user who routes through the public router.

**Broken-functionality path (router not allowlisted):** If the router is not explicitly allowlisted, every allowlisted user who tries to swap through the router is rejected, because the extension sees `sender = router` which is not on the list. The pool's primary user-facing swap path is broken for all legitimate participants.

Both outcomes satisfy the allowed impact gate: the first is a direct guard bypass enabling unauthorized swaps; the second is broken core pool functionality.

### Likelihood Explanation

- `MetricOmmSimpleRouter` is the canonical, publicly documented swap entry point for the protocol.
- Pool admins who configure a `SwapAllowlistExtension` will naturally test direct `pool.swap` calls, not router-mediated calls, and will not observe the mismatch.
- No privileged action is required; any unprivileged user can call the router.
- The router is a fixed, deployed contract — its address is stable and predictable. [4](#0-3) 

### Recommendation

The `SwapAllowlistExtension.beforeSwap` hook should gate on the **economically originating user**, not on the `sender` parameter that the router populates with its own address. Two complementary fixes:

1. **In `MetricOmmSimpleRouter`:** pass `msg.sender` (the originating user) as the `sender` argument to `pool.swap`, not `address(this)`. This preserves the user's identity through the hook chain.

2. **In `SwapAllowlistExtension`:** if the `sender` is a recognized periphery contract (router), fall back to checking an additional `recipient` or `originatingUser` field from `extensionData`, or require the router to attest the real caller in the extension payload.

Fix (1) alone is sufficient if the router is the only supported entry point and the callback settlement does not require `sender == msg.sender`. [5](#0-4) 

### Proof of Concept

```
Setup:
  - Deploy pool with SwapAllowlistExtension configured.
  - allowedSwapper[pool][alice] = true   (alice is allowlisted)
  - allowedSwapper[pool][router] = false (router is not explicitly listed)
  - allowAll[pool] = false

Attack (bypass when router IS allowlisted):
  - Admin adds router to allowlist: allowedSwapper[pool][router] = true
  - Bob (not allowlisted) calls MetricOmmSimpleRouter.exactInputSingle(pool, ...)
  - Router calls pool.swap(address(router), bob_recipient, ...)
  - beforeSwap checks allowedSwapper[pool][router] == true  → passes
  - Bob's swap executes; allowlist is bypassed.

Broken-functionality (router NOT allowlisted):
  - Alice (allowlisted) calls MetricOmmSimpleRouter.exactInputSingle(pool, ...)
  - Router calls pool.swap(address(router), alice_recipient, ...)
  - beforeSwap checks allowedSwapper[pool][router] == false → reverts
  - Alice cannot use the router despite being individually allowlisted.
``` [2](#0-1) [6](#0-5)
