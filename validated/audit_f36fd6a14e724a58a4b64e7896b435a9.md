Audit Report

## Title
`SwapAllowlistExtension` checks router address instead of end-user address, allowing allowlist bypass - (`metric-periphery/contracts/extensions/SwapAllowlistExtension.sol`)

## Summary
When a swap is routed through `MetricOmmSimpleRouter`, the `sender` argument delivered to `SwapAllowlistExtension.beforeSwap` is the router's address, not the originating user's address. A pool admin who allowlists the router (necessary for any user to swap via the router) inadvertently grants every unprivileged address the ability to bypass the per-address swap allowlist entirely.

## Finding Description

**Call path:**

1. User calls `MetricOmmSimpleRouter.exactInputSingle(params)` — `msg.sender` = user EOA.
2. Router calls `IMetricOmmPoolActions(params.pool).swap(...)` — `msg.sender` inside the pool = router.
3. Pool's `swap()` invokes `_beforeSwap(msg.sender, ...)` passing `sender = router`. [1](#0-0) 
4. `ExtensionCalling._beforeSwap` encodes `sender` (= router) and dispatches to the extension. [2](#0-1) 
5. `SwapAllowlistExtension.beforeSwap` receives `sender = router` and checks:
   ```solidity
   if (!allowAllSwappers[msg.sender] && !allowedSwapper[msg.sender][sender]) {
       revert NotAllowedToSwap();
   }
   ```
   where `msg.sender` = pool and `sender` = router. [3](#0-2) 

**Root cause:** The pool passes its own `msg.sender` (the router) as `sender` to the extension. The extension was designed to gate individual traders, but it receives the intermediary router address instead.

**Exploit flow:**
- Pool admin deploys pool with `SwapAllowlistExtension`, allowlists specific user EOAs via `setAllowedToSwap`.
- For any allowlisted user to swap via the router, the admin must also call `setAllowedToSwap(pool, router, true)`.
- Once the router is allowlisted, **any** address — including non-allowlisted attackers — can call `router.exactInputSingle(...)` and the extension check passes because `allowedSwapper[pool][router] == true`.
- The per-user allowlist is completely bypassed.

**Existing guards are insufficient:** `BaseMetricExtension.onlyPool` only verifies the caller is a registered pool; it does not recover the original user. The router stores the original `msg.sender` only in transient storage for payment purposes and never forwards it to the pool or extension. [4](#0-3) 

## Impact Explanation
The swap allowlist — a core pool access-control mechanism — is rendered ineffective for all router-mediated swaps. Any unprivileged trader can bypass the allowlist by routing through `MetricOmmSimpleRouter`, directly violating the admin-boundary invariant (pool admin's access control bypassed by an unprivileged path). This constitutes broken core pool functionality and an admin-boundary break.

## Likelihood Explanation
Exploitation requires only that the pool admin has allowlisted the router (a necessary operational step for any user to use the router). No special privileges, flash loans, or unusual conditions are needed. Any unprivileged EOA can call `exactInputSingle` on the router targeting the allowlisted pool. The condition is trivially reachable in any production deployment where the router is used alongside `SwapAllowlistExtension`.

## Recommendation
Pass the original caller's address through the call chain. One approach: have the router encode `msg.sender` into `extensionData` and have `SwapAllowlistExtension.beforeSwap` decode and check it. A cleaner protocol-level fix is to add an `originator` field to the swap call or have the pool read the original caller from a trusted transient slot set by the router before calling `pool.swap`.

## Proof of Concept

```solidity
// 1. Pool admin sets up allowlist for alice only
swapAllowlist.setAllowedToSwap(pool, alice, true);
// 2. Admin must allowlist the router so alice can use it
swapAllowlist.setAllowedToSwap(pool, address(router), true);

// 3. Attacker (not allowlisted) calls router directly
vm.prank(attacker); // attacker != alice, not in allowlist
router.exactInputSingle(ExactInputSingleParams({
    pool: pool,
    recipient: attacker,
    tokenIn: token0,
    zeroForOne: true,
    amountIn: 1e18,
    amountOutMinimum: 0,
    priceLimitX64: 0,
    deadline: block.timestamp + 1,
    extensionData: ""
}));
// Swap succeeds — allowlist bypassed because extension sees sender=router, which is allowlisted
```
