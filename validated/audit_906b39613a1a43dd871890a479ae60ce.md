### Title
`CongestionInfoV1.allowed_shard` Silently Truncates `ShardId` (u64 → u16) in Borsh-Serialized Chunk Header - (File: `core/primitives/src/congestion_info.rs`)

### Summary

`CongestionInfoV1`, which is Borsh-serialized inside every `ShardChunkHeaderInnerV4`, stores the congestion-control "allowed shard" as a `u16`. `ShardId` is a `u64`. The `Into<u16> for ShardId` conversion silently truncates via `self.0 as u16`. Every call site that writes `allowed_shard` uses this truncating cast. When any shard ID exceeds 65 535, the wrong shard is recorded as the one permitted to forward receipts during full congestion, permanently breaking the progress guarantee of the congestion-control protocol.

### Finding Description

`CongestionInfoV1` is the sole variant of the versioned `CongestionInfo` enum and is Borsh-serialized as part of `ShardChunkHeaderInnerV4`:

```rust
// core/primitives/src/congestion_info.rs  line 460-470
pub struct CongestionInfoV1 {
    pub delayed_receipts_gas: u128,
    pub buffered_receipts_gas: u128,
    pub receipt_bytes: u64,
    pub allowed_shard: u16,   // ← only 16 bits; ShardId is u64
}
```

`ShardId` is `u64` internally, and its `Into<u16>` implementation is a bare truncating cast:

```rust
// core/primitives-core/src/types.rs  line 180-183
impl Into<u16> for ShardId {
    fn into(self) -> u16 {
        self.0 as u16   // silent truncation
    }
}
```

Every write path uses this cast without any bounds check:

**`finalize_allowed_shard`** (called once per chunk application):
```rust
// core/primitives/src/congestion_info.rs  line 366-367
let allowed_shard = Self::get_new_allowed_shard(own_shard, all_shards, congestion_seed);
self.set_allowed_shard(allowed_shard.into());  // ShardId → u16, truncates
```

**`bootstrap_congestion_info`** (called at shard bootstrap):
```rust
// runtime/runtime/src/congestion_control.rs  line 787
allowed_shard: shard_id.into(),  // ShardId → u16, truncates
```

The read path reconstructs a `ShardId` from the stored `u16`:

```rust
// core/primitives/src/congestion_info.rs  line 85
if sender_shard == ShardId::from(self.info.allowed_shard()) {
```

`ShardId::from(u16)` zero-extends, so if the original shard ID was, say, `65 537` (`0x10001`), it is stored as `1` (`0x0001`), and the comparison will match shard `1` instead of shard `65 537`.

### Impact Explanation

`CongestionInfo` is embedded in `ShardChunkHeaderInnerV4`, which is Borsh-serialized and consensus-validated by all validators. When the truncation fires:

1. The chunk header records the wrong shard as `allowed_shard`.
2. `outgoing_gas_limit` grants forwarding capacity to the wrong shard and denies it to the correct one.
3. The correct shard is permanently starved of forwarding capacity during full-congestion epochs, breaking the liveness guarantee of the congestion-control protocol.
4. Because the value is part of the Borsh-serialized header, all validators compute the same wrong value deterministically — there is no fork, but the protocol invariant is silently violated for every subsequent chunk.

### Likelihood Explanation

NEAR currently has ~6 shards; shard IDs are assigned sequentially from 0. Reaching shard ID 65 536 requires 65 536 cumulative shard-split events. This is not reachable in the near term, making the likelihood low. However:

- The type mismatch is baked into the frozen Borsh schema of `CongestionInfoV1`; it cannot be corrected without a new `CongestionInfo::V2` variant and a protocol upgrade.
- `ShardId` is explicitly typed as `u64` throughout the codebase, signalling intent to support large IDs.
- The truncation is completely silent — no panic, no error, no log — so it would not be detected until congestion-control misbehaviour was observed.

### Recommendation

1. Widen `CongestionInfoV1.allowed_shard` to `u32` or `u64` in a new `CongestionInfo::V2` variant (the existing V1 schema is frozen by Borsh compatibility).
2. Replace the bare `Into<u16>` conversion with a checked conversion (`u16::try_from(shard_id.0).expect(...)` or return an error) at both `finalize_allowed_shard` and `bootstrap_congestion_info`.
3. Remove or gate the `impl Into<u16> for ShardId` conversion to prevent future silent truncations.

### Proof of Concept

The truncation path in production code:

```
finalize_allowed_shard(own_shard=ShardId(65537), all_shards=[...], seed)
  → get_new_allowed_shard returns ShardId(65537)
  → set_allowed_shard(ShardId(65537).into())
       = set_allowed_shard(65537u64 as u16)
       = set_allowed_shard(1u16)          ← 65537 & 0xFFFF = 1
  → CongestionInfoV1 { allowed_shard: 1 } serialized into chunk header

outgoing_gas_limit(sender_shard=ShardId(65537)):
  → ShardId::from(self.info.allowed_shard())  // from(1u16) = ShardId(1)
  → ShardId(65537) == ShardId(1)  → false
  → returns Gas::ZERO              ← correct shard is permanently blocked

outgoing_gas_limit(sender_shard=ShardId(1)):
  → ShardId(1) == ShardId(1)  → true
  → returns allowed_shard_outgoing_gas  ← wrong shard gets the slot
```
