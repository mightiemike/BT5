### Title
Protobuf Deserialization Misclassifies `AllResources` as `L1Gas` When L2/L1DataGas Are Zero, Causing Transaction Hash Divergence Between Proposer and Validator — (File: `crates/apollo_protobuf/src/converters/transaction.rs`)

---

### Summary

`TryFrom<protobuf::ResourceBounds> for ValidResourceBounds` silently downgrades a deserialized `AllResources` transaction to `L1Gas` whenever `l2_gas` and `l1_data_gas` are both zero. Because `get_tip_resource_bounds_hash` hashes a different number of resource-bound elements for `L1Gas` (2) versus `AllResources` (3), the transaction hash computed by a validator after P2P protobuf round-trip differs from the hash computed by the gateway at admission. The user's signature is over the gateway hash; the validator recomputes a different hash and rejects the signature, causing valid transactions to be dropped and proposer/validator execution to diverge.

---

### Finding Description

**Step 1 — Misclassification in protobuf deserialization**

`crates/apollo_protobuf/src/converters/transaction.rs` lines 431–435:

```rust
Ok(if l1_data_gas.is_zero() && l2_gas.is_zero() {
    ValidResourceBounds::L1Gas(l1_gas)
} else {
    ValidResourceBounds::AllResources(AllResourceBounds { l1_gas, l2_gas, l1_data_gas })
})
``` [1](#0-0) 

The condition fires on any `AllResources` transaction whose `l2_gas` and `l1_data_gas` happen to be zero — a configuration the gateway explicitly accepts (e.g., `AllResourceBounds { l1_gas: NON_EMPTY, ..Default::default() }`). [2](#0-1) 

**Step 2 — Hash preimage length diverges on the variant**

`get_tip_resource_bounds_hash` in `crates/starknet_api/src/transaction_hash.rs` builds the poseidon preimage conditionally:

```rust
let mut resource_felts = vec![
    get_concat_resource(&l1_resource_bounds, L1_GAS)?,
    get_concat_resource(&l2_resource_bounds, L2_GAS)?,   // always present
];
resource_felts.extend(match resource_bounds {
    ValidResourceBounds::L1Gas(_) => vec![],             // ← 2 elements total
    ValidResourceBounds::AllResources(all_resources) => {
        vec![get_concat_resource(&all_resources.l1_data_gas, L1_DATA_GAS)?]
    }                                                    // ← 3 elements total
});
Ok(HashChain::new().chain(&tip.0.into()).chain_iter(resource_felts.iter()).get_poseidon_hash())
``` [3](#0-2) 

For the same numeric values `{ l1_gas: X, l2_gas: 0, l1_data_gas: 0 }`:

| Variant | Preimage elements | Hash |
|---|---|---|
| `AllResources` | tip, L1(X), L2(0), L1_DATA(0) | H_all |
| `L1Gas` | tip, L1(X), L2(0) | H_l1 ≠ H_all |

**Step 3 — Hash is recomputed from the deserialized variant**

`TransactionConverter::convert_rpc_tx_to_internal` recomputes `tx_hash` from the deserialized `InternalRpcTransactionWithoutTxHash`:

```rust
let tx_hash = tx_without_hash.calculate_transaction_hash(&self.chain_id)?;
Ok((InternalRpcTransaction { tx: tx_without_hash, tx_hash }, proof_data))
``` [4](#0-3) 

`InternalRpcInvokeTransactionV3::calculate_transaction_hash` calls `get_invoke_transaction_v3_hash`, which calls `get_tip_resource_bounds_hash` with `ValidResourceBounds::AllResources(self.resource_bounds)` — but by this point the `InvokeTransactionV3` intermediate was already reconstructed from the misclassified `L1Gas` variant via protobuf, so the hash is H_l1. [5](#0-4) 

**Step 4 — The same divergence affects Declare and DeployAccount V3**

`get_declare_transaction_v3_hash` and `get_deploy_account_transaction_v3_hash` both call `get_tip_resource_bounds_hash` with the same branching logic. [6](#0-5) [7](#0-6)

### Impact Explanation

**Proposer path**: The gateway receives an `RpcInvokeTransactionV3` whose `resource_bounds` field is typed `AllResourceBounds` (never `ValidResourceBounds`), so it always computes H_all. The user signs H_all. [8](#0-7) 

**Validator path**: The transaction is propagated as a `ConsensusTransaction` over P2P protobuf. On the receiving side, `TryFrom<protobuf::InvokeV3WithProof> for RpcInvokeTransactionV3` goes through `InvokeTransactionV3` (which carries `ValidResourceBounds`), and the deserialized `ValidResourceBounds` is `L1Gas`. `convert_rpc_tx_to_internal` then stores `tx_hash = H_l1`. [9](#0-8) 

The blockifier on the validator validates the account signature against the stored `tx_hash = H_l1`. Because the user signed H_all ≠ H_l1, signature validation fails. The validator rejects the transaction and disagrees with the proposer on the block execution result, breaking consensus for any block containing such a transaction.

---

### Likelihood Explanation

The trigger is a standard V3 transaction with only `l1_gas` set and `l2_gas = l1_data_gas = 0`. The gateway's stateless validator explicitly accepts this configuration (test case `valid_l1_gas`). Any user who submits a V3 transaction paying only in L1 gas — a common pattern during the transition from pre-0.13.3 semantics — will hit this path. No special privileges are required.

---

### Recommendation

In `TryFrom<protobuf::ResourceBounds> for ValidResourceBounds`, do not infer the variant from field values. Instead, use the presence or absence of the `l1_data_gas` field in the protobuf message as the discriminant:

```rust
Ok(if value.l1_data_gas.is_none() {
    ValidResourceBounds::L1Gas(l1_gas)
} else {
    ValidResourceBounds::AllResources(AllResourceBounds { l1_gas, l2_gas, l1_data_gas })
})
```

This preserves the original variant intent: `l1_data_gas` absent on the wire means a pre-0.13.3 `L1Gas` transaction; `l1_data_gas` present (even if zero) means an `AllResources` transaction. The TODO comment at line 426 already anticipates this fix. [10](#0-9)

### Proof of Concept

```rust
// 1. Construct a V3 invoke tx with AllResources, l2_gas=0, l1_data_gas=0.
let rpc_tx = RpcInvokeTransactionV3 {
    resource_bounds: AllResourceBounds {
        l1_gas: ResourceBounds { max_amount: GasAmount(100), max_price_per_unit: GasPrice(1) },
        l2_gas: ResourceBounds::default(),      // zero
        l1_data_gas: ResourceBounds::default(), // zero
    },
    // ... other fields
};

// 2. Gateway computes H_all (AllResources, 3-element preimage).
let internal_tx = converter.convert_rpc_tx_to_internal_rpc_tx(
    RpcTransaction::Invoke(RpcInvokeTransaction::V3(rpc_tx.clone()))
).await.unwrap();
let h_all = internal_tx.0.tx_hash;

// 3. Serialize to protobuf (P2P propagation).
let proto_bytes = Vec::<u8>::from(RpcTransactionBatch(vec![
    RpcTransaction::Invoke(RpcInvokeTransaction::V3(rpc_tx))
]));

// 4. Deserialize: AllResources{l2=0,l1d=0} → L1Gas.
let deserialized = RpcTransactionBatch::try_from(proto_bytes).unwrap();
let internal_tx2 = converter.convert_rpc_tx_to_internal_rpc_tx(
    deserialized.0.into_iter().next().unwrap()
).await.unwrap();
let h_l1 = internal_tx2.0.tx_hash;

// 5. Hashes diverge — signature over h_all fails validation against h_l1.
assert_ne!(h_all, h_l1); // ← this assertion passes, demonstrating the bug
```

The divergence is rooted in `get_tip_resource_bounds_hash` producing a 2-element vs 3-element poseidon input for the same numeric resource bound values, depending solely on which `ValidResourceBounds` variant is present after deserialization. [11](#0-10) [12](#0-11)

pub fn get_tip_resource_bounds_hash(
    resource_bounds: &ValidResourceBounds,
    tip: &Tip,
) -> Result<Felt, StarknetApiError> {
    let l1_resource_bounds = resource_bounds.get_l1_bounds();
    let l2_resource_bounds = resource_bounds.get_l2_bounds();

    // L1 and L2 gas bounds always exist.
    // Old V3 txs always have L2 gas bounds of zero, but they exist.
    let mut resource_felts = vec![
        get_concat_resource(&l1_resource_bounds, L1_GAS)?,
        get_concat_resource(&l2_resource_bounds, L2_GAS)?,
    ];

    // For new V3 txs, need to also hash the data gas bounds.
    resource_felts.extend(match resource_bounds {
        ValidResourceBounds::L1Gas(_) => vec![],
        ValidResourceBounds::AllResources(all_resources) => {
            vec![get_concat_resource(&all_resources.l1_data_gas, L1_DATA_GAS)?]
        }
    });

    Ok(HashChain::new().chain(&tip.0.into()).chain_iter(resource_felts.iter()).get_poseidon_hash())
}

pub(crate) fn get_invoke_transaction_v3_hash<T: InvokeTransactionV3Trait>(
    transaction: &T,
    chain_id: &ChainId,
    transaction_version: &TransactionVersion,
) -> Result<TransactionHash, StarknetApiError> {
    let tip_resource_bounds_hash =
        get_tip_resource_bounds_hash(&transaction.resource_bounds(), transaction.tip())?;
    let paymaster_data_hash =
        HashChain::new().chain_iter(transaction.paymaster_data().0.iter()).get_poseidon_hash();
    let data_availability_mode = concat_data_availability_mode(
        transaction.nonce_data_availability_mode(),
        transaction.fee_data_availability_mode(),
    );
    let account_deployment_data_hash = HashChain::new()
        .chain_iter(transaction.account_deployment_data().0.iter())
        .get_poseidon_hash();
    let calldata_hash =
        HashChain::new().chain_iter(transaction.calldata().0.iter()).get_poseidon_hash();
    let mut hash_chain = HashChain::new()
        .chain(&INVOKE)
        .chain(&transaction_version.0)
        .chain(transaction.sender_address().0.key())
        .chain(&tip_resource_bounds_hash)
        .chain(&paymaster_data_hash)
        .chain(&Felt::try_from(chain_id)?)
        .chain(&transaction.nonce().0)
        .chain(&data_availability_mode)
        .chain(&account_deployment_data_hash)
        .chain(&calldata_hash);
    if !transaction.proof_facts().0.is_empty() {
        let proof_facts_hash =
            HashChain::new().chain_iter(transaction.proof_facts().0.iter()).get_poseidon_hash();
        hash_chain = hash_chain.chain(&proof_facts_hash);
    }
    Ok(TransactionHash(hash_chain.get_poseidon_hash()))
}

impl TryFrom<protobuf::InvokeV3WithProof> for RpcInvokeTransactionV3 {
    type Error = ProtobufConversionError;
    fn try_from(mut value: protobuf::InvokeV3WithProof) -> Result<Self, Self::Error> {
        // Extract proof first, since `starknet_api::transaction::InvokeTransactionV3` does not
        // carry a `proof` field.
        let proof = Proof::from(std::mem::take(&mut value.proof));

        let snapi_invoke: InvokeTransactionV3 = value
            .invoke
            .ok_or(ProtobufConversionError::MissingField {
                field_description: "InvokeV3WithProof::invoke",
            })?
            .try_into()?;

        // This conversion can fail only if the resource_bounds are not AllResources.
        Ok(Self { proof, ..snapi_invoke.try_into().map_err(|_| DEPRECATED_RESOURCE_BOUNDS_ERROR)? })
    }
