### Title
Malicious P2P Peer Can Misclassify Cairo-1 Class as Deprecated by Stripping `compiled_class_hash` Field - (`crates/apollo_protobuf/src/converters/state_diff.rs`)

### Summary

The `TryFrom<protobuf::StateDiffsResponse> for DataOrFin<StateDiffChunk>` converter uses the presence/absence of `compiled_class_hash` as the sole discriminant between `StateDiffChunk::DeclaredClass` (Cairo-1) and `StateDiffChunk::DeprecatedDeclaredClass` (Cairo-0). A malicious P2P peer can strip field tag 2 from a `DeclaredClass` wire message, causing the receiving node to commit a `ThinStateDiff` where a Cairo-1 class hash lands in `deprecated_declared_classes` instead of `class_hash_to_compiled_class_hash`. No existing guard in the state-diff sync pipeline rejects this manipulation.

### Finding Description

**Step 1 – Converter branch (the root cause)** [1](#0-0) 

When `compiled_class_hash` is `None` the converter unconditionally produces `StateDiffChunk::DeprecatedDeclaredClass`. There is no cross-check against any trusted source (block hash, state root, Starknet version) at this point.

**Step 2 – Length check does not distinguish class types**

`StateDiffChunk::len()` returns `1` for both `DeclaredClass` and `DeprecatedDeclaredClass`: [2](#0-1) 

`ThinStateDiff::len()` counts both `class_hash_to_compiled_class_hash` entries and `deprecated_declared_classes` entries as `1` each: [3](#0-2) 

The `parse_data_for_block` loop only checks that `current_state_diff_len == target_state_diff_len`: [4](#0-3) 

A legitimate block with one Cairo-1 class has `state_diff_length = 1`. The attacker sends one `DeprecatedDeclaredClass` chunk (also `len() == 1`). The check passes.

**Step 3 – Corrupted `ThinStateDiff` is assembled and committed**

`unite_state_diffs` routes the chunk to `deprecated_declared_classes`: [5](#0-4) 

`write_to_storage` immediately commits it: [6](#0-5) 

`append_state_diff` writes the class hash to `deprecated_declared_classes_block_table` and skips `compiled_class_hash_table` entirely: [7](#0-6) 

**Step 4 – Class sync is poisoned by the corrupted state diff**

The class sync reads the now-committed `ThinStateDiff` to decide which class type to expect: [8](#0-7) 

Because the class hash is in `deprecated_declared_classes` (not `class_hash_to_compiled_class_hash`), the class sync accepts a `DeprecatedContractClass` for it and rejects any honest peer that sends the correct `ContractClass` with `ClassNotInStateDiff`.

### Impact Explanation

After the attack succeeds:

1. `compiled_class_hash_table` has no entry for the class hash → any execution path that looks up the compiled class hash for this class will fail or select the wrong artifact. This is **Critical: Wrong compiled class / CASM artifact selected for execution**.
2. The class hash is recorded in `deprecated_declared_classes_block_table` → the node's state root diverges from the canonical chain, and any RPC call that reads class type (e.g., `starknet_getClass`) returns the wrong class type. This is **Critical: Wrong state committed to storage**.

### Likelihood Explanation

Any single P2P peer that the syncing node connects to for state-diff data can perform this attack. No special privilege is required. The peer is never reported as bad because all validation checks (length, no-conflict, no-duplicate) pass. The attack is silent and permanent until the node is resynced from scratch.

### Recommendation

1. **Reject `None` compiled_class_hash at the converter level** when the Starknet version of the block being synced is ≥ 0.11 (Cairo-1 era). Return a `ProtobufConversionError` instead of silently downgrading to `DeprecatedDeclaredClass`.
2. **Verify the state root** after assembling the `ThinStateDiff` against the block header's `state_root` field before committing to storage.
3. **Cross-check class type** in the class sync: if the stored state diff says a hash is deprecated but the peer sends a `ContractClass`, report the state-diff peer (not the class peer) as bad and revert the committed state diff.

### Proof of Concept

```rust
// Encode DeclaredClass{class_hash=H, compiled_class_hash=Some(X)}
let mut msg = protobuf::DeclaredClass {
    class_hash: Some(H.into()),
    compiled_class_hash: Some(X.into()),
};
let mut bytes = Vec::new();
msg.encode(&mut bytes).unwrap();

// Strip field tag 2 (compiled_class_hash) from wire bytes
// Field 2, wire type 2 (LEN): tag byte = (2 << 3) | 2 = 0x12
let stripped = strip_field_2(&bytes); // remove 0x12 + length + payload

// Re-parse as StateDiffsResponse wrapping a DeclaredClass
let response = protobuf::StateDiffsResponse {
    state_diff_message: Some(
        protobuf::state_diffs_response::StateDiffMessage::DeclaredClass(
            protobuf::DeclaredClass::decode(stripped.as_slice()).unwrap()
        )
    )
};

let result = DataOrFin::<StateDiffChunk>::try_from(response).unwrap();
// Assert: result is DeprecatedDeclaredClass, NOT DeclaredClass
assert!(matches!(
    result.0.unwrap(),
    StateDiffChunk::DeprecatedDeclaredClass(_)
));
```
