### Title
Spread Positions at Maximum Leverage Have a Critically Narrow Liquidation Window, Enabling Bad Debt Accumulation — (File: `core/contracts/libraries/RiskHelper.sol`, `core/contracts/ClearinghouseStorage.sol`)

---

### Summary

The gap between the INITIAL (`0.99`) and MAINTENANCE (`0.994`) spread weight caps in `RiskHelper._getSpreadWeightX18` is only `0.004` — representing a liquidation health buffer of **0.4% of notional** for spread positions opened at maximum leverage. Combined with the minimum spread liquidation penalty of `0.25%` (`MIN_SPREAD_LIQ_PENALTY_X18`) and a `LIQUIDATION_FEE_FRACTION` of `50%`, the net liquidator profit floor is only `0.125%` of notional. In volatile market conditions, the basis can move faster than liquidators can act, leaving spread positions under-collateralized and accumulating bad debt before liquidation occurs.

---

### Finding Description

In `RiskHelper._getSpreadWeightX18`, the spread weight caps are hardcoded as:

```
INITIAL:     maxSpreadWeight = ONE - ONE / 100       = 0.99
MAINTENANCE: maxSpreadWeight = ONE - (ONE * 6) / 1000 = 0.994
``` [1](#0-0) 

The spread health bonus applied in `Clearinghouse.getHealth` is:

```
health += basisAmount * (spotPrice + perpPrice) * (spreadWeight - existingWeight)
``` [2](#0-1) 

For a spread position where the cap is active, the difference in health bonus between INITIAL and MAINTENANCE is:

```
basisAmount * (spotPrice + perpPrice) * (0.994 - 0.99) = 0.004 * notional
```

This means a spread position opened at maximum leverage (initial health ≈ 0) has only a **0.4% adverse basis move** before maintenance health drops below zero and the position becomes liquidatable.

The liquidation incentive for spread positions is computed in `getSpreadLiqPriceX18`:

```solidity
penaltyX18 = (ONE - maintenanceWeight) / 10;
// floored at MIN_SPREAD_LIQ_PENALTY_X18 = ONE / 400 = 0.25%
``` [3](#0-2) 

The liquidator's net profit is further halved by `LIQUIDATION_FEE_FRACTION = 50%`:

```solidity
v.liquidationFees = (v.oraclePriceX18 - v.liquidationPriceX18)
    .mul(LIQUIDATION_FEE_FRACTION)
    .mul(txn.amount);
``` [4](#0-3) [5](#0-4) 

The liquidator receives the spread at a `0.25%` discount but pays `50%` of that discount as fees to the insurance fund, leaving a net profit of only `0.125%` of notional. This is the floor incentive for liquidating a spread position.

The combined window before bad debt is therefore:
- **Liquidation trigger**: basis moves 0.4% (maintenance health < 0)
- **Bad debt threshold**: basis moves an additional ~0.25% (liquidatee's assets exhausted)
- **Total window**: ~0.65% of notional [6](#0-5) 

---

### Impact Explanation

If the basis moves more than ~0.65% before a liquidator acts, the liquidatee's quote balance goes negative beyond what the insurance fund can cover. The `_finalizeSubaccount` path then calls `spotEngine.socializeSubaccount`, distributing losses across all protocol participants. [7](#0-6) 

Additionally, `_assertLiquidationAmount` caps the liquidatable amount by `(quoteBalance.amount + insurance) / liquidationPrice`, meaning that if the quote balance is deeply negative, only a partial liquidation is possible per transaction, further delaying full resolution and compounding bad debt. [8](#0-7) 

**Impact**: Direct bad debt accumulation in the insurance fund, potential socialization of losses to all protocol participants, and permanent solvency degradation.

---

### Likelihood Explanation

Spread positions are explicitly supported and incentivized (up to 100x leverage vs ~11x for individual products). Any unprivileged user can open a spread at maximum leverage through the normal `Endpoint` transaction flow. The 0.4% basis buffer is realistic to breach during:
- Sudden funding rate spikes
- Exchange-wide liquidation cascades
- Low-liquidity periods where the spot/perp basis widens rapidly

The 0.125% net liquidator profit floor may not cover gas costs for small positions, further reducing the likelihood of timely liquidation.

**Likelihood**: Medium-High. The trigger requires volatile market conditions but no privileged access. Spread positions at maximum leverage are a normal, encouraged use case.

---

### Recommendation

1. **Widen the spread weight gap**: Increase the difference between `INITIAL` and `MAINTENANCE` spread weight caps. For example, set `INITIAL` to `0.95` and `MAINTENANCE` to `0.97`, providing a ~2% liquidation buffer instead of 0.4%.
2. **Increase the minimum spread liquidation penalty**: Raise `MIN_SPREAD_LIQ_PENALTY_X18` from `ONE / 400` (0.25%) to at least `ONE / 100` (1%) to ensure liquidators are adequately incentivized.
3. **Run basis volatility simulations**: Measure historical and stress-scenario basis moves for all supported spread pairs to validate that the liquidation buffer is sufficient.
4. **Deploy liquidation bots**: Ensure automated liquidation infrastructure is operational at protocol launch to minimize the window between a position becoming liquidatable and actual liquidation. [9](#0-8)

### Proof of Concept

**Setup**: User opens a spread position (long 1 ETH spot, short 1 ETH perp) at maximum leverage.

- ETH spot price = ETH perp price = $1,000
- Individual product `longWeightInitialX18` = 0.9 for both spot and perp
- `existingWeight` = (0.9 + 0.9) / 2 = 0.9
- Spread weight cap (INITIAL) = 0.99
- Spread health bonus = 1 ETH × $2,000 × (0.99 − 0.9) = **$180**

User borrows $180 against the spread, bringing initial health to exactly 0.

**Trigger**: Basis widens by 0.4% (e.g., spot drops to $998, perp stays at $1,000 — a $2 basis move on a $2,000 notional spread).

- Individual maintenance weights = 0.95 for both
- `existingWeight_maintenance` = 0.95
- Spread weight cap (MAINTENANCE) = 0.994
- Maintenance spread bonus = 1 ETH × $1,998 × (0.994 − 0.95) ≈ **$87.91**
- Individual maintenance health ≈ 1 × $998 × 0.95 + (−1) × $1,000 × 0.95 − $180 ≈ −$181.10
- Total maintenance health ≈ −$181.10 + $87.91 = **−$93.19 < 0** → liquidatable

**Liquidation incentive**: `MIN_SPREAD_LIQ_PENALTY_X18 = 0.25%` → liquidator net profit ≈ $998 × 0.25% × 50% ≈ **$1.25** on a $2,000 position.

**Bad debt**: If the basis widens a further 0.25% before liquidation (total 0.65%), the liquidatee's quote balance is exhausted. The insurance fund absorbs the deficit; if depleted, `socializeSubaccount` is called. [10](#0-9) [11](#0-10) [12](#0-11)

    function _getSpreadWeightX18(
        IProductEngine.CoreRisk memory perpCoreRisk,
        IProductEngine.CoreRisk memory spotCoreRisk,
        IProductEngine.HealthType healthType
    ) internal pure returns (int128) {
        if (healthType == IProductEngine.HealthType.PNL) {
            return ONE;
        }
        int128 spreadWeight;
        if (spotCoreRisk.amount > 0) {
            spreadWeight = ONE - (ONE - perpCoreRisk.longWeight) / 5;
        } else {
            spreadWeight = ONE - (ONE - spotCoreRisk.longWeight) / 5;
        }
        int128 maxSpreadWeight;
        if (healthType == IProductEngine.HealthType.INITIAL) {
            maxSpreadWeight = ONE - ONE / 100; // 0.99
        } else {
            maxSpreadWeight = ONE - (ONE * 6) / 1000; // 0.994
        }
        if (spreadWeight > maxSpreadWeight) {
            spreadWeight = maxSpreadWeight;
        }
        return spreadWeight;
```
