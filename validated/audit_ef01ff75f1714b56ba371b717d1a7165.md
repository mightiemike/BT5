### Title
`ValidResourceBounds` Protobuf Deserialization Collapses `AllResources{l2=0, l1_data=0}` to `L1Gas`, Producing a Divergent Transaction Hash at P2P-Syncing Nodes — (`crates/apollo_protobuf/src/converters/transaction.rs`)

---

### Summary

The protobuf-to-`ValidResourceBounds` conversion silently collapses `AllResources{l2_gas=0, l1_data_gas=0}` into `L1Gas`. Because `get_tip_resource_bounds_hash` hashes **3** resource-bound felts for `AllResources` but only **2** for `L1Gas`, a transaction submitted with `AllResourceBounds{l1_gas: X, l2_gas: 0, l1_data_gas: 0}` receives hash **H1** at the originating gateway and hash **H2 ≠ H1** at any P2P-syncing node that deserializes the same transaction from protobuf. This is a direct hash-domain boundary violation: the same transaction fields bind to two different canonical hashes depending on which side of the protobuf wire the computation occurs.

---

### Finding Description

**Step 1 — Originating node computes H1 with 3 resource bounds.**

`get_tip_resource_bounds_hash` in `crates/starknet_api/src/transaction_hash.rs` builds the fee-fields hash as follows:

```rust
let mut resource_felts = vec![
    get_concat_resource(&l1_resource_bounds, L1_GAS)?,
    get_concat_resource(&l2_resource_bounds, L2_GAS)?,   // ← included even when zero
];
resource_felts.extend(match resource_bounds {
    ValidResourceBounds::L1Gas(_) => vec![],             // ← NOT included
    ValidResourceBounds::AllResources(all) => {
        vec![get_concat_resource(&all.l1_data_gas, L1_DATA_GAS)?]  // ← included
    }
});
Ok(HashChain::new().chain(&tip.0.into()).chain_iter(resource_felts.iter()).get_poseidon_hash())
```

For `AllResources{l1_gas: X, l2_gas: 0, l1_data_gas: 0}` the hash input is `[tip, L1_GAS_packed(X), L2_GAS_packed(0), L1_DATA_GAS_packed(0)]` — **4 felts**. Note that `get_concat_resource(0, L2_GAS)` is non-zero because the resource-name bytes (`b"\0L2_GAS"`) are embedded in the felt even when the amount and price are zero.

The gateway accepts a transaction with only `l1_gas` non-zero (the `valid_l1_gas` test case in `stateless_transaction_validator_test.rs` confirms this). The `InternalRpcInvokeTransactionV3` always carries `AllResourceBounds`, so the hash is always computed with the `AllResources` branch → **H1**.

**Step 2 — Protobuf deserialization at the receiving node collapses to `L1Gas`.**

`TryFrom<protobuf::ResourceBounds> for ValidResourceBounds` in `crates/apollo_protobuf/src/converters/transaction.rs`:

```rust
let l1_data_gas = value.l1_data_gas.unwrap_or_default();   // defaults to zero if absent
...
Ok(if l1_data_gas.is_zero() && l2_gas.is_zero() {
    ValidResourceBounds::L1Gas(l1_gas)                      // ← collapses here
} else {
    ValidResourceBounds::AllResources(AllResourceBounds { l1_gas, l2_gas, l1_data_gas })
})
```

When the wire carries `l2_gas=0` and `l1_data_gas=0` (which is exactly the case for a transaction with only `l1_gas` non-zero), the receiving node produces `ValidResourceBounds::L1Gas(X)`.

**Step 3 — Receiving node computes H2 with 2 resource bounds.**

With `L1Gas(X)`, `get_tip_resource_bounds_hash` produces hash input `[tip, L1_GAS_packed(X), L2_GAS_packed(0)]` — **3 felts** (the `L1_DATA_GAS` felt is absent). Poseidon over 3 felts ≠ Poseidon over 4 felts → **H2 ≠ H1**.

**Step 4 — Hash divergence causes block validation failure.**

The block produced by the sequencer records the transaction with hash H1. When `validate_transaction_hash` is called at the syncing node, it recomputes the hash from the deserialized `ValidResourceBounds::L1Gas` representation and obtains H2. The comparison `possible_hashes.contains(&expected_hash)` fails → the block is rejected.

The Cairo OS `compute_invoke_transaction_hash` in `crates/apollo_starknet_os_program/src/cairo/starkware/starknet/core/os/transaction_hash/transaction_hash.cairo` always asserts `n_resource_bounds = 3`, so it is consistent with the `AllResources` path. The divergence is purely at the Rust protobuf deserialization boundary.

---

### Impact Explanation

Any block that contains a valid `InvokeV3` transaction with `AllResourceBounds{l1_gas: X, l2_gas: 0, l1_data_gas: 0}` will be permanently rejected by P2P-syncing nodes because the transaction hash stored in the block (H1) cannot be reproduced from the deserialized transaction fields (which yield H2). This causes a chain-split between the sequencer and all syncing nodes for any block containing such a transaction. The impact matches: **"Transaction conversion or signature/hash logic binds the wrong signer, hash, type, or executable payload"** (High) and **"Wrong state … from blockifier/syscall/execution logic for accepted input"** (Critical).

---

### Likelihood Explanation

The gateway explicitly accepts transactions with only `l1_gas` non-zero (confirmed by the `valid_l1_gas` test case). Any user who submits such a transaction triggers the divergence. No special privilege is required. The `l1_data_gas.unwrap_or_default()` fallback in the protobuf converter makes the collapse unconditional for any transaction where both `l2_gas` and `l1_data_gas` are zero on the wire.

---

### Recommendation

Remove the `L1Gas`/`AllResources` collapsing logic from the protobuf deserializer, or ensure that the hash function produces identical output for `AllResources{l2=0, l1_data=0}` and `L1Gas`. The cleanest fix is to always deserialize into `AllResources` when all three resource-bound fields are present on the wire (even if zero), reserving `L1Gas` only for the legacy case where `l1_data_gas` is genuinely absent from the protobuf message. Alternatively, `get_tip_resource_bounds_hash` should be made variant-agnostic by always including all three resource-bound felts regardless of the `ValidResourceBounds` discriminant.

---

pub fn get_tip_resource_bounds_hash(
    resource_bounds: &ValidResourceBounds,
    tip: &Tip,
) -> Result<Felt, StarknetApiError> {
    let l1_resource_bounds = resource_bounds.get_l1_bounds();
    let l2_resource_bounds = resource_bounds.get_l2_bounds();

    // L1 and L2 gas bounds always exist.
    // Old V3 txs always have L2 gas bounds of zero, but they exist.
    let mut resource_felts = vec![
        get_concat_resource(&l1_resource_bounds, L1_GAS)?,
        get_concat_resource(&l2_resource_bounds, L2_GAS)?,
    ];

    // For new V3 txs, need to also hash the data gas bounds.
    resource_felts.extend(match resource_bounds {
        ValidResourceBounds::L1Gas(_) => vec![],
        ValidResourceBounds::AllResources(all_resources) => {
            vec![get_concat_resource(&all_resources.l1_data_gas, L1_DATA_GAS)?]
        }
    });

    Ok(HashChain::new().chain(&tip.0.into()).chain_iter(resource_felts.iter()).get_poseidon_hash())
}

pub(crate) fn get_invoke_transaction_v3_hash<T: InvokeTransactionV3Trait>(
    transaction: &T,
    chain_id: &ChainId,
    transaction_version: &TransactionVersion,
) -> Result<TransactionHash, StarknetApiError> {
    let tip_resource_bounds_hash =
        get_tip_resource_bounds_hash(&transaction.resource_bounds(), transaction.tip())?;
    let paymaster_data_hash =
        HashChain::new().chain_iter(transaction.paymaster_data().0.iter()).get_poseidon_hash();
    let data_availability_mode = concat_data_availability_mode(
        transaction.nonce_data_availability_mode(),
        transaction.fee_data_availability_mode(),
    );
    let account_deployment_data_hash = HashChain::new()
        .chain_iter(transaction.account_deployment_data().0.iter())
        .get_poseidon_hash();
    let calldata_hash =
        HashChain::new().chain_iter(transaction.calldata().0.iter()).get_poseidon_hash();
    let mut hash_chain = HashChain::new()
        .chain(&INVOKE)
        .chain(&transaction_version.0)
        .chain(transaction.sender_address().0.key())
        .chain(&tip_resource_bounds_hash)
        .chain(&paymaster_data_hash)
        .chain(&Felt::try_from(chain_id)?)
        .chain(&transaction.nonce().0)
        .chain(&data_availability_mode)
        .chain(&account_deployment_data_hash)
        .chain(&calldata_hash);
    if !transaction.proof_facts().0.is_empty() {
        let proof_facts_hash =
            HashChain::new().chain_iter(transaction.proof_facts().0.iter()).get_poseidon_hash();
        hash_chain = hash_chain.chain(&proof_facts_hash);
    }
    Ok(TransactionHash(hash_chain.get_poseidon_hash()))
}
