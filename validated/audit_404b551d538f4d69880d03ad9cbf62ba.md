### Title
Unbacked Voucher Minting via Double-Prefix ClassId in `processReceivedPacket` — (`x/nft-transfer/keeper/packet.go`)

### Summary

`processReceivedPacket` unconditionally prepends the destination port/channel prefix to `data.ClassId` when `IsAwayFromOrigin` returns `true`. Because `IsAwayFromOrigin` only checks whether `data.ClassId` starts with the **source** port/channel prefix (not the destination), a malicious counterparty chain can craft a packet whose `classId` already begins with the destination port/channel pair. The function then double-prefixes the classId, derives a spurious `voucherClassID`, and mints NFT vouchers for a denom that has no corresponding escrowed NFT on any chain.

### Finding Description

**Entrypoint:** Any IBC channel connected to the `nft-transfer` module. The counterparty chain commits a packet with attacker-controlled `data.ClassId`.

**Exact code path:**

`IsAwayFromOrigin` in `x/nft-transfer/types/trace.go`:

```go
func IsAwayFromOrigin(sourcePort, sourceChannel, fullClassPath string) bool {
    prefixClassID := GetClassPrefix(sourcePort, sourceChannel)
    return !strings.HasPrefix(fullClassPath, prefixClassID)
}
``` [1](#0-0) 

It checks only the **source** port/channel. It does not check whether `fullClassPath` already starts with the **destination** port/channel.

`processReceivedPacket` in `x/nft-transfer/keeper/packet.go`:

```go
isAwayFromOrigin := types.IsAwayFromOrigin(packet.GetSourcePort(), packet.GetSourceChannel(), data.ClassId)
// ...
if isAwayFromOrigin {
    classPrefix := types.GetClassPrefix(packet.GetDestPort(), packet.GetDestChannel())
    prefixedClassID := classPrefix + data.ClassId   // ← unconditional concatenation
    classTrace := types.ParseClassTrace(prefixedClassID)
    // ...
    voucherClassID := classTrace.IBCClassID()
    // IssueDenom + MintNFT with this voucherClassID
}
``` [2](#0-1) 

**No guard exists.** `ValidateBasic()` only checks that `classId` is non-blank, that token arrays are consistent, and that addresses are valid bech32. It does not check whether `classId` already contains the destination prefix. [3](#0-2) 

`OnRecvPacket` calls only `data.ValidateBasic()` before delegating to `processReceivedPacket`. [4](#0-3) 

### Impact Explanation

**Concrete state delta:**

| Step | State change |
|---|---|
| Attacker sends packet with `classId = 'nft/channel-1/nativeClass'` (destPort=`nft`, destChannel=`channel-1`) | No escrow on counterparty |
| `IsAwayFromOrigin('nft','channel-0','nft/channel-1/nativeClass')` → `true` | Enters mint branch |
| `prefixedClassID = 'nft/channel-1/nft/channel-1/nativeClass'` | Spurious trace |
| `voucherClassID = ibc/{sha256('nft/channel-1/nft/channel-1/nativeClass')}` | New denom, distinct from legitimate `ibc/{sha256('nft/channel-1/nativeClass')}` |
| `IssueDenom` + `MintNFT` called | Unbacked NFT vouchers created in victim chain state |

The minted vouchers are fully transferable and usable in any protocol that accepts IBC NFTs. They represent no real escrowed asset.

### Likelihood Explanation

The attacker must control the counterparty chain (or its validators) to commit a packet with the crafted `classId`. This is a realistic threat model: IBC channels are established with external chains whose validator sets are independent. A malicious or compromised counterparty chain can commit arbitrary packet data. No relayer compromise or key leak is required — the malicious chain simply includes the crafted packet in its own block.

### Recommendation

In `processReceivedPacket`, before prepending the destination prefix, validate that `data.ClassId` does **not** already start with `destPort/destChannel/`:

```go
destPrefix := types.GetClassPrefix(packet.GetDestPort(), packet.GetDestChannel())
if strings.HasPrefix(data.ClassId, destPrefix) {
    return sdkerrors.Wrapf(types.ErrInvalidClassID,
        "classId %s already contains destination prefix %s", data.ClassId, destPrefix)
}
```

This mirrors the guard used in the ICS-20 fungible token transfer module in ibc-go.

### Proof of Concept

```
Setup:
  Chain B: destPort='nft', destChannel='channel-1'
  Chain A (attacker): sourcePort='nft', sourceChannel='channel-0'

Packet sent by Chain A:
  data.ClassId  = 'nft/channel-1/nativeClass'
  data.TokenIds = ['token1']
  data.TokenUris = ['uri1']
  data.Sender   = <valid bech32>
  data.Receiver = <valid bech32 on Chain B>

Execution on Chain B (processReceivedPacket):
  IsAwayFromOrigin('nft', 'channel-0', 'nft/channel-1/nativeClass')
    → prefixClassID = 'nft/channel-0/'
    → 'nft/channel-1/nativeClass' does NOT start with 'nft/channel-0/'
    → returns true

  classPrefix      = 'nft/channel-1/'
  prefixedClassID  = 'nft/channel-1/nft/channel-1/nativeClass'   ← double-prefixed
  classTrace.Path  = 'nft/channel-1/nft/channel-1'
  classTrace.Base  = 'nativeClass'
  voucherClassID   = 'ibc/' + sha256('nft/channel-1/nft/channel-1/nativeClass')

  IssueDenom(voucherClassID, ...)   ← new spurious denom created
  MintNFT(voucherClassID, 'token1', ..., receiver)  ← unbacked voucher minted

Legitimate voucherClassID would be:
  'ibc/' + sha256('nft/channel-1/nativeClass')   ← completely different hash

Result: receiver on Chain B holds an NFT voucher backed by nothing.
```

```go
func (nftpd NonFungibleTokenPacketData) ValidateBasic() error {
	if strings.TrimSpace(nftpd.ClassId) == "" {
		return newsdkerrors.Wrap(ErrInvalidClassID, "classId cannot be blank")
	}

	if len(nftpd.TokenIds) == 0 {
		return newsdkerrors.Wrap(ErrInvalidTokenID, "tokenId cannot be blank")
	}

	if len(nftpd.TokenIds) != len(nftpd.TokenUris) {
		return newsdkerrors.Wrap(ErrInvalidPacket, "tokenIds and tokenUris lengths do not match")
	}

	if strings.TrimSpace(nftpd.Sender) == "" {
		return newsdkerrors.Wrap(sdkerrors.ErrInvalidAddress, "sender address cannot be blank")
	}

	// decode the sender address
	if _, err := sdk.AccAddressFromBech32(nftpd.Sender); err != nil {
		return newsdkerrors.Wrap(sdkerrors.ErrInvalidAddress, "invalid sender address")
	}

	if strings.TrimSpace(nftpd.Receiver) == "" {
		return newsdkerrors.Wrap(sdkerrors.ErrInvalidAddress, "receiver address cannot be blank")
	}

	// decode the receiver address
	if _, err := sdk.AccAddressFromBech32(nftpd.Receiver); err != nil {
		return newsdkerrors.Wrap(sdkerrors.ErrInvalidAddress, "invalid receiver address")
	}
	return nil
}
