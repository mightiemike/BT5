### Title
`ValidResourceBounds::AllResources` with zero L2/L1DataGas collapses to `L1Gas` in protobuf round-trip, producing wrong transaction hash - (File: `crates/apollo_protobuf/src/converters/transaction.rs`)

### Summary
The protobuf serialization/deserialization of `ValidResourceBounds` uses a zero-value heuristic to distinguish between the `L1Gas` and `AllResources` variants. An `AllResources` transaction with `l2_gas=0` and `l1_data_gas=0` serializes to protobuf with those zeros and is deserialized back as `L1Gas`. Because `get_tip_resource_bounds_hash` includes the `L1_DATA_GAS` felt in the hash only for `AllResources`, the recomputed hash on the receiving node differs from the hash committed by the proposer, breaking the canonicalization invariant across the P2P/sync boundary.

### Finding Description

**Serialization side** (`From<ValidResourceBounds> for protobuf::ResourceBounds`):

```rust

    l1_data_gas: Some(ResourceBounds::default().into()), // zero
},
```

Both `L1Gas` and `AllResources{l2_gas:0, l1_data_gas:0}` produce identical protobuf wire bytes: `{l1_gas: X, l2_gas: 0, l1_data_gas: 0}`.
