Audit Report

## Title
Missing Minimum Timelock Validation in `initialize()` Allows Pool Admin to Bypass LP-Protection Timelock ‚Äî (`metric-periphery/contracts/extensions/OracleValueStopLossExtension.sol`)

## Summary

`OracleValueStopLossExtension.initialize()` validates `drawdownE6` and `decayPerSecondE8` but accepts any value ‚Äî including zero ‚Äî for `timelock`. With `timelock = 0`, the `_afterTimelock` helper returns `block.timestamp`, causing `_requireElapsed` to pass immediately, so a pool admin can propose and execute any stop-loss parameter change atomically in a single transaction, completely defeating the LP-protection guarantee the timelock is designed to enforce.

## Finding Description

In `initialize()` at lines 56‚Äì62, `_validateDrawdown` and `_validateDecay` are called but no equivalent check exists for `timelock`: [1](#0-0) 

`_afterTimelock` computes `block.timestamp + timelock`. With `timelock = 0` this returns `block.timestamp`: [2](#0-1) 

`_requireElapsed` reverts only when `block.timestamp < executeAfter`. With `executeAfter = block.timestamp`, the condition is `block.timestamp < block.timestamp` ‚Äî always `false` ‚Äî so the guard never triggers: [3](#0-2) 

Every timelocked setter (`proposeOracleStopLossDrawdown` / `executeOracleStopLossDrawdown`, `proposeOracleStopLossDecay` / `executeOracleStopLossDecay`, `proposeOracleStopLossHighWatermarks` / `executeOracleStopLossHighWatermarks`) routes through these two helpers, so all become instant when `timelock = 0`. [4](#0-3) 

The most destructive path: calling `proposeOracleStopLossDrawdown(pool, 0)` then `executeOracleStopLossDrawdown(pool)` in the same transaction sets `drawdownE6 = 0`. The `afterSwap` hook then short-circuits at line 217, silently disabling all stop-loss checks for every subsequent swap: [5](#0-4) 

The contract's own NatSpec at line 16 states: *"Drawdown and decay changes are timelocked so LPs can react"* ‚Äî this invariant is broken when `timelock = 0`. [6](#0-5) 

## Impact Explanation

This is an admin-boundary break: the pool admin is semi-trusted only within caps and timelocks. With `timelock = 0`, the pool admin can (1) disable the stop-loss entirely by setting `drawdownE6 ‚Üí 0` in one atomic transaction, then execute a large swap that drains LP value with no circuit-breaker; (2) maximize decay (`decayPerSecondE8 ‚Üí 1e8`) instantly, collapsing all watermarks to zero; (3) zero out watermarks for any bin instantly. All three paths result in direct loss of LP principal, meeting Sherlock thresholds for High severity.

## Likelihood Explanation

Any public pool creator can supply `timelockSeconds = 0` in `extensionInitData` ‚Äî no special privilege beyond pool creation is required. The missing validation is a single omitted bounds check. A deployer who misreads the field as "number of blocks," uses a default zero value, or intentionally omits it produces the same zero-timelock outcome. Once deployed, the pool admin can exploit the condition at any time without further preconditions.

## Recommendation

Add a minimum timelock constant and enforce it in both `initialize` and `proposeOracleStopLossTimelock`:

```solidity
uint32 private constant MIN_TIMELOCK = 1 days; // 86400 seconds

function _validateTimelock(uint256 t) private pure {
    if (t < MIN_TIMELOCK) revert OracleStopLossTimelockTooShort(t);
}
```

Call `_validateTimelock(timelock)` alongside `_validateDrawdown` and `_validateDecay` in `initialize`, and again inside `proposeOracleStopLossTimelock` before storing `newTimelock`.

    _validateDrawdown(newMaxDrawdownE6);
    PoolStopLossSchedule storage sched = _initializedSchedule(pool_);
    uint32 executeAfter = _afterTimelock(pool_);
    sched.pendingDrawdownE6 = uint32(newMaxDrawdownE6);

  function executeOracleStopLossDrawdown(address pool_) external onlyPoolAdmin(pool_) {
    PoolStopLossSchedule storage sched = _initializedSchedule(pool_);
    if (sched.pendingDrawdownExecuteAfter == 0) revert OracleStopLossNoPendingDrawdown(pool_);
    _requireElapsed(sched.pendingDrawdownExecuteAfter);
    uint32 drawdown = sched.pendingDrawdownE6;
    oracleStopLossConfig[pool_].drawdownE6 = drawdown;

  }
