Audit Report

## Title
SwapAllowlistExtension Checks Router Address Instead of Actual Swapper, Allowing Full Allowlist Bypass via MetricOmmSimpleRouter - (File: metric-periphery/contracts/extensions/SwapAllowlistExtension.sol)

## Summary
`SwapAllowlistExtension.beforeSwap` gates swaps by checking the `sender` argument, which is `msg.sender` at the pool level. When `MetricOmmSimpleRouter` calls `pool.swap()`, the pool's `msg.sender` is the router, so the extension evaluates `allowedSwapper[pool][router]` rather than `allowedSwapper[pool][actualUser]`. Any unpermissioned user can bypass the allowlist entirely by routing through the public router, provided the router is allowlisted — which is a necessary precondition for the router to function on the pool at all.

## Finding Description
**Step 1:** `MetricOmmPool.swap` calls `_beforeSwap(msg.sender, ...)`, passing the immediate caller as `sender`. [1](#0-0) 

**Step 2:** `MetricOmmSimpleRouter.exactInputSingle` calls `IMetricOmmPoolActions(params.pool).swap(...)` directly, making the router itself `msg.sender` at the pool. The original caller's address (`msg.sender` at the router) is stored only in transient callback context for payment purposes and is never forwarded to the pool as the swap initiator. [2](#0-1) 

**Step 3:** `SwapAllowlistExtension.beforeSwap` receives `sender = router` and evaluates `allowedSwapper[msg.sender][sender]`, i.e., `allowedSwapper[pool][router]`. If the router is allowlisted (required for any router-mediated swap to succeed), this check passes unconditionally for every caller regardless of whether the actual user is on the allowlist. [3](#0-2) 

This creates an irreconcilable dilemma: if the router is not allowlisted, legitimate allowlisted users cannot use the router; if the router is allowlisted, any arbitrary user bypasses the gate. There is no configuration that simultaneously enforces the allowlist and permits router-mediated swaps. The `allowAllSwappers` escape hatch does not help — it disables the allowlist entirely. [4](#0-3) 

## Impact Explanation
A pool configured with `SwapAllowlistExtension` to restrict trading to a curated set of addresses (KYC'd users, whitelisted market makers, protocol-owned addresses) is completely unprotected against any public user routing through `MetricOmmSimpleRouter`. The allowlist guard silently passes for the router address, allowing unauthorized swaps to execute at oracle-derived prices and drain LP-owned assets. This constitutes a broken core pool invariant and direct loss of LP principal — matching the "broken core pool functionality causing loss of funds" and "admin-boundary break bypassed by an unprivileged path" impact categories.

## Likelihood Explanation
`MetricOmmSimpleRouter` is the primary documented public swap entrypoint. The attack requires no privileged access, no special setup, and no malicious token behavior. The only precondition — that the router is allowlisted — is a natural and necessary operational step for any pool admin who wants the pool to be usable via the standard periphery. Any user who discovers the bypass can exploit it immediately and repeatedly.

## Recommendation
The pool must receive the original user's address rather than the router's address. Two authenticated approaches:

1. **Router encodes caller in `extensionData`**: `MetricOmmSimpleRouter` prepends `msg.sender` to `extensionData` before calling `pool.swap`. `SwapAllowlistExtension.beforeSwap` decodes this address and verifies that `msg.sender` (the pool) is a known factory-registered pool before trusting the embedded address, then checks `allowedSwapper[pool][decodedCaller]`.

2. **Dedicated router allowlist path**: The extension distinguishes between direct callers and router-mediated calls by checking whether `sender` is a known router, and if so, decoding the actual user from `extensionData` with factory-pool authentication to prevent spoofing.

Either approach must authenticate the pool identity before trusting any embedded address to prevent a malicious pool from injecting arbitrary caller addresses.

## Proof of Concept
```
Setup:
  pool = deploy MetricOmmPool with SwapAllowlistExtension
  admin calls extension.setAllowedToSwap(pool, router, true)   // required for router to work
  admin calls extension.setAllowedToSwap(pool, alice, true)    // intended allowlisted user
  bob = arbitrary non-allowlisted address

Attack:
  bob calls MetricOmmSimpleRouter.exactInputSingle({
    pool: pool, recipient: bob, zeroForOne: true, amountIn: X, ...
  })
  → Router calls pool.swap(bob, true, X, ...) with msg.sender = router
  → Pool calls _beforeSwap(sender=router, ...)
  → Extension checks allowedSwapper[pool][router] → true
  → Swap executes; bob receives output tokens
  → allowedSwapper[pool][bob] was never evaluated
```
