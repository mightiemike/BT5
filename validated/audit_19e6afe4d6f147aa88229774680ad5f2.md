### Title
Deprecated `verify_transaction_inclusion` Accepts Internal Merkle-Tree Nodes as Valid Transaction IDs, Enabling Proof Forgery — (`contract/src/lib.rs`)

---

### Summary

`verify_transaction_inclusion` (v1) is still a live, publicly callable contract method with no caller restriction. It blindly trusts the caller-supplied `tx_id` without validating that it is a leaf node (an actual transaction hash) rather than an internal Merkle-tree node. An unprivileged NEAR caller can supply an internal node hash as `tx_id` together with a crafted `merkle_proof`, causing the function to return `true` for a transaction that does not exist on-chain. Any downstream contract consuming this result is deceived into accepting a forged Bitcoin payment proof.

---

### Finding Description

`verify_transaction_inclusion` is marked `#[deprecated]` (a compile-time warning only, not a runtime restriction) and `#[pause]` (callable by anyone when the contract is not paused). It accepts four caller-controlled fields — `tx_id`, `tx_index`, `tx_block_blockhash`, and `merkle_proof` — and returns `true` if:

```
compute_root_from_merkle_proof(tx_id, tx_index, merkle_proof) == header.block_header.merkle_root
```

The function's own docstring acknowledges the flaw:

> "This function may return `true` if the provided `tx_id` is a hash of an internal node in the Merkle tree rather than a valid transaction hash." [1](#0-0) 

The function performs no check that `tx_id` is a leaf. It delegates directly to `merkle_tools::compute_root_from_merkle_proof`, which is a pure mathematical computation that is equally valid for leaf hashes and internal-node hashes. [2](#0-1) 

The `#[deprecated]` attribute does not remove the method from the compiled WASM binary or restrict who may call it at runtime. [3](#0-2) 

---

### Impact Explanation

A downstream NEAR contract (e.g., a bridge, escrow, or payment gateway) that calls `verify_transaction_inclusion` and acts on a `true` result will be deceived into believing a Bitcoin transaction occurred when it did not. This enables cross-chain fraud: an attacker can claim a Bitcoin payment was made, receive the corresponding NEAR-side asset release, and pay nothing on the Bitcoin side.

The corrupted value is the boolean proof result returned to the caller — the contract's canonical mapping of `tx_id → included-in-block` is falsified.

---

### Likelihood Explanation

- The function is callable by **any unprivileged NEAR account** with no role check.
- The 64-byte transaction Merkle-proof forgery is a well-documented Bitcoin protocol vulnerability (BitMEX research, CVE-2012-2459). Blocks containing 64-byte transactions exist on mainnet.
- The attack requires only knowledge of a real block hash stored in the contract and the ability to compute a crafted proof offline — no privileged access, no leaked keys, no social engineering.
- The function is not removed; it remains in the deployed WASM and is reachable via any NEAR RPC call.

---

### Recommendation

Remove `verify_transaction_inclusion` (v1) from the public ABI entirely. Since `verify_transaction_inclusion_v2` already exists as the safe replacement, the v1 method should be deleted or replaced with a hard `env::panic_str("use verify_transaction_inclusion_v2")` body so it is unreachable at runtime, not merely deprecated at compile time. [4](#0-3) 

---

### Proof of Concept

**Setup:** A Bitcoin block B is stored in the contract's `headers_pool` and `mainchain_header_to_height`. Block B contains a 64-byte transaction T at some position P. T's raw bytes are `L || R` (two 32-byte halves).

**Attack:**

1. Attacker computes `L` = first 32 bytes of T, `R` = last 32 bytes of T.
2. `hash(L || R) = hash(T)` is an internal node in B's Merkle tree.
3. Attacker constructs a `merkle_proof` path from this internal node up to B's Merkle root (using the real sibling hashes from B's Merkle tree).
4. Attacker calls (as any unprivileged NEAR account):

```
verify_transaction_inclusion({
    tx_id:              L,          // first 32 bytes of T — not a real tx
    tx_block_blockhash: B.hash,     // real block in the contract
    tx_index:           P * 2,      // index consistent with the internal node
    merkle_proof:       [R, ...],   // R as sibling, then real upper siblings
    confirmations:      0,
})
```

5. `compute_root_from_merkle_proof(L, index, [R, ...])` computes `hash(L || R) = hash(T)`, which is a real internal node, and the chain of siblings produces B's actual Merkle root.
6. The function returns `true`.
7. A downstream contract that trusted this result releases funds for a Bitcoin transaction that never existed. [5](#0-4)

    #[deprecated(
        since = "0.5.0",
        note = "Use `verify_transaction_inclusion_v2` instead."
    )]
    #[pause]
    pub fn verify_transaction_inclusion(&self, #[serializer(borsh)] args: ProofArgs) -> bool {
        require!(
            args.confirmations <= self.gc_threshold,
            "The required number of confirmations exceeds the number of blocks stored in memory"
        );

        let heaviest_block_header = self
            .headers_pool
            .get(&self.mainchain_tip_blockhash)
            .unwrap_or_else(|| env::panic_str(ERR_KEY_NOT_EXIST));
        let target_block_height = self
            .mainchain_header_to_height
            .get(&args.tx_block_blockhash)
            .unwrap_or_else(|| env::panic_str("block does not belong to the current main chain"));

        // Check requested confirmations. No need to compute proof if insufficient confirmations.
        require!(
            (heaviest_block_header.block_height).saturating_sub(target_block_height) + 1
                >= args.confirmations,
            "Not enough blocks confirmed"
        );

        let header = self
            .headers_pool
            .get(&args.tx_block_blockhash)
            .unwrap_or_else(|| env::panic_str("cannot find requested transaction block"));

        require!(!args.merkle_proof.is_empty(), "Merkle proof is empty");

        // compute merkle tree root and check if it matches block's original merkle tree root
        merkle_tools::compute_root_from_merkle_proof(
            args.tx_id,
            usize::try_from(args.tx_index).unwrap(),
            &args.merkle_proof,
        ) == header.block_header.merkle_root
    }
