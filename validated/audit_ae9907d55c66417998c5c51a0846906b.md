### Title
Fee-Tier Default Override Flag Stored at Tier Granularity Instead of Tier×Product Granularity Causes Zero Taker Fees for Unconfigured Products — (File: `core/contracts/OffchainExchange.sol`)

---

### Summary

`OffchainExchange.sol` uses a single bitmask `nonDefaultFeeTierMask` to record whether a fee tier has been customized. However, the actual fee rates are stored at `feeRates[tier][productId]` — a finer tier×product granularity. When `updateTierFeeRates` is called with a specific `productId` (not `QUOTE_PRODUCT_ID`), it marks the **entire tier** as non-default in `nonDefaultFeeTierMask` while only writing rates for **one product**. Every other product at that tier then returns `{makerRateX18: 0, takerRateX18: 0}` from `getTierFeeRateX18` instead of the intended default of 2 bps taker fee, because the mask causes the default branch to be bypassed.

---

### Finding Description

`nonDefaultFeeTierMask` is a `uint128` with one bit per tier: [1](#0-0) 

`getTierFeeRateX18` uses this mask to decide whether to return the stored rate or the hardcoded default: [2](#0-1) 

`updateTierFeeRates` has two branches. When `txn.productId == QUOTE_PRODUCT_ID`, it iterates all current products and writes rates for each. When `txn.productId` is any other value, it writes rates for **only that one product** — but in both cases it unconditionally sets the tier's bit in `nonDefaultFeeTierMask`: [3](#0-2) 

The result is a granularity mismatch identical in class to the Diamond proxy bug: the "has been customized" flag lives at tier granularity, but the actual data lives at tier×product granularity. After a single-product call, `getTierFeeRateX18(T, P_other)` returns `feeRates[T][P_other]` which is zero-initialized `{0, 0}` for every product that was not explicitly written, instead of the default `{0, 200_000_000_000_000}`.

The same inconsistency arises when a new product is listed after a tier has already been marked non-default: `feeRates[T][newProduct]` is never populated, so the new product silently inherits zero fees for all non-default tiers.

---

### Impact Explanation

Any user whose address has been assigned a non-default fee tier (via `updateFeeTier`) can trade any product that was not explicitly configured for that tier and pay **zero taker fees** instead of 2 bps. The protocol loses all taker-fee revenue on those trades. Because `nonDefaultFeeTierMask` bits are never cleared, the condition is permanent once triggered. The corrupted state is `feeRates[tier][productId]` returning `{0, 0}` for unconfigured products, and `market.collectedFees` accumulating zero instead of the expected fee amount.

---

### Likelihood Explanation

The trigger is a normal, expected admin operation: setting a custom fee rate for a specific product at a specific tier (e.g., granting a market maker a rebate on one product). This is a realistic protocol management action. Once triggered, the zero-fee condition is immediately active for all other products at that tier and affects every user assigned to it. No attacker capability beyond holding a non-default fee tier is required to benefit.

---

### Recommendation

Replace the single-bit-per-tier `nonDefaultFeeTierMask` with a per-tier-per-product tracking structure, or change `getTierFeeRateX18` to fall back to the default only when the stored rate is zero (using a sentinel value or an explicit `isConfigured` flag at the tier×product level). Alternatively, require that `updateTierFeeRates` with a specific `productId` never sets the tier-level bit unless all products for that tier have been explicitly configured.

---

### Proof of Concept

1. Admin calls `updateTierFeeRates` via slow-mode with `tier = 1`, `productId = 5`, `makerRateX18 = 0`, `takerRateX18 = 100_000_000_000_000` (1 bps).
   - `feeRates[1][5] = {0, 1bps}` ✓
   - `nonDefaultFeeTierMask |= (1 << 1)` → tier 1 is now marked non-default for **all** products.
2. Admin assigns tier 1 to address `0xAlice` via `updateFeeTier`.
3. Alice submits a taker order for product 7 (not product 5) through the off-chain exchange.
4. The sequencer matches the order; `applyFee` calls `getUserFeeRateWithBuilder` → `getTierFeeRateX18(1, 7)`.
5. `nonDefaultFeeTierMask & (1 << 1) != 0` → returns `feeRates[1][7]` = `{0, 0}` (zero-initialized).
6. Alice's taker fee is **zero** instead of 2 bps. `market.collectedFees` receives nothing. The protocol loses fee revenue on every subsequent trade Alice makes on product 7. [2](#0-1) [4](#0-3) [5](#0-4)

    function getUserFeeRateWithBuilder(
        bytes32 sender,
        uint32 productId,
        uint128 appendix,
        bool taker
    ) internal view returns (FeeInfo memory) {
        (uint32 builderId, int128 builderFeeRate) = _builderInfo(appendix);
        Builder memory builder;
        if (builderId != 0) {
            builder = builders[builderId];
            if (
                builder.owner == address(0) ||
                builderFeeRate > builder.highestFeeRate ||
                builderFeeRate < builder.lowestFeeRate
            ) {
                revert(ERR_INVALID_BUILDER);
            }
        } else if (builderFeeRate != 0) {
            revert(ERR_INVALID_BUILDER);
        }

        uint32 feeTier = feeTiers[address(uint160(bytes20(sender)))];
        if (feeTier < builder.defaultFeeTier) {
            feeTier = builder.defaultFeeTier;
        }
        FeeRates memory userFeeRates = getTierFeeRateX18(feeTier, productId);
        int128 feeRate = taker
            ? userFeeRates.takerRateX18
            : userFeeRates.makerRateX18;
        return FeeInfo(feeRate, builderId, builderFeeRate);
    }

    function getTierFeeRateX18(uint32 tier, uint32 productId)
        public
        view
        returns (FeeRates memory)
    {
        if (nonDefaultFeeTierMask & (1 << tier) != 0) {
            return feeRates[tier][productId];
        }
        return
            FeeRates({
                makerRateX18: 0,
                takerRateX18: 200_000_000_000_000 // 2 bps
            });
    }

    function updateTierFeeRates(IEndpoint.UpdateTierFeeRates memory txn)
        external
        onlyEndpoint
    {
        if (txn.productId == QUOTE_PRODUCT_ID) {
            uint32[] memory spotProductIds = spotEngine.getProductIds();
            uint32[] memory perpProductIds = perpEngine.getProductIds();
            for (uint32 i = 0; i < spotProductIds.length; i++) {
                if (spotProductIds[i] == QUOTE_PRODUCT_ID) {
                    continue;
                }
                feeRates[txn.tier][spotProductIds[i]] = FeeRates(
                    txn.makerRateX18,
                    txn.takerRateX18
                );
            }
            for (uint32 i = 0; i < perpProductIds.length; i++) {
                feeRates[txn.tier][perpProductIds[i]] = FeeRates(
                    txn.makerRateX18,
                    txn.takerRateX18
                );
            }
        } else {
            feeRates[txn.tier][txn.productId] = FeeRates(
                txn.makerRateX18,
                txn.takerRateX18
            );
        }
        nonDefaultFeeTierMask |= uint128(1) << txn.tier;
    }
