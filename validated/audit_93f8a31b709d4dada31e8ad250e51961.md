### Title
Deferred Cost Multiplier Enforcement in `op_unknown` Allows `run_program` to Return `Ok` with Cost Exceeding `max_cost` — (File: `src/more_ops.rs`)

---

### Summary

`op_unknown` in `src/more_ops.rs` enforces the cost budget **before** applying the opcode's cost multiplier. When an unknown opcode with a non-zero multiplier is the **last** operation in a CLVM program, `run_program` returns `Ok(Reduction(cost, ...))` where `cost > max_cost`, violating the function's contract and corrupting the cost value returned to callers.

---

### Finding Description

`op_unknown` computes a base cost from the opcode's argument list (using one of four cost functions), calls `check_cost(cost, max_cost)` to enforce the budget, and **then** multiplies the cost by `(cost_multiplier + 1)`: [1](#0-0) 

```rust
assert!(cost > 0);

check_cost(cost, max_cost)?;          // ← enforces budget on PRE-multiplied cost
cost *= cost_multiplier + 1;          // ← multiplier applied AFTER the check
if cost > u32::MAX as u64 {
    Err(EvalErr::Invalid(o))?
} else {
    Ok(Reduction(cost as Cost, allocator.nil()))
}
```

The only post-multiplication guard is `cost > u32::MAX`, **not** `cost > max_cost`. The `cost_multiplier` is fully attacker-controlled: it is decoded from the opcode bytes supplied in the CLVM program: [2](#0-1) 

```rust
let cost_multiplier: u64 = match u32_from_u8(&op[0..op.len() - 1]) {
    Some(v) => v as u64,
    None => { return Err(EvalErr::Invalid(o))?; }
};
```

In `run_program`, the cost limit is checked at the **top** of the dispatch loop, **before** each operation executes: [3](#0-2) 

```rust
loop {
    let effective_max_cost = ...;
    if cost > effective_max_cost {          // ← check is at top, before op executes
        return Err(EvalErr::CostExceeded);
    }
    let top = self.op_stack.pop();
    let op = match top { Some(f) => f, None => break };
    cost += match op {
        Operation::Apply => self.apply_op(cost, effective_max_cost - cost)?,
        ...
    };
}
Ok(Reduction(cost, self.pop()?))           // ← no final cost check before returning Ok
```

`apply_op` passes the **remaining budget** (`effective_max_cost - cost`) as `max_cost` to `op_unknown`: [4](#0-3) 

When `op_unknown` is the **last** operation on the op-stack, the sequence is:

1. `check_cost(pre_cost, remaining_budget)` passes — `pre_cost ≤ remaining_budget`
2. `cost *= (multiplier + 1)` — post-multiplied cost can be up to `u32::MAX ≈ 4.3 billion`
3. `apply_op` returns the inflated cost; `run_program` adds it to the running total
4. The loop pops `None` from the op-stack and **breaks** — the top-of-loop check never fires again
5. `Ok(Reduction(total_cost, ...))` is returned with `total_cost > effective_max_cost`

---

### Impact Explanation

`run_program` is the sole execution entry point for all CLVM programs on the Chia blockchain. Its API contract is that returning `Ok` means the program succeeded **within** `max_cost`. Callers rely on this invariant for block cost accounting and fee validation.

When the invariant is broken:

- **Consensus divergence**: A node that re-checks the returned cost will reject the block; a node that trusts `Ok` implies within-budget will accept it. Different implementations or versions of the validator can disagree on block validity.
- **Incorrect cost accounting**: The inflated cost value propagates to block-level cost tallies, potentially allowing a transaction to consume more block space than its declared fee covers, or causing valid transactions to be incorrectly rejected.
- **Corrupted `Reduction` value**: The `Cost` field of the returned `Reduction` is the primary output used by the Chia full node to enforce the 11-billion-cost block limit. An attacker-controlled inflation of this value is a direct attack on that enforcement.

---

### Likelihood Explanation

- Unknown ops are enabled in **consensus mode** (the default for full-node block validation); `MEMPOOL_MODE` sets `NO_UNKNOWN_OPS` and is unaffected.
- The attacker controls the CLVM bytes directly (puzzle or solution bytes in a coin spend).
- Crafting the trigger requires only: (a) a 1–4 byte opcode not in the known set, not starting with `0xffff`, with a large multiplier encoded in the high bytes; (b) the opcode being the last evaluated expression.
- No special privileges, social engineering, or compromised infrastructure are required.

---

### Recommendation

Move the `check_cost` call to **after** the multiplier is applied, or add a second `check_cost` after multiplication:

```rust
// In op_unknown, src/more_ops.rs
check_cost(cost, max_cost)?;          // keep pre-multiplier check for early exit
cost *= cost_multiplier + 1;
if cost > u32::MAX as u64 {
    Err(EvalErr::Invalid(o))?
} else {
    check_cost(cost, max_cost)?;      // ← add post-multiplier enforcement
    Ok(Reduction(cost as Cost, allocator.nil()))
}
```

Alternatively, add a final cost check after the `run_program` loop before returning `Ok`:

```rust
// In run_program, src/run_program.rs, after the loop
if cost > max_cost {
    return Err(EvalErr::CostExceeded);
}
Ok(Reduction(cost, self.pop()?))
```

### Proof of Concept

Construct a CLVM program whose sole evaluated expression is an unknown opcode with:
- `cost_function = 0` (constant cost = 1, encoded in bits 7–6 of the last byte as `0b00`)
- `cost_multiplier` = `remaining_budget` (encoded in the preceding bytes)

For example, with `max_cost = 10_000`:

```
opcode bytes: [0x00, 0x27, 0x10, 0x00]
  ↑ high 3 bytes = 0x002710 = 10000 (cost_multiplier)
  ↑ low byte = 0x00 → cost_function = 0, 6 ignored bits = 0
```

- `pre_cost = 1`
- `check_cost(1, 10_000)` → passes
- `post_cost = 1 × (10_000 + 1) = 10_001 > 10_000`
- `10_001 ≤ u32::MAX` → returns `Ok(Reduction(10_001, nil))`
- `run_program` returns `Ok(Reduction(10_001, nil))` with `max_cost = 10_000`

The returned cost exceeds `max_cost` despite `run_program` returning `Ok`. [5](#0-4) [6](#0-5)

pub fn op_unknown(
    allocator: &mut Allocator,
    o: NodePtr,
    mut args: NodePtr,
    max_cost: Cost,
) -> Response {
    // unknown opcode in lenient mode
    // unknown ops are reserved if they start with 0xffff
    // otherwise, unknown ops are no-ops, but they have costs. The cost is computed
    // like this:

    // byte index (reverse):
    // | 4 | 3 | 2 | 1 | 0          |
    // +---+---+---+---+------------+
    // | multiplier    |XX | XXXXXX |
    // +---+---+---+---+---+--------+
    //  ^               ^    ^
    //  |               |    + 6 bits ignored when computing cost
    // cost_multiplier  |
    // (up to 4 bytes)  + 2 bits
    //                    cost_function

    // 1 is always added to the multiplier before using it to multiply the cost, this
    // is since cost may not be 0.

    // cost_function is 2 bits and defines how cost is computed based on arguments:
    // 0: constant, cost is 1 * (multiplier + 1)
    // 1: computed like operator add, multiplied by (multiplier + 1)
    // 2: computed like operator mul, multiplied by (multiplier + 1)
    // 3: computed like operator concat, multiplied by (multiplier + 1)

    // this means that unknown ops where cost_function is 1, 2, or 3, may still be
    // fatal errors if the arguments passed are not atoms.

    let op_atom = allocator.atom(o);
    let op = op_atom.as_ref();

    if op.is_empty() || (op.len() >= 2 && op[0] == 0xff && op[1] == 0xff) {
        Err(EvalErr::Reserved(o))?;
    }

    let cost_function = (op[op.len() - 1] & 0b11000000) >> 6;
    let cost_multiplier: u64 = match u32_from_u8(&op[0..op.len() - 1]) {
        Some(v) => v as u64,
        None => {
            return Err(EvalErr::Invalid(o))?;
        }
    };

    let mut cost = match cost_function {
        0 => 1,
        1 => {
            let mut cost = ARITH_BASE_COST;
            let mut byte_count: u64 = 0;
            while let Some((arg, rest)) = allocator.next(args) {
                args = rest;
                cost += ARITH_COST_PER_ARG;
                let len = atom_len(allocator, arg, "unknown op")?;
                byte_count += len as u64;
                check_cost(cost + (byte_count as Cost * ARITH_COST_PER_BYTE), max_cost)?;
            }
            cost + (byte_count * ARITH_COST_PER_BYTE)
        }
        2 => {
            let mut cost = MUL_BASE_COST;
            let mut first_iter: bool = true;
            let mut l0: u64 = 0;
            while let Some((arg, rest)) = allocator.next(args) {
                args = rest;
                let len = atom_len(allocator, arg, "unknown op")?;
                if first_iter {
                    l0 = len as u64;
                    first_iter = false;
                    continue;
                }
                let l1 = len as u64;
                cost += MUL_COST_PER_OP;
                cost += (l0 + l1) * MUL_LINEAR_COST_PER_BYTE;
                cost += (l0 * l1) / MUL_SQUARE_COST_PER_BYTE_DIVIDER;
                l0 += l1;
                check_cost(cost, max_cost)?;
            }
            cost
        }
        3 => {
            let mut cost = CONCAT_BASE_COST;
            while let Some((arg, rest)) = allocator.next(args) {
                args = rest;
                let len = atom_len(allocator, arg, "unknown op")?;
                cost += CONCAT_COST_PER_ARG;
                cost += CONCAT_COST_PER_BYTE * (len as Cost);
                check_cost(cost, max_cost)?;
            }
            cost
        }
        _ => 1,
    };

    assert!(cost > 0);

    check_cost(cost, max_cost)?;
    cost *= cost_multiplier + 1;
    if cost > u32::MAX as u64 {
        Err(EvalErr::Invalid(o))?
    } else {
        Ok(Reduction(cost as Cost, allocator.nil()))
    }
}

    pub fn run_program(&mut self, program: NodePtr, env: NodePtr, max_cost: Cost) -> Response {
        self.val_stack = vec![];
        self.op_stack = vec![];

        // max_cost is always in effect, and necessary to prevent wrap-around of
        // the cost integer.
        let max_cost = if max_cost == 0 { Cost::MAX } else { max_cost };
        // We would previously allocate an atom to hold the max cost for the program.
        // Since we don't anymore we need to increment the ghost atom counter to remain
        // backwards compatible with the atom count limit
        self.allocator.add_ghost_atom(1)?;
        let mut cost: Cost = 0;

        cost += self.eval_pair(program, env)?;

        loop {
            // if we are in a softfork guard, temporarily use the guard's
            // expected cost as the upper limit. This lets us fail early in case
            // it's wrong. It's guaranteed to be <= max_cost, because we check
            // that when entering the softfork guard
            let effective_max_cost = if let Some(sf) = self.softfork_stack.last() {
                sf.expected_cost
            } else {
                max_cost
            };

            if cost > effective_max_cost {
                return Err(EvalErr::CostExceeded);
            }
            let top = self.op_stack.pop();
            let op = match top {
                Some(f) => f,
                None => break,
            };
            cost += match op {
                Operation::Apply => self.apply_op(cost, effective_max_cost - cost)?,
                Operation::ExitGuard => self.exit_guard(cost)?,
                Operation::Cons => self.cons_op()?,
                Operation::SwapEval => self.swap_eval_op()?,
                Operation::RestoreAllocator => {
                    let Some(checkpoint) = self.allocator_stack.pop() else {
                        return Err(EvalErr::InternalError(
                            NodePtr::NIL,
                            "allocator checkpoint stack empty".to_string(),
                        ));
                    };
                    let Some(&top) = self.val_stack.last() else {
                        return Err(EvalErr::InternalError(
                            NodePtr::NIL,
                            "value stack empty".to_string(),
                        ));
                    };
                    match self.allocator.maybe_restore_with_node(&checkpoint, top)? {
                        MaybeRestore::NoReplace => {}
                        MaybeRestore::Replace(new_node) => {
                            self.val_stack.pop().unwrap();
                            self.val_stack.push(new_node);
                        }
                        MaybeRestore::Aborted => {}
                    }
                    0
                }
                #[cfg(feature = "pre-eval")]
                Operation::PostEval => {
                    let f = self.posteval_stack.pop().unwrap();
                    let peek: Option<NodePtr> = self.val_stack.last().copied();
                    f(self.allocator, peek);
                    0
                }
            };
        }
        self.allocator.clear_validation_caches();
        Ok(Reduction(cost, self.pop()?))
    }
