### Title
Checkpoint Advance Without Bonus Transfer in `slashRedelegationPosition` Causes Permanent Loss of Accrued Bonus Rewards — (`x/tieredrewards/keeper/slash.go`, `x/tieredrewards/keeper/claim_rewards.go`)

---

### Summary

When `BeforeRedelegationSlashed` fires and the `RewardsPoolName` module account has insufficient balance, `slashRedelegationPosition` swallows `ErrInsufficientBonusPool` but still persists the in-memory `pos` struct whose bonus checkpoints (`LastBonusAccrual`, `LastKnownBonded`, `LastEventSeq`) were already advanced inside `processEventsAndClaimBonus` — and whose validator events were already reference-count-decremented — before the pool check ran. No bonus coins are transferred to the owner, yet the position is saved with fully advanced checkpoints. The accrued bonus is permanently and irrecoverably lost.

---

### Finding Description

**Step 1 — Entry point.**
`BeforeRedelegationSlashed` is a standard staking hook that fires whenever the SDK's `SlashRedelegation` processes a redelegation entry. It is reachable via any double-sign or downtime slash on a validator that has a tiered-rewards position redelegated to it. [1](#0-0) 

**Step 2 — `processEventsAndClaimBonus` mutates checkpoints before the pool check.**
Inside `processEventsAndClaimBonus`, the event loop at lines 172–199 calls `pos.UpdateLastEventSeq(entry.Seq)` and `k.decrementEventRefCount(...)` for every pending event. After the loop, `applyBonusAccrualCheckpoint` advances `LastBonusAccrual` to block time and `pos.UpdateLastKnownBonded` updates `LastKnownBonded`. Only then, at line 230, is `sufficientBonusPoolBalance` called. [2](#0-1) [3](#0-2) 

**Step 3 — Error is swallowed in `slashRedelegationPosition`.**
`slashRedelegationPosition` passes `&pos` (a pointer) to `processEventsAndClaimBonus`. When the pool is insufficient, the function returns `ErrInsufficientBonusPool` with the `pos` struct already mutated in memory. The caller explicitly catches and discards this error: [4](#0-3) 

**Step 4 — Mutated checkpoints are persisted.**
Execution continues with the now-mutated `pos`. For a partial slash, `setPositionWithState(ctx, pos, nil)` is called unconditionally, writing the advanced `LastBonusAccrual`, `LastEventSeq`, and `LastKnownBonded` to the store without any bonus having been paid. [5](#0-4) [6](#0-5) 

**Step 5 — Events are also consumed.**
`decrementEventRefCount` is called inside the loop before the pool check. If an event's reference count reaches zero it is deleted from the store. These deletions are not rolled back when the pool check fails, so the events are gone permanently. [7](#0-6) [8](#0-7) 

**Step 6 — Future `ClaimTierRewards` pays zero additional bonus.**
The next call to `processEventsAndClaimBonus` for this position starts from the already-advanced `LastBonusAccrual` and `LastEventSeq`. The accrual window that was computed but never paid is gone; the owner receives nothing for that period.

---

### Impact Explanation

- **Exact state delta:** `pos.LastBonusAccrual` advances to block time, `pos.LastEventSeq` advances past all processed events, `pos.LastKnownBonded` is updated — all without a corresponding `SendCoinsFromModuleToAccount` to the owner.
- **Fund loss:** The owner permanently loses all bonus rewards that accrued between the previous `LastBonusAccrual` and the slash block time.
- **No recourse:** Refilling the pool and calling `ClaimTierRewards` afterwards pays zero additional bonus for the lost window, because the checkpoints already cover it.
- **Scope:** Matches "permanent, irrecoverable loss of accrued bonus rewards for the position owner."

---

### Likelihood Explanation

The `RewardsPoolName` account balance can fall below the accrued bonus amount through normal operation: the pool is funded by governance/inflation and depleted by ongoing claims. Any validator double-sign or downtime slash that fires `BeforeRedelegationSlashed` while the pool is even 1 token short of the accrued bonus triggers the bug. No attacker privilege is required; a naturally occurring slash event suffices.

---

### Recommendation

Move `applyBonusAccrualCheckpoint`, `pos.UpdateLastKnownBonded`, `pos.UpdateLastEventSeq`, and `decrementEventRefCount` to execute **only after** `sufficientBonusPoolBalance` succeeds and `SendCoinsFromModuleToAccount` completes. Alternatively, in `slashRedelegationPosition`, do not call `setPositionWithState` with the mutated `pos` when `ErrInsufficientBonusPool` is returned; instead revert to the original position state (e.g., re-fetch from store or snapshot before the call).

---

### Proof of Concept

```
1. Create a tier position with a redelegation to validator V.
2. Let bonus accrue for N blocks (e.g., 1000 tokens owed).
3. Fund RewardsPoolName with exactly 999 tokens (1 short).
4. Trigger a double-sign slash on V, causing BeforeRedelegationSlashed to fire.
5. Assert: owner's bank balance is unchanged (no bonus paid). ✓
6. Assert: pos.LastBonusAccrual == block time (checkpoint advanced). ✓
7. Assert: pos.LastEventSeq advanced past all events. ✓
8. Refill RewardsPoolName with 10000 tokens.
9. Call MsgClaimTierRewards for the position.
10. Assert: bonus returned == 0 (the accrual window is gone). ✓
```

```go
func (k Keeper) processEventsAndClaimBonus(ctx context.Context, pos *types.PositionState) (sdk.Coins, error) {
	// Rewards should have been claimed before undelegation
	if !pos.IsDelegated() {
		return sdk.NewCoins(), nil
	}

	valAddr, err := sdk.ValAddressFromBech32(pos.Delegation.ValidatorAddress)
	if err != nil {
		return nil, err
	}

	events, err := k.getValidatorEventsSince(ctx, valAddr, pos.LastEventSeq)
	if err != nil {
		return nil, err
	}

	sdkCtx := sdk.UnwrapSDKContext(ctx)
	blockTime := sdkCtx.BlockTime()

	totalBonus := math.ZeroInt()
	// Use the persisted bonded state from the last replay, not a hardcoded default.
	// This prevents overpaying bonus for unbonded gaps between claims.
	bonded := pos.LastKnownBonded
	segmentStart := pos.LastBonusAccrual

	tier, err := k.getTier(ctx, pos.TierId)
	if err != nil {
		return nil, err
	}

	for _, entry := range events {
		evt := entry.Event

		if bonded {
			// Compute bonus for the bonded segment [segmentStart, eventTime]
			// using the snapshot rate at the event.
			bonus := k.computeSegmentBonus(*pos, tier, segmentStart, evt.Timestamp, evt.TokensPerShare)
			totalBonus = totalBonus.Add(bonus)
		}

		// Update bonded state based on event type.
		switch evt.EventType {
		case types.ValidatorEventType_VALIDATOR_EVENT_TYPE_UNBOND:
			bonded = false
		case types.ValidatorEventType_VALIDATOR_EVENT_TYPE_BOND:
			bonded = true
		case types.ValidatorEventType_VALIDATOR_EVENT_TYPE_SLASH:
			// Slash doesn't change bonded state.
		}

		segmentStart = evt.Timestamp
		pos.UpdateLastEventSeq(entry.Seq)

		// Decrement reference count.
		if err := k.decrementEventRefCount(ctx, valAddr, entry.Seq); err != nil {
			return nil, err
		}
	}

	val, err := k.stakingKeeper.GetValidator(ctx, valAddr)
	if err != nil {
		return nil, err
	}
	// Defensive: validator bond status check
	if bonded && val.IsBonded() {
		currentRate, err := k.getTokensPerShare(ctx, valAddr)
		if err != nil {
			return nil, err
		}
		bonus := k.computeSegmentBonus(*pos, tier, segmentStart, blockTime, currentRate)
		totalBonus = totalBonus.Add(bonus)
	}

	applyBonusAccrualCheckpoint(&pos.Position, blockTime)
	// Persist the bonded state so the next replay starts correctly.
	pos.UpdateLastKnownBonded(bonded)

	if totalBonus.IsZero() {
		return sdk.NewCoins(), nil
	}

	bondDenom, err := k.stakingKeeper.BondDenom(ctx)
	if err != nil {
		return nil, err
	}

	bonusCoins := sdk.NewCoins(sdk.NewCoin(bondDenom, totalBonus))

	if err := k.sufficientBonusPoolBalance(ctx, bonusCoins); err != nil {
		return nil, err
	}
```

```go
func (k Keeper) slashRedelegationPosition(ctx context.Context, unbondingId uint64, sharesToUnbond math.LegacyDec) error {
	positionId, err := k.getRedelegationMapping(ctx, unbondingId)
	if errors.Is(err, collections.ErrNotFound) {
		return nil
	}
	if err != nil {
		return err
	}

	pos, err := k.getPositionState(ctx, positionId)
	if errors.Is(err, types.ErrPositionNotFound) {
		k.logger(ctx).Error("position not found during redelegation slash",
			"position_id", positionId,
			"unbonding_id", unbondingId,
			"error", err.Error(),
		)
		return nil
	}
	if err != nil {
		return err
	}

	if !pos.IsDelegated() {
		// Defensive
		k.logger(ctx).Error("delegation missing during BeforeRedelegationSlashed",
			"position_id", positionId,
			"unbonding_id", unbondingId,
			"shares_to_unbond", sharesToUnbond.String(),
		)
		return nil
	}

	dstValStr := pos.Delegation.ValidatorAddress

	// Settle bonus against PRE-slash shares.
	if _, err := k.processEventsAndClaimBonus(ctx, &pos); err != nil {
		// Deliberately forgo bonus rewards if pool is insufficient to prevent chain halt.
		if errors.Is(err, types.ErrInsufficientBonusPool) {
			k.logger(ctx).Error("insufficient bonus pool during redelegation slash",
				"position_id", pos.Id,
				"error", err.Error(),
			)
		} else {
			return err
		}
	}

	fullSlash := sharesToUnbond.GTE(pos.Delegation.Shares)

	if fullSlash {
		pos.Delegation = nil
		pos.ClearBonusCheckpoints()
		return k.setPositionWithState(ctx, pos, &ValidatorTransition{PreviousAddress: dstValStr})
	}
	// In-memory only: the persisted Position carries no share count, and the
	// live delegation will reflect the post-Unbond shares on the next read.
	// Update the local copy so any follow-up logic in this call sees consistent state.
	pos.Delegation.Shares = pos.Delegation.Shares.Sub(sharesToUnbond)
	return k.setPositionWithState(ctx, pos, nil)
```

```go
func (k Keeper) setPositionWithState(ctx context.Context, state types.PositionState, update *ValidatorTransition) error {
	if err := state.Validate(); err != nil {
		return err
	}

	pos := state.Position

	oldPos, err := k.getPosition(ctx, pos.Id)
	isNew := errors.Is(err, types.ErrPositionNotFound)
	if !isNew && err != nil {
		return err
	}

	if err := k.Positions.Set(ctx, pos.Id, pos); err != nil {
		return err
	}
```
