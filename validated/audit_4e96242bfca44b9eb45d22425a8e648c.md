### Title
Python Object Identity Aliasing in `clvm_tree_to_lazy_node` Causes Corrupted CLVM Tree Output — (File: `wheel/src/api.rs`)

---

### Summary

`clvm_tree_to_lazy_node` in `wheel/src/api.rs` tracks visited Python CLVM nodes using their raw memory address (`pyobj.as_ptr() as usize`) as the key in `identity_map`. Because `LazyNode.pair` creates fresh Python objects on every call, child nodes are freed mid-traversal and their memory addresses are immediately reused by CPython for new, structurally different nodes. The new node is then silently aliased to the old node's `NodePtr`, producing a corrupted CLVM tree with wrong children — a direct analog to the original bug's use of a mutable recipient address instead of a stable operator key.

---

### Finding Description

`clvm_tree_to_lazy_node` converts any Python object implementing the `CLVMStorage` protocol (`.atom` / `.pair`) into a Rust-backed `LazyNode`. To avoid re-traversing shared subtrees it maintains:

```rust
let mut identity_map: HashMap<usize, NodePtr> = HashMap::new();
```

where the key is `pyobj.as_ptr() as usize` — the raw CPython heap address of the Python object. [1](#0-0) 

When a pair node is encountered and one child is already in `identity_map` (`left_done = true, right_done = false`), the code pushes a `BuildPair` work item storing only the raw `usize` IDs, then pushes `Visit(right)` (keeping `right` alive) but **drops `left`**: [2](#0-1) 

The `left` `Bound<'py, PyAny>` is dropped at the end of the match arm. In CPython, reference-counted objects are immediately deallocated when their refcount reaches zero, and the freed memory is placed on a per-size free list for immediate reuse.

`LazyNode.pair` creates brand-new Python objects on every call: [3](#0-2) 

`LazyNode` is a valid `CLVMStorage` input to `clvm_tree_to_lazy_node` (it is declared as a subclass of `CLVMStorage` in the Python stubs): [4](#0-3) 

**Concrete trigger sequence** for a tree `Root = (A . B)` where `A = (C . D)` and `B = (E . F)` with all distinct content:

1. Visit `Root` → `LazyNode.pair` creates `A_obj`, `B_obj`. Push `BuildPair{root_id, A_id, B_id}`, `Visit(B_obj)`, `Visit(A_obj)`.
2. Pop `Visit(A_obj)` → `LazyNode.pair` creates `C_obj`, `D_obj`. Push `BuildPair{A_id, C_id, D_id}`, `Visit(D_obj)`, `Visit(C_obj)`. **`A_obj` is dropped** (moved out of the match arm).
3. Pop `Visit(C_obj)` → atom, insert `identity_map[C_id] = NodePtr_C`. **`C_obj` is dropped**; CPython immediately frees its heap slot.
4. Pop `Visit(D_obj)` → atom, insert `identity_map[D_id] = NodePtr_D`. `D_obj` dropped.
5. Pop `BuildPair{A_id, C_id, D_id}` → builds pair `(NodePtr_C, NodePtr_D)`, inserts `identity_map[A_id]`.
6. Pop `Visit(B_obj)` → `LazyNode.pair` creates `E_obj`, `F_obj`. CPython's free list **reuses the slot freed in step 3**, so `E_obj.as_ptr() == C_obj.as_ptr()`, i.e. `E_id == C_id`.
7. `left_done = identity_map.contains_key(&E_id)` → **`true`** (stale hit from step 3).
8. `identity_map[E_id]` returns `NodePtr_C` — the wrong node. `B` is built as `(NodePtr_C . NodePtr_F)` instead of `(NodePtr_E . NodePtr_F)`. [5](#0-4) 

The `BuildPair` handler then blindly reads the stale entry: [6](#0-5) 

---

### Impact Explanation

**Impact: High.**

The returned `LazyNode` silently contains the wrong child node. Any downstream operation — `ser_legacy`, `ser_backrefs`, `ser_2026`, `run_serialized_chia_program` — operates on the corrupted tree. Concretely:

- **Serialization divergence**: two nodes that deserialize the same blob and re-serialize via `clvm_tree_to_lazy_node` may produce different bytes depending on CPython's allocator state, breaking consensus.
- **Tree-hash mismatch**: the SHA-256 tree hash of the output differs from the hash of the intended tree, invalidating coin-spend signatures that commit to the puzzle hash.
- **Wrong program execution**: if the corrupted tree is executed, a different CLVM program runs, potentially with different cost and output.

---

### Likelihood Explanation

**Likelihood: Low-Medium.**

- `clvm_tree_to_lazy_node` is a public Python API (`from clvm_rs.serde import clvm_tree_to_lazy_node`) and is documented as accepting any `CLVMStorage`, including `LazyNode`.
- `LazyNode` is the direct output of all four deserializers (`deser_legacy`, `deser_backrefs`, `deser_2026`, `deser_auto`), making it a natural input.
- CPython's per-size free list makes address reuse deterministic and immediate for same-size objects; `LazyNode` instances are uniform in size.
- The trigger requires a tree of depth ≥ 2 with at least two sibling subtrees, which is the common case for any real CLVM program.
- The bug is silent — no error is raised, only the wrong `NodePtr` is silently substituted.

---

### Recommendation

Replace the raw-pointer `usize` key in `identity_map` with a stable, content-derived key, or — more directly — store the actual `Bound<'py, PyAny>` references inside `BuildPair` to keep the Python objects alive until their `NodePtr` is consumed:

```rust
enum WorkItem<'py> {
    Visit(Bound<'py, PyAny>),
    BuildPair {
        id: usize,
        left: Bound<'py, PyAny>,   // keep alive
        right: Bound<'py, PyAny>,  // keep alive
    },
}
```

Then in `BuildPair`, look up by `left.as_ptr() as usize` and `right.as_ptr() as usize` — which are guaranteed valid because the objects are still alive. This mirrors the fix in the original report: track by the stable key (the live object reference) rather than the mutable one (the reusable address).

### Proof of Concept

```python
from clvm_rs.clvm_rs import deser_backrefs, ser_legacy, clvm_tree_to_lazy_node
from clvm_rs.serde import ser_backrefs

# Build a 3-level tree: Root = ((C . D) . (E . F))
# where C, D, E, F are distinct atoms.
# Serialize it the "safe" way (Rust-only path).
import struct

def atom(b: bytes) -> bytes:
    # legacy CLVM atom encoding
    if len(b) == 0:
        return b'\x80'
    if len(b) < 0x40:
        return bytes([0x80 | len(b)]) + b
    raise ValueError

def pair() -> bytes:
    return b'\xff'

blob = (pair() +
        pair() + atom(b"CCCC") + atom(b"DDDD") +
        pair() + atom(b"EEEE") + atom(b"FFFF"))

# Deserialize to LazyNode (creates new Python objects on each .pair call)
lazy = deser_backrefs(blob)

# Convert via clvm_tree_to_lazy_node — may corrupt due to address reuse
lazy2 = clvm_tree_to_lazy_node(lazy)

original = ser_legacy(lazy)
converted = ser_legacy(lazy2)

# On a vulnerable build, converted != original because E's slot was
# reused by C's freed LazyNode wrapper, aliasing E -> NodePtr_C.
assert original == converted, f"CORRUPTED: {original.hex()} != {converted.hex()}"

fn clvm_tree_to_lazy_node(obj: Bound<'_, PyAny>) -> PyResult<LazyNode> {
    let mut allocator = Allocator::new();

    let mut identity_map: HashMap<usize, NodePtr> = HashMap::new();
    let mut atom_map: HashMap<Vec<u8>, NodePtr> = HashMap::new();
    let mut pair_map: HashMap<(NodePtr, NodePtr), NodePtr> = HashMap::new();

    enum WorkItem<'py> {
        Visit(Bound<'py, PyAny>),
        BuildPair {
            id: usize,
            left_id: usize,
            right_id: usize,
        },
    }

    let root_ptr = obj.as_ptr() as usize;
    let mut stack: Vec<WorkItem<'_>> = vec![WorkItem::Visit(obj)];

    while let Some(item) = stack.pop() {
        match item {
            WorkItem::Visit(pyobj) => {
                let id = pyobj.as_ptr() as usize;

                if identity_map.contains_key(&id) {
                    continue;
                }

                let atom_val: Option<Vec<u8>> = pyobj.getattr("atom")?.extract()?;

                if let Some(bytes) = atom_val {
                    let node = if let Some(&existing) = atom_map.get(&bytes) {
                        existing
                    } else {
                        let new_node = allocator
                            .new_atom(&bytes)
                            .map_err(|e| pyo3::exceptions::PyMemoryError::new_err(e.to_string()))?;
                        atom_map.insert(bytes, new_node);
                        new_node
                    };
                    identity_map.insert(id, node);
                } else {
                    let pair_val: Option<(Bound<'_, PyAny>, Bound<'_, PyAny>)> =
                        pyobj.getattr("pair")?.extract()?;

                    if let Some((left, right)) = pair_val {
                        let left_id = left.as_ptr() as usize;
                        let right_id = right.as_ptr() as usize;

                        let left_done = identity_map.contains_key(&left_id);
                        let right_done = identity_map.contains_key(&right_id);

                        if left_done && right_done {
                            let l = identity_map[&left_id];
                            let r = identity_map[&right_id];
                            let node = if let Some(&existing) = pair_map.get(&(l, r)) {
                                existing
                            } else {
                                let new_node = allocator.new_pair(l, r).map_err(|e| {
                                    pyo3::exceptions::PyMemoryError::new_err(e.to_string())
                                })?;
                                pair_map.insert((l, r), new_node);
                                new_node
                            };
                            identity_map.insert(id, node);
                        } else {
                            stack.push(WorkItem::BuildPair {
                                id,
                                left_id,
                                right_id,
                            });
                            if !right_done {
                                stack.push(WorkItem::Visit(right));
                            }
                            if !left_done {
                                stack.push(WorkItem::Visit(left));
                            }
                        }
                    } else {
                        return Err(pyo3::exceptions::PyValueError::new_err(
                            "CLVM object has neither .atom nor .pair",
                        ));
                    }
                }
            }
            WorkItem::BuildPair {
                id,
                left_id,
                right_id,
            } => {
                let l = identity_map[&left_id];
                let r = identity_map[&right_id];
                let node = if let Some(&existing) = pair_map.get(&(l, r)) {
                    existing
                } else {
                    let new_node = allocator
                        .new_pair(l, r)
                        .map_err(|e| pyo3::exceptions::PyMemoryError::new_err(e.to_string()))?;
                    pair_map.insert((l, r), new_node);
                    new_node
                };
                identity_map.insert(id, node);
            }
        }
    }

    let root = identity_map[&root_ptr];
    Ok(LazyNode::new(Rc::new(allocator), root))
}
