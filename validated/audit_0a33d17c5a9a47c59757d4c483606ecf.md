Audit Report

## Title
Per-Bin Additional Fee Has No Cap, Allowing Pool Admin to Exceed Factory Fee Limits - (File: metric-core/contracts/MetricOmmPoolFactory.sol)

## Summary
`setPoolBinAdditionalFees` forwards `addFeeBuyE6` and `addFeeSellE6` directly to the pool with no upper-bound validation, while the analogous `setPoolAdminFees` enforces `maxAdminSpreadFeeE6` and `maxAdminNotionalFeeE8` caps. A pool admin can set per-bin additional fees to `type(uint16).max` (65535 in E6 = 6.5535%) on any bin, bypassing the factory's cap system and extracting excess fees from swappers routing through that bin.

## Finding Description
`setPoolAdminFees` enforces caps before writing:

```solidity
// MetricOmmPoolFactory.sol lines 414-415
if (newAdminSpreadFeeE6 > maxAdminSpreadFeeE6) revert AdminFeeTooHigh();
if (newAdminNotionalFeeE8 > maxAdminNotionalFeeE8) revert AdminFeeTooHigh();
```

`setPoolBinAdditionalFees` has no equivalent check and forwards raw caller-supplied values straight to the pool:

```solidity
// MetricOmmPoolFactory.sol lines 450-457
function setPoolBinAdditionalFees(address pool, int8 bin, uint16 addFeeBuyE6, uint16 addFeeSellE6)
    external override nonReentrant onlyPoolAdmin(pool)
{
    IMetricOmmPoolFactoryActions(pool).setBinAdditionalFees(bin, addFeeBuyE6, addFeeSellE6);
}
```

The pool-side handler performs only a bin-index range check, writing fee values unconditionally:

```solidity
// MetricOmmPool.sol lines 464-474
function setBinAdditionalFees(int8 bin, uint16 addFeeBuyE6, uint16 addFeeSellE6)
    external onlyFactory nonReentrant(PoolActions.SET_BIN_ADDITIONAL_FEES)
{
    if (bin < LOWEST_BIN || bin > HIGHEST_BIN) revert InvalidBinIndex(bin);
    BinState storage s = _binStates[bin];
    s.addFeeBuyE6 = addFeeBuyE6;
    s.addFeeSellE6 = addFeeSellE6;
    emit BinAdditionalFeesUpdated(bin, addFeeBuyE6, addFeeSellE6);
}
```

These stored values are then added directly to `baseFeeX64` in every swap path through the affected bin:

```solidity
// MetricOmmPool.sol line 910
params.baseFeeX64 + Math.mulDiv(binState.addFeeBuyE6, ONE_X64, 1e6)

// MetricOmmPool.sol line 1177
params.baseFeeX64 + Math.mulDiv(binState.addFeeSellE6, ONE_X64, 1e6)
```

There is no guard anywhere in the call chain that limits `addFeeBuyE6` or `addFeeSellE6` to `maxAdminSpreadFeeE6` or any other factory-level cap.

## Impact Explanation
Swappers whose trades route through the targeted bin pay the base spread fee plus the uncapped per-bin additional fee. At `uint16.max` (65535), the additional fee is 6.5535% in E6 units, applied on top of whatever base spread fee is already configured. The excess fee accrues to `adminFeeDestination`, constituting a direct loss of user principal. This is an admin-boundary break: the pool admin exceeds the cap system the factory is designed to enforce, which is an allowed impact per the contest rules ("pool admin exceeds caps").

## Likelihood Explanation
Requires a malicious or compromised pool admin. However, the inconsistency is structural: `setPoolAdminFees` is capped while `setPoolBinAdditionalFees` is not, so the cap system the factory exposes as a safety guarantee does not actually cover all fee-setting paths. The attack is repeatable and requires no special on-chain conditions beyond pool admin access.

## Recommendation
Add a cap check inside `setPoolBinAdditionalFees` before forwarding to the pool, mirroring the pattern used in `setPoolAdminFees`:

```solidity
function setPoolBinAdditionalFees(address pool, int8 bin, uint16 addFeeBuyE6, uint16 addFeeSellE6)
    external override nonReentrant onlyPoolAdmin(pool)
{
    if (addFeeBuyE6  > maxAdminSpreadFeeE6) revert AdminFeeTooHigh();
    if (addFeeSellE6 > maxAdminSpreadFeeE6) revert AdminFeeTooHigh();
    IMetricOmmPoolFactoryActions(pool).setBinAdditionalFees(bin, addFeeBuyE6, addFeeSellE6);
}
```

Alternatively, introduce a dedicated `maxAdminBinAdditionalFeeE6` cap that can be tuned independently.

## Proof of Concept
1. Pool is deployed with `spreadFeeE6 = 10_000` (1%) and `maxAdminSpreadFeeE6 = 200_000` (20%).
2. Pool admin calls `setPoolBinAdditionalFees(pool, 0, 65535, 65535)`.
3. Factory forwards the call; pool writes `addFeeBuyE6 = 65535`, `addFeeSellE6 = 65535` for bin 0 with no revert — confirmed by `MetricOmmPool.sol` lines 469–472 which only check bin index range.
4. A victim's swap routing through bin 0 now pays 1% (base) + 6.5535% (per-bin) = 7.5535% total spread fee, computed at `MetricOmmPool.sol` line 910/1177.
5. The excess 6.5535% flows to `adminFeeDestination` — a direct loss of principal to the swapper that the factory's published cap system was supposed to prevent.
