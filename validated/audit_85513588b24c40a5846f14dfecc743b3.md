### Title
Gateway L2 Gas Price Admission Uses Stale `l2_gas_price` Instead of `next_l2_gas_price` — (File: `crates/apollo_gateway/src/stateful_transaction_validator.rs`)

### Summary

The gateway's `validate_resource_bounds` reads `gas_prices.strk_gas_prices.l2_gas_price` from the previous block's `BlockInfo` to threshold-check incoming transactions. The correct reference price for the *next* block is `BlockHeaderWithoutHash::next_l2_gas_price`, which is computed by the EIP-1559 mechanism and stored separately. Using the stale price causes the gateway to accept transactions that will fail during execution (when the price is rising) and to reject transactions that would succeed (when the price is falling).

### Finding Description

`StatefulTransactionValidator::validate_resource_bounds` in `crates/apollo_gateway/src/stateful_transaction_validator.rs` reads the L2 gas price from `BlockInfo`:

```rust
// TODO(Arni): getnext_l2_gas_price from the block header.
let previous_block_l2_gas_price = self
    .gateway_fixed_block_state_reader
    .get_block_info()
    .await?
    .gas_prices
    .strk_gas_prices
    .l2_gas_price;
``` [1](#0-0) 

`BlockInfo::gas_prices.l2_gas_price` is the price *for the already-committed block*. The price that will govern execution of the incoming transaction is `BlockHeaderWithoutHash::next_l2_gas_price`, which is computed by the EIP-1559 formula in `calculate_next_base_gas_price` and stored as a distinct field in the block header. [2](#0-1) 

The two values diverge whenever the block is not at the gas target. The `gas_price_max_change_denominator` in `apollo_versioned_constants` bounds the per-block rate of change, but the divergence is real and non-zero on every congested or under-utilized block. [3](#0-2) 

The threshold check that uses the stale price:

```rust
fn validate_tx_l2_gas_price_within_threshold(
    &self,
    tx_resource_bounds: ValidResourceBounds,
    previous_block_l2_gas_price: NonzeroGasPrice,
) -> StatefulTransactionValidatorResult<()> {
    ...
    let threshold = (gas_price_threshold_multiplier
        * previous_block_l2_gas_price.get().0)
        .to_integer();
    if tx_l2_gas_price.0 < threshold {
        return Err(...);
    }
``` [4](#0-3) 

The `next_l2_gas_price` is available in `StorageBlockHeader` and is propagated through `BlockHeaderWithoutHash`: [5](#0-4) 

The gateway never reads this field for admission control.

### Impact Explanation

**High. Mempool/gateway/RPC admission accepts invalid transactions or rejects valid transactions before sequencing.**

- **Price rising** (`next_l2_gas_price > l2_gas_price`): The gateway uses the lower stale price as the threshold. A transaction with `max_price_per_unit` in the range `[threshold_stale, threshold_correct)` passes gateway admission but will fail during blockifier execution with `MaxGasPriceTooLow`, wasting mempool slots and batcher resources.

- **Price falling** (`next_l2_gas_price < l2_gas_price`): The gateway uses the higher stale price as the threshold. A transaction with `max_price_per_unit` in the range `[threshold_correct, threshold_stale)` is rejected by the gateway even though it would succeed during execution. This is a valid transaction denial.

### Likelihood Explanation

The EIP-1559 mechanism adjusts the L2 gas price every block based on gas consumption vs. the target. Any block that is not exactly at the gas target produces a non-zero divergence between `l2_gas_price` and `next_l2_gas_price`. Under normal network load this divergence is continuous and persistent. The `validate_resource_bounds` flag defaults to `true` in `StatefulTransactionValidatorConfig`. [6](#0-5) 

### Recommendation

Replace the `get_block_info()` call with a read of `next_l2_gas_price` from the latest committed block header (`BlockHeaderWithoutHash::next_l2_gas_price`). The `gateway_fixed_block_state_reader` should expose a method that returns the block header (or at minimum `next_l2_gas_price`) so the admission check uses the price that will actually govern execution of the incoming transaction. The existing TODO comment in the code already identifies this gap.

### Proof of Concept

1. Network is congested; the previous committed block consumed more gas than the target.
2. EIP-1559 computes `next_l2_gas_price = 110` (stored in `BlockHeaderWithoutHash`).
3. The same block's `l2_gas_price = 100` (stored in `BlockInfo::gas_prices`).
4. `StatefulTransactionValidatorConfig::min_gas_price_percentage = 100` (default).
5. User submits an `InvokeV3` transaction with `l2_gas.max_price_per_unit = 105`.
6. Gateway calls `validate_resource_bounds`:
   - reads `previous_block_l2_gas_price = 100` (stale)
   - threshold = `100 * 100% = 100`
   - `105 >= 100` → **PASS**, transaction admitted to mempool
7. Batcher pulls the transaction and executes it in the next block where the actual price is `110`.
8. Blockifier pre-validation: `105 < 110` → `MaxGasPriceTooLow` → transaction **fails during execution**.

The gateway admitted a transaction that the blockifier rejects, consuming mempool capacity and batcher execution budget. The inverse (valid transaction rejected) occurs symmetrically when the price is falling.

    // TODO(Arni): Consider running this validation for all gas prices.
    fn validate_tx_l2_gas_price_within_threshold(
        &self,
        tx_resource_bounds: ValidResourceBounds,
        previous_block_l2_gas_price: NonzeroGasPrice,
    ) -> StatefulTransactionValidatorResult<()> {
        match tx_resource_bounds {
            ValidResourceBounds::AllResources(tx_resource_bounds) => {
                let tx_l2_gas_price = tx_resource_bounds.l2_gas.max_price_per_unit;
                let gas_price_threshold_multiplier =
                    Ratio::new(self.config.min_gas_price_percentage.into(), 100_u128);
                let threshold = (gas_price_threshold_multiplier
                    * previous_block_l2_gas_price.get().0)
                    .to_integer();
                if tx_l2_gas_price.0 < threshold {
                    return Err(StarknetError {
                        // We didn't have this kind of an error.
                        code: StarknetErrorCode::UnknownErrorCode(
                            "StarknetErrorCode.GAS_PRICE_TOO_LOW".to_string(),
                        ),
                        message: format!(
                            "Transaction L2 gas price {tx_l2_gas_price} is below the required \
                             threshold {threshold}.",
                        ),
                    });
                }
            }
            ValidResourceBounds::L1Gas(_) => {
                // No validation required for legacy transactions.
            }
        }
        Ok(())
    }

/// Compute the next L2 gas price (for the fin or for updating state). Respects override when set.
pub fn calculate_next_l2_gas_price_for_fin(
    current_l2_gas_price: GasPrice,
    height: BlockNumber,
    l2_gas_used: GasAmount,
    override_l2_gas_price_fri: Option<u128>,
    min_l2_gas_price_per_height: &[PricePerHeight],
    fee_actual: Option<GasPrice>,
) -> GasPrice {
    if let Some(override_value) = override_l2_gas_price_fri {
        info!(
            "L2 gas price ({}) is not updated, remains on override value of {override_value} fri",
            current_l2_gas_price.0
        );
        return GasPrice(override_value);
    }
    let gas_target = VersionedConstants::latest_constants().gas_target;
    let config_min = get_min_gas_price_for_height(height, min_l2_gas_price_per_height);
    let effective_min = match fee_actual {
        Some(fa) => GasPrice(max(config_min.0, fa.0)),
        None => config_min,
    };
    calculate_next_base_gas_price(current_l2_gas_price, l2_gas_used, gas_target, effective_min)
}
