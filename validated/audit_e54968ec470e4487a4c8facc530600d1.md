### Title
SwapAllowlistExtension gates the router address instead of the actual user, allowing any user to bypass per-user swap restrictions via MetricOmmSimpleRouter — (`metric-periphery/contracts/extensions/SwapAllowlistExtension.sol`)

---

### Summary

`SwapAllowlistExtension.beforeSwap` checks the `sender` argument, which is `msg.sender` of the pool's `swap()` call. When users route through `MetricOmmSimpleRouter`, the pool receives `sender = router address`. If the pool admin allowlists the router to enable router-mediated swaps, every user — including those not individually allowlisted — can bypass the per-user gate by routing through the router.

---

### Finding Description

`SwapAllowlistExtension.beforeSwap` enforces:

```solidity
if (!allowAllSwappers[msg.sender] && !allowedSwapper[msg.sender][sender]) {
    revert IMetricOmmPoolActions.NotAllowedToSwap();
}
``` [1](#0-0) 

Here `msg.sender` is the pool (correct), and `sender` is the value forwarded by `MetricOmmPool._beforeSwap`, which passes `msg.sender` of the pool's `swap()` call:

```solidity
_beforeSwap(
    msg.sender,   // ← router address when called via router
    recipient,
    ...
);
``` [2](#0-1) 

When a user calls `MetricOmmSimpleRouter.exactInputSingle()`, the router calls `pool.swap()` directly:

```solidity
(int128 amount0Delta, int128 amount1Delta) = IMetricOmmPoolActions(params.pool)
    .swap(
        params.recipient,
        params.zeroForOne,
        MetricOmmSwapInputs.asAmountSpecifiedIn(params.amountIn),
        priceLimitX64,
        "",
        params.extensionData
    );
``` [3](#0-2) 

The pool therefore passes `sender = router address` to the extension. The extension has no visibility into which end-user initiated the call. The same identity collapse occurs in `exactInput`, `exactOutputSingle`, and `exactOutput`. [4](#0-3) 

A pool admin who wants to enable router-mediated swaps for any subset of users must allowlist the router address:

```solidity
swapExtension.setAllowedToSwap(pool, address(router), true);
``` [5](#0-4) 

Once the router is allowlisted, the extension cannot distinguish between different end-users passing through it. Every user — including those explicitly not in `allowedSwapper[pool]` — can swap freely by routing through `MetricOmmSimpleRouter`. The per-user allowlist is completely neutralized.

The `DepositAllowlistExtension` does not share this flaw because it gates on `owner` (the position owner, the economically relevant identity), not on `sender` (the immediate caller):

```solidity
function beforeAddLiquidity(address, address owner, uint80, LiquidityDelta calldata, bytes calldata)
    ...
{
    if (!allowAllDepositors[msg.sender] && !allowedDepositor[msg.sender][owner]) {
``` [6](#0-5) 

The swap extension has no equivalent "economically relevant identity" parameter to fall back on.

---

### Impact Explanation

A pool configured with `SwapAllowlistExtension` to restrict trading to a curated set of counterparties (e.g., a private institutional pool) is fully open to any user once the router is allowlisted. Unauthorized users can trade against the pool's LPs, extracting value through adverse selection or simply consuming liquidity the LPs did not intend to provide to the public. This is a direct loss of LP principal and a broken core pool functionality.

---

### Likelihood Explanation

The trigger is a pool admin allowlisting the router — a natural and expected configuration step for any pool that wants to support the standard periphery UX. The admin has no reason to suspect this opens the gate to all users; the allowlist UI (`setAllowedToSwap`) gives no indication that allowlisting a router-type contract is semantically different from allowlisting an EOA. The bypass requires no privileged access, no special tokens, and no flash loan; any user with the router address can exploit it in a single transaction.

---

### Recommendation

Gate on the actual end-user identity rather than the immediate caller. Two concrete options:

1. **Pass the real user through `extensionData`**: Have the router encode `msg.sender` into `extensionData` before forwarding to the pool, and have the extension decode and verify it. This requires a coordinated extension/router convention.

2. **Check `recipient` as a proxy for the user**: For exact-input swaps the recipient is typically the user. This is imperfect (recipient can be a third party) but eliminates the most common bypass path.

3. **Document that the router must never be allowlisted**: Add an explicit guard in `setAllowedToSwap` that reverts if the target address is a registered periphery contract, or emit a warning event.

The cleanest fix is option 1: the router encodes `abi.encode(msg.sender)` as the first word of `extensionData`, and the extension reads it as the authoritative swapper identity.

---

### Proof of Concept

```solidity
// Setup: pool with SwapAllowlistExtension, router allowlisted, Eve not allowlisted
swapExtension.setAllowedToSwap(address(pool), address(router), true);
// Eve is NOT in allowedSwapper[pool]

// Eve bypasses the allowlist via the router
vm.prank(eve);
router.exactInputSingle(
    IMetricOmmSimpleRouter.ExactInputSingleParams({
        pool:            address(pool),
        tokenIn:         address(token1),
        tokenOut:        address(token0),
        zeroForOne:      false,
        amountIn:        1_000,
        amountOutMinimum: 0,
        recipient:       eve,
        deadline:        block.timestamp + 1,
        priceLimitX64:   type(uint128).max,
        extensionData:   ""
    })
);
// Succeeds: pool.swap() sees sender=router, router is allowlisted, Eve trades freely.

// Direct call from Eve reverts as expected
vm.prank(eve);
vm.expectRevert(IMetricOmmPoolActions.NotAllowedToSwap.selector);
pool.swap(eve, false, 1_000, type(uint128).max, "", "");
```

The asymmetry — direct call reverts, router call succeeds — confirms the allowlist is checking the wrong identity for router-mediated swaps.

      address pool = params.pools[i];
      bool zeroForOne = MetricOmmSwapPath.resolveZeroForOneBitmap(params.zeroForOneBitMap, i);

      _setNextCallbackContext(pool, CALLBACK_MODE_JUST_PAY, i == 0 ? msg.sender : address(this), params.tokens[i]);
      (int128 amount0Delta, int128 amount1Delta) = IMetricOmmPoolActions(pool)
        .swap(
          i == last ? params.recipient : address(this),
          zeroForOne,

          MetricOmmSwapPath.openLimit(zeroForOne),
          "",
          params.extensionDatas[i]

      int128 amountInActual = MetricOmmSwapResults.extractAmountIn(zeroForOne, amount0Delta, amount1Delta);
