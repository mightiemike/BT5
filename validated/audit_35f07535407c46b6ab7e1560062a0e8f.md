### Title
Unprivileged Caller Can Aggressively Prune Mainchain Block Headers via `run_mainchain_gc` - (File: contract/src/lib.rs)

### Summary
`run_mainchain_gc` is a public state-mutating function that permanently deletes mainchain block headers from contract storage. It carries only a `#[pause]` decorator, which restricts calls only when the contract is paused. When the contract is running normally, any unprivileged NEAR account can call it with an arbitrarily large `batch_size`, permanently destroying historical header records beyond what the normal relayer flow would remove.

### Finding Description
The function at line 377 of `contract/src/lib.rs` is declared as:

```rust
#[pause(except(roles(Role::UnrestrictedRunGC)))]
pub fn run_mainchain_gc(&mut self, batch_size: u64) {
```

The `#[pause(except(roles(...)))]` macro from `near-plugins` only gates execution when the contract is in a paused state — accounts holding `UnrestrictedRunGC` can bypass the pause. When the contract is **not** paused (its normal operating state), the decorator imposes **no caller restriction at all**. The function is fully open to any NEAR account.

The caller supplies `batch_size: u64` directly. The function computes:

```rust
let total_amount_to_remove = amount_of_headers_we_store - self.gc_threshold;
let selected_amount_to_remove = std::cmp::min(total_amount_to_remove, batch_size);
```

With `batch_size = u64::MAX`, the attacker removes every mainchain header above `gc_threshold` in a single call. Each removed header is permanently deleted from `headers_pool`, `mainchain_height_to_header`, and `mainchain_header_to_height`, and `mainchain_initial_blockhash` is advanced to the new oldest block.

Contrast this with `submit_blocks`, which carries `#[trusted_relayer]` and is correctly restricted to registered relayers or accounts holding `Role::DAO` / `Role::UnrestrictedSubmitBlocks`. No equivalent caller guard exists on `run_mainchain_gc`. [1](#0-0) [2](#0-1) [3](#0-2) 

### Impact Explanation
Permanent deletion of mainchain block headers produces two concrete, irreversible harms:

1. **SPV proof forgery / denial**: `verify_transaction_inclusion` and `verify_transaction_inclusion_v2` look up the target block in `mainchain_header_to_height`. Once a block is GC'd by the attacker, any SPV proof for a transaction in that block will panic with `"block does not belong to the current main chain"`. Bridge contracts or downstream consumers that rely on the light client for settlement will permanently lose the ability to verify historical Bitcoin transactions.

2. **Chain reorganization breakage**: `reorg_chain` walks the fork chain backward until it finds a block present in `mainchain_header_to_height`. If the attacker has pruned the common ancestor, the walk panics with `"previous fork block should be there"`. This permanently disables the contract's ability to process any reorg whose common ancestor falls below the artificially advanced `mainchain_initial_blockhash`. [4](#0-3) [5](#0-4) [6](#0-5) 

### Likelihood Explanation
The attack requires no privileged key, no staked deposit, no special role, and no knowledge beyond the public ABI. Any NEAR account can call `run_mainchain_gc({"batch_size": 18446744073709551615})` in a single transaction. The only cost is NEAR gas. The contract's own documentation confirms the function is intentionally externally callable ("Public call to run GC on a mainchain"), making the missing guard easy to overlook and the attack surface permanently exposed. [7](#0-6) [8](#0-7) 

### Recommendation
Apply a caller-identity guard equivalent to the one on `submit_blocks`. The simplest correct fix is to add an `access_control_any` check requiring at least one of the existing privileged roles (e.g., `Role::DAO` or a new dedicated `GCManager` role) before any state mutation occurs:

```rust
#[pause(except(roles(Role::UnrestrictedRunGC)))]
pub fn run_mainchain_gc(&mut self, batch_size: u64) {
    require!(
        self.acl_has_role_or_greater(
            &[Role::DAO.into(), Role::UnrestrictedRunGC.into()],
            &env::predecessor_account_id()
        ),
        "Unauthorized: caller lacks GC permission"
    );
    // ... existing body
}
```

Alternatively, introduce a dedicated `GCManager` role mirroring the pattern already used for `PauseManager`, `RelayerManager`, etc., and gate the function on that role. [9](#0-8) 

### Proof of Concept
1. Deploy the contract with `gc_threshold = 52704` and a live mainchain of 60,000 headers.
2. From any fresh NEAR account (no roles, no stake), call:
   ```
   run_mainchain_gc({"batch_size": 18446744073709551615})
   ```
3. The contract computes `total_amount_to_remove = 60000 - 52704 = 7296` and deletes all 7,296 oldest mainchain headers in one transaction.
4. Immediately after, call `verify_transaction_inclusion` with a valid SPV proof for any transaction in one of the deleted blocks. The call panics: `"block does not belong to the current main chain"`.
5. Submit a fork whose common ancestor is one of the deleted blocks. The reorg walk panics: `"previous fork block should be there"`.

Both effects are permanent — deleted NEAR storage entries cannot be recovered without a full contract migration and re-submission of all pruned headers from an external source. [10](#0-9) [11](#0-10)

            (heaviest_block_header.block_height).saturating_sub(target_block_height) + 1

            "Not enough blocks confirmed"
        );

            .headers_pool
            .get(&args.tx_block_blockhash)

    /// Public call to run GC on a mainchain.
    /// `batch_size` is how many block headers should be removed in the execution
    ///
    /// # Panics
    /// If initial blockheader or tip blockheader are not in a header pool
    #[pause(except(roles(Role::UnrestrictedRunGC)))]
    pub fn run_mainchain_gc(&mut self, batch_size: u64) {
        let initial_blockheader = self
            .headers_pool
            .get(&self.mainchain_initial_blockhash)
            .unwrap_or_else(|| env::panic_str(ERR_KEY_NOT_EXIST));

        let tip_blockheader = self
            .headers_pool
            .get(&self.mainchain_tip_blockhash)
            .unwrap_or_else(|| env::panic_str(ERR_KEY_NOT_EXIST));

        let amount_of_headers_we_store =
            tip_blockheader.block_height - initial_blockheader.block_height + 1;

        if amount_of_headers_we_store > self.gc_threshold {
            let total_amount_to_remove = amount_of_headers_we_store - self.gc_threshold;
            let selected_amount_to_remove = std::cmp::min(total_amount_to_remove, batch_size);

            let start_removal_height = initial_blockheader.block_height;
            let end_removal_height = initial_blockheader.block_height + selected_amount_to_remove;
            env::log_str(&format!(
                "Num of blocks to remove {selected_amount_to_remove}"
            ));

            for height in start_removal_height..end_removal_height {
                let blockhash = &self
                    .mainchain_height_to_header
                    .get(&height)
                    .unwrap_or_else(|| env::panic_str(ERR_KEY_NOT_EXIST));

                self.remove_block_header(blockhash);
                self.mainchain_height_to_header.remove(&height);
            }

            self.mainchain_initial_blockhash = self
                .mainchain_height_to_header
                .get(&end_removal_height)
                .unwrap_or_else(|| env::panic_str(ERR_KEY_NOT_EXIST));
        }
    }
