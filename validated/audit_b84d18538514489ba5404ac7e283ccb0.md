### Title
`ValidResourceBounds` Type Misclassification in Protobuf Sync Deserialization Produces Divergent Transaction Hash - (File: `crates/apollo_protobuf/src/converters/transaction.rs`)

### Summary

The protobuf deserialization path used during state sync applies a value-based heuristic to classify `ValidResourceBounds` as either `L1Gas` (pre-0.13.3) or `AllResources` (post-0.13.3). When a V3 invoke transaction carries `AllResourceBounds` with zero `l2_gas` and `l1_data_gas`, the sync deserializer silently downgrades it to `ValidResourceBounds::L1Gas`. Because `get_tip_resource_bounds_hash` hashes a different number of resource felts for each variant (2 for `L1Gas`, 3 for `AllResources`), the transaction hash computed during sync diverges from the hash computed and stored by the gateway, breaking the canonicalization invariant.

### Impact Explanation

A syncing node that receives a block containing such a transaction recomputes H‚ÇÇ ‚âÝ H‚ÇÅ (the stored block hash). This causes the syncing node to reject the block as having an invalid transaction hash, breaking state synchronization. The divergence is silent ‚Äî no error is raised at deserialization time; the wrong variant is silently accepted and only manifests when the hash is later verified. This matches: **Wrong state or revert result from blockifier/syscall/execution logic for accepted input** (Critical/High).

### Likelihood Explanation

The trigger requires a V3 invoke transaction with `l2_gas = {0,0}` and `l1_data_gas = {0,0}`. The stateless gateway validator rejects this if `min_gas_price > 0` (because `l2_gas.max_price_per_unit = 0 < min_gas_price`). [6](#0-5) 

However: (a) `min_gas_price = 0` is a valid configuration (default in testing); (b) a gateway with `min_gas_price = 0` would accept and include such a transaction in a block; (c) any peer syncing that block would then fail hash validation. The vulnerability is configuration-gated but the canonicalization invariant is unconditionally broken.

### Recommendation

Replace the value-based heuristic in `TryFrom<protobuf::ResourceBounds> for ValidResourceBounds` (sync path, `transaction.rs`) with an explicit version/protocol-level discriminator. Once support for 0.13.2 is dropped (as the existing TODO notes), remove the `unwrap_or_default()` fallback and always deserialize as `AllResources`. Until then, add a protocol-version field to the protobuf message so the deserializer can select the correct variant without inspecting field values. The mempool P2P path (`rpc_transaction.rs`) already does this correctly by mapping directly to `AllResourceBounds`.

### Proof of Concept

1. Submit a V3 invoke transaction with `resource_bounds = { l1_gas: {max_amount: 1000, max_price_per_unit: 1}, l2_gas: {0,0}, l1_data_gas: {0,0} }` to a gateway with `min_gas_price = 0`.
2. Gateway accepts it; `convert_rpc_tx_to_internal` computes H‚ÇÅ using `AllResources` (3 resource felts in fee hash).
3. Transaction is included in block B with hash H‚ÇÅ.
4. A syncing peer receives block B via the sync protobuf path. `TryFrom<protobuf::ResourceBounds> for ValidResourceBounds` sees `l1_data_gas.is_zero() && l2_gas.is_zero()` ‚Üí returns `L1Gas`.
5. Syncing peer calls `validate_transaction_hash` ‚Üí `get_invoke_transaction_v3_hash` ‚Üí `get_tip_resource_bounds_hash` with `L1Gas` ‚Üí computes H‚ÇÇ (2 resource felts in fee hash).
6. H‚ÇÅ ‚âÝ H‚ÇÇ ‚Üí syncing peer rejects block B ‚Üí sync stalls.

impl TryFrom<protobuf::ResourceBounds> for ValidResourceBounds {
    type Error = ProtobufConversionError;
    fn try_from(value: protobuf::ResourceBounds) -> Result<Self, Self::Error> {
        let Some(l1_gas) = value.l1_gas else {
            return Err(missing("ResourceBounds::l1_gas"));
        };
        let Some(l2_gas) = value.l2_gas else {
            return Err(missing("ResourceBounds::l2_gas"));
        };
        // TODO(Shahak): Assert data gas is not none once we remove support for 0.13.2.
        let l1_data_gas = value.l1_data_gas.unwrap_or_default();
        let l1_gas: ResourceBounds = l1_gas.try_into()?;
        let l2_gas: ResourceBounds = l2_gas.try_into()?;
        let l1_data_gas: ResourceBounds = l1_data_gas.try_into()?;
        Ok(if l1_data_gas.is_zero() && l2_gas.is_zero() {
            ValidResourceBounds::L1Gas(l1_gas)
        } else {
            ValidResourceBounds::AllResources(AllResourceBounds { l1_gas, l2_gas, l1_data_gas })
        })
    }

pub fn get_tip_resource_bounds_hash(
    resource_bounds: &ValidResourceBounds,
    tip: &Tip,
) -> Result<Felt, StarknetApiError> {
    let l1_resource_bounds = resource_bounds.get_l1_bounds();
    let l2_resource_bounds = resource_bounds.get_l2_bounds();

    // L1 and L2 gas bounds always exist.
    // Old V3 txs always have L2 gas bounds of zero, but they exist.
    let mut resource_felts = vec![
        get_concat_resource(&l1_resource_bounds, L1_GAS)?,
        get_concat_resource(&l2_resource_bounds, L2_GAS)?,
    ];

    // For new V3 txs, need to also hash the data gas bounds.
    resource_felts.extend(match resource_bounds {
        ValidResourceBounds::L1Gas(_) => vec![],
        ValidResourceBounds::AllResources(all_resources) => {
            vec![get_concat_resource(&all_resources.l1_data_gas, L1_DATA_GAS)?]
        }
    });

    Ok(HashChain::new().chain(&tip.0.into()).chain_iter(resource_felts.iter()).get_poseidon_hash())
}
