### Title
Hardcoded `LastKnownBonded = true` in `TierRedelegate` Causes Bonus Overpayment for Unbonded Validator Gap - (`File: x/tieredrewards/keeper/msg_server.go`)

### Summary
`MsgTierRedelegate` unconditionally sets `LastKnownBonded = true` on the position after redelegation, regardless of the destination validator's actual bonded state. When a user redelegates to a currently-unbonded validator, the event-replay loop in `processEventsAndClaimBonus` incorrectly treats the post-redelegate gap as a bonded period, paying bonus rewards for time the validator was not bonded and draining the rewards pool.

### Finding Description

In `TierRedelegate`, after settling rewards and executing the staking redelegate, the position's bonus checkpoints are reset: [1](#0-0) 

```go
latestSeq, err := ms.getValidatorEventLatestSeq(ctx, dstValAddr)
...
pos.UpdateBonusCheckpoints(latestSeq, sdkCtx.BlockTime(), true)  // ← hardcoded true
```

The third argument (`true`) hardcodes `LastKnownBonded = true` for the destination validator without checking its actual bonded state.

In `processEventsAndClaimBonus`, the event-replay loop initialises `bonded` from `pos.LastKnownBonded`: [2](#0-1) 

```go
bonded := pos.LastKnownBonded          // ← incorrectly true
segmentStart := pos.LastBonusAccrual

for _, entry := range events {
    evt := entry.Event
    if bonded {
        bonus := k.computeSegmentBonus(*pos, tier, segmentStart, evt.Timestamp, evt.TokensPerShare)
        totalBonus = totalBonus.Add(bonus)
    }
    ...
}
```

If the destination validator was unbonded at redelegate time and later bonds, the first event fetched is a `BOND` event. Because `bonded = true`, the loop computes a bonus segment `[redelegateTime, bondEventTime]` — the entire unbonded gap — using the BOND event's `TokensPerShare`. No bonus should be paid for this period.

The `computeSegmentBonus` function has no guard against unbonded periods; it relies entirely on the `bonded` flag being correct: [3](#0-2) 

### Impact Explanation

The attacker receives bonus rewards from the `RewardsPoolName` module account for a period when their validator was unbonded. The overpayment is:

```
shares × tokensPerShare_at_bond × bonusApy × unbondedDuration / SecondsPerYear
```

For a large position and a long unbonded gap, this directly drains the rewards pool. The corrupted value is the `RewardsPoolName` module account balance and the position's `LastKnownBonded` / `LastBonusAccrual` checkpoints.

### Likelihood Explanation

Validators are routinely jailed and enter the unbonding state. Cosmos SDK's staking module permits redelegation to unbonded validators. Any position owner can call `MsgTierRedelegate` targeting a jailed/unbonded validator, wait for it to unjail, then claim rewards. No privileged access is required.

### Recommendation

Replace the hardcoded `true` with a live check of the destination validator's bonded state:

```go
dstVal, err := ms.stakingKeeper.GetValidator(ctx, dstValAddr)
if err != nil {
    return nil, err
}
pos.UpdateBonusCheckpoints(latestSeq, sdkCtx.BlockTime(), dstVal.IsBonded())
```

### Proof of Concept

1. User holds a tier position on bonded validator V1.
2. Validator V2 is jailed (unbonded). Its latest event seq is N (includes the UNBOND event).
3. User calls `MsgTierRedelegate` → V2. Code sets `pos.LastKnownBonded = true`, `pos.LastEventSeq = N`, `pos.LastBonusAccrual = T_redelegate`.
4. 30 days pass. V2 unjails; a `BOND` event is appended at seq N+1 with timestamp `T_bond`.
5. User calls `MsgClaimTierRewards`. In `processEventsAndClaimBonus`:
   - `bonded = true` (incorrect)
   - Events fetched: `[BOND event at seq N+1]`
   - Loop: `bonded = true` → `computeSegmentBonus(pos, tier, T_redelegate, T_bond, tokensPerShare_at_bond)` → 30-day bonus paid
   - Final segment: `bonded = true && val.IsBonded() = true` → additional bonus paid
6. User receives bonus for the 30-day unbonded gap, draining the rewards pool. [4](#0-3) [5](#0-4) [3](#0-2)

func (k Keeper) computeSegmentBonus(pos types.PositionState, tier types.Tier, segmentStart, segmentEnd time.Time, tokensPerShare math.LegacyDec) math.Int {
	if !pos.ExitUnlockAt.IsZero() && segmentEnd.After(pos.ExitUnlockAt) {
		segmentEnd = pos.ExitUnlockAt
	}

	if !segmentEnd.After(segmentStart) {
		return math.ZeroInt()
	}

	durationSeconds := int64(segmentEnd.Sub(segmentStart) / time.Second)
	if durationSeconds <= 0 {
		return math.ZeroInt()
	}

	tokens := pos.Delegation.Shares.Mul(tokensPerShare)

	return tokens.
		Mul(tier.BonusApy).
		MulInt64(durationSeconds).
		QuoInt64(types.SecondsPerYear).
		TruncateInt()
}
