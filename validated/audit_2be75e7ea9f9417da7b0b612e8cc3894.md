### Title
AuxPoW Parent Block PoW Verified with Scrypt Instead of SHA256d, Enabling Forged Merged-Mining Proofs - (File: `contract/src/dogecoin.rs`)

### Summary

In the Dogecoin build of the light client, `check_aux` verifies the parent Bitcoin block's proof-of-work by calling `aux_data.parent_block.block_hash_pow()`. Because the Dogecoin binary is compiled with the `scrypt_hash` feature, `block_hash_pow()` always computes a **scrypt** hash — even when the receiver is a Bitcoin parent block, which must be verified with **SHA256d**. This is a direct interface-mismatch analog to the Curve pool bug: a single function is called uniformly across two structurally different block types, producing a silently wrong result that corrupts a security-critical check.

### Finding Description

`block_hash_pow()` is defined in `btc_header.rs` with compile-time feature gating: [1](#0-0) 

When the `scrypt_hash` feature is active (as it must be for the Dogecoin binary, which uses Scrypt for its own PoW), the function unconditionally returns `scrypt(header_bytes)` for **any** `Header` value it is called on.

In `check_aux`, the final PoW check calls this function on `aux_data.parent_block`, which is a **Bitcoin** block header: [2](#0-1) 

`aux_data.parent_block` is typed as `Header` (the shared BTC/Dogecoin header struct): [3](#0-2) 

Bitcoin's proof-of-work is SHA256d, not Scrypt. Calling `block_hash_pow()` on a Bitcoin block under the `scrypt_hash` build returns the Scrypt hash of the Bitcoin header bytes — a value that has no relationship to Bitcoin's actual PoW. The contract then compares this wrong hash against the Dogecoin target: [2](#0-1) 

### Impact Explanation

An attacker can forge a complete AuxPoW proof without performing any real Bitcoin merged mining:

1. Craft a Dogecoin block header with the AuxPoW flag set (`version & 0x100 != 0`).
2. Construct a fake "parent block" (a 80-byte header) whose coinbase transaction embeds the Dogecoin block's chain-merkle-root in the format expected by `check_aux` (lines 95–128 of `dogecoin.rs`).
3. **Mine the fake parent block with Scrypt** until `scrypt(fake_parent_bytes) ≤ dogecoin_target`. Because the Dogecoin target is far easier than Bitcoin's SHA256d target, this requires only Dogecoin-level work — orders of magnitude less than real Bitcoin merged mining.
4. Submit the Dogecoin block with this fake `AuxData` via `submit_blocks`.

All other checks in `check_aux` (chain-ID, coinbase merkle proof, merged-mining header format, n_size, index) are structural and can be satisfied by the attacker when constructing the fake parent block. The only cryptographic barrier — the PoW check — is evaluated against the wrong hash function, making it bypassable at Dogecoin cost.

The result is that the attacker can insert arbitrary Dogecoin block headers into the light client's canonical chain (`headers_pool`, `mainchain_tip_blockhash`), corrupting the chain state and enabling false `verify_transaction_inclusion` results for any transaction the attacker chooses.

### Likelihood Explanation

The entry point is the public, unprivileged `submit_blocks` call. No privileged role, leaked key, or social engineering is required. The attacker only needs Dogecoin-level Scrypt mining capacity, which is available to any pool-scale actor. The bug is triggered on every Dogecoin deployment whenever `skip_pow_verification = false` (the recommended production setting).

### Recommendation

The parent block's PoW must be verified with SHA256d regardless of which chain binary is compiled. The fix is to compute the parent block's hash using double-SHA256 explicitly in `check_aux`, rather than delegating to the feature-gated `block_hash_pow()`:

```rust
// In check_aux, replace:
let pow_hash = aux_data.parent_block.block_hash_pow();

// With an explicit SHA256d call:
let pow_hash = btc_types::hash::double_sha256(
    &aux_data.parent_block.get_block_header_vec()
);
```

Alternatively, introduce a separate `block_hash_pow_sha256d()` method on `Header` that always uses SHA256d, and call it specifically for the AuxPoW parent block.

### Proof of Concept

**Attacker-controlled entry path:**

```
NEAR caller
  → submit_blocks([dogecoin_block_with_auxpow_flag], aux_data=Some(fake_aux))
    → submit_block_header((header, Some(fake_aux)), skip_pow=false)   [dogecoin.rs:166]
      → check_target(...)                                              [passes: bits check only]
      → check_aux(&block_header, &fake_aux)                           [dogecoin.rs:180]
          → pow_hash = fake_aux.parent_block.block_hash_pow()         [dogecoin.rs:149]
            // compiled with scrypt_hash → returns scrypt(fake_parent_bytes)
            // attacker mined fake_parent_bytes so scrypt hash ≤ dogecoin_target
          → require!(scrypt(fake_parent) ≤ target)                    [PASSES]
      → submit_block_header_inner(...)                                 [chain state corrupted]
```

**Concrete construction:**

- `fake_aux.parent_block`: 80-byte header with `version=1`, arbitrary `prev_block_hash`, `merkle_root` = hash of a crafted coinbase tx, `time`, `bits` = Dogecoin target bits, `nonce` iterated until `scrypt(header) ≤ target_from_bits(dogecoin_bits)`.
- `fake_aux.coinbase_tx`: a minimal coinbase transaction whose `script_sig` contains the merged-mining header `fabe6d6d` immediately followed by the chain merkle root of the Dogecoin block hash.
- `fake_aux.merkle_proof`: `[]` (coinbase is the only tx, so merkle root = coinbase txid).
- `fake_aux.chain_merkle_proof`: `[]` (chain_id=0 in a single-element tree).
- `fake_aux.chain_id`: `0`.

All structural checks in `check_aux` pass. The only check that should reject this — the PoW check — passes because the contract evaluates `scrypt(fake_parent)` instead of `sha256d(fake_parent)`. [4](#0-3) [1](#0-0)

    pub fn block_hash_pow(&self) -> H256 {
        let block_header = self.get_block_header_vec();
        #[cfg(feature = "scrypt_hash")]
        {
            let params = scrypt::Params::new(10, 1, 1, 32).unwrap(); // N=1024 (2^10), r=1, p=1

            let mut output = [0u8; 32];
            scrypt::scrypt(&block_header, &block_header, &params, &mut output).unwrap();
            H256::from(output)
        }

        #[cfg(not(feature = "scrypt_hash"))]
        {
            double_sha256(&block_header)
        }
    }

    pub(crate) fn check_aux(&mut self, block_header: &Header, aux_data: &AuxData) {
        // The Dogecoin block must have the AuxPoW flag set (bit 8) when AuxPoW data is present.
        // https://github.com/dogecoin/dogecoin/blob/master/src/auxpow.h
        const BLOCK_VERSION_AUXPOW: i32 = 0x100;

        require!(
            aux_data.chain_merkle_proof.len() <= 30,
            "Aux POW chain merkle branch too long"
        );
        require!(
            block_header.version & BLOCK_VERSION_AUXPOW != 0,
            "Aux POW block does not have AuxPoW flag set in version"
        );

        let chain_id = self.get_config().aux_chain_id;

        require!(
            chain_id == block_header.get_chain_id(),
            format!(
                "block does not have our chain ID (got {}, expected {chain_id})",
                block_header.get_chain_id()
            )
        );

        require!(
            chain_id != aux_data.parent_block.get_chain_id(),
            "Aux POW parent has our chain ID"
        );

        let chain_root = merkle_tools::compute_root_from_merkle_proof(
            block_header.block_hash(),
            aux_data.chain_id,
            &aux_data.chain_merkle_proof,
        );

        let coinbase_tx = aux_data.get_coinbase_tx();
        let coinbase_tx_hash = coinbase_tx.compute_txid();

        require!(
            merkle_tools::compute_root_from_merkle_proof(
                H256::from(coinbase_tx_hash.to_raw_hash().to_byte_array()),
                0,
                &aux_data.merkle_proof,
            ) == aux_data.parent_block.merkle_root
        );

        let script_sig = coinbase_tx
            .input
            .first()
            .unwrap()
            .script_sig
            .to_hex_string();
        let pos_merged_mining_header = script_sig.find(MERGED_MINING_HEADER);
        let mut pos_chain_root = script_sig
            .find(&chain_root.to_string())
            .expect("Aux POW missing chain merkle root in parent coinbase");

        match pos_merged_mining_header {
            Some(pos_merged_mining_header) => {
                if script_sig[pos_merged_mining_header + MERGED_MINING_HEADER.len()..]
                    .contains(MERGED_MINING_HEADER)
                {
                    env::panic_str("Multiple merged mining headers in coinbase");
                }

                require!(
                    pos_merged_mining_header + MERGED_MINING_HEADER.len() == pos_chain_root,
                    "Merged mining header is not just before chain merkle root"
                );
            }
            None => {
                require!(pos_chain_root <= 40, "Aux POW chain merkle root must start in the first 20 bytes of the parent coinbase");
            }
        }

        pos_chain_root += chain_root.to_string().len();
        require!(
            script_sig.len() - pos_chain_root >= 16,
            "Aux POW missing chain merkle tree size and nonce in parent coinbase"
        );

        let bytes = hex::decode(&script_sig[pos_chain_root..pos_chain_root + 8]).unwrap();
        let n_size = u32::from_le_bytes([bytes[0], bytes[1], bytes[2], bytes[3]]);
        require!(
            n_size == (1u32 << aux_data.chain_merkle_proof.len()),
            "Aux POW merkle branch size does not match parent coinbase"
        );

        let bytes = hex::decode(&script_sig[pos_chain_root + 8..pos_chain_root + 16]).unwrap();
        let n_nonce = u32::from_le_bytes([bytes[0], bytes[1], bytes[2], bytes[3]]);

        let chain_id = block_header.get_chain_id();

        let expected_index =
            Self::get_expected_index(n_nonce, chain_id, aux_data.chain_merkle_proof.len());

        require!(
            u32::try_from(aux_data.chain_id).ok() == Some(expected_index),
            "Aux POW wrong index"
        );
        let pow_hash = aux_data.parent_block.block_hash_pow();
        require!(
            self.skip_pow_verification
                || U256::from_le_bytes(&pow_hash.0) <= target_from_bits(block_header.bits),
            format!("block should have correct pow")
        );
    }
