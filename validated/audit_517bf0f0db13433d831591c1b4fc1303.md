Audit Report

## Title
`SwapAllowlistExtension.beforeSwap` checks the router address instead of the actual swapper, allowing full allowlist bypass via `MetricOmmSimpleRouter` — (`metric-periphery/contracts/extensions/SwapAllowlistExtension.sol`)

## Summary

`SwapAllowlistExtension.beforeSwap` gates swaps by checking the `sender` argument, which `MetricOmmPool.swap` sets to `msg.sender` — the immediate caller of the pool. When swaps are routed through `MetricOmmSimpleRouter`, `sender` is the router contract address, not the end user. If a pool admin allowlists the router (the only way to let allowlisted users use the standard interface), every address on-chain can bypass the allowlist by calling the router, exposing LP positions to unrestricted counterparties.

## Finding Description

`MetricOmmPool.swap` passes `msg.sender` verbatim as the first argument to `_beforeSwap`: [1](#0-0) 

`SwapAllowlistExtension.beforeSwap` uses that `sender` value to look up the per-pool allowlist: [2](#0-1) 

`MetricOmmSimpleRouter.exactInputSingle` calls `pool.swap(...)` directly, making the router the `msg.sender` of the pool call, and passes `params.extensionData` verbatim without embedding the original caller's address: [3](#0-2) 

The same pattern applies to `exactInput`, `exactOutputSingle`, and `exactOutput` — all call `pool.swap(...)` with the router as `msg.sender` and forward user-supplied `extensionData` unchanged. [4](#0-3) 

The extension has no mechanism to recover the real user identity from a router-mediated swap. The allowlist check passes for any caller as long as the router is allowlisted, because `allowedSwapper[pool][router] == true` satisfies the guard regardless of who called the router. [5](#0-4) 

## Impact Explanation

When the router is allowlisted (the only operational path for allowlisted users to use the standard interface), any non-allowlisted EOA or contract can call `exactInputSingle`/`exactInput`/`exactOutputSingle`/`exactOutput` on the router and execute swaps against the restricted pool. LPs in a pool intended to be access-controlled are exposed to unrestricted counterparties, enabling unauthorized extraction of LP principal and fees through unconstrained swaps. This is a direct loss of LP principal above Sherlock thresholds and constitutes broken core pool functionality (the allowlist invariant is fully defeated).

## Likelihood Explanation

High. `MetricOmmSimpleRouter` is the standard public swap interface. Any pool admin who enables allowlisted users to use the router — a routine operational step — simultaneously opens the pool to every address on-chain. No special privileges, flash loans, or contract deployment are required; a plain EOA call to the router suffices. The bypass is repeatable and unconditional once the router is allowlisted.

## Recommendation

The extension must gate on the actual user identity, not the intermediary. Two sound approaches:

1. **Require the router to embed the caller in `extensionData`**: The router appends `abi.encode(msg.sender)` to `extensionData` before forwarding to the pool; the extension decodes and checks that address. The extension should revert if the embedded address is absent (direct pool calls without the router must also supply it, or be handled separately).

2. **Reject router-mediated calls explicitly**: If `sender` is a known router address, revert unless `allowAllSwappers` is set. This forces allowlisted users to call the pool directly, preserving the security invariant at the cost of UX.

## Proof of Concept

1. Pool admin deploys a pool with `SwapAllowlistExtension` configured.
2. Pool admin calls `setAllowedToSwap(pool, alice, true)` — Alice is the only intended swapper.
3. Pool admin calls `setAllowedToSwap(pool, router, true)` — necessary so Alice can use the router.
4. Charlie (not allowlisted) calls `router.exactInputSingle({pool: pool, ...})`.
5. Router calls `pool.swap(recipient, zeroForOne, amount, limit, "", extensionData)` with `msg.sender = router`.
6. Pool calls `extension.beforeSwap(router, ...)` via `_beforeSwap`.
7. Extension evaluates `allowedSwapper[pool][router]` → `true` → no revert.
8. Charlie's swap executes against the restricted pool.

A Foundry integration test can confirm this by: deploying the pool with the extension, configuring the allowlist as above, impersonating an unlisted address, calling `exactInputSingle` on the router, and asserting the swap succeeds (no `NotAllowedToSwap` revert). [6](#0-5) [7](#0-6)

    _checkDeadline(params.deadline);
    _validatePath(params.tokens, params.pools, params.extensionDatas);

    uint256 last = params.pools.length - 1;
    int128 amount = MetricOmmSwapInputs.asAmountSpecifiedIn(params.amountIn);

    for (uint256 i = 0; i <= last; i++) {
      address pool = params.pools[i];
      bool zeroForOne = MetricOmmSwapPath.resolveZeroForOneBitmap(params.zeroForOneBitMap, i);

      _setNextCallbackContext(pool, CALLBACK_MODE_JUST_PAY, i == 0 ? msg.sender : address(this), params.tokens[i]);
      (int128 amount0Delta, int128 amount1Delta) = IMetricOmmPoolActions(pool)
        .swap(
          i == last ? params.recipient : address(this),
          zeroForOne,
          amount,
          MetricOmmSwapPath.openLimit(zeroForOne),
          "",
          params.extensionDatas[i]
        );

      int128 amountInActual = MetricOmmSwapResults.extractAmountIn(zeroForOne, amount0Delta, amount1Delta);
      if (amountInActual < amount) revert InvalidInputAmountAtHop(uint8(i), amountInActual, amount);

      amount = MetricOmmSwapResults.extractAmountOut(zeroForOne, amount0Delta, amount1Delta);
    }

    if (amount <= 0) revert InvalidSwapDeltas();
    amountOut = MetricOmmSwapInputs.int128ToUint128(amount);
    if (amountOut < params.amountOutMinimum) revert InsufficientOutput(amountOut, params.amountOutMinimum);

    _clearExpectedCallbackPool();
  }
