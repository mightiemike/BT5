Audit Report

## Title
`SwapAllowlistExtension` Checks Router Address Instead of Actual Caller, Enabling Full Allowlist Bypass — (`metric-periphery/contracts/extensions/SwapAllowlistExtension.sol`)

## Summary
`MetricOmmPool.swap()` passes its own `msg.sender` as the `sender` argument to `_beforeSwap`, which the pool forwards verbatim to `SwapAllowlistExtension.beforeSwap`. When a user routes through `MetricOmmSimpleRouter`, the pool's `msg.sender` is the router contract, so the extension evaluates `allowedSwapper[pool][router]` instead of `allowedSwapper[pool][real_user]`. Any pool admin who allowlists the router to support legitimate router-mediated swaps inadvertently grants unrestricted swap access to every address on-chain.

## Finding Description
**Pool passes `msg.sender` as `sender` to the hook:**

`MetricOmmPool.swap()` calls `_beforeSwap(msg.sender, ...)` at L230–240. [1](#0-0) 

**Extension calling encodes `sender` verbatim:**

`ExtensionCalling._beforeSwap` encodes the `sender` argument directly into the call to every configured extension. [2](#0-1) 

**Extension checks the wrong identity:**

`SwapAllowlistExtension.beforeSwap` checks `allowedSwapper[msg.sender][sender]` where `msg.sender` is the pool and `sender` is whoever called `pool.swap()` — the router, not the end user. [3](#0-2) 

**Router is the direct caller of `pool.swap()`:**

`MetricOmmSimpleRouter.exactInputSingle` calls `IMetricOmmPoolActions(params.pool).swap(...)` directly, making the router the pool's `msg.sender`. [4](#0-3) 

**Contrast with the correct pattern in `DepositAllowlistExtension`:**

`DepositAllowlistExtension.beforeAddLiquidity` checks `owner` (the position owner forwarded as a dedicated parameter), not `sender`. The swap hook has no equivalent dedicated "originator" parameter. [5](#0-4) 

The root cause is structural: the swap hook signature exposes only `sender` (the pool's `msg.sender`), which collapses to the router address for all router-mediated swaps. The real user's identity is never forwarded to the extension.

## Impact Explanation
A pool configured with `SwapAllowlistExtension` to restrict trading to specific addresses (e.g., KYC-verified counterparties) is fully bypassed by any user who calls `MetricOmmSimpleRouter.exactInputSingle`, `exactInput`, `exactOutputSingle`, or `exactOutput`. The pool settles the swap normally; the extension never sees the real user. LP assets are exposed to unrestricted trading that the pool admin explicitly intended to prevent — constituting a direct loss of curation control and unauthorized execution against LP positions. This matches the "broken core pool functionality causing loss of funds or unusable swap flows" and "admin-boundary break bypassed by an unprivileged path" impact categories.

## Likelihood Explanation
`MetricOmmSimpleRouter` is a public, permissionless contract requiring no special role. The only precondition is that the pool admin allowlists the router — a natural and necessary operational step to make the router usable for any legitimate allowlisted user. Once taken, the bypass is unconditional and repeatable by any address. The attacker needs no privileged access, no special tokens, and no setup beyond calling the public router.

## Recommendation
The pool must forward the originating user's address to the extension in a verifiable way. Two sound approaches:

1. **Dedicated `originator` parameter**: Add an `originator` field to the `beforeSwap` hook signature. The pool sets `originator = msg.sender` for direct calls; the router encodes the real user's address into `extensionData` with a verifiable mechanism. The extension checks `originator`.

2. **Extension-data convention**: Define a standard encoding where the router prepends the real user's address to `extensionData`. The extension decodes and checks that address. The pool's `onlyPool` guard on the extension ensures only a legitimate pool can invoke the hook, preventing spoofing.

Until fixed, pool admins must not allowlist the router on any pool relying on `SwapAllowlistExtension` for per-user access control.

## Proof of Concept
```
1. Deploy pool with SwapAllowlistExtension configured.
   Pool admin calls:
     setAllowedToSwap(pool, router, true)   // router allowlisted for legitimate users
     setAllowedToSwap(pool, alice, true)    // alice is the only intended trader
     // bob is NOT allowlisted

2. Bob (non-allowlisted) calls:
     MetricOmmSimpleRouter.exactInputSingle({pool: pool, ...})

3. Router calls pool.swap(recipient, ...) — pool sees msg.sender = router

4. Pool calls _beforeSwap(sender=router, ...)

5. SwapAllowlistExtension checks:
     allowedSwapper[pool][router] == true  →  passes

6. Bob's swap executes and settles successfully.

Expected: revert NotAllowedToSwap()
Actual:   swap settles, bob receives output tokens

Foundry test: deploy pool + extension, configure as above, prank as bob,
call router.exactInputSingle, assert no revert occurs.
```
