### Title
Gateway Stateless and Stateful L2 Gas Price Validators Use Uncoordinated Admission Criteria — (`File: crates/apollo_gateway/src/stateless_transaction_validator.rs` / `crates/apollo_gateway/src/stateful_transaction_validator.rs`)

### Summary

The `StatelessTransactionValidator` enforces a **static absolute floor** on `l2_gas.max_price_per_unit`, while the `StatefulTransactionValidator` enforces a **dynamic percentage-based threshold** derived from the previous block's actual L2 gas price. These two criteria are independent, uncoordinated configurations. When the network's L2 gas price is below the static floor, valid transactions (those that would pass the stateful check) are rejected by the stateless check. When the network's L2 gas price is above the static floor (normal production), transactions at the static floor pass the cheap stateless check but fail the expensive stateful check, enabling a resource-exhaustion path at the gateway.

### Finding Description

**Stateless validator** (`StatelessTransactionValidator::validate_resource_bounds`):

```rust
if resource_bounds.l2_gas.max_price_per_unit.0 < self.config.min_gas_price {
    return Err(StatelessTransactionValidatorError::MaxGasPriceTooLow { ... });
}
```

The threshold is `StatelessTransactionValidatorConfig::min_gas_price`, a **static u128 value** (default `8_000_000_000`, i.e. 8 Gwei), stored independently in the gateway config. The code itself acknowledges this is a temporary duplicate:

```rust
// TODO(AlonH): Remove the `min_gas_price` field from this struct and use the one from the
// versioned constants.
pub min_gas_price: u128,
```

**Stateful validator** (`StatefulTransactionValidator::validate_tx_l2_gas_price_within_threshold`):

```rust
let gas_price_threshold_multiplier =
    Ratio::new(self.config.min_gas_price_percentage.into(), 100_u128);
let threshold = (gas_price_threshold_multiplier
    * previous_block_l2_gas_price.get().0)
    .to_integer();
if tx_l2_gas_price.0 < threshold {
    return Err(StarknetError { ... });
}
```

The threshold is `previous_block_l2_gas_price × min_gas_price_percentage / 100` — a **dynamic value** that tracks the actual market price. The two thresholds are completely independent: one is a static config constant, the other is a live oracle read multiplied by a percentage.

The discrepancy creates two divergent admission zones:

| Condition | Stateless result | Stateful result | Net effect |
|---|---|---|---|
| `market_price > static_floor` (normal production) | Accepts tx at static floor | Rejects tx at static floor | Gateway wastes stateful validation resources; DoS vector |
| `market_price < static_floor` (bootstrap / low-activity) | Rejects tx between dynamic threshold and static floor | Would accept same tx | Valid transactions rejected before sequencing |

Additionally, when `StatefulTransactionValidatorConfig::validate_resource_bounds = false` (bootstrap phase), the stateful gas-price check is skipped entirely. Only the static floor applies. Transactions with `l2_gas.max_price_per_unit = 8 Gwei` are admitted to the mempool even when the actual block gas price is far higher, and they will fail during blockifier pre-validation (`check_fee_bounds`), which checks against the live block price.

### Impact Explanation

**High. Mempool/gateway/RPC admission rejects valid transactions before sequencing.**

When the network L2 gas price is below the static floor (e.g., during bootstrap, network restart, or a sustained low-activity period), any transaction whose `l2_gas.max_price_per_unit` satisfies the dynamic stateful threshold but falls below the static stateless floor is rejected at the first gate. The transaction is economically valid (it would clear the stateful check and execute successfully), but the gateway refuses it with `MaxGasPriceTooLow`. Users cannot submit these transactions through the gateway regardless of how they set their gas price, as long as it is below the static constant.

**High. Mempool/gateway/RPC admission accepts invalid transactions before sequencing (bootstrap path).**

When `validate_resource_bounds = false` in the stateful config, the gateway accepts transactions at the static floor (8 Gwei) into the mempool even when the block gas price is orders of magnitude higher. These transactions are invalid from the blockifier's perspective and will be rejected during execution, wasting mempool slots and batcher resources.

### Likelihood Explanation

**Low.** The primary scenario (market price below 8 Gwei) requires an unusual network state. The bootstrap scenario (`validate_resource_bounds = false`) is a documented temporary configuration. However, the root cause — two independent, uncoordinated gas-price thresholds — is a structural invariant violation that persists in the production codebase and is acknowledged by the TODO comment.

### Recommendation

1. Remove `StatelessTransactionValidatorConfig::min_gas_price` and replace it with the value from `VersionedConstants` (as the TODO already notes), so both validators derive their floor from the same canonical source.
2. Alternatively, make the stateless validator read the same dynamic threshold used by the stateful validator (previous block price × percentage), so the two gates are always consistent.
3. Ensure that when `validate_resource_bounds = false` in the stateful config, the stateless validator's floor is also relaxed or disabled, so the two flags are coupled.

### Proof of Concept

```
Setup:
  StatelessTransactionValidatorConfig::min_gas_price = 8_000_000_000  (8 Gwei, default)
  StatefulTransactionValidatorConfig::min_gas_price_percentage = 100   (default)
  previous_block_l2_gas_price = 5_000_000_000                          (5 Gwei, e.g. bootstrap)

Attacker submits invoke V3 tx with:
  l2_gas.max_price_per_unit = 6_000_000_000  (6 Gwei)

Stateless check:
  6_000_000_000 < 8_000_000_000  → REJECTED with MaxGasPriceTooLow

Stateful check (if reached):
  threshold = 5_000_000_000 * 100 / 100 = 5_000_000_000
  6_000_000_000 >= 5_000_000_000  → ACCEPTED

Result: A transaction that is economically valid (clears the stateful threshold and
would execute successfully at the current block price) is permanently rejected by
the gateway because the static floor in the stateless validator is higher than the
live market price. The user cannot submit this transaction through any gateway
instance with the default configuration.

Reverse scenario (production, market = 100 Gwei):
  Attacker submits tx with l2_gas.max_price_per_unit = 8_000_000_000 (8 Gwei)
  Stateless: 8 >= 8  → ACCEPTED (proceeds to expensive stateful validation)
  Stateful:  threshold = 100 Gwei; 8 < 100  → REJECTED
  Gateway ran full stateful validation (state reads + validate entry point) for a
  transaction that the stateless check should have blocked.
```

    // TODO(Arni): Consider running this validation for all gas prices.
    fn validate_tx_l2_gas_price_within_threshold(
        &self,
        tx_resource_bounds: ValidResourceBounds,
        previous_block_l2_gas_price: NonzeroGasPrice,
    ) -> StatefulTransactionValidatorResult<()> {
        match tx_resource_bounds {
            ValidResourceBounds::AllResources(tx_resource_bounds) => {
                let tx_l2_gas_price = tx_resource_bounds.l2_gas.max_price_per_unit;
                let gas_price_threshold_multiplier =
                    Ratio::new(self.config.min_gas_price_percentage.into(), 100_u128);
                let threshold = (gas_price_threshold_multiplier
                    * previous_block_l2_gas_price.get().0)
                    .to_integer();
                if tx_l2_gas_price.0 < threshold {
                    return Err(StarknetError {
                        // We didn't have this kind of an error.
                        code: StarknetErrorCode::UnknownErrorCode(
                            "StarknetErrorCode.GAS_PRICE_TOO_LOW".to_string(),
                        ),
                        message: format!(
                            "Transaction L2 gas price {tx_l2_gas_price} is below the required \
                             threshold {threshold}.",
                        ),
                    });
                }
            }
            ValidResourceBounds::L1Gas(_) => {
                // No validation required for legacy transactions.
            }
        }
        Ok(())
    }
