### Title
Unprivileged Caller Can Force Maximum Garbage Collection via `run_mainchain_gc()` - (File: `contract/src/lib.rs`)

---

### Summary

`run_mainchain_gc()` is a public, state-mutating function with no caller authorization check. Any unprivileged NEAR account can call it with an attacker-controlled `batch_size` (e.g., `u64::MAX`), forcing the contract to immediately prune the maximum number of eligible block headers in a single transaction. This breaks SPV proof verification for any block that is prematurely evicted.

---

### Finding Description

`run_mainchain_gc()` is decorated only with `#[pause(except(roles(Role::UnrestrictedRunGC)))]`, which gates it behind the contract-paused flag but imposes **no restriction on who may call it when the contract is live**. [1](#0-0) 

The function accepts a caller-supplied `batch_size: u64` and uses it directly to determine how many headers to delete: [2](#0-1) 

For each height in the removal range it permanently deletes entries from `mainchain_height_to_header`, `mainchain_header_to_height`, and `headers_pool`, then advances `mainchain_initial_blockhash`: [3](#0-2) 

The intended, controlled path is `submit_blocks`, which calls `run_mainchain_gc(num_of_headers)` where `num_of_headers` equals the small batch submitted by the relayer — a naturally throttled value: [4](#0-3) 

An attacker bypasses this throttle entirely by calling `run_mainchain_gc(u64::MAX)` directly, removing every header that exceeds `gc_threshold` in one atomic call.

---

### Impact Explanation

`verify_transaction_inclusion` and `verify_transaction_inclusion_v2` both require the target block to be present in `mainchain_header_to_height`. If the block has been GC'd, the call panics: [5](#0-4) 

Any downstream NEAR contract or user that submitted a valid SPV proof for a block that was prematurely evicted receives a hard failure. Because `mainchain_initial_blockhash` is also advanced, the pruning is irreversible — the evicted headers cannot be re-queried or re-inserted without re-submitting the entire chain segment.

---

### Likelihood Explanation

The function is unconditionally reachable by any NEAR account when the contract is not paused. No tokens, staking, or special role is required. The call costs only standard NEAR gas. A griefing actor can monitor the chain for the moment `amount_of_headers_we_store > gc_threshold` becomes true and immediately call `run_mainchain_gc(u64::MAX)` to evict the maximum eligible window in one shot, racing ahead of legitimate relayer submissions.

---

### Recommendation

Add a role-based access check so that only authorized accounts (e.g., `Role::DAO` or a dedicated `GCManager` role) can invoke `run_mainchain_gc` directly:

```rust
#[pause(except(roles(Role::UnrestrictedRunGC)))]
pub fn run_mainchain_gc(&mut self, batch_size: u64) {
    require!(
        self.acl_has_role(Role::DAO, &env::predecessor_account_id())
            || self.acl_has_role(Role::UnrestrictedRunGC, &env::predecessor_account_id()),
        "run_mainchain_gc: caller is not authorized"
    );
    // ... existing logic
}
```

Alternatively, make the function `pub(crate)` and remove it from the public NEAR ABI, relying solely on the internal call from `submit_blocks`.

---

### Proof of Concept

1. Deploy the contract with `gc_threshold = 52704` and seed it with 52705+ headers so `amount_of_headers_we_store > gc_threshold`.
2. From any unprivileged NEAR account, call:
   ```
   run_mainchain_gc(18446744073709551615)   // u64::MAX
   ```
3. The contract removes `amount_of_headers_we_store - gc_threshold` headers (e.g., 1 or more of the oldest blocks) in one call, advancing `mainchain_initial_blockhash`.
4. Any subsequent call to `verify_transaction_inclusion` for a block in the evicted range panics with `"block does not belong to the current main chain"`.

The root cause — a public state-mutating function with an attacker-controlled parameter and no caller identity check — is structurally identical to the MapleLoan `closeLoan()` finding: a function intended to be invoked only under controlled conditions is left open to any caller, enabling a griefing attack that forces legitimate users to absorb the consequences. [6](#0-5)

```rust
    /// Public call to run GC on a mainchain.
    /// `batch_size` is how many block headers should be removed in the execution
    ///
    /// # Panics
    /// If initial blockheader or tip blockheader are not in a header pool
    #[pause(except(roles(Role::UnrestrictedRunGC)))]
    pub fn run_mainchain_gc(&mut self, batch_size: u64) {
        let initial_blockheader = self
            .headers_pool
            .get(&self.mainchain_initial_blockhash)
            .unwrap_or_else(|| env::panic_str(ERR_KEY_NOT_EXIST));

        let tip_blockheader = self
            .headers_pool
            .get(&self.mainchain_tip_blockhash)
            .unwrap_or_else(|| env::panic_str(ERR_KEY_NOT_EXIST));

        let amount_of_headers_we_store =
            tip_blockheader.block_height - initial_blockheader.block_height + 1;

        if amount_of_headers_we_store > self.gc_threshold {
            let total_amount_to_remove = amount_of_headers_we_store - self.gc_threshold;
            let selected_amount_to_remove = std::cmp::min(total_amount_to_remove, batch_size);

            let start_removal_height = initial_blockheader.block_height;
            let end_removal_height = initial_blockheader.block_height + selected_amount_to_remove;
            env::log_str(&format!(
                "Num of blocks to remove {selected_amount_to_remove}"
            ));

            for height in start_removal_height..end_removal_height {
                let blockhash = &self
                    .mainchain_height_to_header
                    .get(&height)
                    .unwrap_or_else(|| env::panic_str(ERR_KEY_NOT_EXIST));

                self.remove_block_header(blockhash);
                self.mainchain_height_to_header.remove(&height);
            }

            self.mainchain_initial_blockhash = self
                .mainchain_height_to_header
                .get(&end_removal_height)
                .unwrap_or_else(|| env::panic_str(ERR_KEY_NOT_EXIST));
        }
    }
```
