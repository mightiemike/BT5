### Title
Non-Creator NFT Owner Permanently Denied Burn Rights — (`x/nft/keeper/keeper.go`)

### Summary

`keeper.BurnNFT` enforces two independent authorization checks: the caller must be both the **current owner** of the NFT (`IsOwner`) and the **original creator** of the denomination (`IsDenomCreator`). Because NFT ownership is transferable but denom creator status is immutable, any NFT that has been transferred to a non-creator address becomes permanently non-burnable by its owner via `MsgBurnNFT`.

---

### Finding Description

`BurnNFT` in `x/nft/keeper/keeper.go` applies both checks sequentially: [1](#0-0) 

`IsDenomCreator` in `x/nft/keeper/denom.go` compares the caller against the immutable `denom.Creator` field stored at issuance time: [2](#0-1) 

The denom creator field is set once at `IssueDenom` and never updated. `TransferOwner` only updates the NFT's `Owner` field and the owner index — it does not touch the denom's `Creator`: [3](#0-2) 

The `msgServer.BurnNFT` entry point passes the transaction signer directly as `owner` with no additional privilege escalation: [4](#0-3) 

The module's own spec documentation confirms this is the intended design, explicitly stating the sender must be both owner and creator: [5](#0-4) 

---

### Impact Explanation

Any NFT minted to a recipient address (via `MsgMintNFT` with `recipient != sender`) or transferred to a non-creator address (via `MsgTransferNFT`) is **permanently non-burnable** by its owner through the standard `MsgBurnNFT` path. The NFT is locked in the recipient's account against their will with no on-chain remedy. The only bypass is `BurnNFTUnverified`, which skips the creator check: [6](#0-5) 

But `BurnNFTUnverified` is only called internally by the IBC transfer module (`x/nft-transfer/keeper/packet.go`) and is not exposed as a user-callable message. The victim has no supported production path to burn their own NFT.

---

### Likelihood Explanation

This is triggered by the normal, intended use of the NFT module. Minting to a recipient or transferring an NFT are core advertised features. Every such transfer creates a permanently non-burnable NFT for the recipient. No special privileges, governance, or key compromise are required — only two standard transactions.

---

### Recommendation

Remove the `IsDenomCreator` check from `BurnNFT`. Ownership alone should be sufficient authorization to burn. If the protocol requires creator-only burn semantics, this must be documented as a hard constraint and the `MsgMintNFT`/`MsgTransferNFT` flows must warn users that transferred NFTs cannot be burned by recipients. A separate `MsgBurnNFTByCreator` could be introduced for creator-privileged burns, while `MsgBurnNFT` enforces only `IsOwner`.

---

    err := suite.keeper.IssueDenom(suite.ctx, denomID, denomNm, schema, uri, creator)

    err = suite.keeper.MintNFT(suite.ctx, denomID, tokenID, tokenNm, tokenURI, tokenData, creator, victim)
    suite.NoError(err)

    // 3. Victim (owner) attempts to burn their own NFT

    // RESULT: ErrUnauthorized — victim cannot burn their own NFT
    suite.ErrorContains(err, "is not the creator of")

// TransferOwner transfers the ownership of the given NFT to the new owner
func (k Keeper) TransferOwner(
	ctx sdk.Context, denomID, tokenID string, srcOwner, dstOwner sdk.AccAddress,
) error {
	if !k.HasDenomID(ctx, denomID) {
		return sdkerrors.Wrapf(types.ErrInvalidDenom, "denom ID %s not exists", denomID)
	}

	nft, err := k.IsOwner(ctx, denomID, tokenID, srcOwner)
	if err != nil {
		return err
	}

	nft.Owner = dstOwner.String()

	k.setNFT(ctx, denomID, nft)
	k.swapOwner(ctx, denomID, tokenID, srcOwner, dstOwner)
	return nil
}

func (k Keeper) BurnNFT(ctx sdk.Context, denomID, tokenID string, owner sdk.AccAddress) error {
	if !k.HasDenomID(ctx, denomID) {
		return sdkerrors.Wrapf(types.ErrInvalidDenom, "denom ID %s not exists", denomID)
	}

	nft, err := k.IsOwner(ctx, denomID, tokenID, owner)
	if err != nil {
		return err
	}

	_, err = k.IsDenomCreator(ctx, denomID, owner)
	if err != nil {
		return err
	}

	k.deleteNFT(ctx, denomID, nft)
	k.deleteOwner(ctx, denomID, tokenID, owner)
	k.decreaseSupply(ctx, denomID)

	return nil
}
```

func (k Keeper) IsDenomCreator(ctx sdk.Context, denomID string, address sdk.AccAddress) (types.Denom, error) {
	denom, err := k.GetDenom(ctx, denomID)
	if err != nil {
		return types.Denom{}, err
	}

	creator, err := sdk.AccAddressFromBech32(denom.Creator)
	if err != nil {
		panic(err)
	}

	if !creator.Equals(address) {
		return types.Denom{}, sdkerrors.Wrapf(types.ErrUnauthorized, "%s is not the creator of %s", address, denomID)
	}

	return denom, nil
}
```
