### Title
Gateway `StatelessTransactionValidator` Enforces `min_gas_price` Floor Only on L2 Gas, Allowing Transactions with Sub-Minimum L1/L1-Data Gas Prices to Bypass Admission Control — (`crates/apollo_gateway/src/stateless_transaction_validator.rs`)

### Summary

The `validate_resource_bounds` function in `StatelessTransactionValidator` applies the configured `min_gas_price` floor exclusively to `l2_gas.max_price_per_unit`. The `l1_gas.max_price_per_unit` and `l1_data_gas.max_price_per_unit` fields are never checked against any minimum. A transaction with `l1_gas.max_price_per_unit = 1` (far below the 8 Gwei minimum) and `l2_gas.max_price_per_unit >= min_gas_price` passes stateless gateway validation and is admitted to the mempool, even though it will be rejected by the blockifier at pre-validation time.

### Finding Description

In `crates/apollo_gateway/src/stateless_transaction_validator.rs`, `validate_resource_bounds` performs two checks when `validate_resource_bounds` is enabled:

1. A global zero-fee guard: `max_possible_fee(Tip::ZERO) == Fee(0)` — rejects only if the sum across all three resources is exactly zero.
2. A minimum price floor: `resource_bounds.l2_gas.max_price_per_unit.0 < self.config.min_gas_price` — applied **only to L2 gas**. [1](#0-0) 

No equivalent floor is applied to `l1_gas.max_price_per_unit` or `l1_data_gas.max_price_per_unit`. The production default for `min_gas_price` is 8 Gwei (`8_000_000_000`): [2](#0-1) 

A crafted transaction:
```
l1_gas:      { max_amount: 1000, max_price_per_unit: 1 }        // 1 wei — far below 8 Gwei
l2_gas:      { max_amount: 1000, max_price_per_unit: 8_000_000_000 }  // exactly at floor
l1_data_gas: { max_amount: 1000, max_price_per_unit: 1 }        // 1 wei — far below 8 Gwei
```

passes both gateway checks:
- `max_possible_fee = 1000×1 + 1000×8_000_000_000 + 1000×1 = 8_000_002_000 > 0` → ZeroResourceBounds check passes.
- `l2_gas.max_price_per_unit = 8_000_000_000 >= 8_000_000_000` → `min_gas_price` check passes.

The transaction is admitted to the mempool. However, the blockifier's `check_fee_bounds` (called during `perform_pre_validation_stage` when `charge_fee = true`) checks **all three** resource prices against actual block gas prices: [3](#0-2) 

With actual L1 gas price ≈ 10 Gwei, `l1_gas.max_price_per_unit (1) < actual_l1_gas_price (10_000_000_000)` triggers `ResourceBoundsError::MaxGasPriceTooLow`, and the transaction fails at pre-validation inside the blockifier — after the sequencer has already spent resources sequencing it.

The stateful validator also only checks L2 gas price (as a percentage of the previous block's L2 gas price), providing no additional protection for L1/L1-data gas prices: [4](#0-3) 

The shared `validate_resource_bounds` pointer target controls both validators simultaneously: [5](#0-4) 

### Impact Explanation

**High — Mempool/gateway admission accepts invalid transactions before sequencing.**

The gateway is the first line of defense. Transactions with sub-minimum L1/L1-data gas prices are admitted to the mempool and sequenced into block proposals, only to be rejected by the blockifier at pre-validation. This wastes sequencer CPU, mempool slots, and block space. An attacker can flood the mempool with such transactions (each with a different nonce) at negligible cost, since the gateway does not reject them. The sequencer's throughput for legitimate transactions is degraded.

### Likelihood Explanation

Any unprivileged user can submit a V3 transaction with `l1_gas.max_price_per_unit = 1` and `l2_gas.max_price_per_unit = min_gas_price`. No special account state or privilege is required. The crafted transaction is syntactically valid and passes all other stateless checks (calldata length, signature length, DA mode, etc.). The attack is trivially reproducible.

### Recommendation

Apply the `min_gas_price` floor to all three resource types, not only L2 gas:

```rust
fn validate_resource_bounds(
    &self,
    tx: &RpcTransaction,
) -> StatelessTransactionValidatorResult<()> {
    if !self.config.validate_resource_bounds {
        return Ok(());
    }

    let resource_bounds = *tx.resource_bounds();
    if ValidResourceBounds::AllResources(resource_bounds).max_possible_fee(Tip::ZERO) == Fee(0) {
        return Err(StatelessTransactionValidatorError::ZeroResourceBounds { resource_bounds });
    }

    // Apply min_gas_price to all resource types, not just L2 gas.
    for (price, label) in [
        (resource_bounds.l1_gas.max_price_per_unit.0, "l1_gas"),
        (resource_bounds.l2_gas.max_price_per_unit.0, "l2_gas"),
        (resource_bounds.l1_data_gas.max_price_per_unit.0, "l1_data_gas"),
    ] {
        if price < self.config.min_gas_price {
            return Err(StatelessTransactionValidatorError::MaxGasPriceTooLow {
                gas_price: GasPrice(price),
                min_gas_price: self.config.min_gas_price,
            });
        }
    }
    ...
}
```

Alternatively, introduce separate `min_l1_gas_price` and `min_l1_data_gas_price` fields in `StatelessTransactionValidatorConfig` to allow independent tuning.
