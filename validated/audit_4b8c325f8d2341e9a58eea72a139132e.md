### Title
Deprecated `verify_transaction_inclusion` Remains Publicly Callable, Enabling 64-Byte Transaction Merkle Proof Forgery — (File: contract/src/lib.rs)

---

### Summary

The contract exposes two transaction-inclusion verification entry points. The newer `verify_transaction_inclusion_v2` adds a mandatory coinbase Merkle proof check to defeat the well-known 64-byte-transaction forgery attack. The older `verify_transaction_inclusion` lacks that check entirely and is marked only with a Rust `#[deprecated]` attribute, which emits a compiler warning but imposes **no runtime restriction**. Any unprivileged NEAR caller can invoke the deprecated method directly, bypass the coinbase validation, and receive a `true` proof result for a fabricated transaction.

---

### Finding Description

`verify_transaction_inclusion` is still a `pub` NEAR contract method decorated with `#[pause]` but not with any access-control guard that would block external callers:

```rust
// contract/src/lib.rs  lines 283-323
#[deprecated(
    since = "0.5.0",
    note = "Use `verify_transaction_inclusion_v2` instead."
)]
#[pause]
pub fn verify_transaction_inclusion(&self, #[serializer(borsh)] args: ProofArgs) -> bool {
    ...
    require!(!args.merkle_proof.is_empty(), "Merkle proof is empty");

    merkle_tools::compute_root_from_merkle_proof(
        args.tx_id,
        usize::try_from(args.tx_index).unwrap(),
        &args.merkle_proof,
    ) == header.block_header.merkle_root   // ← no coinbase check
}
```

`verify_transaction_inclusion_v2` (lines 347-369) fixes this by first verifying that the coinbase transaction's Merkle proof resolves to the stored `merkle_root`, then delegating to the deprecated function. Because the coinbase is always at index 0, a valid coinbase proof proves the prover knows the real leaf set and cannot substitute an internal node as a fake leaf.

The deprecated function performs no such check. Bitcoin's Merkle tree hashes pairs of 32-byte values into 32-byte nodes; an internal node is therefore indistinguishable in size from a transaction hash. An attacker can supply any internal node of a real block's Merkle tree as `tx_id`, together with a proof path that correctly reconstructs `merkle_root`, and the function returns `true` — falsely asserting that a non-existent transaction was confirmed in that block.

The only guard present in the deprecated path is `require!(!args.merkle_proof.is_empty())`, which does not prevent the forgery because the forged proof is non-empty.

---

### Impact Explanation

Any bridge, atomic-swap protocol, or cross-chain finance contract that calls `verify_transaction_inclusion` (rather than v2) receives a `true` result for a fabricated Bitcoin transaction. This corrupts the proof result — the exact state value the light client is designed to protect — and allows an attacker to claim settlement of a Bitcoin payment that never occurred, potentially draining funds from the consuming contract.

---

### Likelihood Explanation

The entry path requires no privileged role: any NEAR account can call `verify_transaction_inclusion` at any time the contract is not paused. Constructing a valid 64-byte forgery requires only knowledge of a real block's Merkle tree, which is public data. Consuming contracts that integrated before v0.5.0 or that call the method by name without checking the deprecation notice are silently exposed.

---

### Recommendation

Remove `verify_transaction_inclusion` from the public ABI entirely, or restrict it with `#[private]` / an access-control role so that only the contract itself (as called from `verify_transaction_inclusion_v2`) can reach it. If backward compatibility must be preserved, replicate the coinbase-proof check inside the deprecated function body and document that the function is safe to call only when the coinbase proof is also supplied.

---

### Proof of Concept

1. Select any block already accepted into the light client's mainchain (its hash is in `mainchain_header_to_height`).
2. Obtain the block's full Merkle tree (public Bitcoin data). Pick any internal node `N` at depth `d`; its 32-byte hash is the attacker's `tx_id`.
3. Build a Merkle proof of length `d` that walks from `N` up to `merkle_root` using the sibling hashes at each level — this proof is valid because `N` genuinely exists in the tree.
4. Call `verify_transaction_inclusion` with `tx_id = N`, `tx_index` matching `N`'s position, and the constructed `merkle_proof`.
5. `compute_root_from_merkle_proof` reconstructs `merkle_root` correctly; the function returns `true`, asserting that the fabricated transaction was confirmed in that block. [1](#0-0) [2](#0-1) [3](#0-2)

        require!(
            args.confirmations <= self.gc_threshold,
            "The required number of confirmations exceeds the number of blocks stored in memory"
        );

        let heaviest_block_header = self
            .headers_pool
            .get(&self.mainchain_tip_blockhash)
            .unwrap_or_else(|| env::panic_str(ERR_KEY_NOT_EXIST));
        let target_block_height = self
            .mainchain_header_to_height
            .get(&args.tx_block_blockhash)
            .unwrap_or_else(|| env::panic_str("block does not belong to the current main chain"));

        // Check requested confirmations. No need to compute proof if insufficient confirmations.
        require!(
            (heaviest_block_header.block_height).saturating_sub(target_block_height) + 1
                >= args.confirmations,
            "Not enough blocks confirmed"
        );

        let header = self
            .headers_pool
            .get(&args.tx_block_blockhash)
            .unwrap_or_else(|| env::panic_str("cannot find requested transaction block"));

pub fn compute_root_from_merkle_proof(
    transaction_hash: H256,
    transaction_position: usize,
    merkle_proof: &Vec<H256>,
) -> H256 {
    let mut current_hash = transaction_hash;
    let mut current_position = transaction_position;

    for proof_hash in merkle_proof {
        if current_position % 2 == 0 {
            current_hash = compute_hash(&current_hash, proof_hash);
        } else {
            current_hash = compute_hash(proof_hash, &current_hash);
        }
        current_position /= 2;
    }

    current_hash
}
