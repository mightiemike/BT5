Audit Report

## Title
SwapAllowlistExtension Per-User Allowlist Bypassed via MetricOmmSimpleRouter — (File: metric-periphery/contracts/extensions/SwapAllowlistExtension.sol)

## Summary
`SwapAllowlistExtension.beforeSwap` gates swaps by checking the `sender` argument, which `MetricOmmPool.swap` always sets to `msg.sender` — the immediate caller of `pool.swap()`. When `MetricOmmSimpleRouter` intermediates a swap, the router is `msg.sender`, so the extension checks whether the **router** is allowlisted, not the end user. Any pool admin who allowlists the router (required for router-mediated swaps to succeed) inadvertently opens the pool to every unprivileged user who calls the router.

## Finding Description
`MetricOmmPool.swap` passes `msg.sender` as the `sender` argument to `_beforeSwap`: [1](#0-0) 

`ExtensionCalling._beforeSwap` forwards this `sender` value verbatim to every configured extension via `_callExtensionsInOrder`: [2](#0-1) 

`SwapAllowlistExtension.beforeSwap` then checks `allowedSwapper[msg.sender][sender]`, where `msg.sender` is the pool and `sender` is whoever called `pool.swap()`: [3](#0-2) 

`MetricOmmSimpleRouter.exactInputSingle` calls `pool.swap()` directly, making the router `msg.sender` of that call: [4](#0-3) 

The same pattern holds for `exactInput`, `exactOutputSingle`, and `exactOutput`. There is no point in the call chain where the originating EOA is recovered or checked. The extension's `allowedSwapper` mapping is keyed on the router address, not the end user, so `allowedSwapper[pool][router] == true` satisfies the check for **any** caller of the router.

## Impact Explanation
A pool admin who deploys a pool with `SwapAllowlistExtension` to restrict swaps to a curated set (e.g., KYC'd counterparties) must also allowlist the router for those users to trade via the canonical periphery entry point. The moment the router is allowlisted, the per-user allowlist is entirely defeated: any address can call `MetricOmmSimpleRouter.exactInputSingle` (or any other router entry point), which calls `pool.swap()` with `msg.sender = router`, and the extension passes unconditionally. The invariant `allowedSwapper[pool][Bob] == false` is broken — Bob swaps successfully on a pool that explicitly did not allowlist him. This constitutes broken core pool functionality (the allowlist access control) and enables unauthorized traders to extract value from LP inventory sized and priced for a controlled counterparty set, placing LP principal at risk.

## Likelihood Explanation
`MetricOmmSimpleRouter` is the canonical periphery entry point; most users interact with pools through it. Any pool admin who enables both a swap allowlist and router access — a natural and expected combination — creates the bypass automatically. No special privilege, flash loan, or oracle manipulation is required: a plain router call suffices. The bypass is silent; no event distinguishes a router-mediated swap from a direct one.

## Recommendation
The extension must gate the **economic actor**, not the immediate caller. The most robust fix is to have the router encode the originating user (`msg.sender`) into `extensionData` before forwarding to the pool, and have `SwapAllowlistExtension.beforeSwap` decode and check that address when `sender` is a known router. Alternatively, maintain a router registry in the extension: when `sender` is a registered router, require the actual user address to be supplied and verified (e.g., via transient storage or a signed payload). A third option is to document and enforce that the router must never be added to a per-user allowlist — instead, pools that allow all router users should use `setAllowAllSwappers(pool, true)` rather than adding the router to the per-user list.

## Proof of Concept
```
Setup
─────
1. Admin deploys pool with SwapAllowlistExtension as BEFORE_SWAP extension.
2. Admin calls setAllowedToSwap(pool, Alice, true)   // Alice is the intended user
3. Admin calls setAllowedToSwap(pool, router, true)  // needed so Alice can use the router

Attack
──────
4. Bob (not allowlisted) calls:
       MetricOmmSimpleRouter.exactInputSingle({
           pool:            pool,
           recipient:       Bob,
           amountIn:        X,
           ...
       })

5. Router calls pool.swap(recipient=Bob, ...) → msg.sender = router

6. Pool calls _beforeSwap(sender=router, ...)

7. SwapAllowlistExtension checks:
       allowedSwapper[pool][router]  →  true   ✓

8. Swap executes. Bob receives output tokens.
   allowedSwapper[pool][Bob] == false was never checked.
```
