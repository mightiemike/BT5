### Title
Missing `lastCumulativeFundingX18` Checkpoint Update After Socialization Corrupts Subaccount Accounting — (File: `core/contracts/PerpEngine.sol`)

---

### Summary

In `PerpEngine.socializeSubaccount`, after modifying `state.cumulativeFundingLongX18` and `state.cumulativeFundingShortX18` to spread bad debt across all open-interest holders, the socialized subaccount's `balance.lastCumulativeFundingX18` is not advanced to the new cumulative value. On the next `_updateBalance` call for that subaccount, the stale checkpoint causes the full socialization delta to be re-applied as a funding charge, re-creating a debt that was just cleared.

---

### Finding Description

`PerpEngine.socializeSubaccount` is called during liquidation when insurance is insufficient to cover a subaccount's negative `vQuoteBalance`. The function:

1. Calls `getStateAndBalance`, which internally calls `_updateBalance(state, balance, 0, 0)`. This sets `balance.lastCumulativeFundingX18 = state.cumulativeFundingLongX18` (call this value **C**). [1](#0-0) 

2. Computes `fundingPerShare = -balance.vQuoteBalance / state.openInterest` and increments `state.cumulativeFundingLongX18 += fundingPerShare` (now **C + fundingPerShare**), decrements `state.cumulativeFundingShortX18 -= fundingPerShare`, and sets `balance.vQuoteBalance = 0`. [2](#0-1) 

3. Saves the state and balance via `_setState` and `_setBalanceAndUpdateBitmap`. The saved balance still carries `lastCumulativeFundingX18 = C` — it is **never updated** to `C + fundingPerShare`. [3](#0-2) 

On the next `_updateBalance` call for this subaccount (triggered by any subsequent trade, settlement, or health check):

```
diffX18 = state.cumulativeFundingLongX18 - balance.lastCumulativeFundingX18
        = (C + fundingPerShare) - C
        = fundingPerShare

deltaQuote = vQuoteDelta - diffX18 * balance.amount
           = 0 - fundingPerShare * balance.amount

balance.vQuoteBalance += deltaQuote   // re-creates the debt
``` [4](#0-3) 

The subaccount's debt was cleared to zero by socialization, but the stale checkpoint causes `_updateBalance` to charge it `fundingPerShare * balance.amount` again on the very next interaction.

---

### Impact Explanation

The socialized subaccount accumulates an unintended negative `vQuoteBalance` equal to `fundingPerShare × balance.amount` on its next balance update. This:

- Corrupts the subaccount's PnL accounting.
- Reduces its health score by an amount proportional to its remaining position size and the magnitude of the socialized bad debt.
- Can push the subaccount below maintenance margin, triggering a second, unwarranted liquidation cascade.
- The corrupted `vQuoteBalance` also flows into `settlePnl`, meaning the subaccount would transfer fewer (or negative) quote tokens to the spot engine than it should. [5](#0-4) 

---

### Likelihood Explanation

Socialization is rare but deterministic: it fires whenever a liquidated subaccount's bad debt exceeds available insurance. Once triggered, the double-charge is **guaranteed** on the very next `_updateBalance` call for that subaccount — which occurs on any subsequent trade, `settlePnl`, or health evaluation. Any liquidator can trigger the root cause by liquidating an undercollateralized account with insufficient insurance coverage. No privileged access is required beyond being a liquidator.

---

### Recommendation

After modifying the cumulative funding accumulators inside `socializeSubaccount`, update `balance.lastCumulativeFundingX18` to the new cumulative value before persisting the balance:

```solidity
// after: state.cumulativeFundingLongX18 += fundingPerShare;
//        state.cumulativeFundingShortX18 -= fundingPerShare;
//        balance.vQuoteBalance = 0;

// ADD:
if (balance.amount > 0) {
    balance.lastCumulativeFundingX18 = state.cumulativeFundingLongX18;
} else {
    balance.lastCumulativeFundingX18 = state.cumulativeFundingShortX18;
}
```

This mirrors the checkpoint update already performed at the end of `_updateBalance`. [6](#0-5) 

---

### Proof of Concept

1. Subaccount A holds a long position: `balance.amount = 1000`, `balance.vQuoteBalance = -500` (bad debt), `balance.lastCumulativeFundingX18 = C`. Total `openInterest = 10000`.
2. Insurance is 0. Liquidator calls liquidation → `socializeSubaccount` is invoked.
3. `fundingPerShare = 500 / 10000 = 0.05`. `state.cumulativeFundingLongX18` becomes `C + 0.05`. `balance.vQuoteBalance = 0`. `balance.lastCumulativeFundingX18` remains `C`.
4. State saved with `cumulativeFundingLongX18 = C + 0.05`; balance saved with `lastCumulativeFundingX18 = C`.
5. Any subsequent action on subaccount A calls `_updateBalance`: `diffX18 = 0.05`, `deltaQuote = -0.05 × 1000 = -50`. `balance.vQuoteBalance = -50`.
6. Subaccount A now carries a new, unintended debt of 50 quote units — identical in structure to the original bad debt that was just socialized — corrupting its health and potentially triggering a second liquidation. [7](#0-6)

    function socializeSubaccount(bytes32 subaccount, int128 insurance)
        external
        returns (int128)
    {
        require(msg.sender == address(_clearinghouse), ERR_UNAUTHORIZED);

        uint32[] memory _productIds = getProductIds();
        for (uint128 i = 0; i < _productIds.length; ++i) {
            uint32 productId = _productIds[i];
            (State memory state, Balance memory balance) = getStateAndBalance(
                productId,
                subaccount
            );
            if (balance.vQuoteBalance < 0) {
                int128 insuranceCover = MathHelper.min(
                    insurance,
                    -balance.vQuoteBalance
                );
                insurance -= insuranceCover;
                balance.vQuoteBalance += insuranceCover;
                state.availableSettle += insuranceCover;

                // actually socialize if still not enough
                if (balance.vQuoteBalance < 0) {
                    // socialize across all other participants
                    int128 fundingPerShare = -balance.vQuoteBalance.div(
                        state.openInterest
                    );
                    state.cumulativeFundingLongX18 += fundingPerShare;
                    state.cumulativeFundingShortX18 -= fundingPerShare;
                    balance.vQuoteBalance = 0;
                }
                _setState(productId, state);
                _setBalanceAndUpdateBitmap(productId, subaccount, balance);
            }
        }
        return insurance;
    }
