### Title
`SwapAllowlistExtension` Gates the Router Address Instead of the Originating User, Enabling Full Allowlist Bypass — (`metric-periphery/contracts/extensions/SwapAllowlistExtension.sol`)

---

### Summary

`SwapAllowlistExtension.beforeSwap` checks the `sender` argument passed by the pool, which is always `msg.sender` of `MetricOmmPool.swap()`. When a user routes through `MetricOmmSimpleRouter`, that `msg.sender` is the router contract, not the originating EOA. A pool admin who adds the router to the allowlist (the only way to let allowlisted users trade via the router) simultaneously opens the pool to every user on the network, completely defeating the allowlist.

---

### Finding Description

The call chain is:

```
EOA (not allowlisted)
  → MetricOmmSimpleRouter.exactInputSingle()
      → pool.swap(recipient, ..., extensionData)   // msg.sender = router
          → _beforeSwap(msg.sender=router, ...)
              → SwapAllowlistExtension.beforeSwap(sender=router, ...)
                  → allowedSwapper[pool][router]   // checks router, not EOA
```

**`MetricOmmPool.swap`** passes `msg.sender` as `sender` to `_beforeSwap`:

```solidity
// MetricOmmPool.sol line 230-240
_beforeSwap(
    msg.sender,   // ← router address when called via router
    recipient,
    ...
);
```

**`ExtensionCalling._beforeSwap`** forwards that value unchanged to the extension:

```solidity
// ExtensionCalling.sol line 160-176
_callExtensionsInOrder(
    BEFORE_SWAP_ORDER,
    abi.encodeCall(IMetricOmmExtensions.beforeSwap, (sender, ...))
);
```

**`SwapAllowlistExtension.beforeSwap`** then checks `allowedSwapper[pool][sender]` where `sender` is the router:

```solidity
// SwapAllowlistExtension.sol line 37-39
if (!allowAllSwappers[msg.sender] && !allowedSwapper[msg.sender][sender]) {
    revert IMetricOmmPoolActions.NotAllowedToSwap();
}
```

For allowlisted users to be able to use the router at all, the pool admin must add the router's address to `allowedSwapper[pool][router]`. Once that entry exists, **any** EOA — allowlisted or not — can call `router.exactInputSingle()` and the extension will pass, because it sees `sender = router` which is on the allowlist. The original EOA identity is never inspected.

The same bypass applies to `exactInput`, `exactOutputSingle`, and `exactOutput` on `MetricOmmSimpleRouter`, since all of them call `pool.swap()` with `msg.sender = router`.

---

### Impact Explanation

**Critical / High.** The `SwapAllowlistExtension` is the production mechanism for curated pools — pools restricted to KYC'd counterparties, institutional participants, or specific protocol integrations. A complete bypass means:

- Any unprivileged user can trade on a pool that was designed to be closed.
- Protocol fees accrue from unauthorized trades, and LP positions are exposed to toxic flow the allowlist was meant to exclude.
- The pool admin has no on-chain recourse short of removing the router from the allowlist entirely, which then breaks the UX for legitimate allowlisted users (they can no longer use the router either).

This is a direct loss of the curation guarantee the pool admin paid to configure and the LP depositors relied upon.

---

### Likelihood Explanation

**High.** The `MetricOmmSimpleRouter` is the primary user-facing swap entrypoint documented and deployed by the protocol. Any pool admin who wants allowlisted users to have a normal swap UX must add the router to the allowlist. The bypass requires no special knowledge — any user who reads the allowlist mapping, sees the router address is present, and calls the router directly can exploit it. No privileged access, no malicious setup, and no non-standard tokens are required.

---

### Recommendation

The extension must gate the **originating user**, not the immediate caller of `pool.swap()`. Two complementary fixes:

1. **Pass `tx.origin` or a forwarded-caller field.** The pool could expose a separate `senderOverride` field in `extensionData` that the router populates with `msg.sender` (the EOA), and the extension verifies it against the allowlist. This requires the router to cooperate, but the router is a trusted periphery contract.

2. **Preferred: check `tx.origin` in the extension.** Replace the `sender` check with `tx.origin` for EOA-only pools, or require the router to sign/encode the real caller in `extensionData` and verify it in the extension.

3. **Structural fix:** The pool could record the original `tx.origin` in transient storage at the start of `swap()` and expose it to extensions, so extensions always see the true initiator regardless of intermediaries.

---
