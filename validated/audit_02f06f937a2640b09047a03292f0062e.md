### Title
ML-DSA-65 Gas Key Fee Domain Mismatch: Wire Length Used Where Trie-ID Length Required in Host Function Path — (`runtime/near-vm-runner/src/logic/logic.rs`, `runtime/near-vm-runner/src/wasmtime_runner/logic.rs`, `runtime/runtime/src/config.rs`)

### Summary

The nearcore analog of the "wrong source" bug is a fee-domain invariant violation introduced with the `PostQuantumSignatures` feature. For ML-DSA-65 gas key operations, the **exec fee** in the host-function path is computed from the wire-format public key length (1953 bytes) instead of the on-trie identifier length (33 bytes), overcharging by ~1920 × `gas_key_byte` per nonce. Simultaneously, the **send fee** in the transaction path uses `trie_id_len()` (33) where the wire length (1953) is required, undercharging by the same factor. The two paths use opposite wrong "sources" for the same quantity, breaking the fee-parity invariant that must hold across both entry points.

### Finding Description

The protocol design establishes two distinct length domains for ML-DSA-65 access keys:

- **Wire length** (`PublicKey::len()` = 1953): the borsh-encoded full pubkey, carried in transactions and WASM memory.
- **Trie-ID length** (`PublicKey::trie_id_len()` = 33): the SHA3-256 hash form actually written to the trie. [1](#0-0) 

The `gas_key_add_key_exec_fee` and `gas_key_transfer_exec_fee` helpers are documented to price the **trie** read/write cost, so they must receive `trie_id_len()`: [2](#0-1) 

**Transaction path (correct for exec, wrong for send):** `permission_exec_fees` correctly passes `public_key.trie_id_len()` to `gas_key_add_key_exec_fee`: [3](#0-2) 

But `total_send_fees` passes `action.public_key.trie_id_len()` to `gas_key_transfer_send_fee`, which is documented as "what the sender sees" (wire length). For ML-DSA-65 this undercharges the send fee by 1920 × `gas_key_byte_send`: [4](#0-3) 

**Host-function path (correct for send, wrong for exec):** `promise_batch_action_add_gas_key_with_full_access` passes the raw `public_key_len as usize` (wire length = 1953) to `gas_key_add_key_exec_fee`, which should receive `trie_id_len()` = 33: [5](#0-4) 

The same error appears in `promise_batch_action_add_gas_key_with_function_call`: [6](#0-5) 

And in `promise_batch_action_transfer_to_gas_key`: [7](#0-6) 

The wasmtime runner duplicates the same errors: [8](#0-7) [9](#0-8) 

The design document explicitly flags this as a contract that all fee paths must respect: [10](#0-9) 

### Impact Explanation

**Host-function path overcharge (exec fee):** For `AddKey` with `num_nonces` nonces and an ML-DSA-65 key, the exec fee overcharge is:

```
Δ = (1953 − 33) × gas_key_byte_exec × num_nonces
  = 1920 × 101_435_400 × num_nonces
  ≈ 194.8 Ggas × num_nonces
```

With `num_nonces = 4` (a typical value), the overcharge is ~779 Ggas — 78% of the 1 Tgas block limit. A contract that prepays 300 Tgas for a function call and then schedules `promise_batch_action_add_gas_key_with_full_access` with an ML-DSA-65 key and moderate `num_nonces` will exhaust its gas budget and fail, making ML-DSA-65 gas keys effectively unusable from contracts.

**Transaction path undercharge (send fee):** A user submitting `TransferToGasKey` or `WithdrawFromGasKey` with an ML-DSA-65 key pays ~1920 × `gas_key_byte_send` ≈ 114 Ggas less send fee than the wire size warrants, allowing below-cost action submission.

### Likelihood Explanation

The bug is reachable by any unprivileged user after `PostQuantumSignatures` activates (protocol version 85). No special role is required: any account can hold an ML-DSA-65 access key, and any deployed contract can call the affected host functions. The divergence is deterministic and reproducible with any ML-DSA-65 public key.

### Recommendation

1. In `promise_batch_action_add_gas_key_with_full_access` and `promise_batch_action_add_gas_key_with_function_call`, replace `public_key_len as usize` with `public_key.decode()?.trie_id_len()` when calling `gas_key_add_key_exec_fee`.
2. In `promise_batch_action_transfer_to_gas_key` (both logic.rs and wasmtime_runner/logic.rs), replace `public_key_len as usize` with `public_key.decode()?.trie_id_len()` when calling `gas_key_transfer_exec_fee`.
3. In `total_send_fees` for `TransferToGasKey` and `WithdrawFromGasKey`, replace `action.public_key.trie_id_len()` with `action.public_key.len()` when calling `gas_key_transfer_send_fee`, since the send fee prices the wire-format size.
4. Add a fee-parity test for ML-DSA-65 keys analogous to `test_gas_key_fee_parity` (which currently only tests ED25519 keys). [11](#0-10) 

### Proof of Concept

```
ML-DSA-65 pubkey wire length:  1 + 1952 = 1953 bytes  (PublicKey::len())
ML-DSA-65 trie-id length:      1 +   32 =   33 bytes  (PublicKey::trie_id_len())
Divergence:                              1920 bytes

gas_key_byte exec fee (test config): 101_435_400 gas/byte

Host-fn exec overcharge per nonce: 1920 × 101_435_400 = 194,755,968,000 gas ≈ 194.8 Ggas
With num_nonces = 4:                                                       ≈ 779 Ggas

Transaction send undercharge:      1920 × 59_357_464  = 113,966,331,648 gas ≈ 114 Ggas
```

A contract calling `promise_batch_action_add_gas_key_with_full_access` with an ML-DSA-65 key and `num_nonces = 4` is charged ~779 Ggas more than a transaction performing the identical `AddKey` action — breaking the fee-parity invariant documented in `test_gas_key_fee_parity` and making ML-DSA-65 gas keys non-functional from within contracts.

    /// Length, in bytes, of the on-trie identifier for an access-key
    /// entry owned by this public key. For ed25519/secp256k1 this matches
    /// `len()`; for ML-DSA-65 the trie stores a SHA3-256 hash (33 bytes
    /// including the type tag), not the 1953-byte borsh-encoded pubkey.
    /// Used by storage-fee calculations on the runtime side; cheap to call
    /// (no hashing) - for ML-DSA-65 this returns the size of the digest
    /// form without actually hashing the pubkey.
    pub fn trie_id_len(&self) -> usize {
        match self {
            Self::ED25519(_) => 1 + ed25519_dalek::PUBLIC_KEY_LENGTH,
            Self::SECP256K1(_) => 1 + 64,
            Self::MLDSA65(_) => 1 + ML_DSA_65_HASH_LENGTH,
        }

/// Exec fee when adding a gas key with `num_nonces` nonces, split into base
/// and per-byte components. Each nonce writes a trie entry of
/// (access_key_key_len + NonceIndex) key bytes and NONCE_VALUE_LEN value bytes.
pub fn gas_key_add_key_exec_fee(
    cfg: &RuntimeFeesConfig,
    account_id_len: usize,
    public_key_len: usize,
    num_nonces: NonceIndex,
) -> GasKeyAddFee {
    let num_nonces = num_nonces as u64;
    let base =
        cfg.fee(ActionCosts::gas_key_nonce_write_base).exec_fee().checked_mul(num_nonces).unwrap();
    let nonce_key_len =
        access_key_key_len(account_id_len, public_key_len) + std::mem::size_of::<NonceIndex>();
    let per_byte = cfg
        .fee(ActionCosts::gas_key_byte)
        .exec_fee()
        .checked_mul((nonce_key_len + AccessKey::NONCE_VALUE_LEN) as u64)
        .unwrap()
        .checked_mul(num_nonces)
        .unwrap();
    GasKeyAddFee { base, per_byte }
