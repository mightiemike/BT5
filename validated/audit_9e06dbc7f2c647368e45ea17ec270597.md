### Title
Silent Bonus Forfeit with Permanent Checkpoint Advance on Insufficient Pool During Redelegation Slash - (File: x/tieredrewards/keeper/slash.go)

### Summary

In `slashRedelegationPosition`, when `processEventsAndClaimBonus` returns `ErrInsufficientBonusPool`, the error is deliberately swallowed to avoid a chain halt. However, `processEventsAndClaimBonus` mutates the in-memory `pos` (advancing `LastBonusAccrual`, `LastEventSeq`, `LastKnownBonded`) and decrements event reference counts in the store **before** the pool balance check. Because execution continues after the swallowed error and `setPositionWithState` is called with the already-mutated `pos`, the advanced checkpoints are persisted and the event reference counts are permanently decremented — all without any bonus coins being sent to the owner. The accrued bonus is permanently and irrecoverably lost.

### Finding Description

`processEventsAndClaimBonus` in `x/tieredrewards/keeper/claim_rewards.go` performs state mutations before the pool sufficiency check:

1. Inside the event loop, `pos.UpdateLastEventSeq(entry.Seq)` and `k.decrementEventRefCount(ctx, valAddr, entry.Seq)` are called for every event processed.
2. After the loop, `applyBonusAccrualCheckpoint(&pos.Position, blockTime)` and `pos.UpdateLastKnownBonded(bonded)` advance the position's checkpoints.
3. Only **after** all of the above does `k.sufficientBonusPoolBalance(ctx, bonusCoins)` run. [1](#0-0) 

When the pool check fails with `ErrInsufficientBonusPool`, the function returns an error — but `pos` has already been mutated in-memory and `decrementEventRefCount` has already written to the store.

In `slashRedelegationPosition`, this error is explicitly swallowed:

```go
if _, err := k.processEventsAndClaimBonus(ctx, &pos); err != nil {
    if errors.Is(err, types.ErrInsufficientBonusPool) {
        k.logger(ctx).Error("insufficient bonus pool during redelegation slash", ...)
    } else {
        return err
    }
}
``` [2](#0-1) 

Execution then falls through to `setPositionWithState`, which persists the already-mutated `pos` — including the advanced `LastBonusAccrual`, `LastEventSeq`, and `LastKnownBonded`: [3](#0-2) 

The result is:
- **Event reference counts** are permanently decremented in the store (store writes inside the loop are not rolled back because the hook returns `nil`).
- **Position checkpoints** are advanced and persisted via `setPositionWithState`.
- **No bonus coins** are transferred to the owner.

The owner has no retry path: the checkpoints have moved past the slash point, and the events have been garbage-collected.

### Impact Explanation

A tier position owner who has redelegated their position loses all accrued bonus rewards up to the slash block time whenever a validator slash fires while the bonus pool is insufficient. The loss is permanent — the position's `LastBonusAccrual` is advanced to `blockTime` and persisted, so a subsequent `MsgClaimTierRewards` will compute zero bonus for the already-consumed period. The corrupted value is `affiliateFeesAccrued`-equivalent: `pos.LastBonusAccrual`, `pos.LastEventSeq`, `pos.LastKnownBonded`, and the on-chain `ValidatorEvent` reference counts.

### Likelihood Explanation

The trigger requires two concurrent conditions: (1) a tier position is in an active redelegation (i.e., `TierRedelegate` was called and the redelegation entry has not yet matured), and (2) the destination validator is slashed while the bonus pool balance is below the accrued bonus for that position. Condition (1) is a normal user action. Condition (2) is realistic whenever the pool is underfunded — governance controls pool funding, and there is no on-chain guarantee the pool is always sufficient. A validator downtime slash (the most common slash type) is sufficient to trigger this path.

### Recommendation

Reorder operations in `processEventsAndClaimBonus` so that the pool sufficiency check occurs **before** any state mutations. Specifically:

1. Compute `totalBonus` by walking events (read-only pass).
2. Check `sufficientBonusPoolBalance` before calling `decrementEventRefCount` or advancing any checkpoint.
3. Only if the pool check passes, apply `decrementEventRefCount`, advance checkpoints, and send coins.

This ensures that if the pool is insufficient, `pos` is returned unmodified and no store writes have occurred, preserving the user's ability to retry after the pool is replenished — consistent with the behavior documented for user-driven paths. [4](#0-3) 

### Proof of Concept

1. User calls `MsgLockTier` to create a tier position on validator V1.
2. User calls `MsgTierRedelegate` to redelegate the position to validator V2. A `RedelegationMapping[unbondingId → positionId]` entry is created.
3. Time passes; bonus accrues (e.g., 30 days at 4% APY on a large position).
4. The bonus pool is drained to zero (e.g., by other users claiming rewards, or simply never funded sufficiently).
5. Validator V2 is slashed (e.g., downtime). The Cosmos SDK staking module calls `BeforeRedelegationSlashed(ctx, unbondingId, sharesToUnbond)`.
6. `slashRedelegationPosition` is invoked. `processEventsAndClaimBonus` runs, advances `pos.LastBonusAccrual` to `blockTime`, decrements event reference counts, then returns `ErrInsufficientBonusPool`.
7. The error is swallowed. `setPositionWithState` persists the advanced checkpoints.
8. The user later calls `MsgClaimTierRewards`. `processEventsAndClaimBonus` computes zero bonus because `LastBonusAccrual` already equals the current block time. The accrued bonus is permanently lost. [5](#0-4) [6](#0-5)

func (k Keeper) slashRedelegationPosition(ctx context.Context, unbondingId uint64, sharesToUnbond math.LegacyDec) error {
	positionId, err := k.getRedelegationMapping(ctx, unbondingId)
	if errors.Is(err, collections.ErrNotFound) {
		return nil
	}
	if err != nil {
		return err
	}

	pos, err := k.getPositionState(ctx, positionId)
	if errors.Is(err, types.ErrPositionNotFound) {
		k.logger(ctx).Error("position not found during redelegation slash",
			"position_id", positionId,
			"unbonding_id", unbondingId,
			"error", err.Error(),
		)
		return nil
	}
	if err != nil {
		return err
	}

	if !pos.IsDelegated() {
		// Defensive
		k.logger(ctx).Error("delegation missing during BeforeRedelegationSlashed",
			"position_id", positionId,
			"unbonding_id", unbondingId,
			"shares_to_unbond", sharesToUnbond.String(),
		)
		return nil
	}

	dstValStr := pos.Delegation.ValidatorAddress

	// Settle bonus against PRE-slash shares.
	if _, err := k.processEventsAndClaimBonus(ctx, &pos); err != nil {
		// Deliberately forgo bonus rewards if pool is insufficient to prevent chain halt.
		if errors.Is(err, types.ErrInsufficientBonusPool) {
			k.logger(ctx).Error("insufficient bonus pool during redelegation slash",
				"position_id", pos.Id,
				"error", err.Error(),
			)
		} else {
			return err
		}
	}

	fullSlash := sharesToUnbond.GTE(pos.Delegation.Shares)

	if fullSlash {
		pos.Delegation = nil
		pos.ClearBonusCheckpoints()
		return k.setPositionWithState(ctx, pos, &ValidatorTransition{PreviousAddress: dstValStr})
	}
	// In-memory only: the persisted Position carries no share count, and the
	// live delegation will reflect the post-Unbond shares on the next read.
	// Update the local copy so any follow-up logic in this call sees consistent state.
	pos.Delegation.Shares = pos.Delegation.Shares.Sub(sharesToUnbond)
	return k.setPositionWithState(ctx, pos, nil)
```
