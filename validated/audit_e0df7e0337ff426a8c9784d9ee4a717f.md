### Title
Interface Method Name Mismatch: `IFEndpoint.setPriceX18()` Does Not Exist in `Endpoint` — (`File: core/contracts/interfaces/IFEndpoint.sol`)

---

### Summary

`IFEndpoint` declares `setPriceX18(uint32, int128)`, but `Endpoint.sol` implements `setInitialPrice(uint32, int128)` instead. Any contract that calls `IFEndpoint(endpoint).setPriceX18(...)` will always revert at runtime because no matching function selector exists in `Endpoint`. The access-control guard on `setInitialPrice` confirms the engines are the intended callers, meaning new-product price initialization through this interface path is permanently broken.

---

### Finding Description

`IFEndpoint.sol` declares a single method:

```solidity
interface IFEndpoint {
    function setPriceX18(uint32 productId, int128 priceX18) external;
}
``` [1](#0-0) 

`Endpoint.sol` does **not** implement `setPriceX18`. The corresponding function is named `setInitialPrice`:

```solidity
function setInitialPrice(uint32 productId, int128 initialPriceX18) external {
    require(
        msg.sender == address(spotEngine) ||
            msg.sender == address(perpEngine),
        ERR_UNAUTHORIZED
    );
    require(priceX18[productId] == 0, ERR_UNAUTHORIZED);
    priceX18[productId] = initialPriceX18;
}
``` [2](#0-1) 

The access-control guard (`msg.sender == address(spotEngine) || msg.sender == address(perpEngine)`) proves that `SpotEngine` and `PerpEngine` are the intended callers of this function. The `IFEndpoint` interface exists precisely so those engines can call back into `Endpoint` to register an initial price when a new product is added. Because the interface method name (`setPriceX18`) does not match the actual implementation name (`setInitialPrice`), any call made via `IFEndpoint(endpoint).setPriceX18(...)` will produce a runtime revert — the EVM finds no matching 4-byte selector in `Endpoint`.

The Solidity compiler does **not** catch this: it only verifies that the interface declares the method, not that the target contract actually implements it. The mismatch is invisible at compile time and only surfaces at runtime.

---

### Impact Explanation

When `SpotEngine` or `PerpEngine` adds a new product and attempts to register its initial price by calling `IFEndpoint(endpoint).setPriceX18(...)`, the call reverts unconditionally. As a result:

- `priceX18[productId]` remains `0` for the new product.
- `Endpoint.getPriceX18()` enforces `require(_priceX18 != 0, ERR_INVALID_PRODUCT)`, so every subsequent price query for that product reverts. [3](#0-2) 

- `Clearinghouse._getPriceX18()` calls `IEndpoint(getEndpoint()).getPriceX18(productId)`, so health checks, liquidations, and collateral operations for the affected product all fail. [4](#0-3) 

- `checkMinDeposit` uses `_getPriceX18`, so deposits into the new product also fail. [5](#0-4) 

Any product whose initial price was never set is permanently unusable for trading, deposits, and liquidations.

---

### Likelihood Explanation

This is triggered every time a new spot or perp product is registered. It requires no privileged attacker — the failure occurs automatically during normal protocol operation (product onboarding). The broken code path is reachable by any user who interacts with a newly listed product.

---

### Recommendation

Rename `setPriceX18` in `IFEndpoint` to `setInitialPrice` to match the actual implementation:

```solidity
interface IFEndpoint {
    function setInitialPrice(uint32 productId, int128 initialPriceX18) external;
}
```

Additionally, enforce interface compliance by having `Endpoint` declare `is IFEndpoint` (or equivalent), so the compiler catches future name divergences.

---

1. `IFEndpoint` declares `setPriceX18`. [6](#0-5)

3. The access guard confirms `SpotEngine`/`PerpEngine` are the intended callers via this interface. [8](#0-7) 

4. Any call to `IFEndpoint(endpoint).setPriceX18(...)` dispatches selector `keccak256("setPriceX18(uint32,int128)")[0:4]`, which has no match in `Endpoint`, causing an unconditional revert.

5. `priceX18[productId]` stays `0`; `getPriceX18` reverts with `ERR_INVALID_PRODUCT` for every subsequent query on that product. [9](#0-8)
