### Title
Insufficient Bonus Pool Permanently Blocks All Exit Paths for Tier Positions - (`x/tieredrewards/keeper/claim_rewards.go`)

### Summary

Every exit path from the `x/tieredrewards` module (`MsgTierUndelegate`, `MsgExitTierWithDelegation`, `MsgClearPosition`) mandatorily calls `claimRewards()` before performing the delegation transfer or unbonding. `claimRewards()` calls `processEventsAndClaimBonus()`, which hard-fails with `ErrInsufficientBonusPool` if the rewards pool cannot cover accrued bonus. Because `MsgWithdrawFromTier` — the only exit handler that skips `claimRewards()` — requires the position to already be undelegated, and undelegation itself is blocked by the same check, any user who has accrued non-zero bonus while the pool is empty is permanently unable to exit their tier position and recover their staked tokens.

### Finding Description

`processEventsAndClaimBonus()` computes accrued bonus for a position and then calls `sufficientBonusPoolBalance()`:

```go
if err := k.sufficientBonusPoolBalance(ctx, bonusCoins); err != nil {
    return nil, err
}
```

`sufficientBonusPoolBalance()` returns `ErrInsufficientBonusPool` whenever the pool balance is below the computed bonus amount and the bonus is non-zero:

```go
if !poolBalance.IsAllGTE(bonus) {
    return errorsmod.Wrapf(types.ErrInsufficientBonusPool, ...)
}
```

This error propagates atomically through `claimRewards()` and blocks every exit message:

| Exit message | Calls `claimRewards()`? | Blocked when pool empty? |
|---|---|---|
| `MsgTierUndelegate` | Yes (line 166) | Yes |
| `MsgExitTierWithDelegation` | Yes (line 532) | Yes |
| `MsgClearPosition` (exit triggered) | Yes (line 406) | Yes |
| `MsgWithdrawFromTier` | No | Requires undelegated state first |

`MsgWithdrawFromTier` is the only handler that does not call `claimRewards()`, but `validateWithdrawFromTier()` enforces `!pos.IsDelegated()`. Reaching that state requires `MsgTierUndelegate` first, which is itself blocked. There is no bypass.

Bonus accrues continuously for any position delegated to a bonded validator (`shares × tokensPerShare × bonusApy × duration / SecondsPerYear`). After any non-trivial lock period, `totalBonus` will be non-zero, making the pool check active on every exit attempt.

The `BeforeRedelegationSlashed` hook deliberately forfeits bonus silently when the pool is insufficient to avoid a chain halt, but no equivalent silent-forfeit path exists for user-initiated exits.

### Impact Explanation

When the rewards pool is empty and a position has accrued non-zero bonus, the position owner's staked tokens (the full delegation held at the position's per-position delegator address) are permanently locked. The owner cannot undelegate, exit with delegation, or cancel the exit. The staked tokens remain at `pos.DelegatorAddress` indefinitely with no reachable code path to recover them. This is a direct loss of user funds, not merely a temporary delay.

### Likelihood Explanation

The pool is funded by the `BeginBlocker` top-up (only when `TargetBaseRewardsRate > 0` and the fee collector has a surplus) and by manual governance funding. If `TargetBaseRewardsRate` is set to zero, if the fee collector is consistently empty, or if the pool is drained by a burst of reward claims before it can be replenished, the pool reaches zero. Because bonus accrues continuously, any position that has been active for even a short period will have non-zero pending bonus, making the pool check active on the very next exit attempt. The scenario is reachable without any privileged action: a user simply needs to hold a position while the pool is empty.

### Recommendation

Mirror the pattern already used in `BeforeRedelegationSlashed`: allow exit-critical paths (`MsgTierUndelegate`, `MsgExitTierWithDelegation`) to proceed and forfeit the bonus silently when the pool is insufficient, rather than reverting the entire transaction. The bonus checkpoint (`LastBonusAccrual`, `LastEventSeq`) should still be advanced so the forfeited window is not re-claimed later. Alternatively, add a dedicated "emergency exit" variant that skips bonus settlement entirely and only transfers the delegation/principal.

### Proof of Concept

1. Alice creates a tier position via `MsgLockTier` with `amount = MinLockAmount`.
2. Time advances; bonus accrues (`shares × tokensPerShare × bonusApy × duration > 0`).
3. The rewards pool is drained to zero (naturally or by other claimants).
4. Alice triggers exit via `MsgTriggerExitFromTier` (no `claimRewards()` call here — succeeds).
5. Exit commitment elapses.
6. Alice submits `MsgTierUndelegate`:
   - `validateUndelegatePosition` passes (exit triggered, elapsed, delegated).
   - `claimRewards()` is called → `processEventsAndClaimBonus()` → `sufficientBonusPoolBalance()` → `ErrInsufficientBonusPool` → transaction reverts.
7. Alice submits `MsgExitTierWithDelegation` with `amount = positionAmount`:
   - `validateExitTierWithDelegation` passes.
   - `claimRewards()` is called → same revert.
8. Alice submits `MsgWithdrawFromTier`:
   - `validateWithdrawFromTier` fails with `ErrPositionDelegated` because the position is still delegated.
9. Alice's staked tokens are permanently locked at `pos.DelegatorAddress` with no reachable exit.

func (k Keeper) sufficientBonusPoolBalance(ctx context.Context, bonus sdk.Coins) error {
	if bonus.IsZero() {
		return nil
	}

	poolAddr := k.accountKeeper.GetModuleAddress(types.RewardsPoolName)
	poolBalance := k.bankKeeper.GetAllBalances(ctx, poolAddr)
	if !poolBalance.IsAllGTE(bonus) {
		return errorsmod.Wrapf(types.ErrInsufficientBonusPool,
			"bonus: %s, pool balance: %s",
			bonus.String(), poolBalance.String())
	}

	return nil

func (k Keeper) validateWithdrawFromTier(ctx context.Context, pos types.PositionState, owner string) error {
	if !pos.IsOwner(owner) {
		return types.ErrNotPositionOwner
	}

	if !pos.HasTriggeredExit() {
		return types.ErrExitNotTriggered
	}

	sdkCtx := sdk.UnwrapSDKContext(ctx)
	if !pos.CompletedExitLockDuration(sdkCtx.BlockTime()) {
		return types.ErrExitLockDurationNotReached
	}

	if pos.IsDelegated() {
		return types.ErrPositionDelegated
	}

	isUnbonding, err := k.isUnbonding(ctx, pos.DelegatorAddress)
	if err != nil {
		return err
	}
	if isUnbonding {
		return types.ErrPositionUnbonding
	}

	return nil
}
