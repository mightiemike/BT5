Audit Report

## Title
`SwapAllowlistExtension` Checks Router Address Instead of Originating User, Enabling Full Allowlist Bypass via `MetricOmmSimpleRouter` — (File: `metric-periphery/contracts/extensions/SwapAllowlistExtension.sol`)

## Summary
`SwapAllowlistExtension.beforeSwap` gates swaps by checking `sender` — the `msg.sender` of `pool.swap()` — against the per-pool allowlist. When users route through `MetricOmmSimpleRouter`, `sender` resolves to the router's address rather than the actual user. A pool admin who allowlists the router to support authorized users inadvertently grants swap access to every caller of the router, fully breaking the pool's access-control invariant.

## Finding Description
In `SwapAllowlistExtension.beforeSwap`, the check is:

```solidity
// SwapAllowlistExtension.sol L37
if (!allowAllSwappers[msg.sender] && !allowedSwapper[msg.sender][sender]) {
    revert IMetricOmmPoolActions.NotAllowedToSwap();
}
```

`msg.sender` here is the pool (the pool calls the extension), and `sender` is the value passed from `MetricOmmPool.swap()`:

```solidity
// MetricOmmPool.sol L230-231
_beforeSwap(
    msg.sender,   // ← becomes `sender` in the extension
```

`ExtensionCalling._beforeSwap` encodes it verbatim into the call to the extension (L160-176). When a user calls any router entry point — `exactInputSingle`, `exactInput`, `exactOutputSingle`, or `exactOutput` — the router calls `pool.swap()` directly:

```solidity
// MetricOmmSimpleRouter.sol L72-80 (exactInputSingle)
(int128 amount0Delta, int128 amount1Delta) = IMetricOmmPoolActions(params.pool)
    .swap(params.recipient, params.zeroForOne, ...);
```

This makes `msg.sender` of `pool.swap()` equal to the router's address. The extension therefore evaluates `allowedSwapper[pool][router]`, not `allowedSwapper[pool][actualUser]`. Once the router is allowlisted, the check passes for every caller of the router regardless of individual authorization.

No existing guard compensates for this: `extensionData` is passed through but the extension does not inspect it, and there is no trusted-forwarder or signature mechanism to recover the originating user's identity.

## Impact Explanation
Any user not on the allowlist can bypass the swap gate on a curated pool by routing through `MetricOmmSimpleRouter`. `SwapAllowlistExtension` is the sole mechanism for restricting swap access on a pool; bypassing it means the pool's access-control invariant is fully broken. Pools intended for restricted access (institutional, KYC-gated, or whitelist-only deployments) become open to the public the moment the admin enables router-mediated swaps for their authorized users. This constitutes broken core pool functionality with direct fund-flow consequences — an admin-boundary break reachable by any unprivileged user.

## Likelihood Explanation
The bypass requires the pool admin to allowlist the router — a natural and expected action for any pool that wants to support the standard periphery UX. The admin has no way to simultaneously allow router-mediated swaps for authorized users and block unauthorized users, because the extension cannot see past the router to the originating caller. Once the router is allowlisted, the trigger is trivially reachable by any unprivileged user who calls any of the four router entry points.

## Recommendation
The extension must gate the economically relevant actor, not the immediate caller. Two viable approaches:

1. **Pass actual user identity through `extensionData`**: Require the router to embed the originating user's address (with a signature or trusted-forwarder pattern) in `extensionData`, and have the extension verify and check that identity instead of `sender`.
2. **Document the incompatibility explicitly in code**: If the design cannot be changed, add a NatSpec warning stating that allowlisting the router is equivalent to `setAllowAllSwappers(pool, true)` and that per-user gating is only enforceable on direct pool calls — and consider reverting in `beforeSwap` if `sender` is a known router address.

## Proof of Concept
1. Pool admin deploys a pool with `SwapAllowlistExtension` configured.
2. Admin calls `setAllowedToSwap(pool, userA, true)` — only `userA` is authorized.
3. Admin calls `setAllowedToSwap(pool, router, true)` — to let `userA` use `MetricOmmSimpleRouter`.
4. `userB` (not allowlisted) calls `MetricOmmSimpleRouter.exactInputSingle(...)` targeting the pool.
5. The router calls `pool.swap(...)` with `msg.sender = router` (L72-80 of `MetricOmmSimpleRouter.sol`).
6. `MetricOmmPool.swap()` passes `msg.sender` (= router) as `sender` to `_beforeSwap` (L230-231 of `MetricOmmPool.sol`).
7. `ExtensionCalling._beforeSwap` encodes `sender = router` and calls the extension (L160-176 of `ExtensionCalling.sol`).
8. Extension evaluates `allowedSwapper[pool][router] == true` → check passes (L37 of `SwapAllowlistExtension.sol`).
9. `userB`'s swap executes successfully, bypassing the allowlist entirely.
