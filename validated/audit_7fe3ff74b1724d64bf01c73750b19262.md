Audit Report

## Title
`SwapAllowlistExtension.beforeSwap` Checks Router Address Instead of End-User, Breaking Per-User Allowlist Enforcement — (`metric-periphery/contracts/extensions/SwapAllowlistExtension.sol`)

## Summary
`SwapAllowlistExtension.beforeSwap` receives `sender` as the address that called `pool.swap()`. When a user swaps through `MetricOmmSimpleRouter`, the router is `msg.sender` at the pool, so `sender` is the router address, not the end-user. The allowlist check `allowedSwapper[pool][router]` therefore gates the router contract rather than the actual swapper, making per-user allowlist enforcement impossible for router-mediated swaps and creating a complete bypass when the router is allowlisted.

## Finding Description

**Call chain:**

`MetricOmmPool.swap` passes `msg.sender` as `sender` to `_beforeSwap`: [1](#0-0) 

`ExtensionCalling._beforeSwap` forwards `sender` unchanged to every configured extension: [2](#0-1) 

`SwapAllowlistExtension.beforeSwap` checks `allowedSwapper[msg.sender][sender]`, where `msg.sender` is the pool and `sender` is whoever called `pool.swap()`: [3](#0-2) 

`MetricOmmSimpleRouter.exactInputSingle` calls `pool.swap(...)` directly with no encoding of the original user into `extensionData`: [4](#0-3) 

The same pattern applies to `exactInput`, `exactOutputSingle`, and `exactOutput` — all call `pool.swap` directly, making the router `msg.sender` at the pool in every case. [5](#0-4) 

**Two broken invariants:**

1. **Allowlisted users blocked from router.** If the admin allowlists specific users but not the router, every router-mediated swap reverts with `NotAllowedToSwap` because `allowedSwapper[pool][router] == false`, even though the end-user is explicitly permitted.

2. **Complete bypass when router is allowlisted.** If the admin allowlists the router address to fix the above, `allowedSwapper[pool][router] == true` passes for every caller regardless of their individual allowlist status. Any non-allowlisted user can swap freely through the router.

No existing guard prevents this. The `extensionData` field is available but the router passes it through from the user without encoding the original caller's address, and the extension has no mechanism to decode a user identity from it. [6](#0-5) 

## Impact Explanation
This is a broken core pool functionality and admin-boundary break. A pool deploying `SwapAllowlistExtension` to restrict trading to specific counterparties (KYC, regulatory access control, private liquidity venue) cannot simultaneously allow the standard router and enforce per-user restrictions. In the bypass scenario, non-allowlisted users trade against LP positions in a pool explicitly configured to exclude them, exposing LPs to adverse selection they opted out of — a direct loss of LP principal protection above Sherlock thresholds.

## Likelihood Explanation
`MetricOmmSimpleRouter` is the primary supported periphery for swaps. Any pool admin who configures `SwapAllowlistExtension` and expects users to access the pool through the router will encounter this immediately. The broken-functionality path triggers on the first router swap attempt by an allowlisted user; the bypass path triggers as soon as the admin allowlists the router to fix that problem. No special attacker capability is required — any unprivileged user calling the public router entry points triggers both scenarios.

## Recommendation
The extension must identify the economic actor (the end-user), not the immediate caller of `pool.swap()`. The router should encode `msg.sender` into `extensionData` before calling the pool (e.g., `abi.encode(msg.sender)` prepended or appended), and `SwapAllowlistExtension.beforeSwap` should decode and check that address when `extensionData` is non-empty. Alternatively, the extension can detect whether `sender` is a known router and fall back to a user address embedded in the payload. The NatSpec invariant "Gates `swap` by swapper address, per pool" must hold across all supported entry points. [7](#0-6) 

## Proof of Concept

```
Setup:
  pool P configured with SwapAllowlistExtension E
  admin calls E.setAllowedToSwap(P, alice, true)   // alice is allowlisted
  bob is NOT allowlisted

Scenario A — broken functionality:
  alice calls router.exactInputSingle({pool: P, ...})
  → router calls P.swap(recipient, ...) with msg.sender = router
  → pool calls E.beforeSwap(router, ...)
  → allowedSwapper[P][router] == false → revert NotAllowedToSwap
  alice cannot use the router despite being allowlisted.

Scenario B — bypass:
  admin calls E.setAllowedToSwap(P, router, true)  // to fix Scenario A
  bob calls router.exactInputSingle({pool: P, ...})
  → router calls P.swap(recipient, ...) with msg.sender = router
  → pool calls E.beforeSwap(router, ...)
  → allowedSwapper[P][router] == true → passes
  bob swaps successfully despite not being allowlisted.
```

Foundry test: deploy pool with `SwapAllowlistExtension`, allowlist `alice`, call `router.exactInputSingle` as `alice` and assert revert; then allowlist the router and call as `bob` (not allowlisted) and assert success.

  function exactInput(ExactInputParams calldata params) external payable returns (uint256 amountOut) {
    _checkDeadline(params.deadline);
    _validatePath(params.tokens, params.pools, params.extensionDatas);

    uint256 last = params.pools.length - 1;
    int128 amount = MetricOmmSwapInputs.asAmountSpecifiedIn(params.amountIn);

    for (uint256 i = 0; i <= last; i++) {
      address pool = params.pools[i];
      bool zeroForOne = MetricOmmSwapPath.resolveZeroForOneBitmap(params.zeroForOneBitMap, i);

      _setNextCallbackContext(pool, CALLBACK_MODE_JUST_PAY, i == 0 ? msg.sender : address(this), params.tokens[i]);
      (int128 amount0Delta, int128 amount1Delta) = IMetricOmmPoolActions(pool)
        .swap(
          i == last ? params.recipient : address(this),
          zeroForOne,
          amount,
          MetricOmmSwapPath.openLimit(zeroForOne),
          "",
          params.extensionDatas[i]
        );

      int128 amountInActual = MetricOmmSwapResults.extractAmountIn(zeroForOne, amount0Delta, amount1Delta);
      if (amountInActual < amount) revert InvalidInputAmountAtHop(uint8(i), amountInActual, amount);

      amount = MetricOmmSwapResults.extractAmountOut(zeroForOne, amount0Delta, amount1Delta);
    }

    if (amount <= 0) revert InvalidSwapDeltas();
    amountOut = MetricOmmSwapInputs.int128ToUint128(amount);
    if (amountOut < params.amountOutMinimum) revert InsufficientOutput(amountOut, params.amountOutMinimum);

    _clearExpectedCallbackPool();
  }
