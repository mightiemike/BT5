### Title
`abi: None` vs `abi: Some(vec![])` in `BroadcastedDeclareV1Transaction` produces divergent `abi_length` (0 vs 2) in `simulateTransactions`/`estimateFee` fee estimation — (`crates/papyrus_common/src/deprecated_class_abi.rs`)

---

### Summary

`calculate_deprecated_class_abi_length` has an explicit branch that returns `0` for `abi: None` and serializes the vector (yielding `2` for an empty `[]`) for `abi: Some(vec![])`. Because the RPC v0_8 `BroadcastedDeclareV1Transaction` path passes the caller-supplied `abi` field through without normalization, an unprivileged caller can choose either representation and receive a different `abi_length` — and therefore a different `code_size` and fee estimate — for semantically identical class content.

---

### Finding Description

**Step 1 — The divergence in `calculate_deprecated_class_abi_length`** [1](#0-0) 

```
abi: None        → early-return Ok(0)
abi: Some(vec![]) → serializes [] with PythonJsonFormatter → Ok(2)
```

The two-byte difference is the literal JSON encoding `[]`.

**Step 2 — The RPC type that carries the caller-controlled `abi`**

`apollo_starknet_client::writer::objects::transaction::DeprecatedContractClass` declares: [2](#0-1) 

`#[serde(default)]` means an absent key deserializes to `None`; an explicit `"abi": null` also deserializes to `None`; but `"abi": []` deserializes to `Some(vec![])`. All three are valid JSON that the RPC server will accept.

**Step 3 — The conversion passes `abi` through without normalization** [3](#0-2) 

`value.abi` is assigned directly to `starknet_api::deprecated_contract_class::ContractClass::abi` — no `unwrap_or_default()`, no canonicalization.

**Step 4 — `abi_length` is computed from the un-normalized value and fed into fee calculation** [4](#0-3) 

The resulting `abi_length` is stored in `ExecutableTransactionInput::DeclareV1` and consumed by `ClassInfo::code_size()`: [5](#0-4) 

`code_size` directly drives the gas/fee estimate returned by `simulateTransactions` / `estimateFee`.

---

### Impact Explanation

An unprivileged caller submitting a `BroadcastedDeclareV1Transaction` with `"abi": []` receives a fee estimate that is `2 × gas_price_per_byte` higher than the same class submitted with `"abi": null` (or with the `abi` key absent). The two payloads are semantically identical (empty ABI), but the RPC server returns different authoritative-looking fee values for them. This satisfies the allowed impact: **"High. RPC execution, fee estimation, tracing, simulation, or pending view returns an authoritative-looking wrong value."**

The magnitude per call is small (2 bytes), but the divergence is exact, deterministic, and fully attacker-controlled.

---

### Likelihood Explanation

The path requires no privileges. Any caller of `starknet_simulateTransactions` or `starknet_estimateFee` with a `BROADCASTED_DECLARE_TXN_V1` body can trigger it by toggling `"abi": null` vs `"abi": []`.

---

### Recommendation

Normalize the `abi` field before computing `abi_length`. The simplest fix is to treat `Some(vec![])` identically to `None` inside `calculate_deprecated_class_abi_length`, or to canonicalize in `user_deprecated_contract_class_to_sn_api` by mapping `Some(vec![])` → `None` (mirroring what `deprecated_contract_class.rs` already does with `unwrap_or_default` on the read path): [6](#0-5) 

Alternatively, change `calculate_deprecated_class_abi_length` to treat an empty `Some(vec![])` the same as `None` (return `0`), making the function canonical regardless of which representation the caller chose.

---

### Proof of Concept

```rust
use starknet_api::deprecated_contract_class::ContractClass;
use papyrus_common::deprecated_class_abi::calculate_deprecated_class_abi_length;

#[test]
fn abi_length_none_vs_empty_vec_must_be_equal() {
    let class_none = ContractClass { abi: None, ..Default::default() };
    let class_empty = ContractClass { abi: Some(vec![]), ..Default::default() };

    let len_none  = calculate_deprecated_class_abi_length(&class_none).unwrap();
    let len_empty = calculate_deprecated_class_abi_length(&class_empty).unwrap();

    // Currently fails: len_none == 0, len_empty == 2
    assert_eq!(len_none, len_empty,
        "abi: None and abi: Some(vec![]) must produce the same abi_length");
}
```

This test will fail on the current codebase, confirming the divergence.

pub fn calculate_deprecated_class_abi_length(
    deprecated_class: &starknet_api::deprecated_contract_class::ContractClass,
) -> Result<usize, serde_json::Error> {
    let Some(abi) = deprecated_class.abi.as_ref() else {
        return Ok(0);
    };
    let mut chars = vec![];
    abi.serialize(&mut serde_json::Serializer::with_formatter(&mut chars, PythonJsonFormatter))?;
    Ok(chars.len())
