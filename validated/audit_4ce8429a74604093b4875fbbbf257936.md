### Title
Minimum Burn Fee Floor Destroys 100% of NLP Value for Sub-$1 Burns — (`core/contracts/Clearinghouse.sol`)

---

### Summary

`Clearinghouse.burnNlp` applies a minimum burn fee of `ONE` ($1 in X18 fixed-point). When a user burns NLP whose total value is ≤ $1, the fee equals or exceeds the value, `quoteAmount` is clamped to zero, and the user's NLP is burned with zero quote returned. The NLP share is transferred to `N_ACCOUNT` while the user receives nothing.

---

### Finding Description

In `Clearinghouse.burnNlp`:

```solidity
int128 quoteAmount = nlpAmount.mul(oraclePriceX18);          // X18 fixed-point product
int128 burnFee = MathHelper.max(ONE, quoteAmount / 1000);    // floor = 1e18 = $1
quoteAmount = MathHelper.max(0, quoteAmount - burnFee);      // clamped to 0 if value ≤ $1
``` [1](#0-0) 

`ONE = 10**18` is the X18 representation of $1. [2](#0-1) 

`nlpAmount.mul(oraclePriceX18)` uses `MathSD21x18.mul`, which computes `(x * y) / ONE_X18`. [3](#0-2) 

When `quoteAmount < ONE` (NLP value < $1):
- `quoteAmount / 1000 < ONE/1000 < ONE` → `burnFee = ONE`
- `quoteAmount - ONE < 0` → `quoteAmount = max(0, negative) = 0`

The NLP is then burned unconditionally, but the quote branch is skipped entirely:

```solidity
spotEngine.updateBalance(NLP_PRODUCT_ID, txn.sender, -nlpAmount);  // NLP burned
spotEngine.updateBalance(NLP_PRODUCT_ID, N_ACCOUNT, nlpAmount);    // goes to N_ACCOUNT

if (quoteAmount > 0) {                                              // false → skipped
    spotEngine.updateBalance(QUOTE_PRODUCT_ID, txn.sender, quoteAmount);
    ...
}
``` [4](#0-3) 

The user's NLP share is transferred to `N_ACCOUNT` while the user receives zero quote. There is no minimum NLP amount guard anywhere in the burn path.

---

### Impact Explanation

A user holding NLP worth ≤ $1 (in X18 terms) who submits a `BurnNlp` transaction loses 100% of their NLP value. The underlying quote value remains in the NLP pool (credited to `N_ACCOUNT`), constituting a direct, unrecoverable transfer of user assets to the protocol. The broken invariant: *a user burning NLP must receive quote proportional to NLP value minus a reasonable fee*. A 100% fee is not a reasonable fee.

---

### Likelihood Explanation

The entry path is fully reachable:

1. User signs a `BurnNlp` transaction with any `nlpAmount` such that `nlpAmount * oraclePriceX18 / ONE ≤ ONE`.
2. Sequencer processes it through `EndpointTx` (lines 554–573), validates the signature, charges `HEALTHCHECK_FEE`, and calls `clearinghouse.burnNlp`. [5](#0-4) 

There is no minimum NLP amount check in `mintNlp` or `burnNlp`. A user can arrive at a sub-$1 NLP balance through:
- Minting a small NLP amount (no minimum NLP amount enforced in `mintNlp`)
- NLP oracle price declining after minting

The `SignedBurnNlp` struct carries `oraclePriceX18` set by the sequencer at execution time, so the user cannot manipulate the price, but the condition is met whenever the NLP position is genuinely worth < $1. [6](#0-5) 

---

### Recommendation

Add a minimum NLP burn amount check before computing the fee, reverting if `quoteAmount ≤ ONE` (or more precisely, if `quoteAmount ≤ burnFee`):

```solidity
int128 quoteAmount = nlpAmount.mul(oraclePriceX18);
int128 burnFee = MathHelper.max(ONE, quoteAmount / 1000);
require(quoteAmount > burnFee, "NLP burn value too small");
quoteAmount = quoteAmount - burnFee;
```

This prevents the user from burning NLP and receiving nothing, making the economic constraint explicit rather than silently destroying value.

---

### Proof of Concept

**Setup**: NLP oracle price = $1 (`oraclePriceX18 = 1e18`). User holds `nlpAmount = 5e17` (0.5 NLP in X18 units, worth $0.50).

**Execution**:
```
quoteAmount = 5e17 * 1e18 / 1e18 = 5e17          // $0.50 in X18
burnFee     = max(1e18, 5e17 / 1000) = 1e18       // floor = $1.00
quoteAmount = max(0, 5e17 - 1e18) = max(0, -5e17) = 0
```

**Result**: `spotEngine.updateBalance(NLP_PRODUCT_ID, sender, -5e17)` executes (NLP burned), `spotEngine.updateBalance(QUOTE_PRODUCT_ID, sender, 0)` is skipped. User loses $0.50 of NLP value; `N_ACCOUNT` gains the NLP share. Assert `quoteReceived > 0` fails.
