### Title
`wei_to_fri` silently truncates to `GasPrice(0)` on low ETH/STRK rates, bypassing fallback and causing block proposals to fail — (`crates/starknet_api/src/block.rs`, `crates/apollo_consensus_orchestrator/src/utils.rs`)

---

### Summary

`GasPrice::wei_to_fri` performs integer division `(price_wei * eth_to_fri_rate) / WEI_PER_ETH` and returns `Ok(GasPrice(0))` — not an `Err` — when the numerator is smaller than `WEI_PER_ETH` (10^18). Because `L1PricesInFri::convert_from_wei` only falls back on an `Err` result, a zero FRI price silently propagates into `ProposalInit`. When `convert_to_sn_api_block_info` later calls `NonzeroGasPrice::new(GasPrice(0))`, it returns `Err(ZeroGasPrice)`, aborting block proposal construction. The fallback chain (previous block → config defaults) is never reached.

---

### Finding Description

**Root cause — `wei_to_fri` returns `Ok(GasPrice(0))` on truncation:** [1](#0-0) 

The comment explicitly acknowledges integer division: *"We use integer division since wei × eth_to_fri_rate is expected to be high enough to not cause too much precision loss."* No zero-check is performed on the result. When `price_wei * eth_to_fri_rate < WEI_PER_ETH` (10^18), `checked_div` returns `Some(GasPrice(0))` and the function returns `Ok(GasPrice(0))`.

**Fallback bypass — `convert_from_wei` treats zero as success:** [2](#0-1) 

`convert_from_wei` propagates the `Ok(GasPrice(0))` result. The caller only falls back to previous-block or config-default prices when `convert_from_wei` returns `Err`: [3](#0-2) 

Because the result is `Ok`, the fallback is skipped and `GasPrice(0)` is returned as the live FRI price.

**Zero price placed into `ProposalInit`:** [4](#0-3) 

`l1_gas_price_fri` and `l1_data_gas_price_fri` are set directly from the zero-valued `L1PricesInFri`.

**`NonzeroGasPrice::new` rejects zero, aborting block construction:** [5](#0-4) 

`NonzeroGasPrice::new(GasPrice(0))` returns `Err(StarknetApiError::ZeroGasPrice)`: [6](#0-5) 

The `?` in `convert_to_sn_api_block_info` propagates this error, causing `initiate_build` to fail and the block proposal to be aborted.

---

### Impact Explanation

When the ETH/STRK oracle returns a rate below `WEI_PER_ETH` (10^18, i.e., market condition where 1 ETH < 1 STRK), the proposer cannot build any block. The fallback to safe minimum prices is silently bypassed because `wei_to_fri` returns `Ok` with a zero value rather than `Err`. This is a liveness failure with economic impact: the sequencer halts block production for the duration of the low-rate condition, matching **"Critical. Incorrect fee, gas, bouncer, resource accounting, refund, balance, or L1 gas price effect with economic impact."**

---

### Likelihood Explanation

The trigger requires the ETH/STRK oracle to return `eth_to_fri_rate < 10^18`. No minimum rate is enforced anywhere in the pipeline: [7](#0-6) 

`calculate_eth_to_fri_rate` only rejects zero, not sub-`WEI_PER_ETH` values. A transient oracle misconfiguration, a market dislocation (1 ETH < 1 STRK), or a malicious oracle response can trigger this. The `min_l1_gas_price_wei` clamp does not help because the truncation occurs after the clamp, inside `wei_to_fri`.

---

### Recommendation

Add a zero-result check inside `wei_to_fri` (or inside `convert_from_wei`) and return `Err(StarknetApiError::GasPriceConversionError(...))` when the result is zero. This will cause `get_l1_prices_in_fri_and_wei_and_conversion_rate` to fall through to the previous-block or config-default fallback instead of silently propagating a zero price:

```rust
pub fn wei_to_fri(self, eth_to_fri_rate: u128) -> Result<GasPrice, StarknetApiError> {
    let result = self
        .checked_mul_u128(eth_to_fri_rate)
        .ok_or_else(|| StarknetApiError::GasPriceConversionError(...))?
        .checked_div(WEI_PER_ETH)
        .expect("WEI_PER_ETH must be non-zero");
    if result.0 == 0 && self.0 != 0 {
        return Err(StarknetApiError::GasPriceConversionError(
            format!("wei_to_fri truncated to zero: price={:?}, rate={}", self, eth_to_fri_rate)
        ));
    }
    Ok(result)
}
```

### Proof of Concept

```
eth_to_fri_rate = 5 * 10^17   // oracle returns rate < WEI_PER_ETH (1 ETH = 0.5 STRK)
price_in_wei    = 1            // after apply_fee_transformations clamp to min

wei_to_fri(1, 5*10^17):
  1 * 5*10^17 = 5*10^17
  5*10^17 / 10^18 = 0  (integer division)
  → Ok(GasPrice(0))          // no Err, fallback not triggered

convert_from_wei returns Ok(L1PricesInFri { l1_gas_price: GasPrice(0), ... })
get_l1_prices_in_fri_and_wei_and_conversion_rate returns (GasPrice(0), ...)

ProposalInit { l1_gas_price_fri: GasPrice(0), ... }

convert_to_sn_api_block_info:
  NonzeroGasPrice::new(GasPrice(0)) → Err(ZeroGasPrice)
  initiate_build returns Err(...)   // block proposal aborted
```
