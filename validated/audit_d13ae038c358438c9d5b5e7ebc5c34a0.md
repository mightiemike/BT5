### Title
Accumulated NEAR storage-stake tokens cannot be rescued after GC frees headers — (`contract/src/lib.rs`)

---

### Summary

`submit_blocks` is `#[payable]` and retains NEAR tokens from callers to cover on-chain storage costs. A built-in GC routine (`run_mainchain_gc`) is called inside the same function and removes old headers, freeing storage. The freed storage stake from **prior** transactions returns to the contract's account balance. Because no rescue or withdrawal function exists anywhere in the contract, those NEAR tokens are permanently trapped.

---

### Finding Description

`submit_blocks` captures `env::attached_deposit()` and measures storage before and after header insertion plus GC: [1](#0-0) 

The critical ordering is:

1. `initial_storage` is snapshotted.
2. New headers are inserted (storage grows).
3. `run_mainchain_gc` removes old headers (storage shrinks).
4. `diff_storage_usage = env::storage_usage().saturating_sub(initial_storage)` captures only the **net** change. [2](#0-1) 

`required_deposit` is therefore only the net byte cost. The deposit paid in **previous** calls that covered the now-GC'd headers was already retained by the contract. When GC removes those headers, the freed storage stake flows back into the contract's account balance — but there is no function to transfer it out. [3](#0-2) 

A search across every `.rs` file for `withdraw`, `rescue`, `drain`, `sweep`, or `collect` returns zero matches. The only outbound `Promise::transfer` in the entire contract is the per-call refund inside `submit_blocks`: [4](#0-3) 

That refund only covers the **current** call's overpayment. It cannot reach tokens that accumulated from earlier calls.

---

### Impact Explanation

Every steady-state relay cycle adds headers and then GC removes an equal number of old ones. The net storage change approaches zero, so `required_deposit` approaches zero and the current caller is fully refunded — but the NEAR tokens paid in earlier calls for the now-deleted headers remain locked in the contract balance forever. Over the lifetime of the relay (which is designed to run continuously), this compounds into a non-trivial sum of permanently irrecoverable NEAR.

---

### Likelihood Explanation

`run_mainchain_gc` is called unconditionally on every `submit_blocks` invocation: [5](#0-4) 

Once the chain reaches `gc_threshold` blocks (default suggested value: 52 704 blocks), GC fires on every relay batch. This means token accumulation begins automatically and continuously without any special attacker action — it is an inherent consequence of normal operation.

---

### Recommendation

Add a privileged (e.g., `Role::DAO`-gated) function that transfers the contract's excess balance to a designated recipient:

```rust
#[access_control_any(roles(Role::DAO))]
pub fn rescue_near(&mut self, amount: NearToken, recipient: AccountId) -> Promise {
    Promise::new(recipient).transfer(amount)
}
```

This mirrors the recommendation in the referenced report: add an explicit native-token rescue path alongside the existing ERC-20/storage-token handling.

---

### Proof of Concept

**Setup:** Deploy the contract with `gc_threshold = 10`.

**Step 1 — Fill to threshold:** Relay submits 10 headers across multiple `submit_blocks` calls, each with a deposit covering storage. The contract retains, say, 0.1 NEAR total across those calls.

**Step 2 — Steady-state relay:** Relayer submits 1 new header. Inside `submit_blocks`:
- `initial_storage` is recorded.
- 1 header is inserted (+80 bytes of storage).
- `run_mainchain_gc(1)` removes the oldest header (−80 bytes of storage).
- `diff_storage_usage = 0` (net change is zero due to `saturating_sub`).
- `required_deposit = 0`; the caller's full deposit is refunded.

**Observation:** The 0.1 NEAR retained from Step 1 for the now-deleted headers is still in the contract's balance. It cannot be retrieved. Repeating Step 2 indefinitely never recovers those tokens, and any future calls that trigger GC of headers paid for in Step 2 will add more trapped tokens. [6](#0-5) [7](#0-6)

```rust
    #[payable]
    #[pause]
    #[trusted_relayer]
    pub fn submit_blocks(
        &mut self,
        #[serializer(borsh)] headers: Vec<BlockHeader>,
    ) -> PromiseOrValue<()> {
        let amount = env::attached_deposit();
        let initial_storage = env::storage_usage();
        let num_of_headers = headers.len().try_into().unwrap();

        for header in headers {
            self.submit_block_header(header, self.skip_pow_verification);
        }

        self.run_mainchain_gc(num_of_headers);
        let diff_storage_usage = env::storage_usage().saturating_sub(initial_storage);
        let required_deposit = env::storage_byte_cost().saturating_mul(diff_storage_usage.into());

        require!(
            amount >= required_deposit,
            format!("Required deposit {}", required_deposit)
        );

        let refund = amount.saturating_sub(required_deposit);
        if refund > NearToken::from_near(0) {
            Promise::new(env::predecessor_account_id())
                .transfer(refund)
                .into()
        } else {
            PromiseOrValue::Value(())
        }
    }
```

```rust
    pub fn run_mainchain_gc(&mut self, batch_size: u64) {
        let initial_blockheader = self
            .headers_pool
            .get(&self.mainchain_initial_blockhash)
            .unwrap_or_else(|| env::panic_str(ERR_KEY_NOT_EXIST));

        let tip_blockheader = self
            .headers_pool
            .get(&self.mainchain_tip_blockhash)
            .unwrap_or_else(|| env::panic_str(ERR_KEY_NOT_EXIST));

        let amount_of_headers_we_store =
            tip_blockheader.block_height - initial_blockheader.block_height + 1;

        if amount_of_headers_we_store > self.gc_threshold {
            let total_amount_to_remove = amount_of_headers_we_store - self.gc_threshold;
            let selected_amount_to_remove = std::cmp::min(total_amount_to_remove, batch_size);

            let start_removal_height = initial_blockheader.block_height;
            let end_removal_height = initial_blockheader.block_height + selected_amount_to_remove;
            env::log_str(&format!(
                "Num of blocks to remove {selected_amount_to_remove}"
            ));

            for height in start_removal_height..end_removal_height {
                let blockhash = &self
                    .mainchain_height_to_header
                    .get(&height)
                    .unwrap_or_else(|| env::panic_str(ERR_KEY_NOT_EXIST));

                self.remove_block_header(blockhash);
                self.mainchain_height_to_header.remove(&height);
            }

            self.mainchain_initial_blockhash = self
                .mainchain_height_to_header
                .get(&end_removal_height)
                .unwrap_or_else(|| env::panic_str(ERR_KEY_NOT_EXIST));
        }
    }
```
