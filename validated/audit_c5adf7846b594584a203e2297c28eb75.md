### Title
NFT Permanently Unburnable After Transfer from Denom Creator — (`x/nft/keeper/keeper.go`)

### Summary

`BurnNFT` requires the caller to simultaneously satisfy both `IsOwner` and `IsDenomCreator` for the same address. Once the denom creator transfers an NFT to any third party, no address can ever satisfy both checks simultaneously, making the NFT permanently unburnable via `MsgBurnNFT`.

---

### Finding Description

`BurnNFT` in `x/nft/keeper/keeper.go` applies two sequential guards against the same `owner` address:

```go
// x/nft/keeper/keeper.go lines 141-161
func (k Keeper) BurnNFT(ctx sdk.Context, denomID, tokenID string, owner sdk.AccAddress) error {
    ...
    nft, err := k.IsOwner(ctx, denomID, tokenID, owner)   // owner must hold the NFT
    if err != nil {
        return err
    }
    _, err = k.IsDenomCreator(ctx, denomID, owner)         // same address must be denom creator
    if err != nil {
        return err
    }
    ...
}
``` [1](#0-0) 

`IsOwner` checks that `owner` equals `nft.GetOwner()`: [2](#0-1) 

`IsDenomCreator` checks that `address` equals `denom.Creator`, which is fixed at `IssueDenom` time and never changes: [3](#0-2) 

`TransferOwner` (called by `MsgTransferNFT`) updates `nft.Owner` to the new address but leaves `denom.Creator` unchanged: [4](#0-3) 

**Concrete attack path (no privileges required):**

1. Address A calls `MsgIssueDenom` → A is `denom.Creator`
2. A calls `MsgMintNFT` with `recipient = A` → A is both owner and creator; burn is possible
3. A calls `MsgTransferNFT` to address B → B is now owner; A is still creator
4. B calls `MsgBurnNFT`: `IsOwner` passes, `IsDenomCreator` fails → `ErrUnauthorized`
5. A calls `MsgBurnNFT`: `IsOwner` fails (A no longer owns it) → `ErrUnauthorized`

The NFT is now permanently locked in B's ownership with no burn path through any supported transaction.

---

### Impact Explanation

The invariant "every minted NFT can be burned by some authorized party" is broken. Any NFT transferred away from the denom creator becomes permanently unburnable. This affects:

- **Normal NFTs**: Permanently locked in the recipient's account; supply can never decrease below the transferred count.
- **IBC voucher NFTs on the destination chain**: The denom creator for IBC-received vouchers is the `escrowAddress` (set in `processReceivedPacket` line 168). The escrow address cannot sign transactions, so any IBC voucher NFT transferred from the receiver to a third party is also permanently unburnable via `MsgBurnNFT`. [5](#0-4) 

**Note on IBC escrow locking**: The question's claim that escrowed assets on the source chain are permanently locked is **not accurate**. The IBC return path uses `BurnNFTUnverified` (not `BurnNFT`), which skips the creator check entirely: [6](#0-5) [7](#0-6) 

So IBC round-trip mechanics still function. The impact is confined to the user-facing `MsgBurnNFT` path being permanently broken for any transferred NFT.

---

### Likelihood Explanation

This is triggered by any standard `MsgTransferNFT` followed by `MsgBurnNFT` — two ordinary, supported transactions. No special privileges, governance, or operator access are required. Any user who receives an NFT from the denom creator and later tries to burn it will hit this permanently.

---

### Recommendation

`BurnNFT` should allow burning if the caller is **either** the current owner **or** the denom creator, not require both simultaneously. A reasonable fix:

```go
func (k Keeper) BurnNFT(ctx sdk.Context, denomID, tokenID string, owner sdk.AccAddress) error {
    if !k.HasDenomID(ctx, denomID) {
        return sdkerrors.Wrapf(types.ErrInvalidDenom, "denom ID %s not exists", denomID)
    }
    nft, err := k.IsOwner(ctx, denomID, tokenID, owner)
    if err != nil {
        return err
    }
    // Remove the IsDenomCreator check, or change to OR logic
    k.deleteNFT(ctx, denomID, nft)
    k.deleteOwner(ctx, denomID, tokenID, owner)
    k.decreaseSupply(ctx, denomID)
    return nil
}
```

Alternatively, allow the denom creator to burn regardless of current ownership (similar to how `BurnNFTUnverified` works for IBC).

### Proof of Concept

```go
func TestBurnNFTAfterTransfer(t *testing.T) {
    // setup keeper, ctx, creator, recipient addresses
    creator := sdk.AccAddress([]byte("creator"))
    recipient := sdk.AccAddress([]byte("recipient"))

    k.IssueDenom(ctx, "testdenom", "Test", "", "", creator)
    k.MintNFT(ctx, "testdenom", "token1", "", "", "", creator, creator)

    // Transfer to recipient
    k.TransferOwner(ctx, "testdenom", "token1", creator, recipient)

    // recipient cannot burn: not denom creator
    err := k.BurnNFT(ctx, "testdenom", "token1", recipient)
    require.ErrorIs(t, err, types.ErrUnauthorized) // "not the creator"

    // creator cannot burn: no longer owner
    err = k.BurnNFT(ctx, "testdenom", "token1", creator)
    require.ErrorIs(t, err, types.ErrUnauthorized) // "not the owner"

    // NFT is permanently unburnable
}
```

func (k Keeper) IsOwner(ctx sdk.Context, denomID, tokenID string, owner sdk.AccAddress) (types.BaseNFT, error) {
	nft, err := k.GetNFT(ctx, denomID, tokenID)
	if err != nil {
		return types.BaseNFT{}, err
	}

	if !owner.Equals(nft.GetOwner()) {
		return types.BaseNFT{}, sdkerrors.Wrapf(types.ErrUnauthorized, "%s is not the owner of %s/%s", owner.String(), denomID, tokenID)
	}

	return nft.(types.BaseNFT), nil
}

func (k Keeper) IsDenomCreator(ctx sdk.Context, denomID string, address sdk.AccAddress) (types.Denom, error) {
	denom, err := k.GetDenom(ctx, denomID)
	if err != nil {
		return types.Denom{}, err
	}

	creator, err := sdk.AccAddressFromBech32(denom.Creator)
	if err != nil {
		panic(err)
	}

	if !creator.Equals(address) {
		return types.Denom{}, sdkerrors.Wrapf(types.ErrUnauthorized, "%s is not the creator of %s", address, denomID)
	}

	return denom, nil
}
