Audit Report

## Title
`SwapAllowlistExtension` gates the router address instead of the end-user, allowing any user to bypass a curated pool's swap allowlist via `MetricOmmSimpleRouter` — (`metric-periphery/contracts/extensions/SwapAllowlistExtension.sol`)

## Summary
`SwapAllowlistExtension.beforeSwap` receives `sender` from the pool, which is `msg.sender` of the `pool.swap` call — the router contract address, not the originating user. When a pool admin allowlists the router as the canonical entry point, every unprivileged user passes the check and can swap on a pool intended to be curated. Conversely, individually allowlisted EOAs are locked out when they route through the router.

## Finding Description
`MetricOmmPool.swap` passes `msg.sender` as the `sender` argument to `_beforeSwap`:

```solidity
// MetricOmmPool.sol L230-240
_beforeSwap(
  msg.sender,   // ← router address when called via router
  recipient,
  …
);
```

`ExtensionCalling._beforeSwap` forwards it verbatim via `abi.encodeCall(IMetricOmmExtensions.beforeSwap, (sender, …))`. `SwapAllowlistExtension.beforeSwap` then evaluates:

```solidity
// SwapAllowlistExtension.sol L37-39
if (!allowAllSwappers[msg.sender] && !allowedSwapper[msg.sender][sender]) {
    revert IMetricOmmPoolActions.NotAllowedToSwap();
}
```

Here `msg.sender` is the pool and `sender` is the router. The check resolves to `allowedSwapper[pool][router]`, not `allowedSwapper[pool][user]`. Any user calling `exactInputSingle`, `exactInput`, or `exactOutputSingle` through `MetricOmmSimpleRouter` presents the same `sender` (the router), so the allowlist provides no per-user discrimination. There is no existing guard that recovers the originating user from `extensionData` or any other field.

## Impact Explanation
A curated pool deploying `SwapAllowlistExtension` to restrict trading to a known set of counterparties is fully bypassed. Any user can execute swaps against the pool's liquidity at oracle-derived prices, enabling adverse selection against LP positions or trading on a pool intended to be private. This is a direct loss of LP principal and a broken core pool invariant (curation policy). Meets the "broken core pool functionality causing loss of funds" and "admin-boundary break bypassed by an unprivileged path" criteria.

## Likelihood Explanation
The router is the primary public entry point used by all periphery integrators. A pool admin who configures a swap allowlist and allowlists the router — the natural and documented setup — immediately opens the bypass to every user. The attacker requires no special privilege, no malicious token, and no non-standard setup. A single `exactInputSingle` call suffices and is repeatable indefinitely.

## Recommendation
`SwapAllowlistExtension.beforeSwap` must gate the economic initiator, not the intermediary. Two options:

**Option A:** Check `recipient` instead of `sender` for single-hop swaps where the user is always the recipient (`exactInputSingle`/`exactOutputSingle`). This is incomplete for multi-hop paths where intermediate recipients are the router itself.

**Option B (robust):** Require the router to encode the originating user in `extensionData` and have the extension decode and verify it. The pool already passes `extensionData` through to the hook unchanged, so this requires only a router-side encoding convention and extension-side decoding.

At minimum, NatSpec on `swap` must document that `sender` is the direct caller (router), not the end-user, so extension authors are not misled.

## Proof of Concept
```solidity
// Pool admin sets up a curated pool, intending only the router as entry point:
swapExtension.setAllowedToSwap(address(pool), address(router), true);
// This means ALL users are allowed — sender is always the router.

// Attacker (not individually allowlisted) calls:
router.exactInputSingle(ExactInputSingleParams({
    pool: address(pool),
    tokenIn: token0,
    tokenOut: token1,
    zeroForOne: true,
    amountIn: 1_000e18,
    amountOutMinimum: 0,
    recipient: attacker,
    deadline: block.timestamp,
    priceLimitX64: 0,
    extensionData: ""
}));
// pool.swap is called with msg.sender = router
// _beforeSwap(router, attacker, ...)
// allowedSwapper[pool][router] == true → no revert
// Attacker's swap executes against the curated pool's liquidity.
```
