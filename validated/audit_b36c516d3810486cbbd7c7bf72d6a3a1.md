### Title
Partial Exit from Tier Does Not Update Position's Recorded Delegation Shares — (`x/tieredrewards/keeper/msg_server.go`)

### Summary

In `ExitTierWithDelegation`, when a user partially exits a tier position, the actual staking delegation is correctly reduced via `transferDelegationFromPosition`, but the position's stored `Delegation.Shares` is never updated to reflect the reduced amount before `setPosition` persists the position. The position is saved with the original (inflated) share count, creating a permanent accounting mismatch.

### Finding Description

`ExitTierWithDelegation` in `msg_server.go` handles both full and partial exits from a tiered delegation position. For a partial exit, the flow is:

1. `reconcileAmountFromShares` converts `pos.Delegation.Shares` to a token amount.
2. `transferDelegationFromPosition` unbonds exactly `unbondedShares` from the position's delegator address in the staking module and re-delegates them to the owner. The staking module state is correctly reduced.
3. `remainingShares` is computed as `pos.Delegation.Shares.Sub(unbondedShares)`.
4. `setPosition(ctx, pos.Position, nil)` is called — but `pos.Position.Delegation.Shares` (i.e., `pos.Delegation.Shares`) is **never assigned `remainingShares`** before this call. [1](#0-0) 

The critical lines are:

```go
transferredShares, unbondedShares, transferredAmount, err := ms.transferDelegationFromPosition(ctx, pos, valAddr, msg.Amount)
// ...
} else {
    remainingShares := pos.Delegation.Shares.Sub(unbondedShares)
    // remainingShares is computed but NEVER written back to pos.Delegation.Shares
    // ...
    if err := ms.setPosition(ctx, pos.Position, nil); err != nil {  // saves stale shares
        return nil, err
    }
}
```

`pos.Delegation.Shares` is confirmed stored state: `transferDelegationFromPosition` uses it directly as the unbond amount for full exits (line 141), and `reconcileAmountFromShares` converts it to tokens for the min-lock check. [2](#0-1) 

After the partial exit, the staking module holds `Y` shares for the position's delegator address, but the position record stores the original `X` shares (`X > Y`).

### Impact Explanation

Two concrete impacts follow from the inflated stored shares:

**1. Overpayment of bonus rewards.** `processEventsAndClaimBonus` computes the bonus using `pos.Delegation.Shares` (passed into `computeSegmentBonus`). After a partial exit, every subsequent reward claim pays bonus as if the full original delegation is still active, draining the `RewardsPoolName` module account beyond what is owed. [3](#0-2) 

**2. Position becomes permanently un-exitable.** A future full-exit call reloads the position with the inflated `Delegation.Shares` and passes it directly to `Unbond`. Since the actual staking delegation is smaller, `Unbond` will fail (insufficient shares), permanently locking the position. The owner cannot recover the remaining delegation. [4](#0-3) 

### Likelihood Explanation

Any owner of a tiered position can trigger this by submitting `MsgExitTierWithDelegation` with an amount strictly less than the full position amount. No special privilege is required. The only precondition is that the remaining amount after the partial exit still meets the tier's `MinLockAmount`, which is enforced by the existing check — meaning the bug is reachable in the normal, intended partial-exit flow. [5](#0-4) 

### Recommendation

Assign `remainingShares` back to the position's delegation shares field before calling `setPosition` in the partial-exit branch:

```go
} else {
    remainingShares := pos.Delegation.Shares.Sub(unbondedShares)
    pos.Delegation.Shares = remainingShares   // <-- add this line
    // ... min lock check ...
    if err := ms.setPosition(ctx, pos.Position, nil); err != nil {
        return nil, err
    }
}
```

This mirrors the fix recommended for M-42: track and persist the actual remaining amount rather than assuming the full requested amount was consumed.

### Proof of Concept

1. Alice creates a tier position committing 1 000 000 `basecro` to validator V. The position stores `Delegation.Shares = S` (corresponding to 1 000 000 tokens).
2. Alice calls `MsgExitTierWithDelegation` with `Amount = 400 000`. The tier's `MinLockAmount` is 500 000, so the partial exit is valid.
3. `transferDelegationFromPosition` unbonds `unbondedShares` (≈ 0.4·S) from the position's delegator address and re-delegates them to Alice. The staking module now records ≈ 0.6·S for the position's delegator.
4. `remainingShares ≈ 0.6·S` is computed but never written back. `setPosition` persists `Delegation.Shares = S` (the original full amount).
5. On the next `ClaimTierRewards`, `computeSegmentBonus` uses `S` instead of `0.6·S`, paying Alice ≈ 67% more bonus than she is entitled to.
6. Alice later calls `MsgExitTierWithDelegation` with `Amount = 600 000` (the actual remaining value). `reconcileAmountFromShares(S)` returns ≈ 1 000 000, so `ExitWithFullDelegation(600 000, 1 000 000)` is false. The partial path is taken again, compounding the discrepancy.
7. If Alice attempts a full exit by requesting the full inflated amount, `Unbond` is called with `S` shares but the delegator only holds `0.6·S` — the call fails, and the position is permanently locked. [6](#0-5)

func (k Keeper) processEventsAndClaimBonus(ctx context.Context, pos *types.PositionState) (sdk.Coins, error) {
	// Rewards should have been claimed before undelegation
	if !pos.IsDelegated() {
		return sdk.NewCoins(), nil
	}

	valAddr, err := sdk.ValAddressFromBech32(pos.Delegation.ValidatorAddress)
	if err != nil {
		return nil, err
	}

	events, err := k.getValidatorEventsSince(ctx, valAddr, pos.LastEventSeq)
	if err != nil {
		return nil, err
	}

	sdkCtx := sdk.UnwrapSDKContext(ctx)
	blockTime := sdkCtx.BlockTime()

	totalBonus := math.ZeroInt()
	// Use the persisted bonded state from the last replay, not a hardcoded default.
	// This prevents overpaying bonus for unbonded gaps between claims.
	bonded := pos.LastKnownBonded
	segmentStart := pos.LastBonusAccrual

	tier, err := k.getTier(ctx, pos.TierId)
	if err != nil {
		return nil, err
	}

	for _, entry := range events {
		evt := entry.Event

		if bonded {
			// Compute bonus for the bonded segment [segmentStart, eventTime]
			// using the snapshot rate at the event.
			bonus := k.computeSegmentBonus(*pos, tier, segmentStart, evt.Timestamp, evt.TokensPerShare)
			totalBonus = totalBonus.Add(bonus)
		}
