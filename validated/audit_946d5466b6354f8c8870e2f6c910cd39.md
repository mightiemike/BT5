### Title
`ValidResourceBounds` Protobuf Deserialization Silently Downgrades `AllResources` to `L1Gas` for Zero-Bound Transactions, Breaking Hash Canonicalization Across the P2P Sync Boundary — (`File: crates/apollo_protobuf/src/converters/transaction.rs`)

---

### Summary

The protobuf converter `TryFrom<protobuf::ResourceBounds> for ValidResourceBounds` silently converts `AllResources { l2_gas: 0, l1_data_gas: 0 }` into `L1Gas`. Because `get_tip_resource_bounds_hash` hashes a different number of elements for each variant (3 for `L1Gas`, 4 for `AllResources`), the same transaction produces two distinct Poseidon hashes depending on which representation is used. A transaction submitted through the gateway is hashed with the 4-element preimage; after a P2P block-sync round-trip through protobuf, the same transaction body is deserialized as `L1Gas` and any subsequent hash recomputation produces the 3-element preimage. This is the sequencer-native analog of the bridge bug: an identity (the transaction hash) is assumed to be stable across a representation boundary, but the boundary silently changes the canonical form.

---

### Finding Description

**Step 1 – Submission path (gateway)**

`RpcInvokeTransactionV3.resource_bounds` is typed as `AllResourceBounds` (never `ValidResourceBounds`). When the gateway converts the RPC transaction to its internal form it calls `InternalRpcInvokeTransactionV3::calculate_transaction_hash`, which wraps the bounds as `ValidResourceBounds::AllResources(self.resource_bounds)` before calling `get_tip_resource_bounds_hash`. [1](#0-0) 

`get_tip_resource_bounds_hash` for `AllResources` appends the `l1_data_gas` packed element, producing a 4-element Poseidon input `[tip, l1_gas_packed, l2_gas_packed, l1_data_gas_packed]`: [2](#0-1) 

**Step 2 – P2P block-sync path (protobuf deserialization)**

When a committed block is synced over P2P, `InvokeTransactionV3` is reconstructed via `TryFrom<protobuf::InvokeV3>`, which calls `ValidResourceBounds::try_from(protobuf::ResourceBounds)`: [3](#0-2) 

The decision gate is:

```rust
Ok(if l1_data_gas.is_zero() && l2_gas.is_zero() {
    ValidResourceBounds::L1Gas(l1_gas)   // ← variant changes here
} else {
    ValidResourceBounds::AllResources(...)
})
```

A transaction submitted with `AllResourceBounds { l1_gas: X, l2_gas: 0, l1_data_gas: 0 }` satisfies the condition and is silently downgraded to `ValidResourceBounds::L1Gas(X)`.

**Step 3 – Hash divergence**

For `L1Gas`, `get_tip_resource_bounds_hash` produces a 3-element Poseidon input `[tip, l1_gas_packed, l2_gas_packed(0)]` — the `l1_data_gas` element is absent: [4](#0-3) 

The two Poseidon hashes are distinct. Any component that recomputes the transaction hash from the protobuf-deserialized `InvokeTransactionV3` — including `validate_transaction_hash`, `execute_single_api_tx` in blockifier re-execution, or any RPC handler that derives the hash from the stored transaction body — will produce hash H₂ ≠ H₁ (the hash the user signed and that was committed to the block). [5](#0-4) 

**Contrast with the RPC/consensus path**

The `RpcTransaction` protobuf converter uses a separate `TryFrom<protobuf::ResourceBounds> for AllResourceBounds` that always produces `AllResourceBounds` without the `L1Gas` downgrade: [6](#0-5) 

So the gateway and consensus paths are unaffected; only the committed-block P2P sync path carries the broken conversion.

---

### Impact Explanation

Any node that syncs a block containing a V3 invoke transaction with `l2_gas = 0` and `l1_data_gas = 0` will store the transaction body with `ValidResourceBounds::L1Gas`. If that node subsequently:

- recomputes the transaction hash for block validation (`validate_transaction_hash`), it obtains H₂ ≠ H₁ and rejects the block as invalid;
- serves the transaction via RPC `starknet_getTransactionByHash` and derives the hash from the stored body, it returns H₂ — an authoritative-looking wrong value;
- re-executes the block (blockifier re-execution), the account contract's `get_tx_info()` syscall receives H₂, signature verification fails, and the transaction reverts — a wrong execution result for accepted input.

This matches **High: Transaction conversion or signature/hash logic binds the wrong hash** and **High: RPC execution returns an authoritative-looking wrong value**.

---

### Likelihood Explanation

Any unprivileged user can trigger this by submitting a V3 invoke transaction with `AllResourceBounds { l1_gas: nonzero, l2_gas: 0, l1_data_gas: 0 }`. The gateway accepts it (no validation prevents zero L2/L1DataGas bounds), includes it in a block, and the block is propagated via P2P. Every syncing peer that processes the block through the protobuf path will silently downgrade the resource bounds variant and diverge on hash recomputation.

---

### Recommendation

Remove the `L1Gas` vs `AllResources` branching from the protobuf converter for committed-block transactions. The canonical form of a V3 transaction is always `AllResources`; the `L1Gas` variant exists only for pre-0.13.3 transactions that were never submitted as `AllResources`. The fix is to always deserialize into `AllResources` when the protobuf message originates from a V3 transaction context:

```rust
// In TryFrom<protobuf::ResourceBounds> for ValidResourceBounds (block-sync path):
// Always produce AllResources for V3 transactions; never downgrade to L1Gas.
Ok(ValidResourceBounds::AllResources(AllResourceBounds { l1_gas, l2_gas, l1_data_gas }))
```

Alternatively, gate the `L1Gas` downgrade on an explicit version field so that only pre-0.13.3 transactions are converted to `L1Gas`.

---

### Proof of Concept

```
1. Craft a V3 invoke transaction T with:
     resource_bounds = AllResourceBounds { l1_gas: 1000, l2_gas: 0, l1_data_gas: 0 }
   Submit to the gateway. Gateway computes:
     H1 = poseidon([INVOKE, 3, sender, get_tip_resource_bounds_hash(AllResources{...}), ...])
   where get_tip_resource_bounds_hash hashes [tip, pack(L1Gas,1000), pack(L2Gas,0), pack(L1DataGas,0)]
   → 4-element Poseidon input.

2. T is included in block B. B is propagated via P2P.

3. A syncing node receives B and deserializes T via TryFrom<protobuf::InvokeV3>:
     ValidResourceBounds::try_from(protobuf::ResourceBounds { l1_gas: 1000, l2_gas: 0, l1_data_gas: 0 })
   → l1_data_gas.is_zero() && l2_gas.is_zero() == true
   → ValidResourceBounds::L1Gas(1000)   ← variant changed

4. The syncing node recomputes the hash:
     H2 = poseidon([INVOKE, 3, sender, get_tip_resource_bounds_hash(L1Gas{1000}), ...])
   where get_tip_resource_bounds_hash hashes [tip, pack(L1Gas,1000), pack(L2Gas,0)]
   → 3-element Poseidon input.

5. H1 ≠ H2.
   - validate_transaction_hash(T, block_number, chain_id, H1) recomputes H2, H2 ∉ {H1} → block rejected.
   - RPC starknet_getTransactionByHash returns H2 for T → wrong authoritative value.
   - Re-execution: account contract sees H2, signature over H1 fails → wrong revert.
```

// An implementation of the SNIP: https://github.com/EvyatarO/SNIPs/blob/snip-8/SNIPS/snip-8.md
pub fn get_tip_resource_bounds_hash(
    resource_bounds: &ValidResourceBounds,
    tip: &Tip,
) -> Result<Felt, StarknetApiError> {
    let l1_resource_bounds = resource_bounds.get_l1_bounds();
    let l2_resource_bounds = resource_bounds.get_l2_bounds();

    // L1 and L2 gas bounds always exist.
    // Old V3 txs always have L2 gas bounds of zero, but they exist.
    let mut resource_felts = vec![
        get_concat_resource(&l1_resource_bounds, L1_GAS)?,
        get_concat_resource(&l2_resource_bounds, L2_GAS)?,
    ];

    // For new V3 txs, need to also hash the data gas bounds.
    resource_felts.extend(match resource_bounds {
        ValidResourceBounds::L1Gas(_) => vec![],
        ValidResourceBounds::AllResources(all_resources) => {
            vec![get_concat_resource(&all_resources.l1_data_gas, L1_DATA_GAS)?]
        }
    });

    Ok(HashChain::new().chain(&tip.0.into()).chain_iter(resource_felts.iter()).get_poseidon_hash())
}

impl TryFrom<protobuf::ResourceBounds> for ValidResourceBounds {
    type Error = ProtobufConversionError;
    fn try_from(value: protobuf::ResourceBounds) -> Result<Self, Self::Error> {
        let Some(l1_gas) = value.l1_gas else {
            return Err(missing("ResourceBounds::l1_gas"));
        };
        let Some(l2_gas) = value.l2_gas else {
            return Err(missing("ResourceBounds::l2_gas"));
        };
        // TODO(Shahak): Assert data gas is not none once we remove support for 0.13.2.
        let l1_data_gas = value.l1_data_gas.unwrap_or_default();
        let l1_gas: ResourceBounds = l1_gas.try_into()?;
        let l2_gas: ResourceBounds = l2_gas.try_into()?;
        let l1_data_gas: ResourceBounds = l1_data_gas.try_into()?;
        Ok(if l1_data_gas.is_zero() && l2_gas.is_zero() {
            ValidResourceBounds::L1Gas(l1_gas)
        } else {
            ValidResourceBounds::AllResources(AllResourceBounds { l1_gas, l2_gas, l1_data_gas })
        })
    }
