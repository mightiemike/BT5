### Title
Single `MAX_REF_STALENESS` Applied to Both Legs of Synthetic Pair Allows Stale-Price Execution Against LPs — (`smart-contracts-poc/contracts/AnchoredPriceProvider.sol`)

### Summary

`AnchoredPriceProvider` supports a two-feed synthetic ratio mode (e.g. BTC/USD ÷ ETH/USD → BTC/ETH). Both legs are validated through the same `_readLeg` helper, which applies the single immutable `MAX_REF_STALENESS` to both feeds. Because different oracle feeds have different heartbeats, no single threshold can simultaneously be tight enough to catch stale data on the faster feed and loose enough to avoid DoS on the slower feed. When the deployer accommodates the slower feed, the faster feed can be significantly stale at swap time, producing a bad synthetic price that traders can exploit at LP expense.

### Finding Description

`_readLeg` is called for both `baseFeedId` and `quoteFeedId` and applies the same `MAX_REF_STALENESS` to each:

```solidity
// AnchoredPriceProvider.sol _readLeg (line 283)
if (_isStale(refTime, block.timestamp, MAX_REF_STALENESS)) return (mid, spreadBps, refTime, false);
``` [1](#0-0) 

Both legs are read in `_getBidAndAskPrice`:

```solidity
(uint256 mid, uint256 spreadBps, , bool ok) = _readLeg(baseFeedId);
...
(uint256 mid2, uint256 spreadBps2, , bool ok2) = _readLeg(_quote);
``` [2](#0-1) 

`MAX_REF_STALENESS` is a single immutable set at construction with no per-feed override:

```solidity
if (_maxRefStaleness > 7 days) revert MaxRefStalenessOutOfBounds();
MAX_REF_STALENESS = _maxRefStaleness;
``` [3](#0-2) 

The protocol is explicitly multi-chain (the `smart-contracts-poc` sub-repo contains multi-chain deployment infrastructure). Different chains publish the same asset feeds at different heartbeats. For example, a BTC/USD feed may have a 24-hour heartbeat on one chain while an ETH/USD feed on the same chain has a 1-hour heartbeat. The deployer faces an inescapable dilemma:

| `MAX_REF_STALENESS` choice | Effect on base feed (24 h HB) | Effect on quote feed (1 h HB) |
|---|---|---|
| 1 h (tight) | Reverts whenever base feed is 1–24 h old → DoS | Correct |
| 24 h (loose) | Correct | Quote feed can be up to 24 h stale → bad price |

There is no single value that is simultaneously correct for both legs.

### Impact Explanation

When `MAX_REF_STALENESS` is set to the slower feed's heartbeat (the only way to avoid DoS), the faster feed's `refTime` can lag by nearly the full window. The synthetic ratio `mid1 / mid2` is then computed from a stale `mid2`. A trader who observes that the real ETH/USD price has moved 5–10% since the last oracle update can swap against the pool at the stale synthetic price, extracting that percentage of the pool's liquidity from LPs. This is a direct, quantifiable loss of LP principal — matching the "bad-price execution" and "pool insolvency" impact categories.

### Likelihood Explanation

The `AnchoredPriceProvider` is documented as "the one standard provider for public pools" and explicitly supports synthetic ratio mode via the `quoteFeedId` constructor parameter. The protocol targets any EVM-compatible chain. Feed heartbeats differ across chains for the same asset (e.g. USDT/USD: ~24 h on Linea, ~27 s on Polygon). Any synthetic pool deployed on a chain where the two constituent feeds have materially different heartbeats is affected. The factory admin cannot work around this structurally — the code provides no per-leg staleness parameter.

### Recommendation

Add a separate `MAX_QUOTE_STALENESS` immutable for the quote leg, passed as a distinct constructor argument alongside `_maxRefStaleness`. Pass it to `_readLeg` when reading `quoteFeedId`:

```solidity
uint256 public immutable MAX_REF_STALENESS;   // base leg
uint256 public immutable MAX_QUOTE_STALENESS; // quote leg (synthetic mode)

function _readLeg(bytes32 feedId, uint256 maxStaleness) internal returns (...) {
    ...
    if (_isStale(refTime, block.timestamp, maxStaleness)) return (..., false);
    ...
}
```

Call sites:
```solidity
_readLeg(baseFeedId,  MAX_REF_STALENESS);
_readLeg(quoteFeedId, MAX_QUOTE_STALENESS);
```

For single-feed deployments, `MAX_QUOTE_STALENESS` is unused and can be set to any value (e.g. `type(uint256).max`).

### Proof of Concept

1. Deploy `AnchoredPriceProvider` in synthetic mode: `baseFeedId = BTC/USD` (24 h heartbeat), `quoteFeedId = ETH/USD` (1 h heartbeat), `MAX_REF_STALENESS = 24 hours`.
2. At `t = 0`, oracle publishes BTC/USD = 65 000 and ETH/USD = 3 000; synthetic BTC/ETH = 21.67.
3. At `t = 20 h`, ETH/USD rises to 3 300 (real market). The oracle has not published a new ETH/USD update (still within its 1 h heartbeat window — the feed updated at `t = 19 h 59 m`). The stored `refTime` for ETH/USD is `t = 0` (last push was at `t = 0` for simplicity).
4. A trader calls `swap` on the BTC/ETH pool. `_readLeg(quoteFeedId)` checks `(20 h) > MAX_REF_STALENESS (24 h)` → **false** → staleness check passes. The stale ETH/USD = 3 000 is used.
5. Synthetic price = 65 000 / 3 000 = 21.67 (stale). Real price = 65 000 / 3 300 = 19.70.
6. The trader buys BTC/ETH at 21.67 (pool sells BTC cheaply in ETH terms), extracting ~9% above fair value from LP reserves.
7. LPs suffer a direct loss proportional to the price discrepancy and swap volume. [4](#0-3) [5](#0-4)

    function _getBidAndAskPrice() internal returns (uint128, uint128) {
        (uint256 mid, uint256 spreadBps, , bool ok) = _readLeg(baseFeedId);
        if (!ok) return (0, type(uint128).max);

        bytes32 _quote = quoteFeedId;
        if (_quote != bytes32(0)) {
            (uint256 mid2, uint256 spreadBps2, , bool ok2) = _readLeg(_quote);
            if (!ok2 || mid2 == 0) return (0, type(uint128).max);
            // Synthetic ratio (8-decimal): mid1 / mid2. Relative uncertainties of a ratio add.
            mid = Math.mulDiv(mid, ORACLE_DECIMALS, mid2);
            spreadBps += spreadBps2;
        }

        return _computeBidAsk(mid, spreadBps);
    }

```text
    function _readLeg(bytes32 feedId)
        internal returns (uint256 mid, uint256 spreadBps, uint256 refTime, bool ok)
    {
        (mid, spreadBps, , refTime) = IPricedOracle(address(offchainOracle)).price(feedId, msg.sender);

        // Stale reference → not ok. Clamping to a stale anchor is the one false-safety case.
        if (_isStale(refTime, block.timestamp, MAX_REF_STALENESS)) return (mid, spreadBps, refTime, false);

        // Basic validity — mid positive, spreadBps not the stalled/off-hours marker (the Chainlink oracle
        // writes spreadBps = ORACLE_BPS when an RWA market is closed).
        if (mid == 0 || spreadBps >= ORACLE_BPS) return (mid, spreadBps, refTime, false);

        // Per-leg price guard.
        (uint128 guardMin, uint128 guardMax) = offchainOracle.priceGuard(feedId);
        guardMax = guardMax == 0 ? type(uint128).max : guardMax;
        if (mid < guardMin || mid > guardMax) return (mid, spreadBps, refTime, false);

        ok = true;
    }
