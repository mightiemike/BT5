### Title
Unchecked Return Value of `transferFrom` in `replaceUsdcEWithUsdc` Enables usdcE Drain Without USDC Payment — (File: `core/contracts/ContractOwner.sol`)

---

### Summary

`ContractOwner.replaceUsdcEWithUsdc` calls `IERC20Base(usdc).transferFrom(msg.sender, directDepositV1, balance)` without checking the boolean return value. If the USDC token silently returns `false` instead of reverting, the function continues to withdraw usdcE from the `DirectDepositV1` contract and transfer it to `msg.sender`, allowing an unprivileged caller to receive usdcE without providing any USDC.

---

### Finding Description

`replaceUsdcEWithUsdc` is a migration helper callable by any external user on chain 57073 (Ink Sepolia). Its intended invariant is: caller provides USDC → directDepositV1 receives USDC → caller receives usdcE in exchange.

The function at line 616 performs:

```solidity
IERC20Base(usdc).transferFrom(msg.sender, directDepositV1, balance);
DirectDepositV1(directDepositV1).withdraw(IIERC20Base(usdcE));
IERC20Base(usdcE).safeTransfer(msg.sender, balance);
```

The `transferFrom` call returns a `bool` per the `IERC20Base` interface, but the return value is silently discarded. If the call returns `false` (e.g., caller has no USDC allowance or balance, and the token does not revert), execution continues: usdcE is withdrawn from `directDepositV1` to `ContractOwner`, then forwarded to `msg.sender` via `safeTransfer`.

Every other token transfer in the codebase uses `ERC20Helper.safeTransferFrom`, which wraps the call and requires `success && (data.length == 0 || abi.decode(data, (bool)))`. This one site is the sole exception. [1](#0-0) [2](#0-1) 

---

### Impact Explanation

**Asset delta:** An attacker receives `balance` usdcE tokens from a victim subaccount's `DirectDepositV1` without transferring any USDC. The usdcE balance of `directDepositV1` is permanently drained. The subaccount owner loses their usdcE with no USDC credited in return.

**Broken invariant:** The function's atomicity guarantee — USDC in, usdcE out — is violated. The usdcE leg executes unconditionally regardless of whether the USDC leg succeeded. [3](#0-2) 

---

### Likelihood Explanation

- The function is `external` with no access control beyond `block.chainid == 57073`. Any unprivileged caller can invoke it for any subaccount that has a registered `directDepositV1Address` with a non-zero usdcE balance.
- Exploitability depends on whether the specific USDC token at `0x2D270e6886d130D724215A266106e6832161EAEd` on Ink Sepolia returns `false` on failure rather than reverting. Non-standard or wrapped USDC variants on testnets/new chains frequently exhibit this behavior.
- The attacker needs only to call the function with zero USDC allowance; no privileged access, no governance capture, and no sequencer compromise is required. [4](#0-3) 

---

### Recommendation

Replace the raw `transferFrom` call with the `safeTransferFrom` wrapper already defined in `ERC20Helper`, consistent with every other transfer site in the codebase:

```solidity
// Before (unsafe):
IERC20Base(usdc).transferFrom(msg.sender, directDepositV1, balance);

// After (safe):
IERC20Base(usdc).safeTransferFrom(msg.sender, directDepositV1, balance);
```

`ContractOwner` already imports and uses `ERC20Helper` via `using ERC20Helper for IERC20Base`, so no additional imports are needed. [5](#0-4) [6](#0-5) 

---

### Proof of Concept

1. A subaccount `S` has a registered `directDepositV1Address[S]` holding `N` usdcE tokens.
2. Attacker calls `ContractOwner.replaceUsdcEWithUsdc(S)` with zero USDC allowance.
3. `IERC20Base(usdc).transferFrom(attacker, directDepositV1, N)` returns `false` (no revert).
4. Return value is ignored; execution continues.
5. `DirectDepositV1(directDepositV1).withdraw(usdcE)` transfers `N` usdcE to `ContractOwner`.
6. `IERC20Base(usdcE).safeTransfer(attacker, N)` transfers `N` usdcE to the attacker.
7. Attacker receives `N` usdcE; subaccount `S`'s `directDepositV1` is drained; no USDC was paid. [7](#0-6) [8](#0-7)

        require(block.chainid == 57073, ERR_UNAUTHORIZED);

        require(directDepositV1 != address(0), "no dda");
        address usdcE = 0xF1815bd50389c46847f0Bda824eC8da914045D14;
        address usdc = 0x2D270e6886d130D724215A266106e6832161EAEd;
        uint256 balance = IERC20Base(usdcE).balanceOf(directDepositV1);
        if (balance > 0) {
            IERC20Base(usdc).transferFrom(msg.sender, directDepositV1, balance);
            DirectDepositV1(directDepositV1).withdraw(IIERC20Base(usdcE));
            IERC20Base(usdcE).safeTransfer(msg.sender, balance);
        }
    }

    function safeTransfer(
        IERC20Base self,
        address to,
        uint256 amount
    ) internal {
        (bool success, bytes memory data) = address(self).call(
            abi.encodeWithSelector(IERC20Base.transfer.selector, to, amount)
        );
        require(
            success && (data.length == 0 || abi.decode(data, (bool))),
            ERR_TRANSFER_FAILED
        );
    }
