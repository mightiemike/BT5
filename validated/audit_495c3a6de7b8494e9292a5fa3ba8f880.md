### Title
Accrued Bonus Rewards Permanently Lost When Bonus Pool Is Empty During Redelegation Slash — (`File: x/tieredrewards/keeper/slash.go`)

---

### Summary

In `slashRedelegationPosition`, when `processEventsAndClaimBonus` returns `ErrInsufficientBonusPool`, the error is deliberately swallowed to avoid a chain halt. However, `processEventsAndClaimBonus` has already (a) decremented event reference counts in the KV store and (b) advanced the position's `LastEventSeq`, `LastBonusAccrual`, and `LastKnownBonded` checkpoints in memory — all before the pool-balance check. Because `slashRedelegationPosition` then calls `setPositionWithState` with the mutated position, those advanced checkpoints are persisted. The bonus rewards for the consumed period are never transferred to the owner and can never be reclaimed.

---

### Finding Description

`processEventsAndClaimBonus` performs two categories of side effects before it checks whether the pool can cover the computed bonus:

**Category 1 — KV store writes (immediately durable):**
Inside the event loop, `decrementEventRefCount` is called for every processed event: [1](#0-0) 

`decrementEventRefCount` writes directly to the `ValidatorEvents` collection — it either decrements `ReferenceCount` or deletes the entry entirely when the count reaches zero: [2](#0-1) 

These writes are not wrapped in a `CacheContext` and are therefore committed to the store regardless of whether the function later returns an error.

**Category 2 — In-memory checkpoint advances (persisted by the caller):**
After the event loop, `applyBonusAccrualCheckpoint` and `UpdateLastKnownBonded` advance the position's accrual window on the in-memory `pos` struct: [3](#0-2) 

Only after these mutations does the function check the pool balance and return `ErrInsufficientBonusPool`: [4](#0-3) 

Back in `slashRedelegationPosition`, the error is swallowed: [5](#0-4) 

Execution then falls through to `setPositionWithState`, which persists the mutated `pos` — including the advanced `LastEventSeq`, `LastBonusAccrual`, and `LastKnownBonded`: [6](#0-5) 

The net result: the validator events for the accrual period are consumed (reference counts decremented, entries potentially deleted), the position's accrual window is advanced past the earned period, and the bonus is never sent to the owner. There is no subsequent path that can recover the lost rewards — the events are gone and the checkpoint has moved forward.

---

### Impact Explanation

A delegator who holds a tier position with an active redelegation permanently loses all accrued bonus rewards for the period between their last claim and the slash event, whenever the bonus pool is empty at the time the slash fires. The `RewardsPoolName` module account balance is the exact value that is never transferred. The corrupted invariant is: `owner.balance += accrued_bonus` must hold after every settlement path, but this path silently skips the transfer while advancing the accounting state.

---

### Likelihood Explanation

The trigger requires two concurrent conditions:
1. A validator slash that fires `BeforeRedelegationSlashed` — a normal Cosmos SDK staking event triggered by evidence submission (double-sign, downtime).
2. The bonus pool being empty or underfunded at that block — realistic because the pool is a finite module account that can be drained by concurrent claims or simply not yet replenished.

Both conditions are reachable by unprivileged actors: any delegator can create a tier position and redelegate, and any evidence submitter can trigger a slash. The combination is not exotic.

---

### Recommendation

Wrap the `processEventsAndClaimBonus` call inside `slashRedelegationPosition` in a `CacheContext`. On `ErrInsufficientBonusPool`, discard the cache (rolling back all store writes including `decrementEventRefCount`) and do **not** persist the mutated `pos` checkpoints. This preserves the accrual window so the owner can claim the rewards once the pool is replenished:

```go
cacheCtx, commit := sdk.UnwrapSDKContext(ctx).CacheContext()
_, err = k.processEventsAndClaimBonus(cacheCtx, &pos)
if err != nil {
    if errors.Is(err, types.ErrInsufficientBonusPool) {
        // discard cache — do NOT commit; checkpoints and ref-counts stay unchanged
        k.logger(ctx).Error("insufficient bonus pool during redelegation slash", ...)
        // reload pos without checkpoint mutation for the subsequent setPositionWithState
        pos, _ = k.getPositionState(ctx, positionId)
    } else {
        return err
    }
} else {
    commit()
}
```

Alternatively, move the pool-balance check to before any store writes or checkpoint mutations inside `processEventsAndClaimBonus`.

---

### Proof of Concept

1. Create a tier position and redelegate it to validator B. This creates a `RedelegationMapping` entry.
2. Let bonus accrue for a meaningful period (e.g., 30 days) while the bonus pool remains empty (or drain it just before step 3).
3. Submit evidence that causes validator B to be slashed. The SDK calls `BeforeRedelegationSlashed` → `slashRedelegationPosition`.
4. Inside `processEventsAndClaimBonus`, the event loop runs: `decrementEventRefCount` is called for each pending event (store writes committed), `LastEventSeq`/`LastBonusAccrual`/`LastKnownBonded` are updated on `pos`. The pool check fails with `ErrInsufficientBonusPool`.
5. The error is swallowed at `slash.go:56`. `setPositionWithState` is called with the mutated `pos`.
6. After the block, query the position: `LastBonusAccrual` has advanced to the slash block time. Query the validator events: the entries for the accrual period are gone (ref-count reached zero). Query the owner's balance: no bonus received.
7. Fund the pool and call `MsgClaimTierRewards`. The claim returns zero bonus — the accrual window starts at the slash block time, and the events that covered the earned period no longer exist. [7](#0-6) [8](#0-7) [2](#0-1)

func (k Keeper) decrementEventRefCount(ctx context.Context, valAddr sdk.ValAddress, seq uint64) error {
	key := collections.Join(valAddr, seq)
	event, err := k.ValidatorEvents.Get(ctx, key)
	if errors.Is(err, collections.ErrNotFound) {
		return nil // already cleaned up
	}
	if err != nil {
		return err
	}

	if event.ReferenceCount <= 1 {
		return k.ValidatorEvents.Remove(ctx, key)
	}

	event.ReferenceCount--
	return k.ValidatorEvents.Set(ctx, key, event)
}
