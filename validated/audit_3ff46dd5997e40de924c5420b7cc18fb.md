### Title
Missing Zero Address Validation in `Airdrop.initialize` Allows Sanctioned Addresses to Bypass Sanctions Check — (`File: core/contracts/Airdrop.sol`)

---

### Summary

`Airdrop.initialize` assigns `token` and `sanctions` from caller-supplied parameters with no zero address guards. If `sanctions` is initialized to `address(0)`, every subsequent call to `claim()` silently bypasses the OFAC/sanctions enforcement, allowing sanctioned addresses to receive airdrop tokens.

---

### Finding Description

`Airdrop.initialize` sets both `token` and `sanctions` directly from parameters without any `require(_x != address(0))` guard:

```solidity
function initialize(address _token, address _sanctions)
    external
    initializer
{
    __Ownable_init();
    token = _token;       // no zero-address check
    sanctions = _sanctions; // no zero-address check
}
``` [1](#0-0) 

The `sanctions` variable is later used in `_verifyProof` to gate every claim:

```solidity
require(
    !ISanctionsList(sanctions).isSanctioned(sender),
    "address is sanctioned."
);
``` [2](#0-1) 

If `sanctions == address(0)`, the EVM executes a call to the zero address. Calls to `address(0)` always return `(success=true, data="")`. Decoding empty return data as `bool` yields `false`. Therefore `!false == true`, and the `require` passes unconditionally — every caller, including OFAC-sanctioned addresses, clears the check.

The same initialization path also sets `token` without a zero check. If `token == address(0)`, `SafeERC20.safeTransfer(IERC20(address(0)), ...)` reverts on every `claim()` call, permanently bricking the airdrop (DoS — noted for completeness but not the primary finding). [3](#0-2) 

---

### Impact Explanation

A misconfigured `sanctions = address(0)` silently disables the only sanctions enforcement gate in the `Airdrop` contract. Any address on the OFAC SDN list can call `claim()`, pass `_verifyProof`, and receive airdrop tokens. The corrupted state is `claimed[week][sanctionedAddress]` being set and tokens transferred out — an irreversible asset delta. This is a compliance-critical security failure: the protocol's stated sanctions controls are rendered inoperative. [4](#0-3) 

---

### Likelihood Explanation

Likelihood is low-to-medium. The misconfiguration must occur at initialization time (a one-shot `initializer`-gated call). However, the external report's reference implementation suffered the exact same omission, confirming this is a realistic deployment error. There is no on-chain enforcement preventing `address(0)` from being passed, and no post-deployment setter to correct `sanctions` — making the misconfiguration permanent once deployed. [5](#0-4) 

---

### Recommendation

Add explicit zero address guards in `initialize` for both `_token` and `_sanctions`:

```solidity
function initialize(address _token, address _sanctions)
    external
    initializer
{
    require(_token != address(0), "token is zero address");
    require(_sanctions != address(0), "sanctions is zero address");
    __Ownable_init();
    token = _token;
    sanctions = _sanctions;
}
```

Additionally, consider adding an `onlyOwner` setter for `sanctions` with the same guard, to allow correction without redeployment.

---

### Proof of Concept

1. Deploy `Airdrop` proxy and call `initialize(validToken, address(0))`.
2. Owner calls `registerMerkleRoot(1, merkleRoot)` with a valid Merkle root that includes a sanctioned address `S` with `totalAmount = 1e18`.
3. Address `S` (OFAC-sanctioned) calls `claim([{week: 1, totalAmount: 1e18, proof: validProof}])`.
4. Execution reaches `_verifyProof` → `ISanctionsList(address(0)).isSanctioned(S)` → EVM call to `address(0)` returns `(true, "")` → decoded `bool` = `false` → `require(!false)` passes.
5. `SafeERC20.safeTransfer(IERC20(token), S, 1e18)` executes; `S` receives tokens.
6. `claimed[1][S] = 1e18` is set — state is permanently corrupted. [6](#0-5)

    function _verifyProof(
        uint32 week,
        address sender,
        uint256 totalAmount,
        bytes32[] calldata proof
    ) internal {
        require(claimed[week][sender] == 0, "Already claimed.");
        require(
            merkleRoots[week] != bytes32(0),
            "Week hasn't been registered."
        );
        require(
            !ISanctionsList(sanctions).isSanctioned(sender),
            "address is sanctioned."
        );
        bytes32 leaf = keccak256(
            bytes.concat(keccak256(abi.encode(sender, totalAmount)))
        );
        bool isValidLeaf = MerkleProof.verify(proof, merkleRoots[week], leaf);
        require(isValidLeaf, "Invalid proof.");
        claimed[week][sender] = totalAmount;
    }

    function _claim(
        uint32 week,
        uint256 totalAmount,
        bytes32[] calldata proof
    ) internal {
        _verifyProof(week, msg.sender, totalAmount, proof);
        SafeERC20.safeTransfer(IERC20(token), msg.sender, totalAmount);
        emit Claim(msg.sender, week, totalAmount);
