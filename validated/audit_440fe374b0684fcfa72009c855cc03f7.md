### Title
`ContractOwner` Cannot Receive Native ETH from `DirectDepositV1#withdrawNative`, Permanently Locking Funds — (File: `core/contracts/ContractOwner.sol`)

### Summary

`ContractOwner.withdrawFromDirectDepositV1` attempts to pull native ETH out of a `DirectDepositV1` contract by calling `DirectDepositV1#withdrawNative`, which forwards ETH via `.call{value: balance}("")` to `msg.sender` (i.e., `ContractOwner`). However, `ContractOwner` declares no `receive()` or `fallback()` payable function, so the ETH transfer always reverts. Any native ETH stranded in a `DirectDepositV1` contract is permanently unrecoverable.

### Finding Description

`DirectDepositV1#withdrawNative` pushes ETH to its caller:

```solidity
function withdrawNative() external onlyOwner {
    uint256 balance = address(this).balance;
    (bool success, ) = msg.sender.call{value: balance}("");
    require(success, "Failed to transfer native token to owner");
}
``` [1](#0-0) 

`ContractOwner.withdrawFromDirectDepositV1` is the only caller of this function:

```solidity
uint256 preBalance = address(this).balance;
DirectDepositV1(directDepositV1).withdrawNative();   // sends ETH to address(this)
uint256 postBalance = address(this).balance;
require(postBalance > preBalance, "empty");
(bool success, ) = msg.sender.call{value: postBalance - preBalance}("");
``` [2](#0-1) 

`ContractOwner` inherits only `EIP712Upgradeable` and `OwnableUpgradeable` and defines no `receive()` or `fallback()` function anywhere in the file. [3](#0-2) 

Because `ContractOwner` cannot accept ETH, the `require(success, ...)` inside `withdrawNative` reverts unconditionally, making the entire `withdrawFromDirectDepositV1(subaccount, address(0))` path permanently broken.

### Impact Explanation

Native ETH that accumulates in any `DirectDepositV1` contract — either because the `wrappedNative` wrap failed at construction time (the constructor explicitly handles this case and emits `NativeTokenTransferFailed`, leaving ETH in the DDA) or because ETH was force-sent via `SELFDESTRUCT` — can never be recovered. The `withdrawFromDirectDepositV1` code path for `token == address(0)` always reverts, and there is no alternative withdrawal path for native ETH in `DirectDepositV1`. [4](#0-3) 

### Likelihood Explanation

The `DirectDepositV1` constructor explicitly anticipates a non-zero ETH balance at deploy time and emits `NativeTokenTransferFailed` when the wrap fails, leaving ETH stranded in the contract. Additionally, any ETH force-sent via `SELFDESTRUCT` to an existing DDA cannot be wrapped. Both are realistic on-chain scenarios. Once ETH is stranded, the owner's only recovery path (`withdrawFromDirectDepositV1` with `token == address(0)`) always reverts. [5](#0-4) 

### Recommendation

Add a `receive() external payable {}` function to `ContractOwner` so it can accept the ETH forwarded by `DirectDepositV1#withdrawNative`. The existing logic in `withdrawFromDirectDepositV1` already re-forwards the received ETH to `msg.sender`, so no further changes are needed. [6](#0-5) 

### Proof of Concept

1. A `DirectDepositV1` is deployed (via `createDirectDepositV1`) with a non-zero ETH balance, or ETH is force-sent to it via `SELFDESTRUCT`.
2. The `receive()` hook in `DirectDepositV1` wraps ETH to `wrappedNative`; if that call fails, ETH remains in the DDA and `NativeTokenTransferFailed` is emitted.
3. The owner calls `ContractOwner.withdrawFromDirectDepositV1(subaccount, address(0))`.
4. Internally, `DirectDepositV1#withdrawNative` executes `ContractOwner.call{value: balance}("")`.
5. `ContractOwner` has no `receive()` function → the low-level call returns `success = false`.
6. `require(success, "Failed to transfer native token to owner")` reverts.
7. ETH remains permanently locked in the `DirectDepositV1` contract with no recovery path. [1](#0-0) [2](#0-1)

        external
        onlyOwner

        address payable directDepositV1 = directDepositV1Address[subaccount];
        require(directDepositV1 != address(0), "no dda");
        if (token == address(0)) {
            uint256 preBalance = address(this).balance;
            DirectDepositV1(directDepositV1).withdrawNative();
            uint256 postBalance = address(this).balance;
            require(postBalance > preBalance, "empty");
            (bool success, ) = msg.sender.call{value: postBalance - preBalance}(
                ""
            );
            require(success, "xfer");

            DirectDepositV1(directDepositV1).withdraw(IIERC20Base(token));
            uint256 postBalance = IERC20Base(token).balanceOf(address(this));
            require(postBalance > preBalance, "empty");
            IERC20Base(token).safeTransfer(
                msg.sender,
                postBalance - preBalance
