### Title
Unconditional `LastKnownBonded = true` in `TierRedelegate` Causes Bonus Overpayment for Unbonded Validator Periods — (`x/tieredrewards/keeper/msg_server.go`)

---

### Summary

`TierRedelegate` calls `pos.UpdateBonusCheckpoints(latestSeq, blockTime, true)` unconditionally, setting `LastKnownBonded = true` regardless of the destination validator's actual bonded state. When the destination validator is unbonding at the time of redelegation and later re-bonds, the next `processEventsAndClaimBonus` call treats the entire post-redelegate unbonded gap as a bonded segment and pays bonus for it, violating the invariant that bonus accrues only during bonded validator periods.

---

### Finding Description

**Root cause — `msg_server.go` line 263:**

```go
pos.UpdateBonusCheckpoints(latestSeq, sdkCtx.BlockTime(), true)
```

`UpdateBonusCheckpoints` sets three fields atomically:

```go
// types/position.go:65-69
func (p *Position) UpdateBonusCheckpoints(lastEventSeq uint64, t time.Time, lastKnownBonded bool) {
    p.LastEventSeq = lastEventSeq
    p.LastBonusAccrual = t
    p.LastKnownBonded = lastKnownBonded
}
```

The third argument is hardcoded `true`. No check is made against the destination validator's actual status.

**`validateRedelegatePosition` has no bonded-state guard:**

```go
// msg_validate.go:67-103
func (k Keeper) validateRedelegatePosition(ctx context.Context, pos types.PositionState, owner, dstValidator string) error {
    // checks: owner, delegated, same-validator, exit-lock, close-only, active-redelegation
    // NO check: dstValidator.IsBonded()
    return nil
}
```

Redelegation to an unbonding validator is fully permitted by the SDK and by this module.

**Exploit path — step by step:**

1. Validator V2 is bonded, then jailed/unbonded → `AfterValidatorBeginUnbonding` hook records UNBOND event at seq=N.
2. Attacker calls `MsgTierRedelegate` targeting V2 (currently unbonding). This is accepted — no guard rejects it.
3. `TierRedelegate` reads `latestSeq = N` (the UNBOND event seq) and writes:
   - `pos.LastEventSeq = N`
   - `pos.LastBonusAccrual = T_redelegate`
   - `pos.LastKnownBonded = true` ← **incorrect**
4. V2 is unjailed and re-bonds → `AfterValidatorBonded` hook fires. At this point the position IS counted on V2 (`getPositionCountForValidator > 0`), so a BOND event is recorded at seq=N+1, timestamp=T_bond.
5. Attacker calls `MsgClaimTierRewards`. `processEventsAndClaimBonus` runs:
   - `bonded = pos.LastKnownBonded = true`
   - `segmentStart = T_redelegate`
   - Events since seq N: `[BOND event, seq=N+1, T_bond]`
   - Loop iteration: `bonded == true` → **computes bonus for `[T_redelegate, T_bond]`** — the entire unbonded gap
   - After BOND event: `bonded = true`, `segmentStart = T_bond`
   - Final segment: `bonded && val.IsBonded()` → computes bonus for `[T_bond, now]` (correct)

The segment `[T_redelegate, T_bond]` was entirely unbonded, but bonus is paid for it.

**The final-segment guard (`val.IsBonded()` at line 206) does NOT protect against this case** — it only guards the trailing open segment after all events are processed. The overpayment happens inside the event loop, where the BOND event is the first event seen and `bonded = true` causes the pre-BOND segment to be treated as bonded.

---

### Impact Explanation

- **Direct economic loss:** Bonus coins are transferred from `RewardsPoolName` to the attacker's address for a period during which the validator was not bonded. The amount scales with position size, bonus APY, and the length of the unbonded gap.
- **Affected module account:** `types.RewardsPoolName` (line 239 of `claim_rewards.go`).
- **Repeatability:** The attacker can repeat this by redelegating to a validator that cycles through unbonding/bonding states.

---

### Likelihood Explanation

- Redelegating to an unbonding validator is a standard Cosmos SDK operation — no special privilege is required.
- Validators enter unbonding state routinely (jailing, self-unbonding). The window between unbonding and re-bonding (unjailing) can be hours to days.
- The attacker only needs to time the `MsgTierRedelegate` to land while the destination validator is unbonding, then wait for it to re-bond.

---

### Recommendation

In `TierRedelegate`, replace the hardcoded `true` with the actual bonded state of the destination validator:

```go
dstVal, err := ms.stakingKeeper.GetValidator(ctx, dstValAddr)
if err != nil {
    return nil, err
}
pos.UpdateBonusCheckpoints(latestSeq, sdkCtx.BlockTime(), dstVal.IsBonded())
```

This mirrors the correct behavior used in `processEventsAndClaimBonus` at line 206, which already defensively checks `val.IsBonded()` before computing the trailing segment.

### Proof of Concept

```go
func (s *KeeperSuite) TestTierRedelegate_UnbondingDst_BonusOverpay() {
    lockAmount := sdkmath.NewInt(sdk.DefaultPowerReduction.Int64())
    pos := s.setupNewTierPosition(lockAmount, false)
    delAddr := sdk.MustAccAddressFromBech32(pos.Owner)
    _, bondDenom := s.getStakingData()
    s.fundRewardsPool(sdkmath.NewInt(1_000_000_000), bondDenom)
    msgServer := keeper.NewMsgServerImpl(s.keeper)

    // Create V2 and immediately jail/unbond it.
    dstValAddr, _ := s.createSecondValidator()
    tUnbond := s.ctx.BlockTime().Add(1 * time.Hour)
    s.ctx = s.ctx.WithBlockTime(tUnbond).WithBlockHeight(s.ctx.BlockHeight() + 1)
    s.jailAndUnbondValidator(dstValAddr) // records UNBOND event seq=1 on V2

    // Redelegate to unbonding V2.
    tRedelegate := tUnbond.Add(1 * time.Hour)
    s.ctx = s.ctx.WithBlockTime(tRedelegate).WithBlockHeight(s.ctx.BlockHeight() + 1)
    _, err := msgServer.TierRedelegate(s.ctx, &types.MsgTierRedelegate{
        Owner: delAddr.String(), PositionId: pos.Id, DstValidator: dstValAddr.String(),
    })
    s.Require().NoError(err)

    // Verify LastKnownBonded is incorrectly set to true.
    pos, _ = s.keeper.GetPositionState(s.ctx, pos.Id)
    s.Require().True(pos.LastKnownBonded, "bug: LastKnownBonded=true even though dst is unbonding")

    // Unjail and rebond V2 — records BOND event seq=2.
    tBond := tRedelegate.Add(24 * time.Hour)
    s.ctx = s.ctx.WithBlockTime(tBond).WithBlockHeight(s.ctx.BlockHeight() + 1)
    val, _ := s.app.StakingKeeper.GetValidator(s.ctx, dstValAddr)
    consAddr, _ := val.GetConsAddr()
    s.app.StakingKeeper.Unjail(s.ctx, consAddr)
    s.app.StakingKeeper.ApplyAndReturnValidatorSetUpdates(s.ctx)

    // Advance time and claim.
    tNow := tBond.Add(1 * time.Hour)
    s.ctx = s.ctx.WithBlockTime(tNow).WithBlockHeight(s.ctx.BlockHeight() + 1)

    balBefore := s.app.BankKeeper.GetBalance(s.ctx, delAddr, bondDenom)
    _, err = msgServer.ClaimTierRewards(s.ctx, &types.MsgClaimTierRewards{
        Owner: delAddr.String(), PositionIds: []uint64{pos.Id},
    })
    s.Require().NoError(err)
    balAfter := s.app.BankKeeper.GetBalance(s.ctx, delAddr, bondDenom)

    // Assert: bonus should be ZERO for the unbonded gap [tRedelegate, tBond].
    // With the bug, bonus is non-zero (overpayment).
    s.Require().True(balAfter.Amount.Equal(balBefore.Amount),
        "FAIL: bonus paid for unbonded period — overpayment from RewardsPoolName")
}
```

```go
func (k Keeper) validateRedelegatePosition(ctx context.Context, pos types.PositionState, owner, dstValidator string) error {
	if !pos.IsOwner(owner) {
		return types.ErrNotPositionOwner
	}

	if !pos.IsDelegated() {
		return types.ErrPositionNotDelegated
	}

	if pos.Delegation.ValidatorAddress == dstValidator {
		return types.ErrRedelegationToSameValidator
	}

	sdkCtx := sdk.UnwrapSDKContext(ctx)
	if pos.HasTriggeredExit() && pos.CompletedExitLockDuration(sdkCtx.BlockTime()) {
		return types.ErrExitLockDurationElapsed
	}

	tier, err := k.getTier(ctx, pos.TierId)
	if err != nil {
		return err
	}

	if tier.IsCloseOnly() {
		return types.ErrTierIsCloseOnly
	}

	isRedelegating, err := k.isRedelegating(ctx, pos.DelegatorAddress)
	if err != nil {
		return err
	}
	if isRedelegating {
		return errorsmod.Wrapf(types.ErrActiveRedelegation, "position %d has an active redelegation", pos.Id)
	}

	return nil
}
```

```go
func (h Hooks) AfterValidatorBonded(ctx context.Context, _ sdk.ConsAddress, valAddr sdk.ValAddress) error {
	count, err := h.k.getPositionCountForValidator(ctx, valAddr)
	if err != nil {
		return err
	}
	if count == 0 {
		return nil
	}

	tokensPerShare, err := h.k.getTokensPerShare(ctx, valAddr)
	if err != nil {
		return err
	}

	sdkCtx := sdk.UnwrapSDKContext(ctx)
	_, err = h.k.appendValidatorEvent(ctx, valAddr, types.ValidatorEvent{
		Height:         sdkCtx.BlockHeight(),
		Timestamp:      sdkCtx.BlockTime(),
		EventType:      types.ValidatorEventType_VALIDATOR_EVENT_TYPE_BOND,
		TokensPerShare: tokensPerShare,
		ReferenceCount: count,
	})
	return err
```
