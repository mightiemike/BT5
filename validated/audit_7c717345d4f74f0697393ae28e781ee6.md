Audit Report

## Title
Router-mediated swaps bypass per-user `SwapAllowlistExtension` gate — (`metric-periphery/contracts/extensions/SwapAllowlistExtension.sol`)

## Summary
`SwapAllowlistExtension.beforeSwap` gates swaps on `allowedSwapper[pool][sender]`, where `sender` is the immediate `msg.sender` of `pool.swap()`. When a user swaps through `MetricOmmSimpleRouter`, the pool's `msg.sender` is the router contract, so the extension evaluates the router's allowlist entry rather than the end-user's. Any user individually blocked can bypass the block by routing through an allowlisted router.

## Finding Description
`MetricOmmPool.swap()` captures `msg.sender` and passes it verbatim as the `sender` argument to `_beforeSwap`: [1](#0-0) 

`_beforeSwap` forwards `sender` unchanged into every configured extension via `_callExtensionsInOrder`. `SwapAllowlistExtension.beforeSwap` then evaluates: [2](#0-1) 

Here `msg.sender` is the pool (the extension's caller) and `sender` is whoever called `pool.swap()`. When the call originates from `MetricOmmSimpleRouter.exactInputSingle`, the router calls `pool.swap()` directly: [3](#0-2) 

So the pool sees `msg.sender = router`. The extension check becomes `allowedSwapper[pool][router]`, not `allowedSwapper[pool][end_user]`. If the pool admin has allowlisted the router (the natural configuration for supporting periphery swaps), the check passes unconditionally for every user of that router, including those explicitly blocked via `setAllowedToSwap(..., user, false)`.

No existing guard compensates: `allowAllSwappers[pool]` is a separate flag, and `isAllowedToSwap` mirrors the same flawed logic. The router stores the original `msg.sender` only in transient storage for payment purposes (`_setNextCallbackContext`), never encoding it into `extensionData` for the extension to inspect. [4](#0-3) 

## Impact Explanation
The `SwapAllowlistExtension` is documented as gating `swap` **by swapper address, per pool**. Allowlisting the router to enable standard periphery trading simultaneously and silently removes per-user enforcement for all users of that router. A pool admin who intends to block specific addresses (e.g., sanctioned wallets, competitors, or during incident response) cannot achieve that goal once the router is allowlisted. The core access-control invariant of the extension — that `allowedSwapper[pool][user] == false` prevents `user` from swapping — is broken for the router path. This matches the "Allowlist path" audit pivot: swap allowlist checks must cover the exact actor/action intended and cannot be bypassed through the router.

## Likelihood Explanation
Allowlisting the router is the natural, expected configuration for any pool that wants to support the standard periphery swap flow. The pool admin has no on-chain signal that doing so opens the gate to all router users. The bypass requires no special privileges — any EOA can call `MetricOmmSimpleRouter.exactInputSingle` directly. [5](#0-4) 

## Recommendation
Gate on the economic initiator, not the immediate `pool.swap()` caller. Two concrete options:

1. **Pass the original user through the router**: Have `MetricOmmSimpleRouter` encode the original `msg.sender` into `extensionData`, and have `SwapAllowlistExtension.beforeSwap` read and verify it when `sender` is a known router.
2. **Check both router and user**: Extend the allowlist check so that when `sender` is a known router, the extension additionally verifies the user identity supplied in `extensionData`. [6](#0-5) 

## Proof of Concept
```solidity
// Setup:
// 1. Deploy pool with SwapAllowlistExtension configured on beforeSwap.
// 2. Pool admin allowlists the router (standard config):
//    swapExtension.setAllowedToSwap(address(pool), address(router), true);
// 3. Pool admin does NOT allowlist the attacker (default false):
//    // allowedSwapper[pool][attacker] == false

// Attack:
vm.prank(attacker);
router.exactInputSingle(IMetricOmmSimpleRouter.ExactInputSingleParams({
    pool: address(pool),
    tokenIn: address(token0),
    tokenOut: address(token1),
    zeroForOne: true,
    amountIn: 1000,
    amountOutMinimum: 0,
    recipient: attacker,
    deadline: block.timestamp + 1,
    priceLimitX64: 0,
    extensionData: ""
}));
// Pool sees msg.sender = router
// Extension checks allowedSwapper[pool][router] = true → swap succeeds
// Assert: swap did not revert despite attacker being individually blocked
```
