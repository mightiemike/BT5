### Title
Strict `GT` Instead of `GTE` in Supply Cap Check Allows One Extra Block of Minting Beyond `MaxSupply` - (File: `x/inflation/abci.go`)

### Summary
`BeginBlocker` in the inflation module uses a strict `>` comparison (`totalsupply.GT(maxsupply)`) when enforcing the `MaxSupply` cap. When circulating supply equals `MaxSupply` exactly, the check passes without halting. The Mint module then mints additional tokens in the next block, pushing supply above `MaxSupply` before the halt finally triggers. The governance-set hard cap is therefore not enforced at the stated boundary.

### Finding Description
In `x/inflation/abci.go`, the supply cap enforcement reads:

```go
if maxsupply.IsPositive() && totalsupply.GT(maxsupply) {
    panic(fmt.Sprintf("the total supply has exceeded the maximum supply: %s > %s", totalsupply, maxsupply))
}
``` [1](#0-0) 

The condition `totalsupply.GT(maxsupply)` is `false` when `totalsupply == maxsupply`, so the chain does not halt at that boundary. The test suite explicitly encodes this as the intended behavior with the comment `// GT, not GTE — should NOT panic`: [2](#0-1) 

Per ADR-007, the BeginBlocker runs **after** the Mint module has already minted new tokens for the block:

```
Each block:
  1. Mint module mints new tokens using the decayed inflation rate
  2. Inflation module checks: circulating supply <= max supply
``` [3](#0-2) 

The execution sequence is therefore:
- **Block N**: Mint module mints → supply reaches exactly `MaxSupply` → `GT` check is `false` → no halt → block commits.
- **Block N+1**: Mint module mints again → supply is now `MaxSupply + block_mint_amount` → `GT` check is `true` → chain halts.

One full block's worth of minting is committed to state beyond the governance-approved cap before the halt fires.

### Impact Explanation
The corrupted value is the circulating supply of the bond denomination. Tokens are minted and permanently committed to bank state in excess of `MaxSupply` by the amount minted in one block. Because `MaxSupply` is a governance-enforced hard cap on circulating supply, any supply above it represents unbacked minting that violates the tokenomics invariant the governance vote was intended to enforce.

### Likelihood Explanation
This triggers automatically through normal block production with no attacker action required. As the chain approaches its supply cap (a predictable, observable on-chain event), the off-by-one fires deterministically on the block where supply first equals `MaxSupply`. Likelihood is **high** — it is a guaranteed outcome whenever the cap is reached.

### Recommendation
Change the comparison from `GT` to `GTE` so the chain halts as soon as circulating supply reaches `MaxSupply`:

```go
// Before (incorrect):
if maxsupply.IsPositive() && totalsupply.GT(maxsupply) {

// After (correct):
if maxsupply.IsPositive() && totalsupply.GTE(maxsupply) {
``` [4](#0-3) 

Update `TestBeginBlocker_SupplyExactlyAtMax` to assert that the chain **does** panic when supply equals `MaxSupply`, consistent with the corrected semantics.

### Proof of Concept

1. Governance sets `MaxSupply = S` where `S` is the current circulating supply.
2. Block N begins: Mint module mints `m > 0` tokens → circulating supply becomes `S + m`.  
   Wait — more precisely: if at the start of block N the supply is `S - m` (just below cap), Mint mints `m` tokens, supply becomes `S`. Inflation BeginBlocker runs: `S.GT(S)` → `false` → no halt. Block N commits with supply = `S`.
3. Block N+1 begins: Mint module mints `m'` tokens → supply becomes `S + m'`. Inflation BeginBlocker runs: `(S + m').GT(S)` → `true` → chain halts.
4. State at halt contains `m'` tokens minted above the governance-approved `MaxSupply`.

The root cause is exclusively in `x/inflation/abci.go` line 37. No privileged access, no special transaction, and no configuration beyond a positive `MaxSupply` is required. [5](#0-4)

// TestBeginBlocker_SupplyExactlyAtMax verifies that the chain does NOT halt
// when total supply equals max supply exactly (the check is strictly GT).
func TestBeginBlocker_SupplyExactlyAtMax(t *testing.T) {
	app := testutil.Setup(false, nil)
	ctx := app.BaseApp.NewContext(false).WithBlockHeader(tmproto.Header{ChainID: testutil.ChainID})

	totalSupply, _, err := app.InflationKeeper.GetSupplyAndDenom(ctx)
	require.NoError(t, err)

	// Set max_supply = total_supply (exactly at the boundary)
	params := types.DefaultParams()
	params.MaxSupply = totalSupply
	err = app.InflationKeeper.SetParams(ctx, params)
	require.NoError(t, err)

	// GT, not GTE — should NOT panic
	require.NotPanics(t, func() {
		err := inflation.BeginBlocker(ctx, app.InflationKeeper)
		require.NoError(t, err)
	})
}
