### Title
`ValidResourceBounds` type-switching in protobuf round-trip silently changes transaction hash preimage and gas computation mode — (`crates/apollo_protobuf/src/converters/transaction.rs`)

---

### Summary

The `TryFrom<protobuf::ResourceBounds> for ValidResourceBounds` implementation uses a zero-value heuristic to reconstruct the `L1Gas` vs `AllResources` variant. Because the gateway accepts `AllResourceBounds { l1_gas: X, l2_gas: 0, l1_data_gas: 0 }` as a valid V3 transaction and computes its hash under the `AllResources` path (which includes `L1DataGas=0` in the Poseidon preimage), a protobuf round-trip silently converts the variant to `L1Gas(X)`. This produces a different hash preimage and a different `GasVectorComputationMode`, breaking hash validation and gas accounting on any node that receives the transaction via P2P state sync.

---

### Finding Description

**Step 1 — Gateway accepts `AllResources(l1=X, l2=0, l1_data=0)`**

The stateless validator accepts `AllResourceBounds { l1_gas: NON_EMPTY_RESOURCE_BOUNDS, ..Default::default() }` (l2_gas=0, l1_data_gas=0) as a valid V3 transaction: [1](#0-0) 

**Step 2 — Hash computed under `AllResources` path**

`InternalRpcInvokeTransactionV3` stores `resource_bounds: AllResourceBounds` and its `DeclareTransactionV3Trait` impl always wraps it as `ValidResourceBounds::AllResources(...)`: [2](#0-1) 

`get_tip_resource_bounds_hash` includes `L1DataGas` in the Poseidon preimage only for `AllResources`: [3](#0-2) 

So for `AllResources(l1=X, l2=0, l1_data=0)`:
```
tip_resource_hash = poseidon(tip, concat(L1_GAS, X), concat(L2_GAS, 0), concat(L1_DATA_GAS, 0))
```
(4 elements)

**Step 3 — Protobuf serialization erases the variant**

`From<ValidResourceBounds> for protobuf::ResourceBounds` serializes `L1Gas` and `AllResources(l1, 0, 0)` identically — both produce `{ l1_gas: X, l2_gas: 0, l1_data_gas: 0 }` on the wire: [4](#0-3) 

**Step 4 — Deserialization silently switches the variant**

`TryFrom<protobuf::ResourceBounds> for ValidResourceBounds` reconstructs `L1Gas` whenever `l2_gas=0 AND l1_data_gas=0`: [5](#0-4) 

The comment on line 426 acknowledges this is a compatibility workaround (`// TODO(Shahak): Assert data gas is not none once we remove support for 0.13.2`), but it creates a non-bijective round-trip for any `AllResources` transaction with zero L2 and L1DataGas bounds.

**Step 5 — Deserialized `InvokeTransactionV3` uses `L1Gas` variant**

`TryFrom<protobuf::InvokeV3> for InvokeTransactionV3` calls `ValidResourceBounds::try_from(...)`: [6](#0-5) 

After deserialization, the canonical `InvokeTransactionV3` holds `resource_bounds: L1Gas(X)` instead of `AllResources(X, 0, 0)`.

**Step 6 — Hash preimage diverges**

For `L1Gas(X)`:
```
tip_resource_hash = poseidon(tip, concat(L1_GAS, X), concat(L2_GAS, 0))
```
(3 elements — `L1DataGas` is absent) [7](#0-6) 

The two hashes are structurally different Poseidon outputs. Any node that recomputes the hash from the deserialized transaction (e.g., `validate_transaction_hash`) will produce `H_l1 ≠ H_all`.

**Step 7 — Gas computation mode changes**

`get_gas_vector_computation_mode()` returns `All` for `AllResources` and `NoL2Gas` for `L1Gas`: [8](#0-7) 

This changes fee enforcement: `AllResources` checks L1, L2, and L1DataGas bounds; `L1Gas` checks only L1 gas. A transaction re-executed under `NoL2Gas` mode is not bounded by the L2 gas limit the user signed. [9](#0-8) 

---

### Impact Explanation

- **Wrong hash**: Any synced node that calls `validate_transaction_hash` on a deserialized `InvokeTransactionV3` (or `DeployAccountTransactionV3`) with `AllResources(l1, 0, 0)` computes a hash that does not match the sequencer's stored hash. This causes block/transaction hash validation to fail on synced nodes, or — if validation is skipped — the node silently operates on a transaction with a mismatched hash identity.

- **Wrong gas accounting**: The `GasVectorComputationMode` switches from `All` to `NoL2Gas`, meaning L2 gas is not tracked or bounded during reexecution. A transaction that consumed L2 gas is re-executed without enforcing the L2 gas bound the user signed, constituting incorrect fee/resource accounting with economic impact.

Both impacts fall within the allowed scope:
- *Critical: Wrong state, receipt, or revert result from blockifier/execution logic for accepted input.*
- *High: RPC execution, fee estimation, tracing, or simulation returns an authoritative-looking wrong value.*

---

### Likelihood Explanation

The trigger is fully unprivileged: any user can submit a V3 transaction with `AllResourceBounds { l1_gas: X, l2_gas: 0, l1_data_gas: 0 }`. The gateway explicitly accepts this (confirmed by the `valid_l1_gas` test case). The protobuf round-trip occurs on every P2P state sync. No special permissions, malformed bytes, or adversarial peers are required.

---

### Recommendation

1. **Preserve the variant tag explicitly in protobuf.** Add a boolean or enum field to `protobuf::ResourceBounds` (e.g., `is_all_resources: bool`) so the `L1Gas` vs `AllResources` distinction survives serialization without relying on zero-value heuristics.

2. **Alternatively, reject `AllResources(l1, 0, 0)` at the gateway.** If `l2_gas=0 AND l1_data_gas=0` is semantically equivalent to `L1Gas`, the gateway should normalize the variant before computing the hash, so the hash is always computed under `L1Gas` for such inputs.

3. **Remove the `TODO` workaround** in `TryFrom<protobuf::ResourceBounds> for ValidResourceBounds` once 0.13.2 support is dropped, and replace it with a strict conversion that errors on ambiguous inputs.

### Proof of Concept

```rust
use starknet_api::transaction::fields::{
    AllResourceBounds, ResourceBounds, GasAmount, GasPrice, ValidResourceBounds,
};
use apollo_protobuf::protobuf;
use std::convert::TryFrom;

fn main() {
    // Step 1: Construct AllResources(l1=X, l2=0, l1_data=0) — accepted by gateway
    let original = ValidResourceBounds::AllResources(AllResourceBounds {
        l1_gas: ResourceBounds {
            max_amount: GasAmount(100),
            max_price_per_unit: GasPrice(1),
        },
        l2_gas: ResourceBounds::default(),      // zero
        l1_data_gas: ResourceBounds::default(), // zero
    });

    // Step 2: Serialize to protobuf (as happens during P2P state sync)
    let proto: protobuf::ResourceBounds = original.into();
    // proto.l2_gas = Some(ResourceLimits { max_amount: 0, max_price_per_unit: 0 })
    // proto.l1_data_gas = Some(ResourceLimits { max_amount: 0, max_price_per_unit: 0 })

    // Step 3: Deserialize on receiving node
    let deserialized = ValidResourceBounds::try_from(proto).unwrap();

    // Step 4: Variant has changed!
    assert!(matches!(deserialized, ValidResourceBounds::L1Gas(_)));
    // original was AllResources, deserialized is L1Gas

    // Step 5: Hash preimage differs
    // original hash includes L1DataGas=0 in poseidon → H_all
    // deserialized hash excludes L1DataGas → H_l1
    // H_all ≠ H_l1

    // Step 6: Gas mode differs
    use starknet_api::transaction::fields::GasVectorComputationMode;
    assert_eq!(
        original.get_gas_vector_computation_mode(),
        GasVectorComputationMode::All
    );
    assert_eq!(
        deserialized.get_gas_vector_computation_mode(),
        GasVectorComputationMode::NoL2Gas // wrong — L2 gas no longer bounded
    );
}
```

The divergence in `get_tip_resource_bounds_hash` output is the root cause: `AllResources` hashes 4 elements (tip + L1 + L2 + L1DataGas) while `L1Gas` hashes 3 elements (tip + L1 + L2), producing structurally different Poseidon digests for the same user-signed resource bounds. pub fn get_tip_resource_bounds_hash(
    resource_bounds: &ValidResourceBounds,
    tip: &Tip,
) -> Result<Felt, StarknetApiError> {
    let l1_resource_bounds = resource_bounds.get_l1_bounds();
    let l2_resource_bounds = resource_bounds.get_l2_bounds();

    // L1 and L2 gas bounds always exist.
    // Old V3 txs always have L2 gas bounds of zero, but they exist.
    let mut resource_felts = vec![
        get_concat_resource(&l1_resource_bounds, L1_GAS)?,
        get_concat_resource(&l2_resource_bounds, L2_GAS)?,
    ];

    // For new V3 txs, need to also hash the data gas bounds.
    resource_felts.extend(match resource_bounds {
        ValidResourceBounds::L1Gas(_) => vec![],
        ValidResourceBounds::AllResources(all_resources) => {
            vec![get_concat_resource(&all_resources.l1_data_gas, L1_DATA_GAS)?]
        }
    });

    Ok(HashChain::new().chain(&tip.0.into()).chain_iter(resource_felts.iter()).get_poseidon_hash())
}

            ValidResourceBounds::L1Gas(l1_gas)
        } else {
            ValidResourceBounds::AllResources(AllResourceBounds { l1_gas, l2_gas, l1_data_gas })
        })
    }

#[derive(Clone, Copy, Debug, Eq, PartialEq, Hash, Ord, PartialOrd)]
pub enum ValidResourceBounds {
    L1Gas(ResourceBounds), // Pre 0.13.3. Only L1 gas. L2 bounds are signed but never used.
    AllResources(AllResourceBounds),
}

impl From<AllResourceBounds> for ValidResourceBounds {
    fn from(value: AllResourceBounds) -> Self {
        Self::AllResources(value)
    }
}

impl ValidResourceBounds {
    pub fn get_l1_bounds(&self) -> ResourceBounds {
        match self {
            Self::L1Gas(l1_bounds) => *l1_bounds,
            Self::AllResources(AllResourceBounds { l1_gas, .. }) => *l1_gas,
        }
    }

    pub fn get_l2_bounds(&self) -> ResourceBounds {
        match self {
            Self::L1Gas(_) => ResourceBounds::default(),
            Self::AllResources(AllResourceBounds { l2_gas, .. }) => *l2_gas,
        }
    }

    /// Returns the maximum possible fee that can be charged for the transaction.
    /// The computation is saturating, meaning that if the result is larger than the maximum
    /// possible fee, the maximum possible fee is returned.
    pub fn max_possible_fee(&self, tip: Tip) -> Fee {
        match self {
            ValidResourceBounds::L1Gas(l1_bounds) => {
                l1_bounds.max_amount.saturating_mul(l1_bounds.max_price_per_unit)
            }
            ValidResourceBounds::AllResources(AllResourceBounds {
                l1_gas,
                l2_gas,
                l1_data_gas,
            }) => l1_gas
                .max_amount
                .saturating_mul(l1_gas.max_price_per_unit)
                .saturating_add(
                    l2_gas
                        .max_amount
                        .saturating_mul(l2_gas.max_price_per_unit.saturating_add(tip.into())),
                )
                .saturating_add(
                    l1_data_gas.max_amount.saturating_mul(l1_data_gas.max_price_per_unit),
                ),
        }
    }

    pub fn get_gas_vector_computation_mode(&self) -> GasVectorComputationMode {
        match self {
            Self::AllResources(_) => GasVectorComputationMode::All,
            Self::L1Gas(_) => GasVectorComputationMode::NoL2Gas,
        }
    }
