### Title
Hardcoded `paid_fee_on_l1 = Fee(1)` in Consensus L1Handler Conversion Bypasses Blockifier Fee Sufficiency Check — (`crates/apollo_transaction_converter/src/transaction_converter.rs`)

### Summary

When an `L1HandlerTransaction` arrives via the consensus P2P path, `convert_consensus_l1_handler_to_internal_l1_handler` unconditionally substitutes `Fee(1)` for the actual fee paid on L1. The blockifier's only fee guard checks `paid_fee != Fee(0)`, which `Fee(1)` trivially satisfies. The real `paid_fee_on_l1` value — carried by the L1 event and enforced by the Starknet core contract — is silently discarded and replaced with a constant that always passes the check, breaking the fee-amount invariant at the RPC-to-internal conversion boundary.

### Finding Description

`ConsensusTransaction::L1Handler` wraps a bare `transaction::L1HandlerTransaction` that carries no `paid_fee_on_l1` field. During conversion to the executable form the converter hardcodes the fee:

```rust
// crates/apollo_transaction_converter/src/transaction_converter.rs  lines 473-483
fn convert_consensus_l1_handler_to_internal_l1_handler(
    &self,
    tx: transaction::L1HandlerTransaction,
) -> TransactionConverterResult<executable_transaction::L1HandlerTransaction> {
    Ok(executable_transaction::L1HandlerTransaction::create(
        tx,
        &self.chain_id,
        // TODO(Gilad): Change this once we put real value in paid_fee_on_l1.
        Fee(1),
    )?)
}
``` [1](#0-0) 

The blockifier then executes the transaction and performs the only fee check:

```rust
// crates/blockifier/src/transaction/l1_handler_transaction.rs  lines 103-113
let paid_fee = self.paid_fee_on_l1;
// For now, assert only that any amount of fee was paid.
if paid_fee == Fee(0) {
    return Err(TransactionExecutionError::TransactionFeeError(Box::new(
        TransactionFeeError::InsufficientFee { paid_fee, actual_fee: receipt.fee },
    )));
}
``` [2](#0-1) 

Because `Fee(1) != Fee(0)`, the guard is always satisfied regardless of the actual ETH value attached to the originating `sendMessageToL2` call. The `paid_fee_on_l1` field in `executable_transaction::L1HandlerTransaction` is defined to carry the real L1 fee:

```rust
// crates/starknet_api/src/executable_transaction.rs  lines 380-396
pub struct L1HandlerTransaction {
    pub tx: crate::transaction::L1HandlerTransaction,
    pub tx_hash: TransactionHash,
    pub paid_fee_on_l1: Fee,
}
``` [3](#0-2) 

The protobuf `ConsensusTransaction` message does have a `transaction_hash` field (tag 5) but no `paid_fee_on_l1` field, so the actual fee is structurally absent from the consensus wire format and cannot be recovered at the receiving end: [4](#0-3) 

The conversion from protobuf to `ConsensusTransaction` confirms the field is never populated: [5](#0-4) 

### Impact Explanation

Every `L1HandlerTransaction` that enters the sequencer through the consensus path (i.e., during block validation by a non-proposer node, or when the proposer itself converts the transaction for batcher submission) is executed with `paid_fee_on_l1 = Fee(1)`. The blockifier's fee sufficiency check — the sequencer's own last-resort guard that the L1 caller paid a non-zero fee — is unconditionally bypassed. The actual fee amount paid on L1 is never bound to the executable object, so the fee recorded in the `TransactionExecutionInfo` receipt diverges from the real on-chain value. This constitutes incorrect fee/resource accounting with potential economic impact: the sequencer cannot distinguish a transaction that paid 1 wei from one that paid the full required fee, and the receipt emitted to downstream consumers (state sync, RPC, proof generation) carries the wrong `paid_fee_on_l1`.

### Likelihood Explanation

This code path is exercised for every L1Handler transaction included in any block validated through the consensus orchestrator. Any user who calls `sendMessageToL2` on the L1 Starknet core contract triggers an L1 event that eventually reaches this conversion. No special privilege is required.

### Recommendation

Add `paid_fee_on_l1` to the `ConsensusTransaction::L1Handler` variant and to the protobuf `ConsensusTransaction` message (as an additional field alongside `transaction_hash`). Propagate the real fee through `convert_consensus_l1_handler_to_internal_l1_handler` instead of substituting `Fee(1)`. Until the wire format is updated, at minimum validate that the fee retrieved from the local L1 provider for the given transaction hash matches the fee embedded in the executable object before submitting it to the blockifier.

### Proof of Concept

1. User calls `sendMessageToL2` on the Starknet L1 contract paying 1 wei.
2. The L1 events component records the transaction with `paid_fee_on_l1 = 1 wei`.
3. The proposer node fetches the transaction from the L1 provider (which holds the real fee) and broadcasts it as `ConsensusTransaction::L1Handler(tx)` — without `paid_fee_on_l1`.
4. The validating node receives the message and calls `convert_consensus_tx_to_internal_consensus_tx`, which dispatches to `convert_consensus_l1_handler_to_internal_l1_handler`.
5. `executable_transaction::L1HandlerTransaction::create(tx, &self.chain_id, Fee(1))` is called — the real fee is replaced with `Fee(1)`.
6. The blockifier executes the transaction; `paid_fee == Fee(1) != Fee(0)` → the fee check passes unconditionally.
7. The resulting `TransactionExecutionInfo` receipt records `paid_fee_on_l1 = Fee(1)` regardless of the actual L1 fee, and the fee sufficiency invariant is never enforced by the sequencer.

impl TryFrom<protobuf::ConsensusTransaction> for ConsensusTransaction {
    type Error = ProtobufConversionError;
    fn try_from(value: protobuf::ConsensusTransaction) -> Result<Self, Self::Error> {
        let txn = value.txn.ok_or(missing("ConsensusTransaction::txn"))?;
        let txn = match txn {
            protobuf::consensus_transaction::Txn::DeclareV3(txn) => {
                ConsensusTransaction::RpcTransaction(RpcTransaction::Declare(
                    RpcDeclareTransaction::V3(txn.try_into()?),
                ))
            }
            protobuf::consensus_transaction::Txn::DeployAccountV3(txn) => {
                ConsensusTransaction::RpcTransaction(RpcTransaction::DeployAccount(
                    RpcDeployAccountTransaction::V3(txn.try_into()?),
                ))
            }
            protobuf::consensus_transaction::Txn::InvokeV3(txn) => {
                ConsensusTransaction::RpcTransaction(RpcTransaction::Invoke(
                    RpcInvokeTransaction::V3(txn.try_into()?),
                ))
            }
            protobuf::consensus_transaction::Txn::L1Handler(txn) => {
                ConsensusTransaction::L1Handler(txn.try_into()?)
            }
        };
        Ok(txn)
    }
}
