### Title
Malicious P2P Peer Can Misclassify Cairo-0 Class as Cairo-1 by Sending `compiled_class_hash = Some(zero_hash)`, Corrupting Stored `ThinStateDiff` — (`crates/apollo_protobuf/src/converters/state_diff.rs`)

---

### Summary

The `TryFrom<protobuf::StateDiffsResponse> for DataOrFin<StateDiffChunk>` conversion classifies a `DeclaredClass` protobuf message as either `StateDiffChunk::DeclaredClass` (Cairo-1) or `StateDiffChunk::DeprecatedDeclaredClass` (Cairo-0) based solely on whether `compiled_class_hash` is `Some` or `None`. A malicious peer can send `compiled_class_hash = Some(zero_bytes)` to force Cairo-0 class hash `X` into `class_hash_to_compiled_class_hash` instead of `deprecated_declared_classes`. The p2p sync client validates only `state_diff_length` (which is 1 for both variants) and never validates the assembled `ThinStateDiff` against the block header's `state_diff_commitment`. The corrupted state diff is written directly to storage.

---

### Finding Description

**Step 1 — The classification gate (lines 51–60):** [1](#0-0) 

The branch condition is `declared_class.compiled_class_hash.as_ref()` — `Some(_)` → `DeclaredClass`, `None` → `DeprecatedDeclaredClass`. The value inside `Some` is never inspected at this branch point. A peer sending `compiled_class_hash = Some(Hash { elements: [0u8; 32] })` satisfies `Some(_)` and routes to the Cairo-1 path.

**Step 2 — The Cairo-1 converter accepts zero as a valid `CompiledClassHash` (lines 260–272):** [2](#0-1) 

`Felt::ZERO` is a valid `Felt`, so `CompiledClassHash(Felt::ZERO)` is constructed without error.

**Step 3 — `unite_state_diffs` routes the chunk to the wrong field (lines 164–175):** [3](#0-2) 

A `StateDiffChunk::DeclaredClass` inserts into `class_hash_to_compiled_class_hash`; a `StateDiffChunk::DeprecatedDeclaredClass` pushes into `deprecated_declared_classes`. The misclassified chunk lands in the wrong field.

**Step 4 — The only validation is `state_diff_length`, which is identical for both variants:** [4](#0-3) 

`StateDiffChunk::len()` returns `1` for both `DeclaredClass` and `DeprecatedDeclaredClass`. The length check passes. `validate_deprecated_declared_classes_non_conflicting` only checks for duplicate deprecated classes — it does not detect misclassification.

**Step 5 — No `state_diff_commitment` validation before storage write:** [5](#0-4) 

The assembled `ThinStateDiff` is written directly to storage. The block header carries a `state_diff_commitment` (a Poseidon hash), but `parse_data_for_block` never reads or compares it. The `calculate_state_diff_hash` function hashes `class_hash_to_compiled_class_hash` and `deprecated_declared_classes` differently: [6](#0-5) 

`chain_declared_classes` chains `(class_hash, compiled_class_hash)` pairs; `chain_deprecated_declared_classes` chains only `class_hash`. The two produce different Poseidon outputs, so the stored state diff is inconsistent with the committed `state_diff_commitment` in the block header — but this inconsistency is never detected by the sync client.

---

### Impact Explanation

The concrete divergent values are:

| Field | Expected (Cairo-0) | Actual (attacker-induced) |
|---|---|---|
| `deprecated_declared_classes` | `[class_hash_X]` | `[]` |
| `class_hash_to_compiled_class_hash` | `{}` | `{class_hash_X: CompiledClassHash(Felt::ZERO)}` |

Consequences:
- **Wrong state root commitment**: The stored `ThinStateDiff` produces a different `StateDiffCommitment` than the one committed in the block header, silently corrupting the node's view of the canonical state.
- **Wrong class execution path**: Any subsequent execution that looks up `class_hash_X` will find it in `class_hash_to_compiled_class_hash` with `compiled_class_hash = 0` and attempt to execute it as a Cairo-1 class, rather than as a deprecated Cairo-0 class. This falls under "Critical. Wrong compiled class / class hash selected for execution."

---

### Likelihood Explanation

The attacker only needs to be a reachable p2p peer that the syncing node selects for a state diff query covering a block that contains at least one deprecated (Cairo-0) declared class. No special privileges are required. The attack is silent — the sync client reports no error and advances the state marker normally.

---

### Recommendation

Add a `state_diff_commitment` validation step in `parse_data_for_block` after assembling the full `ThinStateDiff`, before returning `Ok(Some(...))`. Compute `calculate_state_diff_hash(&result)` and compare it against `block_header.state_diff_commitment`. If they differ, return `Err(ParseDataError::BadPeer(...))`.

Alternatively (or additionally), add a zero-value guard in the `Some(_compiled_class_hash)` branch of the `TryFrom<protobuf::StateDiffsResponse> for DataOrFin<StateDiffChunk>` conversion to reject `compiled_class_hash = Some(zero_hash)` as an invalid Cairo-1 declaration, consistent with the Starknet protocol specification that `compiled_class_hash` must be a non-zero Poseidon hash of the compiled class.

---

### Proof of Concept

```rust
use apollo_protobuf::protobuf::{self, Felt252};
use apollo_protobuf::sync::{DataOrFin, StateDiffChunk};

#[test]
fn zero_compiled_class_hash_misclassifies_cairo0_as_cairo1() {
    let class_hash_bytes = [0xABu8; 32];
    let zero_hash = protobuf::Hash { elements: vec![0u8; 32] };

    let proto_msg = protobuf::StateDiffsResponse {
        state_diff_message: Some(
            protobuf::state_diffs_response::StateDiffMessage::DeclaredClass(
                protobuf::DeclaredClass {
                    class_hash: Some(protobuf::Hash { elements: class_hash_bytes.to_vec() }),
                    compiled_class_hash: Some(zero_hash), // Some(zero) — not None
                },
            ),
        ),
    };

    let result: DataOrFin<StateDiffChunk> = proto_msg.try_into().unwrap();
    // Asserts the result is DeclaredClass (Cairo-1), NOT DeprecatedDeclaredClass (Cairo-0)
    assert!(
        matches!(result.0, Some(StateDiffChunk::DeclaredClass(_))),
        "Cairo-0 class was misclassified as Cairo-1 due to Some(zero_hash)"
    );
}
```

pub fn calculate_state_diff_hash(state_diff: &ThinStateDiff) -> StateDiffCommitment {
    let mut hash_chain = HashChain::new();
    hash_chain = hash_chain.chain(&STARKNET_STATE_DIFF0);
    hash_chain = chain_deployed_contracts(&state_diff.deployed_contracts, hash_chain);
    hash_chain = chain_declared_classes(&state_diff.class_hash_to_compiled_class_hash, hash_chain);
    hash_chain =
        chain_deprecated_declared_classes(&state_diff.deprecated_declared_classes, hash_chain);
