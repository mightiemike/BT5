Looking at the codebase, I need to trace the hash domain boundary between `ValidResourceBounds::AllResources` and `ValidResourceBounds::L1Gas` across the protobuf conversion path.

The key finding is in `crates/apollo_protobuf/src/converters/transaction.rs` at the `TryFrom<protobuf::ResourceBounds> for ValidResourceBounds` conversion, combined with the conditional hash preimage in `get_tip_resource_bounds_hash`.

### Title
Protobuf `ValidResourceBounds` Reconstruction Silently Downgrades `AllResources` to `L1Gas` When `l2_gas` and `l1_data_gas` Are Zero, Producing a Different Transaction Hash - (`crates/apollo_protobuf/src/converters/transaction.rs`)

### Summary

The protobuf deserializer for `ValidResourceBounds` infers the variant from field values: if both `l2_gas` and `l1_data_gas` are zero it emits `L1Gas`, otherwise `AllResources`. Because `get_tip_resource_bounds_hash` includes an extra `L1_DATA_GAS` element only for `AllResources`, a valid gateway-submitted `InvokeV3` transaction carrying `AllResources { l1_gas: X, l2_gas: 0, l1_data_gas: 0 }` is assigned hash H₁ at submission time, but any node that reconstructs it from protobuf recomputes hash H₂ ≠ H₁. The two hashes diverge because the hash preimage has a different element count.

### Finding Description

**Step 1 – Submission path (correct hash H₁)**

`RpcInvokeTransactionV3` always carries `resource_bounds: AllResourceBounds`. The gateway's stateless validator explicitly accepts `AllResourceBounds { l1_gas: NON_EMPTY, l2_gas: 0, l1_data_gas: 0 }` as valid: [1](#0-0) 

When the transaction is converted to `InvokeTransactionV3`, `resource_bounds` becomes `ValidResourceBounds::AllResources(...)`: [2](#0-1) 

`get_tip_resource_bounds_hash` is called with `AllResources`. It always appends the `L1_DATA_GAS` element, even when its value is zero: [3](#0-2) 

So the hash preimage for `AllResources { l1_gas: X, l2_gas: 0, l1_data_gas: 0 }` is:
```
H(tip, L1_GAS_concat(X), L2_GAS_concat(0), L1_DATA_GAS_concat(0))   ← 4 elements
```

**Step 2 – Protobuf round-trip (wrong hash H₂)**

When the block is synced via P2P, `InvokeTransactionV3` is serialized to `protobuf::InvokeV3` and then deserialized. The deserializer applies this inference rule: [4](#0-3) 

Because `l2_gas.is_zero() && l1_data_gas.is_zero()` is true, the result is `ValidResourceBounds::L1Gas(l1_gas)` — a different variant.

`get_tip_resource_bounds_hash` for `L1Gas` omits the `L1_DATA_GAS` element entirely: [5](#0-4) 

So the recomputed hash preimage is:
```
H(tip, L1_GAS_concat(X), L2_GAS_concat(0))   ← 3 elements
```

H₁ ≠ H₂. The full transaction hash diverges: [6](#0-5) 

**Step 3 – Divergence confirmed by the Cairo OS**

The Cairo OS hash function mirrors the Rust logic exactly — it also conditionally appends `proof_facts` and uses the same resource-bounds packing. The OS would compute H₁ (the original hash), while a syncing node stores H₂: [7](#0-6) 

### Impact Explanation

Any node that receives the block via P2P protobuf sync reconstructs the transaction with `L1Gas` and recomputes a hash that differs from the one the sequencer committed. Depending on whether the sync layer re-validates the hash:

- **If hash is re-validated**: the syncing node rejects the block, causing a permanent sync failure for any block containing such a transaction.
- **If hash is not re-validated**: the syncing node stores the transaction under the wrong hash key, corrupting its transaction index and receipt lookup.

This matches: **High — Transaction conversion or signature/hash logic binds the wrong hash type or executable payload.**

### Likelihood Explanation

The trigger is a valid, unprivileged `InvokeV3` transaction with `AllResources` where only `l1_gas` is non-zero. The gateway's own test suite confirms this is accepted: [1](#0-0) 

No special permissions or malicious intent are required; any user submitting a standard L1-gas-only V3 invoke triggers the divergence.

### Recommendation

The protobuf deserializer must not infer the `ValidResourceBounds` variant from field values alone. Two options:

1. **Add a discriminant field** to `protobuf::ResourceBounds` (e.g., a boolean `is_all_resources`) so the variant is preserved across serialization.
2. **Always emit `AllResources`** for `InvokeV3` transactions (since the gateway only accepts `AllResources` for new transactions), and use `L1Gas` only for transactions explicitly marked as pre-0.13.3.

The current heuristic `l1_data_gas.is_zero() && l2_gas.is_zero()` is ambiguous and must be removed: [8](#0-7) 

### Proof of Concept

1. Submit an `InvokeV3` transaction with `resource_bounds = AllResourceBounds { l1_gas: ResourceBounds { max_amount: 1000, max_price_per_unit: 1 }, l2_gas: ResourceBounds::default(), l1_data_gas: ResourceBounds::default() }`.
2. The sequencer accepts it (passes stateless validation) and computes hash H₁ using the 4-element `AllResources` preimage.
3. Serialize the resulting `InvokeTransactionV3` to `protobuf::InvokeV3` and deserialize it back.
4. Assert that the deserialized `resource_bounds` is `ValidResourceBounds::L1Gas(...)` — the variant has changed.
5. Recompute the hash from the deserialized transaction; observe H₂ ≠ H₁ because the 3-element `L1Gas` preimage is used.
6. Any syncing node that processes this block via P2P will store or validate against H₂, diverging from the sequencer's committed H₁.

pub fn get_tip_resource_bounds_hash(
    resource_bounds: &ValidResourceBounds,
    tip: &Tip,
) -> Result<Felt, StarknetApiError> {
    let l1_resource_bounds = resource_bounds.get_l1_bounds();
    let l2_resource_bounds = resource_bounds.get_l2_bounds();

    // L1 and L2 gas bounds always exist.
    // Old V3 txs always have L2 gas bounds of zero, but they exist.
    let mut resource_felts = vec![
        get_concat_resource(&l1_resource_bounds, L1_GAS)?,
        get_concat_resource(&l2_resource_bounds, L2_GAS)?,
    ];

    // For new V3 txs, need to also hash the data gas bounds.
    resource_felts.extend(match resource_bounds {
        ValidResourceBounds::L1Gas(_) => vec![],
        ValidResourceBounds::AllResources(all_resources) => {
            vec![get_concat_resource(&all_resources.l1_data_gas, L1_DATA_GAS)?]
        }
    });

    Ok(HashChain::new().chain(&tip.0.into()).chain_iter(resource_felts.iter()).get_poseidon_hash())
}

pub(crate) fn get_invoke_transaction_v3_hash<T: InvokeTransactionV3Trait>(
    transaction: &T,
    chain_id: &ChainId,
    transaction_version: &TransactionVersion,
) -> Result<TransactionHash, StarknetApiError> {
    let tip_resource_bounds_hash =
        get_tip_resource_bounds_hash(&transaction.resource_bounds(), transaction.tip())?;
    let paymaster_data_hash =
        HashChain::new().chain_iter(transaction.paymaster_data().0.iter()).get_poseidon_hash();
    let data_availability_mode = concat_data_availability_mode(
        transaction.nonce_data_availability_mode(),
        transaction.fee_data_availability_mode(),
    );
    let account_deployment_data_hash = HashChain::new()
        .chain_iter(transaction.account_deployment_data().0.iter())
        .get_poseidon_hash();
    let calldata_hash =
        HashChain::new().chain_iter(transaction.calldata().0.iter()).get_poseidon_hash();
    let mut hash_chain = HashChain::new()
        .chain(&INVOKE)
        .chain(&transaction_version.0)
        .chain(transaction.sender_address().0.key())
        .chain(&tip_resource_bounds_hash)
        .chain(&paymaster_data_hash)
        .chain(&Felt::try_from(chain_id)?)
        .chain(&transaction.nonce().0)
        .chain(&data_availability_mode)
        .chain(&account_deployment_data_hash)
        .chain(&calldata_hash);
    if !transaction.proof_facts().0.is_empty() {
        let proof_facts_hash =
            HashChain::new().chain_iter(transaction.proof_facts().0.iter()).get_poseidon_hash();
        hash_chain = hash_chain.chain(&proof_facts_hash);
    }
    Ok(TransactionHash(hash_chain.get_poseidon_hash()))
}

impl TryFrom<protobuf::ResourceBounds> for ValidResourceBounds {
    type Error = ProtobufConversionError;
    fn try_from(value: protobuf::ResourceBounds) -> Result<Self, Self::Error> {
        let Some(l1_gas) = value.l1_gas else {
            return Err(missing("ResourceBounds::l1_gas"));
        };
        let Some(l2_gas) = value.l2_gas else {
            return Err(missing("ResourceBounds::l2_gas"));
        };
        // TODO(Shahak): Assert data gas is not none once we remove support for 0.13.2.
        let l1_data_gas = value.l1_data_gas.unwrap_or_default();
        let l1_gas: ResourceBounds = l1_gas.try_into()?;
        let l2_gas: ResourceBounds = l2_gas.try_into()?;
        let l1_data_gas: ResourceBounds = l1_data_gas.try_into()?;
        Ok(if l1_data_gas.is_zero() && l2_gas.is_zero() {
            ValidResourceBounds::L1Gas(l1_gas)
        } else {
            ValidResourceBounds::AllResources(AllResourceBounds { l1_gas, l2_gas, l1_data_gas })
        })
    }
