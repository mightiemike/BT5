### Title
`StatelessTransactionValidatorConfig` accepts `min_sierra_version > max_sierra_version` with no cross-field validation, permanently rejecting all declare transactions at the gateway - (File: crates/apollo_gateway_config/src/config.rs)

### Summary

`StatelessTransactionValidatorConfig` holds two ordered bounds — `min_sierra_version` and `max_sierra_version` — that are loaded from operator-supplied configuration. The struct derives `Validate` but carries no cross-field constraint enforcing `min_sierra_version <= max_sierra_version`. The gateway's `validate_sierra_version` function requires a submitted Sierra program's version to satisfy **both** `min_sierra_version <= version` **and** `version <= max_sierra_version` simultaneously. When the bounds are inverted, no Sierra version can satisfy both conditions, so every declare transaction is permanently rejected at the gateway admission layer with no way to recover short of restarting with corrected configuration.

### Finding Description

`StatelessTransactionValidatorConfig` is defined as:

```rust
#[derive(Clone, Debug, Deserialize, PartialEq, Serialize, Validate)]
pub struct StatelessTransactionValidatorConfig {
    pub min_sierra_version: VersionId,
    pub max_sierra_version: VersionId,
    // ...
}
``` [1](#0-0) 

The `Validate` derive generates per-field validators only; there is no `#[validate(schema(function = "..."))]` attribute and no `impl Validate for StatelessTransactionValidatorConfig` that checks the cross-field ordering invariant. The admission check in `validate_sierra_version` is:

```rust
if self.config.min_sierra_version <= sierra_version && sierra_version <= max_sierra_version {
    return Ok(());
}
Err(StatelessTransactionValidatorError::UnsupportedSierraVersion { ... })
``` [2](#0-1) 

When `min_sierra_version > max_sierra_version`, the conjunction is unsatisfiable for every possible `sierra_version`:

- If `sierra_version < min_sierra_version` → first predicate fails.
- If `sierra_version >= min_sierra_version` → then `sierra_version >= min_sierra_version > max_sierra_version`, so second predicate fails.

The function therefore always returns `UnsupportedSierraVersion`, permanently blocking all declare transactions.

The existing `config_validate` machinery calls `config.validate()` which only exercises per-field `#[validate(...)]` annotations; the cross-field ordering invariant is never checked. [3](#0-2) 

Contrast with the analogous timing-bound check that **was** added for `build_proposal_margin_millis < proposal_timeout`: [4](#0-3) 

No equivalent cross-field guard exists for the Sierra version window.

### Impact Explanation

Every `RpcDeclareTransaction::V3` submitted to the gateway passes through `validate_declare_tx` → `validate_sierra_version`. With an inverted version window, every declare transaction is rejected at gateway admission with `UnsupportedSierraVersion`, regardless of the actual Sierra version embedded in the contract class. No new contract classes can be declared on the network. This matches the allowed impact: **High — Mempool/gateway/RPC admission rejects valid transactions before sequencing.** [5](#0-4) 

### Likelihood Explanation

The two version fields are independently serialized and deserialized from the JSON config schema. An operator who sets `min_sierra_version.minor = 9` and `max_sierra_version.minor = 1` (or any combination where min exceeds max) will silently produce an inverted window. The config schema exposes all six sub-fields (`major`, `minor`, `patch` for each bound) as independent numeric values with no documented ordering constraint and no runtime rejection at load time. [6](#0-5) 

### Recommendation

Add a cross-field schema validator to `StatelessTransactionValidatorConfig`:

```rust
#[derive(Clone, Debug, Deserialize, PartialEq, Serialize, Validate)]
#[validate(schema(function = "validate_sierra_version_bounds"))]
pub struct StatelessTransactionValidatorConfig { ... }

fn validate_sierra_version_bounds(
    cfg: &StatelessTransactionValidatorConfig,
) -> Result<(), ValidationError> {
    if cfg.min_sierra_version > cfg.max_sierra_version {
        return Err(ValidationError::new(
            "min_sierra_version must be <= max_sierra_version",
        ));
    }
    Ok(())
}
```

This mirrors the pattern already used for `ContextDynamicConfig`: [7](#0-6) 

### Proof of Concept

```rust
use apollo_gateway_config::compiler_version::VersionId;
use apollo_gateway_config::config::StatelessTransactionValidatorConfig;
use apollo_gateway::stateless_transaction_validator::StatelessTransactionValidator;
use starknet_api::state::SierraContractClass;
use starknet_api::test_utils::declare::rpc_declare_tx;
use starknet_api::declare_tx_args;

// Inverted bounds: min (1.9.0) > max (1.1.0)
let config = StatelessTransactionValidatorConfig {
    min_sierra_version: VersionId::new(1, 9, 0),
    max_sierra_version: VersionId::new(1, 1, 0),
    ..StatelessTransactionValidatorConfig::default()
};

// config.validate() passes — no cross-field check exists
assert!(validator::Validate::validate(&config).is_ok());

let validator = StatelessTransactionValidator { config };

// A perfectly valid Sierra 1.5.0 contract class
let sierra_program = create_sierra_program(&VersionId::new(1, 5, 0));
let contract_class = SierraContractClass { sierra_program, ..Default::default() };
let tx = rpc_declare_tx(declare_tx_args!(), contract_class);

// Rejected: 1.9.0 <= 1.5.0 is false → UnsupportedSierraVersion
assert!(validator.validate(&tx).is_err());
// Every Sierra version produces the same error — declares are permanently disabled.

    fn validate_sierra_version(
        &self,
        sierra_program: &[Felt],
    ) -> StatelessTransactionValidatorResult<()> {
        // Any patch version is valid. (i.e. when check version for upper bound, we ignore the Z
        // part in a version X.Y.Z).
        let mut max_sierra_version = self.config.max_sierra_version;
        max_sierra_version.0.patch = usize::MAX;

        let sierra_version = VersionId::from_sierra_program(sierra_program)?;
        if self.config.min_sierra_version <= sierra_version && sierra_version <= max_sierra_version
        {
            return Ok(());
        }

        Err(StatelessTransactionValidatorError::UnsupportedSierraVersion {
            version: sierra_version,
            min_version: self.config.min_sierra_version,
            max_version: self.config.max_sierra_version,
        })
