Audit Report

## Title
SwapAllowlistExtension checks router address instead of end-user identity, enabling allowlist bypass - (File: metric-periphery/contracts/extensions/SwapAllowlistExtension.sol)

## Summary
`SwapAllowlistExtension.beforeSwap` gates swaps by checking the `sender` argument, which the pool sets to `msg.sender` of the `pool.swap()` call. When a user routes through `MetricOmmSimpleRouter`, the pool's `msg.sender` is the router contract, not the end user. If the pool admin allowlists the router address to enable router-based swaps, every unprivileged user can bypass the allowlist by calling through the router.

## Finding Description

**Call path:**

1. Unprivileged user calls `MetricOmmSimpleRouter.exactInputSingle(params)` — `msg.sender = user`.
2. Router calls `IMetricOmmPoolActions(params.pool).swap(params.recipient, ...)` — pool sees `msg.sender = router`.
3. `MetricOmmPool.swap` calls `_beforeSwap(msg.sender, ...)` = `_beforeSwap(router_address, ...)`.
4. `ExtensionCalling._beforeSwap` encodes `sender = router_address` and calls `extension.beforeSwap(router_address, ...)`.
5. `SwapAllowlistExtension.beforeSwap` evaluates `allowedSwapper[msg.sender][sender]` = `allowedSwapper[pool][router_address]`.

The extension never sees the original user address. The pool admin who wants to allow KYC'd users to swap through the router must allowlist the router address itself. Once the router is allowlisted, `allowedSwapper[pool][router] = true` passes for every caller of the router, including completely unauthorized users.

**Root cause in `MetricOmmPool.swap`:**
```solidity
_beforeSwap(
    msg.sender,   // ← this is the router, not the end user
    recipient, ...
);
```

**Root cause in `SwapAllowlistExtension.beforeSwap`:**
```solidity
if (!allowAllSwappers[msg.sender] && !allowedSwapper[msg.sender][sender]) {
    revert IMetricOmmPoolActions.NotAllowedToSwap();
}
```
`sender` here is the router address when the swap originates from the router.

**Existing guards are insufficient:** `BaseMetricExtension.onlyPool` only verifies that `msg.sender` is a registered pool, which is satisfied. There is no mechanism to recover the original EOA or user address from the router call.

## Impact Explanation
A pool configured with `SwapAllowlistExtension` to restrict trading to a curated set of addresses (e.g., KYC'd users) is fully bypassed by any unprivileged user routing through `MetricOmmSimpleRouter`. The attacker receives swap output tokens from a pool they are not authorized to trade on, directly violating the pool's curation policy and constituting a broken core pool functionality / admin-boundary break. This matches the "allowlist bypass" impact category: disallowed users can trade on curated pools.

## Likelihood Explanation
`MetricOmmSimpleRouter` is the primary public swap entrypoint. Any user who discovers that the router address is allowlisted (or who simply tries routing) can exploit this with a single transaction. No special privileges, flash loans, or multi-step setup are required. The condition (router allowlisted) is the natural operational state for any pool that intends to support router-based swaps.

## Recommendation
Pass the original initiator through the call chain. One approach: have the router store the original `msg.sender` in transient storage (alongside the existing callback context) and expose a `swapInitiator()` view. The pool or extension can then read this value. Alternatively, `SwapAllowlistExtension.beforeSwap` should check the `recipient` or a user-supplied identity field, or the pool should pass an additional `initiator` argument that the router populates with `msg.sender` before calling `pool.swap`.

## Proof of Concept

```solidity
// Setup: pool with SwapAllowlistExtension, allowlist is restrictive
swapExtension.setAllowedToSwap(address(pool), address(router), true); // admin enables router
// alice is NOT allowlisted

// Attack: alice calls router
vm.prank(alice);
router.exactInputSingle(ExactInputSingleParams({
    pool: address(pool),
    recipient: alice,
    tokenIn: token0,
    amountIn: 1000,
    amountOutMinimum: 0,
    zeroForOne: true,
    priceLimitX64: 0,
    deadline: block.timestamp,
    extensionData: ""
}));
// Succeeds: extension checks allowedSwapper[pool][router] == true, alice bypasses allowlist
```
