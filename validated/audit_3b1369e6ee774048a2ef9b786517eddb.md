### Title
Deprecated `verify_transaction_inclusion` Remains Publicly Callable and Susceptible to 64-Byte Merkle Proof Forgery — (File: `contract/src/lib.rs`)

---

### Summary

The deprecated `verify_transaction_inclusion` function is still a live, unpermissioned public entry point on the contract. It contains a self-documented proof-verification forgery flaw: it will return `true` when the caller supplies an internal Merkle tree node as `tx_id`, because the function cannot distinguish leaf nodes (real transactions) from internal nodes. Any downstream contract or application consuming this result can be deceived into accepting a fabricated transaction inclusion proof.

---

### Finding Description

`verify_transaction_inclusion` was deprecated in v0.5.0 and replaced by `verify_transaction_inclusion_v2`, which adds a coinbase Merkle proof check specifically to close the 64-byte transaction forgery vector. However, the old function was never removed or gated — it remains a callable `#[pause]`-guarded public method with no access restriction beyond the pause state. [1](#0-0) 

The function's own documentation acknowledges the flaw explicitly: [2](#0-1) 

The verification logic reduces entirely to: [3](#0-2) 

`compute_root_from_merkle_proof` is a pure Merkle path computation. It takes `tx_id`, `tx_index`, and the sibling-hash path and recomputes the root. It has no mechanism to verify that `tx_id` is a leaf (a real transaction) rather than an internal node. Because Bitcoin's Merkle tree is built by hashing 32-byte pairs (64 bytes total) at every level, any internal node `N` at level `l` and position `j` has a valid sibling-hash path to the root. An attacker who supplies `tx_id = N` and the corresponding sibling path will cause `compute_root_from_merkle_proof` to reproduce the block's true `merkle_root`, making the function return `true`.

`verify_transaction_inclusion_v2` closes this by first asserting that the coinbase transaction (always at index 0) produces the same root via a separate proof, making it computationally infeasible to substitute an internal node for a real transaction: [4](#0-3) 

The deprecated path skips this check entirely.

---

### Impact Explanation

Any NEAR contract or off-chain application that calls `verify_transaction_inclusion` and acts on a `true` result — for example, releasing bridged funds, minting tokens, or authorizing a cross-chain action — can be tricked into doing so for a Bitcoin transaction that never existed. The attacker needs only a block already accepted into the light client's mainchain (public information) and the ability to compute its Merkle tree (also public). No privileged role, private key, or social engineering is required.

---

### Likelihood Explanation

All data needed to mount the attack (block headers, Merkle trees) is publicly available from any Bitcoin full node or block explorer. The attacker does not need to mine a block or perform any computationally hard operation. The only prerequisite is that the target block is already in the contract's `headers_pool`, which is the normal operating state. Likelihood is **high** for any deployment where downstream consumers call the deprecated endpoint.

---

### Recommendation

Remove `verify_transaction_inclusion` entirely from the contract. If backward compatibility must be preserved temporarily, replace the function body with `env::panic_str("use verify_transaction_inclusion_v2")` so it is unreachable rather than silently exploitable. All callers should be migrated to `verify_transaction_inclusion_v2`. [5](#0-4) 

---

### Proof of Concept

1. Identify any block `B` already accepted into the light client mainchain (e.g., via `get_block_hash_by_height`).
2. Obtain block `B`'s full transaction list from a Bitcoin node or explorer and reconstruct its Merkle tree.
3. Select any internal node `N` at level `l ≥ 1` and position `j`. Record the sibling-hash path from `N` to the root (this is the forged `merkle_proof`).
4. Compute `tx_index` as the leaf-equivalent index for position `j` at level `l` (i.e., `j * 2^l`).
5. Call `verify_transaction_inclusion` with:
   - `tx_id = N` (the internal node hash — not a real transaction)
   - `tx_block_blockhash = hash(B)`
   - `tx_index` as computed above
   - `merkle_proof` = sibling-hash path from step 3
   - `confirmations = 1`
6. The function computes `compute_root_from_merkle_proof(N, tx_index, merkle_proof)`, which reproduces `B`'s true `merkle_root`, and returns `true` — falsely asserting that the non-existent transaction `N` is included in block `B`. [6](#0-5)

    #[deprecated(
        since = "0.5.0",
        note = "Use `verify_transaction_inclusion_v2` instead."

    #[pause]
    pub fn verify_transaction_inclusion(&self, #[serializer(borsh)] args: ProofArgs) -> bool {
        require!(
            args.confirmations <= self.gc_threshold,
            "The required number of confirmations exceeds the number of blocks stored in memory"
        );

        let heaviest_block_header = self
            .headers_pool
            .get(&self.mainchain_tip_blockhash)
            .unwrap_or_else(|| env::panic_str(ERR_KEY_NOT_EXIST));
        let target_block_height = self
            .mainchain_header_to_height
            .get(&args.tx_block_blockhash)
            .unwrap_or_else(|| env::panic_str("block does not belong to the current main chain"));

        // Check requested confirmations. No need to compute proof if insufficient confirmations.
        require!(
            (heaviest_block_header.block_height).saturating_sub(target_block_height) + 1
                >= args.confirmations,
            "Not enough blocks confirmed"
        );

        let header = self
            .headers_pool
            .get(&args.tx_block_blockhash)
            .unwrap_or_else(|| env::panic_str("cannot find requested transaction block"));

        require!(!args.merkle_proof.is_empty(), "Merkle proof is empty");

        // compute merkle tree root and check if it matches block's original merkle tree root
        merkle_tools::compute_root_from_merkle_proof(
            args.tx_id,
            usize::try_from(args.tx_index).unwrap(),
            &args.merkle_proof,
        ) == header.block_header.merkle_root
    }
