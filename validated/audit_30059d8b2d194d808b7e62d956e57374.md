### Title
Division-by-Zero in `socializeSubaccount` When `openInterest == 0` Permanently Blocks Finalization — (`core/contracts/PerpEngine.sol`)

---

### Summary

`PerpEngine.socializeSubaccount` unconditionally divides by `state.openInterest` when residual bad debt remains after insurance coverage. There is no guard for the case where `state.openInterest == 0`. `MathSD21x18.div` explicitly reverts with `"DBZ"` on a zero denominator. This causes the entire finalization transaction to revert permanently for any subaccount that holds a negative `vQuoteBalance` in a perp product whose open interest has fallen to zero, locking the insurance fund and leaving the subaccount in an unresolvable zombie state.

---

### Finding Description

In `PerpEngine.socializeSubaccount`, after insurance partially covers a negative `vQuoteBalance`, the remaining shortfall is socialized by computing a per-share funding adjustment:

```solidity
if (balance.vQuoteBalance < 0) {
    int128 fundingPerShare = -balance.vQuoteBalance.div(
        state.openInterest          // ← no zero-check
    );
``` [1](#0-0) 

`MathSD21x18.div` enforces `require(y != 0, ERR_DIV_BY_ZERO)`:

```solidity
function div(int128 x, int128 y) internal pure returns (int128) {
    unchecked {
        require(y != 0, ERR_DIV_BY_ZERO);
``` [2](#0-1) 

`state.openInterest` is the sum of `abs(amount)` across all positions in the product. It is perfectly legal for it to be zero: a subaccount can close its position (`amount = 0`) while retaining a negative `vQuoteBalance` from the realized loss, and if every other participant has also closed, `openInterest` reaches zero. The `vQuoteBalance` field is independent of `amount` and is not cleared on position close.

Notably, `Errors.sol` already defines `ERR_NO_OPEN_INTEREST = "NOI"` with the comment *"Socializing product with no open interest"*, confirming the developers anticipated this case but never added the guard to `socializeSubaccount`: [3](#0-2) 

`updateStates` in `PerpEngineState` correctly skips products with zero open interest before dividing, demonstrating the intended pattern:

```solidity
if (state.openInterest == 0) {
    continue;
}
``` [4](#0-3) 

That guard is absent from `socializeSubaccount`.

---

### Impact Explanation

`socializeSubaccount` is called unconditionally during finalization in `ClearinghouseLiq`:

```solidity
v.insurance = perpEngine.socializeSubaccount(
    txn.liquidatee,
    v.insurance
);
``` [5](#0-4) 

When the revert fires:
- The finalization transaction always reverts — the subaccount can never be finalized.
- The insurance fund cannot be applied to cover the bad debt.
- The bad debt is never socialized across remaining participants.
- The subaccount remains permanently underwater with no resolution path.

This breaks the protocol invariant that any underwater subaccount must be finalizable, and locks insurance fund assets from being applied to their intended purpose.

---

### Likelihood Explanation

The trigger state — `openInterest == 0` with `vQuoteBalance < 0` — is reachable through normal trading:

1. Subaccount opens a perp position and accumulates negative `vQuoteBalance` (e.g., buys at a high price, price drops, position is closed at a loss).
2. After closing, `balance.amount = 0` but `balance.vQuoteBalance < 0` persists.
3. All other participants also close their positions → `state.openInterest = 0`.
4. The subaccount's health is negative (negative `vQuoteBalance` reduces health via `positionPnl = price * 0 + vQuoteBalance`).
5. Liquidation/finalization is triggered → `socializeSubaccount` is called → division by zero → permanent revert.

No privileged access, governance capture, or external dependency is required. This is reachable via the standard liquidation path.

---

### Recommendation

Add a zero-check for `state.openInterest` before the division, mirroring the pattern already used in `updateStates`. When `openInterest == 0`, there are no other participants to socialize the loss against; the remaining bad debt should either be absorbed entirely by the insurance fund or written off:

```solidity
if (balance.vQuoteBalance < 0) {
    if (state.openInterest == 0) {
        // No open interest to socialize against; write off remaining bad debt
        balance.vQuoteBalance = 0;
    } else {
        int128 fundingPerShare = -balance.vQuoteBalance.div(
            state.openInterest
        );
        state.cumulativeFundingLongX18 += fundingPerShare;
        state.cumulativeFundingShortX18 -= fundingPerShare;
        balance.vQuoteBalance = 0;
    }
}
```

### Proof of Concept

```solidity
// 1. Deploy PerpEngine with one perp product (productId = 1).
// 2. Give subaccount A a position: amount = 1e18, vQuoteBalance = -500e18.
// 3. Close A's position: amount = 0, vQuoteBalance = -500e18 (loss locked in).
// 4. Ensure no other participants have open positions → openInterest = 0.
// 5. Make A's health negative (negative vQuoteBalance reduces health).
// 6. Trigger finalization: clearinghouse calls perpEngine.socializeSubaccount(A, insurance).
// 7. insurance < 500e18, so balance.vQuoteBalance remains < 0 after coverage.
// 8. Execution reaches: -balance.vQuoteBalance.div(state.openInterest)
//    → state.openInterest == 0 → MathSD21x18.div reverts with "DBZ".
// 9. Assert: transaction reverts, finalization is permanently blocked.
```
