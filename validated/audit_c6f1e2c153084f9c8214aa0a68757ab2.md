### Title
Division-by-Zero in `socializeSubaccount` Permanently Blocks Liquidation Finalization When `openInterest == 0` — (`core/contracts/PerpEngine.sol`)

---

### Summary

`PerpEngine.socializeSubaccount` unconditionally divides by `state.openInterest` when a subaccount has a residual negative `vQuoteBalance` after insurance is exhausted. No guard exists for the case where `openInterest == 0`, causing a hard revert via `ERR_DIV_BY_ZERO`. This permanently blocks `_finalizeSubaccount` for any subaccount that holds a closed perp position (`balance.amount == 0`) with a negative `vQuoteBalance` residual in a product where no other open interest exists.

---

### Finding Description

In `PerpEngine.socializeSubaccount`, after insurance is applied, if `balance.vQuoteBalance` is still negative the function attempts to spread the loss across all open-interest holders:

```solidity
// core/contracts/PerpEngine.sol lines 164–168
if (balance.vQuoteBalance < 0) {
    int128 fundingPerShare = -balance.vQuoteBalance.div(
        state.openInterest   // ← reverts if == 0
    );
``` [1](#0-0) 

`MathSD21x18.div` explicitly enforces a non-zero denominator:

```solidity
// core/contracts/libraries/MathSD21x18.sol line 64
require(y != 0, ERR_DIV_BY_ZERO);
``` [2](#0-1) 

`state.openInterest` is maintained by `_updateBalance`. When `balance.amount == 0` (position fully closed), the pre-update step subtracts `balance.amount.abs() = 0` and the post-update else-branch subtracts `balance.amount = 0`, leaving `openInterest` unchanged in storage:

```solidity
// core/contracts/PerpEngineState.sol lines 30, 49
state.openInterest -= balance.amount.abs();   // -= 0
...
state.openInterest -= balance.amount;          // -= 0
``` [3](#0-2) 

Therefore, if the liquidatee was the sole participant in a product and closed their position, `states[productId].openInterest == 0` in storage, and `getStateAndBalance` returns that zero value unchanged.

The liquidation finalization path in `ClearinghouseLiq._finalizeSubaccount` **requires** `balance.amount == 0` for every perp product before calling `socializeSubaccount`:

```solidity
// core/contracts/ClearinghouseLiq.sol lines 319, 386–389
require(balance.amount == 0, ERR_NOT_FINALIZABLE_SUBACCOUNT);
...
v.insurance = perpEngine.socializeSubaccount(
    txn.liquidatee,
    v.insurance
);
``` [4](#0-3) 

The precondition the protocol enforces to enter `socializeSubaccount` (`balance.amount == 0`) is exactly the precondition that makes `openInterest == 0` reachable, creating a structural deadlock.

---

### Impact Explanation

- `_finalizeSubaccount` permanently reverts for the affected subaccount.
- The negative `vQuoteBalance` (bad debt) can never be socialized across the market.
- The insolvent subaccount remains in the system indefinitely with unresolved bad debt, violating the protocol invariant that every subaccount with negative net value must be finalizable.
- No admin escape hatch exists in the scoped code to bypass or reset this state.

---

### Likelihood Explanation

The state is reachable through normal protocol usage:

1. A subaccount opens a perp position in a product where it is the sole (or last remaining) participant.
2. The position is closed (via trade or partial liquidation) at a loss, leaving `balance.amount = 0` and `balance.vQuoteBalance < 0`.
3. All other participants have also closed, so `states[productId].openInterest == 0`.
4. The subaccount's net value is negative (spot quote balance cannot cover the perp loss).
5. A liquidator calls `liquidateSubaccount` with `productId == type(uint32).max` to finalize — the call reverts unconditionally.

This is especially likely in low-liquidity or newly listed markets where a single trader dominates open interest.

---

### Recommendation

Add a zero-check guard before the division. If `openInterest == 0` there are no other participants to socialize the loss onto; the loss should be absorbed entirely by the insurance fund or written off:

```solidity
if (balance.vQuoteBalance < 0) {
    if (state.openInterest == 0) {
        // No open interest to socialize against; write off the residual
        balance.vQuoteBalance = 0;
    } else {
        int128 fundingPerShare = -balance.vQuoteBalance.div(state.openInterest);
        state.cumulativeFundingLongX18 += fundingPerShare;
        state.cumulativeFundingShortX18 -= fundingPerShare;
        balance.vQuoteBalance = 0;
    }
}
```

### Proof of Concept

State-machine test (Hardhat/Foundry, unmodified deployment):

1. Deploy the full protocol stack (Clearinghouse, PerpEngine, SpotEngine, Endpoint).
2. Add a perp product `P`.
3. Alice deposits USDC and opens a long position of size `S` in product `P`. Assert `states[P].openInterest == S`.
4. Price of `P` drops. Alice closes the position via a matched order (amountDelta = `-S`, vQuoteDelta encodes the loss). Assert `balance.amount == 0`, `balance.vQuoteBalance < 0`, `states[P].openInterest == 0`.
5. Alice's subaccount is now unhealthy (spot quote balance < |vQuoteBalance|).
6. Call `endpoint.liquidateSubaccount` with `productId == type(uint32).max` (finalize path).
7. Assert the transaction reverts with `"DBZ"` (`ERR_DIV_BY_ZERO`).
8. Confirm no subsequent call can finalize Alice's subaccount without an external participant first opening a position in product `P`. [5](#0-4) [6](#0-5) [7](#0-6)
