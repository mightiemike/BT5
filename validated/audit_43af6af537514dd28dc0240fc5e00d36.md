Audit Report

## Title
SwapAllowlistExtension Sender Identity Mismatch: Router-Mediated Swaps Bypass Per-User Allowlist on Curated Pools — (`metric-periphery/contracts/extensions/SwapAllowlistExtension.sol`)

## Summary
`SwapAllowlistExtension.beforeSwap()` checks `allowedSwapper[pool][sender]` where `sender` is the immediate caller of `pool.swap()`. When any EOA routes through `MetricOmmSimpleRouter`, `sender` is the router contract address, not the originating EOA. A pool admin who allowlists the router to support router-mediated swaps on a curated pool inadvertently grants swap access to every EOA, completely defeating the per-user allowlist invariant.

## Finding Description
`MetricOmmPool.swap()` unconditionally passes `msg.sender` as the `sender` argument to `_beforeSwap()` (lines 230–240 of `metric-core/contracts/MetricOmmPool.sol`). `ExtensionCalling._beforeSwap()` forwards that value verbatim to the extension via `abi.encodeCall` (lines 149–177 of `metric-core/contracts/ExtensionCalling.sol`). `SwapAllowlistExtension.beforeSwap()` then checks `allowedSwapper[msg.sender][sender]`, where `msg.sender` is the pool and `sender` is whoever called `pool.swap()` (lines 37–38 of `metric-periphery/contracts/extensions/SwapAllowlistExtension.sol`).

`MetricOmmSimpleRouter.exactInputSingle()` stores the originating `msg.sender` in transient storage only for the payment callback (`_setNextCallbackContext`, line 71), but calls `pool.swap()` directly with no mechanism to forward the original EOA identity (lines 72–80 of `metric-periphery/contracts/MetricOmmSimpleRouter.sol`). The same applies to `exactInput`, `exactOutputSingle`, and `exactOutput`.

Result: the extension always sees `sender = router` for every user who goes through the router. The admin faces an impossible choice — allowlist individual EOAs (those users cannot use the router) or allowlist the router (any EOA bypasses the per-user gate). No configuration simultaneously allows router-mediated swaps and enforces per-user identity checks.

## Impact Explanation
A pool admin who deploys a curated pool with `SwapAllowlistExtension` and allowlists the router address loses all per-user access control. Any non-allowlisted EOA can execute swaps on the curated pool by calling `MetricOmmSimpleRouter.exactInputSingle()` (or any other router entry point). The pool curation invariant — that only individually approved addresses may trade — is permanently broken for all router-mediated paths. This constitutes broken core pool functionality: the primary purpose of the extension (restricting who may swap) is nullified for the canonical swap interface.

## Likelihood Explanation
`MetricOmmSimpleRouter` is the canonical swap interface for the protocol. Pool admins who want to support normal user flows will naturally allowlist the router. The mismatch is not documented in the extension or the router, so the admin has no signal that allowlisting the router is equivalent to `setAllowAllSwappers(pool, true)`. The likelihood of this misconfiguration is high for any curated pool that intends to support router usage. The bypass is repeatable by any EOA at zero cost beyond gas.

## Recommendation
`SwapAllowlistExtension.beforeSwap()` should gate on the originating user, not the immediate `pool.swap()` caller. Two viable approaches:

1. **Pass the original initiator via transient storage** — the router already uses transient storage for callback context; it could additionally write the originating `msg.sender` into a known transient slot readable by extensions, and the pool could expose this alongside `sender`.
2. **Require direct pool interaction for allowlisted pools** — document and enforce that pools using `SwapAllowlistExtension` must not allowlist the router; users must call `pool.swap()` directly. This requires clear documentation and ideally an on-chain guard in the extension that rejects known router addresses as `sender`.

The cleanest fix is for the router to store the originating EOA in transient storage and for the pool/extension to read it, so the extension can check the correct identity.
