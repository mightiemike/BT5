### Title
Double Bonus Claim via Duplicate Position IDs in `ClaimTierRewards` — (`x/tieredrewards/keeper/msg_server.go`, `claim_rewards.go`)

---

### Summary

`ClaimTierRewards` does not deduplicate `msg.PositionIds`. Submitting the same position ID twice causes `claimRewardsAndUpdatesPositions` to process two independent in-memory snapshots of the same position, resulting in the bonus reward being paid twice from `RewardsPoolName` within a single transaction.

---

### Finding Description

**Entry point — `ClaimTierRewards` (msg_server.go:429–468)**

The handler iterates over `msg.PositionIds` with no uniqueness check, fetching each ID from the store and appending to a `positions` slice: [1](#0-0) 

If `msg.PositionIds = [X, X]`, the slice becomes `[snapshot_X, snapshot_X]` — two identical in-memory copies of the same position state, both with the same `LastEventSeq` and `LastBonusAccrual`.

**Processing loop — `claimRewardsAndUpdatesPositions` (claim_rewards.go:106–135)**

Each element is processed independently: [2](#0-1) 

**Iteration 0 (`positions[0]`):**

1. `processEventsAndClaimBonus` walks events since `pos.LastEventSeq` (original value), computes bonus for all segments including the current open segment `[segmentStart, blockTime]`, sends coins from `RewardsPoolName` to owner, and calls `decrementEventRefCount` for each event.
2. `applyBonusAccrualCheckpoint` sets `pos.LastBonusAccrual = blockTime` in memory.
3. `setPosition` writes the updated position (advanced `LastEventSeq`, `LastBonusAccrual = blockTime`) to the store. [3](#0-2) 

**Iteration 1 (`positions[1]` — original snapshot, `LastEventSeq` = old value):**

`processEventsAndClaimBonus` is called again with the original `LastEventSeq`. Two sub-cases:

- **Single-position validator (ReferenceCount = 1):** Events were deleted by iteration 0's `decrementEventRefCount`. The events loop is empty. However, the "current segment" bonus is still computed and paid: [4](#0-3) 

  `segmentStart` = original `pos.LastBonusAccrual` (from snapshot), so the bonus covers `[original_LastBonusAccrual, blockTime]` — the **entire period already paid in iteration 0**.

- **Multi-position validator (ReferenceCount > 1):** Events are not deleted (only decremented). `getValidatorEventsSince` returns them again, and the full bonus for all segments is paid a second time. Additionally, ref counts are decremented twice per event, corrupting the accounting for all other positions on that validator. [5](#0-4) 

After iteration 1, `setPosition` writes `positions[1]`'s state (original `LastEventSeq`, `LastBonusAccrual = blockTime`) to the store, overwriting iteration 0's `LastEventSeq` update. This does not prevent future claims from working correctly (new events have higher seq numbers), but it does mean the double-payment is not detectable from on-chain state alone.

**No deduplication guard exists.** `validateClaimRewards` only checks ownership: [6](#0-5) 

---

### Impact Explanation

Direct fund loss from `RewardsPoolName`. The attacker receives approximately 2× the bonus they are entitled to per `ClaimTierRewards` call. In the multi-position case, other positions on the same validator have their event ref counts decremented twice, potentially causing those positions to permanently lose access to their bonus rewards (events deleted prematurely).

---

### Likelihood Explanation

Any position owner can craft a `MsgClaimTierRewards` with a repeated position ID. No special privileges, governance, or operator access is required. The attack is repeatable every time there are unprocessed validator events (i.e., after any slash, bond, or unbond event on the validator). The only limiting factor is the `RewardsPoolName` balance, which is replenished by inflation.

---

### Recommendation

Deduplicate `msg.PositionIds` before processing, either in `MsgClaimTierRewards.Validate()` (message-level) or at the start of `ClaimTierRewards`:

```go
seen := make(map[uint64]struct{}, len(msg.PositionIds))
for _, id := range msg.PositionIds {
    if _, ok := seen[id]; ok {
        return nil, errorsmod.Wrapf(sdkerrors.ErrInvalidRequest, "duplicate position id: %d", id)
    }
    seen[id] = struct{}{}
}
```

This guard should be placed before the position-fetching loop in `ClaimTierRewards`. [7](#0-6) 

---

### Proof of Concept

```
1. Create position P on validator V (ReferenceCount on new events = 1).
2. Append two ValidatorEvents E1, E2 to V (e.g., via slash hook).
   P.LastEventSeq = 0, E1.seq = 1, E2.seq = 2.
3. Submit MsgClaimTierRewards{Owner: owner, PositionIds: [P.Id, P.Id]}.

Iteration 0:
  - processEventsAndClaimBonus processes E1, E2.
  - decrementEventRefCount(E1): 1→0, deleted.
  - decrementEventRefCount(E2): 1→0, deleted.
  - Pays bonus B = bonus([LastBonusAccrual, E1.t]) + bonus([E1.t, E2.t]) + bonus([E2.t, blockTime]).
  - setPosition: LastEventSeq=2, LastBonusAccrual=blockTime.

Iteration 1 (snapshot: LastEventSeq=0, LastBonusAccrual=T0):
  - getValidatorEventsSince(V, 0) → [] (E1, E2 deleted).
  - segmentStart = T0 (original LastBonusAccrual), bonded = true.
  - Current segment bonus = bonus([T0, blockTime]) ≈ B (same total period).
  - Pays bonus B again.
  - setPosition: LastEventSeq=0 (overwrites!), LastBonusAccrual=blockTime.

Total paid: ~2B. RewardsPoolName drained by 2B instead of B.

4. Assert: owner balance increased by ~2B.
5. Assert: subsequent ClaimTierRewards returns 0 bonus (events gone, LastBonusAccrual=blockTime).
6. Assert: RewardsPoolName balance decreased by ~2B.
```

func (ms msgServer) ClaimTierRewards(ctx context.Context, msg *types.MsgClaimTierRewards) (*types.MsgClaimTierRewardsResponse, error) {
	if err := msg.Validate(); err != nil {
		return nil, err
	}

		if err != nil {
			return nil, err
		}

		if err := ms.validateClaimRewards(pos.Position, msg.Owner); err != nil {
			return nil, err
		}

		positions = append(positions, pos)
	}

	totalBase, totalBonus, err := ms.claimRewardsAndUpdatesPositions(ctx, positions)

func (k Keeper) processEventsAndClaimBonus(ctx context.Context, pos *types.PositionState) (sdk.Coins, error) {
	// Rewards should have been claimed before undelegation
	if !pos.IsDelegated() {
		return sdk.NewCoins(), nil
	}

	valAddr, err := sdk.ValAddressFromBech32(pos.Delegation.ValidatorAddress)
	if err != nil {
		return nil, err
	}

	events, err := k.getValidatorEventsSince(ctx, valAddr, pos.LastEventSeq)
	if err != nil {
		return nil, err
	}

	sdkCtx := sdk.UnwrapSDKContext(ctx)
	blockTime := sdkCtx.BlockTime()

	totalBonus := math.ZeroInt()
	// Use the persisted bonded state from the last replay, not a hardcoded default.
	// This prevents overpaying bonus for unbonded gaps between claims.
	bonded := pos.LastKnownBonded
	segmentStart := pos.LastBonusAccrual

	tier, err := k.getTier(ctx, pos.TierId)
	if err != nil {
		return nil, err
	}

	for _, entry := range events {
		evt := entry.Event

		if bonded {
			// Compute bonus for the bonded segment [segmentStart, eventTime]
			// using the snapshot rate at the event.
			bonus := k.computeSegmentBonus(*pos, tier, segmentStart, evt.Timestamp, evt.TokensPerShare)
			totalBonus = totalBonus.Add(bonus)
		}

		// Update bonded state based on event type.
		switch evt.EventType {
		case types.ValidatorEventType_VALIDATOR_EVENT_TYPE_UNBOND:
			bonded = false
		case types.ValidatorEventType_VALIDATOR_EVENT_TYPE_BOND:
			bonded = true
		case types.ValidatorEventType_VALIDATOR_EVENT_TYPE_SLASH:
			// Slash doesn't change bonded state.
		}

		segmentStart = evt.Timestamp
		pos.UpdateLastEventSeq(entry.Seq)

		// Decrement reference count.
		if err := k.decrementEventRefCount(ctx, valAddr, entry.Seq); err != nil {
			return nil, err
		}
	}

	val, err := k.stakingKeeper.GetValidator(ctx, valAddr)
	if err != nil {
		return nil, err
	}
	// Defensive: validator bond status check
	if bonded && val.IsBonded() {
		currentRate, err := k.getTokensPerShare(ctx, valAddr)
		if err != nil {
			return nil, err
		}
		bonus := k.computeSegmentBonus(*pos, tier, segmentStart, blockTime, currentRate)
		totalBonus = totalBonus.Add(bonus)
	}

	applyBonusAccrualCheckpoint(&pos.Position, blockTime)

// decrementEventRefCount decrements the reference count of a validator event.
// If the reference count reaches zero, the event is deleted.
func (k Keeper) decrementEventRefCount(ctx context.Context, valAddr sdk.ValAddress, seq uint64) error {
	key := collections.Join(valAddr, seq)
	event, err := k.ValidatorEvents.Get(ctx, key)
	if errors.Is(err, collections.ErrNotFound) {
		return nil // already cleaned up
	}
	if err != nil {
		return err
	}

	if event.ReferenceCount <= 1 {
		return k.ValidatorEvents.Remove(ctx, key)
	}

	event.ReferenceCount--
	return k.ValidatorEvents.Set(ctx, key, event)
}
