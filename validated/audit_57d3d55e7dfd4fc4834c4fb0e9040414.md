Audit Report

## Title
`SwapAllowlistExtension` checks router address instead of originating user, allowing any user to bypass per-user swap allowlist via `MetricOmmSimpleRouter` - (`metric-periphery/contracts/extensions/SwapAllowlistExtension.sol`)

## Summary
`SwapAllowlistExtension.beforeSwap` gates on the `sender` argument, which the pool populates with its own `msg.sender` — the direct caller of `pool.swap`. When `MetricOmmSimpleRouter` intermediates a swap, the pool sees the router as `msg.sender`, so the extension checks the router's allowlist entry rather than the originating EOA's. Any pool that allowlists the router (required for router-mediated swaps to function) grants unrestricted swap access to every user, defeating the per-user restriction entirely.

## Finding Description
`MetricOmmPool.swap` passes `msg.sender` as the `sender` argument to `_beforeSwap`:

```solidity
// MetricOmmPool.sol L230-240
_beforeSwap(
    msg.sender,   // ← direct caller of pool.swap
    recipient,
    ...
    extensionData
);
```

`ExtensionCalling._beforeSwap` forwards this value unchanged to the extension:

```solidity
// ExtensionCalling.sol L149-177
function _beforeSwap(address sender, ...) internal {
    _callExtensionsInOrder(
        BEFORE_SWAP_ORDER,
        abi.encodeCall(IMetricOmmExtensions.beforeSwap, (sender, ...))
    );
}
```

`SwapAllowlistExtension.beforeSwap` checks this `sender` against the per-pool allowlist:

```solidity
// SwapAllowlistExtension.sol L37
if (!allowAllSwappers[msg.sender] && !allowedSwapper[msg.sender][sender]) {
    revert IMetricOmmPoolActions.NotAllowedToSwap();
}
```

`MetricOmmSimpleRouter.exactInputSingle` calls `pool.swap` directly, making the router contract the pool's `msg.sender`:

```solidity
// MetricOmmSimpleRouter.sol L72-80
(int128 amount0Delta, int128 amount1Delta) = IMetricOmmPoolActions(params.pool)
    .swap(params.recipient, params.zeroForOne, ...);
```

The same pattern applies in `exactInput` (L104-112), `exactOutputSingle` (L136-137), and `exactOutput` (L165-181). The pool admin faces an impossible configuration: allowlisting individual EOAs blocks all router-mediated swaps (router not allowlisted → reverts); allowlisting the router address passes every user's swap regardless of their individual allowlist status.

## Impact Explanation
A pool configured with `SwapAllowlistExtension` to restrict swaps to specific counterparties (private market-maker pool, KYC-gated pool, curated LP base) can be freely accessed by any EOA through `MetricOmmSimpleRouter`. The attacker executes swaps the pool admin intended to block, enabling unauthorized arbitrage or trades against a pool whose pricing assumes a controlled counterparty set. This constitutes a direct admin-boundary break with potential loss of LP principal — a broken core pool functionality impact meeting Sherlock High/Critical thresholds.

## Likelihood Explanation
`MetricOmmSimpleRouter` is the primary public swap entry point. Any pool deploying `SwapAllowlistExtension` that also needs to support router-mediated swaps must allowlist the router, at which point the allowlist provides zero per-user protection. No special privilege is required: any EOA can call `exactInputSingle` on the router. The condition is trivially reachable and repeatable.

## Recommendation
The extension must gate on the originating user, not the direct caller of `pool.swap`. Two viable approaches:

1. **Pass originator via `extensionData`**: The router populates `extensionData` with `abi.encode(msg.sender)` before calling the pool. The extension decodes and checks this field when `sender` is a known router address. This is safe if the router is the only allowlisted caller.

2. **Dedicated router allowlist slot**: Extend the extension with an `allowedRouter` mapping; when `sender` matches a known router, decode the originating user from `extensionData` and check that address against `allowedSwapper`.

## Proof of Concept

```solidity
// Setup: pool admin configures SwapAllowlistExtension
extension.setAllowedToSwap(pool, router, true);   // required for router-mediated swaps
extension.setAllowedToSwap(pool, alice, true);    // alice is intended user
// eve is NOT allowlisted

// Eve bypasses the allowlist via router:
router.exactInputSingle(ExactInputSingleParams({
    pool:             restrictedPool,
    recipient:        eve,
    zeroForOne:       true,
    amountIn:         1_000e18,
    amountOutMinimum: 0,
    priceLimitX64:    type(uint128).max,
    tokenIn:          token0,
    extensionData:    "",
    deadline:         block.timestamp
}));
// pool.swap called with msg.sender = router
// extension checks allowedSwapper[pool][router] = true → passes
// Eve receives output tokens; LP funds drained without authorization
```
