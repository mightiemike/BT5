### Title
`PauseManager` Role Never Granted During `init`, Making Pause/Unpause Permanently Inaccessible — (`File: contract/src/lib.rs`)

### Summary

`BtcLightClient` derives `Pausable` and applies `#[pause]` to every critical public function, but the `init` function only sets `env::current_account_id()` (the contract account itself) as super admin and grants no external account the `PauseManager` role. Because `acl_grant_role` requires the caller to be a super admin, and the contract account cannot sign transactions in a standard keyless production deployment, the pause/unpause capability is permanently inaccessible after deployment.

### Finding Description

The contract struct is declared with:

```rust
#[pausable(manager_roles(Role::PauseManager))]
```

and the `#[pause]` attribute is applied to all externally reachable state-mutating and proof-returning functions: [1](#0-0) [2](#0-1) [3](#0-2) [4](#0-3) [5](#0-4) 

The `init` function sets only the contract account itself as super admin:

```rust
near_sdk::require!(
    contract.acl_init_super_admin(env::current_account_id()),
    "Failed to initialize super admin",
);
``` [6](#0-5) 

`InitArgs` contains no field for specifying an admin account or granting any role: [7](#0-6) 

After `init` returns, the only account that can call `acl_grant_role` successfully is `env::current_account_id()` — the contract account itself. In a standard NEAR production deployment the contract account holds no full-access key, so no external account can ever be promoted to `PauseManager`. The `near_plugins` `AccessControllable` trait enforces this: `acl_grant_role` checks that `env::predecessor_account_id()` is a super admin or admin before granting any role.

### Impact Explanation

`submit_blocks`, `verify_transaction_inclusion`, and `verify_transaction_inclusion_v2` are all gated by `#[pause]`. If a critical bug is discovered in any of these functions — for example, a header-acceptance flaw that allows invalid chain state to be committed, or a Merkle-proof forgery — the contract **cannot be paused** to stop ongoing exploitation. The pause mechanism exists in the code but is permanently unreachable in production, negating the entire emergency-response capability the `Pausable` design is meant to provide.

**Impact: Medium** — the broken invariant is that the `Pausable` safety net is silently non-functional; exploitation of any future vulnerability in the paused functions cannot be halted.

### Likelihood Explanation

**Likelihood: Medium** — this affects every production deployment of the contract as written. The `init` function is the only initialization path, `InitArgs` has no admin-account field, and no post-init role-granting mechanism is built into the contract itself. Any operator who follows the standard NEAR security practice of removing full-access keys from the contract account after deployment will find the pause controls permanently locked out.

### Recommendation

Grant the `PauseManager` role (and any other required administrative roles) to a designated external account inside `init`. Add an `admin` (or `pause_manager`) field to `InitArgs`:

```rust
pub struct InitArgs {
    // ... existing fields ...
    pub pause_manager: AccountId,
}
```

Then inside `init`, after `acl_init_super_admin`:

```rust
contract.acl_grant_role(
    "PauseManager".to_string(),
    args.pause_manager.clone(),
);
```

This ensures that at least one externally controllable account holds the `PauseManager` role from the moment the contract is live, matching the intent of the `Pausable` design.

### Proof of Concept

1. Deploy the contract and call `init` with any valid `InitArgs`.
2. Attempt to call `pa_pause` (the method generated by the `Pausable` derive macro) from any account other than the contract account itself.
3. The call reverts because no account holds `PauseManager` and no account can be granted it (only the contract account is super admin, and it has no full-access key in production).
4. Simulate an emergency: call `submit_blocks` with adversarial headers — the call succeeds because the contract cannot be paused.

The comment in the test helper confirms the design: *"The contract itself is the super admin (set during `init`), so it can grant any role"* — but the contract never uses this capability to grant `PauseManager` to any external account before returning from `init`. [8](#0-7)
