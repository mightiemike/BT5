Audit Report

## Title
`SwapAllowlistExtension` Checks Router Address Instead of Originating EOA, Allowing Any Caller to Bypass Per-User Swap Allowlist — (`metric-periphery/contracts/extensions/SwapAllowlistExtension.sol`)

## Summary
`SwapAllowlistExtension.beforeSwap` gates swaps by checking `allowedSwapper[msg.sender][sender]`, where `sender` is the direct caller of `pool.swap()`. When swaps are routed through `MetricOmmSimpleRouter`, the pool receives `msg.sender` = router, so the extension checks `allowedSwapper[pool][router]`. Any pool admin who allowlists the router to enable router-based swaps inadvertently grants swap access to every EOA that calls through the router, completely bypassing per-user restrictions.

## Finding Description

**Step 1 — Router calls pool with itself as `msg.sender`:**
`MetricOmmSimpleRouter.exactInputSingle` calls `IMetricOmmPoolActions(params.pool).swap(...)` directly. The pool sees `msg.sender` = router. [1](#0-0) 

**Step 2 — Pool passes `msg.sender` (router) as `sender` to the hook:**
`MetricOmmPool.swap` calls `_beforeSwap(msg.sender, ...)`, binding `sender` = router. [2](#0-1) 

**Step 3 — Extension checks `allowedSwapper[pool][router]`, not `allowedSwapper[pool][EOA]`:**
`SwapAllowlistExtension.beforeSwap` uses `msg.sender` (= pool) as the pool key and `sender` (= router) as the swapper key. The originating EOA is never consulted. [3](#0-2) 

**Step 4 — Test suite confirms the binding:**
`test_allowedSwapSucceeds` allowlists `callers[0]` (the intermediate contract that calls `pool.swap`) and the EOA `users[0]` is never in the allowlist, yet the swap succeeds. This confirms the extension only checks the direct caller of `pool.swap()`, not the originating EOA. [4](#0-3) 

The `_swap` helper confirms that `callers[userIndex]` (not `users[userIndex]`) is the entity that calls `pool.swap()`: [5](#0-4) 

There is no mechanism in `exactInputSingle` to encode the originating `msg.sender` into `extensionData` for the extension to decode — the `extensionData` field is passed through verbatim from the caller-controlled `params.extensionData`. [6](#0-5) 

## Impact Explanation
A pool admin deploying a restricted pool (e.g., a permissioned institutional pool) with `SwapAllowlistExtension` to gate individual counterparties must allowlist the router address to support router-based swaps. Once the router is allowlisted, every EOA on the network can call `exactInputSingle` through the router and trade against the pool's LP reserves, because the extension only checks `allowedSwapper[pool][router]` = `true`. The per-user gate is completely bypassed. LPs in the restricted pool are exposed to trades from unauthorized counterparties, breaking the core access-control invariant of the extension and constituting broken core pool functionality (the allowlist mechanism is rendered non-functional for any router-mediated swap path).

## Likelihood Explanation
`MetricOmmSimpleRouter` is the primary public swap interface. Any pool admin who wants to support router-based swaps must allowlist the router. Once that is done, no privileged access, special tokens, or unusual parameters are required — a standard `exactInputSingle` call from any EOA suffices to bypass the per-user allowlist. The bypass is trivially repeatable.

## Recommendation
Pass the originating user identity through the swap path so the extension can gate on the true end-user:

1. **Preferred:** Have `MetricOmmSimpleRouter` encode the originating `msg.sender` into `extensionData` (e.g., `abi.encode(msg.sender)`) and have `SwapAllowlistExtension.beforeSwap` decode and check it when `sender` is a known trusted router. This requires a trusted router registry in the extension.

2. **Alternative:** If `sender` is a recognized router, check `allowedSwapper[pool][recipient]` instead. This is weaker (recipient is caller-controlled) but prevents the trivial bypass.

The root fix is that `sender` in `beforeSwap` must represent the economically responsible party, not the intermediate contract.

## Proof of Concept

```solidity
// Setup: pool with SwapAllowlistExtension, only router is allowlisted
swapExtension.setAllowedToSwap(address(pool), address(router), true);

// EOA (not in allowlist) calls through router
vm.prank(nonAllowlistedEOA);
router.exactInputSingle(ExactInputSingleParams({
    pool: address(pool),
    tokenIn: token0,
    tokenOut: token1,
    zeroForOne: true,
    amountIn: 1000,
    amountOutMinimum: 0,
    recipient: nonAllowlistedEOA,
    deadline: type(uint256).max,
    priceLimitX64: 0,
    extensionData: ""
}));
// Swap succeeds: allowedSwapper[pool][router] = true, EOA never checked
assertFalse(swapExtension.isAllowedToSwap(address(pool), nonAllowlistedEOA));
// but swap went through anyway
```
