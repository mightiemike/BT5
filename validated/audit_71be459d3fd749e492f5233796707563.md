### Title
`SwapAllowlistExtension` Checks the Router's Address Instead of the Originating Swapper, Allowing Any User to Bypass the Swap Allowlist via `MetricOmmSimpleRouter` — (`metric-periphery/contracts/extensions/SwapAllowlistExtension.sol`)

---

### Summary

`SwapAllowlistExtension.beforeSwap` gates swaps by checking the `sender` argument, which is the `msg.sender` of `pool.swap()`. When a user routes through `MetricOmmSimpleRouter`, the router is the immediate caller of `pool.swap()`, so the extension evaluates `allowedSwapper[pool][router]` rather than `allowedSwapper[pool][original_user]`. If the pool admin allowlists the router address — a natural step to let their whitelisted users access multi-hop routing — every unprivileged user can bypass the allowlist entirely by routing through the same contract.

---

### Finding Description

`ExtensionCalling._beforeSwap` forwards `msg.sender` of `pool.swap()` as the `sender` argument to every configured extension: [1](#0-0) [2](#0-1) 

`SwapAllowlistExtension.beforeSwap` then checks whether that `sender` is allowlisted for the calling pool: [3](#0-2) 

When a user calls `MetricOmmSimpleRouter.exactInputSingle` (or `exactInput`), the router calls `pool.swap()` with `msg.sender = router`: [4](#0-3) 

The extension therefore evaluates `allowedSwapper[pool][router]`, not `allowedSwapper[pool][original_user]`. A pool admin who wants their allowlisted users to be able to use the router must call `setAllowedToSwap(pool, router, true)`. The moment they do, the check becomes `allowedSwapper[pool][router] == true` for every caller of the router — the per-user gate is completely open.

The analog to the external bug is exact: just as `removeStackPosition` increments the loop index one step too early so the intended guard (element shift) never executes on the right slot, `SwapAllowlistExtension` evaluates the allowlist on the wrong identity (the router instead of the originating user), so the intended guard never fires on the actor the pool designer meant to restrict.

---

### Impact Explanation

Any unprivileged user can swap in a pool that is supposed to be restricted to a specific allowlist by routing through `MetricOmmSimpleRouter`. The allowlist is rendered completely ineffective for router-mediated swaps. Pools designed for KYC-gated, institutional, or otherwise restricted participant sets lose their access control, allowing arbitrary counterparties to trade against LP capital that was deployed under the assumption of a controlled participant set.

---

### Likelihood Explanation

The prerequisite — the pool admin allowlisting the router — is a natural operational step. A pool admin who allowlists individual users and then wants those users to access multi-hop routing has no other option under the current design: the extension provides no mechanism to gate the original user through a router intermediary. The admin must either block all router-based swaps (by not allowlisting the router) or open the gate to everyone (by allowlisting the router). The bypass is therefore reachable whenever the pool admin makes the operationally reasonable choice to enable router access.

---

### Recommendation

The extension must gate on the economically relevant actor, not the immediate `msg.sender` of `pool.swap()`. Two sound approaches:

1. **Pass the original user through `extensionData`**: The router encodes `msg.sender` (the original user) into `extensionData` before calling `pool.swap()`. The extension decodes and verifies this address. The pool's `_beforeSwap` already forwards `extensionData` unchanged to the extension.

2. **Check `sender` only for direct pool calls; require the router to attest the original user**: Define a trusted-router registry in the extension. When `sender` is a known router, decode the original user from `extensionData` and apply the allowlist check to that address instead.

---

### Proof of Concept

```
Setup:
  pool configured with SwapAllowlistExtension
  pool admin calls setAllowedToSwap(pool, userA, true)       // allowlist userA
  pool admin calls setAllowedToSwap(pool, router, true)      // enable router for userA

Attack:
  userB (not allowlisted) calls:
    router.exactInputSingle({pool: pool, tokenIn: ..., ...})

Execution:
  router calls pool.swap(...) with msg.sender = router
  pool calls _beforeSwap(sender=router, ...)
  extension evaluates allowedSwapper[pool][router] == true  → passes
  userB's swap executes successfully despite not being allowlisted
``` [5](#0-4) [4](#0-3) [6](#0-5)
