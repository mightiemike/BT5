### Title
Migration v8 Silently Skips Undelegated Vesting-Owned Positions, Enabling Post-Migration Vesting Bypass via `MsgWithdrawFromTier` — (`x/tieredrewards/migrations/v2/migrate.go`, `x/tieredrewards/keeper/force_exit.go`)

---

### Summary

`ForceFullExitWithDelegation` returns `nil` (no-op) when a position is not delegated. `exitVestedAccountsPositions` calls it for every vesting-owned position but never verifies the position was actually deleted. A vesting account owner who calls `MsgTierUndelegate` before the v8 upgrade puts their position into the undelegated state, causing the migration to silently skip it. Post-upgrade the owner calls `MsgWithdrawFromTier`, which has no vesting-account guard, and receives the unbonded principal as spendable bank balance — bypassing the vesting lock that the migration was designed to enforce.

---

### Finding Description

**Step 1 — No-op guard in `ForceFullExitWithDelegation`** [1](#0-0) 

When `pos.IsDelegated()` is false the function logs an error and returns `nil`. It does **not** delete the position, does not transfer any tokens, and does not correct vesting tracking.

**Step 2 — Migration never checks whether the position was deleted** [2](#0-1) 

`exitVestedAccountsPositions` iterates every vesting-owned position ID and calls `ForceFullExitWithDelegation`. It only checks for a returned error. A `nil` return from the no-op path is indistinguishable from a successful exit, so the position survives the migration undetected.

**Step 3 — `validateUndelegatePosition` has no vesting-account guard** [3](#0-2) 

A vesting account owner with a legacy v1 position can freely call `MsgTierUndelegate` (exit triggered + elapsed + delegated is all that is required). After this call the position's staking delegation is cleared and an unbonding entry is created.

**Step 4 — `MsgWithdrawFromTier` has no vesting-account guard** [4](#0-3) 

`validateWithdrawFromTier` checks: owner match, exit triggered, exit elapsed, not delegated, not unbonding. None of these checks prevent a vesting account owner from withdrawing. Once unbonding completes post-upgrade, all five conditions are satisfied.

**Step 5 — `WithdrawFromTier` sends `SpendableCoins(posDelAddr)` to the owner's bank balance** [5](#0-4) 

The unbonded principal lands at `posDelAddr` as ordinary bank balance. `WithdrawFromTier` sweeps it to the owner's bank account.

---

### Impact Explanation

For a **`CommitDelegationToTier`-origin** position the owner originally held a staking delegation that was transferred into the position. The bank-side `TrackDelegation` hook set `DelegatedVesting` at that time, leaving it stale-high after the delegation moved to the position delegator address. The migration was supposed to call `alignVestingDelegationTracking` (inside `ForceFullExitWithDelegation`) to reconcile `DV+DF` with actual on-chain delegations. Because the migration is a no-op for undelegated positions, `DelegatedVesting` remains stale-high post-upgrade. [6](#0-5) 

With `DelegatedVesting ≥ OriginalVesting`, `LockedCoins = OriginalVesting − DelegatedVesting = 0` for a `PermanentLockedAccount`. The owner's bank balance after `MsgWithdrawFromTier` is therefore fully spendable — permanently locked tokens become liquid.

For a **`LockTier`-origin** position the migration invariant ("all vesting-owned positions must be force-exited and deleted") is still violated, and the position survives with no corrective action, though the direct vesting-bypass impact is lower since those tokens were originally spendable.

---

### Likelihood Explanation

The precondition is a v1 legacy vesting-owned position (created before the vesting-account check was added to `MsgLockTier`/`MsgCommitDelegationToTier`). The migration was written precisely because such positions exist on-chain. The attacker only needs to call `MsgTierUndelegate` — a normal, permissionless user transaction — before the upgrade block. No governance control, no operator access, and no leaked keys are required.

---

### Recommendation

In `exitVestedAccountsPositions`, after calling `ForceFullExitWithDelegation`, verify the position no longer exists. If it does (undelegated case), handle it explicitly — either by sweeping the unbonded/bank balance directly from `posDelAddr` to the owner and deleting the position, or by returning an error that halts the migration until the position is in a delegated state.

Alternatively, extend `ForceFullExitWithDelegation` to handle the undelegated case: sweep `SpendableCoins(posDelAddr)` to the owner, call `alignVestingDelegationTracking` with the correct delta, and delete the position — instead of returning `nil`.

---

### Proof of Concept

```
// Seed: vesting owner with a CommitDelegationToTier-origin v1 position
// (created via createCommitPositionV1, bypassing the vesting check)

1. vestingOwner calls MsgTierUndelegate(positionId)
   → position.IsDelegated() becomes false
   → staking unbonding entry created at posDelAddr

2. v8 upgrade block executes Migrate()
   → exitVestedAccountsPositions walks positions, finds vestingOwner's position
   → ForceFullExitWithDelegation(posID) called
   → posState.IsDelegated() == false → returns nil (no-op)
   → position still exists in Positions store

3. Staking unbonding period elapses
   → tokens arrive at posDelAddr as bank balance

4. vestingOwner calls MsgWithdrawFromTier(positionId)
   → validateWithdrawFromTier: owner ✓, exit triggered ✓, exit elapsed ✓,
     not delegated ✓, not unbonding ✓ → passes
   → SpendableCoins(posDelAddr) sent to vestingOwner's bank balance
   → position deleted

5. Assert: vestingOwner.BankBalance increased by position amount
   Assert: DelegatedVesting is stale-high → LockedCoins == 0
   Assert: vestingOwner can now spend the returned tokens (vesting bypass)
```

func (k Keeper) validateUndelegatePosition(ctx context.Context, pos types.PositionState, owner string) error {
	if !pos.IsOwner(owner) {
		return types.ErrNotPositionOwner
	}

	if !pos.IsDelegated() {
		return types.ErrPositionNotDelegated
	}

	if !pos.HasTriggeredExit() {
		return types.ErrExitNotTriggered
	}

	sdkCtx := sdk.UnwrapSDKContext(ctx)
	if !pos.CompletedExitLockDuration(sdkCtx.BlockTime()) {
		return types.ErrExitLockDurationNotReached
	}

	// skip check for zero amount as we want those positions to be able to close their position properly

	return nil
}

func (k Keeper) validateWithdrawFromTier(ctx context.Context, pos types.PositionState, owner string) error {
	if !pos.IsOwner(owner) {
		return types.ErrNotPositionOwner
	}

	if !pos.HasTriggeredExit() {
		return types.ErrExitNotTriggered
	}

	sdkCtx := sdk.UnwrapSDKContext(ctx)
	if !pos.CompletedExitLockDuration(sdkCtx.BlockTime()) {
		return types.ErrExitLockDurationNotReached
	}

	if pos.IsDelegated() {
		return types.ErrPositionDelegated
	}

	isUnbonding, err := k.isUnbonding(ctx, pos.DelegatorAddress)
	if err != nil {
		return err
	}
	if isUnbonding {
		return types.ErrPositionUnbonding
	}

	return nil
}
