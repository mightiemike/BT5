Audit Report

## Title
`SwapAllowlistExtension` checks router address instead of end user, allowing any caller to bypass the swap allowlist via `MetricOmmSimpleRouter` — (`metric-periphery/contracts/extensions/SwapAllowlistExtension.sol`)

## Summary
`SwapAllowlistExtension.beforeSwap` evaluates the `sender` argument forwarded by the pool, which is `msg.sender` of the `pool.swap()` call — the router contract, not the end user. A pool admin who allowlists the router so that allowlisted users can swap through it simultaneously opens the pool to every user on the network. There is no configuration that simultaneously permits allowlisted users to use the router and blocks non-allowlisted users.

## Finding Description
`SwapAllowlistExtension.beforeSwap` enforces:

```solidity
// metric-periphery/contracts/extensions/SwapAllowlistExtension.sol L37
if (!allowAllSwappers[msg.sender] && !allowedSwapper[msg.sender][sender]) {
    revert IMetricOmmPoolActions.NotAllowedToSwap();
}
```

Here `msg.sender` is the pool (the pool calls the extension) and `sender` is the value the pool forwards from its own `msg.sender`. In `MetricOmmPool.swap`, the pool passes `msg.sender` directly as the `sender` argument to `_beforeSwap`:

```solidity
// metric-core/contracts/MetricOmmPool.sol L230-L240
_beforeSwap(
    msg.sender,   // ← whoever called pool.swap()
    recipient, ...
);
```

`ExtensionCalling._beforeSwap` then encodes and forwards this `sender` value unchanged to the extension. When a user calls `MetricOmmSimpleRouter.exactInputSingle`, the router calls `pool.swap(...)` — making the router the `msg.sender` of that call:

```solidity
// metric-periphery/contracts/MetricOmmSimpleRouter.sol L72-L80
(int128 amount0Delta, int128 amount1Delta) = IMetricOmmPoolActions(params.pool)
    .swap(params.recipient, params.zeroForOne, ...);
```

The extension therefore evaluates `allowedSwapper[pool][router]`, not `allowedSwapper[pool][end_user]`. The pool admin faces an impossible choice: not allowlisting the router means allowlisted users cannot use it; allowlisting the router means every user on the network can bypass the allowlist by routing through it.

## Impact Explanation
Any non-allowlisted user can execute swaps on a curated pool intended to restrict trading to a specific set of addresses (e.g., KYC'd wallets, protocol-internal actors). The allowlist protection silently fails open for all router-mediated swaps, breaking the core invariant of `SwapAllowlistExtension`: "only allowlisted addresses may swap on this pool." This is a direct policy bypass that constitutes broken core pool functionality — the access-control mechanism governing who may trade on a restricted pool is rendered ineffective.

## Likelihood Explanation
`MetricOmmSimpleRouter` is the standard, publicly deployed periphery swap entry point callable by any user permissionlessly. A pool admin who deploys a curated pool and wants to support router-mediated swaps for allowlisted users has no choice but to allowlist the router, which opens the gate to everyone. No privileged access, no special setup, and no non-standard tokens are required.

## Recommendation
Pass the original end user through the extension chain rather than the immediate `msg.sender` of `pool.swap()`. One approach: the router already stores the originating user in transient storage via `_setNextCallbackContext(params.pool, CALLBACK_MODE_JUST_PAY, msg.sender, params.tokenIn)` — this `msg.sender` (the true end user) could be forwarded as an additional `originator` field to extensions. Alternatively, redesign the extension to check `sender` only when `sender` is not a known router, and check a router-forwarded user field otherwise. At minimum, document that allowlisting the router defeats the allowlist and provide a router variant that forwards the originating user.

## Proof of Concept
```
Setup:
  - Deploy pool with SwapAllowlistExtension
  - Pool admin calls setAllowedToSwap(pool, alice, true)   // alice is allowlisted
  - Pool admin calls setAllowedToSwap(pool, router, true)  // router allowlisted so alice can use it

Attack (Bob, not allowlisted):
  1. Bob calls router.exactInputSingle({pool: pool, ...})
  2. Router calls pool.swap(...) — msg.sender of pool.swap() is router
  3. Pool calls _beforeSwap(sender=router, ...)
  4. Extension evaluates: allowedSwapper[pool][router] == true → passes
  5. Bob's swap executes on the curated pool despite Bob never being allowlisted

Result:
  - allowedSwapper[pool][bob] == false, yet Bob swapped successfully
  - The allowlist invariant is broken for all router-mediated swaps
```

Foundry test outline:
1. Deploy `SwapAllowlistExtension`, pool, and `MetricOmmSimpleRouter`.
2. Admin calls `setAllowedToSwap(pool, alice, true)` and `setAllowedToSwap(pool, router, true)`.
3. `vm.prank(bob)` → `router.exactInputSingle(...)` — assert swap succeeds.
4. Confirm `allowedSwapper[pool][bob] == false` while the swap completed. [1](#0-0) [2](#0-1) [3](#0-2) [4](#0-3)
