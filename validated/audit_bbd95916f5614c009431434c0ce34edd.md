### Title
`op_multiply` First-Argument Parsing Work Uncharged — Undercharged Execution Per Cost Unit - (File: src/more_ops.rs)

### Summary
`op_multiply` parses its first argument into a bignum via `int_atom` without charging any per-byte cost for that work. Every other arithmetic operator (`op_add`, `op_subtract`, `op_div`, `op_mod`, `op_divmod`, `op_ash`, `op_lsh`, `op_logand`, etc.) charges `ARITH_COST_PER_BYTE` (or equivalent) for every byte of every argument. `op_multiply` silently skips this charge for the first operand, allowing an attacker to consume real CPU parsing work at a fraction of the intended cost.

### Finding Description

In `op_multiply` (`src/more_ops.rs`, lines 592–655), the loop body for the first argument is:

```rust
if first_iter {
    (total, l0) = int_atom(a, arg, "*")?;   // parses up to 256 bytes into a BigInt
    if l0 > 256 {
        return Err(EvalErr::InvalidOpArg(arg, "*".to_string()));
    }
    first_iter = false;
    continue;                                // ← no cost charged for l0 bytes
}
``` [1](#0-0) 

`int_atom` calls `number_from_u8`, which allocates a `BigInt` and copies up to 256 bytes. No `check_cost` is called, and no `ARITH_COST_PER_ARG` or per-byte cost is added before the `continue`. The running `cost` variable remains at `MUL_BASE_COST = 92` after the first argument is fully parsed. [2](#0-1) 

Contrast with `op_add` slow path, which charges both `ARITH_COST_PER_ARG` and `ARITH_COST_PER_BYTE * buf.len()` for every argument including the first:

```rust
cost += ARITH_COST_PER_ARG;
// ...
cost += ARITH_COST_PER_BYTE * (buf.len() as Cost);
check_cost(cost, max_cost)?;
``` [3](#0-2) 

And `op_subtract` slow path, which also charges `ARITH_COST_PER_ARG` and per-byte cost for the first argument: [4](#0-3) 

The cost constants involved:

```
ARITH_BASE_COST      = 99
ARITH_COST_PER_ARG   = 320
ARITH_COST_PER_BYTE  = 3
MUL_BASE_COST        = 92
``` [5](#0-4) 

### Impact Explanation

For a single-argument call `(* x)` where `x` is a 256-byte atom:

| Operator | Charged cost (excl. malloc) | Actual parsing work |
|---|---|---|
| `(+ x)` | `99 + 320 + 256×3 = 1187` | parse 256 bytes |
| `(* x)` | `92` | parse 256 bytes |

Both calls do identical CPU work (`number_from_u8` on 256 bytes), but `op_multiply` charges only 92 cost units vs 1187 for `op_add` — a **12.9× undercharge** for the parsing step. Including `malloc_cost` (256 × 10 = 2560), the totals are 2652 vs 3747, still a **~41% undercharge**.

With a global cost limit of 11 billion (Chia's generator limit), an attacker can execute:
- `(* x)` calls: ~4.1 million (each parsing 256 bytes)
- `(+ x)` calls: ~2.9 million (each parsing 256 bytes)

The attacker performs ~41% more bignum-parsing CPU work than the cost model permits, for the same cost budget. This directly degrades block-validation throughput on full nodes and consensus peers.

`(* x)` with a single argument is semantically equivalent to `(+ x)` — both return `x` re-encoded as a number — so the attacker loses nothing by substituting one for the other. [6](#0-5) 

### Likelihood Explanation

The entry path is direct: any attacker who can submit a CLVM puzzle solution or generator (i.e., any Chia transaction) can craft a program consisting of a long sequence of `(* <256-byte-atom>)` calls. No special privileges, no social engineering, and no dependency on node configuration are required. The `op_multiply` function is dispatched unconditionally for opcode `18` in `ChiaDialect::op`. [7](#0-6) 

### Recommendation

Charge per-byte cost for the first argument in `op_multiply`, consistent with all other arithmetic operators. Before the `continue` on the first iteration, add:

```rust
cost += MUL_COST_PER_OP;                              // or a dedicated LOAD cost
cost += l0 as Cost * MUL_LINEAR_COST_PER_BYTE;
check_cost(cost, max_cost)?;
```

Alternatively, introduce a dedicated `MUL_LOAD_COST_PER_BYTE` constant calibrated to the actual cost of `number_from_u8`, and apply it to `l0` before the `continue`.

### Proof of Concept

```clvm
; CLVM program: repeat (* <256-byte-atom>) N times within cost budget
; Each call costs only 92 + 2560 = 2652 units instead of the ~3747
; that (+ <256-byte-atom>) would cost for identical parsing work.

(a
  (i (= (+ 0x<256-byte-atom>) 0x<256-byte-atom>)   ; baseline: costs 3747
     (q . 0)
     (q x))
  (q . ()))

; vs attacker's version:
(a
  (i (= (* 0x<256-byte-atom>) 0x<256-byte-atom>)   ; undercharged: costs 2652
     (q . 0)
     (q x))
  (q . ()))
```

The corrupted invariant is the cost returned by `op_multiply` for a single large argument: it is `MUL_BASE_COST + malloc_cost` instead of `MUL_BASE_COST + MUL_COST_PER_OP + l0 * MUL_LINEAR_COST_PER_BYTE + malloc_cost`, causing the interpreter to undercount CPU work by up to 1095 cost units per invocation. ```rust
pub fn op_multiply(
    a: &mut Allocator,
    mut input: NodePtr,
    max_cost: Cost,
    _flags: ClvmFlags,
) -> Response {
    let mut cost: Cost = MUL_BASE_COST;
    let mut first_iter: bool = true;
    let mut total: Number = 1.into();
    let mut l0: usize = 0;
    while let Some((arg, rest)) = a.next(input) {
        input = rest;
        if first_iter {
            (total, l0) = int_atom(a, arg, "*")?;
            if l0 > 256 {
                return Err(EvalErr::InvalidOpArg(arg, "*".to_string()));
            }
            first_iter = false;
            continue;
        }

        cost += MUL_COST_PER_OP;
        #[cfg(not(feature = "no-fastpath"))]
        match a.node(arg) {
            NodeVisitor::Buffer(buf) => {
                let l1 = buf.len() as u64;
                if l1 > 256 {
                    return Err(EvalErr::InvalidOpArg(arg, "*".to_string()));
                }
                cost += (l0 as Cost + l1) * MUL_LINEAR_COST_PER_BYTE;
                cost += (l0 as Cost * l1) / MUL_SQUARE_COST_PER_BYTE_DIVIDER;
                check_cost(cost, max_cost)?;

                total *= number_from_u8(buf);
            }
            NodeVisitor::U32(val) => {
                let l1 = len_for_value(val) as u64;
                cost += (l0 as Cost + l1) * MUL_LINEAR_COST_PER_BYTE;
                cost += (l0 as Cost * l1) / MUL_SQUARE_COST_PER_BYTE_DIVIDER;
                check_cost(cost, max_cost)?;

                total *= val;
            }
            NodeVisitor::Pair(_, _) => {
                Err(EvalErr::InvalidOpArg(
                    arg,
                    "Requires Int Argument: *".to_string(),
                ))?;
            }
        }
        #[cfg(feature = "no-fastpath")]
        {
            let (n1, l1) = int_atom(a, arg, "*")?;
            let l1 = l1 as u64;
            if l1 > 256 {
                return Err(EvalErr::InvalidOpArg(arg, "*".to_string()));
            }
            cost += (l0 as Cost + l1) * MUL_LINEAR_COST_PER_BYTE;
            cost += (l0 as Cost * l1) / MUL_SQUARE_COST_PER_BYTE_DIVIDER;
            check_cost(cost, max_cost)?;

            total *= n1;
        }
        l0 = limbs_for_int(&total);
        if l0 > 1024 {
            return Err(EvalErr::InvalidOpArg(arg, "*".to_string()));
        }
    }
    let total = a.new_number(total)?;
    Ok(malloc_cost(a, cost, total))
}
