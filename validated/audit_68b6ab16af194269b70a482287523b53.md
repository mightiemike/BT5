Audit Report

## Title
`SwapAllowlistExtension` Checks Router Address Instead of End-User, Allowing Any Caller to Bypass Swap Allowlist via `MetricOmmSimpleRouter` ‚Äî (`metric-periphery/contracts/extensions/SwapAllowlistExtension.sol`)

## Summary
`SwapAllowlistExtension.beforeSwap` gates swaps by checking the `sender` argument, which the pool sets to `msg.sender` of the `pool.swap` call. When a user routes through `MetricOmmSimpleRouter`, the router is `msg.sender` to the pool, so the extension checks `allowedSwapper[pool][router]` rather than `allowedSwapper[pool][actualUser]`. Any pool that allowlists the router (required for router-mediated swaps to work) grants unrestricted swap access to every address on the network, completely nullifying the allowlist.

## Finding Description
The call chain is confirmed by the production code:

**Step 1:** `MetricOmmSimpleRouter.exactInputSingle` calls `pool.swap(...)` directly with no forwarding of the original `msg.sender` to the pool. [1](#0-0) 

**Step 2:** `ExtensionCalling._beforeSwap` receives `sender` (which is the router address, as it was `msg.sender` to the pool) and encodes it into the extension call. [2](#0-1) 

**Step 3:** `SwapAllowlistExtension.beforeSwap` checks `allowedSwapper[msg.sender][sender]` where `msg.sender` is the pool and `sender` is the router ‚Äî the actual end-user's address is never consulted. [3](#0-2) 

There is no mechanism in the extension, the pool, or the router to recover the original user's address. The `extensionData` field passed by the router is simply forwarded from `params.extensionData` supplied by the caller ‚Äî an untrusted, user-controlled value ‚Äî so it cannot be used to authenticate the real user without a trusted signature scheme. [4](#0-3) 

The dilemma is irresolvable under the current design: if the router is not allowlisted, no user can swap via the router; if it is allowlisted, every user can swap via the router regardless of their individual allowlist status.

## Impact Explanation
This breaks the core access-control invariant of `SwapAllowlistExtension`. Any pool using this extension for KYC, institutional, or permissioned access control is fully bypassed by any unprivileged address calling `exactInputSingle`, `exactInput`, `exactOutputSingle`, or `exactOutput` on the public `MetricOmmSimpleRouter`. The attacker can execute swaps in restricted pools ‚Äî draining liquidity at oracle-anchored prices or trading in pools explicitly intended to exclude them. This constitutes a broken core pool functionality and an admin-boundary break reachable by any unprivileged caller. [5](#0-4) 

## Likelihood Explanation
`MetricOmmSimpleRouter` is the standard, publicly deployed periphery entry point. Any pool using `SwapAllowlistExtension` that also supports router-mediated swaps (the expected common case) must allowlist the router, at which point the bypass is trivially reachable by any EOA or contract with zero special privileges, no setup, and no capital beyond the swap amount itself. The attack is repeatable indefinitely. [6](#0-5) 

## Recommendation
The extension must identify the economic actor, not the immediate `pool.swap` caller. The safest approach within the current architecture is to have the router encode the real user's address into `extensionData` and have the extension verify the caller is a trusted router before accepting the forwarded address. Alternatively, require allowlisted users to call `pool.swap` directly and explicitly document and enforce that the router must not be allowlisted ‚Äî but this breaks standard UX and must be enforced at the protocol level, not left to pool admins to discover. [7](#0-6) 

## Proof of Concept
```
Setup:
  - Pool P uses SwapAllowlistExtension E.
  - Pool admin: E.setAllowedToSwap(P, router, true)   // required for any router swap
  - Pool admin: E.setAllowedToSwap(P, alice, true)    // alice is the intended allowlisted user
  - bob is NOT in the allowlist.

Attack:
  1. bob calls MetricOmmSimpleRouter.exactInputSingle({pool: P, ...})
  2. Router calls P.swap(recipient, zeroForOne, amount, priceLimit, "", extensionData)
     ‚Üí msg.sender to pool = router address
  3. Pool calls _beforeSwap(router, ...)
  4. ExtensionCalling encodes sender=router and calls E.beforeSwap(router, ...)
  5. Extension checks: allowedSwapper[P][router] == true ‚Üí passes
  6. Swap executes for bob with no restriction.

Result: bob swaps in a pool he was explicitly excluded from.
``` [1](#0-0) [3](#0-2)

contract SwapAllowlistExtension is BaseMetricExtension, ISwapAllowlistExtension {
  mapping(address pool => mapping(address swapper => bool)) public allowedSwapper;
  mapping(address pool => bool) public allowAllSwappers;

  constructor(address factory_) BaseMetricExtension(factory_) {}

  function setAllowedToSwap(address pool_, address swapper, bool allowed) external onlyPoolAdmin(pool_) {
    allowedSwapper[pool_][swapper] = allowed;
    emit AllowedToSwapSet(pool_, swapper, allowed);
  }

  function setAllowAllSwappers(address pool_, bool allowed) external onlyPoolAdmin(pool_) {
    allowAllSwappers[pool_] = allowed;
    emit AllowAllSwappersSet(pool_, allowed);
  }

  function isAllowedToSwap(address pool_, address swapper) external view returns (bool) {
    return allowAllSwappers[pool_] || allowedSwapper[pool_][swapper];
  }

  function beforeSwap(address sender, address, bool, int128, uint128, uint256, uint128, uint128, bytes calldata)
    external
    view
    override
    returns (bytes4)
  {
    if (!allowAllSwappers[msg.sender] && !allowedSwapper[msg.sender][sender]) {
      revert IMetricOmmPoolActions.NotAllowedToSwap();
    }
    return IMetricOmmExtensions.beforeSwap.selector;
  }
