Audit Report

## Title
`SwapAllowlistExtension.beforeSwap` checks the intermediary router address instead of the originating user, enabling full allowlist bypass ā€” (`metric-periphery/contracts/extensions/SwapAllowlistExtension.sol`)

## Summary

`SwapAllowlistExtension.beforeSwap` gates swaps by checking `allowedSwapper[pool][sender]`, where `sender` is the pool's `msg.sender` ā€” the direct caller of `pool.swap`. When users route through `MetricOmmSimpleRouter`, `sender` becomes the router address, not the original user. A pool admin who allowlists the router to enable router-mediated swaps inadvertently grants swap access to every user, completely bypassing the curated-pool access control. Conversely, allowlisting individual users blocks them from swapping through the router.

## Finding Description

`SwapAllowlistExtension.beforeSwap` evaluates:

```solidity
if (!allowAllSwappers[msg.sender] && !allowedSwapper[msg.sender][sender]) {
    revert IMetricOmmPoolActions.NotAllowedToSwap();
}
``` [1](#0-0) 

Here `msg.sender` is the pool (the extension caller) and `sender` is the first argument forwarded by `ExtensionCalling._beforeSwap`:

```solidity
_callExtensionsInOrder(
    BEFORE_SWAP_ORDER,
    abi.encodeCall(IMetricOmmExtensions.beforeSwap, (sender, recipient, ...))
);
``` [2](#0-1) 

The pool sets `sender` to its own `msg.sender` ā€” whoever called `pool.swap`. In `MetricOmmSimpleRouter.exactInputSingle`, the router calls `pool.swap` directly:

```solidity
(int128 amount0Delta, int128 amount1Delta) = IMetricOmmPoolActions(params.pool)
    .swap(params.recipient, params.zeroForOne, ..., params.extensionData);
``` [3](#0-2) 

The pool receives `msg.sender = router`. It passes `sender = router` to `_beforeSwap`. The extension evaluates `allowedSwapper[pool][router]`, not `allowedSwapper[pool][user]`.

The same pattern applies to `exactInput`, `exactOutputSingle`, and `exactOutput`. [4](#0-3) 

`DepositAllowlistExtension` does not share this flaw ā€” it ignores `sender` and checks `owner` (the position owner), which is passed explicitly and correctly:

```solidity
function beforeAddLiquidity(address, address owner, ...) {
    if (!allowAllDepositors[msg.sender] && !allowedDepositor[msg.sender][owner]) {
``` [5](#0-4) 

## Impact Explanation

**Mode A (allowlist bypass):** A pool admin who allowlists the router address to enable router-mediated swaps for permitted users has effectively disabled the allowlist for all users. Any non-allowlisted EOA can call `exactInputSingle` / `exactInput` / `exactOutputSingle` / `exactOutput` and the extension passes because `allowedSwapper[pool][router] = true`. This is a complete, deterministic bypass of curated-pool access control ā€” a broken core pool functionality and admin-boundary break.

**Mode B (legitimate user DoS):** If the admin allowlists individual users (not the router), those users are blocked from swapping through the router despite being explicitly permitted, breaking core swap functionality for legitimate users.

## Likelihood Explanation

- `MetricOmmSimpleRouter` is the primary user-facing swap interface for EOAs.
- Pool admins enabling router support will naturally call `setAllowedToSwap(pool, router, true)`, triggering Mode A.
- No privileged access is required by the attacker; any EOA can call the router.
- The bypass is deterministic, requires no timing, oracle manipulation, or multi-step setup, and is repeatable indefinitely.

## Recommendation

The extension must check the original user, not the intermediary. Two options:

1. **Pass the original user through `extensionData`**: the router encodes `msg.sender` into `extensionData`; the extension decodes and verifies it. Requires a trusted encoding convention.
2. **Add an explicit `swapper` field to the pool's `swap` signature**: the pool accepts a `swapper` address (validated against `msg.sender` or a trusted forwarder list) and passes it to `_beforeSwap`. The extension checks that field instead of `sender`.

The `DepositAllowlistExtension` pattern ā€” checking `owner`, not `sender` ā€” is the correct model: gate the economically relevant actor, not the intermediary.

## Proof of Concept

1. Deploy a pool with `SwapAllowlistExtension` configured as a `beforeSwap` hook.
2. Pool admin calls `setAllowedToSwap(pool, router, true)` to enable router-mediated swaps.
3. Non-allowlisted user Alice calls `router.exactInputSingle({pool: pool, ...})`.
4. Router calls `pool.swap(recipient, ...)` ā€” pool's `msg.sender = router`.
5. Pool calls `_beforeSwap(sender=router, ...)`.
6. Extension evaluates `allowedSwapper[pool][router] = true` ā†’ passes.
7. Alice's swap executes on the curated pool despite never being allowlisted.

Foundry test: deploy pool + extension, set `allowedSwapper[pool][router] = true`, call `exactInputSingle` from an address not in the allowlist, assert the swap succeeds (no `NotAllowedToSwap` revert).

    int128 amount = MetricOmmSwapInputs.asAmountSpecifiedIn(params.amountIn);

      address pool = params.pools[i];
      bool zeroForOne = MetricOmmSwapPath.resolveZeroForOneBitmap(params.zeroForOneBitMap, i);

      (int128 amount0Delta, int128 amount1Delta) = IMetricOmmPoolActions(pool)
        .swap(

          zeroForOne,

          "",
