### Title
Bonus Rewards Accrue Over the Post-Exit-Elapse Gap After `ClearPosition` Resets `ExitUnlockAt` - (File: `x/tieredrewards/keeper/msg_server.go`)

---

### Summary

`MsgClearPosition` settles rewards before resetting exit timestamps, but the resulting `LastBonusAccrual` checkpoint is left at the old `ExitUnlockAt` value while `ExitUnlockAt` itself is zeroed out. A subsequent `MsgClaimTierRewards` then computes bonus for the entire interval `[ExitUnlockAt, now]` with no cap, including the "dead" window between exit-elapse and the clearing call — a period during which bonus was explicitly supposed to be zero.

---

### Finding Description

The invariant of the tiered-rewards module is that bonus accrual stops permanently at `ExitUnlockAt` once the exit commitment elapses. The ADR states: *"Bonus stops when the exit commitment elapses."*

`ClearPosition` is designed to cancel a triggered exit and resume bonus accrual. It correctly calls `claimRewards` first to settle all bonus up to `ExitUnlockAt` before clearing the exit state. However, the sequence leaves a corrupted checkpoint:

**Step 1 — `claimRewards` inside `ClearPosition`** (called at block time T₂, where T₂ > `ExitUnlockAt`):

`processEventsAndClaimBonus` calls `applyBonusAccrualCheckpoint`, which detects that the exit has elapsed and caps the checkpoint:

```go
// bonus_rewards.go:15-21
func applyBonusAccrualCheckpoint(pos *types.Position, blockTime time.Time) {
    accrualEnd := blockTime
    if pos.CompletedExitLockDuration(blockTime) {
        accrualEnd = pos.ExitUnlockAt   // ← set to ExitUnlockAt, not T₂
    }
    pos.UpdateLastBonusAccrual(accrualEnd)
}
```

After this call: `pos.LastBonusAccrual = ExitUnlockAt`. [1](#0-0) 

**Step 2 — `ClearExit` zeros out `ExitUnlockAt`**:

```go
// msg_server.go:406-412
pos, _, _, err = ms.claimRewards(ctx, pos)
// ...
pos.ClearExit(sdkCtx.BlockTime())   // sets ExitTriggeredAt = zero, ExitUnlockAt = zero
```

After this call: `pos.ExitUnlockAt = zero`, but `pos.LastBonusAccrual` still holds the old `ExitUnlockAt` timestamp. [2](#0-1) 

**Step 3 — Next `ClaimTierRewards` at time T₃**:

`processEventsAndClaimBonus` starts the segment at `pos.LastBonusAccrual` (= old `ExitUnlockAt`) and applies no cap because `ExitUnlockAt` is now zero:

```go
// claim_rewards.go:165, 26-28
segmentStart := pos.LastBonusAccrual   // = old ExitUnlockAt

// computeSegmentBonus:
if !pos.ExitUnlockAt.IsZero() && segmentEnd.After(pos.ExitUnlockAt) {
    segmentEnd = pos.ExitUnlockAt      // ← skipped because ExitUnlockAt == zero
}
```

Bonus is therefore computed for `[ExitUnlockAt, T₃]`, which includes the interval `[ExitUnlockAt, T₂]` — the dead window when the exit had already elapsed and bonus should have been zero. [3](#0-2) [4](#0-3) 

---

### Impact Explanation

An attacker who owns a tier position can drain the bonus rewards pool by repeatedly:

1. Triggering exit (`MsgTriggerExitFromTier`)
2. Waiting for `ExitUnlockAt` to elapse (bonus stops)
3. Waiting an additional arbitrary duration Δ (the longer, the more bonus stolen)
4. Calling `MsgClearPosition` (resets exit, leaves `LastBonusAccrual = ExitUnlockAt`)
5. Calling `MsgClaimTierRewards` (receives bonus for `[ExitUnlockAt, now]`, including Δ)

The stolen bonus per cycle is approximately:

```
shares × tokensPerShare × bonusApy × Δ / SecondsPerYear
```

The attacker can repeat this cycle indefinitely, extracting bonus from the `RewardsPoolName` module account for time windows during which they were not entitled to any bonus. The corrupted value is the bonus rewards pool balance, which is drained by unauthorized `SendCoinsFromModuleToAccount` calls. [5](#0-4) 

---

### Likelihood Explanation

Any position owner can trigger this with four standard, unprivileged transactions (`MsgTriggerExitFromTier`, wait, `MsgClearPosition`, `MsgClaimTierRewards`). No special role, leaked key, or social engineering is required. The attack is repeatable and scales with position size and waiting time.

---

### Recommendation

After `ClearExit` zeroes `ExitUnlockAt`, advance `LastBonusAccrual` to the current block time so that the next claim only accrues from the moment the exit was cleared, not from the old `ExitUnlockAt`:

```go
// msg_server.go — ClearPosition, after ClearExit
pos.ClearExit(sdkCtx.BlockTime())
pos.UpdateLastBonusAccrual(sdkCtx.BlockTime())  // ← add this
```

This ensures the dead window `[ExitUnlockAt, T_clear]` is never retroactively credited as a bonded segment.

---

### Proof of Concept

```
T=0:   LockTier(amount=X, tier=1, validator=V)
       → position created, LastBonusAccrual = T=0, ExitUnlockAt = zero

T=100: TriggerExitFromTier(position_id)
       → ExitUnlockAt = T=100 + ExitDuration (e.g. T=200)

T=201: Exit has elapsed. Bonus should be zero from T=200 onward.

T=500: ClearPosition(position_id)
       → claimRewards: bonus paid for [T=0, T=200], LastBonusAccrual = T=200
       → ClearExit: ExitUnlockAt = zero, ExitTriggeredAt = zero
       → position saved: LastBonusAccrual=T=200, ExitUnlockAt=zero

T=501: ClaimTierRewards(position_id)
       → processEventsAndClaimBonus: segmentStart = T=200, no cap (ExitUnlockAt=zero)
       → bonus paid for [T=200, T=501]
       → includes [T=200, T=500]: 300 time-units of UNAUTHORIZED bonus
```

The attacker receives bonus for the 300-unit dead window `[T=200, T=500]` during which the exit had elapsed and bonus was supposed to be zero. The `RewardsPoolName` module account is debited for this unauthorized amount. [6](#0-5) [1](#0-0) [7](#0-6)

func (ms msgServer) ClearPosition(ctx context.Context, msg *types.MsgClearPosition) (*types.MsgClearPositionResponse, error) {
	if err := msg.Validate(); err != nil {
		return nil, err
	}

	pos, err := ms.getPositionState(ctx, msg.PositionId)
	if err != nil {
		return nil, err
	}

	if err := ms.validateClearPosition(ctx, pos, msg.Owner); err != nil {
		return nil, err
	}

	if !pos.HasTriggeredExit() {
		return &types.MsgClearPositionResponse{PositionId: pos.Id}, nil
	}

	pos, _, _, err = ms.claimRewards(ctx, pos)
	if err != nil {
		return nil, err
	}

	sdkCtx := sdk.UnwrapSDKContext(ctx)
	pos.ClearExit(sdkCtx.BlockTime())

	if err := ms.setPosition(ctx, pos.Position, nil); err != nil {
		return nil, err
	}

	if err := sdkCtx.EventManager().EmitTypedEvent(&types.EventExitCleared{
		PositionId: pos.Id,
		TierId:     pos.TierId,
		Owner:      pos.Owner,
	}); err != nil {
		return nil, err
	}

	return &types.MsgClearPositionResponse{PositionId: pos.Id}, nil
}

func (k Keeper) processEventsAndClaimBonus(ctx context.Context, pos *types.PositionState) (sdk.Coins, error) {
	// Rewards should have been claimed before undelegation
	if !pos.IsDelegated() {
		return sdk.NewCoins(), nil
	}

	valAddr, err := sdk.ValAddressFromBech32(pos.Delegation.ValidatorAddress)
	if err != nil {
		return nil, err
	}

	events, err := k.getValidatorEventsSince(ctx, valAddr, pos.LastEventSeq)
	if err != nil {
		return nil, err
	}

	sdkCtx := sdk.UnwrapSDKContext(ctx)
	blockTime := sdkCtx.BlockTime()

	totalBonus := math.ZeroInt()
	// Use the persisted bonded state from the last replay, not a hardcoded default.
	// This prevents overpaying bonus for unbonded gaps between claims.
	bonded := pos.LastKnownBonded
	segmentStart := pos.LastBonusAccrual

	tier, err := k.getTier(ctx, pos.TierId)
	if err != nil {
		return nil, err
	}

	for _, entry := range events {
		evt := entry.Event

		if bonded {
			// Compute bonus for the bonded segment [segmentStart, eventTime]
			// using the snapshot rate at the event.
			bonus := k.computeSegmentBonus(*pos, tier, segmentStart, evt.Timestamp, evt.TokensPerShare)
			totalBonus = totalBonus.Add(bonus)
		}

		// Update bonded state based on event type.
		switch evt.EventType {
		case types.ValidatorEventType_VALIDATOR_EVENT_TYPE_UNBOND:
			bonded = false
		case types.ValidatorEventType_VALIDATOR_EVENT_TYPE_BOND:
			bonded = true
		case types.ValidatorEventType_VALIDATOR_EVENT_TYPE_SLASH:
			// Slash doesn't change bonded state.
		}

		segmentStart = evt.Timestamp
		pos.UpdateLastEventSeq(entry.Seq)

		// Decrement reference count.
		if err := k.decrementEventRefCount(ctx, valAddr, entry.Seq); err != nil {
			return nil, err
		}
	}

	val, err := k.stakingKeeper.GetValidator(ctx, valAddr)
	if err != nil {
		return nil, err
	}
	// Defensive: validator bond status check
	if bonded && val.IsBonded() {
		currentRate, err := k.getTokensPerShare(ctx, valAddr)
		if err != nil {
			return nil, err
		}
		bonus := k.computeSegmentBonus(*pos, tier, segmentStart, blockTime, currentRate)
		totalBonus = totalBonus.Add(bonus)
	}

	applyBonusAccrualCheckpoint(&pos.Position, blockTime)
