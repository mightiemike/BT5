### Title
Difficulty Calculation for Fork Blocks Uses Mainchain Ancestor Instead of Fork Ancestor at Adjustment Boundary - (File: `contract/src/bitcoin.rs`, `contract/src/dogecoin.rs`)

---

### Summary

When validating a fork block at a difficulty adjustment boundary, `get_next_work_required` fetches the first block of the difficulty period via `blocks_getter.get_header_by_height(first_block_height)`. This function always resolves against `mainchain_height_to_header`, returning the **mainchain** block at that height — not the fork's actual ancestor. If the fork diverges before `first_block_height`, the difficulty calculation silently uses the wrong block's timestamp, producing an incorrect expected target. The contract then enforces that the submitted block's `bits` matches this incorrect expected target, accepting a fork block whose difficulty does not conform to the protocol's actual rules. The Dogecoin module even contains an explicit TODO comment acknowledging this exact uncertainty.

---

### Finding Description

**Root cause — `bitcoin.rs`:** [1](#0-0) 

`first_block_height` is computed as `prev_block_header.block_height - (difficulty_adjustment_interval - 1)`. The call to `blocks_getter.get_header_by_height(first_block_height)` resolves through: [2](#0-1) 

This always reads from `mainchain_height_to_header`, which maps heights to **mainchain** block hashes. When the block being validated is on a fork that diverged before `first_block_height`, the fork's actual ancestor at that height is a different block with a different timestamp. The code uses the mainchain block's timestamp as `first_block_time`, corrupting the `actual_time_taken` computation in `calculate_next_work_required`. [3](#0-2) 

**Root cause — `dogecoin.rs` (with developer acknowledgment):** [4](#0-3) 

The TODO comment — *"check if it is correct to get block header by height from mainchain without looping to find the ancestor"* — is a direct developer acknowledgment that this assumption may be wrong. The same faulty pattern is present: `get_header_by_height` returns the mainchain block, not the fork's ancestor.

**Broken invariant:** The contract's `check_pow` enforces `expected_bits == block_header.bits`. When `expected_bits` is computed from the wrong ancestor's timestamp, this check enforces a protocol-incorrect difficulty. A fork block whose `bits` satisfies the corrupted expected value — but not the correct protocol value — passes validation. [5](#0-4) 

---

### Impact Explanation

An attacker who submits a fork diverging before a difficulty adjustment boundary can craft the fork's `prev_block_header.time` (the last block before the adjustment) to make `actual_time_taken = prev_block_time - mainchain_first_block_time` appear up to 4× the target timespan. This drives `calculate_next_work_required` to return the maximum-relaxed target (4× easier difficulty). The contract then requires the fork's adjustment block to have `bits` matching this relaxed target — and accepts it. Subsequent fork blocks inherit the relaxed difficulty, allowing the attacker to mine them faster. If the fork accumulates more chainwork than the mainchain, `submit_block_header_inner` promotes it via `reorg_chain`, corrupting the canonical chain state and invalidating SPV proofs that downstream consumers rely on. [6](#0-5) 

---

### Likelihood Explanation

The attacker entry path is `submit_blocks`, which is gated by `#[trusted_relayer]`. Becoming a trusted relayer requires staking NEAR tokens — a permissionless economic action, not a privileged role. Once staked, the attacker can submit arbitrary fork headers. The fork must span at least one full difficulty period (2016 blocks for Bitcoin) before the adjustment boundary, which requires significant but not infeasible mining resources. The Dogecoin path is more accessible because Dogecoin's post-145000 `difficulty_adjustment_interval` is 1, meaning every block is an adjustment boundary — the attacker only needs to diverge one block before the target height. [7](#0-6) 

---

### Recommendation

Replace `get_header_by_height(first_block_height)` with a backward traversal from `prev_block_header` along the fork's own `prev_block_hash` chain until reaching `first_block_height`. This mirrors Bitcoin Core's reference implementation, which walks the ancestor chain rather than looking up by height in a canonical index. The `get_prev_header` method already exists on `BlocksGetter` and can be used for this traversal. [8](#0-7) 

---

### Proof of Concept

**Setup (Dogecoin — adjustment interval = 1 post-height 145000, making every block a boundary):**

1. Attacker stakes NEAR tokens to become a trusted relayer.
2. Attacker submits a valid fork block at height H (diverging from the mainchain at H−1). The fork's `prev_block_hash` points to a mainchain block, so `get_prev_header` resolves correctly.
3. At height H+1 (a difficulty adjustment boundary since interval=1), `get_next_work_required` is called. It computes `height_first = H` and calls `get_header_by_height(H)`.
4. If the fork diverged at H−1, the mainchain block at height H is the mainchain's block, not the fork's block at H. The attacker's fork block at H has an attacker-controlled timestamp.
5. The attacker sets the fork's block-H timestamp such that `prev_block_time - mainchain_block_H_time` equals `retarget_timespan + retarget_timespan/2` (the maximum), driving `modulated_timespan` to `max_timespan` and producing the easiest allowed `bits`.
6. The contract computes `expected_bits` from this manipulated timespan and requires the submitted block to have those bits — which the attacker mines trivially.
7. The fork block is accepted with protocol-incorrect (relaxed) difficulty. Repeated over subsequent blocks, the fork accumulates chainwork and triggers `reorg_chain`, replacing the canonical chain. [9](#0-8) [10](#0-9)

    fn reorg_chain(&mut self, fork_tip_header: ExtendedHeader, last_main_chain_block_height: u64) {
        let fork_tip_height = fork_tip_header.block_height;
        if last_main_chain_block_height > fork_tip_height {
            // If we see that main chain is longer than fork we first garbage collect
            // outstanding main chain blocks:
            //
            //      [m1] - [m2] - [m3] - [m4] <- We should remove [m4]
            //     /
            // [m0]
            //     \
            //      [f1] - [f2] - [f3]
            for height in (fork_tip_height + 1)..=last_main_chain_block_height {
                let current_main_chain_blockhash = self
                    .mainchain_height_to_header
                    .get(&height)
                    .unwrap_or_else(|| env::panic_str("cannot get a block"));
                self.remove_block_header(&current_main_chain_blockhash);
                self.mainchain_height_to_header.remove(&height);
            }
        }

        // Now we are in a situation where mainchain is equivalent to fork size:
        //
        //      [m1] - [m2] - [m3] - [m4] <- main tip
        //     /
        // [m0]
        //     \
        //      [f1] - [f2] - [f3] - [f4] <- fork tip
        //
        //
        // Or in a situation where it is shorter:
        //
        //      [m1] - [m2] - [m3] <- main tip
        //     /
        // [m0]
        //     \
        //      [f1] - [f2] - [f3] - [f4] <- fork tip

        let fork_tip_hash = fork_tip_header.block_hash.clone();
        let mut fork_header_cursor = fork_tip_header;

        while !self
            .mainchain_header_to_height
            .contains_key(&fork_header_cursor.block_hash)
        {
            let prev_block_hash = fork_header_cursor.block_header.prev_block_hash;
            let current_block_hash = fork_header_cursor.block_hash;
            let current_height = fork_header_cursor.block_height;

            // Inserting the fork block into the main chain, if some mainchain block is occupying
            // this height let's save its hashcode
            let main_chain_block = self
                .mainchain_height_to_header
                .insert(&current_height, &current_block_hash);
            self.mainchain_header_to_height
                .insert(&current_block_hash, &current_height);

            // If we found a mainchain block at the current height than remove this block from the
            // header pool and from the header -> height map
            if let Some(current_main_chain_blockhash) = main_chain_block {
                self.remove_block_header(&current_main_chain_blockhash);
            }

            // Switch iterator cursor to the previous block in fork
            fork_header_cursor = self
                .headers_pool
                .get(&prev_block_hash)
                .unwrap_or_else(|| env::panic_str("previous fork block should be there"));
        }

        // Updating tip of the new main chain
        self.mainchain_tip_blockhash = fork_tip_hash;
    }

// source https://github.com/dogecoin/dogecoin/blob/2c513d0172e8bc86fe9a337693b26f2fdf68a013/src/dogecoin.cpp#L41
fn calculate_next_work_required(
    config: &DogecoinConfig,
    prev_block_header: &ExtendedHeader,
    first_block_time: i64,
) -> u32 {
    let retarget_timespan = config.pow_target_timespan;
    let modulated_timespan = i64::from(prev_block_header.block_header.time) - first_block_time;

    let mut modulated_timespan = retarget_timespan + (modulated_timespan - retarget_timespan) / 8;

    let min_timespan = retarget_timespan - (retarget_timespan / 4);
    let max_timespan = retarget_timespan + (retarget_timespan / 2);

    if modulated_timespan < min_timespan {
        modulated_timespan = min_timespan;
    } else if modulated_timespan > max_timespan {
        modulated_timespan = max_timespan;
    }

    let new_target = target_from_bits(prev_block_header.block_header.bits);

    let (mut new_target, new_target_overflow) =
        new_target.overflowing_mul(<i64 as TryInto<u64>>::try_into(modulated_timespan).unwrap());
    require!(!new_target_overflow, "new target overflow");
    new_target =
        new_target / U256::from(<i64 as TryInto<u64>>::try_into(retarget_timespan).unwrap());

    if new_target > config.pow_limit {
        new_target = config.pow_limit;
    }

    new_target.target_to_bits()
}
