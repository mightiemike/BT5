### Title
`calculate_next_base_gas_price` permanently returns `GasPrice(0)` when `price` is zero due to integer division truncation — (`crates/apollo_consensus_orchestrator/src/fee_market/mod.rs`)

### Summary

When `l2_gas_price` is `GasPrice(0)` and below `min_gas_price`, the gradual-recovery formula `price.0 / MIN_GAS_PRICE_INCREASE_DENOMINATOR` integer-divides to zero, so the function returns `GasPrice(0)` on every subsequent call. The gas price is permanently stuck at zero and can never recover toward the minimum, producing a wrong L2 gas price committed to the block header.

### Finding Description

`calculate_next_base_gas_price` in `crates/apollo_consensus_orchestrator/src/fee_market/mod.rs` implements an EIP-1559-style gradual recovery when the current price is below the configured minimum:

```rust
if price < min_gas_price {
    let max_increase = price.0 / MIN_GAS_PRICE_INCREASE_DENOMINATOR;  // line 106
    let adjusted = price.0 + max_increase;                             // line 107
    let adjusted_price = adjusted.min(min_gas_price.0);               // line 109
    return GasPrice(adjusted_price);                                   // line 114
}
```

`MIN_GAS_PRICE_INCREASE_DENOMINATOR` is the constant `333`.

When `price == GasPrice(0)`:

| Step | Expression | Value |
|---|---|---|
| `max_increase` | `0u128 / 333` | `0` (integer truncation) |
| `adjusted` | `0 + 0` | `0` |
| `adjusted_price` | `min(0, min_gas_price.0)` | `0` |
| return | `GasPrice(0)` | stuck forever |

Every subsequent call with `price == 0` returns `GasPrice(0)`. The price can never increase because the only source of growth is `price.0 / 333`, which is zero when `price.0 == 0`. This is the exact integer-division-truncation pattern from the external report: a zero initial value makes the proportional step vanish, permanently freezing the state.
