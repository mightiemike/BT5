### Title
Deprecated `verify_transaction_inclusion` Remains Publicly Callable, Bypassing 64-Byte Transaction Forgery Protection — (File: `contract/src/lib.rs`)

---

### Summary

`verify_transaction_inclusion` was deprecated in favour of `verify_transaction_inclusion_v2`, which adds a mandatory coinbase Merkle proof check to close the 64-byte transaction forgery vulnerability. However, the deprecated function carries no runtime guard: any unprivileged NEAR caller can invoke it directly, bypassing the coinbase validation entirely and obtaining a forged `true` proof result for a transaction that does not exist.

---

### Finding Description

`verify_transaction_inclusion_v2` was introduced specifically to mitigate the 64-byte Merkle-proof forgery attack (documented at https://www.bitmex.com/blog/64-Byte-Transactions). Its fix is a coinbase Merkle proof check that must pass before the inner proof is evaluated: [1](#0-0) 

Internally, after the coinbase check passes, it delegates to the deprecated function: [2](#0-1) 

The deprecated function itself is still a fully public, `#[pause]`-gated method with no access control and no runtime rejection of callers: [3](#0-2) 

The function's own inline warning acknowledges the exact attack surface: [4](#0-3) 

The `#[deprecated]` Rust attribute produces only a compile-time warning for callers in the same codebase. It has zero effect on external NEAR callers invoking the method by name at runtime. The "rejected" status of this function is therefore never enforced at the protocol boundary.

---

### Impact Explanation

An attacker who supplies a 64-byte internal Merkle tree node hash as `tx_id` — with a crafted `merkle_proof` that reconstructs the block's `merkle_root` — will receive `true` from `verify_transaction_inclusion`. Any consumer contract that relies on this return value to authorise a cross-chain action (bridge release, atomic swap settlement, lending collateral unlock) will be deceived into treating a non-existent Bitcoin transaction as confirmed. The corrupted proof result is the exact value that downstream contracts consume.

---

### Likelihood Explanation

Medium. Bitcoin block Merkle trees are fully public. An attacker can compute any internal node hash offline from block data available via any Bitcoin full node or block explorer. No privileged access, leaked key, or social engineering is required. The only prerequisite is knowledge of the 64-byte forgery technique, which is publicly documented.

---

### Recommendation

Remove `verify_transaction_inclusion` from the public ABI entirely, or replace its body with an unconditional `env::panic_str("use verify_transaction_inclusion_v2")`. A compile-time `#[deprecated]` attribute is not a runtime guard and provides no security boundary against external NEAR callers.

---

### Proof of Concept

1. Identify any Bitcoin block `B` whose Merkle tree has at least two transactions.
2. Compute `node = SHA256d(left_child || right_child)` for any internal Merkle node in `B`. This 32-byte value is a valid `H256`.
3. Construct a `merkle_proof` path from `node` upward to `merkle_root` (all siblings are public).
4. Call `verify_transaction_inclusion` on the NEAR contract with:
   - `tx_id = node`
   - `tx_block_blockhash` = hash of block `B` (which is on the mainchain)
   - `tx_index` = the index that places `node` correctly in the path
   - `merkle_proof` = the sibling path computed above
   - `confirmations = 1`
5. The function computes `compute_root_from_merkle_proof(node, index, proof)` and compares it to `header.block_header.merkle_root`. [5](#0-4) 

Because `node` is a genuine internal node, the Merkle root reconstructs correctly and the function returns `true` — confirming a transaction that was never broadcast or mined.

    pub fn verify_transaction_inclusion_v2(&self, #[serializer(borsh)] args: ProofArgsV2) -> bool {
        require!(
            args.merkle_proof.len() == args.coinbase_merkle_proof.len(),
            "Coinbase merkle proof and transaction merkle proof should have the same length"
        );

        let header = self
            .headers_pool
            .get(&args.tx_block_blockhash)
            .unwrap_or_else(|| env::panic_str("cannot find requested transaction block"));

        require!(
            merkle_tools::compute_root_from_merkle_proof(
                args.coinbase_tx_id.clone(),
                0usize,
                &args.coinbase_merkle_proof,
            ) == header.block_header.merkle_root,
            "Incorrect coinbase merkle proof"
        );

        #[allow(deprecated)]
        self.verify_transaction_inclusion(args.into())
    }
