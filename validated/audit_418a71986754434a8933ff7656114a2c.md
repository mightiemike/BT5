### Title
Truncated Merkle Proof Allows Internal-Node Forgery in `verify_transaction_inclusion` — (`contract/src/lib.rs`)

### Summary
`verify_transaction_inclusion` accepts a Merkle proof of any non-zero length and passes it directly to `compute_root_from_merkle_proof`, which performs no depth validation. An attacker can supply a 1-element proof where `tx_id` is a known internal node, causing the function to return `true` for a transaction that does not exist.

### Finding Description
For a 4-transaction block, the Merkle tree is:

```
root = hash(L, R)
  L = hash(tx0, tx1)
  R = hash(tx2, tx3)
```

`L` and `R` are computable from public blockchain data. The attacker calls:

```
verify_transaction_inclusion({
    tx_id:           L,          // internal node, not a real tx
    tx_index:        0,
    merkle_proof:    [R],        // single element — truncated proof
    tx_block_blockhash: <valid block hash>,
    confirmations:   0,
})
```

`compute_root_from_merkle_proof(L, 0, [R])` executes one iteration:
- `current_position % 2 == 0` → `current_hash = hash(L, R) = root` ✓

The computed root equals the stored `merkle_root`, so the function returns `true`. [3](#0-2) 

No cryptographic attack is required. `L` and `R` are deterministically computable from the block's public transaction list.

### Impact Explanation
Any on-chain or off-chain consumer that calls `verify_transaction_inclusion` and trusts its `true` return value to confirm a transaction exists in a block can be deceived into accepting a fabricated inclusion claim. This enables false SPV proofs for non-existent transactions.

### Likelihood Explanation
The function is publicly callable by any NEAR account — no `#[private]`, no relayer role, no access key restriction. [4](#0-3) 

The only prerequisite is knowing the internal node values of the target block's Merkle tree, which are trivially computable from the block's transaction list (public data). The `#[pause]` gate exists but is not enforced by default.

The codebase itself acknowledges this class of attack in `contract/CLAUDE.md`: [5](#0-4) 

The function is `#[deprecated]` since 0.5.0, but it remains callable and is not removed. [6](#0-5) 

### Recommendation
- Remove `verify_transaction_inclusion` entirely, or enforce that it always panics (not just emits a deprecation warning).
- In `verify_transaction_inclusion_v2`, add a proof-length equality check: `require!(args.merkle_proof.len() == args.coinbase_merkle_proof.len())` already exists, but neither path validates that the proof length matches `ceil(log2(tx_count))`. Since the block header contains no transaction count, the coinbase-proof length-equality check in v2 is the practical mitigation — it forces the attacker's proof to be the same depth as a valid coinbase proof, which eliminates the truncation attack. [7](#0-6) 

### Proof of Concept
```rust
// 4-tx block: tx0, tx1, tx2, tx3
let tx0 = H256([0x01; 32]);
let tx1 = H256([0x02; 32]);
let tx2 = H256([0x03; 32]);
let tx3 = H256([0x04; 32]);

let L = compute_hash(&tx0, &tx1);   // internal node
let R = compute_hash(&tx2, &tx3);   // internal node
let root = compute_hash(&L, &R);    // = stored merkle_root

// Truncated 1-element proof using internal node as tx_id
let result = compute_root_from_merkle_proof(
    L,       // tx_id = internal node, not a real transaction
    0,       // tx_index = 0 (even → hash(L, R))
    &vec![R] // single sibling = right subtree root
);

assert_eq!(result, root); // passes — returns true for nonexistent tx
```

This is directly testable in `merkle-tools/src/lib.rs` with no contract deployment required. [8](#0-7)

pub fn compute_root_from_merkle_proof(
    transaction_hash: H256,
    transaction_position: usize,
    merkle_proof: &Vec<H256>,
) -> H256 {
    let mut current_hash = transaction_hash;
    let mut current_position = transaction_position;

    for proof_hash in merkle_proof {
        if current_position % 2 == 0 {
            current_hash = compute_hash(&current_hash, proof_hash);
        } else {
            current_hash = compute_hash(proof_hash, &current_hash);
        }
        current_position /= 2;
    }

    current_hash
