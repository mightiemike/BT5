Audit Report

## Title
SwapAllowlistExtension Checks Router Address Instead of Originating User, Enabling Complete Allowlist Bypass via MetricOmmSimpleRouter - (File: metric-periphery/contracts/extensions/SwapAllowlistExtension.sol)

## Summary
`SwapAllowlistExtension.beforeSwap` gates swaps by checking the `sender` argument, which `MetricOmmPool.swap` populates with `msg.sender` — the direct caller of `pool.swap()`. When swaps are routed through `MetricOmmSimpleRouter`, the router's address is presented as `sender` instead of the originating EOA. A pool admin who allowlists the router to support router-mediated swaps inadvertently grants every user of the router access to the pool, completely defeating the allowlist.

## Finding Description
`MetricOmmPool.swap` passes `msg.sender` as the `sender` argument to `_beforeSwap` (line 231). `ExtensionCalling._beforeSwap` forwards this value unchanged to every configured extension (lines 149–177). `SwapAllowlistExtension.beforeSwap` then evaluates `allowedSwapper[msg.sender][sender]` where `msg.sender` is the pool address and `sender` is whoever called `pool.swap()` (line 37).

When a user calls `MetricOmmSimpleRouter.exactInputSingle`, the router calls `pool.swap()` directly (lines 72–80 of `MetricOmmSimpleRouter.sol`). The pool's `msg.sender` is therefore the router address, so the extension receives `sender = router_address`. The originating EOA is completely invisible to the guard.

A pool admin who wants to support router-mediated swaps must call `setAllowedToSwap(pool, router, true)`, setting `allowedSwapper[pool][router] = true`. Once set, every user who calls through the router passes the check regardless of whether their own address is on the allowlist. The same bypass applies to `exactInput` (every hop), `exactOutputSingle`, and `exactOutput`.

## Impact Explanation
Any pool configured with `SwapAllowlistExtension` to restrict trading to a curated set of addresses (e.g., KYC-verified counterparties, institutional participants, or protocol-controlled addresses) is fully open to all users the moment the router is allowlisted. Non-allowlisted users can execute swaps at oracle-derived prices against a pool designed to be private, constituting broken core pool functionality and potential direct loss of LP principal. This meets the "broken core pool functionality causing loss of funds" and "admin-boundary break bypassed by an unprivileged path" impact criteria.

## Likelihood Explanation
The router is the primary user-facing entry point. Any pool operator who deploys `SwapAllowlistExtension` and also wants users to access the pool through the standard router faces an unavoidable dilemma: allowlist the router (bypass for all) or do not (allowlisted users cannot use the router). The bypass requires only a standard router call with no special privileges, making it trivially reachable by any unprivileged user.

## Recommendation
The extension must receive and check the originating user's address, not the intermediate caller:
1. **Pass the originating payer through the router.** The router already tracks the originating payer in transient storage via `_getPayer()`. Add an `originSender` field to `extensionData` or to the swap call so the pool can forward it to extensions alongside `msg.sender`.
2. **Redesign `SwapAllowlistExtension`** to accept an authenticated origin address from a trusted router (similar to how cross-chain bridges add `originSenderAddress` to message payloads), or check `tx.origin` only for EOA-direct flows with explicit documentation of the limitation.
3. **Document the limitation clearly**: if the router is allowlisted, the allowlist is effectively disabled. Pool admins must allowlist individual EOAs and require direct `pool.swap()` calls if true per-user gating is needed.

## Proof of Concept
```
Setup:
  - Deploy pool with SwapAllowlistExtension
  - Pool admin allowlists router: setAllowedToSwap(pool, router, true)
  - Pool admin does NOT allowlist attacker EOA

Attack:
  - attacker (non-allowlisted EOA) calls:
      MetricOmmSimpleRouter.exactInputSingle({pool: pool, ...})
  - Router calls pool.swap() → msg.sender = router
  - Pool calls _beforeSwap(sender=router, ...)
  - SwapAllowlistExtension checks allowedSwapper[pool][router] == true → PASSES
  - Swap executes; attacker receives tokens from a pool they were never authorized to trade against

Result:
  - allowedSwapper[pool][attacker] == false (never set)
  - allowedSwapper[pool][router]   == true  (set by admin to support router)
  - Attacker bypasses the allowlist entirely via the router
```
