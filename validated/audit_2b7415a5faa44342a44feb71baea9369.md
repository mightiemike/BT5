### Title
Gateway Stateless Validator Accepts ProofVersion::V0 Transactions That Blockifier Rejects Under V0_14_3 Versioned Constants — (`crates/apollo_gateway/src/stateless_transaction_validator.rs`)

### Summary

The gateway's `StatelessTransactionValidator` does not validate the `proof_version` field of `proof_facts` against `allowed_proof_versions` from the versioned constants. The blockifier's `validate_proof_facts` does perform this check. Because `allowed_proof_versions` changed from `["0x50524f4f4630"]` (PROOF0) in V0_14_2 to `["0x50524f4f4631"]` (PROOF1) in V0_14_3, a transaction carrying PROOF0 will pass stateless gateway admission but be rejected by the blockifier when building a V0_14_3 block.

### Finding Description

**Gateway path — no proof version check:**

`validate_client_side_proving_allowed` returns `Ok(())` immediately when `allow_client_side_proving=true`: [1](#0-0) 

`validate_proof_facts_and_proof_consistency` only checks structural consistency (both `proof_facts` and `proof` present or both absent); it never inspects the proof version value: [2](#0-1) 

**Blockifier path — proof version is checked against versioned constants:** [3](#0-2) 

**V0_14_3 only allows PROOF1:** [4](#0-3) 

**V0_14_2 only allowed PROOF0:** [5](#0-4) 

The `allow_client_side_proving` default is `true`: [6](#0-5) 

### Impact Explanation

An unprivileged attacker can submit an `InvokeV3` transaction with `proof_facts` carrying `ProofVersion::V0` (`0x50524f4f4630`) to a gateway with `allow_client_side_proving=true`. The stateless validator accepts it (no proof version check). The transaction enters the mempool. When the sequencer attempts to include it in a V0_14_3 block, the blockifier rejects it with `InvalidProofFacts("Proof version ... is not allowed under this protocol version.")`. The transaction is permanently unsequenceable but occupies mempool slots.

Impact category: **High — Mempool/gateway admission accepts invalid transactions before sequencing.**

### Likelihood Explanation

- `allow_client_side_proving=true` is the **default** config value.
- The version boundary (V0_14_2 → V0_14_3) is a concrete, deployed protocol upgrade.
- No privilege is required; any public RPC caller can submit the transaction.
- The attacker only needs to know the current Starknet version is ≥ V0_14_3 and submit a PROOF0 transaction.

### Recommendation

Add proof version validation to the stateless validator. The validator should either:
1. Reject any `proof_facts` whose version is not in a statically-known set of currently-valid proof versions (e.g., only PROOF1 for the current protocol), or
2. Expose the current `VersionedConstants` (or at minimum `allowed_proof_versions`) to the stateless validator so it can mirror the blockifier's check.

The check should be added inside `validate_proof_facts_and_proof_consistency` or as a new dedicated method called from the `validate` dispatch at lines 45–48. [7](#0-6) 

### Proof of Concept

```rust
// Pseudocode — build a PROOF0 InvokeV3 tx
let proof_facts = ProofFacts(vec![/* SNOS header with proof_version = 0x50524f4f4630 */]);
let tx = RpcInvokeTransaction::V3(RpcInvokeTransactionV3 {
    proof_facts,
    proof: vec![Felt::ONE], // non-empty to pass consistency check
    ..default_invoke_v3()
});

// Gateway stateless validator with allow_client_side_proving=true
let validator = StatelessTransactionValidator {
    config: StatelessTransactionValidatorConfig { allow_client_side_proving: true, ..Default::default() },
};
assert!(validator.validate(&RpcTransaction::Invoke(tx.clone())).is_ok()); // PASSES

// Blockifier with V0_14_3 BlockContext
let block_context = BlockContext::new_for_testing_with_starknet_version(StarknetVersion::V0_14_3);
let account_tx = AccountTransaction::from(executable_invoke_v3(tx));
assert!(account_tx.validate_proof_facts(&block_context, &mut state).is_err()); // REJECTS
// Error: "Proof version 0x50524f4f4630 is not allowed under this protocol version."
```

    fn validate_proof_facts_and_proof_consistency(
        &self,
        tx: &RpcInvokeTransaction,
    ) -> StatelessTransactionValidatorResult<()> {
        let RpcInvokeTransaction::V3(tx) = tx;
        let has_proof_facts = !tx.proof_facts.is_empty();
        let has_proof = !tx.proof.is_empty();
        if has_proof_facts != has_proof {
            return Err(StatelessTransactionValidatorError::ProofFactsAndProofConsistency {
                has_proof_facts,
                has_proof,
            });
        }
        Ok(())
    }
