### Title
Withdrawal Fee (`withdrawFeeX18`) Bypassed via Slow Mode Path — (`File: core/contracts/EndpointTx.sol`)

---

### Summary

The protocol charges a per-product `withdrawFeeX18` fee when a user withdraws collateral through the sequencer path (`processTransactionImpl`). However, the slow mode execution path (`processSlowModeTransactionImpl`) handles the identical `WithdrawCollateral` transaction type and calls `clearinghouse.withdrawCollateral` directly — without ever calling `chargeFee`. Any user can submit a `WithdrawCollateral` transaction via `submitSlowModeTransaction`, wait out the 3-day delay, and receive their full collateral without paying the product-specific withdrawal fee.

---

### Finding Description

`processTransactionImpl` (the sequencer-driven path) correctly charges the withdrawal fee before executing the withdrawal:

```solidity
// EndpointTx.sol lines 425-436
chargeFee(
    signedTx.tx.sender,
    spotEngine.getConfig(signedTx.tx.productId).withdrawFeeX18,
    signedTx.tx.productId
);
clearinghouse.withdrawCollateral(
    signedTx.tx.sender,
    signedTx.tx.productId,
    signedTx.tx.amount,
    address(0),
    nSubmissions
);
``` [1](#0-0) 

The slow mode path in `processSlowModeTransactionImpl` handles the same `WithdrawCollateral` transaction type but omits the `chargeFee` call entirely:

```solidity
// EndpointTx.sol lines 217-229
} else if (txType == IEndpoint.TransactionType.WithdrawCollateral) {
    IEndpoint.WithdrawCollateral memory txn = abi.decode(
        transaction[1:],
        (IEndpoint.WithdrawCollateral)
    );
    validateSender(txn.sender, sender);
    clearinghouse.withdrawCollateral(
        txn.sender,
        txn.productId,
        txn.amount,
        address(0),
        nSubmissions
    );
``` [2](#0-1) 

The entry point is fully open to any user. `submitSlowModeTransaction` on `Endpoint.sol` is `external` with no access control beyond a sanctions check and a flat `SLOW_MODE_FEE` charge: [3](#0-2) 

After the `SLOW_MODE_TX_DELAY` (hardcoded to 3 days), the transaction is executed via `executeSlowModeTransaction()` (also callable by anyone), which routes through `processSlowModeTransactionImpl` — skipping `withdrawFeeX18` entirely. [4](#0-3) 

The flat `SLOW_MODE_FEE` charged at submission time is a separate, unrelated fee (a fixed quote-token amount taken via `chargeSlowModeFee`). It does not substitute for `withdrawFeeX18`, which is a per-product, proportional fee configured in `SpotEngine`. [5](#0-4) 

---

### Impact Explanation

The protocol loses all `withdrawFeeX18` revenue on any withdrawal routed through slow mode. The corrupted state delta is `sequencerFee[productId]` — it is never incremented for slow mode withdrawals, meaning the sequencer fee accounting is permanently understated for every such withdrawal. For products with a non-zero `withdrawFeeX18`, the full fee amount is silently forfeited by the protocol on each slow mode withdrawal.

---

### Likelihood Explanation

High. The attack requires no special privilege — any subaccount holder can call `submitSlowModeTransaction` with a `WithdrawCollateral` payload. The only cost is the flat `SLOW_MODE_FEE` and a 3-day wait. For any withdrawal where `amount × withdrawFeeX18 > SLOW_MODE_FEE`, the slow mode path is strictly cheaper for the user. Rational users withdrawing large amounts will prefer this path.

---

### Recommendation

Add the `withdrawFeeX18` charge inside `processSlowModeTransactionImpl` for the `WithdrawCollateral` branch, mirroring the logic in `processTransactionImpl`:

```solidity
} else if (txType == IEndpoint.TransactionType.WithdrawCollateral) {
    IEndpoint.WithdrawCollateral memory txn = abi.decode(
        transaction[1:],
        (IEndpoint.WithdrawCollateral)
    );
    validateSender(txn.sender, sender);
+   chargeFee(
+       txn.sender,
+       spotEngine.getConfig(txn.productId).withdrawFeeX18,
+       txn.productId
+   );
    clearinghouse.withdrawCollateral(
        txn.sender,
        txn.productId,
        txn.amount,
        address(0),
        nSubmissions
    );
``` [2](#0-1) 

---

### Proof of Concept

1. User holds a subaccount with collateral in product `productId` where `withdrawFeeX18 > 0`.
2. User calls `Endpoint.submitSlowModeTransaction(encodedWithdrawCollateral)` — pays only the flat `SLOW_MODE_FEE`.
3. After 3 days, user (or anyone) calls `Endpoint.executeSlowModeTransaction()`.
4. `processSlowModeTransactionImpl` decodes the `WithdrawCollateral` transaction and calls `clearinghouse.withdrawCollateral` directly — no `chargeFee` is invoked.
5. User receives full `amount` of collateral. `sequencerFee[productId]` is never incremented. The `withdrawFeeX18` fee is entirely avoided. [6](#0-5) [2](#0-1)
