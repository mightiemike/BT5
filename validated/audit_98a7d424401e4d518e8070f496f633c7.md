### Title
Protobuf `ValidResourceBounds` Deserialization Silently Coerces `AllResources` to `L1Gas`, Producing a Different Transaction Hash Preimage - (File: crates/apollo_protobuf/src/converters/transaction.rs)

### Summary

The protobuf deserializer for `ValidResourceBounds` in the P2P/sync path silently maps an `AllResources` transaction (with zero `l2_gas` and zero `l1_data_gas`) to the `L1Gas` variant. Because `get_tip_resource_bounds_hash` hashes a different number of elements for each variant (3 vs 4), the transaction hash computed on the receiving node diverges from the hash computed on the originating node. A user-submitted transaction that was signed over the 4-element hash is stored and propagated with a 3-element hash, breaking signature binding and causing consensus/sync hash-commitment mismatches.

### Finding Description

`TryFrom<protobuf::ResourceBounds> for ValidResourceBounds` applies a value-based heuristic to decide which enum variant to produce:

```rust
// crates/apollo_protobuf/src/converters/transaction.rs
let l1_data_gas = value.l1_data_gas.unwrap_or_default();   // silently defaults to zero
// ...
Ok(if l1_data_gas.is_zero() && l2_gas.is_zero() {
    ValidResourceBounds::L1Gas(l1_gas)          // ← variant changed
} else {
    ValidResourceBounds::AllResources(AllResourceBounds { l1_gas, l2_gas, l1_data_gas })
})
``` [1](#0-0) 

The originating path (`From<RpcInvokeTransactionV3> for InvokeTransactionV3`) always wraps the user-supplied `AllResourceBounds` in `ValidResourceBounds::AllResources`:

`get_tip_resource_bounds_hash` produces structurally different hash preimages for the two variants:

```rust
// crates/starknet_api/src/transaction_hash.rs
let mut resource_felts = vec![
    get_concat_resource(&l1_resource_bounds, L1_GAS)?,
    get_concat_resource(&l2_resource_bounds, L2_GAS)?,
];
resource_felts.extend(match resource_bounds {
    ValidResourceBounds::L1Gas(_) => vec![],                          // 2 elements → 3 total with tip
    ValidResourceBounds::AllResources(all_resources) => {
        vec![get_concat_resource(&all_resources.l1_data_gas, L1_DATA_GAS)?]  // 3 elements → 4 total
    }
});
Ok(HashChain::new().chain(&tip.0.into()).chain_iter(resource_felts.iter()).get_poseidon_hash())
``` [3](#0-2) 

Because Poseidon is a sponge function, `Poseidon(tip, L1, L2_zero)` ≠ `Poseidon(tip, L1, L2_zero, L1DATA_zero)`. The two variants produce provably distinct hashes even when the numeric field values are identical.

### Impact Explanation

**Transaction hash binding is broken across the P2P boundary.** A transaction submitted via JSON-RPC with `AllResourceBounds { l1_gas: X, l2_gas: 0, l1_data_gas: 0 }` is hashed with 4 elements (H1) at the gateway. The user signs H1. When the block containing this transaction is synced to a peer via protobuf, the peer deserializes the same resource-bounds bytes as `L1Gas` and computes H2 (3 elements). H1 ≠ H2.

Consequences:
- **Signature verification failure**: the account contract's `__validate__` receives H2 but the signature covers H1 — the transaction is treated as having an invalid signature.
- **Block/transaction commitment mismatch**: the transaction commitment in the block header was computed over H1; the receiving node computes H2, causing the Merkle commitment check to fail and the block to be rejected.
- **Consensus divergence**: nodes that received the transaction directly (H1) and nodes that received it via P2P sync (H2) disagree on the canonical transaction hash, breaking consensus agreement.

This matches the "High — Transaction conversion or signature/hash logic binds the wrong signer, hash, type, or executable payload" impact.

### Likelihood Explanation

Any user can submit a V3 invoke transaction with `l2_gas.max_amount = 0` and `l1_data_gas.max_amount = 0` (only `l1_gas` non-zero). The gateway accepts it because at least one resource bound is non-zero. The coercion fires deterministically on every P2P peer that deserializes the transaction. No special privileges are required; the trigger is a normal RPC call.

### Recommendation

Remove the value-based variant heuristic from the protobuf deserializer. The variant (`L1Gas` vs `AllResources`) is a semantic property of the transaction that must be preserved across serialization boundaries, not inferred from whether numeric fields happen to be zero.

**Option A** – Encode the variant explicitly in the protobuf schema (a `oneof` or a boolean flag).

**Option B** – Always deserialize to `AllResources` when all three fields are present in the wire message (matching the behavior of the JSON path when `L1DataGas` is present):

```rust
// Always AllResources when l1_data_gas is present on the wire
let l1_data_gas = value.l1_data_gas.unwrap_or_default();
Ok(ValidResourceBounds::AllResources(AllResourceBounds { l1_gas, l2_gas, l1_data_gas }))
```

The `L1Gas` variant should only be produced when the wire message explicitly omits `l1_data_gas` **and** `l2_gas` is zero, mirroring the JSON deserialization rule in `TryFrom<DeprecatedResourceBoundsMapping>`. [4](#0-3) 

### Proof of Concept

1. Submit a V3 invoke transaction via JSON-RPC with:
   ```json
   "resource_bounds": {
     "l1_gas":      { "max_amount": "0x100", "max_price_per_unit": "0x1" },
     "l2_gas":      { "max_amount": "0x0",   "max_price_per_unit": "0x0" },
     "l1_data_gas": { "max_amount": "0x0",   "max_price_per_unit": "0x0" }
   }
   ```
2. The gateway deserializes via `TryFrom<DeprecatedResourceBoundsMapping>`: `L1DataGas` key is present → `AllResources`. Hash H1 = `Poseidon(tip, L1_packed, L2_zero_packed, L1DATA_zero_packed)`.
3. User signs H1 and the transaction is accepted into the mempool.
4. The block is produced and propagated to a peer via protobuf.
5. The peer deserializes via `TryFrom<protobuf::ResourceBounds> for ValidResourceBounds`: `l1_data_gas.is_zero() && l2_gas.is_zero()` → `L1Gas`. Hash H2 = `Poseidon(tip, L1_packed, L2_zero_packed)`.
6. H1 ≠ H2. The peer's transaction-commitment check fails; the block is rejected. Any node that accepted the transaction directly disagrees with any node that received it via P2P sync. [5](#0-4) [6](#0-5)

impl TryFrom<protobuf::ResourceBounds> for ValidResourceBounds {
    type Error = ProtobufConversionError;
    fn try_from(value: protobuf::ResourceBounds) -> Result<Self, Self::Error> {
        let Some(l1_gas) = value.l1_gas else {
            return Err(missing("ResourceBounds::l1_gas"));
        };
        let Some(l2_gas) = value.l2_gas else {
            return Err(missing("ResourceBounds::l2_gas"));
        };
        // TODO(Shahak): Assert data gas is not none once we remove support for 0.13.2.
        let l1_data_gas = value.l1_data_gas.unwrap_or_default();
        let l1_gas: ResourceBounds = l1_gas.try_into()?;
        let l2_gas: ResourceBounds = l2_gas.try_into()?;
        let l1_data_gas: ResourceBounds = l1_data_gas.try_into()?;
        Ok(if l1_data_gas.is_zero() && l2_gas.is_zero() {
            ValidResourceBounds::L1Gas(l1_gas)
        } else {
            ValidResourceBounds::AllResources(AllResourceBounds { l1_gas, l2_gas, l1_data_gas })
        })
    }
}

pub fn get_tip_resource_bounds_hash(
    resource_bounds: &ValidResourceBounds,
    tip: &Tip,
) -> Result<Felt, StarknetApiError> {
    let l1_resource_bounds = resource_bounds.get_l1_bounds();
    let l2_resource_bounds = resource_bounds.get_l2_bounds();

    // L1 and L2 gas bounds always exist.
    // Old V3 txs always have L2 gas bounds of zero, but they exist.
    let mut resource_felts = vec![
        get_concat_resource(&l1_resource_bounds, L1_GAS)?,
        get_concat_resource(&l2_resource_bounds, L2_GAS)?,
    ];

    // For new V3 txs, need to also hash the data gas bounds.
    resource_felts.extend(match resource_bounds {
        ValidResourceBounds::L1Gas(_) => vec![],
        ValidResourceBounds::AllResources(all_resources) => {
            vec![get_concat_resource(&all_resources.l1_data_gas, L1_DATA_GAS)?]
        }
    });

    Ok(HashChain::new().chain(&tip.0.into()).chain_iter(resource_felts.iter()).get_poseidon_hash())
}

impl TryFrom<DeprecatedResourceBoundsMapping> for ValidResourceBounds {
    type Error = StarknetApiError;
    fn try_from(
        resource_bounds_mapping: DeprecatedResourceBoundsMapping,
    ) -> Result<Self, Self::Error> {
        if let (Some(l1_bounds), Some(l2_bounds)) = (
            resource_bounds_mapping.0.get(&Resource::L1Gas),
            resource_bounds_mapping.0.get(&Resource::L2Gas),
        ) {
            match resource_bounds_mapping.0.get(&Resource::L1DataGas) {
                Some(data_bounds) => Ok(Self::AllResources(AllResourceBounds {

                             {resource_bounds_mapping:?}",
                        )))
                    }
                }
            }
