### Title
No Upper Bound on `Tier.ExitDuration` Allows Governance to Lock All Tier Position Funds Indefinitely - (`x/tieredrewards/types/tier.go`)

---

### Summary

`Tier.Validate()` enforces only a lower bound (`ExitDuration > 0`) on the exit commitment period but imposes no upper bound. A governance proposal passing `MsgAddTier` or `MsgUpdateTier` with `ExitDuration = math.MaxInt64` nanoseconds (~292 years) causes every subsequent `MsgTriggerExitFromTier` call to set `ExitUnlockAt` to a timestamp ~292 years in the future. All exit paths gate on `CompletedExitLockDuration(blockTime)`, so the delegated tokens held by each position's per-position delegator address in `x/staking` become permanently unreachable.

---

### Finding Description

`Tier.Validate()` in `x/tieredrewards/types/tier.go` checks:

```go
if t.ExitDuration <= 0 {
    return fmt.Errorf("exit duration must be positive")
}
```

There is no corresponding upper-bound check. The field is typed `time.Duration` (Go `int64` nanoseconds), so any value up to `math.MaxInt64` (~292 years) passes validation. [1](#0-0) 

`SetTier` — the single write path used by both `MsgAddTier` and `MsgUpdateTier` — calls `tier.Validate()` and then stores the tier unconditionally:

```go
func (k Keeper) SetTier(ctx context.Context, tier types.Tier) error {
    if err := tier.Validate(); err != nil {
        return err
    }
    ...
}
``` [2](#0-1) 

When a user calls `MsgTriggerExitFromTier`, the handler reads the live tier and stamps `ExitUnlockAt` onto the position:

```go
pos.TriggerExit(sdkCtx.BlockTime(), tier.ExitDuration)
``` [3](#0-2) 

`TriggerExit` computes:

```go
func (p *Position) TriggerExit(blockTime time.Time, duration time.Duration) {
    p.ExitTriggeredAt = blockTime
    p.ExitUnlockAt = blockTime.Add(duration)
}
``` [4](#0-3) 

Every exit path then gates on `CompletedExitLockDuration`:

```go
func (p Position) CompletedExitLockDuration(blockTime time.Time) bool {
    return !p.ExitUnlockAt.IsZero() && !blockTime.Before(p.ExitUnlockAt)
}
``` [5](#0-4) 

This check is enforced in `validateUndelegatePosition`, `validateExitTierWithDelegation`, and `validateWithdrawFromTier` — covering all three exit paths. [6](#0-5) [7](#0-6) [8](#0-7) 

If governance additionally marks the tier `CloseOnly = true`, `MsgClearPosition` (the only way to cancel a triggered exit) is also blocked: [9](#0-8) 

---

### Impact Explanation

The exact corrupted value is `Position.ExitUnlockAt` — stamped at trigger time as `blockTime + tier.ExitDuration`. With `ExitDuration = math.MaxInt64`, this timestamp is ~292 years in the future. The delegated tokens are held by the position's per-position delegator address inside `x/staking`. Because `CompletedExitLockDuration` will never return `true` within any realistic chain lifetime, `MsgTierUndelegate`, `MsgExitTierWithDelegation`, and `MsgWithdrawFromTier` all revert permanently for every position that triggered exit after the malicious update. The principal (staked `basecro`) is locked in the module's staking delegation and cannot be recovered.

---

### Likelihood Explanation

The trigger is a governance proposal (`MsgUpdateTier` or `MsgAddTier`) that passes with a majority of bonded voting power. This is the same threat model as the original sNOTE finding (compromised or malicious privileged role). On-chain governance is a realistic attack surface: a coordinated validator cartel, a governance attack via flash-loan-style vote accumulation, or a social-engineering campaign targeting large delegators could all pass such a proposal. The `BonusApy` field already has an explicit upper-bound guard (`> 1.0` rejected) showing the developers are aware of the pattern — the omission for `ExitDuration` is an oversight, not a deliberate design choice. [10](#0-9) 

---

### Recommendation

Add an upper-bound check in `Tier.Validate()`, for example:

```go
const MaxExitDuration = 5 * 365 * 24 * time.Hour // 5 years

if t.ExitDuration > MaxExitDuration {
    return fmt.Errorf("exit duration must not exceed %s: got %s", MaxExitDuration, t.ExitDuration)
}
```

A sensible ceiling (e.g. 5 years, matching the longest intended lock tier) ensures governance retains the ability to set meaningful exit periods while guaranteeing users' funds cannot be locked beyond a bounded horizon. [11](#0-10) 

---

### Proof of Concept

1. Governance submits and passes `MsgUpdateTier` with `ExitDuration = 9223372036854775807` (nanoseconds, `math.MaxInt64`).
2. `SetTier` calls `tier.Validate()` — passes because `ExitDuration > 0`.
3. User calls `MsgTriggerExitFromTier` on their position. Handler reads the tier, calls `pos.TriggerExit(blockTime, tier.ExitDuration)`, setting `ExitUnlockAt = blockTime + ~292 years`.
4. User attempts `MsgTierUndelegate` → `validateUndelegatePosition` calls `pos.CompletedExitLockDuration(blockTime)` → returns `false` → `ErrExitLockDurationNotReached`.
5. User attempts `MsgExitTierWithDelegation` → same gate → same error.
6. If governance also set `CloseOnly = true`, `MsgClearPosition` returns `ErrTierIsCloseOnly`, removing the only escape hatch.
7. The position's delegated `basecro` tokens remain locked in the per-position staking delegator address indefinitely. [12](#0-11) [2](#0-1) [4](#0-3)

func (t Tier) Validate() error {
	if t.Id == 0 {
		return ErrInvalidTierID
	}

	if t.ExitDuration <= 0 {
		return fmt.Errorf("exit duration must be positive")
	}

	if t.BonusApy.IsNil() {
		return fmt.Errorf("bonus apy cannot be nil")
	}

	if t.BonusApy.IsNegative() {
		return fmt.Errorf("bonus apy cannot be negative: %s", t.BonusApy)
	}

	// Cap BonusApy at 100% to prevent governance from draining the rewards pool.
	if t.BonusApy.GT(math.LegacyOneDec()) {
		return fmt.Errorf("bonus apy must not exceed 1.0 (100%%): got %s", t.BonusApy)
	}

	if t.MinLockAmount.IsNil() {
		return fmt.Errorf("min lock amount cannot be nil")
	}

	if t.MinLockAmount.IsNegative() {
		return fmt.Errorf("min lock amount cannot be negative: %s", t.MinLockAmount)
	}

	return nil
}

func (ms msgServer) UpdateTier(ctx context.Context, msg *types.MsgUpdateTier) (*types.MsgUpdateTierResponse, error) {
	if err := ms.requireAuthority(msg.Authority); err != nil {
		return nil, err
	}

	oldTier, err := ms.getTier(ctx, msg.Tier.Id)
	if err != nil {
		return nil, err
	}

	if !oldTier.BonusApy.Equal(msg.Tier.BonusApy) {
		if err := ms.claimRewardsAndUpdateTierPositions(ctx, msg.Tier.Id); err != nil {
			return nil, err
		}
	}

	if err := ms.SetTier(ctx, msg.Tier); err != nil {
		return nil, err
	}

	if err := ms.emitTierChangedEvent(ctx, types.TierChangeAction_TIER_CHANGE_ACTION_UPDATE, msg.Tier); err != nil {
		return nil, err
	}

	return &types.MsgUpdateTierResponse{}, nil
}
