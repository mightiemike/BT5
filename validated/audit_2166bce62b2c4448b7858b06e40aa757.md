### Title
Division-by-Zero in `PerpEngine.socializeSubaccount` Blocks Liquidation Finalization When `openInterest` Is Zero — (File: `core/contracts/PerpEngine.sol`)

---

### Summary

`PerpEngine.socializeSubaccount` unconditionally divides by `state.openInterest` when socializing residual negative `vQuoteBalance`. If `openInterest` is zero — which occurs when the liquidatee is the sole remaining participant in a perp market — the division reverts, permanently blocking liquidation finalization for that subaccount.

---

### Finding Description

In `PerpEngine.socializeSubaccount`, after the insurance fund is applied, any remaining negative `vQuoteBalance` is socialized across all other open-interest holders via:

```solidity
// actually socialize if still not enough
if (balance.vQuoteBalance < 0) {
    // socialize across all other participants
    int128 fundingPerShare = -balance.vQuoteBalance.div(state.openInterest);
``` [1](#0-0) 

There is no guard on `state.openInterest == 0` before this division. `openInterest` reaches zero when no other subaccount holds an open position in the market — a realistic condition for newly listed or low-liquidity perp markets where the liquidatee was the sole participant.

The call path to this code is:

1. `Clearinghouse.liquidateSubaccount` → `delegatecall` into `ClearinghouseLiq.liquidateSubaccountImpl` [2](#0-1) 

2. `liquidateSubaccountImpl` calls `_finalizeSubaccount` when `txn.productId == type(uint32).max` [3](#0-2) 

3. `_finalizeSubaccount` first asserts all perp `amount` fields are zero, then calls `perpEngine.socializeSubaccount` [4](#0-3) [5](#0-4) 

The preconditions for the revert are:

- The liquidatee's perp `amount` is already 0 (required by `_finalizeSubaccount`'s own check), meaning their position was closed in prior liquidation steps. When the last position in a market is closed, `openInterest` drops to zero.
- The liquidatee still carries a negative `vQuoteBalance` (unrealized loss from the closed position that was never settled).
- The insurance fund is insufficient to fully cover that negative `vQuoteBalance`, so the inner `if (balance.vQuoteBalance < 0)` branch is entered.
- `state.openInterest == 0` → `div(0)` → revert.

---

### Impact Explanation

The revert occurs inside `_finalizeSubaccount`, which is the only mechanism to clear an insolvent subaccount's residual liabilities and close it out. A permanent revert here means:

- The insolvent subaccount can never be finalized.
- Its bad debt is never socialized or absorbed.
- The protocol accumulates unresolvable bad debt, degrading solvency for all depositors.

This is not a gas-only or observability issue — it directly prevents the protocol from recovering from insolvent positions, which is the core purpose of the liquidation system.

---

### Likelihood Explanation

The condition is reachable without any privileged access or exotic setup:

- Any perp market where the liquidatee is the only open-interest holder (e.g., a market that just launched, or one where all other traders have exited) satisfies `openInterest == 0` after the liquidatee's position is closed.
- A liquidator (unprivileged caller) submits the finalization transaction (`productId == type(uint32).max`) after prior liquidation steps have zeroed the position.
- The insurance fund being insufficient is a normal operating condition for large losses.

---

### Recommendation

Add a zero-check on `state.openInterest` before the division inside `socializeSubaccount`:

```diff
 if (balance.vQuoteBalance < 0) {
-    int128 fundingPerShare = -balance.vQuoteBalance.div(state.openInterest);
-    state.cumulativeFundingLongX18 += fundingPerShare;
-    state.cumulativeFundingShortX18 -= fundingPerShare;
-    balance.vQuoteBalance = 0;
+    if (state.openInterest > 0) {
+        int128 fundingPerShare = -balance.vQuoteBalance.div(state.openInterest);
+        state.cumulativeFundingLongX18 += fundingPerShare;
+        state.cumulativeFundingShortX18 -= fundingPerShare;
+    }
+    // If openInterest == 0, no participants remain to socialize to;
+    // the residual loss is absorbed (zeroed) without distribution.
+    balance.vQuoteBalance = 0;
 }
``` [6](#0-5) 

---

### Proof of Concept

1. A perp market is listed. Only one trader (the liquidatee) opens a large long position.
2. The oracle price drops sharply; the liquidatee's maintenance health goes negative.
3. A liquidator calls `liquidateSubaccountImpl` with the specific `productId` to reduce the position to zero. After this step, `openInterest == 0` for that market, but the liquidatee's `vQuoteBalance` is still negative (unrealized loss).
4. The insurance fund holds less than the absolute value of the remaining `vQuoteBalance`.
5. The liquidator calls `liquidateSubaccountImpl` with `productId == type(uint32).max` to finalize.
6. `_finalizeSubaccount` passes the `require(balance.amount == 0)` check (position is already zero).
7. `perpEngine.socializeSubaccount` is called; it enters the `if (balance.vQuoteBalance < 0)` branch, insurance is partially applied but insufficient, and the inner branch executes `-balance.vQuoteBalance.div(state.openInterest)` where `state.openInterest == 0`.
8. The transaction reverts with a division-by-zero panic.
9. The insolvent subaccount can never be finalized; its bad debt is permanently stuck.
