### Title
Wrong Weight Direction in Spread Health Calculation Inflates INITIAL Health for Long-Spot/Short-Perp Positions — (`core/contracts/libraries/RiskHelper.sol`, `core/contracts/Clearinghouse.sol`)

---

### Summary

`getCoreRisk` in `BaseEngine.sol` always populates `CoreRisk.longWeight` with the **long** weight by hardcoding `amount = 1` in the `_getWeightX18` call, regardless of the subaccount's actual position direction. When `Clearinghouse.getHealth` computes the spread health adjustment for a long-spot/short-perp position, it uses this always-long weight for both `existingWeight` and `spreadWeight`. For a short perp, the correct weight is `shortWeightInitialX18` (> 1), but the code uses `longWeightInitialX18` (< 1). This makes `spreadWeight - existingWeight` significantly larger than it should be, inflating the spread health adjustment and overstating INITIAL health.

---

### Finding Description

**Root cause — `getCoreRisk` always returns the long weight:** [1](#0-0) 

The call `RiskHelper._getWeightX18(risk, 1, healthType)` passes a hardcoded positive `1` as `amount`, so `_getWeightX18` always returns `longWeightInitialX18` or `longWeightMaintenanceX18`. The `CoreRisk` struct has only one weight field named `longWeight`: [2](#0-1) 

**Spread adjustment in `getHealth` uses the wrong weight for the short perp:** [3](#0-2) 

`existingWeight` is `(spotCoreRisk.longWeight + perpCoreRisk.longWeight) / 2`. For a short perp, the individual health contribution computed by `_calculateProductHealth` uses `perpShortWeightInitialX18` (> 1), but `existingWeight` uses `perpLongWeightInitialX18` (< 1).

**`_getSpreadWeightX18` also uses the wrong weight:** [4](#0-3) 

When `spotCoreRisk.amount > 0` (long spot / short perp), `spreadWeight = ONE - (ONE - perpCoreRisk.longWeight) / 5` uses the long weight of the perp instead of its short weight.

**Concrete numerical example** (initial health, typical volatile asset):

| Parameter | Value |
|---|---|
| `spotLongWeightInitial` | 0.90 |
| `perpLongWeightInitial` | 0.90 |
| `perpShortWeightInitial` | 1.10 |

*Buggy path (uses `perpLongWeight`):*
- `spreadWeight = 1 − (1 − 0.90)/5 = 0.98`
- `existingWeight = (0.90 + 0.90)/2 = 0.90`
- `spreadWeight − existingWeight = +0.08` → **large positive health boost**

*Correct path (uses `perpShortWeight`):*
- `spreadWeight = 1 − (1 − 1.10)/5 = 1.02` → capped at `0.99`
- `existingWeight = (0.90 + 1.10)/2 = 1.00`
- `spreadWeight − existingWeight = −0.01` → **small health penalty**

The buggy path produces a health adjustment of `+0.08 × basisAmount × (spotPrice + perpPrice)` instead of `−0.01 × basisAmount × (spotPrice + perpPrice)`. For a $100k notional spread, this is a ~$9k overstatement of INITIAL health.

The short-spot/long-perp direction is not exploitable in the same way: `basisAmount` is negative in that branch, so the wrong weight produces a health *penalty* (conservative), not a boost.

---

### Impact Explanation

A trader holding a long-spot/short-perp spread position has their INITIAL health overstated by approximately `0.09 × basisAmount × (spotPrice + perpPrice)`. This allows them to:

1. **Withdraw collateral** beyond what the true risk parameters permit (via `withdrawCollateral` → `getHealth` check at line 419).
2. **Open additional positions** that would otherwise be blocked by the INITIAL health check (via `_isAboveInitial` → `getHealth`).
3. **Transfer quote** to another subaccount beyond permitted limits (via `transferQuote` → `_isAboveInitial`).

The overstated health scales linearly with position size, so larger spread positions produce proportionally larger exploitable health inflation. [5](#0-4) 

---

### Likelihood Explanation

- The long-spot/short-perp spread (basis trade) is a standard, commonly used strategy on perpetual DEXs.
- No special privileges are required; any trader can open a spread position.
- The exploit is triggered simply by holding the position and calling any health-checked action (withdraw, transfer, open position).
- The `spreads` bitmask in `Clearinghouse` must include the relevant (spotId, perpId) pair, which is a normal protocol configuration.

---

### Recommendation

Fix `getCoreRisk` to pass the actual `amount` (not hardcoded `1`) so that `CoreRisk.longWeight` reflects the directional weight for the subaccount's actual position:

```solidity
// BaseEngine.sol getCoreRisk — pass actual amount, not 1
return IProductEngine.CoreRisk(
    amount,
    risk.priceX18,
    RiskHelper._getWeightX18(risk, amount, healthType)  // was: hardcoded 1
);
```

Then rename the field to `weight` (or `directedWeight`) to avoid confusion. In `Clearinghouse.getHealth`, `existingWeight` will then correctly reflect `perpShortWeight` for a short perp and `spotShortWeight` for a short spot.

Alternatively, add a `shortWeight` field to `CoreRisk` and populate both, then use the correct directional weight in `getHealth` and `_getSpreadWeightX18`.

---

// Step 2c: spreadWeight = 1 - (1-1.1)/5 = 1.02 → capped at 0.99e18
// Step 3c: existingWeight = (0.9 + 1.1)/2 = 1.0e18
// Step 5c: spread health adjustment = 10 * 2000 * (0.99-1.0) = -200e18

// Difference: +1800e18 health units overstated
// Trader can withdraw ~1800 USDC more than permitted per 10 units of spread
```

    function _getSpreadWeightX18(
        IProductEngine.CoreRisk memory perpCoreRisk,
        IProductEngine.CoreRisk memory spotCoreRisk,
        IProductEngine.HealthType healthType
    ) internal pure returns (int128) {
        if (healthType == IProductEngine.HealthType.PNL) {
            return ONE;
        }
        int128 spreadWeight;
        if (spotCoreRisk.amount > 0) {
            spreadWeight = ONE - (ONE - perpCoreRisk.longWeight) / 5;
        } else {
            spreadWeight = ONE - (ONE - spotCoreRisk.longWeight) / 5;
        }
        int128 maxSpreadWeight;
        if (healthType == IProductEngine.HealthType.INITIAL) {
            maxSpreadWeight = ONE - ONE / 100; // 0.99
        } else {
            maxSpreadWeight = ONE - (ONE * 6) / 1000; // 0.994
        }
        if (spreadWeight > maxSpreadWeight) {
            spreadWeight = maxSpreadWeight;
        }
        return spreadWeight;
    }
```
