Audit Report

## Title
Swap Allowlist Keyed on Router `msg.sender` Allows Any User to Bypass Per-User Access Control via `MetricOmmSimpleRouter` - (File: metric-periphery/contracts/extensions/SwapAllowlistExtension.sol)

## Summary
`SwapAllowlistExtension.beforeSwap` gates swaps by checking `allowedSwapper[pool][sender]`, where `sender` is the pool's `msg.sender` — the router address when a user routes through `MetricOmmSimpleRouter`. If the router is allowlisted (a natural admin action to enable legitimate router-mediated trading), every caller of the public router bypasses the per-user allowlist entirely, regardless of whether they are individually excluded. This breaks the core access-control invariant of compliance-gated or KYC-restricted pools.

## Finding Description
`MetricOmmPool.swap` passes `msg.sender` as the `sender` argument to `_beforeSwap`:

```solidity
// MetricOmmPool.sol lines 230–240
_beforeSwap(
    msg.sender,   // ← becomes `sender` in the extension
    recipient, ...
);
```

`ExtensionCalling._beforeSwap` encodes this verbatim as the first argument to `IMetricOmmExtensions.beforeSwap`:

```solidity
// ExtensionCalling.sol lines 160–176
abi.encodeCall(IMetricOmmExtensions.beforeSwap, (sender, recipient, ...))
```

`SwapAllowlistExtension.beforeSwap` then performs its allowlist check keyed on `(msg.sender=pool, sender)`:

```solidity
// SwapAllowlistExtension.sol lines 37–39
if (!allowAllSwappers[msg.sender] && !allowedSwapper[msg.sender][sender]) {
    revert IMetricOmmPoolActions.NotAllowedToSwap();
}
```

When a user calls `MetricOmmSimpleRouter.exactInputSingle()`, the router calls `pool.swap(params.recipient, ...)` directly:

```solidity
// MetricOmmSimpleRouter.sol lines 72–80
IMetricOmmPoolActions(params.pool).swap(params.recipient, params.zeroForOne, ...);
```

The pool's `msg.sender` is the router, so `sender` = router address. The extension looks up `allowedSwapper[pool][router]`, never consulting `allowedSwapper[pool][user]`. If the router is allowlisted, the check passes for every caller of the router unconditionally.

## Impact Explanation
Any user excluded from the swap allowlist can execute swaps on a restricted pool by routing through `MetricOmmSimpleRouter`. The pool receives input tokens and the user receives output tokens — a direct, real token flow the pool admin intended to block. This constitutes broken core pool functionality: the access-control invariant of the `SwapAllowlistExtension` is completely nullified for all router-mediated swaps, rendering compliance-gated, KYC-restricted, or counterparty-limited pools open to the public.

## Likelihood Explanation
`MetricOmmSimpleRouter` is the standard, publicly deployed periphery entry point. A pool admin who wants allowlisted users to trade via the router will naturally allowlist the router address using `setAllowedToSwap(pool, router, true)`. The moment they do, the per-user allowlist is silently nullified for all router-mediated swaps. No special privilege, exploit setup, or malicious configuration is required — any public user can call `exactInputSingle`, `exactInput`, `exactOutputSingle`, or `exactOutput` on the router to bypass the guard.

## Recommendation
The allowlist check must be performed against the end user, not the intermediary contract. Two sound approaches:

1. **Pass the originating user through `extensionData`**: The router encodes the original `msg.sender` into `extensionData`; the extension reads and verifies it. This is acceptable since the router is a known, audited contract.
2. **Key the allowlist on `recipient`**: Change the extension's lookup to use `recipient` (the address economically benefiting from the swap) rather than `sender`. This is already available as the second argument to `beforeSwap`.

Either way, the router address must never be the identity that the allowlist gates.

## Proof of Concept
```
Setup:
  - Deploy pool with SwapAllowlistExtension configured.
  - Pool admin calls setAllowedToSwap(pool, router, true)
    (so legitimate users can trade via MetricOmmSimpleRouter).
  - Pool admin does NOT call setAllowedToSwap(pool, Alice, true).

Attack:
  1. Alice calls MetricOmmSimpleRouter.exactInputSingle({pool, tokenIn, tokenOut, recipient=Alice, ...}).
  2. Router calls pool.swap(recipient=Alice, ...) — pool's msg.sender = router.
  3. Pool calls _beforeSwap(sender=router, recipient=Alice, ...).
  4. SwapAllowlistExtension checks allowedSwapper[pool][router] → true.
  5. Hook passes. Swap executes. Alice receives output tokens.

Result:
  Alice, explicitly excluded from the per-user allowlist, successfully swaps on the
  restricted pool. The allowlist invariant is broken for all router-mediated swaps.
```
