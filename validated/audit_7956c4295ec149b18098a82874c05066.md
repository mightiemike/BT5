### Title
`ValidResourceBounds` protobuf deserialization silently downcasts `AllResources` to `L1Gas` when L2/L1-data gas are zero, producing a divergent transaction hash preimage — (`File: crates/apollo_protobuf/src/converters/transaction.rs`)

---

### Summary

The `TryFrom<protobuf::ResourceBounds> for ValidResourceBounds` implementation in the sync/P2P protobuf converter silently changes the `ValidResourceBounds` variant from `AllResources` to `L1Gas` whenever both `l2_gas` and `l1_data_gas` are zero. Because `get_tip_resource_bounds_hash` hashes a **different number of elements** for `L1Gas` (tip + l1 + l2 = 3 felts) versus `AllResources` (tip + l1 + l2 + l1_data = 4 felts), a transaction accepted by the gateway with `AllResources{l1_gas: X, l2_gas: 0, l1_data_gas: 0}` will produce a **different hash** when re-verified after a protobuf round-trip. This breaks the canonicalization invariant that the hash of a transaction must be identical at every stage of the pipeline.

---

### Finding Description

**Step 1 — Gateway accepts the transaction with `AllResources`.**

`RpcInvokeTransactionV3` and `InternalRpcInvokeTransactionV3` both store `resource_bounds: AllResourceBounds` (not `ValidResourceBounds`). The gateway's `validate_resource_bounds` only requires `max_possible_fee > 0`, which is satisfied by a non-zero `l1_gas` even when `l2_gas` and `l1_data_gas` are both zero. [1](#0-0) 

**Step 2 — Hash is computed as `AllResources` (4-element preimage).**

`InternalRpcInvokeTransactionV3::resource_bounds()` always returns `ValidResourceBounds::AllResources(self.resource_bounds)`. `get_tip_resource_bounds_hash` therefore appends the `l1_data_gas` element to the hash chain, producing a 4-element poseidon hash. [2](#0-1) [3](#0-2) 

**Step 3 — Protobuf deserialization silently changes the variant.**

When the same transaction is received by a syncing peer via the P2P `InvokeV3` protobuf message, `InvokeTransactionV3` is reconstructed using `ValidResourceBounds::try_from(protobuf::ResourceBounds)`. The conversion checks `l1_data_gas.is_zero() && l2_gas.is_zero()` and, if true, returns `ValidResourceBounds::L1Gas(l1_gas)` — discarding the `AllResources` variant entirely. [4](#0-3) [5](#0-4) 

**Step 4 — Hash recomputed as `L1Gas` (3-element preimage) diverges.**

After deserialization, `InvokeTransactionV3.resource_bounds` is `L1Gas`. `get_tip_resource_bounds_hash` now produces a 3-element poseidon hash (no `l1_data_gas` element). The two hashes are structurally different:

- **Original (AllResources):** `poseidon(tip, pack(l1_gas), pack(l2_gas=0), pack(l1_data_gas=0))`
- **After round-trip (L1Gas):** `poseidon(tip, pack(l1_gas), pack(l2_gas=0))` [3](#0-2) 

**Step 5 — Storage serializer preserves the variant; protobuf does not.**

The JSON/binary storage serializer for `ValidResourceBounds` distinguishes `L1Gas` (2-key map) from `AllResources` (3-key map with `L1_DATA_GAS`), so a node that stores the transaction locally preserves the correct variant. Only the protobuf path (used for P2P sync) performs the lossy downcast. [6](#0-5) [7](#0-6) 

---

### Impact Explanation

A syncing node that receives a block containing such a transaction via P2P will:

1. **Compute a wrong transaction hash** from the deserialized `InvokeTransactionV3`, causing `validate_transaction_hash` to return `false` and the block to be rejected.
2. **Select the wrong gas computation mode**: `get_gas_vector_computation_mode()` returns `GasVectorComputationMode::NoL2Gas` for `L1Gas` and `GasVectorComputationMode::All` for `AllResources`. This changes fee-check and resource-bound logic during execution, potentially producing a different execution result or revert outcome than the sequencer produced.

This matches the allowed impact: **"High. Transaction conversion or signature/hash logic binds the wrong signer, hash, type, or executable payload."**

---

### Likelihood Explanation

The trigger requires submitting a V3 invoke transaction with `l2_gas = {max_amount: 0, max_price_per_unit: 0}` and `l1_data_gas = {max_amount: 0, max_price_per_unit: 0}` while keeping `l1_gas` non-zero. This passes the gateway's `validate_resource_bounds` check (non-zero fee from `l1_gas`) and the `min_gas_price` check only when `min_gas_price == 0` (the default in many configurations) or when `validate_resource_bounds` is disabled. The condition is reachable by any unprivileged user who can submit transactions to the RPC.

---

### Recommendation

In `TryFrom<protobuf::ResourceBounds> for ValidResourceBounds`, remove the lossy downcast. Always reconstruct `AllResources` when all three resource fields are present in the protobuf message, regardless of whether their values are zero:

```rust
// crates/apollo_protobuf/src/converters/transaction.rs
impl TryFrom<protobuf::ResourceBounds> for ValidResourceBounds {
    type Error = ProtobufConversionError;
    fn try_from(value: protobuf::ResourceBounds) -> Result<Self, Self::Error> {
        let l1_gas: ResourceBounds = value.l1_gas
            .ok_or(missing("ResourceBounds::l1_gas"))?.try_into()?;
        let l2_gas: ResourceBounds = value.l2_gas
            .ok_or(missing("ResourceBounds::l2_gas"))?.try_into()?;
        // If l1_data_gas is absent (legacy 0.13.2 message), use L1Gas variant.
        // If l1_data_gas is present (even if zero), use AllResources to preserve hash preimage.
        match value.l1_data_gas {
            None => Ok(ValidResourceBounds::L1Gas(l1_gas)),
            Some(l1_data_gas) => {
                let l1_data_gas: ResourceBounds = l1_data_gas.try_into()?;
                Ok(ValidResourceBounds::AllResources(
                    AllResourceBounds { l1_gas, l2_gas, l1_data_gas }
                ))
            }
        }
    }
}
```

The `L1Gas` variant should only be selected when the protobuf message genuinely omits `l1_data_gas` (i.e., a pre-0.13.3 legacy message), not when `l1_data_gas` is present but zero.

### Proof of Concept

```rust
// Demonstrates hash divergence across the protobuf round-trip.
#[test]
fn test_allresources_zero_l2_l1data_hash_diverges_after_protobuf_roundtrip() {
    use starknet_api::transaction::fields::{
        AllResourceBounds, ResourceBounds, ValidResourceBounds, GasAmount, GasPrice, Tip,
    };
    use starknet_api::transaction_hash::get_tip_resource_bounds_hash;
    use apollo_protobuf::protobuf;

    // AllResources with l2_gas=0, l1_data_gas=0 — accepted by gateway.
    let all_resources = ValidResourceBounds::AllResources(AllResourceBounds {
        l1_gas:      ResourceBounds { max_amount: GasAmount(100), max_price_per_unit: GasPrice(1) },
        l2_gas:      ResourceBounds::default(), // zero
        l1_data_gas: ResourceBounds::default(), // zero
    });

    let tip = Tip(0);
    let hash_before = get_tip_resource_bounds_hash(&all_resources, &tip).unwrap();

    // Serialize to protobuf and deserialize back (the P2P sync path).
    let proto: protobuf::ResourceBounds = all_resources.into();
    let after: ValidResourceBounds = proto.try_into().unwrap();

    // After round-trip, variant is L1Gas — only 3 elements in hash preimage.
    assert!(matches!(after, ValidResourceBounds::L1Gas(_)),
        "expected L1Gas after round-trip");

    let hash_after = get_tip_resource_bounds_hash(&after, &tip).unwrap();

    // The two hashes differ — the syncing node computes the wrong tx hash.
    assert_ne!(hash_before, hash_after,
        "hash must diverge: AllResources(4 elems) != L1Gas(3 elems)");
}
```

pub fn get_tip_resource_bounds_hash(
    resource_bounds: &ValidResourceBounds,
    tip: &Tip,
) -> Result<Felt, StarknetApiError> {
    let l1_resource_bounds = resource_bounds.get_l1_bounds();
    let l2_resource_bounds = resource_bounds.get_l2_bounds();

    // L1 and L2 gas bounds always exist.
    // Old V3 txs always have L2 gas bounds of zero, but they exist.
    let mut resource_felts = vec![
        get_concat_resource(&l1_resource_bounds, L1_GAS)?,
        get_concat_resource(&l2_resource_bounds, L2_GAS)?,
    ];

    // For new V3 txs, need to also hash the data gas bounds.
    resource_felts.extend(match resource_bounds {
        ValidResourceBounds::L1Gas(_) => vec![],
        ValidResourceBounds::AllResources(all_resources) => {
            vec![get_concat_resource(&all_resources.l1_data_gas, L1_DATA_GAS)?]
        }
    });

    Ok(HashChain::new().chain(&tip.0.into()).chain_iter(resource_felts.iter()).get_poseidon_hash())
}
