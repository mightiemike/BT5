### Title
Missing Access Control on `run_mainchain_gc` Allows Any Caller to Aggressively Prune Mainchain History — (`contract/src/lib.rs`)

---

### Summary

`run_mainchain_gc` is a public, state-mutating function that permanently deletes mainchain block headers from contract storage. It carries only a `#[pause]` guard (which controls behavior when the contract is paused) but has **no caller restriction**. Any unprivileged NEAR account can invoke it with an arbitrarily large `batch_size`, pruning all excess mainchain history in a single call. This can permanently prevent legitimate chain reorganizations from completing, corrupting the canonical chain selection invariant.

---

### Finding Description

`submit_blocks` is correctly gated behind `#[trusted_relayer]`, which enforces that only registered relayers or accounts holding `Role::DAO` / `Role::UnrestrictedSubmitBlocks` may submit headers. [1](#0-0) 

`run_mainchain_gc`, however, carries only `#[pause(except(roles(Role::UnrestrictedRunGC)))]`. That attribute only controls whether the function is callable when the contract is **paused**. When the contract is live (unpaused), the attribute imposes no caller restriction at all — any NEAR account may call it. [2](#0-1) 

The GC logic removes mainchain blocks from `initial_height` up to `min(total_excess, batch_size)` blocks: [3](#0-2) 

The normal relayer path calls `run_mainchain_gc(num_of_headers)` inside `submit_blocks`, where `num_of_headers` is the small batch just submitted — a conservative, incremental prune. An unprivileged attacker can instead call `run_mainchain_gc` directly with `batch_size = u64::MAX`, which collapses `selected_amount_to_remove` to `total_amount_to_remove` and prunes **all** excess blocks in one transaction.

---

### Impact Explanation

The contract's own documentation explicitly states the consequence of aggressive GC:

> **Caveat**: If mainchain blocks near the fork point have been garbage collected, reorg will fail — the contract panics with `PrevBlockNotFound` when it cannot walk the chain back to the common ancestor. [4](#0-3) 

An attacker who observes a pending fork (all state is public on NEAR) can call `run_mainchain_gc(u64::MAX)` to prune the common ancestor before the reorg completes. The reorg then panics permanently with `PrevBlockNotFound`, locking the contract on the wrong canonical chain. Downstream consumers of `verify_transaction_inclusion` / `verify_transaction_inclusion_v2` will then verify transactions against an incorrect chain tip, producing false inclusion results. [5](#0-4) 

---

### Likelihood Explanation

The entry path requires no privilege: any NEAR account can submit a transaction calling `run_mainchain_gc`. All fork state is publicly visible on-chain, so an attacker can time the call precisely. The attack is cheap (a single NEAR transaction with no deposit required) and irreversible once the ancestor block is deleted.

---

### Recommendation

Add a caller restriction to `run_mainchain_gc` analogous to the `#[trusted_relayer]` guard on `submit_blocks`. The simplest approach is to require the caller to hold an existing privileged role (e.g., `Role::DAO` or a new dedicated `GCManager` role) before the function body executes:

```rust
#[pause(except(roles(Role::UnrestrictedRunGC)))]
pub fn run_mainchain_gc(&mut self, batch_size: u64) {
    require!(
        self.acl_has_role(Role::DAO, &env::predecessor_account_id())
            || self.acl_has_role(Role::UnrestrictedRunGC, &env::predecessor_account_id()),
        "Unauthorized: caller lacks GC permission"
    );
    // ... existing logic
}
```

Alternatively, make `run_mainchain_gc` a private helper and expose a separate, role-gated public wrapper.

---

### Proof of Concept

1. Deploy the contract and initialize it with `gc_threshold = 100`.
2. Submit 200 mainchain headers via a trusted relayer account. The mainchain now has 100 excess blocks.
3. Observe a fork branch submitted at height 50 (within the excess zone).
4. From **any** unprivileged NEAR account, call:
   ```
   run_mainchain_gc(batch_size: 18446744073709551615)
   ```
5. The contract prunes all 100 excess blocks, deleting the common ancestor at height 50.
6. Attempt to submit the fork's tip header. The contract walks back to find the common ancestor, hits the deleted block, and panics with `PrevBlockNotFound`.
7. The reorg is permanently blocked; the canonical chain pointer remains on the pre-reorg chain. [6](#0-5)

    /// Public call to run GC on a mainchain.
    /// `batch_size` is how many block headers should be removed in the execution
    ///
    /// # Panics
    /// If initial blockheader or tip blockheader are not in a header pool
    #[pause(except(roles(Role::UnrestrictedRunGC)))]
    pub fn run_mainchain_gc(&mut self, batch_size: u64) {
        let initial_blockheader = self
            .headers_pool
            .get(&self.mainchain_initial_blockhash)
            .unwrap_or_else(|| env::panic_str(ERR_KEY_NOT_EXIST));

        let tip_blockheader = self
            .headers_pool
            .get(&self.mainchain_tip_blockhash)
            .unwrap_or_else(|| env::panic_str(ERR_KEY_NOT_EXIST));

        let amount_of_headers_we_store =
            tip_blockheader.block_height - initial_blockheader.block_height + 1;

        if amount_of_headers_we_store > self.gc_threshold {
            let total_amount_to_remove = amount_of_headers_we_store - self.gc_threshold;
            let selected_amount_to_remove = std::cmp::min(total_amount_to_remove, batch_size);

            let start_removal_height = initial_blockheader.block_height;
            let end_removal_height = initial_blockheader.block_height + selected_amount_to_remove;
            env::log_str(&format!(
                "Num of blocks to remove {selected_amount_to_remove}"
            ));

            for height in start_removal_height..end_removal_height {
                let blockhash = &self
                    .mainchain_height_to_header
                    .get(&height)
                    .unwrap_or_else(|| env::panic_str(ERR_KEY_NOT_EXIST));

                self.remove_block_header(blockhash);
                self.mainchain_height_to_header.remove(&height);
            }

            self.mainchain_initial_blockhash = self
                .mainchain_height_to_header
                .get(&end_removal_height)
                .unwrap_or_else(|| env::panic_str(ERR_KEY_NOT_EXIST));
        }
    }
