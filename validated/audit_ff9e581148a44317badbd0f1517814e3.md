### Title
Gas-Key Exec-Fee Uses Wire Length Instead of Trie-ID Length for ML-DSA-65 Keys in Host Functions — (File: `runtime/near-vm-runner/src/logic/logic.rs`)

### Summary

The gas-key host functions `promise_batch_action_add_gas_key_with_full_access`, `promise_batch_action_add_gas_key_with_function_call`, and `promise_batch_action_transfer_to_gas_key` compute the exec fee using the raw borsh/wire length of the public key (`public_key_len as usize`). For ML-DSA-65 keys this is 1953 bytes. The actual trie key, however, stores the SHA3-256 hash form (33 bytes, via `trie_id_len()`). The runtime-level fee path correctly uses `trie_id_len()`, but the host-function path does not, producing a divergent exec-fee value of 1953 vs. 33 bytes — a ~1920-byte overcharge per nonce per key.
