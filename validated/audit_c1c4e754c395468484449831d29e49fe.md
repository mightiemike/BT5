Audit Report

## Title
`SwapAllowlistExtension.beforeSwap` gates the router address instead of the actual user, enabling full allowlist bypass via the router — (`metric-periphery/contracts/extensions/SwapAllowlistExtension.sol`)

## Summary

`SwapAllowlistExtension.beforeSwap` checks the `sender` argument, which the pool sets to `msg.sender` of `pool.swap()`. When swaps are routed through `MetricOmmSimpleRouter`, `msg.sender` of `pool.swap()` is the router contract, not the end user. If the pool admin adds the router to the allowlist (the only way to let allowlisted users use the router), every address — including those explicitly excluded — can bypass the per-user gate by routing through the router.

## Finding Description

**Root cause — extension checks the wrong actor:**

`beforeSwap` receives `sender` as its first argument and checks `allowedSwapper[msg.sender][sender]`, where `msg.sender` is the pool and `sender` is whoever called `pool.swap()`. [1](#0-0) 

**What the pool passes as `sender`:**

The pool always passes `msg.sender` — the direct caller of `pool.swap()` — as the first argument to `_beforeSwap`. [2](#0-1) 

**What the router passes to the pool:**

In `exactInputSingle`, the router calls `pool.swap(params.recipient, ...)` directly. The pool's `msg.sender` is therefore the router contract address. The real user's address is only stored in transient callback context via `_setNextCallbackContext(..., msg.sender, ...)`, which is never visible to the extension. [3](#0-2) 

The same pattern holds for `exactInput` multi-hop: [4](#0-3) 

And for `exactOutput` recursive callbacks in `_exactOutputIterateCallback`: [5](#0-4) 

In every router path, `sender` seen by the extension is the router address, not the end user. The existing guard `allowedSwapper[msg.sender][sender]` is therefore checking the router's allowlist status, not the user's.

## Impact Explanation

A pool admin who deploys a curated pool with `SwapAllowlistExtension` and wants allowlisted users to use the standard router has only one option: add the router address to `allowedSwapper[pool]`. Once the router is allowlisted, **every** address — including those explicitly excluded from the per-user allowlist — can bypass the gate by calling any router entry point (`exactInputSingle`, `exactInput`, `exactOutputSingle`, `exactOutput`). The allowlist is rendered completely ineffective for all router-mediated swaps. This breaks the core curation invariant of the extension: that only approved addresses may trade in the pool. The admin has no way to simultaneously allow allowlisted users to use the router and block non-allowlisted users from using the router, because the extension has no visibility into the actual payer.

## Likelihood Explanation

The trigger is a reasonable, expected admin action. The `SwapAllowlistExtension` is the supported periphery mechanism for curated pools. Any admin who wants allowlisted users to access the standard router will add the router to the allowlist. The bypass is then immediately available to any unprivileged user with no further preconditions. No special permissions, tokens, or setup are required beyond calling a public router entry point.

## Recommendation

The `beforeSwap` hook should gate on the economic actor (the payer), not the intermediary. The preferred fix is to require the router to forward the payer identity in a standardized field of `extensionData`, validated against a registry of trusted routers. The extension would then decode the payer from `extensionData` when `sender` is a trusted router, and fall back to checking `sender` directly for non-router callers. This preserves the operator pattern while correctly attributing the economic action to the real user.

## Proof of Concept

```
Setup:
  pool configured with SwapAllowlistExtension
  admin allowlists: userA (KYC'd)
  admin also adds router to allowlist (to let userA use the router)

Attack:
  userB (not KYC'd) calls:
    router.exactInputSingle({pool: pool, ..., extensionData: ""})

  Execution trace:
    router.exactInputSingle()          [msg.sender = userB]
      pool.swap(recipient, ...)        [msg.sender = router]
        _beforeSwap(sender=router, ...)
          SwapAllowlistExtension.beforeSwap(sender=router, ...)
            allowedSwapper[pool][router] == true  ← passes
    swap executes for userB

userB successfully swaps in a pool restricted to userA only.
```
