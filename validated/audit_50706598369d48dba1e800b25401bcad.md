### Title
Incorrect `alreadyMatched` Quote Reconstruction Using Current Maker Price Corrupts Taker Fee Metering for Partially Filled Orders - (File: `core/contracts/OffchainExchange.sol`)

---

### Summary

In `OffchainExchange.matchOrders`, the `alreadyMatched` value passed to `applyFee` is computed by multiplying the **current maker's price** by the cumulative **base amount** already filled (`filledAmounts[digest]`). Because `filledAmounts` accumulates base units across multiple fills that may have executed at different prices, this conversion produces an incorrect quote-denominated `alreadyMatched`. The corrupted value is then used to meter the `minSize` fee threshold, causing takers to be systematically overcharged or undercharged on the percentage fee component whenever their order is partially filled across multiple `matchOrders` calls at different prices.

---

### Finding Description

`matchOrders` calls `applyFee` for the taker with:

```solidity
applyFee(
    callState.productId,
    ordersInfo.taker,
    market,
    -maker.order.priceX18.mul(filledAmounts[ordersInfo.taker.digest]),  // alreadyMatched
    taker.order.appendix,
    true
);
``` [1](#0-0) 

`filledAmounts[ordersInfo.taker.digest]` is the cumulative **base amount** filled across all prior `matchOrders` calls for this taker order digest. It is updated only after `applyFee` returns:

```solidity
if (taker.order.sender != X_ACCOUNT) {
    filledAmounts[ordersInfo.taker.digest] += ordersInfo.taker.amountDelta;
}
``` [2](#0-1) 

The conversion to quote uses `maker.order.priceX18` — the **current** maker's price. If the taker's order was previously partially filled against a different maker at a different price, the reconstructed `alreadyMatched` does not equal the actual quote amount already paid.

Inside `applyFee`, this value drives the `minSize` threshold metering:

```solidity
int128 feeApplied = MathHelper.abs(alreadyMatched + matchQuote) - market.minSize;
feeApplied = MathHelper.min(feeApplied, matchQuote.abs());
``` [3](#0-2) 

The design intent is: charge the percentage fee only on the portion of the cumulative fill that exceeds `minSize` (the first `minSize` of quote is subject only to the flat minimum fee). When `alreadyMatched` is inflated (current price > historical fill price), `feeApplied` is inflated — up to the cap of `abs(matchQuote)` — overcharging the taker. When `alreadyMatched` is deflated (current price < historical fill price), `feeApplied` is deflated, undercharging the taker and reducing protocol fee revenue.

The flat minimum fee branch is gated on `alreadyMatched == 0`:

```solidity
if (alreadyMatched == 0) {
    meteredQuote += market.minSize;
    ...
}
``` [4](#0-3) 

This branch is unaffected (it correctly fires only on the first fill, since `filledAmounts` is zero only then). The corruption is confined to the `feeApplied` threshold calculation.

---

### Impact Explanation

For any taker order partially filled across two or more `matchOrders` calls at different prices:

- **Overcharge path** (current price > prior fill price): `alreadyMatched` is inflated, pushing `feeApplied` toward `abs(matchQuote)`. The taker pays the percentage fee on more quote than the `minSize` threshold logic intends. Maximum overcharge per fill is bounded by `takerFeeRate × abs(matchQuote)`.
- **Undercharge path** (current price < prior fill price): `alreadyMatched` is deflated, reducing `feeApplied` below the correct value. The protocol collects less fee than intended, corrupting `market.collectedFees` and the downstream `dumpFees` accounting.

The corrupted `orderInfo.quoteDelta` is then applied to the taker's balance via `_updateBalances`, making the asset delta incorrect on-chain. [5](#0-4) 

---

### Likelihood Explanation

Partial fills of taker orders across multiple `matchOrders` calls at different prices are routine in any active market. No special attacker action is required — normal sequencer operation matching a large taker order against a sequence of makers at evolving market prices is sufficient to trigger the miscalculation on every subsequent fill after the first.

---

### Recommendation

Track the cumulative **quote** amount filled per order digest in a separate mapping (e.g., `mapping(bytes32 => int128) public filledQuoteAmounts`). Update it alongside `filledAmounts` after each fill:

```solidity
filledQuoteAmounts[ordersInfo.taker.digest] += ordersInfo.taker.quoteDelta; // before fee
```

Pass this directly as `alreadyMatched` instead of reconstructing it from base amounts and the current price:

```solidity
applyFee(
    callState.productId,
    ordersInfo.taker,
    market,
    filledQuoteAmounts[ordersInfo.taker.digest],  // actual quote already matched
    taker.order.appendix,
    true
);
```

### Proof of Concept

**Setup:** `minSize = 100 USDC`, `takerFeeRate = 2 bps`

**Fill 1:** Taker buys 90 units at maker price 1 USDC/unit
- `matchQuote = -90 USDC`, `alreadyMatched = 0` (first fill, correct)
- `feeApplied = abs(0 + (-90)) - 100 = -10` → clamped to 0; only flat minimum fee applies
- `filledAmounts[digest] = 90`

**Fill 2:** Taker buys 20 units at maker price 10 USDC/unit (`matchQuote = -200 USDC`)
- **Wrong:** `alreadyMatched = -(10 × 90) = -900`
  - `feeApplied = abs(-900 + (-200)) - 100 = 1100 - 100 = 1000`
  - `feeApplied = min(1000, 200) = 200` → fee charged on full 200 USDC
- **Correct:** `alreadyMatched = -90` (actual quote already paid)
  - `feeApplied = abs(-90 + (-200)) - 100 = 290 - 100 = 190`
  - `feeApplied = min(190, 200) = 190` → fee charged on 190 USDC
- **Overcharge:** `2 bps × (200 - 190) = 2 bps × 10 USDC = 0.002 USDC` extracted from taker's balance incorrectly; `market.collectedFees` is inflated by the same amount, corrupting the `dumpFees` settlement. [6](#0-5) [1](#0-0)

**File:** core/contracts/OffchainExchange.sol (L509-565)
```text
    function applyFee(
        uint32 productId,
        OrderInfo memory orderInfo,
        MarketInfo memory market,
        int128 alreadyMatched, // in quote
        uint128 appendix,
        bool taker
    ) internal {
        // X account is passthrough for trading and incurs
        // no fees
        if (orderInfo.sender == X_ACCOUNT) {
            return;
        }
        int128 matchQuote = orderInfo.quoteDelta;
        int128 meteredQuote = 0;
        if (taker) {
            // flat minimum fee
            if (alreadyMatched == 0) {
                meteredQuote += market.minSize;
                if (matchQuote < 0) {
                    meteredQuote = -meteredQuote;
                }
            }

            // exclude the portion on [0, self.min_size) for match_quote and
            // add to metered_quote
            // fee is only applied on [minSize, quote_amount)
            int128 feeApplied = MathHelper.abs(alreadyMatched + matchQuote) -
                market.minSize;
            feeApplied = MathHelper.min(feeApplied, matchQuote.abs());
            if (feeApplied > 0) {
                if (matchQuote < 0) {
                    feeApplied = -feeApplied;
                }
                meteredQuote += feeApplied;
            }
        } else {
            // for maker rebates things stay the same
            meteredQuote += matchQuote;
        }
        FeeInfo memory feeInfo = getUserFeeRateWithBuilder(
            orderInfo.sender,
            productId,
            appendix,
            taker
        );

        int128 keepRateX18 = ONE - feeInfo.feeRate;
        int128 newMeteredQuote = (meteredQuote > 0)
            ? meteredQuote.mul(keepRateX18)
            : meteredQuote.div(keepRateX18);
        orderInfo.fee = meteredQuote - newMeteredQuote;
        orderInfo.builderFee = matchQuote.abs().mul(feeInfo.builderFeeRate);
        orderInfo.quoteDelta =
            orderInfo.quoteDelta -
            orderInfo.fee -
            orderInfo.builderFee;
```
