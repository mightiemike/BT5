### Title
`ValidResourceBounds` Protobuf Round-Trip Silently Downgrades `AllResources` to `L1Gas`, Producing a Different Transaction Hash - (File: `crates/apollo_protobuf/src/converters/transaction.rs`)

### Summary

The protobuf deserialization of `ValidResourceBounds` is not idempotent. A V3 transaction carrying `AllResources` with `l2_gas = 0` and `l1_data_gas = 0` serializes to protobuf with all three resource-bound fields present (all zero), but deserializes back as `ValidResourceBounds::L1Gas`. Because `get_tip_resource_bounds_hash` hashes a different number of elements for the two variants, the transaction hash computed after the protobuf round-trip differs from the hash computed before it. Any sequencer component that re-derives the hash from the deserialized `Transaction` object (state-sync reader, RPC trace/simulation, fee-estimation) will bind the wrong hash to the transaction.

---

### Finding Description

**Step 1 – Serialization (correct variant, all fields present)**

`From<ValidResourceBounds> for protobuf::ResourceBounds` always emits all three optional fields:

```rust
// crates/apollo_protobuf/src/converters/transaction.rs  lines 471-489
ValidResourceBounds::AllResources(AllResourceBounds { l1_gas, l2_gas, l1_data_gas }) =>
    protobuf::ResourceBounds {
        l1_gas:      Some(l1_gas.into()),
        l2_gas:      Some(l2_gas.into()),       // zero
        l1_data_gas: Some(l1_data_gas.into()),  // zero
    },
``` [1](#0-0) 

**Step 2 – Deserialization (variant silently changed)**

`TryFrom<protobuf::ResourceBounds> for ValidResourceBounds` inspects the *values* of the fields, not which variant was originally used:

```rust
// crates/apollo_protobuf/src/converters/transaction.rs  lines 417-436
let l1_data_gas = value.l1_data_gas.unwrap_or_default();   // zero
...
Ok(if l1_data_gas.is_zero() && l2_gas.is_zero() {
    ValidResourceBounds::L1Gas(l1_gas)          // ← variant changed!
} else {
    ValidResourceBounds::AllResources(...)
})
``` [2](#0-1) 

An `AllResources` transaction with both `l2_gas = 0` and `l1_data_gas = 0` exits the round-trip as `L1Gas`.

**Step 3 – Hash divergence**

`get_tip_resource_bounds_hash` hashes a different number of elements depending on the variant:

```rust
// crates/starknet_api/src/transaction_hash.rs  lines 188-211
let mut resource_felts = vec![
    get_concat_resource(&l1_resource_bounds, L1_GAS)?,
    get_concat_resource(&l2_resource_bounds, L2_GAS)?,
];
resource_felts.extend(match resource_bounds {
    ValidResourceBounds::L1Gas(_) => vec![],                          // 2 elements
    ValidResourceBounds::AllResources(all) =>
        vec![get_concat_resource(&all.l1_data_gas, L1_DATA_GAS)?],   // 3 elements
});
Ok(HashChain::new().chain(&tip.0.into()).chain_iter(resource_felts.iter()).get_poseidon_hash())
``` [3](#0-2) 

- **Before round-trip** (`AllResources`): `H1 = poseidon(tip, l1_gas_packed, l2_gas_packed(0), l1_data_gas_packed(0))` — 4 elements
- **After round-trip** (`L1Gas`): `H2 = poseidon(tip, l1_gas_packed, l2_gas_packed(0))` — 3 elements
- `H1 ≠ H2` by the collision-resistance of Poseidon.

**Step 4 – Where the wrong hash propagates**

The `TryFrom<protobuf::ResourceBounds> for ValidResourceBounds` converter is used for the regular `Transaction` types (`InvokeTransactionV3`, `DeclareTransactionV3`, `DeployAccountTransactionV3`) in the P2P state-sync path. Any downstream call to `get_transaction_hash` / `validate_transaction_hash` on the deserialized object will compute `H2` instead of `H1`. [4](#0-3) 

The RPC execution / fee-estimation / tracing path also calls `get_transaction_hash` on the stored `Transaction` object: [5](#0-4) 

Note: the *consensus* path is unaffected because `ConsensusTransaction` wraps `RpcTransaction`, which stores `AllResourceBounds` (not `ValidResourceBounds`) and uses a separate converter (`TryFrom<protobuf::ResourceBounds> for AllResourceBounds`) that requires all three fields and never produces `L1Gas`. [6](#0-5) 

---

### Impact Explanation

**High – Transaction conversion or signature/hash logic binds the wrong hash.**

A V3 transaction signed and admitted with hash `H1` (AllResources) is re-identified as hash `H2` (L1Gas) after any protobuf round-trip in the state-sync or RPC layer. An RPC call to `starknet_getTransactionByHash(H1)` on a synced node may return no result (the node stored it under `H2`), or `starknet_estimateFee` / `starknet_simulateTransactions` may compute fees under the wrong `GasVectorComputationMode` (`NoL2Gas` instead of `All`), returning an authoritative-looking but incorrect value.

---

### Likelihood Explanation

**High.** Any V3 transaction that sets `l2_gas = 0` and `l1_data_gas = 0` (a common pattern for pure L1-gas transactions submitted after the `AllResources` format was introduced) triggers the downgrade automatically and silently. No special privilege is required; the trigger is a normal user transaction.

---

### Recommendation

The deserialization logic must preserve the original variant rather than inferring it from field values. Two concrete options:

1. **Add an explicit discriminant field** to `protobuf::ResourceBounds` (e.g., `resource_bounds_type: enum { L1GAS_ONLY = 0, ALL_RESOURCES = 1 }`) and use it during deserialization.
2. **Remove the value-based inference** in `TryFrom<protobuf::ResourceBounds> for ValidResourceBounds`: always produce `AllResources` when all three fields are present (even if zero), and produce `L1Gas` only when `l2_gas` and `l1_data_gas` are absent (`None`), not merely zero.

Option 2 is a minimal fix:

```rust
// Proposed fix
let l1_data_gas_opt = value.l1_data_gas;
let l2_gas_opt      = value.l2_gas;
Ok(match (l2_gas_opt, l1_data_gas_opt) {
    (None, None) => ValidResourceBounds::L1Gas(l1_gas),
    (Some(l2), Some(d)) => ValidResourceBounds::AllResources(AllResourceBounds {
        l1_gas,
        l2_gas:      l2.try_into()?,
        l1_data_gas: d.try_into()?,
    }),
    _ => return Err(missing("ResourceBounds: inconsistent optional fields")),
})
```

### Proof of Concept

```
1. Construct a V3 InvokeTransactionV3 with:
     resource_bounds = AllResources { l1_gas: {max_amount:1000, max_price:1}, l2_gas: ZERO, l1_data_gas: ZERO }
     tip = 0

2. Compute H1 = get_tip_resource_bounds_hash(AllResources{...}, tip=0)
   → poseidon(0, pack(L1_GAS,1000,1), pack(L2_GAS,0,0), pack(L1_DATA_GAS,0,0))   [4 elements]

3. Serialize to protobuf::ResourceBounds:
   { l1_gas: Some({1000,1}), l2_gas: Some({0,0}), l1_data_gas: Some({0,0}) }

4. Deserialize via TryFrom<protobuf::ResourceBounds> for ValidResourceBounds:
   l1_data_gas.is_zero() && l2_gas.is_zero()  → ValidResourceBounds::L1Gas({1000,1})

5. Compute H2 = get_tip_resource_bounds_hash(L1Gas{...}, tip=0)
   → poseidon(0, pack(L1_GAS,1000,1), pack(L2_GAS,0,0))                           [3 elements]

6. Assert H1 ≠ H2  ✓  (Poseidon of 4 elements ≠ Poseidon of 3 elements)

7. Any downstream call to get_transaction_hash on the deserialized Transaction
   embeds H2 instead of H1, diverging from the hash the signer committed to.
```

impl TryFrom<protobuf::ResourceBounds> for ValidResourceBounds {
    type Error = ProtobufConversionError;
    fn try_from(value: protobuf::ResourceBounds) -> Result<Self, Self::Error> {
        let Some(l1_gas) = value.l1_gas else {
            return Err(missing("ResourceBounds::l1_gas"));
        };
        let Some(l2_gas) = value.l2_gas else {
            return Err(missing("ResourceBounds::l2_gas"));
        };
        // TODO(Shahak): Assert data gas is not none once we remove support for 0.13.2.
        let l1_data_gas = value.l1_data_gas.unwrap_or_default();
        let l1_gas: ResourceBounds = l1_gas.try_into()?;
        let l2_gas: ResourceBounds = l2_gas.try_into()?;
        let l1_data_gas: ResourceBounds = l1_data_gas.try_into()?;
        Ok(if l1_data_gas.is_zero() && l2_gas.is_zero() {
            ValidResourceBounds::L1Gas(l1_gas)
        } else {
            ValidResourceBounds::AllResources(AllResourceBounds { l1_gas, l2_gas, l1_data_gas })
        })
    }

pub fn get_tip_resource_bounds_hash(
    resource_bounds: &ValidResourceBounds,
    tip: &Tip,
) -> Result<Felt, StarknetApiError> {
    let l1_resource_bounds = resource_bounds.get_l1_bounds();
    let l2_resource_bounds = resource_bounds.get_l2_bounds();

    // L1 and L2 gas bounds always exist.
    // Old V3 txs always have L2 gas bounds of zero, but they exist.
    let mut resource_felts = vec![
        get_concat_resource(&l1_resource_bounds, L1_GAS)?,
        get_concat_resource(&l2_resource_bounds, L2_GAS)?,
    ];

    // For new V3 txs, need to also hash the data gas bounds.
    resource_felts.extend(match resource_bounds {
        ValidResourceBounds::L1Gas(_) => vec![],
        ValidResourceBounds::AllResources(all_resources) => {
            vec![get_concat_resource(&all_resources.l1_data_gas, L1_DATA_GAS)?]
        }
    });

    Ok(HashChain::new().chain(&tip.0.into()).chain_iter(resource_felts.iter()).get_poseidon_hash())
}
