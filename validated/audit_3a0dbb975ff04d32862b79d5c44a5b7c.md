### Title
Deprecated `verify_transaction_inclusion` Remains Publicly Callable, Bypassing Coinbase Proof Validation — (`contract/src/lib.rs`)

### Summary
The deprecated `verify_transaction_inclusion` function is still a live, publicly callable NEAR entry point. It omits the coinbase Merkle proof check that `verify_transaction_inclusion_v2` was specifically introduced to enforce. Any unprivileged caller can invoke the deprecated path directly, supply a 64-byte internal Merkle-tree node as `tx_id`, and receive a `true` proof result for a transaction that was never broadcast or confirmed on Bitcoin.

### Finding Description
`verify_transaction_inclusion_v2` was introduced to close the well-known 64-byte transaction Merkle-proof forgery vulnerability (documented in the function's own docstring and in the referenced BitMEX post). The mitigation works by requiring the caller to also supply a coinbase Merkle proof; if the coinbase proof is valid, the `tx_id` cannot be an internal node because internal nodes are 64 bytes and real transaction hashes are 32 bytes. [1](#0-0) 

The deprecated predecessor, however, performs no such check: [2](#0-1) 

Rust's `#[deprecated]` attribute emits a compiler warning only; it does not remove the symbol from the compiled WASM binary or prevent NEAR from dispatching calls to it. The function is decorated with `#[pause]` (not `#[private]`), so it remains reachable by any external account whenever the contract is unpaused.

The only validation the deprecated path applies to `tx_id` is: [3](#0-2) 

An internal Merkle node is a valid 32-byte hash and will pass this check. `compute_root_from_merkle_proof` then walks the proof upward from that node and correctly reconstructs the block's `merkle_root`, returning `true`. [4](#0-3) 

### Impact Explanation
The function returns a `bool` that downstream NEAR contracts are expected to treat as authoritative proof of Bitcoin transaction inclusion. A `true` result for a fabricated `tx_id` (an internal node) corrupts the proof result — the exact corrupted value identified in the disqualification checklist. Any bridge, escrow, or cross-chain protocol that gates an action on this return value can be deceived into releasing funds or updating state for a Bitcoin transaction that does not exist.

### Likelihood Explanation
The entry point requires no role, no stake, and no privileged key. Bitcoin block data (including all internal Merkle nodes) is fully public. An attacker needs only to:
1. Pick any confirmed Bitcoin block already stored in `headers_pool`.
2. Compute an internal node at any depth of its Merkle tree.
3. Build the sibling-hash proof path from that node to the root.
4. Call `verify_transaction_inclusion` with that node as `tx_id`, the matching `tx_index`, and the proof path.

All inputs are attacker-controlled; no brute-force or cryptographic work is required.

### Recommendation
Remove `verify_transaction_inclusion` from the public ABI entirely. Because `#[deprecated]` does not prevent dispatch, the function must be either deleted or converted to `#[private]` so that only the contract itself can call it (which it does internally via `verify_transaction_inclusion_v2`). Alternatively, inline the coinbase-proof check into the deprecated function body so both paths enforce the same invariant.

### Proof of Concept
1. Deploy the contract (Bitcoin feature flag) with `skip_pow_verification = true` and submit a real Bitcoin block header whose Merkle tree has at least two transactions.
2. Compute the left child `L` and right child `R` of the Merkle root. The internal node `N = SHA256d(L || R)` equals the root itself for a two-transaction block, or any intermediate node for deeper trees.
3. Call `verify_transaction_inclusion` with:
   - `tx_id` = `N` (the internal node)
   - `tx_block_blockhash` = the submitted block hash
   - `tx_index` = the index corresponding to `N`'s position
   - `merkle_proof` = the sibling hashes from `N` up to the root
   - `confirmations` = 1
4. The call returns `true`, falsely asserting that `N` is a confirmed Bitcoin transaction in that block. [2](#0-1) [4](#0-3)

    #[deprecated(
        since = "0.5.0",
        note = "Use `verify_transaction_inclusion_v2` instead."

    #[pause]
    pub fn verify_transaction_inclusion(&self, #[serializer(borsh)] args: ProofArgs) -> bool {
        require!(
            args.confirmations <= self.gc_threshold,
            "The required number of confirmations exceeds the number of blocks stored in memory"
        );

        let heaviest_block_header = self
            .headers_pool
            .get(&self.mainchain_tip_blockhash)
            .unwrap_or_else(|| env::panic_str(ERR_KEY_NOT_EXIST));
        let target_block_height = self
            .mainchain_header_to_height
            .get(&args.tx_block_blockhash)
            .unwrap_or_else(|| env::panic_str("block does not belong to the current main chain"));

        // Check requested confirmations. No need to compute proof if insufficient confirmations.
        require!(
            (heaviest_block_header.block_height).saturating_sub(target_block_height) + 1
                >= args.confirmations,
            "Not enough blocks confirmed"
        );

        let header = self
            .headers_pool
            .get(&args.tx_block_blockhash)
            .unwrap_or_else(|| env::panic_str("cannot find requested transaction block"));

        require!(!args.merkle_proof.is_empty(), "Merkle proof is empty");

        // compute merkle tree root and check if it matches block's original merkle tree root
        merkle_tools::compute_root_from_merkle_proof(
            args.tx_id,
            usize::try_from(args.tx_index).unwrap(),
            &args.merkle_proof,
        ) == header.block_header.merkle_root
    }

pub fn compute_root_from_merkle_proof(
    transaction_hash: H256,
    transaction_position: usize,
    merkle_proof: &Vec<H256>,
) -> H256 {
    let mut current_hash = transaction_hash;
    let mut current_position = transaction_position;

    for proof_hash in merkle_proof {
        if current_position % 2 == 0 {
            current_hash = compute_hash(&current_hash, proof_hash);
        } else {
            current_hash = compute_hash(proof_hash, &current_hash);
        }
        current_position /= 2;
    }

    current_hash
}
