### Title
Protobuf `ValidResourceBounds` type-inference silently downgrades `AllResources` to `L1Gas` when l2_gas and l1_data_gas are zero, producing a divergent transaction hash and P2P rejection of gateway-accepted transactions — (`File: crates/apollo_protobuf/src/converters/transaction.rs`)

---

### Summary

The protobuf deserializer for `ValidResourceBounds` uses a value-based heuristic to infer the resource-bounds variant: if both `l2_gas` and `l1_data_gas` are zero it emits `ValidResourceBounds::L1Gas`; otherwise `ValidResourceBounds::AllResources`. The gateway, however, accepts `AllResourceBounds { l1_gas: X, l2_gas: 0, l1_data_gas: 0 }` as a valid V3 transaction and computes its hash as `AllResources` (three-element Poseidon preimage). After a protobuf round-trip the same transaction is silently reclassified as `L1Gas` (two-element preimage), producing a different hash. The receiving peer either rejects the conversion outright (because `TryFrom<InvokeTransactionV3> for RpcInvokeTransactionV3` returns `Err` for the `L1Gas` variant) or recomputes a hash that does not match the signer's commitment, causing signature verification to fail.

---

### Finding Description

**Root cause — heuristic type inference in protobuf deserialization**

`TryFrom<protobuf::ResourceBounds> for ValidResourceBounds` in `crates/apollo_protobuf/src/converters/transaction.rs` decides the variant purely from field values:

```rust
Ok(if l1_data_gas.is_zero() && l2_gas.is_zero() {
    ValidResourceBounds::L1Gas(l1_gas)
} else {
    ValidResourceBounds::AllResources(AllResourceBounds { l1_gas, l2_gas, l1_data_gas })
})
``` [1](#0-0) 

The comment `// TODO(Shahak): Assert data gas is not none once we remove support for 0.13.2.` confirms this is a backward-compatibility shim for pre-0.13.3 transactions. The heuristic is correct for old `L1Gas` transactions (which genuinely have zero l2_gas and l1_data_gas) but is incorrect for new `AllResources` transactions that happen to carry zero l2_gas and l1_data_gas.

**Gateway accepts the triggering transaction**

The stateless validator accepts `AllResourceBounds { l1_gas: NON_EMPTY, l2_gas: 0, l1_data_gas: 0 }` as a valid V3 transaction: [2](#0-1) 

The `RpcInvokeTransactionV3` struct carries `resource_bounds: AllResourceBounds` (not `ValidResourceBounds`), so the gateway always computes the hash as `AllResources`: [3](#0-2) 

**Hash divergence in `get_tip_resource_bounds_hash`**

The hash function includes `l1_data_gas` only for the `AllResources` variant:

```rust
resource_felts.extend(match resource_bounds {
    ValidResourceBounds::L1Gas(_) => vec![],
    ValidResourceBounds::AllResources(all_resources) => {
        vec![get_concat_resource(&all_resources.l1_data_gas, L1_DATA_GAS)?]
    }
});
``` [4](#0-3) 

For `AllResources { l1_gas: X, l2_gas: 0, l1_data_gas: 0 }` the preimage is:
`poseidon(tip, concat(L1_GAS, X), concat(L2_GAS, 0), concat(L1_DATA_GAS, 0))` — **4 elements**.

After protobuf round-trip the same data becomes `L1Gas(X)`, so the preimage is:
`poseidon(tip, concat(L1_GAS, X), concat(L2_GAS, 0))` — **3 elements**.

These produce distinct Poseidon digests, i.e. H1 ≠ H2.

**P2P conversion failure**

`TryFrom<InvokeTransactionV3> for RpcInvokeTransactionV3` explicitly rejects the `L1Gas` variant:

```rust
resource_bounds: match value.resource_bounds {
    ValidResourceBounds::AllResources(bounds) => bounds,
    _ => {
        return Err(StarknetApiError::OutOfRange {
            string: "resource_bounds".to_string(),
        });
    }
},
``` [5](#0-4) 

The P2P mempool path converts `protobuf::MempoolTransaction` → `RpcTransaction` via `txn.try_into()?`: [6](#0-5) 

When the intermediate `InvokeTransactionV3` carries `ValidResourceBounds::L1Gas`, this `try_into()` call returns `Err`, and the receiving peer drops the transaction.

**The `get_invoke_transaction_v3_hash` conditional proof_facts path is the same structural pattern**

The hash function also conditionally appends `proof_facts` only when non-empty: [7](#0-6) 

This is the same "assume uniform structure, skip when zero" pattern — the analog of the proportional-reward function assuming equal bonds.

---

### Impact Explanation

A user submits a valid V3 invoke with `AllResourceBounds { l1_gas: X, l2_gas: 0, l1_data_gas: 0 }`. The gateway accepts it and stores it with hash H1. When the transaction is propagated to peers via the P2P mempool layer, the protobuf round-trip silently reclassifies it as `L1Gas`. The receiving peer either:

1. Fails the `TryFrom` conversion and drops the transaction entirely, or
2. Recomputes hash H2 ≠ H1 and fails signature verification (the signer committed to H1).

In both cases a gateway-accepted, correctly-signed transaction is permanently rejected by all peers and never sequenced. This matches the **High** impact: *"Mempool/gateway/RPC admission accepts invalid transactions or rejects valid transactions before sequencing."*

---

### Likelihood Explanation

The trigger is a standard V3 invoke with only `l1_gas` set (l2_gas and l1_data_gas left at zero). The gateway's stateless validator explicitly accepts this combination. Any wallet or SDK that submits a V3 transaction without setting l2_gas (e.g., relying on the default) can hit this path. The protobuf conversion is exercised on every P2P mempool propagation event.

---

### Recommendation

Remove the value-based heuristic from `TryFrom<protobuf::ResourceBounds> for ValidResourceBounds`. Instead, carry an explicit discriminant in the protobuf schema (e.g., a boolean `is_all_resources` flag, or a separate oneof) so the variant is preserved losslessly across serialization. Until the schema is updated, the deserializer should default to `AllResources` (not `L1Gas`) when `l1_data_gas` is absent, since all new V3 transactions use `AllResources`:

} else { ... })

impl TryFrom<protobuf::ResourceBounds> for ValidResourceBounds {
    type Error = ProtobufConversionError;
    fn try_from(value: protobuf::ResourceBounds) -> Result<Self, Self::Error> {
        let Some(l1_gas) = value.l1_gas else {
            return Err(missing("ResourceBounds::l1_gas"));
        };
        let Some(l2_gas) = value.l2_gas else {
            return Err(missing("ResourceBounds::l2_gas"));

        let l1_data_gas = value.l1_data_gas.unwrap_or_default();

        let l2_gas: ResourceBounds = l2_gas.try_into()?;
        let l1_data_gas: ResourceBounds = l1_data_gas.try_into()?;
        Ok(if l1_data_gas.is_zero() && l2_gas.is_zero() {
            ValidResourceBounds::L1Gas(l1_gas)
        } else {
            ValidResourceBounds::AllResources(AllResourceBounds { l1_gas, l2_gas, l1_data_gas })

            }
            protobuf::mempool_transaction::Txn::InvokeV3(txn) => {
                RpcTransaction::Invoke(RpcInvokeTransaction::V3(txn.try_into()?))

        })
