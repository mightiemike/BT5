### Title
Malicious Counterparty Chain Mints Spurious NFT Vouchers via Double-Prefixed ClassId — (`x/nft-transfer/keeper/packet.go`)

---

### Summary

`processReceivedPacket` unconditionally prepends the destination port/channel prefix to `data.ClassId` without first checking whether that prefix is already present. An attacker who controls a counterparty chain can craft a packet whose `ClassId` already embeds the destination chain's prefix, causing the receiving chain to double-prefix it, store a spurious `ClassTrace`, issue a new denom, and mint NFT vouchers under it — with no corresponding escrowed NFTs anywhere.

---

### Finding Description

**Root cause — `processReceivedPacket` (`x/nft-transfer/keeper/packet.go` lines 153–185):**

```
classPrefix     := types.GetClassPrefix(packet.GetDestPort(), packet.GetDestChannel())
prefixedClassID := classPrefix + data.ClassId          // ← blind concatenation
classTrace      := types.ParseClassTrace(prefixedClassID)
if !k.HasClassTrace(ctx, classTrace.Hash()) {
    k.SetClassTrace(ctx, classTrace)                   // ← spurious trace stored
}
voucherClassID := classTrace.IBCClassID()
if !k.nftKeeper.HasDenomID(ctx, voucherClassID) {
    k.nftKeeper.IssueDenom(...)                        // ← new denom created
}
k.nftKeeper.MintNFT(...)                               // ← vouchers minted
``` [1](#0-0) 

**`ParseClassTrace` splits naively on `/` and treats everything except the last segment as the path:**

```go
return ClassTrace{
    Path:        strings.Join(classSplit[:len(classSplit)-1], "/"),
    BaseClassId: classSplit[len(classSplit)-1],
}
``` [2](#0-1) 

**`NonFungibleTokenPacketData.ValidateBasic()` does not inspect `ClassId` content at all** — it only checks that the field is non-blank:

```go
if strings.TrimSpace(nftpd.ClassId) == "" {
    return newsdkerrors.Wrap(ErrInvalidClassID, "classId cannot be blank")
}
``` [3](#0-2) 

**`Validate()` on `ClassTrace` is never called inside `processReceivedPacket`**, so even if it could detect the anomaly, it is not invoked on the receive path. [4](#0-3) 

---

### Impact Explanation

**Concrete state delta:**

| Step | State change |
|---|---|
| Attacker sends packet with `ClassId = "nft-transfer/channel-0/baseClass"` | Packet committed on attacker's chain |
| Victim prepends its prefix → `"nft-transfer/channel-0/nft-transfer/channel-0/baseClass"` | Spurious `ClassTrace{Path:"nft-transfer/channel-0/nft-transfer/channel-0", BaseClassId:"baseClass"}` stored |
| Hash = sha256(`"nft-transfer/channel-0/nft-transfer/channel-0/baseClass"`) ≠ sha256(`"nft-transfer/channel-0/baseClass"`) | New denom `ibc/<spurious-hash>` issued |
| `MintNFT` called for each `tokenId` in the packet | Attacker-chosen receiver holds NFTs under a class that has no escrowed backing |

The legitimate single-hop hash `sha256("nft-transfer/channel-0/baseClass")` is never touched; the spurious double-hop hash creates an entirely new class. The attacker can specify any receiver address on the victim chain, so they can mint NFTs to themselves or any target.

---

### Likelihood Explanation

The precondition — controlling a counterparty chain — is realistic in the permissionless IBC model. Any party can spin up a Cosmos chain, open an IBC channel to the victim chain (channel handshake does not validate packet content), and relay crafted packets. No governance vote, no key compromise, and no social engineering is required beyond the channel being open.

---

### Recommendation

Before prepending the destination prefix, verify that `data.ClassId` does **not** already start with the destination prefix:

```go
destPrefix := types.GetClassPrefix(packet.GetDestPort(), packet.GetDestChannel())
if strings.HasPrefix(data.ClassId, destPrefix) {
    return types.ErrInvalidClassID.Wrapf("classId already contains destination prefix %s", destPrefix)
}
```

Additionally, call `classTrace.Validate()` after `ParseClassTrace` on the receive path to enforce structural correctness of the resulting path.

---

### Proof of Concept

Write a unit test in `x/nft-transfer/types/trace_test.go`:

```go
func TestDoublePrefix_SpuriousTrace(t *testing.T) {
    destPort, destChan := "nft-transfer", "channel-0"
    // Attacker-supplied ClassId already contains the destination prefix
    maliciousClassId := "nft-transfer/channel-0/baseClass"

    classPrefix     := types.GetClassPrefix(destPort, destChan)
    prefixedClassID := classPrefix + maliciousClassId
    // = "nft-transfer/channel-0/nft-transfer/channel-0/baseClass"

    spurious := types.ParseClassTrace(prefixedClassID)
    // Path = "nft-transfer/channel-0/nft-transfer/channel-0"  ← doubled prefix
    // BaseClassId = "baseClass"

    legitimate := types.ParseClassTrace("nft-transfer/channel-0/baseClass")
    // Path = "nft-transfer/channel-0"
    // BaseClassId = "baseClass"

    // Hashes differ → two distinct denoms would be created
    if spurious.Hash().String() == legitimate.Hash().String() {
        t.Fatal("expected different hashes, got same")
    }
    // Spurious path has 4 segments (two port/channel pairs) — passes Validate()
    if err := spurious.Validate(); err != nil {
        t.Fatalf("spurious trace unexpectedly invalid: %v", err)
    }
}
```

This test runs on an unmodified checkout, requires no chain state, and demonstrates that:
1. `ParseClassTrace` produces a doubled-prefix path.
2. The resulting hash differs from the legitimate single-hop hash.
3. `Validate()` accepts the spurious trace (4 identifiers = even, all valid port/channel names).

```go
func ParseClassTrace(rawClassID string) ClassTrace {
	classSplit := strings.Split(rawClassID, "/")

	if classSplit[0] == rawClassID {
		return ClassTrace{
			Path:        "",
			BaseClassId: rawClassID,
		}
	}

	return ClassTrace{
		Path:        strings.Join(classSplit[:len(classSplit)-1], "/"),
		BaseClassId: classSplit[len(classSplit)-1],
	}
}

```go
func (nftpd NonFungibleTokenPacketData) ValidateBasic() error {
	if strings.TrimSpace(nftpd.ClassId) == "" {
		return newsdkerrors.Wrap(ErrInvalidClassID, "classId cannot be blank")
	}

	if len(nftpd.TokenIds) == 0 {
		return newsdkerrors.Wrap(ErrInvalidTokenID, "tokenId cannot be blank")
	}

	if len(nftpd.TokenIds) != len(nftpd.TokenUris) {
		return newsdkerrors.Wrap(ErrInvalidPacket, "tokenIds and tokenUris lengths do not match")
	}

	if strings.TrimSpace(nftpd.Sender) == "" {
		return newsdkerrors.Wrap(sdkerrors.ErrInvalidAddress, "sender address cannot be blank")
	}

	// decode the sender address
	if _, err := sdk.AccAddressFromBech32(nftpd.Sender); err != nil {
		return newsdkerrors.Wrap(sdkerrors.ErrInvalidAddress, "invalid sender address")
	}

	if strings.TrimSpace(nftpd.Receiver) == "" {
		return newsdkerrors.Wrap(sdkerrors.ErrInvalidAddress, "receiver address cannot be blank")
	}

	// decode the receiver address
	if _, err := sdk.AccAddressFromBech32(nftpd.Receiver); err != nil {
		return newsdkerrors.Wrap(sdkerrors.ErrInvalidAddress, "invalid receiver address")
	}
	return nil
}
