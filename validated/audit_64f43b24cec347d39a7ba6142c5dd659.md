### Title
`gas_prices_to_hash` omits `l2_gas_consumed` and `next_l2_gas_price` from block hash, allowing fee-market parameters to be manipulated without changing the block hash — (`crates/starknet_api/src/block_hash/block_hash_calculator.rs`)

---

### Summary

`gas_prices_to_hash` deliberately excludes `l2_gas_consumed` and `next_l2_gas_price` from the block hash preimage. Both fields are present in `BlockHeaderWithoutHash`, stored in `StorageBlockHeader`, and transmitted over P2P, but they are absent from `PartialBlockHashComponents` and therefore from every call to `calculate_block_hash`. A malicious proposer can set either field to an arbitrary value and the resulting block hash is identical to a legitimate block, so consensus validators and syncing peers accept the block without detecting the manipulation. Because `next_l2_gas_price` directly seeds the EIP-1559-style fee market for the following block, the omission has a direct, quantifiable economic effect.

---

### Finding Description

`gas_prices_to_hash` is the sole function that converts gas-price data into the scalar(s) chained into the block hash. For Starknet ≥ 0.13.4 it hashes six values under the `STARKNET_GAS_PRICES0` domain:

```
l1_gas_price_wei, l1_gas_price_fri,
l1_data_gas_price_wei, l1_data_gas_price_fri,
l2_gas_price_wei, l2_gas_price_fri
```

The function carries an explicit TODO at line 416:

> `// TODO(Ayelet): add l2_gas_consumed, next_l2_gas_price after 0.14.0.`

The current production version in the test fixtures is **0.14.4** — past the stated threshold — yet neither field has been added.

`PartialBlockHashComponents` (the struct that feeds `calculate_block_hash`) is constructed from `BlockInfo` via `PartialBlockHashComponents::new`. That constructor copies `l1_gas_price`, `l1_data_gas_price`, and `l2_gas_price` but silently drops `l2_gas_consumed` and `next_l2_gas_price`:

```rust
pub fn new(block_info: &BlockInfo, header_commitments: BlockHeaderCommitments) -> Self {
    Self {
        header_commitments,
        block_number: block_info.block_number,
        l1_gas_price: block_info.gas_prices.l1_gas_price_per_token(),
        l1_data_gas_price: block_info.gas_prices.l1_data_gas_price_per_token(),
        l2_gas_price: block_info.gas_prices.l2_gas_price_per_token(),
        sequencer: SequencerContractAddress(block_info.sequencer_address),
        timestamp: block_info.block_timestamp,
        starknet_version: block_info.starknet_version,
    }
}
```

`StorageBlockHeader` and `BlockHeaderWithoutHash` both carry `l2_gas_consumed: GasAmount` and `next_l2_gas_price: GasPrice`. The protobuf converter serialises both fields over P2P. The `echonet/echo_center.py` script even passes them to the block-hash CLI. None of that matters for hash verification because `calculate_block_hash` never sees them.

The consequence is a broken canonicalization invariant: two block headers that differ only in `next_l2_gas_price` (or `l2_gas_consumed`) produce the **same** block hash. No existing validation path independently recomputes and checks `next_l2_gas_price` against the stored value during block acceptance or P2P sync.

---

### Impact Explanation

`next_l2_gas_price` is the sole input to the EIP-1559-style fee market for the next block (`calculate_next_l2_gas_price_for_fin`). A proposer that sets it to an arbitrarily low value forces all transactions in the next block to pay near-zero L2 fees; setting it arbitrarily high prices out legitimate users. Because the block hash does not commit to this value, consensus validators and syncing nodes cannot detect the manipulation through hash verification alone. This matches the **Critical** impact category: *Incorrect fee, gas, bouncer, resource accounting, refund, balance, or L1 gas price effect with economic impact.*

---

### Likelihood Explanation

In the current centralised deployment the proposer is trusted, so exploitation requires a compromised sequencer. As Starknet moves toward decentralised sequencing (the codebase already contains full BFT consensus infrastructure), any elected proposer can trigger this without any special privilege beyond being selected for a round. The omission is acknowledged in the source code and the fix threshold (0.14.0) has already been passed.

---

### Recommendation

Add `l2_gas_consumed` and `next_l2_gas_price` to `gas_prices_to_hash` under a `BlockHashVersion` gate (e.g., `>= V0_14_0`):

```rust
// For starknet version >= 0.14.0, extend the gas-prices hash with fee-market fields.
if block_hash_version >= &BlockHashVersion::V0_14_0 {
    hash_chain = hash_chain
        .chain(&l2_gas_consumed.0.into())
        .chain(&next_l2_gas_price.0.into());
}
```

`PartialBlockHashComponents` and its constructor must also accept these two fields so callers can supply them.

---

### Proof of Concept

1. A proposer builds a block with `next_l2_gas_price = GasPrice(0)` and `l2_gas_consumed = GasAmount(0)`.
2. `PartialBlockHashComponents::new` is called; neither field is copied into the struct.
3. `calculate_block_hash` calls `gas_prices_to_hash`, which hashes only the six price-per-token values — identical to a legitimate block.
4. The resulting `BlockHash` is indistinguishable from a legitimate block's hash.
5. Consensus validators call `PartialBlockHash::from_partial_block_hash_components` and reach agreement on this hash.
6. The next block's sequencer reads `next_l2_gas_price = 0` from storage and uses it as the base fee, charging zero L2 fees for all transactions in that block.

pub fn calculate_block_hash(
    partial_block_hash_components: &PartialBlockHashComponents,
    state_root: GlobalRoot,
    previous_block_hash: BlockHash,
) -> StarknetApiResult<BlockHash> {
    let block_hash_version: BlockHashVersion =
        partial_block_hash_components.starknet_version.try_into()?;
    let block_commitments = &partial_block_hash_components.header_commitments;
    Ok(BlockHash(
        HashChain::new()
            .chain(&block_hash_version.clone().into())
            .chain(&partial_block_hash_components.block_number.0.into())
            .chain(&state_root.0)
            .chain(&partial_block_hash_components.sequencer.0)
            .chain(&partial_block_hash_components.timestamp.0.into())
            .chain(&block_commitments.concatenated_counts)
            .chain(&block_commitments.state_diff_commitment.0.0)
            .chain(&block_commitments.transaction_commitment.0)
            .chain(&block_commitments.event_commitment.0)
            .chain(&block_commitments.receipt_commitment.0)
            .chain_iter(
                gas_prices_to_hash(
                    &partial_block_hash_components.l1_gas_price,
                    &partial_block_hash_components.l1_data_gas_price,
                    &partial_block_hash_components.l2_gas_price,
                    &block_hash_version,
                )
                .iter(),
            )
            .chain(
                &Felt::try_from(&partial_block_hash_components.starknet_version)
                    .expect("Expect ASCII version"),
            )
            .chain(&Felt::ZERO)
            .chain(&previous_block_hash.0)
            .get_poseidon_hash(),
    ))
}

// TODO(Ayelet): add l2_gas_consumed, next_l2_gas_price after 0.14.0.
pub fn gas_prices_to_hash(
    l1_gas_price: &GasPricePerToken,
    l1_data_gas_price: &GasPricePerToken,
    l2_gas_price: &GasPricePerToken,
    block_hash_version: &BlockHashVersion,
) -> Vec<Felt> {
    if block_hash_version >= &BlockHashVersion::V0_13_4 {
        vec![
            HashChain::new()
                .chain(&STARKNET_GAS_PRICES0)
                .chain(&l1_gas_price.price_in_wei.0.into())
                .chain(&l1_gas_price.price_in_fri.0.into())
                .chain(&l1_data_gas_price.price_in_wei.0.into())
                .chain(&l1_data_gas_price.price_in_fri.0.into())
                .chain(&l2_gas_price.price_in_wei.0.into())
                .chain(&l2_gas_price.price_in_fri.0.into())
                .get_poseidon_hash(),
        ]
    } else {
        vec![
            l1_gas_price.price_in_wei.0.into(),
            l1_gas_price.price_in_fri.0.into(),
            l1_data_gas_price.price_in_wei.0.into(),
            l1_data_gas_price.price_in_fri.0.into(),
        ]
    }
}
