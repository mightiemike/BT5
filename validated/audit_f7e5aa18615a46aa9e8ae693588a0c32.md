Audit Report

## Title
`SwapAllowlistExtension` allowlist bypass via router: `sender` is the router address, not the actual user - (File: metric-periphery/contracts/extensions/SwapAllowlistExtension.sol)

## Summary
`MetricOmmPool.swap()` passes `msg.sender` as the `sender` argument to `_beforeSwap()`, which forwards it to `SwapAllowlistExtension.beforeSwap()`. When a user swaps through `MetricOmmSimpleRouter`, `msg.sender` inside `pool.swap()` is the router contract, not the actual user. The allowlist check therefore gates the router address rather than the real swapper, making the allowlist either completely bypassable (if the router is allowlisted) or blocking all legitimate allowlisted users from using the router.

## Finding Description

**Root cause — `msg.sender` forwarded as `sender`:**

In `MetricOmmPool.swap()` (lines 230–240), the pool calls `_beforeSwap(msg.sender, ...)`. When the user enters through `MetricOmmSimpleRouter.exactInputSingle()` (line 72–80), the router calls `IMetricOmmPoolActions(params.pool).swap(...)` directly, so `msg.sender` inside `pool.swap()` is the router contract address.

`ExtensionCalling._beforeSwap()` (lines 160–176) encodes this router address as the `sender` argument and passes it to every configured extension.

`SwapAllowlistExtension.beforeSwap()` (line 37) then checks:
```solidity
if (!allowAllSwappers[msg.sender] && !allowedSwapper[msg.sender][sender]) {
    revert IMetricOmmPoolActions.NotAllowedToSwap();
}
```
Here `msg.sender` is the pool and `sender` is the router — not the actual user.

**Exploit path A — bypass (router allowlisted):**
1. Pool admin deploys pool with `SwapAllowlistExtension`, intending to allow only specific users.
2. Admin allowlists the router address so that allowlisted users can swap through it: `setAllowedToSwap(pool, router, true)`.
3. Any unprivileged user calls `MetricOmmSimpleRouter.exactInputSingle()`.
4. Router calls `pool.swap()` → `sender = router` → `allowedSwapper[pool][router] == true` → check passes.
5. Non-allowlisted user successfully swaps, bypassing the allowlist entirely.

**Exploit path B — DoS (router not allowlisted):**
1. Pool admin allowlists specific EOAs but not the router.
2. An allowlisted user calls the router.
3. Extension sees `sender = router` → `allowedSwapper[pool][router] == false` → reverts.
4. Allowlisted users cannot use the router at all.

**Existing guards are insufficient:** The `onlyPool` modifier on the extension only verifies the caller is a known pool; it does not recover the original user. The router stores the original `msg.sender` only in transient storage for the payment callback (`_setNextCallbackContext(pool, CALLBACK_MODE_JUST_PAY, msg.sender, ...)`) and never forwards it to the pool's `swap()` call as a verifiable identity.

## Impact Explanation
The `SwapAllowlistExtension` is a core security control for permissioned pools. Its bypass allows any unprivileged user to trade in a pool that was explicitly configured to restrict access. This constitutes broken core pool functionality (the allowlist gate) and an admin-boundary break reachable by an unprivileged path (routing through the public router). In pools where the allowlist is used to enforce KYC, regulatory compliance, or LP-protection policies, the bypass directly undermines the pool's security model.

## Likelihood Explanation
The bypass is trivially reachable: any user with access to the public `MetricOmmSimpleRouter` can exploit it. The only precondition is that the pool admin has allowlisted the router address (a natural and expected configuration for any pool that wants to support router-mediated swaps). The condition is repeatable on every block with no special privileges required.

## Recommendation
Pass the original user identity through the call chain. One approach: add an optional `swapper` parameter to `pool.swap()` that the router populates with `msg.sender` before calling the pool, and have the pool forward this verified identity to extensions. Alternatively, `SwapAllowlistExtension.beforeSwap()` should not rely on the `sender` argument for identity when the caller is a known router; instead, the router should attest the real user via a signed message or the pool should read the original initiator from transient storage set by the router before the `pool.swap()` call.

    tokenIn: token0,
    deadline: block.timestamp,
    extensionData: ""

// pool.swap() sees msg.sender = router
// SwapAllowlistExtension checks allowedSwapper[pool][router] == true → passes
// Attacker swaps successfully despite not being on the allowlist
