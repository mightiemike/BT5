Audit Report

## Title
`SwapAllowlistExtension` Swap Guard Bypassed via `MetricOmmSimpleRouter` — (`metric-periphery/contracts/extensions/SwapAllowlistExtension.sol`)

## Summary
`SwapAllowlistExtension.beforeSwap` gates swaps by checking the `sender` argument, which `MetricOmmPool.swap` sets to `msg.sender` — the direct caller of `pool.swap()`. When a user routes through `MetricOmmSimpleRouter`, the router becomes `msg.sender`, so the extension checks whether the **router** is allowlisted rather than the actual end user. Because the router is a fully public, permissionless contract, any non-allowlisted user can bypass the swap guard by routing through it, rendering the allowlist completely ineffective for the router path.

## Finding Description
**Root cause:** `MetricOmmPool.swap` forwards `msg.sender` verbatim as `sender` to `_beforeSwap`:

```solidity
// MetricOmmPool.sol L230-240
_beforeSwap(
  msg.sender,   // ← direct caller; router when routed
  recipient,
  ...
);
```

`ExtensionCalling._beforeSwap` passes this `sender` unchanged to the extension:

```solidity
// ExtensionCalling.sol L160-176
_callExtensionsInOrder(
  BEFORE_SWAP_ORDER,
  abi.encodeCall(IMetricOmmExtensions.beforeSwap, (sender, ...))
);
```

`SwapAllowlistExtension.beforeSwap` then checks `allowedSwapper[msg.sender][sender]` where `msg.sender` = pool and `sender` = router:

```solidity
// SwapAllowlistExtension.sol L37-39
if (!allowAllSwappers[msg.sender] && !allowedSwapper[msg.sender][sender]) {
  revert IMetricOmmPoolActions.NotAllowedToSwap();
}
```

`MetricOmmSimpleRouter.exactInputSingle` calls `pool.swap()` without encoding or forwarding the original user's identity:

```solidity
// MetricOmmSimpleRouter.sol L72-80
(int128 amount0Delta, int128 amount1Delta) = IMetricOmmPoolActions(params.pool)
  .swap(
    params.recipient,
    params.zeroForOne,
    MetricOmmSwapInputs.asAmountSpecifiedIn(params.amountIn),
    priceLimitX64,
    "",
    params.extensionData   // user-supplied; not authenticated
  );
```

**Exploit flow:**
1. Pool admin configures `SwapAllowlistExtension` to restrict swaps to KYC-verified addresses.
2. Admin allowlists Alice so she can trade; admin also allowlists the router so Alice can use it.
3. Bob (non-allowlisted) calls `router.exactInputSingle(...)` targeting the restricted pool.
4. Router calls `pool.swap(...)` with `msg.sender = router`.
5. `beforeSwap` receives `sender = router` (allowlisted) → check passes.
6. Bob's swap executes against restricted LP liquidity.

**Structural dilemma — no safe configuration exists:**

| Admin choice | Effect |
|---|---|
| Do **not** allowlist the router | Allowlisted users cannot use the router; must call `pool.swap()` directly |
| **Allowlist the router** | Every user on the network bypasses the allowlist via the router |

**Existing guards reviewed and shown insufficient:** The extension has no mechanism to distinguish the original user from the router. The `extensionData` field passed by the router is user-supplied (`params.extensionData`) and unauthenticated — a malicious user could encode any address into it, and the extension does not read it at all.

## Impact Explanation
A pool configured with `SwapAllowlistExtension` to restrict swaps to specific addresses (e.g., KYC-verified counterparties, whitelisted institutions) can be freely accessed by any unprivileged user via the public router. LPs deposited under the assumption that only allowlisted counterparties could trade against them are exposed to unrestricted swaps at live oracle prices. This constitutes **broken core pool functionality** (the swap allowlist — a production extension explicitly designed to gate the swap path — is rendered completely ineffective for the router path) and **direct loss of LP assets** (non-allowlisted users drain liquidity at oracle prices against restricted pools). This meets the Sherlock High/Critical threshold: the guard is structurally broken with no safe configuration available.

## Likelihood Explanation
- `MetricOmmSimpleRouter` is a public, deployed periphery contract with no access control on `exactInputSingle`, `exactInput`, `exactOutputSingle`, or `exactOutput`.
- Any user can call these functions targeting any pool.
- The pool admin **must** allowlist the router to enable router-mediated swaps for legitimate users; once done, the bypass is open to everyone on the network.
- No special privileges, flash loans, or complex setup are required — a single `exactInputSingle` call suffices.
- The bypass is repeatable indefinitely.

## Recommendation
The extension must gate on the **actual end user**, not the direct pool caller. Two viable approaches:

1. **Require the router to forward user identity via `extensionData`** — update `SwapAllowlistExtension.beforeSwap` to decode an authenticated user address from `extensionData` when `sender` is a known router, and update `MetricOmmSimpleRouter` to always encode `msg.sender` into the extension payload it forwards to the pool.

2. **Check `recipient` instead of `sender`** — for single-hop swaps the recipient is the economic beneficiary; however this breaks for multi-hop where intermediate recipients are the router itself.

The most robust fix: update `MetricOmmSimpleRouter` to always encode `msg.sender` into `extensionData` before calling `pool.swap`, and update `SwapAllowlistExtension.beforeSwap` to decode and authenticate the user address from `extensionData` when `sender` is a registered router address.

## Proof of Concept
```solidity
// Pool is configured with SwapAllowlistExtension.
// Admin allowlists Alice (0xAlice) but NOT Bob (0xBob).
// Admin also allowlists the router so Alice can use it.

// Bob (non-allowlisted) calls the router directly:
router.exactInputSingle(
    IMetricOmmSimpleRouter.ExactInputSingleParams({
        pool:             address(pool),
        tokenIn:          address(token0),
        tokenOut:         address(token1),
        zeroForOne:       true,
        amountIn:         1_000e18,
        amountOutMinimum: 0,
        recipient:        address(bob),
        deadline:         block.timestamp + 1,
        priceLimitX64:    0,
        extensionData:    ""
    })
);
// pool.swap() called with msg.sender = router
// beforeSwap receives sender = router (allowlisted) → passes
// Bob's swap executes against restricted LP liquidity
