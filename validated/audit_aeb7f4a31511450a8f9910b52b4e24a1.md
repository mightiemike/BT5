Audit Report

## Title
SwapAllowlistExtension Checks Router Address Instead of End User, Allowing Any Caller to Bypass the Swap Allowlist â€” (`metric-periphery/contracts/extensions/SwapAllowlistExtension.sol`)

## Summary
`SwapAllowlistExtension.beforeSwap` gates swaps by checking the `sender` argument forwarded from the pool. When a swap is routed through `MetricOmmSimpleRouter`, the pool's `msg.sender` is the router contract, so the extension always evaluates the router's address rather than the actual end user. Any pool admin who allowlists the router to permit their KYC'd users to swap via the canonical router simultaneously opens the allowlist to every address on the network.

## Finding Description
In `MetricOmmPool.swap`, the pool passes its own `msg.sender` as the `sender` argument to `_beforeSwap`: [1](#0-0) 

`ExtensionCalling._beforeSwap` encodes that `sender` value directly into the extension call: [2](#0-1) 

`SwapAllowlistExtension.beforeSwap` then checks `allowedSwapper[msg.sender][sender]`, where `msg.sender` is the pool (correct key) and `sender` is whatever called the pool â€” the router: [3](#0-2) 

In `MetricOmmSimpleRouter.exactInputSingle`, the pool's `swap` is called with no originator argument; the pool sees `msg.sender = router`: [4](#0-3) 

The same applies to `exactInput` (all hops), `exactOutputSingle`, and `exactOutput`. For multi-hop `exactInput`, intermediate hops use `address(this)` (the router itself) as the payer, so every hop's `sender` seen by the extension is the router: [5](#0-4) 

For the allowlist to permit any allowlisted user to swap via the router, the admin must add the router to the allowlist. Once `allowedSwapper[pool][router] = true`, the check passes for every caller regardless of their own allowlist status, because the extension never sees the end user's address.

`DepositAllowlistExtension` does not share this flaw because it gates on `owner` (the LP beneficiary), which is an explicit caller-supplied argument independent of the router address: [6](#0-5) 

## Impact Explanation
A pool deployer who restricts swaps to a curated set of addresses (e.g., KYC'd wallets) deploys with `SwapAllowlistExtension`. To allow those users to swap via the canonical router, the admin must add the router to the allowlist. The moment the router is allowlisted, the guard is fully open: any address can call `exactInputSingle` or `exactInput` on the router and the extension passes because it sees the allowlisted router address. Non-permitted actors can execute swaps on curated pools, directly violating the access-control invariant the extension is designed to enforce. This constitutes a complete bypass of a core access-control mechanism with direct fund-impacting consequences.

## Likelihood Explanation
The router is the primary user-facing entry point. Any pool admin who deploys `SwapAllowlistExtension` and wants their allowlisted users to use the router will inevitably add the router to the allowlist, triggering the bypass. The attacker requires no special privilege â€” a single call to `exactInputSingle` suffices. The condition is met in the normal, expected operational setup of the protocol.

## Recommendation
Pass the original end-user address through the swap path so the extension can gate on it. Two options:

1. **Pool-level**: Add an explicit `originator` parameter to `swap()` and forward it as `sender` to extensions, with the router populating it as `msg.sender` before calling the pool.
2. **Extension-level**: Require the router to encode the real user address in `extensionData` and decode it in `SwapAllowlistExtension.beforeSwap`, falling back to `sender` for direct pool calls.

## Proof of Concept

```
Setup:
  - Deploy pool with SwapAllowlistExtension
  - Admin calls setAllowedToSwap(pool, alice, true)   // alice is KYC'd
  - Admin calls setAllowedToSwap(pool, router, true)  // required so alice can use the router

Attack:
  - bob (not allowlisted) calls router.exactInputSingle({pool: pool, ...})
  - Router calls pool.swap(...) â†’ pool's msg.sender = router
  - Pool calls _beforeSwap(sender=router, ...)
  - Extension checks allowedSwapper[pool][router] â†’ true
  - Swap executes for bob despite bob not being on the allowlist

Result:
  - SwapAllowlistExtension is fully bypassed for any caller via the router
```
