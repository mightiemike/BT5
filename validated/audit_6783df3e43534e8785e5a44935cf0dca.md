Audit Report

## Title
`SwapAllowlistExtension` Allowlist Bypassed via `MetricOmmSimpleRouter` — (File: `metric-periphery/contracts/extensions/SwapAllowlistExtension.sol`)

## Summary
`SwapAllowlistExtension.beforeSwap` gates swaps by checking the `sender` argument forwarded by the pool, which is always `msg.sender` of `pool.swap()` — the router address when a user routes through `MetricOmmSimpleRouter`. If the router is allowlisted so that legitimate users can trade, every non-allowlisted address can also trade by calling through the router, fully defeating the curated pool's access control.

## Finding Description
In `MetricOmmPool.swap()`, `_beforeSwap` is called with `msg.sender` as the `sender` argument:

```solidity
// metric-core/contracts/MetricOmmPool.sol L230-240
_beforeSwap(
  msg.sender,   // the router when routed
  recipient, ...
);
```

`ExtensionCalling._beforeSwap` forwards this verbatim to every configured extension:

```solidity
// metric-core/contracts/ExtensionCalling.sol L160-176
_callExtensionsInOrder(
  BEFORE_SWAP_ORDER,
  abi.encodeCall(IMetricOmmExtensions.beforeSwap, (sender, recipient, ...))
);
```

`SwapAllowlistExtension.beforeSwap` then checks that forwarded `sender`:

```solidity
// metric-periphery/contracts/extensions/SwapAllowlistExtension.sol L37-39
if (!allowAllSwappers[msg.sender] && !allowedSwapper[msg.sender][sender]) {
  revert IMetricOmmPoolActions.NotAllowedToSwap();
}
```

`msg.sender` here is the pool; `sender` is the router. Meanwhile, in `MetricOmmSimpleRouter.exactInputSingle`, the actual user's address is stored only in transient callback context and is never passed to `pool.swap()`:

```solidity
// metric-periphery/contracts/MetricOmmSimpleRouter.sol L71-80
_setNextCallbackContext(params.pool, CALLBACK_MODE_JUST_PAY, msg.sender, params.tokenIn);
IMetricOmmPoolActions(params.pool).swap(
  params.recipient, params.zeroForOne, ..., params.extensionData
);
```

The real user's address (`msg.sender` inside the router) is stored only for the payment callback, never forwarded to the pool or extension. The extension therefore checks the router's allowlist status, not the user's.

## Impact Explanation
A curated pool deploying `SwapAllowlistExtension` to restrict trading to KYC'd or otherwise vetted addresses loses that restriction entirely for any user who calls through `MetricOmmSimpleRouter`. The attacker receives pool output tokens and the pool receives input tokens — a direct, fund-impacting bypass of the configured access control. This constitutes an admin-boundary break: the pool admin's allowlist enforcement is circumvented by an unprivileged path available to any address.

## Likelihood Explanation
`MetricOmmSimpleRouter` is the canonical, publicly deployed periphery contract callable by any address without permission. No special role, privileged setup, or non-standard token is required. The bypass is reachable on every swap on every allowlisted pool that also has the router allowlisted, which is the only configuration that permits legitimate users to use the router at all.

## Recommendation
The extension must gate the economic actor, not the intermediary. The most robust fix is to have the router encode `msg.sender` into `extensionData`, and have the extension decode and check it when `msg.sender` (the pool) recognizes the caller as a trusted, factory-verified router. This requires a factory-maintained router registry so the extension can confirm the `extensionData` origin is trustworthy and cannot be forged by an arbitrary caller passing crafted `extensionData` directly to `pool.swap()`.

## Proof of Concept
```
Setup:
  - Deploy pool with SwapAllowlistExtension in BEFORE_SWAP_ORDER
  - Pool admin calls setAllowedToSwap(pool, router, true)
    // router must be allowed for any legitimate user to trade
  - Pool admin does NOT call setAllowedToSwap(pool, attacker, true)

Attack:
  - attacker calls MetricOmmSimpleRouter.exactInputSingle({
        pool: allowlistedPool,
        recipient: attacker,
        zeroForOne: true,
        amountIn: X,
        ...
    })

Trace:
  router.exactInputSingle()
    → pool.swap(recipient=attacker, ...) [msg.sender = router]
      → _beforeSwap(sender=router, ...)
        → SwapAllowlistExtension.beforeSwap(sender=router, ...)
          → allowedSwapper[pool][router] == true  ✓  (no revert)
      → swap executes, attacker receives output tokens

Result:
  attacker bypasses the allowlist and receives swap output.
  The check passed because the router is allowlisted,
  not because the attacker is allowlisted.
```
