### Title
Non-Bond-Denom Tokens Sent to `rewards_pool` Module Account Are Permanently Irrecoverable - (`app/app.go`)

### Summary
The `tieredrewards` `rewards_pool` module account is explicitly whitelisted to receive external token sends from any user. However, the module only ever disburses `bondDenom` tokens. Any other token denomination sent to the pool — accidentally, as an airdrop, or by mistake — is permanently locked with no on-chain recovery path.

### Finding Description
`app/app.go` defines a whitelist of module accounts that bypass the default Cosmos SDK block on external sends:

```go
moduleAccsAllowedToReceiveExternalFunds = map[string]bool{
    tieredrewardstypes.RewardsPoolName: true,
    tieredrewardstypes.ModuleName:      true,
}
```

This means any user can submit a standard `MsgSend` to the `rewards_pool` module address with any token denomination, and the bank module will accept it.

The module's two outbound payment paths are both hard-coded to `bondDenom` only:

1. **Bonus rewards** (`processEventsAndClaimBonus`): `SendCoinsFromModuleToAccount(ctx, types.RewardsPoolName, ownerAddr, bonusCoins)` where `bonusCoins` is always `sdk.NewCoin(bondDenom, totalBonus)`.
2. **Base rewards top-up** (`topUpBaseRewards`): `SendCoinsFromModuleToModule(ctx, types.RewardsPoolName, distributiontypes.ModuleName, sdk.NewCoins(sdk.NewCoin(bondDenom, topUpAmount)))`.

There is no governance message, admin message, or any other on-chain path to sweep non-`bondDenom` tokens out of the `rewards_pool` module account. The `RewardsPoolBalances` query exposes all balances (including non-bond-denom), confirming the module is aware of multi-denom balances, but provides no mechanism to act on them.

### Impact Explanation
Any non-`bondDenom` token sent to the `rewards_pool` module address is permanently irrecoverable. This includes:
- Tokens accidentally sent by users who misidentify the address
- IBC-transferred tokens sent to the pool address
- Airdrops targeting the pool address

The corrupted value is the balance of non-`bondDenom` tokens at `authtypes.NewModuleAddress(tieredrewardstypes.RewardsPoolName)`. These tokens are provably unspendable: no message handler calls `SendCoinsFromModuleToAccount` or `SendCoinsFromModuleToModule` for any denom other than `bondDenom`, and no governance proposal type exists to drain them.

### Likelihood Explanation
The `rewards_pool` address is publicly queryable via the `RewardsPoolBalances` gRPC endpoint. The whitelist in `BlockedAddrs()` explicitly permits `MsgSend` to this address. A user funding the pool with the wrong token denomination, or a protocol sending an IBC token to the pool address, is a realistic scenario. The `tieredrewards` module is actively used for staking rewards, making the pool address a known, funded target.

### Recommendation
Add a governance-gated sweep message (e.g., `MsgSweepRewardsPool`) that allows the `gov` module authority to transfer a specified list of non-`bondDenom` tokens from the `rewards_pool` module account to a designated recipient (e.g., the community pool or a governance-specified address). This mirrors the `sweep()` function recommended in the external report. Alternatively, restrict the `moduleAccsAllowedToReceiveExternalFunds` entry for `RewardsPoolName` to `bondDenom` only at the `AnteHandler` level.

### Proof of Concept
1. Query the `rewards_pool` address: `chain-maind query tieredrewards rewards-pool-balances`
2. Send a non-bond-denom token to it: `chain-maind tx bank send <user> <rewards_pool_addr> 1000ibc/XXXX`
3. The `MsgSend` succeeds (address is whitelisted in `BlockedAddrs()`).
4. Query balances again: the IBC token appears in the pool balance.
5. Attempt any tieredrewards message: none disburse the IBC token.
6. Attempt any governance proposal: no `MsgSweepRewardsPool` type exists.

// BlockedAddrs returns all the app's module account addresses that are not
// allowed to receive external tokens.
func (app *ChainApp) BlockedAddrs() map[string]bool {
	blockedAddrs := make(map[string]bool)
	for acc := range maccPerms {
		blockedAddrs[authtypes.NewModuleAddress(acc).String()] = !moduleAccsAllowedToReceiveExternalFunds[acc]
	}

	return blockedAddrs
