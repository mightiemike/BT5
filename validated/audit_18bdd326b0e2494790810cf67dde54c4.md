### Title
Silent `ValidResourceBounds::L1Gas` Coercion in Protobuf Deserialization Produces Wrong Transaction Hash for V3 Transactions with Zero L2/Data-Gas Bounds — (`crates/apollo_protobuf/src/converters/transaction.rs`)

---

### Summary

The protobuf deserializer for `ValidResourceBounds` silently coerces a V3 `AllResources` transaction (whose `l2_gas` and `l1_data_gas` are both zero) into the `L1Gas` variant. Because `get_tip_resource_bounds_hash` produces a structurally different hash preimage for `L1Gas` (2 resource felts) versus `AllResources` (3 resource felts), the transaction hash computed after a P2P round-trip diverges from the hash computed at submission time. A syncing node therefore rejects the transaction as having an invalid hash, even though it was accepted and committed by the sequencer.

---

### Finding Description

**Step 1 – Hash preimage depends on variant, not on numeric values.**

`get_tip_resource_bounds_hash` in `crates/starknet_api/src/transaction_hash.rs` builds the Poseidon preimage differently for the two variants:

```
L1Gas(l1_gas)          → Poseidon(tip, packed(L1_GAS), packed(L2_GAS=0))          // 3 elements
AllResources(…)        → Poseidon(tip, packed(L1_GAS), packed(L2_GAS=0), packed(L1_DATA_GAS=0))  // 4 elements
```

Even when every numeric field is identical, the two variants produce different Poseidon outputs. [1](#0-0) 

**Step 2 – Protobuf deserializer silently coerces `AllResources` → `L1Gas`.**

`TryFrom<protobuf::ResourceBounds> for ValidResourceBounds` (used in the P2P sync path for `Transaction`) applies `unwrap_or_default()` to the optional `l1_data_gas` field and then checks whether both `l2_gas` and `l1_data_gas` are zero. If they are, it returns `ValidResourceBounds::L1Gas`, discarding the `AllResources` variant that was originally signed:

```rust
let l1_data_gas = value.l1_data_gas.unwrap_or_default();   // silently defaults to zero
...
Ok(if l1_data_gas.is_zero() && l2_gas.is_zero() {
    ValidResourceBounds::L1Gas(l1_gas)                      // ← wrong variant
} else {
    ValidResourceBounds::AllResources(...)
})
``` [2](#0-1) 

**Step 3 – The consensus/gateway path always uses `AllResources`.**

`InternalRpcInvokeTransactionV3` stores `resource_bounds: AllResourceBounds` and its `InvokeTransactionV3Trait` implementation wraps it as `ValidResourceBounds::AllResources(self.resource_bounds)`. The hash stored in `InternalRpcTransaction.tx_hash` is therefore always computed with the 4-element preimage. [3](#0-2) 

The `RpcTransaction` protobuf converter (`TryFrom<protobuf::ResourceBounds> for AllResourceBounds`) requires all three fields to be present and never applies the `is_zero()` coercion, so the consensus path is unaffected. [4](#0-3) 

**Step 4 – Divergence during P2P sync.**

When a syncing node receives a historical block over P2P, it deserializes `Transaction::Invoke(InvokeTransaction::V3(...))` through the `ValidResourceBounds` path. For any V3 transaction whose `l2_gas` and `l1_data_gas` are both zero, the deserializer produces `L1Gas`, and `validate_transaction_hash` recomputes the hash with the 3-element preimage. This hash does not match the 4-element hash stored in the block, so the syncing node rejects the transaction as invalid. [5](#0-4) 

**Step 5 – Gateway admission conditions.**

The gateway's stateless validator rejects a transaction only if `max_possible_fee == 0` or if `l2_gas.max_price_per_unit < min_gas_price`. A transaction with `AllResources { l1_gas: {A>0, P>0}, l2_gas: {0,0}, l1_data_gas: {0,0} }` satisfies `max_possible_fee = A*P > 0`. If `min_gas_price = 0` (a valid configuration), the price check also passes, so the transaction is accepted and committed with hash H₁ (4-element preimage). After P2P sync it is re-hashed as H₂ (3-element preimage), H₁ ≠ H₂. [6](#0-5) 

---

### Impact Explanation

A syncing node computes the wrong transaction hash for any committed V3 transaction whose `l2_gas` and `l1_data_gas` resource bounds are both zero. `validate_transaction_hash` returns `false`, causing the node to treat a valid, sequencer-committed transaction as having an invalid hash. This breaks chain sync for affected blocks and causes the syncing node to diverge from the canonical chain. The impact matches **"High. Transaction conversion or signature/hash logic binds the wrong signer, hash, type, or executable payload."**

---

### Likelihood Explanation

The trigger requires `min_gas_price = 0` in the gateway configuration (so that a transaction with zero `l2_gas.max_price_per_unit` is admitted) and a non-zero `l1_gas` bound (so that `max_possible_fee > 0`). This is a valid configuration for private or test networks and is not prevented by any hard-coded constant. The coercion in the protobuf converter is unconditional and fires on every P2P sync of such a transaction.

---

### Recommendation

1. **Remove the `L1Gas`/`AllResources` coercion from the P2P sync deserializer.** The variant should be preserved as serialized, not inferred from zero-value checks. If backward compatibility with pre-0.13.3 peers is required, use an explicit version field or a separate protobuf message type.

2. **Alternatively, canonicalize `ValidResourceBounds` before hashing.** If `l2_gas` and `l1_data_gas` are both zero, always treat the transaction as `L1Gas` at hash time *and* at submission time, so the hash is consistent across all paths.

3. **Add a gateway check** that rejects `AllResources` transactions where both `l2_gas` and `l1_data_gas` are zero, forcing submitters to use the `L1Gas` variant explicitly and eliminating the ambiguity.

---

### Proof of Concept

```
1. Configure the sequencer with min_gas_price = 0.

2. Submit an Invoke V3 transaction with:
     resource_bounds = AllResources {
         l1_gas:      { max_amount: 1000, max_price_per_unit: 1 },
         l2_gas:      { max_amount: 0,    max_price_per_unit: 0 },
         l1_data_gas: { max_amount: 0,    max_price_per_unit: 0 },
     }

3. Gateway accepts it. TransactionConverter::convert_rpc_tx_to_internal calls
   InternalRpcInvokeTransactionV3::calculate_transaction_hash, which calls
   get_tip_resource_bounds_hash with ValidResourceBounds::AllResources.
   Hash H1 = Poseidon(INVOKE, version, sender, Poseidon(tip, L1_packed, L2_packed(0), DATA_packed(0)), ...).

4. The transaction is included in a block and committed.

5. A syncing node receives the block over P2P. The protobuf message has
   l1_data_gas = Some({max_amount:0, max_price_per_unit:0}).

6. TryFrom<protobuf::ResourceBounds> for ValidResourceBounds:
     l1_data_gas = Some({0,0}).unwrap_or_default() = {0,0}  → is_zero() = true
     l2_gas = {0,0}                                          → is_zero() = true
     → returns ValidResourceBounds::L1Gas(l1_gas)            ← wrong variant

7. validate_transaction_hash recomputes:
   H2 = Poseidon(INVOKE, version, sender, Poseidon(tip, L1_packed, L2_packed(0)), ...).
   H2 ≠ H1 → returns false → syncing node rejects the transaction.
```

pub fn get_tip_resource_bounds_hash(
    resource_bounds: &ValidResourceBounds,
    tip: &Tip,
) -> Result<Felt, StarknetApiError> {
    let l1_resource_bounds = resource_bounds.get_l1_bounds();
    let l2_resource_bounds = resource_bounds.get_l2_bounds();

    // L1 and L2 gas bounds always exist.
    // Old V3 txs always have L2 gas bounds of zero, but they exist.
    let mut resource_felts = vec![
        get_concat_resource(&l1_resource_bounds, L1_GAS)?,
        get_concat_resource(&l2_resource_bounds, L2_GAS)?,
    ];

    // For new V3 txs, need to also hash the data gas bounds.
    resource_felts.extend(match resource_bounds {
        ValidResourceBounds::L1Gas(_) => vec![],
        ValidResourceBounds::AllResources(all_resources) => {
            vec![get_concat_resource(&all_resources.l1_data_gas, L1_DATA_GAS)?]
        }
    });

    Ok(HashChain::new().chain(&tip.0.into()).chain_iter(resource_felts.iter()).get_poseidon_hash())
}

impl TryFrom<protobuf::ResourceBounds> for ValidResourceBounds {
    type Error = ProtobufConversionError;
    fn try_from(value: protobuf::ResourceBounds) -> Result<Self, Self::Error> {
        let Some(l1_gas) = value.l1_gas else {
            return Err(missing("ResourceBounds::l1_gas"));
        };
        let Some(l2_gas) = value.l2_gas else {
            return Err(missing("ResourceBounds::l2_gas"));
        };
        // TODO(Shahak): Assert data gas is not none once we remove support for 0.13.2.
        let l1_data_gas = value.l1_data_gas.unwrap_or_default();
        let l1_gas: ResourceBounds = l1_gas.try_into()?;
        let l2_gas: ResourceBounds = l2_gas.try_into()?;
        let l1_data_gas: ResourceBounds = l1_data_gas.try_into()?;
        Ok(if l1_data_gas.is_zero() && l2_gas.is_zero() {
            ValidResourceBounds::L1Gas(l1_gas)
        } else {
            ValidResourceBounds::AllResources(AllResourceBounds { l1_gas, l2_gas, l1_data_gas })
        })
    }
