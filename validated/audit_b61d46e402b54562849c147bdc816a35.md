### Title
Division-by-Zero in `socializeSubaccount` Permanently Blocks Liquidation Finalization When Last Position Holder Has Negative vQuoteBalance — (`core/contracts/PerpEngine.sol`)

---

### Summary

`PerpEngine.socializeSubaccount` unconditionally divides by `state.openInterest` when socializing a negative `vQuoteBalance` that exceeds insurance. When the liquidated subaccount is the last position holder in a perp market, `state.openInterest` is zero at the time of the call, causing a division-by-zero revert. This permanently blocks the finalization step of the liquidation, leaving the subaccount in an unresolvable state.

---

### Finding Description

The `socializeSubaccount` function in `PerpEngine.sol` is called by `ClearinghouseLiq._finalizeSubaccount` as the last step of liquidating an insolvent subaccount. Its purpose is to spread any residual negative `vQuoteBalance` (after insurance is exhausted) across all remaining open-interest holders via a per-share funding adjustment.

The critical path is:

```solidity
// PerpEngine.sol L164-171
if (balance.vQuoteBalance < 0) {
    // socialize across all other participants
    int128 fundingPerShare = -balance.vQuoteBalance.div(
        state.openInterest   // <-- can be 0
    );
    state.cumulativeFundingLongX18 += fundingPerShare;
    state.cumulativeFundingShortX18 -= fundingPerShare;
    balance.vQuoteBalance = 0;
}
``` [1](#0-0) 

The `state.openInterest` returned by `getStateAndBalance` for a subaccount whose `balance.amount` is already `0` is identical to the stored global `state.openInterest`, because `_updateBalance` subtracts and re-adds `balance.amount.abs() == 0`:

```solidity
// PerpEngineState.sol L30-51
state.openInterest -= balance.amount.abs();  // -= 0, no change
...
balance.amount += balanceDelta;              // += 0, still 0
...
// neither branch fires for amount == 0
``` [2](#0-1) 

`_finalizeSubaccount` requires all perp positions to be `amount == 0` before it proceeds:

```solidity
// ClearinghouseLiq.sol L313-320
for (uint32 i = 0; i < v.perpIds.length; ++i) {
    uint32 perpId = v.perpIds[i];
    IPerpEngine.Balance memory balance = perpEngine.getBalance(perpId, txn.liquidatee);
    require(balance.amount == 0, ERR_NOT_FINALIZABLE_SUBACCOUNT);
}
``` [3](#0-2) 

Then it calls `perpEngine.socializeSubaccount`:

```solidity
// ClearinghouseLiq.sol L386-389
v.insurance = perpEngine.socializeSubaccount(
    txn.liquidatee,
    v.insurance
);
``` [4](#0-3) 

If the liquidatee was the only open-position holder in the market, the prior liquidation step (which closed their `amount` to `0`) already decremented `state.openInterest` to `0`. When `socializeSubaccount` then attempts to divide by `state.openInterest`, it reverts with a division-by-zero panic.

---

### Impact Explanation

The finalization step of the liquidation permanently reverts. The subaccount remains in a state where:
- All perp `amount` fields are `0` (positions closed)
- `vQuoteBalance` is still negative (unresolved loss)
- The subaccount cannot be finalized again (same revert)
- The protocol's accounting is corrupted: the negative `vQuoteBalance` is never cleared, and the insurance fund is never correctly updated

This is a direct analog to M-10: an accounting path that becomes permanently blocked due to a missing guard on a shared-state invariant (here, `openInterest == 0` instead of collateral underflow).

---

### Likelihood Explanation

Any perp market that reaches a state with a single remaining position holder who becomes insolvent (negative `vQuoteBalance` exceeding insurance) triggers this. This is realistic in:
- Low-liquidity or newly launched perp markets
- Markets where all other participants have exited before the insolvent account is finalized
- Adversarial scenarios where an attacker opens a position, waits for all others to exit, then becomes insolvent

The sequencer controls the order of `liquidateSubaccountImpl` calls, but the final `productId == type(uint32).max` finalization call is a required step that any liquidator can trigger.

---

### Recommendation

Add a zero-check on `state.openInterest` before dividing. If `openInterest == 0`, there are no other participants to socialize the loss to; the loss should be absorbed entirely by the insurance fund or written off:

```solidity
if (balance.vQuoteBalance < 0) {
    if (state.openInterest == 0) {
        // No other participants; loss cannot be socialized
        balance.vQuoteBalance = 0;
    } else {
        int128 fundingPerShare = -balance.vQuoteBalance.div(state.openInterest);
        state.cumulativeFundingLongX18 += fundingPerShare;
        state.cumulativeFundingShortX18 -= fundingPerShare;
        balance.vQuoteBalance = 0;
    }
}
```

---

### Proof of Concept

**Attacker-controlled entry path:** `Endpoint.submitTransactionsChecked` → `Clearinghouse.liquidateSubaccount` → `ClearinghouseLiq.liquidateSubaccountImpl` → `_finalizeSubaccount` → `PerpEngine.socializeSubaccount`

**Step-by-step scenario:**

1. Perp market `productId = 2` has a single open long position: subaccount A with `amount = 100e18`, `vQuoteBalance = -50e18` (accumulated funding debt). `state.openInterest = 100e18`.
2. All other participants have exited. `state.openInterest = 100e18` (only A).
3. Price moves against A; A's health falls below maintenance. A is liquidatable.
4. Liquidator calls `liquidateSubaccountImpl` with `productId = 2`, `amount = -100e18` to close A's position. After this call, A's `balance.amount = 0`, `state.openInterest = 0`.
5. Liquidator calls `liquidateSubaccountImpl` with `productId = type(uint32).max` to finalize A.
6. `_finalizeSubaccount` passes the `balance.amount == 0` check.
7. A's `vQuoteBalance` is still negative after insurance is exhausted.
8. `socializeSubaccount` is called. `state.openInterest == 0`. `fundingPerShare = -balance.vQuoteBalance.div(0)` → **division-by-zero revert**.
9. The finalization is permanently blocked. A's subaccount cannot be resolved. [5](#0-4) [6](#0-5)

        external
        returns (int128)
    {
        require(msg.sender == address(_clearinghouse), ERR_UNAUTHORIZED);

        uint32[] memory _productIds = getProductIds();
        for (uint128 i = 0; i < _productIds.length; ++i) {
            uint32 productId = _productIds[i];
            (State memory state, Balance memory balance) = getStateAndBalance(
                productId,
                subaccount
            );
            if (balance.vQuoteBalance < 0) {
                int128 insuranceCover = MathHelper.min(
                    insurance,
                    -balance.vQuoteBalance
                );
                insurance -= insuranceCover;

    function _finalizeSubaccount(
        IEndpoint.LiquidateSubaccount calldata txn,
        ISpotEngine spotEngine,
        IPerpEngine perpEngine
    ) internal returns (bool) {
        if (txn.productId != type(uint32).max) {
            return false;
        }
        // check whether the subaccount can be finalized:
        // - all perps positions have closed
        // - all spread positions have closed
        // - all spot assets have closed
        // - all positive pnls have been settled

        FinalizeVars memory v;

        v.spotIds = spotEngine.getProductIds();
        v.perpIds = perpEngine.getProductIds();

        require(v.spotIds[0] == QUOTE_PRODUCT_ID);

        // all spot assets (except USDC) must be closed out
        for (uint32 i = 1; i < v.spotIds.length; ++i) {
            uint32 spotId = v.spotIds[i];
            if (spotEngine.getRisk(spotId).longWeightInitialX18 == 0) {
                continue;
            }
            ISpotEngine.Balance memory balance = spotEngine.getBalance(
                spotId,
                txn.liquidatee
            );
            require(balance.amount <= 0, ERR_NOT_FINALIZABLE_SUBACCOUNT);
        }

        for (uint32 i = 0; i < v.perpIds.length; ++i) {
            uint32 perpId = v.perpIds[i];
            IPerpEngine.Balance memory balance = perpEngine.getBalance(
                perpId,
                txn.liquidatee
            );
            require(balance.amount == 0, ERR_NOT_FINALIZABLE_SUBACCOUNT);
        }

        // settle all positive pnl
        for (uint32 i = 0; i < v.perpIds.length; ++i) {
            uint32 perpId = v.perpIds[i];
            IPerpEngine.Balance memory balance = perpEngine.getBalance(
                perpId,
                txn.liquidatee
            );
            if (balance.vQuoteBalance > 0) {
                _settlePnlAgainstLiquidator(
                    txn,
                    perpId,
                    balance.vQuoteBalance,
                    spotEngine,
                    perpEngine
                );
            }
        }

        ISpotEngine.Balance memory quoteBalance = spotEngine.getBalance(
            QUOTE_PRODUCT_ID,
            txn.liquidatee
        );

        // settle all negative pnl until quote balance becomes 0
        for (uint32 i = 0; i < v.perpIds.length; ++i) {
            uint32 perpId = v.perpIds[i];
            IPerpEngine.Balance memory balance = perpEngine.getBalance(
                perpId,
                txn.liquidatee
            );
            if (balance.vQuoteBalance < 0 && quoteBalance.amount > 0) {
                int128 canSettle = MathHelper.max(
                    balance.vQuoteBalance,
                    -quoteBalance.amount
                );
                _settlePnlAgainstLiquidator(
                    txn,
                    perpId,
                    canSettle,
                    spotEngine,
                    perpEngine
                );
                quoteBalance.amount += canSettle;
            }
        }

        v.insurance = insurance;
        v.insurance -= lastLiquidationFees;
        v.canLiquidateMore = (quoteBalance.amount + v.insurance) > 0;

        if (v.canLiquidateMore) {
            for (uint32 i = 1; i < v.spotIds.length; ++i) {
                uint32 spotId = v.spotIds[i];
                ISpotEngine.Balance memory balance = spotEngine.getBalance(
                    spotId,
                    txn.liquidatee
                );
                if (spotEngine.getRisk(spotId).longWeightInitialX18 == 0) {
                    continue;
                }
                require(balance.amount == 0, ERR_NOT_FINALIZABLE_SUBACCOUNT);
            }
        }

        v.insurance = perpEngine.socializeSubaccount(
            txn.liquidatee,
            v.insurance
        );

        // we can assure that quoteBalance must be non positive, because if quoteBalance.amount > 0,
        // there must be 1) no negative pnl in perps, and 2) no liabilities in spot after above actions.
        // however, in this case the liquidatee must be healthy and cannot pass the health check at
        // the beginning.
        int128 insuranceCover = MathHelper.min(
            v.insurance,
            -quoteBalance.amount
        );
        if (insuranceCover > 0) {
            v.insurance -= insuranceCover;
            spotEngine.updateBalance(
                QUOTE_PRODUCT_ID,
                txn.liquidatee,
                insuranceCover
            );
        }
        if (v.insurance <= 0) {
            spotEngine.socializeSubaccount(txn.liquidatee);
        }
        v.insurance += lastLiquidationFees;
        insurance = v.insurance;
        return true;
    }
