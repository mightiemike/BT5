### Title
Missing `isRedelegating` Guard in `validateUndelegatePosition` Allows Slash Evasion via Concurrent Redelegation and Unbonding — (`x/tieredrewards/keeper/msg_validate.go`)

---

### Summary

`validateUndelegatePosition` omits the `isRedelegating` check that both `validateRedelegatePosition` and `validateExitTierWithDelegation` perform. This allows `TierUndelegate` to call `stakingKeeper.Undelegate` on a position whose delegator address still has an active redelegation entry, creating an unbonding delegation with a fixed token amount while a redelegation slash from the source validator is still pending. The position thereby escapes the slash.

---

### Finding Description

`validateUndelegatePosition` checks only ownership, delegation status, exit trigger, and exit lock duration: [1](#0-0) 

It does **not** call `isRedelegating`. By contrast, both sibling validators do:

`validateRedelegatePosition`: [2](#0-1) 

`validateExitTierWithDelegation`: [3](#0-2) 

`isRedelegating` queries the staking module for active redelegation entries on the position's delegator address: [4](#0-3) 

`TierUndelegate` calls `validateUndelegatePosition` and then unconditionally calls `ms.undelegate`: [5](#0-4) 

`undelegate` is a thin wrapper over `stakingKeeper.Undelegate`: [6](#0-5) 

The Cosmos SDK's `Undelegate` does **not** check for active redelegations *to* the current validator — it only blocks redelegations *from* the same validator (transitive redelegation prevention). Therefore, calling `Undelegate` while a redelegation to the current validator is still pending is accepted by the SDK and produces a valid unbonding delegation entry with a **fixed token amount** at the time of the call.

---

### Impact Explanation

When `TierUndelegate` succeeds while a redelegation from `valA → valB` is still active:

1. An unbonding delegation from `valB` is created, locking in the token amount at that moment.
2. If `valA` is subsequently slashed during the redelegation window, the Cosmos SDK's slash propagation burns tokens from `valB`'s bonded pool via `SlashRedelegation`.
3. The unbonding delegation entry from step 1 is **not retroactively reduced** — it was created before the slash and holds a fixed balance.
4. The position receives its full pre-slash token amount upon unbonding completion, effectively escaping the slash that should have applied.

Other delegators to `valB` bear the cost of the slash (their share value is diluted), while the exiting position is made whole.

---

### Likelihood Explanation

The exploit requires a specific but achievable sequence:

1. A position has triggered exit but the exit lock duration has **not yet elapsed** (so `validateRedelegatePosition` still permits redelegation — it blocks only when `HasTriggeredExit() && CompletedExitLockDuration()` are both true). [7](#0-6) 

2. The owner redelegates from `valA` to `valB` (creating an active redelegation with `unbondingID != 0`). [8](#0-7) 

3. The exit lock duration elapses (but the redelegation period — typically 21 days — has not).
4. The owner calls `TierUndelegate`; `validateUndelegatePosition` passes because it never calls `isRedelegating`.
5. A slash on `valA` is applied during the remaining redelegation window; the position's unbonding delegation is unaffected.

A sophisticated actor monitoring validator misbehavior evidence in the mempool (e.g., double-sign) can front-run the slash by redelegating before the evidence is committed, then undelegating after the exit lock expires.

---

### Recommendation

Add the same `isRedelegating` guard used in `validateExitTierWithDelegation` and `validateRedelegatePosition` to `validateUndelegatePosition`:

```go
isRedelegating, err := k.isRedelegating(ctx, pos.DelegatorAddress)
if err != nil {
    return err
}
if isRedelegating {
    return errorsmod.Wrapf(types.ErrActiveRedelegation,
        "position %d has an active redelegation", pos.Id)
}
```

Add a corresponding test in `msg_server_undelegate_test.go` that: creates a position, redelegates it, and asserts `TierUndelegate` returns `ErrActiveRedelegation` while the redelegation is active.

---

### Proof of Concept

```
1. Create position, delegate to valA, trigger exit immediately.
2. Before exit lock duration elapses, call TierRedelegate(valA → valB).
   → unbondingID != 0; redelegation entry stored; position.Delegation.ValidatorAddress = valB.
3. Advance block time past exit lock duration (but < 21-day redelegation period).
4. Call TierUndelegate.
   → validateUndelegatePosition passes (no isRedelegating check).
   → stakingKeeper.Undelegate(delAddr, valB, shares) succeeds.
   → Unbonding delegation created with token amount T.
5. Slash valA at infraction factor f.
   → SDK SlashRedelegation burns f*T tokens from valB's pool.
   → Unbonding delegation balance remains T (not T*(1-f)).
6. After unbonding period, position withdraws T tokens — full pre-slash amount.
   → Slash was borne by other valB delegators; position escaped it.
```

func (k Keeper) validateUndelegatePosition(ctx context.Context, pos types.PositionState, owner string) error {
	if !pos.IsOwner(owner) {
		return types.ErrNotPositionOwner
	}

	if !pos.IsDelegated() {
		return types.ErrPositionNotDelegated
	}

	if !pos.HasTriggeredExit() {

	}

	// skip check for zero amount as we want those positions to be able to close their position properly

	return nil
}

func (k Keeper) isRedelegating(ctx context.Context, delegatorAddress string) (bool, error) {
	delAddr, err := sdk.AccAddressFromBech32(delegatorAddress)
	if err != nil {
		return false, errorsmod.Wrap(sdkerrors.ErrInvalidAddress, "invalid delegator address")
	}
	reds, err := k.stakingKeeper.GetRedelegations(ctx, delAddr, 1)
	if err != nil {
		return false, err
	}
	return len(reds) > 0, nil
}
