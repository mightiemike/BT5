Looking at the tiered rewards module, specifically `ClaimTierRewards` in `msg_server.go` and `claimRewardsAndUpdatesPositions` in `claim_rewards.go`, there is a concrete analog to the described vulnerability class.

---

### Title
Double Bonus Reward Claim via Duplicate Position IDs in `ClaimTierRewards` — (File: `x/tieredrewards/keeper/msg_server.go`)

### Summary
`ClaimTierRewards` loads all requested positions into an in-memory slice before processing any of them. If the same `positionId` appears more than once in `msg.PositionIds`, the position is loaded twice with identical stale state. When `claimRewardsAndUpdatesPositions` iterates over the slice, the second (stale) copy recomputes and pays out the same bonus reward segment that was already paid during the first iteration, draining the `RewardsPoolName` module account.

### Finding Description

In `ClaimTierRewards`:

```go
positions := make([]types.PositionState, 0, len(msg.PositionIds))
for _, posId := range msg.PositionIds {
    pos, err := ms.getPositionState(ctx, posId)   // loaded from KV store
    ...
    positions = append(positions, pos)             // appended as value copy
}
totalBase, totalBonus, err := ms.claimRewardsAndUpdatesPositions(ctx, positions)
``` [1](#0-0) 

All positions are snapshotted into memory **before** any state mutation occurs. There is no deduplication of `positionId` values in this loop.

`claimRewardsAndUpdatesPositions` then iterates over the slice:

```go
for i := range positions {
    pos := &positions[i]
    base, err := k.claimBaseRewards(ctx, *pos)
    bonus, err := k.processEventsAndClaimBonus(ctx, pos)
    k.setPosition(ctx, pos.Position, nil)          // writes updated state to KV store
}
``` [2](#0-1) 

When the first copy of a duplicated position is processed, `processEventsAndClaimBonus` advances `pos.LastBonusAccrual` to `blockTime` in memory and `setPosition` persists this to the KV store. However, the **second copy** in the slice still holds the old `LastBonusAccrual` value, because it was snapshotted before any writes occurred.

Inside `processEventsAndClaimBonus`, the current-segment bonus is computed unconditionally for any bonded validator:

```go
if bonded && val.IsBonded() {
    currentRate, err := k.getTokensPerShare(ctx, valAddr)
    bonus := k.computeSegmentBonus(*pos, tier, segmentStart, blockTime, currentRate)
    totalBonus = totalBonus.Add(bonus)
}
applyBonusAccrualCheckpoint(&pos.Position, blockTime)
``` [3](#0-2) 

`segmentStart` is `pos.LastBonusAccrual` from the stale copy. The second iteration computes `computeSegmentBonus(oldLastBonusAccrual → blockTime)` — the exact same interval already paid in the first iteration — and calls:

```go
k.bankKeeper.SendCoinsFromModuleToAccount(ctx, types.RewardsPoolName, ownerAddr, bonusCoins)
``` [4](#0-3) 

This transfers bonus coins from the module pool to the owner a second time for the same accrual period.

The same stale-state problem applies to historical validator events: `getValidatorEventsSince` is called with the old `pos.LastEventSeq`, and if those events are still present in the KV store (their reference count has not yet reached zero because other positions also reference the same validator), they are replayed and bonus is paid again.

### Impact Explanation

An attacker who owns any position on a currently-bonded validator can drain the `RewardsPoolName` module account proportionally to the number of times they repeat their `positionId` in a single `MsgClaimTierRewards` transaction. Each repetition pays out the full accrued bonus for the position's current segment. With N repetitions, the attacker receives N× the legitimate bonus. The corrupted value is the `RewardsPoolName` module account balance and the position's `LastBonusAccrual` / `LastEventSeq` checkpoints.

### Likelihood Explanation

Any position owner can craft a `MsgClaimTierRewards` message with a repeated `positionId`. The only prerequisite is holding a position on a bonded validator with any non-zero accrued bonus. No privileged role, leaked key, or social engineering is required. The entry path is a standard signed Cosmos SDK transaction.

### Recommendation

Deduplicate `msg.PositionIds` before loading positions, or check for duplicates in `MsgClaimTierRewards.Validate()`. A simple approach is to use a `map[uint64]struct{}` to detect repeated IDs and return an error:

```go
seen := make(map[uint64]struct{}, len(msg.PositionIds))
for _, posId := range msg.PositionIds {
    if _, ok := seen[posId]; ok {
        return nil, errorsmod.Wrapf(types.ErrInvalidPositionID, "duplicate position id: %d", posId)
    }
    seen[posId] = struct{}{}
    ...
}
```

Alternatively, enforce uniqueness at the protobuf/`ValidateBasic` layer so the check cannot be bypassed.

### Proof of Concept

1. Attacker creates a position (ID = 42) on a bonded validator via `MsgLockTier`. Bonus accrues over time.
2. Attacker submits `MsgClaimTierRewards{ Owner: attacker, PositionIds: [42, 42, 42] }`.
3. `ClaimTierRewards` loads position 42 three times into `positions[0..2]`, all with the same `LastBonusAccrual = T0`.
4. First iteration: bonus for `[T0, blockTime]` is computed and paid; `setPosition` writes `LastBonusAccrual = blockTime` to KV store.
5. Second iteration: stale copy still has `LastBonusAccrual = T0`; `processEventsAndClaimBonus` recomputes bonus for `[T0, blockTime]` and calls `SendCoinsFromModuleToAccount` again.
6. Third iteration: same as second.
7. Attacker receives 3× the legitimate bonus; `RewardsPoolName` is drained by 3× the correct amount. [5](#0-4) [2](#0-1) [6](#0-5)

func (ms msgServer) ClaimTierRewards(ctx context.Context, msg *types.MsgClaimTierRewards) (*types.MsgClaimTierRewardsResponse, error) {
	if err := msg.Validate(); err != nil {
		return nil, err
	}

	positions := make([]types.PositionState, 0, len(msg.PositionIds))
	for _, posId := range msg.PositionIds {
		pos, err := ms.getPositionState(ctx, posId)
		if err != nil {
			return nil, err
		}

		if err := ms.validateClaimRewards(pos.Position, msg.Owner); err != nil {
			return nil, err
		}

		positions = append(positions, pos)
	}

	totalBase, totalBonus, err := ms.claimRewardsAndUpdatesPositions(ctx, positions)
	if err != nil {
		return nil, err
	}

	sdkCtx := sdk.UnwrapSDKContext(ctx)
	if err := sdkCtx.EventManager().EmitTypedEvent(&types.EventTierRewardsClaimed{
		Owner:        msg.Owner,
		PositionIds:  msg.PositionIds,
		BaseRewards:  totalBase,
		BonusRewards: totalBonus,
	}); err != nil {
		return nil, err
	}

	return &types.MsgClaimTierRewardsResponse{
		BaseRewards:  totalBase,
		BonusRewards: totalBonus,
		PositionIds:  msg.PositionIds,
	}, nil
}

func (k Keeper) claimRewardsAndUpdatesPositions(ctx context.Context, positions []types.PositionState) (sdk.Coins, sdk.Coins, error) {
	totalBase := sdk.NewCoins()
	totalBonus := sdk.NewCoins()

	for i := range positions {
		pos := &positions[i]

		if !pos.IsDelegated() {
			continue
		}

		base, err := k.claimBaseRewards(ctx, *pos)
		if err != nil {
			return nil, nil, err
		}
		totalBase = totalBase.Add(base...)

		bonus, err := k.processEventsAndClaimBonus(ctx, pos)
		if err != nil {
			return nil, nil, err
		}
		totalBonus = totalBonus.Add(bonus...)

		if err := k.setPosition(ctx, pos.Position, nil); err != nil {
			return nil, nil, err
		}
	}

	return totalBase, totalBonus, nil
}

func (k Keeper) processEventsAndClaimBonus(ctx context.Context, pos *types.PositionState) (sdk.Coins, error) {
	// Rewards should have been claimed before undelegation
	if !pos.IsDelegated() {
		return sdk.NewCoins(), nil
	}

	valAddr, err := sdk.ValAddressFromBech32(pos.Delegation.ValidatorAddress)
	if err != nil {
		return nil, err
	}

	events, err := k.getValidatorEventsSince(ctx, valAddr, pos.LastEventSeq)
	if err != nil {
		return nil, err
	}

	sdkCtx := sdk.UnwrapSDKContext(ctx)
	blockTime := sdkCtx.BlockTime()

	totalBonus := math.ZeroInt()
	// Use the persisted bonded state from the last replay, not a hardcoded default.
	// This prevents overpaying bonus for unbonded gaps between claims.
	bonded := pos.LastKnownBonded
	segmentStart := pos.LastBonusAccrual

	tier, err := k.getTier(ctx, pos.TierId)
	if err != nil {
		return nil, err
	}

	for _, entry := range events {
		evt := entry.Event

		if bonded {
			// Compute bonus for the bonded segment [segmentStart, eventTime]
			// using the snapshot rate at the event.
			bonus := k.computeSegmentBonus(*pos, tier, segmentStart, evt.Timestamp, evt.TokensPerShare)
			totalBonus = totalBonus.Add(bonus)
		}

		// Update bonded state based on event type.
		switch evt.EventType {
		case types.ValidatorEventType_VALIDATOR_EVENT_TYPE_UNBOND:
			bonded = false
		case types.ValidatorEventType_VALIDATOR_EVENT_TYPE_BOND:
			bonded = true
		case types.ValidatorEventType_VALIDATOR_EVENT_TYPE_SLASH:
			// Slash doesn't change bonded state.
		}

		segmentStart = evt.Timestamp
		pos.UpdateLastEventSeq(entry.Seq)

		// Decrement reference count.
		if err := k.decrementEventRefCount(ctx, valAddr, entry.Seq); err != nil {
			return nil, err
		}
	}

	val, err := k.stakingKeeper.GetValidator(ctx, valAddr)
	if err != nil {
		return nil, err
	}
	// Defensive: validator bond status check
	if bonded && val.IsBonded() {
		currentRate, err := k.getTokensPerShare(ctx, valAddr)
		if err != nil {
			return nil, err
		}
		bonus := k.computeSegmentBonus(*pos, tier, segmentStart, blockTime, currentRate)
		totalBonus = totalBonus.Add(bonus)
	}

	applyBonusAccrualCheckpoint(&pos.Position, blockTime)
	// Persist the bonded state so the next replay starts correctly.
	pos.UpdateLastKnownBonded(bonded)

	if totalBonus.IsZero() {
		return sdk.NewCoins(), nil
	}

	bondDenom, err := k.stakingKeeper.BondDenom(ctx)
	if err != nil {
		return nil, err
	}

	bonusCoins := sdk.NewCoins(sdk.NewCoin(bondDenom, totalBonus))

	if err := k.sufficientBonusPoolBalance(ctx, bonusCoins); err != nil {
		return nil, err
	}

	ownerAddr, err := sdk.AccAddressFromBech32(pos.Owner)
	if err != nil {
		return nil, errorsmod.Wrap(sdkerrors.ErrInvalidAddress, "invalid owner address")
	}

	if err := k.bankKeeper.SendCoinsFromModuleToAccount(ctx, types.RewardsPoolName, ownerAddr, bonusCoins); err != nil {
		return nil, err
	}

	if err := sdkCtx.EventManager().EmitTypedEvent(&types.EventBonusRewardsClaimed{
		PositionId: pos.Id,
		Owner:      pos.Owner,
		Rewards:    bonusCoins,
	}); err != nil {
		return nil, err
	}

	return bonusCoins, nil
