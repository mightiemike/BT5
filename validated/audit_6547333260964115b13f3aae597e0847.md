### Title
Division-by-Zero in `PerpEngine.socializeSubaccount` When `state.openInterest == 0` Permanently Blocks Bankrupt Subaccount Finalization — (`core/contracts/PerpEngine.sol`)

---

### Summary

`PerpEngine.socializeSubaccount` unconditionally divides by `state.openInterest` when insurance is insufficient to cover a negative `vQuoteBalance`. If `state.openInterest == 0` at the time of finalization — a reachable state when all other participants have closed their positions in that product — the call reverts with `"DBZ"`, permanently blocking finalization of the bankrupt subaccount.

---

### Finding Description

In `PerpEngine.socializeSubaccount`, after exhausting insurance, the residual loss is spread across open-interest holders:

```solidity
if (balance.vQuoteBalance < 0) {
    int128 fundingPerShare = -balance.vQuoteBalance.div(
        state.openInterest   // ← reverts if == 0
    );
``` [1](#0-0) 

`MathSD21x18.div` explicitly requires the denominator to be non-zero:

```solidity
function div(int128 x, int128 y) internal pure returns (int128) {
    unchecked {
        require(y != 0, ERR_DIV_BY_ZERO);
``` [2](#0-1) 

`_finalizeSubaccount` in `ClearinghouseLiq` enforces that all perp `balance.amount` values are zero before calling `socializeSubaccount`:

```solidity
require(balance.amount == 0, ERR_NOT_FINALIZABLE_SUBACCOUNT);
``` [3](#0-2) 

Because `state.openInterest` is the sum of `|balance.amount|` across **all** subaccounts for a product, if the bankrupt subaccount is the last (or only) participant with a non-zero `vQuoteBalance` but has already closed its position (`balance.amount == 0`), and no other subaccount holds an open position, then `state.openInterest == 0`. The finalization check passes (all `balance.amount == 0`), but the subsequent `socializeSubaccount` call reverts. [4](#0-3) 

The call chain is:

```
liquidateSubaccountImpl (productId = type(uint32).max)
  └─ _finalizeSubaccount
       └─ perpEngine.socializeSubaccount   ← reverts: DBZ
``` [5](#0-4) 

---

### Impact Explanation

The revert is unconditional and deterministic: every call to `liquidateSubaccountImpl` with `productId == type(uint32).max` for this subaccount will revert. The bankrupt subaccount can never be finalized, its bad debt can never be socialized, and the insurance fund cannot be properly applied or reclaimed. The protocol is left with permanently unresolved bad debt and a zombie subaccount that blocks the liquidation accounting flow.

---

### Likelihood Explanation

The condition is reachable in normal operation:
1. A subaccount opens and then closes a perp position at a loss (`balance.amount = 0`, `vQuoteBalance < 0`).
2. All other participants in that product also close their positions (`openInterest = 0`).
3. The subaccount's overall health turns negative (the negative `vQuoteBalance` reduces its effective quote balance).
4. A liquidator submits `productId = type(uint32).max` to finalize.

This is especially likely for low-liquidity or deprecated perp markets where open interest naturally drains to zero.

---

### Recommendation

Add a guard in `socializeSubaccount` before the division. If `state.openInterest == 0`, there are no other participants to socialize the loss against; the loss should either be absorbed entirely by insurance or written off:

```solidity
if (balance.vQuoteBalance < 0) {
    if (state.openInterest == 0) {
        // No open interest to socialize against; absorb or write off
        balance.vQuoteBalance = 0;
    } else {
        int128 fundingPerShare = -balance.vQuoteBalance.div(state.openInterest);
        state.cumulativeFundingLongX18 += fundingPerShare;
        state.cumulativeFundingShortX18 -= fundingPerShare;
        balance.vQuoteBalance = 0;
    }
}
``` [6](#0-5) 

---

### Proof of Concept

1. Deploy the protocol with one perp product (e.g., `productId = 2`).
2. SubaccountA opens a long, then closes it at a loss: `balance.amount = 0`, `balance.vQuoteBalance = -100e18`.
3. No other subaccount holds a position in product 2: `state.openInterest = 0`.
4. SubaccountA's overall health is negative (maintenance health < 0).
5. Liquidator calls `liquidateSubaccountImpl({ liquidatee: subaccountA, productId: type(uint32).max, ... })`.
6. `_finalizeSubaccount` passes the `balance.amount == 0` check for all perps.
7. Insurance is set to 0 (or less than 100e18).
8. `perpEngine.socializeSubaccount` is called; reaches `(-(-100e18)).div(0)`.
9. Transaction reverts with `"DBZ"`.
10. Assert: every subsequent finalization attempt for subaccountA reverts identically. [7](#0-6)
