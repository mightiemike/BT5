### Title
Small Bonus Rewards Permanently Lost Due to Integer Division Truncation in `computeSegmentBonus` — (File: `x/tieredrewards/keeper/bonus_rewards.go`)

---

### Summary

`computeSegmentBonus` computes bonus as `shares × tokensPerShare × bonusApy × durationSeconds / SecondsPerYear` and calls `.TruncateInt()`. When the numerator is smaller than `SecondsPerYear` (31,557,600), the result truncates to zero. Critically, `applyBonusAccrualCheckpoint` advances `LastBonusAccrual` unconditionally — before the zero-bonus early-return — so the accrual period that produced zero bonus is permanently consumed and cannot be recovered on a later claim.

---

### Finding Description

In `computeSegmentBonus`:

```go
// x/tieredrewards/keeper/bonus_rewards.go
durationSeconds := int64(segmentEnd.Sub(segmentStart) / time.Second)
if durationSeconds <= 0 {
    return math.ZeroInt()
}

tokens := pos.Delegation.Shares.Mul(tokensPerShare)

return tokens.
    Mul(tier.BonusApy).
    MulInt64(durationSeconds).
    QuoInt64(types.SecondsPerYear).   // ← divides by 31,557,600
    TruncateInt()                      // ← silently drops sub-unit remainder
``` [1](#0-0) 

When `tokens × bonusApy × durationSeconds < SecondsPerYear`, the result is `0`. The formula is documented in the ADR as the promised return:

> **Formula per segment:** `shares * tokensPerShare * bonusApy * durationSeconds / SecondsPerYear` [2](#0-1) 

In `processEventsAndClaimBonus`, `applyBonusAccrualCheckpoint` is called **before** the `totalBonus.IsZero()` guard:

```go
// x/tieredrewards/keeper/claim_rewards.go
applyBonusAccrualCheckpoint(&pos.Position, blockTime)   // ← advances LastBonusAccrual unconditionally
pos.UpdateLastKnownBonded(bonded)

if totalBonus.IsZero() {
    return sdk.NewCoins(), nil                           // ← returns, but checkpoint already advanced
}
``` [3](#0-2) 

`applyBonusAccrualCheckpoint` sets `LastBonusAccrual = blockTime` (or `ExitUnlockAt`):

```go
func applyBonusAccrualCheckpoint(pos *types.Position, blockTime time.Time) {
    accrualEnd := blockTime
    if pos.CompletedExitLockDuration(blockTime) {
        accrualEnd = pos.ExitUnlockAt
    }
    pos.UpdateLastBonusAccrual(accrualEnd)
}
``` [4](#0-3) 

The position is then persisted with the updated `LastBonusAccrual`. The next claim starts from `blockTime`, not from the original `segmentStart`. The truncated accrual period is permanently lost.

---

### Impact Explanation

The module promises bonus rewards for every bonded second. When the product `tokens × bonusApy × durationSeconds` is less than `SecondsPerYear`, the promised bonus truncates to zero and the accrual window is consumed. Concretely:

- A position with 1 basecro locked at 4% APY earns `1 × 0.04 × 86400 / 31,557,600 ≈ 0.00011` basecro per day → `TruncateInt()` = **0**. The user must hold for ~9,132 days to earn 1 basecro of bonus, but each claim resets the window.
- Even for a 1 CRO (10^8 basecro) position, claiming within ~8 seconds of the last claim yields zero bonus and permanently consumes that window.
- The `BeforeRedelegationSlashed` hook calls `processEventsAndClaimBonus` automatically (system-initiated, not user-controlled). If a redelegation slash fires shortly after the last claim, the tiny bonus for that short period is silently forfeited and `LastBonusAccrual` is still advanced.

The module fails to deliver its promised bonus returns for small positions or short accrual windows. Locked tokens are not lost. [5](#0-4) 

---

### Likelihood Explanation

- **User-initiated path**: `MsgClaimTierRewards`, `MsgTierRedelegate`, `MsgAddToTierPosition`, `MsgClearPosition`, `MsgExitTierWithDelegation` all call `processEventsAndClaimBonus`. Any user who claims more frequently than the truncation threshold loses the sub-unit bonus permanently.
- **System-initiated path**: `BeforeRedelegationSlashed` fires automatically before staking's `Unbond` in `SlashRedelegation`. The user has no control over when this fires. If it fires within seconds of the last claim, the bonus for that short period is permanently lost.
- `MinLockAmount` is governance-defined and validated only as non-negative — a value of `0` is accepted, allowing positions that will never earn any bonus at all. [6](#0-5) [5](#0-4) 

---

### Recommendation

1. **Enforce a minimum position size** via `MinLockAmount` that guarantees at least 1 basecro of bonus per expected claim interval (e.g., per epoch).
2. **Only advance `LastBonusAccrual` when `totalBonus > 0`**, or accumulate sub-unit remainders in a fractional accumulator on the position, so truncated amounts are not silently discarded.
3. Alternatively, perform the division before truncation using a higher-precision intermediate (e.g., multiply numerator by a precision factor and divide at the end).

---

### Proof of Concept

```
shares         = 1 (basecro)
tokensPerShare = 1.0
bonusApy       = 0.04 (4%)
durationSeconds = 86400 (1 day)

bonus = 1 × 1.0 × 0.04 × 86400 / 31,557,600
      = 3456 / 31,557,600
      ≈ 0.0001095
      → TruncateInt() = 0

LastBonusAccrual is still advanced to blockTime.
The 1-day accrual window is permanently consumed.
The user earned 0 bonus despite being promised ~0.0001095 basecro.
```

For the `BeforeRedelegationSlashed` path: if a redelegation slash fires 1 second after the last claim on a 1 CRO position:

```
bonus = 1e8 × 1.0 × 0.04 × 1 / 31,557,600
      = 4,000,000 / 31,557,600
      ≈ 0.127
      → TruncateInt() = 0

LastBonusAccrual advanced. 0.127 basecro permanently lost.
``` [7](#0-6) [8](#0-7)

// computeSegmentBonus computes bonus for a time segment using a snapshot rate.
// Formula: shares * tokensPerShare * tier.BonusApy * durationSeconds / SecondsPerYear
func (k Keeper) computeSegmentBonus(pos types.PositionState, tier types.Tier, segmentStart, segmentEnd time.Time, tokensPerShare math.LegacyDec) math.Int {
	if !pos.ExitUnlockAt.IsZero() && segmentEnd.After(pos.ExitUnlockAt) {
		segmentEnd = pos.ExitUnlockAt
	}

	if !segmentEnd.After(segmentStart) {
		return math.ZeroInt()
	}

	durationSeconds := int64(segmentEnd.Sub(segmentStart) / time.Second)
	if durationSeconds <= 0 {
		return math.ZeroInt()
	}

	tokens := pos.Delegation.Shares.Mul(tokensPerShare)

	return tokens.
		Mul(tier.BonusApy).
		MulInt64(durationSeconds).
		QuoInt64(types.SecondsPerYear).
		TruncateInt()
}
```
