### Title
Negative `canSettle` in `settlePnl` Inflates `availableSettle` and Corrupts Perp Accounting — (File: `core/contracts/PerpEngine.sol`)

### Summary
`PerpEngine.settlePnl` computes `canSettle` via `MathHelper.min(positionPnl, state.availableSettle)` without clamping to zero. When a subaccount holds a losing perp position (`positionPnl < 0`) and `state.availableSettle > 0`, `canSettle` is negative. The subsequent arithmetic then inflates `state.availableSettle` and incorrectly reduces `balance.vQuoteBalance`, corrupting the settlement pool and enabling over-settlement by other profitable traders.

### Finding Description
`getSettlementState` returns `availableSettle = min(positionPnl, state.availableSettle)`. [1](#0-0) 

When `positionPnl < 0` and `state.availableSettle > 0`, `canSettle` is a negative value. `settlePnl` then unconditionally applies:

```solidity
state.availableSettle -= canSettle;   // subtracting a negative → pool inflated
balance.vQuoteBalance -= canSettle;   // subtracting a negative → user debt reduced
totalSettled += canSettle;            // negative value returned to clearinghouse
``` [2](#0-1) 

The three-way corruption:
1. `state.availableSettle` increases by `|positionPnl|` — the pool now reports more quote available for settlement than actually exists.
2. `balance.vQuoteBalance` increases by `|positionPnl|` — the user's perp debt is silently reduced.
3. `totalSettled` is negative — the clearinghouse deducts from the user's spot quote balance, making the user's net position appear unchanged while the pool is left insolvent.

The `availableSettle` inflation is permanent and cumulative: every subsequent call to `settlePnl` on a losing position adds more phantom settle capacity, allowing profitable traders to drain more quote than the pool holds.

### Impact Explanation
`state.availableSettle` is the shared pool from which all profitable perp traders settle their gains into real quote tokens. Inflating it allows over-settlement: profitable traders can call `settlePnl` and receive quote tokens that were never deposited, draining the clearinghouse's quote reserves. This is a direct solvency corruption — the protocol can be made to pay out more quote than it holds.

### Likelihood Explanation
`settlePnl` is called by the clearinghouse during normal deposit and withdrawal flows for any subaccount that holds perp positions. Any user who has an open losing perp position and performs a deposit or withdrawal triggers the path. No special privileges are required; the attacker-controlled entry is a standard `WithdrawCollateral` or `DepositCollateral` transaction routed through the endpoint. [3](#0-2) 

### Recommendation
Clamp `canSettle` to be non-negative before applying it. In `getSettlementState`, replace:

```solidity
availableSettle = MathHelper.min(
    calculatePositionPnl(balance, productId),
    state.availableSettle
);
```

with:

```solidity
availableSettle = MathHelper.min(
    MathHelper.max(calculatePositionPnl(balance, productId), 0),
    state.availableSettle
);
```

This ensures that only profitable positions contribute to settlement, and a losing position results in `canSettle = 0` with no state mutation. [4](#0-3) 

### Proof of Concept

1. Alice opens a perp long position. Price drops; her `positionPnl = -100e18`.
2. The settlement pool has `state.availableSettle = 500e18` (funded by other depositors).
3. Alice submits a `WithdrawCollateral` transaction. The clearinghouse calls `settlePnl` on her subaccount.
4. `getSettlementState` returns `canSettle = min(-100e18, 500e18) = -100e18`.
5. `state.availableSettle -= -100e18` → `availableSettle` becomes `600e18` (inflated by 100).
6. `balance.vQuoteBalance -= -100e18` → Alice's perp debt is reduced by 100.
7. `totalSettled = -100e18` is returned; clearinghouse deducts 100 from Alice's spot quote balance.
8. Alice's net value is unchanged, but the pool now reports 600 available when only 500 exists.
9. Bob, a profitable trader with `positionPnl = +500e18`, calls `settlePnl` and successfully settles 600 (the inflated amount), draining 100 more quote than the pool legitimately holds. [3](#0-2) [5](#0-4)

    function settlePnl(bytes32 subaccount, uint256 productIds)
        external
        returns (int128)
    {
        _assertInternal();
        int128 totalSettled = 0;

        while (productIds != 0) {
            uint32 productId = uint32(productIds & ((1 << 32) - 1));
            // otherwise it means the product is a spot.
            if (productId % 2 == 0) {
                (
                    int128 canSettle,
                    State memory state,
                    Balance memory balance
                ) = getSettlementState(productId, subaccount);

                state.availableSettle -= canSettle;
                balance.vQuoteBalance -= canSettle;

                totalSettled += canSettle;

                _setState(productId, state);
                _setBalanceAndUpdateBitmap(productId, subaccount, balance);
            }
            productIds >>= 32;
        }
        return totalSettled;
    }
