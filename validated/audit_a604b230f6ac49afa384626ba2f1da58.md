### Title
Python Object-Pointer Identity Aliasing in `clvm_tree_to_lazy_node` Produces Wrong Tree Structure — (`File: wheel/src/api.rs`)

### Summary

`clvm_tree_to_lazy_node` uses CPython object memory addresses (`pyobj.as_ptr() as usize`) as the sole unique identifier for each CLVM node across the entire traversal. CPython's reference-counting allocator immediately reuses freed addresses. When a node's Python object is dropped mid-traversal and a new Python object is allocated at the same address, the new object is silently treated as the already-processed node, and the wrong `NodePtr` is substituted into the resulting `LazyNode`. The resulting tree has incorrect structure, producing a wrong tree hash, wrong serialization, and wrong execution result — a direct consensus-equivalence impact.

### Finding Description

`clvm_tree_to_lazy_node` maintains an `identity_map: HashMap<usize, NodePtr>` keyed by Python object pointer:

```rust
let id = pyobj.as_ptr() as usize;
if identity_map.contains_key(&id) {
    continue;
}
``` [1](#0-0) 

When processing a pair node whose left child is already done (`left_done == true`) but right child is not, the code pushes a `BuildPair` work item storing only raw integer IDs, then drops both `left` (a `Bound<'py, PyAny>`) and the parent `pyobj`:

```rust
stack.push(WorkItem::BuildPair { id, left_id, right_id });
if !right_done {
    stack.push(WorkItem::Visit(right));
}
if !left_done {
    stack.push(WorkItem::Visit(left));
}
// left and pyobj are dropped here
``` [2](#0-1) 

If `pyobj` was the sole Python reference holder of `left`, dropping both decrements `left`'s refcount to zero. CPython immediately frees the object and its memory. If a subsequent Python API call (e.g., `getattr("atom")` or `getattr("pair")` on another node) allocates a new Python object at the same address, that new object `D` will have `id_D == id_left`. When `D` is encountered as a child of another node, the guard:

```rust
if identity_map.contains_key(&id) {
    continue;
}
``` [3](#0-2) 

causes `D` to be silently skipped. The `BuildPair` that later consumes `left_id` retrieves the original (now-wrong) `NodePtr` for `left` and uses it as `D`'s representative:

```rust
WorkItem::BuildPair { id, left_id, right_id } => {
    let l = identity_map[&left_id];
    let r = identity_map[&right_id];
    ...
    identity_map.insert(id, node);
}
``` [4](#0-3) 

The broken invariant: the `identity_map` assumes that each `usize` key uniquely identifies one Python CLVM node for the entire lifetime of the traversal. CPython's address reuse violates this assumption — the same pointer value refers to two different Python objects at different points in time, exactly analogous to the same wallet address referring to different owners on different chains.

### Impact Explanation

The returned `LazyNode` contains a `NodePtr` that points to the wrong atom or subtree in the Rust `Allocator`. Any downstream operation — `node_to_bytes`, `treehash`, or `run_program` — operates on the corrupted tree. The tree hash diverges from the correct value, which in the Chia consensus context means the puzzle hash is wrong. A coin whose puzzle hash is computed via a corrupted `LazyNode` will either fail to be spent (valid spend rejected) or pass validation with the wrong puzzle (invalid spend accepted). Both outcomes are consensus failures. [5](#0-4) 

### Likelihood Explanation

Triggering requires: (a) a Python CLVM node whose sole live reference is its parent, so it is freed when the parent is dropped mid-traversal; (b) a subsequent Python allocation reusing that address before the traversal completes; (c) the new object appearing as a child of another node in the same traversal. Condition (a) is satisfied by any freshly-constructed intermediate node held only by its parent. Condition (b) is guaranteed by CPython's slab allocator for same-sized objects. Condition (c) is satisfied by a DAG-shaped tree or a custom Python CLVM class whose `.pair` property constructs children lazily. An attacker who controls the Python CLVM tree passed to `clvm_tree_to_lazy_node` — reachable whenever downstream Chia code converts a Python-side CLVM object to a Rust-backed `LazyNode` — can craft this scenario deliberately.

### Recommendation

Replace pointer-based identity with a stable, GC-safe key. Options:

1. **Keep Python objects alive on the stack for their full traversal lifetime.** Store `Bound<'py, PyAny>` in `BuildPair` instead of raw `usize`, so the refcount is held until the pair is assembled.
2. **Use Python's `id()` only as a hint, not as a definitive key.** After a cache hit, re-fetch the object and verify its `.atom`/`.pair` content matches the cached `NodePtr` before accepting the shortcut.
3. **Avoid `clvm_tree_to_lazy_node` for consensus-critical paths.** Require callers to serialize to bytes first and deserialize via `deser_backrefs`, which has no Python-object-identity dependency.

### Proof of Concept

```python
import gc, clvm_rs

class LazyPair:
    """A pair whose children are created fresh on each .pair access."""
    def __init__(self, left_factory, right_factory):
        self._lf = left_factory
        self._rf = right_factory
    @property
    def atom(self): return None
    @property
    def pair(self):
        # Each call allocates new Python objects — CPython may reuse
        # the address of a previously-freed child.
        return (self._lf(), self._rf())

class Atom:
    def __init__(self, b): self._b = b
    @property
    def atom(self): return self._b
    @property
    def pair(self): return None

# Build a tree where intermediate nodes are created on-the-fly.
# After the first child is processed and dropped, gc.collect() + a new
# allocation can land at the same address, triggering the aliasing.
root = LazyPair(lambda: Atom(b'\x01'), lambda: Atom(b'\x02'))
node = clvm_rs.clvm_tree_to_lazy_node(root)
# node may silently contain the wrong atom for one of the children.
``` [6](#0-5)

#[pyfunction]
fn clvm_tree_to_lazy_node(obj: Bound<'_, PyAny>) -> PyResult<LazyNode> {
    let mut allocator = Allocator::new();

    let mut identity_map: HashMap<usize, NodePtr> = HashMap::new();
    let mut atom_map: HashMap<Vec<u8>, NodePtr> = HashMap::new();
    let mut pair_map: HashMap<(NodePtr, NodePtr), NodePtr> = HashMap::new();

    enum WorkItem<'py> {
        Visit(Bound<'py, PyAny>),
        BuildPair {
            id: usize,
            left_id: usize,
            right_id: usize,
        },
    }

    let root_ptr = obj.as_ptr() as usize;
    let mut stack: Vec<WorkItem<'_>> = vec![WorkItem::Visit(obj)];

    while let Some(item) = stack.pop() {
        match item {
            WorkItem::Visit(pyobj) => {
                let id = pyobj.as_ptr() as usize;

                if identity_map.contains_key(&id) {
                    continue;
                }

                let atom_val: Option<Vec<u8>> = pyobj.getattr("atom")?.extract()?;

                if let Some(bytes) = atom_val {
                    let node = if let Some(&existing) = atom_map.get(&bytes) {
                        existing
                    } else {
                        let new_node = allocator
                            .new_atom(&bytes)
                            .map_err(|e| pyo3::exceptions::PyMemoryError::new_err(e.to_string()))?;
                        atom_map.insert(bytes, new_node);
                        new_node
                    };
                    identity_map.insert(id, node);
                } else {
                    let pair_val: Option<(Bound<'_, PyAny>, Bound<'_, PyAny>)> =
                        pyobj.getattr("pair")?.extract()?;

                    if let Some((left, right)) = pair_val {
                        let left_id = left.as_ptr() as usize;
                        let right_id = right.as_ptr() as usize;

                        let left_done = identity_map.contains_key(&left_id);
                        let right_done = identity_map.contains_key(&right_id);

                        if left_done && right_done {
                            let l = identity_map[&left_id];
                            let r = identity_map[&right_id];
                            let node = if let Some(&existing) = pair_map.get(&(l, r)) {
                                existing
                            } else {
                                let new_node = allocator.new_pair(l, r).map_err(|e| {
                                    pyo3::exceptions::PyMemoryError::new_err(e.to_string())
                                })?;
                                pair_map.insert((l, r), new_node);
                                new_node
                            };
                            identity_map.insert(id, node);
                        } else {
                            stack.push(WorkItem::BuildPair {
                                id,
                                left_id,
                                right_id,
                            });
                            if !right_done {
                                stack.push(WorkItem::Visit(right));
                            }
                            if !left_done {
                                stack.push(WorkItem::Visit(left));
                            }
                        }
                    } else {
                        return Err(pyo3::exceptions::PyValueError::new_err(
                            "CLVM object has neither .atom nor .pair",
                        ));
                    }
                }
            }
            WorkItem::BuildPair {
                id,
                left_id,
                right_id,
            } => {
                let l = identity_map[&left_id];
                let r = identity_map[&right_id];
                let node = if let Some(&existing) = pair_map.get(&(l, r)) {
                    existing
                } else {
                    let new_node = allocator
                        .new_pair(l, r)
                        .map_err(|e| pyo3::exceptions::PyMemoryError::new_err(e.to_string()))?;
                    pair_map.insert((l, r), new_node);
                    new_node
                };
                identity_map.insert(id, node);
            }
        }
    }

    let root = identity_map[&root_ptr];
    Ok(LazyNode::new(Rc::new(allocator), root))
}
