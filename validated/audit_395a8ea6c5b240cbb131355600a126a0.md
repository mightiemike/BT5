### Title
`op_substr` Charges Fixed Cost of 1 Regardless of Substring Size, Enabling Undercharged Execution — (`File: src/more_ops.rs`)

### Summary

`op_substr` (opcode 12) charges a hardcoded cost of `1` for any substring operation, regardless of the size of the atom being sliced. Every other atom-producing operator in the VM charges `MALLOC_COST_PER_BYTE` (10 units/byte) for the result. This makes `op_substr` severely undercharged relative to the actual work it performs, breaking the cost model's DoS invariant.

### Finding Description

In `src/more_ops.rs`, `op_substr` validates its arguments and then calls `a.new_substr(a0, start as u32, end as u32)`, returning a fixed cost of exactly `1`:

```rust
} else {
    let r = a.new_substr(a0, start as u32, end as u32)?;
    let cost: Cost = 1;
    Ok(Reduction(cost, r))
}
``` [1](#0-0) 

The `_max_cost` parameter is explicitly ignored (underscore prefix), meaning no cost ceiling check is performed inside the operator at all. [2](#0-1) 

Compare this to every other atom-producing operator in the codebase. `op_concat` charges `CONCAT_COST_PER_BYTE + MALLOC_COST_PER_BYTE` per byte of output:

```rust
cost += len as Cost * (CONCAT_COST_PER_BYTE + MALLOC_COST_PER_BYTE);
``` [3](#0-2) 

`op_strlen` charges `STRLEN_COST_PER_BYTE` per byte just to *read* the length of an atom: [4](#0-3) [5](#0-4) 

Even purely structural O(1) operators that allocate nothing charge far more than 1: `op_if` = 33, `op_cons` = 50, `op_first` = 30, `op_rest` = 30, `op_listp` = 19. [6](#0-5) 

The `MALLOC_COST_PER_BYTE` constant is defined as 10 and is the universal mechanism for accounting for memory pressure: [7](#0-6) 

`op_substr` bypasses this entirely. A substring of 1 MB costs the same as a substring of 1 byte: 1 unit.

### Impact Explanation

An attacker-controlled CLVM program can call `op_substr` in a loop to produce many large atom results at cost 1 each. If `Allocator::new_substr` copies bytes (allocates new heap memory), the attacker can exhaust the node heap while the reported cost stays near zero, defeating the cost model's primary DoS protection. Even if `new_substr` is zero-copy (a slice reference into existing memory), the cost of 1 is still inconsistent with the rest of the model: it is lower than every other operator including purely structural ones that allocate nothing, and it allows an attacker to produce an unbounded number of large-atom `NodePtr` values at negligible cost, which can then be fed into downstream operators. The `LIMIT_HEAP` flag (`ClvmFlags::LIMIT_HEAP`) provides a secondary guard, but it is not always enabled (it is absent from the default `ClvmFlags::empty()` dialect used in consensus mode). [8](#0-7) 

### Likelihood Explanation

The entry path is direct: any caller of `run_program` with attacker-supplied CLVM bytes can invoke opcode 12 (`substr`) with a large atom and arbitrary indices. The operator is dispatched unconditionally in `ChiaDialect::op` with no flag guard: [9](#0-8) 

No special dialect flag, softfork guard, or whitelist is required. The cost of 1 is always returned.

### Recommendation

Apply the same cost formula used by all other atom-producing operators. At minimum, charge `MALLOC_COST_PER_BYTE` per byte of the resulting substring, and add a base cost consistent with other structural operators (≥ 19):

```rust
let substr_len = (end - start) as usize;
let cost: Cost = SUBSTR_BASE_COST + substr_len as Cost * MALLOC_COST_PER_BYTE;
check_cost(cost, max_cost)?;
```

Also remove the `_` prefix from `_max_cost` so the cost ceiling is enforced inside the operator.

### Proof of Concept

```lisp
; CLVM pseudocode: create a 1 MB atom, then substr it 10000 times at cost 1 each
; Total cost charged: ~10,000,010 (initial alloc) + 10,000 (substr loop) = ~10,010,010
; Actual memory touched: 10,000 * 1 MB = 10 GB (if new_substr copies)

(a
  (q . (loop_body))
  (q . (big_atom_1mb 10000)))
```

Concretely: allocate a 1 MB atom via `concat` (paying ~13 million cost units), then call `(substr big_atom 0 1000000)` in a tight loop 10,000 times. Each call costs 1. The total cost budget consumed by the substr calls is 10,000 units, while the actual work (if copying) is 10 GB of memory writes. A node enforcing an 11-billion-cost limit would allow ~1,100,000 such substr calls on a 1 MB atom before hitting the limit, potentially allocating over 1 TB of heap data.
