### Title
Missing Receiver-Equals-Escrow Guard in `processReceivedPacket` Permanently Locks Minted Voucher NFTs — (`x/nft-transfer/keeper/packet.go`)

### Summary

`OnRecvPacket` → `ValidateBasic` → `processReceivedPacket` contains no check that `data.Receiver` is not the module's own escrow address for the destination port/channel. A malicious IBC counterparty can craft a packet whose `Receiver` field is the bech32 encoding of `GetEscrowAddress("nft", "channel-0")` on the receiving chain. `ValidateBasic` accepts it (it is a valid bech32 address), and `processReceivedPacket` mints the voucher NFT directly to the escrow address, where it is permanently unrecoverable.

### Finding Description

**`ValidateBasic`** only verifies that `Receiver` is non-blank and parses as a valid bech32 address. There is no check that the decoded address is not the escrow address. [1](#0-0) 

**`processReceivedPacket`** independently computes `escrowAddress` from the destination port/channel, then in the `isAwayFromOrigin` branch calls:

```go
k.nftKeeper.MintNFT(ctx, voucherClassID, tokenID, "", data.TokenUris[i], "", escrowAddress, receiver)
```

where `receiver` is decoded directly from `data.Receiver` with no comparison against `escrowAddress`. [2](#0-1) 

The escrow address is a deterministic, publicly computable value: [3](#0-2) 

It is created as a **regular account** (not a module account, not a blocked address) via `NewAccountWithAddress`: [4](#0-3) 

The `AccountKeeper` interface exposed to the nft-transfer module does not include `BlockedAddr`, so there is no blocked-address guard at the NFT minting layer either. [5](#0-4) 

### Impact Explanation

When `receiver == escrowAddress`:
- `MintNFT` succeeds — the escrow address is a valid, existing account.
- The voucher NFT is recorded as owned by the escrow address.
- No user holds the private key to the escrow address (it is a hash-derived address per ADR-028).
- The NFT is permanently locked with no recovery path: no `TransferOwner` can be called because no signer can authorize it, and no module logic re-examines NFTs already sitting in the escrow account.

The `isAwayFromOrigin` branch is the direct path: the malicious counterparty sends a packet whose `ClassId` does not carry the receiving chain's port/channel prefix, making `IsAwayFromOrigin` return `true`, and sets `Receiver` to the escrow address string. [6](#0-5) 

### Likelihood Explanation

The escrow address for any channel is deterministic and publicly computable from the channel identifiers. Any chain that has an open IBC channel with the receiving chain can craft such a packet. No governance vote, no privileged key, and no special permission is required — only the ability to submit an IBC packet, which is the normal operation of any counterparty chain relayer.

### Recommendation

Add an explicit guard in `processReceivedPacket` immediately after decoding `receiver`:

```go
receiver, err := sdk.AccAddressFromBech32(data.Receiver)
if err != nil {
    return err
}
// Guard: receiver must not be the escrow address for this channel
escrowAddress := types.GetEscrowAddress(packet.GetDestPort(), packet.GetDestChannel())
if receiver.Equals(escrowAddress) {
    return sdkerrors.Wrapf(sdkerrors.ErrUnauthorized,
        "receiver cannot be the escrow address %s", escrowAddress)
}
```

Alternatively, add the same check inside `ValidateBasic` if the escrow address is available at that point, or enforce it at the IBC module boundary before dispatching to `OnRecvPacket`.

### Proof of Concept

```go
func TestProcessReceivedPacket_ReceiverIsEscrow(t *testing.T) {
    // compute the escrow address for port="nft", channel="channel-0"
    escrowAddr := types.GetEscrowAddress("nft", "channel-0")

    data := types.NonFungibleTokenPacketData{
        ClassId:   "someClass",          // no dest prefix → isAwayFromOrigin = true
        ClassUri:  "",
        TokenIds:  []string{"token1"},
        TokenUris: []string{"uri1"},
        Sender:    validSenderBech32,
        Receiver:  escrowAddr.String(),  // attacker sets receiver = escrow address
    }

    packet := channeltypes.Packet{
        SourcePort:      "nft",
        SourceChannel:   "channel-1",
        DestinationPort: "nft",
        DestinationChannel: "channel-0",
        // ...
    }

    err := keeper.OnRecvPacket(ctx, types.Version, packet, data)
    require.NoError(t, err) // passes today — this is the bug

    // assert the NFT owner is NOT the escrow address
    nft, _ := nftKeeper.GetNFT(ctx, ibcClassID, "token1")
    require.NotEqual(t, escrowAddr, nft.GetOwner(),
        "NFT must not be minted to the escrow address")
}
```

Running this test against the unmodified code will show `err == nil` and `nft.GetOwner() == escrowAddr`, confirming the invariant is broken and the voucher NFT is permanently locked. [7](#0-6) [8](#0-7)

```go
func (k Keeper) processReceivedPacket(ctx sdk.Context, packet channeltypes.Packet,
	data types.NonFungibleTokenPacketData,
) error {
	receiver, err := sdk.AccAddressFromBech32(data.Receiver)
	if err != nil {
		return err
	}

	isAwayFromOrigin := types.IsAwayFromOrigin(packet.GetSourcePort(), packet.GetSourceChannel(), data.ClassId)

	// create the escrow address for creating denom and minting nft
	escrowAddress := types.GetEscrowAddress(packet.GetDestPort(), packet.GetDestChannel())

	if isAwayFromOrigin {
		// since SendPacket did not prefix the classID, we must prefix classID here
		classPrefix := types.GetClassPrefix(packet.GetDestPort(), packet.GetDestChannel())
		// NOTE: sourcePrefix contains the trailing "/"
		prefixedClassID := classPrefix + data.ClassId

		// construct the class trace from the full raw classID
		classTrace := types.ParseClassTrace(prefixedClassID)
		if !k.HasClassTrace(ctx, classTrace.Hash()) {
			k.SetClassTrace(ctx, classTrace)
		}

		voucherClassID := classTrace.IBCClassID()

		if !k.nftKeeper.HasDenomID(ctx, voucherClassID) {
			if err := k.nftKeeper.IssueDenom(ctx, voucherClassID, voucherClassID, "", data.ClassUri, escrowAddress); err != nil {
				return err
			}
		}
		sdkCtx := sdk.UnwrapSDKContext(ctx)
		sdkCtx.EventManager().EmitEvent(
			sdk.NewEvent(
				types.EventTypeClassTrace,
				sdk.NewAttribute(types.AttributeKeyTraceHash, classTrace.Hash().String()),
				sdk.NewAttribute(types.AttributeKeyClassID, voucherClassID),
			),
		)

		for i, tokenID := range data.TokenIds {
			if err := k.nftKeeper.MintNFT(ctx, voucherClassID, tokenID, "", data.TokenUris[i], "", escrowAddress, receiver); err != nil {
				return err
			}
		}
```

```go
func GetEscrowAddress(portID, channelID string) sdk.AccAddress {
	// a slash is used to create domain separation between port and channel identifiers to
	// prevent address collisions between escrow addresses created for different channels
	contents := fmt.Sprintf("%s/%s", portID, channelID)

	// ADR 028 AddressHash construction
	preImage := []byte(Version)
	preImage = append(preImage, 0)
	preImage = append(preImage, contents...)
	hash := sha256.Sum256(preImage)
	return hash[:20]
```
