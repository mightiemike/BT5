### Title
SwapAllowlistExtension Checks Router Address Instead of End-User, Allowing Any User to Bypass the Swap Allowlist via MetricOmmSimpleRouter - (File: metric-periphery/contracts/extensions/SwapAllowlistExtension.sol)

### Summary

`SwapAllowlistExtension.beforeSwap` gates swaps by checking the `sender` argument, which is the pool's `msg.sender`. When a user routes through `MetricOmmSimpleRouter`, the pool's `msg.sender` is the router contract, not the original user. If the pool admin allowlists the router (necessary to support router-mediated swaps for legitimate users), every unprivileged user can bypass the allowlist by routing through the router.

### Finding Description

`MetricOmmPool.swap` passes `msg.sender` as the `sender` argument to `_beforeSwap`: [1](#0-0) 

`ExtensionCalling._beforeSwap` forwards that value unchanged to every configured extension: [2](#0-1) 

`SwapAllowlistExtension.beforeSwap` then checks `allowedSwapper[msg.sender][sender]`, where `msg.sender` is the pool and `sender` is whoever called `pool.swap()`: [3](#0-2) 

`MetricOmmSimpleRouter.exactInputSingle` calls `pool.swap()` directly, making the router itself the pool's `msg.sender`. The original end-user's address is stored only in transient callback context and is never forwarded to the pool: [4](#0-3) 

The same pattern applies to `exactInput`, `exactOutputSingle`, and `exactOutput`: [5](#0-4) 

**Consequence**: The extension always evaluates `allowedSwapper[pool][router]` for any router-mediated swap, regardless of which end-user initiated it. A pool admin who wants allowlisted users to be able to use the router must add the router address to the allowlist. Once the router is allowlisted, the check passes for every caller — including non-allowlisted users — because the extension sees only the router's address.

The `DepositAllowlistExtension` does not share this flaw because it gates on the `owner` argument (the explicit position owner), not on `sender`: [6](#0-5) 

### Impact Explanation

Any user who is not on the swap allowlist can execute swaps on a curated pool by calling `MetricOmmSimpleRouter.exactInputSingle` (or any other router entry point), provided the router address has been added to the allowlist. The pool admin's intent — to restrict trading to a specific set of addresses — is completely defeated. Depending on the pool's purpose (regulatory compliance, exclusive LP access, risk-gated pools), this constitutes a direct loss of the curation guarantee and may expose the pool to trades from actors the admin explicitly intended to block.

### Likelihood Explanation

The router is the standard periphery swap path. Pool admins who deploy a curated pool and want their allowlisted users to be able to use the router will naturally add the router to the allowlist. This is a predictable, non-adversarial configuration step that silently opens the gate to all users. No special privileges or malicious setup are required; any user can call the public router functions.

### Recommendation

The `sender` forwarded to extensions must represent the economic actor, not the intermediary. Two complementary fixes:

1. **Router-side**: Have `MetricOmmSimpleRouter` encode the original `msg.sender` inside `extensionData` so that the extension can decode and check the real user. This requires a convention between the router and the extension.

2. **Extension-side (preferred)**: Change `SwapAllowlistExtension.beforeSwap` to accept an optional "real sender" encoded in `extensionData`. If present, check that address; otherwise fall back to the `sender` argument. The router would always encode `msg.sender` into `extensionData` before forwarding to the pool.

A minimal stopgap is to document that allowlisting the router address defeats per-user gating, and to provide a separate `RouterSwapAllowlistExtension` that decodes the real user from `extensionData`.

### Proof of Concept

```
Setup:
  pool configured with SwapAllowlistExtension
  pool admin calls setAllowedToSwap(pool, alice, true)       // alice is the only allowed swapper
  pool admin calls setAllowedToSwap(pool, router, true)      // router added so alice can use it

Attack (bob is not allowlisted):
  bob calls MetricOmmSimpleRouter.exactInputSingle({
      pool: pool,
      recipient: bob,
      zeroForOne: true,
      amountIn: X,
      ...
  })

Trace:
  router calls pool.swap(recipient=bob, ...)
  pool.swap: msg.sender = router
  pool calls _beforeSwap(sender=router, ...)
  extension checks allowedSwapper[pool][router] → true   ← router is allowlisted
  swap executes for bob despite bob not being on the allowlist
```

Bob successfully trades on the curated pool. The allowlist check evaluated the router's identity, not bob's.
