### Title
Deprecated `verify_transaction_inclusion` Remains Callable and Accepts Internal-Node Hashes as Valid Transaction Proofs — (File: `contract/src/lib.rs`)

---

### Summary

The deprecated `verify_transaction_inclusion` method is still exported and callable by any unprivileged NEAR account. It performs no validation that the supplied `tx_id` is a leaf-level transaction hash rather than an internal Merkle-tree node hash. An attacker can supply an internal node hash as `tx_id` with a shortened proof path and receive a `true` return value, falsely proving inclusion of a non-existent transaction.

---

### Finding Description

`verify_transaction_inclusion` is marked `#[deprecated]` but Rust's `#[deprecated]` attribute only emits a compiler warning — it does not prevent the function from being compiled into the WASM binary or called on-chain. [1](#0-0) 

The function delegates directly to `merkle_tools::compute_root_from_merkle_proof`, passing the caller-supplied `args.tx_id` as the starting hash with no check that it is a leaf node: [2](#0-1) 

`compute_root_from_merkle_proof` is a pure positional hash-chain computation. It accepts any 32-byte value as the starting hash and walks up the tree using the supplied proof hashes: [3](#0-2) 

Because the function does not enforce that `merkle_proof.len()` equals the full tree depth, an attacker can supply an internal node at depth `D` in a tree of depth `N` with only `N - D` proof hashes. If those hashes are valid siblings on the path from depth `D` to the root, `compute_root_from_merkle_proof` returns the correct `merkle_root`, and the function returns `true`.

The contract's own documentation acknowledges this: [4](#0-3) 

The safe replacement, `verify_transaction_inclusion_v2`, fixes this by requiring a coinbase proof of the same length, which pins the tree depth: [5](#0-4) 

But `verify_transaction_inclusion` remains live and unguarded beyond the `#[pause]` decorator.

---

### Impact Explanation

Any recipient contract that calls `verify_transaction_inclusion` to gate a cross-chain action (e.g., releasing funds, minting tokens, updating state) can be deceived into accepting a forged proof. The attacker does not need to mine a block or control any privileged role — they only need a mainchain block whose Merkle tree they can inspect to extract an internal node hash and construct a valid shortened proof path. The corrupted value is the boolean proof result returned to the consuming contract, which treats it as authoritative confirmation of a Bitcoin transaction that was never broadcast.

---

### Likelihood Explanation

The attack requires only:
1. Knowledge of a mainchain block's Merkle tree (publicly available from any Bitcoin node).
2. The ability to call a NEAR contract function (any NEAR account, zero stake required).

No privileged role, no leaked key, no social engineering. The function is callable today on any deployment that has not explicitly paused it via the `PauseManager` role.

---

### Recommendation

Remove `verify_transaction_inclusion` from the contract's public interface entirely. Because `#[deprecated]` does not prevent on-chain calls, the only safe remediation is to delete the method or replace its body with `env::panic_str("use verify_transaction_inclusion_v2")`. All callers should be migrated to `verify_transaction_inclusion_v2`.

---

### Proof of Concept

Given a mainchain block at hash `B` with a Merkle tree of depth 3 (8 transactions):

```
         root
        /    \
      n01    n23
     /  \   /  \
   n0  n1  n2  n3   <- internal nodes at depth 1
  / \ / \ / \ / \
 t0 t1 t2 t3 t4 t5 t6 t7  <- leaves (real txids)
```

The attacker:
1. Computes `n01 = SHA256d(t0 || t1)` — an internal node, not a transaction.
2. Calls `verify_transaction_inclusion` with:
   - `tx_id = n01`
   - `tx_index = 0`
   - `merkle_proof = [n23]` (length 1, not 3)
   - `tx_block_blockhash = B`
   - `confirmations = 1`
3. `compute_root_from_merkle_proof(n01, 0, [n23])` computes `SHA256d(n01 || n23) = root`.
4. `root == header.block_header.merkle_root` → function returns `true`.

The consuming contract receives `true` for a transaction that does not exist. [3](#0-2) [6](#0-5)

    #[deprecated(
        since = "0.5.0",
        note = "Use `verify_transaction_inclusion_v2` instead."

    #[pause]
    pub fn verify_transaction_inclusion(&self, #[serializer(borsh)] args: ProofArgs) -> bool {
        require!(
            args.confirmations <= self.gc_threshold,
            "The required number of confirmations exceeds the number of blocks stored in memory"
        );

        let heaviest_block_header = self
            .headers_pool
            .get(&self.mainchain_tip_blockhash)
            .unwrap_or_else(|| env::panic_str(ERR_KEY_NOT_EXIST));
        let target_block_height = self
            .mainchain_header_to_height
            .get(&args.tx_block_blockhash)
            .unwrap_or_else(|| env::panic_str("block does not belong to the current main chain"));

        // Check requested confirmations. No need to compute proof if insufficient confirmations.
        require!(
            (heaviest_block_header.block_height).saturating_sub(target_block_height) + 1
                >= args.confirmations,
            "Not enough blocks confirmed"
        );

        let header = self
            .headers_pool
            .get(&args.tx_block_blockhash)
            .unwrap_or_else(|| env::panic_str("cannot find requested transaction block"));

        require!(!args.merkle_proof.is_empty(), "Merkle proof is empty");

        // compute merkle tree root and check if it matches block's original merkle tree root
        merkle_tools::compute_root_from_merkle_proof(
            args.tx_id,
            usize::try_from(args.tx_index).unwrap(),
            &args.merkle_proof,
        ) == header.block_header.merkle_root
    }

pub fn compute_root_from_merkle_proof(
    transaction_hash: H256,
    transaction_position: usize,
    merkle_proof: &Vec<H256>,
) -> H256 {
    let mut current_hash = transaction_hash;
    let mut current_position = transaction_position;

    for proof_hash in merkle_proof {
        if current_position % 2 == 0 {
            current_hash = compute_hash(&current_hash, proof_hash);
        } else {
            current_hash = compute_hash(proof_hash, &current_hash);
        }
        current_position /= 2;
    }

    current_hash
}
