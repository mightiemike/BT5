### Title
ICS-721 `ValidateBasic()` Incorrectly Enforces Bech32 Validation on Sender Address, Permanently Blocking NFT Inbound Transfers from Non-Cosmos Chains — (`x/nft-transfer/types/packet.go`)

---

### Summary

`NonFungibleTokenPacketData.ValidateBasic()` carries a comment explicitly stating that address formats are **not** validated because counterparty chains may use different address schemes. Despite this, the function calls `sdk.AccAddressFromBech32()` on **both** the sender and receiver fields. When this chain acts as the IBC packet receiver, the sender address belongs to the counterparty chain and may be in any format (e.g., hex/Ethereum). The bech32 check will fail for any non-Cosmos sender, causing `OnRecvPacket` to return an error acknowledgement and permanently blocking inbound IBC NFT transfers from non-Cosmos counterparty chains.

---

### Finding Description

`ValidateBasic()` in `x/nft-transfer/types/packet.go` lines 38–71 contains a self-contradicting comment and implementation:

```go
// NOTE: The addresses formats are not validated as the sender and recipient can have different
// formats defined by their corresponding chains that are not known to IBC.
func (nftpd NonFungibleTokenPacketData) ValidateBasic() error {
    ...
    // decode the sender address
    if _, err := sdk.AccAddressFromBech32(nftpd.Sender); err != nil {
        return newsdkerrors.Wrap(sdkerrors.ErrInvalidAddress, "invalid sender address")
    }
    ...
    // decode the receiver address
    if _, err := sdk.AccAddressFromBech32(nftpd.Receiver); err != nil {
        return newsdkerrors.Wrap(sdkerrors.ErrInvalidAddress, "invalid receiver address")
    }
    return nil
}
```

The ICS-721 specification explicitly states that address formats must not be validated on the receiving chain because the sender address originates from the counterparty chain and may follow any encoding scheme. The comment in the code acknowledges this, yet the implementation enforces Cosmos bech32 for both fields.

`ValidateBasic()` is called unconditionally inside `keeper.OnRecvPacket()` at `x/nft-transfer/keeper/relay.go` line 105:

```go
func (k Keeper) OnRecvPacket(...) error {
    if err := data.ValidateBasic(); err != nil {
        return err
    }
    return k.processReceivedPacket(ctx, packet, data)
}
```

This error propagates back to `ibc_module.go` `OnRecvPacket`, which converts it into an error acknowledgement written on-chain.

---

### Impact Explanation

Any IBC NFT packet whose `sender` field contains a non-bech32 address (e.g., `0x...` Ethereum address, or any other non-Cosmos encoding) will be rejected with an error acknowledgement every time it is relayed to this chain. The receiving chain will never mint or unescrow the NFT for the intended recipient. The counterparty chain will refund the sender upon receiving the error ack, but the cross-chain NFT transfer is permanently non-functional for all non-Cosmos counterparty chains. The broken invariant is: **ICS-721 packets with a valid non-bech32 sender address are incorrectly rejected**, violating the ICS-721 protocol guarantee that cross-chain NFT movement works between heterogeneous chains.

---

### Likelihood Explanation

The ICS-721 standard is designed for interoperability with non-Cosmos chains. Any operator establishing an IBC channel between this chain and an Ethereum-compatible or other non-Cosmos chain will find that all inbound NFT transfers fail silently (from the user's perspective) with an error ack. The entry path requires only a standard `MsgTransfer` on the counterparty chain relayed by any relayer — no privileged access is needed.

---

### Recommendation

Remove the `sdk.AccAddressFromBech32` check on `nftpd.Sender` in `ValidateBasic()`. The sender address belongs to the counterparty chain and must not be validated against this chain's address format. The receiver address check is correct (since the receiver must be a valid address on this chain) and should be retained. This aligns the implementation with the ICS-721 spec and the function's own comment.

```go
// Remove these lines:
if _, err := sdk.AccAddressFromBech32(nftpd.Sender); err != nil {
    return newsdkerrors.Wrap(sdkerrors.ErrInvalidAddress, "invalid sender address")
}
```

---

### Proof of Concept

1. Chain A (non-Cosmos, e.g., Ethereum-based) establishes an ICS-721 channel with this chain.
2. A user on Chain A calls the NFT transfer entrypoint; the resulting IBC packet has `sender = "0xAbCd..."`.
3. A relayer submits the packet to this chain via `MsgRecvPacket`.
4. IBC core calls `ibc_module.go OnRecvPacket` → `keeper.OnRecvPacket` → `data.ValidateBasic()`.
5. `sdk.AccAddressFromBech32("0xAbCd...")` returns an error.
6. `ValidateBasic()` returns `"invalid sender address"`.
7. `OnRecvPacket` returns an error acknowledgement; the NFT is never minted/unescrowed on this chain.
8. The counterparty chain receives the error ack and refunds the sender, but the transfer is permanently blocked for all such senders.

```go
// ValidateBasic is used for validating the nft transfer.
// NOTE: The addresses formats are not validated as the sender and recipient can have different
// formats defined by their corresponding chains that are not known to IBC.
func (nftpd NonFungibleTokenPacketData) ValidateBasic() error {
	if strings.TrimSpace(nftpd.ClassId) == "" {
		return newsdkerrors.Wrap(ErrInvalidClassID, "classId cannot be blank")
	}

	if len(nftpd.TokenIds) == 0 {
		return newsdkerrors.Wrap(ErrInvalidTokenID, "tokenId cannot be blank")
	}

	if len(nftpd.TokenIds) != len(nftpd.TokenUris) {
		return newsdkerrors.Wrap(ErrInvalidPacket, "tokenIds and tokenUris lengths do not match")
	}

	if strings.TrimSpace(nftpd.Sender) == "" {
		return newsdkerrors.Wrap(sdkerrors.ErrInvalidAddress, "sender address cannot be blank")
	}

	// decode the sender address
	if _, err := sdk.AccAddressFromBech32(nftpd.Sender); err != nil {
		return newsdkerrors.Wrap(sdkerrors.ErrInvalidAddress, "invalid sender address")
	}

	if strings.TrimSpace(nftpd.Receiver) == "" {
		return newsdkerrors.Wrap(sdkerrors.ErrInvalidAddress, "receiver address cannot be blank")
	}

	// decode the receiver address
	if _, err := sdk.AccAddressFromBech32(nftpd.Receiver); err != nil {
		return newsdkerrors.Wrap(sdkerrors.ErrInvalidAddress, "invalid receiver address")
	}
	return nil
```

```go
func (im IBCModule) OnRecvPacket(
	ctx sdk.Context,
	channelVersion string,
	packet channeltypes.Packet,
	relayer sdk.AccAddress,
) ibcexported.Acknowledgement {
	ack := channeltypes.NewResultAcknowledgement([]byte{byte(1)})

	var data types.NonFungibleTokenPacketData
	if err := types.ModuleCdc.UnmarshalJSON(packet.GetData(), &data); err != nil {
		ack = channeltypes.NewErrorAcknowledgement(fmt.Errorf("cannot unmarshal ICS-721 nft-transfer packet data"))
	}

	// only attempt the application logic if the packet data
	// was successfully decoded
	if ack.Success() {
		if err := im.keeper.OnRecvPacket(ctx, channelVersion, packet, data); err != nil {
			ack = types.NewErrorAcknowledgement(err)
		}
	}
	sdkCtx := sdk.UnwrapSDKContext(ctx)
	sdkCtx.EventManager().EmitEvent(
		sdk.NewEvent(
			types.EventTypePacket,
			sdk.NewAttribute(sdk.AttributeKeyModule, types.ModuleName),
			sdk.NewAttribute(sdk.AttributeKeySender, data.Sender),
			sdk.NewAttribute(types.AttributeKeyReceiver, data.Receiver),
			sdk.NewAttribute(types.AttributeKeyClassID, data.ClassId),
			sdk.NewAttribute(types.AttributeKeyTokenIDs, strings.Join(data.TokenIds, ",")),
			sdk.NewAttribute(types.AttributeKeyAckSuccess, fmt.Sprintf("%t", ack.Success())),
		),
	)

	// NOTE: acknowledgement will be written synchronously during IBC handler execution.
	return ack
}
```
