The code path is real and the invariant violation is confirmed. Let me trace it precisely.

### Title
Bonus Checkpoints Advance Without Payment When Pool Is Empty During Redelegation Slash — (`x/tieredrewards/keeper/slash.go`, `x/tieredrewards/keeper/claim_rewards.go`)

---

### Summary

In `slashRedelegationPosition`, when `processEventsAndClaimBonus` returns `ErrInsufficientBonusPool`, the error is intentionally swallowed to avoid a chain halt. However, `processEventsAndClaimBonus` mutates the `pos` pointer's checkpoints (`LastEventSeq`, `LastBonusAccrual`, `LastKnownBonded`) **before** the pool balance check. The subsequent `setPositionWithState` call then persists those advanced checkpoints. The result is that earned bonus for the pre-slash period is permanently destroyed: the checkpoints advance as if bonus was paid, but no coins are transferred.

---

### Finding Description

**Step 1 — Checkpoint mutation happens before pool check in `processEventsAndClaimBonus`:**

Inside the event loop, `pos.UpdateLastEventSeq` is called per event: [1](#0-0) 

After the loop, `applyBonusAccrualCheckpoint` and `UpdateLastKnownBonded` advance the remaining checkpoints: [2](#0-1) 

Only **after** all three checkpoints are mutated does the function check pool balance: [3](#0-2) 

When the pool is empty, `sufficientBonusPoolBalance` returns `ErrInsufficientBonusPool` and the function returns — but the `pos` pointer already carries the advanced checkpoints. [4](#0-3) 

**Step 2 — `slashRedelegationPosition` swallows the error and persists the mutated position:**

`processEventsAndClaimBonus` is called with `&pos` (pointer): [5](#0-4) 

`ErrInsufficientBonusPool` is logged but not returned. Execution falls through to `setPositionWithState`, which persists `pos` — now carrying the advanced checkpoints — to the store: [6](#0-5) 

**Step 3 — The pre-slash bonus period is permanently erased:**

`LastBonusAccrual` is now set to the slash block time. On the next call to `processEventsAndClaimBonus` (e.g., via `MsgClaimTierRewards` after the pool is refilled), `segmentStart` is initialized from the persisted `LastBonusAccrual`: [7](#0-6) 

The pre-slash segment `[original_LastBonusAccrual, slash_block_time]` is gone. No amount of pool refilling can recover it.

---

### Impact Explanation

A position owner whose redelegation is slashed while the bonus pool is empty permanently loses all bonus rewards accrued since their last checkpoint. The loss is proportional to `shares × tokensPerShare × bonusApy × elapsed_duration`. This is a direct, unrecoverable destruction of earned rewards — not a deferral.

The ADR documents this as an intentional trade-off ("chain-halt avoidance"), but the implementation is incorrect: it advances checkpoints as if payment succeeded, rather than leaving them at the pre-slash position so the bonus can be claimed once the pool is refilled. [8](#0-7) 

---

### Likelihood Explanation

The bonus pool can be empty at any time — it is funded externally and depleted by normal reward claims. A validator slash is a normal chain event (double-sign, downtime). The intersection of these two conditions is realistic in production, particularly during pool depletion periods or shortly after chain launch before the pool is seeded.

---

### Recommendation

Move `applyBonusAccrualCheckpoint` and `UpdateLastKnownBonded` to **after** the successful `SendCoinsFromModuleToAccount` call, or — if the forfeit-on-empty-pool design is intentional — explicitly reset the in-memory `pos` checkpoints back to their pre-call values when `ErrInsufficientBonusPool` is caught in `slashRedelegationPosition`, so that `setPositionWithState` persists the original (unpaid) checkpoint state rather than the advanced one.

---

### Proof of Concept

```
1. Create a tier position and redelegate it to a second validator.
2. Advance block time by 30 days (bonus accrues).
3. Ensure the bonus pool is empty (do not fund it).
4. Call keeper.Hooks().BeforeRedelegationSlashed(ctx, unbondingID, partialShares).
5. Assert: no error returned (ErrInsufficientBonusPool is swallowed).
6. Read the persisted position: LastBonusAccrual == slash block time (advanced).
7. Fund the bonus pool.
8. Call MsgClaimTierRewards for the position.
9. Assert: bonus received == 0 for the pre-slash period.
   (Expected: bonus > 0 for the 30-day pre-slash segment.)
```

This matches the existing test structure in `x/tieredrewards/keeper/slash_test.go` — simply omit the `s.fundRewardsPool` call and add the post-refill claim assertion. [9](#0-8)

func (k Keeper) sufficientBonusPoolBalance(ctx context.Context, bonus sdk.Coins) error {
	if bonus.IsZero() {
		return nil
	}

	poolAddr := k.accountKeeper.GetModuleAddress(types.RewardsPoolName)
	poolBalance := k.bankKeeper.GetAllBalances(ctx, poolAddr)

		return errorsmod.Wrapf(types.ErrInsufficientBonusPool,
			"bonus: %s, pool balance: %s",
			bonus.String(), poolBalance.String())
	}

	return nil

// TestSlashRedelegationPosition_ClaimsBonusRewardsUpToSlash verifies that when
// a redelegation slash fires, any bonus accrued on the destination delegation
// since the last accrual checkpoint is paid out to the position owner, and the
// position's bonus-state checkpoints (LastBonusAccrual, LastKnownBonded) are
// advanced.
func (s *KeeperSuite) TestSlashRedelegationPosition_ClaimsBonusRewardsUpToSlash() {
	lockAmount := sdkmath.NewInt(sdk.DefaultPowerReduction.Int64())
	_, bondDenom := s.getStakingData()
	s.fundRewardsPool(sdkmath.NewInt(1_000_000_000), bondDenom)

	pos, _, unbondingID := s.setupRedelegatingPosition(lockAmount)
	owner := sdk.MustAccAddressFromBech32(pos.Owner)
	preAccrual := pos.LastBonusAccrual

	// Advance block time so bonus accrues on the destination validator.
	s.ctx = s.ctx.WithBlockHeight(s.ctx.BlockHeight() + 1)
	s.ctx = s.ctx.WithBlockTime(s.ctx.BlockTime().Add(30 * 24 * time.Hour))

	balBefore := s.app.BankKeeper.GetBalance(s.ctx, owner, bondDenom)

	// Partial slash — a small fraction of shares.
	sharesToUnbond := pos.Delegation.Shares.Quo(sdkmath.LegacyNewDec(10))
	err := s.keeper.Hooks().BeforeRedelegationSlashed(s.ctx, unbondingID, sharesToUnbond)
	s.Require().NoError(err)

	balAfter := s.app.BankKeeper.GetBalance(s.ctx, owner, bondDenom)
	s.Require().True(balAfter.Amount.GT(balBefore.Amount),
		"owner should have received bonus rewards accrued up to slash: before=%s after=%s",
		balBefore.Amount, balAfter.Amount)

	updated, err := s.keeper.GetPositionState(s.ctx, pos.Id)
	s.Require().NoError(err)
	s.Require().True(updated.LastBonusAccrual.After(preAccrual),
		"LastBonusAccrual should have advanced past the pre-slash checkpoint")
	s.Require().Equal(s.ctx.BlockTime(), updated.LastBonusAccrual,
		"LastBonusAccrual should advance to the slash block time")
	s.Require().True(updated.LastKnownBonded,
		"LastKnownBonded should remain true — destination validator is still bonded")
}
