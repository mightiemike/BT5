Audit Report

## Title
`SwapAllowlistExtension` gates the router address instead of the actual user, allowing any user to bypass the per-user swap allowlist via `MetricOmmSimpleRouter` — (`metric-periphery/contracts/extensions/SwapAllowlistExtension.sol`)

## Summary
`MetricOmmPool.swap` unconditionally passes `msg.sender` as the `sender` argument to `_beforeSwap`. When users route through `MetricOmmSimpleRouter`, `msg.sender` to the pool is the router contract, not the end user. A pool admin who allowlists the router to enable router-mediated swaps for their curated users inadvertently grants swap access to every caller of the router, completely neutralising the per-user allowlist.

## Finding Description
`MetricOmmPool.swap` hardcodes `msg.sender` as the `sender` forwarded to every `beforeSwap` extension: [1](#0-0) 

`SwapAllowlistExtension.beforeSwap` then checks that exact value against its per-pool allowlist: [2](#0-1) 

`MetricOmmSimpleRouter.exactInputSingle` (and `exactOutputSingle`, `exactInput`, `exactOutput`) calls `pool.swap()` directly, making the router the `msg.sender` to the pool: [3](#0-2) 

The extension therefore evaluates `allowedSwapper[pool][router]` instead of `allowedSwapper[pool][user]`. A pool admin who wants their allowlisted users to use the standard periphery has only one option: `setAllowedToSwap(pool, router, true)`. The moment they do so, `allowedSwapper[pool][router] = true` and the check passes for **every** caller of the router, regardless of whether that caller is on the per-user allowlist. There is no mechanism in the current design that lets a pool admin simultaneously restrict swaps to a named set of users and allow those users to use `MetricOmmSimpleRouter`.

## Impact Explanation
A curated pool intended to restrict trading to a specific set of counterparties (e.g., KYC'd addresses, whitelisted market-makers) can be accessed by any arbitrary user the moment the pool admin allowlists the router. This is a broken core pool access-control invariant: the `SwapAllowlistExtension` is rendered entirely ineffective for router-mediated swaps, constituting an admin-boundary break where an unprivileged trader bypasses a pool-level restriction set by the pool admin.

## Likelihood Explanation
Exploitation requires no special privileges. Any user can call `MetricOmmSimpleRouter.exactInputSingle` (or any other router entry-point) targeting a pool that has allowlisted the router. The router is the standard periphery contract, so pool admins enabling router access for their users is the expected operational pattern. The condition is therefore likely to be met in production deployments.

## Recommendation
Pass the original end-user address through the call chain rather than `msg.sender`. One approach: encode the originating user in `extensionData` and have the extension read it, but this is forgeable by any caller. A safer approach is to have the pool accept an explicit `sender` parameter (verified via a trusted router registry or a signed context), or have the router pass the real user address through a dedicated field that the pool forwards to extensions instead of `msg.sender`. Alternatively, the extension can be made router-aware: it checks whether `sender` is a known router and, if so, reads the actual payer from a router-provided context (e.g., transient storage written by the router before calling `pool.swap`).

## Proof of Concept
1. Deploy a pool with `SwapAllowlistExtension` configured; `allowAllSwappers[pool] = false`.
2. Pool admin calls `setAllowedToSwap(pool, alice, true)` — only Alice is meant to trade.
3. Pool admin calls `setAllowedToSwap(pool, router, true)` — to let Alice use the standard router.
4. Bob (not allowlisted) calls `MetricOmmSimpleRouter.exactInputSingle({pool: pool, ...})`.
5. The router calls `pool.swap(...)` with `msg.sender = router`.
6. `_beforeSwap(router, ...)` is dispatched; `allowedSwapper[pool][router] == true` → check passes.
7. Bob's swap executes successfully despite not being on the per-user allowlist.

Foundry test sketch:
```solidity
vm.prank(bob); // bob is NOT allowlisted
router.exactInputSingle(ExactInputSingleParams({pool: pool, ...}));
// expect: succeeds (should revert NotAllowedToSwap)
```
