### Title
`SwapAllowlistExtension` Checks Router Address Instead of End-User, Allowing Any EOA to Bypass Per-User Swap Allowlist — (`metric-periphery/contracts/extensions/SwapAllowlistExtension.sol`)

---

### Summary

`SwapAllowlistExtension.beforeSwap` gates swaps by checking `allowedSwapper[pool][sender]`, where `sender` is the value passed by the pool as `msg.sender` at the time `pool.swap()` was called. When a user routes through `MetricOmmSimpleRouter`, `msg.sender` to the pool is always the **router contract**, not the originating EOA. Any pool admin who allowlists the router to permit router-based swaps inadvertently grants swap access to every EOA that calls through the router, regardless of whether those EOAs are individually allowlisted.

---

### Finding Description

The call chain is:

```
EOA → MetricOmmSimpleRouter.exactInputSingle()
        └─ pool.swap(recipient, ...)          // msg.sender to pool = router
              └─ _beforeSwap(msg.sender=router, ...)
                    └─ SwapAllowlistExtension.beforeSwap(sender=router, ...)
                          └─ checks allowedSwapper[msg.sender/*pool*/][sender/*router*/]
```

**Step 1 — Router calls pool with itself as `msg.sender`:**

`exactInputSingle` calls `pool.swap(...)` directly. The pool sees `msg.sender` = router. [1](#0-0) 

**Step 2 — Pool passes `msg.sender` (router) as `sender` to the hook:**

`MetricOmmPool.swap` calls `_beforeSwap(msg.sender, ...)`, binding `sender` = router. [2](#0-1) 

**Step 3 — Extension checks `allowedSwapper[pool][router]`, not `allowedSwapper[pool][EOA]`:**

`SwapAllowlistExtension.beforeSwap` uses `msg.sender` (= pool) as the pool key and `sender` (= router) as the swapper key. The originating EOA is never consulted. [3](#0-2) 

The existing test suite confirms this binding: `test_allowedSwapSucceeds` allowlists `callers[0]` (the intermediate contract that calls `pool.swap`) and the EOA `users[0]` is never in the allowlist, yet the swap succeeds. [4](#0-3) 

---

### Impact Explanation

A pool admin who deploys a restricted pool (e.g., a permissioned institutional pool) and uses `SwapAllowlistExtension` to gate individual counterparties **must** allowlist the router address to support router-based swaps. Once the router is allowlisted, every EOA on the network can call `exactInputSingle` through the router and trade against the pool's LP reserves, because the extension only checks `allowedSwapper[pool][router]` = `true`. The per-user gate is completely bypassed. LPs in the restricted pool suffer direct fund loss from trades executed by unauthorized counterparties.

---

### Likelihood Explanation

The `MetricOmmSimpleRouter` is the primary public swap interface. Any user who discovers the allowlist can trivially route through the router. No privileged access, special tokens, or unusual parameters are required — a standard `exactInputSingle` call suffices.

---

### Recommendation

Pass the originating user identity through the swap path so the extension can gate on the true end-user. Two options:

1. **Preferred:** Have `MetricOmmSimpleRouter` encode the originating `msg.sender` into `extensionData` and have `SwapAllowlistExtension.beforeSwap` decode and check it when `sender` is a known router. This requires a trusted router registry or a signed-origin pattern.

2. **Alternative:** Add a `recipient`-based check as a fallback: if `sender` is a router, check `allowedSwapper[pool][recipient]` instead. This is weaker (recipient is caller-controlled) but prevents the trivial bypass.

The root fix is that `sender` in `beforeSwap` must represent the economically responsible party, not the intermediate contract.

---

### Proof of Concept

```solidity
// Setup: pool with SwapAllowlistExtension, only router is allowlisted
swapExtension.setAllowedToSwap(address(pool), address(router), true);
// EOA (not in allowlist) calls through router
vm.prank(nonAllowlistedEOA);
router.exactInputSingle(ExactInputSingleParams({
    pool: address(pool),
    tokenIn: token0,
    tokenOut: token1,
    zeroForOne: true,
    amountIn: 1000,
    amountOutMinimum: 0,
    recipient: nonAllowlistedEOA,
    deadline: type(uint256).max,
    priceLimitX64: 0,
    extensionData: ""
}));
// Swap succeeds: allowedSwapper[pool][router] = true, EOA never checked
assertFalse(swapExtension.isAllowedToSwap(address(pool), nonAllowlistedEOA)); // true — EOA not allowlisted
// but swap went through anyway
```
