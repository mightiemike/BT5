### Title
Validator Mnemonic Written to Disk in Plaintext JSON — (`File: x/chainmain/client/cli/testnet.go`)

### Summary
The `InitTestnet` function in `x/chainmain/client/cli/testnet.go` generates a BIP39 mnemonic for each validator node and immediately serializes it into a plaintext JSON file (`key_seed.json`) on the filesystem without any encryption. Any party with read access to the output directory — including backup systems, co-located processes running as the same OS user, or root — can recover the full mnemonic and reconstruct the private key to drain the validator's staked funds.

### Finding Description
Inside `InitTestnet`, for each validator node, a fresh mnemonic is generated via `kb.NewMnemonic()`. The returned `secret` (the raw BIP39 mnemonic string) is placed directly into a plain Go `map[string]string` and marshalled to JSON: [1](#0-0) 

The resulting JSON object — `{"secret":"<24-word mnemonic>","addr":"<bech32>"}` — is written to `<outputDir>/<nodeN>/<daemonHome>/key_seed.json` via `writeFile`: [2](#0-1) 

`writeFile` applies `0o600` permissions (owner read/write only), which is a partial mitigation, but the file content itself is completely unencrypted plaintext. The mnemonic is the root secret from which the validator's secp256k1 private key is derived; possessing it is equivalent to possessing the private key.

Additionally, the default keyring backend for the `testnet` command is `keyring.BackendTest`: [3](#0-2) 

`BackendTest` itself stores keys unencrypted on disk, compounding the exposure: both the keyring files and `key_seed.json` are plaintext.

### Impact Explanation
An attacker who obtains read access to the output directory — through a misconfigured backup, a shared NFS mount, a co-located process running as the same OS user, or root access — can read `key_seed.json`, recover the 24-word mnemonic, and use `chain-maind keys add --recover` to reconstruct the validator's account key. From there they can sign and broadcast a `MsgSend` or `MsgDelegate`/`MsgUndelegate` transaction to drain or redirect the validator's bonded and liquid CRO funds. The corrupted value is the validator account balance and delegation shares.

### Likelihood Explanation
The `testnet` command is the standard operator-facing CLI path for bootstrapping multi-validator networks. The output directory is frequently archived, transferred over scp/rsync, or stored in shared infrastructure. The `0o600` permission only prevents other non-root OS users on the same host; it does not protect against root, backup agents, or any process running as the same user. The plaintext mnemonic persists indefinitely on disk unless manually deleted.

### Recommendation
- Do not write the raw mnemonic to disk at all. If a seed backup is required, encrypt it with a user-supplied passphrase (e.g., using `golang.org/x/crypto/nacl/secretbox` or `age`) before writing.
- At minimum, document clearly that `key_seed.json` contains unencrypted key material and must be deleted after the operator has recorded the mnemonic securely.
- Consider switching the default keyring backend from `BackendTest` to `BackendFile` or `BackendOS` so that keyring entries are also encrypted at rest.

### Proof of Concept
```
# 1. Operator runs the testnet init command
chain-maind testnet --v 4 --output-dir ./mytestnet

# 2. Attacker reads the plaintext mnemonic from any node's key_seed.json
cat ./mytestnet/node0/.chain-maind/key_seed.json
# Output: {"addr":"cro1...","secret":"word1 word2 ... word24"}

# 3. Attacker imports the mnemonic into their own keyring
echo "word1 word2 ... word24" | chain-maind keys add stolen --recover --keyring-backend test

# 4. Attacker drains the validator account
chain-maind tx bank send stolen <attacker_addr> 1000000000000basecro \
  --chain-id <chainID> --keyring-backend test --fees 10000basecro
```

func writeFile(name, dir string, contents []byte) error {
	file := filepath.Join(dir, name)

	err := tmos.EnsureDir(dir, 0o755)
	if err != nil {
		return err
	}

	err = tmos.WriteFile(file, contents, 0o600)
	if err != nil {
		return err
	}

	return nil
}
