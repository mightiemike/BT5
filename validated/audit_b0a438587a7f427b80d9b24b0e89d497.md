### Title
Unchecked `transferFrom` Return Value in `replaceUsdcEWithUsdc` Enables Free Drain of USDC.e from User DDAs — (File: `core/contracts/ContractOwner.sol`)

---

### Summary

`ContractOwner.replaceUsdcEWithUsdc()` calls `IERC20Base(usdc).transferFrom(msg.sender, directDepositV1, balance)` without checking the boolean return value. Because the function has no access control beyond a chain-ID check, any unprivileged caller can invoke it. If the `transferFrom` silently returns `false` (e.g., zero allowance, non-standard token), execution continues: the USDC.e is still withdrawn from the victim's `DirectDepositV1` (DDA) and forwarded to the caller — with no USDC ever deposited in exchange.

---

### Finding Description

`replaceUsdcEWithUsdc` is designed as a migration helper: it accepts USDC from the caller, deposits it into a target DDA, then withdraws the DDA's USDC.e balance and returns it to the caller. The three-step sequence is:

1. Pull USDC from `msg.sender` into `directDepositV1` via `transferFrom` ← **return value ignored**
2. Withdraw USDC.e from `directDepositV1` to `ContractOwner` via `DirectDepositV1.withdraw()`
3. Push USDC.e from `ContractOwner` to `msg.sender` via `safeTransfer` [1](#0-0) 

Steps 2 and 3 execute unconditionally regardless of whether step 1 succeeded. The contract already imports and uses `ERC20Helper` (which provides a `safeTransferFrom` that checks the return value), and even uses `safeTransfer` on the very next line (618), making the omission on line 616 a clear inconsistency. [2](#0-1) 

The `using ERC20Helper for IERC20Base` directive is active in `ContractOwner`, so `safeTransferFrom` was available and simply not used. [3](#0-2) 

---

### Impact Explanation

An attacker with zero USDC and zero allowance calls `replaceUsdcEWithUsdc(victimSubaccount)` on chain ID 57073 (Ink). The `transferFrom` returns `false` and is ignored. Steps 2–3 still execute: the victim's USDC.e is pulled out of their DDA and transferred to the attacker. The attacker receives the full USDC.e balance of any targeted DDA for free. This is a direct, complete theft of user collateral held in DDAs.

---

### Likelihood Explanation

The function is `external` with no `onlyOwner` or `onlyDeployer` guard — the sole restriction is `block.chainid == 57073`. [4](#0-3) 

Any address on the Ink chain can call it. The attacker only needs to know a subaccount whose DDA holds USDC.e (discoverable on-chain via `directDepositV1Address` mapping and `balanceOf`). No privileged access, no leaked keys, no governance capture required. Likelihood is **high**.

---

### Recommendation

Replace the raw `transferFrom` call on line 616 with the safe wrapper already available in scope:

```solidity
// Before (unsafe):
IERC20Base(usdc).transferFrom(msg.sender, directDepositV1, balance);

// After (safe):
IERC20Base(usdc).safeTransferFrom(msg.sender, directDepositV1, balance);
```

`safeTransferFrom` in `ERC20Helper` low-level calls the token and requires both `success == true` and a truthy decoded return value, reverting on failure. [2](#0-1) 

---

### Proof of Concept

1. Identify a subaccount `S` whose DDA holds `N` USDC.e (e.g., `N = 1000e6`).
2. Deploy or use any EOA with **zero USDC balance and zero USDC allowance** to `ContractOwner`.
3. Call `ContractOwner.replaceUsdcEWithUsdc(S)` from that address on chain ID 57073.
4. `IERC20Base(usdc).transferFrom(attacker, dda, 1000e6)` returns `false` (no allowance); return value is discarded.
5. `DirectDepositV1(dda).withdraw(usdcE)` transfers 1000 USDC.e from the DDA to `ContractOwner`.
6. `IERC20Base(usdcE).safeTransfer(attacker, 1000e6)` transfers 1000 USDC.e to the attacker.
7. Attacker receives 1000 USDC.e; victim's DDA is emptied; no USDC was ever deposited. [5](#0-4)

```text
    function replaceUsdcEWithUsdc(bytes32 subaccount) external {
        require(block.chainid == 57073, ERR_UNAUTHORIZED);
        address payable directDepositV1 = directDepositV1Address[subaccount];
        require(directDepositV1 != address(0), "no dda");
        address usdcE = 0xF1815bd50389c46847f0Bda824eC8da914045D14;
        address usdc = 0x2D270e6886d130D724215A266106e6832161EAEd;
        uint256 balance = IERC20Base(usdcE).balanceOf(directDepositV1);
        if (balance > 0) {
            IERC20Base(usdc).transferFrom(msg.sender, directDepositV1, balance);
            DirectDepositV1(directDepositV1).withdraw(IIERC20Base(usdcE));
            IERC20Base(usdcE).safeTransfer(msg.sender, balance);
        }
    }
```

**File:** core/contracts/libraries/ERC20Helper.sol (L23-42)
```text
    function safeTransferFrom(
        IERC20Base self,
        address from,
        address to,
        uint256 amount
    ) internal {
        (bool success, bytes memory data) = address(self).call(
            abi.encodeWithSelector(
                IERC20Base.transferFrom.selector,
                from,
                to,
                amount
            )
        );

        require(
            success && (data.length == 0 || abi.decode(data, (bool))),
            ERR_TRANSFER_FAILED
        );
    }
```
