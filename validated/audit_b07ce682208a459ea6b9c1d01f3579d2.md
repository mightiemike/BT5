### Title
Hardcoded `LastKnownBonded = true` in `createDelegatedPosition` Allows Unbacked Bonus Drain When Position Is Created on an Unbonded Validator — (`x/tieredrewards/keeper/position.go`)

---

### Summary

`createDelegatedPosition` unconditionally initialises `LastKnownBonded = true` for every new position. `validateNewPosition` (called by both `LockTier` and `CommitDelegationToTier`) contains no check that the target validator is currently bonded. When a position is created while the validator is unbonded, the stale `true` value causes `processEventsAndClaimBonus` to treat the entire period from position-creation to the first subsequent BOND event as a bonded segment and pay bonus for it, even though the validator was never bonded during that window.

---

### Finding Description

**Root cause — hardcoded initialisation:**

`createDelegatedPosition` passes the literal `true` as the `lastKnownBonded` argument to `NewPosition`:

```go
// x/tieredrewards/keeper/position.go  line 56
pos := types.NewPosition(id, owner, tier.Id, delAddr.String(),
    blockHeight, lastEventSeq, blockTime, true /*lastKnownBonded*/, blockTime)
``` [1](#0-0) 

The value is never derived from the live validator status.

**No bonded-status guard in `validateNewPosition`:**

```go
// x/tieredrewards/keeper/msg_validate.go  lines 28-42
func (k Keeper) validateNewPosition(ctx context.Context, owner string, amount math.Int, tier types.Tier) error {
    if err := k.validateNonVestingAccount(ctx, owner); err != nil { return err }
    if tier.IsCloseOnly() { return types.ErrTierIsCloseOnly }
    if !tier.MeetsMinLockRequirement(amount) { return types.ErrMinLockAmountNotMet }
    return nil
}
``` [2](#0-1) 

There is no call to `stakingKeeper.GetValidator` and no rejection when the validator is unbonded or unbonding.

**Bonus computation reads the stale flag:**

```go
// x/tieredrewards/keeper/claim_rewards.go  line 164
bonded := pos.LastKnownBonded   // starts as true even if validator is unbonded
segmentStart := pos.LastBonusAccrual

    switch evt.EventType {
    case UNBOND: bonded = false
    case BOND:   bonded = true
    }
    segmentStart = evt.Timestamp
}
``` [3](#0-2) 

**BOND event hook only fires when `count > 0`:**

```go

``` [4](#0-3) 

Once the attacker's position exists, `count ≥ 1`, so the BOND event is recorded and the position will process it.

---

### Impact Explanation

When `processEventsAndClaimBonus` walks events starting from `pos.LastEventSeq` (set to the latest seq at creation time), the first new event it sees is the BOND event. Because `bonded` starts as `true`, the loop body executes the bonus computation for the segment `[creation_time, bond_event_time]` — a period during which the validator was entirely unbonded. The resulting `bonusCoins` are sent from `types.RewardsPoolName` to the attacker's address:

```go
// x/tieredrewards/keeper/claim_rewards.go  line 239
k.bankKeeper.SendCoinsFromModuleToAccount(ctx, types.RewardsPoolName, ownerAddr, bonusCoins)
``` [5](#0-4) 

The longer the validator remains unbonded between position creation and rebonding, the larger the unbacked payout. The `RewardsPoolName` module account balance is directly drained.

---

### Likelihood Explanation

The precondition — a validator that is currently unbonded but will eventually bond — is routine: jailed validators that get unjailed, newly-created validators before their first bonding, or validators that temporarily fall out of the active set. The Cosmos SDK `Delegate` call inside `LockTier` succeeds for unbonded validators without error. No governance or privileged role is required; any user can call `MsgLockTier` with an unbonded validator address.

---

### Recommendation

In `createDelegatedPosition`, query the live validator status and initialise `LastKnownBonded` from it:

```go
val, err := k.stakingKeeper.GetValidator(ctx, valAddr)
if err != nil { return types.Position{}, err }
lastKnownBonded := val.IsBonded()

pos := types.NewPosition(id, owner, tier.Id, delAddr.String(),
    blockHeight, lastEventSeq, blockTime, lastKnownBonded, blockTime)
```

Alternatively (and more defensively), reject `LockTier` / `CommitDelegationToTier` outright when the target validator is not bonded by adding a check in `validateNewPosition`.

The same hardcoded `true` appears in `TierRedelegate` at line 263 of `msg_server.go` and should be fixed identically for the destination validator. [6](#0-5)

### Proof of Concept

```
1. Jail validator V so it transitions to unbonded state
   → AfterValidatorBeginUnbonding fires, UNBOND event at seq N recorded
   → ValidatorEventSeq[V] = N

2. Attacker calls MsgLockTier{ValidatorAddress: V, Amount: minLock}
   → validateNewPosition: passes (no bonded check)
   → delegate(delAddr, V, amount): succeeds (SDK allows delegating to unbonded validators)
   → createDelegatedPosition: LastEventSeq = N, LastKnownBonded = true  ← BUG
   → PositionCountByValidator[V] = 1

3. Advance time by D days (validator still unbonded)

4. Unjail V → ApplyAndReturnValidatorSetUpdates → AfterValidatorBonded fires
   → count = 1 > 0, so BOND event at seq N+1 recorded with ReferenceCount = 1

5. Attacker calls MsgClaimTierRewards{PositionIds: [posId]}
   → processEventsAndClaimBonus:
       bonded = pos.LastKnownBonded = true   ← stale
       segmentStart = creation_time
       events = [BOND at seq N+1, timestamp = bond_time]
       loop iteration 1:
           bonded == true → bonus += computeSegmentBonus(creation_time, bond_time, rate)
                            ← pays for D days of unbonded period  ← WRONG
           bonded = true (BOND event)
           segmentStart = bond_time
       final segment: bonded && val.IsBonded() → pays for [bond_time, now]  ← correct
   → SendCoinsFromModuleToAccount(RewardsPoolName, attacker, bonus)

Assert: bonus for [creation_time, bond_time] > 0 even though validator was unbonded
```

func (h Hooks) AfterValidatorBonded(ctx context.Context, _ sdk.ConsAddress, valAddr sdk.ValAddress) error {
	count, err := h.k.getPositionCountForValidator(ctx, valAddr)
	if err != nil {
		return err
	}
	if count == 0 {
		return nil
	}

	tokensPerShare, err := h.k.getTokensPerShare(ctx, valAddr)
	if err != nil {
		return err
	}

	sdkCtx := sdk.UnwrapSDKContext(ctx)
	_, err = h.k.appendValidatorEvent(ctx, valAddr, types.ValidatorEvent{
		Height:         sdkCtx.BlockHeight(),
		Timestamp:      sdkCtx.BlockTime(),
		EventType:      types.ValidatorEventType_VALIDATOR_EVENT_TYPE_BOND,
		TokensPerShare: tokensPerShare,
		ReferenceCount: count,
	})
	return err
}
