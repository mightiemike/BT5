### Title
P2P Protobuf Deserialization Silently Coerces `AllResources` to `L1Gas` Variant, Producing Wrong Transaction Hash During Sync - (`crates/apollo_protobuf/src/converters/transaction.rs`)

---

### Summary

The protobuf deserializer for `ValidResourceBounds` in the P2P sync path silently converts an `AllResources` variant (with zero `l2_gas` and `l1_data_gas`) into the `L1Gas` variant. Because `get_tip_resource_bounds_hash` hashes a different number of Poseidon elements for each variant, any V3 transaction whose `AllResourceBounds` has `l2_gas = 0` and `l1_data_gas = 0` will have its hash recomputed to a different value on a syncing peer than the hash that was computed and stored by the originating node. The codebase itself acknowledges this coercion in a comment inside the consensus round-trip test.

---

### Finding Description

**Serialization boundary — `TryFrom<protobuf::ResourceBounds> for ValidResourceBounds`** [1](#0-0) 

```rust
Ok(if l1_data_gas.is_zero() && l2_gas.is_zero() {
    ValidResourceBounds::L1Gas(l1_gas)          // ← coercion
} else {
    ValidResourceBounds::AllResources(AllResourceBounds { l1_gas, l2_gas, l1_data_gas })
})
```

A transaction submitted via RPC carries `resource_bounds: AllResourceBounds` (the `RpcInvokeTransactionV3` struct always uses `AllResourceBounds`). [2](#0-1) 

When the gateway converts it to `InternalRpcInvokeTransactionV3`, the `AllResourceBounds` is preserved and the hash is computed via `get_invoke_transaction_v3_hash` → `get_tip_resource_bounds_hash(ValidResourceBounds::AllResources(...))`. [3](#0-2) 

For `AllResources`, the hash includes **three** packed resource felts (L1_GAS, L2_GAS, L1_DATA_GAS):

```
poseidon(tip, pack(l1_gas,"L1_GAS"), pack(0,"L2_GAS"), pack(0,"L1_DATA"))  → H_all
```

For `L1Gas`, only **two** packed resource felts are included:

```
poseidon(tip, pack(l1_gas,"L1_GAS"), pack(0,"L2_GAS"))                     → H_l1
```

`H_all ≠ H_l1` because Poseidon is sensitive to the number of absorbed elements.

**P2P sync path uses `InvokeTransactionV3` which carries `ValidResourceBounds`**

When a block is propagated or synced over P2P, the stored transaction type is `InvokeTransactionV3` with `resource_bounds: ValidResourceBounds`. [4](#0-3) 

Its `calculate_transaction_hash` calls `get_invoke_transaction_v3_hash(self, ...)` which passes `self.resource_bounds` directly to `get_tip_resource_bounds_hash`. [5](#0-4) 

After protobuf round-trip, `self.resource_bounds` is `ValidResourceBounds::L1Gas(l1_gas)` (coerced), so the syncing node computes `H_l1` while the originating node stored `H_all`.

**The codebase explicitly acknowledges this coercion** [6](#0-5) 

```rust
// If all the fields of `AllResources` are 0 upon serialization,
// then the deserialized value will be interpreted as the `L1Gas` variant.
fn add_gas_values_to_transaction(transactions: &mut [ConsensusTransaction]) {
    ...
    resource_bounds.l2_gas.max_amount = GasAmount(1);  // workaround
```

The test workaround sets `l2_gas.max_amount = 1` to avoid the coercion. No such guard exists in production code paths.

**Trigger condition**

A V3 transaction with `AllResourceBounds { l1_gas: X, l2_gas: 0, l1_data_gas: 0 }` is valid and accepted by the gateway. The stateless validator explicitly allows transactions where only `l1_gas` is non-zero: [7](#0-6) 

---

### Impact Explanation

A syncing node that receives a block containing a V3 transaction with `AllResourceBounds { l1_gas: X, l2_gas: 0, l1_data_gas: 0 }` will:

1. Deserialize `ValidResourceBounds::L1Gas(X)` from protobuf (coercion).
2. Recompute the transaction hash as `H_l1`.
3. The originating node stored and committed the block with `H_all`.
4. `validate_transaction_hash` will fail: the recomputed hash does not match the committed hash. [8](#0-7) 

This causes the syncing node to reject a valid block or store the transaction under the wrong hash, breaking the transaction commitment invariant. This matches the **Critical** impact category: *Wrong state, receipt, event, L1 message, class hash, storage value, or revert result from blockifier/syscall/execution logic for accepted input.*

---

### Likelihood Explanation

- The trigger is a valid, unprivileged RPC transaction with only `l1_gas` set (a common pattern for pre-0.13.3-style transactions submitted via the new API).
- No special permissions or malicious peer are required; any user can submit such a transaction.
- The coercion is unconditional and silent — no error is returned.
- The codebase already has a workaround in tests, confirming the condition is reachable.

---

### Recommendation

**Short term:** In `TryFrom<protobuf::ResourceBounds> for ValidResourceBounds`, do not use the zero-check heuristic to decide the variant. Instead, use an explicit discriminant field in the protobuf schema, or always deserialize as `AllResources` when all three fields are present (even if zero). The `AllResourceBounds` converter in `rpc_transaction.rs` already does this correctly by requiring all three fields without any coercion. [9](#0-8) 

**Long term:** Add a round-trip hash-preservation test: serialize an `InvokeTransactionV3` with `AllResources { l1_gas: X, l2_gas: 0, l1_data_gas: 0 }` to protobuf, deserialize it, recompute the hash, and assert it equals the original hash.

    AllResourceBounds, ResourceBounds, ValidResourceBounds, GasAmount, GasPrice,
};
use starknet_api::transaction_hash::get_tip_resource_bounds_hash;
use starknet_api::transaction::fields::Tip;

// Transaction as submitted via RPC: AllResources with l2_gas=0, l1_data_gas=0
let all_resources = ValidResourceBounds::AllResources(AllResourceBounds {
    l1_gas: ResourceBounds { max_amount: GasAmount(1000), max_price_per_unit: GasPrice(1) },
    l2_gas: ResourceBounds::default(),       // zero
    l1_data_gas: ResourceBounds::default(),  // zero
});

// Hash computed by originating node (AllResources path: 3 elements)
let h_all = get_tip_resource_bounds_hash(&all_resources, &Tip(0)).unwrap();

// After protobuf round-trip, coerced to L1Gas
let l1_gas_only = ValidResourceBounds::L1Gas(
    ResourceBounds { max_amount: GasAmount(1000), max_price_per_unit: GasPrice(1) },
);

// Hash computed by syncing node (L1Gas path: 2 elements)
let h_l1 = get_tip_resource_bounds_hash(&l1_gas_only, &Tip(0)).unwrap();

// These differ — the syncing node computes the wrong transaction hash
assert_ne!(h_all, h_l1);  // PASSES — confirms the divergence

pub fn get_tip_resource_bounds_hash(
    resource_bounds: &ValidResourceBounds,
    tip: &Tip,
) -> Result<Felt, StarknetApiError> {
    let l1_resource_bounds = resource_bounds.get_l1_bounds();
    let l2_resource_bounds = resource_bounds.get_l2_bounds();

    // L1 and L2 gas bounds always exist.
    // Old V3 txs always have L2 gas bounds of zero, but they exist.
    let mut resource_felts = vec![
        get_concat_resource(&l1_resource_bounds, L1_GAS)?,
        get_concat_resource(&l2_resource_bounds, L2_GAS)?,
    ];

    // For new V3 txs, need to also hash the data gas bounds.
    resource_felts.extend(match resource_bounds {
        ValidResourceBounds::L1Gas(_) => vec![],
        ValidResourceBounds::AllResources(all_resources) => {
            vec![get_concat_resource(&all_resources.l1_data_gas, L1_DATA_GAS)?]
        }
    });

    Ok(HashChain::new().chain(&tip.0.into()).chain_iter(resource_felts.iter()).get_poseidon_hash())
}

pub(crate) fn get_invoke_transaction_v3_hash<T: InvokeTransactionV3Trait>(
    transaction: &T,
    chain_id: &ChainId,
    transaction_version: &TransactionVersion,
) -> Result<TransactionHash, StarknetApiError> {
    let tip_resource_bounds_hash =
        get_tip_resource_bounds_hash(&transaction.resource_bounds(), transaction.tip())?;
    let paymaster_data_hash =
        HashChain::new().chain_iter(transaction.paymaster_data().0.iter()).get_poseidon_hash();
    let data_availability_mode = concat_data_availability_mode(
        transaction.nonce_data_availability_mode(),
        transaction.fee_data_availability_mode(),
    );
    let account_deployment_data_hash = HashChain::new()
        .chain_iter(transaction.account_deployment_data().0.iter())
        .get_poseidon_hash();
    let calldata_hash =
        HashChain::new().chain_iter(transaction.calldata().0.iter()).get_poseidon_hash();
    let mut hash_chain = HashChain::new()
        .chain(&INVOKE)
        .chain(&transaction_version.0)
        .chain(transaction.sender_address().0.key())
        .chain(&tip_resource_bounds_hash)
        .chain(&paymaster_data_hash)
        .chain(&Felt::try_from(chain_id)?)
        .chain(&transaction.nonce().0)
        .chain(&data_availability_mode)
        .chain(&account_deployment_data_hash)
        .chain(&calldata_hash);
    if !transaction.proof_facts().0.is_empty() {
        let proof_facts_hash =
            HashChain::new().chain_iter(transaction.proof_facts().0.iter()).get_poseidon_hash();
        hash_chain = hash_chain.chain(&proof_facts_hash);
    }
    Ok(TransactionHash(hash_chain.get_poseidon_hash()))
