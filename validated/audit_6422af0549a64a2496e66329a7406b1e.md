### Title
Pool Admin Can Instantly Raise Swap Fees Without Timelock to Front-Run Traders — (`metric-core/contracts/MetricOmmPoolFactory.sol`)

---

### Summary

`setPoolAdminFees` allows the pool admin to change `adminSpreadFeeE6` and `adminNotionalFeeE8` instantly and without any timelock. Price-provider changes are protected by a configurable `priceProviderTimelock`, but fee changes carry no equivalent delay. A pool admin can front-run any pending swap by raising fees to the factory-enforced maximum in the same block, causing traders to pay far more than the rate they observed.

---

### Finding Description

`MetricOmmPoolFactory.setPoolAdminFees` (lines 408–435) lets the pool admin update both spread and notional fee components at will: [1](#0-0) 

The function first calls `collectFees` to settle accrued fees at the old rate, then immediately writes the new rates to `poolFeeConfig` and pushes them to the pool via `setPoolFees`. There is no queuing, no delay, and no event that gives traders advance notice before the new rate takes effect.

By contrast, price-provider changes are explicitly time-locked: [2](#0-1) 

The pool-level `setPoolFees` that applies the new rate is equally immediate: [3](#0-2) 

The hard caps the pool admin can reach are: [4](#0-3) 

`HARD_MAX_SPREAD_FEE_E6 = 200_000` → 20 % spread fee  
`HARD_MAX_NOTIONAL_FEE_E8 = 1_000_000` → 1 % notional fee

Both caps are the default values assigned to `maxAdminSpreadFeeE6` and `maxAdminNotionalFeeE8` at construction: [5](#0-4) 

---

### Impact Explanation

A trader who queries the pool's current `spreadFeeE6` / `notionalFeeE8` before submitting a swap sees the benign rate. The pool admin can atomically replace those rates with the maximum values in the block before the swap lands. The swap then executes at up to **20 % spread + 1 % notional** instead of the rate the trader observed. The excess fee is credited to `notionalFeeToken0Scaled` / `notionalFeeToken1Scaled` and to the spread surplus, both of which flow to the pool admin on the next `collectPoolFees` call. This is a direct, quantifiable loss of trader principal with no on-chain recourse.

---

### Likelihood Explanation

- Any pool admin (a permissionless role — anyone who calls `createPool`) can execute this.
- No special privilege beyond pool-admin status is required.
- The attack is a single `setPoolAdminFees` call; it requires only mempool visibility, which is standard on all EVM chains.
- The admin can restore the original fee immediately after the target swap settles, making the manipulation hard to detect retrospectively.

---

### Recommendation

Apply the same timelock pattern already used for price-provider changes to fee updates:

1. Add a `pendingAdminFeeConfig` and `pendingAdminFeeExecuteAfter` mapping per pool.
2. `setPoolAdminFees` writes to the pending slot and records `block.timestamp + feeTimelock`.
3. A separate `executePoolAdminFees` function (callable only after the delay) applies the change and calls `setPoolFees` on the pool.
4. Expose the pending fee change in an event so LPs and traders can react before it takes effect.

This mirrors the existing `proposePoolPriceProvider` / `executePoolPriceProviderUpdate` pattern.

---

### Proof of Concept

1. Pool is deployed with `adminSpreadFeeE6 = 500` (0.05 %) and `adminNotionalFeeE8 = 0`.
2. Alice signs a large `swap` (e.g., 1 000 000 USDC → TOKEN1) and broadcasts it.
3. Pool admin sees Alice's transaction in the mempool and submits `setPoolAdminFees(pool, 200_000, 1_000_000)` with higher gas.
4. Pool admin's transaction lands first:
   - `collectFees` settles the tiny accrued fees at the old 0.05 % rate.
   - `poolFeeConfig[pool].adminSpreadFeeE6` is set to `200_000`.
   - `pool.setPoolFees(protocolSpread + 200_000, protocolNotional + 1_000_000)` is called.
5. Alice's swap executes: she pays ≈ 20 % spread + 1 % notional instead of 0.05 %.
6. Pool admin calls `collectPoolFees` — the inflated fees flow to `poolAdminFeeDestination`.
7. Pool admin calls `setPoolAdminFees(pool, 500, 0)` to restore the normal rate.

Alice loses up to ~21 % of her swap value with no on-chain protection and no advance warning.

  function setPoolAdminFees(address pool, uint24 newAdminSpreadFeeE6, uint24 newAdminNotionalFeeE8)
    external
    override
    nonReentrant
    onlyPoolAdmin(pool)
  {
    if (newAdminSpreadFeeE6 > maxAdminSpreadFeeE6) revert AdminFeeTooHigh();
    if (newAdminNotionalFeeE8 > maxAdminNotionalFeeE8) revert AdminFeeTooHigh();

    PoolFeeConfig memory c = poolFeeConfig[pool];
    IMetricOmmPoolCollectFees(pool)
      .collectFees(
        c.protocolSpreadFeeE6,
        c.adminSpreadFeeE6,
        c.protocolNotionalFeeE8,
        c.adminNotionalFeeE8,
        poolAdminFeeDestination[pool]
      );

    c.adminSpreadFeeE6 = newAdminSpreadFeeE6;
    c.adminNotionalFeeE8 = newAdminNotionalFeeE8;
    poolFeeConfig[pool] = c;

    IMetricOmmPoolFactoryActions(pool)
      .setPoolFees(c.protocolSpreadFeeE6 + c.adminSpreadFeeE6, c.protocolNotionalFeeE8 + c.adminNotionalFeeE8);
    emit PoolAdminSpreadFeeUpdated(pool, newAdminSpreadFeeE6);
    emit PoolAdminNotionalFeeUpdated(pool, newAdminNotionalFeeE8);
  }

  function proposePoolPriceProvider(address pool, address newPriceProvider)
    external
    override
    nonReentrant
    onlyPoolAdmin(pool)
  {
    PoolImmutables memory p = IMetricOmmPool(pool).getImmutables();
    uint256 timelock = priceProviderTimelock[pool];
    if (p.immutablePriceProvider != address(0)) revert PriceProviderImmutable();
    _validatePriceProvider(p.token0, p.token1, newPriceProvider);

    address mutableProvider = PoolStateLibrary._slot3(pool);
    address current = mutableProvider != address(0) ? mutableProvider : p.immutablePriceProvider;
    uint256 executeAfter = block.timestamp + timelock;
    pendingPriceProvider[pool] = newPriceProvider;
    pendingPriceProviderExecuteAfter[pool] = executeAfter;
    emit PoolPriceProviderChangeProposed(pool, current, newPriceProvider, executeAfter);
  }

  /// @inheritdoc IMetricOmmPoolFactoryPoolAdmin
  function executePoolPriceProviderUpdate(address pool) external override nonReentrant onlyPoolAdmin(pool) {
    address pending = pendingPriceProvider[pool];
    if (pending == address(0)) revert NoPriceProviderChangeProposed();
    uint256 execAfter = pendingPriceProviderExecuteAfter[pool];
    // forge-lint: disable-next-line(block-timestamp) -- timelock enforcement legitimately relies on `block.timestamp`.
    if (block.timestamp < execAfter) revert PriceProviderTimelockNotElapsed(execAfter, block.timestamp);
    PoolImmutables memory p = IMetricOmmPool(pool).getImmutables();
    if (p.immutablePriceProvider != address(0)) revert PriceProviderImmutable();
    _validatePriceProvider(p.token0, p.token1, pending);
    IMetricOmmPoolFactoryActions(pool).setPriceProvider(pending);
    delete pendingPriceProvider[pool];
    delete pendingPriceProviderExecuteAfter[pool];
    emit PoolPriceProviderUpdated(pool, pending);
  }
