Audit Report

## Title
`SwapAllowlistExtension.beforeSwap` checks the router address instead of the actual swapper, allowing any user to bypass the per-user allowlist via `MetricOmmSimpleRouter` — (`metric-periphery/contracts/extensions/SwapAllowlistExtension.sol`)

## Summary

`SwapAllowlistExtension.beforeSwap` receives `sender` from the pool, which is `msg.sender` of `MetricOmmPool.swap`. When a user routes through `MetricOmmSimpleRouter`, `sender` is the router address, not the actual user. A pool admin who allowlists the router to enable router-based swaps for their approved users inadvertently opens the gate to every unapproved user who calls through the router, fully breaking the per-user allowlist invariant.

## Finding Description

`MetricOmmPool.swap` passes `msg.sender` as the first argument to `_beforeSwap`: [1](#0-0) 

When the call path is `user → MetricOmmSimpleRouter → pool.swap()`, `msg.sender` inside the pool is the router, so `sender` passed to the extension is the router address.

`SwapAllowlistExtension.beforeSwap` then checks: [2](#0-1) 

Here `msg.sender` is the pool and `sender` is whoever called `pool.swap()` — the router. The check `allowedSwapper[pool][router]` passes if the admin added the router, regardless of who the actual end-user is.

`MetricOmmSimpleRouter.exactInputSingle` stores the real caller only in transient storage for the payment callback and never forwards it to the pool or extension: [3](#0-2) 

The actual user address (`msg.sender`) is captured by `_setNextCallbackContext` for payment purposes only; the extension never sees it. The same pattern applies to `exactInput`, `exactOutputSingle`, and `exactOutput`. [4](#0-3) 

The allowlist mapping is keyed `allowedSwapper[pool][swapper]`. Adding the router as `swapper` grants every caller of the router access, collapsing the per-user gate to a single shared entry.

## Impact Explanation

Any non-allowlisted user can swap in a pool intended to be restricted (KYC-gated, institutional-only, rate-limited) by calling `MetricOmmSimpleRouter.exactInputSingle` (or `exactInput`/`exactOutput`). The allowlist invariant — "only approved addresses may swap" — is broken for every pool that allowlists the router. Unrestricted arbitrage and toxic flow against LP positions causes direct loss of LP principal. This is a broken core pool access-control mechanism with direct fund impact, meeting the Medium/High threshold.

## Likelihood Explanation

Medium. The bypass requires the pool admin to add the router to the allowlist, which is the natural and expected operational step for enabling router-based swaps for approved users. The mistake is non-obvious: the admin expects the router to act as a transparent forwarder of user identity, which it does not. Once the router is allowlisted, any EOA can exploit the bypass with no special privilege. The condition is predictable and repeatable.

## Recommendation

The extension must check the economically relevant actor, not the intermediary. Two options:

1. **Forward the real caller via `extensionData`**: Have `MetricOmmSimpleRouter` encode `msg.sender` into `extensionData` before calling `pool.swap`. Have `SwapAllowlistExtension.beforeSwap` decode and check it when `sender` is a known/trusted router address.
2. **Router-aware fallback in the extension**: If `sender` is an allowlisted router, decode the actual user from `extensionData` and apply the per-user check to that address instead of the router address.

Either approach closes the gap between the configured gate (individual users) and the checked identity (router address).

## Proof of Concept

1. Pool admin deploys a pool with `SwapAllowlistExtension` in the `beforeSwap` slot.
2. Pool admin calls `setAllowedToSwap(pool, alice, true)` — Alice is approved.
3. Pool admin calls `setAllowedToSwap(pool, router, true)` — router is added so Alice can use it.
4. Bob (not allowlisted) calls `MetricOmmSimpleRouter.exactInputSingle({pool: pool, ...})`.
5. Router calls `pool.swap(params.recipient, ...)`. Inside the pool, `msg.sender` = router.
6. Pool calls `_beforeSwap(router, ...)`. Extension evaluates `allowedSwapper[pool][router]` → `true` → passes.
7. Bob's swap executes in the restricted pool. The per-user allowlist is fully bypassed. [5](#0-4) [6](#0-5)
