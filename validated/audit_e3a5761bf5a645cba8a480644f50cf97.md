### Title
Duplicate Position IDs in `ClaimTierRewards` Enable Double-Claiming of Bonus Rewards - (File: `x/tieredrewards/keeper/msg_server.go`)

---

### Summary

`MsgClaimTierRewards` accepts a caller-supplied slice of position IDs with no deduplication check. Position states are loaded into an in-memory slice before any processing begins. When the same ID appears twice, both slots in the slice hold the same stale checkpoint values. The second iteration re-computes and re-pays the current-segment bonus using the old `LastBonusAccrual` timestamp, draining the `RewardsPoolName` module account.

---

### Finding Description

In `ClaimTierRewards` (`msg_server.go` lines 429–468), position states are collected into a local slice before any reward processing: [1](#0-0) 

```go
positions := make([]types.PositionState, 0, len(msg.PositionIds))
for _, posId := range msg.PositionIds {
    pos, err := ms.getPositionState(ctx, posId)
    ...
    positions = append(positions, pos)
}
totalBase, totalBonus, err := ms.claimRewardsAndUpdatesPositions(ctx, positions)
```

There is no deduplication of `msg.PositionIds`. If the caller submits `PositionIds = [X, X]`, both `positions[0]` and `positions[1]` are loaded with the same stale state (identical `LastBonusAccrual`, `LastEventSeq`, `LastKnownBonded`).

`claimRewardsAndUpdatesPositions` then iterates over the slice: [2](#0-1) 

For each element it calls `processEventsAndClaimBonus`, which computes the current-segment bonus from `pos.LastBonusAccrual` to `blockTime`: [3](#0-2) 

```go
if bonded && val.IsBonded() {
    currentRate, err := k.getTokensPerShare(ctx, valAddr)
    ...
    bonus := k.computeSegmentBonus(*pos, tier, segmentStart, blockTime, currentRate)
    totalBonus = totalBonus.Add(bonus)
}
```

After the first iteration, `setPosition` writes the updated `LastBonusAccrual = blockTime` to the store. However, the second iteration's `pos` is a pointer to `positions[1]` — the **stale in-memory copy** — which still holds the old `LastBonusAccrual`. The store update from iteration 1 is never reflected in `positions[1]`. [4](#0-3) 

The second iteration therefore re-computes the identical current-segment bonus and calls: [5](#0-4) 

```go
k.bankKeeper.SendCoinsFromModuleToAccount(ctx, types.RewardsPoolName, ownerAddr, bonusCoins)
```

This sends the same bonus coins a second time from the module account to the owner.

Regarding event-based bonus: `decrementEventRefCount` is called during the first iteration, which may delete events when their ref count reaches zero. The second iteration's `getValidatorEventsSince` would then return empty, so event-based bonus is not double-paid. However, the **current-segment bonus** (the accrual from `LastBonusAccrual` to `blockTime`) is fully double-paid because it depends only on the stale in-memory `LastBonusAccrual` value, not on the event store.

The attacker can include the same position ID `N` times to multiply the payout by `N`, limited only by `sufficientBonusPoolBalance`. [6](#0-5) 

---

### Impact Explanation

The corrupted value is the `RewardsPoolName` module account balance. An attacker with any active tiered-rewards position can drain the bonus pool by submitting a single `MsgClaimTierRewards` transaction with the same position ID repeated many times. Each repetition transfers `bonusCoins` (proportional to locked amount × tier rate × elapsed time) from the module account to the attacker's address. The pool is a shared resource; draining it harms all other position holders who are owed future bonus rewards.

---

### Likelihood Explanation

The entry path is a standard, unprivileged `MsgClaimTierRewards` transaction signed by any position owner. No special role, governance action, or IBC relay is required. The message format accepts a repeated field of position IDs, and there is no on-chain validation preventing duplicates. Any position holder who discovers this can exploit it immediately.

---

### Recommendation

Add a deduplication check on `msg.PositionIds` before loading positions, either in `MsgClaimTierRewards.Validate()` or at the start of `ClaimTierRewards`:

```go
seen := make(map[uint64]struct{}, len(msg.PositionIds))
for _, id := range msg.PositionIds {
    if _, dup := seen[id]; dup {
        return nil, errorsmod.Wrapf(sdkerrors.ErrInvalidRequest, "duplicate position id: %d", id)
    }
    seen[id] = struct{}{}
}
```

Alternatively, re-read each position's state from the store inside `claimRewardsAndUpdatesPositions` rather than operating on pre-loaded in-memory copies, so that the updated `LastBonusAccrual` written by iteration `i` is visible to iteration `i+1`.

---

### Proof of Concept

1. Attacker holds position ID `42` with `LastBonusAccrual` set 30 days ago and a non-zero locked amount in a bonded-validator tier.
2. Attacker submits `MsgClaimTierRewards{ Owner: attacker, PositionIds: [42, 42, 42, ..., 42] }` with the ID repeated 100 times.
3. `ClaimTierRewards` loads 100 identical copies of position 42's state into `positions`.
4. `claimRewardsAndUpdatesPositions` iterates 100 times. Each iteration computes `computeSegmentBonus` from the same old `LastBonusAccrual` to `blockTime` and calls `SendCoinsFromModuleToAccount`.
5. The attacker receives 100× the legitimate bonus for that 30-day period.
6. The `RewardsPoolName` module account is debited 100× the correct amount, leaving other position holders unable to claim their owed rewards. [7](#0-6) [8](#0-7) [9](#0-8)

func (ms msgServer) ClaimTierRewards(ctx context.Context, msg *types.MsgClaimTierRewards) (*types.MsgClaimTierRewardsResponse, error) {
	if err := msg.Validate(); err != nil {
		return nil, err
	}

	positions := make([]types.PositionState, 0, len(msg.PositionIds))
	for _, posId := range msg.PositionIds {
		pos, err := ms.getPositionState(ctx, posId)
		if err != nil {
			return nil, err
		}

		if err := ms.validateClaimRewards(pos.Position, msg.Owner); err != nil {
			return nil, err
		}

		positions = append(positions, pos)
	}

	totalBase, totalBonus, err := ms.claimRewardsAndUpdatesPositions(ctx, positions)
	if err != nil {
		return nil, err
	}

	sdkCtx := sdk.UnwrapSDKContext(ctx)
	if err := sdkCtx.EventManager().EmitTypedEvent(&types.EventTierRewardsClaimed{
		Owner:        msg.Owner,
		PositionIds:  msg.PositionIds,
		BaseRewards:  totalBase,
		BonusRewards: totalBonus,
	}); err != nil {
		return nil, err
	}

	return &types.MsgClaimTierRewardsResponse{
		BaseRewards:  totalBase,
		BonusRewards: totalBonus,
		PositionIds:  msg.PositionIds,
	}, nil
}

func (k Keeper) claimRewardsAndUpdatesPositions(ctx context.Context, positions []types.PositionState) (sdk.Coins, sdk.Coins, error) {
	totalBase := sdk.NewCoins()
	totalBonus := sdk.NewCoins()

	for i := range positions {
		pos := &positions[i]

		if !pos.IsDelegated() {
			continue
		}

		base, err := k.claimBaseRewards(ctx, *pos)
		if err != nil {
			return nil, nil, err
		}
		totalBase = totalBase.Add(base...)

		bonus, err := k.processEventsAndClaimBonus(ctx, pos)
		if err != nil {
			return nil, nil, err
		}
		totalBonus = totalBonus.Add(bonus...)

		if err := k.setPosition(ctx, pos.Position, nil); err != nil {
			return nil, nil, err
		}
	}

	return totalBase, totalBonus, nil
}
