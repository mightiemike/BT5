Audit Report

## Title
`SwapAllowlistExtension` checks router address instead of originating user, enabling allowlist bypass via `MetricOmmSimpleRouter` - (`metric-periphery/contracts/extensions/SwapAllowlistExtension.sol`)

## Summary
`SwapAllowlistExtension.beforeSwap` gates swaps on `allowedSwapper[pool][sender]`, where `sender` is `msg.sender` of the pool's `swap()` call. When a user routes through `MetricOmmSimpleRouter`, the pool receives the router as `msg.sender`, so the check becomes `allowedSwapper[pool][router]`. If the pool admin allowlists the router to enable any legitimate user to swap through it, every non-allowlisted user can also bypass the restriction by routing through the public router. This directly violates the pool admin's access-control invariant.

## Finding Description
**Exact call path:**

`MetricOmmPool.swap()` passes `msg.sender` as the `sender` argument to `_beforeSwap`: [1](#0-0) 

`ExtensionCalling._beforeSwap` forwards `sender` verbatim to the extension: [2](#0-1) 

`SwapAllowlistExtension.beforeSwap` evaluates `allowedSwapper[msg.sender][sender]`, where `msg.sender` = pool and `sender` = router: [3](#0-2) 

`MetricOmmSimpleRouter.exactInputSingle` calls `pool.swap()` directly, making the router the `msg.sender` at the pool: [4](#0-3) 

**Two broken outcomes:**
1. **Allowlist bypass (high impact):** Admin allowlists the router so that legitimate users can swap through it. Because the check is on the router address, every user — including those not on the allowlist — can now swap by routing through `MetricOmmSimpleRouter`.
2. **Broken functionality:** Admin does not allowlist the router. Allowlisted users cannot use the router at all; the periphery is effectively unusable for allowlisted pools.

Existing guards are insufficient: the only check in `beforeSwap` is `allowedSwapper[msg.sender][sender]`, and there is no mechanism to recover the originating EOA from the router call. [5](#0-4) 

## Impact Explanation
A pool configured with `SwapAllowlistExtension` is intended to restrict swapping to a curated set of addresses (e.g., KYC'd counterparties, whitelisted market makers). The bypass allows any unpermissioned address to execute swaps on such a pool by routing through the public `MetricOmmSimpleRouter`. This breaks the admin-boundary invariant and constitutes broken core pool functionality: the allowlist — a primary access-control mechanism — is rendered ineffective, allowing unauthorized parties to trade against the pool's liquidity.

## Likelihood Explanation
`MetricOmmSimpleRouter` is the primary user-facing swap interface and is a public, permissionless contract. Any user who discovers the allowlist can trivially route through the router. The only precondition is that the pool admin has allowlisted the router (the only way to make the router usable for any user), which is the expected operational setup. No privileged access or special setup is required beyond this.

## Recommendation
The extension must gate the economic actor, not the immediate caller. Two concrete options:

1. **Pass the original user through the router.** Modify `MetricOmmSimpleRouter` to encode the originating `msg.sender` in `extensionData`, and have `SwapAllowlistExtension.beforeSwap` decode and check that address. This requires a convention between router and extension.

2. **Check `recipient` instead of `sender`.** If the pool's design intent is that the recipient is the economic beneficiary, check `allowedSwapper[pool][recipient]`. This is simpler but only correct if recipient always equals the user.

## Proof of Concept
```solidity
// Setup: pool with SwapAllowlistExtension; alice is allowlisted, bob is not.
swapExtension.setAllowedToSwap(address(pool), alice, true);
// router is allowlisted so alice can use it
swapExtension.setAllowedToSwap(address(pool), address(router), true);

// Bob (not allowlisted) calls directly → reverts correctly
vm.prank(bob);
vm.expectRevert(IMetricOmmPoolActions.NotAllowedToSwap.selector);
pool.swap(bob, false, 1000, type(uint128).max, "", "");

// Bob routes through the router → succeeds, bypassing the allowlist
vm.prank(bob);
router.exactInputSingle(ExactInputSingleParams({
    pool: address(pool),
    recipient: bob,
    tokenIn: token1,
    zeroForOne: false,
    amountIn: 1000,
    amountOutMinimum: 0,
    priceLimitX64: type(uint128).max,
    deadline: block.timestamp + 1,
    extensionData: ""
}));
// Bob successfully swaps despite not being on the allowlist.
```
