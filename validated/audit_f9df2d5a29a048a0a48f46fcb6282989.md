### Title
`FeeEstimation` Struct Omits `l2_gas_consumed` While Including `l2_gas_price`, Causing Incorrect Fee Breakdown in RPC Estimation Endpoints - (File: `crates/apollo_rpc_execution/src/objects.rs`)

### Summary

The `FeeEstimation` struct returned by `starknet_estimateFee`, `starknet_simulateTransactions`, and `starknet_estimateMessageFee` includes `l2_gas_price` but omits `l2_gas_consumed`. The struct's own documentation states `overall_fee = gas_consumed * gas_price + data_gas_consumed * data_gas_price`, but the actual `overall_fee` is computed from the full `GasVector` including `l2_gas`. Any client that reconstructs the fee from the returned components will compute a value lower than `overall_fee`, receiving an authoritative-looking but internally inconsistent response.

### Finding Description

`FeeEstimation` is defined in `crates/apollo_rpc_execution/src/objects.rs`:

```rust
pub struct FeeEstimation {
    pub gas_consumed: Felt,          // l1_gas
    pub l1_gas_price: GasPrice,
    pub data_gas_consumed: Felt,     // l1_data_gas
    pub l1_data_gas_price: GasPrice,
    // TODO(Tzahi): Add l2_gas_consumed. Verify overall_fee estimation of
    // l1_gas_price only is close enough (as there are roundings) to the fee
    // of both l1_gas_price and l2_gas_price.
    pub l2_gas_price: GasPrice,      // price present, quantity absent
    pub overall_fee: Fee,
    pub unit: PriceUnit,
}
```

The function `tx_execution_output_to_fee_estimation` populates this struct:

```rust
Ok(FeeEstimation {
    gas_consumed: gas_vector.l1_gas.0.into(),
    l1_gas_price,
    data_gas_consumed: gas_vector.l1_data_gas.0.into(),
    l1_data_gas_price,
    l2_gas_price,
    overall_fee: tx_execution_output.execution_info.receipt.fee,  // includes l2_gas cost
    unit: tx_execution_output.price_unit,
})
```

`gas_vector.l2_gas` is read from `receipt.gas` but is never placed into the returned struct. Meanwhile `receipt.fee` is computed via `GasVector::cost()`, which sums all three gas components including `l2_gas * (l2_gas_price + tip)`. The result is that `overall_fee > gas_consumed * l1_gas_price + data_gas_consumed * l1_data_gas_price` for any transaction that consumes L2 gas, contradicting the struct's own doc comment.

### Impact Explanation

Any caller of `starknet_estimateFee`, `starknet_simulateTransactions`, or `starknet_estimateMessageFee` that attempts to verify or reconstruct `overall_fee` from the returned components will compute a systematically lower value. The response is authoritative-looking (it includes `l2_gas_price`) but internally inconsistent: the price for a resource is present while the consumed quantity of that resource is absent. This matches the "High. RPC execution, fee estimation, tracing, simulation, or pending view returns an authoritative-looking wrong value" impact criterion.

### Likelihood Explanation

Any client that parses the fee estimation response and attempts to verify `overall_fee` by multiplying the exposed gas quantities by their prices will observe a discrepancy on every transaction that consumes L2 gas. The TODO comment confirms this is a known, unresolved gap in the production code path.

### Recommendation

- **Short term**: Add `l2_gas_consumed: Felt` to `FeeEstimation` and populate it with `gas_vector.l2_gas.0.into()` in `tx_execution_output_to_fee_estimation`. Update the `overall_fee` doc comment to include the l2_gas term.
- **Long term**: Add a unit test that asserts `overall_fee == gas_consumed * l1_gas_price + data_gas_consumed * l1_data_gas_price + l2_gas_consumed * l2_gas_price` (plus tip) for every fee estimation code path, so any future regression is caught automatically.

### Proof of Concept

1. Submit any V3 invoke transaction to `starknet_estimateFee`.
2. Observe the returned `FeeEstimation`: `l2_gas_price` is non-zero, `l2_gas_consumed` is absent.
3. Compute `reconstructed_fee = gas_consumed * l1_gas_price + data_gas_consumed * l1_data_gas_price`.
4. Compare with `overall_fee`: `overall_fee > reconstructed_fee` by exactly `l2_gas * l2_gas_price`, confirming the missing component.

    #[instrument(skip(self, transactions), level = "debug", err, ret)]
    async fn estimate_fee(
        &self,
        transactions: Vec<BroadcastedTransaction>,
        simulation_flags: Vec<SimulationFlag>,
        block_id: BlockId,
    ) -> RpcResult<Vec<FeeEstimation>> {
        trace!("Estimating fee of transactions: {:#?}", transactions);
        let validate = !simulation_flags.contains(&SimulationFlag::SkipValidate);

        let storage_txn = self.storage_reader.begin_ro_txn().map_err(internal_server_error)?;

        let maybe_pending_data = if let BlockId::Tag(Tag::Pending) = block_id {
            Some(client_pending_data_to_execution_pending_data(
                read_pending_data(&self.pending_data, &storage_txn).await?,
                self.pending_classes.read().await.clone(),
            ))
        } else {
            None
        };

        let executable_txns =
            transactions.into_iter().map(|tx| tx.try_into()).collect::<Result<_, _>>()?;

        let block_number = get_accepted_block_number(&storage_txn, block_id)?;
        let block_not_reverted_validator =
            BlockNotRevertedValidator::new(block_number, &storage_txn)?;
        drop(storage_txn);
        let state_number = StateNumber::unchecked_right_after_block(block_number);
        let execution_config = self.execution_config;

        let chain_id = self.chain_id.clone();
        let reader = self.storage_reader.clone();
        let class_manager_client =
            create_class_manager_client(self.class_manager_client.clone()).await;

        let estimate_fee_result = tokio::task::spawn_blocking(move || {
            exec_estimate_fee(
                executable_txns,
                &chain_id,
                reader,
                maybe_pending_data,
                state_number,
                block_number,
                &execution_config,
                validate,
                DONT_IGNORE_L1_DA_MODE,
                class_manager_client,
            )
        })
        .await
        .map_err(internal_server_error)?;

        block_not_reverted_validator.validate(&self.storage_reader)?;

        match estimate_fee_result {
            Ok(Ok(fees)) => Ok(fees),
            Ok(Err(reverted_tx)) => {
                Err(ErrorObjectOwned::from(JsonRpcError::<TransactionExecutionError>::from(
                    TransactionExecutionError {
                        transaction_index: reverted_tx.index,
                        execution_error: reverted_tx.revert_reason,
                    },
                )))
            }
            Err(err) => Err(internal_server_error(err)),
        }
    }
