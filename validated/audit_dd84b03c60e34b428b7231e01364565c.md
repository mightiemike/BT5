### Title
ML-DSA-65 Wire Length Passed Instead of Trie-ID Length to Gas-Key Host-Function Fee Helpers — (File: `runtime/near-vm-runner/src/logic/logic.rs`)

---

### Summary

`promise_batch_action_transfer_to_gas_key` and `promise_batch_action_add_gas_key_with_full_access` (and their Wasmtime mirrors) pass the raw `public_key_len` argument — the borsh wire length — to `gas_key_transfer_send_fee`, `gas_key_transfer_exec_fee`, and `gas_key_add_key_exec_fee`. For ML-DSA-65 keys the wire length is **1953 bytes** while the on-trie identifier is **33 bytes** (SHA3-256 hash). The transaction-level fee path in `config.rs` correctly calls `action.public_key.trie_id_len()` (= 33) for the same fee helpers. The result is a ~59× fee overcharge for any contract that funds or registers an ML-DSA-65 gas key via a host function, breaking the protocol invariant that the same action costs the same gas regardless of invocation path.

---

### Finding Description

`PublicKey::len()` returns the borsh-encoded wire size (1 + 1952 = **1953** for ML-DSA-65), while `PublicKey::trie_id_len()` returns the on-trie identifier size (1 + 32 = **33** for ML-DSA-65). The two values are identical for ed25519 (33) and secp256k1 (65), so the divergence is invisible until an ML-DSA-65 key is used.

**Transaction path** — `total_send_fees` and `exec_fee` in `runtime/runtime/src/config.rs`:

```rust
// line 115
TransferToGasKey(action) => {
    gas_key_transfer_send_fee(fees, sender_is_receiver, action.public_key.trie_id_len())
        .total()
}
// line 348
TransferToGasKey(action) => {
    gas_key_transfer_exec_fee(fees, receiver_id.len(), action.public_key.trie_id_len())
        .total()
}
```

Both calls use `trie_id_len()` = 33 for ML-DSA-65.

**Host-function path** — `promise_batch_action_transfer_to_gas_key` in `runtime/near-vm-runner/src/logic/logic.rs`:

```rust
// lines 3091-3096
let send = gas_key_transfer_send_fee(&self.fees_config, sir, public_key_len as usize);
let exec = gas_key_transfer_exec_fee(
    &self.fees_config,
    receiver_id.len(),
    public_key_len as usize,   // ← raw wire length, 1953 for ML-DSA-65
);
```

`public_key_len` is the raw u64 passed by the contract — the borsh-encoded wire length (1953 for ML-DSA-65). The decoded `PublicKey` is available at this point (line 3085: `let public_key = self.get_public_key(…)?`) but its `trie_id_len()` is never consulted.

The same pattern appears in `promise_batch_action_add_gas_key_with_full_access` (lines 3155–3160) and `promise_batch_action_add_gas_key_with_function_call` (lines 3226–3231), where `public_key_len as usize` is passed to `gas_key_add_key_exec_fee` instead of the decoded key's `trie_id_len()`. The Wasmtime mirror at `runtime/near-vm-runner/src/wasmtime_runner/logic.rs` lines 3323–3325 and 3395–3400 contains the identical defect.

Inside `gas_key_transfer_exec_fee` the wrong length propagates into `access_key_key_len(account_id_len, public_key_len)`, inflating the computed trie-key length by 1920 bytes and thus the per-byte exec fee by the same factor. Inside `gas_key_add_key_exec_fee` the same inflation applies to `nonce_key_len`.

---

### Impact Explanation

Any smart contract that calls `promise_batch_action_transfer_to_gas_key` or `promise_batch_action_add_gas_key_with_*` with an ML-DSA-65 public key is charged approximately **59× more gas** (1953 / 33) for the per-byte component of both the send and exec fees compared to the identical action submitted as a signed transaction. Because the exec fee is prepaid from the contract's gas budget, a contract that budgets the correct amount (matching the transaction path) will exhaust its gas and fail. A contract that budgets the inflated amount succeeds but burns ~59× the intended gas, draining the gas-key balance far faster than expected. The discrepancy is deterministic and reproducible for every ML-DSA-65 gas-key host-function call after `PostQuantumSignatures` activates (protocol version 85).

---

### Likelihood Explanation

`PostQuantumSignatures` is stabilized at protocol version 85. Any contract deployed on a network running ≥ v85 that attempts to fund or register an ML-DSA-65 gas key via a promise batch action will trigger the overcharge. The trigger requires only a normal contract call — no privileged role is needed. The bug is latent for ed25519/secp256k1 keys (where `len() == trie_id_len()`) and becomes active the moment an ML-DSA-65 key is used in the host-function path.

---

### Recommendation

Replace `public_key_len as usize` with `public_key.trie_id_len()` (after decoding, or by peeking at the tag byte to determine the scheme before full decoding) in all three host-function fee-calculation call sites:

- `promise_batch_action_transfer_to_gas_key` — lines 3091 and 3095 of `logic.rs`
- `promise_batch_action_add_gas_key_with_full_access` — line 3158 of `logic.rs`
- `promise_batch_action_add_gas_key_with_function_call` — line 3229 of `logic.rs`
- Identical sites in `wasmtime_runner/logic.rs`

The transaction path in `config.rs` is the reference: it already calls `action.public_key.trie_id_len()` for every gas-key fee helper.

---

### Proof of Concept

1. Deploy a contract that calls `promise_batch_action_transfer_to_gas_key` with a borsh-encoded ML-DSA-65 public key (1953 bytes, tag byte `0x02`).
2. Observe `gas_burnt` on the resulting receipt.
3. Submit the identical `TransferToGasKey` action as a signed transaction.
4. Observe `gas_burnt` on that receipt.
5. The host-function receipt burns ≈59× more gas in the per-byte component than the transaction receipt, confirming the divergence.

/// Send fee for TransferToGasKey / WithdrawFromGasKey actions.
/// Based on the public key length (what the sender sees).
pub fn gas_key_transfer_send_fee(
    cfg: &RuntimeFeesConfig,
    sender_is_receiver: bool,
    public_key_len: usize,
) -> GasKeyTransferFee {
    let base = cfg.fee(ActionCosts::gas_key_transfer_base).send_fee(sender_is_receiver);
    let per_byte = cfg
        .fee(ActionCosts::gas_key_byte)
        .send_fee(sender_is_receiver)
        .checked_mul(public_key_len as u64)
        .unwrap();
    GasKeyTransferFee { base, per_byte }
}

/// Exec fee for TransferToGasKey / WithdrawFromGasKey actions.
/// Based on the access key trie key length + estimated value length (what the
/// receiver needs to read/write in the trie).
pub fn gas_key_transfer_exec_fee(
    cfg: &RuntimeFeesConfig,
    account_id_len: usize,
    public_key_len: usize,
) -> GasKeyTransferFee {
    let base = cfg.fee(ActionCosts::gas_key_transfer_base).exec_fee();
    let trie_key_len = access_key_key_len(account_id_len, public_key_len);
    let estimated_value_len = AccessKey::min_gas_key_borsh_len();
    let per_byte = cfg
        .fee(ActionCosts::gas_key_byte)
        .exec_fee()
        .checked_mul((trie_key_len + estimated_value_len) as u64)
        .unwrap();
    GasKeyTransferFee { base, per_byte }
}

impl PublicKey {
    /// Length of this public key's borsh encoding, in bytes - that is,
    /// the on-the-wire size of the raw key bytes plus a 1-byte borsh
    /// discriminant tag (the leading `+ 1` in each arm).
    ///
    /// For storage-fee accounting use [`PublicKey::trie_id_len`] instead;
    /// for ML-DSA-65 those two diverge (1953 wire vs 33 on-trie).
    // `is_empty` always returns false, so there is no point in adding it
    #[allow(clippy::len_without_is_empty)]
    pub fn len(&self) -> usize {
        match self {
            Self::ED25519(_) => 1 + ed25519_dalek::PUBLIC_KEY_LENGTH,
            Self::SECP256K1(_) => 1 + 64,
            Self::MLDSA65(_) => 1 + ML_DSA_65_PUBLIC_KEY_LENGTH,
        }
    }

    pub fn empty(key_type: KeyType) -> Self {
        match key_type {
            KeyType::ED25519 => {
                PublicKey::ED25519(ED25519PublicKey([0u8; ed25519_dalek::PUBLIC_KEY_LENGTH]))
            }
            KeyType::SECP256K1 => PublicKey::SECP256K1(Secp256K1PublicKey([0u8; 64])),
            KeyType::MLDSA65 => {
                PublicKey::MLDSA65(MlDsa65PublicKey(Box::new([0u8; ML_DSA_65_PUBLIC_KEY_LENGTH])))
            }
        }
    }

    pub fn key_type(&self) -> KeyType {
        match self {
            Self::ED25519(_) => KeyType::ED25519,
            Self::SECP256K1(_) => KeyType::SECP256K1,
            Self::MLDSA65(_) => KeyType::MLDSA65,
        }
    }

    fn key_tag(&self) -> KeyTag {
        match self {
            PublicKey::ED25519(_) => KeyTag::Ed25519,
            PublicKey::SECP256K1(_) => KeyTag::Secp256k1,
            PublicKey::MLDSA65(_) => KeyTag::MlDsa65Full,
        }
    }

    pub fn key_data(&self) -> &[u8] {
        match self {
            Self::ED25519(key) => key.as_ref(),
            Self::SECP256K1(key) => key.as_ref(),
            Self::MLDSA65(key) => key.as_ref(),
        }
    }

    pub fn unwrap_as_ed25519(&self) -> &ED25519PublicKey {
        match self {
            Self::ED25519(key) => key,
            Self::SECP256K1(_) | Self::MLDSA65(_) => panic!(),
        }
    }

    pub fn unwrap_as_secp256k1(&self) -> &Secp256K1PublicKey {
        match self {
            Self::SECP256K1(key) => key,
            Self::ED25519(_) | Self::MLDSA65(_) => panic!(),
        }
    }

    /// Length, in bytes, of the on-trie identifier for an access-key
    /// entry owned by this public key. For ed25519/secp256k1 this matches
    /// `len()`; for ML-DSA-65 the trie stores a SHA3-256 hash (33 bytes
    /// including the type tag), not the 1953-byte borsh-encoded pubkey.
    /// Used by storage-fee calculations on the runtime side; cheap to call
    /// (no hashing) - for ML-DSA-65 this returns the size of the digest
    /// form without actually hashing the pubkey.
    pub fn trie_id_len(&self) -> usize {
        match self {
            Self::ED25519(_) => 1 + ed25519_dalek::PUBLIC_KEY_LENGTH,
            Self::SECP256K1(_) => 1 + 64,
            Self::MLDSA65(_) => 1 + ML_DSA_65_HASH_LENGTH,
        }
    }

pub fn promise_batch_action_transfer_to_gas_key(
    ctx: &mut Ctx,
    memory: &mut [u8],
    promise_idx: u64,
    public_key_len: u64,
    public_key_ptr: u64,
    amount_ptr: u64,
) -> Result<()> {
    ctx.result_state.gas_counter.pay_base(base)?;
    if ctx.context.is_view() {
        return Err(HostError::ProhibitedInView {
            method_name: "promise_batch_action_transfer_to_gas_key".to_string(),
        }
        .into());
    }
    let public_key = get_public_key(
        &mut ctx.result_state.gas_counter,
        memory,
        &ctx.registers,
        public_key_ptr,
        public_key_len,
    )?;
    let amount =
        Balance::from_yoctonear(get_u128(&mut ctx.result_state.gas_counter, memory, amount_ptr)?);
    let (receipt_idx, sir) = promise_idx_to_receipt_idx_with_sir(ctx, promise_idx)?;
    let receiver_id = ctx.ext.get_receipt_receiver(receipt_idx);
    let send = gas_key_transfer_send_fee(&ctx.fees_config, sir, public_key_len as usize);
    let exec =
        gas_key_transfer_exec_fee(&ctx.fees_config, receiver_id.len(), public_key_len as usize);
    let burn_base = send.base;
    let use_base = burn_base.gas.checked_add(exec.base.gas).ok_or(HostError::IntegerOverflow)?;
    ctx.result_state.gas_counter.pay_action_accumulated(
        burn_base,
        use_base,
        ActionCosts::gas_key_transfer_base,
    )?;
    let burn_byte = send.per_byte;
    let use_byte =
        burn_byte.gas.checked_add(exec.per_byte.gas).ok_or(HostError::IntegerOverflow)?;
    ctx.result_state.gas_counter.pay_action_accumulated(
        burn_byte,
        use_byte,
        ActionCosts::gas_key_byte,
    )?;
    ctx.result_state.deduct_balance(amount)?;
    ctx.ext.append_action_transfer_to_gas_key(receipt_idx, public_key.decode()?, amount);
    Ok(())
}
