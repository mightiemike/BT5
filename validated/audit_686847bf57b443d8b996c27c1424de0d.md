### Title
Unprivileged Caller Can Aggressively Drain the Mainchain Header Store via `run_mainchain_gc` - (File: `contract/src/lib.rs`)

---

### Summary

`run_mainchain_gc` is a public state-mutating function with no caller authentication. Any unprivileged NEAR account can call it with `batch_size = u64::MAX`, immediately removing every header that exceeds `gc_threshold` in a single transaction. This permanently destroys the ability to verify SPV proofs for those blocks.

---

### Finding Description

`run_mainchain_gc` carries only a `#[pause(except(roles(Role::UnrestrictedRunGC)))]` attribute. That attribute controls behavior when the contract is *paused*; it imposes no restriction on who may call the function when the contract is *running*. There is no `#[trusted_relayer]` guard, no `acl_require!`, and no `env::predecessor_account_id()` check. [1](#0-0) 

The function accepts a caller-supplied `batch_size: u64` and removes up to `min(total_excess, batch_size)` of the oldest mainchain headers: [2](#0-1) 

For each removed height it calls `remove_block_header`, which deletes the entry from both `mainchain_header_to_height` and `headers_pool`, and also removes the entry from `mainchain_height_to_header`. It then advances `mainchain_initial_blockhash` to the new oldest block. [3](#0-2) 

In normal operation, `submit_blocks` calls `run_mainchain_gc(num_of_headers)` where `num_of_headers` is the small count of headers submitted in that batch, so pruning is gradual and bounded. [4](#0-3) 

An attacker bypasses this rate control entirely by calling `run_mainchain_gc` directly with `u64::MAX`, collapsing all excess removal into one transaction.

---

### Impact Explanation

After the attacker's call, every pruned block hash is absent from `mainchain_header_to_height` and `headers_pool`. Both `verify_transaction_inclusion` and `verify_transaction_inclusion_v2` then panic with `"block does not belong to the current main chain"` for any transaction whose containing block was pruned: [5](#0-4) [6](#0-5) 

The deletion is permanent. The relayer cannot re-submit those headers because the contract's `mainchain_initial_blockhash` has been advanced past them; any attempt to re-attach them would require the contract to walk back through a chain segment it no longer stores. The light client's core SPV service is irreversibly broken for the pruned range.

Additionally, the CLAUDE.md documentation notes that aggressive GC can break chain reorganization: [7](#0-6) 

---

### Likelihood Explanation

The entry path requires no special role, no staked deposit, and no privileged key. Any NEAR account that can pay the gas cost of a single function call can trigger this. The contract is live and the function is part of its public ABI. The attacker only needs to wait until the chain has grown beyond `gc_threshold` (the recommended value is 52,704 blocks, roughly one year of Bitcoin headers), at which point a single call with `batch_size = u64::MAX` removes all excess headers atomically.

---

### Recommendation

Add a role-based access guard to `run_mainchain_gc` so that only trusted callers (e.g., `Role::DAO` or a dedicated `GCManager` role) can invoke it directly. The internal call from `submit_blocks` can bypass the guard via a private helper. For example:

```rust
// Public entry: restricted
#[pause(except(roles(Role::UnrestrictedRunGC)))]
pub fn run_mainchain_gc(&mut self, batch_size: u64) {
    require!(
        self.acl_has_role(Role::DAO, &env::predecessor_account_id())
            || self.acl_has_role(Role::RelayerManager, &env::predecessor_account_id()),
        "Unauthorized"
    );
    self.run_mainchain_gc_internal(batch_size);
}

// Private helper called by submit_blocks
fn run_mainchain_gc_internal(&mut self, batch_size: u64) { ... }
```

---

### Proof of Concept

1. Deploy the contract with `gc_threshold = 52704` and sync it until it holds more than 52,704 mainchain headers (i.e., the chain has grown past the threshold).
2. From any unprivileged NEAR account, call:
   ```
   run_mainchain_gc(18446744073709551615)   // u64::MAX
   ```
3. The contract immediately removes all `amount_of_headers_we_store - gc_threshold` oldest headers in one transaction, advancing `mainchain_initial_blockhash` to the new oldest block.
4. Any subsequent call to `verify_transaction_inclusion` or `verify_transaction_inclusion_v2` for a transaction in a pruned block panics with `"block does not belong to the current main chain"`, permanently breaking SPV proof verification for those blocks. [8](#0-7)

```rust
    #[pause(except(roles(Role::UnrestrictedRunGC)))]
    pub fn run_mainchain_gc(&mut self, batch_size: u64) {
        let initial_blockheader = self
            .headers_pool
            .get(&self.mainchain_initial_blockhash)
            .unwrap_or_else(|| env::panic_str(ERR_KEY_NOT_EXIST));

        let tip_blockheader = self
            .headers_pool
            .get(&self.mainchain_tip_blockhash)
            .unwrap_or_else(|| env::panic_str(ERR_KEY_NOT_EXIST));

        let amount_of_headers_we_store =
            tip_blockheader.block_height - initial_blockheader.block_height + 1;

        if amount_of_headers_we_store > self.gc_threshold {
            let total_amount_to_remove = amount_of_headers_we_store - self.gc_threshold;
            let selected_amount_to_remove = std::cmp::min(total_amount_to_remove, batch_size);

            let start_removal_height = initial_blockheader.block_height;
            let end_removal_height = initial_blockheader.block_height + selected_amount_to_remove;
            env::log_str(&format!(
                "Num of blocks to remove {selected_amount_to_remove}"
            ));

            for height in start_removal_height..end_removal_height {
                let blockhash = &self
                    .mainchain_height_to_header
                    .get(&height)
                    .unwrap_or_else(|| env::panic_str(ERR_KEY_NOT_EXIST));

                self.remove_block_header(blockhash);
                self.mainchain_height_to_header.remove(&height);
            }

            self.mainchain_initial_blockhash = self
                .mainchain_height_to_header
                .get(&end_removal_height)
                .unwrap_or_else(|| env::panic_str(ERR_KEY_NOT_EXIST));
        }
    }
```
