Looking at the extension system, I need to trace how the `SwapAllowlistExtension` guard is applied when a router intermediates the swap.

**Key code path:**

In `MetricOmmPool.swap`, the `sender` forwarded to extensions is `msg.sender` — the direct pool caller: [1](#0-0) 

In `ExtensionCalling._beforeSwap`, that same `sender` is encoded into the extension call: [2](#0-1) 

`SwapAllowlistExtension.beforeSwap` then checks `allowedSwapper[msg.sender][sender]` — i.e., `allowedSwapper[pool][direct_pool_caller]`: [3](#0-2) 

When `MetricOmmSimpleRouter.exactInputSingle` is used, the router is the direct pool caller: [4](#0-3) 

So the extension checks whether the **router** is allowlisted, not the actual end-user. Compare this to `DepositAllowlistExtension`, which correctly checks `owner` (the actual position owner), not `sender`: [5](#0-4)
