### Title
`ValidResourceBounds::AllResources` with zero L2/L1DataGas silently downcasts to `ValidResourceBounds::L1Gas` during protobuf deserialization, producing a divergent transaction hash - (`File: crates/apollo_protobuf/src/converters/transaction.rs`)

### Summary

The protobuf deserializer for `ValidResourceBounds` uses a zero-value check on `l2_gas` and `l1_data_gas` to decide whether to reconstruct `L1Gas` or `AllResources`. A V3 transaction stored internally as `AllResources { l1_gas: X, l2_gas: 0, l1_data_gas: 0 }` serializes to the same protobuf bytes as a legacy `L1Gas(X)` transaction. On deserialization the peer reconstructs `L1Gas(X)`. Because `get_tip_resource_bounds_hash` includes an extra `L1_DATA_GAS` felt for `AllResources` but not for `L1Gas`, the two representations produce different transaction hashes. The original sequencer node and every peer that re-derives the hash from the block body therefore disagree on the transaction hash, breaking the `transaction_commitment` in the block header.

### Finding Description

**Serialization path** (`crates/apollo_protobuf/src/converters/transaction.rs`, lines 471–490):

```rust
ValidResourceBounds::L1Gas(l1_gas) => protobuf::ResourceBounds {
    l1_gas: Some(l1_gas.into()),
    l2_gas: Some(value.get_l2_bounds().into()),   // returns ResourceBounds::default() → zero
    l1_data_gas: Some(ResourceBounds::default().into()), // zero
},
ValidResourceBounds::AllResources(AllResourceBounds { l1_gas, l2_gas, l1_data_gas }) =>
    protobuf::ResourceBounds { l1_gas: Some(l1_gas.into()), l2_gas: Some(l2_gas.into()),
                                l1_data_gas: Some(l1_data_gas.into()) },
```

When `AllResources { l2_gas: 0, l1_data_gas: 0 }` is serialized, the wire bytes are identical to those produced by `L1Gas`.

**Deserialization path** (`crates/apollo_protobuf/src/converters/transaction.rs`, lines 417–436):

```rust
Ok(if l1_data_gas.is_zero() && l2_gas.is_zero() {
    ValidResourceBounds::L1Gas(l1_gas)          // ← wrong variant chosen
} else {
    ValidResourceBounds::AllResources(AllResourceBounds { l1_gas, l2_gas, l1_data_gas })
})
```

The condition is true for both a genuine `L1Gas` transaction **and** an `AllResources` transaction whose l2/l1_data bounds happen to be zero. The original variant is irrecoverably lost.

**Hash divergence** (`crates/starknet_api/src/transaction_hash.rs`, lines 188–210):

```rust
resource_felts.extend(match resource_bounds {
    ValidResourceBounds::L1Gas(_) => vec![],                          // no L1_DATA_GAS felt
    ValidResourceBounds::AllResources(all_resources) => {
        vec![get_concat_resource(&all_resources.l1_data_gas, L1_DATA_GAS)?]  // extra felt
    }
});
```

`AllResources { l1_data_gas: 0 }` hashes `concat(0, L1_DATA_GAS)` into the preimage; `L1Gas` omits it entirely. The resulting `tip_resource_bounds_hash` values differ, so the full transaction hash differs.

**Trigger condition**: A V3 invoke/declare/deploy-account transaction submitted with `AllResourceBounds { l1_gas: X, l2_gas: 0, l1_data_gas: 0 }` passes gateway admission when `min_gas_price = 0` (the `max_price_per_unit.0 < min_gas_price` check at `crates/apollo_gateway/src/stateless_transaction_validator.rs` line 71 evaluates `0 < 0 = false`). The same transaction can also enter via the consensus path, bypassing the gateway entirely.

### Impact Explanation

When the sequencer proposes a block containing such a transaction, every peer that re-derives the transaction hash from the block body (to verify `transaction_commitment` in the block header) computes a different hash than the proposer. The `transaction_commitment` mismatch causes peers to reject the block. This maps to:

> **Critical. Wrong state, receipt, event, L1 message, class hash, storage value, or revert result from blockifier/syscall/execution logic for accepted input.**

and

> **Critical. Invalid or unauthorized Starknet transaction accepted through account validation, signature, nonce, chain id, fee/resource bound, paymaster, or account-deployment logic.**

### Likelihood Explanation

The trigger requires `min_gas_price = 0` (or consensus-path injection). The default config schema (`crates/apollo_node/resources/config_schema.json`) does not hard-code a non-zero floor, so operator misconfiguration or a test/staging deployment is sufficient. The condition is also reachable from any peer that can inject a consensus transaction directly.

### Recommendation

The serialization must embed a discriminant bit so the deserializer can distinguish `L1Gas` from `AllResources` with zero secondary bounds. One approach: add a boolean `is_all_resources` field to `protobuf::ResourceBounds` and set it unconditionally on serialization. Alternatively, always serialize `AllResources` with a sentinel non-zero value in a reserved field, or use a protobuf `oneof` to encode the variant explicitly.

The deserialization guard should be:

```rust
// Use an explicit variant tag, not a value-based heuristic.
if value.is_all_resources {
    ValidResourceBounds::AllResources(...)
} else {
    ValidResourceBounds::L1Gas(l1_gas)
}
```

### Proof of Concept

1. Craft a V3 invoke transaction with `AllResourceBounds { l1_gas: GasAmount(1000)/GasPrice(1), l2_gas: 0/0, l1_data_gas: 0/0 }`.
2. Submit to a node with `min_gas_price = 0`; the gateway accepts it.
3. The node computes `tx_hash_A` using `AllResources` (includes `concat(0, L1_DATA_GAS)` in the hash preimage).
4. The node proposes a block; the block body is serialized to protobuf via `From<ValidResourceBounds> for protobuf::ResourceBounds`.
5. A peer deserializes the block body; the `TryFrom<protobuf::ResourceBounds> for ValidResourceBounds` zero-check fires and produces `L1Gas(1000/1)`.
6. The peer computes `tx_hash_B` using `L1Gas` (omits `L1_DATA_GAS` felt).
7. `tx_hash_A ≠ tx_hash_B`; the peer's recomputed `transaction_commitment` does not match the block header; the block is rejected. [1](#0-0) [2](#0-1) [3](#0-2) [4](#0-3)

pub fn get_tip_resource_bounds_hash(
    resource_bounds: &ValidResourceBounds,
    tip: &Tip,
) -> Result<Felt, StarknetApiError> {
    let l1_resource_bounds = resource_bounds.get_l1_bounds();
    let l2_resource_bounds = resource_bounds.get_l2_bounds();

    // L1 and L2 gas bounds always exist.
    // Old V3 txs always have L2 gas bounds of zero, but they exist.
    let mut resource_felts = vec![
        get_concat_resource(&l1_resource_bounds, L1_GAS)?,
        get_concat_resource(&l2_resource_bounds, L2_GAS)?,
    ];

    // For new V3 txs, need to also hash the data gas bounds.
    resource_felts.extend(match resource_bounds {
        ValidResourceBounds::L1Gas(_) => vec![],
        ValidResourceBounds::AllResources(all_resources) => {
            vec![get_concat_resource(&all_resources.l1_data_gas, L1_DATA_GAS)?]
        }
    });

    Ok(HashChain::new().chain(&tip.0.into()).chain_iter(resource_felts.iter()).get_poseidon_hash())
