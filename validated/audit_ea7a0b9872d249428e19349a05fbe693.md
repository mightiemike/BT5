### Title
Unvalidated `_cached_sha256_treehash` Accepted from Caller-Supplied Python Objects Produces Forged Puzzle Hashes — (`wheel/python/clvm_rs/program.py`, `wheel/python/clvm_rs/tree_hash.py`)

---

### Summary

`Program.wrap()` blindly copies `_cached_sha256_treehash` from any caller-supplied Python object without verifying that the cached value matches the object's actual tree content. `Program.tree_hash()` then returns that unverified value directly. Any downstream consumer — including `Program.__eq__()` and `Program.curry_hash()` — receives a hash that is inconsistent with the program's real content, exactly mirroring the external report's pattern of two related pieces of data (shnarf ↔ state root hash) being accepted together without cross-validation.

---

### Finding Description

**Vulnerability class:** hash/state mismatch — two correlated values (tree content and its SHA-256 tree hash) are accepted together from an external source without being validated against each other.

**Root cause — `Program.wrap()` (`wheel/python/clvm_rs/program.py`, lines 133–143):**

```python
@classmethod
def wrap(cls, v: CLVMStorage) -> Program:
    if isinstance(v, Program):
        return v
    o = cls()
    o.atom = v.atom
    o._pair = None
    o._unwrapped = v
    o._unwrapped_pair = v.pair
    o._cached_serialization = getattr(v, "_cached_serialization", None)
    o._cached_sha256_treehash = getattr(v, "_cached_sha256_treehash", None)  # ← copied verbatim
    return o
```

`_cached_sha256_treehash` is read from `v` with `getattr` and stored directly. No recomputation or comparison against the actual atom/pair content is performed.

**Root cause — `Program.tree_hash()` (`wheel/python/clvm_rs/program.py`, lines 281–286):**

```python
def tree_hash(self) -> bytes:
    if self._cached_sha256_treehash is None:
        self._cached_sha256_treehash = sha256_treehash(self._unwrapped)
    return self._cached_sha256_treehash
```

If `_cached_sha256_treehash` is already set (because `wrap()` copied it from the caller), `sha256_treehash` is never called. The forged value is returned as-is.

**Root cause — `Treehasher.sha256_treehash()` (`wheel/python/clvm_rs/tree_hash.py`, lines 57–61):**

```python
r = getattr(obj, "_cached_sha256_treehash", None)
if r is not None:
    self.cache_hits += 1
    hash_stack.append(r)
    return
```

Even when `sha256_treehash` is called on `self._unwrapped`, it immediately returns whatever `_cached_sha256_treehash` is set on the underlying object — again without recomputing.

**Downstream consumers that propagate the forged hash:**

- `Program.__eq__()` (line 188): `return self.tree_hash() == other_obj.tree_hash()` — equality is decided by the forged hash.
- `Program.curry_hash()` (lines 358–362): passes `self.tree_hash()` into `calculate_hash_of_quoted_mod_hash`, producing a wrong puzzle hash.

---

### Impact Explanation

An attacker who can pass a Python object with a pre-set `_cached_sha256_treehash` to any code path that calls `Program.wrap()` or `Program.to()` can:

1. **Forge puzzle identity:** `Program.__eq__()` uses `tree_hash()` for equality. Two programs with completely different content but the same forged hash compare as equal.
2. **Forge puzzle hashes:** `Program.curry_hash()` computes the curried puzzle hash from `self.tree_hash()`. A forged hash produces a wrong puzzle hash, which is the primary identifier for Chia coins. Wallet code that derives coin IDs or verifies puzzle reveals using `curry_hash()` or `tree_hash()` would accept an incorrect hash as authoritative.
3. **Corrupt the hash cache for sub-trees:** `Treehasher.handle_pair()` writes the computed pair hash back to `_cached_sha256_treehash` on sub-objects. If a forged sub-tree hash is trusted, all ancestor hashes computed from it are also wrong.

The corrupted value is the `bytes` returned by `Program.tree_hash()` — a 32-byte SHA-256 digest that does not correspond to the actual CLVM tree.

---

### Likelihood Explanation

`Program.to()` and `Program.wrap()` are the primary public entry points for the Python API and are called throughout the Chia wallet and puzzle toolchain. `CastableType` (the accepted input type) includes arbitrary Python objects implementing the `CLVMStorage` protocol (`.atom` / `.pair` attributes). Any Python object can have `_cached_sha256_treehash` set as a plain attribute — no subclassing or special access is required. A malicious integrator, a crafted puzzle library, or any code path that accepts user-supplied `CLVMStorage`-compatible objects can trigger this without any privileged access.

---

### Recommendation

In `Program.wrap()`, do not copy `_cached_sha256_treehash` from the input object. Remove line 142:

```python
# Remove this line:
o._cached_sha256_treehash = getattr(v, "_cached_sha256_treehash", None)
```

Leave `_cached_sha256_treehash` as `None` so that `Program.tree_hash()` always computes it from the actual content on first access. Alternatively, if the performance optimization is required, recompute and verify the hash before caching it:

```python
cached = getattr(v, "_cached_sha256_treehash", None)
if cached is not None:
    # Validate before trusting
    actual = sha256_treehash(v)
    o._cached_sha256_treehash = actual if actual == cached else None
else:
    o._cached_sha256_treehash = None
```

Similarly, `Treehasher.sha256_treehash()` should not short-circuit on `_cached_sha256_treehash` from objects it did not itself hash.

### Proof of Concept

```python
from clvm_rs.program import Program

class ForgeryObject:
    """A CLVMStorage-compatible object with a pre-forged tree hash."""
    atom = b"real content"
    pair = None
    _cached_sha256_treehash = b"\x00" * 32  # forged: all-zero hash

obj = ForgeryObject()
p = Program.wrap(obj)

# tree_hash() returns the forged value, not sha256tree(b"real content")
assert p.tree_hash() == b"\x00" * 32, "Forged hash accepted"

# Equality is now broken: two programs with different content compare equal
# if they share the same forged _cached_sha256_treehash
p2 = Program.new_atom(b"different content")
p2._cached_sha256_treehash = b"\x00" * 32
assert p == p2, "Programs with different content compare as equal"

# curry_hash() produces a wrong puzzle hash
wrong_puzzle_hash = p.curry_hash()
# wrong_puzzle_hash is derived from 0x00..00 instead of the real tree hash
