### Title
Duplicate Position IDs in `MsgClaimTierRewards` Enable Double-Claim of Bonus Rewards - (File: x/tieredrewards/keeper/msg_server.go)

---

### Summary

`ClaimTierRewards` loads all requested positions into an in-memory slice before processing them. There is no deduplication of `PositionIds`. If an attacker submits the same position ID twice, both entries in the slice carry the same stale `LastEventSeq` checkpoint. The second iteration re-processes the same validator events and pays out bonus rewards a second time, draining the `RewardsPoolName` module account.

---

### Finding Description

`ClaimTierRewards` in `x/tieredrewards/keeper/msg_server.go` first loads every requested position into a `[]types.PositionState` slice, then hands the whole slice to `claimRewardsAndUpdatesPositions`:

```go
// msg_server.go lines 434-448
positions := make([]types.PositionState, 0, len(msg.PositionIds))
for _, posId := range msg.PositionIds {
    pos, err := ms.getPositionState(ctx, posId)
    ...
    positions = append(positions, pos)
}
totalBase, totalBonus, err := ms.claimRewardsAndUpdatesPositions(ctx, positions)
``` [1](#0-0) 

`claimRewardsAndUpdatesPositions` iterates the slice sequentially. For each entry it calls `processEventsAndClaimBonus`, which reads validator events from the store starting at `pos.LastEventSeq`, pays out bonus coins via `bankKeeper.SendCoinsFromModuleToAccount`, decrements each event's reference count, and updates `pos.LastEventSeq` **only in the in-memory struct**. The persisted state is written back by `setPosition` at the end of each loop body:

```go
// claim_rewards.go lines 110-134
for i := range positions {
    pos := &positions[i]
    ...
    bonus, err := k.processEventsAndClaimBonus(ctx, pos)
    ...
    if err := k.setPosition(ctx, pos.Position, nil); err != nil { ... }
}
``` [2](#0-1) 

When the same position ID appears twice in `msg.PositionIds`, both `positions[0]` and `positions[1]` are loaded with the identical initial `LastEventSeq = X`. After the first iteration:

- Bonus is paid to the owner.
- Each processed event's reference count is decremented by one.
- `positions[0].LastEventSeq` is advanced to `Y` in memory.
- The updated state is persisted via `setPosition`.

The second iteration starts with `positions[1].LastEventSeq = X` (the stale snapshot). `processEventsAndClaimBonus` queries the store for events since `X`. Because the reference counts were only decremented from `N` to `N-1` (not to zero) — which is the normal case when any other position is also delegated to the same validator — those events are still present. The function computes the same bonus and issues a second `SendCoinsFromModuleToAccount` transfer. [3](#0-2) 

The stale-state pattern is structurally identical to the external report: in the MIMO bug, vault B's value was read before being updated, allowing a second rebalance to validate against the same pre-update figure. Here, `LastEventSeq` is read before being persisted, allowing a second loop iteration to replay the same events.

---

### Impact Explanation

The attacker receives bonus rewards from `types.RewardsPoolName` twice for a single accrual period. Each duplicate entry in `PositionIds` multiplies the payout. Additionally, the second iteration over-decrements the reference counts of validator events that other positions still need to process, silently destroying those events and permanently preventing other legitimate positions from ever claiming their own bonus rewards for the same period.

The corrupted value is: **bonus coin balance of the `RewardsPoolName` module account** and **validator event reference counts** in the tiered-rewards store.

---

### Likelihood Explanation

The attack requires only a standard `MsgClaimTierRewards` transaction with a repeated position ID — no special privilege, no governance access, no key compromise. The only precondition is that at least one other position is delegated to the same validator (so event reference counts are ≥ 2 before the attack), which is the normal production state on any active validator. The attacker controls only their own position and the contents of `PositionIds`.

---

### Recommendation

Deduplicate `msg.PositionIds` before loading positions, or add an explicit duplicate check in `MsgClaimTierRewards.Validate()`. A simple approach:

```go
seen := make(map[uint64]struct{}, len(msg.PositionIds))
for _, id := range msg.PositionIds {
    if _, dup := seen[id]; dup {
        return nil, errorsmod.Wrapf(sdkerrors.ErrInvalidRequest, "duplicate position id %d", id)
    }
    seen[id] = struct{}{}
}
```

This mirrors the fix applied in the MIMO report: preventing re-entry into the same logical resource within a single operation.

---

### Proof of Concept

**Setup:**
- Validator V has two positions delegated to it: Position 1 (owned by attacker) and Position 2 (owned by any other user). Both have `LastEventSeq = 5`. Validator events 1–5 have reference count = 2.
- The `RewardsPoolName` module account holds sufficient bonus tokens.

**Attack:**
1. Attacker submits `MsgClaimTierRewards{ Owner: attacker, PositionIds: [1, 1] }`.
2. Loading phase: `positions[0]` and `positions[1]` are both loaded with `LastEventSeq = 5`.
3. First loop iteration (`i=0`): `processEventsAndClaimBonus` fetches events 1–5 (ref count = 2), computes bonus B, sends B to attacker, decrements ref counts to 1, advances `positions[0].LastEventSeq` to latest. `setPosition` persists the updated state.
4. Second loop iteration (`i=1`): `processEventsAndClaimBonus` fetches events 1–5 again (ref count = 1 > 0), computes the same bonus B, sends B to attacker again, decrements ref counts to 0 (events deleted). `setPosition` persists.
5. Attacker has received 2×B bonus tokens. Position 2 can no longer claim its bonus for events 1–5 because those events have been deleted. [4](#0-3) [5](#0-4) [6](#0-5)

func (ms msgServer) ClaimTierRewards(ctx context.Context, msg *types.MsgClaimTierRewards) (*types.MsgClaimTierRewardsResponse, error) {
	if err := msg.Validate(); err != nil {
		return nil, err
	}

	positions := make([]types.PositionState, 0, len(msg.PositionIds))
	for _, posId := range msg.PositionIds {
		pos, err := ms.getPositionState(ctx, posId)
		if err != nil {
			return nil, err
		}

		if err := ms.validateClaimRewards(pos.Position, msg.Owner); err != nil {
			return nil, err
		}

		positions = append(positions, pos)
	}

	totalBase, totalBonus, err := ms.claimRewardsAndUpdatesPositions(ctx, positions)
	if err != nil {
		return nil, err
	}

	sdkCtx := sdk.UnwrapSDKContext(ctx)
	if err := sdkCtx.EventManager().EmitTypedEvent(&types.EventTierRewardsClaimed{
		Owner:        msg.Owner,
		PositionIds:  msg.PositionIds,
		BaseRewards:  totalBase,
		BonusRewards: totalBonus,
	}); err != nil {
		return nil, err
	}

	return &types.MsgClaimTierRewardsResponse{
		BaseRewards:  totalBase,
		BonusRewards: totalBonus,
		PositionIds:  msg.PositionIds,
	}, nil
}

func (k Keeper) claimRewardsAndUpdatesPositions(ctx context.Context, positions []types.PositionState) (sdk.Coins, sdk.Coins, error) {
	totalBase := sdk.NewCoins()
	totalBonus := sdk.NewCoins()

	for i := range positions {
		pos := &positions[i]

		if !pos.IsDelegated() {
			continue
		}

		base, err := k.claimBaseRewards(ctx, *pos)
		if err != nil {
			return nil, nil, err
		}
		totalBase = totalBase.Add(base...)

		bonus, err := k.processEventsAndClaimBonus(ctx, pos)
		if err != nil {
			return nil, nil, err
		}
		totalBonus = totalBonus.Add(bonus...)

		if err := k.setPosition(ctx, pos.Position, nil); err != nil {
			return nil, nil, err
		}
	}

	return totalBase, totalBonus, nil
}

func (k Keeper) processEventsAndClaimBonus(ctx context.Context, pos *types.PositionState) (sdk.Coins, error) {
	// Rewards should have been claimed before undelegation
	if !pos.IsDelegated() {
		return sdk.NewCoins(), nil
	}

	valAddr, err := sdk.ValAddressFromBech32(pos.Delegation.ValidatorAddress)
	if err != nil {
		return nil, err
	}

	events, err := k.getValidatorEventsSince(ctx, valAddr, pos.LastEventSeq)
	if err != nil {
		return nil, err
	}

	sdkCtx := sdk.UnwrapSDKContext(ctx)
	blockTime := sdkCtx.BlockTime()

	totalBonus := math.ZeroInt()
	// Use the persisted bonded state from the last replay, not a hardcoded default.
	// This prevents overpaying bonus for unbonded gaps between claims.
	bonded := pos.LastKnownBonded
	segmentStart := pos.LastBonusAccrual

	tier, err := k.getTier(ctx, pos.TierId)
	if err != nil {
		return nil, err
	}

	for _, entry := range events {
		evt := entry.Event

		if bonded {
			// Compute bonus for the bonded segment [segmentStart, eventTime]
			// using the snapshot rate at the event.
			bonus := k.computeSegmentBonus(*pos, tier, segmentStart, evt.Timestamp, evt.TokensPerShare)
			totalBonus = totalBonus.Add(bonus)
		}

		// Update bonded state based on event type.
		switch evt.EventType {
		case types.ValidatorEventType_VALIDATOR_EVENT_TYPE_UNBOND:
			bonded = false
		case types.ValidatorEventType_VALIDATOR_EVENT_TYPE_BOND:
			bonded = true
		case types.ValidatorEventType_VALIDATOR_EVENT_TYPE_SLASH:
			// Slash doesn't change bonded state.
		}

		segmentStart = evt.Timestamp
		pos.UpdateLastEventSeq(entry.Seq)

		// Decrement reference count.
		if err := k.decrementEventRefCount(ctx, valAddr, entry.Seq); err != nil {
			return nil, err
		}
	}

	val, err := k.stakingKeeper.GetValidator(ctx, valAddr)
	if err != nil {
		return nil, err
	}
	// Defensive: validator bond status check
	if bonded && val.IsBonded() {
		currentRate, err := k.getTokensPerShare(ctx, valAddr)
		if err != nil {
			return nil, err
		}
		bonus := k.computeSegmentBonus(*pos, tier, segmentStart, blockTime, currentRate)
		totalBonus = totalBonus.Add(bonus)
	}

	applyBonusAccrualCheckpoint(&pos.Position, blockTime)
	// Persist the bonded state so the next replay starts correctly.
	pos.UpdateLastKnownBonded(bonded)

	if totalBonus.IsZero() {
		return sdk.NewCoins(), nil
	}

	bondDenom, err := k.stakingKeeper.BondDenom(ctx)
	if err != nil {
		return nil, err
	}

	bonusCoins := sdk.NewCoins(sdk.NewCoin(bondDenom, totalBonus))

	if err := k.sufficientBonusPoolBalance(ctx, bonusCoins); err != nil {
		return nil, err
	}

	ownerAddr, err := sdk.AccAddressFromBech32(pos.Owner)
	if err != nil {
		return nil, errorsmod.Wrap(sdkerrors.ErrInvalidAddress, "invalid owner address")
	}

	if err := k.bankKeeper.SendCoinsFromModuleToAccount(ctx, types.RewardsPoolName, ownerAddr, bonusCoins); err != nil {
		return nil, err
	}

	if err := sdkCtx.EventManager().EmitTypedEvent(&types.EventBonusRewardsClaimed{
		PositionId: pos.Id,
		Owner:      pos.Owner,
		Rewards:    bonusCoins,
	}); err != nil {
		return nil, err
	}

	return bonusCoins, nil
