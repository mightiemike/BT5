### Title
`SwapAllowlistExtension` gates the router address instead of the actual user, allowing any caller to bypass the per-user swap allowlist via `MetricOmmSimpleRouter` — (`metric-periphery/contracts/extensions/SwapAllowlistExtension.sol`)

---

### Summary

`SwapAllowlistExtension.beforeSwap` checks the `sender` argument against the allowlist. When a swap is routed through `MetricOmmSimpleRouter`, `sender` is the **router address**, not the actual user. A pool admin who allowlists the router (required for router-mediated swaps to function) inadvertently opens the allowlist to every user on earth.

---

### Finding Description

`MetricOmmPool.swap` passes `msg.sender` as the first argument (`sender`) to `_beforeSwap`, which forwards it verbatim to every configured extension: [1](#0-0) 

`ExtensionCalling._beforeSwap` encodes that value as the `sender` field of the `IMetricOmmExtensions.beforeSwap` call: [2](#0-1) 

`SwapAllowlistExtension.beforeSwap` then checks `allowedSwapper[msg.sender][sender]`, where `msg.sender` is the pool and `sender` is the value just described: [3](#0-2) 

When a user calls `MetricOmmSimpleRouter.exactInputSingle`, the router calls `pool.swap(...)` directly with no forwarding of the original caller: [4](#0-3) 

The pool therefore sees `msg.sender = router`. The extension receives `sender = router` and evaluates `allowedSwapper[pool][router]`. The actual user's address is never consulted.

The same pattern applies to `exactInput`, `exactOutputSingle`, and `exactOutput`.

**Contrast with `DepositAllowlistExtension`:** that extension correctly ignores `sender` (the adder contract) and checks `owner` (the LP position owner), which is the economically relevant actor regardless of entry path: [5](#0-4) 

`SwapAllowlistExtension` has no equivalent correction.

---

### Impact Explanation

A pool configured with `SwapAllowlistExtension` is intended to restrict swaps to a curated set of addresses. Once the pool admin allowlists the router (the only way to make router-mediated swaps work), the allowlist is effectively disabled for all users: any address can call `exactInputSingle` through the router and execute a swap that the allowlist was supposed to block. This is a direct policy bypass with fund-impacting consequences on curated pools (e.g., KYC-gated or institutional pools where unrestricted trading is a protocol invariant).

---

### Likelihood Explanation

The `MetricOmmSimpleRouter` is the canonical periphery swap contract. Any pool operator who deploys a `SwapAllowlistExtension`-guarded pool and also wants to support the standard router must allowlist the router. This is the expected operational path, making the bypass reachable by any unprivileged user with no special preconditions.

---

### Recommendation

The extension must check the identity of the **economic actor**, not the intermediary. Two viable approaches:

1. **Check `sender` only for direct pool calls; require the router to embed the real user in `extensionData`** — the extension decodes the real caller from `extensionData` when `sender` is a known router, and falls back to `sender` otherwise.

2. **Mirror the `DepositAllowlistExtension` design** — have the pool (or router) pass the original `msg.sender` as a dedicated `swapper` field that the extension checks, separate from the call-chain `sender`. The router would need to forward `msg.sender` explicitly (e.g., as the first word of `extensionData`).

Either way, the allowlist mapping key must resolve to the address that controls the economic decision to swap, not the address of the contract that mechanically forwards the call.

---

### Proof of Concept

```
Setup:
  pool configured with SwapAllowlistExtension
  allowedSwapper[pool][alice] = true          // alice is the intended gated user
  allowedSwapper[pool][router] = true         // admin must set this for router swaps to work
  allowedSwapper[pool][attacker] = false      // attacker is NOT on the allowlist

Attack:
  attacker calls MetricOmmSimpleRouter.exactInputSingle({
      pool:      pool,
      recipient: attacker,
      tokenIn:   token0,
      amountIn:  X,
      ...
  })

  router calls pool.swap(attacker_recipient, ...)
    pool: msg.sender = router
    pool calls _beforeSwap(sender=router, ...)
    extension checks allowedSwapper[pool][router] == true  ✓
    swap executes — attacker receives output tokens

Result:
  attacker bypassed the per-user allowlist entirely.
  The check that was supposed to block attacker evaluated the router's
  allowlist entry instead of attacker's entry.
```
