### Title
Unprivileged Caller Can Permanently Prune Canonical Chain State via Unrestricted `run_mainchain_gc` - (File: contract/src/lib.rs)

### Summary
`run_mainchain_gc` is a public, state-mutating NEAR contract method with no caller access control. Any unprivileged NEAR account can invoke it to permanently delete verified Bitcoin block headers from the canonical chain, breaking SPV proof verification and chain reorganization for all downstream consumers.

### Finding Description
`run_mainchain_gc` is declared `pub` and carries only a `#[pause(except(roles(Role::UnrestrictedRunGC)))]` attribute:

```rust
#[pause(except(roles(Role::UnrestrictedRunGC)))]
pub fn run_mainchain_gc(&mut self, batch_size: u64) {
```

The `#[pause]` macro gates access only when the contract is **paused** — privileged accounts with `UnrestrictedRunGC` can bypass the pause gate. When the contract is running normally (unpaused, which is the production state), there is **zero caller restriction**: no `#[trusted_relayer]`, no role check, no `#[private]`. Any NEAR account ID can call this method directly. [1](#0-0) 

Contrast this with `submit_blocks`, which carries both `#[pause]` and `#[trusted_relayer]` to enforce that only staked, registered relayers may mutate chain state: [2](#0-1) 

The GC function permanently removes the oldest mainchain block headers from all three storage maps (`headers_pool`, `mainchain_height_to_header`, `mainchain_header_to_height`) and advances `mainchain_initial_blockhash` forward: [3](#0-2) 

### Impact Explanation
An attacker calling `run_mainchain_gc(u64::MAX)` in a single transaction prunes the maximum possible number of mainchain headers (all blocks older than `gc_threshold`) in one shot. This causes:

1. **SPV proof breakage**: `verify_transaction_inclusion` and `verify_transaction_inclusion_v2` look up `tx_block_blockhash` in `mainchain_header_to_height`. After GC, pruned blocks are absent from this map, causing the call to panic with `"block does not belong to the current main chain"` — permanently invalidating all pending or future SPV proofs for those heights. [4](#0-3) 

2. **Chain reorganization breakage**: The reorg walk in `reorg_chain` traverses fork headers backward until it finds a block present in `mainchain_header_to_height`. If GC has pruned the common ancestor, the walk reaches a block whose `prev_block_hash` is no longer in `headers_pool`, causing a panic with `"previous fork block should be there"` — permanently disabling reorg for any fork that diverges before the new `mainchain_initial_blockhash`. [5](#0-4) 

3. **Canonical state corruption**: `mainchain_initial_blockhash` is permanently advanced, destroying the historical anchor of the verified chain. [6](#0-5) 

### Likelihood Explanation
The attack requires no privileged keys, no staking, no special role, and no deposit. Any NEAR account can submit a single transaction calling `run_mainchain_gc` with `batch_size = u64::MAX`. The function is documented as a "Public call" in its own doc comment, making it trivially discoverable. The contract is live on NEAR mainnet and the method is part of its published ABI. [7](#0-6) 

### Recommendation
Add a caller access control guard equivalent to `submit_blocks`. The simplest fix is to require the caller to hold a designated role (e.g., `UnrestrictedRunGC`, `DAO`, or a new `GCManager` role) before the function body executes, or apply `#[trusted_relayer]` so only registered relayers can trigger manual GC. The `#[pause(except(...))]` attribute must not be mistaken for an authorization check — it only controls behavior under the paused state.

    /// Public call to run GC on a mainchain.
    /// `batch_size` is how many block headers should be removed in the execution
    ///
    /// # Panics
    /// If initial blockheader or tip blockheader are not in a header pool
    #[pause(except(roles(Role::UnrestrictedRunGC)))]
    pub fn run_mainchain_gc(&mut self, batch_size: u64) {
        let initial_blockheader = self
            .headers_pool
            .get(&self.mainchain_initial_blockhash)
            .unwrap_or_else(|| env::panic_str(ERR_KEY_NOT_EXIST));

        let tip_blockheader = self
            .headers_pool
            .get(&self.mainchain_tip_blockhash)
            .unwrap_or_else(|| env::panic_str(ERR_KEY_NOT_EXIST));

        let amount_of_headers_we_store =
            tip_blockheader.block_height - initial_blockheader.block_height + 1;

        if amount_of_headers_we_store > self.gc_threshold {
            let total_amount_to_remove = amount_of_headers_we_store - self.gc_threshold;
            let selected_amount_to_remove = std::cmp::min(total_amount_to_remove, batch_size);

            let start_removal_height = initial_blockheader.block_height;
            let end_removal_height = initial_blockheader.block_height + selected_amount_to_remove;
            env::log_str(&format!(
                "Num of blocks to remove {selected_amount_to_remove}"
            ));

            for height in start_removal_height..end_removal_height {
                let blockhash = &self
                    .mainchain_height_to_header
                    .get(&height)
                    .unwrap_or_else(|| env::panic_str(ERR_KEY_NOT_EXIST));

                self.remove_block_header(blockhash);
                self.mainchain_height_to_header.remove(&height);
            }

            self.mainchain_initial_blockhash = self
                .mainchain_height_to_header
                .get(&end_removal_height)
                .unwrap_or_else(|| env::panic_str(ERR_KEY_NOT_EXIST));
        }
    }
```
