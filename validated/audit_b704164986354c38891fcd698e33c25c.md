### Title
`updateTierFeeRates` Accepts Unbounded `takerRateX18` / `makerRateX18`, Breaking Fee Arithmetic for All Traders in Affected Tier - (File: core/contracts/OffchainExchange.sol)

### Summary
`OffchainExchange.updateTierFeeRates` stores `makerRateX18` and `takerRateX18` with no upper-bound check. If either value reaches or exceeds `ONE` (1e18 = 100%), the `applyFee` function computes a non-positive `keepRateX18`, causing fee arithmetic to either divide by zero or invert sign, making every trade in the affected tier revert or drain the trader's quote balance beyond the trade value.

### Finding Description
`updateTierFeeRates` in `OffchainExchange.sol` writes the supplied rates directly into storage:

```solidity
feeRates[txn.tier][txn.productId] = FeeRates(
    txn.makerRateX18,
    txn.takerRateX18
);
```

No check enforces `takerRateX18 < ONE` or `makerRateX18 < ONE`. [1](#0-0) 

These rates are consumed in `applyFee`:

```solidity
int128 keepRateX18 = ONE - feeInfo.feeRate;
int128 newMeteredQuote = (meteredQuote > 0)
    ? meteredQuote.mul(keepRateX18)
    : meteredQuote.div(keepRateX18);
orderInfo.fee = meteredQuote - newMeteredQuote;
``` [2](#0-1) 

When `feeRate >= ONE`:
- `keepRateX18 <= 0`
- For a taker buy (`meteredQuote > 0`): `newMeteredQuote <= 0`, so `fee = meteredQuote - newMeteredQuote >= meteredQuote` — the fee equals or exceeds the entire trade value.
- For a maker (`meteredQuote < 0`): `meteredQuote.div(keepRateX18)` divides by zero when `keepRateX18 == 0`, or flips sign when `keepRateX18 < 0`, producing a corrupted `quoteDelta`.

The default taker rate is `200_000_000_000_000` (2 bps), confirming the intended ceiling is far below `ONE`. [3](#0-2) 

The setter is reached via `ContractOwner.updateTierFeeRates` (onlyOwner) → slow-mode transaction → `EndpointTx` → `OffchainExchange.updateTierFeeRates`. [4](#0-3) 

### Impact Explanation
Every trader assigned to the misconfigured fee tier would have their `quoteDelta` corrupted on every matched order: either the transaction reverts (division by zero) or the fee exceeds the trade value, draining the trader's quote balance beyond what the trade warrants. Because `updateTierFeeRates` can target `QUOTE_PRODUCT_ID` to broadcast the rate across **all** spot and perp products simultaneously, a single misconfigured call can disable trading protocol-wide. [5](#0-4) 

### Likelihood Explanation
The setter requires owner/sequencer access, matching the trust level of the external report's `onlyTreasury` setter. The risk is accidental misconfiguration (e.g., supplying a raw percentage integer like `10000` instead of the X18-scaled value `10000 * 1e14`), which is a realistic operator error given the X18 fixed-point encoding is not enforced anywhere in the call path. No malicious actor is required; a single off-by-scale input suffices.

### Recommendation
Add an upper-bound guard in `updateTierFeeRates` before writing to storage:

```diff
function updateTierFeeRates(IEndpoint.UpdateTierFeeRates memory txn)
    external
    onlyEndpoint
{
+   require(txn.takerRateX18 < ONE, "taker rate cannot reach 100%");
+   require(txn.makerRateX18 < ONE, "maker rate cannot reach 100%");
    ...
}
``` [6](#0-5) 

### Proof of Concept
1. Owner calls `ContractOwner.updateTierFeeRates` with `takerRateX18 = 1e18` (ONE) for tier 0, `productId = QUOTE_PRODUCT_ID` (broadcasts to all products). [4](#0-3) 
2. Slow-mode transaction is processed; `OffchainExchange.updateTierFeeRates` stores `takerRateX18 = 1e18` for every product. [7](#0-6) 
3. Any trader in tier 0 submits a taker order. `getUserFeeRateWithBuilder` returns `feeRate = 1e18`. [8](#0-7) 
4. In `applyFee`: `keepRateX18 = ONE - ONE = 0`. For `meteredQuote > 0`, `newMeteredQuote = meteredQuote.mul(0) = 0`, so `fee = meteredQuote`. For `meteredQuote < 0`, `meteredQuote.div(0)` reverts. [2](#0-1) 
5. All taker trades either revert or have 100% of their quote value taken as fee, making the protocol non-functional for all affected traders.

    function updateTierFeeRates(IEndpoint.UpdateTierFeeRates memory txn)
        external
        onlyEndpoint
    {
        if (txn.productId == QUOTE_PRODUCT_ID) {
            uint32[] memory spotProductIds = spotEngine.getProductIds();
            uint32[] memory perpProductIds = perpEngine.getProductIds();
            for (uint32 i = 0; i < spotProductIds.length; i++) {
                if (spotProductIds[i] == QUOTE_PRODUCT_ID) {
                    continue;
                }
                feeRates[txn.tier][spotProductIds[i]] = FeeRates(
                    txn.makerRateX18,
                    txn.takerRateX18
                );
            }
            for (uint32 i = 0; i < perpProductIds.length; i++) {
                feeRates[txn.tier][perpProductIds[i]] = FeeRates(
                    txn.makerRateX18,
                    txn.takerRateX18
                );
            }
        } else {
            feeRates[txn.tier][txn.productId] = FeeRates(
                txn.makerRateX18,
                txn.takerRateX18
            );
        }
        nonDefaultFeeTierMask |= uint128(1) << txn.tier;
    }

    function updateTierFeeRates(
        uint32[] memory tier,
        uint32[] memory productId,
        int128[] memory makerRateX18,
        int128[] memory takerRateX18
    ) external onlyOwner {
        if (
            tier.length != productId.length ||
            tier.length != makerRateX18.length ||
            tier.length != takerRateX18.length
        ) {
            revert InvalidInput();
        }
        for (uint256 i = 0; i < tier.length; i++) {
            IEndpoint.UpdateTierFeeRates memory _txn = IEndpoint
                .UpdateTierFeeRates(
                    tier[i],
                    productId[i],
                    makerRateX18[i],
                    takerRateX18[i]
                );
            _submitSlowModeTransaction(
                IEndpoint.TransactionType.UpdateTierFeeRates,
                abi.encode(_txn)
            );
        }
    }
