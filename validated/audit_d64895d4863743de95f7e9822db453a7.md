### Title
Governance `MsgUpdateTier` Permanently Blocked by Empty Bonus Pool in `claimRewardsAndUpdateTierPositions` - (File: x/tieredrewards/keeper/claim_rewards.go)

---

### Summary

When governance submits `MsgUpdateTier` to change a tier's `BonusApy`, the handler calls `claimRewardsAndUpdateTierPositions`, which iterates every position in the tier and calls `processEventsAndClaimBonus` for each. Unlike `slashRedelegationPosition`, which explicitly swallows `ErrInsufficientBonusPool` to prevent chain halt, `claimRewardsAndUpdateTierPositions` propagates **all** errors including `ErrInsufficientBonusPool`. If the bonus pool is empty and any position in the tier has accrued non-zero bonus, the entire `UpdateTier` governance execution fails. Governance cannot update `BonusApy` until the pool is externally replenished — a dependency that is nowhere enforced or documented, and that any actor can exploit by draining the pool before a governance proposal executes.

---

### Finding Description

`MsgUpdateTier` in `x/tieredrewards/keeper/msg_server_auth.go` calls `claimRewardsAndUpdateTierPositions` whenever `BonusApy` changes:

```go
// x/tieredrewards/keeper/msg_server_auth.go:67-71
if !oldTier.BonusApy.Equal(msg.Tier.BonusApy) {
    if err := ms.claimRewardsAndUpdateTierPositions(ctx, msg.Tier.Id); err != nil {
        return nil, err
    }
}
``` [1](#0-0) 

`claimRewardsAndUpdateTierPositions` iterates all positions in the tier and calls `processEventsAndClaimBonus` for each, propagating every error unconditionally:

```go
// x/tieredrewards/keeper/claim_rewards.go:71-73
if _, err := k.processEventsAndClaimBonus(ctx, &pos); err != nil {
    return err   // ErrInsufficientBonusPool propagates here
}
``` [2](#0-1) 

`processEventsAndClaimBonus` returns `ErrInsufficientBonusPool` whenever the computed bonus exceeds the pool balance:

```go
// x/tieredrewards/keeper/claim_rewards.go:230-232
if err := k.sufficientBonusPoolBalance(ctx, bonusCoins); err != nil {
    return nil, err
}
``` [3](#0-2) 

The codebase already recognises this exact problem in `slashRedelegationPosition`, which explicitly swallows `ErrInsufficientBonusPool` with the comment *"Deliberately forgo bonus rewards if pool is insufficient to prevent chain halt"*:

```go
// x/tieredrewards/keeper/slash.go:54-64
if _, err := k.processEventsAndClaimBonus(ctx, &pos); err != nil {
    if errors.Is(err, types.ErrInsufficientBonusPool) {
        k.logger(ctx).Error("insufficient bonus pool during redelegation slash", ...)
    } else {
        return err
    }
}
``` [4](#0-3) 

`claimRewardsAndUpdateTierPositions` has no equivalent guard. The asymmetry is the root cause.

---

### Impact Explanation

Governance cannot execute `MsgUpdateTier` to change `BonusApy` for any tier that has at least one delegated position with accrued bonus, whenever the rewards pool balance is below the aggregate accrued bonus. Concretely:

- Governance cannot reduce `BonusApy` to zero to halt further bonus accrual from a depleted pool.
- Governance cannot raise `BonusApy` to attract delegators.
- The tier's parameters are frozen until the pool is externally replenished — a dependency that is not enforced or documented anywhere in the protocol.

The corrupted invariant is the **module-permission assumption**: governance must always be able to update tier parameters. That assumption is violated whenever `claimRewardsAndUpdateTierPositions` fails due to pool insufficiency.

---

### Likelihood Explanation

The bonus pool is a finite external fund. It depletes naturally as positions claim rewards. Any actor who observes a pending governance proposal to update a tier can front-run it by claiming rewards from their own positions (a fully legitimate action) until the pool balance falls below the aggregate accrued bonus for the tier. The governance transaction then fails. The actor can repeat this on every subsequent attempt until the pool is replenished. No privileged access is required; a standard `MsgClaimTierRewards` transaction is sufficient.

---

### Recommendation

Mirror the pattern already used in `slashRedelegationPosition`: in `claimRewardsAndUpdateTierPositions`, catch `ErrInsufficientBonusPool` and skip the bonus settlement for that position (advancing only the checkpoint timestamps), rather than aborting the entire sweep. This ensures governance can always update tier parameters regardless of pool balance, consistent with the design intent documented in the ADR. [4](#0-3) [5](#0-4) 

---

### Proof of Concept

1. Governance adds a tier with `BonusApy = 0.10` (10 %).
2. Several users call `MsgLockTier` and let time pass so bonus accrues.
3. The rewards pool is funded with a small amount — enough for individual claims but not for the full tier sweep.
4. An observer (or the users themselves) calls `MsgClaimTierRewards` repeatedly until the pool balance falls below the aggregate accrued bonus for the remaining positions.
5. Governance submits `MsgUpdateTier` to change `BonusApy` to `0.05`.
6. The proposal passes on-chain, but execution fails:
   - `UpdateTier` → `claimRewardsAndUpdateTierPositions` → `processEventsAndClaimBonus` → `sufficientBonusPoolBalance` → `ErrInsufficientBonusPool` → propagated → `UpdateTier` returns error.
7. The tier's `BonusApy` remains at `0.10`. Governance cannot reduce it until the pool is replenished.
8. Step 4 can be repeated after each replenishment, permanently blocking the governance operation at negligible cost. [6](#0-5) [7](#0-6) [8](#0-7)

func (ms msgServer) UpdateTier(ctx context.Context, msg *types.MsgUpdateTier) (*types.MsgUpdateTierResponse, error) {
	if err := ms.requireAuthority(msg.Authority); err != nil {
		return nil, err
	}

	oldTier, err := ms.getTier(ctx, msg.Tier.Id)
	if err != nil {
		return nil, err
	}

	if !oldTier.BonusApy.Equal(msg.Tier.BonusApy) {
		if err := ms.claimRewardsAndUpdateTierPositions(ctx, msg.Tier.Id); err != nil {
			return nil, err
		}
	}

	if err := ms.SetTier(ctx, msg.Tier); err != nil {
		return nil, err
	}

	if err := ms.emitTierChangedEvent(ctx, types.TierChangeAction_TIER_CHANGE_ACTION_UPDATE, msg.Tier); err != nil {
		return nil, err
	}

	return &types.MsgUpdateTierResponse{}, nil
}

func (k Keeper) claimRewardsAndUpdateTierPositions(ctx context.Context, tierId uint32) error {
	ids, err := k.getPositionsIdsByTier(ctx, tierId)
	if err != nil {
		return err
	}
	if len(ids) == 0 {
		return nil
	}

	for _, id := range ids {
		pos, err := k.getPositionState(ctx, id)
		if err != nil {
			return err
		}
		if !pos.IsDelegated() {
			continue
		}

		if _, err := k.claimBaseRewards(ctx, pos); err != nil {
			return err
		}
		if _, err := k.processEventsAndClaimBonus(ctx, &pos); err != nil {
			return err
		}
		if err := k.setPosition(ctx, pos.Position, nil); err != nil {
			return err
		}
	}

	return nil
}
