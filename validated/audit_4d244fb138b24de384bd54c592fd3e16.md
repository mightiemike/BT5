### Title
Unvalidated `_cached_sha256_treehash` Inheritance in `Program.wrap()` Enables Puzzle Hash Spoofing — (File: `wheel/python/clvm_rs/program.py`)

### Summary
`Program.wrap()` blindly copies `_cached_sha256_treehash` from any caller-supplied `CLVMStorage` object without verifying it matches the actual tree hash. Once copied, the cached value is returned by `tree_hash()` without recomputation. Any Python code that passes a crafted `CLVMStorage` object with a wrong `_cached_sha256_treehash` to `Program.wrap()` causes `tree_hash()` to return an arbitrary hash, corrupting puzzle hash verification for the lifetime of that `Program` object.

### Finding Description
`Program.wrap()` (lines 132–143 of `wheel/python/clvm_rs/program.py`) copies `_cached_sha256_treehash` from the wrapped `CLVMStorage` object with no validation:

```python
o._cached_sha256_treehash = getattr(v, "_cached_sha256_treehash", None)
```

`Program.tree_hash()` (lines 281–286) then uses this cached value directly if it is not `None`, never recomputing it:

```python
def tree_hash(self) -> bytes:
    if self._cached_sha256_treehash is None:
        self._cached_sha256_treehash = sha256_treehash(self._unwrapped)
    return self._cached_sha256_treehash
```

`Treehasher.sha256_treehash()` (lines 57–60 of `wheel/python/clvm_rs/tree_hash.py`) applies the same unconditional trust to every sub-node it visits during traversal:

```python
r = getattr(obj, "_cached_sha256_treehash", None)
if r is not None:
    self.cache_hits += 1
    hash_stack.append(r)
    return
```

The `CLVMStorage` protocol (`wheel/python/clvm_rs/clvm_storage.py`) is intentionally open: any Python object with `atom` and `pair` attributes satisfies it. There is no mechanism that restricts who may supply a pre-populated `_cached_sha256_treehash`, and there is no guard that checks the supplied value against the actual tree content. This is the direct analog of the unprotected `setGauge` one-time setter: the cache slot is written once and then trusted unconditionally, with no access control on who writes it.

The broken invariant is: **`program._cached_sha256_treehash`, once set, must equal `sha256_treehash(program._unwrapped)`**. `Program.wrap()` breaks this invariant by accepting the field from an external object without verification.

### Impact Explanation
`Program.__eq__()` (lines 182–188) delegates entirely to `tree_hash()`:

```python
return self.tree_hash() == other_obj.tree_hash()
```

A spoofed hash therefore corrupts every equality comparison on the affected `Program` object. In the Chia ecosystem, puzzle hashes derived from `tree_hash()` are the canonical identifiers for coins and spending conditions. A node that computes a wrong puzzle hash for a spend will either accept an invalid spend or reject a valid one, producing a consensus split with nodes that computed the correct hash. The impact is equivalent to the external report's "temporary system-breaking" severity: the affected `Program` object is permanently poisoned for the duration of its lifetime, and any downstream coin-validation logic that relies on it produces wrong results.

### Likelihood Explanation
`Program.wrap()` is a public classmethod exposed as the primary entry point for converting any `CLVMStorage`-conforming object into a `Program`. The `CLVMStorage` protocol is structurally typed and requires no registration or approval. Any Python caller — including wallet code, puzzle generators, or third-party libraries — that passes a custom `CLVMStorage` object with a pre-set `_cached_sha256_treehash` triggers the bug. The `Treehasher.sha256_treehash()` path additionally fires for every sub-node encountered during traversal, so a single poisoned leaf in a large tree corrupts the hash of every ancestor. In practice, Chia wallet and puzzle-construction code frequently builds `CLVMStorage` objects programmatically before wrapping them, making this a reachable path without requiring any special privilege.

### Recommendation
Remove the unconditional cache inheritance from `Program.wrap()`. Do not copy `_cached_sha256_treehash` from an external `CLVMStorage` object unless the value has been verified against the actual tree content, or drop the copy entirely and let `tree_hash()` compute the hash lazily on first access:

```python
# Remove or guard these two lines in Program.wrap():
# o._cached_serialization = getattr(v, "_cached_serialization", None)
# o._cached_sha256_treehash = getattr(v, "_cached_sha256_treehash", None)
```

Similarly, `Treehasher.sha256_treehash()` should only trust `_cached_sha256_treehash` values that were written by the hasher itself (e.g., by tagging them with a sentinel or by only trusting values on objects whose type is known to be safe).

### Proof of Concept

```python
from hashlib import sha256
from clvm_rs.program import Program

# Build a legitimate atom program containing b"hello"
legit = Program.new_atom(b"hello")
correct_hash = legit.tree_hash()   # sha256(b'\x01' + b'hello')

# Craft a CLVMStorage with the same atom but a poisoned cached hash
class PoisonedStorage:
    atom = b"hello"
    pair = None
    _cached_sha256_treehash = b'\x00' * 32   # attacker-chosen value

poisoned = Program.wrap(PoisonedStorage())

# tree_hash() returns the attacker's value, not the correct hash
assert poisoned.tree_hash() == b'\x00' * 32          # wrong hash returned
assert poisoned.tree_hash() != correct_hash           # correct hash suppressed

# Equality comparison is now broken
assert poisoned != legit   # same atom, but reports not-equal due to hash mismatch
```

The `_cached_sha256_treehash` field is written once in `Program.wrap()` (line 142) and read unconditionally in `Program.tree_hash()` (line 284), with no intervening validation — the exact "unprotected one-time setter" pattern identified in the reference report.
