### Title
Crafted IBC Packet ClassID Bypasses `IsAwayFromOrigin` Direction Check, Enabling Unauthorized Unescrow of Native NFTs — (`x/nft-transfer/keeper/packet.go`)

---

### Summary

A malicious IBC counterparty chain can craft a packet whose `data.ClassId` is prefixed with its own `sourcePort/sourceChannel/`, causing `IsAwayFromOrigin` to return `false` on the receiving chain. This triggers the "return-to-origin" branch in `processReceivedPacket`, which calls `RemoveClassPrefix` and then `TransferOwner(escrowAddress, receiver)` on a native NFT denom that was legitimately escrowed by a prior `SendTransfer`. The result is unauthorized transfer of an escrowed native NFT to an attacker-controlled receiver, with no corresponding `SendTransfer` from the attacker.

---

### Finding Description

**`IsAwayFromOrigin` logic:** [1](#0-0) 

```go
func IsAwayFromOrigin(sourcePort, sourceChannel, fullClassPath string) bool {
    prefixClassID := GetClassPrefix(sourcePort, sourceChannel)
    return !strings.HasPrefix(fullClassPath, prefixClassID)
}
```

The function returns `false` (i.e., "moving back toward origin") whenever `fullClassPath` starts with `sourcePort/sourceChannel/`. The `fullClassPath` here is `data.ClassId` from the packet — **attacker-controlled data**.

**`processReceivedPacket` call site:** [2](#0-1) 

```go
isAwayFromOrigin := types.IsAwayFromOrigin(packet.GetSourcePort(), packet.GetSourceChannel(), data.ClassId)
escrowAddress := types.GetEscrowAddress(packet.GetDestPort(), packet.GetDestChannel())

// ...
} else {
    unprefixedClassID := types.RemoveClassPrefix(packet.GetSourcePort(), packet.GetSourceChannel(), data.ClassId)
    voucherClassID := types.ParseClassTrace(unprefixedClassID).IBCClassID()
    for _, tokenID := range data.TokenIds {
        if err := k.nftKeeper.TransferOwner(ctx, voucherClassID, tokenID, escrowAddress, receiver); err != nil {
            return err
        }
    }
}
```

**Attack construction:**

Suppose the IBC channel is:
- Attacker's chain (Chain A): `sourcePort = "nft"`, `sourceChannel = "channel-0"`
- Victim chain (Chain B): `destPort = "nft"`, `destChannel = "channel-1"`

A legitimate user on Chain B previously called `SendTransfer` for native NFT `nativeClass/token1`, escrowing it to `escrowAddress = GetEscrowAddress("nft", "channel-1")`.

The attacker crafts a packet with:
```
data.ClassId = "nft/channel-0/nativeClass"
data.TokenIds = ["token1"]
data.Receiver = <attacker-controlled address>
```

On Chain B's `processReceivedPacket`:
1. `IsAwayFromOrigin("nft", "channel-0", "nft/channel-0/nativeClass")` → `false` ✓ (attacker wins the direction check)
2. `RemoveClassPrefix("nft", "channel-0", "nft/channel-0/nativeClass")` → `"nativeClass"`
3. `ParseClassTrace("nativeClass").IBCClassID()` → `"nativeClass"` (empty path, returns base class ID)
4. `TransferOwner(ctx, "nativeClass", "token1", escrowAddress, attackerReceiver)` → **succeeds** because the escrow address legitimately holds `token1`

**`ValidateBasic` provides no protection:** [3](#0-2) 

`ValidateBasic` only checks that `classId` is non-blank, `tokenIds` is non-empty, lengths match, and addresses are valid bech32. There is no check that `data.ClassId` does not start with the source port/channel prefix.

**`TransferOwner` owner check is satisfied by the precondition:** [4](#0-3) 

`TransferOwner` calls `IsOwner(ctx, denomID, tokenID, srcOwner)` — it verifies the NFT is owned by `escrowAddress`. This check is satisfied because the legitimate user's prior `SendTransfer` escrowed the NFT there.

---

### Impact Explanation

An attacker controlling a counterparty IBC chain can steal any native NFT currently held in escrow on the victim chain for the channel connecting to the attacker. The NFT is transferred to an attacker-controlled receiver without any prior `SendTransfer` from the attacker. This is a direct, unauthorized ownership change of a native NFT — a Critical-severity invariant break.

---

### Likelihood Explanation

The precondition (escrow address holds a native NFT) is a normal operational state: it occurs whenever any user on the victim chain initiates an IBC NFT transfer through the channel and the packet is in-flight. An attacker who controls a counterparty chain (or can submit packets on behalf of one) can exploit this at any time such a state exists. The attack requires no governance, no privileged keys, and no social engineering — only the ability to submit a crafted IBC packet, which is the standard capability of any IBC counterparty.

---

### Recommendation

The direction check must be made unforgeable. The standard ICS-20 fungible token transfer fix is to validate that `data.ClassId` does NOT start with the source port/channel prefix before accepting it as a "return to origin" packet. Alternatively, the receiving chain should only enter the unescrow branch if it can verify a corresponding escrow commitment exists (e.g., by checking a stored escrow record keyed by classID + tokenID + channel). The `data.ClassId` in the packet should be treated as untrusted input and never used alone to determine transfer direction.

```go
func (nftpd NonFungibleTokenPacketData) ValidateBasic() error {
	if strings.TrimSpace(nftpd.ClassId) == "" {
		return newsdkerrors.Wrap(ErrInvalidClassID, "classId cannot be blank")
	}

	if len(nftpd.TokenIds) == 0 {
		return newsdkerrors.Wrap(ErrInvalidTokenID, "tokenId cannot be blank")
	}

	if len(nftpd.TokenIds) != len(nftpd.TokenUris) {
		return newsdkerrors.Wrap(ErrInvalidPacket, "tokenIds and tokenUris lengths do not match")
	}

	if strings.TrimSpace(nftpd.Sender) == "" {
		return newsdkerrors.Wrap(sdkerrors.ErrInvalidAddress, "sender address cannot be blank")
	}

	// decode the sender address
	if _, err := sdk.AccAddressFromBech32(nftpd.Sender); err != nil {
		return newsdkerrors.Wrap(sdkerrors.ErrInvalidAddress, "invalid sender address")
	}

	if strings.TrimSpace(nftpd.Receiver) == "" {
		return newsdkerrors.Wrap(sdkerrors.ErrInvalidAddress, "receiver address cannot be blank")
	}

	// decode the receiver address
	if _, err := sdk.AccAddressFromBech32(nftpd.Receiver); err != nil {
		return newsdkerrors.Wrap(sdkerrors.ErrInvalidAddress, "invalid receiver address")
	}
	return nil
}
```

```go
func (k Keeper) TransferOwner(
	ctx sdk.Context, denomID, tokenID string, srcOwner, dstOwner sdk.AccAddress,
) error {
	if !k.HasDenomID(ctx, denomID) {
		return sdkerrors.Wrapf(types.ErrInvalidDenom, "denom ID %s not exists", denomID)
	}

	nft, err := k.IsOwner(ctx, denomID, tokenID, srcOwner)
	if err != nil {
		return err
	}

	nft.Owner = dstOwner.String()

	k.setNFT(ctx, denomID, nft)
	k.swapOwner(ctx, denomID, tokenID, srcOwner, dstOwner)
	return nil
}
```
