### Title
u64 Overflow in `op_unknown` Cost Multiplier Produces Undercharged Execution Cost — (`File: src/more_ops.rs`)

---

### Summary

In `op_unknown`, the base cost computed for cost-functions 1, 2, or 3 is multiplied by `cost_multiplier + 1` (up to `u32::MAX + 1 = 2^32`) without overflow protection. Because Rust wraps on integer overflow in release mode, the product can silently wrap to a value far below `u32::MAX`, bypassing the post-multiplication guard and returning a near-zero cost for what should be a budget-exhausting operation. This is a direct arithmetic-semantic-mismatch analog to M-23: a raw intermediate value (base cost) is used where a properly bounded, overflow-safe converted value (final cost) should be used, making the effective cost threshold far lower than intended.

---

### Finding Description

`op_unknown` in `src/more_ops.rs` handles unknown opcodes in lenient mode. The last byte of the opcode encodes a `cost_function` (bits 6–7) and the preceding bytes encode a `cost_multiplier` (up to `u32::MAX`). [1](#0-0) 

For cost-functions 1, 2, and 3, the base cost is accumulated inside a loop with `check_cost` guards that ensure `cost ≤ max_cost` before the loop exits: [2](#0-1) 

After the loop, a redundant `check_cost` is called, then the multiplication is performed: [3](#0-2) 

The critical line is:

```rust
cost *= cost_multiplier + 1;   // line 261 — no overflow check
```

`cost` is `u64`, `cost_multiplier` is `u64` (cast from `u32`), so `cost_multiplier + 1` is at most `2^32`. Chia's block `max_cost` is 11 × 10⁹ ≈ 2^33.4. Therefore `cost` (bounded by `max_cost`) can exceed `u64::MAX / 2^32 = 2^32 ≈ 4.3 × 10⁹`, making the product exceed `u64::MAX`. In Rust release builds, this wraps silently to a small value. The subsequent guard `if cost > u32::MAX` then passes (wrapped value is small), and the function returns `Ok(Reduction(small_cost, nil))`.

**Concrete trigger for cost-function 2 (mul-like):**

The quadratic term `(l0 * l1) / MUL_SQUARE_COST_PER_BYTE_DIVIDER` with `l0 = l1 ≈ 742 KB` yields:

```
cost ≈ (742000)² / 128 ≈ 4.3 × 10⁹ > u64::MAX / 2^32
```

Two atoms of ~742 KB can be quoted in CLVM for just `2 × QUOTE_COST = 40` cost units. The unknown opcode bytes encode `cost_multiplier = 0xffffffff` and cost-function 2. The base cost passes `check_cost(cost, max_cost)` (since 4.3 × 10⁹ < 11 × 10⁹), then the multiplication wraps, and the returned cost is near zero. [4](#0-3) [5](#0-4) 

---

### Impact Explanation

An attacker-controlled CLVM program can include an unknown opcode that, by design, should consume nearly the entire block cost budget (up to `max_cost × (u32::MAX + 1)` intended cost), but instead returns a cost of near zero due to wrapping. The program then has its full remaining budget available for further operations. This allows a program to perform far more computation than the `max_cost` limit is intended to permit, breaking the consensus-level resource accounting invariant. All nodes executing the program in lenient/consensus mode are affected equally, so this does not cause a consensus split — it causes all nodes to accept programs that exceed the intended computational budget, enabling a resource-exhaustion attack on block validators.

---

### Likelihood Explanation

The trigger requires: (1) an unknown opcode with cost-function 2 or 1, (2) a `cost_multiplier` of `0xffffffff` (4 opcode bytes), and (3) two atom arguments of ~742 KB or many smaller arguments summing to equivalent cost. All of these are fully attacker-controlled via the CLVM bytecode. The quoting cost for two large atoms is negligible (40 cost units). Unknown opcodes are accepted in lenient/consensus mode on Chia mainnet. The overflow is deterministic and reproducible across all nodes running release builds.

---

### Recommendation

Replace the unchecked multiplication with an overflow-safe check before applying the multiplier:

```rust
// Before:
cost *= cost_multiplier + 1;
if cost > u32::MAX as u64 {
    Err(EvalErr::Invalid(o))?
}

// After:
let multiplied = cost.checked_mul(cost_multiplier + 1)
    .ok_or(EvalErr::Invalid(o))?;
if multiplied > u32::MAX as u64 {
    Err(EvalErr::Invalid(o))?
}
cost = multiplied;
```

This ensures that any multiplication that would overflow u64 is treated as an invalid opcode (hard failure), consistent with the existing `u32::MAX` cap intent. [3](#0-2)

### Proof of Concept

**Opcode bytes** (5 bytes total):
- Bytes 0–3: `0xff 0xff 0xff 0xff` → `cost_multiplier = 0xffffffff`
- Byte 4: `0x40` → bits 6–7 = `01` → cost-function 2 (mul-like)

**CLVM program** (pseudocode):
```
(0xffffff ff40
  (q . <atom-A: 742000 bytes of 0x00>)
  (q . <atom-B: 742000 bytes of 0x00>))
```

**Execution trace:**
1. Quote atom-A: cost += 20
2. Quote atom-B: cost += 20
3. Call unknown op: base cost ≈ `(742000 × 742000) / 128 ≈ 4.30 × 10⁹`
4. `check_cost(4.30e9, 11e9)` → passes
5. `cost *= 0xffffffff + 1` → `4.30e9 × 2^32` overflows u64, wraps to small value W
6. `W > u32::MAX` → false → returns `Reduction(W, nil)`
7. Total program cost ≈ 40 + W ≈ 40, well within `max_cost = 11e9`

The program consumes ~40 cost units instead of the ~18.5 × 10¹⁸ that the multiplier was intended to charge, leaving the full budget available for additional operations. [6](#0-5) [7](#0-6)

```rust
pub fn op_unknown(
    allocator: &mut Allocator,
    o: NodePtr,
    mut args: NodePtr,
    max_cost: Cost,
) -> Response {
    // unknown opcode in lenient mode
    // unknown ops are reserved if they start with 0xffff
    // otherwise, unknown ops are no-ops, but they have costs. The cost is computed
    // like this:

    // byte index (reverse):
    // | 4 | 3 | 2 | 1 | 0          |
    // +---+---+---+---+------------+
    // | multiplier    |XX | XXXXXX |
    // +---+---+---+---+---+--------+
    //  ^               ^    ^
    //  |               |    + 6 bits ignored when computing cost
    // cost_multiplier  |
    // (up to 4 bytes)  + 2 bits
    //                    cost_function

    // 1 is always added to the multiplier before using it to multiply the cost, this
    // is since cost may not be 0.

    // cost_function is 2 bits and defines how cost is computed based on arguments:
    // 0: constant, cost is 1 * (multiplier + 1)
    // 1: computed like operator add, multiplied by (multiplier + 1)
    // 2: computed like operator mul, multiplied by (multiplier + 1)
    // 3: computed like operator concat, multiplied by (multiplier + 1)

    // this means that unknown ops where cost_function is 1, 2, or 3, may still be
    // fatal errors if the arguments passed are not atoms.

    let op_atom = allocator.atom(o);
    let op = op_atom.as_ref();

    if op.is_empty() || (op.len() >= 2 && op[0] == 0xff && op[1] == 0xff) {
        Err(EvalErr::Reserved(o))?;
    }

    let cost_function = (op[op.len() - 1] & 0b11000000) >> 6;
    let cost_multiplier: u64 = match u32_from_u8(&op[0..op.len() - 1]) {
        Some(v) => v as u64,
        None => {
            return Err(EvalErr::Invalid(o))?;
        }
    };

    let mut cost = match cost_function {
        0 => 1,
        1 => {
            let mut cost = ARITH_BASE_COST;
            let mut byte_count: u64 = 0;
            while let Some((arg, rest)) = allocator.next(args) {
                args = rest;
                cost += ARITH_COST_PER_ARG;
                let len = atom_len(allocator, arg, "unknown op")?;
                byte_count += len as u64;
                check_cost(cost + (byte_count as Cost * ARITH_COST_PER_BYTE), max_cost)?;
            }
            cost + (byte_count * ARITH_COST_PER_BYTE)
        }
        2 => {
            let mut cost = MUL_BASE_COST;
            let mut first_iter: bool = true;
            let mut l0: u64 = 0;
            while let Some((arg, rest)) = allocator.next(args) {
                args = rest;
                let len = atom_len(allocator, arg, "unknown op")?;
                if first_iter {
                    l0 = len as u64;
                    first_iter = false;
                    continue;
                }
                let l1 = len as u64;
                cost += MUL_COST_PER_OP;
                cost += (l0 + l1) * MUL_LINEAR_COST_PER_BYTE;
                cost += (l0 * l1) / MUL_SQUARE_COST_PER_BYTE_DIVIDER;
                l0 += l1;
                check_cost(cost, max_cost)?;
            }
            cost
        }
        3 => {
            let mut cost = CONCAT_BASE_COST;
            while let Some((arg, rest)) = allocator.next(args) {
                args = rest;
                let len = atom_len(allocator, arg, "unknown op")?;
                cost += CONCAT_COST_PER_ARG;
                cost += CONCAT_COST_PER_BYTE * (len as Cost);
                check_cost(cost, max_cost)?;
            }
            cost
        }
        _ => 1,
    };

    assert!(cost > 0);

    check_cost(cost, max_cost)?;
    cost *= cost_multiplier + 1;
    if cost > u32::MAX as u64 {
        Err(EvalErr::Invalid(o))?
    } else {
        Ok(Reduction(cost as Cost, allocator.nil()))
    }
```

```rust
use crate::error::{EvalErr, Result};

pub type Cost = u64;

pub fn check_cost(cost: Cost, max_cost: Cost) -> Result<()> {
    if cost > max_cost {
        Err(EvalErr::CostExceeded)
    } else {
        Ok(())
    }
}
```
