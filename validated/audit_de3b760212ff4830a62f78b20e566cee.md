Audit Report

## Title
`SwapAllowlistExtension.beforeSwap` Checks Router Address Instead of End User, Breaking Allowlisted Swaps and Enabling Full Allowlist Bypass — (`metric-periphery/contracts/extensions/SwapAllowlistExtension.sol`)

## Summary
`SwapAllowlistExtension.beforeSwap` gates swaps by checking `allowedSwapper[pool][sender]`, where `sender` is bound to `msg.sender` of the `pool.swap()` call. When swaps are routed through `MetricOmmSimpleRouter`, the router contract is `msg.sender` of `pool.swap()`, so the extension checks whether the **router** is allowlisted rather than the **end user**. This produces two mutually exclusive failure modes: allowlisted users are locked out of the standard periphery path entirely, and if the admin allowlists the router to restore access, every address on-chain can bypass the per-user allowlist.

## Finding Description

**Root cause — wrong actor checked in `beforeSwap`:**

`SwapAllowlistExtension.beforeSwap` receives `sender` as its first argument and checks:

```solidity
if (!allowAllSwappers[msg.sender] && !allowedSwapper[msg.sender][sender]) {
    revert IMetricOmmPoolActions.NotAllowedToSwap();
}
```

Here `msg.sender` is the pool (the pool calls the extension) and `sender` is the first argument passed to `beforeSwap`.

**Pool binds `msg.sender` of `swap()` as `sender`:**

`MetricOmmPool.swap` calls `_beforeSwap(msg.sender, ...)`, so `sender` in the extension is whoever called `pool.swap()`.

**Router calls `pool.swap()` directly — its own address becomes `sender`:**

In `MetricOmmSimpleRouter.exactInputSingle`, `exactInput`, `exactOutputSingle`, and `exactOutput`, the router calls `IMetricOmmPoolActions(params.pool).swap(...)` directly. The router is therefore `msg.sender` of `pool.swap()`, making `sender = router` in the extension. The check becomes `allowedSwapper[pool][router]`, not `allowedSwapper[pool][user]`.

**`ExtensionCalling._beforeSwap` passes `sender` verbatim — no user address is threaded through:**

There is no mechanism in the extension call path to recover the original EOA address once the router has become `msg.sender`.

**Exploit flow:**

- **Mode 1 (no admin mistake required):** Alice is allowlisted (`allowedSwapper[pool][alice] = true`). Alice calls `router.exactInputSingle(...)`. The router calls `pool.swap(...)` — `msg.sender` of that call is the router. The pool calls `_beforeSwap(sender=router, ...)`. The extension checks `allowedSwapper[pool][router]` → `false` → reverts with `NotAllowedToSwap`. Alice cannot use the router at all.

- **Mode 2 (one natural admin remediation):** Admin calls `setAllowedToSwap(pool, router, true)` to restore router access. Now `allowedSwapper[pool][router] = true`. Bob (not allowlisted) calls `router.exactInputSingle(...)`. The router calls `pool.swap(...)` — `msg.sender` is the router. Extension checks `allowedSwapper[pool][router]` → `true` → passes. Bob swaps successfully on the curated pool, defeating the entire purpose of the extension.

## Impact Explanation
Mode 1 breaks core pool functionality: any pool deploying `SwapAllowlistExtension` with the public router has every router-mediated swap revert, making the standard periphery path completely unusable for allowlisted users. Mode 2 enables a full allowlist bypass: once the admin allowlists the router (the natural remediation), any unprivileged address can swap on a pool intended to be restricted to specific users. This directly satisfies "Broken core pool functionality causing loss of funds or unusable swap flows" and "Admin-boundary break: pool admin exceeds caps or role checks are bypassed by an unprivileged path."

## Likelihood Explanation
Mode 1 is triggered immediately for every pool that combines `SwapAllowlistExtension` with `MetricOmmSimpleRouter` — no attacker action required. Mode 2 requires the admin to allowlist the router, which is the only available remediation when users report that the router is broken. The admin has no way to grant router access only to allowlisted users; the only knob is the router address itself, which is all-or-nothing. Both modes are reachable by any unprivileged trader using the public router.

## Recommendation
The extension must gate the economically relevant actor — the end user — not the intermediary router. The cleanest fix is to have the router ABI-encode `msg.sender` into the `extensionData` it forwards, and have `SwapAllowlistExtension.beforeSwap` decode and check that address when `sender` is a known router address. Alternatively, the pool's `_beforeSwap` interface could be extended to carry the originating user address as a separate argument, but this requires core changes. A simpler interim approach is to check `recipient` instead of `sender` for single-hop flows, verifying that the router never sets `recipient` to itself for the final hop (confirmed: `exactInputSingle` and `exactOutputSingle` pass `params.recipient` directly).

## Proof of Concept

```
1. Deploy pool with SwapAllowlistExtension configured.
2. Admin calls setAllowedToSwap(pool, alice, true).
   → alice is allowlisted; bob is not.

3. alice calls router.exactInputSingle({pool: pool, ...}).
   → router calls pool.swap(...) — msg.sender of pool.swap = router.
   → pool calls _beforeSwap(sender=router, ...).
   → extension checks allowedSwapper[pool][router] → false.
   → REVERTS with NotAllowedToSwap.
   alice (allowlisted) cannot use the router. [Mode 1]

4. Admin, trying to fix this, calls setAllowedToSwap(pool, router, true).
   → router is now allowlisted.

5. bob (not allowlisted) calls router.exactInputSingle({pool: pool, ...}).
   → router calls pool.swap(...) — msg.sender = router.
   → extension checks allowedSwapper[pool][router] → true.
   → PASSES. bob swaps successfully on the curated pool. [Mode 2]
```

Foundry test plan: deploy pool with `SwapAllowlistExtension` as a `beforeSwap` extension; call `setAllowedToSwap(pool, alice, true)`; `vm.prank(alice)` → `router.exactInputSingle(...)` → assert revert `NotAllowedToSwap`; call `setAllowedToSwap(pool, router, true)`; `vm.prank(bob)` → `router.exactInputSingle(...)` → assert success.   }
