### Title
`SwapAllowlistExtension` gates the router address instead of the originating user, allowing any unprivileged caller to bypass the swap allowlist via `MetricOmmSimpleRouter` — (`metric-periphery/contracts/extensions/SwapAllowlistExtension.sol`)

---

### Summary

`SwapAllowlistExtension.beforeSwap` enforces access control by checking `allowedSwapper[pool][sender]`, where `sender` is the first argument forwarded by the pool — which is `msg.sender` of the pool's `swap` call. When a user routes through `MetricOmmSimpleRouter`, the pool's `msg.sender` is the router contract, not the originating user. This creates an irresolvable dilemma for any pool admin who deploys a curated pool with this extension: either the router is not allowlisted (blocking all router-mediated swaps, including those from legitimately allowlisted users), or the router is allowlisted (allowing every user on-chain to bypass the allowlist by routing through the public router). In the latter case — the only operationally viable configuration — the allowlist guard is fully bypassed for all users.

---

### Finding Description

The `SwapAllowlistExtension` is a `beforeSwap` hook that pool admins attach to restrict which addresses may trade on a curated pool. Its check is:

```
allowAll[pool] || allowedSwapper[pool][sender]
```

where `sender` is the first parameter passed by `ExtensionCalling._beforeSwap`, which is `msg.sender` of the pool's `swap` call. [1](#0-0) 

`MetricOmmSimpleRouter` is the canonical public periphery entry point for swaps. When a user calls any `exact*` function on the router, the router calls `pool.swap(recipient, ...)` directly. At that point, `msg.sender` seen by the pool — and forwarded as `sender` to every `beforeSwap` hook — is the router contract address, not the originating user. [2](#0-1) [3](#0-2) 

The pool's `ExtensionCalling._beforeSwap` dispatcher passes `msg.sender` (the router) as the `sender` argument to every configured extension. The `SwapAllowlistExtension` therefore checks whether the **router** is allowlisted, not whether the **originating user** is allowlisted. [4](#0-3) 

This produces two failure modes:

1. **Router not allowlisted**: All router-mediated swaps revert, even for users who are individually allowlisted. The pool is unusable via the standard periphery.
2. **Router allowlisted** (the only operationally viable choice): Every address on-chain can bypass the allowlist by calling `MetricOmmSimpleRouter.exact*` instead of calling the pool directly. The allowlist guard is completely defeated.

The `DepositAllowlistExtension.beforeAddLiquidity` hook receives `owner` (the LP position owner, passed explicitly by the caller), so it is not identically broken — but the swap path has no equivalent explicit-user forwarding. [5](#0-4) 

---

### Impact Explanation

A pool admin who deploys a curated pool (e.g., KYC-gated, institution-only, or regulatory-restricted) with `SwapAllowlistExtension` and allowlists the router to support standard user flows inadvertently opens the pool to every address on-chain. Any non-allowlisted user can execute swaps against the pool's LP positions by routing through `MetricOmmSimpleRouter`. This constitutes:

- **Direct loss of LP principal**: LP positions priced for a restricted counterparty set are exposed to the full adversarial public, enabling arbitrage or directional trading that drains LP value.
- **Broken core pool functionality**: The access-control invariant the pool was deployed to enforce is silently violated on every router-mediated swap.
- **Admin-boundary break**: The pool admin's configured allowlist policy is bypassed by an unprivileged path (the public router).

---

### Likelihood Explanation

- `MetricOmmSimpleRouter` is the primary user-facing swap interface; virtually all non-technical users and integrations route through it.
- The pool admin has no on-chain mechanism to enforce the allowlist for router users without blocking the router entirely.
- No special knowledge or capital is required: any address can call `MetricOmmSimpleRouter.exact*` with the target pool.
- The bypass is deterministic and repeatable in every block.

---

### Recommendation

The `SwapAllowlistExtension` must check the economically relevant actor, not the immediate `msg.sender` of the pool. Two sound approaches:

1. **Check `recipient` instead of (or in addition to) `sender`**: The router passes the originating user as `recipient`. Gating on `recipient` correctly identifies who receives the output tokens and is the economically relevant party.

2. **Require the router to forward the originating user in `extensionData`**: The extension decodes the first 20 bytes of `extensionData` as the real user and checks that address. The router must be updated to prepend `msg.sender` to any `extensionData` it forwards.

Option 1 is simpler and does not require router changes. Option 2 is more flexible but requires coordinated router and extension updates.

Additionally, the existing test suite only exercises direct pool calls; router-mediated allowlist tests should be added to cover this path. [6](#0-5)

### Proof of Concept

```
Setup:
  1. Deploy MetricOmmPool with SwapAllowlistExtension as a beforeSwap hook.
  2. Pool admin calls swapExtension.setAllowedToSwap(pool, alice, true)
     — only alice is allowlisted.
  3. Pool admin calls swapExtension.setAllowedToSwap(pool, router, true)
     — router must be allowlisted for alice to use it.
  4. LP adds liquidity to the pool.

Attack (bob is NOT allowlisted):
  5. bob calls MetricOmmSimpleRouter.exactInput(pool, ..., recipient=bob, extensionData="")
  6. Router calls pool.swap(bob, zeroForOne, amount, priceLimit, "")
  7. Pool calls ExtensionCalling._beforeSwap(sender=router, recipient=bob, ...)
  8. SwapAllowlistExtension checks allowedSwapper[pool][router] → true (step 3)
  9. Swap executes successfully.

Result:
  bob, a non-allowlisted address, completes a swap on a curated pool.
  The allowlist guard is fully bypassed.

Direct call (control):
  10. bob calls pool.swap(bob, ...) directly.
  11. Pool calls extension with sender=bob.
  12. allowedSwapper[pool][bob] → false → revert NotAllowedToSwap.
``` [1](#0-0) [2](#0-1) [3](#0-2)
