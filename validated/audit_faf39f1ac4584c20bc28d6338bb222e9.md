### Title
IBC NFT Packet Processing Silently Mints and Burns NFTs Without Emitting NFT-Level Events - (File: x/nft-transfer/keeper/packet.go)

### Summary
The `x/nft-transfer` module's packet processing functions (`processReceivedPacket`, `createOutgoingPacket`, `refundPacketToken`) directly call the `x/nft` keeper's `MintNFT`, `BurnNFTUnverified`, and `TransferOwner` functions without emitting the `mint_nft`, `burn_nft`, or `transfer_nft` events that the `x/nft` module's `msg_server` always emits for user-triggered operations. Off-chain indexers and clients that reconstruct NFT ownership history by parsing these events will produce incorrect ownership state for any NFT that has been transferred, minted, or burned via the IBC NFT-transfer lifecycle.

### Finding Description

The `x/nft` module's `msg_server.go` establishes a clear contract: every NFT ownership change emits a corresponding event. `MintNFT` emits `mint_nft` with `token_id`, `denom_id`, `token_uri`, and `recipient`. `BurnNFT` emits `burn_nft` with `denom_id`, `token_id`, and `owner`. `TransferNFT` emits `transfer_nft` with `token_id`, `denom_id`, `sender`, and `recipient`. [1](#0-0) [2](#0-1) [3](#0-2) 

However, the IBC packet processing path bypasses `msg_server` entirely and calls the keeper directly:

**`processReceivedPacket`** (called on `OnRecvPacket`) calls `k.nftKeeper.MintNFT(...)` for each incoming token without emitting any `mint_nft` event: [4](#0-3) 

**`createOutgoingPacket`** (called on `SendTransfer`) calls `k.nftKeeper.BurnNFTUnverified(...)` for sink-chain sends without emitting any `burn_nft` event, and calls `k.nftKeeper.TransferOwner(...)` for source-chain sends without emitting any `transfer_nft` event: [5](#0-4) 

**`refundPacketToken`** (called on `OnAcknowledgementPacket` error and `OnTimeoutPacket`) calls `k.nftKeeper.TransferOwner(...)` or `k.nftKeeper.MintNFT(...)` without emitting any NFT-level event: [6](#0-5) 

The `ibc_module.go` does emit a `non_fungible_token_packet` event in `OnRecvPacket`, `OnAcknowledgementPacket`, and `OnTimeoutPacket`, but this is an IBC-level packet event containing only `classID` and `tokenIDs` — it is not the `mint_nft`, `burn_nft`, or `transfer_nft` event that the `x/nft` module defines and that indexers rely on: [7](#0-6) [8](#0-7) 

The `x/nft` module's own event definitions confirm the expected event schema: [9](#0-8) 

### Impact Explanation

Any off-chain indexer, wallet, or NFT marketplace that reconstructs NFT ownership history by subscribing to `mint_nft`, `burn_nft`, and `transfer_nft` events will produce incorrect ownership records for all NFTs that have been minted, burned, or transferred via the IBC NFT-transfer lifecycle. Specifically:

- An NFT minted on chain B via IBC from chain A will not appear in any `mint_nft` event log on chain B, so indexers will not register the new owner.
- An NFT burned on a sink chain during an IBC send will not appear in any `burn_nft` event log, so indexers will still show the NFT as owned.
- An NFT refunded after a timeout or failed ack will not appear in any `transfer_nft` or `mint_nft` event log, so indexers will show the NFT as lost.

The on-chain KV store state (`GetNFT`, `GetTotalSupply`, `GetTotalSupplyOfOwner`) is correctly updated in all cases. The corruption is exclusively in the event log, which is the standard mechanism for off-chain state reconstruction.

### Likelihood Explanation

Any user who sends an NFT via IBC (`MsgTransfer` in the `nft-transfer` module) triggers this gap. The IBC NFT-transfer module is a production feature of this chain. Every successful IBC NFT send, receive, timeout, or failed acknowledgement exercises the affected code paths. The likelihood is high for any deployment where IBC NFT transfers are used.

### Recommendation

Emit the appropriate `x/nft` module events inside `processReceivedPacket`, `createOutgoingPacket`, and `refundPacketToken` after each successful keeper call, mirroring the event emission pattern in `x/nft/keeper/msg_server.go`. Specifically:

- After `k.nftKeeper.MintNFT(...)` in `processReceivedPacket` and `refundPacketToken`, emit `EventTypeMintNFT` with `token_id`, `denom_id`, `token_uri`, and `recipient`.
- After `k.nftKeeper.BurnNFTUnverified(...)` in `createOutgoingPacket`, emit `EventTypeBurnNFT` with `denom_id`, `token_id`, and `owner`.
- After `k.nftKeeper.TransferOwner(...)` in `createOutgoingPacket` and `refundPacketToken`, emit `EventTypeTransfer` with `token_id`, `denom_id`, `sender`, and `recipient`.

### Proof of Concept

1. On chain A, user mints NFT `tokenid` under denom `denomid` via `MsgMintNFT`. Event log on chain A contains `mint_nft{token_id: tokenid, recipient: userA}`. Indexer records `userA` owns `tokenid`.

2. User sends NFT via `MsgTransfer` (nft-transfer module) from chain A to chain B. On chain A, `createOutgoingPacket` calls `k.nftKeeper.TransferOwner(ctx, classID, tokenid, sender, escrowAddress)` — no `transfer_nft` event is emitted. Indexer on chain A still shows `userA` as owner (incorrect; escrow address now owns it).

3. On chain B, `OnRecvPacket` → `processReceivedPacket` calls `k.nftKeeper.MintNFT(ctx, voucherClassID, tokenid, ...)` — no `mint_nft` event is emitted. Indexer on chain B never registers `userB` as owner of the IBC voucher NFT.

4. User sends NFT back from chain B to chain A. On chain B, `createOutgoingPacket` calls `k.nftKeeper.BurnNFTUnverified(...)` — no `burn_nft` event is emitted. Indexer on chain B still shows `userB` as owner of a burned NFT.

5. On chain A, `processReceivedPacket` calls `k.nftKeeper.TransferOwner(ctx, voucherClassID, tokenid, escrowAddress, receiver)` — no `transfer_nft` event is emitted. Indexer on chain A never registers the return of ownership to `userA`.

At every step, the on-chain KV state is correct, but the event-based ownership reconstruction is wrong for all IBC-transferred NFTs. [10](#0-9) [11](#0-10) [12](#0-11)

func (k Keeper) refundPacketToken(ctx sdk.Context, packet channeltypes.Packet, data types.NonFungibleTokenPacketData) error {
	sender, err := sdk.AccAddressFromBech32(data.Sender)
	if err != nil {
		return err
	}

	classTrace := types.ParseClassTrace(data.ClassId)
	voucherClassID := classTrace.IBCClassID()

	isAwayFromOrigin := types.IsAwayFromOrigin(packet.GetSourcePort(),
		packet.GetSourceChannel(), data.ClassId)

	escrowAddress := types.GetEscrowAddress(packet.GetSourcePort(), packet.GetSourceChannel())

	if isAwayFromOrigin {
		// unescrow tokens back to the sender
		for _, tokenID := range data.TokenIds {
			if err := k.nftKeeper.TransferOwner(ctx, voucherClassID, tokenID, escrowAddress, sender); err != nil {
				return err
			}
		}
	} else {
		// we are sink chain, mint voucher back to sender
		for i, tokenID := range data.TokenIds {
			if err := k.nftKeeper.MintNFT(ctx, voucherClassID, tokenID, "", data.TokenUris[i], "", escrowAddress, sender); err != nil {
				return err
			}
		}
	}

	return nil
}

func (k Keeper) createOutgoingPacket(ctx sdk.Context,
	sourcePort,
	sourceChannel,
	destinationPort,
	destinationChannel,
	classID string,
	tokenIDs []string,
	sender sdk.AccAddress,
	receiver string,
	sequence uint64,
	timeoutHeight clienttypes.Height,
	timeoutTimestamp uint64,
) (channeltypes.Packet, error) {
	denom, err := k.nftKeeper.GetDenom(ctx, classID)
	if err != nil {
		return channeltypes.Packet{}, err
	}

	var (
		// NOTE: class and hex hash correctness checked during msg.ValidateBasic
		fullClassPath = classID
		tokenURIs     = []string{}
	)

	// deconstruct the token denomination into the denomination trace info
	// to determine if the sender is the source chain
	if strings.HasPrefix(classID, nfttypes.IBCPrefix) {
		fullClassPath, err = k.ClassPathFromHash(ctx, classID)
		if err != nil {
			return channeltypes.Packet{}, err
		}
	}

	isAwayFromOrigin := types.IsAwayFromOrigin(sourcePort,
		sourceChannel, fullClassPath)

	for _, tokenID := range tokenIDs {
		nft, err := k.nftKeeper.GetNFT(ctx, classID, tokenID)
		if err != nil {
			return channeltypes.Packet{}, err
		}
		tokenURIs = append(tokenURIs, nft.GetURI())

		owner := nft.GetOwner()
		if !sender.Equals(owner) {
			return channeltypes.Packet{}, newsdkerrors.Wrap(sdkerrors.ErrUnauthorized, "not token owner")
		}

		if isAwayFromOrigin {
			// create the escrow address for the tokens
			escrowAddress := types.GetEscrowAddress(sourcePort, sourceChannel)
			if err := k.nftKeeper.TransferOwner(ctx, classID, tokenID, sender, escrowAddress); err != nil {
				return channeltypes.Packet{}, err
			}
		} else {
			// we are sink chain, burn the voucher
			if err := k.nftKeeper.BurnNFTUnverified(ctx, classID, tokenID, sender); err != nil {
				return channeltypes.Packet{}, err
			}
		}
	}

	packetData := types.NewNonFungibleTokenPacketData(
		fullClassPath, denom.Uri, tokenIDs, tokenURIs, sender.String(), receiver,
	)

	return channeltypes.NewPacket(
		packetData.GetBytes(),
		sequence,
		sourcePort,
		sourceChannel,
		destinationPort,
		destinationChannel,
		timeoutHeight,
		timeoutTimestamp,
	), nil
}

// processReceivedPacket will mint the tokens to receiver account
// if the token was away from origin chain . Otherwise, the sent tokens
// were burnt in the sending chain and will unescrow the token to receiver
// in the destination chain
func (k Keeper) processReceivedPacket(ctx sdk.Context, packet channeltypes.Packet,
	data types.NonFungibleTokenPacketData,
) error {
	receiver, err := sdk.AccAddressFromBech32(data.Receiver)
	if err != nil {
		return err
	}

	isAwayFromOrigin := types.IsAwayFromOrigin(packet.GetSourcePort(), packet.GetSourceChannel(), data.ClassId)

	// create the escrow address for creating denom and minting nft
	escrowAddress := types.GetEscrowAddress(packet.GetDestPort(), packet.GetDestChannel())

	if isAwayFromOrigin {
		// since SendPacket did not prefix the classID, we must prefix classID here
		classPrefix := types.GetClassPrefix(packet.GetDestPort(), packet.GetDestChannel())
		// NOTE: sourcePrefix contains the trailing "/"
		prefixedClassID := classPrefix + data.ClassId

		// construct the class trace from the full raw classID
		classTrace := types.ParseClassTrace(prefixedClassID)
		if !k.HasClassTrace(ctx, classTrace.Hash()) {
			k.SetClassTrace(ctx, classTrace)
		}

		voucherClassID := classTrace.IBCClassID()

		if !k.nftKeeper.HasDenomID(ctx, voucherClassID) {
			if err := k.nftKeeper.IssueDenom(ctx, voucherClassID, voucherClassID, "", data.ClassUri, escrowAddress); err != nil {
				return err
			}
		}
		sdkCtx := sdk.UnwrapSDKContext(ctx)
		sdkCtx.EventManager().EmitEvent(
			sdk.NewEvent(
				types.EventTypeClassTrace,
				sdk.NewAttribute(types.AttributeKeyTraceHash, classTrace.Hash().String()),
				sdk.NewAttribute(types.AttributeKeyClassID, voucherClassID),
			),
		)

		for i, tokenID := range data.TokenIds {
			if err := k.nftKeeper.MintNFT(ctx, voucherClassID, tokenID, "", data.TokenUris[i], "", escrowAddress, receiver); err != nil {
				return err
			}
		}
	} else {
		// If the token moves in the direction of back to origin,
		// we need to unescrow the token and transfer it to the receiver

		// we should remove the prefix. For example:
		// p6/c6/p4/c4/p2/c2/nftClass -> p4/c4/p2/c2/nftClass
		unprefixedClassID := types.RemoveClassPrefix(packet.GetSourcePort(),
			packet.GetSourceChannel(), data.ClassId)

		voucherClassID := types.ParseClassTrace(unprefixedClassID).IBCClassID()
		for _, tokenID := range data.TokenIds {
			if err := k.nftKeeper.TransferOwner(ctx,
				voucherClassID, tokenID, escrowAddress, receiver); err != nil {
				return err
			}
		}
	}

	return nil
}
