Audit Report

## Title
`SwapAllowlistExtension.beforeSwap` gates the immediate caller (router) instead of the end user, allowing allowlist bypass via `MetricOmmSimpleRouter` - (`metric-periphery/contracts/extensions/SwapAllowlistExtension.sol`)

## Summary
`SwapAllowlistExtension.beforeSwap` checks `allowedSwapper[pool][sender]` where `sender` is `msg.sender` of `pool.swap()`. When swaps are routed through `MetricOmmSimpleRouter`, the pool's `msg.sender` is the router contract, not the end user. A pool admin who allowlists the router address to enable router-based swaps for their users inadvertently opens the gate to all users, as any unpermissioned address can call `exactInputSingle()` and pass the allowlist check via the router's allowlisted identity.

## Finding Description
**Step 1 — Pool passes its own `msg.sender` as `sender` to the extension.**

In `MetricOmmPool.swap()`, `_beforeSwap` is called with `msg.sender` as the first argument: [1](#0-0) 

When a user routes through `MetricOmmSimpleRouter`, `msg.sender` here is the router contract, not the end user.

**Step 2 — The extension checks that caller-supplied `sender` argument.**

`SwapAllowlistExtension.beforeSwap` uses `msg.sender` (the pool) as the mapping key and `sender` (the first argument, i.e., the router) as the identity to gate: [2](#0-1) 

**Step 3 — The router is the immediate caller of `pool.swap()`.**

`MetricOmmSimpleRouter.exactInputSingle()` calls `pool.swap()` directly with no mechanism to forward the original `msg.sender`: [3](#0-2) 

The allowlist lookup becomes `allowedSwapper[pool][router]`, not `allowedSwapper[pool][user]`. There is no configuration that simultaneously (a) allows allowlisted users to use the router and (b) blocks non-allowlisted users from using the router. Allowlisting the router grants access to all users; not allowlisting it blocks all router-mediated swaps including for legitimate allowlisted users.

## Impact Explanation
A pool admin deploying a `SwapAllowlistExtension`-gated pool for regulatory compliance or restricted LP access, who allowlists the router to enable router-based swaps for their users, inadvertently opens the pool to all users. Any unpermissioned address can execute swaps against the restricted pool via `exactInputSingle()`. This constitutes a broken core pool invariant — the allowlist guard is misapplied and fails to gate the economically relevant actor — with direct consequences: unauthorized parties can execute trades the pool was designed to prohibit, constituting broken core pool functionality causing potential loss of funds.

## Likelihood Explanation
The trigger requires the pool admin to allowlist the router, which is a natural and expected operational step for any pool that wants its allowlisted users to access the primary user-facing swap interface (`MetricOmmSimpleRouter`). The attacker needs no special privileges — a single call to `exactInputSingle()` suffices. The condition is not adversarial; it is the expected operational configuration.

## Recommendation
The extension should gate the end user, not the immediate caller of `pool.swap()`. Options:

1. **Pass the original `msg.sender` through `extensionData`** — the router encodes the original caller in `extensionData`, and the extension decodes and checks it. This requires a coordinated change to the router and extension.
2. **Preferred:** The pool's `swap()` interface should expose the original initiator as a dedicated argument (separate from `sender`/`recipient`), and the extension should check that field.
3. **Until fixed:** Document that `SwapAllowlistExtension` is incompatible with router-mediated swaps and must only be used with direct pool calls; do not allowlist the router address.

## Proof of Concept
```
Setup:
  - Deploy pool with SwapAllowlistExtension
  - Admin: setAllowedToSwap(pool, userA, true)       // allowlist userA
  - Admin: setAllowedToSwap(pool, router, true)       // enable router swaps for userA

Attack:
  - userC (not allowlisted) calls:
      router.exactInputSingle({pool: pool, ...})
  - Router calls pool.swap() → msg.sender = router
  - Pool calls _beforeSwap(sender=router, ...)
  - Extension checks: allowedSwapper[pool][router] == true → PASSES
  - userC executes an unauthorized swap against the restricted pool

Expected: revert NotAllowedToSwap()
Actual:   swap succeeds
```
