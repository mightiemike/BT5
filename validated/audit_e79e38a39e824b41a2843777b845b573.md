Audit Report

## Title
`SwapAllowlistExtension` Gates on Router Address Instead of Originating EOA, Allowing Any User to Bypass the Swap Allowlist via `MetricOmmSimpleRouter` - (File: `metric-periphery/contracts/extensions/SwapAllowlistExtension.sol`)

## Summary
`SwapAllowlistExtension.beforeSwap` evaluates `allowedSwapper[pool][sender]` where `sender` is the immediate caller of `MetricOmmPool.swap()`. When a user routes through `MetricOmmSimpleRouter`, `sender` is the router's address, not the originating EOA. If the router is allowlisted (required for any user to use it), every non-allowlisted EOA can bypass the curated-pool gate by routing through the router, completely defeating the allowlist invariant.

## Finding Description
In `MetricOmmPool.swap`, the pool passes its own `msg.sender` as the `sender` argument to `_beforeSwap`: [1](#0-0) 

`SwapAllowlistExtension.beforeSwap` then evaluates the allowlist keyed on that `sender` (with `msg.sender` being the pool): [2](#0-1) 

In `MetricOmmSimpleRouter.exactInputSingle`, the router calls `pool.swap()` directly with no forwarding of the original `msg.sender` as the sender identity: [3](#0-2) 

The call chain is: `EOA → MetricOmmSimpleRouter.exactInputSingle → pool.swap(msg.sender=router) → _beforeSwap(sender=router) → SwapAllowlistExtension.beforeSwap(sender=router)`. The check becomes `allowedSwapper[pool][router]`, not `allowedSwapper[pool][EOA]`. The same flaw exists in `exactOutputSingle`, `exactInput`, and `exactOutput`.

## Impact Explanation
Two mutually exclusive failure modes exist, both breaking the core allowlist invariant:

**Mode A (router allowlisted):** Any non-allowlisted EOA calls `exactInputSingle` through the router. The extension sees `sender = router`, finds `allowedSwapper[pool][router] = true`, and passes. The individual-user allowlist is completely defeated — every user on the network can trade on a pool intended to be curated. This is a direct admin-boundary break: an unprivileged path bypasses the pool admin's access control.

**Mode B (router not allowlisted):** Allowlisted EOAs cannot use the router at all; they must call `pool.swap()` directly. This breaks the expected periphery UX and renders the supported swap entrypoint unusable for legitimate allowlisted users.

Both modes represent broken core pool functionality tied to the allowlist access control mechanism.

## Likelihood Explanation
`MetricOmmSimpleRouter` is the primary supported swap entrypoint for EOAs. Pool admins who deploy a `SwapAllowlistExtension` will naturally need to allowlist the router to make the pool usable, triggering Mode A. No special permissions, malicious setup, or non-standard tokens are required. Any EOA can call `exactInputSingle` on a router-allowlisted curated pool. The bypass is reachable on every router-mediated swap function.

## Recommendation
Pass the originating user identity through the swap path. The cleanest fix is to add an `originator` field to the pool's `swap` signature and have the router populate it with `msg.sender`. The extension then checks `allowedSwapper[pool][originator]` instead of `allowedSwapper[pool][sender]`. Alternatively, check `recipient` instead of `sender` if the pool's design guarantees the recipient is always the economic beneficiary, though this is weaker since `recipient` can be set to any address.

## Proof of Concept
1. Deploy a pool with `SwapAllowlistExtension` configured.
2. Pool admin calls `setAllowedToSwap(pool, router, true)` — required for any user to use the router.
3. Non-allowlisted EOA `Bob` calls `MetricOmmSimpleRouter.exactInputSingle({pool: pool, ...})`.
4. Router calls `pool.swap(recipient, ...)` with `msg.sender = router`.
5. Pool calls `_beforeSwap(router, ...)`.
6. Extension evaluates `allowedSwapper[pool][router] == true` → passes.
7. Bob's swap executes on the curated pool despite never being individually allowlisted.

Conversely, if the admin does **not** allowlist the router: allowlisted Alice calls `exactInputSingle`; extension evaluates `allowedSwapper[pool][router] == false` → reverts with `NotAllowedToSwap`; Alice cannot use the router even though she is individually allowlisted.
