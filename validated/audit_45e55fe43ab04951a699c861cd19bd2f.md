### Title
`_finalizeSubaccount` Skips Zero-Weight Spot Liabilities, Leaving Bad Debt Unhandled When Insurance Is Sufficient — (`File: core/contracts/ClearinghouseLiq.sol`)

---

### Summary

`_finalizeSubaccount` in `ClearinghouseLiq.sol` skips spot products with `longWeightInitialX18 == 0` when checking that all liabilities are cleared before finalization. When insurance is sufficient (`canLiquidateMore = true`), `spotEngine.socializeSubaccount` is never called, so any outstanding liability in a zero-weight spot product is left permanently on the liquidatee's account — creating unrecoverable bad debt.

---

### Finding Description

`_finalizeSubaccount` is triggered during liquidation when `txn.productId == type(uint32).max`. It is designed to fully close out an insolvent subaccount: verify all positions are flat, settle PnL, and cover remaining quote debt with insurance or socialization.

The check that enforces "all spot liabilities must be zero before finalization" (executed only when `canLiquidateMore = true`) explicitly skips products whose `longWeightInitialX18 == 0`:

```solidity
if (v.canLiquidateMore) {
    for (uint32 i = 1; i < v.spotIds.length; ++i) {
        uint32 spotId = v.spotIds[i];
        ...
        if (spotEngine.getRisk(spotId).longWeightInitialX18 == 0) {
            continue;   // ← zero-weight product liability is NOT required to be zero
        }
        require(balance.amount == 0, ERR_NOT_FINALIZABLE_SUBACCOUNT);
    }
}
``` [1](#0-0) 

After this check, `spotEngine.socializeSubaccount` — the only mechanism that would absorb remaining spot liabilities — is guarded by `if (v.insurance <= 0)`:

```solidity
if (v.insurance <= 0) {
    spotEngine.socializeSubaccount(txn.liquidatee);
}
``` [2](#0-1) 

When insurance is sufficient (`v.insurance > 0`), `spotEngine.socializeSubaccount` is **never called**. Any negative balance in a zero-weight spot product is therefore neither required to be zero, nor socialized, nor covered by insurance. It remains on the liquidatee's account after `_finalizeSubaccount` returns `true`.

`SpotEngine.socializeSubaccount` only processes `balance.amount < 0` entries: [3](#0-2) 

So the liability is permanently stranded — the liquidatee has no remaining positions to generate income to repay it, and no further liquidation path exists if the product's `shortWeightInitialX18` is also zero (meaning the liability does not affect health).

---

### Impact Explanation

A liquidatee's account is declared "finalized" while still carrying an unresolved spot liability in a zero-weight product. Since all perp positions are required to be zero and all non-zero-weight spot assets must be `<= 0` before finalization, the account has no remaining earning capacity. The liability is permanent bad debt absorbed by the protocol's accounting state, corrupting the invariant that a finalized subaccount has no outstanding obligations.

If the zero-weight product also has `shortWeightInitialX18 == 0`, the liability contributes nothing to health, so the account cannot be re-liquidated to recover it. Insurance funds that were available to cover the liability are left unused while the debt is silently abandoned.

---

### Likelihood Explanation

Zero-weight spot products (`longWeightInitialX18 == 0`) are a realistic protocol state — they correspond to delisted or deprecated assets. A user can accumulate a borrow liability in such a product through normal spot trading before the product is delisted. The finalization path (`productId == type(uint32).max`) is a standard liquidation call reachable by any liquidator. The condition `canLiquidateMore = true` (insurance + quote > 0) is the common case when the insurance fund is healthy. No privileged access is required.

---

### Recommendation

When `canLiquidateMore = true`, remove the `longWeightInitialX18 == 0` skip so that **all** spot product liabilities are required to be zero before finalization proceeds:

```diff
if (v.canLiquidateMore) {
    for (uint32 i = 1; i < v.spotIds.length; ++i) {
        uint32 spotId = v.spotIds[i];
        ISpotEngine.Balance memory balance = spotEngine.getBalance(spotId, txn.liquidatee);
-       if (spotEngine.getRisk(spotId).longWeightInitialX18 == 0) {
-           continue;
-       }
        require(balance.amount == 0, ERR_NOT_FINALIZABLE_SUBACCOUNT);
    }
}
```

Alternatively, call `spotEngine.socializeSubaccount` unconditionally (not only when `v.insurance <= 0`) so that any residual zero-weight liability is always absorbed before the function returns.

---

### Proof of Concept

1. A spot product `P` is configured with `longWeightInitialX18 = 0` (e.g., a delisted asset).
2. A user borrows `P`, creating a negative spot balance (liability) on their subaccount.
3. The user's subaccount becomes insolvent through other positions and falls under maintenance health.
4. A liquidator calls `liquidateSubaccountImpl` with `productId = type(uint32).max`, triggering `_finalizeSubaccount`.
5. All perp positions are already zero; all non-zero-weight spot assets are `<= 0`. The pre-conditions pass.
6. PnL is settled; insurance covers the quote balance. `v.insurance > 0` → `canLiquidateMore = true`.
7. The loop at lines 372–383 iterates spot products. For product `P`, `longWeightInitialX18 == 0` → `continue`. The liability is not checked.
8. `perpEngine.socializeSubaccount` is called. `v.insurance > 0` → `spotEngine.socializeSubaccount` is **not** called.
9. `_finalizeSubaccount` returns `true`. The liquidatee's account is considered finalized.
10. The liability in product `P` remains on the liquidatee's account. With no positions left and no health impact (if `shortWeightInitialX18 == 0` too), it is irrecoverable bad debt. [4](#0-3)

    function _finalizeSubaccount(
        IEndpoint.LiquidateSubaccount calldata txn,
        ISpotEngine spotEngine,
        IPerpEngine perpEngine
    ) internal returns (bool) {
        if (txn.productId != type(uint32).max) {
            return false;
        }
        // check whether the subaccount can be finalized:
        // - all perps positions have closed
        // - all spread positions have closed
        // - all spot assets have closed
        // - all positive pnls have been settled

        FinalizeVars memory v;

        v.spotIds = spotEngine.getProductIds();
        v.perpIds = perpEngine.getProductIds();

        require(v.spotIds[0] == QUOTE_PRODUCT_ID);

        // all spot assets (except USDC) must be closed out
        for (uint32 i = 1; i < v.spotIds.length; ++i) {
            uint32 spotId = v.spotIds[i];
            if (spotEngine.getRisk(spotId).longWeightInitialX18 == 0) {
                continue;
            }
            ISpotEngine.Balance memory balance = spotEngine.getBalance(
                spotId,
                txn.liquidatee
            );
            require(balance.amount <= 0, ERR_NOT_FINALIZABLE_SUBACCOUNT);
        }

        for (uint32 i = 0; i < v.perpIds.length; ++i) {
            uint32 perpId = v.perpIds[i];
            IPerpEngine.Balance memory balance = perpEngine.getBalance(
                perpId,
                txn.liquidatee
            );
            require(balance.amount == 0, ERR_NOT_FINALIZABLE_SUBACCOUNT);
        }

        // settle all positive pnl
        for (uint32 i = 0; i < v.perpIds.length; ++i) {
            uint32 perpId = v.perpIds[i];
            IPerpEngine.Balance memory balance = perpEngine.getBalance(
                perpId,
                txn.liquidatee
            );
            if (balance.vQuoteBalance > 0) {
                _settlePnlAgainstLiquidator(
                    txn,
                    perpId,
                    balance.vQuoteBalance,
                    spotEngine,
                    perpEngine
                );
            }
        }

        ISpotEngine.Balance memory quoteBalance = spotEngine.getBalance(
            QUOTE_PRODUCT_ID,
            txn.liquidatee
        );

        // settle all negative pnl until quote balance becomes 0
        for (uint32 i = 0; i < v.perpIds.length; ++i) {
            uint32 perpId = v.perpIds[i];
            IPerpEngine.Balance memory balance = perpEngine.getBalance(
                perpId,
                txn.liquidatee
            );
            if (balance.vQuoteBalance < 0 && quoteBalance.amount > 0) {
                int128 canSettle = MathHelper.max(
                    balance.vQuoteBalance,
                    -quoteBalance.amount
                );
                _settlePnlAgainstLiquidator(
                    txn,
                    perpId,
                    canSettle,
                    spotEngine,
                    perpEngine
                );
                quoteBalance.amount += canSettle;
            }
        }

        v.insurance = insurance;
        v.insurance -= lastLiquidationFees;
        v.canLiquidateMore = (quoteBalance.amount + v.insurance) > 0;

        if (v.canLiquidateMore) {
            for (uint32 i = 1; i < v.spotIds.length; ++i) {
                uint32 spotId = v.spotIds[i];
                ISpotEngine.Balance memory balance = spotEngine.getBalance(
                    spotId,
                    txn.liquidatee
                );
                if (spotEngine.getRisk(spotId).longWeightInitialX18 == 0) {
                    continue;
                }
                require(balance.amount == 0, ERR_NOT_FINALIZABLE_SUBACCOUNT);
            }
        }

        v.insurance = perpEngine.socializeSubaccount(
            txn.liquidatee,
            v.insurance
        );

        // we can assure that quoteBalance must be non positive, because if quoteBalance.amount > 0,
        // there must be 1) no negative pnl in perps, and 2) no liabilities in spot after above actions.
        // however, in this case the liquidatee must be healthy and cannot pass the health check at
        // the beginning.
        int128 insuranceCover = MathHelper.min(
            v.insurance,
            -quoteBalance.amount
        );
        if (insuranceCover > 0) {
            v.insurance -= insuranceCover;
            spotEngine.updateBalance(
                QUOTE_PRODUCT_ID,
                txn.liquidatee,
                insuranceCover
            );
        }
        if (v.insurance <= 0) {
            spotEngine.socializeSubaccount(txn.liquidatee);
        }
        v.insurance += lastLiquidationFees;
        insurance = v.insurance;
        return true;
    }

    function socializeSubaccount(bytes32 subaccount) external {
        require(msg.sender == address(_clearinghouse), ERR_UNAUTHORIZED);

        uint32[] memory _productIds = getProductIds();
        for (uint128 i = 0; i < _productIds.length; ++i) {
            uint32 productId = _productIds[i];

            State memory state = states[productId];
            Balance memory balance = balanceNormalizedToBalance(
                state,
                balances[productId][subaccount]
            );
            if (balance.amount < 0) {
                int128 totalDeposited = state.totalDepositsNormalized.mul(
                    state.cumulativeDepositsMultiplierX18
                );

                state.cumulativeDepositsMultiplierX18 = (totalDeposited +
                    balance.amount).div(state.totalDepositsNormalized);

                require(state.cumulativeDepositsMultiplierX18 > 0);

                state.totalBorrowsNormalized += balance.amount.div(
                    state.cumulativeBorrowsMultiplierX18
                );

                _setBalanceAndUpdateBitmap(
                    productId,
                    subaccount,
                    BalanceNormalized({amountNormalized: 0})
                );
                _setState(productId, state);
            }
        }
    }
