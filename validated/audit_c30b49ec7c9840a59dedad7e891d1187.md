### Title
Missing Validator Bonded Check in `MsgCommitDelegationToTier` Enables Incorrect Bonus Reward Accrual — (File: `x/tieredrewards/keeper/msg_server.go`)

---

### Summary

`MsgCommitDelegationToTier` creates a tier position with `LastKnownBonded = true` without verifying that the target validator is currently bonded. `MsgLockTier` explicitly enforces a bonded-validator check; `MsgCommitDelegationToTier` does not. When a position is created while the validator is unbonding, the UNBOND event that caused the transition is silently skipped (because `LastEventSeq` is set to the latest event sequence at creation time). When the validator later becomes bonded again, the lazy bonus-replay logic incorrectly treats the entire unbonding window as a bonded period and pays out bonus rewards from the `RewardsPool` for time the validator was not bonded.

---

### Finding Description

**`MsgLockTier` vs `MsgCommitDelegationToTier` asymmetry**

The ADR-006 reference table documents that `MsgLockTier` requires "validator bonded" as a key validation, while `MsgCommitDelegationToTier` does not list this requirement. This asymmetry is reflected in the code.

`CommitDelegationToTier` in `msg_server.go` calls only `validateNewPosition`, which checks three things: non-vesting account, `CloseOnly`, and `MinLockAmount`. There is no validator bonded check. [1](#0-0) 

`validateNewPosition` in `msg_validate.go`: [2](#0-1) 

**`LastKnownBonded = true` hardcoded at position creation**

The ADR documents that `createDelegatedPosition` always sets `LastKnownBonded = true` and `LastEventSeq = latestValidatorEventSeq`:

> Create Position: `DelegatorAddress = delAddr.String()`, `LastEventSeq = latestValidatorEventSeq`, `LastBonusAccrual = block_time`, `LastKnownBonded = true` [3](#0-2) 

**UNBOND event is silently skipped**

The store layout shows `ValidatorEventSeq` tracks the last-used sequence per validator. Events with `seq > LastEventSeq` are pending; events at or below `LastEventSeq` are skipped. If the validator transitioned to unbonding at seq N, and the position is created with `LastEventSeq = N`, the UNBOND event is not in the pending window. [4](#0-3) 

**Incorrect bonus accrual on the first BOND event**

The lazy bonus-replay formula for each segment uses the `bonded` variable, which is initialized from `LastKnownBonded` and updated by event type (`UNBOND → false`, `BOND → true`). Because the UNBOND event was skipped, `bonded` starts as `true`. When the validator is unjailed and a BOND event is recorded at seq N+1, the replay computes bonus for the segment `[LastBonusAccrual, BOND event time]` with `bonded = true`, covering the entire unbonding window as if the validator were bonded. [5](#0-4)

---

### Impact Explanation

The `RewardsPool` module account is debited for bonus rewards computed over the unbonding window. The corrupted value is the `bonus` coin amount sent via `SendCoinsFromModule(rewards_pool, owner, bonus)`. The magnitude is `shares × tokensPerShare × tier.BonusApy × unbondingDuration / SecondsPerYear`. For a large position in a high-APY tier with a long unbonding window, this can be a material drain on the pool. The pool balance is the exact on-chain value corrupted. [7](#0-6) 

---

### Likelihood Explanation

The trigger requires a validator to be in the unbonding (non-jailed) state at the moment `MsgCommitDelegationToTier` is submitted. Cosmos SDK allows delegation to unbonding (non-jailed) validators, so `transferDelegationToPosition` (which calls `Unbond` + `Delegate`) succeeds. Validator unbonding transitions occur in normal chain operation (e.g., a validator drops below the minimum self-delegation threshold). Any delegator with an existing delegation to such a validator can exploit this with a single standard transaction. No privileged access is required.

---

### Recommendation

Add a validator bonded check inside `CommitDelegationToTier`, mirroring the check already present in `LockTier`. Alternatively, derive `LastKnownBonded` from the live validator state at position creation time rather than hardcoding `true`, so that positions created against unbonding validators start with `LastKnownBonded = false`. [8](#0-7) 

---

### Proof of Concept

1. Validator V is bonded. User U has a delegation to V.
2. Validator V transitions to unbonding (not jailed). The module records `ValidatorEvent{type: UNBOND}` at seq N for V.
3. User U submits `MsgCommitDelegationToTier{validator: V, tier: 1, amount: X}`.
4. `CommitDelegationToTier` calls `validateNewPosition` — passes (no bonded check). `transferDelegationToPosition` succeeds (Cosmos SDK permits delegation to unbonding validators). `createDelegatedPosition` stores `LastKnownBonded = true`, `LastEventSeq = N`, `LastBonusAccrual = T0`.
5. Validator V is re-bonded. The module records `ValidatorEvent{type: BOND}` at seq N+1 for V.
6. User U submits `MsgClaimTierRewards{position_ids: [pos]}`.
7. `processEventsAndClaimBonus` walks events from seq N+1. It finds the BOND event. It computes bonus for segment `[T0, BOND event time]` using `bonded = LastKnownBonded = true`.
8. Bonus is paid out for the entire unbonding window `[T0, BOND event time]`, during which the validator was not bonded. The `RewardsPool` is debited for rewards that should not have accrued. [9](#0-8) [6](#0-5) [10](#0-9)

func (ms msgServer) CommitDelegationToTier(ctx context.Context, msg *types.MsgCommitDelegationToTier) (*types.MsgCommitDelegationToTierResponse, error) {
	if err := msg.Validate(); err != nil {
		return nil, err
	}

	tier, err := ms.getTier(ctx, msg.Id)
	if err != nil {
		return nil, err
	}

	if err := ms.validateNewPosition(ctx, msg.DelegatorAddress, msg.Amount, tier); err != nil {
		return nil, err
	}

	valAddr, err := sdk.ValAddressFromBech32(msg.ValidatorAddress)
	if err != nil {
		return nil, err
	}

	delegatorAddr, err := sdk.AccAddressFromBech32(msg.DelegatorAddress)
	if err != nil {
		return nil, err
	}

	id, err := ms.NextPositionId.Peek(ctx)
	if err != nil {
		return nil, err
	}

	delAddr, err := ms.createPositionDelegatorAccount(ctx, delegatorAddr, id)
	if err != nil {
		return nil, err
	}

	if _, err := ms.transferDelegationToPosition(ctx, msg.DelegatorAddress, delAddr, msg.ValidatorAddress, msg.Amount); err != nil {
		return nil, err
	}

	pos, err := ms.createDelegatedPosition(ctx, msg.DelegatorAddress, tier, valAddr, delAddr, msg.TriggerExitImmediately)
	if err != nil {
		return nil, err
	}

	// Defensive, but should not happen since transactions are sequential
	if pos.Id != id {
		return nil, errorsmod.Wrapf(types.ErrInvalidPositionID, "position id mismatch: peeked %d, created %d", id, pos.Id)
	}

	if err := ms.setPosition(ctx, pos, &ValidatorTransition{PreviousAddress: ""}); err != nil {
		return nil, err
	}

	sdkCtx := sdk.UnwrapSDKContext(ctx)

	if err := sdkCtx.EventManager().EmitTypedEvent(&types.EventDelegationCommitted{
		Position: pos,
	}); err != nil {
		return nil, err
	}

	return &types.MsgCommitDelegationToTierResponse{PositionId: pos.Id}, nil

func (k Keeper) validateNewPosition(ctx context.Context, owner string, amount math.Int, tier types.Tier) error {
	if err := k.validateNonVestingAccount(ctx, owner); err != nil {
		return err
	}

	if tier.IsCloseOnly() {
		return types.ErrTierIsCloseOnly
	}

	if !tier.MeetsMinLockRequirement(amount) {
		return types.ErrMinLockAmountNotMet
	}

	return nil
}

func (k Keeper) computeSegmentBonus(pos types.PositionState, tier types.Tier, segmentStart, segmentEnd time.Time, tokensPerShare math.LegacyDec) math.Int {
	if !pos.ExitUnlockAt.IsZero() && segmentEnd.After(pos.ExitUnlockAt) {
		segmentEnd = pos.ExitUnlockAt
	}

	if !segmentEnd.After(segmentStart) {
		return math.ZeroInt()
	}

	durationSeconds := int64(segmentEnd.Sub(segmentStart) / time.Second)
	if durationSeconds <= 0 {
		return math.ZeroInt()
	}

	tokens := pos.Delegation.Shares.Mul(tokensPerShare)

	return tokens.
		Mul(tier.BonusApy).
		MulInt64(durationSeconds).
		QuoInt64(types.SecondsPerYear).
		TruncateInt()
}
