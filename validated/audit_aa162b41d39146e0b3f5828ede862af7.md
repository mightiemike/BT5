### Title
Unprivileged Caller Can Aggressively Drain Mainchain Block History via `run_mainchain_gc` - (File: contract/src/lib.rs)

---

### Summary

`run_mainchain_gc` carries only a `#[pause]` guard, which restricts calls exclusively when the contract is paused. During normal (unpaused) operation, any unprivileged NEAR account can call it with an arbitrarily large `batch_size`, immediately deleting all mainchain block headers that exceed `gc_threshold`. This permanently removes entries from `mainchain_header_to_height` and `headers_pool`, causing `verify_transaction_inclusion` / `verify_transaction_inclusion_v2` to panic for any block that was pruned, and causing chain reorganizations to fail if the common ancestor was erased.

---

### Finding Description

`submit_blocks` is gated by `#[trusted_relayer]`, which enforces that only accounts holding `Role::UnrestrictedSubmitBlocks` (or `Role::DAO`) may write new headers. [1](#0-0) 

`run_mainchain_gc`, by contrast, carries only `#[pause(except(roles(Role::UnrestrictedRunGC)))]`. The `except(roles(...))` clause means that role holders may call the function even while paused — it does **not** restrict who may call the function when the contract is unpaused. In normal operation the function is fully open to any caller. [2](#0-1) 

The role comment confirms this reading: `UnrestrictedRunGC` is described as "Allows to use `run_mainchain_gc` API on a **paused** contract", not as a general access gate. [3](#0-2) 

Inside the function, `batch_size` is capped only by `total_amount_to_remove = amount_of_headers_we_store - gc_threshold`. Passing `u64::MAX` therefore removes every header that is eligible for pruning in a single transaction. [4](#0-3) 

Each removed header is deleted from both `mainchain_height_to_header` and `headers_pool` (via `remove_block_header`), and `mainchain_initial_blockhash` is advanced to the new oldest surviving block. [5](#0-4) 

---

### Impact Explanation

**SPV proof invalidation.** Both `verify_transaction_inclusion` and `verify_transaction_inclusion_v2` look up the target block in `mainchain_header_to_height`. If the block was pruned by the attacker, the call panics with `"block does not belong to the current main chain"`, permanently breaking any downstream contract or user that relies on that proof. [6](#0-5) 

**Chain reorganization failure.** `reorg_chain` walks the fork chain back to the common ancestor by checking `mainchain_header_to_height`. If the attacker has already pruned the common ancestor, the walk panics with `PrevBlockNotFound`, making it impossible to promote a valid heavier fork. The project documentation explicitly acknowledges this failure mode but treats it as a consequence of normal GC depth, not of an adversarial caller. [7](#0-6) 

The corrupted invariant is: *the contract guarantees that all mainchain blocks within `gc_threshold` of the tip are always available for SPV verification*. An attacker calling `run_mainchain_gc(u64::MAX)` immediately collapses the stored window to exactly `gc_threshold` blocks, racing ahead of the relayer's gradual pruning and invalidating proofs for any block that was just beyond the threshold.

---

### Likelihood Explanation

The attack requires no privileged key, no staked token, and no special role. Any NEAR account can call `run_mainchain_gc` with a single transaction. The function is publicly documented and its name makes its effect obvious. A griefing attacker targeting a bridge or cross-chain settlement system that depends on SPV proofs would have a clear economic motive.

---

### Recommendation

Add an explicit caller restriction to `run_mainchain_gc` that mirrors the pattern used for `submit_blocks`. The simplest fix is to require the caller to hold a designated role (e.g., `Role::UnrestrictedRunGC` itself, or a new `GCManager` role) even when the contract is unpaused:

```rust
// Before (only blocks when paused):
#[pause(except(roles(Role::UnrestrictedRunGC)))]
pub fn run_mainchain_gc(&mut self, batch_size: u64) { ... }

// After (always requires a role):
#[pause(except(roles(Role::UnrestrictedRunGC)))]
pub fn run_mainchain_gc(&mut self, batch_size: u64) {
    require!(
        self.acl_has_role(Role::UnrestrictedRunGC, &env::predecessor_account_id())
            || self.acl_has_role(Role::DAO, &env::predecessor_account_id()),
        "Caller is not authorized to run GC"
    );
    ...
}
```

Alternatively, make `run_mainchain_gc` `#[private]` and expose it only through `submit_blocks`, removing the public entry point entirely.

---

### Proof of Concept

1. Deploy the contract with `gc_threshold = 52704` and let the relayer submit enough blocks so that `amount_of_headers_we_store > gc_threshold` (e.g., 52800 blocks).
2. From any unprivileged NEAR account (no role granted), call:
   ```
   run_mainchain_gc(batch_size: 18446744073709551615)  // u64::MAX
   ```
3. The contract immediately removes all 96 eligible headers (52800 − 52704) in one transaction.
4. Any subsequent call to `verify_transaction_inclusion` or `verify_transaction_inclusion_v2` for a transaction in one of those 96 pruned blocks panics with `"block does not belong to the current main chain"`.
5. If a fork whose common ancestor falls within the pruned range is submitted, `reorg_chain` panics with `PrevBlockNotFound`, permanently blocking that reorganization. [8](#0-7) [9](#0-8)

    pub fn verify_transaction_inclusion(&self, #[serializer(borsh)] args: ProofArgs) -> bool {
        require!(
            args.confirmations <= self.gc_threshold,
            "The required number of confirmations exceeds the number of blocks stored in memory"
        );

        let heaviest_block_header = self
            .headers_pool
            .get(&self.mainchain_tip_blockhash)
            .unwrap_or_else(|| env::panic_str(ERR_KEY_NOT_EXIST));
        let target_block_height = self
            .mainchain_header_to_height
            .get(&args.tx_block_blockhash)
            .unwrap_or_else(|| env::panic_str("block does not belong to the current main chain"));

        // Check requested confirmations. No need to compute proof if insufficient confirmations.
        require!(
            (heaviest_block_header.block_height).saturating_sub(target_block_height) + 1
                >= args.confirmations,
            "Not enough blocks confirmed"
        );

        let header = self
            .headers_pool
            .get(&args.tx_block_blockhash)
            .unwrap_or_else(|| env::panic_str("cannot find requested transaction block"));

        require!(!args.merkle_proof.is_empty(), "Merkle proof is empty");

        // compute merkle tree root and check if it matches block's original merkle tree root
        merkle_tools::compute_root_from_merkle_proof(
            args.tx_id,
            usize::try_from(args.tx_index).unwrap(),
            &args.merkle_proof,
        ) == header.block_header.merkle_root
    }

    #[pause(except(roles(Role::UnrestrictedRunGC)))]
    pub fn run_mainchain_gc(&mut self, batch_size: u64) {
        let initial_blockheader = self
            .headers_pool
            .get(&self.mainchain_initial_blockhash)
            .unwrap_or_else(|| env::panic_str(ERR_KEY_NOT_EXIST));

        let tip_blockheader = self
            .headers_pool
            .get(&self.mainchain_tip_blockhash)
            .unwrap_or_else(|| env::panic_str(ERR_KEY_NOT_EXIST));

        let amount_of_headers_we_store =
            tip_blockheader.block_height - initial_blockheader.block_height + 1;

        if amount_of_headers_we_store > self.gc_threshold {
            let total_amount_to_remove = amount_of_headers_we_store - self.gc_threshold;
            let selected_amount_to_remove = std::cmp::min(total_amount_to_remove, batch_size);

            let start_removal_height = initial_blockheader.block_height;
            let end_removal_height = initial_blockheader.block_height + selected_amount_to_remove;
            env::log_str(&format!(
                "Num of blocks to remove {selected_amount_to_remove}"
            ));

            for height in start_removal_height..end_removal_height {
                let blockhash = &self
                    .mainchain_height_to_header
                    .get(&height)
                    .unwrap_or_else(|| env::panic_str(ERR_KEY_NOT_EXIST));

                self.remove_block_header(blockhash);
                self.mainchain_height_to_header.remove(&height);
            }

            self.mainchain_initial_blockhash = self
                .mainchain_height_to_header
                .get(&end_removal_height)
                .unwrap_or_else(|| env::panic_str(ERR_KEY_NOT_EXIST));
        }
    }
