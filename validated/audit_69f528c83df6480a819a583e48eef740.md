Audit Report

## Title
`SwapAllowlistExtension.beforeSwap` Checks Direct Pool Caller Instead of Originating User, Enabling Full Allowlist Bypass via Router — (`metric-periphery/contracts/extensions/SwapAllowlistExtension.sol`)

## Summary
`SwapAllowlistExtension.beforeSwap` gates swaps by checking `allowedSwapper[pool][sender]`, where `sender` is `msg.sender` of the pool's `swap` call. When `MetricOmmSimpleRouter` intermediates, the pool receives `msg.sender = router_address`, so the extension evaluates `allowedSwapper[pool][router]` rather than `allowedSwapper[pool][user]`. A pool admin who allowlists the router to enable router-based swaps inadvertently grants every on-chain user access to the curated pool, completely defeating the allowlist.

## Finding Description
**Root cause:** `MetricOmmPool.swap` passes `msg.sender` as the `sender` argument to `_beforeSwap`: [1](#0-0) 

`MetricOmmSimpleRouter.exactInputSingle` calls `pool.swap` directly, making the router the `msg.sender` in the pool: [2](#0-1) 

The extension then evaluates `allowedSwapper[msg.sender][sender]` where `sender = router_address`: [3](#0-2) 

**Contrast with `DepositAllowlistExtension`:** The deposit guard correctly checks `owner` — an explicit position-owner address passed as a separate parameter — not `sender` (the direct caller of `addLiquidity`): [4](#0-3) 

The `beforeSwap` interface only carries `sender` (direct pool caller) and `recipient` (output destination); neither is the originating user when the router intermediates: [5](#0-4) 

**Two broken scenarios:**
- **Scenario A (bypass):** Admin calls `setAllowedToSwap(pool, router, true)` to enable router-based swaps. Every user who calls `router.exactInputSingle` passes the guard because `allowedSwapper[pool][router] = true`, regardless of whether they are individually allowlisted.
- **Scenario B (lockout):** Admin allowlists individual addresses (`allowedSwapper[pool][alice] = true`) but not the router. Alice cannot use the router because the extension checks `allowedSwapper[pool][router] = false` and reverts `NotAllowedToSwap`.

## Impact Explanation
**Scenario A** constitutes a broken core pool functionality and admin-boundary break: the swap allowlist — the sole access control mechanism for curated pools — is fully bypassed by any unprivileged user via the supported router path. Unauthorized users trade against LP positions at oracle-anchored prices without LP consent, causing direct loss of LP value. This meets the High severity threshold.

**Scenario B** constitutes broken core swap functionality: explicitly allowlisted users cannot execute swaps through the primary periphery path, rendering the allowlist operationally unusable.

## Likelihood Explanation
`MetricOmmSimpleRouter` is the primary and only supported periphery swap path. Pool admins configuring a swap allowlist will naturally need to enable router-based access, making allowlisting the router address the obvious operational step. The bug is invisible during normal testing because the `SwapAllowlistExtension` test suite calls the extension directly from a mock pool address, never exercising the router path. Both scenarios are reachable by any unprivileged user with no special preconditions beyond a pool admin having deployed the extension.

## Recommendation
The `beforeSwap` hook interface must carry the originating user's address as a distinct parameter separate from `sender`. The correct fix is to extend `IMetricOmmExtensions.beforeSwap` with an explicit `origin` parameter (the original EOA or initiating contract), have `MetricOmmPool` populate it (e.g., via `tx.origin` or a router-supplied context), and check `allowedSwapper[pool][origin]` in the extension. Until the interface is extended, `SwapAllowlistExtension` should document that it is incompatible with `MetricOmmSimpleRouter`.

## Proof of Concept
```solidity
// Setup: pool with SwapAllowlistExtension; admin allowlists the router
swapExtension.setAllowedToSwap(address(pool), address(router), true);
// alice is NOT individually allowlisted
// allowedSwapper[pool][alice] == false

// Alice calls the router — extension sees sender = router, passes
vm.prank(alice);
router.exactInputSingle(
    IMetricOmmSimpleRouter.ExactInputSingleParams({
        pool:             address(pool),
        tokenIn:          address(token0),
        tokenOut:         address(token1),
        zeroForOne:       true,
        amountIn:         1_000,
        amountOutMinimum: 0,
        recipient:        alice,
        deadline:         block.timestamp + 1,
        priceLimitX64:    0,
        extensionData:    ""
    })
);
// ✓ swap succeeds — alice bypassed the allowlist
// Pool._beforeSwap received sender = address(router)
// Extension evaluated allowedSwapper[pool][router] = true
// allowedSwapper[pool][alice] = false was never checked
```
