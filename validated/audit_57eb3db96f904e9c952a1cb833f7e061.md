### Title
Attacker Can Steal Escrowed NFTs via Crafted Return-Path ClassId in `processReceivedPacket` — (`x/nft-transfer/keeper/packet.go`)

### Summary

`processReceivedPacket` determines source/sink direction solely by checking whether `data.ClassId` starts with `sourcePort/sourceChannel/`. Because `data.ClassId` is attacker-controlled application-layer packet data (not verified by IBC core), an attacker on chain B can create a fake NFT whose classID begins with the correct channel prefix, send it through a valid IBC channel, and trigger the unescrow path on chain A — stealing any NFT that is legitimately escrowed there.

---

### Finding Description

**Direction check** in `processReceivedPacket`:

```go
isAwayFromOrigin := types.IsAwayFromOrigin(
    packet.GetSourcePort(), packet.GetSourceChannel(), data.ClassId)
``` [1](#0-0) 

`IsAwayFromOrigin` returns `false` (unescrow path) whenever `data.ClassId` has the prefix `sourcePort + "/" + sourceChannel + "/"`:

```go
func IsAwayFromOrigin(sourcePort, sourceChannel, fullClassPath string) bool {
    prefixClassID := GetClassPrefix(sourcePort, sourceChannel)
    return !strings.HasPrefix(fullClassPath, prefixClassID)
}
``` [2](#0-1) 

When `isAwayFromOrigin == false`, the unescrow branch executes:

```go
unprefixedClassID := types.RemoveClassPrefix(
    packet.GetSourcePort(), packet.GetSourceChannel(), data.ClassId)
voucherClassID := types.ParseClassTrace(unprefixedClassID).IBCClassID()
for _, tokenID := range data.TokenIds {
    if err := k.nftKeeper.TransferOwner(ctx,
        voucherClassID, tokenID, escrowAddress, receiver); err != nil {
        return err
    }
}
``` [3](#0-2) 

The `escrowAddress` is derived from the **destination** port/channel (chain A's side):

```go
escrowAddress := types.GetEscrowAddress(packet.GetDestPort(), packet.GetDestChannel())
``` [4](#0-3) 

**`ValidateBasic` does not validate classID format** — it only checks that `ClassId` is non-blank:

```go
if strings.TrimSpace(nftpd.ClassId) == "" {
    return newsdkerrors.Wrap(ErrInvalidClassID, "classId cannot be blank")
}
``` [5](#0-4) 

**`createOutgoingPacket` on chain B** uses the raw classID as `fullClassPath` when it does not start with the IBC prefix `"ibc"`:

```go
fullClassPath = classID
if strings.HasPrefix(classID, nfttypes.IBCPrefix) {
    fullClassPath, err = k.ClassPathFromHash(ctx, classID)
}
``` [6](#0-5) 

And the packet data is built from `fullClassPath`:

```go
packetData := types.NewNonFungibleTokenPacketData(
    fullClassPath, denom.Uri, tokenIDs, tokenURIs, sender.String(), receiver,
)
``` [7](#0-6)

### Impact Explanation

**Concrete attack scenario** (two-chain case, channel `(p2,c2)` on B ↔ `(p1,c1)` on A):

1. Alice legitimately sends NFT (`classID = "nftClass"`, `tokenID = "T"`) from A to B. A escrows it at `escrowAddress("p1","c1")`.
2. Attacker on B creates a denom with `classID = "p2/c2/nftClass"` and mints tokenID `"T"` to themselves. No restriction prevents this — the NFT module accepts arbitrary denom IDs.
3. Attacker calls `SendTransfer` on B. `createOutgoingPacket` computes `IsAwayFromOrigin("p2","c2","p2/c2/nftClass") = false` → burns the fake NFT on B. Packet data: `data.ClassId = "p2/c2/nftClass"`, `data.TokenIds = ["T"]`, `data.Receiver = attacker`.
4. IBC core on A verifies the packet commitment (valid — it was committed on B). `OnRecvPacket` is called.
5. `processReceivedPacket` computes `IsAwayFromOrigin("p2","c2","p2/c2/nftClass") = false` → unescrow path.
6. `unprefixedClassID = RemoveClassPrefix("p2","c2","p2/c2/nftClass") = "nftClass"`.
7. `voucherClassID = ParseClassTrace("nftClass").IBCClassID() = "nftClass"`.
8. `TransferOwner(ctx, "nftClass", "T", escrowAddress("p1","c1"), attacker)` — transfers Alice's escrowed NFT to the attacker.

The attacker has stolen Alice's NFT. Alice's NFT is gone from escrow; the legitimate voucher on B (if any) still exists, creating unbacked supply. The invariant "unescrow only when a legitimate voucher was burned on the sending chain" is never enforced.

---

### Likelihood Explanation

- Requires only a valid IBC channel and the ability to create a denom on the counterparty chain — both are standard, permissionless operations on any ICS721-compatible chain.
- All escrowed NFTs on chain A are visible in public state; the attacker can enumerate targets.
- No privileged role, governance action, or key compromise is needed.
- The attack is repeatable for every escrowed NFT on every channel.

---

### Recommendation

In `processReceivedPacket`, when `isAwayFromOrigin == false`, verify that the `voucherClassID` derived from the packet actually exists as a denom on the receiving chain **and** that the escrow account holds the specific tokenID before calling `TransferOwner`. More robustly, maintain an explicit escrow accounting map `(classID, tokenID) → escrowed` that is written during `createOutgoingPacket` and checked/cleared during `processReceivedPacket`, mirroring the ICS20 fungible token approach.

---

### Proof of Concept

```
// Setup on chain A
IssueDenom(ctx, "nftClass", ...)
MintNFT(ctx, "nftClass", "T", ..., alice, alice)

// Alice sends NFT from A to B via channel (p1,c1)/(p2,c2)
// → A: TransferOwner("nftClass","T", alice, escrowAddress("p1","c1"))
// → B: MintNFT("ibc/{hash(p2/c2/nftClass)}", "T", ..., escrow, alice)

// Attacker on B (independent of Alice's voucher):
IssueDenom(ctx_B, "p2/c2/nftClass", ...)
MintNFT(ctx_B, "p2/c2/nftClass", "T", ..., attacker, attacker)
SendTransfer(ctx_B, "p2", "c2", "p2/c2/nftClass", ["T"], attacker, attacker_on_A, ...)
// → B burns "p2/c2/nftClass"/"T" (sink direction)
// → packet: data.ClassId="p2/c2/nftClass", data.TokenIds=["T"], data.Receiver=attacker_on_A

// Chain A receives packet via OnRecvPacket → processReceivedPacket:
// isAwayFromOrigin = !HasPrefix("p2/c2/nftClass","p2/c2/") = false  ← unescrow
// unprefixedClassID = "nftClass"
// voucherClassID = "nftClass"
// escrowAddress = GetEscrowAddress("p1","c1")
// TransferOwner("nftClass","T", escrowAddress("p1","c1"), attacker_on_A)
// → Alice's NFT is now owned by attacker_on_A
```

Assert: after the attack, `GetNFT("nftClass","T").Owner == attacker_on_A` and `escrowAddress("p1","c1")` no longer holds it. Alice's legitimate voucher on B (`ibc/{hash(p2/c2/nftClass)}`) is now unbacked.
