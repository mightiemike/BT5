### Title
`SwapAllowlistExtension` allowlist bypassed via router: any user can swap in a restricted pool by routing through `MetricOmmSimpleRouter` — (`metric-periphery/contracts/extensions/SwapAllowlistExtension.sol`)

---

### Summary

`SwapAllowlistExtension.beforeSwap` gates swaps by checking the `sender` argument, which the pool sets to `msg.sender` of `pool.swap()`. When a user swaps through `MetricOmmSimpleRouter`, the pool's `msg.sender` is the **router**, not the original user. If the pool admin adds the router to the allowlist (required for allowlisted users to use the router), every user — including non-allowlisted ones — can bypass the allowlist by routing through the public router.

---

### Finding Description

**Step 1 — Pool passes `msg.sender` as `sender` to the extension.**

In `MetricOmmPool.swap()`, `_beforeSwap` is called with `msg.sender` as the first argument: [1](#0-0) 

**Step 2 — Router calls `pool.swap()` directly; its own address becomes `msg.sender`.**

`MetricOmmSimpleRouter.exactInputSingle` calls `IMetricOmmPoolActions(params.pool).swap(...)` with no forwarding of the original caller: [2](#0-1) 

So when a user calls `exactInputSingle`, the pool sees `msg.sender = router`, and passes `sender = router` to every extension.

**Step 3 — `SwapAllowlistExtension` checks `sender`, which is now the router address.** [3](#0-2) 

The check `allowedSwapper[msg.sender][sender]` evaluates `allowedSwapper[pool][router]`, not `allowedSwapper[pool][originalUser]`.

**Step 4 — The trap.**

For allowlisted users to be able to use the router at all, the pool admin must add the router to `allowedSwapper`. Once `allowedSwapper[pool][router] = true`, the condition passes for **every** caller of the router — allowlisted or not — because the extension cannot distinguish between them. The allowlist is fully neutralized.

The same collapse occurs in `exactInput` (multi-hop) and `exactOutputSingle`/`exactOutput`: [4](#0-3) 

---

### Impact Explanation

A pool configured with `SwapAllowlistExtension` (e.g., for KYC compliance, institutional access, or regulatory gating) is completely open to any user who routes through `MetricOmmSimpleRouter`. The attacker receives pool output tokens at oracle-fair prices, draining liquidity that was reserved for authorized counterparties. This is a broken core pool functionality / admin-boundary break with direct loss of LP assets and protocol fee revenue from unauthorized swap volume.

---

### Likelihood Explanation

- `MetricOmmSimpleRouter` is the standard, publicly deployed periphery entry point that all ordinary users are expected to use.
- Any pool admin who wants their allowlisted users to be able to use the router **must** add the router to the allowlist — there is no other option.
- Once the router is allowlisted, the bypass is trivially reachable by any user with no special privileges, no malicious setup, and no non-standard tokens.
- The bypass requires only a single call to `exactInputSingle` on the public router.

---

### Recommendation

The extension must gate on the **original economic actor**, not the immediate pool caller. Two concrete approaches:

1. **Pass original user via `extensionData`:** Have the router encode `msg.sender` into `extensionData` and have the extension verify it (requires a trusted router registry or a signature scheme).
2. **Check `recipient` instead of `sender`:** For swap allowlists, gating on `recipient` (the address receiving output tokens) is a closer proxy to the economic actor and is not collapsed by the router.
3. **Separate router-level allowlist:** Deploy a router wrapper that enforces the allowlist before calling the pool, and configure the pool to only accept calls from that wrapper.

---

### Proof of Concept

```
Setup:
  - Pool deployed with SwapAllowlistExtension as beforeSwap hook
  - Pool admin calls setAllowedToSwap(pool, alice, true)       // alice is KYC'd
  - Pool admin calls setAllowedToSwap(pool, router, true)      // needed so alice can use the router
  - Pool admin does NOT call setAllowedToSwap(pool, bob, true) // bob is NOT KYC'd

Attack:
  - bob calls MetricOmmSimpleRouter.exactInputSingle({pool: pool, ...})
  - Router calls pool.swap(recipient=bob, ...) → msg.sender = router
  - Pool calls _beforeSwap(sender=router, ...)
  - Extension checks allowedSwapper[pool][router] == true → PASSES
  - Bob receives output tokens from the restricted pool
  - The allowlist guard is completely bypassed
``` [5](#0-4) [6](#0-5) [2](#0-1)
