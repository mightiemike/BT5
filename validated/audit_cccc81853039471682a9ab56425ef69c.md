Audit Report

## Title
`SwapAllowlistExtension` gates on the router address instead of the end user, allowing any unprivileged caller to bypass the swap allowlist via `MetricOmmSimpleRouter` - (File: metric-periphery/contracts/extensions/SwapAllowlistExtension.sol)

## Summary

`SwapAllowlistExtension.beforeSwap` checks `allowedSwapper[pool][sender]`, where `sender` is the direct `msg.sender` of `pool.swap()`. When swaps are routed through the public `MetricOmmSimpleRouter`, `sender` is the router contract address, not the end user. Any pool admin who allowlists the router to support normal router-mediated UX simultaneously grants every unprivileged user unrestricted access to the curated pool, completely defeating the allowlist's purpose.

## Finding Description

**Step 1 — Pool passes `msg.sender` as `sender`:**

`MetricOmmPool.swap()` calls `_beforeSwap(msg.sender, ...)` at line 231. When the router calls `pool.swap()`, `msg.sender` inside the pool is the router contract.

**Step 2 — Extension receives the router address as `sender`:**

`ExtensionCalling._beforeSwap` (lines 160–176) encodes `sender` verbatim into `abi.encodeCall(IMetricOmmExtensions.beforeSwap, (sender, ...))` and dispatches it to every configured extension.

**Step 3 — Extension checks the wrong identity:**

`SwapAllowlistExtension.beforeSwap` (line 37) evaluates:
```solidity
if (!allowAllSwappers[msg.sender] && !allowedSwapper[msg.sender][sender])
```
Here `msg.sender` is the pool and `sender` is whoever called `pool.swap()` — the router, not the end user.

**Step 4 — Router is public and permissionless:**

`MetricOmmSimpleRouter.exactInputSingle()` (lines 67–86) is callable by any address. It calls `pool.swap()` directly, making itself the `msg.sender` of that call. There is no access control on who may call the router.

**Exploit path:**
1. Pool admin deploys pool with `SwapAllowlistExtension` on `BEFORE_SWAP_ORDER`.
2. Pool admin calls `setAllowedToSwap(pool, router, true)` to enable normal router UX.
3. Attacker (never allowlisted) calls `MetricOmmSimpleRouter.exactInputSingle(pool, ...)`.
4. Router calls `pool.swap()`; pool passes `sender = router` to the extension.
5. Extension evaluates `allowedSwapper[pool][router] == true` → passes.
6. Attacker's swap executes against the curated pool.

**Contrast with `DepositAllowlistExtension`:**

`DepositAllowlistExtension.beforeAddLiquidity` (line 38) checks `allowedDepositor[msg.sender][owner]`, where `owner` is the position owner passed explicitly — the economically relevant actor regardless of who called `addLiquidity`. The swap allowlist has no equivalent `owner`-style parameter; it only has `sender`, which collapses to the intermediary when a router is used.

## Impact Explanation

A curated pool deploying `SwapAllowlistExtension` to restrict trading to KYC'd counterparties or whitelisted market makers is fully bypassed the moment the router is allowlisted. Any anonymous user can trade against the pool's liquidity, exposing LPs to unauthorized adverse selection and direct loss of principal. This is a broken core pool access-control mechanism causing direct loss of LP funds.

## Likelihood Explanation

`MetricOmmSimpleRouter` is the standard, documented user-facing entry point for swaps. Any pool that configures `SwapAllowlistExtension` and also wants to support router-mediated swaps (the normal UX) must allowlist the router, immediately opening the bypass to all users. No special privileges are required — any public address can call `exactInputSingle()`. The precondition (router allowlisted) is the expected operational state, not an edge case.

## Recommendation

The extension must gate on the end user's identity, not the intermediary's. Two viable approaches:

1. **Pass original caller through `extensionData`:** Have the router encode the original `msg.sender` into `extensionData`; have `SwapAllowlistExtension.beforeSwap` decode and check it. Requires a convention between router and extension.

2. **Add a `recipient`-style parameter:** Expose the swap `recipient` (already available in `beforeSwap` as the second argument, currently ignored with `address`) as an alternative gating identity, or introduce an explicit `originator` field analogous to `owner` in `DepositAllowlistExtension`.

The `DepositAllowlistExtension` pattern — checking `owner` rather than `sender` — is the correct model: the economically relevant actor must be the gated identity.

## Proof of Concept

```
1. Deploy pool with SwapAllowlistExtension on BEFORE_SWAP_ORDER.
2. Pool admin: setAllowedToSwap(pool, router, true).
3. Pool admin does NOT call setAllowedToSwap(pool, attacker, true).
4. Attacker calls MetricOmmSimpleRouter.exactInputSingle({pool: pool, ...}).
5. Router calls pool.swap(...) — msg.sender inside pool = router.
6. _beforeSwap passes sender = router to SwapAllowlistExtension.
7. Extension: allowedSwapper[pool][router] == true → no revert.
8. Attacker's swap executes; curated pool access control is fully bypassed.
```
