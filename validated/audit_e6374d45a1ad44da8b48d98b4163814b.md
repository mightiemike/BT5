### Title
`SwapAllowlistExtension` gates on the router address rather than the actual end-user, allowing any unprivileged swapper to bypass the per-address allowlist — (`metric-periphery/contracts/extensions/SwapAllowlistExtension.sol`)

---

### Summary

`SwapAllowlistExtension` is documented as "Gates `swap` by swapper address, per pool." Its `beforeSwap` hook receives a `sender` argument that the pool populates with `msg.sender` at the time `pool.swap()` is called. When a user routes through `MetricOmmSimpleRouter`, `msg.sender` at the pool is the **router contract**, not the end-user. The extension therefore checks whether the router is allowlisted, not whether the actual human swapper is allowlisted. Any user who is not individually allowlisted can bypass the guard entirely by routing through the router.

---

### Finding Description

`MetricOmmPool.swap` passes `msg.sender` as the `sender` argument to `_beforeSwap`: [1](#0-0) 

`ExtensionCalling._beforeSwap` forwards that value unchanged to the extension: [2](#0-1) 

`SwapAllowlistExtension.beforeSwap` then evaluates:

```solidity
if (!allowAllSwappers[msg.sender] && !allowedSwapper[msg.sender][sender]) {
    revert IMetricOmmPoolActions.NotAllowedToSwap();
}
``` [3](#0-2) 

Here `msg.sender` is the pool and `sender` is whoever called `pool.swap()`. When the call originates from `MetricOmmSimpleRouter`, `sender` equals the router's address, not the end-user's address.

The pool admin faces an inescapable dilemma:

| Router allowlisted? | Effect |
|---|---|
| **Yes** | Every user — including those not individually allowlisted — can swap by routing through the router. The per-address guard is fully bypassed. |
| **No** | Even individually allowlisted users cannot use the router. The pool is effectively router-incompatible. |

There is no configuration that simultaneously (a) allows allowlisted users to use the router and (b) blocks non-allowlisted users from using the router.

---

### Impact Explanation

A pool admin who deploys a permissioned pool (e.g., KYC-gated, institutional-only, or regulatory-restricted) and attaches `SwapAllowlistExtension` to enforce per-address access control receives no protection against router-mediated swaps. Any unprivileged address can execute swaps at the oracle-derived price, draining the pool's liquidity in directions the admin intended to restrict. Because the pool is oracle-priced, the swapper receives fair-value tokens; the harm is the **broken access-control invariant** and the resulting unauthorized outflow of pool assets to parties the admin explicitly excluded.

This matches the allowed-impact criterion: **admin-boundary break — factory/oracle role checks are bypassed by an unprivileged path**.

---

### Likelihood Explanation

- `MetricOmmSimpleRouter` is the primary production entry point for swaps.
- Any user who discovers the restriction can trivially route through the router without any special privilege or setup.
- The bypass requires zero capital beyond the swap itself and is repeatable every block.

Likelihood: **High**.

---

### Recommendation

Pass the actual initiating user through the extension data or add a dedicated `originalSender` field to the hook signature. As a short-term fix, `SwapAllowlistExtension.beforeSwap` should decode the real user from `extensionData` (caller-supplied and router-forwarded) and check that address instead of `sender`. Alternatively, the router should forward the original `msg.sender` in `extensionData` and the extension should verify it:

```solidity
// In SwapAllowlistExtension.beforeSwap:
address realSwapper = extensionData.length >= 20
    ? abi.decode(extensionData, (address))
    : sender;
if (!allowAllSwappers[msg.sender] && !allowedSwapper[msg.sender][realSwapper]) {
    revert IMetricOmmPoolActions.NotAllowedToSwap();
}
```

The router must be updated to always append `abi.encode(msg.sender)` to `extensionData` before calling `pool.swap`.

---

### Proof of Concept

1. Pool admin deploys a pool with `SwapAllowlistExtension` configured.
2. Admin calls `setAllowedToSwap(pool, alice, true)` — only Alice is allowed.
3. Admin calls `setAllowedToSwap(pool, router, true)` — router must be allowlisted for any router-based swap to work.
4. Bob (not allowlisted) calls `router.exactInputSingle({pool: pool, ...})`.
5. Router calls `pool.swap(recipient=bob, ...)` with `msg.sender = router`.
6. Pool calls `extension.beforeSwap(sender=router, ...)`.
7. Extension evaluates `allowedSwapper[pool][router]` → `true` → **no revert**.
8. Bob's swap executes successfully despite not being individually allowlisted. [4](#0-3) [5](#0-4)

contract SwapAllowlistExtension is BaseMetricExtension, ISwapAllowlistExtension {
  mapping(address pool => mapping(address swapper => bool)) public allowedSwapper;
  mapping(address pool => bool) public allowAllSwappers;

  constructor(address factory_) BaseMetricExtension(factory_) {}

  function setAllowedToSwap(address pool_, address swapper, bool allowed) external onlyPoolAdmin(pool_) {
    allowedSwapper[pool_][swapper] = allowed;
    emit AllowedToSwapSet(pool_, swapper, allowed);
  }

  function setAllowAllSwappers(address pool_, bool allowed) external onlyPoolAdmin(pool_) {
    allowAllSwappers[pool_] = allowed;
    emit AllowAllSwappersSet(pool_, allowed);
  }

  function isAllowedToSwap(address pool_, address swapper) external view returns (bool) {
    return allowAllSwappers[pool_] || allowedSwapper[pool_][swapper];
  }

  function beforeSwap(address sender, address, bool, int128, uint128, uint256, uint128, uint128, bytes calldata)
    external
    view
    override
    returns (bytes4)
  {
    if (!allowAllSwappers[msg.sender] && !allowedSwapper[msg.sender][sender]) {
      revert IMetricOmmPoolActions.NotAllowedToSwap();
    }
    return IMetricOmmExtensions.beforeSwap.selector;
  }
