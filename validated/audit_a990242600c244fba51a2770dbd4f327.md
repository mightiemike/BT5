### Title
`StatelessTransactionValidator::validate_resource_bounds` Unconditionally Applies `min_gas_price` to `l2_gas`, Rejecting Valid L1-Only-Bound Transactions at Gateway Admission — (`File: crates/apollo_gateway/src/stateless_transaction_validator.rs`)

---

### Summary

The `validate_resource_bounds` function in `StatelessTransactionValidator` applies a `min_gas_price` floor check unconditionally against `l2_gas.max_price_per_unit` for every incoming `RpcTransaction`. The production default for `min_gas_price` is `8_000_000_000`. A transaction that carries only L1 gas bounds (`l2_gas.max_price_per_unit = 0`) passes the zero-fee guard but is then rejected by the `min_gas_price` check, even though the same transaction type is explicitly marked as valid in the gateway's own test suite. The stateful validator correctly exempts L1-only transactions from the analogous L2 gas price threshold check; the stateless validator does not.

---

### Finding Description

In `crates/apollo_gateway/src/stateless_transaction_validator.rs`, `validate_resource_bounds` executes two sequential checks:

1. **Zero-fee guard** (line 66): wraps `AllResourceBounds` in `ValidResourceBounds::AllResources` and calls `max_possible_fee(Tip::ZERO)`. A transaction with non-zero `l1_gas` bounds passes this check even when `l2_gas` is entirely zero, because `max_possible_fee` sums all three resource components.

2. **`min_gas_price` floor** (line 71): unconditionally tests `resource_bounds.l2_gas.max_price_per_unit.0 < self.config.min_gas_price`. With the production default `min_gas_price = 8_000_000_000`, any transaction whose `l2_gas.max_price_per_unit` is zero fails here with `MaxGasPriceTooLow`. [1](#0-0) 

The production default is set in `StatelessTransactionValidatorConfig::default()`:

```rust
min_gas_price: 8_000_000_000,
``` [2](#0-1) 

The gateway's own positive-flow test `case::valid_l1_gas` explicitly asserts that a transaction with `AllResourceBounds { l1_gas: NON_EMPTY_RESOURCE_BOUNDS, ..Default::default() }` (i.e., `l2_gas.max_price_per_unit = 0`) **must pass** validation: [3](#0-2) 

That test only passes because the test fixture overrides `min_gas_price: 0`, masking the production-config failure: [4](#0-3) 

By contrast, the **stateful** validator's analogous check, `validate_tx_l2_gas_price_within_threshold`, correctly pattern-matches on `ValidResourceBounds::L1Gas` and skips the L2 price check entirely for legacy/L1-only transactions: [5](#0-4) 

The stateless validator operates on `AllResourceBounds` (the RPC wire type) and has no equivalent branch, so the exemption is never applied.

---

### Impact Explanation

**Impact: High — Mempool/gateway/RPC admission rejects valid transactions before sequencing.**

Any user who submits a V3 `RpcTransaction` with `AllResourceBounds` where only `l1_gas` is non-zero (a structurally valid transaction per the protocol and the gateway's own test assertions) will receive a `MaxGasPriceTooLow` rejection from the gateway. The transaction never reaches the mempool. This is a denial-of-service at the admission layer for a class of transactions the protocol explicitly supports.

---

### Likelihood Explanation

**Likelihood: Medium.**

The `RpcTransaction` type only accepts `AllResourceBounds` (not `ValidResourceBounds::L1Gas`), so the affected population is users who set `l2_gas` to zero while providing non-zero `l1_gas` or `l1_data_gas` bounds. This is a valid configuration per the Starknet spec for pre-0.13.3-style V3 transactions. The gateway's own test suite documents this as a passing case, meaning the discrepancy is not intentional. Any user or SDK that constructs such a transaction against a production node will be silently rejected.

---

### Recommendation

Mirror the stateful validator's logic in the stateless validator: skip the `min_gas_price` check on `l2_gas.max_price_per_unit` when `l2_gas.max_price_per_unit` is zero (indicating the transaction does not use L2 gas bounds). Concretely, guard line 71 with:

```rust
if resource_bounds.l2_gas.max_price_per_unit.0 != 0
    && resource_bounds.l2_gas.max_price_per_unit.0 < self.config.min_gas_price
{
    return Err(...);
}
```

Alternatively, align the test fixture `DEFAULT_VALIDATOR_CONFIG_FOR_TESTING` to use the production `min_gas_price` value and add a dedicated test case asserting that L1-only-bound transactions pass with the production config, which would surface the regression immediately.

---

### Proof of Concept

1. Start a gateway node with the default `StatelessTransactionValidatorConfig` (`min_gas_price = 8_000_000_000`, `validate_resource_bounds = true`).

2. Submit any `RpcTransaction` (Invoke, Declare, or DeployAccount) with:
   ```json
   "resource_bounds": {
     "l1_gas":      { "max_amount": "0x1", "max_price_per_unit": "0x1" },
     "l2_gas":      { "max_amount": "0x0", "max_price_per_unit": "0x0" },
     "l1_data_gas": { "max_amount": "0x0", "max_price_per_unit": "0x0" }
   }
   ```

3. **Zero-fee guard passes**: `max_possible_fee(Tip::ZERO) = 1 * 1 + 0 + 0 = 1 ≠ 0`.

4. **`min_gas_price` check fires**: `0 < 8_000_000_000` → `Err(MaxGasPriceTooLow { gas_price: 0, min_gas_price: 8_000_000_000 })`.

5. The transaction is rejected at the gateway. The same transaction submitted to a node with `min_gas_price = 0` (as in the test fixture) is accepted, confirming the divergence between test and production behavior. [6](#0-5) [7](#0-6)

    fn validate_resource_bounds(
        &self,
        tx: &RpcTransaction,
    ) -> StatelessTransactionValidatorResult<()> {
        if !self.config.validate_resource_bounds {
            return Ok(());
        }

        let resource_bounds = *tx.resource_bounds();
        // The resource bounds should be positive even without the tip.
        if ValidResourceBounds::AllResources(resource_bounds).max_possible_fee(Tip::ZERO) == Fee(0)
        {
            return Err(StatelessTransactionValidatorError::ZeroResourceBounds { resource_bounds });
        }

        if resource_bounds.l2_gas.max_price_per_unit.0 < self.config.min_gas_price {
            return Err(StatelessTransactionValidatorError::MaxGasPriceTooLow {
                gas_price: resource_bounds.l2_gas.max_price_per_unit,
                min_gas_price: self.config.min_gas_price,
            });
        }

        // TODO(Arni): Consider adding a validation for max_l2_gas_amount for declare.
        if let RpcTransaction::Declare(_) = tx {
        } else if resource_bounds.l2_gas.max_amount.0 > self.config.max_l2_gas_amount {
            return Err(StatelessTransactionValidatorError::MaxGasAmountTooHigh {
                gas_amount: resource_bounds.l2_gas.max_amount,
                max_gas_amount: self.config.max_l2_gas_amount,
            });
        }

        Ok(())
    }

    // TODO(Arni): Consider running this validation for all gas prices.
    fn validate_tx_l2_gas_price_within_threshold(
        &self,
        tx_resource_bounds: ValidResourceBounds,
        previous_block_l2_gas_price: NonzeroGasPrice,
    ) -> StatefulTransactionValidatorResult<()> {
        match tx_resource_bounds {
            ValidResourceBounds::AllResources(tx_resource_bounds) => {
                let tx_l2_gas_price = tx_resource_bounds.l2_gas.max_price_per_unit;
                let gas_price_threshold_multiplier =
                    Ratio::new(self.config.min_gas_price_percentage.into(), 100_u128);
                let threshold = (gas_price_threshold_multiplier
                    * previous_block_l2_gas_price.get().0)
                    .to_integer();
                if tx_l2_gas_price.0 < threshold {
                    return Err(StarknetError {
                        // We didn't have this kind of an error.
                        code: StarknetErrorCode::UnknownErrorCode(
                            "StarknetErrorCode.GAS_PRICE_TOO_LOW".to_string(),
                        ),
                        message: format!(
                            "Transaction L2 gas price {tx_l2_gas_price} is below the required \
                             threshold {threshold}.",
                        ),
                    });
                }
            }
            ValidResourceBounds::L1Gas(_) => {
                // No validation required for legacy transactions.
            }
        }
        Ok(())
    }
