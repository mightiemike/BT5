### Title
Checkpoint Advances Without Payment When `ErrInsufficientBonusPool` Is Swallowed in `BeforeRedelegationSlashed` — (`x/tieredrewards/keeper/slash.go`, `x/tieredrewards/keeper/claim_rewards.go`)

### Summary

`processEventsAndClaimBonus` mutates the position's `LastBonusAccrual`, `LastEventSeq`, and `LastKnownBonded` fields **in-place on the pointer** before it reaches the pool-balance guard. When the pool is empty, `ErrInsufficientBonusPool` is returned after those mutations have already occurred. `slashRedelegationPosition` deliberately swallows that error and then calls `setPositionWithState` with the already-mutated position, permanently advancing the checkpoint without paying the bonus. The victim cannot retry: the accrual window is gone.

### Finding Description

**Exact mutation order in `processEventsAndClaimBonus`** (`claim_rewards.go`):

`processEventsAndClaimBonus` receives `pos *types.PositionState` (a pointer). Every mutation above is visible to the caller.

**Error swallow in `slashRedelegationPosition`** (`slash.go`):

```go
if _, err := k.processEventsAndClaimBonus(ctx, &pos); err != nil {
    if errors.Is(err, types.ErrInsufficientBonusPool) {
        k.logger(ctx).Error(...)   // swallowed — pos is already mutated
    } else {
        return err
    }
}
// ...
return k.setPositionWithState(ctx, pos, nil)   // persists mutated pos
``` [3](#0-2) 

`setPositionWithState` writes `pos.Position` (including the advanced `LastBonusAccrual` and `LastEventSeq`) to the `Positions` collection. [4](#0-3) 

Because the hook returns `nil` (not the error), the Cosmos SDK cache-store is **not rolled back**. The `decrementEventRefCount` store writes (which may garbage-collect the events) are also committed.

**Result:** `LastBonusAccrual` is now set to `blockTime`, `LastEventSeq` is advanced past all pending events, and the events' reference counts are decremented — all without a single coin being sent to the owner. The accrual window is permanently closed; the victim cannot retry even after the pool is replenished.

### Impact Explanation

Any position that holds a pending redelegation (i.e., has a `RedelegationMappings` entry) and has accrued bonus rewards loses those rewards permanently if the `rewards_pool` balance is below the computed bonus at the moment the source validator is slashed. The loss is proportional to the accrued bonus: `shares × tokensPerShare × bonusApy × duration / SecondsPerYear`. [5](#0-4) 

### Likelihood Explanation

**Triggering the pool drain** is fully unprivileged: any set of position holders can legitimately claim their own bonus rewards until the pool is exhausted.

**Triggering the slash** requires the source validator to be slashed during the redelegation unbonding period. This can occur:
- Naturally (validator downtime → jailing → slash), with no attacker involvement beyond draining the pool first.
- Deliberately, if the attacker is also a validator operator who double-signs.

The ADR documents the "forfeit silently" behavior as an intentional chain-halt-avoidance trade-off:

> *"Bonus forfeits silently if the pool is insufficient (chain-halt avoidance)."* [6](#0-5) 

However, the ADR does **not** document that the checkpoint advances, making the forfeiture permanent and non-retryable. This is the gap between the stated invariant ("user paths fail atomically; retry after pool replenished") and the actual hook behavior. [7](#0-6) 

### Recommendation

Move `applyBonusAccrualCheckpoint` and `pos.UpdateLastKnownBonded` to **after** the pool-balance check and the `SendCoinsFromModuleToAccount` call succeed. If the pool check fails in the hook context, do not advance any checkpoint — leave `LastBonusAccrual` and `LastEventSeq` unchanged so the victim can claim after the pool is replenished. The `decrementEventRefCount` calls inside the event loop should similarly be deferred or skipped on the error path.

### Proof of Concept

1. Create a tier position for the victim; redelegate it to a second validator (this creates a `RedelegationMappings` entry with a non-zero `unbondingID`).
2. Advance block time so bonus accrues on the destination validator.
3. Drain `rewards_pool` to zero via legitimate `ClaimTierRewards` calls from other positions.
4. Call `Hooks().BeforeRedelegationSlashed(ctx, unbondingID, smallFraction)` directly (as the staking module would during a slash).
5. Assert:
   - No coins were sent to the victim's address.
   - `pos.LastBonusAccrual` equals `blockTime` (advanced).
   - `pos.LastEventSeq` is advanced past all pending events.
6. Replenish the pool and call `ClaimTierRewards` for the victim — observe that zero bonus is returned (the accrual window is gone). [8](#0-7)

func (s *KeeperSuite) TestSlashRedelegationPosition_ClaimsBonusRewardsUpToSlash() {
	lockAmount := sdkmath.NewInt(sdk.DefaultPowerReduction.Int64())
	_, bondDenom := s.getStakingData()
	s.fundRewardsPool(sdkmath.NewInt(1_000_000_000), bondDenom)

	pos, _, unbondingID := s.setupRedelegatingPosition(lockAmount)
	owner := sdk.MustAccAddressFromBech32(pos.Owner)
	preAccrual := pos.LastBonusAccrual

	// Advance block time so bonus accrues on the destination validator.
	s.ctx = s.ctx.WithBlockHeight(s.ctx.BlockHeight() + 1)
	s.ctx = s.ctx.WithBlockTime(s.ctx.BlockTime().Add(30 * 24 * time.Hour))

	balBefore := s.app.BankKeeper.GetBalance(s.ctx, owner, bondDenom)

	// Partial slash — a small fraction of shares.
	sharesToUnbond := pos.Delegation.Shares.Quo(sdkmath.LegacyNewDec(10))
	err := s.keeper.Hooks().BeforeRedelegationSlashed(s.ctx, unbondingID, sharesToUnbond)
	s.Require().NoError(err)

	balAfter := s.app.BankKeeper.GetBalance(s.ctx, owner, bondDenom)
	s.Require().True(balAfter.Amount.GT(balBefore.Amount),
		"owner should have received bonus rewards accrued up to slash: before=%s after=%s",
		balBefore.Amount, balAfter.Amount)

	updated, err := s.keeper.GetPositionState(s.ctx, pos.Id)
	s.Require().NoError(err)
	s.Require().True(updated.LastBonusAccrual.After(preAccrual),
		"LastBonusAccrual should have advanced past the pre-slash checkpoint")
	s.Require().Equal(s.ctx.BlockTime(), updated.LastBonusAccrual,
		"LastBonusAccrual should advance to the slash block time")
	s.Require().True(updated.LastKnownBonded,
		"LastKnownBonded should remain true — destination validator is still bonded")
}
