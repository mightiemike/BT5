### Title
IBC Voucher NFTs Permanently Untransferable On-Chain via `MsgTransferNFT` — (`x/nft/types/msgs.go`)

### Summary

`MsgTransferNFT.ValidateBasic()` calls `ValidateDenomID` instead of `ValidateDenomIDWithIBC`, which unconditionally rejects any denom ID containing the `ibc/` prefix. Because the IBC NFT receive path (`OnRecvPacket`) mints NFTs with denom IDs of the form `ibc/<64-hex-hash>`, those NFTs can never be transferred on-chain between users via `MsgTransferNFT`. The same defect affects `MsgEditNFT`, `MsgMintNFT`, and `MsgBurnNFT`.

### Finding Description

**`ValidateDenomID` rejects `ibc/` prefix unconditionally:** [1](#0-0) 

The regex `^[a-z0-9]+$` rejects `/`, so any `ibc/<hash>` denom fails immediately.

**`ValidateDenomIDWithIBC` correctly handles the IBC case:** [2](#0-1) 

**`MsgTransferNFT.ValidateBasic()` calls the wrong validator:** [3](#0-2) 

Line 86 calls `ValidateDenomID(msg.DenomId)` — not `ValidateDenomIDWithIBC`. The same mistake appears in `MsgEditNFT.ValidateBasic()` (line 135), `MsgMintNFT.ValidateBasic()` (line 183), and `MsgBurnNFT.ValidateBasic()` (line 221).

**IBC receive path mints NFTs with `ibc/<hash>` denom IDs:** [4](#0-3) 

`voucherClassID = classTrace.IBCClassID()` produces `ibc/<sha256-hex>`, which is stored directly as the NFT denom in the `x/nft` keeper. `ValidateGenesis` correctly uses `ValidateDenomIDWithIBC` for these: [5](#0-4) 

This asymmetry is the root cause: genesis and the IBC receive path accept `ibc/<hash>` denoms, but every user-facing `Msg*` operation rejects them at `ValidateBasic`.

### Impact Explanation

Any user who receives an IBC NFT on Cronos POS Chain via `OnRecvPacket` holds an NFT whose denom ID begins with `ibc/`. That user:

- **Cannot** call `MsgTransferNFT` to sell or gift it to another on-chain address — `ValidateBasic` returns `ErrInvalidDenom` before the tx reaches the keeper.
- **Cannot** call `MsgEditNFT` or `MsgBurnNFT` for the same reason.
- **Can** still call `MsgTransfer` (the IBC transfer message in `x/nft-transfer`) to send it back cross-chain, because that message's `ValidateBasic` only checks that `ClassId` is non-empty. [6](#0-5) 

The practical impact is that IBC voucher NFTs are **permanently non-transferable on-chain**. A user cannot sell, gift, or otherwise move the NFT to any other address on the same chain. The only escape is IBC re-transfer back to the origin chain, which may not always be available (channel closure, counterparty downtime). This constitutes a functional permanent lock of economically valuable NFTs for the duration the IBC channel is unavailable or the user is unaware of the workaround.

### Likelihood Explanation

The path is fully reachable with no special privileges:
1. Any relayer delivers an IBC NFT packet → `OnRecvPacket` → NFT minted with `ibc/<hash>` denom.
2. Recipient submits `MsgTransferNFT` with `DenomId = "ibc/AABB...64hex"`.
3. `ValidateBasic` → `ValidateDenomID` → `ErrInvalidDenom` → tx rejected.

This is reproducible in a standard Go/Cosmos test environment with no chain modifications.

### Recommendation

Replace `ValidateDenomID` with `ValidateDenomIDWithIBC` in the `ValidateBasic` methods of `MsgTransferNFT`, `MsgEditNFT`, `MsgMintNFT`, and `MsgBurnNFT` in `x/nft/types/msgs.go`. [3](#0-2) [7](#0-6) [8](#0-7) [9](#0-8)

func ValidateDenomIDWithIBC(denomID string) error {
	if strings.HasPrefix(denomID, IBCPrefix) {
		if len(denomID) != IBCDenomLen {
			return sdkerrors.Wrapf(ErrInvalidDenom, "the length of ibc denom(%s) only accepts value [%d]", denomID, IBCDenomLen)
		}
		if _, err := hex.DecodeString(denomID[len(IBCPrefix):]); err != nil {
			return sdkerrors.Wrapf(ErrInvalidDenom, "the hash of ibc denom(%s) must be valid hex", denomID)
		}

		return nil
	}

	return ValidateDenomID(denomID)
}

func (msg MsgTransfer) ValidateBasic() error {
	if err := host.PortIdentifierValidator(msg.SourcePort); err != nil {
		return newsdkerrors.Wrap(err, "invalid source port ID")
	}
	if msg.SourcePort != PortID {
		return newsdkerrors.Wrapf(ErrInvalidSourcePort, "source port must be %q", PortID)
	}
	if err := host.ChannelIdentifierValidator(msg.SourceChannel); err != nil {
		return newsdkerrors.Wrap(err, "invalid source channel ID")
	}

	if strings.TrimSpace(msg.ClassId) == "" {
		return newsdkerrors.Wrap(ErrInvalidClassID, "classId cannot be blank")
	}

	if len(msg.TokenIds) == 0 {
		return newsdkerrors.Wrap(ErrInvalidTokenID, "tokenId cannot be blank")
	}

	for _, tokenID := range msg.TokenIds {
		if strings.TrimSpace(tokenID) == "" {
			return newsdkerrors.Wrap(ErrInvalidTokenID, "tokenId cannot be blank")
		}
	}

	// NOTE: sender format must be validated as it is required by the GetSigners function.
	_, err := sdk.AccAddressFromBech32(msg.Sender)
	if err != nil {
		return newsdkerrors.Wrapf(sdkerrors.ErrInvalidAddress, "string could not be parsed as address: %v", err)
	}
	if strings.TrimSpace(msg.Receiver) == "" {
		return newsdkerrors.Wrap(sdkerrors.ErrInvalidAddress, "missing recipient address")
	}
	return nil
}
