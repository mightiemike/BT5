### Title
`AllResources`→`L1Gas` Collapse at Zero Boundary in Protobuf Deserialization Breaks Consensus P2P Transaction Round-Trip - (File: `crates/apollo_protobuf/src/converters/transaction.rs`)

### Summary

`TryFrom<protobuf::ResourceBounds> for ValidResourceBounds` collapses an `AllResources` variant to `L1Gas` whenever `l2_gas` and `l1_data_gas` are both zero. A valid `RpcInvokeTransactionV3` with only `l1_gas` non-zero (a gateway-accepted form) is serialized as `AllResources` but deserialized as `L1Gas`. The subsequent `TryFrom<InvokeTransactionV3> for RpcInvokeTransactionV3` conversion rejects any non-`AllResources` variant with a hard error, making the `ConsensusTransaction` undeserializable on the receiving peer. Additionally, `get_tip_resource_bounds_hash` produces a structurally different hash for `L1Gas` (2 resource felts) versus `AllResources` (3 resource felts), so even if deserialization were patched, the hash committed by the proposer would diverge from the hash recomputed by validators.

### Finding Description

**Step 1 – Boundary collapse in protobuf deserialization.**

`TryFrom<protobuf::ResourceBounds> for ValidResourceBounds` applies a zero-check:

```rust
Ok(if l1_data_gas.is_zero() && l2_gas.is_zero() {
    ValidResourceBounds::L1Gas(l1_gas)
} else {
    ValidResourceBounds::AllResources(AllResourceBounds { l1_gas, l2_gas, l1_data_gas })
})
``` [1](#0-0) 

**Step 2 – Serialization always emits `AllResources`.**

`RpcInvokeTransactionV3` stores `resource_bounds: AllResourceBounds`. `From<RpcInvokeTransactionV3> for InvokeTransactionV3` wraps it unconditionally as `ValidResourceBounds::AllResources(tx.resource_bounds)`. [2](#0-1) 

`From<ValidResourceBounds> for protobuf::ResourceBounds` faithfully serializes all three fields (l1_gas, l2_gas, l1_data_gas) even when l2_gas and l1_data_gas are zero. [3](#0-2) 

**Step 3 – Deserialization path rejects `L1Gas`.**

`TryFrom<protobuf::InvokeV3WithProof> for RpcInvokeTransactionV3` calls `snapi_invoke.try_into()` which is `TryFrom<InvokeTransactionV3> for RpcInvokeTransactionV3`. That conversion hard-errors on any non-`AllResources` variant:

```rust
resource_bounds: match value.resource_bounds {
    ValidResourceBounds::AllResources(bounds) => bounds,
    _ => {
        return Err(StarknetApiError::OutOfRange { string: "resource_bounds".to_string() });
    }
},
``` [4](#0-3) 

The error is mapped to `DEPRECATED_RESOURCE_BOUNDS_ERROR` and propagated up through `convert_consensus_tx_to_internal_consensus_tx`. [5](#0-4) 

**Step 4 – Hash domain divergence (secondary).**

`get_tip_resource_bounds_hash` hashes 2 resource felts for `L1Gas` and 3 for `AllResources`:

```rust
resource_felts.extend(match resource_bounds {
    ValidResourceBounds::L1Gas(_) => vec![],
    ValidResourceBounds::AllResources(all_resources) => {
        vec![get_concat_resource(&all_resources.l1_data_gas, L1_DATA_GAS)?]
    }
});
``` [6](#0-5) 

The proposer computes the hash with `AllResources` (3 felts); after the protobuf round-trip the validator would recompute with `L1Gas` (2 felts), producing a different `tx_hash`.

**Step 5 – Gateway accepts the triggering transaction.**

The stateless validator accepts `AllResourceBounds` with only `l1_gas` non-zero (l2_gas=0, l1_data_gas=0): [7](#0-6) 

### Impact Explanation

A user submits a valid Invoke V3 transaction with `AllResourceBounds { l1_gas: non-zero, l2_gas: 0, l1_data_gas: 0 }`. The gateway accepts it. The proposer includes it in a block proposal and broadcasts it via consensus P2P. Every receiving validator calls `TryFrom<protobuf::ConsensusTransaction> for ConsensusTransaction`, which internally hits the zero-boundary collapse and then the `AllResources`-only guard, returning `DEPRECATED_RESOURCE_BOUNDS_ERROR`. The validator cannot reconstruct the `InternalConsensusTransaction` and rejects the proposal. This is a **High** impact: transaction conversion logic binds the wrong type for the executable payload, causing valid transactions to be rejected during consensus propagation.

### Likelihood Explanation

The trigger is a standard, gateway-accepted Invoke V3 transaction with only `l1_gas` set. No special privileges are required. Any user can craft such a transaction. The bug fires on every consensus P2P round-trip for that transaction type.

### Recommendation

Remove the zero-value heuristic from `TryFrom<protobuf::ResourceBounds> for ValidResourceBounds`. Since `ConsensusTransaction` and `MempoolTransaction` always carry `AllResources` (as noted in the protobuf comment "Starting from 0.14.0, MempoolTransaction and ConsensusTransaction shouldn't have None here"), the conversion should unconditionally produce `AllResources` when all three fields are present, reserving `L1Gas` only for the legacy sync/storage path that explicitly lacks `l1_data_gas`. Alternatively, introduce a separate `TryFrom` implementation for the RPC/consensus path that always returns `AllResources`. [8](#0-7) 

### Proof of Concept

```
1. Craft RpcInvokeTransactionV3 with:
     resource_bounds = AllResourceBounds {
         l1_gas:      ResourceBounds { max_amount: 1000, max_price_per_unit: 1 },
         l2_gas:      ResourceBounds { max_amount: 0,    max_price_per_unit: 0 },
         l1_data_gas: ResourceBounds { max_amount: 0,    max_price_per_unit: 0 },
     }

2. Submit to gateway → accepted (matches `valid_l1_gas` test case).

3. Proposer wraps in ConsensusTransaction::RpcTransaction and serializes:
     From<ConsensusTransaction> for protobuf::ConsensusTransaction
     → From<RpcInvokeTransactionV3> for protobuf::InvokeV3WithProof
     → From<InvokeTransactionV3> for protobuf::InvokeV3
     → From<ValidResourceBounds::AllResources(...)> for protobuf::ResourceBounds
     Serialized bytes: l1_gas=1000/1, l2_gas=0/0, l1_data_gas=0/0

4. Receiving validator deserializes:
     TryFrom<protobuf::ResourceBounds> for ValidResourceBounds
     → l1_data_gas.is_zero() && l2_gas.is_zero() == true
     → returns ValidResourceBounds::L1Gas(l1_gas)   ← WRONG

5. TryFrom<InvokeTransactionV3> for RpcInvokeTransactionV3
     → resource_bounds is L1Gas, not AllResources
     → returns Err(OutOfRange { "resource_bounds" })
     → mapped to DEPRECATED_RESOURCE_BOUNDS_ERROR

6. convert_consensus_tx_to_internal_consensus_tx returns Err.
   Validator rejects the block proposal containing this transaction.
```

impl TryFrom<protobuf::ResourceBounds> for ValidResourceBounds {
    type Error = ProtobufConversionError;
    fn try_from(value: protobuf::ResourceBounds) -> Result<Self, Self::Error> {
        let Some(l1_gas) = value.l1_gas else {
            return Err(missing("ResourceBounds::l1_gas"));
        };
        let Some(l2_gas) = value.l2_gas else {
            return Err(missing("ResourceBounds::l2_gas"));
        };
        // TODO(Shahak): Assert data gas is not none once we remove support for 0.13.2.
        let l1_data_gas = value.l1_data_gas.unwrap_or_default();
        let l1_gas: ResourceBounds = l1_gas.try_into()?;
        let l2_gas: ResourceBounds = l2_gas.try_into()?;
        let l1_data_gas: ResourceBounds = l1_data_gas.try_into()?;
        Ok(if l1_data_gas.is_zero() && l2_gas.is_zero() {
            ValidResourceBounds::L1Gas(l1_gas)
        } else {
            ValidResourceBounds::AllResources(AllResourceBounds { l1_gas, l2_gas, l1_data_gas })
        })
    }
