### Title
`gas_prices_to_hash` omits `l2_gas_consumed` and `next_l2_gas_price` for V0_14_0+ blocks, producing a wrong block hash — (File: `crates/starknet_api/src/block_hash/block_hash_calculator.rs`)

---

### Summary

`gas_prices_to_hash` computes the gas-prices component that feeds into `calculate_block_hash`. For Starknet version ≥ V0_13_4 it hashes seven fields under the `"STARKNET_GAS_PRICES0"` domain. A TODO comment at line 416 explicitly acknowledges that `l2_gas_consumed` and `next_l2_gas_price` must be added to this hash for V0_14_0+ blocks, but the addition was never made. The current production Starknet version is 0.14.4, so every block produced by the sequencer carries a block hash that omits two canonical fields, diverging from the protocol-specified value.

---

### Finding Description

`gas_prices_to_hash` in `crates/starknet_api/src/block_hash/block_hash_calculator.rs` has exactly one version branch for modern blocks:

```rust
// TODO(Ayelet): add l2_gas_consumed, next_l2_gas_price after 0.14.0.
pub fn gas_prices_to_hash(
    l1_gas_price: &GasPricePerToken,
    l1_data_gas_price: &GasPricePerToken,
    l2_gas_price: &GasPricePerToken,
    block_hash_version: &BlockHashVersion,
) -> Vec<Felt> {
    if block_hash_version >= &BlockHashVersion::V0_13_4 {
        vec![
            HashChain::new()
                .chain(&STARKNET_GAS_PRICES0)
                .chain(&l1_gas_price.price_in_wei.0.into())
                .chain(&l1_gas_price.price_in_fri.0.into())
                .chain(&l1_data_gas_price.price_in_wei.0.into())
                .chain(&l1_data_gas_price.price_in_fri.0.into())
                .chain(&l2_gas_price.price_in_wei.0.into())
                .chain(&l2_gas_price.price_in_fri.0.into())
                .get_poseidon_hash(),
        ]
    } else { ... }
}

The function signature accepts only three price tokens; it has no parameters for `l2_gas_consumed` or `next_l2_gas_price`. `PartialBlockHashComponents`, the struct that feeds `calculate_block_hash`, also does not carry these fields: [2](#0-1) 

Yet both fields are first-class members of `BlockHeaderWithoutHash` (the authoritative on-chain header): [3](#0-2) 

And the echonet replay system explicitly passes them to the external block-hash CLI for every block it processes: [4](#0-3) 

The regression test for `gas_prices_to_hash` only covers `V0_13_2` and `V0_13_4`; no test exercises any V0_14_x version: [5](#0-4) 

The `BlockPostV0_13_1` feeder-gateway struct (used for syncing) carries `l2_gas_consumed` and `next_l2_gas_price` and maps them into `BlockHeaderWithoutHash`, confirming these are live network fields: [6](#0-5) 

---

### Impact Explanation

`calculate_block_hash` is the canonical function that produces the value stored in state, signed by consensus, and verified on L1. For every block at Starknet version ≥ 0.14.0 with non-zero `l2_gas_consumed` or `next_l2_gas_price` (i.e., every normal block), the sequencer computes:

```
gas_prices_hash = Poseidon(
    "STARKNET_GAS_PRICES0",
    l1_gas_price_wei, l1_gas_price_fri,
    l1_data_gas_price_wei, l1_data_gas_price_fri,
    l2_gas_price_wei, l2_gas_price_fri          ← stops here
)
```

The protocol-specified value (as evidenced by the TODO and the echonet CLI call) should be:

```
gas_prices_hash = Poseidon(
    "STARKNET_GAS_PRICES0",
    l1_gas_price_wei, l1_gas_price_fri,
    l1_data_gas_price_wei, l1_data_gas_price_fri,
    l2_gas_price_wei, l2_gas_price_fri,
    l2_gas_consumed,                             ← missing
    next_l2_gas_price                            ← missing
)
```

Because Poseidon is collision-resistant, the two hashes are distinct for any block with `l2_gas_consumed ≠ 0`. The wrong hash propagates into `calculate_block_hash` and is stored as the authoritative block hash. This matches the impact category: **Critical — wrong state value (block hash) produced by execution logic for accepted input**.

---

### Likelihood Explanation

The trigger is unprivileged and automatic: any block sequenced at Starknet version ≥ 0.14.0 with L2 gas usage (every non-empty block) produces the wrong hash. The current production version is 0.14.4. No special transaction or attacker action is required; the omission fires on every normal block.

---

### Recommendation

1. Add `l2_gas_consumed: GasAmount` and `next_l2_gas_price: GasPrice` to `PartialBlockHashComponents`.
2. Populate them in `PartialBlockHashComponents::new` from `BlockHeaderWithoutHash`.
3. Add a new version gate in `gas_prices_to_hash` (e.g., `block_hash_version >= &BlockHashVersion::V0_14_0`) that appends `l2_gas_consumed` and `next_l2_gas_price` to the Poseidon input after `l2_gas_price_fri`.
4. Pass the new fields through `calculate_block_hash` to `gas_prices_to_hash`.
5. Add regression tests covering V0_14_0+ block hashes with non-zero `l2_gas_consumed` and `next_l2_gas_price`.

---

### Proof of Concept

Given a V0_14_4 block with:
- `l2_gas_price_fri = 0x1dcd65000`, `l2_gas_price_wei = 0x7a1200`
- `l2_gas_consumed = 1_000_000`
- `next_l2_gas_price = 30_000_000_000`

Current Rust output (7-element Poseidon):
```
gas_prices_hash_actual = Poseidon("STARKNET_GAS_PRICES0",
    l1_gas_price_wei, l1_gas_price_fri,
    l1_data_gas_price_wei, l1_data_gas_price_fri,
    0x7a1200, 0x1dcd65000)
```

Protocol-expected output (9-element Poseidon):
```
gas_prices_hash_expected = Poseidon("STARKNET_GAS_PRICES0",
    l1_gas_price_wei, l1_gas_price_fri,
    l1_data_gas_price_wei, l1_data_gas_price_fri,
    0x7a1200, 0x1dcd65000,
    1_000_000,            ← omitted
    30_000_000_000)       ← omitted
```

`gas_prices_hash_actual ≠ gas_prices_hash_expected`, so `calculate_block_hash` returns a wrong `BlockHash` for every V0_14_0+ block with L2 gas usage.

#[tokio::test]
async fn test_block_hash_regression(
    #[values(BlockHashVersion::V0_13_2, BlockHashVersion::V0_13_4)]

        &transactions_data,
        state_diff,
        l1_da_mode,
        &block_hash_version.to_owned().into(),

    .await;
    let previous_block_hash = BlockHash(Felt::from(11_u8));

    let partial_block_hash_components = PartialBlockHashComponents {

        sequencer: SequencerContractAddress(ContractAddress(PatriciaKey::from(3_u8))),
        timestamp: BlockTimestamp(4),

        l2_gas_price: GasPricePerToken { price_in_fri: 11_u8.into(), price_in_wei: 12_u8.into() },

        header_commitments: block_commitments,
    };

        calculate_block_hash(&partial_block_hash_components, state_root, previous_block_hash)
