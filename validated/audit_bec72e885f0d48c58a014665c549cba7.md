Audit Report

## Title
`SwapAllowlistExtension` checks router address instead of end-user, allowing full allowlist bypass - (`metric-periphery/contracts/extensions/SwapAllowlistExtension.sol`)

## Summary
`SwapAllowlistExtension.beforeSwap()` gates swaps by checking the `sender` argument, which is always `msg.sender` of `MetricOmmPool.swap()`. When users route through `MetricOmmSimpleRouter`, the router becomes `msg.sender` of `pool.swap()`, so the extension checks `allowedSwapper[pool][router]` rather than the end-user. Any pool admin who allowlists the router to enable legitimate router-mediated swaps inadvertently opens the pool to every caller, including those explicitly excluded.

## Finding Description
`MetricOmmPool.swap()` passes `msg.sender` directly as the `sender` argument to `_beforeSwap()`: [1](#0-0) 

`SwapAllowlistExtension.beforeSwap()` uses that `sender` value to look up the allowlist, where `msg.sender` inside the extension is the pool (correct) and `sender` is whoever called `pool.swap()`: [2](#0-1) 

`MetricOmmSimpleRouter.exactInputSingle()` calls `pool.swap()` directly, making the router the `msg.sender` of that call: [3](#0-2) 

The same pattern holds for `exactInput()` (multi-hop, router is `msg.sender` of each `pool.swap()`): [4](#0-3) 

And for `exactOutputSingle()` and the recursive `exactOutput()` path. In every case, the router is the direct caller of `pool.swap()`, so `sender = router address` reaches the extension. The extension never sees the end-user address. The `allowedSwapper` mapping is keyed on the router, not the user, so once the router is allowlisted, the guard passes unconditionally for all callers.

## Impact Explanation
A pool configured with `SwapAllowlistExtension` to restrict swaps to specific addresses (KYC-gated, institutional-only, whitelist-only) is completely open to any caller who routes through `MetricOmmSimpleRouter`. The router is a public, permissionless contract with no access control. Any address — including those the pool admin explicitly excluded via `setAllowedToSwap(..., false)` — can call `exactInputSingle` or `exactInput` on the router and execute swaps against the restricted pool. The allowlist guard is rendered inoperative, breaking the core access-control invariant the pool admin configured. This constitutes a broken core pool functionality causing loss of funds (unauthorized parties can drain pool liquidity at oracle-quoted prices) and an admin-boundary break (pool admin's access control is bypassed by an unprivileged path).

## Likelihood Explanation
`MetricOmmSimpleRouter` is the standard user-facing entry point for swaps. Any pool admin who wants router-mediated swaps to work for their allowlisted users must allowlist the router address. Once that is done, the bypass is trivially reachable by any address with no special privileges, no malicious setup, and no non-standard tokens — just a normal call to a public router function. The precondition (router allowlisted) is a required operational step for any pool using the extension with the router, making this a near-certain occurrence in production.

## Recommendation
The extension must receive and check the economically relevant actor (the end-user), not the technical `msg.sender` of `pool.swap()`. Two complementary fixes:

1. **In `MetricOmmSimpleRouter`**: encode `msg.sender` (the true end-user) into `extensionData` so the extension can decode and verify it. The extension would then decode the user address from `extensionData` and check it against `allowedSwapper[pool][decodedUser]` rather than the `sender` argument.
2. **In `SwapAllowlistExtension`**: check `recipient` (the address receiving output tokens) instead of `sender` as a simpler alternative, though option 1 is more robust as it preserves full call-chain identity without changing the core pool interface.

## Proof of Concept
```
Setup:
  pool configured with SwapAllowlistExtension
  allowedSwapper[pool][router] = true   (admin enables router-mediated swaps)
  allowedSwapper[pool][alice]  = true   (alice is an approved user)
  allowedSwapper[pool][bob]    = false  (bob is explicitly excluded)

Attack:
  bob calls MetricOmmSimpleRouter.exactInputSingle({pool: pool, ...})
    → router calls pool.swap(recipient=bob, ...)
    → pool calls _beforeSwap(sender=router, ...)   // msg.sender = router
    → SwapAllowlistExtension.beforeSwap(sender=router, ...)
    → checks allowedSwapper[pool][router] == true  ✓
    → swap proceeds — bob receives output tokens

Result:
  bob, who was explicitly excluded from the allowlist, executes a swap
  against the restricted pool. The allowlist guard is completely bypassed.
```
