Audit Report

## Title
Unguarded `refundETH()` allows any caller to drain stranded ETH from `MetricOmmSimpleRouter` across transactions — (`metric-periphery/contracts/base/PeripheryPayments.sol`)

## Summary

`PeripheryPayments.refundETH()` transfers the router's entire native ETH balance to `msg.sender` with no access control. ETH enters the router legitimately via the `payable` `multicall()` entrypoint; if a user's multicall batch consumes only part of the sent ETH and omits a trailing `refundETH()` call, the remainder persists on the contract. Any subsequent unprivileged caller can invoke `refundETH()` in a separate transaction and receive all of it, causing direct loss of user ETH principal.

## Finding Description

`refundETH()` unconditionally sends `address(this).balance` to `msg.sender`: [1](#0-0) 

There is no check that `msg.sender` deposited the ETH, no reentrancy guard scoped to the depositor, and no transient-storage tracking of who sent value.

ETH enters via `multicall`, which is `public payable`: [2](#0-1) 

Because `multicall` uses `delegatecall`, every sub-call shares the original `msg.sender`; an attacker cannot inject a `refundETH()` call into a victim's batch. The attack is therefore **cross-transaction**:

Inside `pay()`, when `token == WETH` and the native balance exceeds the swap's required `value`, only the exact amount needed is wrapped and forwarded; the surplus remains as raw ETH on the contract: [3](#0-2) 

The `receive()` fallback blocks direct ETH sends from non-WETH addresses but does not prevent ETH from accumulating via payable function calls: [4](#0-3) 

**Exploit path:**
1. User A calls `multicall{value: 1 ETH}([exactInputSingle_calldata])` — swap consumes 0.5 ETH via `pay()`; 0.5 ETH remains on the router.
2. User A omits `refundETH()` from the batch (UI bug, direct interaction, or integrator error).
3. Attacker calls `router.refundETH()` in a separate transaction.
4. Attacker receives 0.5 ETH; User A's ETH is unrecoverable.

## Impact Explanation

Direct loss of user ETH principal. Any ETH deposited via a payable multicall that is not fully consumed by the swap and not reclaimed within the same multicall batch is permanently accessible to any external caller. The loss is bounded only by the amount the victim sent. This meets the allowed impact gate of Critical/High direct loss of user principal.

## Likelihood Explanation

Medium. Users following the standard Uniswap-style pattern are expected to append `refundETH()` as the last call in their multicall batch. Users who omit it — through UI bugs, direct contract interaction, or integrator error — leave ETH exposed. MEV bots routinely monitor for stranded ETH on router contracts and can extract it atomically in the next block.

## Recommendation

Track the depositor in transient storage at `multicall` entry and require `msg.sender == depositor` inside `refundETH()`. Alternatively, remove the standalone `refundETH()` external entrypoint and only allow ETH refunds within the multicall context where the original depositor is the `msg.sender`. A minimal fix:

```solidity
// In multicall(), before iterating:
assembly { tstore(DEPOSITOR_SLOT, caller()) }

// In refundETH():
address depositor;
assembly { depositor := tload(DEPOSITOR_SLOT) }
if (msg.sender != depositor) revert Unauthorized();
```

## Proof of Concept

```solidity
// SPDX-License-Identifier: MIT
pragma solidity ^0.8.35;

import "forge-std/Test.sol";
import {MetricOmmSimpleRouter} from "metric-periphery/contracts/MetricOmmSimpleRouter.sol";

contract RefundETHDrainTest is Test {
    MetricOmmSimpleRouter router;
    address userA    = address(0xA);
    address attacker = address(0xB);

    function setUp() public {
        router = new MetricOmmSimpleRouter(address(weth), address(factory));
        vm.deal(userA, 1 ether);
    }

    function test_attackerDrainsUserETH() public {
        // Simulate: userA's multicall consumed 0.5 ETH; 0.5 ETH remains on router.
        vm.deal(address(router), 0.5 ether);

        uint256 attackerBefore = attacker.balance;

        vm.prank(attacker);
        router.refundETH();

        assertEq(attacker.balance, attackerBefore + 0.5 ether);
        assertEq(address(router).balance, 0);
    }
}
```
