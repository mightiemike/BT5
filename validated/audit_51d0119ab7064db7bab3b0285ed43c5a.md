### Title
Bonus Overclaim via Position Creation on Unbonding Validator — (`x/tieredrewards/keeper/position.go`, `x/tieredrewards/keeper/msg_validate.go`)

---

### Summary

`validateNewPosition` does not check the validator's bond status. `createDelegatedPosition` unconditionally sets `LastKnownBonded = true`. When the validator later re-bonds, `processEventsAndClaimBonus` treats the entire interval from position creation to the BOND event as a bonded segment and pays bonus for it, even though the validator was unbonding throughout that period.

---

### Finding Description

**Step 1 — Missing guard in `validateNewPosition`**

`validateNewPosition` only checks for vesting accounts, `CloseOnly` tier flag, and minimum lock amount. There is no check that the target validator is currently bonded. [1](#0-0) 

**Step 2 — `LockTier` delegates to an unbonding validator without rejection**

`LockTier` calls `ms.delegate(ctx, delAddr, valAddr, msg.Amount)` after `validateNewPosition` passes. The Cosmos SDK `Delegate` function accepts delegations to unbonding validators (they land as unbonded delegations). No bond-status check exists anywhere in the `LockTier` path. [2](#0-1) 

**Step 3 — `createDelegatedPosition` hardcodes `LastKnownBonded = true`**

`createDelegatedPosition` calls `types.NewPosition(...)` with the literal `true` for `lastKnownBonded`, regardless of the actual validator bond status. The proto comment even documents the design intent: *"True when the position is first created (only bonded validators accepted)"* — but the enforcement is absent. [3](#0-2) [4](#0-3) 

**Step 4 — `LastEventSeq` is set to the current latest seq, skipping the prior UNBOND event**

`createDelegatedPosition` calls `getValidatorEventLatestSeq` and stores the result as `LastEventSeq`. If the UNBOND event was recorded at seq N before position creation, the new position's `LastEventSeq = N`, so `getValidatorEventsSince(ctx, valAddr, N)` returns only events with seq > N — the UNBOND event is permanently skipped. [5](#0-4) [6](#0-5) 

**Step 5 — `AfterValidatorBonded` records a BOND event that includes the attacker's position**

`setPosition` increments `PositionCountByValidator` for the unbonding validator. When the validator later re-bonds, `AfterValidatorBonded` fires with `count > 0` (the attacker's position is counted), so a BOND event at seq N+1 is recorded with a `ReferenceCount` that includes the attacker's position. [7](#0-6) 

**Step 6 — `processEventsAndClaimBonus` computes a bonded segment for the unbonded gap**

At claim time:
- `bonded = pos.LastKnownBonded = true`
- `segmentStart = pos.LastBonusAccrual = T_creation`
- Events since seq N: `[BOND event at seq N+1, timestamp T_bond]`

The loop processes the BOND event: because `bonded == true`, it computes bonus for `[T_creation, T_bond]` using the BOND event's `TokensPerShare`. This segment spans the entire unbonded period. The validator was actually unbonding during this interval, but the module pays bonus as if it were bonded. [8](#0-7) 

The tail-segment defensive check (`if bonded && val.IsBonded()`) only guards the final open segment after all events are processed — it does not retroactively correct the already-computed segment bonus for the unbonded gap. [9](#0-8) 

---

### Impact Explanation

The attacker receives bonus rewards for a period during which the validator was not bonded. The overclaimed amount is:

```
shares × tokensPerShare × bonusAPY × (T_bond − T_creation) / SecondsPerYear
```

This is a direct, quantifiable drain from the `RewardsPool` module account. The longer the validator remains unbonding after position creation, the larger the overclaim. The attacker can repeat this on any validator that cycles through unbonding/bonding.

---

### Likelihood Explanation

- No special privileges required — any user can submit `MsgLockTier`.
- Cosmos SDK allows delegating to unbonding validators; the transaction succeeds.
- Validators enter unbonding state routinely (jailing, tombstoning, voluntary unbonding).
- The attacker only needs to observe an unbonding validator and submit a transaction before it re-bonds.

---

### Recommendation

Add a validator bond-status check in `validateNewPosition` (or directly in `LockTier` / `CommitDelegationToTier` before calling `createDelegatedPosition`):

```go
val, err := k.stakingKeeper.GetValidator(ctx, valAddr)
if err != nil {
    return err
}
if !val.IsBonded() {
    return types.ErrValidatorNotBonded
}
```

This enforces the documented invariant that `LastKnownBonded = true` is only set when the validator is actually bonded at position creation time.

### Proof of Concept

```go
func (s *KeeperSuite) TestBonusOverclaimOnUnbondingValidator() {
    lockAmount := sdkmath.NewInt(sdk.DefaultPowerReduction.Int64())
    // Setup: create a first position so the validator has count > 0 for hook recording
    pos0 := s.setupNewTierPosition(lockAmount, false)
    valAddr := sdk.MustValAddressFromBech32(pos0.Delegation.ValidatorAddress)
    _, bondDenom := s.getStakingData()
    s.fundRewardsPool(sdkmath.NewInt(1_000_000_000), bondDenom)

    // Unbond the validator (records UNBOND event at seq 1)
    tUnbond := s.ctx.BlockTime().Add(1 * time.Second)
    s.ctx = s.ctx.WithBlockTime(tUnbond).WithBlockHeight(s.ctx.BlockHeight() + 1)
    s.jailAndUnbondValidator(valAddr)
    // seq is now 1 (UNBOND event)

    // Attacker creates a position on the now-unbonding validator
    tCreate := tUnbond.Add(1 * time.Second)
    s.ctx = s.ctx.WithBlockTime(tCreate).WithBlockHeight(s.ctx.BlockHeight() + 1)
    freshAddr := s.fundRandomAddr(bondDenom, lockAmount)
    msgServer := keeper.NewMsgServerImpl(s.keeper)
    resp, err := msgServer.LockTier(s.ctx, &types.MsgLockTier{
        Owner: freshAddr.String(), Id: 1, Amount: lockAmount,
        ValidatorAddress: valAddr.String(),
    })
    s.Require().NoError(err) // succeeds — no bond-status check
    attackerPos, _ := s.keeper.GetPositionState(s.ctx, resp.PositionId)
    s.Require().True(attackerPos.LastKnownBonded) // hardcoded true

    // Advance 30 days (unbonded gap), then re-bond
    tBond := tCreate.Add(30 * 24 * time.Hour)
    s.ctx = s.ctx.WithBlockTime(tBond).WithBlockHeight(s.ctx.BlockHeight() + 1)
    val, _ := s.app.StakingKeeper.GetValidator(s.ctx, valAddr)
    consAddr, _ := val.GetConsAddr()
    s.app.StakingKeeper.Unjail(s.ctx, consAddr)
    s.app.StakingKeeper.ApplyAndReturnValidatorSetUpdates(s.ctx)
    // BOND event recorded at seq 2, ReferenceCount includes attacker's position

    // Advance 1 more day bonded, then claim
    tNow := tBond.Add(24 * time.Hour)
    s.ctx = s.ctx.WithBlockTime(tNow).WithBlockHeight(s.ctx.BlockHeight() + 1)

    attackerPos, _ = s.keeper.GetPositionState(s.ctx, attackerPos.Id)
    bonus, err := s.keeper.ProcessEventsAndClaimBonus(s.ctx, &attackerPos)
    s.Require().NoError(err)

    // ASSERT: bonus should be zero for the 30-day unbonded gap.
    // ACTUAL: bonus is non-zero — the 30-day gap [tCreate, tBond] is paid as bonded.
    s.Require().True(bonus.IsZero(), "bonus must be zero for unbonded period — FAILS")
}
```

// getValidatorEventLatestSeq returns the sequence number of the most recent
// event for a validator. Returns 0 if no events have been appended.
// Used when creating positions to set LastEventSeq so that only future events

func (k Keeper) getValidatorEventLatestSeq(ctx context.Context, valAddr sdk.ValAddress) (uint64, error) {
	seq, err := k.ValidatorEventSeq.Get(ctx, valAddr)
	if errors.Is(err, collections.ErrNotFound) {
		return 0, nil
	}
	return seq, err
}

	count, err := h.k.getPositionCountForValidator(ctx, valAddr)
	if err != nil {
		return err
	}
	if count == 0 {
		return nil
	}

	tokensPerShare, err := h.k.getTokensPerShare(ctx, valAddr)
	if err != nil {
		return err
	}

	sdkCtx := sdk.UnwrapSDKContext(ctx)
	_, err = h.k.appendValidatorEvent(ctx, valAddr, types.ValidatorEvent{
		Height:         sdkCtx.BlockHeight(),
		Timestamp:      sdkCtx.BlockTime(),
		EventType:      types.ValidatorEventType_VALIDATOR_EVENT_TYPE_BOND,
		TokensPerShare: tokensPerShare,
		ReferenceCount: count,
	})
	return err
}
