### Title
64-Byte Merkle Internal-Node Forgery in `verify_transaction_inclusion` — (`contract/src/lib.rs`)

---

### Summary

`verify_transaction_inclusion` (v1) is still a live, publicly callable NEAR contract entrypoint. It performs no validation that the supplied `tx_id` is a leaf-level transaction hash. An attacker can supply an internal Merkle tree node as `tx_id` together with a proof that reconstructs the block's `merkle_root`, causing the function to return `true` for a value that is not a real transaction — a textbook 64-byte SPV forgery.

---

### Finding Description

`verify_transaction_inclusion` accepts `ProofArgs { tx_id, tx_index, merkle_proof, … }` and delegates entirely to:

```rust
merkle_tools::compute_root_from_merkle_proof(
    args.tx_id,
    usize::try_from(args.tx_index).unwrap(),
    &args.merkle_proof,
) == header.block_header.merkle_root
``` [1](#0-0) 

`compute_root_from_merkle_proof` is a pure hash-chain computation with no constraint that the starting value must be a leaf:

```rust
let mut current_hash = transaction_hash;   // accepts any H256
for proof_hash in merkle_proof {
    current_hash = compute_hash(&current_hash, proof_hash);  // or reversed
    current_position /= 2;
}
``` [2](#0-1) 

The function carries a `#[deprecated]` attribute and an explicit `# Warning` comment acknowledging the issue, but `#[deprecated]` is a Rust **compile-time** lint — it imposes zero runtime restriction. The function remains in the contract ABI and is callable by any NEAR account. [3](#0-2) 

The `#[pause]` attribute allows a `PauseManager` to disable the function, but the contract is **not paused by default** — no initialization code sets a paused state for this method. [4](#0-3) 

---

### Impact Explanation

Any caller (no role required) can invoke `verify_transaction_inclusion` with:
- `tx_id` = an internal Merkle node (e.g., `N01 = SHA256d(T0 || T1)`)
- `tx_index` = the position of that node at its level (e.g., `0`)
- `merkle_proof` = the sibling path from that level up to the root

The function returns `true`, falsely asserting that `N01` is an included transaction. Downstream consumers that rely on this return value for SPV proof acceptance (bridges, payment verifiers, etc.) are deceived into treating a non-existent transaction as confirmed.

---

### Likelihood Explanation

- No special role or privilege is required — any NEAR account can call the function.
- The required inputs (internal node values, sibling path) are fully derivable from the public Bitcoin block data already stored in the contract's `headers_pool`.
- The attack requires no hash preimage inversion; it reuses existing tree structure.
- The function is not paused in any default deployment configuration.

---

### Recommendation

1. **Immediate**: Pause `verify_transaction_inclusion` (v1) via the `PauseManager` role on all deployed instances.
2. **Short-term**: Remove the function body entirely (replace with `env::panic_str("deprecated")`) so the ABI entry becomes a hard revert.
3. **Long-term**: Migrate all callers to `verify_transaction_inclusion_v2`, which validates a coinbase merkle proof at the same tree depth, closing the 64-byte forgery window. [5](#0-4) 

---

### Proof of Concept

Given a block with four transactions `[T0, T1, T2, T3]` already stored in the contract:

```
Level 0 (leaves): T0   T1   T2   T3
Level 1 (nodes):  N01=SHA256d(T0||T1)   N23=SHA256d(T2||T3)
Root:             R  =SHA256d(N01||N23)
```

Call:
```
verify_transaction_inclusion({
    tx_id:            N01,          // internal node, NOT a real tx
    tx_block_blockhash: <block_hash>,
    tx_index:         0,
    merkle_proof:     [N23],        // one-element proof
    confirmations:    0,
})
```

`compute_root_from_merkle_proof(N01, 0, [N23])`:
- position 0 is even → `hash(N01, N23)` = `R`
- `R == header.merkle_root` → returns **`true`**

The function confirms inclusion of `N01` — which is not a transaction — against a real, PoW-valid block header. No forgery of block headers is required; all values are read from the existing chain state. [6](#0-5) [7](#0-6)
