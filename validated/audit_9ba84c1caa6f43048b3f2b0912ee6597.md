### Title
`ValidResourceBounds` protobuf deserialization silently downgrades `AllResources` to `L1Gas`, causing transaction hash and gas-mode divergence across P2P-synced nodes — (`crates/apollo_protobuf/src/converters/transaction.rs`)

### Summary
When a V3 transaction is submitted with `AllResourceBounds { l1_gas: X, l2_gas: 0, l1_data_gas: 0 }`, the sequencer computes its hash including the `L1_DATA_GAS` field (because `AllResources` always includes it in `get_tip_resource_bounds_hash`). However, when this transaction is serialized to protobuf and deserialized by a P2P-syncing node, the `ValidResourceBounds::try_from` conversion silently converts `AllResources` with zero `l2_gas` and `l1_data_gas` to `L1Gas`. The `L1Gas` variant excludes `L1_DATA_GAS` from the hash preimage, producing a different hash. Additionally, the gas computation mode changes from `All` to `NoL2Gas`, affecting fee accounting.

### Finding Description

**Root cause — lossy protobuf deserialization:**

In `crates/apollo_protobuf/src/converters/transaction.rs`, the `ValidResourceBounds::try_from(protobuf::ResourceBounds)` implementation contains a lossy downgrade:

```rust
Ok(if l1_data_gas.is_zero() && l2_gas.is_zero() {
    ValidResourceBounds::L1Gas(l1_gas)          // ← drops AllResources variant
} else {
    ValidResourceBounds::AllResources(AllResourceBounds { l1_gas, l2_gas, l1_data_gas })
})
``` [1](#0-0) 

**Hash divergence — `get_tip_resource_bounds_hash` branches on variant:**

`get_tip_resource_bounds_hash` in `crates/starknet_api/src/transaction_hash.rs` includes `L1_DATA_GAS` in the hash preimage only for `AllResources`, not for `L1Gas`:

```rust
resource_felts.extend(match resource_bounds {
    ValidResourceBounds::L1Gas(_) => vec![],                          // ← no L1_DATA_GAS
    ValidResourceBounds::AllResources(all_resources) => {
        vec![get_concat_resource(&all_resources.l1_data_gas, L1_DATA_GAS)?]  // ← included
    }
});
``` [2](#0-1) 

**Concrete divergence path:**

1. A user submits a V3 invoke transaction with `AllResourceBounds { l1_gas: X, l2_gas: 0, l1_data_gas: 0 }`. This is a valid, accepted transaction.
2. The gateway converts it to `InternalRpcInvokeTransactionV3` (stores `AllResourceBounds`). Its `InvokeTransactionV3Trait` impl always wraps as `ValidResourceBounds::AllResources(self.resource_bounds)`. [3](#0-2) 
3. The hash **H1** is computed via `get_invoke_transaction_v3_hash` → `get_tip_resource_bounds_hash` → includes `L1_DATA_GAS = 0` in the Poseidon preimage. [4](#0-3) 
4. The transaction is stored in the block as `InvokeTransactionV3 { resource_bounds: ValidResourceBounds::AllResources(...) }` (the `From<InternalRpcInvokeTransactionV3> for InvokeTransactionV3` conversion always uses `AllResources`). [5](#0-4) 
5. The block is propagated via P2P sync. The `InvokeTransactionV3` is serialized to protobuf with `l2_gas = 0` and `l1_data_gas = 0`.
6. The syncing node deserializes via `ValidResourceBounds::try_from(protobuf::ResourceBounds)`. Because `l2_gas.is_zero() && l1_data_gas.is_zero()`, the result is `L1Gas(X)` — the `AllResources` variant is silently lost. [6](#0-5) 
7. The syncing node computes hash **H2** via the same `get_tip_resource_bounds_hash` path, but now with `L1Gas` → `L1_DATA_GAS` is **not** included → **H1 ≠ H2**.
8. Additionally, `get_gas_vector_computation_mode()` returns `NoL2Gas` for `L1Gas` vs `All` for `AllResources`, changing the fee/gas accounting path. [7](#0-6) 

### Impact Explanation

**Critical/High — two distinct effects:**

1. **Hash divergence (Critical):** The transaction hash computed by the originating sequencer (H1, with `L1_DATA_GAS` in preimage) differs from the hash recomputed by any P2P-syncing node (H2, without `L1_DATA_GAS`). Any node that calls `validate_transaction_hash` or recomputes the hash from the deserialized transaction will get H2 ≠ H1, causing it to reject a valid block or accept a wrong hash as authoritative. This matches: *"Wrong state, receipt, event, L1 message, class hash, storage value, or revert result from blockifier/syscall/execution logic for accepted input."*

2. **Gas-mode divergence (Critical):** The syncing node executes the transaction with `GasVectorComputationMode::NoL2Gas` instead of `All`. This changes which resource bounds are checked during pre-validation and which gas costs are charged, producing a different fee and potentially a different revert/success outcome. This matches: *"Incorrect fee, gas, bouncer, resource accounting, refund, balance, or L1 gas price effect with economic impact."*

### Likelihood Explanation

Any unprivileged user can trigger this by submitting a V3 invoke or declare transaction with `l2_gas = 0` and `l1_data_gas = 0` (a common pattern for users who only want to pay L1 gas). The divergence is automatic and deterministic whenever such a block is synced via P2P. No special permissions or malicious peer behavior is required.

### Recommendation

The `ValidResourceBounds::try_from(protobuf::ResourceBounds)` conversion must not silently downgrade `AllResources` to `L1Gas`. The variant should be preserved canonically, for example by encoding the variant tag explicitly in the protobuf message, or by always deserializing to `AllResources` when all three resource fields are present (even if zero). The hash computation invariant must hold: the same transaction bytes must always produce the same hash regardless of which node processes them.

### Proof of Concept

```
1. Submit RpcInvokeTransactionV3 with:
     resource_bounds = AllResourceBounds { l1_gas: 1000, l2_gas: 0, l1_data_gas: 0 }

2. Originating node:
     InternalRpcInvokeTransactionV3::resource_bounds() 
       → ValidResourceBounds::AllResources({ l1_gas:1000, l2_gas:0, l1_data_gas:0 })
     get_tip_resource_bounds_hash:
       resource_felts = [L1_GAS_concat(1000), L2_GAS_concat(0), L1_DATA_GAS_concat(0)]
     H1 = Poseidon(tip, L1_GAS_concat, L2_GAS_concat, L1_DATA_GAS_concat)

3. Block propagated via P2P → protobuf serialization:
     ResourceBounds { l1_gas: 1000, l2_gas: 0, l1_data_gas: 0 }

4. Syncing node deserializes:
     ValidResourceBounds::try_from → l1_data_gas.is_zero() && l2_gas.is_zero()
       → ValidResourceBounds::L1Gas(1000)   ← AllResources variant lost

5. Syncing node:
     get_tip_resource_bounds_hash:
       resource_felts = [L1_GAS_concat(1000), L2_GAS_concat(0)]  ← no L1_DATA_GAS
     H2 = Poseidon(tip, L1_GAS_concat, L2_GAS_concat)

6. H1 ≠ H2  →  hash mismatch; gas mode = NoL2Gas instead of All
```

impl TryFrom<protobuf::ResourceBounds> for ValidResourceBounds {
    type Error = ProtobufConversionError;
    fn try_from(value: protobuf::ResourceBounds) -> Result<Self, Self::Error> {
        let Some(l1_gas) = value.l1_gas else {
            return Err(missing("ResourceBounds::l1_gas"));
        };
        let Some(l2_gas) = value.l2_gas else {
            return Err(missing("ResourceBounds::l2_gas"));
        };
        // TODO(Shahak): Assert data gas is not none once we remove support for 0.13.2.
        let l1_data_gas = value.l1_data_gas.unwrap_or_default();
        let l1_gas: ResourceBounds = l1_gas.try_into()?;
        let l2_gas: ResourceBounds = l2_gas.try_into()?;
        let l1_data_gas: ResourceBounds = l1_data_gas.try_into()?;
        Ok(if l1_data_gas.is_zero() && l2_gas.is_zero() {
            ValidResourceBounds::L1Gas(l1_gas)
        } else {
            ValidResourceBounds::AllResources(AllResourceBounds { l1_gas, l2_gas, l1_data_gas })
        })
    }

pub(crate) fn get_invoke_transaction_v3_hash<T: InvokeTransactionV3Trait>(
    transaction: &T,
    chain_id: &ChainId,
    transaction_version: &TransactionVersion,
) -> Result<TransactionHash, StarknetApiError> {
    let tip_resource_bounds_hash =
        get_tip_resource_bounds_hash(&transaction.resource_bounds(), transaction.tip())?;
    let paymaster_data_hash =
        HashChain::new().chain_iter(transaction.paymaster_data().0.iter()).get_poseidon_hash();
    let data_availability_mode = concat_data_availability_mode(
        transaction.nonce_data_availability_mode(),
        transaction.fee_data_availability_mode(),
    );
    let account_deployment_data_hash = HashChain::new()
        .chain_iter(transaction.account_deployment_data().0.iter())
        .get_poseidon_hash();
    let calldata_hash =
        HashChain::new().chain_iter(transaction.calldata().0.iter()).get_poseidon_hash();
    let mut hash_chain = HashChain::new()
        .chain(&INVOKE)
        .chain(&transaction_version.0)
        .chain(transaction.sender_address().0.key())
        .chain(&tip_resource_bounds_hash)
        .chain(&paymaster_data_hash)
        .chain(&Felt::try_from(chain_id)?)
        .chain(&transaction.nonce().0)
        .chain(&data_availability_mode)
        .chain(&account_deployment_data_hash)
        .chain(&calldata_hash);
    if !transaction.proof_facts().0.is_empty() {
        let proof_facts_hash =
            HashChain::new().chain_iter(transaction.proof_facts().0.iter()).get_poseidon_hash();
        hash_chain = hash_chain.chain(&proof_facts_hash);
    }
    Ok(TransactionHash(hash_chain.get_poseidon_hash()))
