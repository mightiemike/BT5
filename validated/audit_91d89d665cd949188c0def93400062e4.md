Looking at the full code path for this question, I need to trace:
1. The protobuf conversion behavior for `compiled_class_hash=None`
2. What validation the p2p sync client applies to received state diffs
3. Whether the `state_diff_commitment` stored in the header is verified against the received state diff

### Title
P2P Sync Client Accepts Cairo-1 Class Downgraded to Cairo-0 via `compiled_class_hash=None` Without State Diff Commitment Verification — (`crates/apollo_protobuf/src/converters/state_diff.rs`)

---

### Summary

An attacker-controlled p2p peer can send a `protobuf::DeclaredClass` message with `compiled_class_hash=None` for a class that is genuinely Cairo-1. The p2p sync client accepts this message, stores the class in `deprecated_declared_classes` (Cairo-0 path) instead of `class_hash_to_compiled_class_hash` (Cairo-1 path), and never verifies the received state diff against the `state_diff_commitment` stored in the block header. The syncing node permanently treats the class as Cairo-0, causing wrong or absent compiled class code to be selected for execution.

---

### Finding Description

**Step 1 — Conversion behavior is intentional but exploitable.**

The `TryFrom<protobuf::StateDiffsResponse> for DataOrFin<StateDiffChunk>` conversion routes on `compiled_class_hash`:

```
None  → StateDiffChunk::DeprecatedDeclaredClass  (Cairo-0)
Some  → StateDiffChunk::DeclaredClass            (Cairo-1)
``` [1](#0-0) 

The same logic is present in `TryFrom<protobuf::DeclaredClass> for ThinStateDiff`: [2](#0-1) 

Both paths are by-design per the p2p spec comment ("if compiled_class_hash is missing, the declared class is a cairo-0 class"), but there is no guard that prevents an attacker from sending `None` for a class that is actually Cairo-1.

**Step 2 — The length check does not distinguish Cairo-0 from Cairo-1.**

`StateDiffChunk::DeclaredClass` and `StateDiffChunk::DeprecatedDeclaredClass` both return `len() == 1`: [3](#0-2) 

So if the real block has a Cairo-1 class declaration (contributing 1 to `state_diff_length`), an attacker can substitute it with a `DeprecatedDeclaredClass` (also contributing 1), and the length check in `parse_data_for_block` passes: [4](#0-3) 

**Step 3 — The state diff commitment is never verified against the received state diff.**

The block header carries a `state_diff_commitment` (a Poseidon hash over the full state diff). The p2p sync client stores the header first, then syncs the state diff. However, `parse_data_for_block` performs only three checks: length match, no conflicting parts, and no duplicate deprecated classes. It never calls `calculate_state_diff_hash` and never compares the result against the stored `state_diff_commitment`. [5](#0-4) 

The `write_to_storage` path calls `append_state_diff` directly with no commitment check: [6](#0-5) 

`append_state_diff` in storage also performs no commitment verification: [7](#0-6) 

The `calculate_state_diff_hash` / `verify_state_diff_hash` guard exists only in the committer (sequencer/batcher path), not in the p2p sync client path: [8](#0-7) 

**Step 4 — Downstream effect on execution.**

`append_state_diff` writes class X into `deprecated_declared_classes_block` (Cairo-0 table) instead of `declared_classes_block` + `compiled_class_hash` (Cairo-1 tables): [9](#0-8) 

`get_compiled_class` in the RPC layer first checks the Cairo-1 table (`get_class_definition_block_number`). Since class X was never inserted there, it falls through to the Cairo-0 table. If no Cairo-0 body was synced for X (because the real class body is Sierra), the call returns `CLASS_HASH_NOT_FOUND`. If a Cairo-0 body was somehow present, the wrong code is returned. [10](#0-9) 

---

### Impact Explanation

A syncing node that receives a manipulated `DeclaredClass` message from a malicious peer will permanently store a Cairo-1 class as Cairo-0. Any subsequent execution, fee estimation, simulation, or RPC call that requires the compiled class for that hash will either fail with `UndeclaredClassHash` / `CLASS_HASH_NOT_FOUND`, or (if a stale Cairo-0 body exists) execute the wrong contract code. This directly matches the Critical impact category: **wrong compiled class / contract code selected for execution**.

---

### Likelihood Explanation

The p2p sync client connects to arbitrary peers on the network. Any peer that responds to a `StateDiffsRequest` can craft this message. No special privilege is required. The attack is deterministic: one manipulated message per block is sufficient to corrupt the class type for that block permanently (until a full re-sync). The only prerequisite is that the attacker's peer is selected for the targeted block's state diff session.

---

### Recommendation

In `parse_data_for_block` (or immediately before `write_to_storage`), compute `calculate_state_diff_hash(&result)` and compare it against the `state_diff_commitment` stored in the block header. Reject the peer (report bad peer) if the commitment does not match. This is the same guard already present in the committer's `commit_or_load` and should be applied symmetrically in the p2p sync client. [11](#0-10)

                ..Default::default()
            }),
            None => Ok(ThinStateDiff {
                deprecated_declared_classes: vec![class_hash],

impl<T: StorageTransaction<Mode = RW>> StateStorageWriter for T {
    #[sequencer_latency_histogram(STORAGE_APPEND_THIN_STATE_DIFF_LATENCY, false)]
    fn append_state_diff(
        self,
        block_number: BlockNumber,
        thin_state_diff: ThinStateDiff,
    ) -> StorageResult<Self> {
        let tables = self.tables();
        let inner_txn = self.txn();
        let file_offset_table = inner_txn.open_table(&tables.file_offsets)?;
        let markers_table = self.open_table(&tables.markers)?;
        let state_diffs_table = self.open_table(&tables.state_diffs)?;
        let nonces_table = self.open_table(&tables.nonces)?;
        let deployed_contracts_table = self.open_table(&tables.deployed_contracts)?;
        let storage_table = self.open_table(&tables.contract_storage)?;
        let declared_classes_block_table = self.open_table(&tables.declared_classes_block)?;
        let deprecated_declared_classes_block_table =
            self.open_table(&tables.deprecated_declared_classes_block)?;
        let compiled_class_hash_table = self.open_table(&tables.compiled_class_hash)?;

        // Write state.
        write_deployed_contracts(
            &thin_state_diff.deployed_contracts,
            inner_txn,
            block_number,
            &deployed_contracts_table,
            &nonces_table,
        )?;
        write_storage_diffs(
            &thin_state_diff.storage_diffs,
            inner_txn,
            block_number,
            &storage_table,
        )?;
        // Must be called after write_deployed_contracts since the nonces are updated there.
        write_nonces(&thin_state_diff.nonces, inner_txn, block_number, &nonces_table)?;

        for (class_hash, _) in &thin_state_diff.class_hash_to_compiled_class_hash {
            let not_declared = declared_classes_block_table.get(inner_txn, class_hash)?.is_none();
            if not_declared {
                declared_classes_block_table.insert(inner_txn, class_hash, &block_number)?;
            }
        }

        write_compiled_class_hashes(
            &thin_state_diff.class_hash_to_compiled_class_hash,
            inner_txn,
            block_number,
            &compiled_class_hash_table,
        )?;

        for class_hash in thin_state_diff.deprecated_declared_classes.iter() {
            // Cairo0 classes can be declared in different blocks. The first block to declare the
            // class is recorded here.
            if deprecated_declared_classes_block_table.get(inner_txn, class_hash)?.is_none() {
                deprecated_declared_classes_block_table.insert(
                    inner_txn,
                    class_hash,
                    &block_number,
                )?;
            }
        }

        // Write state diff.
        let location = self.file_handlers().append_state_diff(&thin_state_diff);
        state_diffs_table.append(inner_txn, &block_number, &location)?;
        file_offset_table.upsert(inner_txn, &OffsetKind::ThinStateDiff, &location.next_offset())?;

        update_marker_to_next_block(inner_txn, &markers_table, MarkerKind::State, block_number)?;

        advance_compiled_class_marker_over_blocks_without_classes(
            inner_txn,
            &markers_table,
            &state_diffs_table,
            self.file_handlers(),
        )?;

        Ok(self)
    }

pub fn calculate_state_diff_hash(state_diff: &ThinStateDiff) -> StateDiffCommitment {
    let mut hash_chain = HashChain::new();
    hash_chain = hash_chain.chain(&STARKNET_STATE_DIFF0);
    hash_chain = chain_deployed_contracts(&state_diff.deployed_contracts, hash_chain);
    hash_chain = chain_declared_classes(&state_diff.class_hash_to_compiled_class_hash, hash_chain);
    hash_chain =
        chain_deprecated_declared_classes(&state_diff.deprecated_declared_classes, hash_chain);
    hash_chain = hash_chain.chain(&Felt::ONE) // placeholder.
        .chain(&Felt::ZERO); // placeholder.
    hash_chain = chain_storage_diffs(&state_diff.storage_diffs, hash_chain);
    hash_chain = chain_nonces(&state_diff.nonces, hash_chain);
    StateDiffCommitment(PoseidonHash(hash_chain.get_poseidon_hash()))
}
