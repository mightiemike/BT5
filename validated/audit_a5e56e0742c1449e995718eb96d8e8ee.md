### Title
Division-by-Zero in `socializeSubaccount` When `openInterest == 0` Permanently Locks Bankrupt Subaccount — (`core/contracts/PerpEngine.sol`)

---

### Summary

`PerpEngine.socializeSubaccount` unconditionally divides by `state.openInterest` when a subaccount's residual `vQuoteBalance` is negative after insurance. `MathSD21x18.div` explicitly reverts with `"DBZ"` when the divisor is zero. Because `openInterest` can legitimately be zero while a subaccount still carries a negative `vQuoteBalance` (all positions closed, bad debt remains), every call to `_finalizeSubaccount` for such a subaccount reverts, permanently preventing liquidation finalization and bad-debt socialization.

---

### Finding Description

**Root cause — `PerpEngine.sol:164-168`:**

```solidity
if (balance.vQuoteBalance < 0) {
    // socialize across all other participants
    int128 fundingPerShare = -balance.vQuoteBalance.div(
        state.openInterest          // ← reverts "DBZ" when == 0
    );
``` [1](#0-0) 

`MathSD21x18.div` contains an explicit guard:

```solidity
require(y != 0, ERR_DIV_BY_ZERO);
``` [2](#0-1) 

**How `openInterest` reaches zero while `vQuoteBalance` stays negative:**

`_updateBalance` in `PerpEngineState.sol` maintains `openInterest` as the sum of `abs(amount)` across all positions. When a position is closed (`amount → 0`), its contribution is removed:

```solidity
state.openInterest -= balance.amount.abs();   // pre-update: remove old
// ...
} else {
    state.openInterest -= balance.amount;      // post-update: amount==0 → no addition
}
``` [3](#0-2) 

`vQuoteBalance` is independent of `amount`; it accumulates realized PnL and funding. A subaccount that opened a long, closed it at a loss, and accumulated negative funding retains `vQuoteBalance < 0` even after `amount == 0`. If no other participant holds an open position in that product, `openInterest == 0`.

**Call path to the revert:**

`_finalizeSubaccount` (triggered when `txn.productId == type(uint32).max`) first **requires** `balance.amount == 0` for every perp product:

```solidity
require(balance.amount == 0, ERR_NOT_FINALIZABLE_SUBACCOUNT);
``` [4](#0-3) 

It then calls `perpEngine.socializeSubaccount`:

```solidity
v.insurance = perpEngine.socializeSubaccount(
    txn.liquidatee,
    v.insurance
);
``` [5](#0-4) 

Inside `socializeSubaccount`, `getStateAndBalance` is called, which internally calls `_updateBalance(state, balance, 0, 0)`. Because `balance.amount == 0`, the call subtracts and adds zero — `state.openInterest` is left exactly as stored. If the stored value is `0`, the subsequent division reverts.

---

### Impact Explanation

- `liquidateSubaccountImpl` reverts on every attempt to finalize the bankrupt subaccount.
- The subaccount is permanently stuck: it cannot be finalized, bad debt is never socialized, and the insurance fund cannot absorb the loss.
- Protocol-wide solvency accounting is broken for that product: other participants' `cumulativeFundingLongX18`/`cumulativeFundingShortX18` are never adjusted to absorb the bad debt.
- The impact matches the Critical scope: bad debt bypass and protocol solvency breakage.

---

### Likelihood Explanation

The precondition is reachable through normal trading activity:

1. A subaccount is the last (or only) participant with an open position in a perp product.
2. It closes the position at a loss, leaving `amount == 0` and `vQuoteBalance < 0`.
3. `openInterest` drops to zero.
4. The subaccount's spot quote balance also goes negative (making it unhealthy/liquidatable).
5. Insurance is insufficient to cover the full negative `vQuoteBalance`.
6. Any liquidator calling `liquidateSubaccountImpl` with `productId == type(uint32).max` triggers the revert.

This is not a contrived edge case; it is a natural end-state for a market with low open interest.

---

### Recommendation

Guard the socialization branch against zero `openInterest`. When `openInterest == 0` there are no other participants to socialize against; the bad debt should be absorbed entirely by the insurance fund (or written off):

```solidity
if (balance.vQuoteBalance < 0) {
    if (state.openInterest == 0) {
        // No open positions to socialize against; write off residual bad debt
        balance.vQuoteBalance = 0;
    } else {
        int128 fundingPerShare = -balance.vQuoteBalance.div(state.openInterest);
        state.cumulativeFundingLongX18 += fundingPerShare;
        state.cumulativeFundingShortX18 -= fundingPerShare;
        balance.vQuoteBalance = 0;
    }
}
```

```
