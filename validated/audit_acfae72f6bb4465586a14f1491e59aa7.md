Audit Report

## Title
`SwapAllowlistExtension` Gates the Router Address Instead of the Economic Actor, Allowing Any User to Bypass the Swap Allowlist — (`metric-periphery/contracts/extensions/SwapAllowlistExtension.sol`)

## Summary

`MetricOmmPool.swap()` passes `msg.sender` as the `sender` argument to every configured extension. When swaps are routed through `MetricOmmSimpleRouter`, the pool's `msg.sender` is the router contract, not the originating user. If the pool admin allowlists the router address to enable router-mediated swaps, any unprivileged user can bypass the allowlist entirely by calling the public router, since the extension sees `sender = router` and passes the check regardless of who initiated the call.

## Finding Description

**Pool → Extension identity binding**

`MetricOmmPool.swap()` passes `msg.sender` as the first argument to `_beforeSwap()`: [1](#0-0) 

`SwapAllowlistExtension.beforeSwap()` gates on that `sender` value: [2](#0-1) 

**Router call path**

`MetricOmmSimpleRouter.exactInputSingle()` calls `pool.swap()` directly, making the router the `msg.sender` the pool observes. The original caller's address is stored in transient storage only for the payment callback via `_setNextCallbackContext`, and is never forwarded to the pool or to any extension: [3](#0-2) 

The same pattern applies to `exactInput`, `exactOutputSingle`, and `exactOutput`. [4](#0-3) 

**The bypass**

When a pool admin allowlists the router (`allowedSwapper[pool][router] = true`) to enable router-mediated swaps, `allowedSwapper[pool][router]` evaluates to `true` for every call arriving through the router, regardless of who the original `msg.sender` of the router call was. Any unprivileged user calling `exactInputSingle(...)` passes the extension check because the extension sees `sender = router`. Conversely, if the admin does not allowlist the router, individually allowlisted users cannot use the router at all — the extension sees `sender = router` and reverts with `NotAllowedToSwap`. The extension cannot simultaneously allow specific users through the router and block others. [5](#0-4) 

## Impact Explanation

`SwapAllowlistExtension` is the sole on-chain mechanism for restricting swap access on a pool. Bypassing it allows unauthorized users to execute swaps against oracle-anchored prices on pools intended to be restricted (e.g., institutional-only pools, pools with favorable oracle pricing for specific counterparties). An unauthorized user can drain LP value by swapping at oracle prices favorable relative to the open market, causing direct loss of LP principal. This constitutes a broken core pool functionality causing loss of funds and an admin-boundary break where an unprivileged path bypasses a required access control check. [6](#0-5) 

## Likelihood Explanation

The trigger requires only that the pool admin allowlists the router — a natural and expected configuration step for any pool intended to be accessible via the standard periphery. The router is a public, permissionless contract. No privileged access, special tokens, or malicious setup is required beyond a standard user calling `exactInputSingle` or `exactInput`. The condition is highly likely to occur in production deployments. [7](#0-6) 

## Recommendation

The extension must gate on the economic actor, not the intermediary. Two complementary fixes:

1. **Router-level**: `MetricOmmSimpleRouter` should forward the original `msg.sender` inside `extensionData` so extensions can verify the real user.
2. **Extension-level**: `SwapAllowlistExtension.beforeSwap()` should decode a user address from `extensionData` when `sender` is a known router, or the pool interface should expose a separate `originalSender` argument populated from a trusted source.

The simplest safe default is to treat `sender` as the gated identity and document that the router must never be allowlisted as a blanket entry — but this breaks router usability for allowlisted pools entirely. A proper fix requires propagating the real user identity through the hook call. [2](#0-1) 

## Proof of Concept

1. Deploy a pool with `SwapAllowlistExtension` configured in `BEFORE_SWAP_ORDER`.
2. Pool admin calls `setAllowedToSwap(pool, router, true)` to enable router-mediated swaps.
3. `userA` (not individually allowlisted) calls:
   ```solidity
   router.exactInputSingle(ExactInputSingleParams({
       pool: restrictedPool,
       tokenIn: token0,
       tokenOut: token1,
       zeroForOne: true,
       amountIn: 1_000e18,
       amountOutMinimum: 0,
       recipient: userA,
       deadline: block.timestamp + 1,
       priceLimitX64: 0,
       extensionData: ""
   }));
   ```
4. The pool calls `_beforeSwap(msg.sender=router, ...)`.
5. `SwapAllowlistExtension.beforeSwap` checks `allowedSwapper[pool][router]` → `true` → passes.
6. `userA` receives output tokens. The allowlist is bypassed. [8](#0-7)

  function exactInput(ExactInputParams calldata params) external payable returns (uint256 amountOut) {
    _checkDeadline(params.deadline);
    _validatePath(params.tokens, params.pools, params.extensionDatas);

    uint256 last = params.pools.length - 1;
    int128 amount = MetricOmmSwapInputs.asAmountSpecifiedIn(params.amountIn);

    for (uint256 i = 0; i <= last; i++) {
      address pool = params.pools[i];
      bool zeroForOne = MetricOmmSwapPath.resolveZeroForOneBitmap(params.zeroForOneBitMap, i);

      _setNextCallbackContext(pool, CALLBACK_MODE_JUST_PAY, i == 0 ? msg.sender : address(this), params.tokens[i]);
      (int128 amount0Delta, int128 amount1Delta) = IMetricOmmPoolActions(pool)
        .swap(
          i == last ? params.recipient : address(this),
          zeroForOne,
          amount,
          MetricOmmSwapPath.openLimit(zeroForOne),
          "",
          params.extensionDatas[i]
        );

      int128 amountInActual = MetricOmmSwapResults.extractAmountIn(zeroForOne, amount0Delta, amount1Delta);
      if (amountInActual < amount) revert InvalidInputAmountAtHop(uint8(i), amountInActual, amount);

      amount = MetricOmmSwapResults.extractAmountOut(zeroForOne, amount0Delta, amount1Delta);

    if (amountOut < params.amountOutMinimum) revert InsufficientOutput(amountOut, params.amountOutMinimum);

    _clearExpectedCallbackPool();
  }
