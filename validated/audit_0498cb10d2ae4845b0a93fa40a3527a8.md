### Title
Exiting Tiered Positions Retain Full Governance Voting Power After Exit Is Triggered — (`x/tieredrewards/keeper/gov_tally.go`, `x/tieredrewards/keeper/voting_power.go`)

---

### Summary

The custom governance tally function grants full voting power to tiered-rewards positions that have already had exit triggered (`ExitUnlockAt` set). Neither `positionVotingPower` nor `GetPositionStatesByOwner` checks `HasTriggeredExit()`. A position owner can therefore vote in a governance proposal using the full delegation shares of an exiting position, then—once the exit lock duration elapses—call `ExitTierWithDelegation` to transfer that stake back to themselves, having already used it to influence the vote.

---

### Finding Description

**`positionVotingPower`** (`x/tieredrewards/keeper/voting_power.go`) has exactly one guard before computing voting power:

```go
if !pos.IsDelegated() {
    return math.LegacyZeroDec()
}
``` [1](#0-0) 

There is no check for `pos.HasTriggeredExit()`. A position with `ExitTriggeredAt` set is still delegated (the staking delegation is not removed until `ExitTierWithDelegation` or `TierUndelegate` is called), so it passes this guard and returns full voting power.

**`GetPositionStatesByOwner`** (`x/tieredrewards/keeper/position_state.go`), which is the entry point used by the tally function, returns every position for the owner with no exit-status filter:

```go
func (k Keeper) GetPositionStatesByOwner(ctx context.Context, owner sdk.AccAddress) ([]types.PositionState, error) {
    ids, err := k.getPositionsIdsByOwner(ctx, owner)
    ...
    states = append(states, state)   // no HasTriggeredExit() guard
``` [2](#0-1) 

**`NewCustomTallyTierVotesFn`** (`x/tieredrewards/keeper/gov_tally.go`) calls both without any exit check:

```go
positions, err := tierKeeper.GetPositionStatesByOwner(ctx, voter)
...
for _, pos := range positions {
    posPower := positionVotingPower(pos, validators)
``` [3](#0-2) 

**`validateExitTierWithDelegation`** (`x/tieredrewards/keeper/msg_validate.go`) enforces that `CompletedExitLockDuration` must be true before the stake can be transferred out:

```go
if !pos.CompletedExitLockDuration(sdkCtx.BlockTime()) {
    return types.ErrExitLockDurationNotReached
}
``` [4](#0-3) 

This means the delegation is still live in staking throughout the entire exit lock window, so the position's shares are counted in governance tally for that whole period.

---

### Impact Explanation

The position's delegator account is a module-derived address separate from the owner's own wallet. The owner receives **additional** governance voting power from tiered positions on top of their normal delegations. After triggering exit, that additional power is not removed from governance tallying. The owner can:

1. Call `TriggerExitFromTier` → `ExitUnlockAt` is set; delegation remains active.
2. Cast a governance vote; `positionVotingPower` returns the full share-weighted power.
3. Wait for the exit lock duration to elapse.
4. Call `ExitTierWithDelegation` to transfer the delegation back to themselves.

The governance vote was cast with stake the owner has now fully reclaimed. If the exit lock duration is shorter than the governance voting period, the stake can be withdrawn while the vote is still open. Even if the exit lock duration is longer, the owner has voted with stake they were already committed to exit—stake that no longer represents a long-term alignment with the chain.

---

### Likelihood Explanation

The call sequence is entirely user-controlled and requires no privileged access. `TriggerExitFromTier` is a standard `MsgServer` transaction available to any position owner. The only precondition is owning a tiered position with a non-trivial delegation. Governance proposals are regularly active on Cosmos chains, and exit lock durations are configurable per tier and may be shorter than the governance voting period.

---

### Recommendation

Add an exit-status guard in `positionVotingPower`:

```go
func positionVotingPower(pos types.PositionState, bondedVals map[string]v1.ValidatorGovInfo) math.LegacyDec {
    if !pos.IsDelegated() || pos.HasTriggeredExit() {
        return math.LegacyZeroDec()
    }
    ...
}
``` [5](#0-4) 

Alternatively, filter in `GetPositionStatesByOwner` when called from the governance path, or add the check directly in the tally loop in `gov_tally.go`.

---

### Proof of Concept

```
1. Alice holds a tiered position with 1,000,000 bonded shares.
2. Alice calls TriggerExitFromTier(positionId).
   → pos.ExitTriggeredAt = blockTime; pos.ExitUnlockAt = blockTime + exitDuration
   → Delegation is still active in staking.
3. A governance proposal is submitted (or was already active).
4. Alice calls MsgVote on the proposal.
   → gov_tally calls GetPositionStatesByOwner(alice) → returns the exiting position.
   → positionVotingPower checks IsDelegated() == true → returns full 1,000,000-share power.
   → Alice's vote is counted with full weight.
5. exitDuration elapses.
6. Alice calls ExitTierWithDelegation(positionId, fullAmount).
   → validateExitTierWithDelegation: HasTriggeredExit() ✓, CompletedExitLockDuration() ✓
   → Delegation transferred back to Alice's wallet.
7. Governance tally executes: Alice's 1,000,000-share vote is already recorded.
   Alice has withdrawn the stake that backed that vote.
```

func positionVotingPower(
	pos types.PositionState,
	bondedVals map[string]v1.ValidatorGovInfo,
) math.LegacyDec {
	if !pos.IsDelegated() {
		return math.LegacyZeroDec()
	}
	val, ok := bondedVals[pos.Delegation.ValidatorAddress]
	if !ok || val.DelegatorShares.IsZero() {
		return math.LegacyZeroDec()
	}
	return pos.Delegation.Shares.MulInt(val.BondedTokens).Quo(val.DelegatorShares)
}
```

func (k Keeper) GetPositionStatesByOwner(ctx context.Context, owner sdk.AccAddress) ([]types.PositionState, error) {
	ids, err := k.getPositionsIdsByOwner(ctx, owner)
	if err != nil {
		return nil, err
	}
	states := make([]types.PositionState, 0, len(ids))
	for _, id := range ids {
		state, err := k.getPositionState(ctx, id)
		if errors.Is(err, types.ErrPositionNotFound) {
			continue
		}
		if err != nil {
			return nil, err
		}
		states = append(states, state)
	}
	return states, nil
}
