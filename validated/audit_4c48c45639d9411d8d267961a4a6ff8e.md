Audit Report

## Title
`SwapAllowlistExtension` checks router address instead of end-user, enabling full allowlist bypass via `MetricOmmSimpleRouter` — (`metric-periphery/contracts/extensions/SwapAllowlistExtension.sol`)

## Summary

`MetricOmmPool.swap` passes its own `msg.sender` (the immediate caller) as the `sender` argument to every `beforeSwap` extension hook. When users route through `MetricOmmSimpleRouter`, the pool's `msg.sender` is the router contract. `SwapAllowlistExtension.beforeSwap` then evaluates `allowedSwapper[pool][router]` instead of `allowedSwapper[pool][actual_user]`. Because the pool admin must allowlist the router for any allowlisted user to use it, allowlisting the router grants every caller of the router — including non-allowlisted ones — unrestricted swap access.

## Finding Description

`MetricOmmPool.swap` passes `msg.sender` as the first argument to `_beforeSwap`: [1](#0-0) 

`SwapAllowlistExtension.beforeSwap` uses that argument as `sender` and checks `allowedSwapper[msg.sender][sender]`, where `msg.sender` is the pool and `sender` is whoever called `pool.swap()`: [2](#0-1) 

When a user calls `MetricOmmSimpleRouter.exactInputSingle`, the router calls `pool.swap()` directly — making the pool's `msg.sender` the router address, not the end user: [3](#0-2) 

The same applies to `exactInput` (all hops): [4](#0-3) 

And to `exactOutput` recursive callback hops: [5](#0-4) 

The allowlist storage is keyed `allowedSwapper[pool][swapper]`, confirming the design intent is to gate individual users, not routing intermediaries: [6](#0-5) 

The pool admin faces an impossible choice: not allowlisting the router means allowlisted users cannot use the primary swap interface; allowlisting the router means every user — including non-allowlisted ones — passes the check because the extension only ever sees the router's address.

## Impact Explanation

A pool configured with `SwapAllowlistExtension` (e.g., KYC-gated, institutional-only, or rate-limited) has its access control completely defeated for any user routing through `MetricOmmSimpleRouter`. Non-allowlisted users gain unrestricted swap access to LP funds that were deposited under the assumption of a functioning allowlist. This is a direct, concrete loss of the access-control guarantee LPs relied on — a broken core pool functionality causing potential loss of funds (LP funds exposed to unrestricted counterparties).

## Likelihood Explanation

High. `MetricOmmSimpleRouter` is the primary user-facing swap interface. No special privileges, flash loans, or contract deployment are required. Any non-allowlisted user simply calls `exactInputSingle`, `exactInput`, or `exactOutput` on the router instead of calling `pool.swap()` directly. The bypass is repeatable and unconditional once the router is allowlisted.

## Recommendation

**Short term:** The pool should pass the original end-user's address as `sender` to extension hooks rather than `msg.sender`. The router already tracks the original caller in transient storage via `_getPayer()` / `_setNextCallbackContext`; it should forward that address through `extensionData` or a dedicated field so the extension can check the real swapper identity.

**Long term:** Redesign the `beforeSwap` hook signature in `IMetricOmmExtensions` to include a separate `originator` field populated from a trusted source (e.g., a signed context or a dedicated router forwarding mechanism), so extensions can always distinguish the economic actor from the routing intermediary.

## Proof of Concept

```
Setup:
  pool configured with SwapAllowlistExtension as beforeSwap hook
  swapExtension.setAllowedToSwap(pool, alice, true)   // Alice is KYC'd
  swapExtension.setAllowedToSwap(pool, router, true)  // admin must allowlist router for Alice to use it

Attack (Charlie, not allowlisted):
  1. Charlie calls router.exactInputSingle({pool: pool, recipient: charlie, ...})
  2. Router calls pool.swap(charlie, zeroForOne, amount, ...) — pool's msg.sender = router
  3. Pool calls _beforeSwap(sender=router, ...)
  4. Extension evaluates: allowedSwapper[pool][router] == true → passes
  5. Charlie's swap executes; allowlist is bypassed

Foundry test outline:
  - Deploy pool with SwapAllowlistExtension
  - setAllowedToSwap(pool, alice, true)
  - setAllowedToSwap(pool, router, true)
  - vm.prank(charlie) → router.exactInputSingle(...)
  - Assert swap succeeds (no NotAllowedToSwap revert)
  - Assert charlie received output tokens
```
