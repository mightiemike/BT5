### Title
Unprivileged Caller Can Invoke `run_mainchain_gc` Directly, Bypassing `#[trusted_relayer]` Guard and Prematurely Deleting Mainchain Headers — (File: `contract/src/lib.rs`)

---

### Summary

`submit_blocks` is protected by both `#[pause]` and `#[trusted_relayer]`, restricting block submission to registered, active relayers. The companion state-mutating function `run_mainchain_gc` is exposed as a public NEAR method in the same `#[near]` impl block but carries only `#[pause(except(roles(Role::UnrestrictedRunGC)))]` — no `#[trusted_relayer]` guard. Any unprivileged NEAR account can call `run_mainchain_gc` directly with an arbitrary `batch_size`, prematurely deleting mainchain headers and corrupting `mainchain_initial_blockhash`, which breaks transaction-inclusion proofs for all evicted blocks.

---

### Finding Description

`submit_blocks` is decorated with `#[pause]` and `#[trusted_relayer]`: [1](#0-0) 

The `#[trusted_relayer]` macro (configured at the impl-block level on lines 120–124) enforces that the caller must be a registered, active relayer; accounts without that role receive `"Relayer is not active"` and the call is rejected.

`run_mainchain_gc` is a separate `pub fn` in the same `#[near]` impl block: [2](#0-1) 

It carries only `#[pause(except(roles(Role::UnrestrictedRunGC)))]`. When the contract is **not paused** (the normal operating state), this attribute imposes zero caller restriction — any NEAR account may invoke it. The function then:

1. Computes how many headers exceed `gc_threshold`.
2. Deletes up to `batch_size` of the oldest mainchain headers from `headers_pool` and `mainchain_height_to_header`.
3. Advances `mainchain_initial_blockhash` to the new oldest surviving block. [3](#0-2) 

An attacker passes `batch_size = u64::MAX`; the function caps removal at `total_amount_to_remove = amount_of_headers_we_store - gc_threshold`, so it evicts every header the protocol is willing to drop in one call.

---

### Impact Explanation

`verify_transaction_inclusion` and `verify_transaction_inclusion_v2` look up the target block in `mainchain_header_to_height`: [4](#0-3) 

Once `run_mainchain_gc` removes a header, that lookup panics with `"block does not belong to the current main chain"`. Any consumer contract or off-chain service that relied on proving inclusion of a transaction in an evicted block receives a permanent failure. `mainchain_initial_blockhash` is also permanently advanced, so the deleted window can never be restored without a full re-sync from genesis.

**Impact: High** — canonical header state is irreversibly corrupted; transaction-inclusion proofs for evicted blocks are permanently broken.

---

### Likelihood Explanation

The entry point is a plain public NEAR method callable by any account with no deposit, no role, and no staking requirement. The attacker needs only to know the contract address and the method name. No privileged access, leaked keys, or social engineering is required.

**Likelihood: High**

---

### Recommendation

Add `#[trusted_relayer]` to `run_mainchain_gc` so that only registered, active relayers (or accounts holding `Role::DAO` / `Role::UnrestrictedSubmitBlocks` as bypass roles) can trigger garbage collection:

```rust
#[pause(except(roles(Role::UnrestrictedRunGC)))]
#[trusted_relayer]   // <-- add this
pub fn run_mainchain_gc(&mut self, batch_size: u64) {
```

This mirrors the protection already applied to `submit_blocks` and closes the bypass path.

---

### Proof of Concept

1. Deploy the contract and initialize it with enough blocks so that `amount_of_headers_we_store > gc_threshold`.
2. From any unprivileged NEAR account (no role granted, no staking):
   ```
   near call <contract_id> run_mainchain_gc '{"batch_size": 18446744073709551615}' --accountId attacker.near
   ```
3. The call succeeds. All headers beyond `gc_threshold` are deleted and `mainchain_initial_blockhash` is advanced.
4. Any subsequent call to `verify_transaction_inclusion` or `verify_transaction_inclusion_v2` for a transaction in an evicted block panics with `"block does not belong to the current main chain"`.
5. Contrast with `submit_blocks` called from the same unprivileged account — it fails with `"Relayer is not active"`, confirming the asymmetric access control. [5](#0-4) [1](#0-0) [6](#0-5)

```rust
    #[pause(except(roles(Role::UnrestrictedRunGC)))]
    pub fn run_mainchain_gc(&mut self, batch_size: u64) {
        let initial_blockheader = self
            .headers_pool
            .get(&self.mainchain_initial_blockhash)
            .unwrap_or_else(|| env::panic_str(ERR_KEY_NOT_EXIST));

        let tip_blockheader = self
            .headers_pool
            .get(&self.mainchain_tip_blockhash)
            .unwrap_or_else(|| env::panic_str(ERR_KEY_NOT_EXIST));

        let amount_of_headers_we_store =
            tip_blockheader.block_height - initial_blockheader.block_height + 1;

        if amount_of_headers_we_store > self.gc_threshold {
            let total_amount_to_remove = amount_of_headers_we_store - self.gc_threshold;
            let selected_amount_to_remove = std::cmp::min(total_amount_to_remove, batch_size);

            let start_removal_height = initial_blockheader.block_height;
            let end_removal_height = initial_blockheader.block_height + selected_amount_to_remove;
            env::log_str(&format!(
                "Num of blocks to remove {selected_amount_to_remove}"
            ));

            for height in start_removal_height..end_removal_height {
                let blockhash = &self
                    .mainchain_height_to_header
                    .get(&height)
                    .unwrap_or_else(|| env::panic_str(ERR_KEY_NOT_EXIST));

                self.remove_block_header(blockhash);
                self.mainchain_height_to_header.remove(&height);
            }

            self.mainchain_initial_blockhash = self
                .mainchain_height_to_header
                .get(&end_removal_height)
                .unwrap_or_else(|| env::panic_str(ERR_KEY_NOT_EXIST));
        }
    }
```
