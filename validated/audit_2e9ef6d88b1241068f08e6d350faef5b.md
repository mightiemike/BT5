### Title
Unclaimed Airdrop Tokens Permanently Locked With No Recovery Function - (File: `core/contracts/Airdrop.sol`)

---

### Summary

`Airdrop.sol` holds tokens on behalf of recipients and requires them to actively call `claim()` with a Merkle proof. The contract contains a hard sanctions gate that permanently blocks sanctioned addresses from claiming. Because the contract exposes **no `withdraw`, `recover`, or `rescue` function**, any tokens allocated to sanctioned (or simply non-claiming) addresses are irreversibly locked in the contract.

---

### Finding Description

The `Airdrop` contract is pre-funded with tokens by the owner and distributes them via a weekly Merkle-root scheme. Users must call `claim()` to pull their allocation. Inside `_verifyProof`, the contract enforces a sanctions check: [1](#0-0) 

If `ISanctionsList(sanctions).isSanctioned(sender)` returns `true`, the call reverts and the tokens remain in the contract. The sanctions list is an external, mutable contract — an address can be added to it at any time, including after a merkle root has already been registered for a week that includes that address's allocation.

A grep over `Airdrop.sol` for any `withdraw`, `recover`, `rescue`, or `sweep` function returns **no matches**. The full public/external interface of the contract is: [2](#0-1) 

Neither `claim` nor `getClaimed` moves tokens out to the owner. There is no owner-callable escape hatch anywhere in the contract.

---

### Impact Explanation

Tokens allocated to a sanctioned address for a given week are permanently locked in `Airdrop.sol`. The owner cannot recover them, redistribute them, or burn them. The total supply of the airdrop token is unaffected (tokens exist), but the locked portion is rendered permanently unusable — an exact structural match to the Unikrn analog where tokens were minted and allocated but could never be moved.

---

### Likelihood Explanation

Sanctions lists (e.g., Chainalysis OFAC) are dynamic. An address can be sanctioned at any point after a merkle root is registered. The window between `registerMerkleRoot` and a user's `claim` call can span days or weeks. Any address added to the sanctions list during that window loses its allocation permanently. This is a realistic, externally-triggered condition requiring no privileged access.

---

### Recommendation

Add an owner-callable token recovery function to `Airdrop.sol`, for example:

```solidity
function recoverUnclaimed(address recipient, uint256 amount) external onlyOwner {
    SafeERC20.safeTransfer(IERC20(token), recipient, amount);
}
```

This mirrors the pattern already used in `DirectDepositV1.sol`, which includes both `withdraw(IIERC20Base token)` and `withdrawNative()` for exactly this purpose. [3](#0-2) 

---

### Proof of Concept

1. Owner calls `registerMerkleRoot(1, root)` where `root` encodes Alice → 1000 tokens.
2. Owner funds `Airdrop.sol` with 1000 tokens.
3. Alice is added to the sanctions list (external, owner-independent action).
4. Alice calls `claim([{week: 1, totalAmount: 1000, proof: [...]}])`.
5. Execution reaches `_verifyProof` → line 53-56 reverts: `"address is sanctioned."` [1](#0-0) 
6. No other function in `Airdrop.sol` can move those 1000 tokens out. [4](#0-3) 
7. Tokens are permanently locked.

```text
pragma solidity ^0.8.0;

import "./interfaces/IAirdrop.sol";
import "./Endpoint.sol";
import "@openzeppelin/contracts/utils/cryptography/MerkleProof.sol";
import "@openzeppelin/contracts-upgradeable/access/OwnableUpgradeable.sol";
import "@openzeppelin/contracts/token/ERC20/IERC20.sol";
import "@openzeppelin/contracts/token/ERC20/utils/SafeERC20.sol";

contract Airdrop is OwnableUpgradeable, IAirdrop {
    address internal token;
    address internal sanctions;
    uint32 internal pastWeeks;

    mapping(uint32 => bytes32) internal merkleRoots;
    mapping(uint32 => mapping(address => uint256)) internal claimed;

    /// @custom:oz-upgrades-unsafe-allow constructor
    constructor() {
        _disableInitializers();
    }

    function initialize(address _token, address _sanctions)
        external
        initializer
    {
        __Ownable_init();
        token = _token;
        sanctions = _sanctions;
    }

    function registerMerkleRoot(uint32 week, bytes32 merkleRoot)
        external
        onlyOwner
    {
        pastWeeks += 1;
        require(week == pastWeeks, "Invalid week provided.");
        merkleRoots[week] = merkleRoot;
    }

    function _verifyProof(
        uint32 week,
        address sender,
        uint256 totalAmount,
        bytes32[] calldata proof
    ) internal {
        require(claimed[week][sender] == 0, "Already claimed.");
        require(
            merkleRoots[week] != bytes32(0),
            "Week hasn't been registered."
        );
        require(
            !ISanctionsList(sanctions).isSanctioned(sender),
            "address is sanctioned."
        );
        bytes32 leaf = keccak256(
            bytes.concat(keccak256(abi.encode(sender, totalAmount)))
        );
        bool isValidLeaf = MerkleProof.verify(proof, merkleRoots[week], leaf);
        require(isValidLeaf, "Invalid proof.");
        claimed[week][sender] = totalAmount;
    }

    function _claim(
        uint32 week,
        uint256 totalAmount,
        bytes32[] calldata proof
    ) internal {
        _verifyProof(week, msg.sender, totalAmount, proof);
        SafeERC20.safeTransfer(IERC20(token), msg.sender, totalAmount);
        emit Claim(msg.sender, week, totalAmount);
    }

    function claim(ClaimProof[] calldata claimProofs) external {
        for (uint32 i = 0; i < claimProofs.length; i++) {
            _claim(
                claimProofs[i].week,
                claimProofs[i].totalAmount,
                claimProofs[i].proof
            );
        }
    }

    function getClaimed(address account)
        external
        view
        returns (uint256[] memory)
    {
        uint256[] memory result = new uint256[](pastWeeks + 1);
        for (uint32 week = 1; week <= pastWeeks; week++) {
            result[week] = claimed[week][account];
        }
        return result;
    }
}
```
