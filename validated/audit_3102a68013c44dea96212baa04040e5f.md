### Title
`fee_proposal_fri` Excluded from Block Hash Preimage Allows Malicious P2P Peer to Inject Arbitrary Fee-Proposal Values That Corrupt the SNIP-35 Gas Price Floor - (`File: crates/starknet_api/src/block.rs`, `crates/starknet_api/src/block_hash/block_hash_calculator.rs`)

---

### Summary

The `fee_proposal_fri` field, introduced in `BlockHeaderWithoutHash` for the SNIP-35 dynamic L2 gas pricing mechanism, is propagated over the P2P network in `SignedBlockHeader` protobuf messages and persisted to storage, but is **explicitly excluded from the canonical block hash preimage**. Because block signatures cover only the block hash, a malicious P2P peer can send a `SignedBlockHeader` with an arbitrary `fee_proposal_fri` value that passes all cryptographic verification. The injected value is then stored and fed into `compute_fee_actual`, whose output is used as a mandatory floor for the minimum L2 gas price in `calculate_next_l2_gas_price_for_fin`. An attacker can therefore silently inflate the effective minimum gas price on a syncing node, causing valid transactions to fail fee checks.

---

### Finding Description

**Root cause — field excluded from hash preimage:**

In `crates/starknet_api/src/block.rs`, `BlockHeaderWithoutHash` carries `fee_proposal_fri`:

```rust

/// Proposer's oracle-derived recommended L2 gas fee. `None` for pre-V0_14_3 blocks.
pub fee_proposal_fri: Option<GasPrice>,
``` [1](#0-0) 

`gas_prices_to_hash`, the only function that feeds gas-related data into `calculate_block_hash`, does not include `fee_proposal_fri` (nor `l2_gas_consumed` / `next_l2_gas_price`, also noted with a TODO):

    block_hash_version: &BlockHashVersion,

``` [2](#0-1) 

**Propagation path — field travels over P2P unauthenticated:**

The protobuf converter in `crates/apollo_protobuf/src/converters/header.rs` faithfully round-trips `fee_proposal_fri` in both directions:

```rust
let fee_proposal_fri = value.fee_proposal_fri.map(|v| GasPrice(u128::from(v)));
``` [3](#0-2) 

The `From<(BlockHeader, Vec<BlockSignature>)>` impl serialises it back out:

```rust
fee_proposal_fri: header
    .block_header_without_hash
    .fee_proposal_fri
    .map(|gp| gp.0.into()),
``` [4](#0-3) 

Block signatures sign the block hash. Because `fee_proposal_fri` is absent from the hash, any peer can set it to any value without invalidating the hash or the signatures.

**Storage — injected value is persisted:**

`StorageBlockHeader` stores `fee_proposal_fri`:

```rust
pub fee_proposal_fri: Option<GasPrice>,
``` [5](#0-4) 

**Downstream effect — corrupts SNIP-35 fee_actual and minimum gas price floor:**

On `set_height_and_round`, the orchestrator calls `initialize_fee_proposals_window`, which reads `fee_proposal_fri` from state-sync blocks. `compute_fee_actual` then computes the median of the window:

```rust
pub fn compute_fee_actual(
    fee_proposals_window: &BTreeMap<BlockNumber, Option<GasPrice>>,
    height: BlockNumber,
    window_size: u64,
) -> Option<GasPrice> { ... }
``` [6](#0-5) 

`fee_actual` is then used as a mandatory floor in `calculate_next_l2_gas_price_for_fin`:

```rust
let effective_min = match fee_actual {
    Some(fa) => GasPrice(max(config_min.0, fa.0)),
    None => config_min,
};
calculate_next_base_gas_price(current_l2_gas_price, l2_gas_used, gas_target, effective_min)
``` [7](#0-6) 

An attacker who injects a large `fee_proposal_fri` across the window raises `fee_actual`, which raises `effective_min`, which raises the minimum L2 gas price the node will accept. Transactions whose `max_price_per_unit` is below the inflated floor are then rejected by the gateway's pre-validation check:

```rust
if resource_bounds.max_price_per_unit < actual_gas_price.get() {
    insufficiencies_resource.push(
        ResourceBoundsError::MaxGasPriceTooLow { ... }
    );
}
``` [8](#0-7) 

---

### Impact Explanation

A syncing node that accepts `SignedBlockHeader` messages from a malicious P2P peer will store attacker-controlled `fee_proposal_fri` values. After `window_size` blocks the entire SNIP-35 window is poisoned, `fee_actual` is set to the attacker's chosen value, and the node's minimum L2 gas price floor is permanently elevated. Valid transactions are rejected as `MaxGasPriceTooLow`, making the node's gateway/mempool non-functional for normal users. This matches: **High — Mempool/gateway/RPC admission rejects valid transactions before sequencing**, and **Critical — Incorrect fee/gas accounting with economic impact**.

---

### Likelihood Explanation

Any peer on the P2P network can send `SignedBlockHeader` messages. The field is optional and defaults to `None` for pre-V0_14_3 blocks, so the attack is only effective on nodes running V0_14_3+. No privileged access is required; a single malicious peer that is accepted as a sync source is sufficient.

---

### Recommendation

Include `fee_proposal_fri` (and `l2_gas_consumed`, `next_l2_gas_price`) in the block hash preimage, gated on the appropriate `BlockHashVersion`, so that the field is covered by the canonical hash and therefore by block signatures. Until the hash is updated, syncing nodes should refuse to use `fee_proposal_fri` values received over P2P for any computation that affects admission or gas pricing, or should cross-validate them against a trusted source.

---

### Proof of Concept

1. Attacker runs a P2P node and connects to a victim syncing node.
2. For each `SignedBlockHeader` the attacker relays, it sets `fee_proposal_fri` to `u128::MAX` (or any large value) while keeping the block hash and signatures unchanged (both remain valid because `fee_proposal_fri` is not in the hash).
3. After `fee_proposal_window_size` blocks (default from `VersionedConstants`), the victim's `fee_proposals_window` contains only attacker-supplied values.
4. `compute_fee_actual` returns `u128::MAX / 2` (median of all-max window).
5. `calculate_next_l2_gas_price_for_fin` sets `effective_min = u128::MAX / 2`.
6. `calculate_next_base_gas_price` returns a price near `u128::MAX`.
7. Any transaction with `max_price_per_unit < u128::MAX` fails `MaxGasPriceTooLow` in the gateway pre-validation, effectively DoS-ing the node's transaction admission.

```rust
pub fn compute_fee_actual(
    fee_proposals_window: &BTreeMap<BlockNumber, Option<GasPrice>>,
    height: BlockNumber,
    window_size: u64,
) -> Option<GasPrice> {
    let Some(start) = height.0.checked_sub(window_size) else {
        warn!(
            "Cannot compute fee_actual for height {height}: height is below window_size \
             ({window_size})"
        );
        return None;
    };
    let window_size_usize = usize::try_from(window_size).expect("window_size fits in usize");
    let mut window = Vec::with_capacity(window_size_usize);
    for source_height in (start..height.0).map(BlockNumber) {
        match fee_proposals_window.get(&source_height) {
            Some(Some(price)) => window.push(*price),
            Some(None) | None => {
                warn!(
                    "Cannot compute fee_actual for height {height}: fee_proposals_window has no \
                     recorded fee_proposal for height {source_height}"
                );
                return None;
            }
        }
    }
    window.sort();
    let mid = window_size_usize / 2;
    let median = if window_size_usize.is_multiple_of(2) {
        // Even: average of the two middle values, rounded down.
        // Overflow-safe averaging: a + (b - a) / 2 (safe because sorted, so b >= a).
        GasPrice(window[mid - 1].0 + (window[mid].0 - window[mid - 1].0) / 2)
    } else {
        window[mid]
    };
    Some(median)
}
