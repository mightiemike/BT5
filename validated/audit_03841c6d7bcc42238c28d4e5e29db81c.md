### Title
NLP Mint Has No Minimum Amount Check While BurnNlp Enforces a Minimum Fee That Exceeds Small Positions, Permanently Destroying User Funds — (`core/contracts/Clearinghouse.sol`)

---

### Summary

`mintNlp` in `Clearinghouse.sol` accepts any positive `quoteAmount` with no minimum size check. However, `burnNlp` enforces a minimum burn fee of `ONE` (1 USDC equivalent in 18-decimal engine units) regardless of position size. A user who mints NLP with less than 1 USDC worth of quote receives exactly `0` quote back when burning — their entire deposit is consumed by the fee. The mandatory 4-day NLP lock period (`NLP_LOCK_PERIOD`) means the user cannot exit immediately and must wait before discovering their funds are unrecoverable.

---

### Finding Description

**Root cause — no minimum in `mintNlp`:**

`Clearinghouse.mintNlp` performs no minimum check on `txn.quoteAmount`:

```solidity
require(txn.quoteAmount <= INT128_MAX, ERR_CONVERSION_OVERFLOW);
int128 quoteAmount = int128(txn.quoteAmount);
int128 nlpAmount = quoteAmount.div(oraclePriceX18);
``` [1](#0-0) 

Any positive `quoteAmount` is accepted. The user's quote balance is debited and NLP is credited.

**Root cause — minimum fee in `burnNlp`:**

`Clearinghouse.burnNlp` computes the burn fee as:

```solidity
int128 burnFee = MathHelper.max(ONE, quoteAmount / 1000);
quoteAmount = MathHelper.max(0, quoteAmount - burnFee);
``` [2](#0-1) 

`ONE` is `1e18` in the SD21x18 fixed-point system, representing 1 USDC (the quote token normalized to 18 decimals). When `quoteAmount < ONE` (i.e., the NLP position is worth less than 1 USDC), the fee floor kicks in: `burnFee = ONE`, and `quoteAmount = max(0, quoteAmount - ONE) = 0`. The user burns their NLP and receives **zero** quote back.

**Root cause — 4-day lock prevents immediate exit:**

All NLP minted by non-`N_ACCOUNT` subaccounts is locked for `NLP_LOCK_PERIOD = 4 days`:

```solidity
unlockedAt: getOracleTime() + NLP_LOCK_PERIOD
``` [3](#0-2) 

`burnNlp` enforces this via:

```solidity
require(
    spotEngine.getNlpUnlockedBalance(txn.sender).amount >= nlpAmount,
    ERR_UNLOCKED_NLP_INSUFFICIENT
);
``` [4](#0-3) 

The user cannot exit their position for 4 days. After the lock expires, they discover their entire deposit is consumed by the fee.

---

### Impact Explanation

A user who mints NLP with any `quoteAmount < ONE` (less than 1 USDC):

1. Has their quote balance debited immediately.
2. Receives NLP tokens that are locked for 4 days.
3. After the lock expires, burns their NLP and receives `0` quote back — the entire deposited value is taken as the burn fee.

The corrupted state delta is: user's quote balance decreases by `quoteAmount`, NLP balance decreases by `nlpAmount`, and the recovered quote is `0` instead of `quoteAmount - proportional_fee`. The user suffers a 100% loss on their deposit for any position below the 1 USDC fee floor.

---

### Likelihood Explanation

The entry path is fully unprivileged: any user can submit a `MintNlp` transaction through the sequencer. There is no on-chain guard preventing a `quoteAmount` below `ONE`. The sequencer processes the transaction, the Clearinghouse accepts it, and the NLP lock is applied. The user has no on-chain recourse during the 4-day lock period, and no recovery path after it. This is a realistic scenario for any user minting a small NLP position (e.g., testing the protocol, or a dust position left after trading).

---

### Recommendation

Add a minimum `quoteAmount` check in `mintNlp` that ensures the position value exceeds the minimum burn fee:

```solidity
// In Clearinghouse.mintNlp
require(int128(txn.quoteAmount) >= ONE, ERR_DEPOSIT_TOO_SMALL);
``` [1](#0-0) 

Alternatively, remove the `ONE` floor from the burn fee in `burnNlp` and apply only the proportional fee (`quoteAmount / 1000`), which naturally scales to zero for small positions:

```solidity
// In Clearinghouse.burnNlp
int128 burnFee = quoteAmount / 1000;
``` [2](#0-1) 

---

### Proof of Concept

```
1. Alice calls MintNlp with quoteAmount = 0.5e18 (0.5 USDC in engine units).
   - Clearinghouse.mintNlp: no minimum check, quoteAmount = 0.5e18 accepted.
   - Alice's quote balance: -0.5e18
   - Alice's NLP balance: +nlpAmount (locked for 4 days)

2. Alice waits 4 days for the NLP lock to expire.

3. Alice calls BurnNlp with nlpAmount = her full NLP balance.
   - quoteAmount = nlpAmount * oraclePriceX18 = 0.5e18 (assuming price unchanged)
   - burnFee = max(ONE, 0.5e18 / 1000) = max(1e18, 5e14) = 1e18
   - quoteAmount = max(0, 0.5e18 - 1e18) = max(0, -0.5e18) = 0
   - Alice receives 0 quote back.

4. Alice has lost her entire 0.5 USDC deposit.
   - Net state: Alice's quote balance = -0.5e18 (from mint), NLP balance = 0 (burned).
   - The 0.5 USDC is permanently consumed by the burn fee.
``` [5](#0-4) [6](#0-5)

    function burnNlp(
        IEndpoint.BurnNlp calldata txn,
        int128 oraclePriceX18,
        IEndpoint.NlpPool[] calldata nlpPools,
        int128[] calldata nlpPoolRebalanceX18
    ) external onlyEndpoint {
        require(!RiskHelper.isIsolatedSubaccount(txn.sender), ERR_UNAUTHORIZED);

        ISpotEngine spotEngine = _spotEngine();
        spotEngine.updatePrice(NLP_PRODUCT_ID, oraclePriceX18);

        require(txn.nlpAmount <= INT128_MAX, ERR_CONVERSION_OVERFLOW);
        int128 nlpAmount = int128(txn.nlpAmount);
        require(
            spotEngine.getNlpUnlockedBalance(txn.sender).amount >= nlpAmount,
            ERR_UNLOCKED_NLP_INSUFFICIENT
        );
        int128 quoteAmount = nlpAmount.mul(oraclePriceX18);
        int128 burnFee = MathHelper.max(ONE, quoteAmount / 1000);
        quoteAmount = MathHelper.max(0, quoteAmount - burnFee);

        _validateNlpRebalance(nlpPools, nlpPoolRebalanceX18, -quoteAmount);
        for (uint128 i = 0; i < nlpPoolRebalanceX18.length; i++) {
            require(nlpPoolRebalanceX18[i] <= 0, ERR_INVALID_NLP_REBALANCE);
        }

        spotEngine.updateBalance(NLP_PRODUCT_ID, txn.sender, -nlpAmount);
        spotEngine.updateBalance(NLP_PRODUCT_ID, N_ACCOUNT, nlpAmount);

        if (quoteAmount > 0) {
            spotEngine.updateBalance(QUOTE_PRODUCT_ID, txn.sender, quoteAmount);
            _applyNlpRebalance(spotEngine, nlpPools, nlpPoolRebalanceX18);
        }

        require(
            spotEngine.getBalance(NLP_PRODUCT_ID, txn.sender).amount >= 0,
            ERR_SUBACCT_HEALTH
        );
        // Burning NLP can decrease health if the burn fee exceeds the health improvement
        // from the withdrawal. This check prevents malicious actors from deliberately
        // creating unhealthy subaccounts through NLP burns.
        require(
            getHealth(txn.sender, IProductEngine.HealthType.MAINTENANCE) >= 0,
            ERR_SUBACCT_HEALTH
        );
    }
```
