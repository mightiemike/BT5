### Title
Gateway L2 Gas Price Admission Check Uses Stale Previous-Block Price Instead of Next-Block Price — (`crates/apollo_gateway/src/stateful_transaction_validator.rs`)

### Summary

The gateway's `validate_resource_bounds` function enforces a minimum L2 gas price threshold against the **previous committed block's** L2 gas price, not the **next block's** gas price that the transaction will actually be executed under. Under EIP-1559 dynamics the next block's price can exceed the previous block's price. Transactions whose `max_price_per_unit` falls between the two values pass gateway admission but are invalid for the block they will be sequenced into.

### Finding Description

In `validate_resource_bounds` the gateway reads the L2 gas price from `self.gateway_fixed_block_state_reader.get_block_info()`:

```rust
// TODO(Arni): getnext_l2_gas_price from the block header.
let previous_block_l2_gas_price = self
    .gateway_fixed_block_state_reader
    .get_block_info()
    .await?
    .gas_prices
    .strk_gas_prices
    .l2_gas_price;
``` [1](#0-0) 

The TODO comment is the developer's own acknowledgement that the wrong price is being used. `GatewayFixedBlockSyncStateClient.get_block_info_from_sync_client` populates `BlockInfo` from `block_header_without_hash.l2_gas_price` — the **current** block's price — and never reads `next_l2_gas_price`, which is a separate field on the `Block` object: [2](#0-1) 

The `next_l2_gas_price` field is available on the `Block` struct: [3](#0-2) 

Meanwhile, `run_validate_entry_point` correctly increments the block number to the next block but leaves the gas prices unchanged:

```rust
let mut block_info = self.gateway_fixed_block_state_reader.get_block_info().await?;
block_info.block_number = block_info.block_number.unchecked_next();
``` [4](#0-3) 

This creates an internal inconsistency: the block context used for blockifier validation carries the **next** block number but the **previous** block's gas prices. The threshold computation in `validate_tx_l2_gas_price_within_threshold` is:

```
threshold = (min_gas_price_percentage / 100) * previous_block_l2_gas_price
``` [5](#0-4) 

Any transaction with `max_price_per_unit` in the range `[threshold, next_block_l2_gas_price)` passes the gateway check but is priced below what the next block will charge.

### Impact Explanation

A transaction admitted with `max_price_per_unit < next_block_l2_gas_price` is invalid for the block it will be sequenced into. The gateway's resource-bounds guard — whose explicit purpose is to filter out under-priced transactions before sequencing — fails to do so because it evaluates against a stale price. This matches the allowed impact: **"High. Mempool/gateway/RPC admission accepts invalid transactions or rejects valid transactions before sequencing."**

### Likelihood Explanation

The EIP-1559 mechanism increases the L2 gas price whenever a block's gas usage exceeds the target (`gas_target`). During any period of sustained high throughput the next block's price will exceed the previous block's price by up to `1/gas_price_max_change_denominator`. The gap is bounded but non-zero under normal load, making this condition routinely reachable without any special attacker capability — any user can craft a transaction priced just above the stale threshold.

### Recommendation

Expose `next_l2_gas_price` through the `GatewayFixedBlockStateReader` interface (or add a dedicated method) and replace the `previous_block_l2_gas_price` variable in `validate_resource_bounds` with the next block's price. The value is already present in the `Block` object returned by the state-sync client; it only needs to be threaded through `GatewayFixedBlockSyncStateClient.get_block_info_from_sync_client`.

### Proof of Concept

1. Let the latest committed block have `l2_gas_price = P` and `next_l2_gas_price = P'` where `P' > P` (block was above gas target).
2. Submit an invoke V3 transaction with `l2_gas.max_price_per_unit = P * min_gas_price_percentage / 100` (exactly at the stale threshold).
3. `validate_resource_bounds` computes `threshold = P * min_gas_price_percentage / 100` and passes the transaction because `max_price_per_unit >= threshold`.
4. The blockifier `__validate__` call also runs with the previous block's gas prices and passes.
5. The transaction is admitted to the mempool.
6. When the batcher sequences the transaction into the next block (gas price `P'`), the transaction's `max_price_per_unit < P'`, making it invalid for that block.

The root cause is structurally identical to the referenced external bug: a value (`total_debt_shares` / `l2_gas_price`) that must be refreshed before a guard check is instead read from a stale snapshot, allowing an operation that should be rejected to proceed.

```rust
    fn validate_tx_l2_gas_price_within_threshold(
        &self,
        tx_resource_bounds: ValidResourceBounds,
        previous_block_l2_gas_price: NonzeroGasPrice,
    ) -> StatefulTransactionValidatorResult<()> {
        match tx_resource_bounds {
            ValidResourceBounds::AllResources(tx_resource_bounds) => {
                let tx_l2_gas_price = tx_resource_bounds.l2_gas.max_price_per_unit;
                let gas_price_threshold_multiplier =
                    Ratio::new(self.config.min_gas_price_percentage.into(), 100_u128);
                let threshold = (gas_price_threshold_multiplier
                    * previous_block_l2_gas_price.get().0)
                    .to_integer();
                if tx_l2_gas_price.0 < threshold {
                    return Err(StarknetError {
                        // We didn't have this kind of an error.
                        code: StarknetErrorCode::UnknownErrorCode(
                            "StarknetErrorCode.GAS_PRICE_TOO_LOW".to_string(),
                        ),
                        message: format!(
                            "Transaction L2 gas price {tx_l2_gas_price} is below the required \
                             threshold {threshold}.",
                        ),
                    });
                }
            }
            ValidResourceBounds::L1Gas(_) => {
                // No validation required for legacy transactions.
            }
        }
        Ok(())
    }

    async fn get_block_info_from_sync_client(&self) -> StarknetResult<BlockInfo> {
        let block = self.state_sync_client.get_block(self.block_number).await.map_err(|e| {
            StarknetError::internal_with_logging("Failed to get latest block info", e)
        })?;

        let block_header = block.block_header_without_hash;
        let block_info = BlockInfo {
            block_number: block_header.block_number,
            block_timestamp: block_header.timestamp,
            sequencer_address: block_header.sequencer.0,
            gas_prices: GasPrices {
                eth_gas_prices: GasPriceVector {
                    l1_gas_price: block_header.l1_gas_price.price_in_wei.try_into()?,
                    l1_data_gas_price: block_header.l1_data_gas_price.price_in_wei.try_into()?,
                    l2_gas_price: block_header.l2_gas_price.price_in_wei.try_into()?,
                },
                strk_gas_prices: GasPriceVector {
                    l1_gas_price: block_header.l1_gas_price.price_in_fri.try_into()?,
                    l1_data_gas_price: block_header.l1_data_gas_price.price_in_fri.try_into()?,
                    l2_gas_price: block_header.l2_gas_price.price_in_fri.try_into()?,
                },
            },
            use_kzg_da: block_header.l1_da_mode.is_use_kzg_da(),
            starknet_version: block_header.starknet_version,
        };

        Ok(block_info)
    }
