### Title
Dual-guard deadlock in `EditNFT` and `BurnNFT` permanently locks transferred NFTs — (`x/nft/keeper/keeper.go`)

### Summary

`EditNFT` and `BurnNFT` each require the caller to simultaneously satisfy two mutually exclusive post-transfer conditions: be the **current owner** (`IsOwner`) AND be the **denom creator** (`IsDenomCreator`). Once the denom creator transfers an NFT to any third party, no address can ever satisfy both conditions at once, making every such NFT permanently uneditable and unburnable through the standard message paths.

### Finding Description

Both `EditNFT` and `BurnNFT` in `x/nft/keeper/keeper.go` apply two sequential, conjunctive authorization checks:

**`EditNFT`:** [1](#0-0) 

**`BurnNFT`:** [2](#0-1) 

`IsOwner` verifies the caller equals the NFT's stored `Owner` field: [3](#0-2) 

`IsDenomCreator` verifies the caller equals the denom's immutable `Creator` field: [4](#0-3) 

`TransferOwner` (called by `MsgTransferNFT`) updates `nft.Owner` to the recipient and calls `swapOwner`, permanently removing the creator from the owner index: [5](#0-4) 

After the transfer:
- **Creator (alice)** → `IsOwner` fails (she is no longer owner) → `ErrUnauthorized` before `IsDenomCreator` is even reached.
- **New owner (bob)** → `IsOwner` passes, then `IsDenomCreator` fails (bob is not the creator) → `ErrUnauthorized`.

There is no third address that can be both owner and creator simultaneously. The NFT is permanently frozen.

The only bypass, `BurnNFTUnverified`, skips the `IsDenomCreator` check but is only reachable via the IBC transfer escrow/burn path — not via `MsgBurnNFT`: [6](#0-5) 

### Impact Explanation

Any NFT transferred away from its denom creator becomes:
- **Permanently uneditable** — no address can call `MsgEditNFT` successfully.
- **Permanently unburnable** — no address can call `MsgBurnNFT` successfully, so the supply counter can never be decremented for that token.

This breaks the invariant that the denom creator retains administrative control (edit/burn) over all NFTs in their denom. Metadata is frozen in whatever state it was at transfer time, and the circulating supply of the denom can never be reduced for transferred tokens.

### Likelihood Explanation

The trigger is a single, ordinary `MsgTransferNFT` transaction — a fully supported, user-facing production path. Any denom creator who transfers an NFT (e.g., to sell or gift it) immediately and irreversibly triggers the deadlock. No special privileges, governance, or attacker cooperation is required beyond the creator performing a normal transfer.

### Recommendation

Decouple the two authorization checks so they are alternatives rather than conjunctive requirements:

- **`EditNFT`**: allow the call if the sender is the current owner **OR** the denom creator (or both).
- **`BurnNFT`**: allow the call if the sender is the current owner **OR** the denom creator (or both).

This matches the semantic intent: the owner controls their token, and the creator retains administrative authority over the denom.

### Proof of Concept

```
// Keeper-level test (no mocks needed)
alice issues denom "testdenom"          → denom.Creator = alice
alice mints NFT "token1" to alice       → nft.Owner = alice
alice sends MsgTransferNFT to bob       → nft.Owner = bob

// EditNFT
alice.EditNFT("testdenom","token1",...) → IsOwner fails  → ErrUnauthorized ✓
bob.EditNFT("testdenom","token1",...)   → IsDenomCreator fails → ErrUnauthorized ✓

// BurnNFT
alice.BurnNFT("testdenom","token1")     → IsOwner fails  → ErrUnauthorized ✓
bob.BurnNFT("testdenom","token1")       → IsDenomCreator fails → ErrUnauthorized ✓
// NFT is now permanently locked; supply counter never decremented.
```

```go
// BurnNFTUnverified deletes a specified NFT without verifying if the owner is the creator of denom
// Needed for IBC transfer of NFT
func (k Keeper) BurnNFTUnverified(ctx sdk.Context, denomID, tokenID string, owner sdk.AccAddress) error {
	if !k.HasDenomID(ctx, denomID) {
		return sdkerrors.Wrapf(types.ErrInvalidDenom, "denom ID %s not exists", denomID)
	}

	nft, err := k.IsOwner(ctx, denomID, tokenID, owner)
	if err != nil {
		return err
	}

	k.deleteNFT(ctx, denomID, nft)
	k.deleteOwner(ctx, denomID, tokenID, owner)
	k.decreaseSupply(ctx, denomID)

	return nil
```
