Audit Report

## Title
SwapAllowlistExtension Gates on Router Address Instead of End User, Allowing Full Allowlist Bypass via MetricOmmSimpleRouter — (File: `metric-periphery/contracts/extensions/SwapAllowlistExtension.sol`)

## Summary

`SwapAllowlistExtension.beforeSwap` checks `allowedSwapper[msg.sender][sender]` where `sender` is the pool's `msg.sender` — the direct caller of `pool.swap()`. When swaps are routed through `MetricOmmSimpleRouter`, the router becomes the pool's `msg.sender`, so the extension checks whether the router is allowlisted, not the end user. Any EOA can bypass a curated pool's allowlist by calling the router, which is the only supported periphery path and must be allowlisted for the pool to be usable.

## Finding Description

`MetricOmmPool.swap` passes its own `msg.sender` as the `sender` argument to `_beforeSwap`:

```solidity
// metric-core/contracts/MetricOmmPool.sol L230-240
_beforeSwap(
    msg.sender,   // ← direct caller of pool.swap(), not the end user
    recipient,
    ...
);
```

`ExtensionCalling._beforeSwap` forwards this value unchanged into the extension call:

```solidity
// metric-core/contracts/ExtensionCalling.sol L160-176
_callExtensionsInOrder(
    BEFORE_SWAP_ORDER,
    abi.encodeCall(IMetricOmmExtensions.beforeSwap, (sender, ...))
);
```

`SwapAllowlistExtension.beforeSwap` then checks this value against the per-pool allowlist:

```solidity
// metric-periphery/contracts/extensions/SwapAllowlistExtension.sol L37-39
if (!allowAllSwappers[msg.sender] && !allowedSwapper[msg.sender][sender]) {
    revert IMetricOmmPoolActions.NotAllowedToSwap();
}
```

`MetricOmmSimpleRouter.exactInputSingle` calls `pool.swap(...)` directly, making the router the pool's `msg.sender`:

```solidity
// metric-periphery/contracts/MetricOmmSimpleRouter.sol L72-80
(int128 amount0Delta, int128 amount1Delta) = IMetricOmmPoolActions(params.pool)
    .swap(params.recipient, params.zeroForOne, ...);
```

The same applies to `exactInput` (L104), `exactOutputSingle` (L136), and `exactOutput` (L165). In all cases, the pool receives `msg.sender = router`, passes `sender = router` to the extension, and the extension evaluates `allowedSwapper[pool][router]`. The actual end user's address (`msg.sender` at the router level) is stored only in transient callback context for payment purposes and is never forwarded into the extension check.

## Impact Explanation

A pool admin deploying a curated pool with `SwapAllowlistExtension` must call `setAllowedToSwap(pool, router, true)` to allow any user to interact via the supported periphery path. Once the router is allowlisted, the allowlist is silently nullified: every EOA on the network can call `router.exactInputSingle({pool: pool, ...})`, the extension approves the call because `allowedSwapper[pool][router] == true`, and the swap executes regardless of whether the end user was ever individually allowlisted. This breaks the core access-control invariant of the extension — unauthorized users can trade against oracle-anchored liquidity in a pool intended for a restricted participant set.

## Likelihood Explanation

The trigger requires no privilege, no special timing, no flash loan, and no contract deployment. Any EOA can call the public router. Allowlisting the router is the only way to enable the standard periphery swap path, so the precondition is met in every realistic curated-pool deployment that intends to support router-mediated swaps. The condition is permanent once set and affects every subsequent swap.

## Recommendation

The end-user address must be threaded through the swap call chain so the extension can gate on the actual economic actor. One approach: before calling `pool.swap(...)`, the router writes the originating `msg.sender` into transient storage (a dedicated slot separate from the existing callback context), and the pool reads this value and passes it as `sender` to `_beforeSwap` instead of its own `msg.sender`. Alternatively, the pool's `swap` interface could accept an explicit `swapper` parameter. As a documentation-only stopgap, `SwapAllowlistExtension` should be marked incompatible with any intermediary router, but this does not fix the code-level misbinding.

## Proof of Concept

1. Deploy a pool with `SwapAllowlistExtension` as the `beforeSwap` hook; `allowAllSwappers[pool] = false`.
2. Pool admin calls `setAllowedToSwap(pool, router, true)` to enable router-mediated swaps (required operational step).
3. Attacker (EOA not in the allowlist) calls `router.exactInputSingle({pool: pool, recipient: attacker, ...})`.
4. Router calls `pool.swap(attacker, ...)` — pool's `msg.sender` = router.
5. Pool calls `_beforeSwap(sender=router, ...)` → `extension.beforeSwap(sender=router, ...)`.
6. Extension evaluates `allowedSwapper[pool][router] == true` → passes.
7. Attacker's swap executes on the curated pool despite never being individually allowlisted. [1](#0-0) [2](#0-1) [3](#0-2) [4](#0-3)
