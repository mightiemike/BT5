### Title
Static `min_gas_price` in `StatelessTransactionValidatorConfig` Diverges from Versioned Constants `min_gas_price`, Causing Gateway Admission Mismatch - (File: `crates/apollo_gateway_config/src/config.rs`)

---

### Summary

The gateway's stateless transaction validator uses a static `min_gas_price` field (hardcoded to 8 Gwei in the default config and in `config_schema.json`) that is entirely independent of `apollo_versioned_constants::VersionedConstants.min_gas_price`, which is the authoritative floor used by the EIP-1559 fee market. At Starknet version V0_14_0, the versioned constants set `min_gas_price = 3 Gwei (0xb2d05e00)`, while the stateless validator requires 8 Gwei. This creates a config-validation boundary mismatch: the gateway rejects transactions whose `l2_gas.max_price_per_unit` falls between 3 Gwei and 8 Gwei, even though the fee market would accept them. The inverse risk also exists: if a future versioned-constants update raises `min_gas_price` above 8 Gwei, the stateless gate will admit transactions that the stateful gate will reject.

---

### Finding Description

**Two independent `min_gas_price` sources govern the same admission invariant:**

**Gate 1 — Stateless validator** (`StatelessTransactionValidator::validate_resource_bounds`):

```rust
if resource_bounds.l2_gas.max_price_per_unit.0 < self.config.min_gas_price {
    return Err(StatelessTransactionValidatorError::MaxGasPriceTooLow { ... });
}
```

`self.config.min_gas_price` is a plain `u128` field in `StatelessTransactionValidatorConfig`, defaulting to `8_000_000_000` (8 Gwei). It is read from the static node config file and is **not** derived from the versioned constants. [1](#0-0) [2](#0-1) 

The code carries an explicit acknowledgement of the mismatch:

```rust
// TODO(AlonH): Remove the `min_gas_price` field from this struct and use the one from the
// versioned constants.
pub min_gas_price: u128,
``` [3](#0-2) 

**Gate 2 — Fee market / orchestrator versioned constants** (`apollo_versioned_constants::VersionedConstants.min_gas_price`):

Used by `get_min_gas_price_for_height()` as the fallback floor for the EIP-1559 price calculation:

```rust
let fallback_min_gas_price = VersionedConstants::latest_constants().min_gas_price;
``` [4](#0-3) 

The versioned constants JSON files set:
- **V0_14_0**: `"min_gas_price": "0xb2d05e00"` = **3,000,000,000 fri (3 Gwei)**
- **V0_14_1+**: `"min_gas_price": "0x1dcd65000"` = **8,000,000,000 fri (8 Gwei)** [5](#0-4) [6](#0-5) 

**Gate 3 — Stateful validator** (`StatefulTransactionValidator::validate_tx_l2_gas_price_within_threshold`):

Uses `previous_block_l2_gas_price * min_gas_price_percentage%` (dynamic, block-derived). This is the gate that actually tracks the live network price. [7](#0-6) 

**The divergence at V0_14_0:**

| Source | Value |
|---|---|
| `StatelessTransactionValidatorConfig.min_gas_price` | 8,000,000,000 fri (8 Gwei) |
| `VersionedConstants (V0_14_0).min_gas_price` | 3,000,000,000 fri (3 Gwei) |
| Divergence | **+5 Gwei** (stateless is 2.67× stricter) |

A transaction with `l2_gas.max_price_per_unit = 5 Gwei`:
- Passes the fee market floor (5 ≥ 3 Gwei) → would be valid for sequencing
- **Fails the stateless gate** (5 < 8 Gwei) → rejected at the gateway before reaching the mempool

The inverse scenario: if a future versioned-constants update sets `min_gas_price = 20 Gwei`, the fee market enforces a block price floor of 20 Gwei, but the stateless gate still admits transactions with `max_price = 10 Gwei`. Those transactions pass stateless, enter the stateful validator, and are rejected there — meaning the gateway admitted a transaction that is invalid for the current block price.

---

### Impact Explanation

**High — Mempool/gateway/RPC admission rejects valid transactions or accepts invalid transactions before sequencing.**

- **Current (V0_14_0)**: The stateless gate is 2.67× stricter than the fee market floor. Any transaction with `max_price_per_unit` in `[3 Gwei, 8 Gwei)` is rejected by the gateway even though it would be accepted and executed successfully at the current block price. This is a systematic, unprivileged denial of valid transactions.
- **Future (versioned constants `min_gas_price` > 8 Gwei)**: The stateless gate becomes too permissive. Transactions with `max_price_per_unit` in `[8 Gwei, new_floor)` pass stateless validation and enter the stateful validator, which rejects them. The gateway admits transactions that are invalid for the current block price.

---

### Likelihood Explanation

The divergence is **already present** at V0_14_0 (3 Gwei vs 8 Gwei). The TODO comment confirms the design team is aware the two values should be unified but have not yet done so. Every future versioned-constants update that changes `min_gas_price` without a corresponding update to the static gateway config will re-introduce or widen the divergence. The risk is proportional to the frequency of protocol upgrades.

---

### Recommendation

Remove `min_gas_price` from `StatelessTransactionValidatorConfig` and replace the static check with a lookup against `VersionedConstants::latest_constants().min_gas_price` (or the version-appropriate constant), as the TODO comment already prescribes. This ensures the stateless gate and the fee market always share the same floor, eliminating the divergence across all current and future protocol versions.

---

### Proof of Concept

1. Run a node at Starknet version V0_14_0 (versioned constants `min_gas_price = 3 Gwei`).
2. Submit an invoke transaction with `l2_gas.max_price_per_unit = 5 Gwei` (above the fee market floor, below the static stateless floor).
3. `StatelessTransactionValidator::validate_resource_bounds` evaluates `5_000_000_000 < 8_000_000_000` → returns `Err(MaxGasPriceTooLow)`.
4. The gateway rejects the transaction before it reaches the stateful validator or the mempool.
5. The same transaction submitted with `max_price_per_unit = 8 Gwei` is accepted, even though the block price is only 3 Gwei — demonstrating the 5 Gwei transaction was valid and was incorrectly rejected.

pub fn get_min_gas_price_for_height(
    height: BlockNumber,
    min_l2_gas_price_per_height: &[PricePerHeight],
) -> GasPrice {
    let fallback_min_gas_price = VersionedConstants::latest_constants().min_gas_price;
    min_l2_gas_price_per_height
        .iter()
        .rev()
        .find(|e| e.height <= height.0)
        .map(|e| GasPrice(e.price))
        .unwrap_or(fallback_min_gas_price)
}

    // TODO(Arni): Consider running this validation for all gas prices.
    fn validate_tx_l2_gas_price_within_threshold(
        &self,
        tx_resource_bounds: ValidResourceBounds,
        previous_block_l2_gas_price: NonzeroGasPrice,
    ) -> StatefulTransactionValidatorResult<()> {
        match tx_resource_bounds {
            ValidResourceBounds::AllResources(tx_resource_bounds) => {
                let tx_l2_gas_price = tx_resource_bounds.l2_gas.max_price_per_unit;
                let gas_price_threshold_multiplier =
                    Ratio::new(self.config.min_gas_price_percentage.into(), 100_u128);
                let threshold = (gas_price_threshold_multiplier
                    * previous_block_l2_gas_price.get().0)
                    .to_integer();
                if tx_l2_gas_price.0 < threshold {
                    return Err(StarknetError {
                        // We didn't have this kind of an error.
                        code: StarknetErrorCode::UnknownErrorCode(
                            "StarknetErrorCode.GAS_PRICE_TOO_LOW".to_string(),
                        ),
                        message: format!(
                            "Transaction L2 gas price {tx_l2_gas_price} is below the required \
                             threshold {threshold}.",
                        ),
                    });
                }
            }
            ValidResourceBounds::L1Gas(_) => {
                // No validation required for legacy transactions.
            }
        }
        Ok(())
    }
