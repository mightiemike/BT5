Audit Report

## Title
SwapAllowlistExtension Checks Router Address Instead of Originating User, Allowing Any User to Bypass Swap Allowlist via Allowlisted Router - (`metric-periphery/contracts/extensions/SwapAllowlistExtension.sol`)

## Summary

`SwapAllowlistExtension.beforeSwap` gates swaps by checking `allowedSwapper[msg.sender][sender]`, where `sender` is the address passed by the pool — always `msg.sender` of `pool.swap()`, i.e., the router. When the router is allowlisted, every user who calls the router bypasses the per-pool swap allowlist entirely, because the originating user address is never consulted. The extension's sole purpose — restricting which addresses may swap — is defeated.

## Finding Description

The exploit path is fully traceable through production code:

**Step 1.** A non-allowlisted user calls `MetricOmmSimpleRouter.exactInputSingle`. The router calls `IMetricOmmPoolActions(params.pool).swap(params.recipient, ...)` — so inside the pool, `msg.sender` is the **router**, not the user. [1](#0-0) 

**Step 2.** `MetricOmmPool.swap` calls `_beforeSwap(msg.sender, ...)`, passing the **router address** as `sender`. [2](#0-1) 

**Step 3.** `ExtensionCalling._beforeSwap` encodes and forwards that `sender` (the router) to `extension.beforeSwap(sender, ...)`. [3](#0-2) 

**Step 4.** `SwapAllowlistExtension.beforeSwap` checks `allowedSwapper[msg.sender][sender]`, where `msg.sender` is the pool and `sender` is the router. Since the router is allowlisted, the check passes — the originating user is **never consulted**. [4](#0-3) 

**Root cause:** The `IMetricOmmExtensions.beforeSwap` interface carries only `sender` (the pool's direct caller) and `recipient`; there is no `originator` field. The pool has no mechanism to propagate the true end-user through the router layer. [5](#0-4) 

**Existing guard is insufficient:** `allowAllSwappers[msg.sender]` is a separate bypass flag; the `allowedSwapper` path is the intended per-address gate, but it resolves to the router address, not the user. [4](#0-3) 

## Impact Explanation

Pool admins deploy `SwapAllowlistExtension` specifically to restrict which addresses may swap — this is the extension's sole purpose, as stated in its NatSpec (`@notice Gates swap by swapper address, per pool`). A pool admin who allowlists the router (a natural and expected configuration for any pool that wants to support the official router) inadvertently grants every on-chain user the ability to swap, defeating the allowlist entirely. The swap executes at live oracle prices with real token transfers; the non-allowlisted user receives real output tokens. This constitutes broken core pool functionality: the allowlist access control is completely non-functional when the router is allowlisted, and pool curation cannot be enforced. [6](#0-5) 

## Likelihood Explanation

The router is a public, permissionless contract — any user can call `exactInputSingle`. The only precondition is that the pool admin has allowlisted the router address, which is both natural and expected for any pool that wants to support the official router. No privileged access, no special token behavior, and no malicious pool setup is required. The attack is repeatable by any address on every block.

## Recommendation

The pool's `swap` interface should propagate the true originator. One approach: add an `originator` field to the `beforeSwap` extension call, set to `msg.sender` at the router entry point and forwarded through a dedicated parameter. Alternatively, `SwapAllowlistExtension` should require the router to attest the originating user via `extensionData`, with the router populating that field as `msg.sender` before calling the pool, and the extension verifying both that the router is allowlisted and that the attested user is allowlisted.

## Proof of Concept

```
1. Deploy pool with SwapAllowlistExtension.
2. Pool admin calls setAllowedToSwap(pool, router, true)   // allowlist the router
3. Pool admin does NOT call setAllowedToSwap(pool, user, true)
4. user calls router.exactInputSingle({pool: pool, ...})
5. router calls pool.swap(recipient, ...) — msg.sender in pool = router
6. pool calls _beforeSwap(router, ...) → extension.beforeSwap(router, ...)
7. Extension checks allowedSwapper[pool][router] == true → passes
8. Swap executes; user receives output tokens.
9. USER_IS_ALLOWLISTED_SWAPPER invariant violated: non-allowlisted user
   successfully swapped through an allowlisted router.
```
