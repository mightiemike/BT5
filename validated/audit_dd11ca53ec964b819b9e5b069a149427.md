### Title
Trailing Bytes Silently Accepted by `node_from_bytes` and `node_from_bytes_backrefs` — Non-Canonical Serialization Accepted as Valid — (`src/serde/de.rs`, `src/serde/de_br.rs`)

---

### Summary

`node_from_bytes` and `node_from_bytes_backrefs` deserialize a CLVM node from a caller-supplied byte slice but never verify that all bytes in the slice were consumed. Trailing bytes after a structurally complete CLVM object are silently discarded, and the functions return `Ok(node)` for inputs that are not canonical serializations. This is the direct analog of the Oracle finding: `latestRoundData()` returns five fields but only `price > 0` is checked — the remaining fields (`answeredInRound`, `timestamp`) are silently ignored. Here, the byte stream returns a valid node but the remaining unconsumed bytes are silently ignored, breaking the invariant that a successful parse implies the input was a complete, canonical serialization.

---

### Finding Description

**Root cause — `src/serde/de.rs`, lines 44–46:**

```rust
pub fn node_from_bytes(allocator: &mut Allocator, b: &[u8]) -> Result<NodePtr> {
    let mut buffer = Cursor::new(b);
    node_from_stream(allocator, &mut buffer)
    // ← no check: buffer.position() == b.len() as u64
}
```

`node_from_stream` stops as soon as one complete S-expression has been parsed. The cursor's final position is never compared to `b.len()`. Any bytes after the first valid CLVM object are silently dropped, and the function returns `Ok`.

**Same root cause — `src/serde/de_br.rs`, lines 114–117:**

```rust
pub fn node_from_bytes_backrefs(allocator: &mut Allocator, b: &[u8]) -> Result<NodePtr> {
    let mut buffer = Cursor::new(b);
    node_from_stream_backrefs(allocator, &mut buffer, |_node| {})
    // ← no check: buffer.position() == b.len() as u64
}
```

Identical omission for the back-reference-aware deserializer.

**The correct check already exists elsewhere — `src/serde/tools.rs`, line 235:**

```rust
f.get_ref().len() as u64 == f.position()
```

`is_canonical_serialization` performs exactly this trailing-byte check at the end of its loop. The two primary deserialization entry points do not.

**Attacker-reachable callers — `wheel/src/api.rs`:**

| Function | Line | Uses |
|---|---|---|
| `run_serialized_chia_program` | 55–56 | `node_from_bytes` for both `program` and `args` |
| `deser_legacy` | 104–107 | `node_from_bytes` |
| `deser_backrefs` | 111–114 | `node_from_bytes_backrefs` |
| `deser_auto` | 148–156 | `node_from_bytes_backrefs` (non-2026 path) |

All four are `#[pyfunction]` exports, directly callable from Python with attacker-controlled bytes.

---

### Impact Explanation

An attacker submits a CLVM puzzle reveal (or solution) whose serialized bytes consist of a valid CLVM object followed by arbitrary trailing garbage. `node_from_bytes` accepts the input, returns `Ok(node)`, and `run_serialized_chia_program` executes the program without error. The parsed tree is correct (trailing bytes do not alter it), but the acceptance of a non-canonical byte sequence breaks two invariants:

1. **Non-canonical serialization accepted as valid**: Two distinct byte sequences — one canonical, one with trailing garbage — deserialize to the same `NodePtr`. Any system that uses the raw bytes as an identifier (e.g., for deduplication, mempool keying, or spend-bundle hashing) will treat them as different objects even though they represent the same program.

2. **Consensus divergence**: If a consensus peer or a future version of `chia_rs` enforces canonical serialization of puzzle reveals (as `is_canonical_serialization` is designed to support), it will reject the spend bundle that `clvm_rs` accepted. Nodes running the current `clvm_rs` and nodes running a stricter validator will disagree on the validity of the same spend bundle, producing a chain split.

The corrupted result is the `Ok(NodePtr)` return value from `node_from_bytes` / `node_from_bytes_backrefs` for a non-canonical input — the function signals success where it should signal a serialization error.

---

### Likelihood Explanation

The attacker entry path requires no privileges. Any participant who can submit a spend bundle to a Chia full node can craft a puzzle reveal with trailing bytes. The Python API (`run_serialized_chia_program`, `deser_legacy`, `deser_backrefs`, `deser_auto`) is the standard integration surface for wallet software and block validators. The crafted input is trivially constructed: take any valid serialized CLVM program and append one or more arbitrary bytes.

---

### Recommendation

After `node_from_stream` / `node_from_stream_backrefs` returns, assert that the cursor has consumed the entire input:

```rust
pub fn node_from_bytes(allocator: &mut Allocator, b: &[u8]) -> Result<NodePtr> {
    let mut buffer = Cursor::new(b);
    let node = node_from_stream(allocator, &mut buffer)?;
    if buffer.position() != b.len() as u64 {
        return Err(EvalErr::SerializationError);
    }
    Ok(node)
}
```

Apply the identical fix to `node_from_bytes_backrefs`. This mirrors the check already present in `is_canonical_serialization` at `src/serde/tools.rs` line 235 and in `serialized_length_serde_2026` at `src/serde_2026/de.rs`.

### Proof of Concept

```rust
use clvmr::allocator::Allocator;
use clvmr::serde::node_from_bytes;

let mut allocator = Allocator::new();

// 0x80 = canonical NIL; 0xDE 0xAD = trailing garbage
let bytes: &[u8] = &[0x80, 0xDE, 0xAD];

// Returns Ok(NIL) — trailing bytes silently ignored
let result = node_from_bytes(&mut allocator, bytes);
assert!(result.is_ok(), "non-canonical input accepted without error");

// A stricter validator using is_canonical_serialization rejects the same bytes:
use clvmr::serde::is_canonical_serialization;
assert!(!is_canonical_serialization(bytes), "correctly rejected by canonical check");
// → the two APIs disagree on the same input
```

pub fn is_canonical_serialization(b: &[u8]) -> bool {
    let mut f = Cursor::new(b);
    let mut counter = 1;
    let mut b = [0; 1];
    while counter > 0 {
        counter -= 1;
        if f.read_exact(&mut b).is_err() {
            return false;
        }
        if b[0] == CONS_BOX_MARKER {
            // we expect to parse two more items from the stream
            // the left and right sub tree
            counter += 2;
        } else if b[0] == BACK_REFERENCE {
            // This is a back-ref. We don't actually need to resolve it, just
            // parse the path and move on
            if f.read_exact(&mut b).is_err() {
                return false;
            }
            if !is_canonical_atom(&mut f, b[0]) {
                return false;
            }
        } else if !is_canonical_atom(&mut f, b[0]) {
            return false;
        }
        if (f.get_ref().len() as u64) < f.position() {
            return false;
        }
    }
    f.get_ref().len() as u64 == f.position()
}
