### Title
Fast Withdrawal Fee Bypassed to Zero via Integer Division Truncation on Low-Decimal Tokens — (`core/contracts/BaseWithdrawPool.sol`)

---

### Summary

`BaseWithdrawPool.fastWithdrawalFeeAmount` converts a fee computed in X18 fixed-point back to native token units with a final integer division. For tokens with low decimal counts the divisor (`multiplier`) is large enough that both the proportional fee and the minimum fee floor can truncate to zero, letting any user withdraw repeatedly at zero cost.

---

### Finding Description

`fastWithdrawalFeeAmount` computes the fee in three steps:

```solidity
// BaseWithdrawPool.sol L141-148
int256 multiplier = int256(10**(MAX_DECIMALS - uint8(decimals)));   // e.g. 10**16 for 2-dec token
int128 amountX18  = int128(amount) * int128(multiplier);

int128 proportionalFeeX18 = FAST_WITHDRAWAL_FEE_RATE.mul(amountX18); // 0.1% of amount
int128 minFeeX18           = 5 * spotEngine().getConfig(productId).withdrawFeeX18;

int128 feeX18 = MathHelper.max(proportionalFeeX18, minFeeX18);
return feeX18 / int128(multiplier);                                  // ← truncates to 0
```

`FAST_WITHDRAWAL_FEE_RATE = 1e15` (0.1%).

**For a 2-decimal token** (`multiplier = 10**16`):

| Variable | Value |
|---|---|
| `proportionalFeeX18` | `amount × 10**13` |
| Threshold for zero | `amount < 1000` raw units (< 10 tokens) |
| `minFeeX18` (if `withdrawFeeX18 = 1e15`) | `5e15 < 10**16` → also 0 |

When both `proportionalFeeX18` and `minFeeX18` are smaller than `multiplier`, the Solidity integer division at line 148 truncates to `0`. The caller then pays no fee at all.

`submitFastWithdrawal` is `public` with no caller restriction:

```solidity
// BaseWithdrawPool.sol L81-113
function submitFastWithdrawal(
    uint64 idx,
    bytes calldata transaction,
    bytes[] calldata signatures
) public {
    ...
    int128 fee = fastWithdrawalFeeAmount(token, productId, transferAmount);

    if (sendTo == msg.sender) {
        require(transferAmount > uint128(fee), "Fee larger than balance");
        transferAmount -= uint128(fee);          // subtracts 0
    } else {
        safeTransferFrom(token, msg.sender, uint128(fee)); // transfers 0
    }
    fees[productId] += fee;                      // adds 0
    handleWithdrawTransfer(token, sendTo, transferAmount); // full amount sent
}
```

The only gate is `v.requireValidTxSignatures`, which verifies sequencer signatures over `(chainId, idx, txn)`. Because the sequencer is an off-chain service that processes user-initiated withdrawal requests, a user can legitimately request N small withdrawals (each below the fee threshold), receive N sequencer-signed transactions, and submit them all via `submitFastWithdrawal`, paying zero fee on each.

---

### Impact Explanation

The protocol collects zero fast-withdrawal fees on any low-decimal collateral token whose `withdrawFeeX18` config is set below `multiplier / 5`. An attacker withdraws an arbitrarily large balance by splitting it into sub-threshold chunks. `fees[productId]` is never incremented, so the accounting loss is permanent and unrecoverable.

---

### Likelihood Explanation

Ink Chain is an EVM L2 with low gas costs, making many small transactions economically viable. Any spot product backed by a token with ≤ 2 decimals (e.g., a 2-decimal stablecoin) and a `withdrawFeeX18` below `multiplier / 5` is immediately exploitable by any user who can request withdrawals from the sequencer — no special role or privilege required.

---

### Recommendation

Enforce a minimum returned fee before the division truncates it to zero. One approach: after computing `feeX18`, check that `feeX18 >= multiplier` before dividing, and revert or enforce a minimum native-unit fee otherwise. Alternatively, round up the division:

```solidity
// round up: (feeX18 + multiplier - 1) / multiplier
int128 multiplierI = int128(multiplier);
return (feeX18 + multiplierI - 1) / multiplierI;
```

This mirrors the standard "add a `minFundAmount` check" recommendation from the reference report, applied to the withdrawal context.

---

### Proof of Concept

**Setup:** 2-decimal token, `withdrawFeeX18 = 1e15`, `FAST_WITHDRAWAL_FEE_RATE = 1e15`.

```
multiplier          = 10**(18 - 2) = 10**16

// Withdraw 9 raw units (= 0.09 tokens)
amountX18           = 9 * 10**16 = 9e16
proportionalFeeX18  = (1e15 * 9e16) / 1e18 = 9e13
minFeeX18           = 5 * 1e15 = 5e15

feeX18              = max(9e13, 5e15) = 5e15
fee (native)        = 5e15 / 10**16 = 0   ← integer division truncates
```

The user requests 1,000 such withdrawals of 9 raw units each from the sequencer, receives 1,000 signed transactions, and calls `submitFastWithdrawal` 1,000 times. Total withdrawn: 9,000 raw units (90 tokens). Total fee paid: **0**.

    function fastWithdrawalFeeAmount(
