Looking at the external report, the vulnerability class is **incomplete constraint checking**: a function validates one cap/limit but silently ignores a second, independent cap at a different level, causing operations to fail or behave unexpectedly.

I'll map this to `clvm_rs` by examining allocation functions that enforce one resource limit but skip another.

**Key observation in `src/allocator.rs`:**

`new_atom` and `new_concat` both enforce **two** independent limits before writing to `u8_vec`:
1. Atom count limit via `check_atom_limit()`
2. Heap byte limit via `heap_limit`

### Title
`Allocator::new_substr()` checks atom-count limit but ignores heap-byte limit in SmallAtom non-fitting path — (`src/allocator.rs`)

### Summary
`Allocator::new_substr()` enforces the atom-count limit (`check_atom_limit()`) but never checks `heap_limit` before writing bytes to `u8_vec` in the `SmallAtom`-non-fitting branch. Every other allocation path (`new_atom`, `new_concat`, `new_small_number`) checks both limits. An attacker-controlled CLVM program can call `substr` on SmallAtom values whose sub-slice does not fit back into a SmallAtom, repeatedly bypassing the heap cap.

### Finding Description

`new_atom` and `new_concat` guard `u8_vec` writes with:

```rust
if start as usize + self.ghost_heap + v.len() > self.heap_limit {
    return Err(EvalErr::OutOfMemory);
}
```

`new_substr` calls `check_atom_limit()` at the top but has **no equivalent heap guard**. When the input node is a `SmallAtom` and the resulting sub-slice does not satisfy `fits_in_small_atom` (e.g., the slice starts with a high byte such as `0xFF`, making it a negative signed integer that cannot be stored as an unsigned 26-bit index), execution falls into the `else` branch at line ~856 and calls `self.u8_vec.extend_from_slice(substr)` without any prior heap-limit check.

A concrete trigger: a SmallAtom with value `0x03FFFFFF` has bytes `[0x03, 0xFF, 0xFF, 0xFF]`. Taking `substr(1, 4)` yields `[0xFF, 0xFF, 0xFF]`, which is a negative signed integer and will not pass `fits_in_small_atom`. The non-fitting branch then writes 3 bytes to `u8_vec` with no heap check.

### Impact Explanation

When `Allocator::new_limited(heap_limit)` is used (e.g., in mempool mode where `LIMIT_HEAP` is active), a CLVM program can bypass the heap cap by repeatedly calling `substr` on SmallAtoms. Each call leaks up to ~4 bytes past the limit. Bounded by the atom-count limit (`MAX_NUM_ATOMS = 62_500_000`), the total unchecked heap growth can reach tens to hundreds of megabytes beyond the intended cap. This can cause:
- **Out-of-memory crashes** on resource-constrained nodes that rely on the heap cap for safety.
- **Consensus divergence** if nodes with different heap limits accept or reject the same program differently.

### Likelihood Explanation

`op_substr` is a standard, always-available CLVM operator (opcode 12). Any attacker-controlled CLVM program can invoke it with SmallAtom arguments and carefully chosen start/end indices to trigger the non-fitting path. No special flags or permissions are required.

### Recommendation

Add the same heap-limit guard used in `new_atom` and `new_concat` to the `SmallAtom` non-fitting branch of `new_substr`:

```rust
// before: self.u8_vec.extend_from_slice(substr);
let start = self.u8_vec.len();
if start + self.ghost_heap + substr.len() > self.heap_limit {
    return Err(EvalErr::OutOfMemory);
}
let end = start + substr.len();
self.u8_vec.extend_from_slice(substr);
```

### Proof of Concept

1. Create an `Allocator` with a small heap limit via `Allocator::new_limited(N)`.
2. Allocate a SmallAtom whose bytes contain a high byte in a non-leading position, e.g., value `0x03FFFFFF` → bytes `[0x03, 0xFF, 0xFF, 0xFF]`.
3. Call `new_substr(node, 1, 4)` → sub-slice `[0xFF, 0xFF, 0xFF]` fails `fits_in_small_atom` (negative signed integer).
4. Observe that `new_substr` succeeds and writes 3 bytes to `u8_vec` even when `u8_vec.len() + ghost_heap + 3 > heap_limit`.

    pub fn new_substr(&mut self, node: NodePtr, start: u32, end: u32) -> Result<NodePtr> {
        #[cfg(feature = "allocator-debug")]
        self.validate_node(node);

        self.check_atom_limit()?;

        fn bounds_check(node: NodePtr, start: u32, end: u32, len: u32) -> Result<()> {
            if start > len {
                Err(EvalErr::InvalidAllocArg(
                    node,
                    format!("substr start out of bounds: {start} > {len}"),
                ))?;
            }
            if end > len {
                Err(EvalErr::InvalidAllocArg(
                    node,
                    format!("substr end out of bounds: {end} > {len}"),
                ))?;
            }
            if end < start {
                Err(EvalErr::InvalidAllocArg(
                    node,
                    format!("substr invalid bounds: {end} < {start}"),
                ))?;
            }
            Ok(())
        }

        match node.object_type() {
            ObjectType::Pair => Err(EvalErr::InternalError(
                node,
                "substr expected atom, got pair".to_string(),
            ))?,
            ObjectType::Bytes => {
                let atom = self.atom_vec[node.index() as usize];
                let atom_len = atom.end - atom.start;
                bounds_check(node, start, end, atom_len)?;
                let idx = self.atom_vec.len();
                self.atom_vec.push(AtomBuf {
                    start: atom.start + start,
                    end: atom.start + end,
                });
                #[cfg(feature = "counters")]
                self.update_max_counts();
                Ok(self.mk_node(ObjectType::Bytes, idx))
            }
            ObjectType::SmallAtom => {
                let val = node.index();
                let len = len_for_value(val) as u32;
                bounds_check(node, start, end, len)?;
                let buf: [u8; 4] = val.to_be_bytes();
                let buf = &buf[4 - len as usize..];
                let substr = &buf[start as usize..end as usize];
                if let Some(new_val) = fits_in_small_atom(substr) {
                    self.ghost_atoms += 1;
                    Ok(self.mk_node(ObjectType::SmallAtom, new_val as usize))
                } else {
                    let start = self.u8_vec.len();
                    let end = start + substr.len();
                    self.u8_vec.extend_from_slice(substr);
                    let idx = self.atom_vec.len();
                    self.atom_vec.push(AtomBuf {
                        start: start as u32,
                        end: end as u32,
                    });
                    #[cfg(feature = "counters")]
                    self.update_max_counts();
                    Ok(self.mk_node(ObjectType::Bytes, idx))
                }
            }
        }
    }
