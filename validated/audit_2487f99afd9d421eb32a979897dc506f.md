### Title
`ValidResourceBounds` variant silently collapsed from `AllResources` to `L1Gas` during protobuf deserialization, producing a divergent transaction hash - (File: `crates/apollo_protobuf/src/converters/transaction.rs`)

### Summary
The `TryFrom<protobuf::ResourceBounds> for ValidResourceBounds` implementation in the P2P-sync transaction converter silently defaults the absent `l1_data_gas` field to zero and then uses a value-based heuristic to choose between `ValidResourceBounds::L1Gas` and `ValidResourceBounds::AllResources`. Any `AllResources` transaction whose `l1_data_gas` and `l2_gas` bounds are both zero is reconstructed as `L1Gas` after a protobuf round-trip. Because `get_tip_resource_bounds_hash` hashes a different number of resource felts for the two variants, the recomputed transaction hash diverges from the canonical hash that was committed to the block, and the gas-vector computation mode switches from `All` to `NoL2Gas`, producing wrong fee/execution results.

### Finding Description

**Root cause – protobuf deserializer, `transaction.rs`:**

```rust
// crates/apollo_protobuf/src/converters/transaction.rs  lines 417-436
impl TryFrom<protobuf::ResourceBounds> for ValidResourceBounds {
    fn try_from(value: protobuf::ResourceBounds) -> Result<Self, Self::Error> {
        let Some(l1_gas) = value.l1_gas else { … };
        let Some(l2_gas) = value.l2_gas else { … };
        // TODO(Shahak): Assert data gas is not none once we remove support for 0.13.2.
        let l1_data_gas = value.l1_data_gas.unwrap_or_default();   // ← silent default
        …
        Ok(if l1_data_gas.is_zero() && l2_gas.is_zero() {
            ValidResourceBounds::L1Gas(l1_gas)                      // ← wrong variant
        } else {
            ValidResourceBounds::AllResources(…)
        })
    }
}
```

A transaction submitted via RPC carries `AllResourceBounds` (all three fields present, even when zero). When serialised to protobuf the `From<ValidResourceBounds> for protobuf::ResourceBounds` implementation always emits an explicit `l1_data_gas = Some(zero)`:

```rust
// lines 471-489
ValidResourceBounds::AllResources(AllResourceBounds { l1_gas, l2_gas, l1_data_gas }) =>
    protobuf::ResourceBounds {
        l1_gas: Some(l1_gas.into()),
        l2_gas: Some(l2_gas.into()),
        l1_data_gas: Some(l1_data_gas.into()),   // explicit zero
    },
```

On deserialisation `unwrap_or_default()` yields zero for `l1_data_gas`, and since `l2_gas` is also zero the heuristic fires and returns `L1Gas` — discarding the `AllResources` variant that was originally committed.
