Audit Report

## Title
SwapAllowlistExtension Checks Router Address Instead of Original User, Allowing Any User to Bypass the Swap Allowlist via MetricOmmSimpleRouter - (File: metric-periphery/contracts/extensions/SwapAllowlistExtension.sol)

## Summary

`SwapAllowlistExtension.beforeSwap()` gates swaps by checking the `sender` argument, which is the direct `msg.sender` of `pool.swap()`. When `MetricOmmSimpleRouter` intermediates a swap, `sender` becomes the router's address rather than the original user's address. A pool admin who allowlists the router to enable router-based swaps inadvertently opens the allowlist to every user on the network, as any user routing through the allowlisted router will pass the check regardless of their individual allowlist status.

## Finding Description

`MetricOmmPool.swap()` passes `msg.sender` verbatim as the `sender` argument to `_beforeSwap`: [1](#0-0) 

`ExtensionCalling._beforeSwap()` encodes that `sender` into the call to each configured extension: [2](#0-1) 

`SwapAllowlistExtension.beforeSwap()` then checks `allowedSwapper[msg.sender][sender]`, where `msg.sender` is the pool and `sender` is whoever called `pool.swap()`: [3](#0-2) 

`MetricOmmSimpleRouter.exactInputSingle()` calls `pool.swap()` directly, making the router the `msg.sender` of that call — the original user's address (`msg.sender` at the router level) is never forwarded to the pool: [4](#0-3) 

The same pattern holds for `exactInput` (all hops after the first use `address(this)` as payer): [5](#0-4) 

And for the recursive `exactOutput` callback path: [6](#0-5) 

Consequently, the extension never sees the original user's address — it sees the router's address. A pool admin who wants router-based swaps to work must allowlist the router. Once the router is allowlisted, `allowedSwapper[pool][router] == true` passes for every user who routes through it, regardless of whether that user is individually allowlisted.

## Impact Explanation

`SwapAllowlistExtension` is the production mechanism for restricting swap access to specific counterparties (e.g., KYC-gated pools, institutional-only pools, or pools that must exclude toxic-flow sources). Bypassing it lets any unprivileged user execute swaps against the pool's LP liquidity. LPs who deposited under the assumption that only vetted counterparties could trade against them are exposed to unrestricted toxic flow, MEV, and arbitrage from actors the admin explicitly intended to exclude. This is a direct admin-boundary break reachable by any public user with no special privileges, matching the allowed impact gate: "Admin-boundary break: factory/oracle role checks are bypassed by an unprivileged path."

## Likelihood Explanation

The bypass requires only that the pool admin has allowlisted the router — a necessary step for the router to be usable at all on that pool. Any user who discovers the pool is router-accessible can immediately route through `MetricOmmSimpleRouter` to bypass the allowlist. No privileged access, no special tokens, and no malicious setup are required. The condition (router allowlisted) is a standard operational state for any pool that supports router-based trading.

## Recommendation

The extension must gate on the original user's identity, not the intermediary's. Two viable approaches:

1. **Pass the original payer through `extensionData`**: The router encodes `msg.sender` (the original user) into `extensionData` before calling `pool.swap()`. The extension decodes and checks that address. This requires a convention between the router and the extension.

2. **Introduce a dedicated `originalSender` field** in the `beforeSwap` hook signature so the pool or router can propagate the true initiator without relying on opaque `extensionData`.

3. **Check `recipient` instead of `sender` for the allowlist**: If the pool's intent is to restrict who *receives* output, `recipient` is already the original user. For input-side gating, approach 1 or 2 is required.

## Proof of Concept

```
Setup:
  - Pool deployed with SwapAllowlistExtension
  - Admin allowlists alice (KYC'd user): allowedSwapper[pool][alice] = true
  - Admin allowlists router (to enable router swaps): allowedSwapper[pool][router] = true
  - bob is NOT allowlisted: allowedSwapper[pool][bob] = false

Direct swap attempt by bob:
  bob → pool.swap(...)
    sender = bob
    extension checks allowedSwapper[pool][bob] → false → revert NotAllowedToSwap ✓

Router swap attempt by bob:
  bob → router.exactInputSingle({pool: pool, ...})
    router → pool.swap(...)
      sender = router
      extension checks allowedSwapper[pool][router] → true → PASSES ✗

Result: bob successfully swaps against the restricted pool, bypassing the allowlist entirely.
```
