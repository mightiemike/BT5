### Title
`ValidResourceBounds::L1Gas` vs `AllResources` hash divergence for zero-valued V3 resource bounds in protobuf conversion — (File: `crates/apollo_protobuf/src/converters/transaction.rs`)

### Summary

The protobuf-to-`ValidResourceBounds` conversion silently downgrades a V3 `AllResources` transaction to the pre-0.13.3 `L1Gas` variant whenever `l2_gas = 0` and `l1_data_gas` is absent or zero. Because `get_tip_resource_bounds_hash` hashes a different number of resource fields for each variant (2 for `L1Gas`, 3 for `AllResources`), the transaction hash computed during P2P historical sync diverges from the hash computed at RPC admission time. A valid V3 transaction with zero `l2_gas` and zero `l1_data_gas` therefore fails hash validation on every syncing peer, or — if hash validation is skipped — is executed under the wrong gas-accounting mode.

### Finding Description

**Step 1 — Admission (RPC path, `AllResources`).**

When a V3 invoke transaction arrives at the gateway, `convert_rpc_tx_to_internal` converts `RpcInvokeTransactionV3` (which carries `resource_bounds: AllResourceBounds`) into `InternalRpcTransactionWithoutTxHash::Invoke`. The hash is then computed via `InternalRpcTransactionWithoutTxHash::calculate_transaction_hash`, which calls `get_invoke_transaction_v3_hash` → `get_tip_resource_bounds_hash` with `ValidResourceBounds::AllResources`. That function hashes **three** resource felts: `L1_GAS`, `L2_GAS`, and `L1_DATA_GAS` (even when `l1_data_gas = 0`):

```rust
// crates/starknet_api/src/transaction_hash.rs  lines 203-208
resource_felts.extend(match resource_bounds {
    ValidResourceBounds::L1Gas(_) => vec![],
    ValidResourceBounds::AllResources(all_resources) => {
        vec![get_concat_resource(&all_resources.l1_data_gas, L1_DATA_GAS)?]
    }
});
```

Call this hash **H₁**.

**Step 2 — Historical P2P sync (protobuf path, `L1Gas`).**

When the same transaction is later received over P2P as a `Transaction::Invoke(InvokeTransaction::V3(...))`, the resource bounds are decoded by:

```rust
// crates/apollo_protobuf/src/converters/transaction.rs  lines 427-435
// TODO(Shahak): Assert data gas is not none once we remove support for 0.13.2.
let l1_data_gas = value.l1_data_gas.unwrap_or_default();
...
Ok(if l1_data_gas.is_zero() && l2_gas.is_zero() {
    ValidResourceBounds::L1Gas(l1_gas)          // ← chosen when both are zero
} else {
    ValidResourceBounds::AllResources(...)
})
```

Because `l1_data_gas` is absent (or zero) and `l2_gas` is zero, the conversion produces `ValidResourceBounds::L1Gas`. `get_tip_resource_bounds_hash` now hashes only **two** resource felts (`L1_GAS`, `L2_GAS`), omitting `L1_DATA_GAS`. Call this hash **H₂**.

**H₁ ≠ H₂** for any transaction where `l1_data_gas = 0` and `l2_gas = 0`, because the Poseidon hash over two elements differs from the Poseidon hash over three elements even when the third element is zero.

**Step 3 — Divergence.**

`validate_transaction_hash` (or any peer that recomputes the hash) will compute H₂ and compare it against the stored/announced H₁. The mismatch causes the block or transaction to be rejected. Alternatively, if the hash is trusted from the wire, the transaction is executed under `GasVectorComputationMode::NoL2Gas` instead of `All`, producing wrong fee accounting and wrong execution receipts.

### Impact Explanation

Every V3 transaction whose `l2_gas` and `l1_data_gas` are both zero (a common configuration — many real mainnet transactions set only `l2_gas` non-zero, leaving `l1_data_gas = 0`) will have its hash recomputed incorrectly during historical P2P sync. This either:

- **Rejects valid transactions / blocks** during sync (High: RPC/gateway rejects valid transactions), or
- **Produces wrong execution state** if the hash is not re-validated (Critical: wrong receipt, fee, or gas accounting from blockifier execution logic).

### Likelihood Explanation

The trigger condition (`l2_gas = 0` and `l1_data_gas = 0`) is satisfied by any pre-0.13.3-style V3 transaction and by any 0.13.3+ transaction that sets only `l1_gas`. The `unwrap_or_default()` on `l1_data_gas` is explicitly guarded by a `TODO` acknowledging it is not yet enforced. Any peer running a version that sends `l1_data_gas = None` in the protobuf will trigger this on the receiving side.

### Recommendation

1. **Never produce `ValidResourceBounds::L1Gas` from a V3 protobuf message.** The `L1Gas` variant is a pre-0.13.3 representation; V3 transactions must always deserialize to `AllResources`, even when all three bounds are zero. Remove the `l1_data_gas.is_zero() && l2_gas.is_zero()` downgrade branch from `TryFrom<protobuf::ResourceBounds> for ValidResourceBounds` for the V3 context, or add a version-aware dispatch.

2. **Enforce the TODO.** Once 0.13.2 support is dropped, assert `l1_data_gas` is present and error on absence, eliminating the `unwrap_or_default()` silent default.

3. **Add a round-trip hash test** that submits a V3 transaction with `l2_gas = 0` and `l1_data_gas = 0` via the RPC path, serializes it to protobuf, deserializes it, and asserts the recomputed hash equals the original.

### Proof of Concept

```
1. Submit via RPC:
   InvokeV3 { l1_gas: {max_amount: 1000, max_price: 1}, l2_gas: {0,0}, l1_data_gas: {0,0}, ... }

2. Gateway computes hash H₁ using AllResources (3 resource felts):
   tip_resource_bounds_hash = poseidon(tip, concat(L1_GAS,1000,1), concat(L2_GAS,0,0), concat(L1_DATA,0,0))

3. Transaction included in block N with hash H₁.

4. Peer syncs block N via P2P. Protobuf ResourceBounds arrives with l1_data_gas = None.
   Conversion: l1_data_gas = default() = {0,0}; l2_gas = {0,0}
   → l1_data_gas.is_zero() && l2_gas.is_zero() == true
   → ValidResourceBounds::L1Gas(l1_gas)

5. Peer recomputes hash H₂ using L1Gas (2 resource felts):
   tip_resource_bounds_hash = poseidon(tip, concat(L1_GAS,1000,1), concat(L2_GAS,0,0))
   // L1_DATA_GAS felt is absent

6. H₁ ≠ H₂  →  hash validation fails  →  block N rejected on syncing peer.
```

pub fn get_tip_resource_bounds_hash(
    resource_bounds: &ValidResourceBounds,
    tip: &Tip,
) -> Result<Felt, StarknetApiError> {
    let l1_resource_bounds = resource_bounds.get_l1_bounds();
    let l2_resource_bounds = resource_bounds.get_l2_bounds();

    // L1 and L2 gas bounds always exist.
    // Old V3 txs always have L2 gas bounds of zero, but they exist.
    let mut resource_felts = vec![
        get_concat_resource(&l1_resource_bounds, L1_GAS)?,
        get_concat_resource(&l2_resource_bounds, L2_GAS)?,
    ];

    // For new V3 txs, need to also hash the data gas bounds.
    resource_felts.extend(match resource_bounds {
        ValidResourceBounds::L1Gas(_) => vec![],
        ValidResourceBounds::AllResources(all_resources) => {
            vec![get_concat_resource(&all_resources.l1_data_gas, L1_DATA_GAS)?]
        }
    });

    Ok(HashChain::new().chain(&tip.0.into()).chain_iter(resource_felts.iter()).get_poseidon_hash())
}

    transaction: &T,
    chain_id: &ChainId,
    transaction_version: &TransactionVersion,
) -> Result<TransactionHash, StarknetApiError> {
    let tip_resource_bounds_hash =
        get_tip_resource_bounds_hash(&transaction.resource_bounds(), transaction.tip())?;
    let paymaster_data_hash =
        HashChain::new().chain_iter(transaction.paymaster_data().0.iter()).get_poseidon_hash();
    let data_availability_mode = concat_data_availability_mode(
        transaction.nonce_data_availability_mode(),
        transaction.fee_data_availability_mode(),
    );
    let account_deployment_data_hash = HashChain::new()
        .chain_iter(transaction.account_deployment_data().0.iter())
        .get_poseidon_hash();
    let calldata_hash =
        HashChain::new().chain_iter(transaction.calldata().0.iter()).get_poseidon_hash();
    let mut hash_chain = HashChain::new()
        .chain(&INVOKE)
        .chain(&transaction_version.0)
        .chain(transaction.sender_address().0.key())
        .chain(&tip_resource_bounds_hash)
        .chain(&paymaster_data_hash)
        .chain(&Felt::try_from(chain_id)?)
        .chain(&transaction.nonce().0)
        .chain(&data_availability_mode)
        .chain(&account_deployment_data_hash)
        .chain(&calldata_hash);

        let proof_facts_hash =
            HashChain::new().chain_iter(transaction.proof_facts().0.iter()).get_poseidon_hash();

    }

#[derive(Clone, Copy, Debug, Eq, PartialEq, Hash, Ord, PartialOrd)]
pub enum ValidResourceBounds {
    L1Gas(ResourceBounds), // Pre 0.13.3. Only L1 gas. L2 bounds are signed but never used.
    AllResources(AllResourceBounds),
}

impl From<AllResourceBounds> for ValidResourceBounds {
    fn from(value: AllResourceBounds) -> Self {
        Self::AllResources(value)
    }
}

impl ValidResourceBounds {
    pub fn get_l1_bounds(&self) -> ResourceBounds {
        match self {
            Self::L1Gas(l1_bounds) => *l1_bounds,
            Self::AllResources(AllResourceBounds { l1_gas, .. }) => *l1_gas,
        }
    }

    pub fn get_l2_bounds(&self) -> ResourceBounds {
        match self {
            Self::L1Gas(_) => ResourceBounds::default(),
            Self::AllResources(AllResourceBounds { l2_gas, .. }) => *l2_gas,
        }
    }

    /// Returns the maximum possible fee that can be charged for the transaction.
    /// The computation is saturating, meaning that if the result is larger than the maximum
    /// possible fee, the maximum possible fee is returned.
    pub fn max_possible_fee(&self, tip: Tip) -> Fee {
        match self {
            ValidResourceBounds::L1Gas(l1_bounds) => {
                l1_bounds.max_amount.saturating_mul(l1_bounds.max_price_per_unit)
            }
            ValidResourceBounds::AllResources(AllResourceBounds {
                l1_gas,
                l2_gas,
                l1_data_gas,
            }) => l1_gas
                .max_amount
                .saturating_mul(l1_gas.max_price_per_unit)
                .saturating_add(
                    l2_gas
                        .max_amount
                        .saturating_mul(l2_gas.max_price_per_unit.saturating_add(tip.into())),
                )
                .saturating_add(
                    l1_data_gas.max_amount.saturating_mul(l1_data_gas.max_price_per_unit),
                ),
        }
    }

    pub fn get_gas_vector_computation_mode(&self) -> GasVectorComputationMode {
        match self {
            Self::AllResources(_) => GasVectorComputationMode::All,
            Self::L1Gas(_) => GasVectorComputationMode::NoL2Gas,
        }
    }
