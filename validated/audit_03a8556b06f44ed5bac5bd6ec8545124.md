### Title
`gateway_dynamic_config` Is Stored in `NodeDynamicConfig` but Has No Getter in `ConfigManagerRequest`, Making Gateway Runtime Config Updates Permanently Unreachable — (`crates/apollo_config_manager/src/config_manager.rs`, `crates/apollo_config_manager_types/src/communication.rs`, `crates/apollo_node_config/src/node_config.rs`)

### Summary

`NodeDynamicConfig` carries a `gateway_dynamic_config` field that is populated from the live config file and written into the `ConfigManager` store via `SetNodeDynamicConfig`. However, `ConfigManagerRequest` has no `GetGatewayDynamicConfig` variant, and `ConfigManager` has no corresponding `get_gateway_dynamic_config` method. The gateway component therefore has no mechanism to retrieve its updated dynamic config at runtime. Any operator-initiated change to gateway validation parameters is silently accepted by the config manager but never delivered to the gateway, leaving the gateway permanently frozen at its startup configuration.

### Finding Description

`NodeDynamicConfig` declares nine optional sub-configs, one of which is `gateway_dynamic_config`:

```rust
pub struct NodeDynamicConfig {
    pub batcher_dynamic_config: Option<BatcherDynamicConfig>,
    pub class_manager_dynamic_config: Option<ClassManagerDynamicConfig>,
    pub consensus_dynamic_config: Option<ConsensusDynamicConfig>,
    pub context_dynamic_config: Option<ContextDynamicConfig>,
    pub gateway_dynamic_config: Option<GatewayDynamicConfig>,   // ← present
    pub http_server_dynamic_config: Option<HttpServerDynamicConfig>,
    pub mempool_dynamic_config: Option<MempoolDynamicConfig>,
    pub staking_manager_dynamic_config: Option<StakingManagerDynamicConfig>,
    pub state_sync_dynamic_config: Option<StateSyncDynamicConfig>,
}
``` [1](#0-0) 

`NodeDynamicConfig::from(&SequencerNodeConfig)` explicitly extracts and stores `gateway_dynamic_config`:

```rust
let gateway_dynamic_config = sequencer_node_config
    .gateway_config
    .as_ref()
    .map(|gateway_config| gateway_config.dynamic_config.clone());
``` [2](#0-1) 

`ConfigManagerRunner::update_config` reads the config file, constructs a `NodeDynamicConfig` (including `gateway_dynamic_config`), and calls `set_node_dynamic_config` on the `ConfigManager`: [3](#0-2) 

`ConfigManager::set_node_dynamic_config` stores the entire `NodeDynamicConfig` (including `gateway_dynamic_config`) in its `RwLock`: [4](#0-3) 

However, `ConfigManagerRequest` exposes getters for every other sub-config **except** the gateway:

```rust
pub enum ConfigManagerRequest {
    GetConsensusDynamicConfig,
    GetClassManagerDynamicConfig,
    GetContextDynamicConfig,
    GetHttpServerDynamicConfig,
    GetMempoolDynamicConfig,
    GetBatcherDynamicConfig,
    GetStateSyncDynamicConfig,
    GetStakingManagerDynamicConfig,
    SetNodeDynamicConfig(Box<NodeDynamicConfig>),
    // ← No GetGatewayDynamicConfig
}
``` [5](#0-4) 

`ConfigManager::handle_request` dispatches eight getters but has no arm for the gateway: [6](#0-5) 

There is also no `get_gateway_dynamic_config` method on `ConfigManager`: [7](#0-6) 

The structural invariant is broken: `gateway_dynamic_config` is writable (via `SetNodeDynamicConfig`) but permanently unreadable through the `ConfigManager` API.

### Impact Explanation

The gateway is the admission layer for all user-submitted transactions. Its `GatewayDynamicConfig` (sourced from `gateway_config.dynamic_config`) governs runtime-adjustable validation parameters — including resource-bounds enforcement, nonce-gap limits, and declare-transaction rejection policy — that directly control which transactions are admitted or rejected before sequencing. Because `GetGatewayDynamicConfig` does not exist in `ConfigManagerRequest`, the gateway component can never poll for an updated config. Any operator change to these parameters (e.g., tightening resource-bound validation to block spam, or relaxing nonce-gap limits during a bootstrap phase) is silently swallowed by the config manager and never delivered. The gateway continues to enforce its startup-time parameters indefinitely, causing it to either accept transactions it should reject or reject transactions it should accept, depending on the direction of the intended change.

**Impact: High — Mempool/gateway admission accepts invalid transactions or rejects valid transactions before sequencing.**

### Likelihood Explanation

The `ConfigManagerRunner` is designed to watch config files for changes and push updates automatically. Any operator who edits the gateway section of the config file will observe no error and no warning — the update is accepted by `set_node_dynamic_config` — but the gateway silently continues with its old parameters. The trigger requires only a config file edit, which is a normal operational action. The omission is structural and cannot be worked around without a code change.

### Recommendation

Add `GetGatewayDynamicConfig` to `ConfigManagerRequest` and `ConfigManagerResponse`, implement `get_gateway_dynamic_config` on `ConfigManager` (mirroring the pattern of the eight existing getters), and add the corresponding arm to `handle_config_request!`. The gateway component should then call this endpoint on the same polling/notification cycle used by the other components.

### Proof of Concept

1. `NodeDynamicConfig` contains `gateway_dynamic_config` and `From<&SequencerNodeConfig>` populates it. [8](#0-7) 

2. `ConfigManagerRunner::update_config` calls `set_node_dynamic_config` with the full `NodeDynamicConfig`, including the gateway field. [9](#0-8) 

3. `ConfigManager::set_node_dynamic_config` stores it without error. [4](#0-3) 

4. Search `ConfigManagerRequest` for `GetGatewayDynamicConfig` — it does not exist. [10](#0-9) 

5. Search `ConfigManager` for `get_gateway_dynamic_config` — it does not exist. [7](#0-6) 

6. The gateway component therefore has no callable path to retrieve its updated dynamic config, leaving it frozen at startup values regardless of subsequent config file changes.

    pub(crate) async fn update_config(
        &mut self,
    ) -> Result<NodeDynamicConfig, Box<dyn std::error::Error + Send + Sync>> {
        let config = load_and_validate_config(self.cli_args.clone(), false).map_err(|e| {
            CONFIG_MANAGER_UPDATE_ERRORS.increment(1);
            error!("ConfigManagerRunner: failed to update config: {e}");
            e
        })?;
        let node_dynamic_config = NodeDynamicConfig::from(&config);

        // Compare the previous and the newly read node dynamic config.
        if self.latest_node_dynamic_config == node_dynamic_config {
            // No change, so no action is needed.
            Ok(node_dynamic_config)
        } else {
            // Log the diff between the latest and the new node dynamic config.
            self.log_config_diff(&self.latest_node_dynamic_config, &node_dynamic_config);
            // Update the latest node dynamic config.
            self.latest_node_dynamic_config = node_dynamic_config.clone();
            match self
                .config_manager_client
                .set_node_dynamic_config(node_dynamic_config.clone())
                .await
            {
                Ok(()) => {
                    info!("Successfully updated dynamic config");
                    Ok(node_dynamic_config)
                }
                Err(e) => Err(format!("Failed to update dynamic config: {:?}", e).into()),
            }
        }
    }

    pub(crate) async fn set_node_dynamic_config(
        &self,
        node_dynamic_config: NodeDynamicConfig,
    ) -> ConfigManagerResult<()> {
        info!("ConfigManager: updating node dynamic config");
        if let Err(errors) = node_dynamic_config.validate() {
            error!(
                "ConfigManager: dynamic config update rejected: {errors}. Keeping previous config."
            );
            return Err(ConfigManagerError::InvalidConfig(errors.to_string()));
        }
        let mut config = self.latest_node_dynamic_config.write().await;
        *config = node_dynamic_config;
        Ok(())
    }
