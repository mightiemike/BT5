### Title
Silent Checkpoint Advancement on `ErrInsufficientBonusPool` in `slashRedelegationPosition` Permanently Forfeits Accrued Bonus Rewards — (`x/tieredrewards/keeper/slash.go`)

### Summary

`slashRedelegationPosition` calls `processEventsAndClaimBonus` with a pointer to `pos`. That function advances `pos.LastEventSeq`, `pos.LastBonusAccrual`, and `pos.LastKnownBonded` **before** it checks pool sufficiency. When the pool is empty, `ErrInsufficientBonusPool` is returned with all checkpoints already mutated. `slashRedelegationPosition` swallows that error and immediately calls `setPositionWithState` with the mutated `pos`, persisting the advanced checkpoints. The accrued bonus for the pre-slash period is permanently unclaimable.

### Finding Description

In `processEventsAndClaimBonus`, the event loop runs first:

- `pos.UpdateLastEventSeq(entry.Seq)` is called per event [1](#0-0) 
- `k.decrementEventRefCount` is called per event, consuming the reference [2](#0-1) 

After the loop, checkpoints are advanced unconditionally:

- `applyBonusAccrualCheckpoint(&pos.Position, blockTime)` advances `LastBonusAccrual` [3](#0-2) 
- `pos.UpdateLastKnownBonded(bonded)` updates bonded state [4](#0-3) 

Only **then** is pool sufficiency checked:

- `k.sufficientBonusPoolBalance(ctx, bonusCoins)` — if this fails, `ErrInsufficientBonusPool` is returned with all in-memory mutations already applied [5](#0-4) 

Back in `slashRedelegationPosition`, the error is explicitly swallowed:

```go
if _, err := k.processEventsAndClaimBonus(ctx, &pos); err != nil {
    if errors.Is(err, types.ErrInsufficientBonusPool) {
        k.logger(ctx).Error("insufficient bonus pool during redelegation slash", ...)
    } else {
        return err
    }
}
``` [6](#0-5) 

Then `setPositionWithState` is called with the already-mutated `pos`, persisting the advanced checkpoints: [7](#0-6) 

The position is passed as `&pos` (pointer), so all mutations from `processEventsAndClaimBonus` are visible to the caller and then written to the store. [8](#0-7) 

### Impact Explanation

After this path executes:

1. `pos.LastBonusAccrual` is advanced to the slash block time.
2. `pos.LastEventSeq` is advanced past all events that covered the accrual period.
3. Event reference counts are decremented, potentially allowing those events to be garbage-collected.
4. No bonus coins are transferred to the owner.
5. When the pool is later replenished and the owner calls `MsgClaimTierRewards`, `processEventsAndClaimBonus` starts from the already-advanced `LastBonusAccrual` and `LastEventSeq`, so the pre-slash accrual period is never revisited. The bonus for that period is permanently lost with no on-chain record.

This violates the invariant stated in the code's own comment: bonus rewards owed to a position owner must either be paid or remain claimable.

### Likelihood Explanation

The bonus pool is a finite module account (`RewardsPoolName`). It can be empty or near-empty at any block. A redelegation slash is a normal staking event triggered by evidence submission — no attacker control is needed. Any position owner whose redelegation is slashed while the pool is empty loses their accrued bonus permanently. This is a realistic, non-adversarial scenario.

### Recommendation

Do not advance checkpoints when the bonus cannot be paid. The fix is to restructure `processEventsAndClaimBonus` so that `applyBonusAccrualCheckpoint`, `UpdateLastEventSeq`, `UpdateLastKnownBonded`, and `decrementEventRefCount` are only applied **after** `sufficientBonusPoolBalance` and `SendCoinsFromModuleToAccount` succeed. Alternatively, `slashRedelegationPosition` should restore the original checkpoint values from a snapshot taken before calling `processEventsAndClaimBonus` when `ErrInsufficientBonusPool` is returned, so the owner can claim later.

### Proof of Concept

```go
func (s *KeeperSuite) TestSlashRedelegationPosition_EmptyPool_ForfeitsBonus() {
    lockAmount := sdkmath.NewInt(sdk.DefaultPowerReduction.Int64())
    _, bondDenom := s.getStakingData()
    // Fund pool, then drain it to zero
    s.fundRewardsPool(sdkmath.NewInt(1_000_000_000), bondDenom)
    // ... drain pool ...

    pos, _, unbondingID := s.setupRedelegatingPosition(lockAmount)
    owner := sdk.MustAccAddressFromBech32(pos.Owner)
    preAccrual := pos.LastBonusAccrual
    preSeq := pos.LastEventSeq

    s.ctx = s.ctx.WithBlockTime(s.ctx.BlockTime().Add(30 * 24 * time.Hour))

    // Fire slash with empty pool — should NOT advance checkpoints
    err := s.keeper.Hooks().BeforeRedelegationSlashed(s.ctx, unbondingID, pos.Delegation.Shares.Quo(sdkmath.LegacyNewDec(2)))
    s.Require().NoError(err)

    updated, _ := s.keeper.GetPositionState(s.ctx, pos.Id)
    // BUG: these assertions FAIL — checkpoints are advanced despite no payment
    s.Require().Equal(preAccrual, updated.LastBonusAccrual, "checkpoint must not advance when bonus not paid")
    s.Require().Equal(preSeq, updated.LastEventSeq, "event seq must not advance when bonus not paid")

    // Replenish pool and claim — owner gets zero bonus for the pre-slash period
    s.fundRewardsPool(sdkmath.NewInt(1_000_000_000), bondDenom)
    balBefore := s.app.BankKeeper.GetBalance(s.ctx, owner, bondDenom)
    // MsgClaimTierRewards ...
    balAfter := s.app.BankKeeper.GetBalance(s.ctx, owner, bondDenom)
    s.Require().True(balAfter.Amount.GT(balBefore.Amount), "owner must recover pre-slash bonus after pool replenishment")
}
```
