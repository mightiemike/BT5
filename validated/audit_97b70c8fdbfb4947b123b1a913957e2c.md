### Title
Non-bond-denom tokens sent to `RewardsPoolName` module account are permanently locked with no rescue path - (`x/tieredrewards/keeper/abci.go`, `app/app.go`)

### Summary

The `rewards_pool` module account is explicitly whitelisted to receive external bank sends of any denomination. However, the only outflow paths from the pool are bond-denom-only: `BeginBlocker` drains bond-denom to the distribution module, and bonus reward claims pay out bond-denom to position owners. No governance message exists to rescue non-bond-denom tokens accidentally sent to the pool. Those tokens are permanently locked.

### Finding Description

`app/app.go` registers `tieredrewardstypes.RewardsPoolName` in `moduleAccsAllowedToReceiveExternalFunds` with value `true`, bypassing the Cosmos SDK bank module's default block on sends to module accounts for any denomination: [1](#0-0) 

This means any user can execute a standard `MsgSend` targeting the `rewards_pool` module address with any token (IBC vouchers, other native denoms, etc.).

The `BeginBlocker` (`topUpBaseRewards`) only reads and drains the bond-denom balance: [2](#0-1) 

The bonus reward path (`sufficientBonusPoolBalance`) checks `GetAllBalances` for sufficiency but the bonus coins are always bond-denom, so non-bond-denom tokens in the pool are never touched: [3](#0-2) 

Inspecting the full message server (`msg_server.go`) and the governance messages (`MsgAddTier`, `MsgUpdateTier`, `MsgDeleteTier`, `MsgUpdateParams`), there is no `MsgRescueRewardsPool` or equivalent governance message that would allow withdrawing arbitrary denoms from the pool: [4](#0-3) 

### Impact Explanation

Any non-bond-denom tokens sent to the `rewards_pool` module account are permanently locked. There is no on-chain path — neither user-driven nor governance-driven — to recover them. Recovery would require a chain upgrade. The exact corrupted value is the balance of non-bond-denom coins at the `rewards_pool` module account address.

### Likelihood Explanation

The entry path is a standard `MsgSend` transaction, executable by any unprivileged account. The `rewards_pool` address is publicly derivable (SHA-256 of the module name). A user intending to fund the pool with bond-denom but accidentally specifying the wrong denom, or a user accidentally sending IBC tokens to the pool address instead of their intended recipient, would trigger this. Likelihood is low but the entry path requires no privilege and no special conditions.

### Recommendation

Add a governance-gated message (e.g., `MsgRescueRewardsPool`) that allows the governance authority to withdraw a specified amount of a specified denom from the `rewards_pool` module account, subject to the constraint that it must not withdraw bond-denom in excess of what is needed for outstanding bonus obligations (analogous to the NounsDAO `rescueERC20` fix that checks `amount < tokenBalance() - remainingBalance`). At minimum, allow rescue of any denom that is not the bond denom unconditionally.

### Proof of Concept

1. Derive the `rewards_pool` module account address (SHA-256 of `"rewards_pool"`, first 20 bytes, bech32-encoded with `"cro"` prefix — as done in the integration test helpers): [5](#0-4) 

2. Submit a `MsgSend` from any account sending, e.g., an IBC voucher denom to that address. The send succeeds because `moduleAccsAllowedToReceiveExternalFunds["rewards_pool"] = true`. [6](#0-5) 

3. Observe that `BeginBlocker` never touches the non-bond-denom balance: [7](#0-6) 

4. Confirm no governance message exists to withdraw the tokens. The `QueryRewardsPoolBalances` query will show the non-bond-denom balance sitting in the pool indefinitely: [8](#0-7)

func (k Keeper) sufficientBonusPoolBalance(ctx context.Context, bonus sdk.Coins) error {
	if bonus.IsZero() {
		return nil
	}

	poolAddr := k.accountKeeper.GetModuleAddress(types.RewardsPoolName)
	poolBalance := k.bankKeeper.GetAllBalances(ctx, poolAddr)
	if !poolBalance.IsAllGTE(bonus) {
		return errorsmod.Wrapf(types.ErrInsufficientBonusPool,
			"bonus: %s, pool balance: %s",
			bonus.String(), poolBalance.String())
	}

	return nil
```

def module_address(name):
    """
    get address of module accounts

    :param name: name of module account, values: {options}
    """
    data = hashlib.sha256(ModuleAccount(name).value.encode()).digest()[:20]
    return bech32.bech32_encode("cro", bech32.convertbits(data, 8, 5))
```
