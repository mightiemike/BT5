### Title
`SwapAllowlistExtension` gates the router address instead of the end user, allowing full allowlist bypass via the standard periphery path â€” (`File: metric-periphery/contracts/extensions/SwapAllowlistExtension.sol`)

---

### Summary

`SwapAllowlistExtension.beforeSwap` checks the `sender` argument, which the pool sets to `msg.sender` of the `swap()` call. When a user routes through `MetricOmmSimpleRouter`, `msg.sender` at the pool is the **router contract**, not the end user. Any pool admin who allowlists the router to support router-mediated swaps for their curated users inadvertently opens the pool to every user on-chain.

---

### Finding Description

`MetricOmmPool.swap` passes `msg.sender` as the `sender` argument to `_beforeSwap`: [1](#0-0) 

`ExtensionCalling._beforeSwap` forwards that value unchanged to every configured extension: [2](#0-1) 

`SwapAllowlistExtension.beforeSwap` then checks whether that `sender` is on the allowlist for the calling pool (`msg.sender` of the extension call = the pool): [3](#0-2) 

When a user calls `MetricOmmSimpleRouter.exactInputSingle` (or `exactInput` / `exactOutput`), the router is the direct caller of `pool.swap()`: [4](#0-3) 

So `sender` delivered to the extension is the **router address**, not the end user. The allowlist check becomes `allowedSwapper[pool][router]`, not `allowedSwapper[pool][user]`.

This creates two mutually exclusive broken states:

1. **Router not allowlisted** â€” allowlisted users cannot use the router at all; their swaps revert with `NotAllowedToSwap` even though they are individually permitted.
2. **Router allowlisted** (the natural fix a pool admin would apply) â€” every address on-chain can swap through the router, completely defeating the allowlist.

The same structural problem exists for multi-hop `exactInput` (intermediate hops also call `pool.swap()` from the router) and `exactOutput` (recursive callback calls `pool.swap()` from the router): [5](#0-4) [6](#0-5) 

---

### Impact Explanation

A curated pool that deploys `SwapAllowlistExtension` to restrict trading to a known set of addresses can be fully bypassed by any unprivileged user routing through `MetricOmmSimpleRouter`. The attacker pays no extra cost beyond normal gas. Every swap that should have been blocked executes at the oracle-derived bid/ask price, draining LP liquidity to unauthorized counterparties. This is a direct loss of LP principal and a complete failure of the pool's access-control invariant.

---

### Likelihood Explanation

`MetricOmmSimpleRouter` is the standard, documented periphery entry point for swaps. Pool admins who configure an allowlist and want their permitted users to use the router will allowlist the router address â€” the only apparent workaround â€” which immediately opens the pool to everyone. The trigger requires no special privilege, no unusual token behavior, and no multi-transaction setup: a single `exactInputSingle` call from any address suffices.

---

### Recommendation

The extension must recover the original end-user identity rather than trusting the `sender` argument when an intermediary is involved. Two sound approaches:

1. **Check `tx.origin` as a secondary signal** â€” only appropriate if the pool admin explicitly accepts EOA-only swappers and the threat model excludes contract callers.
2. **Require the router to forward the real user identity in `extensionData`** and have the extension decode and verify it â€” but this requires router cooperation and is fragile.
3. **Preferred**: gate on the `recipient` field instead of (or in addition to) `sender` when the pool's design intent is to restrict who *receives* output, or redesign the allowlist to gate at the router level with a separate `SwapAllowlistRouter` that enforces `msg.sender` before calling the pool.

The cleanest fix matching the protocol's architecture is to have `SwapAllowlistExtension` accept a signed or encoded real-caller proof in `extensionData`, verified against a known router, so the extension can authenticate the true initiator regardless of which intermediary called `swap()`.
