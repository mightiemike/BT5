### Title
Missing Runtime Consistency Check Between `BouncerWeights::receipt_l2_gas` and `VersionedConstants::max_block_size` Causes Systematic EIP-1559 Gas Price Miscalculation - (File: `crates/blockifier/src/bouncer.rs` / `crates/apollo_versioned_constants/src/lib.rs`)

---

### Summary

Two independently-configured values that the codebase explicitly documents as required to be equal — `BouncerWeights::receipt_l2_gas` (the blockifier's block-closing gas limit) and `VersionedConstants::max_block_size` (the orchestrator's EIP-1559 fee-market block-size denominator) — have no runtime assertion or config validation enforcing their consistency. When they diverge, every subsequent block's gas price is computed against the wrong block-size denominator, producing systematically wrong L2 gas prices with direct economic impact.

---

### Finding Description

The codebase contains two separate, independently-configurable values that are documented to be equal:

**Value 1 — `BouncerWeights::receipt_l2_gas`** (blockifier/batcher side):

```rust
// NOTE: Must stay in sync with orchestrator_versioned_constants' max_block_size.
receipt_l2_gas: GasAmount(5800000000),
``` [1](#0-0) [2](#0-1) 

This value is the block capacity limit used by the `Bouncer` to close a block. When accumulated `receipt_l2_gas` across transactions reaches this limit, the block is sealed.

**Value 2 — `VersionedConstants::max_block_size`** (orchestrator/consensus side):

```rust
/// The maximum block size in gas units.
// NOTE: Must stay in sync with BouncerWeights receipt_l2_gas.
pub max_block_size: GasAmount,
``` [3](#0-2) 

This value is loaded from the orchestrator versioned constants JSON files (e.g., `orchestrator_versioned_constants_0_14_4.json` sets it to `5800000000`). [4](#0-3) 

**How `max_block_size` is used in the fee market:**

`calculate_next_base_gas_price` in `crates/apollo_consensus_orchestrator/src/fee_market/mod.rs` uses `gas_target` (which is derived from `max_block_size`) as the EIP-1559 denominator:

```rust
assert!(
    gas_target < versioned_constants.max_block_size,
    "Gas target must be lower than max block size."
);
// ...
let denominator =
    gas_target_u256 * U256::from(versioned_constants.gas_price_max_change_denominator);
let price_change = (price_u256 * gas_delta) / denominator;
``` [5](#0-4) 

The `gas_delta` is `|gas_used - gas_target|`, where `gas_used` is the actual L2 gas consumed in the block as closed by the bouncer (bounded by `receipt_l2_gas`). The fee market assumes blocks can hold up to `max_block_size` gas, but the bouncer actually closes them at `receipt_l2_gas`.

**The gap — no cross-component validation exists:**

`SequencerNodeConfig::validate_node_config` and `cross_member_validations` perform component-presence checks but contain no assertion that `batcher_config.block_max_capacity.receipt_l2_gas == VersionedConstants::latest_constants().max_block_size`. [6](#0-5) 

The two values are configured in completely separate locations:
- `receipt_l2_gas` is set via `batcher_config.static_config.block_builder_config.bouncer_config.block_max_capacity.receipt_l2_gas` in `config_schema.json` [7](#0-6) 
- `max_block_size` is embedded in the versioned constants JSON files loaded by `apollo_versioned_constants` [8](#0-7) 

There is no code path that reads both values and asserts their equality at startup.

---

### Impact Explanation

When `receipt_l2_gas ≠ max_block_size`:

- The bouncer closes blocks at `receipt_l2_gas` gas units.
- The fee market computes the next block's gas price using `gas_used / gas_target` where `gas_target` is a fraction of `max_block_size`.
- If `receipt_l2_gas < max_block_size`: blocks are closed before reaching the fee market's assumed capacity. The fee market sees `gas_used` that is always capped below what it expects as "full", so it systematically underestimates congestion and underprices gas.
- If `receipt_l2_gas > max_block_size`: the bouncer allows more gas than the fee market's denominator, causing the fee market to overestimate congestion and overprice gas.

In both cases, every block's `next_l2_gas_price` is computed against the wrong denominator, producing a persistent, systematic wrong fee value. This matches the impact category: **Incorrect fee, gas, bouncer, resource accounting, refund, balance, or L1 gas price effect with economic impact.**

---

### Likelihood Explanation

The two values are configured in separate subsystems (blockifier config vs. orchestrator versioned constants JSON). During a protocol upgrade that changes `max_block_size` in a new versioned constants file, an operator must also manually update `receipt_l2_gas` in the node config. The only guard is a code comment. This is precisely the class of error the external report identified: multiple related values with no enforced consistency invariant.

---

### Recommendation

Add a startup assertion in `open_storage_internal` or in `SequencerNodeConfig::validate_node_config` / `cross_member_validations` that compares the configured `receipt_l2_gas` against `VersionedConstants::latest_constants().max_block_size`:

```rust
// In cross_member_validations or at batcher startup:
if let Some(batcher_cfg) = &self.batcher_config {
    let bouncer_receipt_l2_gas = batcher_cfg
        .static_config
        .block_builder_config
        .bouncer_config
        .block_max_capacity
        .receipt_l2_gas;
    let vc_max_block_size =
        apollo_versioned_constants::VersionedConstants::latest_constants().max_block_size;
    assert_eq!(
        bouncer_receipt_l2_gas,
        vc_max_block_size,
        "receipt_l2_gas ({}) must equal VersionedConstants::max_block_size ({})",
        bouncer_receipt_l2_gas.0,
        vc_max_block_size.0,
    );
}
```

---

### Proof of Concept

1. Set `batcher_config.static_config.block_builder_config.bouncer_config.block_max_capacity.receipt_l2_gas` = `4000000000` in the node config (e.g., via a config overlay).
2. Leave `orchestrator_versioned_constants_0_14_4.json` with `max_block_size = 5800000000`.
3. The node starts without error — `validate_node_config` does not check this invariant.
4. The bouncer closes blocks at 4 B gas. The fee market computes `gas_delta = |4B - gas_target|` but uses `gas_target` derived from 5.8 B as the denominator.
5. A block filled to its actual capacity (4 B) appears only ~69% full to the fee market, which computes a price decrease instead of a price increase, systematically underpricing L2 gas for every block.

```rust
pub fn calculate_next_base_gas_price(
    price: GasPrice,
    gas_used: GasAmount,
    gas_target: GasAmount,
    min_gas_price: GasPrice,
) -> GasPrice {
    let versioned_constants = VersionedConstants::latest_constants();
    assert!(
        gas_target < versioned_constants.max_block_size,
        "Gas target must be lower than max block size."
    );
    assert!(gas_target.0 > 0, "Gas target must be greater than zero.");
    assert!(
        versioned_constants.gas_price_max_change_denominator > 0,
        "Denominator constant must be greater than zero."
    );

    // If the current price is below the minimum, apply a gradual adjustment and return early.
    // This allows the price to increase by at most 1/MIN_GAS_PRICE_INCREASE_DENOMINATOR per block.
    if price < min_gas_price {
        let max_increase = price.0 / MIN_GAS_PRICE_INCREASE_DENOMINATOR;
        let adjusted = price.0 + max_increase;
        // Cap at min_gas_price to avoid overshooting
        let adjusted_price = adjusted.min(min_gas_price.0);
        info!(
            "Fee Market: Price {} below minimum gas price {}, adjusted price: {} )",
            price.0, min_gas_price.0, adjusted_price
        );
        return GasPrice(adjusted_price);
    }

    // Use U256 to avoid overflow, as multiplying a u128 by a u64 remains within U256 bounds.
    let gas_delta = U256::from(gas_used.0.abs_diff(gas_target.0));
    let gas_target_u256 = U256::from(gas_target.0);
    let price_u256 = U256::from(price.0);

    // Calculate price change by multiplying first, then dividing. This avoids the precision loss
    // that occurs when dividing before multiplying.
    let denominator =
        gas_target_u256 * U256::from(versioned_constants.gas_price_max_change_denominator);
    let price_change = (price_u256 * gas_delta) / denominator;

    let adjusted_price_u256 =
        if gas_used > gas_target { price_u256 + price_change } else { price_u256 - price_change };

    // Sanity check: ensure direction of change is correct
    assert!(
        gas_used > gas_target && adjusted_price_u256 >= price_u256
            || gas_used <= gas_target && adjusted_price_u256 <= price_u256
    );

    // Price should not realistically exceed u128::MAX, bound to avoid theoretical overflow.
    let adjusted_price = u128::try_from(adjusted_price_u256).unwrap_or(u128::MAX);
    GasPrice(max(adjusted_price, min_gas_price.0))
}
