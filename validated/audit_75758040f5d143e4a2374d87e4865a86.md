### Title
Silent Error Handling in `slashRedelegationPosition` Causes Permanent Bonus Reward Loss via State Desynchronization â€” (`File: x/tieredrewards/keeper/slash.go`)

---

### Summary

`slashRedelegationPosition` silently swallows `ErrInsufficientBonusPool` from `processEventsAndClaimBonus`. By the time that error is returned, two persistent/in-memory state mutations have already occurred: validator event reference counts have been decremented in storage, and the position's bonus checkpoints (`LastEventSeq`, `LastBonusAccrual`, `LastKnownBonded`) have been advanced in memory. The caller then persists the modified position via `setPositionWithState`, recording the checkpoints as if the bonus was paid â€” but the bonus coins were never transferred. The owner permanently loses the accrued bonus rewards for that period.

---

### Finding Description

In `slashRedelegationPosition`, after calling `processEventsAndClaimBonus`, the code catches `ErrInsufficientBonusPool` and silently continues:

```go
if _, err := k.processEventsAndClaimBonus(ctx, &pos); err != nil {
    if errors.Is(err, types.ErrInsufficientBonusPool) {
        k.logger(ctx).Error("insufficient bonus pool during redelegation slash", ...)
    } else {
        return err
    }
}
``` [1](#0-0) 

Inside `processEventsAndClaimBonus`, the following mutations happen **before** `sufficientBonusPoolBalance` is checked:

1. **Persistent**: `decrementEventRefCount` is called for every event in the loop â€” these writes to storage are not rolled back.
2. **In-memory (on `pos` via pointer)**: `pos.UpdateLastEventSeq`, `applyBonusAccrualCheckpoint`, and `pos.UpdateLastKnownBonded` all advance the position's checkpoints. [2](#0-1) [3](#0-2) [4](#0-3) 

Only after all of the above does `sufficientBonusPoolBalance` return `ErrInsufficientBonusPool`. Back in `slashRedelegationPosition`, the function continues and calls `setPositionWithState(ctx, pos, ...)`, which persists the already-mutated `pos`:

```go
if fullSlash {
    pos.Delegation = nil
    pos.ClearBonusCheckpoints()
    return k.setPositionWithState(ctx, pos, &ValidatorTransition{PreviousAddress: dstValStr})
}
pos.Delegation.Shares = pos.Delegation.Shares.Sub(sharesToUnbond)
return k.setPositionWithState(ctx, pos, nil)
``` [5](#0-4) 

In the partial-slash path (`!fullSlash`), `ClearBonusCheckpoints` is **not** called, so the advanced checkpoints are persisted verbatim. In the full-slash path, `ClearBonusCheckpoints` resets the in-memory position fields, but the event reference counts in storage have already been decremented â€” if any reach zero, those events are garbage-collected and can never be replayed.

The desynchronization is:
- **Event ref counts**: decremented (persistent) â€” events may be GC'd
- **Position checkpoints**: advanced and persisted â€” future claims start past these events
- **Bonus coins**: never transferred to the owner

---

### Impact Explanation

A position owner whose validator is slashed during an active redelegation, when the bonus pool is insufficient, permanently loses all accrued bonus rewards for the period covered by the events processed in that call. The position's `LastEventSeq` and `LastBonusAccrual` are advanced past those events, so no future `ClaimTierRewards` or `processEventsAndClaimBonus` call can recover them. The corrupted value is the `bonusCoins` amount that should have been sent from `types.RewardsPoolName` to the owner's address. [6](#0-5) 

---

### Likelihood Explanation

The trigger requires two concurrent conditions: (1) a position is in an active redelegation when its destination validator is slashed, and (2) the `RewardsPoolName` module account has insufficient balance to cover the accrued bonus. Both are realistic in production: the pool can be depleted by concurrent claims or simply be low during early operation. Validator slashes (double-sign evidence, downtime) are normal protocol events. No privileged access is required â€” any delegator who has called `TierRedelegate` and whose destination validator is subsequently slashed is affected. [7](#0-6) 

---

### Recommendation

`processEventsAndClaimBonus` must not decrement event reference counts or advance position checkpoints before confirming the pool has sufficient balance. The correct fix is to check `sufficientBonusPoolBalance` **before** the event-processing loop (or before any persistent mutations), so that if the pool is insufficient, no state is mutated and the error can be returned cleanly. Alternatively, if the "prevent chain halt" design intent must be preserved, the function should be split into a read-only accrual phase and a commit phase, with the commit phase only executing after the pool check passes.

---

### Proof of Concept

1. Alice calls `TierRedelegate` on position P (delegated to validator V1, redelegating to V2). An unbonding ID is recorded in `RedelegationMappings`.
2. V2 accrues several validator events (bond/unbond/slash) that entitle Alice to bonus rewards.
3. The `RewardsPoolName` balance drops below Alice's accrued bonus (e.g., due to concurrent claims by other users).
4. V2 is slashed. The staking module fires `BeforeRedelegationSlashed`, which calls `slashRedelegationPosition`.
5. Inside `slashRedelegationPosition`, `processEventsAndClaimBonus` is called. It loops through all events, decrementing their ref counts in storage and advancing `pos.LastEventSeq`, `pos.LastBonusAccrual`, `pos.LastKnownBonded` in memory. It then calls `sufficientBonusPoolBalance`, which returns `ErrInsufficientBonusPool`.
6. `slashRedelegationPosition` catches the error, logs it, and continues. It calls `setPositionWithState(ctx, pos, nil)`, persisting the advanced checkpoints.
7. Alice later calls `ClaimTierRewards`. `processEventsAndClaimBonus` finds no events since `pos.LastEventSeq` (they were already processed and their ref counts decremented to zero, causing GC). Alice receives zero bonus for the entire period. [8](#0-7) [9](#0-8)

func (k Keeper) slashRedelegationPosition(ctx context.Context, unbondingId uint64, sharesToUnbond math.LegacyDec) error {
	positionId, err := k.getRedelegationMapping(ctx, unbondingId)
	if errors.Is(err, collections.ErrNotFound) {
		return nil
	}
	if err != nil {
		return err
	}

	pos, err := k.getPositionState(ctx, positionId)
	if errors.Is(err, types.ErrPositionNotFound) {
		k.logger(ctx).Error("position not found during redelegation slash",
			"position_id", positionId,
			"unbonding_id", unbondingId,
			"error", err.Error(),
		)
		return nil
	}
	if err != nil {
		return err
	}

	if !pos.IsDelegated() {
		// Defensive
		k.logger(ctx).Error("delegation missing during BeforeRedelegationSlashed",
			"position_id", positionId,
			"unbonding_id", unbondingId,
			"shares_to_unbond", sharesToUnbond.String(),
		)
		return nil
	}

	dstValStr := pos.Delegation.ValidatorAddress

	// Settle bonus against PRE-slash shares.
	if _, err := k.processEventsAndClaimBonus(ctx, &pos); err != nil {
		// Deliberately forgo bonus rewards if pool is insufficient to prevent chain halt.
		if errors.Is(err, types.ErrInsufficientBonusPool) {
			k.logger(ctx).Error("insufficient bonus pool during redelegation slash",
				"position_id", pos.Id,
				"error", err.Error(),
			)
		} else {
			return err
		}
	}

	fullSlash := sharesToUnbond.GTE(pos.Delegation.Shares)

	if fullSlash {
		pos.Delegation = nil
		pos.ClearBonusCheckpoints()
		return k.setPositionWithState(ctx, pos, &ValidatorTransition{PreviousAddress: dstValStr})
	}
	// In-memory only: the persisted Position carries no share count, and the
	// live delegation will reflect the post-Unbond shares on the next read.
	// Update the local copy so any follow-up logic in this call sees consistent state.
	pos.Delegation.Shares = pos.Delegation.Shares.Sub(sharesToUnbond)
	return k.setPositionWithState(ctx, pos, nil)
```
