### Title
`ValidResourceBounds::AllResources` Silently Downgraded to `ValidResourceBounds::L1Gas` in Protobuf Round-Trip When L2 and L1_DATA Gas Bounds Are Zero, Causing Wrong Transaction Hash Preimage and Execution Semantics — (`crates/apollo_protobuf/src/converters/transaction.rs`)

### Summary

The protobuf deserializer for `ValidResourceBounds` uses the runtime *values* of the `l2_gas` and `l1_data_gas` fields to infer which variant to reconstruct. A post-0.13.3 `AllResources` transaction whose `l2_gas` and `l1_data_gas` bounds are both zero is indistinguishable from a pre-0.13.3 `L1Gas` transaction after serialization, and is silently reconstructed as `L1Gas`. This changes the transaction hash preimage (2 vs 3 resource elements in `get_tip_resource_bounds_hash`) and the gas computation mode (`GasVectorComputationMode::All` vs `NoL2Gas`), causing wrong execution results during block re-execution or proving.

### Finding Description

**Serialization side** (`From<ValidResourceBounds> for protobuf::ResourceBounds`):

When a `ValidResourceBounds::L1Gas` value is serialized, `l2_gas` and `l1_data_gas` are both written as `ResourceBounds::default()` (all-zero). When a `ValidResourceBounds::AllResources` value with zero `l2_gas` and zero `l1_data_gas` is serialized, the wire bytes are **identical**. [1](#0-0) 

**Deserialization side** (`TryFrom<protobuf::ResourceBounds> for ValidResourceBounds`):

The reconstructor checks `l1_data_gas.is_zero() && l2_gas.is_zero()` to decide the variant. Because the wire encoding is identical for both cases, an `AllResources` transaction with zero L2 and L1_DATA bounds is reconstructed as `L1Gas`. [2](#0-1) 

The TODO comment acknowledges the fragility: `l1_data_gas` is `unwrap_or_default()` to support 0.13.2 blocks where the field is absent. This conflates "field absent (0.13.2 legacy)" with "field present but zero (valid post-0.13.3 AllResources)".

**Hash preimage divergence** (`get_tip_resource_bounds_hash`):

`AllResources` hashes three resource elements (L1_GAS, L2_GAS, L1_DATA_GAS); `L1Gas` hashes only two (L1_GAS, L2_GAS). After the round-trip, the hash computed from the deserialized transaction differs from the hash stored in `InternalRpcTransaction::tx_hash` (computed at gateway ingestion time with `AllResources` semantics). [3](#0-2) 

**Execution mode divergence** (`get_gas_vector_computation_mode`):

`AllResources` → `GasVectorComputationMode::All` (L1, L2, L1_DATA gas tracked and bounded separately). `L1Gas` → `GasVectorComputationMode::NoL2Gas` (L2 gas converted to L1 equivalent, not bounded separately). [4](#0-3) 

A transaction committed with `AllResources(l1_gas=X, l2_gas={amount:0, price:0}, l1_data_gas=0)` semantics (where L2 gas bound = 0 causes immediate revert) is re-executed after P2P sync with `L1Gas` semantics (where L2 gas is not bounded), potentially succeeding and producing different state changes.

### Impact Explanation

**Wrong state, receipt, and revert result from blockifier for accepted input.** A transaction committed to a block as `AllResources` (reverted due to L2 gas bound = 0) is re-executed during block sync or proving as `L1Gas` (succeeds, producing state writes). The state root computed by the syncing/proving node diverges from the sequencer's committed root. This also breaks the proving pipeline: the transaction hash embedded in the proof is computed with the wrong preimage length.

### Likelihood Explanation

**Low-Medium.** The trigger requires a committed transaction with `l2_gas.is_zero() && l1_data_gas.is_zero()`. The gateway's `min_gas_price` check (`l2_gas.max_price_per_unit >= min_gas_price`) blocks this when `min_gas_price > 0`. However:

1. If `min_gas_price = 0` (a valid configuration), the gateway accepts such transactions.
2. The deprecated gateway path (`apollo_http_server/resources/deprecated_gateway/`) may not enforce the same check.
3. Historical blocks committed before the `min_gas_price` check was introduced may contain such transactions, causing divergence during `blockifier_reexecution`. [5](#0-4) 

### Recommendation

Replace the value-based variant inference with an explicit discriminant. Options:

1. Add a boolean `is_all_resources` field to the `protobuf::ResourceBounds` message so the variant is encoded explicitly, not inferred from field values.
2. During deserialization, use the transaction version (available in the enclosing message) to determine the variant: version ≥ 0.13.3 → always `AllResources`.
3. Remove the `unwrap_or_default()` fallback for `l1_data_gas` and instead treat its presence/absence as the discriminant (absent → `L1Gas`, present → `AllResources`), which is the original semantic intent of the TODO comment.

### Proof of Concept

```
1. Craft RpcInvokeTransactionV3 with:
     resource_bounds = AllResourceBounds {
         l1_gas:      { max_amount: 1000, max_price_per_unit: 1000 },
         l2_gas:      { max_amount: 0,    max_price_per_unit: 0    },
         l1_data_gas: { max_amount: 0,    max_price_per_unit: 0    },
     }
   (Passes gateway when min_gas_price = 0; max_possible_fee = 1_000_000 > 0.)

2. Sequencer executes with AllResources semantics:
     GasVectorComputationMode::All → L2 gas bound = 0 → immediate revert.
   Block records: tx reverted, fee charged, no state writes.
   tx_hash = poseidon(INVOKE, v3, sender, poseidon(tip, L1_GAS_packed, L2_GAS_packed(0), L1_DATA_GAS_packed(0)), ...)
                                                      ^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^
                                                      3-element resource hash

3. Block propagated via P2P as protobuf.
   Syncing node deserializes ResourceBounds:
     l1_data_gas.is_zero() = true, l2_gas.is_zero() = true
     → ValidResourceBounds::L1Gas(l1_gas)   ← WRONG variant

4. Syncing node re-executes (blockifier_reexecution / proving):
     GasVectorComputationMode::NoL2Gas → L2 gas not bounded → tx succeeds.
   Re-execution produces state writes absent from the committed block.
   State root diverges. Proof fails or attests wrong state.

5. Hash recomputed from deserialized tx:
     poseidon(INVOKE, v3, sender, poseidon(tip, L1_GAS_packed, L2_GAS_packed(0)), ...)
                                                ^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^^
                                                2-element resource hash  ← DIFFERENT
   Hash mismatch detected if hash is re-verified post-sync.
```

impl TryFrom<protobuf::ResourceBounds> for ValidResourceBounds {
    type Error = ProtobufConversionError;
    fn try_from(value: protobuf::ResourceBounds) -> Result<Self, Self::Error> {
        let Some(l1_gas) = value.l1_gas else {
            return Err(missing("ResourceBounds::l1_gas"));
        };
        let Some(l2_gas) = value.l2_gas else {
            return Err(missing("ResourceBounds::l2_gas"));
        };
        // TODO(Shahak): Assert data gas is not none once we remove support for 0.13.2.
        let l1_data_gas = value.l1_data_gas.unwrap_or_default();
        let l1_gas: ResourceBounds = l1_gas.try_into()?;
        let l2_gas: ResourceBounds = l2_gas.try_into()?;
        let l1_data_gas: ResourceBounds = l1_data_gas.try_into()?;
        Ok(if l1_data_gas.is_zero() && l2_gas.is_zero() {
            ValidResourceBounds::L1Gas(l1_gas)
        } else {
            ValidResourceBounds::AllResources(AllResourceBounds { l1_gas, l2_gas, l1_data_gas })
        })
    }

pub fn get_tip_resource_bounds_hash(
    resource_bounds: &ValidResourceBounds,
    tip: &Tip,
) -> Result<Felt, StarknetApiError> {
    let l1_resource_bounds = resource_bounds.get_l1_bounds();
    let l2_resource_bounds = resource_bounds.get_l2_bounds();

    // L1 and L2 gas bounds always exist.
    // Old V3 txs always have L2 gas bounds of zero, but they exist.
    let mut resource_felts = vec![
        get_concat_resource(&l1_resource_bounds, L1_GAS)?,
        get_concat_resource(&l2_resource_bounds, L2_GAS)?,
    ];

    // For new V3 txs, need to also hash the data gas bounds.
    resource_felts.extend(match resource_bounds {
        ValidResourceBounds::L1Gas(_) => vec![],
        ValidResourceBounds::AllResources(all_resources) => {
            vec![get_concat_resource(&all_resources.l1_data_gas, L1_DATA_GAS)?]
        }
    });

    Ok(HashChain::new().chain(&tip.0.into()).chain_iter(resource_felts.iter()).get_poseidon_hash())
}
