Audit Report

## Title
`SwapAllowlistExtension` Checks Router Address Instead of End User, Enabling Full Per-User Allowlist Bypass — (File: `metric-periphery/contracts/extensions/SwapAllowlistExtension.sol`)

## Summary
`SwapAllowlistExtension.beforeSwap` gates swaps using the `sender` argument, which the pool sets to its own `msg.sender`. When a user routes through `MetricOmmSimpleRouter`, the pool's `msg.sender` is the router contract, not the end user. A pool admin who adds the router to the allowlist (the only way to permit any router-mediated swap) inadvertently grants every router caller an allowlist slot, completely defeating per-user access control.

## Finding Description
**Root cause — wrong entity checked:**

`MetricOmmPool.swap` passes `msg.sender` as `sender` to `_beforeSwap`: [1](#0-0) 

`ExtensionCalling._beforeSwap` forwards that value verbatim into every configured extension: [2](#0-1) 

`SwapAllowlistExtension.beforeSwap` then checks `allowedSwapper[msg.sender][sender]`, where `msg.sender` is the pool and `sender` is whoever called `pool.swap`: [3](#0-2) 

`MetricOmmSimpleRouter.exactInputSingle` calls `pool.swap(...)` directly, making the router the pool's `msg.sender`. The original caller (`msg.sender` inside the router) is stored only in transient callback context for payment purposes and is never forwarded to the pool as the swap initiator: [4](#0-3) 

**Exploit flow:**
1. Pool admin deploys a pool with `SwapAllowlistExtension` to restrict swaps to a specific set of addresses (e.g., KYC'd counterparties).
2. Pool admin calls `setAllowedToSwap(pool, router, true)` — the only way to allow any router-mediated swap.
3. Any unprivileged user (not individually allowlisted) calls `MetricOmmSimpleRouter.exactInputSingle(...)`.
4. The router calls `pool.swap(...)`. The pool sees `msg.sender == router`. The extension checks `allowedSwapper[pool][router]` → `true`. The swap proceeds.
5. The end user's address is never checked. The per-user gate is completely bypassed.

The same path applies to `exactInput`, `exactOutputSingle`, and `exactOutput` in the router.

**Existing guards are insufficient:** The `allowAllSwappers` flag is a separate bypass for all users; it does not fix the router-identity problem. There is no mechanism in the pool, extension, or router to propagate the original caller's identity through the `sender` parameter.

## Impact Explanation
The `SwapAllowlistExtension` is a core access-control extension whose stated invariant is per-user gating of swaps per pool. When the router is allowlisted, that invariant is broken: any address can swap in a restricted pool by routing through `MetricOmmSimpleRouter`. This constitutes broken core pool functionality — the allowlist extension fails to enforce its intended restriction, allowing unauthorized swap access. Pools configured for restricted counterparty sets (compliance, private market-making arrangements, or tiered access) are fully open to any caller via the router.

## Likelihood Explanation
The condition is straightforward: a pool must use `SwapAllowlistExtension` and the pool admin must have added the router to the allowlist. Any pool that intends to allow router-mediated swaps while also restricting individual users will inevitably reach this configuration. An unprivileged attacker needs only to call the public `exactInputSingle` function on the router with the target pool — no special privileges, no front-running, no flash loans required. The attack is repeatable on every block.

## Recommendation
The router should accept an optional `swapperOverride` parameter and pass it through `extensionData`, or the pool/extension interface should be extended to carry the original initiator separately from the immediate caller. A simpler short-term fix: `SwapAllowlistExtension` should expose a secondary check path where the pool admin can allowlist `(router, endUser)` pairs, or the router should encode `msg.sender` into `extensionData` and the extension should decode and check it when the immediate `sender` is a known router. At minimum, the extension's NatSpec and admin tooling must warn that allowlisting a router grants access to all of its callers.

## Proof of Concept
