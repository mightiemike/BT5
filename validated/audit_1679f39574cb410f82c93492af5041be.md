### Title
Unchecked `fee_proposal_fri` During Startup/Transition Window Allows Fee Poisoning — (`crates/apollo_consensus_orchestrator/src/validate_proposal.rs`)

### Summary

`is_proposal_init_valid` skips the `fee_proposal_fri` bounds check whenever `fee_actual` is `None`. `fee_actual` is `None` for the first `window_size` blocks (startup) and for any block whose sliding window still contains pre-V0_14_3 `None` entries (version-transition). During those windows a proposer can set `fee_proposal_fri` to any value, including `u128::MAX`. Validators accept the proposal unconditionally, the extreme value is committed into the `ProposalCommitment` hash and recorded in `fee_proposals_window`. Once the window fills, `compute_fee_actual` returns an extreme median, `calculate_next_l2_gas_price_for_fin` uses it as a floor, and `l2_gas_price` is driven toward `u128::MAX`, causing all subsequent transactions to fail fee checks.

### Finding Description

In `is_proposal_init_valid` the bounds check is wrapped in a pattern-match that fires only when both `fee_actual` and `fee_proposal` are `Some`:

```rust
// During initiation (fee_actual is None, <window_size blocks), bounds are not enforced.
if let (Some(fee_actual), Some(fee_proposal)) =
    (proposal_init_validation.fee_actual, init_proposed.fee_proposal_fri)
{
    let (lower_bound, upper_bound) = fee_proposal_bounds(
        fee_actual,
        VersionedConstants::latest_constants().fee_proposal_margin_ppt,
    );
    if fee_proposal.0 < lower_bound || fee_proposal.0 > upper_bound {
        return Err(...);
    }
}
``` [1](#0-0) 

`fee_actual` is computed by `compute_fee_actual`, which returns `None` in two distinct cases:

1. **Startup window** — `height < window_size` (the window cannot be filled yet).
2. **V0_14_3 transition** — any slot in `[height-window_size, height-1]` holds `None` (pre-V0_14_3 block), causing the entire median to collapse to `None`. [2](#0-1) 

In both cases the validator passes `fee_actual: None` into `ProposalInitValidation`: [3](#0-2) 

The proposer's honest fallback when `fee_actual` is `None` is to freeze at `l2_gas_price`: [4](#0-3) 

But the validator never enforces this. It only checks that `fee_proposal_fri` is `Some` (required for V0_14_3+): [5](#0-4) 

A malicious proposer therefore sets `fee_proposal_fri = u128::MAX`. The value is committed into the `ProposalCommitment` via `Poseidon(partial.0, fee_proposal_fri)`: [6](#0-5) 

It is also recorded in `fee_proposals_window`: [7](#0-6) 

Once the window fills, `compute_fee_actual` returns the extreme median. `calculate_next_l2_gas_price_for_fin` uses it as a floor (`effective_min`): [8](#0-7) 

`calculate_next_base_gas_price` then ratchets `l2_gas_price` upward each block toward `u128::MAX`: [9](#0-8) 

The elevated `l2_gas_price` is placed in every subsequent `ProposalInit.l2_gas_price_fri` and passed to the blockifier via `convert_to_sn_api_block_info`: [10](#0-9) 

Transactions whose `resource_bounds.l2_gas.max_price_per_unit` was set at the pre-attack price fail the fee check in `check_actual_cost_within_bounds`, producing wrong receipts and incorrect fee accounting. [11](#0-10) 

### Impact Explanation

A malicious proposer who controls any block during the startup window (or during the V0_14_3 transition window) can inject an arbitrary `fee_proposal_fri` that all honest validators accept. The injected value is:

- Committed into the consensus-signed `ProposalCommitment` hash (binding it permanently).
- Recorded in `fee_proposals_window`, poisoning `fee_actual` for the next `window_size` blocks.
- Used as a floor in `calculate_next_l2_gas_price_for_fin`, ratcheting `l2_gas_price` upward each block.

The result is **incorrect fee/gas accounting with economic impact** (all transactions priced at the pre-attack level fail) and **wrong receipts from blockifier execution** (transactions that should succeed are rejected). This matches the "Critical. Incorrect fee, gas, bouncer, resource accounting, refund, balance, or L1 gas price effect with economic impact" scope.

### Likelihood Explanation

The attack window is bounded but predictable: the first `fee_proposal_window_size` blocks after genesis, and the first `fee_proposal_window_size` blocks after the V0_14_3 upgrade. Any consensus participant who holds the proposer role during those blocks can trigger the issue. In BFT consensus the proposer role rotates among validators, so no special privilege beyond being a validator is required. The V0_14_3 transition window is a one-time, network-wide event that every node experiences simultaneously, making it a high-value target.

### Recommendation

When `fee_actual` is `None`, enforce a fallback bound on `fee_proposal_fri` anchored to the node's own `l2_gas_price` (the value an honest proposer would emit). For example:

```rust
// When fee_actual is unavailable, the honest proposer freezes at l2_gas_price.
// Enforce the same invariant on the validator side.
if let Some(fee_proposal) = init_proposed.fee_proposal_fri {
    if fee_actual.is_none() {
        let expected = proposal_init_validation.l2_gas_price_fri;
        let (lower, upper) = fee_proposal_bounds(
            expected,
            VersionedConstants::latest_constants().fee_proposal_margin_ppt,
        );
        if fee_proposal.0 < lower || fee_proposal.0 > upper {
            return Err(ValidateProposalError::InvalidProposalInit(...));
        }
    }
}
```

This closes the gap without breaking the honest-proposer path, because an honest proposer already returns `l2_gas_price` when `fee_actual` is `None`.

### Proof of Concept

1. Network launches at height 0 with `starknet_version >= V0_14_3`. `fee_proposals_window` is empty; `compute_fee_actual` returns `None` for all heights `< window_size`.
2. Malicious proposer wins the proposer slot for height 0 and sets `fee_proposal_fri = Some(GasPrice(u128::MAX))` in `ProposalInit`.
3. `is_proposal_init_valid` reaches the bounds-check block: `proposal_init_validation.fee_actual` is `None`, so the `if let (Some(fee_actual), Some(fee_proposal))` arm does not fire. The proposal is accepted.
4. `finalize_decision` calls `record_fee_proposal(BlockNumber(0), Some(GasPrice(u128::MAX)))`.
5. The malicious proposer repeats for heights 1 … `window_size - 1` (or a majority of them to dominate the median).
6. At height `window_size`, `compute_fee_actual` returns `GasPrice(u128::MAX)` (or a large value if only a majority of slots were poisoned).
7. `calculate_next_l2_gas_price_for_fin` sets `effective_min = u128::MAX`; `calculate_next_base_gas_price` ratchets `l2_gas_price` upward each block.
8. Within `O(MIN_GAS_PRICE_INCREASE_DENOMINATOR * log(u128::MAX / initial_price))` blocks, `l2_gas_price` exceeds the `max_price_per_unit` of all normally-priced transactions.
9. All transactions fail `check_actual_cost_within_bounds`; blocks are produced but contain no executed transactions; fee receipts are wrong.

```rust
pub fn compute_fee_actual(
    fee_proposals_window: &BTreeMap<BlockNumber, Option<GasPrice>>,
    height: BlockNumber,
    window_size: u64,
) -> Option<GasPrice> {
    let Some(start) = height.0.checked_sub(window_size) else {
        warn!(
            "Cannot compute fee_actual for height {height}: height is below window_size \
             ({window_size})"
        );
        return None;
    };
    let window_size_usize = usize::try_from(window_size).expect("window_size fits in usize");
    let mut window = Vec::with_capacity(window_size_usize);
    for source_height in (start..height.0).map(BlockNumber) {
        match fee_proposals_window.get(&source_height) {
            Some(Some(price)) => window.push(*price),
            Some(None) | None => {
                warn!(
                    "Cannot compute fee_actual for height {height}: fee_proposals_window has no \
                     recorded fee_proposal for height {source_height}"
                );
                return None;
            }
        }
    }
    window.sort();
    let mid = window_size_usize / 2;
    let median = if window_size_usize.is_multiple_of(2) {
        // Even: average of the two middle values, rounded down.
        // Overflow-safe averaging: a + (b - a) / 2 (safe because sorted, so b >= a).
        GasPrice(window[mid - 1].0 + (window[mid].0 - window[mid - 1].0) / 2)
    } else {
        window[mid]
    };
    Some(median)
}
