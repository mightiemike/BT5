### Title
`ContractOwner` Missing `receive()` Function Permanently Locks Native ETH in `DirectDepositV1` Contracts — (File: `core/contracts/ContractOwner.sol`)

---

### Summary
`ContractOwner` has no `receive()` or `fallback()` function, yet its `withdrawFromDirectDepositV1` recovery path depends on `DirectDepositV1.withdrawNative()` pushing ETH directly to `ContractOwner` via a low-level `.call`. Because `ContractOwner` cannot accept ETH, this call always reverts, permanently locking any native ETH that becomes stranded in a `DirectDepositV1` contract.

---

### Finding Description
`ContractOwner.withdrawFromDirectDepositV1` is the sole on-chain recovery path for native ETH held in a `DirectDepositV1` (DDA) contract. When called with `token == address(0)`, it invokes `DirectDepositV1.withdrawNative()`:

```solidity
// ContractOwner.sol lines 628–636
if (token == address(0)) {
    uint256 preBalance = address(this).balance;
    DirectDepositV1(directDepositV1).withdrawNative();   // <-- pushes ETH to ContractOwner
    uint256 postBalance = address(this).balance;
    require(postBalance > preBalance, "empty");
    (bool success, ) = msg.sender.call{value: postBalance - preBalance}("");
    require(success, "xfer");
}
```

`DirectDepositV1.withdrawNative()` sends ETH to its caller (`ContractOwner`) via:

```solidity
// DirectDepositV1.sol lines 108–112
function withdrawNative() external onlyOwner {
    uint256 balance = address(this).balance;
    (bool success, ) = msg.sender.call{value: balance}("");
    require(success, "Failed to transfer native token to owner");
}
```

`ContractOwner` declares no `receive()` function and no `payable` fallback. The low-level `.call{value: balance}("")` to `ContractOwner` therefore returns `success = false`, causing `withdrawNative()` to revert with `"Failed to transfer native token to owner"`. The entire `withdrawFromDirectDepositV1` call reverts.

ETH can realistically become stranded in a DDA via the constructor's silent-failure path:

```solidity
// DirectDepositV1.sol lines 52–59
uint256 balance = address(this).balance;
if (balance != 0) {
    (bool success, ) = wrappedNative.call{value: balance}("");
    if (!success) {
        emit NativeTokenTransferFailed(balance);   // does NOT revert
    }
}
```

If the initial wrap fails, ETH is silently left in the DDA with no revert. The only designed recovery is `withdrawNative()` → `withdrawFromDirectDepositV1`, which is itself broken.

---

### Impact Explanation
Any ETH stranded in a `DirectDepositV1` contract is permanently unrecoverable. The `withdrawFromDirectDepositV1(subaccount, address(0))` call will always revert because `ContractOwner` cannot receive ETH. The asset delta is the full ETH balance of the affected DDA, permanently locked with no alternative on-chain recovery path.

---

### Likelihood Explanation
Low-to-medium. The precondition is ETH becoming stuck in a DDA, which occurs when the constructor's initial wrap silently fails (emits `NativeTokenTransferFailed` without reverting). This is realistic if `wrappedNative` is temporarily non-functional, misconfigured, or if the DDA is deployed with a pre-existing ETH balance. Once ETH is stuck, the broken recovery path is deterministic and 100% reproducible.

---

### Recommendation
Add a `receive()` function to `ContractOwner` so it can accept ETH forwarded by `DirectDepositV1.withdrawNative()`:

```solidity
receive() external payable {}
```

This mirrors the fix applied in the referenced report and unblocks the `withdrawFromDirectDepositV1` recovery path for native tokens.

---

### Proof of Concept
1. A `DirectDepositV1` is deployed (e.g., via `createDirectDepositV1`) with a non-zero ETH balance at construction time, and `wrappedNative.call{value: balance}("")` fails — the constructor emits `NativeTokenTransferFailed` and continues without reverting. ETH is now stranded in the DDA.
2. The owner calls `ContractOwner.withdrawFromDirectDepositV1(subaccount, address(0))`.
3. Internally, `DirectDepositV1(directDepositV1).withdrawNative()` is called.
4. `withdrawNative()` executes `msg.sender.call{value: balance}("")` where `msg.sender == address(ContractOwner)`.
5. `ContractOwner` has no `receive()` function → the call returns `(false, "")`.
6. `withdrawNative()` hits `require(success, "Failed to transfer native token to owner")` → reverts.
7. The entire `withdrawFromDirectDepositV1` call reverts. ETH is permanently locked in the DDA.
