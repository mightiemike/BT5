### Title
Deprecated `verify_transaction_inclusion` Remains Publicly Callable, Bypassing the 64-Byte Merkle Forgery Protection Added in v2 - (File: `contract/src/lib.rs`)

### Summary
`verify_transaction_inclusion` (v1) is still a live, publicly callable NEAR contract method despite being deprecated. `verify_transaction_inclusion_v2` was introduced specifically to close the 64-byte transaction Merkle proof forgery vulnerability by requiring a coinbase proof. Because v1 remains reachable by any unprivileged caller, the entire coinbase-proof protection can be bypassed by simply calling the old entry point directly — an exact structural analog to the reported "secondary account bypasses the penalty mechanism" pattern.

### Finding Description

`verify_transaction_inclusion_v2` was introduced to mitigate the well-known 64-byte transaction Merkle proof forgery attack (referenced in the code comments). It does so by requiring the caller to also supply a valid coinbase Merkle proof, which anchors the proof tree and prevents an internal node from being presented as a leaf transaction. [1](#0-0) 

However, the original `verify_transaction_inclusion` (v1) is still decorated only with `#[deprecated]` and `#[pause]`, and remains `pub`: [2](#0-1) 

`#[deprecated]` in Rust is a **compiler lint**, not a runtime restriction. It emits a warning to Rust callers at compile time but has zero effect on the compiled WASM binary or on external callers invoking the method via NEAR's JSON-RPC interface. Any NEAR account — with no role, no stake, no permission — can call `verify_transaction_inclusion` directly and receive a `bool` result without the coinbase proof ever being checked.

The code itself acknowledges the danger explicitly: [3](#0-2) 

### Impact Explanation

An attacker can exploit the 64-byte Merkle forgery attack through v1:

1. Craft a 64-byte blob whose double-SHA256 equals an internal Merkle tree node `N` of a real Bitcoin block already accepted by the contract.
2. Construct a valid Merkle proof from `N` up to the block's `merkle_root` (this proof is trivially derivable from the real block's transaction set).
3. Call `verify_transaction_inclusion` with `tx_id = N`, `tx_block_blockhash = <real block>`, and the proof above.
4. The function computes `compute_root_from_merkle_proof(N, index, proof)` and compares it to `header.block_header.merkle_root`. [4](#0-3) [5](#0-4) 

The comparison succeeds and the function returns `true`, falsely asserting that a non-existent transaction was included in a confirmed Bitcoin block. Any downstream contract or bridge that trusts this return value — e.g., to release funds, mint tokens, or authorize a cross-chain action — is directly exploitable.

### Likelihood Explanation

- No privileged role is required; the method is `pub` and callable by any NEAR account.
- The 64-byte forgery technique is publicly documented and has known tooling.
- The contract already stores real mainchain headers, so a valid target block is always available.
- The only on-chain guard is `#[pause]`, which is inactive by default and controlled by `Role::PauseManager`, not by the attacker.

### Recommendation

Remove `verify_transaction_inclusion` from the public NEAR ABI entirely. Because `#[deprecated]` provides no runtime enforcement, the method must either be:

- Deleted from the `#[near]` impl block (making it unreachable via NEAR RPC), or
- Converted to a private helper used only internally by `verify_transaction_inclusion_v2`.

If backward compatibility is required for a transition period, gate the method behind an explicit access-control role so that only trusted callers can invoke it, and document that it is insecure for untrusted use.

### Proof of Concept

```
# 1. Identify a real block accepted by the contract, e.g. block B with merkle_root R.
# 2. Pick any two adjacent leaf transactions T_left, T_right in B.
#    Their parent node P = SHA256d(T_left || T_right) is a 64-byte-preimage internal node.
# 3. Build the Merkle proof from P up to R (standard derivation from the block's tx list).
# 4. Call the NEAR contract:

             "merkle_proof": [<proof_hashes>],
             "confirmations": 1}}' \
  --accountId attacker.near

# Returns: true
# No coinbase proof was required; the v2 protection was never invoked.

    #[deprecated(
        since = "0.5.0",
        note = "Use `verify_transaction_inclusion_v2` instead."

    #[pause]
    pub fn verify_transaction_inclusion(&self, #[serializer(borsh)] args: ProofArgs) -> bool {
        require!(
            args.confirmations <= self.gc_threshold,
            "The required number of confirmations exceeds the number of blocks stored in memory"
        );

        let heaviest_block_header = self
            .headers_pool
            .get(&self.mainchain_tip_blockhash)
            .unwrap_or_else(|| env::panic_str(ERR_KEY_NOT_EXIST));
        let target_block_height = self
            .mainchain_header_to_height
            .get(&args.tx_block_blockhash)
            .unwrap_or_else(|| env::panic_str("block does not belong to the current main chain"));

        // Check requested confirmations. No need to compute proof if insufficient confirmations.
        require!(
            (heaviest_block_header.block_height).saturating_sub(target_block_height) + 1
                >= args.confirmations,
            "Not enough blocks confirmed"
        );

        let header = self
            .headers_pool
            .get(&args.tx_block_blockhash)
            .unwrap_or_else(|| env::panic_str("cannot find requested transaction block"));

        require!(!args.merkle_proof.is_empty(), "Merkle proof is empty");

        // compute merkle tree root and check if it matches block's original merkle tree root
        merkle_tools::compute_root_from_merkle_proof(
            args.tx_id,
            usize::try_from(args.tx_index).unwrap(),
            &args.merkle_proof,
        ) == header.block_header.merkle_root
    }

pub fn compute_root_from_merkle_proof(
    transaction_hash: H256,
    transaction_position: usize,
    merkle_proof: &Vec<H256>,
) -> H256 {
    let mut current_hash = transaction_hash;
    let mut current_position = transaction_position;

    for proof_hash in merkle_proof {
        if current_position % 2 == 0 {
            current_hash = compute_hash(&current_hash, proof_hash);
        } else {
            current_hash = compute_hash(proof_hash, &current_hash);
        }
        current_position /= 2;
    }

    current_hash
}
