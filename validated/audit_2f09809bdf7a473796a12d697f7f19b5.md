### Title
Post-Trade Health Check Permanently Bypassed via `isHealthy` Always Returning `true` — (File: `core/contracts/OffchainExchange.sol`)

---

### Summary

The `isHealthy` function in `OffchainExchange.sol` is a stub that unconditionally returns `true`. After every order match, `matchOrders` calls `require(isHealthy(taker.order.sender))` and `require(isHealthy(maker.order.sender))` to enforce that neither party's subaccount falls below the initial margin requirement. Because `isHealthy` always returns `true`, these checks are permanently disabled, allowing any trade to be settled on-chain regardless of whether it leaves a subaccount in an under-margined state. This is structurally identical to the BeaconKit bug: a validation gate that should reject invalid state instead always accepts it, committing the invalid state.
