### Title
Unchecked `transferFrom` Return Value Enables usdcE Drain Without USDC Payment - (File: `core/contracts/ContractOwner.sol`)

---

### Summary

`ContractOwner.replaceUsdcEWithUsdc` calls `IERC20Base(usdc).transferFrom(...)` directly without checking its return value and without using the project's own `ERC20Helper.safeTransferFrom` wrapper. If the USDC token returns `false` on failure instead of reverting, the function continues executing, withdrawing usdcE from the DirectDepositV1 address and transferring it to the caller — with no USDC ever received in exchange.

---

### Finding Description

`ContractOwner` declares `using ERC20Helper for IERC20Base` and consistently uses `safeTransfer`/`safeTransferFrom` throughout the codebase. However, `replaceUsdcEWithUsdc` breaks this pattern on the critical inbound transfer:

```solidity
// core/contracts/ContractOwner.sol line 616
IERC20Base(usdc).transferFrom(msg.sender, directDepositV1, balance); // ← raw call, return value ignored
DirectDepositV1(directDepositV1).withdraw(IIERC20Base(usdcE));       // ← usdcE pulled to ContractOwner
IERC20Base(usdcE).safeTransfer(msg.sender, balance);                 // ← usdcE sent to caller
```

The outbound transfer on line 618 correctly uses `safeTransfer`, but the inbound `transferFrom` on line 616 is a raw call whose `bool` return is silently discarded. `ERC20Helper.safeTransferFrom` exists precisely to guard against this:

```solidity
// core/contracts/libraries/ERC20Helper.sol lines 23-42
function safeTransferFrom(...) internal {
    (bool success, bytes memory data) = address(self).call(...);
    require(
        success && (data.length == 0 || abi.decode(data, (bool))),
        ERR_TRANSFER_FAILED
    );
}
```

If the USDC token at `0x2D270e6886d130D724215A266106e6832161EAEd` (hardcoded for chain 57073 / Ink) returns `false` on a failed `transferFrom` rather than reverting, execution falls through to lines 617–618 unconditionally.

---

### Impact Explanation

An attacker with zero USDC (or zero allowance) calls `replaceUsdcEWithUsdc(subaccount)` for any subaccount whose DDA holds usdcE:

1. `transferFrom` silently returns `false` — no USDC leaves the attacker.
2. `DirectDepositV1.withdraw(usdcE)` pulls the full usdcE balance from the DDA into `ContractOwner`.
3. `safeTransfer(msg.sender, balance)` sends that usdcE to the attacker.

**Corrupted asset delta:** usdcE tokens are drained from the DDA and delivered to the attacker; the DDA receives no USDC replacement. Every DDA on chain 57073 that holds usdcE is at risk.

---

### Likelihood Explanation

The function is `external` with no access-control modifier — only a chain ID check (`block.chainid == 57073`). Any unprivileged caller on the Ink chain can invoke it. The precondition is simply that a DDA exists for some subaccount and holds a non-zero usdcE balance, both of which are normal operational states during the migration this function is designed to facilitate. Likelihood is **medium-high** during the active migration window.

---

### Recommendation

Replace the raw `transferFrom` call with the project's own safe wrapper, consistent with every other transfer in the codebase:

```solidity
// Replace line 616:
IERC20Base(usdc).transferFrom(msg.sender, directDepositV1, balance);
// With:
IERC20Base(usdc).safeTransferFrom(msg.sender, directDepositV1, balance);
```

`safeTransferFrom` is already in scope via `using ERC20Helper for IERC20Base` declared at line 24 of `ContractOwner.sol`.

---

### Proof of Concept

1. A DDA exists for `subaccount` on chain 57073 with `usdcE.balanceOf(dda) = X > 0`.
2. Attacker calls `ContractOwner.replaceUsdcEWithUsdc(subaccount)` with zero USDC allowance.
3. `IERC20Base(usdc).transferFrom(attacker, dda, X)` returns `false` (non-reverting token) — no USDC transferred.
4. `DirectDepositV1(dda).withdraw(usdcE)` executes: usdcE balance `X` moves from DDA → ContractOwner.
5. `IERC20Base(usdcE).safeTransfer(attacker, X)` executes: usdcE `X` moves from ContractOwner → attacker.
6. Attacker holds `X` usdcE; DDA holds 0 usdcE and 0 USDC. Net theft: `X` usdcE. [1](#0-0) [2](#0-1) [3](#0-2)

```text
    function replaceUsdcEWithUsdc(bytes32 subaccount) external {
        require(block.chainid == 57073, ERR_UNAUTHORIZED);

        require(directDepositV1 != address(0), "no dda");
        address usdcE = 0xF1815bd50389c46847f0Bda824eC8da914045D14;

        uint256 balance = IERC20Base(usdcE).balanceOf(directDepositV1);
        if (balance > 0) {
            IERC20Base(usdc).transferFrom(msg.sender, directDepositV1, balance);
            DirectDepositV1(directDepositV1).withdraw(IIERC20Base(usdcE));
            IERC20Base(usdcE).safeTransfer(msg.sender, balance);
        }
    }
```

```text
    function safeTransferFrom(
        IERC20Base self,
        address from,
        address to,
        uint256 amount
    ) internal {
        (bool success, bytes memory data) = address(self).call(
            abi.encodeWithSelector(
                IERC20Base.transferFrom.selector,
                from,
                to,
                amount
            )
        );

        require(
            success && (data.length == 0 || abi.decode(data, (bool))),
            ERR_TRANSFER_FAILED
        );
    }
```
