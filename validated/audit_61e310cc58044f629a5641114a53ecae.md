### Title
Spread Health Bonus Inflated Due to Wrong Weight Source in `getCoreRisk` — (File: `core/contracts/BaseEngine.sol`, `core/contracts/Clearinghouse.sol`)

### Summary

`BaseEngine.getCoreRisk` always returns the **long weight** regardless of the actual position direction. `Clearinghouse.getHealth` uses this value to compute `existingWeight` for the spread health adjustment. For a long-spot / short-perp spread, the short perp's `shortWeightPerp` (≥ 1) should be used, but `longWeightPerp` (≤ 1) is used instead. This underestimates `existingWeight`, inflates `spreadWeight - existingWeight`, and grants an unearned health bonus to every spread position.

### Finding Description

`BaseEngine.getCoreRisk` always passes the literal `1` (positive) as the `amount` argument to `_getWeightX18`, unconditionally returning the long weight: [1](#0-0) 

`_getWeightX18` returns `longWeightInitialX18` / `longWeightMaintenanceX18` when `amount >= 0`, and `shortWeightInitialX18` / `shortWeightMaintenanceX18` when `amount < 0`: [2](#0-1) 

`Clearinghouse.getHealth` then uses the returned `longWeight` field from both the spot and perp `CoreRisk` structs to compute `existingWeight`: [3](#0-2) 

For a canonical spread (long spot + short perp), the individual health contributions already applied to the basis amount are:
- Spot side: `basisAmount × longWeightSpot × spotPrice`
- Perp side: `−basisAmount × shortWeightPerp × perpPrice`

So the correct `existingWeight` baseline is `(longWeightSpot + shortWeightPerp) / 2`. Because `shortWeightPerp ≥ 1` while `longWeightPerp ≤ 1`, the code substitutes a smaller value, making `spreadWeight − existingWeight` larger than it should be. With typical parameters (e.g., `longWeightPerp = 0.9`, `shortWeightPerp = 1.1`, `spreadWeight ≈ 0.98`):

| | `existingWeight` | `spreadWeight − existingWeight` | Effect |
|---|---|---|---|
| **Buggy** | `(0.9 + 0.9)/2 = 0.90` | `+0.08` | Health **inflated** |
| **Correct** | `(0.9 + 1.1)/2 = 1.00` | `−0.02` | Health slightly reduced |

The sign of the adjustment is reversed by the bug. Instead of a small health reduction, every long-spot / short-perp spread position receives a positive health bonus proportional to `basisAmount × (spotPrice + perpPrice) × 0.08`.

### Impact Explanation

Any subaccount holding a long-spot / short-perp spread position has its health overstated. This directly affects:

1. **`withdrawCollateral`** — the post-withdrawal `INITIAL` health check passes when it should fail, allowing excess collateral withdrawal.
2. **`transferQuote`** — the `_isAboveInitial` check is bypassed, enabling over-transfer.
3. **Liquidation** — `isUnderMaintenance` returns `false` for accounts that are genuinely undercollateralized, blocking timely liquidation and creating bad debt. [4](#0-3) [5](#0-4) 

### Likelihood Explanation

The spread mechanism is a documented, incentivized feature of the protocol. Any user who opens a long-spot / short-perp position (the standard funding-rate arbitrage trade) automatically benefits from the inflated health. No special permissions or admin access are required. The sequencer submits `matchOrders` transactions that create these positions, and the health inflation is applied on every subsequent `getHealth` call.

### Recommendation

In `BaseEngine.getCoreRisk`, pass the actual `amount` to `_getWeightX18` instead of the hardcoded `1`, so the returned weight reflects the real position direction:

```solidity
// Before (always returns longWeight):
RiskHelper._getWeightX18(risk, 1, healthType)

// After (returns direction-specific weight):
RiskHelper._getWeightX18(risk, amount, healthType)
```

This ensures `existingWeight` in `Clearinghouse.getHealth` correctly represents the average weight already applied to the basis amount, eliminating the inflated spread bonus.

### Proof of Concept

1. User opens a long-spot / short-perp spread position (e.g., long 1 BTC spot, short 1 BTC perp).
2. `Clearinghouse.getHealth` is called (e.g., during `withdrawCollateral`).
3. `getCoreRisk` is called for both the spot and perp products. Both return `longWeight = 0.9` (the perp's short weight of `1.1` is never used).
4. `existingWeight = (0.9 + 0.9) / 2 = 0.9`; `spreadWeight ≈ 0.98`; adjustment = `+0.08 × basisAmount × price`.
5. The user's health is inflated by `0.08 × 1 BTC × price` relative to the correct value.
6. The user withdraws collateral that should be locked, or avoids liquidation that should have been triggered. [6](#0-5) [7](#0-6) [8](#0-7)

```text
    function _getWeightX18(
        Risk memory risk,
        int128 amount,
        IProductEngine.HealthType healthType
    ) internal pure returns (int128) {
        if (healthType == IProductEngine.HealthType.PNL) {
            return ONE;
        }

        int128 weight;
        if (amount >= 0) {
            weight = healthType == IProductEngine.HealthType.INITIAL
                ? risk.longWeightInitialX18
                : risk.longWeightMaintenanceX18;
        } else {
            weight = healthType == IProductEngine.HealthType.INITIAL
                ? risk.shortWeightInitialX18
                : risk.shortWeightMaintenanceX18;
        }

        return weight;
    }
```

```text
    function _getSpreadWeightX18(
        IProductEngine.CoreRisk memory perpCoreRisk,
        IProductEngine.CoreRisk memory spotCoreRisk,
        IProductEngine.HealthType healthType
    ) internal pure returns (int128) {
        if (healthType == IProductEngine.HealthType.PNL) {
            return ONE;
        }
        int128 spreadWeight;
        if (spotCoreRisk.amount > 0) {
            spreadWeight = ONE - (ONE - perpCoreRisk.longWeight) / 5;
        } else {
            spreadWeight = ONE - (ONE - spotCoreRisk.longWeight) / 5;
        }
        int128 maxSpreadWeight;
        if (healthType == IProductEngine.HealthType.INITIAL) {
            maxSpreadWeight = ONE - ONE / 100; // 0.99
        } else {
            maxSpreadWeight = ONE - (ONE * 6) / 1000; // 0.994
        }
        if (spreadWeight > maxSpreadWeight) {
            spreadWeight = maxSpreadWeight;
        }
        return spreadWeight;
    }
```
