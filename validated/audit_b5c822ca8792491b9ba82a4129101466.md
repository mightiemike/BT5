### Title
Unsigned `oraclePriceX18` in `SignedMintNlp`/`SignedBurnNlp` Enables Unconstrained NLP Conversion Price Manipulation — (File: `core/contracts/EndpointTx.sol`)

---

### Summary

The `oraclePriceX18` field embedded in `SignedMintNlp` and `SignedBurnNlp` is excluded from the EIP-712 digest that users sign. A user's signature commits only to the amount (`quoteAmount` or `nlpAmount`) and nonce, but not to the price at which the NLP conversion executes. The sequencer can freely substitute any `oraclePriceX18` value when processing these transactions, causing users to receive fewer NLP tokens on mint or less USDC on burn than the fair market rate would yield.

---

### Finding Description

In `Verifier.sol`, `computeDigest` for `MintNlp` hashes only `(sender, quoteAmount, nonce)`:

```
MINT_NLP_SIGNATURE = "MintNlp(bytes32 sender,uint128 quoteAmount,uint64 nonce)"
digest = keccak256(abi.encode(
    keccak256(bytes(MINT_NLP_SIGNATURE)),
    signedTx.tx.sender,
    signedTx.tx.quoteAmount,
    signedTx.tx.nonce
));
``` [1](#0-0) [2](#0-1) 

The `oraclePriceX18` field in `SignedMintNlp` is structurally outside the signed `tx` sub-struct:

```solidity
struct SignedMintNlp {
    MintNlp tx;           // ← only this is signed
    bytes signature;
    int128 oraclePriceX18;  // ← NOT signed
    int128[] nlpPoolRebalanceX18;
}
``` [3](#0-2) 

Identically for `BurnNlp`: [4](#0-3) [5](#0-4) 

In `EndpointTx.processTransactionImpl`, after signature validation passes, the unsigned `oraclePriceX18` is written directly into the global price store and forwarded to `mintNlp`/`burnNlp`:

```solidity
priceX18[NLP_PRODUCT_ID] = signedTx.oraclePriceX18;
clearinghouse.mintNlp(signedTx.tx, signedTx.oraclePriceX18, nlpPools, ...);
``` [6](#0-5) [7](#0-6) 

`Clearinghouse.mintNlp` uses this price to compute the NLP amount the user receives:

```solidity
int128 nlpAmount = quoteAmount.div(oraclePriceX18);
``` [8](#0-7) 

`Clearinghouse.burnNlp` uses it to compute the USDC the user receives:

```solidity
int128 quoteAmount = nlpAmount.mul(oraclePriceX18);
int128 burnFee = MathHelper.max(ONE, quoteAmount / 1000);
quoteAmount = MathHelper.max(0, quoteAmount - burnFee);
``` [9](#0-8) 

There is no on-chain bound check, TWAP guard, or user-specified slippage limit applied to `oraclePriceX18` anywhere in the call chain.

---

### Impact Explanation

**For `mintNlp`:** A user pays `quoteAmount` USDC and receives `quoteAmount / oraclePriceX18` NLP. If `oraclePriceX18` is inflated (e.g., 2× fair value), the user receives half the NLP they should. The unissued NLP value remains in the `N_ACCOUNT` pool.

**For `burnNlp`:** A user burns `nlpAmount` NLP and receives `nlpAmount × oraclePriceX18 − fee` USDC. If `oraclePriceX18` is deflated (e.g., 0.5× fair value), the user receives half the USDC they should. The retained USDC stays in the NLP pool.

In both cases the delta is a direct, measurable asset transfer from the user to the NLP pool. The corrupted state is the user's quote balance and NLP balance after the operation. [10](#0-9) [11](#0-10) 

---

### Likelihood Explanation

The sequencer constructs and submits the full `SignedMintNlp`/`SignedBurnNlp` transaction bytes, including the unsigned `oraclePriceX18`. No on-chain mechanism prevents the sequencer from setting an arbitrary price. The user's signature provides no protection because `oraclePriceX18` is outside the signed domain. Exploitation requires the sequencer to act against user interests, but the protocol design provides zero on-chain recourse for users — unlike every other sensitive parameter (sender, amount, nonce) which are all signature-covered.

---

### Recommendation

Include `oraclePriceX18` in the EIP-712 type hash and digest for both `MintNlp` and `BurnNlp`, or add a user-specified slippage bound (`minNlpAmount` for mint, `minQuoteAmount` for burn) to the signed struct. Either approach ensures the user's signature commits to the price at which the conversion executes.

Updated signatures would be:

```
MintNlp(bytes32 sender, uint128 quoteAmount, uint64 nonce, int128 maxPriceX18)
BurnNlp(bytes32 sender, uint128 nlpAmount, uint64 nonce, int128 minPriceX18)
``` [12](#0-11) 

---

### Proof of Concept

1. Alice signs `MintNlp(sender=Alice, quoteAmount=1000e18, nonce=5)` expecting NLP at the current fair price of $1.00 → expects 1000 NLP.
2. Sequencer constructs `SignedMintNlp` with `oraclePriceX18 = 2e18` ($2.00).
3. Sequencer submits the transaction to `processTransaction`.
4. `validateSignedTx` passes — the signature is valid for `(Alice, 1000e18, 5)`, which is all that is hashed.
5. `priceX18[NLP_PRODUCT_ID]` is set to `2e18`.
6. `mintNlp` computes `nlpAmount = 1000e18 / 2e18 = 500e18`.
7. Alice's quote balance decreases by 1000 USDC; her NLP balance increases by only 500 NLP.
8. The 500 NLP worth of value ($500) remains in `N_ACCOUNT`. [13](#0-12) [14](#0-13)

        IEndpoint.MintNlp calldata txn,
        int128 oraclePriceX18,
        IEndpoint.NlpPool[] calldata nlpPools,
        int128[] calldata nlpPoolRebalanceX18
    ) external onlyEndpoint {
        require(!RiskHelper.isIsolatedSubaccount(txn.sender), ERR_UNAUTHORIZED);

        ISpotEngine spotEngine = _spotEngine();
        spotEngine.updatePrice(NLP_PRODUCT_ID, oraclePriceX18);

        require(txn.quoteAmount <= INT128_MAX, ERR_CONVERSION_OVERFLOW);
        int128 quoteAmount = int128(txn.quoteAmount);
        int128 nlpAmount = quoteAmount.div(oraclePriceX18);

        _validateNlpRebalance(nlpPools, nlpPoolRebalanceX18, quoteAmount);
        for (uint128 i = 0; i < nlpPoolRebalanceX18.length; i++) {
            require(nlpPoolRebalanceX18[i] >= 0, ERR_INVALID_NLP_REBALANCE);
        }

        spotEngine.updateBalance(NLP_PRODUCT_ID, txn.sender, nlpAmount);
        spotEngine.updateBalance(NLP_PRODUCT_ID, N_ACCOUNT, -nlpAmount);

        spotEngine.updateBalance(QUOTE_PRODUCT_ID, txn.sender, -quoteAmount);
        _applyNlpRebalance(spotEngine, nlpPools, nlpPoolRebalanceX18);

        require(
            getHealth(txn.sender, IProductEngine.HealthType.INITIAL) >= 0,

        );

    function burnNlp(
        IEndpoint.BurnNlp calldata txn,
        int128 oraclePriceX18,
        IEndpoint.NlpPool[] calldata nlpPools,
        int128[] calldata nlpPoolRebalanceX18
    ) external onlyEndpoint {
        require(!RiskHelper.isIsolatedSubaccount(txn.sender), ERR_UNAUTHORIZED);

        ISpotEngine spotEngine = _spotEngine();
        spotEngine.updatePrice(NLP_PRODUCT_ID, oraclePriceX18);

        require(txn.nlpAmount <= INT128_MAX, ERR_CONVERSION_OVERFLOW);
        int128 nlpAmount = int128(txn.nlpAmount);
        require(
            spotEngine.getNlpUnlockedBalance(txn.sender).amount >= nlpAmount,
            ERR_UNLOCKED_NLP_INSUFFICIENT
        );
        int128 quoteAmount = nlpAmount.mul(oraclePriceX18);
        int128 burnFee = MathHelper.max(ONE, quoteAmount / 1000);
        quoteAmount = MathHelper.max(0, quoteAmount - burnFee);

        _validateNlpRebalance(nlpPools, nlpPoolRebalanceX18, -quoteAmount);
        for (uint128 i = 0; i < nlpPoolRebalanceX18.length; i++) {
            require(nlpPoolRebalanceX18[i] <= 0, ERR_INVALID_NLP_REBALANCE);
        }

        spotEngine.updateBalance(NLP_PRODUCT_ID, txn.sender, -nlpAmount);
        spotEngine.updateBalance(NLP_PRODUCT_ID, N_ACCOUNT, nlpAmount);

        if (quoteAmount > 0) {
            spotEngine.updateBalance(QUOTE_PRODUCT_ID, txn.sender, quoteAmount);
            _applyNlpRebalance(spotEngine, nlpPools, nlpPoolRebalanceX18);
        }

        require(
            spotEngine.getBalance(NLP_PRODUCT_ID, txn.sender).amount >= 0,
            ERR_SUBACCT_HEALTH
        );
        // Burning NLP can decrease health if the burn fee exceeds the health improvement
        // from the withdrawal. This check prevents malicious actors from deliberately
        // creating unhealthy subaccounts through NLP burns.
        require(
            getHealth(txn.sender, IProductEngine.HealthType.MAINTENANCE) >= 0,
            ERR_SUBACCT_HEALTH
        );
    }
