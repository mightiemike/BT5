Audit Report

## Title
`SwapAllowlistExtension` Checks Immediate Caller Instead of End User, Enabling Router-Mediated Allowlist Bypass — (File: `metric-periphery/contracts/extensions/SwapAllowlistExtension.sol`)

## Summary
`SwapAllowlistExtension.beforeSwap` gates on `sender`, which the pool sets to its own `msg.sender` — the immediate caller of `pool.swap()`. When `MetricOmmSimpleRouter` is used, `sender` is the router address, not the end user. A pool admin who allowlists the router inadvertently grants every user routing through it unrestricted swap access, nullifying the per-user curation the extension is designed to enforce.

## Finding Description
`SwapAllowlistExtension.beforeSwap` performs the check:

```solidity
if (!allowAllSwappers[msg.sender] && !allowedSwapper[msg.sender][sender]) {
    revert IMetricOmmPoolActions.NotAllowedToSwap();
}
``` [1](#0-0) 

Here `msg.sender` is the pool (the extension's caller) and `sender` is the first parameter. `MetricOmmPool.swap` passes its own `msg.sender` as that first argument:

```solidity
_beforeSwap(
    msg.sender,   // ← becomes `sender` in the extension
    recipient,
    ...
);
``` [2](#0-1) 

`ExtensionCalling._beforeSwap` forwards it unchanged to the extension: [3](#0-2) 

When `MetricOmmSimpleRouter.exactInputSingle` is used, the router is the entity that calls `pool.swap()`:

```solidity
(int128 amount0Delta, int128 amount1Delta) = IMetricOmmPoolActions(params.pool)
    .swap(params.recipient, params.zeroForOne, ...);
``` [4](#0-3) 

So `sender` arriving at the extension equals the **router address**, not the actual end user. The check `allowedSwapper[pool][router]` is a single boolean covering every user who routes through that contract.

By contrast, `DepositAllowlistExtension.beforeAddLiquidity` correctly ignores the first argument (the immediate caller) and checks `owner` — the actual economic actor: [5](#0-4) 

The swap extension lacks this distinction entirely.

## Impact Explanation
A pool admin who configures a curated pool with `SwapAllowlistExtension` and then allowlists `MetricOmmSimpleRouter` (a routine operational step to support router-based swaps) silently grants every user on the router the same access as an individually allowlisted address. The allowlist policy — the sole mechanism protecting a curated pool's swap access — is nullified. This constitutes broken core pool functionality and an admin-boundary break where an unprivileged path circumvents the configured access control.

## Likelihood Explanation
Medium. The trigger is a pool admin allowlisting the router contract, which is a plausible and expected operational step for any curated pool that also wants to support the standard periphery router. The admin has no on-chain signal that doing so opens the pool to all users; the extension's naming ("Gates `swap` by swapper address") implies per-user granularity that does not survive router indirection.

## Recommendation
Gate on the actual end user, not the immediate pool caller. Two concrete options:

1. **Pass the real user through `extensionData`**: The router encodes `msg.sender` into `extensionData`; the extension decodes and verifies it. Requires a coordinated change to the router and extension.
2. **Align with `DepositAllowlistExtension`**: Expose the real payer (stored in the router's transient callback context as `_getPayer()`) through `extensionData` so the extension can check it instead of `sender`.

Until fixed, document explicitly that allowlisting the router is equivalent to `setAllowAllSwappers(pool, true)` and that per-user swap gating requires users to call the pool directly.

## Proof of Concept
```
1. Deploy MetricOmmPool with SwapAllowlistExtension as a beforeSwap hook.
2. Pool admin calls:
       swapExtension.setAllowedToSwap(pool, address(router), true)
   — intending to enable router-based swaps for allowlisted users.
3. Attacker (address NOT individually allowlisted) calls:
       router.exactInputSingle(ExactInputSingleParams{
           pool: pool,
           recipient: attacker,
           zeroForOne: true,
           amountIn: X,
           ...
       })
4. Router calls pool.swap(...) — pool's msg.sender = router.
5. Pool calls _beforeSwap(sender=router, ...).
6. Extension evaluates allowedSwapper[pool][router] == true → passes.
7. Swap executes. Attacker receives output tokens from a pool
   they were never individually authorized to trade on.
```
