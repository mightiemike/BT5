Audit Report

## Title
SwapAllowlistExtension Checks Router Address Instead of Actual Swapper, Allowing Any User to Bypass the Swap Allowlist via MetricOmmSimpleRouter - (File: metric-periphery/contracts/extensions/SwapAllowlistExtension.sol)

## Summary
`SwapAllowlistExtension.beforeSwap` gates swaps by checking the `sender` argument, which the pool sets to its own `msg.sender` — the router contract address when swaps are routed through `MetricOmmSimpleRouter`. If a pool admin allowlists the router to enable router-mediated swaps for their users, every non-allowlisted address can bypass the allowlist by calling the public router. The allowlist guard is completely defeated.

## Finding Description
`SwapAllowlistExtension.beforeSwap` performs its identity check at line 37:

```solidity
if (!allowAllSwappers[msg.sender] && !allowedSwapper[msg.sender][sender]) {
    revert IMetricOmmPoolActions.NotAllowedToSwap();
}
```

`msg.sender` is the pool (correct, enforces per-pool namespacing). `sender` is the first argument, which `MetricOmmPool.swap` sets to its own `msg.sender` at line 231:

```solidity
_beforeSwap(
    msg.sender,   // ← whoever called pool.swap()
    ...
);
```

When a user calls `MetricOmmSimpleRouter.exactInputSingle`, the router calls `pool.swap()` directly at lines 72–80 without encoding the original caller into `extensionData`:

```solidity
_setNextCallbackContext(params.pool, CALLBACK_MODE_JUST_PAY, msg.sender, params.tokenIn);
(int128 amount0Delta, int128 amount1Delta) = IMetricOmmPoolActions(params.pool)
    .swap(
        params.recipient,
        params.zeroForOne,
        MetricOmmSwapInputs.asAmountSpecifiedIn(params.amountIn),
        priceLimitX64,
        "",
        params.extensionData   // ← user-supplied, not modified to include original caller
    );
```

The pool's `msg.sender` is the **router address**, so the extension checks `allowedSwapper[pool][router]`, not `allowedSwapper[pool][user]`. This creates an irreconcilable dilemma:

- **Router not allowlisted:** Allowlisted users cannot use the router — their swaps revert because the extension sees `sender = router`.
- **Router allowlisted:** Every non-allowlisted user bypasses the allowlist by calling `router.exactInputSingle()`, because the extension only sees the router address and passes unconditionally.

The same flaw applies to `exactInput` (line 104), `exactOutputSingle` (line 136), and `exactOutput` (line 165), as well as intermediate hops in `_exactOutputIterateCallback` (line 220), all of which call `pool.swap()` with `msg.sender = router`.

## Impact Explanation
A pool admin who configures `SwapAllowlistExtension` and allowlists the router to enable router-mediated swaps for their curated pool inadvertently opens the pool to **all** users. Any non-allowlisted address can call `MetricOmmSimpleRouter.exactInputSingle` targeting the curated pool and execute a swap the allowlist was designed to block. This is an admin-boundary break: the pool admin's configured access control is bypassed by an unprivileged path (the public router). The extension decision value — `allowedSwapper[pool][user]` — is never evaluated; instead `allowedSwapper[pool][router]` is checked, which is the wrong value.

## Likelihood Explanation
`MetricOmmSimpleRouter` is the primary production swap entry point. Pool admins who configure `SwapAllowlistExtension` and want their allowlisted users to have a good UX will naturally allowlist the router. The bypass is then reachable by any address with no special privileges — the attacker only needs to call a public function on a public contract. No special tokens, flash loans, or privileged access are required.

## Recommendation
The extension must check the **original user**, not the intermediate caller. Two approaches:

1. **Pass the original caller through `extensionData`:** The router encodes `msg.sender` into `extensionData` before calling the pool. The extension decodes and checks that address when `sender` is a known router.
2. **Transient storage for original initiator:** The router sets a transient-storage slot for the "original initiator" before calling `pool.swap()`, analogous to how it already stores the payer in transient storage via `_setNextCallbackContext`. The extension reads this slot.

## Proof of Concept
**Setup:**
- Pool deployed with `SwapAllowlistExtension` as `extension2`, `BEFORE_SWAP_ORDER` pointing to extension 2.
- Pool admin calls `swapExtension.setAllowedToSwap(pool, router, true)` to allow router-mediated swaps.
- Alice (`0xAlice`) is NOT in `allowedSwapper[pool]`.

**Attack:**
1. Alice calls `MetricOmmSimpleRouter.exactInputSingle(ExactInputSingleParams({pool: pool, ..., extensionData: ""}))`.
2. Router calls `pool.swap(recipient, zeroForOne, amount, priceLimit, "", "")` with `msg.sender = router`.
3. Pool calls `_beforeSwap(router, ...)` at line 231.
4. `ExtensionCalling._beforeSwap` encodes `sender = router` and calls `SwapAllowlistExtension.beforeSwap(router, ...)`.
5. Extension checks `allowedSwapper[pool][router]` → `true` → passes.
6. Swap executes. Alice has bypassed the allowlist.
