Audit Report

## Title
`SwapAllowlistExtension` Checks Router Address Instead of Actual Swapper, Allowing Any User to Bypass the Swap Allowlist - (File: `metric-periphery/contracts/extensions/SwapAllowlistExtension.sol`)

## Summary

`SwapAllowlistExtension.beforeSwap` gates on the `sender` argument forwarded by the pool, which equals `msg.sender` of the `pool.swap()` call — the router contract, not the end user. When a pool admin allowlists the router to enable router-mediated swaps on a restricted pool, every user on the network can bypass the allowlist by calling `MetricOmmSimpleRouter`, because the extension sees `sender = router` (allowlisted) rather than the actual originating address.

## Finding Description

In `MetricOmmPool.swap`, the pool passes `msg.sender` as `sender` to `_beforeSwap`: [1](#0-0) 

`ExtensionCalling._beforeSwap` encodes and forwards that same `sender` value to every configured extension: [2](#0-1) 

`SwapAllowlistExtension.beforeSwap` receives `sender` as its first argument and checks it against the per-pool allowlist (`allowedSwapper[msg.sender][sender]`, where `msg.sender` is the pool): [3](#0-2) 

In `MetricOmmSimpleRouter.exactInputSingle`, the router calls `pool.swap` directly with no mechanism to forward the original `msg.sender` into the pool's `sender` slot: [4](#0-3) 

The same pattern applies to `exactInput`, `exactOutputSingle`, and `exactOutput`: [5](#0-4) 

**Exploit call chain:**
```
User (not allowlisted)
  → MetricOmmSimpleRouter.exactInputSingle(...)
      → pool.swap(recipient, ...) [msg.sender = router]
          → _beforeSwap(sender = router, ...)
              → SwapAllowlistExtension.beforeSwap(sender = router, ...)
                  → allowedSwapper[pool][router] == true → PASSES
```

The existing guard (`allowedSwapper[msg.sender][sender]`) is structurally insufficient because it cannot distinguish between the router itself swapping and an arbitrary user routing through the router.

## Impact Explanation

A pool deployer who restricts swaps to vetted counterparties (private institutional pool, KYC-gated pool, risk-parameterized pool) configures `SwapAllowlistExtension` as a `beforeSwap` hook. To allow those vetted users to also use the public router (the standard UX path), the admin must allowlist the router address. Once the router is allowlisted, `allowedSwapper[pool][router]` is permanently `true` for every caller, and any unprivileged user can execute swaps in the restricted pool by routing through `MetricOmmSimpleRouter`. The allowlist provides zero protection against router-mediated access. Unauthorized swappers can drain token reserves from a pool whose LP terms were set assuming only vetted counterparties would trade, causing direct loss of LP principal.

## Likelihood Explanation

- `MetricOmmSimpleRouter` is a public, permissionless contract.
- The bypass requires no special privileges: any EOA can call `exactInputSingle`.
- The precondition (router allowlisted) is the natural and necessary configuration for any pool that wants to support both allowlist enforcement and router-mediated swaps.
- No admin action or malicious setup beyond normal pool configuration is required.

## Recommendation

Pass the original end-user address through the swap path so the extension can gate on the actual swapper rather than the intermediary. Two approaches:

1. **Router-side**: Have the router encode the original `msg.sender` into `extensionData` and have the extension decode and check it. This requires a convention between router and extension.
2. **Protocol-side (preferred)**: Add a `payer` or `originator` field to the pool's `swap` signature (analogous to how `addLiquidity` separates `sender` from `owner`), so the router can pass `msg.sender` as the originator and the pool forwards it to extensions as the identity to gate.

Until fixed, pool admins should not rely on `SwapAllowlistExtension` for pools where the router is also allowlisted, as the allowlist is fully bypassable.

## Proof of Concept

```solidity
// Setup: pool with SwapAllowlistExtension, router allowlisted, attacker not allowlisted
address pool = ...; // pool with SwapAllowlistExtension as beforeSwap hook
address router = address(metricOmmSimpleRouter);
address attacker = makeAddr("attacker");

// Admin allowlists the router (required for normal router-mediated swaps)
vm.prank(poolAdmin);
swapAllowlist.setAllowedToSwap(pool, router, true);

// Confirm attacker is NOT individually allowlisted
assertFalse(swapAllowlist.isAllowedToSwap(pool, attacker));

// Attacker bypasses the allowlist by routing through the router
vm.prank(attacker);
// This succeeds — extension sees sender=router, which is allowlisted
metricOmmSimpleRouter.exactInputSingle(
    IMetricOmmSimpleRouter.ExactInputSingleParams({
        pool: pool,
        recipient: attacker,
        zeroForOne: true,
        amountIn: 1000,
        amountOutMinimum: 0,
        priceLimitX64: 0,
        deadline: block.timestamp + 1,
        extensionData: ""
    })
);
// Attacker successfully swapped in a pool they were not authorized to access
```
