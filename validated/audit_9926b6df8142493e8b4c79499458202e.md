### Title
NFT Transferred to Module Account Without Recipient Capability Validation Can Permanently Freeze NFT - (File: `x/nft/keeper/msg_server.go`, `x/nft-transfer/keeper/packet.go`)

---

### Summary

Both the native NFT transfer handler (`MsgTransferNFT`) and the IBC NFT receive path (`processReceivedPacket`) accept arbitrary recipient addresses from user-controlled input and transfer NFT ownership to them without checking whether the recipient is a module account. Cosmos SDK module accounts have no private key and cannot sign transactions, so any NFT transferred to a module account address is permanently frozen and irrecoverable.

---

### Finding Description

**Path 1 — Native NFT Transfer (`MsgTransferNFT`)**

In `x/nft/keeper/msg_server.go`, `TransferNFT` decodes `msg.Recipient` directly from user input and calls `TransferOwner` with no blocked-address guard:

```go
recipient, err := sdk.AccAddressFromBech32(msg.Recipient)
// ...
if err := m.TransferOwner(ctx, msg.DenomId, msg.Id, sender, recipient); err != nil {
```

There is no check equivalent to the Cosmos SDK bank module's `BlockedAddr` map, which prevents coins from being sent to module accounts. The NFT module has no such protection.

**Path 2 — IBC NFT Receive (`processReceivedPacket`)**

In `x/nft-transfer/keeper/packet.go`, `processReceivedPacket` decodes the receiver from the IBC packet data (fully user-controlled on the sending chain) and mints or unescrows NFTs directly to it:

```go
receiver, err := sdk.AccAddressFromBech32(data.Receiver)
// ...
if err := k.nftKeeper.MintNFT(ctx, voucherClassID, tokenID, "", data.TokenUris[i], "", escrowAddress, receiver); err != nil {
```

and for the "back to origin" path:

```go
if err := k.nftKeeper.TransferOwner(ctx, voucherClassID, tokenID, escrowAddress, receiver); err != nil {
```

Neither branch validates that `receiver` is not a module account. The IBC path is the higher-risk scenario: a user on chain A specifies a `receiver` string that happens to match a module account address on chain B (e.g., the staking, distribution, or governance module account). The sender has no reliable way to enumerate all module account addresses on the destination chain.

---

### Impact Explanation

Cosmos SDK module accounts are derived deterministically from module names and hold no private key. Once an NFT's owner field is set to a module account address, no transaction can ever be signed to move it out. The NFT is permanently frozen and the original owner suffers an irreversible asset loss. The corrupted state is the NFT's `owner` field in the NFT store, which is set to an address that can never authorize a `MsgTransferNFT` or any other outgoing transaction.

---

### Likelihood Explanation

- **Native transfer path**: A user would have to deliberately or accidentally supply a module account bech32 address as the recipient. Module account addresses are derivable from well-known module names, so accidental collision is low but not zero.
- **IBC receive path**: The `receiver` field is a free-form string embedded in the IBC packet on the sending chain. A user on chain A who does not know the full set of module account addresses on chain B can inadvertently specify one. Cross-chain address reuse (e.g., copy-pasting an address that is a module account on the destination chain) is a realistic user mistake. This path has moderate likelihood.

---

### Recommendation

1. In `x/nft/keeper/msg_server.go` `TransferNFT`, add a blocked-address check before calling `TransferOwner`, mirroring the bank module's `BlockedAddr` guard:
   ```go
   if k.bankKeeper.BlockedAddr(recipient) {
       return nil, sdkerrors.Wrapf(sdkerrors.ErrUnauthorized, "%s is not allowed to receive NFTs", recipient)
   }
   ```
2. Apply the same guard in `x/nft-transfer/keeper/packet.go` `processReceivedPacket` before minting or transferring to `receiver`.
3. Similarly protect `MintNFT` in `x/nft/keeper/msg_server.go` for the same reason.

---

### Proof of Concept

1. Identify any module account address on the destination chain, e.g., the distribution module account (`cosmos1jv65s3grqf6v6jl3dp4t6c9t9rk99cd88lyufl` on mainnet, or derivable via `sdk.AccAddress(crypto.AddressHash([]byte("distribution")))`).
2. **Native path**: Submit `MsgTransferNFT` with `Recipient` set to that module account address. The call succeeds; `TransferOwner` updates the NFT owner to the module account. No subsequent transaction can move the NFT.
3. **IBC path**: On chain A, submit `MsgTransfer` (IBC NFT) with `Receiver` set to the module account bech32 address on chain B. When the relayer delivers the packet, `processReceivedPacket` mints or unescrows the NFT to that address. The NFT is permanently frozen on chain B with no recovery path. func (m msgServer) TransferNFT(goCtx context.Context, msg *types.MsgTransferNFT) (*types.MsgTransferNFTResponse, error) {
```go
	sender, err := sdk.AccAddressFromBech32(msg.Sender)
	if err != nil {
		return nil, err
	}

	recipient, err := sdk.AccAddressFromBech32(msg.Recipient)
	if err != nil {
		return nil, err
	}

	ctx := sdk.UnwrapSDKContext(goCtx)
	if err := m.TransferOwner(ctx, msg.DenomId, msg.Id,
		sender,
		recipient,
	); err != nil {
		return nil, err
	}
```

```go
func (k Keeper) processReceivedPacket(ctx sdk.Context, packet channeltypes.Packet,
	data types.NonFungibleTokenPacketData,
) error {
	receiver, err := sdk.AccAddressFromBech32(data.Receiver)
	if err != nil {
		return err
	}

	isAwayFromOrigin := types.IsAwayFromOrigin(packet.GetSourcePort(), packet.GetSourceChannel(), data.ClassId)

	// create the escrow address for creating denom and minting nft
	escrowAddress := types.GetEscrowAddress(packet.GetDestPort(), packet.GetDestChannel())

	if isAwayFromOrigin {
		// since SendPacket did not prefix the classID, we must prefix classID here
		classPrefix := types.GetClassPrefix(packet.GetDestPort(), packet.GetDestChannel())
		// NOTE: sourcePrefix contains the trailing "/"
		prefixedClassID := classPrefix + data.ClassId

		// construct the class trace from the full raw classID
		classTrace := types.ParseClassTrace(prefixedClassID)
		if !k.HasClassTrace(ctx, classTrace.Hash()) {
			k.SetClassTrace(ctx, classTrace)
		}

		voucherClassID := classTrace.IBCClassID()

		if !k.nftKeeper.HasDenomID(ctx, voucherClassID) {
			if err := k.nftKeeper.IssueDenom(ctx, voucherClassID, voucherClassID, "", data.ClassUri, escrowAddress); err != nil {
				return err
			}
		}
		sdkCtx := sdk.UnwrapSDKContext(ctx)
		sdkCtx.EventManager().EmitEvent(
			sdk.NewEvent(
				types.EventTypeClassTrace,
				sdk.NewAttribute(types.AttributeKeyTraceHash, classTrace.Hash().String()),
				sdk.NewAttribute(types.AttributeKeyClassID, voucherClassID),
			),
		)

		for i, tokenID := range data.TokenIds {
			if err := k.nftKeeper.MintNFT(ctx, voucherClassID, tokenID, "", data.TokenUris[i], "", escrowAddress, receiver); err != nil {
				return err
			}
		}
	} else {
		// If the token moves in the direction of back to origin,
		// we need to unescrow the token and transfer it to the receiver

		// we should remove the prefix. For example:
		// p6/c6/p4/c4/p2/c2/nftClass -> p4/c4/p2/c2/nftClass
		unprefixedClassID := types.RemoveClassPrefix(packet.GetSourcePort(),
			packet.GetSourceChannel(), data.ClassId)

		voucherClassID := types.ParseClassTrace(unprefixedClassID).IBCClassID()
		for _, tokenID := range data.TokenIds {
			if err := k.nftKeeper.TransferOwner(ctx,
				voucherClassID, tokenID, escrowAddress, receiver); err != nil {
				return err
			}
		}
	}

	return nil
```
