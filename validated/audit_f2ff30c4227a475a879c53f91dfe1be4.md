### Title
Deprecated `verify_transaction_inclusion` Remains Publicly Callable Without Coinbase Proof Enforcement, Enabling Merkle Proof Forgery - (File: `contract/src/lib.rs`)

### Summary
The contract exposes two transaction-inclusion verification endpoints. The v1 function (`verify_transaction_inclusion`) lacks coinbase proof validation and is explicitly documented as vulnerable to the 64-byte Merkle proof forgery attack. Despite being marked `#[deprecated]`, it remains a fully callable public NEAR method. Any unprivileged caller can invoke it directly, bypassing the security enforcement added in v2, and obtain a `true` result for a fabricated transaction inclusion claim.

### Finding Description

`verify_transaction_inclusion` (v1) is a public, `#[pause]`-gated method that any NEAR account can call: [1](#0-0) 

The function's own documentation states the broken invariant explicitly: [2](#0-1) 

The only guard against an empty proof is: [3](#0-2) 

There is no check that `tx_id` is a leaf node rather than an internal Merkle tree node. The verification reduces entirely to:

```
compute_root_from_merkle_proof(tx_id, tx_index, merkle_proof) == header.merkle_root
```

`compute_root_from_merkle_proof` is a pure path-traversal function with no leaf/internal-node distinction: [4](#0-3) 

The v2 function adds coinbase proof enforcement to close this gap: [5](#0-4) 

However, v2 internally delegates to v1 via `self.verify_transaction_inclusion(args.into())`, and v1 remains independently callable. The Rust `#[deprecated]` attribute is a compile-time warning only; it does not restrict runtime invocation of a NEAR contract method. Any account can call `verify_transaction_inclusion` by name over the NEAR RPC.

### Impact Explanation

A recipient contract that calls `verify_transaction_inclusion` to gate a cross-chain action (e.g., token unlock, bridge settlement) can be deceived into accepting a forged proof. The attacker supplies an internal Merkle tree node hash as `tx_id` and a valid sibling path from that node to the root. Because the path is structurally correct, the computed root matches the stored `merkle_root`, and the function returns `true` for a transaction that does not exist. The corrupted value is the boolean proof result consumed by the downstream contract.

### Likelihood Explanation

The entry path requires no privileges: any NEAR account submits a borsh-encoded `ProofArgs` to `verify_transaction_inclusion`. The target block must already be on the mainchain (a public, observable state), and the Merkle tree structure of any Bitcoin block is fully derivable from public chain data. The attack is deterministic and requires no brute force.

### Recommendation

Remove or gate `verify_transaction_inclusion` so it is no longer callable by external accounts. The simplest fix is to change its visibility to `pub(crate)` or add a `#[private]` attribute. All callers should be migrated to `verify_transaction_inclusion_v2`. Alternatively, inline the coinbase proof check directly into v1 and remove the delegation pattern.

### Proof of Concept

1. Identify any mainchain block `B` with at least 4 transactions. Its Merkle tree has internal nodes at depth 1.
2. Let `N` = `hash(tx[0], tx[1])` — the left internal node at depth 1. This value is computable from public Bitcoin block data.
3. The sibling of `N` at depth 1 is `S` = `hash(tx[2], tx[3])`.
4. The root is `hash(N, S)` = `merkle_root` of block `B`.
5. Call `verify_transaction_inclusion` with:
   - `tx_id = N`
   - `tx_index = 0` (even position → left child)
   - `tx_block_blockhash = B`
   - `merkle_proof = [S]`
   - `confirmations = 1`
6. `compute_root_from_merkle_proof(N, 0, [S])` computes `hash(N, S)` = `merkle_root`. The comparison succeeds and the function returns `true` for the non-existent "transaction" `N`. [6](#0-5)

    pub fn verify_transaction_inclusion(&self, #[serializer(borsh)] args: ProofArgs) -> bool {
        require!(
            args.confirmations <= self.gc_threshold,
            "The required number of confirmations exceeds the number of blocks stored in memory"
        );

        let heaviest_block_header = self
            .headers_pool
            .get(&self.mainchain_tip_blockhash)
            .unwrap_or_else(|| env::panic_str(ERR_KEY_NOT_EXIST));
        let target_block_height = self
            .mainchain_header_to_height
            .get(&args.tx_block_blockhash)
            .unwrap_or_else(|| env::panic_str("block does not belong to the current main chain"));

        // Check requested confirmations. No need to compute proof if insufficient confirmations.
        require!(
            (heaviest_block_header.block_height).saturating_sub(target_block_height) + 1
                >= args.confirmations,
            "Not enough blocks confirmed"
        );

        let header = self
            .headers_pool
            .get(&args.tx_block_blockhash)
            .unwrap_or_else(|| env::panic_str("cannot find requested transaction block"));

        require!(!args.merkle_proof.is_empty(), "Merkle proof is empty");

        // compute merkle tree root and check if it matches block's original merkle tree root
        merkle_tools::compute_root_from_merkle_proof(
            args.tx_id,
            usize::try_from(args.tx_index).unwrap(),
            &args.merkle_proof,
        ) == header.block_header.merkle_root
    }

pub fn compute_root_from_merkle_proof(
    transaction_hash: H256,
    transaction_position: usize,
    merkle_proof: &Vec<H256>,
) -> H256 {
    let mut current_hash = transaction_hash;
    let mut current_position = transaction_position;

    for proof_hash in merkle_proof {
        if current_position % 2 == 0 {
            current_hash = compute_hash(&current_hash, proof_hash);
        } else {
            current_hash = compute_hash(proof_hash, &current_hash);
        }
        current_position /= 2;
    }

    current_hash
}

fn compute_hash(first_tx_hash: &H256, second_tx_hash: &H256) -> H256 {
    let mut concat_inputs = Vec::with_capacity(64);
    concat_inputs.extend(first_tx_hash.0);
    concat_inputs.extend(second_tx_hash.0);

    double_sha256(&concat_inputs)
}
