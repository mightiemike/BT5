### Title
Protobuf `ValidResourceBounds` Deserialization Silently Downgrades `AllResources` to `L1Gas`, Producing a Wrong Transaction Hash in the State-Sync Path — (`crates/apollo_protobuf/src/converters/transaction.rs`)

---

### Summary

The `TryFrom<protobuf::ResourceBounds> for ValidResourceBounds` converter in `transaction.rs` uses a value-based heuristic to decide whether to reconstruct `ValidResourceBounds::L1Gas` or `ValidResourceBounds::AllResources`. Because `get_tip_resource_bounds_hash` hashes a **different number of resource elements** for each variant, any transaction that was originally signed and hashed as `AllResources` but happens to carry zero `l2_gas` and zero `l1_data_gas` will be silently re-typed to `L1Gas` after protobuf round-trip, producing a divergent transaction hash.

---

### Finding Description

**Hash function branch — `get_tip_resource_bounds_hash`** [1](#0-0) 

For `ValidResourceBounds::L1Gas` the function hashes **two** resource felts (`L1_GAS`, `L2_GAS`). For `ValidResourceBounds::AllResources` it hashes **three** (`L1_GAS`, `L2_GAS`, `L1_DATA_GAS`). Even when `l1_data_gas` is numerically zero, `get_concat_resource` encodes the 7-byte ASCII tag `L1_DATA` into the felt, so the third element is **never zero**: [2](#0-1) 

**Protobuf converter — wrong heuristic** [3](#0-2) 

The converter decides the variant purely from the deserialized values: if `l1_data_gas.is_zero() && l2_gas.is_zero()` it emits `L1Gas`. A transaction that was submitted with `AllResourceBounds { l1_gas: X, l2_gas: 0, l1_data_gas: 0 }` satisfies this condition and is silently downgraded.

**Contrast with the RPC-transaction converter — no such heuristic** [4](#0-3) 

The RPC-path converter always produces `AllResourceBounds` unconditionally, so the mempool P2P path is unaffected. The vulnerability is isolated to the `transaction.rs` converter used for block-sync / state-sync.

**Hash is computed from the variant**

`InvokeTransactionV3Trait::resource_bounds()` returns the stored `ValidResourceBounds` directly: [5](#0-4) 

`get_invoke_transaction_v3_hash` calls `get_tip_resource_bounds_hash` with whatever variant is stored: [6](#0-5) 

So a transaction deserialized as `L1Gas` will produce a hash that omits the `L1_DATA_GAS` element, diverging from the hash that was computed (and signed) at submission time when the transaction was `AllResources`.

**Gateway accepts the triggering input**

The gateway stateless validator accepts transactions with `AllResourceBounds` where only `l1_gas` is non-zero: [7](#0-6) 

---

### Impact Explanation

Any component that deserializes an `InvokeTransactionV3` from protobuf via the `transaction.rs` path and then calls `calculate_transaction_hash` (e.g., `validate_transaction_hash` during block ingestion, or the prover re-hashing transactions for the OS) will compute a hash that does **not** match the hash the submitter signed and the sequencer stored. This breaks the canonical hash invariant:

- A syncing node that recomputes transaction hashes to validate a synced block will reject the block as invalid.
- A prover that re-hashes transactions to build the OS input will produce a mismatched transaction hash, invalidating the proof.

Impact category: **High — Transaction conversion or signature/hash logic binds the wrong hash/type for an accepted transaction.**

---

### Likelihood Explanation

The triggering condition — an `AllResources` transaction with zero `l2_gas` and zero `l1_data_gas` — is explicitly tested and accepted by the gateway. Any user who sets only `l1_gas` (a common pattern for pre-0.13.3-style usage under the new format) produces exactly this input. The protobuf round-trip occurs on every block sync, making the condition reachable without any privileged access.

---

### Recommendation

Remove the value-based heuristic. The protobuf wire format already carries all three optional fields; the converter should always reconstruct `AllResources` when the message was produced from an `AllResources` transaction. The simplest fix is to align `transaction.rs` with `rpc_transaction.rs`: always decode into `AllResourceBounds` and wrap in `ValidResourceBounds::AllResources`, reserving `L1Gas` only for messages that explicitly lack the `l2_gas` field (i.e., `value.l2_gas.is_none()`), which is the true wire-level signal for pre-0.13.3 transactions.

---

### Proof of Concept

1. Submit `RpcInvokeTransactionV3` with `resource_bounds = AllResourceBounds { l1_gas: 1000/1, l2_gas: 0/0, l1_data_gas: 0/0 }`.
2. Gateway accepts it; `InternalRpcInvokeTransactionV3::resource_bounds()` returns `ValidResourceBounds::AllResources(...)`.
3. `get_tip_resource_bounds_hash` hashes **three** elements: `[tip, l1_gas_felt, l2_gas_felt, l1_data_gas_felt]` where `l1_data_gas_felt = Felt::from_bytes_be([0, 'L','1','_','D','A','T','A', 0…0])` ≠ 0.
4. Transaction is included in a block; the hash `H_orig` is stored.
5. Block is synced; `TryFrom<protobuf::ResourceBounds> for ValidResourceBounds` fires: `l1_data_gas.is_zero() && l2_gas.is_zero()` → emits `ValidResourceBounds::L1Gas(l1_gas)`.
6. `get_tip_resource_bounds_hash` now hashes **two** elements: `[tip, l1_gas_felt, l2_gas_felt]`.
7. Recomputed hash `H_sync` ≠ `H_orig`.
8. `validate_transaction_hash` returns `false`; block is rejected by the syncing node.

pub fn get_tip_resource_bounds_hash(
    resource_bounds: &ValidResourceBounds,
    tip: &Tip,
) -> Result<Felt, StarknetApiError> {
    let l1_resource_bounds = resource_bounds.get_l1_bounds();
    let l2_resource_bounds = resource_bounds.get_l2_bounds();

    // L1 and L2 gas bounds always exist.
    // Old V3 txs always have L2 gas bounds of zero, but they exist.
    let mut resource_felts = vec![
        get_concat_resource(&l1_resource_bounds, L1_GAS)?,
        get_concat_resource(&l2_resource_bounds, L2_GAS)?,
    ];

    // For new V3 txs, need to also hash the data gas bounds.
    resource_felts.extend(match resource_bounds {
        ValidResourceBounds::L1Gas(_) => vec![],
        ValidResourceBounds::AllResources(all_resources) => {
            vec![get_concat_resource(&all_resources.l1_data_gas, L1_DATA_GAS)?]
        }
    });

    Ok(HashChain::new().chain(&tip.0.into()).chain_iter(resource_felts.iter()).get_poseidon_hash())
}

fn get_concat_resource(
    resource_bounds: &ResourceBounds,
    resource_name: &ResourceName,
) -> Result<Felt, StarknetApiError> {
    let max_amount = resource_bounds.max_amount.0.to_be_bytes();
    let max_price = resource_bounds.max_price_per_unit.0.to_be_bytes();
    let concat_bytes =
        [[0_u8].as_slice(), resource_name.as_slice(), max_amount.as_slice(), max_price.as_slice()]
            .concat();
    Ok(Felt::from_bytes_be(&concat_bytes.try_into().expect("Expect 32 bytes")))
}

pub(crate) fn get_invoke_transaction_v3_hash<T: InvokeTransactionV3Trait>(
    transaction: &T,
    chain_id: &ChainId,
    transaction_version: &TransactionVersion,
) -> Result<TransactionHash, StarknetApiError> {
    let tip_resource_bounds_hash =
        get_tip_resource_bounds_hash(&transaction.resource_bounds(), transaction.tip())?;
    let paymaster_data_hash =
        HashChain::new().chain_iter(transaction.paymaster_data().0.iter()).get_poseidon_hash();
    let data_availability_mode = concat_data_availability_mode(
        transaction.nonce_data_availability_mode(),
        transaction.fee_data_availability_mode(),
    );
    let account_deployment_data_hash = HashChain::new()
        .chain_iter(transaction.account_deployment_data().0.iter())
        .get_poseidon_hash();
    let calldata_hash =
        HashChain::new().chain_iter(transaction.calldata().0.iter()).get_poseidon_hash();
    let mut hash_chain = HashChain::new()
        .chain(&INVOKE)
        .chain(&transaction_version.0)
        .chain(transaction.sender_address().0.key())
        .chain(&tip_resource_bounds_hash)
        .chain(&paymaster_data_hash)
        .chain(&Felt::try_from(chain_id)?)
        .chain(&transaction.nonce().0)
        .chain(&data_availability_mode)
        .chain(&account_deployment_data_hash)
        .chain(&calldata_hash);
    if !transaction.proof_facts().0.is_empty() {
        let proof_facts_hash =
            HashChain::new().chain_iter(transaction.proof_facts().0.iter()).get_poseidon_hash();
        hash_chain = hash_chain.chain(&proof_facts_hash);
    }
    Ok(TransactionHash(hash_chain.get_poseidon_hash()))
}

impl TryFrom<protobuf::ResourceBounds> for ValidResourceBounds {
    type Error = ProtobufConversionError;
    fn try_from(value: protobuf::ResourceBounds) -> Result<Self, Self::Error> {
        let Some(l1_gas) = value.l1_gas else {
            return Err(missing("ResourceBounds::l1_gas"));
        };
        let Some(l2_gas) = value.l2_gas else {
            return Err(missing("ResourceBounds::l2_gas"));
        };
        // TODO(Shahak): Assert data gas is not none once we remove support for 0.13.2.
        let l1_data_gas = value.l1_data_gas.unwrap_or_default();
        let l1_gas: ResourceBounds = l1_gas.try_into()?;
        let l2_gas: ResourceBounds = l2_gas.try_into()?;
        let l1_data_gas: ResourceBounds = l1_data_gas.try_into()?;
        Ok(if l1_data_gas.is_zero() && l2_gas.is_zero() {
            ValidResourceBounds::L1Gas(l1_gas)
        } else {
            ValidResourceBounds::AllResources(AllResourceBounds { l1_gas, l2_gas, l1_data_gas })
        })
    }
