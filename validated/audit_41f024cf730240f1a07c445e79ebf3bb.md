### Title
`From<ResourceBoundsMapping> for ValidResourceBounds` Silently Downgrades `AllResources` to `L1Gas`, Producing a Different Transaction Hash Than the Gateway Path — (`File: crates/apollo_rpc/src/v0_8/transaction.rs`)

### Summary

`get_tip_resource_bounds_hash` in `crates/starknet_api/src/transaction_hash.rs` hashes **3 resource elements** for `ValidResourceBounds::AllResources` (L1\_GAS + L2\_GAS + L1\_DATA\_GAS) but only **2 elements** for `ValidResourceBounds::L1Gas` (L1\_GAS + L2\_GAS). The gateway always produces `AllResources` for every incoming V3 transaction. The RPC layer's `From<ResourceBoundsMapping> for ValidResourceBounds` silently converts any `ResourceBoundsMapping` with `l2_gas = 0` and `l1_data_gas = 0` to `L1Gas`. The two paths therefore compute **different Poseidon hashes** for the same transaction data, making the RPC simulation/estimation path return an authoritative-looking wrong value (wrong signature-verification outcome) for any V3 transaction whose resource bounds happen to have zero L2 and L1-data-gas fields.

---

### Finding Description

**Path 1 — Gateway ingestion (always `AllResources`):**

`RpcDeclareTransactionV3` and `RpcInvokeTransactionV3` carry `resource_bounds: AllResourceBounds`. The converter wraps them unconditionally:

```rust
// crates/starknet_api/src/rpc_transaction.rs:408-411
impl DeclareTransactionV3Trait for InternalRpcDeclareTransactionV3 {
    fn resource_bounds(&self) -> ValidResourceBounds {
        ValidResourceBounds::AllResources(self.resource_bounds)  // always AllResources
    }
``` [1](#0-0) 

The hash is then computed via `get_declare_transaction_v3_hash` → `get_tip_resource_bounds_hash`, which for `AllResources` chains **3 packed resource felts** (L1\_GAS, L2\_GAS, L1\_DATA\_GAS) into the Poseidon state:

```rust
// crates/starknet_api/src/transaction_hash.rs:203-208
resource_felts.extend(match resource_bounds {
    ValidResourceBounds::L1Gas(_) => vec![],                          // 2 elements total
    ValidResourceBounds::AllResources(all_resources) => {
        vec![get_concat_resource(&all_resources.l1_data_gas, L1_DATA_GAS)?]  // 3 elements total
    }
});
``` [2](#0-1) 

**Path 2 — RPC layer conversion (may produce `L1Gas`):**

`apollo_rpc::v0_8::transaction::ResourceBoundsMapping` is the wire type for all RPC-facing V3 transactions (simulation, estimation, `starknet_getTransaction` re-ingestion). Its `From` impl silently downgrades to `L1Gas` whenever both optional fields are zero:

```rust
// crates/apollo_rpc/src/v0_8/transaction.rs:188-199
impl From<ResourceBoundsMapping> for ValidResourceBounds {
    fn from(value: ResourceBoundsMapping) -> Self {
        if value.l1_data_gas.is_zero() && value.l2_gas.is_zero() {
            Self::L1Gas(value.l1_gas)          // ← only 2 elements will be hashed
        } else {
            Self::AllResources(AllResourceBounds { ... })
        }
    }
}
``` [3](#0-2) 

The reverse direction (`ValidResourceBounds → ResourceBoundsMapping`) preserves all three fields regardless of variant:

```rust
// crates/apollo_rpc/src/v0_8/transaction.rs:202-216
impl From<ValidResourceBounds> for ResourceBoundsMapping {
    fn from(value: ValidResourceBounds) -> Self {
        match value {
            ValidResourceBounds::L1Gas(l1_gas) => Self {
                l1_gas,
                l1_data_gas: ResourceBounds::default(),   // zero
                l2_gas: ResourceBounds::default(),        // zero
            },
            ValidResourceBounds::AllResources(...) => Self { l1_gas, l1_data_gas, l2_gas },
        }
    }
}
``` [4](#0-3) 

**The lossy round-trip:**

```
ValidResourceBounds::AllResources { l1_gas: X, l2_gas: 0, l1_data_gas: 0 }
  → ResourceBoundsMapping { l1_gas: X, l2_gas: 0, l1_data_gas: 0 }
  → ValidResourceBounds::L1Gas(X)          ← different variant, different hash
```

**Concrete hash divergence:**

| Path | Elements hashed by `get_tip_resource_bounds_hash` | Result |
|---|---|---|
| Gateway (`AllResources`) | `tip ‖ L1_GAS_packed ‖ L2_GAS_packed(0) ‖ L1_DATA_GAS_packed(0)` | H₁ |
| RPC (`L1Gas`) | `tip ‖ L1_GAS_packed ‖ L2_GAS_packed(0)` | H₂ |

H₁ ≠ H₂ because Poseidon is not length-extension safe — a 4-element input and a 3-element input produce distinct outputs even when the first three elements are identical. [5](#0-4) 

---

### Impact Explanation

Any RPC endpoint that deserializes a V3 transaction through `ResourceBoundsMapping → ValidResourceBounds` and then calls `calculate_transaction_hash` (e.g., `starknet_simulateTransactions` with `SKIP_VALIDATE = false`, `starknet_estimateFee` with explicit signature checking, or any re-hash during state-sync ingestion) will compute H₂ instead of H₁. Because the user's ECDSA/Stark signature was produced over H₁ (the gateway hash), the simulation will report a spurious signature-verification failure for a perfectly valid transaction, or — in the opposite direction — accept a simulation whose hash does not match the on-chain hash. This is an authoritative-looking wrong value returned by the RPC layer.

---

### Likelihood Explanation

Any V3 transaction that sets `l2_gas = 0` and `l1_data_gas = 0` in its resource bounds triggers the bug. This is a common pattern for legacy-style V3 transactions that only specify L1 gas. The condition is fully user-controlled and requires no privileged access.

---

### Recommendation

Remove the heuristic downgrade in `From<ResourceBoundsMapping> for ValidResourceBounds`. The RPC layer should always produce `AllResources` for V3 transactions, matching the gateway path:

```rust
impl From<ResourceBoundsMapping> for ValidResourceBounds {
    fn from(value: ResourceBoundsMapping) -> Self {
        Self::AllResources(AllResourceBounds {
            l1_gas: value.l1_gas,
            l1_data_gas: value.l1_data_gas,
            l2_gas: value.l2_gas,
        })
    }
}
```

Alternatively, add a canonical `to_valid_resource_bounds()` method on `ResourceBoundsMapping` that is shared by both the gateway and RPC paths, so the variant-selection logic cannot diverge.

---

### Proof of Concept

1. Construct a V3 Invoke transaction with `resource_bounds = { l1_gas: { max_amount: 1000, max_price_per_unit: 1 }, l2_gas: { 0, 0 }, l1_data_gas: { 0, 0 } }`.
2. Submit it through the gateway. The gateway calls `InternalRpcInvokeTransactionV3::calculate_transaction_hash` → `get_invoke_transaction_v3_hash` → `get_tip_resource_bounds_hash` with `ValidResourceBounds::AllResources(...)`. This produces H₁ (4-element Poseidon input: tip + 3 packed resource felts).
3. Sign the transaction over H₁ and submit it. It is accepted and stored with hash H₁.
4. Now call `starknet_simulateTransactions` with the same transaction body. The RPC layer deserializes `resource_bounds` as `ResourceBoundsMapping { l2_gas: 0, l1_data_gas: 0 }`, converts via `From<ResourceBoundsMapping>` to `ValidResourceBounds::L1Gas(...)`, and recomputes the hash as H₂ (3-element Poseidon input: tip + 2 packed resource felts).
5. H₁ ≠ H₂. The simulation runs signature verification against H₂ and reports "invalid signature" for a transaction that is valid on-chain — a wrong authoritative value from the RPC layer. [3](#0-2) [5](#0-4) [1](#0-0) [6](#0-5)

pub fn get_tip_resource_bounds_hash(
    resource_bounds: &ValidResourceBounds,
    tip: &Tip,
) -> Result<Felt, StarknetApiError> {
    let l1_resource_bounds = resource_bounds.get_l1_bounds();
    let l2_resource_bounds = resource_bounds.get_l2_bounds();

    // L1 and L2 gas bounds always exist.
    // Old V3 txs always have L2 gas bounds of zero, but they exist.
    let mut resource_felts = vec![
        get_concat_resource(&l1_resource_bounds, L1_GAS)?,
        get_concat_resource(&l2_resource_bounds, L2_GAS)?,
    ];

    // For new V3 txs, need to also hash the data gas bounds.
    resource_felts.extend(match resource_bounds {
        ValidResourceBounds::L1Gas(_) => vec![],
        ValidResourceBounds::AllResources(all_resources) => {
            vec![get_concat_resource(&all_resources.l1_data_gas, L1_DATA_GAS)?]
        }
    });

    Ok(HashChain::new().chain(&tip.0.into()).chain_iter(resource_felts.iter()).get_poseidon_hash())
}

    let mut hash_chain = HashChain::new()
