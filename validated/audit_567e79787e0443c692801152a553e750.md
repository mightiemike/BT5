### Title
`AddClassAndExecutableUnsafe` Exposed Without Caller Authentication Allows Arbitrary CASM Injection - (File: crates/apollo_class_manager/src/class_manager.rs)

### Summary

`ClassManager::add_class_and_executable_unsafe` is exposed as a public `ClassManagerRequest` variant on the `RemoteClassManagerServer` with no authentication, no hash-consistency check, and no compilation verification. Any caller that can reach the class manager's remote endpoint can store an arbitrary compiled class (CASM) under any class hash with a forged `executable_class_hash_v2`, causing the blockifier to execute attacker-controlled bytecode for any declared contract.

### Finding Description

`add_class_and_executable_unsafe` stores a caller-supplied `(class_id, sierra, executable_class_hash_v2, executable_class)` tuple directly into `CachedClassStorage` without:

1. Verifying `H(executable_class) == executable_class_hash_v2`
2. Verifying `H(sierra) == class_id`
3. Compiling the Sierra to derive the canonical CASM
4. Checking class length or version
5. Authenticating the caller [1](#0-0) 

This is the direct analog of `addStrategy()` lacking `onlyOwner`: a function that controls which executable artifact is bound to a class identity is publicly callable with no access control.

The function is registered as a first-class `ClassManagerRequest` variant and dispatched by the `ComponentRequestHandler` implementation, meaning it is reachable over the `RemoteClassManagerServer` network endpoint: [2](#0-1) [3](#0-2) 

The `RemoteClassManagerServer` carries no authentication layer; it accepts any serialized `ClassManagerRequest` from any network peer.

The invariant broken: `add_class` (the safe path) compiles the Sierra, derives `executable_class_hash_v2` from the compiler output, and verifies it matches the user-supplied `compiled_class_hash` in the Declare transaction: [4](#0-3) 

`add_class_and_executable_unsafe` skips every one of these steps.

The `add_class` early-exit path compounds the issue: if a class is already present in storage, `add_class` returns the stored `executable_class_hash_v2` without recompiling: [5](#0-4) 

An attacker who pre-populates the class manager with `(class_id, sierra, H(legitimate_CASM), malicious_CASM)` causes `add_class` to return `H(legitimate_CASM)` to the gateway. The gateway then accepts a Declare transaction whose `compiled_class_hash = H(legitimate_CASM)`, but the stored executable is the attacker's CASM.

The transaction converter later calls `get_executable(class_id)` to build the `ClassInfo` for blockifier execution: [6](#0-5) 

It retrieves the malicious CASM. The blockifier executes it.

### Impact Explanation

**Critical. Wrong compiled class, CASM/native artifact, class hash, or contract code selected for execution.**

An attacker can bind an arbitrary CASM artifact to any class hash. Every future invocation of any contract deployed from that class executes the attacker's bytecode. This enables theft of funds, state corruption, or denial of service for any contract whose class was poisoned.

### Likelihood Explanation

In a distributed deployment the class manager runs as a `RemoteClassManagerServer`. Any process that can reach its network port — including a compromised gateway, batcher, or any internal network peer — can issue `AddClassAndExecutableUnsafe` requests. There is no authentication token, TLS client certificate, or IP allowlist enforced in the server code itself. The attacker only needs to know the Sierra source (public) to pre-compute `H(legitimate_CASM)` and craft the poisoned entry before the legitimate Declare transaction arrives.

### Recommendation

1. Remove `AddClassAndExecutableUnsafe` from the public `ClassManagerRequest` enum, or gate it behind a caller-identity check (e.g., a shared secret, mTLS, or a separate internal-only endpoint not exposed by `RemoteClassManagerServer`).
2. Inside `add_class_and_executable_unsafe`, add an assertion that `H(executable_class) == executable_class_hash_v2` before calling `set_class`, mirroring the check already present in the gateway's `convert_rpc_tx_to_internal`: [7](#0-6) 

3. Consider whether `add_class_and_executable_unsafe` needs to be a remote-callable variant at all; the only legitimate caller is the central sync client, which could use a local (in-process) channel instead.

    pub async fn add_class(&mut self, class: RawClass) -> ClassManagerResult<ClassHashes> {
        let sierra_class = SierraContractClass::try_from(&class)?;
        let class_hash = sierra_class.calculate_class_hash();
        if let Ok(Some(executable_class_hash_v2)) =
            self.classes.get_executable_class_hash_v2(class_hash)
        {
            // Class already exists.
            return Ok(ClassHashes { class_hash, executable_class_hash_v2 });
        }

        let compilation_start_time = Instant::now();
        let (raw_executable_class, executable_class_hash_v2) =
            self.compiler.compile(class.clone()).await.map_err(|err| match err {
                SierraCompilerClientError::SierraCompilerError(error) => {
                    ClassManagerError::SierraCompiler { class_hash, error }
                }
                SierraCompilerClientError::ClientError(error) => {
                    ClassManagerError::Client(error.to_string())
                }
            })?;
        debug!(
            %class_hash,
            compiled_class_hash = %executable_class_hash_v2,
            compilation_elapsed_ms = compilation_start_time.elapsed().as_millis(),
            class_size_bytes =
                class.size().map_or("Failed to get class size".to_owned(), |size| size.to_string()),
            casm_size_bytes =
                raw_executable_class.size().map_or("Failed to get casm size".to_owned(), |size| size.to_string()),
            "Finished compiling class."
        );

        self.validate_class_length(&raw_executable_class)?;
        Self::validate_class_version(&sierra_class)?;
        self.classes.set_class(
            class_hash,
            class,
            executable_class_hash_v2,
            raw_executable_class,
        )?;

        let class_hashes = ClassHashes { class_hash, executable_class_hash_v2 };
        Ok(class_hashes)
    }

#[async_trait]
impl ComponentRequestHandler<ClassManagerRequest, ClassManagerResponse> for ClassManager {
    async fn handle_request(&mut self, request: ClassManagerRequest) -> ClassManagerResponse {
        let dynamic_config: apollo_class_manager_config::config::ClassManagerDynamicConfig = self
            .0
            .config_manager_client
            .get_class_manager_dynamic_config()
            .await
            .expect("Should be able to get class manager dynamic config");
        self.0.update_dynamic_config(dynamic_config);

        match request {
            ClassManagerRequest::AddClass(class) => {
                ClassManagerResponse::AddClass(self.0.add_class(class).await)
            }
            ClassManagerRequest::AddClassAndExecutableUnsafe(
                class_id,
                class,
                executable_class_hash_v2,
                executable_class,
            ) => ClassManagerResponse::AddClassAndExecutableUnsafe(
                self.0.add_class_and_executable_unsafe(
                    class_id,
                    class,
                    executable_class_hash_v2,
                    executable_class,
                ),
            ),
            ClassManagerRequest::AddDeprecatedClass(class_id, class) => {
                ClassManagerResponse::AddDeprecatedClass(
                    self.0.add_deprecated_class(class_id, class),
                )
            }
            ClassManagerRequest::GetExecutable(class_id) => {
                ClassManagerResponse::GetExecutable(self.0.get_executable(class_id))
            }
            ClassManagerRequest::GetSierra(class_id) => {
                ClassManagerResponse::GetSierra(self.0.get_sierra(class_id))
            }
            ClassManagerRequest::GetExecutableClassHashV2(class_id) => {
                let result = self.0.get_executable_class_hash_v2(class_id);
                ClassManagerResponse::GetExecutableClassHashV2(result)
            }
        }
    }
