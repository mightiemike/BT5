### Title
`ENABLE_SECP_OPS` Guard Bypassed via 4-Byte Opcode Path in `ChiaDialect::op` — (`src/chia_dialect.rs`)

### Summary

The 4-byte opcode dispatch branch in `ChiaDialect::op` unconditionally routes opcodes `0x13d61f00` and `0x1c3a8f00` to `op_secp256k1_verify` and `op_secp256r1_verify` without checking `ClvmFlags::ENABLE_SECP_OPS`. The 1-byte opcode path (opcodes 64/65) correctly guards behind that flag, but the 4-byte path does not, creating a complete bypass.

### Finding Description

In `src/chia_dialect.rs`, the `op_len == 4` branch: [1](#0-0) 

dispatches directly to `op_secp256k1_verify` / `op_secp256r1_verify` for opcodes `0x13d61f00` and `0x1c3a8f00` with no flag check: [2](#0-1) 

The 1-byte path, by contrast, correctly gates opcodes 64 and 65: [3](#0-2) 

Neither `op_secp256k1_verify` nor `op_secp256r1_verify` checks the flag internally — both accept `_flags: ClvmFlags` and ignore it entirely: [4](#0-3) [5](#0-4) 

On a valid signature, both return `Ok(Reduction(cost, a.nil()))` — nil — not an error: [6](#0-5) [7](#0-6) 

### Impact Explanation

Any caller that constructs `ChiaDialect::new(ClvmFlags::empty())` — or any flags combination that omits `ENABLE_SECP_OPS` — and then runs a CLVM program containing a 4-byte atom `0x13d61f00` or `0x1c3a8f00` will silently execute a full secp256k1 or secp256r1 signature verification. The result is `nil` (success), not `EvalErr::Unimplemented`. The `ENABLE_SECP_OPS` flag, which is documented as the hard-fork gate for secp ops, provides no protection via the 4-byte opcode path. [8](#0-7) 

### Likelihood Explanation

The attacker path is trivially reachable: craft a CLVM program whose operator atom is the 4-byte literal `[0x13, 0xd6, 0x1f, 0x00]` with valid secp arguments. No special privileges, no compromised nodes, no social engineering. Any public API that accepts attacker-controlled CLVM and uses a dialect without `ENABLE_SECP_OPS` is affected. [9](#0-8) 

### Recommendation

Add the `ENABLE_SECP_OPS` guard to the 4-byte dispatch branch, mirroring the 1-byte path:

```rust
let f = match opcode {
    0x13d61f00 if flags.contains(ClvmFlags::ENABLE_SECP_OPS) => op_secp256k1_verify,
    0x1c3a8f00 if flags.contains(ClvmFlags::ENABLE_SECP_OPS) => op_secp256r1_verify,
    _ => {
        return unknown_operator(allocator, o, argument_list, flags, max_cost);
    }
};
``` [10](#0-9)
