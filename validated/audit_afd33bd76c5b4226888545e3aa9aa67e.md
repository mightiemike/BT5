### Title
Checkpoint Advanced Before Pool-Balance Check in `processEventsAndClaimBonus` Causes Permanent Bonus Loss on Redelegation Slash — (`x/tieredrewards/keeper/slash.go`, `x/tieredrewards/keeper/claim_rewards.go`)

---

### Summary

`processEventsAndClaimBonus` mutates the `pos` pointer — advancing `LastBonusAccrual`, `LastEventSeq`, and `LastKnownBonded` — **before** it checks whether the rewards pool has sufficient balance. When the pool is empty and `ErrInsufficientBonusPool` is returned, `slashRedelegationPosition` silently swallows the error and persists the already-mutated position via `setPositionWithState`. The pre-slash bonus segment is permanently erased from the position's accounting with no coins ever paid.

---

### Finding Description

**Exact mutation order in `processEventsAndClaimBonus`** (`claim_rewards.go`):

1. The event loop (lines 172–198) calls `pos.UpdateLastEventSeq(entry.Seq)` and `k.decrementEventRefCount(...)` for every event — consuming them in-place on the pointer.
2. Line 215: `applyBonusAccrualCheckpoint(&pos.Position, blockTime)` advances `pos.LastBonusAccrual` to `blockTime`.
3. Line 217: `pos.UpdateLastKnownBonded(bonded)` updates the bonded-state checkpoint.
4. Line 230: `k.sufficientBonusPoolBalance(ctx, bonusCoins)` — **only now** checks the pool. If the pool is empty it returns `ErrInsufficientBonusPool`. [1](#0-0) 

Because `pos` is passed as a pointer (`&pos`) from `slashRedelegationPosition`, all three mutations are visible to the caller when the error is returned.

**Error swallowing in `slashRedelegationPosition`** (`slash.go`):

```
if _, err := k.processEventsAndClaimBonus(ctx, &pos); err != nil {
    if errors.Is(err, types.ErrInsufficientBonusPool) {
        k.logger(ctx).Error(...)   // swallowed
    } else {
        return err
    }
}
// pos.LastBonusAccrual is already at blockTime here
return k.setPositionWithState(ctx, pos, ...)
``` [2](#0-1) 

`setPositionWithState` then persists the position with the advanced `LastBonusAccrual`. The next call to `processEventsAndClaimBonus` for this position will start its `segmentStart` from the slash block time, so the entire pre-slash accrual window is gone. [3](#0-2) 

The hook entry point is `BeforeRedelegationSlashed` → `slashRedelegationPosition`, a standard staking hook that fires on every redelegation slash: [4](#0-3) 

---

### Impact Explanation

A position owner who has been delegated for N days accrues bonus proportional to `shares × tokensPerShare × BonusApy × N / SecondsPerYear`. If a redelegation slash fires while `RewardsPoolName` is empty (or underfunded), the entire accrued-but-unpaid segment is silently discarded. When the pool is later refilled and the owner calls `ClaimTierRewards`, `LastBonusAccrual` is already at the slash block time, so the pre-slash segment yields zero bonus. The owner permanently loses those rewards with no recourse.

---

### Likelihood Explanation

The `RewardsPoolName` pool can be empty at any point before it is funded, or transiently empty after a large batch of claims. Redelegation slashes are a normal chain event (double-sign or downtime evidence). No privileged access is required; the bug fires automatically whenever both conditions coincide. The comment in `slash.go` ("Deliberately forgo bonus rewards if pool is insufficient to prevent chain halt") confirms the intent was to skip payment, but the implementation also skips resetting the checkpoint, making the loss permanent rather than deferred. [5](#0-4) 

---

### Recommendation

Move `applyBonusAccrualCheckpoint`, `UpdateLastKnownBonded`, and all event-seq/ref-count mutations to **after** the pool-balance check succeeds, or — if the intent is to skip payment when the pool is empty — do not advance the checkpoint at all, so the segment can be claimed once the pool is refilled. A minimal fix:

```go
// In processEventsAndClaimBonus, reorder:
if err := k.sufficientBonusPoolBalance(ctx, bonusCoins); err != nil {
    return nil, err   // return BEFORE mutating pos checkpoints
}
applyBonusAccrualCheckpoint(&pos.Position, blockTime)
pos.UpdateLastKnownBonded(bonded)
```

Alternatively, store the unclaimable bonus amount in a separate "pending bonus" field on the position so it can be paid once the pool is replenished.

func (k Keeper) processEventsAndClaimBonus(ctx context.Context, pos *types.PositionState) (sdk.Coins, error) {
	// Rewards should have been claimed before undelegation
	if !pos.IsDelegated() {
		return sdk.NewCoins(), nil
	}

	valAddr, err := sdk.ValAddressFromBech32(pos.Delegation.ValidatorAddress)
	if err != nil {
		return nil, err
	}

	events, err := k.getValidatorEventsSince(ctx, valAddr, pos.LastEventSeq)
	if err != nil {
		return nil, err
	}

	sdkCtx := sdk.UnwrapSDKContext(ctx)
	blockTime := sdkCtx.BlockTime()

	totalBonus := math.ZeroInt()
	// Use the persisted bonded state from the last replay, not a hardcoded default.
	// This prevents overpaying bonus for unbonded gaps between claims.
	bonded := pos.LastKnownBonded
	segmentStart := pos.LastBonusAccrual

	tier, err := k.getTier(ctx, pos.TierId)
	if err != nil {
		return nil, err
	}

	for _, entry := range events {
		evt := entry.Event

		if bonded {
			// Compute bonus for the bonded segment [segmentStart, eventTime]
			// using the snapshot rate at the event.
			bonus := k.computeSegmentBonus(*pos, tier, segmentStart, evt.Timestamp, evt.TokensPerShare)
			totalBonus = totalBonus.Add(bonus)
		}

		// Update bonded state based on event type.
		switch evt.EventType {
		case types.ValidatorEventType_VALIDATOR_EVENT_TYPE_UNBOND:
			bonded = false
		case types.ValidatorEventType_VALIDATOR_EVENT_TYPE_BOND:
			bonded = true
		case types.ValidatorEventType_VALIDATOR_EVENT_TYPE_SLASH:
			// Slash doesn't change bonded state.
		}

		segmentStart = evt.Timestamp
		pos.UpdateLastEventSeq(entry.Seq)

		// Decrement reference count.
		if err := k.decrementEventRefCount(ctx, valAddr, entry.Seq); err != nil {
			return nil, err
		}
	}

	val, err := k.stakingKeeper.GetValidator(ctx, valAddr)
	if err != nil {
		return nil, err
	}
	// Defensive: validator bond status check
	if bonded && val.IsBonded() {
		currentRate, err := k.getTokensPerShare(ctx, valAddr)
		if err != nil {
			return nil, err
		}
		bonus := k.computeSegmentBonus(*pos, tier, segmentStart, blockTime, currentRate)
		totalBonus = totalBonus.Add(bonus)
	}

	applyBonusAccrualCheckpoint(&pos.Position, blockTime)
	// Persist the bonded state so the next replay starts correctly.
	pos.UpdateLastKnownBonded(bonded)

	if totalBonus.IsZero() {
		return sdk.NewCoins(), nil
	}

	bondDenom, err := k.stakingKeeper.BondDenom(ctx)
	if err != nil {
		return nil, err
	}

	bonusCoins := sdk.NewCoins(sdk.NewCoin(bondDenom, totalBonus))

	if err := k.sufficientBonusPoolBalance(ctx, bonusCoins); err != nil {
		return nil, err
	}

	ownerAddr, err := sdk.AccAddressFromBech32(pos.Owner)
	if err != nil {
		return nil, errorsmod.Wrap(sdkerrors.ErrInvalidAddress, "invalid owner address")
	}

	if err := k.bankKeeper.SendCoinsFromModuleToAccount(ctx, types.RewardsPoolName, ownerAddr, bonusCoins); err != nil {
		return nil, err
	}

	if err := sdkCtx.EventManager().EmitTypedEvent(&types.EventBonusRewardsClaimed{
		PositionId: pos.Id,
		Owner:      pos.Owner,
		Rewards:    bonusCoins,
	}); err != nil {
		return nil, err
	}

	return bonusCoins, nil
}
