Audit Report

## Title
`SwapAllowlistExtension` Checks Router Address Instead of Actual User on Router-Mediated Swaps, Allowing Allowlist Bypass — (`metric-periphery/contracts/extensions/SwapAllowlistExtension.sol`)

## Summary
`SwapAllowlistExtension.beforeSwap` gates swaps by checking `allowedSwapper[pool][sender]`, where `sender` is `msg.sender` of the pool's `swap` call. When a user routes through `MetricOmmSimpleRouter`, the router is `msg.sender` of `pool.swap`, so the extension checks whether the **router** is allowlisted rather than the actual end user. If the pool admin allowlists the router to enable normal routing, any non-allowlisted user can bypass the curated allowlist by routing through the router.

## Finding Description
In `MetricOmmPool.swap`, the pool calls `_beforeSwap(msg.sender, ...)` at line 230, passing its own `msg.sender` (the immediate caller) as `sender`: [1](#0-0) 

`ExtensionCalling._beforeSwap` then forwards that `sender` directly to the extension via `abi.encodeCall`: [2](#0-1) 

`SwapAllowlistExtension.beforeSwap` checks `allowedSwapper[msg.sender][sender]`, where `msg.sender` is the pool and `sender` is the forwarded caller: [3](#0-2) 

When `MetricOmmSimpleRouter.exactInputSingle` (or `exactInput`, `exactOutputSingle`, `exactOutput`) calls `pool.swap(...)`, the router is `msg.sender` of that call: [4](#0-3) 

The pool therefore passes the router address as `sender` to the extension. The extension evaluates `allowedSwapper[pool][router]`, never consulting `allowedSwapper[pool][actualUser]`. If the pool admin allowlists the router (a natural operational step to keep the pool usable), every non-allowlisted user can bypass the access control with a single router call.

**Attack path:**
1. Pool admin deploys pool with `SwapAllowlistExtension`, intending to restrict trading to KYC'd addresses.
2. Pool admin calls `setAllowedToSwap(pool, router, true)` so the router can forward swaps.
3. Non-allowlisted user `bob` calls `MetricOmmSimpleRouter.exactInputSingle(...)`.
4. Router calls `pool.swap(...)` — pool's `msg.sender` = router.
5. Pool dispatches `extension.beforeSwap(router, ...)`.
6. Extension evaluates `allowedSwapper[pool][router]` → `true`. Hook passes.
7. Bob's swap executes on the curated pool, bypassing the intended access control.

## Impact Explanation
Curated pools (institutional, regulatory, or partner-restricted) lose their access-control guarantee. Any address can trade by routing through the supported periphery router, which is the primary public swap entrypoint. This constitutes broken core pool functionality — the allowlist extension provides no meaningful protection on the router path. The exact corrupted value is the extension's authorization decision: `allowedSwapper[pool][router]` is evaluated instead of `allowedSwapper[pool][actualUser]`, causing the hook to return `IMetricOmmExtensions.beforeSwap.selector` (pass) when it should revert with `NotAllowedToSwap`.

## Likelihood Explanation
`MetricOmmSimpleRouter` is the documented, supported swap entrypoint. Pool admins who configure an allowlist will naturally also allowlist the router to keep the pool usable — the exact condition that opens the bypass. No special privilege or unusual transaction ordering is required; a single `exactInputSingle` call suffices. The bypass is repeatable by any address on every router-mediated swap.

## Recommendation
The pool must forward the **original end-user address** — not `msg.sender` of the `swap` call — as `sender` to extension hooks. Two complementary fixes:

1. **In `MetricOmmPool.swap`**: accept an explicit `sender` parameter (the originating user) and forward it to `ExtensionCalling._beforeSwap`, rather than using `msg.sender`.
2. **In `MetricOmmSimpleRouter`**: pass the original `msg.sender` (the end user) as the `sender` argument when calling `pool.swap`, so the pool can forward the correct identity to extensions.

## Proof of Concept
```
Setup:
  pool configured with SwapAllowlistExtension
  allowedSwapper[pool][alice]  = true   // alice is the curated user
  allowedSwapper[pool][bob]    = false  // bob is NOT allowed
  allowedSwapper[pool][router] = true   // router allowlisted for normal operation

Direct swap (bob → pool.swap directly):
  pool.swap(...) called by bob
  → _beforeSwap(bob, ...)
  → extension.beforeSwap(bob, ...)
  → allowedSwapper[pool][bob] == false → revert NotAllowedToSwap ✓

Router swap (bob → router → pool.swap):
  router.exactInputSingle({pool: pool, ...}) called by bob
  → pool.swap(...) called by router (msg.sender = router)
  → _beforeSwap(router, ...)
  → extension.beforeSwap(router, ...)
  → allowedSwapper[pool][router] == true → hook passes ✗
  → bob's swap executes on the curated pool
```
