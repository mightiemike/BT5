Audit Report

## Title
SwapAllowlistExtension Gates the Router Address Instead of the End User, Enabling Full Allowlist Bypass via `MetricOmmSimpleRouter` — (File: `metric-periphery/contracts/extensions/SwapAllowlistExtension.sol`)

## Summary
`SwapAllowlistExtension.beforeSwap` checks the `sender` argument, which `MetricOmmPool.swap` sets to its own `msg.sender`. When swaps are routed through `MetricOmmSimpleRouter`, `msg.sender` of `pool.swap` is the router contract, not the end user. A pool admin who allowlists the router address to enable router-mediated swaps inadvertently grants every caller of the router access to the pool, completely defeating the per-user allowlist.

## Finding Description
`MetricOmmPool.swap` passes `msg.sender` directly as the `sender` argument to `_beforeSwap`: [1](#0-0) 

`SwapAllowlistExtension.beforeSwap` then evaluates `allowedSwapper[msg.sender][sender]`, where `msg.sender` is the pool and `sender` is the value forwarded from the pool — the router address when the router initiates the swap: [2](#0-1) 

`MetricOmmSimpleRouter.exactInputSingle` calls `pool.swap` directly, making the router `msg.sender` of that call, and passes `params.extensionData` verbatim without injecting the original caller's address: [3](#0-2) 

The same pattern applies to `exactInput`, `exactOutputSingle`, and `exactOutput`. The extension has no on-chain path to the real initiator. When the admin calls `setAllowedToSwap(pool, address(router), true)` — the only rational configuration to allow allowlisted users to trade via the router — the single entry allowlists the router contract, meaning every address that calls any router function passes the guard. [4](#0-3) 

## Impact Explanation
Any user who calls `MetricOmmSimpleRouter` on a pool where the router is allowlisted can execute swaps regardless of whether their own address is on the allowlist. On pools designed to restrict trading to a curated set of counterparties (e.g., private institutional pools, KYC-gated pools), this allows arbitrary external actors to drain LP assets at oracle-fair prices and generate unauthorized volume. The loss is bounded only by available pool liquidity and is directly reachable by any unprivileged address. This constitutes a broken core pool functionality causing direct loss of user/LP principal and an admin-boundary break where the allowlist access control is bypassed by an unprivileged path.

## Likelihood Explanation
The trigger is a routine, non-malicious admin action: allowlisting the router so that the pool's own allowlisted users can trade through the standard periphery. No exploit contract, flash loan, or privileged key is required. `MetricOmmSimpleRouter` is the primary user-facing entry point, so the probability that a pool admin allowlists it is high. Any user who discovers the router is allowlisted can immediately exploit it without any special capability.

## Recommendation
The extension must gate the economic initiator, not the intermediary contract. Two sound approaches:

1. **Router-injected identity**: Have `MetricOmmSimpleRouter` ABI-encode `msg.sender` into a standard prefix of `extensionData` before forwarding to the pool. `SwapAllowlistExtension` decodes and checks that address when `sender` is a known router.
2. **Originator field**: Add an explicit `originator` field to `pool.swap` that the router populates with `msg.sender`, and have extensions check that field.

Until fixed, pools that need per-user access control must require users to call `pool.swap` directly and must **not** allowlist the router.

## Proof of Concept
```
1. Deploy pool with SwapAllowlistExtension configured as beforeSwap hook.
2. Admin calls: swapExtension.setAllowedToSwap(pool, address(router), true)
   (intending to allow allowlisted users to trade via the router)
3. Alice (address NOT in allowedSwapper[pool]) calls:
     router.exactInputSingle(ExactInputSingleParams{pool: pool, ...})
4. Router calls: pool.swap(recipient, zeroForOne, amount, limit, "", extensionData)
   → msg.sender of pool.swap = address(router)
5. Pool calls: _beforeSwap(address(router), recipient, ...)
6. Extension evaluates: allowedSwapper[pool][address(router)] == true  ✓
7. Swap executes. Alice receives output tokens. Allowlist is bypassed.
```
Alice's address is never consulted. The allowlist check passes solely because the router is allowlisted, not because Alice is. [5](#0-4) [6](#0-5)
