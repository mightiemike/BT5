Audit Report

## Title
`SwapAllowlistExtension.beforeSwap` checks the router address instead of the originating user, allowing any unprivileged caller to bypass the swap allowlist via `MetricOmmSimpleRouter` — (`metric-periphery/contracts/extensions/SwapAllowlistExtension.sol`)

## Summary
`SwapAllowlistExtension.beforeSwap` gates swaps by checking `allowedSwapper[pool][sender]`, where `sender` is `msg.sender` of the pool's `swap` call. When a user routes through `MetricOmmSimpleRouter`, the pool's `msg.sender` is the router contract, not the originating user. Any pool admin who allowlists the router to support standard user flows inadvertently opens the pool to every address on-chain, as any non-allowlisted user can bypass the guard by routing through the public router.

## Finding Description
`SwapAllowlistExtension.beforeSwap` performs the check:
```solidity
if (!allowAllSwappers[msg.sender] && !allowedSwapper[msg.sender][sender]) {
    revert IMetricOmmPoolActions.NotAllowedToSwap();
}
```
where `msg.sender` is the pool (the extension's caller) and `sender` is the first argument forwarded by `ExtensionCalling._beforeSwap`.

`MetricOmmPool.swap` passes its own `msg.sender` as the `sender` argument to `_beforeSwap`:
```solidity
_beforeSwap(msg.sender, recipient, zeroForOne, amountSpecified, ...);
```

`MetricOmmSimpleRouter.exactInputSingle` (and all other `exact*` functions) calls `pool.swap(params.recipient, ...)` directly, making the **router contract** the `msg.sender` seen by the pool:
```solidity
(int128 amount0Delta, int128 amount1Delta) = IMetricOmmPoolActions(params.pool)
    .swap(params.recipient, params.zeroForOne, ...);
```

`ExtensionCalling._beforeSwap` then forwards this router address as `sender` to every configured extension. The `SwapAllowlistExtension` therefore checks whether the **router** is allowlisted, not whether the **originating user** is allowlisted.

This creates an irresolvable dilemma:
1. **Router not allowlisted**: All router-mediated swaps revert, even for individually allowlisted users.
2. **Router allowlisted** (the only operationally viable choice): Every address on-chain can bypass the allowlist by calling `MetricOmmSimpleRouter.exact*` instead of calling the pool directly.

The existing test suite (`FullMetricExtension.t.sol`) only exercises direct pool calls via `TestCaller`, never router-mediated swaps, so this path is untested.

## Impact Explanation
A pool admin who deploys a curated pool (KYC-gated, institution-only, or regulatory-restricted) with `SwapAllowlistExtension` and allowlists the router to support standard user flows inadvertently opens the pool to every address on-chain. Any non-allowlisted user can execute swaps against the pool's LP positions by routing through `MetricOmmSimpleRouter`. This constitutes a broken core pool functionality and admin-boundary break: the access-control invariant the pool was deployed to enforce is silently violated on every router-mediated swap, exposing LP positions priced for a restricted counterparty set to the full adversarial public.

## Likelihood Explanation
`MetricOmmSimpleRouter` is the primary user-facing swap interface. The pool admin has no on-chain mechanism to enforce the allowlist for router users without blocking the router entirely. No special knowledge or capital is required — any address can call `MetricOmmSimpleRouter.exact*` with the target pool. The bypass is deterministic and repeatable in every block.

## Recommendation
The `SwapAllowlistExtension` must check the economically relevant actor, not the immediate `msg.sender` of the pool. The simplest fix is to check `recipient` (the second parameter of `beforeSwap`) instead of `sender`, since the router passes the originating user as `recipient`. This correctly identifies who receives the output tokens without requiring router changes. Alternatively, require the router to forward the originating user in `extensionData` and decode it in the extension, but this requires coordinated router and extension updates.

## Proof of Concept
```
Setup:
  1. Deploy MetricOmmPool with SwapAllowlistExtension as a beforeSwap hook.
  2. Pool admin calls swapExtension.setAllowedToSwap(pool, alice, true)
     — only alice is allowlisted.
  3. Pool admin calls swapExtension.setAllowedToSwap(pool, router, true)
     — router must be allowlisted for alice to use it.
  4. LP adds liquidity to the pool.

Attack (bob is NOT allowlisted):
  5. bob calls MetricOmmSimpleRouter.exactInputSingle({pool, recipient=bob, ...})
  6. Router calls pool.swap(bob, zeroForOne, amount, priceLimit, "", "")
  7. Pool calls ExtensionCalling._beforeSwap(sender=router, recipient=bob, ...)
  8. SwapAllowlistExtension checks allowedSwapper[pool][router] → true (step 3)
  9. Swap executes successfully.

Result:
  bob, a non-allowlisted address, completes a swap on a curated pool.
  The allowlist guard is fully bypassed.

Direct call (control):
  10. bob calls pool.swap(bob, ...) directly.
  11. Pool calls extension with sender=bob.
  12. allowedSwapper[pool][bob] → false → revert NotAllowedToSwap.
