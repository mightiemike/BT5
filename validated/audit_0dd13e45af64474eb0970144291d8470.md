### Title
Deprecated `verify_transaction_inclusion` Remains Publicly Callable, Enabling 64-Byte Merkle Proof Forgery — (File: `contract/src/lib.rs`)

---

### Summary

`verify_transaction_inclusion` (v1) is marked `#[deprecated]` but is never removed or access-gated. It remains a live, unpermissioned public entry point on the NEAR contract. It lacks the coinbase-proof guard that `verify_transaction_inclusion_v2` adds to defeat the 64-byte Merkle-node forgery. Any unprivileged NEAR caller can invoke v1 directly and obtain a `true` proof result for a Bitcoin transaction that does not exist.

---

### Finding Description

`verify_transaction_inclusion` at `contract/src/lib.rs:288` carries a `#[deprecated]` attribute and an explicit code comment warning that it "may return `true` if the provided `tx_id` is a hash of an internal node in the Merkle tree." [1](#0-0) 

Despite the deprecation, the function is still decorated with `#[pause]` (not `#[private]`, not removed), making it a fully reachable public method callable by any NEAR account. [2](#0-1) 

The v1 proof path is:

```
compute_root_from_merkle_proof(tx_id, tx_index, &merkle_proof) == header.merkle_root
``` [3](#0-2) 

`compute_root_from_merkle_proof` is a pure positional hash-chain walk with no constraint that `tx_id` must be a leaf-level (transaction) hash: [4](#0-3) 

The only guard added in v2 is a coinbase-proof cross-check that forces the prover to also supply a valid proof for the coinbase transaction at index 0, which anchors the tree depth and prevents an internal node from being presented as a leaf: [5](#0-4) 

Because v1 is still callable, that guard is entirely bypassed.

---

### Impact Explanation

An attacker who knows the Merkle tree of any confirmed Bitcoin block can:

1. Pick an internal node `N` at depth `d` (a 32-byte hash that is the double-SHA256 of two child hashes).
2. Compute the remaining `(tree_depth − d)` proof steps from `N` up to the Merkle root.
3. Call `verify_transaction_inclusion` with `tx_id = N`, `tx_index` set to the position that matches `N`'s parity at each level, and `merkle_proof` = those `(tree_depth − d)` hashes.
4. `compute_root_from_merkle_proof` walks from `N` to the root and returns the correct Merkle root.
5. The equality check passes; the function returns `true`.

The corrupted proof result is: **a non-existent Bitcoin transaction is certified as included in a confirmed block**. Any consumer contract (bridge, atomic swap, cross-chain lending protocol) that calls `verify_transaction_inclusion` and acts on a `true` result is vulnerable to fund theft or fraudulent state transitions. [6](#0-5) 

---

### Likelihood Explanation

- The 64-byte Merkle-node forgery is a well-documented Bitcoin attack (referenced in the contract's own comments and linked to the BitMEX research post).
- The attacker needs only a confirmed Bitcoin block and its Merkle tree — both are public.
- No privileged role, leaked key, or social engineering is required.
- The entry point is a standard public NEAR contract call.

Likelihood: **Medium-High**. The technique is known, the data is public, and the entry path is open to any NEAR account.

---

### Recommendation

Remove `verify_transaction_inclusion` (v1) from the contract entirely, mirroring the tBTC resolution of removing the gameable SPV proof path. If backward compatibility must be preserved temporarily, gate the function with `#[private]` or a privileged role so it cannot be called by unprivileged accounts. All consumers should be migrated to `verify_transaction_inclusion_v2`. [7](#0-6) 

---

### Proof of Concept

Given a confirmed Bitcoin block `B` with Merkle root `R` and `2^k` transactions:

1. Obtain the Merkle tree of `B`. Let `N` = the internal node at depth 1, left child (i.e., `hash(tx[0] || tx[1])`).
2. The proof from `N` to `R` is a single hash: the right sibling of `N` at depth 1, call it `S`.
3. Call on NEAR:

```
verify_transaction_inclusion({
    tx_id:             N,          // internal node, not a real tx hash
    tx_block_blockhash: B.hash,
    tx_index:          0,          // even position → left child
    merkle_proof:      [S],        // one-element proof
    confirmations:     1
})
```

4. Inside `compute_root_from_merkle_proof`: `current_hash = hash(N, S) = R`. The equality `R == header.merkle_root` holds. The function returns `true`.

No real transaction with hash `N` exists in block `B`. The proof result is forged. [8](#0-7) [9](#0-8)

    pub fn verify_transaction_inclusion(&self, #[serializer(borsh)] args: ProofArgs) -> bool {
        require!(
            args.confirmations <= self.gc_threshold,
            "The required number of confirmations exceeds the number of blocks stored in memory"
        );

        let heaviest_block_header = self
            .headers_pool
            .get(&self.mainchain_tip_blockhash)
            .unwrap_or_else(|| env::panic_str(ERR_KEY_NOT_EXIST));
        let target_block_height = self
            .mainchain_header_to_height
            .get(&args.tx_block_blockhash)
            .unwrap_or_else(|| env::panic_str("block does not belong to the current main chain"));

        // Check requested confirmations. No need to compute proof if insufficient confirmations.
        require!(
            (heaviest_block_header.block_height).saturating_sub(target_block_height) + 1
                >= args.confirmations,
            "Not enough blocks confirmed"
        );

        let header = self
            .headers_pool
            .get(&args.tx_block_blockhash)
            .unwrap_or_else(|| env::panic_str("cannot find requested transaction block"));

        require!(!args.merkle_proof.is_empty(), "Merkle proof is empty");

        // compute merkle tree root and check if it matches block's original merkle tree root

            args.tx_id,
            usize::try_from(args.tx_index).unwrap(),
            &args.merkle_proof,

    }

pub fn compute_root_from_merkle_proof(
    transaction_hash: H256,
    transaction_position: usize,
    merkle_proof: &Vec<H256>,
) -> H256 {
    let mut current_hash = transaction_hash;
    let mut current_position = transaction_position;

    for proof_hash in merkle_proof {
        if current_position % 2 == 0 {
            current_hash = compute_hash(&current_hash, proof_hash);
        } else {
            current_hash = compute_hash(proof_hash, &current_hash);
        }
        current_position /= 2;
    }

    current_hash
}
