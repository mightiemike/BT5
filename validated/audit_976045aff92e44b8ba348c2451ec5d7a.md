### Title
Unprivileged Caller Can Invoke `run_mainchain_gc` With Arbitrary `batch_size`, Permanently Erasing Canonical Block Headers and Breaking SPV Proof Verification - (`contract/src/lib.rs`)

---

### Summary

`run_mainchain_gc` is a public, state-mutating contract method with no caller access control. Any NEAR account can call it with an arbitrarily large `batch_size`, causing the contract to immediately remove the maximum possible number of canonical block headers from on-chain storage. This permanently and irreversibly corrupts the canonical chain state, causing all subsequent `verify_transaction_inclusion` and `verify_transaction_inclusion_v2` calls for the erased blocks to panic rather than return a valid proof result.

---

### Finding Description

The `run_mainchain_gc` function is decorated only with `#[pause(except(roles(Role::UnrestrictedRunGC)))]` and carries no role-based caller restriction when the contract is unpaused. [1](#0-0) 

When called, it computes how many headers exceed `gc_threshold` and removes up to `min(total_amount_to_remove, batch_size)` of the oldest canonical headers: [2](#0-1) 

For each removed height it calls `remove_block_header`, which deletes the entry from both `mainchain_header_to_height` and `headers_pool`: [3](#0-2) 

It also advances `mainchain_initial_blockhash` to the new oldest surviving block. All three mutations are permanent — no contract method re-inserts removed headers.

The intended caller is `submit_blocks`, which invokes `run_mainchain_gc(num_of_headers)` internally after adding new blocks, keeping removal gradual and proportional: [4](#0-3) 

`submit_blocks` is protected by `#[trusted_relayer]`, so only registered relayers reach it. But `run_mainchain_gc` is separately exposed as a top-level public method with no equivalent guard. An attacker calls it directly with `batch_size = u64::MAX`, collapsing all excess removal into a single transaction instead of the gradual per-batch removal the relayer would perform.

---

### Impact Explanation

`verify_transaction_inclusion` looks up the target block in `mainchain_header_to_height` and `headers_pool`: [5](#0-4) 

Once a block is removed by GC, both lookups panic unconditionally (`unwrap_or_else(|| env::panic_str(...))`). The proof call aborts rather than returning `false` — the consuming dApp receives a hard failure, not a graceful rejection. Because there is no re-insertion path, the damage is permanent for every block that was erased. Any cross-chain bridge, payment channel, or custody protocol relying on SPV proofs for those blocks loses the ability to verify historical transactions forever.

The same applies to `verify_transaction_inclusion_v2`, which delegates to the v1 path after its coinbase check: [6](#0-5) 

---

### Likelihood Explanation

The entry point is a plain public NEAR contract method requiring no stake, no role, and no deposit beyond gas. Any NEAR account — including a freshly created one — can call it. The attacker needs only to observe that the stored chain length exceeds `gc_threshold` (visible via `get_mainchain_size`) and then submit one transaction. The attack is cheap, requires no privileged access, and cannot be reversed by the protocol.

---

### Recommendation

Add a caller restriction to `run_mainchain_gc` so that only accounts holding a designated role (e.g., `Role::DAO` or a new `GCManager` role) or the contract itself (via `#[private]`) may invoke it externally. The internal call from `submit_blocks` already passes through the `#[trusted_relayer]` guard, so restricting the public surface does not break the normal GC flow. For example:

```rust
// Only allow the contract itself to call run_mainchain_gc externally,
// or gate it behind an explicit role check.
#[private]
#[pause(except(roles(Role::UnrestrictedRunGC)))]
pub fn run_mainchain_gc(&mut self, batch_size: u64) { … }
```

Alternatively, expose a separate, role-gated `trigger_gc(batch_size)` entry point and make `run_mainchain_gc` a private helper.

---

### Proof of Concept

1. Deploy the contract with `gc_threshold = 10` and submit 20 blocks via the trusted relayer. The chain now stores 20 headers, 10 of which are excess.
2. From any unprivileged NEAR account, call:
   ```
   run_mainchain_gc(batch_size = 18446744073709551615)  // u64::MAX
   ```
3. The contract removes all 10 excess headers in one call, advancing `mainchain_initial_blockhash` by 10 heights.
4. Call `verify_transaction_inclusion` for any transaction whose block was among the 10 removed headers. The call panics with `"block does not belong to the current main chain"` instead of returning a valid boolean.
5. The state is permanently corrupted — no relayer submission can restore the erased entries, because `submit_block_header_inner` only appends to the tip or stores fork headers; it never re-populates `mainchain_header_to_height` for already-removed heights. [7](#0-6)

```rust
    #[pause(except(roles(Role::UnrestrictedRunGC)))]
    pub fn run_mainchain_gc(&mut self, batch_size: u64) {
        let initial_blockheader = self
            .headers_pool
            .get(&self.mainchain_initial_blockhash)
            .unwrap_or_else(|| env::panic_str(ERR_KEY_NOT_EXIST));

        let tip_blockheader = self
            .headers_pool
            .get(&self.mainchain_tip_blockhash)
            .unwrap_or_else(|| env::panic_str(ERR_KEY_NOT_EXIST));

        let amount_of_headers_we_store =
            tip_blockheader.block_height - initial_blockheader.block_height + 1;

        if amount_of_headers_we_store > self.gc_threshold {
            let total_amount_to_remove = amount_of_headers_we_store - self.gc_threshold;
            let selected_amount_to_remove = std::cmp::min(total_amount_to_remove, batch_size);

            let start_removal_height = initial_blockheader.block_height;
            let end_removal_height = initial_blockheader.block_height + selected_amount_to_remove;
            env::log_str(&format!(
                "Num of blocks to remove {selected_amount_to_remove}"
            ));

            for height in start_removal_height..end_removal_height {
                let blockhash = &self
                    .mainchain_height_to_header
                    .get(&height)
                    .unwrap_or_else(|| env::panic_str(ERR_KEY_NOT_EXIST));

                self.remove_block_header(blockhash);
                self.mainchain_height_to_header.remove(&height);
            }

            self.mainchain_initial_blockhash = self
                .mainchain_height_to_header
                .get(&end_removal_height)
                .unwrap_or_else(|| env::panic_str(ERR_KEY_NOT_EXIST));
        }
    }
```
