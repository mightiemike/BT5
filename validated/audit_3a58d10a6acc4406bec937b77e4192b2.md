### Title
`ContractOwner` Lacks `receive()` Function, Making `withdrawFromDirectDepositV1` Always Revert for Native ETH — (`File: core/contracts/ContractOwner.sol`)

---

### Summary

`ContractOwner.withdrawFromDirectDepositV1()` is the only privileged path to recover native ETH from a `DirectDepositV1` contract. It calls `DirectDepositV1.withdrawNative()`, which sends ETH back to `msg.sender` — the `ContractOwner` contract itself. However, `ContractOwner` implements neither `receive() external payable` nor `fallback() external payable`, so the ETH transfer always reverts. Any native ETH that accumulates in a `DirectDepositV1` contract is permanently unrecoverable.

---

### Finding Description

`ContractOwner.withdrawFromDirectDepositV1()` handles the native-ETH branch as follows: [1](#0-0) 

```solidity
if (token == address(0)) {
    uint256 preBalance = address(this).balance;
    DirectDepositV1(directDepositV1).withdrawNative();
    uint256 postBalance = address(this).balance;
    require(postBalance > preBalance, "empty");
    (bool success, ) = msg.sender.call{value: postBalance - preBalance}("");
    require(success, "xfer");
}
```

This calls `DirectDepositV1.withdrawNative()`, whose implementation is: [2](#0-1) 

```solidity
function withdrawNative() external onlyOwner {
    uint256 balance = address(this).balance;
    (bool success, ) = msg.sender.call{value: balance}("");
    require(success, "Failed to transfer native token to owner");
}
```

`msg.sender` inside `withdrawNative()` is `ContractOwner` (the owner of every `DirectDepositV1`, set at deploy time via `Ownable`). The low-level `.call{value: balance}("")` to `ContractOwner` will always fail because `ContractOwner` has no `receive()` or `fallback()` function anywhere in its 648-line body. [3](#0-2) 

The `require(success, ...)` in `withdrawNative()` then reverts the entire call stack, making `withdrawFromDirectDepositV1(subaccount, address(0))` permanently non-functional.

---

### Impact Explanation

Native ETH can accumulate in a `DirectDepositV1` contract through two documented paths:

1. **Constructor failure path**: If the initial wrapping call to `wrappedNative` fails, the ETH stays in the contract and only a `NativeTokenTransferFailed` event is emitted — no revert. [4](#0-3) 

2. **`selfdestruct` forced ETH**: ETH force-sent via `selfdestruct` bypasses the `receive()` hook entirely and accumulates silently.

In both cases, the only designed recovery mechanism — `ContractOwner.withdrawFromDirectDepositV1(subaccount, address(0))` — always reverts. The ETH is permanently locked in the `DirectDepositV1` contract with no alternative extraction path.

---

### Likelihood Explanation

The constructor path is reachable whenever the `wrappedNative` contract is temporarily non-functional or paused at the moment a new `DirectDepositV1` is deployed. The `selfdestruct` path is reachable by any external actor. Both are realistic on a live chain. The impact (permanent ETH lockup) is concrete and irreversible once triggered.

---

### Recommendation

Add a `receive() external payable {}` function to `ContractOwner.sol`:

```solidity
receive() external payable {}
```

This mirrors the exact fix applied in the referenced Livepeer report and allows `ContractOwner` to transiently hold the ETH before forwarding it to `msg.sender` in the outer call.

---

### Proof of Concept

1. Deploy the protocol normally. `ContractOwner` becomes the owner of all `DirectDepositV1` contracts.
2. At `DirectDepositV1` deploy time, arrange for `wrappedNative.call{value: balance}("")` to fail (e.g., `wrappedNative` is paused). The constructor emits `NativeTokenTransferFailed` and completes — ETH remains in the `DirectDepositV1`.
3. Call `ContractOwner.withdrawFromDirectDepositV1(subaccount, address(0))` (owner-only).
4. Internally, `DirectDepositV1.withdrawNative()` executes `ContractOwner.call{value: balance}("")`.
5. `ContractOwner` has no `receive()` — the call returns `success = false`.
6. `require(success, "Failed to transfer native token to owner")` reverts.
7. The ETH remains locked in `DirectDepositV1` forever with no recovery path. [5](#0-4) [6](#0-5)

        require(directDepositV1 != address(0), "no dda");

            uint256 preBalance = address(this).balance;
