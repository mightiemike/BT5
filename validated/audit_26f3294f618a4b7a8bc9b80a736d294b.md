### Title
Unprivileged Caller Can Aggressively Prune Mainchain Headers via Unrestricted `run_mainchain_gc` — (File: `contract/src/lib.rs`)

### Summary
`run_mainchain_gc` is a public, state-mutating function with no caller restriction when the contract is not paused. Any unprivileged NEAR account can call it with an arbitrarily large `batch_size`, causing immediate bulk deletion of mainchain block headers, permanently corrupting `mainchain_initial_blockhash`, and breaking SPV proof verification for all pruned blocks.

### Finding Description
The analog to the VoterProxy `deposit` bug is `run_mainchain_gc`. In the VoterProxy case, a state-mutating function lacked a caller whitelist, allowing anyone to trigger undesired state changes on behalf of the contract. Here, `run_mainchain_gc` is decorated only with `#[pause(except(roles(Role::UnrestrictedRunGC)))]`: [1](#0-0) 

The `#[pause(...)]` attribute controls access only when the contract is **paused** — it does not restrict callers when the contract is running normally. There is no `#[trusted_relayer]`, no role check, and no `env::predecessor_account_id()` guard on this path. Any NEAR account can call `run_mainchain_gc` with `batch_size = u64::MAX` at any time.

The function removes the oldest mainchain block headers from `headers_pool` and `mainchain_height_to_header`, then advances `mainchain_initial_blockhash` to the new oldest block: [2](#0-1) 

The actual number of blocks removed is bounded by `min(amount_of_headers_we_store - gc_threshold, batch_size)`. By supplying `batch_size = u64::MAX`, an attacker removes every block currently eligible for GC in a single transaction — blocks that would otherwise be removed gradually over many `submit_blocks` calls.

Contrast this with `submit_blocks`, which is properly gated: [3](#0-2) 

The `run_mainchain_gc` function is explicitly documented as a "Public call" and is also invoked internally by `submit_blocks` with `batch_size = num_of_headers`: [4](#0-3) 

The design intent is that GC is gradual — bounded by the number of headers submitted per relayer call. The public entrypoint breaks this invariant by allowing bulk removal with no rate limit and no caller restriction.

### Impact Explanation
1. **Permanent deletion of verifiable blocks**: `verify_transaction_inclusion` and `verify_transaction_inclusion_v2` require `tx_block_blockhash` to be present in `mainchain_header_to_height`. After aggressive GC, any SPV proof for a pruned block panics with "block does not belong to the current main chain": [5](#0-4) 

2. **Corruption of `mainchain_initial_blockhash`**: The canonical chain pointer is permanently advanced, making it impossible to recover pruned blocks without redeployment.

3. **Prevention of valid chain reorganizations**: `CLAUDE.md` explicitly states: *"If mainchain blocks near the fork point have been garbage collected, reorg will fail — the contract panics with `PrevBlockNotFound` when it cannot walk the chain back to the common ancestor."* An attacker can front-run a pending reorg by GC-ing the fork-point blocks first. [6](#0-5) 

The corrupted canonical state values are: `mainchain_initial_blockhash`, `mainchain_height_to_header`, `mainchain_header_to_height`, and the set of blocks for which `verify_transaction_inclusion` returns a valid result.

### Likelihood Explanation
**High.** The entry point is a public NEAR contract method requiring no deposit, no role, and no staking. Any NEAR account can call it in a single transaction. The attacker needs only to know the contract address. No privileged key, social engineering, or external dependency is required.

### Recommendation
Restrict `run_mainchain_gc` to the same caller set as `submit_blocks`. The simplest fix is to add `#[trusted_relayer]` (or an equivalent role check such as `Role::UnrestrictedRunGC` or `Role::DAO`) to the public `run_mainchain_gc` method, so that only authorized relayers or DAO accounts can trigger manual GC. The internal call from `submit_blocks` can remain unrestricted since it is already behind the `#[trusted_relayer]` guard on the outer function.

### Proof of Concept
1. Deploy the contract with `gc_threshold = 52704` and allow the relayer to submit 60000 blocks (8296 blocks are now eligible for GC).
2. From any unprivileged NEAR account, call:
   ```
   run_mainchain_gc({ "batch_size": 18446744073709551615 })
   ```
3. All 8296 eligible blocks are deleted in one transaction. `mainchain_initial_blockhash` now points to block 8296.
4. Any downstream contract calling `verify_transaction_inclusion` for a block at height < 8296 now panics unconditionally, even for proofs that were valid seconds before the attack.
5. If a fork whose common ancestor is at height < 8296 is submitted, `reorg_chain` panics with `PrevBlockNotFound`, permanently preventing the correct chain from being promoted. [7](#0-6)

```rust
    #[pause(except(roles(Role::UnrestrictedRunGC)))]
    pub fn run_mainchain_gc(&mut self, batch_size: u64) {
        let initial_blockheader = self
            .headers_pool
            .get(&self.mainchain_initial_blockhash)
            .unwrap_or_else(|| env::panic_str(ERR_KEY_NOT_EXIST));

        let tip_blockheader = self
            .headers_pool
            .get(&self.mainchain_tip_blockhash)
            .unwrap_or_else(|| env::panic_str(ERR_KEY_NOT_EXIST));

        let amount_of_headers_we_store =
            tip_blockheader.block_height - initial_blockheader.block_height + 1;

        if amount_of_headers_we_store > self.gc_threshold {
            let total_amount_to_remove = amount_of_headers_we_store - self.gc_threshold;
            let selected_amount_to_remove = std::cmp::min(total_amount_to_remove, batch_size);

            let start_removal_height = initial_blockheader.block_height;
            let end_removal_height = initial_blockheader.block_height + selected_amount_to_remove;
            env::log_str(&format!(
                "Num of blocks to remove {selected_amount_to_remove}"
            ));

            for height in start_removal_height..end_removal_height {
                let blockhash = &self
                    .mainchain_height_to_header
                    .get(&height)
                    .unwrap_or_else(|| env::panic_str(ERR_KEY_NOT_EXIST));

                self.remove_block_header(blockhash);
                self.mainchain_height_to_header.remove(&height);
            }

            self.mainchain_initial_blockhash = self
                .mainchain_height_to_header
                .get(&end_removal_height)
                .unwrap_or_else(|| env::panic_str(ERR_KEY_NOT_EXIST));
        }
    }
```
