### Title
Lack of Slippage Protection in `BurnNlp` and `MintNlp` — (`core/contracts/Clearinghouse.sol`, `core/contracts/interfaces/IEndpoint.sol`)

---

### Summary

The `BurnNlp` and `MintNlp` transaction structs that users sign contain no minimum-output field. The `oraclePriceX18` used to convert between quote and NLP is supplied by the sequencer at execution time and is **not** committed to in the user's signature. A user burning NLP to receive quote (or minting NLP by spending quote) has no on-chain guarantee that the conversion rate will be within any bound they accepted.

---

### Finding Description

The user-signed structs are:

```solidity
// IEndpoint.sol
struct MintNlp {
    bytes32 sender;
    uint128 quoteAmount;
    uint64 nonce;
}

struct BurnNlp {
    bytes32 sender;
    uint128 nlpAmount;
    uint64 nonce;
}
```

Neither struct contains a `minNlpAmount` (for `MintNlp`) or a `minQuoteAmount` (for `BurnNlp`). [1](#0-0) 

The outer `SignedMintNlp` / `SignedBurnNlp` structs carry `oraclePriceX18` as a separate field that is **not** part of the signed digest:

```solidity
struct SignedMintNlp {
    MintNlp tx;
    bytes   signature;
    int128  oraclePriceX18;        // sequencer-supplied, not signed by user
    int128[] nlpPoolRebalanceX18;
}
``` [2](#0-1) 

`validateSignedTx` hashes only the inner `tx` struct fields (matching the pattern seen for every other transaction type in `Verifier.computeDigest`), so the user's ECDSA signature does not cover `oraclePriceX18`. [3](#0-2) 

In `Clearinghouse.mintNlp`, the NLP minted is:

```solidity
int128 nlpAmount = quoteAmount.div(oraclePriceX18);
``` [4](#0-3) 

In `Clearinghouse.burnNlp`, the quote returned is:

```solidity
int128 quoteAmount = nlpAmount.mul(oraclePriceX18);
int128 burnFee     = MathHelper.max(ONE, quoteAmount / 1000);
quoteAmount        = MathHelper.max(0, quoteAmount - burnFee);
``` [5](#0-4) 

In both cases the sequencer passes `signedTx.oraclePriceX18` directly into `clearinghouse.mintNlp` / `clearinghouse.burnNlp` with no user-enforced bound. [6](#0-5) 

---

### Impact Explanation

**`BurnNlp`**: A user signs a request to burn `N` NLP tokens. The quote they receive equals `N × oraclePriceX18 − burnFee`. If the oracle price has fallen (legitimately, due to market movement) between the moment the user signed and the moment the sequencer executes the transaction, the user receives materially less quote than they anticipated, with no on-chain recourse. There is no `minQuoteAmount` check anywhere in the execution path.

**`MintNlp`**: A user signs a request to spend `Q` quote. The NLP minted equals `Q / oraclePriceX18`. If the oracle price has risen between signing and execution, the user receives fewer NLP tokens than expected, with no `minNlpAmount` check.

In both cases the corrupted state delta is the user's quote or NLP balance in `SpotEngine`, which is updated without any user-specified floor. [7](#0-6) 

---

### Likelihood Explanation

NLP oracle prices are updated on every `MintNlp`/`BurnNlp` submission (`priceX18[NLP_PRODUCT_ID] = signedTx.oraclePriceX18`). [8](#0-7)  There is inherent latency between a user signing a transaction off-chain and the sequencer including it in a batch. During volatile market conditions this latency can be seconds to minutes, and the oracle price can move meaningfully. No exploit or malicious actor is required — ordinary market movement is sufficient to trigger the loss.

---

### Recommendation

Add minimum-output fields to the user-signed structs and enforce them in `Clearinghouse`:

```solidity
struct MintNlp {
    bytes32 sender;
    uint128 quoteAmount;
    uint64  nonce;
    uint128 minNlpAmount;   // NEW: revert if nlpAmount < minNlpAmount
}

struct BurnNlp {
    bytes32 sender;
    uint128 nlpAmount;
    uint64  nonce;
    uint128 minQuoteAmount; // NEW: revert if quoteAmount < minQuoteAmount
}
```

In `Clearinghouse.mintNlp`:
```solidity
require(nlpAmount >= int128(txn.minNlpAmount), ERR_SLIPPAGE_TOO_HIGH);
```

In `Clearinghouse.burnNlp`:
```solidity
require(quoteAmount >= int128(txn.minQuoteAmount), ERR_SLIPPAGE_TOO_HIGH);
```

Note that `ERR_SLIPPAGE_TOO_HIGH` is already defined in `Errors.sol` but is currently unused. [9](#0-8) 

---

### Proof of Concept

1. User signs `BurnNlp { sender, nlpAmount: 1000e18, nonce: N }` when NLP oracle price is `$1.00`, expecting to receive `~999 USDC` (after 0.1% burn fee).
2. Before the sequencer batches the transaction, the NLP oracle price drops to `$0.80`.
3. Sequencer submits the transaction with `oraclePriceX18 = 0.80e18`.
4. `Clearinghouse.burnNlp` computes `quoteAmount = 1000e18 × 0.80e18 / 1e18 = 800e18`, then deducts burn fee → user receives `~799 USDC`.
5. User loses `~200 USDC` (~20%) with no on-chain protection. The user's signed message contained no `minQuoteAmount` field, so the execution cannot be reverted. [10](#0-9)

```text
    function burnNlp(
        IEndpoint.BurnNlp calldata txn,
        int128 oraclePriceX18,
        IEndpoint.NlpPool[] calldata nlpPools,
        int128[] calldata nlpPoolRebalanceX18
    ) external onlyEndpoint {
        require(!RiskHelper.isIsolatedSubaccount(txn.sender), ERR_UNAUTHORIZED);

        ISpotEngine spotEngine = _spotEngine();
        spotEngine.updatePrice(NLP_PRODUCT_ID, oraclePriceX18);

        require(txn.nlpAmount <= INT128_MAX, ERR_CONVERSION_OVERFLOW);
        int128 nlpAmount = int128(txn.nlpAmount);
        require(
            spotEngine.getNlpUnlockedBalance(txn.sender).amount >= nlpAmount,
            ERR_UNLOCKED_NLP_INSUFFICIENT
        );
        int128 quoteAmount = nlpAmount.mul(oraclePriceX18);
        int128 burnFee = MathHelper.max(ONE, quoteAmount / 1000);
        quoteAmount = MathHelper.max(0, quoteAmount - burnFee);

        _validateNlpRebalance(nlpPools, nlpPoolRebalanceX18, -quoteAmount);
        for (uint128 i = 0; i < nlpPoolRebalanceX18.length; i++) {
            require(nlpPoolRebalanceX18[i] <= 0, ERR_INVALID_NLP_REBALANCE);
        }

        spotEngine.updateBalance(NLP_PRODUCT_ID, txn.sender, -nlpAmount);
        spotEngine.updateBalance(NLP_PRODUCT_ID, N_ACCOUNT, nlpAmount);

        if (quoteAmount > 0) {
            spotEngine.updateBalance(QUOTE_PRODUCT_ID, txn.sender, quoteAmount);
            _applyNlpRebalance(spotEngine, nlpPools, nlpPoolRebalanceX18);
        }

        require(
            spotEngine.getBalance(NLP_PRODUCT_ID, txn.sender).amount >= 0,
            ERR_SUBACCT_HEALTH
        );
        // Burning NLP can decrease health if the burn fee exceeds the health improvement
        // from the withdrawal. This check prevents malicious actors from deliberately
        // creating unhealthy subaccounts through NLP burns.
        require(
            getHealth(txn.sender, IProductEngine.HealthType.MAINTENANCE) >= 0,
            ERR_SUBACCT_HEALTH
        );
    }
