### Title
`delistProduct` Does Not Settle Realized PnL (`vQuoteBalance`) to Spot Quote Balance, Freezing User Funds — (`File: core/contracts/Clearinghouse.sol`)

---

### Summary

`Clearinghouse.delistProduct()` closes perp positions for listed subaccounts by zeroing `balance.amount` and crediting the settlement value into `balance.vQuoteBalance` inside the `PerpEngine`. It never transfers that realized PnL into the `SpotEngine` quote balance. Because the product is delisted, no further trading can realize the `vQuoteBalance`, and no explicit settlement path exists in `delistProduct`. The realized PnL is frozen inside the perp engine.

---

### Finding Description

`delistProduct` iterates over the provided `txn.subaccounts`, reads each perp balance, and calls `perpEngine.updateBalance` with:

```
baseDelta  = -balance.amount          // closes the position
quoteDelta = -baseDelta * priceX18    // settlement value at delist price
``` [1](#0-0) 

Inside `PerpEngineState._updateBalance`, `quoteDelta` is applied exclusively to `balance.vQuoteBalance`:

```
deltaQuote = vQuoteDelta - diffX18.mul(balance.amount);
balance.vQuoteBalance += deltaQuote;
``` [2](#0-1) 

After the call, `balance.amount == 0` but `balance.vQuoteBalance` holds the full realized PnL. The `SpotEngine` quote balance for the subaccount is **never touched** by `delistProduct`.

Contrast this with the protocol's own liquidation PnL settlement path, `_settlePnlAgainstLiquidator`, which explicitly mirrors every `vQuoteBalance` reduction in the perp engine with an equal `spotEngine.updateBalance(QUOTE_PRODUCT_ID, …)` credit: [3](#0-2) 

`delistProduct` performs no equivalent spot-engine credit. The realized PnL therefore remains stranded in the perp engine's `vQuoteBalance` for a product that can no longer be traded.

---

### Impact Explanation

A subaccount whose perp position is closed by `delistProduct` ends up with:

- `perpBalance.amount = 0` — position closed.
- `perpBalance.vQuoteBalance = realized PnL` — non-zero, but trapped in the perp engine.
- `spotBalance(QUOTE_PRODUCT_ID)` — **unchanged**; no new withdrawable funds.

Because the product is delisted, the subaccount cannot open a new position to trade out of the `vQuoteBalance`. Withdrawals draw from the spot quote balance, not from perp `vQuoteBalance`. The realized PnL is effectively frozen: it contributes to health calculations (preventing liquidation) but cannot be withdrawn. Any subaccount with a profitable perp position at delist time loses access to those funds.

---

### Likelihood Explanation

`delistProduct` is a sequencer-submitted slow-mode transaction. It is a routine operational action (product sunset, emergency removal). Every invocation of `delistProduct` on any product with open profitable positions triggers the freeze. No adversarial actor is required; the bug fires on normal, honest sequencer operation.

---

### Recommendation

Mirror the liquidation settlement pattern inside `delistProduct`. After calling `perpEngine.updateBalance`, immediately settle the resulting `vQuoteBalance` to the spot quote balance for each subaccount:

```solidity
// after perpEngine.updateBalance(...)
IPerpEngine.Balance memory settled = perpEngine.getBalance(txn.productId, txn.subaccounts[i]);
if (settled.vQuoteBalance != 0) {
    perpEngine.updateBalance(txn.productId, txn.subaccounts[i], 0, -settled.vQuoteBalance);
    spotEngine.updateBalance(QUOTE_PRODUCT_ID, txn.subaccounts[i], settled.vQuoteBalance);
}
```

This matches the two-sided settlement already used in `_settlePnlAgainstLiquidator`.

---

### Proof of Concept

1. Subaccount `A` holds a long perp position: `balance.amount = 100`, `balance.vQuoteBalance = -9000` (entry cost), current price = `100`.
2. Sequencer calls `delistProduct` with `priceX18 = 100`, `subaccounts = [A]`.
3. `baseDelta = -100`, `quoteDelta = 10000`.
4. After `_updateBalance`: `balance.amount = 0`, `balance.vQuoteBalance = -9000 + 10000 = 1000` (realized profit).
5. `spotEngine.getBalance(QUOTE_PRODUCT_ID, A)` is **unchanged** — no USDC credited.
6. Subaccount `A` attempts withdrawal; the withdrawal check reads the spot quote balance, which has not increased. The 1000 units of realized profit are inaccessible. [4](#0-3) [5](#0-4)

    function delistProduct(bytes calldata transaction) external onlyEndpoint {
        IEndpoint.DelistProduct memory txn = abi.decode(
            transaction[1:],
            (IEndpoint.DelistProduct)
        );
        // only perp can be delisted
        require(
            productToEngine[txn.productId] == _perpEngine(),
            ERR_INVALID_PRODUCT
        );
        require(txn.priceX18 == _getPriceX18(txn.productId), ERR_INVALID_PRICE);
        IPerpEngine perpEngine = _perpEngine();
        for (uint256 i = 0; i < txn.subaccounts.length; i++) {
            IPerpEngine.Balance memory balance = perpEngine.getBalance(
                txn.productId,
                txn.subaccounts[i]
            );
            int128 baseDelta = -balance.amount;
            int128 quoteDelta = -baseDelta.mul(txn.priceX18);
            perpEngine.updateBalance(
                txn.productId,
                txn.subaccounts[i],
                baseDelta,
                quoteDelta
            );
            if (RiskHelper.isIsolatedSubaccount(txn.subaccounts[i])) {
                IOffchainExchange(
                    IEndpoint(getEndpoint()).getOffchainExchange()
                ).tryCloseIsolatedSubaccount(txn.subaccounts[i]);
            }
        }
    }

    function _updateBalance(
        State memory state,
        Balance memory balance,
        int128 balanceDelta,
        int128 vQuoteDelta
    ) internal pure {
        // pre update
        state.openInterest -= balance.amount.abs();
        int128 cumulativeFundingAmountX18 = (balance.amount > 0)
            ? state.cumulativeFundingLongX18
            : state.cumulativeFundingShortX18;
        int128 diffX18 = cumulativeFundingAmountX18 -
            balance.lastCumulativeFundingX18;
        int128 deltaQuote = vQuoteDelta - diffX18.mul(balance.amount);

        // apply delta
        balance.amount += balanceDelta;

        // apply vquote
        balance.vQuoteBalance += deltaQuote;

        // post update
        if (balance.amount > 0) {
            state.openInterest += balance.amount;
            balance.lastCumulativeFundingX18 = state.cumulativeFundingLongX18;
        } else {
            state.openInterest -= balance.amount;
            balance.lastCumulativeFundingX18 = state.cumulativeFundingShortX18;
        }
    }
```
