### Title
Malicious IBC Counterparty Can Inject Pre-Prefixed `ClassId` Causing Double-Prefix and Unbacked Voucher Minting — (`x/nft-transfer/keeper/packet.go`)

### Summary
`processReceivedPacket` unconditionally prepends the destination port/channel prefix to `data.ClassId` without first checking whether the classID already contains that prefix. A malicious counterparty chain can send a packet with a pre-prefixed `ClassId`, causing the receiving chain to construct a double-prefixed classID and mint NFT vouchers under a classID that has no corresponding escrow, making those vouchers permanently unredeemable.

### Finding Description
In `processReceivedPacket` (`x/nft-transfer/keeper/packet.go`):

```go
// line 148
isAwayFromOrigin := types.IsAwayFromOrigin(packet.GetSourcePort(), packet.GetSourceChannel(), data.ClassId)
// line 155-157
classPrefix := types.GetClassPrefix(packet.GetDestPort(), packet.GetDestChannel())
prefixedClassID := classPrefix + data.ClassId
```

`IsAwayFromOrigin` only checks whether `data.ClassId` starts with `sourcePort/sourceChannel/`. A malicious `ClassId = "nft/channel-1/nft/channel-0/nftClass"` (where `nft/channel-1` is the destination prefix) does not start with the source prefix `nft/channel-0/`, so `isAwayFromOrigin = true`. The code then prepends `nft/channel-1/` again, producing `nft/channel-1/nft/channel-1/nft/channel-0/nftClass`.

`ValidateBasic()` (`x/nft-transfer/types/packet.go` lines 41-71) performs no structural validation of `ClassId` beyond checking it is non-blank, so the malicious packet passes all validation.

### Impact Explanation
The receiving chain mints NFT vouchers under `ibc/<sha256("nft/channel-1/nft/channel-1/nft/channel-0/nftClass")>` — a classID that:
- Was never legitimately established through IBC
- Has no corresponding escrow on the source chain
- Can never be redeemed (the source chain has no NFTs escrowed under the double-prefixed classID)
- Pollutes the on-chain classTrace store with a spurious entry

This constitutes unauthorized voucher minting: NFTs are created out of thin air under a wrong classID, violating the invariant that every minted IBC NFT voucher corresponds to an escrowed NFT on the source chain.

### Likelihood Explanation
Any chain that has an open IBC channel with the victim chain can execute this attack by crafting a packet with a pre-prefixed `ClassId`. No governance approval, key compromise, or social engineering is required — only the ability to send IBC packets, which is the normal operation of any IBC-connected chain.

### Recommendation
In `processReceivedPacket`, before prepending the destination prefix, validate that `data.ClassId` does not already start with `destPort/destChannel/`:

```go
classPrefix := types.GetClassPrefix(packet.GetDestPort(), packet.GetDestChannel())
if strings.HasPrefix(data.ClassId, classPrefix) {
    return sdkerrors.Wrapf(types.ErrInvalidClassID,
        "classID already contains destination prefix %s", classPrefix)
}
prefixedClassID := classPrefix + data.ClassId
```

Alternatively, add this check to `ValidateBasic` or as a dedicated validation step in `OnRecvPacket` that compares `data.ClassId` against the packet's destination port/channel.

	if strings.TrimSpace(nftpd.ClassId) == "" {
		return newsdkerrors.Wrap(ErrInvalidClassID, "classId cannot be blank")
	}

	if len(nftpd.TokenIds) == 0 {
		return newsdkerrors.Wrap(ErrInvalidTokenID, "tokenId cannot be blank")
	}

	if len(nftpd.TokenIds) != len(nftpd.TokenUris) {
		return newsdkerrors.Wrap(ErrInvalidPacket, "tokenIds and tokenUris lengths do not match")
	}

	if strings.TrimSpace(nftpd.Sender) == "" {
		return newsdkerrors.Wrap(sdkerrors.ErrInvalidAddress, "sender address cannot be blank")
	}

	// decode the sender address
	if _, err := sdk.AccAddressFromBech32(nftpd.Sender); err != nil {
		return newsdkerrors.Wrap(sdkerrors.ErrInvalidAddress, "invalid sender address")
	}

	if strings.TrimSpace(nftpd.Receiver) == "" {
		return newsdkerrors.Wrap(sdkerrors.ErrInvalidAddress, "receiver address cannot be blank")
	}

	// decode the receiver address
	if _, err := sdk.AccAddressFromBech32(nftpd.Receiver); err != nil {
		return newsdkerrors.Wrap(sdkerrors.ErrInvalidAddress, "invalid receiver address")
	}
	return nil
