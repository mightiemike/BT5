Audit Report

## Title
SwapAllowlistExtension Gates the Router Address Instead of the Originating User, Allowing Any User to Bypass the Allowlist — (`metric-periphery/contracts/extensions/SwapAllowlistExtension.sol`)

## Summary

`SwapAllowlistExtension.beforeSwap` checks `allowedSwapper[pool][sender]` where `sender` is `msg.sender` at the pool — the router contract — not the originating EOA. A pool admin who allowlists the router to enable router-mediated swaps for vetted users simultaneously grants every user on the network the ability to bypass the per-user allowlist by routing through `MetricOmmSimpleRouter`.

## Finding Description

`MetricOmmPool.swap` passes `msg.sender` directly as the `sender` argument to `_beforeSwap`: [1](#0-0) 

`ExtensionCalling._beforeSwap` forwards this value unchanged into every configured extension: [2](#0-1) 

`SwapAllowlistExtension.beforeSwap` then performs the allowlist lookup keyed on `(msg.sender, sender)` — i.e., `(pool, router)`: [3](#0-2) 

When a user calls `MetricOmmSimpleRouter.exactInputSingle` (or any other swap entry point), the router calls `pool.swap(...)` directly: [4](#0-3) 

At the pool, `msg.sender` is the router's address. The extension never receives the originating EOA. The allowlist check resolves to `allowedSwapper[pool][router]`, not `allowedSwapper[pool][userEOA]`. If the router is allowlisted (a natural operational step to enable any router-mediated swap), the check passes for every caller regardless of their individual allowlist status.

## Impact Explanation

Any non-allowlisted user can swap in a pool configured with `SwapAllowlistExtension` simply by routing through `MetricOmmSimpleRouter`. LP providers who deployed capital into a restricted pool expecting only vetted counterparties face unrestricted swap exposure: adverse-selection losses, unauthorized price impact, and extraction of oracle-anchored pricing by parties the pool was designed to exclude. The allowlist appears active and correctly configured but the identity it validates is the router, not the economically relevant actor. This is a direct loss-of-fund-protection failure for LPs in restricted pools, meeting Medium severity under Sherlock thresholds.

## Likelihood Explanation

`MetricOmmSimpleRouter` is the standard swap entry point for end users. A pool admin who wants to allow even a single vetted user to swap via the router must allowlist the router address. Once that entry exists, the bypass is unconditional and requires no special permissions, flash loans, or unusual token behavior — any EOA calling the router qualifies. The bypass is silent: the extension returns `beforeSwap.selector`, the pool emits no anomaly, and the swap settles normally.

## Recommendation

The `beforeSwap` hook must validate the originating user, not the immediate caller. Two concrete approaches:

1. **`extensionData` passthrough**: The router encodes the true `msg.sender` into `extensionData` before calling the pool. The extension reads and validates this field, trusting only calls where `msg.sender` (the pool's caller) is a recognized router. This requires the router to be trusted to populate the field honestly.
2. **Recipient-keyed allowlist**: For swap allowlists, key the check on `recipient` instead of `sender`. The recipient of output tokens is the economically relevant actor and is harder to spoof via routing intermediaries.

## Proof of Concept

```
1. Deploy pool with SwapAllowlistExtension as beforeSwap hook.
2. Pool admin: allowedSwapper[pool][userA] = true   (only userA intended)
3. Pool admin: allowedSwapper[pool][router] = true  (to enable router-mediated swaps for userA)
4. userB (not in allowlist) calls MetricOmmSimpleRouter.exactInputSingle({pool: pool, ...})
5. Router calls pool.swap(...) → msg.sender at pool = router address
6. _beforeSwap(router, ...) → SwapAllowlistExtension checks allowedSwapper[pool][router] → true → passes
7. userB's swap executes successfully despite allowedSwapper[pool][userB] == false
8. Corrupted invariant: allowedSwapper[pool][userB] == false should block userB; instead the
   check resolves to allowedSwapper[pool][router] == true and the guard is silently bypassed.
```
