### Title
Unrestricted Public Access to `run_mainchain_gc` Allows Any Unprivileged Caller to Prematurely Prune Canonical Mainchain Headers — (File: `contract/src/lib.rs`)

---

### Summary

`run_mainchain_gc` is a public, state-mutating contract method with no caller authentication. Any unprivileged NEAR account can invoke it directly with an arbitrary `batch_size`, immediately pruning all mainchain block headers beyond the GC threshold in a single call. This permanently deletes those headers from `headers_pool`, `mainchain_height_to_header`, and `mainchain_header_to_height`, and advances `mainchain_initial_blockhash` — breaking SPV proof verification for any block that is removed.

---

### Finding Description

`submit_blocks` — the only authenticated entry point for header submission — is protected by both `#[pause]` and `#[trusted_relayer]`, restricting callers to accounts on the trusted relayer list or holding `Role::DAO`/`Role::UnrestrictedSubmitBlocks`. [1](#0-0) 

Internally, `submit_blocks` calls `run_mainchain_gc(num_of_headers)` where `num_of_headers` equals the count of headers just submitted — a small, bounded value (typically 1–100). [2](#0-1) 

However, `run_mainchain_gc` is itself a separately callable public method. Its only decorator is `#[pause(except(roles(Role::UnrestrictedRunGC)))]`, which only gates access when the contract is paused. When the contract is running normally, **any NEAR account** can call it directly with an arbitrary `batch_size`: [3](#0-2) 

Inside the function, `selected_amount_to_remove` is capped at `total_amount_to_remove = amount_of_headers_we_store - gc_threshold`. An attacker supplying `batch_size = u64::MAX` removes every header beyond the threshold in a single transaction: [4](#0-3) 

The loop deletes entries from `headers_pool` and `mainchain_height_to_header`, then advances `mainchain_initial_blockhash` to the new oldest block. These deletions are permanent — there is no recovery path.

---

### Impact Explanation

`verify_transaction_inclusion` and `verify_transaction_inclusion_v2` both look up the target block in `headers_pool` and `mainchain_header_to_height`. If the block has been GC'd, the call panics with `"block does not belong to the current main chain"` or `"cannot find requested transaction block"`. [5](#0-4) [6](#0-5) 

An attacker can therefore:
1. Monitor the contract until `amount_of_headers_we_store > gc_threshold` (i.e., the chain has grown past the retention window).
2. Call `run_mainchain_gc(u64::MAX)` to immediately prune all excess headers — headers that the legitimate relayer would have pruned gradually over many future `submit_blocks` calls.
3. Any downstream contract or user that was about to verify a transaction in one of those now-deleted blocks receives a permanent verification failure.

The corrupted state values are: `mainchain_initial_blockhash`, `mainchain_height_to_header`, `mainchain_header_to_height`, and `headers_pool` — the exact data structures that SPV proof verification depends on.

---

### Likelihood Explanation

The attack requires no privileged role, no leaked key, and no social engineering. The only precondition is that the contract has accumulated more than `gc_threshold` headers (e.g., the default 52,704 blocks, roughly one year of Bitcoin blocks). Once that threshold is crossed, the attack is executable by any NEAR account in a single transaction. The function is publicly documented and callable via standard NEAR JSON-RPC.

---

### Recommendation

Add `#[trusted_relayer]` (or at minimum an explicit `acl_check` for `Role::DAO` / `Role::RelayerManager`) to `run_mainchain_gc`, mirroring the protection applied to `submit_blocks`. If the function is intentionally public for permissionless maintenance, restrict `batch_size` to a small protocol-defined constant so that no single external call can prune more headers than one `submit_blocks` batch would.

---

### Proof of Concept

```
# Attacker (any NEAR account) calls:
near call <contract_id> run_mainchain_gc \
  '{"batch_size": 18446744073709551615}' \
  --accountId attacker.near

# Result:
# - All headers beyond gc_threshold are deleted from headers_pool
# - mainchain_initial_blockhash advances to the new oldest block
# - Any subsequent call to verify_transaction_inclusion for a pruned block panics:
#   "block does not belong to the current main chain"
```

The root cause is the absence of any caller authentication on a state-mutating public method, directly analogous to `@csrf_exempt` on a POST endpoint: the guard (`#[pause]`) only fires in one operational mode, leaving the function fully open under normal conditions. [7](#0-6)

    #[pause(except(roles(Role::UnrestrictedRunGC)))]
    pub fn run_mainchain_gc(&mut self, batch_size: u64) {
        let initial_blockheader = self
            .headers_pool
            .get(&self.mainchain_initial_blockhash)
            .unwrap_or_else(|| env::panic_str(ERR_KEY_NOT_EXIST));

        let tip_blockheader = self
            .headers_pool
            .get(&self.mainchain_tip_blockhash)
            .unwrap_or_else(|| env::panic_str(ERR_KEY_NOT_EXIST));

        let amount_of_headers_we_store =
            tip_blockheader.block_height - initial_blockheader.block_height + 1;

        if amount_of_headers_we_store > self.gc_threshold {
            let total_amount_to_remove = amount_of_headers_we_store - self.gc_threshold;
            let selected_amount_to_remove = std::cmp::min(total_amount_to_remove, batch_size);

            let start_removal_height = initial_blockheader.block_height;
            let end_removal_height = initial_blockheader.block_height + selected_amount_to_remove;
            env::log_str(&format!(
                "Num of blocks to remove {selected_amount_to_remove}"
            ));

            for height in start_removal_height..end_removal_height {
                let blockhash = &self
                    .mainchain_height_to_header
                    .get(&height)
                    .unwrap_or_else(|| env::panic_str(ERR_KEY_NOT_EXIST));

                self.remove_block_header(blockhash);
                self.mainchain_height_to_header.remove(&height);
            }

            self.mainchain_initial_blockhash = self
                .mainchain_height_to_header
                .get(&end_removal_height)
                .unwrap_or_else(|| env::panic_str(ERR_KEY_NOT_EXIST));
        }
    }
