Audit Report

## Title
SwapAllowlistExtension Gates on Router Address Instead of End User, Enabling Allowlist Bypass - (File: metric-periphery/contracts/extensions/SwapAllowlistExtension.sol)

## Summary

`SwapAllowlistExtension.beforeSwap` checks `allowedSwapper[msg.sender][sender]`, where `sender` is `msg.sender` of the `pool.swap` call. When a user routes through `MetricOmmSimpleRouter`, the router is `msg.sender` of `pool.swap`, so the extension checks whether the **router** is allowlisted rather than the actual end user. Any pool admin who allowlists the router to support router-mediated swaps for permitted users inadvertently grants every unpermitted user the ability to bypass the allowlist by calling the same public router.

## Finding Description

`MetricOmmPool.swap` passes `msg.sender` as the `sender` argument to `_beforeSwap`: [1](#0-0) 

`ExtensionCalling._beforeSwap` forwards that value unchanged to every configured extension: [2](#0-1) 

`SwapAllowlistExtension.beforeSwap` gates on `allowedSwapper[msg.sender][sender]`, where `msg.sender` is the pool and `sender` is the value received from the pool — the router address when routing through `MetricOmmSimpleRouter`: [3](#0-2) 

`MetricOmmSimpleRouter.exactInputSingle` calls `pool.swap` directly, making the router the `msg.sender` of that call: [4](#0-3) 

The same pattern applies to `exactInput`, `exactOutputSingle`, and `exactOutput`: [5](#0-4) 

No existing guard in the extension decodes or checks the originating user identity. The `extensionData` field is passed through but never inspected by `SwapAllowlistExtension`. The check `allowedSwapper[pool][router] == true` passes for every caller who routes through the router, regardless of whether that caller was ever individually permitted.

## Impact Explanation

A pool admin who deploys `SwapAllowlistExtension` to restrict swaps to KYC'd or permissioned addresses and also allowlists the router (the natural configuration for supporting the standard periphery) creates a condition where `allowedSwapper[pool][router] == true`. The extension's check then passes for **any** address that calls `router.exactInputSingle` (or any other router entry point), including addresses that were never individually permitted. Unauthorized users can execute swaps in a pool designed to be access-controlled, receiving output tokens they are not entitled to and draining LP assets at oracle-derived prices. This constitutes a direct loss of LP principal and a broken core access-control invariant.

## Likelihood Explanation

The precondition — the pool admin allowlisting the router — is the natural and expected configuration for any pool that wants to restrict users but still support the standard periphery. The router is a public, permissionless contract. No special privilege, flash loan, or complex setup is required; a plain `exactInputSingle` call suffices. Any pool that combines `SwapAllowlistExtension` with router support is vulnerable.

## Recommendation

`SwapAllowlistExtension.beforeSwap` must gate on the originating user, not the immediate caller of `pool.swap`. The simplest safe fix is to have `MetricOmmSimpleRouter` encode `msg.sender` (the end user) into `extensionData` and update `SwapAllowlistExtension.beforeSwap` to decode and check that value when present, falling back to `sender` only for direct pool calls. Alternatively, the pool could expose a dedicated entry point that the router calls with an explicit `trueSender` argument, which the extension then checks.

## Proof of Concept

```
Setup:
  pool = deploy MetricOmmPool with SwapAllowlistExtension
  admin calls setAllowedToSwap(pool, alice, true)       // alice is KYC'd
  admin calls setAllowedToSwap(pool, router, true)      // enable router for alice

Attack:
  eve (not allowlisted) calls:
    router.exactInputSingle({pool: pool, recipient: eve, ...})

Execution trace:
  router.exactInputSingle()
    → pool.swap(recipient=eve, ...)          // msg.sender = router
      → _beforeSwap(sender=router, ...)
        → SwapAllowlistExtension.beforeSwap(sender=router, ...)
          → allowedSwapper[pool][router] == true  ✓  (bypass!)
      → swap executes, eve receives tokens

Result:
  Eve, who was never allowlisted, successfully swaps in a restricted pool.
  The allowlist invariant is broken; LP assets flow to an unauthorized recipient.
```
