### Title
Bonus rewards pool tokens permanently locked with no rescue mechanism when all tier positions exit — (`x/tieredrewards/keeper/msg_server.go`)

### Summary
The `x/tieredrewards` module's bonus rewards pool (`RewardsPoolName` module account) has no direct withdrawal or rescue mechanism. When all tier positions exit, any remaining tokens in the pool become permanently locked with no way to recover them directly.

### Finding Description
The bonus rewards pool is funded externally — any account can send tokens to the `RewardsPoolName` module account (the integration tests confirm this via `fund_pool`). Bonus rewards are paid out lazily when positions claim rewards via `processEventsAndClaimBonus`. [1](#0-0) 

When all positions exit through the normal lifecycle (`TierUndelegate` → `WithdrawFromTier`, or `ExitTierWithDelegation`), the pool may still hold tokens that were never claimed (e.g., due to overfunding, or because the pool was funded after all positions had already accrued their maximum bonus up to `ExitUnlockAt`).

The complete set of RPC handlers in the module's `MsgServer` is: [2](#0-1) 

None of these handlers provide a mechanism to withdraw remaining tokens from the pool. The only indirect drain mechanisms are:

1. **BeginBlocker shortfall transfer**: transfers pool tokens to distribution as base rewards top-up, but *only* when fee revenue is insufficient to meet `TargetBaseRewardsRate`. If fee revenue is sufficient, no transfer occurs and the pool is untouched.
2. **New positions**: new positions can eventually drain the pool through bonus claims, but this requires users to lock tokens and go through the full position lifecycle — it is not a direct admin recovery path.

The `WithdrawFromTier` handler only sends `SpendableCoins` from the position's delegator address to the owner; it does not touch the pool: [3](#0-2) 

The `processEventsAndClaimBonus` function sends from the pool to the owner, but only when a position is actively claiming: [4](#0-3) 

Once all positions are deleted, no code path can trigger `SendCoinsFromModuleToAccount` from `RewardsPoolName`.

The `CloseOnly` tier governance flag blocks new positions while allowing exits: [5](#0-4) 

When governance marks all tiers `CloseOnly` and all positions exit, the pool is permanently stranded.

### Impact Explanation
Admin/governance-funded tokens in the `RewardsPoolName` module account become permanently locked with no direct recovery path when all positions exit. The admin has no `MsgWithdrawFromPool` or equivalent message. The only recovery paths are indirect and require either a fee revenue shortfall (BeginBlocker) or new user positions — neither of which is guaranteed or controllable by the pool funder.

### Likelihood Explanation
The trigger is entirely composed of normal, unprivileged user transactions: `MsgTriggerExitFromTier` → `MsgTierUndelegate` → `MsgWithdrawFromTier` (or `MsgExitTierWithDelegation`). All positions exiting is a realistic scenario, especially when governance marks tiers `CloseOnly` to sunset them: [6](#0-5) 

The integration test itself demonstrates the full "all positions exit" lifecycle. After this, any pool balance is stranded.

### Recommendation
Implement a governance-controlled `MsgRescuePoolFunds` (or `MsgWithdrawFromPool`) message that allows the admin/governance to withdraw remaining pool tokens when no delegated positions exist (i.e., `PositionCountByValidator` is zero for all validators). This mirrors the "rescue" mechanism recommended in the external report:

```go
// After verifying no delegated positions remain:
if totalDelegatedPositions == 0 {
    remaining := bankKeeper.GetAllBalances(ctx, poolAddr)
    if !remaining.IsZero() {
        bankKeeper.SendCoinsFromModuleToAccount(ctx, types.RewardsPoolName, authority, remaining)
    }
}
```

Additionally, consider a sunset mechanism analogous to the external report's recommendation: once all positions have exited (or after a governance-set sunset timestamp), allow the authority to call a rescue function that transfers all pool assets to the governance account.

### Proof of Concept
1. Admin funds the bonus rewards pool with 1,000,000 `basecro` (bank send to `RewardsPoolName` module account address).
2. All existing positions trigger exit, undelegate, and withdraw via normal user transactions.
3. Pool still holds tokens (e.g., 500,000 `basecro` remaining — pool was overfunded relative to actual bonus accrued).
4. Governance marks all tiers `CloseOnly` — no new positions can be created.
5. `TargetBaseRewardsRate = 0` (or fee revenue is sufficient) — BeginBlocker does not drain the pool.
6. Admin attempts to recover funds — no message exists to do so.
7. Pool tokens are permanently locked in the `RewardsPoolName` module account. [7](#0-6)

def test_delete_tier_after_withdraw(cluster):
    """After all positions are withdrawn, DeleteTier succeeds."""
    # Withdraw all Tier 3 positions for both signers
    for signer in ("signer1", "signer2"):
        owner = cluster.address(signer)
        result = query_command(cluster, MODULE, "positions-by-owner", owner)
        for pos in result.get("positions", []):
            if int(pos["tier_id"]) != TIER_3_ID:
                continue
            pos_id = int(pos["id"])

            # Trigger exit if not already triggered
            if pos.get("exit_triggered_at", "0001") == "0001-01-01T00:00:00Z":
                rsp = trigger_exit(cluster, owner, pos_id)
                assert rsp["code"] == 0, rsp["raw_log"]

            # Wait for exit to unlock
            pos_data = query_position(cluster, pos_id)["position"]
            exit_unlock_at = isoparse(pos_data["exit_unlock_at"])
            wait_for_block_time(cluster, exit_unlock_at)
            wait_for_new_blocks(cluster, 1)

            # Undelegate if still delegated
            if pos_data.get("delegated_shares", "0") != "0.000000000000000000":
                rsp = tier_undelegate(cluster, owner, pos_id)
                assert rsp["code"] == 0, rsp["raw_log"]

                unbond_data = find_log_event_attrs(
                    rsp["events"],
                    "chainmain.tieredrewards.v1.EventPositionUndelegated",
                    lambda attrs: "completion_time" in attrs,
                )
                if unbond_data:
                    completion = isoparse(
                        unbond_data["completion_time"].strip('"')
                    ) + timedelta(seconds=1)
                    wait_for_block_time(cluster, completion)
                    wait_for_new_blocks(cluster, 1)

            # Withdraw
            rsp = withdraw(cluster, owner, pos_id)
            assert rsp["code"] == 0, rsp["raw_log"]

func (k Keeper) sufficientBonusPoolBalance(ctx context.Context, bonus sdk.Coins) error {
	if bonus.IsZero() {
		return nil
	}

	poolAddr := k.accountKeeper.GetModuleAddress(types.RewardsPoolName)
	poolBalance := k.bankKeeper.GetAllBalances(ctx, poolAddr)
	if !poolBalance.IsAllGTE(bonus) {
		return errorsmod.Wrapf(types.ErrInsufficientBonusPool,
			"bonus: %s, pool balance: %s",
			bonus.String(), poolBalance.String())
	}

	return nil
