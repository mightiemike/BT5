Audit Report

## Title
uint32 Truncation in `_afterTimelock` Allows Pool Admin to Bypass Timelock on All Timelocked Parameter Changes — (`metric-periphery/contracts/extensions/OracleValueStopLossExtension.sol`)

## Summary
`_afterTimelock` returns `uint32(block.timestamp + oracleStopLossConfig[pool_].timelock)`. When `timelock = type(uint32).max`, the 256-bit sum (~6.04B) exceeds `type(uint32).max` and truncates to a value one second before `block.timestamp`. `_requireElapsed` then passes immediately, allowing the pool admin to execute any timelocked parameter change with zero delay after a one-time setup cost of waiting out the current timelock.

## Finding Description
`_afterTimelock` (L297–299) computes the addition in `uint256` (no arithmetic overflow) but truncates to `uint32`:

```solidity
return uint32(block.timestamp + oracleStopLossConfig[pool_].timelock);
```

With `block.timestamp ≈ 1,753,000,000` and `timelock = type(uint32).max = 4,294,967,295`:
- Sum = `6,047,967,295`
- `uint32(6,047,967,295) = 6,047,967,295 mod 4,294,967,296 = 1,752,999,999`

`_requireElapsed` (L301–303) checks `block.timestamp < executeAfter`. Since `1,753,000,000 < 1,752,999,999` is false, execution proceeds immediately. No validation on the `timelock` value exists in `initialize` (L56–62) or `executeOracleStopLossTimelock` (L86–94), so `type(uint32).max` is a valid input. All four propose/execute pairs (`drawdown`, `decay`, `highWatermarks`, `timelock`) call `_afterTimelock` and `_requireElapsed`, so all are affected.

## Impact Explanation
The timelock is the sole LP protection window. Once the pool admin sets `timelock = type(uint32).max` (itself executable after the current timelock elapses — a one-time cost), all subsequent parameter changes bypass the timelock entirely. The pool admin can then set `drawdownE6 = 1_000_000` (100% drawdown, disabling the stop-loss), `decayPerSecondE8 = 1e8` (100%/s decay, collapsing watermarks to zero on next touch), or zero out watermarks directly — removing all LP value-per-share protection without the required waiting period. This is an admin-boundary break: the pool admin exceeds their authorized capability (parameter changes constrained by timelock).

## Likelihood Explanation
Any pool initialized with `timelock = 0` (no validation prevents this) allows the pool admin to immediately set `timelock = type(uint32).max` and bypass all future timelocks in the same block. With a non-zero initial timelock, the pool admin waits once, then all subsequent changes are instant. The exploit requires only the pool admin's own transactions and no external conditions.

## Recommendation
Add an explicit overflow check in `_afterTimelock`:

```solidity
function _afterTimelock(address pool_) private view returns (uint32) {
    uint256 result = block.timestamp + oracleStopLossConfig[pool_].timelock;
    if (result > type(uint32).max) revert TimelockOverflow();
    return uint32(result);
}
```

Alternatively, cap `timelock` at a safe maximum (e.g., 365 days = 31,536,000) in both `initialize` and `executeOracleStopLossTimelock` via a `_validateTimelock` function, mirroring the existing `_validateDrawdown` and `_validateDecay` pattern.

## Proof of Concept

```solidity
function test_timelockOverflowBypassesDrawdownTimelock() public {
    // Pool initialized with timelock = 0
    vm.startPrank(admin);
    // Immediately set timelock to type(uint32).max (passes: current timelock = 0)
    extension.proposeOracleStopLossTimelock(address(pool), type(uint32).max);
    extension.executeOracleStopLossTimelock(address(pool));

    // Propose drawdown change — _afterTimelock wraps to past
    extension.proposeOracleStopLossDrawdown(address(pool), 1_000_000);
    (, uint32 executeAfter,,) = extension.poolStopLossSchedule(address(pool));
    assertLt(executeAfter, block.timestamp); // executeAfter is in the past

    // Execute immediately — should revert but doesn't
    extension.executeOracleStopLossDrawdown(address(pool));
    vm.stopPrank();

    (uint32 drawdown,,,) = extension.oracleStopLossConfig(address(pool));
    assertEq(drawdown, 1_000_000); // stop-loss disabled
}
```
