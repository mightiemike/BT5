### Title
Silent Bonus Reward Forfeiture with Checkpoint Advancement During Redelegation Slash — (`File: x/tieredrewards/keeper/slash.go`)

### Summary

In `slashRedelegationPosition`, when `processEventsAndClaimBonus` returns `ErrInsufficientBonusPool`, the error is deliberately swallowed to avoid a chain halt. However, by the time the error is returned, the function has already advanced the position's bonus checkpoints (`LastBonusAccrual`, `LastEventSeq`, `LastKnownBonded`) in memory **and** decremented validator event reference counts in the KV store. The caller then persists the advanced checkpoints via `setPositionWithState`. The user's accrued bonus for the entire period since their last claim is permanently and irrecoverably lost — not deferred, not retryable.

---

### Finding Description

In `processEventsAndClaimBonus` (`claim_rewards.go`), the function:

1. Walks all pending validator events, decrementing each event's `ReferenceCount` in the KV store (line 196–198) — a **persistent** state mutation.
2. Advances `pos.LastEventSeq` in memory (line 193).
3. Advances `pos.LastBonusAccrual` to `blockTime` via `applyBonusAccrualCheckpoint` (line 215).
4. Updates `pos.LastKnownBonded` (line 217).
5. Only **then** checks pool sufficiency (line 230) and returns `ErrInsufficientBonusPool` if the pool cannot cover the computed bonus. [1](#0-0) 

Back in `slashRedelegationPosition`, the `ErrInsufficientBonusPool` branch swallows the error and falls through to `setPositionWithState`, which persists the now-advanced checkpoints: [2](#0-1) 

The result is a state where:
- `pos.LastBonusAccrual` has been moved forward to the slash block time.
- `pos.LastEventSeq` has been advanced past all events that were processed.
- Validator event `ReferenceCount`s have been decremented (and events may have been garbage-collected if they reached zero).
- **Zero bonus was paid to the owner.**

On any subsequent `MsgClaimTierRewards`, `MsgTierRedelegate`, `MsgTierUndelegate`, or `MsgClearPosition`, `processEventsAndClaimBonus` will start from the already-advanced `LastBonusAccrual` and `LastEventSeq`, so the forfeited period is invisible to the accounting. The accrued bonus is permanently unrecoverable. [3](#0-2) 

---

### Impact Explanation

The corrupted value is the position owner's accrued bonus reward for the segment `[LastBonusAccrual_before_slash, slash_block_time]`. This is a direct, permanent loss of user funds from the bonus pool's perspective: the pool retains the coins that should have been paid, but the user's checkpoints have been advanced so those coins can never be claimed by that position. The amount lost equals `shares × tokensPerShare × bonusApy × duration / SecondsPerYear` for the forfeited segment — proportional to position size and time since last claim. [4](#0-3) 

---

### Likelihood Explanation

The trigger requires two concurrent conditions:

1. **A tier position is in the redelegation window** (21-day period after `MsgTierRedelegate`). This is a normal user action.
2. **The bonus pool is empty or insufficient** at the moment the slash fires. The pool is continuously drained by the `BeginBlocker` top-up and by user bonus claims; it can reach zero between governance replenishments.

Validator downtime slashing is a routine on-chain event. The combination of a redelegating position and a temporarily empty pool is realistic, especially during periods of high bonus claim activity or when governance has not yet replenished the pool. No privileged access is required; the slash is triggered by the consensus engine. [5](#0-4) 

---

### Recommendation

The fix must ensure that either:

**Option A (preferred):** Do not advance checkpoints or decrement event reference counts unless the bonus is actually paid. Restructure `processEventsAndClaimBonus` to compute the bonus amount first (without mutating state), then check pool sufficiency, and only then perform the KV mutations and checkpoint advancement. This makes the function fully atomic.

**Option B:** In `slashRedelegationPosition`, when `ErrInsufficientBonusPool` is caught, do **not** call `setPositionWithState` with the mutated `pos`. Instead, reload the original position from state (or pass the pre-mutation snapshot) so that checkpoints are not advanced. The event reference count decrements that already occurred in the KV store would also need to be reversed, which makes this option more complex.

**Option C:** Treat `ErrInsufficientBonusPool` in the slash hook the same as other errors — return it and let the slash fail. This prevents the chain halt concern but requires the pool to always be funded before slashes can execute, which may be operationally unacceptable.

Option A is the cleanest: separate the read/compute phase from the write/pay phase inside `processEventsAndClaimBonus`. [6](#0-5) 

---

### Proof of Concept

1. Governance sets `BonusApy = 10%` on tier 1.
2. Alice calls `MsgLockTier` (1,000,000 basecro, validator V1). Position P1 is created; `LastBonusAccrual = T0`.
3. Alice calls `MsgTierRedelegate` (P1, V1 → V2). A redelegation entry is created with `unbondingID = U`. `RedelegationMappings[U] = P1.Id`. `LastBonusAccrual` is reset to `T1 = block_time`.
4. 30 days pass. Alice accrues ~8,219 basecro in bonus (1,000,000 × 0.10 × 30/365).
5. The bonus pool is drained to zero by BeginBlocker top-ups or other user claims.
6. V2 is slashed for downtime. The consensus engine calls `BeforeRedelegationSlashed(ctx, U, sharesToUnbond)`.
7. `slashRedelegationPosition` is called. `processEventsAndClaimBonus` runs:
   - Advances `pos.LastBonusAccrual` to `T1 + 30 days`.
   - Decrements event reference counts.
   - Computes `totalBonus = 8219 basecro`.
   - Calls `sufficientBonusPoolBalance` → pool is 0 → returns `ErrInsufficientBonusPool`.
8. The error is swallowed. `setPositionWithState` persists `pos` with `LastBonusAccrual = T1 + 30 days`.
9. Pool is replenished by governance.
10. Alice calls `MsgClaimTierRewards(P1)`. `processEventsAndClaimBonus` starts from `LastBonusAccrual = T1 + 30 days`. The 30-day segment is gone. Alice receives **0 bonus** for that period.

The 8,219 basecro remain in the pool but are permanently unclaimable by Alice for that period. [7](#0-6) [8](#0-7)

func (k Keeper) slashRedelegationPosition(ctx context.Context, unbondingId uint64, sharesToUnbond math.LegacyDec) error {
	positionId, err := k.getRedelegationMapping(ctx, unbondingId)
	if errors.Is(err, collections.ErrNotFound) {
		return nil
	}
	if err != nil {
		return err
	}

	pos, err := k.getPositionState(ctx, positionId)
	if errors.Is(err, types.ErrPositionNotFound) {
		k.logger(ctx).Error("position not found during redelegation slash",
			"position_id", positionId,
			"unbonding_id", unbondingId,
			"error", err.Error(),
		)
		return nil
	}
	if err != nil {
		return err
	}

	if !pos.IsDelegated() {
		// Defensive
		k.logger(ctx).Error("delegation missing during BeforeRedelegationSlashed",
			"position_id", positionId,
			"unbonding_id", unbondingId,
			"shares_to_unbond", sharesToUnbond.String(),
		)
		return nil
	}

	dstValStr := pos.Delegation.ValidatorAddress

	// Settle bonus against PRE-slash shares.
	if _, err := k.processEventsAndClaimBonus(ctx, &pos); err != nil {
		// Deliberately forgo bonus rewards if pool is insufficient to prevent chain halt.
		if errors.Is(err, types.ErrInsufficientBonusPool) {
			k.logger(ctx).Error("insufficient bonus pool during redelegation slash",
				"position_id", pos.Id,
				"error", err.Error(),
			)
		} else {
			return err
		}
	}

	fullSlash := sharesToUnbond.GTE(pos.Delegation.Shares)

	if fullSlash {
		pos.Delegation = nil
		pos.ClearBonusCheckpoints()
		return k.setPositionWithState(ctx, pos, &ValidatorTransition{PreviousAddress: dstValStr})
	}
	// In-memory only: the persisted Position carries no share count, and the
	// live delegation will reflect the post-Unbond shares on the next read.
	// Update the local copy so any follow-up logic in this call sees consistent state.
	pos.Delegation.Shares = pos.Delegation.Shares.Sub(sharesToUnbond)
	return k.setPositionWithState(ctx, pos, nil)
