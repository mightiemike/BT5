Audit Report

## Title
`SwapAllowlistExtension.beforeSwap` Checks Router Address Instead of End User, Enabling Full Allowlist Bypass via `MetricOmmSimpleRouter` — (`metric-periphery/contracts/extensions/SwapAllowlistExtension.sol`)

## Summary
`SwapAllowlistExtension.beforeSwap` gates swaps by checking the `sender` argument, which the pool sets to `msg.sender` — the direct caller of `pool.swap()`. When users route through `MetricOmmSimpleRouter`, `msg.sender` at the pool is the router contract, not the end user. A pool admin who allowlists the router (the only way to enable router-mediated swaps for their users) inadvertently grants unrestricted swap access to every on-chain caller, completely defeating the per-user allowlist.

## Finding Description
`MetricOmmPool.swap` passes `msg.sender` as the `sender` argument to `_beforeSwap`:

```solidity
// MetricOmmPool.sol L230-231
_beforeSwap(
    msg.sender,   // router address when routed
    ...
);
```

`ExtensionCalling._beforeSwap` forwards this value verbatim to the extension via `abi.encodeCall`. `SwapAllowlistExtension.beforeSwap` then checks:

```solidity
// SwapAllowlistExtension.sol L37
if (!allowAllSwappers[msg.sender] && !allowedSwapper[msg.sender][sender]) {
    revert IMetricOmmPoolActions.NotAllowedToSwap();
}
```

Here `msg.sender` is the pool and `sender` is the router. The check becomes `allowedSwapper[pool][router]`. The extension never sees the originating user's address. `MetricOmmSimpleRouter.exactInputSingle` (and `exactInput`, `exactOutputSingle`, `exactOutput`) are all `external` with no caller restriction, so any EOA or contract can invoke them. Once `allowedSwapper[pool][router] = true`, the condition passes for every caller of the router.

## Impact Explanation
A pool configured with `SwapAllowlistExtension` to restrict swaps to a curated set (e.g., KYC-gated, institutional, or market-maker-only pool) loses that restriction entirely the moment the router is allowlisted. Unprivileged users can execute swaps at oracle-derived prices against LP positions provisioned under the assumption of a restricted counterparty set, causing direct loss of LP principal. This is a broken core pool invariant: the allowlist guard is the pool's primary access-control mechanism for swaps.

## Likelihood Explanation
The precondition — admin allowlisting the router — is the natural and only available action for any pool admin who wants their allowlisted users to use the standard periphery router. The extension provides no "allowlist the router but still check the originating user" option. The bypass is therefore a predictable consequence of normal, well-intentioned admin configuration, not a malicious or exotic setup. All four router entry points (`exactInputSingle`, `exactInput`, `exactOutputSingle`, `exactOutput`) are unrestricted.

## Recommendation
The extension must gate on the originating user, not the direct pool caller. Two viable approaches:

1. **Pass the originating user through `extensionData`**: The router encodes `msg.sender` into `extensionData`; the extension decodes and checks it. Requires a convention between router and extension.
2. **Add an `origin` field to the `beforeSwap` interface**: Analogous to `tx.origin` but passed explicitly and validated by the router, so the extension can check the true initiator regardless of intermediary contracts.

A weaker mitigation is to add a factory-level check that rejects the router address from the allowlist, preventing the most common misconfiguration.

## Proof of Concept
```solidity
// Setup
SwapAllowlistExtension ext = new SwapAllowlistExtension(factory);
// Pool admin allowlists the router so trusted users can use it
ext.setAllowedToSwap(pool, address(router), true);
// Alice (0xA11CE) is NOT individually allowlisted
// allowedSwapper[pool][0xA11CE] == false

// Attack: Alice routes through the router
vm.prank(address(0xA11CE));
router.exactInputSingle(
    IMetricOmmSimpleRouter.ExactInputSingleParams({
        pool:             address(pool),
        tokenIn:          token0,
        tokenOut:         token1,
        zeroForOne:       true,
        amountIn:         1_000e18,
        amountOutMinimum: 0,
        recipient:        address(0xA11CE),
        deadline:         block.timestamp + 1,
        priceLimitX64:    0,
        extensionData:    ""
    })
);
// SwapAllowlistExtension.beforeSwap receives sender=router
// allowedSwapper[pool][router] == true → passes
// Alice's swap executes despite never being individually allowlisted
```
