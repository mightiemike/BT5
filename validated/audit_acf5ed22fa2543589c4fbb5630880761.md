Audit Report

## Title
`SwapAllowlistExtension` Allowlist Bypassed for All Router-Mediated Swaps — (`metric-periphery/contracts/extensions/SwapAllowlistExtension.sol`)

## Summary
`MetricOmmPool.swap` passes `msg.sender` (the immediate caller) as `sender` to `_beforeSwap`, so when `MetricOmmSimpleRouter` calls `pool.swap`, the extension receives the router's address as `sender`, not the end user's address. `SwapAllowlistExtension.beforeSwap` checks `allowedSwapper[pool][sender]` against this router address, meaning allowlisting the router to enable normal UX opens the gate to every address on the network.

## Finding Description
`MetricOmmPool.swap` invokes `_beforeSwap` with `msg.sender` as the first argument:

```solidity
// metric-core/contracts/MetricOmmPool.sol L230-240
_beforeSwap(
  msg.sender,   // ← always the immediate caller of pool.swap
  recipient, ...
);
```

`ExtensionCalling._beforeSwap` forwards this value unchanged to every configured extension via `abi.encodeCall(IMetricOmmExtensions.beforeSwap, (sender, ...))`.

`SwapAllowlistExtension.beforeSwap` then checks:

```solidity
// metric-periphery/contracts/extensions/SwapAllowlistExtension.sol L37
if (!allowAllSwappers[msg.sender] && !allowedSwapper[msg.sender][sender]) {
```

Here `msg.sender` is the pool and `sender` is whoever called `pool.swap`. When a user calls `MetricOmmSimpleRouter.exactInputSingle`, the router calls `pool.swap(...)` directly:

```solidity
// metric-periphery/contracts/MetricOmmSimpleRouter.sol L72-80
(int128 amount0Delta, int128 amount1Delta) = IMetricOmmPoolActions(params.pool)
  .swap(params.recipient, params.zeroForOne, ...);
```

The router never forwards the original `msg.sender` (the user) into any field that reaches the extension. The pool's `swap` signature has no explicit `swapper` parameter — it always derives `sender` from `msg.sender`. The resulting check is `allowedSwapper[pool][router]`, not `allowedSwapper[pool][user]`.

Two outcomes:
- **Router not allowlisted**: Legitimate allowlisted users cannot swap through the router (DoS of normal UX).
- **Router allowlisted** (required for any user to use the router): Every address on the network can bypass the allowlist by calling the router.

The same bypass applies to `exactInput`, `exactOutputSingle`, and `exactOutput` in `MetricOmmSimpleRouter`.

## Impact Explanation
A pool configured with `SwapAllowlistExtension` for KYC/compliance, institutional-only access, or rate-limited liquidity can be freely accessed by any address through the standard periphery router. The allowlist guard — the only on-chain enforcement of the access policy — is completely ineffective for router-mediated swaps. Unauthorized users can execute swaps against restricted liquidity, violating the pool admin's intended access control. This constitutes broken core pool functionality (the access control extension) and potential unauthorized drain of LP assets at prices the pool was not designed to offer to arbitrary counterparties. Severity: **Medium**.

## Likelihood Explanation
`MetricOmmSimpleRouter` is the standard, documented user-facing entry point. Any user following normal UX routes through it. The bypass requires no special knowledge, no privileged role, and no unusual token behavior — just calling the public router with any token amount. The condition (router allowlisted) is the only practical configuration that allows legitimate users to use the router at all, making the vulnerable state the expected production state. Likelihood: **High**.

## Recommendation
The extension must verify the economic actor (the end user), not the immediate `msg.sender` of `pool.swap`. The cleanest fix is to add an explicit `swapper` parameter to `MetricOmmPool.swap` (separate from `msg.sender`) that the router populates with `msg.sender` (the user). The extension would then check this field. Alternatively, document that pools using `SwapAllowlistExtension` must require users to call `pool.swap` directly and must never allowlist the router — but this breaks normal UX and is not a real fix.

## Proof of Concept
```
Setup:
  - Deploy pool with SwapAllowlistExtension configured
  - Pool admin calls setAllowedToSwap(pool, alice, true)
  - Pool admin calls setAllowedToSwap(pool, router, true)
    (required for alice to use the router)

Attack:
  - bob (not allowlisted) calls MetricOmmSimpleRouter.exactInputSingle(pool, ...)
  - Router calls pool.swap(...) → msg.sender = router
  - Pool calls _beforeSwap(msg.sender=router, ...)
  - Extension.beforeSwap receives sender=router
  - Checks allowedSwapper[pool][router] → true
  - Bob's swap succeeds despite not being on the allowlist

Result:
  - The allowlist is bypassed; bob swaps freely in a restricted pool
  - If the pool admin does NOT allowlist the router, alice cannot use
    the router either (checks allowedSwapper[pool][router] → false → revert)
