### Title
No upper-bound validation on `min_gas_price_percentage` in `StatefulTransactionValidatorConfig` causes gateway to reject all valid transactions - (`crates/apollo_gateway_config/src/config.rs`)

### Summary

`StatefulTransactionValidatorConfig.min_gas_price_percentage` is a `u8` (range 0–255) with no upper-bound validation constraint. When set above 100, the computed admission threshold exceeds the previous block's L2 gas price, causing the gateway to reject every valid `AllResources` transaction — a complete DOS on transaction admission.

### Finding Description

`StatefulTransactionValidatorConfig` derives `Validate` but the `min_gas_price_percentage` field carries no `#[validate(range(...))]` attribute:

```rust
// crates/apollo_gateway_config/src/config.rs:276-287
#[derive(Clone, Debug, Serialize, Deserialize, Validate, PartialEq)]
pub struct StatefulTransactionValidatorConfig {
    pub validate_resource_bounds: bool,
    ...
    pub min_gas_price_percentage: u8, // E.g., 80 to require 80% of threshold.
}
``` [1](#0-0) 

The admission check in `validate_tx_l2_gas_price_within_threshold` computes the threshold as:

```rust
// crates/apollo_gateway/src/stateful_transaction_validator.rs:367-372
let gas_price_threshold_multiplier =
    Ratio::new(self.config.min_gas_price_percentage.into(), 100_u128);
let threshold = (gas_price_threshold_multiplier
    * previous_block_l2_gas_price.get().0)
    .to_integer();
if tx_l2_gas_price.0 < threshold { return Err(...) }
``` [2](#0-1) 

If `min_gas_price_percentage = 200`, the threshold becomes `2 × previous_block_l2_gas_price`. Because the EIP-1559 fee market only allows the price to change by a bounded fraction per block (controlled by `gas_price_max_change_denominator = 48`), no honest transaction can ever offer a gas price ≥ 200% of the previous block's price. Every `AllResources` transaction is rejected. [3](#0-2) 

By contrast, the analogous `fee_escalation_percentage` field in `MempoolStaticConfig` is correctly guarded:

```rust
// crates/apollo_mempool_config/src/config.rs:65-66
#[validate(range(min = 1, max = 100))]
pub fee_escalation_percentage: u8,
``` [4](#0-3) 

The test suite for `MempoolStaticConfig` explicitly verifies that `fee_escalation_percentage = 0` fails validation, but no equivalent test exists for `min_gas_price_percentage > 100`. [5](#0-4) 

### Impact Explanation

With `min_gas_price_percentage` set to any value above 100, `validate_resource_bounds` (called during gateway admission) rejects every `AllResources` (v3) transaction. The gateway returns `GAS_PRICE_TOO_LOW` for all such transactions. `L1Gas`-only (legacy) transactions are unaffected because the check is skipped for that variant. This is a complete DOS on v3 transaction admission — matching the **High** impact: *Mempool/gateway/RPC admission rejects valid transactions before sequencing*.

### Likelihood Explanation

The field is operator-controlled via the config JSON (`gateway_config.static_config.stateful_tx_validator_config.min_gas_price_percentage`, default 100). A misconfiguration — e.g., an operator intending to set 100% but accidentally writing 200, or a deployment script applying a wrong multiplier — silently passes config validation and takes effect immediately. The absence of a `#[validate(range(max = 100))]` guard means the error is not caught at startup. [6](#0-5) 

### Recommendation

Add an upper-bound validation constraint to `min_gas_price_percentage`, mirroring the pattern already used for `fee_escalation_percentage`:

```rust
#[validate(range(min = 0, max = 100))]
pub min_gas_price_percentage: u8,
```

Add a corresponding test in `apollo_gateway_config` that asserts `StatefulTransactionValidatorConfig { min_gas_price_percentage: 101, ..Default::default() }.validate().is_err()`.

### Proof of Concept

1. Set `gateway_config.static_config.stateful_tx_validator_config.min_gas_price_percentage = 200` in the node config.
2. Start the gateway with `validate_resource_bounds: true`.
3. Submit any `InvokeTransactionV3` with `AllResources` bounds where `l2_gas.max_price_per_unit` equals the current block's L2 gas price (the normal, correct value).
4. `validate_tx_l2_gas_price_within_threshold` computes `threshold = 2 × previous_block_l2_gas_price`, which exceeds the transaction's offered price.
5. The gateway returns `StarknetErrorCode.GAS_PRICE_TOO_LOW` for every such transaction.
6. All v3 transactions are permanently rejected; the mempool receives nothing; block production stalls on v3 traffic. [7](#0-6)

    // TODO(Arni): Consider running this validation for all gas prices.
    fn validate_tx_l2_gas_price_within_threshold(
        &self,
        tx_resource_bounds: ValidResourceBounds,
        previous_block_l2_gas_price: NonzeroGasPrice,
    ) -> StatefulTransactionValidatorResult<()> {
        match tx_resource_bounds {
            ValidResourceBounds::AllResources(tx_resource_bounds) => {
                let tx_l2_gas_price = tx_resource_bounds.l2_gas.max_price_per_unit;
                let gas_price_threshold_multiplier =
                    Ratio::new(self.config.min_gas_price_percentage.into(), 100_u128);
                let threshold = (gas_price_threshold_multiplier
                    * previous_block_l2_gas_price.get().0)
                    .to_integer();
                if tx_l2_gas_price.0 < threshold {
                    return Err(StarknetError {
                        // We didn't have this kind of an error.
                        code: StarknetErrorCode::UnknownErrorCode(
                            "StarknetErrorCode.GAS_PRICE_TOO_LOW".to_string(),
                        ),
                        message: format!(
                            "Transaction L2 gas price {tx_l2_gas_price} is below the required \
                             threshold {threshold}.",
                        ),
                    });
                }
            }
            ValidResourceBounds::L1Gas(_) => {
                // No validation required for legacy transactions.
            }
        }
        Ok(())
    }

**File:** crates/apollo_consensus_orchestrator/src/fee_market/mod.rs (L86-140)
```rust
pub fn calculate_next_base_gas_price(
    price: GasPrice,
    gas_used: GasAmount,
    gas_target: GasAmount,
    min_gas_price: GasPrice,
) -> GasPrice {
    let versioned_constants = VersionedConstants::latest_constants();
    assert!(
        gas_target < versioned_constants.max_block_size,
        "Gas target must be lower than max block size."
    );
    assert!(gas_target.0 > 0, "Gas target must be greater than zero.");
    assert!(
        versioned_constants.gas_price_max_change_denominator > 0,
        "Denominator constant must be greater than zero."
    );

    // If the current price is below the minimum, apply a gradual adjustment and return early.
    // This allows the price to increase by at most 1/MIN_GAS_PRICE_INCREASE_DENOMINATOR per block.
    if price < min_gas_price {
        let max_increase = price.0 / MIN_GAS_PRICE_INCREASE_DENOMINATOR;
        let adjusted = price.0 + max_increase;
        // Cap at min_gas_price to avoid overshooting
        let adjusted_price = adjusted.min(min_gas_price.0);
        info!(
            "Fee Market: Price {} below minimum gas price {}, adjusted price: {} )",
            price.0, min_gas_price.0, adjusted_price
        );
        return GasPrice(adjusted_price);
    }

    // Use U256 to avoid overflow, as multiplying a u128 by a u64 remains within U256 bounds.
    let gas_delta = U256::from(gas_used.0.abs_diff(gas_target.0));
    let gas_target_u256 = U256::from(gas_target.0);
    let price_u256 = U256::from(price.0);

    // Calculate price change by multiplying first, then dividing. This avoids the precision loss
    // that occurs when dividing before multiplying.
    let denominator =
        gas_target_u256 * U256::from(versioned_constants.gas_price_max_change_denominator);
    let price_change = (price_u256 * gas_delta) / denominator;

    let adjusted_price_u256 =
        if gas_used > gas_target { price_u256 + price_change } else { price_u256 - price_change };

    // Sanity check: ensure direction of change is correct
    assert!(
        gas_used > gas_target && adjusted_price_u256 >= price_u256
            || gas_used <= gas_target && adjusted_price_u256 <= price_u256
    );

    // Price should not realistically exceed u128::MAX, bound to avoid theoretical overflow.
    let adjusted_price = u128::try_from(adjusted_price_u256).unwrap_or(u128::MAX);
    GasPrice(max(adjusted_price, min_gas_price.0))
}
