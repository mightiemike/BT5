### Title
User Burns NLP Tokens But Receives No Quote Assets Due to Rounding-to-Zero — (`File: core/contracts/Clearinghouse.sol`)

---

### Summary

In `Clearinghouse.burnNlp()`, a user's NLP tokens are unconditionally burned, but the quote credit is silently skipped when `quoteAmount` rounds to zero or is entirely consumed by the minimum burn fee. The function does not revert in this case, resulting in a permanent loss of the user's NLP balance with no assets returned.

---

### Finding Description

The `burnNlp` function computes the quote amount a user should receive for burning their NLP tokens using fixed-point multiplication:

```solidity
int128 quoteAmount = nlpAmount.mul(oraclePriceX18);
int128 burnFee = MathHelper.max(ONE, quoteAmount / 1000);
quoteAmount = MathHelper.max(0, quoteAmount - burnFee);
```

`mul` is defined as `(int256(x) * y) / ONE_X18` where `ONE_X18 = 1e18`. If `nlpAmount * oraclePriceX18 < 1e18`, the result truncates to zero. Even when `quoteAmount` is non-zero, the minimum burn fee is `ONE` (1e18 in X18 format, representing 1 full unit of quote). Any `quoteAmount` smaller than `ONE` is entirely consumed by the fee, leaving `quoteAmount = 0` after `MathHelper.max(0, quoteAmount - burnFee)`.

The function then conditionally credits the user:

```solidity
spotEngine.updateBalance(NLP_PRODUCT_ID, txn.sender, -nlpAmount); // always executes
spotEngine.updateBalance(NLP_PRODUCT_ID, N_ACCOUNT, nlpAmount);   // always executes

if (quoteAmount > 0) {
    spotEngine.updateBalance(QUOTE_PRODUCT_ID, txn.sender, quoteAmount);
    _applyNlpRebalance(spotEngine, nlpPools, nlpPoolRebalanceX18);
}
```

The NLP debit at line 511 is unconditional. The quote credit at line 515 is gated on `quoteAmount > 0`. When `quoteAmount == 0`, the user's NLP is permanently burned and no quote is credited. The function does not revert.

---

### Impact Explanation

A user who submits a `BurnNlp` transaction with an `nlpAmount` small enough that the resulting `quoteAmount` is zero (or less than `ONE` after fee deduction) will have their NLP tokens permanently destroyed with no quote assets received. This is a direct, irreversible loss of user funds. The corrupted state delta is: `NLP balance of txn.sender` decreases by `nlpAmount`, `QUOTE balance of txn.sender` does not increase.

---

### Likelihood Explanation

Any user holding NLP tokens can submit a `BurnNlp` transaction with a small `nlpAmount`. The threshold at which this triggers is when `nlpAmount * oraclePriceX18 < ONE` (i.e., the NLP position is worth less than 1 quote unit in X18 terms). For example, if NLP price is $1.00 (1e18 X18) and a user burns 0.5 NLP (5e17 X18): `quoteAmount = (5e17 * 1e18) / 1e18 = 5e17`, `burnFee = max(1e18, 5e14) = 1e18`, `quoteAmount = max(0, 5e17 - 1e18) = 0`. The user loses 0.5 NLP with no return. This is reachable by any unprivileged user through the standard `Endpoint` transaction submission path.

---

### Recommendation

Add a zero-check on `quoteAmount` before executing the NLP burn, and revert if the user would receive nothing:

```solidity
int128 quoteAmount = nlpAmount.mul(oraclePriceX18);
int128 burnFee = MathHelper.max(ONE, quoteAmount / 1000);
quoteAmount = MathHelper.max(0, quoteAmount - burnFee);

require(quoteAmount > 0, "zero quote return");

spotEngine.updateBalance(NLP_PRODUCT_ID, txn.sender, -nlpAmount);
...
```

---

### Proof of Concept

1. User holds NLP tokens and submits a signed `BurnNlp` transaction with `nlpAmount = 5e17` (0.5 NLP in X18).
2. Oracle price is `1e18` ($1.00 per NLP).
3. Sequencer submits the transaction to `Endpoint`, which calls `Clearinghouse.burnNlp()`.
4. `quoteAmount = (5e17 * 1e18) / 1e18 = 5e17`.
5. `burnFee = MathHelper.max(1e18, 5e17 / 1000) = MathHelper.max(1e18, 5e14) = 1e18`.
6. `quoteAmount = MathHelper.max(0, 5e17 - 1e18) = 0`.
7. `spotEngine.updateBalance(NLP_PRODUCT_ID, txn.sender, -5e17)` executes — user loses 0.5 NLP.
8. `if (quoteAmount > 0)` is false — no quote is credited.
9. User has permanently lost 0.5 NLP with zero quote received.
