Audit Report

## Title
SwapAllowlistExtension Checks Router Address Instead of End-User, Allowing Any User to Bypass the Swap Allowlist — (File: `metric-periphery/contracts/extensions/SwapAllowlistExtension.sol`)

## Summary
`SwapAllowlistExtension.beforeSwap` gates swaps by checking the `sender` argument, which is `msg.sender` of the pool's `swap` call. When a user swaps through `MetricOmmSimpleRouter`, the router is `msg.sender` at the pool level, so the extension checks the router's address against the allowlist rather than the end user's address. Once the pool admin allowlists the router (the only way to permit router-based swaps for legitimate users), every unpermissioned user can bypass the curated allowlist by routing through the router.

## Finding Description
`MetricOmmPool.swap` passes `msg.sender` as the `sender` argument to `_beforeSwap`:

```solidity
// MetricOmmPool.sol L230-240
_beforeSwap(
  msg.sender,   // ← router address when called via router
  recipient,
  ...
);
```

`ExtensionCalling._beforeSwap` forwards this value verbatim to the extension via `abi.encodeCall(IMetricOmmExtensions.beforeSwap, (sender, ...))`.

`SwapAllowlistExtension.beforeSwap` then checks:

```solidity
// SwapAllowlistExtension.sol L37-38
if (!allowAllSwappers[msg.sender] && !allowedSwapper[msg.sender][sender]) {
  revert IMetricOmmPoolActions.NotAllowedToSwap();
}
```

When a user calls `MetricOmmSimpleRouter.exactInputSingle`, the router calls `pool.swap(...)` directly. At the pool, `msg.sender` is the router, so `sender` forwarded to the extension is the router address. The extension therefore checks `allowedSwapper[pool][router]`, not `allowedSwapper[pool][user]`. A pool admin who wants allowlisted users to access the standard router UX must call `setAllowedToSwap(pool, router, true)`. Once the router is allowlisted, any address can call the router and have the extension pass, because the extension only sees the router as the swapper — not the actual end user.

## Impact Explanation
A pool configured with `SwapAllowlistExtension` is intended to restrict trading to a curated set of addresses (e.g., KYC-verified counterparties, whitelisted market makers). Once the pool admin allowlists the router to support standard UX, the allowlist is completely bypassed for all router-mediated swaps. Any unpermissioned user can trade against the pool's liquidity, violating the pool's curation policy and potentially draining LP funds through trades the pool was designed to reject. This is a direct bypass of a configured access-control guard on a production swap path, reachable by any unprivileged user with no special setup. **Severity: High.**

## Likelihood Explanation
The scenario is highly likely in practice. A pool admin deploys a pool with `SwapAllowlistExtension` to restrict trading, allowlists specific users, and also allowlists the router so those users can use the standard interface. Any non-allowlisted user can then discover they can call the router and bypass the guard. The router is the canonical swap entrypoint in the periphery; allowlisting it is the natural and expected action for any pool admin who wants their allowlisted users to have a normal UX.

## Recommendation
`SwapAllowlistExtension` should check the originating user rather than the immediate pool caller. The cleanest fix is for the router to encode the original `msg.sender` in `extensionData`, and for the extension to decode and verify that address when `sender` is a known router. Alternatively, the extension could maintain a registry of trusted routers and, when `sender` is a trusted router, decode the real caller from `extensionData`.

## Proof of Concept
```
1. Deploy pool with SwapAllowlistExtension as beforeSwap hook.
2. Pool admin calls setAllowedToSwap(pool, alice, true)       // allowlist Alice
3. Pool admin calls setAllowedToSwap(pool, router, true)      // allow router so Alice can use it
4. Bob (not allowlisted) calls MetricOmmSimpleRouter.exactInputSingle(...)
   → router calls pool.swap(recipient=Bob, ...)
   → pool calls _beforeSwap(sender=router, ...)
   → extension checks allowedSwapper[pool][router] == true  ✓
   → swap executes — Bob bypasses the allowlist entirely
```
