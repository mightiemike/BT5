### Title
Minimum Burn Fee Exceeds Redemption Value for Small NLP Burns, Destroying User Value — (`core/contracts/Clearinghouse.sol`)

### Summary

In `burnNlp`, the minimum fee floor of `ONE = 1e18` can exceed the entire `quoteAmount` computed for a small NLP burn. When this happens, `quoteAmount` is clamped to zero, the user's NLP balance is unconditionally decremented, and no quote is returned. The NLP value is silently transferred to `N_ACCOUNT` (the pool), violating the invariant that a burn must return quote proportional to the burned amount minus a reasonable fee.

---

### Finding Description

The relevant sequence in `Clearinghouse.sol`:

```solidity
int128 quoteAmount = nlpAmount.mul(oraclePriceX18);           // line 502
int128 burnFee = MathHelper.max(ONE, quoteAmount / 1000);     // line 503
quoteAmount = MathHelper.max(0, quoteAmount - burnFee);       // line 504
...
spotEngine.updateBalance(NLP_PRODUCT_ID, txn.sender, -nlpAmount);  // line 511 — unconditional
spotEngine.updateBalance(NLP_PRODUCT_ID, N_ACCOUNT, nlpAmount);    // line 512 — unconditional

if (quoteAmount > 0) {                                         // line 514
    spotEngine.updateBalance(QUOTE_PRODUCT_ID, txn.sender, quoteAmount);
    _applyNlpRebalance(...);
}
``` [1](#0-0) 

`ONE` is defined as `10**18`: [2](#0-1) 

`MathSD21x18.mul` computes `(int256(x) * y) / 1e18`: [3](#0-2) 

**Trigger condition:** whenever `nlpAmount * oraclePriceX18 / 1e18 < 1e18`, i.e., the NLP being burned is worth less than 1 USDC in quote terms.

Concrete example at NLP price = $1 (`oraclePriceX18 = 1e18`):
- `nlpAmount = 0.5e18` (0.5 NLP tokens)
- `quoteAmount = 0.5e18 * 1e18 / 1e18 = 0.5e18`
- `burnFee = max(1e18, 0.5e18/1000) = 1e18`
- `quoteAmount = max(0, 0.5e18 − 1e18) = 0`
- Lines 511–512 execute unconditionally: user loses 0.5 NLP
- Line 514 condition is false: user receives 0 quote

There is **no minimum burn amount check** anywhere in the function: [4](#0-3) 

The `_validateNlpRebalance` call with `-quoteAmount = 0` is satisfied by passing an empty `nlpPoolRebalanceX18` array (sum = 0), so the sequencer can construct a valid transaction for this case: [5](#0-4) 

The post-burn health check does not prevent the loss — it only ensures the user remains non-negative in health, which passes if the user holds other collateral: [6](#0-5) 

---

### Impact Explanation

- The user's NLP is destroyed (transferred to `N_ACCOUNT`) with zero quote returned.
- The burned NLP value accrues to the pool, constituting an involuntary donation.
- This is a broken accounting identity: `user_quote_delta` should equal `nlpAmount * price − fee`, but instead equals `0` when `nlpAmount * price < ONE`.
- The maximum loss per transaction is bounded by the NLP value burned (up to just under 1 USDC at current price), but the path is repeatable and affects any user who burns a sub-1-USDC NLP amount — including users who accumulate dust NLP positions.

---

### Likelihood Explanation

The path is reachable by any non-isolated subaccount holder with unlocked NLP. The user signs the `BurnNlp` transaction (including `oraclePriceX18` as part of the full `transaction` bytes passed to `validateSignedTx`): [7](#0-6) 

At any realistic NLP price, burning fewer tokens than `1e18 / price` raw units triggers the condition. This is a normal user action (redeeming a small or dust NLP balance) and requires no special privileges.

---

### Recommendation

Add a minimum burn amount guard that rejects the transaction when `quoteAmount < ONE` before the NLP balance is mutated, rather than silently clamping to zero:

```solidity
int128 quoteAmount = nlpAmount.mul(oraclePriceX18);
int128 burnFee = MathHelper.max(ONE, quoteAmount / 1000);
require(quoteAmount > burnFee, ERR_BURN_AMOUNT_TOO_SMALL);
quoteAmount = quoteAmount - burnFee;
```

This ensures the burn is rejected rather than silently destroying user value.

---

### Proof of Concept

```solidity
// Setup: user holds 0.5e18 unlocked NLP, NLP oracle price = 1e18 ($1)
// quoteAmount = 0.5e18 * 1e18 / 1e18 = 0.5e18
// burnFee    = max(1e18, 0.5e18/1000) = 1e18
// quoteAmount = max(0, 0.5e18 - 1e18) = 0

int128 quoteBefore = spotEngine.getBalance(QUOTE_PRODUCT_ID, user).amount;
int128 nlpBefore   = spotEngine.getBalance(NLP_PRODUCT_ID,   user).amount;

// Submit BurnNlp with nlpAmount = 0.5e18, oraclePriceX18 = 1e18
// nlpPoolRebalanceX18 = [] (empty, sums to 0 = -quoteAmount)
burnNlp(txn, 1e18, [], []);

int128 quoteAfter = spotEngine.getBalance(QUOTE_PRODUCT_ID, user).amount;
int128 nlpAfter   = spotEngine.getBalance(NLP_PRODUCT_ID,   user).amount;

assert(nlpAfter   == nlpBefore - 0.5e18);  // NLP decremented ✓
assert(quoteAfter == quoteBefore);          // Quote unchanged — user received nothing ✓
// Invariant broken: user lost 0.5 NLP worth $0.50 with zero compensation
```
