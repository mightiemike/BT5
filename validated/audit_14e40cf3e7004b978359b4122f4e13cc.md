### Title
Governance Deadlock: `MsgUpdateTier` BonusApy Change Blocked by Depleted Bonus Pool — (File: `x/tieredrewards/keeper/msg_server_auth.go`)

### Summary
`MsgUpdateTier` forces an immediate settlement of all accrued bonus rewards for every position in a tier whenever `BonusApy` changes. If the bonus pool (`RewardsPoolName` module account) is insufficient to cover the total accrued bonus, the governance proposal execution fails with `ErrInsufficientBonusPool`. Because governance is the **only** authority that can change `BonusApy`, and pool depletion is caused by the current high `BonusApy`, a circular deadlock forms: governance cannot lower `BonusApy` to reduce pool drain because the pool is already empty.

### Finding Description

In `UpdateTier`, when the incoming `BonusApy` differs from the stored value, the handler calls `claimRewardsAndUpdateTierPositions` before writing the new tier:

```go
// x/tieredrewards/keeper/msg_server_auth.go:67-71
if !oldTier.BonusApy.Equal(msg.Tier.BonusApy) {
    if err := ms.claimRewardsAndUpdateTierPositions(ctx, msg.Tier.Id); err != nil {
        return nil, err
    }
}
```

`claimRewardsAndUpdateTierPositions` iterates every position in the tier and calls `processEventsAndClaimBonus` for each delegated position:

```go
// x/tieredrewards/keeper/claim_rewards.go:59-77
for _, id := range ids {
    ...
    if _, err := k.processEventsAndClaimBonus(ctx, &pos); err != nil {
        return err
    }
    ...
}
```

`processEventsAndClaimBonus` calls `sufficientBonusPoolBalance`, which returns `ErrInsufficientBonusPool` if the pool cannot cover the computed bonus:

```go
// x/tieredrewards/keeper/claim_rewards.go:230-232
if err := k.sufficientBonusPoolBalance(ctx, bonusCoins); err != nil {
    return nil, err
}
```

This error propagates atomically up through the governance proposal execution, causing the entire `MsgUpdateTier` transaction to revert. The tier's `BonusApy` is never updated.

The `requireAuthority` check in `UpdateTier` hard-codes the governance module account as the only valid caller:

```go
// x/tieredrewards/keeper/msg_server_auth.go:14-18
func (ms msgServer) requireAuthority(msgAuthority string) error {
    if ms.getAuthority() != msgAuthority {
        return errors.Wrapf(govtypes.ErrInvalidSigner, ...)
    }
    return nil
}
```

There is no emergency out-of-band path to change `BonusApy` without passing a governance proposal. The test `TestUpdateTier_BonusApyChange_InsufficientPool` explicitly confirms this failure mode:

```go
// x/tieredrewards/keeper/msg_server_auth_test.go:339-365
_, err := msgServer.UpdateTier(s.ctx, &types.MsgUpdateTier{...})
s.Require().ErrorIs(err, types.ErrInsufficientBonusPool)
// Tier should NOT have been updated (tx failed).
```

### Impact Explanation

When the bonus pool is depleted and governance attempts to reduce `BonusApy` to slow future drain, the proposal execution fails. The tier's `BonusApy` remains at the high value. The pool continues to be drained by future user claims. Governance is deadlocked from making the corrective change.

A secondary consequence: with an empty pool, user-driven paths that settle rewards before state mutation (`MsgTierUndelegate`, `MsgTierRedelegate`, `MsgClearPosition`, `MsgExitTierWithDelegation`) also fail atomically. Users with delegated positions cannot exit the tier until the pool is replenished. Their delegated tokens are effectively locked in the module.

The exact corrupted invariant is: the governance module's ability to execute `MsgUpdateTier` with a `BonusApy` reduction is blocked by the bonus pool balance — a module-permission flaw where the system's own state prevents the only authorized corrective action.

### Likelihood Explanation

The pool can be depleted by:
1. A high `BonusApy` causing large accrued bonuses across many positions over time.
2. The BeginBlocker (`TargetBaseRewardsRate > 0`) continuously transferring from the rewards pool to the distribution module for base reward top-up.
3. Insufficient external replenishment of the pool.

Once the pool is depleted, any governance proposal to reduce `BonusApy` will fail at execution time. The deadlock is self-reinforcing: the high `BonusApy` that caused the depletion cannot be corrected because correction requires paying out all accrued bonuses at the old rate first.

### Recommendation

Decouple the forced settlement from the `BonusApy` change. Instead of paying out all accrued bonuses at the old rate before writing the new tier, snapshot the `BonusApy` change time and allow positions to lazily settle the pre-change segment at the old rate on their next claim. This is consistent with the existing lazy validator-event model already used for `SLASH`/`UNBOND`/`BOND` events.

Alternatively, implement an emergency governance path that allows changing `BonusApy` without triggering the forced settlement sweep, accepting that positions will settle at the new rate going forward (forfeiting the delta). At minimum, document the operational requirement to pre-fund the pool before submitting a `BonusApy`-reducing proposal.

### Proof of Concept

1. Tier 1 is deployed with `BonusApy = 0.10` (10%). Many delegators lock positions over months.
2. The bonus pool is depleted — either by BeginBlocker base-reward top-ups or by user claims.
3. Governance submits `MsgUpdateTier{id: 1, bonus_apy: 0.05, ...}` to halve the APY and slow drain.
4. The proposal passes the voting period.
5. At execution, `UpdateTier` detects `BonusApy` changed and calls `claimRewardsAndUpdateTierPositions(ctx, 1)`.
6. For each delegated position, `processEventsAndClaimBonus` computes accrued bonus and calls `sufficientBonusPoolBalance`.
7. Pool balance is 0; `sufficientBonusPoolBalance` returns `ErrInsufficientBonusPool`.
8. The entire proposal execution reverts. Tier 1 still has `BonusApy = 0.10`.
9. Governance cannot retry without first externally funding the pool with enough tokens to cover all accrued bonuses across all positions — an amount that grows with each passing block at the 10% rate.

func (ms msgServer) UpdateTier(ctx context.Context, msg *types.MsgUpdateTier) (*types.MsgUpdateTierResponse, error) {
	if err := ms.requireAuthority(msg.Authority); err != nil {
		return nil, err
	}

	oldTier, err := ms.getTier(ctx, msg.Tier.Id)
	if err != nil {
		return nil, err
	}

	if !oldTier.BonusApy.Equal(msg.Tier.BonusApy) {
		if err := ms.claimRewardsAndUpdateTierPositions(ctx, msg.Tier.Id); err != nil {
			return nil, err
		}
	}

	if err := ms.SetTier(ctx, msg.Tier); err != nil {
		return nil, err
	}

	if err := ms.emitTierChangedEvent(ctx, types.TierChangeAction_TIER_CHANGE_ACTION_UPDATE, msg.Tier); err != nil {
		return nil, err
	}

	return &types.MsgUpdateTierResponse{}, nil
}

// claimRewardsAndUpdateTierPositions claims base and bonus rewards for all
// delegated positions in the given tier.
func (k Keeper) claimRewardsAndUpdateTierPositions(ctx context.Context, tierId uint32) error {
	ids, err := k.getPositionsIdsByTier(ctx, tierId)
	if err != nil {
		return err
	}
	if len(ids) == 0 {
		return nil
	}

	for _, id := range ids {
		pos, err := k.getPositionState(ctx, id)
		if err != nil {
			return err
		}
		if !pos.IsDelegated() {
			continue
		}

		if _, err := k.claimBaseRewards(ctx, pos); err != nil {
			return err
		}
		if _, err := k.processEventsAndClaimBonus(ctx, &pos); err != nil {
			return err
		}
		if err := k.setPosition(ctx, pos.Position, nil); err != nil {
			return err
		}
	}

	return nil
}

func (k Keeper) sufficientBonusPoolBalance(ctx context.Context, bonus sdk.Coins) error {
	if bonus.IsZero() {
		return nil
	}

	poolAddr := k.accountKeeper.GetModuleAddress(types.RewardsPoolName)
	poolBalance := k.bankKeeper.GetAllBalances(ctx, poolAddr)
	if !poolBalance.IsAllGTE(bonus) {
		return errorsmod.Wrapf(types.ErrInsufficientBonusPool,
			"bonus: %s, pool balance: %s",
			bonus.String(), poolBalance.String())
	}

	return nil

func (s *KeeperSuite) TestUpdateTier_BonusApyChange_InsufficientPool() {
	lockAmount := sdkmath.NewInt(sdk.DefaultPowerReduction.Int64())
	pos := s.setupNewTierPosition(lockAmount, false)
	valAddr := sdk.MustValAddressFromBech32(pos.Delegation.ValidatorAddress)

	s.setValidatorCommission(valAddr, sdkmath.LegacyZeroDec())

	// Advance time so bonus accrues.
	s.ctx = s.ctx.WithBlockHeight(s.ctx.BlockHeight() + 1)
	s.ctx = s.ctx.WithBlockTime(s.ctx.BlockTime().Add(30 * 24 * time.Hour))

	// Update tier with new BonusApy — should fail due to insufficient pool.
	tier := newTestTier(1)
	initialBonusApy := tier.BonusApy
	updatedBonusApy := sdkmath.LegacyNewDecWithPrec(8, 2)
	tier.BonusApy = updatedBonusApy
	msgServer := keeper.NewMsgServerImpl(s.keeper)
	_, err := msgServer.UpdateTier(s.ctx, &types.MsgUpdateTier{
		Authority: s.keeper.GetAuthority(),
		Tier:      tier,
	})
	s.Require().ErrorIs(err, types.ErrInsufficientBonusPool)

	// Tier should NOT have been updated (tx failed).
	got, err := s.keeper.GetTier(s.ctx, 1)
	s.Require().NoError(err)
	s.Require().True(initialBonusApy.Equal(got.BonusApy), "tier should still have old APY")
