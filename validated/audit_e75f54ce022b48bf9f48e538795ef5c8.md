### Title
`gas_prices_to_hash` Omits `next_l2_gas_price` and `l2_gas_consumed` from Block Hash, Allowing Proposer to Manipulate Fee Market Without Detection - (File: `crates/starknet_api/src/block_hash/block_hash_calculator.rs`)

### Summary

The `gas_prices_to_hash` function carries an explicit TODO acknowledging that `l2_gas_consumed` and `next_l2_gas_price` must be added to the block hash "after 0.14.0". The current production Starknet version is 0.14.4. Because neither field is committed to in the block hash, a proposer can write any value for `next_l2_gas_price` into the block header, validators cannot detect the manipulation through hash verification, and the manipulated price propagates as the gas price for the next block.

### Finding Description

`BlockHeaderWithoutHash` carries both `l2_gas_consumed` and `next_l2_gas_price`:

```rust
// crates/starknet_api/src/block.rs
pub struct BlockHeaderWithoutHash {
    ...
    pub l2_gas_consumed: GasAmount,
    pub next_l2_gas_price: GasPrice,
    ...
    // TODO(AndrewL): Add this field into the block hash.
    pub fee_proposal_fri: Option<GasPrice>,
}
```

`PartialBlockHashComponents`, the struct that feeds `calculate_block_hash`, does not include either field:

```rust
// crates/starknet_api/src/block_hash/block_hash_calculator.rs
pub struct PartialBlockHashComponents {
    pub header_commitments: BlockHeaderCommitments,
    pub block_number: BlockNumber,
    pub l1_gas_price: GasPricePerToken,
    pub l1_data_gas_price: GasPricePerToken,
    pub l2_gas_price: GasPricePerToken,   // current block price only
    pub sequencer: SequencerContractAddress,
    pub timestamp: BlockTimestamp,
    pub starknet_version: StarknetVersion,
    // next_l2_gas_price and l2_gas_consumed are absent
}
```

`gas_prices_to_hash`, called inside `calculate_block_hash`, explicitly defers these fields with a stale TODO:

```rust
// TODO(Ayelet): add l2_gas_consumed, next_l2_gas_price after 0.14.0.
pub fn gas_prices_to_hash(...) -> Vec<Felt> {
    if block_hash_version >= &BlockHashVersion::V0_13_4 {
        vec![HashChain::new()
            .chain(&STARKNET_GAS_PRICES0)
            .chain(&l1_gas_price.price_in_wei.0.into())
            .chain(&l1_gas_price.price_in_fri.0.into())
            .chain(&l1_data_gas_price.price_in_wei.0.into())
            .chain(&l1_data_gas_price.price_in_fri.0.into())
            .chain(&l2_gas_price.price_in_wei.0.into())
            .chain(&l2_gas_price.price_in_fri.0.into())
            .get_poseidon_hash()]
        // next_l2_gas_price and l2_gas_consumed never chained
    }
    ...
}
```

`BlockHashInput::to_final_block_hash_components` in the CLI also silently drops both fields when constructing `PartialBlockHashComponents` from a full `BlockHeaderWithoutHash`:

```rust
// crates/starknet_committer_and_os_cli/src/committer_cli/block_hash.rs

```

The `PartialBlockHash` used as the consensus commitment is derived from `PartialBlockHashComponents` via the same `calculate_block_hash` path, so the consensus commitment also does not bind `next_l2_gas_price`.

The proposer computes `next_l2_gas_price` via `calculate_next_l2_gas_price_for_fin` / `calculate_next_base_gas_price` and writes it into the block header. Because this value is not in the hash, a proposer can substitute any value. Validators verify the `PartialBlockHash` commitment but have no hash-level evidence of the correct `next_l2_gas_price`. The stored value then becomes the `l2_gas_price` for the next block.

### Impact Explanation

`next_l2_gas_price` stored in the committed block header is the direct input to the next block's L2 gas price. A proposer who sets it to 1 FRI causes the next block's gas price to be 1 FRI regardless of actual network congestion. Users can then submit transactions at that artificially low price. The fee market's EIP-1559 adjustment (`calculate_next_base_gas_price`) is bypassed entirely for one block, and because the manipulated value seeds the next iteration, the effect can persist across multiple blocks until the price recovers through the gradual `MIN_GAS_PRICE_INCREASE_DENOMINATOR` ramp. This is a wrong state value (incorrect `next_l2_gas_price` in the committed block header) with direct economic impact on gas fees.

### Likelihood Explanation

In the BFT consensus model implemented by `apollo_consensus`, any validator can be elected proposer. The manipulation requires only that the proposer write a non-canonical `next_l2_gas_price` into the block header before broadcasting. No special privilege beyond being the current round's proposer is needed. The TODO comment confirms the gap is known and unresolved at version 0.14.4.

### Recommendation

Add `l2_gas_consumed` and `next_l2_gas_price` to `gas_prices_to_hash` behind the appropriate `BlockHashVersion` gate (e.g., `>= V0_14_0` or a new `V0_14_X` variant), and add the corresponding fields to `PartialBlockHashComponents`. The `BlockHashInput::to_final_block_hash_components` conversion must also forward these fields. The existing test `l2_gas_price_pre_v0_13_4` already demonstrates the pattern for version-gated field inclusion and should be extended to cover the new fields.

### Proof of Concept

1. Proposer builds a block normally, obtaining `l2_gas_used` from the executor.
2. Instead of calling `calculate_next_l2_gas_price_for_fin(current_price, height, l2_gas_used, ...)`, the proposer sets `next_l2_gas_price = GasPrice(1)` in the block header.
3. `PartialBlockHashComponents::new(&block_info, header_commitments)` is called — `block_info` does not carry `next_l2_gas_price`, so the partial hash is identical to the honest value.
4. `PartialBlockHash::from_partial_block_hash_components` produces the same commitment as an honest proposer would.
5. Validators receive the `PartialBlockHash` commitment, verify it matches, and vote to commit the block.
6. The committed `BlockHeaderWithoutHash` contains `next_l2_gas_price = 1`.
7. At the next height, `set_height_and_round` reads `next_l2_gas_price` from the synced block header and sets `context.l2_gas_price = GasPrice(1)`, which is then broadcast in `ProposalInit` for the next block. [1](#0-0) [2](#0-1) [3](#0-2) [4](#0-3) [5](#0-4) [6](#0-5)

```rust
/// Hash of [`PartialBlockHashComponents`] only (no state root or parent hash).
#[derive(Clone, Copy, Debug, Default, PartialEq, Eq, Hash, Serialize, Deserialize)]
pub struct PartialBlockHash(pub StarkHash);

impl PartialBlockHash {
    // TODO(Ariel): Use parent_partial_block_hash instead of zero.
    const GLOBAL_ROOT_FOR_PARTIAL_BLOCK_HASH: GlobalRoot = GlobalRoot(Felt::ZERO);
    const PARENT_HASH_FOR_PARTIAL_BLOCK_HASH: BlockHash = BlockHash(Felt::ZERO);

    /// Hash of [`PartialBlockHashComponents`].
    /// Uses the same formula as [`calculate_block_hash`] with the fixed constants above for the
    /// state root and parent hash.
    pub fn from_partial_block_hash_components(
        partial_block_hash_components: &PartialBlockHashComponents,
    ) -> StarknetApiResult<Self> {
        let block_hash = calculate_block_hash(
            partial_block_hash_components,
            Self::GLOBAL_ROOT_FOR_PARTIAL_BLOCK_HASH,
            Self::PARENT_HASH_FOR_PARTIAL_BLOCK_HASH,
        )?;
        Ok(Self(block_hash.0))
    }
```

```rust
pub fn calculate_block_hash(
    partial_block_hash_components: &PartialBlockHashComponents,
    state_root: GlobalRoot,
    previous_block_hash: BlockHash,
) -> StarknetApiResult<BlockHash> {
    let block_hash_version: BlockHashVersion =
        partial_block_hash_components.starknet_version.try_into()?;
    let block_commitments = &partial_block_hash_components.header_commitments;
    Ok(BlockHash(
        HashChain::new()
            .chain(&block_hash_version.clone().into())
            .chain(&partial_block_hash_components.block_number.0.into())
            .chain(&state_root.0)
            .chain(&partial_block_hash_components.sequencer.0)
            .chain(&partial_block_hash_components.timestamp.0.into())
            .chain(&block_commitments.concatenated_counts)
            .chain(&block_commitments.state_diff_commitment.0.0)
            .chain(&block_commitments.transaction_commitment.0)
            .chain(&block_commitments.event_commitment.0)
            .chain(&block_commitments.receipt_commitment.0)
            .chain_iter(
                gas_prices_to_hash(
                    &partial_block_hash_components.l1_gas_price,
                    &partial_block_hash_components.l1_data_gas_price,
                    &partial_block_hash_components.l2_gas_price,
                    &block_hash_version,
                )
                .iter(),
            )
            .chain(
                &Felt::try_from(&partial_block_hash_components.starknet_version)
                    .expect("Expect ASCII version"),
            )
            .chain(&Felt::ZERO)
            .chain(&previous_block_hash.0)
            .get_poseidon_hash(),
    ))
}
```
