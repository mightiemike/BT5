### Title
Silent `ValidResourceBounds` Variant Downgrade in Protobuf Deserialization Produces Divergent Transaction Hash - (File: `crates/apollo_protobuf/src/converters/transaction.rs`)

### Summary

The protobuf deserializer for `ValidResourceBounds` silently downgrades `AllResources(l2_gas=0, l1_data_gas=0)` to `L1Gas` using a zero-value heuristic. Because `get_tip_resource_bounds_hash` produces structurally different hash preimages for these two variants (2 vs 3 resource felts), a V3 transaction submitted via RPC with `AllResources` and zero L2/L1-data-gas bounds receives hash H1 at admission, but any component that recomputes the hash from the deserialized `InvokeTransactionV3` (P2P sync, central sync `validate_transaction_hash`, RPC trace/simulation) computes H2 ≠ H1.

### Finding Description

**Root cause — lossy protobuf round-trip:**

`TryFrom<protobuf::ResourceBounds> for ValidResourceBounds` in `crates/apollo_protobuf/src/converters/transaction.rs` applies a heuristic:

```rust

...
Ok(if l1_data_gas.is_zero() && l2_gas.is_zero() {
    ValidResourceBounds::L1Gas(l1_gas)          // ← silently drops AllResources

```rust

``` [2](#0-1) 

So `AllResources(l1_gas=X, l2_gas=0, l1_data_gas=0)` → serialise → deserialise → `L1Gas(X)`. The variant is silently changed.

        let Some(l1_gas) = value.l1_gas else {
            return Err(missing("ResourceBounds::l1_gas"));

        let Some(l2_gas) = value.l2_gas else {

        let l2_gas: ResourceBounds = l2_gas.try_into()?;
