### Title
`DepositAllowlistExtension.beforeAddLiquidity` checks `owner` instead of `sender`, allowing any address to bypass the deposit allowlist — (`File: metric-periphery/contracts/extensions/DepositAllowlistExtension.sol`)

### Summary
`DepositAllowlistExtension` is documented as gating `addLiquidity` by *depositor* address. It checks the `owner` parameter (LP-share recipient) while silently discarding `sender` (the actual caller who pays tokens). Because `MetricOmmPool.addLiquidity` lets any caller supply any `owner`, an unauthorized address can bypass the guard by nominating an allowlisted address as `owner`.

### Finding Description
`ExtensionCalling._beforeAddLiquidity` encodes the call as: [4](#0-3) 

So `sender` = `msg.sender` of `pool.addLiquidity()` (the token payer); `owner` = the caller-supplied beneficiary. The extension ignores `sender` entirely: [1](#0-0) 

`allowedDepositor[msg.sender][owner]` passes whenever `owner` is allowlisted, regardless of who is actually calling and paying.

### Impact Explanation
- **Admin-boundary break**: the pool admin's deposit allowlist is rendered ineffective. Any unprivileged address can add liquidity to a permissioned pool by routing through an allowlisted `owner`.
- **Pool-state manipulation**: unauthorized addresses can alter bin balances (token0/token1 per bin), shifting per-share value metrics consumed by `OracleValueStopLossExtension` and affecting stop-loss watermarks.
- **Unsolicited LP positions**: authorized addresses receive LP shares without consent; while removable, this constitutes forced state changes on third-party accounts.

### Likelihood Explanation
Trivially reachable: no special role, no flash-loan, no multi-step setup. Any EOA or contract calls `pool.addLiquidity(allowlisted_address, ...)` directly. The bypass requires only knowledge of one allowlisted address (readable from `allowedDepositor` mapping).

### Recommendation
Check `sender` (the actual depositor) instead of `owner` (the share recipient), mirroring `SwapAllowlistExtension`:

```solidity
function beforeAddLiquidity(address sender, address owner, uint80, LiquidityDelta calldata, bytes calldata)
    external
    view
    override
    returns (bytes4)
{
    if (!allowAllDepositors[msg.sender] && !allowedDepositor[msg.sender][sender]) {
        revert IMetricOmmPoolActions.NotAllowedToDeposit();
    }
    return IMetricOmmExtensions.beforeAddLiquidity.selector;
}
```
