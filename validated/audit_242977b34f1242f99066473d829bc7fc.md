### Title
`_updateBalanceNormalized()` Uses Truncate-Toward-Zero Division When Storing Negative (Borrow) Balances, Understating User Debt - (File: `core/contracts/SpotEngineState.sol`)

---

### Summary

`_updateBalanceNormalized()` in `SpotEngineState.sol` normalizes a negative `newAmount` (a borrow) by calling `newAmount.div(cumulativeMultiplierX18)`. The `MathSD21x18.div` function performs integer division that truncates toward zero. For a negative dividend, this produces a result that is less negative than the true quotient — the stored normalized debt is smaller in absolute value than it should be. The protocol therefore records less debt than the user actually owes, causing a systematic loss of funds to depositors and the protocol.

---

### Finding Description

In `_updateBalanceNormalized`, after computing the new actual balance `newAmount`, the function normalizes it for storage:

```solidity
balance.amountNormalized = newAmount.div(cumulativeMultiplierX18);
``` [1](#0-0) 

`MathSD21x18.div` is implemented as:

```solidity
int256 result = (int256(x) * ONE_X18) / y;
``` [2](#0-1) 

Solidity's `/` operator truncates toward zero. When `newAmount` is negative (a borrow) and `cumulativeMultiplierX18` is positive (always `>= 1e18`), the result is negative but closer to zero than the true value. Concretely:

- True value: `(-14e18) / 4e18 = -3.5` → correct normalized debt = `-3.5e18`
- Truncated result: `(-14e18 * 1e18) / 4e18 = -3e18` (rounds toward zero, i.e., `-3` instead of `-4`)

The stored `balance.amountNormalized` is therefore less negative than it should be. This also corrupts `totalBorrowsNormalized`:

```solidity
state.totalBorrowsNormalized -= balance.amountNormalized;
``` [3](#0-2) 

Since `balance.amountNormalized` is less negative than it should be, `-balance.amountNormalized` is smaller, so `totalBorrowsNormalized` is understated. When the balance is later read back via `balanceNormalizedToBalance`, the `mul` call also truncates toward zero for negative values, compounding the understatement:

```solidity
return Balance(balance.amountNormalized.mul(cumulativeMultiplierX18));
``` [4](#0-3) 

```solidity
int256 result = (int256(x) * y) / ONE_X18;
``` [5](#0-4) 

---

### Impact Explanation

Every time a user's borrow position is written to storage, the normalized debt is truncated toward zero. The user's actual debt as seen by the protocol is less than what they truly owe. Across all borrowers and all balance updates, `totalBorrowsNormalized` is systematically understated. This means:

1. Depositors earn less interest than they are owed (interest is computed from `totalBorrows`).
2. Health checks understate a borrower's liability, allowing users to borrow slightly more than they should.
3. The dust loss is per-operation and accumulates with every borrow-side balance update across all users and products. [6](#0-5) 

---

### Likelihood Explanation

This is triggered by any user who holds a negative spot balance (i.e., any borrower). `_updateBalanceNormalized` is called from `updateBalance()` in `SpotEngine.sol` on every trade or deposit/withdrawal that touches a borrow position.

```solidity
_updateBalanceNormalized(state, balance, amountDelta);
_updateBalanceNormalized(quoteState, quoteBalance, quoteDelta);
``` [7](#0-6) 

No special privileges are required. Any trader whose quote balance goes negative (a standard borrow in spot margin trading) triggers this path on every balance update. The likelihood is **high** — it fires on every borrow-side balance mutation.

---

### Recommendation

When `newAmount` is negative, use ceiling division (round away from zero) so the stored normalized debt is at least as large as the true value, keeping rounding in favor of the protocol:

```diff
- balance.amountNormalized = newAmount.div(cumulativeMultiplierX18);
+ if (newAmount < 0) {
+     // Round away from zero for debt: ceiling of a negative number
+     int256 raw = int256(newAmount) * MathSD21x18.ONE_X18;
+     int256 divisor = int256(cumulativeMultiplierX18);
+     balance.amountNormalized = int128((raw - (divisor - 1)) / divisor);
+ } else {
+     balance.amountNormalized = newAmount.div(cumulativeMultiplierX18);
+ }
```

Alternatively, add a `divUp` (ceiling for negative, i.e., round away from zero) variant to `MathSD21x18` and use it consistently wherever a negative normalized balance is stored. [2](#0-1) 

---

### Proof of Concept

Suppose `cumulativeMultiplierX18 = 1.5e18` (50% interest accrued) and a user borrows such that `newAmount = -10e18` (10 units of debt in actual terms).

**Correct normalized debt** (exact): `-10e18 / 1.5 = -6.666...e18` → should store `-7e18` (round away from zero, i.e., ceiling toward more negative).

**Actual stored value** via `MathSD21x18.div`:
```
result = (-10e18 * 1e18) / 1.5e18 = -10e36 / 1.5e18 = -6.666...e18 → truncated to -6e18
```

The stored normalized debt is `-6e18` instead of `-7e18`. When converted back:
```
actual debt = -6e18 * 1.5e18 / 1e18 = -9e18
```

The user's debt reads as `9` units instead of `10` — the protocol has silently forgiven 1 unit of debt. Multiplied across thousands of borrow updates, this constitutes a material, accumulating loss to the protocol. [8](#0-7)

    function _updateBalanceNormalized(
        State memory state,
        BalanceNormalized memory balance,
        int128 balanceDelta
    ) internal pure {
        if (balance.amountNormalized > 0) {
            state.totalDepositsNormalized -= balance.amountNormalized;
        } else {
            state.totalBorrowsNormalized += balance.amountNormalized;
        }

        int128 cumulativeMultiplierX18;
        if (balance.amountNormalized > 0) {
            cumulativeMultiplierX18 = state.cumulativeDepositsMultiplierX18;
        } else {
            cumulativeMultiplierX18 = state.cumulativeBorrowsMultiplierX18;
        }

        int128 newAmount = balance.amountNormalized.mul(
            cumulativeMultiplierX18
        ) + balanceDelta;

        if (newAmount > 0) {
            cumulativeMultiplierX18 = state.cumulativeDepositsMultiplierX18;
        } else {
            cumulativeMultiplierX18 = state.cumulativeBorrowsMultiplierX18;
        }

        balance.amountNormalized = newAmount.div(cumulativeMultiplierX18);

        if (balance.amountNormalized > 0) {
            state.totalDepositsNormalized += balance.amountNormalized;
        } else {
            state.totalBorrowsNormalized -= balance.amountNormalized;
        }
    }
