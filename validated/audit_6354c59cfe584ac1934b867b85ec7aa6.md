### Title
Unprivileged Caller Can Trigger Premature Mainchain GC, Corrupting SPV Proof State - (File: `contract/src/lib.rs`)

---

### Summary

`run_mainchain_gc` is a public, state-mutating NEAR contract method that carries no caller authorization check. The only decorator it bears is `#[pause(except(roles(Role::UnrestrictedRunGC)))]`, which — per the `near-plugins` `Pausable` trait — only controls whether the function may be called while the contract is **paused**. When the contract is running normally (unpaused), any unprivileged NEAR account can invoke `run_mainchain_gc(batch_size)` with an arbitrary `batch_size`, causing the contract to permanently delete the oldest mainchain block headers from on-chain storage and advance `mainchain_initial_blockhash` forward.

---

### Finding Description

`run_mainchain_gc` is declared `pub` inside the `#[near]` impl block and is therefore a callable NEAR contract method. [1](#0-0) 

The `#[pause(except(roles(Role::UnrestrictedRunGC)))]` attribute is the only guard on the function. The `near-plugins` `Pausable` macro uses this attribute exclusively to decide whether to reject calls **when the contract is paused**. The `UnrestrictedRunGC` role is documented as:

> "Allows to use `run_mainchain_gc` API on a paused contract" [2](#0-1) 

This confirms the role is a pause-bypass, not a general caller restriction. When the contract is not paused, there is no `predecessor_account_id` check, no role check, and no `#[private]` guard. Any NEAR account — with zero stake, no role, and no relationship to the protocol — can call this function.

Inside the function, the following irreversible state mutations occur: [3](#0-2) 

For each height in the removal range, `remove_block_header` deletes the entry from both `mainchain_header_to_height` and `headers_pool`, and `mainchain_height_to_header.remove` deletes the height-to-hash mapping. Finally, `mainchain_initial_blockhash` is advanced to the new oldest block. These deletions are permanent — there is no undo path.

---

### Impact Explanation

The deleted block headers are the inputs to the contract's core SPV service. `verify_transaction_inclusion` and `verify_transaction_inclusion_v2` both look up the target block in `mainchain_header_to_height`: [4](#0-3) 

If the block has been GC'd, this lookup panics with `"block does not belong to the current main chain"`. Any downstream consumer — a bridge, a cross-chain protocol, or an SPV wallet — that calls `verify_transaction_inclusion` for a transaction in a GC'd block receives a hard panic instead of a valid proof result. This is a false negative: a legitimately confirmed Bitcoin transaction is reported as unverifiable.

Additionally, the CLAUDE.md explicitly documents the reorg breakage:

> "If mainchain blocks near the fork point have been garbage collected, reorg will fail — the contract panics with `PrevBlockNotFound` when it cannot walk the chain back to the common ancestor." [5](#0-4) 

An attacker who triggers GC just before a legitimate relayer submits a competing fork can permanently prevent that fork from being promoted, locking the contract onto a stale or attacker-favored chain tip.

---

### Likelihood Explanation

The attack requires only a standard NEAR account and a single function call with no deposit. The function is publicly listed in the contract ABI. The attacker needs no privileged role, no staked funds, and no knowledge beyond the contract's public interface. The only precondition is that the mainchain must have grown beyond `gc_threshold` blocks, which is the normal operating state of a live deployment. Likelihood is **high**.

---

### Recommendation

Add a caller authorization check to `run_mainchain_gc` that restricts invocation to accounts holding an admin-level role (e.g., `Role::DAO` or a new dedicated `GCManager` role). Using `near-plugins`'s `#[access_control_any(roles(Role::DAO, Role::RelayerManager))]` attribute, or an explicit `require!(self.acl_has_role(...))` guard at the top of the function body, would close the gap. The automatic GC path inside `submit_blocks` (which is already gated by `#[trusted_relayer]`) is unaffected and should remain as-is.

---

### Proof of Concept

1. Deploy the contract with `gc_threshold = 10` and submit 25 mainchain headers via a trusted relayer account. The chain now holds 25 headers, 15 above threshold.
2. From a fresh, unprivileged NEAR account (no roles granted), call:
   ```
   run_mainchain_gc({"batch_size": 18446744073709551615})
   ```
3. The contract removes 15 headers (the maximum allowed by `total_amount_to_remove`), advancing `mainchain_initial_blockhash` to block 15.
4. Now call `verify_transaction_inclusion` with a valid Merkle proof for a transaction in block 5. The call panics: `"block does not belong to the current main chain"` — because block 5's hash was deleted from `mainchain_header_to_height` at step 3.
5. The same unprivileged account can repeat step 2 after each new batch of relayer submissions, continuously racing to prune the oldest blocks and deny SPV proofs for any block older than the current `gc_threshold` window. [6](#0-5)

```rust
    #[pause(except(roles(Role::UnrestrictedRunGC)))]
    pub fn run_mainchain_gc(&mut self, batch_size: u64) {
        let initial_blockheader = self
            .headers_pool
            .get(&self.mainchain_initial_blockhash)
            .unwrap_or_else(|| env::panic_str(ERR_KEY_NOT_EXIST));

        let tip_blockheader = self
            .headers_pool
            .get(&self.mainchain_tip_blockhash)
            .unwrap_or_else(|| env::panic_str(ERR_KEY_NOT_EXIST));

        let amount_of_headers_we_store =
            tip_blockheader.block_height - initial_blockheader.block_height + 1;

        if amount_of_headers_we_store > self.gc_threshold {
            let total_amount_to_remove = amount_of_headers_we_store - self.gc_threshold;
            let selected_amount_to_remove = std::cmp::min(total_amount_to_remove, batch_size);

            let start_removal_height = initial_blockheader.block_height;
            let end_removal_height = initial_blockheader.block_height + selected_amount_to_remove;
            env::log_str(&format!(
                "Num of blocks to remove {selected_amount_to_remove}"
            ));

            for height in start_removal_height..end_removal_height {
                let blockhash = &self
                    .mainchain_height_to_header
                    .get(&height)
                    .unwrap_or_else(|| env::panic_str(ERR_KEY_NOT_EXIST));

                self.remove_block_header(blockhash);
                self.mainchain_height_to_header.remove(&height);
            }

            self.mainchain_initial_blockhash = self
                .mainchain_height_to_header
                .get(&end_removal_height)
                .unwrap_or_else(|| env::panic_str(ERR_KEY_NOT_EXIST));
        }
    }
```
