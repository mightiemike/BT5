### Title
Unconditional Trust of `_cached_sha256_treehash` Enables Hash-Content Substitution in Python API — (`wheel/python/clvm_rs/tree_hash.py`)

---

### Summary

`Treehasher.sha256_treehash` in `wheel/python/clvm_rs/tree_hash.py` unconditionally returns `_cached_sha256_treehash` from any caller-supplied Python object without verifying it matches the object's actual content. `Program.wrap()` in `wheel/python/clvm_rs/program.py` blindly copies this attribute from the wrapped object, and `Program.tree_hash()` returns it directly if non-`None`. An attacker who controls a Python object implementing the `CLVMStorage` interface can pre-set `_cached_sha256_treehash` to an arbitrary value — the hash of a different, legitimate program — while the object's actual `atom`/`pair` content is malicious. Every downstream caller that relies on `tree_hash()` for identity or authorization (puzzle-hash verification, `Program.__eq__`, `curry_hash`) then operates on a forged commitment.

---

### Finding Description

**Root cause — `tree_hash.py` lines 57–60:**

```python
r = getattr(obj, "_cached_sha256_treehash", None)
if r is not None:
    self.cache_hits += 1
    hash_stack.append(r)
    return          # ← content is never inspected
```

`sha256_treehash` short-circuits the entire traversal the moment `_cached_sha256_treehash` is present. No check is made that the cached value is consistent with `obj.atom` or `obj.pair`. The attribute is a pure performance hint, but it is treated as an authoritative commitment.

**Propagation — `program.py` line 142:**

```python
o._cached_sha256_treehash = getattr(v, "_cached_sha256_treehash", None)
```

`Program.wrap()` copies the attribute verbatim from any `CLVMStorage`-compatible object `v`. If `v` carries a forged `_cached_sha256_treehash`, the resulting `Program` inherits it.

**Consumption — `program.py` lines 284–286:**

```python
if self._cached_sha256_treehash is None:
    self._cached_sha256_treehash = sha256_treehash(self._unwrapped)
return self._cached_sha256_treehash
```

Because the cached value is non-`None`, `sha256_treehash` is never called; the forged hash is returned directly.

**Analog to the zkSync finding:**
In the zkSync bug, `proposeDiamondCut` commits to `facetCuts + _initAddress` but deliberately omits `_calldata`, letting the governor change it later. An attacker front-runs the execution with malicious `_calldata` that was never part of the commitment. Here, `_cached_sha256_treehash` is the "committed" field that callers trust, while the actual `atom`/`pair` content is the "uncommitted" field that can be freely substituted. The cache is the proposal; the content is the calldata.

---

### Impact Explanation

`Program.tree_hash()` is the canonical puzzle-hash function in the Chia Python stack. It is used for:

- **Puzzle-hash verification** — wallets and coin-spend validators compare `puzzle.tree_hash()` against the on-chain puzzle hash to confirm they are spending the right coin.
- **Equality** — `Program.__eq__` delegates entirely to `tree_hash()` comparison (line 188).
- **`curry_hash`** — computes a puzzle hash from sub-tree hashes, all of which flow through `tree_hash()`.

If an attacker can make `tree_hash()` return the hash of a legitimate puzzle while the program's actual content is malicious, any check of the form `puzzle.tree_hash() == expected_hash` passes, and the malicious program is accepted as the legitimate one. This is a direct puzzle-hash spoofing primitive at the Python API layer.

---

### Likelihood Explanation

`Program.wrap()` is a public classmethod. Any code that accepts an arbitrary Python object implementing `CLVMStorage` (i.e., any object with `.atom` and `.pair` attributes) and passes it to `Program.wrap()` or directly to `sha256_treehash()` is vulnerable. The `CLVMStorage` interface is duck-typed; no registration or subclassing is required. An attacker constructs:

```python
class Forgery:
    atom = b"<malicious CLVM>"
    pair = None
    _cached_sha256_treehash = <hash of legitimate puzzle>
```

and passes it to `Program.wrap(Forgery())` or `sha256_treehash(Forgery())`. Both are reachable from the public Python API without any special privilege.

---

### Recommendation

Remove the unconditional cache short-circuit from `sha256_treehash`. The cache should only be trusted when it was set by the same hasher instance on the same object, not when it arrives from an external caller. One safe approach: strip `_cached_sha256_treehash` from objects that arrive via `Program.wrap()` unless they are already `Program` instances (whose cache was set by trusted code paths), and always recompute for externally-supplied `CLVMStorage` objects.

Alternatively, scope the cache to a `WeakKeyDictionary` owned by the `Treehasher` instance rather than storing it as an attribute on arbitrary Python objects, eliminating the injection surface entirely.

---

### Proof of Concept

```python
from wheel.python.clvm_rs.tree_hash import sha256_treehash, shatree_atom
from wheel.python.clvm_rs.program import Program

# Hash of the legitimate program b"\x01" (the atom `1`)
legitimate_hash = shatree_atom(b"\x01")

# Forgery: content is b"\xff\x01\x80" (a malicious pair), but hash claims to be atom `1`
class Forgery:
    atom = None
    pair = None  # will be resolved lazily; sha256_treehash never reaches it
    _cached_sha256_treehash = legitimate_hash

forgery = Forgery()

# sha256_treehash returns the forged hash without inspecting content
returned = sha256_treehash(forgery)
assert returned == legitimate_hash  # passes — content was never read

# Program.wrap propagates the forged hash
p = Program.wrap(forgery)
assert p.tree_hash() == legitimate_hash  # passes — malicious content, legitimate hash
