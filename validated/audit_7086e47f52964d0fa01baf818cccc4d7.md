### Title
Quote Token (USDC/USDT) Price Hardcoded to $1 Causes Health Overvaluation During De-Peg — (`core/contracts/SpotEngine.sol`, `core/contracts/BaseEngine.sol`)

---

### Summary

The Nado protocol hardcodes the QUOTE token (USDC/USDT) price to exactly `$1.00` (`priceX18: ONE`) at initialization and never updates it via any oracle. The health calculation in `BaseEngine._calculateProductHealth` uses this hardcoded price to value USDC/USDT collateral. During a USDC/USDT de-peg, every subaccount's USDC collateral is overvalued, allowing users to withdraw non-quote assets that should be locked, or to avoid liquidation when they are genuinely insolvent.

---

### Finding Description

In `SpotEngine.initialize`, the QUOTE product's risk store is set with a hardcoded price of `ONE` (i.e., `1e18`, representing $1.00):

```solidity
_risk().value[QUOTE_PRODUCT_ID] = RiskHelper.RiskStore({
    longWeightInitial: 1e9,
    shortWeightInitial: 1e9,
    longWeightMaintenance: 1e9,
    shortWeightMaintenance: 1e9,
    priceX18: ONE   // <-- hardcoded $1, never updated by oracle
});
``` [1](#0-0) 

This price is never overwritten by any on-chain oracle feed. The health calculation in `BaseEngine._calculateProductHealth` then uses `risk.priceX18` to value every product, including the quote token:

```solidity
health += quoteAmount;
if (amount != 0) {
    health += amount.mul(weight).mul(risk.priceX18);
}
``` [2](#0-1) 

For the QUOTE product, `SpotEngineState._getBalance` returns `(balance.amount, 0)`, so the USDC balance flows through the `amount` path and is multiplied by `risk.priceX18 = ONE`. The result is that 1 USDC is always valued at exactly $1 in every health check, regardless of the real market price of USDC.

The same assumption is explicit in `Clearinghouse.checkMinDeposit`, which explicitly skips the oracle call for the quote product and substitutes `ONE`:

```solidity
int128 priceX18 = ONE;
if (productId != QUOTE_PRODUCT_ID) {
    priceX18 = _getPriceX18(productId);
}
``` [3](#0-2) 

---

### Impact Explanation

During a USDC de-peg (e.g., USDC trades at $0.70, as occurred during the March 2023 SVB crisis):

- A user holding 1,000 USDC collateral has real USD value of $700, but the protocol computes health as if it is $1,000.
- The user can submit a slow-mode `WithdrawCollateral` for non-quote assets (e.g., ETH, BTC). After the 3-day delay, `clearinghouse.withdrawCollateral` executes a health check that still values USDC at $1.
- The health check passes even though the user is genuinely insolvent at real market prices.
- The user extracts non-quote collateral that should be locked, leaving the protocol with under-collateralized positions.
- Simultaneously, accounts that should be liquidated remain above the maintenance health threshold, preventing liquidators from acting and accumulating bad debt.

The corrupted state delta is: non-quote asset balances are drained from subaccounts that are insolvent at real prices, and the insurance fund absorbs the resulting bad debt.

---

### Likelihood Explanation

USDC has de-pegged materially in the past (to ~$0.87 in March 2023). The slow-mode withdrawal path (`submitSlowModeTransaction`) is permissionless — any user can submit it by calling `Endpoint.submitSlowModeTransaction` directly on-chain. [4](#0-3) 

No privileged access, leaked keys, or social engineering is required. The attacker only needs to hold USDC collateral and have non-quote assets in their subaccount when a de-peg occurs.

---

### Recommendation

Replace the hardcoded `priceX18: ONE` for the QUOTE product with a live oracle feed (e.g., a Chainlink or Pyth USDC/USD price feed). The health calculation should use the real market price of the quote token, not a fixed $1 assumption. Alternatively, apply a conservative haircut to USDC/USDT collateral that accounts for de-peg risk, consistent with how other collateral assets are weighted.

---

### Proof of Concept

1. USDC de-pegs to $0.70 (real market price).
2. Attacker subaccount holds: 1,000 USDC collateral + short 800 USDC worth of ETH (borrowed).
3. Real net health = $700 − $800 = **−$100** (insolvent).
4. Protocol health = $1,000 − $800 = **+$200** (appears solvent, because `priceX18 = ONE` for USDC).
5. Attacker calls `Endpoint.submitSlowModeTransaction` with a `WithdrawCollateral` transaction for their ETH position.
6. After 3 days, the sequencer executes the slow-mode transaction via `processSlowModeTransactionImpl` → `clearinghouse.withdrawCollateral`.
7. The health check inside `withdrawCollateral` calls `getHealth`, which calls `_calculateProductHealth` with `risk.priceX18 = ONE` for USDC — the check passes.
8. Attacker receives ETH worth $800 while leaving only $700 of real USDC collateral, creating $100 of bad debt absorbed by the insurance fund. [1](#0-0) [5](#0-4) [6](#0-5)

        IProductEngine.HealthType healthType

            productId,
            subaccount
        );

        health += quoteAmount;

        if (amount != 0) {
            if (weight == 2 * ONE) {
                return -INF;
            }
            health += amount.mul(weight).mul(risk.priceX18);
            emit PriceQuery(productId);
        }
    }

        IEndpoint.TransactionType txType = IEndpoint.TransactionType(
            uint8(transaction[0])
        );

        // special case for DepositCollateral because upon
        // slow mode submission we must take custody of the
        // actual funds

        address sender = msg.sender;

        if (txType == IEndpoint.TransactionType.DepositCollateral) {
            revert();
        } else if (txType == IEndpoint.TransactionType.DepositInsurance) {
            IEndpoint.DepositInsurance memory txn = abi.decode(
                transaction[1:],
                (IEndpoint.DepositInsurance)
            );
            require(
                txn.amount >= uint128(SLOW_MODE_FEE),
                ERR_DEPOSIT_TOO_SMALL
            );
            handleDepositTransfer(_getQuote(), sender, uint256(txn.amount));
        } else if (
            txType == IEndpoint.TransactionType.WithdrawInsurance ||
            txType == IEndpoint.TransactionType.DelistProduct ||
            txType == IEndpoint.TransactionType.DumpFees ||
            txType == IEndpoint.TransactionType.RebalanceXWithdraw ||
            txType == IEndpoint.TransactionType.UpdateTierFeeRates ||
            txType == IEndpoint.TransactionType.AddNlpPool ||
            txType == IEndpoint.TransactionType.UpdateNlpPool ||
            txType == IEndpoint.TransactionType.DeleteNlpPool ||
            txType == IEndpoint.TransactionType.ForceRebalanceNlpPool ||
            txType == IEndpoint.TransactionType.NlpProfitShare ||
            txType == IEndpoint.TransactionType.UpdateBuilder

            require(sender == owner());
        } else {
            chargeSlowModeFee(_getQuote(), sender);
            slowModeFees += SLOW_MODE_FEE;

        IEndpoint.SlowModeConfig memory _slowModeConfig = slowModeConfig;
        requireUnsanctioned(sender);
        slowModeTxs[_slowModeConfig.txCount++] = IEndpoint.SlowModeTx({

            tx: transaction

        slowModeConfig = _slowModeConfig;
