### Title
Zero-Margin Isolated Subaccount Creation Bypasses Collateral Invariant — (`core/contracts/OffchainExchange.sol`)

### Summary
`createIsolatedSubaccount` silently accepts a zero isolated margin (appendix bits [127:64] = 0), registers the isolated subaccount in `digestToSubaccount`, and skips the collateral transfer entirely. When `matchOrders` is subsequently called with the same order digest, the sender is redirected to the zero-balance isolated subaccount. Because `isHealthy` unconditionally returns `true` in the production contract with no override, the post-trade health check is a no-op, allowing a leveraged perp position to be opened against an isolated subaccount with zero collateral.

### Finding Description

**Step 1 — Zero-margin path in `createIsolatedSubaccount`:** [1](#0-0) 

```solidity
int128 margin = int128(_isolatedMargin(txn.order.appendix));
if (margin > 0) {
    digestToMargin[digest] = margin;
    spotEngine.updateBalance(QUOTE_PRODUCT_ID, txn.order.sender, -margin);
    spotEngine.updateBalance(QUOTE_PRODUCT_ID, newIsolatedSubaccount, margin);
}
```

When `_isolatedMargin(txn.order.appendix)` returns `0` (bits [127:64] all zero), the `if (margin > 0)` branch is skipped. No balance is transferred from the parent to the isolated subaccount. Critically, the function does **not** revert — it continues and sets `digestToSubaccount[digest] = newIsolatedSubaccount` unconditionally. [2](#0-1) 

**Step 2 — `matchOrders` redirects sender to the zero-balance isolated subaccount:** [3](#0-2) 

When `matchOrders` is called with the same order, `digestToSubaccount[ordersInfo.taker.digest] != bytes32(0)` is true, so `taker.order.sender` is replaced with the isolated subaccount (which holds zero QUOTE balance). The perp engine then applies `baseDelta` and `quoteDelta` to this zero-balance account.

**Step 3 — `isHealthy` is a no-op:** [4](#0-3) 

```solidity
function isHealthy(bytes32 /* subaccount */) internal view virtual returns (bool) {
    return true;
}
```

There is no override of `isHealthy` anywhere in the codebase (confirmed by grep across all `.sol` files). The post-trade health assertions at lines 826–827 are therefore trivially satisfied regardless of the isolated subaccount's actual health. [5](#0-4) 

### Impact Explanation

An isolated subaccount can hold an arbitrarily large leveraged perp position with zero collateral. Any adverse price movement results in a negative balance on the isolated subaccount that is unbacked. When the position is eventually closed or liquidated, the protocol's insurance fund absorbs the shortfall, directly breaking solvency. This matches the Critical scope: arithmetic/balance accounting that destroys value and breaks solvency.

### Likelihood Explanation

The path is fully reachable by any trader: craft an `Order` with appendix bits [127:64] = 0 and the isolated bit set, submit it through the Endpoint as a `CreateIsolatedSubaccount` transaction, then submit a `matchOrders` transaction using the same order digest. No privileged access, admin keys, or governance capture is required.

### Recommendation

1. In `createIsolatedSubaccount`, add an explicit minimum-margin guard:
   ```solidity
   require(margin > 0, "Isolated subaccount requires positive margin");
   ```
   This should appear before `digestToSubaccount[digest] = newIsolatedSubaccount` so that a zero-margin call does not register the subaccount at all.

2. Implement a real `isHealthy` override that calls `clearinghouse.getHealth(subaccount, HealthType.INITIAL) >= 0` for isolated subaccounts, so that post-trade health is actually enforced.

### Proof of Concept

```solidity
// 1. Craft appendix: isolated bit set, bits [127:64] = 0 (zero margin)
uint128 appendix = ISOLATED_BIT; // bits [127:64] = 0

IEndpoint.Order memory order = IEndpoint.Order({
    sender: parentSubaccount,
    priceX18: ...,
    amount: LARGE_AMOUNT,
    expiration: ...,
    nonce: ...,
    appendix: appendix
});

// 2. Submit CreateIsolatedSubaccount through Endpoint
// -> createIsolatedSubaccount called, margin == 0, no transfer, but
//    digestToSubaccount[digest] = isolatedSubaccount is set

// 3. Submit matchOrders with the same order as taker
// -> taker.order.sender replaced with isolatedSubaccount (zero QUOTE balance)
// -> perpEngine.updateBalance applies baseDelta + quoteDelta to zero-balance account
// -> isHealthy(isolatedSubaccount) returns true unconditionally

// 4. Assert
assert(spotEngine.getBalance(QUOTE_PRODUCT_ID, isolatedSubaccount).amount == 0);
// isolatedSubaccount now holds a leveraged perp position with zero collateral
```
