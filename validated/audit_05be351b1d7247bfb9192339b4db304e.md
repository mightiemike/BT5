### Title
SwapAllowlistExtension Checks Router Address Instead of Actual User, Enabling Full Allowlist Bypass via MetricOmmSimpleRouter — (`metric-periphery/contracts/extensions/SwapAllowlistExtension.sol`)

---

### Summary

`SwapAllowlistExtension.beforeSwap` gates swaps by checking the `sender` argument it receives from the pool. When a swap is routed through `MetricOmmSimpleRouter`, the pool's `msg.sender` is the router contract, so the pool forwards the router's address as `sender` to the extension. The extension then checks whether the **router** is allowlisted, never the actual end-user. If the pool admin allowlists the router (the only way to make router-based swaps work at all), every unpermissioned user can bypass the allowlist by routing through the router.

---

### Finding Description

**Step 1 — Pool forwards `msg.sender` as `sender` to the extension.**

In `MetricOmmPool.sol::swap`, the pool calls `_beforeSwap` with its own `msg.sender` as the first argument: [1](#0-0) 

`_beforeSwap` then ABI-encodes that value as the `sender` parameter and dispatches it to every configured extension: [2](#0-1) 

**Step 2 — When the router is the caller, `sender` is always the router.**

`MetricOmmSimpleRouter.exactInputSingle` (and every other swap entry-point) calls `pool.swap(...)` directly. The pool's `msg.sender` is therefore the router contract, not the end-user: [3](#0-2) 

The actual user's address is stored only in the router's transient callback context (`_getPayer()`) and is never forwarded to the pool or to any extension.

**Step 3 — The extension checks the router address, not the user.**

`SwapAllowlistExtension.beforeSwap` checks `allowedSwapper[msg.sender][sender]`, where `msg.sender` is the pool and `sender` is the value forwarded by the pool — the router address: [4](#0-3) 

The real user's identity is never consulted.

**The dilemma this creates (analogous to `totalTransferRequested` always being zero):**

| Router allowlist state | Effect |
|---|---|
| Router **not** allowlisted | Every router-based swap reverts for all users, including individually allowlisted ones — broken functionality |
| Router **allowlisted** | Every user, allowlisted or not, can bypass the gate by routing through the router — security bypass |

There is no configuration that simultaneously enables router-based swaps and enforces per-user allowlist restrictions.

---

### Impact Explanation

A pool deployer who configures `SwapAllowlistExtension` to restrict trading to a curated set of addresses (e.g., KYC'd counterparties, institutional LPs, or a private OTC desk) loses that restriction entirely for any user who calls through `MetricOmmSimpleRouter`. The allowlist guard — the only mechanism protecting the pool from unauthorized swappers — is rendered ineffective. Unauthorized swaps drain pool liquidity at oracle-anchored prices, directly harming LP principal.

---

### Likelihood Explanation

`MetricOmmSimpleRouter` is the primary user-facing entry point documented and deployed for the protocol. Any pool that (a) uses `SwapAllowlistExtension` and (b) needs to support router-based swaps must allowlist the router, which immediately opens the bypass to all users. The trigger requires no special privilege — any public address can call the router.

---

### Recommendation

The actual end-user address must reach the extension. Two sound approaches:

1. **Pass the originating user through `extensionData`**: The router encodes `msg.sender` (the real user) into `extensionData` before calling the pool. The extension decodes and checks that address. This requires a coordinated encoding convention.

2. **Add a `payer` / `originator` field to the swap call**: Extend the pool's `swap` signature or the extension callback signature to carry the economic originator separately from the EVM `msg.sender`, so the extension can gate on the correct identity regardless of intermediary.

---

### Proof of Concept

1. Deploy a pool with `SwapAllowlistExtension` attached as a `beforeSwap` hook.
2. Call `setAllowedToSwap(pool, router, true)` — required for any router swap to succeed.
3. As an address that is **not** individually allowlisted, call `MetricOmmSimpleRouter.exactInputSingle(...)` targeting that pool.
4. The pool passes `msg.sender = router` to `_beforeSwap`; the extension evaluates `allowedSwapper[pool][router] == true` and returns the success selector.
5. The swap executes at full oracle-anchored depth, bypassing the per-user allowlist entirely. [5](#0-4) [6](#0-5)
