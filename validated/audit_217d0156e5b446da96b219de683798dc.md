### Title
`ValidResourceBounds::AllResources(l1, 0, 0)` silently collapses to `L1Gas(l1)` in protobuf round-trip, producing a divergent transaction hash preimage across nodes - (`crates/apollo_protobuf/src/converters/transaction.rs`)

### Summary

The protobuf deserializer for `ValidResourceBounds` classifies any wire message with zero `l2_gas` and zero `l1_data_gas` as `ValidResourceBounds::L1Gas`, regardless of whether the originating node serialized it from `ValidResourceBounds::AllResources`. Because `get_tip_resource_bounds_hash` includes a third element (`L1_DATA_GAS_packed`) in the Poseidon preimage for `AllResources` but omits it for `L1Gas`, the two variants produce structurally different hashes even when all numeric values are identical. A transaction whose hash was computed with the `AllResources` three-element preimage at the gateway will be recomputed with the `L1Gas` two-element preimage at every peer that receives it over P2P, causing a permanent hash mismatch.

### Finding Description

**Step 1 – Hash domain split.**

`get_tip_resource_bounds_hash` in `crates/starknet_api/src/transaction_hash.rs` builds the Poseidon preimage differently for the two variants:

```
L1Gas(l1)          → Poseidon(tip, L1_GAS_packed, L2_GAS_packed(0))
AllResources(l1,0,0) → Poseidon(tip, L1_GAS_packed, L2_GAS_packed(0), L1_DATA_GAS_packed(0))
```

The extra `L1_DATA_GAS_packed(0)` element makes the two hashes distinct even though every numeric field is identical. [1](#0-0) 

**Step 2 – Gateway always produces `AllResources`.**

`RpcInvokeTransactionV3` carries `resource_bounds: AllResourceBounds` (never `ValidResourceBounds`). The gateway converts it to `InternalRpcInvokeTransactionV3`, which also holds `AllResourceBounds`, and calls `get_invoke_transaction_v3_hash` via the `AllResources` path. The resulting hash H₁ is stored in `InternalRpcTransaction.tx_hash`. [2](#0-1) [3](#0-2) 

**Step 3 – Protobuf serialization preserves all three zero fields.**

`From<ValidResourceBounds> for protobuf::ResourceBounds` serializes `AllResources(l1, 0, 0)` with all three fields set to `Some(zero)`:

```rust
ValidResourceBounds::AllResources(AllResourceBounds { l1_gas, l2_gas, l1_data_gas }) =>
    protobuf::ResourceBounds {
        l1_gas: Some(l1_gas.into()),
        l2_gas: Some(l2_gas.into()),          // Some(zero)
        l1_data_gas: Some(l1_data_gas.into()), // Some(zero)
    },
``` [4](#0-3) 

**Step 4 – Protobuf deserialization silently downgrades to `L1Gas`.**

`TryFrom<protobuf::ResourceBounds> for ValidResourceBounds` treats `l1_data_gas = None` as zero (backward-compat shim for 0.13.2), then applies the rule: if both `l2_gas` and `l1_data_gas` are zero → `L1Gas`. A wire message with `Some(zero)` for both fields therefore deserializes to `L1Gas(l1)`:

```rust
let l1_data_gas = value.l1_data_gas.unwrap_or_default(); // None → zero, Some(zero) → zero
// ...
Ok(if l1_data_gas.is_zero() && l2_gas.is_zero() {
    ValidResourceBounds::L1Gas(l1_gas)   // ← AllResources(l1,0,0) lands here
} else {
    ValidResourceBounds::AllResources(...)
})
``` [5](#0-4) 

**Step 5 – Hash recomputed from deserialized form yields H₂ ≠ H₁.**

Any peer that receives the transaction over P2P and recomputes the hash from the deserialized `L1Gas` variant obtains H₂ (two-element preimage). The stored hash is H₁ (three-element preimage). The mismatch causes the transaction to be rejected as having an invalid hash.

### Impact Explanation

A user who submits an `AllResourceBounds { l1_gas: X, l2_gas: 0, l1_data_gas: 0 }` invoke transaction via the RPC/gateway receives a valid transaction hash H₁ and a valid signature over H₁. The originating node stores and propagates the transaction. Every peer that deserializes it over P2P reconstructs `L1Gas(X)` and recomputes H₂ ≠ H₁. The transaction is permanently rejected by all peers, preventing it from being included in any block proposed by a non-originating node. If the originating node is the proposer, the block containing the transaction will be rejected by validators that recompute the hash during block validation, causing a consensus failure.

This matches: **High. Mempool/gateway/RPC admission rejects valid transactions before sequencing.**

### Likelihood Explanation

The trigger requires only that a user submit a V3 invoke transaction with zero `l2_gas` and zero `l1_data_gas`. The gateway imposes no minimum on either field. The `AllResourceBounds` struct defaults both to zero, so any client that omits or zeroes these fields produces the vulnerable transaction. No privileged access is required.

### Recommendation

The deserialization in `TryFrom<protobuf::ResourceBounds> for ValidResourceBounds` must preserve the original variant. One correct approach: keep the `unwrap_or_default` shim for `None` (backward compat), but only downgrade to `L1Gas` when `l1_data_gas` was **absent** (`None`) from the wire, not when it was explicitly sent as zero. Alternatively, remove the downgrade entirely once 0.13.2 support is dropped (as the existing `TODO` comment already notes), and always deserialize into `AllResources` when all three fields are present.

```rust
// Proposed fix: only downgrade when l1_data_gas was absent on the wire
let (l1_data_gas_raw, was_absent) = match value.l1_data_gas {
    None => (protobuf::ResourceLimits::default(), true),
    Some(v) => (v, false),
};
let l1_data_gas: ResourceBounds = l1_data_gas_raw.try_into()?;
Ok(if was_absent && l2_gas.is_zero() {
    ValidResourceBounds::L1Gas(l1_gas)
} else {
    ValidResourceBounds::AllResources(AllResourceBounds { l1_gas, l2_gas, l1_data_gas })
})
```

### Proof of Concept

1. Craft an `RpcInvokeTransactionV3` with `resource_bounds = AllResourceBounds { l1_gas: {max_amount: 1000, max_price_per_unit: 1}, l2_gas: {0,0}, l1_data_gas: {0,0} }`.
2. Submit to the gateway. The gateway computes H₁ = `get_invoke_transaction_v3_hash` with `AllResources` (three-element Poseidon preimage including `L1_DATA_GAS_packed(0)`).
3. Serialize the `InternalRpcTransaction` to protobuf via `From<ValidResourceBounds> for protobuf::ResourceBounds`. All three fields are `Some(zero)`.
4. Deserialize on a peer via `TryFrom<protobuf::ResourceBounds> for ValidResourceBounds`. Both `l2_gas` and `l1_data_gas` are zero → result is `ValidResourceBounds::L1Gas(l1_gas)`.
5. Recompute the hash on the peer: `get_invoke_transaction_v3_hash` with `L1Gas` (two-element Poseidon preimage, no `L1_DATA_GAS_packed`). Result is H₂.
6. Assert H₁ ≠ H₂. The peer rejects the transaction with a hash mismatch error, even though the transaction is valid and correctly signed. [5](#0-4) [1](#0-0) [6](#0-5)

pub fn get_tip_resource_bounds_hash(
    resource_bounds: &ValidResourceBounds,
    tip: &Tip,
) -> Result<Felt, StarknetApiError> {
    let l1_resource_bounds = resource_bounds.get_l1_bounds();
    let l2_resource_bounds = resource_bounds.get_l2_bounds();

    // L1 and L2 gas bounds always exist.
    // Old V3 txs always have L2 gas bounds of zero, but they exist.
    let mut resource_felts = vec![
        get_concat_resource(&l1_resource_bounds, L1_GAS)?,
        get_concat_resource(&l2_resource_bounds, L2_GAS)?,
    ];

    // For new V3 txs, need to also hash the data gas bounds.
    resource_felts.extend(match resource_bounds {
        ValidResourceBounds::L1Gas(_) => vec![],
        ValidResourceBounds::AllResources(all_resources) => {
            vec![get_concat_resource(&all_resources.l1_data_gas, L1_DATA_GAS)?]
        }
    });

    Ok(HashChain::new().chain(&tip.0.into()).chain_iter(resource_felts.iter()).get_poseidon_hash())
}

impl TryFrom<protobuf::ResourceBounds> for ValidResourceBounds {
    type Error = ProtobufConversionError;
    fn try_from(value: protobuf::ResourceBounds) -> Result<Self, Self::Error> {
        let Some(l1_gas) = value.l1_gas else {
            return Err(missing("ResourceBounds::l1_gas"));
        };
        let Some(l2_gas) = value.l2_gas else {
            return Err(missing("ResourceBounds::l2_gas"));
        };
        // TODO(Shahak): Assert data gas is not none once we remove support for 0.13.2.
        let l1_data_gas = value.l1_data_gas.unwrap_or_default();
        let l1_gas: ResourceBounds = l1_gas.try_into()?;
        let l2_gas: ResourceBounds = l2_gas.try_into()?;
        let l1_data_gas: ResourceBounds = l1_data_gas.try_into()?;
        Ok(if l1_data_gas.is_zero() && l2_gas.is_zero() {
            ValidResourceBounds::L1Gas(l1_gas)
        } else {
            ValidResourceBounds::AllResources(AllResourceBounds { l1_gas, l2_gas, l1_data_gas })
        })
    }
