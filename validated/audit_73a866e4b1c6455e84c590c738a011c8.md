### Title
Bonus Accrual Checkpoint Advances Without Payment When Pool Is Empty During Redelegation Slash — (`x/tieredrewards/keeper/claim_rewards.go`, `x/tieredrewards/keeper/slash.go`)

---

### Summary

In `processEventsAndClaimBonus`, `applyBonusAccrualCheckpoint` advances `pos.LastBonusAccrual` to `blockTime` **before** the pool-sufficiency check. When `slashRedelegationPosition` catches and silently ignores `ErrInsufficientBonusPool`, it then calls `setPositionWithState` with the already-advanced checkpoint. The bonus accrued for the entire pre-slash bonded segment is permanently forfeited — the checkpoint says "paid up to now" but no coins were ever sent.

---

### Finding Description

**Step 1 — Entry point:**
`BeforeRedelegationSlashed` (hooks.go:128-130) calls `slashRedelegationPosition`. [1](#0-0) 

**Step 2 — Bonus settlement attempt:**
`slashRedelegationPosition` calls `processEventsAndClaimBonus(ctx, &pos)` and explicitly catches `ErrInsufficientBonusPool`, logging it and continuing. [2](#0-1) 

**Step 3 — Checkpoint advanced before pool check:**
Inside `processEventsAndClaimBonus`, `applyBonusAccrualCheckpoint` is called at line 215, mutating `pos.LastBonusAccrual` to `blockTime`. Only *after* this does the function check `sufficientBonusPoolBalance` at line 230. If the pool is empty, `ErrInsufficientBonusPool` is returned — but `pos.LastBonusAccrual` has already been advanced in the in-memory struct. [3](#0-2) 

`applyBonusAccrualCheckpoint` unconditionally calls `pos.UpdateLastBonusAccrual(accrualEnd)`: [4](#0-3) 

**Step 4 — Mutated position persisted:**
Back in `slashRedelegationPosition`, after the error is swallowed, `setPositionWithState(ctx, pos, ...)` is called with the `pos` that now carries the advanced `LastBonusAccrual`. `setPositionWithState` calls `k.Positions.Set(ctx, pos.Id, pos)`, persisting the full `Position` struct including the advanced checkpoint. [5](#0-4) [6](#0-5) 

**Step 5 — Permanent loss:**
The next time `processEventsAndClaimBonus` runs for this position, `segmentStart` is initialized from `pos.LastBonusAccrual` (claim_rewards.go:165), which is now `blockTime` of the slash. The entire pre-slash bonded segment `[original_LastBonusAccrual, blockTime]` is never re-computed. The bonus is permanently lost. [7](#0-6) 

---

### Impact Explanation

The invariant that `pos.LastBonusAccrual` only advances when the corresponding bonus coins are actually sent from `RewardsPoolName` to the owner is violated. Position owners permanently lose all bonus rewards accrued during the bonded segment preceding a redelegation slash whenever the `RewardsPoolName` module account balance is zero at slash time. This is a loss of backed assets owed to users — the accounting record says "settled" but no transfer occurred.

---

### Likelihood Explanation

The `RewardsPoolName` pool can be empty at any block (e.g., not yet funded, recently drained by normal claims, or temporarily between funding cycles). Redelegation slashes are triggered by validator double-sign or downtime evidence, which is an ordinary on-chain event. No attacker action is required to drain the pool — the condition arises naturally. Any position owner with an active redelegation to a slashed validator at a moment when the pool is empty will silently lose their accrued bonus.

---

### Recommendation

Move `applyBonusAccrualCheckpoint` (and `pos.UpdateLastKnownBonded`) to **after** the successful `SendCoinsFromModuleToAccount` call, or alternatively, do not advance the checkpoint when `ErrInsufficientBonusPool` is returned. In `slashRedelegationPosition`, when the error is caught, the position should be saved with the **original** (pre-call) checkpoint values so the rewards remain claimable once the pool is refunded.

---

```go
func (k Keeper) processEventsAndClaimBonus(ctx context.Context, pos *types.PositionState) (sdk.Coins, error) {
	// Rewards should have been claimed before undelegation
	if !pos.IsDelegated() {
		return sdk.NewCoins(), nil
	}

	valAddr, err := sdk.ValAddressFromBech32(pos.Delegation.ValidatorAddress)
	if err != nil {
		return nil, err
	}

	events, err := k.getValidatorEventsSince(ctx, valAddr, pos.LastEventSeq)
	if err != nil {
		return nil, err
	}

	sdkCtx := sdk.UnwrapSDKContext(ctx)
	blockTime := sdkCtx.BlockTime()

	totalBonus := math.ZeroInt()
	// Use the persisted bonded state from the last replay, not a hardcoded default.
	// This prevents overpaying bonus for unbonded gaps between claims.
	bonded := pos.LastKnownBonded
	segmentStart := pos.LastBonusAccrual

	tier, err := k.getTier(ctx, pos.TierId)
	if err != nil {
		return nil, err
	}

	for _, entry := range events {
		evt := entry.Event

		if bonded {
			// Compute bonus for the bonded segment [segmentStart, eventTime]
			// using the snapshot rate at the event.
			bonus := k.computeSegmentBonus(*pos, tier, segmentStart, evt.Timestamp, evt.TokensPerShare)
			totalBonus = totalBonus.Add(bonus)
		}

		// Update bonded state based on event type.
		switch evt.EventType {
		case types.ValidatorEventType_VALIDATOR_EVENT_TYPE_UNBOND:
			bonded = false
		case types.ValidatorEventType_VALIDATOR_EVENT_TYPE_BOND:
			bonded = true
		case types.ValidatorEventType_VALIDATOR_EVENT_TYPE_SLASH:
			// Slash doesn't change bonded state.
		}

		segmentStart = evt.Timestamp
		pos.UpdateLastEventSeq(entry.Seq)

		// Decrement reference count.
		if err := k.decrementEventRefCount(ctx, valAddr, entry.Seq); err != nil {
			return nil, err
		}
	}

	val, err := k.stakingKeeper.GetValidator(ctx, valAddr)
	if err != nil {
		return nil, err
	}
	// Defensive: validator bond status check
	if bonded && val.IsBonded() {
		currentRate, err := k.getTokensPerShare(ctx, valAddr)
		if err != nil {
			return nil, err
		}
		bonus := k.computeSegmentBonus(*pos, tier, segmentStart, blockTime, currentRate)
		totalBonus = totalBonus.Add(bonus)
	}

	applyBonusAccrualCheckpoint(&pos.Position, blockTime)
	// Persist the bonded state so the next replay starts correctly.
	pos.UpdateLastKnownBonded(bonded)

	if totalBonus.IsZero() {
		return sdk.NewCoins(), nil
	}

	bondDenom, err := k.stakingKeeper.BondDenom(ctx)
	if err != nil {
		return nil, err
	}

	bonusCoins := sdk.NewCoins(sdk.NewCoin(bondDenom, totalBonus))

	if err := k.sufficientBonusPoolBalance(ctx, bonusCoins); err != nil {
		return nil, err
	}

	ownerAddr, err := sdk.AccAddressFromBech32(pos.Owner)
	if err != nil {
		return nil, errorsmod.Wrap(sdkerrors.ErrInvalidAddress, "invalid owner address")
	}

	if err := k.bankKeeper.SendCoinsFromModuleToAccount(ctx, types.RewardsPoolName, ownerAddr, bonusCoins); err != nil {
		return nil, err
	}

	if err := sdkCtx.EventManager().EmitTypedEvent(&types.EventBonusRewardsClaimed{
		PositionId: pos.Id,
		Owner:      pos.Owner,
		Rewards:    bonusCoins,
	}); err != nil {
		return nil, err
	}

	return bonusCoins, nil
```

```go
func applyBonusAccrualCheckpoint(pos *types.Position, blockTime time.Time) {
	accrualEnd := blockTime
	if pos.CompletedExitLockDuration(blockTime) {
		accrualEnd = pos.ExitUnlockAt
	}
	pos.UpdateLastBonusAccrual(accrualEnd)
}
```
