### Title
GC Removes Blocks Still Reachable Under the `confirmations ≤ gc_threshold` Guard, Causing Permanent Proof Lockout — (`contract/src/lib.rs`)

---

### Summary

After every `submit_blocks` call that triggers GC, the oldest mainchain block is deleted from both `mainchain_header_to_height` and `mainchain_header_to_height` (via `remove_block_header`). The guard in `verify_transaction_inclusion` only checks `args.confirmations <= self.gc_threshold`, which passes for the just-removed block. The subsequent `mainchain_header_to_height.get(...)` lookup returns `None` and panics. The block is permanently inaccessible.

**Why the guard is insufficient:**

After GC the removed block (height H) has *actual* confirmations = (H+T) − H + 1 = **T+1**. The guard only rejects `confirmations > gc_threshold`, so a request for T confirmations passes. But the block is gone. The guard's implicit invariant — *"if confirmations ≤ gc_threshold the block is in storage"* — is broken on every GC cycle for the block that was just evicted.

---

### Impact Explanation

Every time a single block is submitted to a chain sitting exactly at `gc_threshold`, GC evicts the oldest block. Any proof consumer holding a reference to that block hash and requesting `confirmations ≤ gc_threshold` receives a panic (NEAR transaction failure) instead of a result. The block is permanently deleted; no future call can succeed for it. This is a **permanent proof lockout** triggered by normal, expected relayer activity.

---

### Likelihood Explanation

This fires on every `submit_blocks` call once the chain reaches `gc_threshold` blocks — the steady-state operating condition of a live deployment. No special privileges beyond the trusted-relayer role are required; the relayer is the intended submitter. The proof consumer needs only to hold a block hash that was valid one block ago.

---

### Recommendation

Replace the single-threshold guard with a check against the actual oldest retained height:

```rust
let initial_height = self
    .headers_pool
    .get(&self.mainchain_initial_blockhash)
    .unwrap_or_else(|| env::panic_str(ERR_KEY_NOT_EXIST))
    .block_height;

let target_block_height = self
    .mainchain_header_to_height
    .get(&args.tx_block_blockhash)
    .unwrap_or_else(|| env::panic_str("block does not belong to the current main chain"));

require!(
    target_block_height >= initial_height,
    "Block has been garbage collected"
);
```

This ensures the guard reflects the real lower bound of retained history rather than the theoretical maximum.
