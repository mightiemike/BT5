Audit Report

## Title
`SwapAllowlistExtension` Checks Router Address Instead of End User, Allowing Allowlist Bypass — (`metric-periphery/contracts/extensions/SwapAllowlistExtension.sol`)

## Summary

`SwapAllowlistExtension.beforeSwap` gates on the `sender` parameter, which resolves to `msg.sender` of the pool's `swap()` call — the router contract, not the end user. When a pool admin allowlists the router to enable router-based swaps, every user can bypass per-user swap restrictions by routing through `MetricOmmSimpleRouter`. The sibling `DepositAllowlistExtension` correctly gates on `owner` (the actual beneficiary), making the two extensions semantically inconsistent.

## Finding Description

`SwapAllowlistExtension.beforeSwap` checks the `sender` argument: [1](#0-0) 

`MetricOmmPool.swap()` passes `msg.sender` (the immediate caller) as the first argument to `_beforeSwap`: [2](#0-1) 

`MetricOmmSimpleRouter.exactInputSingle` calls `pool.swap()` directly, so the pool sees the router as `msg.sender`: [3](#0-2) 

By contrast, `DepositAllowlistExtension.beforeAddLiquidity` correctly checks `owner` (the actual position beneficiary), not the caller: [4](#0-3) 

The root cause is that `beforeSwap` receives `sender` (the pool's `msg.sender`, i.e., the router) but the intended gate is the end user. There is no mechanism in the extension to recover the original EOA.

## Impact Explanation

A pool admin who configures `SwapAllowlistExtension` to restrict swaps to a curated set of addresses faces a forced dilemma: if the router is allowlisted (required for any router-based swap), every user regardless of individual allowlist status can bypass the restriction. On a curated pool restricted to specific market makers or KYC'd counterparties, unauthorized users trading at oracle prices can extract value from LPs, constituting a direct loss of LP principal. This matches the "Admin-boundary break: factory/oracle role checks are bypassed by an unprivileged path" impact gate.

## Likelihood Explanation

`MetricOmmSimpleRouter` is the primary supported swap entrypoint. Any pool that enables `SwapAllowlistExtension` and expects users to use the router will encounter this condition. The pool admin has no in-protocol mechanism to enforce per-user swap restrictions through the router — the flaw is structural and repeatable by any unprivileged user.

## Recommendation

`SwapAllowlistExtension.beforeSwap` should gate on the `recipient` parameter or an explicit originating-user field forwarded through the extension callback context, analogous to how `owner` is forwarded for deposits in `beforeAddLiquidity`. The fix must make the swap allowlist check the same actor that the deposit allowlist checks — the actual economic initiator — not the intermediate router contract.

## Proof of Concept

1. Pool admin deploys a pool with `SwapAllowlistExtension` configured.
2. Pool admin calls `setAllowedToSwap(pool, alice, true)` — only `alice` is approved.
3. Pool admin calls `setAllowedToSwap(pool, router, true)` — required for any router-based swap.
4. `bob` (not allowlisted) calls `router.exactInputSingle(...)`.
5. Router calls `pool.swap(recipient=bob, ...)` — `msg.sender` seen by the pool is the router address.
6. Pool calls `extension.beforeSwap(sender=router, ...)`.
7. Extension evaluates `allowedSwapper[pool][router]` → `true` → swap proceeds.
8. `bob` successfully swaps against the curated pool despite not being individually allowlisted. [5](#0-4) [6](#0-5)

    _checkDeadline(params.deadline);
    uint128 priceLimitX64 = MetricOmmSwapPath.normalizePriceLimit(params.zeroForOne, params.priceLimitX64);

    _setNextCallbackContext(params.pool, CALLBACK_MODE_JUST_PAY, msg.sender, params.tokenIn);
    (int128 amount0Delta, int128 amount1Delta) = IMetricOmmPoolActions(params.pool)
      .swap(
        params.recipient,
        params.zeroForOne,
        MetricOmmSwapInputs.asAmountSpecifiedIn(params.amountIn),
        priceLimitX64,
        "",
        params.extensionData
      );
    int128 out = MetricOmmSwapResults.extractAmountOut(params.zeroForOne, amount0Delta, amount1Delta);
    amountOut = MetricOmmSwapInputs.int128ToUint128(out);
    if (amountOut < params.amountOutMinimum) revert InsufficientOutput(amountOut, params.amountOutMinimum);

    _clearExpectedCallbackPool();
  }
