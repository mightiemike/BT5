Audit Report

## Title
SwapAllowlistExtension Checks Immediate Pool Caller (Router) Instead of Original User, Enabling Allowlist Bypass — (`metric-periphery/contracts/extensions/SwapAllowlistExtension.sol`)

## Summary
`SwapAllowlistExtension.beforeSwap` gates swaps by checking `allowedSwapper[pool][sender]`, where `sender` is the immediate `msg.sender` of `pool.swap()`. When swaps are routed through `MetricOmmSimpleRouter`, the pool's `msg.sender` is the router contract, not the original user. A pool admin who allowlists the router to support normal periphery flows inadvertently grants every user the ability to bypass per-user swap restrictions entirely.

## Finding Description
`SwapAllowlistExtension.beforeSwap` performs the check:

```solidity
if (!allowAllSwappers[msg.sender] && !allowedSwapper[msg.sender][sender]) {
    revert IMetricOmmPoolActions.NotAllowedToSwap();
}
``` [1](#0-0) 

Here `msg.sender` is the pool (used as the namespace key, correct), and `sender` is the first argument forwarded by the pool — which is `msg.sender` of the pool's own `swap()` call:

```solidity
_beforeSwap(
    msg.sender,   // ← immediate caller of pool.swap()
    recipient,
    ...
);
``` [2](#0-1) 

When a user calls `MetricOmmSimpleRouter.exactInputSingle`, the router calls `pool.swap()` directly:

```solidity
(int128 amount0Delta, int128 amount1Delta) = IMetricOmmPoolActions(params.pool)
    .swap(
        params.recipient,
        params.zeroForOne,
        ...
        params.extensionData
    );
``` [3](#0-2) 

The pool's `msg.sender` is now the **router**, so the extension evaluates `allowedSwapper[pool][router]` instead of `allowedSwapper[pool][originalUser]`. The same substitution occurs in `exactInput` (multi-hop) and `exactOutputSingle`/`exactOutput` paths. [4](#0-3) 

There is no mechanism in the extension or the router to forward the original user's identity. The `extensionData` field is caller-supplied and unverified, so it cannot be trusted as an attestation of the original user.

## Impact Explanation
Complete bypass of the swap allowlist on curated pools. Any user can execute swaps on a pool intended to be restricted to specific addresses by routing through `MetricOmmSimpleRouter`. This is a direct admin-boundary break: the pool admin's access-control policy is silently nullified by the standard supported periphery path. LP funds in the pool are exposed to trades from actors the pool admin explicitly intended to exclude. This meets the "admin-boundary break" and "broken core pool functionality" criteria.

## Likelihood Explanation
High. `MetricOmmSimpleRouter` is the standard periphery entry point for swaps. A pool admin who deploys a `SwapAllowlistExtension` and wants to support normal user flows through the router must allowlist the router address — this is the natural and expected configuration. Any user who knows the router address (a public, deployed contract) can exploit this immediately without any privileged access or special setup.

## Recommendation
The allowlist must gate the **original economic actor**, not the immediate pool caller. The most robust fix is to have the router encode `msg.sender` into `extensionData` in a verifiable way, and have the extension decode and verify it — but this requires a trust assumption that the extension is only used with a known, trusted router. Alternatively, define a standard interface where the router attests the original user via a transient-storage slot that the extension reads and verifies, preventing spoofing by arbitrary callers.

## Proof of Concept
```
1. Pool admin deploys pool with SwapAllowlistExtension.
2. Admin allowlists userA (intended trader):
       setAllowedToSwap(pool, userA, true)
3. Admin also allowlists the router to support normal periphery flows:
       setAllowedToSwap(pool, router, true)
4. userB (NOT on the allowlist) calls:
       router.exactInputSingle({pool: pool, recipient: userB, ...})
5. Router calls pool.swap(...) — pool sees msg.sender = router.
6. Extension evaluates: allowedSwapper[pool][router] → true → PASSES.
7. userB's swap executes successfully on the restricted pool.
```

Root cause: `SwapAllowlistExtension.sol` L37, triggered via `MetricOmmSimpleRouter.sol` L72–80, with identity substitution at `MetricOmmPool.sol` L231.
