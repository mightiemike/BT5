Audit Report

## Title
`SwapAllowlistExtension.beforeSwap` Checks Router Address Instead of End User, Allowing Any User to Bypass the Swap Allowlist ‚Äî (`metric-periphery/contracts/extensions/SwapAllowlistExtension.sol`)

## Summary

`SwapAllowlistExtension.beforeSwap` gates swaps by checking the `sender` argument, which `MetricOmmPool.swap` sets to `msg.sender` ‚Äî the direct caller of `pool.swap()`. When users route through `MetricOmmSimpleRouter`, the router is `msg.sender` of `pool.swap()`, so the extension checks the router address rather than the end user. Allowlisting the router (the only way to let legitimate users use the router) opens the gate to every user, including non-allowlisted ones.

## Finding Description

`MetricOmmPool.swap` passes `msg.sender` as `sender` to `_beforeSwap`: [1](#0-0) 

`ExtensionCalling._beforeSwap` forwards that value verbatim to every configured extension via `_callExtensionsInOrder`: [2](#0-1) 

`SwapAllowlistExtension.beforeSwap` then checks `allowedSwapper[msg.sender][sender]`, where `msg.sender` is the pool and `sender` is whoever called `pool.swap()`: [3](#0-2) 

`MetricOmmSimpleRouter.exactInputSingle` calls `pool.swap()` directly, making the router the `msg.sender` of that call: [4](#0-3) 

The same applies to `exactInput`, `exactOutputSingle`, and `exactOutput`. The router stores the original `msg.sender` in transient storage for payment purposes only ‚Äî it is never forwarded to the pool or extension.

The `DepositAllowlistExtension` avoids this flaw by checking `owner` (the position owner parameter), which is passed through unchanged from the caller's explicit argument: [5](#0-4) 

The swap allowlist has no equivalent identity-preserving mechanism. The pool admin faces an impossible choice: if the router is not allowlisted, no user can swap via the router; if the router is allowlisted, every user ‚Äî including non-allowlisted ones ‚Äî passes the gate.

## Impact Explanation

Any user can bypass a `SwapAllowlistExtension`-protected pool by routing through `MetricOmmSimpleRouter`. Pool admins who configure a swap allowlist (e.g., for KYC/AML-gated, institutional-only, or whitelist-only pools) and allowlist the router to enable legitimate participants will inadvertently grant unrestricted swap access to all users. Unauthorized users can execute swaps in restricted pools, extracting value from LP positions priced and sized under the assumption of a controlled participant set. This constitutes a broken core pool access-control mechanism causing potential loss of funds to LPs.

## Likelihood Explanation

`MetricOmmSimpleRouter` is the production-grade, publicly deployed periphery entry point. Pool admins who configure a `SwapAllowlistExtension` will naturally want their allowlisted users to be able to use the router. Allowlisting the router address is the obvious and only available mechanism to achieve this, making the bypass trivially reachable by any unprivileged actor with no special preconditions beyond the admin having taken the expected configuration step.

## Recommendation

Preserve the original caller's identity through the router so the extension can check the actual economic actor. Two standard approaches:

1. **Pass the original caller in `extensionData`**: The router already stores `msg.sender` in transient storage for the payer. Extend the `extensionData` forwarded to the pool to include the original caller, and have the extension decode and check that address instead of `sender` when `msg.sender` (the pool's caller) is a trusted router.

2. **Follow the `DepositAllowlistExtension` pattern**: Introduce a caller-supplied identity parameter (analogous to `owner` in `beforeAddLiquidity`) that the router passes through unchanged, and have the extension check that parameter rather than `sender`.

## Proof of Concept

```
1. Pool admin deploys pool with SwapAllowlistExtension.
2. Admin calls setAllowedToSwap(pool, alice, true)       // alice is KYC'd
3. Admin calls setAllowedToSwap(pool, router, true)      // needed so alice can use the router
4. Bob (not allowlisted) calls:
       router.exactInputSingle({pool: pool, recipient: bob, ...})
5. Router calls pool.swap(bob, ...) ‚Äî msg.sender of pool.swap() = router
6. Pool calls _beforeSwap(sender=router, ...)
7. Extension checks allowedSwapper[pool][router] == true ‚Üí passes
8. Bob's swap executes successfully despite not being allowlisted.
```

Foundry test plan: deploy a pool with `SwapAllowlistExtension`, allowlist only `alice` and the router, call `exactInputSingle` from an unallowlisted `bob` address, and assert the swap succeeds ‚Äî demonstrating the bypass.
