### Title
Unchecked `transferFrom` Return Value in `replaceUsdcEWithUsdc` Enables USDC-E Theft — (File: `core/contracts/ContractOwner.sol`)

---

### Summary

`ContractOwner.replaceUsdcEWithUsdc()` uses a raw, unchecked `IERC20Base.transferFrom()` to pull USDC from the caller before sending USDC-E back. If the USDC token at the hardcoded address is a no-revert-on-failure token, the transfer silently returns `false`, the function continues, and the caller receives USDC-E without ever providing USDC — draining any victim subaccount's `DirectDepositV1` balance.

---

### Finding Description

`replaceUsdcEWithUsdc` is a public, permissionless function (restricted only by `block.chainid == 57073`) that is intended to atomically swap USDC-E held in a `DirectDepositV1` contract for USDC:

```
ContractOwner.sol lines 608–620
```

The critical sequence is:

1. Read the USDC-E balance of `directDepositV1`.
2. **Pull USDC from `msg.sender` via raw `transferFrom` — return value not checked.**
3. Withdraw USDC-E from `directDepositV1` to `ContractOwner` via `DirectDepositV1.withdraw()`.
4. Send USDC-E to `msg.sender` via `safeTransfer`. [1](#0-0) 

Step 2 uses `IERC20Base(usdc).transferFrom(msg.sender, directDepositV1, balance)` — a direct call to the ERC-20 interface whose `bool` return value is silently discarded. The rest of the codebase consistently uses `ERC20Helper.safeTransferFrom`, which wraps the call and requires `success && (data.length == 0 || abi.decode(data, (bool)))`. [2](#0-1) 

`DirectDepositV1.withdraw()` is `onlyOwner` (owner = `ContractOwner`), so step 3 always succeeds and moves USDC-E to `ContractOwner`. Step 4 then unconditionally forwards that USDC-E to `msg.sender`. [3](#0-2) 

---

### Impact Explanation

If the USDC token at `0x2D270e6886d130D724215A266106e6832161EAEd` on chain 57073 is a no-revert-on-failure token (returns `false` instead of reverting on insufficient balance or allowance), an attacker with zero USDC can:

- Call `replaceUsdcEWithUsdc(victimSubaccount)` for any subaccount whose `directDepositV1` holds USDC-E.
- The `transferFrom` silently fails; no USDC is transferred in.
- USDC-E is withdrawn from the victim's `DirectDepositV1` and forwarded to the attacker.
- The victim permanently loses their USDC-E with no recourse.

The corrupted asset delta: `directDepositV1.balanceOf(usdcE)` goes to zero; attacker gains `balance` USDC-E; victim subaccount loses `balance` USDC-E.

---

### Likelihood Explanation

The function is callable by any unprivileged address on chain 57073 (Ink chain) — no `onlyOwner` or similar guard. The exploitability is conditional on the specific USDC deployment at the hardcoded address being a no-revert-on-failure token. Standard Circle USDC reverts on failure, but the Ink-chain deployment at that address is non-standard and its exact behavior is not guaranteed by the contract code. The code-level defect (missing return-value check) is unconditional and present regardless of token behavior. [4](#0-3) 

---

### Recommendation

Replace the raw `transferFrom` call with `ERC20Helper.safeTransferFrom`, which is already imported and used throughout the contract:

```solidity
// Before (unsafe):
IERC20Base(usdc).transferFrom(msg.sender, directDepositV1, balance);

// After (safe):
IERC20Base(usdc).safeTransferFrom(msg.sender, directDepositV1, balance);
``` [5](#0-4) 

---

### Proof of Concept

1. Victim subaccount `S` has `directDepositV1Address[S]` deployed and holding 1000 USDC-E.
2. Attacker calls `ContractOwner.replaceUsdcEWithUsdc(S)` on chain 57073 with zero USDC balance/allowance.
3. `IERC20Base(usdc).transferFrom(attacker, directDepositV1, 1000)` returns `false` (no-revert-on-failure USDC); return value ignored.
4. `DirectDepositV1(directDepositV1).withdraw(usdcE)` transfers 1000 USDC-E to `ContractOwner`.
5. `IERC20Base(usdcE).safeTransfer(attacker, 1000)` sends 1000 USDC-E to attacker.
6. Attacker holds 1000 USDC-E; victim's `directDepositV1` is drained; no USDC was ever provided. [6](#0-5)

    function replaceUsdcEWithUsdc(bytes32 subaccount) external {
        require(block.chainid == 57073, ERR_UNAUTHORIZED);

        require(directDepositV1 != address(0), "no dda");

        uint256 balance = IERC20Base(usdcE).balanceOf(directDepositV1);
        if (balance > 0) {
            IERC20Base(usdc).transferFrom(msg.sender, directDepositV1, balance);
            DirectDepositV1(directDepositV1).withdraw(IIERC20Base(usdcE));
            IERC20Base(usdcE).safeTransfer(msg.sender, balance);
        }
    }

    function safeTransferFrom(
        IERC20Base self,
        address from,
        address to,
        uint256 amount
    ) internal {
        (bool success, bytes memory data) = address(self).call(
            abi.encodeWithSelector(
                IERC20Base.transferFrom.selector,
                from,
                to,
                amount
            )
        );

        require(
            success && (data.length == 0 || abi.decode(data, (bool))),
            ERR_TRANSFER_FAILED
        );
    }
```
