### Title
Bonus Rewards Permanently Lost When Bonus Pool Is Insufficient During Partial Redelegation Slash — (`x/tieredrewards/keeper/slash.go`)

### Summary

In `slashRedelegationPosition`, when `processEventsAndClaimBonus` returns `ErrInsufficientBonusPool`, the error is deliberately swallowed to prevent a chain halt. However, by the time the error is returned, `processEventsAndClaimBonus` has already (1) decremented every pending event's `ReferenceCount` in the store and (2) advanced the position's `LastEventSeq`, `LastBonusAccrual`, and `LastKnownBonded` in memory. For a **partial** slash, `slashRedelegationPosition` then calls `setPositionWithState` with the already-advanced in-memory `pos`, permanently persisting the advanced checkpoints without ever paying the bonus. The accrued bonus is irrecoverably lost.

### Finding Description

`slashRedelegationPosition` calls `processEventsAndClaimBonus` and explicitly swallows `ErrInsufficientBonusPool`: [1](#0-0) 

Inside `processEventsAndClaimBonus`, the event-processing loop runs to completion before the pool check. For every event entry it:
- calls `pos.UpdateLastEventSeq(entry.Seq)` (in-memory)
- calls `k.decrementEventRefCount(ctx, valAddr, entry.Seq)` — a **direct store write** that may garbage-collect the event [2](#0-1) 

After the loop, `applyBonusAccrualCheckpoint` and `pos.UpdateLastKnownBonded` advance the remaining in-memory checkpoints: [3](#0-2) 

Only then is the pool balance checked: [4](#0-3) 

When the pool is insufficient, `ErrInsufficientBonusPool` is returned. Back in `slashRedelegationPosition`, for a **partial** slash the code does **not** call `ClearBonusCheckpoints()` — it simply persists the already-mutated `pos` via `setPositionWithState`: [5](#0-4) 

Contrast with the full-slash branch, which calls `pos.ClearBonusCheckpoints()` before persisting: [6](#0-5) 

The partial-slash branch has no equivalent reset.

The result is a broken invariant: `LastEventSeq` and `LastBonusAccrual` are advanced past events whose reference counts were already decremented (and potentially garbage-collected), while the corresponding bonus was never transferred. On the next claim, `getValidatorEventsSince` returns no events before the new `LastEventSeq`, and `computeSegmentBonus` starts from the new `LastBonusAccrual`, so the lost accrual window is permanently skipped. [7](#0-6) 

### Impact Explanation

The corrupted value is the position owner's accrued bonus reward for the period `[pos.LastBonusAccrual, blockTime]` at the time of the slash. The amount is:

`shares × tokensPerShare × tier.BonusApy × durationSeconds / SecondsPerYear` [8](#0-7) 

This amount is permanently unclaimable. The position owner receives zero bonus for the accrual window, with no recourse. The `RewardsPool` module account retains the funds but they are effectively stranded relative to this position's entitlement.

### Likelihood Explanation

The trigger requires three concurrent conditions:
1. A tier position has an active redelegation entry (created via `MsgTierRedelegate`).
2. The source validator is slashed during the redelegation period, firing `BeforeRedelegationSlashed`.
3. The `RewardsPoolName` module account has insufficient balance to cover the accrued bonus at slash time.

Condition 3 is explicitly anticipated by the codebase — the ADR documents "Bonus forfeits silently if the pool is insufficient (chain-halt avoidance)" — but the implementation does not distinguish between forfeiting the bonus (acceptable) and permanently advancing the checkpoints (the bug). A pool that is temporarily empty or underfunded, combined with a validator slash, is a realistic operational scenario. [9](#0-8) 

### Recommendation

In `slashRedelegationPosition`, when `ErrInsufficientBonusPool` is caught, do **not** use the mutated `pos` returned by `processEventsAndClaimBonus`. Instead, re-fetch the original position from the store (or snapshot it before the call) and use the unmodified checkpoints for the subsequent `setPositionWithState` call. This ensures that when the pool is replenished, the position can still replay the skipped events and claim the owed bonus. Alternatively, mirror the full-slash branch and call `pos.ClearBonusCheckpoints()` before persisting — though this still forfeits the bonus, it at least avoids the inconsistency between the store's decremented reference counts and the position's stale `LastEventSeq`.

### Proof of Concept

1. Alice creates a tier position on validator V1 and redelegates to V2 via `MsgTierRedelegate`. A `RedelegationMappings` entry is created.
2. Time passes; bonus accrues. The `RewardsPool` is drained to zero (e.g., by other claims or simply never funded).
3. V1 is slashed. The staking module fires `BeforeRedelegationSlashed`, which calls `slashRedelegationPosition`.
4. Inside `processEventsAndClaimBonus`, the event loop runs: `decrementEventRefCount` is called for each pending event (store writes committed). `pos.LastEventSeq` and `pos.LastBonusAccrual` are advanced in memory.
5. `sufficientBonusPoolBalance` returns `ErrInsufficientBonusPool`. The error propagates back to `slashRedelegationPosition`.
6. The error is swallowed. Since `sharesToUnbond < pos.Delegation.Shares` (partial slash), `ClearBonusCheckpoints` is **not** called.
7. `setPositionWithState` is called with the mutated `pos` — advanced `LastEventSeq` and `LastBonusAccrual` are persisted to the store.
8. The pool is later replenished. Alice calls `MsgClaimTierRewards`. `processEventsAndClaimBonus` starts from the new `LastEventSeq` and `LastBonusAccrual`, skipping the entire accrual window from step 2. Alice receives zero bonus for that period. [10](#0-9)

```go
func (k Keeper) slashRedelegationPosition(ctx context.Context, unbondingId uint64, sharesToUnbond math.LegacyDec) error {
	positionId, err := k.getRedelegationMapping(ctx, unbondingId)
	if errors.Is(err, collections.ErrNotFound) {
		return nil
	}
	if err != nil {
		return err
	}

	pos, err := k.getPositionState(ctx, positionId)
	if errors.Is(err, types.ErrPositionNotFound) {
		k.logger(ctx).Error("position not found during redelegation slash",
			"position_id", positionId,
			"unbonding_id", unbondingId,
			"error", err.Error(),
		)
		return nil
	}
	if err != nil {
		return err
	}

	if !pos.IsDelegated() {
		// Defensive
		k.logger(ctx).Error("delegation missing during BeforeRedelegationSlashed",
			"position_id", positionId,
			"unbonding_id", unbondingId,
			"shares_to_unbond", sharesToUnbond.String(),
		)
		return nil
	}

	dstValStr := pos.Delegation.ValidatorAddress

	// Settle bonus against PRE-slash shares.
	if _, err := k.processEventsAndClaimBonus(ctx, &pos); err != nil {
		// Deliberately forgo bonus rewards if pool is insufficient to prevent chain halt.
		if errors.Is(err, types.ErrInsufficientBonusPool) {
			k.logger(ctx).Error("insufficient bonus pool during redelegation slash",
				"position_id", pos.Id,
				"error", err.Error(),
			)
		} else {
			return err
		}
	}

	fullSlash := sharesToUnbond.GTE(pos.Delegation.Shares)

	if fullSlash {
		pos.Delegation = nil
		pos.ClearBonusCheckpoints()
		return k.setPositionWithState(ctx, pos, &ValidatorTransition{PreviousAddress: dstValStr})
	}
	// In-memory only: the persisted Position carries no share count, and the
	// live delegation will reflect the post-Unbond shares on the next read.
	// Update the local copy so any follow-up logic in this call sees consistent state.
	pos.Delegation.Shares = pos.Delegation.Shares.Sub(sharesToUnbond)
	return k.setPositionWithState(ctx, pos, nil)
```

```go
// decrementEventRefCount decrements the reference count of a validator event.
// If the reference count reaches zero, the event is deleted.
func (k Keeper) decrementEventRefCount(ctx context.Context, valAddr sdk.ValAddress, seq uint64) error {
	key := collections.Join(valAddr, seq)
	event, err := k.ValidatorEvents.Get(ctx, key)
	if errors.Is(err, collections.ErrNotFound) {
		return nil // already cleaned up
	}
	if err != nil {
		return err
	}

	if event.ReferenceCount <= 1 {
		return k.ValidatorEvents.Remove(ctx, key)
	}

	event.ReferenceCount--
	return k.ValidatorEvents.Set(ctx, key, event)
}
```
