### Title
Protobuf `ValidResourceBounds` Conversion Silently Collapses `AllResources` to `L1Gas`, Producing a Different Transaction Hash — (File: crates/apollo_protobuf/src/converters/transaction.rs)

### Summary

The protobuf deserializer for `ValidResourceBounds` silently converts an `AllResources` variant (with zero `l2_gas` and `l1_data_gas`) into the `L1Gas` variant. Because `get_tip_resource_bounds_hash` hashes a different number of resource-bound felts for each variant (2 for `L1Gas`, 3 for `AllResources`), the transaction hash computed after a protobuf round-trip diverges from the hash computed at ingestion time. This is the sequencer analog of the staking `earned()` bug: a wrong branch is silently taken in a calculation, producing a divergent value instead of the canonical one.

### Finding Description

**Step 1 — Ingestion path (RPC → Internal).**

`RpcInvokeTransactionV3` carries `AllResourceBounds` (always three fields). When the gateway converts it to `InternalRpcInvokeTransactionV3`, the hash is computed via `get_invoke_transaction_v3_hash`, which calls `get_tip_resource_bounds_hash` with `ValidResourceBounds::AllResources`. For `AllResources`, the function builds a three-element felt vector: `[L1_GAS_concat, L2_GAS_concat, L1_DATA_GAS_concat]`, even when `l2_gas = 0` and `l1_data_gas = 0`. [1](#0-0) 

**Step 2 — Protobuf round-trip (P2P sync path).**

The protobuf deserializer for `ValidResourceBounds` applies the following branch:

```rust
Ok(if l1_data_gas.is_zero() && l2_gas.is_zero() {
    ValidResourceBounds::L1Gas(l1_gas)          // ← wrong variant chosen
} else {
    ValidResourceBounds::AllResources(...)
})
``` [2](#0-1) 

A transaction originally stored as `AllResources(l1_gas=X, l2_gas=0, l1_data_gas=0)` is silently re-classified as `L1Gas(l1_gas=X)` after deserialization.

**Step 3 — Hash divergence.**

`get_tip_resource_bounds_hash` produces a **two-element** felt vector for `L1Gas`:

```rust
// L1Gas branch: only L1_GAS and L2_GAS(zero) are hashed
ValidResourceBounds::L1Gas(_) => vec![],   // no L1_DATA_GAS appended
```

versus a **three-element** vector for `AllResources`:

```rust
ValidResourceBounds::AllResources(all_resources) => {
    vec![get_concat_resource(&all_resources.l1_data_gas, L1_DATA_GAS)?]
}
``` [3](#0-2) 

`poseidon(tip, L1_GAS_felt, L2_GAS_felt)` ≠ `poseidon(tip, L1_GAS_felt, L2_GAS_felt, L1_DATA_GAS_felt(zero))`, so the two variants produce different `tip_resource_bounds_hash` values, and therefore different final transaction hashes.

**Step 4 — Attacker-controlled trigger.**

A user submits a valid `RpcInvokeTransactionV3` with `AllResourceBounds { l1_gas: X, l2_gas: 0, l1_data_gas: 0 }`. The gateway's stateless validator explicitly accepts this: [4](#0-3) 

The transaction is accepted, assigned hash H_A (computed with 3 bounds), included in a block, and propagated via P2P. Any node that deserializes the transaction from protobuf recomputes hash H_B (computed with 2 bounds). H_A ≠ H_B.

### Impact Explanation

**High — Transaction conversion or signature/hash logic binds the wrong hash or executable payload.**

When a syncing node deserializes a block containing such a transaction, it recomputes the transaction hash from the deserialized `L1Gas` variant and obtains a value that differs from the hash committed in the block header or mempool. This causes:

- Block-level hash verification failures, causing valid blocks to be rejected by syncing nodes.
- Mempool inconsistency: the same transaction is known under two different hashes on different nodes, breaking deduplication and nonce tracking.
- If the wrong hash is used for receipt/event lookup, RPC queries return wrong or missing results.

### Likelihood Explanation

**Medium.** Any V3 invoke transaction that specifies only `l1_gas` (leaving `l2_gas` and `l1_data_gas` at zero) triggers the bug. This is a valid and common pattern for legacy-style V3 transactions. The protobuf path is exercised by every P2P block sync.

### Recommendation

Remove the `L1Gas` branch from the protobuf `ValidResourceBounds` deserializer for transactions received in a post-0.13.3 context. The `L1Gas` variant should only be reconstructed from explicitly versioned legacy data, not inferred from zero values:

```rust
// Instead of collapsing to L1Gas when l2/l1_data are zero,
// always produce AllResources for protobuf-deserialized data
// that carries all three fields (even if zero).
Ok(ValidResourceBounds::AllResources(AllResourceBounds { l1_gas, l2_gas, l1_data_gas }))
```

Alternatively, add a version tag to the protobuf message so the deserializer knows which variant was originally encoded. [5](#0-4) 

### Proof of Concept

1. Submit an `RpcInvokeTransactionV3` with `resource_bounds = AllResourceBounds { l1_gas: ResourceBounds { max_amount: 1000, max_price_per_unit: 1 }, l2_gas: ResourceBounds::default(), l1_data_gas: ResourceBounds::default() }`.
2. Record the transaction hash H_A returned by the gateway (computed via `get_invoke_transaction_v3_hash` with `AllResources`, 3 bounds).
3. Serialize the transaction to protobuf (`protobuf::ResourceBounds { l1_gas: Some(...), l2_gas: Some(zero), l1_data_gas: Some(zero) }`).
4. Deserialize via `ValidResourceBounds::try_from(...)`. Observe that the result is `ValidResourceBounds::L1Gas(...)` because `l1_data_gas.is_zero() && l2_gas.is_zero()`.
5. Recompute the transaction hash from the deserialized data. Observe hash H_B ≠ H_A because `get_tip_resource_bounds_hash` now hashes only 2 resource bounds instead of 3. [6](#0-5) [7](#0-6)

// An implementation of the SNIP: https://github.com/EvyatarO/SNIPs/blob/snip-8/SNIPS/snip-8.md
pub fn get_tip_resource_bounds_hash(
    resource_bounds: &ValidResourceBounds,
    tip: &Tip,
) -> Result<Felt, StarknetApiError> {
    let l1_resource_bounds = resource_bounds.get_l1_bounds();
    let l2_resource_bounds = resource_bounds.get_l2_bounds();

    // L1 and L2 gas bounds always exist.
    // Old V3 txs always have L2 gas bounds of zero, but they exist.
    let mut resource_felts = vec![
        get_concat_resource(&l1_resource_bounds, L1_GAS)?,
        get_concat_resource(&l2_resource_bounds, L2_GAS)?,
    ];

    // For new V3 txs, need to also hash the data gas bounds.
    resource_felts.extend(match resource_bounds {
        ValidResourceBounds::L1Gas(_) => vec![],
        ValidResourceBounds::AllResources(all_resources) => {
            vec![get_concat_resource(&all_resources.l1_data_gas, L1_DATA_GAS)?]
        }
    });

    Ok(HashChain::new().chain(&tip.0.into()).chain_iter(resource_felts.iter()).get_poseidon_hash())
}

pub(crate) fn get_invoke_transaction_v3_hash<T: InvokeTransactionV3Trait>(
    transaction: &T,
    chain_id: &ChainId,
    transaction_version: &TransactionVersion,
) -> Result<TransactionHash, StarknetApiError> {
    let tip_resource_bounds_hash =
        get_tip_resource_bounds_hash(&transaction.resource_bounds(), transaction.tip())?;
    let paymaster_data_hash =
        HashChain::new().chain_iter(transaction.paymaster_data().0.iter()).get_poseidon_hash();
    let data_availability_mode = concat_data_availability_mode(
        transaction.nonce_data_availability_mode(),
        transaction.fee_data_availability_mode(),
    );
    let account_deployment_data_hash = HashChain::new()
        .chain_iter(transaction.account_deployment_data().0.iter())
        .get_poseidon_hash();
    let calldata_hash =
        HashChain::new().chain_iter(transaction.calldata().0.iter()).get_poseidon_hash();
    let mut hash_chain = HashChain::new()
        .chain(&INVOKE)
        .chain(&transaction_version.0)
        .chain(transaction.sender_address().0.key())
        .chain(&tip_resource_bounds_hash)
        .chain(&paymaster_data_hash)
        .chain(&Felt::try_from(chain_id)?)
        .chain(&transaction.nonce().0)

            HashChain::new().chain_iter(transaction.proof_facts().0.iter()).get_poseidon_hash();

    }

impl TryFrom<protobuf::ResourceBounds> for ValidResourceBounds {
    type Error = ProtobufConversionError;
    fn try_from(value: protobuf::ResourceBounds) -> Result<Self, Self::Error> {
        let Some(l1_gas) = value.l1_gas else {
            return Err(missing("ResourceBounds::l1_gas"));
        };
        let Some(l2_gas) = value.l2_gas else {
            return Err(missing("ResourceBounds::l2_gas"));
        };
        // TODO(Shahak): Assert data gas is not none once we remove support for 0.13.2.
        let l1_data_gas = value.l1_data_gas.unwrap_or_default();
        let l1_gas: ResourceBounds = l1_gas.try_into()?;
        let l2_gas: ResourceBounds = l2_gas.try_into()?;
        let l1_data_gas: ResourceBounds = l1_data_gas.try_into()?;
        Ok(if l1_data_gas.is_zero() && l2_gas.is_zero() {
            ValidResourceBounds::L1Gas(l1_gas)
        } else {
            ValidResourceBounds::AllResources(AllResourceBounds { l1_gas, l2_gas, l1_data_gas })
        })
    }
