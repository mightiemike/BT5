### Title
Division by Zero in `PerpEngine.socializeSubaccount` When `openInterest` Is Zero — (File: `core/contracts/PerpEngine.sol`)

---

### Summary

`PerpEngine.socializeSubaccount` unconditionally divides by `state.openInterest` when spreading residual bad debt across market participants. If `openInterest` is zero at the time of socialization — a reachable condition when the insolvent subaccount is the sole remaining participant — the call reverts with `"DBZ"`, permanently blocking liquidation of that subaccount.

---

### Finding Description

Inside `PerpEngine.socializeSubaccount`, after insurance is applied to a negative `vQuoteBalance`, any remaining shortfall is spread across all open-interest holders via a per-share funding adjustment:

```solidity
// actually socialize if still not enough
if (balance.vQuoteBalance < 0) {
    // socialize across all other participants
    int128 fundingPerShare = -balance.vQuoteBalance.div(
        state.openInterest          // ← no zero-guard
    );
    state.cumulativeFundingLongX18 += fundingPerShare;
    state.cumulativeFundingShortX18 -= fundingPerShare;
    balance.vQuoteBalance = 0;
}
``` [1](#0-0) 

`MathSD21x18.div` explicitly reverts with `"DBZ"` when the divisor is zero:

```solidity
function div(int128 x, int128 y) internal pure returns (int128) {
    unchecked {
        require(y != 0, ERR_DIV_BY_ZERO);
``` [2](#0-1) 

The reachable trigger is:

1. A perp market has a single active participant (or all other participants have closed their positions, reducing `openInterest` to zero).
2. The liquidation flow closes the insolvent subaccount's position first, which decrements `openInterest` to zero.
3. The subaccount still carries a negative `vQuoteBalance` (residual debt) that exceeds available insurance.
4. `socializeSubaccount` is called; `state.openInterest == 0`; the `div` reverts.

The `State` struct initialises `openInterest` at zero when a product is first added:

```solidity
_setState(
    productId,
    State({
        cumulativeFundingLongX18: 0,
        cumulativeFundingShortX18: 0,
        availableSettle: 0,
        openInterest: 0
    })
);
``` [3](#0-2) 

---

### Impact Explanation

When the revert occurs, the entire liquidation transaction is rolled back. The insolvent subaccount cannot be liquidated, its bad debt cannot be cleared, and the protocol accumulates unresolvable bad debt. This is a permanent DoS of the liquidation path for any perp market that reaches zero open interest while an insolvent account still holds a negative `vQuoteBalance`. The impact is directly analogous to the reference report: a critical protocol function is rendered permanently non-functional without redeployment.

---

### Likelihood Explanation

The condition is reachable without any privileged access. A single trader opening a leveraged position in a low-liquidity or newly-launched perp market, followed by adverse price movement that exhausts insurance, is a realistic scenario. The sequencer submits the liquidation transaction through the normal `Endpoint` path; no special permissions are required to trigger the revert.

---

### Recommendation

Add an explicit zero-guard before the division. If `openInterest` is zero there are no other participants to socialize the loss to; the remaining shortfall should be absorbed by the insurance fund (even if it drives insurance negative) or handled as a protocol-level bad-debt write-off:

```solidity
if (balance.vQuoteBalance < 0) {
    if (state.openInterest != 0) {
        int128 fundingPerShare = -balance.vQuoteBalance.div(
            state.openInterest
        );
        state.cumulativeFundingLongX18 += fundingPerShare;
        state.cumulativeFundingShortX18 -= fundingPerShare;
    }
    // If openInterest == 0, absorb residual into insurance or emit an event
    balance.vQuoteBalance = 0;
}
``` [4](#0-3) 

---

### Proof of Concept

1. Deploy the protocol with a single perp market (product ID `P`).
2. A single trader (`user_A`) opens a leveraged long position; `openInterest` becomes non-zero.
3. Price moves sharply against `user_A`; their account becomes insolvent (maintenance health < 0).
4. The sequencer submits a `LiquidateSubaccount` transaction. The liquidation logic closes `user_A`'s position, reducing `openInterest` back to zero.
5. `user_A` still has `vQuoteBalance < 0` exceeding the insurance fund.
6. `ClearinghouseLiq` calls `PerpEngine.socializeSubaccount(user_A, remainingInsurance)`.
7. Inside `socializeSubaccount`, `balance.vQuoteBalance < 0` after insurance, so the code reaches `fundingPerShare = -balance.vQuoteBalance.div(state.openInterest)` where `state.openInterest == 0`.
8. `MathSD21x18.div` reverts with `"DBZ"`.
9. The entire liquidation transaction reverts. `user_A`'s insolvent account is permanently un-liquidatable. [5](#0-4)
