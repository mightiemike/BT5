### Title
Protobuf `ValidResourceBounds` Deserialization Silently Downgrades `AllResources` to `L1Gas` When l2/data-gas Are Zero, Causing Consensus Proposal Rejection - (File: crates/apollo_protobuf/src/converters/transaction.rs)

### Summary

The `TryFrom<protobuf::ResourceBounds> for ValidResourceBounds` conversion uses a zero-value heuristic to distinguish old-format (`L1Gas`) from new-format (`AllResources`) resource bounds. When a V3 invoke transaction carries `AllResourceBounds` with both `l2_gas` and `l1_data_gas` set to zero, the protobuf round-trip silently converts it to `ValidResourceBounds::L1Gas`. The downstream `RpcInvokeTransactionV3::try_from(InvokeTransactionV3)` then hard-rejects any non-`AllResources` variant, causing the receiving validator to mark the entire consensus proposal as `Failed`.

### Finding Description

**Step 1 – Submission and hash computation.**

`RpcInvokeTransactionV3` always carries `AllResourceBounds` (not `ValidResourceBounds`). A user may submit a transaction with all-zero `l2_gas` and `l1_data_gas` (e.g., `AllResourceBounds::default()`). The gateway converts it to `InternalRpcInvokeTransactionV3` and computes the hash via `get_invoke_transaction_v3_hash`, which calls `get_tip_resource_bounds_hash` with `ValidResourceBounds::AllResources(...)`. For `AllResources`, this function appends a fourth element — `concat(L1_DATA_GAS, 0)` — to the Poseidon input, producing hash **H₁**. [1](#0-0) [2](#0-1) 

**Step 2 – Protobuf serialization.**

When the proposer sends the transaction via P2P consensus, `convert_internal_consensus_tx_to_consensus_tx` converts `InternalRpcInvokeTransactionV3` → `RpcInvokeTransactionV3` → `InvokeTransactionV3` → `protobuf::InvokeV3`. The protobuf wire format carries `resource_bounds` with all-zero `l2_gas` and `l1_data_gas` fields. [3](#0-2) 

**Step 3 – Protobuf deserialization and type collapse.**

On the receiving validator, `TryFrom<protobuf::ResourceBounds> for ValidResourceBounds` applies this heuristic:

```rust
Ok(if l1_data_gas.is_zero() && l2_gas.is_zero() {
    ValidResourceBounds::L1Gas(l1_gas)   // ← wrong branch for AllResources with zero values
} else {
    ValidResourceBounds::AllResources(AllResourceBounds { l1_gas, l2_gas, l1_data_gas })
})
```

Because both fields are zero, the result is `ValidResourceBounds::L1Gas` — the pre-0.13.3 variant — even though the original transaction was a post-0.13.3 `AllResources` transaction. [4](#0-3) 

**Step 4 – Hard rejection in `RpcInvokeTransactionV3::try_from`.**

The deserialized `InvokeTransactionV3` (now carrying `L1Gas`) is then converted to `RpcInvokeTransactionV3`:

```rust
resource_bounds: match value.resource_bounds {
    ValidResourceBounds::AllResources(bounds) => bounds,
    _ => {
        return Err(StarknetApiError::OutOfRange { string: "resource_bounds".to_string() });
    }
},
```

This returns `DEPRECATED_RESOURCE_BOUNDS_ERROR` (`MissingField { field_description: "ResourceBounds::l1_data_gas" }`). [5](#0-4) [6](#0-5) 

**Step 5 – Proposal marked `Failed`.**

`convert_consensus_tx_to_internal_consensus_tx` propagates the error. The validator's proposal handler catches it and returns `HandledProposalPart::Failed(...)`, rejecting the entire block proposal. [7](#0-6) [8](#0-7) 

**The hash domain divergence (analog to the external bug).**

`get_tip_resource_bounds_hash` produces structurally different Poseidon inputs for the two variants:

| Variant | Elements hashed |
|---|---|
| `L1Gas` | `[tip, pack(L1_GAS, l1_gas), pack(L2_GAS, 0)]` — 3 elements |
| `AllResources` | `[tip, pack(L1_GAS, l1_gas), pack(L2_GAS, 0), pack(L1_DATA_GAS, 0)]` — 4 elements |

The `l1_data_gas` term is present in the original hash **H₁** (computed from `AllResources`) but absent from any hash that would be computed from the deserialized `L1Gas` form. This is the direct analog of the external report: a cost component (`l1_data_gas`) that should be included in the canonical computation is silently dropped at the serialization boundary. [9](#0-8) 

The developers are aware of the ambiguity in tests — `add_gas_values_to_transaction` explicitly sets `l2_gas.max_amount = GasAmount(1)` to avoid it — but no equivalent guard exists in the production consensus path. [10](#0-9) 

### Impact Explanation

A user submitting a V3 invoke transaction with `AllResourceBounds { l1_gas: {0,0}, l2_gas: {0,0}, l1_data_gas: {0,0} }` (e.g., `AllResourceBounds::default()`) is accepted by the gateway (zero bounds → no fee enforcement, no balance check). When the proposer includes this transaction in a block and broadcasts it via P2P consensus, every validator rejects the proposal with `DEPRECATED_RESOURCE_BOUNDS_ERROR`. The proposer retries with the same mempool transaction, causing repeated consensus failures. This is a **High** impact: "Transaction conversion or signature/hash logic binds the wrong signer, hash, type, or executable payload" and "Mempool/gateway/RPC admission accepts valid transactions [that are] rejected before sequencing."

### Likelihood Explanation

Any unprivileged user can trigger this by submitting a single V3 invoke transaction with zero `l2_gas` and `l1_data_gas` bounds. No special knowledge is required; `AllResourceBounds::default()` satisfies the condition. The gateway has no stateless or stateful check that rejects zero-bound V3 transactions.

### Recommendation

Replace the zero-value heuristic in `TryFrom<protobuf::ResourceBounds> for ValidResourceBounds` with an explicit discriminator field in the protobuf schema (e.g., a `bool is_all_resources` flag or a `oneof` variant), so that `AllResources` with zero values is unambiguously preserved across the wire. Until the schema is updated, add a gateway-level check that rejects V3 transactions where `l2_gas` and `l1_data_gas` are both zero, or add a pre-send guard in the proposer's P2P serialization path that sets a sentinel non-zero value (as the test helper already does).

### Proof of Concept

```
1. Submit RpcInvokeTransactionV3 with:
     resource_bounds = AllResourceBounds {
         l1_gas:      { max_amount: 1, max_price_per_unit: 1 },
         l2_gas:      { max_amount: 0, max_price_per_unit: 0 },  // zero
         l1_data_gas: { max_amount: 0, max_price_per_unit: 0 },  // zero
     }
   Gateway accepts; hash H1 computed with AllResources (4-element Poseidon input).

2. Proposer includes the transaction in a block proposal and serializes to protobuf:
     InvokeV3 { resource_bounds: ResourceBounds { l1_gas: {1,1}, l2_gas: {0,0}, l1_data_gas: {0,0} } }

3. Validator deserializes:
     ValidResourceBounds::try_from(ResourceBounds { l2_gas.is_zero() && l1_data_gas.is_zero() })
     → ValidResourceBounds::L1Gas({1,1})   // wrong branch

4. RpcInvokeTransactionV3::try_from(InvokeTransactionV3 { resource_bounds: L1Gas(...) })
     → Err(OutOfRange { "resource_bounds" })   // hard rejection

5. convert_consensus_tx_to_internal_consensus_tx returns Err(DEPRECATED_RESOURCE_BOUNDS_ERROR)

6. validate_proposal returns HandledProposalPart::Failed(...)
   → Proposal rejected; consensus stalls for this round.
```

pub fn get_tip_resource_bounds_hash(
    resource_bounds: &ValidResourceBounds,
    tip: &Tip,
) -> Result<Felt, StarknetApiError> {
    let l1_resource_bounds = resource_bounds.get_l1_bounds();
    let l2_resource_bounds = resource_bounds.get_l2_bounds();

    // L1 and L2 gas bounds always exist.
    // Old V3 txs always have L2 gas bounds of zero, but they exist.
    let mut resource_felts = vec![
        get_concat_resource(&l1_resource_bounds, L1_GAS)?,
        get_concat_resource(&l2_resource_bounds, L2_GAS)?,
    ];

    // For new V3 txs, need to also hash the data gas bounds.
    resource_felts.extend(match resource_bounds {
        ValidResourceBounds::L1Gas(_) => vec![],
        ValidResourceBounds::AllResources(all_resources) => {
            vec![get_concat_resource(&all_resources.l1_data_gas, L1_DATA_GAS)?]
        }
    });

    Ok(HashChain::new().chain(&tip.0.into()).chain_iter(resource_felts.iter()).get_poseidon_hash())

pub(crate) fn get_invoke_transaction_v3_hash<T: InvokeTransactionV3Trait>(
    transaction: &T,
    chain_id: &ChainId,
    transaction_version: &TransactionVersion,
) -> Result<TransactionHash, StarknetApiError> {
    let tip_resource_bounds_hash =
        get_tip_resource_bounds_hash(&transaction.resource_bounds(), transaction.tip())?;
    let paymaster_data_hash =
        HashChain::new().chain_iter(transaction.paymaster_data().0.iter()).get_poseidon_hash();
    let data_availability_mode = concat_data_availability_mode(
        transaction.nonce_data_availability_mode(),
        transaction.fee_data_availability_mode(),
    );
    let account_deployment_data_hash = HashChain::new()
        .chain_iter(transaction.account_deployment_data().0.iter())
        .get_poseidon_hash();
    let calldata_hash =
        HashChain::new().chain_iter(transaction.calldata().0.iter()).get_poseidon_hash();
    let mut hash_chain = HashChain::new()
        .chain(&INVOKE)
        .chain(&transaction_version.0)
        .chain(transaction.sender_address().0.key())
        .chain(&tip_resource_bounds_hash)
        .chain(&paymaster_data_hash)
        .chain(&Felt::try_from(chain_id)?)
        .chain(&transaction.nonce().0)
        .chain(&data_availability_mode)
        .chain(&account_deployment_data_hash)

        let proof_facts_hash =
            HashChain::new().chain_iter(transaction.proof_facts().0.iter()).get_poseidon_hash();

    }
    Ok(TransactionHash(hash_chain.get_poseidon_hash()))

impl TryFrom<protobuf::ResourceBounds> for ValidResourceBounds {
    type Error = ProtobufConversionError;
    fn try_from(value: protobuf::ResourceBounds) -> Result<Self, Self::Error> {
        let Some(l1_gas) = value.l1_gas else {
            return Err(missing("ResourceBounds::l1_gas"));
        };
        let Some(l2_gas) = value.l2_gas else {
            return Err(missing("ResourceBounds::l2_gas"));
        };
        // TODO(Shahak): Assert data gas is not none once we remove support for 0.13.2.
        let l1_data_gas = value.l1_data_gas.unwrap_or_default();
        let l1_gas: ResourceBounds = l1_gas.try_into()?;
        let l2_gas: ResourceBounds = l2_gas.try_into()?;
        let l1_data_gas: ResourceBounds = l1_data_gas.try_into()?;
        Ok(if l1_data_gas.is_zero() && l2_gas.is_zero() {
            ValidResourceBounds::L1Gas(l1_gas)
        } else {
            ValidResourceBounds::AllResources(AllResourceBounds { l1_gas, l2_gas, l1_data_gas })
        })
    }
