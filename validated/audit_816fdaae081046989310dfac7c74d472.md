### Title
Protobuf Receipt Serialization Always Emits `price_unit = 0` (Wei) for All Transactions, Including V3 (STRK) Transactions, Due to Uncalled Mandatory Fix Function — (`crates/apollo_protobuf/src/converters/receipt.rs`)

---

### Summary

Every `From<*TransactionOutput>` implementation for protobuf receipt types hardcodes `price_unit: 0` (Wei/Unspecified). A separate function `set_price_unit_based_on_transaction` exists to correct this field, but it is **never called** anywhere in production code. As a result, every receipt serialized for P2P block sync carries the wrong fee denomination for V3 transactions (which should be `Fri`/STRK), producing an authoritative-looking wrong value on the receiving peer.

---

### Finding Description

`create_proto_receipt_common_from_txn_output_fields` always sets `price_unit: 0`: [1](#0-0) 

This helper is called by every `From<*TransactionOutput>` impl. Each of those impls carries an explicit doc comment acknowledging the defect: [2](#0-1) [3](#0-2) [4](#0-3) [5](#0-4) 

The intended fix is `set_price_unit_based_on_transaction`, which maps transaction version to the correct `PriceUnit` (`Wei` for V1/V2/Deploy/L1Handler, `Fri` for V3): [6](#0-5) 

A `grep` across the entire repository finds this function referenced only in its own definition and in the doc-comment strings inside `receipt.rs`. It is **never invoked** in any production call site. The `From<TransactionOutput> for protobuf::Receipt` path — used directly by the P2P sync layer — therefore always emits `price_unit = 0`: [7](#0-6) 

The protobuf `PriceUnit` enum value `0` decodes as `Unspecified`/Wei. For V3 transactions (Invoke V3, Declare V3, DeployAccount V3) the correct value is `Fri` (STRK). The receiving peer therefore reads every V3 receipt as having been paid in ETH, not STRK.

---

### Impact Explanation

When a syncing node receives a block via P2P, it obtains receipts with `price_unit = Wei` for all transactions. For V3 transactions the fee was actually paid in STRK (Fri). Any downstream consumer of the synced receipt — RPC `starknet_getTransactionReceipt`, fee estimation cross-checks, or receipt commitment validation — that trusts the `price_unit` field will observe the wrong denomination. This is a wrong receipt value served from an authoritative-looking source, matching:

> **High. RPC execution, fee estimation, tracing, simulation, or pending view returns an authoritative-looking wrong value.**

---

### Likelihood Explanation

Every V3 transaction (the current and only supported transaction version for new transactions) produces a receipt with the wrong `price_unit` when serialized for P2P sync. The trigger is any block containing a V3 transaction being synced over P2P, which is the normal operating condition of the network. No special attacker input is required; the bug fires unconditionally on every V3 receipt serialization.

---

### Recommendation

Call `set_price_unit_based_on_transaction` immediately after constructing the `protobuf::Receipt` from a `TransactionOutput`, passing the corresponding `protobuf::TransactionInBlock`. Alternatively, refactor `create_proto_receipt_common_from_txn_output_fields` to accept the transaction version directly and set `price_unit` correctly at construction time, eliminating the two-step pattern that is trivially forgotten.

---

### Proof of Concept

1. Serialize any `InvokeTransactionOutput` (from a V3 invoke) to `protobuf::Receipt`:
   ```rust
   let receipt: protobuf::Receipt = TransactionOutput::Invoke(invoke_v3_output).into();
   // receipt.type.invoke.common.price_unit == 0  (Wei/Unspecified)
   // Expected: protobuf::PriceUnit::Fri (2)
   ```
2. `set_price_unit_based_on_transaction` is never called, so the value stays `0`.
3. A peer receiving this receipt via P2P decodes `price_unit = 0` as Wei, interprets the STRK fee amount as an ETH amount, and returns a wrong `FEE_PAYMENT.unit = "WEI"` in `starknet_getTransactionReceipt` for every V3 transaction it learned through sync. [8](#0-7)

fn create_proto_receipt_common_from_txn_output_fields(
    actual_fee: Fee,
    messages_sent: Vec<MessageToL1>,
    execution_resources: ExecutionResources,
    execution_status: TransactionExecutionStatus,
) -> protobuf::receipt::Common {
    let actual_fee = Felt::from(actual_fee).into();
    let messages_sent = messages_sent.into_iter().map(protobuf::MessageToL1::from).collect();
    let execution_resources = execution_resources.into();
    let revert_reason =
        if let TransactionExecutionStatus::Reverted(reverted_status) = execution_status {
            Some(reverted_status.revert_reason)
        } else {
            None
        };
    protobuf::receipt::Common {
        actual_fee: Some(actual_fee),
        price_unit: 0,
        messages_sent,
        execution_resources: Some(execution_resources),
        revert_reason,
    }
}

pub fn set_price_unit_based_on_transaction(
    receipt: &mut protobuf::Receipt,
    transaction: &protobuf::TransactionInBlock,
) {
    let price_unit = match &transaction.txn {
        Some(protobuf::transaction_in_block::Txn::DeclareV1(_)) => protobuf::PriceUnit::Wei,
        Some(protobuf::transaction_in_block::Txn::DeclareV2(_)) => protobuf::PriceUnit::Wei,
        Some(protobuf::transaction_in_block::Txn::DeclareV3(_)) => protobuf::PriceUnit::Fri,
        Some(protobuf::transaction_in_block::Txn::Deploy(_)) => protobuf::PriceUnit::Wei,
        Some(protobuf::transaction_in_block::Txn::DeployAccountV1(_)) => protobuf::PriceUnit::Wei,
        Some(protobuf::transaction_in_block::Txn::DeployAccountV3(_)) => protobuf::PriceUnit::Fri,
        Some(protobuf::transaction_in_block::Txn::InvokeV1(_)) => protobuf::PriceUnit::Wei,
        Some(protobuf::transaction_in_block::Txn::InvokeV3(_)) => protobuf::PriceUnit::Fri,
        Some(protobuf::transaction_in_block::Txn::L1Handler(_)) => protobuf::PriceUnit::Wei,
        _ => return,
    };
    let Some(ref mut receipt_type) = receipt.r#type else {
        return;
    };

    let common = match receipt_type {
        protobuf::receipt::Type::Invoke(invoke) => invoke.common.as_mut(),
        protobuf::receipt::Type::L1Handler(l1_handler) => l1_handler.common.as_mut(),
        protobuf::receipt::Type::Declare(declare) => declare.common.as_mut(),
        protobuf::receipt::Type::DeprecatedDeploy(deploy) => deploy.common.as_mut(),
        protobuf::receipt::Type::DeployAccount(deploy_account) => deploy_account.common.as_mut(),
    };

    if let Some(common) = common {
        common.price_unit = price_unit.into();
    }
