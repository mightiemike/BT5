### Title
`SwapAllowlistExtension` Checks Router Address Instead of End-User — Allowlist Fully Bypassed via `MetricOmmSimpleRouter` - (File: `metric-periphery/contracts/extensions/SwapAllowlistExtension.sol`)

---

### Summary

`SwapAllowlistExtension.beforeSwap` gates on the `sender` argument, which the pool sets to `msg.sender` of `pool.swap()`. When a user routes through `MetricOmmSimpleRouter`, the pool's `msg.sender` is the **router contract**, not the end user. The extension therefore checks `allowedSwapper[pool][router]` instead of `allowedSwapper[pool][user]`. If the router is allowlisted (a natural operational step to make the router usable on a curated pool), every unpermissioned user can bypass the per-user allowlist by calling any router entry-point.

---

### Finding Description

**Call chain for `exactInputSingle`:**

1. User calls `MetricOmmSimpleRouter.exactInputSingle(params)` — `msg.sender` = user.
2. Router calls `IMetricOmmPoolActions(params.pool).swap(params.recipient, ...)` — pool's `msg.sender` = **router**.
3. Pool calls `_beforeSwap(msg.sender, recipient, ...)` — `sender` argument = **router**.
4. `ExtensionCalling._beforeSwap` encodes `sender = router` and dispatches to the extension.
5. `SwapAllowlistExtension.beforeSwap` evaluates `allowedSwapper[msg.sender][sender]` = `allowedSwapper[pool][router]`.

The extension never sees the end user's address. The same mis-binding applies to `exactInput` (all hops), `exactOutputSingle`, and `exactOutput`.

**Relevant code:**

`MetricOmmPool.swap` passes `msg.sender` as `sender`: [1](#0-0) 

`ExtensionCalling._beforeSwap` forwards that `sender` verbatim to the extension: [2](#0-1) 

`SwapAllowlistExtension.beforeSwap` checks `allowedSwapper[msg.sender][sender]` — `msg.sender` is the pool, `sender` is whoever called `pool.swap()` (the router): [3](#0-2) 

`MetricOmmSimpleRouter.exactInputSingle` calls `pool.swap()` directly, making the router the pool's `msg.sender`: [4](#0-3) 

For multi-hop `exactInput`, the same holds for every hop — the router is always the direct caller of each pool: [5](#0-4) 

---

### Impact Explanation

**Allowlist bypass (High):** A pool admin configures `SwapAllowlistExtension` to restrict swaps to KYC'd or otherwise curated addresses. To make the router usable on that pool at all, the admin must call `setAllowedToSwap(pool, router, true)`. Once the router is allowlisted, every unpermissioned user can call `router.exactInputSingle(...)` and the extension passes — `allowedSwapper[pool][router]` is `true`. The per-user curation is completely defeated.

**Broken core functionality (High):** If the admin does *not* allowlist the router, then even legitimately allowlisted users cannot swap through the router (the extension sees `sender = router`, which is not in the allowlist). The pool's primary public swap path is broken for all router users.

Both outcomes are fund-impacting: either unauthorized users drain a curated pool, or legitimate LPs cannot exit positions because the swap path is unusable.

---

### Likelihood Explanation

The `MetricOmmSimpleRouter` is the canonical public swap entry-point documented and deployed for all pools. Any pool that enables `SwapAllowlistExtension` and also wants router support will hit this immediately. The trigger requires only a standard `exactInputSingle` call — no special permissions, no flash loans, no price manipulation.

---

### Recommendation

Pass the **originating user** through the swap call so the extension can gate on the correct actor. Two options:

1. **Add a `payer` / `originator` field to `pool.swap()`** that the router populates with `msg.sender` before delegating to the pool. The extension then checks this field instead of `sender`.
2. **Gate on `recipient` instead of `sender`** if the pool's security model is output-side (less common). This still does not solve the general case.
3. **Alternatively, enforce the allowlist at the router level** — the router reads the pool's configured allowlist provider and checks `msg.sender` before forwarding. This keeps the pool interface unchanged but requires the router to be allowlist-aware.

The `DepositAllowlistExtension` does **not** share this bug: it checks `owner` (the explicit position-owner argument), which the pool passes through from the caller's parameter, not from `msg.sender`. [6](#0-5) 

---

### Proof of Concept

```
Setup:
  - Deploy pool with SwapAllowlistExtension configured as beforeSwap hook
  - Pool admin calls setAllowedToSwap(pool, router, true)   // router must be allowed for any router swap to work
  - Pool admin does NOT call setAllowedToSwap(pool, attacker, true)

Attack:
  - attacker calls router.exactInputSingle({pool: pool, ...})
  - Router calls pool.swap(recipient, ...)  →  pool.msg.sender = router
  - Pool calls _beforeSwap(router, recipient, ...)
  - SwapAllowlistExtension checks allowedSwapper[pool][router] == true  →  PASSES
  - Swap executes for attacker despite attacker not being on the allowlist

Result:
  - attacker successfully swaps on a curated pool
  - SwapAllowlistExtension provides zero per-user protection for any router-mediated swap
```
