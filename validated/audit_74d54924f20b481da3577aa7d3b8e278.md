### Title
`ValidResourceBounds::AllResources` with Zero L2/L1DataGas Silently Collapses to `L1Gas` in Protobuf Round-Trip, Producing a Different Transaction Hash - (`File: crates/apollo_protobuf/src/converters/transaction.rs`)

### Summary

The protobuf deserializer for `ValidResourceBounds` uses a value-based heuristic to decide which variant to reconstruct. When an `AllResources` transaction carries zero `l2_gas` and zero `l1_data_gas`, the round-trip through protobuf silently produces `ValidResourceBounds::L1Gas` instead of `ValidResourceBounds::AllResources`. Because `get_tip_resource_bounds_hash` hashes a different number of resource-bound felts for each variant (2 for `L1Gas`, 3 for `AllResources`), the transaction hash computed after deserialization diverges from the hash computed at submission time. Any node that recomputes or verifies the hash after P2P delivery will reject the transaction.

### Finding Description

**Step 1 – Submission (RPC → Internal).**
`RpcInvokeTransactionV3` carries `resource_bounds: AllResourceBounds`, so every new transaction enters the system as `ValidResourceBounds::AllResources`. A user may legitimately set `l2_gas` and `l1_data_gas` to zero while keeping `l1_gas` non-zero. The gateway accepts this, and `convert_rpc_tx_to_internal` computes the canonical hash using the `AllResources` path. [1](#0-0) 

**Step 2 – Hash computation for `AllResources`.**
`get_tip_resource_bounds_hash` appends the `L1_DATA_GAS` felt only for `AllResources`, producing a 4-element Poseidon input `[tip, L1_GAS, L2_GAS, L1_DATA_GAS]`. [2](#0-1) 

For `L1Gas` it produces a 3-element input `[tip, L1_GAS, L2_GAS]` — a structurally different preimage.

**Step 3 – Protobuf serialization is non-injective.**
`From<ValidResourceBounds> for protobuf::ResourceBounds` serializes `L1Gas` with `l1_data_gas = Some(ResourceBounds::default())` (all zeros). It serializes `AllResources` with zero `l2_gas`/`l1_data_gas` identically — both produce the same wire bytes. [3](#0-2) 

**Step 4 – Deserialization collapses the variant.**
`TryFrom<protobuf::ResourceBounds> for ValidResourceBounds` reconstructs `L1Gas` whenever both `l2_gas` and `l1_data_gas` are zero, regardless of which variant was originally serialized. [4](#0-3) 

The comment `// TODO(Shahak): Assert data gas is not none once we remove support for 0.13.2.` confirms this is a known transitional gap, but the collapse condition is applied unconditionally even for post-0.13.3 `AllResources` transactions.

**Step 5 – Hash divergence.**
After deserialization the receiving node holds `ValidResourceBounds::L1Gas`. When it recomputes the transaction hash (e.g., in `InternalRpcTransactionWithoutTxHash::calculate_transaction_hash`), it calls `get_tip_resource_bounds_hash` with the `L1Gas` branch, producing a 3-element Poseidon hash that differs from the 4-element hash computed at submission. [5](#0-4) 

### Impact Explanation

A valid transaction accepted by the gateway is permanently undeliverable via P2P: every peer that recomputes the hash after protobuf deserialization will compute a different value and reject the transaction. This is a **High** impact: valid transactions are rejected before sequencing, matching the scope "Mempool/gateway/RPC admission… rejects valid transactions before sequencing."

### Likelihood Explanation

Any user who submits a V3 `AllResources` transaction with `l2_gas = 0` and `l1_data_gas = 0` (only `l1_gas` non-zero) triggers this path. The gateway stateless validator explicitly allows this combination. [6](#0-5) 

No special privilege is required; the condition is reachable by any unprivileged user.

### Recommendation

Replace the value-based heuristic in the protobuf deserializer with an explicit discriminant field (e.g., a boolean `is_all_resources` or an enum tag in the protobuf schema). Until the schema is updated, the deserializer should default to `AllResources` whenever `l2_gas` is present in the wire message, reserving `L1Gas` only for messages that explicitly omit `l2_gas` (the pre-0.13.3 wire format). This preserves the canonicalization invariant that the variant chosen at hash-computation time survives the protobuf round-trip unchanged.

### Proof of Concept

1. Submit an `RpcInvokeTransactionV3` with `resource_bounds = AllResourceBounds { l1_gas: X, l2_gas: 0, l1_data_gas: 0 }`.
2. Gateway accepts it; hash `H_A` is computed via `get_tip_resource_bounds_hash` with 4 Poseidon elements.
3. Serialize the internal transaction to protobuf via `From<ValidResourceBounds> for protobuf::ResourceBounds`.
4. Deserialize on a peer: `TryFrom<protobuf::ResourceBounds>` returns `ValidResourceBounds::L1Gas(X)`.
5. Peer recomputes hash `H_B` via `get_tip_resource_bounds_hash` with 3 Poseidon elements.
6. `H_A ≠ H_B`; the peer rejects the transaction as having an invalid hash, even though it was legitimately accepted by the originating node.

pub fn get_tip_resource_bounds_hash(
    resource_bounds: &ValidResourceBounds,
    tip: &Tip,
) -> Result<Felt, StarknetApiError> {
    let l1_resource_bounds = resource_bounds.get_l1_bounds();
    let l2_resource_bounds = resource_bounds.get_l2_bounds();

    // L1 and L2 gas bounds always exist.
    // Old V3 txs always have L2 gas bounds of zero, but they exist.
    let mut resource_felts = vec![
        get_concat_resource(&l1_resource_bounds, L1_GAS)?,
        get_concat_resource(&l2_resource_bounds, L2_GAS)?,
    ];

    // For new V3 txs, need to also hash the data gas bounds.
    resource_felts.extend(match resource_bounds {
        ValidResourceBounds::L1Gas(_) => vec![],
        ValidResourceBounds::AllResources(all_resources) => {
            vec![get_concat_resource(&all_resources.l1_data_gas, L1_DATA_GAS)?]
        }
    });

    Ok(HashChain::new().chain(&tip.0.into()).chain_iter(resource_felts.iter()).get_poseidon_hash())
}

impl TryFrom<protobuf::ResourceBounds> for ValidResourceBounds {
    type Error = ProtobufConversionError;
    fn try_from(value: protobuf::ResourceBounds) -> Result<Self, Self::Error> {
        let Some(l1_gas) = value.l1_gas else {
            return Err(missing("ResourceBounds::l1_gas"));
        };
        let Some(l2_gas) = value.l2_gas else {
            return Err(missing("ResourceBounds::l2_gas"));
        };
        // TODO(Shahak): Assert data gas is not none once we remove support for 0.13.2.
        let l1_data_gas = value.l1_data_gas.unwrap_or_default();
        let l1_gas: ResourceBounds = l1_gas.try_into()?;
        let l2_gas: ResourceBounds = l2_gas.try_into()?;
        let l1_data_gas: ResourceBounds = l1_data_gas.try_into()?;
        Ok(if l1_data_gas.is_zero() && l2_gas.is_zero() {
            ValidResourceBounds::L1Gas(l1_gas)
        } else {
            ValidResourceBounds::AllResources(AllResourceBounds { l1_gas, l2_gas, l1_data_gas })
        })
    }
