Audit Report

## Title
Swap Allowlist Bypassed via Router — Any User Can Trade on Curated Pools Through `MetricOmmSimpleRouter` - (File: `metric-periphery/contracts/extensions/SwapAllowlistExtension.sol`)

## Summary

`SwapAllowlistExtension.beforeSwap` gates swaps by checking the `sender` argument forwarded from the pool, which equals `msg.sender` of `pool.swap()`. When a user routes through `MetricOmmSimpleRouter`, the pool's `msg.sender` is the router contract address, not the end user. Any pool admin who allowlists the router — a necessary operational step for allowlisted users to trade via the router — simultaneously opens the pool to every non-allowlisted user who routes through the same contract.

## Finding Description

`MetricOmmPool.swap` calls `_beforeSwap(msg.sender, ...)` at line 231, forwarding the immediate caller as `sender` to all configured extensions. [1](#0-0) 

`ExtensionCalling._beforeSwap` encodes `sender` and passes it verbatim to every extension via `_callExtensionsInOrder`. [2](#0-1) 

`SwapAllowlistExtension.beforeSwap` checks `allowedSwapper[msg.sender][sender]`, where `msg.sender` is the pool and `sender` is the argument received — i.e., the router address when the call originates from `MetricOmmSimpleRouter`. [3](#0-2) 

`MetricOmmSimpleRouter.exactInputSingle` calls `pool.swap(...)` directly, making the router the `msg.sender` at the pool level. The actual end user (`msg.sender` of `exactInputSingle`) is never forwarded to the pool or extension. [4](#0-3) 

The `IMetricOmmExtensions.beforeSwap` interface provides no separate field for the originating EOA; `sender` is the only identity field available to the extension. [5](#0-4) 

The same bypass applies to `exactOutputSingle`, `exactInput`, and `exactOutput` — all router entry points call `pool.swap` directly with the router as `msg.sender`. [6](#0-5) 

## Impact Explanation

A pool admin configures `SwapAllowlistExtension` to restrict trading to a curated set of users (e.g., KYC-verified addresses, institutional counterparties). To let those users trade via the supported periphery router, the admin must call `setAllowedToSwap(pool, router, true)`. Once the router is allowlisted, every user — including those explicitly excluded — can bypass the restriction by calling any router entry point. The curated pool's entire access-control boundary collapses to a single shared router address. This is a broken core pool functionality and admin-boundary break: an unprivileged path (public router call) bypasses a pool admin's access control, enabling unauthorized swaps on pools designed to prevent them. [7](#0-6) 

## Likelihood Explanation

The trigger requires no special privilege. Any public user can call the router. The precondition — the router being allowlisted — is a necessary operational step for any allowlisted user to use the router at all, so it is expected to be present on any production curated pool that supports router-based trading. The bypass is therefore reachable on every such pool without any additional setup, and is repeatable indefinitely.

## Recommendation

`SwapAllowlistExtension` must gate the originating user, not the immediate `msg.sender` of `pool.swap`. Two complementary fixes:

1. **Pass the originating user through the router.** `MetricOmmSimpleRouter` should forward the caller's address as a dedicated field in `extensionData`, and `SwapAllowlistExtension.beforeSwap` should decode and check that field instead of (or in addition to) `sender` when `sender` is a known router.

2. **Maintain a trusted-router registry in the extension.** When `sender` is a registered trusted router, require the originating user identity to be embedded and verified in `extensionData`. When `sender` is not a router, check `sender` directly as today.

Without one of these changes, any allowlist-protected pool that supports router-mediated swaps is effectively unprotected.

## Proof of Concept

```
Setup:
  pool configured with SwapAllowlistExtension
  allowedSwapper[pool][alice] = true          // alice is the intended user
  allowedSwapper[pool][router] = true         // admin must set this for alice to use the router

Attack:
  bob (not allowlisted) calls:
    MetricOmmSimpleRouter.exactInputSingle({pool: pool, ...})
      → pool.swap(msg.sender=router, recipient=bob, ...)
        → _beforeSwap(sender=router, ...)
          → SwapAllowlistExtension.beforeSwap(sender=router, ...)
            → allowedSwapper[pool][router] == true  ✓  (passes)
        → swap executes for bob

Result: bob swaps on a pool he is not allowlisted for.
```

Foundry test plan: deploy pool with `SwapAllowlistExtension`, allowlist only `alice` and the router, call `exactInputSingle` from `bob`, assert the swap succeeds and bob receives output tokens despite not being individually allowlisted.
