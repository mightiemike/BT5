Audit Report

## Title
Pool Admin Can Reduce Timelock to Zero and Immediately Execute Drawdown Weakening — (`metric-periphery/contracts/extensions/OracleValueStopLossExtension.sol`)

## Summary

`OracleValueStopLossExtension` enforces a per-pool timelock so LPs can react to drawdown/decay/watermark changes. Because neither `initialize` nor `proposeOracleStopLossTimelock` validates a minimum timelock value, the pool admin can set `timelock = 0`, after which every subsequent propose+execute pair can be batched in a single transaction. This lets the admin raise `drawdownE6` to `E6` (100%) atomically, disabling the stop-loss entirely with zero LP reaction time — a direct bypass of the sole LP protection mechanism.

## Finding Description

`_afterTimelock` computes the execution timestamp as:

```solidity
// L297-299
function _afterTimelock(address pool_) private view returns (uint32) {
    return uint32(block.timestamp + oracleStopLossConfig[pool_].timelock);
}
``` [1](#0-0) 

When `timelock == 0`, this returns exactly `block.timestamp`. `_requireElapsed` then checks:

```solidity
// L301-303
function _requireElapsed(uint32 executeAfter) private view {
    if (block.timestamp < executeAfter) revert ...;
}
``` [2](#0-1) 

`block.timestamp < block.timestamp` is always `false`, so the revert never fires. Propose and execute can be called in the same block (or same transaction).

`initialize` accepts any `uint32 timelock` with no lower-bound check — only `_validateDrawdown` and `_validateDecay` are called: [3](#0-2) 

`proposeOracleStopLossTimelock` similarly accepts any `uint32 newTimelock` with no floor: [4](#0-3) 

**Attack sequence:**

1. Admin calls `proposeOracleStopLossTimelock(pool, 0)` — `pendingTimelockExecuteAfter = block.timestamp + T`.
2. Admin waits `T` seconds (one-time cost).
3. Admin calls `executeOracleStopLossTimelock(pool)` — `oracleStopLossConfig[pool].timelock = 0`.
4. Admin calls `proposeOracleStopLossDrawdown(pool, 1_000_000)` — `pendingDrawdownExecuteAfter = block.timestamp + 0 = block.timestamp`.
5. Admin calls `executeOracleStopLossDrawdown(pool)` in the **same block/transaction** — `_requireElapsed(block.timestamp)` passes, drawdown is set to `E6`. [5](#0-4) 

With `drawdownE6 = E6 = 1e6`, `floorMultiplier = E6 - drawdown = 0`, so the breach condition `metric < (hwm * 0) / E6` reduces to `metric < 0`, which is always `false` for `uint256`. The stop-loss never triggers again. [6](#0-5) 

## Impact Explanation

The NatDoc explicitly states the invariant: *"Drawdown and decay changes are timelocked so LPs can react."* [7](#0-6) 

Once `drawdownE6 = E6`, the stop-loss guard is permanently disabled: every swap that would have been blocked by `OracleStopLossTriggered` now executes freely, allowing value to drain from the pool without LP recourse. This is a direct admin-boundary break — the pool admin exceeds the intended constraint by eliminating the timelock that was the sole LP protection mechanism, enabling direct loss of LP principal.

## Likelihood Explanation

The pool admin is a real, non-factory role assigned per pool. The attack requires only a malicious or compromised admin. The one-time cost (waiting the initial timelock `T`) is observable on-chain, but LPs who do not monitor continuously will miss the subsequent same-block drawdown change. Steps 4 and 5 can be batched in a single multicall transaction, making front-running by LPs practically impossible. The `onlyPoolAdmin` modifier confirms this path is reachable: [8](#0-7) 

## Recommendation

Add a minimum timelock floor enforced in both `initialize` and `proposeOracleStopLossTimelock`:

```solidity
uint32 private constant MIN_TIMELOCK = 1 hours;

function _validateTimelock(uint32 timelock) private pure {
    if (timelock < MIN_TIMELOCK) revert OracleStopLossTimelockTooShort(timelock);
}
```

Call `_validateTimelock(timelock)` alongside `_validateDrawdown` and `_validateDecay` in `initialize` (L57-58), and call it at the top of `proposeOracleStopLossTimelock` (L78). [9](#0-8) 

## Proof of Concept

```solidity
function test_timelockZeroBypass() public {
    uint32 initialTimelock = 1 days;
    // Pool initialized with timelock = 1 day, drawdownE6 = 100_000 (10%)

    // Step 1: propose timelock = 0
    vm.prank(admin);
    extension.proposeOracleStopLossTimelock(pool, 0);

    // Step 2: wait the existing timelock (one-time cost)
    vm.warp(block.timestamp + initialTimelock);

    // Step 3: execute timelock = 0
    vm.prank(admin);
    extension.executeOracleStopLossTimelock(pool);
    assertEq(extension.oracleStopLossConfig(pool).timelock, 0);

    // Step 4+5: propose and execute drawdown increase atomically (same block)
    vm.prank(admin);
    extension.proposeOracleStopLossDrawdown(pool, 1_000_000); // 100% drawdown
    vm.prank(admin);
    extension.executeOracleStopLossDrawdown(pool); // no revert

    assertEq(extension.oracleStopLossConfig(pool).drawdownE6, 1_000_000);
    // floorMultiplier = 0 → stop-loss never triggers → LPs unprotected
}
```

Total elapsed time from drawdown proposal to execution: 0 seconds. LPs had zero reaction time.
