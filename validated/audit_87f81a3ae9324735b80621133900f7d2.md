### Title
Bonus Reward Checkpoints Advanced Before Pool Sufficiency Check Causes Permanent Reward Loss on Redelegation Slash - (`x/tieredrewards/keeper/claim_rewards.go`, `x/tieredrewards/keeper/slash.go`)

---

### Summary

In `processEventsAndClaimBonus`, the position's bonus checkpoints (`LastBonusAccrual`, `LastEventSeq`, `LastKnownBonded`) are mutated in-place **before** the pool sufficiency check. When `BeforeRedelegationSlashed` fires and the bonus pool is empty, the error is intentionally swallowed to avoid a chain halt, but the position is then persisted with the already-advanced checkpoints. The accrued bonus for the period up to the slash is permanently unclaimable — the user's earned rewards are silently forfeited with no recovery path.

---

### Finding Description

`processEventsAndClaimBonus` in `claim_rewards.go` mutates the `pos *types.PositionState` argument in-place as it walks events:

- Inside the event loop, `pos.UpdateLastEventSeq(entry.Seq)` is called at line 193.
- After the loop, `applyBonusAccrualCheckpoint(&pos.Position, blockTime)` (line 215) and `pos.UpdateLastKnownBonded(bonded)` (line 217) advance the remaining checkpoints.
- Only **after** all mutations does the function call `k.sufficientBonusPoolBalance(ctx, bonusCoins)` at line 230.
- If the pool is insufficient, the function returns `ErrInsufficientBonusPool` — but `pos` has already been fully mutated. [1](#0-0) [2](#0-1) [3](#0-2) 

In `slashRedelegationPosition` (`slash.go`), the `BeforeRedelegationSlashed` hook calls `processEventsAndClaimBonus` and explicitly swallows `ErrInsufficientBonusPool` to prevent a chain halt:

```go
if _, err := k.processEventsAndClaimBonus(ctx, &pos); err != nil {
    if errors.Is(err, types.ErrInsufficientBonusPool) {
        k.logger(ctx).Error("insufficient bonus pool during redelegation slash", ...)
    } else {
        return err
    }
}
``` [4](#0-3) 

For a **partial** redelegation slash, execution then falls through to:

```go
pos.Delegation.Shares = pos.Delegation.Shares.Sub(sharesToUnbond)
return k.setPositionWithState(ctx, pos, nil)
``` [5](#0-4) 

This persists the position with the already-advanced checkpoints (`LastBonusAccrual = blockTime`, `LastEventSeq` advanced, `LastKnownBonded` updated) even though the bonus was never transferred. The accrued bonus for the entire period from the previous checkpoint to the slash block is permanently lost — no future call to `processEventsAndClaimBonus` will re-compute it because the checkpoints have already moved past it.

For a **full** slash, `pos.ClearBonusCheckpoints()` is called anyway, so the same permanent loss occurs (the bonus is forfeited and checkpoints are zeroed). [6](#0-5) 

---

### Impact Explanation

A tier position owner who has redelegated their position loses all accrued bonus rewards for the period between their last claim and the redelegation slash block, whenever the `RewardsPoolName` module account is empty at slash time. The bonus is computed, the checkpoints are advanced, but the `SendCoinsFromModuleToAccount` transfer never executes. There is no recovery path: the position's `LastBonusAccrual` is now at the slash block time, so the lost period can never be re-entered in a future claim.

The pool balance is not corrupted — the funds remain in the pool — but the user's entitlement to their earned bonus is permanently destroyed. This is a direct, irreversible user fund loss (earned rewards that are never paid). [7](#0-6) 

---

### Likelihood Explanation

The conditions required are:

1. A tier position has been redelegated via `MsgTierRedelegate`, creating a `RedelegationMapping` entry.
2. The destination validator is slashed during the redelegation period (triggering `BeforeRedelegationSlashed`).
3. The `RewardsPoolName` module account balance is zero or below the accrued bonus at slash time.

The bonus pool is a separately funded module account with no automatic replenishment mechanism beyond governance or external transfers. It can and does run to zero (the integration tests explicitly test this scenario). Validator slashing during a redelegation period is a standard Cosmos SDK event. All three conditions are independently reachable by normal protocol operation without any privileged access. [8](#0-7) 

---

### Recommendation

Move the `applyBonusAccrualCheckpoint` and `UpdateLastKnownBonded` mutations to occur **only after** a successful pool transfer, or use a local copy of the position for computation and only apply mutations on success. Alternatively, in `slashRedelegationPosition`, when `ErrInsufficientBonusPool` is caught, explicitly reset the position's bonus checkpoints back to their pre-call values (snapshot them before calling `processEventsAndClaimBonus`) so the user can retry the claim once the pool is replenished.

---

### Proof of Concept

1. User creates a tier position via `MsgLockTier` and accrues bonus over 30 days.
2. User calls `MsgTierRedelegate` — a `RedelegationMapping` is created.
3. The `RewardsPoolName` module account is drained to zero (e.g., by normal `BeginBlocker` top-up activity).
4. The destination validator is slashed during the redelegation period. The staking module fires `BeforeRedelegationSlashed`.
5. `slashRedelegationPosition` calls `processEventsAndClaimBonus(&pos)`.
6. Inside `processEventsAndClaimBonus`: `pos.LastEventSeq`, `pos.LastBonusAccrual`, and `pos.LastKnownBonded` are all updated to reflect the current block. `totalBonus > 0`. `sufficientBonusPoolBalance` returns `ErrInsufficientBonusPool`. The function returns the error with `pos` already mutated.
7. Back in `slashRedelegationPosition`: the error is swallowed. For a partial slash, `setPositionWithState` persists the mutated `pos`.
8. User later calls `MsgClaimTierRewards`. `processEventsAndClaimBonus` starts from the now-advanced `LastBonusAccrual` — the 30-day accrual period is gone. The user receives zero bonus for that period. [9](#0-8) [10](#0-9)

func (k Keeper) slashRedelegationPosition(ctx context.Context, unbondingId uint64, sharesToUnbond math.LegacyDec) error {
	positionId, err := k.getRedelegationMapping(ctx, unbondingId)
	if errors.Is(err, collections.ErrNotFound) {
		return nil
	}
	if err != nil {
		return err
	}

	pos, err := k.getPositionState(ctx, positionId)
	if errors.Is(err, types.ErrPositionNotFound) {
		k.logger(ctx).Error("position not found during redelegation slash",
			"position_id", positionId,
			"unbonding_id", unbondingId,
			"error", err.Error(),
		)
		return nil
	}
	if err != nil {
		return err
	}

	if !pos.IsDelegated() {
		// Defensive
		k.logger(ctx).Error("delegation missing during BeforeRedelegationSlashed",
			"position_id", positionId,
			"unbonding_id", unbondingId,
			"shares_to_unbond", sharesToUnbond.String(),
		)
		return nil
	}

	dstValStr := pos.Delegation.ValidatorAddress

	// Settle bonus against PRE-slash shares.
	if _, err := k.processEventsAndClaimBonus(ctx, &pos); err != nil {
		// Deliberately forgo bonus rewards if pool is insufficient to prevent chain halt.
		if errors.Is(err, types.ErrInsufficientBonusPool) {
			k.logger(ctx).Error("insufficient bonus pool during redelegation slash",
				"position_id", pos.Id,
				"error", err.Error(),
			)
		} else {
			return err
		}
	}

	fullSlash := sharesToUnbond.GTE(pos.Delegation.Shares)

	if fullSlash {
		pos.Delegation = nil
		pos.ClearBonusCheckpoints()
		return k.setPositionWithState(ctx, pos, &ValidatorTransition{PreviousAddress: dstValStr})
	}
	// In-memory only: the persisted Position carries no share count, and the
	// live delegation will reflect the post-Unbond shares on the next read.
	// Update the local copy so any follow-up logic in this call sees consistent state.
	pos.Delegation.Shares = pos.Delegation.Shares.Sub(sharesToUnbond)
	return k.setPositionWithState(ctx, pos, nil)

func (k Keeper) sufficientBonusPoolBalance(ctx context.Context, bonus sdk.Coins) error {
	if bonus.IsZero() {
		return nil
	}

	poolAddr := k.accountKeeper.GetModuleAddress(types.RewardsPoolName)
	poolBalance := k.bankKeeper.GetAllBalances(ctx, poolAddr)
	if !poolBalance.IsAllGTE(bonus) {
		return errorsmod.Wrapf(types.ErrInsufficientBonusPool,
			"bonus: %s, pool balance: %s",
			bonus.String(), poolBalance.String())
	}

	return nil
