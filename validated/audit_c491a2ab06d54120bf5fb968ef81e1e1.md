### Title
Missing minimum price validation for `l1_gas` and `l1_data_gas` resource bounds in `StatelessTransactionValidator` allows admission of transactions that will fail execution — (File: `crates/apollo_gateway/src/stateless_transaction_validator.rs`)

---

### Summary

`StatelessTransactionValidator::validate_resource_bounds` enforces a minimum price floor only for `l2_gas.max_price_per_unit`, while the `l1_gas.max_price_per_unit` and `l1_data_gas.max_price_per_unit` fields — which are present in every `AllResourceBounds` transaction and committed into the transaction hash — are never checked against any minimum threshold. A transaction with `l1_gas.max_price_per_unit = 0` passes both the stateless and stateful gateway validators and is admitted to the mempool, but is then rejected by the blockifier's pre-validation with `InsufficientResourceBounds` at execution time.

---

### Finding Description

`StatelessTransactionValidator::validate_resource_bounds` (lines 56–88) performs two checks when `validate_resource_bounds` is enabled:

1. The aggregate `max_possible_fee` across all three resources is non-zero.
2. `l2_gas.max_price_per_unit >= config.min_gas_price`. [1](#0-0) 

Neither `l1_gas.max_price_per_unit` nor `l1_data_gas.max_price_per_unit` is compared against any floor. A transaction that sets both to `0` while keeping `l2_gas.max_price_per_unit >= min_gas_price` satisfies both checks and is accepted.

The stateful validator's `validate_tx_l2_gas_price_within_threshold` mirrors this gap: it reads the previous block's `l2_gas_price` and checks only the L2 bound; the L1 and L1-data bounds are silently ignored. [2](#0-1) 

The `AllResourceBounds` struct carries all three price fields, and all three are committed into the transaction hash via `get_tip_resource_bounds_hash` → `pack_resource_bounds`. [3](#0-2) 

At execution time the blockifier's `check_resource_bounds_and_fee` iterates over all three resources and rejects the transaction with `ResourceBoundsError::MaxGasPriceTooLow` when `max_price_per_unit < actual_gas_price`. [4](#0-3) 

The gateway therefore admits a transaction that the blockifier will deterministically reject, breaking the invariant that the gateway filters out transactions that cannot succeed.

The `StatelessTransactionValidatorConfig` documents `min_gas_price` as applying only to L2 gas, and the `StatefulTransactionValidatorConfig` comment confirms the stateful check is also L2-only. [5](#0-4) 

---

### Impact Explanation

**High — Mempool/gateway/RPC admission accepts invalid transactions before sequencing.**

Any unprivileged caller can craft a well-formed V3 transaction with `l1_gas.max_price_per_unit = 0` and `l2_gas.max_price_per_unit >= min_gas_price`. The gateway accepts it, the mempool stores it, and the batcher dequeues it. The blockifier then rejects it during pre-validation, wasting batcher and mempool capacity. At scale this enables low-cost mempool flooding: the attacker pays only the L2 gas price floor to inject transactions that consume sequencer resources but never execute.

---

### Likelihood Explanation

The trigger requires only a standard RPC `add_transaction` call with a crafted `AllResourceBounds` value. No privileged access, special account, or malformed bytes are needed. The condition is reachable on every production deployment where the actual L1 gas price is non-zero (the normal operating state).

---

### Recommendation

Extend `validate_resource_bounds` to enforce a minimum price floor for `l1_gas.max_price_per_unit` and `l1_data_gas.max_price_per_unit`, analogous to the existing `l2_gas` check. The stateful validator should similarly compare all three prices against the corresponding previous-block gas prices, not only the L2 price.

```rust
// In validate_resource_bounds (stateless):
if resource_bounds.l1_gas.max_price_per_unit.0 < self.config.min_l1_gas_price {
    return Err(...MaxGasPriceTooLow { resource: L1Gas, ... });
}
if resource_bounds.l1_data_gas.max_price_per_unit.0 < self.config.min_l1_data_gas_price {
    return Err(...MaxGasPriceTooLow { resource: L1DataGas, ... });

```

### Proof of Concept

1. Construct a valid `RpcInvokeTransactionV3` with:
   - `l1_gas = ResourceBounds { max_amount: 1, max_price_per_unit: 0 }`
   - `l1_data_gas = ResourceBounds { max_amount: 1, max_price_per_unit: 0 }`
   - `l2_gas = ResourceBounds { max_amount: 1_000_000, max_price_per_unit: min_gas_price }` (passes the only enforced check)
2. Submit via `add_transaction` to the gateway.
3. Observe: `StatelessTransactionValidator::validate_resource_bounds` passes (aggregate fee > 0, l2 price ≥ floor).
4. Observe: `StatefulTransactionValidator::validate_resource_bounds` passes (only l2 price is compared to the previous block price).
5. Transaction enters the mempool.
6. Batcher dequeues it; blockifier `check_resource_bounds_and_fee` fires `ResourceBoundsError::MaxGasPriceTooLow { resource: L1Gas, max_gas_price: 0, actual_gas_price: <non-zero> }` and rejects the transaction. [6](#0-5) [7](#0-6) [8](#0-7)

    fn validate_resource_bounds(
        &self,
        tx: &RpcTransaction,
    ) -> StatelessTransactionValidatorResult<()> {
        if !self.config.validate_resource_bounds {
            return Ok(());
        }

        let resource_bounds = *tx.resource_bounds();
        // The resource bounds should be positive even without the tip.
        if ValidResourceBounds::AllResources(resource_bounds).max_possible_fee(Tip::ZERO) == Fee(0)
        {
            return Err(StatelessTransactionValidatorError::ZeroResourceBounds { resource_bounds });

                gas_price: resource_bounds.l2_gas.max_price_per_unit,
                min_gas_price: self.config.min_gas_price,

    // TODO(Arni): Consider running this validation for all gas prices.
    fn validate_tx_l2_gas_price_within_threshold(
        &self,
        tx_resource_bounds: ValidResourceBounds,
        previous_block_l2_gas_price: NonzeroGasPrice,
    ) -> StatefulTransactionValidatorResult<()> {
        match tx_resource_bounds {
            ValidResourceBounds::AllResources(tx_resource_bounds) => {
                let tx_l2_gas_price = tx_resource_bounds.l2_gas.max_price_per_unit;
                let gas_price_threshold_multiplier =
                    Ratio::new(self.config.min_gas_price_percentage.into(), 100_u128);
                let threshold = (gas_price_threshold_multiplier
                    * previous_block_l2_gas_price.get().0)
                    .to_integer();
                if tx_l2_gas_price.0 < threshold {
                    return Err(StarknetError {
                        // We didn't have this kind of an error.
                        code: StarknetErrorCode::UnknownErrorCode(
                            "StarknetErrorCode.GAS_PRICE_TOO_LOW".to_string(),
                        ),
                        message: format!(
                            "Transaction L2 gas price {tx_l2_gas_price} is below the required \
                             threshold {threshold}.",
                        ),
                    });
                }
            }
            ValidResourceBounds::L1Gas(_) => {
                // No validation required for legacy transactions.
            }
        }
        Ok(())
    }
