### Title
Unhandled Return Value of `transferFrom` in `replaceUsdcEWithUsdc()` Allows Draining usdcE from DirectDepositV1 Without Providing USDC — (File: `core/contracts/ContractOwner.sol`)

---

### Summary

`ContractOwner.replaceUsdcEWithUsdc()` calls `IERC20Base(usdc).transferFrom(msg.sender, directDepositV1, balance)` without checking the return value. The function has no access control beyond a chain-ID check, so any unprivileged caller on chain 57073 can trigger it. If the USDC token's `transferFrom` returns `false` instead of reverting (e.g., due to insufficient allowance or balance), the contract silently continues, withdraws usdcE from the DDA, and sends it to the caller — who provided nothing.

---

### Finding Description

`ContractOwner` imports and uses `ERC20Helper` via `using ERC20Helper for IERC20Base`, which provides a `safeTransferFrom` wrapper that checks both the low-level call success and the decoded boolean return value. [1](#0-0) 

Every other ERC20 pull in the codebase uses this safe wrapper: [2](#0-1) [3](#0-2) 

However, `replaceUsdcEWithUsdc()` at line 616 directly calls the raw `transferFrom` on the USDC token without using the safe wrapper and without checking the return value: [4](#0-3) 

The function then unconditionally proceeds to:
1. Call `DirectDepositV1(directDepositV1).withdraw(IIERC20Base(usdcE))` — which pulls all usdcE from the DDA to `ContractOwner`.
2. Call `IERC20Base(usdcE).safeTransfer(msg.sender, balance)` — which sends that usdcE to the caller.

If the `transferFrom` on line 616 silently fails (returns `false`), the caller receives usdcE without having provided any USDC.

---

### Impact Explanation

An unprivileged caller who has no USDC balance or allowance can drain usdcE tokens from any `DirectDepositV1` address registered under any `subaccount` on chain 57073. The DDA loses its usdcE collateral, the caller receives it for free, and the subaccount's pending deposit is corrupted — the DDA no longer holds the usdcE that was meant to be credited as collateral.

---

### Likelihood Explanation

The function `replaceUsdcEWithUsdc` has no `onlyOwner` or similar modifier — only a `require(block.chainid == 57073)` guard. Any externally owned account on the Ink chain can call it. The exploitability depends on whether the specific USDC deployment at `0x2D270e6886d130D724215A266106e6832161EAEd` on chain 57073 returns `false` on failure rather than reverting. Many non-standard or bridged stablecoin implementations do return `false`. Even if the current deployment reverts, the missing check is a latent defect that violates the contract's own established safe-transfer pattern and could be triggered by any future token upgrade or replacement.

---

### Recommendation

Replace the raw `transferFrom` call on line 616 with the `safeTransferFrom` wrapper already available via the `ERC20Helper` library:

```solidity
// Before (vulnerable):
IERC20Base(usdc).transferFrom(msg.sender, directDepositV1, balance);

// After (safe):
IERC20Base(usdc).safeTransferFrom(msg.sender, directDepositV1, balance);
```

This is consistent with every other ERC20 pull in the codebase and will revert if the transfer fails for any reason. [5](#0-4) 

---

### Proof of Concept

1. A `DirectDepositV1` DDA for `subaccount` holds 1000 usdcE tokens.
2. Attacker (with 0 USDC balance and 0 USDC allowance) calls `replaceUsdcEWithUsdc(subaccount)` on chain 57073.
3. `balance = IERC20Base(usdcE).balanceOf(directDepositV1)` → 1000.
4. `IERC20Base(usdc).transferFrom(attacker, directDepositV1, 1000)` → returns `false` (attacker has no USDC). Return value is **not checked**.
5. `DirectDepositV1(directDepositV1).withdraw(usdcE)` → transfers 1000 usdcE from DDA to `ContractOwner`.
6. `IERC20Base(usdcE).safeTransfer(attacker, 1000)` → transfers 1000 usdcE from `ContractOwner` to attacker.
7. Attacker receives 1000 usdcE. DDA holds 0 usdcE. No USDC was ever transferred. [6](#0-5)

```text
            IERC20Base(usdcE).safeTransfer(msg.sender, balance);
        }
    }
```

```text
    function safeTransferFrom(
        IERC20Base self,
        address from,
        address to,
        uint256 amount
    ) internal {
        (bool success, bytes memory data) = address(self).call(
            abi.encodeWithSelector(
                IERC20Base.transferFrom.selector,
                from,
                to,
                amount
            )
        );

        require(
            success && (data.length == 0 || abi.decode(data, (bool))),
            ERR_TRANSFER_FAILED
        );
    }
```
