### Title
`withdrawFeeX18` Bypassed via Slow Mode Withdrawal Path — (`File: core/contracts/EndpointTx.sol`)

---

### Summary

The `withdrawFeeX18` fee, which is charged on every collateral withdrawal in the fast (sequencer) path, is entirely absent from the slow mode withdrawal path. Any unprivileged user can submit a `WithdrawCollateral` transaction via `submitSlowModeTransaction`, wait 3 days, and execute it — paying only the flat `SLOW_MODE_FEE` ($1) while completely avoiding the proportional `withdrawFeeX18`.

---

### Finding Description

There are two distinct code paths that execute a `WithdrawCollateral` transaction:

**Fast path — fee is charged:**

In `processTransactionImpl`, when `txType == WithdrawCollateral`, the code calls `chargeFee` before delegating to the clearinghouse:

```solidity
chargeFee(
    signedTx.tx.sender,
    spotEngine.getConfig(signedTx.tx.productId).withdrawFeeX18,
    signedTx.tx.productId
);
clearinghouse.withdrawCollateral(...);
``` [1](#0-0) 

**Slow mode path — fee is NOT charged:**

In `processSlowModeTransactionImpl`, the same `WithdrawCollateral` type is handled, but `chargeFee` is never called:

```solidity
} else if (txType == IEndpoint.TransactionType.WithdrawCollateral) {
    IEndpoint.WithdrawCollateral memory txn = abi.decode(...);
    validateSender(txn.sender, sender);
    clearinghouse.withdrawCollateral(
        txn.sender, txn.productId, txn.amount, address(0), nSubmissions
    );
``` [2](#0-1) 

The `WithdrawCollateral` type is not in the admin-only block of `submitSlowModeTransactionImpl`, so it falls into the `else` branch that charges only the flat `SLOW_MODE_FEE`: [3](#0-2) 

`SLOW_MODE_FEE` is a fixed $1 flat fee, not the proportional `withdrawFeeX18`: [4](#0-3) 

After the 3-day delay, `executeSlowModeTransaction` is callable by **anyone** (no access control), which triggers `processSlowModeTransactionImpl` without the fee: [5](#0-4) 

---

### Impact Explanation

The `sequencerFee[productId]` accumulator is never credited for slow mode withdrawals. The protocol loses the entire `withdrawFeeX18` revenue on every withdrawal routed through slow mode. For large withdrawals, the bypassed fee can be materially larger than the $1 flat slow mode fee. The broken invariant is: *every collateral withdrawal must deduct `withdrawFeeX18` from the withdrawing subaccount and credit `sequencerFee`*. [6](#0-5) 

---

### Likelihood Explanation

High. The entry point is `Endpoint.submitSlowModeTransaction`, which is a public, permissionless function. The only cost to the attacker is the flat $1 `SLOW_MODE_FEE` and a 3-day wait. Any user with a balance large enough that `withdrawFeeX18` exceeds $1 is economically incentivized to use this path. No special privileges, signatures beyond sender identity, or off-chain coordination are required. [7](#0-6) 

---

### Recommendation

Add `chargeFee` for `withdrawFeeX18` inside the `WithdrawCollateral` branch of `processSlowModeTransactionImpl`, mirroring the fast path:

```solidity
} else if (txType == IEndpoint.TransactionType.WithdrawCollateral) {
    IEndpoint.WithdrawCollateral memory txn = abi.decode(...);
    validateSender(txn.sender, sender);
+   chargeFee(
+       txn.sender,
+       spotEngine.getConfig(txn.productId).withdrawFeeX18,
+       txn.productId
+   );
    clearinghouse.withdrawCollateral(...);
``` [2](#0-1) 

---

### Proof of Concept

1. User calls `Endpoint.submitSlowModeTransaction(withdrawCollateralTx)` where `withdrawCollateralTx` encodes `TransactionType.WithdrawCollateral` with `sender = userSubaccount`, `productId`, and `amount`.
2. `submitSlowModeTransactionImpl` does not match `WithdrawCollateral` in the admin-only block, so it charges only `SLOW_MODE_FEE` ($1) and queues the transaction with `executableAt = block.timestamp + 3 days`.
3. After 3 days, anyone calls `Endpoint.executeSlowModeTransaction()`.
4. This calls `processSlowModeTransaction` → `processSlowModeTransactionImpl`.
5. The `WithdrawCollateral` branch calls `clearinghouse.withdrawCollateral` directly — **no `chargeFee` call** — so `withdrawFeeX18` is never deducted and `sequencerFee[productId]` is never incremented.
6. The user receives the full withdrawal amount minus only the $1 flat fee, bypassing the proportional `withdrawFeeX18`. [8](#0-7)

    function submitSlowModeTransactionImpl(bytes calldata transaction) public {
        IEndpoint.TransactionType txType = IEndpoint.TransactionType(
            uint8(transaction[0])
        );

        // special case for DepositCollateral because upon
        // slow mode submission we must take custody of the
        // actual funds

        address sender = msg.sender;

        if (txType == IEndpoint.TransactionType.DepositCollateral) {
            revert();
        } else if (txType == IEndpoint.TransactionType.DepositInsurance) {
            IEndpoint.DepositInsurance memory txn = abi.decode(
                transaction[1:],
                (IEndpoint.DepositInsurance)
            );
            require(
                txn.amount >= uint128(SLOW_MODE_FEE),
                ERR_DEPOSIT_TOO_SMALL
            );
            handleDepositTransfer(_getQuote(), sender, uint256(txn.amount));
        } else if (
            txType == IEndpoint.TransactionType.WithdrawInsurance ||
            txType == IEndpoint.TransactionType.DelistProduct ||
            txType == IEndpoint.TransactionType.DumpFees ||
            txType == IEndpoint.TransactionType.RebalanceXWithdraw ||
            txType == IEndpoint.TransactionType.UpdateTierFeeRates ||
            txType == IEndpoint.TransactionType.AddNlpPool ||
            txType == IEndpoint.TransactionType.UpdateNlpPool ||
            txType == IEndpoint.TransactionType.DeleteNlpPool ||
            txType == IEndpoint.TransactionType.ForceRebalanceNlpPool ||
            txType == IEndpoint.TransactionType.NlpProfitShare ||
            txType == IEndpoint.TransactionType.UpdateBuilder
        ) {
            require(sender == owner());
        } else {
            chargeSlowModeFee(_getQuote(), sender);
            slowModeFees += SLOW_MODE_FEE;
        }

        IEndpoint.SlowModeConfig memory _slowModeConfig = slowModeConfig;
        requireUnsanctioned(sender);
        slowModeTxs[_slowModeConfig.txCount++] = IEndpoint.SlowModeTx({
            executableAt: uint64(block.timestamp) + SLOW_MODE_TX_DELAY, // hardcoded to three days
            sender: sender,
            tx: transaction
        });
        // TODO: to save on costs we could potentially just emit something
        // for now, we can just create a separate loop in the engine that queries the remote
        // sequencer for slow mode transactions, and ignore the possibility of a reorgy attack
        slowModeConfig = _slowModeConfig;
    }
```
