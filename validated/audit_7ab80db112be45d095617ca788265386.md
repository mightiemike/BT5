### Title
Unrestricted `run_mainchain_gc` Allows Any Caller to Bulk-Prune Historical Headers, Permanently Breaking Transaction Verification — (`contract/src/lib.rs`)

---

### Summary

`run_mainchain_gc` is a public, state-mutating function with no caller restriction beyond the pause guard. Any unprivileged NEAR account can call it with an attacker-controlled `batch_size`, causing bulk, irreversible removal of historical block headers from `headers_pool` and `mainchain_height_to_header`. This permanently breaks `verify_transaction_inclusion` and `verify_transaction_inclusion_v2` for every pruned block.

---

### Finding Description

`submit_blocks` is correctly double-gated with both `#[pause]` and `#[trusted_relayer]`, restricting callers to whitelisted relayers or DAO-role accounts. [1](#0-0) 

`run_mainchain_gc`, however, carries only the pause decorator: [2](#0-1) 

The `#[pause(except(roles(Role::UnrestrictedRunGC)))]` attribute only restricts access **when the contract is paused**. When the contract is live (the normal operating state), the function is callable by any NEAR account with no role check whatsoever.

The function accepts a caller-supplied `batch_size: u64` and removes up to `amount_of_headers_we_store - gc_threshold` headers from the bottom of the canonical chain: [3](#0-2) 

It then advances `mainchain_initial_blockhash` to the new oldest retained header. The removed entries are permanently deleted from both `mainchain_height_to_header` and `headers_pool` via `remove_block_header`. [4](#0-3) 

---

### Impact Explanation

Both verification entry points look up the target block in `mainchain_header_to_height` and `headers_pool`. After GC, those lookups panic unconditionally:

`verify_transaction_inclusion`: [5](#0-4) 

`verify_transaction_inclusion_v2`: [6](#0-5) 

An attacker calling `run_mainchain_gc(u64::MAX)` when the chain has grown beyond `gc_threshold` will atomically remove every excess header in one transaction. Any downstream NEAR contract that subsequently calls `verify_transaction_inclusion[_v2]` for a block in the pruned range will receive a hard panic — a permanent, irreversible failure. The deleted state cannot be recovered without full contract redeployment and re-submission of all historical headers.

The corrupted invariant is: **every block hash in `mainchain_header_to_height` must remain resolvable in `headers_pool` for the lifetime of the contract's stated `gc_threshold` window**. The attacker collapses that window to zero excess in a single call.

---

### Likelihood Explanation

- No deposit is required (`run_mainchain_gc` is not `#[payable]`).
- No role, stake, or whitelist check exists when the contract is unpaused.
- The only precondition is that the chain has grown beyond `gc_threshold` (inevitable in production; `gc_threshold` is set to ~52 704 blocks, roughly one year of Bitcoin headers).
- The attack is a single NEAR transaction costing only standard gas fees.
- The attacker needs no prior knowledge beyond the contract's public ABI.

---

### Recommendation

Apply the same `#[trusted_relayer]` guard (or at minimum an explicit role check such as `Role::DAO` or `Role::RelayerManager`) to `run_mainchain_gc` so that only authorized accounts can trigger bulk header pruning:

```rust
#[pause(except(roles(Role::UnrestrictedRunGC)))]
#[trusted_relayer]   // add this, mirroring submit_blocks
pub fn run_mainchain_gc(&mut self, batch_size: u64) { … }
```

Alternatively, make the function `pub(crate)` and remove its external NEAR ABI exposure entirely, since it is already called internally by `submit_blocks`. [7](#0-6) 

---

### Proof of Concept

1. Deploy the contract with `gc_threshold = 52704` and sync it past 52 704 blocks (normal production state).
2. From any unprivileged NEAR account, call:
   ```
   run_mainchain_gc(batch_size: 18446744073709551615)  // u64::MAX
   ```
3. The contract removes all `amount_of_headers_we_store - gc_threshold` excess headers in one transaction and advances `mainchain_initial_blockhash`.
4. Any subsequent call to `verify_transaction_inclusion` or `verify_transaction_inclusion_v2` for a block in the pruned range panics with `"block does not belong to the current main chain"` — permanently, with no recovery path. [8](#0-7)

```rust
    #[pause(except(roles(Role::UnrestrictedRunGC)))]
    pub fn run_mainchain_gc(&mut self, batch_size: u64) {
        let initial_blockheader = self
            .headers_pool
            .get(&self.mainchain_initial_blockhash)
            .unwrap_or_else(|| env::panic_str(ERR_KEY_NOT_EXIST));

        let tip_blockheader = self
            .headers_pool
            .get(&self.mainchain_tip_blockhash)
            .unwrap_or_else(|| env::panic_str(ERR_KEY_NOT_EXIST));

        let amount_of_headers_we_store =
            tip_blockheader.block_height - initial_blockheader.block_height + 1;

        if amount_of_headers_we_store > self.gc_threshold {
            let total_amount_to_remove = amount_of_headers_we_store - self.gc_threshold;
            let selected_amount_to_remove = std::cmp::min(total_amount_to_remove, batch_size);

            let start_removal_height = initial_blockheader.block_height;
            let end_removal_height = initial_blockheader.block_height + selected_amount_to_remove;
            env::log_str(&format!(
                "Num of blocks to remove {selected_amount_to_remove}"
            ));

            for height in start_removal_height..end_removal_height {
                let blockhash = &self
                    .mainchain_height_to_header
                    .get(&height)
                    .unwrap_or_else(|| env::panic_str(ERR_KEY_NOT_EXIST));

                self.remove_block_header(blockhash);
                self.mainchain_height_to_header.remove(&height);
            }

            self.mainchain_initial_blockhash = self
                .mainchain_height_to_header
                .get(&end_removal_height)
                .unwrap_or_else(|| env::panic_str(ERR_KEY_NOT_EXIST));
        }
    }
```
