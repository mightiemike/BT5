### Title
Checkpoint Advancement Before Pool Check in `slashRedelegationPosition` Permanently Destroys Accrued Bonus — (`x/tieredrewards/keeper/slash.go`, `x/tieredrewards/keeper/claim_rewards.go`)

---

### Summary

`processEventsAndClaimBonus` mutates `pos.LastBonusAccrual`, `pos.LastEventSeq`, and `pos.LastKnownBonded` **before** it checks whether the rewards pool has sufficient balance. When `ErrInsufficientBonusPool` is returned, `slashRedelegationPosition` swallows the error and then persists the already-mutated position via `setPositionWithState`. The accrued bonus is never paid, and the checkpoint is permanently advanced past the pre-slash accrual window — the owner can never recover it even after the pool is replenished.

---

### Finding Description

**Step 1 — Checkpoint mutation before pool check in `processEventsAndClaimBonus`:** [1](#0-0) 

During the event loop, `pos.UpdateLastEventSeq(entry.Seq)` mutates the in-memory `pos` on every iteration. [2](#0-1) 

After the loop, `applyBonusAccrualCheckpoint` advances `pos.LastBonusAccrual` to `blockTime` and `pos.UpdateLastKnownBonded` updates the bonded flag — both unconditionally, before any pool check. [3](#0-2) 

Only **after** all three mutations does `sufficientBonusPoolBalance` run. If the pool is empty it returns `ErrInsufficientBonusPool` — but `pos` is already dirty.

**Step 2 — Error swallowed in `slashRedelegationPosition`, dirty `pos` persisted:** [4](#0-3) 

`processEventsAndClaimBonus` is called with `&pos` (pointer to the local value). When `ErrInsufficientBonusPool` is returned, the error is caught and **only logged** — execution falls through with `pos` carrying the advanced checkpoints. [5](#0-4) 

`setPositionWithState` is then called unconditionally (both the full-slash and partial-slash branches), persisting the mutated `pos` — including the advanced `LastBonusAccrual` — to the store without the bonus ever being paid.

**Step 3 — Permanent loss:**

On the next `MsgClaimTierRewards` call, `processEventsAndClaimBonus` reads `pos.LastBonusAccrual` as `blockTime` (the slash block). The entire pre-slash accrual window `[original_LastBonusAccrual, blockTime)` is gone. Even if the pool is replenished immediately, the owner cannot recover the bonus.

---

### Impact Explanation

Direct, permanent loss of accrued bonus rewards to the position owner. The magnitude equals `shares × tokensPerShare × bonusApy × duration / SecondsPerYear` for the accrual window since the last claim — potentially 30+ days of bonus on a large position. There is no recourse: the checkpoint has been advanced and the event reference counts decremented, so the data needed to recompute the bonus is also garbage-collected.

---

### Likelihood Explanation

The rewards pool being empty (or below the owed bonus amount) is a realistic operational condition: the pool is funded by governance and can be drained by concurrent claims or simply not yet replenished. The `BeforeRedelegationSlashed` hook fires automatically during any validator double-sign slash that touches a redelegating position — no attacker action is required beyond having a position on a validator that later misbehaves. The combination is plausible on mainnet.

---

### Recommendation

Separate checkpoint advancement from payment. The simplest fix is to snapshot the pre-call checkpoint fields at the top of `slashRedelegationPosition` and restore them on `ErrInsufficientBonusPool` before persisting:

```go
preAccrual := pos.LastBonusAccrual
preEventSeq := pos.LastEventSeq
preKnownBonded := pos.LastKnownBonded

if _, err := k.processEventsAndClaimBonus(ctx, &pos); err != nil {
    if errors.Is(err, types.ErrInsufficientBonusPool) {
        // Restore checkpoints so the bonus remains claimable after pool replenishment.
        pos.LastBonusAccrual = preAccrual
        pos.LastEventSeq = preEventSeq
        pos.LastKnownBonded = preKnownBonded
        k.logger(ctx).Error(...)
    } else {
        return err
    }
}
```

Alternatively, restructure `processEventsAndClaimBonus` to apply checkpoints only after a successful `SendCoinsFromModuleToAccount`.

### Proof of Concept

```go
func (s *KeeperSuite) TestSlashRedelegation_EmptyPool_LosesBonus() {
    lockAmount := sdkmath.NewInt(sdk.DefaultPowerReduction.Int64())
    _, bondDenom := s.getStakingData()
    // Do NOT fund the rewards pool — leave it empty.

    pos, _, unbondingID := s.setupRedelegatingPosition(lockAmount)
    owner := sdk.MustAccAddressFromBech32(pos.Owner)
    preAccrual := pos.LastBonusAccrual

    // Accrue 30 days of bonus.
    s.ctx = s.ctx.WithBlockTime(s.ctx.BlockTime().Add(30 * 24 * time.Hour))

    balBefore := s.app.BankKeeper.GetBalance(s.ctx, owner, bondDenom)

    // Fire BeforeRedelegationSlashed with a partial slash.
    sharesToUnbond := pos.Delegation.Shares.Quo(sdkmath.LegacyNewDec(10))
    err := s.keeper.Hooks().BeforeRedelegationSlashed(s.ctx, unbondingID, sharesToUnbond)
    s.Require().NoError(err) // hook swallows ErrInsufficientBonusPool

    balAfter := s.app.BankKeeper.GetBalance(s.ctx, owner, bondDenom)
    // Owner received nothing.
    s.Require().True(balAfter.Amount.Equal(balBefore.Amount), "no bonus paid")

    updated, err := s.keeper.GetPositionState(s.ctx, pos.Id)
    s.Require().NoError(err)
    // Checkpoint advanced past pre-slash window — bonus permanently lost.
    s.Require().True(updated.LastBonusAccrual.After(preAccrual),
        "LastBonusAccrual advanced without paying bonus: pre=%s post=%s",
        preAccrual, updated.LastBonusAccrual)

    // Now fund the pool and try to claim — the pre-slash window is gone.
    s.fundRewardsPool(sdkmath.NewInt(1_000_000_000), bondDenom)
    _, err = keeper.NewMsgServerImpl(s.keeper).ClaimTierRewards(s.ctx,
        &types.MsgClaimTierRewards{Owner: pos.Owner, PositionIds: []uint64{pos.Id}})
    s.Require().NoError(err)
    balFinal := s.app.BankKeeper.GetBalance(s.ctx, owner, bondDenom)
    // Balance still equals balBefore — the 30-day bonus is gone forever.
    s.Require().True(balFinal.Amount.Equal(balBefore.Amount),
        "pre-slash bonus irrecoverably lost")
}
```
