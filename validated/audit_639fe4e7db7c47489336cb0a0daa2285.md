### Title
`BurnedAddresses` Parameter Accepts Module Account Addresses, Corrupting Circulating Supply and Bypassing `MaxSupply` Cap - (File: `x/inflation/types/params.go`)

---

### Summary

The `validateBurnedAddresses` function in `x/inflation/types/params.go` only checks that each entry is a valid bech32 address and that there are no duplicates. It does **not** check whether any entry is a module account address. A governance proposal can therefore set a high-balance module account (e.g., `bonded_tokens_pool`) as a burned address. The `BeginBlocker` in `x/inflation/abci.go` then subtracts that module account's balance from the total supply when computing circulating supply, artificially deflating the reported circulating supply and silently bypassing the `MaxSupply` hard cap.

---

### Finding Description

`x/inflation` tracks a list of `BurnedAddresses` whose balances are excluded from the circulating supply before the `MaxSupply` check runs every block:

```go
// x/inflation/abci.go:26-34
burned := math.NewInt(0)
for _, ba := range params.BurnedAddresses {
    balance, err := k.GetAddressBalance(ctx, ba, denom)
    ...
    burned = burned.Add(balance)
}
totalsupply = totalsupply.Sub(burned)
``` [1](#0-0) 

The parameter validation in `validateBurnedAddresses` only enforces bech32 validity and uniqueness:

```go
// x/inflation/types/params.go:76-98
func validateBurnedAddresses(v any) error {
    ...
    for i, addr := range burnedAddresses {
        if _, err := sdk.AccAddressFromBech32(addr); err != nil {
            return fmt.Errorf("invalid burned address at index %d: %s, error: %w", i, addr, err)
        }
        if addressMap[addr] {
            return fmt.Errorf("duplicate burned address found: %s", addr)
        }
        addressMap[addr] = true
    }
    return nil
}
``` [2](#0-1) 

There is no check that the address is **not** a module account. The application registers module accounts with large, actively-changing balances:

```go

``` [3](#0-2) 

`GetAddressBalance` resolves any bech32 address via `bankKeeper.GetBalance`, including module accounts: [4](#0-3)

### Impact Explanation

The corrupted value is `totalsupply` in `BeginBlocker` after `totalsupply = totalsupply.Sub(burned)`. If `bonded_tokens_pool` (which holds all staked tokens — potentially tens of billions of `basecro`) is listed as a burned address, the computed circulating supply is:

```
circulating = total_supply − bonded_pool_balance − actual_burned_balance
```

This is far below the actual circulating supply. The `MaxSupply` check:

```go
if maxsupply.IsPositive() && totalsupply.GT(maxsupply) {
    panic(...)
}
``` [5](#0-4) 

…never triggers even when actual circulating supply exceeds `MaxSupply`. The chain's hard cap of `10^19 basecro` (100B CRO) is silently bypassed, and minting continues indefinitely. The corrupted invariant is the `MaxSupply` supply cap, which is the primary deflationary safety mechanism introduced in v7.0.0. [6](#0-5) 

---

### Likelihood Explanation

The entry path is `MsgUpdateParams` gated by the governance authority. Any token holder can submit a governance proposal; passing it requires a majority vote. This is a realistic governance-path attack: a proposal that sets `bonded_tokens_pool`'s bech32 address as a burned address alongside legitimate-looking parameter changes could pass if voters do not carefully inspect the `burned_addresses` field. The same flaw is also reachable at genesis if the genesis file is misconfigured, though that path is operator-controlled.

---

### Recommendation

Add a check in `validateBurnedAddresses` (or in `SetParams`) that rejects any address whose bech32 string matches a known module account address. The application already exposes `ModuleAccountAddrs()` in `app.go` for exactly this purpose. Alternatively, the `Keeper` should be given access to the `AccountKeeper` and reject any address for which `IsModuleAccount` returns true. [7](#0-6) 

---

### Proof of Concept

1. Compute the bech32 address of `bonded_tokens_pool`:
   ```
   authtypes.NewModuleAddress("bonded_tokens_pool").String()
   // → cro1fl48vsnmsdzcv85q5d2q4z5ajdha8yu3h3skl2 (mainnet prefix)
   ```

2. Submit a governance `MsgUpdateParams` proposal:
   ```json
   {
     "authority": "<gov module address>",
     "params": {
       "max_supply": "10000000000000000000",
       "burned_addresses": ["cro1fl48vsnmsdzcv85q5d2q4z5ajdha8yu3h3skl2"],
       "decay_rate": "0.068"
     }
   }
   ```

3. `validateBurnedAddresses` accepts this — the address is valid bech32 and not a duplicate.

4. From the next block onward, `BeginBlocker` subtracts the entire bonded pool balance (e.g., ~50B basecro) from total supply before the `MaxSupply` check. The reported circulating supply drops by ~50B basecro, the cap check never fires, and minting continues past the 100B CRO hard cap. [2](#0-1) [8](#0-7)

func BeginBlocker(ctx context.Context, k keeper.Keeper) error {
	defer telemetry.ModuleMeasureSince(types.ModuleName, telemetry.Now(), telemetry.MetricKeyBeginBlocker)
	params, err := k.GetParams(ctx)
	if err != nil {
		panic("could not get params: " + err.Error())
	}
	totalsupply, denom, err := k.GetSupplyAndDenom(ctx)
	if err != nil {
		panic("could not get supply and denom: " + err.Error())
	}

	burned := math.NewInt(0)
	for _, ba := range params.BurnedAddresses {
		balance, err := k.GetAddressBalance(ctx, ba, denom)
		if err != nil {
			panic(fmt.Sprintf("could not get %s balance: %s", ba, err.Error()))
		}
		burned = burned.Add(balance)
	}
	totalsupply = totalsupply.Sub(burned)

	maxsupply := params.MaxSupply
	if maxsupply.IsPositive() && totalsupply.GT(maxsupply) {
		panic(fmt.Sprintf("the total supply has exceeded the maximum supply: %s > %s", totalsupply, maxsupply))
	}

	return nil

// validateBurnedAddresses validates the BurnedAddresses param
func validateBurnedAddresses(v any) error {
	burnedAddresses, ok := v.([]string)
	if !ok {
		return fmt.Errorf("invalid parameter type: %T", v)
	}

	addressMap := make(map[string]bool)

	for i, addr := range burnedAddresses {
		if _, err := sdk.AccAddressFromBech32(addr); err != nil {
			return fmt.Errorf("invalid burned address at index %d: %s, error: %w", i, addr, err)
		}

		if addressMap[addr] {
			return fmt.Errorf("duplicate burned address found: %s", addr)
		}
		addressMap[addr] = true

	}

	return nil
}

func (k Keeper) GetAddressBalance(ctx context.Context, address, denom string) (math.Int, error) {
	addr, err := sdk.AccAddressFromBech32(address)
	if err != nil {

	}
	return k.bankKeeper.GetBalance(ctx, addr, denom).Amount, nil

## 3. Supply Cap (BeginBlocker)

circulating_supply = bank.TotalSupply(bond_denom) - sum(bank.Balance(addr, bond_denom) for addr in BurnedAddresses)
