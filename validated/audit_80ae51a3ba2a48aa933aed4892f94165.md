### Title
`CongestionInfo::delayed_receipts_gas` Diverges Across Protocol-Upgrade Boundary Due to Config-Dependent Gas Recomputation for Plain Receipts — (`runtime/runtime/src/congestion_control.rs`)

---

### Summary

`CongestionInfo::delayed_receipts_gas` is incremented at receipt enqueue time using `compute_receipt_congestion_gas(receipt, config_at_push)`, but for plain `Receipt` objects (not `StateStoredReceipt`), it is decremented at dequeue time using `compute_receipt_congestion_gas(receipt, config_at_pop)`. If a protocol upgrade changes fee parameters between enqueue and dequeue, the add and subtract values diverge, causing `delayed_receipts_gas` to become permanently incorrect — an exact structural analog to the LIDO `lidoLockedETH` inflation bug.

---

### Finding Description

**Enqueue path** — `DelayedReceiptQueueWrapper::push()`:

```rust
let gas = compute_receipt_congestion_gas(&receipt, &config)?;
// ...
self.new_delayed_gas = self.new_delayed_gas.checked_add(gas).ok_or(IntegerOverflowError)?;
``` [1](#0-0) 

The gas is computed with the **current** `RuntimeConfig` and accumulated into `new_delayed_gas`, which is later added to `CongestionInfo::delayed_receipts_gas` via `apply_congestion_changes`. [2](#0-1) 

**Dequeue path** — `receipt_congestion_gas()`:

```rust
pub(crate) fn receipt_congestion_gas(
    receipt: &ReceiptOrStateStoredReceipt,
    config: &RuntimeConfig,
) -> Result<Gas, IntegerOverflowError> {
    match receipt {
        ReceiptOrStateStoredReceipt::Receipt(receipt) => {
            compute_receipt_congestion_gas(receipt, config)   // ← recomputed with current config
        }
        ReceiptOrStateStoredReceipt::StateStoredReceipt(receipt) => {
            Ok(receipt.metadata().congestion_gas)             // ← reads stored value (safe)
        }
    }
}
``` [3](#0-2) 

For `StateStoredReceipt` objects the stored gas is used — correct. For plain `Receipt` objects (all receipts stored before `use_state_stored_receipt` was enabled), `compute_receipt_congestion_gas` is called again with the **current** `RuntimeConfig`. The code's own comment acknowledges the design intent:

> "Please note that this method should only be used when storing receipts into state. It should not be used for retrieving receipts from the state." [4](#0-3) 

Yet for plain `Receipt` objects, the implementation violates this invariant at pop time.

**The gas formula** uses protocol-version-dependent fee parameters:

```rust
fn action_receipt_congestion_gas(...) -> Result<Gas, IntegerOverflowError> {
    let prepaid_exec_gas =
        total_prepaid_exec_fees(config, &action_receipt.actions(), receipt.receiver_id())?
            .gas
            .checked_add(config.fees.fee(ActionCosts::new_action_receipt).exec_fee().gas)
            ...
    let prepaid_send_cost = total_prepaid_send_fees(config, &action_receipt.actions())?;
    let gas_attached_to_fns = total_prepaid_gas(&action_receipt.actions())?;
    ...
}
``` [5](#0-4) 

Any protocol upgrade that changes action fees (exec fees, send fees, or the `new_action_receipt` base fee) causes `compute_receipt_congestion_gas(receipt, config_old) ≠ compute_receipt_congestion_gas(receipt, config_new)`.

**The subtraction** uses `checked_sub` and returns a hard error on underflow:

```rust
pub fn remove_delayed_receipt_gas(&mut self, gas: Gas) -> Result<(), RuntimeError> {
    match self {
        CongestionInfo::V1(inner) => {
            inner.delayed_receipts_gas = inner
                .delayed_receipts_gas
                .checked_sub(gas.as_gas().into())
                .ok_or_else(|| {
                    RuntimeError::UnexpectedIntegerOverflow("remove_delayed_receipt_gas".into())
                })?;
        }
    }
    Ok(())
}
``` [6](#0-5) 

---

### Impact Explanation

**Scenario A — fee parameters increase across upgrade (gas_new > gas_old):**

Each popped plain `Receipt` subtracts more gas than was added. After popping enough receipts, `delayed_receipts_gas` underflows. `remove_delayed_receipt_gas` returns `RuntimeError::UnexpectedIntegerOverflow`. This propagates through `apply_congestion_changes` and causes chunk application to fail on every validator simultaneously — a consensus-breaking chain halt.

**Scenario B — fee parameters decrease across upgrade (gas_new < gas_old):**

`delayed_receipts_gas` is permanently inflated. The shard signals false congestion to all other shards. Other shards throttle forwarding unnecessarily, degrading throughput. The shard also rejects new transactions it could safely accept.

The `CongestionInfoV1::delayed_receipts_gas` field is propagated in chunk headers and validated by all validators:

```rust
pub fn validate_extra_and_header(extra: &CongestionInfo, header: &CongestionInfo) -> bool {
    match (extra, header) {
        (CongestionInfo::V1(extra), CongestionInfo::V1(header)) => {
            extra.delayed_receipts_gas == header.delayed_receipts_gas
            ...
        }
    }
}
``` [7](#0-6) [8](#0-7) 

An incorrect `delayed_receipts_gas` that diverges between the chunk producer and validators would also cause validation failures.

---

### Likelihood Explanation

The trigger requires:
1. Plain `Receipt` objects in the delayed queue (receipts pushed before `use_state_stored_receipt` was enabled in `RuntimeConfig`)
2. A protocol upgrade that changes any fee parameter used in `compute_receipt_congestion_gas` while those receipts remain unprocessed

Protocol upgrades happen on a regular cadence. The delayed queue can hold receipts for many blocks. The `AccountCostIncrease` feature already demonstrates that fee parameters change across upgrades. The transition window — from when the upgrade activates to when all pre-upgrade plain receipts are drained — is the exact exposure window.

---

### Recommendation

Ensure `compute_receipt_congestion_gas` is called **only at push time**, and the result is always stored with the receipt. When popping receipts, always read the stored gas value. This is already the correct design for `StateStoredReceipt`. The fix is to ensure that plain `Receipt` objects remaining in the queue from before `use_state_stored_receipt` was enabled are either:
- Migrated to `StateStoredReceipt` at upgrade time (recomputing gas once with the new config and storing it), or
- Handled by `bootstrap_congestion_info` being called at upgrade time to reset `delayed_receipts_gas` to the correct value under the new config. [7](#0-6) 

---

### Proof of Concept

1. Before protocol upgrade at version V: receipt R (e.g., a `FunctionCall` action) is pushed to the delayed queue as a plain `Receipt`. `compute_receipt_congestion_gas(R, config_V)` = `gas_V`. `delayed_receipts_gas += gas_V`.

2. Protocol upgrade to version V+1 changes `ActionCosts::new_action_receipt` exec fee or any action send/exec fee. `config_{V+1}.fees.fee(ActionCosts::new_action_receipt).exec_fee().gas > config_V.fees.fee(ActionCosts::new_action_receipt).exec_fee().gas`.

3. After upgrade: receipt R is popped from the delayed queue. Since it was stored as a plain `Receipt`, `receipt_congestion_gas` calls `compute_receipt_congestion_gas(R, config_{V+1})` = `gas_{V+1} > gas_V`. `removed_delayed_gas += gas_{V+1}`.

4. `apply_congestion_changes` calls `congestion.remove_delayed_receipt_gas(gas_{V+1})`. If `delayed_receipts_gas < gas_{V+1}` (possible after several such receipts are popped), `checked_sub` fails, returning `RuntimeError::UnexpectedIntegerOverflow("remove_delayed_receipt_gas")`.

5. Chunk application fails on all validators → consensus-breaking chain halt. [9](#0-8) [3](#0-2) [10](#0-9)

fn action_receipt_congestion_gas(
    receipt: &Receipt,
    config: &RuntimeConfig,
    action_receipt: VersionedActionReceipt,
) -> Result<Gas, IntegerOverflowError> {
    let prepaid_exec_gas =
        total_prepaid_exec_fees(config, &action_receipt.actions(), receipt.receiver_id())?
            .gas
            .checked_add(config.fees.fee(ActionCosts::new_action_receipt).exec_fee().gas)
            .ok_or(IntegerOverflowError)?;
    // account for gas guaranteed to be used for creating new receipts
    let prepaid_send_cost = total_prepaid_send_fees(config, &action_receipt.actions())?;
    let prepaid_gas = prepaid_exec_gas.checked_add_result(prepaid_send_cost.gas)?;

    // account for gas potentially used for dynamic execution
    let gas_attached_to_fns = total_prepaid_gas(&action_receipt.actions())?;
    let gas = gas_attached_to_fns.checked_add_result(prepaid_gas)?;

    Ok(gas)
}

pub fn bootstrap_congestion_info(
    trie: &dyn near_store::TrieAccess,
    config: &RuntimeConfig,
    shard_id: ShardId,
) -> Result<CongestionInfo, StorageError> {
    let mut receipt_bytes: u64 = 0;
    let mut delayed_receipts_gas: u128 = 0;
    let mut buffered_receipts_gas: u128 = 0;

    let delayed_receipt_queue = &DelayedReceiptQueue::load(trie)?;
    for receipt_result in delayed_receipt_queue.iter(trie, true) {
        let receipt = receipt_result?;
        let gas =
            receipt_congestion_gas(&receipt, config).map_err(int_overflow_to_storage_err)?.as_gas();
        delayed_receipts_gas = safe_add_gas_to_u128(delayed_receipts_gas, Gas::from_gas(gas))
            .map_err(int_overflow_to_storage_err)?;

        let memory = receipt_size(&receipt).map_err(int_overflow_to_storage_err)? as u64;
        receipt_bytes = receipt_bytes.checked_add(memory).ok_or_else(overflow_storage_err)?;
    }

    let mut outgoing_buffers = ShardsOutgoingReceiptBuffer::load(trie)?;
    for shard in outgoing_buffers.shards() {
        for receipt_result in outgoing_buffers.to_shard(shard).iter(trie, true) {
            let receipt = receipt_result?;
            let gas = receipt_congestion_gas(&receipt, config)
                .map_err(int_overflow_to_storage_err)?
                .as_gas();
            buffered_receipts_gas = safe_add_gas_to_u128(buffered_receipts_gas, Gas::from_gas(gas))
                .map_err(int_overflow_to_storage_err)?;
            let memory = receipt_size(&receipt).map_err(int_overflow_to_storage_err)? as u64;
            receipt_bytes = receipt_bytes.checked_add(memory).ok_or_else(overflow_storage_err)?;
        }
    }

    Ok(CongestionInfo::V1(CongestionInfoV1 {
        delayed_receipts_gas: delayed_receipts_gas as u128,
        buffered_receipts_gas: buffered_receipts_gas as u128,
        receipt_bytes,
        // For the first chunk, set this to the own id.
        // This allows bootstrapping without knowing all other shards.
        // It is also irrelevant, since the bootstrapped value is only used at
        // the start of applying a chunk on this shard. Other shards will only
        // see and act on the first congestion info after that.
        allowed_shard: shard_id.into(),
    }))
}
