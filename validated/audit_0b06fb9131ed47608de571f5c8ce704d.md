Audit Report

## Title
SwapAllowlistExtension Gates the Router Address Instead of the Originating User, Allowing Any Caller to Bypass the Swap Allowlist - (File: metric-periphery/contracts/extensions/SwapAllowlistExtension.sol)

## Summary

`SwapAllowlistExtension.beforeSwap` evaluates `allowedSwapper[pool][sender]` where `sender` is the immediate caller of `pool.swap()`. When `MetricOmmSimpleRouter` intermediates the call, `sender` resolves to the router contract address, not the originating user. A pool admin who allowlists the router so that legitimate users can trade through it simultaneously grants unrestricted swap access to every non-allowlisted user who calls the same public router.

## Finding Description

**Step 1 ŌĆö Pool binds `msg.sender` as `sender`:**
`MetricOmmPool.swap()` passes `msg.sender` directly as the first argument to `_beforeSwap`: [1](#0-0) 

**Step 2 ŌĆö `_beforeSwap` forwards `sender` verbatim to every extension:**
`ExtensionCalling._beforeSwap` encodes and dispatches `sender` unchanged to all extensions in `BEFORE_SWAP_ORDER`: [2](#0-1) 

**Step 3 ŌĆö Extension checks `allowedSwapper[pool][sender]`:**
`SwapAllowlistExtension.beforeSwap` uses `msg.sender` (the pool) as the pool key and `sender` (the immediate pool caller) as the swapper: [3](#0-2) 

**Step 4 ŌĆö Router calls pool directly without encoding the originating user:**
`MetricOmmSimpleRouter.exactInputSingle` calls `pool.swap()` with `params.extensionData` passed through unmodified ŌĆö the originating `msg.sender` is stored only in transient callback context and is never forwarded to the pool or extension: [4](#0-3) 

At the pool level, `msg.sender` is the router contract. The extension therefore evaluates `allowedSwapper[pool][router]` instead of `allowedSwapper[pool][user]`. The originating user's address is never consulted.

**Existing guards are insufficient:** The only guard is the `allowedSwapper` mapping checked in `beforeSwap`. There is no secondary check on `recipient`, no verification of `extensionData` contents, and no mechanism in the pool or extension to distinguish a router-mediated call from a direct call.

## Impact Explanation

`SwapAllowlistExtension` is the protocol's primary mechanism for restricting swap access to a curated set of addresses (e.g., KYC'd participants, private pools). When the router is allowlisted ŌĆö an operationally necessary step for any allowlisted user who wants to trade through it ŌĆö every non-allowlisted user gains unrestricted swap access to the pool. This constitutes direct, repeatable unauthorized extraction of LP assets from pools explicitly configured to deny unprivileged traders, matching the "broken core pool functionality causing loss of funds" and "unauthorized swap access" impact categories.

## Likelihood Explanation

`MetricOmmSimpleRouter` is a public, permissionless contract. The only precondition is that the pool admin has allowlisted the router address, which is operationally required the moment any allowlisted user wants to trade through the router rather than calling the pool directly. No privileged access, special tokens, or malicious setup are required. Any user can call `exactInputSingle`, `exactInput`, `exactOutputSingle`, or `exactOutput` on the router to trigger the bypass. [4](#0-3) 

## Recommendation

The `sender` value reaching `SwapAllowlistExtension` must represent the originating user, not the intermediate router. The cleanest fix is router-side: `MetricOmmSimpleRouter` appends `abi.encode(msg.sender)` to `extensionData` before forwarding to the pool, and `SwapAllowlistExtension.beforeSwap` decodes and checks that value instead of the raw `sender` argument. Alternatively, document that pools using `SwapAllowlistExtension` must be called directly (no router), and enforce this by having the extension revert when `sender` is a known router address.

## Proof of Concept

```
Setup:
  pool configured with SwapAllowlistExtension
  allowedSwapper[pool][alice] = true   (alice is the only allowlisted user)
  allowAllSwappers[pool]      = false

Step 1 ŌĆö pool admin allowlists the router so alice can use it:
  allowedSwapper[pool][router] = true

Step 2 ŌĆö bob (not allowlisted) calls:
  router.exactInputSingle({
    pool:      pool,
    recipient: bob,
    zeroForOne: true,
    amountIn:  X,
    extensionData: ""
  })

Step 3 ŌĆö router calls pool.swap(bob, true, X, ...)
  pool.msg.sender = router
  _beforeSwap(sender=router, ...)

Step 4 ŌĆö SwapAllowlistExtension.beforeSwap:
  allowedSwapper[pool][router] == true  ŌåÆ  check passes

Step 5 ŌĆö swap executes; bob receives output tokens.
  allowedSwapper[pool][bob] was never consulted.
```

Foundry test: deploy pool with `SwapAllowlistExtension`, allowlist only `alice` and the router, call `exactInputSingle` from `bob`, assert the swap succeeds and `bob` receives tokens despite not being in the allowlist.
