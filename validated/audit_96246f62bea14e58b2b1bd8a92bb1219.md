Audit Report

## Title
`SwapAllowlistExtension` checks router address instead of end-user, allowing full allowlist bypass via `MetricOmmSimpleRouter` - (File: metric-periphery/contracts/extensions/SwapAllowlistExtension.sol)

## Summary
`SwapAllowlistExtension.beforeSwap` receives `sender` which is `msg.sender` of `MetricOmmPool.swap()`. When swaps are routed through `MetricOmmSimpleRouter`, the pool's `msg.sender` is the router contract, not the originating user. If the router is added to the allowlist — a natural admin step for a pool intended to be publicly accessible via the official router — every unprivileged address can bypass the per-user access gate entirely by calling the router.

## Finding Description
`MetricOmmPool.swap()` passes `msg.sender` as the `sender` argument to `_beforeSwap` (lines 230–240). `ExtensionCalling._beforeSwap` encodes that value verbatim into the call to each configured extension (lines 160–176). `SwapAllowlistExtension.beforeSwap` then evaluates `allowedSwapper[msg.sender][sender]` (line 37), where `msg.sender` is the pool address and `sender` is the direct caller of `pool.swap()`.

When a user calls `MetricOmmSimpleRouter.exactInputSingle`, the router calls `IMetricOmmPoolActions(params.pool).swap(...)` directly (lines 72–80) without encoding the originating user anywhere. The pool therefore receives `msg.sender = router`, and the extension evaluates `allowedSwapper[pool][router]` instead of `allowedSwapper[pool][alice]`. No existing guard in the pool, extension, or router corrects this identity substitution. The `extensionData` field passed by the router is an empty string `""` (line 79), so no out-of-band user identity is available to the extension either.

## Impact Explanation
A pool protected by `SwapAllowlistExtension` is deployed to restrict trading to known counterparties (KYC'd users, whitelisted market makers, specific protocols). Once the router is allowlisted, the restriction is void for all router-mediated swaps. Any address can execute swaps against the pool's LP positions at the oracle-derived bid/ask, draining LP value without the access control the pool admin intended to enforce. This is a broken core pool invariant (the allowlist guard) with direct loss of LP principal — a High-severity impact under the allowed impact gate.

## Likelihood Explanation
`MetricOmmSimpleRouter` is the canonical public entry point for swaps. A pool admin who wants end users to trade through the router will naturally call `setAllowedToSwap(pool, router, true)`. The bypass is then reachable by any unprivileged caller with zero additional preconditions: no special tokens, no malicious setup, no privileged role beyond the admin's own legitimate configuration step. The smart audit pivot explicitly identifies this class: "allowlist checks must cover the exact actor/action intended and cannot be bypassed through router."

## Recommendation
The extension must check the economic actor (end user), not the transport layer (direct caller of `pool.swap()`). Two viable options:

1. **Pass the original user through `extensionData`**: The router encodes `msg.sender` into `extensionData`; the extension decodes and checks it. Requires a documented convention between router and extension.
2. **Transient storage originator slot**: The pool exposes a `swapFrom(address user, ...)` entry point that writes the originating user to transient storage; the extension reads that slot. This is the cleanest separation and requires no router–extension convention.

Option 3 (document that admins must allowlist EOAs directly and never the router) is a usage restriction, not a code fix, and breaks router UX for restricted pools.

## Proof of Concept
```
Setup:
  pool configured with SwapAllowlistExtension
  admin calls setAllowedToSwap(pool, router, true)
  alice (not individually allowlisted) wants to swap

Attack:
  alice calls router.exactInputSingle({pool: pool, ...})
  router calls pool.swap(recipient, ...)          // msg.sender = router
  pool calls _beforeSwap(router, ...)
  extension checks allowedSwapper[pool][router]   // true → no revert
  swap executes; alice receives output tokens

Expected:
  extension checks allowedSwapper[pool][alice]    // false → NotAllowedToSwap

Actual:
  alice's swap succeeds; allowlist is bypassed

Foundry test outline:
  1. Deploy pool with SwapAllowlistExtension configured in BEFORE_SWAP_ORDER.
  2. Call setAllowedToSwap(pool, address(router), true).
  3. Prank as alice (not allowlisted). Call router.exactInputSingle(...).
  4. Assert swap succeeds (no NotAllowedToSwap revert).
  5. Assert alice received output tokens.
```
