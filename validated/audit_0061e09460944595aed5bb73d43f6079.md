Audit Report

## Title
SwapAllowlistExtension Checks Router Address Instead of End User, Enabling Full Allowlist Bypass — (`metric-periphery/contracts/extensions/SwapAllowlistExtension.sol`)

## Summary

`SwapAllowlistExtension.beforeSwap` gates swaps by checking the `sender` argument, which is bound to `msg.sender` of `MetricOmmPool.swap`. When a user swaps through `MetricOmmSimpleRouter`, the pool's `msg.sender` is the router contract, not the end user. If a pool admin allowlists the router address to enable standard periphery UX, every user — including those the allowlist was meant to exclude — can bypass the per-user gate by routing through the router.

## Finding Description

**Root cause: wrong-actor binding on the swap allowlist path.**

`MetricOmmPool.swap` passes `msg.sender` as the `sender` argument to `_beforeSwap`:

```solidity
// MetricOmmPool.sol L230-231
_beforeSwap(
    msg.sender,   // ← whoever called pool.swap()
    recipient,
    ...
);
```

`ExtensionCalling._beforeSwap` forwards that value unchanged to every configured extension via `abi.encodeCall(IMetricOmmExtensions.beforeSwap, (sender, recipient, ...))`.

`SwapAllowlistExtension.beforeSwap` then checks that `sender` against the per-pool allowlist:

```solidity
// SwapAllowlistExtension.sol L37
if (!allowAllSwappers[msg.sender] && !allowedSwapper[msg.sender][sender]) {
    revert IMetricOmmPoolActions.NotAllowedToSwap();
}
```

`MetricOmmSimpleRouter.exactInputSingle` calls `pool.swap` directly:

```solidity
// MetricOmmSimpleRouter.sol L72-80
(int128 amount0Delta, int128 amount1Delta) = IMetricOmmPoolActions(params.pool)
    .swap(
        params.recipient,
        params.zeroForOne,
        MetricOmmSwapInputs.asAmountSpecifiedIn(params.amountIn),
        priceLimitX64,
        "",
        params.extensionData
    );
```

At that point `msg.sender` inside the pool is the **router contract**, so `sender` delivered to the extension is the router address. The extension evaluates `allowedSwapper[pool][router]`, not `allowedSwapper[pool][user]`.

The same wrong-actor binding applies to `exactInput`, `exactOutputSingle`, and `exactOutput` — all router entry points call `pool.swap` with the router as `msg.sender`.

**Existing guards are insufficient:** `BaseMetricExtension.beforeSwap` has an `onlyPool` modifier that only verifies the caller is a registered pool — it does not validate the identity of the economic actor. There is no mechanism in the extension, pool, or router to pass the original `msg.sender` (the end user) through to the extension.

**Exploit flow:**
1. Pool admin deploys pool with `SwapAllowlistExtension` as `beforeSwap` hook.
2. Admin calls `setAllowedToSwap(pool, alice, true)` — Alice is KYC'd.
3. Admin calls `setAllowedToSwap(pool, router, true)` — to enable standard periphery UX.
4. Bob (not allowlisted) calls `MetricOmmSimpleRouter.exactInputSingle({pool: restrictedPool, ...})`.
5. Router calls `pool.swap(bob, true, X, ...)` — `msg.sender` inside pool = router.
6. Pool calls `_beforeSwap(router, bob, ...)`.
7. Extension checks `allowedSwapper[pool][router]` → `true` → swap proceeds.
8. `allowedSwapper[pool][bob]` is never consulted. Bob receives tokens from the restricted pool.

## Impact Explanation

A curated pool relying on `SwapAllowlistExtension` as its sole swap gate is fully bypassed once the router is allowlisted. Any unprivileged user can call `exactInputSingle` (or any `exact*` variant) on the router pointing at the restricted pool and execute swaps. LP principal and fee revenue in the restricted pool are exposed to unauthorized traders. This constitutes a broken core pool functionality / admin-boundary break with direct loss of LP assets, meeting the critical/high threshold.

## Likelihood Explanation

`MetricOmmSimpleRouter` is the primary user-facing swap entry point. A pool admin who deploys a curated pool and wants users to interact through the standard periphery will naturally allowlist the router — this is the intended operational pattern. The bypass requires no special privilege, no flash loan, and no unusual token behavior. Any EOA can call `exactInputSingle` on the router pointing at the restricted pool. The precondition (router allowlisted) is a natural and expected admin action, making this highly likely to be triggered in production.

## Recommendation

The extension must check the **economic actor** (the end user), not the intermediary. The cleanest fix mirrors the existing `_getPayer()` / `_getTokenToPay()` transient storage pattern already used in `MetricOmmSwapRouterBase`:

1. Before calling `pool.swap`, the router stores the original `msg.sender` in transient storage (e.g., a new `_setSwapper(msg.sender)` slot).
2. The pool or extension reads the original swapper via a trusted accessor on the router, or the pool passes it as an additional argument.
3. `SwapAllowlistExtension.beforeSwap` checks the original swapper identity rather than the `sender` argument.

Alternatively, if the pool's design guarantees `recipient` is always the end user for single-hop swaps, the extension could gate on `recipient` — but this does not hold for multi-hop paths where intermediate recipients are the router itself.

## Proof of Concept

```
Setup:
  - Deploy pool with SwapAllowlistExtension configured as beforeSwap hook
  - Pool admin calls setAllowedToSwap(pool, alice, true)   // Alice is KYC'd
  - Pool admin calls setAllowedToSwap(pool, router, true)  // Enable standard periphery

Attack (Bob, not allowlisted):
  1. Bob calls MetricOmmSimpleRouter.exactInputSingle({
         pool: restrictedPool,
         recipient: bob,
         zeroForOne: true,
         amountIn: X,
         ...
     })
  2. Router calls pool.swap(bob, true, X, ...) — msg.sender inside pool = router
  3. Pool calls _beforeSwap(router, bob, ...)
  4. Extension checks allowedSwapper[pool][router] → true (router is allowlisted)
  5. Swap executes; Bob receives tokens from the restricted pool

Result:
  - Bob bypassed the per-user allowlist entirely
  - allowedSwapper[pool][bob] was never consulted
  - LP funds in the restricted pool are exposed to unauthorized traders
```
