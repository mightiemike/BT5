### Title
Gas-fee unit mismatch for ML-DSA-65 keys in gas-key host functions: wire-format length used instead of trie-format length — (File: `runtime/near-vm-runner/src/logic/logic.rs`)

### Summary

The host-function implementations of `promise_batch_action_transfer_to_gas_key` and `promise_batch_action_add_gas_key_with_function_call` forward the raw `public_key_len` argument (the borsh wire-format length) directly into `gas_key_transfer_send_fee`, `gas_key_transfer_exec_fee`, and `gas_key_add_key_exec_fee`. For ML-DSA-65 keys this value is **1953 bytes**. The transaction-level path in `runtime/runtime/src/config.rs` correctly calls the same fee helpers with `action.public_key.trie_id_len()` = **33 bytes**. The divergence is ≈ 59×, causing contracts that invoke these host functions with ML-DSA-65 keys to be overcharged by ~1920 extra bytes of per-byte gas on every call.

### Finding Description

`gas_key_transfer_send_fee` and `gas_key_transfer_exec_fee` both accept a `public_key_len: usize` parameter that is multiplied by the per-byte fee rate (`gas_key_byte`).

**Transaction path** (`runtime/runtime/src/config.rs`, lines 114–116 and 347–353):

```rust
TransferToGasKey(action) => {
    gas_key_transfer_send_fee(fees, sender_is_receiver, action.public_key.trie_id_len())
        .total()
}
// ...
TransferToGasKey(action) => {
    gas_key_transfer_exec_fee(fees, receiver_id.len(), action.public_key.trie_id_len())
        .total()
}
```

`trie_id_len()` for `PublicKey::MLDSA65` returns `1 + ML_DSA_65_HASH_LENGTH = 33`.

**Host-function path** (`runtime/near-vm-runner/src/logic/logic.rs`, lines 3091–3096):

```rust
let send = gas_key_transfer_send_fee(&self.fees_config, sir, public_key_len as usize);
let exec = gas_key_transfer_exec_fee(
    &self.fees_config,
    receiver_id.len(),
    public_key_len as usize,   // ← raw borsh length, 1953 for ML-DSA-65
);
```

`public_key_len` is the byte count of the borsh-encoded key as supplied by the contract. For `PublicKey::MLDSA65`, `len()` = `1 + 1952 = 1953`. The same mismatch appears in the wasmtime host-function path (`runtime/near-vm-runner/src/wasmtime_runner/logic.rs`, lines 3323–3325) and in `promise_batch_action_add_gas_key_with_function_call` (lines 3226–3231 / wasmtime lines 3499–3504).

The `gas_key_transfer_exec_fee` doc comment explicitly states it is "Based on the access key trie key length + estimated value length (what the receiver needs to read/write in the trie)." The trie key for ML-DSA-65 uses the 32-byte SHA3-256 hash (33 bytes total), not the 1952-byte raw pubkey. Using 1953 instead of 33 inflates the per-byte component of the exec fee by 1920 bytes × `gas_key_byte` rate.

The exact divergent values:

| Path | `public_key_len` passed | Trie-key contribution |
|---|---|---|
| Transaction (`config.rs`) | 33 (trie_id_len) | correct |
| Host function (`logic.rs`) | 1953 (wire len) | **~59× too large** |

### Impact Explanation

Any contract that calls `promise_batch_action_transfer_to_gas_key` or `promise_batch_action_add_gas_key_with_function_call` with an ML-DSA-65 public key is charged ≈ 59× more gas than the equivalent transaction-level action. Because the per-byte fee is multiplied into both the send and exec components, the total overcharge is proportional to `(1953 − 33) × gas_key_byte_rate`. Contracts that budget gas based on the documented trie-format semantics will exhaust their gas allowance and fail. The fee-consistency invariant — that the same action costs the same gas regardless of submission path — is broken at the protocol boundary introduced by `PostQuantumSignatures` (protocol version 85).

**Impact: High** — legitimate contract calls fail; fee invariant broken across a public protocol upgrade boundary.

### Likelihood Explanation

Both `GasKeys` and `PostQuantumSignatures` stabilized at protocol version 85 (the current stable version). Any unprivileged user can deploy a contract that calls `promise_batch_action_transfer_to_gas_key` with an ML-DSA-65 key. The key is passed as raw borsh bytes from contract memory, so `public_key_len = 1953` is the natural value a contract would supply. No privileged role is required.

**Likelihood: Medium** — requires ML-DSA-65 keys to be used in gas-key host functions, which is a new but fully supported code path.

### Recommendation

Replace `public_key_len as usize` with `public_key.trie_id_len()` (after decoding the key) in all four affected call sites:

- `runtime/near-vm-runner/src/logic/logic.rs` — `promise_batch_action_transfer_to_gas_key` (send + exec fee calls)
- `runtime/near-vm-runner/src/logic/logic.rs` — `promise_batch_action_add_gas_key_with_function_call` (exec fee call)
- `runtime/near-vm-runner/src/wasmtime_runner/logic.rs` — same two functions

The decoded `PublicKey` is already available at those call sites (via `public_key.decode()?`), so `public_key.trie_id_len()` can be called directly, matching the transaction path in `config.rs`.

### Proof of Concept

1. Deploy a contract that calls `promise_batch_action_transfer_to_gas_key` with a borsh-encoded `PublicKey::MLDSA65` (1953 bytes).
2. Measure the gas burnt for the `gas_key_byte` cost component — it will be `1953 × gas_key_byte_rate` instead of `33 × gas_key_byte_rate`.
3. Submit the equivalent `TransferToGasKey` action as a regular transaction with the same ML-DSA-65 key.
4. Observe that the transaction-level gas cost uses 33 bytes, confirming the ≈ 59× divergence between the two paths.

```rust
/// Send fee for TransferToGasKey / WithdrawFromGasKey actions.
/// Based on the public key length (what the sender sees).
pub fn gas_key_transfer_send_fee(
    cfg: &RuntimeFeesConfig,
    sender_is_receiver: bool,
    public_key_len: usize,
) -> GasKeyTransferFee {
    let base = cfg.fee(ActionCosts::gas_key_transfer_base).send_fee(sender_is_receiver);
    let per_byte = cfg
        .fee(ActionCosts::gas_key_byte)
        .send_fee(sender_is_receiver)
        .checked_mul(public_key_len as u64)
        .unwrap();
    GasKeyTransferFee { base, per_byte }
}

/// Exec fee for TransferToGasKey / WithdrawFromGasKey actions.
/// Based on the access key trie key length + estimated value length (what the
/// receiver needs to read/write in the trie).
pub fn gas_key_transfer_exec_fee(
    cfg: &RuntimeFeesConfig,
    account_id_len: usize,
    public_key_len: usize,
) -> GasKeyTransferFee {
    let base = cfg.fee(ActionCosts::gas_key_transfer_base).exec_fee();
    let trie_key_len = access_key_key_len(account_id_len, public_key_len);
    let estimated_value_len = AccessKey::min_gas_key_borsh_len();
    let per_byte = cfg
        .fee(ActionCosts::gas_key_byte)
        .exec_fee()
        .checked_mul((trie_key_len + estimated_value_len) as u64)
        .unwrap();
    GasKeyTransferFee { base, per_byte }
}
```

```rust
    /// Length, in bytes, of the on-trie identifier for an access-key
    /// entry owned by this public key. For ed25519/secp256k1 this matches
    /// `len()`; for ML-DSA-65 the trie stores a SHA3-256 hash (33 bytes
    /// including the type tag), not the 1953-byte borsh-encoded pubkey.
    /// Used by storage-fee calculations on the runtime side; cheap to call
    /// (no hashing) - for ML-DSA-65 this returns the size of the digest
    /// form without actually hashing the pubkey.
    pub fn trie_id_len(&self) -> usize {
        match self {
            Self::ED25519(_) => 1 + ed25519_dalek::PUBLIC_KEY_LENGTH,
            Self::SECP256K1(_) => 1 + 64,
            Self::MLDSA65(_) => 1 + ML_DSA_65_HASH_LENGTH,
        }
    }
```
