### Title
`Clearinghouse#burnNlp()` Burns User NLP Tokens Without Quote Credit When `quoteAmount == 0` - (File: `core/contracts/Clearinghouse.sol`)

### Summary

The `burnNlp()` function in `Clearinghouse.sol` unconditionally burns a user's NLP tokens but only credits the resulting quote amount inside an `if (quoteAmount > 0)` guard. When the NLP position's value is entirely consumed by the burn fee, `quoteAmount` is clamped to `0` and the guard is never entered — the user's NLP is permanently destroyed with zero compensation.

### Finding Description

In `burnNlp()`, the quote amount owed to the user is computed and then reduced by a burn fee:

```solidity
int128 quoteAmount = nlpAmount.mul(oraclePriceX18);
int128 burnFee = MathHelper.max(ONE, quoteAmount / 1000);
quoteAmount = MathHelper.max(0, quoteAmount - burnFee);
```

The minimum burn fee is `ONE` (1×10¹⁸ in X18 fixed-point, equivalent to 1 USDC when the quote token has 6 decimals). If the NLP position is worth ≤ 1 USDC, `quoteAmount` is clamped to `0`.

The NLP tokens are then unconditionally burned:

```solidity
spotEngine.updateBalance(NLP_PRODUCT_ID, txn.sender, -nlpAmount);  // always executes
spotEngine.updateBalance(NLP_PRODUCT_ID, N_ACCOUNT, nlpAmount);
```

But the quote credit is gated:

```solidity
if (quoteAmount > 0) {                                              // skipped when quoteAmount == 0
    spotEngine.updateBalance(QUOTE_PRODUCT_ID, txn.sender, quoteAmount);
    _applyNlpRebalance(spotEngine, nlpPools, nlpPoolRebalanceX18);
}
``` [1](#0-0) 

When `quoteAmount == 0`, the NLP burn at lines 511–512 still executes, but the quote credit at line 515 is never applied. The user's NLP balance is reduced to zero and they receive nothing in return. [2](#0-1) 

### Impact Explanation

A user with a small NLP position (worth ≤ 1 USDC at the time of the burn) will have their entire NLP balance destroyed with no quote tokens credited. This is a direct, irreversible loss of the user's on-chain asset. The broken invariant is: *burning NLP must either credit the user with `quoteAmount` or revert — it must never silently destroy NLP with zero compensation.*

### Likelihood Explanation

Any user whose NLP position has depreciated to a value at or below the minimum burn fee (`ONE` in X18 = 1 USDC equivalent) is affected. This is realistic for:
- Users who minted a very small NLP position.
- Users whose NLP value has declined due to pool losses.
- Users who partially burned NLP and have a dust remainder.

The `burnNlp` path is reachable by any user submitting a signed `BurnNlp` transaction through the sequencer endpoint. [3](#0-2) 

### Recommendation

Replace the `if (quoteAmount > 0)` guard with an explicit revert when the computed quote amount is zero (or negative before clamping), so that the transaction fails rather than silently burning the user's NLP:

```diff
- quoteAmount = MathHelper.max(0, quoteAmount - burnFee);
+ int128 rawQuote = quoteAmount - burnFee;
+ require(rawQuote > 0, "BurnNlp: position value does not exceed burn fee");
+ quoteAmount = rawQuote;
```

Alternatively, if dust burns are intentional, add a minimum NLP amount check before the burn executes, ensuring the user is never left with burned NLP and zero quote credit.

### Proof of Concept

1. A user mints a small NLP position worth exactly 0.5 USDC (in X18: `5e17`).
2. The user submits a `BurnNlp` transaction via the sequencer.
3. Inside `burnNlp()`:
   - `quoteAmount = nlpAmount.mul(oraclePriceX18)` → `5e17` (0.5 USDC in X18).
   - `burnFee = MathHelper.max(ONE, 5e17 / 1000)` → `ONE` = `1e18` (1 USDC in X18).
   - `quoteAmount = MathHelper.max(0, 5e17 - 1e18)` → `0`.
4. `spotEngine.updateBalance(NLP_PRODUCT_ID, txn.sender, -nlpAmount)` executes — NLP is burned.
5. `if (quoteAmount > 0)` is `false` — no quote is credited.
6. The user's NLP balance is now `0` and their quote balance is unchanged. They have lost their NLP with zero compensation. [4](#0-3) [2](#0-1)

```text
    function burnNlp(
        IEndpoint.BurnNlp calldata txn,
        int128 oraclePriceX18,
        IEndpoint.NlpPool[] calldata nlpPools,
        int128[] calldata nlpPoolRebalanceX18
    ) external onlyEndpoint {
        require(!RiskHelper.isIsolatedSubaccount(txn.sender), ERR_UNAUTHORIZED);

        ISpotEngine spotEngine = _spotEngine();
        spotEngine.updatePrice(NLP_PRODUCT_ID, oraclePriceX18);

        require(txn.nlpAmount <= INT128_MAX, ERR_CONVERSION_OVERFLOW);
        int128 nlpAmount = int128(txn.nlpAmount);
        require(
            spotEngine.getNlpUnlockedBalance(txn.sender).amount >= nlpAmount,
            ERR_UNLOCKED_NLP_INSUFFICIENT
        );
        int128 quoteAmount = nlpAmount.mul(oraclePriceX18);
        int128 burnFee = MathHelper.max(ONE, quoteAmount / 1000);
        quoteAmount = MathHelper.max(0, quoteAmount - burnFee);

        _validateNlpRebalance(nlpPools, nlpPoolRebalanceX18, -quoteAmount);
        for (uint128 i = 0; i < nlpPoolRebalanceX18.length; i++) {
            require(nlpPoolRebalanceX18[i] <= 0, ERR_INVALID_NLP_REBALANCE);
        }

        spotEngine.updateBalance(NLP_PRODUCT_ID, txn.sender, -nlpAmount);
        spotEngine.updateBalance(NLP_PRODUCT_ID, N_ACCOUNT, nlpAmount);

        if (quoteAmount > 0) {
            spotEngine.updateBalance(QUOTE_PRODUCT_ID, txn.sender, quoteAmount);
            _applyNlpRebalance(spotEngine, nlpPools, nlpPoolRebalanceX18);
        }

        require(
            spotEngine.getBalance(NLP_PRODUCT_ID, txn.sender).amount >= 0,
            ERR_SUBACCT_HEALTH
        );
        // Burning NLP can decrease health if the burn fee exceeds the health improvement
        // from the withdrawal. This check prevents malicious actors from deliberately
        // creating unhealthy subaccounts through NLP burns.
        require(
            getHealth(txn.sender, IProductEngine.HealthType.MAINTENANCE) >= 0,
            ERR_SUBACCT_HEALTH
        );
    }
```
