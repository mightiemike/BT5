### Title
SwapAllowlistExtension Checks Router Address Instead of End-User — Any User Bypasses Per-Pool Swap Allowlist via `MetricOmmSimpleRouter` - (File: metric-periphery/contracts/extensions/SwapAllowlistExtension.sol)

---

### Summary

`SwapAllowlistExtension.beforeSwap` gates swaps by checking `allowedSwapper[pool][sender]`, where `sender` is the `msg.sender` of `pool.swap()`. When a user routes through `MetricOmmSimpleRouter`, `msg.sender` to the pool is the **router**, not the end user. If the pool admin allowlists the router address (the natural action to enable router-based swaps for their allowlisted users), every unprivileged user can bypass the per-pool swap allowlist by routing through the router.

---

### Finding Description

**Call chain:**

```
User → MetricOmmSimpleRouter.exactInputSingle()
     → IMetricOmmPoolActions(pool).swap(recipient, ...)   // msg.sender = router
     → MetricOmmPool.swap()
     → _beforeSwap(msg.sender, ...)                       // sender = router
     → SwapAllowlistExtension.beforeSwap(sender=router, ...)
     → allowedSwapper[pool][router]  ← checked, NOT the user
```

In `MetricOmmPool.swap()`, the pool passes `msg.sender` (the router) as `sender` to `_beforeSwap`: [1](#0-0) 

`ExtensionCalling._beforeSwap` forwards that value unchanged to every configured extension: [2](#0-1) 

`SwapAllowlistExtension.beforeSwap` then checks `allowedSwapper[msg.sender][sender]`, i.e., `allowedSwapper[pool][router]`: [3](#0-2) 

The router never forwards the original `msg.sender` to the pool; it simply calls `pool.swap()` directly: [4](#0-3) 

The result is a permanent identity mismatch: the allowlist sees the router, not the user.

---

### Impact Explanation

A pool admin who wants to allow router-based swaps for their allowlisted users will naturally call `setAllowedToSwap(pool, router, true)`. Once the router is allowlisted, **every** user — including those explicitly not on the allowlist — can bypass the gate by routing through `MetricOmmSimpleRouter`. The allowlist ceases to function as an access-control boundary for any router-mediated swap. Because the pool is oracle-driven, unauthorized swappers can interact at live oracle prices, draining LP-owned liquidity in directions the pool admin intended to restrict (e.g., blocking specific counterparties, complying with regulatory requirements, or protecting against oracle-latency exploitation by known actors).

---

### Likelihood Explanation

The scenario is reachable by any unprivileged user with no special setup beyond the pool admin having allowlisted the router — a natural and expected administrative action for any pool that intends to support the standard periphery. The `MetricOmmSimpleRouter` is the canonical user-facing entry point documented in the protocol. No malicious initial setup, non-standard tokens, or privileged access is required on the attacker's side.

---

### Recommendation

The extension must receive the **economic actor** (the end user), not the immediate caller of `pool.swap()`. Two approaches:

1. **Pass the original user through the router**: Have `MetricOmmSimpleRouter` encode `msg.sender` into `extensionData` and have the allowlist extension decode and verify it. This requires a protocol-level convention.

2. **Gate on `recipient` instead of `sender`**: If the pool's intent is to restrict who receives output tokens, check `recipient` (the second argument to `beforeSwap`). For exact-input single-hop swaps the user typically sets themselves as recipient.

3. **Preferred — pool-level originator field**: Add an `originator` field to the swap call that the router populates with its own `msg.sender`, and have the allowlist check that field. This is the cleanest fix and mirrors how Uniswap v4 handles hook-visible originators.

---

### Proof of Concept

```solidity
// Pool is configured with SwapAllowlistExtension.
// Admin allowlists the router to support router-based swaps:
swapAllowlist.setAllowedToSwap(pool, address(router), true);

// Alice is NOT on the allowlist:
assertFalse(swapAllowlist.isAllowedToSwap(pool, alice));

// Alice bypasses the allowlist by going through the router:
vm.prank(alice);
router.exactInputSingle(ExactInputSingleParams({
    pool: pool,
    recipient: alice,
    zeroForOne: true,
    amountIn: 1000e18,
    amountOutMinimum: 0,
    priceLimitX64: 0,
    deadline: block.timestamp,
    tokenIn: token0,
    extensionData: ""
}));
// Swap succeeds — allowlist bypassed.
// The extension checked allowedSwapper[pool][router] == true, not allowedSwapper[pool][alice].
```

The `beforeSwap` hook receives `sender = address(router)`, which is allowlisted, so the check passes regardless of who `alice` is. [5](#0-4) [6](#0-5) [7](#0-6)
