### Title
Denom Creator Can Permanently Lock NFT Supply by Minting to the NFT Module Account Address — (`x/nft/keeper/keeper.go`, `x/nft/types/msgs.go`)

---

### Summary

`MsgMintNFT.ValidateBasic` and the entire `MintNFT` keeper call chain contain no guard against minting to a blocked or module account address. A denom creator can submit a `MintNFT` transaction with `Recipient` set to the NFT module account address (`authtypes.NewModuleAddress("nft")`). The NFT is written to the KV store with the module account as owner and the supply counter is incremented, but the NFT can never be burned or transferred because the module account cannot sign transactions. The supply counter is permanently inflated above the recoverable NFT count.

---

### Finding Description

**Entry point — `GetCmdMintNFT` / `MsgMintNFT.ValidateBasic`**

`GetCmdMintNFT` accepts an arbitrary `--recipient` flag, validates only that it is a valid bech32 address, and constructs a `MsgMintNFT`: [1](#0-0) 

`MsgMintNFT.ValidateBasic` performs the same check — valid bech32 for both `Sender` and `Recipient` — and nothing else: [2](#0-1) 

The module account address `authtypes.NewModuleAddress("nft")` is a valid bech32 address, so both checks pass.

**Keeper path — no blocked-address guard exists**

`msgServer.MintNFT` decodes the recipient and passes it directly to `Keeper.MintNFT`: [3](#0-2) 

`Keeper.MintNFT` only verifies the sender is the denom creator, then delegates to `MintNFTUnverified`: [4](#0-3) 

`MintNFTUnverified` only checks that the denom exists and the token ID is not already taken, then writes the NFT and increments the supply counter unconditionally: [5](#0-4) 

The `Keeper` struct holds no `AccountKeeper` or `BankKeeper` reference that could be used to check blocked addresses: [6](#0-5) 

The `expected_keepers.go` defines `AccountKeeper` and `BankKeeper` interfaces but they are never injected into the NFT keeper and are not consulted during minting: [7](#0-6) 

**Why the NFT is permanently locked**

`BurnNFT` requires `IsOwner` (the tx signer must equal the stored owner) **and** `IsDenomCreator`: [8](#0-7) 

`TransferOwner` also requires `IsOwner`: [9](#0-8) 

The module account cannot sign any transaction, so neither `BurnNFT` nor `TransferNFT` can ever succeed for an NFT owned by it. The supply counter, incremented at mint time via `increaseSupply`, can never be decremented: [10](#0-9) 

---

### Impact Explanation

- The NFT is written to the KV store with the module account as owner and the owner index is updated.
- `GetTotalSupply` returns a value permanently higher than the number of recoverable (burnable/transferable) NFTs.
- No subsequent `BurnNFT` or `TransferNFT` can succeed for this token.
- The supply accounting invariant (`total_supply == number of NFTs that can be burned`) is permanently broken for the affected denom.
- This can be repeated for every token ID in a denom, inflating the supply counter arbitrarily.

---

### Likelihood Explanation

Any user who has issued a denom (a permissionless operation via `MsgIssueDenom`) is a denom creator and can execute this attack. No governance vote, operator key, or privileged role is required. The attack is a single signed transaction.

---

### Recommendation

1. In `MsgMintNFT.ValidateBasic`, reject `Recipient` equal to any module account address (or at minimum the NFT module account).
2. Inject a `BlockedAddr` check (via `BankKeeper.BlockedAddr` or `AccountKeeper`) into `Keeper.MintNFT` / `MintNFTUnverified` and return an error if the recipient is a blocked address.
3. Add an explicit check `Recipient != authtypes.NewModuleAddress(types.ModuleName)` in `ValidateBasic`.

---

### Proof of Concept

```go
// keeper_test.go (pseudocode)
moduleAddr := authtypes.NewModuleAddress(types.ModuleName) // sdk.AccAddress for "nft"

// Attacker issues a denom (permissionless)
err := keeper.IssueDenom(ctx, "testdenom", "Test", "", "", attackerAddr)
require.NoError(t, err)

// Attacker mints to the module account address
err = keeper.MintNFT(ctx, "testdenom", "token1", "", "", "", attackerAddr, moduleAddr)
require.NoError(t, err) // succeeds — no guard

// Supply is now 1
supply := keeper.GetTotalSupply(ctx, "testdenom")
require.Equal(t, uint64(1), supply)

// BurnNFT fails — module account cannot be the signer
err = keeper.BurnNFT(ctx, "testdenom", "token1", moduleAddr)
require.Error(t, err) // ErrUnauthorized or similar — but moduleAddr can never sign

// TransferNFT also fails for the same reason
err = keeper.TransferOwner(ctx, "testdenom", "token1", moduleAddr, attackerAddr)
require.Error(t, err)

// Supply remains 1 permanently — invariant broken
```

```go
func (msg MsgMintNFT) ValidateBasic() error {
	if _, err := sdk.AccAddressFromBech32(msg.Sender); err != nil {
		return newsdkerrors.Wrapf(sdkerrors.ErrInvalidAddress, "invalid sender address (%s)", err)
	}
	if _, err := sdk.AccAddressFromBech32(msg.Recipient); err != nil {

	}
	if err := ValidateDenomID(msg.DenomId); err != nil {
		return err
	}
	if err := ValidateTokenURI(msg.URI); err != nil {
		return err
	}
	return ValidateTokenID(msg.Id)
}
```

```go
func (k Keeper) MintNFTUnverified(ctx sdk.Context, denomID, tokenID, tokenNm, tokenURI, tokenData string, owner sdk.AccAddress) error {
	if !k.HasDenomID(ctx, denomID) {
		return sdkerrors.Wrapf(types.ErrInvalidDenom, "denom ID %s not exists", denomID)
	}

	if k.HasNFT(ctx, denomID, tokenID) {
		return sdkerrors.Wrapf(types.ErrNFTAlreadyExists, "NFT %s already exists in collection %s", tokenID, denomID)
	}

	k.setNFT(
		ctx, denomID,
		types.NewBaseNFT(
			tokenID,
			tokenNm,
			owner,
			tokenURI,
			tokenData,
		),
	)
	k.setOwner(ctx, denomID, tokenID, owner)
	k.increaseSupply(ctx, denomID)

	return nil
```

```go
		return sdkerrors.Wrapf(types.ErrInvalidDenom, "denom ID %s not exists", denomID)
	}

	nft, err := k.IsOwner(ctx, denomID, tokenID, srcOwner)
	if err != nil {
		return err
	}

	nft.Owner = dstOwner.String()

	k.setNFT(ctx, denomID, nft)
	k.swapOwner(ctx, denomID, tokenID, srcOwner, dstOwner)
	return nil
}
```

```go
func (k Keeper) BurnNFT(ctx sdk.Context, denomID, tokenID string, owner sdk.AccAddress) error {
	if !k.HasDenomID(ctx, denomID) {
		return sdkerrors.Wrapf(types.ErrInvalidDenom, "denom ID %s not exists", denomID)
	}

	nft, err := k.IsOwner(ctx, denomID, tokenID, owner)
	if err != nil {
		return err
	}

	_, err = k.IsDenomCreator(ctx, denomID, owner)
	if err != nil {
		return err
	}

	k.deleteNFT(ctx, denomID, nft)
	k.deleteOwner(ctx, denomID, tokenID, owner)
	k.decreaseSupply(ctx, denomID)

	return nil
}
```
