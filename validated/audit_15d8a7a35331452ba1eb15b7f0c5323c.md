### Title
`UnrestrictedSubmitBlocks` Bypass Role Rendered Inoperative by Mismatched Pause Guard on Internal `run_mainchain_gc` Call - (File: `contract/src/lib.rs`)

### Summary
`submit_blocks` is decorated with `#[pause]` and grants bypass access to the `UnrestrictedSubmitBlocks` role, but it unconditionally calls `self.run_mainchain_gc(num_of_headers)` internally. `run_mainchain_gc` carries its own independent `#[pause(except(roles(Role::UnrestrictedRunGC)))]` guard. When the contract is paused, a caller holding only `UnrestrictedSubmitBlocks` passes the outer guard on `submit_blocks` but is then rejected by the inner guard on `run_mainchain_gc`, causing the entire transaction to panic. The `UnrestrictedSubmitBlocks` role is therefore completely inoperative whenever the contract is paused.

### Finding Description
`submit_blocks` is the sole public entry point for relayers to advance the canonical Bitcoin chain tracked on NEAR. It is decorated with `#[pause]`, which in near-plugins generates a guard that allows callers holding the `UnrestrictedSubmitBlocks` role to proceed even when the contract is paused. [1](#0-0) 

Inside the function body, after processing all submitted headers, the function unconditionally calls: [2](#0-1) 

`run_mainchain_gc` carries its own separate pause guard: [3](#0-2) 

The bypass role on `run_mainchain_gc` is `UnrestrictedRunGC`, which is a **different** role from `UnrestrictedSubmitBlocks`. The near-plugins `#[pause]` macro inserts a guard at the top of the function body that fires on every call — including internal Rust calls — using `env::predecessor_account_id()` (the original external caller) to evaluate role membership. A caller holding only `UnrestrictedSubmitBlocks` therefore passes the outer guard on `submit_blocks` but is rejected by the inner guard on `run_mainchain_gc`, causing the whole transaction to panic. The two roles are defined as distinct enum variants with no overlap: [4](#0-3) 

### Impact Explanation
**Impact: High** — The `UnrestrictedSubmitBlocks` role exists precisely to allow a trusted relayer to keep the light client's canonical chain advancing during a contract pause (e.g., during an upgrade or incident response). Because `submit_blocks` always calls `run_mainchain_gc`, and `run_mainchain_gc` rejects any caller without `UnrestrictedRunGC`, the bypass role is silently broken. During any pause period, no new Bitcoin block headers can be submitted. The canonical chain tip stored on-chain falls behind the real Bitcoin chain, making all `verify_transaction_inclusion` and `verify_transaction_inclusion_v2` calls return stale or incorrect confirmation counts. Any downstream bridge, atomic swap, or lending protocol relying on SPV proofs from this contract will receive incorrect results for the duration of the pause.

### Likelihood Explanation
**Likelihood: Medium** — The contract has an explicit `PauseManager` role and a `Pausable` derive, indicating pausing is an expected operational event. The `UnrestrictedSubmitBlocks` role would only be granted to a small set of trusted relayers, and the bug only manifests when the contract is paused. However, the pause is a privileged but realistic operational action, and the broken bypass role means the failure is guaranteed every time a paused-state submission is attempted.

### Recommendation
Grant the `UnrestrictedSubmitBlocks` bypass role also on `run_mainchain_gc`, or change the `run_mainchain_gc` guard to `#[pause(except(roles(Role::UnrestrictedRunGC, Role::UnrestrictedSubmitBlocks)))]`. Alternatively, extract the GC logic into a private helper that is not decorated with `#[pause]`, and call that helper from both `submit_blocks` and the public `run_mainchain_gc` entry point. The public `run_mainchain_gc` retains its own guard; the internal path used by `submit_blocks` bypasses it.

### Proof of Concept
1. Deploy the contract and grant account `relayer.near` the `UnrestrictedSubmitBlocks` role only.
2. Grant account `pauser.near` the `PauseManager` role.
3. `pauser.near` calls `pa_pause_feature("submit_blocks")` (or the global pause) — contract is now paused.
4. `relayer.near` calls `submit_blocks` with a valid batch of headers and the required deposit.
5. The near-plugins `#[pause]` guard on `submit_blocks` checks `relayer.near`'s roles → `UnrestrictedSubmitBlocks` found → execution continues into the function body.
6. All headers are processed successfully via `submit_block_header`.
7. Execution reaches `self.run_mainchain_gc(num_of_headers)` at line 181.
8. The near-plugins `#[pause(except(roles(Role::UnrestrictedRunGC)))]` guard on `run_mainchain_gc` checks `relayer.near`'s roles → `UnrestrictedRunGC` **not** found → transaction panics.
9. The entire `submit_blocks` call is rolled back; no headers are stored; the canonical chain tip is not advanced. [5](#0-4)

    #[pause(except(roles(Role::UnrestrictedRunGC)))]
    pub fn run_mainchain_gc(&mut self, batch_size: u64) {
        let initial_blockheader = self
            .headers_pool
            .get(&self.mainchain_initial_blockhash)
            .unwrap_or_else(|| env::panic_str(ERR_KEY_NOT_EXIST));

        let tip_blockheader = self
            .headers_pool
            .get(&self.mainchain_tip_blockhash)
            .unwrap_or_else(|| env::panic_str(ERR_KEY_NOT_EXIST));

        let amount_of_headers_we_store =
            tip_blockheader.block_height - initial_blockheader.block_height + 1;

        if amount_of_headers_we_store > self.gc_threshold {
            let total_amount_to_remove = amount_of_headers_we_store - self.gc_threshold;
            let selected_amount_to_remove = std::cmp::min(total_amount_to_remove, batch_size);

            let start_removal_height = initial_blockheader.block_height;
            let end_removal_height = initial_blockheader.block_height + selected_amount_to_remove;
            env::log_str(&format!(
                "Num of blocks to remove {selected_amount_to_remove}"
            ));

            for height in start_removal_height..end_removal_height {
                let blockhash = &self
                    .mainchain_height_to_header
                    .get(&height)
                    .unwrap_or_else(|| env::panic_str(ERR_KEY_NOT_EXIST));

                self.remove_block_header(blockhash);
                self.mainchain_height_to_header.remove(&height);
            }

            self.mainchain_initial_blockhash = self
                .mainchain_height_to_header
                .get(&end_removal_height)
                .unwrap_or_else(|| env::panic_str(ERR_KEY_NOT_EXIST));
        }
    }
