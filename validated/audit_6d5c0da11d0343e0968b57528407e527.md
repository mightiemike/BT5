### Title
Synthetic-ratio truncation to zero in `AnchoredPriceProvider._getBidAndAskPrice` permanently bricks swaps for low-value base tokens — (`smart-contracts-poc/contracts/AnchoredPriceProvider.sol`)

---

### Summary

In `AnchoredPriceProvider._getBidAndAskPrice`, when `quoteFeedId` is set (synthetic ratio mode), the computed ratio `mid = Math.mulDiv(mid, ORACLE_DECIMALS, mid2)` can silently truncate to zero via integer division when the base token's oracle price is much smaller than the quote token's oracle price. No zero-check is performed on the resulting `mid` before it is forwarded to `_computeBidAsk`, which then returns the `FeedStalled` sentinel on every call, permanently reverting all swaps through the pool.

---

### Finding Description

In `_getBidAndAskPrice`, the synthetic ratio path reads both legs, validates `ok2` and `mid2 != 0`, then computes:

```solidity
mid = Math.mulDiv(mid, ORACLE_DECIMALS, mid2);
``` [1](#0-0) 

`ORACLE_DECIMALS = 1e8`. The formula is `mid_base * 1e8 / mid_quote`. When `mid_base * 1e8 < mid_quote`, Solidity integer division truncates the result to **zero**. No check is performed on the resulting `mid` after this line.

**Concrete example:**
- Base token (e.g. SHIB): $0.00001 → oracle price = `1000` (8-decimal units)
- Quote token (e.g. ETH): $3000 → oracle price = `300_000_000_000` (8-decimal units)
- Ratio: `Math.mulDiv(1000, 1e8, 300_000_000_000)` = `1e11 / 3e11` = **0** (truncated)

The zero `mid` is then passed to `_computeBidAsk`:

```solidity
uint256 refBid = _bandEdge(mid, BPS_BASE_U - half, Math.Rounding.Floor);
uint256 refAsk = _bandEdge(mid, BPS_BASE_U + half, Math.Rounding.Ceil);
if (refBid == 0 || refAsk > type(uint128).max || refBid >= refAsk) {
    return (0, type(uint128).max);
}
``` [2](#0-1) 

`_bandEdge(0, ...) = Math.mulDiv(0, ...) = 0`, so `refBid == 0` triggers the guard and returns `(0, type(uint128).max)`. The public `getBidAndAskPrice()` then reverts with `FeedStalled()` on every call:

```solidity
if (bid == 0 || ask == type(uint128).max) revert FeedStalled();
``` [3](#0-2) 

The existing guard only checks `mid2 != 0` (the denominator), not the result of the division:

```solidity
if (!ok2 || mid2 == 0) return (0, type(uint128).max);
mid = Math.mulDiv(mid, ORACLE_DECIMALS, mid2);
// ← no check that mid != 0 here
``` [4](#0-3) 

This is the direct analog of the external report: the Notional `getOraclePrice` validated that individual Chainlink prices were positive but did not validate that the computed ratio `(basePrice * quoteDecimals * RATE_DECIMALS) / (quotePrice * baseDecimals)` was also positive after truncation.

---

### Impact Explanation

Every swap through a pool whose `AnchoredPriceProvider` is configured in synthetic ratio mode with a low-value base token reverts with `FeedStalled()`. The pool's core swap functionality is permanently broken under normal, valid oracle conditions. LPs cannot trade through the pool; the pool is effectively insolvent as a trading venue for the duration of the price disparity.

---

### Likelihood Explanation

Synthetic ratio mode (`quoteFeedId != 0`) is an explicitly supported and documented feature. The truncation condition (`mid_base * 1e8 < mid_quote`) is met whenever the base token's USD price is less than `quote_price_USD / 1e8`. For ETH at $3000, any base token below $0.00003 triggers this. This covers a large class of real tokens (SHIB, PEPE, and similar micro-cap assets). The condition is persistent — it holds for as long as the price ratio remains large — so the DoS is not transient.

---

### Recommendation

Add a zero-check on the computed ratio immediately after the `Math.mulDiv` call in `_getBidAndAskPrice`:

```solidity
mid = Math.mulDiv(mid, ORACLE_DECIMALS, mid2);
if (mid == 0) return (0, type(uint128).max); // ratio truncated — fail closed
spreadBps += spreadBps2;
``` [5](#0-4) 

This mirrors the fix recommended in the external report: validate that the derived oracle value is strictly positive, not merely that its inputs are non-zero.

---

### Proof of Concept

1. Deploy `AnchoredPriceProvider` with `baseFeedId` = SHIB/USD feed, `quoteFeedId` = ETH/USD feed.
2. Oracle reports SHIB mid = `1000` (8-decimal, ≈ $0.00001), ETH mid = `300_000_000_000` (8-decimal, ≈ $3000).
3. Both `_readLeg` calls return `ok = true`, `mid2 != 0`.
4. `Math.mulDiv(1000, 1e8, 300_000_000_000)` = `0`.
5. `_computeBidAsk(0, spreadBps)` → `_bandEdge(0, ...) = 0` → `refBid == 0` → returns `(0, type(uint128).max)`.
6. `getBidAndAskPrice()` reverts with `FeedStalled()`.
7. Every swap through the pool reverts; the pool is permanently unusable for trading.
