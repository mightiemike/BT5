### Title
Gas-Key Nonce Fee Uses Wire Length Instead of Trie-ID Length for ML-DSA-65 Keys via Host Functions — (`runtime/near-vm-runner/src/logic/logic.rs`)

---

### Summary

`promise_batch_action_add_gas_key_with_full_access` and `promise_batch_action_add_gas_key_with_function_call` pass the raw borsh wire length of the public key (`public_key_len as usize`, 1953 bytes for ML-DSA-65) to `gas_key_add_key_exec_fee`, which uses it to compute the nonce trie-key length. The transaction path correctly passes `public_key.trie_id_len()` (33 bytes for ML-DSA-65). The two values diverge by a factor of ~59 for ML-DSA-65 keys, causing a ~59× overcharge on the nonce-write fee component for every contract that adds an ML-DSA-65 gas key via a host function.

---

### Finding Description

`gas_key_add_key_exec_fee` computes the per-nonce exec fee as:

```rust
let nonce_key_len =
    access_key_key_len(account_id_len, public_key_len) + size_of::<NonceIndex>();
```

The `public_key_len` argument is supposed to be the **on-trie identifier length** — the number of bytes the key actually occupies in the trie. For ML-DSA-65 that is 33 bytes (`[tag=3] || sha3_256(domain || raw_pubkey)`), not the 1953-byte borsh-encoded full pubkey.

**Transaction path** (`runtime/runtime/src/config.rs`, `permission_exec_fees`): correctly passes `public_key.trie_id_len()`.

**Host-function path** (`runtime/near-vm-runner/src/logic/logic.rs`, `promise_batch_action_add_gas_key_with_full_access`): passes `public_key_len as usize`, which is the borsh wire length the contract supplied — 1953 bytes for ML-DSA-65.

The same error appears in the wasmtime path (`runtime/near-vm-runner/src/wasmtime_runner/logic.rs`) and in `promise_batch_action_add_gas_key_with_function_call` in both backends.

The existing fee-parity test (`test_gas_key_fee_parity`) only exercises ED25519 keys (33-byte wire length == 33-byte trie-id length), so the divergence is invisible to it.

---

### Impact Explanation

For each nonce slot, the overcharged fee component is:

```
(1953 - 33) × gas_key_byte_exec_fee × num_nonces
= 1920 × gas_key_byte_exec_fee × num_nonces
```

This is a ~59× overcharge on the nonce-write exec-fee component relative to the transaction path. Contracts that add ML-DSA-65 gas keys via host functions:

1. Pay far more gas than the protocol intends for the operation.
2. May hit out-of-gas errors for operations that should succeed within the gas limit.
3. Violate the explicit fee-parity invariant between the transaction path and the host-function path (tested by `test_gas_key_fee_parity` for ED25519, but not for ML-DSA-65).

The divergence is deterministic and affects every invocation of the two host functions with an ML-DSA-65 key after `PostQuantumSignatures` and `GasKeys` are both active.

---

### Likelihood Explanation

- Triggered by any smart contract that calls `promise_batch_action_add_gas_key_with_full_access` or `promise_batch_action_add_gas_key_with_function_call` with an ML-DSA-65 public key.
- Both `PostQuantumSignatures` (stabilized at protocol version 85) and `GasKeys` must be active; once both are live the path is reachable by any unprivileged contract.
- No special privilege is required; any deployed contract can supply an ML-DSA-65 key blob of the correct borsh length.

---

### Recommendation

Replace `public_key_len as usize` with the decoded key's trie-id length in both host-function implementations, mirroring the transaction path:

```diff
// logic.rs – promise_batch_action_add_gas_key_with_full_access
  let public_key = self.get_public_key(public_key_ptr, public_key_len)?;
+ let decoded_pk = public_key.decode()?;
  let exec_fee = gas_key_add_key_exec_fee(
      &self.fees_config,
      receiver_id.len(),
-     public_key_len as usize,
+     PublicKey::from(decoded_pk.clone()).trie_id_len(),
      num_nonces,
  );
```

Apply the same fix to `promise_batch_action_add_gas_key_with_function_call` in both `logic.rs` and `wasmtime_runner/logic.rs`, and to `gas_key_transfer_exec_fee` / `gas_key_transfer_send_fee` call sites that similarly pass `public_key_len as usize`. Extend `test_gas_key_fee_parity` to cover `KeyType::MLDSA65`.

```rust
    let num_nonces = num_nonces as u64;
    let base =

        .checked_mul(num_nonces)
        .unwrap();
```

```rust
impl PublicKeyHandle {
    /// Length, in bytes, of this handle's on-trie borsh encoding: the raw
    /// handle bytes plus a 1-byte borsh discriminant tag (the leading
    /// `+ 1` in each arm). For ML-DSA-65 the handle is the 32-byte
    /// SHA3-256 digest, not the full 1952-byte pubkey.
    pub fn trie_id_len(&self) -> usize {
        match self {
            Self::ED25519(_) => 1 + ed25519_dalek::PUBLIC_KEY_LENGTH,
            Self::SECP256K1(_) => 1 + 64,
            Self::MlDsa65(_) => 1 + ML_DSA_65_HASH_LENGTH,
        }
    }
```
