### Title
`TryFrom<protobuf::ResourceBounds> for ValidResourceBounds` Silently Downgrades `AllResources` to `L1Gas`, Producing a Different Transaction Hash Than the One Signed and Stored — (`crates/apollo_protobuf/src/converters/transaction.rs`)

### Summary

The protobuf-to-internal conversion for `ResourceBounds` uses the heuristic `l1_data_gas.is_zero() && l2_gas.is_zero()` to decide whether to reconstruct a `ValidResourceBounds::L1Gas` or `ValidResourceBounds::AllResources` variant. Because `get_tip_resource_bounds_hash` includes the `L1_DATA_GAS` element in the Poseidon chain only for `AllResources`, any transaction that was signed and hash-committed as `AllResources` (with zero `l1_data_gas` and `l2_gas`) will be reconstructed as `L1Gas` after a P2P round-trip, yielding a strictly shorter hash preimage and therefore a different transaction hash. This is the direct sequencer analog of the Sherlock `updateYieldStrategy` bug: just as switching to a new yield strategy without calling `withdrawAll()` silently drops the old strategy's balance from `totalTokenBalanceStakers`, switching the resource-bounds variant from `AllResources` to `L1Gas` during protobuf conversion silently drops the `l1_data_gas` element from the hash preimage, breaking the canonicalization invariant that every representation of the same transaction must produce the same hash.

### Finding Description

**Step 1 — Gateway signs the transaction as `AllResources`.**

`RpcInvokeTransactionV3` carries `resource_bounds: AllResourceBounds` (always the three-resource variant). When the gateway converts it to `InternalRpcInvokeTransactionV3`, the field type is still `AllResourceBounds`. The `TransactionHasher` implementation calls `get_invoke_transaction_v3_hash`, which calls `get_tip_resource_bounds_hash` with `ValidResourceBounds::AllResources(self.resource_bounds)`. [1](#0-0) 

Inside `get_tip_resource_bounds_hash`, the `AllResources` arm appends a third element — `concat(L1_DATA_GAS, l1_data_gas)` — to the Poseidon chain, even when `l1_data_gas` is zero: [2](#0-1) 

So the canonical hash for a transaction with `l1_gas = X, l2_gas = 0, l1_data_gas = 0` is:

```
H1 = poseidon(tip, concat(L1_GAS, X), concat(L2_GAS, 0), concat(L1_DATA_GAS, 0))
```

**Step 2 — P2P protobuf conversion drops the variant information.**

When the transaction is serialised to protobuf and deserialised on a peer, `TryFrom<protobuf::ResourceBounds> for ValidResourceBounds` applies the heuristic: [3](#0-2) 

Because `l1_data_gas` and `l2_gas` are both zero, the condition `l1_data_gas.is_zero() && l2_gas.is_zero()` is true and the result is `ValidResourceBounds::L1Gas(l1_gas)`. The `AllResources` variant — and with it the `l1_data_gas` hash element — is silently discarded.

**Step 3 — The peer recomputes a different hash.**

With `ValidResourceBounds::L1Gas`, `get_tip_resource_bounds_hash` takes the `L1Gas(_) => vec![]` arm and produces only two elements:

```
H2 = poseidon(tip, concat(L1_GAS, X), concat(L2_GAS, 0))
```

`H1 ≠ H2`. The peer's computed hash diverges from the hash that was signed, stored, and broadcast by the originating node.

**Step 4 — Gateway does not block the triggering input.**

The stateless validator requires `l2_gas.max_price_per_unit >= min_gas_price` and `max_possible_fee > 0`, but it does not require `l2_gas.max_amount > 0` or `l1_data_gas.max_amount > 0`. A transaction with `l1_gas = { max_amount: 1, max_price_per_unit: 1 }` and both `l2_gas` and `l1_data_gas` zeroed passes all gateway checks when `min_gas_price = 0` (or when `l2_gas.max_price_per_unit` satisfies the price floor while `max_amount` remains zero): [4](#0-3) 

### Impact Explanation

Any node that recomputes the transaction hash from the protobuf-reconstructed body (e.g., for mempool deduplication, signature verification, or block commitment) will derive `H2` instead of `H1`. This matches the **High** impact category: *"Transaction conversion or signature/hash logic binds the wrong signer, hash, type, or executable payload."* Concretely:

- A peer that verifies the hash before admitting the transaction to its mempool will reject a valid, gateway-accepted transaction.
- A peer that does not re-verify will store the transaction under `H1` but execute it with `L1Gas` resource bounds, producing wrong gas accounting and a divergent state diff.

### Likelihood Explanation

The trigger requires `l2_gas.max_amount = 0` and `l1_data_gas.max_amount = 0` (and, depending on the `is_zero()` definition, `max_price_per_unit = 0` for both). This is reachable when `min_gas_price = 0` in the gateway configuration, which is a plausible testnet or early-mainnet setting. The user action is entirely unprivileged (a standard RPC `starknet_addInvokeTransaction` call).

### Recommendation

- **Short term:** In `TryFrom<protobuf::ResourceBounds> for ValidResourceBounds`, do not use a zero-value heuristic to infer the variant. Instead, add an explicit discriminant field to the protobuf `ResourceBounds` message (e.g., `bool has_all_resources`) so the original variant is preserved losslessly across serialisation boundaries.
- **Long term:** Add a round-trip property test asserting that for every `ValidResourceBounds` value, `serialize → deserialize` produces the identical variant and that `calculate_transaction_hash` returns the same value before and after the round-trip.

### Proof of Concept

```
1. Alice submits RpcInvokeTransactionV3 with:
     resource_bounds = AllResourceBounds {
       l1_gas:      { max_amount: 1, max_price_per_unit: 1 },
       l2_gas:      { max_amount: 0, max_price_per_unit: 0 },
       l1_data_gas: { max_amount: 0, max_price_per_unit: 0 },
     }
   (passes gateway stateless validation when min_gas_price = 0)

2. Gateway computes H1 via get_tip_resource_bounds_hash(AllResources):
     preimage = [tip, concat(L1_GAS,1,1), concat(L2_GAS,0,0), concat(L1_DATA,0,0)]
     H1 = poseidon(preimage)   // 4-element chain

3. Transaction is serialised to protobuf::ResourceBounds with
     l1_gas = {1,1}, l2_gas = {0,0}, l1_data_gas = {0,0}
   and broadcast to peer Bob.

4. Bob's TryFrom<protobuf::ResourceBounds>:
     l1_data_gas.is_zero() && l2_gas.is_zero()  →  true
     result = ValidResourceBounds::L1Gas({1,1})

5. Bob computes H2 via get_tip_resource_bounds_hash(L1Gas):
     preimage = [tip, concat(L1_GAS,1,1), concat(L2_GAS,0,0)]
     H2 = poseidon(preimage)   // 3-element chain

6. H1 ≠ H2.
   Bob rejects the transaction (hash mismatch) or stores it under H1
   but executes it with L1Gas bounds (wrong gas accounting).
```

pub fn get_tip_resource_bounds_hash(
    resource_bounds: &ValidResourceBounds,
    tip: &Tip,
) -> Result<Felt, StarknetApiError> {
    let l1_resource_bounds = resource_bounds.get_l1_bounds();
    let l2_resource_bounds = resource_bounds.get_l2_bounds();

    // L1 and L2 gas bounds always exist.
    // Old V3 txs always have L2 gas bounds of zero, but they exist.
    let mut resource_felts = vec![
        get_concat_resource(&l1_resource_bounds, L1_GAS)?,
        get_concat_resource(&l2_resource_bounds, L2_GAS)?,
    ];

    // For new V3 txs, need to also hash the data gas bounds.
    resource_felts.extend(match resource_bounds {
        ValidResourceBounds::L1Gas(_) => vec![],
        ValidResourceBounds::AllResources(all_resources) => {
            vec![get_concat_resource(&all_resources.l1_data_gas, L1_DATA_GAS)?]
        }
    });

    Ok(HashChain::new().chain(&tip.0.into()).chain_iter(resource_felts.iter()).get_poseidon_hash())
}

impl TryFrom<protobuf::ResourceBounds> for ValidResourceBounds {
    type Error = ProtobufConversionError;
    fn try_from(value: protobuf::ResourceBounds) -> Result<Self, Self::Error> {
        let Some(l1_gas) = value.l1_gas else {
            return Err(missing("ResourceBounds::l1_gas"));
        };
        let Some(l2_gas) = value.l2_gas else {
            return Err(missing("ResourceBounds::l2_gas"));
        };
        // TODO(Shahak): Assert data gas is not none once we remove support for 0.13.2.
        let l1_data_gas = value.l1_data_gas.unwrap_or_default();
        let l1_gas: ResourceBounds = l1_gas.try_into()?;
        let l2_gas: ResourceBounds = l2_gas.try_into()?;
        let l1_data_gas: ResourceBounds = l1_data_gas.try_into()?;
        Ok(if l1_data_gas.is_zero() && l2_gas.is_zero() {
            ValidResourceBounds::L1Gas(l1_gas)
        } else {
            ValidResourceBounds::AllResources(AllResourceBounds { l1_gas, l2_gas, l1_data_gas })
        })
    }
}
