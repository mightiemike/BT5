### Title
Protobuf `ValidResourceBounds` Round-Trip Misclassifies `AllResources` as `L1Gas` When `l2_gas` and `l1_data_gas` Are Zero, Breaking Hash Canonicalization Across P2P Sync - (`crates/apollo_protobuf/src/converters/transaction.rs`)

### Summary

The protobuf deserializer for `ValidResourceBounds` silently downgrades an `AllResources` transaction to `L1Gas` whenever both `l2_gas` and `l1_data_gas` are zero. Because `get_tip_resource_bounds_hash` produces structurally different hashes for these two variants (2 vs 3 resource elements), a transaction whose hash was committed under the `AllResources` encoding is stored and served under the `L1Gas` encoding after P2P sync, producing an authoritative-looking wrong resource-bounds representation and a hash that cannot be reproduced from the stored fields.

### Finding Description

**Root cause — `TryFrom<protobuf::ResourceBounds> for ValidResourceBounds`:**

```rust
// crates/apollo_protobuf/src/converters/transaction.rs  lines 417-436
impl TryFrom<protobuf::ResourceBounds> for ValidResourceBounds {
    fn try_from(value: protobuf::ResourceBounds) -> Result<Self, Self::Error> {
        let Some(l1_gas) = value.l1_gas else { ... };
        let Some(l2_gas) = value.l2_gas else { ... };
        // TODO(Shahak): Assert data gas is not none once we remove support for 0.13.2.
        let l1_data_gas = value.l1_data_gas.unwrap_or_default();   // ← silently zero
        let l1_gas: ResourceBounds = l1_gas.try_into()?;
        let l2_gas: ResourceBounds = l2_gas.try_into()?;
        let l1_data_gas: ResourceBounds = l1_data_gas.try_into()?;
        Ok(if l1_data_gas.is_zero() && l2_gas.is_zero() {
            ValidResourceBounds::L1Gas(l1_gas)          // ← wrong variant
        } else {
            ValidResourceBounds::AllResources(...)
        })
    }
}
```

The serializer always emits a present-but-zero `l1_data_gas` for `AllResources` transactions:

```rust
// crates/apollo_protobuf/src/converters/transaction.rs  lines 471-489
ValidResourceBounds::AllResources(AllResourceBounds { l1_gas, l2_gas, l1_data_gas }) =>
    protobuf::ResourceBounds {
        l1_gas: Some(l1_gas.into()),
        l2_gas: Some(l2_gas.into()),
        l1_data_gas: Some(l1_data_gas.into()),   // present, value = 0
    },
```

So when `l2_gas = 0` and `l1_data_gas = 0`, the round-trip is:

```
AllResources { l1_gas: X, l2_gas: 0, l1_data_gas: 0 }
  → protobuf { l1_gas: X, l2_gas: 0, l1_data_gas: 0 }
  → L1Gas(X)                          ← variant changed
```

**Hash divergence — `get_tip_resource_bounds_hash`:**

```rust
// crates/starknet_api/src/transaction_hash.rs  lines 188-211
pub fn get_tip_resource_bounds_hash(resource_bounds: &ValidResourceBounds, tip: &Tip)
    -> Result<Felt, StarknetApiError>
{
    let mut resource_felts = vec![
        get_concat_resource(&l1_resource_bounds, L1_GAS)?,
        get_concat_resource(&l2_resource_bounds, L2_GAS)?,
    ];
    resource_felts.extend(match resource_bounds {
        ValidResourceBounds::L1Gas(_) => vec![],                    // 2 elements
        ValidResourceBounds::AllResources(all) =>
            vec![get_concat_resource(&all.l1_data_gas, L1_DATA_GAS)?],  // 3 elements
    });
    Ok(HashChain::new().chain(&tip.0.into()).chain_iter(resource_felts.iter()).get_poseidon_hash())
}
```

- `AllResources { l1_gas: X, l2_gas: 0, l1_data_gas: 0 }` → `H(tip, L1_GAS_X, L2_GAS_0, L1_DATA_GAS_0)` — **3-element hash**
- `L1Gas(X)` → `H(tip, L1_GAS_X, L2_GAS_0)` — **2-element hash**

These are cryptographically distinct values.

**Trigger path:**

1. User submits `RpcInvokeTransactionV3 { resource_bounds: AllResourceBounds { l1_gas: X, l2_gas: 0, l1_data_gas: 0 } }`. The gateway accepts this (test case `valid_l1_gas` in `stateless_transaction_validator_test.rs` confirms zero `l2_gas`/`l1_data_gas` is valid).
2. `convert_rpc_tx_to_internal` computes `tx_hash = H_allresources` (3-element hash) and stores it.
3. Transaction is included in a block.
4. Block is synced via P2P; the `transaction_in_block` protobuf path uses `TryFrom<protobuf::ResourceBounds> for ValidResourceBounds`.
5. Receiving node deserializes: `l1_data_gas.is_zero() && l2_gas.is_zero()` → `ValidResourceBounds::L1Gas(X)`.
6. Receiving node stores the transaction with `L1Gas` resource bounds and the original `H_allresources` hash.
7. The stored hash `H_allresources` cannot be reproduced from the stored `L1Gas` bounds (would produce `H_l1gas ≠ H_allresources`).
8. RPC serves `L1Gas` resource bounds for a transaction that was signed and committed as `AllResources`.
9. Fee estimation and simulation on the receiving node use `GasVectorComputationMode::NoL2Gas` (L1Gas mode) instead of `GasVectorComputationMode::All` (AllResources mode), producing wrong fee estimates.

### Impact Explanation

The receiving node stores and serves an authoritative-looking but wrong resource-bounds representation for committed transactions. The stored `tx_hash` cannot be reproduced from the stored resource bounds, breaking hash canonicalization. Fee estimation and simulation on the receiving node use the wrong gas computation mode (`NoL2Gas` instead of `All`), returning wrong fee values. This matches:

- **High. RPC execution, fee estimation, tracing, simulation, or pending view returns an authoritative-looking wrong value.**
- **Critical. Wrong state, receipt, event, L1 message, class hash, storage value, or revert result** (the stored resource bounds are wrong for committed transactions).

### Likelihood Explanation

Any user submitting a V3 invoke transaction with only `l1_gas` set (the most common pattern for pre-0.13.3-style usage) triggers this. The gateway explicitly accepts `AllResourceBounds` with zero `l2_gas` and `l1_data_gas`. No special privileges are required. The misclassification occurs on every node that syncs the block via P2P.

### Recommendation

The serializer must distinguish `AllResources` from `L1Gas` in the wire format so the deserializer can reconstruct the correct variant. One approach: add a boolean or version tag to the protobuf `ResourceBounds` message indicating whether the sender intended `AllResources`. Alternatively, the deserializer should not downgrade to `L1Gas` when `l1_data_gas` is present (even if zero) — only downgrade when `l1_data_gas` is absent (the true 0.13.2 case):

```rust
// Proposed fix
let l1_data_gas_opt = value.l1_data_gas;
let l1_data_gas: ResourceBounds = l1_data_gas_opt
    .map(|v| v.try_into())
    .transpose()?
    .unwrap_or_default();
Ok(if l1_data_gas_opt.is_none() && l2_gas.is_zero() {
    // Only downgrade to L1Gas when l1_data_gas was truly absent (0.13.2 peer)
    ValidResourceBounds::L1Gas(l1_gas)
} else {
    ValidResourceBounds::AllResources(AllResourceBounds { l1_gas, l2_gas, l1_data_gas })
})
```

### Proof of Concept

```
1. Submit via RPC:
   RpcInvokeTransactionV3 {
     resource_bounds: AllResourceBounds { l1_gas: {max_amount:100, max_price:1}, l2_gas: {0,0}, l1_data_gas: {0,0} },
     ...
   }

2. Gateway computes tx_hash = get_invoke_transaction_v3_hash(...)
   → get_tip_resource_bounds_hash with AllResources
   → H(tip, L1_GAS_packed, L2_GAS_0, L1_DATA_GAS_0)   [3 elements]
   → tx_hash = H_3

3. Transaction committed to block with tx_hash = H_3.

4. Peer syncs block via P2P (transaction_in_block path):
   protobuf::ResourceBounds { l1_gas: {100,1}, l2_gas: {0,0}, l1_data_gas: {0,0} }
   → TryFrom: l1_data_gas.is_zero() && l2_gas.is_zero() → ValidResourceBounds::L1Gas({100,1})

5. Peer stores: tx_hash = H_3, resource_bounds = L1Gas({100,1})

6. Peer recomputes hash from stored bounds:
   → get_tip_resource_bounds_hash with L1Gas
   → H(tip, L1_GAS_packed, L2_GAS_0)   [2 elements]
   → H_2 ≠ H_3   ← hash mismatch

7. RPC on peer returns resource_bounds = {"L1_GAS": {100,1}}  (missing L2_GAS and L1_DATA_GAS keys)
   instead of {"L1_GAS":{100,1}, "L2_GAS":{0,0}, "L1_DATA_GAS":{0,0}}

8. Fee estimation on peer uses GasVectorComputationMode::NoL2Gas instead of All,
   returning wrong fee for any simulation that depends on the gas mode.

pub fn get_tip_resource_bounds_hash(
    resource_bounds: &ValidResourceBounds,
    tip: &Tip,
) -> Result<Felt, StarknetApiError> {
    let l1_resource_bounds = resource_bounds.get_l1_bounds();
    let l2_resource_bounds = resource_bounds.get_l2_bounds();

    // L1 and L2 gas bounds always exist.
    // Old V3 txs always have L2 gas bounds of zero, but they exist.
    let mut resource_felts = vec![
        get_concat_resource(&l1_resource_bounds, L1_GAS)?,
        get_concat_resource(&l2_resource_bounds, L2_GAS)?,
    ];

    // For new V3 txs, need to also hash the data gas bounds.
    resource_felts.extend(match resource_bounds {
        ValidResourceBounds::L1Gas(_) => vec![],
        ValidResourceBounds::AllResources(all_resources) => {
            vec![get_concat_resource(&all_resources.l1_data_gas, L1_DATA_GAS)?]
        }
    });

    Ok(HashChain::new().chain(&tip.0.into()).chain_iter(resource_felts.iter()).get_poseidon_hash())
}
