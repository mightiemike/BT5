### Title
Malicious Counterparty Can Steal Escrowed NFTs via Crafted `ClassId` Prefix Collision in `processReceivedPacket` — (`x/nft-transfer/keeper/packet.go`)

---

### Summary

`processReceivedPacket` uses `IsAwayFromOrigin` to decide whether to mint (away-from-origin) or unescrow (back-to-origin) an NFT. `IsAwayFromOrigin` is a pure string-prefix check against the **source** port/channel (the counterparty's channel name). A malicious counterparty can craft a `ClassId` that starts with their own `sourcePort/sourceChannel/` prefix, causing `IsAwayFromOrigin` to return `false`, then after `RemoveClassPrefix` strips that prefix, the remainder resolves to the full class path of a **different** legitimately-escrowed NFT on the receiving chain. `TransferOwner` then moves that NFT out of escrow to the attacker's receiver.

---

### Finding Description

**Root cause — `IsAwayFromOrigin` uses the counterparty's channel name, but `voucherClassID` is derived from the receiving chain's channel namespace**

`IsAwayFromOrigin` is defined as:

```go
func IsAwayFromOrigin(sourcePort, sourceChannel, fullClassPath string) bool {
    prefixClassID := GetClassPrefix(sourcePort, sourceChannel)
    return !strings.HasPrefix(fullClassPath, prefixClassID)
}
``` [1](#0-0) 

`GetClassPrefix` simply returns `"portID/channelID/"`. [2](#0-1) 

In `processReceivedPacket`, the `else` branch (back-to-origin) does:

```go
unprefixedClassID := types.RemoveClassPrefix(packet.GetSourcePort(),
    packet.GetSourceChannel(), data.ClassId)

voucherClassID := types.ParseClassTrace(unprefixedClassID).IBCClassID()
for _, tokenID := range data.TokenIds {
    if err := k.nftKeeper.TransferOwner(ctx,
        voucherClassID, tokenID, escrowAddress, receiver); err != nil {
        return err
    }
}
``` [3](#0-2) 

The `escrowAddress` is derived from the **destination** (local) port/channel:

```go
escrowAddress := types.GetEscrowAddress(packet.GetDestPort(), packet.GetDestChannel())
``` [4](#0-3) 

`ValidateBasic` on the packet data does **not** validate the `ClassId` format — it only checks the field is non-blank:

```go
if strings.TrimSpace(nftpd.ClassId) == "" {
    return newsdkerrors.Wrap(ErrInvalidClassID, "classId cannot be blank")
}
``` [5](#0-4) 

**Concrete exploit path:**

| Step | Chain A state |
|------|--------------|
| 1 | Chain A has `channel-0` → chain C; chain A has `channel-1` → chain B (malicious) |
| 2 | Chain C sends `nativeClass` to chain A via chain A's `channel-0` → chain A mints `ibc/H1` where `H1 = sha256("nft/channel-0/nativeClass")` |
| 3 | Some user sends `ibc/H1 / token1` from chain A to chain B via chain A's `channel-1` → `ibc/H1 / token1` is escrowed at `GetEscrowAddress("nft","channel-1")` |
| 4 | Chain B (malicious, its channel to chain A is named `channel-0` on chain B) sends a crafted packet: `data.ClassId = "nft/channel-0/nft/channel-0/nativeClass"`, `data.TokenIds = ["token1"]`, `data.Receiver = attacker_addr` |

Trace through `processReceivedPacket` on chain A:

- `packet.SourcePort="nft"`, `packet.SourceChannel="channel-0"` (chain B's channel name)
- `packet.DestPort="nft"`, `packet.DestChannel="channel-1"` (chain A's local channel)
- `IsAwayFromOrigin("nft","channel-0","nft/channel-0/nft/channel-0/nativeClass")` → `strings.HasPrefix(…, "nft/channel-0/")` = `true` → returns **`false`** → else branch taken
- `escrowAddress = GetEscrowAddress("nft","channel-1")` ← this is where `ibc/H1/token1` sits
- `RemoveClassPrefix("nft","channel-0","nft/channel-0/nft/channel-0/nativeClass")` → `"nft/channel-0/nativeClass"`
- `ParseClassTrace("nft/channel-0/nativeClass").IBCClassID()` → `"ibc/" + sha256("nft/channel-0/nativeClass")` = `"ibc/H1"`
- `TransferOwner(ctx, "ibc/H1", "token1", escrowAddress, attacker_addr)` → **succeeds**, NFT stolen

The collision is that chain B's `channel-0` and chain A's `channel-0` are entirely different channels that happen to share the same local name. `IsAwayFromOrigin` uses chain B's name to decide direction, but the resulting `voucherClassID` resolves against chain A's namespace.

---

### Impact Explanation

A malicious counterparty chain can steal any NFT escrowed at `GetEscrowAddress(destPort, destChannel)` on the receiving chain. The attacker:
1. Gets the target NFT transferred to their address on chain A
2. The legitimate owner's voucher on chain B becomes unredeemable (escrow is now empty)

This is direct, irreversible theft of economically valuable NFTs. The attacker needs no special privileges — only the ability to submit IBC packets from a counterparty chain (which any IBC-connected chain can do).

---

### Likelihood Explanation

- Any IBC-connected chain that is malicious (or whose validators are compromised) can execute this
- All required information (escrowed token IDs, full class paths) is publicly observable on-chain
- No governance, admin key, or social engineering required
- The only precondition is that the target chain has NFTs escrowed on a channel connected to the attacker's chain, which is the normal operating state of any active IBC NFT bridge

---

### Recommendation

The fix must ensure that the `classId` in the packet data was legitimately issued by the source chain for the specific channel. The standard ICS-721 approach is to verify that the `classId` prefix matches the **counterparty** port/channel as recorded in the channel end, not just the source port/channel string. Concretely:

1. When `IsAwayFromOrigin` returns `false`, verify that the `classId` prefix (`sourcePort/sourceChannel/`) corresponds to the **counterparty** of the receiving channel (i.e., `destChannel`'s counterparty must be `sourcePort/sourceChannel`), not just any channel with the same name.
2. Alternatively, validate that the unprefixed class ID after `RemoveClassPrefix` is a known class trace registered on the receiving chain (i.e., `HasClassTrace` check before proceeding to `TransferOwner`).

func RemoveClassPrefix(portID, channelID, classID string) string {
	classPrefix := GetClassPrefix(portID, channelID)
	return classID[len(classPrefix):]
}

// IsAwayFromOrigin determine if non-fungible token is moving away from
// the origin chain (the chain issued by the native nft).
// Note that fullClassPath refers to the full path of the unencoded classID.
// The longer the fullClassPath, the farther it is from the origin chain
func IsAwayFromOrigin(sourcePort, sourceChannel, fullClassPath string) bool {
	prefixClassID := GetClassPrefix(sourcePort, sourceChannel)
	return !strings.HasPrefix(fullClassPath, prefixClassID)
}

```go
func (k Keeper) processReceivedPacket(ctx sdk.Context, packet channeltypes.Packet,
	data types.NonFungibleTokenPacketData,
) error {
	receiver, err := sdk.AccAddressFromBech32(data.Receiver)
	if err != nil {
		return err
	}

	isAwayFromOrigin := types.IsAwayFromOrigin(packet.GetSourcePort(), packet.GetSourceChannel(), data.ClassId)

	// create the escrow address for creating denom and minting nft
	escrowAddress := types.GetEscrowAddress(packet.GetDestPort(), packet.GetDestChannel())

	if isAwayFromOrigin {
		// since SendPacket did not prefix the classID, we must prefix classID here
		classPrefix := types.GetClassPrefix(packet.GetDestPort(), packet.GetDestChannel())
		// NOTE: sourcePrefix contains the trailing "/"
		prefixedClassID := classPrefix + data.ClassId

		// construct the class trace from the full raw classID
		classTrace := types.ParseClassTrace(prefixedClassID)
		if !k.HasClassTrace(ctx, classTrace.Hash()) {
			k.SetClassTrace(ctx, classTrace)
		}

		voucherClassID := classTrace.IBCClassID()

		if !k.nftKeeper.HasDenomID(ctx, voucherClassID) {
			if err := k.nftKeeper.IssueDenom(ctx, voucherClassID, voucherClassID, "", data.ClassUri, escrowAddress); err != nil {
				return err
			}
		}
		sdkCtx := sdk.UnwrapSDKContext(ctx)
		sdkCtx.EventManager().EmitEvent(
			sdk.NewEvent(
				types.EventTypeClassTrace,
				sdk.NewAttribute(types.AttributeKeyTraceHash, classTrace.Hash().String()),
				sdk.NewAttribute(types.AttributeKeyClassID, voucherClassID),
			),
		)

		for i, tokenID := range data.TokenIds {
			if err := k.nftKeeper.MintNFT(ctx, voucherClassID, tokenID, "", data.TokenUris[i], "", escrowAddress, receiver); err != nil {
				return err
			}
		}
	} else {
		// If the token moves in the direction of back to origin,
		// we need to unescrow the token and transfer it to the receiver

		// we should remove the prefix. For example:
		// p6/c6/p4/c4/p2/c2/nftClass -> p4/c4/p2/c2/nftClass
		unprefixedClassID := types.RemoveClassPrefix(packet.GetSourcePort(),
			packet.GetSourceChannel(), data.ClassId)

		voucherClassID := types.ParseClassTrace(unprefixedClassID).IBCClassID()
		for _, tokenID := range data.TokenIds {
			if err := k.nftKeeper.TransferOwner(ctx,
				voucherClassID, tokenID, escrowAddress, receiver); err != nil {
				return err
			}
		}
	}

	return nil
```
