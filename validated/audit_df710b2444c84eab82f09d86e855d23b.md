### Title
`ValidResourceBounds::L1Gas` vs `AllResources` Misclassification in Protobuf Deserialization Causes P2P Rejection of Valid V3 Transactions and Hash Divergence - (File: `crates/apollo_protobuf/src/converters/transaction.rs`)

### Summary

The `TryFrom<protobuf::ResourceBounds> for ValidResourceBounds` converter in `crates/apollo_protobuf/src/converters/transaction.rs` uses a zero-value heuristic to classify a transaction's resource bounds variant: if both `l2_gas` and `l1_data_gas` are zero, it returns `ValidResourceBounds::L1Gas`. This misclassifies a valid V3 (`AllResources`) transaction as a pre-0.13.3 (`L1Gas`) transaction. Because `get_tip_resource_bounds_hash` in `crates/starknet_api/src/transaction_hash.rs` hashes a different number of elements for each variant, the same logical transaction produces two distinct hash values depending on which deserialization path is taken. The immediate observable consequence is that a valid V3 transaction accepted by the gateway is silently dropped by every P2P peer that receives it, because the P2P deserialization path hits this misclassification and then fails the `AllResources`-only guard in `TryFrom<InvokeTransactionV3> for RpcInvokeTransactionV3`.
