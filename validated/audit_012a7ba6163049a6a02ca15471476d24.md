### Title
`Declare` Transaction Bypasses `max_l2_gas_amount` Gateway Admission Check — (`File: crates/apollo_gateway/src/stateless_transaction_validator.rs`)

### Summary

`StatelessTransactionValidator::validate_resource_bounds()` enforces the `max_l2_gas_amount` cap for `Invoke` and `DeployAccount` transactions but explicitly skips it for `Declare` transactions. Any user can submit a `Declare V3` transaction with `l2_gas.max_amount` set to an arbitrarily large value (up to `u64::MAX = 18,446,744,073,709,551,615`) while the production limit is `1,210,000,000`. The gateway accepts and forwards such transactions to the mempool.

### Finding Description

Inside `validate_resource_bounds`, the check for `l2_gas.max_amount > config.max_l2_gas_amount` is guarded by a type branch that silently skips `Declare` transactions:

```rust
// TODO(Arni): Consider adding a validation for max_l2_gas_amount for declare.
if let RpcTransaction::Declare(_) = tx {
} else if resource_bounds.l2_gas.max_amount.0 > self.config.max_l2_gas_amount {
    return Err(StatelessTransactionValidatorError::MaxGasAmountTooHigh { … });
}
``` [1](#0-0) 

The production default for `max_l2_gas_amount` is `1,210,000,000` (≈ 1.21 × 10⁹): [2](#0-1) 

This is also the value deployed in the node's config schema: [3](#0-2) 

The bypass is confirmed by an existing test that explicitly asserts a `Declare` transaction with `max_amount: GasAmount(200)` passes validation when `max_l2_gas_amount: 100`: [4](#0-3) 

The same `validate_resource_bounds` function correctly rejects `Invoke` and `DeployAccount` transactions that exceed the limit: [5](#0-4) 

The `StatelessTransactionValidatorConfig` struct documents `max_l2_gas_amount` as "Maximum allowed L2 gas amount for transactions" — with no carve-out for Declare: [6](#0-5) 

### Impact Explanation

**High. Mempool/gateway/RPC admission accepts invalid transactions before sequencing.**

A `Declare V3` transaction with `l2_gas.max_amount = u64::MAX` passes `StatelessTransactionValidator::validate()` and enters the mempool. Downstream effects include:

1. **Fee estimation distortion**: `starknet_estimateFee` and `starknet_simulateTransactions` simulate against the declared gas bounds. A transaction claiming `u64::MAX` L2 gas forces the RPC layer to return an authoritative-looking wrong fee value, matching the "High. RPC execution, fee estimation … returns an authoritative-looking wrong value" impact.
2. **Bouncer/block capacity corruption**: The batcher's bouncer uses `l2_gas.max_amount` from the transaction to account for block capacity. A Declare with an astronomically large bound can exhaust the block's L2 gas budget, preventing legitimate transactions from being included.
3. **Mempool pollution**: Unlimited Declare transactions with `l2_gas.max_amount = u64::MAX` can be injected at the cost of a valid signature and fee, saturating the mempool with transactions that will revert at execution time (actual L2 gas used will be far below the bound, but the mempool slot is consumed).

### Likelihood Explanation

**High.** The trigger is unprivileged: any account can submit a `Declare V3` transaction. The only cost is a valid STRK fee payment satisfying `min_gas_price`. No special role, no privileged peer, no malformed bytes — the transaction is structurally valid and passes all other gateway checks. The TODO comment in the source confirms the gap is known but unresolved.

### Recommendation

Apply the same `max_l2_gas_amount` check to `Declare` transactions by removing the type-branch exemption:

```diff
-        // TODO(Arni): Consider adding a validation for max_l2_gas_amount for declare.
-        if let RpcTransaction::Declare(_) = tx {
-        } else if resource_bounds.l2_gas.max_amount.0 > self.config.max_l2_gas_amount {
+        if resource_bounds.l2_gas.max_amount.0 > self.config.max_l2_gas_amount {
             return Err(StatelessTransactionValidatorError::MaxGasAmountTooHigh {
                 gas_amount: resource_bounds.l2_gas.max_amount,
                 max_gas_amount: self.config.max_l2_gas_amount,
             });
         }
```

If Declare transactions legitimately require a higher L2 gas ceiling (e.g., because Sierra compilation is more expensive), introduce a separate `max_l2_gas_amount_declare` config field rather than removing the check entirely.

### Proof of Concept

Submit the following `Declare V3` transaction to the gateway's `add_declare_transaction` endpoint:

```json
{
  "type": "DECLARE",
  "version": "0x3",
  "sender_address": "<valid_account>",
  "compiled_class_hash": "<valid_casm_hash>",
  "contract_class": { … },
  "resource_bounds": {
    "l1_gas":      { "max_amount": "0x0",                    "max_price_per_unit": "0x0" },
    "l2_gas":      { "max_amount": "0xffffffffffffffff",     "max_price_per_unit": "<min_gas_price>" },
    "l1_data_gas": { "max_amount": "0x0",                    "max_price_per_unit": "0x0" }
  },
  "tip": "0x0",
  "nonce": "<current_nonce>",
  "nonce_data_availability_mode": "L1",
  "fee_data_availability_mode": "L1",
  "paymaster_data": [],
  "account_deployment_data": [],
  "signature": [ … ]
}
```

`l2_gas.max_amount = 0xffffffffffffffff` (`u64::MAX = 18,446,744,073,709,551,615`) is `~15,245×` the production limit of `1,210,000,000`. The gateway's `StatelessTransactionValidator::validate_resource_bounds()` returns `Ok(())` for this transaction because the `Declare` branch is taken at line 79, bypassing the `MaxGasAmountTooHigh` error that would be raised for an equivalent `Invoke` or `DeployAccount` transaction.     fn validate_resource_bounds(
        &self,
        tx: &RpcTransaction,
    ) -> StatelessTransactionValidatorResult<()> {
        if !self.config.validate_resource_bounds {
            return Ok(());
        }

        let resource_bounds = *tx.resource_bounds();
        // The resource bounds should be positive even without the tip.
        if ValidResourceBounds::AllResources(resource_bounds).max_possible_fee(Tip::ZERO) == Fee(0)
        {
            return Err(StatelessTransactionValidatorError::ZeroResourceBounds { resource_bounds });
        }

        if resource_bounds.l2_gas.max_price_per_unit.0 < self.config.min_gas_price {
            return Err(StatelessTransactionValidatorError::MaxGasPriceTooLow {
                gas_price: resource_bounds.l2_gas.max_price_per_unit,
                min_gas_price: self.config.min_gas_price,
            });
        }

        // TODO(Arni): Consider adding a validation for max_l2_gas_amount for declare.
        if let RpcTransaction::Declare(_) = tx {
        } else if resource_bounds.l2_gas.max_amount.0 > self.config.max_l2_gas_amount {
            return Err(StatelessTransactionValidatorError::MaxGasAmountTooHigh {
                gas_amount: resource_bounds.l2_gas.max_amount,
                max_gas_amount: self.config.max_l2_gas_amount,
            });
        }

        Ok(())
    }
