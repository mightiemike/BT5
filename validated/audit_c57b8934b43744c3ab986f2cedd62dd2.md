Audit Report

## Title
`SwapAllowlistExtension` Gates Router Address Instead of End-User, Allowing Any User to Bypass the Allowlist via `MetricOmmSimpleRouter` - (File: metric-periphery/contracts/extensions/SwapAllowlistExtension.sol)

## Summary
`SwapAllowlistExtension.beforeSwap` receives the `sender` argument from `MetricOmmPool.swap()`, which is always `msg.sender` of the pool call — the router contract when users route through `MetricOmmSimpleRouter`. If the pool admin allowlists the router address (the natural way to enable router-mediated swaps), every user regardless of individual allowlist status can bypass the per-user gate by calling through the router. The intended per-user restriction is completely nullified.

## Finding Description
`MetricOmmPool.swap()` passes `msg.sender` directly as `sender` to `_beforeSwap`: [1](#0-0) 

`ExtensionCalling._beforeSwap` forwards that value unchanged as the first argument to the extension hook: [2](#0-1) 

`SwapAllowlistExtension.beforeSwap` then checks `allowedSwapper[msg.sender][sender]`, where `msg.sender` is the pool and `sender` is the value forwarded above: [3](#0-2) 

When a user calls `MetricOmmSimpleRouter.exactInputSingle()`, the router calls `pool.swap()` directly: [4](#0-3) 

At that point, `msg.sender` inside `pool.swap()` is the router contract address, not the end-user. The allowlist check becomes `allowedSwapper[pool][router]`, not `allowedSwapper[pool][user]`. There is no mechanism in the router or pool to propagate the original end-user address to the extension. The existing guard in `SwapAllowlistExtension` is structurally insufficient because it trusts the direct caller of `pool.swap()` as the identity to gate, which is always the router when users route through it.

## Impact Explanation
A pool admin who sets `allowAllSwappers[pool] = false` and adds the router to `allowedSwapper` (the natural configuration to enable router-mediated swaps for allowlisted users) silently opens the pool to all users. Any unprivileged trader can call `MetricOmmSimpleRouter.exactInputSingle()` and execute swaps against a pool intended to be restricted to specific counterparties. This constitutes a broken core pool functionality (allowlist bypass) that can result in direct loss of LP assets from pools designed to restrict access to specific counterparties, meeting the contest threshold for a Medium/High impact finding.

## Likelihood Explanation
The `MetricOmmSimpleRouter` is the primary user-facing entry point for swaps. A pool admin enabling router-mediated swaps for their allowlisted users will naturally add the router address to `allowedSwapper` — this is a predictable, non-malicious configuration. No special attacker capability is required beyond calling the public `exactInputSingle` function. The condition is easily triggered and repeatable by any user.

## Recommendation
Pass the original end-user address through the call chain. One approach: add a `payer` or `originator` field to `extensionData` that the router populates with `msg.sender` before calling the pool, and have `SwapAllowlistExtension` decode and check that field instead of (or in addition to) the `sender` argument. Alternatively, document explicitly that `sender` is always the direct caller of `pool.swap()` and that per-user gating requires direct pool interaction only — router allowlisting must never be used with `SwapAllowlistExtension` when `allowAllSwappers` is false.

## Proof of Concept
1. Deploy `MetricOmmPool` with `SwapAllowlistExtension` configured.
2. Pool admin calls `setAllowAllSwappers(pool, false)`.
3. Pool admin calls `setAllowedToSwap(pool, router, true)` — intending to enable router-mediated swaps for allowlisted users.
4. Alice (not individually allowlisted) calls `MetricOmmSimpleRouter.exactInputSingle({pool: pool, ...})`.
5. Router calls `pool.swap(recipient, ...)` — `msg.sender` inside `swap()` = router address.
6. `_beforeSwap(sender=router, ...)` is dispatched to `SwapAllowlistExtension`.
7. Extension checks `allowedSwapper[pool][router] == true` → passes.
8. Alice's swap executes despite never being individually allowlisted, bypassing the per-user gate entirely.
