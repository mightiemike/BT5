### Title
`ValidResourceBounds` Protobuf Round-Trip Silently Downgrades `AllResources` to `L1Gas` When L2/L1DataGas Are Zero, Causing P2P Peers to Reject Valid Transactions — (`crates/apollo_protobuf/src/converters/transaction.rs`)

### Summary

A V3 transaction submitted with `AllResourceBounds { l1_gas: X, l2_gas: 0, l1_data_gas: 0 }` is accepted by the originating node and its hash is computed using the `AllResources` variant (which includes `L1_DATA_GAS` in the Poseidon preimage). When the transaction is propagated over P2P via protobuf, the deserialization path silently reclassifies it as `ValidResourceBounds::L1Gas(X)` because both `l2_gas` and `l1_data_gas` are zero. The receiving peer then fails to reconstruct `RpcInvokeTransactionV3` (which requires `AllResourceBounds`) and rejects the transaction entirely, creating a permanent split between the originating node's mempool and every other peer's mempool.

### Finding Description

**Step 1 — Hash domain split in `get_tip_resource_bounds_hash`**

`crates/starknet_api/src/transaction_hash.rs` lines 188–211 compute the fee-fields hash differently for the two variants:

```
L1Gas(X):        poseidon(tip, L1_GAS_packed, L2_GAS_zero_packed)          // 2 resource felts
AllResources{X,0,0}: poseidon(tip, L1_GAS_packed, L2_GAS_zero_packed, L1_DATA_GAS_zero_packed) // 3 resource felts
```

The two hashes are distinct even when the numeric values of all three bounds are identical. [1](#0-0) 

**Step 2 — Originating node always uses `AllResources`**

`RpcInvokeTransactionV3.resource_bounds` is typed as `AllResourceBounds` (not `ValidResourceBounds`). The `InvokeTransactionV3Trait` implementation for `InternalRpcInvokeTransactionV3` always wraps it as `ValidResourceBounds::AllResources(self.resource_bounds)`, so the hash is always computed with 3 resource felts. [2](#0-1) [3](#0-2) 

**Step 3 — Protobuf serialization preserves all three fields**

`From<ValidResourceBounds> for protobuf::ResourceBounds` always emits all three fields, including `l1_data_gas: Some(ResourceBounds::default().into())` for the `L1Gas` case and the actual zero value for `AllResources`. [4](#0-3) 

**Step 4 — Deserialization silently downgrades the variant**

`TryFrom<protobuf::ResourceBounds> for ValidResourceBounds` applies the condition:

```rust
Ok(if l1_data_gas.is_zero() && l2_gas.is_zero() {
    ValidResourceBounds::L1Gas(l1_gas)
} else {
    ValidResourceBounds::AllResources(...)
})
```

When the wire values are `{ l1_gas: X, l2_gas: 0, l1_data_gas: 0 }` — a perfectly legal `AllResources` transaction — the condition is true and the variant is silently changed to `L1Gas`. [5](#0-4) 

**Step 5 — Receiving peer rejects the transaction**

The P2P path reconstructs `RpcInvokeTransactionV3` (which requires `AllResourceBounds`) from the deserialized `ValidResourceBounds`. The same guard seen in the declare converter applies:

```rust
resource_bounds: match common.resource_bounds {
    ValidResourceBounds::AllResources(resource_bounds) => resource_bounds,
    _ => return Err(DEPRECATED_RESOURCE_BOUNDS_ERROR),
},
```

Because the deserialized variant is now `L1Gas`, the conversion returns `DEPRECATED_RESOURCE_BOUNDS_ERROR` and the transaction is dropped. [6](#0-5) 

### Impact Explanation

A user can submit a syntactically valid V3 invoke transaction with zero `l2_gas` and `l1_data_gas` bounds. The originating gateway accepts it, computes a correct hash, and inserts it into the local mempool. Every other sequencer node in the P2P network rejects it at the protobuf deserialization boundary. The transaction is invisible to all peers and can only be sequenced if the originating node happens to be the current block proposer. This matches the **High** impact: *"Mempool/gateway/RPC admission accepts invalid transactions or rejects valid transactions before sequencing."*

### Likelihood Explanation

The trigger requires only a standard V3 invoke transaction with `max_amount = 0` and `max_price_per_unit = 0` for both `l2_gas` and `l1_data_gas`. No privileged access, no malformed bytes, and no special account contract are needed. Any user can craft this transaction through the public RPC endpoint.

### Recommendation

Replace the heuristic variant-selection condition in `TryFrom<protobuf::ResourceBounds> for ValidResourceBounds` with an explicit discriminant field in the protobuf schema (e.g., a `bool is_all_resources` flag or a `oneof` wrapper). Until the schema is updated, the deserializer should default to `AllResources` whenever `l1_data_gas` is present in the wire message, regardless of its numeric value, to preserve the sender's intent and hash domain.

### Proof of Concept

1. Construct an `RpcInvokeTransactionV3` with:
   ```
   resource_bounds = AllResourceBounds {
       l1_gas:      { max_amount: 1000, max_price_per_unit: 1 },
       l2_gas:      { max_amount: 0,    max_price_per_unit: 0 },
       l1_data_gas: { max_amount: 0,    max_price_per_unit: 0 },
   }
   ```
2. Submit via the HTTP gateway. The originating node accepts it and stores it with hash `H_all = poseidon(tip, L1_GAS_packed, L2_GAS_zero, L1_DATA_GAS_zero)`.
3. The node serializes the transaction to `protobuf::MempoolTransaction` and broadcasts it. The wire message contains `{ l1_gas: ..., l2_gas: 0, l1_data_gas: 0 }`.
4. A receiving peer deserializes `protobuf::ResourceBounds` → `ValidResourceBounds::L1Gas(l1_gas)` (condition `l1_data_gas.is_zero() && l2_gas.is_zero()` is true).
5. The peer attempts `ValidResourceBounds::L1Gas → AllResourceBounds` and returns `DEPRECATED_RESOURCE_BOUNDS_ERROR`.
6. The transaction is silently dropped on every peer. The originating node's mempool contains a transaction that no other node will ever sequence.

pub fn get_tip_resource_bounds_hash(
    resource_bounds: &ValidResourceBounds,
    tip: &Tip,
) -> Result<Felt, StarknetApiError> {
    let l1_resource_bounds = resource_bounds.get_l1_bounds();
    let l2_resource_bounds = resource_bounds.get_l2_bounds();

    // L1 and L2 gas bounds always exist.
    // Old V3 txs always have L2 gas bounds of zero, but they exist.
    let mut resource_felts = vec![
        get_concat_resource(&l1_resource_bounds, L1_GAS)?,
        get_concat_resource(&l2_resource_bounds, L2_GAS)?,
    ];

    // For new V3 txs, need to also hash the data gas bounds.
    resource_felts.extend(match resource_bounds {
        ValidResourceBounds::L1Gas(_) => vec![],
        ValidResourceBounds::AllResources(all_resources) => {
            vec![get_concat_resource(&all_resources.l1_data_gas, L1_DATA_GAS)?]
        }
    });

    Ok(HashChain::new().chain(&tip.0.into()).chain_iter(resource_felts.iter()).get_poseidon_hash())
}

impl TryFrom<protobuf::ResourceBounds> for ValidResourceBounds {
    type Error = ProtobufConversionError;
    fn try_from(value: protobuf::ResourceBounds) -> Result<Self, Self::Error> {
        let Some(l1_gas) = value.l1_gas else {
            return Err(missing("ResourceBounds::l1_gas"));
        };
        let Some(l2_gas) = value.l2_gas else {
            return Err(missing("ResourceBounds::l2_gas"));
        };
        // TODO(Shahak): Assert data gas is not none once we remove support for 0.13.2.
        let l1_data_gas = value.l1_data_gas.unwrap_or_default();
        let l1_gas: ResourceBounds = l1_gas.try_into()?;
        let l2_gas: ResourceBounds = l2_gas.try_into()?;
        let l1_data_gas: ResourceBounds = l1_data_gas.try_into()?;
        Ok(if l1_data_gas.is_zero() && l2_gas.is_zero() {
            ValidResourceBounds::L1Gas(l1_gas)
        } else {
            ValidResourceBounds::AllResources(AllResourceBounds { l1_gas, l2_gas, l1_data_gas })
        })
    }
