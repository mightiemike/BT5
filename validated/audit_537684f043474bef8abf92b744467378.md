Audit Report

## Title
`SwapAllowlistExtension` Checks Router Address Instead of Originating User, Enabling Allowlist Bypass ŌĆö (`metric-periphery/contracts/extensions/SwapAllowlistExtension.sol`)

## Summary
`SwapAllowlistExtension.beforeSwap` checks the `sender` argument against the per-pool allowlist, but when swaps are routed through `MetricOmmSimpleRouter`, the pool's `msg.sender` ŌĆö and therefore the `sender` forwarded to the extension ŌĆö is the router contract, not the originating user. This causes two fund-impacting failures: allowlisted users are blocked from using the router, and any user can bypass the allowlist once the router is added to it.

## Finding Description
`MetricOmmPool.swap` passes `msg.sender` as the first argument to `_beforeSwap`: [1](#0-0) 

`ExtensionCalling._beforeSwap` encodes that `sender` verbatim into the call to each configured extension: [2](#0-1) 

`SwapAllowlistExtension.beforeSwap` then evaluates `allowedSwapper[msg.sender][sender]`, where `msg.sender` is the pool and `sender` is the pool's immediate caller: [3](#0-2) 

`MetricOmmSimpleRouter.exactInputSingle` calls `pool.swap(...)` directly with no user identity in `extensionData` (passes `""`): [4](#0-3) 

At that point, `msg.sender` to the pool is the router. The pool passes the router address as `sender` to `_beforeSwap`. The extension evaluates `allowedSwapper[pool][router]` ŌĆö not `allowedSwapper[pool][user]`. The originating user's allowlist status is never consulted. The same mismatch applies to `exactInput`, `exactOutputSingle`, and `exactOutput`. [5](#0-4) [6](#0-5) [7](#0-6) 

## Impact Explanation
Two direct fund-impacting consequences follow:

**1. Allowlisted users cannot swap through the router.** A pool configured with `SwapAllowlistExtension` and a curated allowlist will reject every router-mediated swap because the router address is not in the allowlist. This breaks the core swap flow for the intended user population ŌĆö a broken core pool functionality impact.

**2. Any user can bypass the allowlist once the router is allowlisted.** The natural operator remediation for consequence #1 is to add the router to the allowlist. This sets `allowedSwapper[pool][router] = true`, causing the extension to pass for every swap routed through the router regardless of who the originating user is. A non-allowlisted user (e.g., a sanctioned address) can bypass the curation gate entirely by calling `exactInputSingle` on the public router. The allowlist is the sole access-control mechanism for curated pools; its bypass is a direct policy failure with fund-level consequences ŌĆö the pool receives input tokens from and delivers output tokens to actors the pool admin explicitly excluded. [8](#0-7) 

## Likelihood Explanation
The router is the standard, documented periphery entry point for swaps. Pool admins who configure an allowlist will encounter the broken-router problem immediately when their allowlisted users try to use the router. The natural remediation ŌĆö allowlisting the router ŌĆö is the exact step that opens the bypass. No special knowledge or privileged access is required; any public user can call `MetricOmmSimpleRouter.exactInputSingle` with any pool address. [9](#0-8) 

## Recommendation
The extension must gate the originating user, not the immediate pool caller. The cleanest fix: the router passes the originating `msg.sender` inside `extensionData`, and the extension decodes and checks that address when the immediate `sender` is a recognized router. Alternatively, the router could be extended to accept a user-supplied identity field that is ABI-encoded into `extensionData` before calling `pool.swap`, and `SwapAllowlistExtension.beforeSwap` would decode and validate that field when `sender` matches a known router registry. [10](#0-9) 

## Proof of Concept
```
Setup:
  - Deploy pool with SwapAllowlistExtension
  - Pool admin calls setAllowedToSwap(pool, alice, true)   // alice is KYC'd
  - Pool admin does NOT allowlist bob or the router

Step 1 ŌĆö Allowlisted user blocked through router:
  - alice calls MetricOmmSimpleRouter.exactInputSingle(pool, ...)
  - Router calls pool.swap(..., "", params.extensionData)
    ŌåÆ msg.sender to pool = router address
  - Pool calls _beforeSwap(sender=router, ...)
  - Extension checks allowedSwapper[pool][router] ŌåÆ false
  - Revert: NotAllowedToSwap  ŌåÉ alice cannot use the router

Step 2 ŌĆö Operator "fixes" by allowlisting the router:
  - Pool admin calls setAllowedToSwap(pool, router, true)

Step 3 ŌĆö Non-allowlisted user bypasses the allowlist:
  - bob (not allowlisted) calls MetricOmmSimpleRouter.exactInputSingle(pool, ...)
  - Router calls pool.swap(...) ŌåÆ msg.sender to pool = router
  - Extension checks allowedSwapper[pool][router] ŌåÆ true
  - Swap succeeds ŌåÉ bob bypasses the allowlist
``` [3](#0-2) [11](#0-10)

  function exactInput(ExactInputParams calldata params) external payable returns (uint256 amountOut) {
    _checkDeadline(params.deadline);
    _validatePath(params.tokens, params.pools, params.extensionDatas);

    uint256 last = params.pools.length - 1;
    int128 amount = MetricOmmSwapInputs.asAmountSpecifiedIn(params.amountIn);

    for (uint256 i = 0; i <= last; i++) {
      address pool = params.pools[i];
      bool zeroForOne = MetricOmmSwapPath.resolveZeroForOneBitmap(params.zeroForOneBitMap, i);

      _setNextCallbackContext(pool, CALLBACK_MODE_JUST_PAY, i == 0 ? msg.sender : address(this), params.tokens[i]);
      (int128 amount0Delta, int128 amount1Delta) = IMetricOmmPoolActions(pool)
        .swap(
          i == last ? params.recipient : address(this),
          zeroForOne,
          amount,
          MetricOmmSwapPath.openLimit(zeroForOne),
          "",
          params.extensionDatas[i]
        );

      int128 amountInActual = MetricOmmSwapResults.extractAmountIn(zeroForOne, amount0Delta, amount1Delta);
      if (amountInActual < amount) revert InvalidInputAmountAtHop(uint8(i), amountInActual, amount);

      amount = MetricOmmSwapResults.extractAmountOut(zeroForOne, amount0Delta, amount1Delta);
    }

    if (amount <= 0) revert InvalidSwapDeltas();
    amountOut = MetricOmmSwapInputs.int128ToUint128(amount);
    if (amountOut < params.amountOutMinimum) revert InsufficientOutput(amountOut, params.amountOutMinimum);

    _clearExpectedCallbackPool();
  }

  function exactOutputSingle(ExactOutputSingleParams calldata params) external payable returns (uint256 amountIn) {
    _checkDeadline(params.deadline);
    uint128 priceLimitX64 = MetricOmmSwapPath.normalizePriceLimit(params.zeroForOne, params.priceLimitX64);

    int128 expectedAmountOut = MetricOmmSwapInputs.asAmountSpecifiedIn(params.amountOut);

    if (amountOut != expectedAmountOut) revert InvalidOutputAmount(amountOut, params.amountOut);

    );

    _clearExpectedCallbackPool();
  }
