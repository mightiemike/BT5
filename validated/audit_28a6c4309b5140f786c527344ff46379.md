### Title
`ValidResourceBounds::L1Gas` Misclassification in Protobuf Deserialization Silently Rejects Valid `AllResources` Invoke V3 Transactions via Consensus and Mempool P2P — (`crates/apollo_protobuf/src/converters/transaction.rs`)

### Summary

The protobuf-to-Rust conversion for `ValidResourceBounds` uses a zero-value heuristic to distinguish pre-0.13.3 (`L1Gas`) from post-0.13.3 (`AllResources`) transactions. A valid `AllResources` Invoke V3 transaction whose `l2_gas` and `l1_data_gas` are both zero is silently reclassified as `ValidResourceBounds::L1Gas` during deserialization. The consensus and mempool P2P paths then attempt to convert the resulting `InvokeTransactionV3` into `RpcInvokeTransactionV3`, which hard-rejects the `L1Gas` variant, causing the transaction to be dropped. The same misclassification also produces a different transaction hash in the state-sync path, breaking hash validation for those transactions.

### Finding Description

**Root cause — wrong variant selected during protobuf deserialization**

`TryFrom<protobuf::ResourceBounds> for ValidResourceBounds` in `crates/apollo_protobuf/src/converters/transaction.rs` applies a heuristic: [1](#0-0) 

```rust
Ok(if l1_data_gas.is_zero() && l2_gas.is_zero() {
    ValidResourceBounds::L1Gas(l1_gas)          // ← chosen for ANY zero-valued AllResources tx
} else {
    ValidResourceBounds::AllResources(...)
})
```

A user-submitted `AllResources` transaction with `l2_gas = {0,0}` and `l1_data_gas = {0,0}` (only L1 gas used) is serialized by the sending node with those zero fields present in the protobuf wire format. On the receiving node the heuristic fires and produces `L1Gas` instead of `AllResources`.

**Consensus / Mempool P2P path — conversion hard-fails**

`TryFrom<protobuf::InvokeV3WithProof> for RpcInvokeTransactionV3` first converts the inner `InvokeV3` to `InvokeTransactionV3` (which carries `ValidResourceBounds`), then calls `snapi_invoke.try_into()` to reach `RpcInvokeTransactionV3`: [2](#0-1) 

`TryFrom<InvokeTransactionV3> for RpcInvokeTransactionV3` rejects any variant that is not `AllResources`: [3](#0-2) 

```rust
resource_bounds: match value.resource_bounds {
    ValidResourceBounds::AllResources(bounds) => bounds,
    _ => return Err(StarknetApiError::OutOfRange { ... }),  // ← L1Gas hits this
},
```

The error is mapped to `DEPRECATED_RESOURCE_BOUNDS_ERROR` and the transaction is dropped. [4](#0-3) 

The same path is taken for mempool P2P: [5](#0-4) 

**State-sync path — wrong transaction hash**

`TryFrom<protobuf::InvokeV3> for InvokeTransactionV3` also uses the same `ValidResourceBounds::try_from` call: [6](#0-5) 

The resulting `InvokeTransactionV3` carries `ValidResourceBounds::L1Gas`. `get_tip_resource_bounds_hash` then omits the `L1_DATA_GAS` field from the Poseidon hash: [7](#0-6) 

```rust
resource_felts.extend(match resource_bounds {
    ValidResourceBounds::L1Gas(_) => vec![],          // ← L1_DATA_GAS omitted
    ValidResourceBounds::AllResources(all_resources) => {
        vec![get_concat_resource(&all_resources.l1_data_gas, L1_DATA_GAS)?]
    }
});
```

The original transaction was hashed with `AllResources` (L1_DATA_GAS included). The re-computed hash differs, so `validate_transaction_hash` rejects the transaction. [8](#0-7) 

**Why the gateway is unaffected**

`RpcInvokeTransactionV3` always stores `AllResourceBounds` directly: [9](#0-8) 

The gateway converts `RpcTransaction` → `InternalRpcTransactionWithoutTxHash` → hash, always using `ValidResourceBounds::AllResources`. The divergence only appears when the transaction crosses a P2P boundary and is deserialized through the `InvokeTransactionV3` intermediate type. [10](#0-9) 

### Impact Explanation

- **Consensus P2P**: A valid `AllResources` Invoke V3 transaction with zero `l2_gas` and `l1_data_gas` is rejected by every receiving sequencer node. The transaction can never be included in a block even though it was accepted by the gateway and is cryptographically valid.
- **Mempool P2P**: Same rejection for transactions propagated between mempool peers.
- **State-sync P2P**: The transaction hash computed on the syncing node differs from the canonical hash, causing `validate_transaction_hash` to fail and the synced block to be rejected.

This matches: **High — Mempool/gateway/RPC admission rejects valid transactions before sequencing** and **High — Transaction conversion or signature/hash logic binds the wrong hash or executable payload**.

### Likelihood Explanation

Any user who submits an Invoke V3 transaction specifying only L1 gas (setting `l2_gas` and `l1_data_gas` to `{max_amount: 0, max_price_per_unit: 0}`) triggers this path. The gateway's stateless validator explicitly allows such configurations: [11](#0-10) 

No special privilege is required; any unprivileged user can submit such a transaction.

### Recommendation

Replace the intermediate `InvokeTransactionV3` step in `TryFrom<protobuf::InvokeV3WithProof> for RpcInvokeTransactionV3` with a direct conversion that uses `TryFrom<protobuf::ResourceBounds> for AllResourceBounds` (already defined in `rpc_transaction.rs`): [12](#0-11) 

This converter requires `l1_data_gas` to be present and always produces `AllResourceBounds`, eliminating the misclassification. The `TryFrom<protobuf::ResourceBounds> for ValidResourceBounds` heuristic should be retained only for the legacy state-sync path (`InvokeTransactionV3`) where backward compatibility with pre-0.13.3 wire data is genuinely needed.

### Proof of Concept

1. Submit an Invoke V3 transaction via the gateway with `resource_bounds = AllResourceBounds { l1_gas: {max_amount: 1000, max_price_per_unit: 1}, l2_gas: {0, 0}, l1_data_gas: {0, 0} }`.
2. The gateway accepts it; `convert_rpc_tx_to_internal` computes the hash using `ValidResourceBounds::AllResources` (includes `L1_DATA_GAS` in the Poseidon hash).
3. The transaction is serialized to `protobuf::InvokeV3WithProof` with `resource_bounds.l2_gas = {0,0}` and `resource_bounds.l1_data_gas = {0,0}`.
4. A peer receives it via consensus or mempool P2P. `TryFrom<protobuf::InvokeV3> for InvokeTransactionV3` fires the heuristic and produces `ValidResourceBounds::L1Gas`.
5. `TryFrom<InvokeTransactionV3> for RpcInvokeTransactionV3` returns `Err(OutOfRange)` → mapped to `DEPRECATED_RESOURCE_BOUNDS_ERROR` → transaction dropped.
6. For state-sync: the same `L1Gas` classification causes `get_tip_resource_bounds_hash` to omit `L1_DATA_GAS`, producing a hash that does not match the canonical hash stored in the block, causing `validate_transaction_hash` to return `false`.

impl TryFrom<protobuf::ResourceBounds> for ValidResourceBounds {
    type Error = ProtobufConversionError;
    fn try_from(value: protobuf::ResourceBounds) -> Result<Self, Self::Error> {
        let Some(l1_gas) = value.l1_gas else {
            return Err(missing("ResourceBounds::l1_gas"));
        };
        let Some(l2_gas) = value.l2_gas else {
            return Err(missing("ResourceBounds::l2_gas"));
        };
        // TODO(Shahak): Assert data gas is not none once we remove support for 0.13.2.
        let l1_data_gas = value.l1_data_gas.unwrap_or_default();

        let l2_gas: ResourceBounds = l2_gas.try_into()?;
        let l1_data_gas: ResourceBounds = l1_data_gas.try_into()?;

            ValidResourceBounds::L1Gas(l1_gas)
        } else {
            ValidResourceBounds::AllResources(AllResourceBounds { l1_gas, l2_gas, l1_data_gas })
        })
    }

impl TryFrom<protobuf::InvokeV3WithProof> for RpcInvokeTransactionV3 {
    type Error = ProtobufConversionError;
    fn try_from(mut value: protobuf::InvokeV3WithProof) -> Result<Self, Self::Error> {
        // Extract proof first, since `starknet_api::transaction::InvokeTransactionV3` does not
        // carry a `proof` field.
        let proof = Proof::from(std::mem::take(&mut value.proof));

        let snapi_invoke: InvokeTransactionV3 = value
            .invoke
            .ok_or(ProtobufConversionError::MissingField {
                field_description: "InvokeV3WithProof::invoke",
            })?
            .try_into()?;

        // This conversion can fail only if the resource_bounds are not AllResources.
        Ok(Self { proof, ..snapi_invoke.try_into().map_err(|_| DEPRECATED_RESOURCE_BOUNDS_ERROR)? })
    }

/// Validates the hash of a starknet transaction.
/// For transactions on testnet or those with a low block_number, we validate the
/// transaction hash against all potential historical hash computations. For recent
/// transactions on mainnet, the hash is validated by calculating the precise hash
/// based on the transaction version.
pub fn validate_transaction_hash(
    transaction: &Transaction,
    block_number: &BlockNumber,
    chain_id: &ChainId,
    expected_hash: TransactionHash,
    transaction_options: &TransactionOptions,
) -> Result<bool, StarknetApiError> {
    let mut possible_hashes = get_deprecated_transaction_hashes(
        chain_id,
        block_number,
        transaction,
        transaction_options,
    )?;
    possible_hashes.push(get_transaction_hash(transaction, chain_id, transaction_options)?);
    Ok(possible_hashes.contains(&expected_hash))
}
