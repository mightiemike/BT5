### Title
Unchecked `transferFrom` Return Value in `replaceUsdcEWithUsdc` Allows usdcE Drain Without USDC Payment ‚Äî (`File: core/contracts/ContractOwner.sol`)

---

### Summary

`ContractOwner.replaceUsdcEWithUsdc` is a permissionless migration helper that is supposed to atomically swap USDC-e held in a `DirectDepositV1` (DDA) for native USDC supplied by the caller. The USDC inbound leg at line 616 uses a raw `.transferFrom()` call whose boolean return value is never checked. If the call returns `false` instead of reverting, execution continues, the DDA's USDC-e is withdrawn to `ContractOwner`, and then forwarded to the caller ‚Äî who paid nothing.

---

### Finding Description

`ContractOwner` declares `using ERC20Helper for IERC20Base` at line 24, giving every `IERC20Base` instance access to the audited `safeTransferFrom` wrapper in `ERC20Helper`. That wrapper low-level-calls the token and asserts `success && (data.length == 0 || abi.decode(data, (bool)))`.

The migration function ignores this wrapper for the inbound USDC leg:

```solidity
// core/contracts/ContractOwner.sol  line 616
IERC20Base(usdc).transferFrom(msg.sender, directDepositV1, balance);
DirectDepositV1(directDepositV1).withdraw(IIERC20Base(usdcE));   // pulls usdcE ‚Üí ContractOwner
IERC20Base(usdcE).safeTransfer(msg.sender, balance);              // sends usdcE ‚Üí caller
```

The outbound leg on line 618 correctly uses `safeTransfer`. Only the inbound leg is unguarded.

`replaceUsdcEWithUsdc` carries no `onlyOwner` / `onlyDeployer` modifier ‚Äî the sole gate is `require(block.chainid == 57073)`. Any EOA or contract on the Ink chain (chain ID 57073) can call it for any `subaccount` whose DDA holds a non-zero usdcE balance.

---

### Impact Explanation

If the USDC token at `0x2D270e6886d130D724215A266106e6832161EAEd` returns `false` on a failed `transferFrom` (e.g., caller has zero allowance or zero balance) rather than reverting, the three-step sequence completes:

1. USDC inbound transfer silently fails ‚Äî caller pays nothing.
2. `DirectDepositV1.withdraw` moves all usdcE from the DDA to `ContractOwner`.
3. `safeTransfer` moves that usdcE from `ContractOwner` to the caller.

The DDA is drained of its usdcE balance. The broken invariant is the 1-for-1 exchange guarantee: usdcE must only leave the DDA after an equal amount of USDC has been received. Any DDA on the Ink deployment that holds usdcE is at risk.

---

### Likelihood Explanation

The function is permissionless on chain 57073. The exploitability depends on whether the specific USDC deployment at the hardcoded address reverts or returns `false` on failure. Many USDC forks and bridged variants (including some Circle deployments on newer L2s) do revert, which would prevent exploitation in practice. However, the code is structurally incorrect regardless: the return value is never checked, and the protocol's own `ERC20Helper.safeTransferFrom` exists precisely to handle this class of token. If the token's behavior changes (upgrade, replacement, or a different token is registered at that address), the vulnerability becomes immediately exploitable. Likelihood is **medium** given the permissionless entry and the structural defect, tempered by the probability that the specific token reverts.

---

### Recommendation

Replace the raw call with the safe wrapper already available in scope:

```solidity
// Before (line 616)
IERC20Base(usdc).transferFrom(msg.sender, directDepositV1, balance);

// After
IERC20Base(usdc).safeTransferFrom(msg.sender, directDepositV1, balance);
```

`ERC20Helper.safeTransferFrom` is already imported and bound via `using ERC20Helper for IERC20Base` at line 24 of `ContractOwner`, so no additional imports are required.

6. Attacker holds the DDA's entire usdcE balance; no USDC was paid.

    function safeTransferFrom(
        IERC20Base self,
        address from,
        address to,
        uint256 amount
    ) internal {
        (bool success, bytes memory data) = address(self).call(
            abi.encodeWithSelector(
                IERC20Base.transferFrom.selector,
                from,
                to,
                amount
            )
        );

        require(
            success && (data.length == 0 || abi.decode(data, (bool))),
            ERR_TRANSFER_FAILED
        );
    }
