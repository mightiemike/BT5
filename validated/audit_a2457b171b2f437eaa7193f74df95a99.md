### Title
Bootstrap `validate_resource_bounds = false` Disables All Resource Bounds Checks, Admitting Zero-Fee Transactions to the Priority Queue - (File: `crates/apollo_gateway/src/stateless_transaction_validator.rs`, `crates/apollo_mempool/src/fee_transaction_queue.rs`)

---

### Summary

The shared `validate_resource_bounds` bootstrap flag simultaneously disables every resource-bounds guard across the gateway stateless validator, the gateway stateful validator, and the mempool queue. When the flag is `false`, transactions with all-zero resource bounds pass every admission check, enter the mempool's **priority** queue (not the pending queue), and execute for free because `enforce_fee()` returns `false` when `max_possible_fee == 0`. An attacker who submits during the bootstrap window can flood the priority queue with cost-free transactions.

---

---

### Impact Explanation

During the bootstrap window an unprivileged attacker can:

1. Submit an unlimited number of transactions with `AllResourceBounds { l1_gas: 0/0, l2_gas: 0/0, l1_data_gas: 0/0 }`.
2. Each transaction passes all three gateway/mempool admission gates.
3. Each transaction lands in the **priority** queue and is sequenced ahead of legitimate fee-paying transactions.
4. Each transaction executes for free — no fee is charged, no balance is required.

This matches the allowed impact: **"High. Mempool/gateway/RPC admission accepts invalid transactions or rejects valid transactions before sequencing."**

---

### Likelihood Explanation

The bootstrap phase is a documented, recurring operational event (every new deployment or chain restart). The flag value is publicly visible in the node config schema. An attacker monitoring the network can detect the bootstrap window and exploit it at low cost (only L1 gas for the RPC call itself).

---

### Recommendation

Decouple the "skip gas-price threshold" bootstrap relaxation from the "allow zero resource bounds" guard. The `ZeroResourceBounds` check in the stateless validator and the priority-vs-pending routing in `FeeTransactionQueue` should remain active regardless of the bootstrap flag. Only the dynamic gas-price threshold comparison (which requires a live block header) needs to be disabled during bootstrap.

Concretely, split the single `validate_resource_bounds` flag into two:
- `validate_min_gas_price` — disableable during bootstrap.
- `validate_nonzero_resource_bounds` — always `true`; prevents zero-fee transactions from entering the priority queue.

---

### Proof of Concept

```
1. Operator sets validate_resource_bounds = false (bootstrap mode).

2. Attacker submits N invoke transactions, each with:
     resource_bounds = AllResourceBounds {
         l1_gas:      ResourceBounds { max_amount: 0, max_price_per_unit: 0 },
         l2_gas:      ResourceBounds { max_amount: 0, max_price_per_unit: 0 },
         l1_data_gas: ResourceBounds { max_amount: 0, max_price_per_unit: 0 },
     }
     tip = 0

3. StatelessTransactionValidator.validate_resource_bounds():
     → config.validate_resource_bounds == false → return Ok(())   [bypassed]

4. StatefulTransactionValidator.validate_resource_bounds():
     → config.validate_resource_bounds == false → skip threshold check [bypassed]

5. FeeTransactionQueue.insert():
     → to_pending_queue = false && ... = false
     → priority_queue.insert(tx)   [zero-price tx in priority queue]

6. Blockifier AccountTransaction.perform_pre_validation_stage():
     → enforce_fee() = max_possible_fee(Tip(0)) > Fee(0) = Fee(0) > Fee(0) = false
     → charge_fee block skipped entirely

7. AccountTransaction.handle_fee():
     → actual_fee == Fee(0) → return Ok(None)   [no fee transfer]

Result: N free transactions execute and are committed to the block,
        displacing legitimate fee-paying transactions from the priority queue.
```

    fn validate_resource_bounds(
        &self,
        tx: &RpcTransaction,
    ) -> StatelessTransactionValidatorResult<()> {
        if !self.config.validate_resource_bounds {
            return Ok(());
        }

        let resource_bounds = *tx.resource_bounds();
        // The resource bounds should be positive even without the tip.
        if ValidResourceBounds::AllResources(resource_bounds).max_possible_fee(Tip::ZERO) == Fee(0)
        {
            return Err(StatelessTransactionValidatorError::ZeroResourceBounds { resource_bounds });
        }

        if resource_bounds.l2_gas.max_price_per_unit.0 < self.config.min_gas_price {
            return Err(StatelessTransactionValidatorError::MaxGasPriceTooLow {
                gas_price: resource_bounds.l2_gas.max_price_per_unit,
                min_gas_price: self.config.min_gas_price,
            });
        }

        // TODO(Arni): Consider adding a validation for max_l2_gas_amount for declare.
        if let RpcTransaction::Declare(_) = tx {
        } else if resource_bounds.l2_gas.max_amount.0 > self.config.max_l2_gas_amount {
            return Err(StatelessTransactionValidatorError::MaxGasAmountTooHigh {
                gas_amount: resource_bounds.l2_gas.max_amount,
                max_gas_amount: self.config.max_l2_gas_amount,
            });
        }

        Ok(())
    }
