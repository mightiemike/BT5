### Title
SwapAllowlistExtension Gates the Router Address Instead of the Original User, Enabling Complete Allowlist Bypass — (`metric-periphery/contracts/extensions/SwapAllowlistExtension.sol`)

---

### Summary

`SwapAllowlistExtension.beforeSwap` checks the `sender` argument, which is `msg.sender` of the pool's `swap` call. When a user routes through `MetricOmmSimpleRouter`, `msg.sender` seen by the pool is the **router contract**, not the original user. If the pool admin adds the router to the allowlist (the only way to let allowlisted users use the standard periphery), every unpermissioned user can bypass the curated-pool gate by routing through the same public router.

---

### Finding Description

`MetricOmmPool.swap` passes `msg.sender` as the `sender` argument to `_beforeSwap`: [1](#0-0) 

`ExtensionCalling._beforeSwap` forwards that value unchanged to every configured extension: [2](#0-1) 

`SwapAllowlistExtension.beforeSwap` then checks `allowedSwapper[msg.sender][sender]`, where `msg.sender` is the pool and `sender` is whoever called `pool.swap`: [3](#0-2) 

When a user calls `MetricOmmSimpleRouter.exactInputSingle`, the router is the direct caller of `pool.swap`: [4](#0-3) 

So the extension receives `sender = router address`, not the original user's address. The router address is a single shared public contract. If the pool admin adds it to the allowlist (the only way to let their allowlisted users use the standard periphery), the mapping entry `allowedSwapper[pool][router] = true` grants every caller of the router unconditional access to the curated pool.

The same wrong-actor binding applies to multi-hop `exactInput`: for every hop the router calls `pool.swap` directly, so every hop sees `sender = router`: [5](#0-4) 

There is no mechanism inside the extension to recover the original user's address from `extensionData`, and `extensionData` is user-controlled so it cannot be trusted for authentication.

---

### Impact Explanation

A curated pool deploying `SwapAllowlistExtension` to restrict trading to a known set of addresses is fully bypassed. Any unpermissioned user can trade against the pool's oracle-anchored bins by routing through `MetricOmmSimpleRouter`. This breaks the core pool invariant that only allowlisted actors may execute swaps, and directly exposes LP funds to trades from actors the pool admin explicitly excluded. The loss surface is the full pool depth available to any swap direction.

---

### Likelihood Explanation

The bypass requires the router to be on the allowlist. Pool admins who want their allowlisted users to use the standard periphery (the only supported swap UI) must add the router. This is the natural, expected configuration. The admin has no alternative: if the router is not allowlisted, even their own approved users cannot use the periphery. The trap is therefore encountered in every realistic curated-pool deployment that intends to support router-based trading.

---

### Recommendation

The extension must gate the **original economic actor**, not the immediate caller. Two sound approaches:

1. **Pass the original user through `extensionData` and verify it on-chain**: The router encodes `msg.sender` into `extensionData` before calling the pool; the extension decodes and checks that address. This requires a trusted encoding convention enforced by the router.

2. **Check `sender` only when `sender` is not a known periphery contract, and require the periphery to forward the real user**: Add a `trustedRouter` registry to the extension; when `sender == trustedRouter`, read the real user from a router-side transient slot (e.g., the payer stored by `_setNextCallbackContext`) and check that address against the allowlist instead.

Either way, the invariant must be: **the address checked against `allowedSwapper` is the address that controls the economic decision to swap**, not the address of the forwarding contract.

---

### Proof of Concept

```
Setup:
  pool  = MetricOmmPool with SwapAllowlistExtension
  admin = pool admin
  alice = allowlisted user
  eve   = NOT on the allowlist
  router = MetricOmmSimpleRouter (public, shared)

Step 1 – admin allows alice and the router so alice can use the periphery:
  extension.setAllowedToSwap(pool, alice,  true)   // alice can swap
  extension.setAllowedToSwap(pool, router, true)   // router must be allowed for alice to use it

Step 2 – eve calls the router directly (no special permission):
  router.exactInputSingle(ExactInputSingleParams({
    pool:          pool,
    recipient:     eve,
    zeroForOne:    true,
    amountIn:      X,
    ...
    extensionData: ""
  }))

Execution trace:
  router.exactInputSingle  →  pool.swap(recipient=eve, ...)
    msg.sender inside pool.swap = router
    _beforeSwap(sender=router, ...)
    SwapAllowlistExtension.beforeSwap(sender=router, ...)
      allowedSwapper[pool][router] == true  →  PASSES

Result:
  Eve's swap executes. The allowlist guard is completely bypassed.
  Eve receives token output; the pool's LP funds are exposed to an
  actor the admin explicitly did not permit.
```
