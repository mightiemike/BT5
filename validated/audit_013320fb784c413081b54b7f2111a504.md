The vulnerability is real. Here is the complete analysis:

---

### Title
Missing `onlyPool` Modifier in `PriceVelocityGuardExtension.beforeSwap` Allows Arbitrary State Pre-Seeding via Direct External Call — (`metric-periphery/contracts/extensions/PriceVelocityGuardExtension.sol`)

---

### Summary

`PriceVelocityGuardExtension.beforeSwap` overrides the base class implementation but **silently drops the `onlyPool` modifier**, allowing any external caller to write attacker-controlled values into `priceVelocityState[msg.sender]`. Combined with CREATE2 address prediction, an attacker can pre-seed the velocity state for a not-yet-deployed pool address, corrupting the guard's baseline for that pool's first swap.

---

### Finding Description

`BaseMetricExtension` defines `beforeSwap` with an explicit `onlyPool` guard: [1](#0-0) 

The `onlyPool` modifier enforces that only factory-registered pools may call extension hooks: [2](#0-1) 

`PriceVelocityGuardExtension.beforeSwap` overrides this function but **omits `onlyPool` entirely**: [3](#0-2) 

Inside the function, `msg.sender` is used directly as the pool key, and state is unconditionally written: [4](#0-3) 

Because there is no caller check, any EOA or contract can call `beforeSwap` and write arbitrary `lastMidPriceX64` / `lastUpdateBlock` into `priceVelocityState[msg.sender]`.

---

### Impact Explanation

**Pre-seeding attack via CREATE2:**

1. Attacker computes the deterministic address of a future pool (CREATE2 salt is a user-supplied `params.salt` passed to `MetricOmmPoolDeployer.deploy`). [5](#0-4) 

2. Before the pool is deployed, the attacker calls `PriceVelocityGuardExtension.beforeSwap` from an address equal to the future pool address (achievable by deploying a thin proxy at that address first, or by using a contract whose address matches via CREATE2). Alternatively, the attacker simply calls it from any address they control and targets `priceVelocityState[their_address]` — but the meaningful attack targets the future pool address.

3. The attacker seeds `lastMidPriceX64` to a value far from the actual market price and `lastUpdateBlock` to the current block.

4. The pool is deployed at that address. On the first real swap, `prevMid` is non-zero (attacker's value), so the velocity check runs: [6](#0-5) 

5. If the attacker's seeded price diverges sufficiently from the actual first swap price, `actualSq > allowedSq` and the swap reverts with `PriceVelocityExceeded` — **DoS on the pool's first swap**.

Alternatively, the attacker seeds a price close to the actual first swap price, making the guard pass even when the true price has moved significantly, **weakening the guard's protection for the first swap**.

---

### Likelihood Explanation

- `beforeSwap` is a public external function with no access control — any caller can invoke it at zero cost.
- CREATE2 pool address prediction is straightforward: `params.salt` is caller-supplied and the deployer contract address is known.
- No privileged role is required. The entire attack is permissionless.

---

### Recommendation

Add the `onlyPool` modifier to `PriceVelocityGuardExtension.beforeSwap`, matching the base class contract:

```solidity
function beforeSwap(
    address,
    address,
    bool,
    int128,
    uint128,
    uint256,
    uint128 bidPriceX64,
    uint128 askPriceX64,
    bytes calldata
) external override onlyPool returns (bytes4) {
```

This ensures only factory-registered pools can update `priceVelocityState`, consistent with every other hook in `BaseMetricExtension`. [2](#0-1)

  function beforeSwap(
    address,
    address,
    bool,
    int128,
    uint128,
    uint256,
    uint128 bidPriceX64,
    uint128 askPriceX64,
    bytes calldata
  ) external override returns (bytes4) {
    address pool_ = msg.sender;
    (uint256 midPriceX64,) = SwapMath.midAndSpreadFeeX64FromBidAsk(uint256(bidPriceX64), uint256(askPriceX64));
    // casting to `uint128` is safe: geometric mid of two uint128 bid/ask quotes fits uint128 (same bound as pool)
    // forge-lint: disable-next-line(unsafe-typecast)
    uint128 midPrice = uint128(midPriceX64);

    PriceVelocityState storage s = priceVelocityState[pool_];
    uint128 prevMid = s.lastMidPriceX64;
    uint64 prevBlock = s.lastUpdateBlock;

    s.lastMidPriceX64 = midPrice;
    s.lastUpdateBlock = uint64(block.number);

    if (prevMid != 0) {
      uint64 maxChange = s.maxChangePerBlockE18;
      if (maxChange != 0) {
        uint256 blockDiff = block.number - prevBlock;

        uint256 delta = midPrice > prevMid ? uint256(midPrice - prevMid) : uint256(prevMid - midPrice);

        uint256 changeE18 = (delta * 1e18) / uint256(prevMid);

        uint256 actualSq = changeE18 * changeE18;
        uint256 allowedSq = uint256(maxChange) * uint256(maxChange) * (1 + blockDiff);

        if (actualSq > allowedSq) {
          revert PriceVelocityExceeded(actualSq, allowedSq);
        }
      }
    }

    return IMetricOmmExtensions.beforeSwap.selector;
  }
