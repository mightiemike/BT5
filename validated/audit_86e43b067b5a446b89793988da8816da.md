Audit Report

## Title
SwapAllowlistExtension Checks Router Address Instead of End User, Enabling Full Allowlist Bypass via MetricOmmSimpleRouter - (File: metric-periphery/contracts/extensions/SwapAllowlistExtension.sol)

## Summary
`SwapAllowlistExtension.beforeSwap` gates swaps by checking the `sender` argument — which is always `msg.sender` of the direct `pool.swap()` caller — against a per-pool allowlist. When `MetricOmmSimpleRouter` intermediates a swap, the pool receives the router's address as `sender`, not the end user's address. Any pool admin who allowlists the router (a necessary step for approved users to use router UX) simultaneously opens the pool to every user on the network, because the extension has no mechanism to distinguish which end user is behind the router call.

## Finding Description
**Pool passes `msg.sender` as `sender` to the extension hook:**

In `MetricOmmPool.sol` at L230–240, the pool calls `_beforeSwap(msg.sender, ...)`, so `sender` is always the direct caller of `pool.swap()`. [1](#0-0) 

**Extension checks only the direct caller (`sender`) against the allowlist:**

In `SwapAllowlistExtension.sol` at L37, `msg.sender` is the pool address and `sender` is whoever called `pool.swap()`. There is no inspection of `extensionData` or any other field to recover the end user's identity. [2](#0-1) 

**Router stores the end user only in transient callback context, never forwards it to the pool:**

In `MetricOmmSimpleRouter.sol` at L71–80, `_setNextCallbackContext` stores `msg.sender` (the end user) for payment purposes only. The subsequent `pool.swap()` call is made with the router as `msg.sender`, so the pool passes `router_address` as `sender` to the extension. [3](#0-2) 

**Exploit path:**
1. Pool admin deploys pool with `SwapAllowlistExtension` and allowlists `alice` and `router` (so alice can use the router).
2. `charlie` (not allowlisted) calls `router.exactInputSingle({pool: pool, ...})`.
3. Router calls `pool.swap(charlie, ...)` with `msg.sender = router`.
4. Pool calls `extension.beforeSwap(router, charlie, ...)`.
5. Extension checks `allowedSwapper[pool][router] == true` → no revert.
6. Charlie swaps successfully in a pool he was never approved for.

The exact wrong value is the extension's `beforeSwap` return value: it returns `IMetricOmmExtensions.beforeSwap.selector` (allow) instead of reverting with `NotAllowedToSwap()` for an unapproved user. [4](#0-3) 

## Impact Explanation
A pool configured with `SwapAllowlistExtension` to restrict swaps to specific counterparties (e.g., KYC'd institutions, whitelisted market makers) is fully open to any user who routes through `MetricOmmSimpleRouter`. LP providers deposited under the assumption that only approved counterparties would trade against them; the bypass exposes LP principal to adversarial flow (arbitrage, MEV, toxic order flow) that the allowlist was designed to exclude. This is a direct loss-of-LP-value impact and constitutes broken core pool functionality (the allowlist access control is entirely defeated for the router path).

## Likelihood Explanation
`MetricOmmSimpleRouter` is the primary public entry point for swaps. Any pool admin who wants approved users to enjoy router UX (slippage protection, deadline checks, multi-hop) must allowlist the router address — this is the expected production configuration. The bypass is reachable by any unprivileged user with no special setup, simply by calling the router with a pool address that has the router allowlisted.

## Recommendation
The extension must verify the end user's identity, not the intermediary's. The most robust fix is to have the router encode `msg.sender` (the end user) into `extensionData`, and have the extension decode and check it when `sender` is a known/trusted router. Since the pool admin controls which router is allowlisted, they can require a trusted router that always encodes the caller. Alternatively, introduce a two-level check: if `sender` is a known router, additionally verify the payer stored in the router's transient context via a shared interface.

## Proof of Concept
```
Setup:
  pool = MetricOmmPool with SwapAllowlistExtension
  pool admin: extension.setAllowedToSwap(pool, alice, true)
  pool admin: extension.setAllowedToSwap(pool, router, true)
    ↑ necessary so alice can use the router

Attack:
  charlie (not allowlisted) calls:
    router.exactInputSingle({pool: pool, recipient: charlie, ...})

  router executes (MetricOmmSimpleRouter.sol L71-80):
    _setNextCallbackContext(pool, CALLBACK_MODE_JUST_PAY, charlie, tokenIn)
    pool.swap(charlie, zeroForOne, amount, limit, "", extensionData)
    // msg.sender in pool = router_address

  pool executes (MetricOmmPool.sol L230-240):
    _beforeSwap(router_address, charlie, ...)

  extension checks (SwapAllowlistExtension.sol L37):
    allowedSwapper[pool][router_address] == true  ✓  → no revert

  Result: charlie swaps successfully in a pool he was never approved for.
```
