### Title
NFT Denom ID and Name Can Be Stolen by Front-Running `MsgIssueDenom` — (`x/nft/keeper/denom.go`, `x/nft/keeper/msg_server.go`)

---

### Summary

The `x/nft` module registers NFT denom IDs and names on a strict first-come-first-served basis with no ownership proof. Any unprivileged account can observe a pending `MsgIssueDenom` transaction in the mempool and front-run it by submitting an identical `id`/`name` with a higher fee, becoming the registered `creator` of that denom. The legitimate creator's transaction then fails permanently, and the attacker gains exclusive minting rights over the stolen denom identity.

---

### Finding Description

`MsgIssueDenom` is an open, permissionless transaction. Its handler in `msg_server.go` calls `Keeper.IssueDenom`, which delegates directly to `SetDenom`: [1](#0-0) 

`SetDenom` checks only that the `id` and `name` are not already taken — it performs no validation that the `sender` has any prior claim to the chosen identifier: [2](#0-1) 

The `Denom` struct stores the `creator` field as the address of whoever wins the race: [3](#0-2) 

Because Cosmos SDK mempools order transactions by fee priority, an attacker who monitors the mempool for a pending `MsgIssueDenom` can resubmit the same `id` and `name` with a higher fee. The attacker's transaction is included first, registering their address as `creator`. The victim's transaction then fails with `"denomID X has already exists"` or `"denomName X has already exists"`. [4](#0-3) 

---

### Impact Explanation

The `creator` field written into the `Denom` struct is the authorization anchor for all subsequent privileged operations on that denom. `MintNFT` and `BurnNFT` both call `IsDenomCreator`, which compares the caller against `denom.Creator`: [5](#0-4) [6](#0-5) 

By stealing the denom, the attacker:
1. **Gains exclusive minting rights** — only the attacker can call `MintNFT` under the stolen denom ID.
2. **Can create a honey pot** — the attacker mints NFTs under the denom ID that the legitimate creator publicly announced, deceiving buyers into purchasing fraudulent tokens.
3. **Permanently locks out the legitimate creator** — the victim must choose a different denom ID, breaking any off-chain references, marketplace listings, or application integrations that expected the original ID.

The corrupted on-chain value is `Denom.Creator` — it is set to the attacker's address instead of the legitimate creator's address, corrupting the `IsDenomCreator` authorization invariant for the lifetime of that denom.

---

### Likelihood Explanation

The attack requires only:
- A funded account (any unprivileged address).
- Visibility into the mempool (standard for any Cosmos SDK node operator or public RPC user).
- Submitting a `MsgIssueDenom` with the same `id`/`name` and a marginally higher fee.

No special privileges, leaked keys, or social engineering are required. The attack is trivially automatable by monitoring `EventTypeIssueDenom` events or mempool broadcasts.

---

### Recommendation

Bind the denom ID to the creator's address so that no two creators can collide. Two concrete options:

1. **Derive the denom ID from the creator address**: require `id` to be prefixed with or derived from `sdk.AccAddress`, e.g., `<creator_bech32>/<user_chosen_suffix>`, so the namespace is partitioned per creator.
2. **Validate ownership of the ID**: if the `id` is intended to be a meaningful external identifier (e.g., an IBC class trace), require a signature or proof that the sender controls the corresponding external resource.

---

### Proof of Concept

1. Alice broadcasts `MsgIssueDenom{Id: "mycollection", Name: "My NFT Collection", Sender: alice}` with fee `1000uatom`.
2. Bob's bot detects the pending transaction in the mempool.
3. Bob broadcasts `MsgIssueDenom{Id: "mycollection", Name: "My NFT Collection", Sender: bob}` with fee `2000uatom`.
4. Bob's transaction is ordered first in the block. `SetDenom` stores `Denom{Id: "mycollection", Creator: bob}`. [7](#0-6) 

5. Alice's transaction executes next and fails: `"denomID mycollection has already exists"`.
6. Bob calls `MsgMintNFT{DenomId: "mycollection", Sender: bob, ...}`. `IsDenomCreator` passes because `denom.Creator == bob`. Bob mints NFTs under Alice's intended collection identity. [6](#0-5)

func (k Keeper) SetDenom(ctx sdk.Context, denom types.Denom) error {
	if k.HasDenomID(ctx, denom.Id) {
		return sdkerrors.Wrapf(types.ErrInvalidDenom, "denomID %s has already exists", denom.Id)
	}

	if k.HasDenomNm(ctx, denom.Name) {
		return sdkerrors.Wrapf(types.ErrInvalidDenom, "denomName %s has already exists", denom.Name)
	}

	store := ctx.KVStore(k.storeKey)
	bz := k.cdc.MustMarshal(&denom)
	store.Set(types.KeyDenomID(denom.Id), bz)
	store.Set(types.KeyDenomName(denom.Name), []byte(denom.Id))
	return nil

// IsDenomCreator checks if address is the creator of Denom
// Return the Denom if true, an error otherwise
func (k Keeper) IsDenomCreator(ctx sdk.Context, denomID string, address sdk.AccAddress) (types.Denom, error) {
	denom, err := k.GetDenom(ctx, denomID)
	if err != nil {
		return types.Denom{}, err
	}

	creator, err := sdk.AccAddressFromBech32(denom.Creator)
	if err != nil {
		panic(err)
	}

	if !creator.Equals(address) {
		return types.Denom{}, sdkerrors.Wrapf(types.ErrUnauthorized, "%s is not the creator of %s", address, denomID)
	}

	return denom, nil
}

func (m msgServer) IssueDenom(goCtx context.Context, msg *types.MsgIssueDenom) (*types.MsgIssueDenomResponse, error) {
	sender, err := sdk.AccAddressFromBech32(msg.Sender)
	if err != nil {
		return nil, err
	}

	ctx := sdk.UnwrapSDKContext(goCtx)
	if err := m.Keeper.IssueDenom(ctx, msg.Id, msg.Name, msg.Schema, msg.Uri, sender); err != nil {
		return nil, err
	}

	ctx.EventManager().EmitEvents(sdk.Events{
		sdk.NewEvent(
			types.EventTypeIssueDenom,
			sdk.NewAttribute(types.AttributeKeyDenomID, msg.Id),
			sdk.NewAttribute(types.AttributeKeyDenomName, msg.Name),
			sdk.NewAttribute(types.AttributeKeyCreator, msg.Sender),
		),
		sdk.NewEvent(
			sdk.EventTypeMessage,
			sdk.NewAttribute(sdk.AttributeKeyModule, types.AttributeValueCategory),
			sdk.NewAttribute(sdk.AttributeKeySender, msg.Sender),
		),
	})

	return &types.MsgIssueDenomResponse{}, nil
}
