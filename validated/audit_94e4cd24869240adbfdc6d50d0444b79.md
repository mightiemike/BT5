Audit Report

## Title
SwapAllowlistExtension Gates Router Address Instead of End-User, Enabling Allowlist Bypass — (`metric-periphery/contracts/extensions/SwapAllowlistExtension.sol`)

## Summary
`SwapAllowlistExtension.beforeSwap` checks the `sender` argument, which the pool sets to its own `msg.sender` — the router contract — when a swap is routed through `MetricOmmSimpleRouter`. If the pool admin allowlists the router to enable router-based swaps for permitted users, every caller of the public router bypasses the allowlist gate, including addresses the admin explicitly never permitted.

## Finding Description
`MetricOmmPool.swap` calls `_beforeSwap(msg.sender, ...)` at line 231, passing the pool's `msg.sender` as `sender`. When `MetricOmmSimpleRouter.exactInputSingle` (or `exactInput`, `exactOutputSingle`, `exactOutput`) calls `IMetricOmmPoolActions(params.pool).swap(...)`, the pool's `msg.sender` is the router contract — no end-user identity is forwarded.

`ExtensionCalling._beforeSwap` encodes this `sender` value and passes it to each configured extension via `_callExtensionsInOrder`. `SwapAllowlistExtension.beforeSwap` then evaluates:

```solidity
if (!allowAllSwappers[msg.sender] && !allowedSwapper[msg.sender][sender])
```

where `msg.sender` is the pool and `sender` is the router. The check resolves to `allowedSwapper[pool][router]`, not `allowedSwapper[pool][end_user]`. Once the router is added to `allowedSwapper` — the necessary step for any allowlisted user to trade through the router UI — the gate passes for every caller of the router, including addresses the admin never allowlisted.

Existing guards are insufficient: `allowAllSwappers` has the same problem (opening the pool to all router callers), and there is no mechanism in the pool or router to forward the true end-user identity to the extension.

## Impact Explanation
Any address excluded from the allowlist can execute swaps on a restricted pool by calling `MetricOmmSimpleRouter`. Pools that use `SwapAllowlistExtension` to restrict trading to specific market makers — the primary use case for oracle-anchored OMM pools — are fully bypassed. LP principal is at direct risk because the pool settles trades at the oracle price regardless of counterparty identity; the allowlist is the sole mechanism preventing harmful counterparties from trading against LP positions.

## Likelihood Explanation
`MetricOmmSimpleRouter` is a public, permissionless periphery contract. Any pool admin who wants allowlisted users to access the router UI must add the router to `allowedSwapper`, which simultaneously opens the pool to all users. The misconfiguration is not detectable at the setter call site because the admin's intent ("allow my users to use the router") and the actual effect ("allow everyone") are indistinguishable. No special privilege or front-running is required; any unprivileged address can exploit this by calling the router.

## Recommendation
The extension must resolve the true end-user identity rather than the intermediary's address. Two options:

1. **Forward `msg.sender` of the router as an explicit `sender` field.** The router should pass the original caller's address to the pool as a trusted `sender` parameter, and the pool should forward that value — not its own `msg.sender` — to extensions. This requires a pool-level interface change.

2. **Encode the end-user address in `extensionData`.** The router encodes `msg.sender` into `extensionData`; the extension decodes and verifies it. This requires the extension to trust that the router correctly reports the caller, which may require a router allowlist at the extension level.

## Proof of Concept
1. Pool admin deploys a pool with `SwapAllowlistExtension` and calls `setAllowedToSwap(pool, alice, true)`.
2. Admin calls `setAllowedToSwap(pool, MetricOmmSimpleRouter, true)` so that `alice` can use the router.
3. `bob` (not allowlisted) calls `MetricOmmSimpleRouter.exactInputSingle(...)` targeting the pool.
4. The router calls `IMetricOmmPoolActions(pool).swap(recipient, ...)` — pool's `msg.sender` = router.
5. `MetricOmmPool.swap` calls `_beforeSwap(msg.sender=router, ...)`.
6. `SwapAllowlistExtension.beforeSwap` evaluates `allowedSwapper[pool][router]` → `true`.
7. `bob`'s swap executes successfully, bypassing the allowlist entirely.
