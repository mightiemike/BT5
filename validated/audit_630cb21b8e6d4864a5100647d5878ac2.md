### Title
Double-Prefix Class Trace Corruption via Crafted ClassId in `processReceivedPacket` — (`x/nft-transfer/keeper/packet.go`)

---

### Summary

`processReceivedPacket` determines transfer direction using `IsAwayFromOrigin(packet.GetSourcePort(), packet.GetSourceChannel(), data.ClassId)`, which checks whether `data.ClassId` starts with the **source** port/channel prefix. When a counterparty chain sends a packet whose `ClassId` already begins with the **destination** chain's port/channel prefix (e.g., `nft/channel-1/realNFT`), the source-prefix check returns `true` (away from origin), and the code unconditionally prepends the destination prefix a second time, producing a double-prefixed path (`nft/channel-1/nft/channel-1/realNFT`). This corrupted trace is stored via `SetClassTrace` and the NFT is minted under the wrong `voucherClassID`, making it permanently unretractable.

---

### Finding Description

**Root cause — direction check uses source, not destination:**

`IsAwayFromOrigin` is defined as:

```go
func IsAwayFromOrigin(sourcePort, sourceChannel, fullClassPath string) bool {
    prefixClassID := GetClassPrefix(sourcePort, sourceChannel)
    return !strings.HasPrefix(fullClassPath, prefixClassID)
}
``` [1](#0-0) 

On the receiving side, `processReceivedPacket` calls it with the **source** port/channel:

```go
isAwayFromOrigin := types.IsAwayFromOrigin(packet.GetSourcePort(), packet.GetSourceChannel(), data.ClassId)
``` [2](#0-1) 

When `isAwayFromOrigin == true`, the code unconditionally prepends the destination prefix:

```go
classPrefix := types.GetClassPrefix(packet.GetDestPort(), packet.GetDestChannel())
prefixedClassID := classPrefix + data.ClassId
classTrace := types.ParseClassTrace(prefixedClassID)
if !k.HasClassTrace(ctx, classTrace.Hash()) {
    k.SetClassTrace(ctx, classTrace)
}
``` [3](#0-2) 

There is **no guard** that checks whether `data.ClassId` already starts with the destination prefix before prepending it. [4](#0-3) 

**Attack trace (concrete):**

| Variable | Value |
|---|---|
| Chain B dest port | `nft` |
| Chain B dest channel | `channel-1` |
| Chain A source port | `transfer` |
| Chain A source channel | `channel-0` |
| Attacker's `ClassId` in packet | `nft/channel-1/realNFT` |

Step-by-step on Chain B receiving the packet:

1. `IsAwayFromOrigin("transfer", "channel-0", "nft/channel-1/realNFT")` → `"nft/channel-1/realNFT"` does **not** start with `"transfer/channel-0/"` → returns `true`.
2. `classPrefix = "nft/channel-1/"` (from `GetClassPrefix(destPort, destChannel)`).
3. `prefixedClassID = "nft/channel-1/" + "nft/channel-1/realNFT"` = **`"nft/channel-1/nft/channel-1/realNFT"`**.
4. `ParseClassTrace("nft/channel-1/nft/channel-1/realNFT")` → `{Path: "nft/channel-1/nft/channel-1", BaseClassId: "realNFT"}`.
5. `SetClassTrace` stores this corrupted trace.
6. `voucherClassID = ibc/{sha256("nft/channel-1/nft/channel-1/realNFT")}` — a hash of a path that never existed in any legitimate IBC hop.
7. `MintNFT(ctx, voucherClassID, tokenID, ...)` mints the NFT under this wrong class ID.

`ValidateBasic()` in `OnRecvPacket` only checks that `ClassId` is non-blank — it imposes no structural constraint on the path: [5](#0-4) 

`SetClassTrace` stores the trace without calling `Validate()`, so the double-prefixed path is persisted without any identifier check. [6](#0-5) 

---

### Impact Explanation

- The stored `ClassTrace` path is `nft/channel-1/nft/channel-1` instead of the correct single-hop `nft/channel-1`. This path does not correspond to any real IBC channel sequence.
- The minted NFT on Chain B carries `voucherClassID = ibc/{hash("nft/channel-1/nft/channel-1/realNFT")}`. No legitimate chain will ever produce a packet that resolves this hash back to a real escrow, so the NFT is permanently stuck on Chain B.
- The NFT on Chain A is escrowed and cannot be recovered: a timeout/ack-error refund would call `refundPacketToken`, which re-derives `voucherClassID` from the original `data.ClassId` and attempts `MintNFT` (since `IsAwayFromOrigin` also returns `true` on the refund path), but the escrowed token on Chain A is never released via `TransferOwner`.
- Impact: **permanent loss of the NFT** and **class trace store corruption** on Chain B.

---

### Likelihood Explanation

The precondition is that the counterparty chain sends a packet with a `ClassId` that begins with the destination chain's port/channel prefix. This is achievable by:

- Any chain whose NFT module permits creating a denom with a slash-containing ID (e.g., `nft/channel-1/realNFT`) and then initiating an IBC transfer of that denom.
- A malicious or misconfigured IBC-connected chain — a standard IBC threat model assumption, since IBC is designed to be trustless between chains.

No privileged key, governance action, or social engineering is required. The packet data is IBC-verified (light client proof), so a relayer cannot forge it, but the **sending chain's module** can freely set any non-blank `ClassId`.

---

### Recommendation

In `processReceivedPacket`, before prepending the destination prefix, add a guard that rejects (or correctly routes) packets whose `ClassId` already starts with the destination prefix:

```go
destPrefix := types.GetClassPrefix(packet.GetDestPort(), packet.GetDestChannel())
if strings.HasPrefix(data.ClassId, destPrefix) {
    return sdkerrors.Wrapf(types.ErrInvalidClassID,
        "classId %s already contains destination prefix %s", data.ClassId, destPrefix)
}
```

Alternatively, align the direction check on the receiving side with the destination prefix (as ICS-721 specifies): an NFT is returning to origin if `ClassId` starts with `{destPort}/{destChannel}/`, not if it starts with `{sourcePort}/{sourceChannel}/`.

### Proof of Concept

```go
// Keeper test pseudocode
func TestDoublePrefix(t *testing.T) {
    // Chain B: destPort="nft", destChannel="channel-1"
    // Chain A: sourcePort="transfer", sourceChannel="channel-0"
    packet := channeltypes.Packet{
        SourcePort:      "transfer",
        SourceChannel:   "channel-0",
        DestinationPort: "nft",
        DestinationChannel: "channel-1",
    }
    data := types.NonFungibleTokenPacketData{
        ClassId:   "nft/channel-1/realNFT", // attacker-crafted
        TokenIds:  []string{"token1"},
        TokenUris: []string{"uri1"},
        Sender:    attackerAddr,
        Receiver:  receiverAddr,
    }

    err := keeper.processReceivedPacket(ctx, packet, data)
    require.NoError(t, err)

    // Retrieve stored trace — expect corruption
    traces := keeper.GetAllClassTraces(ctx)
    // ACTUAL:   traces[0].Path == "nft/channel-1/nft/channel-1"  ← double-prefixed BUG
    // EXPECTED: traces[0].Path == "nft/channel-1"
    assert.Equal(t, "nft/channel-1", traces[0].Path) // FAILS
}
```
