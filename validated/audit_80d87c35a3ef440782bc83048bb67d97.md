### Title
SwapAllowlistExtension Checks Router Address Instead of End User, Allowing Any User to Bypass Swap Allowlist via MetricOmmSimpleRouter — (`metric-periphery/contracts/extensions/SwapAllowlistExtension.sol`)

---

### Summary

`SwapAllowlistExtension.beforeSwap` gates swaps by checking `sender`, which is `msg.sender` of the `pool.swap()` call. When a user routes through `MetricOmmSimpleRouter`, `sender` is the router address, not the actual end user. If the pool admin allowlists the router (required for router-based swaps to work), every user on-chain can bypass the per-user swap restriction by routing through the router.

---

### Finding Description

`SwapAllowlistExtension.beforeSwap` receives `sender` as its first argument and checks it against the per-pool allowlist: [1](#0-0) 

`sender` is populated by `MetricOmmPool.swap` as `msg.sender` of the pool call: [2](#0-1) 

When a user calls `MetricOmmSimpleRouter.exactInput` or `exactOutput`, the router is `msg.sender` of `pool.swap`. The extension therefore sees `sender = router`, not the actual end user. If the pool admin has allowlisted the router address (the only way to support router-based swaps on a curated pool), the check `allowedSwapper[pool][router]` returns `true` for every user who routes through it.

Contrast this with `DepositAllowlistExtension`, which correctly checks `owner` — the address to whom the position is attributed — rather than `sender` (the operator/payer): [3](#0-2) 

The swap extension has the wrong actor binding: it gates the intermediary, not the economic principal.

---

### Impact Explanation

A curated pool (e.g., KYC-gated, institutional) deploying `SwapAllowlistExtension` to restrict swaps to specific addresses is fully bypassed. Any user can call `MetricOmmSimpleRouter.exactInput` / `exactOutput` targeting the pool and execute swaps that the allowlist was intended to block. The pool admin cannot simultaneously support router-based swaps for allowlisted users and exclude non-allowlisted users — allowlisting the router opens the gate to everyone.

This is a broken core pool functionality / admin-boundary break: the configured guard fails to enforce the intended policy on the standard supported periphery path.

---

### Likelihood Explanation

- `MetricOmmSimpleRouter` is the canonical periphery router; pool admins deploying curated pools would naturally want to support it.
- The bypass requires no special privilege: any user calls the public `exactInput` / `exactOutput` functions.
- The pool admin has no way to detect or prevent this without removing router support entirely.
- The `DepositAllowlistExtension` does not share this flaw, so the asymmetry is unlikely to be caught by integration tests that only test direct pool calls.

The integration test `test_allowedSwapSucceeds` allowlists `callers[0]` (a `TestCaller` contract that calls the pool directly), not a router intermediary, so the router bypass path is untested: [4](#0-3) 

---

### Recommendation

`SwapAllowlistExtension.beforeSwap` should gate on the actual economic actor. Two options:

1. **Check `recipient` instead of `sender`** — the recipient is the address that receives output tokens and is set by the end user, not the router. This is the closest analog to how `DepositAllowlistExtension` uses `owner`.

2. **Pass the original caller in `extensionData`** — the router encodes `msg.sender` into `extensionData`; the extension decodes and verifies it. This requires a coordinated router + extension design.

The simplest fix consistent with the deposit extension pattern is to change the gated actor from `sender` to `recipient`:

```solidity
// Current (wrong actor):
function beforeSwap(address sender, address, ...)
    if (!allowedSwapper[msg.sender][sender]) revert ...

// Fixed (gate on recipient, the economic beneficiary):
function beforeSwap(address, address recipient, ...)
    if (!allowedSwapper[msg.sender][recipient]) revert ...
```

### Proof of Concept

```
Setup:
  pool configured with SwapAllowlistExtension
  pool admin calls setAllowedToSwap(pool, router, true)   // to enable router swaps
  pool admin calls setAllowedToSwap(pool, alice, true)    // alice is the only allowed user
  bob is NOT allowlisted

Attack:
  bob calls MetricOmmSimpleRouter.exactInput({
      pools: [pool], tokens: [token0, token1], ...
  })
  → router calls pool.swap(recipient=bob, ...) with msg.sender = router
  → pool calls _beforeSwap(sender=router, ...)
  → SwapAllowlistExtension.beforeSwap checks allowedSwapper[pool][router] → true
  → swap executes for bob despite bob not being allowlisted

Result:
  bob receives token1 output; the curated pool's allowlist is fully bypassed.
``` [5](#0-4) [6](#0-5)
