### Title
NFT Denom Registration Can Be Frontrun, Permanently Stealing Creator Rights - (File: x/nft/keeper/denom.go)

### Summary

`MsgIssueDenom` registers a globally unique denom `id` and `name` without binding either to the sender's address. Any unprivileged actor can observe a pending `IssueDenom` transaction in the mempool, resubmit it with the same `id`/`name` at a higher gas price, and become the recorded `creator` of that denom. The honest user's transaction then fails permanently. Because only the `creator` can ever mint NFTs under a denom, the attacker gains exclusive minting rights over the stolen denom at the cost of a single transaction fee.

### Finding Description

`MsgIssueDenom` accepts a user-supplied `id` and `name` that must be globally unique. The message server passes them directly to `IssueDenom` → `SetDenom`: [1](#0-0) 

`SetDenom` enforces uniqueness by checking both keys before writing: [2](#0-1) 

Neither check involves the sender's address. The `creator` field is set to whoever submits the winning transaction: [3](#0-2) 

An attacker who sees a pending `MsgIssueDenom{id: "mytoken", name: "My Token", sender: Alice}` in the mempool can broadcast an identical message with `sender: Attacker` at a higher gas price. The attacker's transaction lands first; Alice's transaction then hits the `HasDenomID` guard and reverts with `"denomID mytoken has already exists"`. [4](#0-3) 

### Impact Explanation

The corrupted state value is the `creator` field of the `Denom` struct stored under `KeyDenomID(id)`. Because `MintNFT` enforces `IsDenomCreator` before allowing any NFT to be minted: [5](#0-4) 

the attacker permanently holds exclusive minting rights over the stolen denom. Alice can never mint NFTs under her intended denom ID or name. The attacker can mint arbitrary NFTs under the stolen denom and transfer or sell them, or simply hold the denom hostage. The impact is permanent loss of denom ownership for the honest user and unauthorized acquisition of creator rights by the attacker.

### Likelihood Explanation

The attack requires only a single `MsgIssueDenom` transaction with a higher gas price — no stake, no special role, no locked capital. Cosmos SDK mempools are publicly observable. The cost is one transaction fee. Any denom registration is vulnerable from the moment it is broadcast until it is included in a block. The attack is trivially repeatable against every new denom registration.

### Recommendation

Derive the denom identifier from the sender's address so that no two senders can collide on the same key. For example, compute `denomID = hash(sender_address || user_chosen_suffix)` and store it under that composite key. Alternatively, enforce that `msg.Id` must be prefixed with the sender's bech32 address, validated in `ValidateBasic`. Either approach makes the registration key unguessable/unclaimable by a third party.

### Proof of Concept

1. Alice broadcasts `MsgIssueDenom{id: "artcollection", name: "Art Collection", sender: alice_addr}`.
2. Attacker observes the mempool, broadcasts `MsgIssueDenom{id: "artcollection", name: "Art Collection", sender: attacker_addr}` with higher gas.
3. Attacker's tx is included first. `SetDenom` writes `creator = attacker_addr` under key `KeyDenomID("artcollection")`. [6](#0-5) 

4. Alice's tx executes `HasDenomID("artcollection")` → `true` → returns `ErrInvalidDenom`. [4](#0-3) 

5. Alice can never register `"artcollection"` or `"Art Collection"` again. Attacker calls `MsgMintNFT{denom_id: "artcollection", sender: attacker_addr, ...}` — `IsDenomCreator` passes, NFTs are minted under the stolen denom. [7](#0-6)

```go
func (m msgServer) IssueDenom(goCtx context.Context, msg *types.MsgIssueDenom) (*types.MsgIssueDenomResponse, error) {

	}

	if err := m.Keeper.IssueDenom(ctx, msg.Id, msg.Name, msg.Schema, msg.Uri, sender); err != nil {
```

```go
func (k Keeper) SetDenom(ctx sdk.Context, denom types.Denom) error {
	if k.HasDenomID(ctx, denom.Id) {
		return sdkerrors.Wrapf(types.ErrInvalidDenom, "denomID %s has already exists", denom.Id)
	}

	if k.HasDenomNm(ctx, denom.Name) {
		return sdkerrors.Wrapf(types.ErrInvalidDenom, "denomName %s has already exists", denom.Name)
	}

	store := ctx.KVStore(k.storeKey)
	bz := k.cdc.MustMarshal(&denom)
	store.Set(types.KeyDenomID(denom.Id), bz)
	store.Set(types.KeyDenomName(denom.Name), []byte(denom.Id))
	return nil
```
