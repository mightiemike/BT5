### Title
Unchecked `transferFrom` Return Value in `replaceUsdcEWithUsdc` Enables USDCe Drain Without Paying USDC - (File: `core/contracts/ContractOwner.sol`)

---

### Summary

`ContractOwner.replaceUsdcEWithUsdc` calls `IERC20Base(usdc).transferFrom(msg.sender, directDepositV1, balance)` without checking the return value. If the USDC token returns `false` instead of reverting on failure, the function proceeds as if the caller paid, withdraws USDCe from the DDA to `ContractOwner`, and sends it to the caller — resulting in free USDCe extraction from the protocol.

---

### Finding Description

The function `replaceUsdcEWithUsdc` in `ContractOwner.sol` is a chain-restricted (Ink, `chainid == 57073`) but otherwise **unpermissioned** swap helper. Its logic is:

1. Read `balance = IERC20Base(usdcE).balanceOf(directDepositV1)` — how much USDCe the DDA holds.
2. Call `IERC20Base(usdc).transferFrom(msg.sender, directDepositV1, balance)` — pull USDC from caller into DDA. **Return value is not checked.**
3. Call `DirectDepositV1(directDepositV1).withdraw(IIERC20Base(usdcE))` — pull all USDCe from DDA into `ContractOwner`.
4. Call `IERC20Base(usdcE).safeTransfer(msg.sender, balance)` — send USDCe to caller.

Step 2 uses a raw `transferFrom` call, not `ERC20Helper.safeTransferFrom`. The `IERC20Base` interface declares `transferFrom` as returning `bool`, but the return value is silently discarded. [1](#0-0) 

The protocol already has `ERC20Helper.safeTransferFrom` which correctly validates the return value: [2](#0-1) 

And `IERC20Base.transferFrom` is explicitly declared as returning `bool`: [3](#0-2) 

The inconsistency is clear: every other token transfer in the codebase (e.g., `Clearinghouse.handleWithdrawTransfer`, `BaseWithdrawPool.safeTransferFrom`) uses `ERC20Helper.safeTransfer` / `safeTransferFrom`, but this one call does not. [4](#0-3) [5](#0-4) 

---

### Impact Explanation

If the USDC token at `0x2D270e6886d130D724215A266106e6832161EAEd` (Ink chain) returns `false` on a failed `transferFrom` rather than reverting — a behavior exhibited by non-standard ERC20 tokens analogous to DAI's non-standard `permit` — an attacker can:

- Call `replaceUsdcEWithUsdc(subaccount)` for any DDA that holds USDCe.
- The `transferFrom` fails silently (attacker pays nothing).
- The DDA's entire USDCe balance is withdrawn to `ContractOwner` and forwarded to the attacker.

**Corrupted asset delta**: The attacker receives `balance` USDCe for free. The DDA loses its entire USDCe balance. The protocol's collateral backing is reduced by `balance` USDCe with no USDC received in exchange.

---

### Likelihood Explanation

- The function has no access control beyond `block.chainid == 57073`; any address on Ink chain can call it.
- The exploitability depends on whether the specific USDC deployment at `0x2D270e6886d130D724215A266106e6832161EAEd` returns `false` on failure rather than reverting. Non-standard token behavior of this kind is the exact class described in the reference report.
- The attacker does not need any prior balance, approval, or privileged role — only the ability to call the function on the correct chain. [6](#0-5) 

---

### Recommendation

Replace the raw `transferFrom` call with `ERC20Helper.safeTransferFrom`, consistent with every other token transfer in the codebase:

```solidity
// Before (vulnerable):
IERC20Base(usdc).transferFrom(msg.sender, directDepositV1, balance);

// After (safe):
IERC20Base(usdc).safeTransferFrom(msg.sender, directDepositV1, balance);
``` [7](#0-6) 

---

### Proof of Concept

1. A DDA for `subaccount` holds `N` USDCe tokens.
2. Attacker calls `ContractOwner.replaceUsdcEWithUsdc(subaccount)` on Ink chain.
3. `balance = N` (USDCe balance of DDA).
4. `IERC20Base(usdc).transferFrom(attacker, directDepositV1, N)` — if USDC returns `false` (non-reverting failure), execution continues.
5. `DirectDepositV1(directDepositV1).withdraw(usdcE)` — `N` USDCe is transferred from DDA to `ContractOwner`.
6. `IERC20Base(usdcE).safeTransfer(attacker, N)` — attacker receives `N` USDCe.
7. Attacker paid 0 USDC; protocol lost `N` USDCe. [8](#0-7)

    function replaceUsdcEWithUsdc(bytes32 subaccount) external {
        require(block.chainid == 57073, ERR_UNAUTHORIZED);
        address payable directDepositV1 = directDepositV1Address[subaccount];
        require(directDepositV1 != address(0), "no dda");
        address usdcE = 0xF1815bd50389c46847f0Bda824eC8da914045D14;
        address usdc = 0x2D270e6886d130D724215A266106e6832161EAEd;
        uint256 balance = IERC20Base(usdcE).balanceOf(directDepositV1);
        if (balance > 0) {
            IERC20Base(usdc).transferFrom(msg.sender, directDepositV1, balance);
            DirectDepositV1(directDepositV1).withdraw(IIERC20Base(usdcE));
            IERC20Base(usdcE).safeTransfer(msg.sender, balance);
        }
    }
```

    function safeTransferFrom(
        IERC20Base self,
        address from,
        address to,
        uint256 amount
    ) internal {
        (bool success, bytes memory data) = address(self).call(
            abi.encodeWithSelector(
                IERC20Base.transferFrom.selector,
                from,
                to,
                amount
            )
        );

        require(
            success && (data.length == 0 || abi.decode(data, (bool))),
            ERR_TRANSFER_FAILED
        );
    }
```
