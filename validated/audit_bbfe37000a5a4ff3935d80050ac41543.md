### Title
`SwapAllowlistExtension` Checks Router Address Instead of Actual Swapper, Enabling Full Allowlist Bypass via `MetricOmmSimpleRouter` — (`metric-periphery/contracts/extensions/SwapAllowlistExtension.sol`)

---

### Summary

`SwapAllowlistExtension.beforeSwap` gates swaps by checking `allowedSwapper[pool][sender]`, where `sender` is the address passed by the pool — which is `msg.sender` of the pool's `swap` call. When a user routes through `MetricOmmSimpleRouter`, the pool's `msg.sender` is the router, not the actual user. The allowlist therefore checks whether the **router** is permitted, not the individual swapper. If the router is allowlisted (the only way legitimate users can use it on a curated pool), the gate is open to every user unconditionally.

---

### Finding Description

`SwapAllowlistExtension.beforeSwap` receives `sender` as its first argument and checks it against the per-pool allowlist:

```solidity
// SwapAllowlistExtension.sol L37-39
if (!allowAllSwappers[msg.sender] && !allowedSwapper[msg.sender][sender]) {
    revert IMetricOmmPoolActions.NotAllowedToSwap();
}
```

`msg.sender` here is the pool (the contract that called the extension). `sender` is whatever the pool forwarded as the first argument to `_beforeSwap`, which is the pool's own `msg.sender` — i.e., the direct caller of `pool.swap`.

`ExtensionCalling._beforeSwap` passes `sender` straight through without modification:

```solidity
// ExtensionCalling.sol L160-176
_callExtensionsInOrder(
    BEFORE_SWAP_ORDER,
    abi.encodeCall(
        IMetricOmmExtensions.beforeSwap,
        (sender, recipient, zeroForOne, amountSpecified, priceLimitX64,
         packedSlot0Initial, bidPriceX64, askPriceX64, extensionData)
    )
);
```

`MetricOmmSimpleRouter.exactInputSingle` calls `pool.swap` directly, making the router the pool's `msg.sender`:

```solidity
// MetricOmmSimpleRouter.sol L72-80
(int128 amount0Delta, int128 amount1Delta) = IMetricOmmPoolActions(params.pool)
    .swap(
        params.recipient,
        params.zeroForOne,
        MetricOmmSwapInputs.asAmountSpecifiedIn(params.amountIn),
        priceLimitX64,
        "",
        params.extensionData
    );
```

The router stores the actual user (`msg.sender`) only in transient storage for the payment callback — it is never forwarded to the pool or the extension as the swapper identity. The extension therefore evaluates `allowedSwapper[pool][router]`, not `allowedSwapper[pool][actualUser]`.

The same structural flaw applies to `exactInput` (multi-hop) and `exactOutputSingle`/`exactOutput`.

---

### Impact Explanation

A pool admin who deploys a curated pool with `SwapAllowlistExtension` faces an inescapable dilemma:

- **Router not allowlisted**: every individually allowlisted user is blocked from using the standard periphery path, making the pool practically unusable via the supported router.
- **Router allowlisted** (the only workable configuration): `allowedSwapper[pool][router] = true` opens the gate for **every** address that calls the router, regardless of whether they appear in the individual allowlist. The curation policy is entirely nullified.

Any non-allowlisted user can execute swaps on a supposedly restricted pool by calling `MetricOmmSimpleRouter.exactInputSingle` or `exactInput`. This constitutes a direct bypass of an access-control guard with fund-impacting consequences: unauthorized parties can trade against LP liquidity on pools that were explicitly configured to restrict participation.

---

### Likelihood Explanation

The router is the primary supported swap entrypoint in the periphery. Pool admins who want their users to be able to swap at all must allowlist the router, which simultaneously removes all individual-user gating. The bypass requires no special privileges, no flash loans, and no unusual token behavior — any EOA or contract can call the router. Likelihood is high.

---

### Recommendation

The pool must forward the original user's identity to extensions in a field that the router cannot spoof. Two sound approaches:

1. **Pass actual user in `extensionData`**: The router encodes `msg.sender` into the `extensionData` it forwards, and the extension decodes and verifies it. This requires a convention between router and extension.
2. **Separate `originator` field**: Add an `originator` parameter to the `beforeSwap` hook signature that the pool populates from a trusted transient-storage slot written by the router before calling `swap`. The extension checks `allowedSwapper[pool][originator]` instead of `allowedSwapper[pool][sender]`.

Either approach must be paired with a check that the `originator` value was written by a trusted router, not supplied arbitrarily by the caller.

---

### Proof of Concept

```
1. Pool admin deploys pool with SwapAllowlistExtension.
2. Pool admin calls setAllowedToSwap(pool, alice, true)   // alice is the only allowed swapper
3. Pool admin calls setAllowedToSwap(pool, router, true)  // required so alice can use the router
4. Bob (not allowlisted) calls MetricOmmSimpleRouter.exactInputSingle({pool: pool, ...})
5. Router calls pool.swap(recipient, ...) — pool's msg.sender = router
6. Pool calls _beforeSwap(sender=router, ...)
7. Extension evaluates: allowedSwapper[pool][router] == true  → no revert
8. Bob's swap executes successfully on the curated pool.
