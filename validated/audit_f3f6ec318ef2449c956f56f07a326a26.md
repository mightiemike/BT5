### Title
Unchecked `transferFrom` Return Value Enables Free Token Drain in `replaceUsdcEWithUsdc` — (`File: core/contracts/ContractOwner.sol`)

### Summary
`ContractOwner.replaceUsdcEWithUsdc` performs a token swap: the caller provides USDC to a `DirectDepositV1` address and receives `usdcE` in return. The USDC leg of this swap uses a raw `IERC20Base.transferFrom` call whose return value is never checked. If the call silently returns `false`, the caller receives `usdcE` without paying USDC, draining the `directDepositV1` contract's entire `usdcE` balance.

### Finding Description
`replaceUsdcEWithUsdc` is an `external` function with no caller access control (only a chain-ID guard). Its intended flow is:

1. Pull `balance` USDC from `msg.sender` into `directDepositV1`.
2. Withdraw `balance` usdcE from `directDepositV1` to `ContractOwner`.
3. Send `balance` usdcE to `msg.sender`.

Step 1 uses a bare `transferFrom`:

```solidity
// ContractOwner.sol line 616
IERC20Base(usdc).transferFrom(msg.sender, directDepositV1, balance);
``` [1](#0-0) 

The return value is discarded. Steps 2 and 3 execute unconditionally regardless of whether the USDC transfer succeeded. The rest of the codebase consistently wraps all token movements through `ERC20Helper.safeTransferFrom`, which checks the return value and reverts on failure:

```solidity
// ERC20Helper.sol lines 23-42
function safeTransferFrom(IERC20Base self, address from, address to, uint256 amount) internal {
    (bool success, bytes memory data) = address(self).call(...);
    require(success && (data.length == 0 || abi.decode(data, (bool))), ERR_TRANSFER_FAILED);
}
``` [2](#0-1) 

`BaseWithdrawPool` and `Clearinghouse` both use `token.safeTransferFrom(...)` via this helper for every token movement. [3](#0-2)  The single call in `replaceUsdcEWithUsdc` is the only place in the production codebase that bypasses this pattern. [4](#0-3) 

### Impact Explanation
If the USDC `transferFrom` returns `false` (e.g., insufficient allowance, non-standard ERC20 behavior, or a future token upgrade on the Ink chain), the function does not revert. It proceeds to:
- Withdraw the full `usdcE` balance from `directDepositV1` into `ContractOwner`.
- Transfer that `usdcE` to `msg.sender` via `safeTransfer`.

The attacker receives `usdcE` tokens at zero cost. The `directDepositV1` contract for the targeted `subaccount` is fully drained of its `usdcE` balance. The legitimate subaccount owner loses their deposited funds.

### Likelihood Explanation
The function is callable by any unprivileged external address on chain 57073 (Ink). The only precondition is that a `directDepositV1` address exists for some `subaccount` and holds a non-zero `usdcE` balance. The USDC token at `0x2D270e6886d130D724215A266106e6832161EAEd` is a hardcoded address on a relatively new chain; if it behaves as a non-reverting ERC20 on failure (as USDT does on mainnet), the attack is trivially executable with zero USDC approval.

### Recommendation
Replace the bare `transferFrom` with the project's own `ERC20Helper.safeTransferFrom`, consistent with every other token transfer in the codebase:

```solidity
// Before (ContractOwner.sol line 616)
IERC20Base(usdc).transferFrom(msg.sender, directDepositV1, balance);

// After
IERC20Base(usdc).safeTransferFrom(msg.sender, directDepositV1, balance);
```

`ContractOwner` already imports and uses `ERC20Helper` via `using ERC20Helper for IERC20Base` at line 24, so no additional imports are needed. [5](#0-4) 

### Proof of Concept
1. A `directDepositV1` for `subaccount` holds 1000 usdcE.
2. Attacker calls `replaceUsdcEWithUsdc(subaccount)` with zero USDC allowance.
3. `IERC20Base(usdc).transferFrom(attacker, directDepositV1, 1000)` returns `false` — no revert.
4. `DirectDepositV1(directDepositV1).withdraw(usdcE)` moves 1000 usdcE to `ContractOwner`.
5. `IERC20Base(usdcE).safeTransfer(attacker, 1000)` sends 1000 usdcE to the attacker.
6. Attacker receives 1000 usdcE; `directDepositV1` holds 0 usdcE and 0 USDC. [4](#0-3)

    function replaceUsdcEWithUsdc(bytes32 subaccount) external {
        require(block.chainid == 57073, ERR_UNAUTHORIZED);
        address payable directDepositV1 = directDepositV1Address[subaccount];
        require(directDepositV1 != address(0), "no dda");
        address usdcE = 0xF1815bd50389c46847f0Bda824eC8da914045D14;
        address usdc = 0x2D270e6886d130D724215A266106e6832161EAEd;
        uint256 balance = IERC20Base(usdcE).balanceOf(directDepositV1);
        if (balance > 0) {
            IERC20Base(usdc).transferFrom(msg.sender, directDepositV1, balance);
            DirectDepositV1(directDepositV1).withdraw(IIERC20Base(usdcE));
            IERC20Base(usdcE).safeTransfer(msg.sender, balance);
        }
    }
```

    function safeTransferFrom(
        IERC20Base self,
        address from,
        address to,
        uint256 amount
    ) internal {
        (bool success, bytes memory data) = address(self).call(
            abi.encodeWithSelector(
                IERC20Base.transferFrom.selector,
                from,
                to,
                amount
            )
        );

        require(
            success && (data.length == 0 || abi.decode(data, (bool))),
            ERR_TRANSFER_FAILED
        );
    }
