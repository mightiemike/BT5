### Title
Unprotected `initialize()` on Engine Contracts Allows Front-Running to Permanently Brick Protocol Setup — (File: `core/contracts/SpotEngine.sol`)

---

### Summary

`SpotEngine.initialize()` (and by the same pattern, `PerpEngine.initialize()`) is an `external` function with no access control. It is intended to be called exclusively by `Clearinghouse.addEngine()`. Because the engine proxy is deployed in a separate transaction from the `addEngine()` call, an attacker can front-run the initialization window, permanently preventing the engine from being correctly registered and forcing a full redeployment.

---

### Finding Description

`Clearinghouse.addEngine()` is the only legitimate caller of `productEngine.initialize()`. It is `onlyOwner`-gated, but the engine's `initialize()` itself has no such guard: [1](#0-0) 

```solidity
function initialize(
    address _clearinghouse,
    address _offchainExchange,
    address _quote,
    address _endpoint,
    address _admin
) external {
    _initialize(_clearinghouse, _offchainExchange, _endpoint, _admin);
```

`_initialize` in `BaseEngine` carries the `initializer` modifier, making it a one-time-only call: [2](#0-1) 

`Clearinghouse.addEngine()` calls `productEngine.initialize(...)` after registering the engine: [3](#0-2) 

Because the protocol uses ERC1967 upgradeable proxies, the proxy is deployed in one transaction and `addEngine()` is called in a subsequent transaction. Between those two transactions, `SpotEngine.initialize()` is callable by anyone with arbitrary parameters.

---

### Impact Explanation

An attacker who calls `SpotEngine.initialize(attacker, attacker, attacker, attacker, attacker)` before `addEngine()` does:

1. Sets `_clearinghouse`, `endpoint`, and `canApplyDeltas` to attacker-controlled addresses.
2. Sets `owner` to the attacker via `transferOwnership(_admin)`.
3. Consumes the `initializer` slot (`_initialized = 1`).

When the legitimate owner subsequently calls `Clearinghouse.addEngine(spotEngineAddr, ...)`, the inner call to `productEngine.initialize(...)` reverts because `_initialize` is already initialized. `addEngine()` reverts entirely, leaving `engineByType[SPOT]` unset. The entire protocol is non-functional until SpotEngine is redeployed — identical in consequence to the Velodrome Bribe redeployment requirement.

---

### Likelihood Explanation

The attack window exists on every deployment: proxy deployment and `addEngine()` are separate transactions. A mempool-watching bot can trivially detect the proxy deployment and front-run the `addEngine()` call. No special privilege is required — any EOA can execute this.

---

### Recommendation

Add an access control guard to `SpotEngine.initialize()` and `PerpEngine.initialize()` so only the deployer (or a factory contract) can call them, or use a factory pattern that deploys and initializes the engine atomically in a single transaction. Alternatively, call `initialize()` directly in the proxy deployment transaction (e.g., via the ERC1967 proxy constructor's `_data` parameter).

---

### Proof of Concept

1. Owner deploys `SpotEngine` proxy (transaction T1).
2. Attacker observes T1 in the mempool and submits `SpotEngine.initialize(attacker, attacker, attacker, attacker, attacker)` with higher gas (transaction T2).
3. T2 is mined before the owner's `Clearinghouse.addEngine(spotEngineAddr, ...)` call (T3).
4. T3 reverts: `_initialize` inside `SpotEngine.initialize()` hits the `initializer` guard and reverts.
5. `engineByType[SPOT]` remains `address(0)`. All downstream calls (`getHealth`, `depositCollateral`, `withdrawCollateral`, etc.) that rely on `_spotEngine()` revert.
6. Protocol requires full SpotEngine redeployment to recover. [4](#0-3) [2](#0-1)

    function _initialize(
        address _clearinghouseAddr,
        address _offchainExchangeAddr,
        address _endpointAddr,
        address _admin
    ) internal initializer {
        __Ownable_init();
        setEndpoint(_endpointAddr);
        transferOwnership(_admin);

        _clearinghouse = IClearinghouse(_clearinghouseAddr);

        canApplyDeltas[_endpointAddr] = true;
        canApplyDeltas[_clearinghouseAddr] = true;
        canApplyDeltas[_offchainExchangeAddr] = true;
    }

    function addEngine(
        address engine,
        address offchainExchange,
        IProductEngine.EngineType engineType
    ) external onlyOwner {
        require(address(engineByType[engineType]) == address(0));
        require(engine != address(0));
        IProductEngine productEngine = IProductEngine(engine);
        // Register
        supportedEngines.push(engineType);
        engineByType[engineType] = productEngine;

        // add quote to product mapping
        if (engineType == IProductEngine.EngineType.SPOT) {
            productToEngine[QUOTE_PRODUCT_ID] = productEngine;
        }

        // Initialize engine
        productEngine.initialize(
            address(this),
            offchainExchange,
            quote,
            getEndpoint(),
            owner()
        );
    }
