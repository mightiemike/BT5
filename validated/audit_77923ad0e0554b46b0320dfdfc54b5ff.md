### Title
Missing AuxPoW Flag Enforcement Allows AuxPoW Bypass via `AuxData=None` Submission — (`contract/src/dogecoin.rs`)

---

### Summary

`submit_block_header` in the Dogecoin build dispatches to `check_aux` only when `aux_data` is `Some`. There is no guard that rejects a header whose `version & 0x100 != 0` (AuxPoW flag set) when `aux_data` is `None`. A trusted relayer can therefore submit a post-371337 Dogecoin block with the AuxPoW version flag set but no `AuxData`, causing the contract to fall through to the standalone scrypt PoW branch and skip every AuxPoW invariant.

---

### Finding Description

`BlockHeader` for the Dogecoin build is defined as `(Header, Option<AuxData>)`: [1](#0-0) 

`submit_blocks` accepts `Vec<BlockHeader>` from a trusted relayer and passes each element directly to `submit_block_header`: [2](#0-1) 

Inside `submit_block_header`, the PoW branch is chosen purely on whether `aux_data` is `Some` or `None`: [3](#0-2) 

When `aux_data` is `None`, the code executes only a standalone scrypt PoW check against `bits`. It never calls `check_aux`, which is the only place the following invariants are enforced:

- `version & BLOCK_VERSION_AUXPOW != 0` (AuxPoW flag present in submitted block)
- `chain_id == block_header.get_chain_id()` (correct Dogecoin chain ID)
- `chain_id != aux_data.parent_block.get_chain_id()` (parent is not Dogecoin)
- Coinbase merkle proof linking the parent block to the Dogecoin block hash
- Chain merkle branch size and nonce-derived index check [4](#0-3) 

`check_aux` itself does verify `version & BLOCK_VERSION_AUXPOW != 0`, but that check is unreachable when `aux_data` is `None`: [5](#0-4) 

The missing guard is: **if `block_header.version & BLOCK_VERSION_AUXPOW != 0`, then `aux_data` must be `Some`**. No such require exists anywhere in `submit_block_header`.

---

### Impact Explanation

An attacker operating as a trusted relayer submits `(header_with_version | 0x100, None)`. The contract:

1. Passes `check_target` (difficulty check — attacker uses a valid `bits` value).
2. Skips `check_aux` entirely (falls into the `else` branch).
3. Passes the standalone scrypt PoW check (`block_hash_pow() <= target_from_bits(bits)`).
4. Stores the header as a valid canonical block via `submit_block_header_inner`.

The accepted header would be rejected by every real Dogecoin node because post-371337 blocks with the AuxPoW flag must carry a valid AuxPoW proof. The light client's canonical chain can therefore diverge from the real Dogecoin chain, enabling false `verify_transaction_inclusion` results for transactions in the fraudulently accepted block.

---

### Likelihood Explanation

The entry point is `submit_blocks`, which is gated by `#[trusted_relayer]`: [6](#0-5) 

The scope rules explicitly list "trusted-relayer submission path with attacker-relevant data" as a valid entry point. A trusted relayer controls the full `(Header, Option<AuxData>)` tuple it submits; omitting `AuxData` while setting the version flag is a deliberate, one-field change. No DAO, key leak, or social engineering is required beyond holding a trusted-relayer access key.

---

### Recommendation

Add an explicit guard at the top of the `!skip_pow_verification` block in `submit_block_header`:

```rust
const BLOCK_VERSION_AUXPOW: i32 = 0x100;
if block_header.version & BLOCK_VERSION_AUXPOW != 0 {
    require!(aux_data.is_some(), "AuxPoW flag set but no AuxData provided");
}
```

This ensures that any header advertising AuxPoW is unconditionally routed through `check_aux`, making the `else` branch unreachable for AuxPoW-flagged headers.

---

### Proof of Concept

```
1. Obtain a trusted-relayer access key for the deployed Dogecoin light-client contract.
2. Construct a Header with:
     version = (DOGE_CHAIN_ID << 16) | 0x100   // AuxPoW flag set
     bits    = <current network target>
     nonce   = <value such that scrypt(header_bytes) <= target_from_bits(bits)>
     (other fields: valid prev_block_hash, arbitrary merkle_root, valid time)
3. Submit via: submit_blocks(headers = [(header, None)])
4. Observe: the call succeeds (no panic), the block is stored in headers_pool,
   and check_aux was never invoked — chain_id, coinbase commitment, and
   chain merkle branch were never validated.
``` [7](#0-6)

    pub(crate) fn check_aux(&mut self, block_header: &Header, aux_data: &AuxData) {
        // The Dogecoin block must have the AuxPoW flag set (bit 8) when AuxPoW data is present.
        // https://github.com/dogecoin/dogecoin/blob/master/src/auxpow.h
        const BLOCK_VERSION_AUXPOW: i32 = 0x100;

        require!(
            aux_data.chain_merkle_proof.len() <= 30,
            "Aux POW chain merkle branch too long"
        );
        require!(
            block_header.version & BLOCK_VERSION_AUXPOW != 0,
            "Aux POW block does not have AuxPoW flag set in version"
        );

        let chain_id = self.get_config().aux_chain_id;

        require!(
            chain_id == block_header.get_chain_id(),
            format!(
                "block does not have our chain ID (got {}, expected {chain_id})",
                block_header.get_chain_id()
            )
        );

        require!(
            chain_id != aux_data.parent_block.get_chain_id(),
            "Aux POW parent has our chain ID"
        );

        let chain_root = merkle_tools::compute_root_from_merkle_proof(
            block_header.block_hash(),
            aux_data.chain_id,
            &aux_data.chain_merkle_proof,
        );

        let coinbase_tx = aux_data.get_coinbase_tx();
        let coinbase_tx_hash = coinbase_tx.compute_txid();

        require!(
            merkle_tools::compute_root_from_merkle_proof(
                H256::from(coinbase_tx_hash.to_raw_hash().to_byte_array()),
                0,
                &aux_data.merkle_proof,
            ) == aux_data.parent_block.merkle_root
        );

        let script_sig = coinbase_tx
            .input
            .first()
            .unwrap()
            .script_sig
            .to_hex_string();
        let pos_merged_mining_header = script_sig.find(MERGED_MINING_HEADER);
        let mut pos_chain_root = script_sig
            .find(&chain_root.to_string())
            .expect("Aux POW missing chain merkle root in parent coinbase");

        match pos_merged_mining_header {
            Some(pos_merged_mining_header) => {
                if script_sig[pos_merged_mining_header + MERGED_MINING_HEADER.len()..]
                    .contains(MERGED_MINING_HEADER)
                {
                    env::panic_str("Multiple merged mining headers in coinbase");
                }

                require!(
                    pos_merged_mining_header + MERGED_MINING_HEADER.len() == pos_chain_root,
                    "Merged mining header is not just before chain merkle root"
                );
            }
            None => {
                require!(pos_chain_root <= 40, "Aux POW chain merkle root must start in the first 20 bytes of the parent coinbase");
            }
        }

        pos_chain_root += chain_root.to_string().len();
        require!(
            script_sig.len() - pos_chain_root >= 16,
            "Aux POW missing chain merkle tree size and nonce in parent coinbase"
        );

        let bytes = hex::decode(&script_sig[pos_chain_root..pos_chain_root + 8]).unwrap();
        let n_size = u32::from_le_bytes([bytes[0], bytes[1], bytes[2], bytes[3]]);
        require!(
            n_size == (1u32 << aux_data.chain_merkle_proof.len()),
            "Aux POW merkle branch size does not match parent coinbase"
        );

        let bytes = hex::decode(&script_sig[pos_chain_root + 8..pos_chain_root + 16]).unwrap();
        let n_nonce = u32::from_le_bytes([bytes[0], bytes[1], bytes[2], bytes[3]]);

        let chain_id = block_header.get_chain_id();

        let expected_index =
            Self::get_expected_index(n_nonce, chain_id, aux_data.chain_merkle_proof.len());

        require!(
            u32::try_from(aux_data.chain_id).ok() == Some(expected_index),
            "Aux POW wrong index"
        );
        let pow_hash = aux_data.parent_block.block_hash_pow();
        require!(
            self.skip_pow_verification
                || U256::from_le_bytes(&pow_hash.0) <= target_from_bits(block_header.bits),
            format!("block should have correct pow")
        );
    }
