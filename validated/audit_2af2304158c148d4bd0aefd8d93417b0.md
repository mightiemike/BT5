### Title
Unchecked `transferFrom` Return Value Enables Free Drain of `usdcE` from DirectDepositV1 — (File: `core/contracts/ContractOwner.sol`)

---

### Summary

`ContractOwner.replaceUsdcEWithUsdc` calls `IERC20Base(usdc).transferFrom(...)` without checking its return value. If the transfer silently fails (returns `false`), execution continues and the caller receives `usdcE` tokens from the `DirectDepositV1` contract without having provided any USDC, draining the DDA's `usdcE` balance.

---

### Finding Description

`replaceUsdcEWithUsdc` is an `external` function with no access control beyond a chain-ID check. Its purpose is to let any caller swap `usdcE` held in a `DirectDepositV1` (DDA) contract for USDC: the caller provides USDC, the DDA's `usdcE` is withdrawn to `ContractOwner`, and then forwarded to the caller.

The critical flaw is at line 616:

```solidity
// ContractOwner.sol lines 608–620
function replaceUsdcEWithUsdc(bytes32 subaccount) external {
    require(block.chainid == 57073, ERR_UNAUTHORIZED);
    address payable directDepositV1 = directDepositV1Address[subaccount];
    require(directDepositV1 != address(0), "no dda");
    address usdcE = 0xF1815bd50389c46847f0Bda824eC8da914045D14;
    address usdc  = 0x2D270e6886d130D724215A266106e6832161EAEd;
    uint256 balance = IERC20Base(usdcE).balanceOf(directDepositV1);
    if (balance > 0) {
        IERC20Base(usdc).transferFrom(msg.sender, directDepositV1, balance); // ← return value ignored
        DirectDepositV1(directDepositV1).withdraw(IIERC20Base(usdcE));
        IERC20Base(usdcE).safeTransfer(msg.sender, balance);                 // ← safe, but too late
    }
}
``` [1](#0-0) 

The `IERC20Base.transferFrom` signature returns `bool`: [2](#0-1) 

The return value is never checked. If `transferFrom` returns `false` (e.g., caller has insufficient allowance or balance, or the token is non-standard), the EVM does not revert. Execution proceeds to `withdraw` and then `safeTransfer`, sending `usdcE` to the caller for free.

This is inconsistent with the rest of the protocol. All other token movements use `ERC20Helper.safeTransferFrom` / `ERC20Helper.safeTransfer`, which wrap the call in a low-level `.call()` and revert on both EVM failure and a `false` return: [3](#0-2) 

`EndpointStorage.chargeSlowModeFee` and `handleDepositTransfer` both use the safe wrapper: [4](#0-3) 

---

### Impact Explanation

An attacker who calls `replaceUsdcEWithUsdc` for any subaccount whose DDA holds `usdcE` — without holding or approving any USDC — will receive the full `usdcE` balance of that DDA for free if `transferFrom` returns `false` instead of reverting. The DDA loses its entire `usdcE` balance; the protocol receives nothing in return. This is a direct, concrete asset loss from a user-accessible entry point.

---

### Likelihood Explanation

The function is `external` with no `onlyOwner` or similar guard — any address on chain 57073 (Ink) can call it. The trigger requires either: (a) the caller has zero USDC allowance/balance and the token returns `false` rather than reverting, or (b) a future token upgrade or redeployment at the hardcoded address changes return behavior. Standard Circle USDC reverts on failure, but the code provides no safety net if that assumption ever breaks. The inconsistency with the rest of the codebase (which universally uses safe wrappers) makes this a clear oversight.

---

### Recommendation

Replace the raw `transferFrom` call with the protocol's own `ERC20Helper.safeTransferFrom`:

```solidity
// Before (unsafe):
IERC20Base(usdc).transferFrom(msg.sender, directDepositV1, balance);

// After (safe):
IERC20Base(usdc).safeTransferFrom(msg.sender, directDepositV1, balance);
```

`ContractOwner` already imports and uses `ERC20Helper` via `using ERC20Helper for IERC20Base`: [5](#0-4) 

---

### Proof of Concept

1. A DDA for `subaccount` holds 1000 `usdcE` on chain 57073.
2. Attacker calls `ContractOwner.replaceUsdcEWithUsdc(subaccount)` with zero USDC allowance.
3. `IERC20Base(usdc).transferFrom(attacker, dda, 1000)` returns `false` (no revert).
4. `DirectDepositV1(dda).withdraw(usdcE)` transfers 1000 `usdcE` to `ContractOwner`.
5. `IERC20Base(usdcE).safeTransfer(attacker, 1000)` sends 1000 `usdcE` to the attacker.
6. Attacker receives 1000 `usdcE` at zero cost; the DDA is drained. [6](#0-5)

    function safeTransferFrom(
        IERC20Base self,
        address from,
        address to,
        uint256 amount
    ) internal {
        (bool success, bytes memory data) = address(self).call(
            abi.encodeWithSelector(
                IERC20Base.transferFrom.selector,
                from,
                to,
                amount
            )
        );

        require(
            success && (data.length == 0 || abi.decode(data, (bool))),
            ERR_TRANSFER_FAILED
        );
    }
```
