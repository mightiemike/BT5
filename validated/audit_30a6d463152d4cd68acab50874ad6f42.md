### Title
Protobuf `ValidResourceBounds` Deserialization Selects Wrong Hash Variant for V3 Transactions with Zero L2/L1DataGas Bounds — (`File: crates/apollo_protobuf/src/converters/transaction.rs`)

### Summary

The protobuf `TryFrom<protobuf::ResourceBounds> for ValidResourceBounds` conversion uses a value-based heuristic — `l1_data_gas.is_zero() && l2_gas.is_zero()` — to select between the `L1Gas` (pre-0.13.3, 3-element hash) and `AllResources` (post-0.13.3, 4-element hash) variants. Because `get_tip_resource_bounds_hash` produces structurally different Poseidon digests for these two variants even when all numeric values are identical, any V3 transaction submitted via RPC with zero L2 and L1DataGas bounds will have its hash recomputed to a different value when the transaction is later deserialized over P2P state sync. This is the direct sequencer analog of M-07: the wrong formula variant is selected by a value-based condition, producing a systematic, attacker-triggerable hash divergence.

---

### Finding Description

**Step 1 — Two variants, two different hash preimages.**

`get_tip_resource_bounds_hash` in `crates/starknet_api/src/transaction_hash.rs` branches on the enum variant:

```
L1Gas      → poseidon([tip, l1_gas_packed, l2_gas_packed])           // 3 elements
AllResources → poseidon([tip, l1_gas_packed, l2_gas_packed, l1_data_gas_packed])  // 4 elements
```

Even when `l2_gas_packed == 0` and `l1_data_gas_packed == 0`, the two digests are numerically different because Poseidon over 3 elements ≠ Poseidon over 4 elements (the last element is `0`, not absent). [1](#0-0) 

**Step 2 — Gateway always assigns `AllResources`.**

`RpcInvokeTransactionV3` carries `resource_bounds: AllResourceBounds` (never `L1Gas`). The `From<RpcInvokeTransactionV3> for InvokeTransactionV3` conversion hard-codes `ValidResourceBounds::AllResources(tx.resource_bounds)`. The `InternalRpcInvokeTransactionV3::resource_bounds()` trait method also hard-codes `ValidResourceBounds::AllResources(self.resource_bounds)`. Therefore, every transaction entering through the gateway is hashed with the 4-element preimage. [2](#0-1) [3](#0-2) 

**Step 3 — Protobuf deserialization applies the wrong heuristic.**

`TryFrom<protobuf::ResourceBounds> for ValidResourceBounds` in the P2P state-sync path decides the variant purely by inspecting numeric values:

```rust
Ok(if l1_data_gas.is_zero() && l2_gas.is_zero() {
    ValidResourceBounds::L1Gas(l1_gas)          // ← 3-element hash path
} else {
    ValidResourceBounds::AllResources(...)       // ← 4-element hash path
})
``` [4](#0-3) 

A V3 transaction with `l2_gas = {0, 0}` and `l1_data_gas = {0, 0}` satisfies the condition and is silently downgraded to `L1Gas`. The `InvokeTransactionV3Trait` implementation for `InvokeTransactionV3` returns `self.resource_bounds` verbatim, so the recomputed hash uses the 3-element preimage. [5](#0-4) 

**Step 4 — The same wrong heuristic is duplicated in the RPC read path.**

`From<ResourceBoundsMapping> for ValidResourceBounds` in `crates/apollo_rpc/src/v0_8/transaction.rs` applies the identical condition, meaning the same hash divergence is reachable through the RPC read/simulation path. [6](#0-5) 

---

### Impact Explanation

**High. Transaction conversion or signature/hash logic binds the wrong hash.**

When a syncing node receives a block containing a V3 transaction with zero L2/L1DataGas bounds, it deserializes the transaction as `L1Gas` and recomputes a 3-element Poseidon hash. The block's recorded hash was computed with the 4-element `AllResources` preimage. `validate_transaction_hash` will find no match among the possible hashes and reject the transaction, stalling state sync for any block that contains such a transaction.

Additionally, if the wrong variant reaches execution, `get_gas_vector_computation_mode` returns `GasVectorComputationMode::NoL2Gas` instead of `GasVectorComputationMode::All`, silently suppressing L2-gas accounting and altering fee/bouncer behavior. [7](#0-6) [8](#0-7) 

---

### Likelihood Explanation

Any user can submit a valid V3 invoke transaction with `l2_gas.max_amount = 0`, `l2_gas.max_price_per_unit = 0`, `l1_data_gas.max_amount = 0`, `l1_data_gas.max_price_per_unit = 0` through the standard RPC endpoint. No privileged access is required. The transaction is accepted by the gateway (it passes all fee checks with zero bounds), included in a block, and then causes every P2P-syncing node to fail hash validation for that block.

---

### Recommendation

Remove the value-based heuristic. The protobuf wire format should carry an explicit discriminant (e.g., a boolean `is_all_resources` field, or a separate oneof) so the deserializer can reconstruct the correct variant without inspecting numeric values. Until the wire format is updated, the conversion should default to `AllResources` for all V3 transactions (version ≥ 0x3) and reserve `L1Gas` only for transactions whose version is strictly below 0.13.3, using a version field already present in the enclosing message rather than a value-based check on the bounds themselves.

---

### Proof of Concept

```
1. Craft a V3 InvokeTransaction:
     resource_bounds = {
       l1_gas:      { max_amount: N, max_price_per_unit: P },
       l2_gas:      { max_amount: 0, max_price_per_unit: 0 },   ← triggers heuristic
       l1_data_gas: { max_amount: 0, max_price_per_unit: 0 },   ← triggers heuristic
     }

2. Submit via starknet_addInvokeTransaction (RPC).
   Gateway path:
     RpcInvokeTransactionV3.resource_bounds = AllResourceBounds { ... }
     → ValidResourceBounds::AllResources(...)
     → get_tip_resource_bounds_hash: poseidon([tip, l1_packed, l2_packed=0, l1data_packed=0])
     → tx_hash = H1  (4-element digest)

3. Transaction is sequenced and included in block B.

4. A peer node syncs block B via P2P state sync.
   Protobuf deserialization path:
     TryFrom<protobuf::ResourceBounds>:
       l1_data_gas.is_zero() && l2_gas.is_zero()  → TRUE
       → ValidResourceBounds::L1Gas(l1_gas)
     → get_tip_resource_bounds_hash: poseidon([tip, l1_packed, l2_packed=0])
     → recomputed hash = H2  (3-element digest, H2 ≠ H1)

5. validate_transaction_hash(tx, block_number, chain_id, H1, options)
     possible_hashes = [H2]
     H1 ∉ [H2]  → returns false → sync node rejects block B.
```

pub fn get_tip_resource_bounds_hash(
    resource_bounds: &ValidResourceBounds,
    tip: &Tip,
) -> Result<Felt, StarknetApiError> {
    let l1_resource_bounds = resource_bounds.get_l1_bounds();
    let l2_resource_bounds = resource_bounds.get_l2_bounds();

    // L1 and L2 gas bounds always exist.
    // Old V3 txs always have L2 gas bounds of zero, but they exist.
    let mut resource_felts = vec![
        get_concat_resource(&l1_resource_bounds, L1_GAS)?,
        get_concat_resource(&l2_resource_bounds, L2_GAS)?,
    ];

    // For new V3 txs, need to also hash the data gas bounds.
    resource_felts.extend(match resource_bounds {
        ValidResourceBounds::L1Gas(_) => vec![],
        ValidResourceBounds::AllResources(all_resources) => {
            vec![get_concat_resource(&all_resources.l1_data_gas, L1_DATA_GAS)?]
        }
    });

    Ok(HashChain::new().chain(&tip.0.into()).chain_iter(resource_felts.iter()).get_poseidon_hash())
}

impl TryFrom<protobuf::ResourceBounds> for ValidResourceBounds {
    type Error = ProtobufConversionError;
    fn try_from(value: protobuf::ResourceBounds) -> Result<Self, Self::Error> {
        let Some(l1_gas) = value.l1_gas else {
            return Err(missing("ResourceBounds::l1_gas"));
        };
        let Some(l2_gas) = value.l2_gas else {
            return Err(missing("ResourceBounds::l2_gas"));
        };
        // TODO(Shahak): Assert data gas is not none once we remove support for 0.13.2.
        let l1_data_gas = value.l1_data_gas.unwrap_or_default();
        let l1_gas: ResourceBounds = l1_gas.try_into()?;
        let l2_gas: ResourceBounds = l2_gas.try_into()?;
        let l1_data_gas: ResourceBounds = l1_data_gas.try_into()?;
        Ok(if l1_data_gas.is_zero() && l2_gas.is_zero() {
            ValidResourceBounds::L1Gas(l1_gas)
        } else {
            ValidResourceBounds::AllResources(AllResourceBounds { l1_gas, l2_gas, l1_data_gas })
        })
    }
