### Title
Permanent Unburnable NFT After Transfer — `BurnNFT` Requires Simultaneous Owner AND Denom Creator — (`x/nft/keeper/keeper.go`)

---

### Summary

`Keeper.BurnNFT` enforces two independent guards in sequence: the caller must be both the current NFT owner **and** the denom creator. Once the denom creator transfers an NFT to any third party, neither the new owner nor the original creator can ever satisfy both conditions simultaneously. The NFT is permanently locked in state and the denom's supply counter never decreases.

---

### Finding Description

`Keeper.BurnNFT` applies two sequential authorization checks with no logical OR between them:

```go
// x/nft/keeper/keeper.go lines 141–161
func (k Keeper) BurnNFT(ctx sdk.Context, denomID, tokenID string, owner sdk.AccAddress) error {
    ...
    nft, err := k.IsOwner(ctx, denomID, tokenID, owner)   // must be current owner
    if err != nil {
        return err
    }

    _, err = k.IsDenomCreator(ctx, denomID, owner)         // must ALSO be denom creator
    if err != nil {
        return err
    }
    ...
}
``` [1](#0-0) 

`IsOwner` checks that `owner` equals `nft.Owner` in KV store: [2](#0-1) 

`IsDenomCreator` checks that `address` equals `denom.Creator`, which is set once at `IssueDenom` time and never changes: [3](#0-2) 

`TransferOwner` only checks `IsOwner` — no `IsDenomCreator` guard — so transfers succeed freely: [4](#0-3) 

After a transfer from A → B:

| Caller | `IsOwner` | `IsDenomCreator` | Result |
|--------|-----------|-----------------|--------|
| B (new owner) | ✅ | ❌ | `ErrUnauthorized` |
| A (denom creator) | ❌ | ✅ | `ErrUnauthorized` |

No address can satisfy both conditions. The only bypass, `BurnNFTUnverified`, is not reachable via `MsgBurnNFT` — it is called exclusively from the IBC transfer path: [5](#0-4) 

The public `msgServer.BurnNFT` routes directly to `Keeper.BurnNFT` with no alternative: [6](#0-5) 

---

### Impact Explanation

- The NFT is permanently frozen in state — no one can burn it.
- `decreaseSupply` is never called, so the denom's on-chain supply counter is permanently inflated by 1 for every such NFT.
- Any off-chain or on-chain logic that relies on `GetTotalSupply` for that denom (e.g., IBC class-trace accounting, marketplace integrations, governance snapshots) will observe a corrupted value indefinitely.
- The denom creator cannot reclaim or destroy the token even if they re-acquire ownership via a second transfer back, because the moment they hold it again they satisfy both checks — but that requires the third party's cooperation and is not a fix to the protocol logic.

---

### Likelihood Explanation

This is triggered by the standard, fully supported `MsgTransferNFT` → `MsgBurnNFT` sequence. No special privileges, governance, or operator access are required. Any user who receives an NFT from a denom creator via a normal transfer is immediately in this state. The path is reachable from any signed transaction on a live chain.

---

### Recommendation

Remove the `IsDenomCreator` check from `BurnNFT`. Burning should be an owner-only right. The denom creator's privilege is already correctly scoped to minting and editing. The corrected guard:

```go
func (k Keeper) BurnNFT(ctx sdk.Context, denomID, tokenID string, owner sdk.AccAddress) error {
    if !k.HasDenomID(ctx, denomID) {
        return sdkerrors.Wrapf(types.ErrInvalidDenom, "denom ID %s not exists", denomID)
    }
    nft, err := k.IsOwner(ctx, denomID, tokenID, owner)
    if err != nil {
        return err
    }
    k.deleteNFT(ctx, denomID, nft)
    k.deleteOwner(ctx, denomID, tokenID, owner)
    k.decreaseSupply(ctx, denomID)
    return nil
}
```

This mirrors the already-correct `BurnNFTUnverified` logic.

### Proof of Concept

Standard keeper test (no mocks, no special setup):

```go
func TestBurnNFTAfterTransfer(t *testing.T) {
    ctx, keeper := setupKeeper(t)

    addrA := sdk.AccAddress([]byte("creator"))
    addrB := sdk.AccAddress([]byte("thirdparty"))

    // 1. A issues denom
    err := keeper.IssueDenom(ctx, "denom1", "TestDenom", "", "", addrA)
    require.NoError(t, err)

    // 2. A mints NFT to A
    err = keeper.MintNFT(ctx, "denom1", "nft1", "NFT1", "", "", addrA, addrA)
    require.NoError(t, err)

    // 3. A transfers NFT to B
    err = keeper.TransferOwner(ctx, "denom1", "nft1", addrA, addrB)
    require.NoError(t, err)

    // 4. B tries to burn — fails: B is owner but not creator
    err = keeper.BurnNFT(ctx, "denom1", "nft1", addrB)
    require.ErrorContains(t, err, "ErrUnauthorized") // IsDenomCreator fails

    // 5. A tries to burn — fails: A is creator but no longer owner
    err = keeper.BurnNFT(ctx, "denom1", "nft1", addrA)
    require.ErrorContains(t, err, "ErrUnauthorized") // IsOwner fails

    // 6. Supply is still 1 — permanently corrupted
    supply := keeper.GetTotalSupply(ctx, "denom1")
    require.Equal(t, uint64(1), supply)
}
```

func (k Keeper) IsDenomCreator(ctx sdk.Context, denomID string, address sdk.AccAddress) (types.Denom, error) {
	denom, err := k.GetDenom(ctx, denomID)
	if err != nil {
		return types.Denom{}, err
	}

	creator, err := sdk.AccAddressFromBech32(denom.Creator)
	if err != nil {
		panic(err)
	}

	if !creator.Equals(address) {
		return types.Denom{}, sdkerrors.Wrapf(types.ErrUnauthorized, "%s is not the creator of %s", address, denomID)
	}

	return denom, nil
