### Title
No Token Recovery Mechanism for Unclaimed Airdrop Allocations — (`File: core/contracts/Airdrop.sol`)

---

### Summary
The `Airdrop` contract holds pre-funded tokens that users claim via weekly Merkle proofs. The contract contains no token recovery function of any kind. Any tokens deposited into the contract that are not claimed — whether due to user inaction, sanctioned addresses being blocked from claiming, or any other reason — are permanently locked with no path to retrieve them.

---

### Finding Description
The `Airdrop` contract is funded by direct token transfers from the protocol. The owner registers a Merkle root per week via `registerMerkleRoot()`, and users call `claim()` to withdraw their allocation. The contract enforces a sanctions check in `_verifyProof()`:

```solidity
require(
    !ISanctionsList(sanctions).isSanctioned(sender),
    "address is sanctioned."
);
```

If a user is sanctioned, their claim reverts and their allocated tokens remain in the contract. More broadly, any tokens deposited for a given week that go unclaimed — for any reason — are permanently stuck. The contract has no `recoverToken`, `sweep`, `withdrawUnclaimed`, or equivalent function. There is no expiry window, no admin escape hatch, and no mechanism to redirect unclaimed balances.

This is structurally identical to the SpiceAuction issue: a specific terminal state (no bids / no claims) leaves tokens locked because the only exit path (claiming) is unavailable, and no alternative recovery path exists.

---

### Impact Explanation
Tokens deposited into `Airdrop` that are not claimed are permanently irrecoverable. The magnitude scales with:
- The total weekly airdrop allocation deposited by the protocol
- The fraction of users who do not claim (historically, airdrop claim rates are well below 100%)
- Any addresses that become sanctioned after a Merkle root is registered

The result is a permanent, irreversible loss of protocol-owned tokens with no on-chain remedy.

---

### Likelihood Explanation
Airdrop non-participation is common in practice. Even a modest non-claim rate (e.g., 10–30%) across multiple weekly rounds results in a meaningful and growing pool of permanently locked tokens. The sanctions path is a concrete, deterministic trigger: any address sanctioned between Merkle root registration and claim execution loses its allocation permanently.

---

### Recommendation
Add an owner-callable recovery function that can sweep unclaimed tokens after a defined expiry window (e.g., after N weeks have passed since a Merkle root was registered). For example:

```solidity
function recoverUnclaimed(address to) external onlyOwner {
    // Only callable after a sufficient expiry period
    SafeERC20.safeTransfer(IERC20(token), to, IERC20(token).balanceOf(address(this)));
}
```

Alternatively, track total allocated vs. total claimed per week and allow recovery of the difference after expiry.

---

### Proof of Concept

1. Protocol owner calls `registerMerkleRoot(1, root)` and transfers 1,000,000 tokens to `Airdrop`.
2. 30% of eligible users never call `claim()` (300,000 tokens remain).
3. One eligible user's address is added to the OFAC sanctions list before they claim — their 10,000 tokens revert at the `isSanctioned` check.
4. After all claim windows have passed, 310,000 tokens sit in the contract.
5. The owner attempts to recover them — there is no function to call. The tokens are permanently locked.

// SPDX-License-Identifier: GPL-2.0-or-later
pragma solidity ^0.8.0;

import "./interfaces/IAirdrop.sol";
import "./Endpoint.sol";
import "@openzeppelin/contracts/utils/cryptography/MerkleProof.sol";
import "@openzeppelin/contracts-upgradeable/access/OwnableUpgradeable.sol";
import "@openzeppelin/contracts/token/ERC20/IERC20.sol";
import "@openzeppelin/contracts/token/ERC20/utils/SafeERC20.sol";

contract Airdrop is OwnableUpgradeable, IAirdrop {
    address internal token;
    address internal sanctions;
    uint32 internal pastWeeks;

    mapping(uint32 => bytes32) internal merkleRoots;
    mapping(uint32 => mapping(address => uint256)) internal claimed;

    /// @custom:oz-upgrades-unsafe-allow constructor
    constructor() {
        _disableInitializers();
    }

    function initialize(address _token, address _sanctions)
        external
        initializer
    {
        __Ownable_init();
        token = _token;
        sanctions = _sanctions;
    }

    function registerMerkleRoot(uint32 week, bytes32 merkleRoot)
        external
        onlyOwner
    {
        pastWeeks += 1;
        require(week == pastWeeks, "Invalid week provided.");
        merkleRoots[week] = merkleRoot;
    }

    function _verifyProof(
        uint32 week,
        address sender,
        uint256 totalAmount,
        bytes32[] calldata proof
    ) internal {
        require(claimed[week][sender] == 0, "Already claimed.");
        require(
            merkleRoots[week] != bytes32(0),
            "Week hasn't been registered."
        );
        require(
            !ISanctionsList(sanctions).isSanctioned(sender),
            "address is sanctioned."
        );
        bytes32 leaf = keccak256(
            bytes.concat(keccak256(abi.encode(sender, totalAmount)))
        );
        bool isValidLeaf = MerkleProof.verify(proof, merkleRoots[week], leaf);
        require(isValidLeaf, "Invalid proof.");
        claimed[week][sender] = totalAmount;
    }

    function _claim(
        uint32 week,
        uint256 totalAmount,
        bytes32[] calldata proof
    ) internal {
        _verifyProof(week, msg.sender, totalAmount, proof);
        SafeERC20.safeTransfer(IERC20(token), msg.sender, totalAmount);
        emit Claim(msg.sender, week, totalAmount);
    }

    function claim(ClaimProof[] calldata claimProofs) external {
        for (uint32 i = 0; i < claimProofs.length; i++) {
            _claim(
                claimProofs[i].week,
                claimProofs[i].totalAmount,
                claimProofs[i].proof
            );
        }
    }

    function getClaimed(address account)
        external
        view
        returns (uint256[] memory)
    {
        uint256[] memory result = new uint256[](pastWeeks + 1);
        for (uint32 week = 1; week <= pastWeeks; week++) {
            result[week] = claimed[week][account];
        }
        return result;
    }
}
