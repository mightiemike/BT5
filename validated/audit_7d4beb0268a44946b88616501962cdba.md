Audit Report

## Title
`SwapAllowlistExtension.beforeSwap` checks the router address instead of the end user, allowing any caller to bypass the per-user swap allowlist via `MetricOmmSimpleRouter` - (File: metric-periphery/contracts/extensions/SwapAllowlistExtension.sol)

## Summary
`SwapAllowlistExtension.beforeSwap` gates swaps by checking the `sender` argument, which the pool sets to `msg.sender` of `pool.swap()`. When a user routes through `MetricOmmSimpleRouter`, `sender` becomes the router address. If the pool admin allowlists the router (required to enable any router-based swap for permitted users), every unprivileged address can bypass the per-user allowlist by calling the router. The admin's intended access control — KYC gating, institutional-only pools, compliance restrictions — is silently nullified for all router-mediated swaps.

## Finding Description

`SwapAllowlistExtension.beforeSwap` checks `allowedSwapper[msg.sender][sender]`, where `msg.sender` is the pool and `sender` is the first argument passed by the pool:

```solidity
// metric-periphery/contracts/extensions/SwapAllowlistExtension.sol L37
if (!allowAllSwappers[msg.sender] && !allowedSwapper[msg.sender][sender]) {
    revert IMetricOmmPoolActions.NotAllowedToSwap();
}
```

The pool passes its own `msg.sender` as `sender` to `_beforeSwap`:

```solidity
// metric-core/contracts/MetricOmmPool.sol L230-240
_beforeSwap(msg.sender, recipient, zeroForOne, ...);
```

`ExtensionCalling._beforeSwap` forwards this value directly to the extension:

```solidity
// metric-core/contracts/ExtensionCalling.sol L162-165
abi.encodeCall(IMetricOmmExtensions.beforeSwap, (sender, recipient, ...))
```

When a user calls `MetricOmmSimpleRouter.exactInputSingle`, the router calls `pool.swap()` directly, making itself `msg.sender`:

```solidity
// metric-periphery/contracts/MetricOmmSimpleRouter.sol L71-80
_setNextCallbackContext(params.pool, CALLBACK_MODE_JUST_PAY, msg.sender, params.tokenIn);
(int128 amount0Delta, int128 amount1Delta) = IMetricOmmPoolActions(params.pool)
    .swap(params.recipient, params.zeroForOne, ...);
```

The original caller's address is stored in transient storage via `_setNextCallbackContext` only for payment purposes (`_getPayer()` in the callback). It is never forwarded to `extensionData` or any argument visible to the extension. The `beforeSwap` function ignores `extensionData` entirely (the last `bytes calldata` parameter is unnamed and unused).

The same applies to `exactInput` (L103-112), `exactOutputSingle` (L135-137), and `exactOutput` (L165-181) — all call `pool.swap()` with `msg.sender = router`.

**Exploit path:**
1. Pool is configured with `SwapAllowlistExtension`.
2. Admin allowlists Alice (direct user) and the router address (required to enable router-based swaps for Alice).
3. Mallory (not allowlisted) calls `router.exactInputSingle({pool: pool, ...})`.
4. Router calls `pool.swap()` → pool calls `_beforeSwap(router, ...)`.
5. Extension checks `allowedSwapper[pool][router] == true` → passes.
6. Mallory's swap executes despite not being individually permitted.

Direct call by Mallory is correctly blocked: `pool.swap()` → `_beforeSwap(Mallory, ...)` → `allowedSwapper[pool][Mallory] == false` → revert.

## Impact Explanation
This is a direct admin-boundary break. An unprivileged public path (`MetricOmmSimpleRouter`, a permissionless contract) causes the extension guard to authorize actors the admin explicitly did not permit. Any address can bypass a curated pool's swap allowlist by routing through the router. The pool admin's intended access control (KYC gating, institutional-only pools, compliance restrictions) is silently nullified for all router-mediated swaps. This falls squarely within the allowed impact gate: "Admin-boundary break: factory/oracle role checks are bypassed by an unprivileged path."

## Likelihood Explanation
The precondition — the admin allowlisting the router — is not a malicious or unusual setup; it is the only way to enable router-based swaps for any permitted user. Once the router is allowlisted (an expected operational step), the bypass is immediately available to every address with no further preconditions. The attacker needs only to call `router.exactInputSingle()` targeting the pool. The bypass is repeatable and requires no special privileges.

## Recommendation
`SwapAllowlistExtension.beforeSwap` must gate on the end user, not the direct caller of `pool.swap()`. The cleanest fix is for the router to encode the original `msg.sender` into `extensionData` before calling `pool.swap()`, and for the extension to decode and verify it. The extension should only trust this encoded address when `msg.sender` (the pool's direct caller, i.e., the router) is itself a registered trusted router in a factory-attested registry. Alternatively, the pool could expose the payer from transient storage to extensions, but this requires a core change. A simpler interim approach: require callers to pass the end-user address in `extensionData` and validate it against a trusted-router registry in the extension.

## Proof of Concept

```
Setup:
  pool configured with SwapAllowlistExtension
  admin calls setAllowedToSwap(pool, Alice, true)
  admin calls setAllowedToSwap(pool, router, true)  // required for Alice to use router
  Mallory is NOT allowlisted

Attack:
  Mallory calls router.exactInputSingle({pool: pool, recipient: Mallory, ...})
  → router calls pool.swap(Mallory, ...) with msg.sender = router
  → pool calls _beforeSwap(router, Mallory, ...)
  → SwapAllowlistExtension: allowedSwapper[pool][router] == true  ✓
  → swap executes for Mallory

Direct call (correctly blocked):
  Mallory calls pool.swap(...) directly
  → pool calls _beforeSwap(Mallory, ...)
  → SwapAllowlistExtension: allowedSwapper[pool][Mallory] == false  ✗
  → revert NotAllowedToSwap

Foundry test plan:
  1. Deploy pool with SwapAllowlistExtension.
  2. Allowlist Alice and the router; do NOT allowlist Mallory.
  3. Assert Mallory's direct pool.swap() reverts with NotAllowedToSwap.
  4. Assert Mallory's router.exactInputSingle() succeeds (demonstrating bypass).
```
