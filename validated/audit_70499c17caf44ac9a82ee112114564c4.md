### Title
`AllResources`-to-`L1Gas` silent downcast in protobuf round-trip drops `L1_DATA_GAS` from the transaction hash preimage, causing hash divergence across P2P boundaries - (File: crates/apollo_protobuf/src/converters/transaction.rs)

### Summary

When an `InvokeTransactionV3` is submitted with `AllResourceBounds { l1_gas: X, l2_gas: 0, l1_data_gas: 0 }`, the gateway computes its hash including `L1_DATA_GAS_concat(zero)` in the resource-bounds sub-hash. The P2P protobuf converter silently downcasts that `AllResources` representation to `ValidResourceBounds::L1Gas` on the receiving side (because both zero-valued fields satisfy the `l1_data_gas.is_zero() && l2_gas.is_zero()` guard). The receiving node then recomputes the hash without `L1_DATA_GAS_concat(zero)`, producing a divergent hash. Any node that receives the transaction via P2P will reject it, and a validator that receives a block proposal containing such a transaction will fail to reproduce the proposer's transaction commitment.

---

### Finding Description

**Step 1 – Hash preimage depends on the `ValidResourceBounds` variant.**

`get_tip_resource_bounds_hash` in `crates/starknet_api/src/transaction_hash.rs` builds a different-length resource-felts vector depending on the enum variant:

```
L1Gas(l1_bounds)          → [tip, L1_GAS_concat(l1_bounds), L2_GAS_concat(zero)]          // 2 resource felts
AllResources{l1,l2=0,d=0} → [tip, L1_GAS_concat(l1),       L2_GAS_concat(zero), L1_DATA_GAS_concat(zero)]  // 3 resource felts
``` [1](#0-0) 

The two variants therefore produce **different Poseidon hashes** even when l2_gas and l1_data_gas are both zero.

**Step 2 – The gateway always stores `AllResources`.**

`InternalRpcInvokeTransactionV3` carries `resource_bounds: AllResourceBounds` (never `L1Gas`). Its `InvokeTransactionV3Trait` implementation wraps it unconditionally as `ValidResourceBounds::AllResources(self.resource_bounds)`. [2](#0-1) 

The hash stored at admission time therefore always uses the 3-felt `AllResources` path.

**Step 3 – The protobuf round-trip silently downcasts to `L1Gas`.**

`TryFrom<protobuf::ResourceBounds> for ValidResourceBounds` in `crates/apollo_protobuf/src/converters/transaction.rs` applies:

```rust
Ok(if l1_data_gas.is_zero() && l2_gas.is_zero() {
    ValidResourceBounds::L1Gas(l1_gas)
} else {
    ValidResourceBounds::AllResources(AllResourceBounds { l1_gas, l2_gas, l1_data_gas })
})
``` [3](#0-2) 

An `AllResources { l1_gas: X, l2_gas: 0, l1_data_gas: 0 }` transaction serialised to protobuf and deserialised on the receiving node becomes `L1Gas(X)`. The receiving node then recomputes the hash using the 2-felt path, producing a value that differs from the hash the originating node stored and broadcast.

**Step 4 – The conditional `proof_facts` inclusion compounds the non-canonical domain.**

`get_invoke_transaction_v3_hash` appends `proof_facts_hash` only when `proof_facts` is non-empty, with no domain separator distinguishing the two cases:

```rust
if !transaction.proof_facts().0.is_empty() {
    hash_chain = hash_chain.chain(&proof_facts_hash);
}
``` [4](#0-3) 

This means the hash domain for "no proof_facts" and "with proof_facts" is not separated by a version marker, creating a second axis of non-canonical hash preimage construction that interacts with the resource-bounds downcast.

---

### Impact Explanation

A user can submit a valid `RpcInvokeTransactionV3` with `AllResourceBounds { l1_gas: X, l2_gas: 0, l1_data_gas: 0 }` to the gateway. The gateway accepts it and computes hash H₁ (3-felt path). When the transaction is propagated via P2P, every receiving node recomputes hash H₂ (2-felt path, H₂ ≠ H₁) and rejects the transaction as having an invalid hash. If the proposer node (which received the transaction directly) includes it in a block proposal, validators that received the transaction via P2P will fail to reproduce the transaction commitment, causing them to reject the block. This maps to:

- **High**: Mempool/gateway/RPC admission rejects valid transactions before sequencing (P2P propagation failure).
- **High**: Transaction conversion or signature/hash logic binds the wrong hash (divergent hash across the P2P boundary).

---

### Likelihood Explanation

The gateway's stateless validator explicitly accepts `AllResourceBounds` with only `l1_gas` set and all other fields zero: [5](#0-4) 

Any user who submits such a transaction triggers the divergence. No privileged role is required. The condition is reachable on every P2P hop.

---

### Recommendation

1. **Fix the protobuf round-trip**: preserve the `AllResources` variant even when l2_gas and l1_data_gas are zero. Remove the `l1_data_gas.is_zero() && l2_gas.is_zero()` downcast guard, or add an explicit discriminant field to the protobuf `ResourceBounds` message so the variant survives serialisation.

2. **Add a domain separator for `proof_facts`**: instead of conditionally appending `proof_facts_hash`, always include a felt that encodes whether proof_facts are present (e.g., chain a `0` or `1` flag before the hash), preventing the non-canonical hash domain.

3. **Add a round-trip hash-equality test**: a property test that serialises an `AllResources { l1_gas: X, l2_gas: 0, l1_data_gas: 0 }` transaction to protobuf and back, then asserts the recomputed hash equals the original, would have caught this.

---

### Proof of Concept

```
1. Craft RpcInvokeTransactionV3 with:
     resource_bounds = AllResourceBounds { l1_gas: {max_amount:1, max_price:1}, l2_gas: zero, l1_data_gas: zero }

2. Submit to gateway (node A).
   Node A computes:
     tip_resource_bounds_hash = Poseidon(tip, L1_GAS_concat(1,1), L2_GAS_concat(0,0), L1_DATA_GAS_concat(0,0))
     tx_hash H₁ = Poseidon(INVOKE, version, sender, tip_resource_bounds_hash, ...)

3. Node A propagates via P2P protobuf to node B.
   Protobuf ResourceBounds: l1_gas=Some({1,1}), l2_gas=Some({0,0}), l1_data_gas=Some({0,0})

4. Node B deserialises:
     l1_data_gas.is_zero() && l2_gas.is_zero() == true
     → ValidResourceBounds::L1Gas({1,1})

5. Node B recomputes:
     tip_resource_bounds_hash' = Poseidon(tip, L1_GAS_concat(1,1), L2_GAS_concat(0,0))   // missing L1_DATA_GAS
     tx_hash H₂ = Poseidon(INVOKE, version, sender, tip_resource_bounds_hash', ...)

6. H₁ ≠ H₂  →  node B rejects the transaction / block proposal.
```

The root cause is structurally identical to the CollateralizationOracle bug: a field (`L1_DATA_GAS_concat(zero)`) that belongs to the `AllResources` hash domain is silently "excluded" after the protobuf boundary, just as excluded-deposit FEI was silently excluded from `protocolControlledFei` while still implicitly inflating `userCirculatingFei`. pub fn get_tip_resource_bounds_hash(
    resource_bounds: &ValidResourceBounds,
    tip: &Tip,
) -> Result<Felt, StarknetApiError> {
    let l1_resource_bounds = resource_bounds.get_l1_bounds();
    let l2_resource_bounds = resource_bounds.get_l2_bounds();

    // L1 and L2 gas bounds always exist.
    // Old V3 txs always have L2 gas bounds of zero, but they exist.
    let mut resource_felts = vec![
        get_concat_resource(&l1_resource_bounds, L1_GAS)?,
        get_concat_resource(&l2_resource_bounds, L2_GAS)?,
    ];

    // For new V3 txs, need to also hash the data gas bounds.
    resource_felts.extend(match resource_bounds {
        ValidResourceBounds::L1Gas(_) => vec![],
        ValidResourceBounds::AllResources(all_resources) => {
            vec![get_concat_resource(&all_resources.l1_data_gas, L1_DATA_GAS)?]
        }
    });

    Ok(HashChain::new().chain(&tip.0.into()).chain_iter(resource_felts.iter()).get_poseidon_hash())
}

impl TryFrom<protobuf::ResourceBounds> for ValidResourceBounds {
    type Error = ProtobufConversionError;
    fn try_from(value: protobuf::ResourceBounds) -> Result<Self, Self::Error> {
        let Some(l1_gas) = value.l1_gas else {
            return Err(missing("ResourceBounds::l1_gas"));
        };
        let Some(l2_gas) = value.l2_gas else {
            return Err(missing("ResourceBounds::l2_gas"));
        };
        // TODO(Shahak): Assert data gas is not none once we remove support for 0.13.2.
        let l1_data_gas = value.l1_data_gas.unwrap_or_default();
        let l1_gas: ResourceBounds = l1_gas.try_into()?;
        let l2_gas: ResourceBounds = l2_gas.try_into()?;
        let l1_data_gas: ResourceBounds = l1_data_gas.try_into()?;
        Ok(if l1_data_gas.is_zero() && l2_gas.is_zero() {
            ValidResourceBounds::L1Gas(l1_gas)
        } else {
            ValidResourceBounds::AllResources(AllResourceBounds { l1_gas, l2_gas, l1_data_gas })
        })
    }
}
