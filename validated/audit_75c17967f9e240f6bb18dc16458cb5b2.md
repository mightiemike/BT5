### Title
Sanctioned Airdrop Recipients' Tokens Permanently Locked With No Recovery Path — (`core/contracts/Airdrop.sol`)

---

### Summary

The `Airdrop` contract enforces a sanctions check at claim time but provides no mechanism to recover tokens allocated to addresses that become sanctioned after a Merkle root is registered. Once a recipient is sanctioned, their allocated tokens are permanently locked in the contract with no path for recovery by the owner, treasury, or any other party.

---

### Finding Description

The `_verifyProof` function in `Airdrop.sol` hard-blocks any claim from a sanctioned address:

```solidity
require(
    !ISanctionsList(sanctions).isSanctioned(sender),
    "address is sanctioned."
);
``` [1](#0-0) 

The claim flow then marks the address as having claimed and transfers tokens:

```solidity
claimed[week][sender] = totalAmount;
``` [2](#0-1) 

Because `claimed[week][sender]` is only set inside `_verifyProof` — which reverts for sanctioned addresses — a sanctioned user's slot is never marked, and no other address can claim on their behalf (the Merkle leaf is bound to the specific `sender` address via `keccak256(abi.encode(sender, totalAmount))`). [3](#0-2) 

The entire `Airdrop` contract exposes no `recoverTokens`, `sweep`, or equivalent administrative function. The only functions are `initialize`, `registerMerkleRoot`, `claim`, and `getClaimed`. The `OwnableUpgradeable` base class adds no token recovery either. [4](#0-3) 

This is structurally identical to the Y2K M-4 bug: tokens are pre-allocated to a distribution mechanism, a protocol-enforced condition (sanctions check, analogous to `finalTVL == 0`) makes those tokens permanently unclaimable, and no recovery function exists.

---

### Impact Explanation

Any tokens allocated in a Merkle root to an address that is subsequently sanctioned are permanently locked in the `Airdrop` contract. Because the Merkle leaf encodes the recipient address, no other party can claim those tokens. The owner cannot redirect them. The tokens are irrecoverable.

The magnitude scales with the number of sanctioned recipients and the size of their allocations. For a protocol distributing governance or reward tokens weekly, even a single sanctioned high-allocation address results in a material permanent loss.

---

### Likelihood Explanation

Sanctions lists (e.g., Chainalysis oracle, OFAC-derived lists) are updated continuously and unpredictably. The window between Merkle root registration (`registerMerkleRoot`) and a user claiming can span days or weeks. It is realistic that one or more airdrop recipients are added to the sanctions list during this window, particularly for a protocol operating on a public chain with a broad user base. This is not a theoretical edge case — it is a known operational risk for any protocol integrating a live sanctions oracle. [5](#0-4) 

---

### Recommendation

Add an owner-callable token recovery function to `Airdrop.sol` that allows the protocol treasury to reclaim tokens allocated to sanctioned or unclaimed addresses after a defined expiry window. For example:

```solidity
function recoverTokens(address recipient, uint256 amount) external onlyOwner {
    SafeERC20.safeTransfer(IERC20(token), recipient, amount);
}
```

Additionally, consider whether the sanctions check should be applied at root registration time (off-chain filtering) rather than at claim time, to avoid the race condition between root registration and sanctions list updates.

---

### Proof of Concept

1. Owner calls `registerMerkleRoot(1, root)` where `root` encodes an allocation of 10,000 tokens for `address(0xAlice)`.
2. The `Airdrop` contract holds 10,000 tokens to cover Alice's allocation.
3. Before Alice calls `claim(...)`, her address is added to the `ISanctionsList` by the sanctions list administrator.
4. Alice calls `claim([ClaimProof({week: 1, totalAmount: 10000, proof: aliceProof})])`.
5. `_verifyProof` reverts at line 53–56: `"address is sanctioned."`.
6. No other address can construct a valid Merkle proof for Alice's leaf (the leaf is `keccak256(bytes.concat(keccak256(abi.encode(alice, 10000))))`).
7. `claimed[1][alice]` remains `0` forever; the 10,000 tokens remain in the contract with no callable path to move them. [6](#0-5)

```text
contract Airdrop is OwnableUpgradeable, IAirdrop {
    address internal token;
    address internal sanctions;
    uint32 internal pastWeeks;

    mapping(uint32 => bytes32) internal merkleRoots;
    mapping(uint32 => mapping(address => uint256)) internal claimed;

    /// @custom:oz-upgrades-unsafe-allow constructor
    constructor() {
        _disableInitializers();
    }

    function initialize(address _token, address _sanctions)
        external
        initializer
    {
        __Ownable_init();
        token = _token;
        sanctions = _sanctions;
    }

    function registerMerkleRoot(uint32 week, bytes32 merkleRoot)
        external
        onlyOwner
    {
        pastWeeks += 1;
        require(week == pastWeeks, "Invalid week provided.");
        merkleRoots[week] = merkleRoot;
    }

    function _verifyProof(
        uint32 week,
        address sender,
        uint256 totalAmount,
        bytes32[] calldata proof
    ) internal {
        require(claimed[week][sender] == 0, "Already claimed.");
        require(
            merkleRoots[week] != bytes32(0),
            "Week hasn't been registered."
        );
        require(
            !ISanctionsList(sanctions).isSanctioned(sender),
            "address is sanctioned."
        );
        bytes32 leaf = keccak256(
            bytes.concat(keccak256(abi.encode(sender, totalAmount)))
        );
        bool isValidLeaf = MerkleProof.verify(proof, merkleRoots[week], leaf);
        require(isValidLeaf, "Invalid proof.");
        claimed[week][sender] = totalAmount;
    }

    function _claim(
        uint32 week,
        uint256 totalAmount,
        bytes32[] calldata proof
    ) internal {
        _verifyProof(week, msg.sender, totalAmount, proof);
        SafeERC20.safeTransfer(IERC20(token), msg.sender, totalAmount);
        emit Claim(msg.sender, week, totalAmount);
    }

    function claim(ClaimProof[] calldata claimProofs) external {
        for (uint32 i = 0; i < claimProofs.length; i++) {
            _claim(
                claimProofs[i].week,
                claimProofs[i].totalAmount,
                claimProofs[i].proof
            );
        }
    }

    function getClaimed(address account)
        external
        view
        returns (uint256[] memory)
    {
        uint256[] memory result = new uint256[](pastWeeks + 1);
        for (uint32 week = 1; week <= pastWeeks; week++) {
            result[week] = claimed[week][account];
        }
        return result;
    }
```
