Audit Report

## Title
`SwapAllowlistExtension` evaluates the router's address instead of the original user's address, allowing any caller to bypass the per-pool swap allowlist via `MetricOmmSimpleRouter` — (`metric-periphery/contracts/extensions/SwapAllowlistExtension.sol`)

## Summary
`SwapAllowlistExtension.beforeSwap` gates swaps by checking `allowedSwapper[pool][sender]`, where `sender` is the immediate `msg.sender` of `pool.swap()`. When `MetricOmmSimpleRouter` intermediates a swap, it becomes `msg.sender` of `pool.swap()`, so the extension evaluates `allowedSwapper[pool][router]` rather than `allowedSwapper[pool][original_user]`. Any pool admin who allowlists the router to enable router-mediated swaps for permitted users simultaneously opens the gate to every unprivileged caller of the router.

## Finding Description

**Root cause — wrong actor bound in the hook:**

`SwapAllowlistExtension.beforeSwap` receives `sender` as its first argument and checks:

```solidity
if (!allowAllSwappers[msg.sender] && !allowedSwapper[msg.sender][sender]) {
    revert IMetricOmmPoolActions.NotAllowedToSwap();
}
``` [1](#0-0) 

Here `msg.sender` is the pool (the extension's caller) and `sender` is the argument forwarded by the pool. The pool always forwards its own `msg.sender` as `sender` to `_beforeSwap`:

```solidity
_beforeSwap(
    msg.sender,   // ← whoever called pool.swap()
    recipient,
    ...
);
``` [2](#0-1) 

**The router substitution:**

`MetricOmmSimpleRouter.exactInputSingle` calls `pool.swap()` directly without encoding the original user anywhere:

```solidity
(int128 amount0Delta, int128 amount1Delta) = IMetricOmmPoolActions(params.pool)
    .swap(
        params.recipient,
        params.zeroForOne,
        ...
        params.extensionData   // user-supplied; original msg.sender not injected
    );
``` [3](#0-2) 

When this call reaches the pool, `msg.sender` = router. The pool therefore passes `sender = router` to `_beforeSwap`, and the extension evaluates `allowedSwapper[pool][router]` — the original user's identity is never consulted. The `beforeSwap` function receives `extensionData` as its last `bytes calldata` parameter but does not decode an original sender from it. [4](#0-3) 

The same structural flaw exists in `exactInput` (pool.swap called with `msg.sender = router`), `exactOutputSingle`, and `exactOutput`. [5](#0-4) 

**Existing guards are insufficient:** The only guard is `allowedSwapper[pool][sender]`. There is no secondary check on `extensionData`, no trusted-router registry, and no `originalSender` field in the hook interface. Once the router is allowlisted, the check trivially passes for every caller.

## Impact Explanation

A curated pool deploying `SwapAllowlistExtension` to restrict trading to a known set of addresses loses that restriction entirely for any user who routes through `MetricOmmSimpleRouter`. An unauthorized user can execute swaps at oracle prices against LP-owned inventory, directly extracting LP principal from a pool that was designed to be closed to them. This is a direct loss of LP funds and a broken core pool invariant (the allowlist). The corrupted value is `allowedSwapper[pool][router]` being evaluated in place of `allowedSwapper[pool][original_user]`.

## Likelihood Explanation

The scenario requires the pool admin to allowlist the router — the natural, expected action for any curated pool that intends to support the standard periphery for its permitted users. The admin has no on-chain signal that doing so opens the gate to all users; `isAllowedToSwap(pool, router)` returns `true` and gives no indication that the original caller is not checked. [6](#0-5) 

The bypass is reachable by any unprivileged user with no special setup beyond calling the public router. It is repeatable on every swap.

## Recommendation

Pass the original user's address through the extension rather than the immediate pool caller. Two concrete options:

1. **Encode the original user in `extensionData`**: the router encodes `msg.sender` into `extensionData` before forwarding to the pool; the extension decodes and checks that address. This requires the extension to trust the router, which must be verified separately (e.g., via a factory-maintained router registry).

2. **Dedicated `originalSender` field in the hook interface**: add an `originalSender` field to the `beforeSwap` interface that the pool populates from a trusted periphery registry, so the extension always sees the economic actor regardless of routing depth.

Until fixed, document that `SwapAllowlistExtension` only enforces the allowlist for direct `pool.swap()` calls and must not be used with the public router on curated pools.

## Proof of Concept

```
Setup:
  pool admin deploys pool with SwapAllowlistExtension
  admin calls setAllowedToSwap(pool, alice, true)       // alice is permitted
  admin calls setAllowedToSwap(pool, router, true)      // router allowlisted so alice can use it
  LP adds liquidity

Attack (Eve, not allowlisted):
  1. Eve calls router.exactInputSingle({pool: pool, ...})
  2. Router calls pool.swap(recipient=eve, ...) — msg.sender = router
  3. Pool calls _beforeSwap(sender=router, ...)
  4. Extension evaluates: allowedSwapper[pool][router] → true  ✓
  5. Swap executes; Eve receives tokens from the curated pool

Expected: revert NotAllowedToSwap()
Actual:   swap succeeds — allowlist bypassed
```

Foundry test plan: deploy `SwapAllowlistExtension`, configure a pool with it, allowlist only `alice` and the router, then call `router.exactInputSingle` from an address that is not allowlisted and assert the swap succeeds (demonstrating the bypass) rather than reverting with `NotAllowedToSwap`.
