### Title
Protobuf `ValidResourceBounds` Deserialization Silently Downgrades `AllResources` to `L1Gas`, Producing a Different Transaction Hash - (`crates/apollo_protobuf/src/converters/transaction.rs`)

### Summary

The protobuf deserializer for `ValidResourceBounds` classifies any transaction whose `l2_gas` and `l1_data_gas` fields are both zero as `ValidResourceBounds::L1Gas`. However, `get_tip_resource_bounds_hash` produces a structurally different Poseidon preimage for `L1Gas` (3 elements) versus `AllResources` (4 elements). A V3 transaction submitted via RPC as `AllResources` with zero `l2_gas`/`l1_data_gas` bounds has its hash computed over 4 elements by the gateway, but after P2P propagation the receiving node recomputes the hash over 3 elements, causing a permanent hash mismatch and rejection of a valid transaction.

### Finding Description

**Hash domain boundary — `get_tip_resource_bounds_hash`**

`get_tip_resource_bounds_hash` in `crates/starknet_api/src/transaction_hash.rs` produces two structurally distinct Poseidon inputs depending on the `ValidResourceBounds` variant:

- `ValidResourceBounds::L1Gas` → hashes `[tip, l1_gas_packed, l2_gas_packed]` (3 elements)
- `ValidResourceBounds::AllResources` → hashes `[tip, l1_gas_packed, l2_gas_packed, l1_data_gas_packed]` (4 elements) [1](#0-0) 

**RPC ingestion path always uses `AllResources`**

`RpcInvokeTransactionV3` carries `resource_bounds: AllResourceBounds` (not `ValidResourceBounds`). The conversion to `InternalRpcInvokeTransactionV3` preserves this, and `calculate_transaction_hash` wraps it unconditionally in `ValidResourceBounds::AllResources(...)`, so the gateway always computes the 4-element hash for new V3 transactions. [2](#0-1) 

**P2P protobuf deserialization silently downgrades the variant**

`TryFrom<protobuf::ResourceBounds> for ValidResourceBounds` in `crates/apollo_protobuf/src/converters/transaction.rs` applies the rule: if `l1_data_gas.is_zero() && l2_gas.is_zero()` → emit `ValidResourceBounds::L1Gas`. The comment even acknowledges this is a temporary backward-compatibility measure for 0.13.2 nodes that omit `l1_data_gas`: [3](#0-2) 

When a new V3 `AllResources` transaction with zero `l2_gas` and `l1_data_gas` is serialized to protobuf and deserialized on a peer, the peer reconstructs it as `L1Gas`. The peer then recomputes the transaction hash over 3 elements, obtaining a value that differs from the 4-element hash the originating node stored and signed over.

**Serialization is lossy — both variants produce identical protobuf bytes**

The `From<ValidResourceBounds> for protobuf::ResourceBounds` serializer emits `l1_data_gas = ResourceBounds::default()` (all-zero) for `L1Gas`, and emits the actual (also all-zero) `l1_data_gas` for `AllResources` with zero bounds. The resulting protobuf bytes are byte-for-byte identical, so the round-trip is irreversibly lossy for this case. [4](#0-3) 

### Impact Explanation

A valid V3 transaction submitted via RPC with `AllResources { l2_gas: {0,0}, l1_data_gas: {0,0} }` is accepted by the gateway with hash `H_allresources = Poseidon(tip, l1, l2, l1_data)`. After P2P propagation, every receiving node recomputes `H_l1gas = Poseidon(tip, l1, l2)` (3 elements), which is a different field element. Hash validation fails and the transaction is permanently rejected by all peers. The transaction can never be sequenced despite being accepted by the originating gateway.

This matches: **High. Transaction conversion or signature/hash logic binds the wrong signer, hash, type, or executable payload.**

### Likelihood Explanation

The trigger condition — `AllResources` with both `l2_gas` and `l1_data_gas` fully zero — is reachable by any unprivileged user. `ValidResourceBounds::default()` produces exactly this state. Any SDK or tooling that constructs a V3 transaction using default resource bounds and submits it via RPC would trigger the mismatch. The TODO comment in the converter (`// TODO(Shahak): Assert data gas is not none once we remove support for 0.13.2`) confirms the team is aware the `unwrap_or_default()` path is a temporary measure, but the hash-domain consequence of the downgrade is not addressed.

### Recommendation

The protobuf deserializer must not use the zero-check heuristic to decide the `ValidResourceBounds` variant. Instead:

1. Add an explicit discriminant field to the protobuf `ResourceBounds` message (e.g., `bool is_all_resources`) so the variant survives the round-trip without relying on value inspection.
2. Alternatively, remove the `L1Gas` variant from the P2P wire format entirely for post-0.13.3 transactions and always deserialize as `AllResources`, matching the RPC ingestion path.
3. Until the wire format is updated, add a guard in `get_tip_resource_bounds_hash` or in the hash-computation callers that canonicalizes `AllResources` with zero `l2_gas`/`l1_data_gas` to `L1Gas` before hashing, so both paths produce the same preimage for the same logical transaction.

### Proof of Concept

```
1. Construct a V3 invoke transaction with:
     resource_bounds = AllResourceBounds {
         l1_gas:      { max_amount: 1000, max_price_per_unit: 1 },
         l2_gas:      { max_amount: 0,    max_price_per_unit: 0 },
         l1_data_gas: { max_amount: 0,    max_price_per_unit: 0 },
     }

2. Submit via starknet_addInvokeTransaction (RPC).
   Gateway path:
     resource_bounds variant = AllResources
     tip_resource_bounds_hash = Poseidon(tip, pack(l1_gas), pack(l2_gas), pack(l1_data_gas))
                              = Poseidon(tip, X, 0, 0)   [4 elements]
     tx_hash = H_allresources  ← stored in mempool

3. Transaction is broadcast over P2P as protobuf.
   Receiving node deserializes ResourceBounds:
     l1_data_gas = unwrap_or_default() → {0,0}
     l2_gas = {0,0}
     l1_data_gas.is_zero() && l2_gas.is_zero() == true
     → variant = ValidResourceBounds::L1Gas(l1_gas)

4. Receiving node recomputes hash:
     tip_resource_bounds_hash = Poseidon(tip, pack(l1_gas), pack(l2_gas))
                              = Poseidon(tip, X, 0)      [3 elements]
     tx_hash = H_l1gas  ← H_l1gas ≠ H_allresources

5. validate_transaction_hash(tx, expected=H_allresources) returns false.
   Transaction is rejected by all P2P peers.
```

The divergence is rooted in `get_tip_resource_bounds_hash` at lines 203–208 of `crates/starknet_api/src/transaction_hash.rs` and the lossy downgrade at lines 431–432 of `crates/apollo_protobuf/src/converters/transaction.rs`. [5](#0-4) [6](#0-5)

pub fn get_tip_resource_bounds_hash(
    resource_bounds: &ValidResourceBounds,
    tip: &Tip,
) -> Result<Felt, StarknetApiError> {
    let l1_resource_bounds = resource_bounds.get_l1_bounds();
    let l2_resource_bounds = resource_bounds.get_l2_bounds();

    // L1 and L2 gas bounds always exist.
    // Old V3 txs always have L2 gas bounds of zero, but they exist.
    let mut resource_felts = vec![
        get_concat_resource(&l1_resource_bounds, L1_GAS)?,
        get_concat_resource(&l2_resource_bounds, L2_GAS)?,
    ];

    // For new V3 txs, need to also hash the data gas bounds.
    resource_felts.extend(match resource_bounds {
        ValidResourceBounds::L1Gas(_) => vec![],
        ValidResourceBounds::AllResources(all_resources) => {
            vec![get_concat_resource(&all_resources.l1_data_gas, L1_DATA_GAS)?]
        }
    });

    Ok(HashChain::new().chain(&tip.0.into()).chain_iter(resource_felts.iter()).get_poseidon_hash())
}

impl TryFrom<protobuf::ResourceBounds> for ValidResourceBounds {
    type Error = ProtobufConversionError;
    fn try_from(value: protobuf::ResourceBounds) -> Result<Self, Self::Error> {
        let Some(l1_gas) = value.l1_gas else {
            return Err(missing("ResourceBounds::l1_gas"));
        };
        let Some(l2_gas) = value.l2_gas else {
            return Err(missing("ResourceBounds::l2_gas"));
        };
        // TODO(Shahak): Assert data gas is not none once we remove support for 0.13.2.
        let l1_data_gas = value.l1_data_gas.unwrap_or_default();
        let l1_gas: ResourceBounds = l1_gas.try_into()?;
        let l2_gas: ResourceBounds = l2_gas.try_into()?;
        let l1_data_gas: ResourceBounds = l1_data_gas.try_into()?;
        Ok(if l1_data_gas.is_zero() && l2_gas.is_zero() {
            ValidResourceBounds::L1Gas(l1_gas)
        } else {
            ValidResourceBounds::AllResources(AllResourceBounds { l1_gas, l2_gas, l1_data_gas })
        })
    }
