### Title
Unspent ETH Stranded in Router When `priceLimitX64` Causes Partial Exact-Input Swap - (`metric-periphery/contracts/MetricOmmSimpleRouter.sol`)

---

### Summary

`MetricOmmSimpleRouter.exactInputSingle` accepts native ETH (`payable`), forwards only the pool-consumed portion to the pool via the swap callback, but never refunds the remainder to the caller when a `priceLimitX64` causes the swap to terminate early. The unspent ETH is permanently stranded in the router and is extractable by any address.

---

### Finding Description

`exactInputSingle` stores `msg.sender` as the payer and calls `pool.swap` with the full `params.amountIn` as a positive (exact-input) `amountSpecified`: [1](#0-0) 

When a non-zero `priceLimitX64` is supplied, `MetricOmmPool._swapToken0ForToken1SpecifiedInput` (or its token1-for-token0 counterpart) exits the swap loop early once the marginal price reaches the limit, consuming only a fraction of `amountSpecified`: [2](#0-1) 

The pool then fires `metricOmmSwapCallback` with the **actual** (partial) deltas. The router's `_justPayCallback` pays only `extractPositiveAmount(amount0Delta, amount1Delta)` — the real consumed amount — not the full `params.amountIn`: [3](#0-2) 

`PeripheryPayments.pay` (inherited) follows the standard Uniswap periphery pattern: when `token == WETH` and `address(this).balance >= value`, it wraps ETH from the router's own balance and sends WETH to the pool. This means:

1. User sends 1 ETH with the call → router holds 1 ETH.
2. Price limit is hit; pool consumed only 0.7 ETH worth.
3. Callback wraps and forwards 0.7 ETH to the pool.
4. **0.3 ETH remains in the router.**

After the swap, `exactInputSingle` performs only a slippage check on `amountOut` and clears the callback context — **no ETH refund is issued**: [4](#0-3) 

The stranded ETH is then recoverable by any caller via the public `refundETH` function inherited from `PeripheryPayments`.

---

### Impact Explanation

Direct loss of user principal. A user who sells native ETH with a price limit set loses the unconsumed portion of their ETH to the router contract. Any third party (MEV bot or otherwise) can immediately drain it via `refundETH`. The loss scales linearly with how far the price limit is from the current price and the size of the trade.

---

### Likelihood Explanation

Price limits (`priceLimitX64`) are a standard, documented feature of the router interface. Any user who sets a price limit while selling ETH triggers the bug. No privileged access, no malicious setup, and no non-standard token behavior is required — only a normal `exactInputSingle` call with `msg.value > 0` and a non-zero `priceLimitX64`.

---

### Recommendation

After the swap returns, compute the unspent ETH as `msg.value - actualAmountIn` and refund it to `msg.sender`. The actual input consumed can be derived from the returned deltas:

```solidity
// inside exactInputSingle, after the swap call:
int128 actualIn = MetricOmmSwapResults.extractAmountIn(params.zeroForOne, amount0Delta, amount1Delta);
if (address(params.tokenIn) == address(WETH9) && msg.value > uint128(actualIn)) {
    refundETH(); // or: Address.sendValue(payable(msg.sender), msg.value - uint128(actualIn));
}
```

Alternatively, document that callers **must** append a `refundETH()` call via `multicall` whenever a price limit is set and ETH is the input token.

---

### Proof of Concept

```
1. Pool ETH/TKN is at price P0. Alice wants to buy TKN but only up to price P_limit < P0 + spread.
2. Alice calls:
     router.exactInputSingle{value: 1 ether}(ExactInputSingleParams{
       tokenIn:        WETH,
       tokenOut:       TKN,
       pool:           ethTknPool,
       recipient:      alice,
       amountIn:       1e18,
       amountOutMinimum: 0,
       priceLimitX64:  <price between current and upper bin boundary>,
       ...
     });
3. Pool swap loop exits early at P_limit; only 0.6 ETH consumed.
4. metricOmmSwapCallback fires with amount0Delta = 0.6e18 (positive).
5. _justPayCallback wraps 0.6 ETH from router balance and sends WETH to pool.
6. exactInputSingle returns; 0.4 ETH remains in router. No refund.
7. Bob calls router.refundETH() and receives Alice's 0.4 ETH.
``` [5](#0-4)

    _checkDeadline(params.deadline);
    uint128 priceLimitX64 = MetricOmmSwapPath.normalizePriceLimit(params.zeroForOne, params.priceLimitX64);

    _setNextCallbackContext(params.pool, CALLBACK_MODE_JUST_PAY, msg.sender, params.tokenIn);

      .swap(

        "",
        params.extensionData

    amountOut = MetricOmmSwapInputs.int128ToUint128(out);

  }
