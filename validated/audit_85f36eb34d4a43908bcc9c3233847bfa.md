### Title
Bonus Checkpoints Advance Without Payment When Pool Is Insufficient During Redelegation Slash — (`x/tieredrewards/keeper/slash.go`, `x/tieredrewards/keeper/claim_rewards.go`)

### Summary

In `slashRedelegationPosition`, when `processEventsAndClaimBonus` returns `ErrInsufficientBonusPool`, the error is deliberately swallowed to prevent a chain halt. However, by the time that error is returned, `processEventsAndClaimBonus` has already (a) decremented all traversed event `ReferenceCount` entries directly in the store, and (b) advanced `pos.LastEventSeq`, `pos.LastBonusAccrual`, and `pos.LastKnownBonded` in the in-memory `pos` struct. The caller then unconditionally persists that modified `pos` via `setPositionWithState`. The result is that the position's bonus checkpoints are permanently advanced with no corresponding bonus payment, and the consumed events may be garbage-collected, making the lost bonus irrecoverable.

---

### Finding Description

**Call chain:**

`BeforeRedelegationSlashed` → `slashRedelegationPosition` → `processEventsAndClaimBonus`

Inside `processEventsAndClaimBonus`, the event loop runs first:

```
for _, entry := range events {
    // compute bonus ...
    pos.UpdateLastEventSeq(entry.Seq)          // in-memory checkpoint advance
    k.decrementEventRefCount(ctx, valAddr, entry.Seq)  // STORE WRITE — not rolled back
}
applyBonusAccrualCheckpoint(&pos.Position, blockTime)  // in-memory checkpoint advance
pos.UpdateLastKnownBonded(bonded)                      // in-memory checkpoint advance

// ... only THEN:
if err := k.sufficientBonusPoolBalance(ctx, bonusCoins); err != nil {
    return nil, err   // returns ErrInsufficientBonusPool
}
``` [1](#0-0) 

The `decrementEventRefCount` calls are direct store writes that happen inside the loop, before the pool check. They are not part of any deferred rollback. [2](#0-1) 

Back in `slashRedelegationPosition`, the `ErrInsufficientBonusPool` error is caught and swallowed:

```go
if _, err := k.processEventsAndClaimBonus(ctx, &pos); err != nil {
    if errors.Is(err, types.ErrInsufficientBonusPool) {
        k.logger(ctx).Error(...)   // swallowed
    } else {
        return err
    }
}
// pos now has advanced checkpoints; no bonus was paid
...
return k.setPositionWithState(ctx, pos, ...)  // persists the advanced checkpoints
``` [3](#0-2) 

The `setPositionWithState` call at lines 71 or 77 persists the in-memory `pos` with its advanced `LastEventSeq`, `LastBonusAccrual`, and `LastKnownBonded` — even though no bonus coins were transferred. [4](#0-3) 

---

### Impact Explanation

The position owner permanently loses all bonus rewards that accrued between `pos.LastBonusAccrual` and the current block time. Because:

1. `pos.LastBonusAccrual` is now set to the current block time — future calls to `processEventsAndClaimBonus` will compute bonus only from this new (post-slash) timestamp onward.
2. `pos.LastEventSeq` is advanced past all processed events — those events will never be replayed for this position.
3. The event `ReferenceCount` entries were already decremented in the store — if this position was the last reference, the events are deleted and the historical rate snapshots are gone.
4. A subsequent `ClaimTierRewards` call will yield zero bonus for the lost accrual window.

This is a permanent, irrecoverable loss of accrued bonus rewards for the position owner, matching the High scope: direct economic loss from staking hook logic flaw.

---

### Likelihood Explanation

The precondition is:
- A position exists on a validator that undergoes a redelegation slash (a real on-chain event triggered by double-signing during the redelegation unbonding period).
- The `RewardsPoolName` module account balance is below the computed bonus at the time of the slash.

The second condition is realistic: the rewards pool can be depleted by normal claim activity, and the pool balance is not reserved per-position. No attacker action is required to trigger the slash itself; the position owner is the victim of the incorrect checkpoint advancement.

---

### Recommendation

The fix must ensure that checkpoint advancement and event reference-count decrements are only committed if the bonus is actually paid. Options:

1. **Check pool balance before the loop**: call `sufficientBonusPoolBalance` with the pre-computed total bonus before entering the event loop and before any `decrementEventRefCount` store writes. If insufficient, return the error without modifying any state.
2. **Separate computation from commitment**: split `processEventsAndClaimBonus` into a pure computation phase (no store writes) and a commit phase (store writes + transfer), and only enter the commit phase after the pool check passes.
3. **In `slashRedelegationPosition`, do not persist `pos` when the pool is insufficient**: if `ErrInsufficientBonusPool` is caught, call `ClearBonusCheckpoints` or skip `setPositionWithState` so the position retains its original checkpoints for a future retry. [5](#0-4) 

---

### Proof of Concept

1. Create a keeper test with a small (near-zero) `RewardsPoolName` balance.
2. Create a position on a validator with accrued bonus (advance block time so `LastBonusAccrual` is in the past).
3. Call `BeforeRedelegationSlashed` (or directly `slashRedelegationPosition`) with a valid `unbondingId` mapping to the position.
4. Assert that `pos.LastBonusAccrual` has advanced to the current block time.
5. Assert that the owner's balance did not increase.
6. Call `ClaimTierRewards` for the position and assert that bonus returned is zero.
7. Confirm that the event `ReferenceCount` entries were decremented (or events deleted) in the store. [6](#0-5) [7](#0-6)

func (k Keeper) decrementEventRefCount(ctx context.Context, valAddr sdk.ValAddress, seq uint64) error {
	key := collections.Join(valAddr, seq)
	event, err := k.ValidatorEvents.Get(ctx, key)
	if errors.Is(err, collections.ErrNotFound) {
		return nil // already cleaned up
	}
	if err != nil {
		return err
	}

	if event.ReferenceCount <= 1 {
		return k.ValidatorEvents.Remove(ctx, key)
	}

	event.ReferenceCount--
	return k.ValidatorEvents.Set(ctx, key, event)
}

func (k Keeper) slashRedelegationPosition(ctx context.Context, unbondingId uint64, sharesToUnbond math.LegacyDec) error {
	positionId, err := k.getRedelegationMapping(ctx, unbondingId)
	if errors.Is(err, collections.ErrNotFound) {
		return nil
	}
	if err != nil {
		return err
	}

	pos, err := k.getPositionState(ctx, positionId)
	if errors.Is(err, types.ErrPositionNotFound) {
		k.logger(ctx).Error("position not found during redelegation slash",
			"position_id", positionId,
			"unbonding_id", unbondingId,
			"error", err.Error(),
		)
		return nil
	}
	if err != nil {
		return err
	}

	if !pos.IsDelegated() {
		// Defensive
		k.logger(ctx).Error("delegation missing during BeforeRedelegationSlashed",
			"position_id", positionId,
			"unbonding_id", unbondingId,
			"shares_to_unbond", sharesToUnbond.String(),
		)
		return nil
	}

	dstValStr := pos.Delegation.ValidatorAddress

	// Settle bonus against PRE-slash shares.
	if _, err := k.processEventsAndClaimBonus(ctx, &pos); err != nil {
		// Deliberately forgo bonus rewards if pool is insufficient to prevent chain halt.
		if errors.Is(err, types.ErrInsufficientBonusPool) {
			k.logger(ctx).Error("insufficient bonus pool during redelegation slash",
				"position_id", pos.Id,
				"error", err.Error(),
			)
		} else {
			return err
		}
	}

	fullSlash := sharesToUnbond.GTE(pos.Delegation.Shares)

	if fullSlash {
		pos.Delegation = nil
		pos.ClearBonusCheckpoints()
		return k.setPositionWithState(ctx, pos, &ValidatorTransition{PreviousAddress: dstValStr})
	}
	// In-memory only: the persisted Position carries no share count, and the
	// live delegation will reflect the post-Unbond shares on the next read.
	// Update the local copy so any follow-up logic in this call sees consistent state.
	pos.Delegation.Shares = pos.Delegation.Shares.Sub(sharesToUnbond)
	return k.setPositionWithState(ctx, pos, nil)
```
