### Title
Missing Minimum L2 Gas Amount Validation in Gateway Stateless Validator Allows Admission of Transactions That Always Fail at Blockifier Pre-Validation — (`crates/apollo_gateway/src/stateless_transaction_validator.rs`)

### Summary

The `StatelessTransactionValidator.validate_resource_bounds()` enforces an **upper** bound on `l2_gas.max_amount` but no **lower** bound. A transaction with `l2_gas.max_amount = 0` combined with a non-zero L1 gas fee passes every gateway check, enters the mempool, and then deterministically fails at blockifier pre-validation with `InsufficientResourceBounds { MaxGasAmountTooLow { resource: L2Gas } }`. This is the direct Sequencer analog of the LayerZero `GAS_FOR_RELAY` bug: a hardcoded/config-defined constant (the zero lower bound) combined with an unchecked user-controlled field (`l2_gas.max_amount`) produces an insufficient resource specification that is accepted at the admission layer but rejected at the execution layer.

### Finding Description

`StatelessTransactionValidator.validate_resource_bounds()` performs three checks when `validate_resource_bounds = true`:

1. `max_possible_fee(Tip::ZERO) > 0` — at least one resource has both non-zero amount and non-zero price.
2. `l2_gas.max_price_per_unit >= config.min_gas_price` — L2 gas price floor.
3. `l2_gas.max_amount <= config.max_l2_gas_amount` — L2 gas amount **ceiling only**. [1](#0-0) 

There is no `l2_gas.max_amount >= some_minimum` check. The code even carries an explicit TODO acknowledging the gap: [2](#0-1) 

The `StatefulTransactionValidator.validate_resource_bounds()` likewise only checks the L2 gas **price** threshold against the previous block; it never inspects `l2_gas.max_amount`: [3](#0-2) 

The `StatefulTransactionValidatorConfig` mirrors this: only `min_gas_price_percentage` (a price ratio) is present; there is no `min_l2_gas_amount` field: [4](#0-3) 

In contrast, the blockifier's pre-validation **does** enforce a minimum L2 gas amount by calling `estimate_minimal_gas_vector` and comparing it against `resource_bounds.max_amount`: [5](#0-4) 

For every V3 invoke transaction that executes Cairo code, `estimate_minimal_gas_vector` returns a non-zero L2 gas value. A transaction with `l2_gas.max_amount = 0` therefore always fails at blockifier pre-validation with `MaxGasAmountTooLow { resource: L2Gas }`, but only **after** it has already been admitted by the gateway and placed in the mempool.

The `FeeTransactionQueue` also propagates the `validate_resource_bounds` flag without adding any amount floor: [6](#0-5) 

### Impact Explanation

**High — Mempool/gateway/RPC admission accepts invalid transactions before sequencing.**

An attacker can cheaply craft transactions that pass all gateway admission checks but are guaranteed to fail at blockifier pre-validation. Because the mempool allows only one transaction per account address at a time, each such transaction occupies a mempool slot for that account until the batcher attempts execution and discards it. With many accounts, an attacker can continuously saturate the mempool and waste batcher cycles on transactions that will never succeed, degrading sequencer throughput without paying meaningful fees (the minimum cost is a single unit of L1 gas at price 1).

### Likelihood Explanation

**Medium-High.** The crafted transaction requires only that `l1_gas = { max_amount: 1, max_price_per_unit: 1 }` (to satisfy `max_possible_fee > 0`) and `l2_gas = { max_amount: 0, max_price_per_unit: min_gas_price }` (to satisfy the L2 price floor while setting amount to zero). No privileged access, special knowledge, or contract deployment is required. The attack is reproducible with any standard V3 invoke transaction.

### Recommendation

Add a configurable `min_l2_gas_amount` field to `StatelessTransactionValidatorConfig` and enforce it in `validate_resource_bounds()`, analogous to the existing `min_gas_price` check for L2 gas price:

```rust
if resource_bounds.l2_gas.max_amount.0 < self.config.min_l2_gas_amount {
    return Err(StatelessTransactionValidatorError::MaxGasAmountTooLow {
        gas_amount: resource_bounds.l2_gas.max_amount,
        min_gas_amount: self.config.min_l2_gas_amount,
    });
}
```

The minimum value should be set to at least the L2 gas cost of a trivial account validation call (obtainable from `estimate_minimal_gas_vector` on a no-op transaction). This mirrors the LayerZero recommendation of enforcing a minimum gas floor at the source-chain admission layer rather than discovering the insufficiency only at the destination.

### Proof of Concept

1. Construct a V3 invoke transaction with:
   - `l1_gas = { max_amount: 1, max_price_per_unit: 1 }` — satisfies `max_possible_fee(Tip::ZERO) = 1 > 0`.
   - `l2_gas = { max_amount: 0, max_price_per_unit: min_gas_price }` — satisfies `l2_gas.max_price_per_unit >= min_gas_price` and `0 <= max_l2_gas_amount`.
   - `l1_data_gas = { max_amount: 0, max_price_per_unit: 0 }`.

2. Submit via `starknet_addInvokeTransaction`. The `StatelessTransactionValidator` passes all three checks: [7](#0-6) 

3. The `StatefulTransactionValidator` passes because only `l2_gas_price` is compared against the previous block threshold.

4. The transaction enters the mempool's priority queue (since `validate_resource_bounds` is true but only the price is checked against the threshold).

5. The batcher selects the transaction. The blockifier calls `estimate_minimal_gas_vector`, which returns a non-zero L2 gas value for any real invoke. The check `0 < minimal_l2_gas` triggers: [8](#0-7) 
   returning `TransactionFeeError::InsufficientResourceBounds { errors: [MaxGasAmountTooLow { resource: L2Gas, ... }] }`.

6. The transaction is discarded post-admission, having consumed a mempool slot and a batcher execution attempt at near-zero cost to the attacker.

    fn validate_resource_bounds(
        &self,
        tx: &RpcTransaction,
    ) -> StatelessTransactionValidatorResult<()> {
        if !self.config.validate_resource_bounds {
            return Ok(());
        }

        let resource_bounds = *tx.resource_bounds();
        // The resource bounds should be positive even without the tip.
        if ValidResourceBounds::AllResources(resource_bounds).max_possible_fee(Tip::ZERO) == Fee(0)
        {
            return Err(StatelessTransactionValidatorError::ZeroResourceBounds { resource_bounds });
        }

        if resource_bounds.l2_gas.max_price_per_unit.0 < self.config.min_gas_price {
            return Err(StatelessTransactionValidatorError::MaxGasPriceTooLow {
                gas_price: resource_bounds.l2_gas.max_price_per_unit,
                min_gas_price: self.config.min_gas_price,
            });
        }

        // TODO(Arni): Consider adding a validation for max_l2_gas_amount for declare.
        if let RpcTransaction::Declare(_) = tx {
        } else if resource_bounds.l2_gas.max_amount.0 > self.config.max_l2_gas_amount {
            return Err(StatelessTransactionValidatorError::MaxGasAmountTooHigh {
                gas_amount: resource_bounds.l2_gas.max_amount,
                max_gas_amount: self.config.max_l2_gas_amount,
            });
        }

        Ok(())
    }
