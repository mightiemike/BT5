Audit Report

## Title
SwapAllowlistExtension Checks Router Address Instead of End User, Allowing Allowlist Bypass via Router — (File: `metric-periphery/contracts/extensions/SwapAllowlistExtension.sol`)

## Summary
`SwapAllowlistExtension.beforeSwap` gates swaps by checking `allowedSwapper[pool][sender]`, where `sender` is the direct caller of `pool.swap()`. When swaps are routed through `MetricOmmSimpleRouter`, `sender` is the router address, not the end user. Allowlisting the router — a necessary step for any allowlisted user to use the router — grants every unprivileged user the ability to bypass the per-user allowlist, exposing LP funds to adverse selection from counterparties the allowlist was designed to exclude.

## Finding Description
`SwapAllowlistExtension.beforeSwap` performs:
```solidity
if (!allowAllSwappers[msg.sender] && !allowedSwapper[msg.sender][sender]) {
    revert IMetricOmmPoolActions.NotAllowedToSwap();
}
```
`msg.sender` is the pool (correct). `sender` is the first argument forwarded by `MetricOmmPool.swap`, which passes its own `msg.sender`:
```solidity
_beforeSwap(
    msg.sender,   // whoever called pool.swap()
    recipient, ...
);
```
When a user calls `MetricOmmSimpleRouter.exactInputSingle`, the router calls `pool.swap(...)` directly with no forwarding of the originating user:
```solidity
_setNextCallbackContext(params.pool, CALLBACK_MODE_JUST_PAY, msg.sender, params.tokenIn);
(int128 amount0Delta, int128 amount1Delta) = IMetricOmmPoolActions(params.pool)
    .swap(params.recipient, params.zeroForOne, ...);
```
The full call chain is: `user → router.exactInputSingle() → pool.swap() [msg.sender = router] → _beforeSwap(sender = router) → extension.beforeSwap(sender = router) → allowedSwapper[pool][router]`. The extension never sees the end user. If the router is allowlisted (required for any allowlisted user to use the router), the check passes for every caller regardless of individual allowlist status. There is no configuration that simultaneously permits specific users to use the router while blocking others.

## Impact Explanation
A curated pool deploying `SwapAllowlistExtension` to restrict swap counterparties (e.g., to prevent adverse selection by limiting swaps to known market participants) is structurally bypassed whenever the router is allowlisted. Any unprivileged user can execute swaps at the oracle-anchored price against the curated pool's LP liquidity. LPs suffer direct principal loss from adverse selection by counterparties the allowlist was deployed to exclude — the exact impact the guard was designed to prevent. This constitutes a broken core pool functionality (allowlist guard) causing direct loss of LP funds.

## Likelihood Explanation
`MetricOmmSimpleRouter` is a public, permissionless periphery contract callable by any user. The only precondition is that the pool admin has allowlisted the router, which is a natural and expected configuration for any pool intending to support router-mediated swaps for its allowlisted users. No special privileges, flash loans, or complex setup are required. The attack is repeatable by any address at any time once the router is allowlisted.

## Recommendation
The `beforeSwap` hook must check the end user identity, not the direct caller. Two approaches:
1. **`extensionData` forwarding**: The router encodes `msg.sender` into `extensionData`; the extension decodes and validates it against the allowlist. This requires a protocol-level convention between routers and extensions.
2. **Trusted router registry**: Extend the extension so that for known trusted routers, it reads a forwarded originating user address and validates that against the allowlist instead of `sender`.

At minimum, document that `SwapAllowlistExtension` is incompatible with router-mediated swaps and must only be used with direct pool calls until a forwarding mechanism is implemented.

## Proof of Concept
1. Deploy a pool with `SwapAllowlistExtension` configured.
2. Pool admin calls `setAllowedToSwap(pool, userA, true)` and `setAllowedToSwap(pool, router, true)` (so `userA` can use the router).
3. `userB` (not individually allowlisted) calls `router.exactInputSingle({ pool: curatedPool, ... })`.
4. Router calls `pool.swap(...)` — `msg.sender` seen by pool is `router`.
5. Pool calls `_beforeSwap(sender = router, ...)`.
6. Extension checks `allowedSwapper[pool][router] == true` → passes.
7. `userB`'s swap executes against the curated pool's LP liquidity at the oracle price.
8. LPs suffer losses from an unauthorized counterparty the allowlist was designed to exclude.
