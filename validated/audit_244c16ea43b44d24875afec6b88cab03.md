### Title
ML-DSA-65 gas-key host-function fee paths use wire length instead of `trie_id_len`, breaking gas-fee parity with the transaction path — (`core/parameters/src/cost.rs`, `runtime/near-vm-runner/src/logic/logic.rs`)

---

### Summary

The `PostQuantumSignatures` feature (protocol version 85, now stable) introduced ML-DSA-65 access keys whose on-trie identifier is a 33-byte SHA3-256 hash, not the 1953-byte borsh-encoded full pubkey. The transaction path correctly uses `PublicKey::trie_id_len()` (33) for all gas-key fee calculations. The host-function path passes the raw `public_key_len` argument (1953 for ML-DSA-65) to the same fee helpers, causing a ~59× overcharge for every ML-DSA-65 gas-key action dispatched from a WASM contract.

---

### Finding Description

**Transaction path** (`runtime/runtime/src/config.rs`):

```rust
TransferToGasKey(action) => {
    gas_key_transfer_send_fee(fees, sender_is_receiver, action.public_key.trie_id_len())
        .total()
}
// exec_fee:
TransferToGasKey(action) => {
    gas_key_transfer_exec_fee(fees, receiver_id.len(), action.public_key.trie_id_len())
        .total()
}
```

`trie_id_len()` returns **33** for ML-DSA-65. [1](#0-0) 

**Host-function path** (`runtime/near-vm-runner/src/logic/logic.rs`):

```rust
let send = gas_key_transfer_send_fee(&self.fees_config, sir, public_key_len as usize);
let exec = gas_key_transfer_exec_fee(
    &self.fees_config,
    receiver_id.len(),
    public_key_len as usize,   // ← raw borsh length: 1953 for ML-DSA-65
);
```

`public_key_len` is the byte-length of the borsh blob the WASM contract wrote into memory — **1953** for ML-DSA-65 (`1 tag + 1952 raw bytes`). [2](#0-1) 

Inside `gas_key_transfer_exec_fee`, that value is fed directly into the trie-key-length formula:

```rust
let trie_key_len = access_key_key_len(account_id_len, public_key_len);
``` [3](#0-2) 

The actual trie key for an ML-DSA-65 access key is `2 + account_id_len + 33` bytes (the hash form), not `2 + account_id_len + 1953`. The overcharge per action is **1920 bytes × `gas_key_byte` exec fee**.

The same wrong-length substitution appears in both host-function implementations of `promise_batch_action_add_gas_key_with_full_access` and `promise_batch_action_add_gas_key_with_function_call`:

```rust
let exec_fee = gas_key_add_key_exec_fee(
    &self.fees_config,
    receiver_id.len(),
    public_key_len as usize,   // ← 1953 instead of 33
    num_nonces,
);
``` [4](#0-3) [5](#0-4) 

Inside `gas_key_add_key_exec_fee`, the nonce-key length is computed as:

```rust
let nonce_key_len =
    access_key_key_len(account_id_len, public_key_len) + std::mem::size_of::<NonceIndex>();
``` [6](#0-5) 

For ML-DSA-65 with `num_nonces` nonces, the overcharge is `num_nonces × 1920 × gas_key_byte exec fee`.

The design document explicitly states that every storage-stake and trie-byte-priced fee path must call `trie_id_len()`:

> "Callers that still use `len()` for trie-storage costing will misprice ML-DSA-65 keys by ~1900 bytes." [7](#0-6) 

The existing fee-parity integration test uses only ED25519 keys (where `len() == trie_id_len()`), so the divergence for ML-DSA-65 is not caught: [8](#0-7) 

The `trie_id_len` / `len` split is defined here: [9](#0-8) 

---

### Impact Explanation

Any WASM contract that dispatches `TransferToGasKey`, `WithdrawFromGasKey`, or gas-key `AddKey` actions for an ML-DSA-65 key via the host-function API is charged ~59× more gas than the identical action submitted as a signed transaction. The divergence is deterministic and consensus-breaking: two nodes computing gas costs for the same receipt will agree (both use `public_key_len`), but the protocol invariant that "the same action costs the same gas regardless of submission path" is violated. Contracts that budget gas based on the transaction-path cost will run out of gas when the action is dispatched via a promise. The overcharge scales with `num_nonces` for `AddKey`, making gas-key creation especially affected.

---

### Likelihood Explanation

`PostQuantumSignatures` is stabilized at protocol version 85 and the current stable version is 86, so the feature is live on mainnet. Any unprivileged user can deploy a WASM contract and call the affected host functions with an ML-DSA-65 public key. No special role or privilege is required.

---

### Recommendation

In all three host-function implementations (`promise_batch_action_transfer_to_gas_key`, `promise_batch_action_add_gas_key_with_full_access`, `promise_batch_action_add_gas_key_with_function_call`), decode the public key before computing fees and pass `public_key.trie_id_len()` instead of `public_key_len as usize` to `gas_key_transfer_send_fee`, `gas_key_transfer_exec_fee`, and `gas_key_add_key_exec_fee`. The wasmtime mirror of each function (`runtime/near-vm-runner/src/wasmtime_runner/logic.rs`) requires the same fix. Extend `test_gas_key_fee_parity` to cover `KeyType::MLDSA65` to prevent regression.

---

### Proof of Concept

1. Deploy a WASM contract that calls `promise_batch_action_transfer_to_gas_key` with a 1953-byte ML-DSA-65 borsh blob.
2. Submit the identical `TransferToGasKey` action as a signed transaction using the same ML-DSA-65 key.
3. Compare `gas_burnt` on the two execution outcomes.

**Transaction path fee** (per `gas_key_byte` unit):
- send: `33 × gas_key_byte_send`
- exec: `(2 + account_id_len + 33 + value_len) × gas_key_byte_exec`

**Host-function path fee** (per `gas_key_byte` unit):
- send: `1953 × gas_key_byte_send`
- exec: `(2 + account_id_len + 1953 + value_len) × gas_key_byte_exec`

The exact divergent values are **1953** (host-function path) vs **33** (transaction path) — a difference of **1920 bytes** per fee dimension, sourced from `PublicKey::len()` vs `PublicKey::trie_id_len()` for `PublicKey::MLDSA65`. [10](#0-9) [9](#0-8)

pub fn gas_key_transfer_exec_fee(
    cfg: &RuntimeFeesConfig,
    account_id_len: usize,
    public_key_len: usize,
) -> GasKeyTransferFee {
    let base = cfg.fee(ActionCosts::gas_key_transfer_base).exec_fee();
    let trie_key_len = access_key_key_len(account_id_len, public_key_len);
    let estimated_value_len = AccessKey::min_gas_key_borsh_len();
    let per_byte = cfg
        .fee(ActionCosts::gas_key_byte)
        .exec_fee()
        .checked_mul((trie_key_len + estimated_value_len) as u64)
        .unwrap();
    GasKeyTransferFee { base, per_byte }
}
