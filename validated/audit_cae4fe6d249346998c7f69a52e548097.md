### Title
SwapAllowlistExtension Gates the Router Address Instead of the Actual User, Allowing Any Caller to Bypass a Curated Pool's Swap Allowlist — (`metric-periphery/contracts/extensions/SwapAllowlistExtension.sol`)

---

### Summary

`SwapAllowlistExtension.beforeSwap` checks the `sender` argument forwarded by the pool, which equals `msg.sender` of the `pool.swap()` call. When a user routes through `MetricOmmSimpleRouter`, the pool's `msg.sender` is the router contract, not the original user. The extension therefore checks whether the **router** is allowlisted, not whether the **user** is allowlisted. Any non-allowlisted user can bypass a curated pool's swap gate by routing through the public router.

---

### Finding Description

**Call chain for a direct swap (correct):**

```
User → pool.swap()
  pool: msg.sender = User
  _beforeSwap(sender=User, ...)
  SwapAllowlistExtension.beforeSwap(sender=User, ...)
    checks allowedSwapper[pool][User]  ✓
```

**Call chain through the router (bypassed):**

```
User → MetricOmmSimpleRouter.exactInputSingle(...)
  router → pool.swap(recipient, ...)
    pool: msg.sender = Router
    _beforeSwap(sender=Router, ...)
    SwapAllowlistExtension.beforeSwap(sender=Router, ...)
      checks allowedSwapper[pool][Router]  ← wrong actor
```

In `MetricOmmPool.swap`, the pool passes `msg.sender` as `sender` to `_beforeSwap`: [1](#0-0) 

`ExtensionCalling._beforeSwap` forwards that `sender` verbatim to every configured extension: [2](#0-1) 

`SwapAllowlistExtension.beforeSwap` then checks `allowedSwapper[msg.sender][sender]`, where `msg.sender` is the pool and `sender` is whoever called `pool.swap()`: [3](#0-2) 

`MetricOmmSimpleRouter.exactInputSingle` calls `pool.swap()` directly, making the router the pool's `msg.sender`: [4](#0-3) 

The same applies to `exactInput` (all hops), `exactOutputSingle`, and `exactOutput`: [5](#0-4) 

The pool admin faces an impossible choice:

- **Do not allowlist the router** → legitimate allowlisted users cannot use the router at all (broken core functionality).
- **Allowlist the router** → every user on the network can bypass the allowlist by routing through the router (complete policy bypass).

There is no configuration that simultaneously allows allowlisted users to use the router and blocks non-allowlisted users.

---

### Impact Explanation

A curated pool deploying `SwapAllowlistExtension` to restrict trading to KYC'd, whitelisted, or otherwise vetted addresses is completely ineffective against any user who routes through `MetricOmmSimpleRouter`. The non-allowlisted user receives the full swap output from the pool's LP reserves, causing direct loss to LPs who deposited under the assumption that only vetted counterparties would trade against them. This matches the **allowlist bypass** and **wrong-actor binding** impact categories: a disallowed user can still trade, and the economic action is attributed to the router rather than the actual user.

---

### Likelihood Explanation

`MetricOmmSimpleRouter` is a publicly deployed, permissionless periphery contract. No special privilege, front-running, or flash loan is required. Any user who knows the pool address can call `exactInputSingle` with their own address as `recipient` and bypass the allowlist in a single transaction. The attack is trivially repeatable on every block.

---

### Recommendation

The extension must gate the **original user**, not the immediate caller of `pool.swap()`. Two sound approaches:

1. **Pass `tx.origin` as an additional argument** in the extension hook so the extension can check the true initiator. This requires a protocol-level change to the `beforeSwap` signature.

2. **Require the router to forward the original caller** as an explicit parameter and have the pool pass it as `sender` instead of `msg.sender`. The router already tracks the original `msg.sender` in transient storage (as the payer); the same identity should be forwarded as the swap sender.

3. **Alternatively**, the `SwapAllowlistExtension` should check `sender` against a router-aware registry: if `sender` is a known router, look up the actual payer from the router's transient context. This is complex and fragile.

The cleanest fix is option 2: the pool's `swap` entry point should accept an explicit `swapper` address (defaulting to `msg.sender` for direct calls) and the router should pass `msg.sender` (the original user) as that argument.

---

### Proof of Concept

```
Setup:
  - Deploy pool with SwapAllowlistExtension configured as beforeSwap hook.
  - Pool admin calls setAllowedToSwap(pool, alice, true).
  - Pool admin does NOT allowlist bob or the router.
  - Pool has liquidity.

Attack (bob bypasses allowlist):
  1. bob calls MetricOmmSimpleRouter.exactInputSingle({
       pool: pool,
       recipient: bob,
       zeroForOne: true,
       amountIn: X,
       ...
     })
  2. Router calls pool.swap(bob, true, X, ...) — pool sees msg.sender = router.
  3. pool._beforeSwap(sender=router, ...) is called.
  4. SwapAllowlistExtension.beforeSwap(sender=router, ...) checks:
       allowAllSwappers[pool] == false
       allowedSwapper[pool][router] == false  ← router not allowlisted
     → reverts NotAllowedToSwap.

  If pool admin allowlists the router to let alice use it:
  5. Pool admin calls setAllowedToSwap(pool, router, true).
  6. bob repeats step 1.
  7. SwapAllowlistExtension checks allowedSwapper[pool][router] == true → passes.
  8. bob receives swap output. Allowlist is fully bypassed.

Result: The pool admin cannot allowlist alice-via-router without also allowlisting bob-via-router.
        The SwapAllowlistExtension provides zero protection against router-mediated swaps.
```
