### Title
Gas-Key Host-Function Fee Computed in Borsh Wire Length Instead of Trie-ID Length for ML-DSA-65 Keys — (File: `runtime/near-vm-runner/src/logic/logic.rs`)

### Summary

The gas-key host functions (`promise_batch_action_transfer_to_gas_key`, `promise_batch_action_add_gas_key_with_full_access`, `promise_batch_action_add_gas_key_with_function_call`) compute per-byte fees using `public_key_len as usize` — the raw borsh wire length of the public key as supplied by the calling contract — rather than the trie-ID length (`trie_id_len()`). For ML-DSA-65 keys the borsh wire length is 1953 bytes while the trie-ID length is 33 bytes, producing a fee that is ~59× larger than the fee charged for the identical action submitted as a signed transaction.

### Finding Description

The transaction path in `runtime/runtime/src/config.rs` correctly uses `action.public_key.trie_id_len()` for every gas-key fee helper:

```rust
// total_send_fees
TransferToGasKey(action) => {
    gas_key_transfer_send_fee(fees, sender_is_receiver, action.public_key.trie_id_len())
        .total()
}
// exec_fee
TransferToGasKey(action) => {
    gas_key_transfer_exec_fee(fees, receiver_id.len(), action.public_key.trie_id_len())
        .total()
}

The design document explicitly states that every storage-stake and trie-byte-priced fee path must call `trie_id_len()`, not `len()`:

> `len()` reports the borsh-encoded length (33 / 65 / **1953** across the three `PublicKey` variants). `trie_id_len()` reports the on-trie length (33 / 65 / **33**). The two diverge only for `PublicKey::MLDSA65`. Every storage-stake and trie-byte-priced fee path was updated to call `trie_id_len()`. [9](#0-8) 

The runtime `access_keys.rs` correctly uses `trie_id_len()`: [10](#0-9) 

But the host-function path never decodes the key before computing the fee, so it uses the borsh wire length (1953) instead of the trie-ID length (33).

**Exact divergent values for ML-DSA-65:**

| Path | `public_key_len` used for fee | Correct value |
|---|---|---|
| Signed transaction | `trie_id_len()` = 33 bytes | 33 bytes |
| Host function | `public_key_len` = 1953 bytes | 33 bytes |

Ratio: 1953 / 33 ≈ **59×** overcharge on every per-byte gas-key fee component.

For ED25519 and SECP256K1 keys `len()` equals `trie_id_len()`, so those key types are unaffected.

### Impact Explanation

Any unprivileged contract that calls `promise_batch_action_transfer_to_gas_key`, `promise_batch_action_add_gas_key_with_full_access`, or `promise_batch_action_add_gas_key_with_function_call` with an ML-DSA-65 public key is charged ~59× more gas than the protocol intends for that action. This breaks the protocol invariant that the same action costs the same gas regardless of submission path (transaction vs. host function). Contracts relying on gas-key operations with ML-DSA-65 keys will fail with `GasExceeded` at gas budgets that would succeed for the equivalent signed transaction, making the feature effectively unusable via the host-function API.

### Likelihood Explanation

ML-DSA-65 (`PostQuantumSignatures`) was stabilized at protocol version 85 and is active on mainnet. Gas keys are also a production feature. Any contract that combines both — e.g., a contract that programmatically manages ML-DSA-65 gas keys — triggers the divergence. The bug is deterministic and reproducible with any ML-DSA-65 public key passed to the affected host functions.

### Recommendation

After decoding the public key from memory, use `decoded_public_key.trie_id_len()` instead of `public_key_len as usize` when calling the gas-key fee helpers. For example, in `promise_batch_action_transfer_to_gas_key`:

```rust
let public_key = self.get_public_key(public_key_ptr, public_key_len)?;
let decoded = public_key.decode()?;
let key_fee_len = decoded.trie_id_len();   // 33 for ML-DSA-65, same as before for ED/SECP
let send = gas_key_transfer_send_fee(&self.fees_config, sir, key_fee_len);
let exec = gas_key_transfer_exec_fee(&self.fees_config, receiver_id.len(), key_fee_len);
```

Apply the same fix to `promise_batch_action_add_gas_key_with_full_access` and `promise_batch_action_add_gas_key_with_function_call` in both `logic.rs` and `wasmtime_runner/logic.rs`.

### Proof of Concept

1. Deploy a contract that calls `promise_batch_action_transfer_to_gas_key` targeting an ML-DSA-65 public key (borsh-serialized: 1 tag byte + 1952 key bytes = 1953 bytes total).
2. Submit the identical `TransferToGasKey` action as a signed transaction using the same ML-DSA-65 key.
3. Compare `gas_burnt` on the action execution receipt:
   - Transaction path: fee computed with `trie_id_len()` = 33 bytes.
   - Host-function path: fee computed with `public_key_len` = 1953 bytes → ~59× larger.
4. The host-function call will exhaust any gas budget that the transaction path handles successfully, confirming the divergence. [11](#0-10) [12](#0-11) [13](#0-12)

            burn_byte.gas.checked_add(exec.per_byte.gas).ok_or(HostError::IntegerOverflow)?;

            burn_byte,
            use_byte,
            ActionCosts::gas_key_byte,
        )?;
        self.result_state.deduct_balance(amount)?;
        self.ext.append_action_transfer_to_gas_key(receipt_idx, public_key.decode()?, amount);

pub fn gas_key_transfer_exec_fee(
    cfg: &RuntimeFeesConfig,
    account_id_len: usize,
    public_key_len: usize,
) -> GasKeyTransferFee {
    let base = cfg.fee(ActionCosts::gas_key_transfer_base).exec_fee();
    let trie_key_len = access_key_key_len(account_id_len, public_key_len);
    let estimated_value_len = AccessKey::min_gas_key_borsh_len();
    let per_byte = cfg
        .fee(ActionCosts::gas_key_byte)
        .exec_fee()
        .checked_mul((trie_key_len + estimated_value_len) as u64)
        .unwrap();
    GasKeyTransferFee { base, per_byte }
}

fn access_key_storage_usage(
    fee_config: &RuntimeFeesConfig,
    public_key: &PublicKey,
    access_key: &AccessKey,
) -> StorageUsage {
    let storage_usage_config = &fee_config.storage_usage_config;
    // Use the on-trie identifier length, not the borsh-serialized pubkey
    // length: ML-DSA-65 access keys live in the trie as a SHA3-256 hash
    // (33 bytes incl. type tag), not as a 1953-byte full pubkey.
    public_key.trie_id_len() as u64
        + borsh::object_length(access_key).unwrap() as u64
        + storage_usage_config.num_extra_bytes_record
}
