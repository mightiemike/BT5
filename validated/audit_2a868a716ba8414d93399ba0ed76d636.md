### Title
Consensus Vote Signature Never Verified — Any Peer Can Forge Votes for Any Committee Member - (File: `crates/apollo_consensus/src/single_height_consensus.rs`)

### Summary

`handle_vote` in `SingleHeightConsensus` accepts BFT consensus votes from any network peer claiming to be any committee member without verifying the cryptographic signature on the vote. The signature field is simultaneously never populated when votes are created. Any unprivileged network peer can inject forged prevotes or precommits attributed to arbitrary validators, potentially manufacturing a false quorum and causing the sequencer to commit an unauthorized block.

### Finding Description

The analog to the external `mintWithPermit` bug is exact: just as `mintWithPermit` calls `permit()` and proceeds without checking whether the approval was actually granted, `handle_vote` receives a `Vote` struct carrying a `signature` field and proceeds to count it toward quorum without ever checking whether that signature is valid.

In `crates/apollo_consensus/src/single_height_consensus.rs`, `handle_vote` performs only two checks before accepting a vote:

1. `vote.height == current_height`
2. `vote.voter` is a member of the committee

The signature is explicitly skipped with a `TODO`:

The root cause is compounded by the fact that votes are also *created* with a default (zero) signature, confirming the signing infrastructure is not yet wired in:

The `SignatureManager` and `verify_precommit_vote_signature` / `verify_identity` functions exist and are tested in isolation, but are never called on the vote path. [3](#0-2) 

### Impact Explanation

An unprivileged network peer can send `Vote` protobuf messages claiming `voter = <any_committee_member_address>`. Because the committee membership check passes (the address is real) and the signature is never verified, the vote is counted toward quorum. By sending forged prevotes and precommits attributed to 2/3+ of the committee, an attacker can manufacture a false quorum and cause the sequencer to commit a block it should not, producing wrong committed state. This matches the **Critical** impact: *Wrong state from blockifier/syscall/execution logic for accepted input*, and *Invalid or unauthorized transaction accepted through signature logic*.

### Likelihood Explanation

The trigger requires only network-level access to the sequencer's P2P broadcast channel — no privileged credentials. The committee member addresses are public (they are part of the consensus committee configuration). The `TODO` comment confirms this is a known, unimplemented control, not a design choice.

### Recommendation

Before forwarding any received `Vote` to the state machine, call `verify_precommit_vote_signature` (or the equivalent prevote verifier) using the committee member's registered public key. Reject votes whose signature does not verify. Simultaneously, wire `SignatureManager::sign_precommit_vote` (and the prevote equivalent) into `make_self_vote` to replace `RawSignature::default()`. [4](#0-3) 

### Proof of Concept

1. Obtain the committee member addresses for the current height from any public source (e.g., the consensus configuration or a prior block).
2. Construct a `Vote` protobuf message with `voter = <target_validator_address>`, `height = current_height`, `round = 0`, `vote_type = Precommit`, `proposal_commitment = <attacker_chosen_block_id>`, and `signature = []` (empty bytes).
3. Broadcast this message to the sequencer's consensus P2P topic.
4. Repeat for 2/3 of the committee weight.
5. Observe that `handle_vote` accepts each message (the `// TODO(Asmaa): verify the signature` line is never reached as a gate), the state machine accumulates quorum weight, and `DecisionReached` fires for the attacker-chosen block commitment. [5](#0-4) [6](#0-5)

    pub(crate) fn handle_vote(&mut self, vote: Vote) -> Requests {
        // TODO(Asmaa): verify the signature
        trace!("Received {:?}", vote);
        let height = self.state_machine.height();
        if vote.height != height {
            warn!("Invalid vote height: expected {:?}, got {:?}", height, vote.height);
            return VecDeque::new();
        }
        if !self.committee.members().iter().any(|s| s.address == vote.voter) {
            debug!("Ignoring vote from non validator: vote={:?}", vote);
            return VecDeque::new();
        }

        // Check if vote has already been received.
        match self.state_machine.received_vote(&vote) {
            VoteStatus::Duplicate => {
                // Duplicate - ignore.
                trace_every_n_ms!(
                    DUPLICATE_VOTE_LOG_PERIOD_MS,
                    "Ignoring duplicate vote: {vote:?}"
                );
                return VecDeque::new();
            }
            VoteStatus::Conflict(old_vote, new_vote) => {
                // Conflict - ignore and record.
                warn!("Conflicting votes: old={old_vote:?}, new={new_vote:?}");
                CONSENSUS_CONFLICTING_VOTES.increment(1);
                return VecDeque::new();
            }
            VoteStatus::New => {
                // Vote is new, proceed to process it.
            }
        }

        info!("Accepting {:?}", vote);
        let sm_vote = match vote.vote_type {
            VoteType::Prevote => StateMachineEvent::Prevote(vote),
            VoteType::Precommit => StateMachineEvent::Precommit(vote),
        };
        self.state_machine.handle_event(sm_vote)
    }

    async fn sign(&self, message_digest: MessageDigest) -> SignatureManagerResult<RawSignature> {
        let private_key = self.keystore.get_key().await?;
        let signature = ecdsa_sign(&private_key, &message_digest)
