### Title
`SwapAllowlistExtension` Gates the Router Address Instead of the Originating User, Allowing Any User to Bypass the Swap Allowlist via `MetricOmmSimpleRouter` - (File: `metric-periphery/contracts/extensions/SwapAllowlistExtension.sol`)

---

### Summary

`SwapAllowlistExtension.beforeSwap` checks `allowedSwapper[pool][sender]` where `sender` is the immediate caller of `MetricOmmPool.swap()`. When a user routes through `MetricOmmSimpleRouter`, `sender` is the **router's address**, not the originating EOA. If the router is allowlisted (required for any user to use it), every non-allowlisted user can bypass the curated-pool gate by routing through the router.

---

### Finding Description

The call chain for a router-mediated swap is:

```
EOA → MetricOmmSimpleRouter.exactInputSingle(params)
        → IMetricOmmPoolActions(params.pool).swap(recipient, zeroForOne, amount, limit, "", extensionData)
              // msg.sender to pool = MetricOmmSimpleRouter
          → MetricOmmPool._beforeSwap(msg.sender, ...)
              // sender = MetricOmmSimpleRouter address
            → ExtensionCalling._callExtensionsInOrder(...)
              → SwapAllowlistExtension.beforeSwap(sender=router, ...)
                  // checks allowedSwapper[pool][router], NOT allowedSwapper[pool][EOA]
```

In `MetricOmmPool.swap`, the pool passes its own `msg.sender` as the `sender` argument to `_beforeSwap`: [1](#0-0) 

`ExtensionCalling._beforeSwap` forwards that value unchanged to every configured extension: [2](#0-1) 

`SwapAllowlistExtension.beforeSwap` then evaluates the allowlist keyed on that `sender`: [3](#0-2) 

When the router calls `pool.swap()`, `sender` is the router's address, so the check becomes `allowedSwapper[pool][router]`, not `allowedSwapper[pool][EOA]`.

In `MetricOmmSimpleRouter.exactInputSingle`, the router calls `pool.swap` directly with no forwarding of the original `msg.sender` as the `sender` identity: [4](#0-3) 

---

### Impact Explanation

Two mutually exclusive failure modes exist:

**Mode A — Router is allowlisted (necessary for any user to use it):**  
Any non-allowlisted EOA calls `exactInputSingle` through the router. The extension sees `sender = router`, finds `allowedSwapper[pool][router] = true`, and passes. The individual-user allowlist is completely defeated. Every user on the network can trade on a pool that was intended to be curated.

**Mode B — Router is not allowlisted:**  
Allowlisted EOAs cannot use the router at all; they must call `pool.swap()` directly. This breaks the expected periphery UX and forces allowlisted users to implement their own callback settlement, which is not a supported pattern.

Both modes represent a broken core invariant: the allowlist policy is unenforceable through the supported public periphery path.

---

### Likelihood Explanation

- The `MetricOmmSimpleRouter` is the primary supported swap entrypoint for EOAs.
- Pool admins who deploy a `SwapAllowlistExtension` will naturally need to allowlist the router to make the pool usable, triggering Mode A.
- No special permissions, malicious setup, or non-standard tokens are required. Any EOA can call `exactInputSingle` on a router-allowlisted curated pool.
- The bypass is reachable on every router-mediated swap: `exactInputSingle`, `exactOutputSingle`, `exactInput`, and `exactOutput`.

---

### Recommendation

Pass the originating user identity through the swap path. Two options:

1. **Add an `originator` field to the pool's `swap` signature** and have the router populate it with `msg.sender`. The extension then checks `allowedSwapper[pool][originator]`.

2. **Check `recipient` instead of `sender`** if the pool's design guarantees the recipient is always the economic beneficiary. This is weaker because `recipient` can be set to any address.

The cleanest fix is option 1: the pool passes both `msg.sender` (the immediate caller) and an explicit `originator` (the EOA), and the extension is free to gate on either.

---

### Proof of Concept

1. Deploy a pool with `SwapAllowlistExtension` configured.
2. Pool admin calls `setAllowedToSwap(pool, router, true)` — required for any user to use the router.
3. Non-allowlisted EOA `Bob` calls `MetricOmmSimpleRouter.exactInputSingle({pool: pool, ...})`.
4. Router calls `pool.swap(recipient, ...)` with `msg.sender = router`.
5. Pool calls `_beforeSwap(router, ...)`.
6. Extension evaluates `allowedSwapper[pool][router] == true` → passes.
7. Bob's swap executes on the curated pool despite never being individually allowlisted.

Alternatively, if the admin does **not** allowlist the router:

1. Pool admin calls `setAllowedToSwap(pool, alice, true)` for a legitimate user Alice.
2. Alice calls `MetricOmmSimpleRouter.exactInputSingle({pool: pool, ...})`.
3. Extension evaluates `allowedSwapper[pool][router] == false` → reverts with `NotAllowedToSwap`.
4. Alice cannot use the router even though she is individually allowlisted.
