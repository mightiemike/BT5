### Title
Bonus Reward Accounting Desynchronization in `slashRedelegationPosition` When Pool Is Insufficient — (File: `x/tieredrewards/keeper/slash.go`)

---

### Summary

In `slashRedelegationPosition`, when `processEventsAndClaimBonus` returns `ErrInsufficientBonusPool`, the error is silently swallowed to avoid a chain halt. However, by the time the error is returned, the position's bonus checkpoints (`LastEventSeq`, `LastBonusAccrual`, `LastKnownBonded`) have already been updated in-memory, and validator event reference counts have already been decremented in the store. The caller then persists the position with the advanced checkpoints. This creates a permanent disparity: the position's state records that bonus accrual was processed up to the current block, but the owner never received the bonus coins — and can never recover them.

---

### Finding Description

**Root cause in `processEventsAndClaimBonus`** (`x/tieredrewards/keeper/claim_rewards.go`):

The function performs two classes of side effects before checking pool balance:

1. **Persisted immediately (store writes):** Inside the event loop, `decrementEventRefCount` is called for each processed event. These writes go to the cache store and are committed with the transaction.
2. **In-memory updates:** `pos.UpdateLastEventSeq`, `applyBonusAccrualCheckpoint`, and `pos.UpdateLastKnownBonded` update the in-memory `PositionState`.

The pool balance check (`sufficientBonusPoolBalance`) happens **after** both of the above:

```go
// All checkpoints updated, all ref counts decremented — THEN:
if err := k.sufficientBonusPoolBalance(ctx, bonusCoins); err != nil {
    return nil, err   // ErrInsufficientBonusPool returned here
}
// SendCoinsFromModuleToAccount — never reached
``` [1](#0-0) 

**Root cause in `slashRedelegationPosition`** (`x/tieredrewards/keeper/slash.go`):

The hook swallows `ErrInsufficientBonusPool` and then calls `setPositionWithState` with the in-memory `pos` that already has advanced checkpoints:

```go
if _, err := k.processEventsAndClaimBonus(ctx, &pos); err != nil {
    if errors.Is(err, types.ErrInsufficientBonusPool) {
        k.logger(ctx).Error("insufficient bonus pool during redelegation slash", ...)
        // execution continues — pos has advanced checkpoints
    } else {
        return err
    }
}
// ...
return k.setPositionWithState(ctx, pos, ...)  // persists advanced checkpoints
``` [2](#0-1) 

The two accounting values that must always move together are:
- **Position checkpoints** (`LastEventSeq`, `LastBonusAccrual`, `LastKnownBonded`) — advance to record "bonus processed through block T"
- **Bonus coins sent to owner** — the actual payment corresponding to that accrual period

In the `ErrInsufficientBonusPool` path inside `slashRedelegationPosition`, the checkpoints advance but the coins are never sent. This is the exact same class of desynchronization as the reference report.

---

### Impact Explanation

The position owner permanently loses all bonus rewards that had accrued since their last claim up to the slash block. Because:

1. `LastBonusAccrual` is advanced to `blockTime` (or `ExitUnlockAt` if exiting), so the next call to `processEventsAndClaimBonus` starts from that timestamp — the forfeited period is gone.
2. Event reference counts are decremented and potentially garbage-collected, so the validator events covering the forfeited period may no longer exist in the store.
3. There is no retry mechanism: the user-driven paths (ClaimTierRewards, etc.) fail atomically and allow retry, but the hook path silently advances state, making recovery impossible.

The magnitude of loss scales with `shares × tokensPerShare × BonusApy × duration / SecondsPerYear` — for a large, long-running position this can be a material amount of the bond denom. [3](#0-2) 

---

### Likelihood Explanation

The trigger requires two concurrent conditions:

1. A tier position undergoes a redelegation slash (`BeforeRedelegationSlashed` fires). This requires the position owner to have called `MsgTierRedelegate` within the redelegation window before the source validator is slashed — a realistic lifecycle event.
2. The bonus rewards pool (`RewardsPoolName`) is insufficient to cover the accrued bonus at slash time. The pool is a shared resource drained by all bonus claims; it can be depleted by normal user activity and may not be replenished immediately.

Both conditions are reachable by unprivileged actors through normal protocol transactions (`MsgTierRedelegate` + validator downtime/double-sign). The combination is low-to-medium probability but not exotic. [4](#0-3) [5](#0-4) 

---

### Recommendation

Move the pool balance check **before** any checkpoint updates and event reference count decrements. If the pool is insufficient, return the error without modifying any state — the position's checkpoints remain at their pre-call values, and the user can retry after the pool is replenished (consistent with the user-driven path behavior).

Alternatively, if the silent-forfeit design is intentional for chain-halt avoidance, the checkpoints must **not** be advanced when the bonus is forfeited. The in-memory `pos` should be restored to its pre-call checkpoint values before `setPositionWithState` is called, so the forfeited period remains claimable once the pool is replenished. [6](#0-5) 

---

### Proof of Concept

1. User calls `MsgLockTier` → position created, delegated to validator V1.
2. User calls `MsgTierRedelegate` → position redelegated to V2; `RedelegationMappings[unbondingId] = positionId` is set.
3. Time passes; bonus accrues on V2.
4. All other users claim rewards, draining the bonus pool to zero.
5. V1 is slashed for downtime while the redelegation entry is still within the redelegation window.
6. Staking calls `BeforeRedelegationSlashed(unbondingId, sharesToUnbond)` → `slashRedelegationPosition` → `processEventsAndClaimBonus`.
7. Inside `processEventsAndClaimBonus`: event ref counts are decremented (store write), `LastEventSeq`/`LastBonusAccrual`/`LastKnownBonded` are updated in-memory, then `sufficientBonusPoolBalance` returns `ErrInsufficientBonusPool`.
8. `slashRedelegationPosition` logs the error and continues; calls `setPositionWithState` with the in-memory `pos` that has advanced checkpoints.
9. Transaction commits: position checkpoints are now at block T, events are garbage-collected, but the owner received zero bonus coins.
10. User calls `MsgClaimTierRewards` later (after pool is replenished) — `processEventsAndClaimBonus` starts from the advanced `LastBonusAccrual`, skipping the forfeited period entirely. Bonus is permanently lost. [2](#0-1) [7](#0-6) [8](#0-7)

func (k Keeper) slashRedelegationPosition(ctx context.Context, unbondingId uint64, sharesToUnbond math.LegacyDec) error {
	positionId, err := k.getRedelegationMapping(ctx, unbondingId)
	if errors.Is(err, collections.ErrNotFound) {
		return nil
	}
	if err != nil {
		return err
	}

	pos, err := k.getPositionState(ctx, positionId)
	if errors.Is(err, types.ErrPositionNotFound) {
		k.logger(ctx).Error("position not found during redelegation slash",
			"position_id", positionId,
			"unbonding_id", unbondingId,
			"error", err.Error(),
		)
		return nil
	}
	if err != nil {
		return err
	}

	if !pos.IsDelegated() {
		// Defensive
		k.logger(ctx).Error("delegation missing during BeforeRedelegationSlashed",
			"position_id", positionId,
			"unbonding_id", unbondingId,
			"shares_to_unbond", sharesToUnbond.String(),
		)
		return nil
	}

	dstValStr := pos.Delegation.ValidatorAddress

	// Settle bonus against PRE-slash shares.
	if _, err := k.processEventsAndClaimBonus(ctx, &pos); err != nil {
		// Deliberately forgo bonus rewards if pool is insufficient to prevent chain halt.
		if errors.Is(err, types.ErrInsufficientBonusPool) {
			k.logger(ctx).Error("insufficient bonus pool during redelegation slash",
				"position_id", pos.Id,
				"error", err.Error(),
			)
		} else {
			return err
		}
	}

	fullSlash := sharesToUnbond.GTE(pos.Delegation.Shares)

	if fullSlash {
		pos.Delegation = nil
		pos.ClearBonusCheckpoints()
		return k.setPositionWithState(ctx, pos, &ValidatorTransition{PreviousAddress: dstValStr})
	}
	// In-memory only: the persisted Position carries no share count, and the
	// live delegation will reflect the post-Unbond shares on the next read.
	// Update the local copy so any follow-up logic in this call sees consistent state.
	pos.Delegation.Shares = pos.Delegation.Shares.Sub(sharesToUnbond)
	return k.setPositionWithState(ctx, pos, nil)
```

func applyBonusAccrualCheckpoint(pos *types.Position, blockTime time.Time) {
	accrualEnd := blockTime
	if pos.CompletedExitLockDuration(blockTime) {
		accrualEnd = pos.ExitUnlockAt
	}
	pos.UpdateLastBonusAccrual(accrualEnd)
}

func (k Keeper) computeSegmentBonus(pos types.PositionState, tier types.Tier, segmentStart, segmentEnd time.Time, tokensPerShare math.LegacyDec) math.Int {
	if !pos.ExitUnlockAt.IsZero() && segmentEnd.After(pos.ExitUnlockAt) {
		segmentEnd = pos.ExitUnlockAt
	}

	if !segmentEnd.After(segmentStart) {
		return math.ZeroInt()
	}

	durationSeconds := int64(segmentEnd.Sub(segmentStart) / time.Second)
	if durationSeconds <= 0 {
		return math.ZeroInt()
	}

	tokens := pos.Delegation.Shares.Mul(tokensPerShare)

	return tokens.
		Mul(tier.BonusApy).
		MulInt64(durationSeconds).
		QuoInt64(types.SecondsPerYear).
		TruncateInt()
}
