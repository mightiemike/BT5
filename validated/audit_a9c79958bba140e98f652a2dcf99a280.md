Audit Report

## Title
`SwapAllowlistExtension.beforeSwap` receives router address as `sender` instead of actual user, enabling allowlist bypass — (`metric-periphery/contracts/extensions/SwapAllowlistExtension.sol`)

## Summary
When a user swaps through `MetricOmmSimpleRouter`, the pool's `swap()` function passes `msg.sender` (the router contract address) as the `sender` argument to all configured extensions. `SwapAllowlistExtension.beforeSwap` uses this `sender` to enforce per-user access control, but it always receives the router address. If the router is allowlisted for a pool, every user — including those not individually allowlisted — can bypass the restriction by routing through the router contract.

## Finding Description

**Call chain:**

1. User calls `MetricOmmSimpleRouter.exactInputSingle()` (or `exactInput`, `exactOutputSingle`, `exactOutput`).
2. The router calls `IMetricOmmPoolActions(pool).swap(recipient, zeroForOne, amount, priceLimit, "", extensionData)` — `msg.sender` inside the pool is the **router address**.
3. `MetricOmmPool.swap()` calls `_beforeSwap(msg.sender, ...)` — passing the **router address** as `sender`.
4. `ExtensionCalling._beforeSwap` encodes `sender` (router address) and calls `CallExtension.callExtension(extension, data)` via a plain `.call()`.
5. Inside `SwapAllowlistExtension.beforeSwap`, `msg.sender` is the pool and `sender` is the **router address**:

```solidity
// SwapAllowlistExtension.sol line 37
if (!allowAllSwappers[msg.sender] && !allowedSwapper[msg.sender][sender]) {
    revert IMetricOmmPoolActions.NotAllowedToSwap();
}
```

The check `allowedSwapper[pool][sender]` evaluates to `allowedSwapper[pool][router]`, not `allowedSwapper[pool][actualUser]`.

**Exploit path:**
- Pool admin deploys a pool with `SwapAllowlistExtension` to restrict swaps to KYC'd/approved addresses.
- Pool admin allowlists the router address so that approved users can swap via the router.
- Any unapproved user calls `MetricOmmSimpleRouter.exactInputSingle()` targeting the restricted pool.
- The extension sees `sender = router`, which is allowlisted → check passes → swap executes.
- The unapproved user successfully swaps despite not being on the allowlist.

**Existing guards are insufficient:** The `onlyPool` modifier in `BaseMetricExtension` only verifies that the caller is a registered pool — it does not verify the identity of the actual end user. There is no mechanism in the pool or router to forward the real `msg.sender` (the end user) to extensions.

## Impact Explanation
The `SwapAllowlistExtension` access control is completely bypassed for any user who routes through `MetricOmmSimpleRouter`. A pool configured to restrict swaps to a specific set of addresses (e.g., institutional traders, KYC'd users, or whitelisted counterparties) can be freely accessed by any unprivileged address. This breaks the admin-boundary invariant: the pool admin's configured access restriction is circumvented by an unprivileged path (the public router). Severity: **Medium** — broken core extension functionality; the allowlist provides no protection when the router is allowlisted.

## Likelihood Explanation
Any unprivileged user can trigger this by calling the public router. The only precondition is that the pool admin has allowlisted the router address (a natural and expected configuration for any pool that intends to support router-based trading). The attack is trivially repeatable with zero special privileges.

## Recommendation
Pass the actual end user's address through the swap path. One approach: add a `payer` or `originator` field to the extension data that the router populates with `msg.sender` before calling the pool, and have `SwapAllowlistExtension` decode and verify it. Alternatively, the pool could accept an explicit `originator` parameter in `swap()` and pass it as `sender` to extensions instead of `msg.sender`. The router would then set `originator = msg.sender` (the actual user).

## Proof of Concept

```solidity
// 1. Deploy pool with SwapAllowlistExtension
// 2. Pool admin allowlists the router: allowedSwapper[pool][router] = true
// 3. Attacker (not individually allowlisted) calls:
router.exactInputSingle(ExactInputSingleParams({
    pool: restrictedPool,
    recipient: attacker,
    tokenIn: token0,
    zeroForOne: true,
    amountIn: 1e18,
    amountOutMinimum: 0,
    priceLimitX64: 0,
    deadline: block.timestamp,
    extensionData: ""
}));
// SwapAllowlistExtension.beforeSwap receives sender=router (allowlisted) → passes
// Attacker swaps successfully despite not being on the per-user allowlist
```
