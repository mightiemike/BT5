### Title
Trailing-Slash `ClassId` Bypasses Validation, Minting Unbacked NFT Vouchers with Empty `BaseClassId` — (`x/nft-transfer/types/trace.go`, `x/nft-transfer/keeper/packet.go`)

---

### Summary

A malicious IBC counterparty can send a packet whose `ClassId` ends with a trailing slash (e.g., `"port/channel/"`). The receiving chain's `processReceivedPacket` — when in the `isAwayFromOrigin=true` branch — prepends the destination prefix, producing `"destPort/destChannel/port/channel/"`. `ParseClassTrace` splits on `/` and assigns the last (empty) segment as `BaseClassId`, yielding a `ClassTrace{Path: "destPort/destChannel/port/channel", BaseClassId: ""}`. This invalid trace is stored and used to mint NFT vouchers without ever calling `ClassTrace.Validate()`, which would have rejected it with "base class_id cannot be blank."

---

### Finding Description

**Step 1 — Entry point passes `ValidateBasic()`.**

`OnRecvPacket` calls `data.ValidateBasic()` first:

```go
if strings.TrimSpace(nftpd.ClassId) == "" {
    return newsdkerrors.Wrap(ErrInvalidClassID, "classId cannot be blank")
}
```

`"port/channel/"` is non-blank after `TrimSpace`, so this guard passes. [1](#0-0) 

**Step 2 — `isAwayFromOrigin=true` branch prefixes the ClassId.**

```go
classPrefix := types.GetClassPrefix(packet.GetDestPort(), packet.GetDestChannel())
// = "destPort/destChannel/"
prefixedClassID := classPrefix + data.ClassId
// = "destPort/destChannel/port/channel/"
classTrace := types.ParseClassTrace(prefixedClassID)
``` [2](#0-1) 

**Step 3 — `ParseClassTrace` produces `BaseClassId = ""`.**

```go
classSplit := strings.Split("destPort/destChannel/port/channel/", "/")
// → ["destPort", "destChannel", "port", "channel", ""]
return ClassTrace{
    Path:        "destPort/destChannel/port/channel",
    BaseClassId: "",   // ← last element of split
}
``` [3](#0-2) 

**Step 4 — `Validate()` is never called; the invalid trace is stored.**

```go
if !k.HasClassTrace(ctx, classTrace.Hash()) {
    k.SetClassTrace(ctx, classTrace)   // no Validate() call
}
```

`ClassTrace.Validate()` explicitly rejects an empty `BaseClassId`:

```go
case strings.TrimSpace(ct.BaseClassId) == "":
    return fmt.Errorf("base class_id cannot be blank")
``` [4](#0-3) [5](#0-4) 

**Step 5 — `IBCClassID()` returns a well-formed `ibc/{hash}` despite empty base.**

Because `Path != ""`, `IBCClassID()` returns `"ibc/{hash}"` — a syntactically valid classID — hiding the corruption from downstream callers. [6](#0-5) 

**Step 6 — `IssueDenom` and `MintNFT` execute against the corrupted classID.**

```go
k.nftKeeper.IssueDenom(ctx, voucherClassID, ...)
k.nftKeeper.MintNFT(ctx, voucherClassID, tokenID, ...)
```

NFTs are minted under `ibc/{hash}` whose stored trace resolves to an empty base class — no corresponding NFT was ever escrowed on the sending chain. [7](#0-6) 

---

### Impact Explanation

The invariant that every minted ICS-721 voucher classID must correspond to a valid, non-empty base NFT class on the origin chain is violated. The attacker mints NFT vouchers on the receiving chain without escrowing anything on the sending chain. The stored `ClassTrace` with `BaseClassId=""` also corrupts the trace store: `GetFullClassPath()` returns `"destPort/destChannel/port/channel/"` (trailing slash), which would propagate the malformed path if the NFT is forwarded further. Genesis `Validate()` would reject this state on export/import, potentially causing a chain halt on upgrade. [8](#0-7) [9](#0-8) 

---

### Likelihood Explanation

Requires a malicious IBC counterparty chain — an entity that has established a valid IBC channel with the victim chain and deliberately crafts a packet with a trailing-slash `ClassId`. This is within the IBC threat model: the receiving chain is responsible for rejecting malformed packets. No governance, key compromise, or social engineering is needed beyond controlling the sending chain.

---

### Recommendation

1. **Call `classTrace.Validate()` in `processReceivedPacket`** immediately after `ParseClassTrace`, before `SetClassTrace` or `IssueDenom`:

```go
classTrace := types.ParseClassTrace(prefixedClassID)
if err := classTrace.Validate(); err != nil {
    return sdkerrors.Wrapf(types.ErrInvalidClassID, "invalid class trace: %s", err)
}
```

2. **Add a trailing-slash check in `NonFungibleTokenPacketData.ValidateBasic()`**:

```go
if strings.HasSuffix(nftpd.ClassId, "/") {
    return newsdkerrors.Wrap(ErrInvalidClassID, "classId cannot have a trailing slash")
}
``` [10](#0-9) [11](#0-10)

### Proof of Concept

```go
// Unit test — no chain needed
func TestTrailingSlashMint(t *testing.T) {
    // Attacker sends ClassId = "port/channel/"
    // Receiver prepends dest prefix
    prefixed := "destPort/destChannel/" + "port/channel/"
    // = "destPort/destChannel/port/channel/"

    ct := types.ParseClassTrace(prefixed)
    require.Equal(t, "", ct.BaseClassId)          // empty base — BUG
    require.Equal(t, "destPort/destChannel/port/channel", ct.Path)

    err := ct.Validate()
    require.Error(t, err)                          // "base class_id cannot be blank"

    ibcID := ct.IBCClassID()
    require.True(t, strings.HasPrefix(ibcID, "ibc/")) // still returns valid-looking ibc/hash

    // In processReceivedPacket, Validate() is never called,
    // so IssueDenom(ibcID) and MintNFT(ibcID, tokenID) execute
    // against a trace with empty BaseClassId — unbacked voucher minted.
}

```go
func ParseClassTrace(rawClassID string) ClassTrace {
	classSplit := strings.Split(rawClassID, "/")

	if classSplit[0] == rawClassID {
		return ClassTrace{
			Path:        "",
			BaseClassId: rawClassID,
		}
	}

	return ClassTrace{
		Path:        strings.Join(classSplit[:len(classSplit)-1], "/"),
		BaseClassId: classSplit[len(classSplit)-1],
	}
}
