Audit Report

## Title
Synthetic-ratio integer truncation in `_getBidAndAskPrice` silently zeroes `mid`, permanently reverting all swaps on low-ratio synthetic pools — (File: `smart-contracts-poc/contracts/AnchoredPriceProvider.sol`)

## Summary

In two-feed (synthetic) mode, `AnchoredPriceProvider._getBidAndAskPrice` computes the ratio `mid = Math.mulDiv(mid, ORACLE_DECIMALS, mid2)` using only 8-decimal precision. When `mid1 × 1e8 < mid2`, integer division truncates `mid` to zero. Both `_readLeg` calls can return `ok = true` (feeds are fresh, non-zero, within guard bounds), so no per-leg guard catches the truncation. The zero `mid` propagates through `_computeBidAsk`, which returns the stalled sentinel `(0, type(uint128).max)`, causing `getBidAndAskPrice` to revert with `FeedStalled` and blocking every swap on the pool.

## Finding Description

**Root cause — line 267:** [1](#0-0) 

`ORACLE_DECIMALS = 1e8` (line 45). `Math.mulDiv(mid1, 1e8, mid2)` performs exact integer division; when `mid1 × 1e8 < mid2`, the result is 0.

**Why per-leg guards are insufficient:** [2](#0-1) 

`_readLeg` validates staleness, non-zero `mid`, `spreadBps < ORACLE_BPS`, and the per-feed `priceGuard` range — none of which involve the ratio between the two feeds. Both legs pass with `ok = true`.

**Zero `mid` propagation through `_computeBidAsk`:** [3](#0-2) 

`_bandEdge(0, …)` returns 0 for both `refBid` and `refAsk`. The guard `refBid == 0` fires immediately, returning `(0, type(uint128).max)`.

**`getBidAndAskPrice` reverts:** [4](#0-3) 

**Every swap on the pool reverts:** [5](#0-4) 

`_getBidAndAskPriceX64` wraps the call in a `try/catch` that re-reverts with `PriceProviderFailed`, blocking all swaps.

**Concrete arithmetic:**
- PEPE/USD: `mid1 = 1137` ($0.00001137 × 1e8)
- BTC/USD: `mid2 = 12_000_000_000_000` ($120,000 × 1e8)
- `Math.mulDiv(1137, 1e8, 12_000_000_000_000)` = `113_700_000_000 / 12_000_000_000_000` = **0**

## Impact Explanation

All swaps on any pool whose `AnchoredPriceProvider` is deployed in synthetic two-feed mode with a price ratio below `1e-8` are permanently blocked for the duration of the price condition. The pool's primary function (swapping) is completely unusable. This matches the allowed impact: **broken core pool functionality causing unusable swap flows**.

## Likelihood Explanation

Synthetic pairs with extreme price ratios (meme-coin/BTC, meme-coin/ETH) are realistic production configurations. No privileged action is required — ordinary market prices trigger the condition. The per-leg staleness and guard checks pass normally; the truncation is invisible to them. Once triggered, the DoS persists until the ratio recovers above `1e-8`. The condition is repeatable and requires no attacker — it is triggered by normal oracle price updates.

## Recommendation

Replace the 8-decimal intermediate with an 18-decimal scale to preserve precision across the full realistic price range:

```solidity
// Before (line 267):
mid = Math.mulDiv(mid, ORACLE_DECIMALS, mid2);   // 1e8 — truncates to 0

// After:
mid = Math.mulDiv(mid, 1e18, mid2);              // 1e18 — survives ratios down to ~1e-18
```

All downstream consumers of `mid` in `_computeBidAsk` and `_bandEdge` that reference `STEP_DENOM = ORACLE_DECIMALS * BPS_BASE_U` must be updated to match the new scale. As a minimal defensive patch, add an explicit zero-check immediately after the ratio computation:

```solidity
mid = Math.mulDiv(mid, ORACLE_DECIMALS, mid2);
if (mid == 0) return (0, type(uint128).max); // explicit fail-closed
```

## Proof of Concept

1. Deploy `AnchoredPriceProvider` with `baseFeedId = PEPE/USD`, `quoteFeedId = BTC/USD`.
2. Configure the mock oracle: PEPE/USD returns `mid = 1137`, `spreadBps = 1`, valid `refTime`; BTC/USD returns `mid = 12_000_000_000_000`, `spreadBps = 1`, valid `refTime`. Both pass all `_readLeg` guards.
3. Call `getBidAndAskPrice()`.
4. Observe: `Math.mulDiv(1137, 1e8, 12_000_000_000_000) = 0`; `_computeBidAsk(0, 2)` hits `refBid == 0` guard and returns `(0, type(uint128).max)`; `getBidAndAskPrice` reverts with `FeedStalled`.
5. Any `pool.swap(...)` using this provider also reverts via `PriceProviderFailed`, permanently blocking all swaps on the pool.

        internal returns (uint256 mid, uint256 spreadBps, uint256 refTime, bool ok)
    {
        (mid, spreadBps, , refTime) = IPricedOracle(address(offchainOracle)).price(feedId, msg.sender);

        // Stale reference → not ok. Clamping to a stale anchor is the one false-safety case.
        if (_isStale(refTime, block.timestamp, MAX_REF_STALENESS)) return (mid, spreadBps, refTime, false);

        // Basic validity — mid positive, spreadBps not the stalled/off-hours marker (the Chainlink oracle
        // writes spreadBps = ORACLE_BPS when an RWA market is closed).
        if (mid == 0 || spreadBps >= ORACLE_BPS) return (mid, spreadBps, refTime, false);

        // Per-leg price guard.
        (uint128 guardMin, uint128 guardMax) = offchainOracle.priceGuard(feedId);
        guardMax = guardMax == 0 ? type(uint128).max : guardMax;
        if (mid < guardMin || mid > guardMax) return (mid, spreadBps, refTime, false);

        ok = true;
    }
