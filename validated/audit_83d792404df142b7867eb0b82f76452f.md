### Title
Missing chain ID in precommit vote signature domain enables cross-chain replay of consensus votes — (`crates/apollo_signature_manager/src/signature_manager.rs`)

### Summary

`build_precommit_vote_message_digest` constructs the signed payload as `blake2s(b"PRECOMMIT_VOTE" || block_hash)` with no chain identifier. The Starknet block hash itself also omits chain ID from its preimage. Together, a precommit vote signature produced on one Starknet deployment is cryptographically valid on any other deployment that reaches the same block hash, enabling cross-chain replay of consensus votes once signature verification is wired into the consensus layer.

### Finding Description

`build_precommit_vote_message_digest` in `crates/apollo_signature_manager/src/signature_manager.rs` builds the message digest as:

```rust
fn build_precommit_vote_message_digest(block_hash: BlockHash) -> MessageDigest {
    let block_hash = block_hash.to_bytes_be();
    let mut message = Vec::with_capacity(PRECOMMIT_VOTE.len() + block_hash.len());
    message.extend_from_slice(PRECOMMIT_VOTE);   // b"PRECOMMIT_VOTE"
    message.extend_from_slice(&block_hash);
    MessageDigest(blake2s_to_felt(&message))
}
``` [1](#0-0) 

The domain separator is the static string `b"PRECOMMIT_VOTE"` and the only variable input is the block hash. No chain ID is mixed in.

`calculate_block_hash` in `crates/starknet_api/src/block_hash/block_hash_calculator.rs` hashes: `block_hash_version`, `block_number`, `state_root`, `sequencer_address`, `block_timestamp`, `concatenated_counts`, `state_diff_commitment`, `transaction_commitment`, `event_commitment`, `receipt_commitment`, `gas_prices`, `starknet_version`, `0`, `previous_block_hash`. [2](#0-1) 

**Chain ID is absent from the block hash preimage.** This is confirmed by the Cairo OS implementation as well: [3](#0-2) 

Because neither the block hash nor the signature domain binds to a chain identifier, the ECDSA signature `sign(blake2s(b"PRECOMMIT_VOTE" || H))` is identical for any two Starknet networks that happen to produce a block with hash `H`.

The signing path is already live: [4](#0-3) 

Verification is defined and tested: [5](#0-4) 

The consensus layer currently carries a `// TODO(Asmaa): verify the signature` placeholder, meaning the vulnerability becomes reachable the moment that TODO is resolved: [6](#0-5) 

By contrast, every Starknet *transaction* hash correctly binds to chain ID (e.g., `Felt::try_from(chain_id)?` in every `get_invoke_transaction_v3_hash`, `get_declare_transaction_v3_hash`, etc.): [7](#0-6) 

The consensus vote signature is the only signing domain in the sequencer that omits chain ID.

### Impact Explanation

Matches **High — signature/hash logic binds the wrong hash/signer**. A validator's precommit vote for block hash `H` on network A is a valid precommit vote for block hash `H` on network B. An attacker who observes a quorum of precommit signatures on network A and can present the same block hash on network B (possible when both networks share genesis state or are forks) can replay those signatures to falsely satisfy the precommit quorum on network B, causing the consensus layer to accept a block it should not finalize.

### Likelihood Explanation

Low-to-medium. The attack requires the same 32-byte block hash to appear on two distinct deployments. Because the block hash commits to state root, transaction commitment, and parent hash, accidental collision is negligible for long-running independent chains. The risk is elevated when:

1. The sequencer is deployed on multiple chains from the same genesis (the project's stated intent).
2. An adversary controls enough of the early block content on the target chain to engineer a hash collision.
3. Signature verification is activated (the current TODO is resolved).

### Recommendation

Include the chain ID in the precommit vote message digest, mirroring the pattern already used for transaction hashes:

```rust
// Message domain separators.
pub(crate) const PRECOMMIT_VOTE: &[u8] = b"PRECOMMIT_VOTE";

fn build_precommit_vote_message_digest(
    block_hash: BlockHash,
    chain_id: &ChainId,
) -> MessageDigest {
    let block_hash = block_hash.to_bytes_be();
    let chain_id_felt = Felt::try_from(chain_id).expect("chain_id as felt");
    let chain_id_bytes = chain_id_felt.to_bytes_be();
    let mut message = Vec::with_capacity(
        PRECOMMIT_VOTE.len() + block_hash.len() + chain_id_bytes.len(),
    );
    message.extend_from_slice(PRECOMMIT_VOTE);
    message.extend_from_slice(&block_hash);
    message.extend_from_slice(&chain_id_bytes);
    MessageDigest(blake2s_to_felt(&message))
}
```

`SignatureManager` should receive the `ChainId` at construction time (analogous to `TransactionConverter` which already stores `chain_id`): [8](#0-7) 

### Proof of Concept

1. Deploy the sequencer on **Chain A** (mainnet) and **Chain B** (a fork or testnet) starting from the same genesis block.
2. On Chain A, block `N` is finalized. Validators broadcast precommit votes; each vote carries `sig_i = ECDSA_sign(key_i, blake2s(b"PRECOMMIT_VOTE" || H_A_N))`.
3. An attacker collects a quorum of `(sig_i)` from Chain A's P2P layer.
4. On Chain B, the attacker crafts or waits for block `N` to reach hash `H_B_N = H_A_N` (feasible when both chains share the same block content up to block `N`, e.g., during the early life of a fork).
5. The attacker injects the replayed `(sig_i)` into Chain B's consensus layer as precommit votes for `H_B_N`.
6. Because `blake2s(b"PRECOMMIT_VOTE" || H_A_N) == blake2s(b"PRECOMMIT_VOTE" || H_B_N)`, `verify_precommit_vote_signature` returns `true` for each replayed signature.
7. Chain B's consensus layer counts a false quorum and finalizes a block it should not have accepted.

func calculate_block_hash{poseidon_ptr: PoseidonBuiltin*}(
    block_info: BlockInfo*,
    header_commitments: BlockHeaderCommitments*,
    gas_prices_hash: felt,
    state_root: felt,
    previous_block_hash: felt,
    starknet_version: felt,
) -> felt {
    static_assert BlockInfo.SIZE == 3;
    static_assert BlockHeaderCommitments.SIZE == 5;

    let hash_state = hash_init();
    with hash_state {
        hash_update_single(BLOCK_HASH_VERSION);
        hash_update_single(block_info.block_number);
        hash_update_single(state_root);
        hash_update_single(block_info.sequencer_address);
        hash_update_single(block_info.block_timestamp);
        hash_update_single(header_commitments.packed_lengths);
        hash_update_single(header_commitments.state_diff_commitment);
        hash_update_single(header_commitments.transaction_commitment);
        hash_update_single(header_commitments.event_commitment);
        hash_update_single(header_commitments.receipt_commitment);
        hash_update_single(gas_prices_hash);
        hash_update_single(starknet_version);
        hash_update_single(0);
        hash_update_single(previous_block_hash);
    }

    let block_hash = hash_finalize(hash_state=hash_state);
    return block_hash;
}
