### Title
`gas_prices_to_hash` omits `l2_gas_consumed` and `next_l2_gas_price` from block hash for all Starknet versions ≥ 0.13.4 including current production 0.14.x blocks due to missing `BlockHashVersion` variant - (File: crates/starknet_api/src/block_hash/block_hash_calculator.rs)

### Summary

The `gas_prices_to_hash` function uses a two-variant `BlockHashVersion` enum (`V0_13_2`, `V0_13_4`) where every Starknet version ≥ 0.13.4 — including all current production 0.14.x blocks — maps to `V0_13_4`. For that variant the function hashes only six gas price fields and silently omits `l2_gas_consumed` and `next_l2_gas_price`. A TODO comment in the same function explicitly acknowledges these fields must be added "after 0.14.0", but no `V0_14_0` variant exists in `BlockHashVersion`, making the gate structurally impossible to implement. The echonet replay/verification system already passes both fields to the external block-hash CLI, indicating the protocol has already defined them as part of the canonical hash for 0.14.x blocks. The result is that the sequencer produces a structurally wrong block hash for every block at version ≥ 0.14.0.

### Finding Description

`BlockHashVersion` is the sole version discriminator for block hash computation:

```rust
pub enum BlockHashVersion {
    V0_13_2,
    V0_13_4,
}

impl TryFrom<StarknetVersion> for BlockHashVersion {
    fn try_from(value: StarknetVersion) -> StarknetApiResult<Self> {
        if value < Self::V0_13_2.into() {
            Err(...)
        } else if value < Self::V0_13_4.into() {
            Ok(Self::V0_13_2)
        } else {
            Ok(Self::V0_13_4)   // ← all 0.14.x falls here
        }
    }
}
```

`gas_prices_to_hash`, called from `calculate_block_hash`, branches on this enum:

```rust
// TODO(Ayelet): add l2_gas_consumed, next_l2_gas_price after 0.14.0.
pub fn gas_prices_to_hash(..., block_hash_version: &BlockHashVersion) -> Vec<Felt> {
    if block_hash_version >= &BlockHashVersion::V0_13_4 {
        vec![
            HashChain::new()
                .chain(&STARKNET_GAS_PRICES0)
                .chain(&l1_gas_price.price_in_wei.0.into())
                .chain(&l1_gas_price.price_in_fri.0.into())
                .chain(&l1_data_gas_price.price_in_wei.0.into())
                .chain(&l1_data_gas_price.price_in_fri.0.into())
                .chain(&l2_gas_price.price_in_wei.0.into())
                .chain(&l2_gas_price.price_in_fri.0.into())
                // l2_gas_consumed and next_l2_gas_price are ABSENT
                .get_poseidon_hash(),
        ]
    } else { ... }
}
```

`PartialBlockHashComponents`, the struct that feeds `calculate_block_hash`, carries `l1_gas_price`, `l1_data_gas_price`, and `l2_gas_price` but has **no fields** for `l2_gas_consumed` or `next_l2_gas_price`:

```rust
pub struct PartialBlockHashComponents {
    pub l1_gas_price: GasPricePerToken,
    pub l1_data_gas_price: GasPricePerToken,
    pub l2_gas_price: GasPricePerToken,
    // l2_gas_consumed and next_l2_gas_price are absent
    ...
}
```

Meanwhile `BlockHeaderWithoutHash` — the authoritative on-chain header — carries both fields:

```rust
pub struct BlockHeaderWithoutHash {
    pub l2_gas_consumed: GasAmount,
    pub next_l2_gas_price: GasPrice,
    ...
}
```

The echonet replay/verification system already passes both fields to the external block-hash CLI for every block it processes:

```python
result = self._run_block_hash_cli("block-hash", {
    "header": {
        ...
        "l2_gas_consumed": fee_market_info["l2_gas_consumed"],
        "next_l2_gas_price": fee_market_info["next_l2_gas_price"],
        ...
    },
})
```

The current `LATEST` Starknet version is `V0_14_4`. Because `BlockHashVersion` has no `V0_14_0` (or later) variant, there is no code path that can ever include `l2_gas_consumed` and `next_l2_gas_price` in the hash, regardless of the block's actual version string.

### Impact Explanation

Every block produced at Starknet version ≥ 0.14.0 receives a block hash that omits two protocol-mandated fields. This is a **Critical** wrong-state impact:

- The block hash committed to storage and propagated through consensus is canonically wrong.
- The echonet verification system (and any external verifier that follows the 0.14.x hash spec) will compute a different hash, causing verification failures.
- If the L1 contract or proof system expects the updated hash format, every block since 0.14.0 will fail on-chain verification.
- Because `PartialBlockHashComponents` is also used for `PartialBlockHash` (the pre-finalization commitment used during consensus proposal), the wrong hash propagates into the consensus layer before the block is even finalized.

### Likelihood Explanation

The current production `LATEST` version is `V0_14_4`. Every block sequenced today triggers this path. The TODO comment confirms the developers are aware the fields must be added but the structural prerequisite (a new `BlockHashVersion` variant) has not been created, so the omission is active in all current production blocks.

### Recommendation

1. Add a `V0_14_0` variant to `BlockHashVersion` and update `TryFrom<StarknetVersion>` to map versions in `[0.14.0, ...)` to it.
2. Add `l2_gas_consumed: GasAmount` and `next_l2_gas_price: GasPrice` to `PartialBlockHashComponents`.
3. Update `gas_prices_to_hash` to chain these two fields into the Poseidon hash when `block_hash_version >= V0_14_0`.
4. Update `PartialBlockHashComponents::new` to populate the new fields from `BlockHeaderWithoutHash`.
5. Remove the TODO comment once the fix is in place.

### Proof of Concept

**Divergent value**: For any block at version 0.14.x, the Rust sequencer computes:

```
Poseidon("STARKNET_GAS_PRICES0",
    l1_gas_price_wei, l1_gas_price_fri,
    l1_data_gas_price_wei, l1_data_gas_price_fri,
    l2_gas_price_wei, l2_gas_price_fri)
```

The echonet CLI (and the protocol spec for 0.14.x) expects:

```
Poseidon("STARKNET_GAS_PRICES0",
    l1_gas_price_wei, l1_gas_price_fri,
    l1_data_gas_price_wei, l1_data_gas_price_fri,
    l2_gas_price_wei, l2_gas_price_fri,
    l2_gas_consumed,          ← missing in Rust
    next_l2_gas_price)        ← missing in Rust
```

These two Poseidon hashes are different for any block where `l2_gas_consumed != 0` or `next_l2_gas_price != 0`, which is every non-empty block. The wrong hash is then fed into `calculate_block_hash` and stored as the canonical block hash.

**Trigger**: Unprivileged — any transaction submitted to the sequencer causes a block to be built at the current version (0.14.4), which unconditionally takes the broken `V0_13_4` code path. [1](#0-0) [2](#0-1) [3](#0-2) [4](#0-3) [5](#0-4) [6](#0-5)

```rust
pub enum BlockHashVersion {
    V0_13_2,

impl From<BlockHashVersion> for StarknetVersion {
    fn from(value: BlockHashVersion) -> Self {
        match value {
            BlockHashVersion::V0_13_2 => StarknetVersion::V0_13_2,
            BlockHashVersion::V0_13_4 => StarknetVersion::V0_13_4,
        }
    }
}

impl TryFrom<StarknetVersion> for BlockHashVersion {
    type Error = StarknetApiError;

    fn try_from(value: StarknetVersion) -> StarknetApiResult<Self> {
        if value < Self::V0_13_2.into() {
            Err(StarknetApiError::BlockHashVersion { version: value.to_string() })
        } else if value < Self::V0_13_4.into() {
            // Starknet versions 0.13.2 and 0.13.3 both have the same block hash mechanism.
            Ok(Self::V0_13_2)
        } else {
            Ok(Self::V0_13_4)
        }
    }
```

```rust
pub fn calculate_block_hash(
    partial_block_hash_components: &PartialBlockHashComponents,
    state_root: GlobalRoot,
    previous_block_hash: BlockHash,
) -> StarknetApiResult<BlockHash> {
    let block_hash_version: BlockHashVersion =
        partial_block_hash_components.starknet_version.try_into()?;
    let block_commitments = &partial_block_hash_components.header_commitments;
    Ok(BlockHash(
        HashChain::new()
            .chain(&block_hash_version.clone().into())
            .chain(&partial_block_hash_components.block_number.0.into())
            .chain(&state_root.0)
            .chain(&partial_block_hash_components.sequencer.0)
            .chain(&partial_block_hash_components.timestamp.0.into())
            .chain(&block_commitments.concatenated_counts)
            .chain(&block_commitments.state_diff_commitment.0.0)
            .chain(&block_commitments.transaction_commitment.0)
            .chain(&block_commitments.event_commitment.0)
            .chain(&block_commitments.receipt_commitment.0)
            .chain_iter(
                gas_prices_to_hash(
                    &partial_block_hash_components.l1_gas_price,
                    &partial_block_hash_components.l1_data_gas_price,
                    &partial_block_hash_components.l2_gas_price,
                    &block_hash_version,
                )
                .iter(),
            )
            .chain(
                &Felt::try_from(&partial_block_hash_components.starknet_version)
                    .expect("Expect ASCII version"),
            )
            .chain(&Felt::ZERO)
            .chain(&previous_block_hash.0)
            .get_poseidon_hash(),
    ))
}
```
