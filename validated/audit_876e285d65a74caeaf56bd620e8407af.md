### Title
Protobuf `ValidResourceBounds` Conversion Silently Demotes `AllResources` to `L1Gas`, Producing a Divergent Transaction Hash — (`crates/apollo_protobuf/src/converters/transaction.rs`)

### Summary
The P2P sync protobuf converter for `ValidResourceBounds` silently collapses an `AllResources` variant (with zero `l2_gas` and `l1_data_gas`) into the legacy `L1Gas` variant. Because `get_tip_resource_bounds_hash` hashes a different number of elements for each variant, a V3 invoke transaction whose hash was computed at the gateway with `AllResources` (4-element Poseidon input) will produce a different hash when re-derived from the synced `InvokeTransactionV3` object (3-element Poseidon input). This is a direct hash-domain boundary break: the same on-chain transaction yields two distinct `TransactionHash` values depending on which representation is used.

### Finding Description

**Root cause — protobuf converter (`transaction.rs`):** [1](#0-0) 

```rust
Ok(if l1_data_gas.is_zero() && l2_gas.is_zero() {
    ValidResourceBounds::L1Gas(l1_gas)          // ← silently drops l1_data_gas
} else {
    ValidResourceBounds::AllResources(AllResourceBounds { l1_gas, l2_gas, l1_data_gas })
})
```

When a committed V3 invoke transaction with `AllResourceBounds { l1_gas: X, l2_gas: 0, l1_data_gas: 0 }` is received over P2P sync, the deserialized `InvokeTransactionV3` carries `ValidResourceBounds::L1Gas(X)` instead of `ValidResourceBounds::AllResources(...)`.

**Hash divergence — `get_tip_resource_bounds_hash`:** [2](#0-1) 

```
L1Gas variant      → Poseidon([tip, l1_gas, l2_gas=0])           // 3 elements
AllResources(0,0)  → Poseidon([tip, l1_gas, l2_gas=0, l1_data=0]) // 4 elements
```

These are distinct Poseidon digests.

**Gateway path always uses `AllResources`:**

`InternalRpcInvokeTransactionV3::InvokeTransactionV3Trait::resource_bounds()` unconditionally wraps the stored `AllResourceBounds` in `ValidResourceBounds::AllResources(...)`: [3](#0-2) 

So the hash stored in `InternalRpcTransaction.tx_hash` (computed at the gateway via `convert_rpc_tx_to_internal`) always uses the 4-element Poseidon path.

impl TryFrom<protobuf::ResourceBounds> for ValidResourceBounds {
    type Error = ProtobufConversionError;
    fn try_from(value: protobuf::ResourceBounds) -> Result<Self, Self::Error> {
        let Some(l1_gas) = value.l1_gas else {
            return Err(missing("ResourceBounds::l1_gas"));
        };
        let Some(l2_gas) = value.l2_gas else {
            return Err(missing("ResourceBounds::l2_gas"));
        };
        // TODO(Shahak): Assert data gas is not none once we remove support for 0.13.2.
        let l1_data_gas = value.l1_data_gas.unwrap_or_default();
        let l1_gas: ResourceBounds = l1_gas.try_into()?;
        let l2_gas: ResourceBounds = l2_gas.try_into()?;
        let l1_data_gas: ResourceBounds = l1_data_gas.try_into()?;

            ValidResourceBounds::L1Gas(l1_gas)
        } else {
            ValidResourceBounds::AllResources(AllResourceBounds { l1_gas, l2_gas, l1_data_gas })

pub fn get_tip_resource_bounds_hash(
    resource_bounds: &ValidResourceBounds,
    tip: &Tip,
) -> Result<Felt, StarknetApiError> {
    let l1_resource_bounds = resource_bounds.get_l1_bounds();
    let l2_resource_bounds = resource_bounds.get_l2_bounds();

    // L1 and L2 gas bounds always exist.
    // Old V3 txs always have L2 gas bounds of zero, but they exist.
    let mut resource_felts = vec![
        get_concat_resource(&l1_resource_bounds, L1_GAS)?,
        get_concat_resource(&l2_resource_bounds, L2_GAS)?,
    ];

    // For new V3 txs, need to also hash the data gas bounds.
    resource_felts.extend(match resource_bounds {
        ValidResourceBounds::L1Gas(_) => vec![],
        ValidResourceBounds::AllResources(all_resources) => {
            vec![get_concat_resource(&all_resources.l1_data_gas, L1_DATA_GAS)?]
        }
    });

    Ok(HashChain::new().chain(&tip.0.into()).chain_iter(resource_felts.iter()).get_poseidon_hash())
}
