### Title
Duplicate Position IDs in `ClaimTierRewards` Enable Double-Claiming of Bonus Rewards - (File: x/tieredrewards/keeper/msg_server.go)

### Summary
`ClaimTierRewards` loads all requested positions into memory before processing any of them. If a caller includes the same position ID more than once in `msg.PositionIds`, the stale in-memory copy is processed a second time, causing `processEventsAndClaimBonus` to re-pay the same bonus rewards and `decrementEventRefCount` to fire twice for the same validator events.

### Finding Description

`ClaimTierRewards` first collects all positions into a slice, then hands the slice to `claimRewardsAndUpdatesPositions`:

```go
// msg_server.go
positions := make([]types.PositionState, 0, len(msg.PositionIds))
for _, posId := range msg.PositionIds {
    pos, err := ms.getPositionState(ctx, posId)   // loaded from store
    ...
    positions = append(positions, pos)
}
totalBase, totalBonus, err := ms.claimRewardsAndUpdatesPositions(ctx, positions)
``` [1](#0-0) 

`claimRewardsAndUpdatesPositions` iterates the slice and, for each entry, calls `processEventsAndClaimBonus` and then `setPosition` to persist the updated checkpoint:

```go
// claim_rewards.go
for i := range positions {
    pos := &positions[i]
    ...
    bonus, err := k.processEventsAndClaimBonus(ctx, pos)
    ...
    if err := k.setPosition(ctx, pos.Position, nil); err != nil { ... }
}
``` [2](#0-1) 

`processEventsAndClaimBonus` computes bonus from `pos.LastEventSeq` / `pos.LastBonusAccrual`, pays out via `bankKeeper.SendCoinsFromModuleToAccount`, and decrements each event's reference count: [3](#0-2) 

**The race between memory and store:**

When `PositionIds = [N, N]`, both entries in `positions` are loaded from the store *before* any processing begins. They are identical stale snapshots. Iteration 1 processes the first copy, pays bonus, advances `LastEventSeq`/`LastBonusAccrual`, and writes the updated position back to the store. Iteration 2 then processes the *second, stale* copy â€” which still carries the old `LastEventSeq` â€” re-walks the same validator events, pays the same bonus a second time, and overwrites the store with this stale-updated state. The final on-chain checkpoint is identical to what iteration 1 wrote, but the `RewardsPoolName` module account has been debited twice.

Additionally, `decrementEventRefCount` is called twice for every event in the replayed window, potentially triggering premature garbage-collection of validator events that other positions still reference. [4](#0-3) 

### Impact Explanation

**Impact: High.** The `RewardsPoolName` module account is drained by double the legitimate bonus amount per duplicated position. Repeated exploitation across many positions can exhaust the bonus pool, leaving honest users unable to claim their earned bonus rewards. The corrupted value is the `RewardsPoolName` balance and the validator-event reference counts. [5](#0-4) 

### Likelihood Explanation

**Likelihood: Medium.** Any position owner can craft a `MsgClaimTierRewards` transaction with a repeated position ID. No special privilege, leaked key, or social engineering is required. The entry path is a standard signed transaction available to every user of the module. [6](#0-5) 

### Recommendation

Add a deduplication check in `ClaimTierRewards` before the positions are loaded, or alternatively add a uniqueness assertion in `MsgClaimTierRewards.Validate()`:

```go
seen := make(map[uint64]struct{}, len(msg.PositionIds))
for _, id := range msg.PositionIds {
    if _, dup := seen[id]; dup {
        return nil, errorsmod.Wrapf(sdkerrors.ErrInvalidRequest,
            "duplicate position id %d", id)
    }
    seen[id] = struct{}{}
}
```

This mirrors the fix recommended in the external report: add a guard that prevents the same accounting operation from being applied more than once per call.

### Proof of Concept

1. Alice owns position `42` with accrued bonus rewards of 100 CRO.
2. Alice submits `MsgClaimTierRewards{Owner: alice, PositionIds: [42, 42]}`.
3. Both entries are loaded from the store as identical stale snapshots before any processing.
4. Iteration 1: `processEventsAndClaimBonus` walks events since `LastEventSeq=S`, computes 100 CRO, sends 100 CRO from `RewardsPoolName` to Alice, advances checkpoint to `S'`, saves position.
5. Iteration 2: processes the stale copy (still `LastEventSeq=S`), re-walks the same events, computes 100 CRO again, sends another 100 CRO from `RewardsPoolName` to Alice, saves position (overwriting with the same checkpoint `S'`).
6. Alice receives 200 CRO; the bonus pool is debited 200 CRO; the on-chain position state looks identical to a legitimate single claim. [7](#0-6) [6](#0-5)

func (ms msgServer) ClaimTierRewards(ctx context.Context, msg *types.MsgClaimTierRewards) (*types.MsgClaimTierRewardsResponse, error) {
	if err := msg.Validate(); err != nil {
		return nil, err
	}

	positions := make([]types.PositionState, 0, len(msg.PositionIds))
	for _, posId := range msg.PositionIds {
		pos, err := ms.getPositionState(ctx, posId)
		if err != nil {
			return nil, err
		}

		if err := ms.validateClaimRewards(pos.Position, msg.Owner); err != nil {
			return nil, err
		}

		positions = append(positions, pos)
	}

	totalBase, totalBonus, err := ms.claimRewardsAndUpdatesPositions(ctx, positions)
	if err != nil {
		return nil, err
	}

	sdkCtx := sdk.UnwrapSDKContext(ctx)
	if err := sdkCtx.EventManager().EmitTypedEvent(&types.EventTierRewardsClaimed{
		Owner:        msg.Owner,
		PositionIds:  msg.PositionIds,
		BaseRewards:  totalBase,
		BonusRewards: totalBonus,
	}); err != nil {
		return nil, err
	}

	return &types.MsgClaimTierRewardsResponse{
		BaseRewards:  totalBase,
		BonusRewards: totalBonus,
		PositionIds:  msg.PositionIds,
	}, nil
}

func (k Keeper) claimRewardsAndUpdatesPositions(ctx context.Context, positions []types.PositionState) (sdk.Coins, sdk.Coins, error) {
	totalBase := sdk.NewCoins()
	totalBonus := sdk.NewCoins()

	for i := range positions {
		pos := &positions[i]

		if !pos.IsDelegated() {
			continue
		}

		base, err := k.claimBaseRewards(ctx, *pos)
		if err != nil {
			return nil, nil, err
		}
		totalBase = totalBase.Add(base...)

		bonus, err := k.processEventsAndClaimBonus(ctx, pos)
		if err != nil {
			return nil, nil, err
		}
		totalBonus = totalBonus.Add(bonus...)

		if err := k.setPosition(ctx, pos.Position, nil); err != nil {
			return nil, nil, err
		}
	}

	return totalBase, totalBonus, nil
}
