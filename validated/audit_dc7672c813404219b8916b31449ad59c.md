### Title
NFTs Permanently Locked in IBC Escrow Address on Channel Close — (`x/nft-transfer/ibc_module.go`)

### Summary
The `OnChanCloseConfirm` callback in the ICS-721 NFT transfer module returns `nil` without unescrow-ing any NFTs held in the channel's escrow address. If the counterparty chain closes the channel, all NFTs that were escrowed on this chain for that channel are permanently locked with no recovery path.

### Finding Description
When a user sends an NFT cross-chain via ICS-721, `createOutgoingPacket` transfers ownership of the NFT to the deterministic escrow address `types.GetEscrowAddress(sourcePort, sourceChannel)`: [1](#0-0) 

The only paths that unescrow NFTs back to their original owners are `refundPacketToken`, which is called exclusively on packet timeout or error acknowledgement: [2](#0-1) [3](#0-2) 

However, `OnChanCloseConfirm` — the callback invoked when the counterparty chain closes the channel — performs no unescrow operation at all: [4](#0-3) 

While `OnChanCloseInit` correctly blocks user-initiated channel closure: [5](#0-4) 

...this protection is one-sided. The counterparty chain can still initiate closure on its side, which triggers `OnChanCloseConfirm` on this chain. At that point, all NFTs sitting in the escrow address for that `(portID, channelID)` pair are permanently inaccessible — there is no governance proposal, admin function, or user-callable message that can recover them.

### Impact Explanation
Any NFT that was escrowed on this chain (i.e., the chain was acting as the source/away-from-origin chain) for a channel that subsequently closes is permanently locked. The escrow address is a module-derived address with no private key and no `MsgServer` endpoint to release its holdings outside of the normal IBC packet lifecycle. The corrupted value is the NFT ownership record: `nftKeeper` records the escrow address as owner, and no code path ever reassigns ownership back to the original sender after a channel close.

### Likelihood Explanation
IBC channels on UNORDERED ports (which nft-transfer requires) can be closed by a counterparty chain governance proposal or by a buggy/malicious counterparty implementation. This is a realistic production scenario: any chain that has an open ICS-721 channel with this chain can trigger the closure, locking all escrowed NFTs on this side. No special privilege on *this* chain is required — the trigger is entirely external.

### Recommendation
Implement unescrow logic inside `OnChanCloseConfirm` that iterates over all NFTs owned by `GetEscrowAddress(portID, channelID)` and transfers them back to their original owners, or alternatively stores a mapping of `(channelID, tokenID) → originalOwner` at escrow time to enable recovery. At minimum, a governance-callable message to drain a closed channel's escrow address should be added.

### Proof of Concept
1. User on chain X calls `MsgTransferNFT` for `classID="myNFT"`, `tokenID="1"` to chain Y via `(port=nft, channel=channel-0)`.
2. `createOutgoingPacket` transfers NFT ownership to `escrowAddr = GetEscrowAddress("nft", "channel-0")` on chain X.
3. Chain Y governance passes a proposal to close the channel on their side, triggering `OnChanCloseInit` on chain Y.
4. IBC relayer submits the `MsgChannelCloseConfirm` to chain X.
5. `OnChanCloseConfirm` is called on chain X — it returns `nil` immediately with no unescrow.
6. `nftKeeper` on chain X still records `escrowAddr` as the owner of `("myNFT", "1")`.
7. The original sender has no message type available to reclaim the NFT; it is permanently locked. [4](#0-3) [6](#0-5)

func (k Keeper) refundPacketToken(ctx sdk.Context, packet channeltypes.Packet, data types.NonFungibleTokenPacketData) error {
	sender, err := sdk.AccAddressFromBech32(data.Sender)
	if err != nil {
		return err
	}

	classTrace := types.ParseClassTrace(data.ClassId)
	voucherClassID := classTrace.IBCClassID()

	isAwayFromOrigin := types.IsAwayFromOrigin(packet.GetSourcePort(),
		packet.GetSourceChannel(), data.ClassId)

	escrowAddress := types.GetEscrowAddress(packet.GetSourcePort(), packet.GetSourceChannel())

	if isAwayFromOrigin {
		// unescrow tokens back to the sender
		for _, tokenID := range data.TokenIds {
			if err := k.nftKeeper.TransferOwner(ctx, voucherClassID, tokenID, escrowAddress, sender); err != nil {
				return err
			}
		}
	} else {
		// we are sink chain, mint voucher back to sender
		for i, tokenID := range data.TokenIds {
			if err := k.nftKeeper.MintNFT(ctx, voucherClassID, tokenID, "", data.TokenUris[i], "", escrowAddress, sender); err != nil {
				return err
			}
		}
	}

	return nil

func (k Keeper) OnAcknowledgementPacket(ctx sdk.Context, channelVersion string, packet channeltypes.Packet, data types.NonFungibleTokenPacketData, ack channeltypes.Acknowledgement) error {
	switch ack.Response.(type) {
	case *channeltypes.Acknowledgement_Error:
		return k.refundPacketToken(ctx, packet, data)
	default:
		// the acknowledgement succeeded on the receiving chain so nothing
		// needs to be executed and no error needs to be returned
		return nil
	}
}
