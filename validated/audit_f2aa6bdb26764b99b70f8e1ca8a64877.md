### Title
Gateway `validate_resource_bounds` Checks L2 Gas Price Threshold Against Stale Previous-Block Price Instead of `next_l2_gas_price` - (File: crates/apollo_gateway/src/stateful_transaction_validator.rs)

### Summary

`StatefulTransactionValidator::validate_resource_bounds` enforces a minimum L2 gas price threshold by comparing the transaction's `max_price_per_unit` against a fraction of the **previous committed block's** L2 gas price. The block header already carries a `next_l2_gas_price` field that encodes the price for the block being built, but the gateway never reads it. A developer TODO in the code explicitly acknowledges this gap. As a result, when the L2 gas price rises between blocks, the gateway admits transactions whose gas price is too low for the next block, and those transactions fail at execution time.

### Finding Description

`StatefulTransactionValidatorFactory::instantiate_validator` calls `get_blockifier_state_reader_and_gateway_fixed_block_from_latest_block`, which resolves the latest committed block number N and constructs a `GatewayFixedBlockSyncStateClient` pinned to that block. [1](#0-0) 

`GatewayFixedBlockSyncStateClient::get_block_info_from_sync_client` converts the block header into a `BlockInfo` struct. It copies `block_header.l2_gas_price` (the price **of** block N) into `block_info.gas_prices.strk_gas_prices.l2_gas_price`, but it never reads `block_header.next_l2_gas_price` (the price **for** block N+1, the block the transaction will actually land in). [2](#0-1) 

`validate_resource_bounds` then reads that stale price and uses it as the reference for the threshold check:

```rust
// TODO(Arni): getnext_l2_gas_price from the block header.
let previous_block_l2_gas_price = self
    .gateway_fixed_block_state_reader
    .get_block_info()
    .await?
    .gas_prices
    .strk_gas_prices
    .l2_gas_price;
self.validate_tx_l2_gas_price_within_threshold(
    executable_tx.resource_bounds(),
    previous_block_l2_gas_price,
)?;
``` [3](#0-2) 

`validate_tx_l2_gas_price_within_threshold` computes `threshold = (min_gas_price_percentage / 100) * previous_block_l2_gas_price` and rejects the transaction only if `tx_l2_gas_price < threshold`. [4](#0-3) 

The same stale `BlockInfo` is also used to build the `BlockContext` inside `run_validate_entry_point`, so the blockifier's own pre-validation fee check at the gateway level is equally stale. [5](#0-4) 

The `next_l2_gas_price` field is present on `BlockHeaderWithoutHash` and is actively used by the consensus orchestrator, confirming it is available at the point where the gateway reads the block header. [6](#0-5) 

### Impact Explanation

When the L2 gas price rises from block N to block N+1, any transaction whose `max_price_per_unit` satisfies:

```
(min_gas_price_percentage / 100) * price_N  <=  tx_price  <  price_{N+1}
```

passes the gateway's admission check (because the threshold is computed from the stale `price_N`) but will fail the blockifier's execution-time fee check (which uses the actual `price_{N+1}`). The gateway therefore admits transactions that are invalid for the block they will be sequenced into. This matches the impact category: **High — Mempool/gateway/RPC admission accepts invalid transactions before sequencing.**

### Likelihood Explanation

The L2 gas price is adjusted every block by the consensus orchestrator based on actual gas usage. Any period of sustained high throughput causes the price to rise monotonically. During such periods every transaction submitted with a gas price between the old and new floor passes gateway admission and fails execution, making this condition routinely reachable by any unprivileged user simply by submitting a transaction.

### Recommendation

In `GatewayFixedBlockSyncStateClient::get_block_info_from_sync_client`, extract `block_header.next_l2_gas_price` and surface it separately (or replace the `l2_gas_price` field in the returned `BlockInfo` with it). Then in `validate_resource_bounds`, use that `next_l2_gas_price` value as the reference price, consistent with the existing TODO comment. The same value should be used when constructing the `BlockContext` inside `run_validate_entry_point` so that the blockifier's gateway-level pre-validation also reflects the price the transaction will actually face.

### Proof of Concept

1. Observe that block N has `l2_gas_price = P` and `next_l2_gas_price = P'` where `P' > P` (price is rising).
2. Submit an invoke transaction with `resource_bounds.l2_gas.max_price_per_unit = P` (equal to the old price, below the new price).
3. `instantiate_validator` creates a `GatewayFixedBlockSyncStateClient` pinned to block N.
4. `validate_resource_bounds` computes `threshold = (min_gas_price_percentage / 100) * P`. With `min_gas_price_percentage = 100`, `threshold = P`. The transaction's price equals `P`, so `tx_price >= threshold` — the check passes.
5. The transaction is forwarded to the mempool and eventually sequenced into block N+1.
6. At execution time the blockifier uses `price_{N+1} = P'`. The transaction's `max_price_per_unit = P < P'`, so `check_fee_bounds` raises `ResourceBoundsError::MaxGasPriceTooLow` and the transaction fails.

The gateway admitted a transaction that was invalid for the block it was sequenced into, because it validated against the stale price `P` rather than the correct next-block price `P'`.

    // TODO(Arni): Consider running this validation for all gas prices.
    fn validate_tx_l2_gas_price_within_threshold(
        &self,
        tx_resource_bounds: ValidResourceBounds,
        previous_block_l2_gas_price: NonzeroGasPrice,
    ) -> StatefulTransactionValidatorResult<()> {
        match tx_resource_bounds {
            ValidResourceBounds::AllResources(tx_resource_bounds) => {
                let tx_l2_gas_price = tx_resource_bounds.l2_gas.max_price_per_unit;
                let gas_price_threshold_multiplier =
                    Ratio::new(self.config.min_gas_price_percentage.into(), 100_u128);
                let threshold = (gas_price_threshold_multiplier
                    * previous_block_l2_gas_price.get().0)
                    .to_integer();
                if tx_l2_gas_price.0 < threshold {
                    return Err(StarknetError {
                        // We didn't have this kind of an error.
                        code: StarknetErrorCode::UnknownErrorCode(
                            "StarknetErrorCode.GAS_PRICE_TOO_LOW".to_string(),
                        ),
                        message: format!(
                            "Transaction L2 gas price {tx_l2_gas_price} is below the required \
                             threshold {threshold}.",
                        ),
                    });
                }
            }
            ValidResourceBounds::L1Gas(_) => {
                // No validation required for legacy transactions.
            }
        }
        Ok(())
    }

    async fn get_block_info_from_sync_client(&self) -> StarknetResult<BlockInfo> {
        let block = self.state_sync_client.get_block(self.block_number).await.map_err(|e| {
            StarknetError::internal_with_logging("Failed to get latest block info", e)
        })?;

        let block_header = block.block_header_without_hash;
        let block_info = BlockInfo {
            block_number: block_header.block_number,
            block_timestamp: block_header.timestamp,
            sequencer_address: block_header.sequencer.0,
            gas_prices: GasPrices {
                eth_gas_prices: GasPriceVector {
                    l1_gas_price: block_header.l1_gas_price.price_in_wei.try_into()?,
                    l1_data_gas_price: block_header.l1_data_gas_price.price_in_wei.try_into()?,
                    l2_gas_price: block_header.l2_gas_price.price_in_wei.try_into()?,
                },

                    l1_gas_price: block_header.l1_gas_price.price_in_fri.try_into()?,

            starknet_version: block_header.starknet_version,
        };
