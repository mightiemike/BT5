Audit Report

## Title
SwapAllowlistExtension Checks Router Address Instead of Actual User, Enabling Full Allowlist Bypass via MetricOmmSimpleRouter - (File: metric-periphery/contracts/extensions/SwapAllowlistExtension.sol)

## Summary
`SwapAllowlistExtension.beforeSwap` gates swaps by checking the `sender` argument, which `MetricOmmPool.swap` sets to its own `msg.sender`. When a user routes through `MetricOmmSimpleRouter`, the pool's `msg.sender` is the router contract, not the end user. A pool admin who allowlists the router to support standard periphery swaps inadvertently opens the pool to every user, completely defeating the allowlist.

## Finding Description
In `MetricOmmPool.swap`, the pool passes `msg.sender` as the `sender` argument to `_beforeSwap`: [1](#0-0) 

`SwapAllowlistExtension.beforeSwap` then checks that `sender` against the allowlist: [2](#0-1) 

When `MetricOmmSimpleRouter.exactInputSingle` is called, it calls `pool.swap(params.recipient, ...)` directly — making the router the pool's `msg.sender`: [3](#0-2) 

The full call chain is:
```
User → MetricOmmSimpleRouter.exactInputSingle()
     → pool.swap(recipient, ...)          // msg.sender = router
     → _beforeSwap(msg.sender=router, ...)
     → SwapAllowlistExtension.beforeSwap(sender=router, ...)
     → allowedSwapper[pool][router]       // user identity never consulted
```

This creates an irreconcilable dilemma: allowlisting individual users blocks them from using the router (router not allowlisted → revert), while allowlisting the router opens the pool to all users. The `DepositAllowlistExtension` avoids this by checking `owner` (the economic actor) rather than `sender` (the intermediary caller): [4](#0-3) 

The `beforeSwap` interface only exposes `sender` (= pool's `msg.sender`), which collapses to the router address on any router-mediated swap, with no mechanism to recover the original user identity.

## Impact Explanation
A curated pool deploying `SwapAllowlistExtension` to restrict trading to specific addresses (e.g., KYC'd counterparties, whitelisted market makers) can have that restriction completely bypassed. Any unprivileged user routes through `MetricOmmSimpleRouter`, which the admin allowlisted to support normal periphery usage. The user receives swap output, the pool's token balances change, and the curation policy is violated — a direct loss of the pool's intended access-control invariant with fund-impacting consequences (unauthorized parties trade against the pool's liquidity). This meets the admin-boundary break criterion: the pool admin's configured access control is bypassed by an unprivileged path.

## Likelihood Explanation
`MetricOmmSimpleRouter` is the canonical periphery entry point for swaps. Any pool admin who deploys a swap-allowlisted pool and also wants to support normal user tooling will allowlist the router. The bypass requires no special knowledge or privileged access — any user calls `exactInputSingle` on the router with the target pool. The admin's mistake is structurally forced by the design.

## Recommendation
1. **Preferred fix:** Introduce a `swapper` parameter distinct from `sender` in the `beforeSwap` hook interface, where `swapper` is the original transaction initiator. The router would pass `msg.sender` (the user) as `swapper` when calling the pool, and the pool would forward it to the extension — mirroring how `DepositAllowlistExtension` uses `owner` rather than `sender`.
2. **Short-term:** Change `SwapAllowlistExtension.beforeSwap` to check `recipient` instead of `sender` if the intent is to gate who receives the output.
3. **Documentation guard (insufficient alone):** Document that allowlisting the router opens the pool to all users.

## Proof of Concept
```solidity
// Setup: pool with SwapAllowlistExtension; only allowedUser is allowlisted.
// Admin also allowlists the router to support periphery swaps.
swapExtension.setAllowedToSwap(address(pool), allowedUser, true);
swapExtension.setAllowedToSwap(address(pool), address(router), true); // admin enables router

// Attack: bannedUser (not on allowlist) swaps through the router.
vm.prank(bannedUser);
router.exactInputSingle(
    IMetricOmmSimpleRouter.ExactInputSingleParams({
        pool: address(pool),
        tokenIn: address(token0),
        tokenOut: address(token1),
        zeroForOne: true,
        amountIn: 1_000,
        amountOutMinimum: 0,
        recipient: bannedUser,
        deadline: block.timestamp + 1,
        priceLimitX64: 0,
        extensionData: ""
    })
);
// bannedUser receives token1 output. Allowlist bypassed.
// Extension checked allowedSwapper[pool][router] == true → passes.
// bannedUser's identity was never consulted.
```
