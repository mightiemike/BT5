### Title
Deprecated `ContractClass` with `abi: None` Cannot Be Received via P2P Class Sync Due to Broken Protobuf Round-Trip — (`crates/apollo_protobuf/src/converters/class.rs`)

### Summary

`From<ContractClass> for protobuf::Cairo0Class` encodes `abi: None` as the empty string `""`. `TryFrom<protobuf::Cairo0Class>` unconditionally calls `serde_json::from_str("")`, which returns `Err` because `""` is not valid JSON. The `?` operator propagates the error, making the conversion permanently fail. Any receiving node syncing via P2P is therefore unable to store any deprecated class whose ABI could not be parsed at declaration time, leaving that class permanently absent from its state.

### Finding Description

**Serialization side** — `From<ContractClass> for protobuf::Cairo0Class`: [1](#0-0) 

`abi: None` is mapped to `"".to_string()` and placed in `protobuf::Cairo0Class::abi`.

**Deserialization side** — `TryFrom<protobuf::Cairo0Class> for ContractClass`: [2](#0-1) 

`serde_json::from_str("")` always returns `Err` (empty string is not valid JSON). The `?` propagates the error as a `ProtobufConversionError`, causing `TryFrom` to return `Err` for every class that had `abi: None`.

**`abi: None` is a legitimate, on-chain state.** The `ContractClass` definition explicitly documents this: [3](#0-2) 

"Starknet does not verify the abi. If we can't parse it, we set it to None." Real deprecated classes on Starknet mainnet carry unparseable ABIs and are stored with `abi: None`.

**P2P server path** — the server serializes the class via `From<ContractClass> for protobuf::Cairo0Class` when serving class queries: [4](#0-3) 

**P2P client path** — the client calls `TryFrom<protobuf::Cairo0Class>` during `parse_data_for_block`. A `ProtobufConversionError` surfaces as a `ParseDataError`, the peer is reported as bad, and the sync retries. Because every honest peer encodes the same `abi: ""`, the retry loop never succeeds; the class is never written to the receiving node's state. [5](#0-4) 

**The existing round-trip test does not cover `abi: None`** — it uses `GetTestInstance` which always generates a non-`None` ABI: [6](#0-5) 

### Impact Explanation

A node that syncs via P2P and encounters a block that declared a deprecated class with `abi: None` will permanently fail to receive that class. The class is absent from the node's class manager. Any subsequent transaction that invokes a contract whose class hash maps to that missing class will either be rejected or executed against incorrect state. This satisfies the **Critical** impact category: *wrong compiled class / contract code selected for execution*.

### Likelihood Explanation

Deprecated classes with unparseable ABIs exist on Starknet mainnet today (the code comment confirms this is a known, handled case). Any node that bootstraps or re-syncs via P2P will hit this for every such class in the chain history. No attacker action is required; the condition is triggered by ordinary chain data.

### Recommendation

In `TryFrom<protobuf::Cairo0Class>`, treat an empty `abi` string as `None` instead of passing it to `serde_json::from_str`:

```rust
// crates/apollo_protobuf/src/converters/class.rs, line 131
let abi = if value.abi.is_empty() {
    None
} else {
    Some(serde_json::from_str(&value.abi)?)
};
```

Add a dedicated round-trip test with `abi: None` to `class_test.rs`.
