### Title
`SplitKeyDenom` Misparses Owner-Index Keys for Denom ID `"ibc"`, Permanently Corrupting the Owner Index for All NFT Holders — (`x/nft/types/keys.go`)

---

### Summary

`ValidateDenomID` accepts the string `"ibc"` as a valid denom ID (3 lowercase alphanumeric chars, begins with a letter). When an NFT is minted under that denom, `setOwner` writes a KV key of the form `PrefixOwners / <address> / ibc / <tokenID>`. When `GetOwner` or `GetOwners` later iterates those keys and calls `SplitKeyOwner`, the inner call to `SplitKeyDenom` receives the segment `"ibc/<tokenID>"`, which `bytes.HasPrefix(…, []byte("ibc/"))` matches as an IBC denom. The IBC branch then fails its length invariant (`idx < 0`), returning `errors.New("wrong KeyDenom")`. Both `GetOwner` and `GetOwners` propagate that error immediately, making the entire owner index unreadable.

---

### Finding Description

**Step 1 — `"ibc"` passes `ValidateDenomID`.**

`ValidateDenomID` only checks length `[3, 64]`, begins-with-alpha, and alphanumeric. `"ibc"` satisfies all three. [1](#0-0) 

`ValidateDenomIDWithIBC` checks `strings.HasPrefix(denomID, "ibc/")` first; `"ibc"` (no trailing slash) does **not** match, so it falls through to `ValidateDenomID` and also passes. [2](#0-1) 

**Step 2 — `IssueDenom` + `MintNFT` are standard unprivileged transactions.**

`MsgIssueDenom` and `MsgMintNFT` are handled by `msgServer.IssueDenom` and `msgServer.MintNFT` with no privileged-role check; any account can call them. [3](#0-2) 

**Step 3 — `setOwner` writes the poisoned key.**

`setOwner` calls `KeyOwner(owner, "ibc", tokenID)`, which produces:

```
PrefixOwners | "/" | <bech32-address> | "/" | "ibc" | "/" | <tokenID>
``` [4](#0-3) [5](#0-4) 

**Step 4 — `SplitKeyOwner` feeds `"ibc/<tokenID>"` to `SplitKeyDenom`.**

`SplitKeyOwner` strips `PrefixOwners+"/"`, then does `bytes.SplitN(key, "/", 2)`, yielding `keys[0] = <address>` and `keys[1] = "ibc/<tokenID>"`. It then calls `SplitKeyDenom(keys[1])`. [6](#0-5) 

**Step 5 — `SplitKeyDenom` incorrectly enters the IBC branch and fails.**

`IBCPrefix = "ibc/"`. `bytes.HasPrefix("ibc/<tokenID>", "ibc/")` is `true`. The IBC branch strips the prefix, leaving `<tokenID>` (alphanumeric, no `/`), so `bytes.Index(rest, delimiter)` returns `-1`. The guard `idx < 0 || idx != IBCDenomLen-len(IBCPrefix)` fires and returns `errors.New("wrong KeyDenom")`. [7](#0-6) 

**Step 6 — Both `GetOwner` and `GetOwners` propagate the error immediately.**

`GetOwner` returns `types.Owner{}, err` on the first bad key. [8](#0-7) 

`GetOwners` iterates **all** owner keys across all denoms; hitting any `"ibc"` denom entry causes it to return `types.Owners{}, err`, breaking enumeration for every owner on the chain. [9](#0-8) 

---

### Impact Explanation

- **`GetOwner(ctx, A, "ibc")`** returns a non-nil error for every holder of NFTs in the `"ibc"` denom — ownership queries are permanently broken for that denom.
- **`GetOwners(ctx)`** returns a non-nil error for **all** owners on the chain as soon as one `"ibc"`-denom NFT exists in the store, because the iterator is global and the error is returned immediately.
- Any downstream module, IBC NFT transfer path, or genesis export that calls `GetOwners` or `GetOwner` will fail.
- The NFT data itself (stored under `KeyNFT`) is unaffected, but the owner index — the authoritative record of who holds what — is permanently unreadable without a migration.

---

### Likelihood Explanation

The attack requires two standard, fee-paying, unprivileged transactions (`MsgIssueDenom` with `id="ibc"`, then `MsgMintNFT`). No governance, no special role, no leaked key. The denom `"ibc"` is not reserved anywhere in the validation logic. A single motivated attacker or even an accidental user choosing a short denom name triggers the corruption irreversibly.

---

### Recommendation

1. **Reserve `"ibc"` as a forbidden denom ID** in `ValidateDenomID`: reject any denom whose ID equals `"ibc"` (or more generally, any ID that, when followed by `/`, would be a prefix of a valid IBC denom key).
2. **Fix `SplitKeyDenom`** to require the full IBC format (`len(key) == IBCDenomLen`) before entering the IBC branch, rather than relying solely on `bytes.HasPrefix`.

---

```go
// keeper integration test (unmodified Go/Cosmos test setup)
func TestIBCDenomPoisoning(t *testing.T) {
    ctx, keeper := setupKeeper(t)
    creator := sdk.AccAddress([]byte("creator"))
    recipient := sdk.AccAddress([]byte("recipient"))

    // Step 1: issue denom "ibc" — passes ValidateDenomID
    err := keeper.IssueDenom(ctx, "ibc", "ibcname", "", "", creator)
    require.NoError(t, err)

    // Step 2: mint one NFT — writes poisoned owner key
    err = keeper.MintNFT(ctx, "ibc", "token1", "t", "", "", creator, recipient)
    require.NoError(t, err)

    // Step 3: GetOwner now returns an error
    _, err = keeper.GetOwner(ctx, recipient, "ibc")
    require.Error(t, err) // "wrong KeyDenom"

    // Step 4: GetOwners is broken for ALL owners
    _, err = keeper.GetOwners(ctx)
    require.Error(t, err) // "wrong KeyDenom"
}

func SplitKeyDenom(key []byte) (denomID, tokenID string, err error) {
	if bytes.HasPrefix(key, []byte(IBCPrefix)) {
		// IBC denom: ibc/hash/tokenID (take remainder after second delimiter as tokenID)
		rest := key[len(IBCPrefix):]
		idx := bytes.Index(rest, delimiter)
		if idx < 0 || idx != IBCDenomLen-len(IBCPrefix) {
			return denomID, tokenID, errors.New("wrong KeyDenom")
		}
		denomID = IBCPrefix + string(rest[:idx])
		tokenID = string(rest[idx+len(delimiter):])
	} else {
		// Standard denom: denom/tokenID (take remainder after first delimiter as tokenID)
		idx := bytes.Index(key, delimiter)
		if idx < 0 {
			return denomID, tokenID, errors.New("wrong KeyDenom")
		}
		denomID = string(key[:idx])
		tokenID = string(key[idx+len(delimiter):])
	}

	return denomID, tokenID, nil
}

func KeyOwner(address sdk.AccAddress, denomID, tokenID string) []byte {
	key := append(PrefixOwners, delimiter...)
	if address != nil {
		key = append(key, []byte(address.String())...)
		key = append(key, delimiter...)
	}

	if address != nil && len(denomID) > 0 {
		key = append(key, []byte(denomID)...)
		key = append(key, delimiter...)
	}

	if address != nil && len(denomID) > 0 && len(tokenID) > 0 {
		key = append(key, []byte(tokenID)...)
	}
	return key
}
