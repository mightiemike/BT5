### Title
Migration Aborts with `ErrValidatorNotBonded` When Vesting Position's Validator Is Unbonding at v8 Upgrade Time — (`x/tieredrewards/keeper/transfer_delegation.go`)

---

### Summary

`ForceFullExitWithDelegation`, called during the v8 `Migrate1to2` migration for every vesting-owned position, unconditionally delegates through `transferDelegationFromPosition`, which contains a hard `IsBonded()` guard. If the position's validator has begun unbonding before the upgrade block, this guard returns `ErrValidatorNotBonded`, the error propagates through the entire migration call stack without any recovery, the upgrade handler fails, and the vesting owner's funds remain locked in the position's delegator account on the unbonding validator.

---

### Finding Description

**Exact call chain:**

1. `Migrate1to2` → `v2.Migrate` → `exitVestedAccountsPositions` iterates all vesting-owned positions and calls `ForceFullExitWithDelegation` for each. [1](#0-0) 

2. `ForceFullExitWithDelegation` calls `transferDelegationFromPosition` and returns any error it receives. [2](#0-1) 

3. `transferDelegationFromPosition` fetches the validator and immediately checks `IsBonded()`. If the validator is in `Unbonding` or `Unbonded` status, it returns `ErrValidatorNotBonded` — no fallback, no skip, no alternative path. [3](#0-2) 

4. Every caller in the chain propagates the error with `fmt.Errorf(...%w...)` — there is no `continue`-on-error logic anywhere in `exitVestedAccountsPositions`. [4](#0-3) 

**Precondition reachability:**

A validator transitions to `Unbonding` status via normal chain operations: jailing for downtime or double-signing, or an explicit `MsgUndelegate` that drops the validator below the minimum power threshold. None of these require attacker privilege — a validator can begin unbonding at any block before the upgrade. The `AfterValidatorBeginUnbonding` hook only records a validator event; it does not exit or migrate any positions. [5](#0-4) 

**No guard prevents the scenario:**

`reconcileAmountFromShares` (called before `transferDelegationFromPosition`) succeeds for unbonding validators — it only reads `TokensFromShares` and does not check bond status. [6](#0-5) 

The `IsBonded()` check in `transferDelegationFromPosition` is the sole gate and it is unconditional. [7](#0-6) 

---

### Impact Explanation

When the migration returns an error, the Cosmos SDK upgrade handler panics and the chain halts. The vesting account owner's tokens remain locked inside the position's module-derived delegator account (`tieredrewards/position/<id>`), delegated to an unbonding validator. The owner has no user-facing message to recover them: `ForceFullExitWithDelegation` is a migration-only internal function with no `MsgServer` entry point. The position record persists in state. Funds are inaccessible until a coordinated emergency upgrade is deployed. [8](#0-7) 

---

### Likelihood Explanation

Validator jailing for liveness faults is a routine chain event. On any active Cronos POS Chain network, the probability that at least one validator holding a vesting-owned tieredrewards position begins unbonding in the window before the v8 upgrade block is non-negligible. No attacker action is required; the condition arises from normal validator lifecycle events.

---

### Recommendation

In `ForceFullExitWithDelegation` (or in `transferDelegationFromPosition` when called from the migration context), replace the hard `ErrValidatorNotBonded` rejection with a fallback that calls `stakingKeeper.Undelegate` directly on the position's delegator address when the validator is not bonded. This initiates a standard unbonding period for the position's tokens and transfers the resulting unbonding delegation record to the owner, allowing the migration to complete. Alternatively, add a migration-specific exit path that skips `transferDelegationFromPosition` entirely for unbonding validators and instead reassigns the existing unbonding delegation to the owner's address.

---

func exitVestedAccountsPositions(
	ctx context.Context,
	positions collections.Map[uint64, types.Position],
	ak AccountKeeper,
	pk PositionForceExiter,
) error {
	sdkCtx := sdk.UnwrapSDKContext(ctx)

	var toExit []uint64
	if err := positions.Walk(ctx, nil, func(posID uint64, pos types.Position) (bool, error) {
		ownerAddr, err := sdk.AccAddressFromBech32(pos.Owner)
		if err != nil {
			return false, fmt.Errorf("parse owner address for position %d: %w", posID, err)
		}
		acc := ak.GetAccount(ctx, ownerAddr)
		if acc == nil {
			return false, fmt.Errorf("owner account not found for position %d: %s", posID, ownerAddr.String())
		}
		if _, ok := acc.(sdkvesting.VestingAccount); ok {
			toExit = append(toExit, posID)
		}
		return false, nil
	}); err != nil {
		return fmt.Errorf("walk positions: %w", err)
	}

	for _, posID := range toExit {
		sdkCtx.Logger().Info("v8 migration: force-exit vesting-owned position", "position_id", posID)
		if err := pk.ForceFullExitWithDelegation(ctx, posID); err != nil {
			return fmt.Errorf("force-exit position %d: %w", posID, err)
		}
	}

	sdkCtx.Logger().Info("v8 migration: vesting-owned positions exited", "count", len(toExit))
	return nil

func (k Keeper) transferDelegationFromPosition(ctx context.Context, pos types.PositionState, valAddr sdk.ValAddress, amount math.Int) (math.LegacyDec, math.LegacyDec, math.Int, error) {
	owner, err := sdk.AccAddressFromBech32(pos.Owner)
	if err != nil {
		return math.LegacyDec{}, math.LegacyDec{}, math.Int{}, errorsmod.Wrap(sdkerrors.ErrInvalidAddress, "invalid owner address")
	}

	posDelAddr, err := sdk.AccAddressFromBech32(pos.DelegatorAddress)
	if err != nil {
		return math.LegacyDec{}, math.LegacyDec{}, math.Int{}, errorsmod.Wrap(sdkerrors.ErrInvalidAddress, "invalid delegator address")
	}

	// Defensive
	if !pos.IsDelegated() {
		return math.LegacyDec{}, math.LegacyDec{}, math.Int{}, errorsmod.Wrapf(types.ErrPositionNotDelegated, "position %d is not delegated", pos.Id)
	}

	// Defensive
	isRedelegating, err := k.isRedelegating(ctx, pos.DelegatorAddress)
	if err != nil {
		return math.LegacyDec{}, math.LegacyDec{}, math.Int{}, err
	}
	if isRedelegating {
		return math.LegacyDec{}, math.LegacyDec{}, math.Int{}, errorsmod.Wrapf(types.ErrActiveRedelegation, "position %d has an active redelegation", pos.Id)
	}

	validator, err := k.stakingKeeper.GetValidator(ctx, valAddr)
	if errors.Is(err, stakingtypes.ErrNoValidatorFound) {
		return math.LegacyDec{}, math.LegacyDec{}, math.Int{}, types.ErrTransferDelegationDestNotFound
	} else if err != nil {
		return math.LegacyDec{}, math.LegacyDec{}, math.Int{}, err
	}

	if !validator.IsBonded() {
		return math.LegacyDec{}, math.LegacyDec{}, math.Int{}, types.ErrValidatorNotBonded
	}

func (h Hooks) AfterValidatorBeginUnbonding(ctx context.Context, _ sdk.ConsAddress, valAddr sdk.ValAddress) error {
	count, err := h.k.getPositionCountForValidator(ctx, valAddr)
	if err != nil {
		return err
	}
	if count == 0 {
		return nil
	}

	tokensPerShare, err := h.k.getTokensPerShare(ctx, valAddr)
	if err != nil {
		return err
	}

	sdkCtx := sdk.UnwrapSDKContext(ctx)
	_, err = h.k.appendValidatorEvent(ctx, valAddr, types.ValidatorEvent{
		Height:         sdkCtx.BlockHeight(),
		Timestamp:      sdkCtx.BlockTime(),
		EventType:      types.ValidatorEventType_VALIDATOR_EVENT_TYPE_UNBOND,
		TokensPerShare: tokensPerShare,
		ReferenceCount: count,
	})
	return err
}

		return math.ZeroInt(), nil
