Audit Report

## Title
`SwapAllowlistExtension.beforeSwap` Checks Router Address Instead of End User, Allowing Any Caller to Bypass the Swap Allowlist via `MetricOmmSimpleRouter` - (File: `metric-periphery/contracts/extensions/SwapAllowlistExtension.sol`)

## Summary

`SwapAllowlistExtension.beforeSwap` gates swaps by checking `allowedSwapper[pool][sender]`, where `sender` is the value forwarded from `MetricOmmPool.swap()` as `msg.sender`. When a user routes through `MetricOmmSimpleRouter`, the pool's `msg.sender` is the router contract, not the end user. If the pool admin allowlists the router to enable router-mediated swaps, every unprivileged user can bypass the allowlist entirely by calling `router.exactInputSingle()`, because the extension evaluates `allowedSwapper[pool][router] == true` regardless of who the actual caller is.

## Finding Description

**Step 1 — Pool captures `msg.sender` and forwards it as `sender` to extensions:**

`MetricOmmPool.swap()` passes `msg.sender` directly to `_beforeSwap`:

```solidity
// metric-core/contracts/MetricOmmPool.sol L230-240
_beforeSwap(
  msg.sender,   // whoever called pool.swap()
  recipient,
  ...
  extensionData
);
```

**Step 2 — `ExtensionCalling._beforeSwap` relays `sender` verbatim:**

```solidity
// metric-core/contracts/ExtensionCalling.sol L160-176
_callExtensionsInOrder(
  BEFORE_SWAP_ORDER,
  abi.encodeCall(IMetricOmmExtensions.beforeSwap, (sender, recipient, ...))
);
```

**Step 3 — `SwapAllowlistExtension.beforeSwap` gates on that `sender` value:**

```solidity
// metric-periphery/contracts/extensions/SwapAllowlistExtension.sol L37
if (!allowAllSwappers[msg.sender] && !allowedSwapper[msg.sender][sender]) {
  revert IMetricOmmPoolActions.NotAllowedToSwap();
}
```

Here `msg.sender` is the pool (correct), and `sender` is whoever called `pool.swap()`.

**Step 4 — `MetricOmmSimpleRouter.exactInputSingle` calls `pool.swap()` without forwarding the original caller:**

```solidity
// metric-periphery/contracts/MetricOmmSimpleRouter.sol L71-80
_setNextCallbackContext(params.pool, CALLBACK_MODE_JUST_PAY, msg.sender, params.tokenIn);
(int128 amount0Delta, int128 amount1Delta) = IMetricOmmPoolActions(params.pool)
  .swap(
    params.recipient,
    params.zeroForOne,
    MetricOmmSwapInputs.asAmountSpecifiedIn(params.amountIn),
    priceLimitX64,
    "",
    params.extensionData   // real user identity NOT encoded here
  );
```

The router stores `msg.sender` only in transient storage for the payment callback (`_setNextCallbackContext`), not in `extensionData` or any argument visible to extensions. The pool therefore sees `msg.sender = router`, and the extension checks `allowedSwapper[pool][router]`.

**Why existing guards fail:** `SwapAllowlistExtension` has no mechanism to decode the real user from `extensionData`. The `extensionData` passed by the router is `params.extensionData`, which is user-supplied and not populated with the caller's address by the router. There is no fallback path that recovers the true originator.

## Impact Explanation

A pool admin who deploys a pool with `SwapAllowlistExtension` to restrict swaps to a specific set of users (e.g., KYC-verified traders) faces an irreconcilable dilemma:

- **If the router is NOT allowlisted**: allowlisted users cannot use the router at all; they must call `pool.swap()` directly, making the primary public entry point unusable for this pool.
- **If the router IS allowlisted** (the natural operational step): `allowedSwapper[pool][router] = true`, so `beforeSwap` passes for **any** caller who routes through the router. Every unprivileged user can call `router.exactInputSingle()` and the check resolves to `allowedSwapper[pool][router] == true` — the allowlist is fully bypassed.

Consequence: unauthorized users execute swaps on a pool designed to be restricted, draining LP-owned token reserves at oracle-anchored prices without the LP's consent to trade with those counterparties. This is a direct loss of LP principal and constitutes broken core pool functionality (the allowlist protection is rendered inoperative).

## Likelihood Explanation

- `SwapAllowlistExtension` is a production periphery contract, not a mock or test artifact.
- `MetricOmmSimpleRouter` is the primary public swap entry point.
- Any operator who configures both on the same pool and allowlists the router (a natural and expected operational step to enable router-mediated swaps for allowlisted users) is immediately vulnerable.
- No special privilege, flash loan, or unusual token behavior is required — a plain `exactInputSingle` call from any EOA suffices.
- The bypass is repeatable on every swap and requires zero preconditions beyond the router being allowlisted.

## Recommendation

The extension must recover the true end-user identity. The cleanest fix is for the router to encode `msg.sender` into `extensionData` before forwarding to `pool.swap()`, and for `SwapAllowlistExtension.beforeSwap` to decode and check that value when present, falling back to `sender` for direct pool calls. Specifically:

1. In `MetricOmmSimpleRouter.exactInputSingle` (and `exactInput`, `exactOutputSingle`, `exactOutput`), encode `msg.sender` into `extensionData` before passing it to `pool.swap()`.
2. In `SwapAllowlistExtension.beforeSwap`, decode the originator from `extensionData` when it is non-empty and use that address for the allowlist check; use `sender` only when `extensionData` is empty (direct pool call).

Alternatively, define a standard originator-forwarding convention in `extensionData` at the protocol level so all extensions can recover the true caller when routed through periphery contracts.

## Proof of Concept

```
Setup:
  - Deploy pool with SwapAllowlistExtension configured as beforeSwap hook
  - Pool admin calls setAllowedToSwap(pool, alice, true)   // only alice allowed
  - Pool admin calls setAllowedToSwap(pool, router, true)  // router allowlisted so alice can use it

Attack (executed by bob, who is NOT allowlisted):
  1. bob calls MetricOmmSimpleRouter.exactInputSingle({pool: pool, ...})
  2. Router calls pool.swap(recipient, zeroForOne, amount, priceLimit, "", extensionData)
     → pool sees msg.sender = router
  3. Pool calls _beforeSwap(sender=router, ...)
  4. SwapAllowlistExtension.beforeSwap checks allowedSwapper[pool][router] == true  ✓
  5. Swap executes — bob has bypassed the allowlist entirely

Foundry test outline:
  - Deploy SwapAllowlistExtension, pool, and MetricOmmSimpleRouter
  - setAllowedToSwap(pool, alice, true)
  - setAllowedToSwap(pool, address(router), true)
  - vm.prank(bob); router.exactInputSingle(...)
  - Assert swap succeeds (no NotAllowedToSwap revert) despite bob not being allowlisted
```
