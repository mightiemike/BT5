### Title
Trading Fees Included in Spot Borrow Position Cause Users to Pay Interest on Fees Never Received â€” (`core/contracts/OffchainExchange.sol`, `core/contracts/SpotEngineState.sol`)

---

### Summary

When a user executes a spot trade, the trading fee is deducted from their `quoteDelta` before it is applied to their spot quote balance. If the resulting quote balance is negative (a borrow), the fee amount is embedded in that borrow position and accrues borrow interest via `cumulativeBorrowsMultiplierX18`. The fee was never loaned to the user â€” it was routed to `market.collectedFees` and eventually to `X_ACCOUNT` â€” yet the user is charged ongoing borrow interest on it.

---

### Finding Description

In `OffchainExchange.applyFee()`, the protocol deducts the trading fee and builder fee from `orderInfo.quoteDelta`:

```solidity
// OffchainExchange.sol lines 560-565
orderInfo.fee = meteredQuote - newMeteredQuote;
orderInfo.builderFee = matchQuote.abs().mul(feeInfo.builderFeeRate);
orderInfo.quoteDelta =
    orderInfo.quoteDelta -
    orderInfo.fee -
    orderInfo.builderFee;
``` [1](#0-0) 

The reduced `quoteDelta` is then applied to the user's spot balance via `_updateBalances()` â†’ `spotEngine.updateBalance()`: [2](#0-1) 

The fee itself is accumulated in `market.collectedFees` and later dumped to `X_ACCOUNT` via `dumpFees()` â€” it is never credited to the user: [3](#0-2) 

In `SpotEngineState._updateBalanceNormalized()`, when the resulting quote balance is negative, it is normalized using `cumulativeBorrowsMultiplierX18`:

```solidity
// SpotEngineState.sol lines 37-43
if (newAmount > 0) {
    cumulativeMultiplierX18 = state.cumulativeDepositsMultiplierX18;
} else {
    cumulativeMultiplierX18 = state.cumulativeBorrowsMultiplierX18;
}
balance.amountNormalized = newAmount.div(cumulativeMultiplierX18);
``` [4](#0-3) 

`cumulativeBorrowsMultiplierX18` grows over time in `_updateState()` based on utilization and the configured borrow rate (`interestFloorX18`, `interestSmallCapX18`, `interestLargeCapX18`): [5](#0-4) 

This means the fee amount embedded in the user's negative balance compounds over time. The user is effectively paying borrow interest on a sum they never received as a loan.

---

### Impact Explanation

A user who trades spot and whose quote balance is negative (or goes negative after the trade) pays borrow interest on the fee portion of their borrow. The fee was routed to the protocol, not loaned to the user. The user's effective borrow cost is overstated by the interest accrued on the fee amount. For active traders with persistent negative quote balances, this overcharge compounds continuously. The corrupted state is `balance.amountNormalized` for the user's spot quote product: it is more negative than the actual economic loan, causing `balanceNormalizedToBalance()` to return a larger-magnitude negative balance than warranted. [6](#0-5) 

---

### Likelihood Explanation

Any unprivileged user who trades on a spot market (via `matchOrders`) and holds a negative quote balance is affected. Leveraged spot trading â€” where users borrow the quote asset to buy base â€” is a core use case of the protocol. The default taker fee is 2 bps (`200_000_000_000_000`), applied on every fill. The effect is continuous and automatic; no special attacker action is required beyond normal trading. [7](#0-6) 

---

### Recommendation

Track the fee component of the quote delta separately from the user's borrow balance. When applying `quoteDelta` to the user's spot balance, the fee should be credited directly to `FEES_ACCOUNT` (or `market.collectedFees`) as a separate balance update, rather than being embedded in the user's net `quoteDelta`. This ensures the user's borrow position reflects only the actual economic loan, not the fee that was immediately taken by the protocol.

---

### Proof of Concept

1. User has spot quote balance of `0`.
2. User places a taker buy order: `quoteDelta = -1,000,000` (paying 1 USDC for base).
3. `applyFee()` computes `fee = 200` (2 bps), sets `quoteDelta = -1,000,200`.
4. `_updateBalances()` applies `-1,000,200` to the user's spot quote balance.
5. User's balance is now `-1,000,200`, normalized via `cumulativeBorrowsMultiplierX18`.
6. The 200-unit fee was credited to `market.collectedFees`, not to the user.
7. As `cumulativeBorrowsMultiplierX18` grows each epoch, the user's owed balance grows on the full `-1,000,200`, including the 200 fee units they never received.
8. Over time, the user repays more than the 1,000,000 they actually borrowed. [8](#0-7) [9](#0-8)

    function applyFee(
        uint32 productId,
        OrderInfo memory orderInfo,
        MarketInfo memory market,
        int128 alreadyMatched, // in quote
        uint128 appendix,
        bool taker
    ) internal {
        // X account is passthrough for trading and incurs
        // no fees
        if (orderInfo.sender == X_ACCOUNT) {
            return;
        }
        int128 matchQuote = orderInfo.quoteDelta;
        int128 meteredQuote = 0;
        if (taker) {
            // flat minimum fee
            if (alreadyMatched == 0) {
                meteredQuote += market.minSize;
                if (matchQuote < 0) {
                    meteredQuote = -meteredQuote;
                }
            }

            // exclude the portion on [0, self.min_size) for match_quote and
            // add to metered_quote
            // fee is only applied on [minSize, quote_amount)
            int128 feeApplied = MathHelper.abs(alreadyMatched + matchQuote) -
                market.minSize;
            feeApplied = MathHelper.min(feeApplied, matchQuote.abs());
            if (feeApplied > 0) {
                if (matchQuote < 0) {
                    feeApplied = -feeApplied;
                }
                meteredQuote += feeApplied;
            }
        } else {
            // for maker rebates things stay the same
            meteredQuote += matchQuote;
        }
        FeeInfo memory feeInfo = getUserFeeRateWithBuilder(
            orderInfo.sender,
            productId,
            appendix,
            taker
        );

        int128 keepRateX18 = ONE - feeInfo.feeRate;
        int128 newMeteredQuote = (meteredQuote > 0)
            ? meteredQuote.mul(keepRateX18)
            : meteredQuote.div(keepRateX18);
        orderInfo.fee = meteredQuote - newMeteredQuote;
        orderInfo.builderFee = matchQuote.abs().mul(feeInfo.builderFeeRate);
        orderInfo.quoteDelta =
            orderInfo.quoteDelta -
            orderInfo.fee -
            orderInfo.builderFee;
        if (orderInfo.builderFee > 0) {
            collectedBuilderFee[market.quoteId][feeInfo.builderId] += orderInfo
                .builderFee;
            emitBuilderEvent(orderInfo, feeInfo.builderId, productId);
        }
    }

    function _updateBalanceNormalized(
        State memory state,
        BalanceNormalized memory balance,
        int128 balanceDelta
    ) internal pure {
        if (balance.amountNormalized > 0) {
            state.totalDepositsNormalized -= balance.amountNormalized;
        } else {
            state.totalBorrowsNormalized += balance.amountNormalized;
        }

        int128 cumulativeMultiplierX18;
        if (balance.amountNormalized > 0) {
            cumulativeMultiplierX18 = state.cumulativeDepositsMultiplierX18;
        } else {
            cumulativeMultiplierX18 = state.cumulativeBorrowsMultiplierX18;
        }

        int128 newAmount = balance.amountNormalized.mul(
            cumulativeMultiplierX18
        ) + balanceDelta;

        if (newAmount > 0) {
            cumulativeMultiplierX18 = state.cumulativeDepositsMultiplierX18;
        } else {
            cumulativeMultiplierX18 = state.cumulativeBorrowsMultiplierX18;
        }

        balance.amountNormalized = newAmount.div(cumulativeMultiplierX18);

        if (balance.amountNormalized > 0) {
            state.totalDepositsNormalized += balance.amountNormalized;
        } else {
            state.totalBorrowsNormalized -= balance.amountNormalized;
        }
    }
