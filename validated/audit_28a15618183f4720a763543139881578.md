### Title
Missing `ClassTrace.Validate()` in `processReceivedPacket` allows malicious counterparty to store empty `BaseClassId` trace and permanently lock NFTs â€” (`x/nft-transfer/keeper/packet.go`)

### Summary
A malicious counterparty chain can send an IBC NFT packet with `data.ClassId` ending in a trailing slash (e.g., `"nft/channel-0/"`). `ParseClassTrace` splits on `/` and assigns `BaseClassId = ""`. Because `processReceivedPacket` never calls `classTrace.Validate()`, the malformed trace is stored, NFTs are minted under a class ID derived from an empty base, and any subsequent attempt to return those NFTs burns them on Cronos POS while the counterparty mints new ones â€” permanently locking the original escrowed NFTs.

### Finding Description
`ParseClassTrace` in `x/nft-transfer/types/trace.go` (line 62â€“74) splits `rawClassID` on `/` and takes the last element as `BaseClassId`. A trailing slash produces an empty last element, yielding `ClassTrace{Path: "...", BaseClassId: ""}`. `ClassTrace.Validate()` (line 115) explicitly rejects this with `"base class_id cannot be blank"`, but `Validate()` is never called in the receive path. `OnRecvPacket` in `relay.go` (line 105) only calls `data.ValidateBasic()`, which checks that `ClassId` is not blank â€” a trailing slash passes. `processReceivedPacket` in `packet.go` (lines 160â€“168) calls `ParseClassTrace`, then `SetClassTrace` (which stores without validation), then `IBCClassID()` (which returns a valid-looking `"ibc/<hash>"`), then `IssueDenom` and `MintNFT` â€” all without ever calling `Validate()`.

### Impact Explanation
NFTs are minted on Cronos POS under a class ID whose stored trace has `BaseClassId = ""`. `GetFullClassPath()` returns a path with a trailing slash. When the receiver attempts to return these NFTs, `IsAwayFromOrigin` evaluates to `false` (the path starts with the dest port/channel prefix), so the NFTs are burned on Cronos POS. The outgoing packet carries the trailing-slash class ID; the counterparty chain evaluates `IsAwayFromOrigin` as `true` and mints new NFTs rather than unescrow-ing originals. The original escrowed NFTs on the counterparty chain are permanently locked, and the Cronos POS NFTs are burned â€” a net permanent loss for the holder.

### Likelihood Explanation
Any IBC-connected chain (including a chain running modified or malicious software) can craft a packet with a trailing-slash `ClassId`. No privileged access, governance, or key compromise is required. The IBC relayer submits the packet on behalf of the counterparty; the receiving chain has no mechanism to reject it before `processReceivedPacket` executes.

### Recommendation
Add a `classTrace.Validate()` call in `processReceivedPacket` immediately after `ParseClassTrace`, before `SetClassTrace` and `IBCClassID()`:

```go
// in processReceivedPacket, after line 160
classTrace := types.ParseClassTrace(prefixedClassID)
if err := classTrace.Validate(); err != nil {
    return err
}
```

This mirrors the existing `Validate()` logic already present in `ClassTrace` and `Traces.Validate()` and would reject any trace with an empty `BaseClassId`.

func ParseClassTrace(rawClassID string) ClassTrace {
	classSplit := strings.Split(rawClassID, "/")

	if classSplit[0] == rawClassID {
		return ClassTrace{
			Path:        "",
			BaseClassId: rawClassID,
		}
	}

	return ClassTrace{
		Path:        strings.Join(classSplit[:len(classSplit)-1], "/"),
		BaseClassId: classSplit[len(classSplit)-1],
	}
}
