### Title
Precommit Vote Digest Lacks Chain ID — Cross-Chain Replay of Consensus Signatures Possible - (File: crates/apollo_signature_manager/src/signature_manager.rs)

### Summary

`build_precommit_vote_message_digest` constructs the signed payload for Tendermint precommit votes as `blake2s(b"PRECOMMIT_VOTE" || block_hash_bytes)`. Neither the block hash computation (`calculate_block_hash`) nor the digest itself folds in a chain ID. A valid precommit signature produced on one Starknet network is therefore structurally identical to a valid signature on any other network that reaches the same block hash value, enabling cross-chain replay of consensus votes.

### Finding Description

`build_precommit_vote_message_digest` in `crates/apollo_signature_manager/src/signature_manager.rs` builds the ECDSA preimage as:

```rust
fn build_precommit_vote_message_digest(block_hash: BlockHash) -> MessageDigest {
    let block_hash = block_hash.to_bytes_be();
    let mut message = Vec::with_capacity(PRECOMMIT_VOTE.len() + block_hash.len());
    message.extend_from_slice(PRECOMMIT_VOTE);
    message.extend_from_slice(&block_hash);
    MessageDigest(blake2s_to_felt(&message))
}
``` [1](#0-0) 

The only domain material is the ASCII tag `b"PRECOMMIT_VOTE"` and the 32-byte block hash. No chain ID, no network identifier, no sequencer address outside the hash.

`calculate_block_hash` — the function that produces the `BlockHash` value passed here — commits to: `block_hash_version`, `block_number`, `state_root`, `sequencer_address`, `block_timestamp`, `concat_counts`, `state_diff_commitment`, `transaction_commitment`, `event_commitment`, `receipt_commitment`, `gas_prices`, `starknet_version`, `0`, `parent_block_hash`. **Chain ID is absent from the block hash preimage.** [2](#0-1) 

Because neither the block hash nor the vote digest binds to a chain ID, the 32-byte `MessageDigest` value is identical for the same logical block on any two networks that happen to share the same block hash.

The `SignatureManager` component exposes `sign_precommit_vote(block_hash)` as a production RPC endpoint: [3](#0-2) [4](#0-3) 

The corresponding verification function `verify_precommit_vote_signature` reconstructs the same chain-ID-free digest: [5](#0-4) 

`handle_vote` in `single_height_consensus.rs` currently carries an explicit `// TODO(Asmaa): verify the signature` comment, meaning verification is not yet wired into the consensus path: [6](#0-5) 

However, the signing infrastructure is fully production-deployed, the digest design is already fixed in the codebase, and the TODO makes clear that verification will be added. Once it is, the broken domain boundary becomes immediately exploitable.

### Impact Explanation

When signature verification is enabled (the stated next step), an attacker who observes a valid precommit vote `(block_hash, sig, pubkey)` from validator V on network A can submit the identical `(block_hash, sig, pubkey)` tuple on network B. Because `verify_precommit_vote_signature` recomputes the same chain-ID-free digest, the check passes. The attacker can therefore inject forged-but-cryptographically-valid precommit votes from real validators into a different network's consensus round, potentially contributing to a spurious quorum or disrupting the BFT safety guarantee.

This matches the allowed impact: **High — Transaction conversion or signature/hash logic binds the wrong signer, hash, type, or executable payload.**

### Likelihood Explanation

- The block hash does not include chain ID, so two networks with the same genesis and early history (e.g., a fork, a shadow network, or a testnet that mirrors mainnet block structure) will produce identical block hashes for the same block number.
- The signing key is the same validator key across networks if operators reuse keys (common practice).
- The attack requires only passive observation of consensus traffic on one network and the ability to submit a vote message on another — both are unprivileged operations.
- The TODO comment confirms verification is planned, making the window of exploitation a matter of when, not if.

### Recommendation

Fold chain ID into the precommit vote digest. The `SignatureManager` already holds or can be given the chain ID at construction time. The fix is:

```rust
fn build_precommit_vote_message_digest(
    block_hash: BlockHash,
    chain_id: &ChainId,
) -> MessageDigest {
    let block_hash = block_hash.to_bytes_be();
    let chain_id_bytes = chain_id.as_hex_str().as_bytes(); // or canonical encoding
    let mut message = Vec::with_capacity(
        PRECOMMIT_VOTE.len() + chain_id_bytes.len() + block_hash.len()
    );
    message.extend_from_slice(PRECOMMIT_VOTE);
    message.extend_from_slice(chain_id_bytes);
    message.extend_from_slice(&block_hash);
    MessageDigest(blake2s_to_felt(&message))
}
```

Both `sign_precommit_vote` and `verify_precommit_vote_signature` must be updated to accept and pass the chain ID. The same fix should be applied to `build_peer_identity_message_digest` for defence-in-depth.

### Proof of Concept

1. Run two Starknet sequencer instances with different chain IDs (`SN_MAIN` and `SN_SEPOLIA`) but identical genesis and block-building parameters so that block N on both chains produces the same `BlockHash` value (trivially achievable on a local devnet).
2. On chain A, observe a validator's precommit vote for block N: `(block_hash_N, sig_A, pubkey_V)`.
3. On chain B, inject a `Vote { vote_type: Precommit, proposal_commitment: Some(block_hash_N), signature: sig_A, voter: V, ... }` into the consensus message stream.
4. Once `handle_vote` calls `verify_precommit_vote_signature(block_hash_N, sig_A, pubkey_V)`, the function recomputes `blake2s(b"PRECOMMIT_VOTE" || block_hash_N)` — identical on both chains — and `ecdsa_verify` returns `true`.
5. The replayed vote is accepted as a genuine precommit from validator V on chain B, contributing to a quorum that V never intended to cast on that network. [1](#0-0) [7](#0-6) [6](#0-5)

            .chain(&block_commitments.event_commitment.0)
            .chain(&block_commitments.receipt_commitment.0)

                gas_prices_to_hash(

            )
            .chain(&Felt::ZERO)
            .chain(&previous_block_hash.0)
            .get_poseidon_hash(),
    ))
