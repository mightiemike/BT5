### Title
Silent Bonus Forfeiture via Checkpoint Advance Before Pool Check in `slashRedelegationPosition` — (`x/tieredrewards/keeper/slash.go`, `x/tieredrewards/keeper/claim_rewards.go`)

---

### Summary

`processEventsAndClaimBonus` advances `pos.LastBonusAccrual` in memory **before** checking whether the rewards pool has sufficient balance. When `slashRedelegationPosition` catches `ErrInsufficientBonusPool` and swallows it, it then persists the position with the already-advanced checkpoint. The owner's accrued bonus is permanently forfeited — neither paid nor recoverable.

---

### Finding Description

The ordering of operations inside `processEventsAndClaimBonus` is the root cause:

```
claim_rewards.go:215  applyBonusAccrualCheckpoint(&pos.Position, blockTime)  // checkpoint advances HERE
claim_rewards.go:217  pos.UpdateLastKnownBonded(bonded)
...
claim_rewards.go:230  if err := k.sufficientBonusPoolBalance(ctx, bonusCoins); err != nil {
claim_rewards.go:231      return nil, err   // ErrInsufficientBonusPool returned HERE
```

`processEventsAndClaimBonus` takes `pos *types.PositionState` — a pointer. By the time `ErrInsufficientBonusPool` is returned at line 231, the caller's `pos` struct already has `LastBonusAccrual` advanced to `blockTime` and `LastKnownBonded` updated. No coins were transferred. [1](#0-0) 

Back in `slashRedelegationPosition`, the error is caught and swallowed:

```go
if _, err := k.processEventsAndClaimBonus(ctx, &pos); err != nil {
    if errors.Is(err, types.ErrInsufficientBonusPool) {
        k.logger(ctx).Error(...)   // logged only
    } else {
        return err
    }
}
// pos.LastBonusAccrual is already advanced; no bonus was paid
...
return k.setPositionWithState(ctx, pos, nil)   // persists the advanced checkpoint
``` [2](#0-1) 

The position is then saved to the store with the advanced `LastBonusAccrual`. On the next call to `processEventsAndClaimBonus` for this position (e.g., via `MsgClaimTierRewards`), `segmentStart` is initialized from `pos.LastBonusAccrual`:

```go
segmentStart := pos.LastBonusAccrual   // already advanced past the owed period
``` [3](#0-2) 

The owed period `[old_LastBonusAccrual, blockTime_of_slash]` is never replayed again. The bonus is permanently gone.

---

### Impact Explanation

- **Owner** permanently loses all bonus rewards accrued since the last checkpoint up to the slash block time.
- **`RewardsPoolName`** retains the funds that should have been paid; they are not burned or redistributed — they simply stay in the pool, inaccessible to the owner.
- There is no on-chain event or record of the forfeiture; only a node log message is emitted.
- The owner cannot recover the loss by waiting for pool replenishment and retrying, because the checkpoint has already advanced past the owed period. [4](#0-3) 

---

### Likelihood Explanation

The preconditions are operationally realistic:

1. A position has been redelegated (creating a `RedelegationMappings` entry).
2. The `RewardsPoolName` balance is below the bonus owed to the position — a normal condition if the pool is depleted by BeginBlocker top-ups or other claims.
3. The source validator is slashed while the redelegation entry is still active (within the SDK's redelegation completion window, typically 21 days).

No governance action, privileged key, or attacker-controlled parameter is required. The slash is initiated by the staking module's evidence handling, which is a standard production path. [5](#0-4) 

---

### Recommendation

Move `applyBonusAccrualCheckpoint` and `pos.UpdateLastKnownBonded` to **after** the pool balance check and the `SendCoinsFromModuleToAccount` call succeeds. This ensures the checkpoint only advances when the bonus is actually paid.

Alternatively, in `slashRedelegationPosition`, if `ErrInsufficientBonusPool` is caught, explicitly **do not** call `setPositionWithState` with the modified `pos` — instead reload the original position from the store (or save a snapshot before calling `processEventsAndClaimBonus`) so the checkpoint is not persisted.

The ADR documents this as an intentional "chain-halt avoidance" trade-off, but the current implementation goes further than necessary: it permanently destroys the owner's claim rather than merely deferring it. The fix preserves chain-halt avoidance while keeping the owner's future claim intact. [6](#0-5)

// Keeper test (unmodified Go/Cosmos test setup):
func (s *KeeperSuite) TestSlashRedelegation_InsufficientPool_PermanentlyForfeitsBonus() {
    lockAmount := sdkmath.NewInt(sdk.DefaultPowerReduction.Int64())
    _, bondDenom := s.getStakingData()

    // Fund pool with only 1uatom — far below any accrued bonus.
    s.fundRewardsPool(sdkmath.NewInt(1), bondDenom)

    pos, _, unbondingID := s.setupRedelegatingPosition(lockAmount)
    owner := sdk.MustAccAddressFromBech32(pos.Owner)
    preAccrual := pos.LastBonusAccrual

    // Advance 30 days so a large bonus accrues.
    s.ctx = s.ctx.WithBlockTime(s.ctx.BlockTime().Add(30 * 24 * time.Hour))

    balBefore := s.app.BankKeeper.GetBalance(s.ctx, owner, bondDenom)

    // Trigger BeforeRedelegationSlashed — pool is insufficient.
    err := s.keeper.Hooks().BeforeRedelegationSlashed(
        s.ctx, unbondingID, pos.Delegation.Shares.Quo(sdkmath.LegacyNewDec(10)))
    s.Require().NoError(err) // hook swallows the error

    // Owner received zero bonus.
    balAfter := s.app.BankKeeper.GetBalance(s.ctx, owner, bondDenom)
    s.Require().True(balAfter.Amount.Equal(balBefore.Amount), "owner received no bonus")

    // Checkpoint has advanced past the owed period — bonus is permanently lost.
    updated, _ := s.keeper.GetPositionState(s.ctx, pos.Id)
    s.Require().True(updated.LastBonusAccrual.After(preAccrual),
        "checkpoint advanced even though bonus was not paid")

    // Now fund the pool generously and claim — owner still gets zero for the lost period.
    s.fundRewardsPool(sdkmath.NewInt(1_000_000_000), bondDenom)
    s.ctx = s.ctx.WithBlockTime(s.ctx.BlockTime()) // same block time
    bonus, _ := s.keeper.ProcessEventsAndClaimBonus(s.ctx, &updated)
    s.Require().True(bonus.IsZero(), "forfeited period is unrecoverable")
}

// TestSlashRedelegationPosition_ClaimsBonusRewardsUpToSlash verifies that when
// a redelegation slash fires, any bonus accrued on the destination delegation
// since the last accrual checkpoint is paid out to the position owner, and the
// position's bonus-state checkpoints (LastBonusAccrual, LastKnownBonded) are
// advanced.
func (s *KeeperSuite) TestSlashRedelegationPosition_ClaimsBonusRewardsUpToSlash() {
	lockAmount := sdkmath.NewInt(sdk.DefaultPowerReduction.Int64())
	_, bondDenom := s.getStakingData()
	s.fundRewardsPool(sdkmath.NewInt(1_000_000_000), bondDenom)

	pos, _, unbondingID := s.setupRedelegatingPosition(lockAmount)
	owner := sdk.MustAccAddressFromBech32(pos.Owner)
	preAccrual := pos.LastBonusAccrual

	// Advance block time so bonus accrues on the destination validator.
	s.ctx = s.ctx.WithBlockHeight(s.ctx.BlockHeight() + 1)
	s.ctx = s.ctx.WithBlockTime(s.ctx.BlockTime().Add(30 * 24 * time.Hour))

	balBefore := s.app.BankKeeper.GetBalance(s.ctx, owner, bondDenom)

	// Partial slash — a small fraction of shares.
	sharesToUnbond := pos.Delegation.Shares.Quo(sdkmath.LegacyNewDec(10))
	err := s.keeper.Hooks().BeforeRedelegationSlashed(s.ctx, unbondingID, sharesToUnbond)
	s.Require().NoError(err)

	balAfter := s.app.BankKeeper.GetBalance(s.ctx, owner, bondDenom)
	s.Require().True(balAfter.Amount.GT(balBefore.Amount),
		"owner should have received bonus rewards accrued up to slash: before=%s after=%s",
		balBefore.Amount, balAfter.Amount)

	updated, err := s.keeper.GetPositionState(s.ctx, pos.Id)
	s.Require().NoError(err)
	s.Require().True(updated.LastBonusAccrual.After(preAccrual),
		"LastBonusAccrual should have advanced past the pre-slash checkpoint")
	s.Require().Equal(s.ctx.BlockTime(), updated.LastBonusAccrual,
		"LastBonusAccrual should advance to the slash block time")
	s.Require().True(updated.LastKnownBonded,
		"LastKnownBonded should remain true — destination validator is still bonded")
}
