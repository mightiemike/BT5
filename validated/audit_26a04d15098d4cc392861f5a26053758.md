### Title
Division by Zero in `socializeSubaccount` When `openInterest` Is Zero Permanently Blocks Subaccount Finalization — (File: `core/contracts/PerpEngine.sol`)

---

### Summary

`PerpEngine.socializeSubaccount` unconditionally divides by `state.openInterest` when distributing residual negative `vQuoteBalance` across market participants. If `openInterest` is zero at the time of socialization — a reachable state when all perp positions in a market are closed — the transaction reverts with a divide-by-zero error, permanently blocking the finalization and bad-debt resolution of any insolvent subaccount in that market.

---

### Finding Description

In `PerpEngine.socializeSubaccount`, after insurance is exhausted, the remaining negative `vQuoteBalance` is spread across open-interest holders via:

```solidity
// PerpEngine.sol lines 164–171
if (balance.vQuoteBalance < 0) {
    // socialize across all other participants
    int128 fundingPerShare = -balance.vQuoteBalance.div(
        state.openInterest   // ← no zero-guard
    );
    state.cumulativeFundingLongX18 += fundingPerShare;
    state.cumulativeFundingShortX18 -= fundingPerShare;
    balance.vQuoteBalance = 0;
}
```

`MathSD21x18.div` enforces `require(y != 0, "DBZ")`, so if `state.openInterest == 0` the entire call reverts. [1](#0-0) 

`openInterest` is the running sum of absolute position sizes, maintained in `_updateBalance`:

```solidity
// PerpEngineState.sol lines 30, 45–50
state.openInterest -= balance.amount.abs();   // pre-update
...
if (balance.amount > 0) {
    state.openInterest += balance.amount;
} else {
    state.openInterest -= balance.amount;
}
``` [2](#0-1) 

When every subaccount closes its position, `openInterest` reaches exactly zero. A subaccount that closed its position at a loss retains a negative `vQuoteBalance` (trading loss + accumulated funding) even though `amount == 0`. If that subaccount later becomes insolvent, `socializeSubaccount` is called with `openInterest == 0` and reverts.

The call to `socializeSubaccount` is made unconditionally inside `_finalizeSubaccount` in `ClearinghouseLiq`:

```solidity
// ClearinghouseLiq.sol lines 386–389
v.insurance = perpEngine.socializeSubaccount(
    txn.liquidatee,
    v.insurance
);
``` [3](#0-2) 

There is no pre-check for `openInterest == 0` anywhere in the call chain.

---

### Impact Explanation

**Impact: High**

When the revert occurs, the entire `liquidateSubaccountImpl` transaction reverts. The insolvent subaccount cannot be finalized. Its negative `vQuoteBalance` — representing real bad debt — remains permanently unresolved in the protocol. No liquidator can ever successfully finalize this subaccount while `openInterest` stays at zero, because every attempt hits the same revert. The protocol's bad-debt socialization mechanism is completely bypassed for this market state, leaving the accounting in a corrupted, unrecoverable condition. [4](#0-3) 

---

### Likelihood Explanation

**Likelihood: Low**

The condition requires two simultaneous facts: (1) a subaccount with `amount == 0` and `vQuoteBalance < 0` (closed position with residual loss), and (2) `openInterest == 0` for that perp product (no other open positions). This is most likely in low-activity or newly-launched markets where a single trader opens and closes a position at a loss and no other traders are active. It is not a contrived edge case — it is a natural market state.

---

### Recommendation

Add a zero-guard before the division in `PerpEngine.socializeSubaccount`. If `openInterest == 0`, there are no other participants to socialize the loss across; the debt should simply be written off (set `vQuoteBalance = 0`) without adjusting the cumulative funding accumulators:

```solidity
if (balance.vQuoteBalance < 0) {
    if (state.openInterest != 0) {
        int128 fundingPerShare = -balance.vQuoteBalance.div(
            state.openInterest
        );
        state.cumulativeFundingLongX18 += fundingPerShare;
        state.cumulativeFundingShortX18 -= fundingPerShare;
    }
    balance.vQuoteBalance = 0;
}
``` [5](#0-4) 

---

### Proof of Concept

1. Market has one perp product (e.g., `productId = 2`).
2. Alice opens a long position: `amount = 100`, `vQuoteBalance = -10000` (paid quote to open).
3. Price drops. Alice closes the position: `amount = 0`, `vQuoteBalance = -500` (net loss after receiving proceeds).
4. `openInterest` is now `0` (Alice was the only participant).
5. Alice's quote balance is also negative (she has no other assets). She is under maintenance margin.
6. Liquidator submits `liquidateSubaccountImpl` with `productId = type(uint32).max` to finalize Alice.
7. `_finalizeSubaccount` confirms all perp `amount == 0` ✓, settles positive PnL (none), exhausts insurance against Alice's quote deficit.
8. `perpEngine.socializeSubaccount(alice, remainingInsurance)` is called.
9. Inside: `balance.vQuoteBalance < 0` → true; insurance insufficient → enters the inner `if`; calls `.div(state.openInterest)` where `state.openInterest == 0`.
10. `MathSD21x18.div` reverts with `"DBZ"`.
11. The entire liquidation transaction reverts. Alice's bad debt is permanently stuck. [6](#0-5) [7](#0-6)
