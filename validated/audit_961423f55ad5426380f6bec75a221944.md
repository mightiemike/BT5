### Title
`MsgUpdateTier` Retroactively Applies New `ExitDuration` to Existing Locked Positions — (`x/tieredrewards/keeper/msg_server.go`, `x/tieredrewards/keeper/tier.go`)

### Summary

The `ExitDuration` field of a tier is a mutable global parameter that governance can change via `MsgUpdateTier`. However, `TriggerExitFromTier` reads the **current** tier's `ExitDuration` at trigger time rather than the duration that was in effect when the position was created. Because `ExitDuration` is never stored per-position, any governance update to a tier's `ExitDuration` retroactively changes the lock-out period for all existing positions that have not yet triggered their exit.

---

### Finding Description

When a user locks tokens via `MsgLockTier`, the tier's `ExitDuration` is **not** copied into the `Position` struct. The `Position` only stores `TierId` as a foreign key. [1](#0-0) 

Later, when the user calls `TriggerExitFromTier`, the keeper fetches the **live** tier record and uses its current `ExitDuration` to compute `ExitUnlockAt`:

```go
tier, err := ms.getTier(ctx, pos.TierId)
...
pos.TriggerExit(sdkCtx.BlockTime(), tier.ExitDuration)
``` [2](#0-1) 

`TriggerExit` writes the computed `ExitUnlockAt` into the position:

```go
func (p *Position) TriggerExit(blockTime time.Time, duration time.Duration) {
    p.ExitTriggeredAt = blockTime
    p.ExitUnlockAt = blockTime.Add(duration)
}
``` [3](#0-2) 

`SetTier` — called by `MsgUpdateTier` — validates only the tier's own fields and writes the new tier unconditionally, with **no check for active positions**:

```go
func (k Keeper) SetTier(ctx context.Context, tier types.Tier) error {
    if err := tier.Validate(); err != nil {
        return err
    }
    if err := k.Tiers.Set(ctx, tier.Id, tier); err != nil { ... }
    return nil
}
``` [4](#0-3) 

By contrast, `deleteTier` **does** guard against active positions:

```go
func (k Keeper) deleteTier(ctx context.Context, tierId uint32) error {
    hasPositions, err := k.hasPositionsForTier(ctx, tierId)
    ...
    if hasPositions {
        return types.ErrTierHasActivePositions
    }
``` [5](#0-4) 

This asymmetry — deletion is guarded, update is not — is the structural root cause, directly mirroring the `assetToken` pattern in the external report.

---

### Impact Explanation

A governance proposal that increases `ExitDuration` (e.g., from 365 days to 730 days or to `math.MaxInt64`) takes effect immediately for **all** existing positions that have not yet called `TriggerExitFromTier`. Users who locked tokens under the original terms are retroactively subjected to a longer (potentially unbounded) lock period. Because `ExitUnlockAt` is the gate for both `TierUndelegate` and `WithdrawFromTier`, the principal is inaccessible until the new, longer duration elapses. In the extreme case (duration set to a very large value), this constitutes permanent fund lockup for all pre-existing positions.

The corrupted invariant is: **`pos.ExitUnlockAt` must reflect the `ExitDuration` that was in effect when the position was created**, not the duration at trigger time.

---

### Likelihood Explanation

`MsgUpdateTier` is a standard governance message reachable via any on-chain governance proposal. A legitimate governance action to adjust exit duration for **new** positions (e.g., to encourage longer-term staking) silently and unintentionally applies the new duration to all existing positions that have not yet triggered exit. No malicious intent is required; the breakage is a design flaw, not an attack. Any validator or token holder can submit and pass such a proposal. [6](#0-5) 

---

### Recommendation

Store `ExitDuration` inside the `Position` struct at creation time (analogous to the external report's recommendation to store `assetToken` inside the `Application` struct):

```go
// In Position proto / struct
ExitDuration time.Duration  // snapshot from tier at lock time
```

Populate it in `createDelegatedPosition` from the tier's current `ExitDuration`, and use `pos.ExitDuration` instead of `tier.ExitDuration` inside `TriggerExitFromTier`:

```go
// Before (reads live tier)
tier, err := ms.getTier(ctx, pos.TierId)
pos.TriggerExit(sdkCtx.BlockTime(), tier.ExitDuration)

// After (uses snapshotted value)
pos.TriggerExit(sdkCtx.BlockTime(), pos.ExitDuration)
```

This ensures that governance changes to a tier's `ExitDuration` only affect positions created **after** the change, preserving the terms under which existing positions were locked.

---

### Proof of Concept

1. Governance sets Tier 1 with `ExitDuration = 365 days`.
2. Alice calls `MsgLockTier` for Tier 1. Her `Position` stores `TierId = 1` but no `ExitDuration`.
3. Governance passes `MsgUpdateTier` setting Tier 1's `ExitDuration = 730 days`.
4. Alice calls `MsgTriggerExitFromTier`. `TriggerExitFromTier` fetches the live tier and computes `ExitUnlockAt = now + 730 days`.
5. Alice's funds are locked for 730 days instead of the 365 days she agreed to at lock time.
6. `validateWithdrawFromTier` checks `pos.CompletedExitLockDuration(blockTime)` against the now-730-day `ExitUnlockAt`, so withdrawal is blocked for the full extended period. [7](#0-6) [4](#0-3)

func (ms msgServer) TriggerExitFromTier(ctx context.Context, msg *types.MsgTriggerExitFromTier) (*types.MsgTriggerExitFromTierResponse, error) {
	if err := msg.Validate(); err != nil {
		return nil, err
	}

	pos, err := ms.getPositionState(ctx, msg.PositionId)
	if err != nil {
		return nil, err
	}

	if err := ms.validateTriggerExit(pos.Position, msg.Owner); err != nil {
		return nil, err
	}

	tier, err := ms.getTier(ctx, pos.TierId)
	if err != nil {
		return nil, err
	}

	sdkCtx := sdk.UnwrapSDKContext(ctx)
	pos.TriggerExit(sdkCtx.BlockTime(), tier.ExitDuration)

	if err := ms.setPosition(ctx, pos.Position, nil); err != nil {
		return nil, err
	}

	if err := sdkCtx.EventManager().EmitTypedEvent(&types.EventExitTriggered{
		PositionId:   pos.Id,
		TierId:       pos.TierId,
		Owner:        pos.Owner,
		ExitUnlockAt: pos.ExitUnlockAt,
	}); err != nil {
		return nil, err
	}

	return &types.MsgTriggerExitFromTierResponse{
		ExitUnlockAt: pos.ExitUnlockAt,
		PositionId:   pos.Id,
	}, nil
}
