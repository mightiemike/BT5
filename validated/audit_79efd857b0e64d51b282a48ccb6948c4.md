Audit Report

## Title
`SwapAllowlistExtension` checks router address instead of end-user, allowing any caller to bypass per-pool swap allowlist via `MetricOmmSimpleRouter` - (`metric-periphery/contracts/extensions/SwapAllowlistExtension.sol`)

## Summary
`SwapAllowlistExtension.beforeSwap()` checks the `sender` argument forwarded by the pool, which is always `msg.sender` of `MetricOmmPool.swap()`. When a user routes through `MetricOmmSimpleRouter`, the router is the direct caller of `pool.swap()`, so `sender = router address`. If the pool admin allowlists the router to enable router-mediated swaps, the allowlist check passes for every caller regardless of identity, completely bypassing the intended per-user access control.

## Finding Description
`MetricOmmPool.swap()` passes `msg.sender` as the `sender` argument to `_beforeSwap()`:

```solidity
// metric-core/contracts/MetricOmmPool.sol L230-240
_beforeSwap(
    msg.sender,   // always the direct caller of pool.swap()
    recipient,
    ...
);
```

`SwapAllowlistExtension.beforeSwap()` uses that `sender` value to look up the allowlist:

```solidity
// metric-periphery/contracts/extensions/SwapAllowlistExtension.sol L37
if (!allowAllSwappers[msg.sender] && !allowedSwapper[msg.sender][sender]) {
    revert IMetricOmmPoolActions.NotAllowedToSwap();
}
```

`msg.sender` inside the extension is the pool (correct). `sender` is whoever called `pool.swap()`.

All four router entry points call `pool.swap()` directly, making the router the `msg.sender` of each call:

- `exactInputSingle()` — [1](#0-0) 
- `exactInput()` — [2](#0-1) 
- `exactOutputSingle()` — [3](#0-2) 
- `exactOutput()` — [4](#0-3) 

In every path, `sender = router address` reaches the extension. The extension never sees the end-user address. The pool admin, wanting to allow legitimate router-mediated swaps for allowlisted users, must set `allowedSwapper[pool][router] = true`. Once set, `allowedSwapper[pool][router] == true` passes the guard for every caller, including those explicitly excluded. [5](#0-4) 

## Impact Explanation
A pool configured with `SwapAllowlistExtension` to restrict swaps to specific addresses (KYC-gated, institutional-only, whitelist-only) is completely open to any caller who routes through `MetricOmmSimpleRouter`. The router is a public, permissionless contract with no access control of its own. Any address — including those the pool admin explicitly excluded — can call `exactInputSingle` or any other router function and execute swaps against the restricted pool at oracle-quoted prices. The allowlist guard is rendered inoperative, breaking the core access-control invariant the pool admin configured and allowing unauthorized parties to trade against restricted pool liquidity. This constitutes a broken core pool functionality causing potential loss of funds and an admin-boundary break where the pool admin's access control configuration is bypassed by an unprivileged path.

## Likelihood Explanation
The `MetricOmmSimpleRouter` is the standard user-facing entry point for swaps. Any pool admin who wants router-mediated swaps to work for their allowlisted users must allowlist the router address. Once that is done, the bypass is trivially reachable by any address with no special privileges, no malicious setup, no non-standard tokens — just a normal call to a public router function. The precondition (router allowlisted) is a necessary operational step for any pool using both `SwapAllowlistExtension` and the router, making this a near-certain occurrence in production.

## Recommendation
The extension must receive and check the economically relevant actor (the end-user), not the technical `msg.sender` of `pool.swap()`. Two complementary fixes:

1. **In `MetricOmmSimpleRouter`**: encode `msg.sender` (the true end-user) into `extensionData` for each `pool.swap()` call. The extension would decode and verify the user address against the allowlist rather than the `sender` argument. This is more robust as it preserves full call-chain identity without changing the core pool interface.
2. **In `SwapAllowlistExtension`**: check `recipient` (the address receiving output tokens) instead of `sender` as a simpler alternative, though this is weaker since `recipient` can be set to any address by the caller.

## Proof of Concept
```
Setup:
  pool configured with SwapAllowlistExtension
  allowedSwapper[pool][router] = true   (admin enables router-mediated swaps)
  allowedSwapper[pool][alice]  = true   (alice is an approved user)
  allowedSwapper[pool][bob]    = false  (bob is explicitly excluded)

Attack:
  bob calls MetricOmmSimpleRouter.exactInputSingle({pool: pool, ...})
    → router calls pool.swap(recipient=bob, ...)          [MetricOmmSimpleRouter.sol L72-80]
    → pool calls _beforeSwap(sender=router, ...)          [MetricOmmPool.sol L230-240]
    → SwapAllowlistExtension.beforeSwap(sender=router, ...)
    → checks allowedSwapper[pool][router] == true  ✓      [SwapAllowlistExtension.sol L37]
    → swap proceeds — bob receives output tokens

Result:
  bob, explicitly excluded from the allowlist, executes a swap
  against the restricted pool. The allowlist guard is completely bypassed.
```
