### Title
`TryFrom<protobuf::ResourceBounds> for ValidResourceBounds` Silently Downgrades `AllResources` to `L1Gas`, Breaking P2P Mempool Round-Trip for Valid Invoke Transactions - (`crates/apollo_protobuf/src/converters/transaction.rs`)

---

### Summary

The protobuf deserialization of `ValidResourceBounds` in `crates/apollo_protobuf/src/converters/transaction.rs` unconditionally downgrades any `AllResources` variant whose `l2_gas` and `l1_data_gas` are both zero into the `L1Gas` variant. A valid `RpcInvokeTransactionV3` submitted with `AllResourceBounds { l1_gas: X, l2_gas: 0, l1_data_gas: 0 }` is accepted by the gateway, serialized to protobuf, and then silently type-shifted on deserialization. The downstream `TryFrom<InvokeTransactionV3> for RpcInvokeTransactionV3` rejects any `L1Gas` variant with `DEPRECATED_RESOURCE_BOUNDS_ERROR`, so every P2P peer that receives the transaction via mempool propagation drops it before sequencing.

---

### Finding Description

**Root cause — `TryFrom<protobuf::ResourceBounds> for ValidResourceBounds`:**

```rust
// crates/apollo_protobuf/src/converters/transaction.rs  lines 417-436
impl TryFrom<protobuf::ResourceBounds> for ValidResourceBounds {
    fn try_from(value: protobuf::ResourceBounds) -> Result<Self, Self::Error> {
        ...
        let l1_data_gas = value.l1_data_gas.unwrap_or_default();
        ...
        Ok(if l1_data_gas.is_zero() && l2_gas.is_zero() {
            ValidResourceBounds::L1Gas(l1_gas)          // ← downgrade
        } else {
            ValidResourceBounds::AllResources(AllResourceBounds { l1_gas, l2_gas, l1_data_gas })
        })
    }
}
```

This conversion is used inside `TryFrom<protobuf::InvokeV3> for InvokeTransactionV3` (line 596-598), which is the inner step of the mempool P2P deserialization chain:

```
protobuf::MempoolTransaction
  → TryFrom<protobuf::InvokeV3WithProof> for RpcInvokeTransactionV3   (rpc_transaction.rs:115-131)
      → TryFrom<protobuf::InvokeV3> for InvokeTransactionV3            (transaction.rs:593-660)
          → ValidResourceBounds::try_from(protobuf::ResourceBounds)    (transaction.rs:417-436)  ← downgrade here
      → TryFrom<InvokeTransactionV3> for RpcInvokeTransactionV3        (rpc_transaction.rs:586-611)
          → FAILS if resource_bounds == L1Gas                           ← error here
```

**The broken invariant:**

`RpcInvokeTransactionV3` stores resource bounds as `AllResourceBounds` (always three resources). The `From<RpcInvokeTransactionV3> for InvokeTransactionV3` conversion wraps it as `ValidResourceBounds::AllResources(...)`. After protobuf serialization the wire bytes carry `l2_gas=0, l1_data_gas=0`. On deserialization the condition `l1_data_gas.is_zero() && l2_gas.is_zero()` is true, so the type is changed to `ValidResourceBounds::L1Gas`. The reverse conversion `TryFrom<InvokeTransactionV3> for RpcInvokeTransactionV3` then hard-rejects `L1Gas`:

```rust
// crates/starknet_api/src/rpc_transaction.rs  lines 591-597
resource_bounds: match value.resource_bounds {
    ValidResourceBounds::AllResources(bounds) => bounds,
    _ => {
        return Err(StarknetApiError::OutOfRange { string: "resource_bounds".to_string() });
    }
},
```

The comment in `TryFrom<protobuf::InvokeV3WithProof> for RpcInvokeTransactionV3` (line 129) reads:

> "This conversion can fail only if the resource_bounds are not AllResources."

The developers assumed `AllResources` is always preserved across the protobuf round-trip, but the downgrade in `TryFrom<protobuf::ResourceBounds> for ValidResourceBounds` violates that assumption.

**Serialization path (no loss):**

`From<ValidResourceBounds> for protobuf::ResourceBounds` (transaction.rs lines 471-489) correctly serializes `AllResources` with zero `l2_gas`/`l1_data_gas` as three explicit zero fields. The loss only occurs on the deserialization side.

**Hash domain divergence (secondary):**

`get_tip_resource_bounds_hash` (transaction_hash.rs lines 188-211) produces different digests for the two variants:

- `L1Gas` → `poseidon(tip, l1_gas_packed, l2_gas_packed)` — two resource felts
- `AllResources` → `poseidon(tip, l1_gas_packed, l2_gas_packed, l1_data_gas_packed)` — three resource felts

If any code path were to recompute the hash after the downgrade, it would produce a hash that does not match the one stored in `InternalRpcTransaction.tx_hash`, binding the wrong hash to the executable payload.

---

### Impact Explanation

Any valid `RpcInvokeTransactionV3` with `AllResourceBounds { l1_gas: non-zero, l2_gas: 0, l1_data_gas: 0 }` — a configuration explicitly accepted by the gateway's stateless validator (confirmed by test case `valid_l1_gas` in `stateless_transaction_validator_test.rs`) — is silently dropped by every P2P peer that receives it via mempool propagation. The transaction exists only in the mempool of the single node that accepted it via RPC. This matches:

> **High. Mempool/gateway/RPC admission accepts invalid transactions or rejects valid transactions before sequencing.**

---

### Likelihood Explanation

The trigger requires only that a user submit a standard invoke transaction specifying only L1 gas (zero L2 gas and zero L1 data gas). This is a normal pre-0.13.3 usage pattern that the gateway explicitly permits. No special privilege or knowledge of internals is required.

---

### Recommendation

In `TryFrom<protobuf::ResourceBounds> for ValidResourceBounds` (`crates/apollo_protobuf/src/converters/transaction.rs`), do not downgrade to `L1Gas` when all three resource fields are present in the protobuf message. Reserve the `L1Gas` variant only for legacy wire messages that genuinely omit `l2_gas` and `l1_data_gas` (i.e., when `value.l1_data_gas` is `None` and `value.l2_gas` is `None`). When all three fields are present — even if zero — reconstruct `AllResources`:

```rust
Ok(match (value.l1_data_gas, value.l2_gas) {
    (None, _) | (_, None) if l2_gas.is_zero() => ValidResourceBounds::L1Gas(l1_gas),
    _ => ValidResourceBounds::AllResources(AllResourceBounds { l1_gas, l2_gas, l1_data_gas }),
})
```

This preserves the round-trip invariant that `AllResources` serialized to protobuf always deserializes back to `AllResources`.

### Proof of Concept

```
1. User submits RpcInvokeTransactionV3 with:
       resource_bounds = AllResourceBounds {
           l1_gas:      ResourceBounds { max_amount: 1000, max_price_per_unit: 1 },
           l2_gas:      ResourceBounds { max_amount: 0,    max_price_per_unit: 0 },
           l1_data_gas: ResourceBounds { max_amount: 0,    max_price_per_unit: 0 },
       }

2. Gateway accepts it (stateless validator: valid_l1_gas case).
   TransactionConverter::convert_rpc_tx_to_internal() stores it as
   InternalRpcTransactionWithoutTxHash::Invoke(InternalRpcInvokeTransactionV3 {
       resource_bounds: AllResourceBounds { ... },   // AllResources preserved
       ...
   })

3. MempoolP2pPropagator serializes via:
   RpcTransaction → From<RpcInvokeTransactionV3> for InvokeTransactionV3
                  → resource_bounds = ValidResourceBounds::AllResources(...)
                  → From<ValidResourceBounds> for protobuf::ResourceBounds
                  → { l1_gas: 1000/1, l2_gas: 0/0, l1_data_gas: 0/0 }   (all present, all zero for l2/l1d)

4. Receiving peer deserializes via:
   TryFrom<protobuf::InvokeV3> for InvokeTransactionV3
     → ValidResourceBounds::try_from(protobuf::ResourceBounds)
     → l1_data_gas.is_zero() && l2_gas.is_zero() == true
     → returns ValidResourceBounds::L1Gas(l1_gas)   ← DOWNGRADED

5. TryFrom<InvokeTransactionV3> for RpcInvokeTransactionV3
     → match value.resource_bounds { ValidResourceBounds::AllResources(b) => b, _ => Err(...) }
     → Err(OutOfRange { "resource_bounds" })   ← REJECTED

6. Transaction is silently dropped by all P2P peers.
   Only the originating node holds it in its mempool.
```

```rust
    type Error = ProtobufConversionError;
```

```rust
        // Extract proof first, since `starknet_api::transaction::InvokeTransactionV3` does not
        // carry a `proof` field.

        let snapi_invoke: InvokeTransactionV3 = value
            .invoke
            .ok_or(ProtobufConversionError::MissingField {
                field_description: "InvokeV3WithProof::invoke",
            })?
            .try_into()?;

        // This conversion can fail only if the resource_bounds are not AllResources.
        Ok(Self { proof, ..snapi_invoke.try_into().map_err(|_| DEPRECATED_RESOURCE_BOUNDS_ERROR)? })
    }
```

```rust
pub fn get_tip_resource_bounds_hash(
    resource_bounds: &ValidResourceBounds,
    tip: &Tip,
) -> Result<Felt, StarknetApiError> {
    let l1_resource_bounds = resource_bounds.get_l1_bounds();
    let l2_resource_bounds = resource_bounds.get_l2_bounds();

    // L1 and L2 gas bounds always exist.
    // Old V3 txs always have L2 gas bounds of zero, but they exist.
    let mut resource_felts = vec![
        get_concat_resource(&l1_resource_bounds, L1_GAS)?,
        get_concat_resource(&l2_resource_bounds, L2_GAS)?,
    ];

    // For new V3 txs, need to also hash the data gas bounds.
    resource_felts.extend(match resource_bounds {
        ValidResourceBounds::L1Gas(_) => vec![],
        ValidResourceBounds::AllResources(all_resources) => {
            vec![get_concat_resource(&all_resources.l1_data_gas, L1_DATA_GAS)?]
        }
    });

    Ok(HashChain::new().chain(&tip.0.into()).chain_iter(resource_felts.iter()).get_poseidon_hash())
}
```
