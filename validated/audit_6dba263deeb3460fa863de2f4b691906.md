### Title
Missing `HasReceivingRedelegation` guard on owner in exit path allows slash escape on returned delegation — (`x/tieredrewards/keeper/transfer_delegation.go`)

### Summary
`transferDelegationFromPosition` / `ExitTierWithDelegation` does not check whether the owner has an active incoming redelegation to the position's validator. The symmetric guard present in `transferDelegationToPosition` (which calls `HasReceivingRedelegation(ctx, owner, valAddr)`) is absent in the exit path. An owner who has a pending redelegation from valB → valA can call `ExitTierWithDelegation` on a position at valA, receiving a fresh direct delegation at valA that is not covered by the pending redelegation's slash liability.

### Finding Description
`transferDelegationToPosition` explicitly blocks the entry path when the owner has an active incoming redelegation to the target validator:

```go
// transfer_delegation.go:60-66
hasRedel, err := k.stakingKeeper.HasReceivingRedelegation(ctx, from, valAddr)
if hasRedel {
    return math.LegacyDec{}, types.ErrActiveRedelegation
}
```

The exit path (`transferDelegationFromPosition`) only checks whether the *position's* delegator address is redelegating:

```go
// transfer_delegation.go:117-123
isRedelegating, err := k.isRedelegating(ctx, pos.DelegatorAddress)
if isRedelegating {
    return ..., errorsmod.Wrapf(types.ErrActiveRedelegation, ...)
}
```

`isRedelegating` calls `GetRedelegations(ctx, posDelAddr, 1)` — the position's sub-account, which never has its own redelegations. The owner's pending redelegation from valB → valA is invisible to this check. The call then proceeds to `Delegate(ctx, owner, transferredAmount, valA, false)`, creating a new direct delegation for the owner at valA outside the redelegation entry.

### Impact Explanation
When valB is subsequently slashed for an infraction that predates the redelegation, the Cosmos SDK slash logic propagates the slash to the redelegation entry at valA. The owner's new direct delegation at valA (returned from the position) is a separate staking record and is not touched by that propagation. The owner effectively receives `transferredAmount` tokens at valA shielded from the pending slash fraction.

Concrete delta: if the slash fraction is `f` and the transferred amount is `T`, the owner retains `T` instead of `T*(1-f)` — a direct token gain of `T*f` at the expense of the protocol/other delegators who absorb the slash.

### Likelihood Explanation
The preconditions are reachable through standard Cosmos SDK transactions with no privileged access:
1. `MsgBeginRedelegate(owner, valB → valA)` — standard staking tx
2. Wait (redelegation still pending, not yet mature)
3. `MsgExitTierWithDelegation(positionId, amount)` — position at valA, exit lock elapsed

The redelegation maturity window is typically 21 days, giving a wide window. The attacker only needs to know a slash event is likely (e.g., double-sign evidence is pending).

### Recommendation
Add a `HasReceivingRedelegation` check on the owner against the position's validator inside `validateExitTierWithDelegation` (or at the top of `transferDelegationFromPosition`), mirroring the guard already present in `transferDelegationToPosition`:

```go
hasRedel, err := k.stakingKeeper.HasReceivingRedelegation(ctx, owner, valAddr)
if err != nil { return err }
if hasRedel {
    return errorsmod.Wrapf(types.ErrActiveRedelegation,
        "owner has an active incoming redelegation to validator %s", valAddr)
}
```

### Proof of Concept
1. Setup: bonded valA, bonded valB; owner has delegation at valB.
2. `MsgBeginRedelegate(owner, valB → valA)` — creates pending redelegation entry.
3. Create a tiered position at valA (via `CommitDelegationToTier` or `LockTier`).
4. Trigger exit, advance time past exit lock duration (but keep redelegation pending).
5. Call `ExitTierWithDelegation(positionId, amount)` — succeeds because only `pos.DelegatorAddress` is checked.
6. Slash valB for a past infraction — SDK propagates slash to the redelegation entry at valA.
7. Assert: owner's direct delegation at valA (from step 5) retains full `amount`; only the redelegation entry shares are reduced. Owner's net tokens at valA exceed what they should hold post-slash.
