### Title
`ValidResourceBounds::AllResources` with zero L2/L1DataGas silently downgrades to `L1Gas` variant through protobuf round-trip, producing a divergent transaction hash and causing P2P rejection of valid transactions — (`crates/apollo_protobuf/src/converters/transaction.rs`)

---

### Summary

A V3 invoke transaction submitted with `AllResourceBounds { l1_gas: X, l2_gas: 0, l1_data_gas: 0 }` is accepted by the gateway and hashed using the `AllResources` branch of `get_tip_resource_bounds_hash` (which includes the `L1_DATA_GAS` resource name encoding). When the transaction is serialized to protobuf for mempool P2P propagation and deserialized by a receiving peer, the `TryFrom<protobuf::ResourceBounds> for ValidResourceBounds` converter silently downgrades the variant to `ValidResourceBounds::L1Gas`. The receiving peer then fails to reconstruct `RpcInvokeTransactionV3` (which requires `AllResources`) and drops the transaction with `DEPRECATED_RESOURCE_BOUNDS_ERROR`. Any node that does reconstruct the transaction from the downgraded variant computes a different hash, breaking the hash-identity invariant.

---

### Finding Description

**Step 1 — Originating gateway accepts and hashes with `AllResources`.**

`RpcInvokeTransactionV3.resource_bounds` is typed as `AllResourceBounds` (never `L1Gas`). When `convert_rpc_tx_to_internal` calls `tx_without_hash.calculate_transaction_hash`, it reaches `get_invoke_transaction_v3_hash`, which calls `get_tip_resource_bounds_hash` with `ValidResourceBounds::AllResources(AllResourceBounds { l1_gas: X, l2_gas: 0, l1_data_gas: 0 })`. [1](#0-0) 

Inside `get_tip_resource_bounds_hash`, the `AllResources` branch appends a third element — the `L1_DATA_GAS` resource name encoding — to the hash chain, even when `l1_data_gas` amounts are zero: [2](#0-1) 

The `L1Gas` branch does **not** append this element, so the two variants produce structurally different Poseidon hashes for the same numeric values.

**Step 2 — Protobuf serialization preserves zero values.**

`From<ValidResourceBounds> for protobuf::ResourceBounds` serializes `AllResources` with zero `l2_gas` and `l1_data_gas` as explicit zero `ResourceLimits` fields: [3](#0-2) 

**Step 3 — Protobuf deserialization silently changes the variant.**

`TryFrom<protobuf::ResourceBounds> for ValidResourceBounds` uses a value-based heuristic: if both `l2_gas` and `l1_data_gas` are zero, it returns `ValidResourceBounds::L1Gas`, discarding the `AllResources` identity: [4](#0-3) 

**Step 4 — Receiving peer rejects the transaction.**

`TryFrom<protobuf::InvokeV3WithProof> for RpcInvokeTransactionV3` calls `snapi_invoke.try_into()` to convert the deserialized `InvokeTransactionV3` (now carrying `L1Gas`) back to `RpcInvokeTransactionV3`. Because `RpcInvokeTransactionV3.resource_bounds` is typed as `AllResourceBounds`, this conversion fails when the variant is `L1Gas`, returning `DEPRECATED_RESOURCE_BOUNDS_ERROR`: [5](#0-4) 

The transaction is dropped by every receiving peer.

---

### Impact Explanation

A valid V3 invoke transaction with `l2_gas = 0` and `l1_data_gas = 0` (accepted by the stateless validator, as confirmed by the `valid_l1_gas` test case) is permanently rejected by all peers that receive it via mempool P2P. The originating node holds the transaction in its local mempool but cannot propagate it. If the originating node is not the current proposer, the transaction is never sequenced. If a peer somehow reconstructs the transaction from the downgraded `L1Gas` variant, it computes a hash that differs from the one stored in `InternalRpcTransaction.tx_hash`, breaking the hash-identity invariant used for deduplication, receipt lookup, and signature binding.

**Matching impact**: *High — Mempool/gateway/RPC admission rejects valid transactions before sequencing.*

---

### Likelihood Explanation

Any user submitting a V3 invoke transaction that specifies only L1 gas (setting L2 gas and L1 data gas to zero) triggers this path. This is a common pattern for users who do not need L2 gas or data gas. The stateless validator explicitly accepts such transactions. No special privileges or malicious intent are required; the bug is triggered by ordinary, well-formed input.

---

### Recommendation

**Short term**: In `TryFrom<protobuf::ResourceBounds> for ValidResourceBounds`, do not use a value-based heuristic to select the variant. Instead, add an explicit discriminant field to the protobuf `ResourceBounds` message (e.g., a boolean `is_all_resources` or an enum `bounds_type`) and use it to preserve the original variant identity across serialization boundaries.

**Long term**: Audit all `ValidResourceBounds` serialization/deserialization paths (protobuf, JSON, storage) to ensure the `L1Gas` vs `AllResources` distinction is preserved as a first-class field, not inferred from numeric values. Add a round-trip test that asserts `AllResources { l1_gas: X, l2_gas: 0, l1_data_gas: 0 }` survives protobuf serialization with its variant and hash unchanged.

---

### Proof of Concept

1. Construct a V3 invoke transaction with `AllResourceBounds { l1_gas: ResourceBounds { max_amount: 100, max_price_per_unit: 1 }, l2_gas: ResourceBounds::default(), l1_data_gas: ResourceBounds::default() }`.

2. Submit it to the gateway. The gateway calls `get_invoke_transaction_v3_hash` → `get_tip_resource_bounds_hash` with `AllResources`. The hash chain includes `get_concat_resource(&zero_bounds, L1_DATA_GAS)` — a non-zero felt encoding the `"L1_DATA"` resource name — producing hash **H_all**.

3. The gateway propagates the transaction via mempool P2P. `RpcInvokeTransactionV3 → InvokeTransactionV3 → protobuf::InvokeV3` serializes `l2_gas = 0`, `l1_data_gas = 0`.

4. The receiving peer deserializes: `protobuf::InvokeV3 → InvokeTransactionV3` with `resource_bounds = ValidResourceBounds::L1Gas(X)` (because `l1_data_gas.is_zero() && l2_gas.is_zero()`).

5. `InvokeTransactionV3.try_into::<RpcInvokeTransactionV3>()` fails → `DEPRECATED_RESOURCE_BOUNDS_ERROR` → transaction dropped.

6. If the receiving peer were to compute the hash from the downgraded `L1Gas` variant, `get_tip_resource_bounds_hash` would produce hash **H_l1** ≠ **H_all** (the `L1_DATA_GAS` element is absent), confirming the hash divergence. [6](#0-5) [1](#0-0) [7](#0-6) [8](#0-7) [9](#0-8)

pub fn get_tip_resource_bounds_hash(
    resource_bounds: &ValidResourceBounds,
    tip: &Tip,
) -> Result<Felt, StarknetApiError> {
    let l1_resource_bounds = resource_bounds.get_l1_bounds();
    let l2_resource_bounds = resource_bounds.get_l2_bounds();

    // L1 and L2 gas bounds always exist.
    // Old V3 txs always have L2 gas bounds of zero, but they exist.
    let mut resource_felts = vec![
        get_concat_resource(&l1_resource_bounds, L1_GAS)?,
        get_concat_resource(&l2_resource_bounds, L2_GAS)?,
    ];

    // For new V3 txs, need to also hash the data gas bounds.
    resource_felts.extend(match resource_bounds {
        ValidResourceBounds::L1Gas(_) => vec![],
        ValidResourceBounds::AllResources(all_resources) => {
            vec![get_concat_resource(&all_resources.l1_data_gas, L1_DATA_GAS)?]
        }
    });

    Ok(HashChain::new().chain(&tip.0.into()).chain_iter(resource_felts.iter()).get_poseidon_hash())
}

            return Err(missing("ResourceBounds::l1_gas"));

        let Some(l2_gas) = value.l2_gas else {

            ValidResourceBounds::L1Gas(l1_gas)
        } else {
            ValidResourceBounds::AllResources(AllResourceBounds { l1_gas, l2_gas, l1_data_gas })
        })
    }

impl TryFrom<protobuf::InvokeV3WithProof> for RpcInvokeTransactionV3 {
    type Error = ProtobufConversionError;
    fn try_from(mut value: protobuf::InvokeV3WithProof) -> Result<Self, Self::Error> {
        // Extract proof first, since `starknet_api::transaction::InvokeTransactionV3` does not
        // carry a `proof` field.
        let proof = Proof::from(std::mem::take(&mut value.proof));

        let snapi_invoke: InvokeTransactionV3 = value
            .invoke
            .ok_or(ProtobufConversionError::MissingField {
                field_description: "InvokeV3WithProof::invoke",
            })?
            .try_into()?;

        // This conversion can fail only if the resource_bounds are not AllResources.
        Ok(Self { proof, ..snapi_invoke.try_into().map_err(|_| DEPRECATED_RESOURCE_BOUNDS_ERROR)? })
    }
