### Title
Bonus Accrual Checkpoint Advanced Before Pool Balance Check Causes Permanent Reward Loss on Partial Redelegation Slash — (File: `x/tieredrewards/keeper/claim_rewards.go`, `x/tieredrewards/keeper/slash.go`)

---

### Summary

In `processEventsAndClaimBonus`, `applyBonusAccrualCheckpoint` unconditionally advances `pos.LastBonusAccrual` **before** the `sufficientBonusPoolBalance` check. In the `slashRedelegationPosition` hook, `ErrInsufficientBonusPool` is deliberately swallowed to avoid a chain halt, and the position is then persisted with the already-advanced checkpoint. For partial redelegation slashes, this permanently forfeits the user's accrued bonus rewards — the period is marked as processed but no coins are transferred.

---

### Finding Description

`processEventsAndClaimBonus` computes the total bonus, then unconditionally advances the accrual checkpoint, and only afterward checks whether the pool can cover the payout:

```go
// x/tieredrewards/keeper/claim_rewards.go
applyBonusAccrualCheckpoint(&pos.Position, blockTime)   // line 215 — always runs
pos.UpdateLastKnownBonded(bonded)                        // line 217 — always runs

if totalBonus.IsZero() {
    return sdk.NewCoins(), nil
}
...
if err := k.sufficientBonusPoolBalance(ctx, bonusCoins); err != nil {
    return nil, err   // returns error AFTER checkpoint was already advanced in memory
}
```

For user-driven message paths (`MsgClaimTierRewards`, `MsgTierRedelegate`, etc.) this ordering is harmless: the entire Cosmos SDK transaction reverts atomically on error, so the in-memory checkpoint change is never persisted.

The slash hook path is different. `slashRedelegationPosition` explicitly swallows `ErrInsufficientBonusPool` to prevent a chain halt, then unconditionally calls `setPositionWithState` to persist the updated position:

```go
// x/tieredrewards/keeper/slash.go
if _, err := k.processEventsAndClaimBonus(ctx, &pos); err != nil {
    if errors.Is(err, types.ErrInsufficientBonusPool) {
        k.logger(ctx).Error("insufficient bonus pool ...")
        // error swallowed — execution continues
    } else {
        return err
    }
}

fullSlash := sharesToUnbond.GTE(pos.Delegation.Shares)

if fullSlash {
    pos.Delegation = nil
    pos.ClearBonusCheckpoints()   // full slash: checkpoint zeroed — OK
    return k.setPositionWithState(ctx, pos, &ValidatorTransition{...})
}
// PARTIAL SLASH PATH:
pos.Delegation.Shares = pos.Delegation.Shares.Sub(sharesToUnbond)
return k.setPositionWithState(ctx, pos, nil)   // persists advanced LastBonusAccrual
```

For a **partial slash** with an insufficient pool:

1. `processEventsAndClaimBonus` advances `pos.LastBonusAccrual` to `blockTime` in memory.
2. `sufficientBonusPoolBalance` returns `ErrInsufficientBonusPool`.
3. The error is swallowed in `slashRedelegationPosition`.
4. `pos.LastBonusAccrual == blockTime` is now the in-memory state.
5. `setPositionWithState(ctx, pos, nil)` persists the advanced checkpoint.

The accrued bonus for the entire period `[old_LastBonusAccrual, blockTime]` is permanently lost. The pool retains the coins, but the position's checkpoint has moved past the period, so no future claim can recover them.

For a **full slash**, `ClearBonusCheckpoints()` zeroes `LastBonusAccrual` after the swallowed error, so the checkpoint is reset correctly. The partial-slash branch has no such correction.

---

### Impact Explanation

A position owner whose redelegation entry is partially slashed while the rewards pool is insufficient permanently loses all bonus rewards accrued since their last claim. The corrupted value is `pos.LastBonusAccrual` — it is advanced to `blockTime` without a corresponding `SendCoinsFromModuleToAccount`. The pool balance is not reduced, but the user can never reclaim the forfeited bonus because the accrual window is closed.

**Impact: Medium** — direct, permanent loss of earned bonus rewards for affected position owners.

---

### Likelihood Explanation

The trigger requires two concurrent conditions:
1. A validator slash that affects a tier position currently in a redelegation (i.e., `BeforeRedelegationSlashed` fires with a matching `RedelegationMappings` entry).
2. The `tieredrewards` rewards pool balance is insufficient to cover the accrued bonus at the time of the slash.

Condition 1 is a normal protocol event (validator misbehavior/jailing). Condition 2 can occur if the pool is depleted by prior payouts or was never adequately funded. Neither condition requires privileged access or attacker control — a delegator who redelegates to a validator that is later slashed while the pool is low will silently lose their bonus.

**Likelihood: Low-Medium** — both conditions must coincide, but neither is exotic.

---

### Recommendation

Move `applyBonusAccrualCheckpoint` and `UpdateLastKnownBonded` to execute **only after** a successful `SendCoinsFromModuleToAccount`, or alternatively, in `slashRedelegationPosition`, explicitly reset `pos.LastBonusAccrual` back to its pre-call value when `ErrInsufficientBonusPool` is caught on a partial slash, so the user can reclaim the bonus once the pool is replenished.

---

### Proof of Concept

**Sequence:**

1. User calls `MsgTierRedelegate` — position enters redelegation, `RedelegationMappings[unbondingId] = positionId` is set.
2. Time passes; bonus accrues. Pool balance is drained to near zero by other claimants.
3. The destination validator is slashed. The staking module calls `BeforeRedelegationSlashed(ctx, unbondingId, partialShares)`.
4. `slashRedelegationPosition` is invoked. `processEventsAndClaimBonus` runs:
   - `applyBonusAccrualCheckpoint` sets `pos.LastBonusAccrual = blockTime` in memory.
   - `sufficientBonusPoolBalance` returns `ErrInsufficientBonusPool`.
5. Error is swallowed. `pos.Delegation.Shares` is reduced by `partialShares`.
6. `setPositionWithState(ctx, pos, nil)` writes `LastBonusAccrual = blockTime` to the store.
7. User later calls `MsgClaimTierRewards`. `processEventsAndClaimBonus` starts from `segmentStart = blockTime` — the entire prior accrual window is gone. Zero bonus is paid.

func applyBonusAccrualCheckpoint(pos *types.Position, blockTime time.Time) {
	accrualEnd := blockTime
	if pos.CompletedExitLockDuration(blockTime) {
		accrualEnd = pos.ExitUnlockAt
	}
	pos.UpdateLastBonusAccrual(accrualEnd)
}
