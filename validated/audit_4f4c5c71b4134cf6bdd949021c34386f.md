### Title
`node_from_bytes_backrefs` Accepts Non-Canonical Backref Path Encodings That `is_canonical_serialization` Rejects — (`src/serde/de_br.rs`, `src/serde/tools.rs`)

### Summary

`node_from_bytes_backrefs` and `is_canonical_serialization` disagree on the validity of backref path atoms with non-canonical length-prefix encodings (e.g., `0x81 0x01` instead of `0x01`). Two distinct byte streams deserialize to the same CLVM tree, violating the one-canonical-stream invariant.

---

### Finding Description

**`is_canonical_serialization` path for a backref:**

In `tools.rs`, when a `BACK_REFERENCE` (`0xfe`) byte is encountered, the next byte is read and passed to `is_canonical_atom`: [1](#0-0) 

`is_canonical_atom` enforces that a 1-byte-prefixed atom (`0x81`–`0xbf`) with `atom_len == 1` must have its single content byte `>= 0x80`, otherwise the encoding is non-canonical (the value could have been expressed as a bare single byte): [2](#0-1) 

For input `0x81 0x01`: `prefix_len=1`, `atom_len=1`, content byte `0x01 < 0x80` → `is_canonical_atom` returns `false` → `is_canonical_serialization` returns `false`.

---

**`node_from_bytes_backrefs` path for the same backref:**

When `0xfe` is encountered, `parse_path` is called: [3](#0-2) 

`parse_path` delegates to `parse_atom_ptr` with no canonicality check: [4](#0-3) 

`parse_atom_ptr` with `first_byte = 0x81` calls `decode_size(f, 0x81)` → `atom_size = 1`, reads the next byte `0x01`, and returns `&[0x01]`: [5](#0-4) 

This is **byte-for-byte identical** to what the canonical encoding `0x01` produces (a single-byte path `[0x01]`). `traverse_path_with_vec` then resolves the backref normally with no error.

### Proof of Concept

**Stream A (canonical):** `[0xff, 0x01, 0xfe, 0x01]`
- `is_canonical_serialization` → `true`
- `node_from_bytes_backrefs` → `Ok((1 . (1 . ())))`

**Stream B (non-canonical path encoding):** `[0xff, 0x01, 0xfe, 0x81, 0x01]`
- `is_canonical_serialization` → `false` (rejects at `is_canonical_atom` because `0x01 < 0x80`)
- `node_from_bytes_backrefs` → `Ok((1 . (1 . ())))` — **same tree, no error**

The path bytes extracted from both `0x01` and `0x81 0x01` are identical (`[0x01]`), so `traverse_path_with_vec` produces the same result in both cases. [6](#0-5) 

---

### Impact Explanation

The one-canonical-stream invariant is broken: two distinct byte streams deserialize to the same CLVM tree. Any system component that gates on `is_canonical_serialization` (exported from `src/serde/mod.rs`) before calling `node_from_bytes_backrefs` will reject Stream B, while a component that skips the canonical check will accept it and produce an identical tree. This creates a split between nodes depending on which validation path they use — a consensus-layer discrepancy. [7](#0-6) 

---

### Likelihood Explanation

The attacker only needs to control the serialized bytes passed to the deserialization API — a standard, fully attacker-controlled input surface. The non-canonical encoding is trivially constructible (replace any single-byte path value `v < 0x80` with `0x81 v`). No special privileges or internal access are required.

---

### Recommendation

Add a canonicality check inside `parse_path` (or immediately after it in `node_from_stream_backrefs`) that mirrors the logic in `is_canonical_atom`. Specifically, after `parse_atom_ptr` returns the path bytes, verify that the length-prefix encoding used was minimal — i.e., if the path fits in a single byte `< 0x80`, it must not have been encoded with a multi-byte prefix. Alternatively, unify the path-parsing logic so both functions share a single canonical-aware parser. [4](#0-3) [8](#0-7)

fn is_canonical_atom(f: &mut Cursor<&[u8]>, first_byte: u8) -> bool {
    if first_byte == 0x80 || first_byte <= MAX_SINGLE_BYTE {
        return true;
    }

    let Ok((prefix_len, atom_len)) = decode_size_with_offset(f, first_byte) else {
        return false;
    };

    let min_value = match prefix_len {
        1 => 1,
        2 => 1 << 6,
        3 => 1 << (5 + 8),
        4 => 1 << (4 + 8 + 8),
        5 => 1 << (4 + 8 + 8 + 8),
        6 => 1 << (4 + 8 + 8 + 8 + 8),
        _ => panic!("unexpected atom length prefix {prefix_len}"),
    };

    if atom_len == 1 {
        let mut value = [0_u8];
        if f.read_exact(&mut value).is_err() {
            return false;
        }
        if value[0] < 0x80 {
            return false;
        }
    } else if f.seek(SeekFrom::Current(atom_len as i64)).is_err() {
        return false;
    }
    atom_len >= min_value
}

pub fn is_canonical_serialization(b: &[u8]) -> bool {
    let mut f = Cursor::new(b);
    let mut counter = 1;
    let mut b = [0; 1];
    while counter > 0 {
        counter -= 1;
        if f.read_exact(&mut b).is_err() {
            return false;
        }
        if b[0] == CONS_BOX_MARKER {
            // we expect to parse two more items from the stream
            // the left and right sub tree
            counter += 2;
        } else if b[0] == BACK_REFERENCE {
            // This is a back-ref. We don't actually need to resolve it, just
            // parse the path and move on
            if f.read_exact(&mut b).is_err() {
                return false;
            }
            if !is_canonical_atom(&mut f, b[0]) {
                return false;
            }
        } else if !is_canonical_atom(&mut f, b[0]) {
            return false;
        }
        if (f.get_ref().len() as u64) < f.position() {
            return false;
        }
    }
    f.get_ref().len() as u64 == f.position()
}

fn parse_atom_ptr<'a>(f: &'a mut Cursor<&[u8]>, first_byte: u8) -> Result<&'a [u8]> {
    let blob = if first_byte <= MAX_SINGLE_BYTE {
        let pos = f.position() as usize;
        &f.get_ref()[pos - 1..pos]
    } else {
        let blob_size = decode_size(f, first_byte)?;
        let pos = f.position() as usize;
        if f.get_ref().len() < pos + blob_size as usize {
            return Err(EvalErr::SerializationError);
        }
        f.seek(SeekFrom::Current(blob_size as i64))?;
        &f.get_ref()[pos..(pos + blob_size as usize)]
    };
    Ok(blob)
}
