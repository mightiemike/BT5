Audit Report

## Title
SwapAllowlistExtension Checks Router Address Instead of Originating EOA, Enabling Full Allowlist Bypass - (File: metric-periphery/contracts/extensions/SwapAllowlistExtension.sol)

## Summary
`SwapAllowlistExtension.beforeSwap` gates swaps by checking the `sender` argument, which resolves to `msg.sender` of `pool.swap()`. When swaps are routed through `MetricOmmSimpleRouter`, the pool's `msg.sender` is the router contract, not the originating EOA. A pool admin who allowlists the router address — the natural action to enable router-mediated swaps for a curated pool — inadvertently grants every public user unrestricted swap access, fully bypassing the allowlist.

## Finding Description
`SwapAllowlistExtension.beforeSwap` checks `allowedSwapper[msg.sender][sender]` where `msg.sender` is the calling pool and `sender` is the first argument passed by the pool:

```solidity
// SwapAllowlistExtension.sol L37
if (!allowAllSwappers[msg.sender] && !allowedSwapper[msg.sender][sender]) {
    revert IMetricOmmPoolActions.NotAllowedToSwap();
}
```

`ExtensionCalling._beforeSwap` (L149–177) encodes and forwards `sender` to the extension unchanged. `MetricOmmPool.swap` (L230–240) passes `msg.sender` as `sender`:

```solidity
_beforeSwap(msg.sender, recipient, ...);
```

`MetricOmmSimpleRouter.exactInputSingle` (L72–80) calls `pool.swap(params.recipient, ...)` directly — the router is `msg.sender` to the pool. The originating EOA is stored only in `_setNextCallbackContext` for payment routing and is never forwarded to the pool or extension. `params.extensionData` is passed through but `SwapAllowlistExtension.beforeSwap` ignores it entirely (the `bytes calldata` parameter is unnamed and unused).

**Exploit flow:**
1. Pool admin deploys pool with `SwapAllowlistExtension` as `beforeSwap` hook.
2. Admin calls `setAllowedToSwap(pool, router, true)` to enable router-mediated swaps.
3. Attacker (not individually allowlisted) calls `MetricOmmSimpleRouter.exactInputSingle({pool: curated_pool, recipient: attacker, ...})`.
4. Router calls `pool.swap(attacker, ...)` — pool sees `msg.sender = router`.
5. Extension checks `allowedSwapper[pool][router] == true` → passes.
6. Swap executes; attacker receives output tokens from LP reserves.

**Two broken states exist:**
- **Bypass:** Admin allowlists the router → all public users can swap.
- **Lockout:** Admin allowlists specific EOAs → those EOAs cannot use the router (router not allowlisted → revert).

The "fix" for the lockout (allowlisting the router) is exactly the action that triggers the bypass.

## Impact Explanation
A curated pool deploying `SwapAllowlistExtension` to restrict trading to approved counterparties loses all access control the moment the router is allowlisted. Any public user can execute swaps against the pool's LP reserves at oracle-derived prices. LPs who deposited under the assumption that only vetted counterparties could trade suffer direct loss of principal and protocol fees. This is broken core pool functionality causing loss of funds, meeting Sherlock Critical/High thresholds.

## Likelihood Explanation
The scenario requires a single, well-motivated admin action: `setAllowedToSwap(pool, router, true)`. A pool admin who observes that allowlisted EOAs cannot use the standard router will reach for this exact remediation. No attacker setup, special permissions, or off-chain coordination is required — any address can call `exactInputSingle` on the public router. The bypass is repeatable on every swap after the admin action.

## Recommendation
The extension must gate the economically relevant actor — the originating EOA — not the intermediate contract. Two viable approaches:

1. **Pass the original caller through `extensionData`:** The router encodes `msg.sender` into `extensionData` before forwarding to the pool. The extension decodes and checks this value. Requires a convention between router and extension, and the extension must reject calls where `sender` is a known router but `extensionData` does not contain a valid allowlisted address.

2. **Document and enforce direct-call-only:** Add a revert in `beforeSwap` if `sender` matches any registered router address, and document that `SwapAllowlistExtension` is only safe for direct `pool.swap()` calls. Pool admins must never allowlist a router address.
