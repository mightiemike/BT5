### Title
`alignVestingDelegationTracking` Ignores Stale-High DV+DF After Slashed CommitDelegationToTier Force-Exit — (`x/tieredrewards/keeper/force_exit.go`)

---

### Summary

`alignVestingDelegationTracking` only corrects a stale-**low** DV+DF (top-up when `actualDelegated > tracked`). It has no branch for the stale-**high** case (`actualDelegated < tracked`). For a `CommitDelegationToTier`-origin position whose validator is slashed before the v8 migration, the returning delegation is smaller than the pre-migration stale-high DV+DF, so the gap is never closed. The vesting account's `LockedCoins` is permanently underestimated, allowing the owner to spend tokens that should be permanently locked.

---

### Finding Description

**Pre-migration state (CommitDelegationToTier, pre-v8 code)**

When `CommitDelegationToTier` was called in the pre-v8 code, `transferDelegationToPosition` unbonded the owner's delegation and re-delegated it to the position's module address with `subtractAccount=false`. The old code did **not** call `TrackUndelegation` on the owner's vesting account when moving the delegation out. As a result, the owner's `DelegatedVesting + DelegatedFree` (DV+DF) remained at `lockedAmount` even though the actual on-chain delegation was now held by the position address — a stale-high condition.

The code comment in `alignVestingDelegationTracking` acknowledges this explicitly:

> "for CommitDelegationToTier-origin positions DV+DF was already stale-high pre-migration and the returning delegation closes the gap." [1](#0-0) 

**The slash breaks the "closing the gap" assumption**

If the validator is slashed 10% before the v8 migration runs, the position's delegation is worth `0.9 × lockedAmount`. `ForceFullExitWithDelegation` calls `transferDelegationFromPosition`, which unbonds `0.9 × lockedAmount` from the position address and re-delegates it to the owner with `subtractAccount=false` — so `TrackDelegation` is **not** called on the owner. [2](#0-1) 

After the transfer:
- Owner's on-chain delegation = `0.9 × lockedAmount`
- Owner's DV+DF = `lockedAmount` (unchanged, stale-high by `0.1 × lockedAmount`)

**`alignVestingDelegationTracking` does nothing**

```
actualDelegated = 0.9 × lockedAmount
tracked         = lockedAmount
actualDelegated.GT(tracked) → false  →  return nil
```

The function only has a top-up path. There is no `TrackUndelegation` call for the stale-high case. [3](#0-2) 

**Resulting invariant break**

For a `PermanentLockedAccount` with `OriginalVesting = lockedAmount`:

```
LockedCoins = max(0, OriginalVesting − DV)
            = max(0, lockedAmount − lockedAmount)   ← stale-high DV
            = 0                                      ← should be 0.1 × lockedAmount
```

`SpendableCoins` is overestimated by `0.1 × lockedAmount`. Any bank balance the owner holds (staking rewards, IBC transfers, etc.) is spendable without the correct locked-coin constraint.

---

### Impact Explanation

- **Direct financial loss**: the owner can transfer/spend up to `slashFraction × lockedAmount` of tokens that the `PermanentLockedAccount` invariant requires to remain permanently locked.
- The stale-high DV persists indefinitely after the migration because `alignVestingDelegationTracking` is the only correction point and it silently skips the stale-high case.
- `ExitTierWithDelegation` (the normal post-migration exit path) does **not** call `alignVestingDelegationTracking` at all, so there is no secondary correction. [4](#0-3) 

---

### Likelihood Explanation

- Requires a `CommitDelegationToTier`-origin position owned by a vesting account — a supported, documented production flow.
- Requires the validator to be slashed before the v8 migration upgrade block — a realistic on-chain event (double-sign or downtime slash).
- The migration itself (`exitVestedAccountsPositions`) is the only trigger; no attacker action is needed after the slash. [5](#0-4) 

---

### Recommendation

Add a stale-high correction branch in `alignVestingDelegationTracking`. When `tracked > actualDelegated`, call `vacc.TrackUndelegation` with the surplus `(tracked − actualDelegated)` and persist the account:

```go
if tracked.GT(actualDelegated) {
    surplus := tracked.Sub(actualDelegated)
    surplusCoins := sdk.NewCoins(sdk.NewCoin(bondDenom, surplus))
    vacc.TrackUndelegation(surplusCoins)
    k.accountKeeper.SetAccount(ctx, vacc)
}
```

This mirrors the existing top-up logic and ensures DV+DF == Σ on-chain delegations regardless of slash history. [6](#0-5) 

---

### Proof of Concept

1. Create a `PermanentLockedAccount` owner with `OriginalVesting = 1000 CRO`, all delegated to validator V.
2. Call `CommitDelegationToTier` (pre-v8 code path): delegation moves to position, `TrackUndelegation` is **not** called → `DV = 1000`, `DF = 0`.
3. Slash validator V by 10% (`slashValidatorDirect(0.1)`): position's delegation becomes `900 CRO`.
4. Run v8 migration (`exitVestedAccountsPositions` → `ForceFullExitWithDelegation`).
5. After migration:
   - `GetDelegatorDelegations(owner)` → `900 CRO`
   - `vacc.GetDelegatedVesting()` → `1000 CRO` (unchanged)
   - `LockedCoins(owner)` → `0` (should be `100 CRO`)
   - `SpendableCoins(owner)` → full bank balance (should be `bank_balance − 100 CRO`)
6. Assert `trackedTotal(owner) > totalDelegated(owner)` and `SpendableCoins(owner) > expected`. [7](#0-6) [8](#0-7)

```go
func (ms msgServer) ExitTierWithDelegation(ctx context.Context, msg *types.MsgExitTierWithDelegation) (*types.MsgExitTierWithDelegationResponse, error) {
	if err := msg.Validate(); err != nil {
		return nil, err
	}

	pos, err := ms.getPositionState(ctx, msg.PositionId)
	if err != nil {
		return nil, err
	}

	if err := ms.validateExitTierWithDelegation(ctx, pos, msg.Owner, msg.Amount); err != nil {
		return nil, err
	}

	pos, _, _, err = ms.claimRewards(ctx, pos)
	if err != nil {
		return nil, err
	}

	validator := pos.Delegation.ValidatorAddress
	valAddr, err := sdk.ValAddressFromBech32(validator)
	if err != nil {
		return nil, err
	}

	positionAmount, err := ms.reconcileAmountFromShares(ctx, valAddr, pos.Delegation.Shares)
	if err != nil {
		return nil, err
	}

	transferredShares, unbondedShares, transferredAmount, err := ms.transferDelegationFromPosition(ctx, pos, valAddr, msg.Amount)
	if err != nil {
		return nil, err
	}

	// Capture for event before potential deletion.
	posId := pos.Id
	tierId := pos.TierId

	fullExit := pos.ExitWithFullDelegation(msg.Amount, positionAmount)

	if fullExit {
		ownerAddr, err := sdk.AccAddressFromBech32(msg.Owner)
		if err != nil {
			return nil, err
		}

		delAddr, err := sdk.AccAddressFromBech32(pos.DelegatorAddress)
		if err != nil {
			return nil, errorsmod.Wrap(sdkerrors.ErrInvalidAddress, "invalid delegator address")
		}

		// sweeps any remaining dust if any (usually zero)
		balances := ms.bankKeeper.SpendableCoins(ctx, delAddr)
		if !balances.IsZero() {
			if err := ms.bankKeeper.SendCoins(ctx, delAddr, ownerAddr, balances); err != nil {
				return nil, err
			}
		}

		if err := ms.deletePosition(ctx, pos.Position, &ValidatorTransition{PreviousAddress: validator}); err != nil {
			return nil, err
		}

	} else {
		remainingShares := pos.Delegation.Shares.Sub(unbondedShares)
		// Compute remaining token value for min lock check.
		remainingPositionAmount, err := ms.reconcileAmountFromShares(ctx, valAddr, remainingShares)
		if err != nil {
			return nil, err
		}

		tier, err := ms.getTier(ctx, pos.TierId)
		if err != nil {
			return nil, err
		}
		// actual remaining amount (post-transfer) must meet min lock.
		if !tier.MeetsMinLockRequirement(remainingPositionAmount) {
			return nil, errorsmod.Wrapf(types.ErrMinLockAmountNotMet,
				"remaining amount %s is below tier minimum %s", remainingPositionAmount, tier.MinLockAmount)
		}

		if err := ms.setPosition(ctx, pos.Position, nil); err != nil {
			return nil, err
		}
	}

	sdkCtx := sdk.UnwrapSDKContext(ctx)
	if err := sdkCtx.EventManager().EmitTypedEvent(&types.EventExitTierWithDelegation{
		PositionId:        posId,
		TierId:            tierId,
		Owner:             msg.Owner,
		Validator:         validator,
		TransferredAmount: transferredAmount,
		TransferredShares: transferredShares,
		FullExit:          fullExit,
	}); err != nil {
		return nil, err
	}

	return &types.MsgExitTierWithDelegationResponse{
		PositionId:        posId,
		TransferredAmount: transferredAmount,
		TransferredShares: transferredShares,
		FullExit:          fullExit,
	}, nil
}
```

```go
func exitVestedAccountsPositions(
	ctx context.Context,
	positions collections.Map[uint64, types.Position],
	ak AccountKeeper,
	pk PositionForceExiter,
) error {
	sdkCtx := sdk.UnwrapSDKContext(ctx)

	var toExit []uint64
	if err := positions.Walk(ctx, nil, func(posID uint64, pos types.Position) (bool, error) {
		ownerAddr, err := sdk.AccAddressFromBech32(pos.Owner)
		if err != nil {
			return false, fmt.Errorf("parse owner address for position %d: %w", posID, err)
		}
		acc := ak.GetAccount(ctx, ownerAddr)
		if acc == nil {
			return false, fmt.Errorf("owner account not found for position %d: %s", posID, ownerAddr.String())
		}
		if _, ok := acc.(sdkvesting.VestingAccount); ok {
			toExit = append(toExit, posID)
		}
		return false, nil
	}); err != nil {
		return fmt.Errorf("walk positions: %w", err)
	}

	for _, posID := range toExit {
		sdkCtx.Logger().Info("v8 migration: force-exit vesting-owned position", "position_id", posID)
		if err := pk.ForceFullExitWithDelegation(ctx, posID); err != nil {
			return fmt.Errorf("force-exit position %d: %w", posID, err)
		}
	}

	sdkCtx.Logger().Info("v8 migration: vesting-owned positions exited", "count", len(toExit))
	return nil
```
