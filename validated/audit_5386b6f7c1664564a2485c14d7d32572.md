### Title
`MsgUpdateTier` `ExitDuration` Change Desynchronizes Stored `ExitUnlockAt` on Existing Exiting Positions — (File: `x/tieredrewards/keeper/msg_server_auth.go`)

---

### Summary

`MsgUpdateTier` allows governance to change a tier's `ExitDuration` at any time. When `ExitDuration` changes without a simultaneous `BonusApy` change, the handler performs no settlement and no update of existing positions' stored `ExitUnlockAt` timestamps. Because `ExitUnlockAt` is the hard cap for bonus accrual and the gate for undelegation/withdrawal, this desynchronization causes either (a) bonus over-payment from the rewards pool when `ExitDuration` is shortened, or (b) premature exit bypass of the new lock-up period when `ExitDuration` is lengthened.

---

### Finding Description

When a user calls `MsgTriggerExitFromTier`, the handler reads the current `tier.ExitDuration` and stores an absolute timestamp on the position:

```go
pos.TriggerExit(sdkCtx.BlockTime(), tier.ExitDuration)
// sets: pos.ExitTriggeredAt = blockTime
//       pos.ExitUnlockAt    = blockTime.Add(duration)
``` [1](#0-0) [2](#0-1) 

This stored `ExitUnlockAt` serves two critical roles:

1. **Bonus accrual cap** — `computeSegmentBonus` clamps `segmentEnd` to `pos.ExitUnlockAt`, and `applyBonusAccrualCheckpoint` advances `LastBonusAccrual` only up to `ExitUnlockAt` once the exit lock has elapsed. [3](#0-2) [4](#0-3) 

2. **Exit gate** — `CompletedExitLockDuration` checks `!blockTime.Before(pos.ExitUnlockAt)` to allow undelegation and withdrawal. [5](#0-4) 

The `UpdateTier` handler only triggers reward settlement when `BonusApy` changes. A change to `ExitDuration` alone passes through with no settlement and no update of any existing position's `ExitUnlockAt`:

```go
if !oldTier.BonusApy.Equal(msg.Tier.BonusApy) {
    if err := ms.claimRewardsAndUpdateTierPositions(ctx, msg.Tier.Id); err != nil {
        return nil, err
    }
}
// ExitDuration change: no action on existing positions
if err := ms.SetTier(ctx, msg.Tier); err != nil { ... }
``` [6](#0-5) 

After `SetTier` writes the new `ExitDuration`, every existing position that has already triggered exit retains its old absolute `ExitUnlockAt`, which was computed from the previous `ExitDuration`. The tier's `ExitDuration` and the positions' `ExitUnlockAt` are now desynchronized.

---

### Impact Explanation

**Path A — `ExitDuration` shortened (e.g., 1 year → 5 s):**

Existing exiting positions retain `ExitUnlockAt = ExitTriggeredAt + 1 year`. Bonus continues to accrue until that old timestamp. Under the new tier parameters the intent is that bonus should stop much sooner. The rewards pool (`tieredrewards` module account `RewardsPoolName`) is over-drained by up to ~1 year of bonus at the tier's `BonusApy` per affected position. The corrupted value is the rewards pool balance.

**Path B — `ExitDuration` lengthened (e.g., 5 s → 1 year):**

Existing exiting positions retain `ExitUnlockAt = ExitTriggeredAt + 5 s`. After 5 seconds `CompletedExitLockDuration` returns `true`, and the position owner can call `MsgTierUndelegate` or `MsgExitTierWithDelegation` to exit immediately, bypassing the governance-mandated 1-year lock-up. The corrupted invariant is the lock-up enforcement guarantee. [7](#0-6) [8](#0-7) 

---

### Likelihood Explanation

Changing `ExitDuration` via `MsgUpdateTier` is an explicitly supported and tested governance operation. The integration test `test_update_tier_non_apy_no_claim` confirms that changing `exit_duration` without changing APY is a normal governance flow, and the upgrade test `assert_v8_precreated_position_delegator_vesting_acc_lifecycle` uses exactly this pattern to shorten `ExitDuration` for an existing position. The desynchronization occurs silently on every such governance proposal whenever any position in the tier is already in the exit phase. [9](#0-8) [10](#0-9) 

---

### Recommendation

When `ExitDuration` changes, one of the following mitigations should be applied:

1. **Settle and recompute**: Before writing the new tier, iterate all exiting positions (those with non-zero `ExitTriggeredAt`) and recompute `ExitUnlockAt = pos.ExitTriggeredAt + newExitDuration`, persisting the updated position. This mirrors the existing `BonusApy`-change settlement pattern.
2. **Prevent mid-course change**: Reject `MsgUpdateTier` if any position in the tier has a non-zero `ExitTriggeredAt`, analogous to the external report's remediation of removing the option to change `time_unit` mid-course.

---

### Proof of Concept

1. Tier 1 is configured with `ExitDuration = 1 year`, `BonusApy = 4%`.
2. User calls `MsgTriggerExitFromTier` at block time `T`. Position stores `ExitUnlockAt = T + 1 year`.
3. Governance submits and passes `MsgUpdateTier` with `ExitDuration = 5s`, same `BonusApy = 4%`.
4. `UpdateTier` evaluates `!oldTier.BonusApy.Equal(msg.Tier.BonusApy)` → `false`; `claimRewardsAndUpdateTierPositions` is **not** called.
5. `SetTier` writes the new tier with `ExitDuration = 5s`.
6. The position still holds `ExitUnlockAt = T + 1 year`.
7. Every subsequent `ClaimTierRewards` or mutation call invokes `processEventsAndClaimBonus`, which caps `segmentEnd` at `pos.ExitUnlockAt = T + 1 year` — accruing bonus for the full original year instead of stopping at `T + 5s`.
8. The rewards pool is drained by approximately `shares × tokensPerShare × 0.04 × (1 year − 5 s) / SecondsPerYear` per affected position. [11](#0-10) [12](#0-11) [13](#0-12)

func (k Keeper) computeSegmentBonus(pos types.PositionState, tier types.Tier, segmentStart, segmentEnd time.Time, tokensPerShare math.LegacyDec) math.Int {
	if !pos.ExitUnlockAt.IsZero() && segmentEnd.After(pos.ExitUnlockAt) {
		segmentEnd = pos.ExitUnlockAt
	}

	if !segmentEnd.After(segmentStart) {
		return math.ZeroInt()
	}

	durationSeconds := int64(segmentEnd.Sub(segmentStart) / time.Second)
	if durationSeconds <= 0 {
		return math.ZeroInt()
	}

	tokens := pos.Delegation.Shares.Mul(tokensPerShare)

	return tokens.
		Mul(tier.BonusApy).
		MulInt64(durationSeconds).
		QuoInt64(types.SecondsPerYear).
		TruncateInt()
}

func (ms msgServer) UpdateTier(ctx context.Context, msg *types.MsgUpdateTier) (*types.MsgUpdateTierResponse, error) {
	if err := ms.requireAuthority(msg.Authority); err != nil {
		return nil, err
	}

	oldTier, err := ms.getTier(ctx, msg.Tier.Id)
	if err != nil {
		return nil, err
	}

	if !oldTier.BonusApy.Equal(msg.Tier.BonusApy) {
		if err := ms.claimRewardsAndUpdateTierPositions(ctx, msg.Tier.Id); err != nil {
			return nil, err
		}
	}

	if err := ms.SetTier(ctx, msg.Tier); err != nil {
		return nil, err
	}

	if err := ms.emitTierChangedEvent(ctx, types.TierChangeAction_TIER_CHANGE_ACTION_UPDATE, msg.Tier); err != nil {
		return nil, err
	}

	return &types.MsgUpdateTierResponse{}, nil
}

// claimRewardsAndUpdateTierPositions claims base and bonus rewards for all
// delegated positions in the given tier.
func (k Keeper) claimRewardsAndUpdateTierPositions(ctx context.Context, tierId uint32) error {
	ids, err := k.getPositionsIdsByTier(ctx, tierId)
	if err != nil {
		return err
	}
	if len(ids) == 0 {
		return nil
	}

	for _, id := range ids {
		pos, err := k.getPositionState(ctx, id)
		if err != nil {
			return err
		}
		if !pos.IsDelegated() {
			continue
		}

		if _, err := k.claimBaseRewards(ctx, pos); err != nil {
			return err
		}
		if _, err := k.processEventsAndClaimBonus(ctx, &pos); err != nil {
			return err
		}
		if err := k.setPosition(ctx, pos.Position, nil); err != nil {
			return err
		}
	}

	return nil
}

def test_update_tier_non_apy_no_claim(cluster):
    """Changing exit_duration without changing APY does not trigger rewards claiming.

    Verified by checking that the owner balance is unchanged.
    """
    owner = cluster.address("signer1")

    # Confirm the signer1 position on Tier 3 exists
    result = query_command(cluster, MODULE, "positions-by-owner", owner)
    tier3_pos = next(
        p for p in result.get("positions", []) if int(p["tier_id"]) == TIER_3_ID
    )
    assert tier3_pos is not None
    balance_before = cluster.balance(owner, DENOM)

    rsp = submit_gov_proposal(
        cluster,
        "community",
        MSG_UPDATE_TIER,
        {
            "tier": {
                "id": TIER_3_ID,
                "exit_duration": "10s",
                "bonus_apy": "0.030000000000000000",
                "min_lock_amount": "2000000",
                "close_only": False,
            }
