### Title
Deprecated `verify_transaction_inclusion` Remains Publicly Callable Without Coinbase Proof Guard, Enabling Merkle Proof Forgery — (File: `contract/src/lib.rs`)

---

### Summary
The deprecated `verify_transaction_inclusion` function is still a fully reachable NEAR contract method. It is missing the coinbase Merkle proof validation that `verify_transaction_inclusion_v2` introduced to close the 64-byte transaction Merkle forgery attack. Any unprivileged NEAR caller can invoke the deprecated path directly, bypassing the security fix, and receive a `true` return value for a Bitcoin transaction that does not exist.

### Finding Description

In Rust, `#[deprecated]` is a compile-time lint only; it does not restrict runtime access.
