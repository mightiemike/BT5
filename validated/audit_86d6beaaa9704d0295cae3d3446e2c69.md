### Title
Difficulty Adjustment Reads Mainchain Ancestor Instead of Fork Ancestor for Fork Blocks — (`contract/src/dogecoin.rs`, `contract/src/bitcoin.rs`)

---

### Summary

The `get_next_work_required` function in both `dogecoin.rs` and `bitcoin.rs` resolves the first block of the difficulty-adjustment interval by calling `blocks_getter.get_header_by_height(height_first)`. That call always reads from `mainchain_height_to_header`, which only contains mainchain blocks. When the contract is validating a fork block whose ancestor at `height_first` is a fork block (not a mainchain block), the wrong timestamp is used, producing an incorrect expected-`bits` value. The contract then accepts or rejects the fork block based on that wrong value. A `TODO` comment in `dogecoin.rs` explicitly flags this unresolved concern.

---

### Finding Description

`get_header_by_height` is implemented in `lib.rs` as:

```rust
fn get_header_by_height(&self, height: u64) -> ExtendedHeader {
    self.mainchain_height_to_header          // ← always mainchain
        .get(&height)
        .and_then(|hash| self.headers_pool.get(&hash))
        .unwrap_or_else(|| env::panic_str(ERR_KEY_NOT_EXIST))
}
``` [1](#0-0) 

Fork blocks are stored only in `headers_pool`, never in `mainchain_height_to_header`:

```rust
fn store_fork_header(&mut self, header: &ExtendedHeader) {
    self.headers_pool.insert(&header.block_hash, header);
}
``` [2](#0-1) 

In `dogecoin.rs`, after Dogecoin's height-145,000 protocol switch, `difficulty_adjustment_interval` becomes `1`, so every block triggers a retarget. The retarget reads the first-interval block via:

```rust
// TODO: check if it is correct to get block header by height from mainchain without looping to find the ancestor
let first_block_time = blocks_getter
    .get_header_by_height(height_first)
    .block_header
    .time;
``` [3](#0-2) 

With `blocks_to_go_back = 1`, `height_first = prev_block_header.block_height - 1`. [4](#0-3) 

**Concrete desynchronization scenario (Dogecoin, height ≥ 145,000):**

| Block | Chain | Stored in mainchain map? | Used as `height_first` for next fork block? |
|---|---|---|---|
| H | mainchain | yes | yes (correct — fork diverges here) |
| H+1 | fork | **no** | yes (correct — parent is mainchain H) |
| H+2 | fork | **no** | yes (correct — parent is fork H+1, `height_first = H`, mainchain H == fork ancestor H ✓) |
| H+3 | fork | **no** | `height_first = H+1` → returns **mainchain H+1**, not **fork H+1** ✗ |

Starting at the third fork block (H+3), `get_header_by_height(H+1)` returns the mainchain block at H+1, whose timestamp may differ from the fork block at H+1. The Digishield formula:

```rust
let modulated_timespan = retarget_timespan + (modulated_timespan - retarget_timespan) / 8;
``` [5](#0-4) 

…uses `modulated_timespan = prev_block_time − first_block_time`. Substituting the wrong `first_block_time` (mainchain H+1 timestamp instead of fork H+1 timestamp) produces a wrong `expected_bits`. The contract then enforces:

```rust
require!(
    expected_bits == block_header.bits,
    ...
);
``` [6](#0-5) 

So the contract accepts a fork block whose `bits` matches the **wrong** expected difficulty, not the difficulty the real Dogecoin network would require.

The same structural issue exists in `bitcoin.rs`:

```rust
let interval_tail_extend_header = blocks_getter.get_header_by_height(first_block_height);
``` [7](#0-6) 

For Bitcoin the adjustment interval is 2016 blocks, so the desynchronization only manifests for forks deeper than 2016 blocks — far less likely in practice.

---

### Impact Explanation

The contract's `verify_transaction_inclusion` and `verify_transaction_inclusion_v2` trust that every block in `mainchain_height_to_header` passed a correct difficulty check. If a fork chain is promoted to the mainchain after accumulating more chainwork, and some of its blocks were accepted with an incorrect `bits` value (because the difficulty check used the wrong ancestor timestamp), then:

1. The contract's canonical chain diverges from the real Dogecoin network's canonical chain.
2. `verify_transaction_inclusion` returns `true` for transactions in blocks that the real Dogecoin network considers invalid or non-canonical.
3. Any downstream contract or application consuming these verification results is deceived. [8](#0-7) 

---

### Likelihood Explanation

An unprivileged proof submitter / relayer can trigger this by:

1. Submitting a valid fork block at H+1 (diverging from mainchain at H) with a timestamp `T_f` that differs from the mainchain block at H+1's timestamp `T_m` (within MTP constraints — the MTP check uses `prev_block_header`, which is the mainchain block at H, so the attacker has a window of several hundred seconds).
2. Submitting fork blocks at H+2 and H+3. For H+3, the contract computes expected bits using `T_m` instead of `T_f`, accepting a `bits` value that the real network would reject.
3. Repeating for subsequent fork blocks, each time benefiting from the wrong ancestor lookup.

No privileged role, leaked key, or social engineering is required. The `submit_blocks` entry point is open to any caller with sufficient deposit. [9](#0-8) 

---

### Recommendation

Replace the `get_header_by_height` call inside `get_next_work_required` with an ancestor-traversal that walks `headers_pool` backwards via `prev_block_hash` from `prev_block_header` until reaching the target height. This mirrors how the reference Dogecoin implementation finds the retarget ancestor and ensures fork blocks use their own ancestor chain rather than the mainchain mapping.

---

### Proof of Concept

```
Setup (Dogecoin mainnet, height ≥ 145,000):
  Mainchain: ... → [H-1] → [H, time=T0] → [H+1, time=T_m] → [H+2, time=T_m+60]

Attack:
  1. Submit fork block F1 at height H+1:
       prev_block_hash = hash(mainchain[H])
       time = T_f  (T_f << T_m, e.g. T_f = MTP(mainchain[H]) + 1)
       bits = correct (same as mainchain[H+1], since height_first=H, mainchain[H] is shared)
       PoW: mine valid hash ≤ target_from_bits(bits)

  2. Submit fork block F2 at height H+2:
       prev_block_hash = hash(F1)
       height_first = H+1
       get_header_by_height(H+1) → mainchain[H+1], time = T_m   ← WRONG (should be F1, time=T_f)
       actual_timespan (wrong) = F2.time - T_m  (small, since T_m is close to F2.time)
       actual_timespan (correct) = F2.time - T_f  (large, since T_f << T_m)
       expected_bits (wrong) = lower difficulty (higher target) than correct
       Attacker sets F2.bits = expected_bits_wrong
       PoW: mine valid hash ≤ target_from_bits(expected_bits_wrong)  ← easier than correct

  3. Repeat for F3, F4, … each benefiting from the wrong ancestor lookup.

  4. If fork accumulates more chainwork than mainchain, reorg_chain() promotes it.
     verify_transaction_inclusion() now returns true for transactions in this
     incorrectly-validated fork chain.
``` [10](#0-9) [11](#0-10)

    pub fn verify_transaction_inclusion(&self, #[serializer(borsh)] args: ProofArgs) -> bool {
        require!(
            args.confirmations <= self.gc_threshold,
            "The required number of confirmations exceeds the number of blocks stored in memory"
        );

        let heaviest_block_header = self
            .headers_pool
            .get(&self.mainchain_tip_blockhash)
            .unwrap_or_else(|| env::panic_str(ERR_KEY_NOT_EXIST));
        let target_block_height = self
            .mainchain_header_to_height
            .get(&args.tx_block_blockhash)
            .unwrap_or_else(|| env::panic_str("block does not belong to the current main chain"));

        // Check requested confirmations. No need to compute proof if insufficient confirmations.
        require!(
            (heaviest_block_header.block_height).saturating_sub(target_block_height) + 1
                >= args.confirmations,
            "Not enough blocks confirmed"
        );

        let header = self
            .headers_pool
            .get(&args.tx_block_blockhash)
            .unwrap_or_else(|| env::panic_str("cannot find requested transaction block"));

        require!(!args.merkle_proof.is_empty(), "Merkle proof is empty");

        // compute merkle tree root and check if it matches block's original merkle tree root
        merkle_tools::compute_root_from_merkle_proof(
            args.tx_id,
            usize::try_from(args.tx_index).unwrap(),
            &args.merkle_proof,
        ) == header.block_header.merkle_root
    }

    fn reorg_chain(&mut self, fork_tip_header: ExtendedHeader, last_main_chain_block_height: u64) {
        let fork_tip_height = fork_tip_header.block_height;
        if last_main_chain_block_height > fork_tip_height {
            // If we see that main chain is longer than fork we first garbage collect
            // outstanding main chain blocks:
            //
            //      [m1] - [m2] - [m3] - [m4] <- We should remove [m4]
            //     /
            // [m0]
            //     \
            //      [f1] - [f2] - [f3]
            for height in (fork_tip_height + 1)..=last_main_chain_block_height {
                let current_main_chain_blockhash = self
                    .mainchain_height_to_header
                    .get(&height)
                    .unwrap_or_else(|| env::panic_str("cannot get a block"));
                self.remove_block_header(&current_main_chain_blockhash);
                self.mainchain_height_to_header.remove(&height);
            }
        }

        // Now we are in a situation where mainchain is equivalent to fork size:
        //
        //      [m1] - [m2] - [m3] - [m4] <- main tip
        //     /
        // [m0]
        //     \
        //      [f1] - [f2] - [f3] - [f4] <- fork tip
        //
        //
        // Or in a situation where it is shorter:
        //
        //      [m1] - [m2] - [m3] <- main tip
        //     /
        // [m0]
        //     \
        //      [f1] - [f2] - [f3] - [f4] <- fork tip

        let fork_tip_hash = fork_tip_header.block_hash.clone();
        let mut fork_header_cursor = fork_tip_header;

        while !self
            .mainchain_header_to_height
            .contains_key(&fork_header_cursor.block_hash)
        {
            let prev_block_hash = fork_header_cursor.block_header.prev_block_hash;
            let current_block_hash = fork_header_cursor.block_hash;
            let current_height = fork_header_cursor.block_height;

            // Inserting the fork block into the main chain, if some mainchain block is occupying
            // this height let's save its hashcode
            let main_chain_block = self
                .mainchain_height_to_header
                .insert(&current_height, &current_block_hash);
            self.mainchain_header_to_height
                .insert(&current_block_hash, &current_height);

            // If we found a mainchain block at the current height than remove this block from the
            // header pool and from the header -> height map
            if let Some(current_main_chain_blockhash) = main_chain_block {
                self.remove_block_header(&current_main_chain_blockhash);
            }

            // Switch iterator cursor to the previous block in fork
            fork_header_cursor = self
                .headers_pool
                .get(&prev_block_hash)
                .unwrap_or_else(|| env::panic_str("previous fork block should be there"));
        }

        // Updating tip of the new main chain
        self.mainchain_tip_blockhash = fork_tip_hash;
    }

```rust
fn get_next_work_required(
    config: &DogecoinConfig,
    block_header: &Header,
    prev_block_header: &ExtendedHeader,
    blocks_getter: &impl BlocksGetter,
) -> u32 {
    // Dogecoin: Special rules for minimum difficulty blocks with Digishield
    if allow_min_difficulty_for_block(config, block_header, prev_block_header) {
        // Special difficulty rule for testnet:
        // If the new block's timestamp is more than 2* nTargetSpacing minutes
        // then allow mining of a min-difficulty block.
        return config.proof_of_work_limit_bits;
    }

    // Only change once per difficulty adjustment interval
    let new_difficulty_protocol = prev_block_header.block_height >= 145_000;
    let difficulty_adjustment_interval = if new_difficulty_protocol {
        1
    } else {
        config.difficulty_adjustment_interval
    };

    if (prev_block_header.block_height + 1) % difficulty_adjustment_interval != 0 {
        if config.pow_allow_min_difficulty_blocks {
            // Special difficulty rule for testnet:
            // If the new block's timestamp is more than 2* 10 minutes
            // then allow mining of a min-difficulty block.
            if block_header.time
                > prev_block_header.block_header.time + config.pow_target_spacing * 2
            {
                return config.proof_of_work_limit_bits;
            }

            // Return the last non-special-min-difficulty-rules-block
            let mut current_block_header = prev_block_header.clone();

            while current_block_header.block_header.bits == config.proof_of_work_limit_bits
                && current_block_header.block_height % config.difficulty_adjustment_interval != 0
            {
                current_block_header =
                    blocks_getter.get_prev_header(&current_block_header.block_header);
            }

            return current_block_header.block_header.bits;
        }

        return prev_block_header.block_header.bits;
    }

    // Litecoin: This fixes an issue where a 51% attack can change difficulty at will.
    // Go back the full period unless it's the first retarget after genesis. Code courtesy of Art Forz
    let mut blocks_to_go_back = difficulty_adjustment_interval - 1;
    if prev_block_header.block_height + 1 != difficulty_adjustment_interval {
        blocks_to_go_back = difficulty_adjustment_interval;
    }

    // Go back by what we want to be 14 days worth of blocks
    let height_first = prev_block_header
        .block_height
        .checked_sub(blocks_to_go_back)
        .unwrap_or_else(|| env::panic_str("Height underflow when calculating first block height"));

    // TODO: check if it is correct to get block header by height from mainchain without looping to find the ancestor
    let first_block_time = blocks_getter
        .get_header_by_height(height_first)
        .block_header
        .time;

    calculate_next_work_required(config, prev_block_header, i64::from(first_block_time))
