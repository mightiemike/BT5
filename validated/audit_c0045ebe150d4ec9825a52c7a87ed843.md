### Title
BurnNFT Requires Simultaneous IsOwner AND IsDenomCreator, Making Transferred NFTs Permanently Unburnable — (`x/nft/keeper/keeper.go`)

### Summary

`Keeper.BurnNFT` gates deletion on **both** `IsOwner` and `IsDenomCreator` being true for the same address simultaneously. Once an NFT is transferred away from the denom creator, no single address can satisfy both conditions, making the NFT unburnable and permanently inflating the denom's supply counter.

### Finding Description

`BurnNFT` in `x/nft/keeper/keeper.go` applies two sequential AND-gated checks: [1](#0-0) 

```
IsOwner(owner)       → must be current NFT owner
IsDenomCreator(owner) → must be denom creator
```

Both must pass for the **same** `owner` argument. There is no OR path, no admin override, and no separate creator-privilege burn path.

`IsOwner` returns `ErrUnauthorized` if the caller is not the current NFT owner: [2](#0-1) 

`IsDenomCreator` returns `ErrUnauthorized` if the caller is not the denom creator: [3](#0-2) 

`MintNFT` explicitly allows minting to a recipient different from the sender/creator: [4](#0-3) 

`TransferOwner` only checks `IsOwner` (no creator restriction), so any owner can freely transfer: [5](#0-4) 

**Concrete reachable state:**

1. Alice (creator) calls `MsgIssueDenom` → denom created, `creator = Alice`.
2. Alice calls `MsgMintNFT(recipient=Bob)` → NFT owned by Bob.
3. Bob calls `MsgTransferNFT(recipient=Carol)` → NFT owned by Carol.

Now:
- `MsgBurnNFT(sender=Alice)` → `IsOwner(Alice)` fails (`Carol` is owner) → `ErrUnauthorized`.
- `MsgBurnNFT(sender=Carol)` → `IsOwner(Carol)` passes, then `IsDenomCreator(Carol)` fails (`Alice` is creator) → `ErrUnauthorized`.

The NFT cannot be burned by any party unless Carol voluntarily transfers it back to Alice. If Carol is uncooperative, lost, or is a module/escrow account, the NFT is permanently stuck.

### Impact Explanation

- The denom's supply counter (`GetTotalSupply`) is permanently inflated for every such stuck NFT, since `decreaseSupply` is never called.
- The denom creator loses all administrative lifecycle control over NFTs once transferred.
- `EditNFT` has the same dual AND-check pattern (`IsOwner` + `IsDenomCreator`), so transferred NFTs are also permanently uneditable by either party. [6](#0-5) 

### Likelihood Explanation

Any standard NFT workflow where the creator mints to a recipient (the common case — see `MsgMintNFT`'s explicit `recipient` field) and that recipient transfers the NFT triggers this. No special privileges or governance are required. The path is reachable via ordinary signed transactions.

### Recommendation

Replace the AND logic in `BurnNFT` with an OR: allow burn if the caller is **either** the current owner **or** the denom creator. Alternatively, add a separate creator-only admin burn path that bypasses the ownership check. The same fix applies to `EditNFT`.

```go
// Proposed fix for BurnNFT
isOwner := owner.Equals(nft.GetOwner())
_, creatorErr := k.IsDenomCreator(ctx, denomID, owner)
if !isOwner && creatorErr != nil {
    return sdkerrors.Wrapf(types.ErrUnauthorized, ...)
}
```

### Proof of Concept

```go
// keeper_test.go (pseudocode)
creator := addrs[0]; bob := addrs[1]; carol := addrs[2]

k.MintNFT(ctx, "testdenom", "nft1", ..., creator, bob)   // minted to bob

// supply counter is still 1, NFT is stuck

// EditNFT updates an already existing NFT
func (k Keeper) EditNFT(
	ctx sdk.Context, denomID, tokenID, tokenNm,
	tokenURI, tokenData string, owner sdk.AccAddress,
) error {
	if !k.HasDenomID(ctx, denomID) {
		return sdkerrors.Wrapf(types.ErrInvalidDenom, "denom ID %s not exists", denomID)
	}

	nft, err := k.IsOwner(ctx, denomID, tokenID, owner)
	if err != nil {
		return err
	}

	_, err = k.IsDenomCreator(ctx, denomID, owner)
	if err != nil {
		return err
	}

	if types.Modified(tokenNm) {
		nft.Name = tokenNm
	}

	if types.Modified(tokenURI) {
		nft.URI = tokenURI
	}

	if types.Modified(tokenData) {
		nft.Data = tokenData
	}

	k.setNFT(ctx, denomID, nft)

	return nil
}

func (k Keeper) BurnNFT(ctx sdk.Context, denomID, tokenID string, owner sdk.AccAddress) error {
	if !k.HasDenomID(ctx, denomID) {
		return sdkerrors.Wrapf(types.ErrInvalidDenom, "denom ID %s not exists", denomID)
	}

	nft, err := k.IsOwner(ctx, denomID, tokenID, owner)
	if err != nil {
		return err
	}

	_, err = k.IsDenomCreator(ctx, denomID, owner)
	if err != nil {
		return err
	}

	k.deleteNFT(ctx, denomID, nft)
	k.deleteOwner(ctx, denomID, tokenID, owner)
	k.decreaseSupply(ctx, denomID)

	return nil
}

func (k Keeper) IsOwner(ctx sdk.Context, denomID, tokenID string, owner sdk.AccAddress) (types.BaseNFT, error) {
	nft, err := k.GetNFT(ctx, denomID, tokenID)
	if err != nil {
		return types.BaseNFT{}, err
	}

	if !owner.Equals(nft.GetOwner()) {
		return types.BaseNFT{}, sdkerrors.Wrapf(types.ErrUnauthorized, "%s is not the owner of %s/%s", owner.String(), denomID, tokenID)
	}

	return nft.(types.BaseNFT), nil
}

func (k Keeper) IsDenomCreator(ctx sdk.Context, denomID string, address sdk.AccAddress) (types.Denom, error) {
	denom, err := k.GetDenom(ctx, denomID)
	if err != nil {
		return types.Denom{}, err
	}

	creator, err := sdk.AccAddressFromBech32(denom.Creator)
	if err != nil {
		panic(err)
	}

	if !creator.Equals(address) {
		return types.Denom{}, sdkerrors.Wrapf(types.ErrUnauthorized, "%s is not the creator of %s", address, denomID)
	}

	return denom, nil
```
