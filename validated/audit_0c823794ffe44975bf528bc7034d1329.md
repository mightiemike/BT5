### Title
NFT Metadata Permanently Frozen After Any Transfer Due to Conjunctive `IsOwner` AND `IsDenomCreator` Guard in `EditNFT` — (`x/nft/keeper/keeper.go`)

---

### Summary

`EditNFT` in the NFT keeper requires the caller to simultaneously satisfy both `IsOwner` and `IsDenomCreator` for the same address. After any `TransferNFT`, no single address can ever satisfy both conditions, permanently freezing NFT metadata for the lifetime of the token.

---

### Finding Description

`EditNFT` applies two sequential ownership checks against the same `owner` address: [1](#0-0) 

```go
func (k Keeper) EditNFT(..., owner sdk.AccAddress) error {
    nft, err := k.IsOwner(ctx, denomID, tokenID, owner)   // must be current NFT owner
    if err != nil { return err }

    _, err = k.IsDenomCreator(ctx, denomID, owner)         // must ALSO be denom creator
    if err != nil { return err }
    ...
}
```

`IsOwner` verifies the caller holds the NFT: [2](#0-1) 

`IsDenomCreator` verifies the caller created the denom (a fixed, immutable field): [3](#0-2) 

`TransferOwner` changes the NFT's owner record but never changes the denom's `Creator` field: [4](#0-3) 

After `MintNFT(creator, recipient=victim)` followed by `TransferNFT`:

| Caller | `IsOwner` | `IsDenomCreator` | Result |
|---|---|---|---|
| `victim` (new owner) | PASS | FAIL (not creator) | `ErrUnauthorized` |
| `creator` (original) | FAIL (not owner) | PASS | `ErrUnauthorized` |

The existing keeper test already demonstrates this exact deadlock — after `TransferOwner`, `EditNFT` fails for the new owner with "not the creator": [5](#0-4) 

But the test never asserts that the original creator also fails `EditNFT` post-transfer (they do, because `IsOwner` now fails for them). The deadlock is total and permanent.

---

### Impact Explanation

Any NFT that has ever been transferred has its metadata permanently frozen. No party — not the current owner, not the denom creator — can call `EditNFT` successfully. If NFT metadata encodes mutable state (game attributes, license terms, dynamic URIs), the economic value of the NFT is destroyed by a simple transfer. A malicious denom creator can also deliberately mint to a victim address to grief them with an immutable, potentially defamatory or worthless metadata record.

`BurnNFT` has the identical conjunctive guard: [6](#0-5) 

This means a transferred NFT also cannot be burned by its new owner, compounding the impact.

---

### Likelihood Explanation

The exploit path requires only standard, permissionless Cosmos SDK transactions available to any user:
1. `MsgIssueDenom` — any address
2. `MsgMintNFT(recipient=victim)` — denom creator mints to any address
3. `MsgTransferNFT` — victim transfers to anyone (or creator self-transfers after minting to themselves, then transfers out)

No governance, privileged role, or key compromise is required. The condition triggers on every transfer, making it a near-certain occurrence in any real NFT marketplace.

---

### Recommendation

Decouple the authorization logic. `EditNFT` should permit the call if the sender is **either** the current NFT owner **or** the denom creator, not both simultaneously:

```go
_, ownerErr := k.IsOwner(ctx, denomID, tokenID, owner)
_, creatorErr := k.IsDenomCreator(ctx, denomID, owner)
if ownerErr != nil && creatorErr != nil {
    return sdkerrors.Wrapf(types.ErrUnauthorized, "sender must be NFT owner or denom creator")
}
```

Apply the same fix to `BurnNFT` if the intent is to allow the owner to burn without being the creator.

---

### Proof of Concept

The existing test in `x/nft/keeper/keeper_test.go` already proves the post-transfer failure for the new owner. Extending it to also assert the creator's failure after transfer confirms the full deadlock:

```go
// After TransferOwner(address -> address2):
err = suite.keeper.EditNFT(suite.ctx, denomID, tokenID, tokenNm, tokenURI, tokenData, address2)
suite.Error(err) // address2 is owner but not creator — CONFIRMED by existing test line 135

err = suite.keeper.EditNFT(suite.ctx, denomID, tokenID, tokenNm, tokenURI, tokenData, address)
suite.Error(err) // address is creator but no longer owner — deadlock confirmed
```

Both assertions pass, proving no address can ever edit the NFT again after a transfer.

func (k Keeper) IsDenomCreator(ctx sdk.Context, denomID string, address sdk.AccAddress) (types.Denom, error) {
	denom, err := k.GetDenom(ctx, denomID)
	if err != nil {
		return types.Denom{}, err
	}

	creator, err := sdk.AccAddressFromBech32(denom.Creator)
	if err != nil {
		panic(err)
	}

	if !creator.Equals(address) {
		return types.Denom{}, sdkerrors.Wrapf(types.ErrUnauthorized, "%s is not the creator of %s", address, denomID)
	}

	return denom, nil
