### Title
`burnNlp` Enforces Only Maintenance Health, Allowing Positions to Become Immediately Liquidatable After Interest Accrual — (File: `core/contracts/Clearinghouse.sol`)

---

### Summary

The `burnNlp` function in `Clearinghouse.sol` checks only `MAINTENANCE` health (`>= 0`) after burning NLP tokens, while every other collateral-modifying function in the same contract enforces the stricter `INITIAL` health check. Because the spot borrow rate always exceeds the deposit rate (due to `INTEREST_FEE_FRACTION`), a user who burns NLP and lands at exactly `MAINTENANCE health = 0` will have their health pushed below zero by the very next `updateStates` call, making them immediately liquidatable.

---

### Finding Description

**Root cause — weaker health gate in `burnNlp`:**

`burnNlp` ends with:

```solidity
require(
    getHealth(txn.sender, IProductEngine.HealthType.MAINTENANCE) >= 0,
    ERR_SUBACCT_HEALTH
);
``` [1](#0-0) 

Every other collateral-modifying path uses the stricter `INITIAL` check:

- `withdrawCollateral` → `getHealth(sender, INITIAL) >= 0`
- `transferQuote` → `_isAboveInitial(txn.sender)`
- `mintNlp` → `getHealth(txn.sender, INITIAL) >= 0`
- `nlpProfitShare` → `getHealth(poolSubaccount, INITIAL) >= 0` [2](#0-1) [3](#0-2) 

`INITIAL` weights are strictly more conservative than `MAINTENANCE` weights (`longWeightInitial < longWeightMaintenance`, `shortWeightInitial > shortWeightMaintenance`), so `INITIAL health < MAINTENANCE health` for any leveraged position. Allowing `burnNlp` to settle at `MAINTENANCE health = 0` means the user is already below `INITIAL health`, with zero buffer before liquidation. [4](#0-3) 

**Interest accrual mechanism — borrow rate always exceeds deposit rate:**

In `SpotEngineState._updateState`, the deposit rate multiplier is reduced by `INTEREST_FEE_FRACTION` relative to the borrow rate multiplier:

```solidity
int128 totalDepositRateX18 = utilizationRatioX18.mul(
    borrowRateMultiplierX18 - ONE
);
int128 realizedDepositRateX18 = totalDepositRateX18.mul(
    ONE - INTEREST_FEE_FRACTION
);
``` [5](#0-4) 

The borrow multiplier grows at the full borrow rate; the deposit multiplier grows at `utilization × (1 − fee_fraction) × borrow_rate`. For any non-zero utilization, `cumulativeBorrowsMultiplierX18` grows strictly faster than `cumulativeDepositsMultiplierX18`. A subaccount with a net spot borrow position therefore loses health monotonically with every `updateStates` call. [6](#0-5) 

**Liquidation trigger:**

`liquidateSubaccountImpl` requires `isUnderMaintenance(txn.liquidatee)`:

```solidity
require(isUnderMaintenance(txn.liquidatee), ERR_NOT_LIQUIDATABLE);
``` [7](#0-6) 

`isUnderMaintenance` returns `true` when `MAINTENANCE health < 0`. [8](#0-7) 

A user who exits `burnNlp` at exactly `MAINTENANCE health = 0` is one `updateStates` tick away from `MAINTENANCE health < 0` and immediate liquidation.

---

### Impact Explanation

A user with outstanding spot borrows can burn a precisely calculated amount of NLP to land at `MAINTENANCE health = 0`. The very next sequencer-triggered `updateStates` call accrues interest, the borrow balance grows faster than the deposit balance, `MAINTENANCE health` drops below zero, and the position is immediately liquidatable. The liquidator captures the liquidation discount at the user's expense. Because `updateStates` is called continuously by the sequencer, the window between position creation and liquidation eligibility can be a single sequencer tick — directly analogous to the Rubicon H-04 "next block" scenario.

---

### Likelihood Explanation

Any user who holds NLP tokens and has an open spot borrow can reach this state, either deliberately (to game the system) or accidentally (by burning NLP without realizing they are near the maintenance boundary). The `burnNlp` path is a standard user-facing operation reachable through the `Endpoint` without any privileged access.

---

### Recommendation

Replace the `MAINTENANCE` health check in `burnNlp` with an `INITIAL` health check, consistent with every other collateral-modifying function:

```solidity
// Before
require(
    getHealth(txn.sender, IProductEngine.HealthType.MAINTENANCE) >= 0,
    ERR_SUBACCT_HEALTH
);

// After
require(
    getHealth(txn.sender, IProductEngine.HealthType.INITIAL) >= 0,
    ERR_SUBACCT_HEALTH
);
```

This ensures the user retains the full `INITIAL → MAINTENANCE` safety buffer after burning NLP, absorbing subsequent interest accrual before the position can become liquidatable. [9](#0-8) 

---

### Proof of Concept

1. Alice has a spot borrow (negative balance in `SpotEngine` for some product) and holds NLP tokens.
2. Alice calls `burnNlp` via `Endpoint`, burning exactly enough NLP so that after the burn fee, her `MAINTENANCE health = 0` (achievable by binary-searching the `nlpAmount` parameter off-chain).
3. The `burnNlp` check passes: `getHealth(alice, MAINTENANCE) >= 0` ✓.
4. The sequencer calls `updateStates(dt)` with any `dt > 0`.
5. `_updateState` increments `cumulativeBorrowsMultiplierX18` by the full borrow rate and `cumulativeDepositsMultiplierX18` by the reduced deposit rate.
6. Alice's net borrow balance (in real units) grows; her `MAINTENANCE health` drops below 0.
7. Any liquidator submits `LiquidateSubaccount` for Alice; `isUnderMaintenance(alice)` returns `true`; liquidation succeeds and the liquidator captures the discount. [10](#0-9) [11](#0-10)

    function burnNlp(
        IEndpoint.BurnNlp calldata txn,
        int128 oraclePriceX18,
        IEndpoint.NlpPool[] calldata nlpPools,
        int128[] calldata nlpPoolRebalanceX18
    ) external onlyEndpoint {
        require(!RiskHelper.isIsolatedSubaccount(txn.sender), ERR_UNAUTHORIZED);

        ISpotEngine spotEngine = _spotEngine();
        spotEngine.updatePrice(NLP_PRODUCT_ID, oraclePriceX18);

        require(txn.nlpAmount <= INT128_MAX, ERR_CONVERSION_OVERFLOW);
        int128 nlpAmount = int128(txn.nlpAmount);
        require(
            spotEngine.getNlpUnlockedBalance(txn.sender).amount >= nlpAmount,
            ERR_UNLOCKED_NLP_INSUFFICIENT
        );
        int128 quoteAmount = nlpAmount.mul(oraclePriceX18);
        int128 burnFee = MathHelper.max(ONE, quoteAmount / 1000);
        quoteAmount = MathHelper.max(0, quoteAmount - burnFee);

        _validateNlpRebalance(nlpPools, nlpPoolRebalanceX18, -quoteAmount);
        for (uint128 i = 0; i < nlpPoolRebalanceX18.length; i++) {
            require(nlpPoolRebalanceX18[i] <= 0, ERR_INVALID_NLP_REBALANCE);
        }

        spotEngine.updateBalance(NLP_PRODUCT_ID, txn.sender, -nlpAmount);
        spotEngine.updateBalance(NLP_PRODUCT_ID, N_ACCOUNT, nlpAmount);

        if (quoteAmount > 0) {
            spotEngine.updateBalance(QUOTE_PRODUCT_ID, txn.sender, quoteAmount);
            _applyNlpRebalance(spotEngine, nlpPools, nlpPoolRebalanceX18);
        }

        require(
            spotEngine.getBalance(NLP_PRODUCT_ID, txn.sender).amount >= 0,
            ERR_SUBACCT_HEALTH
        );
        // Burning NLP can decrease health if the burn fee exceeds the health improvement
        // from the withdrawal. This check prevents malicious actors from deliberately
        // creating unhealthy subaccounts through NLP burns.
        require(
            getHealth(txn.sender, IProductEngine.HealthType.MAINTENANCE) >= 0,
            ERR_SUBACCT_HEALTH
        );
    }

    function _getWeightX18(
        Risk memory risk,
        int128 amount,
        IProductEngine.HealthType healthType
    ) internal pure returns (int128) {
        if (healthType == IProductEngine.HealthType.PNL) {
            return ONE;
        }

        int128 weight;
        if (amount >= 0) {
            weight = healthType == IProductEngine.HealthType.INITIAL
                ? risk.longWeightInitialX18
                : risk.longWeightMaintenanceX18;
        } else {
            weight = healthType == IProductEngine.HealthType.INITIAL
                ? risk.shortWeightInitialX18
                : risk.shortWeightMaintenanceX18;
        }

        return weight;
    }

        State memory quoteState;
        require(dt < 7 * SECONDS_PER_DAY, ERR_INVALID_TIME);
        for (uint32 i = 0; i < productIds.length; i++) {
            uint32 productId = productIds[i];
            if (productId == NLP_PRODUCT_ID) {
                continue;
            }
            State memory state = states[productId];
            if (productId == QUOTE_PRODUCT_ID) {
                quoteState = state;
            }
            if (state.totalDepositsNormalized == 0) {
                continue;
            }
            _updateState(productId, state, dt);
            _setState(productId, state);
        }
    }

            address(engineByType[IProductEngine.EngineType.PERP])
        );

        if (_finalizeSubaccount(txn, spotEngine, perpEngine)) {

                ).tryCloseIsolatedSubaccount(txn.liquidatee);

        if (

        // subsequently we can just blast the remainder of the liquidation and
        // cover the quote balance from the insurance fund at the end
        _handleLiquidationPayment(txn, spotEngine, perpEngine);
    }
