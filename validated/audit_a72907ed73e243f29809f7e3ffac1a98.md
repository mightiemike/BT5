Looking at the `x/tieredrewards` module, specifically `ClaimTierRewards` in `msg_server.go` and `claimRewardsAndUpdatesPositions` in `claim_rewards.go`, there is a concrete analog.

### Title
Duplicate Position IDs in `MsgClaimTierRewards` Enable Multiple Bonus Reward Payouts from Protocol Pool — (`File: x/tieredrewards/keeper/msg_server.go`)

---

### Summary

`ClaimTierRewards` accepts a caller-supplied list of position IDs and processes each entry independently without deduplication. When the same position ID appears more than once, the position's original (pre-update) state is used for every duplicate entry, causing `processEventsAndClaimBonus` to compute and transfer bonus coins from the `RewardsPoolName` module account multiple times for the same position within a single transaction.

---

### Finding Description

In `ClaimTierRewards` (msg_server.go lines 429–468), the handler iterates over `msg.PositionIds`, calls `getPositionState` for each ID, and appends the result to a local `positions` slice:

```go
positions := make([]types.PositionState, 0, len(msg.PositionIds))
for _, posId := range msg.PositionIds {
    pos, err := ms.getPositionState(ctx, posId)
    ...
    positions = append(positions, pos)
}
``` [1](#0-0) 

There is no deduplication check. If `positionId X` appears at index 0 and index 1, both `positions[0]` and `positions[1]` hold the **same original state** loaded from the store before any mutation.

The slice is then passed to `claimRewardsAndUpdatesPositions`:

```go
for i := range positions {
    pos := &positions[i]
    ...
    bonus, err := k.processEventsAndClaimBonus(ctx, pos)
    ...
    if err := k.setPosition(ctx, pos.Position, nil); err != nil {
``` [2](#0-1) 

Each slice element is processed independently. `setPosition` in iteration 0 persists the updated checkpoints to the store, but `positions[1]` is a separate in-memory copy still holding the original `LastEventSeq` and `LastBonusAccrual`. In iteration 1, `processEventsAndClaimBonus` re-reads validator events from the original sequence and re-computes the current-segment bonus:

```go
if bonded && val.IsBonded() {
    currentRate, err := k.getTokensPerShare(ctx, valAddr)
    bonus := k.computeSegmentBonus(*pos, tier, segmentStart, blockTime, currentRate)
    totalBonus = totalBonus.Add(bonus)
}
``` [3](#0-2) 

This bonus is then transferred from the module pool to the owner a second time:

```go
if err := k.bankKeeper.SendCoinsFromModuleToAccount(ctx, types.RewardsPoolName, ownerAddr, bonusCoins); err != nil {
``` [4](#0-3) 

Base rewards (`claimBaseRewards` → `distributionKeeper.WithdrawDelegationRewards`) are not double-paid because the distribution module resets outstanding rewards after the first withdrawal. However, the bonus pool transfer is driven entirely by the stale in-memory position state and is executed once per duplicate entry.

---

### Impact Explanation

A position owner can drain the `RewardsPoolName` module account by submitting a single `MsgClaimTierRewards` transaction with the same position ID repeated N times. The bonus accrued since the position's last `LastBonusAccrual` checkpoint is paid out N times. The only practical bound is the transaction byte/gas limit. This directly corrupts the bonus pool balance, causing a loss of funds for the protocol and other legitimate stakers who depend on that pool.

---

### Likelihood Explanation

Any position owner can construct this transaction with no special privileges. The entry path is a standard signed Cosmos SDK transaction (`MsgClaimTierRewards`). The `msg.Validate()` call and `validateClaimRewards` check ownership but neither enforces uniqueness of position IDs in the list. The attack is deterministic and repeatable. [5](#0-4) 

---

### Recommendation

Add a deduplication check in `MsgClaimTierRewards.Validate()` (or at the top of `ClaimTierRewards`) that returns an error if any position ID appears more than once in `msg.PositionIds`. A simple seen-set over the ID list is sufficient.

---

### Proof of Concept

1. Alice holds position ID `42` with accrued bonus rewards.
2. Alice submits `MsgClaimTierRewards{Owner: alice, PositionIds: [42, 42, 42, ..., 42]}` (N entries).
3. The handler loads position `42`'s state N times into `positions[0..N-1]`, all identical.
4. `claimRewardsAndUpdatesPositions` iterates:
   - **i=0**: computes bonus B for position 42, calls `SendCoinsFromModuleToAccount(RewardsPoolName, alice, B)`, persists updated checkpoints via `setPosition`.
   - **i=1**: `positions[1]` still holds the original state (original `LastBonusAccrual`, original `LastEventSeq`). Computes the same bonus B again, calls `SendCoinsFromModuleToAccount(RewardsPoolName, alice, B)` again.
   - **i=2..N-1**: repeats, paying B each time.
5. Alice receives `N × B` bonus tokens; the `RewardsPoolName` pool is debited `N × B` instead of `B`. [6](#0-5) [7](#0-6)

func (ms msgServer) ClaimTierRewards(ctx context.Context, msg *types.MsgClaimTierRewards) (*types.MsgClaimTierRewardsResponse, error) {
	if err := msg.Validate(); err != nil {
		return nil, err
	}

	positions := make([]types.PositionState, 0, len(msg.PositionIds))
	for _, posId := range msg.PositionIds {
		pos, err := ms.getPositionState(ctx, posId)
		if err != nil {
			return nil, err
		}

		if err := ms.validateClaimRewards(pos.Position, msg.Owner); err != nil {
			return nil, err
		}

		positions = append(positions, pos)
	}

	totalBase, totalBonus, err := ms.claimRewardsAndUpdatesPositions(ctx, positions)
	if err != nil {
		return nil, err
	}

	sdkCtx := sdk.UnwrapSDKContext(ctx)
	if err := sdkCtx.EventManager().EmitTypedEvent(&types.EventTierRewardsClaimed{
		Owner:        msg.Owner,
		PositionIds:  msg.PositionIds,
		BaseRewards:  totalBase,
		BonusRewards: totalBonus,
	}); err != nil {
		return nil, err
	}

	return &types.MsgClaimTierRewardsResponse{
		BaseRewards:  totalBase,
		BonusRewards: totalBonus,
		PositionIds:  msg.PositionIds,
	}, nil
}

func (k Keeper) claimRewardsAndUpdatesPositions(ctx context.Context, positions []types.PositionState) (sdk.Coins, sdk.Coins, error) {
	totalBase := sdk.NewCoins()
	totalBonus := sdk.NewCoins()

	for i := range positions {
		pos := &positions[i]

		if !pos.IsDelegated() {
			continue
		}

		base, err := k.claimBaseRewards(ctx, *pos)
		if err != nil {
			return nil, nil, err
		}
		totalBase = totalBase.Add(base...)

		bonus, err := k.processEventsAndClaimBonus(ctx, pos)
		if err != nil {
			return nil, nil, err
		}
		totalBonus = totalBonus.Add(bonus...)

		if err := k.setPosition(ctx, pos.Position, nil); err != nil {
			return nil, nil, err
		}
	}

	return totalBase, totalBonus, nil
}
