Looking at the ICS-721 NFT transfer module, specifically `processReceivedPacket` in `x/nft-transfer/keeper/packet.go`, there is a direct structural analog to the reported vulnerability class.

---

### Title
IBC NFT Voucher Denom Pre-Creation Allows Attacker to Seize Creator Privileges Over IBC Denom - (File: `x/nft-transfer/keeper/packet.go`)

### Summary

`processReceivedPacket` checks whether the IBC voucher denom already exists before creating it. Because the `voucherClassID` is fully deterministic from public on-chain data, an unprivileged attacker can pre-create that denom with themselves as creator before the first legitimate IBC NFT packet arrives. The protocol then silently skips denom creation, leaving the attacker as the permanent creator of the IBC voucher denom.

### Finding Description

In `processReceivedPacket`, when an IBC NFT packet arrives in the away-from-origin direction, the code computes a deterministic `voucherClassID` and then conditionally issues the denom only if it does not already exist:

```go
// x/nft-transfer/keeper/packet.go lines 167–171
if !k.nftKeeper.HasDenomID(ctx, voucherClassID) {
    if err := k.nftKeeper.IssueDenom(ctx, voucherClassID, voucherClassID, "", data.ClassUri, escrowAddress); err != nil {
        return err
    }
}
``` [1](#0-0) 

In the normal flow, `escrowAddress` (derived deterministically from the destination port and channel) becomes the denom creator. The `voucherClassID` is computed as:

```
classPrefix  = GetClassPrefix(destPort, destChannel)   // public
prefixedID   = classPrefix + data.ClassId              // public
classTrace   = ParseClassTrace(prefixedID)             // deterministic
voucherClassID = classTrace.IBCClassID()               // deterministic hash
``` [2](#0-1) 

All inputs are observable on-chain before any NFT transfer occurs. An attacker who knows the IBC channel configuration and the source `classId` can pre-compute `voucherClassID` and submit a `MsgIssueDenom` transaction naming themselves as creator before the first IBC packet is relayed.

When the legitimate IBC packet subsequently arrives, `HasDenomID` returns `true`, the `IssueDenom` call is skipped, and the attacker remains the permanent creator of the IBC voucher denom. The NFT module's `IsDenomCreator` authorization check then grants the attacker creator-level privileges over that denom:

```go
// x/nft/keeper/denom.go lines 81–96
func (k Keeper) IsDenomCreator(ctx sdk.Context, denomID string, address sdk.AccAddress) (types.Denom, error) {
    denom, err := k.GetDenom(ctx, denomID)
    ...
    if !creator.Equals(address) {
        return types.Denom{}, sdkerrors.Wrapf(types.ErrUnauthorized, ...)
    }
    return denom, nil
}
``` [3](#0-2) 

The `SetDenom` function imposes no restriction on the denom ID format, so there is no barrier preventing a user from registering a denom whose ID matches the IBC voucher format: [4](#0-3) 

### Impact Explanation

**Impact: High.** The attacker becomes the permanent creator of the IBC voucher denom. In the NFT module, the denom creator is the authorized party for minting new NFTs within that denom. This allows the attacker to mint arbitrary unbacked NFT vouchers in the IBC denom — NFTs that appear to be legitimate cross-chain IBC NFTs but are not backed by any escrowed asset on the source chain. The corrupted value is the NFT supply and ownership integrity of the IBC voucher denom. Any downstream protocol or marketplace that trusts the IBC denom's NFT supply is affected.

### Likelihood Explanation

**Likelihood: Medium.** The attack requires the attacker to act before the first IBC NFT transfer on a given channel for a given source classID. IBC channel openings are publicly observable, and the source classID is known from the `MsgTransfer` that initiates the transfer. A monitoring attacker can front-run the first relayed packet. The cost is a single `MsgIssueDenom` transaction. No privileged access is required.

### Recommendation

In `processReceivedPacket`, after the `HasDenomID` check, verify that the existing denom's creator matches the expected escrow address. If it does not, the packet should be rejected with an error rather than silently proceeding:

```go
if !k.nftKeeper.HasDenomID(ctx, voucherClassID) {
    if err := k.nftKeeper.IssueDenom(ctx, voucherClassID, voucherClassID, "", data.ClassUri, escrowAddress); err != nil {
        return err
    }
} else {
    // Verify the existing denom was created by the expected escrow address
    denom, err := k.nftKeeper.GetDenom(ctx, voucherClassID)
    if err != nil {
        return err
    }
    if denom.Creator != escrowAddress.String() {
        return types.ErrInvalidDenom.Wrapf("IBC voucher denom %s has unexpected creator", voucherClassID)
    }
}
```

Additionally, consider restricting `MsgIssueDenom` from accepting denom IDs that match the IBC voucher prefix (`ibc/`) to prevent pre-registration entirely.

### Proof of Concept

1. Observe an IBC channel open between chain A (source) and chain B (destination) for NFT port.
2. Note the destination port (`nft`) and channel ID (e.g., `channel-0`), and the source `classId` (e.g., `myNFTClass`).
3. Compute `voucherClassID`:
   - `prefixedClassID = "nft/channel-0/myNFTClass"`
   - `classTrace = ParseClassTrace(prefixedClassID)` → `{Path: "nft/channel-0", BaseClassId: "myNFTClass"}`
   - `voucherClassID = "ibc/" + hex(sha256("nft/channel-0/myNFTClass"))`
4. On chain B, submit `MsgIssueDenom` with `id = voucherClassID` and attacker address as creator, before any NFT transfer occurs.
5. Relay a legitimate IBC NFT transfer from chain A to chain B. `processReceivedPacket` calls `HasDenomID` → `true`, skips `IssueDenom`, mints the NFT to the legitimate receiver.
6. Attacker, as denom creator, calls `MsgMintNFT` with `denom_id = voucherClassID` to mint additional unbacked NFT vouchers to any address, bypassing the IBC escrow invariant. [5](#0-4) [6](#0-5)

func (k Keeper) processReceivedPacket(ctx sdk.Context, packet channeltypes.Packet,
	data types.NonFungibleTokenPacketData,
) error {
	receiver, err := sdk.AccAddressFromBech32(data.Receiver)
	if err != nil {
		return err
	}

	isAwayFromOrigin := types.IsAwayFromOrigin(packet.GetSourcePort(), packet.GetSourceChannel(), data.ClassId)

	// create the escrow address for creating denom and minting nft
	escrowAddress := types.GetEscrowAddress(packet.GetDestPort(), packet.GetDestChannel())

	if isAwayFromOrigin {
		// since SendPacket did not prefix the classID, we must prefix classID here
		classPrefix := types.GetClassPrefix(packet.GetDestPort(), packet.GetDestChannel())
		// NOTE: sourcePrefix contains the trailing "/"
		prefixedClassID := classPrefix + data.ClassId

		// construct the class trace from the full raw classID
		classTrace := types.ParseClassTrace(prefixedClassID)
		if !k.HasClassTrace(ctx, classTrace.Hash()) {
			k.SetClassTrace(ctx, classTrace)
		}

		voucherClassID := classTrace.IBCClassID()

		if !k.nftKeeper.HasDenomID(ctx, voucherClassID) {
			if err := k.nftKeeper.IssueDenom(ctx, voucherClassID, voucherClassID, "", data.ClassUri, escrowAddress); err != nil {
				return err
			}
		}
		sdkCtx := sdk.UnwrapSDKContext(ctx)
		sdkCtx.EventManager().EmitEvent(
			sdk.NewEvent(
				types.EventTypeClassTrace,
				sdk.NewAttribute(types.AttributeKeyTraceHash, classTrace.Hash().String()),
				sdk.NewAttribute(types.AttributeKeyClassID, voucherClassID),
			),
		)

		for i, tokenID := range data.TokenIds {
			if err := k.nftKeeper.MintNFT(ctx, voucherClassID, tokenID, "", data.TokenUris[i], "", escrowAddress, receiver); err != nil {

func (k Keeper) SetDenom(ctx sdk.Context, denom types.Denom) error {
	if k.HasDenomID(ctx, denom.Id) {
		return sdkerrors.Wrapf(types.ErrInvalidDenom, "denomID %s has already exists", denom.Id)
	}

	if k.HasDenomNm(ctx, denom.Name) {
		return sdkerrors.Wrapf(types.ErrInvalidDenom, "denomName %s has already exists", denom.Name)
	}

	store := ctx.KVStore(k.storeKey)
	bz := k.cdc.MustMarshal(&denom)
	store.Set(types.KeyDenomID(denom.Id), bz)
	store.Set(types.KeyDenomName(denom.Name), []byte(denom.Id))
	return nil

func (k Keeper) IsDenomCreator(ctx sdk.Context, denomID string, address sdk.AccAddress) (types.Denom, error) {

	if err != nil {
