### Title
Sanctioned EOA Bypasses On-Chain Sanctions Check via Proxy Contract — (`core/contracts/Airdrop.sol`)

### Summary

The `_verifyProof` function checks `ISanctionsList(sanctions).isSanctioned(sender)` where `sender` is always `msg.sender` — the immediate caller. When a sanctioned EOA deploys a contract and calls `claim()` through it, `msg.sender` is the contract address, which is not on the sanctions list. The on-chain guard is therefore trivially bypassed.

---

### Finding Description

In `_claim`, `sender` is hardcoded to `msg.sender`: [1](#0-0) 

This `sender` value is passed directly to `_verifyProof`, which performs the sanctions check against it: [2](#0-1) 

The same `sender` (the contract address) is then used to construct the Merkle leaf: [3](#0-2) 

And tokens are transferred to it: [4](#0-3) 

Sanctions lists (e.g., Chainalysis) screen **EOA addresses**, not contract addresses. A freshly deployed contract will return `isSanctioned(contractAddress) == false`. The Merkle tree is built off-chain from protocol activity — any contract address that traded or earned rewards will naturally appear in it. The owner building the tree has no reliable on-chain signal that a given contract is controlled by a sanctioned EOA.

**Concrete attack path:**
1. Sanctioned EOA deploys `ProxyClaimer` contract.
2. `ProxyClaimer` interacts with the Nado protocol (e.g., trades), earning a weekly reward allocation.
3. Owner builds the Merkle tree from on-chain activity; `ProxyClaimer`'s address is included with `totalAmount`.
4. Owner calls `registerMerkleRoot(week, root)` — no on-chain screening of contract addresses occurs.
5. Sanctioned EOA calls `ProxyClaimer.triggerClaim()`, which calls `Airdrop.claim(...)`.
6. `msg.sender` = `ProxyClaimer` address → `isSanctioned(ProxyClaimer) == false` → check passes.
7. Merkle proof verifies (leaf was built with `ProxyClaimer` address).
8. `safeTransfer(token, ProxyClaimer, totalAmount)` executes.
9. Sanctioned EOA withdraws tokens from `ProxyClaimer`.

---

### Impact Explanation

A sanctioned principal receives their full airdrop token allocation. The invariant that "sanctioned actors must never receive airdrop tokens" is broken. The tokens are now under the sanctioned actor's control via the proxy contract.

---

### Likelihood Explanation

Sanctions lists screen EOAs, not contracts. Any sanctioned actor who has previously interacted with the protocol via a contract (a common pattern for privacy) will have a contract address in the Merkle tree. The off-chain tree builder has no reliable way to trace contract ownership back to a sanctioned EOA without additional tooling. The on-chain code provides zero defense against this pattern.

---

### Recommendation

Add a check that the caller is not a contract (i.e., enforce EOA-only claiming), or additionally check `tx.origin`:

```solidity
require(
    !ISanctionsList(sanctions).isSanctioned(tx.origin),
    "origin is sanctioned."
);
require(msg.sender == tx.origin, "contracts cannot claim");
```

Alternatively, the off-chain Merkle tree construction pipeline must trace contract ownership and exclude contracts controlled by sanctioned actors — but this is not enforceable on-chain and is therefore an insufficient sole defense.

```solidity
pragma solidity ^0.8.0;

interface IAirdrop {
    struct ClaimProof { uint32 week; uint256 totalAmount; bytes32[] proof; }
    function claim(ClaimProof[] calldata) external;
}

contract ProxyClaimer {
    IAirdrop airdrop;
    address owner;
    constructor(address _airdrop) { airdrop = IAirdrop(_airdrop); owner = msg.sender; }
    function triggerClaim(IAirdrop.ClaimProof[] calldata proofs) external {
        require(msg.sender == owner);
        airdrop.claim(proofs); // msg.sender = address(this), not the sanctioned EOA
    }
    function withdraw(address token) external {
        require(msg.sender == owner);
        IERC20(token).transfer(owner, IERC20(token).balanceOf(address(this)));
    }
}
```

```text
    function _verifyProof(
        uint32 week,
        address sender,
        uint256 totalAmount,
        bytes32[] calldata proof
    ) internal {
        require(claimed[week][sender] == 0, "Already claimed.");
        require(
            merkleRoots[week] != bytes32(0),
            "Week hasn't been registered."
        );
        require(
            !ISanctionsList(sanctions).isSanctioned(sender),
            "address is sanctioned."
        );
        bytes32 leaf = keccak256(
            bytes.concat(keccak256(abi.encode(sender, totalAmount)))
        );
        bool isValidLeaf = MerkleProof.verify(proof, merkleRoots[week], leaf);
        require(isValidLeaf, "Invalid proof.");
        claimed[week][sender] = totalAmount;
    }
```
