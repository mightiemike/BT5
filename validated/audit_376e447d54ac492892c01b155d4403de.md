Audit Report

## Title
`SwapAllowlistExtension` checks router address instead of real user, enabling allowlist bypass via `MetricOmmSimpleRouter` — (`metric-periphery/contracts/extensions/SwapAllowlistExtension.sol`)

## Summary

`SwapAllowlistExtension.beforeSwap` gates swaps on `allowedSwapper[pool][sender]`, where `sender` is the `msg.sender` of the pool's `swap()` call. When a user routes through `MetricOmmSimpleRouter`, the pool's `msg.sender` is the router contract, not the user. If the pool admin whitelists the router to enable router-mediated swaps, every unprivileged user bypasses the per-user allowlist entirely. If the router is not whitelisted, all individually-whitelisted users are blocked from using the router.

## Finding Description

**Root cause — `MetricOmmPool.swap` passes `msg.sender` (the router) as `sender`:** [1](#0-0) 

**`ExtensionCalling._beforeSwap` forwards that value unchanged to every configured extension:** [2](#0-1) 

**`SwapAllowlistExtension.beforeSwap` evaluates `allowedSwapper[msg.sender][sender]`, i.e. `allowedSwapper[pool][router]`, never the real user:** [3](#0-2) 

**`MetricOmmSimpleRouter.exactInputSingle` calls `pool.swap` directly, making the router `msg.sender` to the pool:** [4](#0-3) 

The same identity mismatch applies to `exactInput` (all hops), `exactOutputSingle`, and `exactOutput` (all recursive hops via `_exactOutputIterateCallback`), since every pool call originates from the router contract. [5](#0-4) 

**Outcome A — bypass:** Admin whitelists the router so router-mediated swaps work. Because the check resolves to `allowedSwapper[pool][router] == true`, every caller of `exactInputSingle` passes regardless of individual allowlist status. The per-user allowlist is completely inoperative.

**Outcome B — DoS:** Admin whitelists individual users but not the router. Every whitelisted user is blocked from using the router because `allowedSwapper[pool][router] == false`.

No existing guard in `SwapAllowlistExtension`, `ExtensionCalling`, or `MetricOmmPool` recovers the real user identity from the call context.

## Impact Explanation

Outcome A is the fund-impacting path. A pool restricted to a curated set of counterparties (e.g., KYC'd addresses, protocol-owned addresses) can be freely traded against by any public EOA through the public router. The pool's LP assets are exposed to unrestricted swap flow, defeating the economic protection the allowlist was meant to provide. This matches the "broken core pool functionality causing loss of funds" and "admin-boundary break" impact categories: an unprivileged user bypasses a pool-admin-configured access control, gaining swap access to a restricted pool.

## Likelihood Explanation

`MetricOmmSimpleRouter` is a public, permissionless contract — any EOA can call `exactInputSingle`. A pool admin who deploys `SwapAllowlistExtension` and wants to support the standard router flow will naturally whitelist the router address, directly triggering Outcome A. No privileged cooperation is required; the bypass is reachable by any EOA in a single transaction.

## Recommendation

The extension must gate the economically relevant actor, not the intermediary. Two options:

1. **Encode the real user in `extensionData`:** Have the router forward `msg.sender` (the real user) inside `extensionData`. `SwapAllowlistExtension.beforeSwap` decodes this field when `sender` is a recognized router address and checks `allowedSwapper[pool][decodedUser]` instead.

2. **Check `recipient` as a fallback:** When `sender` is a known router, fall back to checking the `recipient` argument (the second parameter of `beforeSwap`), which in `exactInputSingle` is the user-supplied `params.recipient`. This is simpler but requires the extension to maintain a registry of trusted router addresses.

The simplest safe fix is option 1: the router appends `abi.encode(msg.sender)` to `extensionData`, and the extension decodes and verifies it when `sender` matches a known router.

## Proof of Concept

```solidity
// Pool configured with SwapAllowlistExtension.
// Admin whitelists alice and the router (to allow router-mediated swaps).
extension.setAllowedToSwap(address(pool), alice, true);
extension.setAllowedToSwap(address(pool), address(router), true);

// bob is NOT whitelisted.
vm.startPrank(bob);
token0.approve(address(router), type(uint256).max);
router.exactInputSingle(
    IMetricOmmSimpleRouter.ExactInputSingleParams({
        pool: address(pool),
        recipient: bob,
        tokenIn: address(token0),
        zeroForOne: true,
        amountIn: 1000,
        amountOutMinimum: 0,
        priceLimitX64: 0,
        deadline: block.timestamp,
        extensionData: ""
    })
);
// Swap succeeds — pool sees msg.sender = router (whitelisted), never checks bob.
vm.stopPrank();

assertFalse(extension.isAllowedToSwap(address(pool), bob)); // bob was never whitelisted
```
