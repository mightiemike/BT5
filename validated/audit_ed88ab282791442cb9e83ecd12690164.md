### Title
Deprecated `verify_transaction_inclusion` Remains Publicly Callable Without Coinbase Proof Validation, Enabling 64-Byte Merkle Forgery - (`contract/src/lib.rs`)

### Summary

`verify_transaction_inclusion` (v1) and `verify_transaction_inclusion_v2` both verify Bitcoin transaction inclusion against the same on-chain `headers_pool`, but only v2 validates a coinbase merkle proof to prevent the 64-byte internal-node forgery attack. v1 is still a live, publicly callable NEAR contract method. Any unprivileged NEAR caller can invoke v1 with a crafted internal Merkle tree node as `tx_id` and receive a `true` result for a transaction that does not exist.

### Finding Description

`verify_transaction_inclusion_v2` was introduced specifically to close the 64-byte transaction Merkle proof forgery vulnerability (referenced in the code comment: https://www.bitmex.com/blog/64-Byte-Transactions). It does so by requiring the caller to also supply a coinbase merkle proof and validating it against the block's `merkle_root` before delegating to v1:

```rust
// v2: coinbase proof validated first
require!(
    merkle_tools::compute_root_from_merkle_proof(
        args.coinbase_tx_id.clone(),
        0usize,
        &args.coinbase_merkle_proof,
    ) == header.block_header.merkle_root,
    "Incorrect coinbase merkle proof"
);
#[allow(deprecated)]
self.verify_transaction_inclusion(args.into())
```

v1, however, performs no such check. It accepts any `tx_id` and any `merkle_proof`, computes the Merkle root, and returns `true` if it matches the stored block header's `merkle_root`:

```rust
merkle_tools::compute_root_from_merkle_proof(
    args.tx_id,
    usize::try_from(args.tx_index).unwrap(),
    &args.merkle_proof,
) == header.block_header.merkle_root
```

The `#[deprecated]` Rust attribute is a compiler hint only. It does not remove the function from the compiled WASM or prevent it from being dispatched as a NEAR contract call. The function carries `#[pause]` but no access-control restriction, so any NEAR account can call it.

The 64-byte forgery works as follows: Bitcoin's Merkle tree computes internal nodes by hashing the 32-byte concatenation of two child hashes. An attacker who knows the Merkle tree of any mainchain block can pick any internal node `N` at depth `d`, treat it as a `tx_id`, and supply the `d`-step sibling path to the root. `compute_root_from_merkle_proof` will reconstruct the correct root and v1 will return `true`, even though `N` is not a transaction hash.

### Impact Explanation

Any downstream NEAR contract that calls `verify_transaction_inclusion` (v1) to gate a fund release, a bridge mint, or any other irreversible action can be deceived into accepting a proof for a Bitcoin transaction that never existed. The attacker does not need to mine any block or control any privileged role; they only need knowledge of the Merkle tree of any already-accepted mainchain block, which is public information.

### Likelihood Explanation

The entry path is fully permissionless: any NEAR account can call `verify_transaction_inclusion` with a crafted `ProofArgs`. The required Merkle tree data is publicly available from any Bitcoin node. The only prerequisite is that the target block is already in the contract's `headers_pool`, which is the normal operating state of the light client.

### Recommendation

Remove `verify_transaction_inclusion` from the public contract interface entirely, or gate it with an access-control role so it cannot be called by arbitrary accounts. If backward compatibility must be preserved, apply the same coinbase-proof validation inside v1 itself rather than only in v2. The `#[deprecated]` attribute alone provides no on-chain protection.

### Proof of Concept

1. Identify any block `B` in the contract's mainchain (e.g., via `get_block_hash_by_height`).
2. Obtain the full Merkle tree of `B` from a Bitcoin node.
3. Select any internal node `N` at depth `d` (e.g., the root's left child). Compute the sibling path of length `d` from `N` to the Merkle root.
4. Call `verify_transaction_inclusion` with:
   - `tx_id = N` (a 32-byte internal node hash, not a real txid)
   - `tx_block_blockhash = B`
   - `tx_index` = the index corresponding to `N`'s position
   - `merkle_proof` = the `d`-step sibling path
   - `confirmations = 1`
5. The call returns `true`. No such transaction exists in block `B`.

`verify_transaction_inclusion_v2` rejects the same input because the coinbase proof check anchors the tree structure to a real transaction at index 0, preventing internal-node substitution. [1](#0-0) [2](#0-1) [3](#0-2) [4](#0-3)

    #[deprecated(
        since = "0.5.0",
        note = "Use `verify_transaction_inclusion_v2` instead."
    )]
    #[pause]
    pub fn verify_transaction_inclusion(&self, #[serializer(borsh)] args: ProofArgs) -> bool {
        require!(
            args.confirmations <= self.gc_threshold,
            "The required number of confirmations exceeds the number of blocks stored in memory"
        );

        let heaviest_block_header = self
            .headers_pool
            .get(&self.mainchain_tip_blockhash)
            .unwrap_or_else(|| env::panic_str(ERR_KEY_NOT_EXIST));
        let target_block_height = self
            .mainchain_header_to_height
            .get(&args.tx_block_blockhash)
            .unwrap_or_else(|| env::panic_str("block does not belong to the current main chain"));

        // Check requested confirmations. No need to compute proof if insufficient confirmations.
        require!(
            (heaviest_block_header.block_height).saturating_sub(target_block_height) + 1
                >= args.confirmations,
            "Not enough blocks confirmed"
        );

        let header = self
            .headers_pool
            .get(&args.tx_block_blockhash)
            .unwrap_or_else(|| env::panic_str("cannot find requested transaction block"));

        require!(!args.merkle_proof.is_empty(), "Merkle proof is empty");

        // compute merkle tree root and check if it matches block's original merkle tree root
        merkle_tools::compute_root_from_merkle_proof(
            args.tx_id,
            usize::try_from(args.tx_index).unwrap(),
            &args.merkle_proof,
        ) == header.block_header.merkle_root
    }
