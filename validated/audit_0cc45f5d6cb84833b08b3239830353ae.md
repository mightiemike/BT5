### Title
SwapAllowlistExtension Checks Router Address Instead of End User, Allowing Any User to Bypass Swap Allowlist — (`metric-periphery/contracts/extensions/SwapAllowlistExtension.sol`)

### Summary

`SwapAllowlistExtension.beforeSwap` gates swaps by checking the `sender` argument passed by the pool. The pool passes `msg.sender` of `pool.swap()` as `sender`. When a user routes through `MetricOmmSimpleRouter`, `msg.sender` of `pool.swap()` is the router contract, not the end user. If the pool admin allowlists the router address (required for any router-mediated swap to succeed), every user on the network can bypass the allowlist by calling through the router.

### Finding Description

**Pool passes `msg.sender` as `sender` to the extension:** [1](#0-0) 

`_beforeSwap` is called with `msg.sender` as the first argument. When the router calls `pool.swap(...)`, `msg.sender` is the router contract address.

**`ExtensionCalling._beforeSwap` forwards this as `sender`:** [2](#0-1) 

**`SwapAllowlistExtension.beforeSwap` checks `allowedSwapper[pool][sender]`:** [3](#0-2) 

`msg.sender` here is the pool (correct), and `sender` is the router address (not the end user). The check becomes `allowedSwapper[pool][router]`.

**The router calls `pool.swap()` directly with no user identity forwarding:** [4](#0-3) 

The router passes `params.recipient` (the output destination) but the pool records `msg.sender` (the router) as `sender`. There is no mechanism to forward the actual end user's address.

**The dilemma for pool admins:**

- If the admin does **not** allowlist the router → allowlisted users cannot use the standard router UX at all; they must call `pool.swap()` directly.
- If the admin **does** allowlist the router (the only way to enable router-based swaps) → `allowedSwapper[pool][router] = true`, and every user on the network can bypass the allowlist by routing through `MetricOmmSimpleRouter`.

### Impact Explanation

A curated pool's swap allowlist is completely ineffective for router-mediated swaps. Any unprivileged user can bypass the allowlist by calling `MetricOmmSimpleRouter.exactInputSingle` (or `exactInput`, `exactOutputSingle`, `exactOutput`) targeting the restricted pool. This allows unauthorized parties to trade against LP funds in pools that were intended to be restricted (e.g., KYC-gated, counterparty-restricted, or compliance-controlled pools). LP funds are exposed to swaps from actors the pool admin explicitly intended to exclude.

### Likelihood Explanation

High. `MetricOmmSimpleRouter` is the standard user-facing swap entrypoint. Any pool admin who wants allowlisted users to have a normal UX must allowlist the router, which simultaneously opens the gate to all users. The bypass requires no special privileges, no flash loans, and no complex setup — a single `exactInputSingle` call suffices.

### Recommendation

Pass the actual end user's address through the extension call chain rather than the immediate `msg.sender`. One approach: add a `swapper` parameter to `pool.swap()` that callers supply explicitly (validated against `msg.sender` or a trusted router registry), and pass that as `sender` to extensions. Alternatively, `SwapAllowlistExtension` should check `recipient` or require the router to forward the originating user via `extensionData`, with the extension decoding and verifying it.

### Proof of Concept

```
1. Pool admin deploys pool with SwapAllowlistExtension.
2. Admin calls setAllowedToSwap(pool, alice, true)  — only alice should swap.
3. Admin calls setAllowedToSwap(pool, router, true) — required for alice to use the router.
4. Bob (not allowlisted) calls:
       router.exactInputSingle(ExactInputSingleParams({
           pool: restrictedPool,
           recipient: bob,
           zeroForOne: true,
           amountIn: X,
           ...
       }))
5. pool.swap() is called with msg.sender = router.
6. _beforeSwap(router, ...) → SwapAllowlistExtension checks allowedSwapper[pool][router] == true → passes.
7. Bob's swap executes successfully despite not being on the allowlist.
``` [3](#0-2) [1](#0-0) [5](#0-4)
