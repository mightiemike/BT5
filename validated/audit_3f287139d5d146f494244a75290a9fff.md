### Title
Unchecked `transferFrom` Return Value in `replaceUsdcEWithUsdc` Enables usdcE Drain Without Providing USDC - (File: `core/contracts/ContractOwner.sol`)

---

### Summary

`ContractOwner.replaceUsdcEWithUsdc` calls `IERC20Base(usdc).transferFrom(msg.sender, directDepositV1, balance)` without checking the return value. The function then unconditionally withdraws usdcE from the `DirectDepositV1` address and sends it to `msg.sender` via `safeTransfer`. If the `transferFrom` returns `false` (silent failure), the caller receives usdcE tokens without having provided any USDC, draining the DDA's usdcE balance.

---

### Finding Description

`replaceUsdcEWithUsdc` is a public, permissionless function (no access modifier) that is intended to swap usdcE held in a `DirectDepositV1` (DDA) address for USDC by pulling USDC from the caller and returning usdcE. [1](#0-0) 

The critical sequence is:

1. **Line 616** — `IERC20Base(usdc).transferFrom(msg.sender, directDepositV1, balance)` — pulls USDC from caller. **Return value is not checked.**
2. **Line 617** — `DirectDepositV1(directDepositV1).withdraw(IIERC20Base(usdcE))` — withdraws all usdcE from the DDA to `ContractOwner`.
3. **Line 618** — `IERC20Base(usdcE).safeTransfer(msg.sender, balance)` — sends usdcE to caller using the safe wrapper that **does** check the return value.

The inconsistency is stark: `safeTransfer` is used for the outbound usdcE transfer but the inbound USDC `transferFrom` is unchecked. The `ERC20Helper.safeTransferFrom` wrapper exists in the codebase precisely for this purpose but is not used here. [2](#0-1) 

If the USDC token at `0x2D270e6886d130D724215A266106e6832161EAEd` on chain 57073 returns `false` on a failed `transferFrom` (e.g., insufficient allowance, paused token, or a non-standard implementation), execution continues uninterrupted and the caller receives usdcE for free.

---

### Impact Explanation

An attacker who calls `replaceUsdcEWithUsdc` for any subaccount whose DDA holds usdcE, without having approved USDC or with zero USDC balance, will receive the full usdcE balance of that DDA if the `transferFrom` silently returns `false`. This is a direct, unrecoverable token loss for the DDA owner — the usdcE is transferred out and the USDC is never deposited in exchange.

---

### Likelihood Explanation

The function is callable by any unprivileged address with no preconditions beyond the chain ID check (`block.chainid == 57073`) and the DDA existing. The USDC token at the hardcoded address on chain 57073 (Ink chain) may follow the standard Circle implementation that reverts on failure, which would prevent exploitation in the common case. However, the code is structurally broken: it does not enforce the invariant that USDC was actually received before releasing usdcE. Any deviation in token behavior (pause, upgrade, non-standard return) directly enables the drain. The inconsistency with `safeTransfer` on the same line confirms this is an unintentional omission.

---

### Recommendation

Replace the bare `transferFrom` call with the `ERC20Helper.safeTransferFrom` wrapper already present in the codebase:

```solidity
// Before (line 616):
IERC20Base(usdc).transferFrom(msg.sender, directDepositV1, balance);

// After:
IERC20Base(usdc).safeTransferFrom(msg.sender, directDepositV1, balance);
``` [3](#0-2) 

---

### Proof of Concept

1. Identify any DDA (`directDepositV1Address[subaccount]`) on chain 57073 that holds a non-zero usdcE balance.
2. Call `ContractOwner.replaceUsdcEWithUsdc(subaccount)` from an EOA with zero USDC allowance or zero USDC balance.
3. If the USDC token returns `false` instead of reverting on the failed `transferFrom`, execution proceeds to step 2 and 3 of the function body.
4. `DirectDepositV1.withdraw` transfers all usdcE from the DDA to `ContractOwner`.
5. `safeTransfer` sends the full usdcE balance to the attacker's address.
6. The DDA's usdcE is drained; no USDC was deposited. [4](#0-3)

    function replaceUsdcEWithUsdc(bytes32 subaccount) external {
        require(block.chainid == 57073, ERR_UNAUTHORIZED);
        address payable directDepositV1 = directDepositV1Address[subaccount];
        require(directDepositV1 != address(0), "no dda");
        address usdcE = 0xF1815bd50389c46847f0Bda824eC8da914045D14;
        address usdc = 0x2D270e6886d130D724215A266106e6832161EAEd;
        uint256 balance = IERC20Base(usdcE).balanceOf(directDepositV1);
        if (balance > 0) {
            IERC20Base(usdc).transferFrom(msg.sender, directDepositV1, balance);
            DirectDepositV1(directDepositV1).withdraw(IIERC20Base(usdcE));
            IERC20Base(usdcE).safeTransfer(msg.sender, balance);
        }
    }
```

**File:** core/contracts/libraries/ERC20Helper.sol (L23-42)
```text
    function safeTransferFrom(
        IERC20Base self,
        address from,
        address to,
        uint256 amount
    ) internal {
        (bool success, bytes memory data) = address(self).call(
            abi.encodeWithSelector(
                IERC20Base.transferFrom.selector,
                from,
                to,
                amount
            )
        );

        require(
            success && (data.length == 0 || abi.decode(data, (bool))),
            ERR_TRANSFER_FAILED
        );
    }
```
