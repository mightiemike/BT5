Audit Report

## Title
SwapAllowlistExtension Bypass via MetricOmmSimpleRouter: Router Address Replaces End-User in Allowlist Check — (File: metric-periphery/contracts/extensions/SwapAllowlistExtension.sol)

## Summary
`SwapAllowlistExtension.beforeSwap` gates swaps by checking the `sender` argument, which is the immediate `msg.sender` of `pool.swap()`. When a user routes through `MetricOmmSimpleRouter`, the pool receives `msg.sender = router` and the extension checks `allowedSwapper[pool][router]` instead of `allowedSwapper[pool][user]`. Any pool admin who allowlists the router to enable router-mediated swaps on a restricted pool inadvertently grants every public user the ability to bypass the allowlist.

## Finding Description
`MetricOmmPool.swap` passes `msg.sender` verbatim as the `sender` argument to `_beforeSwap`: [1](#0-0) 

`SwapAllowlistExtension.beforeSwap` then checks `allowedSwapper[msg.sender][sender]`, where `msg.sender` is the pool and `sender` is whoever called `pool.swap()`: [2](#0-1) 

`MetricOmmSimpleRouter.exactInputSingle` calls `pool.swap()` directly, making the router the `msg.sender` of that call: [3](#0-2) 

The same pattern applies to `exactInput` (line 104) and `exactOutputSingle` (line 136). For router-mediated swaps to work at all on a restricted pool, the pool admin must call `setAllowedToSwap(pool, router, true)`. Once the router is allowlisted, the check `allowedSwapper[pool][router]` returns `true` for every user who routes through it, regardless of whether that user is individually allowlisted. The extension has no mechanism to inspect the originating user (`tx.origin` is not used, and `extensionData` is not decoded for an end-user address). [4](#0-3) 

## Impact Explanation
Any address not on the allowlist can trade against restricted LP funds by calling `MetricOmmSimpleRouter.exactInputSingle` (or `exactInput`/`exactOutputSingle`) targeting the restricted pool. The pool admin's intent — to restrict which addresses can deplete LP reserves — is silently nullified. This is a direct loss-of-access-control over LP principal, matching the "allowlist path" impact gate: swap allowlist checks must cover the exact actor intended and cannot be bypassed through the router.

## Likelihood Explanation
The trigger is unprivileged: any user can call the public router. The only precondition is that the pool admin has allowlisted the router, which is the natural and necessary production step to support router-mediated swaps on a restricted pool. No malicious setup is required; the condition is created by the pool admin's own legitimate configuration action.

## Recommendation
The extension must gate the actual end user, not the intermediary. Two viable approaches:

1. **Forward the originating user through `extensionData`**: the router encodes `msg.sender` (the end user) into `extensionData` before calling `pool.swap()`, and the extension decodes and checks that address against the allowlist. This requires a trusted encoding convention between the router and extension.

2. **Router registry with user forwarding**: the extension maintains a registry of trusted routers; when `sender` is a known router, it requires the actual user address to be present in `extensionData` and checks that address instead of the router address.

Either approach requires the extension to not treat the router as the gated identity.

## Proof of Concept
```
Setup:
  pool admin deploys pool with SwapAllowlistExtension
  pool admin calls swapExtension.setAllowedToSwap(pool, address(router), true)
    → router is allowlisted so that router-mediated swaps work
  pool admin does NOT allowlist alice (alice is an unauthorized user)

Attack:
  alice calls router.exactInputSingle({pool: restrictedPool, ...})
  router calls pool.swap(recipient, ...) with msg.sender = router
  pool calls _beforeSwap(sender=router, ...)
  extension checks allowedSwapper[pool][router] → true
  swap executes; alice receives output tokens from restricted LP funds

Expected: revert NotAllowedToSwap (alice is not allowlisted)
Actual:   swap succeeds (router is allowlisted, alice is invisible to the check)
```
