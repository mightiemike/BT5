### Title
Silent Permanent Loss of Accrued Bonus Rewards When Bonus Pool Is Empty During Redelegation Slash - (File: `x/tieredrewards/keeper/slash.go`)

---

### Summary

In `slashRedelegationPosition`, when `processEventsAndClaimBonus` returns `ErrInsufficientBonusPool`, the error is deliberately swallowed to prevent a chain halt. However, by the time the error is returned, the function has already (1) advanced the in-memory `pos.LastBonusAccrual` and `pos.LastEventSeq` checkpoints, and (2) decremented validator event reference counts in the store. The caller then persists the position with these advanced checkpoints via `setPositionWithState`, permanently destroying the user's ability to ever claim the accrued bonus.

---

### Finding Description

`processEventsAndClaimBonus` performs state mutations before it checks pool sufficiency: [1](#0-0) 

For each pending event the loop calls `pos.UpdateLastEventSeq(entry.Seq)` (in-memory) and `k.decrementEventRefCount(ctx, valAddr, entry.Seq)` (store write). After the loop: [2](#0-1) 

`applyBonusAccrualCheckpoint` advances `pos.LastBonusAccrual` to the current block time. Only **after** all of these mutations does the pool balance check fire: [3](#0-2) 

When `ErrInsufficientBonusPool` is returned, the caller `slashRedelegationPosition` swallows it: [4](#0-3) 

Execution then falls through to `setPositionWithState`, which persists the position with the already-advanced `LastBonusAccrual` and `LastEventSeq`: [5](#0-4) 

The validator events whose reference counts were decremented are now garbage-collected or have lower counts. The position's checkpoints are advanced past the accrual window. The bonus is permanently unclaimable.

The `applyBonusAccrualCheckpoint` helper confirms the checkpoint is set to `blockTime` (or `ExitUnlockAt` if exiting) unconditionally before the pool check: [6](#0-5) 

---

### Impact Explanation

A position owner whose position is actively redelegating at the time of a validator slash, and whose accrued bonus exceeds the current pool balance, permanently loses all bonus rewards that accrued since their last claim. The corrupted value is the owner's earned bonus tokens (denominated in the bond denom, drawn from the `RewardsPoolName` module account). There is no recovery path: the checkpoints are advanced, the events are consumed, and no retry mechanism exists for the hook path.

---

### Likelihood Explanation

Two realistic conditions must coincide:

1. **Active redelegation**: Any user who called `MsgTierRedelegate` has a live `RedelegationMappings` entry until the SDK unbonding period completes (up to 28 days per `solo_machine.jsonnet`). [7](#0-6) 

2. **Empty or insufficient pool**: The pool is funded externally and can be depleted by normal reward claims. An adversary who controls a validator can time a downtime slash (by stopping their node) to coincide with a depleted pool, targeting victims with large accrued-but-unclaimed bonuses.

Both conditions are normal operational states. The window of exposure for each redelegation is the full SDK redelegation unbonding period.

---

### Recommendation

Move `applyBonusAccrualCheckpoint` and `pos.UpdateLastEventSeq` updates to **after** the pool balance check succeeds, so that if the pool is insufficient the in-memory position state is not advanced. Alternatively, when `ErrInsufficientBonusPool` is caught in `slashRedelegationPosition`, restore the original checkpoint values before calling `setPositionWithState`, so the user can claim the bonus once the pool is replenished.

---

### Proof of Concept

1. User calls `MsgTierRedelegate` on position P to validator V2. A `RedelegationMappings[unbondingId → P]` entry is created.
2. Bonus accrues on P for several days (large `totalBonus` pending).
3. The bonus pool is depleted by other claimants.
4. Validator V2 is slashed for downtime. The SDK calls `BeforeRedelegationSlashed(ctx, unbondingId, sharesToUnbond)`.
5. `slashRedelegationPosition` is entered. `processEventsAndClaimBonus` is called with `&pos`.
6. Inside `processEventsAndClaimBonus`: all pending events are walked; `decrementEventRefCount` writes to the store for each; `applyBonusAccrualCheckpoint` advances `pos.LastBonusAccrual` to `blockTime`; `sufficientBonusPoolBalance` returns `ErrInsufficientBonusPool`.
7. Back in `slashRedelegationPosition`: the error is logged and swallowed. `setPositionWithState(ctx, pos, nil)` persists the position with the advanced `LastBonusAccrual` and `LastEventSeq`.
8. The user's accrued bonus is permanently destroyed. Future calls to `processEventsAndClaimBonus` for this position start from the advanced checkpoint and find no pending events. [8](#0-7) [9](#0-8) [6](#0-5)

func (k Keeper) processEventsAndClaimBonus(ctx context.Context, pos *types.PositionState) (sdk.Coins, error) {
	// Rewards should have been claimed before undelegation
	if !pos.IsDelegated() {
		return sdk.NewCoins(), nil
	}

	valAddr, err := sdk.ValAddressFromBech32(pos.Delegation.ValidatorAddress)
	if err != nil {
		return nil, err
	}

	events, err := k.getValidatorEventsSince(ctx, valAddr, pos.LastEventSeq)
	if err != nil {
		return nil, err
	}

	sdkCtx := sdk.UnwrapSDKContext(ctx)
	blockTime := sdkCtx.BlockTime()

	totalBonus := math.ZeroInt()
	// Use the persisted bonded state from the last replay, not a hardcoded default.
	// This prevents overpaying bonus for unbonded gaps between claims.
	bonded := pos.LastKnownBonded
	segmentStart := pos.LastBonusAccrual

	tier, err := k.getTier(ctx, pos.TierId)
	if err != nil {
		return nil, err
	}

	for _, entry := range events {
		evt := entry.Event

		if bonded {
			// Compute bonus for the bonded segment [segmentStart, eventTime]
			// using the snapshot rate at the event.
			bonus := k.computeSegmentBonus(*pos, tier, segmentStart, evt.Timestamp, evt.TokensPerShare)
			totalBonus = totalBonus.Add(bonus)
		}

		// Update bonded state based on event type.
		switch evt.EventType {
		case types.ValidatorEventType_VALIDATOR_EVENT_TYPE_UNBOND:
			bonded = false
		case types.ValidatorEventType_VALIDATOR_EVENT_TYPE_BOND:
			bonded = true
		case types.ValidatorEventType_VALIDATOR_EVENT_TYPE_SLASH:
			// Slash doesn't change bonded state.
		}

		segmentStart = evt.Timestamp
		pos.UpdateLastEventSeq(entry.Seq)

		// Decrement reference count.
		if err := k.decrementEventRefCount(ctx, valAddr, entry.Seq); err != nil {
			return nil, err
		}
	}

	val, err := k.stakingKeeper.GetValidator(ctx, valAddr)
	if err != nil {
		return nil, err
	}
	// Defensive: validator bond status check
	if bonded && val.IsBonded() {
		currentRate, err := k.getTokensPerShare(ctx, valAddr)
		if err != nil {
			return nil, err
		}
		bonus := k.computeSegmentBonus(*pos, tier, segmentStart, blockTime, currentRate)
		totalBonus = totalBonus.Add(bonus)
	}

	applyBonusAccrualCheckpoint(&pos.Position, blockTime)
	// Persist the bonded state so the next replay starts correctly.
	pos.UpdateLastKnownBonded(bonded)

	if totalBonus.IsZero() {
		return sdk.NewCoins(), nil
	}

	bondDenom, err := k.stakingKeeper.BondDenom(ctx)
	if err != nil {
		return nil, err
	}

	bonusCoins := sdk.NewCoins(sdk.NewCoin(bondDenom, totalBonus))

	if err := k.sufficientBonusPoolBalance(ctx, bonusCoins); err != nil {
		return nil, err
	}

func (k Keeper) slashRedelegationPosition(ctx context.Context, unbondingId uint64, sharesToUnbond math.LegacyDec) error {
	positionId, err := k.getRedelegationMapping(ctx, unbondingId)
	if errors.Is(err, collections.ErrNotFound) {
		return nil
	}
	if err != nil {
		return err
	}

	pos, err := k.getPositionState(ctx, positionId)
	if errors.Is(err, types.ErrPositionNotFound) {
		k.logger(ctx).Error("position not found during redelegation slash",
			"position_id", positionId,
			"unbonding_id", unbondingId,
			"error", err.Error(),
		)
		return nil
	}
	if err != nil {
		return err
	}

	if !pos.IsDelegated() {
		// Defensive
		k.logger(ctx).Error("delegation missing during BeforeRedelegationSlashed",
			"position_id", positionId,
			"unbonding_id", unbondingId,
			"shares_to_unbond", sharesToUnbond.String(),
		)
		return nil
	}

	dstValStr := pos.Delegation.ValidatorAddress

	// Settle bonus against PRE-slash shares.
	if _, err := k.processEventsAndClaimBonus(ctx, &pos); err != nil {
		// Deliberately forgo bonus rewards if pool is insufficient to prevent chain halt.
		if errors.Is(err, types.ErrInsufficientBonusPool) {
			k.logger(ctx).Error("insufficient bonus pool during redelegation slash",
				"position_id", pos.Id,
				"error", err.Error(),
			)
		} else {
			return err
		}
	}

	fullSlash := sharesToUnbond.GTE(pos.Delegation.Shares)

	if fullSlash {
		pos.Delegation = nil
		pos.ClearBonusCheckpoints()
		return k.setPositionWithState(ctx, pos, &ValidatorTransition{PreviousAddress: dstValStr})
	}
	// In-memory only: the persisted Position carries no share count, and the
	// live delegation will reflect the post-Unbond shares on the next read.
	// Update the local copy so any follow-up logic in this call sees consistent state.
	pos.Delegation.Shares = pos.Delegation.Shares.Sub(sharesToUnbond)
	return k.setPositionWithState(ctx, pos, nil)

func applyBonusAccrualCheckpoint(pos *types.Position, blockTime time.Time) {
	accrualEnd := blockTime
	if pos.CompletedExitLockDuration(blockTime) {
		accrualEnd = pos.ExitUnlockAt
	}
	pos.UpdateLastBonusAccrual(accrualEnd)
}
