### Title
Dogecoin Fork Difficulty Validation Uses Mainchain Ancestor Instead of Fork Ancestor, Enabling Incorrect PoW Acceptance — (`contract/src/dogecoin.rs`)

---

### Summary

The `get_next_work_required` function in `dogecoin.rs` uses `get_header_by_height` to retrieve the "first block" timestamp for Dogecoin's per-block difficulty adjustment. This function always returns the **mainchain's** block at the given height. For fork blocks that are 2+ blocks deep into a fork, the mainchain's block at `height_first` is a **different block** than the fork's actual ancestor at that height. This causes the difficulty calculation to use the wrong timestamp, producing an incorrect expected difficulty for Dogecoin fork blocks.
