Audit Report

## Title
`SwapAllowlistExtension` checks the router address instead of the end-user, allowing any caller to bypass the curated-pool swap allowlist via `MetricOmmSimpleRouter` — (File: metric-periphery/contracts/extensions/SwapAllowlistExtension.sol)

## Summary
`SwapAllowlistExtension.beforeSwap` gates on the `sender` argument, which is `msg.sender` of the pool's `swap` call. When a user routes through `MetricOmmSimpleRouter`, the router becomes `msg.sender` to the pool, so the allowlist checks whether the **router** is permitted rather than the actual end-user. If the router is allowlisted — the natural production configuration for any pool that wants its vetted users to use the standard periphery — every address on-chain can bypass the curated-pool restriction with a single router call.

## Finding Description
`MetricOmmPool.swap` passes `msg.sender` as the `sender` argument to `_beforeSwap`:

```solidity
// metric-core/contracts/MetricOmmPool.sol L230-240
_beforeSwap(
    msg.sender,   // whoever called pool.swap()
    recipient, ...
);
``` [1](#0-0) 

`ExtensionCalling._beforeSwap` forwards that value verbatim to every configured extension via `_callExtensionsInOrder`: [2](#0-1) 

`SwapAllowlistExtension.beforeSwap` then gates on that `sender`:

```solidity
// metric-periphery/contracts/extensions/SwapAllowlistExtension.sol L37
if (!allowAllSwappers[msg.sender] && !allowedSwapper[msg.sender][sender]) {
    revert IMetricOmmPoolActions.NotAllowedToSwap();
}
``` [3](#0-2) 

`MetricOmmSimpleRouter.exactInputSingle` calls `pool.swap(...)` directly, making the router `msg.sender` to the pool: [4](#0-3) 

The result: `allowedSwapper[pool][router]` is checked, not `allowedSwapper[pool][user]`. The extension has no mechanism to recover the original initiator — the router stores the real payer in transient storage (`_getPayer()`) but never encodes it into `extensionData` or any field visible to the extension.

## Impact Explanation
A pool admin deploys a curated pool with `SwapAllowlistExtension` to restrict trading to a known set of addresses. To allow those vetted addresses to use the standard periphery, the admin calls `setAllowedToSwap(pool, router, true)`. At that point, every address on-chain can bypass the allowlist by calling `exactInputSingle`, `exactInput`, `exactOutputSingle`, or `exactOutput` on the router. The guard is silently misapplied to the wrong actor and fails open for all router-mediated swaps. Unauthorized users trade against the pool's liquidity, directly harming LPs who deposited under the assumption that only vetted counterparties could trade — a direct loss of LP principal value and a broken core pool invariant enforced by the extension.

## Likelihood Explanation
`MetricOmmSimpleRouter` is the canonical public swap entry point for the protocol. Any production pool that wants its allowlisted users to be able to use the router must allowlist the router itself. The bypass requires no special privileges, no unusual token behavior, and no multi-transaction setup — a single `exactInputSingle` call from any EOA suffices. The condition (router allowlisted) is the expected and natural production configuration.

## Recommendation
The extension must gate on the economic actor (the end-user), not the intermediary. The minimal fix is to require the router to encode the original `msg.sender` into `extensionData` and have the extension decode and verify it. Alternatively, document that pools using `SwapAllowlistExtension` must not allowlist the router and that allowlisted users must call `pool.swap` directly — but this eliminates standard periphery usability for curated pools. The preferred fix is a protocol-level convention where the router populates `extensionData` with the original caller, and the extension decodes it when `sender` is a known router address.

## Proof of Concept
```
Setup:
  - Deploy pool with SwapAllowlistExtension
  - Pool admin calls setAllowedToSwap(pool, router, true)   // allowlist the router
  - Pool admin does NOT call setAllowedToSwap(pool, attacker, true)

Attack:
  - attacker calls MetricOmmSimpleRouter.exactInputSingle({pool, ...})
  - Router calls pool.swap(recipient, ...) — msg.sender to pool = router
  - Pool calls _beforeSwap(sender=router, ...)
  - SwapAllowlistExtension checks allowedSwapper[pool][router] → true
  - Swap executes for attacker despite attacker not being on the allowlist

Result:
  - attacker successfully trades on a curated pool
  - SwapAllowlistExtension guard is completely bypassed
```
