Looking at the code carefully, I need to trace the `TrackDelegation` call in `alignVestingDelegationTracking` for a `PeriodicVestingAccount` with a LockTier-origin position.

### Title
Incorrect DV/DF Split in `alignVestingDelegationTracking` Allows Vesting Account Owners to Spend Locked Tokens After v8 Migration - (`x/tieredrewards/keeper/force_exit.go`)

### Summary

`alignVestingDelegationTracking` calls `TrackDelegation(blockTime, coins, coins)` with `balance == amount == deficit`. The inline comment claims `balance` is only used for an invariant check (`amount <= balance`). This is factually wrong. For any time-based vesting account (`PeriodicVestingAccount`, `ContinuousVestingAccount`, `DelayedVestingAccount`), the `balance` parameter feeds directly into the DV/DF split via `x = min(vestingCoins(blockTime), balance)`. For a LockTier-origin position — where the delegation came from the owner's **free/spendable** coins, not from locked coins — the correct split is `DV += 0, DF += deficit`. Instead, the code sets `DV += min(vestingCoins(blockTime), deficit)`, reducing `lockedCoins = vestingCoins - DV` by that amount and making that many locked tokens appear spendable in the owner's bank.

### Finding Description

**Entrypoint**: The v8 upgrade handler calls `Migrate1to2` → `exitVestedAccountsPositions` → `ForceFullExitWithDelegation` for every position whose owner is a vesting account. This is a genesis/migration path, not a user-triggered transaction, but the state it writes is permanent and immediately exploitable by the owner post-upgrade.

**Root cause** in `alignVestingDelegationTracking`:

```go
// force_exit.go:163-171
deficit := actualDelegated.Sub(tracked)
coins := sdk.NewCoins(sdk.NewCoin(bondDenom, deficit))
// Pass balance == amount: TrackDelegation only uses balance for an
// invariant check (amount <= balance); the DV/DF split is computed from
// vestingCoins(blockTime) and DelegatedVesting. The owner's actual bank
// balance is irrelevant here because the delegation came from the position
// pool, not from the owner's balance.
sdkCtx := sdk.UnwrapSDKContext(ctx)
vacc.TrackDelegation(sdkCtx.BlockTime(), coins, coins)
```

The Cosmos SDK `PeriodicVestingAccount.TrackDelegation` delegates to `BaseVestingAccount.TrackDelegation(balance, vestingCoins(blockTime), amount)`, which computes:

```
x = min(vestingCoins(blockTime), balance)
y = min(x − DV_current, amount)
DV += y
DF += amount − y
```

With `balance == amount == deficit` and `DV_current == 0` (LockTier-origin, never touched DV/DF):

```
x = min(vestingCoins(blockTime), deficit)
y = min(vestingCoins(blockTime), deficit)   // since DV_current = 0
DV += min(vestingCoins(blockTime), deficit) // WRONG: should be 0
DF += deficit − min(vestingCoins(blockTime), deficit) // WRONG: should be deficit
```

The Cosmos SDK `LockedCoinsFromVesting` formula is `lockedCoins = vestingCoins − DV`. Inflating DV by `min(vestingCoins(blockTime), deficit)` reduces `lockedCoins` by the same amount, making that many locked tokens appear spendable in the owner's bank account.

**Preconditions** (all reachable before v8):
1. Owner holds a `PeriodicVestingAccount` (or `ContinuousVestingAccount`) with `vestingCoins(migrationBlockTime) > 0`.
2. Owner created a LockTier-origin position before v8 (no vesting filter existed pre-v8; `validateNonVestingAccount` was added in v8).
3. The LockTier used the owner's spendable (free) coins — so `DV = 0, DF = 0` pre-migration.

**Migration path** (`exitVestedAccountsPositions`) handles all `sdkvesting.VestingAccount` subtypes generically — it does not restrict to `PermanentLockedAccount`.

### Impact Explanation

After the v8 migration, the owner's `lockedCoins` is understated by `min(vestingCoins(blockTime), positionAmount)`. The owner can immediately send that many tokens from their bank balance that should remain locked. The tokens are not recovered until the owner undelegates (at which point `TrackUndelegation` restores the lock), but the window between migration and undelegation is sufficient to transfer or spend the tokens.

Concrete example:
- `PeriodicVestingAccount`: `OriginalVesting = 1000`, `vestingCoins(T_migration) = 600`
- LockTier used 200 spendable coins at T1 (when `vestingCoins(T1) = 800`, `spendable = 200`)
- Post-migration: `DV = 200` (should be 0), `lockedCoins = 400` (should be 600), `spendable = 400` (should be 200)
- Owner can spend 200 extra tokens that should be locked

### Likelihood Explanation

The tieredrewards module was introduced in v7. Between v7 and v8, any vesting account type could call `MsgLockTier` (no vesting filter existed). The migration in `exitVestedAccountsPositions` handles all `sdkvesting.VestingAccount` subtypes. If any `PeriodicVestingAccount` or `ContinuousVestingAccount` holder created a LockTier position between v7 and v8, they are affected. The exploit requires no special privileges — just sending a bank transfer after the upgrade.

### Recommendation

For LockTier-origin positions, the delegation came from free coins. The correct call is `TrackDelegation(blockTime, sdk.NewCoins(sdk.NewCoin(bondDenom, sdkmath.ZeroInt())), coins)` — passing `balance = 0` forces `x = min(vestingCoins, 0) = 0`, `y = 0`, so `DV += 0` and `DF += deficit`. Alternatively, track position origin (LockTier vs CommitDelegationToTier) and call `TrackDelegation` only for CommitDelegationToTier-origin positions (where the delegation genuinely came from the owner's vesting coins), and directly increment `DelegatedFree` for LockTier-origin positions.

func (s *KeeperSuite) TestAlignVestingTracking_PeriodicVesting_LockOrigin_SpendableInvariant() {
    s.setupTier(1)
    vals, bondDenom := s.getStakingData()
    val := vals[0]
    valAddr := sdk.MustValAddressFromBech32(val.GetOperator())
    s.setValidatorCommission(valAddr, sdkmath.LegacyZeroDec())

    // PeriodicVestingAccount: OriginalVesting=1000, two periods of 500 each
    // At migration blockTime, vestingCoins(blockTime) = 500 (one period elapsed)
    originalVesting := sdkmath.NewInt(1000)
    spendableAtCreation := sdkmath.NewInt(500) // first period vested
    lockAmount := sdkmath.NewInt(200)          // uses spendable coins

    owner := sdk.AccAddress(secp256k1.GenPrivKey().PubKey().Address())
    baseAcc, _ := s.app.AccountKeeper.NewAccountWithAddress(s.ctx, owner).(*authtypes.BaseAccount)
    now := s.ctx.BlockTime()
    periodicAcc := vestingtypes.NewPeriodicVestingAccount(baseAcc,
        sdk.NewCoins(sdk.NewCoin(bondDenom, originalVesting)),
        now.Unix(),
        vestingtypes.Periods{
            {Length: 1, Amount: sdk.NewCoins(sdk.NewCoin(bondDenom, sdkmath.NewInt(500)))},
            {Length: 86400 * 365, Amount: sdk.NewCoins(sdk.NewCoin(bondDenom, sdkmath.NewInt(500)))},
        },
    )
    s.app.AccountKeeper.SetAccount(s.ctx, periodicAcc)
    // Fund: originalVesting + lockAmount (lockAmount is spendable after first period)
    s.Require().NoError(banktestutil.FundAccount(s.ctx, s.app.BankKeeper,
        owner, sdk.NewCoins(sdk.NewCoin(bondDenom, originalVesting.Add(lockAmount)))))

    // Advance past first period so lockAmount is spendable
    s.ctx = s.ctx.WithBlockTime(now.Add(2 * time.Second))

    pos := s.createLockTierPositionV1(owner, valAddr, lockAmount)
    s.advanceForRewards(valAddr, bondDenom)

    // Migration blockTime: vestingCoins = 500 (second period not yet elapsed)
    s.Require().NoError(s.keeper.ForceFullExitWithDelegation(s.ctx, pos.Id))

    // Assert: spendable <= bank_balance - vestingCoins(blockTime)
    spendable := s.app.BankKeeper.SpendableCoins(s.ctx, owner)
    bankBal := s.app.BankKeeper.GetBalance(s.ctx, owner, bondDenom).Amount

    vestingNow := vacc.GetVestingCoins(s.ctx.BlockTime()).AmountOf(bondDenom)
    correctSpendable := bankBal.Sub(vestingNow)
    if correctSpendable.IsNegative() { correctSpendable = sdkmath.ZeroInt() }

    // BUG: spendable > correctSpendable because DV was incorrectly inflated
    s.Require().True(spendable.AmountOf(bondDenom).LTE(correctSpendable),
        "DelegatedFree <= spendable invariant violated: spendable=%s, correct=%s",
        spendable, correctSpendable)
}

```go
func exitVestedAccountsPositions(
	ctx context.Context,
	positions collections.Map[uint64, types.Position],
	ak AccountKeeper,
	pk PositionForceExiter,
) error {
	sdkCtx := sdk.UnwrapSDKContext(ctx)

	var toExit []uint64
	if err := positions.Walk(ctx, nil, func(posID uint64, pos types.Position) (bool, error) {
		ownerAddr, err := sdk.AccAddressFromBech32(pos.Owner)
		if err != nil {
			return false, fmt.Errorf("parse owner address for position %d: %w", posID, err)
		}
		acc := ak.GetAccount(ctx, ownerAddr)
		if acc == nil {
			return false, fmt.Errorf("owner account not found for position %d: %s", posID, ownerAddr.String())
		}
		if _, ok := acc.(sdkvesting.VestingAccount); ok {
			toExit = append(toExit, posID)
		}
		return false, nil
	}); err != nil {
		return fmt.Errorf("walk positions: %w", err)
	}

	for _, posID := range toExit {
		sdkCtx.Logger().Info("v8 migration: force-exit vesting-owned position", "position_id", posID)
		if err := pk.ForceFullExitWithDelegation(ctx, posID); err != nil {
			return fmt.Errorf("force-exit position %d: %w", posID, err)
		}
	}

	sdkCtx.Logger().Info("v8 migration: vesting-owned positions exited", "count", len(toExit))
	return nil
```
