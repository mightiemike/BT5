### Title
Missing Validation for Zero-Timeout IBC NFT Transfer Packets Allows Indefinite Packet Withholding - (File: x/nft-transfer/types/msgs.go)

---

### Summary

`MsgTransfer.ValidateBasic()` in the `x/nft-transfer` module does not enforce that at least one of `TimeoutHeight` or `TimeoutTimestamp` is non-zero. An NFT owner can submit a `MsgTransfer` with both timeouts set to zero, creating an IBC packet that never expires. The NFT is immediately escrowed or burned on the source chain, but the packet can be withheld and relayed by a malicious relayer at any arbitrarily future time — a direct Cosmos-SDK analog to the `block.timestamp`-as-deadline vulnerability.

---

### Finding Description

In `x/nft-transfer/types/msgs.go`, `ValidateBasic()` validates port, channel, classID, tokenIDs, sender, and receiver — but performs **no check** that at least one of `TimeoutHeight` or `TimeoutTimestamp` is non-zero: [1](#0-0) 

The CLI in `x/nft-transfer/client/cli/tx.go` enforces a non-zero timestamp only for the **relative** timeout path: [2](#0-1) 

This guard is bypassed in two ways:
1. Passing `--absolute-timeouts=true --packet-timeout-height=0-0 --packet-timeout-timestamp=0` via CLI — the absolute-timeout branch has no zero-check.
2. Submitting `MsgTransfer` directly via gRPC or REST, which invokes `ValidateBasic()` and then `SendTransfer()` without any CLI layer.

In `keeper/packet.go`, `createOutgoingPacket` immediately escrows (source chain) or burns (sink chain) the NFT tokens before the packet is sent: [3](#0-2) 

The resulting `channeltypes.Packet` is constructed with the caller-supplied zero timeouts and handed to `ics4Wrapper.SendPacket`: [4](#0-3) 

With both timeouts at zero, the IBC core layer treats the packet as having no expiry. The `OnTimeoutPacket` refund path — which would return the escrowed/re-minted NFT to the sender — can never be triggered: [5](#0-4) 

---

### Impact Explanation

- The NFT is **immediately and irrevocably** removed from the sender's control at the moment the `MsgTransfer` transaction is included in a block.
- A malicious or colluding relayer can **withhold the packet indefinitely** and relay it at any future time of their choosing — e.g., after the NFT's value has changed, after the receiver address has been compromised, or to front-run a secondary-market sale.
- Because the timeout never fires, the sender has **no on-chain recourse** to recover the NFT. The `OnTimeoutPacket` → `refundPacketToken` path is permanently unreachable for this packet.
- Corrupted value: the NFT owner record on the source chain (escrowed to the module account) and the eventual destination-chain mint/unescrow are both permanently deferred to relayer discretion.

---

### Likelihood Explanation

The attack is reachable by any unprivileged NFT owner submitting a standard `MsgTransfer` transaction. The zero-timeout path is reachable via:
- Direct gRPC/REST submission (no CLI involved).
- CLI with `--absolute-timeouts=true` and both timeout flags at their zero defaults.

No privileged role, leaked key, or social engineering is required. The only external actor needed is a relayer, which is a standard, expected participant in any IBC-enabled chain.

---

### Recommendation

Add a check in `MsgTransfer.ValidateBasic()` (or equivalently in `SendTransfer`) that rejects messages where both `TimeoutHeight` and `TimeoutTimestamp` are zero:

```go
// In x/nft-transfer/types/msgs.go, inside ValidateBasic():
if msg.TimeoutHeight.IsZero() && msg.TimeoutTimestamp == 0 {
    return newsdkerrors.Wrap(ErrInvalidTimeout,
        "both timeout height and timeout timestamp cannot be zero")
}
```

This mirrors the protection that the standard IBC fungible token transfer module (`x/transfer`) applies and closes the gap between CLI-enforced and protocol-enforced timeout validation. [1](#0-0) 

---

### Proof of Concept

1. Alice owns NFT `(classID="myClass", tokenID="token1")` on the source chain.
2. Alice (or an attacker who has observed Alice's intent) submits a `MsgTransfer` via gRPC with `TimeoutHeight = {RevisionNumber:0, RevisionHeight:0}` and `TimeoutTimestamp = 0`.
3. `ValidateBasic()` passes — no zero-timeout check exists.
4. `createOutgoingPacket` escrows `token1` to the IBC escrow address; Alice no longer owns it.
5. The IBC packet is committed on-chain with no expiry.
6. The malicious relayer withholds the packet. Alice cannot recover `token1` because `OnTimeoutPacket` requires the packet to have timed out, which it never will.
7. Weeks later, the relayer relays the packet to the destination chain, delivering the NFT to the receiver at a time of the relayer's choosing. [6](#0-5) [7](#0-6)

func (msg MsgTransfer) ValidateBasic() error {
	if err := host.PortIdentifierValidator(msg.SourcePort); err != nil {
		return newsdkerrors.Wrap(err, "invalid source port ID")
	}
	if msg.SourcePort != PortID {
		return newsdkerrors.Wrapf(ErrInvalidSourcePort, "source port must be %q", PortID)
	}
	if err := host.ChannelIdentifierValidator(msg.SourceChannel); err != nil {
		return newsdkerrors.Wrap(err, "invalid source channel ID")
	}

	if strings.TrimSpace(msg.ClassId) == "" {
		return newsdkerrors.Wrap(ErrInvalidClassID, "classId cannot be blank")
	}

	if len(msg.TokenIds) == 0 {
		return newsdkerrors.Wrap(ErrInvalidTokenID, "tokenId cannot be blank")
	}

	for _, tokenID := range msg.TokenIds {
		if strings.TrimSpace(tokenID) == "" {
			return newsdkerrors.Wrap(ErrInvalidTokenID, "tokenId cannot be blank")
		}
	}

	// NOTE: sender format must be validated as it is required by the GetSigners function.
	_, err := sdk.AccAddressFromBech32(msg.Sender)
	if err != nil {
		return newsdkerrors.Wrapf(sdkerrors.ErrInvalidAddress, "string could not be parsed as address: %v", err)
	}
	if strings.TrimSpace(msg.Receiver) == "" {
		return newsdkerrors.Wrap(sdkerrors.ErrInvalidAddress, "missing recipient address")
	}
	return nil
}

func (k Keeper) createOutgoingPacket(ctx sdk.Context,
	sourcePort,
	sourceChannel,
	destinationPort,
	destinationChannel,
	classID string,
	tokenIDs []string,
	sender sdk.AccAddress,
	receiver string,
	sequence uint64,
	timeoutHeight clienttypes.Height,
	timeoutTimestamp uint64,
) (channeltypes.Packet, error) {
	denom, err := k.nftKeeper.GetDenom(ctx, classID)
	if err != nil {
		return channeltypes.Packet{}, err
	}

	var (
		// NOTE: class and hex hash correctness checked during msg.ValidateBasic
		fullClassPath = classID
		tokenURIs     = []string{}
	)

	// deconstruct the token denomination into the denomination trace info
	// to determine if the sender is the source chain
	if strings.HasPrefix(classID, nfttypes.IBCPrefix) {
		fullClassPath, err = k.ClassPathFromHash(ctx, classID)
		if err != nil {
			return channeltypes.Packet{}, err
		}
	}

	isAwayFromOrigin := types.IsAwayFromOrigin(sourcePort,
		sourceChannel, fullClassPath)

	for _, tokenID := range tokenIDs {
		nft, err := k.nftKeeper.GetNFT(ctx, classID, tokenID)
		if err != nil {
			return channeltypes.Packet{}, err
		}
		tokenURIs = append(tokenURIs, nft.GetURI())

		owner := nft.GetOwner()
		if !sender.Equals(owner) {
			return channeltypes.Packet{}, newsdkerrors.Wrap(sdkerrors.ErrUnauthorized, "not token owner")
		}

		if isAwayFromOrigin {
			// create the escrow address for the tokens
			escrowAddress := types.GetEscrowAddress(sourcePort, sourceChannel)
			if err := k.nftKeeper.TransferOwner(ctx, classID, tokenID, sender, escrowAddress); err != nil {
				return channeltypes.Packet{}, err
			}
		} else {
			// we are sink chain, burn the voucher
			if err := k.nftKeeper.BurnNFTUnverified(ctx, classID, tokenID, sender); err != nil {
				return channeltypes.Packet{}, err
			}
		}
	}

	packetData := types.NewNonFungibleTokenPacketData(
		fullClassPath, denom.Uri, tokenIDs, tokenURIs, sender.String(), receiver,
	)

	return channeltypes.NewPacket(
		packetData.GetBytes(),
		sequence,
		sourcePort,
		sourceChannel,
		destinationPort,
		destinationChannel,
		timeoutHeight,
		timeoutTimestamp,
	), nil
}
