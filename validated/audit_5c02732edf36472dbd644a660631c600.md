### Title
Denom Creator Can Permanently Lock NFT by Minting to a Blocked Module Account — (`x/nft/keeper/keeper.go`)

### Summary

`MsgMintNFT` allows the denom creator to specify any valid bech32 address as `Recipient`, including blocked module account addresses (e.g., `distribution`, `nft-transfer` escrow). No guard exists anywhere in the call chain to reject such addresses. Once minted, the NFT is permanently irrecoverable because every subsequent mutation (`TransferNFT`, `BurnNFT`) requires the owner to sign, which a module account cannot do.

---

### Finding Description

**Call chain:**

```
MsgMintNFT.ValidateBasic()
  → sdk.AccAddressFromBech32(msg.Recipient)   // only bech32 validity
msgServer.MintNFT()
  → Keeper.MintNFT()
      → IsDenomCreator(ctx, denomID, sender)  // only checks sender
      → MintNFTUnverified()
          → setNFT(ctx, denomID, nft)         // no blocked-account check
          → setOwner(ctx, denomID, tokenID, owner)
```

`ValidateBasic` on `MsgMintNFT` only validates that `Recipient` is a parseable bech32 address: [1](#0-0) 

`msgServer.MintNFT` passes the decoded recipient directly to `Keeper.MintNFT` with no further validation: [2](#0-1) 

`Keeper.MintNFT` only checks that the sender is the denom creator, then delegates to `MintNFTUnverified`: [3](#0-2) 

`MintNFTUnverified` calls `setNFT` and `setOwner` with no blocked-account check: [4](#0-3) 

Critically, the `Keeper` struct holds only a `storeKey` and `cdc` — there is no `AccountKeeper` or `BankKeeper` reference, so the module is structurally incapable of performing a blocked-account check: [5](#0-4) 

**Lock is permanent.** `TransferOwner` requires `IsOwner(srcOwner == sender)`: [6](#0-5) 

`BurnNFT` requires the sender to be both the owner **and** the denom creator: [7](#0-6) 

A module account cannot sign either message, so there is no recovery path.

---

### Impact Explanation

An NFT minted to a blocked module account (e.g., `distribution`, `bonded_tokens_pool`, or the IBC NFT-transfer escrow account) is permanently locked. No on-chain transaction can transfer or burn it because every mutation path requires the owner to be the transaction signer. The NFT's supply counter is incremented but the token is unspendable forever.

---

### Likelihood Explanation

Module account addresses are deterministic and publicly documented. The denom creator is an unprivileged user who can issue a denom and immediately call `MsgMintNFT`. The precondition (knowing a module account address) is trivially satisfied. The action requires only a single signed transaction.

---

### Recommendation

1. Add an `AccountKeeper` (with a `BlockedAddr` map) to the NFT `Keeper`.
2. In `MintNFT` (and `MintNFTUnverified` for the IBC path), reject the call if `accountKeeper.BlockedAddr(owner)` returns `true`.
3. Add the same check in `TransferOwner` for the `dstOwner` to prevent transfers into module accounts as well.

---

### Proof of Concept

```go
// keeper integration test (unmodified Go/Cosmos test setup)
func TestMintToModuleAccountLocksNFT(t *testing.T) {
    ctx, keeper := setupKeeper(t)

    // Issue denom as creator
    creator := sdk.AccAddress([]byte("creator"))
    keeper.IssueDenom(ctx, "testdenom", "Test", "", "", creator)

    // Distribution module address — publicly known, no private key
    distAddr := authtypes.NewModuleAddress("distribution")

    // Mint NFT to the distribution module account — succeeds with no error
    err := keeper.MintNFT(ctx, "testdenom", "token1", "", "", "", creator, distAddr)
    require.NoError(t, err) // ← minting succeeds

    // Attempt transfer FROM distribution module — fails: no signer
    err = keeper.TransferOwner(ctx, "testdenom", "token1", distAddr, creator)
    require.Error(t, err) // ← ErrUnauthorized, but distAddr can never sign

    // Attempt burn FROM distribution module — fails: no signer
    err = keeper.BurnNFT(ctx, "testdenom", "token1", distAddr)
    require.Error(t, err) // ← permanently locked, no recovery path
}
```

// Needed during genesis initialization
func (k Keeper) MintNFTUnverified(ctx sdk.Context, denomID, tokenID, tokenNm, tokenURI, tokenData string, owner sdk.AccAddress) error {
	if !k.HasDenomID(ctx, denomID) {
		return sdkerrors.Wrapf(types.ErrInvalidDenom, "denom ID %s not exists", denomID)
	}

	if k.HasNFT(ctx, denomID, tokenID) {
		return sdkerrors.Wrapf(types.ErrNFTAlreadyExists, "NFT %s already exists in collection %s", tokenID, denomID)
	}

	k.setNFT(
		ctx, denomID,
		types.NewBaseNFT(
			tokenID,
			tokenNm,
			owner,
			tokenURI,
			tokenData,
		),
	)
	k.setOwner(ctx, denomID, tokenID, owner)
	k.increaseSupply(ctx, denomID)

	return nil

func (k Keeper) TransferOwner(
	ctx sdk.Context, denomID, tokenID string, srcOwner, dstOwner sdk.AccAddress,
) error {
	if !k.HasDenomID(ctx, denomID) {
		return sdkerrors.Wrapf(types.ErrInvalidDenom, "denom ID %s not exists", denomID)
	}

	nft, err := k.IsOwner(ctx, denomID, tokenID, srcOwner)
	if err != nil {
		return err
	}

	nft.Owner = dstOwner.String()

	k.setNFT(ctx, denomID, nft)
	k.swapOwner(ctx, denomID, tokenID, srcOwner, dstOwner)
	return nil
}

func (k Keeper) BurnNFT(ctx sdk.Context, denomID, tokenID string, owner sdk.AccAddress) error {
	if !k.HasDenomID(ctx, denomID) {
		return sdkerrors.Wrapf(types.ErrInvalidDenom, "denom ID %s not exists", denomID)
	}

	nft, err := k.IsOwner(ctx, denomID, tokenID, owner)
	if err != nil {
		return err
	}

	_, err = k.IsDenomCreator(ctx, denomID, owner)
	if err != nil {
		return err
	}

	k.deleteNFT(ctx, denomID, nft)
	k.deleteOwner(ctx, denomID, tokenID, owner)
	k.decreaseSupply(ctx, denomID)

	return nil
}
