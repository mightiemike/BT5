### Title
`ValidResourceBounds` Protobuf Deserialization Silently Downgrades `AllResources` to `L1Gas`, Producing a Divergent Transaction Hash — (`File: crates/apollo_protobuf/src/converters/transaction.rs`)

---

### Summary

The protobuf deserializer for `ValidResourceBounds` silently converts an `AllResources` variant (with zero `l2_gas` and `l1_data_gas`) into the `L1Gas` variant. Because `get_tip_resource_bounds_hash` includes the `L1_DATA_GAS` element in the Poseidon hash chain only for `AllResources`, the same on-chain transaction produces two structurally different hash preimages depending on which code path processes it. A transaction submitted via the gateway is hashed as `AllResources` (3-element resource chain), but after a protobuf round-trip through state sync it is hashed as `L1Gas` (2-element resource chain), yielding a different `TransactionHash`.

---

### Finding Description

**Step 1 — Hash computation branches on the `ValidResourceBounds` variant.**

`get_tip_resource_bounds_hash` in `crates/starknet_api/src/transaction_hash.rs` builds the Poseidon hash chain differently for the two variants:

```rust
// lines 197-208
let mut resource_felts = vec![
    get_concat_resource(&l1_resource_bounds, L1_GAS)?,
    get_concat_resource(&l2_resource_bounds, L2_GAS)?,
];
resource_felts.extend(match resource_bounds {
    ValidResourceBounds::L1Gas(_) => vec![],                          // 2 elements
    ValidResourceBounds::AllResources(all_resources) => {
        vec![get_concat_resource(&all_resources.l1_data_gas, L1_DATA_GAS)?]  // 3 elements
    }
});
```

`L1Gas` → `hash(tip, L1_GAS_packed, L2_GAS_packed)`
`AllResources` → `hash(tip, L1_GAS_packed, L2_GAS_packed, L1_DATA_GAS_packed)`

These are distinct Poseidon outputs even when `l2_gas = 0` and `l1_data_gas = 0`, because the hash chain length differs.

**Step 2 — The protobuf deserializer silently changes the variant.**

`TryFrom<protobuf::ResourceBounds> for ValidResourceBounds` in `crates/apollo_protobuf/src/converters/transaction.rs`:

```rust
// lines 431-435
Ok(if l1_data_gas.is_zero() && l2_gas.is_zero() {
    ValidResourceBounds::L1Gas(l1_gas)
} else {
    ValidResourceBounds::AllResources(AllResourceBounds { l1_gas, l2_gas, l1_data_gas })
})
```

Any `AllResources` transaction whose `l2_gas` and `l1_data_gas` are both zero is silently re-classified as `L1Gas` on deserialization. The comment on line 426 even acknowledges this is a temporary compatibility measure: `// TODO(Shahak): Assert data gas is not none once we remove support for 0.13.2.`

**Step 3 — The gateway path always uses `AllResources`.**

`InternalRpcInvokeTransactionV3` stores `resource_bounds: AllResourceBounds` and its `InvokeTransactionV3Trait` impl always returns `ValidResourceBounds::AllResources(self.resource_bounds)`. The hash computed at gateway admission time therefore always uses the 3-element chain.

**Step 4 — The state-sync path uses `ValidResourceBounds` and triggers the downgrade.**

`InvokeTransactionV3` (the full blockifier type used during state sync) stores `resource_bounds: ValidResourceBounds`. When a syncing node deserializes a block over P2P, it calls `TryFrom<protobuf::ResourceBounds> for ValidResourceBounds`, which produces `L1Gas` for the zero-bounds case. The syncing node then recomputes the hash using the 2-element chain, obtaining `H_l1 ≠ H_all`.

**Concrete divergence:**

| Path | Variant | Hash chain elements | Hash |
|---|---|---|---|
| Gateway / proposer | `AllResources` | tip, L1_GAS, L2_GAS(0), L1_DATA_GAS(0) | H_all |
| State-sync deserializer | `L1Gas` | tip, L1_GAS, L2_GAS(0) | H_l1 |

`H_all ≠ H_l1` because Poseidon over 4 elements ≠ Poseidon over 3 elements.

---

### Impact Explanation

When `validate_transaction_hash` is called on the syncing node with `expected_hash = H_all` (from the block header), it recomputes `H_l1` from the deserialized `Transaction` object and returns `false`. The syncing node either:

1. **Rejects the block** — a valid, sequenced block is treated as invalid, breaking liveness for any node that syncs rather than participates in consensus.
2. **Stores the transaction with the wrong hash** — if the node trusts the block-header hash but stores the deserialized `Transaction` with `L1Gas` variant, subsequent RPC calls (`starknet_getTransactionByHash`, fee estimation, tracing) operate on a transaction object whose resource-bounds variant does not match the signed hash, returning authoritative-looking wrong values.

This matches the allowed impact: **High — Transaction conversion or signature/hash logic binds the wrong hash or executable payload** and **High — RPC execution, fee estimation, tracing, simulation, or pending view returns an authoritative-looking wrong value**.

---

### Likelihood Explanation

The trigger condition — an `AllResources` transaction with `l2_gas = 0` and `l1_data_gas = 0` — is trivially constructable. The gateway's `StatelessTransactionValidator` accepts it as long as `l1_gas` is non-zero (satisfying the non-zero fee check). No special privilege is required. Any user can submit such a transaction, and it will be included in a block by the proposer with hash `H_all`. Every syncing node will then compute `H_l1` and diverge.

---

### Recommendation

Remove the variant-switching heuristic from the protobuf deserializer. The `ValidResourceBounds` variant must be preserved across serialization boundaries, not inferred from zero-value fields. Two options:

1. **Encode the variant explicitly in the protobuf message** (add a `bounds_type` enum field) so the deserializer can reconstruct the exact variant without inspecting field values.
2. **Always deserialize as `AllResources`** in the `TryFrom<protobuf::ResourceBounds> for ValidResourceBounds` impl, removing the `L1Gas` branch entirely for new transactions. The `L1Gas` variant should only be produced when reading pre-0.13.3 historical data through a separate, version-gated path.

The TODO comment at line 426 (`Assert data gas is not none once we remove support for 0.13.2`) confirms the intent to eventually remove this compatibility path; the fix should be applied now to close the hash-domain split.

---

### Proof of Concept

1. Construct an `InvokeTransactionV3` with `resource_bounds = AllResources { l1_gas: X, l2_gas: 0, l1_data_gas: 0 }` where `X > 0`.

2. Compute `H_all = get_invoke_transaction_v3_hash(tx, chain_id, version)`. The `get_tip_resource_bounds_hash` call hashes `[tip, L1_GAS_packed(X), L2_GAS_packed(0), L1_DATA_GAS_packed(0)]` — 4 Poseidon inputs.

3. Serialize the transaction to protobuf via `From<ValidResourceBounds> for protobuf::ResourceBounds` (line 471–489 of `transaction.rs`): `l1_data_gas` is set to `ResourceBounds::default()` (zero).

4. Deserialize via `TryFrom<protobuf::ResourceBounds> for ValidResourceBounds` (line 431): `l1_data_gas.is_zero() && l2_gas.is_zero()` is `true` → returns `ValidResourceBounds::L1Gas(l1_gas)`.

5. Compute `H_l1 = get_invoke_transaction_v3_hash(deserialized_tx, chain_id, version)`. The `get_tip_resource_bounds_hash` call hashes `[tip, L1_GAS_packed(X), L2_GAS_packed(0)]` — 3 Poseidon inputs.

6. Assert `H_all ≠ H_l1`. The assertion holds because `poseidon([tip, A, B, C]) ≠ poseidon([tip, A, B])` for any values.

7. Call `validate_transaction_hash(deserialized_tx, block_number, chain_id, H_all, options)` → returns `false`, confirming the syncing node rejects the block. [1](#0-0) [2](#0-1) [3](#0-2) [4](#0-3) [5](#0-4)

// An implementation of the SNIP: https://github.com/EvyatarO/SNIPs/blob/snip-8/SNIPS/snip-8.md
pub fn get_tip_resource_bounds_hash(
    resource_bounds: &ValidResourceBounds,
    tip: &Tip,
) -> Result<Felt, StarknetApiError> {
    let l1_resource_bounds = resource_bounds.get_l1_bounds();
    let l2_resource_bounds = resource_bounds.get_l2_bounds();

    // L1 and L2 gas bounds always exist.
    // Old V3 txs always have L2 gas bounds of zero, but they exist.
    let mut resource_felts = vec![
        get_concat_resource(&l1_resource_bounds, L1_GAS)?,
        get_concat_resource(&l2_resource_bounds, L2_GAS)?,
    ];

    // For new V3 txs, need to also hash the data gas bounds.
    resource_felts.extend(match resource_bounds {
        ValidResourceBounds::L1Gas(_) => vec![],
        ValidResourceBounds::AllResources(all_resources) => {
            vec![get_concat_resource(&all_resources.l1_data_gas, L1_DATA_GAS)?]
        }
    });

    Ok(HashChain::new().chain(&tip.0.into()).chain_iter(resource_felts.iter()).get_poseidon_hash())
}

pub(crate) fn get_invoke_transaction_v3_hash<T: InvokeTransactionV3Trait>(
    transaction: &T,
    chain_id: &ChainId,
    transaction_version: &TransactionVersion,
) -> Result<TransactionHash, StarknetApiError> {
    let tip_resource_bounds_hash =
        get_tip_resource_bounds_hash(&transaction.resource_bounds(), transaction.tip())?;
    let paymaster_data_hash =
        HashChain::new().chain_iter(transaction.paymaster_data().0.iter()).get_poseidon_hash();
    let data_availability_mode = concat_data_availability_mode(
        transaction.nonce_data_availability_mode(),
        transaction.fee_data_availability_mode(),
    );
    let account_deployment_data_hash = HashChain::new()
        .chain_iter(transaction.account_deployment_data().0.iter())
        .get_poseidon_hash();
    let calldata_hash =
        HashChain::new().chain_iter(transaction.calldata().0.iter()).get_poseidon_hash();
    let mut hash_chain = HashChain::new()
        .chain(&INVOKE)
        .chain(&transaction_version.0)
        .chain(transaction.sender_address().0.key())
        .chain(&tip_resource_bounds_hash)
        .chain(&paymaster_data_hash)
        .chain(&Felt::try_from(chain_id)?)
        .chain(&transaction.nonce().0)
        .chain(&data_availability_mode)
        .chain(&account_deployment_data_hash)
        .chain(&calldata_hash);
    if !transaction.proof_facts().0.is_empty() {
        let proof_facts_hash =
            HashChain::new().chain_iter(transaction.proof_facts().0.iter()).get_poseidon_hash();
        hash_chain = hash_chain.chain(&proof_facts_hash);
    }
    Ok(TransactionHash(hash_chain.get_poseidon_hash()))
