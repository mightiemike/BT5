Audit Report

## Title
`SwapAllowlistExtension` allowlist bypassed via router — `sender` is router address, not original user - (File: metric-periphery/contracts/extensions/SwapAllowlistExtension.sol)

## Summary
`MetricOmmPool.swap()` passes `msg.sender` as the `sender` argument to `_beforeSwap`. When a user swaps through `MetricOmmSimpleRouter`, `msg.sender` inside the pool is the router contract, not the original user. `SwapAllowlistExtension.beforeSwap` checks `allowedSwapper[pool][sender]`, which resolves to `allowedSwapper[pool][router]`. Any pool admin who allowlists the router to enable router-mediated swaps inadvertently opens the allowlist to every unprivileged user.

## Finding Description
**Call path:**
1. Unprivileged user calls `MetricOmmSimpleRouter.exactInputSingle` / `exactInput` / `exactOutputSingle` / `exactOutput`.
2. Router calls `IMetricOmmPoolActions(pool).swap(recipient, ...)` — so inside `MetricOmmPool.swap`, `msg.sender` = router address.
3. `MetricOmmPool.swap` calls `_beforeSwap(msg.sender, ...)` (line 231), forwarding the router address as `sender`.
4. `ExtensionCalling._beforeSwap` encodes `sender` (= router) and dispatches to `SwapAllowlistExtension.beforeSwap`.
5. `SwapAllowlistExtension.beforeSwap` evaluates `allowedSwapper[msg.sender][sender]` = `allowedSwapper[pool][router]`.

If the pool admin has allowlisted the router (a natural step to let any user reach the pool via the router), the check passes for every caller regardless of their individual allowlist status. The original user's address is never inspected.

**Existing guard is insufficient:** `SwapAllowlistExtension` has no mechanism to recover the original `msg.sender` of the router call; it only sees the `sender` argument supplied by the pool, which is always the pool's own `msg.sender`.

**Exact wrong value:** `allowedSwapper[pool][sender]` is evaluated against the router address instead of the end-user address.

## Impact Explanation
The swap allowlist — the sole on-chain gate restricting which addresses may trade against a pool — is rendered ineffective for all router-mediated swaps whenever the router is allowlisted. Any unprivileged user can trade on a pool that was intended to be restricted, leading to unauthorized access to pool liquidity and potential loss of LP assets or protocol fees if the restriction was economically motivated (e.g., private/institutional pools, KYC-gated pools).

## Likelihood Explanation
The precondition is that the pool admin allowlists the router. This is a routine and expected administrative action: without it, no allowlisted user can use the router at all, making the router unusable for any allowlisted pool. The admin faces a forced choice — either block the router entirely or open the allowlist to everyone. The bypass is therefore reachable in any realistic deployment that combines `SwapAllowlistExtension` with `MetricOmmSimpleRouter`.

## Recommendation
Pass the original end-user identity through the call chain. One approach: add a `payer` / `originator` field to the swap callback context in the router and forward it as `extensionData`; the extension can then decode and check it. Alternatively, redesign `SwapAllowlistExtension` to check `msg.sender` of the extension call (the pool) against a router registry and, when the caller is a known router, require the extension data to carry a signed or verified user address. The simplest correct fix is to have the pool accept an explicit `sender` parameter from the caller and validate it (e.g., require `msg.sender == sender || isApprovedRouter(msg.sender)`), then pass that validated address to hooks.

## Proof of Concept
```solidity
// Setup: pool with SwapAllowlistExtension; router is allowlisted, attacker is not.
swapAllowlist.setAllowedToSwap(pool, address(router), true);
// attacker is NOT in allowedSwapper

// Attacker bypasses allowlist via router:
vm.prank(attacker);
router.exactInputSingle(IMetricOmmSimpleRouter.ExactInputSingleParams({
    pool: pool,
    tokenIn: token0,
    tokenOut: token1,
    zeroForOne: true,
    amountIn: 1_000,
    amountOutMinimum: 0,
    recipient: attacker,
    deadline: block.timestamp + 1,
    priceLimitX64: 0,
    extensionData: ""
}));
// Swap succeeds — allowlist check passed because sender == router (allowlisted),
// not attacker (not allowlisted). Assert no revert.
```
