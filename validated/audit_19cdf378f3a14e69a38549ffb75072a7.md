### Title
NLP Lock Period Bypass via `withdrawCollateral` — (`File: core/contracts/Clearinghouse.sol`)

---

### Summary

The Nado protocol enforces an `NLP_LOCK_PERIOD` on newly minted NLP tokens, requiring users to wait before burning them. The `burnNlp` path correctly checks `getNlpUnlockedBalance` before reducing a user's NLP balance. However, `withdrawCollateral` in `Clearinghouse.sol` is a second, unguarded path that also calls `spotEngine.updateBalance(NLP_PRODUCT_ID, sender, -amount)` — without any lock-period check — allowing a user to exit a locked NLP position immediately.

---

### Finding Description

When NLP is minted, `handleNlpLockedBalance` in `SpotEngine.sol` enqueues the minted amount with `unlockedAt = getOracleTime() + NLP_LOCK_PERIOD`. [1](#0-0) 

The intended exit path, `burnNlp`, enforces the lock by calling `getNlpUnlockedBalance` before proceeding:

```solidity
require(
    spotEngine.getNlpUnlockedBalance(txn.sender).amount >= nlpAmount,
    ERR_UNLOCKED_NLP_INSUFFICIENT
);
``` [2](#0-1) 

However, `withdrawCollateral` is a separate Endpoint-dispatched function that reduces a spot-product balance by calling `spotEngine.updateBalance(productId, sender, amountRealized)` with no NLP-specific lock check:

```solidity
spotEngine.updateBalance(productId, sender, amountRealized);
spotEngine.assertUtilization(productId);
``` [3](#0-2) 

Because NLP is registered as a product in `SpotEngine` with a token address, `withdrawCollateral` can be invoked with `NLP_PRODUCT_ID`. This routes into `handleNlpLockedBalance` with a negative delta, which unconditionally subtracts from `unlockedBalanceSum` — even when the balance is entirely locked:

```solidity
} else if (amountDelta < 0) {
    Balance memory balanceSum = nlpLockedBalanceQueues[subaccount]
        .unlockedBalanceSum;
    balanceSum.amount += amountDelta;
    nlpLockedBalanceQueues[subaccount].unlockedBalanceSum = balanceSum;
}
``` [4](#0-3) 

The result is that `unlockedBalanceSum` goes negative, the actual NLP tokens are transferred out to the user, and the lock-period invariant is silently broken. [5](#0-4) 

---

### Impact Explanation

A user who has just minted NLP (all balance locked) can immediately call `withdrawCollateral` with `NLP_PRODUCT_ID` to receive their NLP tokens back without waiting for `NLP_LOCK_PERIOD` to expire. This:

- Breaks the liquidity-stability guarantee of the NLP lock period.
- Allows users to mint NLP, collect any in-period yield or profit-share, then immediately exit — undermining the economic design of the NLP pool.
- Corrupts `unlockedBalanceSum` (drives it negative), which can cause subsequent legitimate `burnNlp` calls to fail or produce incorrect accounting for the affected subaccount. [6](#0-5) 

---

### Likelihood Explanation

The entry path is fully unprivileged: any user who holds NLP balance can submit a signed `WithdrawCollateral` transaction through the Endpoint targeting `NLP_PRODUCT_ID`. No special role, governance action, or leaked key is required. The sequencer dispatches it to `Clearinghouse.withdrawCollateral` in the normal transaction flow. [7](#0-6) 

---

### Recommendation

Disable `withdrawCollateral` for `NLP_PRODUCT_ID` by adding an explicit guard at the top of the function, mirroring how `burnNlp` is the sole sanctioned exit path:

```solidity
require(productId != NLP_PRODUCT_ID, "use burnNlp to exit NLP position");
```

Alternatively, add the same `getNlpUnlockedBalance` check inside `withdrawCollateral` when `productId == NLP_PRODUCT_ID`, consistent with the fix applied to the stFLUO `withdraw` function. [8](#0-7) 

---

### Proof of Concept

1. User calls `mintNlp` — NLP balance is fully locked in `NlpLockedBalanceQueue` with `unlockedAt = now + NLP_LOCK_PERIOD`. `unlockedBalanceSum = 0`. [9](#0-8) 
2. Immediately after, user signs and submits a `WithdrawCollateral` transaction with `productId = NLP_PRODUCT_ID` and `amount = nlpAmount`.
3. Sequencer calls `Clearinghouse.withdrawCollateral(sender, NLP_PRODUCT_ID, amount, sendTo, idx)`. No lock check is performed.
4. `spotEngine.updateBalance(NLP_PRODUCT_ID, sender, -amount)` is called → `handleNlpLockedBalance(sender, -amount)` → `unlockedBalanceSum.amount -= amount` (now negative). [10](#0-9) 
5. NLP tokens are transferred to `sendTo` via `handleWithdrawTransfer`. Lock period is fully bypassed. [11](#0-10)

    function withdrawCollateral(
        bytes32 sender,
        uint32 productId,
        uint128 amount,
        address sendTo,
        uint64 idx
    ) public virtual onlyEndpoint {
        require(!RiskHelper.isIsolatedSubaccount(sender), ERR_UNAUTHORIZED);
        require(amount <= INT128_MAX, ERR_CONVERSION_OVERFLOW);
        ISpotEngine spotEngine = _spotEngine();
        IERC20Base token = IERC20Base(spotEngine.getConfig(productId).token);
        require(address(token) != address(0));

        if (sendTo == address(0)) {
            sendTo = address(uint160(bytes20(sender)));
        }

        handleWithdrawTransfer(token, sendTo, amount, idx);

        int256 multiplier = int256(10**(MAX_DECIMALS - _decimals(productId)));
        int128 amountRealized = -int128(amount) * int128(multiplier);
        spotEngine.updateBalance(productId, sender, amountRealized);
        spotEngine.assertUtilization(productId);

        IProductEngine.HealthType healthType = sender == X_ACCOUNT
            ? IProductEngine.HealthType.PNL
            : IProductEngine.HealthType.INITIAL;

        require(getHealth(sender, healthType) >= 0, ERR_SUBACCT_HEALTH);
        emit ModifyCollateral(amountRealized, sender, productId);
    }

    function tryUnlockNlpBalance(bytes32 subaccount)
        public
        returns (Balance memory)
    {
        NlpLockedBalanceQueue storage queue = nlpLockedBalanceQueues[
            subaccount
        ];
        while (
            queue.unlockedUpTo < queue.balanceCount &&
            queue.balances[queue.unlockedUpTo].unlockedAt <= getOracleTime()
        ) {
            // we can unlock this balance
            queue.unlockedBalanceSum.amount += queue
                .balances[queue.unlockedUpTo]
                .balance
                .amount;
            delete queue.balances[queue.unlockedUpTo];
            queue.unlockedUpTo++;
        }

        return queue.unlockedBalanceSum;
    }
