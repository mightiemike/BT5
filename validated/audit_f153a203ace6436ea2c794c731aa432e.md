### Title
Wrong Escrow Address in `refundPacketToken()` Causes Permanent NFT Loss on Sink-Chain Refund — (File: `x/nft-transfer/keeper/packet.go`)

### Summary

`refundPacketToken()` uses the **source-port/channel escrow address** as the `sender` argument to `MintNFT()` when refunding a burned NFT on a sink chain. Because `MintNFT()` enforces a denom-creator check (`IsDenomCreator`), and the denom was created by a **different** escrow address (the destination-port/channel escrow of the original inbound packet), the mint fails. The NFT was already burned in the prior send transaction and cannot be recovered, producing a permanent loss of the user's asset with a corrupted NFT supply.

---

### Finding Description

When an IBC NFT packet is sent from a **sink chain** (Chain B, which holds a voucher NFT originally received from Chain A), the NFT is burned in `createOutgoingPacket()`: [1](#0-0) 

On a failed packet (error acknowledgement or timeout), `refundPacketToken()` is called to restore the NFT. For the sink-chain case (`isAwayFromOrigin == false`), it attempts to mint the NFT back: [2](#0-1) 

The `escrowAddress` used here is: [3](#0-2) 

This is the **B→C channel escrow** (source port/channel of the failing packet). However, the voucher denom on Chain B was originally created in `processReceivedPacket()` using the **A→B channel escrow** (destination port/channel of the original inbound packet): [4](#0-3) 

`MintNFT()` enforces a denom-creator check before minting: [5](#0-4) 

Because the B→C escrow address is **not** the denom creator (the A→B escrow is), `IsDenomCreator` returns `ErrUnauthorized`, `MintNFT` fails, and the refund transaction reverts. The burn from the original send is already committed in a prior block and is permanent.

---

### Impact Explanation

The NFT is burned on Chain B when the outbound packet is sent. When the packet fails (error ack or timeout), the refund path is broken: `MintNFT` always rejects the B→C escrow address as the denom creator. The NFT supply is permanently decremented without the token being returned to the sender. The user loses their NFT with no recovery path.

Corrupted invariant: `NFT.supply(denomID)` is decremented by the burn but never restored; the NFT record is deleted and cannot be re-created by the user.

---

### Likelihood Explanation

Any user performing a multi-hop IBC NFT transfer (Chain A → Chain B → Chain C) where the B→C packet fails (network timeout, relayer delay, or destination chain rejection) triggers this path. Multi-hop IBC NFT transfers are a supported and documented use case of ICS-721. The failure condition (timeout or error ack) is a normal operational event, not an adversarial one. An unprivileged user can trigger this by simply sending an IBC NFT transfer that times out.

---

### Recommendation

Replace `MintNFT` with `MintNFTUnverified` in the `else` branch of `refundPacketToken()`. The refund is a trusted IBC-module-initiated operation and does not require a denom-creator check. `MintNFTUnverified` is already used in `processReceivedPacket()` for the analogous forward-path mint and bypasses the creator check correctly: [6](#0-5) 

Change:
```go
k.nftKeeper.MintNFT(ctx, voucherClassID, tokenID, "", data.TokenUris[i], "", escrowAddress, sender)
```
To:
```go
k.nftKeeper.MintNFTUnverified(ctx, voucherClassID, tokenID, "", data.TokenUris[i], "", sender)
```

---

### Proof of Concept

1. Chain A holds `nftClass/token1` (origin chain).
2. Chain A sends `token1` to Chain B via IBC (A→B). Chain B receives it, creates denom `ibc/HASH_AB` with **A→B dest escrow** as creator, mints `token1` to the user.
3. User on Chain B sends `token1` to Chain C via IBC (B→C). `createOutgoingPacket` burns `token1` on Chain B (`BurnNFTUnverified`).
4. The B→C packet times out (or Chain C sends an error ack).
5. `OnTimeoutPacket` → `refundPacketToken` is called on Chain B.
6. `escrowAddress = GetEscrowAddress(B_port, B_C_channel)` ≠ A→B dest escrow (the denom creator).
7. `MintNFT(ctx, "ibc/HASH_AB", "token1", ..., B_C_escrow, user)` → `IsDenomCreator` fails → `ErrUnauthorized`.
8. Transaction reverts. Packet remains unacknowledged. Relayer retries indefinitely — always fails.
9. `token1` is permanently lost. `GetTotalSupply(ctx, "ibc/HASH_AB")` is 0 but no owner holds the token. [7](#0-6) [8](#0-7)

```go
// refundPacketToken will unescrow and send back the tokens back to sender
// if the sending chain was the source chain. Otherwise, the sent tokens
// were burnt in the original send so new tokens are minted and sent to
// the sending address.
func (k Keeper) refundPacketToken(ctx sdk.Context, packet channeltypes.Packet, data types.NonFungibleTokenPacketData) error {
	sender, err := sdk.AccAddressFromBech32(data.Sender)
	if err != nil {
		return err
	}

	classTrace := types.ParseClassTrace(data.ClassId)
	voucherClassID := classTrace.IBCClassID()

	isAwayFromOrigin := types.IsAwayFromOrigin(packet.GetSourcePort(),
		packet.GetSourceChannel(), data.ClassId)

	escrowAddress := types.GetEscrowAddress(packet.GetSourcePort(), packet.GetSourceChannel())

	if isAwayFromOrigin {
		// unescrow tokens back to the sender
		for _, tokenID := range data.TokenIds {
			if err := k.nftKeeper.TransferOwner(ctx, voucherClassID, tokenID, escrowAddress, sender); err != nil {
				return err
			}
		}
	} else {
		// we are sink chain, mint voucher back to sender
		for i, tokenID := range data.TokenIds {
			if err := k.nftKeeper.MintNFT(ctx, voucherClassID, tokenID, "", data.TokenUris[i], "", escrowAddress, sender); err != nil {
				return err
			}
		}
	}

	return nil
}
```

```go
// MintNFT mints an NFT and manages the NFT's existence within Collections and Owners
func (k Keeper) MintNFT(
	ctx sdk.Context, denomID, tokenID, tokenNm,
	tokenURI, tokenData string, sender, owner sdk.AccAddress,
) error {
	_, err := k.IsDenomCreator(ctx, denomID, sender)
	if err != nil {
		return err
	}

	return k.MintNFTUnverified(ctx, denomID, tokenID, tokenNm, tokenURI, tokenData, owner)
}
```

```go
// BurnNFTUnverified deletes a specified NFT without verifying if the owner is the creator of denom
// Needed for IBC transfer of NFT
func (k Keeper) BurnNFTUnverified(ctx sdk.Context, denomID, tokenID string, owner sdk.AccAddress) error {
	if !k.HasDenomID(ctx, denomID) {
		return sdkerrors.Wrapf(types.ErrInvalidDenom, "denom ID %s not exists", denomID)
	}

	nft, err := k.IsOwner(ctx, denomID, tokenID, owner)
	if err != nil {
		return err
	}

	k.deleteNFT(ctx, denomID, nft)
	k.deleteOwner(ctx, denomID, tokenID, owner)
	k.decreaseSupply(ctx, denomID)

	return nil
```
