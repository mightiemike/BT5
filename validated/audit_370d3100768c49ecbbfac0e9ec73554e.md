### Title
Multi-Hop IBC NFT Refund Permanently Destroys NFTs Due to Wrong Escrow Address in `refundPacketToken` - (File: x/nft-transfer/keeper/packet.go)

### Summary
When an IBC NFT transfer fails on the destination chain and the source chain is a **sink chain** (not the NFT's origin), `refundPacketToken` calls `MintNFT` using the **outgoing channel's escrow address** as the `sender`. However, the IBC voucher denom was originally created with the **incoming channel's escrow address** as its creator. The `IsDenomCreator` check inside `MintNFT` rejects the call, the refund transaction fails, and the NFT is permanently destroyed.

### Finding Description

When Chain B first receives an NFT from Chain A via channel `(p1,c1)→(p2,c2)`, `processReceivedPacket` creates the IBC voucher denom `ibc/{hash(p2/c2/nftClass)}` with `GetEscrowAddress(p2, c2)` as its creator: [1](#0-0) 

When Chain B later forwards this voucher to Chain C via channel `(p3,c3)→(p4,c4)`, `createOutgoingPacket` determines `isAwayFromOrigin = false` (Chain B is a sink chain) and **burns** the NFT via `BurnNFTUnverified`: [2](#0-1) 

If the packet to Chain C fails (error ack or timeout), `OnAcknowledgementPacket` calls `refundPacketToken`: [3](#0-2) 

Inside `refundPacketToken`, `isAwayFromOrigin` is again `false` (the class path `p2/c2/nftClass` does not start with `p3/c3/`), so the `else` branch executes. It computes `escrowAddress = GetEscrowAddress(packet.GetSourcePort(), packet.GetSourceChannel())` = `GetEscrowAddress(p3, c3)` and calls: [4](#0-3) 

`MintNFT` immediately calls `IsDenomCreator(ctx, voucherClassID, GetEscrowAddress(p3,c3))`: [5](#0-4) 

The denom creator is `GetEscrowAddress(p2, c2)` ≠ `GetEscrowAddress(p3, c3)`. The check fails, `MintNFT` returns an error, and the refund reverts. The NFT was already burned in the outgoing send and can never be recovered.

The test suite confirms `MintNFT` errors when the sender is not the denom creator: [6](#0-5) 

### Impact Explanation

**Impact: High.** Any NFT that has been relayed through at least two hops (Chain A → Chain B → Chain C) is permanently destroyed if the second-hop packet fails. The NFT is burned on Chain B during the outgoing send, and the refund path is broken. There is no recovery mechanism. The corrupted invariant is: `nftKeeper.HasNFT(voucherClassID, tokenID)` should be `true` after a failed refund, but it is `false` because the mint was rejected.

### Likelihood Explanation

**Likelihood: Medium.** Multi-hop NFT transfers are a standard ICS-721 use case. Packet failure can occur naturally (timeout, invalid receiver address, out-of-gas on destination) or can be deliberately triggered by an attacker who controls the receiver address on Chain C (e.g., setting it to an invalid bech32 string that passes `ValidateBasic` on Chain B but fails on Chain C). The attacker's entry path is a standard `MsgTransfer` on Chain B with a crafted receiver field.

### Recommendation

In `refundPacketToken`, replace the call to `MintNFT` (which enforces `IsDenomCreator`) with `MintNFTUnverified`, which skips the creator check and is already used in genesis initialization for exactly this kind of privileged internal mint: [7](#0-6) 

Alternatively, derive the correct escrow address (the one that created the denom) from the stored `ClassTrace` path rather than from the current packet's source port/channel.

### Proof of Concept

1. Chain A holds native NFT `nftClass/token1`, owned by Alice.
2. Alice sends `nftClass/token1` from Chain A to Chain B via channel `(p1,c1)→(p2,c2)`.
   - Chain B: `processReceivedPacket` creates denom `ibc/{hash(p2/c2/nftClass)}` with creator = `GetEscrowAddress(p2,c2)`. Mints `token1` to Alice on Chain B.
3. Alice sends `ibc/{hash(p2/c2/nftClass)}/token1` from Chain B to Chain C via channel `(p3,c3)→(p4,c4)`.
   - Chain B: `createOutgoingPacket` detects sink chain, calls `BurnNFTUnverified`. NFT is gone from Chain B.
4. Attacker sets receiver on Chain C to an address that fails Chain C's ante-handler. Chain C returns an error ack.
5. Chain B: `OnAcknowledgementPacket` → `refundPacketToken`.
   - `isAwayFromOrigin = false`, `escrowAddress = GetEscrowAddress(p3,c3)`.
   - `MintNFT(ctx, "ibc/{hash(p2/c2/nftClass)}", "token1", ..., GetEscrowAddress(p3,c3), Alice)`.
   - `IsDenomCreator` fails: creator is `GetEscrowAddress(p2,c2)` ≠ `GetEscrowAddress(p3,c3)`.
   - Refund reverts. `token1` is permanently destroyed. [8](#0-7)

func (k Keeper) refundPacketToken(ctx sdk.Context, packet channeltypes.Packet, data types.NonFungibleTokenPacketData) error {
	sender, err := sdk.AccAddressFromBech32(data.Sender)
	if err != nil {
		return err
	}

	classTrace := types.ParseClassTrace(data.ClassId)
	voucherClassID := classTrace.IBCClassID()

	isAwayFromOrigin := types.IsAwayFromOrigin(packet.GetSourcePort(),
		packet.GetSourceChannel(), data.ClassId)

	escrowAddress := types.GetEscrowAddress(packet.GetSourcePort(), packet.GetSourceChannel())

	if isAwayFromOrigin {
		// unescrow tokens back to the sender
		for _, tokenID := range data.TokenIds {
			if err := k.nftKeeper.TransferOwner(ctx, voucherClassID, tokenID, escrowAddress, sender); err != nil {
				return err
			}
		}
	} else {
		// we are sink chain, mint voucher back to sender
		for i, tokenID := range data.TokenIds {
			if err := k.nftKeeper.MintNFT(ctx, voucherClassID, tokenID, "", data.TokenUris[i], "", escrowAddress, sender); err != nil {
				return err
			}
		}
	}

	return nil
}

// MintNFTUnverified mints an NFT without verifying if the owner is the creator of denom
// Needed during genesis initialization
func (k Keeper) MintNFTUnverified(ctx sdk.Context, denomID, tokenID, tokenNm, tokenURI, tokenData string, owner sdk.AccAddress) error {
	if !k.HasDenomID(ctx, denomID) {
		return sdkerrors.Wrapf(types.ErrInvalidDenom, "denom ID %s not exists", denomID)
	}

	if k.HasNFT(ctx, denomID, tokenID) {
		return sdkerrors.Wrapf(types.ErrNFTAlreadyExists, "NFT %s already exists in collection %s", tokenID, denomID)
	}

	k.setNFT(
		ctx, denomID,
		types.NewBaseNFT(
			tokenID,
			tokenNm,
			owner,
			tokenURI,
			tokenData,
		),
	)
	k.setOwner(ctx, denomID, tokenID, owner)
	k.increaseSupply(ctx, denomID)

	return nil
}

func (k Keeper) MintNFT(
	ctx sdk.Context, denomID, tokenID, tokenNm,
	tokenURI, tokenData string, sender, owner sdk.AccAddress,
) error {
	_, err := k.IsDenomCreator(ctx, denomID, sender)
	if err != nil {
		return err
	}

	return k.MintNFTUnverified(ctx, denomID, tokenID, tokenNm, tokenURI, tokenData, owner)
}
