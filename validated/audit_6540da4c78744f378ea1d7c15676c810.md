Audit Report

## Title
`SwapAllowlistExtension.beforeSwap` Checks Router Address Instead of Actual Swapper, Allowing Any User to Bypass the Allowlist — (`metric-periphery/contracts/extensions/SwapAllowlistExtension.sol`)

## Summary

`MetricOmmPool.swap` passes `msg.sender` as the `sender` argument to `_beforeSwap`, which forwards it to `SwapAllowlistExtension.beforeSwap`. When a swap is routed through `MetricOmmSimpleRouter`, `msg.sender` inside the pool is the router contract address, not the end user. If the pool admin allowlists the router to permit router-mediated swaps, every unprivileged user can bypass the allowlist by calling the public, permissionless router — the extension checks the router's identity, not the actual swapper's.

## Finding Description

**Root cause — three-step call chain:**

**Step 1:** `MetricOmmPool.swap` passes `msg.sender` as `sender` to `_beforeSwap`: [1](#0-0) 

**Step 2:** `ExtensionCalling._beforeSwap` forwards that `sender` value verbatim to every registered extension: [2](#0-1) 

**Step 3:** `SwapAllowlistExtension.beforeSwap` checks `allowedSwapper[msg.sender][sender]` — where `msg.sender` is the pool and `sender` is whatever the pool received as its own `msg.sender`: [3](#0-2) 

**Router entry point:** `MetricOmmSimpleRouter.exactInputSingle` calls `pool.swap(params.recipient, ...)` directly, making the pool's `msg.sender` = `address(router)`: [4](#0-3) 

The same pattern applies to `exactInput`, `exactOutputSingle`, and `exactOutput`. [5](#0-4) 

**Exploit flow:**

1. Pool admin deploys pool with `SwapAllowlistExtension`.
2. Admin calls `setAllowedToSwap(pool, address(router), true)` so that approved users can swap via the router.
3. Attacker (not on allowlist) calls `router.exactInputSingle(...)`.
4. Router calls `pool.swap(...)` → pool's `msg.sender` = `address(router)`.
5. Pool calls `_beforeSwap(address(router), ...)` → extension receives `sender = address(router)`.
6. Extension checks `allowedSwapper[pool][address(router)]` → `true` → passes.
7. Attacker's swap executes against the restricted pool.

**Existing guards are insufficient:** The only access-control mechanism is the `allowedSwapper` mapping keyed by the `sender` argument. There is no secondary check on `tx.origin` or any trusted-forwarder mechanism. The `allowAllSwappers` flag is a separate escape hatch and not relevant to the bypass path. [6](#0-5) 

## Impact Explanation

**Admin-boundary break / broken core pool functionality causing loss of funds:**

- The allowlist guard — the sole access-control mechanism for swap-restricted pools — is rendered completely ineffective for all router-mediated swaps.
- Any unprivileged user can swap against a pool the admin intended to restrict, enabling arbitrage or front-running in a pool designed for a closed participant set, draining LP value.
- The pool admin's configuration is meaningless: allowlisting the router (the only way to support router-mediated swaps) simultaneously opens the pool to all users.

This matches the allowed impact gate: *Admin-boundary break — pool admin role check bypassed by an unprivileged path* and *Broken core pool functionality causing loss of funds.*

## Likelihood Explanation

- Trigger is fully unprivileged: any address can call `MetricOmmSimpleRouter` with no special permissions.
- The only required precondition is that the pool has `SwapAllowlistExtension` configured and the admin has allowlisted the router — the exact configuration needed for legitimate router-mediated swaps to work at all.
- No malicious initial pool setup is assumed; the admin acts in good faith per the documented design.
- Repeatable on every swap; no one-time setup cost for the attacker.

## Recommendation

`SwapAllowlistExtension.beforeSwap` must check the **original caller**, not the intermediary. Two concrete options:

1. **Pass the real caller via `extensionData`:** `MetricOmmSimpleRouter` populates `extensionData` with `abi.encode(msg.sender)` before calling `pool.swap`. The extension decodes and checks this value. This requires the router to be trusted not to forge the identity (acceptable given it is a protocol-controlled contract).

2. **Dedicated trusted-forwarder pattern:** The extension maintains a registry of trusted routers. When `sender` is a trusted router, it decodes the real swapper from `extensionData`; otherwise it checks `sender` directly.

The cleanest production fix is option 1: the router already has `msg.sender` available at call time and can encode it into `extensionData` at zero additional trust cost.

## Proof of Concept

```solidity
function test_swapAllowlist_bypassViaRouter() public {
    // Setup: pool has SwapAllowlistExtension; router is allowlisted, attacker is not
    address attacker = makeAddr("attacker");
    deal(token0, attacker, 1e18);

    vm.startPrank(attacker);
    IERC20(token0).approve(address(router), type(uint256).max);

    // Direct swap reverts: attacker not allowlisted
    vm.expectRevert(IMetricOmmPoolActions.NotAllowedToSwap.selector);
    pool.swap(attacker, false, 1000, type(uint128).max, "", "");

    // Router-mediated swap succeeds: router IS allowlisted
    // extension receives sender=address(router), not address(attacker)
    (int128 d0, int128 d1) = router.exactInputSingle(
        IMetricOmmSimpleRouter.ExactInputSingleParams({
            pool: address(pool),
            zeroForOne: false,
            amountIn: 1000,
            amountOutMinimum: 0,
            priceLimitX64: type(uint128).max,
            recipient: attacker,
            extensionData: ""
        })
    );
    // Swap succeeded — allowlist bypassed
    assertTrue(d0 != 0 || d1 != 0);
    vm.stopPrank();
}
```

The test mirrors the existing unit test structure in `metric-periphery/test/extensions/SwapAllowlistSubExtension.t.sol` and requires only a pool configured with `SwapAllowlistExtension` and the router allowlisted. [7](#0-6)

    _checkDeadline(params.deadline);
    _validatePath(params.tokens, params.pools, params.extensionDatas);

    uint256 last = params.pools.length - 1;
    int128 amount = MetricOmmSwapInputs.asAmountSpecifiedIn(params.amountIn);

    for (uint256 i = 0; i <= last; i++) {
      address pool = params.pools[i];
      bool zeroForOne = MetricOmmSwapPath.resolveZeroForOneBitmap(params.zeroForOneBitMap, i);

      _setNextCallbackContext(pool, CALLBACK_MODE_JUST_PAY, i == 0 ? msg.sender : address(this), params.tokens[i]);
      (int128 amount0Delta, int128 amount1Delta) = IMetricOmmPoolActions(pool)
        .swap(
          i == last ? params.recipient : address(this),
          zeroForOne,
          amount,
          MetricOmmSwapPath.openLimit(zeroForOne),
          "",
          params.extensionDatas[i]
        );

      int128 amountInActual = MetricOmmSwapResults.extractAmountIn(zeroForOne, amount0Delta, amount1Delta);
      if (amountInActual < amount) revert InvalidInputAmountAtHop(uint8(i), amountInActual, amount);

      amount = MetricOmmSwapResults.extractAmountOut(zeroForOne, amount0Delta, amount1Delta);
    }

    if (amount <= 0) revert InvalidSwapDeltas();
    amountOut = MetricOmmSwapInputs.int128ToUint128(amount);
    if (amountOut < params.amountOutMinimum) revert InsufficientOutput(amountOut, params.amountOutMinimum);

    _clearExpectedCallbackPool();
  }
