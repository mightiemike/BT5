### Title
`block_casm_hash_v1_declares` Version Gate Skips Declare V2 Transactions, Allowing V1 (Poseidon) Compiled Class Hash Into State - (File: `crates/blockifier/src/transaction/transactions.rs`)

---

### Summary

The `block_casm_hash_v1_declares` versioned-constant flag is intended to reject any new Declare transaction that supplies a V1 (Poseidon-based) compiled class hash once the network has migrated to V2 (Blake-based) hashes. The enforcement check in `DeclareTransaction::run_execute` is gated on `self.version() >= TransactionVersion::THREE`, but `DeclareTransaction::V2` shares the same match arm and is never checked. A V2 Declare carrying a V1 compiled class hash bypasses the guard entirely and writes the wrong hash into state.

---

### Finding Description

In `crates/blockifier/src/transaction/transactions.rs`, the `run_execute` implementation for `DeclareTransaction` handles both V2 and V3 in a single match arm:

```rust
starknet_api::transaction::DeclareTransaction::V2(DeclareTransactionV2 {
    compiled_class_hash, ..
})
| starknet_api::transaction::DeclareTransaction::V3(DeclareTransactionV3 {
    compiled_class_hash, ..
}) => {
    if context.tx_context.block_context.versioned_constants.block_casm_hash_v1_declares
        && self.version() >= TransactionVersion::THREE   // ← V2 is excluded
    {
        self.check_compile_class_hash_v2_declaration()?
    }
    try_declare(self, state, class_hash, Some(*compiled_class_hash))?
}
``` [1](#0-0) 

The condition `self.version() >= TransactionVersion::THREE` explicitly skips the V2 case. `check_compile_class_hash_v2_declaration` verifies that the supplied `compiled_class_hash` equals the Blake-based (V2) hash of the contract class: [2](#0-1) 

When `block_casm_hash_v1_declares = true` (enabled in all versioned constants from `0.14.1` onward), a V2 Declare with a Poseidon (V1) compiled class hash is accepted without this check and the V1 hash is committed to state via `try_declare`. [3](#0-2) 

The `TransactionConverter` in the RPC gateway path only handles `RpcDeclareTransaction::V3`, so this path is not reachable through the public RPC gateway. However, the `native_blockifier` Python wrapper exposes `set_block_casm_hash_v1_declares_in_vc` and accepts arbitrary `AccountTransaction` objects directly, making V2 Declares reachable in that execution path. [4](#0-3) 

---

### Impact Explanation

A V2 Declare transaction with a V1 (Poseidon) compiled class hash is accepted and the wrong hash is written into the `class_hash_to_compiled_class_hash` state map. This produces two concrete wrong-value effects:

1. **Wrong compiled class hash in state / state diff**: The committed state diff contains a V1 hash for the class. The OS migration logic (`migrate_classes_to_v2_casm_hash`) expects to find V1 hashes only for classes declared before the migration epoch; a newly declared class with a V1 hash will be treated as a migration candidate, producing an incorrect `contract_class_changes` dict update and a wrong state root.

2. **Wrong class selected for execution**: `should_migrate` compares `get_compiled_class_hash` (state, V1) against `get_compiled_class_hash_v2` (computed, V2). Because they differ, the class is queued for migration even though it was just declared. If `enable_casm_hash_migration = false` but `block_casm_hash_v1_declares = true`, the V1 hash persists permanently, and any subsequent execution that relies on the compiled class hash to locate the CASM artifact will resolve the wrong artifact. [5](#0-4) 

This matches the impact category: **Critical. Wrong compiled class, CASM/native artifact, class hash, or contract code selected for execution.**

---

### Likelihood Explanation

The RPC gateway rejects all non-V3 transactions, so the vulnerability is not reachable through the public API. It is reachable through the `native_blockifier` Python interface, which is used by the Python-based sequencer layer and in testing. An operator or test harness that submits a V2 Declare with a V1 hash while `block_casm_hash_v1_declares = true` will silently corrupt state. Likelihood is **Medium** given the restricted entry point.

---

### Recommendation

Remove the `self.version() >= TransactionVersion::THREE` guard so that the `block_casm_hash_v1_declares` check applies to all Declare transactions that carry a `compiled_class_hash` (i.e., V2 and V3):

```rust
// Before
if context.tx_context.block_context.versioned_constants.block_casm_hash_v1_declares
    && self.version() >= TransactionVersion::THREE
{
    self.check_compile_class_hash_v2_declaration()?
}

// After
if context.tx_context.block_context.versioned_constants.block_casm_hash_v1_declares {
    self.check_compile_class_hash_v2_declaration()?
}
``` [6](#0-5) 

This mirrors the pattern already enforced in the `TransactionConverter` gateway path, which rejects any Declare whose `compiled_class_hash` does not equal `executable_class_hash_v2`: [7](#0-6) 

---

### Proof of Concept

```rust
// With block_casm_hash_v1_declares = true (default since 0.14.1):
let casm = get_casm_contract_class();
let v1_hash = casm.hash(&HashVersion::V1);  // Poseidon hash
let v2_hash = casm.hash(&HashVersion::V2);  // Blake hash
assert_ne!(v1_hash, v2_hash);

// Construct a V2 Declare with the V1 hash
let declare_v2 = DeclareTransactionV2 {
    compiled_class_hash: v1_hash,  // wrong hash version
    class_hash: ...,
    ..
};
let executable = DeclareTransaction { tx: DeclareTransaction::V2(declare_v2), .. };

// Execute — check_compile_class_hash_v2_declaration is NOT called for V2
let result = executable_tx.execute(&mut state, &block_context);
assert!(result.is_ok());  // succeeds despite block_casm_hash_v1_declares = true

// State now contains the V1 hash
assert_eq!(state.get_compiled_class_hash(class_hash).unwrap(), v1_hash);
// should_migrate will flag this class for migration, corrupting the state diff
```

The existing test `test_declare_tx` in `crates/blockifier/src/transaction/transactions_test.rs` confirms that `DeclareTransactionCasmHashMissMatch` is only expected for `TransactionVersion::THREE` with a V1 hash — no equivalent test case exists for `TransactionVersion::TWO`. [8](#0-7)

    pub fn check_compile_class_hash_v2_declaration(&self) -> Result<(), StarknetApiError> {
        let compiled_class = &self.class_info.contract_class;
        let compiled_class_hash_v2 = match &compiled_class {
            ContractClass::V0(_) => return Ok(()),
            ContractClass::V1((casm, _)) => casm.hash(&HashVersion::V2),
        };
        let compiled_class_hash = self.compiled_class_hash();
        if compiled_class_hash_v2 != compiled_class_hash {
            let err_var = CasmHashMismatch {
                hash: self.class_hash(),
                actual: compiled_class_hash,
                expected: compiled_class_hash_v2,
            };
            return Err(StarknetApiError::DeclareTransactionCasmHashMissMatch(Box::new(err_var)));
        }
        Ok(())
    }

pub fn should_migrate(
    state_reader: &impl StateReader,
    class_hash: ClassHash,
) -> StateResult<Option<(ClassHash, CompiledClassHashV2ToV1)>> {
    let state_compiled_class_hash = state_reader.get_compiled_class_hash(class_hash)?;
    match state_compiled_class_hash {
        // Class hash does not exist in the state, or is a Cairo 0 class.
        CompiledClassHash(hash) if hash == StarkHash::ZERO => Ok(None),
        state_compiled_class_hash => {
            let compiled_class_hash_v2 = state_reader.get_compiled_class_hash_v2(
                class_hash,
                &state_reader.get_compiled_class(class_hash)?,
            )?;
            // If the state compiled class hash is compiled class hash v2, the class should not
            // migrate.
            if state_compiled_class_hash == compiled_class_hash_v2 {
                return Ok(None);
            }
            Ok(Some((class_hash, (compiled_class_hash_v2, state_compiled_class_hash))))
        }
    }
