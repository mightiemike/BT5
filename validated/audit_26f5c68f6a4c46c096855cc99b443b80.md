### Title
Division-by-Zero in `socializeSubaccount` When `openInterest` Is Zero Permanently Blocks Subaccount Finalization ŌĆö (`File: core/contracts/PerpEngine.sol`)

---

### Summary

`PerpEngine.socializeSubaccount` divides by `state.openInterest` without first checking whether it is zero. When a perp product has no remaining open positions (`openInterest == 0`) but a subaccount still carries a negative `vQuoteBalance`, the division reverts with `"DBZ"`, permanently blocking the finalization path for that subaccount.

---

### Finding Description

In `PerpEngine.socializeSubaccount`, when insurance is insufficient to cover a subaccount's negative `vQuoteBalance`, the code attempts to spread the remaining loss across all open-interest holders:

```solidity
if (balance.vQuoteBalance < 0) {
    // socialize across all other participants
    int128 fundingPerShare = -balance.vQuoteBalance.div(
        state.openInterest   // ŌåÉ no zero-check
    );
    state.cumulativeFundingLongX18 += fundingPerShare;
    state.cumulativeFundingShortX18 -= fundingPerShare;
    balance.vQuoteBalance = 0;
}
```

`MathSD21x18.div` enforces `require(y != 0, ERR_DIV_BY_ZERO)`, so if `state.openInterest == 0` the call reverts unconditionally.

The state that triggers this is reachable without any privileged action:

1. A subaccount opens a perp position, accumulates negative funding (`vQuoteBalance < 0`), then closes the position (`amount = 0`). Because closing a position does not automatically settle `vQuoteBalance`, the subaccount retains the negative balance.
2. If this subaccount was the last (or only) participant, `openInterest` drops to `0`.
3. The subaccount's quote balance falls below zero, making it unhealthy and eligible for finalization.
4. A liquidator submits a `LiquidateSubaccount` transaction with `productId = type(uint32).max`, which routes through `_finalizeSubaccount` ŌåÆ `perpEngine.socializeSubaccount`.
5. The division by `state.openInterest == 0` reverts, and the finalization is permanently blocked.

The protocol itself acknowledges this edge case ŌĆö `ERR_NO_OPEN_INTEREST = "NOI"` is defined in `Errors.sol` ŌĆö but no guard is present in `socializeSubaccount`.

By contrast, `PerpEngineState.updateStates` correctly skips products with zero open interest:

```solidity
if (state.openInterest == 0) {
    continue;
}
```

No equivalent guard exists in `socializeSubaccount`.

---

### Impact Explanation

- The insolvent subaccount can never be finalized: its liabilities cannot be socialized, the insurance fund cannot absorb the loss, and the accounting state for that perp product is permanently corrupted.
- Any subsequent attempt to finalize the same subaccount will also revert, because the on-chain state does not change.
- This is a settlement bypass and solvency/accounting corruption: the protocol's last-resort loss-recovery mechanism is rendered inoperable for the affected product.

---

### Likelihood Explanation

The precondition ŌĆö a closed position with residual negative `vQuoteBalance` in a product that has reached zero open interest ŌĆö arises naturally during the lifecycle of any perp product with low participation (e.g., a newly listed or recently delisted product). No special attacker capability is required; a single user who opens and closes a position while accumulating negative funding is sufficient to create the state. The finalization code path is exercised by any liquidator submitting a standard `LiquidateSubaccount` transaction.

---

### Recommendation

Add an explicit guard in `socializeSubaccount` before dividing by `state.openInterest`. If `openInterest == 0` and `balance.vQuoteBalance` is still negative after insurance coverage, the remaining loss has no counterparties to absorb it and should either be written off against the insurance fund entirely or cause a hard revert with a descriptive error (consistent with `ERR_NO_OPEN_INTEREST`):

```solidity
if (balance.vQuoteBalance < 0) {
    require(state.openInterest != 0, ERR_NO_OPEN_INTEREST);
    int128 fundingPerShare = -balance.vQuoteBalance.div(state.openInterest);
    ...
}
```

---

### Proof of Concept

1. Deploy the protocol with a single perp product.
2. Alice opens a long position; negative funding accumulates (`vQuoteBalance < 0`).
3. Alice closes the position (`amount = 0`); `openInterest` drops to `0`; `vQuoteBalance` remains negative.
4. Alice's quote balance goes negative; her subaccount is under maintenance health.
5. Bob (liquidator) submits `LiquidateSubaccount { productId: type(uint32).max, liquidatee: Alice }`.
6. Execution path: `Clearinghouse.liquidateSubaccount` ŌåÆ `delegatecall` ŌåÆ `ClearinghouseLiq.liquidateSubaccountImpl` ŌåÆ `_finalizeSubaccount` ŌåÆ `perpEngine.socializeSubaccount`.
7. Inside `socializeSubaccount`, `balance.vQuoteBalance < 0` and `state.openInterest == 0`; `MathSD21x18.div` reverts with `"DBZ"`.

    function socializeSubaccount(bytes32 subaccount, int128 insurance)
        external
        returns (int128)
    {
        require(msg.sender == address(_clearinghouse), ERR_UNAUTHORIZED);

        uint32[] memory _productIds = getProductIds();
        for (uint128 i = 0; i < _productIds.length; ++i) {
            uint32 productId = _productIds[i];
            (State memory state, Balance memory balance) = getStateAndBalance(
                productId,
                subaccount
            );
            if (balance.vQuoteBalance < 0) {
                int128 insuranceCover = MathHelper.min(
                    insurance,
                    -balance.vQuoteBalance
                );
                insurance -= insuranceCover;
                balance.vQuoteBalance += insuranceCover;
                state.availableSettle += insuranceCover;

                // actually socialize if still not enough
                if (balance.vQuoteBalance < 0) {
                    // socialize across all other participants
                    int128 fundingPerShare = -balance.vQuoteBalance.div(
                        state.openInterest
                    );
                    state.cumulativeFundingLongX18 += fundingPerShare;
                    state.cumulativeFundingShortX18 -= fundingPerShare;
                    balance.vQuoteBalance = 0;
                }
                _setState(productId, state);
                _setBalanceAndUpdateBitmap(productId, subaccount, balance);
            }
        }
        return insurance;
    }
