### Title
Proof-Facts Storage Key Does Not Bind to Transaction Sender — Any Submitter Can Reuse Another User's Verified Proof to Bypass Gateway Proof Verification - (File: crates/apollo_transaction_converter/src/transaction_converter.rs)

### Summary

The `ProofManager` stores and looks up proofs exclusively by `H(proof_facts)`. Because `proof_facts` contains only `(proof_version, program_hash, block_hash, block_number, config_hash)` and never the `sender_address` or transaction hash, any user who copies another user's `proof_facts` and submits a transaction with arbitrary (garbage) proof bytes will have proof verification silently skipped by the gateway. This is the direct sequencer analog of the `OmnichainStaking.unstakeToken` bug: just as that contract burns the caller's power and transfers *any* NFT without checking who deposited it, the gateway burns the cost of proof verification and grants "client-side proving" admission to *any* transaction that presents proof_facts already stored under a matching hash — regardless of who generated the proof or for which transaction.

Two transactions from different senders can carry identical `proof_facts` and therefore share the same `H(proof_facts)` storage key, even though their transaction hashes differ.

**Stateless validator requires proof to be non-empty when proof_facts is non-empty**, so the attacker must supply *some* bytes — but those bytes are never verified once `contains_proof` returns true: [5](#0-4) 

**Gateway flow — proof verification is the only gate; blockifier only checks proof_facts fields, not the proof itself:** [6](#0-5) 

### Impact Explanation

An attacker (User B) who observes a legitimately verified transaction from User A on-chain or in the mempool can:

1. Extract User A's `proof_facts` (public, included in the transaction).
2. Construct their own `InvokeTransactionV3` with the same `proof_facts` and any non-empty garbage `proof` bytes.
3. Submit to the gateway. The stateless validator passes (both fields non-empty). `convert_rpc_tx_to_internal` calls `run_proof_verification`; `contains_proof(proof_facts)` returns `true`; verification is skipped entirely.
4. The transaction is admitted to the mempool and forwarded to the batcher with an unverified proof.

The blockifier's `validate_proof_facts` only checks the structural fields (`block_hash`, `program_hash`, `config_hash`) — it never re-verifies the cryptographic proof. So the transaction executes with "client-side proving" status despite carrying a proof that was never verified for it.

**Matching impact:** *High — Mempool/gateway/RPC admission accepts invalid transactions before sequencing.*

### Likelihood Explanation

- Proof_facts are fully public (included in the serialized transaction, visible on-chain and in the mempool).
- No privileged access is required; any unprivileged user can copy proof_facts from any observed transaction.
- The attacker only needs to supply non-empty bytes for the `proof` field to pass the stateless consistency check.
- The window is open for the entire lifetime of the stored proof (persistent filesystem storage).

### Recommendation

Bind the proof storage key to the transaction identity. The simplest fix is to key the proof manager on `H(tx_hash)` or `H(proof_facts || sender_address || nonce)` instead of `H(proof_facts)` alone. Concretely:

- In `run_proof_verification`, pass the `tx_hash` (or `sender_address + nonce`) alongside `proof_facts` and include it in the lookup key.
- In `ProofManager::set_proof` / `contains_proof`, use a composite key `H(proof_facts || tx_hash)`.
- Alternatively, verify the proof unconditionally on every submission (no caching skip), accepting the CPU cost.

    fn validate_proof_facts(
        &self,
        block_context: &BlockContext,
        state: &mut dyn State,
    ) -> TransactionPreValidationResult<()> {
        // Only Invoke V3 transactions can carry proof facts.
        let Transaction::Invoke(invoke_tx) = &self.tx else {
            return Ok(());
        };
        if invoke_tx.version() < TransactionVersion::THREE {
            return Ok(());
        }

        // Parse proof facts.
        let proof_facts = invoke_tx.proof_facts();
        let snos_proof_facts = match ProofFactsVariant::try_from(&proof_facts)
            .map_err(|e| TransactionPreValidationError::InvalidProofFacts(e.to_string()))?
        {
            ProofFactsVariant::Empty => return Ok(()),
            ProofFactsVariant::Snos(snos_proof_facts) => snos_proof_facts,
        };
        let os_constants = &block_context.versioned_constants.os_constants;

        if !os_constants.allowed_proof_versions.contains(&snos_proof_facts.proof_version.as_felt())
        {
            return Err(TransactionPreValidationError::InvalidProofFacts(format!(
                "Proof version {} is not allowed under this protocol version.",
                snos_proof_facts.proof_version
            )));
        }

        // Validate the program hash.
        let allowed = &os_constants.allowed_virtual_os_program_hashes;
        if !allowed.contains(&snos_proof_facts.program_hash) {
            return Err(TransactionPreValidationError::InvalidProofFacts(format!(
                "Virtual OS program hash {} is not allowed",
                snos_proof_facts.program_hash
            )));
        }

        // Validate the block hash and block number.
        let proof_block_hash = snos_proof_facts.block_hash.0;
        let proof_block_number = snos_proof_facts.block_number.0;
        Self::validate_proof_block_number(
            proof_block_number,
            block_context.block_info.block_number,
        )?;
        Self::validate_proof_block_hash(proof_block_hash, proof_block_number, os_constants, state)?;

        // Validate the config hash.
        let virtual_os_config_hash = block_context.virtual_os_config_hash();
        let proof_config_hash = snos_proof_facts.config_hash;
        if virtual_os_config_hash != proof_config_hash {
            return Err(TransactionPreValidationError::InvalidProofFacts(format!(
                "Virtual OS config hash mismatch. Computed virtual OS config hash: \
                 {virtual_os_config_hash}, expected virtual OS config hash: {proof_config_hash}."
            )));
        }

        Ok(())
    }
