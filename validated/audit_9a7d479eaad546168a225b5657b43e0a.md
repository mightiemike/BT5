### Title
`SwapAllowlistExtension` gates the router address instead of the actual user, allowing full allowlist bypass via the official periphery router - (`File: metric-periphery/contracts/extensions/SwapAllowlistExtension.sol`)

---

### Summary

`SwapAllowlistExtension.beforeSwap` checks the `sender` parameter, which the pool sets to `msg.sender` — the direct caller of `pool.swap()`. When a user routes through `MetricOmmSimpleRouter`, the pool's `msg.sender` is the router contract, not the actual user. A pool admin who allowlists the router (the natural action to enable the official periphery) inadvertently grants every user on-chain unrestricted access to the curated pool, completely defeating the allowlist.

---

### Finding Description

`MetricOmmPool.swap` passes `msg.sender` as the `sender` argument to `_beforeSwap`:

```solidity
// metric-core/contracts/MetricOmmPool.sol
_beforeSwap(
    msg.sender,   // ← router address when called via router
    recipient,
    ...
);
```

`ExtensionCalling._beforeSwap` forwards that value unchanged to every configured extension:

```solidity
// metric-core/contracts/ExtensionCalling.sol
abi.encodeCall(
    IMetricOmmExtensions.beforeSwap,
    (sender, recipient, ...)   // sender = router
)
```

`SwapAllowlistExtension.beforeSwap` then checks whether that `sender` is on the per-pool allowlist:

```solidity
// metric-periphery/contracts/extensions/SwapAllowlistExtension.sol
function beforeSwap(address sender, address, ...)
    external view override returns (bytes4)
{
    if (!allowAllSwappers[msg.sender] && !allowedSwapper[msg.sender][sender]) {
        revert IMetricOmmPoolActions.NotAllowedToSwap();
    }
    ...
}
```

`msg.sender` here is the pool (correct — selects the right config namespace). `sender` is whoever called `pool.swap()`. When the call path is `user → MetricOmmSimpleRouter → pool.swap()`, `sender` equals the router's address, not the user's address. The extension therefore checks `allowedSwapper[pool][router]`, not `allowedSwapper[pool][user]`.

---

### Impact Explanation

**Critical/High — direct bypass of curated-pool access control.**

A pool admin who deploys a curated pool (e.g., KYC-only, institution-only) and allowlists the official `MetricOmmSimpleRouter` — the expected action to let users interact via the supported periphery — grants every address on-chain the ability to swap in that pool. The allowlist provides zero protection once the router is allowlisted. Any non-allowlisted user can execute swaps, drain LP liquidity at oracle prices, and extract value that the pool's curation policy was designed to prevent.

---

### Likelihood Explanation

**High.** The `MetricOmmSimpleRouter` is the documented, supported swap entrypoint. A pool admin who wants users to be able to use the periphery must allowlist the router. The admin has no way to simultaneously allowlist the router and restrict individual users, because the extension only sees the router's address. The misconfiguration is the natural and expected configuration, not an edge case.

---

### Recommendation

The extension must gate the **economically relevant actor** — the address that benefits from the swap — not the intermediary that relays the call. Two options:

1. **Pass the actual user through the pool.** Add a `swapper` or `originator` field to the `swap` call that the router populates with `msg.sender` before forwarding to the pool, and have the pool pass that field (not `msg.sender`) as `sender` to extensions.

2. **Check `recipient` instead of `sender`.** If the router always sets `recipient` to the actual user, the extension can check `recipient`. However, this requires a protocol-level convention that `recipient` always equals the beneficial owner, which must be enforced in the router.

---

### Proof of Concept

```
Setup:
  - Pool deployed with SwapAllowlistExtension configured as beforeSwap hook.
  - Pool admin calls setAllowedToSwap(pool, router, true)
    (to enable the official periphery path).
  - Alice (non-allowlisted) wants to swap.

Attack:
  1. Alice calls MetricOmmSimpleRouter.exactInput(..., recipient=Alice, ...).
  2. Router calls pool.swap(recipient=Alice, ...).
  3. Pool calls _beforeSwap(msg.sender=router, recipient=Alice, ...).
  4. SwapAllowlistExtension.beforeSwap receives sender=router.
  5. allowedSwapper[pool][router] == true → check passes.
  6. Swap executes. Alice receives output tokens.

Result:
  Alice, who is not on the allowlist, successfully swaps in a curated pool.
  The allowlist is entirely bypassed via the official router.
```
