### Title
Transferred NFTs Are Permanently Unburnable — Supply Counter Permanently Overstated (`x/nft/keeper/keeper.go`)

### Summary

`BurnNFT` requires **both** `IsOwner` and `IsDenomCreator` to pass for the **same address**. After `TransferOwner` moves an NFT to a non-creator address, no single address can satisfy both checks simultaneously. The NFT is permanently locked and `decreaseSupply` is never called, causing `GetTotalSupply` to permanently overstate the live collection size.

---

### Finding Description

`BurnNFT` enforces two sequential guards on the single `owner` parameter:

```go
// keeper/keeper.go lines 146–154
nft, err := k.IsOwner(ctx, denomID, tokenID, owner)
if err != nil {
    return err
}

_, err = k.IsDenomCreator(ctx, denomID, owner)
if err != nil {
    return err
}
```

`IsOwner` requires `owner == nft.Owner` (current holder).
`IsDenomCreator` requires `owner == denom.Creator` (original issuer). [1](#0-0) 

After `TransferOwner` executes:

```go
// keeper/keeper.go lines 133–136
nft.Owner = dstOwner.String()
k.setNFT(ctx, denomID, nft)
k.swapOwner(ctx, denomID, tokenID, srcOwner, dstOwner)
``` [2](#0-1) 

The NFT owner is now `dstOwner` (non-creator). The denom creator field is immutable (set once in `SetDenom`, never updated). [3](#0-2) 

Post-transfer state:

| Caller | `IsOwner` | `IsDenomCreator` | Result |
|---|---|---|---|
| `dstOwner` (new owner) | PASS | FAIL | `ErrUnauthorized` |
| `creator` (original) | FAIL | PASS | `ErrUnauthorized` |

Neither party can ever call `BurnNFT` successfully. The `decreaseSupply` call at line 158 is never reached. [4](#0-3) 

Note: `BurnNFTUnverified` (lines 165–179) skips the `IsDenomCreator` check but is only reachable via the IBC transfer path — it is not exposed through `MsgBurnNFT`. [5](#0-4) [6](#0-5) 

---

### Impact Explanation

- Any NFT transferred to a non-creator address becomes permanently unburnable via the `MsgBurnNFT` transaction path.
- `decreaseSupply` is never called for those tokens, so the on-chain supply counter is permanently inflated.
- `GetTotalSupply` returns a value higher than the number of actually live/circulating NFTs for the affected denom, violating the collection accounting invariant.

---

### Likelihood Explanation

This is triggered by the standard, fully supported user flow: issue denom → mint NFT → transfer to another user → attempt burn. No special privileges, governance, or operator access are required. Any user who receives an NFT via `MsgTransferNFT` triggers this condition.

---

### Recommendation

`BurnNFT` should allow burning if the caller is **either** the current owner **or** the denom creator (not both simultaneously). A corrected guard:

```go
func (k Keeper) BurnNFT(ctx sdk.Context, denomID, tokenID string, owner sdk.AccAddress) error {
    if !k.HasDenomID(ctx, denomID) {
        return sdkerrors.Wrapf(types.ErrInvalidDenom, "denom ID %s not exists", denomID)
    }

    nft, err := k.GetNFT(ctx, denomID, tokenID)
    if err != nil {
        return err
    }

    isOwner := owner.Equals(nft.GetOwner())
    _, creatorErr := k.IsDenomCreator(ctx, denomID, owner)
    isDenomCreator := creatorErr == nil

    if !isOwner && !isDenomCreator {
        return sdkerrors.Wrapf(types.ErrUnauthorized, "%s is neither the owner nor the creator", owner)
    }

    k.deleteNFT(ctx, denomID, nft)
    k.deleteOwner(ctx, denomID, tokenID, nft.GetOwner().(sdk.AccAddress))
    k.decreaseSupply(ctx, denomID)
    return nil
}
```

---

### Proof of Concept

Reproducible in a standard Go keeper unit test (no chain required):

```
1. IssueDenom(denomID, creator)
2. MintNFT(denomID, tokenID, sender=creator, owner=creator)
3. TransferOwner(denomID, tokenID, srcOwner=creator, dstOwner=alice)
4. BurnNFT(denomID, tokenID, owner=alice)   → ErrUnauthorized (IsDenomCreator fails)
5. BurnNFT(denomID, tokenID, owner=creator) → ErrUnauthorized (IsOwner fails)
6. Assert GetTotalSupply(denomID) == 1       → supply permanently overstated
```

Both burn attempts fail. The NFT is permanently locked and the supply counter is stuck at 1.

func (k Keeper) SetDenom(ctx sdk.Context, denom types.Denom) error {
	if k.HasDenomID(ctx, denom.Id) {
		return sdkerrors.Wrapf(types.ErrInvalidDenom, "denomID %s has already exists", denom.Id)
	}

	if k.HasDenomNm(ctx, denom.Name) {
		return sdkerrors.Wrapf(types.ErrInvalidDenom, "denomName %s has already exists", denom.Name)
	}

	store := ctx.KVStore(k.storeKey)
	bz := k.cdc.MustMarshal(&denom)
	store.Set(types.KeyDenomID(denom.Id), bz)
	store.Set(types.KeyDenomName(denom.Name), []byte(denom.Id))
	return nil
```

func (m msgServer) BurnNFT(goCtx context.Context, msg *types.MsgBurnNFT) (*types.MsgBurnNFTResponse, error) {
	sender, err := sdk.AccAddressFromBech32(msg.Sender)
	if err != nil {
		return nil, err
	}

	ctx := sdk.UnwrapSDKContext(goCtx)
	if err := m.Keeper.BurnNFT(ctx, msg.DenomId, msg.Id, sender); err != nil {
		return nil, err
	}

	ctx.EventManager().EmitEvents(sdk.Events{
		sdk.NewEvent(
			types.EventTypeBurnNFT,
			sdk.NewAttribute(types.AttributeKeyDenomID, msg.DenomId),
			sdk.NewAttribute(types.AttributeKeyTokenID, msg.Id),
			sdk.NewAttribute(types.AttributeKeyOwner, msg.Sender),
		),
		sdk.NewEvent(
			sdk.EventTypeMessage,
			sdk.NewAttribute(sdk.AttributeKeyModule, types.AttributeValueCategory),
			sdk.NewAttribute(sdk.AttributeKeySender, msg.Sender),
		),
	})

	return &types.MsgBurnNFTResponse{}, nil
