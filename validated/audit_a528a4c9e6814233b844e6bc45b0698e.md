### Title
GC Prunes Fork-Point From `headers_pool`, Permanently Blocking Any Reorg Anchored at the Retention Boundary — (`contract/src/lib.rs`)

---

### Summary

`run_mainchain_gc` is a public, unpermissioned function that removes canonical blocks from **both** `mainchain_header_to_height` and `headers_pool`. `reorg_chain` terminates its walk-back loop only when it finds a block whose hash is present in `mainchain_header_to_height`, and it fetches each predecessor unconditionally from `headers_pool`. If GC prunes the common ancestor (fork point) before the heavier fork tip is submitted, `reorg_chain` walks past the pruned block, calls `headers_pool.get()` on a hash that no longer exists, and panics with `"previous fork block should be there"`. The transaction reverts, the fork can never be applied, and the contract is permanently locked to the lighter chain.

---

### Finding Description

**`run_mainchain_gc` access control**

The function is decorated only with `#[pause(except(roles(Role::UnrestrictedRunGC)))]`. When the contract is not paused, any NEAR account — including an attacker with no staked relayer role — can call it with an arbitrary `batch_size`. [1](#0-0) 

**What GC deletes**

For every height in `start_removal_height..end_removal_height`, GC calls `remove_block_header`, which removes the block from **both** `mainchain_header_to_height` **and** `headers_pool`. [2](#0-1) [3](#0-2) 

**`reorg_chain` termination condition**

The walk-back loop stops only when `mainchain_header_to_height.contains_key(&fork_header_cursor.block_hash)` is true — i.e., when the cursor lands on a block that is still registered as canonical. [4](#0-3) 

**Unconditional `headers_pool` fetch inside the loop**

On every iteration, the loop advances the cursor by fetching the predecessor from `headers_pool` with an `unwrap_or_else(panic)`. There is no bounds check against `mainchain_initial_blockhash`. [5](#0-4) 

**Fork blocks are stored only in `headers_pool`**

`store_fork_header` inserts only into `headers_pool`, not into `mainchain_header_to_height`. So fork blocks are invisible to the loop's termination check. [6](#0-5) 

---

### Impact Explanation

Once the fork point is pruned:

1. The relayer cannot submit the fork tip — `reorg_chain` panics and reverts every attempt.
2. The relayer cannot re-anchor the fork from a higher block — `get_prev_header` for the first fork block (`F_{H+1}`) looks up the pruned common ancestor in `headers_pool` and panics with `"PrevBlockNotFound"`.
3. The contract is permanently stuck on the lighter chain.
4. `verify_transaction_inclusion` and `verify_transaction_inclusion_v2` return `true` for transactions confirmed only on the lighter chain, enabling false SPV proofs.
5. Any downstream bridge or settlement contract that trusts these proofs can be drained or permanently frozen. [7](#0-6) [8](#0-7) 

---

### Likelihood Explanation

- `run_mainchain_gc` requires zero privileges and is callable by any NEAR account.
- The attacker only needs to observe that the chain has grown one block past `gc_threshold` (trivially detectable via `get_mainchain_size` or `get_last_block_height`) and that a fork is in progress (detectable by watching `headers_pool` entries that are not in `mainchain_header_to_height`).
- The attacker calls `run_mainchain_gc(1)` in a single transaction. No staking, no key compromise, no social engineering.
- Bitcoin forks at depth ≈ `gc_threshold` are rare but not impossible (selfish mining, network partitions). An attacker who can also influence Bitcoin mining can manufacture the fork at the exact depth.

---

### Recommendation

1. **Do not delete from `headers_pool` during GC.** GC should only remove entries from `mainchain_height_to_header` and `mainchain_header_to_height`. The `headers_pool` entry for the oldest canonical block must remain until it is confirmed to be unreachable by any pending fork.
2. **Bound `reorg_chain` against `mainchain_initial_blockhash`.** Before fetching `headers_pool.get(&prev_block_hash)`, assert that `prev_block_hash` is not older than `mainchain_initial_blockhash`. If it is, the fork is unresolvable and the submission should be rejected with a clear error rather than a panic.
3. **Gate `run_mainchain_gc` with a role check** (e.g., require `Role::UnrestrictedRunGC` or trusted-relayer status) so that unprivileged accounts cannot race against the relayer.

---

### Proof of Concept

```
State: gc_threshold = K, tip at height T, mainchain_initial_blockhash = block H (H = T-K+1)

Step 1 – Relayer submits fork blocks F_{H+1} … F_T
         (each stored via store_fork_header → headers_pool only)

Step 2 – Relayer submits main-chain block M_{T+1}
         → amount_of_headers_we_store = K+1 > K  (GC is now eligible)

Step 3 – Attacker calls: run_mainchain_gc(1)
         → remove_block_header(block_H_hash)
              mainchain_header_to_height.remove(block_H_hash)  ✓
              headers_pool.remove(block_H_hash)                ✓  ← DELETED
         → mainchain_initial_blockhash = block H+1

Step 4 – Relayer submits F_{T+1} (fork tip, chain_work > main chain)
         → submit_block_header_inner detects fork, calls reorg_chain(F_{T+1}, T+1)
         → reorg_chain walks back: F_{T+1} → F_T → … → F_{H+1}
         → at F_{H+1}: prev_block_hash = block_H_hash
         → mainchain_header_to_height.contains_key(block_H_hash) == false  (pruned)
         → headers_pool.get(block_H_hash) == None
         → PANIC: "previous fork block should be there"
         → transaction reverts

Step 5 – Relayer retries F_{T+1}: same panic, forever.
         Relayer tries to re-submit F_{H+1}: get_prev_header looks up block_H_hash
         → headers_pool.get(block_H_hash) == None
         → PANIC: "PrevBlockNotFound"

Result: contract permanently on lighter chain; all SPV proofs on lighter chain accepted as valid.
``` [9](#0-8) [10](#0-9)

        require!(
            args.confirmations <= self.gc_threshold,
            "The required number of confirmations exceeds the number of blocks stored in memory"
        );

        let heaviest_block_header = self
            .headers_pool
            .get(&self.mainchain_tip_blockhash)
            .unwrap_or_else(|| env::panic_str(ERR_KEY_NOT_EXIST));
        let target_block_height = self
            .mainchain_header_to_height
            .get(&args.tx_block_blockhash)
            .unwrap_or_else(|| env::panic_str("block does not belong to the current main chain"));

        // Check requested confirmations. No need to compute proof if insufficient confirmations.
        require!(
            (heaviest_block_header.block_height).saturating_sub(target_block_height) + 1
                >= args.confirmations,
            "Not enough blocks confirmed"
        );

        let header = self
            .headers_pool
            .get(&args.tx_block_blockhash)
            .unwrap_or_else(|| env::panic_str("cannot find requested transaction block"));

            usize::try_from(args.tx_index).unwrap(),

    }
