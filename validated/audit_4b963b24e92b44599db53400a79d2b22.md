### Title
Missing `isRedelegating` Guard in `validateUndelegatePosition` Allows Slash Evasion — (`x/tieredrewards/keeper/msg_validate.go`)

---

### Summary

`validateUndelegatePosition` does not check whether the position has an active redelegation. An attacker who has triggered exit and redelegated can call `TierUndelegate` after the exit lock duration elapses, removing the staking delegation while the redelegation entry is still pending. When `BeforeRedelegationSlashed` subsequently fires, `slashRedelegationPosition` finds `IsDelegated() == false` and silently returns `nil`, skipping all tier-level slash accounting. The Cosmos SDK's `SlashRedelegation` also skips the slash at the staking level because the destination delegation no longer exists. The attacker's tokens survive in an unbonding delegation and are returned in full after the unbonding period.

---

### Finding Description

**Root cause — missing guard in `validateUndelegatePosition`:**

`validateUndelegatePosition` enforces four checks: owner, `IsDelegated()`, `HasTriggeredExit()`, and `CompletedExitLockDuration()`. [1](#0-0) 

It does **not** call `isRedelegating`. By contrast, every other exit path that touches the delegation does:

- `validateRedelegatePosition` — checks `isRedelegating` and returns `ErrActiveRedelegation` [2](#0-1) 

- `validateExitTierWithDelegation` — same guard [3](#0-2) 

**Slash hook silently skips when `IsDelegated() == false`:**

`slashRedelegationPosition` treats a missing delegation as a defensive no-op and returns `nil` without reducing any stake: [4](#0-3) 

`IsDelegated()` is simply `Delegation != nil`: [5](#0-4) 

After `TierUndelegate` the staking delegation is removed and `setPosition` is called with `ValidatorTransition{PreviousAddress: srcValidator}`, which causes the next `getPositionState` to return `Delegation: nil`. [6](#0-5) 

**Redelegation mapping is not cleaned up by `TierUndelegate`**, so `slashRedelegationPosition` still finds the `unbondingID → posId` entry, loads the position, and hits the `!IsDelegated()` branch: [7](#0-6) 

---

### Impact Explanation

The attacker avoids the redelegation slash at **both** layers:

- **Tier layer**: `slashRedelegationPosition` returns `nil` without adjusting `pos.Delegation` or `PositionCountByValidator`.
- **Staking layer**: The Cosmos SDK's `SlashRedelegation` calls `GetDelegation` on the destination validator; finding none, it skips the slash entry. The tokens remain in the unbonding delegation and are returned in full after the unbonding period.

The invariant stated in the ADR — *"a slashed redelegation always reduces the position's effective stake"* — is violated: [8](#0-7) 

---

### Likelihood Explanation

The attack requires:
1. A position in a tier with a short exit lock duration (e.g., 60 s for Tier 2 in tests).
2. The attacker observes that the source validator is accumulating downtime and will be slashed.
3. The attacker triggers exit and redelegates within the exit lock window (note: `validateRedelegatePosition` blocks redelegate once exit lock has elapsed, so the order must be: trigger exit → redelegate → wait for exit lock → undelegate).
4. After the exit lock elapses, the attacker calls `TierUndelegate` — no `isRedelegating` check blocks this.
5. The slash fires at any point during the 21-day redelegation unbonding period.

Steps 1–4 are fully attacker-controlled. Step 5 is guaranteed if the source validator is already offline. The window between exit lock elapsing and the slash firing can be days or weeks, making the timing trivially achievable.

---

### Recommendation

Add the same `isRedelegating` guard to `validateUndelegatePosition` that already exists in `validateRedelegatePosition` and `validateExitTierWithDelegation`:

```go
// in validateUndelegatePosition, after CompletedExitLockDuration check:
isRedelegating, err := k.isRedelegating(ctx, pos.DelegatorAddress)
if err != nil {
    return err
}
if isRedelegating {
    return errorsmod.Wrapf(types.ErrActiveRedelegation,
        "position %d has an active redelegation", pos.Id)
}
```

This mirrors the pattern already applied consistently to every other delegation-modifying path. [2](#0-1)

### Proof of Concept

```go
// Keeper test sketch (package keeper_test)
func (s *KeeperSuite) TestTierUndelegate_BypassesRedelegationSlash() {
    lockAmount := sdkmath.NewInt(sdk.DefaultPowerReduction.Int64())
    _, bondDenom := s.getStakingData()
    s.fundRewardsPool(sdkmath.NewInt(1_000_000_000), bondDenom)

    // 1. Lock with immediate exit trigger (exit lock = tier duration, e.g. 1s in test tier).
    pos := s.setupNewTierPosition(lockAmount, true /*triggerExit*/)
    owner := sdk.MustAccAddressFromBech32(pos.Owner)

    // 2. Redelegate before exit lock elapses → RedelegationMappings populated.
    dstValAddr, _ := s.createSecondValidator()
    msgServer := keeper.NewMsgServerImpl(s.keeper)
    _, err := msgServer.TierRedelegate(s.ctx, &types.MsgTierRedelegate{
        Owner: pos.Owner, PositionId: pos.Id, DstValidator: dstValAddr.String(),
    })
    s.Require().NoError(err)

    // Confirm mapping exists.
    isRed, _ := s.keeper.IsRedelegating(s.ctx, pos.DelegatorAddress)
    s.Require().True(isRed)

    // 3. Advance past exit lock duration.
    s.advancePastExitDuration()

    // 4. TierUndelegate succeeds — no isRedelegating guard.
    _, err = msgServer.TierUndelegate(s.ctx, &types.MsgTierUndelegate{
        Owner: pos.Owner, PositionId: pos.Id,
    })
    s.Require().NoError(err) // BUG: should return ErrActiveRedelegation

    posAfter, _ := s.keeper.GetPositionState(s.ctx, pos.Id)
    s.Require().False(posAfter.IsDelegated()) // delegation cleared

    // 5. Simulate BeforeRedelegationSlashed — slash is silently skipped.
    var unbondingID uint64
    _ = s.keeper.RedelegationMappings.Walk(s.ctx, nil, func(id, posId uint64) (bool, error) {
        if posId == pos.Id { unbondingID = id; return true, nil }
        return false, nil
    })
    s.Require().NotZero(unbondingID)

    balBefore := s.app.BankKeeper.GetBalance(s.ctx, owner, bondDenom)
    err = s.keeper.Hooks().BeforeRedelegationSlashed(s.ctx, unbondingID, posAfter.Delegation.Shares /* would panic — delegation is nil */)
    // Hook returns nil without slashing — invariant violated.
    s.Require().NoError(err)

    // Tokens remain in unbonding delegation, unslashed.
    ubds, _ := s.app.StakingKeeper.GetUnbondingDelegations(s.ctx, sdk.MustAccAddressFromBech32(pos.DelegatorAddress), 1)
    s.Require().NotEmpty(ubds)
    _ = balBefore // attacker recovers full amount after unbonding period
}
```

func (k Keeper) validateUndelegatePosition(ctx context.Context, pos types.PositionState, owner string) error {
	if !pos.IsOwner(owner) {
		return types.ErrNotPositionOwner
	}

	if !pos.IsDelegated() {
		return types.ErrPositionNotDelegated
	}

	if !pos.HasTriggeredExit() {
		return types.ErrExitNotTriggered
	}

	sdkCtx := sdk.UnwrapSDKContext(ctx)
	if !pos.CompletedExitLockDuration(sdkCtx.BlockTime()) {
		return types.ErrExitLockDurationNotReached
	}

	// skip check for zero amount as we want those positions to be able to close their position properly

	return nil
}

func (k Keeper) deletePositionRedelegationMappings(ctx context.Context, positionId uint64) error {
	iter, err := k.RedelegationMappings.Indexes.ByPosition.MatchExact(ctx, positionId)
	if err != nil {
		return err
	}
	defer iter.Close()

	unbondingIds, err := iter.PrimaryKeys()
	if err != nil {
		return err
	}
	for _, unbondingId := range unbondingIds {
		if err := k.RedelegationMappings.Remove(ctx, unbondingId); err != nil {
			if errors.Is(err, collections.ErrNotFound) {
				continue
			}
			return err
		}
	}
	return nil

```markdown
## 9. Security Invariants

- Only governance can manage tiers and params.
- Rewards accrue only when delegated to a bonded validator.
- No undelegation until exit commitment has elapsed.
- `MsgClearPosition` settles rewards before clearing exit to prevent bonus manipulation past `ExitUnlockAt`.
- Pool balance is never exceeded. User paths fail atomically; the `BeforeRedelegationSlashed` hook forfeits bonus silently if the pool is short, to avoid chain halt.
- Base-reward routing: `SetWithdrawAddr(posDelAddr, owner)` is installed at position creation and cleared on deletion, so every base-reward withdrawal lands on the owner.
- Auth-account materialisation: before the first delegate or undelegate on a position's delegator address, a `BaseAccount` must exist at that address. Both `MsgLockTier` and `MsgCommitDelegationToTier` reserve the account up front via `createPositionDelegatorAccount`.
- CloseOnly tiers block new positions while allowing exits.
- Bonus cannot be double-claimed: `LastEventSeq` prevents event replay, `LastBonusAccrual` prevents segment replay, `LastKnownBonded` prevents unbonded gap overpay.
- All delegation paths reject non-bonded validators (delegate, redelegate, transferDelegationToPosition, transferDelegationFromPosition).
- Vesting accounts are not allowed to create a new tier position.
```
