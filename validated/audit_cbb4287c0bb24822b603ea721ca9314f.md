### Title
`ValidResourceBounds` Variant Flip in Protobuf Round-Trip Produces Wrong Transaction Hash for `AllResources` Transactions with Zero L2/L1DataGas Bounds — (`crates/apollo_protobuf/src/converters/transaction.rs`)

---

### Summary

The protobuf deserializer for `ValidResourceBounds` uses **value-based** discrimination to decide between the `L1Gas` and `AllResources` variants. The transaction hash function `get_invoke_transaction_v3_hash` produces **structurally different hashes** for these two variants even when the numeric field values are identical. A transaction accepted by the gateway as `AllResources { l1_gas: X, l2_gas: {0,0}, l1_data_gas: {0,0} }` receives hash H1 (which includes the `l1_data_gas` felt in the hash preimage). After a protobuf round-trip in the P2P sync path, the same transaction is silently re-classified as `L1Gas(X)` and produces hash H2 ≠ H1. Any node that recomputes the transaction hash from the deserialized content will disagree with the stored/transmitted hash, breaking block verification for syncing nodes.

---

### Finding Description

**Step 1 – Gateway accepts and hashes as `AllResources`.**

`RpcInvokeTransactionV3` always carries `resource_bounds: AllResourceBounds` (never `ValidResourceBounds`). The `From<RpcInvokeTransactionV3> for InternalRpcInvokeTransactionV3` conversion preserves this, and `InternalRpcInvokeTransactionV3::resource_bounds()` always returns `ValidResourceBounds::AllResources(self.resource_bounds)`. [1](#0-0) 

So `calculate_transaction_hash` on the internal type always calls `get_invoke_transaction_v3_hash` with `ValidResourceBounds::AllResources`.

**Step 2 – Hash preimage includes `l1_data_gas` for `AllResources`, omits it for `L1Gas`.**

`get_tip_resource_bounds_hash` conditionally appends the `l1_data_gas` felt only for the `AllResources` variant: [2](#0-1) 

For `AllResources { l1_gas: X, l2_gas: {0,0}, l1_data_gas: {0,0} }` the hash preimage contains `[tip, l1_gas_felt, l2_gas_felt, l1_data_gas_felt]`. For `L1Gas(X)` it contains only `[tip, l1_gas_felt, l2_gas_felt]`. The `l1_data_gas_felt` encodes the 7-byte resource name `L1_DATA` plus two zero 64/128-bit fields, so it is a non-zero felt; the two hashes are distinct.

**Step 3 – Protobuf deserialization silently flips the variant.**

`TryFrom<protobuf::ResourceBounds> for ValidResourceBounds` uses value-based discrimination: [3](#0-2) 

If `l1_data_gas.is_zero() && l2_gas.is_zero()` the result is `ValidResourceBounds::L1Gas`, regardless of whether the sender intended `AllResources`. The same logic appears in the RPC v0.8 path: [4](#0-3) 

**Step 4 – The JSON/storage path uses presence-based discrimination (inconsistent).**

`TryFrom<DeprecatedResourceBoundsMapping> for ValidResourceBounds` produces `AllResources` whenever the `L1DataGas` key is present in the map, even if its value is zero: [5](#0-4) 

So the same transaction content produces `AllResources` via JSON/storage deserialization but `L1Gas` via protobuf deserialization.

**Step 5 – The P2P sync path uses `InvokeTransactionV3` with `ValidResourceBounds`.**

`From<InvokeTransactionV3> for protobuf::InvokeV3` serializes the `resource_bounds` field: [6](#0-5) 

A syncing node that receives a block containing an `InvokeTransactionV3` with `AllResources { l1_gas: X, l2_gas: {0,0}, l1_data_gas: {0,0} }` will deserialize it as `L1Gas(X)`, recompute the hash as H2, and find it does not match the stored hash H1 that was computed at the gateway.

---

### Impact Explanation

Any node syncing via P2P that recomputes transaction hashes from deserialized content will compute H2 ≠ H1 for transactions with `AllResources { l2_gas: {0,0}, l1_data_gas: {0,0} }`. Because the block hash is derived from transaction hashes, the block hash verification will also fail, causing the syncing node to reject a valid block. An attacker who can submit such a transaction (which passes all gateway validation since `l1_gas` is non-zero) can craft a block that full nodes accept but syncing nodes reject, producing a chain-split or persistent sync failure.

This matches: **High. Transaction conversion or signature/hash logic binds the wrong signer, hash, type, or executable payload.**

---

### Likelihood Explanation

The gateway accepts `RpcInvokeTransactionV3` with `AllResourceBounds` where `l2_gas` and `l1_data_gas` are both zero as long as `l1_gas` is non-zero (the `ZeroResourceBounds` check only requires total fee > 0). No special privilege is required. Any external user can submit such a transaction. [7](#0-6) 

---

### Recommendation

Replace value-based discrimination in `TryFrom<protobuf::ResourceBounds> for ValidResourceBounds` with explicit presence/intent signaling. The protobuf schema should carry a discriminant field (e.g., a boolean `is_all_resources`) so that `AllResources` with zero `l2_gas`/`l1_data_gas` is preserved across serialization boundaries. Alternatively, canonicalize the hash computation so that `AllResources { l2_gas: {0,0}, l1_data_gas: {0,0} }` and `L1Gas` produce identical hashes (by treating zero-value `l1_data_gas` as absent in the hash preimage for `AllResources` as well). The inconsistency between the JSON/storage path (presence-based) and the protobuf path (value-based) must be resolved to a single canonical rule.

---

### Proof of Concept

1. Submit an `InvokeV3` transaction via RPC with:
   ```json
   "resource_bounds": {
     "l1_gas": { "max_amount": "0x64", "max_price_per_unit": "0x1" },
     "l2_gas": { "max_amount": "0x0",  "max_price_per_unit": "0x0" },
     "l1_data_gas": { "max_amount": "0x0", "max_price_per_unit": "0x0" }
   }
   ```
   The gateway accepts it (l1_gas fee > 0). Hash H1 is computed via `get_invoke_transaction_v3_hash` with `AllResources`, including the `l1_data_gas` felt in the preimage.

2. The transaction is included in a block. A syncing node requests the block via P2P sync. The `InvokeTransactionV3` is serialized to `protobuf::InvokeV3`.

3. The syncing node deserializes: `l1_data_gas.is_zero() && l2_gas.is_zero()` → `ValidResourceBounds::L1Gas`. It recomputes the hash as H2 (without `l1_data_gas` in the preimage).

4. H1 ≠ H2. The syncing node's block hash verification fails; the valid block is rejected.

The divergence can be confirmed in a unit test by constructing an `InvokeTransactionV3` with `AllResources { l2_gas: {0,0}, l1_data_gas: {0,0} }`, serializing it to protobuf, deserializing it back, and asserting that `original.resource_bounds != deserialized.resource_bounds` and that their hashes differ.

pub fn get_tip_resource_bounds_hash(
    resource_bounds: &ValidResourceBounds,
    tip: &Tip,
) -> Result<Felt, StarknetApiError> {
    let l1_resource_bounds = resource_bounds.get_l1_bounds();
    let l2_resource_bounds = resource_bounds.get_l2_bounds();

    // L1 and L2 gas bounds always exist.
    // Old V3 txs always have L2 gas bounds of zero, but they exist.
    let mut resource_felts = vec![
        get_concat_resource(&l1_resource_bounds, L1_GAS)?,
        get_concat_resource(&l2_resource_bounds, L2_GAS)?,
    ];

    // For new V3 txs, need to also hash the data gas bounds.
    resource_felts.extend(match resource_bounds {
        ValidResourceBounds::L1Gas(_) => vec![],
        ValidResourceBounds::AllResources(all_resources) => {
            vec![get_concat_resource(&all_resources.l1_data_gas, L1_DATA_GAS)?]
        }
    });

    Ok(HashChain::new().chain(&tip.0.into()).chain_iter(resource_felts.iter()).get_poseidon_hash())
}

impl TryFrom<DeprecatedResourceBoundsMapping> for ValidResourceBounds {
    type Error = StarknetApiError;
    fn try_from(
        resource_bounds_mapping: DeprecatedResourceBoundsMapping,
    ) -> Result<Self, Self::Error> {
        if let (Some(l1_bounds), Some(l2_bounds)) = (
            resource_bounds_mapping.0.get(&Resource::L1Gas),
            resource_bounds_mapping.0.get(&Resource::L2Gas),
        ) {
            match resource_bounds_mapping.0.get(&Resource::L1DataGas) {
                Some(data_bounds) => Ok(Self::AllResources(AllResourceBounds {
                    l1_gas: *l1_bounds,
                    l1_data_gas: *data_bounds,
                    l2_gas: *l2_bounds,
                })),
                None => {
                    if l2_bounds.is_zero() {
                        Ok(Self::L1Gas(*l1_bounds))
                    } else {
                        Err(StarknetApiError::InvalidResourceMappingInitializer(format!(
                            "Missing data gas bounds but L2 gas bound is not zero: \
                             {resource_bounds_mapping:?}",
                        )))
                    }
                }
            }
        } else {
            Err(StarknetApiError::InvalidResourceMappingInitializer(format!(
                "{resource_bounds_mapping:?}",
            )))
        }
    }
