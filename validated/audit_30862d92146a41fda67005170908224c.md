### Title
Unconditional Trust of `_cached_sha256_treehash` Without Integrity Validation Allows Stale or Attacker-Controlled Hashes to Propagate — (`File: wheel/python/clvm_rs/tree_hash.py`, `wheel/python/clvm_rs/program.py`)

---

### Summary

The Python wheel layer caches SHA-256 tree hashes on arbitrary Python objects via the `_cached_sha256_treehash` attribute. Both `Treehasher.sha256_treehash()` and `Program.tree_hash()` read this attribute unconditionally and return it without any integrity check against the object's actual content. An attacker-controlled Python object implementing `CLVMStorage` can pre-set `_cached_sha256_treehash` to any 32-byte value, causing every downstream caller — including puzzle-hash derivation, coin-address computation, and `curry_hash` — to operate on a fabricated hash.

---

### Finding Description

In `wheel/python/clvm_rs/tree_hash.py`, the `Treehasher.sha256_treehash()` method begins by reading `_cached_sha256_treehash` from the supplied object:

```python
r = getattr(obj, "_cached_sha256_treehash", None)
if r is not None:
    self.cache_hits += 1
    hash_stack.append(r)
    return
``` [1](#0-0) 

If the attribute is present and non-`None`, the value is accepted as the correct tree hash with no verification that it was computed from the object's actual `.atom` / `.pair` content, that it has the correct length, or that it was produced by the correct algorithm. There is no timestamp, version counter, content digest, or any other freshness signal.

`Program.wrap()` propagates the same unverified attribute from any wrapped `CLVMStorage` object:

```python
o._cached_sha256_treehash = getattr(v, "_cached_sha256_treehash", None)
``` [2](#0-1) 

`Program.tree_hash()` then short-circuits on the cached value without recomputing:

```python
def tree_hash(self) -> bytes:
    if self._cached_sha256_treehash is None:
        self._cached_sha256_treehash = sha256_treehash(self._unwrapped)
    return self._cached_sha256_treehash
``` [3](#0-2) 

The broken invariant is: **the cache is trusted as authoritative solely because it is non-`None`**, with no mechanism to detect that the cached value is stale (object content mutated after caching) or was never correct (attacker-supplied object with a pre-set attribute).

The `Treehasher` also writes the computed hash back to sub-objects via `setattr`, propagating any stale seed hash further into the tree:

```python
setattr(obj, "_cached_sha256_treehash", r)
``` [4](#0-3) [5](#0-4) 

---

### Impact Explanation

Tree hashes in Chia are puzzle hashes — they are the on-chain coin addresses. Every downstream operation that depends on `Program.tree_hash()` is affected:

- **`curry_hash()`** calls `self.tree_hash()` to derive the quoted-mod hash, then uses it to compute the curried puzzle hash. A wrong base hash produces a wrong curried puzzle hash. [6](#0-5) 

- **`CurryTreehasher`** and any wallet code that computes coin addresses from puzzle hashes will silently operate on the fabricated value. [7](#0-6) 

- **`clvm_tree_to_lazy_node`** in the Rust FFI layer reads `.atom` and `.pair` from the Python object to build the Rust-side tree, but the Python-side `tree_hash()` path is entirely separate and still returns the stale cached value. The Rust tree and the Python hash are therefore inconsistent. [8](#0-7) 

The concrete corrupted result is: **`Program.tree_hash()` returns a 32-byte value that does not correspond to the SHA-256 tree hash of the program's actual CLVM content**, causing coin addresses, puzzle hash comparisons, and curried puzzle hashes to be wrong.

---

### Likelihood Explanation

The `CLVMStorage` protocol is a duck-typed Python interface — any object with `.atom` and `.pair` attributes is accepted. [9](#0-8)  Any library or application that accepts externally-supplied CLVM-like objects (e.g., from deserialized network data, user-provided puzzle objects, or third-party CLVM libraries) and passes them through `Program.to()` / `Program.wrap()` / `sha256_treehash()` is exposed. The `_cached_sha256_treehash` attribute is a plain Python attribute with no access control; any caller can set it. The `CLVMTree` class itself sets it unconditionally from deserialized `tree_hashes` data without re-verifying the hash against the blob content:

```python
if self.tree_hashes:
    self._cached_sha256_treehash = self.tree_hashes[index]
``` [10](#0-9) 

This means a malformed or adversarially crafted serialized blob that supplies incorrect tree hashes will have those hashes accepted and propagated without recomputation.

---

### Recommendation

1. **Remove unconditional cache trust in `sha256_treehash()`**: Do not short-circuit on `_cached_sha256_treehash` for objects whose provenance is unknown. Only trust the cache for objects that were produced internally (e.g., `Program` instances created by `new_atom` / `new_pair`).

2. **Validate cache entries on write, not on read**: When `CLVMTree.__init__` sets `_cached_sha256_treehash` from deserialized `tree_hashes`, verify the hash against the atom bytes for leaf nodes before storing it. [11](#0-10) 

3. **Bind the cache to content**: Store a content fingerprint (e.g., `id(self.atom)` or a hash of the atom bytes) alongside the cached hash, and invalidate the cache if the fingerprint changes.

4. **Do not propagate `_cached_sha256_treehash` in `Program.wrap()`** from arbitrary external objects without first verifying the hash is consistent with the object's `.atom` content (for atoms) or recomputing it. [12](#0-11) 

---

### Proof of Concept

```python
from clvm_rs.clvm_storage import CLVMStorage
from clvm_rs.program import Program

# Attacker-controlled object: atom b"foo" but with a fabricated tree hash
class MaliciousStorage(CLVMStorage):
    def __init__(self):
        self.atom = b"foo"
        self._pair = None
        # Pre-set a completely wrong hash
        self._cached_sha256_treehash = b"\xff" * 32

obj = MaliciousStorage()
p = Program.wrap(obj)

# Returns b"\xff" * 32 — not the real SHA-256 tree hash of b"foo"
print(p.tree_hash().hex())  # ffffffffffffffffffffffffffffffffffffffffffffffffffffffffffffffff

# The real hash would be:
from clvm_rs.tree_hash import sha256_treehash
real = Program.new_atom(b"foo")
real._cached_sha256_treehash = None  # force recompute
print(real.tree_hash().hex())  # 0080b50a51ecd0ccfaaa4d49dba866fe58724f18445d30202bafb03e21eef6cb

# curry_hash now derives a wrong puzzle hash silently
mod = Program.new_atom(b"\x02")  # some mod
wrong_curry_hash = mod.curry_hash(p.tree_hash())
# wrong_curry_hash is derived from 0xff*32 instead of the real hash
```

The fabricated hash propagates through `Program.wrap()` at line 142 of `program.py`, is returned by `Program.tree_hash()` at line 286, and is consumed by `curry_hash()` at line 359 — all without any integrity check against the actual CLVM content.

fn clvm_tree_to_lazy_node(obj: Bound<'_, PyAny>) -> PyResult<LazyNode> {
    let mut allocator = Allocator::new();

    let mut identity_map: HashMap<usize, NodePtr> = HashMap::new();
    let mut atom_map: HashMap<Vec<u8>, NodePtr> = HashMap::new();
    let mut pair_map: HashMap<(NodePtr, NodePtr), NodePtr> = HashMap::new();

    enum WorkItem<'py> {
        Visit(Bound<'py, PyAny>),
        BuildPair {
            id: usize,
            left_id: usize,
            right_id: usize,
        },
    }

    let root_ptr = obj.as_ptr() as usize;
    let mut stack: Vec<WorkItem<'_>> = vec![WorkItem::Visit(obj)];

    while let Some(item) = stack.pop() {
        match item {
            WorkItem::Visit(pyobj) => {
                let id = pyobj.as_ptr() as usize;

                if identity_map.contains_key(&id) {
                    continue;
                }

                let atom_val: Option<Vec<u8>> = pyobj.getattr("atom")?.extract()?;

                if let Some(bytes) = atom_val {
                    let node = if let Some(&existing) = atom_map.get(&bytes) {
                        existing
                    } else {
                        let new_node = allocator
                            .new_atom(&bytes)
                            .map_err(|e| pyo3::exceptions::PyMemoryError::new_err(e.to_string()))?;
                        atom_map.insert(bytes, new_node);
                        new_node
                    };
                    identity_map.insert(id, node);
                } else {
                    let pair_val: Option<(Bound<'_, PyAny>, Bound<'_, PyAny>)> =
                        pyobj.getattr("pair")?.extract()?;

                    if let Some((left, right)) = pair_val {
                        let left_id = left.as_ptr() as usize;
                        let right_id = right.as_ptr() as usize;

                        let left_done = identity_map.contains_key(&left_id);
                        let right_done = identity_map.contains_key(&right_id);

                        if left_done && right_done {
                            let l = identity_map[&left_id];
                            let r = identity_map[&right_id];
                            let node = if let Some(&existing) = pair_map.get(&(l, r)) {
                                existing
                            } else {
                                let new_node = allocator.new_pair(l, r).map_err(|e| {
                                    pyo3::exceptions::PyMemoryError::new_err(e.to_string())
                                })?;
                                pair_map.insert((l, r), new_node);
                                new_node
                            };
                            identity_map.insert(id, node);
                        } else {
                            stack.push(WorkItem::BuildPair {
                                id,
                                left_id,
                                right_id,
                            });
                            if !right_done {
                                stack.push(WorkItem::Visit(right));
                            }
                            if !left_done {
                                stack.push(WorkItem::Visit(left));
                            }
                        }
                    } else {
                        return Err(pyo3::exceptions::PyValueError::new_err(
                            "CLVM object has neither .atom nor .pair",
                        ));
                    }
                }
            }
            WorkItem::BuildPair {
                id,
                left_id,
                right_id,
            } => {
                let l = identity_map[&left_id];
                let r = identity_map[&right_id];
                let node = if let Some(&existing) = pair_map.get(&(l, r)) {
                    existing
                } else {
                    let new_node = allocator
                        .new_pair(l, r)
                        .map_err(|e| pyo3::exceptions::PyMemoryError::new_err(e.to_string()))?;
                    pair_map.insert((l, r), new_node);
                    new_node
                };
                identity_map.insert(id, node);
            }
        }
    }

    let root = identity_map[&root_ptr];
    Ok(LazyNode::new(Rc::new(allocator), root))
}
