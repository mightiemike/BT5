### Title
Protobuf `ValidResourceBounds` Deserialization Silently Downgrades `AllResources` to `L1Gas`, Causing P2P Rejection of Valid V3 Transactions - (File: `crates/apollo_protobuf/src/converters/transaction.rs`)

### Summary

The protobuf deserialization of `ValidResourceBounds` uses a value-based heuristic — `l1_data_gas.is_zero() && l2_gas.is_zero()` — to decide whether to reconstruct `AllResources` or `L1Gas`. This is the exact analog of the PoolMath bug: an ordering invariant (`AllResources` for all RPC V3 transactions) is established at the RPC/gateway boundary, but a subsequent normalization step (protobuf round-trip) silently violates it when both `l1_data_gas` and `l2_gas` are zero. The downstream conversion from `InvokeTransactionV3` back to `RpcInvokeTransactionV3` then hard-fails, causing valid transactions to be rejected by every P2P-receiving node.

### Finding Description

**Step 1 — Invariant established at the RPC boundary.**

`RpcInvokeTransactionV3` carries `resource_bounds: AllResourceBounds` (a concrete struct, not the enum). Every RPC V3 invoke transaction is therefore always `ValidResourceBounds::AllResources` when the gateway computes its hash:

```rust
// crates/starknet_api/src/rpc_transaction.rs:636-638
impl InvokeTransactionV3Trait for InternalRpcInvokeTransactionV3 {
    fn resource_bounds(&self) -> ValidResourceBounds {
        ValidResourceBounds::AllResources(self.resource_bounds)  // always AllResources
    }
```

The hash is computed via `get_invoke_transaction_v3_hash`, which calls `get_tip_resource_bounds_hash`. For `AllResources`, the hash **includes** the `L1_DATA_GAS` field (even when zero); for `L1Gas`, it does **not**:

```rust
// crates/starknet_api/src/transaction_hash.rs:203-208
resource_felts.extend(match resource_bounds {
    ValidResourceBounds::L1Gas(_) => vec![],
    ValidResourceBounds::AllResources(all_resources) => {
        vec![get_concat_resource(&all_resources.l1_data_gas, L1_DATA_GAS)?]
    }
});
```

**Step 2 — Normalization step (protobuf round-trip) violates the invariant.**

When the transaction is propagated via mempool P2P, it is serialized to `protobuf::InvokeV3WithProof`. On the receiving side, `TryFrom<protobuf::ResourceBounds> for ValidResourceBounds` applies the following heuristic:

```rust
// crates/apollo_protobuf/src/converters/transaction.rs:427-435
let l1_data_gas = value.l1_data_gas.unwrap_or_default();
// ...
Ok(if l1_data_gas.is_zero() && l2_gas.is_zero() {
    ValidResourceBounds::L1Gas(l1_gas)          // ← invariant broken
} else {
    ValidResourceBounds::AllResources(AllResourceBounds { l1_gas, l2_gas, l1_data_gas })
})
```

A valid `AllResources` transaction with `l1_data_gas = 0` and `l2_gas = 0` is silently downgraded to `L1Gas`.

**Step 3 — Downstream conversion hard-fails.**

The receiving node then calls `TryFrom<InvokeTransactionV3> for RpcInvokeTransactionV3`:

```rust
// crates/starknet_api/src/rpc_transaction.rs:591-597
resource_bounds: match value.resource_bounds {
    ValidResourceBounds::AllResources(bounds) => bounds,
    _ => {
        return Err(StarknetApiError::OutOfRange {
            string: "resource_bounds".to_string(),
        });
    }
},
```

This error is caught in the mempool P2P converter and returned as `DEPRECATED_RESOURCE_BOUNDS_ERROR`:

```rust
// crates/apollo_protobuf/src/converters/rpc_transaction.rs:129-130
// This conversion can fail only if the resource_bounds are not AllResources.
Ok(Self { proof, ..snapi_invoke.try_into().map_err(|_| DEPRECATED_RESOURCE_BOUNDS_ERROR)? })
```

The transaction is rejected by every P2P peer.

### Impact Explanation

A valid `RpcInvokeTransactionV3` with `AllResourceBounds { l1_data_gas: 0, l2_gas: 0, l1_gas: <nonzero> }` is accepted by the gateway (hash computed correctly with `AllResources`), but every P2P-receiving node rejects it with `DEPRECATED_RESOURCE_BOUNDS_ERROR`. The transaction can never propagate beyond the originating node. If the originating node is the proposer and includes the transaction in a block, validators that receive the block via P2P sync cannot reconstruct or validate the transaction, producing a divergent view. This matches the **High** impact scope: "Mempool/gateway/RPC admission accepts invalid transactions or rejects valid transactions before sequencing."

### Likelihood Explanation

Any user who submits a V3 invoke transaction with zero `l2_gas` and zero `l1_data_gas` (a plausible configuration for a simple ETH-only fee payment) triggers this path. No privileged access is required; the RPC endpoint accepts the transaction normally. The condition is reachable with ordinary user input.

### Recommendation

Remove the value-based heuristic from `TryFrom<protobuf::ResourceBounds> for ValidResourceBounds`. The variant (`L1Gas` vs `AllResources`) must be encoded explicitly in the protobuf schema (e.g., a dedicated enum field or a presence flag for `l1_data_gas`), not inferred from whether the numeric values happen to be zero. Until the schema is updated, the deserializer should default to `AllResources` for all V3 transactions, since `L1Gas` is only valid for pre-0.13.3 transactions that would never appear in the mempool P2P path.

### Proof of Concept

```
1. Submit RpcInvokeTransactionV3 with:
     resource_bounds = AllResourceBounds {
         l1_gas:      { max_amount: 1000, max_price_per_unit: 1 },
         l2_gas:      { max_amount: 0,    max_price_per_unit: 0 },
         l1_data_gas: { max_amount: 0,    max_price_per_unit: 0 },
     }

2. Gateway accepts; hash H is computed using AllResources
   (includes L1_DATA_GAS=0 in the Poseidon preimage).

3. Originating node propagates RpcTransaction via mempool P2P.

4. Receiving node deserializes protobuf::ResourceBounds:
     l1_data_gas.is_zero() && l2_gas.is_zero()  →  ValidResourceBounds::L1Gas(l1_gas)

5. TryFrom<InvokeTransactionV3> for RpcInvokeTransactionV3 hits the
   non-AllResources arm → Err(OutOfRange) → DEPRECATED_RESOURCE_BOUNDS_ERROR.

6. Receiving node drops the transaction; it never enters the remote mempool.
   Any block containing this transaction cannot be validated by P2P peers.

pub fn get_tip_resource_bounds_hash(
    resource_bounds: &ValidResourceBounds,
    tip: &Tip,
) -> Result<Felt, StarknetApiError> {
    let l1_resource_bounds = resource_bounds.get_l1_bounds();
    let l2_resource_bounds = resource_bounds.get_l2_bounds();

    // L1 and L2 gas bounds always exist.
    // Old V3 txs always have L2 gas bounds of zero, but they exist.
    let mut resource_felts = vec![
        get_concat_resource(&l1_resource_bounds, L1_GAS)?,
        get_concat_resource(&l2_resource_bounds, L2_GAS)?,
    ];

    // For new V3 txs, need to also hash the data gas bounds.
    resource_felts.extend(match resource_bounds {
        ValidResourceBounds::L1Gas(_) => vec![],
        ValidResourceBounds::AllResources(all_resources) => {
            vec![get_concat_resource(&all_resources.l1_data_gas, L1_DATA_GAS)?]
        }
    });

    Ok(HashChain::new().chain(&tip.0.into()).chain_iter(resource_felts.iter()).get_poseidon_hash())

pub(crate) fn get_invoke_transaction_v3_hash<T: InvokeTransactionV3Trait>(
    transaction: &T,
    chain_id: &ChainId,
    transaction_version: &TransactionVersion,
) -> Result<TransactionHash, StarknetApiError> {
    let tip_resource_bounds_hash =
        get_tip_resource_bounds_hash(&transaction.resource_bounds(), transaction.tip())?;
    let paymaster_data_hash =
        HashChain::new().chain_iter(transaction.paymaster_data().0.iter()).get_poseidon_hash();
    let data_availability_mode = concat_data_availability_mode(
        transaction.nonce_data_availability_mode(),
        transaction.fee_data_availability_mode(),
    );
    let account_deployment_data_hash = HashChain::new()
        .chain_iter(transaction.account_deployment_data().0.iter())
        .get_poseidon_hash();
    let calldata_hash =
        HashChain::new().chain_iter(transaction.calldata().0.iter()).get_poseidon_hash();
    let mut hash_chain = HashChain::new()
        .chain(&INVOKE)
        .chain(&transaction_version.0)
        .chain(transaction.sender_address().0.key())
        .chain(&tip_resource_bounds_hash)
        .chain(&paymaster_data_hash)
        .chain(&Felt::try_from(chain_id)?)
        .chain(&transaction.nonce().0)
        .chain(&data_availability_mode)
        .chain(&account_deployment_data_hash)
        .chain(&calldata_hash);
    if !transaction.proof_facts().0.is_empty() {
        let proof_facts_hash =
            HashChain::new().chain_iter(transaction.proof_facts().0.iter()).get_poseidon_hash();
        hash_chain = hash_chain.chain(&proof_facts_hash);
    }
    Ok(TransactionHash(hash_chain.get_poseidon_hash()))
}
