### Title
`minDepositRate` Applies Multiplier to Both Deposit and Borrow Accumulators, Creating Unbacked Net Liquidity — (`core/contracts/SpotEngineState.sol`)

---

### Summary

The `minDepositRate` block in `_updateState` multiplies **both** `cumulativeDepositsMultiplierX18` and `cumulativeBorrowsMultiplierX18` by `minDepositRateMultiplierX18`. This inflates the net withdrawable liquidity (`totalDeposits − totalBorrows`) by `T × (minDepositRateMultiplierX18 − 1)` per tick without any corresponding token inflow to the Clearinghouse, violating the protocol's own documented invariant.

---

### Finding Description

The protocol documents its core accounting invariant at lines 102–113:

> "if we don't take fees into account, the liquidity, which is `(deposits - borrows)` should remain the same after updating state"
> `depositRateMultiplier = utilization × (borrowRateMultiplier − 1) + 1`

The normal interest path (lines 129–137) correctly preserves this: borrowers pay interest that flows to depositors, so net liquidity `T = D − B` is unchanged. [1](#0-0) 

The `minDepositRate` block then applies `minDepositRateMultiplierX18` to **both** accumulators: [2](#0-1) 

After this block:
- New net withdrawable = `(D × depositRateMultiplier − B × borrowRateMultiplier) × minDepositRateMultiplierX18`
- = `T × minDepositRateMultiplierX18`
- > `T` (vault balance, which is unchanged — no tokens moved)

The excess `T × (minDepositRateMultiplierX18 − 1)` is unbacked. No insurance fund is debited, no protocol reserve is drawn, and no new tokens enter the Clearinghouse.

**Worst case — zero utilization:**

When `utilizationRatioX18 == 0`, the code explicitly sets `borrowerRateX18 = 0` (line 75), so the normal interest path is a no-op. The minDepositRate block then grows `cumulativeDepositsMultiplierX18` while `totalBorrowsNormalized = 0`, meaning the entire vault balance `D` becomes unbacked by `D × (minDepositRateMultiplierX18 − 1)` per tick. [3](#0-2) 

---

### Impact Explanation

Each `SpotTick` that triggers `updateStates(dt)` with a product configured with nonzero `minDepositRateX18` inflates depositor balances beyond what the Clearinghouse holds. Over time, depositors can collectively withdraw more tokens than were ever deposited. The shortfall is `T × (minDepositRateMultiplierX18^N − 1)` after N ticks, growing exponentially. This is unbacked collateral creation — a Critical-scope impact.

---

### Likelihood Explanation

`minDepositRateX18` is a configurable field in `Config` and is already present in the deployed `QUOTE_PRODUCT_ID` config (initialized to `0`, but settable to nonzero for other products). [4](#0-3)  `SpotTick` transactions are submitted by the sequencer as part of normal protocol operation — no attacker action is needed to trigger the ticks. Any product with `minDepositRateX18 > 0` is continuously affected. [5](#0-4) 

---

### Recommendation

The `minDepositRate` subsidy must be funded from somewhere. Two correct approaches:

1. **Charge borrowers only:** Apply `minDepositRateMultiplierX18` only to `cumulativeBorrowsMultiplierX18` (not deposits). This passes the cost to borrowers and preserves net liquidity.
2. **Fund from insurance/protocol reserve:** Deduct `T × (minDepositRateMultiplierX18 − 1)` from the insurance fund or a designated protocol reserve account each tick, crediting it to depositors. This preserves the vault invariant.

The current code applies the multiplier symmetrically to both sides, which is mathematically equivalent to creating tokens from nothing.

---

### Proof of Concept

```
Setup:
  - Product P configured with minDepositRateX18 = 0.10e18 (10% APY)
  - Alice deposits 1000 USDC → Clearinghouse holds 1000 USDC
  - No borrows: utilizationRatio = 0

Tick 1 (dt = 31536000, i.e., 1 year):
  - minDepositRateMultiplierX18 = (1 + 0.10/31536000)^31536000 ≈ 1.10517
  - cumulativeDepositsMultiplierX18 *= 1.10517
  - cumulativeBorrowsMultiplierX18 *= 1.10517 (no effect, totalBorrowsNormalized = 0)
  - Alice's withdrawable = 1000 × 1.10517 = 1105.17 USDC
  - Clearinghouse balance = 1000 USDC

Assert: Clearinghouse.balanceOf(USDC) >= sum(withdrawable) → FAILS
  1000 < 1105.17  ✗

Alice calls WithdrawCollateral for 1105.17 USDC → vault drained by 105.17 USDC
that was never deposited.
```

The `updateStates` entry point is `onlyEndpoint`, but the sequencer submits `SpotTick` transactions as routine protocol operation — no privilege escalation is required. [6](#0-5)
