Looking at the full `ClearinghouseLiq.sol` code, I can now trace the exact storage mutation and determine whether the invariant breaks.

### Title
Stale `lastLiquidationFees` from Prior Subaccount Contaminates Insurance Accounting During Unrelated Subaccount Finalization — (`core/contracts/ClearinghouseLiq.sol`)

---

### Summary

`lastLiquidationFees` is a single shared storage slot that is written by every call to `_handleLiquidationPayment` and read by `_finalizeSubaccount`. Because it is never scoped to a specific liquidatee and is never reset at the start of `liquidateSubaccountImpl`, a non-zero value left by liquidating subaccount A is silently consumed when a different subaccount B is finalized. This causes `perpEngine.socializeSubaccount` to receive `insurance − lastLiquidationFees_from_A` instead of the full `insurance`, potentially triggering excess socialization of losses onto other traders even when the insurance fund is sufficient to cover B's bad debt in full.

---

### Finding Description

**Storage declaration** — `lastLiquidationFees` is a plain `int128` field in `ClearinghouseStorage`, shared across all liquidation calls with no per-subaccount scoping.

**Write path** — every call to `_handleLiquidationPayment` unconditionally overwrites it:

```solidity
// ClearinghouseLiq.sol line 579-586
insurance += v.liquidationFees;
lastLiquidationFees = v.liquidationFees;   // ← always overwritten, never cleared
``` [1](#0-0) 

**Read path** — `_finalizeSubaccount` subtracts it from the working insurance copy before passing it to `perpEngine.socializeSubaccount`, then adds it back only *after* socialization has already executed:

```solidity
// ClearinghouseLiq.sol lines 368-411
v.insurance = insurance;
v.insurance -= lastLiquidationFees;          // ← stale value from A applied to B
...
v.insurance = perpEngine.socializeSubaccount(txn.liquidatee, v.insurance);
...
if (v.insurance <= 0) {
    spotEngine.socializeSubaccount(txn.liquidatee);
}
v.insurance += lastLiquidationFees;          // ← added back, but socialization already happened
insurance = v.insurance;
``` [2](#0-1) 

**Design intent (comment lines 581–585):** the mechanism is meant to prevent the fees just collected during the *same* subaccount's last liquidation step from blocking socialization of *that same* subaccount. The comment explicitly describes a single-subaccount sequence. [3](#0-2) 

**The cross-subaccount contamination:** `liquidateSubaccountImpl` has no guard that resets `lastLiquidationFees` to zero at entry, and no check that the stored value belongs to the current `txn.liquidatee`. When the call sequence is:

1. `liquidateSubaccountImpl(A, productId=P)` → `_handleLiquidationPayment` → `lastLiquidationFees = F_A`
2. `liquidateSubaccountImpl(B, productId=type(uint32).max)` → `_finalizeSubaccount` reads `lastLiquidationFees = F_A`

B's finalization passes `insurance − F_A` to `perpEngine.socializeSubaccount` instead of `insurance`. [4](#0-3) 

**`perpEngine.socializeSubaccount` behavior** — it uses the passed `insurance` value to cover `vQuoteBalance` deficits; any shortfall is socialized by adjusting `cumulativeFundingLongX18` / `cumulativeFundingShortX18` across all open-interest holders. Once socialized, the loss is permanent and cannot be reversed. [5](#0-4) 

---

### Impact Explanation

Let:
- `insurance = I`
- `lastLiquidationFees` (from A) = `F_A > 0`
- B's total bad debt = `D_B`
- Condition: `I − F_A < D_B ≤ I`

Under this condition the insurance fund is sufficient to cover B's bad debt in full, but `perpEngine.socializeSubaccount` receives only `I − F_A`, leaving `D_B − (I − F_A)` to be socialized across all other perp participants. After socialization, `v.insurance += lastLiquidationFees` restores the insurance storage to `I − D_B + (D_B − (I − F_A)) + F_A = I` — the insurance fund balance is arithmetically preserved, but the socialization loss has already been permanently distributed to innocent traders. This is an incorrect value transfer: losses that the protocol's own insurance fund was obligated to absorb are instead pushed onto other participants.

Additionally, if `I − F_A ≤ 0` while `I > 0`, `spotEngine.socializeSubaccount` is triggered (line 408), diluting all spot depositors' `cumulativeDepositsMultiplierX18`, again when the fund had sufficient coverage. [6](#0-5) [7](#0-6) 

---

### Likelihood Explanation

The attack requires no special privileges. Any liquidator can:

1. Identify subaccount B that is under maintenance health and has all positions closed (ready to finalize).
2. Submit a normal liquidation of any subaccount A (even a trivially small one) to set `lastLiquidationFees = F_A`.
3. Immediately submit `liquidateSubaccountImpl(B, productId=type(uint32).max)`.

Because the sequencer processes signed liquidation transactions in submission order, the liquidator controls the interleaving. The precondition `I − F_A < D_B ≤ I` is achievable by choosing A's liquidation size to produce a suitably large `F_A`. The window is not race-dependent; the liquidator submits both transactions in the same batch.

---

### Recommendation

Reset `lastLiquidationFees` to zero at the start of `_finalizeSubaccount` (or at the entry of `liquidateSubaccountImpl` when `productId == type(uint32).max`), so that a finalization call that was not immediately preceded by a `_handleLiquidationPayment` call for the same subaccount does not inherit stale fees:

```solidity
// At the top of _finalizeSubaccount, before line 368:
int128 feesToRestore = lastLiquidationFees;
lastLiquidationFees = 0;
// ... use feesToRestore in place of lastLiquidationFees below
```

Alternatively, tag `lastLiquidationFees` with the subaccount it belongs to (`lastLiquidationFeesSubaccount`) and only apply it when `txn.liquidatee == lastLiquidationFeesSubaccount`, defaulting to zero otherwise.

    function socializeSubaccount(bytes32 subaccount, int128 insurance)
        external
        returns (int128)
    {
        require(msg.sender == address(_clearinghouse), ERR_UNAUTHORIZED);

        uint32[] memory _productIds = getProductIds();
        for (uint128 i = 0; i < _productIds.length; ++i) {
            uint32 productId = _productIds[i];
            (State memory state, Balance memory balance) = getStateAndBalance(
                productId,
                subaccount
            );
            if (balance.vQuoteBalance < 0) {
                int128 insuranceCover = MathHelper.min(
                    insurance,
                    -balance.vQuoteBalance
                );
                insurance -= insuranceCover;
                balance.vQuoteBalance += insuranceCover;
                state.availableSettle += insuranceCover;

                // actually socialize if still not enough
                if (balance.vQuoteBalance < 0) {
                    // socialize across all other participants
                    int128 fundingPerShare = -balance.vQuoteBalance.div(
                        state.openInterest
                    );
                    state.cumulativeFundingLongX18 += fundingPerShare;
                    state.cumulativeFundingShortX18 -= fundingPerShare;
                    balance.vQuoteBalance = 0;
                }
                _setState(productId, state);
                _setBalanceAndUpdateBitmap(productId, subaccount, balance);
            }
        }
        return insurance;

    function socializeSubaccount(bytes32 subaccount) external {
        require(msg.sender == address(_clearinghouse), ERR_UNAUTHORIZED);

        uint32[] memory _productIds = getProductIds();
        for (uint128 i = 0; i < _productIds.length; ++i) {
            uint32 productId = _productIds[i];

            State memory state = states[productId];
            Balance memory balance = balanceNormalizedToBalance(
                state,
                balances[productId][subaccount]
            );
            if (balance.amount < 0) {
                int128 totalDeposited = state.totalDepositsNormalized.mul(
                    state.cumulativeDepositsMultiplierX18
                );

                state.cumulativeDepositsMultiplierX18 = (totalDeposited +
                    balance.amount).div(state.totalDepositsNormalized);

                require(state.cumulativeDepositsMultiplierX18 > 0);

                state.totalBorrowsNormalized += balance.amount.div(
                    state.cumulativeBorrowsMultiplierX18
                );

                _setBalanceAndUpdateBitmap(
                    productId,
                    subaccount,
                    BalanceNormalized({amountNormalized: 0})
                );
                _setState(productId, state);
            }
        }
    }
