### Title
Incorrect `LastKnownBonded = true` assumption after `MsgTierRedelegate` to an unbonded validator — (`x/tieredrewards/keeper/msg_server.go`)

---

### Summary

After `MsgTierRedelegate`, the position's `LastKnownBonded` is unconditionally hardcoded to `true` regardless of the destination validator's actual bonding status. If the destination validator is currently unbonded, the lazy bonus-replay logic will treat the entire gap between the redelegation and the validator's next BOND event as a bonded period, paying out bonus rewards for time the validator was not bonded.

---

### Finding Description

In `TierRedelegate`, after the redelegation succeeds, the position's bonus checkpoint is reset:

```go
// x/tieredrewards/keeper/msg_server.go
latestSeq, err := ms.getValidatorEventLatestSeq(ctx, dstValAddr)
...
pos.UpdateBonusCheckpoints(latestSeq, sdkCtx.BlockTime(), true)  // ← hardcoded true
``` [1](#0-0) 

`UpdateBonusCheckpoints` stores `LastKnownBonded = true` unconditionally:

```go
func (p *Position) UpdateBonusCheckpoints(lastEventSeq uint64, t time.Time, lastKnownBonded bool) {
    p.LastEventSeq = lastEventSeq
    p.LastBonusAccrual = t
    p.LastKnownBonded = lastKnownBonded
}
``` [2](#0-1) 

`LastEventSeq` is set to the destination validator's **current** latest event sequence. This means any prior UNBOND event for that validator (recorded at seq ≤ `latestSeq`) is permanently skipped — it will never be replayed for this position. [3](#0-2) 

When `processEventsAndClaimBonus` later runs, it initialises `bonded = pos.LastKnownBonded` (i.e., `true`) and walks only events with seq > `LastEventSeq`. If the destination validator was unbonded at redelegation time and later emits a BOND event, the code computes a bonus segment for the entire gap `[redelegation_time, bond_time]` with `bonded = true`:

```go
bonded := pos.LastKnownBonded   // true — incorrect if dst was unbonded
segmentStart := pos.LastBonusAccrual

for _, entry := range events {
    evt := entry.Event
    if bonded {
        bonus := k.computeSegmentBonus(*pos, tier, segmentStart, evt.Timestamp, evt.TokensPerShare)
        totalBonus = totalBonus.Add(bonus)   // ← paid for unbonded gap
    }
    ...
}
``` [4](#0-3) 

`validateRedelegatePosition` does **not** check whether the destination validator is bonded: [5](#0-4) 

The `redelegate` helper calls `BeginRedelegation` directly, which the Cosmos SDK permits to any validator regardless of bonding status: [6](#0-5) 

By contrast, `LockTier` and `CommitDelegationToTier` both call `createDelegatedPosition`, which also hardcodes `LastKnownBonded = true`, but those paths enforce `validator.IsBonded()` before delegating, making the assumption correct at creation time. `TierRedelegate` has no such guard for the destination. [7](#0-6) 

---

### Impact Explanation

An attacker can claim bonus rewards for a period during which their validator was unbonded. The bonus is paid from the `RewardsPoolName` module account. Repeated exploitation drains the pool, causing all subsequent legitimate bonus claims to fail atomically (`ErrInsufficientBonusPool`).

The corrupted value is the `RewardsPoolName` module account balance and the `totalBonus` computed per position — both are concrete on-chain balances. [8](#0-7) 

---

### Likelihood Explanation

The attack requires:
1. Holding a tier position on any bonded validator.
2. Redelegating to a currently-unbonded (e.g., jailed) validator — a normal Cosmos SDK operation that `validateRedelegatePosition` does not block.
3. Waiting for that validator to be unjailed and re-bond, which emits a BOND event.
4. Calling `MsgClaimTierRewards`.

Jailed validators are a routine chain event. The attacker controls the timing and needs no privileged access.

---

### Recommendation

Replace the hardcoded `true` with the destination validator's actual bonding status at the time of redelegation:

```go
dstVal, err := ms.stakingKeeper.GetValidator(ctx, dstValAddr)
if err != nil {
    return nil, err
}
pos.UpdateBonusCheckpoints(latestSeq, sdkCtx.BlockTime(), dstVal.IsBonded())
``` [1](#0-0) 

Additionally, `validateRedelegatePosition` should reject redelegation to a non-bonded destination validator, consistent with the invariant that tier positions only accrue bonus while their validator is bonded.

---

### Proof of Concept

1. Validator B is jailed (unbonded). Its latest event seq is `N` (the UNBOND event).
2. Attacker holds a tier position on bonded validator A.
3. Attacker sends `MsgTierRedelegate` with `dst_validator = B`.
   - `validateRedelegatePosition` passes (no bonding check on dst).
   - `BeginRedelegation` succeeds.
   - `latestSeq = N`, `LastKnownBonded = true` (incorrect — B is unbonded).
4. Validator B is unjailed. `AfterValidatorBonded` fires, appending a BOND event at seq `N+1`.
5. Attacker calls `MsgClaimTierRewards`.
   - `bonded = true` (from `LastKnownBonded`).
   - Processes BOND event at seq `N+1`: computes bonus for `[redelegation_time, bond_time]` with `bonded = true`.
   - Bonus is paid from the rewards pool for the entire unbonded gap — an overpayment. [9](#0-8)

```go
func (k Keeper) validateRedelegatePosition(ctx context.Context, pos types.PositionState, owner, dstValidator string) error {
	if !pos.IsOwner(owner) {
		return types.ErrNotPositionOwner
	}

	if !pos.IsDelegated() {
		return types.ErrPositionNotDelegated
	}

	if pos.Delegation.ValidatorAddress == dstValidator {
		return types.ErrRedelegationToSameValidator
	}

	sdkCtx := sdk.UnwrapSDKContext(ctx)
	if pos.HasTriggeredExit() && pos.CompletedExitLockDuration(sdkCtx.BlockTime()) {
		return types.ErrExitLockDurationElapsed
	}

	tier, err := k.getTier(ctx, pos.TierId)
	if err != nil {
		return err
	}

	if tier.IsCloseOnly() {
		return types.ErrTierIsCloseOnly
	}

	isRedelegating, err := k.isRedelegating(ctx, pos.DelegatorAddress)
	if err != nil {
		return err
	}
	if isRedelegating {
		return errorsmod.Wrapf(types.ErrActiveRedelegation, "position %d has an active redelegation", pos.Id)
	}

	return nil
}
