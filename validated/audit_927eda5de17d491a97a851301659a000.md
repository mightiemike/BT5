### Title
Permanently Unburnable NFT After Transfer — (`x/nft/keeper/keeper.go`)

### Summary

`Keeper.BurnNFT` requires the caller to simultaneously satisfy **both** `IsOwner` and `IsDenomCreator`. After the denom creator transfers an NFT to any third party, neither the new owner nor the original creator can ever satisfy both checks at once, making the NFT permanently unburnable and the denom's supply counter permanently inflated.

### Finding Description

`BurnNFT` enforces two sequential, conjunctive authorization checks: [1](#0-0) 

First it calls `IsOwner`, which returns `ErrUnauthorized` if the caller is not the **current** NFT owner: [2](#0-1) 

Then it calls `IsDenomCreator`, which returns `ErrUnauthorized` if the caller is not the **original denom creator**: [3](#0-2) 

`TransferOwner`, however, only checks `IsOwner` — it does **not** require the sender to be the denom creator: [4](#0-3) 

This creates a permanent deadlock after any transfer:

| Caller | `IsOwner` | `IsDenomCreator` | Result |
|---|---|---|---|
| B (new owner) | ✅ pass | ❌ fail | `ErrUnauthorized` |
| A (denom creator) | ❌ fail | ✅ pass | `ErrUnauthorized` |

The externally reachable path is `MsgBurnNFT → msgServer.BurnNFT → Keeper.BurnNFT`: [5](#0-4) 

### Impact Explanation

After A transfers an NFT to B, the token is permanently locked in existence. `k.decreaseSupply` is never reached, so the denom's supply counter never decrements for that token. The supply invariant is broken: the on-chain supply count permanently exceeds the number of actually burnable tokens. The NFT cannot be destroyed by any standard transaction path — no governance override or admin escape hatch exists in this module.

### Likelihood Explanation

This is triggered by the ordinary, fully supported `MsgTransferNFT` flow followed by `MsgBurnNFT`. No special privileges, governance, or key compromise are required. Any denom creator who mints and then transfers an NFT — a completely normal usage pattern — produces an unburnable token.

### Recommendation

Remove the `IsDenomCreator` check from `BurnNFT`. Ownership alone should be sufficient authorization to burn. The denom creator's privilege is already correctly scoped to minting (`MintNFT`) and editing (`EditNFT`). Burning is a destructive act that should be controlled by the current token owner, not the denom creator.

```go
func (k Keeper) BurnNFT(ctx sdk.Context, denomID, tokenID string, owner sdk.AccAddress) error {
    if !k.HasDenomID(ctx, denomID) {
        return sdkerrors.Wrapf(types.ErrInvalidDenom, "denom ID %s not exists", denomID)
    }

    nft, err := k.IsOwner(ctx, denomID, tokenID, owner)
    if err != nil {
        return err
    }

    // Remove the IsDenomCreator check — ownership is sufficient to burn.

    k.deleteNFT(ctx, denomID, nft)
    k.deleteOwner(ctx, denomID, tokenID, owner)
    k.decreaseSupply(ctx, denomID)

    return nil
}
```

### Proof of Concept

```go
// keeper_test.go (standard Go keeper unit test, no special setup)
func TestBurnNFTAfterTransfer(t *testing.T) {
    ctx, keeper := setupKeeper(t)

    addrA := sdk.AccAddress([]byte("addrA"))
    addrB := sdk.AccAddress([]byte("addrB"))

    // 1. A issues denom
    err := keeper.IssueDenom(ctx, "denom1", "Denom One", "", "", addrA)
    require.NoError(t, err)

    // 2. A mints NFT to A
    err = keeper.MintNFT(ctx, "denom1", "nft1", "NFT One", "", "", addrA, addrA)
    require.NoError(t, err)

    // 3. A transfers NFT to B
    err = keeper.TransferOwner(ctx, "denom1", "nft1", addrA, addrB)
    require.NoError(t, err)

    // 4. B tries to burn — IsOwner passes, IsDenomCreator fails
    err = keeper.BurnNFT(ctx, "denom1", "nft1", addrB)
    require.ErrorIs(t, err, types.ErrUnauthorized) // B is not denom creator

    // 5. A tries to burn — IsOwner fails (A no longer owns it)
    err = keeper.BurnNFT(ctx, "denom1", "nft1", addrA)
    require.ErrorIs(t, err, types.ErrUnauthorized) // A is not owner

    // 6. Supply is permanently stuck at 1
    supply := keeper.GetTotalSupply(ctx, "denom1")
    require.Equal(t, uint64(1), supply) // never decrements
}
```

func (k Keeper) IsOwner(ctx sdk.Context, denomID, tokenID string, owner sdk.AccAddress) (types.BaseNFT, error) {
	nft, err := k.GetNFT(ctx, denomID, tokenID)
	if err != nil {
		return types.BaseNFT{}, err
	}

	if !owner.Equals(nft.GetOwner()) {
		return types.BaseNFT{}, sdkerrors.Wrapf(types.ErrUnauthorized, "%s is not the owner of %s/%s", owner.String(), denomID, tokenID)
	}

	return nft.(types.BaseNFT), nil
}

func (k Keeper) IsDenomCreator(ctx sdk.Context, denomID string, address sdk.AccAddress) (types.Denom, error) {
	denom, err := k.GetDenom(ctx, denomID)
	if err != nil {
		return types.Denom{}, err
	}

	creator, err := sdk.AccAddressFromBech32(denom.Creator)
	if err != nil {
		panic(err)
	}

	if !creator.Equals(address) {
		return types.Denom{}, sdkerrors.Wrapf(types.ErrUnauthorized, "%s is not the creator of %s", address, denomID)
	}

	return denom, nil

func (m msgServer) BurnNFT(goCtx context.Context, msg *types.MsgBurnNFT) (*types.MsgBurnNFTResponse, error) {
	sender, err := sdk.AccAddressFromBech32(msg.Sender)
	if err != nil {
		return nil, err
	}

	ctx := sdk.UnwrapSDKContext(goCtx)
	if err := m.Keeper.BurnNFT(ctx, msg.DenomId, msg.Id, sender); err != nil {
		return nil, err
	}

	ctx.EventManager().EmitEvents(sdk.Events{
		sdk.NewEvent(
			types.EventTypeBurnNFT,
			sdk.NewAttribute(types.AttributeKeyDenomID, msg.DenomId),
			sdk.NewAttribute(types.AttributeKeyTokenID, msg.Id),
			sdk.NewAttribute(types.AttributeKeyOwner, msg.Sender),
		),
		sdk.NewEvent(
			sdk.EventTypeMessage,
			sdk.NewAttribute(sdk.AttributeKeyModule, types.AttributeValueCategory),
			sdk.NewAttribute(sdk.AttributeKeySender, msg.Sender),
		),
	})

	return &types.MsgBurnNFTResponse{}, nil
}
