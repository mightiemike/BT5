### Title
Divide-by-Zero in `PerpEngine.socializeSubaccount` When `openInterest == 0` Permanently Blocks Insolvent Subaccount Finalization — (`core/contracts/PerpEngine.sol`)

---

### Summary

`PerpEngine.socializeSubaccount` unconditionally divides by `state.openInterest` when socializing residual bad debt. No guard exists for the case where `openInterest == 0`. `MathSD21x18.div` explicitly reverts on a zero denominator. This causes `ClearinghouseLiq._finalizeSubaccount` to permanently revert, leaving bad debt unresolved and the insurance fund locked.

---

### Finding Description

In `PerpEngine.socializeSubaccount`, after insurance coverage is exhausted, the code attempts to spread the remaining loss across all open-interest holders:

```solidity
// core/contracts/PerpEngine.sol lines 164–168
if (balance.vQuoteBalance < 0) {
    int128 fundingPerShare = -balance.vQuoteBalance.div(
        state.openInterest   // <-- reverts if == 0
    );
``` [1](#0-0) 

`MathSD21x18.div` enforces a hard zero-check:

```solidity
// core/contracts/libraries/MathSD21x18.sol line 64
require(y != 0, ERR_DIV_BY_ZERO);
``` [2](#0-1) 

`state.openInterest` is the sum of absolute position sizes across all participants. It is decremented to zero whenever all participants close their positions:

```solidity
// core/contracts/PerpEngineState.sol lines 30, 46–50
state.openInterest -= balance.amount.abs();   // pre-update: remove old contribution
// ...
state.openInterest += balance.amount;         // post-update: add new contribution (0 if closed)
``` [3](#0-2) 

`vQuoteBalance` is independent of `openInterest`. A subaccount can have `balance.amount == 0` (position closed) while retaining a negative `vQuoteBalance` from prior trading losses. If all other participants have also closed their positions, `openInterest == 0` while the insolvent subaccount still carries negative `vQuoteBalance`.

`_finalizeSubaccount` requires `balance.amount == 0` for all perp products before calling `socializeSubaccount`:

```solidity
// core/contracts/ClearinghouseLiq.sol lines 319, 386–389
require(balance.amount == 0, ERR_NOT_FINALIZABLE_SUBACCOUNT);
// ...
v.insurance = perpEngine.socializeSubaccount(txn.liquidatee, v.insurance);
``` [4](#0-3) [5](#0-4) 

The protocol itself already handles `openInterest == 0` safely in `updateStates` (funding rate computation skips the product), confirming this is a known possible state — but `socializeSubaccount` has no equivalent guard. [6](#0-5) 

---

### Impact Explanation

- `_finalizeSubaccount` reverts every time it is called for this subaccount.
- The insolvent subaccount can never be finalized.
- Bad debt is permanently unresolved.
- The insurance fund balance is locked: `insurance` is only updated inside `_finalizeSubaccount` after `socializeSubaccount` returns, which it never does.
- This is a permanent, irrecoverable protocol accounting failure matching the Critical scope: solvency/accounting failure in liquidation/insurance handling that leaves the protocol undercollateralized.

---

### Likelihood Explanation

The precondition is realistic and requires no privileged access:

1. A single trader opens a leveraged perp position and accumulates negative PnL (`vQuoteBalance < 0`).
2. The trader closes the position (`balance.amount = 0`), but `vQuoteBalance` remains negative.
3. All other participants in that product also close their positions, driving `openInterest` to 0.
4. The trader's account becomes insolvent (health check fails due to negative `vQuoteBalance` with insufficient collateral).
5. Any liquidator calls `LiquidateSubaccount` with `productId = type(uint32).max` to finalize.
6. The call reverts with `"DBZ"` every time.

This can also be triggered deliberately by a sophisticated attacker who opens a position against a victim, then closes their own position to zero out `openInterest` before the victim is liquidated.

---

### Recommendation

Add a zero-check guard before the division in `socializeSubaccount`. If `openInterest == 0`, there are no other participants to socialize the loss against; the bad debt should be absorbed entirely by the insurance fund or written off:

```solidity
if (balance.vQuoteBalance < 0) {
    if (state.openInterest == 0) {
        // No open interest to socialize against; write off remaining bad debt
        balance.vQuoteBalance = 0;
    } else {
        int128 fundingPerShare = -balance.vQuoteBalance.div(state.openInterest);
        state.cumulativeFundingLongX18 += fundingPerShare;
        state.cumulativeFundingShortX18 -= fundingPerShare;
        balance.vQuoteBalance = 0;
    }
}
```

### Proof of Concept

```solidity
// Hardhat test (pseudocode)
it("divide-by-zero in socializeSubaccount when openInterest == 0", async () => {
    // 1. Alice opens a long perp position (openInterest > 0)
    await endpoint.submitOrder(aliceLongOrder);

    // 2. Price moves against Alice; vQuoteBalance becomes negative
    await endpoint.updateStates(...);

    // 3. Alice closes her position (amount = 0, vQuoteBalance still < 0)
    await endpoint.submitOrder(aliceCloseOrder);
    // openInterest is now 0 (Alice was the only participant)

    // 4. Alice's account is insolvent (negative vQuoteBalance, insufficient collateral)
    // Health check confirms unhealthy

    // 5. Liquidator attempts to finalize Alice's subaccount
    await expect(
        endpoint.liquidateSubaccount({
            liquidatee: alice,
            productId: ethers.constants.MaxUint32, // finalize
            ...
        })
    ).to.be.revertedWith("DBZ"); // MathSD21x18 divide-by-zero
});
```
