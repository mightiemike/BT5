### Title
NFT Permanently Unburnable via Native Path After Transfer — (`x/nft/keeper/keeper.go`)

### Summary
`BurnNFT` requires the caller to simultaneously satisfy both `IsOwner` and `IsDenomCreator`. After any `MsgTransferNFT`, ownership moves to the buyer while the denom creator field is immutable. No single address can satisfy both checks, making the NFT permanently unburnable via `MsgBurnNFT`.

### Finding Description
`Keeper.BurnNFT` enforces two sequential guards: [5](#0-4) 

`IsOwner` checks `nft.Owner == caller`; `IsDenomCreator` checks `denom.Creator == caller`. After `TransferOwner` updates `nft.Owner` to the buyer: [6](#0-5) 

- Buyer: passes `IsOwner`, fails `IsDenomCreator` → `ErrUnauthorized`
- Creator: fails `IsOwner` immediately → `ErrUnauthorized`

The denom creator field is set once at `IssueDenom` and never updated: [7](#0-6) 

### Impact Explanation
Any NFT that has been transferred at least once is permanently unburnable via the native `MsgBurnNFT` transaction path. The NFT supply counter (`decreaseSupply`) is never called for such tokens via the native path. The only escape is `BurnNFTUnverified`, which is called exclusively by the IBC NFT-transfer module and is not directly accessible to users as a standalone transaction: [4](#0-3) 

The "supply inflation" framing in the question is imprecise — the supply count already reflects the existing NFT; the actual impact is **permanent loss of burn capability** for transferred NFTs, breaking the expected invariant that NFT owners can destroy their own tokens.

### Likelihood Explanation
Triggered by any ordinary `MsgTransferNFT` followed by `MsgBurnNFT`. No special privileges, governance, or IBC required. Affects every NFT denom on the chain where transfers have occurred.

### Recommendation
`BurnNFT` should require `IsOwner` OR `IsDenomCreator`, not both. The current AND-logic is inconsistent with `TransferOwner` (which only checks `IsOwner`) and `MintNFT` (which only checks `IsDenomCreator`). A corrected guard would allow either the current owner or the denom creator to burn:

```go
nft, ownerErr := k.IsOwner(ctx, denomID, tokenID, owner)
if ownerErr != nil {
    _, creatorErr := k.IsDenomCreator(ctx, denomID, owner)
    if creatorErr != nil {
        return ownerErr // or a combined error
    }
    // fetch nft directly if not owner but is creator
    ...
}
```

### Proof of Concept
1. Creator issues denom, mints NFT to themselves.
2. Creator calls `MsgTransferNFT` → buyer becomes owner.
3. Buyer calls `MsgBurnNFT` → fails `IsDenomCreator` at line 151.
4. Creator calls `MsgBurnNFT` → fails `IsOwner` at line 146.
5. Both return `ErrUnauthorized`; NFT is permanently locked in supply. [1](#0-0)

func (k Keeper) IsDenomCreator(ctx sdk.Context, denomID string, address sdk.AccAddress) (types.Denom, error) {
	denom, err := k.GetDenom(ctx, denomID)
	if err != nil {
		return types.Denom{}, err
	}

	creator, err := sdk.AccAddressFromBech32(denom.Creator)
	if err != nil {
		panic(err)
	}

	if !creator.Equals(address) {
		return types.Denom{}, sdkerrors.Wrapf(types.ErrUnauthorized, "%s is not the creator of %s", address, denomID)
	}

	return denom, nil
