Audit Report

## Title
SwapAllowlistExtension Checks the Direct Pool Caller (Router) Instead of the End User, Allowing Allowlist Bypass — (File: `metric-periphery/contracts/extensions/SwapAllowlistExtension.sol`)

## Summary

`SwapAllowlistExtension.beforeSwap` gates swaps on `sender`, which is the direct caller of `pool.swap()` (e.g., `MetricOmmSimpleRouter`), not the end user who initiated the transaction. When a pool admin allowlists the router to enable standard-periphery access, the check `allowedSwapper[pool][router]` returns `true` for every caller of the router, unconditionally bypassing the per-user restriction the allowlist was configured to enforce.

## Finding Description

In `MetricOmmPool.swap`, the pool invokes `_beforeSwap` with `msg.sender` as the `sender` argument:

```solidity
// metric-core/contracts/MetricOmmPool.sol L230-240
_beforeSwap(
  msg.sender,   // ← direct pool caller, not the end user
  recipient,
  ...
);
```

`SwapAllowlistExtension.beforeSwap` then checks whether that `sender` is on the allowlist keyed by pool (`msg.sender` inside the extension is the pool):

```solidity
// metric-periphery/contracts/extensions/SwapAllowlistExtension.sol L37
if (!allowAllSwappers[msg.sender] && !allowedSwapper[msg.sender][sender]) {
  revert IMetricOmmPoolActions.NotAllowedToSwap();
}
```

When a user calls `MetricOmmSimpleRouter.exactInputSingle`, the router calls `pool.swap(params.recipient, ...)` directly:

```solidity
// metric-periphery/contracts/MetricOmmSimpleRouter.sol L72-80
(int128 amount0Delta, int128 amount1Delta) = IMetricOmmPoolActions(params.pool)
  .swap(
    params.recipient,
    params.zeroForOne,
    ...
  );
```

The pool sees `msg.sender = router`, so the extension receives `sender = router`. If the pool admin has allowlisted the router (`allowedSwapper[pool][router] = true`), the check passes for **every caller of the router**, regardless of whether the actual end user is authorized. The `recipient` parameter (second, unnamed argument in `beforeSwap`) — the address that actually receives output tokens — is never checked.

The Smart Audit Pivot for the allowlist path explicitly requires: "deposit/swap allowlist checks must cover the exact actor/action intended and cannot be bypassed through router." This check fails that requirement.

## Impact Explanation

The swap allowlist access control is completely nullified for any pool that allowlists the router. Any unprivileged user can swap in a restricted pool by calling `MetricOmmSimpleRouter.exactInputSingle` (or `exactInput`, `exactOutputSingle`, `exactOutput`). The pool receives and sends real tokens; the unauthorized caller receives the output. This is a direct, fund-impacting breach of the access-control invariant the pool admin configured — unauthorized parties interact with the pool's liquidity and receive output tokens.

## Likelihood Explanation

High. `MetricOmmSimpleRouter` is the standard swap periphery. A pool admin who wants end users to swap through the standard interface must allowlist the router. Once the router is allowlisted, the restriction is unconditionally bypassed for all callers. No special privilege or setup is required from the attacker beyond calling the public router functions. The bypass is repeatable and deterministic.

## Recommendation

Check the **end user** rather than the direct pool caller. Two complementary fixes:

1. **In `SwapAllowlistExtension.beforeSwap`**: check the `recipient` (the second parameter, currently unnamed/ignored) instead of — or in addition to — `sender`. The recipient is the address that benefits from the swap output and is the actor the allowlist should gate.

2. **Alternatively**, require that every end user be individually allowlisted and have the router pass the end user's address as `sender` (which would require a protocol-level change to how `sender` is forwarded through the pool — e.g., passing `tx.origin` or an explicit `payer` field through `extensionData`).

## Proof of Concept

```
Setup:
  pool configured with SwapAllowlistExtension
  allowedSwapper[pool][router] = true    ← admin allowlists the router
  allowedSwapper[pool][attacker] = false ← attacker is NOT allowlisted

Attack:
  attacker calls:
    MetricOmmSimpleRouter.exactInputSingle({
      pool:      pool,
      recipient: attacker,
      zeroForOne: true,
      amountIn:  X,
      ...
    })

  Router calls:
    pool.swap(recipient=attacker, zeroForOne=true, amountSpecified=X, ...)
    → msg.sender to pool = router

  Pool calls _beforeSwap(sender=router, recipient=attacker, ...)
  Extension checks:
    allowedSwapper[pool][router] == true  ✓ passes

  Swap executes; attacker receives output tokens.
  Allowlist is bypassed with zero privilege.
```

Foundry test plan: deploy pool with `SwapAllowlistExtension`, set `allowedSwapper[pool][router] = true`, set `allowedSwapper[pool][attacker] = false`, call `exactInputSingle` from `attacker` address, assert swap succeeds and attacker receives output tokens.
