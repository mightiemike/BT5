### Title
Bonus Rewards Permanently Lost When Pool Is Insufficient During Partial Redelegation Slash ‚Äî (`x/tieredrewards/keeper/slash.go`)

---

### Summary

In `slashRedelegationPosition`, when `processEventsAndClaimBonus` returns `ErrInsufficientBonusPool`, the error is silently swallowed. However, by the time the error is returned, `processEventsAndClaimBonus` has already advanced the position's `LastBonusAccrual`, `LastKnownBonded`, and `LastEventSeq` checkpoints in memory **and** decremented validator event reference counts in the store. The caller then persists the position with these advanced checkpoints via `setPositionWithState`, making the accrued bonus permanently unrecoverable ‚Äî even after the pool is later replenished.

---

### Finding Description

`BeforeRedelegationSlashed` fires inside the staking module's `SlashRedelegation` path and calls `slashRedelegationPosition`: [1](#0-0) 

The call to `processEventsAndClaimBonus` at line 54 performs the following **before** the pool check:

1. **Decrements event reference counts in the store** (line 196 of `claim_rewards.go`) ‚Äî these are direct store writes inside the event loop, not in-memory: [2](#0-1) 

2. **Advances `LastBonusAccrual` and `LastKnownBonded` in memory** (lines 215‚Äì217): [3](#0-2) 

3. **Only then** checks the pool balance (line 230), which fails and returns `ErrInsufficientBonusPool`: [4](#0-3) 

Back in `slashRedelegationPosition`, the `ErrInsufficientBonusPool` is caught and ignored. For a partial slash, execution falls through to `setPositionWithState`, which **persists the position with the already-advanced checkpoints**: [5](#0-4) 

The result is:
- `pos.LastBonusAccrual` is now set to the slash block time.
- `pos.LastEventSeq` is advanced past all processed events.
- Validator event reference counts are decremented (events may be garbage-collected).
- The bonus coins are **never sent** to the owner.

Any subsequent call to `ClaimTierRewards` will compute bonus only from `LastBonusAccrual` (the slash block time) forward, permanently skipping the accrual period before the slash.

---

### Impact Explanation

Accrued bonus rewards for a tier position are permanently forfeited when:
1. The position is in a redelegating state (active `RedelegationMapping` entry exists).
2. The destination validator is partially slashed.
3. The `RewardsPoolName` module account has insufficient balance to cover the accrued bonus at the moment of the slash.

Even after the pool is replenished by governance, the affected position can never recover the lost bonus because the accounting checkpoints have been irrevocably advanced and the validator events have been decremented/garbage-collected. The corrupted value is `pos.LastBonusAccrual` and the associated event reference counts in `ValidatorEvents`.

---

### Likelihood Explanation

The scenario is realistic:
- Tier positions are routinely redelegated (`MsgTierRedelegate` is a standard user operation).
- Validator slashing is a normal chain event (double-sign, downtime).
- The bonus pool can be depleted by concurrent legitimate claims from other users, or simply not yet replenished after a prior drain.

No privileged access is required. The trigger is a combination of two unprivileged/normal chain events (user redelegate + validator slash) coinciding with a pool state that can arise organically.

---

### Recommendation

In `slashRedelegationPosition`, when `ErrInsufficientBonusPool` is caught, the position's in-memory checkpoints should be **reset to their pre-call values** before calling `setPositionWithState`, so that the accrual period is not skipped. Alternatively, the event reference count decrements should be deferred until after the pool check succeeds. The simplest fix is to snapshot `pos` before calling `processEventsAndClaimBonus` and restore the snapshot on `ErrInsufficientBonusPool`:

```go
posSnapshot := pos  // snapshot before call
if _, err := k.processEventsAndClaimBonus(ctx, &pos); err != nil {
    if errors.Is(err, types.ErrInsufficientBonusPool) {
        pos = posSnapshot  // restore checkpoints; events not yet GC'd
        k.logger(ctx).Error(...)
    } else {
        return err
    }
}
```

This requires that `decrementEventRefCount` calls be moved to after the pool check, or that the event loop be restructured to separate computation from side effects.

func (k Keeper) slashRedelegationPosition(ctx context.Context, unbondingId uint64, sharesToUnbond math.LegacyDec) error {
	positionId, err := k.getRedelegationMapping(ctx, unbondingId)
	if errors.Is(err, collections.ErrNotFound) {
		return nil
	}
	if err != nil {
		return err
	}

	pos, err := k.getPositionState(ctx, positionId)
	if errors.Is(err, types.ErrPositionNotFound) {
		k.logger(ctx).Error("position not found during redelegation slash",
			"position_id", positionId,
			"unbonding_id", unbondingId,
			"error", err.Error(),
		)
		return nil
	}
	if err != nil {
		return err
	}

	if !pos.IsDelegated() {
		// Defensive
		k.logger(ctx).Error("delegation missing during BeforeRedelegationSlashed",
			"position_id", positionId,
			"unbonding_id", unbondingId,
			"shares_to_unbond", sharesToUnbond.String(),
		)
		return nil
	}

	dstValStr := pos.Delegation.ValidatorAddress

	// Settle bonus against PRE-slash shares.
	if _, err := k.processEventsAndClaimBonus(ctx, &pos); err != nil {
		// Deliberately forgo bonus rewards if pool is insufficient to prevent chain halt.
		if errors.Is(err, types.ErrInsufficientBonusPool) {
			k.logger(ctx).Error("insufficient bonus pool during redelegation slash",
				"position_id", pos.Id,
				"error", err.Error(),
			)
		} else {
			return err
		}
	}

	fullSlash := sharesToUnbond.GTE(pos.Delegation.Shares)

	if fullSlash {
		pos.Delegation = nil
		pos.ClearBonusCheckpoints()
		return k.setPositionWithState(ctx, pos, &ValidatorTransition{PreviousAddress: dstValStr})
	}
	// In-memory only: the persisted Position carries no share count, and the
	// live delegation will reflect the post-Unbond shares on the next read.
	// Update the local copy so any follow-up logic in this call sees consistent state.
	pos.Delegation.Shares = pos.Delegation.Shares.Sub(sharesToUnbond)
	return k.setPositionWithState(ctx, pos, nil)

func (k Keeper) processEventsAndClaimBonus(ctx context.Context, pos *types.PositionState) (sdk.Coins, error) {
	// Rewards should have been claimed before undelegation
	if !pos.IsDelegated() {
		return sdk.NewCoins(), nil
	}

	valAddr, err := sdk.ValAddressFromBech32(pos.Delegation.ValidatorAddress)
	if err != nil {
		return nil, err
	}

	events, err := k.getValidatorEventsSince(ctx, valAddr, pos.LastEventSeq)
	if err != nil {
		return nil, err
	}

	sdkCtx := sdk.UnwrapSDKContext(ctx)
	blockTime := sdkCtx.BlockTime()

	totalBonus := math.ZeroInt()
	// Use the persisted bonded state from the last replay, not a hardcoded default.
	// This prevents overpaying bonus for unbonded gaps between claims.
	bonded := pos.LastKnownBonded
	segmentStart := pos.LastBonusAccrual

	tier, err := k.getTier(ctx, pos.TierId)
	if err != nil {
		return nil, err
	}

	for _, entry := range events {
		evt := entry.Event

		if bonded {
			// Compute bonus for the bonded segment [segmentStart, eventTime]
			// using the snapshot rate at the event.
			bonus := k.computeSegmentBonus(*pos, tier, segmentStart, evt.Timestamp, evt.TokensPerShare)
			totalBonus = totalBonus.Add(bonus)
		}

		// Update bonded state based on event type.
		switch evt.EventType {
		case types.ValidatorEventType_VALIDATOR_EVENT_TYPE_UNBOND:
			bonded = false
		case types.ValidatorEventType_VALIDATOR_EVENT_TYPE_BOND:
			bonded = true
		case types.ValidatorEventType_VALIDATOR_EVENT_TYPE_SLASH:
			// Slash doesn't change bonded state.
		}

		segmentStart = evt.Timestamp
		pos.UpdateLastEventSeq(entry.Seq)

		// Decrement reference count.
		if err := k.decrementEventRefCount(ctx, valAddr, entry.Seq); err != nil {
			return nil, err
		}
	}

	val, err := k.stakingKeeper.GetValidator(ctx, valAddr)
	if err != nil {
		return nil, err
	}
	// Defensive: validator bond status check
	if bonded && val.IsBonded() {
		currentRate, err := k.getTokensPerShare(ctx, valAddr)
		if err != nil {
			return nil, err
		}
		bonus := k.computeSegmentBonus(*pos, tier, segmentStart, blockTime, currentRate)
		totalBonus = totalBonus.Add(bonus)
	}

	applyBonusAccrualCheckpoint(&pos.Position, blockTime)
	// Persist the bonded state so the next replay starts correctly.
	pos.UpdateLastKnownBonded(bonded)

	if totalBonus.IsZero() {
		return sdk.NewCoins(), nil
	}

	bondDenom, err := k.stakingKeeper.BondDenom(ctx)
	if err != nil {
		return nil, err
	}

	bonusCoins := sdk.NewCoins(sdk.NewCoin(bondDenom, totalBonus))

	if err := k.sufficientBonusPoolBalance(ctx, bonusCoins); err != nil {
		return nil, err
	}
