### Title
`MintNlp` Oracle Price Is Unsigned — User Has No Slippage Protection When Minting NLP Tokens - (File: `core/contracts/Verifier.sol`, `core/contracts/Clearinghouse.sol`)

---

### Summary

The EIP-712 digest for `MintNlp` covers only `sender`, `quoteAmount`, and `nonce`. The `oraclePriceX18` field — which directly determines how many NLP tokens the user receives — is appended by the sequencer and is never committed to by the user's signature. There is no minimum NLP amount check anywhere in the execution path. A user who signs a `MintNlp` transaction has no on-chain protection against receiving far fewer NLP tokens than they expected if the oracle price moves between signing and sequencer execution.

---

### Finding Description

`SignedMintNlp` contains two fields beyond the user-signed `MintNlp` struct:

```solidity
struct SignedMintNlp {
    MintNlp tx;           // signed: sender, quoteAmount, nonce
    bytes signature;
    int128 oraclePriceX18;        // NOT signed
    int128[] nlpPoolRebalanceX18; // NOT signed
}
``` [1](#0-0) 

The EIP-712 type string and digest computation confirm that `oraclePriceX18` is excluded from the user's commitment:

```solidity
string internal constant MINT_NLP_SIGNATURE =
    "MintNlp(bytes32 sender,uint128 quoteAmount,uint64 nonce)";
``` [2](#0-1) 

```solidity
digest = keccak256(abi.encode(
    keccak256(bytes(MINT_NLP_SIGNATURE)),
    signedTx.tx.sender,
    signedTx.tx.quoteAmount,
    signedTx.tx.nonce
    // oraclePriceX18 is absent
));
``` [3](#0-2) 

In `Clearinghouse.mintNlp`, the NLP amount is computed as:

```solidity
int128 nlpAmount = quoteAmount.div(oraclePriceX18);
``` [4](#0-3) 

There is no floor check, no `minNlpAmount` parameter, and no assertion that the resulting `nlpAmount` meets any user-specified threshold. The full `quoteAmount` is always debited from the user regardless of what `nlpAmount` they receive. [5](#0-4) 

---

### Impact Explanation

A user signs a `MintNlp` transaction committing to spend `quoteAmount` of quote tokens. The NLP tokens they receive are `quoteAmount / oraclePriceX18`. Because `oraclePriceX18` is not bound by the signature, the user cannot enforce a minimum NLP output. If the NLP oracle price rises between signing and sequencer execution, the user's full `quoteAmount` is consumed but they receive proportionally fewer NLP tokens — a direct, unrecoverable asset loss with no on-chain protection. The corrupted state delta is: `quoteAmount` fully debited, `nlpAmount` silently reduced below the user's expectation.

---

### Likelihood Explanation

Any user minting NLP tokens during a period of NLP price volatility is exposed. The sequencer processes transactions in batches; the oracle price used at execution time is whatever `oraclePriceX18` the sequencer supplies in the `SignedMintNlp` envelope. Even an honest sequencer using the current market price at batch-submission time will produce a different price than what the user saw when they signed. No user action can prevent this — the signed transaction contains no price bound.

---

### Recommendation

Add a `minNlpAmount` field to the `MintNlp` struct and include it in the EIP-712 type string and digest:

```solidity
string internal constant MINT_NLP_SIGNATURE =
    "MintNlp(bytes32 sender,uint128 quoteAmount,uint128 minNlpAmount,uint64 nonce)";
```

In `Clearinghouse.mintNlp`, assert:

```solidity
require(nlpAmount >= int128(txn.minNlpAmount), ERR_SLIPPAGE_EXCEEDED);
```

This mirrors the `amountAMin`/`amountBMin` pattern in UniswapV2Router02 and gives users a signed, enforceable slippage bound.

---

### Proof of Concept

1. NLP oracle price is 1.00 USDC/NLP. User signs `MintNlp { quoteAmount: 10_000e18, nonce: N }` expecting ~10,000 NLP tokens.
2. Before the sequencer batches the transaction, NLP price rises to 1.10 USDC/NLP.
3. Sequencer submits `SignedMintNlp` with `oraclePriceX18 = 1.10e18`.
4. `mintNlp` executes: `nlpAmount = 10_000e18 / 1.10e18 ≈ 9,090 NLP`.
5. User's full `10,000 USDC` is debited; they receive only `~9,090 NLP` — a ~9% value loss with no revert and no recourse.
6. The user had no way to express "revert if I receive fewer than 9,900 NLP" because `oraclePriceX18` is outside the signed envelope. [6](#0-5) [7](#0-6)
