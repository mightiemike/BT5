### Title
Unprivileged Caller Can Trigger `run_mainchain_gc` to Prune Canonical Chain State — (`contract/src/lib.rs`)

### Summary
`run_mainchain_gc` is a state-mutating function that permanently deletes mainchain block headers from contract storage. It carries only a `#[pause]` guard, which restricts calls only when the contract is paused. When the contract is running normally, **any unprivileged NEAR account** can call it with an arbitrary `batch_size`, triggering aggressive pruning of the oldest mainchain blocks. This is a direct analog to the reported pattern: a privileged state-mutation is reachable without any caller restriction.

### Finding Description
`run_mainchain_gc` is decorated with `#[pause(except(roles(Role::UnrestrictedRunGC)))]`: [1](#0-0) 

The `#[pause]` attribute from `near-plugins` only blocks callers when the contract is in a paused state; the `except(roles(...))` clause merely allows privileged accounts to bypass that pause. When the contract is **not** paused — the normal production state — there is no caller check at all. Any NEAR account can call:

```
run_mainchain_gc(batch_size: u64)
```

with `batch_size = u64::MAX` and the function will remove up to `amount_of_headers_we_store - gc_threshold` of the oldest mainchain block headers in a single transaction, updating `mainchain_initial_blockhash` to the new oldest surviving block. [2](#0-1) 

By contrast, `submit_blocks` — the only other path that calls `run_mainchain_gc` — is protected by `#[trusted_relayer]`, which enforces that the caller is a registered, active relayer: [3](#0-2) 

The test suite confirms this asymmetry: unauthorized accounts are explicitly rejected from `submit_blocks` with "Relayer is not active", but no equivalent test or guard exists for direct calls to `run_mainchain_gc`. [4](#0-3) 

### Impact Explanation
The GC function permanently removes entries from three pieces of canonical state:

- `headers_pool` (the `ExtendedHeader` record)
- `mainchain_height_to_header` (height → hash index)
- `mainchain_initial_blockhash` (the oldest-block pointer) [5](#0-4) 

Two concrete harms follow:

1. **SPV proof invalidation.** `verify_transaction_inclusion` and `verify_transaction_inclusion_v2` look up the target block in `mainchain_header_to_height`. If an attacker prunes that block before a downstream contract or user submits their proof, the call panics with "block does not belong to the current main chain", permanently invalidating a legitimate proof. [6](#0-5) 

2. **Chain reorganization failure.** The CLAUDE.md explicitly documents: "If mainchain blocks near the fork point have been garbage collected, reorg will fail — the contract panics with `PrevBlockNotFound`." An attacker who observes a competing fork building up can call `run_mainchain_gc` at the moment the common ancestor becomes eligible for removal, ensuring the legitimate reorg can never complete and locking the contract onto the attacker-preferred chain tip. [7](#0-6) 

### Likelihood Explanation
The precondition is that the mainchain has grown beyond `gc_threshold` (the recommended value is 52,704 blocks — roughly one year). On a live deployment tracking Bitcoin mainnet, this threshold is crossed in normal operation. Once crossed, the attack requires a single permissionless NEAR transaction with no deposit. The attacker needs no keys, no role, and no prior interaction with the contract.

### Recommendation
Add a caller restriction to `run_mainchain_gc` equivalent to the one on `submit_blocks`. The simplest fix is to require the caller to hold the `UnrestrictedSubmitBlocks` or a dedicated `GCManager` role, or to apply `#[trusted_relayer]` to the public entry point. Alternatively, make the public `run_mainchain_gc` a view-only trigger that internally enforces `env::predecessor_account_id()` is a registered relayer before mutating state.

### Proof of Concept
1. Deploy the contract with `gc_threshold = 52704` and submit enough Bitcoin headers to exceed the threshold (or use a low threshold in a test environment).
2. From any unprivileged NEAR account (no role granted), call:
   ```
   near call <contract_id> run_mainchain_gc '{"batch_size": 18446744073709551615}' --accountId attacker.near
   ```
3. Observe that the call succeeds, `mainchain_initial_blockhash` advances, and the pruned block hashes are no longer present in `mainchain_height_to_header`.
4. Attempt `verify_transaction_inclusion` for any pruned block — the call panics with "block does not belong to the current main chain", confirming the SPV proof is permanently invalidated. [8](#0-7)

```rust
    #[pause(except(roles(Role::UnrestrictedRunGC)))]
    pub fn run_mainchain_gc(&mut self, batch_size: u64) {
        let initial_blockheader = self
            .headers_pool
            .get(&self.mainchain_initial_blockhash)
            .unwrap_or_else(|| env::panic_str(ERR_KEY_NOT_EXIST));

        let tip_blockheader = self
            .headers_pool
            .get(&self.mainchain_tip_blockhash)
            .unwrap_or_else(|| env::panic_str(ERR_KEY_NOT_EXIST));

        let amount_of_headers_we_store =
            tip_blockheader.block_height - initial_blockheader.block_height + 1;

        if amount_of_headers_we_store > self.gc_threshold {
            let total_amount_to_remove = amount_of_headers_we_store - self.gc_threshold;
            let selected_amount_to_remove = std::cmp::min(total_amount_to_remove, batch_size);

            let start_removal_height = initial_blockheader.block_height;
            let end_removal_height = initial_blockheader.block_height + selected_amount_to_remove;
            env::log_str(&format!(
                "Num of blocks to remove {selected_amount_to_remove}"
            ));

            for height in start_removal_height..end_removal_height {
                let blockhash = &self
                    .mainchain_height_to_header
                    .get(&height)
                    .unwrap_or_else(|| env::panic_str(ERR_KEY_NOT_EXIST));

                self.remove_block_header(blockhash);
                self.mainchain_height_to_header.remove(&height);
            }

            self.mainchain_initial_blockhash = self
                .mainchain_height_to_header
                .get(&end_removal_height)
                .unwrap_or_else(|| env::panic_str(ERR_KEY_NOT_EXIST));
        }
    }
```
