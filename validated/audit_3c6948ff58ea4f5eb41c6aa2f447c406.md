### Title
`mintNlp()` and `burnNlp()` Lack User-Specified Minimum Output (Slippage) Check — (`File: core/contracts/Clearinghouse.sol`)

---

### Summary

`mintNlp()` and `burnNlp()` in `Clearinghouse.sol` execute NLP token conversions at a sequencer-supplied `oraclePriceX18` that is **not part of the user's signed payload**. Neither the `MintNlp` nor `BurnNlp` inner structs contain a `minNlpAmount` or `minQuoteAmount` field, so users have no on-chain protection against receiving an unfavorable output. Minted NLP is subject to a 4-day lock (`NLP_LOCK_PERIOD`), compounding the harm.

---

### Finding Description

The user-signed structs are:

```solidity
struct MintNlp {
    bytes32 sender;
    uint128 quoteAmount;
    uint64 nonce;
}

struct BurnNlp {
    bytes32 sender;
    uint128 nlpAmount;
    uint64 nonce;
}
```

Neither contains a minimum acceptable output field. The outer `SignedMintNlp` / `SignedBurnNlp` structs carry `oraclePriceX18` and `nlpPoolRebalanceX18` **outside** the inner signed struct — these are sequencer-appended execution parameters, not user commitments. [1](#0-0) 

At execution time, `mintNlp` computes:

```solidity
int128 nlpAmount = quoteAmount.div(oraclePriceX18);
``` [2](#0-1) 

And `burnNlp` computes:

```solidity
int128 quoteAmount = nlpAmount.mul(oraclePriceX18);
int128 burnFee = MathHelper.max(ONE, quoteAmount / 1000);
quoteAmount = MathHelper.max(0, quoteAmount - burnFee);
``` [3](#0-2) 

The sequencer supplies `oraclePriceX18` at submission time via `EndpointTx.sol`:

```solidity
priceX18[NLP_PRODUCT_ID] = signedTx.oraclePriceX18;
clearinghouse.mintNlp(signedTx.tx, signedTx.oraclePriceX18, ...);
``` [4](#0-3) 

There is no on-chain check that `nlpAmount >= userMinNlpAmount` (for mint) or `quoteAmount >= userMinQuoteAmount` (for burn). The user has zero recourse if the price used at execution differs from what they anticipated.

Minted NLP is locked for `NLP_LOCK_PERIOD = 4 days` before it can be burned:

```solidity
require(
    spotEngine.getNlpUnlockedBalance(txn.sender).amount >= nlpAmount,
    ERR_UNLOCKED_NLP_INSUFFICIENT
);
``` [5](#0-4) [6](#0-5) 

---

### Impact Explanation

**Impact: Medium.**

- For `mintNlp`: a user spending `quoteAmount` receives `quoteAmount / oraclePriceX18` NLP. If `oraclePriceX18` is higher than the user expected (NLP price rose between signing and execution), the user receives materially fewer NLP tokens and cannot exit for 4 days.
- For `burnNlp`: a user burning `nlpAmount` receives `nlpAmount * oraclePriceX18 - burnFee` quote. If `oraclePriceX18` is lower than expected (NLP price fell), the user receives materially less quote with no recourse.
- In both cases the user's balance is directly reduced relative to their expectation, with no on-chain protection.

---

### Likelihood Explanation

**Likelihood: Medium.**

NLP price (`oraclePriceX18`) is a live oracle value that fluctuates with protocol PnL and pool composition. There is always a gap between when a user signs a transaction and when the sequencer executes it. Even with an honest sequencer, the price used at execution can differ meaningfully from the price the user observed when signing. No minimum-output field exists in the signed payload to bound this exposure.

---

### Recommendation

Add `minNlpAmount` to `MintNlp` and `minQuoteAmount` to `BurnNlp`:

```solidity
struct MintNlp {
    bytes32 sender;
    uint128 quoteAmount;
    uint128 minNlpAmount;   // slippage guard
    uint64 nonce;
}

struct BurnNlp {
    bytes32 sender;
    uint128 nlpAmount;
    uint128 minQuoteAmount; // slippage guard
    uint64 nonce;
}
```

In `Clearinghouse.mintNlp`, after computing `nlpAmount`, add:

```solidity
require(nlpAmount >= int128(txn.minNlpAmount), "STH");
```

In `Clearinghouse.burnNlp`, after computing the final `quoteAmount`, add:

```solidity
require(quoteAmount >= int128(txn.minQuoteAmount), "STH");
```

---

### Proof of Concept

1. Alice signs a `MintNlp` transaction with `quoteAmount = 1000e18` when the NLP oracle price is `1.00e18` (expecting ~1000 NLP).
2. Before the sequencer executes the transaction, the NLP price rises to `1.10e18` due to protocol PnL accrual.
3. The sequencer submits the transaction with `oraclePriceX18 = 1.10e18`.
4. `mintNlp` computes `nlpAmount = 1000e18 / 1.10e18 ≈ 909 NLP`.
5. Alice receives ~91 fewer NLP tokens than expected, and her minted NLP is locked for 4 days.
6. No on-chain check exists to revert this outcome — Alice's signed payload contained no `minNlpAmount`. [7](#0-6) [8](#0-7)

```text
    function burnNlp(
        IEndpoint.BurnNlp calldata txn,
        int128 oraclePriceX18,
        IEndpoint.NlpPool[] calldata nlpPools,
        int128[] calldata nlpPoolRebalanceX18
    ) external onlyEndpoint {
        require(!RiskHelper.isIsolatedSubaccount(txn.sender), ERR_UNAUTHORIZED);

        ISpotEngine spotEngine = _spotEngine();
        spotEngine.updatePrice(NLP_PRODUCT_ID, oraclePriceX18);

        require(txn.nlpAmount <= INT128_MAX, ERR_CONVERSION_OVERFLOW);
        int128 nlpAmount = int128(txn.nlpAmount);
        require(
            spotEngine.getNlpUnlockedBalance(txn.sender).amount >= nlpAmount,
            ERR_UNLOCKED_NLP_INSUFFICIENT
        );
        int128 quoteAmount = nlpAmount.mul(oraclePriceX18);
        int128 burnFee = MathHelper.max(ONE, quoteAmount / 1000);
        quoteAmount = MathHelper.max(0, quoteAmount - burnFee);

        _validateNlpRebalance(nlpPools, nlpPoolRebalanceX18, -quoteAmount);
        for (uint128 i = 0; i < nlpPoolRebalanceX18.length; i++) {
            require(nlpPoolRebalanceX18[i] <= 0, ERR_INVALID_NLP_REBALANCE);
        }

        spotEngine.updateBalance(NLP_PRODUCT_ID, txn.sender, -nlpAmount);
        spotEngine.updateBalance(NLP_PRODUCT_ID, N_ACCOUNT, nlpAmount);

        if (quoteAmount > 0) {
            spotEngine.updateBalance(QUOTE_PRODUCT_ID, txn.sender, quoteAmount);
            _applyNlpRebalance(spotEngine, nlpPools, nlpPoolRebalanceX18);
        }

        require(
            spotEngine.getBalance(NLP_PRODUCT_ID, txn.sender).amount >= 0,
            ERR_SUBACCT_HEALTH
        );
        // Burning NLP can decrease health if the burn fee exceeds the health improvement
        // from the withdrawal. This check prevents malicious actors from deliberately
        // creating unhealthy subaccounts through NLP burns.
        require(
            getHealth(txn.sender, IProductEngine.HealthType.MAINTENANCE) >= 0,
            ERR_SUBACCT_HEALTH
        );
    }
