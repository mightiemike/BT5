### Title
`ValidResourceBounds::AllResources` Silently Downgraded to `L1Gas` in Protobuf Deserialization, Producing Wrong Transaction Hash - (File: `crates/apollo_protobuf/src/converters/transaction.rs`)

### Summary

The protobuf-to-`ValidResourceBounds` converter in `crates/apollo_protobuf/src/converters/transaction.rs` classifies a V3 `AllResources` transaction as the legacy `L1Gas` variant whenever `l2_gas` and `l1_data_gas` are both zero. Because `get_tip_resource_bounds_hash` branches on this variant to decide whether to include `l1_data_gas` in the Poseidon hash preimage, the transaction hash computed after P2P deserialization diverges from the hash computed at the originating node, causing syncing nodes to reject valid blocks or store wrong hashes.

### Finding Description

`ValidResourceBounds` has two variants:

- `L1Gas(ResourceBounds)` — pre-0.13.3 transactions; hash preimage contains only `[tip, L1_GAS, L2_GAS]`.
- `AllResources(AllResourceBounds)` — post-0.13.3 transactions; hash preimage contains `[tip, L1_GAS, L2_GAS, L1_DATA_GAS]`.

The serializer correctly preserves the distinction:

```rust
// crates/apollo_protobuf/src/converters/transaction.rs:471-489
ValidResourceBounds::AllResources(AllResourceBounds { l1_gas, l2_gas, l1_data_gas }) =>
    protobuf::ResourceBounds {
        l1_gas: Some(l1_gas.into()),
        l2_gas: Some(l2_gas.into()),
        l1_data_gas: Some(l1_data_gas.into()),
    },
``` [1](#0-0) 

But the deserializer re-classifies by inspecting the *values*, not the wire-level type:

```rust
// crates/apollo_protobuf/src/converters/transaction.rs:431-435
Ok(if l1_data_gas.is_zero() && l2_gas.is_zero() {
    ValidResourceBounds::L1Gas(l1_gas)
} else {
    ValidResourceBounds::AllResources(AllResourceBounds { l1_gas, l2_gas, l1_data_gas })
})
``` [2](#0-1) 

A V3 transaction with `AllResourceBounds { l1_gas: X, l2_gas: 0, l1_data_gas: 0 }` — valid and accepted by the gateway — is serialized with all three fields present, but deserialized as `L1Gas(X)`.

The hash function branches on this variant:

```rust
// crates/starknet_api/src/transaction_hash.rs:203-208
resource_felts.extend(match resource_bounds {
    ValidResourceBounds::L1Gas(_) => vec![],                          // l1_data_gas omitted
    ValidResourceBounds::AllResources(all_resources) => {
        vec![get_concat_resource(&all_resources.l1_data_gas, L1_DATA_GAS)?]  // l1_data_gas included
    }
});
``` [3](#0-2) 

- **Originating node** computes: `Poseidon(tip, L1_GAS_packed, L2_GAS_packed=0, L1_DATA_GAS_packed=0)` — 4-element input.
- **Syncing node** computes: `Poseidon(tip, L1_GAS_packed, L2_GAS_packed=0)` — 3-element input.

These are distinct Poseidon outputs. The same mismatch exists in the RPC-layer converter:

```rust
// crates/apollo_rpc/src/v0_8/transaction.rs:188-199
impl From<ResourceBoundsMapping> for ValidResourceBounds {
    fn from(value: ResourceBoundsMapping) -> Self {
        if value.l1_data_gas.is_zero() && value.l2_gas.is_zero() {
            Self::L1Gas(value.l1_gas)
        } else { ... }
    }
}
``` [4](#0-3) 

### Impact Explanation

A syncing node that receives a block containing a V3 transaction with `l2_gas = 0` and `l1_data_gas = 0` will:

1. Deserialize the transaction's resource bounds as `L1Gas` instead of `AllResources`.
2. Recompute the transaction hash via `get_invoke_transaction_v3_hash` → `get_tip_resource_bounds_hash`, producing a hash that omits the `L1_DATA_GAS` term.
3. The computed hash diverges from the hash stored in the block receipt/header.
4. `validate_transaction_hash` returns `false`, causing the syncing node to reject the valid block or store a wrong hash that is then served authoritatively via RPC.

This matches: **High — RPC execution, fee estimation, tracing, simulation, or pending view returns an authoritative-looking wrong value** (wrong transaction hash served to clients), and **High — Mempool/gateway/RPC admission accepts invalid transactions or rejects valid transactions before sequencing** (valid blocks rejected during sync).

### Likelihood Explanation

V3 transactions with `l2_gas = 0` and `l1_data_gas = 0` are common and explicitly accepted by the gateway's stateless validator (test cases `valid_l1_gas` confirm `AllResourceBounds { l1_gas: X, ..Default::default() }` is valid). [5](#0-4) 

Any such transaction included in a block will trigger the mismatch on every syncing peer.

### Recommendation

Remove the value-based heuristic from the protobuf deserializer. The `AllResources` vs `L1Gas` distinction must be preserved structurally, not inferred from zero values. One approach: add a boolean discriminant field to the protobuf `ResourceBounds` message, or use a protobuf `oneof`. At minimum, the deserializer should always produce `AllResources` when all three fields are present on the wire (regardless of their values), reserving `L1Gas` only for legacy messages where `l2_gas` is absent:

```rust
// Correct: presence of l2_gas field, not its value, determines the variant
Ok(if value.l2_gas.is_none() {
    ValidResourceBounds::L1Gas(l1_gas)
} else {
    ValidResourceBounds::AllResources(AllResourceBounds { l1_gas, l2_gas, l1_data_gas })
})
```

Apply the same fix to `From<ResourceBoundsMapping> for ValidResourceBounds` in `crates/apollo_rpc/src/v0_8/transaction.rs`.

### Proof of Concept

1. Submit a V3 invoke transaction with `resource_bounds = { l1_gas: { max_amount: 1000, max_price_per_unit: 1 }, l2_gas: { max_amount: 0, max_price_per_unit: 0 }, l1_data_gas: { max_amount: 0, max_price_per_unit: 0 } }`.
2. The gateway accepts it and computes hash `H_orig` using `AllResources` → 4-element Poseidon input.
3. The transaction is included in a block and propagated via P2P protobuf.
4. A syncing node deserializes: `l1_data_gas.is_zero() && l2_gas.is_zero()` → `L1Gas(l1_gas)`.
5. The syncing node computes hash `H_sync` using `L1Gas` → 3-element Poseidon input.
6. `H_orig ≠ H_sync`. The syncing node's `validate_transaction_hash` returns `false`.
7. The syncing node rejects the block or stores `H_sync` and serves it via `starknet_getTransactionByHash`, returning a hash that does not match the one on the originating node.
