Audit Report

## Title
`SwapAllowlistExtension.beforeSwap` Checks Router Address as `sender`, Allowing Complete Allowlist Bypass via `MetricOmmSimpleRouter` — (`metric-periphery/contracts/extensions/SwapAllowlistExtension.sol`)

## Summary

`SwapAllowlistExtension.beforeSwap` gates swaps by checking the `sender` argument against `allowedSwapper[pool][sender]`. When a swap is routed through `MetricOmmSimpleRouter`, the pool's `msg.sender` — and therefore the `sender` forwarded to the extension — is the router contract, not the end user. A pool admin who allowlists the router to support legitimate router-mediated swaps inadvertently grants every on-chain user the ability to bypass the per-user allowlist, completely nullifying the access-control invariant.

## Finding Description

**Root cause — `MetricOmmPool.swap` passes its own `msg.sender` as `sender` to the extension:**

`MetricOmmPool.swap` calls `_beforeSwap(msg.sender, ...)`: [1](#0-0) 

`ExtensionCalling._beforeSwap` forwards that value unchanged as the first argument to the extension hook: [2](#0-1) 

**`SwapAllowlistExtension.beforeSwap` checks that forwarded `sender`:** [3](#0-2) 

**`MetricOmmSimpleRouter.exactInputSingle` calls `pool.swap(...)` directly, making the router the pool's `msg.sender`:** [4](#0-3) 

The full call chain is:

```
Bob → router.exactInputSingle(...)
        → pool.swap(...)          [pool's msg.sender = router]
            → _beforeSwap(msg.sender=router, ...)
                → extension.beforeSwap(sender=router, ...)
                    → allowedSwapper[pool][router] == true → PASSES
```

The extension evaluates `allowedSwapper[pool][router]`, not `allowedSwapper[pool][Bob]`. There is no existing guard that recovers the originating user: `extensionData` is passed through unchanged from the router call, but the router passes `""` (empty bytes) for `exactInputSingle`, and the extension does not decode it regardless. [4](#0-3) 

## Impact Explanation

A pool deploying `SwapAllowlistExtension` intends to restrict swaps to a curated set of counterparties (e.g., KYC'd addresses, institutional traders). LPs deposit expecting only authorized counterparties. Once the pool admin allowlists the router — a necessary operational step for any pool that wants to support the standard periphery swap path — every on-chain user can call `router.exactInputSingle(...)` and execute swaps against the pool's liquidity. This results in direct LP principal loss from unrestricted swap volume and adverse selection, and complete nullification of the access-control invariant the pool was deployed to enforce. The same bypass applies to `exactInput`, `exactOutputSingle`, and `exactOutput` entry points on the router. [5](#0-4) 

## Likelihood Explanation

The trigger requires only that the pool admin allowlists the router — a routine operational step with no on-chain warning that it opens the pool to all users. The `SwapAllowlistExtension` interface provides no indication of this behavior. Any pool that (a) deploys `SwapAllowlistExtension` and (b) allowlists the router is fully exposed. The attacker needs only to call the public router with a valid swap; no special privileges, flash loans, or multi-step setup are required. The attack is repeatable on every block. [6](#0-5) 

## Recommendation

The extension must gate on the economic actor, not the immediate `msg.sender` of the pool. The most robust fix is to require the router to encode the originating user (`msg.sender`) into `extensionData`, and have the extension decode and verify it. The `beforeSwap` hook already receives `extensionData` unchanged: [7](#0-6) 

Concretely: (1) `MetricOmmSimpleRouter` should ABI-encode `msg.sender` into `extensionData` before calling `pool.swap`; (2) `SwapAllowlistExtension.beforeSwap` should decode the originating user from `extensionData` when `sender` is a known router, and check `allowedSwapper[pool][originatingUser]` instead. Alternatively, maintain a separate `trustedRouter` mapping and require routers to always supply the originating user in a standardized `extensionData` field.

## Proof of Concept

```
1. Deploy pool with SwapAllowlistExtension configured on beforeSwap.
2. Pool admin calls setAllowedToSwap(pool, Alice, true)
   // Alice is the only intended swapper
3. Pool admin calls setAllowedToSwap(pool, router, true)
   // necessary for Alice to use MetricOmmSimpleRouter
4. Bob (not allowlisted) calls:
       router.exactInputSingle({pool: pool, recipient: Bob, amountIn: X, ...})
5. Router calls pool.swap(Bob, ...) → pool's msg.sender = router
6. Pool calls _beforeSwap(msg.sender=router, ...)
7. Extension evaluates: allowedSwapper[pool][router] == true → PASSES
8. Bob's swap executes against LP liquidity despite not being on the allowlist.
``` [8](#0-7) [9](#0-8)

    _checkDeadline(params.deadline);
    _validatePath(params.tokens, params.pools, params.extensionDatas);

    uint256 last = params.pools.length - 1;
    int128 amount = MetricOmmSwapInputs.asAmountSpecifiedIn(params.amountIn);

    for (uint256 i = 0; i <= last; i++) {
      address pool = params.pools[i];
      bool zeroForOne = MetricOmmSwapPath.resolveZeroForOneBitmap(params.zeroForOneBitMap, i);

      _setNextCallbackContext(pool, CALLBACK_MODE_JUST_PAY, i == 0 ? msg.sender : address(this), params.tokens[i]);
      (int128 amount0Delta, int128 amount1Delta) = IMetricOmmPoolActions(pool)
        .swap(
          i == last ? params.recipient : address(this),
          zeroForOne,
          amount,
          MetricOmmSwapPath.openLimit(zeroForOne),
          "",
          params.extensionDatas[i]
        );

      int128 amountInActual = MetricOmmSwapResults.extractAmountIn(zeroForOne, amount0Delta, amount1Delta);
      if (amountInActual < amount) revert InvalidInputAmountAtHop(uint8(i), amountInActual, amount);

      amount = MetricOmmSwapResults.extractAmountOut(zeroForOne, amount0Delta, amount1Delta);
    }

    if (amount <= 0) revert InvalidSwapDeltas();
    amountOut = MetricOmmSwapInputs.int128ToUint128(amount);
    if (amountOut < params.amountOutMinimum) revert InsufficientOutput(amountOut, params.amountOutMinimum);

    _clearExpectedCallbackPool();
  }
