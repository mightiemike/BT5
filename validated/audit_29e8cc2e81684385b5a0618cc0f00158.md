### Title
`gas_prices_to_hash` omits `l2_gas_consumed` and `next_l2_gas_price` from the block hash preimage, allowing a malicious P2P peer to serve manipulated gas-pricing fields that pass hash verification - (`File: crates/starknet_api/src/block_hash/block_hash_calculator.rs`)

---

### Summary

`gas_prices_to_hash` does not include `l2_gas_consumed` or `next_l2_gas_price` in the block hash preimage for any `BlockHashVersion::V0_13_4` block (all Starknet versions ≥ 0.13.4, including the current 0.14.x series). Both fields are present in `BlockHeaderWithoutHash`, transmitted in the P2P protobuf `SignedBlockHeader`, and stored in `StorageBlockHeader`. Because they are absent from the hash, a malicious P2P peer can serve a `SignedBlockHeader` with an arbitrary `next_l2_gas_price` while the `block_hash` remains valid. A syncing node that accepts this header seeds its EIP-1559 gas price from the manipulated value, causing every subsequent block it proposes or validates to use the wrong gas price.

---

### Finding Description

`gas_prices_to_hash` in `crates/starknet_api/src/block_hash/block_hash_calculator.rs` is the sole function that maps gas-price fields into the Poseidon block hash preimage. For `BlockHashVersion::V0_13_4` (the only version used for all blocks from Starknet 0.13.4 onward, including 0.14.4) it hashes exactly seven felts:

```
STARKNET_GAS_PRICES0 | l1_gas_price_wei | l1_gas_price_fri
| l1_data_gas_price_wei | l1_data_gas_price_fri
| l2_gas_price_wei | l2_gas_price_fri
``` [1](#0-0) 

A developer TODO at line 416 explicitly acknowledges the omission:

```rust
// TODO(Ayelet): add l2_gas_consumed, next_l2_gas_price after 0.14.0.
``` [2](#0-1) 

The current latest version is `V0_14_4`, so the TODO deadline has passed without the fix being applied.

`BlockHeaderWithoutHash` carries both missing fields as first-class members:

```rust
pub l2_gas_consumed: GasAmount,
pub next_l2_gas_price: GasPrice,
``` [3](#0-2) 

The P2P protobuf converter deserializes both fields from `SignedBlockHeader` and places them directly into `BlockHeaderWithoutHash` without any hash-based integrity check: [4](#0-3) 

The serializer writes them back out unconditionally: [5](#0-4) 

`StorageBlockHeader` persists both fields: [6](#0-5) 

The consensus orchestrator reads `next_l2_gas_price` from a synced block header and uses it directly as the starting gas price for the next height: [7](#0-6) 

The full hash calculation path that never touches these fields: [8](#0-7)

### Impact Explanation

A syncing node queries block headers from P2P peers. A malicious peer returns a `SignedBlockHeader` with a forged `next_l2_gas_price` (e.g., set to 1 wei or to `u128::MAX`). The node verifies `block_hash` against the header fields; because `next_l2_gas_price` is not part of the hash preimage, the check passes. The node stores the header and seeds its EIP-1559 gas price from the forged value. Every block it subsequently proposes or validates uses the wrong base gas price, causing:

- Incorrect fee estimation returned by RPC (`starknet_estimateFee`, `starknet_simulateTransactions`).
- Wrong `next_l2_gas_price` propagated to further syncing peers, amplifying the divergence.
- Economic harm: users underpay or overpay fees; the sequencer's bouncer may accept or reject blocks based on incorrect gas accounting.

This matches **High – RPC execution, fee estimation, tracing, simulation, or pending view returns an authoritative-looking wrong value** and **Critical – Incorrect fee, gas, bouncer, resource accounting, refund, balance, or L1 gas price effect with economic impact**.

---

### Likelihood Explanation

The trigger is any P2P peer the syncing node connects to. No privileged access is required. The manipulation is invisible to the victim because the block hash check passes. The `BlockHashVersion` enum has only two variants and the vulnerable one (`V0_13_4`) covers every block from Starknet 0.13.4 onward, so all production traffic is affected. [9](#0-8) 

---

### Recommendation

Introduce a new `BlockHashVersion::V0_14_0` (or later) variant and extend `gas_prices_to_hash` to include `l2_gas_consumed` and `next_l2_gas_price` in the Poseidon sub-hash for that version:

```rust
// For BlockHashVersion >= V0_14_0:
HashChain::new()
    .chain(&STARKNET_GAS_PRICES0)
    .chain(&l1_gas_price.price_in_wei.0.into())
    .chain(&l1_gas_price.price_in_fri.0.into())
    .chain(&l1_data_gas_price.price_in_wei.0.into())
    .chain(&l1_data_gas_price.price_in_fri.0.into())
    .chain(&l2_gas_price.price_in_wei.0.into())
    .chain(&l2_gas_price.price_in_fri.0.into())
    .chain(&l2_gas_consumed.0.into())      // add
    .chain(&next_l2_gas_price.0.into())    // add
    .get_poseidon_hash()
```

`PartialBlockHashComponents` must be extended to carry both fields, and `calculate_block_hash` must pass them through. The Cairo OS `block_hash.cairo` constant `BLOCK_HASH_VERSION` must be bumped to `STARKNET_BLOCK_HASH2` in lockstep, and the existing test `test_block_hash_version` will enforce the invariant. [10](#0-9)

### Proof of Concept

1. A syncing node at height N connects to a malicious P2P peer.
2. The peer returns a valid `SignedBlockHeader` for block N with `next_l2_gas_price` set to `1` (instead of the legitimate value, e.g., `100_000_000_000`).
3. The node calls `calculate_block_hash` on the received header. `gas_prices_to_hash` does not include `next_l2_gas_price`, so the computed hash matches the peer-supplied `block_hash`. The header is accepted and stored.
4. The consensus orchestrator calls `try_sync` → reads `block_header_without_hash.next_l2_gas_price` → sets `self.l2_gas_price = GasPrice(1)`.
5. For every subsequent block the node proposes, `calculate_next_l2_gas_price_for_fin` starts from `GasPrice(1)`, producing a gas price orders of magnitude below the legitimate value.
6. `starknet_estimateFee` returns fees based on this wrong price; users submit transactions with insufficient fees; the sequencer's bouncer makes incorrect admission decisions. [11](#0-10) [12](#0-11)

pub enum BlockHashVersion {
    V0_13_2,
    V0_13_4,
}

impl From<BlockHashVersion> for StarknetVersion {
    fn from(value: BlockHashVersion) -> Self {
        match value {
            BlockHashVersion::V0_13_2 => StarknetVersion::V0_13_2,
            BlockHashVersion::V0_13_4 => StarknetVersion::V0_13_4,
        }
    }
}

impl TryFrom<StarknetVersion> for BlockHashVersion {
    type Error = StarknetApiError;

    fn try_from(value: StarknetVersion) -> StarknetApiResult<Self> {
        if value < Self::V0_13_2.into() {
            Err(StarknetApiError::BlockHashVersion { version: value.to_string() })
        } else if value < Self::V0_13_4.into() {
            // Starknet versions 0.13.2 and 0.13.3 both have the same block hash mechanism.
            Ok(Self::V0_13_2)
        } else {
            Ok(Self::V0_13_4)
        }
    }
}

pub fn calculate_block_hash(
    partial_block_hash_components: &PartialBlockHashComponents,
    state_root: GlobalRoot,
    previous_block_hash: BlockHash,
) -> StarknetApiResult<BlockHash> {
    let block_hash_version: BlockHashVersion =
        partial_block_hash_components.starknet_version.try_into()?;
    let block_commitments = &partial_block_hash_components.header_commitments;
    Ok(BlockHash(
        HashChain::new()
            .chain(&block_hash_version.clone().into())
            .chain(&partial_block_hash_components.block_number.0.into())
            .chain(&state_root.0)
            .chain(&partial_block_hash_components.sequencer.0)
            .chain(&partial_block_hash_components.timestamp.0.into())
            .chain(&block_commitments.concatenated_counts)
            .chain(&block_commitments.state_diff_commitment.0.0)
            .chain(&block_commitments.transaction_commitment.0)
            .chain(&block_commitments.event_commitment.0)
            .chain(&block_commitments.receipt_commitment.0)
            .chain_iter(
                gas_prices_to_hash(
                    &partial_block_hash_components.l1_gas_price,
                    &partial_block_hash_components.l1_data_gas_price,
                    &partial_block_hash_components.l2_gas_price,
                    &block_hash_version,
                )
                .iter(),
            )
            .chain(
                &Felt::try_from(&partial_block_hash_components.starknet_version)
                    .expect("Expect ASCII version"),
            )
            .chain(&Felt::ZERO)
            .chain(&previous_block_hash.0)
            .get_poseidon_hash(),
    ))

/// Compute the next L2 gas price (for the fin or for updating state). Respects override when set.
pub fn calculate_next_l2_gas_price_for_fin(
    current_l2_gas_price: GasPrice,
    height: BlockNumber,
    l2_gas_used: GasAmount,
    override_l2_gas_price_fri: Option<u128>,
    min_l2_gas_price_per_height: &[PricePerHeight],
    fee_actual: Option<GasPrice>,
) -> GasPrice {
    if let Some(override_value) = override_l2_gas_price_fri {
        info!(
            "L2 gas price ({}) is not updated, remains on override value of {override_value} fri",
            current_l2_gas_price.0
        );
        return GasPrice(override_value);
    }
    let gas_target = VersionedConstants::latest_constants().gas_target;
    let config_min = get_min_gas_price_for_height(height, min_l2_gas_price_per_height);
    let effective_min = match fee_actual {
        Some(fa) => GasPrice(max(config_min.0, fa.0)),
        None => config_min,
    };
    calculate_next_base_gas_price(current_l2_gas_price, l2_gas_used, gas_target, effective_min)
}
