### Title
Unclaimed Airdrop Tokens Permanently Locked in `Airdrop.sol` With No Recovery Mechanism - (File: `core/contracts/Airdrop.sol`)

---

### Summary

The `Airdrop.sol` contract is pre-funded with tokens that eligible users claim via weekly Merkle-proof distributions. Once tokens are deposited and a week's Merkle root is registered, any tokens that go unclaimed — due to sanctioned addresses, lost keys, or user inaction — are permanently locked in the contract. There is no `withdraw`, `sweep`, `recover`, or equivalent function anywhere in the contract or its interface.

---

### Finding Description

The `Airdrop` contract holds a pool of ERC-20 tokens and distributes them to eligible users via `claim()`. The owner registers a Merkle root per week via `registerMerkleRoot()`, and users call `claim()` with a proof to receive their allocation.

The contract's only outbound token transfer is in `_claim()`: [1](#0-0) 

There is no function to recover tokens that were never claimed. The `IAirdrop` interface exposes only `claim` and `getClaimed`: [2](#0-1) 

The `Airdrop.sol` contract itself has no `withdraw`, `recover`, `sweep`, or `rescue` function of any kind: [3](#0-2) 

Additionally, the `_verifyProof` function explicitly reverts for sanctioned addresses: [4](#0-3) 

This means tokens allocated to sanctioned addresses are provably unclaimable and permanently locked — there is no path for the protocol to recover them.

---

### Impact Explanation

Any tokens deposited into `Airdrop.sol` that are not claimed are permanently locked. This includes:

1. Tokens allocated to sanctioned addresses (blocked by the `isSanctioned` check — unclaimable by design).
2. Tokens allocated to users who lose their private keys or simply never claim.
3. Any surplus tokens deposited beyond what the Merkle tree allocates.

The asset delta is: `token.balanceOf(address(Airdrop)) - sum(all claimed amounts)` is permanently irrecoverable. Over multiple weeks of operation, this accumulates into a material loss of protocol-owned tokens.

---

### Likelihood Explanation

Likelihood is high. Airdrop non-participation rates are routinely 20–50%+ in practice. Sanctioned addresses are a guaranteed non-zero set given the explicit sanctions check. The protocol has no time-bounded claim window, so there is no natural expiry that would trigger a recovery path — tokens simply accumulate indefinitely.

---

### Recommendation

Add an owner-callable recovery function that allows sweeping unclaimed tokens after a configurable claim deadline has passed:

```solidity
function recoverUnclaimed(address recipient, uint256 amount) external onlyOwner {
    SafeERC20.safeTransfer(IERC20(token), recipient, amount);
}
```

Optionally, track per-week total allocations and enforce a claim window (e.g., 90 days after `registerMerkleRoot`) before allowing recovery, to protect legitimate late claimants.

---

### Proof of Concept

1. Owner deploys `Airdrop.sol`, pre-funds it with 1,000,000 tokens.
2. Owner calls `registerMerkleRoot(1, root)` where the Merkle tree allocates 1,000,000 tokens across 1,000 addresses.
3. 200 of those addresses are sanctioned — their `claim()` calls revert at: [4](#0-3) 
4. 300 additional addresses never call `claim()`.
5. After week 1 concludes and week 2 is registered, the 500,000 tokens allocated to non-claimants remain in the contract.
6. No function exists to recover them. `token.balanceOf(address(airdrop))` permanently exceeds the sum of all `claimed` mappings with no path to zero the difference.

```text
// SPDX-License-Identifier: GPL-2.0-or-later
pragma solidity ^0.8.0;

import "./interfaces/IAirdrop.sol";
import "./Endpoint.sol";
import "@openzeppelin/contracts/utils/cryptography/MerkleProof.sol";
import "@openzeppelin/contracts-upgradeable/access/OwnableUpgradeable.sol";
import "@openzeppelin/contracts/token/ERC20/IERC20.sol";
import "@openzeppelin/contracts/token/ERC20/utils/SafeERC20.sol";

contract Airdrop is OwnableUpgradeable, IAirdrop {
    address internal token;
    address internal sanctions;
    uint32 internal pastWeeks;

    mapping(uint32 => bytes32) internal merkleRoots;
    mapping(uint32 => mapping(address => uint256)) internal claimed;

    /// @custom:oz-upgrades-unsafe-allow constructor
    constructor() {
        _disableInitializers();
    }

    function initialize(address _token, address _sanctions)
        external
        initializer
    {
        __Ownable_init();
        token = _token;
        sanctions = _sanctions;
    }

    function registerMerkleRoot(uint32 week, bytes32 merkleRoot)
        external
        onlyOwner
    {
        pastWeeks += 1;
        require(week == pastWeeks, "Invalid week provided.");
        merkleRoots[week] = merkleRoot;
    }

    function _verifyProof(
        uint32 week,
        address sender,
        uint256 totalAmount,
        bytes32[] calldata proof
    ) internal {
        require(claimed[week][sender] == 0, "Already claimed.");
        require(
            merkleRoots[week] != bytes32(0),
            "Week hasn't been registered."
        );
        require(
            !ISanctionsList(sanctions).isSanctioned(sender),
            "address is sanctioned."
        );
        bytes32 leaf = keccak256(
            bytes.concat(keccak256(abi.encode(sender, totalAmount)))
        );
        bool isValidLeaf = MerkleProof.verify(proof, merkleRoots[week], leaf);
        require(isValidLeaf, "Invalid proof.");
        claimed[week][sender] = totalAmount;
    }

    function _claim(
        uint32 week,
        uint256 totalAmount,
        bytes32[] calldata proof
    ) internal {
        _verifyProof(week, msg.sender, totalAmount, proof);
        SafeERC20.safeTransfer(IERC20(token), msg.sender, totalAmount);
        emit Claim(msg.sender, week, totalAmount);
    }

    function claim(ClaimProof[] calldata claimProofs) external {
        for (uint32 i = 0; i < claimProofs.length; i++) {
            _claim(
                claimProofs[i].week,
                claimProofs[i].totalAmount,
                claimProofs[i].proof
            );
        }
    }

    function getClaimed(address account)
        external
        view
        returns (uint256[] memory)
    {
        uint256[] memory result = new uint256[](pastWeeks + 1);
        for (uint32 week = 1; week <= pastWeeks; week++) {
            result[week] = claimed[week][account];
        }
        return result;
    }
}
```
