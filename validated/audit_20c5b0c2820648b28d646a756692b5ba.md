Audit Report

## Title
SwapAllowlistExtension Checks Router Address Instead of Actual User, Allowing Allowlist Bypass via MetricOmmSimpleRouter — (`metric-periphery/contracts/extensions/SwapAllowlistExtension.sol`)

## Summary

`SwapAllowlistExtension.beforeSwap` gates swaps by checking the `sender` parameter against a per-pool allowlist. When a swap is routed through `MetricOmmSimpleRouter`, the pool's `msg.sender` is the router, so the pool passes the router address as `sender` to the extension. The extension therefore checks whether the router is allowlisted — not the actual end user. Any non-allowlisted user can bypass the swap allowlist by routing through the router.

## Finding Description

`SwapAllowlistExtension.beforeSwap` performs the check at line 37:

```solidity
if (!allowAllSwappers[msg.sender] && !allowedSwapper[msg.sender][sender]) {
    revert IMetricOmmPoolActions.NotAllowedToSwap();
}
```

Here `msg.sender` is the pool (the caller of the extension), and `sender` is the first parameter — whatever the pool passes as the initiating swapper. [1](#0-0) 

`ExtensionCalling._beforeSwap` forwards the `sender` parameter directly to the extension without modification: [2](#0-1) 

When a user calls `MetricOmmSimpleRouter.exactInputSingle`, the router calls `pool.swap(params.recipient, ...)`. At that point, `msg.sender` inside the pool is the router address, so the pool passes `sender = router` into `_beforeSwap`. The extension evaluates `allowedSwapper[pool][router]` — never seeing the actual user's address. [3](#0-2) 

The same call path applies to `exactInput`, `exactOutputSingle`, and `exactOutput` — all call `pool.swap(...)` with the router as `msg.sender`. [4](#0-3) 

For any allowlisted user to use the router, the pool admin is forced to allowlist the router address. Once `allowedSwapper[pool][router] = true`, every user — including those explicitly excluded — can bypass the restriction by calling the router instead of the pool directly. No existing guard prevents this: the extension has no mechanism to recover the original `msg.sender` from the router call.

## Impact Explanation

A pool admin deploys `SwapAllowlistExtension` to restrict swaps to a curated set of addresses (e.g., KYC-verified counterparties). The admin must allowlist the router for any allowlisted user to use the standard periphery path. Once the router is allowlisted, the allowlist provides zero protection against router-mediated swaps — any excluded user can execute swaps freely. This is a direct, complete bypass of the access-control invariant the extension was deployed to enforce, with fund-impacting consequences on curated pools (unauthorized parties trade against restricted liquidity).

## Likelihood Explanation

- `MetricOmmSimpleRouter` is the standard, publicly documented swap entrypoint.
- Any user aware of the router can exploit this without any privileged access or special setup.
- Pool admins who want allowlisted users to use the router are forced to allowlist the router, which opens the bypass for everyone.
- No special tokens, malicious setup, or admin cooperation is required beyond the router being allowlisted.

## Recommendation

The extension must resolve the actual end user's identity rather than trusting the `sender` parameter, which reflects the direct caller of `pool.swap()`. Two sound approaches:

1. **Pass the original initiator through `extensionData`**: The router should encode the original `msg.sender` (the end user) into `extensionData`, and the extension should decode and check that address. This requires the extension to trust only calls from known routers.
2. **Forward original caller in the router**: Add an explicit `originator` field to the swap call that the pool passes as `sender` to extensions, with the router setting it to `msg.sender` before calling the pool.

The root fix is ensuring the identity checked by the allowlist is the same actor who economically initiates the swap — not the intermediary contract that relays the call.

## Proof of Concept

1. Deploy a pool with `SwapAllowlistExtension` configured as a `beforeSwap` hook.
2. As pool admin, call `setAllowedToSwap(pool, router, true)` — required for any allowlisted user to use the router.
3. As a non-allowlisted `attacker`, call `router.exactInputSingle(...)` targeting the pool.
4. The pool calls `extension.beforeSwap(sender=router, ...)`.
5. The extension evaluates `allowedSwapper[pool][router] == true` → passes.
6. The attacker's swap executes successfully despite not being on the allowlist. [5](#0-4)
