### Title
Unsigned `oraclePriceX18` in `SignedMintNlp`/`SignedBurnNlp` Enables Arbitrary NLP Price Manipulation via Slow Mode — (`core/contracts/Verifier.sol`, `core/contracts/Clearinghouse.sol`)

---

### Summary

The `oraclePriceX18` field embedded in `SignedMintNlp` and `SignedBurnNlp` is **not included** in the EIP-712 digest that the user signs. Because any user can submit `MintNlp` or `BurnNlp` transactions through the slow-mode path with fully attacker-controlled `oraclePriceX18` and `nlpPoolRebalanceX18` values, an unprivileged caller can mint NLP tokens at an artificially low price and redeem them at an artificially high price, draining NLP pool quote balances.

---

### Finding Description

**The unsigned price field**

`Verifier.computeDigest` constructs the EIP-712 digest for `MintNlp` and `BurnNlp` as follows:

```
MintNlp digest = hash(MINT_NLP_SIGNATURE, sender, quoteAmount, nonce)
BurnNlp digest = hash(BURN_NLP_SIGNATURE, sender, nlpAmount,   nonce)
``` [1](#0-0) 

`oraclePriceX18` and `nlpPoolRebalanceX18` — both present in `SignedMintNlp` / `SignedBurnNlp` — are **absent from the digest**. The user's signature therefore commits to nothing about the price used to value NLP tokens. [2](#0-1) 

**The slow-mode entry path**

`submitSlowModeTransaction` accepts any transaction type that is not in the owner-only list. `MintNlp` and `BurnNlp` are not restricted, so any user can queue them: [3](#0-2) 

When the slow-mode transaction is later executed, `processTransactionImpl` decodes the full `SignedMintNlp` from the stored bytes — including the attacker-supplied `oraclePriceX18` — and passes it directly to `Clearinghouse.mintNlp`: [4](#0-3) 

**How the price is consumed**

`mintNlp` uses `oraclePriceX18` to compute how many NLP tokens the user receives:

```solidity
int128 nlpAmount = quoteAmount.div(oraclePriceX18);
``` [5](#0-4) 

`burnNlp` uses it to compute how much quote the user receives back:

```solidity
int128 quoteAmount = nlpAmount.mul(oraclePriceX18);
``` [6](#0-5) 

Both functions also call `spotEngine.updatePrice(NLP_PRODUCT_ID, oraclePriceX18)`, so the health check that follows uses the same attacker-supplied price — meaning the health check does not catch the manipulation. [7](#0-6) 

**The rebalance drain**

`_applyNlpRebalance` blindly applies the attacker-controlled `nlpPoolRebalanceX18` deltas to NLP pool subaccounts with no floor check: [8](#0-7) 

There is no assertion that NLP pool subaccounts hold sufficient quote before the debit is applied.

---

### Impact Explanation

An attacker can:

1. **Mint NLP at near-zero price** — submit a slow-mode `MintNlp` with `oraclePriceX18 = 1`. For a deposit of `Q` quote units, they receive `Q / 1 = Q` NLP tokens. The health check passes because the NLP price is simultaneously set to `1`, so the NLP position is valued at `Q * 1 = Q`, exactly offsetting the quote debit.

2. **Wait for the NLP lock period to expire** (enforced by `getNlpUnlockedBalance`).

3. **Burn NLP at an inflated price** — submit a slow-mode `BurnNlp` with `oraclePriceX18 = P_high`. They receive `Q * P_high / 1e18` in quote. The health check passes because the NLP tokens are now valued at the same inflated price. The NLP pool subaccounts absorb the corresponding negative rebalance, going into bad debt.

The net result is that the attacker extracts `Q * (P_high - 1) / 1e18` in quote from the NLP pools — limited only by int128 overflow and the quote balances held by NLP pool subaccounts. [9](#0-8) 

---

### Likelihood Explanation

The slow-mode path is a documented, permissionless fallback mechanism. Any registered subaccount can submit `MintNlp` and `BurnNlp` slow-mode transactions. The only delays are the slow-mode queue delay (3 days) and the NLP lock period. No privileged access, flashloan, or oracle compromise is required. The attacker only needs to craft the transaction bytes with a chosen `oraclePriceX18` value, which is trivial off-chain.

---

### Recommendation

Include `oraclePriceX18` and `nlpPoolRebalanceX18` in the EIP-712 type hash and digest for both `MintNlp` and `BurnNlp`. Concretely, update `MINT_NLP_SIGNATURE` and `BURN_NLP_SIGNATURE` in `Verifier.sol` and the corresponding `computeDigest` branches to commit to these fields:

```
MintNlp(bytes32 sender,uint128 quoteAmount,uint64 nonce,int128 oraclePriceX18,int128[] nlpPoolRebalanceX18)
BurnNlp(bytes32 sender,uint128 nlpAmount,uint64 nonce,int128 oraclePriceX18,int128[] nlpPoolRebalanceX18)
``` [10](#0-9) 

This ensures that a user's signature binds them to a specific price and rebalance distribution, preventing post-signing substitution via the slow-mode path.

---

### Proof of Concept

1. Attacker registers a subaccount and deposits `Q = 1000e18` quote.
2. Attacker calls `submitSlowModeTransaction` with a `MintNlp` transaction where:
   - `sender` = attacker subaccount, `quoteAmount = 1000e18`, `nonce = 0`
   - `oraclePriceX18 = 1` (1 wei, i.e., effectively zero price)
   - `nlpPoolRebalanceX18 = [1000e18]` (sum = `quoteAmount`)
   - Signature is valid over `(sender, quoteAmount, nonce)` only
3. After the slow-mode delay, the transaction executes. Attacker receives `1000e18 / 1 = 1000e18` NLP tokens. Health check passes because NLP price is set to `1`.
4. Attacker waits for the NLP lock period to expire.
5. Attacker calls `submitSlowModeTransaction` with a `BurnNlp` transaction where:
   - `sender` = attacker subaccount, `nlpAmount = 1000e18`, `nonce = 1`
   - `oraclePriceX18 = 1e27` (1e9 in real terms)
   - `nlpPoolRebalanceX18 = [-1000e27]` (sum = `-quoteAmount`)
   - Signature is valid over `(sender, nlpAmount, nonce)` only
6. After the slow-mode delay, the transaction executes. Attacker receives `1000e18 * 1e27 / 1e18 = 1000e27` quote. NLP pool subaccount is debited `1000e27`, going deeply negative. Health check passes because NLP price is set to `1e27`.
7. Attacker withdraws `1000e27` quote, having deposited only `1000e18`. [1](#0-0) [11](#0-10)

    function mintNlp(
        IEndpoint.MintNlp calldata txn,
        int128 oraclePriceX18,
        IEndpoint.NlpPool[] calldata nlpPools,
        int128[] calldata nlpPoolRebalanceX18
    ) external onlyEndpoint {
        require(!RiskHelper.isIsolatedSubaccount(txn.sender), ERR_UNAUTHORIZED);

        ISpotEngine spotEngine = _spotEngine();
        spotEngine.updatePrice(NLP_PRODUCT_ID, oraclePriceX18);

        require(txn.quoteAmount <= INT128_MAX, ERR_CONVERSION_OVERFLOW);
        int128 quoteAmount = int128(txn.quoteAmount);
        int128 nlpAmount = quoteAmount.div(oraclePriceX18);

        _validateNlpRebalance(nlpPools, nlpPoolRebalanceX18, quoteAmount);
        for (uint128 i = 0; i < nlpPoolRebalanceX18.length; i++) {
            require(nlpPoolRebalanceX18[i] >= 0, ERR_INVALID_NLP_REBALANCE);
        }

        spotEngine.updateBalance(NLP_PRODUCT_ID, txn.sender, nlpAmount);
        spotEngine.updateBalance(NLP_PRODUCT_ID, N_ACCOUNT, -nlpAmount);

        spotEngine.updateBalance(QUOTE_PRODUCT_ID, txn.sender, -quoteAmount);
        _applyNlpRebalance(spotEngine, nlpPools, nlpPoolRebalanceX18);

        require(
            getHealth(txn.sender, IProductEngine.HealthType.INITIAL) >= 0,
            ERR_SUBACCT_HEALTH
        );
    }

    function burnNlp(
        IEndpoint.BurnNlp calldata txn,
        int128 oraclePriceX18,
        IEndpoint.NlpPool[] calldata nlpPools,
        int128[] calldata nlpPoolRebalanceX18
    ) external onlyEndpoint {
        require(!RiskHelper.isIsolatedSubaccount(txn.sender), ERR_UNAUTHORIZED);

        ISpotEngine spotEngine = _spotEngine();
        spotEngine.updatePrice(NLP_PRODUCT_ID, oraclePriceX18);

        require(txn.nlpAmount <= INT128_MAX, ERR_CONVERSION_OVERFLOW);
        int128 nlpAmount = int128(txn.nlpAmount);
        require(
            spotEngine.getNlpUnlockedBalance(txn.sender).amount >= nlpAmount,
            ERR_UNLOCKED_NLP_INSUFFICIENT
        );
        int128 quoteAmount = nlpAmount.mul(oraclePriceX18);
        int128 burnFee = MathHelper.max(ONE, quoteAmount / 1000);
        quoteAmount = MathHelper.max(0, quoteAmount - burnFee);

        _validateNlpRebalance(nlpPools, nlpPoolRebalanceX18, -quoteAmount);
        for (uint128 i = 0; i < nlpPoolRebalanceX18.length; i++) {
            require(nlpPoolRebalanceX18[i] <= 0, ERR_INVALID_NLP_REBALANCE);
        }

        spotEngine.updateBalance(NLP_PRODUCT_ID, txn.sender, -nlpAmount);
        spotEngine.updateBalance(NLP_PRODUCT_ID, N_ACCOUNT, nlpAmount);

        if (quoteAmount > 0) {
            spotEngine.updateBalance(QUOTE_PRODUCT_ID, txn.sender, quoteAmount);
            _applyNlpRebalance(spotEngine, nlpPools, nlpPoolRebalanceX18);
        }

        require(
            spotEngine.getBalance(NLP_PRODUCT_ID, txn.sender).amount >= 0,
            ERR_SUBACCT_HEALTH
        );
        // Burning NLP can decrease health if the burn fee exceeds the health improvement
        // from the withdrawal. This check prevents malicious actors from deliberately
        // creating unhealthy subaccounts through NLP burns.
        require(
            getHealth(txn.sender, IProductEngine.HealthType.MAINTENANCE) >= 0,
            ERR_SUBACCT_HEALTH
        );
    }
