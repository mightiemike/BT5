### Title
Phantom Duplicate-Leaf Index Bypass in `verify_transaction_inclusion_v2` Allows Proof of Non-Existent Transaction — (`contract/src/lib.rs`, `merkle-tools/src/lib.rs`)

---

### Summary

`verify_transaction_inclusion_v2` returns `true` for a transaction at index `N` (which does not exist) in a block containing exactly `N` transactions, when the last real transaction `tx[N-1]` is the odd-leaf duplicate. An unprivileged NEAR caller can supply `tx_id=tx[N-1]`, `tx_index=N`, and the identical merkle proof used for `tx_index=N-1`, and the root computation produces the same value as the legitimate proof.

---

### Finding Description

The root of the problem is in `compute_root_from_merkle_proof`: [1](#0-0) 

The function uses `current_position % 2` to decide left/right placement at each level. For the phantom index `N` (odd) vs. the real index `N-1` (even), the parity differs at the leaf level — but because the odd-leaf duplication rule makes both the left and right child identical (`tx[N-1]`), the hash `H(tx[N-1], tx[N-1])` is produced regardless of which side the current hash is placed on.

**Concrete trace for a 5-tx block** (`tx0..tx4`):

```
Tree:
  Level 0 (padded): [tx0, tx1, tx2, tx3, tx4, tx4]
  Level 1:          [A=H(tx0,tx1), B=H(tx2,tx3), C=H(tx4,tx4), C]
  Level 2:          [D=H(A,B), E=H(C,C)]
  Root:             H(D, E)

Proof for index 4 = [tx4, C, D]

Verify(tx4, index=4, [tx4,C,D]):
  pos=4 (even): H(tx4, tx4) = C;  pos→2
  pos=2 (even): H(C, C)    = E;  pos→1
  pos=1 (odd):  H(D, E)    = Root ✓

Verify(tx4, index=5, [tx4,C,D]):   ← phantom
  pos=5 (odd):  H(tx4, tx4) = C;  pos→2   ← same hash, different parity, same result
  pos=2 (even): H(C, C)    = E;  pos→1
  pos=1 (odd):  H(D, E)    = Root ✓
```

Both calls produce `Root`. The function has no guard on `tx_index < actual_tx_count`. [2](#0-1) 

The only guard present is `require!(!args.merkle_proof.is_empty())`. There is no upper-bound check on `tx_index`.

The v2 coinbase check does not prevent this: [3](#0-2) 

The coinbase proof is verified independently at index `0`. It passes correctly for any valid block, and its validity is entirely orthogonal to the phantom `tx_index=N` claim. After the coinbase check passes, the function delegates to `verify_transaction_inclusion` with the attacker-controlled `tx_index`. [4](#0-3) 

---

### Impact Explanation

`verify_transaction_inclusion_v2` returns `true` for a transaction at index `N` that does not exist in the block. Any downstream contract or application that relies on this function to confirm transaction inclusion — e.g., to release funds, mint tokens, or record a settlement — can be deceived using a real transaction hash (`tx[N-1]`) at a fabricated index. The attacker does not need any privileged role; the function is a public `view` call.

---

### Likelihood Explanation

- The attack requires only a canonical block with an odd transaction count (extremely common on any Bitcoin-family chain).
- The attacker needs no special keys, relayer access, or DAO approval.
- The proof construction is deterministic and computable offline.
- The coinbase mitigation added in v2 does not address this vector at all.

---

### Recommendation

Add an explicit upper-bound check on `tx_index` before computing the merkle root. Since the block header does not store the transaction count, the proof length encodes the tree depth; the maximum valid index is `2^proof_len - 1`. A tighter fix is to reject any `tx_index` that, when traced through the proof, would land on a position that only exists due to odd-leaf duplication. The most robust fix is to store or pass the transaction count and assert `tx_index < tx_count`.

pub fn merkle_proof_calculator(tx_hashes: Vec<H256>, transaction_position: usize) -> Vec<H256> {
    let mut transaction_position = transaction_position;
    let mut merkle_proof = Vec::new();
    let mut current_hashes = tx_hashes;

    while current_hashes.len() > 1 {
        if current_hashes.len() % 2 == 1 {
            current_hashes.push(current_hashes[current_hashes.len() - 1].clone());
        }

        if transaction_position % 2 == 1 {
            merkle_proof.push(current_hashes[transaction_position - 1].clone());
        } else {
            merkle_proof.push(current_hashes[transaction_position + 1].clone());
        }

        let mut new_hashes = Vec::new();

        for i in (0..current_hashes.len() - 1).step_by(2) {
            new_hashes.push(compute_hash(&current_hashes[i], &current_hashes[i + 1]));
        }

        current_hashes = new_hashes;
        transaction_position /= 2;
    }

    merkle_proof
}

pub fn compute_root_from_merkle_proof(
    transaction_hash: H256,
    transaction_position: usize,
    merkle_proof: &Vec<H256>,
) -> H256 {
    let mut current_hash = transaction_hash;
    let mut current_position = transaction_position;

    for proof_hash in merkle_proof {
        if current_position % 2 == 0 {
            current_hash = compute_hash(&current_hash, proof_hash);
        } else {
            current_hash = compute_hash(proof_hash, &current_hash);
        }
        current_position /= 2;
    }

    current_hash
}
