### Title
`getCoreRisk` Always Returns `longWeight`, Causing Inflated Spread Health Bonus — (`core/contracts/BaseEngine.sol` / `core/contracts/libraries/RiskHelper.sol`)

---

### Summary

`BaseEngine.getCoreRisk` hardcodes `1` as the `amount` argument to `_getWeightX18`, so `CoreRisk.longWeight` is always the **long-side** risk weight regardless of the actual position direction. The spread health adjustment in `Clearinghouse.getHealth` then uses this field for both legs of the spread — including the leg that is actually **short** — producing an inflated health bonus for every spread position.

---

### Finding Description

`BaseEngine.getCoreRisk` constructs a `CoreRisk` struct and always passes the literal `1` to `_getWeightX18`: [1](#0-0) 

```solidity
return IProductEngine.CoreRisk(
    amount,
    risk.priceX18,
    RiskHelper._getWeightX18(risk, 1, healthType)   // ← always 1, always longWeight
);
```

`_getWeightX18` branches on the sign of `amount`: [2](#0-1) 

Because `1 > 0`, it always returns `longWeightInitialX18` or `longWeightMaintenanceX18`, never the short-side weight. The `CoreRisk` struct only carries a single `longWeight` field: [3](#0-2) 

`Clearinghouse.getHealth` enforces that the two legs of a spread are on **opposite sides** (one long, one short): [4](#0-3) 

It then computes `existingWeight` and calls `_getSpreadWeightX18`, both of which use `CoreRisk.longWeight` for both legs: [5](#0-4) 

`_getSpreadWeightX18` also uses `longWeight` for the leg that is actually short: [6](#0-5) 

**Concrete numeric example** (INITIAL health, typical weights: `longWeight = 0.9`, `shortWeight = 1.1`):

| | Correct | Actual code |
|---|---|---|
| `existingWeight` | `(0.9 + 1.1) / 2 = 1.0` | `(0.9 + 0.9) / 2 = 0.9` |
| `spreadWeight` (perp short) | `1 − (1 − 1.1)/5 = 1.02` | `1 − (1 − 0.9)/5 = 0.98` |
| `spreadWeight − existingWeight` | `1.02 − 1.0 = 0.02` | `0.98 − 0.9 = 0.08` |

The health delta applied to the subaccount is **4× larger than intended**, directly inflating the health score of every spread holder.

---

### Impact Explanation

The spread health adjustment `basisAmount × (spotPrice + perpPrice) × (spreadWeight − existingWeight)` is overstated for every spread position. A user holding a long-spot / short-perp (or short-spot / long-perp) spread receives more health credit than the risk model intends. This allows:

1. **Excess leverage** — a user can open a larger spread position than the protocol's risk parameters permit.
2. **Liquidation bypass** — a spread position that should be below the maintenance threshold may appear healthy, blocking liquidators from acting.

The corrupted state is the subaccount's health score, which gates every collateral withdrawal, new order, and liquidation check.

---

### Likelihood Explanation

Spread positions are a first-class, explicitly supported feature (the `spreads` bitmap is set at initialization and iterated in every `getHealth` call). Any unprivileged user who opens a matched long-spot / short-perp position triggers the inflated calculation. No special role or privileged access is required.

---

### Recommendation

`getCoreRisk` must pass the **actual position amount** (not the hardcoded `1`) to `_getWeightX18`, so that `CoreRisk.longWeight` reflects the correct directional weight:

```solidity
// BaseEngine.sol – getCoreRisk
(int128 amount, ) = _getBalance(productId, subaccount);
return IProductEngine.CoreRisk(
    amount,
    risk.priceX18,
    RiskHelper._getWeightX18(risk, amount, healthType)  // use actual amount
);
```

Alternatively, rename the field to `weight` and document that it is direction-aware, and update `_getSpreadWeightX18` and the `existingWeight` computation in `Clearinghouse.getHealth` accordingly.

---

### Proof of Concept

1. Alice opens a spread: long 1 BTC-SPOT (productId = `_spotId`) and short 1 BTC-PERP (productId = `_perpId`).
2. `Clearinghouse.getHealth` is called (e.g., during a withdrawal or liquidation check).
3. `perpEngine.getCoreRisk(alice, perpId, INITIAL)` returns `CoreRisk{amount: -1, price: P, longWeight: 0.9}` — the short weight `1.1` is never used.
4. `existingWeight = (0.9 + 0.9) / 2 = 0.9` instead of the correct `(0.9 + 1.1) / 2 = 1.0`.
5. `_getSpreadWeightX18` computes `spreadWeight = 0.98` instead of `1.02`.
6. The health delta is `1 × 2P × 0.08 = 0.16P` instead of the correct `1 × 2P × 0.02 = 0.04P`.
7. Alice's health is overstated by `0.12P` per unit of spread, allowing her to withdraw collateral or avoid liquidation that the protocol's risk parameters would otherwise prevent. [7](#0-6) [8](#0-7) [6](#0-5)

    function _getWeightX18(
        Risk memory risk,
        int128 amount,
        IProductEngine.HealthType healthType
    ) internal pure returns (int128) {
        if (healthType == IProductEngine.HealthType.PNL) {
            return ONE;
        }

        int128 weight;
        if (amount >= 0) {
            weight = healthType == IProductEngine.HealthType.INITIAL
                ? risk.longWeightInitialX18
                : risk.longWeightMaintenanceX18;
        } else {
            weight = healthType == IProductEngine.HealthType.INITIAL
                ? risk.shortWeightInitialX18
                : risk.shortWeightMaintenanceX18;
        }

        return weight;
    }
```

    function _getSpreadWeightX18(
        IProductEngine.CoreRisk memory perpCoreRisk,
        IProductEngine.CoreRisk memory spotCoreRisk,
        IProductEngine.HealthType healthType
    ) internal pure returns (int128) {
        if (healthType == IProductEngine.HealthType.PNL) {
            return ONE;
        }
        int128 spreadWeight;
        if (spotCoreRisk.amount > 0) {
            spreadWeight = ONE - (ONE - perpCoreRisk.longWeight) / 5;
        } else {
            spreadWeight = ONE - (ONE - spotCoreRisk.longWeight) / 5;
        }
        int128 maxSpreadWeight;
        if (healthType == IProductEngine.HealthType.INITIAL) {
            maxSpreadWeight = ONE - ONE / 100; // 0.99
        } else {
            maxSpreadWeight = ONE - (ONE * 6) / 1000; // 0.994
        }
        if (spreadWeight > maxSpreadWeight) {
            spreadWeight = maxSpreadWeight;
        }
        return spreadWeight;
    }
```
