### Title
Partial Redelegation Slash With Empty Bonus Pool Permanently Forfeits Accrued Bonus — (`x/tieredrewards/keeper/slash.go`, `x/tieredrewards/keeper/claim_rewards.go`)

---

### Summary

`slashRedelegationPosition` calls `processEventsAndClaimBonus`, which advances all three bonus checkpoints (`LastBonusAccrual`, `LastEventSeq`, `LastKnownBonded`) on the in-memory `pos` object **before** it checks whether the pool can cover the computed bonus. When the pool is empty the function returns `ErrInsufficientBonusPool`. `slashRedelegationPosition` deliberately swallows that error (to prevent a chain halt), then calls `setPositionWithState` with the already-mutated `pos`, persisting the advanced checkpoints. The accrued bonus for the entire period since the last claim is permanently forfeited with no compensation to the position owner.

---

### Finding Description

**Step 1 — Checkpoints are advanced before the pool check inside `processEventsAndClaimBonus`.** [1](#0-0) 

Inside the event loop `pos.UpdateLastEventSeq(entry.Seq)` is called for every pending event, advancing `LastEventSeq` on the in-memory `pos`. [2](#0-1) 

After the loop, `applyBonusAccrualCheckpoint` advances `LastBonusAccrual` to `blockTime` and `UpdateLastKnownBonded` updates `LastKnownBonded` — both mutations happen unconditionally, before the pool balance is ever tested. [3](#0-2) 

Only then is `sufficientBonusPoolBalance` called. If the pool is empty it returns `ErrInsufficientBonusPool`, but the three checkpoint fields on `pos` are already mutated.

**Step 2 — `slashRedelegationPosition` swallows the error and persists the mutated position.** [4](#0-3) 

`ErrInsufficientBonusPool` is caught, logged, and execution continues. The `pos` variable still carries the advanced checkpoints written by `processEventsAndClaimBonus`. [5](#0-4) 

For a **partial slash** (`sharesToUnbond < pos.Delegation.Shares`), `setPositionWithState` is called with the mutated `pos`, writing the advanced `LastBonusAccrual`, `LastEventSeq`, and `LastKnownBonded` to the store. No bonus was ever paid.

*(For a full slash the subsequent `pos.ClearBonusCheckpoints()` at line 70 overwrites the advanced values, so the full-slash path is not affected.)*

**Step 3 — The bonus is permanently lost.**

On the next claim the position's `LastBonusAccrual` already equals the slash block time, so `computeSegmentBonus` returns zero for the entire forfeited period. There is no deferred-payment record, no retry mechanism, and no way for the owner to recover the lost bonus. [6](#0-5) 

---

### Impact Explanation

Every position owner whose redelegation is partially slashed while `RewardsPoolName` holds less than the computed bonus loses all bonus accrued since their last checkpoint. The loss is proportional to `shares × tokensPerShare × BonusAPY × elapsed_time`. For a position that has been delegated for months without claiming, this can be a material amount. The funds are not redistributed — they remain in the pool and accrue to no one for the forfeited period.

---

### Likelihood Explanation

The pool can reach zero through normal operation: a large simultaneous claim by many positions, a governance decision to stop refilling the pool, or a deliberate griefing strategy where an actor with many positions drains the pool immediately before a known redelegation slash fires. Redelegation slashes are a standard Cosmos SDK event triggered by double-signing on the source validator during the redelegation unbonding window. No privileged access is required; the pool state and the slash event are both reachable through normal production paths.

---

### Recommendation

Move `applyBonusAccrualCheckpoint` and `UpdateLastKnownBonded` to **after** the successful `SendCoinsFromModuleToAccount` call, so checkpoints are only advanced when the bonus is actually paid. Alternatively, if the "forgo and continue" policy is intentional, record the unpaid bonus amount in a separate store entry keyed by position ID so the owner can claim it once the pool is refunded.

func (s *KeeperSuite) TestSlashRedelegation_EmptyPool_ForfeitsBonus() {
    lockAmount := sdkmath.NewInt(sdk.DefaultPowerReduction.Int64())
    _, bondDenom := s.getStakingData()
    // Do NOT fund the rewards pool — leave it at zero.

    pos, _, unbondingID := s.setupRedelegatingPosition(lockAmount)
    owner := sdk.MustAccAddressFromBech32(pos.Owner)
    accrualBefore := pos.LastBonusAccrual

    // Advance 30 days so a non-zero bonus accrues.
    s.ctx = s.ctx.WithBlockTime(s.ctx.BlockTime().Add(30 * 24 * time.Hour))

    balBefore := s.app.BankKeeper.GetBalance(s.ctx, owner, bondDenom)

    // Partial slash — pool is empty, ErrInsufficientBonusPool is swallowed.
    sharesToUnbond := pos.Delegation.Shares.Quo(sdkmath.LegacyNewDec(2))
    err := s.keeper.Hooks().BeforeRedelegationSlashed(s.ctx, unbondingID, sharesToUnbond)
    s.Require().NoError(err)

    balAfter := s.app.BankKeeper.GetBalance(s.ctx, owner, bondDenom)
    // Owner received nothing.
    s.Require().True(balAfter.Amount.Equal(balBefore.Amount), "no bonus paid")

    updated, err := s.keeper.GetPositionState(s.ctx, pos.Id)
    s.Require().NoError(err)
    // Checkpoint was advanced despite no payment — bonus is permanently lost.
    s.Require().True(updated.LastBonusAccrual.After(accrualBefore),
        "LastBonusAccrual advanced even though no bonus was paid: accrued period forfeited")

    // Now fund the pool and claim — yields zero because checkpoint already advanced.
    s.fundRewardsPool(sdkmath.NewInt(1_000_000_000), bondDenom)
    updated, _ = s.keeper.GetPositionState(s.ctx, updated.Id)
    bonus, err := s.keeper.ProcessEventsAndClaimBonus(s.ctx, &updated)
    s.Require().NoError(err)
    s.Require().True(bonus.IsZero(), "forfeited period cannot be reclaimed")
}

func applyBonusAccrualCheckpoint(pos *types.Position, blockTime time.Time) {
	accrualEnd := blockTime
	if pos.CompletedExitLockDuration(blockTime) {
		accrualEnd = pos.ExitUnlockAt
	}
	pos.UpdateLastBonusAccrual(accrualEnd)
}
