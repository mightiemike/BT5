### Title
Protobuf `ValidResourceBounds` Round-Trip Silently Downgrades `AllResources` to `L1Gas` When l2_gas and l1_data_gas Are Zero, Causing Consensus Validators to Reject Valid Proposals — (`File: crates/apollo_protobuf/src/converters/transaction.rs`)

---

### Summary

The `TryFrom<protobuf::ResourceBounds> for ValidResourceBounds` conversion uses a **value-based heuristic** to select the enum variant: if the deserialized `l1_data_gas` and `l2_gas` are both zero, it produces `ValidResourceBounds::L1Gas` regardless of whether the sender originally used `AllResources`. Because the gateway accepts invoke transactions with `AllResourceBounds { l1_gas: X, l2_gas: 0, l1_data_gas: 0 }` and computes their hash under the `AllResources` domain (3-element Poseidon chain), but the consensus P2P deserialization path silently converts those bounds to `L1Gas`, the subsequent `TryFrom<InvokeTransactionV3> for RpcInvokeTransactionV3` conversion fails with `DEPRECATED_RESOURCE_BOUNDS_ERROR`. Validators reject the entire proposal, breaking consensus liveness.

---

### Finding Description

**Root cause — value-based variant selection in protobuf deserialization:** [1](#0-0) 

```rust
let l1_data_gas = value.l1_data_gas.unwrap_or_default();   // None → zero
...
Ok(if l1_data_gas.is_zero() && l2_gas.is_zero() {
    ValidResourceBounds::L1Gas(l1_gas)          // ← wrong for AllResources with zero values
} else {
    ValidResourceBounds::AllResources(AllResourceBounds { l1_gas, l2_gas, l1_data_gas })
})
```

The protobuf field `l1_data_gas` is declared `optional`: [2](#0-1) 

When a proposer serializes `ValidResourceBounds::AllResources { l1_gas: X, l2_gas: 0, l1_data_gas: 0 }`, the serializer emits `l1_data_gas = Some(zero)`: [3](#0-2) 

On the validator side, `unwrap_or_default()` collapses `Some(zero)` and `None` to the same zero value, so the heuristic fires and produces `L1Gas` — a different variant than the proposer used.

**Conversion failure:**

The consensus deserialization path for invoke transactions is: [4](#0-3) 

```rust
let snapi_invoke: InvokeTransactionV3 = value.invoke...?.try_into()?;
// This conversion can fail only if the resource_bounds are not AllResources.
Ok(Self { proof, ..snapi_invoke.try_into().map_err(|_| DEPRECATED_RESOURCE_BOUNDS_ERROR)? })
```

When `snapi_invoke.resource_bounds` is `L1Gas` (produced by the heuristic), `snapi_invoke.try_into::<RpcInvokeTransactionV3>()` returns `Err(DEPRECATED_RESOURCE_BOUNDS_ERROR)`, propagating up through: [5](#0-4) 

causing the entire `ConsensusTransaction` deserialization to fail.

**Hash domain divergence (secondary):**

The hash function includes `L1_DATA_GAS` only for `AllResources`: [6](#0-5) 

The gateway always computes the hash under `AllResources` (3-element chain) because `InternalRpcInvokeTransactionV3` stores `AllResourceBounds`: [7](#0-6) 

If a validator were to accept the transaction after the downgrade, it would compute a different hash (2-element chain for `L1Gas`), binding the wrong hash to the executable payload.

---

### Impact Explanation

A validator receiving a consensus proposal that contains a valid invoke transaction with `AllResourceBounds { l1_gas: X, l2_gas: 0, l1_data_gas: 0 }` fails to deserialize it. The `DEPRECATED_RESOURCE_BOUNDS_ERROR` propagates out of `TryFrom<protobuf::ConsensusTransaction>`, causing the validator to reject the proposal. This breaks consensus liveness for any block containing such a transaction.

Matches: **High — Transaction conversion logic binds the wrong type/executable payload.**

---

### Likelihood Explanation

The gateway's stateless validator explicitly accepts transactions with zero l2_gas and l1_data_gas: [8](#0-7) 

Any unprivileged user can submit such a transaction. Once it enters the mempool and is included in a block proposal, the liveness failure is deterministic.

---

### Recommendation

Replace the value-based heuristic with a structural check on the presence of the optional `l1_data_gas` field:

```rust
impl TryFrom<protobuf::ResourceBounds> for ValidResourceBounds {
    fn try_from(value: protobuf::ResourceBounds) -> Result<Self, Self::Error> {
        let l1_gas: ResourceBounds = value.l1_gas.ok_or(missing("ResourceBounds::l1_gas"))?.try_into()?;
        let l2_gas: ResourceBounds = value.l2_gas.ok_or(missing("ResourceBounds::l2_gas"))?.try_into()?;
        Ok(match value.l1_data_gas {
            // l1_data_gas absent → legacy L1Gas transaction (pre-0.13.3)
            None => {
                if !l2_gas.is_zero() {
                    return Err(missing("ResourceBounds::l1_data_gas"));
                }
                ValidResourceBounds::L1Gas(l1_gas)
            }
            // l1_data_gas present (even if zero) → AllResources transaction
            Some(l1_data_gas) => {
                let l1_data_gas: ResourceBounds = l1_data_gas.try_into()?;
                ValidResourceBounds::AllResources(AllResourceBounds { l1_gas, l2_gas, l1_data_gas })
            }
        })
    }
}
```

This preserves the `AllResources` variant when `l1_data_gas` is explicitly set to zero, matching the serializer's behavior.

---

### Proof of Concept

1. Craft an invoke V3 transaction with `resource_bounds = AllResourceBounds { l1_gas: { max_amount: 1000, max_price_per_unit: 1 }, l2_gas: { 0, 0 }, l1_data_gas: { 0, 0 } }`.
2. Submit to the gateway — accepted (stateless validator passes, hash H₁ computed under `AllResources` 3-element Poseidon chain).
3. Transaction enters mempool; proposer includes it in a block proposal and broadcasts via consensus P2P.
4. Validator deserializes `protobuf::ResourceBounds { l1_gas: Some(X), l2_gas: Some(0), l1_data_gas: Some(0) }`:
   - `l1_data_gas.unwrap_or_default()` → zero
   - `l1_data_gas.is_zero() && l2_gas.is_zero()` → `true`
   - Result: `ValidResourceBounds::L1Gas(X)` ← wrong variant
5. `snapi_invoke.try_into::<RpcInvokeTransactionV3>()` returns `Err(DEPRECATED_RESOURCE_BOUNDS_ERROR)`.
6. `TryFrom<protobuf::ConsensusTransaction>` returns `Err(...)`.
7. Validator rejects the proposal; consensus stalls for this block height.

// An implementation of the SNIP: https://github.com/EvyatarO/SNIPs/blob/snip-8/SNIPS/snip-8.md
pub fn get_tip_resource_bounds_hash(
    resource_bounds: &ValidResourceBounds,
    tip: &Tip,
) -> Result<Felt, StarknetApiError> {
    let l1_resource_bounds = resource_bounds.get_l1_bounds();
    let l2_resource_bounds = resource_bounds.get_l2_bounds();

    // L1 and L2 gas bounds always exist.
    // Old V3 txs always have L2 gas bounds of zero, but they exist.
    let mut resource_felts = vec![
        get_concat_resource(&l1_resource_bounds, L1_GAS)?,
        get_concat_resource(&l2_resource_bounds, L2_GAS)?,
    ];

    // For new V3 txs, need to also hash the data gas bounds.
    resource_felts.extend(match resource_bounds {
        ValidResourceBounds::L1Gas(_) => vec![],
        ValidResourceBounds::AllResources(all_resources) => {
            vec![get_concat_resource(&all_resources.l1_data_gas, L1_DATA_GAS)?]
        }
    });

    Ok(HashChain::new().chain(&tip.0.into()).chain_iter(resource_felts.iter()).get_poseidon_hash())
}
