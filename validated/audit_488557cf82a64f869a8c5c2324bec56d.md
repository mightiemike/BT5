### Title
`ValidResourceBounds` Variant Collapse in Protobuf Deserialization Causes P2P Rejection of Valid V3 Declare Transactions and Transaction Hash Divergence — (`File: crates/apollo_protobuf/src/converters/transaction.rs`)

### Summary

A V3 declare transaction submitted with `AllResourceBounds { l1_gas: X, l2_gas: 0, l1_data_gas: 0 }` is accepted by the gateway and assigned a canonical hash computed over three resource-bound fields. When the same transaction is later propagated over the P2P mempool layer, the protobuf deserializer silently collapses the `AllResources` variant into `ValidResourceBounds::L1Gas`, which (a) causes an immediate hard rejection in the `RpcDeclareTransactionV3` converter and (b) would produce a different transaction hash if the conversion were allowed to proceed, breaking signature-domain canonicalization.

### Finding Description

**Root cause — variant collapse in protobuf deserialization**

`crates/apollo_protobuf/src/converters/transaction.rs` lines 431–435:

```rust
Ok(if l1_data_gas.is_zero() && l2_gas.is_zero() {
    ValidResourceBounds::L1Gas(l1_gas)
} else {
    ValidResourceBounds::AllResources(AllResourceBounds { l1_gas, l2_gas, l1_data_gas })
})
```

When a `protobuf::ResourceBounds` message carries `l2_gas = 0` and `l1_data_gas = 0` (or an absent `l1_data_gas` field, which is defaulted to zero via `unwrap_or_default` on line 427), the converter silently produces `ValidResourceBounds::L1Gas` regardless of whether the sender originally used the `AllResources` variant. [1](#0-0) 

**Downstream rejection in the P2P mempool path**

`DeclareTransactionV3Common` is constructed from `protobuf::DeclareV3Common` using the above converter. It is then consumed by `TryFrom<protobuf::DeclareV3WithClass> for RpcDeclareTransactionV3`, which explicitly rejects the `L1Gas` variant:

```rust
resource_bounds: match common.resource_bounds {
    ValidResourceBounds::AllResources(resource_bounds) => resource_bounds,
    _ => {
        return Err(DEPRECATED_RESOURCE_BOUNDS_ERROR);
    }
},
``` [2](#0-1) 

Any V3 declare transaction whose `l2_gas` and `l1_data_gas` are both zero is therefore unconditionally rejected by every peer that receives it over P2P, even though the gateway accepted it.

**Hash-domain divergence if the conversion were allowed**

`get_tip_resource_bounds_hash` in `crates/starknet_api/src/transaction_hash.rs` produces structurally different hash preimages for the two variants:

- `L1Gas(X)` → Poseidon of `[tip, concat(L1_GAS, X), concat(L2_GAS, 0)]` — **2 resource felts**
- `AllResources { l1_gas: X, l2_gas: 0, l1_data_gas: 0 }` → Poseidon of `[tip, concat(L1_GAS, X), concat(L2_GAS, 0), concat(L1_DATA_GAS, 0)]` — **3 resource felts** [3](#0-2) 

The gateway always wraps `InternalRpcDeclareTransactionV3::resource_bounds: AllResourceBounds` in `ValidResourceBounds::AllResources` before hashing: [4](#0-3) 

So the canonical hash H₁ (three-element preimage) stored in the mempool and signed by the user differs from the hash H₂ (two-element preimage) that any peer would compute after the variant collapse. Even if the `DEPRECATED_RESOURCE_BOUNDS_ERROR` guard were removed, the peer would store the transaction under the wrong hash, and the account's `__validate__` call would fail because the sequencer would pass H₂ to the contract while the user signed H₁.

**The gateway accepts the triggering input**

The gateway's stateless validator accepts `AllResourceBounds` with zero L2 and L1DataGas bounds (only L1 gas non-zero is explicitly tested as valid): [5](#0-4) 

### Impact Explanation

Any user who submits a V3 declare transaction specifying only an L1 gas bound (setting L2 gas and L1DataGas to zero) triggers the issue. The gateway accepts the transaction and assigns it hash H₁. Every peer that receives the transaction via the P2P mempool layer immediately rejects it with `DEPRECATED_RESOURCE_BOUNDS_ERROR`. The transaction is therefore confined to the single node that received it via RPC. If that node proposes a block containing the transaction, peers performing block-level hash validation will compute H₂ ≠ H₁ and reject the block, creating a potential chain split.

This matches: **High — Mempool/gateway/RPC admission accepts invalid transactions or rejects valid transactions before sequencing**, and potentially **Critical — Wrong state or revert result** if the hash divergence reaches execution.

### Likelihood Explanation

The trigger is an ordinary, unprivileged RPC submission. No special permissions or malformed bytes are required. Any wallet or SDK that submits a V3 declare transaction without explicitly setting L2 gas or L1DataGas bounds (defaulting them to zero) will hit this path. The condition `l2_gas.is_zero() && l1_data_gas.is_zero()` is common for users who have not yet migrated to the full three-resource model.

### Recommendation

1. **Preserve the `AllResources` variant unconditionally for V3 transactions.** The `TryFrom<protobuf::ResourceBounds> for ValidResourceBounds` converter should not collapse to `L1Gas` when the wire format carries explicit (even if zero-valued) L2 and L1DataGas fields. The collapse should only occur for legacy pre-0.13.3 messages that genuinely omit those fields.

2. **Align the protobuf serializer.** When serializing `ValidResourceBounds::AllResources`, always emit all three `ResourceLimits` sub-messages so that the round-trip is lossless.

3. **Add a round-trip hash invariant test** asserting that `hash(deserialize(serialize(tx))) == hash(tx)` for V3 transactions with zero L2/L1DataGas bounds.

### Proof of Concept

```
1. Submit RPC declare V3 transaction:
     resource_bounds = AllResourceBounds {
         l1_gas:      { max_amount: 1000, max_price_per_unit: 1 },
         l2_gas:      { max_amount: 0,    max_price_per_unit: 0 },
         l1_data_gas: { max_amount: 0,    max_price_per_unit: 0 },
     }

2. Gateway accepts; stores InternalRpcDeclareTransactionV3 with
   resource_bounds = AllResourceBounds { ... }.
   Hash H₁ = get_declare_transaction_v3_hash(AllResources { ... })
             includes concat(L1_DATA_GAS, 0) in the Poseidon preimage.

3. Mempool P2P serializes the transaction as protobuf DeclareV3WithClass.
   Wire bytes: l1_gas=1000/1, l2_gas=0/0, l1_data_gas=0/0.

4. Peer deserializes:
     ValidResourceBounds::try_from(proto_resource_bounds)
     → l1_data_gas.is_zero() && l2_gas.is_zero() == true
     → returns ValidResourceBounds::L1Gas(l1_gas)

5. RpcDeclareTransactionV3 converter hits:
     _ => return Err(DEPRECATED_RESOURCE_BOUNDS_ERROR)
   Transaction rejected. Peer never adds it to its mempool.

6. Hash divergence (if guard removed):
     H₂ = get_declare_transaction_v3_hash(L1Gas { ... })
          omits concat(L1_DATA_GAS, 0) from the Poseidon preimage.
     H₁ ≠ H₂  →  __validate__ receives wrong hash  →  signature check fails.
```

impl TryFrom<protobuf::ResourceBounds> for ValidResourceBounds {
    type Error = ProtobufConversionError;
    fn try_from(value: protobuf::ResourceBounds) -> Result<Self, Self::Error> {
        let Some(l1_gas) = value.l1_gas else {
            return Err(missing("ResourceBounds::l1_gas"));
        };
        let Some(l2_gas) = value.l2_gas else {
            return Err(missing("ResourceBounds::l2_gas"));
        };
        // TODO(Shahak): Assert data gas is not none once we remove support for 0.13.2.
        let l1_data_gas = value.l1_data_gas.unwrap_or_default();
        let l1_gas: ResourceBounds = l1_gas.try_into()?;
        let l2_gas: ResourceBounds = l2_gas.try_into()?;
        let l1_data_gas: ResourceBounds = l1_data_gas.try_into()?;
        Ok(if l1_data_gas.is_zero() && l2_gas.is_zero() {
            ValidResourceBounds::L1Gas(l1_gas)
        } else {
            ValidResourceBounds::AllResources(AllResourceBounds { l1_gas, l2_gas, l1_data_gas })
        })
    }

// An implementation of the SNIP: https://github.com/EvyatarO/SNIPs/blob/snip-8/SNIPS/snip-8.md
pub fn get_tip_resource_bounds_hash(
    resource_bounds: &ValidResourceBounds,
    tip: &Tip,
) -> Result<Felt, StarknetApiError> {
    let l1_resource_bounds = resource_bounds.get_l1_bounds();
    let l2_resource_bounds = resource_bounds.get_l2_bounds();

    // L1 and L2 gas bounds always exist.
    // Old V3 txs always have L2 gas bounds of zero, but they exist.
    let mut resource_felts = vec![
        get_concat_resource(&l1_resource_bounds, L1_GAS)?,
        get_concat_resource(&l2_resource_bounds, L2_GAS)?,
    ];

    // For new V3 txs, need to also hash the data gas bounds.
    resource_felts.extend(match resource_bounds {
        ValidResourceBounds::L1Gas(_) => vec![],
        ValidResourceBounds::AllResources(all_resources) => {
            vec![get_concat_resource(&all_resources.l1_data_gas, L1_DATA_GAS)?]
        }
    });

    Ok(HashChain::new().chain(&tip.0.into()).chain_iter(resource_felts.iter()).get_poseidon_hash())
}
