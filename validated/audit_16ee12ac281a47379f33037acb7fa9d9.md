### Title
Deprecated `verify_transaction_inclusion` Remains Publicly Callable, Bypassing 64-Byte Internal Node Mitigation — (`contract/src/lib.rs`)

### Summary

`verify_transaction_inclusion` (v1) is marked `#[deprecated]` in Rust but carries **no NEAR access restriction** (`#[private]` is absent). Any external caller can invoke it directly, bypassing the coinbase-proof guard added in `verify_transaction_inclusion_v2`. An attacker can supply an internal Merkle node as `tx_id` with a proof of depth D-1 and receive `true`, falsely confirming inclusion of a non-existent transaction.

---

### Finding Description

`verify_transaction_inclusion_v2` was introduced to mitigate the [64-byte transaction Merkle proof forgery](https://www.bitmex.com/blog/64-Byte-Transactions) by requiring a coinbase proof of the same depth before delegating to v1: [1](#0-0) 

The coinbase-depth check enforces that the proof depth matches a real leaf-level proof. However, v1 remains a fully public `#[pause]`-gated method with no `#[private]` restriction: [2](#0-1) 

Rust's `#[deprecated]` attribute emits a **compiler warning only** — it does not restrict runtime callability from external NEAR accounts. The function's own doc-comment acknowledges the gap: [3](#0-2) 

The root computation in v1 is a plain Merkle path walk with no leaf-vs-node distinction: [4](#0-3) 

For a tree of depth D, an internal node H at level 1 (one step below the root) has exactly one sibling. Supplying `tx_id = H`, `tx_index` matching H's position, and `merkle_proof = [sibling]` (length 1) causes `compute_root_from_merkle_proof` to return the real `merkle_root`, so the function returns `true`.

---

### Impact Explanation

Any downstream NEAR contract that calls `verify_transaction_inclusion` (v1) to gate asset release receives a `true` result for a fabricated "transaction" that is actually an internal Merkle node from a legitimately stored block. This satisfies the scoped critical impact: **false transaction inclusion confirmation enabling downstream asset release without coinbase proof validation**.

---

### Likelihood Explanation

- No special privileges are required; the function is open to any NEAR account.
- The attacker only needs a block already stored in the light client (publicly available on-chain) and its Merkle tree structure (derivable from public Bitcoin data).
- The attack is deterministic and requires no mining or chain manipulation.

---

### Recommendation

Add `#[private]` to `verify_transaction_inclusion` to restrict it to internal cross-contract calls only, or remove it entirely from the public ABI. All external callers must be migrated to `verify_transaction_inclusion_v2`. Alternatively, add the same coinbase-depth guard directly inside v1 so the mitigation cannot be bypassed regardless of which entry point is used.

---

### Proof of Concept

```rust
// In a near-sdk integration test (bitcoin feature, skip_pow=true):
// 1. Initialize contract with a real block whose Merkle tree has ≥2 txs.
// 2. Compute internal node H = double_sha256(tx0 || tx1).
// 3. The merkle_root = H (for a 2-tx block, H IS the root — use a deeper tree).
//    For depth D=2: H_internal = double_sha256(tx0||tx1); root = double_sha256(H_internal||H_internal).
//    Supply tx_id=H_internal, tx_index=0, merkle_proof=[H_internal] (sibling = itself, odd tree).
// 4. Call verify_transaction_inclusion directly (not v2).
// 5. Assert returns true — confirmed bypass of coinbase mitigation.
```

The call sequence is exactly:

```
verify_transaction_inclusion(ProofArgs { tx_id: H_internal, tx_block_blockhash: <valid>, tx_index: 0, merkle_proof: [sibling], confirmations: 1 })
  → compute_root_from_merkle_proof(H_internal, 0, [sibling]) == merkle_root
  → true
``` [5](#0-4)

pub fn compute_root_from_merkle_proof(
    transaction_hash: H256,
    transaction_position: usize,
    merkle_proof: &Vec<H256>,
) -> H256 {
    let mut current_hash = transaction_hash;
    let mut current_position = transaction_position;

    for proof_hash in merkle_proof {
        if current_position % 2 == 0 {
            current_hash = compute_hash(&current_hash, proof_hash);
        } else {
            current_hash = compute_hash(proof_hash, &current_hash);
        }
        current_position /= 2;
    }

    current_hash
}
