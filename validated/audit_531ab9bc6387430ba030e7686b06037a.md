### Title
Any Unprivileged Account Can Front-Run NFT Denom Registration and Permanently Block Legitimate Creators - (File: `x/nft/keeper/msg_server.go`, `x/nft/keeper/denom.go`)

---

### Summary

`MsgIssueDenom` in the `x/nft` module carries no access-control check. Any account can call it with an arbitrary denom ID and name. Because `SetDenom` enforces global uniqueness on both `denomID` and `denomName`, a single front-running transaction permanently blocks the legitimate project from ever registering their intended denom. The attacker becomes the recorded `Creator` of the denom, which is the authorization gate for all subsequent minting under that namespace.

---

### Finding Description

`IssueDenom` in `x/nft/keeper/msg_server.go` accepts `msg.Sender` as the creator and immediately delegates to `m.Keeper.IssueDenom`: [1](#0-0) 

There is no check that `msg.Sender` is a privileged address, a governance-approved issuer, or any specific account. Any valid Cosmos address can submit this message.

`SetDenom` in `x/nft/keeper/denom.go` enforces uniqueness on both the ID and the name: [2](#0-1) 

Once either `denomID` or `denomName` is registered, the store entry is permanent and immutable — there is no `UpdateDenom` or admin-override path. The first caller wins, and all subsequent callers with the same ID or name are rejected with `ErrInvalidDenom`.

The `Creator` field stored in the denom record is the `msg.Sender` of the winning call: [3](#0-2) 

`IsDenomCreator` is the authorization gate used by `MintNFT` and other denom-scoped operations. Whoever registers the denom first owns the minting rights permanently.

---

### Impact Explanation

An attacker who front-runs a legitimate project's `MsgIssueDenom` transaction achieves two simultaneous effects:

1. **Permanent namespace hijack**: The legitimate project can never register their intended denom ID or name. The error `"denomID X has already exists"` is final — there is no recourse short of choosing a different ID.
2. **Creator authority seizure**: The attacker's address is stored as `denom.Creator`. All `MintNFT` calls under that denom require the sender to pass the `IsDenomCreator` check, so the attacker controls who can mint NFTs in that collection. The attacker can mint counterfeit NFTs under the hijacked denom and present them as originating from the legitimate project.

The corrupted on-chain value is `denom.Creator` — a string stored in the KV store under `types.KeyDenomID(id)`. [4](#0-3) 

---

### Likelihood Explanation

The attack requires only a funded Cosmos account and knowledge of the target denom ID (which is typically announced publicly before launch, e.g., in documentation, social media, or a pending governance proposal). The attacker pays only the standard transaction fee. Mempool front-running is straightforward on a CometBFT chain where transactions are broadcast publicly before inclusion. The cost is negligible; the impact is permanent.

---

### Recommendation

Restrict `MsgIssueDenom` to a governance-controlled whitelist or require a governance proposal to register a new denom, analogous to how `x25519` token factory modules gate denom creation. At minimum, add an admin address parameter to the module params (settable via governance) and enforce `msg.Sender == admin` in `IssueDenom` before calling `SetDenom`. [5](#0-4) 

---

### Proof of Concept

**Setup**: Legitimate project `alice` intends to issue denom ID `"my-nft-collection"` with name `"My NFT Collection"`.

**Attack**:
1. Attacker `eve` monitors the public mempool and observes alice's pending `MsgIssueDenom{Id: "my-nft-collection", Name: "My NFT Collection", Sender: alice}`.
2. `eve` broadcasts her own `MsgIssueDenom{Id: "my-nft-collection", Name: "My NFT Collection", Sender: eve}` with a higher fee to ensure priority inclusion.
3. `eve`'s transaction is included first. `SetDenom` writes `denom.Creator = eve` to `KeyDenomID("my-nft-collection")`. [6](#0-5) 

4. Alice's transaction is included next. `SetDenom` returns `ErrInvalidDenom: "denomID my-nft-collection has already exists"`. Alice's `IssueDenom` call fails permanently. [7](#0-6) 

5. `eve` is now the sole authorized minter under `"my-nft-collection"` and can mint counterfeit NFTs or ransom the denom back to alice. Alice must abandon her intended denom ID entirely.

func (m msgServer) IssueDenom(goCtx context.Context, msg *types.MsgIssueDenom) (*types.MsgIssueDenomResponse, error) {
	sender, err := sdk.AccAddressFromBech32(msg.Sender)
	if err != nil {
		return nil, err
	}

	ctx := sdk.UnwrapSDKContext(goCtx)
	if err := m.Keeper.IssueDenom(ctx, msg.Id, msg.Name, msg.Schema, msg.Uri, sender); err != nil {
		return nil, err
	}

	ctx.EventManager().EmitEvents(sdk.Events{
		sdk.NewEvent(
			types.EventTypeIssueDenom,
			sdk.NewAttribute(types.AttributeKeyDenomID, msg.Id),
			sdk.NewAttribute(types.AttributeKeyDenomName, msg.Name),
			sdk.NewAttribute(types.AttributeKeyCreator, msg.Sender),
		),
		sdk.NewEvent(
			sdk.EventTypeMessage,
			sdk.NewAttribute(sdk.AttributeKeyModule, types.AttributeValueCategory),
			sdk.NewAttribute(sdk.AttributeKeySender, msg.Sender),
		),
	})

	return &types.MsgIssueDenomResponse{}, nil
}

func (k Keeper) SetDenom(ctx sdk.Context, denom types.Denom) error {
	if k.HasDenomID(ctx, denom.Id) {
		return sdkerrors.Wrapf(types.ErrInvalidDenom, "denomID %s has already exists", denom.Id)
	}

	if k.HasDenomNm(ctx, denom.Name) {
		return sdkerrors.Wrapf(types.ErrInvalidDenom, "denomName %s has already exists", denom.Name)
	}

	store := ctx.KVStore(k.storeKey)
	bz := k.cdc.MustMarshal(&denom)
	store.Set(types.KeyDenomID(denom.Id), bz)
	store.Set(types.KeyDenomName(denom.Name), []byte(denom.Id))
	return nil

// IsDenomCreator checks if address is the creator of Denom
// Return the Denom if true, an error otherwise
func (k Keeper) IsDenomCreator(ctx sdk.Context, denomID string, address sdk.AccAddress) (types.Denom, error) {
	denom, err := k.GetDenom(ctx, denomID)
	if err != nil {
		return types.Denom{}, err
	}

	creator, err := sdk.AccAddressFromBech32(denom.Creator)
	if err != nil {
		panic(err)
	}

	if !creator.Equals(address) {
		return types.Denom{}, sdkerrors.Wrapf(types.ErrUnauthorized, "%s is not the creator of %s", address, denomID)
	}

	return denom, nil
