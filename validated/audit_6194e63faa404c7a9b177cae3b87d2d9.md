### Title
Silent Checkpoint Advance on Insufficient Bonus Pool During Redelegation Slash Permanently Loses Owner's Accrued Bonus Rewards — (`x/tieredrewards/keeper/slash.go`, `x/tieredrewards/keeper/claim_rewards.go`)

### Summary
When `BeforeRedelegationSlashed` fires and the `RewardsPoolName` module account has insufficient balance to cover accrued bonus rewards, `processEventsAndClaimBonus` advances the position's `LastEventSeq` and `LastBonusAccrual` checkpoints and decrements event reference counts before discovering the pool is empty. `slashRedelegationPosition` then swallows the `ErrInsufficientBonusPool` error and persists the mutated position. The owner's accrued bonus rewards are permanently unrecoverable.

### Finding Description

The root cause is a sequencing error in `processEventsAndClaimBonus`: [1](#0-0) 

The event loop advances `pos.LastEventSeq` and decrements event ref counts (potentially deleting events) for every processed event before the pool balance check at line 230: [2](#0-1) 

`applyBonusAccrualCheckpoint` (line 215) and `UpdateLastKnownBonded` (line 217) both mutate `pos` before `sufficientBonusPoolBalance` is called. If the pool is empty, `ErrInsufficientBonusPool` is returned with `pos` already fully checkpointed.

In `slashRedelegationPosition`, this error is explicitly caught and swallowed: [3](#0-2) 

The function then calls `setPositionWithState` with the mutated `pos`, persisting the advanced checkpoints: [4](#0-3) 

`setPositionWithState` writes the position to the store unconditionally: [5](#0-4) 

After this, the position's `LastEventSeq` points past all previously accrued events, `LastBonusAccrual` is set to the current block time, and the event ref counts have been decremented (events may be deleted). There is no way for the owner to ever claim the skipped bonus.

### Impact Explanation
Permanent loss of the position owner's accrued bonus rewards. The `RewardsPoolName` module account balance is not reduced (no payment was made), but the owner's entitlement is silently erased. This is a direct accounting violation: value that was owed to the owner is destroyed with no on-chain record of the skipped payment.

### Likelihood Explanation
The `RewardsPoolName` pool can be at zero in normal operation: at chain launch before the pool is seeded, after a period of heavy claims, or if the pool is intentionally underfunded. A validator slash is a standard production event (double-sign or downtime). Any position in a redelegation that is slashed while the pool is empty triggers this path. No privileged access or governance action is required by the attacker.

### Recommendation
Move all checkpoint mutations (`UpdateLastEventSeq`, `decrementEventRefCount`, `applyBonusAccrualCheckpoint`, `UpdateLastKnownBonded`) to occur **after** the `sufficientBonusPoolBalance` check and the `SendCoinsFromModuleToAccount` call succeed. Alternatively, perform the pool balance check at the start of `processEventsAndClaimBonus` before any state mutation, or use a transactional pattern that rolls back `pos` mutations on error. In `slashRedelegationPosition`, if `ErrInsufficientBonusPool` is caught, the position's checkpoints must **not** be persisted in their mutated state.

### Proof of Concept
1. Create a tier position with an active redelegation.
2. Drain `RewardsPoolName` to zero (e.g., by claiming all existing bonus rewards or simply not funding it).
3. Let some blocks pass so bonus accrues (`totalBonus > 0`).
4. Trigger a validator slash that fires `BeforeRedelegationSlashed` for the position's unbonding ID.
5. Assert: `pos.LastBonusAccrual` advanced to the current block time, `pos.LastEventSeq` advanced past all events, but no bonus coins were transferred to the owner's address. The owner's accrued bonus is permanently lost. [6](#0-5) [7](#0-6)

func (k Keeper) sufficientBonusPoolBalance(ctx context.Context, bonus sdk.Coins) error {
	if bonus.IsZero() {
		return nil
	}

	poolAddr := k.accountKeeper.GetModuleAddress(types.RewardsPoolName)
	poolBalance := k.bankKeeper.GetAllBalances(ctx, poolAddr)
	if !poolBalance.IsAllGTE(bonus) {
		return errorsmod.Wrapf(types.ErrInsufficientBonusPool,
			"bonus: %s, pool balance: %s",
			bonus.String(), poolBalance.String())
	}

	return nil
