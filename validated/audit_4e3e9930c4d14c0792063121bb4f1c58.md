I need to trace the exact code path for this claimed vulnerability, checking `IsAwayFromOrigin`, `processReceivedPacket`, and the `TransferOwner` guard.

### Title
Crafted `ClassId` with Source Port/Channel Prefix Bypasses Direction Check, Enabling Unauthorized Unescrow of Victim NFTs — (`x/nft-transfer/keeper/packet.go`)

---

### Summary

An attacker controlling a counterparty chain can craft a `NonFungibleTokenPacketData.ClassId` that begins with the **source** port/channel prefix (their own port/channel), causing `IsAwayFromOrigin` to return `false` in `processReceivedPacket`. This triggers the unescrow branch unconditionally, allowing the attacker to transfer any NFT currently held in the destination chain's escrow address to an arbitrary receiver — without that NFT having been legitimately sent by the attacker.

---

### Finding Description

`IsAwayFromOrigin` determines transfer direction by a simple string prefix check: [1](#0-0) 

```go
func IsAwayFromOrigin(sourcePort, sourceChannel, fullClassPath string) bool {
    prefixClassID := GetClassPrefix(sourcePort, sourceChannel)
    return !strings.HasPrefix(fullClassPath, prefixClassID)
}
```

In `processReceivedPacket`, this is called with the **source** (attacker-controlled) port/channel and the attacker-supplied `data.ClassId`: [2](#0-1) 

There is no validation anywhere — not in `ValidateBasic`, not in `OnRecvPacket` — that prevents `data.ClassId` from being prefixed with the source port/channel: [3](#0-2) [4](#0-3) 

`ValidateBasic` only checks that `ClassId` is non-blank, token/URI counts match, and addresses are valid. It does not reject a `ClassId` that starts with the source port/channel prefix.

When `isAwayFromOrigin` is `false`, the unescrow branch executes: [5](#0-4) 

```go
unprefixedClassID := types.RemoveClassPrefix(packet.GetSourcePort(), packet.GetSourceChannel(), data.ClassId)
voucherClassID := types.ParseClassTrace(unprefixedClassID).IBCClassID()
for _, tokenID := range data.TokenIds {
    if err := k.nftKeeper.TransferOwner(ctx, voucherClassID, tokenID, escrowAddress, receiver); err != nil {
        return err
    }
}
```

The `escrowAddress` used here is `GetEscrowAddress(packet.GetDestPort(), packet.GetDestChannel())` — the destination chain's escrow address for this channel.

The only guard inside `TransferOwner` is an ownership check: [6](#0-5) 

```go
nft, err := k.IsOwner(ctx, denomID, tokenID, srcOwner)
```

This check passes whenever the NFT is legitimately held in escrow — which is exactly the case after any honest user has sent an NFT via `SendTransfer` on the destination chain.

---

### Impact Explanation

**Concrete exploit scenario:**

1. Alice on chain B (victim) sends NFT `("victimClass", "victimTokenID")` via IBC to chain A (attacker). This escrows the NFT to `GetEscrowAddress("nft", "channel-0")` on chain B.
2. Attacker on chain A crafts a packet with:
   - `sourcePort = "nft"`, `sourceChannel = "channel-1"` (attacker's own channel)
   - `data.ClassId = "nft/channel-1/victimClass"` (starts with source prefix `"nft/channel-1/"`)
   - `data.TokenIds = ["victimTokenID"]`
   - `data.Receiver = attackerAddress`
3. Packet is committed on chain A and relayed to chain B.
4. On chain B, `IsAwayFromOrigin("nft", "channel-1", "nft/channel-1/victimClass")` → `strings.HasPrefix("nft/channel-1/victimClass", "nft/channel-1/")` is `true` → returns `false`.
5. Unescrow branch: `RemoveClassPrefix` strips `"nft/channel-1/"` → `"victimClass"`. `IBCClassID()` returns `"victimClass"` (no path).
6. `TransferOwner(ctx, "victimClass", "victimTokenID", escrowAddress, attackerAddress)` succeeds because the NFT is owned by the escrow address.
7. Alice's NFT is now owned by the attacker.

**Impact:** Unauthorized unescrow of any NFT held in the channel's escrow address. Any NFT escrowed via a legitimate `SendTransfer` on the destination chain is at risk.

---

### Likelihood Explanation

- Requires the attacker to control a counterparty chain or be able to relay a crafted IBC packet — a realistic threat model for IBC (any chain operator or relayer with write access to the source chain).
- The precondition (an NFT in escrow) is a normal operational state whenever any user has sent an NFT via IBC.
- No governance, key compromise, or social engineering is required.
- The IBC core layer authenticates that the packet was committed on the source chain via light client, but does NOT validate packet data content — that is entirely the application's responsibility, and the application has no guard here.

---

### Recommendation

In `processReceivedPacket`, before calling `IsAwayFromOrigin`, validate that if `data.ClassId` starts with the source port/channel prefix, a corresponding escrow record (or class trace) exists that was created by a legitimate outbound `SendTransfer` on this chain. Alternatively, maintain an explicit escrow ledger mapping `(classID, tokenID)` to the amount escrowed per channel, and require that ledger to be decremented on unescrow — rejecting any unescrow attempt for which no prior escrow entry exists.

---

```go
func (nftpd NonFungibleTokenPacketData) ValidateBasic() error {
	if strings.TrimSpace(nftpd.ClassId) == "" {
		return newsdkerrors.Wrap(ErrInvalidClassID, "classId cannot be blank")
	}

	if len(nftpd.TokenIds) == 0 {
		return newsdkerrors.Wrap(ErrInvalidTokenID, "tokenId cannot be blank")
	}

	if len(nftpd.TokenIds) != len(nftpd.TokenUris) {
		return newsdkerrors.Wrap(ErrInvalidPacket, "tokenIds and tokenUris lengths do not match")
	}

	if strings.TrimSpace(nftpd.Sender) == "" {
		return newsdkerrors.Wrap(sdkerrors.ErrInvalidAddress, "sender address cannot be blank")
	}

	// decode the sender address
	if _, err := sdk.AccAddressFromBech32(nftpd.Sender); err != nil {
		return newsdkerrors.Wrap(sdkerrors.ErrInvalidAddress, "invalid sender address")
	}

	if strings.TrimSpace(nftpd.Receiver) == "" {
		return newsdkerrors.Wrap(sdkerrors.ErrInvalidAddress, "receiver address cannot be blank")
	}

	// decode the receiver address
	if _, err := sdk.AccAddressFromBech32(nftpd.Receiver); err != nil {
		return newsdkerrors.Wrap(sdkerrors.ErrInvalidAddress, "invalid receiver address")
	}
	return nil
```

```go
func (k Keeper) TransferOwner(
	ctx sdk.Context, denomID, tokenID string, srcOwner, dstOwner sdk.AccAddress,
) error {
	if !k.HasDenomID(ctx, denomID) {
		return sdkerrors.Wrapf(types.ErrInvalidDenom, "denom ID %s not exists", denomID)
	}

	nft, err := k.IsOwner(ctx, denomID, tokenID, srcOwner)
	if err != nil {
		return err
	}

	nft.Owner = dstOwner.String()

	k.setNFT(ctx, denomID, nft)
	k.swapOwner(ctx, denomID, tokenID, srcOwner, dstOwner)
	return nil
}
```
