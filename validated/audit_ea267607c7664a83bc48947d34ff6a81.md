Audit Report

## Title
SwapAllowlistExtension Gates Router Address Instead of Actual User, Enabling Allowlist Bypass — (`metric-periphery/contracts/extensions/SwapAllowlistExtension.sol`)

## Summary

`SwapAllowlistExtension.beforeSwap` checks the `sender` argument against the per-pool allowlist, but `MetricOmmPool.swap` sets `sender` to `msg.sender` — which is the router when a user routes through `MetricOmmSimpleRouter`. If the router is allowlisted (the natural operational choice), every non-allowlisted user can bypass the restriction by calling the router. If the router is not allowlisted, every individually-allowlisted user is blocked from using the router.

## Finding Description

`MetricOmmPool.swap` captures `msg.sender` and passes it verbatim as the `sender` argument to `_beforeSwap`:

```solidity
// MetricOmmPool.sol L230-231
_beforeSwap(
    msg.sender,   // whoever called pool.swap()
    ...
);
``` [1](#0-0) 

`SwapAllowlistExtension.beforeSwap` then checks that `sender` against the per-pool allowlist, using `msg.sender` (the pool) as the pool key:

```solidity
// SwapAllowlistExtension.sol L37
if (!allowAllSwappers[msg.sender] && !allowedSwapper[msg.sender][sender]) {
    revert IMetricOmmPoolActions.NotAllowedToSwap();
}
``` [2](#0-1) 

When a user calls `MetricOmmSimpleRouter.exactInputSingle`, the router calls `pool.swap()` directly:

```solidity
// MetricOmmSimpleRouter.sol L72-80
(int128 amount0Delta, int128 amount1Delta) = IMetricOmmPoolActions(params.pool)
    .swap(
        params.recipient,
        params.zeroForOne,
        MetricOmmSwapInputs.asAmountSpecifiedIn(params.amountIn),
        priceLimitX64,
        "",
        params.extensionData
    );
``` [3](#0-2) 

At this point `msg.sender` inside the pool is the **router**, not the end user. The extension therefore evaluates `allowedSwapper[pool][router]` instead of `allowedSwapper[pool][user]`. The same flaw applies to `exactInput`, `exactOutputSingle`, and `exactOutput`. [4](#0-3) 

No existing guard corrects this: the extension has no mechanism to decode the original caller from `extensionData`, and the router passes `""` as `extensionData` for single-hop swaps. [5](#0-4) 

## Impact Explanation

A pool deploying `SwapAllowlistExtension` to restrict trading to a curated set of addresses (KYC'd counterparties, whitelisted market makers, protocol-owned addresses) loses that restriction entirely for any user who routes through `MetricOmmSimpleRouter`. An attacker can execute swaps on a pool that was designed to be closed to them, interacting with LP assets at oracle-derived prices without authorization. This is a direct, broken access-control impact on LP assets and pool integrity — matching the "broken core pool functionality causing loss of funds" and "admin-boundary break bypassed by an unprivileged path" allowed impact criteria. [6](#0-5) 

## Likelihood Explanation

`MetricOmmSimpleRouter` is the primary user-facing entry point for swaps. Any pool that enables `SwapAllowlistExtension` and also allowlists the router (the expected production configuration so that allowlisted users can use the router) is immediately and completely vulnerable. No privileged action, malicious setup, non-standard token, or special precondition is required — a normal user calling `exactInputSingle` through the public router is sufficient. The attack is repeatable on every swap. [7](#0-6) 

## Recommendation

The extension must gate on the original user, not the intermediary. The cleanest fix is for the router to encode the original `msg.sender` into `extensionData`, and for `SwapAllowlistExtension.beforeSwap` to decode and verify that value when present, falling back to `sender` for direct pool calls. Concretely:

1. In `MetricOmmSimpleRouter`, replace the empty `""` `extensionData` argument (or prepend to caller-supplied `extensionData`) with `abi.encode(msg.sender)`.
2. In `SwapAllowlistExtension.beforeSwap`, decode the first 32 bytes of `extensionData` as the original caller when the data is non-empty and the caller is a known router, and check that address against the allowlist instead of `sender`. [6](#0-5) 

## Proof of Concept

```solidity
// 1. Deploy pool with SwapAllowlistExtension
// 2. Admin allowlists only alice
vm.prank(poolAdmin);
swapAllowlist.setAllowedToSwap(address(pool), alice, true);
// 3. Admin also allowlists the router so alice can use it
vm.prank(poolAdmin);
swapAllowlist.setAllowedToSwap(address(pool), address(router), true);

// 4. Attacker (not allowlisted) routes through the router
vm.prank(attacker);
// Should revert — does NOT. Router address passes the check.
router.exactInputSingle(ExactInputSingleParams({
    pool: address(pool),
    recipient: attacker,
    tokenIn: token0,
    zeroForOne: true,
    amountIn: 1000,
    amountOutMinimum: 0,
    priceLimitX64: 0,
    deadline: block.timestamp,
    extensionData: ""
}));
// Attacker successfully swapped on a restricted pool
```

Call path: `attacker` → `router.exactInputSingle` → `pool.swap(router as msg.sender)` → `_beforeSwap(router, ...)` → `allowedSwapper[pool][router] == true` → check passes → swap executes. [8](#0-7)
