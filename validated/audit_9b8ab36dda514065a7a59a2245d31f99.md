### Title
`SwapAllowlistExtension` gates the router address instead of the end user, allowing any user to bypass the per-user swap allowlist via `MetricOmmSimpleRouter` - (`metric-periphery/contracts/extensions/SwapAllowlistExtension.sol`)

---

### Summary

`SwapAllowlistExtension.beforeSwap` checks the `sender` argument forwarded by the pool, which equals `msg.sender` of `pool.swap()`. When a swap is routed through `MetricOmmSimpleRouter`, `msg.sender` of `pool.swap()` is the router contract, not the end user. If the pool admin allowlists the router address to enable router-mediated swaps on a curated pool, every user — including those not individually allowlisted — can bypass the per-user gate by routing through the public router.

---

### Finding Description

**Call path:**

1. Pool admin deploys a curated pool with `SwapAllowlistExtension` attached to the `beforeSwap` hook.
2. Admin allowlists specific users directly: `setAllowedToSwap(pool, userA, true)`.
3. To also allow router-mediated swaps for those users, admin allowlists the router: `setAllowedToSwap(pool, router, true)`.
4. Non-allowlisted `userB` calls `MetricOmmSimpleRouter.exactInputSingle(params)` targeting the curated pool.

**Inside the router:**

```
MetricOmmSimpleRouter.exactInputSingle
  → IMetricOmmPoolActions(params.pool).swap(params.recipient, ..., params.extensionData)
```

`msg.sender` of `pool.swap()` is the router. [1](#0-0) 

**Inside the pool:**

```
MetricOmmPool.swap
  → _beforeSwap(msg.sender, ...)   // msg.sender = router
``` [2](#0-1) 

**Inside `ExtensionCalling._beforeSwap`**, `sender` = router is forwarded to the extension: [3](#0-2) 

**Inside `SwapAllowlistExtension.beforeSwap`**, the check is:

```solidity
if (!allowAllSwappers[msg.sender] && !allowedSwapper[msg.sender][sender]) {
    revert IMetricOmmPoolActions.NotAllowedToSwap();
}
```

`msg.sender` here is the pool; `sender` is the router. The extension evaluates `allowedSwapper[pool][router]` — not `allowedSwapper[pool][userB]`. [4](#0-3) 

Because the router is allowlisted (required for any router-mediated swap to work on this pool), `userB`'s swap succeeds despite never being individually authorized.

---

### Impact Explanation

A curated pool using `SwapAllowlistExtension` to enforce KYC, regulatory, or protocol-specific access control loses that control entirely for any user who routes through `MetricOmmSimpleRouter`. The allowlist is bypassed without any privileged action by the attacker. LP funds in the pool are exposed to swaps from actors the pool admin explicitly intended to exclude, which constitutes a broken core pool functionality and a direct policy bypass with fund-impacting consequences (unauthorized parties can drain arbitrage value or execute swaps against LP positions).

---

### Likelihood Explanation

`MetricOmmSimpleRouter` is the primary public swap entrypoint documented for end users. Any non-allowlisted user who discovers the curated pool can trivially route through the router. The only precondition is that the pool admin has allowlisted the router address — which is the only way to make router-mediated swaps work at all on an allowlisted pool, making this a near-certain operational configuration.

---

### Recommendation

The extension must identify the true end user, not the intermediary. Two approaches:

1. **Pass the original caller in `extensionData`**: Have the router encode `msg.sender` (the end user) into `extensionData` and have the extension decode and check that address. This requires a trusted router convention.

2. **Check `sender` against a trusted-forwarder registry**: The extension can maintain a mapping of trusted routers and, when `sender` is a trusted router, require the actual user identity to be supplied in `extensionData`.

3. **Do not allowlist the router**: Instead, require all allowlisted users to call `pool.swap()` directly. Document this restriction explicitly. This is the simplest fix but removes router usability for curated pools.

---

### Proof of Concept

```
Setup:
  - Deploy pool with SwapAllowlistExtension on beforeSwap hook
  - setAllowedToSwap(pool, router, true)       // enable router-mediated swaps
  - setAllowedToSwap(pool, userA, true)        // only userA is individually allowed
  - userB is NOT allowlisted

Attack:
  - userB calls MetricOmmSimpleRouter.exactInputSingle({pool: pool, ...})
  - Router calls pool.swap(...) with msg.sender = router
  - Pool calls _beforeSwap(router, ...)
  - Extension checks allowedSwapper[pool][router] → true
  - Swap executes for userB despite userB not being allowlisted

Expected: revert NotAllowedToSwap
Actual:   swap succeeds
``` [5](#0-4) [6](#0-5) [2](#0-1)
