### Title
Missing Oracle Price Bound in `MintNlp`/`BurnNlp` Signed Digest Allows Sequencer to Manipulate NLP Exchange Rate — (`core/contracts/Verifier.sol`, `core/contracts/Clearinghouse.sol`)

---

### Summary

The `MintNlp` and `BurnNlp` EIP-712 signed digests commit only to `sender`, `amount`, and `nonce`. The `oraclePriceX18` field — which directly determines how many NLP tokens a user receives (or how much quote they receive on burn) — is supplied by the sequencer outside the signed struct and is never committed to in the digest. A user who signs a `MintNlp` or `BurnNlp` message has no on-chain guarantee about the price at which their transaction will execute.

---

### Finding Description

`SignedMintNlp` and `SignedBurnNlp` are defined in `IEndpoint.sol` as:

```solidity
struct SignedMintNlp {
    MintNlp tx;          // signed: sender, quoteAmount, nonce
    bytes signature;
    int128 oraclePriceX18;       // NOT signed
    int128[] nlpPoolRebalanceX18; // NOT signed
}

struct SignedBurnNlp {
    BurnNlp tx;          // signed: sender, nlpAmount, nonce
    bytes signature;
    int128 oraclePriceX18;       // NOT signed
    int128[] nlpPoolRebalanceX18; // NOT signed
}
``` [1](#0-0) 

The `Verifier.computeDigest` function for `MintNlp` hashes only `sender`, `quoteAmount`, and `nonce`:

```solidity
digest = keccak256(
    abi.encode(
        keccak256(bytes(MINT_NLP_SIGNATURE)),
        signedTx.tx.sender,
        signedTx.tx.quoteAmount,
        signedTx.tx.nonce
    )
);
``` [2](#0-1) 

And for `BurnNlp`, only `sender`, `nlpAmount`, and `nonce`:

```solidity
digest = keccak256(
    abi.encode(
        keccak256(bytes(BURN_NLP_SIGNATURE)),
        signedTx.tx.sender,
        signedTx.tx.nlpAmount,
        signedTx.tx.nonce
    )
);
``` [3](#0-2) 

In `EndpointTx.sol`, after signature validation, the sequencer-supplied `oraclePriceX18` is passed directly to `Clearinghouse`:

```solidity
priceX18[NLP_PRODUCT_ID] = signedTx.oraclePriceX18;
clearinghouse.mintNlp(signedTx.tx, signedTx.oraclePriceX18, nlpPools, signedTx.nlpPoolRebalanceX18);
``` [4](#0-3) 

In `Clearinghouse.mintNlp`, the NLP tokens minted are computed as:

```solidity
int128 nlpAmount = quoteAmount.div(oraclePriceX18);
``` [5](#0-4) 

In `Clearinghouse.burnNlp`, the quote returned is:

```solidity
int128 quoteAmount = nlpAmount.mul(oraclePriceX18);
``` [6](#0-5) 

Because `oraclePriceX18` is not part of the signed digest, the sequencer can substitute any price value when submitting the transaction. The user's signature remains valid regardless of what price is used.

---

### Impact Explanation

**For `MintNlp`:** A user signs a commitment to spend `quoteAmount` (e.g., 1000 USDC). The NLP tokens received equal `quoteAmount / oraclePriceX18`. If the sequencer inflates `oraclePriceX18`, the user receives far fewer NLP tokens than the fair market rate — a direct, quantifiable asset loss proportional to the price deviation.

**For `BurnNlp`:** A user signs a commitment to burn `nlpAmount` NLP tokens. The quote returned equals `nlpAmount * oraclePriceX18 - burnFee`. If the sequencer deflates `oraclePriceX18`, the user receives far less USDC than the fair market rate for their burned NLP — again a direct asset loss.

In both cases, the user's collateral balance in the `SpotEngine` is mutated by an amount the user never consented to, with no on-chain mechanism to enforce a minimum acceptable price. [7](#0-6) [8](#0-7) 

---

### Likelihood Explanation

The sequencer is the only party that can submit `MintNlp`/`BurnNlp` transactions (via `submitTransactionsChecked`). Exploitation requires a malicious or compromised sequencer. The protocol acknowledges sequencer trust risk by providing a slow-mode escape hatch for censorship resistance, but slow-mode does not cover `MintNlp`/`BurnNlp` — those transactions can only be submitted by the sequencer. The likelihood is **medium**: the sequencer is a privileged off-chain component, but the protocol's own architecture treats sequencer misbehavior as a realistic threat class (hence slow-mode), and the absence of a price bound in the signed digest means there is zero on-chain protection for users even if the sequencer acts adversarially.

---

### Recommendation

Include `oraclePriceX18` as a minimum acceptable price bound in the signed digest for both `MintNlp` and `BurnNlp`. Specifically:

1. Add a `minPriceX18` (for `MintNlp`) or `maxPriceX18` (for `BurnNlp`) field to the `MintNlp` and `BurnNlp` structs.
2. Include this field in the EIP-712 type string and `abi.encode` call in `Verifier.computeDigest`.
3. Enforce the bound in `Clearinghouse.mintNlp` / `Clearinghouse.burnNlp` before computing the token amounts.

This mirrors the standard slippage protection pattern and ensures the user's signature commits to the financial terms of the operation.

---

### Proof of Concept

1. Alice signs a `MintNlp` message: `{ sender: alice, quoteAmount: 1000e18, nonce: 5 }`. At the time of signing, the fair NLP price is `1e18` (1:1), so she expects ~1000 NLP tokens.
2. The sequencer constructs `SignedMintNlp { tx: aliceTx, signature: aliceSig, oraclePriceX18: 1000e18, nlpPoolRebalanceX18: [...] }` — setting the price 1000× higher than fair value.
3. `EndpointTx` validates Alice's signature against the digest of `(sender, quoteAmount, nonce)` — the signature is valid.
4. `Clearinghouse.mintNlp` computes `nlpAmount = 1000e18 / 1000e18 = 1e18` — Alice receives only **1 NLP token** instead of 1000.
5. Alice's 1000 USDC is debited; she has no recourse because the nonce is consumed and the transaction is final. [2](#0-1) [9](#0-8)

    function mintNlp(
        IEndpoint.MintNlp calldata txn,
        int128 oraclePriceX18,
        IEndpoint.NlpPool[] calldata nlpPools,
        int128[] calldata nlpPoolRebalanceX18
    ) external onlyEndpoint {
        require(!RiskHelper.isIsolatedSubaccount(txn.sender), ERR_UNAUTHORIZED);

        ISpotEngine spotEngine = _spotEngine();
        spotEngine.updatePrice(NLP_PRODUCT_ID, oraclePriceX18);

        require(txn.quoteAmount <= INT128_MAX, ERR_CONVERSION_OVERFLOW);
        int128 quoteAmount = int128(txn.quoteAmount);
        int128 nlpAmount = quoteAmount.div(oraclePriceX18);

        _validateNlpRebalance(nlpPools, nlpPoolRebalanceX18, quoteAmount);
        for (uint128 i = 0; i < nlpPoolRebalanceX18.length; i++) {
            require(nlpPoolRebalanceX18[i] >= 0, ERR_INVALID_NLP_REBALANCE);
        }

        spotEngine.updateBalance(NLP_PRODUCT_ID, txn.sender, nlpAmount);
        spotEngine.updateBalance(NLP_PRODUCT_ID, N_ACCOUNT, -nlpAmount);

        spotEngine.updateBalance(QUOTE_PRODUCT_ID, txn.sender, -quoteAmount);
        _applyNlpRebalance(spotEngine, nlpPools, nlpPoolRebalanceX18);

        require(
            getHealth(txn.sender, IProductEngine.HealthType.INITIAL) >= 0,
            ERR_SUBACCT_HEALTH
        );

```text
    function burnNlp(
        IEndpoint.BurnNlp calldata txn,
        int128 oraclePriceX18,
        IEndpoint.NlpPool[] calldata nlpPools,
        int128[] calldata nlpPoolRebalanceX18
    ) external onlyEndpoint {
        require(!RiskHelper.isIsolatedSubaccount(txn.sender), ERR_UNAUTHORIZED);

        ISpotEngine spotEngine = _spotEngine();
        spotEngine.updatePrice(NLP_PRODUCT_ID, oraclePriceX18);

        require(txn.nlpAmount <= INT128_MAX, ERR_CONVERSION_OVERFLOW);
        int128 nlpAmount = int128(txn.nlpAmount);
        require(
            spotEngine.getNlpUnlockedBalance(txn.sender).amount >= nlpAmount,
            ERR_UNLOCKED_NLP_INSUFFICIENT
        );
        int128 quoteAmount = nlpAmount.mul(oraclePriceX18);
        int128 burnFee = MathHelper.max(ONE, quoteAmount / 1000);
        quoteAmount = MathHelper.max(0, quoteAmount - burnFee);

        _validateNlpRebalance(nlpPools, nlpPoolRebalanceX18, -quoteAmount);
        for (uint128 i = 0; i < nlpPoolRebalanceX18.length; i++) {
            require(nlpPoolRebalanceX18[i] <= 0, ERR_INVALID_NLP_REBALANCE);
        }

        spotEngine.updateBalance(NLP_PRODUCT_ID, txn.sender, -nlpAmount);
        spotEngine.updateBalance(NLP_PRODUCT_ID, N_ACCOUNT, nlpAmount);

        if (quoteAmount > 0) {
            spotEngine.updateBalance(QUOTE_PRODUCT_ID, txn.sender, quoteAmount);
            _applyNlpRebalance(spotEngine, nlpPools, nlpPoolRebalanceX18);
        }

        require(
            spotEngine.getBalance(NLP_PRODUCT_ID, txn.sender).amount >= 0,
            ERR_SUBACCT_HEALTH
        );
        // Burning NLP can decrease health if the burn fee exceeds the health improvement
        // from the withdrawal. This check prevents malicious actors from deliberately
        // creating unhealthy subaccounts through NLP burns.
        require(
            getHealth(txn.sender, IProductEngine.HealthType.MAINTENANCE) >= 0,
            ERR_SUBACCT_HEALTH
        );
    }
