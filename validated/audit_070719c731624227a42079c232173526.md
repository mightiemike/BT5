### Title
`SwapAllowlistExtension.beforeSwap` checks the router address as the swapper identity, making per-user allowlist enforcement dead code for router-mediated swaps - (File: metric-periphery/contracts/extensions/SwapAllowlistExtension.sol)

### Summary
`SwapAllowlistExtension.beforeSwap` gates swaps by checking `sender`, which is `msg.sender` of `MetricOmmPool.swap()`. When a user routes through `MetricOmmSimpleRouter`, `sender` is the router contract address, not the actual user. If the pool admin allowlists the router to enable router-mediated swaps for their curated users, the per-user allowlist check becomes dead code: any user—including non-allowlisted ones—can bypass the gate by calling the router.

### Finding Description

`MetricOmmPool.swap()` passes `msg.sender` as the `sender` argument to `_beforeSwap`:

```solidity
// MetricOmmPool.sol:230-240
_beforeSwap(
    msg.sender,   // ← direct caller of pool.swap()
    recipient,
    zeroForOne,
    amountSpecified,
    priceLimitX64,
    packedSlot0Initial,
    bidPriceX64,
    askPriceX64,
    extensionData
);
```

`ExtensionCalling._beforeSwap` forwards this value verbatim to every configured extension:

```solidity
// ExtensionCalling.sol:160-176
abi.encodeCall(
    IMetricOmmExtensions.beforeSwap,
    (sender, recipient, zeroForOne, ...)
)
```

`SwapAllowlistExtension.beforeSwap` then checks that `sender` is allowlisted for the calling pool:

```solidity
// SwapAllowlistExtension.sol:37-39
if (!allowAllSwappers[msg.sender] && !allowedSwapper[msg.sender][sender]) {
    revert IMetricOmmPoolActions.NotAllowedToSwap();
}
```

When `MetricOmmSimpleRouter.exactInputSingle()` (or `exactInput`, `exactOutputSingle`, `exactOutput`) calls `pool.swap()`, the pool's `msg.sender` is the router contract:

```solidity
// MetricOmmSimpleRouter.sol:72-80
(int128 amount0Delta, int128 amount1Delta) = IMetricOmmPoolActions(params.pool)
    .swap(
        params.recipient,
        params.zeroForOne,
        MetricOmmSwapInputs.asAmountSpecifiedIn(params.amountIn),
        priceLimitX64,
        "",
        params.extensionData
    );
```

So `sender` arriving at the extension is the router address, not the actual user (`msg.sender` of the router call).

The pool admin faces an irresolvable dilemma:

| Configuration | Effect |
|---|---|
| Router **not** allowlisted | Allowlisted users cannot use the router at all |
| Router **allowlisted** | Any user (including non-allowlisted) can swap through the router |

There is no configuration of `SwapAllowlistExtension` that allows specific users to swap through the router while blocking others. The per-user allowlist check is structurally dead code for router-mediated swaps: the checked identity (`sender` = router) is never the economic actor the pool admin intended to gate.

This is the direct analog to the external bug: just as `.onConflictDoNothing()` is dead code because the unique constraint it targets does not exist on `tradeId`, the per-user allowlist check in `beforeSwap` is dead code for router-mediated swaps because the identity it checks (`sender`) is never the actual user when the router is involved.

### Impact Explanation

Any non-allowlisted user can bypass the swap allowlist on a curated pool by routing through `MetricOmmSimpleRouter`. The pool admin's intended access control policy is silently nullified for all router-mediated swaps. Curated pools (e.g., KYC-gated, institutional, or restricted-access pools) lose their curation guarantee. Unauthorized users can trade against the pool's LPs, which may violate regulatory or contractual obligations and expose LPs to counterparty risk they explicitly opted out of.

### Likelihood Explanation

Medium. The pool admin must allowlist the router to enable router-mediated swaps for their allowlisted users—this is a natural, expected configuration
