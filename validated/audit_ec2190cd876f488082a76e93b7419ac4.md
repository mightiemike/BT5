### Title
Authz GenericAuthorization for MsgMintNFT Allows Grantee to Mint NFTs to Arbitrary Recipients Under Granter's Denom â€” (`x/nft/keeper/msg_server.go`, `x/nft/keeper/keeper.go`)

---

### Summary

The NFT module's `MsgMintNFT` handler trusts `msg.Sender` as the authority for the denom-creator check. When a denom creator (alice) grants a `GenericAuthorization` for `MsgMintNFT` to an unprivileged account (bob), bob can submit `authz.MsgExec` wrapping `MsgMintNFT{Sender: alice, Recipient: bob}`. The authz module validates the grant using `GetSigners()` â†’ alice, finds the grant, and executes the inner message. The keeper then calls `IsDenomCreator(ctx, denomID, alice)`, which passes because alice is the creator, and mints the NFT to bob. The actual transaction signer (bob) is never verified against the denom creator.

---

### Finding Description

**Entrypoint:** `authz.MsgExec` wrapping `MsgMintNFT` â€” a standard, externally reachable Cosmos SDK transaction path.

**Step-by-step trace:**

1. Alice issues a denom; she is stored as `denom.Creator`.
2. Alice grants bob a `GenericAuthorization` for `/chainmain.nft.v1.Msg/MintNFT`.
3. Bob constructs `MsgMintNFT{Sender: alice, Recipient: bob, DenomId: X, Id: Y, ...}` and wraps it in `authz.MsgExec`, signing with his own key.
4. The authz module calls `GetSigners()` on the inner message, which returns `alice` (from `msg.Sender`). [1](#0-0) 

5. The authz module looks up: "does bob have a grant from alice for `MsgMintNFT`?" â†’ YES. It executes the inner message.
6. `msgServer.MintNFT` reads `msg.Sender` (alice) and passes it as `sender` to `Keeper.MintNFT`. [2](#0-1) 

7. `Keeper.MintNFT` calls `IsDenomCreator(ctx, denomID, sender)` where `sender = alice`. [3](#0-2) 

8. `IsDenomCreator` compares `denom.Creator` to alice â†’ equal â†’ returns no error. [4](#0-3) 

9. `MintNFTUnverified` mints the NFT to `owner = recipient = bob`. Bob now owns the NFT.

**Root cause:** The proto annotation `option (cosmos.msg.v1.signer) = "sender"` means the authz module uses `msg.Sender` as the granter identity for grant lookup. [5](#0-4) 

The keeper never verifies that the actual transaction signer equals `msg.Sender`. In a direct transaction the SDK enforces this implicitly (only the key holder can sign), but under `authz.MsgExec` the actual signer is the grantee (bob), not the granter (alice). The keeper's `IsDenomCreator` check against `msg.Sender` is therefore bypassable by any grantee holding a `GenericAuthorization`.

---

### Impact Explanation

Bob (an unprivileged account) can mint an unlimited number of NFTs under alice's denom to any recipient address, including himself. This directly violates the NFT ownership invariant: the `KeyOwner(bob, denomID, tokenID)` index is set for tokens bob never legitimately earned. Bob can then transfer or sell these NFTs. The denom creator's exclusive minting right is completely nullified for any denom where a `GenericAuthorization` grant exists.

---

### Likelihood Explanation

`GenericAuthorization` for message types is a standard, documented Cosmos SDK feature. A denom creator might legitimately grant minting rights to a trusted co-operator (e.g., a hot wallet or a minting service), expecting that the grant only allows minting on their behalf in a controlled way. The attacker only needs to be the recipient of such a grant â€” no key compromise, governance action, or privileged role is required. The attack is a single `MsgExec` transaction.

---

### Recommendation

The keeper must verify that the actual transaction signer is the denom creator, not just `msg.Sender`. Two concrete approaches:

1. **Reject authz delegation for minting:** Register a custom `Authorization` type for `MsgMintNFT` that always returns `ErrUnauthorized`, preventing `GenericAuthorization` from being used for this message type.
2. **Bind sender to tx signer in the handler:** In `msgServer.MintNFT`, extract the actual signer from the SDK context (e.g., via `sdk.UnwrapSDKContext(goCtx).TxBytes()` or by requiring the keeper to receive the verified signer from the ante-handler) and assert it equals `msg.Sender` before proceeding to the creator check.

func (k Keeper) IsDenomCreator(ctx sdk.Context, denomID string, address sdk.AccAddress) (types.Denom, error) {
	denom, err := k.GetDenom(ctx, denomID)
	if err != nil {
		return types.Denom{}, err
	}

	creator, err := sdk.AccAddressFromBech32(denom.Creator)
	if err != nil {
		panic(err)
	}

	if !creator.Equals(address) {
		return types.Denom{}, sdkerrors.Wrapf(types.ErrUnauthorized, "%s is not the creator of %s", address, denomID)
	}

	return denom, nil
