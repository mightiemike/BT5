### Title
Protobuf `ValidResourceBounds` Deserialization Silently Downgrades `AllResources` to `L1Gas` When l2_gas and l1_data_gas Are Zero, Causing P2P Rejection of Valid Transactions - (File: `crates/apollo_protobuf/src/converters/transaction.rs`)

---

### Summary

The protobuf deserializer for `ValidResourceBounds` applies a heuristic that converts any wire representation with zero `l2_gas` and zero `l1_data_gas` into the `L1Gas` variant. However, the RPC/internal pipeline always produces `AllResources` for V3 transactions. A V3 transaction with non-zero `l1_gas` but zero `l2_gas` and `l1_data_gas` is accepted by the gateway, propagated over P2P, and then irrecoverably rejected by every receiving peer because the downstream `TryFrom<InvokeTransactionV3> for RpcInvokeTransactionV3` conversion hard-rejects the `L1Gas` variant. The entire batch containing that transaction is dropped and the originating peer is reported.

---

### Finding Description

**Step 1 – Gateway admission.**
`StatelessTransactionValidator::validate_resource_bounds` accepts a V3 transaction whose `AllResourceBounds` has non-zero `l1_gas` but zero `l2_gas` and `l1_data_gas`:

```rust
// crates/apollo_gateway/src/stateless_transaction_validator.rs:66
if ValidResourceBounds::AllResources(resource_bounds).max_possible_fee(Tip::ZERO) == Fee(0) {
    return Err(ZeroResourceBounds { resource_bounds });
}
```

The `max_possible_fee` for `AllResources { l1_gas: X, l2_gas: 0, l1_data_gas: 0 }` equals `X.max_amount * X.max_price_per_unit`, which is non-zero. The test case `valid_l1_gas` in the validator test suite explicitly confirms this path is valid. [1](#0-0) [2](#0-1) 

**Step 2 – Hash computed as `AllResources`.**
`convert_rpc_tx_to_internal` calls `tx_without_hash.calculate_transaction_hash`, which calls `get_invoke_transaction_v3_hash`. That function calls `get_tip_resource_bounds_hash` with `ValidResourceBounds::AllResources(...)`, which includes the `L1_DATA_GAS` element in the Poseidon hash:

```rust
// crates/starknet_api/src/transaction_hash.rs:203-208
resource_felts.extend(match resource_bounds {
    ValidResourceBounds::L1Gas(_) => vec![],
    ValidResourceBounds::AllResources(all_resources) => {
        vec![get_concat_resource(&all_resources.l1_data_gas, L1_DATA_GAS)?]
    }
});
```

The user signed this 4-element hash (tip + L1_GAS + L2_GAS + L1_DATA_GAS). [3](#0-2) [4](#0-3) 

**Step 3 – P2P serialization.**
The propagator converts `InternalRpcTransaction → RpcTransaction → protobuf::MempoolTransaction`. The `RpcInvokeTransactionV3 → InvokeTransactionV3 → protobuf::InvokeV3` path serializes `AllResourceBounds { l1_gas: X, l2_gas: 0, l1_data_gas: 0 }` faithfully as three `ResourceLimits` fields, all with zero amounts/prices for l2 and l1_data. [5](#0-4) [6](#0-5) 

**Step 4 – Protobuf deserialization at the receiving peer.**
`TryFrom<protobuf::ResourceBounds> for ValidResourceBounds` applies:

```rust
// crates/apollo_protobuf/src/converters/transaction.rs:431-435
Ok(if l1_data_gas.is_zero() && l2_gas.is_zero() {
    ValidResourceBounds::L1Gas(l1_gas)
} else {
    ValidResourceBounds::AllResources(AllResourceBounds { l1_gas, l2_gas, l1_data_gas })
})
```

Because both `l2_gas` and `l1_data_gas` are zero, the result is `ValidResourceBounds::L1Gas(l1_gas)`. [7](#0-6) 

**Step 5 – Hard rejection.**
`TryFrom<protobuf::InvokeV3WithProof> for RpcInvokeTransactionV3` calls `snapi_invoke.try_into()`, which is `TryFrom<InvokeTransactionV3> for RpcInvokeTransactionV3`. That conversion hard-rejects any variant other than `AllResources`:

```rust
// crates/starknet_api/src/rpc_transaction.rs:591-597
resource_bounds: match value.resource_bounds {
    ValidResourceBounds::AllResources(bounds) => bounds,
    _ => {
        return Err(StarknetApiError::OutOfRange { string: "resource_bounds".to_string() });
    }
},
```

The error is mapped to `DEPRECATED_RESOURCE_BOUNDS_ERROR` and propagated up. [8](#0-7) [9](#0-8) 

**Step 6 – Entire batch dropped.**
`TryFrom<protobuf::MempoolTransactionBatch> for RpcTransactionBatch` uses `collect::<Result<_, _>>()?`, so a single failing transaction aborts the entire batch:

```rust
// crates/apollo_protobuf/src/converters/rpc_transaction.rs:80-88
Ok(RpcTransactionBatch(
    value.transactions.into_iter()
        .map(RpcTransaction::try_from)
        .collect::<Result<_, _>>()?,
))
```

The runner then reports the originating peer. [10](#0-9) 

---

### Impact Explanation

A valid V3 transaction (accepted by the gateway on node A) is silently dropped by every P2P peer that receives it. The transaction never enters any other node's mempool. If node A is not the current block proposer, the transaction is never sequenced. Additionally, node A is reported as a misbehaving peer by every node that receives the batch, potentially leading to blacklisting. A single such transaction poisons an entire batch, causing collateral loss of all other transactions in that batch.

This matches: **High. Mempool/gateway/RPC admission accepts invalid transactions or rejects valid transactions before sequencing.**

---

### Likelihood Explanation

The trigger requires a V3 transaction with non-zero `l1_gas` but zero `l2_gas` and `l1_data_gas`. The gateway's `min_gas_price` check (`l2_gas.max_price_per_unit >= min_gas_price`) blocks this when `min_gas_price > 0`. However:

- The test suite explicitly marks `valid_l1_gas` (zero l2/l1_data bounds) as a passing case.
- Any deployment with `min_gas_price = 0` (e.g., private/test networks, early mainnet configurations) is directly vulnerable.
- An attacker who can influence the gateway configuration or who operates against a node with permissive settings can trigger this deterministically.

---

### Recommendation

**Short term:** In `TryFrom<protobuf::ResourceBounds> for ValidResourceBounds`, always produce `AllResources` for V3 mempool transactions, regardless of whether `l2_gas` and `l1_data_gas` are zero. The `L1Gas` variant should only be produced when deserializing historical (pre-0.13.3) block-sync transactions, not mempool transactions.

```rust
// Proposed fix for mempool path:
Ok(ValidResourceBounds::AllResources(AllResourceBounds { l1_gas, l2_gas, l1_data_gas }))
```

**Long term:** Add a round-trip serialization test that covers `AllResourceBounds { l1_gas: X, l2_gas: 0, l1_data_gas: 0 }` through the full protobuf mempool path and asserts that the deserialized `RpcTransaction` is identical to the original, including the `resource_bounds` variant.

---

### Proof of Concept

1. Craft a V3 invoke transaction: `AllResourceBounds { l1_gas: { max_amount: 1000, max_price_per_unit: 1 }, l2_gas: { max_amount: 0, max_price_per_unit: 0 }, l1_data_gas: { max_amount: 0, max_price_per_unit: 0 } }`.
2. Submit to node A (gateway with `min_gas_price = 0`). Stateless validation passes because `max_possible_fee(Tip::ZERO) = 1000 > 0`.
3. Node A computes hash H using `get_tip_resource_bounds_hash` with `AllResources` (4-element hash: tip + L1_GAS + L2_GAS + L1_DATA_GAS). User signature covers H.
4. Node A propagates the transaction via P2P gossipsub.
5. Node B receives the `MempoolTransactionBatch` protobuf.
6. `TryFrom<protobuf::ResourceBounds> for ValidResourceBounds`: `l1_data_gas.is_zero() && l2_gas.is_zero()` → produces `ValidResourceBounds::L1Gas(l1_gas)`.
7. `TryFrom<InvokeTransactionV3> for RpcInvokeTransactionV3`: `L1Gas` arm → returns `Err(OutOfRange)` → mapped to `DEPRECATED_RESOURCE_BOUNDS_ERROR`.
8. `collect::<Result<_, _>>()?` in `TryFrom<protobuf::MempoolTransactionBatch>` propagates the error.
9. The entire batch is dropped. Node B reports node A as a bad peer.
10. The transaction is never added to node B's mempool and will not be sequenced if node B is the proposer.

pub fn get_tip_resource_bounds_hash(
    resource_bounds: &ValidResourceBounds,
    tip: &Tip,
) -> Result<Felt, StarknetApiError> {
    let l1_resource_bounds = resource_bounds.get_l1_bounds();
    let l2_resource_bounds = resource_bounds.get_l2_bounds();

    // L1 and L2 gas bounds always exist.
    // Old V3 txs always have L2 gas bounds of zero, but they exist.
    let mut resource_felts = vec![
        get_concat_resource(&l1_resource_bounds, L1_GAS)?,
        get_concat_resource(&l2_resource_bounds, L2_GAS)?,
    ];

    // For new V3 txs, need to also hash the data gas bounds.
    resource_felts.extend(match resource_bounds {
        ValidResourceBounds::L1Gas(_) => vec![],
        ValidResourceBounds::AllResources(all_resources) => {
            vec![get_concat_resource(&all_resources.l1_data_gas, L1_DATA_GAS)?]
        }
    });

    Ok(HashChain::new().chain(&tip.0.into()).chain_iter(resource_felts.iter()).get_poseidon_hash())
}

impl TryFrom<protobuf::InvokeV3WithProof> for RpcInvokeTransactionV3 {
    type Error = ProtobufConversionError;
    fn try_from(mut value: protobuf::InvokeV3WithProof) -> Result<Self, Self::Error> {
        // Extract proof first, since `starknet_api::transaction::InvokeTransactionV3` does not
        // carry a `proof` field.
        let proof = Proof::from(std::mem::take(&mut value.proof));

        let snapi_invoke: InvokeTransactionV3 = value
            .invoke
            .ok_or(ProtobufConversionError::MissingField {
                field_description: "InvokeV3WithProof::invoke",
            })?
            .try_into()?;

        // This conversion can fail only if the resource_bounds are not AllResources.
        Ok(Self { proof, ..snapi_invoke.try_into().map_err(|_| DEPRECATED_RESOURCE_BOUNDS_ERROR)? })
    }
}

impl TryFrom<protobuf::ResourceBounds> for ValidResourceBounds {
    type Error = ProtobufConversionError;
    fn try_from(value: protobuf::ResourceBounds) -> Result<Self, Self::Error> {
        let Some(l1_gas) = value.l1_gas else {
            return Err(missing("ResourceBounds::l1_gas"));
        };
        let Some(l2_gas) = value.l2_gas else {
            return Err(missing("ResourceBounds::l2_gas"));
        };
        // TODO(Shahak): Assert data gas is not none once we remove support for 0.13.2.
        let l1_data_gas = value.l1_data_gas.unwrap_or_default();
        let l1_gas: ResourceBounds = l1_gas.try_into()?;
        let l2_gas: ResourceBounds = l2_gas.try_into()?;
        let l1_data_gas: ResourceBounds = l1_data_gas.try_into()?;
        Ok(if l1_data_gas.is_zero() && l2_gas.is_zero() {
            ValidResourceBounds::L1Gas(l1_gas)
        } else {
            ValidResourceBounds::AllResources(AllResourceBounds { l1_gas, l2_gas, l1_data_gas })
        })
    }
}
