### Title
Unsigned `oraclePriceX18` in `SignedMintNlp` Enables Unbacked NLP Minting via Slow-Mode — (`core/contracts/EndpointTx.sol` / `core/contracts/Clearinghouse.sol`)

---

### Summary

The `oraclePriceX18` field in `SignedMintNlp` is **not included in the EIP-712 digest** that the user signs. Any caller can submit a `MintNlp` transaction via the slow-mode path with `oraclePriceX18 = 1`, causing `nlpAmount = quoteAmount / 1`, minting an unbounded quantity of NLP tokens for a trivial quote deposit and diluting all existing NLP holders.

---

### Finding Description

**Step 1 — The signed data does not cover `oraclePriceX18`.**

`SignedMintNlp` has four fields: the inner `MintNlp tx` struct, `signature`, `oraclePriceX18`, and `nlpPoolRebalanceX18`. [1](#0-0) 

In `Verifier.sol`, the EIP-712 digest for `MintNlp` hashes only `sender`, `quoteAmount`, and `nonce` — `oraclePriceX18` is decoded from the struct but **never included**: [2](#0-1) 

**Step 2 — `MintNlp` is reachable via slow-mode by any user.**

`submitSlowModeTransactionImpl` only restricts a specific set of admin-only transaction types to `owner`. `MintNlp` is not in that list, so it falls through to the `else` branch and is accepted from any caller who pays the slow-mode fee: [3](#0-2) 

**Step 3 — The unsigned `oraclePriceX18` is used verbatim to compute `nlpAmount`.**

In `processTransactionImpl`, after `validateSignedTx` passes (it only validates the signature over `{sender, quoteAmount, nonce}`), `signedTx.oraclePriceX18` is forwarded directly to `clearinghouse.mintNlp` with no oracle cross-check: [4](#0-3) 

Inside `mintNlp`, the NLP amount is computed as:

```solidity
int128 nlpAmount = quoteAmount.div(oraclePriceX18);
``` [5](#0-4) 

Setting `oraclePriceX18 = 1` makes `nlpAmount = quoteAmount`, i.e., a 1:1 ratio regardless of the true NLP price.

---

### Impact Explanation

An attacker deposits a small `quoteAmount` (e.g., 100 USDC in X18 units), constructs `SignedMintNlp` with `oraclePriceX18 = 1`, signs only the inner `MintNlp{sender, quoteAmount, nonce}` struct (which is all the signature covers), and submits via slow-mode. After the 3-day delay, they receive `nlpAmount = quoteAmount / 1` NLP tokens — orders of magnitude more than the fair share. This:

- Creates unbacked NLP supply (the NLP token balance of `N_ACCOUNT` is debited by the same inflated amount)
- Dilutes every existing NLP holder's redemption value proportionally
- Allows the attacker to immediately `burnNlp` at the fair price, extracting real quote from the NLP pool

---

### Likelihood Explanation

The slow-mode path is permissionless (any non-sanctioned address can submit). The attacker needs only to: (a) hold a registered subaccount, (b) pay the slow-mode fee, and (c) wait 3 days. No sequencer cooperation, no admin key, no oracle manipulation is required. The attack is fully self-contained on-chain.

---

### Recommendation

Include `oraclePriceX18` in the EIP-712 digest so that the price is committed to by the signer and cannot be altered post-signing:

```solidity
// Verifier.sol — MintNlp digest
digest = keccak256(
    abi.encode(
        keccak256(bytes(MINT_NLP_SIGNATURE)),
        signedTx.tx.sender,
        signedTx.tx.quoteAmount,
        signedTx.tx.nonce,
        signedTx.oraclePriceX18   // ADD THIS
    )
);
```

Additionally, add an on-chain oracle sanity bound in `mintNlp` (e.g., require `oraclePriceX18` to be within X% of a trusted on-chain price feed) as a defense-in-depth measure.

---

### Proof of Concept

```solidity
// Hardhat test (pseudocode)
const fairPrice = ethers.utils.parseUnits("1", 18); // 1e18 = $1 in X18
const quoteAmount = ethers.utils.parseUnits("100", 18); // 100 USDC

// Victim mints at fair price (submitted by sequencer normally)
// attacker.nlpBalance_before = 0

// Attacker constructs SignedMintNlp with oraclePriceX18 = 1
const maliciousTx = {
    tx: { sender: attacker, quoteAmount, nonce: attackerNonce },
    signature: sign({ sender, quoteAmount, nonce }),  // valid sig, no price
    oraclePriceX18: 1,                                // NOT covered by sig
    nlpPoolRebalanceX18: [quoteAmount]                // sums to quoteAmount ✓
};

// Submit via slow-mode (pays fee, waits 3 days)
await endpoint.submitSlowModeTransaction(encode(MintNlp, maliciousTx));
// ... advance time 3 days ...
await endpoint.executeSlowModeTransaction();

// nlpAmount = quoteAmount / 1 = 100e18 NLP
// Fair nlpAmount would be = 100e18 / fairPrice = 100 NLP
// Attacker received 1e18x more NLP than entitled
// assert(attackerNlpBalance == quoteAmount)  // 100e18 instead of 100
```
