Audit Report

## Title
`SwapAllowlistExtension.beforeSwap` checks the router address as `sender` instead of the originating user, enabling full allowlist bypass via router-mediated swaps — (`metric-periphery/contracts/extensions/SwapAllowlistExtension.sol`)

## Summary
`SwapAllowlistExtension` is intended to restrict pool swaps to a per-pool allowlist of user addresses. However, the `sender` argument it checks is the pool's `msg.sender` at the time `pool.swap(...)` is called — which is `MetricOmmSimpleRouter` when swaps are routed through it, not the end user. Because the router must itself be allowlisted for any router-mediated swap to succeed, any unprivileged user can bypass the per-user restriction simply by routing through the router.

## Finding Description

In `MetricOmmPool.swap()`, `_beforeSwap` is called with `msg.sender` as the `sender` argument:

```solidity
_beforeSwap(
    msg.sender,   // ← router address when called via MetricOmmSimpleRouter
    recipient,
    ...
);
``` [1](#0-0) 

`ExtensionCalling._beforeSwap` forwards this value unchanged as the first argument to `IMetricOmmExtensions.beforeSwap`: [2](#0-1) 

`SwapAllowlistExtension.beforeSwap` then checks:

```solidity
if (!allowAllSwappers[msg.sender] && !allowedSwapper[msg.sender][sender]) {
    revert IMetricOmmPoolActions.NotAllowedToSwap();
}
``` [3](#0-2) 

Here `msg.sender` is the pool (the extension's caller) and `sender` is the router address. The check therefore asks: "is the router allowlisted for this pool?" — not "is the originating user allowlisted?"

**Exploit path:**
1. Pool admin deploys a pool with `SwapAllowlistExtension` and adds only specific trusted addresses to `allowedSwapper[pool]`.
2. Pool admin also adds `MetricOmmSimpleRouter` to `allowedSwapper[pool]` (required for any router-mediated swap to work).
3. An unprivileged user — not on the allowlist — calls `MetricOmmSimpleRouter.exactInputSingle(...)` targeting the restricted pool.
4. The router calls `pool.swap(recipient, ...)`, so the pool's `msg.sender` is the router.
5. `beforeSwap` receives `sender = router`, finds `allowedSwapper[pool][router] == true`, and passes — the swap executes despite the user not being allowlisted.

`MetricOmmSimpleRouter.exactInputSingle` calls `pool.swap` directly with no forwarding of the original `msg.sender`: [4](#0-3) 

## Impact Explanation
The `SwapAllowlistExtension` is rendered completely ineffective for any pool that permits router-mediated swaps. Any user can bypass the per-user swap restriction, defeating the core access-control invariant the extension is designed to enforce. This constitutes broken core pool functionality — the allowlist gate — which is a required impact under the contest rules (broken core pool functionality causing loss of funds or unusable flows). Pools intended to be restricted to KYC'd or whitelisted counterparties are fully open to any caller via the router.

## Likelihood Explanation
Exploitation requires no special privileges. Any user with access to `MetricOmmSimpleRouter` can trigger this. The only precondition is that the router is allowlisted on the target pool, which is a necessary operational condition for the router to be usable at all. The bypass is repeatable and deterministic.

## Recommendation
Pass the originating user through the call chain rather than the immediate `msg.sender`. One approach: have the pool accept an explicit `originator` parameter in `swap()` (verified against `msg.sender` or a trusted router registry), or have the router pass the end user's address in `extensionData` and have the extension decode it. Alternatively, the extension should check `tx.origin` only as a last resort (with reentrancy caveats), or the pool should expose a trusted-forwarder pattern so the router can attest the real user address in a verifiable way.

## Proof of Concept
```solidity
// 1. Deploy pool with SwapAllowlistExtension
// 2. allowedSwapper[pool][alice] = true   (alice is the only allowed user)
// 3. allowedSwapper[pool][router] = true  (router must be allowed for routing)
// 4. Bob (not allowlisted) calls:
router.exactInputSingle(ExactInputSingleParams({
    pool: restrictedPool,
    recipient: bob,
    tokenIn: token0,
    ...
}));
// pool.swap(msg.sender=router) → beforeSwap(sender=router) → allowedSwapper[pool][router]=true → PASSES
// Bob's swap executes despite not being on the allowlist.
```
