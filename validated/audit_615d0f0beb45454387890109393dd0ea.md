### Title
Unauthenticated Bootstrap Declare Allows Any Caller to Occupy Class Hash Slots and Bypass Account Validation — (`crates/starknet_api/src/executable_transaction.rs`, `crates/apollo_gateway/src/gateway.rs`)

---

### Summary

The `bootstrap_address()` (`0x424f4f545354524150` = ASCII "BOOTSTRAP") is a hardcoded, publicly documented constant. Any external user can craft a `DeclareTransactionV3` with `sender_address = BOOTSTRAP`, `nonce = 0`, and zero resource bounds. When the gateway is in bootstrap mode (`validate_resource_bounds = false`), this transaction bypasses signature verification, fee enforcement, and nonce checks, and permanently occupies a class hash slot. The gateway's `check_declare_permissions` does not specifically restrict the BOOTSTRAP address, and the default `authorized_declarer_accounts = None` authorizes all accounts unconditionally.

---

### Finding Description

The bootstrap declare mechanism is designed to let the system bootstrapper declare classes without fee or nonce requirements. However, the mechanism relies entirely on a well-known constant address rather than any cryptographic authentication.

**`is_bootstrap_declare()` in `crates/starknet_api/src/executable_transaction.rs`:**

```rust
pub fn is_bootstrap_declare(&self, charge_fee: bool) -> bool {
    if let crate::transaction::DeclareTransaction::V3(tx) = &self.tx {
        return tx.sender_address == Self::bootstrap_address()
            && tx.nonce == Nonce(Felt::ZERO)
            && !charge_fee;
    }
    false
}

pub fn bootstrap_address() -> ContractAddress {
    // A felt representation of the string 'BOOTSTRAP'.
    ContractAddress::from(0x424f4f545354524150_u128)
}
```

None of the three conditions (`sender_address`, `nonce`, `!charge_fee`) require cryptographic proof of identity. The BOOTSTRAP address has no deployed contract, so there is no `__validate_declare__` entry point to run. The execution path in `crates/blockifier/src/transaction/account_transaction.rs` explicitly skips all account-related actions:

```rust
if let Transaction::Declare(tx) = &self.tx {
    if tx.is_bootstrap_declare(self.execution_flags.charge_fee) {
        // ... runs declare only, no validation, no fee, no nonce bump
        return Ok(TransactionExecutionInfo::default());
    }
}
```

The OS Cairo code in `crates/apollo_starknet_os_program/src/cairo/starkware/starknet/core/os/execution/transaction_impls.cairo` enforces the same path and writes the class hash with `prev_value=0`, meaning **a class can only be declared once**:

**The gateway's `check_declare_permissions` in `crates/apollo_gateway/src/gateway.rs` does not restrict the BOOTSTRAP address:**


And `is_authorized_declarer` in `crates/apollo_gateway_config/src/config.rs`:

```rust
pub fn is_authorized_declarer(&self, declarer_address: &ContractAddress) -> bool {
    match &self.static_config.authorized_declarer_accounts {
        Some(allowed_accounts) => allowed_accounts.contains(declarer_address),
        None => true,   // <-- default: ALL accounts authorized
    }
}
```

The default production config has `authorized_declarer_accounts: None`, meaning every address — including the BOOTSTRAP address — is authorized to declare.

---

### Impact Explanation

When bootstrap mode is active (`validate_resource_bounds = false`), any external user can:

1. Submit a `DeclareTransactionV3` from the BOOTSTRAP address with zero resource bounds — no private key required, no signature checked, no fee paid, no nonce consumed.
2. Permanently occupy a class hash slot (`prev_value=0` enforces once-only declaration in the OS).
3. Front-run the legitimate bootstrapper's pending declarations observed in the mempool, causing the bootstrapper's transactions to fail with "class already declared."
4. Flood the system with bootstrap declares, consuming block space and delaying legitimate bootstrap declarations.

This matches: **High — Mempool/gateway/RPC admission accepts invalid (unauthorized) transactions before sequencing**, and **Critical — Invalid or unauthorized Starknet transaction accepted through account validation logic** (the bootstrap path explicitly bypasses `__validate_declare__`).

---

### Likelihood Explanation

The BOOTSTRAP address (`0x424f4f545354524150`) is publicly documented in the codebase and trivially derivable from the ASCII string "BOOTSTRAP". Any attacker who monitors the mempool during the bootstrap phase can observe pending bootstrap declares and front-run them. No special resources, tokens, or privileges are required — only the ability to submit a standard RPC transaction to the gateway.

---

### Recommendation

1. **Restrict the BOOTSTRAP address at the gateway level**: When `authorized_declarer_accounts` is set, explicitly include the BOOTSTRAP address only if the operator intends to allow bootstrap declares. When `authorized_declarer_accounts = None`, the BOOTSTRAP address should still require explicit opt-in.
2. **Add a dedicated config flag** (e.g., `allow_bootstrap_declares: bool`) that must be explicitly set to `true` to permit bootstrap declares, independent of the general `authorized_declarer_accounts` list.
3. **Alternatively**, require bootstrap declares to carry a signature from a known operator key, verified before the bootstrap path is taken.

---

### Proof of Concept

1. Configure the gateway with `validate_resource_bounds = false` (bootstrap mode).
2. Obtain any valid Sierra class and compute its `class_hash` and `compiled_class_hash` (Blake2, `HashVersion::V2`).
3. Craft a `DeclareTransactionV3`:
   - `sender_address = 0x424f4f545354524150` (BOOTSTRAP)
   - `nonce = 0`
   - `resource_bounds = AllResourceBounds { l1_gas: 0, l2_gas: 0, l1_data_gas: 0 }`
   - `compiled_class_hash` = Blake2 hash of the CASM
   - `contract_class` = the Sierra class
4. Submit to the gateway's `add_tx` endpoint.
5. The gateway's `check_declare_permissions` passes (all accounts authorized by default).
6. `convert_rpc_tx_to_internal_rpc_tx` compiles the class and verifies `compiled_class_hash`.
7. The stateful validator runs `execute_raw`, detects `is_bootstrap_declare(charge_fee=false) = true`, and declares the class with no signature check, no fee, no nonce increment.
8. The class hash slot is permanently occupied.
9. The legitimate bootstrapper's subsequent transaction for the same class fails: `DeclareTransactionError { class_hash }` — "class already declared." [1](#0-0) [2](#0-1) [3](#0-2) [4](#0-3) [5](#0-4) [6](#0-5)

```rust
fn try_declare<S: State>(
    tx: &DeclareTransaction,
    state: &mut S,
    class_hash: ClassHash,
    compiled_class_hash: Option<CompiledClassHash>,
) -> TransactionExecutionResult<()> {
    match state.get_compiled_class(class_hash) {
        Err(StateError::UndeclaredClassHash(_)) => {
            // Class is undeclared; declare it.
            state.set_contract_class(class_hash, tx.contract_class().try_into()?)?;
            if let Some(compiled_class_hash) = compiled_class_hash {
                state.set_compiled_class_hash(class_hash, compiled_class_hash)?;
            }
            Ok(())
        }
        Err(error) => Err(error)?,
        Ok(_) => {
            // Class is already declared, cannot redeclare.
            Err(TransactionExecutionError::DeclareTransactionError { class_hash })
        }
    }
```
