### Title
Consensus Precommit Votes Accepted Without Signature Verification, Enabling Quorum Forgery by Any Peer — (`crates/apollo_consensus/src/single_height_consensus.rs`)

### Summary

Two symmetric TODOs leave the consensus vote path completely unsigned and unverified in production code. Self-votes are broadcast with `RawSignature::default()` (empty `Vec`), and incoming votes from peers are accepted without any call to `verify_precommit_vote_signature`. Any network peer can forge a precommit vote for any committee-member address and have it counted toward quorum.

### Finding Description

**Unsigned self-votes.** `make_self_vote` in `state_machine.rs` constructs every outgoing vote with an explicit empty signature: [1](#0-0) 

The `// TODO(Asmaa): sign the vote` comment confirms this is intentionally deferred, not accidentally omitted.

**No verification on incoming votes.** `handle_vote` in `single_height_consensus.rs` performs only two checks before accepting a peer vote: height equality and committee membership of the `voter` field: [2](#0-1) 

The `// TODO(Asmaa): verify the signature` comment at line 242 is the only reference to verification. `verify_precommit_vote_signature` — which is correctly implemented in `signature_manager.rs` — is never called anywhere in the consensus crate: [3](#0-2) 

**The committee-membership check does not authenticate the sender.** Line 249 checks that `vote.voter` is a known committee address, but it does not verify that the peer who sent the message controls the private key for that address. Without signature verification, the `voter` field is fully attacker-controlled. [4](#0-3) 

### Impact Explanation

An unprivileged peer (no validator key required) can:

1. Connect to any consensus node over the P2P network.
2. Craft `Vote` messages with `vote_type = Precommit`, `voter = <any_committee_member_address>`, `proposal_commitment = <target_block_hash>`, and `signature = []`.
3. Send one such message per committee member until the forged weight sum crosses the quorum threshold.
4. `upon_decision` in `state_machine.rs` fires, emitting `SMRequest::DecisionReached` for a block that was never actually approved by the real validators. [5](#0-4) 

This maps to **Critical — wrong state committed** because the decided block hash drives all subsequent state, receipt, event, and L1-message derivation.

### Likelihood Explanation

The path is reachable by any peer that can open a P2P connection. No key material, staking, or operator access is required. The attack is deterministic: forge one vote per committee member, observe the `DecisionReached` event. The only practical constraint is knowing the current committee addresses, which are public on-chain.

### Recommendation

1. **Sign outgoing votes**: replace `RawSignature::default()` in `make_self_vote` with an actual call to `SignatureManager::sign_precommit_vote` (or the prevote equivalent once that domain separator is defined).
2. **Verify incoming votes**: in `handle_vote`, after the committee-membership check, resolve the voter's public key from the committee, call `verify_precommit_vote_signature(block_hash, vote.signature.clone(), public_key)`, and drop the vote if verification fails or returns `false`.
3. **Remove the TODO stubs** once both sides are implemented and covered by integration tests that exercise the rejection path.

    fn upon_decision(&mut self, round: u32) -> VecDeque<SMRequest> {
        let Some((Some(proposal_id), _)) = self.proposals.get(&round) else {
            return VecDeque::new();
        };
        if !self.value_has_enough_votes(&self.precommits, round, &Some(*proposal_id), &self.quorum)
        {
            return VecDeque::new();
        }
        if !self.virtual_proposer_in_favor(&self.precommits, round, &Some(*proposal_id)) {
            return VecDeque::new();
        }
        // Collect all supporting precommits for this proposal and round.
        let supporting_precommits: Vec<Vote> = self
            .precommits
            .iter()
            .filter(|(&(r, _voter), (v, _w))| {
                r == round && v.proposal_commitment == Some(*proposal_id)
            })
            .map(|(_vote_key, (v, _w))| v.clone())
            .collect();

        let decision = Decision { precommits: supporting_precommits, block: *proposal_id, round };
        VecDeque::from([SMRequest::DecisionReached(decision)])
    }

    pub(crate) fn handle_vote(&mut self, vote: Vote) -> Requests {
        // TODO(Asmaa): verify the signature
        trace!("Received {:?}", vote);
        let height = self.state_machine.height();
        if vote.height != height {
            warn!("Invalid vote height: expected {:?}, got {:?}", height, vote.height);
            return VecDeque::new();
        }
        if !self.committee.members().iter().any(|s| s.address == vote.voter) {
            debug!("Ignoring vote from non validator: vote={:?}", vote);
            return VecDeque::new();
        }

        // Check if vote has already been received.
        match self.state_machine.received_vote(&vote) {
            VoteStatus::Duplicate => {
                // Duplicate - ignore.
                trace_every_n_ms!(
                    DUPLICATE_VOTE_LOG_PERIOD_MS,
                    "Ignoring duplicate vote: {vote:?}"
                );
                return VecDeque::new();
            }
            VoteStatus::Conflict(old_vote, new_vote) => {
                // Conflict - ignore and record.
                warn!("Conflicting votes: old={old_vote:?}, new={new_vote:?}");
                CONSENSUS_CONFLICTING_VOTES.increment(1);
                return VecDeque::new();
            }
            VoteStatus::New => {
                // Vote is new, proceed to process it.
            }
        }

        info!("Accepting {:?}", vote);
        let sm_vote = match vote.vote_type {
            VoteType::Prevote => StateMachineEvent::Prevote(vote),
            VoteType::Precommit => StateMachineEvent::Precommit(vote),
        };
        self.state_machine.handle_event(sm_vote)
    }
