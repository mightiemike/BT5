### Title
ML-DSA-65 Gas-Key Host Functions Use Wire Length Instead of Trie-ID Length for Exec-Fee Calculation — (`File: runtime/near-vm-runner/src/logic/logic.rs`, `runtime/near-vm-runner/src/wasmtime_runner/logic.rs`)

---

### Summary

When a contract invokes `promise_batch_action_transfer_to_gas_key`, `promise_batch_action_add_gas_key_with_full_access`, or `promise_batch_action_add_gas_key_with_function_call` with an ML-DSA-65 public key, the exec-fee calculation uses the raw wire-format length supplied by the contract (`public_key_len as usize` = 1953 bytes) instead of the on-trie identifier length (`public_key.trie_id_len()` = 33 bytes). The same action sent as a direct transaction uses the correct `trie_id_len()` path. The divergence is ~58×, producing a protocol-level fee inconsistency that makes ML-DSA-65 gas keys economically unusable from contracts and breaks the invariant that exec fees reflect actual trie-write costs.

---

### Finding Description

`PublicKey::len()` returns the borsh-encoded wire size (1 + 1952 = 1953 bytes for ML-DSA-65), while `PublicKey::trie_id_len()` returns the on-trie identifier size (1 + 32 = 33 bytes — the SHA3-256 hash form). The two diverge only for `PublicKey::MLDSA65`. [1](#0-0) 

The exec-fee helpers `gas_key_transfer_exec_fee` and `gas_key_add_key_exec_fee` accept a `public_key_len: usize` parameter that is documented and used as the **trie key length** component: [2](#0-1) [3](#0-2) 

The **correct** call site in `runtime/runtime/src/config.rs` passes `action.public_key.trie_id_len()`: [4](#0-3) 

The **incorrect** call sites in both VM logic backends pass `public_key_len as usize` — the raw byte count of the borsh-encoded key as supplied by the contract's host-function argument:

```
// logic.rs (interpreter path)
let send = gas_key_transfer_send_fee(&self.fees_config, sir, public_key_len as usize);
let exec = gas_key_transfer_exec_fee(
    &self.fees_config,
    receiver_id.len(),
    public_key_len as usize,   // ← wire length, not trie_id_len
);
``` [5](#0-4) 

```
// wasmtime_runner/logic.rs (wasmtime path)
let send = gas_key_transfer_send_fee(&ctx.fees_config, sir, public_key_len as usize);
let exec =
    gas_key_transfer_exec_fee(&ctx.fees_config, receiver_id.len(), public_key_len as usize);
``` [6](#0-5) 

The same wrong argument appears in `promise_batch_action_add_gas_key_with_full_access`: [7](#0-6) [8](#0-7) 

And in `promise_batch_action_add_gas_key_with_function_call`: [9](#0-8) 

The exact divergent values for ML-DSA-65:

| Path | `public_key_len` passed | Trie key length computed |
|---|---|---|
| Direct transaction (`exec_fee` in `config.rs`) | `trie_id_len()` = **33** | correct |
| Contract host function (both VM backends) | wire `len()` = **1953** | ~58× inflated |

---

### Impact Explanation

**High.** The exec fee for `TransferToGasKey`, `WithdrawFromGasKey`, and `AddKey` (gas-key variants) when invoked from a contract with an ML-DSA-65 key is computed as if the trie key contains a 1953-byte pubkey instead of a 33-byte hash. The `gas_key_byte` rate is multiplied by the full trie-key length, so the overcharge is approximately `(1953 − 33) × gas_key_byte × (account_id_len + 2)` gas units per action. This makes ML-DSA-65 gas keys economically unusable from contracts, breaks the protocol invariant that exec fees reflect actual trie-write costs, and creates a deterministic but incorrect fee schedule that diverges from the direct-transaction path for the same action.

---

### Likelihood Explanation

**Medium.** The `PostQuantumSignatures` protocol feature must be active. Once active, any contract that manages gas keys using ML-DSA-65 public keys will hit this on every `TransferToGasKey`, `WithdrawFromGasKey`, or `AddKey` (gas-key) promise. The discrepancy is large enough (~58×) to cause gas-budget exhaustion or make the feature economically unviable from contracts, so it will surface immediately in any integration test or real usage.

---

### Recommendation

In both `runtime/near-vm-runner/src/logic/logic.rs` and `runtime/near-vm-runner/src/wasmtime_runner/logic.rs`, after decoding the public key from the host-function argument, replace `public_key_len as usize` with `public_key.trie_id_len()` in all calls to `gas_key_transfer_exec_fee` and `gas_key_add_key_exec_fee`. The decoded `public_key` is already available at those call sites (e.g. `public_key.decode()?`). This mirrors the correct pattern already used in `runtime/runtime/src/config.rs` at lines 348 and 352. [4](#0-3)

### Proof of Concept

1. Deploy a contract that calls `promise_batch_action_transfer_to_gas_key` with an ML-DSA-65 public key (1953-byte borsh encoding, so `public_key_len = 1953`).
2. Observe the gas charged for the exec portion of the action.
3. Send the identical `TransferToGasKey` action as a direct transaction (not via a contract promise).
4. Compare exec fees: the contract path charges `access_key_key_len(account_id_len, 1953) × gas_key_byte` while the direct path charges `access_key_key_len(account_id_len, 33) × gas_key_byte` — a ratio of approximately `(2 + account_id_len + 1953) / (2 + account_id_len + 33)`, which for a 10-character account ID is `(1965) / (45)` ≈ **43.7×** overcharge on the per-byte component. impl PublicKey {
    /// Length of this public key's borsh encoding, in bytes - that is,
    /// the on-the-wire size of the raw key bytes plus a 1-byte borsh
    /// discriminant tag (the leading `+ 1` in each arm).
    ///
    /// For storage-fee accounting use [`PublicKey::trie_id_len`] instead;
    /// for ML-DSA-65 those two diverge (1953 wire vs 33 on-trie).
    // `is_empty` always returns false, so there is no point in adding it
    #[allow(clippy::len_without_is_empty)]
    pub fn len(&self) -> usize {
        match self {
            Self::ED25519(_) => 1 + ed25519_dalek::PUBLIC_KEY_LENGTH,
            Self::SECP256K1(_) => 1 + 64,
            Self::MLDSA65(_) => 1 + ML_DSA_65_PUBLIC_KEY_LENGTH,
        }
    }

    pub fn empty(key_type: KeyType) -> Self {
        match key_type {
            KeyType::ED25519 => {
                PublicKey::ED25519(ED25519PublicKey([0u8; ed25519_dalek::PUBLIC_KEY_LENGTH]))
            }
            KeyType::SECP256K1 => PublicKey::SECP256K1(Secp256K1PublicKey([0u8; 64])),
            KeyType::MLDSA65 => {
                PublicKey::MLDSA65(MlDsa65PublicKey(Box::new([0u8; ML_DSA_65_PUBLIC_KEY_LENGTH])))
            }
        }
    }

    pub fn key_type(&self) -> KeyType {
        match self {
            Self::ED25519(_) => KeyType::ED25519,
            Self::SECP256K1(_) => KeyType::SECP256K1,
            Self::MLDSA65(_) => KeyType::MLDSA65,
        }
    }

    fn key_tag(&self) -> KeyTag {
        match self {
            PublicKey::ED25519(_) => KeyTag::Ed25519,
            PublicKey::SECP256K1(_) => KeyTag::Secp256k1,
            PublicKey::MLDSA65(_) => KeyTag::MlDsa65Full,
        }
    }

    pub fn key_data(&self) -> &[u8] {
        match self {
            Self::ED25519(key) => key.as_ref(),
            Self::SECP256K1(key) => key.as_ref(),
            Self::MLDSA65(key) => key.as_ref(),
        }
    }

    pub fn unwrap_as_ed25519(&self) -> &ED25519PublicKey {
        match self {
            Self::ED25519(key) => key,
            Self::SECP256K1(_) | Self::MLDSA65(_) => panic!(),
        }
    }

    pub fn unwrap_as_secp256k1(&self) -> &Secp256K1PublicKey {
        match self {
            Self::SECP256K1(key) => key,
            Self::ED25519(_) | Self::MLDSA65(_) => panic!(),
        }
    }

    /// Length, in bytes, of the on-trie identifier for an access-key
    /// entry owned by this public key. For ed25519/secp256k1 this matches
    /// `len()`; for ML-DSA-65 the trie stores a SHA3-256 hash (33 bytes
    /// including the type tag), not the 1953-byte borsh-encoded pubkey.
    /// Used by storage-fee calculations on the runtime side; cheap to call
    /// (no hashing) - for ML-DSA-65 this returns the size of the digest
    /// form without actually hashing the pubkey.
    pub fn trie_id_len(&self) -> usize {
        match self {
            Self::ED25519(_) => 1 + ed25519_dalek::PUBLIC_KEY_LENGTH,
            Self::SECP256K1(_) => 1 + 64,
            Self::MLDSA65(_) => 1 + ML_DSA_65_HASH_LENGTH,
        }
    }

/// Based on the access key trie key length + estimated value length (what the
/// receiver needs to read/write in the trie).
pub fn gas_key_transfer_exec_fee(
    cfg: &RuntimeFeesConfig,
    account_id_len: usize,
    public_key_len: usize,
) -> GasKeyTransferFee {
    let base = cfg.fee(ActionCosts::gas_key_transfer_base).exec_fee();
    let trie_key_len = access_key_key_len(account_id_len, public_key_len);
    let estimated_value_len = AccessKey::min_gas_key_borsh_len();
    let per_byte = cfg
        .fee(ActionCosts::gas_key_byte)
        .exec_fee()
        .checked_mul((trie_key_len + estimated_value_len) as u64)
        .unwrap();
    GasKeyTransferFee { base, per_byte }
}

pub fn gas_key_add_key_exec_fee(
    cfg: &RuntimeFeesConfig,
    account_id_len: usize,
    public_key_len: usize,
    num_nonces: NonceIndex,
) -> GasKeyAddFee {
    let num_nonces = num_nonces as u64;
    let base =
        cfg.fee(ActionCosts::gas_key_nonce_write_base).exec_fee().checked_mul(num_nonces).unwrap();
    let nonce_key_len =
        access_key_key_len(account_id_len, public_key_len) + std::mem::size_of::<NonceIndex>();
    let per_byte = cfg
        .fee(ActionCosts::gas_key_byte)
        .exec_fee()
        .checked_mul((nonce_key_len + AccessKey::NONCE_VALUE_LEN) as u64)
        .unwrap()
        .checked_mul(num_nonces)
        .unwrap();
    GasKeyAddFee { base, per_byte }
}

/// Returns the length of the trie key for an access key.
///
/// The trie key format is: `[col_prefix] [account_id] [separator] [public_key]`
/// where the prefix and separator are each a single `u8` byte.
pub fn access_key_key_len(account_id_len: usize, public_key_len: usize) -> usize {
    COLUMN_PREFIX_LEN + account_id_len + ACCESS_KEY_SEPARATOR_LEN + public_key_len
}
