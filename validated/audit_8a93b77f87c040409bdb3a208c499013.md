### Title
Transferred NFTs Become Permanently Uneditable and Unburnable Due to Dual-Guard Deadlock — (`x/nft/keeper/keeper.go`)

### Summary

`EditNFT` and `BurnNFT` both require the caller to satisfy **two independent guards simultaneously**: `IsOwner` AND `IsDenomCreator`. Once the denom creator transfers an NFT to any third party via `TransferOwner`, no address can ever satisfy both guards for that token simultaneously, permanently locking it from being edited or burned.

### Finding Description

`EditNFT` enforces two sequential checks: [1](#0-0) 

`BurnNFT` enforces the same two sequential checks: [2](#0-1) 

`IsOwner` returns an error if the caller is not the current NFT owner: [3](#0-2) 

`IsDenomCreator` returns an error if the caller is not the address stored in `denom.Creator` at issuance time — this field is **never updated**: [4](#0-3) 

`TransferOwner` only checks `IsOwner` (not `IsDenomCreator`), so the denom creator can freely transfer any NFT they own: [5](#0-4) 

After `swapOwner` executes, the denom creator is no longer the owner: [6](#0-5) 

**Resulting deadlock:**

| Caller | `IsOwner` | `IsDenomCreator` | Result |
|---|---|---|---|
| Alice (creator, ex-owner) | FAIL | PASS | `ErrUnauthorized` |
| Bob (new owner, non-creator) | PASS | FAIL | `ErrUnauthorized` |

Neither party can satisfy both guards. The NFT is permanently frozen.

### Impact Explanation

- **Supply invariant broken**: `BurnNFT` is permanently blocked for the transferred token. The on-chain supply counter for that denom can never be decremented for that token, even if the denom creator wants to retire it.
- **Metadata permanently frozen**: `EditNFT` is permanently blocked. URI, name, and data fields are immutable for the lifetime of the chain for that token.
- `BurnNFTUnverified` (which skips `IsDenomCreator`) is only reachable via the IBC transfer path, not via `MsgBurnNFT`: [7](#0-6) 

### Likelihood Explanation

Any denom creator who mints NFTs and transfers them to users (the primary intended use case) triggers this condition. It requires no special privileges, no governance action, and no attacker cooperation — the denom creator's own normal usage of `MsgTransferNFT` is sufficient.

### Recommendation

Decouple the authorization model. Two reasonable fixes:

1. **Allow the denom creator to edit/burn regardless of current ownership** — change `EditNFT` and `BurnNFT` to check `IsOwner` OR `IsDenomCreator` (not AND).
2. **Allow the current owner to burn their own NFT** — remove the `IsDenomCreator` check from `BurnNFT`, keeping it only in `EditNFT` if metadata control is intended to remain with the creator.

### Proof of Concept

```go
// keeper_test.go
alice := sdk.AccAddress([]byte("alice"))
bob   := sdk.AccAddress([]byte("bob"))

// Alice issues denom and mints NFT to herself
k.IssueDenom(ctx, "denom1", "Denom", "", "", alice)
k.MintNFT(ctx, "denom1", "nft1", "NFT", "", "", alice, alice)

// Alice transfers to Bob
k.TransferOwner(ctx, "denom1", "nft1", alice, bob)

// Both EditNFT calls fail
err := k.EditNFT(ctx, "denom1", "nft1", "new", "", "", alice)
require.ErrorIs(t, err, types.ErrUnauthorized) // alice not owner

err = k.EditNFT(ctx, "denom1", "nft1", "new", "", "", bob)
require.ErrorIs(t, err, types.ErrUnauthorized) // bob not creator

// Both BurnNFT calls fail
err = k.BurnNFT(ctx, "denom1", "nft1", alice)
require.ErrorIs(t, err, types.ErrUnauthorized) // alice not owner

err = k.BurnNFT(ctx, "denom1", "nft1", bob)
require.ErrorIs(t, err, types.ErrUnauthorized) // bob not creator
// NFT is now permanently locked
```

```go
func (k Keeper) TransferOwner(
	ctx sdk.Context, denomID, tokenID string, srcOwner, dstOwner sdk.AccAddress,
) error {
	if !k.HasDenomID(ctx, denomID) {
		return sdkerrors.Wrapf(types.ErrInvalidDenom, "denom ID %s not exists", denomID)
	}

	nft, err := k.IsOwner(ctx, denomID, tokenID, srcOwner)
	if err != nil {
		return err
	}

	nft.Owner = dstOwner.String()

	k.setNFT(ctx, denomID, nft)
	k.swapOwner(ctx, denomID, tokenID, srcOwner, dstOwner)
	return nil
}
```

```go
// BurnNFTUnverified deletes a specified NFT without verifying if the owner is the creator of denom
// Needed for IBC transfer of NFT
func (k Keeper) BurnNFTUnverified(ctx sdk.Context, denomID, tokenID string, owner sdk.AccAddress) error {
	if !k.HasDenomID(ctx, denomID) {
		return sdkerrors.Wrapf(types.ErrInvalidDenom, "denom ID %s not exists", denomID)
	}

	nft, err := k.IsOwner(ctx, denomID, tokenID, owner)
	if err != nil {
		return err
	}

	k.deleteNFT(ctx, denomID, nft)
	k.deleteOwner(ctx, denomID, tokenID, owner)
	k.decreaseSupply(ctx, denomID)

	return nil
```

```go
func (k Keeper) IsDenomCreator(ctx sdk.Context, denomID string, address sdk.AccAddress) (types.Denom, error) {
	denom, err := k.GetDenom(ctx, denomID)
	if err != nil {
		return types.Denom{}, err
	}

	creator, err := sdk.AccAddressFromBech32(denom.Creator)
	if err != nil {
		panic(err)
	}

	if !creator.Equals(address) {
		return types.Denom{}, sdkerrors.Wrapf(types.ErrUnauthorized, "%s is not the creator of %s", address, denomID)
	}

	return denom, nil
```
