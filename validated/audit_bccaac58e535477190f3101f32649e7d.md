Audit Report

## Title
SwapAllowlistExtension Gates the Router Address Instead of the End User, Allowing Any User to Bypass a Curated Pool's Swap Allowlist via MetricOmmSimpleRouter - (File: metric-periphery/contracts/extensions/SwapAllowlistExtension.sol)

## Summary
`SwapAllowlistExtension.beforeSwap` checks the `sender` argument, which is `msg.sender` of the pool's `swap` call — the immediate caller, not the economic actor. When swaps are routed through `MetricOmmSimpleRouter`, `msg.sender` at the pool is the router contract address. A pool admin who allowlists the router to enable router-based swaps for permitted users inadvertently opens the pool to every user, because all router-mediated swaps appear as the router address to the extension.

## Finding Description
`MetricOmmPool.swap` passes `msg.sender` as the `sender` argument to `_beforeSwap`:

```solidity
// MetricOmmPool.sol L230-240
_beforeSwap(
  msg.sender,   // immediate caller of pool.swap()
  recipient,
  ...
);
```

`SwapAllowlistExtension.beforeSwap` checks this `sender` against the per-pool allowlist (keyed by `msg.sender` = the pool):

```solidity
// SwapAllowlistExtension.sol L37
if (!allowAllSwappers[msg.sender] && !allowedSwapper[msg.sender][sender]) {
  revert IMetricOmmPoolActions.NotAllowedToSwap();
}
```

`MetricOmmSimpleRouter.exactInputSingle` calls `pool.swap()` directly, making the router the `msg.sender` at the pool:

```solidity
// MetricOmmSimpleRouter.sol L71-80
_setNextCallbackContext(params.pool, CALLBACK_MODE_JUST_PAY, msg.sender, params.tokenIn);
(int128 amount0Delta, int128 amount1Delta) = IMetricOmmPoolActions(params.pool)
  .swap(params.recipient, params.zeroForOne, ...);
```

The end user's identity is stored only in the router's transient callback context (`_getPayer()`) and in `params.recipient`, neither of which is checked by the extension. The extension therefore evaluates `allowedSwapper[pool][router]`, not `allowedSwapper[pool][end_user]`. The same mismatch applies to `exactInput`, `exactOutputSingle`, and `exactOutput`. No existing guard in the extension or pool corrects for this indirection.

## Impact Explanation
A pool admin who configures `SwapAllowlistExtension` intends to restrict swaps to a curated set of addresses. To allow those permitted users to also swap via the standard periphery router, the admin must allowlist the router address. Once the router is allowlisted, every user — including those explicitly excluded — can bypass the gate by routing through `MetricOmmSimpleRouter`. This constitutes a direct policy bypass: unauthorized users can execute swaps against LP funds on a pool designed to be restricted, causing unauthorized price impact on LP positions and unauthorized extraction of LP value. This meets the "admin-boundary break bypassed by an unprivileged path" and "broken core pool functionality causing loss of funds" impact criteria.

## Likelihood Explanation
The trigger requires no special permissions. Any user can call `MetricOmmSimpleRouter.exactInputSingle` or `exactInput`. The only precondition — that the pool admin has allowlisted the router — is a natural and expected operational step for any restricted pool that wants to support the standard periphery. The scenario is therefore highly likely in any real deployment of a curated pool.

## Recommendation
The extension must gate the economic actor, not the immediate caller. The preferred fix is for the router to encode `msg.sender` (the end user) into `extensionData`, and for the extension to decode and verify it. Alternatively, the extension can expose a separate "trusted operator" allowlist for addresses like the router, requiring the real user identity to be passed and verified via `extensionData`. Checking `recipient` instead of `sender` is insufficient because `recipient` can be set to any address by the caller.

## Proof of Concept
1. Deploy a pool with `SwapAllowlistExtension` as a `beforeSwap` hook.
2. Pool admin calls `setAllowedToSwap(pool, alice, true)` — only alice is permitted.
3. Pool admin calls `setAllowedToSwap(pool, router, true)` — to let alice use the router.
4. Bob (not allowlisted) calls `MetricOmmSimpleRouter.exactInputSingle({pool: pool, recipient: bob, ...})`.
5. The router calls `pool.swap(bob, ...)` with `msg.sender = router`.
6. `_beforeSwap(router, bob, ...)` is called; the extension checks `allowedSwapper[pool][router]` → `true` → swap proceeds.
7. Bob successfully swaps on a pool he was explicitly excluded from, with no special permissions required.
