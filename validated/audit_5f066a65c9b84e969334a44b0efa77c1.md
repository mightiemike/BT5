### Title
Unprivileged Caller Can Trigger `run_mainchain_gc` to Permanently Delete Canonical Chain State — (`contract/src/lib.rs`)

### Summary
`run_mainchain_gc` is a `&mut self` (state-mutating) public function that permanently removes block headers from `headers_pool` and `mainchain_height_to_header`, and advances `mainchain_initial_blockhash`. It carries only a `#[pause]` decorator, which gates access only when the contract is paused. When the contract is running normally, any unprivileged NEAR account can call it with an arbitrary `batch_size`, triggering aggressive pruning of the canonical chain state. The analogous state-write restriction (`#[trusted_relayer]`) present on `submit_blocks` is entirely absent here.

### Finding Description
In `contract/src/lib.rs`, `submit_blocks` (line 169) is protected by both `#[pause]` and `#[trusted_relayer]`, meaning only registered relayers may mutate chain state through that path. [1](#0-0) 

`run_mainchain_gc` (line 377) carries only `#[pause(except(roles(Role::UnrestrictedRunGC)))]`. The `#[pause]` attribute from `near_plugins` exclusively controls whether the function is callable while the contract is in a paused state — it imposes no restriction on the identity of the caller when the contract is live. [2](#0-1) 

Inside the function, three irreversible state mutations occur:
- `self.remove_block_header(blockhash)` — deletes the entry from `headers_pool` and `mainchain_header_to_height`
- `self.mainchain_height_to_header.remove(&height)` — removes the height-to-hash mapping
- `self.mainchain_initial_blockhash = ...` — advances the oldest tracked block pointer [3](#0-2) 

The attacker-controlled entry path is direct: any NEAR account calls `run_mainchain_gc` with `batch_size = u64::MAX`. The function caps removal at `total_amount_to_remove = amount_of_headers_we_store - self.gc_threshold`, so it removes every block currently eligible for pruning in a single transaction, rather than the small incremental batches the relayer would normally issue via `submit_blocks`. [4](#0-3) 

### Impact Explanation
Once a block header is removed from `headers_pool` and `mainchain_height_to_header`, `verify_transaction_inclusion` and `verify_transaction_inclusion_v2` will permanently panic for any SPV proof referencing that block ("block does not belong to the current main chain"). [5](#0-4) 

Additionally, any chain reorg that must walk back past the new `mainchain_initial_blockhash` will panic with `PrevBlockNotFound`, as documented in `contract/CLAUDE.md`: "If mainchain blocks near the fork point have been garbage collected, reorg will fail." [6](#0-5) 

The corrupted state variables are `mainchain_initial_blockhash`, `mainchain_height_to_header`, and `headers_pool` — the exact canonical-chain mappings that all SPV proof consumers depend on. Deletion is irreversible; there is no re-insertion path.

### Likelihood Explanation
Likelihood is high. The attacker requires no privileged role, no staked deposit, no special key, and no prior interaction with the contract. A single NEAR transaction suffices. The function is callable at any time the contract is not paused, and the contract is expected to be unpaused during normal operation.

### Recommendation
Apply the same `#[trusted_relayer]` guard used on `submit_blocks` to `run_mainchain_gc`, or introduce an explicit role check (e.g., `Role::DAO` or a dedicated `GCManager` role) that must be held by the caller. The `#[pause]` decorator alone is not a caller-identity control.

### Proof of Concept
1. Deploy the contract with `gc_threshold = 52704` and sync enough blocks so that `amount_of_headers_we_store > gc_threshold`.
2. From any fresh NEAR account (no roles granted), call:
   ```
   near call <contract_id> run_mainchain_gc '{"batch_size": 18446744073709551615}' --accountId attacker.near
   ```
3. The call succeeds. All blocks from `mainchain_initial_blockhash` up to `tip - gc_threshold` are deleted from `headers_pool` and `mainchain_height_to_header`, and `mainchain_initial_blockhash` is advanced.
4. Any subsequent `verify_transaction_inclusion` call for a pruned block panics: "block does not belong to the current main chain."
5. Any reorg touching the pruned range panics: "PrevBlockNotFound." [7](#0-6)

```rust
    #[pause(except(roles(Role::UnrestrictedRunGC)))]
    pub fn run_mainchain_gc(&mut self, batch_size: u64) {
        let initial_blockheader = self
            .headers_pool
            .get(&self.mainchain_initial_blockhash)
            .unwrap_or_else(|| env::panic_str(ERR_KEY_NOT_EXIST));

        let tip_blockheader = self
            .headers_pool
            .get(&self.mainchain_tip_blockhash)
            .unwrap_or_else(|| env::panic_str(ERR_KEY_NOT_EXIST));

        let amount_of_headers_we_store =
            tip_blockheader.block_height - initial_blockheader.block_height + 1;

        if amount_of_headers_we_store > self.gc_threshold {
            let total_amount_to_remove = amount_of_headers_we_store - self.gc_threshold;
            let selected_amount_to_remove = std::cmp::min(total_amount_to_remove, batch_size);

            let start_removal_height = initial_blockheader.block_height;
            let end_removal_height = initial_blockheader.block_height + selected_amount_to_remove;
            env::log_str(&format!(
                "Num of blocks to remove {selected_amount_to_remove}"
            ));

            for height in start_removal_height..end_removal_height {
                let blockhash = &self
                    .mainchain_height_to_header
                    .get(&height)
                    .unwrap_or_else(|| env::panic_str(ERR_KEY_NOT_EXIST));

                self.remove_block_header(blockhash);
                self.mainchain_height_to_header.remove(&height);
            }

            self.mainchain_initial_blockhash = self
                .mainchain_height_to_header
                .get(&end_removal_height)
                .unwrap_or_else(|| env::panic_str(ERR_KEY_NOT_EXIST));
        }
    }
```
