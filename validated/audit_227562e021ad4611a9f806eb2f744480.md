### Title
Wrong-Channel Send Causes IBC Voucher Escrow Instead of Burn, Minting Unbacked NFT Supply — (`x/nft-transfer/keeper/packet.go`)

### Summary

`IsAwayFromOrigin` is a pure string-prefix check. When a sink-chain holder sends an IBC voucher back via a **different** channel than the one it arrived on, the prefix check returns `true` (away from origin) instead of `false`, causing the voucher to be **escrowed** rather than burned. The receiving chain simultaneously **mints** a new NFT, creating unbacked supply.

---

### Finding Description

`IsAwayFromOrigin` in `x/nft-transfer/types/trace.go` is:

```go
func IsAwayFromOrigin(sourcePort, sourceChannel, fullClassPath string) bool {
    prefixClassID := GetClassPrefix(sourcePort, sourceChannel)
    return !strings.HasPrefix(fullClassPath, prefixClassID)
}
``` [1](#0-0) 

It returns `false` (sink chain, should burn) **only** when `fullClassPath` starts with `sourcePort/sourceChannel/`. If the attacker uses any other channel, the prefix never matches and the function returns `true`.

In `createOutgoingPacket`, when `isAwayFromOrigin = true`, the code escrows instead of burning:

```go
if isAwayFromOrigin {
    escrowAddress := types.GetEscrowAddress(sourcePort, sourceChannel)
    if err := k.nftKeeper.TransferOwner(ctx, classID, tokenID, sender, escrowAddress); err != nil { ...
} else {
    if err := k.nftKeeper.BurnNFTUnverified(ctx, classID, tokenID, sender); err != nil { ...
}
``` [2](#0-1) 

`ValidateBasic` in `MsgTransfer` performs no check that the chosen `sourceChannel` matches the NFT's trace prefix: [3](#0-2) 

`SendTransfer` only verifies the channel exists, not that it is consistent with the class trace: [4](#0-3) 

---

### Impact Explanation

**Concrete state delta:**

| Chain | Before exploit | After exploit |
|---|---|---|
| Chain A (origin) | NFT locked in channel-0 escrow | NFT still locked in channel-0 escrow **+ new NFT minted** with class `ibc/{hash(transfer/channel-1/transfer/channel-0/nativeClass)}` |
| Chain B (sink) | Attacker holds IBC voucher `ibc/{hash(transfer/channel-0/nativeClass)}` | Voucher locked in channel-1 escrow (not burned) |

The new NFT on Chain A is **unbacked**: no corresponding locked NFT exists for it. The ICS-721 supply invariant is broken. The attacker receives a new NFT on Chain A without surrendering the original value (the original NFT on Chain A remains locked, and the voucher on Chain B is merely escrowed, not destroyed).

---

### Likelihood Explanation

Any user who holds an IBC NFT voucher and has access to two channels between the same chain pair can trigger this. No special privileges are required — only a standard `MsgTransfer` signed by the voucher owner. The attack is fully reachable through the production `MsgServer.Transfer` → `SendTransfer` → `createOutgoingPacket` path. [5](#0-4) 

---

### Recommendation

Before calling `IsAwayFromOrigin`, validate that if `classID` is an IBC-prefixed voucher (`ibc/...`), the resolved `fullClassPath` must start with `sourcePort/sourceChannel/`. If it does not, reject the transaction with an error — the voucher cannot be legitimately sent back via a channel that does not match its outermost trace hop. Concretely, in `createOutgoingPacket`, after resolving `fullClassPath`:

```go
if strings.HasPrefix(classID, nfttypes.IBCPrefix) {
    expectedPrefix := types.GetClassPrefix(sourcePort, sourceChannel)
    if !strings.HasPrefix(fullClassPath, expectedPrefix) {
        return channeltypes.Packet{}, sdkerrors.Wrapf(
            types.ErrInvalidClassID,
            "IBC voucher must be returned via its original channel; expected prefix %s", expectedPrefix)
    }
}
``` [6](#0-5) 

---

### Proof of Concept

1. Chain A mints `nativeClass/token-1` and sends it to Chain B via `(transfer, channel-0)`.
2. Chain B receives and stores class trace `transfer/channel-0/nativeClass` → `ibc/HASH0`. Attacker on Chain B holds `ibc/HASH0/token-1`.
3. Attacker submits `MsgTransfer{SourceChannel: "channel-1", ClassId: "ibc/HASH0", TokenIds: ["token-1"], Receiver: attackerOnA}` on Chain B.
4. `ClassPathFromHash` resolves `ibc/HASH0` → `transfer/channel-0/nativeClass`.
5. `IsAwayFromOrigin("transfer", "channel-1", "transfer/channel-0/nativeClass")` → `true` (wrong).
6. `TransferOwner` moves `ibc/HASH0/token-1` to channel-1's escrow on Chain B (not burned).
7. Packet arrives at Chain A; `processReceivedPacket` also computes `isAwayFromOrigin = true` and calls `MintNFT` for class `ibc/{hash(transfer/channel-1/transfer/channel-0/nativeClass)}`, minting `token-1` to the attacker.
8. Assert: Chain A now has two NFTs for `token-1` (one locked in channel-0 escrow, one owned by attacker under a new class). Unbacked supply confirmed. [7](#0-6) [8](#0-7)

func (msg MsgTransfer) ValidateBasic() error {
	if err := host.PortIdentifierValidator(msg.SourcePort); err != nil {
		return newsdkerrors.Wrap(err, "invalid source port ID")
	}
	if msg.SourcePort != PortID {
		return newsdkerrors.Wrapf(ErrInvalidSourcePort, "source port must be %q", PortID)
	}
	if err := host.ChannelIdentifierValidator(msg.SourceChannel); err != nil {
		return newsdkerrors.Wrap(err, "invalid source channel ID")
	}

	if strings.TrimSpace(msg.ClassId) == "" {
		return newsdkerrors.Wrap(ErrInvalidClassID, "classId cannot be blank")
	}

	if len(msg.TokenIds) == 0 {
		return newsdkerrors.Wrap(ErrInvalidTokenID, "tokenId cannot be blank")
	}

	for _, tokenID := range msg.TokenIds {
		if strings.TrimSpace(tokenID) == "" {
			return newsdkerrors.Wrap(ErrInvalidTokenID, "tokenId cannot be blank")
		}
	}

	// NOTE: sender format must be validated as it is required by the GetSigners function.
	_, err := sdk.AccAddressFromBech32(msg.Sender)
	if err != nil {
		return newsdkerrors.Wrapf(sdkerrors.ErrInvalidAddress, "string could not be parsed as address: %v", err)
	}
	if strings.TrimSpace(msg.Receiver) == "" {
		return newsdkerrors.Wrap(sdkerrors.ErrInvalidAddress, "missing recipient address")
	}
	return nil
}
