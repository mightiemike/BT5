### Title
Unpermissioned `run_mainchain_gc` Allows Any Caller to Prune Mainchain Block Headers — (File: `contract/src/lib.rs`)

### Summary
`run_mainchain_gc` is a public, state-mutating function with no caller access control. Any unprivileged NEAR account can invoke it with an arbitrarily large `batch_size`, permanently deleting mainchain block headers from contract storage. This breaks SPV proof verification for pruned blocks and can prevent legitimate chain reorganizations.

### Finding Description
The function `run_mainchain_gc` is declared `pub` and carries only a `#[pause(except(roles(Role::UnrestrictedRunGC)))]` attribute: [1](#0-0) 

The `#[pause(...)]` macro controls behavior only when the contract is paused — it does not restrict which accounts may call the function when the contract is running normally. There is no `#[trusted_relayer]`, no `#[private]`, and no `require!` guard on `env::predecessor_account_id()`.

When called, the function permanently removes entries from three critical data structures: [2](#0-1) 

Specifically, it calls `remove_block_header` (which removes from both `mainchain_header_to_height` and `headers_pool`) and removes from `mainchain_height_to_header`, then advances `mainchain_initial_blockhash` forward: [3](#0-2) 

Compare this to `submit_blocks`, which is correctly gated behind `#[trusted_relayer]`: [4](#0-3) 

And `init` / `migrate`, which are gated behind `#[private]`: [5](#0-4) 

### Impact Explanation
An attacker calling `run_mainchain_gc(u64::MAX)` will prune all mainchain blocks down to `gc_threshold`. The consequences are:

1. **SPV proof forgery/denial**: `verify_transaction_inclusion` and `verify_transaction_inclusion_v2` look up the target block in `mainchain_header_to_height`. If the block has been GC'd, the call panics with `"block does not belong to the current main chain"`. Any downstream NEAR contract relying on SPV proofs for pruned blocks will permanently fail. [6](#0-5) 

2. **Chain reorganization prevention**: `reorg_chain` walks the fork chain back to the common ancestor by checking `mainchain_header_to_height`. If the common ancestor has been GC'd, the walk panics with `PrevBlockNotFound`. This means a legitimate deep reorg submitted by the relayer will be permanently blocked. [7](#0-6) 

3. **Irreversible state corruption**: Deleted entries in `LookupMap` cannot be restored without a migration. The attacker's action is permanent.

### Likelihood Explanation
The entry path requires only a standard NEAR account and a single transaction call. No staking, no role, no deposit beyond gas is needed. The function is publicly callable on a live contract. Any motivated attacker (e.g., one who wants to invalidate a specific Bitcoin transaction SPV proof used by a DeFi bridge) can execute this at negligible cost.

### Recommendation
Add an access-control guard to `run_mainchain_gc` consistent with the pattern used for `submit_blocks`. The simplest fix is to require the caller to hold the `DAO` or a new dedicated `GCManager` role:

```rust
#[pause(except(roles(Role::UnrestrictedRunGC)))]
pub fn run_mainchain_gc(&mut self, batch_size: u64) {
    require!(
        self.acl_has_role(Role::DAO, &env::predecessor_account_id())
            || self.acl_has_role(Role::RelayerManager, &env::predecessor_account_id()),
        "Unauthorized: caller lacks GC permission"
    );
    // ... existing body
}
```

Alternatively, apply `#[trusted_relayer]` or a dedicated role attribute matching the project's existing RBAC pattern.

    #[pause(except(roles(Role::UnrestrictedRunGC)))]
    pub fn run_mainchain_gc(&mut self, batch_size: u64) {
        let initial_blockheader = self
            .headers_pool
            .get(&self.mainchain_initial_blockhash)
            .unwrap_or_else(|| env::panic_str(ERR_KEY_NOT_EXIST));

        let tip_blockheader = self
            .headers_pool
            .get(&self.mainchain_tip_blockhash)
            .unwrap_or_else(|| env::panic_str(ERR_KEY_NOT_EXIST));

        let amount_of_headers_we_store =
            tip_blockheader.block_height - initial_blockheader.block_height + 1;

        if amount_of_headers_we_store > self.gc_threshold {
            let total_amount_to_remove = amount_of_headers_we_store - self.gc_threshold;
            let selected_amount_to_remove = std::cmp::min(total_amount_to_remove, batch_size);

            let start_removal_height = initial_blockheader.block_height;
            let end_removal_height = initial_blockheader.block_height + selected_amount_to_remove;
            env::log_str(&format!(
                "Num of blocks to remove {selected_amount_to_remove}"
            ));

            for height in start_removal_height..end_removal_height {
                let blockhash = &self
                    .mainchain_height_to_header
                    .get(&height)
                    .unwrap_or_else(|| env::panic_str(ERR_KEY_NOT_EXIST));

                self.remove_block_header(blockhash);
                self.mainchain_height_to_header.remove(&height);
            }

            self.mainchain_initial_blockhash = self
                .mainchain_height_to_header
                .get(&end_removal_height)
                .unwrap_or_else(|| env::panic_str(ERR_KEY_NOT_EXIST));
        }
    }
