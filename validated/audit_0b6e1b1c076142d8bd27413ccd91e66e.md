Audit Report

## Title
`SwapAllowlistExtension.beforeSwap` Checks Router Address Instead of End-User, Allowing Any User to Bypass the Swap Allowlist — (`metric-periphery/contracts/extensions/SwapAllowlistExtension.sol`)

## Summary

`SwapAllowlistExtension.beforeSwap` gates swaps by checking `allowedSwapper[msg.sender][sender]`, where `sender` is whoever called `pool.swap()`. When `MetricOmmSimpleRouter` intermediates a swap, it calls `pool.swap()` directly, making the router the `sender` seen by the extension. Any pool admin who allowlists the router to enable normal UX inadvertently grants every user on the internet the ability to bypass the swap allowlist by routing through the router.

## Finding Description

`SwapAllowlistExtension.beforeSwap` performs:

```solidity
// SwapAllowlistExtension.sol line 37
if (!allowAllSwappers[msg.sender] && !allowedSwapper[msg.sender][sender]) {
    revert IMetricOmmPoolActions.NotAllowedToSwap();
}
```

`msg.sender` here is the pool (the extension is called by the pool via `_callExtensionsInOrder`), and `sender` is the address passed through `_beforeSwap` — which originates as `msg.sender` at the pool level, i.e., whoever called `pool.swap(...)`. [1](#0-0) 

In `MetricOmmSimpleRouter.exactInputSingle`, the router calls:

```solidity
IMetricOmmPoolActions(params.pool).swap(params.recipient, ...)
``` [2](#0-1) 

At this call site, `msg.sender` inside the pool is the **router's address**. The pool then passes `sender = router_address` into `_beforeSwap`: [3](#0-2) 

The extension therefore checks `allowedSwapper[pool][router_address]`. If the router is allowlisted (required for any router-mediated swap to work), this check passes for **every caller** regardless of who the actual end-user is. The same flaw applies to `exactInput`, `exactOutputSingle`, and `exactOutput`. [4](#0-3) 

`DepositAllowlistExtension` does not share this flaw because it checks the explicit `owner` parameter, not `sender`: [5](#0-4) 

The swap path has no equivalent explicit "real swapper" parameter — the router passes no end-user identity to the extension.

## Impact Explanation

Direct policy bypass on curated pools. The swap allowlist exists to restrict which addresses may trade (KYC-gated, institutional-only, whitelist-only pools). Once the router is allowlisted — required for normal UX — any unprivileged user can trade on the pool by routing through `MetricOmmSimpleRouter`. The pool admin's curation is completely nullified. LPs on curated pools face unauthorized counterparties, potential adversarial swaps, and regulatory/compliance failure if the pool is gated for legal reasons. Severity: **High** — direct loss of access-control invariant with fund-impacting consequences for LPs. [6](#0-5) 

## Likelihood Explanation

High. The attack requires no special privilege. `MetricOmmSimpleRouter` is a standard public periphery contract. Any pool admin who wants users to swap via the router must allowlist the router address — this is the expected operational pattern. Once the router is allowlisted, any user calls `router.exactInput(...)` targeting the curated pool. No admin action, no special token, no flash loan needed. The only non-vulnerable scenario is if the pool admin intentionally blocks the router and forces all swappers to call the pool directly, which breaks normal UX. [7](#0-6) 

## Recommendation

The extension must gate the **end-user**, not the intermediary. Two sound approaches:

1. **Trusted forwarder pattern:** Allowlist the router as a trusted forwarder. When `sender` is a known trusted forwarder, read the real user from `extensionData` and check that address instead. Requires the router to cooperate by encoding `msg.sender` into `extensionData`.
2. **Explicit swapper parameter at pool level:** Add an explicit `swapper` parameter to `pool.swap()` that the router populates with its `msg.sender`. The extension checks that address. Requires a pool-level interface change.
3. **Short-term mitigation:** Do not allowlist the router; require direct pool calls for allowlisted pools. Document this constraint clearly. Breaks composability but is safe. [8](#0-7) 

## Proof of Concept

```
Setup:
  - Deploy pool with SwapAllowlistExtension as beforeSwap hook.
  - Pool admin calls setAllowedToSwap(pool, router, true)   // allowlist the router
  - Pool admin does NOT call setAllowedToSwap(pool, alice, true)  // alice is NOT allowlisted

Attack:
  - alice calls MetricOmmSimpleRouter.exactInputSingle({pool: pool, ...})
  - Router calls IMetricOmmPoolActions(pool).swap(recipient=alice, ...)
  - Pool passes sender=router_address to _beforeSwap → extension
  - Extension checks: allowedSwapper[pool][router_address] == true → PASSES
  - Alice's swap executes on the curated pool despite not being allowlisted

Expected:
  - Extension should check allowedSwapper[pool][alice] == false → REVERT NotAllowedToSwap

Contrast (direct call):
  - alice calls pool.swap(...) directly
  - Pool passes sender=alice to _beforeSwap
  - Extension checks: allowedSwapper[pool][alice] == false → REVERTS correctly
``` [9](#0-8) [8](#0-7)
