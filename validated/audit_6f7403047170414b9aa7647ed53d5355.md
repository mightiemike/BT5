### Title
Gas-key host-function exec-fee uses wire-format key length instead of trie-id length for ML-DSA-65 keys, causing ~59× fee overcharge - (File: `runtime/near-vm-runner/src/logic/logic.rs`)

### Summary

The `promise_batch_action_transfer_to_gas_key`, `promise_batch_action_add_gas_key_with_full_access`, and `promise_batch_action_add_gas_key_with_function_call` host functions compute the exec-side gas fee using the raw borsh wire length of the public key (`public_key_len as usize`). For ML-DSA-65 keys this is 1953 bytes. The correct value — used by the transaction path — is `trie_id_len()` = 33 bytes. The two diverge by a factor of ~59× for every ML-DSA-65 gas-key host-function call.

### Finding Description

**Transaction path** (`runtime/runtime/src/config.rs`, `total_send_fees`):

```rust
TransferToGasKey(action) => {
    gas_key_transfer_send_fee(fees, sender_is_receiver, action.public_key.trie_id_len())
        .total()
}
```

`action.public_key.trie_id_len()` returns 33 for `PublicKey::MLDSA65` (1 tag byte + 32-byte SHA3-256 hash), matching the actual on-trie key size. [1](#0-0) 

**Host-function path** (`runtime/near-vm-runner/src/logic/logic.rs`, `promise_batch_action_transfer_to_gas_key`):

```rust
let public_key = self.get_public_key(public_key_ptr, public_key_len)?;
...
let send = gas_key_transfer_send_fee(&self.fees_config, sir, public_key_len as usize);
let exec = gas_key_transfer_exec_fee(
    &self.fees_config,
    receiver_id.len(),
    public_key_len as usize,   // ← wire size, not trie-id size
);
```

`public_key_len` is the raw byte count of the borsh-encoded key as supplied by the contract. For `PublicKey::MLDSA65` this is 1953 bytes (1 tag + 1952 raw pubkey bytes). [2](#0-1) 

`gas_key_transfer_exec_fee` uses its `public_key_len` argument to compute the trie key length:

```rust
pub fn gas_key_transfer_exec_fee(
    cfg: &RuntimeFeesConfig,
    account_id_len: usize,
    public_key_len: usize,          // supposed to be trie-id length
) -> GasKeyTransferFee {
    let trie_key_len = access_key_key_len(account_id_len, public_key_len);
    ...
}
``` [3](#0-2) 

The same pattern appears in `promise_batch_action_add_gas_key_with_full_access` and `promise_batch_action_add_gas_key_with_function_call`, which pass `public_key_len as usize` to `gas_key_add_key_exec_fee`: [4](#0-3) 

The divergence is rooted in the ML-DSA-65 key-length split documented in `PublicKey`:

```rust
pub fn len(&self) -> usize {
    Self::MLDSA65(_) => 1 + ML_DSA_65_PUBLIC_KEY_LENGTH,  // 1953
}
pub fn trie_id_len(&self) -> usize {
    Self::MLDSA65(_) => 1 + ML_DSA_65_HASH_LENGTH,        // 33
}
``` [5](#0-4) 

The design document explicitly states: "Callers that still use `len()` for trie-storage costing will misprice ML-DSA-65 keys by ~1900 bytes." The host-function fee path is exactly such a caller. [6](#0-5) 

The existing fee-parity test (`test_gas_key_fee_parity`) only exercises ED25519 keys, so the divergence is not caught: [7](#0-6) 

### Impact Explanation

For every call to `promise_batch_action_transfer_to_gas_key` (or the add/withdraw variants) with an ML-DSA-65 key, the exec-fee per-byte component is computed against 1953 bytes instead of 33 bytes — a ~59× overcharge on that component. Concretely:

- `trie_key_len` inside `gas_key_transfer_exec_fee` becomes `access_key_key_len(account_id_len, 1953)` instead of `access_key_key_len(account_id_len, 33)`, inflating the per-byte exec gas by 1920 extra bytes × `gas_key_byte` exec rate per call.
- The inflated `used_gas` is prepaid from the calling contract's gas budget. If the contract does not attach enough gas, the host-function call fails with gas exhaustion.
- The fee is deterministic and consistent across all nodes (no consensus break), but it creates a permanent, unprivileged-user-observable fee asymmetry: the same logical operation costs ~59× more gas when invoked via host function than via a signed transaction.

### Likelihood Explanation

Both `GasKeys` and `PostQuantumSignatures` stabilized at protocol version 85. Any contract running on a post-v85 network can call `promise_batch_action_transfer_to_gas_key` with a borsh-encoded ML-DSA-65 public key (1953 bytes). No privileged role is required. The trigger is a normal cross-contract call pattern.

### Recommendation

After decoding the public key, use `decoded_key.trie_id_len()` for the exec-fee argument instead of the raw `public_key_len`:

```rust
let public_key = self.get_public_key(public_key_ptr, public_key_len)?;
let decoded = public_key.decode()?;
let send = gas_key_transfer_send_fee(&self.fees_config, sir, public_key_len as usize);
let exec = gas_key_transfer_exec_fee(
    &self.fees_config,
    receiver_id.len(),
    decoded.trie_id_len(),   // ← trie-id length, not wire length
);
```

Apply the same fix to `gas_key_add_key_exec_fee` calls in `promise_batch_action_add_gas_key_with_full_access` and `promise_batch_action_add_gas_key_with_function_call`. Add a fee-parity test that exercises ML-DSA-65 keys alongside ED25519 keys.

### Proof of Concept

1. Deploy a contract that calls `promise_batch_action_transfer_to_gas_key` with a borsh-encoded `PublicKey::MLDSA65` (1953 bytes).
2. Observe the `gas_burnt` on the resulting receipt.
3. Compare with the `gas_burnt` of a signed `TransferToGasKey` transaction carrying the same ML-DSA-65 key.
4. The host-function receipt burns ~59× more gas on the per-byte exec component than the transaction receipt, confirming the domain mismatch.

The exact byte counts are fixed by the protocol schema: `ML_DSA_65_PUBLIC_KEY_LENGTH = 1952`, `ML_DSA_65_HASH_LENGTH = 32`, so wire length = 1953, trie-id length = 33, ratio = 59.18×. [8](#0-7)

pub fn gas_key_transfer_exec_fee(
    cfg: &RuntimeFeesConfig,
    account_id_len: usize,
    public_key_len: usize,
) -> GasKeyTransferFee {
    let base = cfg.fee(ActionCosts::gas_key_transfer_base).exec_fee();
    let trie_key_len = access_key_key_len(account_id_len, public_key_len);
    let estimated_value_len = AccessKey::min_gas_key_borsh_len();
    let per_byte = cfg
        .fee(ActionCosts::gas_key_byte)
        .exec_fee()
        .checked_mul((trie_key_len + estimated_value_len) as u64)
        .unwrap();
    GasKeyTransferFee { base, per_byte }
}
