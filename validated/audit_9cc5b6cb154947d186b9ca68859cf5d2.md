### Title
`ValidResourceBounds` Silently Mutates from `AllResources` to `L1Gas` Across Protobuf Round-Trip, Producing a Divergent Transaction Hash — (`crates/apollo_protobuf/src/converters/transaction.rs`)

---

### Summary

The protobuf deserializer for `ValidResourceBounds` silently collapses `AllResources{l2_gas=0, l1_data_gas=0}` into the `L1Gas` variant. Because `get_tip_resource_bounds_hash` hashes a different number of resource elements for each variant (2 for `L1Gas`, 3 for `AllResources`), any transaction submitted with `AllResourceBounds` where both `l2_gas` and `l1_data_gas` are zero will carry a gateway-computed hash that cannot be reproduced after a protobuf round-trip through the P2P block-sync layer. The gateway accepts such transactions today.

---

### Finding Description

**Step 1 — Gateway always hashes with `AllResources`.**

`InternalRpcInvokeTransactionV3::resource_bounds()` unconditionally wraps its stored `AllResourceBounds` in `ValidResourceBounds::AllResources`:

```rust
// crates/starknet_api/src/rpc_transaction.rs  line 637-639
fn resource_bounds(&self) -> ValidResourceBounds {
    ValidResourceBounds::AllResources(self.resource_bounds)
}
```

`get_tip_resource_bounds_hash` then hashes **three** resource felts for `AllResources` (L1\_GAS, L2\_GAS, L1\_DATA\_GAS) and only **two** for `L1Gas` (L1\_GAS, L2\_GAS):

```rust
// crates/starknet_api/src/transaction_hash.rs  lines 197-208
let mut resource_felts = vec![
    get_concat_resource(&l1_resource_bounds, L1_GAS)?,
    get_concat_resource(&l2_resource_bounds, L2_GAS)?,
];
resource_felts.extend(match resource_bounds {
    ValidResourceBounds::L1Gas(_) => vec![],
    ValidResourceBounds::AllResources(all_resources) => {
        vec![get_concat_resource(&all_resources.l1_data_gas, L1_DATA_GAS)?]
    }
});
```

So for `AllResources{l1_gas: X, l2_gas: 0, l1_data_gas: 0}` the hash preimage is `poseidon(tip, L1_GAS_X, L2_GAS_0, L1_DATA_GAS_0)` — three elements.

**Step 2 — Protobuf serialization preserves all three fields.**

`From<ValidResourceBounds> for protobuf::ResourceBounds` always emits all three fields, even for `L1Gas`:

```rust
// crates/apollo_protobuf/src/converters/transaction.rs  lines 471-489
ValidResourceBounds::L1Gas(l1_gas) => protobuf::ResourceBounds {
    l1_gas: Some(l1_gas.into()),
    l2_gas: Some(value.get_l2_bounds().into()),
    l1_data_gas: Some(ResourceBounds::default().into()),  // zero, but present
},
```

For `AllResources{l2_gas=0, l1_data_gas=0}` the wire bytes are identical to the `L1Gas` case: all three fields present, two of them zero.

**Step 3 — Protobuf deserialization silently collapses the variant.**

`TryFrom<protobuf::ResourceBounds> for ValidResourceBounds` applies a zero-check and returns `L1Gas` whenever both `l2_gas` and `l1_data_gas` are zero:

```rust
// crates/apollo_protobuf/src/converters/transaction.rs  lines 417-436
let l1_data_gas = value.l1_data_gas.unwrap_or_default();
// ...
Ok(if l1_data_gas.is_zero() && l2_gas.is_zero() {
    ValidResourceBounds::L1Gas(l1_gas)   // ← variant changed
} else {
    ValidResourceBounds::AllResources(AllResourceBounds { l1_gas, l2_gas, l1_data_gas })
})
```

After this deserialization the hash preimage becomes `poseidon(tip, L1_GAS_X, L2_GAS_0)` — only two elements. The Poseidon hash of two elements differs from the Poseidon hash of three elements even when the third element is zero, so **H\_gateway ≠ H\_sync**.

**Step 4 — The gateway accepts the triggering transaction.**

The stateless validator explicitly allows `AllResourceBounds` with `l2_gas=0` and `l1_data_gas=0` (test case `valid_l1_gas` in `apollo_gateway/src/stateless_transaction_validator_test.rs` lines 70-82). No privileged access is required.

---

### Impact Explanation

A syncing node that deserializes a block containing such a transaction will recompute a transaction hash that differs from the one stored in the block header and in the mempool. Depending on which hash is used for state commitment, receipt generation, or RPC responses, this produces one of:

- **Wrong receipt / wrong transaction hash returned by RPC** — the syncing node indexes the transaction under H\_sync while the sequencer committed H\_gateway.
- **Block validation failure** — if the syncing node verifies that the transaction hash in the body matches the one in the header, it will reject a valid block.
- **Wrong execution context** — `ValidResourceBounds::L1Gas` selects `GasVectorComputationMode::NoL2Gas`, while `AllResources` selects `GasVectorComputationMode::All`, changing fee-checking and gas-accounting behavior for the same transaction.

This matches the **Critical** impact class: *Wrong state, receipt, event, or revert result from blockifier/syscall/execution logic for accepted input*, and the **High** class: *RPC execution, fee estimation, tracing, simulation, or pending view returns an authoritative-looking wrong value*.

---

### Likelihood Explanation

The trigger is any V3 Invoke or Declare transaction submitted with `AllResourceBounds` where `l2_gas = 0` and `l1_data_gas = 0`. This is the natural form for a user who only cares about L1 gas (e.g., a simple token transfer). The gateway accepts it without restriction. No special privileges, malformed bytes, or adversarial peer are required — a normal user transaction is sufficient.

---

### Recommendation

1. **Remove the zero-collapse heuristic** from `TryFrom<protobuf::ResourceBounds> for ValidResourceBounds`. The variant should be determined by the presence or absence of the `l1_data_gas` field in the wire format, not by whether its value is zero. The proto comment already acknowledges this: `"This can be None only in transactions that don't support l2 gas."` Use `None` vs `Some` as the discriminant, not the numeric value.

2. **Add a round-trip hash invariant test**: serialize an `AllResources{l2_gas=0, l1_data_gas=0}` transaction to protobuf, deserialize it, recompute the hash, and assert it equals the original hash.

### Proof of Concept

```rust
use starknet_api::transaction::fields::{
    AllResourceBounds, ResourceBounds, ValidResourceBounds, GasAmount, GasPrice,
};
use starknet_api::transaction_hash::get_tip_resource_bounds_hash;
use starknet_api::transaction::fields::Tip;
use apollo_protobuf::protobuf;

// Transaction as submitted to the gateway
let all_resources = ValidResourceBounds::AllResources(AllResourceBounds {
    l1_gas: ResourceBounds { max_amount: GasAmount(1000), max_price_per_unit: GasPrice(1) },
    l2_gas: ResourceBounds::default(),      // zero
    l1_data_gas: ResourceBounds::default(), // zero
});

// Hash computed by the gateway (AllResources path — 3 elements)
let h_gateway = get_tip_resource_bounds_hash(&all_resources, &Tip(0)).unwrap();

// Protobuf round-trip (P2P block sync)
let proto: protobuf::ResourceBounds = all_resources.into();
let after_roundtrip = ValidResourceBounds::try_from(proto).unwrap();

// after_roundtrip is now L1Gas, not AllResources
assert!(matches!(after_roundtrip, ValidResourceBounds::L1Gas(_)));

// Hash computed by syncing node (L1Gas path — 2 elements)
let h_sync = get_tip_resource_bounds_hash(&after_roundtrip, &Tip(0)).unwrap();

// Divergence: h_gateway != h_sync
assert_ne!(h_gateway, h_sync);
```

The divergence is rooted in `get_tip_resource_bounds_hash` at [1](#0-0)  which hashes a different number of resource felts depending on the variant, and the silent variant collapse in [2](#0-1) . The gateway always produces `AllResources` because `InternalRpcInvokeTransactionV3::resource_bounds()` hard-wraps the stored `AllResourceBounds` [3](#0-2) , while the block-sync deserializer uses the collapsing `TryFrom` path [4](#0-3) . The gateway accepts the triggering transaction class as shown by the `valid_l1_gas` test case [5](#0-4) .

        let Some(l1_gas) = value.l1_gas else {
            return Err(missing("ResourceBounds::l1_gas"));

        let Some(l2_gas) = value.l2_gas else {
            return Err(missing("ResourceBounds::l2_gas"));

            ValidResourceBounds::L1Gas(l1_gas)
        } else {
            ValidResourceBounds::AllResources(AllResourceBounds { l1_gas, l2_gas, l1_data_gas })
