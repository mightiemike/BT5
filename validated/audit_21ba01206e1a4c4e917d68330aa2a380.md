### Title
NFT Transferred Away From Denom Creator Becomes Permanently Unburnable — (`x/nft/keeper/keeper.go`)

### Summary

`BurnNFT` requires the caller to simultaneously pass **both** `IsOwner` and `IsDenomCreator`. Once an NFT is transferred from the denom creator to any other address, no party can ever satisfy both checks at the same time, making the NFT permanently unburnable.

### Finding Description

`BurnNFT` in `keeper.go` performs two sequential authorization checks: [1](#0-0) 

1. **Line 146–149** — `IsOwner`: the caller must be the current NFT owner.
2. **Line 151–154** — `IsDenomCreator`: the caller must also be the denom creator.

`IsDenomCreator` in `denom.go` compares the caller against the immutable `denom.Creator` field: [2](#0-1) 

After `TransferOwner` (called by `MsgTransferNFT`) updates `nft.Owner` to the recipient: [3](#0-2) 

the state becomes:

| Caller | `IsOwner` | `IsDenomCreator` | Result |
|---|---|---|---|
| Bob (new owner) | ✅ | ❌ | `ErrUnauthorized` |
| Alice (denom creator) | ❌ | ✅ | `ErrUnauthorized` |

Neither party can ever satisfy both guards simultaneously. The NFT is permanently unburnable.

Note that `BurnNFTUnverified` (lines 163–179) exists and intentionally skips the `IsDenomCreator` check — it is used for IBC flows. The regular `BurnNFT` path used by `MsgBurnNFT` via `msg_server.go` line 173 does not use this bypass: [4](#0-3) 

### Impact Explanation

Any NFT transferred from its denom creator to a third party becomes permanently unburnable through the standard `MsgBurnNFT` transaction path. If the NFT represents a consumable, ticket, or any asset whose destruction is economically meaningful, the owner loses that right entirely. The NFT supply for that token ID can never be reduced via the burn path.

### Likelihood Explanation

This is triggered by the completely normal and supported sequence: `MsgIssueDenom` → `MsgMintNFT` → `MsgTransferNFT` → `MsgBurnNFT`. No special privileges or unusual conditions are required. Any user who receives an NFT via transfer is affected.

### Recommendation

Remove the `IsDenomCreator` check from `BurnNFT`. Burning should require only that the caller is the current owner (`IsOwner`). The denom creator restriction is appropriate for minting and editing, but not for burning. The existing `BurnNFTUnverified` pattern already demonstrates the correct ownership-only check.

```go
// BurnNFT deletes a specified NFT
func (k Keeper) BurnNFT(ctx sdk.Context, denomID, tokenID string, owner sdk.AccAddress) error {
    if !k.HasDenomID(ctx, denomID) {
        return sdkerrors.Wrapf(types.ErrInvalidDenom, "denom ID %s not exists", denomID)
    }

    nft, err := k.IsOwner(ctx, denomID, tokenID, owner)
    if err != nil {
        return err
    }
    // Remove IsDenomCreator check — owner should be able to burn their own NFT

    k.deleteNFT(ctx, denomID, nft)
    k.deleteOwner(ctx, denomID, tokenID, owner)
    k.decreaseSupply(ctx, denomID)
    return nil
}
```

### Proof of Concept

```go
// keeper_test.go
func TestBurnNFTAfterTransfer(t *testing.T) {
    ctx, keeper := setupKeeper(t)
    alice := sdk.AccAddress([]byte("alice"))
    bob   := sdk.AccAddress([]byte("bob"))

    // Alice issues denom and mints NFT to herself
    keeper.IssueDenom(ctx, "testdenom", "Test", "", "", alice)
    keeper.MintNFT(ctx, "testdenom", "token1", "", "", "", alice, alice)

    // Alice transfers to Bob
    keeper.TransferOwner(ctx, "testdenom", "token1", alice, bob)

    // Bob (owner) tries to burn — fails: not denom creator
    err := keeper.BurnNFT(ctx, "testdenom", "token1", bob)
    require.Error(t, err) // ErrUnauthorized

    // Alice (creator) tries to burn — fails: no longer owner
    err = keeper.BurnNFT(ctx, "testdenom", "token1", alice)
    require.Error(t, err) // ErrUnauthorized

    // NFT still exists — permanently unburnable
    _, err = keeper.GetNFT(ctx, "testdenom", "token1")
    require.NoError(t, err)
}
```

func (k Keeper) IsDenomCreator(ctx sdk.Context, denomID string, address sdk.AccAddress) (types.Denom, error) {
	denom, err := k.GetDenom(ctx, denomID)
	if err != nil {
		return types.Denom{}, err
	}

	creator, err := sdk.AccAddressFromBech32(denom.Creator)
	if err != nil {
		panic(err)
	}

	if !creator.Equals(address) {
		return types.Denom{}, sdkerrors.Wrapf(types.ErrUnauthorized, "%s is not the creator of %s", address, denomID)
	}

	return denom, nil
