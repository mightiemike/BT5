Audit Report

## Title
SwapAllowlistExtension gates the router address instead of the end user, allowing any user to bypass the per-pool swap allowlist via the router - (File: metric-periphery/contracts/extensions/SwapAllowlistExtension.sol)

## Summary
`MetricOmmPool.swap()` passes `msg.sender` as the `sender` argument to `_beforeSwap`, so when `MetricOmmSimpleRouter` calls `pool.swap()`, the `sender` seen by `SwapAllowlistExtension.beforeSwap` is the router's address, not the originating user's address. A pool admin who allowlists the router address to restore router usability for their intended allowlisted users inadvertently grants swap access to every user who routes through `MetricOmmSimpleRouter`, completely defeating the per-pool allowlist.

## Finding Description

**Root cause — wrong identity propagated to the extension:**

`MetricOmmPool.swap()` calls `_beforeSwap(msg.sender, ...)`:

```solidity
// metric-core/contracts/MetricOmmPool.sol:230-240
_beforeSwap(
    msg.sender,   // <-- whoever called pool.swap()
    recipient,
    ...
);
```

`ExtensionCalling._beforeSwap` forwards that value unchanged as `sender` to the extension:

```solidity
// metric-core/contracts/ExtensionCalling.sol:160-176
_callExtensionsInOrder(
    BEFORE_SWAP_ORDER,
    abi.encodeCall(IMetricOmmExtensions.beforeSwap, (sender, ...))
);
```

`SwapAllowlistExtension.beforeSwap` then checks:

```solidity
// metric-periphery/contracts/extensions/SwapAllowlistExtension.sol:37
if (!allowAllSwappers[msg.sender] && !allowedSwapper[msg.sender][sender]) {
    revert IMetricOmmPoolActions.NotAllowedToSwap();
}
```

Here `msg.sender` is the pool (correct) and `sender` is whoever called `pool.swap()`.

**Router call path:**

`MetricOmmSimpleRouter.exactInputSingle` (and `exactInput`, `exactOutputSingle`, `exactOutput`) calls `pool.swap()` directly:

```solidity
// metric-periphery/contracts/MetricOmmSimpleRouter.sol:72-80
(int128 amount0Delta, int128 amount1Delta) = IMetricOmmPoolActions(params.pool)
    .swap(
        params.recipient,
        params.zeroForOne,
        ...
    );
```

When this executes, `msg.sender` inside `pool.swap()` is the router contract, so `sender` passed to `beforeSwap` is the router's address — not the originating EOA.

**Exploit flow:**

1. Pool admin deploys a pool with `SwapAllowlistExtension` and allowlists only specific trusted EOAs.
2. Allowlisted users want to use the router. The pool admin calls `setAllowedToSwap(pool, routerAddress, true)` to enable router-mediated swaps.
3. Any unprivileged user (not in the allowlist) calls `MetricOmmSimpleRouter.exactInputSingle` targeting that pool.
4. The router calls `pool.swap()`, making the router the `msg.sender` to the pool.
5. `beforeSwap` receives `sender = routerAddress`, checks `allowedSwapper[pool][routerAddress]` → `true` → passes.
6. The unprivileged user's swap executes successfully, bypassing the allowlist entirely.

**Existing guards are insufficient:** There is no mechanism in the router or the pool to forward the originating EOA's address as `sender`. The pool has no concept of a "real caller" beyond `msg.sender`. The extension has no way to recover the true end user from the calldata.

## Impact Explanation
The swap allowlist is a core access-control mechanism. Its bypass allows any unprivileged user to trade in a pool that the admin intended to restrict to specific counterparties. This breaks the pool's intended access model and can expose restricted liquidity to unauthorized traders, constituting broken core pool functionality and a direct admin-boundary break reachable by any unprivileged router caller.

## Likelihood Explanation
The condition is easily triggered: any user who calls `MetricOmmSimpleRouter` targeting an allowlist-gated pool where the admin has allowlisted the router address can exploit this. The router is a public, permissionless contract. No special privileges, flash loans, or complex setup are required. The pool admin's allowlisting of the router is a natural and expected operational step once they want their allowlisted users to use the router, making this a realistic production scenario.

## Recommendation
The extension must check the originating user, not the immediate caller. Two viable approaches:

1. **Pass the originating caller through the router:** Have the router encode the originating `msg.sender` in `extensionData` and have `SwapAllowlistExtension.beforeSwap` decode and check it. This requires a convention between the router and the extension.

2. **Check `recipient` instead of `sender`:** If the pool's design intent is that the recipient is the economically relevant party, gate on `recipient`. However, this changes semantics and may not match all use cases.

3. **Preferred — check both sender and recipient:** Require that either the direct caller or the recipient is allowlisted, so router-mediated swaps are only permitted when the recipient is an allowlisted address.

The cleanest fix is option 1: the router encodes `msg.sender` into `extensionData`, and the extension reads and validates it, so the true end user is always checked regardless of intermediary.

## Proof of Concept

```solidity
// Foundry test sketch
function test_routerBypassesSwapAllowlist() public {
    // Setup: pool with SwapAllowlistExtension, only `trustedUser` allowlisted
    vm.prank(poolAdmin);
    swapAllowlist.setAllowedToSwap(pool, trustedUser, true);

    // Admin allowlists the router so trustedUser can use it
    vm.prank(poolAdmin);
    swapAllowlist.setAllowedToSwap(pool, address(router), true);

    // Attacker (not in allowlist) calls the router
    vm.prank(attacker);
    router.exactInputSingle(ExactInputSingleParams({
        pool: pool,
        recipient: attacker,
        zeroForOne: true,
        amountIn: 1e18,
        amountOutMinimum: 0,
        priceLimitX64: 0,
        extensionData: "",
        deadline: block.timestamp
    }));
    // Swap succeeds — attacker bypassed the allowlist
}
```

The test demonstrates that `attacker`, who is not in `allowedSwapper`, successfully swaps because the extension sees `sender = address(router)` which is allowlisted.
