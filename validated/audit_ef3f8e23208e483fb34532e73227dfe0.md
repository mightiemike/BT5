### Title
Malicious IBC Counterparty Can Steal Escrowed NFTs via Crafted `class_id` Prefix — (`x/nft-transfer/keeper/packet.go`)

---

### Summary

`processReceivedPacket` calls `IsAwayFromOrigin` using the attacker-controlled `data.ClassId` field directly. A malicious counterparty chain can craft a packet whose `class_id` starts with `sourcePort/sourceChannel/`, causing `IsAwayFromOrigin` to return `false` and triggering the unescrow branch — `TransferOwner(escrowAddress → receiver)` — for any NFT the local escrow address legitimately holds, without a corresponding prior send on the source chain.

---

### Finding Description

`IsAwayFromOrigin` is a pure string-prefix check: [1](#0-0) 

```go
func IsAwayFromOrigin(sourcePort, sourceChannel, fullClassPath string) bool {
    prefixClassID := GetClassPrefix(sourcePort, sourceChannel)
    return !strings.HasPrefix(fullClassPath, prefixClassID)
}
```

In `processReceivedPacket`, this is called with the **attacker-supplied** `data.ClassId`: [2](#0-1) 

When `isAwayFromOrigin` is `false`, the code executes the unescrow branch: [3](#0-2) 

The `escrowAddress` is derived from the **destination** port/channel (the local chain's channel): [4](#0-3) 

`ValidateBasic()` performs no check on whether `class_id` is consistent with the packet's routing: [5](#0-4) 

`TransferOwner` in the NFT keeper only verifies that `srcOwner` (the escrow address) currently owns the token — it does not verify the transfer was authorized by the original depositor: [6](#0-5) 

---

### Impact Explanation

**Concrete exploit path:**

1. User on local chain sends NFT (`classID = "someClass"`, `tokenID = "token1"`) to counterparty via channel `(nft-transfer, channel-0)`. The NFT is escrowed at `GetEscrowAddress("nft-transfer", "channel-0")`.

2. Malicious counterparty chain crafts and commits a packet:
   - `SourcePort = "nft-transfer"`, `SourceChannel = "channel-1"` (counterparty side)
   - `DestPort = "nft-transfer"`, `DestChannel = "channel-0"` (local side)
   - `class_id = "nft-transfer/channel-1/someClass"`
   - `token_ids = ["token1"]`
   - `receiver = attacker_address`

3. IBC core verifies the packet commitment against the counterparty light client (passes — the counterparty chain committed to this packet).

4. `IsAwayFromOrigin("nft-transfer", "channel-1", "nft-transfer/channel-1/someClass")` → `false`.

5. `unprefixedClassID = RemoveClassPrefix("nft-transfer", "channel-1", "nft-transfer/channel-1/someClass") = "someClass"`.

6. `voucherClassID = ParseClassTrace("someClass").IBCClassID()` → `"someClass"` (empty path, returns `BaseClassId` directly per `IBCClassID()`). [7](#0-6) 

7. `TransferOwner(ctx, "someClass", "token1", escrowAddress, attacker)` — succeeds because the escrow address legitimately owns `token1`.

**Result:** The attacker receives `token1` without ever having sent it. The original sender's NFT is stolen. The escrow invariant (unescrow requires a prior escrow deposit from the same counterparty) is broken.

---

### Likelihood Explanation

Requires a malicious or compromised IBC counterparty chain. This is a realistic threat in the IBC model — chains are expected to validate packet data defensively because counterparty chains are not fully trusted at the application layer. Any chain that establishes an IBC channel with the victim chain can execute this attack if it controls its own packet commitment logic.

---

### Recommendation

In `processReceivedPacket`, do not use `data.ClassId` directly in `IsAwayFromOrigin`. Instead, validate that a "return to origin" packet (`isAwayFromOrigin = false`) is only accepted if the `class_id` starts with the **destination** chain's own port/channel prefix (i.e., the local chain's `destPort/destChannel/`), not the source chain's prefix. The correct check should be:

```go
// A packet is returning to origin only if the class_id was prefixed by THIS chain
// when it originally sent the NFT out. The prefix must match destPort/destChannel.
destPrefix := types.GetClassPrefix(packet.GetDestPort(), packet.GetDestChannel())
isAwayFromOrigin := !strings.HasPrefix(data.ClassId, destPrefix)
```

This mirrors how ICS-20 fungible token transfer handles the analogous `isSource` check — using the **destination** port/channel to determine if the receiving chain is the source.

### Proof of Concept

```go
// Integration test sketch (unmodified Go/Cosmos test setup)
func TestMaliciousCounterpartyUnescrow(t *testing.T) {
    // 1. Setup: local chain has channel (nft-transfer, channel-0)
    //    counterparty has (nft-transfer, channel-1)

    // 2. Legitimate escrow: user sends NFT to counterparty
    //    => NFT "someClass"/"token1" is now owned by escrowAddr = GetEscrowAddress("nft-transfer","channel-0")
    escrowAddr := types.GetEscrowAddress("nft-transfer", "channel-0")
    nftKeeper.MintNFT(ctx, "someClass", "token1", "", "", "", escrowAddr, escrowAddr)

    // 3. Craft malicious packet from counterparty
    maliciousData := types.NewNonFungibleTokenPacketData(
        "nft-transfer/channel-1/someClass", // class_id starts with sourcePort/sourceChannel/
        "",
        []string{"token1"},
        []string{""},
        legitimateSender.String(),
        attacker.String(), // receiver = attacker
    )
    packet := channeltypes.NewPacket(
        maliciousData.GetBytes(), 1,
        "nft-transfer", "channel-1", // source (counterparty)
        "nft-transfer", "channel-0", // dest (local)
        clienttypes.ZeroHeight(), 0,
    )

    // 4. Deliver packet — IsAwayFromOrigin returns false, unescrow executes
    err := keeper.OnRecvPacket(ctx, "", packet, maliciousData)
    require.NoError(t, err)

    // 5. Assert: attacker now owns token1, escrow address lost it
    nft, _ := nftKeeper.GetNFT(ctx, "someClass", "token1")
    require.Equal(t, attacker.String(), nft.GetOwner().String()) // PASSES — exploit confirmed
}
```

```go
func (nftpd NonFungibleTokenPacketData) ValidateBasic() error {
	if strings.TrimSpace(nftpd.ClassId) == "" {
		return newsdkerrors.Wrap(ErrInvalidClassID, "classId cannot be blank")
	}

	if len(nftpd.TokenIds) == 0 {
		return newsdkerrors.Wrap(ErrInvalidTokenID, "tokenId cannot be blank")
	}

	if len(nftpd.TokenIds) != len(nftpd.TokenUris) {
		return newsdkerrors.Wrap(ErrInvalidPacket, "tokenIds and tokenUris lengths do not match")
	}

	if strings.TrimSpace(nftpd.Sender) == "" {
		return newsdkerrors.Wrap(sdkerrors.ErrInvalidAddress, "sender address cannot be blank")
	}

	// decode the sender address
	if _, err := sdk.AccAddressFromBech32(nftpd.Sender); err != nil {
		return newsdkerrors.Wrap(sdkerrors.ErrInvalidAddress, "invalid sender address")
	}

	if strings.TrimSpace(nftpd.Receiver) == "" {
		return newsdkerrors.Wrap(sdkerrors.ErrInvalidAddress, "receiver address cannot be blank")
	}

	// decode the receiver address
	if _, err := sdk.AccAddressFromBech32(nftpd.Receiver); err != nil {
		return newsdkerrors.Wrap(sdkerrors.ErrInvalidAddress, "invalid receiver address")
	}
	return nil
}
```

```go
func (k Keeper) TransferOwner(
	ctx sdk.Context, denomID, tokenID string, srcOwner, dstOwner sdk.AccAddress,
) error {
	if !k.HasDenomID(ctx, denomID) {
		return sdkerrors.Wrapf(types.ErrInvalidDenom, "denom ID %s not exists", denomID)
	}

	nft, err := k.IsOwner(ctx, denomID, tokenID, srcOwner)
	if err != nil {
		return err
	}

	nft.Owner = dstOwner.String()

	k.setNFT(ctx, denomID, nft)
	k.swapOwner(ctx, denomID, tokenID, srcOwner, dstOwner)
	return nil
}
```
