### Title
SwapAllowlistExtension Checks Router Address Instead of Original User, Allowing Any User to Bypass Swap Allowlist via Router — (`metric-periphery/contracts/extensions/SwapAllowlistExtension.sol`)

---

### Summary

`SwapAllowlistExtension.beforeSwap` gates swaps by checking the `sender` argument, which is the pool's `msg.sender`. When swaps are routed through `MetricOmmSimpleRouter`, `sender` is the router address, not the originating user. A pool admin who allowlists the router (required for router-mediated swaps to function) inadvertently grants every user of the router the ability to bypass the allowlist.

---

### Finding Description

`MetricOmmPool.swap` passes `msg.sender` as the `sender` argument to `_beforeSwap`: [1](#0-0) 

`ExtensionCalling._beforeSwap` forwards that value unchanged to the extension: [2](#0-1) 

`SwapAllowlistExtension.beforeSwap` then checks `allowedSwapper[msg.sender][sender]`, where `msg.sender` is the pool and `sender` is whoever called the pool: [3](#0-2) 

When a user routes through `MetricOmmSimpleRouter.exactInputSingle`, the router is the direct caller of `pool.swap`: [4](#0-3) 

So `sender` delivered to the extension is the router address, not the originating EOA. For router-mediated swaps to work at all on an allowlisted pool, the pool admin must add the router to `allowedSwapper`. Once the router is allowlisted, the check `allowedSwapper[pool][router]` passes for every user who routes through it, regardless of whether that user is individually permitted.

The same issue applies to the multi-hop `exactInput` path and to `exactOutput` / `exactOutputSingle`, all of which call `pool.swap` with `msg.sender = router`. [5](#0-4) 

---

### Impact Explanation

The `SwapAllowlistExtension` is the sole on-chain mechanism for a pool admin to restrict which counterparties may trade against the pool's LP liquidity. Bypassing it means every user of the public router can trade against a pool that was configured to accept only specific, vetted counterparties. LP funds are directly exposed to unauthorized order flow, which can include adversarial or toxic flow the allowlist was specifically designed to exclude. This satisfies the "admin-boundary break: factory/oracle role checks are bypassed by an unprivileged path" and "broken core pool functionality causing loss of funds" impact criteria.

---

### Likelihood Explanation

The bypass is trivially reachable: any user calls `MetricOmmSimpleRouter.exactInputSingle` (or any other router entry point) targeting the allowlisted pool. No special permissions, flash loans, or multi-step setup are required. The only precondition — the router being allowlisted — is a normal operational necessity for any pool that wants to support router-mediated swaps, making it a realistic and common deployment state.

---

### Recommendation

The extension must gate on the true originating user, not the immediate pool caller. Two sound approaches:

1. **Pass the original user in `extensionData`**: The router encodes `msg.sender` (the EOA) into `extensionData` before forwarding to the pool. The extension decodes and checks that address. This requires a trusted router convention.
2. **Check `sender` and `recipient` together**: Require that the allowlisted address is the `recipient` (the economic beneficiary) rather than the intermediate caller, or require both `sender` and `recipient` to be allowlisted.
3. **Disallow router allowlisting**: Document that allowlisting the router defeats the guard and enforce this at the extension level by rejecting known router addresses from the allowlist.

---

### Proof of Concept

```
1. Pool admin deploys pool with SwapAllowlistExtension configured.
2. Pool admin calls setAllowedToSwap(pool, router, true)
   — necessary so that router-mediated swaps do not revert.
3. Non-allowlisted user Alice (not in allowedSwapper[pool]) calls:
     MetricOmmSimpleRouter.exactInputSingle({
       pool: pool,
       recipient: Alice,
       zeroForOne: true,
       amountIn: X,
       ...
     })
4. Router calls pool.swap(Alice, true, X, limit, "", extensionData).
   pool.msg.sender = router.
5. Pool calls _beforeSwap(sender=router, ...).
6. Extension evaluates: allowedSwapper[pool][router] == true → passes.
7. Swap executes. Alice receives output tokens.
   Alice was never individually allowlisted; the guard was bypassed entirely.
```
