### Title
`SwapAllowlistExtension` checks the router's address as the swapper identity, allowing any user to bypass the swap allowlist via `MetricOmmSimpleRouter` — (`metric-periphery/contracts/extensions/SwapAllowlistExtension.sol`)

---

### Summary

`SwapAllowlistExtension.beforeSwap` gates swaps by checking the `sender` argument, which is the `msg.sender` of the pool's `swap` call. When a user routes through `MetricOmmSimpleRouter`, `sender` is the **router contract address**, not the actual user. A pool admin who allowlists the router (the natural step to let allowlisted users swap through the standard interface) simultaneously opens the pool to every user on-chain, because the allowlist check passes for anyone who calls the router.

---

### Finding Description

`MetricOmmPool.swap` forwards `msg.sender` as the `sender` argument to every `beforeSwap` hook: [1](#0-0) 

`SwapAllowlistExtension.beforeSwap` uses that `sender` value as the identity to check against the per-pool allowlist: [2](#0-1) 

`MetricOmmSimpleRouter.exactInputSingle` calls `pool.swap(...)` directly, making the router the `msg.sender` of the pool call: [3](#0-2) 

The router does **not** embed the original caller's address in `extensionData`; it passes the user-supplied `params.extensionData` verbatim. The extension therefore has no way to recover the real user identity from a router-mediated swap.

This creates an irresolvable dilemma for the pool admin:

| Router allowlisted? | Effect |
|---|---|
| **Yes** | Every user on-chain can bypass the allowlist by calling the router |
| **No** | Allowlisted users cannot use the standard swap interface at all |

The `SwapAllowlistExtension` interface documents the intent as gating "by swapper address": [4](#0-3) 

But the implementation gates by the **intermediary contract address**, not the swapper.

---

### Impact Explanation

When the router is allowlisted (the only way to let allowlisted users use the standard interface), any non-allowlisted address can call `exactInputSingle` / `exactInput` / `exactOutputSingle` / `exactOutput` on the router and execute swaps against the restricted pool. LPs in a pool that was intended to be access-controlled are exposed to unrestricted counterparties, enabling unauthorized extraction of value from LP positions through unconstrained swaps. This is a direct loss of LP principal/fees above Sherlock thresholds.

---

### Likelihood Explanation

High. `MetricOmmSimpleRouter` is the standard public swap interface. Any pool admin who enables allowlisted users to use the router (a routine operational step) triggers the bypass. No special privileges, flash loans, or contract deployment are required — a plain EOA call to the router suffices.

---

### Recommendation

The extension must gate on the **actual user identity**, not the intermediary. Two sound approaches:

1. **Require the router to embed the caller in `extensionData`**: The router appends `abi.encode(msg.sender)` to `extensionData` before forwarding to the pool; the extension decodes and checks that address. The extension should revert if the embedded address is absent (i.e., direct pool calls without the router must also supply it, or be handled separately).

2. **Check `sender` for direct calls and reject router-mediated calls**: If `sender` is a known router, revert unless `allowAllSwappers` is set. This forces allowlisted users to call the pool directly, which is a UX trade-off but preserves the security invariant.

---

### Proof of Concept

1. Pool admin deploys a pool with `SwapAllowlistExtension` configured.
2. Pool admin calls `setAllowedToSwap(pool, alice, true)` — Alice is the only intended swapper.
3. Pool admin calls `setAllowedToSwap(pool, router, true)` — necessary so Alice can use the router.
4. Charlie (not allowlisted) calls `router.exactInputSingle({pool: pool, ...})`.
5. Router calls `pool.swap(recipient, zeroForOne, amount, limit, "", extensionData)` with `msg.sender = router`.
6. Pool calls `extension.beforeSwap(router, ...)`.
7. Extension checks `allowedSwapper[pool][router]` → **true** → no revert.
8. Charlie's swap executes against the restricted pool, draining LP value without authorization. [2](#0-1) [1](#0-0) [5](#0-4)
