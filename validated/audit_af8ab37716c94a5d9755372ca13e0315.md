### Title
`socializeSubaccount` Reverts When Insolvent Borrower's Debt Exceeds Total Deposits, Permanently Blocking Liquidation Finalization — (`core/contracts/SpotEngine.sol`)

---

### Summary

`SpotEngine.socializeSubaccount` contains an arithmetic check that reverts when a liquidatee's negative balance (debt) exceeds the total deposited amount in a product. Because this function is called unconditionally during liquidation finalization when insurance is exhausted, any insolvent subaccount whose debt exceeds total deposits permanently blocks the liquidation pipeline for that subaccount, leaving bad debt uncleared in the system.

---

### Finding Description

In `SpotEngine.socializeSubaccount`, when a subaccount has a negative balance (a liability) in a product, the function recomputes `cumulativeDepositsMultiplierX18` — the deposit-side accounting multiplier analogous to ECG's `creditMultiplier` — as follows:

```solidity
// SpotEngine.sol lines 256–263
int128 totalDeposited = state.totalDepositsNormalized.mul(
    state.cumulativeDepositsMultiplierX18
);

state.cumulativeDepositsMultiplierX18 = (totalDeposited +
    balance.amount).div(state.totalDepositsNormalized);

require(state.cumulativeDepositsMultiplierX18 > 0);
```

`balance.amount` is negative here (it is the liquidatee's debt). The expression `totalDeposited + balance.amount` computes `totalDeposited - |debt|`. If `|debt| >= totalDeposited`, the numerator becomes zero or negative, the resulting multiplier is `<= 0`, and the `require` reverts. [1](#0-0) 

This revert propagates up through the entire liquidation call stack:

1. `ClearinghouseLiq._finalizeSubaccount` calls `spotEngine.socializeSubaccount(txn.liquidatee)` unconditionally when `v.insurance <= 0`.
2. `liquidateSubaccountImpl` calls `_finalizeSubaccount` when `txn.productId == type(uint32).max`.
3. `Clearinghouse.liquidateSubaccount` delegatecalls `liquidateSubaccountImpl` and propagates the revert via `require(success, string(result))`. [2](#0-1) [3](#0-2) [4](#0-3) 

---

### Impact Explanation

When the condition is triggered, every attempt to finalize the insolvent subaccount via `liquidateSubaccount` with `productId = type(uint32).max` reverts. The bad debt is permanently stuck: the subaccount cannot be socialized, the `cumulativeDepositsMultiplierX18` is never updated to reflect the loss, and depositors in that product cannot recover their proportional share of remaining assets. The liquidation system is rendered non-functional for that subaccount, mirroring the ECG finding where `notifyPnL` was permanently blocked.

---

### Likelihood Explanation

The condition `|balance.amount| >= totalDeposited` is reachable without any privileged action:

- `cumulativeBorrowsMultiplierX18` grows faster than `cumulativeDepositsMultiplierX18` because borrowers pay a higher rate than depositors earn (the spread goes to fees). Over time, a large borrower's actual debt (denominated in `cumulativeBorrowsMultiplierX18` units) can exceed the total deposited amount (denominated in `cumulativeDepositsMultiplierX18` units), even if the normalized borrow amount was within limits at origination.
- If other depositors withdraw (reducing `totalDepositsNormalized`) while the large borrower's position remains open, `totalDeposited` shrinks further.
- High utilization markets with sustained high interest rates accelerate this divergence. [5](#0-4) [6](#0-5) 

---

### Recommendation

Replace the hard `require` with a floor clamp, mirroring the ECG mitigation of setting the multiplier to `0` when loss exceeds supply:

```solidity
int128 newMultiplier = (totalDeposited + balance.amount).div(
    state.totalDepositsNormalized
);
// Clamp to 0 if the debt exceeds total deposits (full socialization)
state.cumulativeDepositsMultiplierX18 = MathHelper.max(0, newMultiplier);
```

If `totalDepositsNormalized` is zero, the division should be skipped entirely and the multiplier set to `0`. This prevents the revert and allows the liquidation finalization to complete, clearing the bad debt from the system even in extreme insolvency scenarios. [7](#0-6) 

---

### Proof of Concept

**Attacker-controlled entry path (no privileged keys required):**

1. A large borrower (`liquidatee`) takes out a large loan in a spot product, borrowing close to the total deposits.
2. Other depositors withdraw over time, reducing `totalDepositsNormalized` and thus `totalDeposited`.
3. Interest accrues: `cumulativeBorrowsMultiplierX18` grows faster than `cumulativeDepositsMultiplierX18`, so the borrower's actual debt (`balance.amount` in absolute terms) grows beyond `totalDeposited`.
4. The borrower becomes undercollateralized. Insurance is insufficient to cover the shortfall (`v.insurance <= 0` after `perpEngine.socializeSubaccount`).
5. Any liquidator submits a `LiquidateSubaccount` transaction with `productId = type(uint32).max` through `Endpoint.submitTransactionsChecked`.
6. `_finalizeSubaccount` calls `spotEngine.socializeSubaccount(txn.liquidatee)`.
7. Inside `socializeSubaccount`, `totalDeposited + balance.amount < 0`, the computed multiplier is `<= 0`, and `require(state.cumulativeDepositsMultiplierX18 > 0)` reverts.
8. The revert propagates: `liquidateSubaccountImpl` → `liquidateSubaccount` → the sequencer's transaction fails.
9. The insolvent subaccount can never be finalized. Bad debt is permanently frozen in the system. [8](#0-7) [9](#0-8)

    function socializeSubaccount(bytes32 subaccount) external {
        require(msg.sender == address(_clearinghouse), ERR_UNAUTHORIZED);

        uint32[] memory _productIds = getProductIds();
        for (uint128 i = 0; i < _productIds.length; ++i) {
            uint32 productId = _productIds[i];

            State memory state = states[productId];
            Balance memory balance = balanceNormalizedToBalance(
                state,
                balances[productId][subaccount]
            );
            if (balance.amount < 0) {
                int128 totalDeposited = state.totalDepositsNormalized.mul(
                    state.cumulativeDepositsMultiplierX18
                );

                state.cumulativeDepositsMultiplierX18 = (totalDeposited +
                    balance.amount).div(state.totalDepositsNormalized);

                require(state.cumulativeDepositsMultiplierX18 > 0);

                state.totalBorrowsNormalized += balance.amount.div(
                    state.cumulativeBorrowsMultiplierX18
                );

                _setBalanceAndUpdateBitmap(
                    productId,
                    subaccount,
                    BalanceNormalized({amountNormalized: 0})
                );
                _setState(productId, state);
            }
        }
