### Title
Empty Merkle Proof Incorrectly Rejected for Single-Transaction Blocks — (`contract/src/lib.rs`)

---

### Summary

`verify_transaction_inclusion` unconditionally panics when `merkle_proof` is empty. For a Bitcoin block containing only one transaction (a coinbase-only block), an empty proof is **mathematically correct**: the merkle root equals the transaction hash and no proof elements are needed. The underlying `compute_root_from_merkle_proof` already handles this correctly, but the guard at line 315 fires first and kills the call.

---

### Finding Description

In `contract/src/lib.rs`, `verify_transaction_inclusion` contains:

```rust
require!(!args.merkle_proof.is_empty(), "Merkle proof is empty");
``` [1](#0-0) 

This check fires **before** the merkle root comparison and unconditionally rejects any call that supplies an empty proof vector.

However, `compute_root_from_merkle_proof` in `merkle-tools/src/lib.rs` is defined as:

```rust
pub fn compute_root_from_merkle_proof(
    transaction_hash: H256,
    transaction_position: usize,
    merkle_proof: &Vec<H256>,
) -> H256 {
    let mut current_hash = transaction_hash;
    // ...
    for proof_hash in merkle_proof { /* loop body */ }
    current_hash   // returns transaction_hash unchanged when proof is empty
}
``` [2](#0-1) 

When `merkle_proof` is empty the loop does not execute and `transaction_hash` is returned unchanged. For a single-transaction block, `merkle_root == tx_id`, so the final comparison

```rust
compute_root_from_merkle_proof(args.tx_id, ..., &args.merkle_proof)
    == header.block_header.merkle_root
``` [3](#0-2) 

would evaluate to `true` — but it is never reached because the guard at line 315 panics first.

The same root cause propagates into `verify_transaction_inclusion_v2`, which calls `verify_transaction_inclusion` internally after its own length-equality check passes (both proofs are empty, so `0 == 0`). [4](#0-3) 

**Analog to the FOK bug:** The FOK check `newOrder.amount > 0` incorrectly rejects dust amounts below `lotSize`. Here, `!merkle_proof.is_empty()` incorrectly rejects a valid empty proof for a single-tx block. Both are threshold/boundary comparison errors that cause unnecessary panics on valid inputs.

---

### Impact Explanation

Any NEAR caller invoking `verify_transaction_inclusion` (or `_v2`) with a valid transaction from a single-transaction Bitcoin block receives a panic instead of `true`. Downstream contracts — cross-chain bridges, payment verification systems, or any dApp that gates logic on SPV proof results — will fail to process valid Bitcoin transactions in coinbase-only blocks. If such a contract holds funds contingent on confirmation, the funds can be permanently locked or the action permanently blocked.

---

### Likelihood Explanation

Coinbase-only blocks are a documented, recurring phenomenon on Bitcoin mainnet. Miners produce them when the mempool is empty or when they choose to mine empty blocks for latency reasons. This is not a theoretical edge case: it occurs on mainnet with measurable frequency. Any user attempting to prove a transaction in such a block triggers the bug with no ability to work around it — the proof is mathematically empty and cannot be made non-empty.

---

### Recommendation

Remove the guard at line 315:

```rust
// Remove this line:
require!(!args.merkle_proof.is_empty(), "Merkle proof is empty");
```

`compute_root_from_merkle_proof` already handles the empty-proof case correctly. The merkle root comparison that follows is the authoritative check and is sufficient on its own.

---

### Proof of Concept

1. A Bitcoin block is mined containing only the coinbase transaction. Its `merkle_root` equals `coinbase_tx_id`.
2. The block header is submitted to the contract and stored.
3. A user calls `verify_transaction_inclusion` with:
   - `tx_id = coinbase_tx_id`
   - `tx_block_blockhash = block_hash`
   - `tx_index = 0`
   - `merkle_proof = []`
4. Execution reaches line 315: `require!(!args.merkle_proof.is_empty(), ...)` → **panics** with `"Merkle proof is empty"`.
5. Correct behavior: `compute_root_from_merkle_proof(coinbase_tx_id, 0, &[])` returns `coinbase_tx_id`; `coinbase_tx_id == merkle_root` → `true`. [5](#0-4)

    pub fn verify_transaction_inclusion(&self, #[serializer(borsh)] args: ProofArgs) -> bool {
        require!(
            args.confirmations <= self.gc_threshold,
            "The required number of confirmations exceeds the number of blocks stored in memory"
        );

        let heaviest_block_header = self
            .headers_pool
            .get(&self.mainchain_tip_blockhash)
            .unwrap_or_else(|| env::panic_str(ERR_KEY_NOT_EXIST));
        let target_block_height = self
            .mainchain_header_to_height
            .get(&args.tx_block_blockhash)
            .unwrap_or_else(|| env::panic_str("block does not belong to the current main chain"));

        // Check requested confirmations. No need to compute proof if insufficient confirmations.
        require!(
            (heaviest_block_header.block_height).saturating_sub(target_block_height) + 1
                >= args.confirmations,
            "Not enough blocks confirmed"
        );

        let header = self
            .headers_pool
            .get(&args.tx_block_blockhash)
            .unwrap_or_else(|| env::panic_str("cannot find requested transaction block"));

        require!(!args.merkle_proof.is_empty(), "Merkle proof is empty");

        // compute merkle tree root and check if it matches block's original merkle tree root
        merkle_tools::compute_root_from_merkle_proof(
            args.tx_id,
            usize::try_from(args.tx_index).unwrap(),
            &args.merkle_proof,
        ) == header.block_header.merkle_root
    }

pub fn compute_root_from_merkle_proof(
    transaction_hash: H256,
    transaction_position: usize,
    merkle_proof: &Vec<H256>,
) -> H256 {
    let mut current_hash = transaction_hash;
    let mut current_position = transaction_position;

    for proof_hash in merkle_proof {
        if current_position % 2 == 0 {
            current_hash = compute_hash(&current_hash, proof_hash);
        } else {
            current_hash = compute_hash(proof_hash, &current_hash);
        }
        current_position /= 2;
    }

    current_hash
}
