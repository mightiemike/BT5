Audit Report

## Title
`SwapAllowlistExtension` checks router address instead of end-user, allowing any caller to bypass per-user swap allowlist via `MetricOmmSimpleRouter` - (`metric-periphery/contracts/extensions/SwapAllowlistExtension.sol`)

## Summary
`SwapAllowlistExtension.beforeSwap` gates swaps by checking `allowedSwapper[pool][sender]`, where `sender` is the direct caller of `pool.swap()`. When `MetricOmmSimpleRouter` is used, the router contract is that direct caller, not the end user. A pool admin who adds the router to the allowlist (the only way to enable router-mediated swaps for legitimate users) simultaneously grants every address on the network the ability to swap on the curated pool, completely defeating the allowlist's purpose.

## Finding Description
**Root cause — `MetricOmmPool.swap` passes `msg.sender` as `sender`:**
`MetricOmmPool.swap` calls `_beforeSwap(msg.sender, ...)` at line 231. [1](#0-0) 

**`ExtensionCalling._beforeSwap` forwards `sender` unchanged:**
The value is passed directly as the first argument of the encoded `beforeSwap` call. [2](#0-1) 

**`SwapAllowlistExtension.beforeSwap` checks that forwarded `sender`:**
The check is `allowedSwapper[msg.sender][sender]` — `msg.sender` is the pool (correct), `sender` is whoever called `pool.swap()`. [3](#0-2) 

**`MetricOmmSimpleRouter.exactInputSingle` calls `pool.swap()` directly**, making the router `msg.sender` of the pool: [4](#0-3) 

The same pattern applies to `exactInput` (L104–112), `exactOutputSingle` (L136–137), and `exactOutput` (L165–181) — in every case the pool sees the router as `msg.sender`, so the extension checks `allowedSwapper[pool][router]` instead of `allowedSwapper[pool][real_user]`. [5](#0-4) [6](#0-5) [7](#0-6) 

**Existing guards are insufficient:** There is no mechanism in the router to forward the original `msg.sender` identity to the extension, and no mechanism in the extension to decode an originator from `extensionData`. The `allowedSwapper` mapping is keyed on the direct pool caller, which is structurally always the router when routing through `MetricOmmSimpleRouter`.

## Impact Explanation
A pool admin deploying a curated pool with `SwapAllowlistExtension` faces an inescapable dilemma: if the router is not on the allowlist, all allowlisted users are silently blocked from using the primary supported swap interface; if the router is added to the allowlist (the only way to re-enable router swaps), the check degenerates to `allowedSwapper[pool][router] == true` for every call, so any address on the network can swap on the curated pool by routing through the router. The per-user allowlist is completely bypassed. This is broken core pool functionality — a pool designed to restrict trading to a specific set of counterparties is open to the public — matching the "Admin-boundary break bypassed by an unprivileged path" and "Broken core pool functionality" allowed impacts.

## Likelihood Explanation
`MetricOmmSimpleRouter` is the primary supported swap interface for EOAs. Any pool that uses `SwapAllowlistExtension` and also wants to support router users will add the router to the allowlist, triggering the bypass. No special privilege is required — any public user can call `exactInputSingle` on the router pointing at the curated pool. The attack is repeatable and requires no setup beyond the pool admin having taken the expected action of allowlisting the router.

## Recommendation
The extension must receive the original end-user identity, not the intermediary. Two complementary fixes:

1. **Pass the original sender through the router**: `MetricOmmSimpleRouter` should forward `msg.sender` (the real user) as a dedicated `originator` field in `extensionData`, and `SwapAllowlistExtension` should decode and check that field when present.

2. **Alternatively, check `recipient` instead of `sender`**: for swap allowlists the economically relevant actor is often the recipient of output tokens. The extension could be parameterized to check `recipient` (second argument of `beforeSwap`) rather than `sender`.

The cleanest long-term fix is for the pool to pass both the direct caller and an optional `originator` so extensions can choose which identity to gate.

## Proof of Concept
```
Setup:
  - Deploy pool with SwapAllowlistExtension.
  - Pool admin calls setAllowedToSwap(pool, alice, true)   // alice is the only allowed swapper
  - Pool admin calls setAllowedToSwap(pool, router, true)  // required so alice can use the router

Attack:
  - bob (not on allowlist) calls MetricOmmSimpleRouter.exactInputSingle(
        pool    = curated_pool,
        sender  = bob,          // bob is msg.sender of the router
        ...
    )
  - Router calls pool.swap(recipient, ...) with msg.sender = router
  - Pool calls _beforeSwap(sender=router, ...)
  - ExtensionCalling forwards sender=router to SwapAllowlistExtension.beforeSwap
  - Extension checks allowedSwapper[pool][router] == true  → passes
  - Bob's swap executes on the curated pool despite not being on the allowlist.

Verification: remove router from allowlist → alice's router swaps revert.
              add router back → bob's swaps succeed. The allowlist is fully bypassed.
```
