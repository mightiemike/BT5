Looking at the actual code paths carefully:

**Path 1: Sequencer execution state reader (`ApolloReader::get_compiled_class_hash`)** [1](#0-0) 
This calls `get_compiled_class_hash_at(state_number, class_hash)` which queries the `compiled_class_hash_table` keyed by `(class_hash, block_number)` — a time-series table that records every update, including migration writes. [2](#0-1) 

**Path 2: RPC execution state reader (`ExecutionStateReader::get_compiled_class_hash`)** [3](#0-2) 
This calls `get_class_definition_block_number` (which returns the **original declaration block** B_decl, never updated after first write), then reads `get_state_diff(B_decl)` and extracts `class_hash_to_compiled_class_hash` from that historical state diff — returning H1 forever.

**The declaration block table is never updated on migration:** [4](#0-3) 
The `declared_classes_block_table` is only written when `not_declared` (first time). When the migration block's state diff is stored with `class_hash → H2`, the `declared_classes_block_table` already has B_decl for this class, so it is not updated. The RPC reader will always resolve to B_decl and return H1.

**The migration writes H2 to the compiled_class_hash_table at B_migration:** [5](#0-4) [6](#0-5) 

After `finalize_block`, `to_state_diff()` emits `class_hash → H2` (since H2 ≠ H1 is asserted), which gets stored in the `compiled_class_hash_table` at B_migration. `ApolloReader::get_compiled_class_hash` at any state after B_migration returns H2. `ExecutionStateReader::get_compiled_class_hash` always returns H1 (from B_decl's state diff).

**The divergence is concrete and structural.** The two readers use fundamentally different storage paths:
- Sequencer: time-series `compiled_class_hash_table` (latest value wins) → H2
- RPC: `declared_classes_block_table` → B_decl → `state_diffs_table[B_decl]` → H1

### Title
RPC `ExecutionStateReader::get_compiled_class_hash` returns stale pre-migration H1 while sequencer reads post-migration H2 after CASM hash migration — (`crates/apollo_rpc_execution/src/state_reader.rs`)

### Summary
After a CASM hash migration block commits H2 for class C, the RPC execution state reader permanently returns H1 (from the original declaration block's state diff) while the sequencer's `ApolloReader` returns H2 (from the `compiled_class_hash_table`). This produces a concrete, persistent divergence between RPC simulation and actual sequencer execution.

### Finding Description
`ExecutionStateReader::get_compiled_class_hash` resolves the compiled class hash by:
1. Looking up the class's declaration block B_decl via `get_class_definition_block_number` (reads `declared_classes_block_table`, written once at first declaration, never updated)
2. Fetching the raw `ThinStateDiff` stored at B_decl
3. Returning `state_diff.class_hash_to_compiled_class_hash[class_hash]` = H1 [7](#0-6) 

When `set_compiled_class_hash_migration` writes H2 and `finalize_block` calls `to_state_diff()`, the resulting state diff contains `class_hash → H2` (since `strict_subtract_mappings` emits it because H2 ≠ H1). This diff is stored via `append_state_diff` at B_migration, writing `(class_hash, B_migration) → H2` into the `compiled_class_hash_table`. However, `declared_classes_block_table` is not updated (the `not_declared` guard prevents it). [8](#0-7) 

`ApolloReader::get_compiled_class_hash` uses `get_compiled_class_hash_at(state_number_after_migration, class_hash)` which scans the `compiled_class_hash_table` and returns H2. The two readers permanently disagree.

### Impact Explanation
Any RPC call that uses `ExecutionStateReader` (simulation, fee estimation, `starknet_call`) after a migration block will use H1 as the compiled class hash for class C. The sequencer uses H2. This affects:
- RPC simulation of declare transactions for class C (wrong "already declared" check)
- RPC fee estimation for transactions touching class C's compiled class hash
- Any protocol logic that reads `get_compiled_class_hash` through the RPC execution path

This matches the allowed impact: **High — RPC execution, fee estimation, tracing, simulation, or pending view returns an authoritative-looking wrong value.**

### Likelihood Explanation
Requires `enable_casm_hash_migration = true` (a versioned constants flag) and at least one class declared with a v1 (Pedersen) compiled class hash to be executed in a block. An unprivileged user can trigger the migration for an eligible class by submitting an invoke transaction that calls it. Once the migration block is committed, the divergence is permanent for that class.

### Recommendation
Change `ExecutionStateReader::get_compiled_class_hash` to use the same storage path as `ApolloReader`: query `get_compiled_class_hash_at(state_number, class_hash)` from the `compiled_class_hash_table` directly, rather than resolving through the declaration block's historical state diff. The `compiled_class_hash_table` is the authoritative time-series record and correctly reflects migration updates.

### Proof of Concept
1. Declare class C with compiled class hash H1 (v1/Pedersen) at block B_decl.
2. Enable `enable_casm_hash_migration`.
3. Submit an invoke transaction executing class C; `finalize_block` migrates it to H2 at block B_migration.
4. Call `ExecutionStateReader::get_compiled_class_hash(C)` → resolves B_decl → reads state diff at B_decl → returns H1.
5. Call `ApolloReader::get_compiled_class_hash(C)` at state after B_migration → reads `compiled_class_hash_table` → returns H2.
6. H1 ≠ H2 (asserted by the migration code itself at line 26–29 of `compiled_class_hash_migration.rs`).

fn get_compiled_class_hash_at<'env, Mode: TransactionKind>(
    first_irrelevant_block: BlockNumber,
    class_hash: &ClassHash,
    txn: &'env DbTransaction<'env, Mode>,
    compiled_class_hash_table: &'env CompiledClassHashTable<'env>,
) -> StorageResult<Option<CompiledClassHash>> {
    let db_key = (*class_hash, first_irrelevant_block);
    // Find the previous db item.
    let mut cursor = compiled_class_hash_table.cursor(txn)?;
    cursor.lower_bound(&db_key)?;
    let res = cursor.prev()?;
    match res {
        None => Ok(None),
        Some(((got_class_hash, _got_block_number), value)) => {
            if got_class_hash != *class_hash {
                // The previous item belongs to different class hash, which means there is no
                // previous state diff for this item.
                return Ok(None);
            };
            // The previous db item indeed belongs to this address and key.
            Ok(Some(value))
        }
    }

    fn get_compiled_class_hash(&self, class_hash: ClassHash) -> StateResult<CompiledClassHash> {
        if let Some(pending_data) = &self.maybe_pending_data {
            for DeclaredClassHashEntry { class_hash: other_class_hash, compiled_class_hash } in
                &pending_data.declared_classes
            {
                if class_hash == *other_class_hash {
                    return Ok(*compiled_class_hash);
                }
            }
        }

        let maybe_block_number = self
            .storage_reader
            .begin_ro_txn()
            .map_err(storage_err_to_state_err)?
            .get_state_reader()
            .map_err(storage_err_to_state_err)?
            .get_class_definition_block_number(&class_hash)
            .map_err(storage_err_to_state_err)?;

        // Cairo 0 classes (and undeclared classes) do not have a compiled class hash.
        // According to the trait, return the default value.
        let Some(block_number) = maybe_block_number else {
            return Ok(CompiledClassHash::default());
        };

        let state_diff = self
            .storage_reader
            .begin_ro_txn()
            .map_err(storage_err_to_state_err)?
            .get_state_diff(block_number)
            .map_err(storage_err_to_state_err)?
            .ok_or(StateError::StateReadError(format!(
                "Inner storage error. Missing state diff at block {block_number}."
            )))?;

        let compiled_class_hash = state_diff
            .class_hash_to_compiled_class_hash
            .get(&class_hash)
            .ok_or(StateError::StateReadError(format!(
                "Inner storage error. Missing class declaration at block {block_number}, class \
                 {class_hash}."
            )))?;

        Ok(*compiled_class_hash)
    }
