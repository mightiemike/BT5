### Title
Wrong Block Used in Fork Difficulty Calculation (Mainchain Lookup Instead of Fork Ancestor) - (File: `contract/src/bitcoin.rs`, `contract/src/litecoin.rs`, `contract/src/dogecoin.rs`)

---

### Summary

When computing the required difficulty for a fork block at a difficulty-adjustment boundary, all three chain implementations call `get_header_by_height()` to retrieve the first block in the adjustment interval. That helper always reads from `mainchain_height_to_header`, so it returns the **mainchain** block at the requested height, not the fork's actual ancestor. If the fork diverged before that height, the timestamps differ, the computed `actual_time_taken` is wrong, and the expected `bits` value is wrong — allowing a fork block with an incorrect (easier) difficulty to pass the PoW check.

---

### Finding Description

**Root cause — `get_header_by_height` is mainchain-only** [1](#0-0) 

```rust

    self.mainchain_height_to_header          // ← only mainchain
        .get(&height)
        .and_then(|hash| self.headers_pool.get(&hash))
        .unwrap_or_else(|| env::panic_str(ERR_KEY_NOT_EXIST))
}
```

The `check_pow` / `check_target` call happens **before** `submit_block_header_inner` stores the block, so the fork's own block at `height_first` is already in `headers_pool` but is never consulted — only the mainchain entry is. [5](#0-4) 

---

### Impact Explanation

**Incorrect `actual_time_taken` → wrong expected `bits`**

In `calculate_next_work_required` (all three chains):

```
actual_time_taken = prev_block_header.time − first_block_time   (mainchain value, wrong)
correct           = prev_block_header.time − fork_ancestor.time (fork value, correct)
```

If the attacker sets the fork's block at `height_first` to a timestamp **later** than the mainchain block at the same height, then `actual_time_taken` (using the earlier mainchain timestamp) is **larger** than the correct value, producing an **easier** target. The fork block at the adjustment boundary then passes `require!(expected_bits == block_header.bits)` with a lower difficulty than Bitcoin/Dogecoin/Litecoin actually requires.

A fork that accumulates more `chain_work` than the mainchain tip triggers `reorg_chain`, replacing the canonical chain and corrupting `mainchain_tip_blockhash`, `mainchain_height_to_header`, and all subsequent `verify_transaction_inclusion` / `verify_transaction_inclusion_v2` results. [6](#0-5) 

---

### Likelihood Explanation

**Dogecoin (highest likelihood):** With `difficulty_adjustment_interval = 1` for heights ≥ 145 000, `height_first = prev_block_height − 1`. The bug activates for the fork block at height `D+2` after diverging at height `D`. The attacker needs only **two** fork blocks with valid PoW before the miscalculation takes effect. The Dogecoin difficulty cap limits the easement to ≈ 1.5× (max\_timespan / retarget\_timespan), but that is still a meaningful reduction achievable by a modestly resourced attacker.

**Bitcoin / Litecoin (lower likelihood):** The adjustment interval is 2 016 blocks, so the attacker must produce ≈ 2 016 fork blocks with valid PoW before the miscalculation applies. The difficulty can be eased up to 4× (the protocol cap). This requires substantial hash power but is not impossible for a well-resourced adversary.

---

### Recommendation

Replace the `get_header_by_height` call with a backwards traversal of the fork chain starting from `prev_block_header`, stopping at `first_block_height`. Because `get_prev_header` follows `prev_block_hash` through `headers_pool` (not the mainchain map), it correctly walks the fork ancestry:

```rust
// Instead of:
let first_block = blocks_getter.get_header_by_height(first_block_height);

// Use:
let mut cursor = prev_block_header.clone();
while cursor.block_height > first_block_height {
    cursor = blocks_getter.get_prev_header(&cursor.block_header);
}
// cursor is now the fork's true ancestor at first_block_height
let first_block_time = cursor.block_header.time;
```

This mirrors the reference Bitcoin Core implementation, which walks `pindexFirst` backwards through the block index rather than looking up by height.

---

### Proof of Concept

**Dogecoin scenario (heights ≥ 145 000, every-block adjustment):**

1. Mainchain has block at height `D` with timestamp `T_main`.
2. Attacker mines a fork block at height `D` with timestamp `T_fork = T_main + 7 000 s` (within the 2-hour future-timestamp limit relative to the NEAR block time). This block has valid Scrypt PoW against the current difficulty.
3. Attacker mines a fork block at height `D+1` with valid PoW (difficulty unchanged, since `height_first = D−1` which is the same in both chains).
4. Attacker submits the fork block at height `D+2`. The contract calls `get_header_by_height(D)` → returns mainchain block with timestamp `T_main`.
   - `actual_time_taken = fork_block_{D+1}.time − T_main` (inflated by ≈ 7 000 s)
   - `modulated_timespan` is clamped to `max_timespan = 90 s` (vs correct ≈ 60 s)
   - New target is ≈ 1.5× easier than the true required target.
5. The fork block at `D+2` passes `require!(expected_bits == block_header.bits)` with the easier bits value.
6. Repeating this pattern, the fork accumulates chain\_work faster than the mainchain, eventually triggering `reorg_chain` and replacing the canonical chain. [7](#0-6) [8](#0-7)

```rust
fn get_next_work_required(
    config: &DogecoinConfig,
    block_header: &Header,
    prev_block_header: &ExtendedHeader,
    blocks_getter: &impl BlocksGetter,
) -> u32 {
    // Dogecoin: Special rules for minimum difficulty blocks with Digishield
    if allow_min_difficulty_for_block(config, block_header, prev_block_header) {
        // Special difficulty rule for testnet:
        // If the new block's timestamp is more than 2* nTargetSpacing minutes
        // then allow mining of a min-difficulty block.
        return config.proof_of_work_limit_bits;
    }

    // Only change once per difficulty adjustment interval
    let new_difficulty_protocol = prev_block_header.block_height >= 145_000;
    let difficulty_adjustment_interval = if new_difficulty_protocol {
        1
    } else {
        config.difficulty_adjustment_interval
    };

    if (prev_block_header.block_height + 1) % difficulty_adjustment_interval != 0 {
        if config.pow_allow_min_difficulty_blocks {
            // Special difficulty rule for testnet:
            // If the new block's timestamp is more than 2* 10 minutes
            // then allow mining of a min-difficulty block.
            if block_header.time
                > prev_block_header.block_header.time + config.pow_target_spacing * 2
            {
                return config.proof_of_work_limit_bits;
            }

            // Return the last non-special-min-difficulty-rules-block
            let mut current_block_header = prev_block_header.clone();

            while current_block_header.block_header.bits == config.proof_of_work_limit_bits
                && current_block_header.block_height % config.difficulty_adjustment_interval != 0
            {
                current_block_header =
                    blocks_getter.get_prev_header(&current_block_header.block_header);
            }

            return current_block_header.block_header.bits;
        }

        return prev_block_header.block_header.bits;
    }

    // Litecoin: This fixes an issue where a 51% attack can change difficulty at will.
    // Go back the full period unless it's the first retarget after genesis. Code courtesy of Art Forz
    let mut blocks_to_go_back = difficulty_adjustment_interval - 1;
    if prev_block_header.block_height + 1 != difficulty_adjustment_interval {
        blocks_to_go_back = difficulty_adjustment_interval;
    }

    // Go back by what we want to be 14 days worth of blocks
    let height_first = prev_block_header
        .block_height
        .checked_sub(blocks_to_go_back)
        .unwrap_or_else(|| env::panic_str("Height underflow when calculating first block height"));

    // TODO: check if it is correct to get block header by height from mainchain without looping to find the ancestor
    let first_block_time = blocks_getter
        .get_header_by_height(height_first)
        .block_header
        .time;

    calculate_next_work_required(config, prev_block_header, i64::from(first_block_time))
```
