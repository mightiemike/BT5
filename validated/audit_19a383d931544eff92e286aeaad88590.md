### Title
`ValidResourceBounds` Variant Collapse in Protobuf Round-Trip Causes Valid Invoke V3 Transaction to Be Rejected by All Peers — (`File: crates/apollo_protobuf/src/converters/transaction.rs`)

### Summary

The protobuf deserialization of `ResourceBounds` irreversibly collapses `AllResources { l1_gas: X, l2_gas: 0, l1_data_gas: 0 }` into `L1Gas(X)`. A valid invoke V3 transaction with only `l1_gas` non-zero is accepted by the gateway (hash computed under the `AllResources` 3-felt preimage), but every peer that receives it via P2P or consensus protobuf rejects it because the conversion back to `RpcInvokeTransactionV3` hard-fails on `L1Gas` input. The transaction is permanently stuck and can never be sequenced.

### Finding Description

**Root cause — lossy `TryFrom<protobuf::ResourceBounds> for ValidResourceBounds`** [1](#0-0) 

```rust
let l1_data_gas = value.l1_data_gas.unwrap_or_default();   // absent → zero
...
Ok(if l1_data_gas.is_zero() && l2_gas.is_zero() {
    ValidResourceBounds::L1Gas(l1_gas)          // ← collapses AllResources
} else {
    ValidResourceBounds::AllResources(...)
})
```

Any `AllResources` transaction whose `l2_gas` and `l1_data_gas` are both zero is indistinguishable from a legacy `L1Gas` transaction after protobuf round-trip. The protobuf wire format carries no discriminant bit.

**Gateway accepts the transaction under `AllResources` hash**

The stateless validator explicitly allows `AllResourceBounds { l1_gas: NON_EMPTY, l2_gas: 0, l1_data_gas: 0 }`: [2](#0-1) 

The hash is computed by `get_invoke_transaction_v3_hash`, which chains `[tip, l1_gas, l2_gas=0, l1_data_gas=0]` — three resource felts — for `AllResources`: [3](#0-2) 

**Peer conversion hard-fails on `L1Gas`**

When the transaction is serialized to `protobuf::InvokeV3WithProof` and deserialized by a peer, the `resource_bounds` field becomes `ValidResourceBounds::L1Gas(X)`. The conversion to `RpcInvokeTransactionV3` then fails: [4](#0-3) 

```rust
// This conversion can fail only if the resource_bounds are not AllResources.
Ok(Self { proof, ..snapi_invoke.try_into().map_err(|_| DEPRECATED_RESOURCE_BOUNDS_ERROR)? })
```

`RpcInvokeTransactionV3.resource_bounds` is typed `AllResourceBounds`, so `L1Gas` is structurally incompatible: [5](#0-4) 

This rejection propagates through both the mempool P2P path: [6](#0-5) 

and the consensus path: [7](#0-6) 

**Hash domain divergence**

Even if the conversion did not fail, the hash computed by a peer after deserialization would differ from the gateway's hash. `get_tip_resource_bounds_hash` produces a 2-felt preimage for `L1Gas` and a 3-felt preimage for `AllResources`: [8](#0-7) 

```
AllResources: poseidon([tip, l1_gas_felt, l2_gas_felt=0, l1_data_gas_felt=0])
L1Gas:        poseidon([tip, l1_gas_felt, l2_gas_felt=0])
```

These are distinct field elements, so the transaction hash the gateway stored differs from any hash a peer would compute.

### Impact Explanation

A user who submits a valid invoke V3 transaction with `AllResourceBounds { l1_gas > 0, l2_gas = 0, l1_data_gas = 0 }` has it accepted by the gateway and placed in the mempool. Every other node in the network rejects it on receipt via P2P or consensus protobuf. If the proposer includes it in a block, validators reject the entire block proposal, stalling consensus for that height. The transaction can never be sequenced. This matches **High — Mempool/gateway/RPC admission rejects valid transactions before sequencing** and **High — Transaction conversion or signature/hash logic binds the wrong type or executable payload**.

### Likelihood Explanation

The trigger requires no privilege: any user can submit an invoke V3 transaction with only `l1_gas` non-zero. The gateway's stateless validator explicitly permits this configuration. The condition `l2_gas = 0 && l1_data_gas = 0` is the natural default for users who only care about L1 execution cost.

### Recommendation

In `TryFrom<protobuf::ResourceBounds> for ValidResourceBounds`, produce `AllResources` whenever `l1_data_gas` is explicitly present in the wire message (even if its value is zero), and reserve `L1Gas` only for the legacy case where `l1_data_gas` is absent (`None`). This preserves the variant discriminant across the serialization boundary:

```rust
let l1_data_gas_opt = value.l1_data_gas;
let l1_data_gas: ResourceBounds = l1_data_gas_opt.unwrap_or_default().try_into()?;
Ok(if l1_data_gas_opt.is_none() && l2_gas.is_zero() {
    ValidResourceBounds::L1Gas(l1_gas)
} else {
    ValidResourceBounds::AllResources(AllResourceBounds { l1_gas, l2_gas, l1_data_gas })
})
```

Alternatively, add an explicit discriminant field to the `ResourceBounds` protobuf message.

### Proof of Concept

1. Craft an invoke V3 transaction with `AllResourceBounds { l1_gas: { max_amount: 1000, max_price_per_unit: 1 }, l2_gas: { max_amount: 0, max_price_per_unit: 0 }, l1_data_gas: { max_amount: 0, max_price_per_unit: 0 } }`.
2. Submit to the gateway via `starknet_addInvokeTransaction`. The stateless validator accepts it (matches the `valid_l1_gas` test case). The gateway computes hash H₁ using the `AllResources` 3-felt preimage and stores the transaction in the mempool.
3. The mempool P2P propagator serializes the transaction to `protobuf::MempoolTransaction` → `protobuf::InvokeV3WithProof`. The `resource_bounds` wire fields are `{ l1_gas: X, l2_gas: 0, l1_data_gas: 0 }`.
4. A peer receives the protobuf. `TryFrom<protobuf::ResourceBounds>` fires: `l1_data_gas.is_zero() && l2_gas.is_zero()` → `ValidResourceBounds::L1Gas(X)`.
5. `TryFrom<protobuf::InvokeV3WithProof> for RpcInvokeTransactionV3` calls `snapi_invoke.try_into()` which fails because `resource_bounds` is `L1Gas`, not `AllResources`. The peer returns `DEPRECATED_RESOURCE_BOUNDS_ERROR` and drops the transaction.
6. The same rejection occurs on the consensus path when the proposer includes the transaction in a block proposal: every validator's `TryFrom<protobuf::ConsensusTransaction>` fails at the same point, and the block is rejected.

            return Err(missing("ResourceBounds::l1_gas"));

pub fn get_tip_resource_bounds_hash(
    resource_bounds: &ValidResourceBounds,
    tip: &Tip,
) -> Result<Felt, StarknetApiError> {
    let l1_resource_bounds = resource_bounds.get_l1_bounds();
    let l2_resource_bounds = resource_bounds.get_l2_bounds();

    // L1 and L2 gas bounds always exist.
    // Old V3 txs always have L2 gas bounds of zero, but they exist.
    let mut resource_felts = vec![
        get_concat_resource(&l1_resource_bounds, L1_GAS)?,
        get_concat_resource(&l2_resource_bounds, L2_GAS)?,
    ];

    // For new V3 txs, need to also hash the data gas bounds.
    resource_felts.extend(match resource_bounds {
        ValidResourceBounds::L1Gas(_) => vec![],
        ValidResourceBounds::AllResources(all_resources) => {
            vec![get_concat_resource(&all_resources.l1_data_gas, L1_DATA_GAS)?]
        }
    });

    Ok(HashChain::new().chain(&tip.0.into()).chain_iter(resource_felts.iter()).get_poseidon_hash())
}

impl TryFrom<protobuf::InvokeV3WithProof> for RpcInvokeTransactionV3 {
    type Error = ProtobufConversionError;
    fn try_from(mut value: protobuf::InvokeV3WithProof) -> Result<Self, Self::Error> {
        // Extract proof first, since `starknet_api::transaction::InvokeTransactionV3` does not
        // carry a `proof` field.
        let proof = Proof::from(std::mem::take(&mut value.proof));

        let snapi_invoke: InvokeTransactionV3 = value
            .invoke
            .ok_or(ProtobufConversionError::MissingField {
                field_description: "InvokeV3WithProof::invoke",
            })?
            .try_into()?;

        // This conversion can fail only if the resource_bounds are not AllResources.
        Ok(Self { proof, ..snapi_invoke.try_into().map_err(|_| DEPRECATED_RESOURCE_BOUNDS_ERROR)? })
    }
