Audit Report

## Title
SwapAllowlistExtension Checks Router Address Instead of End-User, Allowing Any User to Bypass the Swap Allowlist via MetricOmmSimpleRouter - (File: metric-periphery/contracts/extensions/SwapAllowlistExtension.sol)

## Summary
`SwapAllowlistExtension.beforeSwap` gates swaps by checking the `sender` argument forwarded by the pool, which is always the pool's immediate `msg.sender`. When a user routes through `MetricOmmSimpleRouter`, the pool's `msg.sender` is the router contract, so the allowlist checks whether the router is permitted rather than whether the individual user is permitted. Any user can bypass a curated pool's swap allowlist by calling through the public, permissionless router.

## Finding Description
`MetricOmmPool.swap` passes `msg.sender` as the `sender` argument to `_beforeSwap` (L230–231). `SwapAllowlistExtension.beforeSwap` then evaluates `allowedSwapper[msg.sender][sender]` (L37), where `msg.sender` is the pool and `sender` is the pool's own `msg.sender` — the router. `MetricOmmSimpleRouter.exactInputSingle` (L71–80) calls `pool.swap()` directly without encoding the real user into `extensionData`; the router stores the real user only in transient callback context for payment purposes, which is never visible to the extension. The same structural problem applies to `exactInput` (L103–112), `exactOutputSingle` (L135–137), and `exactOutput` (L165–181). For multi-hop `exactInput`, intermediate hops use `address(this)` as payer, making the router the pool's `msg.sender` for every hop. The allowlist check therefore becomes `allowedSwapper[pool][router]`, creating an impossible configuration: not allowlisting the router breaks all router-mediated swaps for legitimately allowlisted users; allowlisting the router grants every user on the network the ability to bypass the allowlist.

## Impact Explanation
A pool configured with `SwapAllowlistExtension` to restrict trading to a curated set of addresses (e.g., KYC'd counterparties, whitelisted market makers) provides zero protection against any user who calls through `MetricOmmSimpleRouter`. The bypass is unconditional and requires no special privilege. This constitutes broken core pool functionality: the allowlist policy is completely unenforceable through the primary user-facing swap entry point, allowing unauthorized parties to drain LP value through swaps the pool admin explicitly intended to block.

## Likelihood Explanation
`MetricOmmSimpleRouter` is the primary user-facing swap entry point for the protocol and is a public, permissionless contract. Any user aware that a pool has a swap allowlist can trivially route through the router to bypass it. No special knowledge, capital, or timing is required. The bypass is deterministic and repeatable on every block.

## Recommendation
The extension must gate on the economically relevant actor — the end-user — not the immediate caller of `pool.swap()`. The preferred fix is a router-aware allowlist: the extension checks `sender` first; if `sender` is a registered trusted router, it decodes the real user from `extensionData` and checks that address instead. This requires the router to encode `msg.sender` into `extensionData` before calling `pool.swap()`, and the factory to maintain a registry of approved routers that the extension can trust. Alternatively, the router could be modified to always ABI-encode the originating user address into `extensionData` as a protocol-level convention, and the extension updated to decode and verify it when `sender` is a known router.

## Proof of Concept
```
Setup:
  - Deploy pool with SwapAllowlistExtension
  - Pool admin calls setAllowedToSwap(pool, alice, true)
  - Pool admin does NOT allowlist bob or the router

Attack (router not allowlisted):
  1. bob calls MetricOmmSimpleRouter.exactInputSingle({pool: pool, ...})
  2. Router calls pool.swap(recipient=bob, ...)
  3. Pool calls _beforeSwap(sender=router, ...)
  4. SwapAllowlistExtension checks allowedSwapper[pool][router] → false → revert NotAllowedToSwap()
  (alice also cannot use the router — all router-mediated swaps are broken)

Attack (router allowlisted to fix alice):
  5. Pool admin calls setAllowedToSwap(pool, router, true)
  6. bob calls MetricOmmSimpleRouter.exactInputSingle({pool: pool, ...})
  7. Router calls pool.swap(recipient=bob, ...)
  8. Pool calls _beforeSwap(sender=router, ...)
  9. SwapAllowlistExtension checks allowedSwapper[pool][router] → true → PASSES
  10. bob's swap executes successfully despite bob never being allowlisted

There is no configuration that enforces per-user allowlisting through the router.
```
