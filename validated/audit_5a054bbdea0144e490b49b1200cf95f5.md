### Title
Unchecked `transferFrom` Return Value in `replaceUsdcEWithUsdc` Enables usdcE Drain from Direct Deposit Accounts — (File: `core/contracts/ContractOwner.sol`)

---

### Summary

`ContractOwner.replaceUsdcEWithUsdc` calls `IERC20Base(usdc).transferFrom(msg.sender, directDepositV1, balance)` without checking the boolean return value. If the USDC token returns `false` on failure rather than reverting, execution continues, allowing any caller to drain usdcE from a Direct Deposit Account (DDA) without providing any USDC.

---

### Finding Description

`replaceUsdcEWithUsdc` is an unrestricted `external` function (gated only by `block.chainid == 57073`) intended to swap usdcE held in a DDA for USDC. The intended flow is:

1. Pull USDC from `msg.sender` into the DDA via `transferFrom`.
2. Withdraw usdcE from the DDA to `ContractOwner`.
3. Send usdcE to `msg.sender`.

At step 1, the return value of `transferFrom` is silently discarded:

```solidity
IERC20Base(usdc).transferFrom(msg.sender, directDepositV1, balance);
``` [1](#0-0) 

Steps 2 and 3 execute unconditionally regardless of whether step 1 succeeded:

```solidity
DirectDepositV1(directDepositV1).withdraw(IIERC20Base(usdcE));
IERC20Base(usdcE).safeTransfer(msg.sender, balance);
``` [2](#0-1) 

By contrast, every other ERC20 transfer in the codebase uses `ERC20Helper.safeTransfer` / `ERC20Helper.safeTransferFrom`, which wrap the call and assert the return value:

```solidity
(bool success, bytes memory data) = address(self).call(
    abi.encodeWithSelector(IERC20Base.transferFrom.selector, from, to, amount)
);
require(
    success && (data.length == 0 || abi.decode(data, (bool))),
    ERR_TRANSFER_FAILED
);
``` [3](#0-2) 

The DDA is owned by `ContractOwner` (set during `DirectDepositV1` construction), so `ContractOwner` has the authority to call `withdraw` on it: [4](#0-3) 

---

### Impact Explanation

If the USDC token at `0x2D270e6886d130D724215A266106e6832161EAEd` on Ink chain (chainid 57073) returns `false` on a failed `transferFrom` (e.g., insufficient allowance or balance) rather than reverting, an attacker can:

1. Call `replaceUsdcEWithUsdc(subaccount)` for any subaccount whose DDA holds usdcE.
2. The `transferFrom` silently fails — no USDC is moved.
3. `withdraw(usdcE)` pulls the DDA's entire usdcE balance into `ContractOwner`.
4. `safeTransfer(usdcE)` sends that balance to the attacker.

**Corrupted asset delta**: The DDA loses its full usdcE balance. The attacker receives usdcE without providing USDC. The subaccount's collateral backing is permanently reduced.

---

### Likelihood Explanation

The function is callable by any address on Ink chain with no access control beyond the chain ID check. The attacker needs no special privileges, no prior relationship with the protocol, and no capital — only the ability to call the function. Exploitability depends on whether the specific USDC deployment on Ink returns `false` vs. reverts on failure. Non-reverting ERC20 tokens are common (USDT, older USDC versions), and the code pattern is unconditionally unsafe regardless of current token behavior, since token implementations can change or be replaced.

---

### Recommendation

Replace the bare `transferFrom` call with `ERC20Helper.safeTransferFrom`, consistent with every other transfer in the codebase:

```solidity
// Before (unsafe):
IERC20Base(usdc).transferFrom(msg.sender, directDepositV1, balance);

// After (safe):
IERC20Base(usdc).safeTransferFrom(msg.sender, directDepositV1, balance);
``` [5](#0-4) 

---

### Proof of Concept

**Precondition**: A DDA exists for `subaccount` on Ink chain (chainid 57073) with a non-zero usdcE balance. The USDC token at the hardcoded address returns `false` (rather than reverting) when `transferFrom` is called with zero allowance.

**Attack steps**:

1. Attacker calls `ContractOwner.replaceUsdcEWithUsdc(subaccount)` with zero USDC balance and zero allowance.
2. `IERC20Base(usdc).transferFrom(attacker, dda, balance)` returns `false` — not checked, execution continues.
3. `DirectDepositV1(dda).withdraw(usdcE)` transfers the DDA's usdcE to `ContractOwner`.
4. `IERC20Base(usdcE).safeTransfer(attacker, balance)` sends the usdcE to the attacker.
5. Attacker receives the DDA's full usdcE balance at zero cost. The DDA's collateral is drained. [5](#0-4)

    function replaceUsdcEWithUsdc(bytes32 subaccount) external {
        require(block.chainid == 57073, ERR_UNAUTHORIZED);

        uint256 balance = IERC20Base(usdcE).balanceOf(directDepositV1);
        if (balance > 0) {
            IERC20Base(usdc).transferFrom(msg.sender, directDepositV1, balance);
            DirectDepositV1(directDepositV1).withdraw(IIERC20Base(usdcE));
            IERC20Base(usdcE).safeTransfer(msg.sender, balance);
        }
    }

    function safeTransferFrom(
        IERC20Base self,
        address from,
        address to,
        uint256 amount
    ) internal {
        (bool success, bytes memory data) = address(self).call(
            abi.encodeWithSelector(
                IERC20Base.transferFrom.selector,
                from,
                to,
                amount
            )
        );

        );
