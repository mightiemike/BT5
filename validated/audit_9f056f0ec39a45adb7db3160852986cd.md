### Title
Borrow Interest Accrues During Slow-Mode Deposit Delay, Unfairly Charging the Depositing User — (File: `core/contracts/Endpoint.sol`)

---

### Summary

When a user with a negative spot balance (active borrow) deposits collateral via `depositCollateralWithReferral()`, ERC-20 tokens are pulled from the user immediately, but the on-chain balance credit is queued as a slow-mode transaction with a hardcoded **3-day delay** (`SLOW_MODE_TX_DELAY`). During this window, the user's balance remains negative and continues to accrue borrow interest via `SpotEngineState._updateState()`, even though the user has already committed the repayment funds. The accrued interest flows to existing depositors — not to the depositing user — mirroring the exact unfairness described in the reference report.

---

### Finding Description

In `Endpoint.depositCollateralWithReferral()`, the deposit flow is split across two phases:

**Phase 1 — Immediate token transfer:**
`handleDepositTransfer()` pulls tokens from the user immediately, and a `SlowModeTx` is enqueued with `executableAt = block.timestamp + SLOW_MODE_TX_DELAY` (3 days). [1](#0-0) 

**Phase 2 — Delayed balance credit (up to 3 days later):**
Only when the slow-mode transaction is executed does `Clearinghouse.depositCollateral()` call `spotEngine.updateBalance()` to credit the user's balance. [2](#0-1) 

Between Phase 1 and Phase 2, the sequencer periodically calls `SpotEngineState.updateStates(dt)`, which advances `cumulativeBorrowsMultiplierX18` for all products. Any subaccount with a negative `amountNormalized` has its effective debt grow with each call. [3](#0-2) 

The borrow interest accrued during the delay is distributed to existing depositors via the `cumulativeDepositsMultiplierX18` increase — the depositing user's committed tokens sit idle in the clearinghouse, credited to no one, while the user's debt silently grows. [4](#0-3) 

The `SLOW_MODE_TX_DELAY` is hardcoded to 3 days and `NLP_LOCK_PERIOD` to 4 days: [5](#0-4) 

---

### Impact Explanation

A user with a negative balance who calls `depositCollateralWithReferral()` pays up to **3 days of additional borrow interest** that they do not owe, because their funds are already in the protocol. This interest is redistributed to existing depositors, constituting a direct, quantifiable asset loss for the depositing user. At typical borrow rates on a large position, 3 days of interest is a non-trivial amount. The broken invariant is: *a user who has committed funds to repay a borrow should not continue to accrue borrow interest on those committed funds.*

---

### Likelihood Explanation

`depositCollateralWithReferral()` is a publicly callable, permissionless entrypoint. Any regular subaccount with a negative balance that uses this path is affected. The sequencer can execute slow-mode transactions early, but there is **no on-chain guarantee** it will do so promptly. If the sequencer is delayed, congested, or the user is being censored (a scenario the slow-mode path is explicitly designed to handle as a fallback), the full 3-day delay applies. This is a realistic, protocol-acknowledged scenario. [6](#0-5) 

---

### Recommendation

When tokens are deposited via `depositCollateralWithReferral()`, immediately credit the user's balance in the `SpotEngine` (or at minimum snapshot the borrow balance and freeze interest accrual on the committed amount). The slow-mode queue can still be used for sequencer ordering purposes, but the interest clock on the committed funds should stop at the moment of token transfer, not at the moment of sequencer execution.

---

### Proof of Concept

1. User has a negative quote balance of −1,000 USDC (active borrow).
2. User calls `depositCollateralWithReferral(subaccount, QUOTE_PRODUCT_ID, 1000e6, "")`.
3. 1,000 USDC is transferred from the user to the clearinghouse immediately via `handleDepositTransfer()`.
4. A `SlowModeTx` is enqueued with `executableAt = block.timestamp + 3 days`.
5. The sequencer calls `updateStates(dt)` periodically; `cumulativeBorrowsMultiplierX18` advances.
6. After 3 days, the user's effective debt has grown from −1,000 USDC to approximately −1,000 × (1 + borrowRate × 3/365) USDC.
7. The slow-mode transaction executes: `spotEngine.updateBalance(QUOTE_PRODUCT_ID, subaccount, +1000e18)` credits the original deposit amount.
8. The user's net balance is now negative by the 3-day interest amount — they paid the full 1,000 USDC three days ago but still owe the interest accrued during the delay. That interest has already been distributed to existing depositors.

        uint32 productId,
        State memory state,
        uint128 dt
    ) internal {
        int128 borrowRateMultiplierX18;
        int128 totalDeposits = state.totalDepositsNormalized.mul(
            state.cumulativeDepositsMultiplierX18
        );
        int128 totalBorrows = state.totalBorrowsNormalized.mul(
            state.cumulativeBorrowsMultiplierX18
        );
        int128 utilizationRatioX18 = totalBorrows.div(totalDeposits);
        int128 minDepositRateX18;
        {
            Config memory config = configs[productId];

            // annualized borrower rate
            int128 borrowerRateX18 = config.interestFloorX18;
            if (utilizationRatioX18 == 0) {
                // setting borrowerRateX18 to 0 here has the property that
                // adding a product at the beginning of time and not using it until time T
                // results in the same state as adding the product at time T
                borrowerRateX18 = 0;
            } else if (utilizationRatioX18 < config.interestInflectionUtilX18) {
                borrowerRateX18 += config
                    .interestSmallCapX18
                    .mul(utilizationRatioX18)
                    .div(config.interestInflectionUtilX18);
            } else {
                borrowerRateX18 +=
                    config.interestSmallCapX18 +
                    config.interestLargeCapX18.mul(
                        (
                            (utilizationRatioX18 -
                                config.interestInflectionUtilX18).div(
                                    ONE - config.interestInflectionUtilX18
                                )
                        )
                    );
            }

                MathSD21x18.fromInt(31536000)
            );
