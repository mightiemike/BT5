## Title
Stale `lastDecayTs` After `executeOracleStopLossDecay` Retroactively Misapplies New Rate, Silently Disabling the Stop-Loss Guard — (`metric-periphery/contracts/extensions/OracleValueStopLossExtension.sol`)

## Summary
`executeOracleStopLossDecay()` writes the new `decayPerSecondE8` into `oracleStopLossConfig` but never checkpoints `highWatermarks[pool][binIdx].lastDecayTs` for any bin. On the next swap, `_checkAndUpdateWatermarks` computes `dt = block.timestamp - hwmS.lastDecayTs` using the stale timestamp and applies the new (higher) rate over the entire elapsed interval, including the pre-change period. When `newRate × dt ≥ 1e8`, `_decayed()` returns 0, zeroing the watermark and causing `_applyWatermark` to never detect a breach, silently disabling the stop-loss guard for those bins.

## Finding Description
`executeOracleStopLossDecay` at L139–147 only writes `oracleStopLossConfig[pool_].decayPerSecondE8 = decay` (L144) and clears the pending schedule. No per-bin `lastDecayTs` is reset or checkpointed.

On the next swap, `_afterSwapOracleStopLoss` reads `cfg.decayPerSecondE8` (L235) — now the new higher rate — and passes it to `_checkAndUpdateWatermarks` (L241). Inside that function (L267–285), `dt = block.timestamp - hwmS.lastDecayTs` (L268) spans the full idle period since the last swap, which includes the pre-change window. `_decayed(hwmS.token0, decayRate, dt)` at L270 computes `factor = ratePerSecondE8 * dt`; when `factor >= E8` (L322), it returns 0. With a zeroed watermark, `_applyWatermark(metric, 0, floorMultiplier)` at L333 evaluates `metric >= 0` — always true for a `uint256` — and returns `(metric, false)`, so `breach0` and `breach1` are never set. The revert at L271–273 and L276–278 is never reached. The stop-loss guard is silently disabled for those bins.

No existing guard prevents this: `_validateDecay` (L309–311) only enforces `decayPerSecondE8 <= E8`; the timelock (L142) delays execution but does not prevent retroactive misapplication; and there is no checkpoint mechanism in `executeOracleStopLossDecay`.

## Impact Explanation
With the stop-loss guard disabled, value-draining swaps proceed without triggering `OracleStopLossTriggered`. LP principal can drain arbitrarily beyond the configured `drawdownE6` threshold. This is a direct loss of LP principal — a core protection invariant of the extension is broken. The impact satisfies the "broken core pool functionality causing loss of funds" and "direct loss of user principal" criteria.

## Likelihood Explanation
The pool admin is a semi-trusted role that legitimately increases the decay rate within the allowed cap (`decayPerSecondE8 <= E8`) and after the timelock elapses — both conditions are normal operational actions. Any bin that has not been touched since `lastDecayTs` is vulnerable. In low-activity pools or during quiet periods, bins can be idle for days. With the PoC parameters (`newRate = 580`, `dt = 3 days + 1s`), `factor = 150,336,580 ≥ 1e8`, which is easily achievable. After the admin's rate change, any unprivileged trader triggering a swap in an idle bin causes the retroactive zeroing and guard bypass. The condition is repeatable across all idle bins simultaneously.

## Recommendation
Before writing the new `decayPerSecondE8`, checkpoint every affected bin's watermark by applying the current (old) rate up to `block.timestamp` and resetting `lastDecayTs`. Since active bins are not enumerated on-chain, the minimal safe fix is one of:

1. **Piecewise decay**: Store the rate-change timestamp alongside the rate and apply rate₁ for `[lastDecayTs, changeTs]` and rate₂ for `[changeTs, now]` inside `_decayed`.
2. **Checkpoint function**: Add `checkpointBinWatermark(address pool_, int8 binIdx)` that applies the current rate and resets `lastDecayTs`, and require all active bins to be checkpointed before `executeOracleStopLossDecay` is called.
3. **Conservative reset**: Inside `executeOracleStopLossDecay`, reset `lastDecayTs` for all bins to `block.timestamp` (treating the pre-change period as zero-decay), preventing retroactive misapplication at the cost of forgoing decay credit for the idle period.
