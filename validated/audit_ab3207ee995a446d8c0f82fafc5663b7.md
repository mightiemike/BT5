Audit Report

## Title
`SwapAllowlistExtension.beforeSwap` checks router address instead of end user, enabling allowlist bypass via `MetricOmmSimpleRouter` — (File: `metric-periphery/contracts/extensions/SwapAllowlistExtension.sol`)

## Summary
`SwapAllowlistExtension.beforeSwap` gates swaps by checking `sender`, which resolves to `msg.sender` of `pool.swap()`. When swaps are routed through `MetricOmmSimpleRouter`, `sender` is the router contract address, not the originating user. A pool admin who allowlists the router to enable router-mediated swaps for permitted users inadvertently grants every user the ability to bypass the allowlist, because the extension cannot distinguish between different end users transiting the same router.

## Finding Description
`SwapAllowlistExtension.beforeSwap` performs the check:

```solidity
if (!allowAllSwappers[msg.sender] && !allowedSwapper[msg.sender][sender]) {
    revert IMetricOmmPoolActions.NotAllowedToSwap();
}
```

`msg.sender` here is the pool (the extension's caller), and `sender` is the first argument forwarded by the pool — which is `msg.sender` of the pool's own `swap()` call (`metric-core/contracts/MetricOmmPool.sol`, lines 230–231):

```solidity
_beforeSwap(
    msg.sender,   // ← becomes `sender` in the extension
    ...
```

`MetricOmmSimpleRouter.exactInputSingle` calls `pool.swap()` directly (`metric-periphery/contracts/MetricOmmSimpleRouter.sol`, lines 72–80), making the router the `msg.sender` to the pool. Therefore `sender` in the extension is always the router address for any router-mediated swap, regardless of which end user initiated the call.

The swap interface has no separate "actual user" field analogous to the `owner` parameter in `beforeAddLiquidity` (`metric-periphery/contracts/extensions/DepositAllowlistExtension.sol`, lines 32–41), which correctly checks the position owner rather than the direct caller. The swap hook receives only `sender` (the direct caller), collapsing all router users into a single identity.

A pool admin who wants allowlisted users to be able to use the router has no alternative but to call `setAllowedToSwap(pool, router, true)`. The moment that entry is set, `allowedSwapper[pool][router] == true`, and `beforeSwap` passes for every caller — allowlisted or not — who routes through the router.

## Impact Explanation
LPs on a curated pool (swap allowlist configured to restrict counterparties) who also need allowlisted users to access the canonical `MetricOmmSimpleRouter` are exposed to unrestricted oracle-priced trading by all users. Non-allowlisted actors can drain LP principal through adversarial or uninformed order flow that the pool was specifically designed to exclude. This is a direct loss of LP principal and a broken core pool invariant (the allowlist cap on who may trade).

## Likelihood Explanation
The `MetricOmmSimpleRouter` is the canonical, publicly documented swap entrypoint. A pool admin configuring a curated pool will naturally add the router to the allowlist to support allowlisted users — this is the only mechanism available to make router-mediated swaps work for permitted users. The bypass is a predictable consequence of a reasonable and expected admin configuration, not an exotic edge case. Any unprivileged user can exploit it by simply calling `router.exactInputSingle` targeting the curated pool.

## Recommendation
The `beforeSwap` hook interface should expose the actual end user's identity separately from `sender`, mirroring how `beforeAddLiquidity` exposes `owner` independently of `sender`. Until the interface is extended, `SwapAllowlistExtension` must decode the actual caller from `extensionData` (requiring the router to encode `msg.sender` into `extensionData` before calling `pool.swap()`). As an interim measure, the extension and router documentation must warn that allowlisting the router on a curated pool grants unrestricted access to all router users.

## Proof of Concept
1. Pool admin deploys a pool with `SwapAllowlistExtension` in the `beforeSwap` order.
2. Pool admin allowlists Alice: `setAllowedToSwap(pool, alice, true)`.
3. Pool admin allowlists the router so Alice can use it: `setAllowedToSwap(pool, router, true)`.
4. Bob (not allowlisted) calls `router.exactInputSingle({pool: pool, ...})`.
5. Router calls `pool.swap(recipient, zeroForOne, amountIn, priceLimit, "", extensionData)` — `msg.sender` to the pool is the router address.
6. Pool calls `_beforeSwap(msg.sender=router, ...)` (`MetricOmmPool.sol` lines 230–240).
7. `SwapAllowlistExtension.beforeSwap` evaluates `allowedSwapper[pool][router]` = `true` → does not revert (`SwapAllowlistExtension.sol` line 37).
8. Bob's swap executes on the curated pool, bypassing the allowlist entirely.
