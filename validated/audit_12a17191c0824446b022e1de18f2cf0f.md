### Title
Protobuf `ValidResourceBounds` Deserialization Silently Downgrades `AllResources` to `L1Gas` When `l2_gas` and `l1_data_gas` Are Zero, Causing Consensus P2P Deserialization Failure for Valid Transactions - (File: crates/apollo_protobuf/src/converters/transaction.rs)

### Summary

The `TryFrom<protobuf::ResourceBounds> for ValidResourceBounds` conversion silently reclassifies an `AllResources` transaction as `L1Gas` whenever both `l2_gas` and `l1_data_gas` are zero. The gateway accepts such transactions and computes their hash under the `AllResources` branch. When the proposer later broadcasts the block proposal over P2P consensus, receiving nodes fail to deserialize the transaction and reject the proposal, breaking consensus liveness.

### Finding Description

**Root cause — `crates/apollo_protobuf/src/converters/transaction.rs` line 431:**

```rust
let l1_data_gas = value.l1_data_gas.unwrap_or_default();   // silently zero if absent
...
Ok(if l1_data_gas.is_zero() && l2_gas.is_zero() {
    ValidResourceBounds::L1Gas(l1_gas)                      // ← wrong variant
} else {
    ValidResourceBounds::AllResources(AllResourceBounds { l1_gas, l2_gas, l1_data_gas })
})
``` [1](#0-0) 

Any `AllResources` transaction whose `l2_gas` and `l1_data_gas` fields are both zero is silently demoted to `L1Gas` on deserialization.

**Gateway accepts the transaction under `AllResources`:**

`RpcTransaction` and `InternalRpcTransaction` always carry `AllResourceBounds` (never `ValidResourceBounds`). The gateway's stateless validator explicitly allows `l2_gas = 0` and `l1_data_gas = 0`: [2](#0-1) 

`convert_rpc_tx_to_internal` computes the transaction hash from `InternalRpcTransactionWithoutTxHash`, which always wraps `AllResourceBounds` and therefore always calls `get_invoke_transaction_v3_hash` with `ValidResourceBounds::AllResources`. That hash includes the `L1_DATA_GAS` packed field: [3](#0-2) 

**P2P consensus path fails on the receiving node:**

When the proposer serializes the block proposal, `From<ValidResourceBounds> for protobuf::ResourceBounds` emits all three fields (l1_gas, l2_gas=0, l1_data_gas=0): [4](#0-3) 

On the receiving node, `TryFrom<protobuf::ConsensusTransaction> for ConsensusTransaction` calls `TryFrom<protobuf::InvokeV3WithProof> for RpcInvokeTransactionV3`: [5](#0-4) 

The inner `snapi_invoke.try_into()` converts `InvokeTransactionV3` (now carrying `ValidResourceBounds::L1Gas`) to `RpcInvokeTransactionV3`. That conversion requires `AllResources` and returns `DEPRECATED_RESOURCE_BOUNDS_ERROR` otherwise: [6](#0-5) 

The same failure path exists for `DeployAccountV3` and `DeclareV3WithClass`: [7](#0-6) 

**The test suite already acknowledges the bug** but only papers over it in tests by forcing `l2_gas.max_amount = GasAmount(1)`: [8](#0-7) 

No equivalent guard exists in production code.

### Impact Explanation

A valid transaction (accepted by the gateway, correctly hashed, placed in the mempool) causes every receiving consensus node to return a `ProtobufConversionError` when deserializing the block proposal. The proposal is rejected, the round fails, and the network stalls until a new proposer is elected. Because the proposer's own node never re-deserializes the transaction from protobuf, the divergence is invisible to the proposer.

This matches: **High — Mempool/gateway/RPC admission accepts invalid transactions or rejects valid transactions before sequencing.**

### Likelihood Explanation

Any unprivileged user can submit an invoke, declare, or deploy-account transaction with `l2_gas = 0` and `l1_data_gas = 0`. This is a normal configuration for transactions that only consume L1 gas (pre-0.13.3 style). No special knowledge or privileged access is required. The gateway provides no error or warning.

### Recommendation

The `AllResources` / `L1Gas` distinction must be preserved across the protobuf boundary independently of whether the numeric values are zero. Two concrete fixes:

1. **Preserve variant by field presence:** Use the presence/absence of the `l1_data_gas` field in the protobuf message (not its value) to decide the variant. Remove the `unwrap_or_default()` fallback and instead map `None` → `L1Gas`, `Some(_)` → `AllResources`.

2. **Reject at the gateway:** Add a stateless validation rule that rejects `AllResources` transactions where both `l2_gas` and `l1_data_gas` are zero, forcing users to submit them as the `L1Gas` variant (which round-trips correctly).

### Proof of Concept

1. Submit an invoke V3 transaction via the RPC gateway with `resource_bounds = { l1_gas: {max_amount: 1000, max_price_per_unit: 1}, l2_gas: {max_amount: 0, max_price_per_unit: 0}, l1_data_gas: {max_amount: 0, max_price_per_unit: 0} }`.
2. Gateway passes stateless validation (zero l2/l1_data gas is explicitly allowed).
3. `convert_rpc_tx_to_internal` computes hash H using `AllResources` (4-element poseidon: tip + L1_GAS + L2_GAS + L1_DATA_GAS).
4. Transaction enters the mempool; proposer includes it in a block proposal.
5. Proposer serializes the proposal to protobuf: `l2_gas = 0`, `l1_data_gas = 0` are emitted as present-but-zero fields.
6. Receiving validator deserializes: `l1_data_gas.is_zero() && l2_gas.is_zero()` → `ValidResourceBounds::L1Gas`.
7. `RpcInvokeTransactionV3::try_from(InvokeTransactionV3 { resource_bounds: L1Gas, .. })` → `Err(DEPRECATED_RESOURCE_BOUNDS_ERROR)`.
8. `TryFrom<protobuf::ConsensusTransaction>` returns an error; the receiving node rejects the entire block proposal.
9. Consensus round fails; repeat for every round where this transaction is included.

impl TryFrom<protobuf::InvokeV3WithProof> for RpcInvokeTransactionV3 {
    type Error = ProtobufConversionError;
    fn try_from(mut value: protobuf::InvokeV3WithProof) -> Result<Self, Self::Error> {
        // Extract proof first, since `starknet_api::transaction::InvokeTransactionV3` does not
        // carry a `proof` field.
        let proof = Proof::from(std::mem::take(&mut value.proof));

        let snapi_invoke: InvokeTransactionV3 = value
            .invoke
            .ok_or(ProtobufConversionError::MissingField {
                field_description: "InvokeV3WithProof::invoke",
            })?
            .try_into()?;

        // This conversion can fail only if the resource_bounds are not AllResources.
        Ok(Self { proof, ..snapi_invoke.try_into().map_err(|_| DEPRECATED_RESOURCE_BOUNDS_ERROR)? })
    }
