### Title
ML-DSA-65 Gas-Key Nonce Exec Fee Uses Wire Length Instead of Trie-ID Length in Host Functions, Causing ~59× Overcharge — (`runtime/near-vm-runner/src/logic/logic.rs`)

---

### Summary

The host functions `promise_batch_action_add_gas_key_with_full_access` and `promise_batch_action_add_gas_key_with_function_call` pass the raw wire-format `public_key_len` (1953 bytes for ML-DSA-65) to `gas_key_add_key_exec_fee`, but the actual trie key for each nonce entry uses the SHA3-256 hash form (33 bytes, `trie_id_len()`). The runtime path for the same action correctly calls `public_key.trie_id_len()`. The mismatch causes a ~59× overcharge on the per-byte exec-fee component for every ML-DSA-65 gas key added via a contract host function.

---

### Finding Description

`gas_key_add_key_exec_fee` computes the exec fee for writing nonce trie entries when adding a gas key. It takes `public_key_len` and uses it to compute:

```rust
let nonce_key_len =
    access_key_key_len(account_id_len, public_key_len) + std::mem::size_of::<NonceIndex>();
```

The actual trie key for a nonce entry uses `trie_id_len()` — 33 bytes for ML-DSA-65 (the SHA3-256 hash form), not the 1953-byte borsh-encoded full pubkey.

**Host function path — wrong value:**

In `promise_batch_action_add_gas_key_with_full_access` and `promise_batch_action_add_gas_key_with_function_call`, the raw `public_key_len` from the host ABI (1953 for ML-DSA-65) is forwarded directly:

```rust
let exec_fee = gas_key_add_key_exec_fee(
    &self.fees_config,
    receiver_id.len(),
    public_key_len as usize,   // ← wire length: 1953 for ML-DSA-65
    num_nonces,
);
```

The same pattern appears in the wasmtime runner at the same call site.

**Runtime path — correct value:**

`permission_exec_fees` in `runtime/runtime/src/config.rs` correctly calls `trie_id_len()`:

```rust
let nonce_fee = gas_key_add_key_exec_fee(
    fees,
    account_id.len(),
    public_key.trie_id_len(),  // ← 33 bytes for ML-DSA-65
    gas_key_info.num_nonces,
);
```

The design document explicitly states: *"Every storage-stake and trie-byte-priced fee path was updated to call `trie_id_len()`."* The host function path is a missed update.

The divergent value is exact: `1953` (wire) vs `33` (trie-id), a difference of **1920 bytes per nonce key**. For a gas key with `N` nonces the per-byte exec overcharge is `gas_key_byte_exec_fee × 1920 × N`.

---

### Impact Explanation

Any contract that calls `promise_batch_action_add_gas_key_with_full_access` or `promise_batch_action_add_gas_key_with_function_call` with an ML-DSA-65 public key is charged ~59× more exec gas than the actual trie work warrants. This:

1. Causes systematic, deterministic overcharging for every such call — a direct financial loss to the contract caller.
2. Can cause the transaction to exhaust its gas budget unexpectedly, making the operation fail even when the caller has budgeted correctly based on the documented fee model.
3. Creates a fee-invariant break: the same `AddKey` action costs materially different amounts depending on whether it is submitted as a transaction (correct) or emitted by a contract via host functions (overcharged).

---

### Likelihood Explanation

Both `GasKeys` and `PostQuantumSignatures` stabilized at protocol version 85. After that version activates on mainnet, any contract that programmatically manages ML-DSA-65 gas keys hits this path unconditionally. No special privilege is required — any unprivileged contract caller can trigger it. The overcharge is deterministic and reproducible on every invocation.

---

### Recommendation

Decode the public key before computing the exec fee and pass `trie_id_len()` instead of the raw wire length. In both `promise_batch_action_add_gas_key_with_full_access` and `promise_batch_action_add_gas_key_with_function_call`:

```rust
let decoded_key = public_key.decode()?;
let exec_fee = gas_key_add_key_exec_fee(
    &self.fees_config,
    receiver_id.len(),
    decoded_key.trie_id_len(),   // ← 33 for ML-DSA-65, unchanged for ed25519/secp256k1
    num_nonces,
);
// then pass decoded_key to append_action_add_gas_key_*
```

This aligns the host-function path with the runtime path in `permission_exec_fees` and with the documented invariant that all trie-byte-priced fee paths use `trie_id_len()`.
