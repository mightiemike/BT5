### Title
Unprivileged Caller Can Aggressively Prune Canonical Chain State via `run_mainchain_gc()` - (File: `contract/src/lib.rs`)

### Summary
`run_mainchain_gc(batch_size: u64)` is a public, state-mutating function callable by any unprivileged NEAR account when the contract is not paused. It accepts an unsanitized `batch_size` parameter and permanently overwrites the critical `mainchain_initial_blockhash` state variable while deleting block headers from `headers_pool` and `mainchain_height_to_header`. This is a direct analog to the reported `createGauge4Pool()` bug: missing access control on a function that overwrites a critical singleton state variable with attacker-influenced inputs.

### Finding Description

`run_mainchain_gc` is decorated only with `#[pause(except(roles(Role::UnrestrictedRunGC)))]`: [1](#0-0) 

The `#[pause]` decorator restricts callers **only when the contract is paused**. During normal (unpaused) operation, any NEAR account — with no role, no stake, no registration — can call this function. The `batch_size` parameter receives no validation: it is not bounded, not checked against zero, and not checked against any maximum. The function then:

1. Computes `selected_amount_to_remove = min(total_amount_to_remove, batch_size)` where `total_amount_to_remove = amount_of_headers_we_store - gc_threshold`.
2. Deletes every mainchain block header in the range `[start_removal_height, end_removal_height)` from both `mainchain_height_to_header` and `headers_pool`.
3. **Overwrites `mainchain_initial_blockhash`** with the new oldest block hash. [2](#0-1) 

By passing `batch_size = u64::MAX`, an attacker forces the contract to prune the maximum possible number of mainchain blocks in a single transaction — all blocks that exceed `gc_threshold` — and permanently advances `mainchain_initial_blockhash` to the new tail. This is irreversible: deleted entries in `LookupMap` cannot be recovered.

The intended design is that GC runs automatically inside `submit_blocks` with `batch_size = num_of_headers` (a small, relayer-controlled value), or is called manually by the relayer with a controlled batch size. The public exposure with no caller restriction breaks this invariant. [3](#0-2) 

### Impact Explanation

- **SPV proof forgery/denial**: `verify_transaction_inclusion` and `verify_transaction_inclusion_v2` look up `args.tx_block_blockhash` in `headers_pool`. Any block header deleted by the attacker-triggered GC is permanently gone; all SPV proofs for those blocks will panic with `"cannot find requested transaction block"`. Downstream contracts or bridges relying on these proofs are permanently broken for the pruned range.
- **Chain reorg failure**: `reorg_chain` walks fork blocks back to the common ancestor via `headers_pool`. If the common ancestor was in the GC'd range, the reorg panics with `"PrevBlockNotFound"`, permanently breaking the contract's ability to follow the longest chain.
- **`mainchain_initial_blockhash` corruption**: The canonical "oldest tracked block" pointer is permanently overwritten to an attacker-chosen advancement, corrupting `get_mainchain_size()` and any logic depending on the initial block. [4](#0-3) 

### Likelihood Explanation

The entry point is a public NEAR contract method requiring no deposit, no role, and no stake. Any NEAR account can call it in a single transaction. The contract is expected to be deployed on mainnet with real Bitcoin SPV proofs. The attack requires only knowledge of the contract's account ID, which is public. Likelihood is **high**.

### Recommendation

Add a role-based access control guard to `run_mainchain_gc` so that only privileged callers (e.g., `Role::DAO` or a dedicated `GCManager` role) can invoke it directly. The automatic invocation from within `submit_blocks` (which is already gated by `#[trusted_relayer]`) is safe and should remain. Additionally, validate that `batch_size > 0` to prevent no-op calls from consuming gas unnecessarily.

```rust
// Example fix: restrict direct public calls
#[pause(except(roles(Role::UnrestrictedRunGC)))]
#[access_control_any(roles(Role::DAO, Role::GCManager))]  // add this
pub fn run_mainchain_gc(&mut self, batch_size: u64) {
    require!(batch_size > 0, "batch_size must be positive");
    // ...
}
```

### Proof of Concept

1. Deploy the contract normally (Bitcoin feature, `skip_pow_verification = false`, `gc_threshold = 52704`).
2. Relayer submits 60,000 blocks over time. The mainchain now has 60,000 headers; 7,296 exceed `gc_threshold`.
3. Attacker (any NEAR account) calls:
   ```
   run_mainchain_gc({ "batch_size": 18446744073709551615 })
   ```
4. The contract computes `selected_amount_to_remove = min(7296, u64::MAX) = 7296` and deletes all 7,296 oldest mainchain headers from `headers_pool` and `mainchain_height_to_header`.
5. `mainchain_initial_blockhash` is overwritten to the new oldest block.
6. Any subsequent call to `verify_transaction_inclusion` for a block in the deleted range panics: `"cannot find requested transaction block"`.
7. If a fork exists whose common ancestor is in the deleted range, `reorg_chain` panics: `"PrevBlockNotFound"`, permanently breaking chain-following. [5](#0-4)

```rust
    #[pause(except(roles(Role::UnrestrictedRunGC)))]
    pub fn run_mainchain_gc(&mut self, batch_size: u64) {
        let initial_blockheader = self
            .headers_pool
            .get(&self.mainchain_initial_blockhash)
            .unwrap_or_else(|| env::panic_str(ERR_KEY_NOT_EXIST));

        let tip_blockheader = self
            .headers_pool
            .get(&self.mainchain_tip_blockhash)
            .unwrap_or_else(|| env::panic_str(ERR_KEY_NOT_EXIST));

        let amount_of_headers_we_store =
            tip_blockheader.block_height - initial_blockheader.block_height + 1;

        if amount_of_headers_we_store > self.gc_threshold {
            let total_amount_to_remove = amount_of_headers_we_store - self.gc_threshold;
            let selected_amount_to_remove = std::cmp::min(total_amount_to_remove, batch_size);

            let start_removal_height = initial_blockheader.block_height;
            let end_removal_height = initial_blockheader.block_height + selected_amount_to_remove;
            env::log_str(&format!(
                "Num of blocks to remove {selected_amount_to_remove}"
            ));

            for height in start_removal_height..end_removal_height {
                let blockhash = &self
                    .mainchain_height_to_header
                    .get(&height)
                    .unwrap_or_else(|| env::panic_str(ERR_KEY_NOT_EXIST));

                self.remove_block_header(blockhash);
                self.mainchain_height_to_header.remove(&height);
            }

            self.mainchain_initial_blockhash = self
                .mainchain_height_to_header
                .get(&end_removal_height)
                .unwrap_or_else(|| env::panic_str(ERR_KEY_NOT_EXIST));
        }
    }
```
