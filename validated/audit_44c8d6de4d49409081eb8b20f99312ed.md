### Title
`Role::UnrestrictedSubmitBlocks` Bypasses Trusted-Relayer Whitelist Beyond Its Documented Scope - (`contract/src/lib.rs`)

---

### Summary

`Role::UnrestrictedSubmitBlocks` is documented as granting the ability to submit blocks **only when the contract is paused**. However, because it is also listed in the `bypass_roles` of the `#[trusted_relayer]` macro, it unconditionally bypasses the trusted-relayer whitelist check on `submit_blocks` — even when the contract is fully operational. Any account granted this role can submit arbitrary block headers without being a staked, authorized relayer, corrupting the canonical chain and poisoning `verify_transaction_inclusion` results.

---

### Finding Description

The `Role` enum documents `UnrestrictedSubmitBlocks` with a single, narrow purpose:

> "Allows to use contract API even after contract is paused" [1](#0-0) 

The `submit_blocks` method is guarded by two independent macros: `#[pause]` and `#[trusted_relayer]`. [2](#0-1) 

The `trusted_relayer` macro is applied at the impl level with `bypass_roles` that includes `Role::UnrestrictedSubmitBlocks`: [3](#0-2) 

Placing `UnrestrictedSubmitBlocks` in `bypass_roles` causes it to skip the trusted-relayer whitelist check entirely — not just the pause check. This is confirmed by the integration test helper, whose comment explicitly states the role is used to pass the `#[trusted_relayer]` guard, not the pause guard: [4](#0-3) 

The same pattern is repeated across all chain variants (Dogecoin, Zcash), confirming this is a systemic design issue, not an isolated test artifact. [5](#0-4) 

Compare with `run_mainchain_gc`, which correctly uses `#[pause(except(roles(Role::UnrestrictedRunGC)))]` to grant only a pause bypass — without touching the relayer whitelist: [6](#0-5) 

`UnrestrictedSubmitBlocks` lacks this scoped treatment. Its placement in `bypass_roles` of `trusted_relayer` grants it a second, undocumented capability: submitting blocks as a non-relayer at any time.

---

### Impact Explanation

An account holding `UnrestrictedSubmitBlocks` can call `submit_blocks` with a crafted sequence of block headers at any time, regardless of whether the contract is paused. Because `submit_block_header_inner` promotes a fork to the canonical chain whenever its `chain_work` exceeds the current tip's, the attacker can:

1. Submit a fabricated fork with artificially high chainwork.
2. Trigger `reorg_chain`, replacing the canonical chain with the attacker's fake chain.
3. Cause `verify_transaction_inclusion` and `verify_transaction_inclusion_v2` to return `true` for transactions that never existed on the real Bitcoin chain. [7](#0-6) 

Any downstream NEAR contract (bridge, atomic swap, lending protocol) that trusts the light client's verification result is directly exploitable.

---

### Likelihood Explanation

`UnrestrictedSubmitBlocks` is intended to be granted to emergency or backup relayers who need to operate during a pause. Once granted, the role persists and is active even when the contract is not paused. The admin granting this role for its documented purpose (pause bypass) would not expect it to also remove the trusted-relayer restriction. The role is designed to be granted to multiple accounts across deployments, making the attack surface non-trivial.

---

### Recommendation

Remove `Role::UnrestrictedSubmitBlocks` from the `bypass_roles` of the `trusted_relayer` macro. Instead, apply the pause bypass using the scoped `except(roles(...))` syntax on the `#[pause]` attribute, mirroring the pattern already used for `run_mainchain_gc`:

```rust
// Before (overly broad):
#[trusted_relayer(
    bypass_roles(Role::DAO, Role::UnrestrictedSubmitBlocks),
    ...
)]
#[pause]
#[trusted_relayer]
pub fn submit_blocks(...) { ... }

// After (scoped):
#[trusted_relayer(
    bypass_roles(Role::DAO),   // only DAO bypasses relayer check
    ...
)]
#[pause(except(roles(Role::UnrestrictedSubmitBlocks, Role::DAO)))]
#[trusted_relayer]
pub fn submit_blocks(...) { ... }
```

This ensures `UnrestrictedSubmitBlocks` grants only the pause bypass it documents, while the trusted-relayer whitelist remains enforced for all non-DAO callers.

---

### Proof of Concept

1. Admin grants `UnrestrictedSubmitBlocks` to account `attacker.near` (for legitimate pause-bypass purposes).
2. Contract is **not** paused — normal operation.
3. `attacker.near` calls `submit_blocks` with a sequence of crafted headers whose cumulative `chain_work` exceeds the current canonical tip.
4. The `#[trusted_relayer]` guard is bypassed because `UnrestrictedSubmitBlocks` is in `bypass_roles`.
5. `submit_block_header_inner` detects `current_header.chain_work > total_main_chain_chainwork` and calls `reorg_chain`, replacing the canonical chain.
6. `mainchain_height_to_header` and `mainchain_header_to_height` now map to the attacker's fake headers.
7. A downstream contract calls `verify_transaction_inclusion_v2` with a fake `tx_id` and a crafted Merkle proof against the attacker's fake `merkle_root` — the call returns `true`. [8](#0-7)

        &mut self,
        current_header: ExtendedHeader,
        prev_block_header: &ExtendedHeader,
    ) {
        // Main chain submission
        if prev_block_header.block_hash == self.mainchain_tip_blockhash {
            // Probably we should check if it is not in a mainchain?
            // chainwork > highScore
            log!("Block {}: saving to mainchain", current_header.block_hash);
            // Validate chain
            assert_eq!(
                self.mainchain_tip_blockhash,
                current_header.block_header.prev_block_hash
            );

            self.store_block_header(&current_header);
            self.mainchain_tip_blockhash = current_header.block_hash;
        } else {
            log!("Block {}: saving to fork", current_header.block_hash);
            // Fork submission
            let main_chain_tip_header = self
                .headers_pool
                .get(&self.mainchain_tip_blockhash)
                .unwrap_or_else(|| env::panic_str("tip should be in a header pool"));

            let last_main_chain_block_height = main_chain_tip_header.block_height;
            let total_main_chain_chainwork = main_chain_tip_header.chain_work;

            self.store_fork_header(&current_header);

            // Current chainwork is higher than on a current mainchain, let's promote the fork
            if current_header.chain_work > total_main_chain_chainwork {
                log!("Chain reorg");
                self.reorg_chain(current_header, last_main_chain_block_height);
            }
        }
    }
