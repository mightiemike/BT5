### Title
Bonus Reward Checkpoints Advance Before Pool Sufficiency Check, Causing Permanent Loss of Pre-Slash Bonus on Empty Pool — (`x/tieredrewards/keeper/slash.go`, `x/tieredrewards/keeper/claim_rewards.go`)

---

### Summary

`processEventsAndClaimBonus` mutates the position's `LastBonusAccrual`, `LastEventSeq`, and `LastKnownBonded` checkpoints — and decrements persistent event reference counts — **before** it checks whether the bonus pool has sufficient balance. When the pool is empty, `ErrInsufficientBonusPool` is returned with the position already mutated. `slashRedelegationPosition` silently swallows that error and then calls `setPositionWithState` with the mutated position, persisting the advanced checkpoints. All bonus accrued between the previous `LastBonusAccrual` and the slash block time is permanently unrecoverable.

---

### Finding Description

**Step 1 — Checkpoint advancement happens before pool check.**

Inside `processEventsAndClaimBonus`: [1](#0-0) 

The events loop advances `pos.LastEventSeq` (line 193) and decrements event reference counts in persistent store (line 196) for every event processed. [2](#0-1) 

After the loop, `applyBonusAccrualCheckpoint` advances `pos.LastBonusAccrual` to `blockTime` and `UpdateLastKnownBonded` updates the bonded state — both on the in-memory `pos` pointer.

Only **after** all of this does the pool sufficiency check run: [3](#0-2) 

If the pool is empty, `ErrInsufficientBonusPool` is returned with `pos` already carrying advanced checkpoints and with event ref counts already decremented in the store.

**Step 2 — The error is silently swallowed and the mutated position is persisted.** [4](#0-3) 

The error is caught and execution continues. The function then calls `setPositionWithState` with the already-mutated `pos`: [5](#0-4) 

`setPositionWithState` writes the position to the `Positions` collection: [6](#0-5) 

The persisted position now has `LastBonusAccrual = slashBlockTime` and `LastEventSeq` past the slash event, but the owner received zero bonus.

**Step 3 — The pre-slash bonus is permanently unrecoverable.**

Any subsequent call to `processEventsAndClaimBonus` (e.g., via `ClaimTierRewards`) reads `segmentStart = pos.LastBonusAccrual` and `pos.LastEventSeq`: [7](#0-6) 

Because both were advanced past the slash event, the pre-slash accrual window `[original_LastBonusAccrual, slashBlockTime]` is never revisited. Even if the pool is later refunded, the bonus for that window is gone.

---

### Impact Explanation

Position owners permanently lose all bonus rewards accrued between their last `LastBonusAccrual` checkpoint and the slash block time whenever a redelegation slash fires while the bonus pool is empty. The loss is proportional to the position's delegated shares, the tier's `BonusApy`, and the elapsed time since the last claim. There is no recovery path once the position is persisted with the advanced checkpoints.

---

### Likelihood Explanation

The bonus pool is a module account funded by governance. It can be empty at genesis before any funding transaction, or it can be depleted by normal reward claims over time. Redelegation slashes are triggered by evidence submission (`MsgSubmitEvidence`), which is permissionless in Cosmos SDK — any actor can submit valid double-sign evidence. The `BeforeRedelegationSlashed` hook is the production entry point: [8](#0-7) 

The two conditions (empty pool + redelegation slash) can co-occur naturally without any privileged action.

---

### Recommendation

Move the `sufficientBonusPoolBalance` check to **before** any checkpoint mutation or event ref-count decrement. If the pool is insufficient, return the error immediately without advancing `LastBonusAccrual`, `LastEventSeq`, or `LastKnownBonded`, and without calling `decrementEventRefCount`. This preserves the invariant that checkpoints only advance when the corresponding bonus has actually been paid.

Alternatively, if the "prevent chain halt" design intent must be preserved, the position's checkpoints must **not** be advanced when the bonus payment is skipped — the position should be left in its pre-call state so the accrual window remains claimable once the pool is refunded.

---

### Proof of Concept

```
1. Create a tier position and redelegate it to a second validator.
2. Advance block time by 30 days so bonus accrues.
3. Drain the RewardsPool module account to zero (or start with an unfunded pool).
4. Call Hooks().BeforeRedelegationSlashed(ctx, unbondingID, sharesToUnbond).
   → processEventsAndClaimBonus advances LastBonusAccrual to blockTime,
     advances LastEventSeq, decrements event ref counts, then returns
     ErrInsufficientBonusPool. The error is swallowed.
   → setPositionWithState persists the position with advanced checkpoints.
   → Owner balance is unchanged.
5. Refund the RewardsPool.
6. Call MsgClaimTierRewards for the position.
   → processEventsAndClaimBonus starts from the new LastBonusAccrual (= slashBlockTime).
   → The [original_LastBonusAccrual, slashBlockTime] window is never computed.
   → Owner receives zero bonus for the pre-slash period.
7. Assert owner balance delta == 0 for the pre-slash window. Test passes, confirming permanent loss.
```

The existing test `TestSlashRedelegationPosition_ClaimsBonusRewardsUpToSlash` in `slash_test.go` funds the pool before the slash and passes. The same test with an empty pool would demonstrate the loss. [9](#0-8)

```go
func (s *KeeperSuite) TestSlashRedelegationPosition_ClaimsBonusRewardsUpToSlash() {
	lockAmount := sdkmath.NewInt(sdk.DefaultPowerReduction.Int64())
	_, bondDenom := s.getStakingData()
	s.fundRewardsPool(sdkmath.NewInt(1_000_000_000), bondDenom)

	pos, _, unbondingID := s.setupRedelegatingPosition(lockAmount)
	owner := sdk.MustAccAddressFromBech32(pos.Owner)
	preAccrual := pos.LastBonusAccrual

	// Advance block time so bonus accrues on the destination validator.
	s.ctx = s.ctx.WithBlockHeight(s.ctx.BlockHeight() + 1)
	s.ctx = s.ctx.WithBlockTime(s.ctx.BlockTime().Add(30 * 24 * time.Hour))

	balBefore := s.app.BankKeeper.GetBalance(s.ctx, owner, bondDenom)

	// Partial slash — a small fraction of shares.
	sharesToUnbond := pos.Delegation.Shares.Quo(sdkmath.LegacyNewDec(10))
	err := s.keeper.Hooks().BeforeRedelegationSlashed(s.ctx, unbondingID, sharesToUnbond)
	s.Require().NoError(err)

	balAfter := s.app.BankKeeper.GetBalance(s.ctx, owner, bondDenom)
	s.Require().True(balAfter.Amount.GT(balBefore.Amount),
		"owner should have received bonus rewards accrued up to slash: before=%s after=%s",
		balBefore.Amount, balAfter.Amount)

	updated, err := s.keeper.GetPositionState(s.ctx, pos.Id)
	s.Require().NoError(err)
	s.Require().True(updated.LastBonusAccrual.After(preAccrual),
		"LastBonusAccrual should have advanced past the pre-slash checkpoint")
	s.Require().Equal(s.ctx.BlockTime(), updated.LastBonusAccrual,
		"LastBonusAccrual should advance to the slash block time")
	s.Require().True(updated.LastKnownBonded,
		"LastKnownBonded should remain true — destination validator is still bonded")
}
```
