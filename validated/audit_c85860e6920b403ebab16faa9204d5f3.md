### Title
Unpermissioned `run_mainchain_gc` with Attacker-Controlled `batch_size` Allows Any Caller to Bulk-Prune Canonical Chain State — (`File: contract/src/lib.rs`)

---

### Summary

`BtcLightClient::run_mainchain_gc` is a public, state-mutating function that carries **no caller-identity restriction** when the contract is unpaused, and accepts a fully attacker-controlled `batch_size` parameter. Any unprivileged NEAR account can call it with `batch_size = u64::MAX` to immediately remove the maximum possible number of canonical block headers in a single transaction, permanently destroying the contract's ability to verify transaction inclusion for those blocks.

---

### Finding Description

The function is declared as:

```rust
#[pause(except(roles(Role::UnrestrictedRunGC)))]
pub fn run_mainchain_gc(&mut self, batch_size: u64) {
``` [1](#0-0) 

The `#[pause]` decorator only restricts callers when the contract is in a **paused** state (where only `UnrestrictedRunGC` role holders may proceed). When the contract is **unpaused** — its normal operating state — the decorator is a no-op and the function is callable by any NEAR account with no role check whatsoever.

The `batch_size` parameter is entirely caller-supplied. Internally, `submit_blocks` calls `run_mainchain_gc` with `batch_size = num_of_headers` (the count of headers in the submitted batch, typically 1–100):

```rust
self.run_mainchain_gc(num_of_headers);
``` [2](#0-1) 

This design intent is gradual, incremental pruning. However, the public surface allows any caller to bypass this by passing `u64::MAX`, causing the GC to compute:

```rust
let total_amount_to_remove = amount_of_headers_we_store - self.gc_threshold;
let selected_amount_to_remove = std::cmp::min(total_amount_to_remove, batch_size);
``` [3](#0-2) 

With `batch_size = u64::MAX`, `selected_amount_to_remove` equals `total_amount_to_remove` — the entire removable surplus — and the loop permanently deletes all of those headers from `mainchain_height_to_header` and `headers_pool` in one call:

```rust
self.remove_block_header(blockhash);
self.mainchain_height_to_header.remove(&height);
``` [4](#0-3) 

It also advances `mainchain_initial_blockhash` to the new oldest stored block:

```rust
self.mainchain_initial_blockhash = self
    .mainchain_height_to_header
    .get(&end_removal_height)
    .unwrap_or_else(|| env::panic_str(ERR_KEY_NOT_EXIST));
``` [5](#0-4) 

These removals are **irreversible**. Once a header is gone from `headers_pool` and `mainchain_height_to_header`, no subsequent call can restore it.

---

### Impact Explanation

`verify_transaction_inclusion` and `verify_transaction_inclusion_v2` both require the target block's hash to be present in `headers_pool` and in `mainchain_header_to_height`:

```rust
let target_block_height = self
    .mainchain_header_to_height
    .get(&args.tx_block_blockhash)
    .unwrap_or_else(|| env::panic_str("block does not belong to the current main chain"));
``` [6](#0-5) 

After an attacker bulk-prunes with `run_mainchain_gc(u64::MAX)`, every block header that was removed will cause these verification calls to **panic and return false**, permanently breaking cross-chain proof verification for all transactions in those blocks. Downstream contracts or protocols relying on the light client's `verify_transaction_inclusion_v2` result will receive incorrect (failed) verification for transactions that were legitimately confirmed on Bitcoin.

The corrupted state values are:
- `headers_pool` — entries deleted for all pruned block hashes
- `mainchain_height_to_header` — entries deleted for all pruned heights
- `mainchain_initial_blockhash` — advanced to the post-prune boundary

---

### Likelihood Explanation

The entry path requires no special role, no staking, no private key, and no social engineering. Any NEAR account can call `run_mainchain_gc` directly on the deployed contract at any time the contract is not paused. The call is cheap (no deposit required). A motivated attacker — e.g., one who wants to invalidate a specific Bitcoin transaction proof — can execute this in a single transaction.

---

### Recommendation

Add an explicit role restriction to `run_mainchain_gc` so it is callable only by trusted roles (e.g., `Role::DAO`, `Role::RelayerManager`, or a new dedicated `GCManager` role) regardless of pause state. The `#[pause]` decorator alone is insufficient because it only gates access when the contract is paused, not when it is live.

```rust
// Example: restrict to DAO or a dedicated GC role even when unpaused
#[access_control_any(roles(Role::DAO, Role::RelayerManager))]
#[pause(except(roles(Role::UnrestrictedRunGC)))]
pub fn run_mainchain_gc(&mut self, batch_size: u64) { ... }
```

Alternatively, make `run_mainchain_gc` a private function and remove its public surface entirely, since it is already called internally by `submit_blocks`.

---

### Proof of Concept

1. Deploy the contract with `gc_threshold = 52704` and sync 100,000 headers (so `amount_of_headers_we_store = 100000`, `total_amount_to_remove = 47296`).
2. As an **unprivileged** NEAR account (no roles assigned), call:
   ```
   run_mainchain_gc(18446744073709551615)  // u64::MAX
   ```
3. The contract computes `selected_amount_to_remove = min(47296, u64::MAX) = 47296` and deletes all 47,296 oldest canonical headers in one transaction, advancing `mainchain_initial_blockhash` by 47,296 blocks.
4. Any subsequent call to `verify_transaction_inclusion_v2` for a transaction in any of those 47,296 pruned blocks will panic with `"block does not belong to the current main chain"`, permanently breaking proof verification for those blocks. [7](#0-6)

    #[pause(except(roles(Role::UnrestrictedRunGC)))]
    pub fn run_mainchain_gc(&mut self, batch_size: u64) {
        let initial_blockheader = self
            .headers_pool
            .get(&self.mainchain_initial_blockhash)
            .unwrap_or_else(|| env::panic_str(ERR_KEY_NOT_EXIST));

        let tip_blockheader = self
            .headers_pool
            .get(&self.mainchain_tip_blockhash)
            .unwrap_or_else(|| env::panic_str(ERR_KEY_NOT_EXIST));

        let amount_of_headers_we_store =
            tip_blockheader.block_height - initial_blockheader.block_height + 1;

        if amount_of_headers_we_store > self.gc_threshold {
            let total_amount_to_remove = amount_of_headers_we_store - self.gc_threshold;
            let selected_amount_to_remove = std::cmp::min(total_amount_to_remove, batch_size);

            let start_removal_height = initial_blockheader.block_height;
            let end_removal_height = initial_blockheader.block_height + selected_amount_to_remove;
            env::log_str(&format!(
                "Num of blocks to remove {selected_amount_to_remove}"
            ));

            for height in start_removal_height..end_removal_height {
                let blockhash = &self
                    .mainchain_height_to_header
                    .get(&height)
                    .unwrap_or_else(|| env::panic_str(ERR_KEY_NOT_EXIST));

                self.remove_block_header(blockhash);
                self.mainchain_height_to_header.remove(&height);
            }

            self.mainchain_initial_blockhash = self
                .mainchain_height_to_header
                .get(&end_removal_height)
                .unwrap_or_else(|| env::panic_str(ERR_KEY_NOT_EXIST));
        }
    }
```
