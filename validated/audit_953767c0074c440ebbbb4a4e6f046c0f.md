Audit Report

## Title
SwapAllowlistExtension Per-User Allowlist Bypassed via Router-Mediated Swaps — (File: `metric-periphery/contracts/extensions/SwapAllowlistExtension.sol`)

## Summary
`SwapAllowlistExtension.beforeSwap` checks `allowedSwapper[pool][sender]` where `sender` is the direct caller of `pool.swap()`. When swaps are routed through `MetricOmmSimpleRouter`, `sender` is the router address. Allowlisting the router — required for any allowlisted user to use the router — grants every unprivileged user the ability to bypass the per-user allowlist and swap against the curated pool, directly exposing LP funds to the adverse selection the allowlist was deployed to prevent.

## Finding Description
`SwapAllowlistExtension.beforeSwap` performs its access check against the `sender` argument: [1](#0-0) 

`MetricOmmPool.swap` passes `msg.sender` (the direct caller of `swap()`) as the `sender` argument to `_beforeSwap`: [2](#0-1) 

`MetricOmmSimpleRouter.exactInputSingle` calls `pool.swap()` directly, making the router the `msg.sender` from the pool's perspective: [3](#0-2) 

The resulting call chain is:

```
userB (not allowlisted) → router.exactInputSingle()
  → pool.swap()          [msg.sender = router]
      → _beforeSwap(sender = router, ...)
          → extension.beforeSwap(sender = router)
              → allowedSwapper[pool][router] == true → PASSES
```

The extension never observes the originating user. It only sees the router address. If the pool admin allowlists the router (necessary for any allowlisted user to use the router), the check degenerates into a single shared gate: any caller of the public router can swap against the curated pool. There is no configuration that simultaneously permits specific users to use the router while blocking others — the guard is structurally bypassed whenever the router is allowlisted.

## Impact Explanation
A pool deploying `SwapAllowlistExtension` restricts swap counterparties to prevent adverse selection against LP liquidity. Once the router is allowlisted, any unprivileged user can call `router.exactInputSingle` targeting the curated pool and execute swaps at the oracle-anchored price. LPs suffer direct principal loss from counterparties the allowlist was explicitly deployed to exclude. This matches the allowed impact category: direct loss of LP assets and broken core pool functionality (the allowlist guard).

## Likelihood Explanation
`MetricOmmSimpleRouter` is a public, permissionless periphery contract callable by any address. The only precondition — that the pool admin has allowlisted the router — is a natural and expected operational step for any pool that intends to support router-mediated swaps for its allowlisted users. No special privileges, flash loans, or complex setup are required. The bypass is repeatable by any address on every block.

## Recommendation
The `beforeSwap` hook must check the end-user identity, not the direct caller. Two concrete approaches:

1. **Router-forwarded identity via `extensionData`**: The router encodes `msg.sender` into `extensionData`; the extension decodes and validates it against the allowlist. This requires a protocol-level convention for trusted routers.
2. **Signed/attested user identity**: Extend the extension interface so that trusted routers attest the originating user address, and the extension validates that attested identity.

As an immediate safe measure, document that `SwapAllowlistExtension` is incompatible with router-mediated swaps and must only be used with direct `pool.swap()` calls until a forwarded-identity mechanism is implemented.

## Proof of Concept
1. Deploy a pool with `SwapAllowlistExtension` configured.
2. Pool admin calls `setAllowedToSwap(pool, userA, true)` and `setAllowedToSwap(pool, router, true)` so that `userA` can use the router.
3. `userB` (not allowlisted) calls `router.exactInputSingle({ pool: curatedPool, ... })`.
4. The router calls `pool.swap(...)` — `msg.sender` from the pool's perspective is the router.
5. `_beforeSwap(sender = router, ...)` is forwarded to the extension.
6. `allowedSwapper[pool][router] == true` → check passes.
7. `userB`'s swap executes against the curated pool's LP liquidity at the oracle price.
8. LPs suffer losses from an unauthorized counterparty the allowlist was designed to exclude.
