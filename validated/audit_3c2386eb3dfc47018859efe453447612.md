### Title
Locked NLP Balance Inflates Health Score, Allowing Over-Leveraging Against Illiquid Collateral — (`core/contracts/SpotEngine.sol` / `core/contracts/Clearinghouse.sol`)

---

### Summary

The health calculation in `Clearinghouse.getHealth` uses the **total** NLP balance (locked + unlocked) as collateral, but `burnNlp` enforces that only the **unlocked** portion can be redeemed. This is a direct structural analog to the Unitas portfolio-in-reserve-ratio bug: an accounting metric includes assets that are not actually available for the user to act on, overstating solvency.

---

### Finding Description

When a user mints NLP via `Clearinghouse.mintNlp`, two parallel accounting paths are updated:

**Path 1 — Lock queue** (`nlpLockedBalanceQueues`): the minted NLP is enqueued with `unlockedAt = getOracleTime() + NLP_LOCK_PERIOD` (4 days). This is the gate used by `burnNlp`.

**Path 2 — Regular balance** (`balances[NLP_PRODUCT_ID][subaccount]`): the same amount is also written into the standard spot balance via `_updateBalanceNormalized`. [1](#0-0) 

The health engine reads exclusively from Path 2: [2](#0-1) 

`_getBalance` → `getBalance` → `balanceNormalizedToBalance(state, balances[NLP_PRODUCT_ID][subaccount])` — this is the **total** NLP balance, with no distinction between locked and unlocked.

`burnNlp`, however, gates redemption on the **unlocked** portion only: [3](#0-2) 

The lock period constant is 4 days: [4](#0-3) 

So for up to 4 days after minting, a user's health score includes NLP collateral that they **cannot burn** to repay debt or improve their position.

---

### Impact Explanation

A user can mint NLP (all of it locked), receive full health credit for it, and immediately borrow against it. During the 4-day lock window, if NLP price declines, the user **cannot** burn locked NLP to repay debt or restore health — the only exit is forced liquidation. The health check at borrow time overstated the user's actual redeemable collateral, allowing them to take on more leverage than the unlocked liquidity supports.

This mirrors the Unitas invariant break exactly: the reserve ratio (health) includes assets (portfolio / locked NLP) that are not available for the redemption path (USDT withdrawal / NLP burn).

---

### Likelihood Explanation

Any user can trigger this by minting NLP and immediately borrowing against the full balance. The 4-day lock window is a fixed, predictable constraint. No special permissions or admin access are required. The entry path is `Endpoint → Clearinghouse.mintNlp` followed by any borrow-creating action (e.g., `withdrawCollateral` of quote).

---

### Recommendation

The health contribution of NLP for a given subaccount should be capped at the **unlocked** NLP balance, not the total balance. Concretely, `_calculateProductHealth` (or a NLP-specific override) should call `getNlpUnlockedBalance(subaccount)` instead of `_getBalance(NLP_PRODUCT_ID, subaccount)` when computing the NLP health contribution.

```diff
// In BaseEngine._calculateProductHealth (or a SpotEngine override for NLP_PRODUCT_ID):
- (int128 amount, int128 quoteAmount) = _getBalance(productId, subaccount);
+ int128 amount = (productId == NLP_PRODUCT_ID)
+     ? getNlpUnlockedBalance(subaccount).amount
+     : _getBalance(productId, subaccount);
```

---

### Proof of Concept

1. User calls `mintNlp` with `quoteAmount = 10_000e18`. They receive `nlpAmount` NLP tokens, all locked for 4 days. Their `balances[NLP_PRODUCT_ID][user]` = `nlpAmount`; `nlpLockedBalanceQueues[user].unlockedBalanceSum.amount` = 0.

2. `getHealth(user, INITIAL)` is called. `_getBalance(NLP_PRODUCT_ID, user)` returns `nlpAmount` (full locked balance). Health = `nlpAmount * priceX18 * longWeight - 0` > 0. Health check passes.

3. User calls `withdrawCollateral` for 8,000 USDC. `getHealth` again uses full locked NLP balance. Health still positive. Withdrawal succeeds.

4. NLP price drops 15% over the next 2 days (still within lock window). User's maintenance health goes negative.

5. User attempts `burnNlp`. `getNlpUnlockedBalance(user).amount == 0` → `ERR_UNLOCKED_NLP_INSUFFICIENT`. User cannot self-rescue.

6. User is liquidated. The health check at step 2–3 included 10,000 USDC worth of collateral that was never burnable during the risk window. [5](#0-4) [6](#0-5) [7](#0-6)

    function handleNlpLockedBalance(bytes32 subaccount, int128 amountDelta)
        internal
    {
        _assertInternal();

        // N_ACCOUNT is not limited by lock period
        if (subaccount == N_ACCOUNT) return;

        tryUnlockNlpBalance(subaccount);
        if (amountDelta > 0) {
            NlpLockedBalanceQueue storage queue = nlpLockedBalanceQueues[
                subaccount
            ];
            if (
                queue.balanceCount > 0 &&
                queue.balances[queue.balanceCount - 1].unlockedAt ==
                getOracleTime() + NLP_LOCK_PERIOD
            ) {
                queue
                    .balances[queue.balanceCount - 1]
                    .balance
                    .amount += amountDelta;
            } else {
                queue.balances[queue.balanceCount] = NlpLockedBalance({
                    balance: Balance({amount: amountDelta}),
                    unlockedAt: getOracleTime() + NLP_LOCK_PERIOD
                });
                queue.balanceCount++;
            }
        } else if (amountDelta < 0) {
            Balance memory balanceSum = nlpLockedBalanceQueues[subaccount]
                .unlockedBalanceSum;
            balanceSum.amount += amountDelta;
            nlpLockedBalanceQueues[subaccount].unlockedBalanceSum = balanceSum;
        }
    }

    function mintNlp(
        IEndpoint.MintNlp calldata txn,
        int128 oraclePriceX18,
        IEndpoint.NlpPool[] calldata nlpPools,
        int128[] calldata nlpPoolRebalanceX18
    ) external onlyEndpoint {
        require(!RiskHelper.isIsolatedSubaccount(txn.sender), ERR_UNAUTHORIZED);

        ISpotEngine spotEngine = _spotEngine();
        spotEngine.updatePrice(NLP_PRODUCT_ID, oraclePriceX18);

        require(txn.quoteAmount <= INT128_MAX, ERR_CONVERSION_OVERFLOW);
        int128 quoteAmount = int128(txn.quoteAmount);
        int128 nlpAmount = quoteAmount.div(oraclePriceX18);

        _validateNlpRebalance(nlpPools, nlpPoolRebalanceX18, quoteAmount);
        for (uint128 i = 0; i < nlpPoolRebalanceX18.length; i++) {
            require(nlpPoolRebalanceX18[i] >= 0, ERR_INVALID_NLP_REBALANCE);
        }

        spotEngine.updateBalance(NLP_PRODUCT_ID, txn.sender, nlpAmount);
        spotEngine.updateBalance(NLP_PRODUCT_ID, N_ACCOUNT, -nlpAmount);

        spotEngine.updateBalance(QUOTE_PRODUCT_ID, txn.sender, -quoteAmount);
        _applyNlpRebalance(spotEngine, nlpPools, nlpPoolRebalanceX18);

        require(
            getHealth(txn.sender, IProductEngine.HealthType.INITIAL) >= 0,
            ERR_SUBACCT_HEALTH
        );
    }

    function burnNlp(
        IEndpoint.BurnNlp calldata txn,
        int128 oraclePriceX18,
        IEndpoint.NlpPool[] calldata nlpPools,
        int128[] calldata nlpPoolRebalanceX18
    ) external onlyEndpoint {
        require(!RiskHelper.isIsolatedSubaccount(txn.sender), ERR_UNAUTHORIZED);

        ISpotEngine spotEngine = _spotEngine();
        spotEngine.updatePrice(NLP_PRODUCT_ID, oraclePriceX18);

        require(txn.nlpAmount <= INT128_MAX, ERR_CONVERSION_OVERFLOW);
        int128 nlpAmount = int128(txn.nlpAmount);
        require(
            spotEngine.getNlpUnlockedBalance(txn.sender).amount >= nlpAmount,
            ERR_UNLOCKED_NLP_INSUFFICIENT
        );
        int128 quoteAmount = nlpAmount.mul(oraclePriceX18);
        int128 burnFee = MathHelper.max(ONE, quoteAmount / 1000);
        quoteAmount = MathHelper.max(0, quoteAmount - burnFee);

        _validateNlpRebalance(nlpPools, nlpPoolRebalanceX18, -quoteAmount);
        for (uint128 i = 0; i < nlpPoolRebalanceX18.length; i++) {
            require(nlpPoolRebalanceX18[i] <= 0, ERR_INVALID_NLP_REBALANCE);
        }

        spotEngine.updateBalance(NLP_PRODUCT_ID, txn.sender, -nlpAmount);
        spotEngine.updateBalance(NLP_PRODUCT_ID, N_ACCOUNT, nlpAmount);

        if (quoteAmount > 0) {
            spotEngine.updateBalance(QUOTE_PRODUCT_ID, txn.sender, quoteAmount);
            _applyNlpRebalance(spotEngine, nlpPools, nlpPoolRebalanceX18);
        }

        require(
            spotEngine.getBalance(NLP_PRODUCT_ID, txn.sender).amount >= 0,
            ERR_SUBACCT_HEALTH
        );
        // Burning NLP can decrease health if the burn fee exceeds the health improvement
        // from the withdrawal. This check prevents malicious actors from deliberately
        // creating unhealthy subaccounts through NLP burns.
        require(
            getHealth(txn.sender, IProductEngine.HealthType.MAINTENANCE) >= 0,
            ERR_SUBACCT_HEALTH
        );
    }
