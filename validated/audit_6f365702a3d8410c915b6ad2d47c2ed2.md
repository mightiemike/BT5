Audit Report

## Title
Swap allowlist checks router address instead of actual user, enabling full bypass via `MetricOmmSimpleRouter` - (File: metric-periphery/contracts/extensions/SwapAllowlistExtension.sol)

## Summary
`MetricOmmPool.swap()` passes `msg.sender` (the direct caller) as `sender` to `_beforeSwap`, which forwards it to `SwapAllowlistExtension.beforeSwap()`. When a user swaps through `MetricOmmSimpleRouter`, the router is `msg.sender` to the pool, so the allowlist checks the router's address rather than the actual user's address. If the router is allowlisted for a pool, any unprivileged user can bypass the swap allowlist entirely by routing through it.

## Finding Description

**Root cause — `MetricOmmPool.swap()` binds `msg.sender` as `sender`:**

```solidity
// metric-core/contracts/MetricOmmPool.sol:230-240
_beforeSwap(
  msg.sender,   // <-- direct caller of pool.swap(), not the end user
  recipient,
  ...
);
```

**`SwapAllowlistExtension.beforeSwap()` gates on that `sender`:**

```solidity
// metric-periphery/contracts/extensions/SwapAllowlistExtension.sol:37
if (!allowAllSwappers[msg.sender] && !allowedSwapper[msg.sender][sender]) {
  revert IMetricOmmPoolActions.NotAllowedToSwap();
}
```

Here `msg.sender` is the pool (the extension's caller) and `sender` is the first argument — the direct caller of `pool.swap()`.

**Router-mediated call path:**

```
User (not allowlisted)
  → MetricOmmSimpleRouter.exactInput*(...)
    → pool.swap(recipient, ...) [msg.sender = router]
      → _beforeSwap(router_address, ...)
        → SwapAllowlistExtension.beforeSwap(router_address, ...)
          → allowedSwapper[pool][router_address]  ← checks router, not user
```

If the pool admin allowlists the router address (a natural operational choice to let users interact via the standard periphery), `allowedSwapper[pool][router]` is `true`, and the check passes for **every** user regardless of their individual allowlist status.

Existing guards are insufficient: the `onlyPool` modifier in `BaseMetricExtension` only verifies the extension is called by a registered pool — it does not verify the identity of the original swap initiator.

## Impact Explanation
The swap allowlist is the sole on-chain gate controlling which addresses may trade in a restricted pool. When bypassed via the router, any unprivileged address can execute swaps in a pool that was configured to be permissioned. This breaks core pool access-control functionality and constitutes a broken allowlist invariant with direct fund-flow impact (trades that should be rejected are settled, fees are collected from unauthorized parties, and pool state changes in ways the admin did not authorize).

## Likelihood Explanation
The bypass requires only that the router address is allowlisted for the target pool — a realistic and expected operational configuration since the router is the standard user-facing entry point. Any unprivileged user can then call `MetricOmmSimpleRouter.exactInputSingle` or `exactInput` targeting the restricted pool. No special privileges, flash loans, or complex setup are needed. The attack is repeatable every block.

## Recommendation
Pass the original user identity through the call chain. One approach: have the router encode the original `msg.sender` in `extensionData` and have the extension decode and check it. A cleaner approach is to add an `originator` parameter to the pool's `swap()` interface (similar to how `addLiquidity` separates `msg.sender` payer from `owner`), so the pool passes the true initiator to extension hooks. The allowlist should gate on the address that economically initiates the swap, not the intermediate contract.

## Proof of Concept

1. Deploy a pool with `SwapAllowlistExtension` configured as a before-swap hook.
2. Call `swapExtension.setAllowedToSwap(pool, routerAddress, true)` — allowlist the router (simulating a pool admin who wants router users to be able to swap).
3. As an address that is **not** individually allowlisted, call `MetricOmmSimpleRouter.exactInputSingle(...)` targeting the pool.
4. Observe the swap succeeds — `SwapAllowlistExtension.beforeSwap` receives `sender = routerAddress`, which is allowlisted, so the check passes.
5. Confirm that calling `pool.swap(...)` directly from the same non-allowlisted address reverts with `NotAllowedToSwap`, proving the bypass is router-specific.
