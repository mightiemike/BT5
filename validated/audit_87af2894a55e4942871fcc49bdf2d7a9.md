### Title
ML-DSA-65 Gas-Key Host-Function Fee Uses Wire Length (1953 B) Instead of Trie-ID Length (33 B), Causing ~59× Overcharge vs. Transaction Path — (`runtime/near-vm-runner/src/logic/logic.rs`)

---

### Summary

The host-function path for `TransferToGasKey`, `WithdrawFromGasKey`, and `AddKey` (gas-key) actions passes the raw borsh wire length of the caller-supplied public key (`public_key_len as usize`) to `gas_key_transfer_send_fee`, `gas_key_transfer_exec_fee`, and `gas_key_add_key_exec_fee`. The transaction path for the same actions correctly passes `public_key.trie_id_len()`. For ML-DSA-65 keys these two values diverge by a factor of ~59 (1953 wire bytes vs. 33 trie bytes), producing a systematic ~59× overcharge on every gas-key operation invoked through a WASM host function with a post-quantum key.

---

The same divergence appears in the wasmtime path at `runtime/near-vm-runner/src/wasmtime_runner/logic.rs` lines 3323, 3325, 3398, and 3502.

**Why the values differ for ML-DSA-65:**

`PublicKey::len()` returns the borsh wire size: `1 + ML_DSA_65_PUBLIC_KEY_LENGTH = 1953`.
`PublicKey::trie_id_len()` returns the on-trie SHA3-256 hash size: `1 + ML_DSA_65_HASH_LENGTH = 33`.

```rust
// core/crypto/src/signature.rs line 268-273
pub fn len(&self) -> usize {
    match self {
        Self::MLDSA65(_) => 1 + ML_DSA_65_PUBLIC_KEY_LENGTH,  // 1953
    }
}
// line 333-338
pub fn trie_id_len(&self) -> usize {
    match self {
        Self::MLDSA65(_) => 1 + ML_DSA_65_HASH_LENGTH,        // 33
    }
}
```

The fee helper `gas_key_transfer_exec_fee` is documented as covering "what the receiver needs to read/write in the trie" — the trie-id length (33 bytes). Passing the wire length (1953 bytes) inflates the per-byte multiplier by 1953/33 ≈ 59.2×.

For `gas_key_add_key_exec_fee`, the nonce trie key length is computed as `access_key_key_len(account_id_len, public_key_len) + size_of::<NonceIndex>()`. Substituting 1953 instead of 33 inflates every nonce key length by 1920 bytes, multiplied by `num_nonces`.

---

### Impact Explanation

With the testnet `action_gas_key_byte` rate of `59_357_464` gas/byte:

| Path | Bytes used | Per-byte gas (send+exec) |
|------|-----------|--------------------------|
| Transaction (`trie_id_len`) | 33 | ~1.96 Ggas |
| Host function (`wire len`) | 1953 | ~115.9 Ggas |

A WASM contract calling `promise_batch_action_transfer_to_gas_key` or `promise_batch_action_add_gas_key_with_full_access` with an ML-DSA-65 key is charged ~59× more gas than the equivalent transaction-level action. This:

1. Makes ML-DSA-65 gas-key operations via host functions prohibitively expensive relative to the transaction path, breaking fee-parity invariants.
2. Can cause contracts that budget gas based on the transaction-path cost to run out of gas and fail.
3. Creates a protocol-level inconsistency: the same semantic action has wildly different gas costs depending on whether it is submitted as a transaction or dispatched from a contract.

---

### Likelihood Explanation

The bug is reachable by any unprivileged WASM contract that holds or receives an ML-DSA-65 public key and calls the affected host functions. The contract controls `public_key_len` (the borsh wire length it passes), and for ML-DSA-65 that value is always 1953. The `PostQuantumSignatures` protocol feature gates ML-DSA-65 key acceptance, so the bug activates at the protocol version that enables that feature. No privileged role is required.

---

### Recommendation

Replace `public_key_len as usize` with `public_key.decode()?.trie_id_len()` (after the key is decoded and validated) in all three host-function call sites:

- `promise_batch_action_transfer_to_gas_key` (both `logic.rs` and `wasmtime_runner/logic.rs`)
- `promise_batch_action_add_gas_key_with_full_access`
- `promise_batch_action_add_gas_key_with_function_call`

The decoded key is already available at those call sites (as `public_key.decode()?`), so no additional parsing is needed. This aligns the host-function fee with the transaction-path fee and with the documented intent of the exec-fee helper ("what the receiver needs to read/write in the trie").

```rust
pub fn gas_key_transfer_send_fee(
    cfg: &RuntimeFeesConfig,
    sender_is_receiver: bool,
    public_key_len: usize,
) -> GasKeyTransferFee {
    let base = cfg.fee(ActionCosts::gas_key_transfer_base).send_fee(sender_is_receiver);
    let per_byte = cfg
        .fee(ActionCosts::gas_key_byte)
        .send_fee(sender_is_receiver)
        .checked_mul(public_key_len as u64)
        .unwrap();
    GasKeyTransferFee { base, per_byte }
}

/// Exec fee for TransferToGasKey / WithdrawFromGasKey actions.
/// Based on the access key trie key length + estimated value length (what the
/// receiver needs to read/write in the trie).
pub fn gas_key_transfer_exec_fee(
    cfg: &RuntimeFeesConfig,
    account_id_len: usize,
    public_key_len: usize,
) -> GasKeyTransferFee {
    let base = cfg.fee(ActionCosts::gas_key_transfer_base).exec_fee();
    let trie_key_len = access_key_key_len(account_id_len, public_key_len);
    let estimated_value_len = AccessKey::min_gas_key_borsh_len();
    let per_byte = cfg
        .fee(ActionCosts::gas_key_byte)
        .exec_fee()
        .checked_mul((trie_key_len + estimated_value_len) as u64)
        .unwrap();
    GasKeyTransferFee { base, per_byte }
}
```

pub fn gas_key_add_key_exec_fee(
    cfg: &RuntimeFeesConfig,
    account_id_len: usize,
    public_key_len: usize,
    num_nonces: NonceIndex,
) -> GasKeyAddFee {
    let num_nonces = num_nonces as u64;
    let base =
        cfg.fee(ActionCosts::gas_key_nonce_write_base).exec_fee().checked_mul(num_nonces).unwrap();
    let nonce_key_len =

        .unwrap()
        .checked_mul(num_nonces)
        .unwrap();
