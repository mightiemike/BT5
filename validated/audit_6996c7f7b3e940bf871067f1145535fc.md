### Title
`SwapAllowlistExtension` gates the router's address instead of the actual user, allowing any unprivileged user to bypass the per-pool swap allowlist via `MetricOmmSimpleRouter` — (`metric-periphery/contracts/extensions/SwapAllowlistExtension.sol`)

---

### Summary

`SwapAllowlistExtension.beforeSwap` checks `allowedSwapper[pool][sender]`, where `sender` is the `msg.sender` of the pool's `swap()` call. When a user routes through `MetricOmmSimpleRouter`, the pool's `msg.sender` is the router contract, not the end user. If the pool admin allowlists the router (the natural action to enable router-mediated swaps for permitted users), every unprivileged user can bypass the allowlist by calling the router, because the extension only sees the router's address and approves it.

---

### Finding Description

**Call chain:**

```
User → MetricOmmSimpleRouter.exactInputSingle()
     → pool.swap(recipient, ..., extensionData)   [msg.sender = router]
     → _beforeSwap(msg.sender=router, recipient, ...)
     → ExtensionCalling._callExtensionsInOrder(BEFORE_SWAP_ORDER, ...)
     → SwapAllowlistExtension.beforeSwap(sender=router, ...)
```

In `MetricOmmPool.swap`, the pool passes `msg.sender` (the router) as the `sender` argument to `_beforeSwap`: [1](#0-0) 

`ExtensionCalling._beforeSwap` forwards that value unchanged as the first argument to every configured extension: [2](#0-1) 

`SwapAllowlistExtension.beforeSwap` then checks `allowedSwapper[msg.sender][sender]`, where `msg.sender` is the pool and `sender` is the router: [3](#0-2) 

The actual end-user's address is never inspected. The router itself is the identity the allowlist evaluates.

**Bypass scenario:**

A pool admin deploys a pool with `SwapAllowlistExtension` to restrict swaps to a whitelist of counterparties. To let those counterparties use the router, the admin calls `setAllowedToSwap(pool, router, true)`. From that moment, any address — including completely unpermissioned users — can call `MetricOmmSimpleRouter.exactInputSingle` (or `exactInput`, `exactOutputSingle`, `exactOutput`) against the restricted pool and the extension will approve the call, because it only checks whether the router is allowlisted.

**Alternate broken scenario (DoS):**

If the admin does *not* allowlist the router, then individually allowlisted users cannot use the router at all, because the extension sees the router's address and rejects it. The allowlist becomes incompatible with the router for any user.

---

### Impact Explanation

The `SwapAllowlistExtension` is the sole on-chain mechanism for restricting who may trade against a pool's LP liquidity. A complete bypass means any unpermissioned user can execute swaps against a pool whose LP depositors expected only trusted counterparties. LP principal is directly at risk: the pool's oracle-driven bid/ask spread is calibrated for a known set of traders; adversarial flow from unpermissioned users can extract value from LP bins through informed or manipulative trading that the allowlist was designed to prevent.

---

### Likelihood Explanation

The trigger is fully unprivileged: any EOA or contract can call `MetricOmmSimpleRouter`. The only precondition is that the pool admin has allowlisted the router — a natural and expected configuration step for any pool that intends to support router-mediated swaps for its permitted users. No special timing, oracle manipulation, or admin collusion is required.

---

### Recommendation

Pass the original end-user's address through the swap path so the extension can gate the correct identity. One approach: add a `payer` or `originator` field to the extension data that the router populates with `msg.sender` before forwarding to the pool, and have `SwapAllowlistExtension` decode and check that field. A cleaner alternative is for the pool to accept an explicit `originator` parameter in `swap()` (defaulting to `msg.sender` for direct calls) and forward it as the `sender` argument to extensions, so the allowlist always sees the economic actor rather than the intermediary.

---

### Proof of Concept

```solidity
// Pool admin sets up a restricted pool
SwapAllowlistExtension ext = new SwapAllowlistExtension(factory);
// Deploy pool with ext as beforeSwap extension
address pool = factory.createPool(..., ext, ...);

// Admin allowlists the router so permitted users can swap through it
ext.setAllowedToSwap(pool, address(router), true);
// Admin does NOT allowlist attacker
// ext.setAllowedToSwap(pool, attacker, false); // never called

// Attacker (not on allowlist) bypasses the gate via the router
vm.prank(attacker);
router.exactInputSingle(
    IMetricOmmSimpleRouter.ExactInputSingleParams({
        pool: pool,
        recipient: attacker,
        tokenIn: token0,
        zeroForOne: true,
        amountIn: 1_000e18,
        amountOutMinimum: 0,
        priceLimitX64: 0,
        deadline: block.timestamp + 1,
        extensionData: ""
    })
);
// Swap succeeds: extension saw allowedSwapper[pool][router] == true
// Attacker extracted output tokens from LP bins without being on the allowlist
```
