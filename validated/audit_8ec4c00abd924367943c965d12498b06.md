### Title
ML-DSA-65 Gas-Key Host-Function Fee Uses Wire-Format Length Instead of Trie-ID Length, Causing ~1920-Byte-Per-Nonce Exec-Fee Overcharge — (`core/parameters/src/cost.rs`)

### Summary

`gas_key_add_key_exec_fee` and `gas_key_transfer_exec_fee` receive `public_key_len` as the raw wire/borsh byte count of the public key. For ML-DSA-65 keys that value is **1953** (1 tag + 1952 key bytes). The functions feed it directly into `access_key_key_len(account_id_len, public_key_len)` to price the trie write. But the actual trie key for an ML-DSA-65 access-key entry is the SHA3-256 hash form — **33 bytes** (tag 3 + 32-byte digest). The result is a per-nonce exec-fee overcharge of **1920 × `gas_key_byte`** whenever a contract adds or funds an ML-DSA-65 gas key through the host-function interface.

### Finding Description

`PublicKey::trie_id_len()` was introduced precisely because ML-DSA-65 wire length (1953) and trie-id length (33) diverge. The runtime storage-stake path (`access_key_storage_usage`, `gas_key_storage_cost`) was updated to call `trie_id_len()`. The host-function fee helpers were not.

**Divergent call chain — add-key path:**

```
promise_batch_action_add_gas_key_with_full_access(…, public_key_len=1953, …)
  └─ gas_key_add_key_exec_fee(cfg, account_id_len, public_key_len=1953, num_nonces)
       └─ nonce_key_len = access_key_key_len(account_id_len, 1953) + sizeof(NonceIndex)
          // actual trie key uses access_key_key_len(account_id_len, 33) + sizeof(NonceIndex)

The `trie_id_len` / `len` divergence for ML-DSA-65 is documented as a known contract: [6](#0-5)

pub fn trie_id_len(&self) -> usize { … Self::MLDSA65(_) => 1 + ML_DSA_65_HASH_LENGTH, … }

### Impact Explanation

**High.** The overcharge per nonce is `1920 × gas_key_byte` exec gas. With `num_nonces` up to `u16::MAX` (65 535) the total overcharge can reach `1920 × gas_key_byte × 65535`. Even at modest `gas_key_byte` rates this exceeds the per-receipt gas limit for any non-trivial nonce count, making it **impossible for a contract to add an ML-DSA-65 gas key with more than a small number of nonces via the host-function interface**. The same overcharge applies to `TransferToGasKey` / `WithdrawFromGasKey` host-function calls targeting an ML-DSA-65 key. Core gas-key functionality is broken for the post-quantum key type at the host-function boundary, while the transaction path works correctly.

### Likelihood Explanation

`PostQuantumSignatures` and `GasKeys` both activate at protocol version 85 (stable). Any contract deployed after that point can call `promise_batch_action_add_gas_key_with_full_access` with an ML-DSA-65 public key. No privileged role is required; any unprivileged account can deploy such a contract. The trigger is the combination of two already-stabilized features.

### Recommendation

In `gas_key_add_key_exec_fee` and `gas_key_transfer_exec_fee`, replace the raw `public_key_len` parameter (wire length) with the trie-id length. The cleanest fix is to decode the public key before computing the fee and call `public_key.trie_id_len()`, or to add a separate `public_key_trie_id_len: usize` parameter that callers populate from `public_key.trie_id_len()` after decoding. The host-function sites already decode the key (`self.get_public_key(…)`) before calling the fee helper, so the decoded key is available.

Add a fee-parity test that uses `KeyType::MLDSA65` to catch future regressions.

### Proof of Concept

1. After protocol version 85 activates, deploy a contract that calls:
   ```
   promise_batch_action_add_gas_key_with_full_access(
       promise_idx,
       public_key_len = 1953,   // borsh-encoded ML-DSA-65 pubkey
       public_key_ptr,
       num_nonces = 500,
   )
   ```
2. The host function charges exec gas using `gas_key_add_key_exec_fee(…, 1953, 500)`.
   - `nonce_key_len = access_key_key_len(account_id_len, 1953) + 2`
   - Actual trie nonce key length = `access_key_key_len(account_id_len, 33) + 2`
   - Overcharge = `(1953 − 33) × gas_key_byte × 500 = 960 000 × gas_key_byte` exec gas
3. The same operation submitted as a signed `AddKey` transaction charges exec gas using `trie_id_len()` = 33, producing a far lower fee.
4. The divergence confirms the domain mismatch: wire-format length (1953) is used where trie-id length (33) is required. [2](#0-1) [1](#0-0) [8](#0-7) [5](#0-4)

    /// Length, in bytes, of the on-trie identifier for an access-key
    /// entry owned by this public key. For ed25519/secp256k1 this matches
    /// `len()`; for ML-DSA-65 the trie stores a SHA3-256 hash (33 bytes
    /// including the type tag), not the 1953-byte borsh-encoded pubkey.
    /// Used by storage-fee calculations on the runtime side; cheap to call
    /// (no hashing) - for ML-DSA-65 this returns the size of the digest
    /// form without actually hashing the pubkey.
    pub fn trie_id_len(&self) -> usize {
        match self {
            Self::ED25519(_) => 1 + ed25519_dalek::PUBLIC_KEY_LENGTH,
            Self::SECP256K1(_) => 1 + 64,
            Self::MLDSA65(_) => 1 + ML_DSA_65_HASH_LENGTH,
        }
    }
