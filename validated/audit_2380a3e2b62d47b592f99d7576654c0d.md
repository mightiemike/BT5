Audit Report

## Title
SwapAllowlistExtension Checks Router Address Instead of Actual User, Allowlist Fully Bypassed via MetricOmmSimpleRouter - (File: metric-periphery/contracts/extensions/SwapAllowlistExtension.sol)

## Summary
`SwapAllowlistExtension.beforeSwap` gates swaps by checking the `sender` argument, which `MetricOmmPool.swap` always sets to its own `msg.sender`. When `MetricOmmSimpleRouter` calls `pool.swap(...)`, the pool's `msg.sender` is the router contract, so the extension evaluates `allowedSwapper[pool][router]` instead of `allowedSwapper[pool][actualUser]`. Any pool admin who allowlists the router to enable router-based trading inadvertently opens the allowlist to every user on the network.

## Finding Description
`MetricOmmPool.swap()` calls `_beforeSwap(msg.sender, ...)`, hardcoding its own `msg.sender` as the `sender` argument forwarded to every configured extension: [1](#0-0) 

`ExtensionCalling._beforeSwap` passes that `sender` value unchanged into the ABI-encoded extension call: [2](#0-1) 

`SwapAllowlistExtension.beforeSwap` receives `sender` and checks it against the per-pool allowlist, where `msg.sender` inside the extension is the pool: [3](#0-2) 

`MetricOmmSimpleRouter.exactInputSingle` (and `exactInput`, `exactOutputSingle`, `exactOutput`) calls `pool.swap(recipient, ...)` directly with no mechanism to propagate the original EOA: [4](#0-3) 

The `swap` interface has no `sender` parameter, so there is no in-protocol path to supply the originating user address: [5](#0-4) 

When the router calls the pool, the extension evaluates `allowedSwapper[pool][router]`, not `allowedSwapper[pool][actualUser]`. If the router is allowlisted (the natural operational state for any pool supporting router-mediated trading), the check passes for every caller of the router regardless of whether they are individually allowlisted.

## Impact Explanation
A pool configured with `SwapAllowlistExtension` is a curated pool where only explicitly approved addresses may trade. Allowlisting `MetricOmmSimpleRouter` — the necessary step to enable router-based multi-hop swaps — renders the allowlist entirely meaningless. Every user on the network can call `router.exactInput(...)` or `router.exactOutput(...)` targeting that pool and the extension will pass because it sees `sender = router`, which is allowlisted. This constitutes broken core pool functionality: the curated-pool invariant is violated, unauthorized users can trade against LP liquidity, and any KYC, compliance, or access-control intent the pool admin configured is silently bypassed.

## Likelihood Explanation
The trigger requires no special privilege. Any user can call the public router functions. The precondition — that the router is allowlisted — is the natural operational state for any pool that wants to support router-mediated swaps. Pool admins have no documented warning that allowlisting the router opens the gate to all users. The bypass is therefore reachable on every production pool that combines `SwapAllowlistExtension` with router support.

## Recommendation
The pool should forward the original caller's identity through the swap interface, or the router should pass the originating user address as part of `extensionData` and the extension should decode and verify it. The cleanest fix is to add an explicit `originator` field to `pool.swap()` that the router populates with `msg.sender` before calling the pool, and to have the pool validate that only trusted periphery contracts may supply a non-`msg.sender` value for that field. Alternatively, the extension can be redesigned to decode the true originator from `extensionData`, with the router committing to an authenticated user identity in that field.

## Proof of Concept
1. Deploy a pool with `SwapAllowlistExtension` as the `beforeSwap` hook.
2. Pool admin calls `setAllowedToSwap(pool, router, true)` to enable router-based trading.
3. Attacker (address not in the allowlist) calls `MetricOmmSimpleRouter.exactInputSingle(...)` targeting the pool.
4. The router calls `pool.swap(recipient, ...)` — pool sees `msg.sender = router`.
5. Pool calls `extension.beforeSwap(router, ...)` — extension checks `allowedSwapper[pool][router]` → `true`.
6. Swap executes. Attacker successfully trades on a pool they were never authorized to access.
7. Repeat until LP position is drained or arbitrage profit is exhausted.
