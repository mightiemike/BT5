### Title
`ContractOwner` Lacks `receive()` Function, Permanently Locking Native ETH in `DirectDepositV1` — (File: `core/contracts/ContractOwner.sol`)

---

### Summary

`ContractOwner` is the owner of every `DirectDepositV1` (DDA) contract it deploys. It exposes `withdrawFromDirectDepositV1` as the sole mechanism to recover native ETH from a DDA. That function calls `DirectDepositV1.withdrawNative()`, which pushes ETH back to `msg.sender` (i.e., `ContractOwner`). Because `ContractOwner` declares no `receive()` or `fallback()` function, the ETH transfer always reverts, making native ETH in any DDA permanently irrecoverable.

---

### Finding Description

`ContractOwner.createDirectDepositV1` deploys each `DirectDepositV1` via `new … {salt: …}(…)`, making `ContractOwner` the `Ownable` owner of the resulting contract. [1](#0-0) 

`DirectDepositV1.withdrawNative()` is `onlyOwner` and pushes the contract's ETH balance to `msg.sender` with a hard `require`: [2](#0-1) 

The intended recovery path in `ContractOwner` is: [3](#0-2) 

When `withdrawNative()` executes, `msg.sender` is `ContractOwner`. Because `ContractOwner` has no `receive()` or `payable fallback()`, the low-level `.call{value: balance}("")` returns `success = false`, and `require(success, "Failed to transfer native token to owner")` reverts the entire call stack. `withdrawFromDirectDepositV1(subaccount, address(0))` is therefore permanently broken.

---

### Impact Explanation

Any native ETH that lands in a `DirectDepositV1` contract is unrecoverable:

1. **Constructor silent-failure path** — the DDA constructor attempts to forward a pre-existing ETH balance to `wrappedNative`; on failure it emits `NativeTokenTransferFailed` and continues, leaving ETH stranded. [4](#0-3) 

2. **`selfdestruct` griefing** — because `createDirectDepositV1` uses a fixed `salt: bytes32(uint256(1))`, the CREATE2 address is deterministic and publicly computable. An attacker can `selfdestruct`-send ETH to that address before deployment, triggering the silent-failure path above. [5](#0-4) 

In both cases the only privileged recovery function (`withdrawFromDirectDepositV1` with `token == address(0)`) always reverts, so the ETH is permanently locked.

---

### Likelihood Explanation

The `selfdestruct` griefing vector requires only knowledge of the deterministic CREATE2 address (publicly derivable from on-chain data) and a trivial ETH amount. The constructor silent-failure path is triggered whenever `wrappedNative` is temporarily unavailable or misconfigured at deployment time. Neither path requires any privileged access.

---

### Recommendation

Add a `receive()` function to `ContractOwner` so it can accept ETH forwarded by `DirectDepositV1.withdrawNative()`:

```solidity
receive() external payable {}
```

Alternatively, refactor `DirectDepositV1.withdrawNative()` to accept a `recipient` address parameter and send ETH directly to the intended final recipient rather than routing it through `ContractOwner`.

---

### Proof of Concept

1. Attacker computes the CREATE2 address of the DDA for a target `subaccount` using the fixed salt `bytes32(uint256(1))`.
2. Attacker deploys a contract that `selfdestruct`s, sending 1 wei to that address.
3. `ContractOwner.createDirectDepositV1(subaccount)` is called; the constructor finds `address(this).balance == 1`, calls `wrappedNative.call{value: 1}("")`, which fails (e.g., `wrappedNative` not yet live), emits `NativeTokenTransferFailed(1)`, and continues — 1 wei is now stuck.
4. Owner calls `withdrawFromDirectDepositV1(subaccount, address(0))`.
5. Internally: `DirectDepositV1.withdrawNative()` executes `ContractOwner.call{value: 1}("")` → `ContractOwner` has no `receive()` → returns `false` → `require(success, "Failed to transfer native token to owner")` reverts.
6. ETH is permanently locked; no other code path can call `withdrawNative()` because it is `onlyOwner` and the owner is `ContractOwner`. [6](#0-5) [7](#0-6)

        external
        onlyOwner
    {
        address payable directDepositV1 = directDepositV1Address[subaccount];
        require(directDepositV1 != address(0), "no dda");
        if (token == address(0)) {
            uint256 preBalance = address(this).balance;
            DirectDepositV1(directDepositV1).withdrawNative();
            uint256 postBalance = address(this).balance;
            require(postBalance > preBalance, "empty");
            (bool success, ) = msg.sender.call{value: postBalance - preBalance}(
                ""

            );
        }
    }
