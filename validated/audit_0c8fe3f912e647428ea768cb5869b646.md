### Title
Deprecated `verify_transaction_inclusion` Remains Publicly Callable, Bypassing the Coinbase Merkle Proof Security Check - (File: contract/src/lib.rs)

### Summary

The `verify_transaction_inclusion` function is marked `#[deprecated]` but remains a live, publicly callable NEAR contract method. It lacks the coinbase Merkle proof validation that was introduced in `verify_transaction_inclusion_v2` specifically to mitigate the 64-byte transaction Merkle proof forgery vulnerability. Any unprivileged NEAR caller can invoke the deprecated endpoint directly, bypassing the security gate entirely and obtaining a `true` return value for a fabricated transaction ID.

### Finding Description

`verify_transaction_inclusion_v2` was introduced to close the 64-byte transaction forgery attack. Its defence is a mandatory coinbase proof step: [1](#0-0) 

This check anchors the Merkle tree structure by requiring the caller to prove the real coinbase transaction sits at index 0. Without it, an attacker can supply an internal Merkle tree node (a 64-byte concatenation of two child hashes) as the `tx_id` and provide a valid sibling path that reconstructs the block's Merkle root, making the contract believe a non-existent transaction was included.

The deprecated function retains the `pub` visibility and the `#[pause]` decorator (so it is live whenever the contract is not paused), but has **no coinbase proof requirement**: [2](#0-1) 

The `#[deprecated]` Rust attribute is a compile-time lint only; it imposes zero runtime restriction. The function is fully reachable as a NEAR contract call.

The only checks inside `verify_transaction_inclusion` are:
1. `confirmations <= gc_threshold`
2. Block hash exists in `mainchain_header_to_height` (block is on the canonical chain)
3. `merkle_proof` is non-empty
4. `compute_root_from_merkle_proof(tx_id, tx_index, proof) == header.merkle_root`

None of these prevent an internal Merkle node from being passed as `tx_id`.

### Impact Explanation

Any downstream NEAR contract that calls `verify_transaction_inclusion` (rather than v2) to gate a cross-chain action — e.g., a Bitcoin bridge that releases wrapped tokens upon proof of deposit — can be deceived. The attacker supplies an internal Merkle node of a real canonical-chain block as `tx_id`, constructs the matching sibling path, and receives `true`. This can be used to claim funds for a Bitcoin transaction that never occurred.

The `verify_transaction_inclusion_v2` path is safe because it enforces the coinbase anchor: [3](#0-2) 

But the deprecated path is a fully independent entry point that skips this anchor entirely.

### Likelihood Explanation

- No special role or permission is required; any NEAR account can call the deprecated method.
- The attack requires only knowledge of a canonical-chain block's Merkle tree, which is public Bitcoin data.
- The deprecated function is not removed or access-gated, so the attack surface persists across every deployment that has not explicitly paused the contract.
- Integrators who read the ABI or SDK bindings may not notice the deprecation warning and call the older, simpler endpoint.

### Recommendation

Remove `verify_transaction_inclusion` from the public ABI entirely, or gate it with an access-control role (e.g., `Role::DAO`) so it cannot be called by arbitrary accounts. The preferred fix is deletion; if backward compatibility is required, replace the body with an unconditional `env::panic_str("use verify_transaction_inclusion_v2")`.

### Proof of Concept

1. Identify any canonical-chain block `B` stored in the contract (e.g., via `get_last_block_header`).
2. Obtain `B`'s full transaction list from a Bitcoin node and compute its Merkle tree.
3. Pick any internal node `N` at depth `d` (a 64-byte value). Compute the sibling path from `N` up to the root.
4. Call `verify_transaction_inclusion` with:
   - `tx_id = N`
   - `tx_block_blockhash = B.block_hash`
   - `tx_index` = the leaf-index that corresponds to `N`'s position
   - `merkle_proof` = the sibling path computed in step 3
   - `confirmations = 1`
5. The function computes `compute_root_from_merkle_proof(N, tx_index, proof)`, which equals `B.merkle_root` by construction, and returns `true` — confirming a transaction that does not exist. [4](#0-3)

    #[deprecated(
        since = "0.5.0",
        note = "Use `verify_transaction_inclusion_v2` instead."
    )]
    #[pause]
    pub fn verify_transaction_inclusion(&self, #[serializer(borsh)] args: ProofArgs) -> bool {
        require!(
            args.confirmations <= self.gc_threshold,
            "The required number of confirmations exceeds the number of blocks stored in memory"
        );

        let heaviest_block_header = self
            .headers_pool
            .get(&self.mainchain_tip_blockhash)
            .unwrap_or_else(|| env::panic_str(ERR_KEY_NOT_EXIST));
        let target_block_height = self
            .mainchain_header_to_height
            .get(&args.tx_block_blockhash)
            .unwrap_or_else(|| env::panic_str("block does not belong to the current main chain"));

        // Check requested confirmations. No need to compute proof if insufficient confirmations.
        require!(
            (heaviest_block_header.block_height).saturating_sub(target_block_height) + 1
                >= args.confirmations,
            "Not enough blocks confirmed"
        );

        let header = self
            .headers_pool
            .get(&args.tx_block_blockhash)
            .unwrap_or_else(|| env::panic_str("cannot find requested transaction block"));

        require!(!args.merkle_proof.is_empty(), "Merkle proof is empty");

        // compute merkle tree root and check if it matches block's original merkle tree root
        merkle_tools::compute_root_from_merkle_proof(
            args.tx_id,
            usize::try_from(args.tx_index).unwrap(),
            &args.merkle_proof,
        ) == header.block_header.merkle_root
    }

    #[pause]
    pub fn verify_transaction_inclusion_v2(&self, #[serializer(borsh)] args: ProofArgsV2) -> bool {
        require!(
            args.merkle_proof.len() == args.coinbase_merkle_proof.len(),
            "Coinbase merkle proof and transaction merkle proof should have the same length"
        );

        let header = self
            .headers_pool
            .get(&args.tx_block_blockhash)
            .unwrap_or_else(|| env::panic_str("cannot find requested transaction block"));

        require!(
            merkle_tools::compute_root_from_merkle_proof(
                args.coinbase_tx_id.clone(),
                0usize,
                &args.coinbase_merkle_proof,
            ) == header.block_header.merkle_root,
            "Incorrect coinbase merkle proof"
        );

        #[allow(deprecated)]
        self.verify_transaction_inclusion(args.into())
    }
