Audit Report

## Title
`SwapAllowlistExtension` Per-User Gate Bypassed When Router Is Allowlisted — (`metric-periphery/contracts/extensions/SwapAllowlistExtension.sol`)

## Summary
`SwapAllowlistExtension.beforeSwap` gates swaps by checking the `sender` argument, which `MetricOmmPool.swap` sets to `msg.sender` — the immediate caller of the pool. When `MetricOmmSimpleRouter` intermediates a swap, the pool's `msg.sender` is the router contract, not the end-user. Allowlisting the router (required for any router-mediated swap to succeed on a gated pool) collapses the per-user gate to a single shared entry, allowing every address to bypass the allowlist by routing through the public router.

## Finding Description
`MetricOmmPool.swap` passes `msg.sender` verbatim as the `sender` argument to `_beforeSwap`: [1](#0-0) 

`ExtensionCalling._beforeSwap` encodes that `sender` into the ABI-encoded call dispatched to every configured extension: [2](#0-1) 

`SwapAllowlistExtension.beforeSwap` then evaluates `allowedSwapper[msg.sender][sender]`, where `msg.sender` is the pool and `sender` is whoever called `pool.swap()`: [3](#0-2) 

`MetricOmmSimpleRouter.exactInputSingle` calls `pool.swap()` directly with no user-identity forwarding: [4](#0-3) 

The actual end-user (`msg.sender` of the router call) is stored only in transient callback context for payment settlement and is never surfaced to the extension layer. The extension therefore sees `sender = router_address` for every user who routes through the router. If the pool admin allowlists the router (the only way to permit router-mediated swaps on a gated pool), the check `allowedSwapper[pool][router]` passes for every caller regardless of their individual allowlist status. There is no configuration that simultaneously enables router-mediated swaps and enforces per-user allowlist policy.

## Impact Explanation
A pool admin who deploys a curated pool with `SwapAllowlistExtension` to restrict trading to KYC-verified counterparties, whitelisted market makers, or protocol-internal accounts, and who allowlists the router to support the standard periphery UI, inadvertently opens the pool to every address. Unauthorized users can execute swaps at oracle-anchored prices, extract LP value, and violate the pool's access policy. This constitutes direct loss of LP principal and a broken core pool invariant (curated access enforcement).

## Likelihood Explanation
`MetricOmmSimpleRouter` is the primary documented swap interface. A pool admin who wants allowlisted users to use the standard router UI must allowlist the router. Once the router is allowlisted, the bypass is reachable by any address with no special privileges, no malicious setup, and no non-standard token behavior. The only prerequisite is the natural administrative step of allowlisting the router.

## Recommendation
The extension must recover the true end-user identity when the immediate caller is a trusted router. The simplest safe fix mirrors the callback-context pattern already used for payment settlement: the router writes the originating `msg.sender` into a transient storage slot before calling `pool.swap()`, and `SwapAllowlistExtension.beforeSwap` reads that slot when `sender` is a recognized router address. Alternatively, the router encodes `msg.sender` into `extensionData` under a convention the extension decodes and checks. Either approach must be authenticated (the extension must verify the router wrote the slot, not an arbitrary caller spoofing it).

## Proof of Concept
```
Setup:
  pool configured with SwapAllowlistExtension as beforeSwap hook
  allowedSwapper[pool][alice] = true   // alice is the only intended user
  allowedSwapper[pool][router] = true  // admin allowlists router so alice can use the UI

Attack:
  bob (not allowlisted) calls:
    router.exactInputSingle({pool: pool, ..., recipient: bob})

Execution trace:
  router.exactInputSingle (msg.sender = bob)
    → pool.swap (msg.sender = router)
      → _beforeSwap(sender = router, ...)
        → SwapAllowlistExtension.beforeSwap(sender = router)
          → allowedSwapper[pool][router] == true  ✓  (no revert)
      → swap executes, bob receives output tokens

Result: bob bypasses the per-user allowlist and trades on a curated pool.
```
