### Title
`gas_prices_to_hash` omits `l2_gas_consumed` and `next_l2_gas_price` from block hash for all Starknet versions including 0.14.x — (File: crates/starknet_api/src/block_hash/block_hash_calculator.rs)

### Summary

The `gas_prices_to_hash` function and `PartialBlockHashComponents` struct permanently omit `l2_gas_consumed` and `next_l2_gas_price` from the block hash preimage for every live Starknet version, including 0.14.x. Both fields are stored in `BlockHeaderWithoutHash` and `StorageBlockHeader` and drive the EIP-1559 fee market. Because the block hash does not commit to them, a syncing node that receives a block from a malicious peer cannot detect manipulation of these fields through hash verification alone, causing fee estimation to return authoritative-looking wrong values.

### Finding Description

`BlockHashVersion` has exactly two variants:

```rust
pub enum BlockHashVersion {
    V0_13_2,
    V0_13_4,
}
```

The `TryFrom<StarknetVersion> for BlockHashVersion` implementation maps every version ≥ 0.13.4 — including all 0.14.x releases — to `V0_13_4`:

### Impact Explanation

`next_l2_gas_price` is the EIP-1559 base fee for the following block. It is read by every fee-estimation and simulation call. Because the block hash does not commit to it, a node syncing over P2P cannot distinguish a legitimately computed `next_l2_gas_price` from a manipulated one by verifying the hash. The block signature covers only the block hash; it therefore also fails to authenticate this field. A syncing node that stores a manipulated value will subsequently serve wrong results from `starknet_estimateFee`, `starknet_simulateTransactions`, and pending-block gas-price queries — all of which read `next_l2_gas_price` from the stored header. This matches the allowed impact: **High — RPC execution, fee estimation, tracing, simulation, or pending view returns an authoritative-looking wrong value.**

`l2_gas_consumed` is the gas-usage input to the EIP-1559 price-adjustment formula (`calculate_next_base_gas_price`). Manipulating it shifts the computed `next_l2_gas_price` for the block after the manipulated one, compounding the effect across multiple blocks.

### Likelihood Explanation

The vulnerability is reachable by any peer that can serve blocks to a syncing node — a standard P2P network participant. No privileged sequencer key is required; the attacker only needs to be a peer that the victim node trusts for block data. The current Starknet version (0.14.4) is already past the "after 0.14.0" threshold named in the TODO, so the omission is active in production today.

### Recommendation

1. Add a `V0_14_0` (or later) variant to `BlockHashVersion` and update `TryFrom<StarknetVersion>` to map 0.14.x versions to it.
2. Add `l2_gas_consumed: GasAmount` and `next_l2_gas_price: GasPrice` to `PartialBlockHashComponents`.
3. Update `gas_prices_to_hash` to include both fields in the Poseidon preimage for the new variant, matching the echonet CLI reference.
4. Similarly resolve the `// TODO(AndrewL): Add this field into the block hash.` for `fee_proposal_fri` in `BlockHeaderWithoutHash`.

### Proof of Concept

```
1. Attacker runs a P2P node and serves a 0.14.x block to a victim syncing node.
2. The block has a valid block hash H (computed by gas_prices_to_hash without
   l2_gas_consumed / next_l2_gas_price).
3. The attacker sets next_l2_gas_price = 10 × correct_value in the block header
   while keeping all other fields identical.
4. The victim node calls calculate_block_hash → produces the same H (the two
   omitted fields are not in PartialBlockHashComponents).
5. Hash verification passes; the block is stored with the inflated next_l2_gas_price.
6. All subsequent starknet_estimateFee / starknet_simulateTransactions calls on
   the victim node read next_l2_gas_price from the stored header and return fee
   estimates 10× too high, causing users to overpay or have transactions rejected.
```

```rust
pub fn calculate_block_hash(
    partial_block_hash_components: &PartialBlockHashComponents,
    state_root: GlobalRoot,
    previous_block_hash: BlockHash,
) -> StarknetApiResult<BlockHash> {
    let block_hash_version: BlockHashVersion =
        partial_block_hash_components.starknet_version.try_into()?;
    let block_commitments = &partial_block_hash_components.header_commitments;
    Ok(BlockHash(
        HashChain::new()
            .chain(&block_hash_version.clone().into())
            .chain(&partial_block_hash_components.block_number.0.into())
            .chain(&state_root.0)
            .chain(&partial_block_hash_components.sequencer.0)
            .chain(&partial_block_hash_components.timestamp.0.into())
            .chain(&block_commitments.concatenated_counts)
            .chain(&block_commitments.state_diff_commitment.0.0)
            .chain(&block_commitments.transaction_commitment.0)
            .chain(&block_commitments.event_commitment.0)
            .chain(&block_commitments.receipt_commitment.0)
            .chain_iter(
                gas_prices_to_hash(
                    &partial_block_hash_components.l1_gas_price,
                    &partial_block_hash_components.l1_data_gas_price,
                    &partial_block_hash_components.l2_gas_price,
                    &block_hash_version,
                )
                .iter(),
            )
            .chain(
                &Felt::try_from(&partial_block_hash_components.starknet_version)
                    .expect("Expect ASCII version"),
            )
            .chain(&Felt::ZERO)
            .chain(&previous_block_hash.0)
            .get_poseidon_hash(),
    ))
}
