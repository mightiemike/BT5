### Title
Stale `digestToSubaccount` and `filledAmounts` After Isolated Subaccount Close Enables Undercollateralized Fill — (`core/contracts/OffchainExchange.sol`)

### Summary
`_tryCloseIsolatedSubaccount` does not clear `digestToSubaccount[digest]` or `filledAmounts[digest]`. A subsequent `createIsolatedSubaccount` call with the same order returns the old closed subaccount via early-return without transferring margin. `matchOrders` then fills against that subaccount — which holds zero margin — while `filledAmounts` reduces the remaining fillable amount to whatever was not yet filled in the prior lifecycle.

### Finding Description
The state-machine invariant is: when an isolated subaccount is closed, all digest-keyed state that gates re-creation and fill accounting must be reset. `_tryCloseIsolatedSubaccount` resets the slot/mask/parent mappings but leaves two critical entries intact:

1. `digestToSubaccount[digest]` — still maps to the now-closed isolated subaccount address.
2. `filledAmounts[digest]` — still holds the cumulative fill from the previous lifecycle.

When `createIsolatedSubaccount` is called again with the identical order:

```
if (digestToSubaccount[digest] != bytes32(0)) {
    return digestToSubaccount[digest];   // early return, no margin transfer
}
``` [1](#0-0) 

The function returns the stale closed subaccount. Margin transfer at lines 1074–1087 is never reached. [2](#0-1) 

`matchOrders` then resolves `taker.order.sender` to the closed isolated subaccount:

```
if (digestToSubaccount[ordersInfo.taker.digest] != bytes32(0)) {
    taker.order.sender = digestToSubaccount[ordersInfo.taker.digest];
}
``` [3](#0-2) 

`_validateOrder` reduces the fillable amount by the stale `filledAmounts`:

```
int128 filledAmount = filledAmounts[orderDigest];
order.amount -= filledAmount;
``` [4](#0-3) 

The fill then executes against the isolated subaccount, which has zero quote balance (all margin was returned to parent during close at lines 190–200). [5](#0-4) 

The post-fill health check uses a virtual `isHealthy` that returns `true` unconditionally in the base contract:

```
function isHealthy(bytes32) internal view virtual returns (bool) {
    return true;
}
``` [6](#0-5) 

No override was found. The fill succeeds, leaving the isolated subaccount holding a perp position with no margin.

### Impact Explanation
The isolated subaccount accumulates a perp position without any quote collateral backing it. This is an undercollateralized position that the protocol cannot recover from through normal liquidation (the subaccount has no margin to seize). The protocol absorbs the loss if the position moves adversely. Additionally, the trader's margin (sitting in the parent account) is effectively decoupled from the position — the trader retains the margin while the protocol bears the risk of the open position.

### Likelihood Explanation
The call sequence is fully externally reachable: `CreateIsolatedSubaccount` and `CloseIsolatedSubaccount` are both supported `TransactionType` values processed by `EndpointTx`. [7](#0-6) 

Any trader can submit the same signed order after a close. The sequencer processes these as standard slow-mode or fast-path transactions. No privileged access is required.

### Recommendation
In `_tryCloseIsolatedSubaccount`, after clearing the slot/mask/parent mappings, also clear all digest-keyed state associated with the subaccount being closed:

```solidity
// pseudocode — requires reverse mapping from subaccount → digest(es)
delete filledAmounts[digest];
delete digestToSubaccount[digest];
delete digestToMargin[digest];
```

Alternatively, maintain a reverse mapping `subaccountToDigests` populated at `createIsolatedSubaccount` time, and iterate over it during close to clear all associated digests.

   → tryCloseIsolatedSubaccount(isoA)
   → isolatedSubaccounts[alice][0] = bytes32(0)

   → digestToSubaccount[D] still = isoA  ← BUG
   → filledAmounts[D] still = 50%        ← BUG

   → digestToSubaccount[D] != 0 → early return isoA
   → NO margin transferred
   → spotEngine.balance(isoA, QUOTE) = 0

   → _validateOrder: order.amount -= filledAmounts[D]=50% → 50% remaining → valid
   → taker.order.sender = isoA (from digestToSubaccount)
   → fill executes: perpEngine.balance(isoA).amount = +50%
   → isHealthy(isoA) = true (base impl)
   → filledAmounts[D] = 100%

Assert: isoA has perp position but spotEngine.balance(isoA, QUOTE) == 0
Assert: alice.balance unchanged (margin never re-transferred)
→ Undercollateralized position confirmed
