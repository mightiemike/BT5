### Title
`get_tip_resource_bounds_hash` Produces a Different Fee-Field Hash for `ValidResourceBounds::AllResources` with Zero L2/L1DataGas vs `ValidResourceBounds::L1Gas`, Silently Broken by Protobuf Round-Trip — (`crates/starknet_api/src/transaction_hash.rs` / `crates/apollo_protobuf/src/converters/transaction.rs`)

---

### Summary

`get_tip_resource_bounds_hash` hashes a **different number of resource-bound elements** depending on which `ValidResourceBounds` variant is present. The protobuf deserializer silently **downgrades** `AllResources { l2_gas: 0, l1_data_gas: 0 }` to `L1Gas`, changing the element count and therefore the hash. A transaction whose hash was computed with `AllResources` (4 elements) will produce a different hash after a protobuf round-trip (3 elements), causing the receiving node to bind the wrong hash to the transaction.

---

### Finding Description

**Hash boundary — `get_tip_resource_bounds_hash`**

`crates/starknet_api/src/transaction_hash.rs` lines 188–211:

```rust
pub fn get_tip_resource_bounds_hash(
    resource_bounds: &ValidResourceBounds,
    tip: &Tip,
) -> Result<Felt, StarknetApiError> {
    let mut resource_felts = vec![
        get_concat_resource(&l1_resource_bounds, L1_GAS)?,
        get_concat_resource(&l2_resource_bounds, L2_GAS)?,
    ];
    resource_felts.extend(match resource_bounds {
        ValidResourceBounds::L1Gas(_) => vec![],                          // 3-element poseidon
        ValidResourceBounds::AllResources(all_resources) => {
            vec![get_concat_resource(&all_resources.l1_data_gas, L1_DATA_GAS)?]  // 4-element poseidon
        }
    });
    Ok(HashChain::new().chain(&tip.0.into()).chain_iter(resource_felts.iter()).get_poseidon_hash())
}
```

For `L1Gas`: `poseidon(tip, l1_packed, l2_packed)` — **3 elements**.
For `AllResources` with `l1_data_gas = 0`: `poseidon(tip, l1_packed, l2_packed, 0)` — **4 elements**.

These are cryptographically distinct values even when `l2_gas = 0` and `l1_data_gas = 0`.

**Protobuf conversion boundary — silent variant downgrade**

`crates/apollo_protobuf/src/converters/transaction.rs` lines 417–436:

```rust
impl TryFrom<protobuf::ResourceBounds> for ValidResourceBounds {
    fn try_from(value: protobuf::ResourceBounds) -> Result<Self, Self::Error> {
        let l1_data_gas = value.l1_data_gas.unwrap_or_default();
        ...
        Ok(if l1_data_gas.is_zero() && l2_gas.is_zero() {
            ValidResourceBounds::L1Gas(l1_gas)          // ← variant changed
        } else {
            ValidResourceBounds::AllResources(AllResourceBounds { l1_gas, l2_gas, l1_data_gas })
        })
    }
}
```

Any `AllResources` transaction whose `l2_gas` and `l1_data_gas` are both zero is silently re-typed as `L1Gas` on deserialization. The comment on the same line acknowledges this is a compatibility shim: `// TODO(Shahak): Assert data gas is not none once we remove support for 0.13.2.`

**End-to-end hash divergence**

1. A user submits an `InvokeV3` transaction with `AllResourceBounds { l1_gas: X, l2_gas: 0, l1_data_gas: 0 }` via RPC.
2. The gateway converts it to `InternalRpcInvokeTransactionV3` (which always carries `AllResourceBounds`) and computes hash **H₁** using `AllResources` → 4-element poseidon.
3. The transaction is included in a block and propagated over P2P as a protobuf `InvokeV3` message.
4. The receiving/syncing node deserializes the protobuf into `InvokeTransactionV3` with `ValidResourceBounds`. Because `l2_gas = 0` and `l1_data_gas = 0`, the converter produces `ValidResourceBounds::L1Gas(X)`.
5. The receiving node recomputes the hash using `L1Gas` → 3-element poseidon → **H₂ ≠ H₁**.
6. The stored `tx_hash` in the block is H₁; the locally recomputed hash is H₂. The node either rejects the block or stores the transaction under the wrong hash.

This is structurally identical to the external bug's "zero-state" branch: the `is_zero()` guard on `l2_gas` and `l1_data_gas` selects a shorter hash preimage, just as `totalSupply == 0` selected a different share-price formula.

---

### Impact Explanation

A syncing node that receives a valid block containing an `AllResources` transaction with `l2_gas = 0` and `l1_data_gas = 0` will recompute a hash that does not match the hash committed in the block. Depending on how the node uses the recomputed hash:

- **Wrong state / receipt**: the transaction is stored under hash H₂ instead of H₁, making `starknet_getTransactionByHash(H₁)` return nothing and `starknet_getTransactionReceipt(H₁)` return nothing — the RPC layer serves authoritative-looking wrong values.
- **Block rejection**: if the node validates `recomputed_hash == stored_hash`, it rejects the block and cannot sync past it — a liveness failure for any node that syncs via P2P.

This matches the allowed impact: *"Wrong state, receipt, event … storage value … from blockifier/syscall/execution logic for accepted input"* and *"Transaction conversion or signature/hash logic binds the wrong … hash … or executable payload."*

---

### Likelihood Explanation

The trigger is an `InvokeV3` transaction with `AllResources` where both `l2_gas` and `l1_data_gas` are zero. This is a valid, well-formed transaction (a user who only cares about L1 gas may set the other bounds to zero). Any such transaction propagated over P2P hits the downgrade path unconditionally. No privileged access is required; the attacker only needs to submit such a transaction and have it included in a block.

---

### Recommendation

**Option A (preferred):** Remove the variant-conditional branch in `get_tip_resource_bounds_hash`. Always hash all three resource bounds regardless of variant, matching the Cairo OS `hash_fee_fields` which unconditionally asserts `n_resource_bounds = 3` and hashes all three:

```rust
// Always include L1DataGas, using zero for L1Gas variant.
let l1_data_gas = match resource_bounds {
    ValidResourceBounds::L1Gas(_) => ResourceBounds::default(),
    ValidResourceBounds::AllResources(r) => r.l1_data_gas,
};
resource_felts.push(get_concat_resource(&l1_data_gas, L1_DATA_GAS)?);
```

**Option B:** Remove the silent downgrade in the protobuf converter. Preserve the `AllResources` variant even when `l2_gas` and `l1_data_gas` are zero, so the hash computed after deserialization matches the hash computed before serialization.

### Proof of Concept

```rust
use starknet_api::transaction::fields::{
    AllResourceBounds, ResourceBounds, ValidResourceBounds, GasAmount, GasPrice, Tip,
};
use starknet_api::transaction_hash::get_tip_resource_bounds_hash;

let l1_gas = ResourceBounds { max_amount: GasAmount(100), max_price_per_unit: GasPrice(1) };
let tip = Tip(0);

// Hash computed at gateway (AllResources, 4-element poseidon)
let all_resources = ValidResourceBounds::AllResources(AllResourceBounds {
    l1_gas,
    l2_gas: ResourceBounds::default(),      // zero
    l1_data_gas: ResourceBounds::default(), // zero
});
let h1 = get_tip_resource_bounds_hash(&all_resources, &tip).unwrap();

// Hash computed after protobuf round-trip (L1Gas, 3-element poseidon)
let l1_only = ValidResourceBounds::L1Gas(l1_gas);
let h2 = get_tip_resource_bounds_hash(&l1_only, &tip).unwrap();

assert_ne!(h1, h2, "same semantic bounds, different hashes — hash mismatch after protobuf round-trip");
```

func hash_fee_fields{range_check_ptr, poseidon_ptr: PoseidonBuiltin*}(
    tip: felt, resource_bounds: ResourceBounds*, n_resource_bounds: felt
) -> felt {
    alloc_locals;

    let (local data_to_hash: felt*) = alloc();
    assert data_to_hash[0] = tip;
    assert_nn_le(tip, 2 ** 64 - 1);

    static_assert L1_GAS_INDEX == 0;
    static_assert L2_GAS_INDEX == 1;
    static_assert L1_DATA_GAS_INDEX == 2;

    with_attr error_message("Invalid number of resource bounds: {n_resource_bounds}.") {
        assert n_resource_bounds = 3;
    }

    // L1 gas.
    let l1_gas_bounds = resource_bounds[L1_GAS_INDEX];
    assert l1_gas_bounds.resource = L1_GAS;
    assert data_to_hash[1] = pack_resource_bounds(l1_gas_bounds);

    // L2 gas.
    let l2_gas_bounds = resource_bounds[L2_GAS_INDEX];
    assert l2_gas_bounds.resource = L2_GAS;
    assert data_to_hash[2] = pack_resource_bounds(l2_gas_bounds);

    // L1 data gas.
    let l1_data_gas_bounds = resource_bounds[L1_DATA_GAS_INDEX];
    assert l1_data_gas_bounds.resource = L1_DATA_GAS;
    assert data_to_hash[3] = pack_resource_bounds(l1_data_gas_bounds);

    let (hash) = poseidon_hash_many(n=n_resource_bounds + 1, elements=data_to_hash);
    return hash;
