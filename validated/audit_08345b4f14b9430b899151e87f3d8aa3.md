### Title
Silent Truncating Cast from `ShardId` (`u64`) to `u16` in `CongestionInfoV1.allowed_shard` Breaks Congestion-Control Shard Identity at Shard-Layout Upgrade Boundary — (`File: core/primitives/src/congestion_info.rs`, `core/primitives-core/src/types.rs`)

---

### Summary

`CongestionInfoV1` stores `allowed_shard` as a `u16`, but `ShardId` is a `u64`-backed newtype. The `Into<u16>` implementation for `ShardId` performs a silent truncating cast (`self.0 as u16`). When `finalize_allowed_shard` converts the selected `ShardId` into `u16` for storage in the Borsh-serialized `ChunkHeader`, any shard whose numeric ID exceeds `65535` is silently mapped to a different shard. The round-trip `ShardId(u64) → u16 → ShardId(u64)` is lossy, causing the wrong shard to be granted the congestion-control "allowed shard" privilege during full-congestion epochs.

---

### Finding Description

**Root cause — truncating `Into<u16>` for `ShardId`:**

`ShardId` is defined as a `u64` newtype: [1](#0-0) 

The `Into<u16>` implementation performs an unchecked narrowing cast: [2](#0-1) 

```rust
impl Into<u16> for ShardId {
    fn into(self) -> u16 {
        self.0 as u16   // silent truncation — no overflow check
    }
}
```

**Storage site — `CongestionInfoV1.allowed_shard` is `u16`:** [3](#0-2) 

```rust
pub struct CongestionInfoV1 {

    pub receipt_bytes: u64,
    pub allowed_shard: u16,   // ← only 16 bits, ShardId is 64 bits

```

**Call site — `finalize_allowed_shard` invokes the truncating conversion:** [4](#0-3) 

```rust

The `CongestionInfo` enum (containing `CongestionInfoV1`) is `BorshSerialize`/`BorshDeserialize` and is embedded in the `ChunkHeader`, making it a consensus-critical, protocol-versioned field: [7](#0-6) 

---

### Impact Explanation

During a fully-congested epoch, `finalize_allowed_shard` selects one shard that is permitted to forward receipts to the congested shard. If the selected `ShardId` has a numeric value above `65535`, the stored `allowed_shard` field silently encodes a *different* shard (the low 16 bits of the original ID). Consequences:

1. **Wrong shard is privileged.** The shard that actually holds the `allowed_shard_outgoing_gas` budget is not the one the protocol intended; the intended shard is blocked at zero gas.
2. **Liveness degradation.** Under full congestion the protocol guarantees forward progress by allowing exactly one shard to send. If the stored ID maps to a non-existent or unintended shard, no shard can make progress, stalling the congested shard indefinitely.
3. **Consensus-serialized field.** Because `CongestionInfoV1` is Borsh-serialized inside `ChunkHeader`, any node that independently recomputes `allowed_shard` from the full `ShardId` and compares it to the stored `u16` will observe a mismatch, potentially causing chunk validation failures across the network.

---

### Likelihood Explanation

Current mainnet shard IDs are small integers (0–7), so the truncation is presently a no-op. However:

- `ShardId` is explicitly documented as an *arbitrary* `u64` — not constrained to `0..NUM_SHARDS`: [8](#0-7) 

- Dynamic resharding (already in production) generates new shard IDs at each split. There is no protocol-level gate preventing a future shard layout from assigning IDs above `65535`.
- The `Into<u16>` conversion carries no `debug_assert`, no `checked` variant, and no compile-time bound — the truncation is entirely silent.
- The upgrade boundary is public: any protocol upgrade that introduces shard IDs > `65535` activates the bug without any code change.

---

### Recommendation

1. **Replace the truncating cast with a checked conversion.** In `finalize_allowed_shard`, assert or return an error if the selected `ShardId` exceeds `u16::MAX`:

```rust
let allowed_shard_u16 = u16::try_from(u64::from(allowed_shard))
    .expect("ShardId exceeds u16::MAX; CongestionInfoV1.allowed_shard cannot represent it");
self.set_allowed_shard(allowed_shard_u16);
```

2. **Widen `CongestionInfoV1.allowed_shard` to `u64`** (requires a new `CongestionInfoV2` variant, since `CongestionInfoV1` is a versioned, Borsh-serialized protocol schema that must not be mutated).

3. **Remove the unchecked `Into<u16>` implementation** for `ShardId` in `core/primitives-core/src/types.rs` (lines 180–184) or replace it with a `TryInto<u16>` to force callers to handle the overflow case explicitly.

---

### Proof of Concept

```
Given: a shard layout containing ShardId(0x10001) = 65537

finalize_allowed_shard selects ShardId(65537) as the allowed shard.

allowed_shard.into() → u16:
    65537u64 as u16 = 1u16   ← silent truncation

CongestionInfoV1 { allowed_shard: 1 } is Borsh-serialized into ChunkHeader.

At read-back:
    self.info.allowed_shard() → 1u16
    ShardId::from(1u16)       → ShardId(1)

Comparison in outgoing_gas_limit:
    sender_shard == ShardId(1)   // true for shard 1, false for shard 65537

Result:
    ShardId(65537) — the intended allowed shard — receives Gas::ZERO outgoing limit.
    ShardId(1)     — an unintended shard — receives allowed_shard_outgoing_gas.
    Forward progress guarantee is broken for the congested shard.
``` [2](#0-1) [4](#0-3) [9](#0-8)
