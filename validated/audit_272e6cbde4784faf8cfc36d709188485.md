### Title
Duplicate Position IDs in `MsgClaimTierRewards` Enable Double-Claiming Bonus Rewards from Pool - (`File: x/tieredrewards/keeper/msg_server.go`)

### Summary
`ClaimTierRewards` loads all requested positions into an in-memory slice before processing them. Because there is no deduplication check on `msg.PositionIds`, a position owner can submit the same position ID multiple times in a single transaction. The second (and subsequent) copies in the slice retain the stale pre-claim `LastEventSeq` and `LastBonusAccrual` checkpoints, causing `processEventsAndClaimBonus` to replay the same validator events and transfer bonus coins from the `RewardsPoolName` module account a second time.

### Finding Description

`ClaimTierRewards` in `msg_server.go` first collects all positions into a slice:

```go
positions := make([]types.PositionState, 0, len(msg.PositionIds))
for _, posId := range msg.PositionIds {
    pos, err := ms.getPositionState(ctx, posId)
    ...
    if err := ms.validateClaimRewards(pos.Position, msg.Owner); err != nil {
        return nil, err
    }
    positions = append(positions, pos)
}
totalBase, totalBonus, err := ms.claimRewardsAndUpdatesPositions(ctx, positions)
```

`validateClaimRewards` only checks ownership — it performs no deduplication:

```go
func (k Keeper) validateClaimRewards(pos types.Position, owner string) error {
    if !pos.IsOwner(owner) {
        return types.ErrNotPositionOwner
    }
    return nil
}
```

`claimRewardsAndUpdatesPositions` then iterates the slice and, for each entry, calls `processEventsAndClaimBonus` followed by `setPosition`:

```go
for i := range positions {
    pos := &positions[i]
    ...
    bonus, err := k.processEventsAndClaimBonus(ctx, pos)
    ...
    if err := k.setPosition(ctx, pos.Position, nil); err != nil { ... }
}
```

`processEventsAndClaimBonus` uses `pos.LastEventSeq` (read from the in-memory copy) to determine which validator events to replay and how much bonus to pay:

```go
events, err := k.getValidatorEventsSince(ctx, valAddr, pos.LastEventSeq)
...
if err := k.bankKeeper.SendCoinsFromModuleToAccount(ctx, types.RewardsPoolName, ownerAddr, bonusCoins); err != nil {
    return nil, err
}
```

When the same position ID appears twice in `msg.PositionIds`:
- **First iteration**: loads position with `LastEventSeq = X`, replays events, sends bonus coins, calls `setPosition` to persist `LastEventSeq = Y` to the store.
- **Second iteration**: operates on the **stale in-memory copy** still holding `LastEventSeq = X`. `getValidatorEventsSince` returns the same events again. `SendCoinsFromModuleToAccount` transfers the same bonus amount a second time.

Base rewards (`claimBaseRewards` → `distributionKeeper.WithdrawDelegationRewards`) are not double-spent because the distribution module drains the accrued rewards on the first call and returns zero on the second. The vulnerability is isolated to the bonus pool transfer.

### Impact Explanation

The `RewardsPoolName` module account balance is drained at a multiple of the legitimate bonus owed. An attacker who owns a position with N pending bonus events can include that position ID K times in a single `MsgClaimTierRewards` and receive K× the correct bonus payout. Repeated across blocks, this can fully exhaust the bonus rewards pool, permanently denying bonus rewards to all other position holders.

The corrupted value is the `RewardsPoolName` module account coin balance and the `LastEventSeq` / `LastBonusAccrual` checkpoints stored per position.

### Likelihood Explanation

Any position owner can craft this transaction with no special privileges. The `MsgClaimTierRewards` message accepts a repeated `PositionIds` field, and neither the message-level `Validate()` nor `validateClaimRewards` rejects duplicates. The attack requires only a funded position with accrued bonus events, which is the normal operating state of any long-term staker.

### Recommendation

Add a deduplication check before processing. The simplest fix is to reject the message if `PositionIds` contains duplicates:

```go
seen := make(map[uint64]struct{}, len(msg.PositionIds))
for _, id := range msg.PositionIds {
    if _, exists := seen[id]; exists {
        return nil, errorsmod.Wrapf(types.ErrInvalidPositionID, "duplicate position id: %d", id)
    }
    seen[id] = struct{}{}
}
```

This check should be placed in `MsgClaimTierRewards.Validate()` in `types/msgs.go` so it is enforced at the ante-handler level, or at minimum at the top of `ClaimTierRewards` before positions are loaded.

### Proof of Concept

1. Owner creates a tier position (ID = 42) and accrues bonus rewards over several blocks (validator events accumulate).
2. Owner submits `MsgClaimTierRewards{Owner: owner, PositionIds: [42, 42, 42]}`.
3. The handler loads position 42 three times into the `positions` slice; all three copies share `LastEventSeq = X`.
4. `claimRewardsAndUpdatesPositions` processes each copy:
   - Copy 1: `processEventsAndClaimBonus` replays events X→Y, sends `bonusCoins` to owner, `setPosition` writes `LastEventSeq = Y` to store.
   - Copy 2: in-memory copy still has `LastEventSeq = X`; `getValidatorEventsSince` returns the same events; `SendCoinsFromModuleToAccount` sends `bonusCoins` again.
   - Copy 3: same as copy 2; sends `bonusCoins` a third time.
5. Owner receives 3× the legitimate bonus; `RewardsPoolName` balance is reduced by 3× the correct amount. [1](#0-0) [2](#0-1) [3](#0-2) [4](#0-3)

```go
func (ms msgServer) ClaimTierRewards(ctx context.Context, msg *types.MsgClaimTierRewards) (*types.MsgClaimTierRewardsResponse, error) {
	if err := msg.Validate(); err != nil {

		positions = append(positions, pos)
	}

		return nil, err
	}
```

```go
func (k Keeper) claimRewardsAndUpdatesPositions(ctx context.Context, positions []types.PositionState) (sdk.Coins, sdk.Coins, error) {
	totalBase := sdk.NewCoins()
	totalBonus := sdk.NewCoins()

	for i := range positions {
		pos := &positions[i]

		if !pos.IsDelegated() {
			continue
		}

		base, err := k.claimBaseRewards(ctx, *pos)
		if err != nil {
			return nil, nil, err
		}
		totalBase = totalBase.Add(base...)

		bonus, err := k.processEventsAndClaimBonus(ctx, pos)
		if err != nil {
			return nil, nil, err
		}
		totalBonus = totalBonus.Add(bonus...)

		if err := k.setPosition(ctx, pos.Position, nil); err != nil {
			return nil, nil, err
		}
	}

	return totalBase, totalBonus, nil
}
```

```go
func (k Keeper) processEventsAndClaimBonus(ctx context.Context, pos *types.PositionState) (sdk.Coins, error) {
	// Rewards should have been claimed before undelegation
	if !pos.IsDelegated() {
		return sdk.NewCoins(), nil
	}

	valAddr, err := sdk.ValAddressFromBech32(pos.Delegation.ValidatorAddress)
	if err != nil {
		return nil, err
	}

	events, err := k.getValidatorEventsSince(ctx, valAddr, pos.LastEventSeq)
	if err != nil {
		return nil, err
	}

	sdkCtx := sdk.UnwrapSDKContext(ctx)
	blockTime := sdkCtx.BlockTime()

	totalBonus := math.ZeroInt()
	// Use the persisted bonded state from the last replay, not a hardcoded default.
	// This prevents overpaying bonus for unbonded gaps between claims.
	bonded := pos.LastKnownBonded
	segmentStart := pos.LastBonusAccrual

	tier, err := k.getTier(ctx, pos.TierId)
	if err != nil {
		return nil, err
	}

	for _, entry := range events {
		evt := entry.Event

		if bonded {
			// Compute bonus for the bonded segment [segmentStart, eventTime]
			// using the snapshot rate at the event.
			bonus := k.computeSegmentBonus(*pos, tier, segmentStart, evt.Timestamp, evt.TokensPerShare)
			totalBonus = totalBonus.Add(bonus)
		}

		// Update bonded state based on event type.
		switch evt.EventType {
		case types.ValidatorEventType_VALIDATOR_EVENT_TYPE_UNBOND:
			bonded = false
		case types.ValidatorEventType_VALIDATOR_EVENT_TYPE_BOND:
			bonded = true
		case types.ValidatorEventType_VALIDATOR_EVENT_TYPE_SLASH:
			// Slash doesn't change bonded state.
		}

		segmentStart = evt.Timestamp
		pos.UpdateLastEventSeq(entry.Seq)

		// Decrement reference count.
		if err := k.decrementEventRefCount(ctx, valAddr, entry.Seq); err != nil {
			return nil, err
		}
	}

	val, err := k.stakingKeeper.GetValidator(ctx, valAddr)
	if err != nil {
		return nil, err
	}
	// Defensive: validator bond status check
	if bonded && val.IsBonded() {
		currentRate, err := k.getTokensPerShare(ctx, valAddr)
		if err != nil {
			return nil, err
		}
		bonus := k.computeSegmentBonus(*pos, tier, segmentStart, blockTime, currentRate)
		totalBonus = totalBonus.Add(bonus)
	}

	applyBonusAccrualCheckpoint(&pos.Position, blockTime)
	// Persist the bonded state so the next replay starts correctly.
	pos.UpdateLastKnownBonded(bonded)

	if totalBonus.IsZero() {
		return sdk.NewCoins(), nil
	}

	bondDenom, err := k.stakingKeeper.BondDenom(ctx)
	if err != nil {
		return nil, err
	}

	bonusCoins := sdk.NewCoins(sdk.NewCoin(bondDenom, totalBonus))

	if err := k.sufficientBonusPoolBalance(ctx, bonusCoins); err != nil {
		return nil, err
	}

	ownerAddr, err := sdk.AccAddressFromBech32(pos.Owner)
	if err != nil {
		return nil, errorsmod.Wrap(sdkerrors.ErrInvalidAddress, "invalid owner address")
	}

	if err := k.bankKeeper.SendCoinsFromModuleToAccount(ctx, types.RewardsPoolName, ownerAddr, bonusCoins); err != nil {
		return nil, err
	}
```
