### Title
NFT Permanently Unburnable After Transfer to Non-Creator — (`x/nft/keeper/keeper.go`)

### Summary
`BurnNFT` enforces a conjunctive guard requiring the caller to be **both** the current NFT owner (`IsOwner`) **and** the denom creator (`IsDenomCreator`). Because `TransferOwner` freely moves ownership to any address without restriction, any NFT transferred away from the creator enters a state where **no address satisfies both conditions simultaneously**, making the NFT permanently unburnable and the collection supply counter permanently inflated.

### Finding Description

`BurnNFT` in `x/nft/keeper/keeper.go` lines 141–161 enforces two sequential guards:

```go
nft, err := k.IsOwner(ctx, denomID, tokenID, owner)   // caller must be current owner
if err != nil { return err }

_, err = k.IsDenomCreator(ctx, denomID, owner)         // caller must also be denom creator
if err != nil { return err }
``` [1](#0-0) 

`IsDenomCreator` compares `address` against `denom.Creator`, which is set once at denom issuance and **never updated**: [2](#0-1) 

`TransferOwner` only checks `IsOwner(srcOwner)` — no creator restriction — so it freely transfers to any address: [3](#0-2) 

After `TransferOwner(creator → victim)`:

| Caller | `IsOwner` | `IsDenomCreator` | `BurnNFT` result |
|--------|-----------|-----------------|-----------------|
| victim | ✓ (current owner) | ✗ (not creator) | **error** |
| creator | ✗ (no longer owner) | ✓ | **error** |

There is no other user-facing burn path. `BurnNFTUnverified` exists but is only called internally for IBC escrow mechanics, not exposed via any `MsgServer` method: [4](#0-3) 

The `MsgBurnNFT` handler routes exclusively through `k.Keeper.BurnNFT`: [5](#0-4) 

### Impact Explanation

- The NFT record persists in KV store indefinitely with no deletion path.
- `decreaseSupply` is never called, so `GetTotalSupply` for the denom is permanently inflated by one unit per such NFT.
- Neither the owner nor the creator can recover or destroy the asset.
- This is reachable via two standard production `MsgServer` calls: `MsgMintNFT` → `MsgTransferNFT` → `MsgBurnNFT` (fails for both parties). [6](#0-5) 

### Likelihood Explanation

Any creator who mints an NFT and transfers it (the primary intended use case for NFT marketplaces) triggers this condition. No special privileges, governance, or key compromise required — only two standard signed transactions.

### Recommendation

Remove the `IsDenomCreator` check from `BurnNFT`. Ownership alone should be sufficient authorization to burn. If the design intent is that only the creator can burn, then `TransferOwner` must be blocked or `BurnNFT` must allow the creator to burn regardless of current ownership. A minimal fix:

```go
func (k Keeper) BurnNFT(ctx sdk.Context, denomID, tokenID string, owner sdk.AccAddress) error {
    if !k.HasDenomID(ctx, denomID) {
        return sdkerrors.Wrapf(types.ErrInvalidDenom, "denom ID %s not exists", denomID)
    }
    nft, err := k.IsOwner(ctx, denomID, tokenID, owner)
    if err != nil {
        return err
    }
    // Remove IsDenomCreator check — owner authorization is sufficient
    k.deleteNFT(ctx, denomID, nft)
    k.deleteOwner(ctx, denomID, tokenID, owner)
    k.decreaseSupply(ctx, denomID)
    return nil
}
```

### Proof of Concept

```go
func TestBurnAfterTransferLocked(t *testing.T) {
    // setup keeper, ctx, creator, victim addresses
    k.IssueDenom(ctx, "testdenom", "Test", "", "", creator)
    k.MintNFT(ctx, "testdenom", "token1", "", "", "", creator, creator)
    // Transfer to victim
    err := k.TransferOwner(ctx, "testdenom", "token1", creator, victim)
    require.NoError(t, err)
    // Victim cannot burn (not creator)
    err = k.BurnNFT(ctx, "testdenom", "token1", victim)
    require.Error(t, err) // ErrUnauthorized: victim is not creator
    // Creator cannot burn (not owner)
    err = k.BurnNFT(ctx, "testdenom", "token1", creator)
    require.Error(t, err) // ErrUnauthorized: creator is not owner
    // Supply is permanently inflated
    require.Equal(t, uint64(1), k.GetTotalSupply(ctx, "testdenom"))
}

func (k Keeper) increaseSupply(ctx sdk.Context, denomID string) {
	supply := k.GetTotalSupply(ctx, denomID)
	supply++

	store := ctx.KVStore(k.storeKey)
	bz := types.MustMarshalSupply(k.cdc, supply)
	store.Set(types.KeyCollection(denomID), bz)
}

func (k Keeper) decreaseSupply(ctx sdk.Context, denomID string) {
	supply := k.GetTotalSupply(ctx, denomID)
	supply--

	store := ctx.KVStore(k.storeKey)
	if supply == 0 {
		store.Delete(types.KeyCollection(denomID))
		return
	}

	bz := types.MustMarshalSupply(k.cdc, supply)
	store.Set(types.KeyCollection(denomID), bz)
