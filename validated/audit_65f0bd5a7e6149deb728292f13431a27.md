### Title
Division-by-Zero in `PerpEngine.socializeSubaccount` When `openInterest == 0` Permanently Blocks Liquidation Finalization — (`core/contracts/PerpEngine.sol`)

---

### Summary

`PerpEngine.socializeSubaccount` unconditionally divides by `state.openInterest` when residual bad debt remains after insurance coverage. `MathSD21x18.div` explicitly reverts with `"DBZ"` when the divisor is zero. Because `state.openInterest` can legitimately be zero while a subaccount still carries a negative `vQuoteBalance`, the finalization path in `ClearinghouseLiq._finalizeSubaccount` permanently reverts, leaving the protocol with unresolvable bad debt.

---

### Finding Description

**The division site** — `PerpEngine.sol` lines 164–168:

```solidity
if (balance.vQuoteBalance < 0) {
    int128 fundingPerShare = -balance.vQuoteBalance.div(
        state.openInterest   // ← no zero-guard
    );
``` [1](#0-0) 

**`MathSD21x18.div` reverts on zero** — `MathSD21x18.sol` line 64:

```solidity
function div(int128 x, int128 y) internal pure returns (int128) {
    unchecked {
        require(y != 0, ERR_DIV_BY_ZERO);   // ← hard revert "DBZ"
``` [2](#0-1) 

**How `state.openInterest` is computed** — `getStateAndBalance` calls `_updateBalance(state, balance, 0, 0)`, which subtracts then re-adds `balance.amount.abs()`. When the subaccount being socialized has `balance.amount == 0` (required by `_finalizeSubaccount`), its contribution to `state.openInterest` is zero. If no other subaccount holds an open position in that product, `state.openInterest == 0`. [3](#0-2) 

**`_finalizeSubaccount` enforces `amount == 0` before calling `socializeSubaccount`:**

```solidity
require(balance.amount == 0, ERR_NOT_FINALIZABLE_SUBACCOUNT);
``` [4](#0-3) 

Then unconditionally calls:

```solidity
v.insurance = perpEngine.socializeSubaccount(txn.liquidatee, v.insurance);
``` [5](#0-4) 

---

### Impact Explanation

When `socializeSubaccount` reverts, `liquidateSubaccountImpl` reverts. The subaccount can never be finalized. The negative `vQuoteBalance` is never cleared. The protocol accumulates permanent bad debt with no recovery path, since every subsequent finalization attempt hits the same revert.

---

### Likelihood Explanation

The precondition is reachable through normal protocol operation with no privileged access:

1. SubA opens a long at price P₁; SubB opens a matching short at P₁. `openInterest = 2 * |X|`.
2. Price falls to P₂ < P₁.
3. Both SubA and SubB close their positions. `openInterest → 0`. SubA's `vQuoteBalance = (P₂ − P₁) * X < 0`. SubB's `vQuoteBalance = (P₁ − P₂) * X > 0`.
4. SubA's spot quote balance is also negative (or zero). SubA is under maintenance health.
5. A liquidator submits `LiquidateSubaccount` with `productId == type(uint32).max` to finalize SubA.
6. `_finalizeSubaccount` verifies `balance.amount == 0` ✓, attempts to settle negative PnL against zero quote balance (no-op), then calls `socializeSubaccount`.
7. Inside `socializeSubaccount`, insurance is insufficient, `balance.vQuoteBalance` remains negative, and `state.openInterest == 0` → `div` reverts with `"DBZ"`.

No admin keys, sequencer compromise, or governance capture required.

---

### Recommendation

Add a zero-guard before the division. If `state.openInterest == 0` and residual bad debt remains, there are no other participants to socialize against; the loss should be absorbed entirely by the insurance fund or written off:

```solidity
if (balance.vQuoteBalance < 0) {
    if (state.openInterest == 0) {
        // No open interest to socialize against; absorb or write off
        balance.vQuoteBalance = 0;
    } else {
        int128 fundingPerShare = -balance.vQuoteBalance.div(state.openInterest);
        state.cumulativeFundingLongX18 += fundingPerShare;
        state.cumulativeFundingShortX18 -= fundingPerShare;
        balance.vQuoteBalance = 0;
    }
}
```

### Proof of Concept

State-machine test (Hardhat/Foundry):

1. Deploy `PerpEngine` with one product (productId = 2).
2. Call `updateBalance(2, subA, +1e18, -100e18)` — SubA opens long, `vQuoteBalance = -100e18`.
3. Call `updateBalance(2, subA, -1e18, +80e18)` — SubA closes at a loss, `amount = 0`, `vQuoteBalance = -20e18`.
4. Verify `states[2].openInterest == 0` (no other open positions).
5. Call `socializeSubaccount(subA, 0)` (zero insurance).
6. Assert: transaction reverts with `"DBZ"`.

This proves the invariant that "socialization always completes without reverting" is broken under reachable on-chain state.
