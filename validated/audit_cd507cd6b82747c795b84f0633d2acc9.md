Audit Report

## Title
`SwapAllowlistExtension` checks router address instead of end user, enabling full allowlist bypass via `MetricOmmSimpleRouter` â€” (File: metric-periphery/contracts/extensions/SwapAllowlistExtension.sol)

## Summary

`SwapAllowlistExtension.beforeSwap` gates on the `sender` argument, which `MetricOmmPool.swap` sets to `msg.sender` â€” the direct caller of `pool.swap()`. When a user routes through `MetricOmmSimpleRouter`, the router contract is `msg.sender` at the pool, so the allowlist check becomes `allowedSwapper[pool][router]` instead of `allowedSwapper[pool][user]`. Any pool admin who allowlists the router to enable router-mediated swaps for their permitted users inadvertently opens the pool to every address on-chain.

## Finding Description

`MetricOmmPool.swap` passes `msg.sender` as the `sender` argument to `_beforeSwap`: [1](#0-0) 

`_beforeSwap` (via `ExtensionCalling`) forwards that value unchanged to every configured extension. `SwapAllowlistExtension.beforeSwap` then checks whether that `sender` is on the allowlist for the calling pool (`msg.sender` of the extension call = the pool): [2](#0-1) 

When a user calls `MetricOmmSimpleRouter.exactInputSingle`, the router is the direct caller of `pool.swap()`: [3](#0-2) 

So `sender` delivered to the extension is the **router address**, not the end user. The same structural problem exists for multi-hop `exactInput` (all hops call `pool.swap()` from the router): [4](#0-3) 

And for `exactOutput` recursive callback hops: [5](#0-4) 

This creates two mutually exclusive broken states:

1. **Router not allowlisted** â€” allowlisted users cannot use the router at all; their swaps revert with `NotAllowedToSwap` even though they are individually permitted.
2. **Router allowlisted** (the natural fix a pool admin would apply) â€” every address on-chain can swap through the router, completely defeating the allowlist.

No existing guard in `SwapAllowlistExtension` or `MetricOmmPool` recovers the original end-user identity. The `recipient` field is ignored by the extension check. There is no signed proof or forwarded caller identity mechanism in the router.

## Impact Explanation

A curated pool deploying `SwapAllowlistExtension` to restrict trading to a known set of addresses can be fully bypassed by any unprivileged user routing through `MetricOmmSimpleRouter`. The attacker pays no extra cost beyond normal gas. Every swap that should have been blocked executes at the oracle-derived bid/ask price, draining LP liquidity to unauthorized counterparties. This is a direct loss of LP principal and a complete failure of the pool's access-control invariant â€” matching the "Admin-boundary break" and "Broken core pool functionality causing loss of funds" allowed impacts.

## Likelihood Explanation

`MetricOmmSimpleRouter` is the standard periphery entry point for swaps. Pool admins who configure an allowlist and want their permitted users to use the router will allowlist the router address â€” the only apparent workaround â€” which immediately opens the pool to everyone. The trigger requires no special privilege, no unusual token behavior, and no multi-transaction setup: a single `exactInputSingle` call from any address suffices. The condition is highly likely to be reached in production.

## Recommendation

The extension must recover the original end-user identity rather than trusting the `sender` argument when an intermediary is involved:

1. **Preferred**: Redesign `MetricOmmSimpleRouter` to encode the real caller (`msg.sender`) into `extensionData` and have `SwapAllowlistExtension.beforeSwap` decode and verify it against a registry of trusted routers. This preserves composability without relying on `tx.origin`.
2. **Alternative**: Gate on `tx.origin` as a secondary signal â€” only appropriate if the pool admin explicitly accepts EOA-only swappers and the threat model excludes contract callers.
3. **Alternative**: Gate at the router level with a separate `SwapAllowlistRouter` that enforces `msg.sender` before calling the pool, removing the need for the extension to recover caller identity.

## Proof of Concept

