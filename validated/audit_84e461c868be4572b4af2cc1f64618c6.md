Audit Report

## Title
Deposit Allowlist Checks `owner` (LP Recipient) Instead of `sender` (Token Provider), Allowing Any Caller to Bypass Pool Curation — (`metric-periphery/contracts/extensions/DepositAllowlistExtension.sol`)

## Summary
`DepositAllowlistExtension.beforeAddLiquidity` silently discards the `sender` argument and gates on `owner` instead. Because `owner` is a free caller-supplied parameter in `MetricOmmPool.addLiquidity`, any unlisted address can bypass the allowlist by passing an allowlisted address as `owner`, injecting tokens into a curated pool while LP shares are minted to that allowlisted address.

## Finding Description
`MetricOmmPool.addLiquidity` accepts a caller-supplied `owner` parameter and passes both `msg.sender` (the actual token provider) and `owner` (the LP recipient) to the extension hook: [1](#0-0) 

`ExtensionCalling._beforeAddLiquidity` faithfully encodes both `sender` and `owner` into the call: [2](#0-1) 

However, `DepositAllowlistExtension.beforeAddLiquidity` declares the first parameter (`sender`) as unnamed and ignores it entirely, then gates only on `owner`: [3](#0-2) 

The token pull happens via callback on `msg.sender` (the actual caller), not on `owner`: [4](#0-3) 

LP shares are recorded under `owner`'s position key, not the caller's: [5](#0-4) 

`removeLiquidity` enforces `msg.sender == owner`, so the allowlisted address (not the attacker) controls the resulting position: [6](#0-5) 

**Attack path:**
1. Pool admin configures `DepositAllowlistExtension` with `allowedDepositor[pool][alice] = true` and `allowedDepositor[pool][attacker] = false`.
2. Attacker calls `pool.addLiquidity(owner=alice, salt, deltas, callbackData, "")`.
3. Extension receives `(sender=attacker, owner=alice)`, checks `allowedDepositor[pool][alice]` → `true` → passes.
4. Attacker's callback contract provides the tokens (tokens pulled from `msg.sender` = attacker).
5. LP shares are minted to `alice`'s position. Attacker's tokens are consumed. Pool curation is violated.

## Impact Explanation
The deposit allowlist is the primary mechanism for pool curation — it is supposed to ensure only trusted, vetted addresses can inject liquidity into a pool. The check on `owner` instead of `sender` completely nullifies this guarantee: any address can deposit into any allowlist-gated pool by naming an allowlisted address as `owner`. This constitutes a broken core pool functionality and an admin-boundary break: the pool admin's curation control is bypassed by any unprivileged caller with no special access required.

## Likelihood Explanation
The bypass requires only a single public call to `pool.addLiquidity` with a known allowlisted address as `owner`. No privileged access, no special token behavior, and no off-chain data are needed. Any observer of on-chain allowlist state (via `allowedDepositor` public mapping or `AllowedToDepositSet` events) can execute this immediately and repeatably. [7](#0-6) 

## Recommendation
Change the allowlist check to gate on `sender` (the actual token provider) rather than `owner` (the LP recipient):

```solidity
// Before (incorrect):
function beforeAddLiquidity(address, address owner, uint80, LiquidityDelta calldata, bytes calldata)
    external view override returns (bytes4)
{
    if (!allowAllDepositors[msg.sender] && !allowedDepositor[msg.sender][owner]) {

// After (correct):
function beforeAddLiquidity(address sender, address, uint80, LiquidityDelta calldata, bytes calldata)
    external view override returns (bytes4)
{
    if (!allowAllDepositors[msg.sender] && !allowedDepositor[msg.sender][sender]) {
```

## Proof of Concept

```solidity
function test_allowlistBypass_ownerVsSender() public {
    address allowlisted = makeAddr("allowlisted");
    address attacker    = makeAddr("attacker");

    vm.prank(admin);
    depositExtension.setAllowedToDeposit(address(pool), allowlisted, true);

    deal(address(token0), attacker, 1_000 ether);
    deal(address(token1), attacker, 1_000 ether);
    vm.startPrank(attacker);
    token0.approve(address(pool), type(uint256).max);
    token1.approve(address(pool), type(uint256).max);

    LiquidityDelta memory d = _deltaAbovePrice(4, 10_000);
    // attacker passes allowlisted as owner — extension checks allowlisted, passes
    pool.addLiquidity(allowlisted, 1, d, callbackData, "");
    vm.stopPrank();

    uint256 shares = stateView.positionBinShares(address(pool), allowlisted, 1, int8(4));
    assertGt(shares, 0, "allowlisted received LP shares");
    assertLt(token0.balanceOf(attacker) + token1.balanceOf(attacker), 2_000 ether);
}
```
