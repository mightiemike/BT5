### Title
Unchecked `.unwrap()` on `Option` dynamic-config fields after `None`-permissive schema validation in `ConfigManager` — (`crates/apollo_config_manager/src/config_manager.rs`)

---

### Summary

`ConfigManager` stores a `NodeDynamicConfig` whose every sub-config field is `Option<T>`. The schema-level validator (`validate_node_dynamic_config`) explicitly returns `Ok(())` whenever either of the two checked fields is `None`, meaning a `NodeDynamicConfig` with **all fields set to `None`** passes validation and is committed to the live store. Every `get_*_dynamic_config` accessor then calls `.unwrap()` on the stored `Option` without a prior `is_some()` guard, causing an unconditional panic the next time any consumer (consensus, mempool, gateway, context) requests its dynamic config.

---

### Finding Description

`NodeDynamicConfig` is defined with all sub-configs as `Option`:

```rust

    pub consensus_dynamic_config:        Option<ConsensusDynamicConfig>,
    pub context_dynamic_config:          Option<ContextDynamicConfig>,
    ...
}
``` [1](#0-0) 

The schema-level validator that guards `set_node_dynamic_config` is:

```rust
// crates/apollo_node_config/src/node_config.rs  L461-475

    else {
        return Ok(());   // ← silently accepts all-None

`set_node_dynamic_config` calls `node_dynamic_config.validate()` and, if it passes, atomically replaces the live store:

```rust
// crates/apollo_config_manager/src/config_manager.rs  L59-73
pub(crate) async fn set_node_dynamic_config(...) -> ConfigManagerResult<()> {
    if let Err(errors) = node_dynamic_config.validate() { ... return Err(...); }
    let mut config = self.latest_node_dynamic_config.write().await;
    *config = node_dynamic_config;   // ← all-None config committed here
    Ok(())
}
``` [3](#0-2) 

Every accessor then calls `.unwrap()` on the stored `Option` with no guard:

```rust
// crates/apollo_config_manager/src/config_manager.rs  L75-129
pub(crate) async fn get_consensus_dynamic_config(&self) -> ... {
    let config = self.latest_node_dynamic_config.read().await;
    Ok(config.consensus_dynamic_config.as_ref().unwrap().clone())  // panics if None
}
// identical pattern for get_class_manager_dynamic_config, get_context_dynamic_config,
// get_http_server_dynamic_config, get_mempool_dynamic_config, get_batcher_dynamic_config,
// get_state_sync_dynamic_config, get_staking_manager_dynamic_config
``` [4](#0-3) 

The `ConfigManagerRunner` pushes a new `NodeDynamicConfig` derived from the full `SequencerNodeConfig` on every file-watch tick. If the config file is written with `consensus_manager_config.#is_none = true` (a valid serialization the config system supports), `NodeDynamicConfig::from(&config)` produces `consensus_dynamic_config: None`, which passes validation and is committed. [5](#0-4) 

---

### Impact Explanation

After the all-`None` config is committed, the next call to any `get_*_dynamic_config` method panics the `ConfigManager` task. Because `ConfigManager` is a `ConcurrentLocalComponentServer`, the panic propagates through the tokio task and crashes the component. All downstream consumers — `run_consensus` (reads `validator_id` and timeout parameters at every height), `SequencerConsensusContext::update_dynamic_config` (reads gas-price and proposal-margin parameters), the mempool, and the gateway — lose access to live dynamic config. Consensus continues with the last-known stale `validator_id`, which may no longer match the node's actual identity in the committee, causing the node to silently skip proposer duties or cast votes under the wrong identity. This matches the **High** impact: "Mempool/gateway/RPC admission accepts invalid transactions or rejects valid transactions before sequencing."

---

### Likelihood Explanation

The trigger requires a config-file update that sets one or more top-level component configs to `None` (via the `#is_none` mechanism the config system already supports). This is analogous to the original report's "series of misconfigurations/failures." It does not require network access or cryptographic material — only the ability to write the node's config file, which is within the operator's normal update workflow. The `ConfigManagerRunner` polls for file changes and automatically pushes the result, so the window between a bad file write and the panic is bounded only by the poll interval.

---

### Recommendation

1. **In `validate_node_dynamic_config`**: replace the early-return-on-`None` with an explicit check that every field required by a running component is `Some`. Alternatively, make the fields non-optional and use a separate `PartialNodeDynamicConfig` for partial updates.

2. **In every `get_*_dynamic_config` accessor**: replace `.unwrap()` with a proper `ok_or` that returns a `ConfigManagerError::MissingConfig` variant instead of panicking:

```rust
pub(crate) async fn get_consensus_dynamic_config(&self) -> ConfigManagerResult<ConsensusDynamicConfig> {
    let config = self.latest_node_dynamic_config.read().await;
    config.consensus_dynamic_config.clone()
        .ok_or(ConfigManagerError::MissingConfig("consensus_dynamic_config"))
}
```

3. **In `set_node_dynamic_config`**: add a pre-commit check that no field transitions from `Some` to `None` unless the corresponding component is disabled.

---

### Proof of Concept

```rust
// Construct an all-None NodeDynamicConfig — passes validate() because
// validate_node_dynamic_config returns Ok(()) when fields are None.
let all_none = NodeDynamicConfig::default(); // all fields are None by Default
assert!(all_none.validate().is_ok());        // ← validation passes

let config_manager = ConfigManager::new(ConfigManagerConfig::default(), all_none);

// Any get_* call now panics:
// thread 'test' panicked at 'called `Option::unwrap()` on a `None` value'
let _ = config_manager.get_consensus_dynamic_config().await; // PANIC
```

The `NodeDynamicConfig::default()` produces all-`None` fields (the `#[derive(Default)]` on `Option<T>` yields `None`). The `validate()` call returns `Ok(())` because `validate_node_dynamic_config` exits early when `consensus_dynamic_config` is `None`. The subsequent `.unwrap()` in `get_consensus_dynamic_config` panics unconditionally. [6](#0-5) [7](#0-6)

fn validate_node_dynamic_config(config: &NodeDynamicConfig) -> Result<(), ValidationError> {
    let (Some(consensus), Some(context)) =
        (&config.consensus_dynamic_config, &config.context_dynamic_config)
    else {
        return Ok(());
    };
    let min_timeout = consensus.timeouts.get_proposal_timeout(0);
    let margin = context.build_proposal_margin_millis;
    if margin >= min_timeout {
        return Err(ValidationError::new(
            "build_proposal_margin_millis must be less than the base proposal timeout",
        ));
    }
    Ok(())
}

    pub(crate) async fn update_config(
        &mut self,
    ) -> Result<NodeDynamicConfig, Box<dyn std::error::Error + Send + Sync>> {
        let config = load_and_validate_config(self.cli_args.clone(), false).map_err(|e| {
            CONFIG_MANAGER_UPDATE_ERRORS.increment(1);
            error!("ConfigManagerRunner: failed to update config: {e}");
            e
        })?;
        let node_dynamic_config = NodeDynamicConfig::from(&config);

        // Compare the previous and the newly read node dynamic config.
        if self.latest_node_dynamic_config == node_dynamic_config {
            // No change, so no action is needed.
            Ok(node_dynamic_config)
        } else {
            // Log the diff between the latest and the new node dynamic config.
            self.log_config_diff(&self.latest_node_dynamic_config, &node_dynamic_config);
            // Update the latest node dynamic config.
            self.latest_node_dynamic_config = node_dynamic_config.clone();
            match self
                .config_manager_client
                .set_node_dynamic_config(node_dynamic_config.clone())
                .await
            {
                Ok(()) => {
                    info!("Successfully updated dynamic config");
                    Ok(node_dynamic_config)
                }
                Err(e) => Err(format!("Failed to update dynamic config: {:?}", e).into()),
            }
        }
    }
