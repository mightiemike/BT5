Audit Report

## Title
`SwapAllowlistExtension.beforeSwap` checks router address instead of end-user, enabling allowlist bypass for any caller — (File: `metric-periphery/contracts/extensions/SwapAllowlistExtension.sol`)

## Summary
`MetricOmmPool.swap` passes `msg.sender` (the router) as `sender` to `_beforeSwap`, which forwards it verbatim to `SwapAllowlistExtension.beforeSwap`. The extension evaluates `allowedSwapper[pool][router]` rather than `allowedSwapper[pool][end-user]`. Any pool admin who allowlists the router so that their curated users can swap through it simultaneously opens unrestricted swap access to every caller who routes through `MetricOmmSimpleRouter`.

## Finding Description
**Call chain:**

`MetricOmmPool.swap` captures `msg.sender` at L231 and passes it as `sender` to `_beforeSwap`: [1](#0-0) 

`ExtensionCalling._beforeSwap` encodes that `sender` value (= router) into the ABI call forwarded to every configured extension: [2](#0-1) 

`SwapAllowlistExtension.beforeSwap` then evaluates:
```solidity
if (!allowAllSwappers[msg.sender] && !allowedSwapper[msg.sender][sender]) {
    revert IMetricOmmPoolActions.NotAllowedToSwap();
}
```
where `msg.sender` = pool and `sender` = router — the original end-user address is never consulted: [3](#0-2) 

In `MetricOmmSimpleRouter.exactInputSingle`, the original caller (`msg.sender`) is stored only in transient callback context for payment purposes and is never forwarded to `pool.swap`: [4](#0-3) 

**Irreconcilable conflict:** A pool admin who wants allowlisted users (Alice) to swap through the router must call `setAllowedToSwap(pool, router, true)`. Once set, `allowedSwapper[pool][router]` is `true` for every call arriving through the router — regardless of who the actual caller is. Charlie (not individually allowlisted) calls `exactInputSingle`, the router calls `pool.swap`, the extension sees `sender = router`, the check passes, and the swap executes. There is no mechanism in the current extension interface for the extension to recover the true end-user address. [5](#0-4) 

## Impact Explanation
Broken core pool access-control functionality: any unprivileged user can execute swaps against a pool restricted by `SwapAllowlistExtension` by routing through `MetricOmmSimpleRouter`. The pool admin's intended per-user allowlist is fully unenforceable whenever the router is in the call path, allowing unauthorized users to extract value (arbitrage, front-running) reserved for a curated participant set.

## Likelihood Explanation
High. `MetricOmmSimpleRouter` is the primary user-facing swap entrypoint. Any pool admin who deploys `SwapAllowlistExtension` and wants their allowlisted users to use the router — a normal operational requirement — must allowlist the router. The bypass is triggered by ordinary, good-faith pool administration with no malicious precondition required.

## Recommendation
The extension must gate the original end-user, not the intermediary. Viable approaches:

1. **Encode original caller in `extensionData`**: The router encodes `msg.sender` (end-user) into `extensionData` forwarded to the pool. `SwapAllowlistExtension` decodes and checks that address, establishing a convention between router and extension.
2. **Check `recipient`**: If the economically relevant actor is the output recipient, the extension can check `recipient` instead of `sender`. This changes semantics for direct swaps where `sender != recipient`.
3. **Router-aware allowlist variant**: Introduce an extension that accepts a signed or encoded user identity from a known router address, verified on-chain.

At minimum, documentation for `SwapAllowlistExtension` must warn that allowlisting the router grants unrestricted swap access to all users.

## Proof of Concept
```
Setup:
  pool configured with SwapAllowlistExtension as beforeSwap hook
  pool admin: setAllowedToSwap(pool, Alice, true)   // Alice allowlisted
  pool admin: setAllowedToSwap(pool, router, true)  // router allowlisted so Alice can use it

Attack:
  Charlie (not allowlisted) calls:
    MetricOmmSimpleRouter.exactInputSingle({pool: pool, recipient: Charlie, ...})

  Router calls: pool.swap(Charlie, zeroForOne, amount, ..., extensionData)
    → pool passes msg.sender (= router) as sender to _beforeSwap
    → SwapAllowlistExtension.beforeSwap(sender = router, ...)
    → allowedSwapper[pool][router] == true  ✓
    → swap executes for Charlie despite Charlie not being allowlisted

Result:
  Charlie successfully swaps on a pool intended to restrict access to Alice only.
  The allowlist is fully bypassed for any user routing through MetricOmmSimpleRouter.
```
