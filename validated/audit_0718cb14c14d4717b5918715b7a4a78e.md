### Title
`SwapAllowlistExtension.beforeSwap` checks the router address instead of the actual swapper, allowing any user to bypass the per-user swap allowlist — (`metric-periphery/contracts/extensions/SwapAllowlistExtension.sol`)

---

### Summary

`SwapAllowlistExtension` is designed to gate `swap` by individual swapper address. However, `beforeSwap` checks the `sender` parameter, which is `msg.sender` of `pool.swap()`. When a user routes through `MetricOmmSimpleRouter`, the router is the direct caller of `pool.swap()`, so `sender = router`. The allowlist then checks `allowedSwapper[pool][router]` instead of `allowedSwapper[pool][actualUser]`. If the pool admin allowlists the router to enable periphery-based swaps, every user — including non-allowlisted ones — can bypass the per-user gate by routing through the router.

---

### Finding Description

`MetricOmmPool.swap()` passes `msg.sender` as the `sender` argument to `_beforeSwap`: [1](#0-0) 

`_beforeSwap` forwards that value verbatim to every configured extension: [2](#0-1) 

`SwapAllowlistExtension.beforeSwap` then gates on that `sender`: [3](#0-2) 

When a user calls `MetricOmmSimpleRouter.exactInputSingle`, the router is the direct caller of `pool.swap()`: [4](#0-3) 

So `sender` arriving at the extension is the **router address**, not the user. The allowlist check `allowedSwapper[pool][router]` is evaluated instead of `allowedSwapper[pool][user]`.

The same mismatch applies to `exactInput`, `exactOutputSingle`, and `exactOutput` — all of them call `pool.swap()` with `msg.sender = router`.

By contrast, `DepositAllowlistExtension.beforeAddLiquidity` correctly gates on `owner` (the actual position owner, explicitly passed through the call stack), not on `sender`: [5](#0-4) 

This inconsistency confirms the swap allowlist binds to the wrong actor.

---

### Impact Explanation

**High.** Two fund-impacting outcomes arise from the wrong-actor binding:

1. **Allowlist bypass (primary impact):** A pool admin who allowlists the router address — the natural step to enable periphery-based swaps on a curated pool — inadvertently opens the gate to every user. Any non-allowlisted address can call `router.exactInputSingle(...)` and the extension sees `sender = router` (allowlisted), so the swap proceeds. The per-user curation is completely defeated for all router-routed swaps.

2. **Broken core functionality (secondary impact):** A pool admin who allowlists individual user addresses (the intended usage) blocks those same users from swapping through the router, because the router address is not in the allowlist. The standard periphery path becomes unusable for allowlisted users.

Both outcomes violate the invariant that a curated pool must enforce the same allowlist policy regardless of which supported public entrypoint reaches it.

---

### Likelihood Explanation

**Medium.** The `MetricOmmSimpleRouter` is the primary supported swap entrypoint in the periphery layer. Any pool that deploys `SwapAllowlistExtension` and expects users to interact via the router will encounter this bug. The bypass path (calling the router) requires no special privilege — it is the normal user flow.

---

### Recommendation

The `beforeSwap` hook must gate on the actual economic actor, not the intermediary. Two options:

1. **Preferred:** Mirror the `addLiquidity` pattern — add an explicit `swapper` parameter to `pool.swap()` (analogous to `owner` in `addLiquidity`) so the router can forward the originating user address. The extension then checks `allowedSwapper[pool][swapper]`.

2. **Short-term:** Document that `SwapAllowlistExtension` is incompatible with router-based flows and require direct `pool.swap()` calls only, or gate by allowlisting the router and enforcing user identity off-chain.

---

### Proof of Concept

```
1. Deploy a pool with SwapAllowlistExtension configured on beforeSwap.
2. Pool admin calls swapExtension.setAllowedToSwap(pool, router, true)
   — intending to enable router-based swaps.
3. Non-allowlisted user Alice calls:
     router.exactInputSingle({pool: pool, ..., extensionData: ""})
4. Router calls pool.swap(recipient, ...) — msg.sender = router.
5. Pool calls _beforeSwap(msg.sender=router, recipient, ...).
6. SwapAllowlistExtension.beforeSwap receives sender=router.
7. Check: allowedSwapper[pool][router] == true → passes.
8. Alice's swap executes despite never being individually allowlisted.

Alternatively, if the admin allowlists Alice directly
(setAllowedToSwap(pool, alice, true)) but NOT the router:
- Alice calls router.exactInputSingle(...).
- Extension sees sender=router, checks allowedSwapper[pool][router] == false → reverts.
- Alice is blocked from using the standard periphery path even though she is allowlisted.
``` [3](#0-2) [4](#0-3) [1](#0-0)
