### Title
Integer Multiplication Overflow in `op_unknown` Cost Calculation Allows Cost Metering Bypass - (File: src/more_ops.rs)

### Summary
The `op_unknown` function in `src/more_ops.rs` computes the final cost of an unknown opcode by multiplying an accumulated cost by `(cost_multiplier + 1)`. This multiplication is performed on `u64` values without overflow protection. In a Rust release build, the multiplication silently wraps, producing a cost value far below the true cost. The subsequent guard (`if cost > u32::MAX as u64`) can be bypassed when the wrapped result is ≤ `u32::MAX`, causing the function to return `Ok(Reduction(0, nil))` — zero cost — for an operation whose true cost would exceed `max_cost` and should be rejected.

### Finding Description

In `op_unknown`, after computing a base cost from the opcode's arguments (bounded by `check_cost` to ≤ `max_cost`), the code multiplies by the opcode-encoded multiplier:

```rust
check_cost(cost, max_cost)?;
cost *= cost_multiplier + 1;          // line 261 — no overflow check
if cost > u32::MAX as u64 {
    Err(EvalErr::Invalid(o))?
} else {
    Ok(Reduction(cost as Cost, allocator.nil()))
}
```

`cost_multiplier` is a `u64` derived from up to 4 bytes of the opcode atom via `u32_from_u8`, so it ranges from 0 to `u32::MAX` (4,294,967,295). `cost_multiplier + 1` therefore ranges from 1 to 2^32. The pre-multiplication `cost` can be up to `max_cost` (≈ 11 billion in Chia consensus), which is larger than 2^32.

When `cost = 2^33 = 8,589,934,592` and `cost_multiplier = 2^31 − 1 = 2,147,483,647`:

```
cost * (cost_multiplier + 1) = 2^33 * 2^31 = 2^64 ≡ 0  (mod 2^64, wrapping)
```

The guard sees `0 > u32::MAX` → false, and returns `Ok(Reduction(0, nil))`.

The opcode bytes that produce `cost_multiplier = 2^31 − 1` are `[0x7f, 0xff, 0xff, 0xff, 0xXX]`. This does **not** start with `0xffff`, so the reserved-opcode check does not fire.

To make the pre-multiplication cost equal exactly `2^33 = 8,589,934,592` using `cost_function = 1` (arithmetic-like):

```
cost = ARITH_BASE_COST + 1 * ARITH_COST_PER_ARG + s * ARITH_COST_PER_BYTE
     = 99 + 320 + 3s = 8,589,934,592
     → s = 2,863,311,391 bytes
```

A single atom of 2,863,311,391 bytes is within the allocator's `u32::MAX`-byte heap limit. The `check_cost` call inside the loop passes because `8,589,934,592 ≤ max_cost`.

### Impact Explanation

An attacker-controlled CLVM program can invoke an unknown opcode whose true cost (pre-overflow) is `2^64`, which should be rejected as exceeding `max_cost`. Instead, the wrapped cost is 0, the call succeeds, and the operation is accepted with zero cost charged. This bypasses the cost metering that enforces block-size limits in Chia consensus. Nodes running release builds silently accept the program; nodes running debug builds panic. This is a concrete consensus-divergence and undercharged-execution impact.

### Likelihood Explanation

The attacker fully controls both the opcode bytes (setting `cost_multiplier`) and the argument list (setting the pre-multiplication cost). No privileged access is required. The only practical constraint is constructing a CLVM program containing a multi-gigabyte atom, which is large but within protocol limits. The overflow is silent in release builds (Rust's default wrapping semantics), making it undetectable without explicit checked arithmetic.

### Recommendation

Replace the unchecked multiplication with a checked or saturating variant. If the product overflows or exceeds `u32::MAX`, return `EvalErr::Invalid`:

```rust
let final_cost = cost
    .checked_mul(cost_multiplier + 1)
    .filter(|&c| c <= u32::MAX as u64)
    .ok_or(EvalErr::Invalid(o))?;
Ok(Reduction(final_cost as Cost, allocator.nil()))
```

### Proof of Concept

Craft a 5-byte unknown opcode `[0x7f, 0xff, 0xff, 0xff, 0x40]`:
- Bytes 0–3: `cost_multiplier = 0x7fffffff = 2,147,483,647`; `cost_multiplier + 1 = 2^31`
- Byte 4 bits 7–6: `01` → `cost_function = 1` (arithmetic-like)

Pass one atom argument of exactly 2,863,311,391 bytes. The loop computes:

```
cost = 99 + 320 + 2,863,311,391 * 3 = 8,589,934,592 = 2^33
check_cost(8,589,934,592, 11_000_000_000) → Ok   // passes
cost *= 2^31  →  2^33 * 2^31 = 2^64 ≡ 0  (u64 wrapping)
0 > u32::MAX  →  false
return Ok(Reduction(0, nil))   // zero cost, program accepted
```
