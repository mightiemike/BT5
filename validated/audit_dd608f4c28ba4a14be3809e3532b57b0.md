### Title
NFT Transfer to Module Account Address Permanently Locks NFT - (File: x/nft/types/msgs.go)

### Summary

`MsgTransferNFT.ValidateBasic()` and `MsgMintNFT.ValidateBasic()` validate only that the recipient is a syntactically valid bech32 address. Neither the message validation layer nor the NFT keeper's `TransferOwner`/`MintNFT` execution path checks whether the recipient is a module account address. An NFT sent to any module account address (e.g., the nft-transfer escrow address, the tieredrewards rewards pool, the fee collector) is permanently locked there with no recovery path, because module accounts have no private key and the NFT module has no equivalent of the bank module's `BlockedAddresses` guard.

### Finding Description

`MsgTransferNFT.ValidateBasic()` accepts any valid bech32 address as the recipient:

```go
func (msg MsgTransferNFT) ValidateBasic() error {
    if err := ValidateDenomID(msg.DenomId); err != nil { return err }
    if _, err := sdk.AccAddressFromBech32(msg.Sender); err != nil { ... }
    if _, err := sdk.AccAddressFromBech32(msg.Recipient); err != nil { ... }
    return ValidateTokenID(msg.Id)
}
```

The message server then calls `TransferOwner` directly with the parsed recipient:

```go
func (m msgServer) TransferNFT(...) {
    recipient, err := sdk.AccAddressFromBech32(msg.Recipient)
    ...
    if err := m.TransferOwner(ctx, msg.DenomId, msg.Id, sender, recipient); err != nil { ... }
}
```

`TransferOwner` updates the NFT's owner field in the KV store without any module-account guard. The same gap exists in `MsgMintNFT.ValidateBasic()` and in `processReceivedPacket` inside the IBC nft-transfer module, where the receiver address decoded from the IBC packet data is passed directly to `MintNFT` or `TransferOwner` without a module-account check.

The Cosmos SDK bank module protects against this class of mistake via `BlockedAddresses`, which rejects `SendCoins` to module accounts. The NFT module has no analogous protection.

### Impact Explanation

An NFT transferred to a module account address is permanently lost. The NFT owner field in the store is set to the module account address; no private key exists to sign a subsequent transfer out. The corrupted invariant is the NFT owner record: it points to an address that can never sign a `MsgTransferNFT`, making the token irrecoverable. This is a direct user-fund loss (NFT asset loss).

The IBC path is the most severe variant: when a user on chain A sends an IBC NFT transfer to chain B with the receiver set to a module account address on chain B, the NFT is escrowed or burned on chain A and minted to the module account on chain B. Both the source-chain escrow and the destination-chain NFT are permanently inaccessible.

### Likelihood Explanation

The likelihood is low but realistic. Module account addresses on Cronos POS Chain are deterministic and publicly derivable (e.g., the nft-transfer escrow address for a given port/channel, the tieredrewards `RewardsPoolName` address). A user who mistakenly copies a module account address as the recipient, or a cross-chain application that programmatically constructs a receiver address with a bug, can trigger this path with a normal signed transaction. No privileged role is required.

### Recommendation

Add a module-account blocklist check in `MsgTransferNFT.ValidateBasic()`, `MsgMintNFT.ValidateBasic()`, and in `processReceivedPacket` before calling `MintNFT`/`TransferOwner`. The check should mirror the bank module's `BlockedAddresses` pattern: reject any recipient whose address matches a registered module account. Alternatively, add the guard inside the NFT keeper's `TransferOwner` and `MintNFT` functions so that all callers (including IBC) are protected uniformly.

### Proof of Concept

1. Derive the nft-transfer escrow address for channel-0: `types.GetEscrowAddress("nft-transfer", "channel-0")`.
2. Sign and broadcast a `MsgTransferNFT` with `Recipient` set to that escrow address.
3. `ValidateBasic()` passes (valid bech32). The message server calls `TransferOwner`, updating the NFT owner to the escrow address.
4. The NFT is now owned by the escrow module account. No private key exists to sign a recovery transfer. The NFT is permanently locked.

The same steps apply to `MsgMintNFT` (denom creator mints directly to a module account) and to the IBC receive path (`processReceivedPacket` with `data.Receiver` set to a module account bech32 string).

func (k Keeper) processReceivedPacket(ctx sdk.Context, packet channeltypes.Packet,
	data types.NonFungibleTokenPacketData,
) error {
	receiver, err := sdk.AccAddressFromBech32(data.Receiver)
	if err != nil {
		return err
	}

	isAwayFromOrigin := types.IsAwayFromOrigin(packet.GetSourcePort(), packet.GetSourceChannel(), data.ClassId)

	// create the escrow address for creating denom and minting nft
	escrowAddress := types.GetEscrowAddress(packet.GetDestPort(), packet.GetDestChannel())

	if isAwayFromOrigin {
		// since SendPacket did not prefix the classID, we must prefix classID here
		classPrefix := types.GetClassPrefix(packet.GetDestPort(), packet.GetDestChannel())
		// NOTE: sourcePrefix contains the trailing "/"
		prefixedClassID := classPrefix + data.ClassId

		// construct the class trace from the full raw classID
		classTrace := types.ParseClassTrace(prefixedClassID)
		if !k.HasClassTrace(ctx, classTrace.Hash()) {
			k.SetClassTrace(ctx, classTrace)
		}

		voucherClassID := classTrace.IBCClassID()

		if !k.nftKeeper.HasDenomID(ctx, voucherClassID) {
			if err := k.nftKeeper.IssueDenom(ctx, voucherClassID, voucherClassID, "", data.ClassUri, escrowAddress); err != nil {
				return err
			}
		}
		sdkCtx := sdk.UnwrapSDKContext(ctx)
		sdkCtx.EventManager().EmitEvent(
			sdk.NewEvent(
				types.EventTypeClassTrace,
				sdk.NewAttribute(types.AttributeKeyTraceHash, classTrace.Hash().String()),
				sdk.NewAttribute(types.AttributeKeyClassID, voucherClassID),
			),
		)

		for i, tokenID := range data.TokenIds {
			if err := k.nftKeeper.MintNFT(ctx, voucherClassID, tokenID, "", data.TokenUris[i], "", escrowAddress, receiver); err != nil {
				return err
			}
		}
	} else {
		// If the token moves in the direction of back to origin,
		// we need to unescrow the token and transfer it to the receiver

		// we should remove the prefix. For example:
		// p6/c6/p4/c4/p2/c2/nftClass -> p4/c4/p2/c2/nftClass
		unprefixedClassID := types.RemoveClassPrefix(packet.GetSourcePort(),
			packet.GetSourceChannel(), data.ClassId)

		voucherClassID := types.ParseClassTrace(unprefixedClassID).IBCClassID()
		for _, tokenID := range data.TokenIds {
			if err := k.nftKeeper.TransferOwner(ctx,
				voucherClassID, tokenID, escrowAddress, receiver); err != nil {
				return err
			}
		}
	}
