### Title
Deprecated `verify_transaction_inclusion` Remains Publicly Callable with Incomplete Proof Validation — (File: `contract/src/lib.rs`)

---

### Summary

`verify_transaction_inclusion` is marked `#[deprecated]` since v0.5.0 because it omits the coinbase-proof leaf-node check needed to defeat the 64-byte Merkle forgery attack. Despite the deprecation, the function remains a live, publicly callable `#[pause]`-gated endpoint. Any unprivileged NEAR caller — including a recipient dApp — can invoke it directly and receive a `true` result for a forged proof, bypassing the fix introduced in `verify_transaction_inclusion_v2`.

---

### Finding Description

`verify_transaction_inclusion` is defined and appears fully functional, but its implementation is incomplete with respect to its security contract: it never validates that the supplied `tx_id` is a leaf node (a real transaction hash) rather than an internal Merkle-tree node.

The function's own docstring acknowledges this:

> "This function may return `true` if the provided `tx_id` is a hash of an internal node in the Merkle tree rather than a valid transaction hash." [1](#0-0) 

The function is still decorated with `#[pause]` (not removed, not access-controlled beyond the pause flag), making it reachable by any NEAR account: [2](#0-1) 

The fix — requiring a coinbase Merkle proof to anchor the tree's leftmost leaf — exists only in `verify_transaction_inclusion_v2`: [3](#0-2) 

`verify_transaction_inclusion_v2` itself calls the broken v1 internally after the coinbase check passes, so the coinbase check is the only guard. A caller who bypasses v2 and calls v1 directly gets no such guard. [4](#0-3) 

---

### Impact Explanation

A recipient smart contract that calls `verify_transaction_inclusion` (v1) can be deceived into accepting a forged transaction-inclusion proof. The attacker supplies an internal Merkle-tree node hash as `tx_id` together with a valid sibling path for that node; the function computes the root correctly and returns `true`, even though no real transaction with that ID exists in the block. This corrupts the proof-verification result consumed by any downstream contract.

---

### Likelihood Explanation

The attack is well-documented (https://www.bitmex.com/blog/64-Byte-Transactions) and requires no privileged role — only the ability to call a public NEAR contract method and supply crafted `ProofArgs`. Any relayer-path user or dApp integrator who reads the ABI and calls v1 instead of v2 is exposed.

---

### Recommendation

Remove `verify_transaction_inclusion` from the public ABI entirely, or gate it with an access-control role that prevents unprivileged callers from reaching it. The replacement `verify_transaction_inclusion_v2` should be the sole public verification endpoint.

---

### Proof of Concept

1. Identify any Bitcoin block whose Merkle tree has at least two transactions (so internal nodes exist).
2. Pick any internal node `N = SHA256d(left || right)` where `left` and `right` are 32-byte child hashes.
3. Construct a `ProofArgs` with:
   - `tx_id = N`
   - `tx_block_blockhash` = the block's hash (already in the contract's `headers_pool`)
   - `tx_index` = the index that `N` occupies in the tree
   - `merkle_proof` = the sibling path from `N` up to the Merkle root
   - `confirmations` = any value ≤ `gc_threshold`
4. Call `verify_transaction_inclusion(args)` on the NEAR contract.
5. The function computes `compute_root_from_merkle_proof(N, index, proof)`, which equals the stored `merkle_root`, and returns `true` — despite no real transaction with ID `N` existing. [5](#0-4)

    ///
    /// # Warning
    /// This function may return `true` if the provided `tx_id` is a hash of an internal node in the Merkle tree rather than a valid transaction hash.
    /// We assume that validation of whether the `tx_id` corresponds to a valid transaction hash is performed at a higher level of verification.
    ///
    /// # Panics
    /// Multiple cases
    #[deprecated(
        since = "0.5.0",
        note = "Use `verify_transaction_inclusion_v2` instead."
    )]
    #[pause]
    pub fn verify_transaction_inclusion(&self, #[serializer(borsh)] args: ProofArgs) -> bool {
        require!(
            args.confirmations <= self.gc_threshold,
            "The required number of confirmations exceeds the number of blocks stored in memory"
        );

        let heaviest_block_header = self
            .headers_pool
            .get(&self.mainchain_tip_blockhash)
            .unwrap_or_else(|| env::panic_str(ERR_KEY_NOT_EXIST));
        let target_block_height = self
            .mainchain_header_to_height
            .get(&args.tx_block_blockhash)
            .unwrap_or_else(|| env::panic_str("block does not belong to the current main chain"));

        // Check requested confirmations. No need to compute proof if insufficient confirmations.
        require!(
            (heaviest_block_header.block_height).saturating_sub(target_block_height) + 1
                >= args.confirmations,
            "Not enough blocks confirmed"
        );

        let header = self
            .headers_pool
            .get(&args.tx_block_blockhash)
            .unwrap_or_else(|| env::panic_str("cannot find requested transaction block"));

        require!(!args.merkle_proof.is_empty(), "Merkle proof is empty");

        // compute merkle tree root and check if it matches block's original merkle tree root
        merkle_tools::compute_root_from_merkle_proof(
            args.tx_id,
            usize::try_from(args.tx_index).unwrap(),
            &args.merkle_proof,
        ) == header.block_header.merkle_root
    }

    pub fn verify_transaction_inclusion_v2(&self, #[serializer(borsh)] args: ProofArgsV2) -> bool {
        require!(
            args.merkle_proof.len() == args.coinbase_merkle_proof.len(),
            "Coinbase merkle proof and transaction merkle proof should have the same length"
        );

        let header = self
            .headers_pool
            .get(&args.tx_block_blockhash)
            .unwrap_or_else(|| env::panic_str("cannot find requested transaction block"));

        require!(
            merkle_tools::compute_root_from_merkle_proof(
                args.coinbase_tx_id.clone(),
                0usize,
                &args.coinbase_merkle_proof,
            ) == header.block_header.merkle_root,
            "Incorrect coinbase merkle proof"
        );

        #[allow(deprecated)]
        self.verify_transaction_inclusion(args.into())
    }
