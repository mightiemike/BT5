Audit Report

## Title
Pool Admin Can Reduce Timelock to Zero and Immediately Disable Stop-Loss Protection â€” (`metric-periphery/contracts/extensions/OracleValueStopLossExtension.sol`)

## Summary
The `OracleValueStopLossExtension` uses a per-pool timelock to delay parameter changes, giving LPs a reaction window. However, the pool admin can reduce the timelock to zero by waiting the current timelock once. After that, all subsequent parameter changes â€” including setting `drawdownE6 = 0` â€” can be proposed and executed atomically in the same block, permanently eliminating LP protection.

## Finding Description
`proposeOracleStopLossTimelock` computes `executeAfter = _afterTimelock(pool_)` which returns `block.timestamp + oracleStopLossConfig[pool_].timelock` â€” the **current** timelock value. [1](#0-0) 

After waiting that duration, `executeOracleStopLossTimelock` sets `oracleStopLossConfig[pool_].timelock = 0`. [2](#0-1) 

From that point, `proposeOracleStopLossDrawdown(pool, 0)` stores `pendingDrawdownExecuteAfter = block.timestamp + 0 = block.timestamp`. [3](#0-2) 

`executeOracleStopLossDrawdown` then calls `_requireElapsed(block.timestamp)`, which checks `block.timestamp < block.timestamp` â€” this is `false`, so it does not revert. `drawdownE6` is set to `0` in the same block. [4](#0-3) [5](#0-4) 

In `_afterSwapOracleStopLoss`, the very first check is `if (drawdown == 0) return;`, which causes the entire watermark/breach logic to be skipped for all future swaps. [6](#0-5) 

There is no minimum timelock validation â€” `_validateDrawdown` and `_validateDecay` enforce upper bounds, but no analogous `_validateTimelock` exists, so `timelock = 0` is accepted at both initialization and via the propose/execute flow. [7](#0-6) 

## Impact Explanation
With `drawdownE6 = 0`, `_afterSwapOracleStopLoss` returns immediately on every swap â€” the watermark/drawdown breach check is entirely skipped. LPs are exposed to unlimited value extraction through adversarial swaps at manipulated oracle prices with no on-chain protection. The pool admin can also reset watermarks to zero immediately after disabling the timelock. This is a direct admin-boundary break: the timelock was the only mechanism preventing the pool admin from making instant, LP-hostile parameter changes, and it has been permanently eliminated.

## Likelihood Explanation
Any pool admin willing to wait the initial timelock period (e.g., 1 day) can execute this deterministically. No external conditions, oracle manipulation, or special token behavior are required. The attack is permanent â€” once the timelock is 0, LPs cannot restore it. The steps are fully on-chain and repeatable across any pool using this extension.

## Recommendation
Enforce a non-zero minimum timelock floor at the protocol level (e.g., a `MIN_TIMELOCK` constant) validated in both `initialize` and `executeOracleStopLossTimelock`. Alternatively, require that any timelock reduction waits `max(current_timelock, new_timelock)` before taking effect, so reducing to 0 requires waiting 0 seconds under the new value â€” but the correct fix is a hard protocol-level floor that cannot be set to zero by any admin.

## Proof of Concept
```solidity
// 1. Pool initialized with timelock=86400, drawdown=500_000
extension.initialize(pool, abi.encode(uint32(500_000), uint32(0), uint32(86400)));

// 2. Pool admin proposes timelock=0 (executeAfter = now + 86400)
vm.prank(admin);
extension.proposeOracleStopLossTimelock(pool, 0);

// 3. Wait the current timelock
vm.warp(block.timestamp + 86400);

// 4. Execute â€” timelock is now 0
vm.prank(admin);
extension.executeOracleStopLossTimelock(pool);
(,, uint32 tl,) = extension.oracleStopLossConfig(pool);
assertEq(tl, 0);

// 5. Same block: propose + execute drawdown=0
vm.startPrank(admin);
extension.proposeOracleStopLossDrawdown(pool, 0);   // executeAfter = block.timestamp
extension.executeOracleStopLossDrawdown(pool);       // block.timestamp < block.timestamp == false â†’ passes
vm.stopPrank();
(uint32 d,,,) = extension.oracleStopLossConfig(pool);
assertEq(d, 0);

// 6. Stop-loss disabled: afterSwap returns immediately, all swaps pass unconditionally
```
