Audit Report

## Title
`SwapAllowlistExtension.beforeSwap` checks router address as swapper identity, enabling full allowlist bypass for any router caller - (File: metric-periphery/contracts/extensions/SwapAllowlistExtension.sol)

## Summary
`SwapAllowlistExtension.beforeSwap` gates swaps by checking `allowedSwapper[pool][sender]`, where `sender` is `msg.sender` of `pool.swap()`. When swaps are routed through `MetricOmmSimpleRouter`, `sender` is always the router contract address, not the actual user. A pool admin who allowlists the router address (the only way to let allowlisted users trade through the router) simultaneously opens the pool to every unprivileged user who calls the router, completely defeating the per-user allowlist.

## Finding Description

**Root cause — identity mismatch between router and actual user:**

`MetricOmmPool.swap` passes `msg.sender` as the `sender` argument to `_beforeSwap`:

```solidity
// metric-core/contracts/MetricOmmPool.sol:230-240
_beforeSwap(
    msg.sender,   // ← caller of pool.swap(), i.e. the router
    recipient,
    ...
);
```

`ExtensionCalling._beforeSwap` forwards this verbatim to the extension:

```solidity
// metric-core/contracts/ExtensionCalling.sol:160-176
_callExtensionsInOrder(
    BEFORE_SWAP_ORDER,
    abi.encodeCall(IMetricOmmExtensions.beforeSwap, (sender, recipient, ...))
);
```

`SwapAllowlistExtension.beforeSwap` then checks:

```solidity
// metric-periphery/contracts/extensions/SwapAllowlistExtension.sol:37
if (!allowAllSwappers[msg.sender] && !allowedSwapper[msg.sender][sender]) {
    revert IMetricOmmPoolActions.NotAllowedToSwap();
}
```

Here `msg.sender` is the pool and `sender` is whoever called `pool.swap()`.

**Router call path:**

`MetricOmmSimpleRouter.exactInputSingle` calls `pool.swap(...)` directly:

```solidity
// metric-periphery/contracts/MetricOmmSimpleRouter.sol:72-80
IMetricOmmPoolActions(params.pool).swap(
    params.recipient,
    params.zeroForOne,
    ...,
    params.extensionData
);
```

The router does not forward the original `msg.sender` (the actual user) anywhere into the pool call. The pool sees `msg.sender = router`. The extension therefore evaluates `allowedSwapper[pool][router]`, not `allowedSwapper[pool][actual_user]`.

**Exploit flow:**

1. Pool admin deploys a pool with `SwapAllowlistExtension` configured as a `beforeSwap` hook, intending to restrict swaps to KYC'd addresses.
2. Pool admin allowlists specific user addresses: `setAllowedToSwap(pool, alice, true)`.
3. To let Alice trade through the router, the admin must also allowlist the router: `setAllowedToSwap(pool, router, true)`.
4. Any unprivileged user (Bob, who is not allowlisted) calls `router.exactInputSingle(...)` targeting the pool.
5. The extension checks `allowedSwapper[pool][router]` → `true` → swap proceeds. Bob bypasses the allowlist entirely.

**Existing guards are insufficient:**

The router validates that the pool is factory-registered and that the swap callback comes from the expected pool, but it performs no identity forwarding to the extension layer. The extension has no mechanism to recover the original caller's address — `extensionData` is opaque and user-controlled, so it cannot be trusted as an identity proof.

## Impact Explanation

An unprivileged, non-allowlisted user can execute swaps on a pool that the pool admin intended to restrict to specific addresses. This is a direct admin-boundary break: the pool admin's access-control configuration is bypassed by routing through a public intermediary contract. The allowlist — the sole mechanism for restricting swap access on such pools — provides no protection for router-mediated swaps once the router address is allowlisted.

## Likelihood Explanation

The condition is realistic and likely in production: any pool admin who wants allowlisted users to trade through the standard router must allowlist the router address. This is the natural, expected configuration. Once that single admin action is taken, the bypass is unconditionally available to any address that calls the router — no special privileges, no front-running, no timing dependency. The attack is repeatable on every block.

## Recommendation

The extension must gate on the actual end-user identity, not the intermediary. Two viable approaches:

1. **Router forwards original caller via `extensionData`**: The router encodes `msg.sender` into `extensionData` and the extension verifies it against a trusted router registry. This requires the extension to trust the router's encoding.
2. **Pool passes `tx.origin` or a separate `initiator` field**: Extend the `beforeSwap` hook signature to include an `initiator` address distinct from `sender`, populated by the pool from a router-supplied context (e.g., via transient storage set before the pool call). The extension then gates on `initiator`.

The simplest safe fix is for `SwapAllowlistExtension` to reject calls where `sender` is a known router unless the actual user identity is verifiably embedded and authenticated in `extensionData`.

## Proof of Concept

```solidity
// Foundry test sketch
function test_swapAllowlist_bypass_via_router() public {
    // Setup: pool with SwapAllowlistExtension as beforeSwap hook
    SwapAllowlistExtension ext = new SwapAllowlistExtension(address(factory));
    MetricOmmPool pool = _deployPoolWithExtension(address(ext), beforeSwapOrder);
    ext.bindPool(address(pool));
    _addLiquidity(...);

    address alice = makeAddr("alice");
    address bob   = makeAddr("bob");   // NOT allowlisted

    // Admin allowlists alice and the router (so alice can use the router)
    vm.prank(poolAdmin);
    ext.setAllowedToSwap(address(pool), alice, true);
    vm.prank(poolAdmin);
    ext.setAllowedToSwap(address(pool), address(router), true);

    // Bob (not allowlisted) swaps through the router — should revert, but does not
    vm.prank(bob);
    router.exactInputSingle(IMetricOmmSimpleRouter.ExactInputSingleParams({
        pool: address(pool),
        tokenIn: address(token0),
        tokenOut: address(token1),
        zeroForOne: true,
        amountIn: 1000,
        amountOutMinimum: 0,
        recipient: bob,
        deadline: block.timestamp + 1,
        priceLimitX64: 0,
        extensionData: ""
    }));
    // Bob receives tokens — allowlist bypassed
}
```
