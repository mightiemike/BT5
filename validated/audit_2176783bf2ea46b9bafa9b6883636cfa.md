Audit Report

## Title
SwapAllowlistExtension Gates the Router Address Instead of the End User, Enabling Full Allowlist Bypass via MetricOmmSimpleRouter — (`metric-periphery/contracts/extensions/SwapAllowlistExtension.sol`)

## Summary

`SwapAllowlistExtension.beforeSwap` checks the `sender` argument, which resolves to `msg.sender` of the `pool.swap` call. When a user routes through `MetricOmmSimpleRouter`, the router is `msg.sender`, so the extension gates the router address rather than the actual end user. A pool admin who allowlists the router to support normal swap UX inadvertently grants every on-chain user access to the curated pool, completely defeating the allowlist.

## Finding Description

**Step 1 — Pool captures `msg.sender` and passes it as `sender` to `_beforeSwap`:**

`MetricOmmPool.swap` at lines 230–240 calls `_beforeSwap(msg.sender, recipient, ...)`. [1](#0-0) 

**Step 2 — `ExtensionCalling._beforeSwap` forwards `sender` verbatim to the hook:**

The value is encoded directly into the `beforeSwap` hook call without modification. [2](#0-1) 

**Step 3 — `SwapAllowlistExtension.beforeSwap` checks `allowedSwapper[pool][sender]`:**

`msg.sender` here is the pool; `sender` is the immediate caller of `pool.swap`. The check is `allowedSwapper[pool][sender]`. [3](#0-2) 

**Step 4 — `MetricOmmSimpleRouter.exactInputSingle` calls `pool.swap` directly, making the router `msg.sender`:**

The router calls `IMetricOmmPoolActions(params.pool).swap(params.recipient, ...)` with no user identity forwarded. [4](#0-3) 

**Consequence:** When the pool admin allowlists the router (e.g., `setAllowedToSwap(pool, router, true)`) to enable standard UX, `allowedSwapper[pool][router]` becomes `true`. Any unpermissioned user who calls the router passes the check because the extension sees `sender = router`, not the actual end user.

**Asymmetry with `DepositAllowlistExtension`:** The deposit extension correctly gates `owner` (the second parameter, the economically relevant actor), not `sender` (the immediate caller). [5](#0-4) 

## Impact Explanation

A curated pool using `SwapAllowlistExtension` to restrict trading to KYC'd or permissioned addresses is fully bypassed. Any unpermissioned user routes through `MetricOmmSimpleRouter` and executes swaps the pool admin explicitly prohibited. Because the pool's oracle-priced bins hold real token balances, unauthorized swaps directly extract LP principal. This constitutes a direct loss of user/LP funds above Sherlock thresholds, qualifying as High severity.

## Likelihood Explanation

The bypass requires the pool admin to allowlist the router — a natural and expected operational step for any pool that wants to support the standard user-facing entry point. The admin has no indication that doing so voids the per-user allowlist guarantee. The attack is then permissionless and repeatable by any on-chain address.

## Recommendation

The extension must gate the actual end user, not the immediate caller. Three options:

1. **Pass original user through `extensionData`**: The router encodes `msg.sender` (the true end user) into `extensionData`; the extension decodes and checks it. Requires a convention between router and extension.
2. **Align with the deposit pattern**: Introduce a dedicated `swapper` identity field (analogous to `owner` in `addLiquidity`) that the pool populates with the true end user, and have the extension check that field.
3. **Document incompatibility**: At minimum, clearly document that `SwapAllowlistExtension` is incompatible with `MetricOmmSimpleRouter` and that allowlisting the router voids the allowlist guarantee.

## Proof of Concept

1. Deploy a pool with `SwapAllowlistExtension` configured in `beforeSwap`.
2. Pool admin calls `setAllowedToSwap(pool, router, true)` to enable router-based swaps.
3. Unpermissioned user `alice` (not in the per-user allowlist) calls `MetricOmmSimpleRouter.exactInputSingle(...)` targeting the pool.
4. The router calls `pool.swap(recipient, ...)` — `msg.sender` of this call is the router.
5. Pool calls `_beforeSwap(router, ...)` → extension evaluates `allowedSwapper[pool][router]` → `true` → swap proceeds.
6. Alice successfully swaps on a pool she was never authorized to access, bypassing the curated allowlist entirely.
