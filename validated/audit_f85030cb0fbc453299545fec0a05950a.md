### Title
NFT metadata and burn permanently locked after first transfer — (`x/nft/keeper/keeper.go`)

### Summary
`EditNFT` and `BurnNFT` require the caller to simultaneously satisfy `IsOwner` **and** `IsDenomCreator` on the same address. Once an NFT is transferred, ownership and denom-creator are held by two different addresses, so neither party can ever satisfy both checks. The NFT's metadata is permanently frozen and the NFT is unburnable.

### Finding Description
In `keeper.go`, `EditNFT` (lines 85–118) calls `IsOwner` (line 93) and then `IsDenomCreator` (line 98) on the same `owner` argument. Both must succeed. `IsDenomCreator` compares against `denom.Creator`, which is set once at `IssueDenom` and never updated. `TransferOwner` (lines 121–138) updates `nft.Owner` and the owner index but leaves `denom.Creator` untouched. After one transfer:

- **Buyer** passes `IsOwner` but fails `IsDenomCreator` → `ErrUnauthorized`
- **Denom creator** fails `IsOwner` (no longer the NFT owner) → `ErrUnauthorized`

No address can ever satisfy both conditions simultaneously again. The same logic applies identically to `BurnNFT` (lines 141–161).

### Impact Explanation
- NFT metadata (`Name`, `URI`, `Data`) is permanently frozen after the first transfer. Any NFT whose value depends on updatable metadata (e.g., evolving game assets, dynamic art) loses that capability entirely and irreversibly.
- `BurnNFT` is also permanently blocked, meaning the NFT cannot be destroyed by anyone — the supply counter can never be decremented for that token.
- Both effects are permanent and require no further attacker action after the transfer.

### Likelihood Explanation
Any standard NFT transfer via `MsgTransferNFT` triggers this. The denom creator transferring an NFT to a buyer (a completely normal, intended operation) is sufficient. No special privileges, governance, or unusual configuration required.

### Recommendation
Decouple the authorization model for `EditNFT` and `BurnNFT`. Two reasonable options:

1. **Owner-only edit**: Allow `EditNFT` if the caller is the current NFT owner (drop the `IsDenomCreator` check). This matches the standard expectation that the owner controls their asset.
2. **Creator-only edit**: Allow `EditNFT` if the caller is the denom creator (drop the `IsOwner` check). This matches a model where the issuer controls metadata.

For `BurnNFT`, the same fix applies — require only `IsOwner` (the owner should be able to burn their own NFT).

### Proof of Concept
```
1. Creator A calls MsgIssueDenom  → denom.Creator = A
2. Creator A calls MsgMintNFT     → nft.Owner = A
3. Creator A calls MsgTransferNFT → nft.Owner = B (buyer)
4. Buyer B calls MsgEditNFT       → IsOwner(B) ✓, IsDenomCreator(B) ✗ → ErrUnauthorized
5. Creator A calls MsgEditNFT     → IsOwner(A) ✗ → ErrUnauthorized
   (A is no longer the NFT owner after step 3)
```

Both step 4 and step 5 return `ErrUnauthorized`. No address can call `EditNFT` or `BurnNFT` on this NFT ever again.

func (k Keeper) TransferOwner(
	ctx sdk.Context, denomID, tokenID string, srcOwner, dstOwner sdk.AccAddress,
) error {
	if !k.HasDenomID(ctx, denomID) {
		return sdkerrors.Wrapf(types.ErrInvalidDenom, "denom ID %s not exists", denomID)
	}

	nft, err := k.IsOwner(ctx, denomID, tokenID, srcOwner)
	if err != nil {
		return err
	}

	nft.Owner = dstOwner.String()

	k.setNFT(ctx, denomID, nft)
	k.swapOwner(ctx, denomID, tokenID, srcOwner, dstOwner)
	return nil
}

func (k Keeper) BurnNFT(ctx sdk.Context, denomID, tokenID string, owner sdk.AccAddress) error {
	if !k.HasDenomID(ctx, denomID) {
		return sdkerrors.Wrapf(types.ErrInvalidDenom, "denom ID %s not exists", denomID)
	}

	nft, err := k.IsOwner(ctx, denomID, tokenID, owner)
	if err != nil {
		return err
	}

	_, err = k.IsDenomCreator(ctx, denomID, owner)
	if err != nil {
		return err
	}

	k.deleteNFT(ctx, denomID, nft)
	k.deleteOwner(ctx, denomID, tokenID, owner)
	k.decreaseSupply(ctx, denomID)

	return nil
}

func (k Keeper) IsOwner(ctx sdk.Context, denomID, tokenID string, owner sdk.AccAddress) (types.BaseNFT, error) {
	nft, err := k.GetNFT(ctx, denomID, tokenID)
	if err != nil {
		return types.BaseNFT{}, err
	}

	if !owner.Equals(nft.GetOwner()) {
		return types.BaseNFT{}, sdkerrors.Wrapf(types.ErrUnauthorized, "%s is not the owner of %s/%s", owner.String(), denomID, tokenID)
	}

	return nft.(types.BaseNFT), nil

func (k Keeper) IsDenomCreator(ctx sdk.Context, denomID string, address sdk.AccAddress) (types.Denom, error) {
	denom, err := k.GetDenom(ctx, denomID)
	if err != nil {
		return types.Denom{}, err
	}

	creator, err := sdk.AccAddressFromBech32(denom.Creator)
	if err != nil {
		panic(err)
	}

	if !creator.Equals(address) {
		return types.Denom{}, sdkerrors.Wrapf(types.ErrUnauthorized, "%s is not the creator of %s", address, denomID)
	}

	return denom, nil
