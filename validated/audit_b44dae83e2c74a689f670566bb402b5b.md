Audit Report

## Title
`SwapAllowlistExtension` checks router address instead of end user, enabling full allowlist bypass via `MetricOmmSimpleRouter` — (File: metric-periphery/contracts/extensions/SwapAllowlistExtension.sol)

## Summary
`SwapAllowlistExtension.beforeSwap` gates on the `sender` argument, which the pool sets to `msg.sender` of `pool.swap()`. When a user routes through `MetricOmmSimpleRouter`, that `msg.sender` is the router contract. A pool admin who allowlists the router — a natural action to enable periphery usage — inadvertently grants every on-chain address the ability to bypass the allowlist, completely voiding the pool's curation policy and exposing LP funds to unauthorized counterparties.

## Finding Description
`MetricOmmPool.swap()` passes `msg.sender` as `sender` to `_beforeSwap` (L230–231). `ExtensionCalling._beforeSwap` forwards it verbatim to each configured extension (L160–176). `SwapAllowlistExtension.beforeSwap` then evaluates `allowedSwapper[msg.sender][sender]`, where `msg.sender` is the pool and `sender` is whoever called `pool.swap()` (L37). `MetricOmmSimpleRouter.exactInputSingle` (and all other router entry points) calls `pool.swap()` directly (L72–80), so the pool sees `sender = router`. The real caller's identity is stored only in transient storage via `_setNextCallbackContext` for payment settlement (L71) and is never forwarded to the pool or extension. When the admin calls `setAllowedToSwap(pool, router, true)`, the check `allowedSwapper[pool][router]` returns `true` for every user routing through the router, regardless of whether that user is individually excluded. `DepositAllowlistExtension` avoids this by ignoring `sender` and checking `owner` (the position owner), but the swap interface carries no equivalent identity-preserving field.

## Impact Explanation
A pool configured with `SwapAllowlistExtension` for access curation has its invariant broken: any address, including those explicitly excluded via `setAllowedToSwap(pool, user, false)`, can swap against the pool by routing through `MetricOmmSimpleRouter`. LP funds in the curated pool are exposed to unauthorized counterparties, defeating the intended access control. This constitutes broken core pool functionality causing potential direct LP losses — the allowlist extension's sole purpose is nullified.

## Likelihood Explanation
The trigger is a pool admin calling `setAllowedToSwap(pool, router, true)`, a non-malicious, semi-trusted action any operator of a curated pool would reasonably take to enable the standard periphery flow. No documentation or interface signals that allowlisting the router is semantically equivalent to disabling the allowlist for all users. Once the router is allowlisted, the bypass requires zero additional preconditions and is available to any unprivileged address.

## Recommendation
The extension must identify the true economic actor, not the intermediary. Two viable approaches:
1. **Explicit swapper field in `extensionData`**: require the router to ABI-encode the end user's address (`msg.sender` at router entry) into `extensionData`; the extension decodes and checks it. The router already captures `msg.sender` at L71.
2. **Trusted-router registry with per-user forwarding**: add a trusted-router registry to the extension; when `sender` is a trusted router, decode the real swapper from `extensionData` and check that address instead.

Additionally, document clearly that allowlisting the router address is equivalent to opening the pool to all users.

## Proof of Concept
```
Setup:
  pool configured with SwapAllowlistExtension
  allowedSwapper[pool][alice] = true   // alice is the only allowed swapper
  allowedSwapper[pool][bob]   = false  // bob is explicitly excluded

Step 1 (admin, reasonable action):
  swapExtension.setAllowedToSwap(pool, address(router), true)
  // intended: "let users trade through the router"
  // actual effect: allowedSwapper[pool][router] = true

Step 2 (bob, unprivileged):
  router.exactInputSingle(ExactInputSingleParams{pool: pool, ..., extensionData: ""})
  // router calls pool.swap(...) → msg.sender seen by pool = router
  // pool calls _beforeSwap(sender=router, ...)
  // extension checks allowedSwapper[pool][router] → true
  // swap executes; bob receives output tokens

Result:
  bob, explicitly excluded, successfully swaps against the curated pool.
  The allowlist invariant is broken.
```
