### Title
`SwapAllowlistExtension` Allowlist Bypassed via Router: Any User Can Swap on Restricted Pools — (`metric-periphery/contracts/extensions/SwapAllowlistExtension.sol`)

---

### Summary

`SwapAllowlistExtension.beforeSwap` gates swaps by checking `allowedSwapper[pool][sender]`, where `sender` is the direct `msg.sender` of the pool's `swap` call. When swaps are routed through `MetricOmmSimpleRouter`, `sender` is the router address, not the end user. For router-mediated swaps to function on an allowlisted pool, the pool admin must allowlist the router — but doing so grants every user access through the router, completely defeating the allowlist.

---

### Finding Description

`MetricOmmPool.swap` passes `msg.sender` as the `sender` argument to `_beforeSwap`: [1](#0-0) 

`ExtensionCalling._beforeSwap` forwards that value unchanged to the extension: [2](#0-1) 

`SwapAllowlistExtension.beforeSwap` then checks `allowedSwapper[msg.sender][sender]`, where `msg.sender` is the pool and `sender` is whoever called the pool: [3](#0-2) 

When a user routes through `MetricOmmSimpleRouter`, the call chain is:

```
user → router.exactInput*() → pool.swap(msg.sender=router, ...)
                                    ↓
                         extension.beforeSwap(sender=router, ...)
                                    ↓
                         allowedSwapper[pool][router]  ← checked, NOT the end user
```

For router-mediated swaps to work at all on an allowlisted pool, the admin must add the router to `allowedSwapper`. Once the router is allowlisted, the check `allowedSwapper[pool][router]` is always `true` regardless of which end user initiated the call. Every user on the network can bypass the allowlist by routing through the public router.

There is no mechanism for the pool to pass the original end-user address as `sender`; the pool unconditionally uses `msg.sender`. [4](#0-3) 

---

### Impact Explanation

A pool deployer who configures `SwapAllowlistExtension` to restrict trading to a curated set of counterparties (e.g., KYC'd addresses, protocol-owned contracts, or whitelisted market makers) loses that restriction entirely for any user who routes through the public `MetricOmmSimpleRouter`. The allowlist guard — the sole access-control boundary on the swap path — is rendered inoperative. Unauthorized users can execute swaps, drain liquidity at oracle prices, and interact with pools that were explicitly designed to be closed to them. This is an admin-boundary break: an unprivileged path (the public router) bypasses a pool-admin-configured access control.

---

### Likelihood Explanation

The `MetricOmmSimpleRouter` is a public, permissionless periphery contract. Any user who discovers the router address can route through it. The pool admin is forced into the dilemma: either allowlist the router (breaking the allowlist for all users) or do not allowlist it (making the router unusable on the pool). There is no configuration that simultaneously supports router-mediated swaps and per-user allowlisting. Likelihood is high for any pool that intends to use both the allowlist extension and the router.

---

### Recommendation

The extension must verify the economically relevant actor — the end user — not the intermediary. Two approaches:

1. **Pass the original initiator through the call chain**: Modify `MetricOmmPool.swap` to accept an explicit `swapper` parameter (or read it from `extensionData`) and pass it as `sender` to `_beforeSwap`, allowing routers to forward the original `msg.sender`.

2. **Check both sender and recipient in the extension**: The extension can require that either `sender` or a user-supplied address embedded in `extensionData` is allowlisted, and routers must attest to the original caller.

The simplest safe fix is to have the router embed the original `msg.sender` in `extensionData` and have the extension decode and check that value, with the pool enforcing that the embedded address matches the callback payer.

---

### Proof of Concept

```
Setup:
  - Deploy pool with SwapAllowlistExtension
  - Pool admin calls setAllowedToSwap(pool, alice, true)   // only alice allowed
  - Pool admin calls setAllowedToSwap(pool, router, true)  // required for router to work

Attack:
  - bob (not allowlisted) calls router.exactInputSingle({pool: pool, ...})
  - router calls pool.swap(msg.sender=router, ...)
  - pool calls extension.beforeSwap(sender=router, ...)
  - extension checks allowedSwapper[pool][router] == true  → passes
  - bob's swap executes successfully on a pool he is not allowlisted for

Result:
  - SwapAllowlistExtension is bypassed for all users via the public router
  - Pool admin's access control is completely defeated
``` [3](#0-2) [5](#0-4)
