### Title
Bonus Checkpoints Advanced and Event Ref-Counts Decremented Without Payment When Pool Is Insufficient During Redelegation Slash - (`x/tieredrewards/keeper/slash.go`)

---

### Summary

In `slashRedelegationPosition`, when `processEventsAndClaimBonus` returns `ErrInsufficientBonusPool`, the caller silently swallows the error and then persists the position with already-mutated checkpoints (`LastBonusAccrual`, `LastEventSeq`, `LastKnownBonded`) and already-decremented event reference counts. The bonus is never paid, but the accounting state is advanced as if it were. The accrued bonus is permanently and irrecoverably lost for the affected position.

---

### Finding Description

`processEventsAndClaimBonus` in `x/tieredrewards/keeper/claim_rewards.go` performs two categories of side effects **before** it checks whether the pool can cover the computed bonus:

**Category 1 — in-memory mutations to `pos` (pointer receiver):**
- `pos.UpdateLastEventSeq(entry.Seq)` at line 193 — advances the event cursor
- `applyBonusAccrualCheckpoint(&pos.Position, blockTime)` at line 215 — advances `LastBonusAccrual` to current block time
- `pos.UpdateLastKnownBonded(bonded)` at line 217 — updates bonded state

**Category 2 — direct KV-store writes:**
- `k.decrementEventRefCount(ctx, valAddr, entry.Seq)` at line 196 — decrements the reference count of each processed event; when the count reaches zero the event is garbage-collected from `ValidatorEvents`

Only **after** all of the above does the function check pool sufficiency:

```go
// claim_rewards.go:230-232
if err := k.sufficientBonusPoolBalance(ctx, bonusCoins); err != nil {
    return nil, err
}
``` [1](#0-0) 

When `ErrInsufficientBonusPool` is returned, control returns to `slashRedelegationPosition`:

```go
// slash.go:54-64
if _, err := k.processEventsAndClaimBonus(ctx, &pos); err != nil {
    // Deliberately forgo bonus rewards if pool is insufficient to prevent chain halt.
    if errors.Is(err, types.ErrInsufficientBonusPool) {
        k.logger(ctx).Error(...)
    } else {
        return err
    }
}
``` [2](#0-1) 

The error is swallowed. For a **partial slash**, execution falls through to:

```go
// slash.go:76-77
pos.Delegation.Shares = pos.Delegation.Shares.Sub(sharesToUnbond)
return k.setPositionWithState(ctx, pos, nil)
``` [3](#0-2) 

`setPositionWithState` persists the already-mutated `pos` — with `LastBonusAccrual` advanced to the current block time, `LastEventSeq` advanced past all processed events, and `LastKnownBonded` updated — even though no bonus was transferred to the owner. [4](#0-3) 

The KV-store writes from `decrementEventRefCount` are also already committed; events whose reference count reached zero have been garbage-collected from `ValidatorEvents`. [5](#0-4) 

For a **full slash**, `ClearBonusCheckpoints()` is called immediately after (line 70), which resets the in-memory checkpoint fields to zero before `setPositionWithState`. The in-memory damage is overwritten, but the KV-store `decrementEventRefCount` writes have already been committed and cannot be undone. [6](#0-5) 

---

### Impact Explanation

For a **partial redelegation slash** while the bonus pool is insufficient:

- The position's `LastBonusAccrual` is advanced to the slash block time. The entire accrual window from the previous checkpoint to the slash block is permanently erased — the owner can never claim bonus for that period.
- `LastEventSeq` is advanced past all processed events. Those events' reference counts have been decremented; if this position was the last referencing them, the events are garbage-collected. Even if the pool is later replenished, the position cannot replay those events because they no longer exist and the cursor has moved past them.
- The corrupted value is the position's `LastBonusAccrual` timestamp and `LastEventSeq` stored in `x/tieredrewards`'s `Positions` collection — they reflect a claim that was never paid.

The invariant broken is stated in the module's own security documentation: *"Pool balance is never exceeded. User paths fail atomically; the `BeforeRedelegationSlashed` hook forfeits bonus silently if the pool is short, to avoid chain halt."* The documentation says the bonus is "forfeited" (implying a one-time loss), but the actual effect is that the position's accounting state is permanently corrupted: the owner loses not just the current accrual but the ability to ever claim it after the pool is replenished. [7](#0-6) 

---

### Likelihood Explanation

The trigger requires two concurrent conditions:
1. A tier position is in an active redelegation (i.e., `MsgTierRedelegate` was submitted and the redelegation period has not yet completed).
2. A governance-initiated slash or a double-sign evidence slash fires on the source validator during that redelegation window, causing `BeforeRedelegationSlashed` to fire.
3. The `RewardsPoolName` module account has insufficient balance to cover the accrued bonus.

Condition 3 is realistic: the pool is funded externally and can be depleted by normal bonus claims or simply not yet replenished. Conditions 1 and 2 are normal protocol events. The combination is uncommon but not negligible, especially as the protocol scales. [8](#0-7) 

---

### Recommendation

Move all side effects — both the in-memory checkpoint mutations and the KV-store `decrementEventRefCount` writes — to occur **after** the pool sufficiency check, or wrap the entire `processEventsAndClaimBonus` body in a cache context that is only committed on success. The simplest fix is to restructure `processEventsAndClaimBonus` so that:

1. The event loop accumulates `totalBonus` and the new checkpoint values in local variables without mutating `pos` or writing to the KV store.
2. The pool check is performed.
3. Only on success are `pos` mutated and `decrementEventRefCount` called.

Alternatively, in `slashRedelegationPosition`, when `ErrInsufficientBonusPool` is caught, explicitly restore `pos` to its pre-call state (re-read from store) before calling `setPositionWithState`, so that the persisted checkpoints are not advanced. [9](#0-8) 

---

### Proof of Concept

**State before the slash:**
- Position P has `LastBonusAccrual = T0`, `LastEventSeq = 5`, accrued bonus = 100 basecro.
- `RewardsPoolName` balance = 50 basecro (< 100 basecro).
- A redelegation for P is active with `unbondingId = X`.

**Execution of `BeforeRedelegationSlashed(ctx, X, partialShares)`:**

1. `slashRedelegationPosition` is called.
2. `processEventsAndClaimBonus(ctx, &pos)` is called:
   - Events since seq 5 are walked; `pos.LastEventSeq` is updated to 8 (in-memory).
   - `decrementEventRefCount` is called for seqs 6, 7, 8 — KV-store writes committed.
   - `pos.LastBonusAccrual` is set to `blockTime` (in-memory).
   - `totalBonus = 100 basecro`.
   - `sufficientBonusPoolBalance` fails: pool has 50 < 100.
   - Returns `ErrInsufficientBonusPool`.
3. Error is swallowed (line 56-63).
4. `pos.Delegation.Shares -= partialShares`.
5. `setPositionWithState(ctx, pos, nil)` persists `pos` with `LastBonusAccrual = blockTime`, `LastEventSeq = 8`.

**State after the slash:**
- Position P has `LastBonusAccrual = blockTime`, `LastEventSeq = 8` — as if 100 basecro was paid.
- Owner received 0 basecro.
- Events 6, 7, 8 may be garbage-collected.
- Even after the pool is replenished to 1000 basecro, the owner can never claim the 100 basecro for the period `[T0, blockTime]`. [10](#0-9) [11](#0-10) [12](#0-11)

// decrementEventRefCount decrements the reference count of a validator event.
// If the reference count reaches zero, the event is deleted.
func (k Keeper) decrementEventRefCount(ctx context.Context, valAddr sdk.ValAddress, seq uint64) error {
	key := collections.Join(valAddr, seq)
	event, err := k.ValidatorEvents.Get(ctx, key)
	if errors.Is(err, collections.ErrNotFound) {
		return nil // already cleaned up
	}
	if err != nil {
		return err
	}

	if event.ReferenceCount <= 1 {
		return k.ValidatorEvents.Remove(ctx, key)
	}

	event.ReferenceCount--
	return k.ValidatorEvents.Set(ctx, key, event)
}

func (k Keeper) sufficientBonusPoolBalance(ctx context.Context, bonus sdk.Coins) error {
	if bonus.IsZero() {
		return nil
	}

	poolAddr := k.accountKeeper.GetModuleAddress(types.RewardsPoolName)
	poolBalance := k.bankKeeper.GetAllBalances(ctx, poolAddr)
	if !poolBalance.IsAllGTE(bonus) {
		return errorsmod.Wrapf(types.ErrInsufficientBonusPool,
			"bonus: %s, pool balance: %s",
			bonus.String(), poolBalance.String())
	}

	return nil
