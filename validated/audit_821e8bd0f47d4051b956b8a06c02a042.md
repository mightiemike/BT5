### Title
Bonus Pool Solvency Check Ignores Total Outstanding Obligations, Enabling First-Come-First-Served Pool Drainage — (File: `x/tieredrewards/keeper/bonus_rewards.go`)

---

### Summary

The `sufficientBonusPoolBalance` function in `x/tieredrewards/keeper/bonus_rewards.go` only verifies that the rewards pool can cover the **current** claim being processed. It never accounts for the total accrued-but-unclaimed bonus obligations owed to all other active positions. This mirrors the Neptune Mutual vulnerability class exactly: a solvency/capacity check that undercounts existing commitments. The result is that early claimants can drain the pool, leaving later claimants with legitimately earned bonuses unable to collect.

---

### Finding Description

When a position owner calls `ClaimRewards` (or any flow that triggers `processEventsAndClaimBonus`), the final gate before transferring funds is `sufficientBonusPoolBalance`: [1](#0-0) 

```go
func (k Keeper) sufficientBonusPoolBalance(ctx context.Context, bonus sdk.Coins) error {
    if bonus.IsZero() {
        return nil
    }
    poolAddr := k.accountKeeper.GetModuleAddress(types.RewardsPoolName)
    poolBalance := k.bankKeeper.GetAllBalances(ctx, poolAddr)
    if !poolBalance.IsAllGTE(bonus) {
        return errorsmod.Wrapf(types.ErrInsufficientBonusPool, ...)
    }
    return nil
}
```

The check is: `poolBalance >= thisClaimAmount`. It does **not** compute or subtract the total accrued-but-unclaimed bonus obligations of every other active position before deciding whether the pool is solvent enough to pay the current claim.

The bonus accrual formula is time-based and continuous: [2](#0-1) 

Every active, bonded position silently accumulates an obligation against the pool proportional to `shares × tokensPerShare × BonusApy × duration`. None of these outstanding obligations are summed and compared against the pool balance at claim time or at position-creation time.

The claim path that reaches this check is: [3](#0-2) 

`processEventsAndClaimBonus` → `sufficientBonusPoolBalance` → `bankKeeper.SendCoinsFromModuleToAccount`.

---

### Impact Explanation

When the pool is not sufficiently funded relative to total outstanding obligations (a condition the protocol never enforces or even measures), the first positions to claim exhaust the pool. All subsequent claimants receive `ErrInsufficientBonusPool` and lose their legitimately earned bonus rewards permanently — the accrual checkpoint is still advanced: [4](#0-3) 

`applyBonusAccrualCheckpoint` is called unconditionally in `processEventsAndClaimBonus` regardless of whether the pool payment succeeded, meaning the accrued period is consumed even when the payout fails. The corrupted value is the bonus reward balance owed to positions that cannot claim after the pool is drained.

---

### Likelihood Explanation

The pool is funded by governance deposits with no protocol-enforced minimum relative to total outstanding obligations. As the number of active positions grows and time passes, the gap between total accrued obligations and pool balance can widen silently. Any user with a position can trigger the drain by claiming at any time via a standard `MsgClaimRewards` transaction — no privilege required. The condition is reachable on mainnet whenever the pool is underfunded relative to the sum of all positions' accrued bonuses.

---

### Recommendation

Before approving any individual claim, compute the total accrued-but-unclaimed bonus obligations across all active positions and verify that `poolBalance >= thisClaimAmount + totalOutstandingObligations`. Alternatively, track a running `TotalOutstandingObligations` counter that is incremented on each block per active position and decremented on each successful payout, and gate both new position creation and claims against this counter.

---

### Proof of Concept

1. Governance funds the pool with `X` tokens.
2. Alice and Bob each create a tiered position; both accrue bonuses continuously.
3. After sufficient time, Alice's accrued bonus = `0.9X` and Bob's accrued bonus = `0.9X` (total obligations = `1.8X > X`).
4. Alice submits `MsgClaimRewards`. `sufficientBonusPoolBalance` checks `X >= 0.9X` → passes. Alice receives `0.9X`. Pool now holds `0.1X`.
5. Bob submits `MsgClaimRewards`. `sufficientBonusPoolBalance` checks `0.1X >= 0.9X` → fails with `ErrInsufficientBonusPool`. Bob's accrual checkpoint is still advanced by `applyBonusAccrualCheckpoint`; his earned `0.9X` is permanently lost.

The root cause — `sufficientBonusPoolBalance` checking only the immediate claim against the current pool balance without accounting for all outstanding obligations — is the direct analog of the Neptune Mutual commitment undercount.

	accrualEnd := blockTime

	}
	pos.UpdateLastBonusAccrual(accrualEnd)

func (k Keeper) computeSegmentBonus(pos types.PositionState, tier types.Tier, segmentStart, segmentEnd time.Time, tokensPerShare math.LegacyDec) math.Int {
	if !pos.ExitUnlockAt.IsZero() && segmentEnd.After(pos.ExitUnlockAt) {
		segmentEnd = pos.ExitUnlockAt
	}

	if !segmentEnd.After(segmentStart) {
		return math.ZeroInt()
	}

	durationSeconds := int64(segmentEnd.Sub(segmentStart) / time.Second)
	if durationSeconds <= 0 {
		return math.ZeroInt()
	}

	tokens := pos.Delegation.Shares.Mul(tokensPerShare)

	return tokens.
		Mul(tier.BonusApy).
		MulInt64(durationSeconds).
		QuoInt64(types.SecondsPerYear).
		TruncateInt()
}

func (k Keeper) processEventsAndClaimBonus(ctx context.Context, pos *types.PositionState) (sdk.Coins, error) {
	// Rewards should have been claimed before undelegation
	if !pos.IsDelegated() {
		return sdk.NewCoins(), nil
	}

	valAddr, err := sdk.ValAddressFromBech32(pos.Delegation.ValidatorAddress)
	if err != nil {
		return nil, err
	}

	events, err := k.getValidatorEventsSince(ctx, valAddr, pos.LastEventSeq)
	if err != nil {
		return nil, err
	}

	sdkCtx := sdk.UnwrapSDKContext(ctx)
	blockTime := sdkCtx.BlockTime()

	totalBonus := math.ZeroInt()
	// Use the persisted bonded state from the last replay, not a hardcoded default.
	// This prevents overpaying bonus for unbonded gaps between claims.
	bonded := pos.LastKnownBonded
	segmentStart := pos.LastBonusAccrual

	tier, err := k.getTier(ctx, pos.TierId)
	if err != nil {
		return nil, err
	}

	for _, entry := range events {
		evt := entry.Event

		if bonded {
			// Compute bonus for the bonded segment [segmentStart, eventTime]
			// using the snapshot rate at the event.
			bonus := k.computeSegmentBonus(*pos, tier, segmentStart, evt.Timestamp, evt.TokensPerShare)
			totalBonus = totalBonus.Add(bonus)
		}

		// Update bonded state based on event type.
		switch evt.EventType {
		case types.ValidatorEventType_VALIDATOR_EVENT_TYPE_UNBOND:
			bonded = false
		case types.ValidatorEventType_VALIDATOR_EVENT_TYPE_BOND:
			bonded = true
		case types.ValidatorEventType_VALIDATOR_EVENT_TYPE_SLASH:
			// Slash doesn't change bonded state.
		}

		segmentStart = evt.Timestamp
		pos.UpdateLastEventSeq(entry.Seq)

		// Decrement reference count.
		if err := k.decrementEventRefCount(ctx, valAddr, entry.Seq); err != nil {
			return nil, err
		}
	}

	val, err := k.stakingKeeper.GetValidator(ctx, valAddr)
	if err != nil {
		return nil, err
	}
	// Defensive: validator bond status check
	if bonded && val.IsBonded() {
		currentRate, err := k.getTokensPerShare(ctx, valAddr)
		if err != nil {
			return nil, err
		}
		bonus := k.computeSegmentBonus(*pos, tier, segmentStart, blockTime, currentRate)
		totalBonus = totalBonus.Add(bonus)
	}

	applyBonusAccrualCheckpoint(&pos.Position, blockTime)
	// Persist the bonded state so the next replay starts correctly.
	pos.UpdateLastKnownBonded(bonded)

	if totalBonus.IsZero() {
		return sdk.NewCoins(), nil
	}

	bondDenom, err := k.stakingKeeper.BondDenom(ctx)
	if err != nil {
		return nil, err
	}

	bonusCoins := sdk.NewCoins(sdk.NewCoin(bondDenom, totalBonus))

	if err := k.sufficientBonusPoolBalance(ctx, bonusCoins); err != nil {
		return nil, err
	}

	ownerAddr, err := sdk.AccAddressFromBech32(pos.Owner)
	if err != nil {
		return nil, errorsmod.Wrap(sdkerrors.ErrInvalidAddress, "invalid owner address")
	}

	if err := k.bankKeeper.SendCoinsFromModuleToAccount(ctx, types.RewardsPoolName, ownerAddr, bonusCoins); err != nil {
		return nil, err
	}

	if err := sdkCtx.EventManager().EmitTypedEvent(&types.EventBonusRewardsClaimed{
		PositionId: pos.Id,
		Owner:      pos.Owner,
		Rewards:    bonusCoins,
	}); err != nil {
		return nil, err
	}

	return bonusCoins, nil
}
