### Title
`AllResourceBounds` with zero L2/data-gas silently collapses to `ValidResourceBounds::L1Gas` on protobuf round-trip, breaking consensus deserialization - (File: `crates/apollo_protobuf/src/converters/transaction.rs`)

### Summary

The protobuf deserializer for `ValidResourceBounds` uses a value-based heuristic to decide which enum variant to reconstruct. When `l2_gas = 0` and `l1_data_gas = 0`, it always produces `ValidResourceBounds::L1Gas`, even when the original transaction was `AllResources`. The consensus receive path then rejects the transaction because `TryFrom<InvokeTransactionV3> for RpcInvokeTransactionV3` only accepts `AllResources`. A valid transaction accepted by the gateway therefore causes every consensus peer to fail to deserialize the proposal that contains it.

### Finding Description

`RpcInvokeTransactionV3` and `InternalRpcInvokeTransactionV3` both store resource bounds as `AllResourceBounds` (always the `AllResources` variant). The gateway stateless validator explicitly accepts transactions where only `l1_gas` is non-zero and both `l2_gas` and `l1_data_gas` are zero. [1](#0-0) 

When such a transaction is serialized to protobuf for consensus propagation, `From<RpcInvokeTransactionV3> for protobuf::InvokeV3WithProof` converts it through `InvokeTransactionV3` (wrapping the bounds as `ValidResourceBounds::AllResources`) and then through `From<ValidResourceBounds> for protobuf::ResourceBounds`, which emits `l2_gas = 0` and `l1_data_gas = 0` on the wire — identical to what a legacy `L1Gas` transaction would produce. [2](#0-1) 

On the receiving side, `TryFrom<protobuf::ResourceBounds> for ValidResourceBounds` applies the heuristic: if `l1_data_gas.is_zero() && l2_gas.is_zero()`, reconstruct as `ValidResourceBounds::L1Gas`. [3](#0-2) 

The reconstructed `InvokeTransactionV3` now carries `ValidResourceBounds::L1Gas`. The next step in the consensus receive path, `TryFrom<InvokeTransactionV3> for RpcInvokeTransactionV3`, explicitly rejects any variant other than `AllResources`: [4](#0-3) 

The error is mapped to `DEPRECATED_RESOURCE_BOUNDS_ERROR` and propagated through `TryFrom<protobuf::InvokeV3WithProof> for RpcInvokeTransactionV3`: [5](#0-4) 

This causes `TryFrom<protobuf::ConsensusTransaction> for ConsensusTransaction` to return an error, and the entire proposal is rejected by every receiving peer. [6](#0-5) 

### Impact Explanation

A proposer that includes a valid `AllResources` transaction with `l2_gas = 0` and `l1_data_gas = 0` in a block proposal will have that proposal rejected by all peers during consensus deserialization. This is a **High** impact: "Transaction conversion or signature/hash logic binds the wrong signer, hash, type, or executable payload" — the conversion binds the wrong resource-bounds type (`L1Gas` instead of `AllResources`), causing the executable payload to be unrecoverable from the wire format.

### Likelihood Explanation

The gateway stateless validator explicitly allows transactions with only `l1_gas` set: [7](#0-6) 

Any user can submit such a transaction through the normal RPC path. Once it enters the mempool and is selected by the batcher, the proposer will include it in a consensus proposal. Every peer will then fail to deserialize the proposal, stalling consensus for that round. The condition (`l2_gas = 0`, `l1_data_gas = 0`) is trivially constructable and requires no special privilege.

### Recommendation

Replace the value-based heuristic in `TryFrom<protobuf::ResourceBounds> for ValidResourceBounds` with an explicit discriminator field in the protobuf schema, or always reconstruct as `AllResources` when all three fields are present on the wire (regardless of their values). The `unwrap_or_default` for `l1_data_gas` combined with the zero-check is the root cause:

```rust
// Current (broken): collapses AllResources{l2=0, data=0} → L1Gas
Ok(if l1_data_gas.is_zero() && l2_gas.is_zero() {
    ValidResourceBounds::L1Gas(l1_gas)
} else {
    ValidResourceBounds::AllResources(...)
})

// Fix: always AllResources when all three fields are present
Ok(ValidResourceBounds::AllResources(AllResourceBounds { l1_gas, l2_gas, l1_data_gas }))
```

The `L1Gas` variant should only be reconstructed when the protobuf message explicitly signals a pre-0.13.3 transaction (e.g., via a missing `l1_data_gas` field), not based on the numeric value of the bounds.

### Proof of Concept

1. Submit an `RpcInvokeTransactionV3` to the gateway with `resource_bounds = AllResourceBounds { l1_gas: { max_amount: 1000, max_price_per_unit: 1 }, l2_gas: { max_amount: 0, max_price_per_unit: 0 }, l1_data_gas: { max_amount: 0, max_price_per_unit: 0 } }`. The gateway accepts it (stateless validator passes because `l1_gas` is non-zero).

2. The transaction enters the mempool and is selected by the batcher into a proposal.

3. The proposer serializes the proposal to protobuf. `From<ValidResourceBounds> for protobuf::ResourceBounds` emits `l2_gas = 0` and `l1_data_gas = 0` on the wire.

4. A peer receives the proposal and calls `TryFrom<protobuf::ConsensusTransaction> for ConsensusTransaction`. Inside, `TryFrom<protobuf::ResourceBounds> for ValidResourceBounds` sees `l2_gas.is_zero() && l1_data_gas.is_zero()` and returns `ValidResourceBounds::L1Gas(l1_gas)`.

5. `TryFrom<InvokeTransactionV3> for RpcInvokeTransactionV3` matches on `ValidResourceBounds::L1Gas` and returns `Err(DEPRECATED_RESOURCE_BOUNDS_ERROR)`.

6. The `?` in `TryFrom<protobuf::ConsensusTransaction>` propagates the error; the peer rejects the entire proposal.

7. Consensus for this round fails. Repeating with any such transaction stalls the sequencer.

impl TryFrom<protobuf::ResourceBounds> for ValidResourceBounds {
    type Error = ProtobufConversionError;
    fn try_from(value: protobuf::ResourceBounds) -> Result<Self, Self::Error> {
        let Some(l1_gas) = value.l1_gas else {
            return Err(missing("ResourceBounds::l1_gas"));
        };
        let Some(l2_gas) = value.l2_gas else {
            return Err(missing("ResourceBounds::l2_gas"));
        };
        // TODO(Shahak): Assert data gas is not none once we remove support for 0.13.2.
        let l1_data_gas = value.l1_data_gas.unwrap_or_default();
        let l1_gas: ResourceBounds = l1_gas.try_into()?;
        let l2_gas: ResourceBounds = l2_gas.try_into()?;
        let l1_data_gas: ResourceBounds = l1_data_gas.try_into()?;
        Ok(if l1_data_gas.is_zero() && l2_gas.is_zero() {
            ValidResourceBounds::L1Gas(l1_gas)
        } else {
            ValidResourceBounds::AllResources(AllResourceBounds { l1_gas, l2_gas, l1_data_gas })
        })
    }

impl TryFrom<protobuf::ConsensusTransaction> for ConsensusTransaction {

    fn try_from(value: protobuf::ConsensusTransaction) -> Result<Self, Self::Error> {
        let txn = value.txn.ok_or(missing("ConsensusTransaction::txn"))?;
        let txn = match txn {
            protobuf::consensus_transaction::Txn::DeclareV3(txn) => {
                ConsensusTransaction::RpcTransaction(RpcTransaction::Declare(
                    RpcDeclareTransaction::V3(txn.try_into()?),
                ))
            }
            protobuf::consensus_transaction::Txn::DeployAccountV3(txn) => {
                ConsensusTransaction::RpcTransaction(RpcTransaction::DeployAccount(
                    RpcDeployAccountTransaction::V3(txn.try_into()?),
                ))
            }
            protobuf::consensus_transaction::Txn::InvokeV3(txn) => {
                ConsensusTransaction::RpcTransaction(RpcTransaction::Invoke(
                    RpcInvokeTransaction::V3(txn.try_into()?),
                ))
            }
            protobuf::consensus_transaction::Txn::L1Handler(txn) => {
                ConsensusTransaction::L1Handler(txn.try_into()?)
            }
        };
        Ok(txn)
    }
}

impl TryFrom<protobuf::InvokeV3WithProof> for RpcInvokeTransactionV3 {
    type Error = ProtobufConversionError;
    fn try_from(mut value: protobuf::InvokeV3WithProof) -> Result<Self, Self::Error> {
        // Extract proof first, since `starknet_api::transaction::InvokeTransactionV3` does not
        // carry a `proof` field.
        let proof = Proof::from(std::mem::take(&mut value.proof));

        let snapi_invoke: InvokeTransactionV3 = value
            .invoke
            .ok_or(ProtobufConversionError::MissingField {
                field_description: "InvokeV3WithProof::invoke",
            })?
            .try_into()?;

        // This conversion can fail only if the resource_bounds are not AllResources.
        Ok(Self { proof, ..snapi_invoke.try_into().map_err(|_| DEPRECATED_RESOURCE_BOUNDS_ERROR)? })
    }
