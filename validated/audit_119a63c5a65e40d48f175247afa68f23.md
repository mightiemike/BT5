The core of this question is whether `ExecutionStateReader::get_compiled_class_hash` (used in RPC simulation/fee estimation) returns a stale V1 hash after a migration has been committed to storage, and whether that stale value feeds `should_migrate` into incorrectly re-triggering migration logic.

## Tracing the Two Implementations

**`ExecutionStateReader::get_compiled_class_hash`** (RPC path): [1](#0-0) 

The logic is:
1. Call `get_class_definition_block_number(&class_hash)` — returns the **declaration block N** (stored once, never updated on migration).
2. Read `get_state_diff(block_number)` at block N.
3. Return `class_hash_to_compiled_class_hash.get(&class_hash)` from that state diff — always the **V1 hash**.

`get_class_definition_block_number` is a simple key-value lookup that returns the first block where the class was declared and is never updated: [2](#0-1) 

The `declared_classes_block_table` is written once at declaration time and never updated on migration: [3](#0-2) 

**`ApolloReader::get_compiled_class_hash`** (sequencer path) — the correct implementation: [4](#0-3) 

This calls `get_compiled_class_hash_at(state_number, &class_hash)` which does a cursor-based `lower_bound` + `prev` scan on the `compiled_class_hash_table` keyed by `(ClassHash, BlockNumber)`: [5](#0-4) 

When migration runs at block N+1, `write_compiled_class_hashes` inserts `(class_hash, N+1) → V2` into this table: [6](#0-5) 

So `ApolloReader` correctly returns V2 after migration; `ExecutionStateReader` always returns V1 (from the declaration block's state diff).

## The `should_migrate` Call Chain

`should_migrate` compares `get_compiled_class_hash` against `get_compiled_class_hash_v2`: [7](#0-6) 

If `get_compiled_class_hash` returns V1 (stale), `state_compiled_class_hash != compiled_class_hash_v2`, so `should_migrate` returns `Some(...)` — indicating migration is needed even for an already-migrated class.

This is called from `CasmHashMigrationData::from_state` during bouncer weight computation: [8](#0-7) 

Which is called from `get_tx_weights` during every transaction execution: [9](#0-8) 

## Reachability

Any unprivileged user calling `starknet_estimateFee` or `starknet_simulateTransactions` for a transaction that invokes a contract whose class was migrated at block N+1 will trigger this path. The `ExecutionStateReader` is the state reader for all RPC execution in `crates/apollo_rpc_execution`.

## Concrete Divergence

- Block N: class C declared with V1 hash → `declared_classes_block_table[C] = N`, `state_diff[N].class_hash_to_compiled_class_hash[C] = V1`
- Block N+1: migration → `compiled_class_hash_table[(C, N+1)] = V2`
- RPC simulation at state after N+1: `ExecutionStateReader::get_compiled_class_hash(C)` → reads `state_diff[N]` → returns V1 → `should_migrate` returns `Some(...)` → migration gas added to fee estimate → inflated fee returned to caller

### Title
`ExecutionStateReader::get_compiled_class_hash` reads declaration-block state diff instead of time-ordered storage, returning stale V1 hash after migration and causing inflated RPC fee estimates — (`crates/apollo_rpc_execution/src/state_reader.rs`)

### Summary
`ExecutionStateReader::get_compiled_class_hash` always reads the compiled class hash from the **declaration block's state diff** (via `get_class_definition_block_number` + `get_state_diff`), ignoring any subsequent migration updates written to the `compiled_class_hash_table`. After a class is migrated from V1 to V2 at block N+1, RPC simulation/fee estimation still sees V1, causing `should_migrate` to incorrectly flag the class for re-migration and inflate the fee estimate.

### Finding Description
The `compiled_class_hash_table` is keyed by `(ClassHash, BlockNumber)` and supports time-ordered lookups via cursor scan. When migration runs at block N+1, `write_compiled_class_hashes` inserts `(class_hash, N+1) → V2`. The correct reader (`ApolloReader`) uses `get_compiled_class_hash_at(state_number, ...)` which finds the most recent entry ≤ state_number, returning V2 post-migration.

`ExecutionStateReader::get_compiled_class_hash` instead: (1) looks up `declared_classes_block_table[class_hash]` → always returns N (declaration block, never updated); (2) reads `state_diff[N].class_hash_to_compiled_class_hash[class_hash]` → always returns V1. The `state_number` field of `ExecutionStateReader` is completely ignored by this method. [10](#0-9) 

### Impact Explanation
`should_migrate` receives V1 from `get_compiled_class_hash` and V2 from `get_compiled_class_hash_v2`; since they differ, it returns `Some(...)` for every already-migrated class that is executed during simulation. Migration gas is added to `total_sierra_gas` in `get_tx_weights`, inflating the fee estimate returned by `starknet_estimateFee` and `starknet_simulateTransactions`. The simulation state diff also incorrectly shows a compiled class hash update (migration) that would not occur in actual block execution.

### Likelihood Explanation
Triggered by any unprivileged RPC caller simulating or estimating fees for a transaction that invokes a contract whose class was migrated. No special privileges required. Becomes active as soon as `enable_casm_hash_migration=true` and at least one migration block has been committed.

### Recommendation
Replace the declaration-block state diff lookup in `ExecutionStateReader::get_compiled_class_hash` with a time-ordered storage lookup using `get_compiled_class_hash_at(self.state_number, &class_hash)`, consistent with how `ApolloReader` implements the same method.

### Proof of Concept
1. Declare class C at block N with V1 compiled class hash; commit state diff with `class_hash_to_compiled_class_hash[C] = V1`.
2. At block N+1 with `enable_casm_hash_migration=true`, execute a transaction invoking C; finalize block → migration writes `compiled_class_hash_table[(C, N+1)] = V2`.
3. Call `starknet_estimateFee` for a transaction invoking C at block N+1.
4. `ExecutionStateReader::get_compiled_class_hash(C)` → `get_class_definition_block_number(C)` = N → `get_state_diff(N).class_hash_to_compiled_class_hash[C]` = V1.
5. `should_migrate` compares V1 ≠ V2 → returns `Some(...)` → migration gas added.
6. Fee estimate is inflated by the migration gas cost of class C.
7. Assert: `get_compiled_class_hash_at(state_after_N+1, C)` = V2 (correct), but `ExecutionStateReader::get_compiled_class_hash(C)` = V1 (wrong).

fn get_compiled_class_hash_at<'env, Mode: TransactionKind>(
    first_irrelevant_block: BlockNumber,
    class_hash: &ClassHash,
    txn: &'env DbTransaction<'env, Mode>,
    compiled_class_hash_table: &'env CompiledClassHashTable<'env>,
) -> StorageResult<Option<CompiledClassHash>> {
    let db_key = (*class_hash, first_irrelevant_block);
    // Find the previous db item.
    let mut cursor = compiled_class_hash_table.cursor(txn)?;
    cursor.lower_bound(&db_key)?;
    let res = cursor.prev()?;
    match res {
        None => Ok(None),
        Some(((got_class_hash, _got_block_number), value)) => {
            if got_class_hash != *class_hash {
                // The previous item belongs to different class hash, which means there is no
                // previous state diff for this item.
                return Ok(None);
            };
            // The previous db item indeed belongs to this address and key.
            Ok(Some(value))
        }
    }
}

pub fn should_migrate(
    state_reader: &impl StateReader,
    class_hash: ClassHash,
) -> StateResult<Option<(ClassHash, CompiledClassHashV2ToV1)>> {
    let state_compiled_class_hash = state_reader.get_compiled_class_hash(class_hash)?;
    match state_compiled_class_hash {
        // Class hash does not exist in the state, or is a Cairo 0 class.
        CompiledClassHash(hash) if hash == StarkHash::ZERO => Ok(None),
        state_compiled_class_hash => {
            let compiled_class_hash_v2 = state_reader.get_compiled_class_hash_v2(
                class_hash,
                &state_reader.get_compiled_class(class_hash)?,
            )?;
            // If the state compiled class hash is compiled class hash v2, the class should not
            // migrate.
            if state_compiled_class_hash == compiled_class_hash_v2 {
                return Ok(None);
            }
            Ok(Some((class_hash, (compiled_class_hash_v2, state_compiled_class_hash))))
        }
    }
