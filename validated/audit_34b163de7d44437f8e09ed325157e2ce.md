## Analysis

I need to carefully examine both `overflowing_add` and `overflowing_mul` in `btc-types/src/u256.rs`, trace how they reach production paths, and determine whether a carry-propagation error can cause a security-critical divergence.

---

### `overflowing_add` — Carry Propagation

```rust
pub fn overflowing_add(self, rhs: Self) -> (Self, bool) {
    let (high, overflow) = self.0.overflowing_add(rhs.0);   // high words first
    ret.0 = high;
    ret_overflow |= overflow;

    let (low, overflow) = self.1.overflowing_add(rhs.1);    // low words second
    ret.1 = low;
    if overflow {                                             // carry from low → high
        let (high, overflow) = ret.0.overflowing_add(1);
        ret.0 = high;
        ret_overflow |= overflow;
    }
    (ret, ret_overflow)
}
``` [1](#0-0) 

The carry from the low 128-bit limb is correctly propagated to the high 128-bit limb. The overflow flag is correctly ORed at both the initial high-word addition and the carry-propagation step. **No bug here.**

### `overflowing_mul` — Carry Approximation

```rust
pub fn overflowing_mul(self, rhs: u64) -> (Self, bool) {
    let (high, overflow) = self.0.overflowing_mul(u128::from(rhs));
    let (low, overflow_low) = self.1.overflowing_mul(u128::from(rhs));

    if !overflow_low {
        return (Self(high, low), overflow);
    }

    let carry = ((self.1 >> 64) * (u128::from(rhs))) >> 64;
    let (high, overflow_add) = high.overflowing_add(carry);
    (Self(high, low), overflow | overflow_add)
}
``` [2](#0-1) 

Let `self.1 = a·2⁶⁴ + b` (where `a = self.1 >> 64`, `b = self.1 & (2⁶⁴−1)`). The true carry from `self.1 * rhs` overflowing u128 is:

```
true_carry = floor(a·rhs / 2⁶⁴) + floor((r₁·2⁶⁴ + b·rhs) / 2¹²⁸)
```

where `r₁ = (a·rhs) mod 2⁶⁴`. The code computes only `floor(a·rhs / 2⁶⁴)`, omitting the correction term. The correction is 1 when `r₁·2⁶⁴ + b·rhs ≥ 2¹²⁸`.

**This is a real arithmetic defect** — the carry can be underestimated by exactly 1 in specific cases. However, whether it is reachable and exploitable in production requires checking the actual input domain.
