### Title
`ValidResourceBounds` Protobuf Deserialization Silently Downgrades `AllResources` to `L1Gas` When L2/L1DataGas Are Zero, Causing Hash Mismatch and P2P/Consensus Rejection - (File: `crates/apollo_protobuf/src/converters/transaction.rs`)

### Summary

The protobuf deserializer for `ValidResourceBounds` uses a zero-value heuristic to distinguish legacy `L1Gas` transactions from modern `AllResources` transactions. When an `AllResources` invoke V3 transaction is submitted with `l2_gas = {0, 0}` and `l1_data_gas = {0, 0}`, the gateway accepts it and computes a hash over three resource-bound elements (the `AllResources` path). However, after a protobuf round-trip the same bytes are decoded as `ValidResourceBounds::L1Gas`, causing `RpcInvokeTransactionV3::try_from` to return `DEPRECATED_RESOURCE_BOUNDS_ERROR`. The transaction is therefore un-propagatable via the mempool P2P layer and, if the local node is the block proposer, causes every validator to fail deserialization of the proposal, disrupting consensus.

### Finding Description

**Root cause — `TryFrom<protobuf::ResourceBounds> for ValidResourceBounds`** [1](#0-0) 

When `l1_data_gas` is absent or zero **and** `l2_gas` is zero, the deserializer returns `ValidResourceBounds::L1Gas(l1_gas)` instead of `ValidResourceBounds::AllResources(...)`. This is a backward-compatibility path for pre-0.13.3 transactions (noted by the `TODO(Shahak)` comment on line 426), but it is also triggered by any post-0.13.3 `AllResources` transaction whose L2 and L1-data-gas bounds happen to be zero.

**Hash computation is branch-sensitive**

`get_tip_resource_bounds_hash` hashes a different number of elements depending on the variant: [2](#0-1) 

- `AllResources` → three elements: `[L1_GAS_concat, L2_GAS_concat(0), L1_DATA_GAS_concat(0)]`
- `L1Gas` → two elements: `[L1_GAS_concat, L2_GAS_concat(0)]`

These produce distinct Poseidon digests, so the hash committed at the gateway (H₁, `AllResources`) differs from the hash a peer would compute after deserialization (H₂, `L1Gas`).

**`RpcInvokeTransactionV3` requires `AllResources`** [3](#0-2) 

`TryFrom<InvokeTransactionV3> for RpcInvokeTransactionV3` returns `Err(OutOfRange)` for any `L1Gas` variant. The mempool P2P converter wraps this as `DEPRECATED_RESOURCE_BOUNDS_ERROR`: [4](#0-3) 

**Consensus path**

The consensus proposal serializer converts `InternalConsensusTransaction → ConsensusTransaction → protobuf::ConsensusTransaction`: [5](#0-4) 

A validator receiving the proposal calls `convert_consensus_tx_to_internal_consensus_tx`, which calls `convert_rpc_tx_to_internal`. The deserialization failure propagates up and the proposal is marked `Failed`: [6](#0-5) 

**Gateway always accepts `AllResourceBounds`**

`RpcInvokeTransactionV3.resource_bounds` is typed `AllResourceBounds` (never `ValidResourceBounds`), so the gateway always computes H₁ under the `AllResources` branch: [7](#0-6) 

The only relevant gateway check is `validate_tx_l2_gas_price_within_threshold`, which only validates `max_price_per_unit` and is skipped entirely for `L1Gas` variants: [8](#0-7) 

When `min_gas_price_percentage = 0` (threshold = 0), a transaction with `l2_gas.max_price_per_unit = 0` passes this check.

### Impact Explanation

**Matching impact: High — Mempool/gateway/RPC admission accepts valid transactions that cannot be propagated; High — Transaction conversion binds the wrong hash.**

1. The gateway accepts the transaction and stores it with hash H₁ (`AllResources`).
2. The mempool P2P propagator calls `convert_internal_rpc_tx_to_rpc_tx` → serializes to protobuf → peers deserialize as `L1Gas` → `RpcInvokeTransactionV3::try_from` fails → the transaction is silently dropped by every peer.
3. If the local node is the block proposer, it includes the transaction in a proposal. Every validator fails to deserialize the `InvokeV3WithProof` message and rejects the proposal, stalling consensus for that round.
4. The hash H₁ committed in the mempool and batcher is permanently inconsistent with the hash H₂ that any peer would derive from the same wire bytes, violating the canonical-hash invariant across the public-to-internal conversion boundary.

### Likelihood Explanation

Any unprivileged user can craft and submit an invoke V3 transaction with `AllResourceBounds { l2_gas: {0,0}, l1_data_gas: {0,0} }` via the standard `starknet_addInvokeTransaction` RPC endpoint. No special privilege, key material, or prior state is required. The condition is reachable whenever `min_gas_price_percentage = 0` (or the L2 gas price threshold is not enforced), which is a plausible configuration for testing or early-stage deployments.

### Recommendation

**Short term:** In `TryFrom<protobuf::ResourceBounds> for ValidResourceBounds`, remove the zero-value heuristic for the `MempoolTransaction` and `ConsensusTransaction` paths. For these message types, `l1_data_gas` should be treated as mandatory (assert it is `Some` and non-default), returning `DEPRECATED_RESOURCE_BOUNDS_ERROR` only for the legacy `TransactionInBlock` path where pre-0.13.3 transactions may legitimately appear.

**Long term:** Add a version or type discriminator field to the `ResourceBounds` protobuf message so that `AllResources` and `L1Gas` are unambiguously distinguishable on the wire, eliminating the collision between the two representations when all optional fields are zero.

### Proof of Concept

```
1. Craft an RpcInvokeTransactionV3 with:
     resource_bounds = AllResourceBounds {
         l1_gas:      { max_amount: 1000, max_price_per_unit: 1 },
         l2_gas:      { max_amount: 0,    max_price_per_unit: 0 },
         l1_data_gas: { max_amount: 0,    max_price_per_unit: 0 },
     }
   Sign it; the gateway computes hash H₁ using AllResources (3-element Poseidon chain).

2. Submit via starknet_addInvokeTransaction. Gateway accepts; transaction enters mempool
   with hash H₁.

3. Observe mempool P2P propagation: the MempoolP2pPropagator calls
   convert_internal_rpc_tx_to_rpc_tx → serializes to protobuf::InvokeV3WithProof
   with l2_gas={0,0}, l1_data_gas={0,0}.

4. Peer receives the protobuf bytes:
     TryFrom<protobuf::ResourceBounds> for ValidResourceBounds
       → l1_data_gas.is_zero() && l2_gas.is_zero() == true
       → returns ValidResourceBounds::L1Gas(l1_gas)
     TryFrom<InvokeTransactionV3> for RpcInvokeTransactionV3
       → resource_bounds is L1Gas, not AllResources
       → returns Err(DEPRECATED_RESOURCE_BOUNDS_ERROR)
   Transaction is dropped; it never reaches the peer's mempool.

5. If the originating node proposes a block containing this transaction:
     Validator calls TryFrom<protobuf::ConsensusTransaction> for ConsensusTransaction
       → same deserialization failure
     validate_proposal returns HandledProposalPart::Failed(...)
   The proposal is rejected; consensus round fails.

6. Hash divergence: H₁ (AllResources, 3 elements) ≠ H₂ (L1Gas, 2 elements).
   Any node that could deserialize the transaction would compute a different hash
   than the one stored in the originating node's mempool and batcher.
```

impl TryFrom<protobuf::ResourceBounds> for ValidResourceBounds {
    type Error = ProtobufConversionError;
    fn try_from(value: protobuf::ResourceBounds) -> Result<Self, Self::Error> {
        let Some(l1_gas) = value.l1_gas else {
            return Err(missing("ResourceBounds::l1_gas"));
        };
        let Some(l2_gas) = value.l2_gas else {
            return Err(missing("ResourceBounds::l2_gas"));
        };
        // TODO(Shahak): Assert data gas is not none once we remove support for 0.13.2.
        let l1_data_gas = value.l1_data_gas.unwrap_or_default();
        let l1_gas: ResourceBounds = l1_gas.try_into()?;
        let l2_gas: ResourceBounds = l2_gas.try_into()?;
        let l1_data_gas: ResourceBounds = l1_data_gas.try_into()?;
        Ok(if l1_data_gas.is_zero() && l2_gas.is_zero() {
            ValidResourceBounds::L1Gas(l1_gas)
        } else {
            ValidResourceBounds::AllResources(AllResourceBounds { l1_gas, l2_gas, l1_data_gas })
        })
    }

// An implementation of the SNIP: https://github.com/EvyatarO/SNIPs/blob/snip-8/SNIPS/snip-8.md
pub fn get_tip_resource_bounds_hash(
    resource_bounds: &ValidResourceBounds,
    tip: &Tip,
) -> Result<Felt, StarknetApiError> {
    let l1_resource_bounds = resource_bounds.get_l1_bounds();
    let l2_resource_bounds = resource_bounds.get_l2_bounds();

    // L1 and L2 gas bounds always exist.
    // Old V3 txs always have L2 gas bounds of zero, but they exist.
    let mut resource_felts = vec![
        get_concat_resource(&l1_resource_bounds, L1_GAS)?,
        get_concat_resource(&l2_resource_bounds, L2_GAS)?,
    ];

    // For new V3 txs, need to also hash the data gas bounds.
    resource_felts.extend(match resource_bounds {
        ValidResourceBounds::L1Gas(_) => vec![],
        ValidResourceBounds::AllResources(all_resources) => {
            vec![get_concat_resource(&all_resources.l1_data_gas, L1_DATA_GAS)?]
        }
    });

    Ok(HashChain::new().chain(&tip.0.into()).chain_iter(resource_felts.iter()).get_poseidon_hash())
}
