### Title
Dual `IsOwner` AND `IsDenomCreator` Guard in `EditNFT` Permanently Freezes NFT Metadata When Creator Mints to Non-Creator — (`x/nft/keeper/keeper.go`)

### Summary

`EditNFT` requires the caller to simultaneously satisfy both `IsOwner` and `IsDenomCreator`. Because `MintNFT` allows the denom creator to mint an NFT to any recipient address, a state is trivially reachable where no address satisfies both conditions at once, permanently freezing the NFT's metadata with no authorized editor.

### Finding Description

In `x/nft/keeper/keeper.go`, `EditNFT` enforces two sequential AND-gated authorization checks:

```go
nft, err := k.IsOwner(ctx, denomID, tokenID, owner)   // line 93
if err != nil { return err }

_, err = k.IsDenomCreator(ctx, denomID, owner)         // line 98
if err != nil { return err }
``` [1](#0-0) 

`IsOwner` returns `ErrUnauthorized` unless the caller is the current NFT owner. [2](#0-1) 

`IsDenomCreator` returns `ErrUnauthorized` unless the caller is the original denom creator. [3](#0-2) 

`MintNFT` only checks `IsDenomCreator` for the **sender**, and freely sets any address as the NFT **owner** (recipient):

```go
_, err := k.IsDenomCreator(ctx, denomID, sender)
...
return k.MintNFTUnverified(ctx, denomID, tokenID, tokenNm, tokenURI, tokenData, owner)
``` [4](#0-3) 

This means after creator A mints `tokenID` to non-creator B:

| Caller | `IsOwner` | `IsDenomCreator` | `EditNFT` result |
|--------|-----------|-----------------|-----------------|
| A | FAIL (B owns it) | PASS | `ErrUnauthorized` |
| B | PASS | FAIL (A is creator) | `ErrUnauthorized` |

No address can ever satisfy both guards simultaneously. The metadata is permanently frozen.

The same dual-check pattern also affects `BurnNFT`, making the NFT unburnable by either party as well: [5](#0-4) 

### Impact Explanation

Once a denom creator mints an NFT to any non-creator address, the NFT's metadata (`Name`, `URI`, `Data`) is permanently immutable — no transaction can ever update it. The NFT also cannot be burned. This destroys the utility of the NFT for any use case requiring metadata updates (e.g., dynamic NFTs, game assets, certificates). The state is irreversible without a chain upgrade.

### Likelihood Explanation

This is the **default and expected usage pattern**: a creator issues a denom and mints NFTs to users/buyers. Every such mint to a non-creator address triggers this freeze. No special attacker action is required — the creator's own normal workflow produces the broken state.

### Recommendation

Replace the AND-gate with an OR-gate: allow `EditNFT` if the caller is **either** the current NFT owner **or** the denom creator. The most semantically correct fix is to allow only the current owner to edit their own NFT's metadata:

```go
nft, err := k.IsOwner(ctx, denomID, tokenID, owner)
if err != nil {
    return err
}
// Remove the IsDenomCreator check entirely, or use OR logic
```

If the intent is to allow the denom creator to also edit metadata, use:

```go
_, ownerErr := k.IsOwner(ctx, denomID, tokenID, owner)
_, creatorErr := k.IsDenomCreator(ctx, denomID, owner)
if ownerErr != nil && creatorErr != nil {
    return types.ErrUnauthorized
}
```

func (k Keeper) EditNFT(
	ctx sdk.Context, denomID, tokenID, tokenNm,
	tokenURI, tokenData string, owner sdk.AccAddress,
) error {
	if !k.HasDenomID(ctx, denomID) {
		return sdkerrors.Wrapf(types.ErrInvalidDenom, "denom ID %s not exists", denomID)
	}

	nft, err := k.IsOwner(ctx, denomID, tokenID, owner)
	if err != nil {
		return err
	}

	_, err = k.IsDenomCreator(ctx, denomID, owner)
	if err != nil {
		return err
	}

	if types.Modified(tokenNm) {
		nft.Name = tokenNm
	}

	if types.Modified(tokenURI) {
		nft.URI = tokenURI
	}

	if types.Modified(tokenData) {
		nft.Data = tokenData
	}

	k.setNFT(ctx, denomID, nft)

	return nil
}

func (k Keeper) IsOwner(ctx sdk.Context, denomID, tokenID string, owner sdk.AccAddress) (types.BaseNFT, error) {
	nft, err := k.GetNFT(ctx, denomID, tokenID)
	if err != nil {
		return types.BaseNFT{}, err
	}

	if !owner.Equals(nft.GetOwner()) {
		return types.BaseNFT{}, sdkerrors.Wrapf(types.ErrUnauthorized, "%s is not the owner of %s/%s", owner.String(), denomID, tokenID)
	}

	return nft.(types.BaseNFT), nil
}

func (k Keeper) IsDenomCreator(ctx sdk.Context, denomID string, address sdk.AccAddress) (types.Denom, error) {
	denom, err := k.GetDenom(ctx, denomID)
	if err != nil {
		return types.Denom{}, err
	}

	creator, err := sdk.AccAddressFromBech32(denom.Creator)
	if err != nil {
		panic(err)
	}

	if !creator.Equals(address) {
		return types.Denom{}, sdkerrors.Wrapf(types.ErrUnauthorized, "%s is not the creator of %s", address, denomID)
	}

	return denom, nil
