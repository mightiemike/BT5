Audit Report

## Title
`SwapAllowlistExtension` checks the router address instead of the end-user, allowing any user to bypass a curated pool's swap allowlist - (File: metric-periphery/contracts/extensions/SwapAllowlistExtension.sol)

## Summary
`SwapAllowlistExtension.beforeSwap` gates swaps by checking the `sender` argument against `allowedSwapper[pool][sender]`. When `MetricOmmSimpleRouter` is used, `sender` equals the router's address because `MetricOmmPool.swap` passes `msg.sender` directly. A pool admin who allowlists the router to enable standard swap UX inadvertently grants every user of the router the ability to swap, defeating the individual allowlist entirely and exposing LP capital to unvetted traders.

## Finding Description
`MetricOmmPool.swap` passes `msg.sender` as the first argument to `_beforeSwap`: [1](#0-0) 

`ExtensionCalling._beforeSwap` forwards that value as `sender` to every configured extension via `abi.encodeCall`: [2](#0-1) 

`SwapAllowlistExtension.beforeSwap` then checks `allowedSwapper[msg.sender][sender]`, where `msg.sender` is the pool (the extension's caller) and `sender` is whoever called `pool.swap`: [3](#0-2) 

`MetricOmmSimpleRouter.exactInputSingle` (and all other router entry-points) calls `pool.swap` directly, making the router the `msg.sender` of that call: [4](#0-3) 

Therefore the extension evaluates `allowedSwapper[pool][router]`, not `allowedSwapper[pool][actual_user]`. The router is a single address that proxies all users; allowlisting it collapses the per-user allowlist into an all-or-nothing gate. There is no mechanism in the extension or the pool to recover the true end-user identity from the call stack.

By contrast, `DepositAllowlistExtension` correctly checks the `owner` argument (the position owner explicitly passed by the caller), not `sender`, so it is not affected: [5](#0-4) 

## Impact Explanation
A curated pool using `SwapAllowlistExtension` to restrict trading to vetted counterparties loses that protection the moment the router is allowlisted. Any unprivileged user can route through `MetricOmmSimpleRouter` and execute swaps at oracle prices against LP capital that was never intended to be exposed to them. LP assets are directly at risk from adverse selection, constituting a direct loss of LP principal above Sherlock thresholds.

## Likelihood Explanation
Allowlisting the router is the expected operational step for any pool that wants to support the protocol's standard swap UX. The admin has no way to simultaneously allow router-based swaps for vetted users only and block unvetted users, because the extension cannot distinguish the two cases. The precondition (router allowlisted) is a normal, non-malicious configuration that any pool admin would apply. The attack requires no special privileges and is repeatable by any user.

## Recommendation
The extension must identify the true economic actor, not the intermediary contract. Two viable approaches:

1. **Pass the real user through `extensionData`**: Have the router ABI-encode `msg.sender` (the end-user) into `extensionData` and have the extension decode and verify it. The pool already forwards `extensionData` unmodified to every hook via `_beforeSwap`.
2. **Check `recipient` instead of `sender`**: For swap allowlists the recipient is often the meaningful actor; the extension already receives `recipient` as its second argument and could gate on that instead of (or in addition to) `sender`.

## Proof of Concept
```
Setup:
  pool configured with SwapAllowlistExtension (beforeSwap order)
  admin calls extension.setAllowedToSwap(pool, router, true)
    → allowedSwapper[pool][router] = true
  alice (not individually allowlisted) holds token0

Attack:
  alice calls router.exactInputSingle({pool: pool, ...})
  router calls pool.swap(recipient, zeroForOne, amount, ...)
    → msg.sender inside pool = router
  pool calls _beforeSwap(msg.sender=router, ...)
  extension checks allowedSwapper[pool][router] → true → passes
  swap executes at oracle price against LP capital
  alice receives token1 output

Result:
  alice, who is not on the individual allowlist, successfully swaps
  on a pool whose admin intended to restrict access to vetted traders
```
