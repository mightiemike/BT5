Audit Report

## Title
`SwapAllowlistExtension.beforeSwap` checks the router address as `sender` instead of the actual end-user, enabling allowlist bypass - (File: metric-periphery/contracts/extensions/SwapAllowlistExtension.sol)

## Summary
`MetricOmmPool.swap()` passes `msg.sender` (the immediate caller, e.g. the router) as the `sender` argument to `_beforeSwap`. `SwapAllowlistExtension.beforeSwap` then checks `allowedSwapper[pool][sender]` against that router address rather than the actual end-user. If the pool admin allowlists the router to enable router-based swaps, every unprivileged user can bypass the per-user allowlist by routing through `MetricOmmSimpleRouter`.

## Finding Description

**Call chain:**

1. User calls `MetricOmmSimpleRouter.exactInputSingle(...)`.
2. Router calls `IMetricOmmPoolActions(pool).swap(recipient, zeroForOne, amountSpecified, priceLimitX64, "", extensionData)` — here `msg.sender` inside the pool is the **router address**.
3. `MetricOmmPool.swap()` (line 230) calls:
   ```solidity
   _beforeSwap(
     msg.sender,   // ← router address, NOT the end-user
     recipient, zeroForOne, amountSpecified, priceLimitX64,
     packedSlot0Initial, bidPriceX64, askPriceX64, extensionData
   );
   ```
4. `SwapAllowlistExtension.beforeSwap` receives `sender = router_address` and checks:
   ```solidity
   if (!allowAllSwappers[msg.sender] && !allowedSwapper[msg.sender][sender]) {
     revert NotAllowedToSwap();
   }
   ```
   `msg.sender` here is the pool (correct), but `sender` is the **router**, not the actual trader.

**Root cause:** The allowlist is keyed on the immediate caller of `pool.swap()`, not on the originating user. When the pool admin allowlists the router address (a natural step to enable router-based swaps), the check `allowedSwapper[pool][router] == true` passes for **every** user who calls through the router, regardless of whether that user is individually allowlisted.

**Existing guards are insufficient:** The extension has no mechanism to unwrap the true originator. The pool only forwards `msg.sender`, and the router does not pass the original `msg.sender` through `callbackData` or `extensionData` in any authenticated way.

## Impact Explanation
The `SwapAllowlistExtension` is documented as "Gates `swap` by swapper address, per pool." When the router is allowlisted, this gate is completely bypassed for all router callers. Any unprivileged trader can swap in a pool that the admin intended to restrict to a specific set of addresses. This is an admin-boundary break: the pool admin's access-control invariant is circumvented by an unprivileged path through the router.

## Likelihood Explanation
The router is the primary user-facing entry point. A pool admin who deploys a swap allowlist and also wants to support router-based swaps for their allowlisted users must allowlist the router address. Once the router is allowlisted, the bypass is trivially reachable by any caller of `exactInputSingle`, `exactInput`, `exactOutputSingle`, or `exactOutput` — no special privileges or setup required. The condition (router allowlisted) is a natural and expected operational state.

## Recommendation
Pass the true originating user through the extension data or a dedicated transient-storage slot, and have the extension read that value instead of the raw `sender` argument. Alternatively, the pool could accept an explicit `originator` parameter that the router populates with `msg.sender` before calling `pool.swap`, and the extension checks `allowedSwapper[pool][originator]`. The router's own `msg.sender` capture at entry (before any delegate-call or callback) is the correct value to authenticate.

## Proof of Concept

```solidity
// 1. Deploy pool with SwapAllowlistExtension configured.
// 2. Pool admin allowlists the router:
extension.setAllowedToSwap(pool, address(router), true);
// 3. Pool admin does NOT allowlist `attacker`.
// 4. Attacker calls:
router.exactInputSingle(ExactInputSingleParams({
    pool: pool,
    tokenIn: token0,
    recipient: attacker,
    zeroForOne: true,
    amountIn: 1e18,
    amountOutMinimum: 0,
    priceLimitX64: 0,
    deadline: block.timestamp,
    extensionData: ""
}));
// 5. Pool calls _beforeSwap(msg.sender=router, ...).
// 6. Extension checks allowedSwapper[pool][router] == true → passes.
// 7. Attacker swaps successfully despite not being individually allowlisted.
```
