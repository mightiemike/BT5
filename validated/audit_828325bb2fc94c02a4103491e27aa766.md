### Title
`validate_resource_bounds` Skips `max_l2_gas_amount` Enforcement for Declare Transactions, Allowing Unbounded Gas Bound Admission — (File: `crates/apollo_gateway/src/stateless_transaction_validator.rs`)

### Summary

`StatelessTransactionValidator::validate_resource_bounds` enforces a `max_l2_gas_amount` ceiling on the `l2_gas.max_amount` field for Invoke and DeployAccount transactions, but explicitly skips this check for Declare transactions via a no-op branch. Any unprivileged user can submit a `RpcDeclareTransaction::V3` with `l2_gas.max_amount` set to `u64::MAX`, bypassing the gateway's admission-control ceiling and causing the transaction to be accepted into the mempool with a resource bound that would be rejected for every other transaction type.

### Finding Description

In `crates/apollo_gateway/src/stateless_transaction_validator.rs`, the `validate_resource_bounds` function applies two checks to all incoming transactions:

1. Non-zero resource bounds (all types).
2. `l2_gas.max_price_per_unit >= min_gas_price` (all types).
3. `l2_gas.max_amount <= max_l2_gas_amount` — **Invoke and DeployAccount only**.

The third check is gated by an explicit no-op branch:

```rust
// TODO(Arni): Consider adding a validation for max_l2_gas_amount for declare.
if let RpcTransaction::Declare(_) = tx {
} else if resource_bounds.l2_gas.max_amount.0 > self.config.max_l2_gas_amount {
    return Err(StatelessTransactionValidatorError::MaxGasAmountTooHigh { ... });
}
```

The `if let RpcTransaction::Declare(_) = tx {}` arm does nothing and falls through to `Ok(())`, so a Declare transaction with `l2_gas.max_amount = u64::MAX` passes `validate_resource_bounds` without error. The transaction then proceeds through `convert_rpc_tx_to_internal_rpc_tx` → `calculate_transaction_hash` → mempool admission, all of which accept it.

This is structurally identical to the `recoverErc20` bug: a restriction function that blocks some members of a set (Invoke, DeployAccount) but leaves one member (Declare) unguarded, allowing an unprivileged caller to bypass the intended invariant.

### Impact Explanation

**Impact: High — Mempool/gateway admission accepts invalid transactions before sequencing.**

A Declare transaction with `l2_gas.max_amount > max_l2_gas_amount` is invalid under the gateway's own admission policy, yet it is accepted. The gateway's `max_l2_gas_amount` ceiling is the sole stateless guard against transactions that commit to an unrealistically large L2 gas budget. Bypassing it for Declare transactions means:

- The mempool accepts Declare transactions that would be rejected for any other type.
- The batcher picks them up and attempts execution; the blockifier then rejects them at pre-validation (insufficient balance to cover `max_amount × max_price`), wasting batcher and execution resources.
- Because the gateway is the admission boundary, a sustained stream of such transactions cannot be filtered before they consume mempool slots and batcher cycles.

### Likelihood Explanation

Likelihood is **high**: the trigger requires no privilege, no special key, and no knowledge beyond the public RPC API. Any caller who can submit a `starknet_addDeclareTransaction` RPC call can set `l2_gas.max_amount` to an arbitrary value. The TODO comment confirms the gap is known but unresolved.

### Recommendation

Remove the no-op `if let RpcTransaction::Declare(_) = tx {}` branch and apply the `max_l2_gas_amount` check uniformly to all transaction types, or introduce a separate, appropriately-sized ceiling for Declare transactions:

```rust
// Apply to all transaction types, including Declare.
if resource_bounds.l2_gas.max_amount.0 > self.config.max_l2_gas_amount {
    return Err(StatelessTransactionValidatorError::MaxGasAmountTooHigh {
        gas_amount: resource_bounds.l2_gas.max_amount,
        max_gas_amount: self.config.max_l2_gas_amount,
    });
}
```

If Declare transactions legitimately require a higher ceiling (e.g., for large Sierra programs), introduce a dedicated `max_l2_gas_amount_declare` config field rather than removing the check entirely.

### Proof of Concept

1. Construct a valid `RpcDeclareTransactionV3` with `resource_bounds.l2_gas.max_amount = u64::MAX` and `resource_bounds.l2_gas.max_price_per_unit = 1` (satisfies the non-zero and min-price checks).
2. Submit it via `POST /gateway/add_transaction` or the `starknet_addDeclareTransaction` JSON-RPC endpoint.
3. Observe that `StatelessTransactionValidator::validate_resource_bounds` returns `Ok(())` — the `max_l2_gas_amount` branch is skipped.
4. The transaction is forwarded to `convert_rpc_tx_to_internal_rpc_tx`, a transaction hash is computed, and the transaction is inserted into the mempool.
5. Repeat with a fresh nonce to fill the mempool with Declare transactions whose committed fee (`u64::MAX × price`) is unpayable, causing repeated pre-validation failures in the batcher.

    fn validate_resource_bounds(
        &self,
        tx: &RpcTransaction,
    ) -> StatelessTransactionValidatorResult<()> {
        if !self.config.validate_resource_bounds {
            return Ok(());
        }

        let resource_bounds = *tx.resource_bounds();
        // The resource bounds should be positive even without the tip.
        if ValidResourceBounds::AllResources(resource_bounds).max_possible_fee(Tip::ZERO) == Fee(0)
        {
            return Err(StatelessTransactionValidatorError::ZeroResourceBounds { resource_bounds });
        }

        if resource_bounds.l2_gas.max_price_per_unit.0 < self.config.min_gas_price {
            return Err(StatelessTransactionValidatorError::MaxGasPriceTooLow {
                gas_price: resource_bounds.l2_gas.max_price_per_unit,
                min_gas_price: self.config.min_gas_price,
            });
        }

        // TODO(Arni): Consider adding a validation for max_l2_gas_amount for declare.
        if let RpcTransaction::Declare(_) = tx {
        } else if resource_bounds.l2_gas.max_amount.0 > self.config.max_l2_gas_amount {
            return Err(StatelessTransactionValidatorError::MaxGasAmountTooHigh {
                gas_amount: resource_bounds.l2_gas.max_amount,
                max_gas_amount: self.config.max_l2_gas_amount,
            });
        }

        Ok(())
    }
