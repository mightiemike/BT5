### Title
Bonus Reward Over-Accrual When Position Shares Increase Without Prior Settlement — (`x/tieredrewards/keeper/bonus_rewards.go`)

### Summary

The `x/tieredrewards` module computes bonus rewards using the position's **current** delegation shares for every historical time segment. Because `BeforeDelegationSharesModified` is a no-op hook, adding tokens to an existing position (e.g., via `MsgAddToPosition`) without first settling accrued rewards causes the module to retroactively apply the larger post-increase share count to all past segments, over-paying bonus rewards from the `RewardsPoolName` pool.

---

### Finding Description

`computeSegmentBonus` in `bonus_rewards.go` computes the bonus for a time segment as:

```
tokens = pos.Delegation.Shares × tokensPerShare
bonus  = tokens × tier.BonusApy × durationSeconds / SecondsPerYear
``` [1](#0-0) 

`pos.Delegation.Shares` is not a stored snapshot — it is fetched **live** from the staking module every time `getPositionState` is called: [2](#0-1) 

`processEventsAndClaimBonus` calls `computeSegmentBonus` for every bonded segment since `LastBonusAccrual`, passing the same `*pos` (and therefore the same `pos.Delegation.Shares`) to every segment: [3](#0-2) 

The natural place to settle rewards before shares change is the `BeforeDelegationSharesModified` staking hook. That hook is explicitly a **no-op**: [4](#0-3) 

By contrast, the developers correctly settled rewards *before* a slash in `slashRedelegationPosition` via the `BeforeRedelegationSlashed` hook: [5](#0-4) 

This shows the developers understand the pattern — settle first, then modify shares — but it was not applied to the share-increase path.

---

### Impact Explanation

An attacker who holds a position with a small share amount `S1` can:

1. Wait for a long unclaimed period `[T0, T1]` to accumulate.
2. Add a large amount of tokens to the position (increasing shares to `S2 >> S1`) via `MsgAddToPosition` **without** a prior reward settlement.
3. Call `MsgClaimTierRewards`. `processEventsAndClaimBonus` now computes the bonus for the entire period `[T0, T1]` using `S2` instead of `S1`.

The over-payment per claim is:

```
(S2 − S1) × tokensPerShare × tier.BonusApy × (T1 − T0) / SecondsPerYear
```

This drains the `RewardsPoolName` module account faster than intended, directly reducing the bonus rewards available to all other legitimate position holders. The corrupted value is the `RewardsPoolName` balance and the `tokenRewards` credited to the attacker.

---

### Likelihood Explanation

The entry path is a standard, unprivileged user transaction (`MsgAddToPosition`). No special role or leaked key is required. The attacker only needs an existing position and enough tokens to add to it. The longer they wait before adding tokens, the larger the over-payment. This is a realistic, low-barrier attack.

---

### Recommendation

Before modifying a position's delegation shares (in `MsgAddToPosition` and any other share-increasing message handler), call `processEventsAndClaimBonus` to settle accrued rewards at the current share amount and advance `LastBonusAccrual` to the current block time. This is the same pattern already used in `slashRedelegationPosition`.

Alternatively, implement `BeforeDelegationSharesModified` to trigger reward settlement whenever the staking module is about to modify the delegation shares of a position's delegator address.

---

### Proof of Concept

1. Alice creates a position with `1_000 uatom` (shares `S1`). `LastBonusAccrual = T0`.
2. 365 days pass with no claims. The validator remains bonded throughout.
3. Alice calls `MsgAddToPosition` with `999_000 uatom`, bringing shares to `S2 = 1000 × S1`. No reward settlement occurs before the delegation is modified.
4. Alice calls `MsgClaimTierRewards`. `processEventsAndClaimBonus` computes:
   - `bonus = S2 × tokensPerShare × BonusApy × 365days / SecondsPerYear`
   - Expected (correct): `S1 × tokensPerShare × BonusApy × 365days / SecondsPerYear`
   - Over-payment factor: **1000×**
5. Alice receives 1000× the bonus she legitimately earned, draining the `RewardsPoolName` pool. [6](#0-5) [7](#0-6) [8](#0-7)

func (k Keeper) computeSegmentBonus(pos types.PositionState, tier types.Tier, segmentStart, segmentEnd time.Time, tokensPerShare math.LegacyDec) math.Int {
	if !pos.ExitUnlockAt.IsZero() && segmentEnd.After(pos.ExitUnlockAt) {
		segmentEnd = pos.ExitUnlockAt
	}

	if !segmentEnd.After(segmentStart) {
		return math.ZeroInt()
	}

	durationSeconds := int64(segmentEnd.Sub(segmentStart) / time.Second)
	if durationSeconds <= 0 {
		return math.ZeroInt()
	}

	tokens := pos.Delegation.Shares.Mul(tokensPerShare)

	return tokens.
		Mul(tier.BonusApy).
		MulInt64(durationSeconds).
		QuoInt64(types.SecondsPerYear).
		TruncateInt()
}
```

func (k Keeper) processEventsAndClaimBonus(ctx context.Context, pos *types.PositionState) (sdk.Coins, error) {
	// Rewards should have been claimed before undelegation
	if !pos.IsDelegated() {
		return sdk.NewCoins(), nil
	}

	valAddr, err := sdk.ValAddressFromBech32(pos.Delegation.ValidatorAddress)
	if err != nil {
		return nil, err
	}

	events, err := k.getValidatorEventsSince(ctx, valAddr, pos.LastEventSeq)
	if err != nil {
		return nil, err
	}

	sdkCtx := sdk.UnwrapSDKContext(ctx)
	blockTime := sdkCtx.BlockTime()

	totalBonus := math.ZeroInt()
	// Use the persisted bonded state from the last replay, not a hardcoded default.
	// This prevents overpaying bonus for unbonded gaps between claims.
	bonded := pos.LastKnownBonded
	segmentStart := pos.LastBonusAccrual

	tier, err := k.getTier(ctx, pos.TierId)
	if err != nil {
		return nil, err
	}

	for _, entry := range events {
		evt := entry.Event

		if bonded {
			// Compute bonus for the bonded segment [segmentStart, eventTime]
			// using the snapshot rate at the event.
			bonus := k.computeSegmentBonus(*pos, tier, segmentStart, evt.Timestamp, evt.TokensPerShare)
			totalBonus = totalBonus.Add(bonus)
		}

		// Update bonded state based on event type.
		switch evt.EventType {
		case types.ValidatorEventType_VALIDATOR_EVENT_TYPE_UNBOND:
			bonded = false
		case types.ValidatorEventType_VALIDATOR_EVENT_TYPE_BOND:
			bonded = true
		case types.ValidatorEventType_VALIDATOR_EVENT_TYPE_SLASH:
			// Slash doesn't change bonded state.
		}

		segmentStart = evt.Timestamp
		pos.UpdateLastEventSeq(entry.Seq)

		// Decrement reference count.
		if err := k.decrementEventRefCount(ctx, valAddr, entry.Seq); err != nil {
			return nil, err
		}
	}

	val, err := k.stakingKeeper.GetValidator(ctx, valAddr)
	if err != nil {
		return nil, err
	}
	// Defensive: validator bond status check
	if bonded && val.IsBonded() {
		currentRate, err := k.getTokensPerShare(ctx, valAddr)
		if err != nil {
			return nil, err
		}
		bonus := k.computeSegmentBonus(*pos, tier, segmentStart, blockTime, currentRate)
		totalBonus = totalBonus.Add(bonus)
	}

	applyBonusAccrualCheckpoint(&pos.Position, blockTime)
