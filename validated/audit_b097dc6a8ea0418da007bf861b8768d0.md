### Title
Malicious Counterparty Chain Can Unescrow Arbitrary Escrowed NFTs via Crafted ClassId — (`x/nft-transfer/keeper/packet.go`)

### Summary

`processReceivedPacket` determines the transfer direction using `IsAwayFromOrigin(packet.GetSourcePort(), packet.GetSourceChannel(), data.ClassId)`. Because `data.ClassId` is fully attacker-controlled application-layer data (IBC core does not validate it), a malicious counterparty chain can craft a `ClassId` that starts with its own port/channel prefix, forcing `IsAwayFromOrigin` to return `false`. The unescrow branch then calls `TransferOwner` from the receiving chain's escrow address — which holds legitimately escrowed NFTs — to the attacker's receiver, with no verification that a prior escrow in the same packet lifecycle exists.

---

### Finding Description

**Root cause — `IsAwayFromOrigin` is trivially spoofable via `data.ClassId`:** [1](#0-0) 

```go
func IsAwayFromOrigin(sourcePort, sourceChannel, fullClassPath string) bool {
    prefixClassID := GetClassPrefix(sourcePort, sourceChannel)
    return !strings.HasPrefix(fullClassPath, prefixClassID)
}
```

`sourcePort` and `sourceChannel` here are `packet.GetSourcePort()` / `packet.GetSourceChannel()` — the counterparty's own port and channel, which the attacker controls. `fullClassPath` is `data.ClassId`, which the attacker also controls. The attacker simply sets `data.ClassId = sourcePort + "/" + sourceChannel + "/" + nativeClass` to make the prefix check succeed and force `isAwayFromOrigin = false`.

`escrowAddress` is derived from `packet.GetDestPort()` / `packet.GetDestChannel()` — the receiving chain's own port/channel. This is the same address that holds NFTs legitimately escrowed by real users on the receiving chain. `TransferOwner` only checks that `escrowAddress` is the current owner; there is no invariant tying this unescrow to a prior escrow in the same packet lifecycle. ### Impact Explanation

**Concrete attack steps (chain B = victim, chain A = attacker):**

1. A legitimate user on chain B sends `nativeClass/tokenID` to chain A via channel `(nft-transfer, channel-B)` on chain B / `(nft-transfer, channel-A)` on chain A. Chain B's `createOutgoingPacket` escrows the NFT to `GetEscrowAddress("nft-transfer", "channel-B")`.

2. The attacker (chain A) crafts a packet to chain B with:
   - `packet.SourcePort = "nft-transfer"`, `packet.SourceChannel = "channel-A"` (fixed by IBC channel)
   - `packet.DestPort = "nft-transfer"`, `packet.DestChannel = "channel-B"` (fixed by IBC channel)
   - `data.ClassId = "nft-transfer/channel-A/nativeClass"` (attacker-controlled)
   - `data.TokenIds = ["tokenID"]`
   - `data.Receiver = <attacker address on chain B>`

3. In `processReceivedPacket` on chain B:
   - `IsAwayFromOrigin("nft-transfer", "channel-A", "nft-transfer/channel-A/nativeClass")` → `false` (prefix matches)
   - `escrowAddress = GetEscrowAddress("nft-transfer", "channel-B")` — holds the victim's NFT
   - `unprefixedClassID = RemoveClassPrefix("nft-transfer", "channel-A", "nft-transfer/channel-A/nativeClass")` = `"nativeClass"`
   - `voucherClassID = "nativeClass"` (no path → `IBCClassID()` returns base class)
   - `TransferOwner(ctx, "nativeClass", "tokenID", escrowAddress, attackerReceiver)` — **succeeds**

4. The victim's NFT is transferred to the attacker. The attacker also retains the voucher minted on chain A from step 1, enabling a double-spend.

**State delta:** `nativeClass/tokenID` ownership changes from `escrowAddress("nft-transfer","channel-B")` to attacker's address on chain B, without any corresponding burn of a voucher on chain A.

---

### Likelihood Explanation

Any chain connected via IBC NFT transfer that is operated by a malicious or compromised validator set can execute this. The attacker only needs to wait for any user on the victim chain to escrow an NFT outbound. The IBC core layer does not inspect `data.ClassId` — it is purely application-layer data. No governance or privileged access is required beyond controlling the counterparty chain's block production.

---

### Recommendation

The receiving chain must verify that a "back-to-origin" unescrow is legitimate. The standard ICS-721 fix is to check that the `ClassId` in the incoming packet was actually prefixed by the **destination** port/channel (not the source), meaning the NFT was previously sent from this chain to the counterparty. Specifically:

- `IsAwayFromOrigin` in `processReceivedPacket` should be evaluated against `packet.GetDestPort()` / `packet.GetDestChannel()` and the full class path, not the source port/channel.
- Alternatively, maintain an explicit escrow record (class + tokenID → escrowed-via-channel) and require it to exist before unescrow.

### Proof of Concept

```go
// In processReceivedPacket on chain B:
// packet.SourcePort = "nft-transfer", packet.SourceChannel = "channel-A"
// packet.DestPort   = "nft-transfer", packet.DestChannel   = "channel-B"
// data.ClassId      = "nft-transfer/channel-A/nativeClass"  ← attacker-crafted
// data.TokenIds     = ["tokenID"]
// data.Receiver     = attackerAddr

// Step 1: IsAwayFromOrigin check
prefixClassID := "nft-transfer/channel-A/"
strings.HasPrefix("nft-transfer/channel-A/nativeClass", "nft-transfer/channel-A/") // true
// → isAwayFromOrigin = false  ← attacker wins the branch

// Step 2: escrow address is the victim chain's own escrow
escrowAddress := GetEscrowAddress("nft-transfer", "channel-B")
// This address holds nativeClass/tokenID escrowed by a real user

// Step 3: unprefixed class
unprefixedClassID := RemoveClassPrefix("nft-transfer", "channel-A", "nft-transfer/channel-A/nativeClass")
// = "nativeClass"
voucherClassID := ParseClassTrace("nativeClass").IBCClassID()
// = "nativeClass"  (no path → returns base class directly)

// Step 4: TransferOwner succeeds — escrow address owns the NFT
TransferOwner(ctx, "nativeClass", "tokenID", escrowAddress, attackerAddr)
// NFT stolen. Attacker also retains the chain-A voucher → double spend.
```

```go
func (k Keeper) TransferOwner(
	ctx sdk.Context, denomID, tokenID string, srcOwner, dstOwner sdk.AccAddress,
) error {
	if !k.HasDenomID(ctx, denomID) {
		return sdkerrors.Wrapf(types.ErrInvalidDenom, "denom ID %s not exists", denomID)
	}

	nft, err := k.IsOwner(ctx, denomID, tokenID, srcOwner)
	if err != nil {
		return err
	}

	nft.Owner = dstOwner.String()

	k.setNFT(ctx, denomID, nft)
	k.swapOwner(ctx, denomID, tokenID, srcOwner, dstOwner)
	return nil
}
```
