Audit Report

## Title
`SwapAllowlistExtension.beforeSwap` checks router identity instead of end-user identity, allowing any trader to bypass the swap allowlist - (File: metric-periphery/contracts/extensions/SwapAllowlistExtension.sol)

## Summary
`SwapAllowlistExtension.beforeSwap` receives `sender` as the address of whoever called `MetricOmmPool.swap` — which is the router (`MetricOmmSimpleRouter`), not the end user. Because the allowlist check uses this `sender` value, allowlisting the router grants every user unrestricted swap access, completely defeating the per-user gating the extension is designed to enforce.

## Finding Description

**Call chain:**

1. `MetricOmmSimpleRouter.exactInputSingle` (line 72) calls `IMetricOmmPoolActions(params.pool).swap(params.recipient, ...)`. The router is `msg.sender` from the pool's perspective.

2. `MetricOmmPool.swap` (line 230–240) calls `_beforeSwap(msg.sender, recipient, ...)`. Here `msg.sender` is the **router address**, not the end user.

3. `ExtensionCalling._beforeSwap` (line 160–176) encodes `(sender, ...)` and dispatches to the extension via `CallExtension.callExtension`. `sender` = router address.

4. `SwapAllowlistExtension.beforeSwap` (line 37):
```solidity
if (!allowAllSwappers[msg.sender] && !allowedSwapper[msg.sender][sender]) {
    revert IMetricOmmPoolActions.NotAllowedToSwap();
}
```
- `msg.sender` = pool address (the pool calls the extension)
- `sender` = **router address** (not the end user)

The extension is documented as "Gates `swap` by swapper address, per pool." The intended `sender` is the end user, but the actual `sender` is the router.

**Exploit path:**

- Pool admin deploys a pool with `SwapAllowlistExtension` to restrict swaps to a whitelist of addresses.
- Pool admin allowlists the router (necessary for normal router-based swaps to work).
- Any non-allowlisted user calls `MetricOmmSimpleRouter.exactInputSingle` → pool receives `msg.sender = router` → `allowedSwapper[pool][router] == true` → check passes → swap executes.
- The allowlist is completely bypassed for every user routing through `MetricOmmSimpleRouter`.

**Existing guards are insufficient:** The only guard is the `allowedSwapper[pool][sender]` check in `SwapAllowlistExtension.beforeSwap`, but `sender` is structurally bound to the immediate caller of `pool.swap`, not the originating user. There is no mechanism in `ExtensionCalling._beforeSwap` or `MetricOmmPool.swap` to propagate the original `tx.origin` or a user-supplied identity.

## Impact Explanation
The swap allowlist — a core access-control feature — is rendered ineffective for any pool using `MetricOmmSimpleRouter`. Any unprivileged trader can execute swaps in a pool that the admin intended to restrict to specific addresses. This is a broken core pool functionality / admin-boundary break: the pool admin's configured access control is bypassed by an unprivileged path (the public router), allowing unauthorized swaps that drain pool liquidity against the admin's intent.

## Likelihood Explanation
Exploitability requires only that: (1) a pool is configured with `SwapAllowlistExtension`, and (2) the router is allowlisted (the standard operational setup). Both conditions are the expected production configuration. Any unprivileged trader can exploit this immediately and repeatedly with no special capability.

## Recommendation
Pass the originating user identity through the call chain. One approach: have `MetricOmmPool.swap` accept an explicit `swapper` parameter (or read it from a trusted transient-storage slot set by the router before calling `swap`), and pass that address as `sender` to `_beforeSwap`. The router would write `msg.sender` (the end user) into transient storage before calling `pool.swap`, and the pool would read it for extension dispatch. Alternatively, `SwapAllowlistExtension` could use `tx.origin` as a fallback, though that has its own limitations. The fix must ensure the `sender` value reaching `beforeSwap` is the end user, not the router.

// pool.swap is called with msg.sender = router
// _beforeSwap passes sender = router to SwapAllowlistExtension
// allowedSwapper[pool][router] == true → check passes
// Attacker receives token1 output despite not being allowlisted
// SwapAllowlistExtension invariant broken
