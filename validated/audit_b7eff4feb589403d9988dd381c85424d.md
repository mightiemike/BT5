### Title
Slow Mode Fees Permanently Locked in Endpoint Contract — (`core/contracts/EndpointStorage.sol`)

---

### Summary

Every slow mode fee paid by users is transferred to `address(this)` (the `Endpoint` contract) and tracked in `slowModeFees`, but no function exists anywhere in the protocol to withdraw or recover these accumulated ERC-20 tokens. They are permanently locked.

---

### Finding Description

In `EndpointStorage.sol`, `chargeSlowModeFee` pulls the quote token from the user directly into the `Endpoint` contract:

```solidity
// EndpointStorage.sol L83-93
function chargeSlowModeFee(IERC20Base token, address from) internal virtual {
    require(address(token) != address(0));
    token.safeTransferFrom(
        from,
        address(this),   // <-- tokens land in Endpoint itself
        clearinghouse.getSlowModeFee()
    );
}
``` [1](#0-0) 

This is called from `EndpointTx.submitSlowModeTransactionImpl` for every non-privileged slow mode transaction type, and the running total is recorded in `slowModeFees`:

```solidity
// EndpointTx.sol L370-371
chargeSlowModeFee(_getQuote(), sender);
slowModeFees += SLOW_MODE_FEE;
``` [2](#0-1) 

`slowModeFees` is declared as a plain storage variable with no associated withdrawal path: [3](#0-2) 

The `DumpFees` slow mode transaction — the only fee-recovery path in the protocol — exclusively handles `sequencerFee[productId]` (off-chain accounting balances) and never touches `slowModeFees` or the actual ERC-20 balance sitting in the `Endpoint` contract:

```solidity
// EndpointTx.sol L244-253
IOffchainExchange(offchainExchange).dumpFees();
uint32[] memory spotIds = spotEngine.getProductIds();
int128[] memory fees = new int128[](spotIds.length);
for (uint256 i = 0; i < spotIds.length; i++) {
    fees[i] = sequencerFee[spotIds[i]];
    sequencerFee[spotIds[i]] = 0;
}
requireSubaccount(X_ACCOUNT);
clearinghouse.claimSequencerFees(fees);
``` [4](#0-3) 

No function in `Endpoint.sol`, `EndpointTx.sol`, or `EndpointStorage.sol` transfers the quote token balance out of the `Endpoint` contract. `handleDepositTransfer` immediately forwards deposited collateral to the `Clearinghouse`, so the only tokens that remain in `Endpoint` are the slow mode fees. [5](#0-4) 

---

### Impact Explanation

Every slow mode fee paid by any user is permanently locked in the `Endpoint` contract. The affected transaction types include `WithdrawCollateral`, `LinkSigner`, `ClaimBuilderFee`, and any other type that falls into the `else` branch of `submitSlowModeTransactionImpl`. The quote token (USDC or equivalent) accumulates in the contract indefinitely with no recovery path. The corrupted asset delta is: `Endpoint.balanceOf(quoteToken) += SLOW_MODE_FEE` per slow mode submission, with no corresponding outflow. [6](#0-5) 

---

### Likelihood Explanation

High. Any unprivileged user who submits a slow mode transaction (e.g., a withdrawal or signer link) triggers this path. It requires no special access, no exploit setup, and no coordination. The fee accumulates with every normal protocol interaction that uses the slow mode path.

---

### Recommendation

Add a privileged withdrawal function (e.g., `onlyOwner`) to the `Endpoint` contract that transfers the accumulated `slowModeFees` balance of the quote token to a designated recipient (e.g., the protocol treasury or `X_ACCOUNT` via the clearinghouse), mirroring how `DumpFees` handles `sequencerFee`. Alternatively, route `chargeSlowModeFee` to transfer directly to the `Clearinghouse` or a fee recipient address rather than `address(this)`.

---

### Proof of Concept

1. User calls `Endpoint.submitSlowModeTransaction(withdrawCollateralTx)`.
2. `submitSlowModeTransactionImpl` reaches the `else` branch (line 370).
3. `chargeSlowModeFee` calls `token.safeTransferFrom(user, address(this), fee)` — tokens land in `Endpoint`.
4. `slowModeFees += SLOW_MODE_FEE` increments the counter.
5. Sequencer later submits `DumpFees` — only `sequencerFee` balances are moved; `slowModeFees` and the actual ERC-20 balance in `Endpoint` are untouched.
6. Repeat for every slow mode submission. Tokens accumulate in `Endpoint` with no exit path. [1](#0-0) [2](#0-1)

    function submitSlowModeTransactionImpl(bytes calldata transaction) public {
        IEndpoint.TransactionType txType = IEndpoint.TransactionType(
            uint8(transaction[0])
        );

        // special case for DepositCollateral because upon
        // slow mode submission we must take custody of the
        // actual funds

        address sender = msg.sender;

        if (txType == IEndpoint.TransactionType.DepositCollateral) {
            revert();
        } else if (txType == IEndpoint.TransactionType.DepositInsurance) {
            IEndpoint.DepositInsurance memory txn = abi.decode(
                transaction[1:],
                (IEndpoint.DepositInsurance)
            );
            require(
                txn.amount >= uint128(SLOW_MODE_FEE),
                ERR_DEPOSIT_TOO_SMALL
            );
            handleDepositTransfer(_getQuote(), sender, uint256(txn.amount));
        } else if (
            txType == IEndpoint.TransactionType.WithdrawInsurance ||
            txType == IEndpoint.TransactionType.DelistProduct ||
            txType == IEndpoint.TransactionType.DumpFees ||
            txType == IEndpoint.TransactionType.RebalanceXWithdraw ||
            txType == IEndpoint.TransactionType.UpdateTierFeeRates ||
            txType == IEndpoint.TransactionType.AddNlpPool ||
            txType == IEndpoint.TransactionType.UpdateNlpPool ||
            txType == IEndpoint.TransactionType.DeleteNlpPool ||
            txType == IEndpoint.TransactionType.ForceRebalanceNlpPool ||
            txType == IEndpoint.TransactionType.NlpProfitShare ||
            txType == IEndpoint.TransactionType.UpdateBuilder
        ) {
            require(sender == owner());
        } else {
            chargeSlowModeFee(_getQuote(), sender);
            slowModeFees += SLOW_MODE_FEE;
        }

        IEndpoint.SlowModeConfig memory _slowModeConfig = slowModeConfig;
        requireUnsanctioned(sender);
        slowModeTxs[_slowModeConfig.txCount++] = IEndpoint.SlowModeTx({
            executableAt: uint64(block.timestamp) + SLOW_MODE_TX_DELAY, // hardcoded to three days
            sender: sender,
            tx: transaction
        });
        // TODO: to save on costs we could potentially just emit something
        // for now, we can just create a separate loop in the engine that queries the remote
        // sequencer for slow mode transactions, and ignore the possibility of a reorgy attack
        slowModeConfig = _slowModeConfig;
    }
```
