### Title
Division by Zero in `socializeSubaccount` When `openInterest == 0` Permanently Blocks Insolvent Subaccount Finalization — (`File: core/contracts/PerpEngine.sol`)

---

### Summary

`PerpEngine.socializeSubaccount` unconditionally divides by `state.openInterest` without a zero-guard. When a perp product has no remaining open interest (e.g., the liquidatee was the sole participant), the division reverts, permanently blocking the finalization of an insolvent subaccount and preventing bad-debt socialization.

---

### Finding Description

In `PerpEngine.socializeSubaccount`, after insurance is exhausted and `balance.vQuoteBalance` is still negative, the protocol attempts to spread the remaining loss across all open-interest holders:

```solidity
// PerpEngine.sol lines 164–171
if (balance.vQuoteBalance < 0) {
    // socialize across all other participants
    int128 fundingPerShare = -balance.vQuoteBalance.div(
        state.openInterest   // <-- no zero-check
    );
    state.cumulativeFundingLongX18 += fundingPerShare;
    state.cumulativeFundingShortX18 -= fundingPerShare;
    balance.vQuoteBalance = 0;
}
``` [1](#0-0) 

If `state.openInterest == 0` at this point, the `MathSD21x18.div` call reverts (Solidity 0.8.x reverts on integer division by zero). There is no guard of the form `require(state.openInterest > 0)` before this division.

`socializeSubaccount` is invoked from `ClearinghouseLiq._finalizeSubaccount`:

```solidity
// ClearinghouseLiq.sol line 386
v.insurance = perpEngine.socializeSubaccount(txn.liquidatee, v.insurance);
``` [2](#0-1) 

`_finalizeSubaccount` is reached when a liquidator submits `txn.productId == type(uint32).max` to `liquidateSubaccountImpl`:

```solidity
// ClearinghouseLiq.sol lines 620–627
if (_finalizeSubaccount(txn, spotEngine, perpEngine)) {
    if (RiskHelper.isIsolatedSubaccount(txn.liquidatee)) {
        IOffchainExchange(...).tryCloseIsolatedSubaccount(txn.liquidatee);
    }
    return;
}
``` [3](#0-2) 

`liquidateSubaccountImpl` is itself called via `delegatecall` from `Clearinghouse.liquidateSubaccount`, which is an `onlyEndpoint` function reachable by the sequencer processing a user-signed `LiquidateSubaccount` transaction. [4](#0-3) 

---

### Impact Explanation

When `state.openInterest == 0` for a given perp product and the liquidatee still carries a negative `vQuoteBalance` that insurance cannot fully cover, every call to finalize that subaccount reverts. The concrete corrupted state:

- The insolvent subaccount is permanently stuck — it cannot be finalized, its bad debt cannot be written off, and its positions cannot be cleared.
- The protocol's accounting for that perp product is frozen: `cumulativeFundingLongX18` / `cumulativeFundingShortX18` are never updated to absorb the loss, leaving the system with unresolved bad debt.
- Any future participants in that perp product inherit a corrupted funding baseline.

---

### Likelihood Explanation

The condition is realistic in low-activity or newly listed perp markets:

1. A single trader opens a perp position (they are the only participant; `openInterest` equals their position size).
2. Their position is closed during finalization (all perp `amount` values are zeroed out by prior liquidation steps), driving `openInterest` to zero.
3. They still carry a negative `vQuoteBalance` (unrealized loss) that exceeds the insurance fund.
4. Any liquidator submitting `productId == type(uint32).max` triggers the revert.

The liquidator is an unprivileged caller — no special role is required.

---

### Recommendation

Add a zero-guard before the division in `PerpEngine.socializeSubaccount`. If `openInterest == 0`, there are no counterparties to socialize against; the loss should either be absorbed entirely by the insurance fund or the function should handle this edge case explicitly (e.g., skip socialization and leave `vQuoteBalance` for the insurance path):

```solidity
if (balance.vQuoteBalance < 0) {
    if (state.openInterest > 0) {
        int128 fundingPerShare = -balance.vQuoteBalance.div(state.openInterest);
        state.cumulativeFundingLongX18 += fundingPerShare;
        state.cumulativeFundingShortX18 -= fundingPerShare;
    }
    balance.vQuoteBalance = 0;
}
``` [5](#0-4) 

---

### Proof of Concept

1. Deploy the protocol with a single perp product (e.g., `productId = 1`).
2. Trader A opens a long position; `state.openInterest > 0`.
3. Price crashes; Trader A's maintenance health goes negative.
4. Liquidator calls `liquidateSubaccountImpl` with `productId = 1` to close Trader A's perp position (`amount → 0`). After this step, `state.openInterest == 0`.
5. Trader A still has a large negative `vQuoteBalance` (unrealized loss) exceeding the insurance fund.
6. Liquidator calls `liquidateSubaccountImpl` with `productId = type(uint32).max` to finalize.
7. `_finalizeSubaccount` → `perpEngine.socializeSubaccount(txn.liquidatee, v.insurance)` → division by `state.openInterest == 0` → revert.
8. Every subsequent finalization attempt reverts. The insolvent subaccount is permanently stuck. [6](#0-5) [7](#0-6)

    function socializeSubaccount(bytes32 subaccount, int128 insurance)
        external
        returns (int128)
    {
        require(msg.sender == address(_clearinghouse), ERR_UNAUTHORIZED);

        uint32[] memory _productIds = getProductIds();
        for (uint128 i = 0; i < _productIds.length; ++i) {
            uint32 productId = _productIds[i];
            (State memory state, Balance memory balance) = getStateAndBalance(
                productId,
                subaccount
            );
            if (balance.vQuoteBalance < 0) {
                int128 insuranceCover = MathHelper.min(
                    insurance,
                    -balance.vQuoteBalance
                );
                insurance -= insuranceCover;
                balance.vQuoteBalance += insuranceCover;
                state.availableSettle += insuranceCover;

                // actually socialize if still not enough
                if (balance.vQuoteBalance < 0) {
                    // socialize across all other participants
                    int128 fundingPerShare = -balance.vQuoteBalance.div(
                        state.openInterest
                    );
                    state.cumulativeFundingLongX18 += fundingPerShare;
                    state.cumulativeFundingShortX18 -= fundingPerShare;
                    balance.vQuoteBalance = 0;
                }
                _setState(productId, state);
                _setBalanceAndUpdateBitmap(productId, subaccount, balance);
            }
        }
        return insurance;
    }

    function liquidateSubaccountImpl(IEndpoint.LiquidateSubaccount calldata txn)
        external
    {
        require(!RiskHelper.isIsolatedSubaccount(txn.sender), ERR_UNAUTHORIZED);
        require(txn.sender != txn.liquidatee, ERR_UNAUTHORIZED);
        require(isUnderMaintenance(txn.liquidatee), ERR_NOT_LIQUIDATABLE);
        require(
            txn.liquidatee != X_ACCOUNT && txn.liquidatee != N_ACCOUNT,
            ERR_NOT_LIQUIDATABLE
        );
        require(
            txn.productId != QUOTE_PRODUCT_ID,
            ERR_INVALID_LIQUIDATION_PARAMS
        );

        ISpotEngine spotEngine = ISpotEngine(
            address(engineByType[IProductEngine.EngineType.SPOT])
        );
        IPerpEngine perpEngine = IPerpEngine(
            address(engineByType[IProductEngine.EngineType.PERP])
        );

        if (_finalizeSubaccount(txn, spotEngine, perpEngine)) {
            if (RiskHelper.isIsolatedSubaccount(txn.liquidatee)) {
                IOffchainExchange(
                    IEndpoint(getEndpoint()).getOffchainExchange()
                ).tryCloseIsolatedSubaccount(txn.liquidatee);
            }
            return;
        }

        if (
            (txn.amount < 0) &&
            (txn.isEncodedSpread ||
                address(productToEngine[txn.productId]) == address(spotEngine))
        ) {
            // when it's spread or spot liquidation, we need to make sure the liquidatee has
            // enough quote to buyback the liquidated amount.
            _assertCanLiquidateLiability(txn, spotEngine, perpEngine);
            _settlePositivePerpPnl(txn, spotEngine, perpEngine);
        }

        _assertLiquidationAmount(txn, spotEngine, perpEngine);

        // beyond this point, we can be sure that we can liquidate the entire
        // liquidation amount knowing that the insurance fund will remain solvent
        // subsequently we can just blast the remainder of the liquidation and
        // cover the quote balance from the insurance fund at the end
        _handleLiquidationPayment(txn, spotEngine, perpEngine);
    }
