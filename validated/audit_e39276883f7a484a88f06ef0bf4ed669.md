### Title
`ValidResourceBounds::L1Gas` Downgrade in Protobuf Deserialization Causes Consensus/Mempool Rejection of Valid InvokeV3 Transactions - (`File: crates/apollo_protobuf/src/converters/transaction.rs`)

---

### Summary

The protobuf deserializer for `ResourceBounds` silently downgrades an `AllResources` variant with zero `l2_gas` and `l1_data_gas` to `ValidResourceBounds::L1Gas`. A subsequent mandatory conversion step from `InvokeTransactionV3` to `RpcInvokeTransactionV3` hard-rejects any `L1Gas` variant. The result is that a valid `InvokeV3` transaction accepted by the gateway (only `l1_gas` non-zero) is permanently rejected by every peer node that receives it over the consensus or mempool P2P channel, breaking consensus for any block that contains such a transaction.

---

### Finding Description

**Step 1 – Gateway accepts the transaction.**

`RpcInvokeTransactionV3` carries `resource_bounds: AllResourceBounds`, which is always the `AllResources` variant. The stateless validator explicitly allows transactions where only `l1_gas` is non-zero:

```
#[case::valid_l1_gas(
    RpcTransactionArgs {
        resource_bounds: AllResourceBounds { l1_gas: NON_EMPTY_RESOURCE_BOUNDS, ..Default::default() },
        ..
    }
)]
```

The gateway converts this to `InternalRpcInvokeTransactionV3` (also `AllResourceBounds`) and computes the hash via `InvokeTransactionV3Trait::resource_bounds()` returning `ValidResourceBounds::AllResources(...)`. The hash includes the `l1_data_gas` packed field (3-element resource array). [1](#0-0) 

**Step 2 – Serialisation to protobuf is lossless.**

`From<ValidResourceBounds> for protobuf::ResourceBounds` serialises `AllResources` with all three fields present (even when zero). [2](#0-1) 

**Step 3 – Deserialisation silently downgrades the variant.**

`TryFrom<protobuf::ResourceBounds> for ValidResourceBounds` inspects the wire values and, when `l2_gas` and `l1_data_gas` are both zero, returns `ValidResourceBounds::L1Gas(l1_gas)` instead of `ValidResourceBounds::AllResources`:

```rust
Ok(if l1_data_gas.is_zero() && l2_gas.is_zero() {
    ValidResourceBounds::L1Gas(l1_gas)          // ← wrong variant
} else {
    ValidResourceBounds::AllResources(...)
})
``` [3](#0-2) 

**Step 4 – The mandatory upward conversion hard-fails.**

The consensus and mempool P2P paths both route through `TryFrom<protobuf::InvokeV3WithProof> for RpcInvokeTransactionV3`, which first converts to `InvokeTransactionV3` (now carrying `L1Gas`) and then calls `snapi_invoke.try_into()`:

```rust
// This conversion can fail only if the resource_bounds are not AllResources.
Ok(Self { proof, ..snapi_invoke.try_into().map_err(|_| DEPRECATED_RESOURCE_BOUNDS_ERROR)? })
``` [4](#0-3) 

`TryFrom<InvokeTransactionV3> for RpcInvokeTransactionV3` returns `Err` for any non-`AllResources` variant:

```rust
resource_bounds: match value.resource_bounds {
    ValidResourceBounds::AllResources(bounds) => bounds,
    _ => return Err(StarknetApiError::OutOfRange { string: "resource_bounds".to_string() }),
},
``` [5](#0-4) 

The `?` propagates the error, causing the entire `ConsensusTransaction` or `MempoolTransaction` deserialization to fail.

**Step 5 – Consensus path.**

`TryFrom<protobuf::ConsensusTransaction> for ConsensusTransaction` uses `?` on the inner conversion:

```rust
protobuf::consensus_transaction::Txn::InvokeV3(txn) => {
    ConsensusTransaction::RpcTransaction(RpcTransaction::Invoke(
        RpcInvokeTransaction::V3(txn.try_into()?),   // ← propagates error
    ))
}
``` [6](#0-5) 

A validator node that receives a block proposal containing such a transaction cannot deserialize it, causing it to reject the proposal.

---

### Impact Explanation

**Consensus (Critical/High):** A proposer node that accepted a valid `InvokeV3` with only `l1_gas` non-zero will include it in a block proposal. Every validator node will fail to deserialize the `ConsensusTransaction` for that entry, reject the proposal, and the block cannot be finalized. This matches: *"High. Transaction conversion or signature/hash logic binds the wrong signer, hash, type, or executable payload."*

**Mempool P2P (High):** The same failure occurs in `TryFrom<protobuf::MempoolTransaction> for RpcTransaction`, so the transaction cannot propagate to any peer node. Matches: *"High. Mempool/gateway/RPC admission accepts invalid transactions or rejects valid transactions before sequencing."*

---

### Likelihood Explanation

The gateway's stateless validator explicitly accepts `AllResourceBounds` with only `l1_gas` non-zero (test case `valid_l1_gas`). Any user who submits such a transaction triggers the bug deterministically. No special privileges or malformed bytes are required.

---

### Recommendation

Remove the `ValidResourceBounds::L1Gas` downgrade from the protobuf deserializer. The `L1Gas` variant is a pre-0.13.3 semantic distinction; new transactions arriving over the P2P wire should always be treated as `AllResources`. Replace the conditional in `TryFrom<protobuf::ResourceBounds> for ValidResourceBounds` with an unconditional `AllResources` construction:

```rust
Ok(ValidResourceBounds::AllResources(AllResourceBounds { l1_gas, l2_gas, l1_data_gas }))
```

Alternatively, add a separate `TryFrom` path for the consensus/mempool protobuf messages that constructs `AllResourceBounds` directly (as the `rpc_transaction.rs` converter already does for `AllResourceBounds` in `TryFrom<protobuf::ResourceBounds> for AllResourceBounds`).

---

### Proof of Concept

1. Construct a valid `RpcInvokeTransactionV3` with `resource_bounds = AllResourceBounds { l1_gas: ResourceBounds { max_amount: GasAmount(1), max_price_per_unit: GasPrice(1) }, l2_gas: ResourceBounds::default(), l1_data_gas: ResourceBounds::default() }`.
2. Submit to the gateway — it is accepted (matches `valid_l1_gas` test case).
3. Serialize to `protobuf::ConsensusTransaction` via `From<ConsensusTransaction>` — succeeds, wire bytes contain `l2_gas = 0`, `l1_data_gas = 0`.
4. Deserialize via `TryFrom<protobuf::ConsensusTransaction> for ConsensusTransaction`:
   - `protobuf::InvokeV3` → `InvokeTransactionV3`: `ValidResourceBounds::try_from(...)` sees `l2_gas.is_zero() && l1_data_gas.is_zero()` → returns `ValidResourceBounds::L1Gas(l1_gas)`.
   - `InvokeTransactionV3` → `RpcInvokeTransactionV3`: `L1Gas` arm hits `return Err(...)`.
   - `?` propagates → `TryFrom<protobuf::ConsensusTransaction>` returns `Err`.
5. Any validator node receiving this proposal rejects it. [3](#0-2) [4](#0-3) [5](#0-4) [1](#0-0)

impl TryFrom<protobuf::InvokeV3WithProof> for RpcInvokeTransactionV3 {
    type Error = ProtobufConversionError;
    fn try_from(mut value: protobuf::InvokeV3WithProof) -> Result<Self, Self::Error> {
        // Extract proof first, since `starknet_api::transaction::InvokeTransactionV3` does not
        // carry a `proof` field.
        let proof = Proof::from(std::mem::take(&mut value.proof));

        let snapi_invoke: InvokeTransactionV3 = value
            .invoke
            .ok_or(ProtobufConversionError::MissingField {
                field_description: "InvokeV3WithProof::invoke",
            })?
            .try_into()?;

        // This conversion can fail only if the resource_bounds are not AllResources.
        Ok(Self { proof, ..snapi_invoke.try_into().map_err(|_| DEPRECATED_RESOURCE_BOUNDS_ERROR)? })
    }
