### Title
Swap Allowlist Keyed on `sender` (msg.sender) Lets Any User Bypass the Guard via `MetricOmmSimpleRouter` - (File: metric-periphery/contracts/extensions/SwapAllowlistExtension.sol)

### Summary
`SwapAllowlistExtension.beforeSwap` enforces its allowlist by looking up `(pool, sender)`, where `sender` is `msg.sender` as seen by the pool. When a user routes through `MetricOmmSimpleRouter`, the pool's `msg.sender` is the router, not the end user. If the router address is allowlisted (or `allowAll` is set for it), every user—including those explicitly excluded from the per-user allowlist—can execute swaps on the restricted pool by routing through the public router contract.

### Finding Description
`MetricOmmPool.swap` passes `msg.sender` as the `sender` argument to `_beforeSwap`, which forwards it verbatim to every registered extension:

```solidity
// MetricOmmPool.sol
_beforeSwap(
    msg.sender,   // ← becomes `sender` in the hook
    recipient,
    zeroForOne,
    amountSpecified,
    priceLimitX64,
    packedSlot0Initial,
    bidPriceX64,
    askPriceX64,
    extensionData
);
```

`ExtensionCalling._beforeSwap` encodes this as the first argument to `IMetricOmmExtensions.beforeSwap`:

```solidity
// ExtensionCalling.sol
abi.encodeCall(
    IMetricOmmExtensions.beforeSwap,
    (sender, recipient, zeroForOne, amountSpecified, priceLimitX64,
     packedSlot0Initial, bidPriceX64, askPriceX64, extensionData)
)
```

`SwapAllowlistExtension.beforeSwap` then performs its allowlist lookup keyed on `(pool, sender)`. When the user calls `MetricOmmSimpleRouter.exactInputSingle()` (or any multi-hop variant), the router calls `pool.swap()`, so `sender` = router address. The allowlist entry for the individual user (`(pool, user)`) is never consulted. If the pool admin has allowlisted the router address (a natural step to let legitimate users access the pool through the standard periphery), the guard is completely bypassed for every caller of the router, regardless of whether they appear in the per-user allowlist.

### Impact Explanation
Any user excluded from the swap allowlist can execute swaps on a restricted pool by routing through `MetricOmmSimpleRouter`. This defeats the entire purpose of the allowlist: compliance-gated pools, KYC-restricted pools, or pools that limit swap access to specific counterparties are all rendered open to the public. The attacker receives output tokens from the pool and the pool receives input tokens, so there is a direct, real token flow that the pool admin intended to block. This constitutes broken core pool functionality causing loss of the intended access-control invariant with direct fund-flow consequences.

### Likelihood Explanation
The `MetricOmmSimpleRouter` is the standard, publicly deployed periphery entry point. Pool admins who want their allowlisted users to be able to trade will naturally allowlist the router. The moment they do, the per-user allowlist is silently nullified for all router-mediated swaps. No special privilege or malicious setup is required—any public user can call the router.

### Recommendation
The allowlist check must be performed against the **end user**, not the intermediary contract. Two sound approaches:

1. **Pass the originating user through `extensionData`**: The router encodes the original `msg.sender` into `extensionData`; the extension reads and verifies it. This requires the router to be trusted to set this field honestly (acceptable since the router is a known, audited contract).
2. **Key the allowlist on `recipient` or a dedicated `swapper` field**: Change the hook signature or the extension's lookup to use the address that economically benefits from the swap rather than the address that called the pool.

Either way, the router address must never be the identity that the allowlist gates.

### Proof of Concept

```
Setup:
  - Deploy pool with SwapAllowlistExtension configured.
  - Pool admin allowlists the router address (so legitimate users can trade via router).
  - Pool admin does NOT allowlist Alice.

Attack:
  1. Alice calls MetricOmmSimpleRouter.exactInputSingle({pool, tokenIn, tokenOut, ...}).
  2. Router calls pool.swap(recipient=Alice, ...).
  3. Pool calls _beforeSwap(sender=router, ...).
  4. SwapAllowlistExtension looks up allowedSwapper[pool][router] → true (router is allowlisted).
  5. Hook passes. Swap executes. Alice receives output tokens.

Result:
  Alice, who is explicitly excluded from the per-user allowlist, successfully swaps on the
  restricted pool. The allowlist invariant is broken for all router-mediated swaps.
```
