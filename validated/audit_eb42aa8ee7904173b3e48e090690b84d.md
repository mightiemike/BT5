### Title
Bonus Accrual Checkpoint Advanced Before Payment in `slashRedelegationPosition` Permanently Forfeits User Rewards — (File: `x/tieredrewards/keeper/slash.go`)

---

### Summary

In `x/tieredrewards/keeper/slash.go`, the `slashRedelegationPosition` function calls `processEventsAndClaimBonus`, which unconditionally advances `pos.LastBonusAccrual` to the current block time **before** checking whether the bonus pool has sufficient funds. When the pool is insufficient, `ErrInsufficientBonusPool` is returned, but the error is deliberately swallowed to prevent a chain halt. The modified position — with its checkpoint already advanced — is then persisted. The user's entire accrued bonus for the period `[original_LastBonusAccrual, blockTime]` is permanently forfeited with no recourse.

---

### Finding Description

**Root cause — checkpoint advanced before payment:**

In `processEventsAndClaimBonus`, `applyBonusAccrualCheckpoint` is called at line 215, unconditionally setting `pos.LastBonusAccrual = blockTime` in memory, **before** the pool balance check and the `SendCoinsFromModule` transfer that follows: [1](#0-0) 

The pool check and transfer happen only after line 228, meaning any error from an insufficient pool is returned **after** the checkpoint has already been written to the in-memory `pos`.

**Error swallowed in `slashRedelegationPosition`:** [2](#0-1) 

When `ErrInsufficientBonusPool` is caught, execution continues. The `pos` struct in memory now carries `LastBonusAccrual = blockTime` (advanced by the failed call), but zero bonus was paid.

**Modified position persisted with advanced checkpoint:**

For a partial slash (the common case), the position is saved immediately after: [3](#0-2) 

For a full slash, `ClearBonusCheckpoints()` is called, which resets the checkpoint entirely — so the full-slash path does not exhibit this issue. The partial-slash path is the vulnerable one.

**Consequence — desynchronized checkpoint vs. payment state:**

After the failed-payment path, `pos.LastBonusAccrual` is `blockTime` (the time of the failed claim). On the user's next `MsgClaimTierRewards`, `processEventsAndClaimBonus` initialises `segmentStart = pos.LastBonusAccrual = blockTime`. All bonus that accrued in the window `[original_LastBonusAccrual, blockTime]` is gone: the segment is never replayed because `LastEventSeq` was also advanced past those events during the failed call. [4](#0-3) 

This is the direct analog to the external report's bug class: **two different sources for the same logical time reference** — the actual last-paid-up time (which did not advance because no payment occurred) and the persisted checkpoint time (which was advanced to `blockTime`) — are desynchronized, breaking the invariant that `LastBonusAccrual` always reflects the last time bonus was successfully paid.

---

### Impact Explanation

A user with a tier position that has an active redelegation permanently loses all accrued bonus rewards for the period between their last successful claim and the block in which the redelegation slash fires, whenever the bonus pool is depleted at that moment. The lost amount equals `shares × tokensPerShare × bonusApy × duration / SecondsPerYear` for the affected window — a direct, irreversible loss of earned token rewards from the user's locked position. [5](#0-4) 

---

### Likelihood Explanation

The trigger requires two concurrent conditions:

1. **A redelegation slash fires** — this is a normal protocol event whenever a validator is slashed while a position's redelegation is in the unbonding window. Any validator slash on a validator that was the source of a tier-position redelegation suffices.
2. **The bonus pool is depleted** — the pool is a shared module account funded by governance. It can be drained to zero by other users legitimately claiming their own rewards, or simply by not being replenished. There is no minimum-balance enforcement.

Neither condition requires privileged access. A user with a tier position and an active redelegation is exposed to this loss whenever both conditions coincide, which is a realistic operational scenario. [6](#0-5) 

---

### Recommendation

Move `applyBonusAccrualCheckpoint` (and `UpdateLastKnownBonded`) to **after** the successful `SendCoinsFromModule` transfer, so the checkpoint is only advanced when payment actually succeeds. Alternatively, save the original `LastBonusAccrual` and `LastEventSeq` values before the call and restore them on `ErrInsufficientBonusPool` in `slashRedelegationPosition`, so the user can retry the claim once the pool is replenished.

---

### Proof of Concept

1. User creates a tier position delegating to validator V1 and calls `MsgTierRedelegate` to redelegate to V2. A `RedelegationMapping` entry is created.
2. The bonus pool is drained to zero (e.g., by other users claiming their own rewards).
3. V1 is slashed. The Cosmos SDK staking module fires `BeforeRedelegationSlashed` for the in-progress redelegation.
4. `slashRedelegationPosition` is called. It calls `processEventsAndClaimBonus(ctx, &pos)`.
5. Inside `processEventsAndClaimBonus`, `applyBonusAccrualCheckpoint(&pos.Position, blockTime)` advances `pos.LastBonusAccrual` to the current block time and `pos.LastEventSeq` past all pending events.
6. The pool check fails; `ErrInsufficientBonusPool` is returned.
7. `slashRedelegationPosition` catches the error, logs it, and continues.
8. `setPositionWithState(ctx, pos, nil)` persists the position with `LastBonusAccrual = blockTime` and the advanced `LastEventSeq`.
9. The user later calls `MsgClaimTierRewards`. `processEventsAndClaimBonus` starts from `segmentStart = blockTime`; all bonus accrued before that block is permanently gone. [7](#0-6) [8](#0-7)

func (k Keeper) processEventsAndClaimBonus(ctx context.Context, pos *types.PositionState) (sdk.Coins, error) {
	// Rewards should have been claimed before undelegation
	if !pos.IsDelegated() {
		return sdk.NewCoins(), nil
	}

	valAddr, err := sdk.ValAddressFromBech32(pos.Delegation.ValidatorAddress)
	if err != nil {
		return nil, err
	}

	events, err := k.getValidatorEventsSince(ctx, valAddr, pos.LastEventSeq)
	if err != nil {
		return nil, err
	}

	sdkCtx := sdk.UnwrapSDKContext(ctx)
	blockTime := sdkCtx.BlockTime()

	totalBonus := math.ZeroInt()
	// Use the persisted bonded state from the last replay, not a hardcoded default.
	// This prevents overpaying bonus for unbonded gaps between claims.
	bonded := pos.LastKnownBonded
	segmentStart := pos.LastBonusAccrual

	tier, err := k.getTier(ctx, pos.TierId)
	if err != nil {
		return nil, err
	}

	for _, entry := range events {
		evt := entry.Event

		if bonded {
			// Compute bonus for the bonded segment [segmentStart, eventTime]
			// using the snapshot rate at the event.
			bonus := k.computeSegmentBonus(*pos, tier, segmentStart, evt.Timestamp, evt.TokensPerShare)
			totalBonus = totalBonus.Add(bonus)
		}

		// Update bonded state based on event type.
		switch evt.EventType {
		case types.ValidatorEventType_VALIDATOR_EVENT_TYPE_UNBOND:
			bonded = false
		case types.ValidatorEventType_VALIDATOR_EVENT_TYPE_BOND:
			bonded = true
		case types.ValidatorEventType_VALIDATOR_EVENT_TYPE_SLASH:
			// Slash doesn't change bonded state.
		}

		segmentStart = evt.Timestamp
		pos.UpdateLastEventSeq(entry.Seq)

		// Decrement reference count.
		if err := k.decrementEventRefCount(ctx, valAddr, entry.Seq); err != nil {
			return nil, err
		}
	}

	val, err := k.stakingKeeper.GetValidator(ctx, valAddr)
	if err != nil {
		return nil, err
	}
	// Defensive: validator bond status check
	if bonded && val.IsBonded() {
		currentRate, err := k.getTokensPerShare(ctx, valAddr)
		if err != nil {
			return nil, err
		}
		bonus := k.computeSegmentBonus(*pos, tier, segmentStart, blockTime, currentRate)
		totalBonus = totalBonus.Add(bonus)
	}

	applyBonusAccrualCheckpoint(&pos.Position, blockTime)
	// Persist the bonded state so the next replay starts correctly.
	pos.UpdateLastKnownBonded(bonded)

	if totalBonus.IsZero() {
		return sdk.NewCoins(), nil
	}

func (k Keeper) slashRedelegationPosition(ctx context.Context, unbondingId uint64, sharesToUnbond math.LegacyDec) error {
	positionId, err := k.getRedelegationMapping(ctx, unbondingId)
	if errors.Is(err, collections.ErrNotFound) {
		return nil
	}
	if err != nil {
		return err
	}

	pos, err := k.getPositionState(ctx, positionId)
	if errors.Is(err, types.ErrPositionNotFound) {
		k.logger(ctx).Error("position not found during redelegation slash",
			"position_id", positionId,
			"unbonding_id", unbondingId,
			"error", err.Error(),
		)
		return nil
	}
	if err != nil {
		return err
	}

	if !pos.IsDelegated() {
		// Defensive
		k.logger(ctx).Error("delegation missing during BeforeRedelegationSlashed",
			"position_id", positionId,
			"unbonding_id", unbondingId,
			"shares_to_unbond", sharesToUnbond.String(),
		)
		return nil
	}

	dstValStr := pos.Delegation.ValidatorAddress

	// Settle bonus against PRE-slash shares.
	if _, err := k.processEventsAndClaimBonus(ctx, &pos); err != nil {
		// Deliberately forgo bonus rewards if pool is insufficient to prevent chain halt.
		if errors.Is(err, types.ErrInsufficientBonusPool) {
			k.logger(ctx).Error("insufficient bonus pool during redelegation slash",
				"position_id", pos.Id,
				"error", err.Error(),
			)
		} else {
			return err
		}
	}

	fullSlash := sharesToUnbond.GTE(pos.Delegation.Shares)

	if fullSlash {
		pos.Delegation = nil
		pos.ClearBonusCheckpoints()
		return k.setPositionWithState(ctx, pos, &ValidatorTransition{PreviousAddress: dstValStr})
	}
	// In-memory only: the persisted Position carries no share count, and the
	// live delegation will reflect the post-Unbond shares on the next read.
	// Update the local copy so any follow-up logic in this call sees consistent state.
	pos.Delegation.Shares = pos.Delegation.Shares.Sub(sharesToUnbond)
	return k.setPositionWithState(ctx, pos, nil)
