### Title
Duplicate Position IDs in `MsgClaimTierRewards` Enable Double Bonus Reward Claim - (File: x/tieredrewards/keeper/msg_server.go)

### Summary

`ClaimTierRewards` loads all requested positions from state into a local slice **before** any state updates occur, with no deduplication of `msg.PositionIds`. Submitting the same position ID twice causes the stale pre-update copy to be processed a second time, paying out the same bonus reward segment twice from the `RewardsPoolName` module account.

### Finding Description

In `ClaimTierRewards` (`x/tieredrewards/keeper/msg_server.go`, lines 434–448), all positions are fetched from state and appended to a local slice before `claimRewardsAndUpdatesPositions` is called:

```go
positions := make([]types.PositionState, 0, len(msg.PositionIds))
for _, posId := range msg.PositionIds {
    pos, err := ms.getPositionState(ctx, posId)
    // ...
    positions = append(positions, pos)
}
totalBase, totalBonus, err := ms.claimRewardsAndUpdatesPositions(ctx, positions)
```

There is no deduplication of `msg.PositionIds`. If the same ID (e.g., `1`) appears twice, both entries in `positions` are identical stale copies of the on-chain state at the time of loading.

`claimRewardsAndUpdatesPositions` (`x/tieredrewards/keeper/claim_rewards.go`, lines 106–135) then iterates over the slice and, for each entry, calls `processEventsAndClaimBonus` and then `setPosition`:

```go
for i := range positions {
    pos := &positions[i]
    bonus, err := k.processEventsAndClaimBonus(ctx, pos)
    // ...
    if err := k.setPosition(ctx, pos.Position, nil); err != nil { ... }
}
```

Inside `processEventsAndClaimBonus` (`x/tieredrewards/keeper/claim_rewards.go`, lines 165, 206–215), the current-segment bonus is computed using `pos.LastBonusAccrual` as the segment start, and the checkpoint is advanced to `blockTime` only in the local copy:

```go
segmentStart := pos.LastBonusAccrual
// ...
if bonded && val.IsBonded() {
    bonus := k.computeSegmentBonus(*pos, tier, segmentStart, blockTime, currentRate)
    totalBonus = totalBonus.Add(bonus)
}
applyBonusAccrualCheckpoint(&pos.Position, blockTime)
```

When position `1` appears at slice index `0` and index `1`:

- **First iteration (index 0):** computes bonus for `[old_LastBonusAccrual, blockTime]`, advances the local checkpoint to `blockTime`, saves the updated position to state via `setPosition`.
- **Second iteration (index 1):** this entry is the **stale pre-update copy** with `LastBonusAccrual = old_LastBonusAccrual`. It recomputes bonus for the same segment `[old_LastBonusAccrual, blockTime]` and pays it again via `bankKeeper.SendCoinsFromModuleToAccount(ctx, types.RewardsPoolName, ownerAddr, bonusCoins)`.

The on-chain state written by the first `setPosition` call is never re-read for the second slice entry, because the slice was populated before any writes occurred. This is the same check-effects-interactions violation as the ERC777 report: a stale balance/state snapshot is used for a second withdrawal.

### Impact Explanation

An attacker can drain the `RewardsPoolName` module account by submitting `MsgClaimTierRewards` with a position ID repeated `N` times, receiving `N×` the legitimately accrued bonus rewards. The `RewardsPoolName` pool is a shared module account funding bonus rewards for all participants; draining it harms every other position holder.

### Likelihood Explanation

The attack requires only a standard signed `MsgClaimTierRewards` transaction with a repeated position ID — no privileged role, no special token, no governance action. Any position owner can trigger it at any time after bonus rewards have accrued. The message format does not enforce uniqueness of `PositionIds`, and `msg.Validate()` and `validateClaimRewards` check ownership but not deduplication.

### Recommendation

Deduplicate `msg.PositionIds` before loading positions, or re-fetch each position from state inside `claimRewardsAndUpdatesPositions` rather than operating on a pre-loaded slice. The simplest fix is to add a uniqueness check in `ClaimTierRewards` before the loop:

```go
seen := make(map[uint64]struct{}, len(msg.PositionIds))
for _, posId := range msg.PositionIds {
    if _, dup := seen[posId]; dup {
        return nil, errorsmod.Wrapf(sdkerrors.ErrInvalidRequest, "duplicate position id: %d", posId)
    }
    seen[posId] = struct{}{}
    // ... existing load logic
}
```

Alternatively, follow the checks-effects-interactions pattern by re-reading each position from state immediately before processing it, rather than pre-loading all positions into a slice.

### Proof of Concept

1. Attacker owns position ID `1`, delegated to a bonded validator. `LastBonusAccrual = T0`, current block time = `T1 > T0`.
2. Attacker submits `MsgClaimTierRewards{Owner: attacker, PositionIds: [1, 1]}`.
3. `ClaimTierRewards` loads position `1` twice into `positions[0]` and `positions[1]`, both with `LastBonusAccrual = T0`.
4. First iteration (`positions[0]`): `processEventsAndClaimBonus` computes bonus for `[T0, T1]`, sends `bonusCoins` from `RewardsPoolName` to attacker, advances local checkpoint to `T1`, saves updated position (with `LastBonusAccrual = T1`) to state.
5. Second iteration (`positions[1]`): stale copy still has `LastBonusAccrual = T0`. `processEventsAndClaimBonus` recomputes bonus for `[T0, T1]` and sends the same `bonusCoins` from `RewardsPoolName` to attacker again.
6. Attacker receives `2×bonusCoins` while only being entitled to `1×bonusCoins`. Repeating with `N` copies of the same ID yields `N×` payout. [1](#0-0) [2](#0-1) [3](#0-2) [4](#0-3)

func (k Keeper) claimRewardsAndUpdatesPositions(ctx context.Context, positions []types.PositionState) (sdk.Coins, sdk.Coins, error) {
	totalBase := sdk.NewCoins()
	totalBonus := sdk.NewCoins()

	for i := range positions {
		pos := &positions[i]

		if !pos.IsDelegated() {
			continue
		}

		base, err := k.claimBaseRewards(ctx, *pos)
		if err != nil {
			return nil, nil, err
		}
		totalBase = totalBase.Add(base...)

		bonus, err := k.processEventsAndClaimBonus(ctx, pos)
		if err != nil {
			return nil, nil, err
		}
		totalBonus = totalBonus.Add(bonus...)

		if err := k.setPosition(ctx, pos.Position, nil); err != nil {
			return nil, nil, err
		}
	}

	return totalBase, totalBonus, nil
}
