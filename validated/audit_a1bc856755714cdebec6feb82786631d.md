### Title
Division by Zero in `PerpEngine.socializeSubaccount()` When `openInterest` Is Zero Blocks Liquidation Finalization — (`core/contracts/PerpEngine.sol`)

---

### Summary

`PerpEngine.socializeSubaccount()` unconditionally divides by `state.openInterest` when socializing residual negative `vQuoteBalance`. If `openInterest` is zero at the time of the call, `MathSD21x18.div` reverts with `"DBZ"`, causing the entire `liquidateSubaccountImpl` call to revert. This permanently blocks finalization of any insolvent subaccount in a perp market where all other participants have closed their positions.

---

### Finding Description

In `PerpEngine.socializeSubaccount()`, after exhausting insurance coverage, the protocol attempts to spread the remaining loss across all open-interest holders:

```solidity
// core/contracts/PerpEngine.sol  lines 164–171
if (balance.vQuoteBalance < 0) {
    // socialize across all other participants
    int128 fundingPerShare = -balance.vQuoteBalance.div(
        state.openInterest   // ← reverts if openInterest == 0
    );
    state.cumulativeFundingLongX18 += fundingPerShare;
    state.cumulativeFundingShortX18 -= fundingPerShare;
    balance.vQuoteBalance = 0;
}
```

`MathSD21x18.div` enforces a non-zero divisor:

```solidity
// core/contracts/libraries/MathSD21x18.sol  lines 62–69
function div(int128 x, int128 y) internal pure returns (int128) {
    unchecked {
        require(y != 0, ERR_DIV_BY_ZERO);   // reverts here
        ...
    }
}
```

`socializeSubaccount` is invoked unconditionally inside `_finalizeSubaccount` in `ClearinghouseLiq`:

```solidity
// core/contracts/ClearinghouseLiq.sol  lines 386–389
v.insurance = perpEngine.socializeSubaccount(
    txn.liquidatee,
    v.insurance
);
```

`_finalizeSubaccount` is called at the top of `liquidateSubaccountImpl`:

```solidity
// core/contracts/ClearinghouseLiq.sol  lines 620–627
if (_finalizeSubaccount(txn, spotEngine, perpEngine)) {
    ...
    return;
}
```

**Trigger condition**: A subaccount has `balance.vQuoteBalance < 0` for some perp product AND `state.openInterest == 0` for that product AND insurance is insufficient to cover the deficit. This is reachable because:

- `openInterest` tracks the aggregate absolute position size across all participants.
- A user's `vQuoteBalance` can remain negative even after their own position amount reaches zero (unrealized losses are not automatically settled).
- If every other participant closes their position, `openInterest` drops to zero while the insolvent subaccount still carries a negative `vQuoteBalance`.

No zero-guard exists before the division.

---

### Impact Explanation

When the condition is met, every call to `liquidateSubaccountImpl` with `productId == type(uint32).max` (the finalization path) reverts. The insolvent subaccount can never be finalized:

- The protocol cannot zero out the bad debt.
- The insurance fund cannot be correctly debited or replenished.
- The subaccount remains permanently un-finalizable, corrupting the solvency accounting of the protocol.

This is a solvency/accounting corruption issue, not merely a transient DoS: the bad debt is frozen in state indefinitely.

---

### Likelihood Explanation

The condition is reachable in any perp market that experiences a full wind-down of open interest — for example, a newly listed market with a single early participant who accumulates losses and then closes their position size (amount → 0) while retaining negative `vQuoteBalance`, or a market where all other participants exit. The insolvent subaccount owner does not need to act maliciously; normal trading activity can produce this state.

---

### Recommendation

Add a zero-guard for `openInterest` before the division, mirroring the fix recommended in the reference report:

```solidity
if (balance.vQuoteBalance < 0) {
    if (state.openInterest == 0) {
        // No participants to socialize against; absorb via insurance or leave for governance
        balance.vQuoteBalance = 0;
    } else {
        int128 fundingPerShare = -balance.vQuoteBalance.div(
            state.openInterest
        );
        state.cumulativeFundingLongX18 += fundingPerShare;
        state.cumulativeFundingShortX18 -= fundingPerShare;
        balance.vQuoteBalance = 0;
    }
}
```

---

### Proof of Concept

1. Deploy the protocol; create a single perp market.
2. User A opens a long position; price moves against them, accumulating negative `vQuoteBalance`.
3. User A closes their position (amount → 0); `openInterest` drops to zero. `vQuoteBalance` remains negative.
4. User A's subaccount health falls below maintenance margin.
5. Sequencer submits `LiquidateSubaccount` with `productId == type(uint32).max` to finalize the subaccount.
6. `_finalizeSubaccount` → `perpEngine.socializeSubaccount` → `balance.vQuoteBalance.div(state.openInterest)` → `MathSD21x18.div` reverts with `"DBZ"`.
7. The liquidation transaction reverts; the insolvent subaccount cannot be finalized.
