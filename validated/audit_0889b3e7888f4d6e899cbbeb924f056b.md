### Title
Silent Bonus Forfeit with Checkpoint Advancement in `BeforeRedelegationSlashed` Hook When Rewards Pool Is Insufficient — (File: `x/tieredrewards/keeper/hooks.go`)

---

### Summary

When a redelegating tier position is slashed and the `RewardsPoolName` module account has insufficient balance to cover the accrued bonus, the `BeforeRedelegationSlashed` hook silently forfeits the bonus while still advancing the position's reward checkpoints (`LastBonusAccrual`, `LastEventSeq`). The position is persisted with the advanced checkpoints as if the bonus was fully paid, permanently destroying the user's accrued bonus entitlement. This is the direct analog of the external report's "partial transfer but full accounting" class: the accounting marks the reward as settled, but the actual transfer never occurs.

---

### Finding Description

`processEventsAndClaimBonus` advances the position's in-memory checkpoints unconditionally during event replay, then checks pool sufficiency only afterward: [1](#0-0) 

```go
segmentStart = evt.Timestamp
pos.UpdateLastEventSeq(entry.Seq)
// Decrement reference count.
if err := k.decrementEventRefCount(ctx, valAddr, entry.Seq); err != nil {
    return nil, err
}
``` [2](#0-1) 

```go
applyBonusAccrualCheckpoint(&pos.Position, blockTime)
pos.UpdateLastKnownBonded(bonded)
...
if err := k.sufficientBonusPoolBalance(ctx, bonusCoins); err != nil {
    return nil, err
}
```

In all **user-driven paths** (`MsgClaimTierRewards`, `MsgTierRedelegate`, etc.), returning an error causes the entire transaction to roll back, so the in-memory checkpoint advances are discarded and the user can retry later. This is the correct behavior, confirmed by the test: [3](#0-2) 

However, the **`BeforeRedelegationSlashed` hook** path is fundamentally different. It fires inside the consensus-level `SlashRedelegation` call, where returning an error would halt the chain. The ADR explicitly documents the consequence: [4](#0-3) 

> "Bonus forfeits silently if the pool is insufficient (chain-halt avoidance)."

The hook calls `slashRedelegationPosition`: [5](#0-4) 

```go
func (h Hooks) BeforeRedelegationSlashed(ctx context.Context, unbondingID uint64, sharesToUnbond sdkmath.LegacyDec) error {
    return h.k.slashRedelegationPosition(ctx, unbondingID, sharesToUnbond)
}
```

Because the hook must not return an error (chain-halt avoidance), `slashRedelegationPosition` catches the `ErrInsufficientBonusPool` error, persists the position with the already-advanced checkpoints (`LastBonusAccrual`, `LastEventSeq`), and returns `nil`. The result is:

- `LastBonusAccrual` is advanced to the slash block time — the accrual window is permanently closed.
- `LastEventSeq` is advanced past the processed events — those events' reference counts were already decremented and may be garbage-collected.
- No `SendCoinsFromModuleToAccount` is executed — the user receives zero bonus tokens.

The position's accounting now reflects a settled bonus that was never paid. The user has no mechanism to re-claim it.

This is the exact structural analog of the external report: `safeRewardTransfer` sends `min(balance, amount)` while the caller sets `user.accClaim = 0`; here, `processEventsAndClaimBonus` sends zero while the caller persists checkpoints as if the full bonus was paid.

---

### Impact Explanation

A tier position owner who has accrued bonus rewards and whose position is in an active redelegation permanently loses all accrued bonus rewards if the `RewardsPoolName` module account balance is insufficient at the moment the destination validator is slashed. The corrupted invariant is:

> **For every position, `LastBonusAccrual` must only advance when the corresponding bonus has been successfully transferred to the owner.**

After this event, the invariant is broken: `LastBonusAccrual` has advanced but no transfer occurred. The user's bonus entitlement for the elapsed period is permanently destroyed. [6](#0-5) 

---

### Likelihood Explanation

The trigger requires two concurrent conditions:

1. **A tier position is in an active redelegation** — `MsgTierRedelegate` is a normal, unprivileged user action.
2. **The destination validator is slashed** — validator slashes (double-sign evidence, downtime) are normal consensus events, not attacker-controlled.
3. **The rewards pool is insufficient** — the pool is funded by governance/external deposits and can be depleted over time, especially if many positions are claiming bonus rewards concurrently.

None of these conditions require privileged access. The combination is realistic in a live network, particularly during periods of high bonus claim activity that drain the pool, followed by a validator slash event.

---

### Recommendation

The checkpoint advancement in `processEventsAndClaimBonus` must not be committed unless the bonus transfer succeeds. Two safe approaches:

1. **Defer checkpoint advancement**: Move `applyBonusAccrualCheckpoint` and `UpdateLastKnownBonded` to execute only after `SendCoinsFromModuleToAccount` succeeds. In the hook path, if the pool is insufficient, do not advance checkpoints and do not persist the position — the user retains the ability to claim later.

2. **Separate checkpoint from payment in the hook path**: In `slashRedelegationPosition`, if `ErrInsufficientBonusPool` is returned, skip persisting the updated position (leave checkpoints unchanged) and emit a warning event. The user can re-claim when the pool is replenished.

---

### Proof of Concept

1. User calls `MsgTierRedelegate` on position P from validator A to validator B. A `RedelegationMappings` entry is created mapping the unbonding ID to position P.
2. The rewards pool is drained (e.g., by many concurrent bonus claims or simply not replenished).
3. Validator B commits a double-sign infraction. The consensus layer calls `SlashRedelegation`, which fires `BeforeRedelegationSlashed(ctx, unbondingID, sharesToUnbond)`.
4. `slashRedelegationPosition` resolves position P via `RedelegationMappings`, calls `processEventsAndClaimBonus` against pre-slash shares.
5. `processEventsAndClaimBonus` walks events, advances `LastEventSeq` and decrements event reference counts, then calls `applyBonusAccrualCheckpoint` advancing `LastBonusAccrual` to the slash block time.
6. `sufficientBonusPoolBalance` returns `ErrInsufficientBonusPool`. The function returns the error.
7. `slashRedelegationPosition` catches the error, persists position P with the advanced checkpoints, and returns `nil` to avoid chain halt.
8. Position P's `LastBonusAccrual` now equals the slash block time. All bonus accrued since the last claim is permanently lost. The user's subsequent `MsgClaimTierRewards` will compute zero bonus for the already-elapsed window. [7](#0-6) [8](#0-7) [9](#0-8)

func (k Keeper) processEventsAndClaimBonus(ctx context.Context, pos *types.PositionState) (sdk.Coins, error) {
	// Rewards should have been claimed before undelegation
	if !pos.IsDelegated() {
		return sdk.NewCoins(), nil
	}

	valAddr, err := sdk.ValAddressFromBech32(pos.Delegation.ValidatorAddress)
	if err != nil {
		return nil, err
	}

	events, err := k.getValidatorEventsSince(ctx, valAddr, pos.LastEventSeq)
	if err != nil {
		return nil, err
	}

	sdkCtx := sdk.UnwrapSDKContext(ctx)
	blockTime := sdkCtx.BlockTime()

	totalBonus := math.ZeroInt()
	// Use the persisted bonded state from the last replay, not a hardcoded default.
	// This prevents overpaying bonus for unbonded gaps between claims.
	bonded := pos.LastKnownBonded
	segmentStart := pos.LastBonusAccrual

	tier, err := k.getTier(ctx, pos.TierId)
	if err != nil {
		return nil, err
	}

	for _, entry := range events {
		evt := entry.Event

		if bonded {
			// Compute bonus for the bonded segment [segmentStart, eventTime]
			// using the snapshot rate at the event.
			bonus := k.computeSegmentBonus(*pos, tier, segmentStart, evt.Timestamp, evt.TokensPerShare)
			totalBonus = totalBonus.Add(bonus)
		}

		// Update bonded state based on event type.
		switch evt.EventType {
		case types.ValidatorEventType_VALIDATOR_EVENT_TYPE_UNBOND:
			bonded = false
		case types.ValidatorEventType_VALIDATOR_EVENT_TYPE_BOND:
			bonded = true
		case types.ValidatorEventType_VALIDATOR_EVENT_TYPE_SLASH:
			// Slash doesn't change bonded state.
		}

		segmentStart = evt.Timestamp
		pos.UpdateLastEventSeq(entry.Seq)

		// Decrement reference count.
		if err := k.decrementEventRefCount(ctx, valAddr, entry.Seq); err != nil {
			return nil, err
		}
	}

	val, err := k.stakingKeeper.GetValidator(ctx, valAddr)
	if err != nil {
		return nil, err
	}
	// Defensive: validator bond status check
	if bonded && val.IsBonded() {
		currentRate, err := k.getTokensPerShare(ctx, valAddr)
		if err != nil {
			return nil, err
		}
		bonus := k.computeSegmentBonus(*pos, tier, segmentStart, blockTime, currentRate)
		totalBonus = totalBonus.Add(bonus)
	}

	applyBonusAccrualCheckpoint(&pos.Position, blockTime)
	// Persist the bonded state so the next replay starts correctly.
	pos.UpdateLastKnownBonded(bonded)

	if totalBonus.IsZero() {
		return sdk.NewCoins(), nil
	}

	bondDenom, err := k.stakingKeeper.BondDenom(ctx)
	if err != nil {
		return nil, err
	}

	bonusCoins := sdk.NewCoins(sdk.NewCoin(bondDenom, totalBonus))

	if err := k.sufficientBonusPoolBalance(ctx, bonusCoins); err != nil {
		return nil, err
	}
```

// TestMsgClaimTierRewards_FailsWhenBonusPoolInsufficient verifies that ClaimTierRewards
// returns ErrInsufficientBonusPool when accrued bonus cannot be paid, so the tx rolls
// back and the user can retry later without losing base rewards to a partial claim.
func (s *KeeperSuite) TestMsgClaimTierRewards_FailsWhenBonusPoolInsufficient() {
	lockAmount := sdkmath.NewInt(sdk.DefaultPowerReduction.Int64())
	pos := s.setupNewTierPosition(lockAmount, false)
	delAddr := sdk.MustAccAddressFromBech32(pos.Owner)
	valAddr := sdk.MustValAddressFromBech32(pos.Delegation.ValidatorAddress)
	_, bondDenom := s.getStakingData()
	msgServer := keeper.NewMsgServerImpl(s.keeper)

	s.setValidatorCommission(valAddr, sdkmath.LegacyZeroDec())

	// Advance time and allocate base rewards, but intentionally leave bonus pool empty.
	s.ctx = s.ctx.WithBlockHeight(s.ctx.BlockHeight() + 1)
	s.ctx = s.ctx.WithBlockTime(s.ctx.BlockTime().Add(time.Hour * 24 * 365))
	s.allocateRewardsToValidator(valAddr, sdkmath.NewInt(100), bondDenom)

	// Bonus pool remains at 0 — bonus accrued but pool cannot cover it.
	balBefore := s.app.BankKeeper.GetBalance(s.ctx, delAddr, bondDenom)

	// Use a branched context so a failed message does not persist state (matches DeliverTx rollback).
	cacheCtx, _ := s.ctx.CacheContext()
	resp, err := msgServer.ClaimTierRewards(cacheCtx, &types.MsgClaimTierRewards{
		Owner:       delAddr.String(),
		PositionIds: []uint64{pos.Id},
	})
	s.Require().Error(err)
	s.Require().True(errors.Is(err, types.ErrInsufficientBonusPool))
	s.Require().Nil(resp)

	balAfter := s.app.BankKeeper.GetBalance(s.ctx, delAddr, bondDenom)
	s.Require().True(balAfter.Amount.Equal(balBefore.Amount), "failed claim must not transfer rewards")

func (k Keeper) sufficientBonusPoolBalance(ctx context.Context, bonus sdk.Coins) error {
	if bonus.IsZero() {
		return nil
	}

	poolAddr := k.accountKeeper.GetModuleAddress(types.RewardsPoolName)
	poolBalance := k.bankKeeper.GetAllBalances(ctx, poolAddr)
	if !poolBalance.IsAllGTE(bonus) {
		return errorsmod.Wrapf(types.ErrInsufficientBonusPool,
			"bonus: %s, pool balance: %s",
			bonus.String(), poolBalance.String())
	}

	return nil
