### Title
Attacker-Controlled Proof Inputs Enable Merkle Second-Preimage Forgery in `verify_transaction_inclusion` — (File: `contract/src/lib.rs`)

---

### Summary

The deprecated-but-still-callable `verify_transaction_inclusion` function accepts fully attacker-controlled proof inputs and performs no tree-depth validation. Any unprivileged NEAR caller can supply an internal Merkle node hash as `tx_id` with a crafted proof that correctly computes to a real block's `merkle_root`, causing the function to return `true` for a Bitcoin transaction that was never mined. Downstream contracts that gate fund releases or cross-chain actions on this oracle's result are directly exploitable.

---

### Finding Description

`verify_transaction_inclusion` is publicly callable with no role restriction: [1](#0-0) 

It delegates proof evaluation entirely to `compute_root_from_merkle_proof`: [2](#0-1) 

`compute_root_from_merkle_proof` iterates over the caller-supplied `merkle_proof` elements and hashes them together with the caller-supplied `tx_id`, starting at the caller-supplied `tx_index`. It performs **no validation of tree depth, transaction count, or whether `tx_id` is a leaf node**. The only guard in `verify_transaction_inclusion` against a trivially empty proof is: [3](#0-2) 

This does not prevent the second-preimage attack. The function's own documentation acknowledges the root cause: [4](#0-3) 

The function is marked `#[deprecated]` (a Rust compiler hint only) and `#[pause]` (pausable by a privileged role, but active by default). It remains a live, publicly callable NEAR view/call method. The `ProofArgs` struct exposes every field to the caller: [5](#0-4) 

---

### Impact Explanation

Any NEAR contract that calls `verify_transaction_inclusion` and uses its `true` return value to authorize an action — releasing wrapped BTC, unlocking collateral, processing a cross-chain transfer — can be deceived. The attacker does not need to mine a block or break any cryptographic primitive; they only need the public merkle tree structure of any real Bitcoin block already stored in the contract's `headers_pool`.

The corrupted invariant is the **proof result**: the function is supposed to return `true` only when `tx_id` is a real leaf-level transaction hash at the claimed position, but it returns `true` for any internal node hash whose subtree root, combined with the supplied proof siblings, equals the block's `merkle_root`.

---

### Likelihood Explanation

- `verify_transaction_inclusion` is callable by any unprivileged NEAR account with no deposit or role requirement.
- The Bitcoin merkle tree structure for every stored block is derivable from public blockchain data.
- The attack requires only constructing a valid `ProofArgs` struct — no special tooling, no privileged access, no social engineering.
- The function is not removed; it is only deprecated at the Rust source level.

---

### Recommendation

1. **Remove `verify_transaction_inclusion` entirely** from the public ABI. A `#[deprecated]` annotation does not prevent on-chain calls. Replace all call sites with `verify_transaction_inclusion_v2`, which enforces equal proof lengths to bind the tree depth via the coinbase proof.
2. If removal is not immediately feasible, add a runtime `require!(false, "use verify_transaction_inclusion_v2")` guard inside the function body so that any direct call panics.
3. Ensure all downstream integrators are notified that any historical call to `verify_transaction_inclusion` returning `true` cannot be trusted without re-verification via v2.

---

### Proof of Concept

Consider a real Bitcoin block stored in `headers_pool` whose `merkle_root = R` was computed from four transactions `[t0, t1, t2, t3]`:

```
        R
       / \
      A   B          A = hash(t0, t1),  B = hash(t2, t3)
     / \ / \
    t0 t1 t2 t3
```

An attacker calls:

```rust
verify_transaction_inclusion(ProofArgs {
    tx_id:              A,          // internal node — NOT a real transaction
    tx_block_blockhash: <any valid mainchain block hash with merkle_root = R>,
    tx_index:           0,          // even index → left child
    merkle_proof:       vec![B],    // one element satisfies the non-empty check
    confirmations:      0,
})
```

Inside `compute_root_from_merkle_proof`:
- `current_hash = A`, `current_position = 0` (even) → `current_hash = hash(A, B) = R`
- Loop ends; returns `R`

`R == header.block_header.merkle_root` → function returns **`true`**.

The internal node `A` is not a Bitcoin transaction. No Bitcoin transaction with txid `A` exists. The contract has certified a fabricated inclusion proof. [6](#0-5) [7](#0-6)

    pub fn verify_transaction_inclusion(&self, #[serializer(borsh)] args: ProofArgs) -> bool {
        require!(
            args.confirmations <= self.gc_threshold,
            "The required number of confirmations exceeds the number of blocks stored in memory"
        );

        let heaviest_block_header = self
            .headers_pool
            .get(&self.mainchain_tip_blockhash)
            .unwrap_or_else(|| env::panic_str(ERR_KEY_NOT_EXIST));
        let target_block_height = self
            .mainchain_header_to_height
            .get(&args.tx_block_blockhash)
            .unwrap_or_else(|| env::panic_str("block does not belong to the current main chain"));

        // Check requested confirmations. No need to compute proof if insufficient confirmations.
        require!(
            (heaviest_block_header.block_height).saturating_sub(target_block_height) + 1
                >= args.confirmations,
            "Not enough blocks confirmed"
        );

        let header = self
            .headers_pool
            .get(&args.tx_block_blockhash)
            .unwrap_or_else(|| env::panic_str("cannot find requested transaction block"));

        require!(!args.merkle_proof.is_empty(), "Merkle proof is empty");

        // compute merkle tree root and check if it matches block's original merkle tree root
        merkle_tools::compute_root_from_merkle_proof(
            args.tx_id,
            usize::try_from(args.tx_index).unwrap(),
            &args.merkle_proof,
        ) == header.block_header.merkle_root
    }

pub fn compute_root_from_merkle_proof(
    transaction_hash: H256,
    transaction_position: usize,
    merkle_proof: &Vec<H256>,
) -> H256 {
    let mut current_hash = transaction_hash;
    let mut current_position = transaction_position;

    for proof_hash in merkle_proof {
        if current_position % 2 == 0 {
            current_hash = compute_hash(&current_hash, proof_hash);
        } else {
            current_hash = compute_hash(proof_hash, &current_hash);
        }
        current_position /= 2;
    }

    current_hash
}
