### Title
`ValidResourceBounds` Variant Collapse in Protobuf Deserialization Produces Wrong Transaction Hash for `AllResources` Transactions with Zero L2/L1_DATA_GAS — (File: `crates/apollo_protobuf/src/converters/transaction.rs`)

---

### Summary

The `TryFrom<protobuf::ResourceBounds> for ValidResourceBounds` implementation silently collapses `AllResources { l1_gas: X, l2_gas: 0, l1_data_gas: 0 }` into `L1Gas(X)` whenever both L2 and L1_DATA_GAS are zero. Because `get_tip_resource_bounds_hash` produces structurally different Poseidon preimages for these two variants (the `AllResources` path appends a third `L1_DATA_GAS` element; the `L1Gas` path does not), any Invoke V3 transaction submitted via the RPC gateway with `AllResourceBounds` but zero L2/L1_DATA_GAS will carry a hash computed under the `AllResources` domain, yet be re-hashed under the `L1Gas` domain by every node that receives it through the P2P state-sync protobuf path. The two hashes are irreconcilably different.

---

### Finding Description

**Step 1 — Submission path (RPC → Internal → hash)**

`RpcInvokeTransactionV3.resource_bounds` is typed `AllResourceBounds` (never `ValidResourceBounds`). Any transaction submitted via the JSON-RPC gateway is converted to `InternalRpcInvokeTransactionV3`, whose `InvokeTransactionV3Trait::resource_bounds()` always returns `ValidResourceBounds::AllResources(self.resource_bounds)`: [1](#0-0) 

The hash is then computed by `get_invoke_transaction_v3_hash`, which calls `get_tip_resource_bounds_hash`: [2](#0-1) 

For `AllResources { l1_gas: X, l2_gas: 0, l1_data_gas: 0 }`, `get_tip_resource_bounds_hash` builds a three-element vector `[concat(X, L1_GAS), concat(0, L2_GAS), concat(0, L1_DATA_GAS)]` and hashes it: [3](#0-2) 

Call this hash **H₁**.

**Step 2 — State-sync path (protobuf → `InvokeTransactionV3` → hash)**

When a syncing node receives the same block over P2P, it deserializes the transaction via `TryFrom<protobuf::InvokeV3> for InvokeTransactionV3`, which calls `ValidResourceBounds::try_from(protobuf::ResourceBounds)`: [4](#0-3) 

The critical branch:
```rust
Ok(if l1_data_gas.is_zero() && l2_gas.is_zero() {
    ValidResourceBounds::L1Gas(l1_gas)   // ← variant collapsed
} else {
    ValidResourceBounds::AllResources(...)
})
```

Because the submitted transaction has `l2_gas = 0` and `l1_data_gas = 0`, the deserialized `InvokeTransactionV3` carries `ValidResourceBounds::L1Gas(X)`.

`get_tip_resource_bounds_hash` for `L1Gas(X)` builds only a two-element vector `[concat(X, L1_GAS), concat(0, L2_GAS)]` — the `L1_DATA_GAS` element is absent: [5](#0-4) 

Call this hash **H₂**. Because Poseidon is length-sensitive, **H₁ ≠ H₂** for any non-trivial `X`.

**Step 3 — Divergence is permanent**

The storage serializer for `InvokeTransactionV3` uses the JSON-derived `Serialize`/`Deserialize` for `ValidResourceBounds`, which preserves the `AllResources` variant (it writes the `L1DataGas` key into the map even when zero): [6](#0-5) 

So the sequencer's own storage round-trips correctly. Only the protobuf path collapses the variant. The mempool P2P path (`TryFrom<protobuf::MempoolTransaction> for RpcTransaction`) is unaffected because it converts to `AllResourceBounds` directly: [7](#0-6) 

The divergence is therefore isolated to the **block state-sync protobuf path** (`TryFrom<protobuf::InvokeV3> for InvokeTransactionV3`).

---

### Impact Explanation

Any node that receives a block via P2P state sync and re-derives the transaction hash from the deserialized `InvokeTransactionV3` (e.g., for block-hash commitment verification, re-execution, or RPC `starknet_getTransactionByHash` consistency checks) will compute **H₂** instead of **H₁**. Concretely:

- **Block hash verification failure**: `calculate_block_hash` commits to `transaction_commitment`, which is a Merkle root over per-transaction leaf hashes. If a syncing node recomputes leaf hashes from deserialized transactions, the commitment diverges from the sequencer's, causing the block to be rejected as invalid.
- **Wrong executable payload**: `convert_internal_rpc_tx_to_executable_tx` stores `tx_hash` alongside the transaction. If the hash stored by a syncing node is H₂ while the canonical hash is H₁, any downstream logic keyed on the hash (receipts, events, proof-fact lookup) will silently bind the wrong value.
- **RPC tracing/simulation divergence**: `starknet_getTransactionReceipt` and `starknet_traceTransaction` keyed on H₁ will not find the entry stored under H₂, returning authoritative-looking wrong results.

This matches the allowed impact scope: **"Transaction conversion or signature/hash logic binds the wrong signer, hash, type, or executable payload"** (High) and **"Wrong state, receipt, event … or revert result from blockifier/syscall/execution logic for accepted input"** (Critical).

---

### Likelihood Explanation

The trigger requires submitting an Invoke V3 transaction with `AllResourceBounds` where both `l2_gas` and `l1_data_gas` are zero. This is a fully valid RPC submission — the gateway's stateless validator checks only that at least one resource bound is non-zero (L1 gas), and the stateful validator checks only that the L2 gas price is acceptable. No existing gate rejects `AllResources` with zero L2/L1_DATA_GAS. A user can craft this transaction without any privileged access.

---

### Recommendation

In `TryFrom<protobuf::ResourceBounds> for ValidResourceBounds`, remove the variant-collapsing heuristic. The protobuf wire format already carries all three resource fields; the receiver should always reconstruct `AllResources` when all three fields are present (even if zero), reserving `L1Gas` only for legacy messages that genuinely omit `l2_gas` and `l1_data_gas`:

```rust
// Before (collapses AllResources → L1Gas when l2 and l1_data are zero):
Ok(if l1_data_gas.is_zero() && l2_gas.is_zero() {
    ValidResourceBounds::L1Gas(l1_gas)
} else {
    ValidResourceBounds::AllResources(AllResourceBounds { l1_gas, l2_gas, l1_data_gas })
})

// After (preserve AllResources when all three fields were present on the wire):
Ok(if value.l1_data_gas.is_none() && value.l2_gas.is_none() {
    ValidResourceBounds::L1Gas(l1_gas)   // truly legacy: fields absent
} else {
    ValidResourceBounds::AllResources(AllResourceBounds { l1_gas, l2_gas, l1_data_gas })
})
```

Alternatively, add a boolean tag to the protobuf `ResourceBounds` message to explicitly encode which `ValidResourceBounds` variant was intended.

// An implementation of the SNIP: https://github.com/EvyatarO/SNIPs/blob/snip-8/SNIPS/snip-8.md
pub fn get_tip_resource_bounds_hash(
    resource_bounds: &ValidResourceBounds,
    tip: &Tip,
) -> Result<Felt, StarknetApiError> {
    let l1_resource_bounds = resource_bounds.get_l1_bounds();
    let l2_resource_bounds = resource_bounds.get_l2_bounds();

    // L1 and L2 gas bounds always exist.
    // Old V3 txs always have L2 gas bounds of zero, but they exist.
    let mut resource_felts = vec![
        get_concat_resource(&l1_resource_bounds, L1_GAS)?,
        get_concat_resource(&l2_resource_bounds, L2_GAS)?,
    ];

    // For new V3 txs, need to also hash the data gas bounds.
    resource_felts.extend(match resource_bounds {
        ValidResourceBounds::L1Gas(_) => vec![],
        ValidResourceBounds::AllResources(all_resources) => {
            vec![get_concat_resource(&all_resources.l1_data_gas, L1_DATA_GAS)?]
        }
    });

    Ok(HashChain::new().chain(&tip.0.into()).chain_iter(resource_felts.iter()).get_poseidon_hash())
}

impl TryFrom<protobuf::ResourceBounds> for ValidResourceBounds {
    type Error = ProtobufConversionError;
    fn try_from(value: protobuf::ResourceBounds) -> Result<Self, Self::Error> {
        let Some(l1_gas) = value.l1_gas else {
            return Err(missing("ResourceBounds::l1_gas"));
        };
        let Some(l2_gas) = value.l2_gas else {
            return Err(missing("ResourceBounds::l2_gas"));
        };
        // TODO(Shahak): Assert data gas is not none once we remove support for 0.13.2.
        let l1_data_gas = value.l1_data_gas.unwrap_or_default();
        let l1_gas: ResourceBounds = l1_gas.try_into()?;
        let l2_gas: ResourceBounds = l2_gas.try_into()?;
        let l1_data_gas: ResourceBounds = l1_data_gas.try_into()?;
        Ok(if l1_data_gas.is_zero() && l2_gas.is_zero() {
            ValidResourceBounds::L1Gas(l1_gas)
        } else {
            ValidResourceBounds::AllResources(AllResourceBounds { l1_gas, l2_gas, l1_data_gas })
        })
    }
