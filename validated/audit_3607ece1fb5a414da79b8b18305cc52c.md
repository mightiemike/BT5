Audit Report

## Title
`SwapAllowlistExtension.beforeSwap` Checks Router Address Instead of Originating Caller, Enabling Full Per-User Allowlist Bypass via `MetricOmmSimpleRouter` — (`metric-periphery/contracts/extensions/SwapAllowlistExtension.sol`)

## Summary
`SwapAllowlistExtension.beforeSwap` gates swaps by checking the `sender` argument, which `MetricOmmPool.swap` sets to `msg.sender` — the immediate caller of `pool.swap()`. When swaps are routed through `MetricOmmSimpleRouter`, the router is `msg.sender` at the pool boundary, not the originating user. If the pool admin allowlists the router (a routine operational step), every unprivileged address can bypass the per-user allowlist by routing through the router, completely defeating the access-control invariant.

## Finding Description

`MetricOmmPool.swap` passes `msg.sender` as the `sender` argument to `_beforeSwap`: [1](#0-0) 

`ExtensionCalling._beforeSwap` forwards that value unchanged to the extension: [2](#0-1) 

`SwapAllowlistExtension.beforeSwap` checks whether `sender` (the pool's `msg.sender`) is on the per-pool allowlist: [3](#0-2) 

`MetricOmmSimpleRouter.exactInputSingle` calls `pool.swap()` directly with no forwarding of the original caller's identity — the router is `msg.sender` at the pool boundary: [4](#0-3) 

The same identity-loss occurs in `exactInput`, `exactOutputSingle`, and `exactOutput`: [5](#0-4) 

The extension has no mechanism to recover the originating user's address; `extensionData` is passed through from the router call but the router never encodes `msg.sender` into it.

## Impact Explanation

A pool admin who deploys `SwapAllowlistExtension` to restrict swaps to specific addresses will naturally also allowlist the router so that permitted users can use it. Once the router is allowlisted, `allowedSwapper[pool][router] == true`, and any unprivileged address calling `router.exactInputSingle(...)` causes the extension to see `sender = router`, which passes the check. The per-user allowlist is completely defeated. This is a direct admin-boundary break: an unprivileged path (the public router) bypasses the pool admin's access-control gate, matching the "admin-boundary break: factory/oracle role checks are bypassed by an unprivileged path" allowed impact class.

## Likelihood Explanation

The trigger is the pool admin allowlisting the router — a routine, non-malicious operational step required to make the pool usable through the standard periphery for any permitted user. No special attacker capability is required beyond calling the public router. Once the router is allowlisted, the bypass is immediately and repeatedly exploitable by any address.

## Recommendation

The extension must gate the **originating user**, not the intermediary. The router should encode `msg.sender` into `extensionData` on every `pool.swap()` call. `SwapAllowlistExtension.beforeSwap` should decode and verify that address when `sender` is a known router (or always, with a fallback to `sender` for direct calls). This ensures the allowlist check always targets the actual economic actor regardless of routing path.

## Proof of Concept

```solidity
// Setup: pool with SwapAllowlistExtension; only permittedUser is allowlisted.
// Admin also allowlists the router so permittedUser can use it.
extension.setAllowedToSwap(address(pool), address(router), true);
extension.setAllowedToSwap(address(pool), permittedUser, true);

// Attacker is NOT allowlisted.
// Direct swap reverts as expected:
vm.prank(attacker);
vm.expectRevert(IMetricOmmPoolActions.NotAllowedToSwap.selector);
pool.swap(attacker, false, 1000, type(uint128).max, "", "");

// Router-mediated swap succeeds — allowlist bypassed:
vm.prank(attacker);
router.exactInputSingle(
    IMetricOmmSimpleRouter.ExactInputSingleParams({
        pool: address(pool),
        recipient: attacker,
        tokenIn: address(token1),
        zeroForOne: false,
        amountIn: 1000,
        amountOutMinimum: 0,
        priceLimitX64: type(uint128).max,
        deadline: block.timestamp,
        extensionData: ""
    })
);
// Attacker receives token0 — allowlist was not enforced.
// Root cause: extension sees sender = router (allowlisted), not attacker.
```
