Audit Report

## Title
`SwapAllowlistExtension` checks router address instead of end user, allowing full allowlist bypass via `MetricOmmSimpleRouter` — (File: metric-periphery/contracts/extensions/SwapAllowlistExtension.sol)

## Summary
`SwapAllowlistExtension.beforeSwap` gates swaps by checking the `sender` argument, which `MetricOmmPool.swap` sets to `msg.sender` — the immediate caller of `pool.swap()`. When swaps are routed through `MetricOmmSimpleRouter`, the router contract is `msg.sender`, so the extension checks `allowedSwapper[pool][router]` rather than `allowedSwapper[pool][actual_user]`. Any pool admin who allowlists the router to enable router-mediated swaps for legitimate users simultaneously grants unrestricted swap access to every non-allowlisted user.

## Finding Description
`MetricOmmPool.swap` passes `msg.sender` as the `sender` argument to `_beforeSwap`:

```solidity
_beforeSwap(
    msg.sender,   // whoever called pool.swap()
    recipient, ...
);
```

`SwapAllowlistExtension.beforeSwap` checks that value against the per-pool allowlist:

```solidity
if (!allowAllSwappers[msg.sender] && !allowedSwapper[msg.sender][sender]) {
    revert IMetricOmmPoolActions.NotAllowedToSwap();
}
```

`MetricOmmSimpleRouter.exactInputSingle` (and `exactInput`, `exactOutputSingle`, `exactOutput`) calls `pool.swap()` directly with no originator forwarded through `extensionData`:

```solidity
_setNextCallbackContext(params.pool, CALLBACK_MODE_JUST_PAY, msg.sender, params.tokenIn);
(int128 amount0Delta, int128 amount1Delta) = IMetricOmmPoolActions(params.pool)
    .swap(params.recipient, params.zeroForOne, ..., params.extensionData);
```

`_setNextCallbackContext` stores `msg.sender` only for the payment callback; it is never forwarded to the extension. The extension therefore receives `address(router)` as `sender` and evaluates `allowedSwapper[pool][router]`. If the router is allowlisted (the only way to let any user use the router on a restricted pool), the check passes for every caller regardless of individual allowlist status. No existing guard in the router or pool corrects this.

## Impact Explanation
A pool configured with `SwapAllowlistExtension` to restrict swaps to KYC-verified counterparties, institutional LPs, or whitelisted market makers is fully bypassed by any user who calls `MetricOmmSimpleRouter` instead of `pool.swap()` directly, provided the router is allowlisted. The unauthorized user executes swaps against the pool's liquidity at oracle-derived prices, enabling unauthorized fund extraction from LP positions in restricted pools. This is a broken access-control invariant with direct fund-impacting consequences — matching the "broken core pool functionality causing loss of funds" and "admin-boundary break bypassed by an unprivileged path" allowed impacts.

## Likelihood Explanation
The router is the primary user-facing entry point. Any pool admin enabling router access for allowlisted users must allowlist the router, which simultaneously opens the gate to all users. The bypass requires no special privileges, no flash loans, and no unusual token behavior — only a standard router call. Any non-allowlisted user who observes the pool is restricted can trivially route around the guard. The condition is met in the normal operational configuration of any restricted pool that supports router usage.

## Recommendation
The `sender` forwarded to extensions must represent the economic actor (the end user), not the immediate caller of `pool.swap()`.

1. **Short term:** Document that pools using `SwapAllowlistExtension` cannot be used with `MetricOmmSimpleRouter` for per-user gating, and enforce this at the extension level by reverting if `sender` is a known router address.

2. **Long term:** Introduce an `originator` field that the router populates by appending `msg.sender` to `extensionData` before calling `pool.swap()`. Have `SwapAllowlistExtension.beforeSwap` decode and verify the originator from `extensionData` when the immediate `sender` is a known router, checking `allowedSwapper[pool][originator]` instead of `allowedSwapper[pool][sender]`. Alternatively, expose a `swapWithOriginator(address originator, ...)` entry point on the pool that the router calls, passing the true user address, and have the extension check `originator`.

## Proof of Concept
1. Deploy a pool with `SwapAllowlistExtension` configured as a `beforeSwap` hook.
2. Pool admin calls `swapExtension.setAllowedToSwap(pool, address(router), true)` to enable router-mediated swaps.
3. Alice (not individually allowlisted) calls `router.exactInputSingle(...)` targeting the pool.
4. The router calls `pool.swap(...)` — `msg.sender` of that call is `address(router)`.
5. The pool calls `_beforeSwap(address(router), ...)`. The extension checks `allowedSwapper[pool][router]` → `true`. Alice's swap succeeds despite not being individually allowlisted.
6. Bob (individually allowlisted via `setAllowedToSwap(pool, bob, true)`) calls `pool.swap()` directly → also succeeds.
7. The allowlist provides no per-user gating for any user who routes through the router.
