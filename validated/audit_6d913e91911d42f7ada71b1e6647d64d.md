Audit Report

## Title
`SwapAllowlistExtension` checks the immediate pool caller (router) instead of the end user, allowing any user to bypass the per-user swap allowlist via `MetricOmmSimpleRouter` — (`metric-periphery/contracts/extensions/SwapAllowlistExtension.sol`)

## Summary
`SwapAllowlistExtension.beforeSwap` gates swaps by checking `allowedSwapper[msg.sender][sender]`, where `sender` is the direct caller of `pool.swap`. When `MetricOmmSimpleRouter` is used, the pool receives `msg.sender = router`, so the extension checks the router's allowlist status rather than the end user's. Any pool that allowlists the router to support router-mediated swaps simultaneously grants every on-chain address unrestricted swap access, completely defeating the per-user allowlist.

## Finding Description
**Root cause — three confirmed code facts:**

1. `MetricOmmPool.swap` passes `msg.sender` verbatim as the `sender` argument to `_beforeSwap`: [1](#0-0) 

2. `SwapAllowlistExtension.beforeSwap` checks `allowedSwapper[msg.sender][sender]`, where `msg.sender` is the pool and `sender` is whoever called `pool.swap`: [2](#0-1) 

3. `MetricOmmSimpleRouter.exactInputSingle` calls `pool.swap` directly, making the pool see `msg.sender = router`, not the end user: [3](#0-2) 

The same pattern holds for `exactInput`, `exactOutputSingle`, and `exactOutput` — in every case the pool receives the router as `msg.sender`. [4](#0-3) 

**Exploit flow:**
- Pool admin deploys pool with `SwapAllowlistExtension` as a `beforeSwap` hook.
- Admin calls `setAllowedToSwap(pool, router, true)` so that allowlisted users can use the router.
- Admin does NOT call `setAllowedToSwap(pool, alice, true)`.
- Alice calls `MetricOmmSimpleRouter.exactInputSingle({pool: pool, ...})`.
- Router calls `pool.swap(recipient, ...)` → pool calls `_beforeSwap(msg.sender=router, ...)` → extension checks `allowedSwapper[pool][router] == true` → swap executes.
- Alice, who is not individually allowlisted, successfully swaps on the curated pool.

No existing guard prevents this: the extension has no mechanism to recover the originating user from `extensionData` or transient storage, and the router stores the original `msg.sender` only in transient storage for callback payment purposes — it is never forwarded to the extension. [5](#0-4) 

## Impact Explanation
This is an admin-boundary break triggered by an unprivileged path. The pool admin's allowlist configuration — intended to restrict swaps to a curated set of counterparties — is fully bypassed by any user calling through the canonical router. LPs who deposited under the assumption that only approved counterparties could trade against them are exposed to unapproved traders. The pool admin faces an impossible choice: either block the router entirely (breaking core swap functionality for allowlisted users) or allowlist the router (opening the pool to all users). In the natural configuration the allowlist provides zero protection, constituting a direct policy bypass with fund-impact consequences for LPs.

## Likelihood Explanation
`MetricOmmSimpleRouter` is the canonical, documented periphery entry point for swaps. Any pool operator who deploys `SwapAllowlistExtension` and wants end users to use the router will naturally allowlist the router address. The bypass requires no special privileges, no flash loans, and no unusual token behavior — a single standard router call suffices. The precondition (router allowlisted) is the expected production configuration, making this highly likely to be triggered.

## Recommendation
The `sender` argument delivered to `beforeSwap` must represent the economic actor, not the immediate `msg.sender` of `pool.swap`. Two complementary fixes:

1. **In `MetricOmmSimpleRouter`**: encode the originating `msg.sender` into `extensionData` (or a dedicated transient slot) so extensions can recover the true initiator.
2. **In `SwapAllowlistExtension`**: decode the true initiator from `extensionData` when present and check that address, falling back to `sender` for direct pool calls.

Alternatively, provide a separate `OriginAllowlistExtension` that enforces per-user identity through a signed payload or transient-storage originator field, and document clearly that `SwapAllowlistExtension` only gates the immediate pool caller.

## Proof of Concept
```
Setup:
  1. Deploy pool with SwapAllowlistExtension as beforeSwap hook.
  2. Pool admin: setAllowedToSwap(pool, router, true)   // router allowlisted for UX
  3. Pool admin does NOT call setAllowedToSwap(pool, alice, true)

Attack:
  4. Alice calls MetricOmmSimpleRouter.exactInputSingle({pool: pool, ...})
     → router calls pool.swap(recipient, ...)                    [MetricOmmSimpleRouter.sol L72-80]
     → pool calls _beforeSwap(msg.sender=router, ...)            [MetricOmmPool.sol L230-240]
     → extension checks allowedSwapper[pool][router] == true ✓  [SwapAllowlistExtension.sol L37]
     → swap executes successfully

Result:
  Alice, not individually allowlisted, swaps on the curated pool
  because the extension checked the router's status, not hers.
```

```text
    _checkDeadline(params.deadline);
    _validatePath(params.tokens, params.pools, params.extensionDatas);

    uint256 last = params.pools.length - 1;
    int128 amount = MetricOmmSwapInputs.asAmountSpecifiedIn(params.amountIn);

    for (uint256 i = 0; i <= last; i++) {
      address pool = params.pools[i];
      bool zeroForOne = MetricOmmSwapPath.resolveZeroForOneBitmap(params.zeroForOneBitMap, i);

      _setNextCallbackContext(pool, CALLBACK_MODE_JUST_PAY, i == 0 ? msg.sender : address(this), params.tokens[i]);
      (int128 amount0Delta, int128 amount1Delta) = IMetricOmmPoolActions(pool)
        .swap(
          i == last ? params.recipient : address(this),
          zeroForOne,
          amount,
          MetricOmmSwapPath.openLimit(zeroForOne),
          "",
          params.extensionDatas[i]
        );

      int128 amountInActual = MetricOmmSwapResults.extractAmountIn(zeroForOne, amount0Delta, amount1Delta);
      if (amountInActual < amount) revert InvalidInputAmountAtHop(uint8(i), amountInActual, amount);

    if (amountOut < params.amountOutMinimum) revert InsufficientOutput(amountOut, params.amountOutMinimum);

    _clearExpectedCallbackPool();
  }
```
