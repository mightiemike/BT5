### Title
Deprecated `verify_transaction_inclusion` Remains Publicly Callable Without Coinbase Proof Guard, Enabling 64-Byte Merkle Forgery — (`contract/src/lib.rs`)

### Summary

`verify_transaction_inclusion` (deprecated since v0.5.0) applies the same single-pass Merkle proof check for every transaction index, including index 0. `verify_transaction_inclusion_v2` was introduced specifically to add a coinbase-proof guard that closes the 64-byte internal-node forgery path. Because the deprecated function is still `pub` and carries only a Rust compiler warning — not an access restriction — any unprivileged NEAR caller can invoke it directly and receive a `true` result for a fabricated transaction inclusion claim.

### Finding Description

`verify_transaction_inclusion` computes the Merkle root from the caller-supplied `tx_id` and `merkle_proof` and compares it to the stored `merkle_root`: [1](#0-0) 

No coinbase-level proof is required. `verify_transaction_inclusion_v2` fixes this by first demanding that a separate `coinbase_merkle_proof` correctly reconstructs the same `merkle_root` at index 0, then delegating to the deprecated function: [2](#0-1) 

The deprecated function is still declared `pub` with only `#[pause]` gating (not role-gated), so any NEAR account can call it directly, bypassing the coinbase guard entirely: [3](#0-2) 

### Impact Explanation

The 64-byte transaction forgery (documented at https://www.bitmex.com/blog/64-Byte-Transactions) allows an attacker to supply a 64-byte value that is a valid internal Merkle-tree node as `tx_id`. Because `compute_root_from_merkle_proof` treats it as a leaf, the computed root can match the genuine `merkle_root` stored in a real, mainchain block header. `verify_transaction_inclusion` then returns `true` for a transaction that was never broadcast or mined. Any bridge, atomic-swap contract, or other consumer that calls the deprecated entry point will accept this forged proof as valid, potentially releasing funds or triggering irreversible cross-chain state changes.

The corrupted protocol value is the **proof result** returned by `verify_transaction_inclusion`: it is `true` when it must be `false`.

### Likelihood Explanation

The attack requires no privileged role, no key material, and no social engineering. The attacker needs only:
1. A real mainchain block hash accepted by the contract (publicly readable via `get_block_hash_by_height`).
2. Knowledge of the block's Merkle tree structure (available from any Bitcoin node).
3. A crafted 64-byte internal node and a matching partial Merkle path.

The 64-byte forgery technique is publicly documented and has known tooling. Any NEAR account can submit the call.

### Recommendation

Remove the public surface of `verify_transaction_inclusion` entirely, or gate it with a role that prevents external calls (e.g., `#[private]`). All external callers must be directed to `verify_transaction_inclusion_v2`. Alternatively, add the coinbase-proof check directly inside `verify_transaction_inclusion` so both entry points enforce the same security invariant, eliminating the divergence.

### Proof of Concept

1. Identify a mainchain block hash `B` via `get_block_hash_by_height(h)`.
2. Obtain the block's Merkle tree from a Bitcoin node; locate a 64-byte internal node `N` at some tree position `(idx, proof_path)` such that `compute_root_from_merkle_proof(N, idx, proof_path) == merkle_root(B)`.
3. Call `verify_transaction_inclusion({ tx_id: N, tx_block_blockhash: B, tx_index: idx, merkle_proof: proof_path, confirmations: 1 })`.
4. The function returns `true` despite `N` not being a real transaction, because:
   - `B` is in `mainchain_header_to_height` ✓
   - `compute_root_from_merkle_proof(N, idx, proof_path) == header.block_header.merkle_root` ✓
   - No coinbase guard is applied ✓ [4](#0-3)

    #[deprecated(
        since = "0.5.0",
        note = "Use `verify_transaction_inclusion_v2` instead."
    )]
    #[pause]
    pub fn verify_transaction_inclusion(&self, #[serializer(borsh)] args: ProofArgs) -> bool {
        require!(
            args.confirmations <= self.gc_threshold,
            "The required number of confirmations exceeds the number of blocks stored in memory"
        );

        let heaviest_block_header = self
            .headers_pool
            .get(&self.mainchain_tip_blockhash)
            .unwrap_or_else(|| env::panic_str(ERR_KEY_NOT_EXIST));
        let target_block_height = self
            .mainchain_header_to_height
            .get(&args.tx_block_blockhash)
            .unwrap_or_else(|| env::panic_str("block does not belong to the current main chain"));

        // Check requested confirmations. No need to compute proof if insufficient confirmations.
        require!(
            (heaviest_block_header.block_height).saturating_sub(target_block_height) + 1
                >= args.confirmations,
            "Not enough blocks confirmed"
        );

        let header = self
            .headers_pool
            .get(&args.tx_block_blockhash)
            .unwrap_or_else(|| env::panic_str("cannot find requested transaction block"));

        require!(!args.merkle_proof.is_empty(), "Merkle proof is empty");

        // compute merkle tree root and check if it matches block's original merkle tree root
        merkle_tools::compute_root_from_merkle_proof(
            args.tx_id,
            usize::try_from(args.tx_index).unwrap(),
            &args.merkle_proof,
        ) == header.block_header.merkle_root
    }

    pub fn verify_transaction_inclusion_v2(&self, #[serializer(borsh)] args: ProofArgsV2) -> bool {
        require!(
            args.merkle_proof.len() == args.coinbase_merkle_proof.len(),
            "Coinbase merkle proof and transaction merkle proof should have the same length"
        );

        let header = self
            .headers_pool
            .get(&args.tx_block_blockhash)
            .unwrap_or_else(|| env::panic_str("cannot find requested transaction block"));

        require!(
            merkle_tools::compute_root_from_merkle_proof(
                args.coinbase_tx_id.clone(),
                0usize,
                &args.coinbase_merkle_proof,
            ) == header.block_header.merkle_root,
            "Incorrect coinbase merkle proof"
        );

        #[allow(deprecated)]
        self.verify_transaction_inclusion(args.into())
    }
