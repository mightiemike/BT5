### Title
Internal-Node Merkle Forgery in `verify_transaction_inclusion` Returns `true` for Non-Leaf Hash — (`contract/src/lib.rs`)

---

### Summary

`verify_transaction_inclusion` accepts any 32-byte value as `tx_id` and any-length `merkle_proof`. Because `compute_root_from_merkle_proof` performs no proof-depth or leaf-position validation, an attacker can supply an **internal node hash** (e.g., the left subtree root of a 4-tx tree) together with a proof that is one level shorter than the real tree depth. The function reconstructs the correct Merkle root and returns `true`, falsely asserting that a non-existent transaction was included in the block.

---

### Finding Description

`verify_transaction_inclusion` in `contract/src/lib.rs` is a public, unpermissioned NEAR view call. Its only guards are:

- `confirmations <= gc_threshold`
- block hash present in `mainchain_header_to_height` (i.e., on the canonical chain)
- `!merkle_proof.is_empty()` [1](#0-0) 

It then delegates entirely to `compute_root_from_merkle_proof`: [2](#0-1) 

That function iterates over whatever proof elements are supplied, hashing upward, with **no check** that `merkle_proof.len()` equals `log2(num_txs)` and **no check** that `transaction_hash` is a leaf. The `ProofArgs` struct imposes no structural constraints either: [3](#0-2) 

The code itself acknowledges the flaw in a warning comment but provides no runtime guard: [4](#0-3) 

The `#[deprecated]` attribute is a compile-time Rust lint; it does not restrict on-chain callability. The `#[pause]` attribute allows a `PauseManager` to disable the function, but the function is **not paused by default**. [5](#0-4) 

---

### Impact Explanation

Any downstream NEAR contract that calls `verify_transaction_inclusion` and trusts its `bool` return value will accept a forged inclusion claim. The attacker does not need to forge a block header or break SHA-256; they only need to know the internal node hashes of a real, confirmed block's Merkle tree, which are fully computable from public blockchain data.

---

### Likelihood Explanation

The function is callable by any unprivileged NEAR account. The required inputs (internal node hash, right-sibling subtree root, a real confirmed block hash) are all derivable from public Bitcoin RPC data. No privileged role, key leak, or social engineering is required.

---

### Recommendation

1. **Remove `verify_transaction_inclusion`** from the deployed contract entirely, or enforce `#[pause]` to be active by default for this method.
2. In `compute_root_from_merkle_proof`, add a `tree_depth` parameter and assert `merkle_proof.len() == tree_depth`, rejecting proofs that are shorter than the full path from leaf to root.
3. Callers should migrate exclusively to `verify_transaction_inclusion_v2`, which enforces equal proof lengths for the coinbase and target transaction, making the short-proof attack infeasible. [6](#0-5) 

---

### Proof of Concept

Build a 4-transaction Merkle tree. Let:

```
tx0, tx1, tx2, tx3  — four real leaf hashes from a confirmed block
h01 = double_sha256(tx0 || tx1)   // left internal node
h23 = double_sha256(tx2 || tx3)   // right internal node
root = double_sha256(h01 || h23)  // Merkle root stored in block header
```

Call `verify_transaction_inclusion` with:

```
tx_id             = h01          // internal node, NOT a real transaction
tx_block_blockhash = <confirmed block hash>
tx_index          = 0
merkle_proof      = [h23]        // proof of length 1, not 2
confirmations     = 1
```

Execution trace through `compute_root_from_merkle_proof`:

```
current_hash     = h01
current_position = 0   (even → left)
iteration 1: current_hash = compute_hash(h01, h23) = root
``` [7](#0-6) 

Result: `root == header.block_header.merkle_root` → **returns `true`**.

The function has accepted `h01` — an internal node, not a transaction — as a valid transaction inclusion proof, violating the invariant that `tx_id` must be a real leaf transaction hash. [8](#0-7)

    pub fn verify_transaction_inclusion(&self, #[serializer(borsh)] args: ProofArgs) -> bool {
        require!(
            args.confirmations <= self.gc_threshold,
            "The required number of confirmations exceeds the number of blocks stored in memory"
        );

        let heaviest_block_header = self
            .headers_pool
            .get(&self.mainchain_tip_blockhash)
            .unwrap_or_else(|| env::panic_str(ERR_KEY_NOT_EXIST));
        let target_block_height = self
            .mainchain_header_to_height
            .get(&args.tx_block_blockhash)
            .unwrap_or_else(|| env::panic_str("block does not belong to the current main chain"));

        // Check requested confirmations. No need to compute proof if insufficient confirmations.
        require!(
            (heaviest_block_header.block_height).saturating_sub(target_block_height) + 1
                >= args.confirmations,
            "Not enough blocks confirmed"
        );

        let header = self
            .headers_pool
            .get(&args.tx_block_blockhash)
            .unwrap_or_else(|| env::panic_str("cannot find requested transaction block"));

        require!(!args.merkle_proof.is_empty(), "Merkle proof is empty");

        // compute merkle tree root and check if it matches block's original merkle tree root
        merkle_tools::compute_root_from_merkle_proof(
            args.tx_id,
            usize::try_from(args.tx_index).unwrap(),
            &args.merkle_proof,
        ) == header.block_header.merkle_root
    }

#[must_use]
pub fn compute_root_from_merkle_proof(
    transaction_hash: H256,
    transaction_position: usize,
    merkle_proof: &Vec<H256>,
) -> H256 {
    let mut current_hash = transaction_hash;
    let mut current_position = transaction_position;

    for proof_hash in merkle_proof {
        if current_position % 2 == 0 {
            current_hash = compute_hash(&current_hash, proof_hash);
        } else {
            current_hash = compute_hash(proof_hash, &current_hash);
        }
        current_position /= 2;
    }

    current_hash
}
