### Title
Division-by-Zero in `socializeSubaccount` When `openInterest == 0` Permanently Blocks Liquidation Finalization — (`core/contracts/PerpEngine.sol`)

---

### Summary

`PerpEngine.socializeSubaccount` unconditionally divides by `state.openInterest` at line 166 when insurance cannot fully cover a negative `vQuoteBalance`. There is no guard for the case where `state.openInterest == 0`, which is reachable when a subaccount has closed all its positions (setting `amount = 0`) but retains a negative `vQuoteBalance` (realized loss). `MathSD21x18.div` explicitly reverts with `"DBZ"` on a zero denominator, permanently blocking the liquidation finalization path for that subaccount.

---

### Finding Description

`openInterest` in `PerpEngineState._updateBalance` tracks the sum of `|balance.amount|` across all subaccounts: [1](#0-0) 

When a subaccount closes its position (`amountDelta = -balance.amount`), `balance.amount` becomes `0` and `state.openInterest` is decremented by the full position size. The `vQuoteBalance` (realized PnL) is **not** zeroed — it retains the accumulated loss. If no other participants hold open positions, `state.openInterest` reaches `0`.

In `socializeSubaccount`, after insurance is partially applied, the code enters the socialization branch when `balance.vQuoteBalance < 0` still holds: [2](#0-1) 

`MathSD21x18.div` enforces a hard `require(y != 0, ERR_DIV_BY_ZERO)`: [3](#0-2) 

This reverts the entire `socializeSubaccount` call. Since `socializeSubaccount` is called unconditionally inside `_finalizeSubaccount` in `ClearinghouseLiq`: [4](#0-3) 

...the revert propagates up through `liquidateSubaccountImpl`, permanently blocking finalization of the insolvent subaccount.

---

### Impact Explanation

The insolvent subaccount can never be finalized. The loss cannot be socialized or written off. The protocol is left with an unresolvable insolvent subaccount. Any future attempt to finalize it via the liquidation path will always revert. The insurance fund cannot be properly applied or recovered.

---

### Likelihood Explanation

This is reachable in any perp market where:
1. A participant opens a position, then closes it at a realized loss (setting `amount = 0`, `vQuoteBalance < 0`).
2. No other participants hold open positions in that product (`openInterest == 0`).
3. The insurance fund is insufficient to cover the full loss.

This is a realistic single-participant market scenario (e.g., early-stage product, thin market, or after all other participants have exited). No special privileges or admin access are required — only normal trading activity followed by a liquidation attempt.

---

### Recommendation

Add a guard before the division: if `state.openInterest == 0`, there is no one to socialize the loss to, so simply zero out `balance.vQuoteBalance` (absorbing the loss into the protocol) and continue:

```solidity
if (balance.vQuoteBalance < 0) {
    if (state.openInterest == 0) {
        // No open positions to socialize against; write off the loss
        balance.vQuoteBalance = 0;
    } else {
        int128 fundingPerShare = -balance.vQuoteBalance.div(state.openInterest);
        state.cumulativeFundingLongX18 += fundingPerShare;
        state.cumulativeFundingShortX18 -= fundingPerShare;
        balance.vQuoteBalance = 0;
    }
}
```

### Proof of Concept

```solidity
// Single-participant market: participant opens, then closes at a loss.
// state.openInterest == 0, balance.vQuoteBalance < 0, insurance < -vQuoteBalance.
// Assert: socializeSubaccount does NOT revert.

function testSocializeZeroOpenInterest() public {
    // 1. Open a long position for subaccountA in productId=2
    perpEngine.updateBalance(2, subaccountA, 1e18, -2000e18); // amount=1, vQuote=-2000

    // 2. Close the position at a loss (price dropped to 1500)
    perpEngine.updateBalance(2, subaccountA, -1e18, 1500e18); // amount=0, vQuote=-500

    // Verify: openInterest == 0, vQuoteBalance == -500e18
    (IPerpEngine.State memory s, IPerpEngine.Balance memory b) =
        perpEngine.getStateAndBalance(2, subaccountA);
    assertEq(s.openInterest, 0);
    assertLt(b.vQuoteBalance, 0);

    // 3. Insurance is insufficient (e.g., only 100 available)
    int128 insurance = 100e18;

    // 4. This call currently reverts with "DBZ" — it should not
    vm.prank(address(clearinghouse));
    perpEngine.socializeSubaccount(subaccountA, insurance); // REVERTS
}
```
