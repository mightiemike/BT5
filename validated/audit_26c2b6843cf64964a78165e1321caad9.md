### Title
SwapAllowlistExtension Checks Router Address Instead of End User, Allowing Full Allowlist Bypass via Router ‚Äî (`metric-periphery/contracts/extensions/SwapAllowlistExtension.sol`)

### Summary

`SwapAllowlistExtension.beforeSwap` gates swaps by checking the `sender` argument passed by the pool. The pool always passes `msg.sender` (its direct caller) as `sender`. When a user routes through `MetricOmmSimpleRouter`, the pool's `msg.sender` is the router contract, not the end user. If a pool admin allowlists the router address to enable router-mediated swaps for their curated pool, every unprivileged user can bypass the allowlist by routing through the same public, permissionless router.

### Finding Description

`MetricOmmPool.swap` passes `msg.sender` as the `sender` argument to `_beforeSwap`: [1](#0-0) 

`ExtensionCalling._beforeSwap` forwards that value verbatim to every configured extension: [2](#0-1) 

`SwapAllowlistExtension.beforeSwap` then checks `allowedSwapper[msg.sender][sender]`, where `msg.sender` is the pool and `sender` is whatever the pool forwarded: [3](#0-2) 

`MetricOmmSimpleRouter.exactInputSingle` calls `pool.swap(...)` directly, making the router the pool's `msg.sender`: [4](#0-3) 

The router is a public, permissionless contract ‚Äî any EOA can call it. A pool admin who wants allowlisted users to be able to use the router (the standard periphery path) must add the router to `allowedSwapper`. Once the router is allowlisted, the check `allowedSwapper[pool][router]` returns `true` for every caller, regardless of who the actual end user is. The extension has no way to recover the original EOA from the router call.

The same structural problem exists for `exactInput`, `exactOutputSingle`, and `exactOutput`: all of them call `pool.swap` from the router's address. [5](#0-4) 

### Impact Explanation

Any user who is not on the allowlist can trade on a curated, access-controlled pool by routing through `MetricOmmSimpleRouter`. The pool receives and pays out real tokens; the swap executes at live oracle prices. The allowlist ‚Äî the sole access-control mechanism on the swap path ‚Äî is silently bypassed. Unauthorized actors can drain LP value, front-run allowlisted traders, or violate compliance requirements (e.g., KYC-gated pools). This is a direct loss-of-policy impact with fund-level consequences for LP providers who deployed the pool expecting restricted access.

### Likelihood Explanation

The `MetricOmmSimpleRouter` is the canonical, documented periphery swap path. A pool admin who wants allowlisted users to be able to use the router ‚Äî the normal UX ‚Äî must allowlist the router address. There is no other supported mechanism to grant router access to a subset of users, because the extension only sees the router's address. The misconfiguration is therefore the natural, expected configuration for any pool that combines the allowlist extension with router support. The bypass requires no special privileges, no flash loans, and no unusual token behavior ‚Äî any EOA can exploit it in a single transaction.

### Recommendation

Pass the original end-user identity through the swap path so the extension can gate on it. Two options:

1. **Preferred ‚Äî pass `tx.origin` or a signed identity in `extensionData`**: Extend the extension to decode a user-supplied identity from `extensionData` and verify it (e.g., via a signature or a trusted forwarder pattern). The router would forward the caller's address inside `extensionData`.

2. **Simpler ‚Äî check `recipient` instead of `sender`**: For single-hop swaps the recipient is often the end user. This is not general (multi-hop intermediate recipients are the router itself), but it closes the single-hop case.

The core invariant that must hold: the identity checked by `SwapAllowlistExtension` must be the economic actor who benefits from the swap, not the intermediate contract that relays the call.

### Proof of Concept

```
Setup:
  pool = MetricOmmPool with SwapAllowlistExtension (beforeSwap order)
  admin calls setAllowedToSwap(pool, router, true)   // to enable router swaps
  admin does NOT call setAllowedToSwap(pool, attacker, true)

Attack:
  attacker (not allowlisted) calls:
    MetricOmmSimpleRouter.exactInputSingle({
      pool: pool,
      recipient: attacker,
      ...
    })

  Router calls pool.swap(attacker, ...) ‚Äî pool's msg.sender = router
  Pool calls _beforeSwap(router, attacker, ...)
  Extension checks allowedSwapper[pool][router] ‚Üí true  ‚úì
  Swap executes; attacker receives output tokens.

Result:
  attacker bypassed the allowlist and traded on a curated pool.
  allowedSwapper[pool][attacker] was never set to true.
