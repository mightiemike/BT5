### Title
SwapAllowlistExtension Checks Router Address Instead of End-User, Allowing Any User to Bypass Per-User Swap Allowlist — (`metric-periphery/contracts/extensions/SwapAllowlistExtension.sol`)

---

### Summary

`SwapAllowlistExtension.beforeSwap` gates swaps by checking the `sender` argument passed by the pool, which is `msg.sender` of the `pool.swap()` call — the **direct caller**, not the originating user. When swaps are routed through `MetricOmmSimpleRouter`, the direct caller is the router contract. If the pool admin allowlists the router (the only way to let allowlisted users use multi-hop routing), every unprivileged user can bypass the per-user restriction by routing through the same public router.

---

### Finding Description

`SwapAllowlistExtension.beforeSwap` performs its identity check as follows:

```solidity
// metric-periphery/contracts/extensions/SwapAllowlistExtension.sol
function beforeSwap(address sender, address, bool, int128, uint128, uint256, uint128, uint128, bytes calldata)
    external view override returns (bytes4)
{
    if (!allowAllSwappers[msg.sender] && !allowedSwapper[msg.sender][sender]) {
        revert IMetricOmmPoolActions.NotAllowedToSwap();
    }
    return IMetricOmmExtensions.beforeSwap.selector;
}
``` [1](#0-0) 

The `sender` argument is whatever the pool passes as the first argument to the extension hook. The pool passes `msg.sender` of the `pool.swap()` call — i.e., the **direct caller of the pool**, not the originating EOA.

`MetricOmmSimpleRouter` is a public, permissionless contract. When any user calls `exactInputSingle` or a multi-hop variant, the router calls `pool.swap(recipient, ...)` directly. From the pool's perspective, `msg.sender = router`. The pool then calls `extension.beforeSwap(router, recipient, ...)`.

The allowlist check therefore evaluates `allowedSwapper[pool][router]`, not `allowedSwapper[pool][user]`.

**The impossible dilemma for the pool admin:**

| Admin intent | Admin action | Actual result |
|---|---|---|
| Allow only user A to swap directly | `setAllowedToSwap(pool, userA, true)` | userA can swap directly; router-mediated swaps by userA revert |
| Allow userA to also use the router | `setAllowedToSwap(pool, router, true)` | **Every user** can now swap through the router — allowlist is fully bypassed |

There is no configuration that achieves "only userA can swap, including via the router." The allowlist design makes per-user gating and router usage mutually exclusive. [2](#0-1) 

---

### Impact Explanation

A pool configured with `SwapAllowlistExtension` to restrict swaps to a set of trusted counterparties (e.g., a private institutional pool, a KYC-gated pool, or a pool where only the protocol's own router is meant to trade) can have its allowlist fully bypassed. Any unprivileged user can execute swaps against the pool's liquidity by routing through `MetricOmmSimpleRouter`. This allows unauthorized price-taking against LP positions, draining LP principal and fees beyond what the pool admin intended to permit.

---

### Likelihood Explanation

The trigger is entirely unprivileged: `MetricOmmSimpleRouter` is a public contract callable by anyone. The only precondition is that the pool admin has allowlisted the router address — a natural and expected action for any pool that wants its allowlisted users to access multi-hop routing. The admin has no way to know that allowlisting the router opens the pool to all users, because the `isAllowedToSwap` view function does not expose this implication. [3](#0-2) 

---

### Recommendation

The pool should forward the **originating sender** to the extension rather than `msg.sender`. One approach: add an explicit `sender` parameter to `pool.swap()` that the pool validates (e.g., must equal `msg.sender` for direct calls, or a forwarded address for trusted routers). The router would then pass the original `msg.sender` as the forwarded sender, and the extension would check that value.

Alternatively, the `SwapAllowlistExtension` can be redesigned to check `tx.origin` as a fallback when the direct caller is a known router, though this introduces its own risks. The cleanest fix is a forwarded-sender pattern already used in protocols like Uniswap v4's `msgSender` hook argument. [1](#0-0)

  - Deploy pool with SwapAllowlistExtension as extension1, beforeSwap order = extension1
  - Pool admin calls setAllowedToSwap(pool, userA, true)

Attack:
  - userB (not allowlisted) calls MetricOmmSimpleRouter.exactInputSingle(pool, userB, ...)

  - Pool calls extension.beforeSwap(router, userB, ...)
  - Extension checks allowedSwapper[pool][router] == true  →  passes
  - userB's swap executes against LP funds

Expected: revert NotAllowedToSwap

contract SwapAllowlistExtension is BaseMetricExtension, ISwapAllowlistExtension {
  mapping(address pool => mapping(address swapper => bool)) public allowedSwapper;
  mapping(address pool => bool) public allowAllSwappers;

  constructor(address factory_) BaseMetricExtension(factory_) {}

  function setAllowedToSwap(address pool_, address swapper, bool allowed) external onlyPoolAdmin(pool_) {
    allowedSwapper[pool_][swapper] = allowed;
    emit AllowedToSwapSet(pool_, swapper, allowed);
  }

  function setAllowAllSwappers(address pool_, bool allowed) external onlyPoolAdmin(pool_) {
    allowAllSwappers[pool_] = allowed;
    emit AllowAllSwappersSet(pool_, allowed);
  }

  function isAllowedToSwap(address pool_, address swapper) external view returns (bool) {
    return allowAllSwappers[pool_] || allowedSwapper[pool_][swapper];
