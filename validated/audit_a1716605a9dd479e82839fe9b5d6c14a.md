### Title
SwapAllowlistExtension Checks Router Address Instead of End User — Allowlist Bypassed or Broken for All Router-Mediated Swaps - (File: metric-periphery/contracts/extensions/SwapAllowlistExtension.sol)

---

### Summary

`SwapAllowlistExtension.beforeSwap` gates swaps by checking the `sender` argument, which the pool sets to `msg.sender` of the `swap()` call. When any user routes through `MetricOmmSimpleRouter`, `msg.sender` at the pool is the router contract, not the end user. The extension therefore checks the router's allowlist entry, not the user's. This produces two fund-impacting failure modes: (1) if the router is allowlisted, every public user bypasses the curated-pool gate; (2) if the router is not allowlisted, every legitimately allowlisted user is silently blocked from using the supported periphery.

---

### Finding Description

`SwapAllowlistExtension.beforeSwap` receives `sender` as its first argument and checks it against the per-pool allowlist:

```solidity
function beforeSwap(address sender, address, bool, int128, uint128, uint256, uint128, uint128, bytes calldata)
    external view override returns (bytes4)
{
    if (!allowAllSwappers[msg.sender] && !allowedSwapper[msg.sender][sender]) {
        revert IMetricOmmPoolActions.NotAllowedToSwap();
    }
    return IMetricOmmExtensions.beforeSwap.selector;
}
``` [1](#0-0) 

The pool populates `sender` with `msg.sender` of its own `swap()` entry point:

```solidity
_beforeSwap(
    msg.sender,   // ← always the immediate caller of pool.swap()
    recipient, zeroForOne, amountSpecified, priceLimitX64,
    packedSlot0Initial, bidPriceX64, askPriceX64, extensionData
);
``` [2](#0-1) 

`MetricOmmSimpleRouter.exactInputSingle` calls `pool.swap()` directly, making the router the `msg.sender` at the pool for every hop:

```solidity
_setNextCallbackContext(params.pool, CALLBACK_MODE_JUST_PAY, msg.sender, params.tokenIn);
(int128 amount0Delta, int128 amount1Delta) = IMetricOmmPoolActions(params.pool)
    .swap(params.recipient, params.zeroForOne, ...);
``` [3](#0-2) 

The same pattern holds for `exactInput` (all hops), `exactOutputSingle`, and `exactOutput`: [4](#0-3) 

The extension therefore evaluates `allowedSwapper[pool][router]`, never `allowedSwapper[pool][end_user]`.

---

### Impact Explanation

**Bypass path (High):** A pool admin configures `SwapAllowlistExtension` to restrict swaps to a curated set of addresses. To let those users access the router, the admin calls `setAllowedToSwap(pool, router, true)`. Because the extension checks `sender = router`, every public user who calls any `exact*` function on the router now passes the gate. The allowlist is entirely ineffective. In a pool with oracle-anchored pricing or subsidized rates, unauthorized traders can extract LP value at will.

**Broken-functionality path (Medium):** If the admin does not allowlist the router, every legitimately allowlisted user who calls the router is blocked (`NotAllowedToSwap`). The only working path is a direct `pool.swap()` call, which requires the user to implement their own callback — the supported periphery is unusable for the pool's intended participants.

Both outcomes are reachable by any public user without any privileged action beyond the normal pool configuration the admin already performed.

---

### Likelihood Explanation

The `MetricOmmSimpleRouter` is the primary user-facing swap interface. Any pool that deploys `SwapAllowlistExtension` and expects users to interact through the router will immediately hit one of the two failure modes. The admin's natural remediation attempt (allowlisting the router) triggers the bypass. No special timing, flash loan, or oracle manipulation is required.

---

### Recommendation

The extension must resolve the actual end user, not the proximate caller. Two options:

1. **Pass the original initiator through the router.** Add a `recipient`/`originator` field to the swap call or use a transient-storage slot (analogous to how the router already stores the payer) so the pool can forward the true initiator to the extension.

2. **Check `sender` only for direct calls; for router calls, check a forwarded identity.** The extension could accept an `extensionData` payload containing the verified end-user address, signed or validated by the router, and fall back to `sender` when no payload is present.

The simplest safe fix is option 1: store `msg.sender` of the router call in transient storage before calling the pool, expose it via a view function, and have the extension read it when `sender` is a known router address.

### Proof of Concept

```
Setup:
  pool = deploy MetricOmmPool with SwapAllowlistExtension as beforeSwap hook
  admin calls setAllowedToSwap(pool, router, true)   // intending to enable router access

Attack:
  attacker (not in allowlist) calls:
    router.exactInputSingle({pool: pool, recipient: attacker, ...})

Trace:
  router.exactInputSingle()
    → pool.swap(recipient=attacker, ...)          // msg.sender = router
      → _beforeSwap(sender=router, ...)
        → SwapAllowlistExtension.beforeSwap(sender=router, ...)
          → allowedSwapper[pool][router] == true  // ✓ passes
      → swap executes, attacker receives tokens

Result:
  Non-allowlisted attacker completes swap in a curated pool.
  If pool offers subsidized rates or is vulnerable to oracle-price extraction,
  LP principal is drained by unauthorized traders.
