Audit Report

## Title
`SwapAllowlistExtension` checks router address instead of end-user, allowing full allowlist bypass via `MetricOmmSimpleRouter` - (File: `metric-periphery/contracts/extensions/SwapAllowlistExtension.sol`)

## Summary
`SwapAllowlistExtension.beforeSwap()` gates swaps by checking the `sender` argument, which is always `msg.sender` of `MetricOmmPool.swap()`. When a user routes through `MetricOmmSimpleRouter`, the router is the direct caller of `pool.swap()`, so the extension checks `allowedSwapper[pool][router]` rather than the end-user. Any pool admin who allowlists the router to enable legitimate router-mediated swaps inadvertently opens the pool to every caller, including explicitly excluded addresses.

## Finding Description
`MetricOmmPool.swap()` passes `msg.sender` as `sender` to `_beforeSwap()`: [1](#0-0) 

`SwapAllowlistExtension.beforeSwap()` uses that `sender` value for the allowlist lookup: [2](#0-1) 

In every router entry point — `exactInputSingle`, `exactInput`, `exactOutputSingle`, `exactOutput`, and the recursive `_exactOutputIterateCallback` — the router calls `pool.swap()` directly, making itself `msg.sender`: [3](#0-2) [4](#0-3) [5](#0-4) [6](#0-5) 

In all paths, `sender = router address` reaches the extension. The extension has no mechanism to recover the original end-user. The `allowedSwapper` and `allowAllSwappers` mappings are keyed only on `pool` and `sender`: [7](#0-6) 

Once `allowedSwapper[pool][router] = true`, the guard passes for every caller regardless of identity.

## Impact Explanation
A pool configured with `SwapAllowlistExtension` to restrict swaps to specific addresses (KYC-gated, institutional-only, whitelist-only) is completely open to any caller who routes through `MetricOmmSimpleRouter`. The router is a public, permissionless contract with no access control. Any address — including those the pool admin explicitly excluded — can execute swaps against the restricted pool at oracle-quoted prices. This breaks the core access-control invariant the pool admin configured and constitutes a direct bypass of an admin-configured security boundary, qualifying as a high-severity admin-boundary break.

## Likelihood Explanation
`MetricOmmSimpleRouter` is the standard user-facing entry point for swaps. Any pool admin who wants router-mediated swaps to work for their allowlisted users must allowlist the router address. Once that is done, the bypass is trivially reachable by any address with no special privileges, no malicious setup, and no non-standard tokens — just a normal call to a public router function. No exploit tooling or elevated access is required.

## Recommendation
The extension must check the economically relevant actor (the end-user), not the technical `msg.sender` of `pool.swap()`. Two complementary fixes:

1. **In `MetricOmmSimpleRouter`**: encode `msg.sender` (the true end-user) into `extensionData` for each `pool.swap()` call. The extension decodes and verifies the user address against the allowlist instead of `sender`.
2. **In `SwapAllowlistExtension`**: decode the initiator from `extensionData` (the last parameter of `beforeSwap`) and check that address against `allowedSwapper[msg.sender][initiator]`.

Option 1+2 together are the most robust fix, preserving full call-chain identity without changing the core pool interface.

## Proof of Concept
```
Setup:
  pool configured with SwapAllowlistExtension
  allowedSwapper[pool][router] = true   // admin enables router-mediated swaps
  allowedSwapper[pool][alice]  = true   // alice is approved
  allowedSwapper[pool][bob]    = false  // bob is explicitly excluded

Attack:
  bob calls MetricOmmSimpleRouter.exactInputSingle({pool: pool, ...})
    → router calls pool.swap(recipient=bob, ...)          [MetricOmmPool.sol L230]
    → pool calls _beforeSwap(sender=router, ...)
    → SwapAllowlistExtension.beforeSwap(sender=router, ...)
    → checks allowedSwapper[pool][router] == true  ✓      [SwapAllowlistExtension.sol L37]
    → swap proceeds — bob receives output tokens

Result:
  bob, explicitly excluded from the allowlist, executes a swap against the
  restricted pool. The allowlist guard is completely bypassed.
```
