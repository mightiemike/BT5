### Title
`max_l2_gas_amount` Admission Check Missing for `Declare` Transactions in `StatelessTransactionValidator` — (`crates/apollo_gateway/src/stateless_transaction_validator.rs`)

### Summary

The gateway's `StatelessTransactionValidator::validate_resource_bounds` enforces an upper-bound cap (`max_l2_gas_amount`) on `l2_gas.max_amount` for `Invoke` and `DeployAccount` transactions, but explicitly skips this check for `Declare` transactions. A `Declare` transaction with an arbitrarily large `l2_gas.max_amount` (up to `u64::MAX`) passes gateway admission unchanged, bypassing the DoS-protection invariant the cap is meant to enforce.

### Finding Description

In `validate_resource_bounds`, the check reads:

```rust
// TODO(Arni): Consider adding a validation for max_l2_gas_amount for declare.
if let RpcTransaction::Declare(_) = tx {
} else if resource_bounds.l2_gas.max_amount.0 > self.config.max_l2_gas_amount {
    return Err(StatelessTransactionValidatorError::MaxGasAmountTooHigh { … });
}
```

The `if let RpcTransaction::Declare(_) = tx { }` branch is an intentional no-op that causes the entire `max_l2_gas_amount` guard to be skipped for every `Declare` transaction. The production config sets `max_l2_gas_amount = 1_210_000_000`. A `Declare` transaction with `l2_gas.max_amount = u64::MAX` passes this validator without error.

The test `valid_l2_gas_amount_on_declare` explicitly confirms the bypass: a `Declare` with `max_amount: GasAmount(200)` passes when `max_l2_gas_amount: 100` is configured. [1](#0-0)

### Impact Explanation

`l2_gas.max_amount` is used as `initial_sierra_gas` for execution inside the blockifier. A `Declare` transaction with `l2_gas.max_amount = u64::MAX` is admitted to the mempool and forwarded to the batcher. While the blockifier's `validate_max_sierra_gas` constant caps actual execution steps, the gateway's admission invariant — that no transaction may claim more than `max_l2_gas_amount` of L2 gas — is violated for the entire `Declare` type. An attacker can flood the mempool with `Declare` transactions carrying unbounded gas claims, bypassing the rate-limiting and resource-accounting that `max_l2_gas_amount` is designed to provide.

**Impact: High. Mempool/gateway admission accepts invalid transactions before sequencing.**

### Likelihood Explanation

Any unprivileged user can submit a `Declare` transaction via the public RPC endpoint. Constructing a `Declare` with `l2_gas.max_amount = u64::MAX` requires no special access. The bypass is unconditional — it applies regardless of any other field values.

### Recommendation

Remove the `Declare` exemption and apply the same `max_l2_gas_amount` upper-bound check uniformly across all three transaction types:

```rust
if resource_bounds.l2_gas.max_amount.0 > self.config.max_l2_gas_amount {
    return Err(StatelessTransactionValidatorError::MaxGasAmountTooHigh {
        gas_amount: resource_bounds.l2_gas.max_amount,
        max_gas_amount: self.config.max_l2_gas_amount,
    });
}
```

If `Declare` transactions legitimately require a higher gas ceiling (e.g., for large Sierra programs), introduce a separate `max_l2_gas_amount_declare` config field rather than removing the check entirely.

### Proof of Concept

The existing test already demonstrates the bypass. Extending it to show the full admission path:

```rust
#[test]
fn test_declare_bypasses_max_l2_gas_amount() {
    let config = StatelessTransactionValidatorConfig {
        validate_resource_bounds: true,
        max_l2_gas_amount: 100,  // production: 1_210_000_000
        ..*DEFAULT_VALIDATOR_CONFIG_FOR_TESTING
    };
    let tx_validator = StatelessTransactionValidator { config };

    // Declare with max_amount = u64::MAX — should be rejected, but is accepted.
    let rpc_tx_args = RpcTransactionArgs {
        resource_bounds: AllResourceBounds {
            l2_gas: ResourceBounds {
                max_amount: GasAmount(u64::MAX),
                max_price_per_unit: GasPrice(DEFAULT_VALIDATOR_CONFIG.min_gas_price),
            },
            ..Default::default()
        },
        ..Default::default()
    };
    let tx = rpc_tx_for_testing(TransactionType::Declare, rpc_tx_args);

    // Passes — the max_l2_gas_amount guard is skipped for Declare.
    assert_matches!(tx_validator.validate(&tx), Ok(()));

    // Contrast: the same bounds on Invoke are correctly rejected.
    let invoke_tx = rpc_tx_for_testing(TransactionType::Invoke, rpc_tx_args);
    assert_matches!(
        tx_validator.validate(&invoke_tx),
        Err(StatelessTransactionValidatorError::MaxGasAmountTooHigh { .. })
    );
}
