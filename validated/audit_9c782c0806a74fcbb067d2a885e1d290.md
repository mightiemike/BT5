### Title
`ValidResourceBounds` Protobuf Round-Trip Silently Changes Variant, Producing Wrong Transaction Hash for V3 Transactions with Zero L2/Data Gas Bounds - (File: `crates/apollo_protobuf/src/converters/transaction.rs`)

### Summary

The `get_tip_resource_bounds_hash` function in `crates/starknet_api/src/transaction_hash.rs` produces **structurally different hash preimages** depending on whether `ValidResourceBounds::L1Gas` or `ValidResourceBounds::AllResources` is used, even when the numeric values are identical. The protobuf deserializer in `crates/apollo_protobuf/src/converters/transaction.rs` silently converts `AllResources { l1_gas: X, l2_gas: 0, l1_data_gas: 0 }` to `L1Gas(X)` based solely on zero-value checks. This means a V3 transaction that was hashed under the `AllResources` path (3 resource felts) will be recomputed under the `L1Gas` path (2 resource felts) after a protobuf round-trip, producing a different transaction hash.

### Finding Description

**Step 1 – Two variants, two different hash preimages.**

`get_tip_resource_bounds_hash` in `crates/starknet_api/src/transaction_hash.rs` branches on the variant:

```rust
let mut resource_felts = vec![
    get_concat_resource(&l1_resource_bounds, L1_GAS)?,
    get_concat_resource(&l2_resource_bounds, L2_GAS)?,
];
resource_felts.extend(match resource_bounds {
    ValidResourceBounds::L1Gas(_) => vec![],                          // 2 felts total
    ValidResourceBounds::AllResources(all_resources) => {
        vec![get_concat_resource(&all_resources.l1_data_gas, L1_DATA_GAS)?]  // 3 felts total
    }
});
```

`L1Gas(X)` → Poseidon over `[tip, l1_felt, l2_felt]`
`AllResources(X, 0, 0)` → Poseidon over `[tip, l1_felt, l2_felt, l1_data_felt]`

These are **different field-element sequences** and produce **different Poseidon hashes**, even when all numeric values are identical. [1](#0-0) 

**Step 2 – The protobuf serializer always emits all three fields.**

When `ValidResourceBounds::L1Gas` is serialized to protobuf, the serializer explicitly sets `l1_data_gas` to the zero default:

```rust
ValidResourceBounds::L1Gas(l1_gas) => protobuf::ResourceBounds {
    l1_gas: Some(l1_gas.into()),
    l2_gas: Some(value.get_l2_bounds().into()),
    l1_data_gas: Some(ResourceBounds::default().into()),  // zero
},
``` [2](#0-1) 

**Step 3 – The protobuf deserializer silently collapses `AllResources` to `L1Gas`.**

When a V3 transaction with `AllResources { l1_gas: X, l2_gas: 0, l1_data_gas: 0 }` is deserialized from protobuf, the deserializer checks the zero condition and silently changes the variant:

```rust
let l1_data_gas = value.l1_data_gas.unwrap_or_default();
// ...
Ok(if l1_data_gas.is_zero() && l2_gas.is_zero() {
    ValidResourceBounds::L1Gas(l1_gas)   // ← variant changed
} else {
    ValidResourceBounds::AllResources(AllResourceBounds { l1_gas, l2_gas, l1_data_gas })
})
``` [3](#0-2) 

**Step 4 – The hash is recomputed from the deserialized `ValidResourceBounds`.**

The `Transaction::Invoke(InvokeTransaction::V3)` type stores `resource_bounds: ValidResourceBounds`. After protobuf deserialization, `validate_transaction_hash` recomputes the hash from this field: [4](#0-3) 

The `InvokeTransactionV3Trait` implementation for `InvokeTransactionV3` returns `self.resource_bounds` directly, so the deserialized `L1Gas` variant flows into `get_tip_resource_bounds_hash` and produces hash H2 ≠ H1. [5](#0-4) 

**Step 5 – The gateway/RPC path always uses `AllResources`.**

`InternalRpcInvokeTransactionV3` stores `resource_bounds: AllResourceBounds` and its trait implementation always wraps it in `ValidResourceBounds::AllResources`:

```rust
fn resource_bounds(&self) -> ValidResourceBounds {
    ValidResourceBounds::AllResources(self.resource_bounds)
}
``` [6](#0-5) 

So the proposer always computes H1 (3-felt path). The validator, after protobuf deserialization, computes H2 (2-felt path). H1 ≠ H2.

### Impact Explanation

A V3 transaction with `l2_gas = 0` and `l1_data_gas = 0` (a valid transaction type used before Starknet 0.13.3) is accepted and hashed by the proposer using the `AllResources` path (H1). When this transaction is propagated via P2P protobuf and the receiving node recomputes the hash, it uses the `L1Gas` path (H2). Since H1 ≠ H2, `validate_transaction_hash` returns false. The receiving node rejects the transaction or the block containing it as having an invalid hash. This causes valid blocks proposed by the sequencer to be rejected by validators, breaking consensus for any block containing such transactions.

### Likelihood Explanation

V3 transactions with zero L2 gas and zero L1 data gas are valid and occur in practice (they are the standard form for pre-0.13.3 V3 transactions). Any such transaction propagated via P2P sync triggers the mismatch. The trigger requires no special privileges — any user can submit a V3 transaction with zero L2/data gas bounds.

### Recommendation

The protobuf deserializer must not use zero-value checks to select the `ValidResourceBounds` variant. Instead, the variant should be encoded explicitly in the protobuf schema (e.g., a `oneof` or a version tag), or the deserializer should always produce `AllResources` for V3 transactions and reserve `L1Gas` only for explicitly versioned pre-0.13.3 transactions. Alternatively, `get_tip_resource_bounds_hash` should be made variant-agnostic by always including the `l1_data_gas` felt (using zero when absent), so that `L1Gas(X)` and `AllResources(X, 0, 0)` produce the same hash.

### Proof of Concept

```
1. Construct a V3 InvokeTransaction with:
     resource_bounds = AllResourceBounds { l1_gas: X, l2_gas: 0, l1_data_gas: 0 }

2. Compute H1 = get_tip_resource_bounds_hash(AllResources(X, 0, 0), tip)
   → Poseidon([tip, l1_felt, l2_felt, l1_data_felt])   // 3 resource felts

3. Serialize to protobuf::ResourceBounds:
   { l1_gas: X, l2_gas: 0, l1_data_gas: 0 }  // all three fields present, l2 and l1_data are zero

4. Deserialize via TryFrom<protobuf::ResourceBounds> for ValidResourceBounds:
   l1_data_gas.is_zero() && l2_gas.is_zero() → true
   → ValidResourceBounds::L1Gas(X)            // variant silently changed

5. Compute H2 = get_tip_resource_bounds_hash(L1Gas(X), tip)
   → Poseidon([tip, l1_felt, l2_felt])         // only 2 resource felts

6. Assert H1 ≠ H2  ← hash mismatch confirmed

7. validate_transaction_hash(tx_with_L1Gas_variant, block_number, chain_id, H1, options)
   → recomputes H2 ≠ H1 → returns false → block/transaction rejected
```

pub fn get_tip_resource_bounds_hash(
    resource_bounds: &ValidResourceBounds,
    tip: &Tip,
) -> Result<Felt, StarknetApiError> {
    let l1_resource_bounds = resource_bounds.get_l1_bounds();
    let l2_resource_bounds = resource_bounds.get_l2_bounds();

    // L1 and L2 gas bounds always exist.
    // Old V3 txs always have L2 gas bounds of zero, but they exist.
    let mut resource_felts = vec![
        get_concat_resource(&l1_resource_bounds, L1_GAS)?,
        get_concat_resource(&l2_resource_bounds, L2_GAS)?,
    ];

    // For new V3 txs, need to also hash the data gas bounds.
    resource_felts.extend(match resource_bounds {
        ValidResourceBounds::L1Gas(_) => vec![],
        ValidResourceBounds::AllResources(all_resources) => {
            vec![get_concat_resource(&all_resources.l1_data_gas, L1_DATA_GAS)?]
        }
    });

    Ok(HashChain::new().chain(&tip.0.into()).chain_iter(resource_felts.iter()).get_poseidon_hash())
}

impl TryFrom<protobuf::ResourceBounds> for ValidResourceBounds {
    type Error = ProtobufConversionError;
    fn try_from(value: protobuf::ResourceBounds) -> Result<Self, Self::Error> {
        let Some(l1_gas) = value.l1_gas else {
            return Err(missing("ResourceBounds::l1_gas"));
        };
        let Some(l2_gas) = value.l2_gas else {
            return Err(missing("ResourceBounds::l2_gas"));
        };
        // TODO(Shahak): Assert data gas is not none once we remove support for 0.13.2.
        let l1_data_gas = value.l1_data_gas.unwrap_or_default();
        let l1_gas: ResourceBounds = l1_gas.try_into()?;
        let l2_gas: ResourceBounds = l2_gas.try_into()?;
        let l1_data_gas: ResourceBounds = l1_data_gas.try_into()?;
        Ok(if l1_data_gas.is_zero() && l2_gas.is_zero() {
            ValidResourceBounds::L1Gas(l1_gas)
        } else {
            ValidResourceBounds::AllResources(AllResourceBounds { l1_gas, l2_gas, l1_data_gas })
        })
    }
