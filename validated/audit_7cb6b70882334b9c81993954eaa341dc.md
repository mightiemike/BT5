### Title
Unauthenticated `ContractCodeResponse::V1` Accepted Without Signature Verification Before `SignedContractCodeResponse` Activates ŌĆö (`File: chain/client/src/stateless_validation/validate.rs`)

### Summary

`validate_contract_code_response()` conditionally skips chunk-producer signature verification based on `ProtocolFeature::SignedContractCodeResponse`. Before that feature activates, any network peer can inject a `ContractCodeResponse::V1` carrying arbitrary contract bytes for any relevant chunk. The response passes validation, is stored, and causes the receiving chunk validator's state-witness validation to fail ŌĆö preventing it from endorsing the chunk.

### Finding Description

`ContractCodeResponse` is a versioned enum with two variants:

- `V1` ŌĆö no signature field, `responder()` returns `None`, `verify_signature()` always returns `false`
- `V2` ŌĆö carries a chunk-producer signature over a Borsh-serialized inner payload [1](#0-0) 

The receiver-side validation function is:

```rust
pub fn validate_contract_code_response(...) -> Result<ChunkRelevance, Error> {
    let key = response.chunk_production_key();
    require_relevant!(validate_chunk_relevant(epoch_manager, key, store)?);
    let protocol_version = epoch_manager.get_epoch_protocol_version(&key.epoch_id)?;
    if ProtocolFeature::SignedContractCodeResponse.enabled(protocol_version) {
        validate_witness_contract_code_response_signature(epoch_manager, response)?;
    }
    Ok(ChunkRelevance::Relevant)
}
``` [2](#0-1) 

Before `SignedContractCodeResponse` activates, the only gate is `validate_chunk_relevant` ŌĆö a height/shard/epoch range check that any peer can satisfy by targeting an in-flight chunk. No sender identity or signature is verified. The accepted response is immediately stored:

```rust
fn handle_contract_code_response(&self, response: ContractCodeResponse) -> Result<(), Error> {
    if !validate_contract_code_response(...)?.is_relevant() { return Ok(()); }
    let key = response.chunk_production_key().clone();
    let contracts = response.decompress_contracts()?;
    self.partial_witness_tracker.store_accessed_contract_codes(key, contracts)
}
``` [3](#0-2) 

The stored bytes are later used directly for state-witness validation. If the bytes do not match the contracts actually executed in the chunk, the witness validation fails and the validator does not emit a chunk endorsement.

The `ContractCodeRequest` path ŌĆö which is always signature-verified ŌĆö provides no protection here because the response path is separate and unauthenticated pre-feature. [4](#0-3) 

The exact divergent Borsh bytes are the `compressed_contracts` field of `ContractCodeResponseV1`: an attacker supplies an arbitrary valid zstd-compressed payload whose decompressed contract code does not match the code hashes in `ChunkContractAccesses`. [5](#0-4) 

### Impact Explanation

A network peer that races the legitimate chunk producer's response can cause a chunk validator to store wrong contract code. The validator then fails state-witness validation for that chunk and withholds its endorsement. If enough validators are targeted simultaneously, the chunk fails to accumulate the required endorsement quorum, stalling block production for the affected shard. This is a High-severity liveness attack requiring no privileged role.

### Likelihood Explanation

`SignedContractCodeResponse` is a nightly/unreleased feature (it appears after the stable feature list in `version.rs`, which caps at protocol version 86). [6](#0-5) [7](#0-6) 

On mainnet (stable protocol version 86), the feature is not yet active, so the unauthenticated path is the live production path. An attacker needs only to be a network peer of a chunk validator and to deliver a fake `ContractCodeResponse::V1` before the legitimate chunk producer's response arrives ŌĆö a race that is feasible given P2P routing latency.

### Recommendation

1. **Short-term**: In `validate_contract_code_response`, reject `V1` responses whose `responder()` is `None` even before `SignedContractCodeResponse` activates, or add a sender-identity check using the epoch's chunk-producer set.
2. **Long-term**: Accelerate stabilization of `SignedContractCodeResponse` so that V2 (signed) responses are required on mainnet. Once stable, remove the `V1` variant from the Borsh enum or stop accepting it.
3. **Defensive**: In `handle_contract_code_response`, verify that each decompressed contract's `sha256` hash matches a hash from the corresponding `ChunkContractAccesses` message before storing it, providing a content-integrity check independent of sender authentication.

### Proof of Concept

1. Observe a `ContractCodeRequest` on the P2P network targeting chunk validator `V` for chunk `C` at height `H`, shard `S`.
2. Construct `ContractCodeResponse::V1 { next_chunk: ChunkProductionKey { shard_id: S, epoch_id: E, height_created: H }, compressed_contracts: <zstd-compressed garbage bytes> }`.
3. Send this message to `V` before the legitimate chunk producer responds.
4. `validate_contract_code_response` passes: `validate_chunk_relevant` succeeds (chunk is in range), `SignedContractCodeResponse` is not enabled, so no signature check is performed.
5. `decompress_contracts` succeeds (the payload is valid zstd).
6. `store_accessed_contract_codes` stores the garbage code under key `C`.
7. When `V` assembles the state witness for `C`, it uses the garbage code; execution results diverge from the witness; validation fails; `V` does not endorse `C`. [2](#0-1) [8](#0-7) [3](#0-2)

pub enum ContractCodeResponse {
    V1(ContractCodeResponseV1) = 0,
    V2(ContractCodeResponseV2) = 1,
}

impl ContractCodeResponse {
    pub fn encode(
        next_chunk: ChunkProductionKey,
        contracts: &Vec<CodeBytes>,
        signer: &ValidatorSigner,
        protocol_version: ProtocolVersion,
    ) -> std::io::Result<Self> {
        if ProtocolFeature::SignedContractCodeResponse.enabled(protocol_version) {
            ContractCodeResponseV2::encode(next_chunk, contracts, signer).map(Self::V2)
        } else {
            ContractCodeResponseV1::encode(next_chunk, contracts).map(Self::V1)
        }
    }

    pub fn chunk_production_key(&self) -> &ChunkProductionKey {
        match self {
            Self::V1(v1) => &v1.next_chunk,
            Self::V2(v2) => &v2.inner.next_chunk,
        }
    }

    /// Account that produced this response. Available only for signed variants.
    pub fn responder(&self) -> Option<&AccountId> {
        match self {
            Self::V1(_) => None,
            Self::V2(v2) => Some(&v2.inner.responder),
        }
    }

    pub fn decompress_contracts(&self) -> std::io::Result<Vec<CodeBytes>> {
        let compressed_contracts = match self {
            Self::V1(v1) => &v1.compressed_contracts,
            Self::V2(v2) => &v2.inner.compressed_contracts,
        };
        compressed_contracts.decode().map(|(data, _size)| data)
    }

    /// Verifies the signature for signed variants. Returns `false` for
    /// unsigned variants since there is nothing to verify.
    pub fn verify_signature(&self, public_key: &PublicKey) -> bool {
        match self {
            Self::V1(_) => false,
            Self::V2(v2) => v2.verify_signature(public_key),
        }
    }
}
