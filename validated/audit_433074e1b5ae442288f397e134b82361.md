### Title
AuxPoW Flag Not Checked When `aux_data` Is Absent, Allowing PoW Bypass on Dogecoin Blocks — (File: `contract/src/dogecoin.rs`)

---

### Summary

In `dogecoin.rs`, `submit_block_header` accepts `(Header, Option<AuxData>)`. When `aux_data` is `None`, the function falls back to checking the Dogecoin block's own hash against the target. However, it never verifies that the block's version field does **not** have the AuxPoW flag set (`version & BLOCK_VERSION_AUXPOW != 0`). A block that declares itself as an AuxPoW block (via the version flag) but is submitted without `aux_data` bypasses all AuxPoW chain validation and is instead evaluated only on its own raw hash — which is the wrong PoW witness for an AuxPoW block.

---

### Finding Description

In `submit_block_header` for Dogecoin:

```rust
if let Some(ref aux_data) = aux_data {
    self.check_aux(&block_header, aux_data);
} else {
    let pow_hash = block_header.block_hash_pow();
    require!(
        U256::from_le_bytes(&pow_hash.0) <= target_from_bits(block_header.bits),
        format!("block should have correct pow")
    );
}
``` [1](#0-0) 

The `else` branch is intended for pre-AuxPoW Dogecoin blocks, where the block's own hash is the PoW witness. But there is no guard requiring `block_header.version & BLOCK_VERSION_AUXPOW == 0` before entering this branch.

Contrast this with `check_aux`, which explicitly enforces the flag in the opposite direction:

```rust
require!(
    block_header.version & BLOCK_VERSION_AUXPOW != 0,
    "Aux POW block does not have AuxPoW flag set in version"
);
``` [2](#0-1) 

For a genuine AuxPoW block, the block's own hash is **not** the PoW witness — the parent Bitcoin block's SHA256d hash is. Checking the Dogecoin block's own hash against the target is therefore semantically wrong for any block with the AuxPoW flag set.

The analog to the original report is direct:

| Original (Solana pool) | This repo (Dogecoin light client) |
|---|---|
| `pool.amount` is always 0 when `shared_escrow` is set | Block's own hash is not the PoW witness when AuxPoW flag is set |
| Check `pool.amount < price` always passes → pool closed | Check `block_hash_pow() <= target` can be made to pass → fake block accepted |
| Missing guard: `shared_escrow == pubkey::default` | Missing guard: `version & BLOCK_VERSION_AUXPOW == 0` |

---

### Impact Explanation

An attacker acting as a registered relayer can craft a Dogecoin block with:
- `version & BLOCK_VERSION_AUXPOW != 0` (AuxPoW flag set, signalling merge-mined block)
- `aux_data = None` (no AuxPoW chain proof provided)
- A block body whose own hash (SHA256d, if the `scrypt_hash` feature is not compiled in for the Dogecoin build) meets the Dogecoin difficulty target

The contract accepts this block without validating any AuxPoW chain linkage, coinbase commitment, or parent block PoW. The accepted block is stored in `headers_pool` and can become part of the canonical chain via the chainwork comparison in `submit_block_header_inner`. [3](#0-2) 

Once a fake block is on the canonical chain, `verify_transaction_inclusion` and `verify_transaction_inclusion_v2` will accept SPV proofs anchored to it, allowing false transaction inclusion proofs to be certified on-chain. [4](#0-3) 

---

### Likelihood Explanation

- The entry path is `submit_blocks` → `submit_block_header` (Dogecoin build). The `#[trusted_relayer]` gate requires the caller to be a registered relayer, but registration is managed by `RelayerManager`/`DAO` roles and does not inherently prevent a malicious actor from obtaining relayer status.
- The computational cost of the attack depends on whether the `scrypt_hash` feature is compiled into the Dogecoin WASM. If it is not (plausible, since AuxPoW blocks are validated via the parent block's SHA256d hash in `check_aux`), `block_hash_pow()` falls back to SHA256d, which is orders of magnitude faster than Scrypt and makes meeting the Dogecoin target feasible with commodity hardware.
- Even if `scrypt_hash` is enabled, the missing flag check is still a structural correctness flaw that violates the Dogecoin protocol invariant. [5](#0-4) 

---

### Recommendation

Add an explicit guard in the `else` branch of `submit_block_header` to reject any block that has the AuxPoW flag set but was submitted without `aux_data`:

```rust
} else {
    require!(
        block_header.version & 0x100 == 0,
        "AuxPoW flag is set but no AuxPoW data was provided"
    );
    let pow_hash = block_header.block_hash_pow();
    require!(
        U256::from_le_bytes(&pow_hash.0) <= target_from_bits(block_header.bits),
        format!("block should have correct pow")
    );
}
```

This mirrors the fix described in the original report: check for the alternative configuration (`shared_escrow` / AuxPoW flag) before using the local field (`pool.amount` / block's own hash). [6](#0-5)

### Proof of Concept

1. Deploy the Dogecoin build of the contract and register a relayer account.
2. Craft a `Header` with `version = 0x00620100` (chain ID 0x62, AuxPoW flag 0x100 set).
3. Set `prev_block_hash` to the current tip's hash, set `bits` to the current expected target, and set `time` to a valid timestamp.
4. Mine the block's SHA256d hash (via `block_hash()` / `block_hash_pow()`) until it is below `target_from_bits(bits)`.
5. Call `submit_blocks` with `(crafted_header, None)` as the `BlockHeader` tuple.
6. The contract calls `check_target` (passes — `bits` is correct), then enters the `else` branch, checks the SHA256d hash against the target (passes — mined to satisfy it), and stores the block in `headers_pool` as a valid mainchain block.
7. Call `verify_transaction_inclusion` with a Merkle proof anchored to the fake block's `merkle_root` — the contract returns `true` for a transaction that never existed on the real Dogecoin chain. [7](#0-6) [8](#0-7)

    pub(crate) fn submit_block_header(
        &mut self,
        header: (Header, Option<AuxData>),
        skip_pow_verification: bool,
    ) {
        let (block_header, aux_data) = header;

        let prev_block_header = self.get_prev_header(&block_header);
        let current_block_hash = block_header.block_hash();

        if !skip_pow_verification {
            self.check_target(&block_header, &prev_block_header);

            if let Some(ref aux_data) = aux_data {
                self.check_aux(&block_header, aux_data);
            } else {
                let pow_hash = block_header.block_hash_pow();
                // Check if the block hash is less than or equal to the target
                require!(
                    U256::from_le_bytes(&pow_hash.0) <= target_from_bits(block_header.bits),
                    format!("block should have correct pow")
                );
            }
        }

        let (current_block_computed_chain_work, overflow) = prev_block_header
            .chain_work
            .overflowing_add(work_from_bits(block_header.bits));
        require!(!overflow, "Addition of U256 values overflowed");

        let current_header = ExtendedHeader {
            block_header: block_header.clone().into_light(),
            block_hash: current_block_hash,
            chain_work: current_block_computed_chain_work,
            block_height: 1 + prev_block_header.block_height,
        };

        self.submit_block_header_inner(current_header, &prev_block_header);
    }

    pub fn verify_transaction_inclusion(&self, #[serializer(borsh)] args: ProofArgs) -> bool {
        require!(
            args.confirmations <= self.gc_threshold,
            "The required number of confirmations exceeds the number of blocks stored in memory"
        );

        let heaviest_block_header = self
            .headers_pool
            .get(&self.mainchain_tip_blockhash)
            .unwrap_or_else(|| env::panic_str(ERR_KEY_NOT_EXIST));
        let target_block_height = self
            .mainchain_header_to_height
            .get(&args.tx_block_blockhash)
            .unwrap_or_else(|| env::panic_str("block does not belong to the current main chain"));

        // Check requested confirmations. No need to compute proof if insufficient confirmations.
        require!(
            (heaviest_block_header.block_height).saturating_sub(target_block_height) + 1
                >= args.confirmations,
            "Not enough blocks confirmed"
        );

        let header = self
            .headers_pool
            .get(&args.tx_block_blockhash)
            .unwrap_or_else(|| env::panic_str("cannot find requested transaction block"));

        require!(!args.merkle_proof.is_empty(), "Merkle proof is empty");

        // compute merkle tree root and check if it matches block's original merkle tree root
        merkle_tools::compute_root_from_merkle_proof(
            args.tx_id,
            usize::try_from(args.tx_index).unwrap(),
            &args.merkle_proof,
        ) == header.block_header.merkle_root
    }

    fn submit_block_header_inner(
        &mut self,
        current_header: ExtendedHeader,
        prev_block_header: &ExtendedHeader,
    ) {
        // Main chain submission
        if prev_block_header.block_hash == self.mainchain_tip_blockhash {
            // Probably we should check if it is not in a mainchain?
            // chainwork > highScore
            log!("Block {}: saving to mainchain", current_header.block_hash);
            // Validate chain
            assert_eq!(
                self.mainchain_tip_blockhash,
                current_header.block_header.prev_block_hash
            );

            self.store_block_header(&current_header);
            self.mainchain_tip_blockhash = current_header.block_hash;
        } else {
            log!("Block {}: saving to fork", current_header.block_hash);
            // Fork submission
            let main_chain_tip_header = self
                .headers_pool
                .get(&self.mainchain_tip_blockhash)
                .unwrap_or_else(|| env::panic_str("tip should be in a header pool"));

            let last_main_chain_block_height = main_chain_tip_header.block_height;
            let total_main_chain_chainwork = main_chain_tip_header.chain_work;

            self.store_fork_header(&current_header);

            // Current chainwork is higher than on a current mainchain, let's promote the fork
            if current_header.chain_work > total_main_chain_chainwork {
                log!("Chain reorg");
                self.reorg_chain(current_header, last_main_chain_block_height);
            }
        }
    }

    pub fn block_hash_pow(&self) -> H256 {
        let block_header = self.get_block_header_vec();
        #[cfg(feature = "scrypt_hash")]
        {
            let params = scrypt::Params::new(10, 1, 1, 32).unwrap(); // N=1024 (2^10), r=1, p=1

            let mut output = [0u8; 32];
            scrypt::scrypt(&block_header, &block_header, &params, &mut output).unwrap();
            H256::from(output)
        }

        #[cfg(not(feature = "scrypt_hash"))]
        {
            double_sha256(&block_header)
        }
    }
