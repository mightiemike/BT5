### Title
ML-DSA-65 Gas-Key Host-Function Exec Fee Uses Wire Length Instead of Trie-ID Length — (File: `runtime/near-vm-runner/src/logic/logic.rs`)

---

### Summary

The `promise_batch_action_transfer_to_gas_key`, `promise_batch_action_add_gas_key_with_full_access`, and `promise_batch_action_add_gas_key_with_function_call` host functions compute exec fees using the raw caller-supplied wire length (`public_key_len as usize`), while the transaction path correctly uses `public_key.trie_id_len()`. For ML-DSA-65 keys the two values diverge by a factor of ~59 (1953 wire bytes vs 33 trie-id bytes), breaking the documented fee-parity invariant between the transaction and host-function paths.

---

### Finding Description

The codebase explicitly documents that every storage-stake and trie-byte-priced fee path must call `trie_id_len()` rather than `len()` for ML-DSA-65 keys, because the trie stores a 33-byte SHA3-256 hash, not the 1953-byte borsh-encoded full pubkey.

**Transaction path — correct:**

`runtime/runtime/src/config.rs` lines 347–354 call `gas_key_transfer_exec_fee` with `action.public_key.trie_id_len()`:

```rust
TransferToGasKey(action) => {
    gas_key_transfer_exec_fee(fees, receiver_id.len(), action.public_key.trie_id_len())
        .total()
}
``` [1](#0-0) 

**Host-function path — incorrect:**

`runtime/near-vm-runner/src/logic/logic.rs` lines 3091–3096 pass the raw caller-supplied `public_key_len as usize` to both `gas_key_transfer_send_fee` and `gas_key_transfer_exec_fee`:

```rust
let send = gas_key_transfer_send_fee(&self.fees_config, sir, public_key_len as usize);
let exec = gas_key_transfer_exec_fee(
    &self.fees_config,
    receiver_id.len(),
    public_key_len as usize,   // ← wire length, not trie_id_len
);
``` [2](#0-1) 

The same pattern appears in `promise_batch_action_add_gas_key_with_full_access` (lines 3155–3160) and `promise_batch_action_add_gas_key_with_function_call` (lines 3226–3231): [3](#0-2) [4](#0-3) 

The wasmtime runner duplicates the same bug at lines 3323–3325 and 3499–3503: [5](#0-4) 

**Root cause — the divergence:**

`PublicKey::len()` returns 1953 for ML-DSA-65 (1 tag + 1952 raw key bytes). `PublicKey::trie_id_len()` returns 33 (1 tag + 32 SHA3-256 hash bytes). The two are equal for ed25519 (33) and secp256k1 (65), so the bug is invisible for those schemes. [6](#0-5) 

`gas_key_transfer_exec_fee` passes `public_key_len` directly into `access_key_key_len`, which is supposed to model the actual trie key length: [7](#0-6) 

For ML-DSA-65 the trie key is 33 bytes; the host function feeds 1953 bytes, producing an exec-fee overcharge of `(1953 − 33) × gas_key_byte_exec_fee × (1 + num_nonces)` per operation.

The documentation explicitly flags this as a contract that all fee paths must honour:

> "`PublicKey::trie_id_len()` is a new contract that all storage-cost code must respect. Callers that still use `len()` for trie-storage costing will misprice ML-DSA-65 keys by ~1900 bytes." [8](#0-7) 

The fee-parity integration test (`test_gas_key_fee_parity`) only exercises ED25519 keys, so the ML-DSA-65 divergence is not caught: [9](#0-8) 

---

### Impact Explanation

When both `PostQuantumSignatures` and `GasKeys` protocol features are active, any contract that calls `promise_batch_action_transfer_to_gas_key`, `promise_batch_action_add_gas_key_with_full_access`, or `promise_batch_action_add_gas_key_with_function_call` with an ML-DSA-65 public key will be charged ~59× the correct exec gas. This:

1. Breaks the documented fee-parity invariant between the transaction and host-function paths.
2. Can cause unexpected `GasExceeded` aborts for contracts that budget gas based on the transaction-path fee schedule.
3. Makes ML-DSA-65 gas keys economically unviable via the host-function path, silently degrading the feature for on-chain contract use.

---

### Likelihood Explanation

Requires both `PostQuantumSignatures` (protocol version 85) and `GasKeys` to be simultaneously active, and a contract that uses ML-DSA-65 keys with the gas-key host functions. The combination is reachable on mainnet once both features stabilise. The bug is latent today and will surface as soon as a contract exercises the path.

---

### Recommendation

Replace `public_key_len as usize` with `public_key.trie_id_len()` (after decoding the key) in all three host-function sites in both `logic.rs` and `wasmtime_runner/logic.rs`. The decoded `PublicKey` is already available at those call sites (as `public_key.decode()?` is called immediately after the fee computation). Extend `test_gas_key_fee_parity` to cover `KeyType::MLDSA65` to prevent regression.

---

### Proof of Concept

For an ML-DSA-65 gas key on account `"alice.near"` (10 bytes):

| Path | `public_key_len` fed to `gas_key_transfer_exec_fee` | `access_key_key_len` result | Exec-fee multiplier |
|---|---|---|---|
| Transaction (`exec_fee`) | `trie_id_len()` = 33 | 2 + 10 + 33 = 45 | 45 + value_len |
| Host function | wire `len()` = 1953 | 2 + 10 + 1953 = 1965 | 1965 + value_len |

The exec fee for `TransferToGasKey` via host function is `(1965 + value_len) / (45 + value_len)` ≈ **43× higher** than via transaction for a typical `AccessKey` value. For `AddKey` with `num_nonces` nonces the overcharge compounds per nonce.

impl PublicKey {
    /// Length of this public key's borsh encoding, in bytes - that is,
    /// the on-the-wire size of the raw key bytes plus a 1-byte borsh
    /// discriminant tag (the leading `+ 1` in each arm).
    ///
    /// For storage-fee accounting use [`PublicKey::trie_id_len`] instead;
    /// for ML-DSA-65 those two diverge (1953 wire vs 33 on-trie).
    // `is_empty` always returns false, so there is no point in adding it
    #[allow(clippy::len_without_is_empty)]
    pub fn len(&self) -> usize {
        match self {
            Self::ED25519(_) => 1 + ed25519_dalek::PUBLIC_KEY_LENGTH,
            Self::SECP256K1(_) => 1 + 64,
            Self::MLDSA65(_) => 1 + ML_DSA_65_PUBLIC_KEY_LENGTH,
        }
    }

    pub fn empty(key_type: KeyType) -> Self {
        match key_type {
            KeyType::ED25519 => {
                PublicKey::ED25519(ED25519PublicKey([0u8; ed25519_dalek::PUBLIC_KEY_LENGTH]))
            }
            KeyType::SECP256K1 => PublicKey::SECP256K1(Secp256K1PublicKey([0u8; 64])),
            KeyType::MLDSA65 => {
                PublicKey::MLDSA65(MlDsa65PublicKey(Box::new([0u8; ML_DSA_65_PUBLIC_KEY_LENGTH])))
            }
        }
    }

    pub fn key_type(&self) -> KeyType {
        match self {
            Self::ED25519(_) => KeyType::ED25519,
            Self::SECP256K1(_) => KeyType::SECP256K1,
            Self::MLDSA65(_) => KeyType::MLDSA65,
        }
    }

    fn key_tag(&self) -> KeyTag {
        match self {
            PublicKey::ED25519(_) => KeyTag::Ed25519,
            PublicKey::SECP256K1(_) => KeyTag::Secp256k1,
            PublicKey::MLDSA65(_) => KeyTag::MlDsa65Full,
        }
    }

    pub fn key_data(&self) -> &[u8] {
        match self {
            Self::ED25519(key) => key.as_ref(),
            Self::SECP256K1(key) => key.as_ref(),
            Self::MLDSA65(key) => key.as_ref(),
        }
    }

    pub fn unwrap_as_ed25519(&self) -> &ED25519PublicKey {
        match self {
            Self::ED25519(key) => key,
            Self::SECP256K1(_) | Self::MLDSA65(_) => panic!(),
        }
    }

    pub fn unwrap_as_secp256k1(&self) -> &Secp256K1PublicKey {
        match self {
            Self::SECP256K1(key) => key,
            Self::ED25519(_) | Self::MLDSA65(_) => panic!(),
        }
    }

    /// Length, in bytes, of the on-trie identifier for an access-key
    /// entry owned by this public key. For ed25519/secp256k1 this matches
    /// `len()`; for ML-DSA-65 the trie stores a SHA3-256 hash (33 bytes
    /// including the type tag), not the 1953-byte borsh-encoded pubkey.
    /// Used by storage-fee calculations on the runtime side; cheap to call
    /// (no hashing) - for ML-DSA-65 this returns the size of the digest
    /// form without actually hashing the pubkey.
    pub fn trie_id_len(&self) -> usize {
        match self {
            Self::ED25519(_) => 1 + ed25519_dalek::PUBLIC_KEY_LENGTH,
            Self::SECP256K1(_) => 1 + 64,
            Self::MLDSA65(_) => 1 + ML_DSA_65_HASH_LENGTH,
        }
    }

**File:** core/parameters/src/cost.rs (L833-847)
```rust
pub fn gas_key_transfer_exec_fee(
    cfg: &RuntimeFeesConfig,
    account_id_len: usize,
    public_key_len: usize,
) -> GasKeyTransferFee {
    let base = cfg.fee(ActionCosts::gas_key_transfer_base).exec_fee();
    let trie_key_len = access_key_key_len(account_id_len, public_key_len);
    let estimated_value_len = AccessKey::min_gas_key_borsh_len();
    let per_byte = cfg
        .fee(ActionCosts::gas_key_byte)
        .exec_fee()
        .checked_mul((trie_key_len + estimated_value_len) as u64)
        .unwrap();
    GasKeyTransferFee { base, per_byte }
}
