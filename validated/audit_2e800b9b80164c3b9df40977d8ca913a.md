Audit Report

## Title
`SwapAllowlistExtension` Bypass via Router Address Substitution When Router Is Allowlisted — (File: `metric-periphery/contracts/extensions/SwapAllowlistExtension.sol`)

## Summary
`SwapAllowlistExtension.beforeSwap` checks the `sender` argument, which `MetricOmmPool.swap` always sets to `msg.sender` — the immediate caller of `pool.swap()`. When swaps are routed through `MetricOmmSimpleRouter`, `msg.sender` inside `pool.swap()` is the router contract, not the originating EOA. If the pool admin allowlists the router address to enable router-mediated swaps for legitimate users, every unprivileged user can bypass the allowlist entirely by routing through the same public router.

## Finding Description
**Root cause — three confirmed code facts:**

1. `MetricOmmPool.swap` passes `msg.sender` verbatim as `sender` to `_beforeSwap`: [1](#0-0) 

2. `SwapAllowlistExtension.beforeSwap` gates on `allowedSwapper[msg.sender][sender]` — i.e., `allowedSwapper[pool][router]` when the router is the caller: [2](#0-1) 

3. `MetricOmmSimpleRouter.exactInputSingle` calls `pool.swap()` directly with no mechanism to thread the original EOA through as `sender`: [3](#0-2) 

**Exploit flow:**
- Admin deploys pool with `SwapAllowlistExtension`, allowlists `alice` (KYC'd user).
- Admin also calls `setAllowedToSwap(pool, router, true)` so `alice` can use the standard router UX.
- `allowedSwapper[pool][router] = true` now passes for **any** caller of `exactInputSingle` pointing at that pool.
- Attacker (not allowlisted) calls `router.exactInputSingle(pool=restrictedPool, ...)` → pool sees `msg.sender = router` → extension checks `allowedSwapper[pool][router]` → `true` → swap executes.

**Why existing guards fail:** The extension has no concept of a "transparent forwarder." It cannot distinguish between the router acting on behalf of `alice` versus an arbitrary attacker. The `allowAllSwappers` flag is the only alternative, but it removes the allowlist entirely. [4](#0-3) 

## Impact Explanation
A pool configured with `SwapAllowlistExtension` to restrict swaps to a curated set of counterparties (KYC'd users, specific market makers, whitelisted protocols) is fully bypassed by any unprivileged user who routes through `MetricOmmSimpleRouter`. The attacker executes real swaps against the pool's liquidity, receiving output tokens at oracle-derived prices. LP funds are consumed by unauthorized counterparties, directly violating the pool admin's access control policy and exposing LPs to adversarial flow they explicitly opted out of. This constitutes a broken core pool access-control mechanism with direct fund impact on LPs.

## Likelihood Explanation
`MetricOmmSimpleRouter` is the primary user-facing swap interface. Any pool admin who deploys `SwapAllowlistExtension` and wants allowlisted users to have a normal UX will naturally allowlist the router — this is the only way to make the extension compatible with the router. The bypass requires no special privileges, no flash loans, and no multi-step setup: any EOA can call `exactInputSingle` on the public router pointing at the restricted pool.

## Recommendation
The extension must gate the economically relevant actor, not the immediate caller. Two viable approaches:

1. **Trusted-router registry in the extension**: When `sender` is a recognized router, decode the original EOA from `extensionData` (passed by the router) and check that address instead. Requires the router to forward the original `msg.sender` in `extensionData` and the extension to maintain a trusted-router registry.

2. **Prohibit router use for allowlisted pools**: Document and enforce that `SwapAllowlistExtension` is incompatible with `MetricOmmSimpleRouter`. Do not allowlist the router; allowlisted users must call `pool.swap()` directly or through a custom wrapper that is itself allowlisted and enforces per-user identity checks.

## Proof of Concept
```solidity
// Setup: pool with SwapAllowlistExtension, only alice is allowlisted
swapExtension.setAllowedToSwap(address(pool), alice, true);
// Admin allowlists the router so alice can use it
swapExtension.setAllowedToSwap(address(pool), address(router), true);

// Attacker (not allowlisted) bypasses the gate via the router
vm.startPrank(attacker);
token0.approve(address(router), type(uint256).max);
// Succeeds: allowedSwapper[pool][router] == true
router.exactInputSingle(IMetricOmmSimpleRouter.ExactInputSingleParams({
    pool: address(pool),
    recipient: attacker,
    zeroForOne: true,
    amountIn: 1000e18,
    amountOutMinimum: 0,
    priceLimitX64: 0,
    tokenIn: address(token0),
    deadline: block.timestamp,
    extensionData: ""
}));
// Attacker receives token1 from the restricted pool — allowlist bypassed
```
