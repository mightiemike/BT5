### Title
Fast Withdrawal Fee Truncates to Zero for Low-Decimal Tokens, Enabling Fee-Free Withdrawals — (File: `core/contracts/BaseWithdrawPool.sol`)

---

### Summary

In `BaseWithdrawPool.fastWithdrawalFeeAmount`, the fee computed in 18-decimal precision (`feeX18`) is converted back to native token decimals via plain integer division `feeX18 / int128(multiplier)`. For low-decimal tokens such as USDC (6 decimals), `multiplier = 10**12`. When `feeX18 < 10**12`, the division truncates to zero, allowing fast withdrawal providers to be exploited into providing the withdrawal service for free and causing the protocol to collect no fee.

---

### Finding Description

`BaseWithdrawPool.fastWithdrawalFeeAmount` computes the fast withdrawal fee as follows:

```solidity
function fastWithdrawalFeeAmount(
    IERC20Base token,
    uint32 productId,
    uint128 amount
) public view returns (int128) {
    uint8 decimals = token.decimals();
    require(decimals <= MAX_DECIMALS);
    int256 multiplier = int256(10**(MAX_DECIMALS - uint8(decimals)));
    int128 amountX18 = int128(amount) * int128(multiplier);

    int128 proportionalFeeX18 = FAST_WITHDRAWAL_FEE_RATE.mul(amountX18);
    int128 minFeeX18 = 5 * spotEngine().getConfig(productId).withdrawFeeX18;

    int128 feeX18 = MathHelper.max(proportionalFeeX18, minFeeX18);
    return feeX18 / int128(multiplier);   // ← plain integer division, truncates toward zero
}
``` [1](#0-0) 

For USDC (6 decimals), `MAX_DECIMALS = 18`, so `multiplier = 10**(18-6) = 10**12`. The final line performs plain integer division. Any `feeX18 < 10**12` (i.e., a fee smaller than $0.000001 in 18-decimal representation) truncates to **zero**.

`FAST_WITHDRAWAL_FEE_RATE = 1_000_000_000_000_000` (0.1%) and `SLOW_MODE_FEE = 1000000` ($1 in 6-decimal) are defined in `Constants.sol`. [2](#0-1) 

The caller path is `submitFastWithdrawal → fastWithdrawalFeeAmount`. When `fee = 0` is returned:

```solidity
if (sendTo == msg.sender) {
    require(transferAmount > uint128(fee), "Fee larger than balance");
    transferAmount -= uint128(fee);   // subtracts 0
} else {
    safeTransferFrom(token, msg.sender, uint128(fee));  // transfers 0 from provider
}
fees[productId] += fee;   // adds 0 to protocol fees
handleWithdrawTransfer(token, sendTo, transferAmount);  // full amount sent to recipient
``` [3](#0-2) 

The recipient receives the full `transferAmount`, the protocol collects zero fee, and the fast withdrawal provider (who fronts the liquidity) earns zero compensation.

---

### Impact Explanation

**Impact: Medium.** Fast withdrawal providers front liquidity in exchange for a fee. When the fee truncates to zero, they provide an interest-free, risk-bearing liquidity service with no compensation. The protocol also loses all fee revenue for these withdrawals. Unlike the external report where the bidder loses their entire bid principal, here the provider loses only their fee compensation — but for a service that carries real counterparty and opportunity-cost risk, a zero fee is a meaningful and concrete asset delta.

---

### Likelihood Explanation

**Likelihood: Medium.** USDC (6 decimals) is one of the most commonly used tokens in DeFi. The truncation to zero occurs whenever `feeX18 < multiplier = 10**12`. Concretely:

- For the proportional path: `proportionalFeeX18 = 0.001 × amountX18`. For USDC, `amountX18 = amount × 10**12`. So `proportionalFeeX18 < 10**12` when `amount < 1000` USDC units = $0.001. Any withdrawal under $0.001 USDC triggers this.
- For the minimum fee path: `minFeeX18 = 5 × withdrawFeeX18`. If `withdrawFeeX18` is configured below `2×10**11` (i.e., $0.0000002 in 18-decimal), the minimum fee also truncates to zero regardless of withdrawal size.

Small withdrawal amounts are common in automated or programmatic contexts.

---

### Recommendation

Replace the truncating floor division with ceiling division when converting `feeX18` back to native token decimals, ensuring at least 1 native unit is charged whenever a non-zero fee is computed:

```solidity
int128 nativeFee = feeX18 / int128(multiplier);
if (nativeFee == 0 && feeX18 > 0) {
    nativeFee = 1;
}
return nativeFee;
```

Alternatively, enforce a protocol-level minimum withdrawal amount in `submitFastWithdrawal` to prevent dust withdrawals that produce zero fees.

---

### Proof of Concept

1. Quote token is USDC (6 decimals); `multiplier = 10**12`.
2. A user submits a fast withdrawal of `amount = 500` USDC units ($0.0005).
3. `amountX18 = 500 × 10**12 = 5×10**14`.
4. `proportionalFeeX18 = 0.001 × 5×10**14 = 5×10**11`.
5. Assume `withdrawFeeX18 = 1×10**11`, so `minFeeX18 = 5×10**11`.
6. `feeX18 = max(5×10**11, 5×10**11) = 5×10**11`.
7. `return 5×10**11 / 10**12 = 0` ← fee is zero.
8. Fast withdrawal provider calls `submitFastWithdrawal`, fronts $0.0005 USDC to the recipient, earns zero fee, and the protocol collects nothing. [1](#0-0)
