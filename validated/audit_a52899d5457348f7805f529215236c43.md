Audit Report

## Title
SwapAllowlistExtension Gates on Router Address Instead of End User, Enabling Full Allowlist Bypass — (File: `metric-periphery/contracts/extensions/SwapAllowlistExtension.sol`)

## Summary
`SwapAllowlistExtension.beforeSwap` checks `allowedSwapper[pool][sender]` where `sender` is `msg.sender` of the pool's `swap` call — the router contract — not the originating user. Any pool admin who allowlists `MetricOmmSimpleRouter` to support router-mediated swaps simultaneously opens the gate to every address on the network, completely neutralising the per-user allowlist.

## Finding Description
`MetricOmmPool.swap` captures `msg.sender` and passes it as `sender` to `_beforeSwap`:

```solidity
// MetricOmmPool.sol L230-231
_beforeSwap(
    msg.sender,   // sender = whoever called pool.swap (the router)
    recipient,
    ...
);
```

`ExtensionCalling._beforeSwap` encodes that value and forwards it to the extension unchanged. `SwapAllowlistExtension.beforeSwap` then evaluates:

```solidity
// SwapAllowlistExtension.sol L37
if (!allowAllSwappers[msg.sender] && !allowedSwapper[msg.sender][sender]) {
```

Here `msg.sender` is the pool and `sender` is the router. When `MetricOmmSimpleRouter.exactInputSingle` calls `pool.swap(params.recipient, ...)`, the pool records `sender = address(router)`. The check resolves to `allowedSwapper[pool][router]` — a single boolean shared by every user of the router.

A pool admin who wants router-mediated swaps to work for their curated set of users must add the router to the allowlist. The moment they do, `allowedSwapper[pool][router] == true`, and any unprivileged address can call `MetricOmmSimpleRouter.exactInputSingle` and pass the check. The per-user key (`allowedSwapper[pool][user]`) is structurally unreachable through the router path because the pool never exposes the originating user as `sender`.

## Impact Explanation
A pool configured with `SwapAllowlistExtension` to restrict swaps to a curated set (KYC'd counterparties, whitelisted market makers, protocol-internal actors) loses that restriction entirely for any caller who routes through the public `MetricOmmSimpleRouter`. This constitutes broken core pool functionality: the access-control extension produces the wrong decision (`allowedSwapper[pool][router]` instead of `allowedSwapper[pool][user]`), allowing unrestricted token swaps against a pool whose admin believed access was gated. LP principal and protocol fees are exposed to actors the pool admin explicitly excluded.

## Likelihood Explanation
`MetricOmmSimpleRouter` is a public, permissionless periphery contract. Any pool admin enabling router-based swaps for their allowlisted users must add the router to the allowlist, which simultaneously opens the gate to everyone. No privileged role, flash loan, or multi-step setup is required — a single `exactInputSingle` call suffices. The bypass is deterministic and repeatable on every block.

## Recommendation
The allowlist must be keyed on the end user, not the immediate pool caller. Two complementary fixes:

1. **Pass the originating user through the router.** Have `MetricOmmSimpleRouter` pass `msg.sender` (the human caller) as an explicit `sender` argument to `pool.swap`, and have the pool forward that value — rather than its own `msg.sender` — to `_beforeSwap`. This requires adding a `sender` parameter to the pool's `swap` interface.

2. **Gate on `recipient` as a fallback.** If the pool architecture cannot expose the originating user as `sender`, the extension should check `recipient` (the address receiving output tokens) as the economically relevant identity, since the router always sets `recipient = msg.sender` (the user). The `recipient` argument is already forwarded to `beforeSwap` as the second parameter and is available for this check.

## Proof of Concept
```
Setup:
  Pool P configured with SwapAllowlistExtension E
  Pool admin allowlists only Alice: allowedSwapper[P][Alice] = true
  Pool admin also allowlists the router (required for router swaps): allowedSwapper[P][router] = true

Attack (Bob, not allowlisted):
  Bob calls MetricOmmSimpleRouter.exactInputSingle({pool: P, recipient: Bob, ...})
  Router calls P.swap(recipient=Bob, ...)                    // router is msg.sender to pool
  Pool sets sender = address(router), calls E.beforeSwap(sender=router, recipient=Bob, ...)
  E checks: allowedSwapper[P][router] == true  → passes
  Bob's swap executes against the allowlisted pool
  Pool admin's per-user gate is bypassed
```

The wrong value is `allowedSwapper[pool][router]` at `SwapAllowlistExtension.sol` line 37 — a single boolean that unlocks the gate for the entire user population instead of the intended per-user boolean `allowedSwapper[pool][user]`.
