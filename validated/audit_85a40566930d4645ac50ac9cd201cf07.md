### Title
Swallowed `ErrInsufficientBonusPool` in `slashRedelegationPosition` Persists Partially-Mutated Position Checkpoints, Causing Permanent Bonus Reward Loss - (File: `x/tieredrewards/keeper/slash.go`)

### Summary
In `slashRedelegationPosition`, when `processEventsAndClaimBonus` returns `ErrInsufficientBonusPool`, the error is intentionally swallowed to prevent a chain halt. However, `processEventsAndClaimBonus` receives `&pos` (a pointer) and, per the module's own ADR, advances `pos.LastEventSeq`, `pos.LastBonusAccrual`, and `pos.LastKnownBonded` **before** the pool-balance check. The mutated `pos` is then unconditionally persisted via `setPositionWithState`. The bonus is never paid, but the checkpoints are advanced as if it were, permanently erasing the user's entitlement to those rewards.

### Finding Description
`slashRedelegationPosition` in `x/tieredrewards/keeper/slash.go` calls `processEventsAndClaimBonus(ctx, &pos)` and explicitly swallows `ErrInsufficientBonusPool`:

```go
if _, err := k.processEventsAndClaimBonus(ctx, &pos); err != nil {
    if errors.Is(err, types.ErrInsufficientBonusPool) {
        k.logger(ctx).Error("insufficient bonus pool during redelegation slash", ...)
    } else {
        return err
    }
}
// ... pos is used below regardless
return k.setPositionWithState(ctx, pos, ...)
``` [1](#0-0) 

The module's own ADR (`doc/architecture/adr-006.md`) documents the bonus-accrual flow as:

> → Advance `LastBonusAccrual`, `LastEventSeq`, `LastKnownBonded`  
> → If pool balance < bonus: **fail atomically** (user retries later) [2](#0-1) 

"Fail atomically" is safe in the normal claim path because an unswallowed error causes the Cosmos SDK cached-context to be discarded, rolling back all mutations. In `slashRedelegationPosition`, the error is swallowed, so:

1. `processEventsAndClaimBonus` advances `pos.LastEventSeq`, `pos.LastBonusAccrual`, `pos.LastKnownBonded` in the in-memory `pos` struct (passed by pointer).
2. It also decrements event reference counts in the store (same `ctx`, no branch), potentially garbage-collecting events prematurely.
3. The error is swallowed; no rollback occurs.
4. `setPositionWithState(ctx, pos, ...)` persists the advanced checkpoints.

On the next `MsgClaimTierRewards`, `processEventsAndClaimBonus` starts from the already-advanced `LastEventSeq` and skips all events that were processed (but not paid) during the slash. The bonus for that entire period is permanently lost. [3](#0-2) [4](#0-3) 

### Impact Explanation
A delegator whose position is subject to a redelegation slash at a moment when the bonus pool is insufficient permanently loses all bonus rewards accrued since their last claim. The corrupted invariant is `pos.LastEventSeq` / `pos.LastBonusAccrual`: they advance past events for which no payment was made, so those events can never be replayed for this position. Additionally, prematurely decremented event reference counts may cause events to be garbage-collected before other positions have processed them, corrupting bonus accounting for unrelated positions on the same validator.

### Likelihood Explanation
The trigger requires two concurrent conditions: (1) a validator slash on a redelegation entry that maps to a tiered-rewards position, and (2) an insufficient bonus pool at that block. Both are realistic in production — validators are slashed for downtime or double-signing, and the bonus pool can be depleted by high claim volume or delayed replenishment. Neither condition requires any privileged access; a delegator who redelegated to a misbehaving validator is affected automatically via the `BeforeRedelegationSlashed` hook. [5](#0-4) 

### Recommendation
Do not trust the in-memory state of `pos` after `processEventsAndClaimBonus` returns `ErrInsufficientBonusPool`. Re-fetch the position from the store before persisting, or reset the mutated fields explicitly:

```go
if _, err := k.processEventsAndClaimBonus(ctx, &pos); err != nil {
    if errors.Is(err, types.ErrInsufficientBonusPool) {
        k.logger(ctx).Error("insufficient bonus pool during redelegation slash", ...)
        // Re-fetch pos to discard any partial checkpoint mutations
        pos, err = k.getPositionState(ctx, positionId)
        if err != nil {
            return err
        }
    } else {
        return err
    }
}
```

Alternatively, `processEventsAndClaimBonus` should be refactored to check pool balance **before** advancing any checkpoints, making it safe to swallow the error without re-fetching.

### Proof of Concept
1. A delegator creates a tiered-rewards position and redelegates it to validator V.
2. The bonus pool is drained (e.g., by other claimants) to below the position's accrued bonus.
3. Validator V is slashed for a redelegation entry. The `BeforeRedelegationSlashed` hook fires, calling `slashRedelegationPosition`.
4. `processEventsAndClaimBonus(ctx, &pos)` walks events, advances `pos.LastEventSeq` and `pos.LastBonusAccrual`, then returns `ErrInsufficientBonusPool`.
5. The error is swallowed; `setPositionWithState` persists the advanced checkpoints.
6. The delegator later calls `MsgClaimTierRewards`. `processEventsAndClaimBonus` starts from the advanced `LastEventSeq` and finds no new events to process — the bonus for the entire pre-slash period is gone. [1](#0-0) [6](#0-5)

func (k Keeper) sufficientBonusPoolBalance(ctx context.Context, bonus sdk.Coins) error {
	if bonus.IsZero() {
		return nil
	}

	poolAddr := k.accountKeeper.GetModuleAddress(types.RewardsPoolName)
	poolBalance := k.bankKeeper.GetAllBalances(ctx, poolAddr)
	if !poolBalance.IsAllGTE(bonus) {
		return errorsmod.Wrapf(types.ErrInsufficientBonusPool,
			"bonus: %s, pool balance: %s",
			bonus.String(), poolBalance.String())
	}

	return nil

func (k Keeper) claimRewards(ctx context.Context, pos types.PositionState) (types.PositionState, sdk.Coins, sdk.Coins, error) {
	if !pos.IsDelegated() {
		return pos, sdk.NewCoins(), sdk.NewCoins(), nil
	}

	base, err := k.claimBaseRewards(ctx, pos)
	if err != nil {
		return types.PositionState{}, nil, nil, err
	}

	bonus, err := k.processEventsAndClaimBonus(ctx, &pos)
	if err != nil {
		return types.PositionState{}, nil, nil, err
	}

	return pos, base, bonus, nil
}
