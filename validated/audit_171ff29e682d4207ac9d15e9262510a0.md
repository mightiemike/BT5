### Title
Missing Slippage Protection in NLP Mint and Burn — (`core/contracts/Clearinghouse.sol`, `core/contracts/interfaces/IEndpoint.sol`)

---

### Summary

The `mintNlp` and `burnNlp` functions in `Clearinghouse.sol` exchange quote tokens for NLP tokens (and vice versa) at an `oraclePriceX18` that is injected by the sequencer and is not part of the user's signed transaction payload. Neither the `MintNlp` nor the `BurnNlp` struct includes a user-specified minimum output amount, leaving users with no on-chain slippage protection for either direction of the NLP exchange.

---

### Finding Description

The `MintNlp` struct contains only `sender`, `quoteAmount`, and `nonce`. The `BurnNlp` struct contains only `sender`, `nlpAmount`, and `nonce`. Neither includes a field for a minimum acceptable output amount. [1](#0-0) 

The outer wrappers `SignedMintNlp` and `SignedBurnNlp` carry `oraclePriceX18` as a sequencer-appended field — outside the user's signed inner struct. [2](#0-1) 

In `mintNlp`, the NLP amount credited to the user is computed as:

```solidity
int128 nlpAmount = quoteAmount.div(oraclePriceX18);
``` [3](#0-2) 

In `burnNlp`, the quote amount returned to the user is computed as:

```solidity
int128 quoteAmount = nlpAmount.mul(oraclePriceX18);
``` [4](#0-3) 

In both cases, `oraclePriceX18` is provided by the sequencer at execution time and is not validated against any on-chain oracle bound. The user's signature covers only the inner struct (`MintNlp` / `BurnNlp`), which does not commit to any price or minimum output. There is no `require` in either function that enforces a user-specified minimum NLP or minimum quote amount. [5](#0-4) [6](#0-5) 

The `ERR_SLIPPAGE_TOO_HIGH` error constant is defined in `Errors.sol` but is not applied in the NLP mint or burn execution paths. [7](#0-6) 

---

### Impact Explanation

A user who signs a `MintNlp` transaction committing `quoteAmount` of quote tokens has no on-chain guarantee about the minimum NLP they will receive. If the sequencer supplies a depressed `oraclePriceX18`, the user receives fewer NLP tokens for the same quote spend. Symmetrically, a user burning `nlpAmount` of NLP has no on-chain guarantee about the minimum quote returned; a suppressed `oraclePriceX18` reduces the quote credited. In both cases the user's subaccount balance is silently reduced relative to their expectation, with no revert path available to them.

---

### Likelihood Explanation

Exploitation requires the sequencer to supply an `oraclePriceX18` that deviates unfavorably from the true market price. The sequencer is a privileged but externally operated component; the protocol's own architecture places no on-chain bound on the price value it may submit. Even absent malicious intent, a stale or lagged oracle price submitted during a period of rapid NLP NAV movement produces the same adverse outcome for the user. The absence of any user-controlled minimum-output guard means there is no circuit-breaker regardless of the cause of the price deviation.

---

### Recommendation

Add a `minNlpAmount` field to `MintNlp` and a `minQuoteAmount` field to `BurnNlp`. In `mintNlp`, assert `nlpAmount >= txn.minNlpAmount` before crediting the balance. In `burnNlp`, assert `quoteAmount >= txn.minQuoteAmount` before crediting the balance. Because these fields would be part of the user-signed inner struct, the user's EIP-712 signature would commit to the acceptable price range, providing enforceable on-chain slippage protection without requiring any trust in the sequencer's price submission.

---

### Proof of Concept

1. User signs `MintNlp { sender: alice, quoteAmount: 1000e18, nonce: 1 }` expecting NLP at price 1.0 (i.e., 1000 NLP).
2. Sequencer wraps it as `SignedMintNlp { tx: <above>, signature: <alice_sig>, oraclePriceX18: 2e18, nlpPoolRebalanceX18: [...] }`.
3. `mintNlp` executes: `nlpAmount = 1000e18 / 2e18 = 500`. Alice receives 500 NLP instead of 1000.
4. No `require` in `mintNlp` checks a user-specified minimum; the transaction succeeds and Alice's quote balance is debited 1000 while her NLP balance is credited only 500.
5. The same attack applies symmetrically to `burnNlp` by supplying a suppressed `oraclePriceX18`.

    function burnNlp(
        IEndpoint.BurnNlp calldata txn,
        int128 oraclePriceX18,
        IEndpoint.NlpPool[] calldata nlpPools,
        int128[] calldata nlpPoolRebalanceX18
    ) external onlyEndpoint {
        require(!RiskHelper.isIsolatedSubaccount(txn.sender), ERR_UNAUTHORIZED);

        ISpotEngine spotEngine = _spotEngine();
        spotEngine.updatePrice(NLP_PRODUCT_ID, oraclePriceX18);

        require(txn.nlpAmount <= INT128_MAX, ERR_CONVERSION_OVERFLOW);
        int128 nlpAmount = int128(txn.nlpAmount);
        require(
            spotEngine.getNlpUnlockedBalance(txn.sender).amount >= nlpAmount,
            ERR_UNLOCKED_NLP_INSUFFICIENT
        );
        int128 quoteAmount = nlpAmount.mul(oraclePriceX18);
        int128 burnFee = MathHelper.max(ONE, quoteAmount / 1000);
        quoteAmount = MathHelper.max(0, quoteAmount - burnFee);

        _validateNlpRebalance(nlpPools, nlpPoolRebalanceX18, -quoteAmount);
        for (uint128 i = 0; i < nlpPoolRebalanceX18.length; i++) {
            require(nlpPoolRebalanceX18[i] <= 0, ERR_INVALID_NLP_REBALANCE);
        }

        spotEngine.updateBalance(NLP_PRODUCT_ID, txn.sender, -nlpAmount);
        spotEngine.updateBalance(NLP_PRODUCT_ID, N_ACCOUNT, nlpAmount);

        if (quoteAmount > 0) {
            spotEngine.updateBalance(QUOTE_PRODUCT_ID, txn.sender, quoteAmount);
            _applyNlpRebalance(spotEngine, nlpPools, nlpPoolRebalanceX18);
        }

        require(
            spotEngine.getBalance(NLP_PRODUCT_ID, txn.sender).amount >= 0,
            ERR_SUBACCT_HEALTH
        );
        // Burning NLP can decrease health if the burn fee exceeds the health improvement
        // from the withdrawal. This check prevents malicious actors from deliberately
        // creating unhealthy subaccounts through NLP burns.
        require(
            getHealth(txn.sender, IProductEngine.HealthType.MAINTENANCE) >= 0,
            ERR_SUBACCT_HEALTH
        );
    }
