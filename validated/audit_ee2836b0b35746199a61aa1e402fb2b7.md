Audit Report

## Title
`SwapAllowlistExtension` Bypassed via Router — `sender` Is Always Router Address, Not Original User - (File: metric-periphery/contracts/extensions/SwapAllowlistExtension.sol)

## Summary
`MetricOmmPool.swap` passes `msg.sender` as the `sender` argument to `_beforeSwap`, and therefore to every configured extension. When a swap is routed through `MetricOmmSimpleRouter` (any of `exactInputSingle`, `exactInput`, `exactOutputSingle`, `exactOutput`), `msg.sender` inside the pool is always the router contract, not the original user. `SwapAllowlistExtension.beforeSwap` gates access by checking `allowedSwapper[pool][sender]`, so it checks whether the **router** is allowlisted, never the actual trader. Any unprivileged user can bypass a pool's swap allowlist by routing through `MetricOmmSimpleRouter`.

## Finding Description

**Exact call path:**

1. User calls `MetricOmmSimpleRouter.exactOutput(params)`. [1](#0-0) 

2. The router calls `IMetricOmmPoolActions(pool).swap(params.recipient, ...)`. At this point `msg.sender` inside the pool's `swap` is the **router address**. [2](#0-1) 

3. `MetricOmmPool.swap` immediately calls `_beforeSwap(msg.sender, ...)`, forwarding the router address as `sender`. [3](#0-2) 

4. `_beforeSwap` dispatches to `SwapAllowlistExtension.beforeSwap(sender=router, ...)`. [4](#0-3) 

5. The extension checks `allowedSwapper[msg.sender][sender]` = `allowedSwapper[pool][router]`. The original user's address is never consulted. [5](#0-4) 

**Recursive hops (`_exactOutputIterateCallback`):** The same issue applies. The router calls `pool.swap(msg.sender, ...)` where `msg.sender` is the prior pool (the callback caller), but the new pool still sees `msg.sender` = router as the caller of its `swap` function, so `sender` = router again. [6](#0-5) 

**Why existing guards fail:** The router correctly stores the original user as the *payer* in transient storage (`_setNextCallbackContext(..., msg.sender, ...)`) for payment settlement, but this payer identity is never forwarded to the pool's `swap` call or to any extension hook. The `SwapAllowlistExtension` has no mechanism to look through the router to the originating user. [7](#0-6) 

## Impact Explanation
A pool admin deploys a pool with `SwapAllowlistExtension` to restrict swapping to a curated set of addresses (e.g., KYC'd traders or protocol-owned contracts). If the router is allowlisted (the natural configuration to allow any approved user to swap via the router), **every unprivileged address** can bypass the allowlist by calling `exactInputSingle` / `exactInput` / `exactOutputSingle` / `exactOutput` on `MetricOmmSimpleRouter`. The allowlist access-control invariant is completely broken: the pool admin's intended gate is never evaluated against the actual trader. This constitutes a broken core pool functionality and admin-boundary break with direct fund-flow impact (unauthorized parties execute swaps the pool was designed to reject).

## Likelihood Explanation
Exploitable by any unprivileged EOA or contract with no special preconditions beyond the router being allowlisted on the target pool. The router is a public, permissionless contract. The attack is repeatable every block, requires no capital beyond the swap input, and leaves no on-chain trace distinguishing it from a legitimate router swap.

## Recommendation
Pass the original user's address through the router to the pool as a dedicated `sender` parameter, or encode it in `callbackData`/`extensionData` and have the pool expose it to extensions. The cleanest fix is to add a `sender` field to the pool's `swap` interface (separate from `msg.sender`) that the router populates with `msg.sender` at entry, and that `_beforeSwap` forwards to extensions. Alternatively, `SwapAllowlistExtension` can be extended to allowlist the router itself and require the router to attest the real sender via a signed or transient-storage mechanism — but this requires a coordinated change across the router and extension.

pragma solidity ^0.8.35;
// Foundry integration test sketch

function test_swapAllowlist_bypassViaRouter() public {
    // 1. Deploy pool with SwapAllowlistExtension; allowlist only `trustedTrader`.
    //    Also allowlist the router so legitimate users can swap through it.
    extension.setAllowedToSwap(address(pool), address(router), true);
    // trustedTrader is NOT set; attacker is NOT set.

    // 2. Attacker (not on allowlist) calls exactInputSingle through the router.
    vm.prank(attacker);
    router.exactInputSingle(IMetricOmmSimpleRouter.ExactInputSingleParams({
        pool: address(pool),
        tokenIn: tokenIn,
        tokenOut: tokenOut,
        zeroForOne: true,
        amountIn: 1_000,
        amountOutMinimum: 0,
        recipient: attacker,
        deadline: block.timestamp + 1,
        priceLimitX64: 0,
        extensionData: ""
    }));
    // Expected: revert NotAllowedToSwap
    // Actual:   swap succeeds — extension saw sender=router (allowlisted), not attacker
}
