### Title
`ContractCodeResponse::V1` Accepted Without Authentication Pre-`SignedContractCodeResponse` — Event-Based Gate Allows Any Peer to Inject Arbitrary Contract Code into Chunk Validators - (File: `chain/client/src/stateless_validation/validate.rs`)

### Summary

`validate_contract_code_response` skips all signature verification when `ProtocolFeature::SignedContractCodeResponse` is not yet enabled (protocol version < 85). Any network peer can send a `ContractCodeResponse::V1` (Borsh discriminant `0`) carrying arbitrary compressed contract bytes for any valid `ChunkProductionKey`, and the receiving chunk validator will accept and store those bytes without any authentication. This is the exact nearcore analog of the "event-based" authentication pattern described in the external report: the gate is a relevance check (`validate_chunk_relevant`) rather than a cryptographic binding to a specific chunk producer.

### Finding Description

`ContractCodeResponse` is a versioned enum with two variants:

- `V1(ContractCodeResponseV1)` — unsigned; carries only `next_chunk` + `compressed_contracts`
- `V2(ContractCodeResponseV2)` — signed; carries an inner payload + a `Signature` over `borsh(ContractCodeResponseV2Inner)` which includes a `signature_differentiator` [1](#0-0) 

`ContractCodeResponse::verify_signature` explicitly returns `false` for `V1` ("nothing to verify"): [2](#0-1) 

The validation entry point is `validate_contract_code_response`:

```rust
pub fn validate_contract_code_response(...) -> Result<ChunkRelevance, Error> {
    let key = response.chunk_production_key();
    require_relevant!(validate_chunk_relevant(epoch_manager, key, store)?);
    let protocol_version = epoch_manager.get_epoch_protocol_version(&key.epoch_id)?;
    if ProtocolFeature::SignedContractCodeResponse.enabled(protocol_version) {
        validate_witness_contract_code_response_signature(epoch_manager, response)?;
    }
    Ok(ChunkRelevance::Relevant)
}
``` [3](#0-2) 

When `SignedContractCodeResponse` is not enabled, the function returns `Ok(ChunkRelevance::Relevant)` after only checking that the `ChunkProductionKey` is within the expected height window — no sender identity check, no signature check. The caller then unconditionally decompresses and stores the payload:

```rust
fn handle_contract_code_response(&self, response: ContractCodeResponse) -> Result<(), Error> {
    if !validate_contract_code_response(...)?.is_relevant() { return Ok(()); }
    let key = response.chunk_production_key().clone();
    let contracts = response.decompress_contracts()?;
    self.partial_witness_tracker.store_accessed_contract_codes(key, contracts)
}
``` [4](#0-3) 

The `SignedContractCodeResponse` feature is stabilized at protocol version 85: [5](#0-4) 

`ContractCodeResponse::encode` emits `V1` pre-feature and `V2` post-feature: [6](#0-5) 

### Impact Explanation

During the protocol upgrade window (any node running at protocol version < 85), any network peer that can reach a chunk validator's P2P port can:

1. Craft a `ContractCodeResponse::V1` (Borsh tag `0x00`) with a valid `ChunkProductionKey` for an in-flight chunk and arbitrary `compressed_contracts` bytes.
2. Send it to the target validator.
3. `validate_contract_code_response` passes (relevance check only), `decompress_contracts()` succeeds, and `store_accessed_contract_codes` stores the attacker-controlled bytes.

The stored bytes are indexed by the `ChunkProductionKey`. If the injected bytes arrive before the legitimate response, the validator may use wrong contract code when assembling or validating the chunk, producing an incorrect endorsement or failing to endorse a valid chunk. At minimum, the attacker can prevent the validator from ever receiving the legitimate code (cache-slot poisoning), stalling chunk validation for that slot.

### Likelihood Explanation

`SignedContractCodeResponse` is at protocol version 85 (stable). Any node that has not yet upgraded past version 84 is reachable. The attack requires only a valid `ChunkProductionKey` (publicly observable from the chain) and the ability to send a P2P message — no validator key, no stake, no privileged role. The Borsh encoding of a `V1` response is straightforward to construct.

### Recommendation

1. **Short term**: In `validate_contract_code_response`, reject `V1` responses unconditionally once the network has fully upgraded past protocol version 85. Add an explicit check: if the response is `V1` and the feature is enabled, return an error rather than silently accepting.
2. **Medium term**: Remove the `V1` deserialization path entirely once no node on any supported network can be at a protocol version that emits `V1`. This eliminates the "event-based" gate entirely.
3. **Long term**: Ensure all future versioned P2P messages that carry sensitive data (contract code, witness parts) are signed from their first version, so there is never a pre-feature window with unauthenticated payloads.

let fake_contracts = vec![CodeBytes(Arc::from(b"(module)".as_slice()))]; // arbitrary WASM

let wire = borsh::to_vec(&response).unwrap();
// wire[0] == 0x00  (V1 discriminant)

// validate_contract_code_response returns Ok(Relevant) pre-v85.
// handle_contract_code_response stores the fake bytes under `target_key`.
```

The exact divergent Borsh bytes are: `[0x00, <next_chunk borsh>, <compressed_contracts borsh>]` — the `V1` discriminant `0x00` is accepted without any signature field, while `V2` (`0x01`) requires a valid ed25519 signature over `borsh(ContractCodeResponseV2Inner)` including the `"ContractCodeResponseV2Inner"` differentiator string. pub enum ContractCodeResponse {
    V1(ContractCodeResponseV1) = 0,
    V2(ContractCodeResponseV2) = 1,
}

impl ContractCodeResponse {
    pub fn encode(
        next_chunk: ChunkProductionKey,
        contracts: &Vec<CodeBytes>,
        signer: &ValidatorSigner,
        protocol_version: ProtocolVersion,
    ) -> std::io::Result<Self> {
        if ProtocolFeature::SignedContractCodeResponse.enabled(protocol_version) {
            ContractCodeResponseV2::encode(next_chunk, contracts, signer).map(Self::V2)
        } else {
            ContractCodeResponseV1::encode(next_chunk, contracts).map(Self::V1)
        }
    }

    pub fn chunk_production_key(&self) -> &ChunkProductionKey {
        match self {
            Self::V1(v1) => &v1.next_chunk,
            Self::V2(v2) => &v2.inner.next_chunk,
        }
    }

    /// Account that produced this response. Available only for signed variants.
    pub fn responder(&self) -> Option<&AccountId> {
        match self {
            Self::V1(_) => None,
            Self::V2(v2) => Some(&v2.inner.responder),
        }
    }

    pub fn decompress_contracts(&self) -> std::io::Result<Vec<CodeBytes>> {
        let compressed_contracts = match self {
            Self::V1(v1) => &v1.compressed_contracts,
            Self::V2(v2) => &v2.inner.compressed_contracts,
        };
        compressed_contracts.decode().map(|(data, _size)| data)
    }

    /// Verifies the signature for signed variants. Returns `false` for
    /// unsigned variants since there is nothing to verify.
    pub fn verify_signature(&self, public_key: &PublicKey) -> bool {
        match self {
            Self::V1(_) => false,
            Self::V2(v2) => v2.verify_signature(public_key),
        }
    }
