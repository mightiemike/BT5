Audit Report

## Title
Missing Zero-Address Validation for `_oracle` Constructor Parameter Permanently Breaks Pool Swap Functionality — (`File: smart-contracts-poc/contracts/PriceProvider.sol` and `smart-contracts-poc/contracts/AnchoredPriceProvider.sol`)

## Summary
Both `PriceProvider` and `AnchoredPriceProvider` assign `_oracle` to the `immutable` field `offchainOracle` without a zero-address guard, while `_factory`, `_baseToken`, and `_quoteToken` are all explicitly validated. If `address(0)` is passed, the broken oracle binding is permanently baked in. Any pool using this provider will have every swap permanently revert with `PriceProviderFailed`, with no on-chain recovery path for immutable-provider pools.

## Finding Description
In `PriceProvider.sol`, the constructor validates `_factory` (L70) and both token addresses (L76) but performs no check on `_oracle` before assigning it to the `immutable` `offchainOracle` (L73). The identical pattern exists in `AnchoredPriceProvider.sol` at L136–139.

When `getBidAndAskPrice()` is called on a provider with `offchainOracle = address(0)`, the internal call at `PriceProvider.sol` L194–195:
```solidity
(uint256 mid, uint256 spread, , uint256 refTime) =
    IPricedOracle(address(offchainOracle)).price(offchainFeedId, msg.sender);
```
issues a high-level call to `address(0)`. The EVM returns success with empty returndata; Solidity's ABI decoder reverts attempting to decode four `uint256` values from zero bytes.

`MetricOmmPool._getBidAndAskPriceX64()` (L806–811) wraps this in a `try/catch` and re-reverts with `PriceProviderFailed`, blocking every swap.

The factory's `_validatePriceProvider` (MetricOmmPoolFactory.sol L541–546) only checks that the provider address is non-zero and that `token0()`/`token1()` match the pool tokens. Since those values are stored as separate immutables (not derived from the oracle), they return correctly even when `offchainOracle = address(0)`, allowing the broken provider to pass pool-creation validation silently.

For pools created with `immutablePriceProvider` set (i.e., `priceProviderTimelock == type(uint256).max`), `proposePoolPriceProvider` reverts with `PriceProviderImmutable` (MetricOmmPoolFactory.sol L482), leaving no recovery path.

## Impact Explanation
Every `swap()` call on the affected pool permanently reverts with `PriceProviderFailed`. Swap is the core revenue and utility function of the pool; its permanent failure renders the pool commercially dead. For immutable-provider pools, there is no on-chain recovery — the pool must be redeployed and all LP positions migrated. This constitutes broken core pool functionality causing unusable swap flows, matching the allowed impact gate.

## Likelihood Explanation
The `_oracle` parameter is the most operationally critical address in the provider constructor, yet it is the only address-type parameter lacking a zero guard. A deployer who accidentally passes `address(0)` (e.g., from a misconfigured deployment script or a missing environment variable) receives no on-chain error at construction time. The factory's post-deployment validation does not catch the defect. The mistake is easy to make, silent, and irreversible for immutable-provider pools.

## Recommendation
Add an explicit zero-address check for `_oracle` in both constructors, consistent with the existing guards:

**`PriceProvider.sol`** (after line 70):
```solidity
require(_oracle != address(0));
offchainOracle = IOffchainOracle(_oracle);
```

**`AnchoredPriceProvider.sol`** (after line 136):
```solidity
require(_oracle != address(0));
offchainOracle = IOffchainOracle(_oracle);
```

Optionally, extend `_validatePriceProvider` in the factory to verify the oracle address is non-zero via a public getter on the provider, so a broken provider is rejected at pool-creation time rather than discovered only when the first swap is attempted.

## Proof of Concept
1. Deploy `PriceProvider` with `_oracle = address(0)`, valid `_factory`, `_baseToken = WETH`, `_quoteToken = USDC`, a valid `_offchainFeedId`, and any non-zero `_maxTimeDelta`. Construction succeeds; `offchainOracle` is permanently `address(0)`.
2. Call `MetricOmmPoolFactory.createPool(...)` with this provider as `params.priceProvider`. `_validatePriceProvider` calls `provider.token0()` → `WETH` ✓ and `provider.token1()` → `USDC` ✓. Pool is created successfully.
3. Any caller invokes `pool.swap(...)`. The pool calls `_getBidAndAskPriceX64()`, which calls `provider.getBidAndAskPrice()`. Internally, `IPricedOracle(address(0)).price(feedId, pool)` is issued; the EVM returns empty bytes; Solidity's ABI decoder reverts. The `catch` block fires and the pool reverts with `PriceProviderFailed`.
4. If the pool was created with `priceProviderTimelock == type(uint256).max`, `proposePoolPriceProvider` reverts with `PriceProviderImmutable`; the pool is permanently broken and must be redeployed.
