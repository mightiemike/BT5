Audit Report

## Title
`SwapAllowlistExtension` gates the router address instead of the actual user, allowing any user to bypass the per-user swap allowlist via `MetricOmmSimpleRouter` — (`metric-periphery/contracts/extensions/SwapAllowlistExtension.sol`)

## Summary

`SwapAllowlistExtension.beforeSwap` checks `allowedSwapper[pool][sender]` where `sender` is the immediate caller of `pool.swap()`. When swaps are routed through `MetricOmmSimpleRouter`, the router is `msg.sender` of `pool.swap()`, so the extension checks the router's allowlist status rather than the end-user's. A pool admin who allowlists the router to enable router-mediated swaps for approved users inadvertently opens the gate to every public user, breaking the per-user access-control invariant of the extension.

## Finding Description

**Step 1 — Pool passes its own `msg.sender` as `sender` to the extension.**

`MetricOmmPool.swap()` calls `_beforeSwap(msg.sender, recipient, ...)`: [1](#0-0) 

`ExtensionCalling._beforeSwap` forwards that value unchanged as the `sender` parameter of `IMetricOmmExtensions.beforeSwap`: [2](#0-1) 

**Step 2 — The extension checks `sender`, not the actual end-user.**

`SwapAllowlistExtension.beforeSwap` uses `msg.sender` (the pool) as the pool dimension and `sender` (the caller of `pool.swap()`) as the swapper dimension: [3](#0-2) 

**Step 3 — The router is the caller of `pool.swap()`, not the end-user.**

`MetricOmmSimpleRouter.exactInputSingle` calls `pool.swap(params.recipient, ...)` directly, making the router `msg.sender` inside the pool: [4](#0-3) 

The same applies to `exactInput` multi-hop paths: [5](#0-4) 

**Step 4 — Contrast with `DepositAllowlistExtension`.**

`DepositAllowlistExtension.beforeAddLiquidity` correctly checks `owner` (the second argument — the economic beneficiary), not `sender` (the immediate caller), so the deposit allowlist cannot be bypassed via a router: [6](#0-5) 

The swap extension has no equivalent — it only sees the immediate caller. The `recipient` parameter (second argument to `beforeSwap`) is available but ignored by the extension.

## Impact Explanation

Any public user can bypass a pool's per-user swap allowlist by calling `MetricOmmSimpleRouter.exactInputSingle`, `exactInput`, or `exactOutput` on a pool configured with `SwapAllowlistExtension`, provided the admin has allowlisted the router. Allowlisting the router is the only way to enable router-mediated swaps for approved users, so the admin faces an impossible choice: either approved users cannot use the router, or the allowlist is silently opened to the entire public. This breaks the core access-control invariant of the extension and allows unauthorized parties to trade in pools intended to be restricted (e.g., KYC-gated, institutional-only, or compliance-restricted pools). The corrupted value is the `allowedSwapper[pool][sender]` gate decision — it evaluates `true` for the router when it should evaluate against the actual end-user address.

## Likelihood Explanation

The router is the primary user-facing entry point for swaps. A pool admin deploying `SwapAllowlistExtension` who wants approved users to have normal UX will naturally allowlist the router — this is the expected operational pattern. The bypass requires no special privileges, no flash loans, and no unusual token behavior. Any public user can trigger it with a standard router call, making it trivially repeatable.

## Recommendation

The extension must gate the actual end-user, not the immediate caller of `pool.swap()`. The `recipient` parameter (second argument to `beforeSwap`) is the address that receives swap output and is set by the end-user, not the router. The simplest fix is to change `beforeSwap` to check `allowedSwapper[msg.sender][recipient]` instead of `allowedSwapper[msg.sender][sender]`, mirroring how `DepositAllowlistExtension` gates on `owner`. Alternatively, pass the real user address through `extensionData` with router attestation, or check `tx.origin` for EOA-only pools.

## Proof of Concept

```
Setup:
  pool P configured with SwapAllowlistExtension E
  admin calls E.setAllowedToSwap(P, alice, true)   // alice is allowlisted
  admin calls E.setAllowedToSwap(P, router, true)  // router allowlisted so alice can use it

Attack:
  bob (not allowlisted) calls:
    router.exactInputSingle({pool: P, tokenIn: T0, tokenOut: T1, recipient: bob, ...})

  Execution trace:
    router.exactInputSingle()
      → pool.swap(recipient=bob, ...) [msg.sender = router]
        → _beforeSwap(sender=router, recipient=bob, ...)
          → E.beforeSwap(sender=router, ...)
            → allowedSwapper[P][router] == true  ✓  (no revert)
        → swap executes for bob

Result: bob, who is not on the allowlist, successfully swaps in a pool
        restricted to alice only.
```
