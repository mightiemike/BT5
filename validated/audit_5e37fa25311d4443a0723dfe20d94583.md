### Title
Malicious IBC Counterparty Can Drain Escrowed NFTs via Crafted ClassId Prefix — (`x/nft-transfer/keeper/packet.go`)

### Summary

`processReceivedPacket` determines whether to mint or unescrow an NFT by calling `IsAwayFromOrigin(packet.GetSourcePort(), packet.GetSourceChannel(), data.ClassId)`. Both `sourcePort`/`sourceChannel` and `data.ClassId` are fully attacker-controlled (they come from the counterparty chain). A malicious counterparty can craft `data.ClassId` to begin with its own `sourcePort/sourceChannel/` prefix, forcing `IsAwayFromOrigin` to return `false` and triggering the unescrow branch, which calls `TransferOwner` from the local escrow address to the attacker-specified receiver.

---

### Finding Description

`IsAwayFromOrigin` is defined as: [1](#0-0) 

```go
func IsAwayFromOrigin(sourcePort, sourceChannel, fullClassPath string) bool {
    prefixClassID := GetClassPrefix(sourcePort, sourceChannel)
    return !strings.HasPrefix(fullClassPath, prefixClassID)
}
```

`GetClassPrefix` returns `"<portID>/<channelID>/"`. [2](#0-1) 

In `processReceivedPacket`, the branch decision is: [3](#0-2) 

```go
isAwayFromOrigin := types.IsAwayFromOrigin(packet.GetSourcePort(), packet.GetSourceChannel(), data.ClassId)
escrowAddress := types.GetEscrowAddress(packet.GetDestPort(), packet.GetDestChannel())
```

When `isAwayFromOrigin == false`, the unescrow path executes: [4](#0-3) 

```go
unprefixedClassID := types.RemoveClassPrefix(packet.GetSourcePort(),
    packet.GetSourceChannel(), data.ClassId)
voucherClassID := types.ParseClassTrace(unprefixedClassID).IBCClassID()
for _, tokenID := range data.TokenIds {
    if err := k.nftKeeper.TransferOwner(ctx,
        voucherClassID, tokenID, escrowAddress, receiver); err != nil {
        return err
    }
}
```

**Attack construction:**

| Field | Attacker-set value |
|---|---|
| `packet.SourcePort` | `"nft"` |
| `packet.SourceChannel` | `"channel-0"` (attacker's channel ID on their chain) |
| `data.ClassId` | `"nft/channel-0/myclass"` |
| `data.TokenIds` | `["t1"]` (a token known to be escrowed on the victim chain) |
| `data.Receiver` | attacker's address |

Evaluation on the receiving chain:
- `prefixClassID = "nft/channel-0/"`
- `strings.HasPrefix("nft/channel-0/myclass", "nft/channel-0/")` → `true`
- `IsAwayFromOrigin` → `false` → unescrow branch taken
- `unprefixedClassID = "myclass"`, `voucherClassID = "myclass"`
- `escrowAddress = GetEscrowAddress(destPort, destChannel)` — the legitimate escrow address holding Alice's NFT
- `TransferOwner(ctx, "myclass", "t1", escrowAddress, attacker)` — succeeds because `escrowAddress` owns `t1`

`ValidateBasic` on the packet data performs no validation of the classId trace structure or its relationship to the source port/channel: [5](#0-4) 

There is no guard anywhere in `OnRecvPacket` → `processReceivedPacket` that cross-checks whether `data.ClassId` was legitimately prefixed by the sending chain.

---

### Impact Explanation

Any NFT escrowed on the receiving chain (i.e., sent outbound via the same channel) can be stolen by the malicious counterparty chain. The attacker only needs to know the `classID` and `tokenID` of an escrowed NFT (both are observable on-chain). This is a direct, irreversible theft of NFT assets — a Critical impact.

---

### Likelihood Explanation

The attack requires the counterparty chain to be malicious or compromised. This is a realistic IBC threat model: IBC explicitly does not trust counterparty chains. The exploit requires no governance, no privileged keys, and no social engineering — only the ability to submit a valid IBC packet from the counterparty chain.

---

### Recommendation

Validate that `data.ClassId` does **not** start with the receiving chain's own `destPort/destChannel/` prefix (which would be a forged "return" path). Alternatively, mirror the ICS-20 fungible token transfer approach: only accept a classId as "returning to origin" if the receiving chain can verify it previously issued a voucher for that exact class trace. A concrete fix is to check, before entering the unescrow branch, that a class trace record for `unprefixedClassID` actually exists in the local store (i.e., the chain previously minted a voucher for it), and reject the packet otherwise.

---

### Proof of Concept

```
1. Chain B (victim) has NFT classID="myclass", tokenID="t1" owned by Alice.
2. Alice calls SendTransfer on chain B → NFT is escrowed at
   GetEscrowAddress("nft", "channel-0") on chain B.
3. Malicious chain A constructs and commits a packet:
     SourcePort="nft", SourceChannel="channel-0"
     DestPort="nft",   DestChannel="channel-0"  (chain B's channel)
     data.ClassId="nft/channel-0/myclass"
     data.TokenIds=["t1"]
     data.Receiver=<attacker address on chain B>
4. IBC relayer delivers the packet to chain B.
5. OnRecvPacket → processReceivedPacket:
     IsAwayFromOrigin("nft","channel-0","nft/channel-0/myclass") = false
     unprefixedClassID = "myclass"
     voucherClassID    = "myclass"
     escrowAddress     = GetEscrowAddress("nft","channel-0")
     TransferOwner("myclass","t1", escrowAddress, attacker) → SUCCESS
6. Attacker now owns t1. Alice's NFT is gone.
``` [6](#0-5)

```go
func (k Keeper) processReceivedPacket(ctx sdk.Context, packet channeltypes.Packet,
	data types.NonFungibleTokenPacketData,
) error {
	receiver, err := sdk.AccAddressFromBech32(data.Receiver)
	if err != nil {
		return err
	}

	isAwayFromOrigin := types.IsAwayFromOrigin(packet.GetSourcePort(), packet.GetSourceChannel(), data.ClassId)

	// create the escrow address for creating denom and minting nft
	escrowAddress := types.GetEscrowAddress(packet.GetDestPort(), packet.GetDestChannel())

	if isAwayFromOrigin {
		// since SendPacket did not prefix the classID, we must prefix classID here
		classPrefix := types.GetClassPrefix(packet.GetDestPort(), packet.GetDestChannel())
		// NOTE: sourcePrefix contains the trailing "/"
		prefixedClassID := classPrefix + data.ClassId

		// construct the class trace from the full raw classID
		classTrace := types.ParseClassTrace(prefixedClassID)
		if !k.HasClassTrace(ctx, classTrace.Hash()) {
			k.SetClassTrace(ctx, classTrace)
		}

		voucherClassID := classTrace.IBCClassID()

		if !k.nftKeeper.HasDenomID(ctx, voucherClassID) {
			if err := k.nftKeeper.IssueDenom(ctx, voucherClassID, voucherClassID, "", data.ClassUri, escrowAddress); err != nil {
				return err
			}
		}
		sdkCtx := sdk.UnwrapSDKContext(ctx)
		sdkCtx.EventManager().EmitEvent(
			sdk.NewEvent(
				types.EventTypeClassTrace,
				sdk.NewAttribute(types.AttributeKeyTraceHash, classTrace.Hash().String()),
				sdk.NewAttribute(types.AttributeKeyClassID, voucherClassID),
			),
		)

		for i, tokenID := range data.TokenIds {
			if err := k.nftKeeper.MintNFT(ctx, voucherClassID, tokenID, "", data.TokenUris[i], "", escrowAddress, receiver); err != nil {
				return err
			}
		}
	} else {
		// If the token moves in the direction of back to origin,
		// we need to unescrow the token and transfer it to the receiver

		// we should remove the prefix. For example:
		// p6/c6/p4/c4/p2/c2/nftClass -> p4/c4/p2/c2/nftClass
		unprefixedClassID := types.RemoveClassPrefix(packet.GetSourcePort(),
			packet.GetSourceChannel(), data.ClassId)

		voucherClassID := types.ParseClassTrace(unprefixedClassID).IBCClassID()
		for _, tokenID := range data.TokenIds {
			if err := k.nftKeeper.TransferOwner(ctx,
				voucherClassID, tokenID, escrowAddress, receiver); err != nil {
				return err
			}
		}
	}

	return nil
```
