### Title
SwapAllowlistExtension Checks Router Address Instead of Actual User, Allowing Any User to Bypass Swap Allowlist via Router — (File: `metric-periphery/contracts/extensions/SwapAllowlistExtension.sol`)

---

### Summary

`SwapAllowlistExtension.beforeSwap` gates swaps by checking the `sender` argument against a per-pool allowlist. However, when a user swaps through `MetricOmmSimpleRouter`, the pool's `swap()` function passes `msg.sender` (the router address) as `sender` to the extension. The extension therefore checks whether the **router** is allowlisted, not the actual user. Any pool admin who allowlists the router to enable router-mediated swaps inadvertently opens the pool to every user, completely defeating the per-user allowlist.

---

### Finding Description

**Step 1 — Pool passes `msg.sender` as `sender` to the extension.**

In `MetricOmmPool.swap()`, the `sender` forwarded to `_beforeSwap` is always `msg.sender` — the immediate caller of the pool: [1](#0-0) 

**Step 2 — Router is the immediate caller of the pool.**

When a user calls `exactInputSingle()` on `MetricOmmSimpleRouter`, the router calls `pool.swap()` directly. Inside the pool, `msg.sender` is therefore the **router address**, not the user: [2](#0-1) 

The same applies to every hop of `exactInput()` and `exactOutput()`: [3](#0-2) 

**Step 3 — Extension checks the router address, not the user.**

`SwapAllowlistExtension.beforeSwap` receives `sender` (the router) and checks `allowedSwapper[msg.sender][sender]`, where `msg.sender` is the pool and `sender` is the router: [4](#0-3) 

The allowlist lookup is therefore `allowedSwapper[pool][router]`. If the router is in the allowlist, the check passes for **every user** who routes through it, regardless of whether that user is individually permitted.

**Step 4 — The admin faces an impossible choice.**

A pool admin who wants to allow router-mediated swaps must add the router to `allowedSwapper`. Doing so grants every user on the network the ability to swap on the curated pool. Not doing so means no user can ever use the router on that pool. There is no configuration that enforces per-user allowlist policy through the router.

---

### Impact Explanation

The `SwapAllowlistExtension` is the primary access-control mechanism for curated pools. Its bypass allows any unprivileged user to execute swaps on a pool that was explicitly configured to restrict access. Depending on the pool's purpose (e.g., institutional counterparty pools, KYC-gated pools, or pools with favorable oracle pricing reserved for specific LPs), unauthorized swaps can drain LP value, violate access policy, or allow adversarial price-taking against the pool's oracle-anchored bid/ask. This matches the **admin-boundary break** and **broken core pool functionality** impact categories.

---

### Likelihood Explanation

The `MetricOmmSimpleRouter` is the standard, documented periphery entry point for swaps. Users are expected to interact with pools through it. Any pool admin who deploys a `SwapAllowlistExtension` and wants router-mediated swaps to work will naturally allowlist the router, triggering the bypass. The attacker requires no special privilege — only a call to a public router function.

---

### Recommendation

The extension must gate on the **actual user**, not the intermediate caller. Two viable approaches:

1. **Pass the original user through `extensionData`**: The router encodes `msg.sender` into `extensionData`; the extension decodes and checks it. This requires a trusted router convention.
2. **Add a `swapper` override to the pool's swap interface**: The pool accepts an explicit `swapper` address (validated against `msg.sender` or a trusted-router registry) and forwards it as `sender` to extensions, replacing the raw `msg.sender`.

Either approach must ensure the override cannot be spoofed by an untrusted caller.

---

### Proof of Concept

```
1. Deploy pool with SwapAllowlistExtension configured as a beforeSwap hook.
2. Pool admin calls setAllowedToSwap(pool, router, true)
   — intending to enable router-mediated swaps for allowlisted users.
3. Pool admin does NOT call setAllowedToSwap(pool, alice, true).
4. Alice (not allowlisted) calls MetricOmmSimpleRouter.exactInputSingle(...).
5. Router calls pool.swap(recipient, ...) — pool's msg.sender = router.
6. Pool calls _beforeSwap(router, recipient, ...).
7. Extension evaluates: allowedSwapper[pool][router] == true → passes.
8. Alice's swap executes on the curated pool despite being explicitly excluded.
```
