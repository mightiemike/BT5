### Title
NFT Metadata Permanently Frozen and Unburnable After Transfer for Non-Creator Owners — (`x/nft/keeper/keeper.go`)

### Summary

`EditNFT` and `BurnNFT` enforce a dual guard requiring the caller to be simultaneously the NFT owner **and** the denom creator. After `TransferOwner`, the new owner passes `IsOwner` but always fails `IsDenomCreator`, making it impossible for any non-creator to edit or burn an NFT they legitimately own.

### Finding Description

In `EditNFT` (lines 85–118), the keeper applies two sequential authorization checks:

```go
nft, err := k.IsOwner(ctx, denomID, tokenID, owner)   // line 93
if err != nil { return err }

_, err = k.IsDenomCreator(ctx, denomID, owner)         // line 98
if err != nil { return err }
``` [1](#0-0) 

`IsDenomCreator` checks that `address` equals `denom.Creator`, a field that is set once at `IssueDenom` time and never updated:

```go
if !creator.Equals(address) {
    return types.Denom{}, sdkerrors.Wrapf(types.ErrUnauthorized, ...)
}
``` [2](#0-1) 

`TransferOwner` updates only the NFT's `Owner` field and the owner index — it does not touch `denom.Creator`:

```go
nft.Owner = dstOwner.String()
k.setNFT(ctx, denomID, nft)
k.swapOwner(ctx, denomID, tokenID, srcOwner, dstOwner)
``` [3](#0-2) 

The same dual guard exists in `BurnNFT` (lines 141–161), so the new owner also cannot burn their own NFT: [4](#0-3) 

The entry point is the standard `MsgEditNFT` transaction handled by `msgServer.EditNFT`, which passes `msg.Sender` directly to `keeper.EditNFT` with no additional bypass: [5](#0-4) 

### Impact Explanation

After any `MsgTransferNFT`, the receiving address:
- **Cannot edit** the NFT's name, URI, or data (`EditNFT` → `ErrUnauthorized`)
- **Cannot burn** the NFT they own (`BurnNFT` → `ErrUnauthorized`)

NFT metadata is permanently frozen from the new owner's perspective. The only party who can ever edit or burn the NFT is the original denom creator — even if they no longer own it. This breaks the core ownership invariant: owning an NFT does not grant control over it.

The denom creator retains the ability to edit/burn any NFT in their denom **only if they also hold it**, but cannot act on transferred NFTs either (they fail `IsOwner`). The result is a state where no one can edit or burn a transferred NFT.

### Likelihood Explanation

This is triggered by any standard `MsgTransferNFT` followed by `MsgEditNFT` from the recipient — a completely normal, supported user flow. No special privileges, governance, or key compromise is required. The path is reachable via any CLI-signed transaction.

### Recommendation

`EditNFT` should require **only** `IsOwner`. The denom creator check is appropriate for `MintNFT` (controlling who can create tokens in a collection) but is semantically incorrect for editing or burning — those operations should be gated solely on current ownership. Alternatively, introduce a separate `UpdateDenom` operation for creator-level metadata changes, and restrict `EditNFT`/`BurnNFT` to the current NFT owner only.

### Proof of Concept

```
1. Creator issues denom "testdenom" (creator = addr_A)
2. Creator mints NFT "token1" in "testdenom", owner = addr_A
3. addr_A sends MsgTransferNFT → recipient = addr_B
   → keeper.TransferOwner sets nft.Owner = addr_B, denom.Creator unchanged = addr_A
4. addr_B sends MsgEditNFT{DenomId:"testdenom", Id:"token1", URI:"new-uri", Sender:addr_B}
   → keeper.EditNFT:
       IsOwner(addr_B) → PASS (addr_B is owner)
       IsDenomCreator(addr_B) → FAIL (creator is addr_A)
       → returns ErrUnauthorized
5. addr_B cannot edit or burn their NFT. Metadata is permanently frozen.
```

func (k Keeper) BurnNFT(ctx sdk.Context, denomID, tokenID string, owner sdk.AccAddress) error {
	if !k.HasDenomID(ctx, denomID) {
		return sdkerrors.Wrapf(types.ErrInvalidDenom, "denom ID %s not exists", denomID)
	}

	nft, err := k.IsOwner(ctx, denomID, tokenID, owner)
	if err != nil {
		return err
	}

	_, err = k.IsDenomCreator(ctx, denomID, owner)
	if err != nil {
		return err
	}

	k.deleteNFT(ctx, denomID, nft)
	k.deleteOwner(ctx, denomID, tokenID, owner)
	k.decreaseSupply(ctx, denomID)

	return nil
}

func (m msgServer) EditNFT(goCtx context.Context, msg *types.MsgEditNFT) (*types.MsgEditNFTResponse, error) {
	sender, err := sdk.AccAddressFromBech32(msg.Sender)
	if err != nil {
		return nil, err
	}

	ctx := sdk.UnwrapSDKContext(goCtx)
	if err := m.Keeper.EditNFT(ctx, msg.DenomId, msg.Id,
		msg.Name,
		msg.URI,
		msg.Data,
		sender,
	); err != nil {
		return nil, err
	}

	ctx.EventManager().EmitEvents(sdk.Events{
		sdk.NewEvent(
			types.EventTypeEditNFT,
			sdk.NewAttribute(types.AttributeKeyTokenID, msg.Id),
			sdk.NewAttribute(types.AttributeKeyDenomID, msg.DenomId),
			sdk.NewAttribute(types.AttributeKeyTokenURI, msg.URI),
			sdk.NewAttribute(types.AttributeKeyOwner, msg.Sender),
		),
		sdk.NewEvent(
			sdk.EventTypeMessage,
			sdk.NewAttribute(sdk.AttributeKeyModule, types.AttributeValueCategory),
			sdk.NewAttribute(sdk.AttributeKeySender, msg.Sender),
		),
	})

	return &types.MsgEditNFTResponse{}, nil
}
