### Title
Child `ChunkExtra.proposed_split` Inherits Stale Parent Value at Shard Split Boundary, Causing `InvalidChunkHeaderShardSplit` on Child's First Chunk - (File: chain/chain/src/resharding/manager.rs)

### Summary

During dynamic resharding, `process_memtrie_resharding_storage_update` clones the parent shard's `ChunkExtra` to seed each child shard's initial `ChunkExtra`, but only resets `state_root` and `congestion_info`. The `proposed_split` field — which holds the `TrieSplit` that triggered the split — is never cleared. The child shard therefore starts epoch N+2 with `ChunkExtra.proposed_split = Some(parent_split)`. The child's first chunk producer calls `compute_proposed_split`, which returns `None` (cooldown blocks re-splitting). The chunk-header validation then compares `prev_chunk_extra.proposed_split() = Some(...)` against `chunk_header.proposed_split() = None`, producing `InvalidChunkHeaderShardSplit` and rejecting every first chunk of every child shard after a dynamic split.

### Finding Description

In `process_memtrie_resharding_storage_update` (`chain/chain/src/resharding/manager.rs`), the child `ChunkExtra` is constructed by cloning the parent and patching only two fields:

```rust
let mut child_chunk_extra = ChunkExtra::clone(&parent_chunk_extra);
*child_chunk_extra.state_root_mut() = trie_changes.new_root;
*child_chunk_extra.congestion_info_mut() = child_congestion_info;
// proposed_split is NOT reset — inherits parent's Some(TrieSplit{...})
```

The parent's last chunk (the resharding block, last block of epoch N) has `proposed_split = Some(TrieSplit { boundary_account, left_memory, right_memory })` because `compute_proposed_split` set it near the epoch boundary. This value is cloned verbatim into both child `ChunkExtra` records and persisted to the store.

When the child shard's first chunk is produced in epoch N+2, `compute_proposed_split` returns `None` because `can_reshard` enforces `epoch_height - last_resharding >= min_epochs_between_resharding` (which is `>= 1`). The chunk header therefore carries `proposed_split = None`.

`validate_chunk_with_chunk_extra_and_receipts_root` then executes:

```rust
if prev_chunk_extra.proposed_split() != chunk_header.proposed_split() {
    return Err(Error::InvalidChunkHeaderShardSplit(...));
}
```

`Some(parent_split) != None` is `true`, so every validator rejects the child shard's first chunk with `InvalidChunkHeaderShardSplit`.

The project's own architecture document acknowledges this at `docs/architecture/how/dynamic_resharding.md` (TODO #10):
> `chain/chain/src/resharding/manager.rs:249` — The resharding manager doesn't set all `ChunkExtra` fields (notably the new `proposed_split` field).

### Impact Explanation

**High.** Every child shard produced by a dynamic split will have its first chunk rejected by all validators. Because the child shard's `ChunkExtra` at the resharding block is the only `prev_chunk_extra` available for the child's first chunk, and it permanently carries the stale `proposed_split`, no valid first chunk can ever be accepted. This halts chunk processing for all child shards after any dynamic split, effectively stalling the affected portion of the chain.

### Likelihood Explanation

**Low.** The bug is only reachable when `ProtocolFeature::DynamicResharding` is enabled (gated by protocol version) and a shard split actually occurs. No special attacker action is required — the bug fires automatically on the first dynamic split. The cooldown (`min_epochs_between_resharding > 0`) does not mitigate it; it only prevents a second split, not the stale-field mismatch.

### Recommendation

In `process_memtrie_resharding_storage_update`, after cloning the parent `ChunkExtra`, explicitly clear `proposed_split` on the child:

```diff
  let mut child_chunk_extra = ChunkExtra::clone(&parent_chunk_extra);
  *child_chunk_extra.state_root_mut() = trie_changes.new_root;
  *child_chunk_extra.congestion_info_mut() = child_congestion_info;
+ *child_chunk_extra.proposed_split_mut() = None;
```

A `proposed_split_mut()` accessor analogous to the existing `state_root_mut()` and `congestion_info_mut()` must be added to `ChunkExtra` in `core/primitives/src/types.rs`.

### Proof of Concept

1. Enable `ProtocolFeature::DynamicResharding` (set `min_epochs_between_resharding = 1`, add a shard to `force_split_shards`).
2. Run the chain until epoch N ends with a shard split. The resharding block's parent `ChunkExtra` has `proposed_split = Some(TrieSplit{...})`.
3. `process_memtrie_resharding_storage_update` clones it into both child `ChunkExtra` records without clearing `proposed_split`.
4. Epoch N+2 begins. The child shard's chunk producer calls `compute_proposed_split` → `can_reshard` returns `false` → returns `None` → chunk header has `proposed_split = None`.
5. `validate_chunk_with_chunk_extra_and_receipts_root` compares `Some(TrieSplit{...})` (from child `ChunkExtra`) against `None` (from chunk header) → `InvalidChunkHeaderShardSplit` → chunk rejected.
6. All validators reject the child shard's first chunk; the child shard never advances past its first block.
