### Title
Non-Canonical Backref Path Encoding Accepted by Deserializer but Rejected by Canonical Check â€” (`src/serde/parse_atom.rs`, `src/serde/tools.rs`)

### Summary

`node_from_bytes_backrefs` accepts backref path atoms with non-minimal length prefixes (e.g., `0x81 0x01` for path `[0x01]`), while `is_canonical_serialization` correctly rejects them. This breaks the invariant that every accepted byte stream must be canonical, allowing two distinct byte streams to deserialize to the same consensus tree.

### Finding Description

**Root cause:** `parse_path` delegates to `parse_atom_ptr`, which calls `decode_size` to read the length prefix but performs **no canonicality check** on the encoding. [1](#0-0) 

`parse_atom_ptr` simply decodes the size and reads the bytes â€” it never validates that the length prefix is minimal: [2](#0-1) 

By contrast, `is_canonical_atom` in `tools.rs` explicitly checks that a 1-byte-length-prefixed atom (`prefix_len == 1`) has a body byte `>= 0x80` (i.e., the value could not have been encoded as a single byte): [3](#0-2) 

And `is_canonical_serialization` applies this check to backref path atoms: [4](#0-3)

### Impact Explanation

The one-canonical-stream-per-tree invariant is broken. Any code path that calls `node_from_bytes_backrefs` without first gating on `is_canonical_serialization` will silently accept non-canonical encodings. If the Chia full node or mempool accepts serialized programs via `node_from_bytes_backrefs` without the canonical check, an attacker can submit a program in non-canonical form that deserializes identically to a canonical one â€” potentially causing consensus splits between nodes that enforce canonicality and those that do not, or enabling transaction malleability at the serialization layer.

### Likelihood Explanation

The bug is locally testable and requires only crafting a byte stream with `0x81 0x01` instead of `0x01` as a backref path. No special privileges are needed. The attacker-controlled entry point is any API that accepts serialized CLVM bytes and passes them to `node_from_bytes_backrefs`.

### Recommendation

Add a canonicality check inside `parse_path` (or `parse_atom_ptr` when called from the backref path). Specifically, after decoding the path atom, verify that the length prefix is minimal â€” i.e., that a single-byte-prefix atom has a body byte `>= 0x80`, and that the atom length meets the `min_value` threshold for its prefix length, mirroring the logic already present in `is_canonical_atom`. [5](#0-4)

fn parse_atom_ptr<'a>(f: &'a mut Cursor<&[u8]>, first_byte: u8) -> Result<&'a [u8]> {
    let blob = if first_byte <= MAX_SINGLE_BYTE {
        let pos = f.position() as usize;
        &f.get_ref()[pos - 1..pos]
    } else {
        let blob_size = decode_size(f, first_byte)?;
        let pos = f.position() as usize;
        if f.get_ref().len() < pos + blob_size as usize {
            return Err(EvalErr::SerializationError);
        }
        f.seek(SeekFrom::Current(blob_size as i64))?;
        &f.get_ref()[pos..(pos + blob_size as usize)]
    };
    Ok(blob)
}
