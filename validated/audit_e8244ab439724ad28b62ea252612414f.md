### Title
P2P Mempool Rejects Valid `AllResources` Transactions Whose L2/L1-Data Gas Bounds Are Zero Due to Asymmetric Protobuf Deserialization - (File: `crates/apollo_protobuf/src/converters/transaction.rs`)

---

### Summary

There are two distinct `TryFrom<protobuf::ResourceBounds>` implementations that apply different canonicalization rules to the same wire bytes. The block-sync path silently collapses `AllResources { l2_gas: zero, l1_data_gas: zero }` into `L1Gas`, while the mempool P2P path subsequently rejects any transaction that arrives as `L1Gas` because `RpcInvokeTransactionV3` requires `AllResources`. A transaction accepted by the gateway with `AllResourceBounds { l1_gas: X, l2_gas: {0,0}, l1_data_gas: {0,0} }` is therefore permanently rejected by every P2P peer that tries to admit it through the mempool propagation path.

---

### Finding Description

**Two divergent deserialization paths for the same protobuf message**

`crates/apollo_protobuf/src/converters/transaction.rs` implements `TryFrom<protobuf::ResourceBounds> for ValidResourceBounds`:

```rust
// TODO(Shahak): Assert data gas is not none once we remove support for 0.13.2.
let l1_data_gas = value.l1_data_gas.unwrap_or_default();
// ...
Ok(if l1_data_gas.is_zero() && l2_gas.is_zero() {
    ValidResourceBounds::L1Gas(l1_gas)   // ← collapses AllResources to L1Gas
} else {
    ValidResourceBounds::AllResources(AllResourceBounds { l1_gas, l2_gas, l1_data_gas })
})
``` [1](#0-0) 

`crates/apollo_protobuf/src/converters/rpc_transaction.rs` implements a separate `TryFrom<protobuf::ResourceBounds> for AllResourceBounds` that is strict:

```rust
l1_data_gas: value
    .l1_data_gas
    .ok_or(missing("ResourceBounds::l1_data_gas"))?
    .try_into()?,
``` [2](#0-1) 

**The mempool P2P deserialization chain**

`TryFrom<protobuf::InvokeV3WithProof> for RpcInvokeTransactionV3` first converts the inner `InvokeV3` to `InvokeTransactionV3` (using the `ValidResourceBounds` path with `unwrap_or_default`), then calls `snapi_invoke.try_into()` to reach `RpcInvokeTransactionV3`: [3](#0-2) 

`TryFrom<InvokeTransactionV3> for RpcInvokeTransactionV3` rejects anything that is not `AllResources`:

```rust
resource_bounds: match value.resource_bounds {
    ValidResourceBounds::AllResources(bounds) => bounds,
    _ => {
        return Err(DEPRECATED_RESOURCE_BOUNDS_ERROR);
    }
},
``` [4](#0-3) [5](#0-4) 

**The gateway accepts the transaction with `AllResources`**

`RpcInvokeTransactionV3` carries `resource_bounds: AllResourceBounds` (not `ValidResourceBounds`), so the gateway always receives and stores the transaction as `AllResources`. The hash is computed via `get_tip_resource_bounds_hash`, which includes `l1_data_gas` in the Poseidon preimage for `AllResources` but omits it for `L1Gas`: [6](#0-5) [7](#0-6) 

**The collapse is irreversible**

When the originating node serializes the transaction for P2P broadcast, `From<ValidResourceBounds> for protobuf::ResourceBounds` emits `l1_data_gas = Some(zero)` for `AllResources` with zero data-gas: [8](#0-7) 

On the receiving node, `ValidResourceBounds::try_from` sees `l1_data_gas.is_zero() && l2_gas.is_zero()` and collapses back to `L1Gas`. The subsequent `TryFrom<InvokeTransactionV3> for RpcInvokeTransactionV3` then returns `DEPRECATED_RESOURCE_BOUNDS_ERROR`, and the transaction is dropped.

---

### Impact Explanation

A transaction submitted via the HTTP gateway with `AllResourceBounds { l1_gas: X, l2_gas: {0,0}, l1_data_gas: {0,0} }` is accepted, assigned a hash, and placed in the local mempool. When the originating node broadcasts it to peers via the mempool P2P path, every peer's deserialization collapses the resource bounds to `L1Gas` and then rejects the transaction. The transaction is permanently stuck on the originating node and cannot propagate to the rest of the network, preventing it from being sequenced by any other proposer.

This matches: **High — Mempool/gateway/RPC admission rejects valid transactions before sequencing.**

---

### Likelihood Explanation

Any user who submits a V3 invoke transaction with `l2_gas.max_price_per_unit = 0` and `l1_data_gas.max_price_per_unit = 0` (a plausible configuration for a transaction that only pays L1 gas) triggers this path. The gateway imposes no check preventing such a submission. The collapse condition (`l1_data_gas.is_zero() && l2_gas.is_zero()`) is a silent, non-error code path, so neither the sender nor the originating node receives any indication that propagation will fail.

---

### Recommendation

Remove the collapsing heuristic from `TryFrom<protobuf::ResourceBounds> for ValidResourceBounds`. Once support for pre-0.13.3 transactions is dropped (as the `TODO(Shahak)` comment already anticipates), `l1_data_gas` should be required and the result should always be `AllResources`. Until then, the collapse should only apply when `l1_data_gas` is **absent** from the wire (i.e., `value.l1_data_gas.is_none()`), not when it is present but zero:

```rust
let l1_data_gas = value.l1_data_gas.unwrap_or_default();
let l1_data_gas: ResourceBounds = l1_data_gas.try_into()?;
// Only collapse to L1Gas when l1_data_gas was absent (legacy wire format).
Ok(if value.l1_data_gas.is_none() && l2_gas.is_zero() {
    ValidResourceBounds::L1Gas(l1_gas)
} else {
    ValidResourceBounds::AllResources(AllResourceBounds { l1_gas, l2_gas, l1_data_gas })
})
```

This preserves backward compatibility with old peers that omit `l1_data_gas` while correctly round-tripping modern transactions that explicitly set it to zero.

---

### Proof of Concept

1. Submit a V3 invoke transaction to the gateway with:
   ```json
   "resource_bounds": {
     "l1_gas":      { "max_amount": "0x1000", "max_price_per_unit": "0x1" },
     "l2_gas":      { "max_amount": "0x0",    "max_price_per_unit": "0x0" },
     "l1_data_gas": { "max_amount": "0x0",    "max_price_per_unit": "0x0" }
   }
   ```
   The gateway accepts it and stores `tx_hash = H_allresources` (Poseidon over `[INVOKE, 3, sender, hash(tip, l1_gas, l2_gas, l1_data_gas), ...]`).

2. The originating node serializes the transaction for P2P broadcast. `From<ValidResourceBounds> for protobuf::ResourceBounds` emits `l1_data_gas = Some({max_amount:0, max_price_per_unit:0})`. [8](#0-7) 

3. A receiving peer calls `TryFrom<protobuf::MempoolTransaction> for RpcTransaction` → `TryFrom<protobuf::InvokeV3WithProof> for RpcInvokeTransactionV3`. [9](#0-8) 

4. Inside that conversion, `ValidResourceBounds::try_from` evaluates `l1_data_gas.is_zero() && l2_gas.is_zero()` → `true` → returns `ValidResourceBounds::L1Gas(X)`. [10](#0-9) 

5. `TryFrom<InvokeTransactionV3> for RpcInvokeTransactionV3` matches `ValidResourceBounds::L1Gas` against the `_ => return Err(DEPRECATED_RESOURCE_BOUNDS_ERROR)` arm. The transaction is rejected. [11](#0-10) 

6. The transaction never enters the peer's mempool and cannot be sequenced by any node other than the originator.

impl TryFrom<protobuf::InvokeV3WithProof> for RpcInvokeTransactionV3 {
    type Error = ProtobufConversionError;
    fn try_from(mut value: protobuf::InvokeV3WithProof) -> Result<Self, Self::Error> {
        // Extract proof first, since `starknet_api::transaction::InvokeTransactionV3` does not
        // carry a `proof` field.
        let proof = Proof::from(std::mem::take(&mut value.proof));

        let snapi_invoke: InvokeTransactionV3 = value
            .invoke
            .ok_or(ProtobufConversionError::MissingField {
                field_description: "InvokeV3WithProof::invoke",
            })?
            .try_into()?;

        // This conversion can fail only if the resource_bounds are not AllResources.
        Ok(Self { proof, ..snapi_invoke.try_into().map_err(|_| DEPRECATED_RESOURCE_BOUNDS_ERROR)? })
    }

// An implementation of the SNIP: https://github.com/EvyatarO/SNIPs/blob/snip-8/SNIPS/snip-8.md
pub fn get_tip_resource_bounds_hash(
    resource_bounds: &ValidResourceBounds,
    tip: &Tip,
) -> Result<Felt, StarknetApiError> {
    let l1_resource_bounds = resource_bounds.get_l1_bounds();
    let l2_resource_bounds = resource_bounds.get_l2_bounds();

    // L1 and L2 gas bounds always exist.
    // Old V3 txs always have L2 gas bounds of zero, but they exist.
    let mut resource_felts = vec![
        get_concat_resource(&l1_resource_bounds, L1_GAS)?,
        get_concat_resource(&l2_resource_bounds, L2_GAS)?,
    ];

    // For new V3 txs, need to also hash the data gas bounds.
    resource_felts.extend(match resource_bounds {
        ValidResourceBounds::L1Gas(_) => vec![],
        ValidResourceBounds::AllResources(all_resources) => {
            vec![get_concat_resource(&all_resources.l1_data_gas, L1_DATA_GAS)?]
        }
    });

    Ok(HashChain::new().chain(&tip.0.into()).chain_iter(resource_felts.iter()).get_poseidon_hash())
}
