### Title
Protobuf Round-Trip Lossy Conversion of `AllResources` to `L1Gas` Silently Drops Valid Transactions from P2P and Consensus - (File: crates/apollo_protobuf/src/converters/transaction.rs)

### Summary

A valid V3 invoke transaction with `AllResourceBounds` where `l2_gas = 0` and `l1_data_gas = 0` (only `l1_gas` non-zero) is accepted by the gateway but is permanently rejected by every peer node when propagated via mempool P2P or included in a consensus proposal. The root cause is a strict-equality zero-check in the protobuf `ValidResourceBounds` deserializer that silently collapses `AllResources` into `L1Gas`, after which the mandatory `AllResources` requirement in `RpcInvokeTransactionV3` causes a hard conversion failure.

### Finding Description

**Step 1 – Serialization (lossless).**
`RpcInvokeTransactionV3` carries `resource_bounds: AllResourceBounds`. When serialized to protobuf it is first converted to `InvokeTransactionV3` via:

```rust
// crates/starknet_api/src/rpc_transaction.rs:568-583
impl From<RpcInvokeTransactionV3> for InvokeTransactionV3 {
    fn from(tx: RpcInvokeTransactionV3) -> Self {
        Self {
            resource_bounds: ValidResourceBounds::AllResources(tx.resource_bounds),
            ...
        }
    }
}
```

`ValidResourceBounds::AllResources` is then serialized to `protobuf::ResourceBounds` with all three fields present (even if zero):

```rust
// crates/apollo_protobuf/src/converters/transaction.rs:479-488
ValidResourceBounds::AllResources(AllResourceBounds { l1_gas, l2_gas, l1_data_gas }) =>
    protobuf::ResourceBounds {
        l1_gas: Some(l1_gas.into()),
        l2_gas: Some(l2_gas.into()),
        l1_data_gas: Some(l1_data_gas.into()),
    },
```

**Step 2 – Deserialization (lossy, the bug).**
On the receiving node, `protobuf::InvokeV3` is deserialized to `InvokeTransactionV3` via:

```rust
// crates/apollo_protobuf/src/converters/transaction.rs:596-598
let resource_bounds = ValidResourceBounds::try_from(
    value.resource_bounds.ok_or(missing("InvokeV3::resource_bounds"))?,
)?;
```

`ValidResourceBounds::try_from(protobuf::ResourceBounds)` applies a strict equality check:

```rust
// crates/apollo_protobuf/src/converters/transaction.rs:431-435
Ok(if l1_data_gas.is_zero() && l2_gas.is_zero() {
    ValidResourceBounds::L1Gas(l1_gas)          // ← wrong variant chosen
} else {
    ValidResourceBounds::AllResources(...)
})
```

When `l2_gas = 0` and `l1_data_gas = 0`, the deserializer produces `ValidResourceBounds::L1Gas` even though the wire bytes came from an `AllResources` transaction.

**Step 3 – Conversion failure.**
The deserialized `InvokeTransactionV3` (now carrying `L1Gas`) is then converted back to `RpcInvokeTransactionV3`:

```rust
// crates/starknet_api/src/rpc_transaction.rs:591-597
resource_bounds: match value.resource_bounds {
    ValidResourceBounds::AllResources(bounds) => bounds,
    _ => {
        return Err(StarknetApiError::OutOfRange { string: "resource_bounds".to_string() });
    }
},
```

This hard-fails. The error is surfaced as `DEPRECATED_RESOURCE_BOUNDS_ERROR` in both the mempool P2P path:

```rust
// crates/apollo_protobuf/src/converters/rpc_transaction.rs:115-131
impl TryFrom<protobuf::InvokeV3WithProof> for RpcInvokeTransactionV3 {
    fn try_from(mut value: protobuf::InvokeV3WithProof) -> Result<Self, Self::Error> {
        let snapi_invoke: InvokeTransactionV3 = value.invoke...try_into()?;
        Ok(Self { proof, ..snapi_invoke.try_into().map_err(|_| DEPRECATED_RESOURCE_BOUNDS_ERROR)? })
    }
}
```

and the consensus path:

```rust
// crates/apollo_protobuf/src/converters/transaction.rs:1042-1045
protobuf::consensus_transaction::Txn::InvokeV3(txn) => {
    ConsensusTransaction::RpcTransaction(RpcTransaction::Invoke(
        RpcInvokeTransaction::V3(txn.try_into()?),  // fails here
    ))
}
```

Note that a parallel `TryFrom<protobuf::ResourceBounds> for AllResourceBounds` exists in `crates/apollo_protobuf/src/converters/rpc_transaction.rs` (lines 212–224) that correctly preserves all three fields without the zero-check, but it is **never reached** in either propagation path because both paths route through `InvokeTransactionV3` first.

### Impact Explanation

Any valid V3 invoke transaction with `l2_gas = 0` and `l1_data_gas = 0` (e.g., a pure L1-gas-only transaction, which the gateway explicitly accepts per its test suite) is silently dropped by every peer node. In a multi-sequencer deployment:

- **Mempool P2P**: the transaction cannot propagate beyond the originating node; it is permanently stuck.
- **Consensus proposals**: if the originating sequencer is the proposer and includes such a transaction in a proposal, every other validator rejects the entire proposal deserialization, causing a consensus round failure.

This matches the allowed impact: **High – Mempool/gateway/RPC admission rejects valid transactions before sequencing** and **High – Transaction conversion logic binds the wrong type or executable payload**.

### Likelihood Explanation

The gateway's stateless validator explicitly accepts transactions with only `l1_gas` non-zero (test cases `valid_l1_gas` in `stateless_transaction_validator_test.rs`). Any unprivileged user can craft such a transaction. The failure is deterministic and reproducible on every peer node receiving the protobuf-encoded form.

### Recommendation

In `TryFrom<protobuf::ResourceBounds> for ValidResourceBounds` (`crates/apollo_protobuf/src/converters/transaction.rs`), remove the zero-equality heuristic and instead always produce `AllResources` when all three fields are present in the wire message. Reserve `L1Gas` only for the legacy case where `l1_data_gas` is absent (`None`) in the protobuf, which is the actual backward-compatibility signal (as the existing `TODO(Shahak)` comment acknowledges):

```rust
// Proposed fix
Ok(if value.l1_data_gas.is_none() {
    ValidResourceBounds::L1Gas(l1_gas)
} else {
    ValidResourceBounds::AllResources(AllResourceBounds { l1_gas, l2_gas, l1_data_gas })
})
```

### Proof of Concept

1. Submit a V3 invoke transaction via RPC with `AllResourceBounds { l1_gas: <non-zero>, l2_gas: 0, l1_data_gas: 0 }`. The gateway accepts it.
2. The mempool P2P propagator calls `From<RpcTransaction> for protobuf::MempoolTransaction`, which serializes `l2_gas = 0` and `l1_data_gas = 0` as explicit zero `ResourceLimits` fields.
3. A peer node receives the bytes and calls `RpcTransaction::try_from(protobuf::MempoolTransaction)` → `RpcInvokeTransactionV3::try_from(protobuf::InvokeV3WithProof)` → `InvokeTransactionV3::try_from(protobuf::InvokeV3)`.
4. Inside step 3, `ValidResourceBounds::try_from(protobuf::ResourceBounds)` fires the `l1_data_gas.is_zero() && l2_gas.is_zero()` branch and returns `ValidResourceBounds::L1Gas`.
5. `RpcInvokeTransactionV3::try_from(InvokeTransactionV3)` hits the `_ => Err(OutOfRange)` arm and returns `DEPRECATED_RESOURCE_BOUNDS_ERROR`.
6. The peer node drops the transaction. The same failure occurs for any consensus proposal containing this transaction. [1](#0-0) [2](#0-1) [3](#0-2) [4](#0-3) [5](#0-4) [6](#0-5)

    }

// An implementation of the SNIP: https://github.com/EvyatarO/SNIPs/blob/snip-8/SNIPS/snip-8.md
pub fn get_tip_resource_bounds_hash(
    resource_bounds: &ValidResourceBounds,
    tip: &Tip,
) -> Result<Felt, StarknetApiError> {
    let l1_resource_bounds = resource_bounds.get_l1_bounds();
    let l2_resource_bounds = resource_bounds.get_l2_bounds();

    // L1 and L2 gas bounds always exist.
    // Old V3 txs always have L2 gas bounds of zero, but they exist.
    let mut resource_felts = vec![
        get_concat_resource(&l1_resource_bounds, L1_GAS)?,
        get_concat_resource(&l2_resource_bounds, L2_GAS)?,
    ];

    // For new V3 txs, need to also hash the data gas bounds.
    resource_felts.extend(match resource_bounds {
        ValidResourceBounds::L1Gas(_) => vec![],
        ValidResourceBounds::AllResources(all_resources) => {
            vec![get_concat_resource(&all_resources.l1_data_gas, L1_DATA_GAS)?]
        }
    });

    Ok(HashChain::new().chain(&tip.0.into()).chain_iter(resource_felts.iter()).get_poseidon_hash())
}
