### Title
`ContractCodeResponse::V1` Accepted Without Sender Identity Check Pre-`SignedContractCodeResponse` Feature — (File: `chain/client/src/stateless_validation/validate.rs`)

### Summary

`validate_contract_code_response` skips all sender-identity verification when `ProtocolFeature::SignedContractCodeResponse` is not yet active for the epoch referenced in the response. Because `ContractCodeResponse::V1` carries no responder field and `verify_signature` unconditionally returns `false` for it, any network peer can send a `V1` response carrying arbitrary contract bytes for any relevant `ChunkProductionKey` and have those bytes accepted and stored by chunk validators without any check against the epoch manager's authoritative chunk-producer list.

### Finding Description

`ContractCodeResponse` is a versioned enum with two Borsh-discriminated variants:

- `V1` (discriminant `0`): unsigned, no `responder` field, no identity information.
- `V2` (discriminant `1`): signed by the responder; carries a `responder: AccountId` and a `Signature` over the inner payload. [1](#0-0) 

The receiver-side gate in `validate_contract_code_response` reads the protocol version from the epoch embedded in the response's own `ChunkProductionKey`, then conditionally calls `validate_witness_contract_code_response_signature` only when `SignedContractCodeResponse` is enabled: [2](#0-1) 

When the feature is **not** enabled, the only check performed is `validate_chunk_relevant` — a relevance window check that verifies the epoch/shard/height are known and recent, but performs **no identity check** on who sent the response. A `V1` message with a valid `ChunkProductionKey` passes straight through.

`validate_witness_contract_code_response_signature`, when it is called, verifies that `response.responder()` is a current chunk producer for the shard and that the signature over the inner payload is valid: [3](#0-2) 

For `V1`, `response.responder()` returns `None` and `verify_signature` returns `false` — but neither is ever evaluated on the pre-feature path. [4](#0-3) 

After passing validation, the handler stores the unverified contract bytes: [5](#0-4) 

### Impact Explanation

Any network peer — not just an authorized chunk producer — can craft a `ContractCodeResponse::V1` with a valid `ChunkProductionKey` referencing a pre-`SignedContractCodeResponse` epoch and arbitrary `compressed_contracts` bytes. The bytes are stored in the chunk validator's contract-code cache under that key. If the injected bytes arrive before the legitimate response, the validator may use wrong contract code during chunk state-witness validation, causing validation to fail. This is a targeted liveness attack: a single malicious peer can prevent specific chunk validators from successfully validating chunks, stalling chunk endorsement for those shards.

**Severity: High** — targeted, unprivileged liveness disruption of chunk validation; no safety (state-correctness) impact because the contract bytes are ultimately identified by hash during execution.

### Likelihood Explanation

The attack window is the epoch(s) during which the network protocol version is below 85 (the version at which `SignedContractCodeResponse` stabilized). Any peer with network access can send a `ContractCodeResponse::V1` message; no validator key or privileged role is required. The `ContractCodeResponse::V1` Borsh encoding (discriminant byte `0x00` followed by the `ChunkProductionKey` and compressed payload) is fully constructible from public protocol documentation. [6](#0-5) 

### Recommendation

Reject `ContractCodeResponse::V1` unconditionally once `SignedContractCodeResponse` is the minimum supported protocol version, or — more immediately — add a version-independent check that rejects any `V1` response when the epoch's protocol version is ≥ 85. Concretely, in `validate_contract_code_response`, add an `else` branch that rejects unsigned `V1` responses when the feature is active:

```rust
if ProtocolFeature::SignedContractCodeResponse.enabled(protocol_version) {
    validate_witness_contract_code_response_signature(epoch_manager, response)?;
} else if matches!(response, ContractCodeResponse::V2(_)) {
    // V2 sent before the feature is active: reject to prevent version confusion.
    return Err(Error::Other("V2 response before SignedContractCodeResponse feature".into()));
}
```

Additionally, once the minimum supported protocol version exceeds 85, remove the `V1` variant and the pre-feature branch entirely.

### Proof of Concept

1. Observe a `ContractCodeRequest` on the network for `ChunkProductionKey { epoch_id: E, shard_id: S, height_created: H }` where epoch `E` has protocol version < 85.
2. Craft a `ContractCodeResponse::V1` (Borsh discriminant `0x00`) with the same `ChunkProductionKey` and arbitrary `compressed_contracts` bytes (e.g., a valid zstd-compressed empty list, or a list of garbage bytes).
3. Send the message to one or more chunk validators before the legitimate chunk producer responds.
4. `validate_contract_code_response` passes: `validate_chunk_relevant` accepts the known epoch/shard/height; the `SignedContractCodeResponse` branch is skipped; `Ok(ChunkRelevance::Relevant)` is returned.
5. `handle_contract_code_response` calls `store_accessed_contract_codes(key, contracts)` with the attacker-supplied bytes.
6. The chunk validator now holds wrong contract bytes for key `(E, S, H)`. When it attempts to validate the chunk state witness using those bytes, the hash check fails and validation is rejected, preventing the validator from endorsing the chunk. [2](#0-1) [7](#0-6) [8](#0-7)

#[derive(Debug, Clone, PartialEq, Eq, BorshSerialize, BorshDeserialize, ProtocolSchema)]
#[borsh(use_discriminant = true)]
#[repr(u8)]
pub enum ContractCodeResponse {
    V1(ContractCodeResponseV1) = 0,
    V2(ContractCodeResponseV2) = 1,
}

impl ContractCodeResponse {
    pub fn encode(
        next_chunk: ChunkProductionKey,
        contracts: &Vec<CodeBytes>,
        signer: &ValidatorSigner,
        protocol_version: ProtocolVersion,
    ) -> std::io::Result<Self> {
        if ProtocolFeature::SignedContractCodeResponse.enabled(protocol_version) {
            ContractCodeResponseV2::encode(next_chunk, contracts, signer).map(Self::V2)
        } else {
            ContractCodeResponseV1::encode(next_chunk, contracts).map(Self::V1)
        }
    }

    pub fn chunk_production_key(&self) -> &ChunkProductionKey {
        match self {
            Self::V1(v1) => &v1.next_chunk,
            Self::V2(v2) => &v2.inner.next_chunk,
        }
    }

    /// Account that produced this response. Available only for signed variants.
    pub fn responder(&self) -> Option<&AccountId> {
        match self {
            Self::V1(_) => None,
            Self::V2(v2) => Some(&v2.inner.responder),
        }
    }

    pub fn decompress_contracts(&self) -> std::io::Result<Vec<CodeBytes>> {
        let compressed_contracts = match self {
            Self::V1(v1) => &v1.compressed_contracts,
            Self::V2(v2) => &v2.inner.compressed_contracts,
        };
        compressed_contracts.decode().map(|(data, _size)| data)
    }

    /// Verifies the signature for signed variants. Returns `false` for
    /// unsigned variants since there is nothing to verify.
    pub fn verify_signature(&self, public_key: &PublicKey) -> bool {
        match self {
            Self::V1(_) => false,
            Self::V2(v2) => v2.verify_signature(public_key),
        }
    }
}

pub fn validate_contract_code_response(
    epoch_manager: &dyn EpochManagerAdapter,
    response: &ContractCodeResponse,
    store: &Store,
) -> Result<ChunkRelevance, Error> {
    let key = response.chunk_production_key();
    require_relevant!(validate_chunk_relevant(epoch_manager, key, store)?);
    let protocol_version = epoch_manager.get_epoch_protocol_version(&key.epoch_id)?;
    if ProtocolFeature::SignedContractCodeResponse.enabled(protocol_version) {
        validate_witness_contract_code_response_signature(epoch_manager, response)?;
    }

    Ok(ChunkRelevance::Relevant)
}

fn validate_witness_contract_code_response_signature(
    epoch_manager: &dyn EpochManagerAdapter,
    response: &ContractCodeResponse,
) -> Result<(), Error> {
    let Some(responder) = response.responder() else {
        return Err(Error::Other(
            "Unsigned contract code response in epoch where signature is required".to_owned(),
        ));
    };
    let key = response.chunk_production_key();
    let chunk_producers =
        epoch_manager.get_epoch_chunk_producers_for_shard(&key.epoch_id, key.shard_id)?;
    if !chunk_producers.contains(responder) {
        return Err(Error::Other(format!(
            "Contract code response responder {responder} is not a chunk producer for shard {} in epoch {:?}",
            key.shard_id, key.epoch_id,
        )));
    }
    let validator = epoch_manager.get_validator_by_account_id(&key.epoch_id, responder)?;
    if !response.verify_signature(validator.public_key()) {
        return Err(Error::Other("Invalid witness contract code response signature".to_owned()));
    }
    Ok(())
