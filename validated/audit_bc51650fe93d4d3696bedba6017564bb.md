### Title
Missing Response `timestamp` Validation in Exchange Rate Oracle Allows Stale Rate to Be Silently Accepted - (File: `crates/apollo_l1_gas_price/src/exchange_rate_oracle.rs`)

### Summary
`resolve_query` in `ExchangeRateOracleClient` parses the oracle HTTP response and validates `price` and `decimals`, but never validates the `timestamp` field present in the response against the `adjusted_timestamp` that was sent in the request. A stale or mismatched oracle response is silently accepted, cached under the correct quantized key, and used to compute L1 gas prices in FRI for block proposals.

### Finding Description

`spawn_query` sends the oracle request with a specific `adjusted_timestamp`:

```rust
url.query_pairs_mut().append_pair("timestamp", &adjusted_timestamp.to_string());
```

The oracle API contract (documented by the integration test mock) returns a `timestamp` field in the response body:

```rust
// crates/apollo_integration_tests/src/utils.rs:646
json!({ "timestamp": query.timestamp, "price": price, "decimals": EXCHANGE_RATE_DECIMALS })
```

However, `resolve_query` only reads `price` and `decimals` — the `timestamp` field is never extracted or compared to `adjusted_timestamp`:

```rust
// crates/apollo_l1_gas_price/src/exchange_rate_oracle.rs:166-213
fn resolve_query(body: String, metrics: &ExchangeRateOracleMetrics) -> Result<u128, ...> {
    let price = match json.get("price").and_then(|v| v.as_str()) { ... };
    // rate != 0 check
    let decimals = match json.get("decimals").and_then(|v| v.as_u64()) { ... };
    // decimals == 18 check
    // *** NO check: json.get("timestamp") == adjusted_timestamp ***
    Ok(rate)
}
```

The result is then cached under `quantized_timestamp`:

```rust
// crates/apollo_l1_gas_price/src/exchange_rate_oracle.rs:273
cache.put(quantized_timestamp, rate);
```

If the oracle returns a response whose `timestamp` corresponds to a different time bucket (e.g., a stale cached response, a buggy implementation, or a response from a degraded endpoint), the sequencer accepts the wrong rate, stores it under the correct key, and uses it for all subsequent gas price conversions in that bucket.

### Impact Explanation

The accepted rate flows directly into L1 gas price computation:

```rust
// crates/apollo_consensus_orchestrator/src/utils.rs:171-175
let l1_gas_prices_fri_result =
    L1PricesInFri::convert_from_wei(&prices_in_wei, eth_to_fri_rate);
if let Ok(prices_in_fri) = l1_gas_prices_fri_result {
    return (prices_in_fri, prices_in_wei, eth_to_fri_rate);
}
```

Both the proposer and validator call `get_l1_prices_in_fri_and_wei` with the same block timestamp, so if both resolve the same stale rate from their respective oracle instances, the proposal passes the gas-price margin check and is committed with wrong FRI-denominated L1 gas prices. This directly affects the fee charged to every transaction in the block — matching the "Incorrect fee, gas, bouncer, resource accounting, refund, balance, or L1 gas price effect with economic impact" scope.

### Likelihood Explanation

The oracle endpoint is operator-configured, so a fully adversarial oracle requires operator compromise. However, the realistic trigger is a buggy or degraded oracle that returns a cached response for a different time bucket — a common failure mode for HTTP price feeds. Because `resolve_query` accepts any non-zero price with correct decimals, this failure is completely silent: no error is logged, no fallback is triggered, and the wrong rate is cached as authoritative for the requested bucket.

### Recommendation

Pass `adjusted_timestamp` into `resolve_query` and validate the returned `timestamp` field:

```rust
fn resolve_query(
    body: String,
    metrics: &ExchangeRateOracleMetrics,
    expected_timestamp: u64,
) -> Result<u128, ExchangeRateOracleClientError> {
    // ... existing price/decimals parsing ...

    // Add: validate response timestamp matches the requested bucket
    let response_timestamp = match json.get("timestamp").and_then(|v| v.as_u64()) {
        Some(ts) => ts,
        None => return Err(ExchangeRateOracleClientError::MissingFieldError(
            "timestamp".to_string(), body,
        )),
    };
    if response_timestamp != expected_timestamp {
        return Err(ExchangeRateOracleClientError::InvalidRateError(format!(
            "Stale oracle response: requested timestamp {expected_timestamp}, \
             got {response_timestamp}"
        )));
    }
    Ok(rate)
}
```

Update the call site in `spawn_query` to pass `adjusted_timestamp`:

```rust
let rate = resolve_query(body, &metrics, adjusted_timestamp)?;
```

### Proof of Concept

1. Configure the oracle endpoint to return `{ "timestamp": <old_bucket>, "price": "0x<stale_rate>", "decimals": 18 }` for any request.
2. `resolve_query` accepts the response — `price` is non-zero, `decimals` is 18, no timestamp check exists.
3. The stale `rate` is stored in `cached_prices` under the current `quantized_timestamp`.
4. `get_l1_prices_in_fri_and_wei` uses this rate to convert WEI prices to FRI.
5. Both proposer and validator compute the same wrong FRI gas prices; the proposal passes the margin check and is committed.

```rust
fn resolve_query(
    body: String,
    metrics: &ExchangeRateOracleMetrics,
) -> Result<u128, ExchangeRateOracleClientError> {
    let Ok(json): Result<serde_json::Value, _> = serde_json::from_str(&body) else {
        return Err(ExchangeRateOracleClientError::ParseError(format!(
            "Failed to parse JSON: {body}"
        )));
    };
    // Extract price from API response. Also returns MissingFieldError if value is not a string.
    let price = match json.get("price").and_then(|v| v.as_str()) {
        Some(price) => price,
        None => {
            return Err(ExchangeRateOracleClientError::MissingFieldError(
                "price".to_string(),
                body,
            ));
        }
    };
    let rate = u128::from_str_radix(price.trim_start_matches("0x"), 16).map_err(|e| {
        ExchangeRateOracleClientError::ParseError(format!("Failed to parse price {price}: {e}"))
    })?;
    if rate == 0 {
        return Err(ExchangeRateOracleClientError::InvalidRateError(
            "rate must be non-zero".to_string(),
        ));
    }
    // Extract decimals from API response. Also returns MissingFieldError if value is not a number.
    let decimals = match json.get("decimals").and_then(|v| v.as_u64()) {
        Some(decimals) => decimals,
        None => {
            return Err(ExchangeRateOracleClientError::MissingFieldError(
                "decimals".to_string(),
                body,
            ));
        }
    };
    if decimals != EXCHANGE_RATE_DECIMALS {
        return Err(ExchangeRateOracleClientError::InvalidDecimalsError(
            EXCHANGE_RATE_DECIMALS,
            decimals,
        ));
    }
    metrics.success_count.increment(1);
    set_unix_now_seconds(metrics.last_success_timestamp);
    metrics.rate.set_lossy(rate);
    Ok(rate)
}
```
