### Title
Missing `chain_id` Non-Zero/Non-Empty Validation in Node Config Allows Wrong Transaction Hash Domain — (`crates/apollo_node_config/src/node_config.rs`, `crates/blockifier/src/context.rs`, `crates/native_blockifier/src/py_block_executor.rs`)

---

### Summary

The sequencer's `validate_node_config()` function performs extensive cross-component validation but never checks that `chain_id` is a valid, non-empty, non-zero value. Two separate `Default` implementations supply a zero/empty `chain_id` as a fallback, and the production deployment template ships with `"chain_id": ""`. If a node is started with this unchecked default, every transaction hash is computed with `chain_id = Felt::ZERO`, breaking the signature domain and enabling cross-chain replay.

---

### Finding Description

**Default values that produce a zero chain_id:**

`ChainInfo::default()` in `crates/blockifier/src/context.rs` sets:

```rust
chain_id: ChainId::Other("0x0".to_string()),
```

with an open TODO: *"should we remove the default value for chain_id?"* [1](#0-0) 

`PyOsConfig::default()` in `crates/native_blockifier/src/py_block_executor.rs` sets:

```rust
chain_id: ChainId::Other("".to_string()),
``` [2](#0-1) 

**Deployment template ships with empty chain_id:**

The production deployment config `crates/apollo_deployments/resources/app_configs/general_config.json` ships with:

```json
"chain_id": "",
``` [3](#0-2) 

The config schema marks `chain_id` as `"TemporaryValue"` / `"PointerTarget"`, meaning it is expected to be filled in by deployment scripts. If a deployment script omits or misconfigures this pointer, the node starts with `chain_id = ""`. [4](#0-3) 

**No validation gate in `validate_node_config()`:**

`SequencerNodeConfig::validate_node_config()` calls `config_validate(self)` (which runs `#[derive(Validate)]` annotations) and `cross_member_validations()`. Neither checks that `chain_id` is non-empty or non-zero. The `cross_member_validations()` function validates dozens of component-config pairings, idle-detection timing, and `config_manager` locality, but has no `chain_id` guard. [5](#0-4) 

**How the zero chain_id propagates into transaction hashes:**

`GatewayStaticConfig::default()` uses `chain_info: ChainInfo::default()`, which carries the `"0x0"` chain_id into the gateway. [6](#0-5) 

Every transaction hash function (`get_declare_transaction_v3_hash`, `get_invoke_transaction_v3_hash`, `get_deploy_account_transaction_v3_hash`, etc.) embeds `chain_id` via `Felt::try_from(chain_id)?`. For `ChainId::Other("")` or `ChainId::Other("0x0")`, this resolves to `Felt::ZERO`. [7](#0-6) 

`InternalRpcTransactionWithoutTxHash::calculate_transaction_hash` is called during RPC→Internal conversion in `convert_rpc_tx_to_internal`, using `self.chain_id` from the converter's config. [8](#0-7) 

---

### Impact Explanation

When `chain_id = Felt::ZERO` is used as the hash domain:

1. **Wrong transaction hashes are stored in the DB and emitted in receipts.** Every `InternalRpcTransaction.tx_hash` is computed over the wrong domain, so the stored hash diverges from what the Starknet protocol expects.
2. **Signature verification accepts transactions signed for chain_id = 0.** An account's `__validate__` receives `tx_info.transaction_hash` computed with `chain_id = 0`. Any wallet that signs a transaction for a network whose chain_id encodes to `Felt::ZERO` (or an attacker who deliberately signs for `chain_id = 0`) will pass validation on this misconfigured node.
3. **Legitimate mainnet/sepolia transactions are rejected.** Their signatures were computed with the real chain_id, so the hash mismatch causes `__validate__` to fail.

This matches: **High — Mempool/gateway/RPC admission accepts invalid transactions or rejects valid transactions before sequencing**, and **Critical — Wrong state, receipt, or revert result from blockifier/syscall/execution logic for accepted input**.

---

### Likelihood Explanation

The deployment template `general_config.json` ships with `"chain_id": ""` as a placeholder. The config schema marks it `"TemporaryValue"` / `"PointerTarget"`, relying entirely on deployment scripts to fill it in. A single scripting error, a missing overlay, or a new deployment topology that omits the pointer resolution leaves the node running with `chain_id = ""`. The `validate_node_config()` function, which is the only runtime gate, does not catch this. The open TODO comment in `ChainInfo::default()` confirms the team is aware the default is unsafe but has not yet added a guard.

---

### Recommendation

**Short term:** Add a `chain_id` non-empty/non-zero check inside `cross_member_validations()` (or as a `#[validate]` annotation on the `chain_id` field):

```rust
if self.chain_id.as_hex() == "0x0" || self.chain_id.to_string().is_empty() {
    return Err(ConfigError::ComponentConfigMismatch {
        component_config_mismatch: "chain_id must not be zero or empty".to_string(),
    });
}
```

Remove or gate the `ChainId::Other("0x0")` and `ChainId::Other("")` defaults so they cannot silently propagate into production paths.

**Long term:** Replace the `"PointerTarget"` pattern for `chain_id` with a required field that has no default, forcing every deployment to supply an explicit value and failing fast at config load time rather than at transaction-hash time.

---

### Proof of Concept

1. Start the sequencer node without overriding the `chain_id` pointer (use `general_config.json` as-is). `validate_node_config()` succeeds.
2. Submit an `InvokeV3` transaction whose signature was computed with `chain_id = Felt::ZERO` (i.e., `ChainId::Other("")`).
3. `convert_rpc_tx_to_internal` calls `tx_without_hash.calculate_transaction_hash(&self.chain_id)` with `chain_id = ChainId::Other("")`, producing `tx_hash = H(... || Felt::ZERO)`.
4. The account's `__validate__` receives this hash; the signature matches because the wallet also used `chain_id = 0`.
5. The transaction is accepted, stored with the wrong hash, and executed — while a legitimate mainnet transaction with the same nonce and correct chain_id is rejected. [9](#0-8) [10](#0-9)

    pub fn validate_node_config(&self) -> Result<(), ConfigError> {
        // Validate each config member using its `Validate` trait derivation.
        config_validate(self)?;

        // Custom cross member validations.
        self.cross_member_validations()

    fn cross_member_validations(&self) -> Result<(), ConfigError> {
        macro_rules! validate_component_config_is_set_iff_running_locally {
            ($component_field:ident, $config_field:ident) => {{
                // The component config should be set iff its running locally.
                if self.components.$component_field.is_running_locally()
                    != self.$config_field.is_some()
                {
                    let execution_mode = &self.components.$component_field.execution_mode;
                    let component_config_availability =
                        if self.$config_field.is_some() { "available" } else { "not available" };
                    return Err(ConfigError::ComponentConfigMismatch {
                        component_config_mismatch: format!(
                            "{} component configs mismatch: execution mode {:?} while config is {}",
                            stringify!($component_field),
                            execution_mode,
                            component_config_availability
                        ),
                    });
                }
            }};
        }

        // TODO(Tsabary): should be based on iteration of `ComponentConfig` fields.
        validate_component_config_is_set_iff_running_locally!(batcher, batcher_config);
        validate_component_config_is_set_iff_running_locally!(class_manager, class_manager_config);
        validate_component_config_is_set_iff_running_locally!(committer, committer_config);
        validate_component_config_is_set_iff_running_locally!(
            config_manager,
            config_manager_config
        );
        validate_component_config_is_set_iff_running_locally!(
            consensus_manager,
            consensus_manager_config
        );
        validate_component_config_is_set_iff_running_locally!(gateway, gateway_config);
        validate_component_config_is_set_iff_running_locally!(http_server, http_server_config);
        validate_component_config_is_set_iff_running_locally!(
            l1_gas_price_provider,
            l1_gas_price_provider_config
        );
        validate_component_config_is_set_iff_running_locally!(
            l1_gas_price_scraper,
            l1_gas_price_scraper_config
        );
        validate_component_config_is_set_iff_running_locally!(
            l1_events_provider,
            l1_events_provider_config
        );
        validate_component_config_is_set_iff_running_locally!(
            l1_events_scraper,
            l1_events_scraper_config
        );
        validate_component_config_is_set_iff_running_locally!(mempool, mempool_config);
        validate_component_config_is_set_iff_running_locally!(mempool_p2p, mempool_p2p_config);
        validate_component_config_is_set_iff_running_locally!(
            monitoring_endpoint,
            monitoring_endpoint_config
        );
        validate_component_config_is_set_iff_running_locally!(proof_manager, proof_manager_config);
        validate_component_config_is_set_iff_running_locally!(
            sierra_compiler,
            sierra_compiler_config
        );
        validate_component_config_is_set_iff_running_locally!(state_sync, state_sync_config);

        // The config manager is a local infrastructure component: every node runs its own instance
        // and its consumers (e.g. the mempool) reach it over an in-process channel. Allowing it to
        // run remotely would turn the per-request `get_*_dynamic_config` calls into network RPCs,
        // where a transient failure would crash the consuming component. Enforce here that it is
        // never exposed over (or reached over) the network so that client is always local.
        match self.components.config_manager.execution_mode {
            ReactiveComponentExecutionMode::Remote
            | ReactiveComponentExecutionMode::LocalExecutionWithRemoteEnabled => {
                return Err(ConfigError::ComponentConfigMismatch {
                    component_config_mismatch: format!(
                        "config_manager must run locally without a remote server (execution mode \
                         must be Disabled or LocalExecutionWithRemoteDisabled), but is {:?}. It \
                         is a local infrastructure component and must be co-located with its \
                         consumers.",
                        self.components.config_manager.execution_mode
                    ),
                });
            }
            ReactiveComponentExecutionMode::LocalExecutionWithRemoteDisabled
            | ReactiveComponentExecutionMode::Disabled => {}
        }

        // Validate proposer_idle_detection_delay < batcher_deadline.
        // The batcher_deadline = proposal_timeout - build_proposal_margin.
        // If idle_delay >= batcher_deadline, idle detection never triggers (hard deadline fires
        // first).
        if let (Some(batcher_config), Some(consensus_manager_config)) =
            (&self.batcher_config, &self.consensus_manager_config)
        {
            let idle_delay = batcher_config.dynamic_config.proposer_idle_detection_delay_millis;
            let proposal_timeout = consensus_manager_config
                .consensus_manager_config
                .dynamic_config
                .timeouts
                .get_proposal_timeout(0); // base timeout (round 0)
            let build_margin =
                consensus_manager_config.context_config.dynamic_config.build_proposal_margin_millis;
            let batcher_deadline = proposal_timeout.saturating_sub(build_margin);

            if idle_delay >= batcher_deadline {
                return Err(ConfigError::ComponentConfigMismatch {
                    component_config_mismatch: format!(
                        "proposer_idle_detection_delay_millis ({:?}) must be less than \
                         batcher_deadline ({:?}) = proposal_timeout ({:?}) - build_margin ({:?})",
                        idle_delay, batcher_deadline, proposal_timeout, build_margin
                    ),
                });
            }
        }

        self.validate_validation_only_config()?;

        Ok(())
    }

pub(crate) fn get_declare_transaction_v3_hash<T: DeclareTransactionV3Trait>(
    transaction: &T,
    chain_id: &ChainId,
    transaction_version: &TransactionVersion,
) -> Result<TransactionHash, StarknetApiError> {
    let tip_resource_bounds_hash =
        get_tip_resource_bounds_hash(&transaction.resource_bounds(), transaction.tip())?;
    let paymaster_data_hash =
        HashChain::new().chain_iter(transaction.paymaster_data().0.iter()).get_poseidon_hash();
    let data_availability_mode = concat_data_availability_mode(
        transaction.nonce_data_availability_mode(),
        transaction.fee_data_availability_mode(),
    );
    let account_deployment_data_hash = HashChain::new()
        .chain_iter(transaction.account_deployment_data().0.iter())
        .get_poseidon_hash();

    Ok(TransactionHash(
        HashChain::new()
            .chain(&DECLARE)
            .chain(&transaction_version.0)
            .chain(transaction.sender_address().0.key())
            .chain(&tip_resource_bounds_hash)
            .chain(&paymaster_data_hash)
            .chain(&Felt::try_from(chain_id)?)
            .chain(&transaction.nonce().0)
            .chain(&data_availability_mode)
            .chain(&account_deployment_data_hash)
            .chain(&transaction.class_hash().0)
            .chain(&transaction.compiled_class_hash().0)
            .get_poseidon_hash(),
    ))
}

    async fn convert_rpc_tx_to_internal(
        &self,
        tx: RpcTransaction,
    ) -> TransactionConverterResult<(InternalRpcTransaction, Option<(ProofFacts, Proof)>)> {
        let (tx_without_hash, proof_data) = match tx {
            RpcTransaction::Invoke(RpcInvokeTransaction::V3(tx)) => {
                let proof_data = if tx.proof_facts.is_empty() {
                    None
                } else {
                    Some((tx.proof_facts.clone(), tx.proof.clone()))
                };
                (InternalRpcTransactionWithoutTxHash::Invoke(tx.into()), proof_data)
            }
            RpcTransaction::Declare(RpcDeclareTransaction::V3(tx)) => {
                let ClassHashes { class_hash, executable_class_hash_v2 } =
                // TODO(Dori): Make this async and spawn a task to compile and add it to the class manager.
                    self.class_manager_client.add_class(tx.contract_class).await?;
                // TODO(Aviv): Ensure that we do not want to
                // allow declare with compiled class hash v1.
                if tx.compiled_class_hash != executable_class_hash_v2 {
                    return Err(TransactionConverterError::ValidateCompiledClassHashError(
                        ValidateCompiledClassHashError::CompiledClassHashMismatch {
                            computed_class_hash: executable_class_hash_v2,
                            supplied_class_hash: tx.compiled_class_hash,
                        },
                    ));
                }
                (
                    InternalRpcTransactionWithoutTxHash::Declare(InternalRpcDeclareTransactionV3 {
                        sender_address: tx.sender_address,
                        compiled_class_hash: tx.compiled_class_hash,
                        signature: tx.signature,
                        nonce: tx.nonce,
                        class_hash,
                        resource_bounds: tx.resource_bounds,
                        tip: tx.tip,
                        paymaster_data: tx.paymaster_data,
                        account_deployment_data: tx.account_deployment_data,
                        nonce_data_availability_mode: tx.nonce_data_availability_mode,
                        fee_data_availability_mode: tx.fee_data_availability_mode,
                    }),
                    None,
                )
            }
            RpcTransaction::DeployAccount(RpcDeployAccountTransaction::V3(tx)) => {
                let contract_address = tx.calculate_contract_address()?;
                (
                    InternalRpcTransactionWithoutTxHash::DeployAccount(
                        InternalRpcDeployAccountTransaction {
                            tx: RpcDeployAccountTransaction::V3(tx),
                            contract_address,
                        },
                    ),
                    None,
                )
            }
        };
        let tx_hash = tx_without_hash.calculate_transaction_hash(&self.chain_id)?;
        Ok((InternalRpcTransaction { tx: tx_without_hash, tx_hash }, proof_data))
    }
