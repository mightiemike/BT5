### Title
SwapAllowlistExtension Gates the Router Address Instead of the Real Swapper, Allowing Any User to Bypass the Allowlist via MetricOmmSimpleRouter — (File: `metric-periphery/contracts/extensions/SwapAllowlistExtension.sol`)

---

### Summary

`SwapAllowlistExtension.beforeSwap` checks the `sender` argument, which the pool sets to `msg.sender` of the `swap()` call. When a user routes through `MetricOmmSimpleRouter`, the pool's `msg.sender` is the router contract, not the real user. If the pool admin allowlists the router (the only way to permit any router-mediated swap), every user on the internet can bypass the allowlist by calling the router.

---

### Finding Description

`MetricOmmPool.swap` passes `msg.sender` as the `sender` argument to `_beforeSwap`, which forwards it verbatim to every configured extension:

```solidity
// MetricOmmPool.sol line 230-240
_beforeSwap(
    msg.sender,   // ← router address when called via router
    recipient,
    zeroForOne,
    amountSpecified,
    priceLimitX64,
    packedSlot0Initial,
    bidPriceX64,
    askPriceX64,
    extensionData
);
``` [1](#0-0) 

`SwapAllowlistExtension.beforeSwap` then checks whether that `sender` is allowlisted for the calling pool (`msg.sender` inside the extension is the pool):

```solidity
// SwapAllowlistExtension.sol lines 31-41
function beforeSwap(address sender, address, bool, int128, uint128, uint256, uint128, uint128, bytes calldata)
    external view override returns (bytes4)
{
    if (!allowAllSwappers[msg.sender] && !allowedSwapper[msg.sender][sender]) {
        revert IMetricOmmPoolActions.NotAllowedToSwap();
    }
    return IMetricOmmExtensions.beforeSwap.selector;
}
``` [2](#0-1) 

`MetricOmmSimpleRouter.exactInputSingle` (and every other router entry point) calls `pool.swap(...)` directly, making the pool see `msg.sender = router`:

```solidity
// MetricOmmSimpleRouter.sol lines 71-80
_setNextCallbackContext(params.pool, CALLBACK_MODE_JUST_PAY, msg.sender, params.tokenIn);
(int128 amount0Delta, int128 amount1Delta) = IMetricOmmPoolActions(params.pool)
    .swap(
        params.recipient,
        params.zeroForOne,
        MetricOmmSwapInputs.asAmountSpecifiedIn(params.amountIn),
        priceLimitX64,
        "",
        params.extensionData
    );
``` [3](#0-2) 

The extension therefore checks `allowedSwapper[pool][router]`, not `allowedSwapper[pool][real_user]`. To permit any router-mediated swap at all, the pool admin must call `setAllowedToSwap(pool, router, true)`. Once that entry exists, the check passes for **every** caller who routes through the router, regardless of whether the real user is on the allowlist.

The `DepositAllowlistExtension` does not share this flaw: it checks `owner` (the LP-share recipient), which is the economically relevant actor and is supplied directly by the caller, not overwritten by the router. [4](#0-3) 

---

### Impact Explanation

A pool configured with `SwapAllowlistExtension` to restrict trading to specific counterparties (e.g., KYC-verified market makers, institutional partners, or whitelisted arbitrageurs) has its access control completely nullified for any user who calls through `MetricOmmSimpleRouter`. The attacker can:

1. Execute swaps the pool admin explicitly intended to block.
2. Extract value from the pool at the expense of LPs who deposited under the assumption that only vetted counterparties could trade.
3. Drain favorable pricing or arbitrage opportunities reserved for allowlisted parties.

This is a direct loss of LP principal and a broken core pool invariant: the allowlist guard fails open for the entire public router path.

---

### Likelihood Explanation

- **Trigger is unprivileged**: any EOA or contract can call `MetricOmmSimpleRouter`.
- **Setup is natural**: a pool admin who deploys `SwapAllowlistExtension` and also wants to support the standard router *must* allowlist the router; there is no other mechanism to allow router-mediated swaps while keeping the allowlist active.
- **No special conditions**: the bypass works on every block, for every swap direction, with any token amount.

Likelihood is **High**.

---

### Recommendation

The `SwapAllowlistExtension` must gate the **real user**, not the intermediary. Two complementary fixes:

1. **Pass the real payer through `extensionData`**: have the router encode `msg.sender` into `extensionData` and have the extension decode and verify it. This requires the extension to trust the router's encoding, which reintroduces a trust assumption.

2. **Preferred — check `sender` at the router level and reject non-allowlisted callers before the pool call**: add an `isAllowedToSwap` pre-check in the router, or require callers to prove allowlist membership via a signed permit forwarded in `extensionData`.

3. **Structural fix**: redesign `beforeSwap` to receive the original `tx.origin` or a router-attested caller identity. Because `tx.origin` is manipulable in some contexts, the cleanest solution is to have the router sign or encode the real caller and have the extension verify the signature or the router's identity before trusting the embedded address.

Until fixed, pools that rely on `SwapAllowlistExtension` for access control should **not** allowlist `MetricOmmSimpleRouter`.

---

### Proof of Concept

```solidity
// Setup
SwapAllowlistExtension ext = new SwapAllowlistExtension(factory);
// Pool deployed with ext as beforeSwap extension
// Admin allowlists the router so legitimate users can swap via router:
ext.setAllowedToSwap(pool, address(router), true);
// Admin does NOT allowlist attacker:
// allowedSwapper[pool][attacker] == false

// Attack — attacker is not allowlisted but calls through the router:
router.exactInputSingle(ExactInputSingleParams({
    pool:            pool,
    tokenIn:         token0,
    recipient:       attacker,
    amountIn:        1_000e18,
    amountOutMinimum: 0,
    zeroForOne:      true,
    priceLimitX64:   0,
    deadline:        block.timestamp,
    extensionData:   ""
}));
// Pool calls ext.beforeSwap(sender=router, ...)
// ext checks allowedSwapper[pool][router] == true  ✓
// Swap executes — attacker bypassed the allowlist.
```

The pool's `_beforeSwap` receives `sender = address(router)`. [5](#0-4) 

The extension approves because the router is allowlisted, not the real user. [6](#0-5)
