### Title
`BurnNFT` Requires Simultaneous `IsOwner` AND `IsDenomCreator` — NFT Permanently Locked After Transfer — (`x/nft/keeper/keeper.go`)

### Summary

`keeper.BurnNFT` enforces two sequential authorization checks on the same `owner` address: it must be both the current NFT owner **and** the denom creator. Once the denom creator transfers an NFT to any other address, neither party can ever satisfy both conditions simultaneously, permanently locking the NFT.

### Finding Description

In `x/nft/keeper/keeper.go`, `BurnNFT` is implemented as: [1](#0-0) 

Line 146 calls `IsOwner`, which returns `ErrUnauthorized` if the caller is not the current NFT owner: [2](#0-1) 

Line 151 then calls `IsDenomCreator`, which returns `ErrUnauthorized` if the caller is not the denom's creator: [3](#0-2) 

Both checks are applied to the **same** `owner` argument with no `||` branch. After `TransferOwner` moves the NFT from Alice (denom creator) to Bob: [4](#0-3) 

- `BurnNFT(Alice)` → `IsOwner` fails (Alice is no longer owner)
- `BurnNFT(Bob)` → `IsDenomCreator` fails (Bob is not the creator)

There is no code path through which either party can burn the NFT. The only escape is Bob voluntarily transferring back to Alice, which Bob is not obligated to do.

Note that `BurnNFTUnverified` (used for IBC) correctly skips the `IsDenomCreator` check and only requires `IsOwner`: [5](#0-4) 

This confirms the dual-check in `BurnNFT` is an unintentional design error, not a deliberate policy.

### Impact Explanation

Any NFT minted by a denom creator and subsequently transferred to a non-creator address becomes permanently unburnable via the standard `MsgBurnNFT` path. The NFT is locked in the recipient's account indefinitely. If the NFT has economic value (e.g., represents a redeemable asset, access right, or scarce collectible), this constitutes a permanent lock of an economically valuable NFT — matching the Critical scoped impact.

### Likelihood Explanation

The transfer of NFTs from creator to non-creator is the normal, intended use case (e.g., minting to a buyer). This scenario is trivially reachable by any user through standard `MsgMintNFT` + `MsgTransferNFT` + `MsgBurnNFT` transactions. No special privileges or unusual chain state are required.

### Recommendation

Change `BurnNFT` to require **either** `IsOwner` **or** `IsDenomCreator`, not both. The owner of an NFT should always be able to destroy their own property. A corrected guard:

```go
_, ownerErr := k.IsOwner(ctx, denomID, tokenID, owner)
_, creatorErr := k.IsDenomCreator(ctx, denomID, owner)
if ownerErr != nil && creatorErr != nil {
    return sdkerrors.Wrapf(types.ErrUnauthorized, "%s is neither the owner nor the creator", owner)
}
```

This matches the semantics of `BurnNFTUnverified` (owner-only) while also permitting the denom creator to burn any NFT in their collection.

### Proof of Concept

```
1. Alice calls MsgIssueDenom  → denom "testdenom" created, Alice = creator
2. Alice calls MsgMintNFT     → NFT "token1" minted, owner = Alice
3. Alice calls MsgTransferNFT → NFT "token1" transferred to Bob
4. Alice calls MsgBurnNFT     → keeper.IsOwner(Alice) fails → ErrUnauthorized
5. Bob   calls MsgBurnNFT     → keeper.IsDenomCreator(Bob) fails → ErrUnauthorized
6. NFT "token1" is now permanently locked in Bob's account.
```

All five steps are reachable through standard, unprivileged Cosmos SDK transactions on an unmodified chain deployment.

	return nil
}

func (k Keeper) IsOwner(ctx sdk.Context, denomID, tokenID string, owner sdk.AccAddress) (types.BaseNFT, error) {
	nft, err := k.GetNFT(ctx, denomID, tokenID)
	if err != nil {
		return types.BaseNFT{}, err
	}

	if !owner.Equals(nft.GetOwner()) {
		return types.BaseNFT{}, sdkerrors.Wrapf(types.ErrUnauthorized, "%s is not the owner of %s/%s", owner.String(), denomID, tokenID)
	}

	return nft.(types.BaseNFT), nil
}

func (k Keeper) IsDenomCreator(ctx sdk.Context, denomID string, address sdk.AccAddress) (types.Denom, error) {
	denom, err := k.GetDenom(ctx, denomID)
	if err != nil {
		return types.Denom{}, err
	}

	creator, err := sdk.AccAddressFromBech32(denom.Creator)
	if err != nil {
		panic(err)
	}

	if !creator.Equals(address) {
		return types.Denom{}, sdkerrors.Wrapf(types.ErrUnauthorized, "%s is not the creator of %s", address, denomID)
	}

	return denom, nil
