### Title
Bonus Accrual Checkpoint Advanced Before Pool Check in `BeforeRedelegationSlashed` Hook Permanently Destroys Accrued Reward Claims - (File: `x/tieredrewards/keeper/claim_rewards.go`, `x/tieredrewards/keeper/slash.go`)

---

### Summary

In `processEventsAndClaimBonus`, the position's `LastBonusAccrual` checkpoint is advanced **before** the bonus pool sufficiency check. When this function is called from `slashRedelegationPosition` (the `BeforeRedelegationSlashed` hook) and the pool is insufficient, the error is silently swallowed and the modified position state — with the checkpoint already advanced — is persisted. The position owner permanently loses their accrued bonus rewards with no recourse.

---

### Finding Description

`processEventsAndClaimBonus` in `claim_rewards.go` follows this order:

1. Walk pending validator events, accumulate `totalBonus`, advance `pos.LastEventSeq` per event, decrement event reference counts (lines 172–199).
2. Compute the final open segment bonus (lines 201–213).
3. **Advance `pos.LastBonusAccrual` and `pos.LastKnownBonded`** (lines 215–217).
4. Check pool sufficiency — return `ErrInsufficientBonusPool` if the pool cannot cover the bonus (lines 230–232). [1](#0-0) 

The checkpoint mutation at line 215 happens unconditionally before the pool check at line 230. When the pool is insufficient, the function returns an error with `pos.LastBonusAccrual` already advanced to `blockTime`.

In `slashRedelegationPosition`, this error is explicitly swallowed for chain-halt avoidance:

```go
if _, err := k.processEventsAndClaimBonus(ctx, &pos); err != nil {
    if errors.Is(err, types.ErrInsufficientBonusPool) {
        k.logger(ctx).Error("insufficient bonus pool during redelegation slash", ...)
        // error swallowed — pos.LastBonusAccrual is already advanced
    } else {
        return err
    }
}
// partial slash path:
pos.Delegation.Shares = pos.Delegation.Shares.Sub(sharesToUnbond)
return k.setPositionWithState(ctx, pos, nil)  // persists the advanced checkpoint
``` [2](#0-1) 

`setPositionWithState` is then called with the already-mutated `pos`, persisting `LastBonusAccrual` advanced to the slash block time, `LastEventSeq` advanced past all processed events, and `LastKnownBonded` updated — all without any bonus having been paid.

For user-driven paths (`ClaimTierRewards`, `TierRedelegate`, etc.), `processEventsAndClaimBonus` also returns `ErrInsufficientBonusPool` with the checkpoint already advanced, but those transactions fail atomically (the entire tx is rolled back by the Cosmos SDK), so the checkpoint mutation is never committed. The hook path has no such rollback: `slashRedelegationPosition` is called inside the staking module's `SlashRedelegation`, which is itself inside `BeginBlock`. The state is committed regardless. [3](#0-2) 

---

### Impact Explanation

The exact corrupted value is `pos.LastBonusAccrual` (and `pos.LastEventSeq`, `pos.LastKnownBonded`) stored in the `Positions` collection. These fields are advanced to the slash block time without the corresponding bonus payment. On the next `ClaimTierRewards` call, `processEventsAndClaimBonus` starts from the advanced checkpoint, so the entire accrual period before the slash is permanently skipped. The position owner loses all bonus rewards accrued since their last successful claim, with no mechanism to recover them. The bonus pool retains the funds, but the user's claim is destroyed.

The magnitude scales with: position size × `tier.BonusApy` × time since last claim. For a large long-running position in a high-APY tier, this can be a material loss. [4](#0-3) 

---

### Likelihood Explanation

Three conditions must coincide:

1. **Active redelegation on a tier position** — any user who calls `MsgTierRedelegate` creates a redelegation entry and a `RedelegationMappings` row. This is a normal user action.
2. **Source validator is slashed** — validator downtime or double-signing triggers `SlashRedelegation`, which fires `BeforeRedelegationSlashed`. This is a normal staking event requiring no attacker privilege.
3. **Bonus pool is insufficient** — the `RewardsPoolName` module account has less than the accrued bonus. The pool is funded externally and depletes over time via normal bonus claims and `BeginBlocker` top-ups. An empty or near-empty pool is a realistic operational state, especially if the pool has not been replenished recently.

All three conditions are reachable by normal protocol operation without any privileged access. A validator slash is an unprivileged external event (downtime is automatic; double-sign is a validator action). The pool being insufficient is a normal operational state. [5](#0-4) 

---

### Recommendation

Move `applyBonusAccrualCheckpoint` and `pos.UpdateLastKnownBonded` to **after** the pool sufficiency check and the `SendCoinsFromModuleToAccount` call in `processEventsAndClaimBonus`. The checkpoint should only be advanced when the bonus is actually paid (or when bonus is zero, which is already safe). Concretely:

```go
// After SendCoinsFromModuleToAccount succeeds:
applyBonusAccrualCheckpoint(&pos.Position, blockTime)
pos.UpdateLastKnownBonded(bonded)
```

For the `BeforeRedelegationSlashed` hook path specifically, if the pool is insufficient and the bonus must be forfeited to avoid chain halt, the checkpoint should still be advanced (to prevent double-claiming the same period later), but the position owner should be made aware via an on-chain event rather than only a log message. Alternatively, the checkpoint advancement can be separated from the payment so that the hook path can advance the checkpoint without attempting payment. [6](#0-5) 

---

### Proof of Concept

1. Governance sets `BonusApy = 10%` on Tier 1. The `RewardsPoolName` account is funded with a small amount (e.g., 1 `basecro`).
2. Alice calls `MsgLockTier` with 1,000,000 `basecro` on validator V1. Position P1 is created with `LastBonusAccrual = T0`.
3. 30 days pass. Alice's accrued bonus ≈ `1,000,000 × 0.10 × (30/365)` ≈ 8,219 `basecro`. The pool has only 1 `basecro`.
4. Alice calls `MsgTierRedelegate` from V1 to V2. A `RedelegationMappings[unbondingId] = P1` entry is created.
5. V1 is slashed (downtime). `BeforeRedelegationSlashed` fires → `slashRedelegationPosition` → `processEventsAndClaimBonus`.
6. Inside `processEventsAndClaimBonus`: `pos.LastBonusAccrual` is advanced to the slash block time (line 215). Then `sufficientBonusPoolBalance` returns `ErrInsufficientBonusPool` (pool has 1 `basecro`, bonus is 8,219).
7. Back in `slashRedelegationPosition`: error is swallowed. `setPositionWithState` persists `pos` with `LastBonusAccrual = slashTime`.
8. Alice calls `MsgClaimTierRewards` after the pool is replenished. `processEventsAndClaimBonus` starts from `LastBonusAccrual = slashTime` — the 30-day accrual period is gone. Alice receives 0 bonus for that period. [2](#0-1) [7](#0-6)

func (k Keeper) processEventsAndClaimBonus(ctx context.Context, pos *types.PositionState) (sdk.Coins, error) {
	// Rewards should have been claimed before undelegation
	if !pos.IsDelegated() {
		return sdk.NewCoins(), nil
	}

	valAddr, err := sdk.ValAddressFromBech32(pos.Delegation.ValidatorAddress)
	if err != nil {
		return nil, err
	}

	events, err := k.getValidatorEventsSince(ctx, valAddr, pos.LastEventSeq)
	if err != nil {
		return nil, err
	}

	sdkCtx := sdk.UnwrapSDKContext(ctx)
	blockTime := sdkCtx.BlockTime()

	totalBonus := math.ZeroInt()
	// Use the persisted bonded state from the last replay, not a hardcoded default.
	// This prevents overpaying bonus for unbonded gaps between claims.
	bonded := pos.LastKnownBonded
	segmentStart := pos.LastBonusAccrual

	tier, err := k.getTier(ctx, pos.TierId)
	if err != nil {
		return nil, err
	}

	for _, entry := range events {
		evt := entry.Event

		if bonded {
			// Compute bonus for the bonded segment [segmentStart, eventTime]
			// using the snapshot rate at the event.
			bonus := k.computeSegmentBonus(*pos, tier, segmentStart, evt.Timestamp, evt.TokensPerShare)
			totalBonus = totalBonus.Add(bonus)
		}

		// Update bonded state based on event type.
		switch evt.EventType {
		case types.ValidatorEventType_VALIDATOR_EVENT_TYPE_UNBOND:
			bonded = false
		case types.ValidatorEventType_VALIDATOR_EVENT_TYPE_BOND:
			bonded = true
		case types.ValidatorEventType_VALIDATOR_EVENT_TYPE_SLASH:
			// Slash doesn't change bonded state.
		}

		segmentStart = evt.Timestamp
		pos.UpdateLastEventSeq(entry.Seq)

		// Decrement reference count.
		if err := k.decrementEventRefCount(ctx, valAddr, entry.Seq); err != nil {
			return nil, err
		}
	}

	val, err := k.stakingKeeper.GetValidator(ctx, valAddr)
	if err != nil {
		return nil, err
	}
	// Defensive: validator bond status check
	if bonded && val.IsBonded() {
		currentRate, err := k.getTokensPerShare(ctx, valAddr)
		if err != nil {
			return nil, err
		}
		bonus := k.computeSegmentBonus(*pos, tier, segmentStart, blockTime, currentRate)
		totalBonus = totalBonus.Add(bonus)
	}

	applyBonusAccrualCheckpoint(&pos.Position, blockTime)
	// Persist the bonded state so the next replay starts correctly.
	pos.UpdateLastKnownBonded(bonded)

	if totalBonus.IsZero() {
		return sdk.NewCoins(), nil
	}

	bondDenom, err := k.stakingKeeper.BondDenom(ctx)
	if err != nil {
		return nil, err
	}

	bonusCoins := sdk.NewCoins(sdk.NewCoin(bondDenom, totalBonus))

	if err := k.sufficientBonusPoolBalance(ctx, bonusCoins); err != nil {
		return nil, err
	}

func (k Keeper) computeSegmentBonus(pos types.PositionState, tier types.Tier, segmentStart, segmentEnd time.Time, tokensPerShare math.LegacyDec) math.Int {
	if !pos.ExitUnlockAt.IsZero() && segmentEnd.After(pos.ExitUnlockAt) {
		segmentEnd = pos.ExitUnlockAt
	}

	if !segmentEnd.After(segmentStart) {
		return math.ZeroInt()
	}

	durationSeconds := int64(segmentEnd.Sub(segmentStart) / time.Second)
	if durationSeconds <= 0 {
		return math.ZeroInt()
	}

	tokens := pos.Delegation.Shares.Mul(tokensPerShare)

	return tokens.
		Mul(tier.BonusApy).
		MulInt64(durationSeconds).
		QuoInt64(types.SecondsPerYear).
		TruncateInt()
