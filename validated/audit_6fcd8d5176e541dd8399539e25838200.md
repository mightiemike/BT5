### Title
`ValidResourceBounds` Protobuf Round-Trip Conflates `AllResources{l2=0,l1_data=0}` with `L1Gas`, Causing Valid Transactions to Be Rejected by Peer Nodes — (`crates/apollo_protobuf/src/converters/transaction.rs`)

### Summary

The protobuf deserialization of `ValidResourceBounds` uses a value-based heuristic — checking whether `l2_gas` and `l1_data_gas` are zero — to decide which enum variant to reconstruct. This conflates a legitimately submitted `AllResources { l1_gas: X, l2_gas: 0, l1_data_gas: 0 }` transaction with the deprecated `L1Gas(X)` variant. The originating gateway accepts the transaction, computes its hash under `AllResources` semantics, and adds it to the mempool. When the transaction is propagated via P2P, the receiving node deserializes the resource bounds as `L1Gas`, then immediately rejects the transaction with `DEPRECATED_RESOURCE_BOUNDS_ERROR` because the `RpcDeclareTransactionV3` (and `RpcInvokeTransactionV3`) protobuf converter requires `AllResources`. Valid transactions are thus silently dropped at every peer node.

### Finding Description

**Root cause — deserialization heuristic in `TryFrom<protobuf::ResourceBounds> for ValidResourceBounds`:**

```rust
// crates/apollo_protobuf/src/converters/transaction.rs  lines 417-436
Ok(if l1_data_gas.is_zero() && l2_gas.is_zero() {
    ValidResourceBounds::L1Gas(l1_gas)          // ← misclassifies AllResources with zero L2/L1_data
} else {
    ValidResourceBounds::AllResources(AllResourceBounds { l1_gas, l2_gas, l1_data_gas })
})
``` [1](#0-0) 

The condition tests the *value* of the fields, not their presence. Any `AllResources` transaction whose L2 and L1_data_gas bounds happen to be zero is silently downgraded to `L1Gas` after a protobuf round-trip.

**Rejection site — `TryFrom<protobuf::DeclareV3WithClass> for RpcDeclareTransactionV3`:**

```rust
// crates/apollo_protobuf/src/converters/rpc_transaction.rs  lines 169-191
resource_bounds: match common.resource_bounds {
    ValidResourceBounds::AllResources(resource_bounds) => resource_bounds,
    _ => return Err(DEPRECATED_RESOURCE_BOUNDS_ERROR),   // ← fires for the misclassified L1Gas
},
``` [2](#0-1) 

The same pattern applies to `RpcInvokeTransactionV3` and `RpcDeployAccountTransactionV3` converters.

**Originating gateway accepts the transaction** because `RpcDeclareTransactionV3` carries `resource_bounds: AllResourceBounds` (not `ValidResourceBounds`), so the heuristic is never applied there: [3](#0-2) 

The gateway's stateless validator explicitly permits `AllResourceBounds` with only L1 gas set: [4](#0-3) 

**Hash domain divergence (secondary):** Even if the rejection were bypassed, the hash computed at the originating node uses `ValidResourceBounds::AllResources` (includes `l1_data_gas` in the Poseidon preimage), while any re-hash at the peer would use `ValidResourceBounds::L1Gas` (omits `l1_data_gas`), producing a different `tip_resource_bounds_hash`: [5](#0-4) 

`InternalRpcDeclareTransactionV3` and `InternalRpcInvokeTransactionV3` always return `ValidResourceBounds::AllResources(self.resource_bounds)` from their trait impls, so the originating hash is always computed under `AllResources` semantics: [6](#0-5) [7](#0-6) 

### Impact Explanation

A valid `Declare`, `Invoke`, or `DeployAccount` V3 transaction with `AllResourceBounds { l1_gas: X, l2_gas: 0, l1_data_gas: 0 }` is accepted and hashed by the originating gateway, added to its mempool, and forwarded to peers via P2P. Every peer node deserializes the resource bounds as `L1Gas` and immediately rejects the transaction with `DEPRECATED_RESOURCE_BOUNDS_ERROR`. The transaction never enters any peer's mempool. In a distributed sequencer deployment, only the originating node can propose the transaction; all other validators lack it, which can stall or delay finality for that transaction.

This matches: **High — Mempool/gateway/RPC admission rejects valid transactions before sequencing.**

### Likelihood Explanation

The trigger requires only submitting a standard V3 transaction with zero L2 and L1_data_gas bounds — a configuration explicitly permitted by the gateway's stateless validator (test case `valid_l1_gas` above). No special privileges, no malformed bytes, no adversarial peer required. Any user who sets only `l1_gas` in an `AllResourceBounds` transaction (a natural choice when not using L2 gas or blob DA) hits this path.

### Recommendation

Replace the value-based heuristic with an explicit field-presence check. The protobuf schema should use a discriminator field or a wrapper message to unambiguously encode which `ValidResourceBounds` variant is intended. At minimum, the deserialization should only produce `L1Gas` when the protobuf message was serialized from a `L1Gas` variant — not whenever the numeric values happen to be zero. One concrete fix: add a boolean `is_all_resources` flag to the protobuf `ResourceBounds` message, or use a `oneof` to distinguish the two variants, so that `AllResources { l2_gas: 0, l1_data_gas: 0 }` round-trips faithfully.

### Proof of Concept

1. Submit a `RpcDeclareTransactionV3` to the gateway with `resource_bounds = AllResourceBounds { l1_gas: ResourceBounds { max_amount: 1000, max_price_per_unit: 1 }, l2_gas: ResourceBounds::default(), l1_data_gas: ResourceBounds::default() }`.
2. Gateway accepts it (stateless validator passes; `AllResourceBounds` with non-zero L1 gas is valid).
3. Gateway computes `tx_hash` using `get_declare_transaction_v3_hash` → `get_tip_resource_bounds_hash` with `ValidResourceBounds::AllResources(...)` → hash includes `l1_data_gas = 0` element → hash_A.
4. Transaction is serialized to `protobuf::MempoolTransaction::DeclareV3` for P2P propagation. The `AllResourceBounds` is serialized as `protobuf::ResourceBounds { l1_gas: ..., l2_gas: zero, l1_data_gas: zero }`.
5. Peer deserializes: `l1_data_gas.is_zero() && l2_gas.is_zero()` → `ValidResourceBounds::L1Gas(l1_gas)`.
6. Peer attempts `RpcDeclareTransactionV3::try_from(...)` → `match common.resource_bounds { ValidResourceBounds::AllResources(...) => ..., _ => Err(DEPRECATED_RESOURCE_BOUNDS_ERROR) }` → **transaction rejected**.
7. Transaction is absent from all peer mempools; only the originating node can propose it. [8](#0-7)

impl TryFrom<protobuf::ResourceBounds> for ValidResourceBounds {
    type Error = ProtobufConversionError;
    fn try_from(value: protobuf::ResourceBounds) -> Result<Self, Self::Error> {
        let Some(l1_gas) = value.l1_gas else {
            return Err(missing("ResourceBounds::l1_gas"));
        };
        let Some(l2_gas) = value.l2_gas else {
            return Err(missing("ResourceBounds::l2_gas"));
        };
        // TODO(Shahak): Assert data gas is not none once we remove support for 0.13.2.
        let l1_data_gas = value.l1_data_gas.unwrap_or_default();
        let l1_gas: ResourceBounds = l1_gas.try_into()?;
        let l2_gas: ResourceBounds = l2_gas.try_into()?;
        let l1_data_gas: ResourceBounds = l1_data_gas.try_into()?;
        Ok(if l1_data_gas.is_zero() && l2_gas.is_zero() {
            ValidResourceBounds::L1Gas(l1_gas)
        } else {
            ValidResourceBounds::AllResources(AllResourceBounds { l1_gas, l2_gas, l1_data_gas })
        })
    }

```rust
pub fn get_tip_resource_bounds_hash(
    resource_bounds: &ValidResourceBounds,
    tip: &Tip,
) -> Result<Felt, StarknetApiError> {
    let l1_resource_bounds = resource_bounds.get_l1_bounds();
    let l2_resource_bounds = resource_bounds.get_l2_bounds();

    // L1 and L2 gas bounds always exist.
    // Old V3 txs always have L2 gas bounds of zero, but they exist.
    let mut resource_felts = vec![
        get_concat_resource(&l1_resource_bounds, L1_GAS)?,
        get_concat_resource(&l2_resource_bounds, L2_GAS)?,
    ];

    // For new V3 txs, need to also hash the data gas bounds.
    resource_felts.extend(match resource_bounds {
        ValidResourceBounds::L1Gas(_) => vec![],
        ValidResourceBounds::AllResources(all_resources) => {
            vec![get_concat_resource(&all_resources.l1_data_gas, L1_DATA_GAS)?]
        }
    });

    Ok(HashChain::new().chain(&tip.0.into()).chain_iter(resource_felts.iter()).get_poseidon_hash())
}
