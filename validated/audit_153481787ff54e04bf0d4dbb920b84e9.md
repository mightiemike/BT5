Audit Report

## Title
`SwapAllowlistExtension.beforeSwap` Checks Router Address Instead of Actual User, Allowing Full Allowlist Bypass — (`metric-periphery/contracts/extensions/SwapAllowlistExtension.sol`)

## Summary
`SwapAllowlistExtension.beforeSwap` gates swaps by checking `allowedSwapper[pool][sender]`, where `sender` is the pool's `msg.sender` — the router, not the originating user. When the router is allowlisted (the natural configuration for any pool accessible via the supported periphery), every unprivileged user can bypass the per-user swap allowlist by routing through `MetricOmmSimpleRouter`. The allowlist — the sole access-control mechanism on the swap path — silently fails open.

## Finding Description
The full call chain is confirmed in production code:

1. `MetricOmmSimpleRouter.exactInputSingle` calls `IMetricOmmPoolActions(params.pool).swap(params.recipient, ...)` — the router is `msg.sender` to the pool. [1](#0-0) 

2. `MetricOmmPool.swap` calls `_beforeSwap(msg.sender, ...)`, passing the router as `sender`. [2](#0-1) 

3. `ExtensionCalling._beforeSwap` encodes `sender` (the router) verbatim and dispatches to the extension. [3](#0-2) 

4. `SwapAllowlistExtension.beforeSwap` checks `allowedSwapper[msg.sender][sender]`, which resolves to `allowedSwapper[pool][router]` — never `allowedSwapper[pool][alice]`. [4](#0-3) 

There is no secondary check, no `tx.origin` guard, and no mechanism to recover the original user identity. The `allowAllSwappers` flag is the only alternative path, but it is a pool-wide bypass, not per-user. Once `allowedSwapper[pool][router] = true`, the check passes for every caller of the router regardless of their identity.

A secondary breakage also exists: a pool admin who allowlists `alice` directly (`setAllowedToSwap(pool, alice, true)`) will find that Alice cannot swap through the router, because the check sees `router`, not `alice`. This forces direct pool interaction, which requires implementing `IMetricOmmSwapCallback` — unreasonable for an EOA. [5](#0-4) 

## Impact Explanation
A curated pool deploying `SwapAllowlistExtension` to restrict trading to a specific set of counterparties loses that restriction entirely for any user who routes through `MetricOmmSimpleRouter`. Unauthorized users can execute swaps, interact with LP-provided liquidity at oracle prices, and extract value from a pool designed to be closed to them. This constitutes broken core pool functionality causing direct loss of LP assets — the allowlist is the sole access-control mechanism on the swap path and it fails open silently.

## Likelihood Explanation
`MetricOmmSimpleRouter` is the primary supported swap entrypoint for the protocol. Any pool admin who configures a swap allowlist and also wants their pool accessible via the router will naturally call `setAllowedToSwap(pool, router, true)`. The bypass requires no special privileges, no malicious setup, no non-standard tokens — only a call to the public `exactInputSingle` or `exactInput` function on the router. The precondition (router allowlisted) is the expected and natural operational configuration.

## Recommendation
The extension must check the economically relevant actor — the original user — not the intermediary router. Options in order of cleanliness:

1. **Preferred:** Add a dedicated `swapper` field to the extension hook interface that the pool populates with the original `msg.sender` before any router indirection, keeping `sender` as the direct caller.
2. **Alternative:** The router forwards the original `msg.sender` via `extensionData`, and `SwapAllowlistExtension` decodes it when `sender` is a known router.
3. **Minimal (partial):** Check `recipient` instead of `sender` if the intended gated identity is the output recipient — but this is only correct if recipient always equals the actual user, which is not guaranteed.

## Proof of Concept
1. Deploy a pool with `SwapAllowlistExtension` configured.
2. Pool admin calls `setAllowedToSwap(pool, router, true)` — intending to allow router-mediated swaps.
3. Pool admin does **not** call `setAllowedToSwap(pool, alice, true)` — alice is not an authorized swapper.
4. Alice calls `MetricOmmSimpleRouter.exactInputSingle(pool, ...)`.
5. Pool calls `_beforeSwap(router, ...)` — `msg.sender` of `pool.swap()` is the router.
6. Extension evaluates `allowedSwapper[pool][router]` → `true` → swap proceeds.
7. Alice successfully swaps on a pool she was never authorized to access.
