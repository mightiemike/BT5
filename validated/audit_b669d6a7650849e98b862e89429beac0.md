### Title
Attacker-Controlled `data.ClassId` Prefix Bypasses `IsAwayFromOrigin` to Drain Victim Escrow — (`x/nft-transfer/keeper/packet.go`, `x/nft-transfer/types/trace.go`)

---

### Summary

`processReceivedPacket` determines whether to mint a voucher or unescrow an NFT solely by calling `IsAwayFromOrigin(packet.GetSourcePort(), packet.GetSourceChannel(), data.ClassId)`. Because `data.ClassId` is attacker-supplied packet data and `ValidateBasic` imposes no structural constraint on it, a malicious counterparty chain can craft `data.ClassId` to start with its own `sourcePort/sourceChannel/` prefix, forcing `IsAwayFromOrigin` to return `false` and triggering the unescrow branch. If the victim chain holds any NFT in the escrow address for that channel, `TransferOwner(escrowAddress, receiver)` will transfer it to the attacker.

---

### Finding Description

**`IsAwayFromOrigin` logic** (`x/nft-transfer/types/trace.go`, lines 49–52):

```go
func IsAwayFromOrigin(sourcePort, sourceChannel, fullClassPath string) bool {
    prefixClassID := GetClassPrefix(sourcePort, sourceChannel)
    return !strings.HasPrefix(fullClassPath, prefixClassID)
}
```

It returns `false` (i.e., "going back to origin") whenever `fullClassPath` starts with `sourcePort + "/" + sourceChannel + "/"`. The `fullClassPath` is taken directly from `data.ClassId` in the packet — a field the attacker writes freely.

**`processReceivedPacket` unescrow branch** (`x/nft-transfer/keeper/packet.go`, lines 148–201):

```go
isAwayFromOrigin := types.IsAwayFromOrigin(
    packet.GetSourcePort(), packet.GetSourceChannel(), data.ClassId)
escrowAddress := types.GetEscrowAddress(
    packet.GetDestPort(), packet.GetDestChannel())

// ...
} else {
    unprefixedClassID := types.RemoveClassPrefix(
        packet.GetSourcePort(), packet.GetSourceChannel(), data.ClassId)
    voucherClassID := types.ParseClassTrace(unprefixedClassID).IBCClassID()
    for _, tokenID := range data.TokenIds {
        k.nftKeeper.TransferOwner(ctx, voucherClassID, tokenID, escrowAddress, receiver)
    }
}
```

`escrowAddress` is `GetEscrowAddress(destPort, destChannel)` — the victim chain's escrow for the channel. If the victim chain previously sent any NFT to the attacker's chain via this channel, that NFT sits in exactly this escrow address.

**`ValidateBasic`** (`x/nft-transfer/types/packet.go`, lines 41–72) only checks that `ClassId` is non-blank, token/URI lengths match, and addresses are valid bech32. It performs **no structural validation** of the `ClassId` prefix.

**Exploit construction:**

| Field | Value |
|---|---|
| Attacker source port | `nft` |
| Attacker source channel | `channel-1` |
| Victim dest port | `nft` |
| Victim dest channel | `channel-0` |
| Crafted `data.ClassId` | `nft/channel-1/victimClass` |

- `IsAwayFromOrigin("nft", "channel-1", "nft/channel-1/victimClass")` → `false` ✓
- `RemoveClassPrefix("nft", "channel-1", "nft/channel-1/victimClass")` → `"victimClass"` ✓
- `ParseClassTrace("victimClass").IBCClassID()` → `"victimClass"` (no path, returns base) ✓
- `escrowAddress` = `GetEscrowAddress("nft", "channel-0")` — exactly where victim's escrowed NFTs live ✓
- `TransferOwner(ctx, "victimClass", tokenID, escrowAddress, attackerReceiver)` → succeeds if `victimClass/tokenID` is owned by that escrow ✓

---

### Impact Explanation

An attacker who controls a counterparty chain can steal any NFT that the victim chain has escrowed for transfers over the shared channel. The attacker can simultaneously hold the legitimate IBC voucher on their chain (received from the victim's original send) **and** reclaim the original NFT on the victim chain — a complete double-spend. The victim chain's escrow invariant ("an NFT may only be unescrow'd if the corresponding send packet was committed on this chain") is broken without any corresponding burn or acknowledgement.

---

### Likelihood Explanation

The precondition — victim chain has an NFT escrowed at `GetEscrowAddress(destPort, destChannel)` — is the normal steady state whenever any user on the victim chain sends an NFT to the attacker's chain. The attacker controls their own chain and can commit arbitrary packet data. IBC core validates packet commitment proofs but not application-layer packet content. No privileged access, governance, or key compromise is required.

---

### Recommendation

`IsAwayFromOrigin` must not trust the attacker-supplied `data.ClassId` to determine transfer direction. The correct fix mirrors the ICS-20 fungible token transfer approach: the receiving chain should only enter the unescrow branch if it can verify that it previously committed a send packet for the given class and token. Concretely:

1. **Verify against committed send packets**: before calling `TransferOwner`, check that a packet commitment exists (or existed) for the given `(classID, tokenID, destPort, destChannel)` tuple, proving the victim chain itself initiated the original escrow.
2. **Alternatively**, reject any incoming `data.ClassId` that starts with the receiving chain's own port/channel prefix unless it was registered in the local class-trace store as a result of a prior outbound send — i.e., validate that `voucherClassID` exists in the local NFT store and is owned by `escrowAddress` before proceeding, and cross-check against a committed-send record.

```go
func (nftpd NonFungibleTokenPacketData) ValidateBasic() error {
	if strings.TrimSpace(nftpd.ClassId) == "" {
		return newsdkerrors.Wrap(ErrInvalidClassID, "classId cannot be blank")
	}

	if len(nftpd.TokenIds) == 0 {
		return newsdkerrors.Wrap(ErrInvalidTokenID, "tokenId cannot be blank")
	}

	if len(nftpd.TokenIds) != len(nftpd.TokenUris) {
		return newsdkerrors.Wrap(ErrInvalidPacket, "tokenIds and tokenUris lengths do not match")
	}

	if strings.TrimSpace(nftpd.Sender) == "" {
		return newsdkerrors.Wrap(sdkerrors.ErrInvalidAddress, "sender address cannot be blank")
	}

	// decode the sender address
	if _, err := sdk.AccAddressFromBech32(nftpd.Sender); err != nil {
		return newsdkerrors.Wrap(sdkerrors.ErrInvalidAddress, "invalid sender address")
	}

	if strings.TrimSpace(nftpd.Receiver) == "" {
		return newsdkerrors.Wrap(sdkerrors.ErrInvalidAddress, "receiver address cannot be blank")
	}

	// decode the receiver address
	if _, err := sdk.AccAddressFromBech32(nftpd.Receiver); err != nil {
		return newsdkerrors.Wrap(sdkerrors.ErrInvalidAddress, "invalid receiver address")
	}
	return nil
}
```

```go
func GetEscrowAddress(portID, channelID string) sdk.AccAddress {
	// a slash is used to create domain separation between port and channel identifiers to
	// prevent address collisions between escrow addresses created for different channels
	contents := fmt.Sprintf("%s/%s", portID, channelID)

	// ADR 028 AddressHash construction
	preImage := []byte(Version)
	preImage = append(preImage, 0)
	preImage = append(preImage, contents...)
	hash := sha256.Sum256(preImage)
	return hash[:20]
```
