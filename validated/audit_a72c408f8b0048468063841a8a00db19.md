### Title
`add_class_and_executable_unsafe` Exposed Without Access Control Allows Arbitrary Class/Hash Injection — (`crates/apollo_class_manager/src/class_manager.rs`)

### Summary

`ClassManager::add_class_and_executable_unsafe` bypasses all compilation and hash-consistency validation and is exposed through the same `ClassManagerRequest` enum and `RemoteComponentServer` as the normal `add_class` path. There is no access-control gate separating the two callers. Any network-reachable actor can inject an arbitrary `(class_id, sierra, executable_class_hash_v2, casm)` tuple where none of the three hash relationships are verified, causing the class manager to store a permanently inconsistent mapping. Downstream, the gateway accepts declare transactions whose `compiled_class_hash` matches the injected fake hash, and the blockifier executes the wrong CASM artifact for that class hash.

### Finding Description

`ClassManager::add_class` (the normal path) compiles the Sierra class, derives `executable_class_hash_v2` from the compiler output, validates class length and version, and only then stores the triple `(class_id → sierra, executable_class_hash_v2, casm)`.

`ClassManager::add_class_and_executable_unsafe` skips every one of those steps:

```rust
pub fn add_class_and_executable_unsafe(
    &mut self,
    class_id: ClassId,
    class: RawClass,
    executable_class_hash_v2: ExecutableClassHash,
    executable_class: RawExecutableClass,
) -> ClassManagerResult<()> {
    Ok(self.classes.set_class(class_id, class, executable_class_hash_v2, executable_class)?)
}
``` [1](#0-0) 

`set_class` in `CachedClassStorage` stores the caller-supplied `executable_class_hash_v2` directly as the existence marker without hashing `executable_class`: [2](#0-1) 

The function is documented as "should only be used through state sync … bypasses compilation – thus unsafe," but this is a comment, not an enforcement: [3](#0-2) 

Both `AddClass` and `AddClassAndExecutableUnsafe` are variants of the same `ClassManagerRequest` enum and are dispatched by the same `ComponentRequestHandler`: [4](#0-3) 

The `RemoteComponentServer` that serves this enum has no authentication layer. Any caller that can reach the class manager's network port can send `ClassManagerRequest::AddClassAndExecutableUnsafe`.

The gateway's `convert_rpc_tx_to_internal` enforces that `tx.compiled_class_hash == executable_class_hash_v2` returned by the class manager: [5](#0-4) 

If the class manager has been poisoned with a fake `executable_class_hash_v2`, this check passes for a declare transaction carrying that fake hash, while the blockifier will execute the injected CASM artifact.

### Impact Explanation

**Critical – Wrong compiled class / CASM artifact selected for execution.**

After injection, `get_executable(class_id)` returns the attacker-supplied CASM, and `get_executable_class_hash_v2(class_id)` returns the attacker-supplied hash. The blockifier's `ClassInfo` is built from these two values: [6](#0-5) 

The executed bytecode is the injected CASM, not the bytecode whose hash was committed to in the transaction. The state diff records the fake hash as the `compiled_class_hash` for the declared class, permanently corrupting the class registry.

**High – Gateway/mempool admission accepts an invalid declare transaction.**

The gateway's compiled-class-hash mismatch check is defeated because both sides of the comparison (`tx.compiled_class_hash` and the value returned by the class manager) are the attacker-controlled fake hash.

### Likelihood Explanation

In a distributed deployment the class manager runs as a separate process behind a `RemoteComponentServer`. The server port is reachable by any component (or attacker) on the same network segment. There is no token, TLS client certificate, or IP allowlist enforced at the `RemoteComponentServer` layer. The `AddClassAndExecutableUnsafe` variant is indistinguishable from `AddClass` at the transport level.

### Recommendation

1. **Remove `AddClassAndExecutableUnsafe` from the public `ClassManagerRequest` enum** and expose it only through a separate, internal-only interface (e.g., a dedicated `StateSyncClassManagerClient` trait that is never backed by a `RemoteComponentServer`).
2. **Add a hash-consistency assertion** inside `add_class_and_executable_unsafe`: compute `hash(executable_class)` and assert it equals the supplied `executable_class_hash_v2` before storing.
3. **Add a class-id consistency assertion**: compute `sierra.calculate_class_hash()` and assert it equals the supplied `class_id`.
4. Analogous to the missing `purgeDeployer()` in M-15, add a mechanism (feature flag or compile-time separation) that prevents the unsafe path from being reachable after the bootstrapping/sync phase is complete.

### Proof of Concept

```
1. Attacker connects to the class manager's RemoteComponentServer port.

2. Attacker sends:
   ClassManagerRequest::AddClassAndExecutableUnsafe(
       class_id   = H(legitimate_sierra),   // real class hash
       class      = legitimate_sierra,
       executable_class_hash_v2 = 0xDEAD,  // fake hash
       executable_class = malicious_casm,   // arbitrary bytecode
   )

3. Class manager stores:
   executable_class_hashes_v2[H(legitimate_sierra)] = 0xDEAD
   executable_classes[H(legitimate_sierra)]          = malicious_casm

4. User submits a DeclareV3 transaction with:
   compiled_class_hash = 0xDEAD

5. Gateway calls class_manager.add_class(legitimate_sierra):
   → class already cached, returns ClassHashes { class_hash, executable_class_hash_v2: 0xDEAD }
   → check: tx.compiled_class_hash (0xDEAD) == executable_class_hash_v2 (0xDEAD) → PASSES

6. Blockifier builds ClassInfo with contract_class = malicious_casm.
   State diff records: H(legitimate_sierra) → compiled_class_hash 0xDEAD.
   All future calls to the declared class execute malicious_casm.
```

    #[instrument(skip(self, class, executable_class), level = "debug", ret, err)]
    fn set_class(
        &mut self,
        class_id: ClassId,
        class: RawClass,
        executable_class_hash_v2: ExecutableClassHash,
        executable_class: RawExecutableClass,
    ) -> Result<(), Self::Error> {
        if self.class_cached(class_id) {
            return Ok(());
        }

        self.storage.set_class(
            class_id,
            class.clone(),
            executable_class_hash_v2,
            executable_class.clone(),
        )?;

        increment_n_classes(CairoClassType::Regular);
        record_class_size(ClassObjectType::Sierra, &class);
        record_class_size(ClassObjectType::Casm, &executable_class);

        // Cache the class.
        // Done after successfully writing to storage as an optimization;
        // does not require atomicity.
        self.classes.set(class_id, class);
        self.executable_classes.set(class_id, executable_class);
        // Cache the executable class hash last; acts as an existence marker.
        self.executable_class_hashes_v2.set(class_id, executable_class_hash_v2);

        Ok(())
    }
