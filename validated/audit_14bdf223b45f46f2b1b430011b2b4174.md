Audit Report

## Title
`SwapAllowlistExtension` checks the router's address instead of the end user's address, allowing any caller to bypass the per-pool swap allowlist when the router is allowlisted — (`metric-periphery/contracts/extensions/SwapAllowlistExtension.sol`)

## Summary
`SwapAllowlistExtension.beforeSwap` gates swaps on `allowedSwapper[pool][sender]`, where `sender` is `msg.sender` from the pool's perspective — the direct caller of `pool.swap`. When any user routes through `MetricOmmSimpleRouter`, the pool's `msg.sender` is the router, so the extension evaluates `allowedSwapper[pool][router]`. A pool admin who allowlists the router to enable router-mediated swaps for their approved users inadvertently opens the gate to every caller of the router, fully defeating the allowlist.

## Finding Description
`MetricOmmPool.swap` passes `msg.sender` (the direct caller) as `sender` to `_beforeSwap`: [1](#0-0) 

`ExtensionCalling._beforeSwap` forwards `sender` unchanged to every configured extension: [2](#0-1) 

`SwapAllowlistExtension.beforeSwap` then evaluates:
```solidity
if (!allowAllSwappers[msg.sender] && !allowedSwapper[msg.sender][sender]) {
    revert IMetricOmmPoolActions.NotAllowedToSwap();
}
```
where `msg.sender` is the pool and `sender` is whoever called the pool: [3](#0-2) 

`MetricOmmSimpleRouter.exactInputSingle` calls `pool.swap(...)` directly, making the router `msg.sender` at the pool level: [4](#0-3) 

The same substitution occurs for `exactInput` (all hops call `pool.swap` from the router): [5](#0-4) 

And for `exactOutputSingle`: [6](#0-5) 

There is no mechanism in the extension or pool to recover the original EOA once the router is the direct caller. The `extensionData` bytes are caller-controlled and cannot be trusted as identity proof.

## Impact Explanation
Any unpermitted user can swap in a pool that has `SwapAllowlistExtension` active and the router allowlisted. The allowlist — the sole access-control gate on the swap path — is fully bypassed. Pools designed for restricted participants (e.g., institutional, KYC-gated, or partner-only pools) become open to the public. This is a direct admin-boundary break: an unprivileged path (routing through the public router) defeats a pool-admin-configured guard.

## Likelihood Explanation
The pool admin must allowlist the router. This is the natural and expected action when a pool admin wants their approved users to be able to use the standard periphery router rather than calling the pool directly. The admin's mental model — "I allowlist the router so my users can route through it" — is incorrect; the actual effect is "I allowlist every user who routes through it." The misunderstanding is easy to make because the router is a trusted, factory-registered contract, and `setAllowedToSwap` gives no indication that allowlisting an intermediary opens the gate to all downstream callers.

## Recommendation
The `SwapAllowlistExtension` should gate the economically relevant actor — the end user — not the intermediary. Two viable approaches:

1. **Router-forwarded identity**: Have `MetricOmmSimpleRouter` encode `msg.sender` into `extensionData` and have the extension decode and verify it against a trusted router registry to prevent spoofing.
2. **Direct-only allowlist**: Document clearly that `SwapAllowlistExtension` only works for direct pool calls, and that allowlisting the router defeats the gate. Pools that need router support should allowlist individual users, not the router contract.

## Proof of Concept
```
1. Pool admin deploys pool with SwapAllowlistExtension at extension slot 1,
   BEFORE_SWAP_ORDER = extension 1.

2. Admin allowlists the router:
   swapExtension.setAllowedToSwap(pool, address(router), true);
   // Intent: "let my approved users route through the router"

3. Admin does NOT allowlist attacker:
   swapExtension.isAllowedToSwap(pool, attacker) == false

4. Attacker calls:
   router.exactInputSingle(ExactInputSingleParams({
       pool: pool,
       recipient: attacker,
       zeroForOne: true,
       amountIn: X,
       ...
   }));

5. Pool calls extension.beforeSwap(router, attacker, ...):
   allowedSwapper[pool][router] == true  →  check passes

6. Attacker receives output tokens. Allowlist fully bypassed.
```
