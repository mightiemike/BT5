### Title
Depleted Bonus Rewards Pool Bricks User Exits, Undelegations, and Redelegations - (`x/tieredrewards/keeper/claim_rewards.go`)

---

### Summary

In `x/tieredrewards`, the `processEventsAndClaimBonus` function hard-reverts when the `RewardsPool` module account lacks sufficient balance to pay accrued bonus rewards. Because `claimRewards` (which calls `processEventsAndClaimBonus`) is invoked as a mandatory prerequisite inside `TierUndelegate`, `TierRedelegate`, `AddToTierPosition`, `ClearPosition`, and `ExitTierWithDelegation`, any user who has accrued non-zero bonus rewards is completely unable to exit, undelegate, or redelegate their position if the bonus pool is depleted. There is no fallback path.

---

### Finding Description

`processEventsAndClaimBonus` computes the total bonus owed to a position and, when that total is non-zero, calls `sufficientBonusPoolBalance` before transferring funds: [1](#0-0) 

If `sufficientBonusPoolBalance` returns an error (i.e., the `RewardsPool` module account balance is below the owed bonus), the function returns that error immediately with no fallback: [2](#0-1) 

This error propagates through `claimRewards`: [3](#0-2) 

`claimRewards` is called as a **mandatory, non-skippable step** inside five separate message handlers:

- `TierUndelegate` — line 166 [4](#0-3) 
- `TierRedelegate` — line 229 [5](#0-4) 
- `AddToTierPosition` — line 314 [6](#0-5) 
- `ClearPosition` — line 406 [7](#0-6) 
- `ExitTierWithDelegation` — line 532 [8](#0-7) 

In every case, if `claimRewards` returns an error, the entire transaction reverts and the user's requested operation (undelegate, redelegate, exit) does not execute. There is no alternative code path that skips or defers the bonus payment and proceeds with the principal operation.

---

### Impact Explanation

Any user who has been in a tier long enough to accrue a non-zero bonus reward is permanently blocked from:

- Undelegating their staked tokens (`TierUndelegate`)
- Redelegating to a different validator (`TierRedelegate`)
- Exiting their position with delegation intact (`ExitTierWithDelegation`)
- Clearing a triggered exit (`ClearPosition`)
- Adding more funds to a position (`AddToTierPosition`)

Their principal (staked tokens) is effectively locked in the protocol with no recovery path until the bonus pool is replenished by an external governance/admin action. This is a direct user-fund-locking impact, not merely a reward dilution.

The exact corrupted invariant: the protocol promises users can always exit a tier position (via `TierUndelegate` or `ExitTierWithDelegation`), but this invariant is broken whenever `RewardsPool` balance < accrued bonus for any position attempting to exit.

---

### Likelihood Explanation

The `RewardsPool` is a finite module account funded by governance or inflation. It can be depleted by:

1. A large number of users simultaneously claiming rewards via `ClaimTierRewards` (which also calls `claimRewardsAndUpdatesPositions`).
2. The pool simply not being topped up frequently enough relative to accrual rates.
3. The ABCI end-blocker `claimRewardsAndUpdateTierPositions` draining the pool automatically each block for all positions in a tier. [9](#0-8) 

Once the pool is depleted, every user with accrued bonus is affected simultaneously — the condition is not user-specific but pool-wide.

---

### Recommendation

Decouple the bonus payment from the principal operation. When `sufficientBonusPoolBalance` fails (or when the pool balance is less than the owed bonus), the implementation should either:

1. **Skip the bonus payment and proceed**: Record the owed bonus as a claimable debt, allow the undelegate/exit to complete, and let the user claim the bonus separately once the pool is refilled.
2. **Pay a partial bonus**: Transfer only what the pool currently holds and proceed with the operation.

The critical invariant to preserve is: **a user must always be able to exit or undelegate their position regardless of the bonus pool state**. Bonus rewards are secondary to principal recovery.

---

### Proof of Concept

1. Alice locks tokens in a tier via `MsgLockTier` and accrues bonus rewards over several blocks.
2. The `RewardsPool` module account is drained (e.g., by many concurrent `ClaimTierRewards` calls or by the ABCI end-blocker).
3. Alice submits `MsgTierUndelegate` to recover her principal.
4. Execution reaches `ms.claimRewards(ctx, pos)` at line 166 of `msg_server.go`.
5. Inside `processEventsAndClaimBonus`, `totalBonus > 0`, so `sufficientBonusPoolBalance` is called.
6. The check fails because the pool is empty; the function returns an error.
7. `TierUndelegate` reverts. Alice's staked tokens remain locked with no recovery path.
8. Alice cannot redelegate, exit with delegation, or clear her position either — all five exit paths are blocked by the same root cause. [10](#0-9)

func (k Keeper) claimRewardsAndUpdateTierPositions(ctx context.Context, tierId uint32) error {
	ids, err := k.getPositionsIdsByTier(ctx, tierId)
	if err != nil {
		return err
	}
	if len(ids) == 0 {
		return nil
	}

	for _, id := range ids {
		pos, err := k.getPositionState(ctx, id)
		if err != nil {
			return err
		}
		if !pos.IsDelegated() {
			continue
		}

		if _, err := k.claimBaseRewards(ctx, pos); err != nil {
			return err
		}
		if _, err := k.processEventsAndClaimBonus(ctx, &pos); err != nil {
			return err
		}
		if err := k.setPosition(ctx, pos.Position, nil); err != nil {
			return err
		}
	}

	return nil

func (k Keeper) claimRewards(ctx context.Context, pos types.PositionState) (types.PositionState, sdk.Coins, sdk.Coins, error) {
	if !pos.IsDelegated() {
		return pos, sdk.NewCoins(), sdk.NewCoins(), nil
	}

	base, err := k.claimBaseRewards(ctx, pos)
	if err != nil {
		return types.PositionState{}, nil, nil, err
	}

	bonus, err := k.processEventsAndClaimBonus(ctx, &pos)
	if err != nil {
		return types.PositionState{}, nil, nil, err
	}

	return pos, base, bonus, nil
}

func (k Keeper) processEventsAndClaimBonus(ctx context.Context, pos *types.PositionState) (sdk.Coins, error) {
	// Rewards should have been claimed before undelegation
	if !pos.IsDelegated() {
		return sdk.NewCoins(), nil
	}

	valAddr, err := sdk.ValAddressFromBech32(pos.Delegation.ValidatorAddress)
	if err != nil {
		return nil, err
	}

	events, err := k.getValidatorEventsSince(ctx, valAddr, pos.LastEventSeq)
	if err != nil {
		return nil, err
	}

	sdkCtx := sdk.UnwrapSDKContext(ctx)
	blockTime := sdkCtx.BlockTime()

	totalBonus := math.ZeroInt()
	// Use the persisted bonded state from the last replay, not a hardcoded default.
	// This prevents overpaying bonus for unbonded gaps between claims.
	bonded := pos.LastKnownBonded
	segmentStart := pos.LastBonusAccrual

	tier, err := k.getTier(ctx, pos.TierId)
	if err != nil {
		return nil, err
	}

	for _, entry := range events {
		evt := entry.Event

		if bonded {
			// Compute bonus for the bonded segment [segmentStart, eventTime]
			// using the snapshot rate at the event.
			bonus := k.computeSegmentBonus(*pos, tier, segmentStart, evt.Timestamp, evt.TokensPerShare)
			totalBonus = totalBonus.Add(bonus)
		}

		// Update bonded state based on event type.
		switch evt.EventType {
		case types.ValidatorEventType_VALIDATOR_EVENT_TYPE_UNBOND:
			bonded = false
		case types.ValidatorEventType_VALIDATOR_EVENT_TYPE_BOND:
			bonded = true
		case types.ValidatorEventType_VALIDATOR_EVENT_TYPE_SLASH:
			// Slash doesn't change bonded state.
		}

		segmentStart = evt.Timestamp
		pos.UpdateLastEventSeq(entry.Seq)

		// Decrement reference count.
		if err := k.decrementEventRefCount(ctx, valAddr, entry.Seq); err != nil {
			return nil, err
		}
	}

	val, err := k.stakingKeeper.GetValidator(ctx, valAddr)
	if err != nil {
		return nil, err
	}
	// Defensive: validator bond status check
	if bonded && val.IsBonded() {
		currentRate, err := k.getTokensPerShare(ctx, valAddr)
		if err != nil {
			return nil, err
		}
		bonus := k.computeSegmentBonus(*pos, tier, segmentStart, blockTime, currentRate)
		totalBonus = totalBonus.Add(bonus)
	}

	applyBonusAccrualCheckpoint(&pos.Position, blockTime)
	// Persist the bonded state so the next replay starts correctly.
	pos.UpdateLastKnownBonded(bonded)

	if totalBonus.IsZero() {
		return sdk.NewCoins(), nil
	}

	bondDenom, err := k.stakingKeeper.BondDenom(ctx)
	if err != nil {
		return nil, err
	}

	bonusCoins := sdk.NewCoins(sdk.NewCoin(bondDenom, totalBonus))

	if err := k.sufficientBonusPoolBalance(ctx, bonusCoins); err != nil {
		return nil, err
	}

	ownerAddr, err := sdk.AccAddressFromBech32(pos.Owner)
	if err != nil {
		return nil, errorsmod.Wrap(sdkerrors.ErrInvalidAddress, "invalid owner address")
	}

	if err := k.bankKeeper.SendCoinsFromModuleToAccount(ctx, types.RewardsPoolName, ownerAddr, bonusCoins); err != nil {
		return nil, err
	}

	if err := sdkCtx.EventManager().EmitTypedEvent(&types.EventBonusRewardsClaimed{
		PositionId: pos.Id,
		Owner:      pos.Owner,
		Rewards:    bonusCoins,
	}); err != nil {
		return nil, err
	}

	return bonusCoins, nil
}
