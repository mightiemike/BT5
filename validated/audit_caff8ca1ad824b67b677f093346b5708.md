### Title
Duplicate Position IDs in `MsgClaimTierRewards` Enable Multiple Bonus Reward Claims from the Same Position — (File: `x/tieredrewards/keeper/msg_server.go`, `x/tieredrewards/keeper/claim_rewards.go`)

---

### Summary

`ClaimTierRewards` loads all requested positions into a slice **before** any state is updated, then iterates over that slice to claim and persist rewards. Because there is no deduplication of `PositionIds` in the handler, a caller who submits the same position ID multiple times receives the current-segment bonus reward for each duplicate entry, draining the `RewardsPoolName` module account.

---

### Finding Description

`ClaimTierRewards` in `msg_server.go` collects all positions into a local slice and then delegates to `claimRewardsAndUpdatesPositions`: [1](#0-0) 

```go
positions := make([]types.PositionState, 0, len(msg.PositionIds))
for _, posId := range msg.PositionIds {
    pos, err := ms.getPositionState(ctx, posId)
    ...
    positions = append(positions, pos)   // snapshot loaded here
}
totalBase, totalBonus, err := ms.claimRewardsAndUpdatesPositions(ctx, positions)
```

If `msg.PositionIds = [X, X]`, two identical snapshots of position `X` (same `LastBonusAccrual`, same `LastEventSeq`, same `LastKnownBonded`) are appended to the slice **before any claiming occurs**.

`claimRewardsAndUpdatesPositions` then iterates over the slice: [2](#0-1) 

```go
for i := range positions {
    pos := &positions[i]
    base, err := k.claimBaseRewards(ctx, *pos)
    bonus, err := k.processEventsAndClaimBonus(ctx, pos)
    k.setPosition(ctx, pos.Position, nil)   // persists updated checkpoint
}
```

**Iteration 0** (first copy of position X):
- `processEventsAndClaimBonus` computes the current-segment bonus using `pos.LastBonusAccrual` (old value) as `segmentStart` and `blockTime` as the end.
- `bankKeeper.SendCoinsFromModuleToAccount` transfers bonus coins from `RewardsPoolName` to the owner.
- `applyBonusAccrualCheckpoint` advances `pos.LastBonusAccrual` to `blockTime` **in the in-memory copy**.
- `setPosition` persists the updated checkpoint to the store. [3](#0-2) 

**Iteration 1** (second copy of position X, loaded before iteration 0 ran):
- `pos.LastBonusAccrual` is still the **old** value (the snapshot was taken before iteration 0).
- `processEventsAndClaimBonus` recomputes the identical current-segment bonus `[old_LastBonusAccrual, blockTime]`.
- `bankKeeper.SendCoinsFromModuleToAccount` transfers the same bonus coins **again**.
- `setPosition` overwrites the store with the second copy's checkpoint (which is identical to what iteration 0 already wrote, so no net harm to the checkpoint — but the coins are already gone).

The root cause is that the **checks-effects-interactions** pattern is violated: all positions are loaded (checked) before any state is updated (effects), so duplicate entries in the input slice each see the pre-claim checkpoint and each trigger a full bonus payout.

---

### Impact Explanation

An attacker who owns any position with non-zero accrued bonus can submit a single `MsgClaimTierRewards` transaction with the same `positionId` repeated `N` times. Each repetition independently computes and transfers the current-segment bonus from the `RewardsPoolName` module account to the owner. The attacker receives `N × bonus_amount` instead of `1 × bonus_amount`, limited only by the pool balance (guarded by `sufficientBonusPoolBalance`, which is checked per-iteration against the live pool balance — so the loop terminates when the pool is exhausted, but by then the pool may be fully drained).

The corrupted value is the `RewardsPoolName` module account balance, which is shared across all tier participants. Draining it denies bonus rewards to every other position holder.

---

### Likelihood Explanation

The entry path is a standard, unprivileged Cosmos SDK transaction (`MsgClaimTierRewards`) signed by any position owner. No special role, governance action, or privileged key is required. The only prerequisite is owning at least one position with a non-zero current-segment bonus, which is the normal operating state for any active position. The attack is a single transaction.

---

### Recommendation

1. **Deduplicate `PositionIds` in `MsgClaimTierRewards.Validate()`** — reject the message if any ID appears more than once.
2. **Alternatively, re-read position state from the store inside `claimRewardsAndUpdatesPositions`** rather than operating on pre-loaded snapshots, so that the second encounter of the same ID sees the already-advanced checkpoint.
3. **Follow checks-effects-interactions**: persist the updated checkpoint (`setPosition`) before sending coins, or at minimum before processing the next position in the loop.

---

### Proof of Concept

1. Alice owns position ID `42` with a non-zero current-segment bonus of `100 BASECRO`.
2. Alice submits `MsgClaimTierRewards{Owner: alice, PositionIds: [42, 42, 42, 42, 42]}`.
3. `ClaimTierRewards` loads five identical snapshots of position 42 (all with `LastBonusAccrual = T0`).
4. `claimRewardsAndUpdatesPositions` iterates five times; each iteration computes bonus for `[T0, blockTime]` = `100 BASECRO` and calls `SendCoinsFromModuleToAccount`.
5. Alice receives `500 BASECRO` instead of `100 BASECRO`; the `RewardsPoolName` account is debited `500 BASECRO`.
6. Repeating with larger `N` or across multiple transactions drains the pool entirely, breaking bonus rewards for all other participants.

func (ms msgServer) ClaimTierRewards(ctx context.Context, msg *types.MsgClaimTierRewards) (*types.MsgClaimTierRewardsResponse, error) {
	if err := msg.Validate(); err != nil {
		return nil, err
	}

	positions := make([]types.PositionState, 0, len(msg.PositionIds))
	for _, posId := range msg.PositionIds {
		pos, err := ms.getPositionState(ctx, posId)
		if err != nil {
			return nil, err
		}

		if err := ms.validateClaimRewards(pos.Position, msg.Owner); err != nil {
			return nil, err
		}

		positions = append(positions, pos)
	}

	totalBase, totalBonus, err := ms.claimRewardsAndUpdatesPositions(ctx, positions)
	if err != nil {
		return nil, err
	}

func (k Keeper) claimRewardsAndUpdatesPositions(ctx context.Context, positions []types.PositionState) (sdk.Coins, sdk.Coins, error) {
	totalBase := sdk.NewCoins()
	totalBonus := sdk.NewCoins()

	for i := range positions {
		pos := &positions[i]

		if !pos.IsDelegated() {
			continue
		}

		base, err := k.claimBaseRewards(ctx, *pos)
		if err != nil {
			return nil, nil, err
		}
		totalBase = totalBase.Add(base...)

		bonus, err := k.processEventsAndClaimBonus(ctx, pos)
		if err != nil {
			return nil, nil, err
		}
		totalBonus = totalBonus.Add(bonus...)

		if err := k.setPosition(ctx, pos.Position, nil); err != nil {
			return nil, nil, err
		}
	}

	return totalBase, totalBonus, nil
}
