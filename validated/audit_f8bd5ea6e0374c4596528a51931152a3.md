### Title
Missing Destination-Prefix Guard in `processReceivedPacket` Enables Malicious Counterparty to Permanently Lock Escrowed NFTs — (`x/nft-transfer/keeper/packet.go`)

---

### Summary

`processReceivedPacket` unconditionally prepends the destination port/channel prefix to `data.ClassId` when `isAwayFromOrigin` is `true`, with no check that the incoming `data.ClassId` does not already carry that prefix. A malicious IBC counterparty can exploit this to cause double-prefixing, minting vouchers under a permanently wrong `voucherClassID` and leaving the originally escrowed NFT on Cronos POS Chain locked forever.

---

### Finding Description

**Entry point:** `IBCModule.OnRecvPacket` → `keeper.OnRecvPacket` → `processReceivedPacket`

**`ValidateBasic` does not guard the classID structure.**

`NonFungibleTokenPacketData.ValidateBasic()` only checks that `ClassId` is non-blank, that token arrays are consistent, and that addresses are valid bech32. It performs zero structural validation of the classID path. [1](#0-0) 

**`IsAwayFromOrigin` is fooled by a pre-prefixed classID.**

```go
isAwayFromOrigin := types.IsAwayFromOrigin(
    packet.GetSourcePort(), packet.GetSourceChannel(), data.ClassId)
```

`IsAwayFromOrigin` returns `true` when `data.ClassId` does **not** start with `sourcePort/sourceChannel/`. If the malicious counterparty's source channel is `channel-1` but it sends `data.ClassId = "nft/channel-0/nftClass"` (using the *destination* prefix `nft/channel-0/`), the check evaluates `strings.HasPrefix("nft/channel-0/nftClass", "nft/channel-1/")` → `false`, so `isAwayFromOrigin = true`. [2](#0-1) 

**Double-prefixing occurs unconditionally.**

```go
classPrefix := types.GetClassPrefix(packet.GetDestPort(), packet.GetDestChannel())
// classPrefix = "nft/channel-0/"
prefixedClassID := classPrefix + data.ClassId
// = "nft/channel-0/" + "nft/channel-0/nftClass"
// = "nft/channel-0/nft/channel-0/nftClass"
``` [3](#0-2) 

**`ParseClassTrace` silently accepts the double-prefixed string.**

`ParseClassTrace("nft/channel-0/nft/channel-0/nftClass")` splits on `/` and takes everything except the last segment as `Path`:

- `Path = "nft/channel-0/nft/channel-0"`
- `BaseClassId = "nftClass"` [4](#0-3) 

**`SetClassTrace` is called without any `Validate()` call.**

```go
classTrace := types.ParseClassTrace(prefixedClassID)
if !k.HasClassTrace(ctx, classTrace.Hash()) {
    k.SetClassTrace(ctx, classTrace)   // no Validate() call
}
```

The resulting trace `{Path:"nft/channel-0/nft/channel-0", BaseClassId:"nftClass"}` even passes `validateTraceIdentifiers` (4 identifiers, even count, valid port/channel names), so it is stored without error. [5](#0-4) [6](#0-5) 

**NFT is minted under the wrong `voucherClassID`.**

```go
voucherClassID := classTrace.IBCClassID()
// = "ibc/" + sha256("nft/channel-0/nft/channel-0/nftClass")
// instead of "ibc/" + sha256("nft/channel-0/nftClass")
``` [7](#0-6) 

---

### Impact Explanation

**Permanent lock of escrowed NFTs on Cronos POS Chain.**

Concrete scenario:

1. User on Cronos POS Chain sends `nftClass` to counterparty via `nft/channel-0`. The NFT is escrowed on Cronos POS Chain; the counterparty mints `nft/channel-1/nftClass`.
2. Malicious counterparty sends back a packet with `data.ClassId = "nft/channel-0/nftClass"` (pre-prefixed with the *destination* prefix) instead of the correct `"nft/channel-1/nftClass"`.
3. Cronos POS Chain evaluates `isAwayFromOrigin = true` and mints a new voucher under `ibc/sha256("nft/channel-0/nft/channel-0/nftClass")` — it never calls `TransferOwner` to release the escrowed NFT.
4. The original `nftClass` NFT remains permanently locked in the escrow account. No future packet can release it: any subsequent packet with `data.ClassId = "nft/channel-0/nftClass"` from the counterparty will again be treated as away-from-origin and will again mint a new (worthless) voucher.

The wrongly minted voucher also cannot be correctly unwound: sending it back burns it on Cronos POS Chain (sink-chain path) and sends `ClassId = "nft/channel-0/nft/channel-0/nftClass"` to the counterparty, which the counterparty will treat as away-from-origin and mint yet another layer of vouchers.

---

### Likelihood Explanation

Requires a malicious IBC counterparty chain — a chain that deliberately crafts packet data violating ICS-721 semantics. This is within the IBC threat model: the receiving chain is responsible for validating all incoming packet fields. The exploit is trivially constructable (one crafted packet), requires no special privileges on Cronos POS Chain, and is reachable through the standard IBC NFT packet lifecycle path.

---

### Recommendation

In `processReceivedPacket`, before prepending the destination prefix, assert that `data.ClassId` does **not** already start with the destination prefix:

```go
classPrefix := types.GetClassPrefix(packet.GetDestPort(), packet.GetDestChannel())
if strings.HasPrefix(data.ClassId, classPrefix) {
    return sdkerrors.Wrapf(types.ErrInvalidClassID,
        "classId %s already contains destination prefix %s", data.ClassId, classPrefix)
}
prefixedClassID := classPrefix + data.ClassId
```

Additionally, call `classTrace.Validate()` before `SetClassTrace` to enforce the invariant that the path consists of valid, even-count port/channel pairs representing the true hop history. [8](#0-7)

### Proof of Concept

```go
// Keeper integration test (unmodified Go/Cosmos test setup)
func TestDoublePrefix(t *testing.T) {
    // Setup: channel-0 on destination (Cronos POS Chain)
    destPort, destChannel := "nft", "channel-0"
    srcPort,  srcChannel  := "nft", "channel-1"   // counterparty

    // Malicious packet: classId already carries the destination prefix
    data := types.NonFungibleTokenPacketData{
        ClassId:   "nft/channel-0/nftClass",   // pre-prefixed
        TokenIds:  []string{"token1"},
        TokenUris: []string{"uri1"},
        Sender:    maliciousSender.String(),
        Receiver:  victim.String(),
    }
    packet := channeltypes.NewPacket(data.GetBytes(), 1,
        srcPort, srcChannel, destPort, destChannel, ...)

    err := keeper.OnRecvPacket(ctx, "", packet, data)
    require.NoError(t, err)

    // Assert: stored trace has FOUR path segments (double-prefixed)
    trace := types.ParseClassTrace("nft/channel-0/" + data.ClassId)
    // trace.Path == "nft/channel-0/nft/channel-0"  ← two identical hops
    segments := strings.Split(trace.Path, "/")
    require.Equal(t, 4, len(segments), "path must have exactly 2 hops (4 segments)")

    // Assert: voucherClassID is wrong — sha256 of double-prefixed path
    voucherClassID := trace.IBCClassID()
    correctTrace := types.ParseClassTrace("nft/channel-0/nftClass")
    require.NotEqual(t, correctTrace.IBCClassID(), voucherClassID,
        "voucherClassID must differ from the correct single-hop classID")

    // Assert: original escrowed NFT is still locked (TransferOwner was never called)
    _, found := nftKeeper.GetNFT(ctx, "nftClass", "token1")
    require.True(t, found, "original NFT must still be in escrow — it was never released")
}
```

func ParseClassTrace(rawClassID string) ClassTrace {
	classSplit := strings.Split(rawClassID, "/")

	if classSplit[0] == rawClassID {
		return ClassTrace{
			Path:        "",
			BaseClassId: rawClassID,
		}
	}

	return ClassTrace{
		Path:        strings.Join(classSplit[:len(classSplit)-1], "/"),
		BaseClassId: classSplit[len(classSplit)-1],
	}
}
