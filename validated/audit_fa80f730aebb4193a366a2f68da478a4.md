### Title
Non-Creator NFT Owner Permanently Blocked from Burning Their Own NFT — (`x/nft/keeper/keeper.go`)

### Summary
`Keeper.BurnNFT` requires the caller to be **both** the current NFT owner **and** the denom creator. Once an NFT is transferred to any address that is not the denom creator, that address can never burn the NFT via `MsgBurnNFT`, even though they legitimately own it. The NFT becomes permanently unburnable by the new owner.

### Finding Description
`BurnNFT` at [5](#0-4)  applies two sequential guards: `IsOwner` (line 146) and `IsDenomCreator` (line 151). `TransferOwner` only enforces `IsOwner`, so ownership can freely move to non-creators. After such a transfer, the new owner passes `IsOwner` but fails `IsDenomCreator`, returning `ErrUnauthorized` with "not the creator of". There is no production `MsgServer` path that calls `BurnNFTUnverified` for user-initiated burns — that function is reserved for IBC sink-chain voucher destruction. [6](#0-5) 

### Impact Explanation
Any NFT transferred away from the denom creator becomes permanently unburnable by the recipient through any externally reachable transaction. The recipient holds an NFT they cannot dispose of by burning. The denom creator retains exclusive burn authority regardless of who holds the token. This breaks the ownership invariant: ownership (via `TransferOwner`) and burn authority are decoupled with no mechanism for the new owner to recover burn rights.

### Likelihood Explanation
This is triggered by the normal, supported `MsgTransferNFT` → `MsgBurnNFT` sequence. No special privileges are needed. The existing keeper test at [4](#0-3)  explicitly asserts this failure, confirming it is the intended (but broken) behavior baked into production code.

### Recommendation
Remove the `IsDenomCreator` check from `BurnNFT`. Burning should only require `IsOwner`. The denom creator check is appropriate for `MintNFT` and `EditNFT` (where creator-only control is a design choice), but not for burning — the owner of an NFT should always be able to destroy it. Alternatively, expose `BurnNFTUnverified` via a separate `MsgServer` handler gated only on ownership.

### Proof of Concept
```
1. Alice calls MsgIssueDenom  → alice is denom creator
2. Alice calls MsgMintNFT     → token1 minted, owner = alice
3. Alice calls MsgTransferNFT → token1 owner = bob
4. Bob calls MsgBurnNFT(token1)
   → BurnNFT: IsOwner(bob) ✓
   → BurnNFT: IsDenomCreator(bob) ✗ → ErrUnauthorized "bob is not the creator of <denomID>"
5. Bob can never burn token1 via any production transaction path.
```

This matches the exact sequence already tested at [7](#0-6) , confirming the invariant mismatch is real and reachable.

func (k Keeper) BurnNFT(ctx sdk.Context, denomID, tokenID string, owner sdk.AccAddress) error {
	if !k.HasDenomID(ctx, denomID) {
		return sdkerrors.Wrapf(types.ErrInvalidDenom, "denom ID %s not exists", denomID)
	}

	nft, err := k.IsOwner(ctx, denomID, tokenID, owner)
	if err != nil {
		return err
	}

	_, err = k.IsDenomCreator(ctx, denomID, owner)
	if err != nil {
		return err
	}

	k.deleteNFT(ctx, denomID, nft)
	k.deleteOwner(ctx, denomID, tokenID, owner)
	k.decreaseSupply(ctx, denomID)

	return nil
}

func (suite *KeeperSuite) TestBurnNFT() {

	suite.NoError(err)

	// BurnNFT should fail when address is not the owner of nft
	err = suite.keeper.BurnNFT(suite.ctx, denomID, tokenID, address2)
	if suite.Error(err) {
		suite.EqualError(err, fmt.Sprintf("%s is not the owner of %s/%s: %s", address2, denomID, tokenID, types.ErrUnauthorized))
	}

	// Transfer nft to an address which is not the creator of denom
	err = suite.keeper.TransferOwner(suite.ctx, denomID, tokenID, address, address2)
	suite.NoError(err)

	// BurnNFT should fail when address is not the creator of denom
	err = suite.keeper.BurnNFT(suite.ctx, denomID, tokenID, address2)
	if suite.Error(err) {
		suite.EqualError(err, fmt.Sprintf("%s is not the creator of %s: %s", address2, denomID, types.ErrUnauthorized))
	}

	// Transfer nft back to the creator of denom
	err = suite.keeper.TransferOwner(suite.ctx, denomID, tokenID, address2, address)
	suite.NoError(err)

	// BurnNFT shouldn't fail when NFT exists and address is owner of nft and creator of denom
	err = suite.keeper.BurnNFT(suite.ctx, denomID, tokenID, address)
	suite.NoError(err)

	// NFT should no longer exist
	isNFT := suite.keeper.HasNFT(suite.ctx, denomID, tokenID)
	suite.False(isNFT)
