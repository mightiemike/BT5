Audit Report

## Title
SwapAllowlistExtension Per-User Gate Bypassed via Router Allowlist — (File: metric-periphery/contracts/extensions/SwapAllowlistExtension.sol)

## Summary
`SwapAllowlistExtension.beforeSwap` gates swaps by checking `allowedSwapper[pool][sender]`, where `sender` is the direct caller of `pool.swap()`. When swaps are routed through `MetricOmmSimpleRouter`, the pool receives `msg.sender = router`, so the extension evaluates `allowedSwapper[pool][router]` rather than `allowedSwapper[pool][user]`. Any pool admin who allowlists the router address to permit router-mediated swaps inadvertently grants every unprivileged user the ability to bypass the per-user allowlist entirely.

## Finding Description

`SwapAllowlistExtension.beforeSwap` performs its identity check as:

```solidity
if (!allowAllSwappers[msg.sender] && !allowedSwapper[msg.sender][sender]) {
    revert IMetricOmmPoolActions.NotAllowedToSwap();
}
``` [1](#0-0) 

Here `msg.sender` is the pool (the extension's caller) and `sender` is the first argument forwarded by the pool from its own `msg.sender`:

```solidity
_beforeSwap(
    msg.sender,   // ← direct caller of pool.swap(), not the originating EOA
    recipient, ...
);
``` [2](#0-1) 

`MetricOmmSimpleRouter.exactInputSingle` calls `pool.swap()` directly with no mechanism to surface the originating user's address to the pool or extension:

```solidity
_setNextCallbackContext(params.pool, CALLBACK_MODE_JUST_PAY, msg.sender, params.tokenIn);
(int128 amount0Delta, int128 amount1Delta) = IMetricOmmPoolActions(params.pool)
  .swap(params.recipient, params.zeroForOne, ...);
``` [3](#0-2) 

The originating user's address is stored only in transient storage for the payment callback (`_getPayer()`); it is never passed to the pool or the extension. The same structural gap exists in `exactInput` (multi-hop) and `exactOutputSingle`/`exactOutput` paths. [4](#0-3) 

**Exploit flow:**
1. Pool admin deploys a pool with `SwapAllowlistExtension` to restrict swaps to a curated set of addresses.
2. Admin calls `setAllowedToSwap(pool, router, true)` to permit router-mediated swaps for allowlisted users.
3. Any unprivileged user calls `MetricOmmSimpleRouter.exactInputSingle(...)` targeting the allowlist-gated pool.
4. The pool receives `msg.sender = router`; the extension evaluates `allowedSwapper[pool][router] == true` and passes — the per-user allowlist is fully bypassed.

The extension has no way to distinguish which EOA initiated the router call, and the router provides no mechanism to forward the originating user's identity to the extension.

## Impact Explanation
The `SwapAllowlistExtension` is a core access-control hook. Its bypass allows any unprivileged user to execute swaps in a pool that the admin explicitly intended to restrict. This breaks the admin-set access boundary and constitutes broken core pool functionality: the allowlist extension cannot enforce per-user swap restrictions when the router is involved, rendering the extension's primary purpose ineffective for any pool that permits router-mediated swaps.

## Likelihood Explanation
The precondition — the router being allowlisted — is a natural and expected admin action for any pool that intends to support router-mediated swaps alongside per-user access control. Once the router is allowlisted (a single admin transaction), the bypass is trivially reachable by any user with no special privileges, is repeatable, and requires no additional setup.

## Recommendation
The extension should verify the originating user rather than the direct caller. Options include:
1. **Pass the originating user via `extensionData`**: The router encodes `msg.sender` into `extensionData`; the extension decodes and verifies it. This requires a coordinated change to both the router and extension.
2. **Reject contract callers**: The extension reverts if `sender` is a contract (e.g., `sender.code.length > 0`) unless `allowAllSwappers[pool]` is set, forcing direct EOA swaps only.
3. **Dedicated router with identity forwarding**: Replace the public router with one that passes the originating user's address through a verified channel the extension can trust.

## Proof of Concept
```solidity
// Setup
SwapAllowlistExtension ext = ...;
address pool = ...; // pool configured with SwapAllowlistExtension
address router = address(metricOmmSimpleRouter);
address alice = makeAddr("alice"); // NOT in allowlist
address bob = makeAddr("bob");     // IS in allowlist

// Admin allowlists bob and the router (to allow router-mediated swaps)
ext.setAllowedToSwap(pool, bob, true);
ext.setAllowedToSwap(pool, router, true);

// Alice (not allowlisted) routes through the router
vm.prank(alice);
metricOmmSimpleRouter.exactInputSingle(ExactInputSingleParams({
    pool: pool,
    recipient: alice,
    tokenIn: token0,
    amountIn: 1e18,
    amountOutMinimum: 0,
    zeroForOne: true,
    priceLimitX64: 0,
    deadline: block.timestamp,
    extensionData: ""
}));
// ✓ Swap succeeds — alice bypassed the allowlist via the router
```
