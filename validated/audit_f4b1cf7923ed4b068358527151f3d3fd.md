### Title
`ValidResourceBounds::AllResources` with zero L2/L1-data bounds serializes identically to `L1Gas` over protobuf, causing type mutation and transaction hash divergence in block sync — (`File: crates/apollo_protobuf/src/converters/transaction.rs`)

---

### Summary

The protobuf serializer for `ValidResourceBounds::L1Gas` emits `l1_data_gas: Some(zero)` instead of `None`. The deserializer then uses a zero-value check to decide which variant to reconstruct. As a result, a valid V3 transaction carrying `AllResources({l1_gas: X, l2_gas: 0, l1_data_gas: 0})` round-trips through protobuf as `L1Gas(X)`. Because `get_tip_resource_bounds_hash` hashes a different number of resource-bound felts for each variant, the transaction hash changes after the round-trip. Any node that recomputes transaction hashes from deserialized block-sync data will derive a hash that disagrees with the proposer's hash, causing valid blocks to be rejected or wrong hashes to be stored.

---

### Finding Description

**Serialization side** (`From<ValidResourceBounds> for protobuf::ResourceBounds`):

```rust
ValidResourceBounds::L1Gas(l1_gas) => protobuf::ResourceBounds {
    l1_gas: Some(l1_gas.into()),
    l2_gas: Some(value.get_l2_bounds().into()),       // → zero
    l1_data_gas: Some(ResourceBounds::default().into()), // → zero (should be None)
},
``` [1](#0-0) 

Both `L1Gas(X)` and `AllResources({X, 0, 0})` produce the identical wire bytes `{l1_gas: X, l2_gas: 0, l1_data_gas: 0}`.

**Deserialization side** (`TryFrom<protobuf::ResourceBounds> for ValidResourceBounds`):

```rust
Ok(if l1_data_gas.is_zero() && l2_gas.is_zero() {
    ValidResourceBounds::L1Gas(l1_gas)          // ← AllResources mutates to L1Gas here
} else {
    ValidResourceBounds::AllResources(...)
})
``` [2](#0-1) 

**Hash divergence** (`get_tip_resource_bounds_hash`):

For `L1Gas`, the poseidon input is `[tip, L1_GAS_packed, L2_GAS_packed(0)]` — **3 elements**.  
For `AllResources`, the poseidon input is `[tip, L1_GAS_packed, L2_GAS_packed(0), L1_DATA_GAS_packed(0)]` — **4 elements**. [3](#0-2) 

Poseidon over 3 elements ≠ Poseidon over 4 elements even when the 4th element is zero, so the transaction hash changes after the round-trip.

**The gateway accepts the trigger transaction.** The stateless validator explicitly allows `AllResourceBounds { l1_gas: NON_EMPTY, l2_gas: default(0), l1_data_gas: default(0) }`: [4](#0-3) 

The `InternalRpcTransactionWithoutTxHash` always stores `AllResources` (since `RpcTransaction` only carries `AllResourceBounds`), so the gateway computes hash H_all. After protobuf block-sync serialization and deserialization, the receiving node reconstructs `L1Gas` and computes hash H_l1 ≠ H_all. [5](#0-4) 

---

### Impact Explanation

**High — Transaction conversion binds the wrong hash and type.**

A receiving node that recomputes transaction hashes from deserialized block-sync data (e.g., to validate the block hash, which in Starknet commits to all transaction hashes) will derive H_l1 for a transaction whose canonical hash is H_all. This causes:

1. **Valid block rejected**: block hash validation fails on the receiving node, preventing it from advancing its chain tip.
2. **Wrong hash stored**: if hash validation is skipped, the transaction is indexed under H_l1, making it unreachable by the original hash H_all.

Both outcomes match the allowed impact: *"Transaction conversion or signature/hash logic binds the wrong signer, hash, type, or executable payload."*

---

### Likelihood Explanation

**Medium.** The trigger condition — a V3 transaction with `AllResourceBounds { l1_gas: non-zero, l2_gas: zero, l1_data_gas: zero }` — is explicitly accepted by the gateway's stateless validator (test case `valid_l1_gas`). No special privilege is required; any user can submit such a transaction. The bug is latent until such a transaction is included in a block and that block is propagated via P2P block sync.

---

### Recommendation

1. **Fix the serializer**: emit `l1_data_gas: None` (not `Some(zero)`) for `ValidResourceBounds::L1Gas`, matching the proto schema comment *"This can be None only in transactions that don't support l2 gas"*:

```rust
ValidResourceBounds::L1Gas(l1_gas) => protobuf::ResourceBounds {
    l1_gas: Some(l1_gas.into()),
    l2_gas: Some(value.get_l2_bounds().into()),
    l1_data_gas: None,   // ← correct: absence signals L1Gas variant
},
```

2. **Fix the deserializer**: distinguish `L1Gas` from `AllResources` by the *presence* of `l1_data_gas`, not by its zero-value:

```rust
Ok(if value.l1_data_gas.is_none() {
    ValidResourceBounds::L1Gas(l1_gas)
} else {
    ValidResourceBounds::AllResources(AllResourceBounds { l1_gas, l2_gas, l1_data_gas })
})
```

3. **Add a gateway guard**: reject `AllResources` transactions where both `l2_gas` and `l1_data_gas` are entirely zero (amount and price), since they are semantically equivalent to `L1Gas` but produce a different hash.

---

### Proof of Concept

```
1. Attacker submits RPC InvokeV3 with:
     resource_bounds = AllResourceBounds {
         l1_gas:      { max_amount: 1000, max_price_per_unit: 1 },
         l2_gas:      { max_amount: 0,    max_price_per_unit: 0 },
         l1_data_gas: { max_amount: 0,    max_price_per_unit: 0 },
     }

2. Gateway (TransactionConverter::convert_rpc_tx_to_internal):
     tx_without_hash = InternalRpcTransactionWithoutTxHash::Invoke(...)
     tx_hash = tx_without_hash.calculate_transaction_hash(&chain_id)
     // Uses get_invoke_transaction_v3_hash → get_tip_resource_bounds_hash(AllResources)
     // Poseidon([tip, L1_GAS_packed, L2_GAS_packed(0), L1_DATA_GAS_packed(0)]) = H_all

3. Transaction included in block B. Block hash commits to H_all.

4. Block B propagated via P2P block sync. Receiving node deserializes InvokeV3:
     ValidResourceBounds::try_from(protobuf::ResourceBounds {
         l1_gas: Some({1000, 1}), l2_gas: Some({0,0}), l1_data_gas: Some({0,0})
     })
     // l1_data_gas.is_zero() && l2_gas.is_zero() → ValidResourceBounds::L1Gas({1000,1})

5. Receiving node recomputes hash:
     // Uses get_tip_resource_bounds_hash(L1Gas)
     // Poseidon([tip, L1_GAS_packed, L2_GAS_packed(0)]) = H_l1 ≠ H_all

6. Block hash validation: H_l1 ≠ H_all → block rejected / wrong hash stored.
```

// An implementation of the SNIP: https://github.com/EvyatarO/SNIPs/blob/snip-8/SNIPS/snip-8.md
pub fn get_tip_resource_bounds_hash(
    resource_bounds: &ValidResourceBounds,
    tip: &Tip,
) -> Result<Felt, StarknetApiError> {
    let l1_resource_bounds = resource_bounds.get_l1_bounds();
    let l2_resource_bounds = resource_bounds.get_l2_bounds();

    // L1 and L2 gas bounds always exist.
    // Old V3 txs always have L2 gas bounds of zero, but they exist.
    let mut resource_felts = vec![
        get_concat_resource(&l1_resource_bounds, L1_GAS)?,
        get_concat_resource(&l2_resource_bounds, L2_GAS)?,
    ];

    // For new V3 txs, need to also hash the data gas bounds.
    resource_felts.extend(match resource_bounds {
        ValidResourceBounds::L1Gas(_) => vec![],
        ValidResourceBounds::AllResources(all_resources) => {
            vec![get_concat_resource(&all_resources.l1_data_gas, L1_DATA_GAS)?]
        }
    });

    Ok(HashChain::new().chain(&tip.0.into()).chain_iter(resource_felts.iter()).get_poseidon_hash())
}
