Audit Report

## Title
`SwapAllowlistExtension` checks router address as `sender`, allowing any user to bypass per-user allowlist via router - (File: metric-periphery/contracts/extensions/SwapAllowlistExtension.sol)

## Summary
When a user swaps through `MetricOmmSimpleRouter`, the pool receives `msg.sender = router` and passes that as `sender` to `_beforeSwap` in `ExtensionCalling`. `SwapAllowlistExtension.beforeSwap` then checks `allowedSwapper[msg.sender][sender]` = `allowedSwapper[pool][router]`. If the router is allowlisted (a natural configuration), every unprivileged user bypasses the per-user allowlist gate entirely by routing through `MetricOmmSimpleRouter`.

## Finding Description

**Call path:**

1. User calls `MetricOmmSimpleRouter.exactInputSingle(params)`. [1](#0-0) 

2. The router calls `IMetricOmmPoolActions(params.pool).swap(recipient, zeroForOne, amount, priceLimit, "", extensionData)`. No explicit `sender` argument is passed; the pool derives `sender = msg.sender`, which is the **router address**. [2](#0-1) 

3. The pool calls `_beforeSwap(sender=router, ...)` in `ExtensionCalling`, which ABI-encodes and forwards `sender` to the extension. [3](#0-2) 

4. `SwapAllowlistExtension.beforeSwap` receives `sender = router` and checks:
   ```solidity
   if (!allowAllSwappers[msg.sender] && !allowedSwapper[msg.sender][sender]) revert NotAllowedToSwap();
   ```
   Here `msg.sender` = pool, `sender` = router. The check is `allowedSwapper[pool][router]`, **not** `allowedSwapper[pool][actualUser]`. [4](#0-3) 

**Root cause:** The extension's `sender` parameter is the immediate caller of `pool.swap()` (the router), not the originating user. The allowlist is designed to gate individual swapper addresses, but the router collapses all users into a single address.

**Exploit flow:**
- Pool admin configures `SwapAllowlistExtension` to restrict swaps to KYC'd addresses.
- Pool admin allowlists the router (reasonable: to let KYC'd users use the router).
- Any non-allowlisted attacker calls `router.exactInputSingle(...)`.
- The pool sees `sender = router`, which is allowlisted → `beforeSwap` passes → swap executes.
- The attacker swaps in a pool they are not authorized to access.

**Existing guards are insufficient:** `setAllowedToSwap` and `setAllowAllSwappers` operate on `allowedSwapper[pool][swapper]`, but the swapper checked at runtime is always the router, not the user. There is no mechanism in the extension or the pool to recover the original `msg.sender` of the router call. [5](#0-4) 

## Impact Explanation
An unprivileged, non-allowlisted trader can execute swaps in a pool gated by `SwapAllowlistExtension` simply by routing through `MetricOmmSimpleRouter`. This breaks the core allowlist functionality — a pool operator's primary tool for restricting swap access (e.g., for compliance, whitelist-only liquidity pools, or exploit-mitigation). Depending on pool configuration, this can result in unauthorized fund flows and violation of LP-agreed access controls. Severity: **Medium** (broken core pool functionality; allowlist bypass reachable by any unprivileged caller with no special preconditions beyond the router being allowlisted, which is the natural operational state).

## Likelihood Explanation
The bypass is trivially reachable by any unprivileged user with no special capability. The only precondition — the router being allowlisted — is the expected production configuration for any pool that intends to support router-based swaps. The attack is repeatable and requires no privileged access, flash loans, or oracle manipulation.

## Recommendation
The extension should not rely on the `sender` argument passed by the pool when the actual originator matters. Options:

1. **Pass `tx.origin` or a verified caller context:** Modify the pool's `_beforeSwap` to also pass `tx.origin` or have the router embed the real user in `extensionData`, and have the extension verify it (with appropriate anti-spoofing).
2. **Check `tx.origin` in the extension:** Replace `sender` with `tx.origin` in the allowlist check. This is simple but prevents contract-based swappers.
3. **Router-level allowlist:** Move the allowlist check into the router itself before calling the pool, so the router enforces per-user access before forwarding.
4. **Disallow router allowlisting:** Document that allowlisting the router address defeats per-user gating, and enforce this in the extension (e.g., revert if `sender` is a known router).
