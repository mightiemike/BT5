### Title
Minimum Burn Fee Exceeds Small Burn Value, Causing 100% NLP Value Destruction — (`core/contracts/Clearinghouse.sol`)

### Summary

In `Clearinghouse.burnNlp`, the minimum fee is hardcoded to `ONE = 1e18` ($1). For any burn where `nlpAmount * oraclePriceX18 / ONE < ONE` (i.e., the NLP being burned is worth less than $1), the fee equals or exceeds the entire burn value. The result is `quoteAmount = MathHelper.max(0, quoteAmount - burnFee) = 0`. The user's NLP is burned and credited to `N_ACCOUNT`, but they receive zero quote — a 100% loss of the burned NLP's value.

### Finding Description

In `Clearinghouse.burnNlp`:

```solidity
int128 quoteAmount = nlpAmount.mul(oraclePriceX18);          // X18 fixed-point multiply
int128 burnFee = MathHelper.max(ONE, quoteAmount / 1000);    // minimum fee = $1
quoteAmount = MathHelper.max(0, quoteAmount - burnFee);      // clamp to 0
```

`MathHelper.max(ONE, quoteAmount / 1000)` sets a minimum fee of `ONE = 1e18` (representing $1 in X18 format). The `quoteAmount / 1000` branch only exceeds `ONE` when `quoteAmount > 1000 * ONE` (i.e., burn value > $1000). For any burn worth between $0 and $1000, the fee is exactly $1.

When `quoteAmount <= ONE` (burn value ≤ $1):
- `burnFee = ONE`
- `quoteAmount - burnFee <= 0`
- `MathHelper.max(0, ...) = 0`

Then:
```solidity
spotEngine.updateBalance(NLP_PRODUCT_ID, txn.sender, -nlpAmount);  // user loses NLP
spotEngine.updateBalance(NLP_PRODUCT_ID, N_ACCOUNT, nlpAmount);    // N_ACCOUNT gains NLP

if (quoteAmount > 0) {   // FALSE — skipped entirely
    spotEngine.updateBalance(QUOTE_PRODUCT_ID, txn.sender, quoteAmount);
    _applyNlpRebalance(...);
}
```

The user's NLP is burned and transferred to `N_ACCOUNT`, but no quote is returned. The NLP value is destroyed from the user's perspective.

There is no minimum `nlpAmount` check, no revert when `quoteAmount <= burnFee`, and no guard preventing a user from burning NLP worth less than the minimum fee.

### Impact Explanation

A user burning NLP with a value ≤ $1 loses 100% of that NLP's value. The NLP balance is debited from the user and credited to `N_ACCOUNT` with no corresponding quote payment. This is a concrete, measurable asset delta: user's NLP balance decreases by `nlpAmount`, user's quote balance does not increase, and `N_ACCOUNT`'s NLP balance increases by `nlpAmount` for free.

Realistic scenarios:
1. A user holds a residual NLP balance worth < $1 (e.g., after a partial burn or after NLP price decline) and attempts to fully exit.
2. A user mints a small NLP position (no minimum mint amount is enforced in `mintNlp`) and later burns it.
3. NLP price drops significantly after minting, making a previously reasonable position worth < $1.

### Likelihood Explanation

The `BurnNlp` transaction is user-initiated and user-signed. The user must sign `(sender, nlpAmount, nonce)` and submit it. However, the `oraclePriceX18` is **not** part of the signed digest — it is provided by the sequencer. A user who signs a burn expecting a certain price could receive 0 quote if the sequencer applies a lower oracle price. Additionally, users with residual balances or positions that have declined in value will naturally encounter this condition. No special privileges or external compromise are required — only a valid user signature and a small NLP balance.

### Recommendation

Add a revert guard when the computed `quoteAmount` after fee deduction is zero (or negative before clamping):

```solidity
int128 quoteAmount = nlpAmount.mul(oraclePriceX18);
int128 burnFee = MathHelper.max(ONE, quoteAmount / 1000);
require(quoteAmount > burnFee, "ERR_BURN_AMOUNT_TOO_SMALL");
quoteAmount = quoteAmount - burnFee;
```

Alternatively, enforce a minimum `nlpAmount` such that `nlpAmount.mul(oraclePriceX18) > ONE` always holds.

### Proof of Concept

```
Setup:
  - NLP oracle price = 1e18 ($1 per NLP in X18)
  - User holds 5e17 NLP (0.5 NLP, worth $0.50 in X18 terms)
  - User's NLP is fully unlocked (past NLP_LOCK_PERIOD)

Execution:
  1. User signs BurnNlp { sender, nlpAmount = 5e17, nonce }
  2. Sequencer submits transaction with oraclePriceX18 = 1e18

In burnNlp:
  quoteAmount = 5e17 * 1e18 / 1e18 = 5e17
  burnFee     = max(1e18, 5e17 / 1000) = max(1e18, 5e14) = 1e18
  quoteAmount = max(0, 5e17 - 1e18) = max(0, -5e17) = 0

Result:
  - spotEngine.updateBalance(NLP_PRODUCT_ID, user, -5e17)   → user loses 0.5 NLP
  - spotEngine.updateBalance(NLP_PRODUCT_ID, N_ACCOUNT, 5e17) → N_ACCOUNT gains 0.5 NLP
  - quoteAmount == 0 → no quote credited to user
  - Assert: user received 0 quote for NLP worth $0.50 → 100% value destruction
```
