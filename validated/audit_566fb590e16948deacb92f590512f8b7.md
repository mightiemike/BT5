### Title
`MsgUpdateTier` Governance Execution Permanently Blocked When Bonus Pool Is Insufficient - (`x/tieredrewards/keeper/msg_server_auth.go`)

### Summary

When governance submits `MsgUpdateTier` to change a tier's `BonusApy`, the handler mandatorily settles all accrued bonus rewards for every position in the tier at the old rate before applying the new rate. This settlement calls `sufficientBonusPoolBalance` for each position. If the pool cannot cover any single position's accrued bonus, the entire governance proposal execution fails with `ErrInsufficientBonusPool`. This creates a concrete deadlock: governance cannot reduce the APY (to slow pool drain) precisely when the pool is already depleted.

### Finding Description

In `msg_server_auth.go`, `UpdateTier` detects a `BonusApy` change and calls `claimRewardsAndUpdateTierPositions`:

```go
// x/tieredrewards/keeper/msg_server_auth.go:67-71
if !oldTier.BonusApy.Equal(msg.Tier.BonusApy) {
    if err := ms.claimRewardsAndUpdateTierPositions(ctx, msg.Tier.Id); err != nil {
        return nil, err
    }
}
```

`claimRewardsAndUpdateTierPositions` iterates every position in the tier and calls `processEventsAndClaimBonus` for each:

```go
// x/tieredrewards/keeper/claim_rewards.go:59-77
for _, id := range ids {
    ...
    if _, err := k.processEventsAndClaimBonus(ctx, &pos); err != nil {
        return err
    }
    ...
}
```

Inside `processEventsAndClaimBonus`, `sufficientBonusPoolBalance` is called before any payout:

```go
// x/tieredrewards/keeper/claim_rewards.go:230-232
if err := k.sufficientBonusPoolBalance(ctx, bonusCoins); err != nil {
    return nil, err
}
```

`sufficientBonusPoolBalance` returns `ErrInsufficientBonusPool` if the pool cannot cover the accrued bonus:

```go
// x/tieredrewards/keeper/bonus_rewards.go:48-61
func (k Keeper) sufficientBonusPoolBalance(ctx context.Context, bonus sdk.Coins) error {
    ...
    if !poolBalance.IsAllGTE(bonus) {
        return errorsmod.Wrapf(types.ErrInsufficientBonusPool, ...)
    }
    return nil
}
```

This error propagates all the way back through `claimRewardsAndUpdateTierPositions` → `UpdateTier`, causing the governance proposal execution to fail and the tier APY to remain unchanged.

**Deadlock scenario:**
1. The bonus pool is depleted by BeginBlocker top-ups or user claims.
2. Positions in the tier have accrued bonus rewards (time has passed since last claim).
3. Governance submits `MsgUpdateTier` to **reduce** `BonusApy` (the natural response to a depleted pool).
4. The proposal passes the voting period but fails execution with `ErrInsufficientBonusPool`.
5. The tier APY cannot be reduced; the pool continues to drain at the old higher rate.
6. The only resolution is a separate governance action to fund the pool — which itself requires the pool to be funded externally before the APY can be corrected.

This is structurally identical to the external report: an internal balance check (`sufficientBonusPoolBalance`) is applied uniformly to both user-driven reward claims and governance-triggered reward settlements, causing the governance operation to fail when the pool is depleted by other activity.

Notably, the `BeforeRedelegationSlashed` hook already handles this correctly — it silently forfeits bonus if the pool is insufficient to avoid a chain halt:

```go
// x/tieredrewards/keeper/slash.go:54-63
if _, err := k.processEventsAndClaimBonus(ctx, &pos); err != nil {
    if errors.Is(err, types.ErrInsufficientBonusPool) {
        k.logger(ctx).Error("insufficient bonus pool during redelegation slash", ...)
    } else {
        return err
    }
}
```

The `UpdateTier` path has no equivalent guard.

### Impact Explanation

A governance proposal to change `BonusApy` on any tier fails execution whenever the bonus pool cannot cover all accrued rewards for all positions in that tier. The tier APY is not updated. The pool continues to drain at the old rate. Governance loses the ability to manage the tier's reward rate precisely when management is most needed (pool depleted). The corrupted invariant is: the `Tier.BonusApy` value in state cannot be updated to its governance-approved value, and the `LastBonusAccrual` checkpoints for all positions in the tier are not advanced, meaning the same accrued-but-unpaid window will be re-computed on the next attempt.

**Impact: Medium. Likelihood: Low.**

### Likelihood Explanation

The pool is drained continuously by the BeginBlocker top-up mechanism and by user claims. A tier with many long-running positions accumulates large aggregate accrued bonus. If governance attempts an APY change during a period of pool depletion — a realistic operational scenario — the proposal execution fails. The condition is not attacker-controlled but arises from normal protocol operation.

### Recommendation

In `claimRewardsAndUpdateTierPositions` (called only from `UpdateTier`), apply the same silent-forfeit pattern already used in `slashRedelegationPosition`: if `ErrInsufficientBonusPool` is returned, advance `LastBonusAccrual` and `LastEventSeq` checkpoints without paying out, so the position's accrual window is reset to the new APY boundary. This prevents double-counting while allowing the governance operation to succeed regardless of pool balance.

### Proof of Concept

The existing unit test `TestUpdateTier_BonusApyChange_InsufficientPool` in `x/tieredrewards/keeper/msg_server_auth_test.go` directly demonstrates the failure:

**On-chain trigger path:**
1. Any unprivileged user creates positions in a tier via `MsgLockTier` or `MsgCommitDelegationToTier`.
2. Time passes; bonus accrues. The pool drains via BeginBlocker or user claims.
3. Governance submits `MsgUpdateTier` (e.g., to reduce `BonusApy` from 10% to 1%).
4. Proposal passes voting. Execution calls `UpdateTier` → `claimRewardsAndUpdateTierPositions` → `processEventsAndClaimBonus` → `sufficientBonusPoolBalance` → `ErrInsufficientBonusPool`.

func (ms msgServer) UpdateTier(ctx context.Context, msg *types.MsgUpdateTier) (*types.MsgUpdateTierResponse, error) {
	if err := ms.requireAuthority(msg.Authority); err != nil {
		return nil, err
	}

	oldTier, err := ms.getTier(ctx, msg.Tier.Id)
	if err != nil {
		return nil, err

		return nil, err
	}

		return nil, err
	}

func (k Keeper) claimRewardsAndUpdateTierPositions(ctx context.Context, tierId uint32) error {
	ids, err := k.getPositionsIdsByTier(ctx, tierId)
	if err != nil {
		return err
	}
	if len(ids) == 0 {
		return nil
	}

	for _, id := range ids {
		pos, err := k.getPositionState(ctx, id)
		if err != nil {
			return err
		}
		if !pos.IsDelegated() {
			continue
		}

		if _, err := k.claimBaseRewards(ctx, pos); err != nil {
			return err
		}
		if _, err := k.processEventsAndClaimBonus(ctx, &pos); err != nil {
			return err
		}
		if err := k.setPosition(ctx, pos.Position, nil); err != nil {
			return err
		}
	}

	return nil
}

func (k Keeper) sufficientBonusPoolBalance(ctx context.Context, bonus sdk.Coins) error {
	if bonus.IsZero() {
		return nil
	}

	poolAddr := k.accountKeeper.GetModuleAddress(types.RewardsPoolName)
	poolBalance := k.bankKeeper.GetAllBalances(ctx, poolAddr)
	if !poolBalance.IsAllGTE(bonus) {
		return errorsmod.Wrapf(types.ErrInsufficientBonusPool,
			"bonus: %s, pool balance: %s",
			bonus.String(), poolBalance.String())
	}

	return nil

func (s *KeeperSuite) TestUpdateTier_BonusApyChange_InsufficientPool() {
	lockAmount := sdkmath.NewInt(sdk.DefaultPowerReduction.Int64())
	pos := s.setupNewTierPosition(lockAmount, false)
	valAddr := sdk.MustValAddressFromBech32(pos.Delegation.ValidatorAddress)

	s.setValidatorCommission(valAddr, sdkmath.LegacyZeroDec())

	// Advance time so bonus accrues.
	s.ctx = s.ctx.WithBlockHeight(s.ctx.BlockHeight() + 1)
	s.ctx = s.ctx.WithBlockTime(s.ctx.BlockTime().Add(30 * 24 * time.Hour))

	// Update tier with new BonusApy — should fail due to insufficient pool.
	tier := newTestTier(1)
	initialBonusApy := tier.BonusApy
	updatedBonusApy := sdkmath.LegacyNewDecWithPrec(8, 2)
	tier.BonusApy = updatedBonusApy
	msgServer := keeper.NewMsgServerImpl(s.keeper)
	_, err := msgServer.UpdateTier(s.ctx, &types.MsgUpdateTier{
		Authority: s.keeper.GetAuthority(),
		Tier:      tier,
	})
	s.Require().ErrorIs(err, types.ErrInsufficientBonusPool)

	// Tier should NOT have been updated (tx failed).
	got, err := s.keeper.GetTier(s.ctx, 1)
	s.Require().NoError(err)
	s.Require().True(initialBonusApy.Equal(got.BonusApy), "tier should still have old APY")
