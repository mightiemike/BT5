### Title
Incomplete `ProofArgs` Structure Enables 64-Byte Transaction Merkle Proof Forgery via `verify_transaction_inclusion` — (File: `contract/src/lib.rs`)

---

### Summary

The `verify_transaction_inclusion` function accepts a `ProofArgs` structure that is missing the coinbase proof fields required to distinguish Merkle tree leaf nodes (real transactions) from internal nodes. Any unprivileged NEAR caller can supply an internal Merkle tree node hash as `tx_id` with a valid upward proof path and receive a `true` return value, forging transaction inclusion for a non-existent Bitcoin transaction.

---

### Finding Description

`ProofArgs` is defined as: [1](#0-0) 

It contains only `tx_id`, `tx_block_blockhash`, `tx_index`, `merkle_proof`, and `confirmations`. It is structurally analogous to the incomplete `PythPrice` in the external report: it lacks the fields (`coinbase_tx_id`, `coinbase_merkle_proof`) that are necessary to anchor the proof to a known leaf depth and distinguish leaf nodes from internal nodes.

`verify_transaction_inclusion` uses this incomplete structure directly: [2](#0-1) 

The verification reduces to:

```
compute_root_from_merkle_proof(tx_id, tx_index, merkle_proof) == header.block_header.merkle_root
``` [3](#0-2) 

Because `compute_root_from_merkle_proof` is purely positional — it hashes the supplied hash upward through the proof path without any leaf/internal-node distinction — an attacker can supply any internal node hash as `tx_id` together with a valid upward proof path and the equation holds. The function returns `true`.

The fixed version, `verify_transaction_inclusion_v2`, uses `ProofArgsV2`: [4](#0-3) 

which adds `coinbase_tx_id` and `coinbase_merkle_proof` to anchor the tree depth. But `verify_transaction_inclusion` with the incomplete `ProofArgs` remains a live, callable public method — `#[deprecated]` in Rust is a compiler lint, not a runtime access control.

---

### Impact Explanation

Any bridge, atomic-swap contract, or other recipient that calls `verify_transaction_inclusion` to authorize an action (e.g., releasing wrapped BTC) can be made to authorize that action for a Bitcoin transaction that does not exist. The attacker needs only to identify an internal Merkle tree node in a real, confirmed Bitcoin block and construct the upward proof path to the stored `merkle_root`. The contract's stored header state is not corrupted, but the proof result returned to the caller is forged.

---

### Likelihood Explanation

Medium. The 64-byte transaction Merkle forgery attack is well-documented and the inputs are fully attacker-controlled. The only prerequisite is identifying a suitable internal node in any confirmed block already stored by the light client, which is trivially achievable by inspecting the Bitcoin blockchain. No privileged role or key is required; any NEAR account can call `verify_transaction_inclusion`.

---

### Recommendation

Remove `verify_transaction_inclusion` from the contract entirely, or gate it with an access-control role so it cannot be called by unprivileged accounts. All callers must migrate to `verify_transaction_inclusion_v2`. The `ProofArgs` structure should either be deleted or extended with the coinbase proof fields to match `ProofArgsV2`.

---

### Proof of Concept

1. Select any Bitcoin block already accepted by the light client (its `tx_block_blockhash` is in `mainchain_header_to_height`).
2. Obtain the block's full transaction list and compute its Merkle tree. Pick any internal node `N` at depth `d` (e.g., the parent of two leaf transactions).
3. Compute the upward Merkle proof from `N` to the root (a path of length `depth - d`).
4. Call `verify_transaction_inclusion` with:
   - `tx_id` = hash of internal node `N`
   - `tx_block_blockhash` = the chosen block hash
   - `tx_index` = the positional index of `N` at its level
   - `merkle_proof` = the upward path from `N` to the root
   - `confirmations` = any value ≤ current chain depth
5. `compute_root_from_merkle_proof(N, tx_index, proof)` reconstructs the stored `merkle_root` exactly, so the function returns `true` — falsely asserting that a transaction with hash `N` is confirmed in that block. [5](#0-4)

    pub fn verify_transaction_inclusion(&self, #[serializer(borsh)] args: ProofArgs) -> bool {
        require!(
            args.confirmations <= self.gc_threshold,
            "The required number of confirmations exceeds the number of blocks stored in memory"
        );

        let heaviest_block_header = self
            .headers_pool
            .get(&self.mainchain_tip_blockhash)
            .unwrap_or_else(|| env::panic_str(ERR_KEY_NOT_EXIST));
        let target_block_height = self
            .mainchain_header_to_height
            .get(&args.tx_block_blockhash)
            .unwrap_or_else(|| env::panic_str("block does not belong to the current main chain"));

        // Check requested confirmations. No need to compute proof if insufficient confirmations.
        require!(
            (heaviest_block_header.block_height).saturating_sub(target_block_height) + 1
                >= args.confirmations,
            "Not enough blocks confirmed"
        );

        let header = self
            .headers_pool
            .get(&args.tx_block_blockhash)
            .unwrap_or_else(|| env::panic_str("cannot find requested transaction block"));

        require!(!args.merkle_proof.is_empty(), "Merkle proof is empty");

        // compute merkle tree root and check if it matches block's original merkle tree root
        merkle_tools::compute_root_from_merkle_proof(
            args.tx_id,
            usize::try_from(args.tx_index).unwrap(),
            &args.merkle_proof,
        ) == header.block_header.merkle_root
    }

pub fn compute_root_from_merkle_proof(
    transaction_hash: H256,
    transaction_position: usize,
    merkle_proof: &Vec<H256>,
) -> H256 {
    let mut current_hash = transaction_hash;
    let mut current_position = transaction_position;

    for proof_hash in merkle_proof {
        if current_position % 2 == 0 {
            current_hash = compute_hash(&current_hash, proof_hash);
        } else {
            current_hash = compute_hash(proof_hash, &current_hash);
        }
        current_position /= 2;
    }

    current_hash
}
