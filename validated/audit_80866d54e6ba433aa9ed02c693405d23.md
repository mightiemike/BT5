### Title
`ContractOwner` Cannot Receive Native ETH from `DirectDepositV1.withdrawNative()` — (`core/contracts/ContractOwner.sol`)

---

### Summary

`ContractOwner` lacks a `receive()` function, yet it is the designated owner of every `DirectDepositV1` (DDA) contract it deploys. When `ContractOwner.withdrawFromDirectDepositV1` is called with `token == address(0)`, it invokes `DirectDepositV1.withdrawNative()`, which attempts to push native ETH to `ContractOwner` via a low-level call. Because `ContractOwner` has no `receive()` or `fallback()`, the ETH transfer always fails, the call reverts, and any native ETH stranded inside a DDA is permanently unrecoverable.

---

### Finding Description

`ContractOwner` deploys `DirectDepositV1` instances and becomes their Ownable owner: [1](#0-0) 

`DirectDepositV1.withdrawNative()` pushes the contract's entire native balance to `msg.sender` (i.e., `ContractOwner`) and hard-requires success: [2](#0-1) 

`ContractOwner.withdrawFromDirectDepositV1` calls this function when `token == address(0)`, then checks that its own balance increased: [3](#0-2) 

`ContractOwner` has no `receive()` or `fallback()` function anywhere in its body: [4](#0-3) 

The ETH push from `DirectDepositV1.withdrawNative()` therefore always reverts inside `ContractOwner`, causing `require(success, "Failed to transfer native token to owner")` to fire, which in turn causes `withdrawFromDirectDepositV1` to revert at `require(postBalance > preBalance, "empty")`.

Native ETH can accumulate inside a DDA in a concrete, non-hypothetical way: the DDA constructor attempts to wrap any pre-existing balance but explicitly does **not** revert on failure — it only emits `NativeTokenTransferFailed`: [5](#0-4) 

If the `wrappedNative` contract is temporarily non-functional at deployment time, ETH is silently left in the DDA with no recovery path.

---

### Impact Explanation

Any native ETH that ends up in a `DirectDepositV1` contract — whether from a failed wrap at construction time or any other mechanism — is permanently locked. The only designated recovery function, `withdrawFromDirectDepositV1(subaccount, address(0))`, always reverts because `ContractOwner` cannot accept ETH. The function is entirely non-operational for native token, mirroring the QV strategy's inability to receive native ETH.

---

### Likelihood Explanation

The trigger condition (native ETH present in a DDA with a non-reverting constructor wrap failure) is realistic on any chain where the wrapped-native contract is paused, upgraded, or temporarily unavailable at DDA deployment time. The `DirectDepositV1` constructor is explicitly written to tolerate this failure silently, confirming the protocol designers anticipated it. Once ETH is stranded, the recovery path is permanently broken.

---

### Recommendation

Add a `receive()` function to `ContractOwner` so it can accept native ETH forwarded from `DirectDepositV1.withdrawNative()`:

```solidity
receive() external payable {}
```

This mirrors the pattern already present in `DirectDepositV1` itself: [6](#0-5) 

---

### Proof of Concept

1. Deploy the system normally; `ContractOwner` becomes the owner of a `DirectDepositV1` for some `subaccount`.
2. At DDA construction time, `wrappedNative.call{value: balance}("")` fails (e.g., wrapped-native contract is paused). The constructor emits `NativeTokenTransferFailed` and continues — ETH remains in the DDA.
3. The multisig (owner of `ContractOwner`) calls `withdrawFromDirectDepositV1(subaccount, address(0))`.
4. `ContractOwner` calls `DirectDepositV1.withdrawNative()`.
5. `withdrawNative()` executes `ContractOwner.call{value: balance}("")` — reverts because `ContractOwner` has no `receive()`.
6. `require(success, "Failed to transfer native token to owner")` fires; the entire call reverts.
7. ETH is permanently locked in the DDA with no alternative recovery path. [7](#0-6) [2](#0-1)

    function withdrawFromDirectDepositV1(bytes32 subaccount, address token)
        external
        onlyOwner
    {
        address payable directDepositV1 = directDepositV1Address[subaccount];
        require(directDepositV1 != address(0), "no dda");
        if (token == address(0)) {
            uint256 preBalance = address(this).balance;
            DirectDepositV1(directDepositV1).withdrawNative();
            uint256 postBalance = address(this).balance;
            require(postBalance > preBalance, "empty");
            (bool success, ) = msg.sender.call{value: postBalance - preBalance}(
                ""
            );
            require(success, "xfer");
        } else {
            uint256 preBalance = IERC20Base(token).balanceOf(address(this));
            DirectDepositV1(directDepositV1).withdraw(IIERC20Base(token));
            uint256 postBalance = IERC20Base(token).balanceOf(address(this));
            require(postBalance > preBalance, "empty");
            IERC20Base(token).safeTransfer(
                msg.sender,
                postBalance - preBalance
            );
        }
    }
