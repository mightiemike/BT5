### Title
Linked Signer Not Cleared After Subaccount Finalization Allows Stale Authorized Signer to Control Re-Deposited Funds - (File: `core/contracts/ClearinghouseLiq.sol`)

---

### Summary

When a subaccount is fully liquidated and finalized via `_finalizeSubaccount`, all on-chain balances are zeroed out, but the `linkedSigners` mapping entry for that subaccount is never cleared. Because the subaccount registration (`subaccountIds`) is also never removed, if the same user re-deposits to the same subaccount after finalization, the previously authorized linked signer retains full signing authority over the re-funded account.

---

### Finding Description

`_finalizeSubaccount` in `ClearinghouseLiq.sol` performs a complete accounting cleanup: it settles positive PnL, covers negative balances from insurance, and socializes residual losses. However, it never touches the `linkedSigners` mapping stored in `EndpointStorage`. [1](#0-0) 

The `linkedSigners` mapping is set when a user submits a `LinkSigner` transaction: [2](#0-1) 

It is stored persistently in `EndpointStorage`: [3](#0-2) 

After finalization, the subaccount remains registered because `subaccountIds` is never cleared: [4](#0-3) 

This means `requireSubaccount(sender)` continues to pass for the finalized subaccount. When the user re-deposits to the same subaccount, `validateSignedTx` will accept signatures from the old linked signer because:

1. `requireSubaccount` passes (subaccount still registered)
2. `getLinkedSigner` returns the stale linked signer address
3. `validateSignature` accepts the stale signer's signature [5](#0-4) 

The stale linked signer can sign `WithdrawCollateral`, `MatchOrders`, `TransferQuote`, and even a new `LinkSigner` transaction (since `allowLinkedSigner = true` is passed for `LinkSigner`), allowing it to entrench itself and prevent the legitimate owner from revoking access. [6](#0-5) 

---

### Impact Explanation

A user whose subaccount was finalized (fully liquidated) and who re-deposits to the same subaccount will have their new funds immediately accessible to the old linked signer. The stale signer can withdraw all collateral or trade the account to zero. Because the stale signer can also re-sign a `LinkSigner` transaction to re-set itself as the signer (using the current nonce read from chain), the legitimate owner may be unable to revoke access in time. This constitutes a direct loss of deposited funds.

---

### Likelihood Explanation

Medium. The scenario requires: (1) a subaccount to be fully finalized through liquidation, (2) the user to re-deposit to the same subaccount (a natural action — the subaccount name is tied to the user's address), and (3) the previously linked signer to be compromised or malicious. A user who experienced a hot-wallet compromise, had their subaccount liquidated as a result, and then re-deposited expecting a clean slate is the realistic victim. The nonce is publicly readable on-chain, so the stale signer can act immediately upon re-deposit without any additional information.

---

### Recommendation

In `_finalizeSubaccount`, after all balance cleanup is complete, clear the linked signer for the liquidated subaccount:

```solidity
// In ClearinghouseLiq.sol, _finalizeSubaccount, before returning true:
IEndpoint(getEndpoint()).clearLinkedSigner(txn.liquidatee);
```

Alternatively, add a direct storage clear in the finalization path:

```solidity
linkedSigners[txn.liquidatee] = address(0);
```

This mirrors the fix recommended in the Paladin analog: explicitly zeroing the stale state variable in the "special exit" operation so it cannot be exploited if the subaccount is later re-activated.

---

### Proof of Concept

1. User A controls subaccount `X` and sets linked signer to hot wallet `H` via `LinkSigner`.
2. Hot wallet `H` is compromised by attacker.
3. Subaccount `X` is fully liquidated and finalized via `liquidateSubaccountImpl` → `_finalizeSubaccount`. All balances become zero.
4. `linkedSigners[X]` still equals `H`. `subaccountIds[X]` is still non-zero.
5. User A re-deposits USDC to subaccount `X` via `depositCollateral`, believing the account is clean.
6. Attacker, controlling `H`, reads the current nonce for `X` from `nonces[address(uint160(bytes20(X)))]`.
7. Attacker signs a `WithdrawCollateral` transaction for the full deposited amount using `H`.
8. Sequencer submits the transaction; `validateSignedTx` passes because `getLinkedSigner(X)` returns `H`.
9. Funds are withdrawn to the attacker's address. User A loses the re-deposited collateral. [7](#0-6) [8](#0-7)

    function _finalizeSubaccount(
        IEndpoint.LiquidateSubaccount calldata txn,
        ISpotEngine spotEngine,
        IPerpEngine perpEngine
    ) internal returns (bool) {
        if (txn.productId != type(uint32).max) {
            return false;
        }
        // check whether the subaccount can be finalized:
        // - all perps positions have closed
        // - all spread positions have closed
        // - all spot assets have closed
        // - all positive pnls have been settled

        FinalizeVars memory v;

        v.spotIds = spotEngine.getProductIds();
        v.perpIds = perpEngine.getProductIds();

        require(v.spotIds[0] == QUOTE_PRODUCT_ID);

        // all spot assets (except USDC) must be closed out
        for (uint32 i = 1; i < v.spotIds.length; ++i) {
            uint32 spotId = v.spotIds[i];
            if (spotEngine.getRisk(spotId).longWeightInitialX18 == 0) {
                continue;
            }
            ISpotEngine.Balance memory balance = spotEngine.getBalance(
                spotId,
                txn.liquidatee
            );
            require(balance.amount <= 0, ERR_NOT_FINALIZABLE_SUBACCOUNT);
        }

        for (uint32 i = 0; i < v.perpIds.length; ++i) {
            uint32 perpId = v.perpIds[i];
            IPerpEngine.Balance memory balance = perpEngine.getBalance(
                perpId,
                txn.liquidatee
            );
            require(balance.amount == 0, ERR_NOT_FINALIZABLE_SUBACCOUNT);
        }

        // settle all positive pnl
        for (uint32 i = 0; i < v.perpIds.length; ++i) {
            uint32 perpId = v.perpIds[i];
            IPerpEngine.Balance memory balance = perpEngine.getBalance(
                perpId,
                txn.liquidatee
            );
            if (balance.vQuoteBalance > 0) {
                _settlePnlAgainstLiquidator(
                    txn,
                    perpId,
                    balance.vQuoteBalance,
                    spotEngine,
                    perpEngine
                );
            }
        }

        ISpotEngine.Balance memory quoteBalance = spotEngine.getBalance(
            QUOTE_PRODUCT_ID,
            txn.liquidatee
        );

        // settle all negative pnl until quote balance becomes 0
        for (uint32 i = 0; i < v.perpIds.length; ++i) {
            uint32 perpId = v.perpIds[i];
            IPerpEngine.Balance memory balance = perpEngine.getBalance(
                perpId,
                txn.liquidatee
            );
            if (balance.vQuoteBalance < 0 && quoteBalance.amount > 0) {
                int128 canSettle = MathHelper.max(
                    balance.vQuoteBalance,
                    -quoteBalance.amount
                );
                _settlePnlAgainstLiquidator(
                    txn,
                    perpId,
                    canSettle,
                    spotEngine,
                    perpEngine
                );
                quoteBalance.amount += canSettle;
            }
        }

        v.insurance = insurance;
        v.insurance -= lastLiquidationFees;
        v.canLiquidateMore = (quoteBalance.amount + v.insurance) > 0;

        if (v.canLiquidateMore) {
            for (uint32 i = 1; i < v.spotIds.length; ++i) {
                uint32 spotId = v.spotIds[i];
                ISpotEngine.Balance memory balance = spotEngine.getBalance(
                    spotId,
                    txn.liquidatee
                );
                if (spotEngine.getRisk(spotId).longWeightInitialX18 == 0) {
                    continue;
                }
                require(balance.amount == 0, ERR_NOT_FINALIZABLE_SUBACCOUNT);
            }
        }

        v.insurance = perpEngine.socializeSubaccount(
            txn.liquidatee,
            v.insurance
        );

        // we can assure that quoteBalance must be non positive, because if quoteBalance.amount > 0,
        // there must be 1) no negative pnl in perps, and 2) no liabilities in spot after above actions.
        // however, in this case the liquidatee must be healthy and cannot pass the health check at
        // the beginning.
        int128 insuranceCover = MathHelper.min(
            v.insurance,
            -quoteBalance.amount
        );
        if (insuranceCover > 0) {
            v.insurance -= insuranceCover;
            spotEngine.updateBalance(
                QUOTE_PRODUCT_ID,
                txn.liquidatee,
                insuranceCover
            );
        }
        if (v.insurance <= 0) {
            spotEngine.socializeSubaccount(txn.liquidatee);
        }
        v.insurance += lastLiquidationFees;
        insurance = v.insurance;
        return true;
    }

    function liquidateSubaccountImpl(IEndpoint.LiquidateSubaccount calldata txn)
        external
    {
        require(!RiskHelper.isIsolatedSubaccount(txn.sender), ERR_UNAUTHORIZED);
        require(txn.sender != txn.liquidatee, ERR_UNAUTHORIZED);
        require(isUnderMaintenance(txn.liquidatee), ERR_NOT_LIQUIDATABLE);
        require(
            txn.liquidatee != X_ACCOUNT && txn.liquidatee != N_ACCOUNT,
            ERR_NOT_LIQUIDATABLE
        );
        require(
            txn.productId != QUOTE_PRODUCT_ID,
            ERR_INVALID_LIQUIDATION_PARAMS
        );

        ISpotEngine spotEngine = ISpotEngine(
            address(engineByType[IProductEngine.EngineType.SPOT])
        );
        IPerpEngine perpEngine = IPerpEngine(
            address(engineByType[IProductEngine.EngineType.PERP])
        );

        if (_finalizeSubaccount(txn, spotEngine, perpEngine)) {
            if (RiskHelper.isIsolatedSubaccount(txn.liquidatee)) {
                IOffchainExchange(
                    IEndpoint(getEndpoint()).getOffchainExchange()
                ).tryCloseIsolatedSubaccount(txn.liquidatee);
            }
            return;
        }

        bytes32 sender,
        uint64 nonce,
        bytes calldata transaction,
        bytes memory signature,
        bool allowLinkedSigner

        validateSignature(

                computeDigest(

                    transaction[1:]
                )
            ),
            signature,
            allowLinkedSigner
        );

    }
