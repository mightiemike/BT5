### Title
`ContractOwner` Missing `receive()` Function Permanently Breaks Native Token Recovery from `DirectDepositV1` — (File: `core/contracts/ContractOwner.sol`)

---

### Summary

`ContractOwner` has no `receive()` function. When `withdrawFromDirectDepositV1(subaccount, address(0))` is called to recover native tokens from a `DirectDepositV1` contract, it calls `DirectDepositV1.withdrawNative()`, which attempts to push ETH to `msg.sender` (i.e., `ContractOwner`). Because `ContractOwner` cannot accept ETH, the call always reverts, permanently locking any native tokens that accumulate in `DirectDepositV1`.

---

### Finding Description

`ContractOwner` inherits only from `EIP712Upgradeable` and `OwnableUpgradeable`, neither of which defines a `receive()` function. The contract has no `receive()` or `fallback()` of its own across its 648 lines. [1](#0-0) 

The recovery path for native tokens is:

```
ContractOwner.withdrawFromDirectDepositV1(subaccount, address(0))
  → DirectDepositV1.withdrawNative()
      → msg.sender.call{value: balance}("")   // msg.sender == ContractOwner
      → require(success, ...)                 // ALWAYS REVERTS
```

`DirectDepositV1.withdrawNative()` unconditionally pushes ETH to its caller and reverts on failure: [2](#0-1) 

`ContractOwner.withdrawFromDirectDepositV1` then measures the pre/post ETH balance of `ContractOwner` and requires it increased — a check that can never pass because the ETH transfer already reverted: [3](#0-2) 

Native tokens can accumulate in `DirectDepositV1` in two ways:

1. **Constructor failure**: If `wrappedNative.call{value: balance}("")` fails during deployment, the ETH is silently retained (the constructor emits `NativeTokenTransferFailed` and continues). [4](#0-3) 

2. **`receive()` revert**: If `wrappedNative` is temporarily broken, any ETH sent directly to `DirectDepositV1` is rejected by the `receive()` function (which `require`s success), so ETH cannot enter that way — but the constructor path remains. [5](#0-4) 

---

### Impact Explanation

Any native tokens that end up in a `DirectDepositV1` contract (via the constructor's silent-failure path) are permanently unrecoverable. The designated recovery function `withdrawFromDirectDepositV1(subaccount, address(0))` will always revert because `ContractOwner` cannot accept ETH. The funds are irreversibly locked with no alternative withdrawal path.

---

### Likelihood Explanation

The scenario requires `wrappedNative.call{value: balance}("")` to fail during `DirectDepositV1` construction (e.g., a misconfigured or temporarily broken wrapped-native contract at deployment time). This is a low-probability edge case, but the recovery mechanism is **unconditionally broken** regardless of how the ETH arrived — making the severity of the broken invariant high once triggered.

---

### Recommendation

Add a `receive()` function to `ContractOwner` so it can accept ETH forwarded from `DirectDepositV1.withdrawNative()`:

```solidity
receive() external payable {}
```

Alternatively, refactor `withdrawNative()` in `DirectDepositV1` to accept a recipient address parameter, bypassing the need for `ContractOwner` to hold ETH transiently:

```solidity
function withdrawNative(address payable recipient) external onlyOwner {
    uint256 balance = address(this).balance;
    (bool success, ) = recipient.call{value: balance}("");
    require(success, "Failed to transfer native token");
}
```

---

### Proof of Concept

1. Deploy `DirectDepositV1` with a `wrappedNative` address whose `receive()` reverts (e.g., a contract with no `receive()`). The constructor silently retains the ETH and emits `NativeTokenTransferFailed`.
2. Call `ContractOwner.withdrawFromDirectDepositV1(subaccount, address(0))`.
3. Internally, `DirectDepositV1.withdrawNative()` executes `ContractOwner.call{value: balance}("")`.
4. `ContractOwner` has no `receive()` → the call returns `success = false`.
5. `withdrawNative()` reverts: `"Failed to transfer native token to owner"`.
6. The ETH remains permanently locked in `DirectDepositV1`. [6](#0-5) [2](#0-1)

        external
        onlyOwner
    {
        address payable directDepositV1 = directDepositV1Address[subaccount];
        require(directDepositV1 != address(0), "no dda");
        if (token == address(0)) {
            uint256 preBalance = address(this).balance;
            DirectDepositV1(directDepositV1).withdrawNative();
            uint256 postBalance = address(this).balance;
            require(postBalance > preBalance, "empty");
            (bool success, ) = msg.sender.call{value: postBalance - preBalance}(
                ""
            );
            require(success, "xfer");

            uint256 preBalance = IERC20Base(token).balanceOf(address(this));
            DirectDepositV1(directDepositV1).withdraw(IIERC20Base(token));
            uint256 postBalance = IERC20Base(token).balanceOf(address(this));
            require(postBalance > preBalance, "empty");
            IERC20Base(token).safeTransfer(
                msg.sender,
                postBalance - preBalance
            );
        }
    }
