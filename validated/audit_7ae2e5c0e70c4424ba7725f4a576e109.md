### Title
AuxPoW-Flagged Dogecoin Block Accepted Without AuxPoW Validation When `aux_data` Is `None` - (File: `contract/src/dogecoin.rs`)

### Summary
In the Dogecoin build of the BTC light client, `submit_block_header` accepts a block with the AuxPoW version flag set but with `aux_data = None`, silently skipping all AuxPoW validation — including the critical parent-block PoW hash check — and instead running only a standalone hash check that is irrelevant for merged-mined blocks. This is a direct analog to the reference report: a resource (AuxPoW block) is processed without the sanity checks that are only reachable when the associated data (aux_data) is explicitly provided.

### Finding Description

`BlockHeader` for the Dogecoin build is `(Header, Option<AuxData>)`. [1](#0-0) 

In `submit_block_header`, when `skip_pow_verification` is `false`, the code branches on whether `aux_data` is `Some` or `None`: [2](#0-1) 

When `aux_data` is `None`, the `else` branch fires and checks only the Dogecoin block's own `block_hash_pow()` against the target. All of `check_aux` is skipped entirely.

`check_aux` is the function that performs the real AuxPoW proof-of-work verification — specifically, it checks the **parent block's** hash against the target: [3](#0-2) 

For any AuxPoW block (version bit `0x100` set), the Dogecoin block's own hash is **not** required to meet the difficulty target; only the parent block's hash must. The `else` branch therefore applies the wrong check. An attacker can craft a Dogecoin block with:

- `version & 0x100 != 0` (AuxPoW flag set, as required by Dogecoin consensus after block 371337)
- A standalone scrypt hash that happens to meet the current target (feasible at low difficulty or on testnet)
- `aux_data = None`

The contract accepts this block, even though the real Dogecoin network would reject it (AuxPoW-flagged blocks must carry valid AuxPoW data; standalone mining of such blocks is invalid).

The full list of `check_aux` validations that are skipped when `aux_data` is `None`: [4](#0-3) 

Skipped checks include:
1. AuxPoW flag presence in version
2. Chain ID match / parent chain ID exclusion
3. Chain Merkle proof verification (Dogecoin block hash in parent coinbase)
4. Coinbase transaction Merkle proof against parent block's `merkle_root`
5. Merged-mining header position and nonce/index validation
6. **Parent block PoW hash ≤ target** (the actual PoW proof)

### Impact Explanation

The light client's `mainchain_height_to_header` and `headers_pool` are corrupted with a block that has no valid Dogecoin PoW. Any downstream consumer calling `verify_transaction_inclusion` or `verify_transaction_inclusion_v2` against a height occupied by this fraudulent block will receive a result based on an unverified block header, enabling false SPV proof acceptance. [5](#0-4) 

### Likelihood Explanation

The `submit_blocks` entry point is gated by `#[trusted_relayer]`, but the staking mechanism (`omni_utils::macros::trusted_relayer`) is permissionless — any NEAR account can stake tokens to register as a trusted relayer. [6](#0-5) [7](#0-6) 

On testnet or during low-difficulty periods, mining a standalone scrypt hash that meets the target is computationally feasible. The attacker only needs to register as a relayer (open to any NEAR account) and submit one crafted header.

### Recommendation

Add an explicit guard at the top of `submit_block_header` (or inside the PoW verification block) that rejects any block whose version has the AuxPoW flag set (`version & BLOCK_VERSION_AUXPOW != 0`) but whose `aux_data` is `None`. The check should mirror the one already present inside `check_aux`:

```rust
if !skip_pow_verification {
    const BLOCK_VERSION_AUXPOW: i32 = 0x100;
    if block_header.version & BLOCK_VERSION_AUXPOW != 0 {
        require!(aux_data.is_some(), "AuxPoW block must include aux_data");
    }
    ...
}
```

This ensures that AuxPoW-flagged blocks always go through the full `check_aux` path, including the parent-block PoW hash check, exactly as the reference report recommends prohibiting seizure of assets not explicitly listed by the borrower.

### Proof of Concept

1. Stake NEAR tokens via the trusted-relayer staking API to register as a relayer.
2. Construct a Dogecoin `Header` with `version = 0x00620100` (chain ID 98, AuxPoW flag set) and mine a scrypt hash that satisfies `scrypt_hash <= target_from_bits(bits)` (feasible on testnet).
3. Call `submit_blocks` with `headers = [(crafted_header, None)]`.
4. The contract executes `check_target` (passes — bits are correct) then enters the `else` branch, checks `block_hash_pow() <= target` (passes — crafted to satisfy this), and stores the block in `headers_pool` and `mainchain_height_to_header`.
5. `check_aux` is never called; no parent-block PoW, no coinbase Merkle proof, no chain-ID validation is performed.
6. A subsequent call to `verify_transaction_inclusion_v2` with a transaction in this fraudulent block returns `true`. [8](#0-7)

    pub(crate) fn check_aux(&mut self, block_header: &Header, aux_data: &AuxData) {
        // The Dogecoin block must have the AuxPoW flag set (bit 8) when AuxPoW data is present.
        // https://github.com/dogecoin/dogecoin/blob/master/src/auxpow.h
        const BLOCK_VERSION_AUXPOW: i32 = 0x100;

        require!(
            aux_data.chain_merkle_proof.len() <= 30,
            "Aux POW chain merkle branch too long"
        );
        require!(
            block_header.version & BLOCK_VERSION_AUXPOW != 0,
            "Aux POW block does not have AuxPoW flag set in version"
        );

        let chain_id = self.get_config().aux_chain_id;

        require!(
            chain_id == block_header.get_chain_id(),
            format!(
                "block does not have our chain ID (got {}, expected {chain_id})",
                block_header.get_chain_id()
            )
        );

        require!(
            chain_id != aux_data.parent_block.get_chain_id(),
            "Aux POW parent has our chain ID"
        );

        let chain_root = merkle_tools::compute_root_from_merkle_proof(
            block_header.block_hash(),
            aux_data.chain_id,
            &aux_data.chain_merkle_proof,
        );

        let coinbase_tx = aux_data.get_coinbase_tx();
        let coinbase_tx_hash = coinbase_tx.compute_txid();

        require!(
            merkle_tools::compute_root_from_merkle_proof(
                H256::from(coinbase_tx_hash.to_raw_hash().to_byte_array()),
                0,
                &aux_data.merkle_proof,
            ) == aux_data.parent_block.merkle_root
        );

        let script_sig = coinbase_tx
            .input
            .first()
            .unwrap()
            .script_sig
            .to_hex_string();
        let pos_merged_mining_header = script_sig.find(MERGED_MINING_HEADER);
        let mut pos_chain_root = script_sig
            .find(&chain_root.to_string())
            .expect("Aux POW missing chain merkle root in parent coinbase");

        match pos_merged_mining_header {
            Some(pos_merged_mining_header) => {
                if script_sig[pos_merged_mining_header + MERGED_MINING_HEADER.len()..]
                    .contains(MERGED_MINING_HEADER)
                {
                    env::panic_str("Multiple merged mining headers in coinbase");
                }

                require!(
                    pos_merged_mining_header + MERGED_MINING_HEADER.len() == pos_chain_root,
                    "Merged mining header is not just before chain merkle root"
                );
            }
            None => {
                require!(pos_chain_root <= 40, "Aux POW chain merkle root must start in the first 20 bytes of the parent coinbase");
            }
        }

        pos_chain_root += chain_root.to_string().len();
        require!(
            script_sig.len() - pos_chain_root >= 16,
            "Aux POW missing chain merkle tree size and nonce in parent coinbase"
        );

        let bytes = hex::decode(&script_sig[pos_chain_root..pos_chain_root + 8]).unwrap();
        let n_size = u32::from_le_bytes([bytes[0], bytes[1], bytes[2], bytes[3]]);
        require!(
            n_size == (1u32 << aux_data.chain_merkle_proof.len()),
            "Aux POW merkle branch size does not match parent coinbase"
        );

        let bytes = hex::decode(&script_sig[pos_chain_root + 8..pos_chain_root + 16]).unwrap();
        let n_nonce = u32::from_le_bytes([bytes[0], bytes[1], bytes[2], bytes[3]]);

        let chain_id = block_header.get_chain_id();

        let expected_index =
            Self::get_expected_index(n_nonce, chain_id, aux_data.chain_merkle_proof.len());

        require!(
            u32::try_from(aux_data.chain_id).ok() == Some(expected_index),
            "Aux POW wrong index"
        );
        let pow_hash = aux_data.parent_block.block_hash_pow();
        require!(
            self.skip_pow_verification
                || U256::from_le_bytes(&pow_hash.0) <= target_from_bits(block_header.bits),
            format!("block should have correct pow")
        );
    }

    pub(crate) fn submit_block_header(
        &mut self,
        header: (Header, Option<AuxData>),
        skip_pow_verification: bool,
    ) {
        let (block_header, aux_data) = header;

        let prev_block_header = self.get_prev_header(&block_header);
        let current_block_hash = block_header.block_hash();

        if !skip_pow_verification {
            self.check_target(&block_header, &prev_block_header);

            if let Some(ref aux_data) = aux_data {
                self.check_aux(&block_header, aux_data);
            } else {
                let pow_hash = block_header.block_hash_pow();
                // Check if the block hash is less than or equal to the target
                require!(
                    U256::from_le_bytes(&pow_hash.0) <= target_from_bits(block_header.bits),
                    format!("block should have correct pow")
                );
            }
        }

        let (current_block_computed_chain_work, overflow) = prev_block_header
            .chain_work
            .overflowing_add(work_from_bits(block_header.bits));
        require!(!overflow, "Addition of U256 values overflowed");

        let current_header = ExtendedHeader {
            block_header: block_header.clone().into_light(),
            block_hash: current_block_hash,
            chain_work: current_block_computed_chain_work,
            block_height: 1 + prev_block_header.block_height,
        };

        self.submit_block_header_inner(current_header, &prev_block_header);
    }
