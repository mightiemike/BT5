### Title
Malicious Counterparty Chain Can Drain Escrowed NFTs via Crafted `data.ClassId` Prefix — (`x/nft-transfer/keeper/packet.go`, `x/nft-transfer/types/trace.go`)

---

### Summary

`processReceivedPacket` determines whether to mint a new voucher or unescrow an existing NFT solely by checking whether `data.ClassId` starts with the packet's source port/channel prefix via `IsAwayFromOrigin`. There is no validation that the `data.ClassId` in an incoming packet actually corresponds to a class trace that was legitimately minted on the counterparty chain. A malicious counterparty chain can craft a packet whose `data.ClassId` is prefixed with its own source port/channel (e.g., `"nft/channel-X/nftClass"`), causing `IsAwayFromOrigin` to return `false` and triggering `TransferOwner` from the local escrow address to the attacker — without any corresponding burn on the counterparty.

---

### Finding Description

**`IsAwayFromOrigin` is a pure string-prefix check with no on-chain state validation:** [1](#0-0) 

It returns `false` (i.e., "returning to origin") whenever `data.ClassId` starts with `"{sourcePort}/{sourceChannel}/"`. This value is fully attacker-controlled application-layer data.

**`processReceivedPacket` branches on this result without any additional guard:** [2](#0-1) 

When `isAwayFromOrigin == false`, it calls:
1. `RemoveClassPrefix(packet.GetSourcePort(), packet.GetSourceChannel(), data.ClassId)` — strips the prefix to get the base class ID
2. `ParseClassTrace(unprefixedClassID).IBCClassID()` — resolves to the native class ID (e.g., `"nftClass"`) when there is no remaining path
3. `TransferOwner(ctx, voucherClassID, tokenID, escrowAddress, receiver)` — transfers the NFT out of the escrow address to the attacker

**`OnRecvPacket` only calls `ValidateBasic`, which does not check the ClassId prefix:** [3](#0-2) 

`ValidateBasic` only verifies that `ClassId` is non-empty, `TokenIds` is non-empty, lengths match, and addresses are valid: [4](#0-3) 

There is no check that `data.ClassId` does not start with the source port/channel prefix (which would be semantically invalid for a first-hop transfer from the counterparty).

---

### Impact Explanation

**Concrete state delta:** NFT `nftClass/token1` owned by `GetEscrowAddress("nft", "channel-0")` is transferred to the attacker's address. The original owner's NFT is permanently stolen. The escrow invariant — "NFTs in escrow are only released by a corresponding burn on the counterparty" — is violated. [5](#0-4) 

---

### Likelihood Explanation

**Attacker-controlled entrypoint:** IBC channels are established permissionlessly. A malicious chain can open a channel with Cronos POS Chain, wait for users to escrow NFTs via `SendTransfer` on that channel, then craft a packet with `data.ClassId = "{sourcePort}/{sourceChannel}/{nativeClassID}"`. The IBC core layer only verifies packet commitment proofs — it does not validate application-layer semantics of `data.ClassId`.

**Exact attack steps:**

1. Attacker controls chain B, which has a legitimate IBC channel to Cronos POS Chain (`channel-0` on Cronos, `channel-X` on chain B).
2. A legitimate user on Cronos POS Chain calls `SendTransfer` for `nftClass/token1` via `channel-0`. `createOutgoingPacket` calls `IsAwayFromOrigin("nft", "channel-0", "nftClass") == true`, so the NFT is escrowed at `GetEscrowAddress("nft", "channel-0")`. [6](#0-5) 

3. Attacker on chain B crafts and commits a packet:
   - `SourcePort = "nft"`, `SourceChannel = "channel-X"`
   - `DestPort = "nft"`, `DestChannel = "channel-0"`
   - `data.ClassId = "nft/channel-X/nftClass"` ← starts with `"nft/channel-X/"`
   - `data.TokenIds = ["token1"]`
   - `data.Receiver = attacker`

4. On Cronos POS Chain, `processReceivedPacket` evaluates:
   - `IsAwayFromOrigin("nft", "channel-X", "nft/channel-X/nftClass")` → `prefixClassID = "nft/channel-X/"`, `strings.HasPrefix("nft/channel-X/nftClass", "nft/channel-X/") == true` → returns `false`
   - `RemoveClassPrefix("nft", "channel-X", "nft/channel-X/nftClass")` → `"nftClass"`
   - `ParseClassTrace("nftClass").IBCClassID()` → `"nftClass"` (empty path, returns base class ID) [7](#0-6) 

   - `escrowAddress = GetEscrowAddress("nft", "channel-0")`
   - `TransferOwner(ctx, "nftClass", "token1", escrowAddress, attacker)` — **succeeds** because `nftClass/token1` is owned by the escrow address from step 2.

---

### Recommendation

Validate that an incoming packet claiming to be a "return" transfer (`isAwayFromOrigin == false`) corresponds to a class trace that was actually registered on this chain (i.e., `HasClassTrace` for the full path including the dest port/channel prefix). Specifically, before calling `TransferOwner` in the `else` branch of `processReceivedPacket`, verify that the full class path `"{destPort}/{destChannel}/{unprefixedClassID}"` exists in the class trace store, proving that this chain previously minted a voucher for this class via a legitimate forward transfer. This mirrors the ICS-20 fungible token transfer defense of checking the denomination trace store before unescrowing.

---

```go
// Keeper integration test (pseudocode)
func TestEscrowDrain(t *testing.T) {
    // Step 1: Legitimate SendTransfer escrows nftClass/token1 at GetEscrowAddress("nft","channel-0")
    err := keeper.SendTransfer(ctx, "nft", "channel-0", "nftClass", []string{"token1"}, sender, "receiver_on_B", ...)
    require.NoError(t, err)
    escrow := types.GetEscrowAddress("nft", "channel-0")
    // Verify NFT is escrowed
    nft, _ := nftKeeper.GetNFT(ctx, "nftClass", "token1")
    require.Equal(t, escrow, nft.GetOwner())

    // Step 2: Attacker on chain B crafts malicious packet
    craftedData := types.NewNonFungibleTokenPacketData(
        "nft/channel-X/nftClass",  // ClassId starts with sourcePort/sourceChannel prefix
        "",
        []string{"token1"},
        []string{""},
        legitimateSender.String(),
        attacker.String(),
    )
    craftedPacket := channeltypes.NewPacket(
        craftedData.GetBytes(), 1,
        "nft", "channel-X",   // source: chain B
        "nft", "channel-0",   // dest: Cronos (channel-0 is counterparty of channel-X)
        ...,
    )

    // Step 3: Deliver packet — no error expected
    err = keeper.OnRecvPacket(ctx, "ics721-1", craftedPacket, craftedData)
    require.NoError(t, err)

    // Step 4: Assert attacker owns the NFT — escrow drained
    nft, _ = nftKeeper.GetNFT(ctx, "nftClass", "token1")
    require.Equal(t, attacker, nft.GetOwner()) // PASSES — theft confirmed
}
```

```go
func (ct ClassTrace) IBCClassID() string {

		return fmt.Sprintf("%s/%s", ClassPrefix, ct.Hash())
```

```go
func (nftpd NonFungibleTokenPacketData) ValidateBasic() error {
	if strings.TrimSpace(nftpd.ClassId) == "" {
		return newsdkerrors.Wrap(ErrInvalidClassID, "classId cannot be blank")
	}

	if len(nftpd.TokenIds) == 0 {
		return newsdkerrors.Wrap(ErrInvalidTokenID, "tokenId cannot be blank")
	}

	if len(nftpd.TokenIds) != len(nftpd.TokenUris) {
		return newsdkerrors.Wrap(ErrInvalidPacket, "tokenIds and tokenUris lengths do not match")
	}

	if strings.TrimSpace(nftpd.Sender) == "" {
		return newsdkerrors.Wrap(sdkerrors.ErrInvalidAddress, "sender address cannot be blank")
	}

	// decode the sender address
	if _, err := sdk.AccAddressFromBech32(nftpd.Sender); err != nil {
		return newsdkerrors.Wrap(sdkerrors.ErrInvalidAddress, "invalid sender address")
	}

	if strings.TrimSpace(nftpd.Receiver) == "" {
		return newsdkerrors.Wrap(sdkerrors.ErrInvalidAddress, "receiver address cannot be blank")
	}

	// decode the receiver address
	if _, err := sdk.AccAddressFromBech32(nftpd.Receiver); err != nil {
		return newsdkerrors.Wrap(sdkerrors.ErrInvalidAddress, "invalid receiver address")
	}
	return nil
}
```

```go
func GetEscrowAddress(portID, channelID string) sdk.AccAddress {
	// a slash is used to create domain separation between port and channel identifiers to
	// prevent address collisions between escrow addresses created for different channels
	contents := fmt.Sprintf("%s/%s", portID, channelID)

	// ADR 028 AddressHash construction
	preImage := []byte(Version)
	preImage = append(preImage, 0)
	preImage = append(preImage, contents...)
	hash := sha256.Sum256(preImage)
	return hash[:20]
```
