### Title
Protobuf `ValidResourceBounds` deserialization silently downgrades `AllResources` to `L1Gas`, producing a divergent transaction hash — (`crates/apollo_protobuf/src/converters/transaction.rs`)

### Summary

The `TryFrom<protobuf::ResourceBounds> for ValidResourceBounds` converter classifies a deserialized transaction as `ValidResourceBounds::L1Gas` whenever both `l2_gas.is_zero()` and `l1_data_gas.is_zero()`. However, `ValidResourceBounds::AllResources` with all-zero `l2_gas` and `l1_data_gas` fields produces a **different** Poseidon hash than `ValidResourceBounds::L1Gas` with the same `l1_gas` bounds, because `get_tip_resource_bounds_hash` includes the `L1_DATA_GAS` element only for `AllResources`. Any transaction originally hashed as `AllResources(l2_gas={0,0}, l1_data_gas={0,0})` is silently re-classified as `L1Gas` after a protobuf round-trip, yielding a divergent hash that the sequencer then serves authoritatively via RPC.

### Finding Description

**Step 1 — `ResourceBounds::is_zero()` definition**

`is_zero()` returns `true` iff **both** `max_amount == 0` AND `max_price_per_unit == 0`:

```rust
// crates/starknet_api/src/transaction/fields.rs:263-265
pub fn is_zero(&self) -> bool {
    self.max_amount == GasAmount(0) && self.max_price_per_unit == GasPrice(0)
}
``` [1](#0-0) 

**Step 2 — Protobuf converter collapses `AllResources` to `L1Gas`**

```rust
// crates/apollo_protobuf/src/converters/transaction.rs:431-435
Ok(if l1_data_gas.is_zero() && l2_gas.is_zero() {
    ValidResourceBounds::L1Gas(l1_gas)
} else {
    ValidResourceBounds::AllResources(AllResourceBounds { l1_gas, l2_gas, l1_data_gas })
})
``` [2](#0-1) 

When a transaction carries `AllResources(l1_gas=X, l2_gas={0,0}, l1_data_gas={0,0})`, the serializer (`From<ValidResourceBounds> for protobuf::ResourceBounds`) emits `l2_gas = {0,0}` and `l1_data_gas = {0,0}` — identical bytes to what `L1Gas` would emit. The deserializer then re-classifies it as `L1Gas`, silently changing the type. [3](#0-2) 

**Step 3 — Hash function produces different outputs for the two variants**

`get_tip_resource_bounds_hash` appends the `L1_DATA_GAS` element **only** for `AllResources`:

```rust
// crates/starknet_api/src/transaction_hash.rs:203-208
resource_felts.extend(match resource_bounds {
    ValidResourceBounds::L1Gas(_) => vec![],                          // 2-element hash
    ValidResourceBounds::AllResources(all_resources) => {
        vec![get_concat_resource(&all_resources.l1_data_gas, L1_DATA_GAS)?]  // 3-element hash
    }
});
``` [4](#0-3) 

Concretely:
- `L1Gas(l1_gas)` → `poseidon(tip, L1_GAS_concat, L2_GAS_concat_zero)` — **2 elements**
- `AllResources(l1_gas, l2_gas={0,0}, l1_data_gas={0,0})` → `poseidon(tip, L1_GAS_concat, L2_GAS_concat_zero, L1_DATA_GAS_concat_zero)` — **3 elements**

These are cryptographically distinct Poseidon outputs.

**Step 4 — How `AllResources(l2_gas={0,0}, l1_data_gas={0,0})` enters the system**

The JSON deserialization path (`TryFrom<DeprecatedResourceBoundsMapping> for ValidResourceBounds`) produces `AllResources` whenever the `L1DataGas` key is **present** in the JSON, regardless of its value:

```rust
// crates/starknet_api/src/transaction/fields.rs:584-589
match resource_bounds_mapping.0.get(&Resource::L1DataGas) {
    Some(data_bounds) => Ok(Self::AllResources(AllResourceBounds {
        l1_gas: *l1_bounds,
        l1_data_gas: *data_bounds,
        l2_gas: *l2_bounds,
    })),
``` [5](#0-4) 

A user submitting a V3 transaction via the JSON RPC API with `L1DataGas: {max_amount: "0x0", max_price_per_unit: "0x0"}` and `L2_GAS: {max_amount: "0x0", max_price_per_unit: "0x0"}` causes the gateway to compute the hash using `AllResources` (3-element Poseidon). That hash is stored in the block. When the block is later synced over P2P, the transaction is deserialized as `L1Gas` (2-element Poseidon), and any subsequent hash recalculation — e.g., in `validate_transaction_hash` or RPC `starknet_getTransactionByHash` — returns the wrong value. [6](#0-5) 

The `TryFrom<protobuf::ResourceBounds> for AllResourceBounds` converter used in the **mempool** P2P path (`rpc_transaction.rs`) always produces `AllResourceBounds` and is not affected; the bug is isolated to the **state-sync** path that deserializes historical `Transaction` objects. [7](#0-6) 

### Impact Explanation

After a protobuf round-trip the sequencer holds a `ValidResourceBounds::L1Gas` object for a transaction whose canonical hash was computed over three resource-bound elements. Every downstream consumer that calls `get_tip_resource_bounds_hash` on the deserialized object — including RPC `starknet_getTransactionByHash`, fee estimation, and re-execution — computes a two-element Poseidon hash that diverges from the on-chain hash. This matches:

- **High**: RPC execution, fee estimation, tracing, simulation, or pending view returns an authoritative-looking wrong value.
- **High**: Transaction conversion or signature/hash logic binds the wrong hash or type.

### Likelihood Explanation

The trigger requires a V3 transaction submitted with `L1DataGas` present in the JSON but with both `max_amount = 0` and `max_price_per_unit = 0`, and `L2_GAS` similarly all-zero. The gateway accepts this when the minimal L2 gas requirement for the transaction is zero (e.g., bootstrap or no-fee-enforcement transactions). The `AllResourceBounds::new_unlimited_gas_no_fee_enforcement()` path used in integration tests sets `l1_gas.max_price_per_unit = GasPrice(0)` and `l1_data_gas` to zero, confirming the pattern is exercised in production-adjacent code. Historical blocks synced from a feeder gateway may also contain such transactions. [8](#0-7) 

### Recommendation

Remove the `is_zero()` heuristic from the protobuf deserializer. Instead, use an explicit discriminant field in the protobuf schema (e.g., a `bounds_type` enum) to distinguish `L1Gas` from `AllResources`, or always deserialize to `AllResources` when `l1_data_gas` is present in the message (matching the JSON path). The serializer should also be audited to ensure `L1Gas` and `AllResources(l2_gas={0,0}, l1_data_gas={0,0})` produce distinguishable wire bytes.

### Proof of Concept

```
1. Craft a V3 InvokeTransaction via starknet_addInvokeTransaction with:
     resource_bounds = {
       L1_GAS:      { max_amount: "0x1000", max_price_per_unit: "0x1" },
       L2_GAS:      { max_amount: "0x0",    max_price_per_unit: "0x0" },
       L1_DATA_GAS: { max_amount: "0x0",    max_price_per_unit: "0x0" }
     }

2. Gateway deserializes via DeprecatedResourceBoundsMapping:
     L1DataGas key IS present → ValidResourceBounds::AllResources
   Hash = poseidon(tip, L1_GAS_concat, L2_GAS_concat_zero, L1_DATA_GAS_concat_zero)
   → stored as tx_hash_A in the block.

3. Block is propagated over P2P state sync.
   Receiving node deserializes via TryFrom<protobuf::ResourceBounds> for ValidResourceBounds:
     l2_gas.is_zero() = true, l1_data_gas.is_zero() = true
     → ValidResourceBounds::L1Gas

4. Receiving node calls get_tip_resource_bounds_hash on the deserialized object:
   Hash = poseidon(tip, L1_GAS_concat, L2_GAS_concat_zero)   ← only 2 elements
   → tx_hash_B ≠ tx_hash_A

5. starknet_getTransactionByHash(tx_hash_A) on the receiving node either:
   (a) returns the stored tx with a recalculated hash of tx_hash_B (wrong value), or
   (b) fails validate_transaction_hash because tx_hash_B ≠ tx_hash_A.
```

impl TryFrom<protobuf::ResourceBounds> for ValidResourceBounds {
    type Error = ProtobufConversionError;
    fn try_from(value: protobuf::ResourceBounds) -> Result<Self, Self::Error> {
        let Some(l1_gas) = value.l1_gas else {
            return Err(missing("ResourceBounds::l1_gas"));
        };
        let Some(l2_gas) = value.l2_gas else {
            return Err(missing("ResourceBounds::l2_gas"));
        };
        // TODO(Shahak): Assert data gas is not none once we remove support for 0.13.2.
        let l1_data_gas = value.l1_data_gas.unwrap_or_default();
        let l1_gas: ResourceBounds = l1_gas.try_into()?;
        let l2_gas: ResourceBounds = l2_gas.try_into()?;
        let l1_data_gas: ResourceBounds = l1_data_gas.try_into()?;
        Ok(if l1_data_gas.is_zero() && l2_gas.is_zero() {
            ValidResourceBounds::L1Gas(l1_gas)
        } else {
            ValidResourceBounds::AllResources(AllResourceBounds { l1_gas, l2_gas, l1_data_gas })
        })
    }

pub fn get_tip_resource_bounds_hash(
    resource_bounds: &ValidResourceBounds,
    tip: &Tip,
) -> Result<Felt, StarknetApiError> {
    let l1_resource_bounds = resource_bounds.get_l1_bounds();
    let l2_resource_bounds = resource_bounds.get_l2_bounds();

    // L1 and L2 gas bounds always exist.
    // Old V3 txs always have L2 gas bounds of zero, but they exist.
    let mut resource_felts = vec![
        get_concat_resource(&l1_resource_bounds, L1_GAS)?,
        get_concat_resource(&l2_resource_bounds, L2_GAS)?,
    ];

    // For new V3 txs, need to also hash the data gas bounds.
    resource_felts.extend(match resource_bounds {
        ValidResourceBounds::L1Gas(_) => vec![],
        ValidResourceBounds::AllResources(all_resources) => {
            vec![get_concat_resource(&all_resources.l1_data_gas, L1_DATA_GAS)?]
        }
    });

    Ok(HashChain::new().chain(&tip.0.into()).chain_iter(resource_felts.iter()).get_poseidon_hash())
}
