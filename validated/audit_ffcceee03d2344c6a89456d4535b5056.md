### Title
Asymmetric Fee Calculation Overcharges Buy-Side Orders — (`core/contracts/OffchainExchange.sol`)

---

### Summary

The `applyFee` function in `OffchainExchange.sol` uses two different mathematical formulas depending on the sign of `meteredQuote`. For sell-side orders (`meteredQuote > 0`) it correctly multiplies by `keepRateX18`, but for buy-side orders (`meteredQuote <= 0`) it divides by `keepRateX18`. This produces a systematically higher fee for buyers than for sellers on equivalent trade sizes, mirroring the JuiceBox M-05 bug class of imprecise fee calculation.

---

### Finding Description

In `OffchainExchange.applyFee`, the fee is computed as:

```solidity
int128 keepRateX18 = ONE - feeInfo.feeRate;
int128 newMeteredQuote = (meteredQuote > 0)
    ? meteredQuote.mul(keepRateX18)
    : meteredQuote.div(keepRateX18);
orderInfo.fee = meteredQuote - newMeteredQuote;
``` [1](#0-0) 

**Sell path (`meteredQuote > 0`):**

```
newMeteredQuote = meteredQuote × (1 − feeRate)
fee             = meteredQuote − meteredQuote × (1 − feeRate)
                = meteredQuote × feeRate          ✓
```

**Buy path (`meteredQuote < 0`):**

```
newMeteredQuote = meteredQuote / (1 − feeRate)
fee             = meteredQuote − meteredQuote / (1 − feeRate)
                = |meteredQuote| × feeRate / (1 − feeRate)   ✗
```

The correct fee for a buyer should be `|meteredQuote| × feeRate`, not `|meteredQuote| × feeRate / (1 − feeRate)`.

**Concrete example (50 bps taker fee, trade size 1000 USDC):**

| | Seller | Buyer (current) | Buyer (correct) |
|---|---|---|---|
| `meteredQuote` | +1000 | −1000 | −1000 |
| `fee` | 5.00 | **5.025** | 5.00 |
| `quoteDelta` after fee | +995 | **−1005.025** | −1005.00 |

The buyer is overcharged by `feeRate² / (1 − feeRate)` per unit of trade value. For 50 bps this is ~0.025 USDC per 1000 USDC traded; for 100 bps it is ~0.101 USDC per 1000 USDC.

The `meteredQuote` sign is determined by `matchQuote = orderInfo.quoteDelta`, which is negative for any buy order (buyer pays quote). This path is reached for every taker buy and every maker buy. [2](#0-1) 

---

### Impact Explanation

Every buy-side taker or maker order pays a fee of `feeRate / (1 − feeRate)` instead of `feeRate`. The protocol collects more fees from buyers than from sellers for identical notional sizes. The overcharge is systematic and cumulative across all buy orders. At higher fee tiers the absolute overcharge per trade grows proportionally. The corrupted state is `orderInfo.fee` and the resulting `orderInfo.quoteDelta`, which directly reduces the buyer's spot or perp balance by more than the protocol's stated fee schedule.

---

### Likelihood Explanation

Every taker buy order and every maker buy order (where `makerAccruesTakerFee` is false) reaches the `meteredQuote.div(keepRateX18)` branch. No special conditions, permissions, or attacker setup are required. Any unprivileged trader submitting a buy order through the `OffchainExchange` triggers this path on every fill. [3](#0-2) 

---

### Recommendation

Replace the conditional `mul`/`div` with a uniform `mul` on the absolute value, preserving the sign:

```solidity
// Correct: fee = |meteredQuote| × feeRate, sign follows meteredQuote
int128 keepRateX18 = ONE - feeInfo.feeRate;
int128 newMeteredQuote = meteredQuote.mul(keepRateX18);
orderInfo.fee = meteredQuote - newMeteredQuote;
```

`meteredQuote.mul(keepRateX18)` for negative `meteredQuote` yields a value that is more negative, so `fee = meteredQuote − newMeteredQuote` is positive (protocol receives positive fee) and equals `|meteredQuote| × feeRate`, matching the sell-side formula exactly.

---

### Proof of Concept

Using the default 2 bps taker fee (`takerRateX18 = 200_000_000_000_000`): [4](#0-3) 

```
feeRate     = 2e14 / 1e18 = 0.0002
keepRateX18 = 1e18 − 2e14 = 9998e14

Sell 1000 USDC (meteredQuote = +1000e18):
  newMeteredQuote = 1000e18 × 9998e14 / 1e18 = 999.8e18
  fee             = 1000e18 − 999.8e18 = 0.2e18  ✓

Buy 1000 USDC (meteredQuote = −1000e18):
  newMeteredQuote = −1000e18 × 1e18 / 9998e14 = −1000.2004e18
  fee             = −1000e18 − (−1000.2004e18) = +0.2004e18  ✗ (should be 0.2e18)

Overcharge per trade = 0.0004 USDC per 1000 USDC (2 bps tier)
Overcharge per trade = 0.025  USDC per 1000 USDC (50 bps tier)
Overcharge per trade = 0.101  USDC per 1000 USDC (100 bps tier)
```

The root cause is exclusively in `OffchainExchange.applyFee` at the `meteredQuote.div(keepRateX18)` branch. [5](#0-4)
