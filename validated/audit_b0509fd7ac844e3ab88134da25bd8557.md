### Title
`TryFrom<protobuf::InvokeV3WithProof> for RpcInvokeTransactionV3` silently rejects valid V3 transactions whose `l2_gas` and `l1_data_gas` bounds are both zero during P2P/consensus deserialization — (`crates/apollo_protobuf/src/converters/rpc_transaction.rs`)

---

### Summary

A V3 invoke transaction submitted via RPC with `AllResources` bounds where both `l2_gas` and `l1_data_gas` are zero is accepted by the gateway and assigned a hash computed under the `AllResources` domain. When that transaction is propagated to other nodes via P2P (mempool gossip or consensus), the protobuf deserialization path silently downgrades the resource-bounds variant from `AllResources` to the deprecated `L1Gas`, causing the subsequent conversion back to `RpcInvokeTransactionV3` to fail. The transaction is rejected by every receiving node and is never sequenced.

---

### Finding Description

**Step 1 — Gateway accepts the transaction as `AllResources`.**

`RpcInvokeTransactionV3` carries `resource_bounds: AllResourceBounds` (not `ValidResourceBounds`). The gateway's stateless validator explicitly allows a transaction whose only non-zero bound is `l1_gas`: [1](#0-0) 

The `TransactionConverter` converts the incoming `RpcInvokeTransactionV3` to `InternalRpcInvokeTransactionV3`, which also stores `resource_bounds: AllResourceBounds`, and computes the hash via `get_invoke_transaction_v3_hash`. That function calls `get_tip_resource_bounds_hash` with `ValidResourceBounds::AllResources(...)`, which includes the `L1_DATA_GAS` field (even when zero) in the Poseidon hash: [2](#0-1) 

So the stored hash commits to the `AllResources` domain (three resource fields).

---

**Step 2 — P2P serialization is lossless.**

When the transaction is serialized to protobuf for P2P propagation, `ValidResourceBounds::AllResources` is serialized with all three fields present: [3](#0-2) 

The wire format therefore carries `l1_gas`, `l2_gas = 0`, `l1_data_gas = 0`.

---

**Step 3 — Receiving node's protobuf converter applies a value-based discriminator.**

`TryFrom<protobuf::InvokeV3WithProof> for RpcInvokeTransactionV3` first converts the inner `InvokeV3` to `InvokeTransactionV3`: [4](#0-3) 

That call reaches `TryFrom<protobuf::ResourceBounds> for ValidResourceBounds`, which applies a **value-based** discriminator: [5](#0-4) 

Because `l1_data_gas.is_zero() && l2_gas.is_zero()`, the result is `ValidResourceBounds::L1Gas(l1_gas)` — the deprecated pre-0.13.3 variant — even though the sender intended `AllResources`.

---

**Step 4 — Conversion back to `RpcInvokeTransactionV3` fails.**

`TryFrom<protobuf::InvokeV3WithProof> for RpcInvokeTransactionV3` then tries to convert the `InvokeTransactionV3` (now carrying `L1Gas`) back to `RpcInvokeTransactionV3`: [6](#0-5) 

That calls `TryFrom<InvokeTransactionV3> for RpcInvokeTransactionV3`, which hard-fails on any non-`AllResources` variant: [7](#0-6) 

The error is mapped to `DEPRECATED_RESOURCE_BOUNDS_ERROR` and the transaction is dropped. The same path is taken for both mempool P2P propagation and consensus: [8](#0-7) 

---

**The broken invariant (analog to the external bug).**

In the external Solidity bug, when both `feeCollector` and `owner` are "renounced" (set to `address(0)`), `claimFees()` still computes a share for `address(0)` and the value is lost. The analog here: when both `l2_gas` and `l1_data_gas` are "renounced" (set to zero), the protobuf converter silently changes the resource-bounds *domain* from `AllResources` to `L1Gas`. The hash was committed under `AllResources` (three fields), but the receiving node reconstructs the transaction under `L1Gas` (two fields) — and then rejects it entirely. A valid transaction is permanently lost from the P2P network.

A separate, correct converter for `AllResourceBounds` already exists but is not used in this path: [9](#0-8) 

---

### Impact Explanation

**High — Mempool/gateway/RPC admission rejects valid transactions before sequencing.**

Any V3 invoke transaction with `l2_gas = 0` and `l1_data_gas = 0` (a common pattern when the sender only cares about L1 gas) is accepted by the gateway, assigned a canonical hash, and then silently dropped by every peer that receives it via P2P or consensus. The transaction can never be included in a block.

---

### Likelihood Explanation

**Medium.** The gateway explicitly accepts transactions with only `l1_gas` non-zero (the `valid_l1_gas` test case). Any wallet or SDK that submits a V3 transaction without specifying L2 or L1-data gas bounds will produce exactly this pattern. The trigger requires no special privilege — any unprivileged user submitting a standard V3 transaction can hit it.

---

### Recommendation

In `TryFrom<protobuf::InvokeV3WithProof> for RpcInvokeTransactionV3` (`crates/apollo_protobuf/src/converters/rpc_transaction.rs`), bypass the intermediate `InvokeTransactionV3` step and deserialize `resource_bounds` directly using `TryFrom<protobuf::ResourceBounds> for AllResourceBounds`, which already exists and preserves all three fields regardless of their values. This eliminates the value-based `L1Gas`/`AllResources` discrimination that causes the mismatch.

---

### Proof of Concept

```
1. User submits via JSON-RPC:
   InvokeV3 {
     resource_bounds: AllResourceBounds {
       l1_gas:      { max_amount: 1000, max_price_per_unit: 1 },
       l2_gas:      { max_amount: 0,    max_price_per_unit: 0 },  // "renounced"
       l1_data_gas: { max_amount: 0,    max_price_per_unit: 0 },  // "renounced"
     },
     ...
   }

2. Gateway accepts; hash H computed via AllResources domain
   (includes L1_DATA_GAS=0 in Poseidon preimage).

3. Transaction propagated as protobuf InvokeV3WithProof to peer nodes.

4. Peer calls TryFrom<protobuf::InvokeV3WithProof> for RpcInvokeTransactionV3:
   a. protobuf::InvokeV3 → InvokeTransactionV3:
      ValidResourceBounds::try_from(resource_bounds)
        → l1_data_gas.is_zero() && l2_gas.is_zero()
        → returns ValidResourceBounds::L1Gas(l1_gas)   ← wrong variant
   b. InvokeTransactionV3 → RpcInvokeTransactionV3:
      match value.resource_bounds {
        ValidResourceBounds::AllResources(bounds) => bounds,  // not taken
        _ => return Err(OutOfRange)                           // taken → DEPRECATED_RESOURCE_BOUNDS_ERROR
      }

5. Transaction rejected on all peers; never sequenced.
   The gateway-computed hash H is orphaned.
```

pub fn get_tip_resource_bounds_hash(
    resource_bounds: &ValidResourceBounds,
    tip: &Tip,
) -> Result<Felt, StarknetApiError> {
    let l1_resource_bounds = resource_bounds.get_l1_bounds();
    let l2_resource_bounds = resource_bounds.get_l2_bounds();

    // L1 and L2 gas bounds always exist.
    // Old V3 txs always have L2 gas bounds of zero, but they exist.
    let mut resource_felts = vec![
        get_concat_resource(&l1_resource_bounds, L1_GAS)?,
        get_concat_resource(&l2_resource_bounds, L2_GAS)?,
    ];

    // For new V3 txs, need to also hash the data gas bounds.
    resource_felts.extend(match resource_bounds {
        ValidResourceBounds::L1Gas(_) => vec![],
        ValidResourceBounds::AllResources(all_resources) => {
            vec![get_concat_resource(&all_resources.l1_data_gas, L1_DATA_GAS)?]
        }
    });

    Ok(HashChain::new().chain(&tip.0.into()).chain_iter(resource_felts.iter()).get_poseidon_hash())
}

impl TryFrom<protobuf::ResourceBounds> for ValidResourceBounds {
    type Error = ProtobufConversionError;
    fn try_from(value: protobuf::ResourceBounds) -> Result<Self, Self::Error> {
        let Some(l1_gas) = value.l1_gas else {
            return Err(missing("ResourceBounds::l1_gas"));
        };
        let Some(l2_gas) = value.l2_gas else {
            return Err(missing("ResourceBounds::l2_gas"));
        };
        // TODO(Shahak): Assert data gas is not none once we remove support for 0.13.2.
        let l1_data_gas = value.l1_data_gas.unwrap_or_default();
        let l1_gas: ResourceBounds = l1_gas.try_into()?;
        let l2_gas: ResourceBounds = l2_gas.try_into()?;
        let l1_data_gas: ResourceBounds = l1_data_gas.try_into()?;
        Ok(if l1_data_gas.is_zero() && l2_gas.is_zero() {
            ValidResourceBounds::L1Gas(l1_gas)
        } else {
            ValidResourceBounds::AllResources(AllResourceBounds { l1_gas, l2_gas, l1_data_gas })
        })
    }

impl TryFrom<protobuf::InvokeV3WithProof> for RpcInvokeTransactionV3 {
    type Error = ProtobufConversionError;
    fn try_from(mut value: protobuf::InvokeV3WithProof) -> Result<Self, Self::Error> {
        // Extract proof first, since `starknet_api::transaction::InvokeTransactionV3` does not
        // carry a `proof` field.
        let proof = Proof::from(std::mem::take(&mut value.proof));

        let snapi_invoke: InvokeTransactionV3 = value
            .invoke
            .ok_or(ProtobufConversionError::MissingField {
                field_description: "InvokeV3WithProof::invoke",
            })?
            .try_into()?;

        // This conversion can fail only if the resource_bounds are not AllResources.
        Ok(Self { proof, ..snapi_invoke.try_into().map_err(|_| DEPRECATED_RESOURCE_BOUNDS_ERROR)? })
    }
}
