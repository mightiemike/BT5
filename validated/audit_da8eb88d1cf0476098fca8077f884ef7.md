### Title
`AllResources`→`L1Gas` Protobuf Round-Trip Silently Changes Transaction Hash, Causing Valid Transactions to Be Rejected by Receiving Nodes - (`crates/apollo_protobuf/src/converters/transaction.rs`)

### Summary

The protobuf deserialization of `ValidResourceBounds` silently downgrades an `AllResources` variant (with zero `l2_gas` and `l1_data_gas`) to `L1Gas`. Because `get_tip_resource_bounds_hash` hashes a different number of resource fields depending on the variant, the transaction hash computed at the gateway diverges from the hash recomputed by any node that receives the transaction over P2P. The receiving node rejects the transaction as having an invalid hash, so a legitimately submitted transaction is permanently dropped from the network.

### Finding Description

**Step 1 – Serialization (sender side)**

`From<ValidResourceBounds> for protobuf::ResourceBounds` always emits all three fields when the variant is `AllResources`:

```rust
ValidResourceBounds::AllResources(AllResourceBounds { l1_gas, l2_gas, l1_data_gas }) =>
    protobuf::ResourceBounds {
        l1_gas: Some(l1_gas.into()),
        l2_gas: Some(l2_gas.into()),
        l1_data_gas: Some(l1_data_gas.into()),   // always Some
    },
``` [1](#0-0) 

**Step 2 – Deserialization (receiver side)**

`TryFrom<protobuf::ResourceBounds> for ValidResourceBounds` applies a zero-check and silently collapses the variant:

```rust
let l1_data_gas = value.l1_data_gas.unwrap_or_default();
...
Ok(if l1_data_gas.is_zero() && l2_gas.is_zero() {
    ValidResourceBounds::L1Gas(l1_gas)          // ← variant changed
} else {
    ValidResourceBounds::AllResources(...)
})
``` [2](#0-1) 

When a user submits an `AllResources` V3 transaction with `l2_gas = {0, 0}` and `l1_data_gas = {0, 0}`, the wire bytes carry all three fields as `Some(zero)`. The receiving node's deserializer sees both as zero and returns `L1Gas` instead of `AllResources`.

**Step 3 – Hash divergence**

`get_tip_resource_bounds_hash` hashes a different number of resource entries depending on the variant:

```rust
resource_felts.extend(match resource_bounds {
    ValidResourceBounds::L1Gas(_) => vec![],                          // 2 entries
    ValidResourceBounds::AllResources(all_resources) => {
        vec![get_concat_resource(&all_resources.l1_data_gas, L1_DATA_GAS)?]  // 3 entries
    }
});
``` [3](#0-2) 

- Gateway computes **H1** = `poseidon(tip, l1_gas_packed, l2_gas_packed_zero, l1_data_gas_packed_zero)` (3 resource entries, `AllResources` path).
- Receiving node computes **H2** = `poseidon(tip, l1_gas_packed, l2_gas_packed_zero)` (2 resource entries, `L1Gas` path).
- **H1 ≠ H2**.

The `InvokeTransactionV3` struct stores `proof_facts` with `#[serde(default, skip_serializing_if = "ProofFacts::is_empty")]`, and `get_invoke_transaction_v3_hash` conditionally appends the proof-facts hash only when non-empty — this is a separate but structurally identical conditional-inclusion pattern that compounds the hash-domain fragility. [4](#0-3) [5](#0-4) 

**Step 4 – Trigger is unprivileged**

`RpcInvokeTransactionV3` always carries `AllResourceBounds` (never `L1Gas`), so any user who submits a V3 invoke with zero `l2_gas` and zero `l1_data_gas` triggers this path. No special privilege is required. [6](#0-5) 

### Impact Explanation

A valid transaction accepted by the gateway (hash H1 stored, signature over H1) is propagated via P2P. Every receiving node recomputes H2 ≠ H1 and rejects the transaction. The transaction is never sequenced. This matches the allowed impact: **"Mempool/gateway/RPC admission accepts invalid transactions or rejects valid transactions before sequencing."**

### Likelihood Explanation

Any user who submits an `AllResources` V3 invoke with zero `l2_gas` and zero `l1_data_gas` — a configuration that passes all gateway validation — triggers the bug. The `enforce_fee()` path only skips fee enforcement; it does not reject the transaction. The condition is reachable without any attacker coordination.

### Recommendation

In `TryFrom<protobuf::ResourceBounds> for ValidResourceBounds`, do not collapse `AllResources` to `L1Gas` based on zero-value fields. The variant must be preserved exactly as serialized. One approach: add an explicit discriminant field to the protobuf message, or require that the `L1Gas` variant is only produced when the sender explicitly serialized it as such (i.e., when `l1_data_gas` is `None`, not `Some(zero)`).

Additionally, `get_tip_resource_bounds_hash` should produce the same hash for `AllResources { l2_gas: 0, l1_data_gas: 0 }` and `L1Gas` only if they are intended to be semantically equivalent — if they are not, the hash function correctly distinguishes them, and the conversion must not silently change the variant.

### Proof of Concept

```
1. User submits RpcInvokeTransactionV3 with:
     resource_bounds = AllResourceBounds {
         l1_gas:      { max_amount: 1000, max_price_per_unit: 1 },
         l2_gas:      { max_amount: 0,    max_price_per_unit: 0 },
         l1_data_gas: { max_amount: 0,    max_price_per_unit: 0 },
     }

2. Gateway converts to InvokeTransactionV3 with ValidResourceBounds::AllResources(...).
   get_tip_resource_bounds_hash hashes [tip, l1_gas_packed, l2_gas_packed_zero,
   l1_data_gas_packed_zero] → H1.
   Signature is over H1. Transaction stored with tx_hash = H1.

3. Transaction serialized to protobuf::ResourceBounds:
     l1_gas = Some(1000, 1), l2_gas = Some(0, 0), l1_data_gas = Some(0, 0).

4. Receiving node deserializes:
     l1_data_gas.is_zero() && l2_gas.is_zero() == true
     → ValidResourceBounds::L1Gas({ max_amount: 1000, max_price_per_unit: 1 })

5. Receiving node calls get_tip_resource_bounds_hash with L1Gas:
   hashes [tip, l1_gas_packed, l2_gas_packed_zero] → H2.
   H2 ≠ H1.

6. Receiving node rejects the transaction (hash mismatch / signature invalid over H2).
   Transaction is never sequenced.
```

```rust
pub fn get_tip_resource_bounds_hash(
    resource_bounds: &ValidResourceBounds,
    tip: &Tip,
) -> Result<Felt, StarknetApiError> {
    let l1_resource_bounds = resource_bounds.get_l1_bounds();
    let l2_resource_bounds = resource_bounds.get_l2_bounds();

    // L1 and L2 gas bounds always exist.
    // Old V3 txs always have L2 gas bounds of zero, but they exist.
    let mut resource_felts = vec![
        get_concat_resource(&l1_resource_bounds, L1_GAS)?,
        get_concat_resource(&l2_resource_bounds, L2_GAS)?,
    ];

    // For new V3 txs, need to also hash the data gas bounds.
    resource_felts.extend(match resource_bounds {
        ValidResourceBounds::L1Gas(_) => vec![],
        ValidResourceBounds::AllResources(all_resources) => {
            vec![get_concat_resource(&all_resources.l1_data_gas, L1_DATA_GAS)?]
        }
    });

    Ok(HashChain::new().chain(&tip.0.into()).chain_iter(resource_felts.iter()).get_poseidon_hash())
}
