### Title
Protobuf `ValidResourceBounds` Round-Trip Silently Downgrades `AllResources` to `L1Gas`, Producing a Different Transaction Hash for V3 Transactions with Zero L2/Data-Gas Bounds — (`File: crates/apollo_protobuf/src/converters/transaction.rs`)

---

### Summary

The protobuf deserializer for `ValidResourceBounds` silently converts an `AllResources` variant (with zero `l1_data_gas` and `l2_gas`) into the `L1Gas` variant. Because `get_tip_resource_bounds_hash` includes the `L1_DATA_GAS` packed field in the Poseidon preimage only for `AllResources`, the transaction hash computed before serialization differs from the hash recomputed after deserialization. A V3 transaction whose resource bounds are `AllResources{l1_gas: X, l2_gas: 0, l1_data_gas: 0}` is accepted and hashed by the gateway with the three-resource preimage, but any node that receives it over P2P and recomputes the hash from the deserialized form will derive the two-resource `L1Gas` hash — a different value — breaking the signature-to-hash binding.

---

### Finding Description

**Step 1 — Hash preimage depends on the `ValidResourceBounds` variant.**

`get_tip_resource_bounds_hash` in `crates/starknet_api/src/transaction_hash.rs` builds the Poseidon preimage differently for the two variants:

```
L1Gas     → [tip, packed(L1_GAS), packed(L2_GAS)]          // 2 resource felts
AllResources → [tip, packed(L1_GAS), packed(L2_GAS), packed(L1_DATA_GAS)]  // 3 resource felts
``` [1](#0-0) 

Even when `l1_data_gas = 0`, the packed encoding of `L1_DATA_GAS` is a non-zero felt (the resource name occupies the upper 56 bits), so the two hashes are numerically distinct.

**Step 2 — The gateway always stores `AllResources` for any `AllResourceBounds` input.**

`AllResourceBounds` (the type used in `RpcInvokeTransactionV3` and `InternalRpcTransaction`) converts to `ValidResourceBounds::AllResources` unconditionally:

```rust
impl From<AllResourceBounds> for ValidResourceBounds {
    fn from(value: AllResourceBounds) -> Self {
        Self::AllResources(value)
    }
}
``` [2](#0-1) 

So a user who submits a V3 invoke with `l2_gas = 0, l1_data_gas = 0` causes the gateway to compute and store the **three-resource** hash (including the zero `L1_DATA_GAS` packed felt). The user signs this hash.

**Step 3 — The protobuf deserializer silently downgrades the variant.**

When the stored `ValidResourceBounds::AllResources{l1_gas: X, l2_gas: 0, l1_data_gas: 0}` is serialized to protobuf and then deserialized, the converter applies:

```rust
Ok(if l1_data_gas.is_zero() && l2_gas.is_zero() {
    ValidResourceBounds::L1Gas(l1_gas)   // ← variant changed
} else {
    ValidResourceBounds::AllResources(...)
})
``` [3](#0-2) 

The receiving node now holds `ValidResourceBounds::L1Gas(X)` for a transaction whose committed hash was computed as `AllResources`.

**Step 4 — Any hash recomputation on the receiving side produces a different value.**

`validate_transaction_hash` recomputes the hash from the deserialized `Transaction` object and checks it against the stored hash: [4](#0-3) 

Because the deserialized object now carries `L1Gas`, the recomputed hash omits the `L1_DATA_GAS` packed felt, yielding a value that does not match the hash the proposer committed and the user signed.

---

### Impact Explanation

**Impact: High — Transaction conversion or signature/hash logic binds the wrong hash.**

Any node that receives a block over P2P containing a V3 transaction with `AllResources{l2_gas=0, l1_data_gas=0}` and then validates or re-derives the transaction hash will compute a hash that differs from the one the proposer committed. Concretely:

- A syncing node calling `validate_transaction_hash` will reject the transaction as having an invalid hash, causing it to diverge from the canonical chain state.
- An RPC node serving `starknet_getTransactionByHash` or `starknet_getTransactionReceipt` after re-deriving the hash will return an authoritative-looking wrong value (wrong hash, wrong receipt linkage).
- Any component that re-derives the hash for signature verification will conclude the signature is invalid for a legitimately signed transaction.

This matches the allowed impact: *"Transaction conversion or signature/hash logic binds the wrong signer, hash, type, or executable payload."*

---

### Likelihood Explanation

**Likelihood: Medium.**

The trigger is a standard V3 transaction where the user sets `l2_gas = 0` and `l1_data_gas = 0` (e.g., a pre-0.13.3-style transaction submitted through the new V3 interface, or any transaction that only consumes L1 gas). This is a valid, common transaction shape. No privileged access is required; any unprivileged user can submit such a transaction. The conversion happens automatically in the P2P sync path for every such transaction.

---

### Recommendation

Remove the variant-downgrade heuristic from the protobuf deserializer. The `L1Gas` vs `AllResources` distinction must be carried explicitly in the wire format (e.g., via a boolean flag or a separate protobuf `oneof`) rather than inferred from whether the zero-valued fields are present. Alternatively, always deserialize to `AllResources` and let the hash computation handle both cases uniformly:

```rust
// Instead of:
Ok(if l1_data_gas.is_zero() && l2_gas.is_zero() {
    ValidResourceBounds::L1Gas(l1_gas)
} else {
    ValidResourceBounds::AllResources(AllResourceBounds { l1_gas, l2_gas, l1_data_gas })
})

// Use:
Ok(ValidResourceBounds::AllResources(AllResourceBounds { l1_gas, l2_gas, l1_data_gas }))
// and let the hash function always include all three resource felts for V3 transactions.
```

The same fix should be applied to the RPC-layer converter in `crates/apollo_rpc/src/v0_8/transaction.rs`: [5](#0-4)

### Proof of Concept

1. Submit a V3 invoke transaction with `resource_bounds = AllResourceBounds { l1_gas: {max_amount: 1000, max_price_per_unit: 1}, l2_gas: {0, 0}, l1_data_gas: {0, 0} }` to the gateway.
2. The gateway calls `convert_rpc_tx_to_internal`, which calls `tx_without_hash.calculate_transaction_hash(chain_id)`. Because `AllResourceBounds` converts to `ValidResourceBounds::AllResources`, `get_tip_resource_bounds_hash` hashes `[tip, packed(L1_GAS), packed(L2_GAS=0), packed(L1_DATA_GAS=0)]` — **four felts** — producing hash `H_all`.
3. The transaction is included in a block and stored as `InvokeTransactionV3 { resource_bounds: ValidResourceBounds::AllResources{...}, ... }` with committed hash `H_all`.
4. A syncing peer requests the block. The transaction is serialized via `From<ValidResourceBounds> for protobuf::ResourceBounds` (l1_data_gas=0, l2_gas=0 are written as zero `ResourceLimits`).
5. The peer deserializes via `TryFrom<protobuf::ResourceBounds> for ValidResourceBounds`. Because `l1_data_gas.is_zero() && l2_gas.is_zero()`, the result is `ValidResourceBounds::L1Gas(l1_gas)`.
6. The peer calls `get_transaction_hash` on the deserialized transaction. `get_tip_resource_bounds_hash` now hashes `[tip, packed(L1_GAS), packed(L2_GAS=0)]` — **three felts** — producing hash `H_l1 ≠ H_all`.
7. `validate_transaction_hash` compares `H_l1` against the stored `H_all` and returns `false`, causing the peer to reject or mishandle the transaction. [6](#0-5) [1](#0-0) [7](#0-6)

pub fn get_tip_resource_bounds_hash(
    resource_bounds: &ValidResourceBounds,
    tip: &Tip,
) -> Result<Felt, StarknetApiError> {
    let l1_resource_bounds = resource_bounds.get_l1_bounds();
    let l2_resource_bounds = resource_bounds.get_l2_bounds();

    // L1 and L2 gas bounds always exist.
    // Old V3 txs always have L2 gas bounds of zero, but they exist.
    let mut resource_felts = vec![
        get_concat_resource(&l1_resource_bounds, L1_GAS)?,
        get_concat_resource(&l2_resource_bounds, L2_GAS)?,
    ];

    // For new V3 txs, need to also hash the data gas bounds.
    resource_felts.extend(match resource_bounds {
        ValidResourceBounds::L1Gas(_) => vec![],
        ValidResourceBounds::AllResources(all_resources) => {
            vec![get_concat_resource(&all_resources.l1_data_gas, L1_DATA_GAS)?]
        }
    });

    Ok(HashChain::new().chain(&tip.0.into()).chain_iter(resource_felts.iter()).get_poseidon_hash())
}

impl TryFrom<protobuf::ResourceBounds> for ValidResourceBounds {
    type Error = ProtobufConversionError;
    fn try_from(value: protobuf::ResourceBounds) -> Result<Self, Self::Error> {
        let Some(l1_gas) = value.l1_gas else {
            return Err(missing("ResourceBounds::l1_gas"));
        };
        let Some(l2_gas) = value.l2_gas else {
            return Err(missing("ResourceBounds::l2_gas"));
        };
        // TODO(Shahak): Assert data gas is not none once we remove support for 0.13.2.
        let l1_data_gas = value.l1_data_gas.unwrap_or_default();
        let l1_gas: ResourceBounds = l1_gas.try_into()?;
        let l2_gas: ResourceBounds = l2_gas.try_into()?;
        let l1_data_gas: ResourceBounds = l1_data_gas.try_into()?;
        Ok(if l1_data_gas.is_zero() && l2_gas.is_zero() {
            ValidResourceBounds::L1Gas(l1_gas)
        } else {
            ValidResourceBounds::AllResources(AllResourceBounds { l1_gas, l2_gas, l1_data_gas })
        })
    }
}
