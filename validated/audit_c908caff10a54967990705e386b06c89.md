### Title
`verify_transaction_inclusion` Accepts Internal Merkle Node as `tx_id`, Enabling Proof Forgery — (File: `contract/src/lib.rs`)

---

### Summary

`verify_transaction_inclusion` assumes the caller-supplied `tx_id` is a leaf-level transaction hash. An unprivileged NEAR caller can instead supply an internal Merkle tree node hash paired with a correspondingly shorter proof. Because the function blindly feeds `tx_id` into `compute_root_from_merkle_proof` without validating that it is a leaf, the computed root can still equal the stored `merkle_root`, causing the function to return `true` for a Bitcoin transaction that does not exist.

---

### Finding Description

`verify_transaction_inclusion` in `contract/src/lib.rs` delegates all Merkle verification to `merkle_tools::compute_root_from_merkle_proof`:

```rust
merkle_tools::compute_root_from_merkle_proof(
    args.tx_id,
    usize::try_from(args.tx_index).unwrap(),
    &args.merkle_proof,
) == header.block_header.merkle_root
``` [1](#0-0) 

`compute_root_from_merkle_proof` iterates over the proof hashes and hashes upward without any check that the starting hash is a leaf:

```rust
pub fn compute_root_from_merkle_proof(
    transaction_hash: H256,
    transaction_position: usize,
    merkle_proof: &Vec<H256>,
) -> H256 {
    let mut current_hash = transaction_hash;
    ...
    for proof_hash in merkle_proof { ... }
    current_hash
}
``` [2](#0-1) 

The only guard in `verify_transaction_inclusion` is that the proof must be non-empty:

```rust
require!(!args.merkle_proof.is_empty(), "Merkle proof is empty");
``` [3](#0-2) 

The function's own docstring acknowledges the broken invariant:

> "This function may return `true` if the provided `tx_id` is a hash of an internal node in the Merkle tree rather than a valid transaction hash." [4](#0-3) 

Despite being marked `#[deprecated]`, the function remains `pub` and is reachable by any NEAR caller. Rust's `#[deprecated]` is a compiler lint, not a runtime restriction. [5](#0-4) 

The analog to the original report is direct:

| Original (PayoutManager) | This repo (BtcLightClient) |
|---|---|
| Assumes `safeTransferFrom` delivers exactly `cc.amount` | Assumes `tx_id` is a leaf-level transaction hash |
| Fee-on-transfer tokens silently deliver less | Attacker supplies an internal node hash |
| Deal tokens minted against the wrong amount | `verify_transaction_inclusion` returns `true` for a non-existent transaction |
| Mismatch between tokens issued and funds received | Mismatch between certified transaction and actual Bitcoin ledger |

---

### Impact Explanation

Any downstream consumer — a bridge, atomic swap, or cross-chain lending protocol — that calls `verify_transaction_inclusion` and acts on a `true` result (e.g., releasing funds) can be deceived into authorizing a payout for a Bitcoin transaction that was never broadcast. The corrupted value is the **proof result**: the function certifies inclusion of a non-existent transaction. [6](#0-5) 

---

### Likelihood Explanation

The attack requires no privileged role. Any NEAR account can call `verify_transaction_inclusion` with a real block hash (already on-chain), an internal Merkle node hash derived from public block data, and a valid shorter proof. The 64-byte transaction Merkle forgery technique is publicly documented and tooled. The only prerequisite is access to a Bitcoin block's transaction list, which is public. [7](#0-6) 

---

### Recommendation

Remove `verify_transaction_inclusion` from the public API entirely, or replace its body with `env::panic_str("use verify_transaction_inclusion_v2")` to enforce migration at runtime. The `#[deprecated]` attribute alone does not prevent on-chain calls. `verify_transaction_inclusion_v2` correctly mitigates the attack by requiring a coinbase proof of the same length, which pins the tree depth and prevents an internal node from being passed off as a leaf. [8](#0-7) 

---

### Proof of Concept

1. Select any Bitcoin block already accepted by the contract (its hash is in `mainchain_header_to_height`).
2. Obtain the block's full transaction list from a public Bitcoin node.
3. Compute the level-1 internal node: `H = double_sha256(tx0 || tx1)`.
4. Construct a Merkle proof of depth `d-1` (one level shorter than the full tree) that hashes `H` up to the stored `merkle_root`. This proof is valid because `H` is a genuine internal node.
5. Call `verify_transaction_inclusion` with:
   - `tx_id = H` (the internal node hash, not a real transaction)
   - `tx_block_blockhash` = the real block hash
   - `tx_index = 0`
   - `merkle_proof` = the `d-1`-element proof
   - `confirmations = 1`
6. `compute_root_from_merkle_proof(H, 0, proof)` produces the correct `merkle_root`; the function returns `true`.
7. A downstream consumer acting on this result releases funds for a Bitcoin transaction that does not exist. [2](#0-1) [9](#0-8)

            .get(&self.mainchain_tip_blockhash)
            .unwrap_or_else(|| env::panic_str(ERR_KEY_NOT_EXIST));

            .mainchain_header_to_height
            .get(&args.tx_block_blockhash)
            .unwrap_or_else(|| env::panic_str("block does not belong to the current main chain"));

        // Check requested confirmations. No need to compute proof if insufficient confirmations.
        require!(
            (heaviest_block_header.block_height).saturating_sub(target_block_height) + 1
                >= args.confirmations,
            "Not enough blocks confirmed"
        );

        let header = self
            .headers_pool
            .get(&args.tx_block_blockhash)
            .unwrap_or_else(|| env::panic_str("cannot find requested transaction block"));

            usize::try_from(args.tx_index).unwrap(),

pub fn compute_root_from_merkle_proof(
    transaction_hash: H256,
    transaction_position: usize,
    merkle_proof: &Vec<H256>,
) -> H256 {
    let mut current_hash = transaction_hash;
    let mut current_position = transaction_position;

    for proof_hash in merkle_proof {
        if current_position % 2 == 0 {
            current_hash = compute_hash(&current_hash, proof_hash);
        } else {
            current_hash = compute_hash(proof_hash, &current_hash);
        }
        current_position /= 2;
    }

    current_hash
}
