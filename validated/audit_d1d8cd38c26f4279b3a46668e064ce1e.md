### Title
Missing AuxPoW-flag consistency check allows AuxPoW-flagged Dogecoin headers to bypass `check_aux` — (`contract/src/dogecoin.rs`)

---

### Summary

`submit_block_header` in the Dogecoin build branches on whether `aux_data` is `Some` or `None` to decide between `check_aux` and bare Scrypt PoW. It never checks whether `header.version & 0x100 != 0` (the AuxPoW flag) when `aux_data` is `None`. An attacker who submits `(header_with_auxpow_flag_set, None)` with a valid Scrypt PoW hash bypasses all of `check_aux` — chain-merkle-root, coinbase-commitment, chain-ID, and merged-mining-header verification — and gets the header canonically accepted.

---

### Finding Description

The dispatch logic in `submit_block_header` is:

```rust
if let Some(ref aux_data) = aux_data {
    self.check_aux(&block_header, aux_data);
} else {
    let pow_hash = block_header.block_hash_pow();
    require!(
        U256::from_le_bytes(&pow_hash.0) <= target_from_bits(block_header.bits),
        format!("block should have correct pow")
    );
}
``` [1](#0-0) 

The branch is purely on `aux_data.is_some()`. There is no guard of the form:

```rust
require!(
    aux_data.is_some() || (block_header.version & 0x100 == 0),
    "AuxPoW flag set but no AuxPoW data provided"
);
```

The `check_aux` function does verify the flag internally:

```rust
require!(
    block_header.version & BLOCK_VERSION_AUXPOW != 0,
    "Aux POW block does not have AuxPoW flag set in version"
);
``` [2](#0-1) 

But this check is only reachable when `aux_data` is `Some`. When `aux_data` is `None`, the entire `check_aux` body — including chain-ID verification, coinbase-commitment, chain-merkle-root, and merged-mining-header checks — is skipped entirely. [3](#0-2) 

The public entry point `submit_blocks` accepts `Vec<(Header, Option<AuxData>)>` from any account holding the `UnrestrictedSubmitBlocks` role, with no further filtering on the version field before calling `submit_block_header`. [4](#0-3) 

---

### Impact Explanation

A successfully submitted AuxPoW-flagged header with `aux_data = None` is stored in `headers_pool` and can become the canonical tip. The stored `ExtendedHeader` carries the attacker-supplied `merkle_root` field verbatim: [5](#0-4) 

`verify_transaction_inclusion` and `verify_transaction_inclusion_v2` both compare a caller-supplied Merkle proof against `header.block_header.merkle_root`: [6](#0-5) 

Because the attacker controls `merkle_root` (no coinbase-commitment check was performed), they can subsequently prove inclusion of an arbitrary, non-existent transaction. The invariant that every AuxPoW-flagged Dogecoin header must pass `check_aux` before canonical acceptance is broken.

---

### Likelihood Explanation

On mainnet Dogecoin difficulty, finding a standalone Scrypt PoW solution for an AuxPoW-flagged header is computationally equivalent to mining a real Dogecoin block — very expensive. However:

- On testnet or any deployment where `proof_of_work_limit_bits = 0x1e0fffff`, the Scrypt target is near-maximum and a valid nonce is found in milliseconds.
- The `check_target` / `check_pow` path enforces only that `block_header.bits` matches the expected difficulty; it does not reject AuxPoW-flagged headers submitted without `aux_data`. [7](#0-6) 

The bug is locally testable on an unmodified NEAR sandbox with the `dogecoin` feature and low-difficulty genesis blocks.

---

### Recommendation

Add an explicit consistency check in `submit_block_header` before the `if let Some` branch:

```rust
const BLOCK_VERSION_AUXPOW: i32 = 0x100;
if !skip_pow_verification {
    require!(
        aux_data.is_some() || (block_header.version & BLOCK_VERSION_AUXPOW == 0),
        "AuxPoW flag set in header version but no AuxPoW data provided"
    );
    // ... existing dispatch ...
}
```

This mirrors the check already present inside `check_aux` and closes the bypass path.

---

### Proof of Concept

```rust

#[test]
fn auxpow_flag_no_aux_data_bypasses_check_aux() {
    // 1. Initialize contract with skip_pow=false, low-difficulty genesis chain.
    // 2. Craft a header: version = 0x0101 (AuxPoW flag | version 1),

    //    At 0x1e0fffff this takes < 1 second on commodity hardware.
    // 4. Call submit_blocks([(crafted_header, None)]).
    // 5. Assert: call succeeds (no panic), get_last_block_header() returns the header.
    // 6. Call verify_transaction_inclusion with a fake tx_id and a merkle proof
    //    constructed against the attacker-chosen merkle_root.
    // 7. Assert: returns true — fake transaction "proven" included.
}
```

The `submit_block_header` function at lines 179–188 will take the `else` branch, perform only the Scrypt hash check, and store the header. `check_aux` — and its chain-ID, coinbase-commitment, and chain-merkle-root verifications — is never called. [8](#0-7)

        // The Dogecoin block must have the AuxPoW flag set (bit 8) when AuxPoW data is present.
        // https://github.com/dogecoin/dogecoin/blob/master/src/auxpow.h
        const BLOCK_VERSION_AUXPOW: i32 = 0x100;

        require!(
            aux_data.chain_merkle_proof.len() <= 30,
            "Aux POW chain merkle branch too long"
        );
        require!(
            block_header.version & BLOCK_VERSION_AUXPOW != 0,
            "Aux POW block does not have AuxPoW flag set in version"
        );

        let chain_id = self.get_config().aux_chain_id;

        require!(
            chain_id == block_header.get_chain_id(),
            format!(
                "block does not have our chain ID (got {}, expected {chain_id})",
                block_header.get_chain_id()
            )
        );

        require!(
            chain_id != aux_data.parent_block.get_chain_id(),
            "Aux POW parent has our chain ID"
        );

        let chain_root = merkle_tools::compute_root_from_merkle_proof(
            block_header.block_hash(),
            aux_data.chain_id,
            &aux_data.chain_merkle_proof,
        );

        let coinbase_tx = aux_data.get_coinbase_tx();
        let coinbase_tx_hash = coinbase_tx.compute_txid();

        require!(
            merkle_tools::compute_root_from_merkle_proof(
                H256::from(coinbase_tx_hash.to_raw_hash().to_byte_array()),
                0,
                &aux_data.merkle_proof,
            ) == aux_data.parent_block.merkle_root
        );

        let script_sig = coinbase_tx
            .input
            .first()
            .unwrap()
            .script_sig
            .to_hex_string();
        let pos_merged_mining_header = script_sig.find(MERGED_MINING_HEADER);
        let mut pos_chain_root = script_sig
            .find(&chain_root.to_string())
            .expect("Aux POW missing chain merkle root in parent coinbase");

        match pos_merged_mining_header {
            Some(pos_merged_mining_header) => {
                if script_sig[pos_merged_mining_header + MERGED_MINING_HEADER.len()..]
                    .contains(MERGED_MINING_HEADER)
                {
                    env::panic_str("Multiple merged mining headers in coinbase");
                }

                require!(
                    pos_merged_mining_header + MERGED_MINING_HEADER.len() == pos_chain_root,
                    "Merged mining header is not just before chain merkle root"
                );
            }
            None => {
                require!(pos_chain_root <= 40, "Aux POW chain merkle root must start in the first 20 bytes of the parent coinbase");
            }
        }

        pos_chain_root += chain_root.to_string().len();
        require!(
            script_sig.len() - pos_chain_root >= 16,
            "Aux POW missing chain merkle tree size and nonce in parent coinbase"
        );

        let bytes = hex::decode(&script_sig[pos_chain_root..pos_chain_root + 8]).unwrap();
        let n_size = u32::from_le_bytes([bytes[0], bytes[1], bytes[2], bytes[3]]);
        require!(

            "Aux POW merkle branch size does not match parent coinbase"
        );

        let n_nonce = u32::from_le_bytes([bytes[0], bytes[1], bytes[2], bytes[3]]);

        let chain_id = block_header.get_chain_id();

        let expected_index =
            Self::get_expected_index(n_nonce, chain_id, aux_data.chain_merkle_proof.len());

    pub(crate) fn submit_block_header(
        &mut self,
        header: (Header, Option<AuxData>),
        skip_pow_verification: bool,
    ) {
        let (block_header, aux_data) = header;

        let prev_block_header = self.get_prev_header(&block_header);
        let current_block_hash = block_header.block_hash();

        if !skip_pow_verification {
            self.check_target(&block_header, &prev_block_header);

            if let Some(ref aux_data) = aux_data {
                self.check_aux(&block_header, aux_data);
            } else {
                let pow_hash = block_header.block_hash_pow();
                // Check if the block hash is less than or equal to the target
                require!(
                    U256::from_le_bytes(&pow_hash.0) <= target_from_bits(block_header.bits),
                    format!("block should have correct pow")
                );
            }
        }

        let (current_block_computed_chain_work, overflow) = prev_block_header
            .chain_work
            .overflowing_add(work_from_bits(block_header.bits));
        require!(!overflow, "Addition of U256 values overflowed");

        let current_header = ExtendedHeader {
            block_header: block_header.clone().into_light(),
            block_hash: current_block_hash,
            chain_work: current_block_computed_chain_work,
            block_height: 1 + prev_block_header.block_height,
        };

        self.submit_block_header_inner(current_header, &prev_block_header);
    }
