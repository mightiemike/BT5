Audit Report

## Title
SwapAllowlistExtension Checks Router Address Instead of Actual Swapper, Allowing Any User to Bypass the Swap Allowlist via MetricOmmSimpleRouter - (File: metric-periphery/contracts/extensions/SwapAllowlistExtension.sol)

## Summary

`SwapAllowlistExtension.beforeSwap()` gates swaps by checking the `sender` argument, which the pool sets to its own `msg.sender` — the direct caller of `pool.swap()`. When a user routes through `MetricOmmSimpleRouter`, the router is `msg.sender` to the pool, so the allowlist checks the router's identity rather than the real end user's. Any non-allowlisted user can bypass the curated-pool restriction by calling the router instead of the pool directly.

## Finding Description

**Step 1 — Pool passes `msg.sender` as `sender` to the hook.**

In `MetricOmmPool.swap()`, `_beforeSwap` is called with `msg.sender` as the first argument: [1](#0-0) 

**Step 2 — SwapAllowlistExtension checks `sender` against the allowlist.**

`SwapAllowlistExtension.beforeSwap()` uses `msg.sender` (the pool) as the mapping key and `sender` (the direct caller of `pool.swap()`) as the identity being gated: [2](#0-1) 

**Step 3 — MetricOmmSimpleRouter calls `pool.swap()` directly, making itself `msg.sender`.**

In `exactInputSingle`, the router calls `pool.swap()` directly. The pool's `msg.sender` is therefore the router, not the end user: [3](#0-2) 

The same pattern applies to `exactInput`, `exactOutputSingle`, and `exactOutput`: [4](#0-3) 

**Result:** The allowlist check resolves to `allowedSwapper[pool][router]` — it checks whether the **router** is allowlisted, not whether the **actual user** is allowlisted. This creates an irreconcilable dilemma for pool admins: if the router is allowlisted (required for any router-mediated swap), every user bypasses the allowlist by routing through the router; if the router is not allowlisted, allowlisted users cannot use the router at all.

## Impact Explanation

A pool admin deploys a curated pool with `SwapAllowlistExtension` to restrict swaps to KYC'd or otherwise vetted addresses. If the router is allowlisted (the only way to enable router-mediated swaps), any non-allowlisted user can call `MetricOmmSimpleRouter.exactInputSingle()` and swap freely against the pool's liquidity. The allowlist guard is completely defeated. LP funds in the curated pool are exposed to unauthorized swappers, which can result in direct loss of LP principal through adverse-selection trades the pool was specifically configured to prevent. This is an admin-boundary break: an unprivileged path (calling a public router function) bypasses a security control the pool admin explicitly configured.

## Likelihood Explanation

`MetricOmmSimpleRouter` is the primary user-facing swap entrypoint. Any user who reads the interface can call it. No privileged access, no special setup, and no malicious initial configuration is required. The attacker only needs to call a public router function with a valid swap path. The condition is trivially reachable by any unprivileged trader.

## Recommendation

The `beforeSwap` hook signature receives `sender` (the pool's `msg.sender`) but not the original transaction initiator. The correct fix is to have the pool record the original transaction initiator via transient storage (analogous to how the router already uses `_setNextCallbackContext` for callback context) and expose it to extensions. The router should set this context before calling `pool.swap()`, and the pool should pass the verified original sender to extensions rather than raw `msg.sender`. Alternatively, the `beforeSwap` hook signature should be extended to carry the verified original sender set by the pool from a trusted periphery context. Until fixed, `SwapAllowlistExtension` should not be used with `MetricOmmSimpleRouter` on pools where the allowlist is a security boundary.

## Proof of Concept

```
1. Pool admin deploys a pool with SwapAllowlistExtension.
2. Pool admin calls setAllowedToSwap(pool, router, true)
   — required so that any router-mediated swap can pass the hook.
3. Alice (allowlisted) and Bob (NOT allowlisted) both want to swap.
4. Bob calls MetricOmmSimpleRouter.exactInputSingle({pool: pool, ...}).
5. Router calls pool.swap(...) — msg.sender to pool = router.
6. Pool calls _beforeSwap(router, recipient, ...).
7. SwapAllowlistExtension.beforeSwap checks allowedSwapper[pool][router] == true → passes.
8. Bob's swap executes successfully despite not being on the allowlist.
9. The curated pool's access control is completely bypassed.
```
