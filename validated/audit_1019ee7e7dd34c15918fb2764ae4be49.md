### Title
Protobuf `ValidResourceBounds` Variant Flip Produces Divergent Transaction Hash Across P2P Sync Boundary — (`crates/apollo_protobuf/src/converters/transaction.rs`)

### Summary

The `TryFrom<protobuf::ResourceBounds> for ValidResourceBounds` conversion in `crates/apollo_protobuf/src/converters/transaction.rs` silently converts `AllResources { l1_gas: X, l2_gas: 0, l1_data_gas: 0 }` to `L1Gas(X)`. Because `get_tip_resource_bounds_hash` in `crates/starknet_api/src/transaction_hash.rs` produces **different Poseidon digests** for these two variants — `AllResources` appends a `L1_DATA_GAS` field while `L1Gas` does not — a V3 transaction submitted with `AllResources` and zero l2/l1_data bounds will carry a different hash after a protobuf round-trip. This breaks the canonicalization invariant that the same transaction bytes must hash to the same value regardless of the deserialization path.

### Finding Description

**Step 1 — Two variants, two hashes.**

`get_tip_resource_bounds_hash` in `crates/starknet_api/src/transaction_hash.rs` (lines 188–211) branches on the enum variant:

```
L1Gas(X)                                  → poseidon(tip, L1_GAS_packed, L2_GAS_packed(0))
AllResources { l1_gas:X, l2_gas:0, l1_data_gas:0 } → poseidon(tip, L1_GAS_packed, L2_GAS_packed(0), L1_DATA_GAS_packed(0))
```

The two inputs differ only in the trailing `L1_DATA_GAS_packed(0)` element, so the Poseidon digests are distinct: **H1 ≠ H2**. [1](#0-0) 

**Step 2 — Protobuf deserialization silently flips the variant.**

`TryFrom<protobuf::ResourceBounds> for ValidResourceBounds` (lines 431–435) maps any wire message where both `l2_gas` and `l1_data_gas` are zero to `L1Gas`, regardless of whether the sender originally used `AllResources`:

```rust
Ok(if l1_data_gas.is_zero() && l2_gas.is_zero() {
    ValidResourceBounds::L1Gas(l1_gas)
} else {
    ValidResourceBounds::AllResources(...)
})
``` [2](#0-1) 

The serializer (`From<ValidResourceBounds> for protobuf::ResourceBounds`, lines 471–490) emits identical wire bytes for both `L1Gas(X)` and `AllResources { l1_gas:X, l2_gas:0, l1_data_gas:0 }`, so the round-trip is **not lossless** for this case. [3](#0-2) 

**Step 3 — The gateway accepts the triggering transaction.**

`StatelessTransactionValidator::validate_resource_bounds` (lines 56–88) accepts `AllResourceBounds { l1_gas: NON_EMPTY, l2_gas: default(), l1_data_gas: default() }` when `min_gas_price = 0` (the default in `DEFAULT_VALIDATOR_CONFIG_FOR_TESTING`). The positive test case `valid_l1_gas` explicitly confirms this path: [4](#0-3) [5](#0-4) 

**Step 4 — End-to-end hash divergence.**

1. User submits `RpcInvokeTransactionV3` with `AllResourceBounds { l1_gas: X, l2_gas: 0, l1_data_gas: 0 }`.
2. Gateway wraps it as `ValidResourceBounds::AllResources` and computes **H1** (AllResources hash). User signs over H1.
3. Transaction is sequenced and committed to a block.
4. Block is propagated via P2P. The transaction is serialized to protobuf and deserialized on the receiving node as `ValidResourceBounds::L1Gas(X)`.
5. Receiving node recomputes the hash → **H2** (L1Gas hash). **H1 ≠ H2**. [6](#0-5) 

The same divergence affects `valid_resource_bounds_as_felts` (used in OS execution context), which also branches on the variant and omits `L1DataGas` for `L1Gas`: [7](#0-6) 

### Impact Explanation

**High — Transaction conversion or signature/hash logic binds the wrong hash.**

- If the P2P sync path validates the hash (via `validate_transaction_hash`), it will reject the block containing the transaction, since the recomputed hash H2 does not match the stored hash H1.
- If the sync path trusts the transmitted hash, the wrong `ValidResourceBounds` variant (`L1Gas` instead of `AllResources`) is stored. Re-execution (e.g., for proving) then uses `GasVectorComputationMode::NoL2Gas` instead of `All`, altering fee computation and resource-bounds enforcement. The account's `__validate__` syscall receives H2 (recomputed from the stored `L1Gas` variant) instead of H1 (the value the user signed), causing signature verification to fail.

### Likelihood Explanation

**Medium.** The trigger requires `min_gas_price = 0` in the gateway's `StatelessTransactionValidatorConfig`, which is the default in test/bootstrap configurations and may appear in production deployments. The user-controlled input (zero l2_gas and l1_data_gas in an `AllResources` transaction) is accepted by the gateway under this configuration, as confirmed by the `valid_l1_gas` positive test case.

### Recommendation

Fix `TryFrom<protobuf::ResourceBounds> for ValidResourceBounds` to always produce `AllResources` for V3 transactions, removing the value-dependent variant flip:

```rust
// Instead of branching on is_zero(), always return AllResources for V3 context:
Ok(ValidResourceBounds::AllResources(AllResourceBounds { l1_gas, l2_gas, l1_data_gas }))
```

Alternatively, fix `get_tip_resource_bounds_hash` to always include the `L1_DATA_GAS` field regardless of the variant (treating `L1Gas` as `AllResources` with zero data-gas), so the hash is canonical over the underlying field values rather than the Rust enum discriminant.

### Proof of Concept

```rust
use starknet_api::transaction::fields::{AllResourceBounds, ResourceBounds, ValidResourceBounds};
use starknet_api::block::GasPrice;
use starknet_api::execution_resources::GasAmount;
use starknet_api::transaction_hash::get_tip_resource_bounds_hash;
use starknet_api::transaction::fields::Tip;

let l1_gas = ResourceBounds { max_amount: GasAmount(100), max_price_per_unit: GasPrice(1) };

// Hash as AllResources (how the gateway computes it at submission)
let all_resources = ValidResourceBounds::AllResources(AllResourceBounds {
    l1_gas,
    l2_gas: ResourceBounds::default(),   // zero
    l1_data_gas: ResourceBounds::default(), // zero
});
let h1 = get_tip_resource_bounds_hash(&all_resources, &Tip::default()).unwrap();

// Hash as L1Gas (how the P2P sync node computes it after protobuf round-trip)
let l1_gas_variant = ValidResourceBounds::L1Gas(l1_gas);
let h2 = get_tip_resource_bounds_hash(&l1_gas_variant, &Tip::default()).unwrap();

// h1 != h2 — same underlying data, different hashes
assert_ne!(h1, h2, "Hash divergence confirmed: AllResources hash != L1Gas hash");
```

The protobuf round-trip that triggers the flip:

```rust
// Serialize AllResources with zero l2/l1_data → protobuf bytes
let proto: protobuf::ResourceBounds = all_resources.into();
// Deserialize → silently becomes L1Gas
let deserialized = ValidResourceBounds::try_from(proto).unwrap();
assert!(matches!(deserialized, ValidResourceBounds::L1Gas(_)));
// Recomputed hash now equals h2, not h1
```

pub fn get_tip_resource_bounds_hash(
    resource_bounds: &ValidResourceBounds,
    tip: &Tip,
) -> Result<Felt, StarknetApiError> {
    let l1_resource_bounds = resource_bounds.get_l1_bounds();
    let l2_resource_bounds = resource_bounds.get_l2_bounds();

    // L1 and L2 gas bounds always exist.
    // Old V3 txs always have L2 gas bounds of zero, but they exist.
    let mut resource_felts = vec![
        get_concat_resource(&l1_resource_bounds, L1_GAS)?,
        get_concat_resource(&l2_resource_bounds, L2_GAS)?,
    ];

    // For new V3 txs, need to also hash the data gas bounds.
    resource_felts.extend(match resource_bounds {
        ValidResourceBounds::L1Gas(_) => vec![],
        ValidResourceBounds::AllResources(all_resources) => {
            vec![get_concat_resource(&all_resources.l1_data_gas, L1_DATA_GAS)?]
        }
    });

    Ok(HashChain::new().chain(&tip.0.into()).chain_iter(resource_felts.iter()).get_poseidon_hash())
}

impl TryFrom<protobuf::ResourceBounds> for ValidResourceBounds {
    type Error = ProtobufConversionError;
    fn try_from(value: protobuf::ResourceBounds) -> Result<Self, Self::Error> {
        let Some(l1_gas) = value.l1_gas else {
            return Err(missing("ResourceBounds::l1_gas"));
        };
        let Some(l2_gas) = value.l2_gas else {
            return Err(missing("ResourceBounds::l2_gas"));
        };
        // TODO(Shahak): Assert data gas is not none once we remove support for 0.13.2.
        let l1_data_gas = value.l1_data_gas.unwrap_or_default();
        let l1_gas: ResourceBounds = l1_gas.try_into()?;
        let l2_gas: ResourceBounds = l2_gas.try_into()?;
        let l1_data_gas: ResourceBounds = l1_data_gas.try_into()?;
        Ok(if l1_data_gas.is_zero() && l2_gas.is_zero() {
            ValidResourceBounds::L1Gas(l1_gas)
        } else {
            ValidResourceBounds::AllResources(AllResourceBounds { l1_gas, l2_gas, l1_data_gas })
        })
    }
}

pub fn valid_resource_bounds_as_felts(
    resource_bounds: &ValidResourceBounds,
    exclude_l1_data_gas: bool,
) -> Result<Vec<ResourceAsFelts>, StarknetApiError> {
    let mut resource_bounds_vec: Vec<_> = vec![
        ResourceAsFelts::new(Resource::L1Gas, &resource_bounds.get_l1_bounds())?,
        ResourceAsFelts::new(Resource::L2Gas, &resource_bounds.get_l2_bounds())?,
    ];
    if exclude_l1_data_gas {
        return Ok(resource_bounds_vec);
    }
    if let ValidResourceBounds::AllResources(AllResourceBounds { l1_data_gas, .. }) =
        resource_bounds
    {
        resource_bounds_vec.push(ResourceAsFelts::new(Resource::L1DataGas, l1_data_gas)?)
    }
    Ok(resource_bounds_vec)
}
