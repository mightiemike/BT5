Audit Report

## Title
`SwapAllowlistExtension.beforeSwap` checks router address instead of originating user, enabling full allowlist bypass — (`metric-periphery/contracts/extensions/SwapAllowlistExtension.sol`)

## Summary
`SwapAllowlistExtension.beforeSwap` receives `sender` from the pool, which equals the pool's `msg.sender` — the immediate caller of `pool.swap`. When `MetricOmmSimpleRouter` is used, that caller is the router contract, not the end user. If the pool admin allowlists the router address (necessary for allowlisted users to swap via the router), every unpermissioned address can bypass the allowlist by routing through `MetricOmmSimpleRouter`. The sole swap-path access-control guard is rendered inoperative.

## Finding Description
`MetricOmmPool.swap` passes its own `msg.sender` as the first argument to `_beforeSwap`: [1](#0-0) 

`SwapAllowlistExtension.beforeSwap` receives this value as `sender` and checks it against the per-pool allowlist, where `msg.sender` inside the extension is the pool address: [2](#0-1) 

`MetricOmmSimpleRouter.exactInputSingle` calls `pool.swap` directly, making the router the pool's `msg.sender`: [3](#0-2) 

The same pattern applies to `exactOutputSingle` and the multi-hop `exactInput`/`exactOutput` paths. [4](#0-3) 

The result is a two-sided failure: allowlisted users cannot use the router (their address is not the `sender` the extension sees), so the admin must allowlist the router address itself. Once `allowedSwapper[pool][router] == true`, the check passes for **any** caller who routes through the router, regardless of whether they are individually allowlisted.

By contrast, `DepositAllowlistExtension.beforeAddLiquidity` checks `owner` — an explicit argument passed independently of `msg.sender` — so the liquidity path does not share this flaw: [5](#0-4) 

## Impact Explanation
A pool deploying `SwapAllowlistExtension` to restrict trading to KYC'd or permissioned counterparties loses that restriction entirely once the router is allowlisted. Any unpermissioned address can execute swaps against the pool's liquidity at oracle-derived prices. This constitutes a broken core pool access-control mechanism causing potential direct loss of LP value and violation of the admin-boundary invariant: the pool admin's intended restriction is bypassed by an unprivileged caller via a public router entry point.

## Likelihood Explanation
`MetricOmmSimpleRouter` is the primary user-facing swap entry point. Any pool operator who deploys `SwapAllowlistExtension` and wants their allowlisted users to be able to use the router will inevitably allowlist the router address, triggering the bypass. No special privilege is required: calling `router.exactInputSingle` with a standard token approval is sufficient.

## Recommendation
The extension must check the economic actor, not the immediate pool caller. The cleanest fix mirrors `DepositAllowlistExtension`: have `MetricOmmPool.swap` accept an explicit `swapper` argument (distinct from `msg.sender`) and forward it to extensions, so the router can pass its own `msg.sender` as the true swapper. Alternatively, the router can encode `msg.sender` into `extensionData` and the extension can decode and verify it, but this requires a coordinated convention and is fragile if new routers are added without updating the extension.

## Proof of Concept
```
Setup:
  pool configured with SwapAllowlistExtension
  admin calls setAllowedToSwap(pool, alice, true)   // alice is KYC'd
  admin calls setAllowedToSwap(pool, router, true)  // required so alice can use the router

Attack:
  bob (not allowlisted) calls:
    router.exactInputSingle({pool: pool, recipient: bob, ...})
      → router calls pool.swap(bob, ...)
        pool.msg.sender = router
        _beforeSwap(sender=router, ...)
          SwapAllowlistExtension.beforeSwap(sender=router, ...)
            allowedSwapper[pool][router] == true  ✓  (passes)
        swap executes at oracle price
        bob receives output tokens

Result:
  bob, who was never individually allowlisted, completes a swap against the
  restricted pool. The SwapAllowlistExtension guard is fully bypassed.
```
