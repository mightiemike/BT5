### Title
Child `ChunkExtra.proposed_split` Inherits Parent's Stale Value After Shard Split, Causing `InvalidChunkHeaderShardSplit` Chain Stall - (File: chain/chain/src/resharding/manager.rs)

---

### Summary

When dynamic resharding splits a parent shard into two children, the resharding manager creates each child's initial `ChunkExtra` by cloning the parent's `ChunkExtra` and updating only `state_root` and `congestion_info`. The `proposed_split` field — which carries the `TrieSplit` value that triggered the resharding — is never reset to `None` for the children. The child shard's first chunk header will carry `proposed_split = None` (freshly computed; the child is new and below threshold), but `validate_chunk_with_chunk_extra_and_receipts_root` compares it against `prev_chunk_extra.proposed_split() = Some(TrieSplit)` (stale, inherited from parent). The mismatch produces `InvalidChunkHeaderShardSplit`, causing every honest chunk producer's first chunk for each child shard to be rejected — a chain stall at every dynamic-resharding boundary.

---

### Finding Description

**Root cause — missing field reset in child `ChunkExtra` construction:**

```rust
// chain/chain/src/resharding/manager.rs ~L255-265

// typing. Clarify where it should happen when `State` and
// `FlatState` update is implemented.

*child_chunk_extra.state_root_mut() = trie_changes.new_root;
*child_chunk_extra.congestion_info_mut() = child_congestion_info;
// proposed_split is NOT reset — it inherits parent's Some(TrieSplit)
``` [1](#0-0) 

The parent shard's last chunk (the one that triggered resharding) has `proposed_split = Some(TrieSplit)` in its `ChunkExtra`. The child `ChunkExtra` is a clone of this, so both children start life with `proposed_split = Some(TrieSplit)`.

**Validation path that rejects the child's first chunk:**

```rust
// chain/chain/src/validate.rs L176-185
if prev_chunk_extra.proposed_split() != chunk_header.proposed_split() {
    DYNAMIC_RESHARDING_VALIDATION_FAILURES.with_label_values(&["chunk_header"]).inc();
    return Err(Error::InvalidChunkHeaderShardSplit(...));
}
``` [2](#0-1) 

This check is reached for every new chunk via `validate_chunk_with_chunk_extra` → `validate_chunk_with_chunk_extra_and_receipts_root`, called unconditionally in `apply_chunks_preprocessing`. [3](#0-2) 

**Child's first chunk header carries `None`:**

`compute_proposed_split` returns `None` for the child shard's first chunk because:
- `is_next_block_possibly_last_in_epoch` is false for most blocks, so the function returns early with `Ok(None)`.
- Even near the epoch boundary, the child shard is freshly created and its memory usage is far below the threshold. [4](#0-3) 

**The divergent values:**

| Source | `proposed_split` value |
|--------|----------------------|
| `prev_chunk_extra` (child's initial `ChunkExtra`) | `Some(TrieSplit { boundary_account, left_memory, right_memory })` — inherited from parent |
| `chunk_header.proposed_split()` (child's first chunk) | `None` — freshly computed |

**The bug is explicitly acknowledged in the codebase:**

> `chain/chain/src/resharding/manager.rs:249` — The resharding manager doesn't set all `ChunkExtra` fields (notably the new `proposed_split` field). [5](#0-4)

### Impact Explanation

Every honest chunk producer for a child shard will produce a chunk with `proposed_split = None`. Every validator will reject it with `InvalidChunkHeaderShardSplit`. No valid chunk can be produced for either child shard until the stale `ChunkExtra` is overwritten — which requires a valid chunk to be accepted first. This is a deadlock: the chain stalls permanently at the resharding boundary for both child shards.

`DynamicResharding` is assigned to protocol version 85 and `STABLE_PROTOCOL_VERSION = 86`, so the feature is active in the current stable binary. [7](#0-6) 

---

### Likelihood Explanation

The bug triggers the first time a shard split executes under `ShardLayoutConfig::Dynamic`. The mainnet `EpochConfigStore` includes a config entry at version 85 (the `DynamicResharding` stabilization version), and the `EpochConfig` at that version determines whether `ShardLayoutConfig::Dynamic` is active. Once any shard's memory usage crosses `memory_usage_threshold` and a split is committed, both child shards are immediately broken. No privileged role or attacker action is required — the stall is caused by honest chunk producers following the protocol correctly. [8](#0-7) 

---

### Recommendation

In `process_memtrie_resharding_storage_update` (or wherever child `ChunkExtra` is constructed), explicitly reset `proposed_split` to `None` after cloning the parent:

```rust
let mut child_chunk_extra = ChunkExtra::clone(&parent_chunk_extra);
*child_chunk_extra.state_root_mut() = trie_changes.new_root;
*child_chunk_extra.congestion_info_mut() = child_congestion_info;
*child_chunk_extra.proposed_split_mut() = None;  // ADD THIS
```

Alternatively, use `ChunkExtra::new_with_only_state_root` as the base and populate the required fields explicitly, which already sets `proposed_split = None` by construction. [9](#0-8) 

Add a regression test that performs a shard split and verifies the child shards' first chunks are accepted without `InvalidChunkHeaderShardSplit`.

---

### Proof of Concept

1. Configure a network with `ShardLayoutConfig::Dynamic` and a low `memory_usage_threshold`.
2. Load a shard past the threshold so `compute_proposed_split` returns `Some(TrieSplit)` near the epoch boundary.
3. Observe the split committed in the last block of epoch N: `BlockHeaderInnerRestV6.shard_split = Some((shard_id, boundary_account))`.
4. At epoch N+2, the child shards begin producing chunks. Each child's `prev_chunk_extra.proposed_split()` is `Some(TrieSplit)` (inherited from parent).
5. Each child's chunk header carries `proposed_split = None` (freshly computed by `compute_proposed_split`).
6. `validate_chunk_with_chunk_extra_and_receipts_root` fires `InvalidChunkHeaderShardSplit` for every chunk produced for either child shard.
7. The chain stalls; no further blocks can be finalized for the affected shards. [10](#0-9) [1](#0-0)

/// Validate that all next chunk information matches previous chunk extra.
pub fn validate_chunk_with_chunk_extra_and_receipts_root(
    prev_chunk_extra: &ChunkExtra,
    chunk_header: &ShardChunkHeader,
    outgoing_receipts_root: &CryptoHash,
) -> Result<(), Error> {
    if *prev_chunk_extra.state_root() != chunk_header.prev_state_root() {
        return Err(Error::InvalidStateRoot);
    }

    if prev_chunk_extra.outcome_root() != chunk_header.prev_outcome_root() {
        return Err(Error::InvalidOutcomesProof);
    }

    let chunk_extra_proposals = prev_chunk_extra.validator_proposals();
    let chunk_header_proposals = chunk_header.prev_validator_proposals();
    if chunk_header_proposals.len() != chunk_extra_proposals.len()
        || !chunk_extra_proposals.eq(chunk_header_proposals)
    {
        return Err(Error::InvalidValidatorProposals);
    }

    if prev_chunk_extra.gas_limit() != chunk_header.gas_limit() {
        return Err(Error::InvalidGasLimit);
    }

    if prev_chunk_extra.gas_used() != chunk_header.prev_gas_used() {
        return Err(Error::InvalidGasUsed);
    }

    if prev_chunk_extra.balance_burnt() != chunk_header.prev_balance_burnt() {
        return Err(Error::InvalidBalanceBurnt);
    }

    if outgoing_receipts_root != chunk_header.prev_outgoing_receipts_root() {
        return Err(Error::InvalidReceiptsProof);
    }

    validate_congestion_info(prev_chunk_extra.congestion_info(), chunk_header.congestion_info())?;
    validate_bandwidth_requests(
        prev_chunk_extra.bandwidth_requests(),
        chunk_header.bandwidth_requests(),
    )?;

    if prev_chunk_extra.proposed_split() != chunk_header.proposed_split() {
        DYNAMIC_RESHARDING_VALIDATION_FAILURES.with_label_values(&["chunk_header"]).inc();
        return Err(Error::InvalidChunkHeaderShardSplit(format!(
            "header has {:?}, expected {:?} (prev block hash: {:?} height created: {:?})",
            chunk_header.proposed_split(),
            prev_chunk_extra.proposed_split(),
            chunk_header.prev_block_hash(),
            chunk_header.height_created(),
        )));
    }

    Ok(())
}
