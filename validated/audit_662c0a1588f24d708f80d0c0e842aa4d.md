### Title
Difficulty Retarget Uses Mainchain Ancestor Timestamp Instead of Fork Ancestor Timestamp — (`contract/src/dogecoin.rs`, `contract/src/litecoin.rs`, `contract/src/bitcoin.rs`)

---

### Summary

When computing the required difficulty for a fork block at a retarget boundary, all three chain implementations call `get_header_by_height()` to retrieve the first block of the retarget interval. That function always reads from the **mainchain** height-to-hash mapping, not from the fork's actual ancestor chain. When the fork diverges before the start of the retarget interval, the timestamp used in the difficulty calculation belongs to a different block than the fork's true ancestor, producing an incorrect expected difficulty target. This is the direct analog of H-28: a stored/stale value (mainchain ancestor timestamp) is used in a calculation instead of the correct current value (fork ancestor timestamp).

---

### Finding Description

`get_header_by_height()` is implemented as:

```rust
fn get_header_by_height(&self, height: u64) -> ExtendedHeader {
    self.mainchain_height_to_header
        .get(&height)
        .and_then(|hash| self.headers_pool.get(&hash))
        .unwrap_or_else(|| env::panic_str(ERR_KEY_NOT_EXIST))
}
``` [1](#0-0) 

It unconditionally resolves height → hash via `mainchain_height_to_header`. This is correct for mainchain submissions, but wrong for fork submissions.

**Bitcoin** (`bitcoin.rs` lines 78–86): at every 2016-block retarget, `get_next_work_required` fetches the first block of the interval by height:

```rust
let first_block_height =
    prev_block_header.block_height - (config.difficulty_adjustment_interval - 1);
let interval_tail_extend_header = blocks_getter.get_header_by_height(first_block_height);
calculate_next_work_required(
    config,
    prev_block_header,
    interval_tail_extend_header.block_header.time.into(),
)
``` [2](#0-1) 

**Litecoin** (`litecoin.rs` lines 86–93) has the identical pattern: [3](#0-2) 

**Dogecoin** (`dogecoin.rs` lines 286–297) has the same pattern, and the developers themselves flagged it with an explicit TODO:

```rust
// TODO: check if it is correct to get block header by height from mainchain without looping to find the ancestor
let first_block_time = blocks_getter
    .get_header_by_height(height_first)
    .block_header
    .time;
``` [4](#0-3) 

For Dogecoin post-height 145,000, `difficulty_adjustment_interval = 1`, so `blocks_to_go_back = 1` and `height_first = prev_block_height - 1`. [5](#0-4) 

This means: for every Dogecoin fork block at height H+2 (where the fork diverges at H), the difficulty is computed using the **mainchain** block at height H, not the fork's block at height H. Those two blocks can have different timestamps, producing a different `modulated_timespan` and therefore a different `expected_bits`.

The `calculate_next_work_required` functions for all three chains derive the new target directly from this timestamp difference: [6](#0-5) [7](#0-6) 

---

### Impact Explanation

The contract computes `expected_bits` from the wrong timestamp and then enforces `block_header.bits == expected_bits`. A fork block whose `bits` field matches the incorrectly computed target is accepted even though it does not satisfy the correct PoW consensus rules for that fork chain. The `chain_work` accumulated for that block is then `work_from_bits(incorrect_bits)`, which is wrong relative to what the correct difficulty would have produced. [8](#0-7) 

If the mainchain ancestor's timestamp is older than the fork ancestor's timestamp, `modulated_timespan` is inflated, the computed target is easier, and the fork block is accepted with less PoW than required. Each such block contributes less chainwork, but the block itself passes the PoW check with less actual mining effort. Across many fork blocks (especially in Dogecoin's every-block retarget regime), this systematic undercount of required work can allow a fork to be promoted to mainchain via `reorg_chain` with a total chainwork that does not reflect the correct PoW rules, corrupting the canonical chain mapping. [9](#0-8) 

---

### Likelihood Explanation

- **Dogecoin (post-145,000)**: Every block is a retarget. Any fork block at height ≥ fork_divergence + 2 triggers the bug. This is essentially every fork block beyond the first two, making the condition nearly universal for any fork submission.
- **Bitcoin / Litecoin**: The fork must diverge before the start of the 2016-block retarget interval. This is a deeper fork but is still a realistic scenario for a relayer submitting a competing chain.
- The entry point (`submit_blocks`) is reachable by any registered relayer, which is an unprivileged role requiring only a stake deposit. [10](#0-9) 

---

### Recommendation

Replace `get_header_by_height(first_block_height)` with an ancestor traversal that walks back through `get_prev_header()` from `prev_block_header` until reaching `first_block_height`. This ensures the timestamp used in the difficulty calculation always belongs to the actual ancestor of the block being validated, regardless of whether it is on the mainchain or a fork.

---

### Proof of Concept

**Dogecoin scenario (post-145,000):**

1. Mainchain has blocks at heights 0 … N. Mainchain block at height N has timestamp `T_main`.
2. A fork diverges at height N: the fork's block at height N has timestamp `T_fork ≠ T_main` (both valid under MTP and future-time checks).
3. Relayer submits fork block at height N+1 (prev = fork block at N). `height_first = N`.
4. `get_header_by_height(N)` returns the **mainchain** block at N with timestamp `T_main`.
5. `modulated_timespan = time(fork block N+1) - T_main` instead of `time(fork block N+1) - T_fork`.
6. If `T_main < T_fork`, `modulated_timespan` is larger → easier target → fork block N+1 accepted with less PoW than required by correct rules.
7. Repeated across subsequent fork blocks, the fork accumulates chainwork under incorrect difficulty rules.
8. If fork chainwork exceeds mainchain chainwork, `reorg_chain` promotes the fork, corrupting `mainchain_tip_blockhash` and the height-to-hash mapping. [11](#0-10)

    fn reorg_chain(&mut self, fork_tip_header: ExtendedHeader, last_main_chain_block_height: u64) {
        let fork_tip_height = fork_tip_header.block_height;
        if last_main_chain_block_height > fork_tip_height {
            // If we see that main chain is longer than fork we first garbage collect
            // outstanding main chain blocks:
            //
            //      [m1] - [m2] - [m3] - [m4] <- We should remove [m4]
            //     /
            // [m0]
            //     \
            //      [f1] - [f2] - [f3]
            for height in (fork_tip_height + 1)..=last_main_chain_block_height {
                let current_main_chain_blockhash = self
                    .mainchain_height_to_header
                    .get(&height)
                    .unwrap_or_else(|| env::panic_str("cannot get a block"));
                self.remove_block_header(&current_main_chain_blockhash);
                self.mainchain_height_to_header.remove(&height);
            }
        }

        // Now we are in a situation where mainchain is equivalent to fork size:
        //
        //      [m1] - [m2] - [m3] - [m4] <- main tip
        //     /
        // [m0]
        //     \
        //      [f1] - [f2] - [f3] - [f4] <- fork tip
        //
        //
        // Or in a situation where it is shorter:
        //
        //      [m1] - [m2] - [m3] <- main tip
        //     /
        // [m0]
        //     \
        //      [f1] - [f2] - [f3] - [f4] <- fork tip

        let fork_tip_hash = fork_tip_header.block_hash.clone();
        let mut fork_header_cursor = fork_tip_header;

        while !self
            .mainchain_header_to_height
            .contains_key(&fork_header_cursor.block_hash)
        {
            let prev_block_hash = fork_header_cursor.block_header.prev_block_hash;
            let current_block_hash = fork_header_cursor.block_hash;
            let current_height = fork_header_cursor.block_height;

            // Inserting the fork block into the main chain, if some mainchain block is occupying
            // this height let's save its hashcode
            let main_chain_block = self
                .mainchain_height_to_header
                .insert(&current_height, &current_block_hash);
            self.mainchain_header_to_height
                .insert(&current_block_hash, &current_height);

            // If we found a mainchain block at the current height than remove this block from the
            // header pool and from the header -> height map
            if let Some(current_main_chain_blockhash) = main_chain_block {
                self.remove_block_header(&current_main_chain_blockhash);
            }

            // Switch iterator cursor to the previous block in fork
            fork_header_cursor = self
                .headers_pool
                .get(&prev_block_hash)
                .unwrap_or_else(|| env::panic_str("previous fork block should be there"));
        }

        // Updating tip of the new main chain
        self.mainchain_tip_blockhash = fork_tip_hash;
    }

fn calculate_next_work_required(
    config: &NetworkConfig,
    prev_block_header: &ExtendedHeader,
    first_block_time: i64,
) -> u32 {
    let prev_block_time: i64 = prev_block_header.block_header.time.into();

    let mut actual_time_taken: i64 = prev_block_time - first_block_time;
    if actual_time_taken < config.pow_target_timespan / 4 {
        actual_time_taken = config.pow_target_timespan / 4;
    }
    if actual_time_taken > config.pow_target_timespan * 4 {
        actual_time_taken = config.pow_target_timespan * 4;
    }

    let new_target = target_from_bits(prev_block_header.block_header.bits);

    let (mut new_target, new_target_overflow) =
        new_target.overflowing_mul(<i64 as TryInto<u64>>::try_into(actual_time_taken).unwrap());
    require!(!new_target_overflow, "new target overflow");
    new_target = new_target
        / U256::from(<i64 as TryInto<u64>>::try_into(config.pow_target_timespan).unwrap());

    if new_target > config.pow_limit {
        new_target = config.pow_limit;
    }

    new_target.target_to_bits()
}

```rust
fn calculate_next_work_required(
    config: &DogecoinConfig,
    prev_block_header: &ExtendedHeader,
    first_block_time: i64,
) -> u32 {
    let retarget_timespan = config.pow_target_timespan;
    let modulated_timespan = i64::from(prev_block_header.block_header.time) - first_block_time;

    let mut modulated_timespan = retarget_timespan + (modulated_timespan - retarget_timespan) / 8;

    let min_timespan = retarget_timespan - (retarget_timespan / 4);
    let max_timespan = retarget_timespan + (retarget_timespan / 2);

    if modulated_timespan < min_timespan {
        modulated_timespan = min_timespan;
    } else if modulated_timespan > max_timespan {
        modulated_timespan = max_timespan;
    }

    let new_target = target_from_bits(prev_block_header.block_header.bits);

    let (mut new_target, new_target_overflow) =
        new_target.overflowing_mul(<i64 as TryInto<u64>>::try_into(modulated_timespan).unwrap());
    require!(!new_target_overflow, "new target overflow");
    new_target =
        new_target / U256::from(<i64 as TryInto<u64>>::try_into(retarget_timespan).unwrap());

    if new_target > config.pow_limit {
        new_target = config.pow_limit;
    }

    new_target.target_to_bits()
