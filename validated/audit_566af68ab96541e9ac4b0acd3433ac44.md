### Title
`l2_gas_price_wei` Derived from Proposer-Controlled L1 Price Ratio Bypasses Independent Rate Validation — (`File: crates/apollo_consensus_orchestrator/src/utils.rs`)

---

### Summary

`convert_to_sn_api_block_info` derives `l2_gas_price_wei` (the ETH-denominated L2 gas price used for execution) by back-calculating `eth_to_fri_rate` from the proposer-supplied `l1_gas_price_fri / l1_gas_price_wei` ratio. The validator only checks each L1 price component independently within a ±10% margin; it never validates the ratio between them. A malicious proposer can therefore skew the ratio by up to ~20% while keeping both individual values inside the margin, producing a `l2_gas_price_wei` that is up to ~20% wrong. This wrong price is passed directly to the batcher as the authoritative `BlockInfo.eth_gas_prices.l2_gas_price`, causing incorrect ETH-denominated fees for every transaction in the block.

---

### Finding Description

**Invariant broken:** The ETH-denominated L2 gas price committed to the block must equal `l2_gas_price_fri` converted at the validator's own oracle rate, not at the proposer's supplied ratio.

**Root cause in `convert_to_sn_api_block_info`:**

```rust
// crates/apollo_consensus_orchestrator/src/utils.rs  lines 318-328
let proposal_init_info = PreviousProposalInitInfo::from(init);   // uses proposer's L1 prices
let eth_to_fri_rate = calculate_eth_to_fri_rate(&proposal_init_info)?;
//   = init.l1_gas_price_fri * WEI_PER_ETH / init.l1_gas_price_wei   (proposer-controlled)

let l2_gas_price_wei =
    NonzeroGasPrice::new(init.l2_gas_price_fri.fri_to_wei(eth_to_fri_rate)?)?;
//   = l2_gas_price_fri * l1_gas_price_wei / l1_gas_price_fri         (proposer-controlled ratio)
``` [1](#0-0) 

`calculate_eth_to_fri_rate` computes `l1_gas_price_fri * WEI_PER_ETH / l1_gas_price_wei` entirely from the proposer-supplied `ProposalInit` fields: [2](#0-1) 

**What the validator actually checks** (`is_proposal_init_valid`): each of the four L1 price fields is validated independently within `l1_gas_price_margin_percent = 10%`: [3](#0-2) 

The margin is 10% across all deployed versioned-constants files: [4](#0-3) 

**What is never checked:** the ratio `l1_gas_price_fri / l1_gas_price_wei`. A proposer can set `l1_gas_price_fri` to `+10%` of the reference and `l1_gas_price_wei` to `-10%` of the reference simultaneously. Both pass `within_margin`, but the derived rate is `(+10%) / (-10%) ≈ +22%` off. The resulting `l2_gas_price_wei` is therefore `≈ 18%` lower than the correct value (or `≈ 22%` higher in the opposite direction).

The wrong `l2_gas_price_wei` is then placed into `eth_gas_prices.l2_gas_price` in the `BlockInfo` sent to the batcher: [5](#0-4) 

This `BlockInfo` is passed to `batcher.validate_block` (validator path) and `batcher.propose_block` (proposer path): [6](#0-5) 

**`ProposalInit` does not carry `l2_gas_price_wei`** — it is absent from the wire format: [7](#0-6) 

So there is no field the validator could compare against; the value is silently derived from the proposer's ratio.

---

### Impact Explanation

`eth_gas_prices.l2_gas_price` is the price used to charge ETH fees for L2 gas consumption (V1/V2 transactions). A wrong value here means:

- **Undercharge path** (proposer sets `l1_gas_price_fri` high, `l1_gas_price_wei` low): `l2_gas_price_wei` is reduced by up to ~18%. Every ETH-paying transaction in the block pays less than the correct L2 gas fee. The sequencer absorbs the shortfall.
- **Overcharge path** (proposer sets `l1_gas_price_fri` low, `l1_gas_price_wei` high): `l2_gas_price_wei` is inflated by up to ~22%. Users are overcharged for L2 gas in ETH.

This matches: **Critical — Incorrect fee, gas, bouncer, resource accounting, refund, balance, or L1 gas price effect with economic impact.**

---

### Likelihood Explanation

Any validator that wins a proposal round can exploit this. No special privilege beyond being a proposer is required. The manipulation is within the allowed margin so it passes all existing checks. The attacker controls the exact skew direction and magnitude (up to the margin limit) and can repeat it every block they propose.

---

### Recommendation

Derive `l2_gas_price_wei` from the validator's own independently-fetched `eth_to_fri_rate` (the same rate used in `get_l1_prices_in_fri_and_wei`) rather than back-calculating it from the proposer-supplied L1 price ratio. Concretely, `is_proposal_init_valid` should compute the local `eth_to_fri_rate` from its own oracle call and pass it to `convert_to_sn_api_block_info`, replacing the current `calculate_eth_to_fri_rate(&PreviousProposalInitInfo::from(init))` call that trusts the proposer's ratio.

---

### Proof of Concept

Assume the validator's oracle returns:
- `l1_gas_price_wei_ref = 1000`, `l1_gas_price_fri_ref = 1_000_000`
- True `eth_to_fri_rate = 1_000_000 * WEI_PER_ETH / 1000 = 10^21`
- `l2_gas_price_fri = 3_000_000_000` (3 Gfri)

A malicious proposer sends `ProposalInit` with:
- `l1_gas_price_fri = 1_100_000` (+10%, within margin ✓)
- `l1_gas_price_wei = 900` (-10%, within margin ✓)

Both pass `within_margin` checks in `is_proposal_init_valid`.

`convert_to_sn_api_block_info` then computes:
```
eth_to_fri_rate = 1_100_000 * WEI_PER_ETH / 900 ≈ 1.222 × 10^21   (+22.2%)
l2_gas_price_wei = 3_000_000_000 * WEI_PER_ETH / 1.222×10^21
                 ≈ 2_454_000_000                                     (−18.2% vs correct 3_000_000_000)
```

Every ETH-paying transaction in this block is charged `l2_gas_price_wei = 2.454 Gwei` instead of the correct `3 Gwei` for L2 gas, an 18% undercharge per unit of L2 gas consumed.

fn calculate_eth_to_fri_rate(
    proposal_init_info: &PreviousProposalInitInfo,
) -> Result<u128, StarknetApiError> {
    let eth_to_fri_rate = proposal_init_info
        .l1_prices_fri
        .l1_gas_price
        .0
        .checked_mul(WEI_PER_ETH)
        .ok_or_else(|| {
            StarknetApiError::GasPriceConversionError(format!(
                "Gas price in Fri should be small enough to multiply by WEI_PER_ETH. Previous \
                 proposal init info: {:?}",
                proposal_init_info
            ))
        })?
        .checked_div(proposal_init_info.l1_prices_wei.l1_gas_price.0)
        .ok_or_else(|| {
            StarknetApiError::GasPriceConversionError(format!(
                "Gas price in Wei should be non-zero. Previous proposal init info: {:?}",
                proposal_init_info
            ))
        })?;
    if eth_to_fri_rate == 0 {
        return Err(StarknetApiError::GasPriceConversionError(format!(
            "Eth to fri rate is zero. Previous proposal init info: {:?}",
            proposal_init_info
        )));
    }
    Ok(eth_to_fri_rate)
