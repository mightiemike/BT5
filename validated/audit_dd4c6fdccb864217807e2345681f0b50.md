### Title
Unclaimed Airdrop Tokens Are Permanently Locked With No Recovery Mechanism - (File: core/contracts/Airdrop.sol)

### Summary
The Nado `Airdrop` contract holds the airdrop token and distributes it via Merkle-proof claims, but provides no function for the owner to recover unclaimed tokens. Any tokens not claimed by users are permanently locked in the contract.

### Finding Description
`Airdrop.sol` is a 96-line contract. The only path by which the airdrop token ever leaves the contract is through `_claim()`, which calls `SafeERC20.safeTransfer(IERC20(token), msg.sender, totalAmount)` after verifying a Merkle proof. [1](#0-0) 

There is no `recoverWrongToken`, no `sweep`, no `withdrawUnclaimed`, and no time-gated owner withdrawal of any kind. The full contract surface is:

- `registerMerkleRoot` — owner-only, registers a weekly Merkle root
- `claim` — user-facing, transfers tokens to the caller
- `getClaimed` — view only [2](#0-1) 

Once tokens are deposited into the contract to fund a weekly airdrop round, any portion not claimed by users is irrecoverable. The `token` address is set at initialization and never used in any withdrawal path accessible to the owner. [3](#0-2) 

### Impact Explanation
**Impact: High.** Unclaimed airdrop tokens accumulate in the contract across every weekly round. Because some users will inevitably fail to claim (lost keys, sanctioned addresses, disinterest, or simply missing the window), a non-trivial portion of the airdrop supply becomes permanently inaccessible. There is no on-chain path — for the owner or anyone else — to recover these tokens.

### Likelihood Explanation
**Likelihood: Low.** The scenario requires that at least some users do not claim their allocation, which is a near-certainty over many weekly rounds. The loss is passive and accumulates silently.

### Recommendation
Add an owner-callable recovery function that can only be invoked after a defined claim window has closed (e.g., after `N` weeks have elapsed since a round was registered). Example:

```solidity
function recoverUnclaimed(address to) external onlyOwner {
    // e.g., require a minimum elapsed time since last round
    uint256 balance = IERC20(token).balanceOf(address(this));
    SafeERC20.safeTransfer(IERC20(token), to, balance);
}
```

A time-lock ensures users retain a fair window to claim before the owner can sweep residual tokens.

### Proof of Concept
1. Owner calls `registerMerkleRoot(1, root)` and funds the contract with 1,000,000 tokens for week 1.
2. Only 800,000 tokens are claimed by eligible users over the claim period.
3. The remaining 200,000 tokens sit in the contract.
4. Owner attempts to recover them — no function exists to do so.
5. Tokens are permanently locked. The same occurs for every subsequent weekly round. [4](#0-3)

    function registerMerkleRoot(uint32 week, bytes32 merkleRoot)
        external
        onlyOwner
    {
        pastWeeks += 1;
        require(week == pastWeeks, "Invalid week provided.");
        merkleRoots[week] = merkleRoot;
    }

    function _verifyProof(
        uint32 week,
        address sender,
        uint256 totalAmount,
        bytes32[] calldata proof
    ) internal {
        require(claimed[week][sender] == 0, "Already claimed.");
        require(
            merkleRoots[week] != bytes32(0),
            "Week hasn't been registered."
        );
        require(
            !ISanctionsList(sanctions).isSanctioned(sender),
            "address is sanctioned."
        );
        bytes32 leaf = keccak256(
            bytes.concat(keccak256(abi.encode(sender, totalAmount)))
        );
        bool isValidLeaf = MerkleProof.verify(proof, merkleRoots[week], leaf);
        require(isValidLeaf, "Invalid proof.");
        claimed[week][sender] = totalAmount;
    }

    function _claim(
        uint32 week,
        uint256 totalAmount,
        bytes32[] calldata proof
    ) internal {
        _verifyProof(week, msg.sender, totalAmount, proof);
        SafeERC20.safeTransfer(IERC20(token), msg.sender, totalAmount);
        emit Claim(msg.sender, week, totalAmount);
    }

    function claim(ClaimProof[] calldata claimProofs) external {
        for (uint32 i = 0; i < claimProofs.length; i++) {
            _claim(
                claimProofs[i].week,
                claimProofs[i].totalAmount,
                claimProofs[i].proof
            );
        }
    }

    function getClaimed(address account)
        external
        view
        returns (uint256[] memory)
    {
        uint256[] memory result = new uint256[](pastWeeks + 1);
        for (uint32 week = 1; week <= pastWeeks; week++) {
            result[week] = claimed[week][account];
        }
        return result;
    }
}
