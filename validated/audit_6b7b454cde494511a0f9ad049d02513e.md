### Title
Single-step NFT transfer by denom creator permanently and irrecoverably locks edit/burn capabilities - (File: x/nft/keeper/keeper.go)

### Summary
The `x/nft` module's `MsgTransferNFT` is a single-step, immediately-final transfer. Because `EditNFT` and `BurnNFT` both require the caller to be simultaneously the NFT **owner** and the denom **creator**, any transfer of an NFT by the denom creator to a non-creator address permanently and irrecoverably removes the ability to edit or burn that NFT. There is no pending-transfer, two-step confirmation, or recovery path.

### Finding Description
The denom creator is the privileged role in the `x/nft` module. It is set once at `IssueDenom` time and is never transferable: [1](#0-0) 

`EditNFT` enforces a dual requirement — the caller must be both the NFT owner **and** the denom creator: [2](#0-1) 

`BurnNFT` enforces the same dual requirement: [3](#0-2) 

`TransferOwner`, called by `MsgTransferNFT`, is a single-step, immediately-final operation with no pending state or confirmation: [4](#0-3) 

The message server wires this directly: [5](#0-4) 

Once the denom creator transfers an NFT to any non-creator address:
- The new owner cannot edit or burn the NFT (fails `IsDenomCreator`).
- The original creator cannot edit or burn the NFT (fails `IsOwner`).
- The NFT is permanently frozen in its current state with no recovery path.

The denom creator role itself has no transfer mechanism, so the state is irrecoverable.

### Impact Explanation
The corrupted invariant is: **the NFT's editability and burnability are permanently destroyed**. Any NFT that was intended to be mutable (metadata updates) or redeemable (burn-on-use) becomes permanently locked the moment the denom creator transfers it to a non-creator address. This is an irreversible on-chain state change with no administrative override.

### Likelihood Explanation
Transferring an NFT is a normal, expected operation (selling, gifting, IBC transfer). The denom creator is the natural first holder of minted NFTs. Any creator who transfers their NFT — even to a legitimate buyer — triggers this condition. The IBC transfer path in `processReceivedPacket` also issues denoms with the escrow address as creator and mints to the receiver, meaning IBC-received NFTs are immediately in this frozen state for the receiver. [6](#0-5) 

### Recommendation
1. Introduce a two-step denom creator transfer: a `PendingCreator` field in `Denom` that must be claimed by the new address before the creator role changes.
2. Alternatively, decouple `EditNFT`/`BurnNFT` authorization so that either the current owner **or** the denom creator can perform these operations independently.
3. At minimum, add a `TransferDenomCreator` message so the creator role can follow the NFT when it is transferred.

### Proof of Concept
1. Alice issues denom `myDenom` via `MsgIssueDenom` — Alice is now `Creator`.
2. Alice mints `token1` in `myDenom` via `MsgMintNFT` with herself as recipient.
3. Alice transfers `token1` to Bob via `MsgTransferNFT` (single-step, immediately final).
4. Bob attempts `MsgEditNFT` on `token1` → fails: `IsDenomCreator` rejects Bob.
5. Alice attempts `MsgEditNFT` on `token1` → fails: `IsOwner` rejects Alice.
6. Bob attempts `MsgBurnNFT` on `token1` → fails: `IsDenomCreator` rejects Bob.
7. Alice attempts `MsgBurnNFT` on `token1` → fails: `IsOwner` rejects Alice.
8. `token1` is permanently frozen with no on-chain recovery path. [7](#0-6)

func (k Keeper) TransferOwner(
	ctx sdk.Context, denomID, tokenID string, srcOwner, dstOwner sdk.AccAddress,
) error {
	if !k.HasDenomID(ctx, denomID) {
		return sdkerrors.Wrapf(types.ErrInvalidDenom, "denom ID %s not exists", denomID)
	}

	nft, err := k.IsOwner(ctx, denomID, tokenID, srcOwner)
	if err != nil {
		return err
	}

	nft.Owner = dstOwner.String()

	k.setNFT(ctx, denomID, nft)
	k.swapOwner(ctx, denomID, tokenID, srcOwner, dstOwner)
	return nil
}

func (k Keeper) BurnNFT(ctx sdk.Context, denomID, tokenID string, owner sdk.AccAddress) error {
	if !k.HasDenomID(ctx, denomID) {
		return sdkerrors.Wrapf(types.ErrInvalidDenom, "denom ID %s not exists", denomID)
	}

	nft, err := k.IsOwner(ctx, denomID, tokenID, owner)
	if err != nil {
		return err
	}

	_, err = k.IsDenomCreator(ctx, denomID, owner)
	if err != nil {
		return err
	}

	k.deleteNFT(ctx, denomID, nft)
	k.deleteOwner(ctx, denomID, tokenID, owner)
	k.decreaseSupply(ctx, denomID)

	return nil
}

func (m msgServer) TransferNFT(goCtx context.Context, msg *types.MsgTransferNFT) (*types.MsgTransferNFTResponse, error) {
	sender, err := sdk.AccAddressFromBech32(msg.Sender)
	if err != nil {
		return nil, err
	}

	recipient, err := sdk.AccAddressFromBech32(msg.Recipient)
	if err != nil {
		return nil, err
	}

	ctx := sdk.UnwrapSDKContext(goCtx)
	if err := m.TransferOwner(ctx, msg.DenomId, msg.Id,
		sender,
		recipient,
	); err != nil {
		return nil, err
	}

	ctx.EventManager().EmitEvents(sdk.Events{
		sdk.NewEvent(
			types.EventTypeTransfer,
			sdk.NewAttribute(types.AttributeKeyTokenID, msg.Id),
			sdk.NewAttribute(types.AttributeKeyDenomID, msg.DenomId),
			sdk.NewAttribute(types.AttributeKeySender, msg.Sender),
			sdk.NewAttribute(types.AttributeKeyRecipient, msg.Recipient),
		),
		sdk.NewEvent(
			sdk.EventTypeMessage,
			sdk.NewAttribute(sdk.AttributeKeyModule, types.AttributeValueCategory),
			sdk.NewAttribute(sdk.AttributeKeySender, msg.Sender),
		),
	})

	return &types.MsgTransferNFTResponse{}, nil
}

// IsDenomCreator checks if address is the creator of Denom
// Return the Denom if true, an error otherwise
func (k Keeper) IsDenomCreator(ctx sdk.Context, denomID string, address sdk.AccAddress) (types.Denom, error) {
	denom, err := k.GetDenom(ctx, denomID)
	if err != nil {
		return types.Denom{}, err
	}

	creator, err := sdk.AccAddressFromBech32(denom.Creator)
	if err != nil {
		panic(err)
	}

	if !creator.Equals(address) {
		return types.Denom{}, sdkerrors.Wrapf(types.ErrUnauthorized, "%s is not the creator of %s", address, denomID)
	}

	return denom, nil
}
