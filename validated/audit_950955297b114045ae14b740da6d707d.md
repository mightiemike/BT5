### Title
Missing Liquidator Health Check in Finalization Path — (`core/contracts/ClearinghouseLiq.sol`)

---

### Summary

When `txn.productId == type(uint32).max`, `liquidateSubaccountImpl` takes the finalization branch, calls `_finalizeSubaccount`, and returns early at line 626 **without ever calling `_handleLiquidationPayment`**. The sole liquidator health guard (`require(!isUnderInitial(txn.sender))`) lives exclusively inside `_handleLiquidationPayment`. `_finalizeSubaccount` itself debits the liquidator's quote balance via `_settlePnlAgainstLiquidator` but performs no health check on `txn.sender`, so the invariant that a liquidator must remain above INITIAL health is silently violated.

---

### Finding Description

`liquidateSubaccountImpl` has two mutually exclusive exit paths:

**Path A — finalization (`txn.productId == type(uint32).max`):** [1](#0-0) 

`_finalizeSubaccount` returns `true`, and the function returns immediately at line 626. `_handleLiquidationPayment` is never reached.

**Path B — normal liquidation:** [2](#0-1) 

`_handleLiquidationPayment` is called, which contains the only liquidator health check in the contract: [3](#0-2) 

Inside `_finalizeSubaccount`, for every perp with a positive `vQuoteBalance`, `_settlePnlAgainstLiquidator` is called: [4](#0-3) 

`_settlePnlAgainstLiquidator` debits the liquidator's quote balance by `pnl`: [5](#0-4) 

After all PnL settlements complete, `_finalizeSubaccount` returns `true` at line 412 with no call to `isUnderInitial(txn.sender)` anywhere in its body. [6](#0-5) 

---

### Impact Explanation

A liquidator can submit a finalization transaction against a liquidatee that holds large positive `vQuoteBalance` across one or more perp products. The liquidator absorbs that PnL (gaining `vQuoteBalance`) while paying quote. If the quote debit is large enough to push the liquidator below INITIAL health, the transaction still succeeds. The liquidator's account is left in an undercollateralized state, violating the protocol invariant and potentially creating a cascading liquidation risk or bad debt.

---

### Likelihood Explanation

The finalization path is a normal, documented liquidation step reachable by any external caller through `Endpoint.processTransactionImpl(LiquidateSubaccount)` with `productId == type(uint32).max`. No special privileges are required. A liquidator only needs to construct a valid signed `LiquidateSubaccount` transaction targeting a liquidatee whose perp positions have been fully closed but still carry positive `vQuoteBalance`.

---

### Recommendation

Add a liquidator health check at the end of `_finalizeSubaccount`, before returning `true`, mirroring the guard already present in `_handleLiquidationPayment`:

```solidity
require(
    txn.sender == N_ACCOUNT || !isUnderInitial(txn.sender),
    ERR_SUBACCT_HEALTH
);
return true;
```

### Proof of Concept

1. Deploy the protocol locally (Hardhat).
2. Create `liquidatee` with: spot positions closed, all perp `amount == 0`, one perp with `vQuoteBalance = +1_000_000e18`, quote balance negative enough to be under maintenance health.
3. Create `liquidator` with: quote balance = `500_000e18` (above INITIAL health before settlement, below INITIAL health after absorbing the `1_000_000e18` PnL debit).
4. Submit `LiquidateSubaccount` with `productId = type(uint32).max`, `sender = liquidator`, `liquidatee = liquidatee`.
5. `_finalizeSubaccount` calls `_settlePnlAgainstLiquidator` with `pnl = 1_000_000e18`, debiting `liquidator`'s quote by `1_000_000e18`.
6. No health check fires. Transaction succeeds.
7. Assert `getHealth(liquidator, INITIAL) < 0` — confirmed.

The missing guard is at: [7](#0-6)

    function _finalizeSubaccount(
        IEndpoint.LiquidateSubaccount calldata txn,
        ISpotEngine spotEngine,
        IPerpEngine perpEngine
    ) internal returns (bool) {
        if (txn.productId != type(uint32).max) {
            return false;
        }
        // check whether the subaccount can be finalized:
        // - all perps positions have closed
        // - all spread positions have closed
        // - all spot assets have closed
        // - all positive pnls have been settled

        FinalizeVars memory v;

        v.spotIds = spotEngine.getProductIds();
        v.perpIds = perpEngine.getProductIds();

        require(v.spotIds[0] == QUOTE_PRODUCT_ID);

        // all spot assets (except USDC) must be closed out
        for (uint32 i = 1; i < v.spotIds.length; ++i) {
            uint32 spotId = v.spotIds[i];
            if (spotEngine.getRisk(spotId).longWeightInitialX18 == 0) {
                continue;
            }
            ISpotEngine.Balance memory balance = spotEngine.getBalance(
                spotId,
                txn.liquidatee
            );
            require(balance.amount <= 0, ERR_NOT_FINALIZABLE_SUBACCOUNT);
        }

        for (uint32 i = 0; i < v.perpIds.length; ++i) {
            uint32 perpId = v.perpIds[i];
            IPerpEngine.Balance memory balance = perpEngine.getBalance(
                perpId,
                txn.liquidatee
            );
            require(balance.amount == 0, ERR_NOT_FINALIZABLE_SUBACCOUNT);
        }

        // settle all positive pnl
        for (uint32 i = 0; i < v.perpIds.length; ++i) {
            uint32 perpId = v.perpIds[i];
            IPerpEngine.Balance memory balance = perpEngine.getBalance(
                perpId,
                txn.liquidatee
            );
            if (balance.vQuoteBalance > 0) {
                _settlePnlAgainstLiquidator(
                    txn,
                    perpId,
                    balance.vQuoteBalance,
                    spotEngine,
                    perpEngine
                );
            }
        }

        ISpotEngine.Balance memory quoteBalance = spotEngine.getBalance(
            QUOTE_PRODUCT_ID,
            txn.liquidatee
        );

        // settle all negative pnl until quote balance becomes 0
        for (uint32 i = 0; i < v.perpIds.length; ++i) {
            uint32 perpId = v.perpIds[i];
            IPerpEngine.Balance memory balance = perpEngine.getBalance(
                perpId,
                txn.liquidatee
            );
            if (balance.vQuoteBalance < 0 && quoteBalance.amount > 0) {
                int128 canSettle = MathHelper.max(
                    balance.vQuoteBalance,
                    -quoteBalance.amount
                );
                _settlePnlAgainstLiquidator(
                    txn,
                    perpId,
                    canSettle,
                    spotEngine,
                    perpEngine
                );
                quoteBalance.amount += canSettle;
            }
        }

        v.insurance = insurance;
        v.insurance -= lastLiquidationFees;
        v.canLiquidateMore = (quoteBalance.amount + v.insurance) > 0;

        if (v.canLiquidateMore) {
            for (uint32 i = 1; i < v.spotIds.length; ++i) {
                uint32 spotId = v.spotIds[i];
                ISpotEngine.Balance memory balance = spotEngine.getBalance(
                    spotId,
                    txn.liquidatee
                );
                if (spotEngine.getRisk(spotId).longWeightInitialX18 == 0) {
                    continue;
                }
                require(balance.amount == 0, ERR_NOT_FINALIZABLE_SUBACCOUNT);
            }
        }

        v.insurance = perpEngine.socializeSubaccount(
            txn.liquidatee,
            v.insurance
        );

        // we can assure that quoteBalance must be non positive, because if quoteBalance.amount > 0,
        // there must be 1) no negative pnl in perps, and 2) no liabilities in spot after above actions.
        // however, in this case the liquidatee must be healthy and cannot pass the health check at
        // the beginning.
        int128 insuranceCover = MathHelper.min(
            v.insurance,
            -quoteBalance.amount
        );
        if (insuranceCover > 0) {
            v.insurance -= insuranceCover;
            spotEngine.updateBalance(
                QUOTE_PRODUCT_ID,
                txn.liquidatee,
                insuranceCover
            );
        }
        if (v.insurance <= 0) {
            spotEngine.socializeSubaccount(txn.liquidatee);
        }
        v.insurance += lastLiquidationFees;
        insurance = v.insurance;
        return true;
    }
