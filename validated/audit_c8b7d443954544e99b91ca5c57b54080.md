### Title
Unprivileged Caller Can Trigger Mainchain GC and Permanently Delete Block Headers - (File: contract/src/lib.rs)

### Summary
`run_mainchain_gc` is a public, state-mutating function on `BtcLightClient` that carries no caller-identity restriction when the contract is not paused. Any unprivileged NEAR account can call it with an arbitrarily large `batch_size`, immediately pruning all GC-eligible mainchain block headers from storage. This permanently corrupts the canonical chain index, breaks `verify_transaction_inclusion` for pruned blocks, and can cause reorg resolution to panic irreversibly.

### Finding Description
`run_mainchain_gc` is decorated only with `#[pause(except(roles(Role::UnrestrictedRunGC)))]`. [1](#0-0) 

The `#[pause]` attribute from `near-plugins` only gates the function when the contract is in a paused state; it imposes no restriction on the caller's identity when the contract is running normally. Compare this with `submit_blocks`, which carries both `#[pause]` and `#[trusted_relayer]`: [2](#0-1) 

`run_mainchain_gc` has no equivalent `#[trusted_relayer]` or any other caller check. The function body deletes entries from `mainchain_height_to_header`, calls `remove_block_header` on each, and advances `mainchain_initial_blockhash`: [3](#0-2) 

An attacker supplies `batch_size = u64::MAX`. The function caps removal at `min(total_amount_to_remove, batch_size)`, so in a single transaction it removes every block header that is currently eligible for GC (i.e., every block older than `gc_threshold` from the tip). This is state that the legitimate relayer flow would only remove gradually, one batch per `submit_blocks` call.

### Impact Explanation
Two concrete broken invariants:

1. **`verify_transaction_inclusion` / `verify_transaction_inclusion_v2` permanently fail for pruned blocks.** Both functions look up the target block in `mainchain_header_to_height` and panic with `"block does not belong to the current main chain"` if the entry is absent: [4](#0-3) 

Once GC'd, the block is gone from both `mainchain_header_to_height` and `headers_pool`. The deletion is irreversible; there is no re-insertion path.

2. **Reorg resolution panics permanently.** The CLAUDE.md documents: *"If mainchain blocks near the fork point have been garbage collected, reorg will fail ‚Äî the contract panics with `PrevBlockNotFound`."* [5](#0-4) 

An attacker who observes a competing fork being built can call `run_mainchain_gc(u64::MAX)` to prune the common-ancestor blocks before the reorg walk completes, causing every subsequent `submit_blocks` on that fork to panic and permanently blocking the reorg.

### Likelihood Explanation
The entry path requires zero privileges: any NEAR account with enough gas can call `run_mainchain_gc` directly. No staking, no role, no deposit is required. The function is part of the public contract ABI. The attacker only needs to know the contract account ID, which is public on-chain.

### Recommendation
Add the `#[trusted_relayer]` macro (or an equivalent ACL role check) to `run_mainchain_gc`, consistent with how `submit_blocks` is protected. Alternatively, restrict the function to callers holding `Role::UnrestrictedRunGC` or `Role::DAO` by adding an explicit `acl_require` guard at the top of the function body, so that unprivileged accounts cannot trigger mainchain state deletion.

### Proof of Concept
```rust
// Any unprivileged NEAR account executes this call:
let outcome = attacker_account
    .call(contract.id(), "run_mainchain_gc")
    .args_json(json!({"batch_size": u64::MAX}))
    .max_gas()
    .transact()
    .await?;
// outcome.is_success() == true
// All GC-eligible mainchain headers are now deleted.

// Subsequent verify_transaction_inclusion for any pruned block panics:
let result = contract
    .view("verify_transaction_inclusion")
    .args_borsh(ProofArgs {
        tx_id: known_tx_id,
        tx_block_blockhash: pruned_block_hash, // now absent from mainchain_header_to_height
        tx_index: 0,
        merkle_proof: valid_proof,
        confirmations: 1,
    })
    .await;
// Panics: "block does not belong to the current main chain"
```
