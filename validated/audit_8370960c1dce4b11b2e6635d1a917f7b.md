### Title
SwapAllowlistExtension Checks Router Address Instead of Original User, Enabling Full Allowlist Bypass — (`metric-periphery/contracts/extensions/SwapAllowlistExtension.sol`)

---

### Summary

`SwapAllowlistExtension.beforeSwap` gates swaps by checking the `sender` argument, which is the direct caller of `pool.swap()`. When users route through `MetricOmmSimpleRouter`, `sender` is the router's address, not the original user. If the pool admin allowlists the router (a natural step to let allowlisted users use the standard periphery), every user — including those not individually allowlisted — can bypass the curated-pool guard.

---

### Finding Description

`SwapAllowlistExtension.beforeSwap` performs its identity check as:

```solidity
if (!allowAllSwappers[msg.sender] && !allowedSwapper[msg.sender][sender]) {
    revert IMetricOmmPoolActions.NotAllowedToSwap();
}
``` [1](#0-0) 

Here `msg.sender` is the pool (correct) and `sender` is whatever the pool received as `msg.sender` when `swap()` was called. The pool passes its own `msg.sender` directly to `_beforeSwap`: [2](#0-1) 

`MetricOmmSimpleRouter.exactInputSingle` calls `pool.swap()` directly, making the pool's `msg.sender` the router contract, not the original user:

```solidity
_setNextCallbackContext(params.pool, CALLBACK_MODE_JUST_PAY, msg.sender, params.tokenIn);
(int128 amount0Delta, int128 amount1Delta) = IMetricOmmPoolActions(params.pool)
    .swap(params.recipient, params.zeroForOne, ..., params.extensionData);
``` [3](#0-2) 

The same pattern holds for `exactInput` (all hops call `pool.swap()` from the router), `exactOutputSingle`, and the recursive `exactOutput` callback — in every case `sender` seen by the extension is the router address. [4](#0-3) 

This creates an impossible configuration for pool admins:

| Admin action | Effect |
|---|---|
| Allowlist individual users only | Allowlisted users **cannot** use the router (router not in allowlist → revert) |
| Allowlist the router | **Every** user can swap through the router, allowlist is nullified |

There is no configuration that simultaneously (a) allows allowlisted users to use the router and (b) blocks non-allowlisted users from using the router.

---

### Impact Explanation

A curated pool whose admin allowlists the router (the standard periphery) loses all swap-access control. Any unprivileged user can call `MetricOmmSimpleRouter.exactInputSingle` (or any other router entry point) targeting the pool and the `SwapAllowlistExtension` will pass because `allowedSwapper[pool][router] == true`. The pool's curation policy — which may exist to protect LP capital, enforce KYC, or limit counterparties — is completely bypassed. LPs in the curated pool are exposed to trades from actors the pool admin explicitly intended to exclude.

---

### Likelihood Explanation

The `MetricOmmSimpleRouter` is the canonical periphery swap contract. A pool admin who wants allowlisted users to be able to use the standard UX (router) will naturally add the router to the allowlist. The admin has no way to know this opens the gate to everyone, because the extension's interface (`setAllowedToSwap(pool, router, true)`) gives no indication that it grants universal access. The misconfiguration is easy to make and hard to detect without reading the extension source.

---

### Recommendation

The extension must gate on the **economically relevant actor** — the original user — not the intermediary router. Two complementary fixes:

1. **Router-side:** Have the router encode the original `msg.sender` into `extensionData` in a trusted, authenticated way (e.g., a dedicated field the extension can read and verify came from a known router).
2. **Extension-side:** `SwapAllowlistExtension` should expose a separate allowlist for trusted routers that maps `router → bool`, and when `sender` is a trusted router, fall back to checking an address extracted from `extensionData` rather than `sender` itself.

Until fixed, pool admins must be warned that allowlisting the router grants universal swap access and that per-user allowlisting only works for direct pool calls.

---

### Proof of Concept

```
Setup:
  pool = curated MetricOmmPool with SwapAllowlistExtension as beforeSwap hook
  admin calls: extension.setAllowedToSwap(pool, alice, true)   // alice is allowlisted
  admin calls: extension.setAllowedToSwap(pool, router, true)  // to let alice use the router

Attack (charlie, not allowlisted):
  charlie calls router.exactInputSingle({pool: pool, ...})
    → router calls pool.swap(recipient, ...)   // msg.sender to pool = router
    → pool calls _beforeSwap(sender=router, ...)
    → SwapAllowlistExtension checks allowedSwapper[pool][router] == true  ✓
    → swap executes for charlie with no revert

Result:
  charlie trades on a pool he was never authorized to access.
  alice's allowlist entry is irrelevant; the router entry is the only check that matters.
``` [5](#0-4) [6](#0-5)
