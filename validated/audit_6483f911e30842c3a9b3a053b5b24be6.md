### Title
`ValidResourceBounds` protobuf round-trip silently downgrades `AllResources{l2_gas:0, l1_data_gas:0}` to `L1Gas`, producing a wrong transaction hash preimage and wrong `initial_sierra_gas` on synced nodes — (`File: crates/apollo_protobuf/src/converters/transaction.rs`)

---

### Summary

The `TryFrom<protobuf::ResourceBounds> for ValidResourceBounds` implementation in `transaction.rs` collapses `AllResources { l1_gas: X, l2_gas: 0, l1_data_gas: 0 }` into `L1Gas(X)` whenever both zero-valued fields are present. Because the serializer emits identical protobuf bytes for both variants, the round-trip is lossy: the `AllResources` discriminant is permanently discarded. This changes (a) the Poseidon hash preimage used in `get_tip_resource_bounds_hash` (L1_DATA_GAS element is omitted for `L1Gas`), and (b) the `initial_sierra_gas` supplied to the VM (0 for `AllResources{l2_gas:0}` vs. `initial_gas_no_user_l2_bound()` ≈ 200 M for `L1Gas`). A synced node that re-executes such a transaction for `starknet_traceTransaction` / `starknet_traceBlockTransactions` will therefore produce a different revert/success result and a different transaction hash than the sequencer that originally executed it.

---

### Finding Description

**Serializer produces identical bytes for two distinct variants**

`From<ValidResourceBounds> for protobuf::ResourceBounds` (lines 471–489) always emits all three fields:

```
L1Gas(X)                                  → { l1_gas: X, l2_gas: 0, l1_data_gas: 0 }
AllResources { l1_gas: X, l2_gas: 0, l1_data_gas: 0 } → { l1_gas: X, l2_gas: 0, l1_data_gas: 0 }
```

Both produce **identical protobuf bytes**. [1](#0-0) 

**Deserializer always picks `L1Gas` for the ambiguous case**

`TryFrom<protobuf::ResourceBounds> for ValidResourceBounds` (lines 431–432) resolves the ambiguity by choosing `L1Gas` whenever `l2_gas.is_zero() && l1_data_gas.is_zero()`:

```rust
Ok(if l1_data_gas.is_zero() && l2_gas.is_zero() {
    ValidResourceBounds::L1Gas(l1_gas)
} else { ... })
``` [2](#0-1) 

An `AllResources` transaction with zero L2/data-gas bounds is therefore permanently re-labelled `L1Gas` after any P2P sync round-trip.

**Consequence 1 — wrong transaction hash preimage**

`get_tip_resource_bounds_hash` branches on the variant discriminant:

```rust
resource_felts.extend(match resource_bounds {
    ValidResourceBounds::L1Gas(_) => vec![],                          // ← no L1_DATA_GAS element
    ValidResourceBounds::AllResources(all_resources) => {
        vec![get_concat_resource(&all_resources.l1_data_gas, L1_DATA_GAS)?]  // ← zero element present
    }
});
``` [3](#0-2) 

`AllResources { l1_gas: X, l2_gas: 0, l1_data_gas: 0 }` hashes as `Poseidon(tip, L1_GAS‖X, L2_GAS‖0, L1_DATA_GAS‖0)`, while `L1Gas(X)` hashes as `Poseidon(tip, L1_GAS‖X, L2_GAS‖0)`. These are distinct Poseidon outputs, so the stored transaction hash H₁ (computed at the sequencer from `AllResources`) diverges from H₂ (recomputed on the synced node from `L1Gas`).

**Consequence 2 — wrong `initial_sierra_gas` and execution mode**

`get_gas_vector_computation_mode()` returns `All` for `AllResources` and `NoL2Gas` for `L1Gas`: [4](#0-3) 

For `GasVectorComputationMode::All`, `initial_sierra_gas = l2_gas.max_amount`. When `l2_gas.max_amount = 0`, the VM starts with **zero Sierra gas**, causing an immediate out-of-gas revert for any Cairo 1 contract. After the P2P round-trip the variant becomes `L1Gas`, so `initial_sierra_gas = initial_gas_no_user_l2_bound()` (≈ `execute_max_sierra_gas + validate_max_sierra_gas`, on the order of 200 M gas): [5](#0-4) 

The test `test_initial_sierra_gas` confirms this: `GasVectorComputationMode::All` with `l2_gas.max_amount = N` yields `initial_sierra_gas = N`, while `NoL2Gas` yields `initial_gas_no_user_l2_bound()`: [6](#0-5) 

---

### Impact Explanation

A user submits `RpcInvokeTransactionV3` with `AllResourceBounds { l1_gas: X, l2_gas: 0, l1_data_gas: 0 }`. The gateway accepts it (the `valid_l1_gas` test case confirms zero L2/data-gas bounds are valid). The sequencer executes it with `initial_sierra_gas = 0`; any Cairo 1 account or callee immediately runs out of gas and the transaction reverts. The block is committed with a revert receipt.

A synced node receives the block via P2P. The `InvokeTransactionV3.resource_bounds` field is deserialized as `L1Gas(X)`. When the synced node serves `starknet_traceTransaction` or `starknet_traceBlockTransactions` (both of which re-execute transactions from storage), it uses `initial_sierra_gas ≈ 200 M` and `GasVectorComputationMode::NoL2Gas`. The same transaction now **succeeds** instead of reverting, and the returned trace, gas consumption, and fee estimation are all wrong. The RPC response is authoritative-looking but incorrect.

Additionally, any component that recomputes the transaction hash from the deserialized `ValidResourceBounds` (e.g., `blockifier_reexecution`) will produce H₂ ≠ H₁, causing a hash mismatch against the stored block data.

---

### Likelihood Explanation

The trigger is an unprivileged user submitting a V3 invoke transaction with `l2_gas.max_amount = 0` and `l1_data_gas.max_amount = 0` — a valid, accepted input. The gateway test suite explicitly covers this case (`valid_l1_gas`). Any synced node (non-sequencer) that serves trace RPC endpoints is affected. The condition is reachable without any special privileges or malformed bytes.

---

### Recommendation

The deserializer must preserve the `AllResources` variant whenever the original transaction was a post-0.13.3 V3 transaction. The TODO comment at line 426 already acknowledges this:

```
// TODO(Shahak): Assert data gas is not none once we remove support for 0.13.2.
```

The fix is to remove the zero-value heuristic and instead always produce `AllResources` when `l1_data_gas` is present in the protobuf message (even if its value is zero), reserving `L1Gas` only for legacy messages where `l1_data_gas` is absent:

```rust
Ok(if value.l1_data_gas.is_none() && l2_gas.is_zero() {
    ValidResourceBounds::L1Gas(l1_gas)
} else {
    ValidResourceBounds::AllResources(AllResourceBounds { l1_gas, l2_gas, l1_data_gas })
})
```

This matches the intent of the TODO and mirrors the `rpc_transaction.rs` converter, which already requires `l1_data_gas` to be present and always produces `AllResourceBounds` without any zero-value downgrade. [7](#0-6)

### Proof of Concept

1. Construct an `InvokeTransactionV3` with `ValidResourceBounds::AllResources(AllResourceBounds { l1_gas: ResourceBounds { max_amount: 1000, max_price_per_unit: 1 }, l2_gas: ResourceBounds::default(), l1_data_gas: ResourceBounds::default() })`.

2. Compute its hash via `get_invoke_transaction_v3_hash` → H₁. Observe that `get_tip_resource_bounds_hash` includes the `L1_DATA_GAS‖0` element.

3. Serialize to `protobuf::ResourceBounds` via `From<ValidResourceBounds>`. Observe the wire bytes contain `l1_data_gas = { max_amount: 0, max_price_per_unit: 0 }`.

4. Deserialize back via `TryFrom<protobuf::ResourceBounds> for ValidResourceBounds`. Observe the result is `ValidResourceBounds::L1Gas(ResourceBounds { max_amount: 1000, max_price_per_unit: 1 })` — the `AllResources` discriminant is gone.

5. Recompute the hash → H₂. Observe H₁ ≠ H₂ because `L1_DATA_GAS‖0` is absent from the Poseidon chain.

6. Confirm `get_gas_vector_computation_mode()` returns `NoL2Gas` for the deserialized value, and that `initial_sierra_gas` is `initial_gas_no_user_l2_bound()` instead of 0. [8](#0-7) [9](#0-8) [10](#0-9)

    transaction: &T,
    chain_id: &ChainId,
    transaction_version: &TransactionVersion,
) -> Result<TransactionHash, StarknetApiError> {
    let tip_resource_bounds_hash =
        get_tip_resource_bounds_hash(&transaction.resource_bounds(), transaction.tip())?;
    let paymaster_data_hash =
        HashChain::new().chain_iter(transaction.paymaster_data().0.iter()).get_poseidon_hash();
    let data_availability_mode = concat_data_availability_mode(
        transaction.nonce_data_availability_mode(),
        transaction.fee_data_availability_mode(),
    );
    let account_deployment_data_hash = HashChain::new()
        .chain_iter(transaction.account_deployment_data().0.iter())
        .get_poseidon_hash();
    let calldata_hash =
        HashChain::new().chain_iter(transaction.calldata().0.iter()).get_poseidon_hash();
    let mut hash_chain = HashChain::new()
        .chain(&INVOKE)
        .chain(&transaction_version.0)
        .chain(transaction.sender_address().0.key())
        .chain(&tip_resource_bounds_hash)
        .chain(&paymaster_data_hash)
        .chain(&Felt::try_from(chain_id)?)
        .chain(&transaction.nonce().0)
        .chain(&data_availability_mode)
        .chain(&account_deployment_data_hash)
        .chain(&calldata_hash);
    if !transaction.proof_facts().0.is_empty() {
        let proof_facts_hash =
            HashChain::new().chain_iter(transaction.proof_facts().0.iter()).get_poseidon_hash();
        hash_chain = hash_chain.chain(&proof_facts_hash);
    }
    Ok(TransactionHash(hash_chain.get_poseidon_hash()))

#[derive(Clone, Copy, Debug, Eq, PartialEq, Hash, Ord, PartialOrd)]
pub enum ValidResourceBounds {
    L1Gas(ResourceBounds), // Pre 0.13.3. Only L1 gas. L2 bounds are signed but never used.
    AllResources(AllResourceBounds),
}

impl From<AllResourceBounds> for ValidResourceBounds {
    fn from(value: AllResourceBounds) -> Self {
        Self::AllResources(value)
    }
}

impl ValidResourceBounds {
    pub fn get_l1_bounds(&self) -> ResourceBounds {
        match self {
            Self::L1Gas(l1_bounds) => *l1_bounds,
            Self::AllResources(AllResourceBounds { l1_gas, .. }) => *l1_gas,
        }
    }

    pub fn get_l2_bounds(&self) -> ResourceBounds {
        match self {
            Self::L1Gas(_) => ResourceBounds::default(),
            Self::AllResources(AllResourceBounds { l2_gas, .. }) => *l2_gas,
        }
    }

    /// Returns the maximum possible fee that can be charged for the transaction.
    /// The computation is saturating, meaning that if the result is larger than the maximum
    /// possible fee, the maximum possible fee is returned.
    pub fn max_possible_fee(&self, tip: Tip) -> Fee {
        match self {
            ValidResourceBounds::L1Gas(l1_bounds) => {
                l1_bounds.max_amount.saturating_mul(l1_bounds.max_price_per_unit)
            }
            ValidResourceBounds::AllResources(AllResourceBounds {
                l1_gas,
                l2_gas,
                l1_data_gas,
            }) => l1_gas
                .max_amount
                .saturating_mul(l1_gas.max_price_per_unit)
                .saturating_add(
                    l2_gas
                        .max_amount
                        .saturating_mul(l2_gas.max_price_per_unit.saturating_add(tip.into())),
                )
                .saturating_add(
                    l1_data_gas.max_amount.saturating_mul(l1_data_gas.max_price_per_unit),
                ),
        }
    }

    pub fn get_gas_vector_computation_mode(&self) -> GasVectorComputationMode {
        match self {
            Self::AllResources(_) => GasVectorComputationMode::All,
            Self::L1Gas(_) => GasVectorComputationMode::NoL2Gas,
        }
    }
