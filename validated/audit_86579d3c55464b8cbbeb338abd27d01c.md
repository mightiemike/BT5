### Title
Malicious IBC Counterparty Can Drain Escrowed NFTs via Crafted ClassId in `processReceivedPacket` — (`x/nft-transfer/keeper/packet.go`)

### Summary

A malicious IBC counterparty (chain B) can craft a `NonFungibleTokenPacketData` packet whose `ClassId` is prefixed with the packet's own source port/channel (e.g., `"nft/channel-X/localclass"`), causing `IsAwayFromOrigin` to return `false` on the receiving chain (chain A). This forces `processReceivedPacket` into the unescrow branch, which calls `TransferOwner(escrowAddress, receiver)` for a token the escrow address legitimately holds from a prior outbound transfer — effectively stealing it.

---

### Finding Description

**Entrypoint:** `IBCModule.OnRecvPacket` → `keeper.OnRecvPacket` → `keeper.processReceivedPacket`

**Step 1 — Direction check is purely string-based.**

`IsAwayFromOrigin` in `x/nft-transfer/types/trace.go` checks only whether `data.ClassId` starts with `srcPort/srcChannel/`: [1](#0-0) 

There is no cryptographic or state-based verification that the prefix was legitimately added by a prior receive on chain A.

**Step 2 — Attacker crafts the packet.**

Attacker controls chain B (source). They submit a transaction on chain B that emits a packet with:
- `SourcePort = "nft"`, `SourceChannel = "channel-X"` (chain B's side of the channel)
- `data.ClassId = "nft/channel-X/localclass"` (prefixed with the packet's own source port/channel)
- `data.TokenIds = ["tok1"]`, `data.Receiver = attacker_address`

**Step 3 — `IsAwayFromOrigin` returns `false`.**

On chain A receiving the packet:
```
IsAwayFromOrigin("nft", "channel-X", "nft/channel-X/localclass")
  → prefixClassID = "nft/channel-X/"
  → "nft/channel-X/localclass" HasPrefix "nft/channel-X/" → true
  → returns !true = false
```

**Step 4 — Unescrow branch executes.** [2](#0-1) 

```
unprefixedClassID = RemoveClassPrefix("nft", "channel-X", "nft/channel-X/localclass") = "localclass"
voucherClassID    = ParseClassTrace("localclass").IBCClassID() = "localclass"
escrowAddress     = GetEscrowAddress("nft", "channel-0")   // chain A's dest channel
TransferOwner(ctx, "localclass", "tok1", escrowAddress, attacker)
```

**Step 5 — Precondition is satisfied by any prior legitimate transfer.**

When Alice legitimately sent `tok1` (class `localclass`) from chain A to chain B, `createOutgoingPacket` escrowed it: [3](#0-2) 

`escrowAddress = GetEscrowAddress("nft", "channel-0")` now owns `tok1` in class `localclass`. The attacker's `TransferOwner` call passes the ownership check (escrow address legitimately owns the token) and transfers `tok1` to the attacker.

**No guard prevents this.** `OnRecvPacket` only calls `data.ValidateBasic()` before delegating: [4](#0-3) 

`ValidateBasic` checks that fields are non-empty and addresses are valid — it does not validate that `ClassId` is consistent with the channel's escrow history. There is no state lookup confirming that a prior outbound transfer through this channel locked the token.

---

### Impact Explanation

Any NFT currently held by the escrow address for a given channel can be stolen by the counterparty chain. The attacker receives the NFT on chain A while the voucher on chain B (minted during the legitimate transfer) remains outstanding, breaking the invariant that total supply across chains is conserved. This is an unauthorized transfer of arbitrary NFTs — a Critical NFT ownership invariant break.

---

### Likelihood Explanation

Requires the attacker to control chain B (or hold sufficient validator power to produce a fraudulent packet commitment that passes IBC light-client verification). This is a realistic threat model for any chain that establishes an IBC channel with an untrusted or compromised counterparty. The attack is deterministic and requires no brute force.

---

### Recommendation

In the unescrow branch of `processReceivedPacket`, verify that the derived `voucherClassID` and `tokenID` are actually owned by the escrow address before calling `TransferOwner`. Alternatively, maintain an on-chain escrow ledger (mapping `classID → tokenID → escrowed bool`) that is written during `createOutgoingPacket` and checked/cleared during `processReceivedPacket`. This mirrors the approach used in ICS-20 fungible token transfer, which tracks escrowed amounts per channel.

---

### Proof of Concept

1. Chain A has NFT class `localclass`, token `tok1`, owned by Alice.
2. Alice calls `SendTransfer` on chain A → `tok1` is escrowed to `GetEscrowAddress("nft", "channel-0")` on chain A; a packet with `ClassId = "localclass"` is sent to chain B.
3. Chain B receives the packet, mints a voucher `ibc/{hash(nft/channel-X/localclass)}` for `tok1` to Alice on chain B.
4. Attacker (controlling chain B) submits a transaction on chain B that sends a new packet:
   - `ClassId = "nft/channel-X/localclass"` (chain B's source port/channel + `localclass`)
   - `TokenIds = ["tok1"]`
   - `Receiver = attacker_address_on_chain_A`
5. Relayer submits this packet to chain A.
6. Chain A's `processReceivedPacket` enters the unescrow branch, calls `TransferOwner("localclass", "tok1", escrowAddress, attacker)`.
7. Since `escrowAddress` owns `tok1`, the transfer succeeds. Attacker now owns `tok1` on chain A.
8. Alice's voucher on chain B still exists; the escrow on chain A is now empty. The invariant is broken. [5](#0-4) [6](#0-5)

```go
func RemoveClassPrefix(portID, channelID, classID string) string {
	classPrefix := GetClassPrefix(portID, channelID)
	return classID[len(classPrefix):]
}

// IsAwayFromOrigin determine if non-fungible token is moving away from
// the origin chain (the chain issued by the native nft).
// Note that fullClassPath refers to the full path of the unencoded classID.
// The longer the fullClassPath, the farther it is from the origin chain
func IsAwayFromOrigin(sourcePort, sourceChannel, fullClassPath string) bool {
	prefixClassID := GetClassPrefix(sourcePort, sourceChannel)
	return !strings.HasPrefix(fullClassPath, prefixClassID)
}
```

```go
func (k Keeper) processReceivedPacket(ctx sdk.Context, packet channeltypes.Packet,
	data types.NonFungibleTokenPacketData,
) error {
	receiver, err := sdk.AccAddressFromBech32(data.Receiver)
	if err != nil {
		return err
	}

	isAwayFromOrigin := types.IsAwayFromOrigin(packet.GetSourcePort(), packet.GetSourceChannel(), data.ClassId)

	// create the escrow address for creating denom and minting nft
	escrowAddress := types.GetEscrowAddress(packet.GetDestPort(), packet.GetDestChannel())

	if isAwayFromOrigin {
		// since SendPacket did not prefix the classID, we must prefix classID here
		classPrefix := types.GetClassPrefix(packet.GetDestPort(), packet.GetDestChannel())
		// NOTE: sourcePrefix contains the trailing "/"
		prefixedClassID := classPrefix + data.ClassId

		// construct the class trace from the full raw classID
		classTrace := types.ParseClassTrace(prefixedClassID)
		if !k.HasClassTrace(ctx, classTrace.Hash()) {
			k.SetClassTrace(ctx, classTrace)
		}

		voucherClassID := classTrace.IBCClassID()

		if !k.nftKeeper.HasDenomID(ctx, voucherClassID) {
			if err := k.nftKeeper.IssueDenom(ctx, voucherClassID, voucherClassID, "", data.ClassUri, escrowAddress); err != nil {
				return err
			}
		}
		sdkCtx := sdk.UnwrapSDKContext(ctx)
		sdkCtx.EventManager().EmitEvent(
			sdk.NewEvent(
				types.EventTypeClassTrace,
				sdk.NewAttribute(types.AttributeKeyTraceHash, classTrace.Hash().String()),
				sdk.NewAttribute(types.AttributeKeyClassID, voucherClassID),
			),
		)

		for i, tokenID := range data.TokenIds {
			if err := k.nftKeeper.MintNFT(ctx, voucherClassID, tokenID, "", data.TokenUris[i], "", escrowAddress, receiver); err != nil {
				return err
			}
		}
	} else {
		// If the token moves in the direction of back to origin,
		// we need to unescrow the token and transfer it to the receiver

		// we should remove the prefix. For example:
		// p6/c6/p4/c4/p2/c2/nftClass -> p4/c4/p2/c2/nftClass
		unprefixedClassID := types.RemoveClassPrefix(packet.GetSourcePort(),
			packet.GetSourceChannel(), data.ClassId)

		voucherClassID := types.ParseClassTrace(unprefixedClassID).IBCClassID()
		for _, tokenID := range data.TokenIds {
			if err := k.nftKeeper.TransferOwner(ctx,
				voucherClassID, tokenID, escrowAddress, receiver); err != nil {
				return err
			}
		}
	}
```
