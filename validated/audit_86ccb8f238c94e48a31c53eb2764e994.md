### Title
New Product Added After Fee Tier Customization Inherits Zero Fee Rates for All Customized Tiers — (`File: core/contracts/OffchainExchange.sol`)

---

### Summary

When `updateTierFeeRates` is called with `QUOTE_PRODUCT_ID` to bulk-set fee rates across all existing products for a given tier, it permanently marks that tier as "non-default" via `nonDefaultFeeTierMask`. Any product added **after** this call has uninitialized (`0`) fee rates for that tier. Because `getTierFeeRateX18` unconditionally returns the stored (zero) value for any tier whose bit is set in `nonDefaultFeeTierMask`, all traders at that tier pay zero protocol fees on the new product until the admin separately re-runs `updateTierFeeRates` for it.

---

### Finding Description

`OffchainExchange.sol` stores per-tier, per-product fee rates in a nested mapping and tracks which tiers have been customized with a bitmask:

```
mapping(uint32 => mapping(uint32 => FeeRates)) internal feeRates;
uint128 internal nonDefaultFeeTierMask;
``` [1](#0-0) 

`getTierFeeRateX18` uses the bitmask to decide which path to take:

```solidity
if (nonDefaultFeeTierMask & (1 << tier) != 0) {
    return feeRates[tier][productId];   // returns 0 if never written
}
return FeeRates({ makerRateX18: 0, takerRateX18: 200_000_000_000_000 }); // 2 bps default
``` [2](#0-1) 

`updateTierFeeRates`, when called with `txn.productId == QUOTE_PRODUCT_ID`, iterates over **currently registered** spot and perp products and writes their fee rates, then sets the tier's bit in `nonDefaultFeeTierMask`:

```solidity
for (uint32 i = 0; i < spotProductIds.length; i++) { ... feeRates[txn.tier][spotProductIds[i]] = ...; }
for (uint32 i = 0; i < perpProductIds.length; i++) { ... feeRates[txn.tier][perpProductIds[i]] = ...; }
nonDefaultFeeTierMask |= uint128(1) << txn.tier;
``` [3](#0-2) 

`addOrUpdateProducts` in `ContractOwner.sol` adds new products to the engines with no corresponding `updateTierFeeRates` call:

```solidity
spotEngine.addOrUpdateProduct(call.productId, call.quoteId, ...);
perpEngine.addOrUpdateProduct(call.productId, ...);
``` [4](#0-3) 

After a new product is registered, `feeRates[tier][newProductId]` is zero for every tier whose bit is already set in `nonDefaultFeeTierMask`. `getTierFeeRateX18` will return `{makerRateX18: 0, takerRateX18: 0}` for those tiers on the new product, because the bitmask check routes to the stored (uninitialized) mapping slot instead of the hardcoded 2 bps default.

---

### Impact Explanation

Inside `applyFee`, the fee charged to a trader is computed as:

```solidity
int128 keepRateX18 = ONE - feeInfo.feeRate;
orderInfo.fee = meteredQuote - newMeteredQuote;
``` [5](#0-4) 

When `feeInfo.feeRate == 0`, `keepRateX18 == ONE`, `newMeteredQuote == meteredQuote`, and `orderInfo.fee == 0`. The protocol collects **zero fees** on every trade of the new product for every user whose fee tier has been customized (including tier 0, the default tier for all users). The corrupted state is `marketInfo[newProductId].collectedFees`, which remains 0 instead of accumulating protocol revenue. This persists until the admin separately calls `updateTierFeeRates` for the new product.

---

### Likelihood Explanation

The protocol is designed to grow its product set over time. `updateTierFeeRates` with `QUOTE_PRODUCT_ID` is the standard mechanism to apply a fee schedule globally, and it will have been called at least once before any new product is added in a live deployment. Tier 0 is the default tier for every address (mapping default), so once bit 0 of `nonDefaultFeeTierMask` is set, **all users** pay zero fees on any subsequently added product. There is no on-chain enforcement requiring `updateTierFeeRates` to be called alongside `addOrUpdateProducts`, and no revert or event signals the missing initialization.

---

### Recommendation

In `updateTierFeeRates`, when `txn.productId == QUOTE_PRODUCT_ID`, the function already iterates all existing products. The same pattern should be enforced in reverse: when a new product is added via `addOrUpdateProducts` (or the underlying engine `addOrUpdateProduct`), the contract should iterate all tiers whose bits are set in `nonDefaultFeeTierMask` and initialize `feeRates[tier][newProductId]` to a sensible default (e.g., copying from tier 0 or from a stored "global default" rate). Alternatively, `addOrUpdateProducts` should require a companion `updateTierFeeRates` call for the new product before it becomes tradeable.

---

### Proof of Concept

1. Admin calls `updateTierFeeRates` with `tier=0, productId=QUOTE_PRODUCT_ID, takerRateX18=200_000_000_000_000`. This writes `feeRates[0][existingProductA]` and sets bit 0 of `nonDefaultFeeTierMask`.
2. Admin calls `addOrUpdateProducts` to register `newProductB`. No fee rates are written for `newProductB`. `feeRates[0][newProductB]` remains `{0, 0}`.
3. Trader (any address, default tier 0) submits a taker order on `newProductB` through the sequencer path.
4. `applyFee` calls `getUserFeeRateWithBuilder` → `getTierFeeRateX18(0, newProductB)`. Since bit 0 of `nonDefaultFeeTierMask` is set, it returns `feeRates[0][newProductB]` = `{makerRateX18: 0, takerRateX18: 0}`.
5. `keepRateX18 = ONE - 0 = ONE`. `orderInfo.fee = meteredQuote - meteredQuote = 0`. Protocol receives zero fees.
6. `marketInfo[newProductB].collectedFees` stays at 0 across all trades until admin separately calls `updateTierFeeRates` for `newProductB`. [6](#0-5) [2](#0-1) [7](#0-6) [8](#0-7)

    function updateTierFeeRates(IEndpoint.UpdateTierFeeRates memory txn)
        external
        onlyEndpoint
    {
        if (txn.productId == QUOTE_PRODUCT_ID) {
            uint32[] memory spotProductIds = spotEngine.getProductIds();
            uint32[] memory perpProductIds = perpEngine.getProductIds();
            for (uint32 i = 0; i < spotProductIds.length; i++) {
                if (spotProductIds[i] == QUOTE_PRODUCT_ID) {
                    continue;
                }
                feeRates[txn.tier][spotProductIds[i]] = FeeRates(
                    txn.makerRateX18,
                    txn.takerRateX18
                );
            }
            for (uint32 i = 0; i < perpProductIds.length; i++) {
                feeRates[txn.tier][perpProductIds[i]] = FeeRates(
                    txn.makerRateX18,
                    txn.takerRateX18
                );
            }
        } else {
            feeRates[txn.tier][txn.productId] = FeeRates(
                txn.makerRateX18,
                txn.takerRateX18
            );
        }
        nonDefaultFeeTierMask |= uint128(1) << txn.tier;
    }
