### Title
`MsgExitTierWithDelegation` reverts when validator token-per-share ratio changes between query and execution — (`x/tieredrewards/keeper/msg_server.go`)

### Summary

`MsgExitTierWithDelegation` accepts a token `amount` that is validated against a live `positionAmount` derived from `TokensFromShares(delegation.Shares)`. A user who queries `positionAmount` at block N and submits that exact value as `amount` may have their transaction revert at block N+1 if the validator's token-per-share ratio changes between the query and the transaction's inclusion. This is the direct Cosmos SDK analog of the external report's stale-shares-to-amount conversion causing transaction failure.

### Finding Description

The handler `ExitTierWithDelegation` in `msg_server.go` computes `positionAmount` live from the validator's current exchange rate:

```go
positionAmount, err := ms.reconcileAmountFromShares(ctx, valAddr, pos.Delegation.Shares)
``` [1](#0-0) 

This value is also computed inside `validateExitTierWithDelegation` to enforce the upper bound:

```go
if amount.GT(positionAmount) {
    return errorsmod.Wrapf(types.ErrInvalidAmount, "amount %s exceeds position token value %s", amount, positionAmount)
}
``` [2](#0-1) 

The `positionAmount` is computed by `reconcileAmountFromShares`, which calls `val.TokensFromShares(shares).TruncateInt()` — a live, block-time value: [3](#0-2) 

The full-vs-partial exit decision in `transferDelegationFromPosition` uses an exact equality check (`ExitWithFullDelegation`):

```go
unbondedShares := pos.Delegation.Shares
if !pos.ExitWithFullDelegation(amount, positionAmount) {
    unbondedShares, err = k.stakingKeeper.ValidateUnbondAmount(ctx, posDelAddr, valAddr, amount)
``` [4](#0-3) 

Two concrete failure modes arise from this design:

**Failure Mode 1 — Validator slash between query and execution:**
1. User queries `positionAmount = 10000` at block N.
2. User submits `MsgExitTierWithDelegation{Amount: 10000}`.
3. Before the tx is included, the validator is slashed; `positionAmount` drops to `9000`.
4. Validation: `10000 > 9000` → **revert** with `ErrInvalidAmount`.

**Failure Mode 2 — New delegation to the same validator between query and execution:**
1. User queries `positionAmount = 10000` at block N, intending a full exit.
2. User submits `MsgExitTierWithDelegation{Amount: 10000}`.
3. Before the tx is included, another delegator delegates to the same validator; `positionAmount` becomes `10001` due to Cosmos SDK's integer truncation in `TokensFromShares`.
4. Validation passes (`10000 <= 10001`).
5. `ExitWithFullDelegation(10000, 10001)` → `false` (partial exit path taken).
6. `ValidateUnbondAmount(posDelAddr, valAddr, 10000)` unbonds shares worth 10000 tokens; 1 token worth of shares remains.
7. `MeetsMinLockRequirement(1)` → `false` (tier `MinLockAmount` is typically much larger).
8. **Revert** with `ErrMinLockAmountNotMet`.

The same stale-amount problem affects `MsgCommitDelegationToTier`, where `transferDelegationToPosition` calls `ValidateUnbondAmount(ctx, from, valAddr, amount)` with a user-supplied token amount that may exceed the delegation's current token value after a slash: [5](#0-4) 

The ADR documents that `positionAmount` is intentionally computed on the fly and never stored:

> The `Position` proto does not store token amount — it is computed on the fly from staking state on every read. [6](#0-5) 

### Impact Explanation

A user who follows the standard workflow — query `positionAmount` via gRPC, then submit `MsgExitTierWithDelegation{Amount: positionAmount}` — may have their transaction revert if the validator's token-per-share ratio changes between the query and the block in which the transaction is included. In Failure Mode 2, the user's intended full exit silently becomes a partial exit that then fails the minimum lock check, leaving the position open and requiring the user to re-query and retry. No funds are permanently lost, but the user's exit is blocked until they retry with the updated amount.

### Likelihood Explanation

Validator slashes (Failure Mode 1) are rare but protocol-defined events. New delegations to the same validator (Failure Mode 2) are routine on any active network — any delegation to the validator between the user's query and the next block triggers the issue. The window is the block time (typically 5–6 seconds on Cronos POS Chain), which is long enough for other transactions to land. Any user attempting a full exit via the standard CLI or frontend path is exposed.

### Recommendation

1. **Accept `amount >= positionAmount` as a full exit** in `ExitWithFullDelegation` (i.e., use `>=` instead of `==`). This ensures that if the position's token value has decreased since the user's query, the full delegation shares are still used, avoiding the partial-exit path.
2. **Add a `full_exit` boolean flag** to `MsgExitTierWithDelegation` that, when set, ignores the `amount` field and always unbonds all `pos.Delegation.Shares`. This is the Cosmos SDK analog of the external report's recommendation to accept an amount rather than shares.
3. For `MsgCommitDelegationToTier`, document that `amount` should be queried immediately before submission, or accept a `max_amount` with a slippage tolerance.

func (ms msgServer) ExitTierWithDelegation(ctx context.Context, msg *types.MsgExitTierWithDelegation) (*types.MsgExitTierWithDelegationResponse, error) {
	if err := msg.Validate(); err != nil {
		return nil, err
	}

	pos, err := ms.getPositionState(ctx, msg.PositionId)
	if err != nil {
		return nil, err
	}

	if err := ms.validateExitTierWithDelegation(ctx, pos, msg.Owner, msg.Amount); err != nil {
		return nil, err
	}

	pos, _, _, err = ms.claimRewards(ctx, pos)
	if err != nil {
		return nil, err
	}

	validator := pos.Delegation.ValidatorAddress
	valAddr, err := sdk.ValAddressFromBech32(validator)
	if err != nil {
		return nil, err
	}

	positionAmount, err := ms.reconcileAmountFromShares(ctx, valAddr, pos.Delegation.Shares)
	if err != nil {
		return nil, err
	}

	transferredShares, unbondedShares, transferredAmount, err := ms.transferDelegationFromPosition(ctx, pos, valAddr, msg.Amount)
	if err != nil {
		return nil, err
	}

	// Capture for event before potential deletion.
	posId := pos.Id
	tierId := pos.TierId

	fullExit := pos.ExitWithFullDelegation(msg.Amount, positionAmount)

	if fullExit {
		ownerAddr, err := sdk.AccAddressFromBech32(msg.Owner)
		if err != nil {
			return nil, err
		}

		delAddr, err := sdk.AccAddressFromBech32(pos.DelegatorAddress)
		if err != nil {
			return nil, errorsmod.Wrap(sdkerrors.ErrInvalidAddress, "invalid delegator address")
		}

		// sweeps any remaining dust if any (usually zero)
		balances := ms.bankKeeper.SpendableCoins(ctx, delAddr)
		if !balances.IsZero() {
			if err := ms.bankKeeper.SendCoins(ctx, delAddr, ownerAddr, balances); err != nil {
				return nil, err
			}
		}

		if err := ms.deletePosition(ctx, pos.Position, &ValidatorTransition{PreviousAddress: validator}); err != nil {
			return nil, err
		}

	} else {
		remainingShares := pos.Delegation.Shares.Sub(unbondedShares)
		// Compute remaining token value for min lock check.
		remainingPositionAmount, err := ms.reconcileAmountFromShares(ctx, valAddr, remainingShares)
		if err != nil {
			return nil, err
		}

		tier, err := ms.getTier(ctx, pos.TierId)
		if err != nil {
			return nil, err
		}
		// actual remaining amount (post-transfer) must meet min lock.
		if !tier.MeetsMinLockRequirement(remainingPositionAmount) {
			return nil, errorsmod.Wrapf(types.ErrMinLockAmountNotMet,
				"remaining amount %s is below tier minimum %s", remainingPositionAmount, tier.MinLockAmount)
		}

		if err := ms.setPosition(ctx, pos.Position, nil); err != nil {
			return nil, err
		}
	}

func (k Keeper) validateExitTierWithDelegation(ctx context.Context, pos types.PositionState, owner string, amount math.Int) error {
	if !pos.IsOwner(owner) {
		return types.ErrNotPositionOwner
	}

	if !pos.IsDelegated() {
		return types.ErrPositionNotDelegated
	}

	if !pos.HasTriggeredExit() {
		return types.ErrExitNotTriggered
	}

	sdkCtx := sdk.UnwrapSDKContext(ctx)
	if !pos.CompletedExitLockDuration(sdkCtx.BlockTime()) {
		return types.ErrExitLockDurationNotReached
	}

	positionAmount, err := k.getPositionAmount(ctx, pos)
	if err != nil {
		return err
	}

	if amount.GT(positionAmount) {
		return errorsmod.Wrapf(types.ErrInvalidAmount, "amount %s exceeds position token value %s", amount, positionAmount)
	}

	isRedelegating, err := k.isRedelegating(ctx, pos.DelegatorAddress)
	if err != nil {
		return err
	}
	if isRedelegating {
		return errorsmod.Wrapf(types.ErrActiveRedelegation, "position %d has an active redelegation", pos.Id)
	}

	return nil

func (k Keeper) reconcileAmountFromShares(ctx context.Context, valAddr sdk.ValAddress, shares math.LegacyDec) (math.Int, error) {
	val, err := k.stakingKeeper.GetValidator(ctx, valAddr)
	if err != nil {
		return math.Int{}, err
	}
	if val.GetDelegatorShares().IsZero() {
		return math.ZeroInt(), nil
	}
	return val.TokensFromShares(shares).TruncateInt(), nil
}
