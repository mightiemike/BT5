### Title
Bonus Accrual Checkpoints Permanently Advanced Without Payment on Insufficient Pool During Redelegation Slash — (`x/tieredrewards/keeper/slash.go`)

---

### Summary

In `slashRedelegationPosition`, when `processEventsAndClaimBonus` returns `ErrInsufficientBonusPool`, the error is deliberately swallowed to avoid a chain halt. However, `processEventsAndClaimBonus` has already advanced the position's `LastEventSeq`, `LastBonusAccrual`, and `LastKnownBonded` checkpoints **and** decremented event reference counts in the store before the pool-sufficiency check is reached. The position is then saved with these advanced checkpoints. The user permanently loses all bonus that accrued up to the slash point, with no ability to retry.

---

### Finding Description

**Root cause — ordering in `processEventsAndClaimBonus`** (`claim_rewards.go:142–251`):

The function processes events in a loop, updating in-memory checkpoints and writing reference-count decrements to the store on every iteration, **before** the pool-sufficiency check:

```
for _, entry := range events {
    // bonus accumulation (in-memory)
    pos.UpdateLastEventSeq(entry.Seq)          // checkpoint advanced (line 193)
    k.decrementEventRefCount(ctx, ...)         // STORE WRITE (line 196)
}
applyBonusAccrualCheckpoint(&pos.Position, blockTime)  // LastBonusAccrual advanced (line 215)
pos.UpdateLastKnownBonded(bonded)                       // LastKnownBonded advanced (line 217)

// pool check comes AFTER all the above
if err := k.sufficientBonusPoolBalance(ctx, bonusCoins); err != nil {
    return nil, err   // ErrInsufficientBonusPool returned here (line 230–232)
}
``` [1](#0-0) 

**Error swallowing in `slashRedelegationPosition`** (`slash.go:54–64`):

```go

**What is permanently corrupted:**

| Field | Effect |
|---|---|
| `pos.LastEventSeq` | Advanced to latest event seq; events can never be replayed for this position |
| `pos.LastBonusAccrual` | Advanced to `blockTime`; the accrued-but-unpaid segment is erased |
| `pos.LastKnownBonded` | Updated; future replay starts from wrong bonded state |
| Event `ReferenceCount` | Decremented in the store even though no payment occurred; events may be garbage-collected prematurely |

**Contrast with user-driven paths:** The ADR explicitly states that user-driven paths (ClaimTierRewards, AddToPosition, Undelegate, Redelegate, ClearPosition) "fail atomically — user retries after pool is replenished." [4](#0-3) 

In those paths, the transaction rolls back entirely, so no checkpoints are advanced and the user can retry. In the hook path, the error is swallowed and the position is saved with advanced checkpoints, making the loss permanent and irrecoverable.

---

### Impact Explanation

A delegator who holds a tier position on a validator that undergoes a redelegation slash while the bonus pool is insufficient permanently loses all bonus rewards that accrued from `LastBonusAccrual` to the slash timestamp. The `LastBonusAccrual` checkpoint is advanced to `blockTime` with no payment, so no future claim can recover the lost amount. The corrupted value is the `bonus` field of the position's accrual state, and the lost amount equals `shares × tokensPerShare × bonusApy × duration / SecondsPerYear` for the unpaid segment. [5](#0-4) 

---

### Likelihood Explanation

The trigger requires two concurrent conditions:
1. A tier position exists on a validator with an active redelegation entry (i.e., the user called `MsgTierRedelegate` and the unbonding period has not yet elapsed).
2. The validator is slashed during that redelegation window while the `RewardsPoolName` module account balance is below the accrued bonus.

Both conditions are realistic in production: redelegation windows last the full unbonding period (typically 21 days), and the bonus pool can be depleted by normal bonus payouts or by governance inaction. An adversary who can observe pool balance and validator misbehavior timing can reliably trigger this path.

---

### Recommendation

Apply `processEventsAndClaimBonus` inside a cache context in `slashRedelegationPosition`. If the pool is insufficient, discard the cache context (rolling back all checkpoint and reference-count writes) and log the event without advancing any state. This preserves the chain-halt-avoidance property while allowing the user to retry the claim once the pool is replenished — consistent with the behavior of all user-driven paths.

Alternatively, restructure `processEventsAndClaimBonus` to perform the pool-sufficiency check **before** advancing any checkpoints or decrementing reference counts, so that returning an error leaves the position state fully intact.

---

### Proof of Concept

1. User calls `MsgTierRedelegate` for position P on validator V1 → V2. An unbonding entry is created and a `RedelegationMapping` entry is stored.
2. The `RewardsPoolName` module account is drained (e.g., by many concurrent bonus claims).
3. Validator V2 commits a double-sign infraction. The staking module calls `SlashRedelegation`, which fires `BeforeRedelegationSlashed(ctx, unbondingId, sharesToUnbond)`.
4. `slashRedelegationPosition` is entered. `processEventsAndClaimBonus` runs the full event loop, advancing `pos.LastEventSeq`, `pos.LastBonusAccrual`, `pos.LastKnownBonded`, and decrementing all event reference counts in the store.
5. `sufficientBonusPoolBalance` returns `ErrInsufficientBonusPool`. The error is swallowed. `setPositionWithState` saves the position with the advanced checkpoints.
6. The pool is later replenished. The user calls `MsgClaimTierRewards`. `processEventsAndClaimBonus` starts from the now-advanced `LastBonusAccrual` and `LastEventSeq`, finding no unpaid segment. Zero bonus is returned. The bonus accrued before the slash is permanently lost. [6](#0-5) [7](#0-6)

func (k Keeper) slashRedelegationPosition(ctx context.Context, unbondingId uint64, sharesToUnbond math.LegacyDec) error {
	positionId, err := k.getRedelegationMapping(ctx, unbondingId)
	if errors.Is(err, collections.ErrNotFound) {
		return nil
	}
	if err != nil {
		return err
	}

	pos, err := k.getPositionState(ctx, positionId)
	if errors.Is(err, types.ErrPositionNotFound) {
		k.logger(ctx).Error("position not found during redelegation slash",
			"position_id", positionId,
			"unbonding_id", unbondingId,
			"error", err.Error(),
		)
		return nil
	}
	if err != nil {
		return err
	}

	if !pos.IsDelegated() {
		// Defensive
		k.logger(ctx).Error("delegation missing during BeforeRedelegationSlashed",
			"position_id", positionId,
			"unbonding_id", unbondingId,
			"shares_to_unbond", sharesToUnbond.String(),
		)
		return nil
	}

	dstValStr := pos.Delegation.ValidatorAddress

	// Settle bonus against PRE-slash shares.
	if _, err := k.processEventsAndClaimBonus(ctx, &pos); err != nil {
		// Deliberately forgo bonus rewards if pool is insufficient to prevent chain halt.
		if errors.Is(err, types.ErrInsufficientBonusPool) {
			k.logger(ctx).Error("insufficient bonus pool during redelegation slash",
				"position_id", pos.Id,
				"error", err.Error(),
			)
		} else {
			return err
		}
	}

	fullSlash := sharesToUnbond.GTE(pos.Delegation.Shares)

	if fullSlash {
		pos.Delegation = nil
		pos.ClearBonusCheckpoints()
		return k.setPositionWithState(ctx, pos, &ValidatorTransition{PreviousAddress: dstValStr})
	}
	// In-memory only: the persisted Position carries no share count, and the
	// live delegation will reflect the post-Unbond shares on the next read.
	// Update the local copy so any follow-up logic in this call sees consistent state.
	pos.Delegation.Shares = pos.Delegation.Shares.Sub(sharesToUnbond)
	return k.setPositionWithState(ctx, pos, nil)

func (k Keeper) computeSegmentBonus(pos types.PositionState, tier types.Tier, segmentStart, segmentEnd time.Time, tokensPerShare math.LegacyDec) math.Int {
	if !pos.ExitUnlockAt.IsZero() && segmentEnd.After(pos.ExitUnlockAt) {
		segmentEnd = pos.ExitUnlockAt
	}

	if !segmentEnd.After(segmentStart) {
		return math.ZeroInt()
	}

	durationSeconds := int64(segmentEnd.Sub(segmentStart) / time.Second)
	if durationSeconds <= 0 {
		return math.ZeroInt()
	}

	tokens := pos.Delegation.Shares.Mul(tokensPerShare)

	return tokens.
		Mul(tier.BonusApy).
		MulInt64(durationSeconds).
		QuoInt64(types.SecondsPerYear).
		TruncateInt()
}
