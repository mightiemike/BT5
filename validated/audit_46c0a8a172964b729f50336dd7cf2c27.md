### Title
Crafted `ClassId` Prefix Bypasses `IsAwayFromOrigin` Direction Check, Enabling Unauthorized Unescrow of Victim NFTs — (`x/nft-transfer/keeper/packet.go`)

---

### Summary

An attacker who controls a counterparty chain (or can relay a crafted IBC packet) can set `NonFungibleTokenPacketData.ClassId` to a value that begins with the packet's own source port/channel prefix. This causes `IsAwayFromOrigin` to return `false`, routing execution into the unescrow branch of `processReceivedPacket`. Because the escrow address used in that branch is derived from the **destination** port/channel — which matches the address where legitimate users' NFTs are held — the attacker can transfer any escrowed NFT to an arbitrary receiver without having previously escrowed it.

---

### Finding Description

**`IsAwayFromOrigin` logic** in `x/nft-transfer/types/trace.go`:

```go
func IsAwayFromOrigin(sourcePort, sourceChannel, fullClassPath string) bool {
    prefixClassID := GetClassPrefix(sourcePort, sourceChannel)
    return !strings.HasPrefix(fullClassPath, prefixClassID)
}
```

It returns `false` (i.e., "moving back toward origin") whenever `fullClassPath` starts with `sourcePort/sourceChannel/`. The `sourcePort` and `sourceChannel` here are the **attacker's** port/channel (from the packet header). The attacker fully controls `data.ClassId` in the packet payload. [1](#0-0) 

**`processReceivedPacket` unescrow branch** in `x/nft-transfer/keeper/packet.go`:

```go

### Impact Explanation

**Concrete attack scenario:**

1. Victim on chain D sends NFT `victimClass/victimTokenID` via `SendTransfer` over channel `channel-0` (port `nft`). The NFT is escrowed to `GetEscrowAddress("nft", "channel-0")` on chain D.

2. Attacker controls chain A, connected to chain D via `channel-0` on chain D and `channel-X` on chain A.

3. Attacker crafts a packet from chain A → chain D:
   - `packet.SourcePort = "nft"`, `packet.SourceChannel = "channel-X"`
   - `packet.DestPort = "nft"`, `packet.DestChannel = "channel-0"`
   - `data.ClassId = "nft/channel-X/victimClass"`
   - `data.TokenIds = ["victimTokenID"]`
   - `data.Receiver = attacker_address_on_D`

4. On chain D, `processReceivedPacket` evaluates:
   - `IsAwayFromOrigin("nft", "channel-X", "nft/channel-X/victimClass")` → `false` (prefix matches)
   - `escrowAddress = GetEscrowAddress("nft", "channel-0")` ← same address holding victim's NFT
   - `unprefixedClassID = RemoveClassPrefix("nft", "channel-X", "nft/channel-X/victimClass")` = `"victimClass"`
   - `voucherClassID = "victimClass"`
   - `TransferOwner(ctx, "victimClass", "victimTokenID", escrowAddress, attacker)` → **succeeds**

5. Victim's NFT is transferred to the attacker. The victim's original `SendTransfer` packet, if it eventually completes or times out, will attempt to unescrow or refund an NFT that no longer belongs to the escrow address, causing a chain error or permanent loss.

**State delta:** `victimClass/victimTokenID` owner changes from `GetEscrowAddress("nft", "channel-0")` to attacker's address. The invariant "an NFT in escrow can only be released to the original sender (on timeout/ack error) or to the legitimate receiver (on successful relay)" is broken.

---

### Likelihood Explanation

The precondition — attacker controls a counterparty chain with an established IBC channel to the victim chain — is a standard IBC deployment scenario. Any chain operator can open a channel. The attack requires no governance access, no leaked keys, and no privileged role. The only additional precondition is that a victim NFT is currently in escrow (i.e., a `SendTransfer` is in-flight), which is a normal operational state.

---

### Recommendation

In `processReceivedPacket`, before entering the unescrow branch, verify that the `data.ClassId` is consistent with a legitimate prior escrow. Specifically:

1. **Validate that `data.ClassId` starts with the source port/channel prefix only if the source chain legitimately issued that prefix.** The receiving chain should check that the `ClassId` after stripping the source prefix corresponds to a class trace that was previously registered via a legitimate outbound transfer on this chain (i.e., `HasClassTrace` for the expected trace, or the base class exists natively).

2. Alternatively, enforce that the unescrow branch only executes when the `voucherClassID` derived from the packet matches a class trace that was registered by a prior outbound `SendTransfer` on this chain over the same channel.

---

```go
func (nftpd NonFungibleTokenPacketData) ValidateBasic() error {
	if strings.TrimSpace(nftpd.ClassId) == "" {
		return newsdkerrors.Wrap(ErrInvalidClassID, "classId cannot be blank")
	}

	if len(nftpd.TokenIds) == 0 {
		return newsdkerrors.Wrap(ErrInvalidTokenID, "tokenId cannot be blank")
	}

	if len(nftpd.TokenIds) != len(nftpd.TokenUris) {
		return newsdkerrors.Wrap(ErrInvalidPacket, "tokenIds and tokenUris lengths do not match")
	}

	if strings.TrimSpace(nftpd.Sender) == "" {
		return newsdkerrors.Wrap(sdkerrors.ErrInvalidAddress, "sender address cannot be blank")
	}

	// decode the sender address
	if _, err := sdk.AccAddressFromBech32(nftpd.Sender); err != nil {
		return newsdkerrors.Wrap(sdkerrors.ErrInvalidAddress, "invalid sender address")
	}

	if strings.TrimSpace(nftpd.Receiver) == "" {
		return newsdkerrors.Wrap(sdkerrors.ErrInvalidAddress, "receiver address cannot be blank")
	}

	// decode the receiver address
	if _, err := sdk.AccAddressFromBech32(nftpd.Receiver); err != nil {
		return newsdkerrors.Wrap(sdkerrors.ErrInvalidAddress, "invalid receiver address")
	}
	return nil
}

```go
func (k Keeper) TransferOwner(
	ctx sdk.Context, denomID, tokenID string, srcOwner, dstOwner sdk.AccAddress,
) error {
	if !k.HasDenomID(ctx, denomID) {
		return sdkerrors.Wrapf(types.ErrInvalidDenom, "denom ID %s not exists", denomID)
	}

	nft, err := k.IsOwner(ctx, denomID, tokenID, srcOwner)
	if err != nil {
		return err
	}

	nft.Owner = dstOwner.String()

	k.setNFT(ctx, denomID, nft)
	k.swapOwner(ctx, denomID, tokenID, srcOwner, dstOwner)
	return nil
}
