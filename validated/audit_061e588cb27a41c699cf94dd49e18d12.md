Audit Report

## Title
`SwapAllowlistExtension` checks router address instead of end-user, allowing any caller to bypass per-pool swap allowlist — (`metric-periphery/contracts/extensions/SwapAllowlistExtension.sol`)

## Summary
`SwapAllowlistExtension.beforeSwap` gates on the `sender` argument, which is `msg.sender` at the `pool.swap()` call boundary. When `MetricOmmSimpleRouter` intermediates, that value is the router contract address, not the originating user. A pool admin who allowlists the router to enable user trading inadvertently opens the allowlist to every address on-chain, completely nullifying the access control.

## Finding Description
The call chain is confirmed by the production code:

**Step 1 — Pool passes `msg.sender` (the router) to `_beforeSwap`:**
`MetricOmmPool.swap` at line 230 calls `_beforeSwap(msg.sender, ...)`. When `MetricOmmSimpleRouter.exactInputSingle` is the caller, `msg.sender` is the router contract.

**Step 2 — `ExtensionCalling._beforeSwap` forwards `sender` unchanged:**
`ExtensionCalling._beforeSwap` (lines 149–177) encodes `sender` directly into the `abi.encodeCall` payload and dispatches it to every configured extension without modification.

**Step 3 — `SwapAllowlistExtension.beforeSwap` checks the wrong address:**
```solidity
if (!allowAllSwappers[msg.sender] && !allowedSwapper[msg.sender][sender]) {
    revert IMetricOmmPoolActions.NotAllowedToSwap();
}
```
Here `msg.sender` is the pool (correct for keying the mapping) and `sender` is the router (wrong — should be the end user). The check `allowedSwapper[pool][router]` passes for any caller as long as the router is allowlisted.

**Step 4 — Router has no mechanism to inject the real user:**
`MetricOmmSimpleRouter.exactInputSingle` (lines 72–80) calls `pool.swap(params.recipient, ...)` with no field carrying `msg.sender` of the router into the pool's `sender` slot. The same applies to `exactInput` (line 104), `exactOutputSingle` (line 136), and `exactOutput` (line 165).

**Existing guards are insufficient:** There is no secondary check in the pool, the extension, or the router that recovers or validates the original user identity before the allowlist decision is made.

## Impact Explanation
**Scenario A (Critical — allowlist fully bypassed):** A pool admin calls `setAllowedToSwap(pool, router, true)` to let permitted users trade through the router. Because `beforeSwap` sees only the router address, every EOA on-chain can call `exactInputSingle` and pass the check. LP assets are exposed to unrestricted swaps at oracle-quoted prices — a direct fund-loss path meeting the "broken core pool functionality causing loss of funds" criterion.

**Scenario B (High — allowlisted users locked out of router):** A pool admin allowlists individual EOAs but not the router. Every router-mediated swap reverts with `NotAllowedToSwap`, making the primary user-facing swap interface unusable for all permitted users — broken core swap functionality.

## Likelihood Explanation
- `MetricOmmSimpleRouter` is the primary user-facing interface; pool admins are expected to support it.
- Allowlisting the router is the only way to enable router-mediated swaps on a restricted pool — a natural and expected admin action.
- No special privilege, flash loan, or unusual token behavior is required; any EOA calling `exactInputSingle` triggers the bypass.
- `SwapAllowlistExtension` is a production periphery contract, not a mock or test artifact.

## Recommendation
The extension must gate on the economically relevant actor. The lowest-friction fix is to check `recipient` instead of `sender` in `beforeSwap`, since `recipient` is set by the end user in `ExactInputSingleParams` and represents the economic beneficiary. Alternatively, the router can ABI-encode `msg.sender` into `extensionData` before forwarding, and the extension can decode and verify it. A third option is adding an explicit `swapper` parameter to `pool.swap()` that the pool validates and passes to extensions.

## Proof of Concept
```
Setup:
  pool configured with SwapAllowlistExtension
  allowedSwapper[pool][router] = true   // admin enables router for permitted users
  allowedSwapper[pool][alice]  = true   // alice is a permitted user
  allowedSwapper[pool][bob]    = false  // bob is NOT permitted

Attack:
  bob calls MetricOmmSimpleRouter.exactInputSingle({
      pool:      pool,
      recipient: bob,
      ...
  })

  // MetricOmmSimpleRouter.exactInputSingle (L72-80):
  IMetricOmmPoolActions(params.pool).swap(params.recipient, ...)
  // msg.sender inside pool = router

  // MetricOmmPool.swap (L230-240):
  _beforeSwap(msg.sender /*= router*/, recipient /*= bob*/, ...)

  // SwapAllowlistExtension.beforeSwap (L37):
  allowedSwapper[pool][router] → true  ← bob bypasses check

Result:
  bob, a non-allowlisted address, executes a swap on a restricted pool.
  LP assets are transferred to bob at oracle-quoted prices.
  The allowlist guard is completely bypassed.
```
