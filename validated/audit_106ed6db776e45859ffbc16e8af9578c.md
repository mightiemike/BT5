### Title
Repeated-Prefix ClassID Bypass in `processReceivedPacket` Allows Unauthorized NFT Unescrow — (`x/nft-transfer/types/trace.go`, `x/nft-transfer/keeper/packet.go`)

---

### Summary

A malicious counterparty chain can craft a packet with a deliberately repeated port/channel prefix in `data.ClassId` (e.g., `nft/channel-0/nft/channel-0/realClass`). This causes `IsAwayFromOrigin` to return `false` (sink/back-to-origin direction) while `RemoveClassPrefix` strips only one prefix level, producing `nft/channel-0/realClass` as the `unprefixedClassID`. The resulting `voucherClassID` (`ibc/HASH(nft/channel-0/realClass)`) maps to a legitimately escrowed NFT belonging to a different user, and `TransferOwner` transfers it to the attacker.

---

### Finding Description

**`IsAwayFromOrigin`** uses a simple `strings.HasPrefix` check: [1](#0-0) 

A classID of `nft/channel-0/nft/channel-0/realClass` starts with `nft/channel-0/`, so `IsAwayFromOrigin("nft", "channel-0", "nft/channel-0/nft/channel-0/realClass")` returns `false` — the sink branch is taken.

**`RemoveClassPrefix`** performs a single, unconditional byte-slice strip: [2](#0-1) 

`"nft/channel-0/nft/channel-0/realClass"[len("nft/channel-0/"):]` = `"nft/channel-0/realClass"` — one prefix level remains.

**`processReceivedPacket`** then computes `voucherClassID` from this residual path and calls `TransferOwner` against the escrow address: [3](#0-2) 

**`OnRecvPacket`** only calls `data.ValidateBasic()` before delegating to `processReceivedPacket`: [4](#0-3) 

**`ValidateBasic`** checks only that classID is non-blank, token arrays are consistent, and addresses are valid bech32 — it performs **no structural validation** of the classID against the channel direction: [5](#0-4) 

There is no guard anywhere in the receive path that rejects a classID whose prefix appears more than once or that verifies the post-strip result is consistent with what was originally escrowed.

---

### Impact Explanation

**Concrete theft scenario:**

1. Chain B receives NFT `realClass` from chain A via dest channel `channel-0` on chain B → minted as `ibc/HASH(nft/channel-0/realClass)`.
2. A legitimate user on chain B sends that NFT to chain C via source channel `channel-2`. `IsAwayFromOrigin("nft", "channel-2", "nft/channel-0/realClass")` = `true`, so the NFT is escrowed in `GetEscrowAddress("nft", "channel-2")`.
3. Attacker controls a chain with source channel `channel-0` connected to chain B's `channel-2`.
4. Attacker sends packet: source=`nft/channel-0`, dest=`nft/channel-2`, `data.ClassId="nft/channel-0/nft/channel-0/realClass"`, `data.TokenIds=[victimTokenID]`, `data.Receiver=attacker`.
5. On chain B: `IsAwayFromOrigin("nft","channel-0","nft/channel-0/nft/channel-0/realClass")` → `false`; `RemoveClassPrefix` → `"nft/channel-0/realClass"`; `voucherClassID` = `ibc/HASH(nft/channel-0/realClass)`; `escrowAddress` = `GetEscrowAddress("nft","channel-2")`.
6. `TransferOwner(ibc/HASH(nft/channel-0/realClass), victimTokenID, escrowAddress, attacker)` succeeds — attacker receives the victim's NFT.

The escrow address used in `processReceivedPacket` is `GetEscrowAddress(packet.GetDestPort(), packet.GetDestChannel())`: [6](#0-5) 

This is the same address used when chain B escrowed the NFT in step 2, so the NFT is present and the transfer succeeds.

---

### Likelihood Explanation

Requires the attacker to operate or compromise a counterparty chain (or a chain that has an IBC channel to chain B). This is a realistic threat model for IBC: any chain that has established a channel with chain B can send arbitrary packet data. The tokenID of an escrowed NFT is public on-chain state. The attack is deterministic and requires no brute force.

---

### Recommendation

1. **Validate classID direction consistency in `OnRecvPacket`:** In the sink branch, assert that `data.ClassId` starts with exactly one occurrence of the source prefix and that the remainder does **not** start with the same prefix again.
2. **Strengthen `RemoveClassPrefix`:** After stripping, verify the result does not still start with the same `portID/channelID/` prefix, and return an error if it does.
3. **Add classID structural validation to `ValidateBasic`:** Reject classIDs where the same `port/channel/` segment appears consecutively.

---

### Proof of Concept

```
// Precondition on chain B:
// - NFT classID="ibc/HASH(nft/channel-0/realClass)", tokenID="token1"
//   is held by GetEscrowAddress("nft", "channel-2")
//   (escrowed when a user sent it to chain C via channel-2)

// Attacker (controls chain with channel-0 → chain B's channel-2) sends:
packet := {
    SourcePort:    "nft",
    SourceChannel: "channel-0",
    DestPort:      "nft",
    DestChannel:   "channel-2",
    Data: NonFungibleTokenPacketData{
        ClassId:   "nft/channel-0/nft/channel-0/realClass",
        TokenIds:  ["token1"],
        TokenUris: [""],
        Sender:    attacker,
        Receiver:  attacker,
    },
}

// On chain B, processReceivedPacket:
// IsAwayFromOrigin("nft","channel-0","nft/channel-0/nft/channel-0/realClass") = false
// unprefixedClassID = RemoveClassPrefix(...) = "nft/channel-0/realClass"
// voucherClassID = ibc/HASH("nft/channel-0/realClass")
// escrowAddress = GetEscrowAddress("nft","channel-2")
// TransferOwner(voucherClassID, "token1", escrowAddress, attacker) → SUCCESS
// Attacker now owns token1 that belonged to the legitimate user.
```

```go
func (nftpd NonFungibleTokenPacketData) ValidateBasic() error {
	if strings.TrimSpace(nftpd.ClassId) == "" {
		return newsdkerrors.Wrap(ErrInvalidClassID, "classId cannot be blank")
	}

	if len(nftpd.TokenIds) == 0 {
		return newsdkerrors.Wrap(ErrInvalidTokenID, "tokenId cannot be blank")
	}

	if len(nftpd.TokenIds) != len(nftpd.TokenUris) {
		return newsdkerrors.Wrap(ErrInvalidPacket, "tokenIds and tokenUris lengths do not match")
	}

	if strings.TrimSpace(nftpd.Sender) == "" {
		return newsdkerrors.Wrap(sdkerrors.ErrInvalidAddress, "sender address cannot be blank")
	}

	// decode the sender address
	if _, err := sdk.AccAddressFromBech32(nftpd.Sender); err != nil {
		return newsdkerrors.Wrap(sdkerrors.ErrInvalidAddress, "invalid sender address")
	}

	if strings.TrimSpace(nftpd.Receiver) == "" {
		return newsdkerrors.Wrap(sdkerrors.ErrInvalidAddress, "receiver address cannot be blank")
	}

	// decode the receiver address
	if _, err := sdk.AccAddressFromBech32(nftpd.Receiver); err != nil {
		return newsdkerrors.Wrap(sdkerrors.ErrInvalidAddress, "invalid receiver address")
	}
	return nil
}
```
