### Title
Stateless Gateway Validator Unconditionally Applies `min_gas_price` Check to `l2_gas.max_price_per_unit` Even When L2 Gas Is Zero, Rejecting Valid Transactions - (File: `crates/apollo_gateway/src/stateless_transaction_validator.rs`)

---

### Summary

`StatelessTransactionValidator::validate_resource_bounds` applies the `min_gas_price` threshold check to `l2_gas.max_price_per_unit` unconditionally, even when a transaction legitimately sets L2 gas to zero (using only L1 or L1-data gas bounds). In production, `min_gas_price` defaults to `8_000_000_000`, so any transaction with `l2_gas.max_price_per_unit = 0` is rejected with `MaxGasPriceTooLow`, even though the protocol explicitly permits transactions with only L1 gas bounds set.

---

### Finding Description

In `crates/apollo_gateway/src/stateless_transaction_validator.rs`, `validate_resource_bounds` performs two sequential checks:

```rust
// Line 66: Correctly allows transactions with any non-zero resource bound
if ValidResourceBounds::AllResources(resource_bounds).max_possible_fee(Tip::ZERO) == Fee(0) {
    return Err(StatelessTransactionValidatorError::ZeroResourceBounds { resource_bounds });
}

// Line 71: Unconditionally checks l2_gas price — even when l2_gas is zero
if resource_bounds.l2_gas.max_price_per_unit.0 < self.config.min_gas_price {
    return Err(StatelessTransactionValidatorError::MaxGasPriceTooLow { ... });
}
```

The first check correctly passes a transaction with only `l1_gas` set (non-zero fee). But the second check then unconditionally compares `l2_gas.max_price_per_unit` (which is `0` when L2 gas is not used) against `min_gas_price` (production default: `8_000_000_000`). Since `0 < 8_000_000_000`, the transaction is rejected.

The test suite confirms this is a valid transaction type — `valid_l1_gas` explicitly tests a transaction with only L1 gas set and expects `Ok(())`:

```rust
#[case::valid_l1_gas(
    StatelessTransactionValidatorConfig {
        validate_resource_bounds: true,
        ..*DEFAULT_VALIDATOR_CONFIG_FOR_TESTING  // min_gas_price: 0
    },
    RpcTransactionArgs {
        resource_bounds: AllResourceBounds {
            l1_gas: NON_EMPTY_RESOURCE_BOUNDS,
            ..Default::default()  // l2_gas = 0
        },
        ..Default::default()
    }
)]
```

This test only passes because `DEFAULT_VALIDATOR_CONFIG_FOR_TESTING` sets `min_gas_price: 0`. In production, `StatelessTransactionValidatorConfig::default()` sets `min_gas_price: 8_000_000_000`, causing the same transaction to be rejected.

The stateful validator correctly handles this asymmetry — `validate_tx_l2_gas_price_within_threshold` explicitly skips the L2 gas price check for `ValidResourceBounds::L1Gas(_)`:

```rust
ValidResourceBounds::L1Gas(_) => {
    // No validation required for legacy transactions.
}
```

But the stateless validator has no such guard.

---

### Impact Explanation

**High. Mempool/gateway/RPC admission rejects valid transactions before sequencing.**

Any user submitting a transaction with only L1 gas bounds set (a valid protocol configuration) will have their transaction rejected at the gateway stateless validation stage with a misleading `MaxGasPriceTooLow` error for L2 gas, even though L2 gas is not being used. This is a liveness/admission failure: valid, correctly-formed transactions are permanently blocked from entering the mempool.

---

### Likelihood Explanation

The production default `min_gas_price = 8_000_000_000` is always active. Any transaction with `AllResourceBounds` where `l2_gas` is zero (only L1 or L1-data gas used) triggers this path. The test suite masks the bug by using `min_gas_price: 0` in test configs. The condition is deterministic and reproducible by any unprivileged user.

---

### Recommendation

Guard the `min_gas_price` check so it only applies when `l2_gas` is actually being used (i.e., `l2_gas.max_amount > 0` or `l2_gas.max_price_per_unit > 0`):

```rust
if resource_bounds.l2_gas.max_amount.0 > 0
    && resource_bounds.l2_gas.max_price_per_unit.0 < self.config.min_gas_price
{
    return Err(StatelessTransactionValidatorError::MaxGasPriceTooLow { ... });
}
```

This mirrors the pattern used in the stateful validator, which skips the L2 gas price threshold check entirely when L2 gas is not part of the transaction's resource bounds.

---

### Proof of Concept

1. Submit an `RpcInvokeTransactionV3` with:
   ```
   resource_bounds = AllResourceBounds {
       l1_gas: ResourceBounds { max_amount: 1000, max_price_per_unit: 1_000_000_000 },
       l2_gas: ResourceBounds { max_amount: 0, max_price_per_unit: 0 },
       l1_data_gas: ResourceBounds { max_amount: 0, max_price_per_unit: 0 },
   }
   ```
2. The gateway calls `StatelessTransactionValidator::validate`.
3. `validate_resource_bounds` is called. Line 66 check passes (fee = `1000 * 1_000_000_000 > 0`).
4. Line 71 check: `0 < 8_000_000_000` → `true` → returns `Err(MaxGasPriceTooLow { gas_price: 0, min_gas_price: 8_000_000_000 })`.
5. The transaction is rejected despite being valid per the protocol.

    fn validate_resource_bounds(
        &self,
        tx: &RpcTransaction,
    ) -> StatelessTransactionValidatorResult<()> {
        if !self.config.validate_resource_bounds {
            return Ok(());
        }

        let resource_bounds = *tx.resource_bounds();
        // The resource bounds should be positive even without the tip.
        if ValidResourceBounds::AllResources(resource_bounds).max_possible_fee(Tip::ZERO) == Fee(0)
        {
            return Err(StatelessTransactionValidatorError::ZeroResourceBounds { resource_bounds });
        }

        if resource_bounds.l2_gas.max_price_per_unit.0 < self.config.min_gas_price {
            return Err(StatelessTransactionValidatorError::MaxGasPriceTooLow {
                gas_price: resource_bounds.l2_gas.max_price_per_unit,
                min_gas_price: self.config.min_gas_price,
            });
        }

        // TODO(Arni): Consider adding a validation for max_l2_gas_amount for declare.
        if let RpcTransaction::Declare(_) = tx {
        } else if resource_bounds.l2_gas.max_amount.0 > self.config.max_l2_gas_amount {
            return Err(StatelessTransactionValidatorError::MaxGasAmountTooHigh {
                gas_amount: resource_bounds.l2_gas.max_amount,
                max_gas_amount: self.config.max_l2_gas_amount,
            });
        }

        Ok(())
    }
