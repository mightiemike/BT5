### Title
AuxPoW Parent Block PoW Verified Against Child Chain Difficulty Instead of Parent Chain Difficulty - (`contract/src/dogecoin.rs`)

### Summary

In `check_aux`, the PoW hash of the AuxPoW parent block is validated against the **Dogecoin block's** difficulty target (`block_header.bits`) instead of the **parent block's own** difficulty target (`aux_data.parent_block.bits`). This is a direct analog to the external report's "wrong identifier used for wrapped vs. original asset" class: the code uses the wrong `bits` field when crossing the child-chain / parent-chain boundary, allowing an attacker to forge AuxPoW proofs with only Dogecoin-level computational work.

### Finding Description

In `check_aux` in `contract/src/dogecoin.rs`, the final PoW check reads:

```rust
let pow_hash = aux_data.parent_block.block_hash_pow();
require!(
    self.skip_pow_verification
        || U256::from_le_bytes(&pow_hash.0) <= target_from_bits(block_header.bits),
    ...
);
``` [1](#0-0) 

`block_header` is the **Dogecoin** block being submitted. `aux_data.parent_block` is the **Bitcoin/parent-chain** block that contains the merged-mining commitment. The check should verify that the parent block's PoW hash meets the **parent block's own difficulty target** (`aux_data.parent_block.bits`), not Dogecoin's difficulty target.

The reference Dogecoin implementation (`auxpow.cpp`) performs:
```cpp
CheckProofOfWork(parentBlock.GetHash(), parentBlock.nBits, params)
```
i.e., it uses `parentBlock.nBits` — the parent chain's own bits field.

The `AuxData` struct carries the full parent block header, which includes its own `bits` field: [2](#0-1) 

The `Header` type (used for `parent_block`) has a `bits` field: [3](#0-2) 

So `aux_data.parent_block.bits` is available but is never used in the PoW check.

### Impact Explanation

Dogecoin's mainnet `proof_of_work_limit_bits` is `0x1e0fffff`: [4](#0-3) 

Bitcoin's `proof_of_work_limit_bits` is `0x1d00ffff`, and its live difficulty is orders of magnitude higher than Dogecoin's. Because the check uses Dogecoin's difficulty (`block_header.bits`) instead of Bitcoin's difficulty (`aux_data.parent_block.bits`), an attacker only needs to find a parent block header whose scrypt hash meets Dogecoin's (much lower) difficulty. This is computationally trivial compared to meeting Bitcoin's actual difficulty.

A successful exploit injects a fraudulent Dogecoin block header into the light client's `headers_pool` and potentially into the canonical mainchain. Once accepted, the attacker can call `verify_transaction_inclusion` or `verify_transaction_inclusion_v2` with a fabricated Merkle proof against the fraudulent block, causing the contract to return `true` for a transaction that was never included in any real Dogecoin block. [5](#0-4) 

### Likelihood Explanation

The `submit_blocks` public entry point accepts `(Header, Option<AuxData>)` tuples from any caller (subject to the trusted-relayer check, but the relayer role can be held by any staked account): [6](#0-5) 

The `dogecoin` feature enables `btc-types/scrypt_hash`: [7](#0-6) 

So `block_hash_pow()` uses scrypt for the parent block hash computation. An attacker needs to mine a scrypt hash below Dogecoin's current difficulty — a task achievable with commodity GPU hardware in seconds to minutes. All other components of the AuxPoW (coinbase transaction embedding the Dogecoin block hash, chain merkle proof, transaction merkle proof) are constructed deterministically with no computational cost.

### Recommendation

Replace `block_header.bits` with `aux_data.parent_block.bits` in the PoW check:

```rust
let pow_hash = aux_data.parent_block.block_hash_pow();
require!(
    self.skip_pow_verification
        || U256::from_le_bytes(&pow_hash.0) <= target_from_bits(aux_data.parent_block.bits),
    "block should have correct pow"
);
```

This mirrors the reference Dogecoin implementation, which validates the parent block's hash against the parent block's own `nBits`.

### Proof of Concept

1. Obtain the current Dogecoin mainchain tip and compute the expected `bits` for the next block (this passes `check_pow`).
2. Construct a fake Dogecoin `Header` with valid `prev_block_hash`, `version` (AuxPoW flag set), and the expected `bits`.
3. Construct a fake Bitcoin-format `parent_block` header with `bits = 0x1e0fffff` (Dogecoin's minimum difficulty, far easier than Bitcoin's real difficulty). Mine its scrypt hash until it satisfies `hash <= target_from_bits(dogecoin_block.bits)` — this takes seconds on a GPU.
4. Build a coinbase transaction whose `script_sig` embeds the Dogecoin block hash (preceded by `fabe6d6d`) and a valid `chain_merkle_proof` of length 0 (single-entry tree).
5. Set `aux_data.chain_id = get_expected_index(nonce, chain_id, 0)` to pass the index check.
6. Call `submit_blocks` with `(fake_dogecoin_header, Some(aux_data))`.
7. The contract accepts the block because `parent_block.block_hash_pow() <= target_from_bits(dogecoin_block.bits)` passes (both use Dogecoin's easy difficulty), even though the parent block would never satisfy Bitcoin's real difficulty.
8. Call `verify_transaction_inclusion` with a fabricated `tx_id` and Merkle proof against the accepted fraudulent block — the contract returns `true`. [8](#0-7)

    pub(crate) fn check_aux(&mut self, block_header: &Header, aux_data: &AuxData) {
        // The Dogecoin block must have the AuxPoW flag set (bit 8) when AuxPoW data is present.
        // https://github.com/dogecoin/dogecoin/blob/master/src/auxpow.h
        const BLOCK_VERSION_AUXPOW: i32 = 0x100;

        require!(
            aux_data.chain_merkle_proof.len() <= 30,
            "Aux POW chain merkle branch too long"
        );
        require!(
            block_header.version & BLOCK_VERSION_AUXPOW != 0,
            "Aux POW block does not have AuxPoW flag set in version"
        );

        let chain_id = self.get_config().aux_chain_id;

        require!(
            chain_id == block_header.get_chain_id(),
            format!(
                "block does not have our chain ID (got {}, expected {chain_id})",
                block_header.get_chain_id()
            )
        );

        require!(
            chain_id != aux_data.parent_block.get_chain_id(),
            "Aux POW parent has our chain ID"
        );

        let chain_root = merkle_tools::compute_root_from_merkle_proof(
            block_header.block_hash(),
            aux_data.chain_id,
            &aux_data.chain_merkle_proof,
        );

        let coinbase_tx = aux_data.get_coinbase_tx();
        let coinbase_tx_hash = coinbase_tx.compute_txid();

        require!(
            merkle_tools::compute_root_from_merkle_proof(
                H256::from(coinbase_tx_hash.to_raw_hash().to_byte_array()),
                0,
                &aux_data.merkle_proof,
            ) == aux_data.parent_block.merkle_root
        );

        let script_sig = coinbase_tx
            .input
            .first()
            .unwrap()
            .script_sig
            .to_hex_string();
        let pos_merged_mining_header = script_sig.find(MERGED_MINING_HEADER);
        let mut pos_chain_root = script_sig
            .find(&chain_root.to_string())
            .expect("Aux POW missing chain merkle root in parent coinbase");

        match pos_merged_mining_header {
            Some(pos_merged_mining_header) => {
                if script_sig[pos_merged_mining_header + MERGED_MINING_HEADER.len()..]
                    .contains(MERGED_MINING_HEADER)
                {
                    env::panic_str("Multiple merged mining headers in coinbase");
                }

                require!(
                    pos_merged_mining_header + MERGED_MINING_HEADER.len() == pos_chain_root,
                    "Merged mining header is not just before chain merkle root"
                );
            }
            None => {
                require!(pos_chain_root <= 40, "Aux POW chain merkle root must start in the first 20 bytes of the parent coinbase");
            }
        }

        pos_chain_root += chain_root.to_string().len();
        require!(
            script_sig.len() - pos_chain_root >= 16,
            "Aux POW missing chain merkle tree size and nonce in parent coinbase"
        );

        let bytes = hex::decode(&script_sig[pos_chain_root..pos_chain_root + 8]).unwrap();
        let n_size = u32::from_le_bytes([bytes[0], bytes[1], bytes[2], bytes[3]]);
        require!(
            n_size == (1u32 << aux_data.chain_merkle_proof.len()),
            "Aux POW merkle branch size does not match parent coinbase"
        );

        let bytes = hex::decode(&script_sig[pos_chain_root + 8..pos_chain_root + 16]).unwrap();
        let n_nonce = u32::from_le_bytes([bytes[0], bytes[1], bytes[2], bytes[3]]);

        let chain_id = block_header.get_chain_id();

        let expected_index =
            Self::get_expected_index(n_nonce, chain_id, aux_data.chain_merkle_proof.len());

        require!(
            u32::try_from(aux_data.chain_id).ok() == Some(expected_index),
            "Aux POW wrong index"
        );
        let pow_hash = aux_data.parent_block.block_hash_pow();
        require!(
            self.skip_pow_verification
                || U256::from_le_bytes(&pow_hash.0) <= target_from_bits(block_header.bits),
            format!("block should have correct pow")
        );
    }

    pub fn verify_transaction_inclusion(&self, #[serializer(borsh)] args: ProofArgs) -> bool {
        require!(
            args.confirmations <= self.gc_threshold,
            "The required number of confirmations exceeds the number of blocks stored in memory"
        );

        let heaviest_block_header = self
            .headers_pool
            .get(&self.mainchain_tip_blockhash)
            .unwrap_or_else(|| env::panic_str(ERR_KEY_NOT_EXIST));
        let target_block_height = self
            .mainchain_header_to_height
            .get(&args.tx_block_blockhash)
            .unwrap_or_else(|| env::panic_str("block does not belong to the current main chain"));

        // Check requested confirmations. No need to compute proof if insufficient confirmations.
        require!(
            (heaviest_block_header.block_height).saturating_sub(target_block_height) + 1
                >= args.confirmations,
            "Not enough blocks confirmed"
        );

        let header = self
            .headers_pool
            .get(&args.tx_block_blockhash)
            .unwrap_or_else(|| env::panic_str("cannot find requested transaction block"));

        require!(!args.merkle_proof.is_empty(), "Merkle proof is empty");

        // compute merkle tree root and check if it matches block's original merkle tree root
        merkle_tools::compute_root_from_merkle_proof(
            args.tx_id,
            usize::try_from(args.tx_index).unwrap(),
            &args.merkle_proof,
        ) == header.block_header.merkle_root
    }
