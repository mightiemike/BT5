### Title
NFT Permanently Locked in Module Account via Unchecked `MsgTransferNFT` Recipient — (`x/nft/keeper/keeper.go`, `x/nft/types/msgs.go`)

---

### Summary

`MsgTransferNFT.ValidateBasic()` only validates that `msg.Recipient` is a syntactically valid bech32 address. Neither the message validation layer nor the keeper's `TransferOwner` function checks whether the recipient is a blocked module account. The `Keeper` struct holds no `AccountKeeper` reference capable of calling `BlockedAddr`. As a result, any NFT owner can irrevocably transfer their NFT to a module account address. Because module accounts have no private key, no subsequent `MsgTransferNFT` or `MsgBurnNFT` can ever be signed on their behalf, permanently locking the NFT while the supply counter remains inflated.

---

### Finding Description

**Entrypoint:** `MsgTransferNFT` submitted by any NFT owner.

**`ValidateBasic` — no blocked-address guard:** [1](#0-0) 

The only recipient check is `sdk.AccAddressFromBech32(msg.Recipient)` — a pure syntactic check. Module account addresses are valid bech32 addresses and pass this check unconditionally.

**`msgServer.TransferNFT` — no blocked-address guard:** [2](#0-1) 

After decoding the recipient, the server calls `TransferOwner` directly with no interstitial check.

**`Keeper.TransferOwner` — no blocked-address guard:** [3](#0-2) 

The function only verifies the denom exists and that `srcOwner` is the current owner, then unconditionally writes `dstOwner` as the new owner and calls `swapOwner`. There is no call to any `BlockedAddr`-style predicate.

**`Keeper` struct has no `AccountKeeper`:** [4](#0-3) 

The keeper holds only `storeKey` and `cdc`. The `expected_keepers.go` defines `AccountKeeper` and `BankKeeper` interfaces, but neither is injected into the `Keeper` struct, so no blocked-address check is structurally possible anywhere in the transfer path. [5](#0-4) 

**`BurnNFT` cannot recover the NFT:** [6](#0-5) 

`BurnNFT` requires the caller to pass both `IsOwner` (owner must match) and `IsDenomCreator` (owner must be the denom creator). A module account satisfies neither condition as a signer, and it cannot sign any transaction at all.

---

### Impact Explanation

- The NFT's `Owner` field and the owner-index KV store entry are permanently set to the module account address.
- The supply counter (`increaseSupply`/`decreaseSupply`) is never decremented — the NFT remains counted in the collection supply but is unrecoverable.
- No on-chain path exists to transfer or burn the NFT out of the module account: `TransferOwner` requires the sender to be the current owner (the module account), and `BurnNFT` additionally requires the owner to be the denom creator — neither condition can be satisfied by a module account signer.
- This corrupts the invariant that every NFT in the supply is owned by an address capable of signing transfer or burn messages.

---

### Likelihood Explanation

The exploit requires only a standard `MsgTransferNFT` transaction from any NFT owner. The recipient address of any registered module account (e.g., the `nft` module account, `bank`, `distribution`, etc.) is publicly derivable from the module name. No special privileges, governance, or operator access are needed. The path is fully reachable from a CLI-signed transaction.

---

### Recommendation

Inject an `AccountKeeper` (or a `BlockedAddrs map[string]bool`) into the `Keeper` struct and call a blocked-address check inside `TransferOwner` before writing the new owner:

```go
// In keeper.go TransferOwner:
if k.accountKeeper.IsModuleAccount(ctx, dstOwner) {
    return sdkerrors.Wrapf(types.ErrUnauthorized, "recipient %s is a module account", dstOwner)
}
```

Alternatively, mirror the Cosmos SDK bank module pattern and pass a `blockedAddrs map[string]bool` at keeper construction time, checking `blockedAddrs[dstOwner.String()]` before the state write. Apply the same guard to `MintNFT`/`MintNFTUnverified` for the `owner` parameter.

---

### Proof of Concept

```go
// Integration test sketch (unmodified Go/Cosmos test setup)
func TestTransferNFTToModuleAccount(t *testing.T) {
    // Setup: issue denom, mint NFT to attacker address
    // ...

    // Derive the nft module account address
    nftModuleAddr := authtypes.NewModuleAddress(nfttypes.ModuleName)

    // Submit MsgTransferNFT with Recipient = nft module account
    msg := nfttypes.NewMsgTransferNFT(tokenID, denomID, attacker.String(), nftModuleAddr.String())
    _, err := msgServer.TransferNFT(ctx, msg)
    require.NoError(t, err) // transfer succeeds

    // NFT owner is now the module account
    nft, err := keeper.GetNFT(ctx, denomID, tokenID)
    require.NoError(t, err)
    require.Equal(t, nftModuleAddr.String(), nft.GetOwner().String())

    // Supply is NOT decremented — NFT is counted but unrecoverable
    supply := keeper.GetTotalSupply(ctx, denomID)
    require.Equal(t, uint64(1), supply)

    // No recovery: BurnNFT fails (module account cannot sign)
    burnMsg := nfttypes.NewMsgBurnNFT(nftModuleAddr.String(), tokenID, denomID)
    _, err = msgServer.BurnNFT(ctx, burnMsg) // would fail: not denom creator, and unsignable
    require.Error(t, err)
}
```

func (k Keeper) TransferOwner(
	ctx sdk.Context, denomID, tokenID string, srcOwner, dstOwner sdk.AccAddress,
) error {
	if !k.HasDenomID(ctx, denomID) {
		return sdkerrors.Wrapf(types.ErrInvalidDenom, "denom ID %s not exists", denomID)
	}

	nft, err := k.IsOwner(ctx, denomID, tokenID, srcOwner)
	if err != nil {
		return err
	}

	nft.Owner = dstOwner.String()

	k.setNFT(ctx, denomID, nft)
	k.swapOwner(ctx, denomID, tokenID, srcOwner, dstOwner)
	return nil
}

func (k Keeper) BurnNFT(ctx sdk.Context, denomID, tokenID string, owner sdk.AccAddress) error {
	if !k.HasDenomID(ctx, denomID) {
		return sdkerrors.Wrapf(types.ErrInvalidDenom, "denom ID %s not exists", denomID)
	}

	nft, err := k.IsOwner(ctx, denomID, tokenID, owner)
	if err != nil {
		return err
	}

	_, err = k.IsDenomCreator(ctx, denomID, owner)
	if err != nil {
		return err
	}

	k.deleteNFT(ctx, denomID, nft)
	k.deleteOwner(ctx, denomID, tokenID, owner)
	k.decreaseSupply(ctx, denomID)

	return nil
}
