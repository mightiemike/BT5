### Title
Insufficient Bonus Pool Permanently Blocks Principal Withdrawal via `MsgTierUndelegate` and `MsgExitTierWithDelegation` - (File: `x/tieredrewards/keeper/msg_server.go`)

---

### Summary

`MsgTierUndelegate` and `MsgExitTierWithDelegation` both call `claimRewards()` as a mandatory step before performing the actual unstaking/exit. `claimRewards()` calls `processEventsAndClaimBonus()`, which calls `sufficientBonusPoolBalance()`. If the bonus rewards pool holds less than the accrued bonus, the entire transaction reverts with `ErrInsufficientBonusPool`. This means any user with a delegated position that has accrued non-zero bonus rewards **cannot undelegate or exit their principal** until the pool is externally replenished — a condition entirely outside the user's control.

---

### Finding Description

In `MsgTierUndelegate`, the handler unconditionally calls `claimRewards` before performing the staking unbond:

```go
pos, _, _, err = ms.claimRewards(ctx, pos)
if err != nil {
    return nil, err
}
``` [1](#0-0) 

Similarly, `MsgExitTierWithDelegation` calls `claimRewards` before transferring the delegation back to the owner:

```go
pos, _, _, err = ms.claimRewards(ctx, pos)
if err != nil {
    return nil, err
}
``` [2](#0-1) 

`claimRewards` calls `processEventsAndClaimBonus`, which computes accrued bonus and then checks pool solvency:

```go
if err := k.sufficientBonusPoolBalance(ctx, bonusCoins); err != nil {
    return nil, err
}
``` [3](#0-2) 

`sufficientBonusPoolBalance` returns `ErrInsufficientBonusPool` if the pool cannot cover the accrued bonus:

```go
if !poolBalance.IsAllGTE(bonus) {
    return errorsmod.Wrapf(types.ErrInsufficientBonusPool, ...)
}
``` [4](#0-3) 

The ADR explicitly documents this as the intended behavior for user-driven paths: *"Pool empty (user-driven) | Message fails atomically. No state change. Retry after pool replenished."* [5](#0-4) 

However, this design breaks the invariant that users must always be able to exit and recover their principal after the exit commitment elapses. The pool is funded externally (governance/community) and is also drained by the BeginBlocker for base-rewards top-up. If the pool runs dry — a realistic operational scenario — every user with accrued bonus rewards is blocked from undelegating or exiting with delegation, trapping their principal indefinitely.

Note that `MsgWithdrawFromTier` (the final principal-recovery step) does **not** call `claimRewards` and would succeed on its own, but it requires the position to already be undelegated. The blocking occurs at the mandatory prior step: `MsgTierUndelegate`. [6](#0-5) 

---

### Impact Explanation

A user who has locked tokens in a tier position, triggered exit, and waited for the exit commitment to elapse **cannot recover their principal** if the bonus pool is empty at the time they attempt to undelegate or exit with delegation. Their tokens remain locked in the module's per-position delegator address indefinitely. The user has no recourse: they cannot skip the reward claim, they cannot forfeit the bonus to proceed, and they cannot call `MsgWithdrawFromTier` directly because the position is still delegated.

The corrupted invariant is: **user's principal balance is inaccessible** despite the exit commitment having fully elapsed.

---

### Likelihood Explanation

The bonus pool is a module account funded by external actors (governance, community). The BeginBlocker also drains it for base-rewards top-up when the fee collector is insufficient. [7](#0-6) 

A pool depletion event is realistic: the pool can be drained by the BeginBlocker over time, or simply never replenished after initial funding. Any user whose position has accrued non-zero bonus (i.e., any delegated position on a bonded validator after any time has passed) is affected. This is the normal operating state for all active positions.

---

### Recommendation

Separate the reward-claim step from the exit/undelegate step. Provide a path for users to undelegate or exit with delegation while **forgoing** uncollectable bonus rewards when the pool is insufficient. Concretely:

- In `TierUndelegate` and `ExitTierWithDelegation`, if `claimRewards` fails with `ErrInsufficientBonusPool`, allow the operation to proceed by advancing the position's bonus checkpoints (updating `LastBonusAccrual`, `LastEventSeq`, `LastKnownBonded`) without transferring any bonus coins, so the user can exit at the cost of forfeiting the unpayable bonus.
- Alternatively, add a dedicated `EmergencyUndelegate` message that skips bonus claiming entirely, mirroring the recommendation in the referenced report.

---

### Proof of Concept

1. Alice locks tokens in a tier position on a bonded validator. Time passes; bonus accrues.
2. Alice calls `MsgTriggerExitFromTier`. She waits for the exit commitment (e.g., 1 year) to elapse.
3. The bonus pool is drained to zero (by BeginBlocker top-ups or simply not replenished).
4. Alice calls `MsgTierUndelegate`. The call stack is:
   - `TierUndelegate` → `claimRewards` → `processEventsAndClaimBonus` → `sufficientBonusPoolBalance` → returns `ErrInsufficientBonusPool`.
   - Transaction reverts. Alice's delegation is unchanged.
5. Alice calls `MsgExitTierWithDelegation`. Same failure path at line 532.
6. Alice cannot call `MsgWithdrawFromTier` because the position is still delegated (`ErrPositionDelegated`).
7. Alice's principal is permanently locked until an external actor replenishes the pool. [8](#0-7) [9](#0-8) [10](#0-9) [11](#0-10)

func (ms msgServer) TierUndelegate(ctx context.Context, msg *types.MsgTierUndelegate) (*types.MsgTierUndelegateResponse, error) {
	if err := msg.Validate(); err != nil {
		return nil, err
	}

	pos, err := ms.getPositionState(ctx, msg.PositionId)
	if err != nil {
		return nil, err
	}

	if err := ms.validateUndelegatePosition(ctx, pos, msg.Owner); err != nil {
		return nil, err
	}

	pos, _, _, err = ms.claimRewards(ctx, pos)
	if err != nil {
		return nil, err
	}

	srcValidator := pos.Delegation.ValidatorAddress
	valAddr, err := sdk.ValAddressFromBech32(srcValidator)
	if err != nil {
		return nil, err
	}

	delAddr, err := sdk.AccAddressFromBech32(pos.DelegatorAddress)
	if err != nil {
		return nil, errorsmod.Wrap(sdkerrors.ErrInvalidAddress, "invalid delegator address")
	}

	completionTime, _, err := ms.undelegate(ctx, delAddr, valAddr, pos.Delegation.Shares)
	if err != nil {
		return nil, err
	}

	pos.ClearBonusCheckpoints()

	if err := ms.setPosition(ctx, pos.Position, &ValidatorTransition{PreviousAddress: srcValidator}); err != nil {
		return nil, err
	}

	sdkCtx := sdk.UnwrapSDKContext(ctx)
	if err := sdkCtx.EventManager().EmitTypedEvent(&types.EventPositionUndelegated{
		PositionId:     pos.Id,
		TierId:         pos.TierId,
		Owner:          pos.Owner,
		Validator:      srcValidator,
		CompletionTime: completionTime,
	}); err != nil {
		return nil, err
	}

	return &types.MsgTierUndelegateResponse{
		CompletionTime: completionTime,
		PositionId:     pos.Id,
	}, nil
}

func (ms msgServer) WithdrawFromTier(ctx context.Context, msg *types.MsgWithdrawFromTier) (*types.MsgWithdrawFromTierResponse, error) {
	if err := msg.Validate(); err != nil {
		return nil, err
	}

	pos, err := ms.getPositionState(ctx, msg.PositionId)
	if err != nil {
		return nil, err
	}

	if err := ms.validateWithdrawFromTier(ctx, pos, msg.Owner); err != nil {
		return nil, err
	}

	ownerAddr, err := sdk.AccAddressFromBech32(pos.Owner)
	if err != nil {
		return nil, errorsmod.Wrap(sdkerrors.ErrInvalidAddress, "invalid owner address")
	}

	delAddr, err := sdk.AccAddressFromBech32(pos.DelegatorAddress)
	if err != nil {
		return nil, errorsmod.Wrap(sdkerrors.ErrInvalidAddress, "invalid delegator address")
	}

	balances := ms.bankKeeper.SpendableCoins(ctx, delAddr)
	if !balances.IsZero() {
		if err := ms.bankKeeper.SendCoins(ctx, delAddr, ownerAddr, balances); err != nil {
			return nil, err
		}
	}

	if err := ms.deletePosition(ctx, pos.Position, nil); err != nil {
		return nil, err
	}

	sdkCtx := sdk.UnwrapSDKContext(ctx)
	if err := sdkCtx.EventManager().EmitTypedEvent(&types.EventPositionWithdrawn{
		Position: pos.Position,
		Amount:   balances,
	}); err != nil {
		return nil, err
	}

	return &types.MsgWithdrawFromTierResponse{
		Amount: balances,
	}, nil
}

func (k Keeper) claimRewards(ctx context.Context, pos types.PositionState) (types.PositionState, sdk.Coins, sdk.Coins, error) {
	if !pos.IsDelegated() {
		return pos, sdk.NewCoins(), sdk.NewCoins(), nil
	}

	base, err := k.claimBaseRewards(ctx, pos)
	if err != nil {
		return types.PositionState{}, nil, nil, err
	}

	bonus, err := k.processEventsAndClaimBonus(ctx, &pos)
	if err != nil {
		return types.PositionState{}, nil, nil, err
	}

	return pos, base, bonus, nil
}
