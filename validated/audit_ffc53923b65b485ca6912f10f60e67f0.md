### Title
`StateReaderAndContractManager::new` Silently Hardcodes `NativeClassesWhitelist::All`, Bypassing the Configured Native Compiler Whitelist in Production Execution Paths - (File: `crates/blockifier/src/state/state_reader_and_contract_manager.rs`)

### Summary

`StateReaderAndContractManager::new` unconditionally sets `native_classes_whitelist` to `NativeClassesWhitelist::All`, ignoring the operator-configured whitelist. Production callers — `native_blockifier/src/py_block_executor.rs` and `starknet_transaction_prover/src/running/runner.rs` — use this constructor instead of `new_with_native_classes_whitelist`. As a result, every Cairo 1 class that has been compiled to native is executed as native in those paths, even when the deployment config restricts native execution to a specific set of class hashes. This is the direct sequencer analog of the Ion Protocol bug: the whitelist check is performed against the wrong identity (a hardcoded `All` instead of the caller-supplied configured value).

### Finding Description

`StateReaderAndContractManager` exposes two constructors:

```rust
// crates/blockifier/src/state/state_reader_and_contract_manager.rs:42-53
pub fn new(...) -> Self {
    Self::new_with_native_classes_whitelist(
        state_reader,
        contract_class_manager,
        NativeClassesWhitelist::All,   // ← hardcoded, ignores config
        class_cache_metrics,
    )
}
``` [1](#0-0) 

The correct constructor, `new_with_native_classes_whitelist`, accepts the configured value and passes it through to `NativeClassManager::get_runnable`:

```rust
// crates/blockifier/src/state/native_class_manager.rs:141-146
let cached_class = match cached_class {
    CompiledClasses::V1Native(CachedCairoNative::Compiled(native))
        if !native_classes_whitelist.contains(class_hash) =>
    {
        CompiledClasses::into_non_native_class(native)  // downgrade to CASM

``` [2](#0-1) 

When `NativeClassesWhitelist::All` is used, `contains` always returns `true`, so the downgrade branch is never taken and every cached native class is returned as `RunnableCompiledClass::V1Native`.

The batcher and gateway correctly use `new_with_native_classes_whitelist` and receive the operator-configured value: [3](#0-2) [4](#0-3) 

But two production callers use the plain `new` constructor:

1. `crates/native_blockifier/src/py_block_executor.rs` — the Python-facing block executor used by the legacy sequencer path.
2. `crates/starknet_transaction_prover/src/running/runner.rs:429-433` — the transaction prover that re-executes blocks to generate ZK proofs. [5](#0-4) 

### Impact Explanation

The mainnet deployment config restricts native execution to a single class hash:

```yaml
native_classes_whitelist: '["0x054c5afe61ed27be53b1e4dec5707209a9fcabdb14712fb800fbc60439090115"]'
``` [6](#0-5) 

Any other Cairo 1 class that the compilation worker has compiled to native will be executed as `V1Native` in the prover and `py_block_executor` paths, while the batcher executes the same class as `V1` (CASM). If native compilation introduces any behavioral divergence from CASM (a known risk with ahead-of-time native compilation), the prover produces a proof over a different execution trace than the one the sequencer committed to. This matches the impact category: **Wrong compiled class, CASM/native artifact, or contract code selected for execution**.

Additionally, the `BatcherDynamicConfig` default also sets `native_classes_whitelist: NativeClassesWhitelist::All`: [7](#0-6) 

meaning any batcher instance that does not receive an explicit config override also silently runs all classes as native, compounding the scope of the bypass.

### Likelihood Explanation

The `NativeClassesWhitelist` config pointer is explicitly wired through `config_schema.json` to both `batcher_config.dynamic_config.native_classes_whitelist` and `gateway_config.dynamic_config.native_classes_whitelist`: [8](#0-7) 

Operators who set a `Limited` whitelist in production (as mainnet does) have a reasonable expectation that it is enforced everywhere. The prover and `py_block_executor` silently ignore it. Any transaction invoking a non-whitelisted Cairo 1 class that has been compiled to native triggers the divergence — no privileged access is required.

### Recommendation

Remove the `new` constructor or make it private and replace all call sites with `new_with_native_classes_whitelist`, passing the configured `NativeClassesWhitelist` value. For the transaction prover, the whitelist should be threaded through `RpcRunnerFactory` from the node config. For `py_block_executor`, the whitelist should be passed from the Python layer. The `BatcherDynamicConfig` default should be changed from `All` to `Limited(vec![])` (deny-by-default) so that native execution requires explicit opt-in.

### Proof of Concept

1. Deploy the sequencer with `native_classes_whitelist = '["0xAAAA"]'` (only class `0xAAAA` whitelisted).
2. Declare and deploy a Cairo 1 contract with class hash `0xBBBB`.
3. The compilation worker compiles `0xBBBB` to native and caches it as `CompiledClasses::V1Native(Compiled(...))`.
4. Submit a transaction invoking `0xBBBB`.
5. **Batcher path** (`new_with_native_classes_whitelist` with `Limited(["0xAAAA"])`): `get_runnable` hits the downgrade branch → returns `V1` (CASM). Execution uses CASM.
6. **Prover path** (`new` → `All`): `get_runnable` skips the downgrade branch → returns `V1Native`. Execution uses native.
7. If native and CASM produce any divergent output (gas, storage writes, events), the prover generates a proof over a trace that does not match the committed state diff, causing proof verification failure or silent state corruption. [1](#0-0) [9](#0-8) [10](#0-9)

    /// Returns the runnable compiled class for the given class hash, if it exists in class_cache.
    pub fn get_runnable(
        &self,
        class_hash: &ClassHash,
        native_classes_whitelist: &NativeClassesWhitelist,
    ) -> Option<RunnableCompiledClass> {
        let cached_class = self.class_cache.get(class_hash)?;
        if let CompiledClasses::V1(..) = cached_class {
            // When native mode is WaitOnCompilation, all V1 classes should have been
            // compiled to native synchronously. A V1 cache entry indicates a pipeline bug.
            assert_ne!(
                self.cairo_native_mode(),
                CairoNativeMode::WaitOnCompilation,
                "Manager did not wait on native compilation."
            );
        }

        let cached_class = match cached_class {
            CompiledClasses::V1Native(CachedCairoNative::Compiled(native))
                if !native_classes_whitelist.contains(class_hash) =>
            {
                CompiledClasses::into_non_native_class(native)
            }
            CompiledClasses::V1Native(..) | CompiledClasses::V1(..) | CompiledClasses::V0(..) => {
                cached_class
            }
        };

        Some(cached_class.to_runnable())
    }
