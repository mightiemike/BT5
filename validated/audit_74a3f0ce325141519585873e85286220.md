Audit Report

## Title
`SwapAllowlistExtension.beforeSwap` checks router address instead of actual swapper, enabling full allowlist bypass — (`metric-periphery/contracts/extensions/SwapAllowlistExtension.sol`)

## Summary

`SwapAllowlistExtension` gates swaps by checking the `sender` argument passed by the pool, which is the pool's `msg.sender` — the immediate caller of `pool.swap()`. When `MetricOmmSimpleRouter` is used, `sender` is the router address, not the end user. A pool admin who allowlists the router (the only way to let allowlisted users trade via the router) inadvertently grants swap access to every public caller of the router, fully bypassing the allowlist.

## Finding Description

`MetricOmmPool.swap()` passes `msg.sender` as the `sender` argument to `_beforeSwap`: [1](#0-0) 

`ExtensionCalling._beforeSwap` encodes that `sender` verbatim into the extension hook call: [2](#0-1) 

`SwapAllowlistExtension.beforeSwap` then evaluates `allowedSwapper[msg.sender][sender]`, where `msg.sender` is the pool and `sender` is whoever called the pool: [3](#0-2) 

When `MetricOmmSimpleRouter.exactInputSingle` (or any `exact*` variant) calls the pool, the pool's `msg.sender` is the router: [4](#0-3) 

The check therefore evaluates `allowedSwapper[pool][router]`, not `allowedSwapper[pool][actualUser]`. Once the router is allowlisted — the only configuration that permits allowlisted users to trade via the router — the check passes for **any** caller of the router.

A secondary weakness: the `beforeSwap` override drops the `onlyPool` modifier present on the base class stub: [5](#0-4) 

The override at line 31 carries no such guard, making the function callable by any address (though the allowlist check still fails for non-pool callers with no entries). [6](#0-5) 

`DepositAllowlistExtension` correctly avoids this problem by checking `owner` (the LP position owner) rather than `sender` (the immediate caller): [7](#0-6) 

## Impact Explanation

A curated pool using `SwapAllowlistExtension` to restrict trading to KYC'd or otherwise vetted addresses is fully bypassed. Any public user can call `MetricOmmSimpleRouter.exactInputSingle`, `exactInput`, `exactOutputSingle`, or `exactOutput` targeting the pool. If the router is allowlisted — the only way to let allowlisted users trade via the router — the extension passes for every caller. Unauthorized users gain full swap access to the pool, breaking the LP's curation guarantee and any regulatory or risk-management intent behind the allowlist. This constitutes broken core pool functionality (access control) causing direct unauthorized fund flows through the pool.

## Likelihood Explanation

The trigger requires the pool admin to have allowlisted the router address. This is the natural and expected configuration for any curated pool that intends to support the standard periphery UX. The router is a public, permissionless contract, so once it is allowlisted, the bypass is reachable by any address with no special privileges or setup. The attacker only needs to call a standard router function with the target pool address.

## Recommendation

The `sender` forwarded to extensions must represent the economically relevant actor, not the immediate pool caller. Two viable fixes:

1. **Check `recipient` instead of `sender`**: For swap allowlists, gate on `recipient` (the address receiving tokens). The router always sets `recipient` to the user-supplied address, correctly identifying the economic beneficiary. This mirrors the pattern `DepositAllowlistExtension` uses with `owner`.
2. **Extension-data forwarding**: The router encodes the original `msg.sender` into `extensionData`; the extension decodes and verifies it (requires a trusted router registry or signature scheme).

Additionally, restore the `onlyPool` modifier on the `beforeSwap` override to match the base class guard.

## Proof of Concept

1. Pool admin deploys a pool with `SwapAllowlistExtension` configured.
2. Admin calls `setAllowedToSwap(pool, alice, true)` — Alice is the only intended swapper.
3. Admin calls `setAllowedToSwap(pool, router, true)` — necessary so Alice can use the router.
4. Bob (not allowlisted) calls `MetricOmmSimpleRouter.exactInputSingle({pool: pool, recipient: bob, ...})`.
5. The router calls `pool.swap(recipient=bob, ...)` with `msg.sender = router`.
6. The pool calls `extension.beforeSwap(sender=router, ...)`.
7. The extension evaluates `allowedSwapper[pool][router]` → `true` → no revert.
8. Bob's swap executes successfully on the curated pool, bypassing the allowlist entirely.
