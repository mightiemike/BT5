### Title
`verify_transaction_inclusion` Accepts Internal Merkle Tree Nodes as Valid Transaction IDs, Enabling SPV Proof Forgery — (File: `contract/src/lib.rs`)

---

### Summary

The deprecated but still publicly callable `verify_transaction_inclusion` function does not validate that the caller-supplied `tx_id` is a leaf-level transaction hash rather than an internal Merkle tree node. Any unprivileged NEAR caller can supply an internal node as `tx_id` with a shortened but arithmetically valid Merkle proof and receive a `true` return value, falsely proving inclusion of a non-existent transaction.

---

### Finding Description

`verify_transaction_inclusion` is annotated `#[deprecated(since = "0.5.0")]` but remains a fully public, externally callable contract method. The function's own inline documentation explicitly acknowledges the missing check:

> "This function may return `true` if the provided `tx_id` is a hash of an internal node in the Merkle tree rather than a valid transaction hash. We assume that validation of whether the `tx_id` corresponds to a valid transaction hash is performed at a higher level of verification." [1](#0-0) 

The function performs five checks before computing the proof:

1. `confirmations <= gc_threshold`
2. Block hash exists in the main chain
3. Sufficient confirmations
4. Merkle proof is non-empty
5. `compute_root_from_merkle_proof(tx_id, tx_index, merkle_proof) == header.merkle_root` [2](#0-1) 

None of these checks distinguish a leaf-level transaction hash from an internal Merkle tree node. The `compute_root_from_merkle_proof` function in `merkle-tools` is purely positional — it hashes the supplied value with each proof element according to parity of `current_position`, with no awareness of tree depth or whether the starting value is a leaf: [3](#0-2) 

`verify_transaction_inclusion_v2` was introduced to mitigate the 64-byte Merkle forgery attack via a coinbase proof length constraint, but it does not remove or gate the v1 function. V1 remains independently reachable. [4](#0-3) 

---

### Impact Explanation

For a block whose Merkle tree has leaves `[A, B, C, D]`:

- Internal node `AB = hash(A, B)` exists at depth 1.
- A valid proof for `AB` at `tx_index = 0` is `[CD]`, because `hash(AB, CD) = ABCD = merkle_root`.

An attacker submits `tx_id = AB`, `tx_index = 0`, `merkle_proof = [CD]`. `compute_root_from_merkle_proof` computes `hash(AB, CD) = merkle_root` and the function returns `true`. The attacker has proven inclusion of a transaction that does not exist. Any downstream NEAR contract or off-chain system that calls `verify_transaction_inclusion` and acts on a `true` result (e.g., releasing bridged funds, minting wrapped assets, authorizing withdrawals) is exploitable without any Bitcoin-side action.

---

### Likelihood Explanation

The function is `pub` with no role guard, no `#[pause]` restriction beyond the contract-level pause, and no deposit requirement. Any unprivileged NEAR account can call it. The required inputs — a real block hash in the main chain and the block's public Merkle tree — are freely available from any Bitcoin block explorer. The construction of a valid internal-node proof requires only arithmetic on public data; no hash preimage search or privileged access is needed.

---

### Recommendation

Remove `verify_transaction_inclusion` from the public ABI entirely, or gate it with a hard `env::panic_str` that redirects callers to v2. The deprecation attribute is a compile-time hint only; it provides no runtime protection. `verify_transaction_inclusion_v2` enforces equal proof lengths for the target transaction and the coinbase, which prevents the internal-node substitution attack and should be the sole externally callable proof endpoint.

---

### Proof of Concept

```
# Block with 4 txs: A, B, C, D
# merkle_root = hash(hash(A,B), hash(C,D))
# Internal node AB = hash(A, B)

near call <contract> verify_transaction_inclusion '{
  "args": {
    "tx_id":             "<AB = hash(A,B)>",
    "tx_block_blockhash":"<real block hash in mainchain>",
    "tx_index":          0,
    "merkle_proof":      ["<CD = hash(C,D)>"],
    "confirmations":     1
  }
}' --accountId attacker.near
# Returns: true
# No such transaction exists in the block.
```

    pub fn verify_transaction_inclusion(&self, #[serializer(borsh)] args: ProofArgs) -> bool {
        require!(
            args.confirmations <= self.gc_threshold,
            "The required number of confirmations exceeds the number of blocks stored in memory"
        );

        let heaviest_block_header = self
            .headers_pool
            .get(&self.mainchain_tip_blockhash)
            .unwrap_or_else(|| env::panic_str(ERR_KEY_NOT_EXIST));
        let target_block_height = self
            .mainchain_header_to_height
            .get(&args.tx_block_blockhash)
            .unwrap_or_else(|| env::panic_str("block does not belong to the current main chain"));

        // Check requested confirmations. No need to compute proof if insufficient confirmations.
        require!(
            (heaviest_block_header.block_height).saturating_sub(target_block_height) + 1
                >= args.confirmations,
            "Not enough blocks confirmed"
        );

        let header = self
            .headers_pool
            .get(&args.tx_block_blockhash)
            .unwrap_or_else(|| env::panic_str("cannot find requested transaction block"));

        require!(!args.merkle_proof.is_empty(), "Merkle proof is empty");

        // compute merkle tree root and check if it matches block's original merkle tree root
        merkle_tools::compute_root_from_merkle_proof(
            args.tx_id,
            usize::try_from(args.tx_index).unwrap(),
            &args.merkle_proof,
        ) == header.block_header.merkle_root
    }

            "Coinbase merkle proof and transaction merkle proof should have the same length"

        let header = self
            .headers_pool
            .get(&args.tx_block_blockhash)
            .unwrap_or_else(|| env::panic_str("cannot find requested transaction block"));

        require!(
            merkle_tools::compute_root_from_merkle_proof(
                args.coinbase_tx_id.clone(),
                0usize,
                &args.coinbase_merkle_proof,
            ) == header.block_header.merkle_root,
            "Incorrect coinbase merkle proof"
        );

        #[allow(deprecated)]
        self.verify_transaction_inclusion(args.into())
    }

pub fn compute_root_from_merkle_proof(
    transaction_hash: H256,
    transaction_position: usize,
    merkle_proof: &Vec<H256>,
) -> H256 {
    let mut current_hash = transaction_hash;
    let mut current_position = transaction_position;

    for proof_hash in merkle_proof {
        if current_position % 2 == 0 {
            current_hash = compute_hash(&current_hash, proof_hash);
        } else {
            current_hash = compute_hash(proof_hash, &current_hash);
        }
        current_position /= 2;
    }

    current_hash
}
