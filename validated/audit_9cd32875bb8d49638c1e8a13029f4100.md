Audit Report

## Title
`SwapAllowlistExtension` checks the direct pool caller (router address) instead of the end user, allowing any user to bypass the swap allowlist via `MetricOmmSimpleRouter` — (`metric-periphery/contracts/extensions/SwapAllowlistExtension.sol`)

## Summary

`SwapAllowlistExtension.beforeSwap` gates on `sender`, which is `msg.sender` of `pool.swap()`. When `MetricOmmSimpleRouter` mediates a swap, `sender` equals the router's address, not the end user's. A pool admin who allowlists the router to enable router-mediated swaps for their permitted users inadvertently opens the allowlist to all users, because the router is a shared public contract with no per-user access control.

## Finding Description

**Step 1 — Pool passes `msg.sender` as `sender` to extensions.**

In `MetricOmmPool.swap()`, `_beforeSwap` is called with `msg.sender` as the first argument: [1](#0-0) 

**Step 2 — `ExtensionCalling._beforeSwap` forwards `sender` unchanged.** [2](#0-1) 

**Step 3 — `SwapAllowlistExtension.beforeSwap` checks `allowedSwapper[pool][sender]`.**

`msg.sender` here is the pool (the extension's caller), and `sender` is the direct caller of `pool.swap()`: [3](#0-2) 

**Step 4 — `MetricOmmSimpleRouter.exactInputSingle` calls `pool.swap()` directly, making itself `msg.sender` to the pool.**

The router does not encode the end user's address into `extensionData` before forwarding it: [4](#0-3) 

**Root cause:** The extension sees `sender = router_address`. If the admin allowlists the router (a natural action to enable router UX for permitted users), `allowedSwapper[pool][router]` is `true`, and the check passes for *any* end user who routes through the router. There is no mechanism in the extension or the router to distinguish "router called on behalf of Alice (allowlisted)" from "router called on behalf of Eve (not allowlisted)."

The `setAllowedToSwap` setter only gates by address, with no per-user-via-router granularity: [5](#0-4) 

The Smart Audit Pivot for the allowlist path explicitly states checks "cannot be bypassed through router" — this is exactly the bypass demonstrated here.

## Impact Explanation

High. Any user can trade on a curated pool (e.g., KYC-gated, institutional) by routing through `MetricOmmSimpleRouter`, completely defeating the pool's access control policy. LP funds are directly at risk: unauthorized swappers can extract value at oracle-anchored prices without the pool admin's consent, violating the pool's intended participation constraints. This constitutes a broken core pool invariant (swap allowlist) with direct fund impact on LPs.

## Likelihood Explanation

Medium. The bypass requires the pool admin to allowlist the router address. This is a predictable and natural operational action for any curated pool that wants to support router-mediated swaps for their permitted users. The admin has no way to simultaneously enable router UX and maintain allowlist security with the current design, making this a foreseeable operational mistake. No existing test covers the router-mediated bypass path.

## Recommendation

The router should encode the end user's address (`msg.sender`) into `extensionData` before calling `pool.swap()`, and `SwapAllowlistExtension.beforeSwap` should decode and gate on that address when present:

```solidity
function beforeSwap(
    address sender,
    address,
    bool, int128, uint128, uint256, uint128, uint128,
    bytes calldata extensionData
) external view override returns (bytes4) {
    address swapper = (extensionData.length >= 32)
        ? abi.decode(extensionData, (address))
        : sender;
    if (!allowAllSwappers[msg.sender] && !allowedSwapper[msg.sender][swapper]) {
        revert IMetricOmmPoolActions.NotAllowedToSwap();
    }
    return IMetricOmmExtensions.beforeSwap.selector;
}
```

The router would prepend `abi.encode(msg.sender)` to `extensionData` before calling `pool.swap()`. Alternatively, document explicitly that allowlisting the router is equivalent to `setAllowAllSwappers(pool, true)`.

## Proof of Concept

```solidity
function test_SwapAllowlistBypassViaRouter() public {
    // Pool admin allowlists Alice and the router
    // (reasonable: admin wants Alice to be able to use the router)
    swapExtension.setAllowedToSwap(address(pool), alice, true);
    swapExtension.setAllowedToSwap(address(pool), address(router), true);

    // Eve (not allowlisted) bypasses the allowlist via the router
    vm.prank(eve);
    router.exactInputSingle(IMetricOmmSimpleRouter.ExactInputSingleParams({
        pool:             address(pool),
        tokenIn:          address(token0),
        tokenOut:         address(token1),
        zeroForOne:       true,
        amountIn:         1_000,
        amountOutMinimum: 0,
        recipient:        eve,
        deadline:         block.timestamp + 1,
        priceLimitX64:    0,
        extensionData:    ""
    }));
    // Eve successfully swaps despite not being allowlisted.
    // The extension saw sender = router (allowlisted), not Eve (not allowlisted).
}
```
