Audit Report

## Title
`SwapAllowlistExtension.beforeSwap` gates on router address instead of end-user when swaps route through `MetricOmmSimpleRouter`, enabling full allowlist bypass or blocking all router-based swaps for individually allowlisted users - (`metric-periphery/contracts/extensions/SwapAllowlistExtension.sol`)

## Summary

`SwapAllowlistExtension.beforeSwap` checks the `sender` parameter against `allowedSwapper[pool][sender]`, but `sender` is `msg.sender` of the pool's `swap()` call ŌĆö which is the `MetricOmmSimpleRouter` contract address, not the end-user, whenever the standard periphery path is used. A pool admin configuring per-user swap restrictions faces two mutually exclusive broken states: either the router is allowlisted and every user bypasses the restriction, or individual users are allowlisted and cannot use the router at all.

## Finding Description

**Root cause ŌĆö wrong actor bound to `sender`:**

In `MetricOmmPool.swap()`, `_beforeSwap` is called with `msg.sender` as the first argument: [1](#0-0) 

This `msg.sender` propagates through `ExtensionCalling._beforeSwap` unchanged: [2](#0-1) 

`SwapAllowlistExtension.beforeSwap` then checks this value as the swapper identity: [3](#0-2) 

**Router call path ŌĆö `msg.sender` inside the pool is the router:**

When a user calls `MetricOmmSimpleRouter.exactInputSingle()`, the router calls `pool.swap()` directly: [4](#0-3) 

So `msg.sender` inside `MetricOmmPool.swap()` is the router address. The extension receives `sender = router` and evaluates `allowedSwapper[pool][router]` ŌĆö not `allowedSwapper[pool][actual_user]`. The same misbinding applies to `exactInput`, `exactOutputSingle`, and `exactOutput`: [5](#0-4) 

**Contrast with `DepositAllowlistExtension`:**

`DepositAllowlistExtension.beforeAddLiquidity` correctly gates on `owner` ŌĆö the position owner explicitly passed through the call chain, independent of the intermediary sender: [6](#0-5) 

`SwapAllowlistExtension` has no equivalent stable identity; it uses `sender`, which changes depending on whether the user calls the pool directly or through the router.

**Exploit path ŌĆö allowlist bypass:**

1. Pool deployed with `SwapAllowlistExtension`.
2. Admin calls `setAllowedToSwap(pool, router, true)` to enable router-based swaps for allowlisted users.
3. Mallory (not individually allowlisted) calls `router.exactInputSingle({pool: pool, ...})`.
4. Router calls `pool.swap(mallory, ...)` ŌĆö `msg.sender` inside pool = router.
5. Pool calls `extension.beforeSwap(router, mallory, ...)`.
6. Extension checks `allowedSwapper[pool][router] == true` ŌåÆ passes.
7. Mallory receives tokens from the curated pool despite never being individually authorized.

**Exploit path ŌĆö authorized user blocked:**

1. Admin calls `setAllowedToSwap(pool, alice, true)`.
2. Alice calls `router.exactInputSingle({pool: pool, ...})`.
3. Extension checks `allowedSwapper[pool][router] == false` ŌåÆ reverts `NotAllowedToSwap`.
4. Alice cannot use the standard periphery path despite being allowlisted.

## Impact Explanation

In the bypass scenario, unauthorized parties receive token output from a curated pool configured to exclude them ŌĆö a direct policy bypass with fund-impacting consequences (broken core pool functionality: swap allowlist is the sole gating mechanism for curated pools). In the blocking scenario, individually allowlisted users cannot use the primary supported periphery path, making the pool's swap flow unusable for the standard entry point. Both states constitute broken core pool functionality meeting the contest threshold.

## Likelihood Explanation

`MetricOmmSimpleRouter` is the primary supported periphery entry point for swaps. Any user of a curated pool who uses the router ŌĆö the expected and documented path ŌĆö triggers this misbinding. The pool admin has no way to configure `SwapAllowlistExtension` to correctly enforce per-user restrictions while also allowing router-based swaps, because the extension has no access to the true originating user address. No special attacker capability is required; a normal `exactInputSingle` call suffices.

## Recommendation

`SwapAllowlistExtension` must identify the true end-user, not the intermediary router. Two approaches:

1. **`extensionData` forwarding:** Have `MetricOmmSimpleRouter` encode the actual `msg.sender` (the end-user) into `extensionData`, and have `SwapAllowlistExtension.beforeSwap` decode and check that address instead of (or in addition to) `sender`. This requires a convention between the router and the extension.
2. **Dual-check:** Check both `sender` and a user address decoded from `extensionData`, falling back to `sender` when `extensionData` is empty (for direct pool callers).

// Foundry integration test sketch
function test_allowlistBypassViaRouter() public {
    // Setup: pool with SwapAllowlistExtension, router allowlisted
    swapExtension.setAllowedToSwap(address(pool), address(router), true);

    // Mallory is NOT individually allowlisted
    address mallory = makeAddr("mallory");
    assertFalse(swapExtension.isAllowedToSwap(address(pool), mallory));

    // Mallory swaps via router ŌĆö extension sees sender=router, passes
    vm.prank(mallory);
    router.exactInputSingle(IMetricOmmSimpleRouter.ExactInputSingleParams({
        pool: address(pool),
        recipient: mallory,
        tokenIn: token0,
        zeroForOne: true,
        amountIn: 1e18,
        amountOutMinimum: 0,
        priceLimitX64: 0,
        deadline: block.timestamp,
        extensionData: ""
    }));
    // Mallory receives tokens ŌĆö allowlist bypassed
}

function test_allowlistedUserBlockedViaRouter() public {
    address alice = makeAddr("alice");
    swapExtension.setAllowedToSwap(address(pool), alice, true);

    // Alice swaps via router ŌĆö extension sees sender=router, reverts
    vm.prank(alice);
    vm.expectRevert(IMetricOmmPoolActions.NotAllowedToSwap.selector);
    router.exactInputSingle(IMetricOmmSimpleRouter.ExactInputSingleParams({
        pool: address(pool),
        recipient: alice,
        tokenIn: token0,
        zeroForOne: true,
        amountIn: 1e18,
        amountOutMinimum: 0,
        priceLimitX64: 0,
        deadline: block.timestamp,
        extensionData: ""
    }));
}
