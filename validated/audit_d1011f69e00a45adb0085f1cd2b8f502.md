### Title
`run_mainchain_gc` Permanently Deletes Block Headers Without Any Caller Role Check — (`contract/src/lib.rs`)

### Summary
`run_mainchain_gc` is a public, state-mutating NEAR contract function that permanently erases verified block headers from the light client's canonical chain storage. It carries only a `#[pause]` decorator, which blocks execution when the contract is paused but imposes **no restriction whatsoever on who may call it when the contract is running normally**. Any unprivileged NEAR account can invoke it at any time with an arbitrary `batch_size`, forcing immediate deletion of all GC-eligible headers in a single transaction and permanently breaking SPV proof verification for those blocks.

### Finding Description

`run_mainchain_gc` is declared `pub fn` and annotated only with `#[pause(except(roles(Role::UnrestrictedRunGC)))]`: [1](#0-0) 

The `#[pause]` macro's sole effect is to block the call when the contract is in a paused state; it performs no caller identity or role check when the contract is unpaused. There is no `#[trusted_relayer]`, no `require!(self.acl_has_role(...))`, and no `env::predecessor_account_id()` guard anywhere in the function body. [2](#0-1) 

When `amount_of_headers_we_store > gc_threshold`, the function removes up to `min(total_amount_to_remove, batch_size)` headers. An attacker supplying `batch_size = u64::MAX` removes every GC-eligible header — all headers older than `gc_threshold` blocks from the current tip — in a single call. For each removed height the function:

- Calls `remove_block_header`, which erases the entry from both `mainchain_header_to_height` and `headers_pool`
- Removes the entry from `mainchain_height_to_header`
- Advances `mainchain_initial_blockhash` past the deleted range [3](#0-2) 

These deletions are **permanent and irreversible** — there is no undo path in the contract.

By contrast, `submit_blocks` — the only other caller of `run_mainchain_gc` — is protected by both `#[pause]` and `#[trusted_relayer]`, restricting it to accounts holding `Role::UnrestrictedSubmitBlocks` or `Role::DAO`: [4](#0-3) 

The `UnrestrictedRunGC` role exists in the role enum specifically to allow GC on a paused contract: [5](#0-4) 

This confirms the design intent that GC access should be role-gated, but the gate was only applied to the paused path, leaving the unpaused path completely open.

### Impact Explanation

After an attacker calls `run_mainchain_gc(u64::MAX)`:

1. All headers older than `gc_threshold` blocks are permanently erased from `headers_pool` and both height-index maps.
2. `verify_transaction_inclusion` and `verify_transaction_inclusion_v2` panic with `"block does not belong to the current main chain"` or `"cannot find requested transaction block"` for any deleted block hash. [6](#0-5) 

3. Any downstream NEAR contract that calls these verification functions for a deleted block receives a permanent failure — Bitcoin transaction inclusion proofs for those blocks can never be verified again on this deployment.
4. Chain reorgs that require walking back past the new `mainchain_initial_blockhash` will panic with `PrevBlockNotFound`, as documented in `CLAUDE.md`. [7](#0-6) 

### Likelihood Explanation

The entry point is a standard NEAR contract call requiring no deposit, no special account, and no prior interaction with the contract. Any NEAR account — including a freshly created one — can call `run_mainchain_gc` on a live, unpaused deployment. The only precondition is that the chain has grown beyond `gc_threshold` (the recommended production value is 52,704 blocks, approximately one year of Bitcoin blocks), which is a normal operational state for a long-running deployment. The attack is a single transaction.

### Recommendation

Add an explicit caller role check to `run_mainchain_gc` so that only authorized accounts (e.g., `Role::DAO`, `Role::UnrestrictedRunGC`, or a dedicated `GCManager` role) may invoke it when the contract is unpaused. The simplest fix is to add an `acl_require_role` guard at the top of the function body, or to apply `#[trusted_relayer]` (or an equivalent access-control attribute) alongside the existing `#[pause]` decorator. The internal call from `submit_blocks` should be routed through a private helper to avoid the role check on the internal path.

### Proof of Concept

1. Deploy the contract with `gc_threshold = 52704` and submit enough headers that `mainchain_size > gc_threshold` (normal production state after ~1 year).
2. From any unprivileged NEAR account, call:
   ```
   run_mainchain_gc({ "batch_size": 18446744073709551615 })
   ```
3. Observe that all headers older than `gc_threshold` blocks are deleted in one transaction and `mainchain_initial_blockhash` is advanced.
4. Attempt `verify_transaction_inclusion` for any deleted block hash — the call panics with `"block does not belong to the current main chain"`, confirming permanent loss of SPV verification for those blocks. [8](#0-7)

```rust
    #[pause(except(roles(Role::UnrestrictedRunGC)))]
    pub fn run_mainchain_gc(&mut self, batch_size: u64) {
        let initial_blockheader = self
            .headers_pool
            .get(&self.mainchain_initial_blockhash)
            .unwrap_or_else(|| env::panic_str(ERR_KEY_NOT_EXIST));

        let tip_blockheader = self
            .headers_pool
            .get(&self.mainchain_tip_blockhash)
            .unwrap_or_else(|| env::panic_str(ERR_KEY_NOT_EXIST));

        let amount_of_headers_we_store =
            tip_blockheader.block_height - initial_blockheader.block_height + 1;

        if amount_of_headers_we_store > self.gc_threshold {
            let total_amount_to_remove = amount_of_headers_we_store - self.gc_threshold;
            let selected_amount_to_remove = std::cmp::min(total_amount_to_remove, batch_size);

            let start_removal_height = initial_blockheader.block_height;
            let end_removal_height = initial_blockheader.block_height + selected_amount_to_remove;
            env::log_str(&format!(
                "Num of blocks to remove {selected_amount_to_remove}"
            ));

            for height in start_removal_height..end_removal_height {
                let blockhash = &self
                    .mainchain_height_to_header
                    .get(&height)
                    .unwrap_or_else(|| env::panic_str(ERR_KEY_NOT_EXIST));

                self.remove_block_header(blockhash);
                self.mainchain_height_to_header.remove(&height);
            }

            self.mainchain_initial_blockhash = self
                .mainchain_height_to_header
                .get(&end_removal_height)
                .unwrap_or_else(|| env::panic_str(ERR_KEY_NOT_EXIST));
        }
    }
```
