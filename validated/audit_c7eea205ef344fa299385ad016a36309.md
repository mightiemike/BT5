### Title
Unchecked `transferFrom` Return Value and Missing Access Control in `replaceUsdcEWithUsdc` Enables Zero-Cost Theft of All DDA usdcE — (`core/contracts/ContractOwner.sol`)

---

### Summary

`ContractOwner.replaceUsdcEWithUsdc` is callable by any EOA on chain 57073, and its USDC `transferFrom` call is made directly on the `IERC20Base` interface (not via `ERC20Helper.safeTransferFrom`), so a silent `false` return is never checked. An attacker with zero USDC can drain usdcE from every DDA in a single transaction at zero cost.

---

### Finding Description

The function at issue:

```solidity
// ContractOwner.sol lines 608-620
function replaceUsdcEWithUsdc(bytes32 subaccount) external {
    require(block.chainid == 57073, ERR_UNAUTHORIZED);
    address payable directDepositV1 = directDepositV1Address[subaccount];
    require(directDepositV1 != address(0), "no dda");
    address usdcE = 0xF1815bd50389c46847f0Bda824eC8da914045D14;
    address usdc  = 0x2D270e6886d130D724215A266106e6832161EAEd;
    uint256 balance = IERC20Base(usdcE).balanceOf(directDepositV1);
    if (balance > 0) {
        IERC20Base(usdc).transferFrom(msg.sender, directDepositV1, balance); // ← bare call, return ignored
        DirectDepositV1(directDepositV1).withdraw(IIERC20Base(usdcE));       // ← usdcE → ContractOwner
        IERC20Base(usdcE).safeTransfer(msg.sender, balance);                 // ← usdcE → attacker
    }
}
```

Two independent flaws combine:

**Flaw 1 — No access control.**
The function is `external` with no `onlyOwner` or equivalent guard. [1](#0-0) 

Compare with the sibling function `withdrawFromDirectDepositV1`, which correctly gates on `onlyOwner`: [2](#0-1) 

**Flaw 2 — Bare `transferFrom` call; return value silently discarded.**
Line 616 calls `IERC20Base(usdc).transferFrom(...)` directly on the interface. Although `ContractOwner` imports `ERC20Helper` and declares `using ERC20Helper for IERC20Base`, the extension methods added are `safeTransfer` and `safeTransferFrom`. The bare `.transferFrom()` call is a plain interface dispatch whose `bool` return is never captured or checked. [3](#0-2) 

`ERC20Helper.safeTransferFrom` would have reverted on a `false` return: [4](#0-3) 

**Flaw 3 — `DirectDepositV1.withdraw` sends tokens to its `msg.sender` (ContractOwner).**
`withdraw` is `onlyOwner`, and the owner of every DDA is `ContractOwner`. When `ContractOwner` calls `withdraw`, the DDA's `safeTransfer` sends the full usdcE balance to `ContractOwner` (`msg.sender` in that context). [5](#0-4) 

**Flaw 4 — `safeTransfer` then forwards usdcE to the attacker.**
Line 618 uses `ERC20Helper.safeTransfer` (the safe variant, which does check the return value) to send the usdcE that just arrived in `ContractOwner` to the original `msg.sender` — the attacker. [6](#0-5)

### Impact Explanation

For every DDA with a non-zero usdcE balance:

| Step | Actor | Token flow |
|------|-------|-----------|
| `transferFrom(attacker → dda, balance)` | USDC contract | Returns `false`; no USDC moves |
| `dda.withdraw(usdcE)` | DDA → ContractOwner | All usdcE leaves DDA |
| `safeTransfer(ContractOwner → attacker, balance)` | ContractOwner | All usdcE reaches attacker |

Net result: attacker gains `Σ usdcE_balance(dda_i)` across all DDAs; attacker spends zero USDC. The DDAs are fully drained. Any subaccount whose DDA held usdcE loses those funds permanently with no recourse.

---

### Likelihood Explanation

- Callable by any EOA on chain 57073 (no special role, no signature, no prior deposit).
- Zero cost: no USDC required, no approval required.
- Repeatable in a single transaction across all DDAs by iterating over known subaccounts.
- The `directDepositV1Address` mapping is public, so all DDA addresses are enumerable on-chain. [7](#0-6) 

---

### Recommendation

Apply both fixes independently; neither alone is sufficient:

1. **Add `onlyOwner`** to `replaceUsdcEWithUsdc`, consistent with every other DDA-mutating function in `ContractOwner`.

2. **Replace the bare `transferFrom` with `ERC20Helper.safeTransferFrom`**:
   ```solidity
   // Before (unsafe):
   IERC20Base(usdc).transferFrom(msg.sender, directDepositV1, balance);
   
   // After (safe):
   IERC20Base(usdc).safeTransferFrom(msg.sender, directDepositV1, balance);
   ```
   This uses the already-imported `ERC20Helper` library and will revert if the caller has insufficient USDC or has not approved the contract. [4](#0-3)
