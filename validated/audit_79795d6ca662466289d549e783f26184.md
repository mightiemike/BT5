### Title
Asymmetric Fee Formula Overcharges Taker Buy Orders — (`File: core/contracts/OffchainExchange.sol`)

---

### Summary

In `OffchainExchange.applyFee`, the fee calculation uses two different formulas depending on the sign of `meteredQuote`. When `meteredQuote > 0` (user receiving quote / selling base), the fee is correctly computed as `meteredQuote * feeRate`. When `meteredQuote < 0` (user paying quote / buying base), the code divides by `(1 - feeRate)` instead of multiplying by `feeRate`, producing a systematically inflated fee. This is the same arithmetic class as JuiceBox M-14: using a complement-based division that yields `feeRate / (1 - feeRate)` instead of `feeRate`.

---

### Finding Description

The vulnerable code is in `applyFee` at lines 556–560:

```solidity
int128 keepRateX18 = ONE - feeInfo.feeRate;
int128 newMeteredQuote = (meteredQuote > 0)
    ? meteredQuote.mul(keepRateX18)
    : meteredQuote.div(keepRateX18);   // ← bug
orderInfo.fee = meteredQuote - newMeteredQuote;
``` [1](#0-0) 

**When `meteredQuote > 0`** (taker selling base, receiving quote):
- `newMeteredQuote = meteredQuote × (1 − feeRate)`
- `fee = meteredQuote − newMeteredQuote = meteredQuote × feeRate` ✓ correct

**When `meteredQuote < 0`** (taker buying base, paying quote):
- `newMeteredQuote = meteredQuote ÷ (1 − feeRate)`
- `fee = meteredQuote − meteredQuote ÷ (1 − feeRate)`
- Simplifying: `fee = |meteredQuote| × feeRate ÷ (1 − feeRate)` ✗ inflated

The correct formula for both directions is simply `fee = |meteredQuote| × feeRate`.

**Concrete example** (using `feeRate = 0.003` i.e. 0.3%, `meteredQuote = −1000`):

| | Formula | Result |
|---|---|---|
| Correct | `1000 × 0.003` | `3.000` |
| Current code | `1000 × 0.003 / 0.997` | `3.009` |

The overcharge factor is `1 / (1 − feeRate)`. For a 0.3% fee rate the overcharge is ~0.3% of the fee itself; for a 1% fee rate it is ~1% of the fee. The excess is silently credited to the protocol fee account via `updateCollectedFees`. [2](#0-1) 

The `quoteDelta` is then reduced by the inflated fee:

```solidity
orderInfo.quoteDelta =
    orderInfo.quoteDelta -
    orderInfo.fee -
    orderInfo.builderFee;
``` [3](#0-2) 

So the buyer's subaccount balance is debited more quote than the advertised fee rate implies.

---

### Impact Explanation

Every taker buy order (paying quote, `meteredQuote < 0`) is overcharged relative to the published fee tier. The excess quote is transferred to the protocol fee account, constituting a direct, systematic loss of user funds on every buy trade. The magnitude scales with `feeRate² / (1 − feeRate)` per trade. The maker path is also affected when a maker is buying base (`matchQuote < 0`), producing a smaller-than-expected rebate.

---

### Likelihood Explanation

Taker buy orders are one of the two most common operations in any DEX. The condition `meteredQuote < 0` is triggered on every such trade without any special precondition. Any unprivileged user submitting a buy order through the sequencer path triggers this path. Likelihood is high.

---

### Recommendation

Replace the asymmetric `mul`/`div` branch with a uniform multiplication by `feeRate` applied to the absolute value of `meteredQuote`:

```solidity
// Correct: fee = |meteredQuote| * feeRate, always positive
orderInfo.fee = meteredQuote.abs().mul(feeInfo.feeRate);
```

This produces the same result as the current code for `meteredQuote > 0` and fixes the overcharge for `meteredQuote < 0`. The `keepRateX18` variable and the ternary branch can be removed entirely.

---

### Proof of Concept

Given:
- `feeRate = 3_000_000_000_000_000` (0.3%, i.e. `0.003 × 1e18`)
- `ONE = 1e18`
- `meteredQuote = −1_000_000_000_000_000_000_000` (−1000 in X18)

**Current code path (`meteredQuote < 0` branch):**
```
keepRateX18 = 1e18 - 0.003e18 = 0.997e18
newMeteredQuote = (-1000e18 * 1e18) / 0.997e18 = -1003.009...e18
fee = -1000e18 - (-1003.009e18) = 3.009e18   ← overcharge
```

**Correct result:**
```
fee = 1000e18 * 0.003 = 3.000e18
```

The user is overcharged `0.009e18` (0.9% excess over the correct fee). This excess accrues to `marketInfo[productId].collectedFees` on every buy order. [4](#0-3)

    function applyFee(
        uint32 productId,
        OrderInfo memory orderInfo,
        MarketInfo memory market,
        int128 alreadyMatched, // in quote
        uint128 appendix,
        bool taker
    ) internal {
        // X account is passthrough for trading and incurs
        // no fees
        if (orderInfo.sender == X_ACCOUNT) {
            return;
        }
        int128 matchQuote = orderInfo.quoteDelta;
        int128 meteredQuote = 0;
        if (taker) {
            // flat minimum fee
            if (alreadyMatched == 0) {
                meteredQuote += market.minSize;
                if (matchQuote < 0) {
                    meteredQuote = -meteredQuote;
                }
            }

            // exclude the portion on [0, self.min_size) for match_quote and
            // add to metered_quote
            // fee is only applied on [minSize, quote_amount)
            int128 feeApplied = MathHelper.abs(alreadyMatched + matchQuote) -
                market.minSize;
            feeApplied = MathHelper.min(feeApplied, matchQuote.abs());
            if (feeApplied > 0) {
                if (matchQuote < 0) {
                    feeApplied = -feeApplied;
                }
                meteredQuote += feeApplied;
            }
        } else {
            // for maker rebates things stay the same
            meteredQuote += matchQuote;
        }
        FeeInfo memory feeInfo = getUserFeeRateWithBuilder(
            orderInfo.sender,
            productId,
            appendix,
            taker
        );

        int128 keepRateX18 = ONE - feeInfo.feeRate;
        int128 newMeteredQuote = (meteredQuote > 0)
            ? meteredQuote.mul(keepRateX18)
            : meteredQuote.div(keepRateX18);
        orderInfo.fee = meteredQuote - newMeteredQuote;
        orderInfo.builderFee = matchQuote.abs().mul(feeInfo.builderFeeRate);
        orderInfo.quoteDelta =
            orderInfo.quoteDelta -
            orderInfo.fee -
            orderInfo.builderFee;
        if (orderInfo.builderFee > 0) {
            collectedBuilderFee[market.quoteId][feeInfo.builderId] += orderInfo
                .builderFee;
            emitBuilderEvent(orderInfo, feeInfo.builderId, productId);
        }
    }
