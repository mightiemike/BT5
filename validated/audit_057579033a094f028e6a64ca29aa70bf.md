### Title
Missing `ClassTrace.Validate()` in `processReceivedPacket` Allows Malicious Counterparty to Mint Voucher NFTs with Empty `BaseClassId` — (`x/nft-transfer/keeper/packet.go`)

### Summary

A malicious IBC counterparty chain can send a packet whose `data.ClassId` ends with a trailing `/` (e.g., `"portA/chanA/"`). Because `ValidateBasic()` only checks that `ClassId` is not blank, and `processReceivedPacket` never calls `classTrace.Validate()` after parsing, the receiving chain stores a `ClassTrace` with an empty `BaseClassId` and mints voucher NFTs under it — violating the invariant that `BaseClassId` must be non-empty for any IBC class trace.

### Finding Description

**Entry point:** `OnRecvPacket` → `OnRecvPacket` (keeper) → `processReceivedPacket`

**Step 1 — `ValidateBasic()` does not reject trailing-slash ClassId**

`ValidateBasic()` only checks `strings.TrimSpace(nftpd.ClassId) == ""`. A `ClassId` of `"portA/chanA/"` is non-blank and passes without error. [1](#0-0) 

**Step 2 — `IsAwayFromOrigin` evaluates to `true`**

With `data.ClassId = "portA/chanA/"` and source port/channel `transfer/channel-0`, `GetClassPrefix` produces `"transfer/channel-0/"`. Since `"portA/chanA/"` does not start with `"transfer/channel-0/"`, `isAwayFromOrigin = true` and the minting branch is taken. [2](#0-1) 

**Step 3 — `ParseClassTrace` produces empty `BaseClassId`**

The dest-chain prefix (e.g., `"nft-transfer/channel-1/"`) is prepended, giving `prefixedClassID = "nft-transfer/channel-1/portA/chanA/"`. `strings.Split` on `/` yields `["nft-transfer","channel-1","portA","chanA",""]`. The last element is `""`, so:

```
ClassTrace{Path: "nft-transfer/channel-1/portA/chanA", BaseClassId: ""}
``` [3](#0-2) 

**Step 4 — `classTrace.Validate()` is never called; invalid trace is stored**

`processReceivedPacket` calls `ParseClassTrace` and immediately calls `SetClassTrace` without any validation. The `Validate()` method exists and would return `"base class_id cannot be blank"` for this trace, but it is never invoked in the receive path. [4](#0-3) 

**Step 5 — `IBCClassID()` returns a well-formed `ibc/{hash}` string; `IssueDenom` and `MintNFT` succeed**

Because `Path != ""`, `IBCClassID()` returns `"ibc/{sha256("nft-transfer/channel-1/portA/chanA/")}"` — a syntactically valid class ID. `IssueDenom` and `MintNFT` are called with this ID and succeed, minting NFTs on the receiving chain with no valid backing on the counterparty. [5](#0-4) 

The `Validate()` guard that would catch this is only used in the query path (`ClassHash` gRPC handler), not in the packet receive path. [6](#0-5) 

### Impact Explanation

- **Unauthorized voucher minting**: A malicious counterparty chain mints NFTs on the victim chain with no valid escrowed or burned backing.
- **Corrupted class trace state**: `ClassTrace{BaseClassId: ""}` is stored permanently, violating the module invariant. `GetFullClassPath()` returns a trailing-slash path (`"nft-transfer/channel-1/portA/chanA/"`), which propagates incorrect behavior into any subsequent send or refund that calls `ClassPathFromHash`.
- **Permanent state corruption**: The invalid trace cannot be removed without a governance migration; any NFTs minted under it are stranded (cannot be properly returned to origin).

### Likelihood Explanation

Any chain that has an open IBC channel with the victim chain can exploit this. The attacker only needs to craft a packet with a trailing-slash `ClassId`. No privileged access, governance, or key compromise is required — only the ability to relay an IBC packet, which is the normal IBC threat model for counterparty validation.

### Recommendation

Add a `classTrace.Validate()` call immediately after `ParseClassTrace` in `processReceivedPacket`, before `SetClassTrace` or any NFT operations:

```go
classTrace := types.ParseClassTrace(prefixedClassID)
if err := classTrace.Validate(); err != nil {
    return err
}
```

Additionally, `ValidateBasic()` on `NonFungibleTokenPacketData` should reject `ClassId` values that end with `/` or that, when parsed, produce an empty `BaseClassId`.

func ParseClassTrace(rawClassID string) ClassTrace {
	classSplit := strings.Split(rawClassID, "/")

	if classSplit[0] == rawClassID {
		return ClassTrace{
			Path:        "",
			BaseClassId: rawClassID,
		}
	}

	return ClassTrace{
		Path:        strings.Join(classSplit[:len(classSplit)-1], "/"),
		BaseClassId: classSplit[len(classSplit)-1],
	}
}
