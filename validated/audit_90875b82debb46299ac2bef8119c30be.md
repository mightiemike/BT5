### Title
Malicious Counterparty Can Unescrow Native NFTs via Crafted `ClassId` in `processReceivedPacket` — (`x/nft-transfer/keeper/packet.go`)

---

### Summary

When both chains share the same channel ID (e.g., both have `channel-0`), a malicious counterparty chain can send a crafted `NonFungibleTokenPacketData` with `ClassId = "nft/channel-0/nativeClass"`. This causes `IsAwayFromOrigin` to return `false`, the code strips the prefix to recover `nativeClass`, and then calls `TransferOwner` from the legitimate escrow address to the attacker's receiver — bypassing the invariant that escrowed NFTs may only be unescrow on a valid return packet to the original sender.

---

### Finding Description

The vulnerability lives in `processReceivedPacket`: [1](#0-0) 

**Step 1 — Direction check uses attacker-controlled `data.ClassId`:**

```go
isAwayFromOrigin := types.IsAwayFromOrigin(
    packet.GetSourcePort(), packet.GetSourceChannel(), data.ClassId)
```

`IsAwayFromOrigin` simply checks whether `data.ClassId` starts with `sourcePort/sourceChannel/`: [2](#0-1) 

There is no validation that `data.ClassId` was legitimately produced by the local chain's outgoing `createOutgoingPacket`. The attacker fully controls `data.ClassId`.

**Step 2 — Prefix strip recovers the native class ID:**

```go
unprefixedClassID := types.RemoveClassPrefix(
    packet.GetSourcePort(), packet.GetSourceChannel(), data.ClassId)
// RemoveClassPrefix("nft","channel-0","nft/channel-0/nativeClass") = "nativeClass"
``` [3](#0-2) 

**Step 3 — `IBCClassID()` returns the bare native class ID:**

```go
voucherClassID := types.ParseClassTrace("nativeClass").IBCClassID()
// Path="" → returns "nativeClass" directly
``` [4](#0-3) 

**Step 4 — Escrow address is derived from `packet.DestPort/DestChannel` (Chain A's own channel):**

```go
escrowAddress := types.GetEscrowAddress(packet.GetDestPort(), packet.GetDestChannel())
// = GetEscrowAddress("nft", "channel-0")
``` [5](#0-4) 

This is the **same** escrow address where Chain A locked the native NFT when it originally sent it to Chain B. [6](#0-5) 

**Step 5 — `TransferOwner` succeeds because the escrow address legitimately owns the NFT:**

```go
k.nftKeeper.TransferOwner(ctx, "nativeClass", tokenID, escrowAddress, receiver)
``` [7](#0-6) 

`TransferOwner` calls `IsOwner` which verifies `srcOwner` owns the token — and it does, because Chain A escrowed it there: [8](#0-7) 

**Step 6 — `ValidateBasic` does not guard against crafted `ClassId`:**

`ValidateBasic` only checks that `ClassId` is non-empty. It performs no structural or prefix validation: [9](#0-8) 

---

### Impact Explanation

An attacker controlling a counterparty chain can drain any NFT that Chain A has escrowed into `GetEscrowAddress("nft", "channel-0")` — transferring ownership to an arbitrary attacker-controlled address — without ever having been the original sender of those NFTs. This is unauthorized unescrow of native NFTs, a Critical-severity asset theft.

---

### Likelihood Explanation

The precondition is that Chain B's channel ID to Chain A is also `channel-0`. IBC channel IDs are assigned sequentially starting from 0, so if both chains open their first channel to each other, both receive `channel-0`. This is the common case in a two-chain setup and is not an unusual or contrived configuration. The attacker only needs to control the counterparty chain's consensus (e.g., a malicious or compromised chain), which is the standard IBC threat model for cross-chain attacks.

---

### Recommendation

In `processReceivedPacket`, before entering the unescrow branch, verify that the `data.ClassId` was legitimately produced by this chain's own outgoing transfer. Concretely, check that a class trace record exists for the stripped class ID, or — more robustly — validate that `data.ClassId` does **not** start with the local destination port/channel prefix (i.e., `packet.GetDestPort()/packet.GetDestChannel()/`). A crafted `ClassId` of the form `destPort/destChannel/nativeClass` should be rejected because the local chain never prefixes outgoing native class IDs with its own port/channel.

Additionally, the ICS-721 spec requires that a receiving chain must reject packets whose `classId` is prefixed with the receiving chain's own port/channel pair, since that would indicate a spoofed return path.

---

### Proof of Concept

```
Precondition:
  Chain A: port=nft, channel=channel-0, counterparty=(nft, channel-0) on Chain B
  Chain A escrows NFT nativeClass/token1 to GetEscrowAddress("nft","channel-0")
    via SendTransfer → createOutgoingPacket → IsAwayFromOrigin=true → TransferOwner(sender→escrow)

Attack:
  Chain B (attacker-controlled) commits a packet:
    SourcePort    = "nft"
    SourceChannel = "channel-0"   ← Chain B's channel ID (same as Chain A's)
    DestPort      = "nft"
    DestChannel   = "channel-0"   ← Chain A's channel ID
    Data.ClassId  = "nft/channel-0/nativeClass"   ← crafted
    Data.TokenIds = ["token1"]
    Data.Receiver = attacker_address

  IBC core on Chain A:
    ✓ Verifies packet proof against Chain B's light client
    ✓ DestPort/DestChannel match Chain A's channel-0
    ✓ SourcePort/SourceChannel match stored counterparty (nft, channel-0)

  processReceivedPacket on Chain A:
    IsAwayFromOrigin("nft","channel-0","nft/channel-0/nativeClass")
      = !strings.HasPrefix("nft/channel-0/nativeClass","nft/channel-0/")
      = !true = false                          ← unescrow branch taken

    escrowAddress = GetEscrowAddress("nft","channel-0")  ← holds token1
    unprefixedClassID = "nativeClass"
    voucherClassID    = "nativeClass"
    TransferOwner(ctx, "nativeClass", "token1", escrowAddress, attacker_address)
      → IsOwner check passes (escrow owns token1)
      → NFT ownership changed to attacker ✓

Result: attacker_address now owns nativeClass/token1 without authorization.
```

func GetEscrowAddress(portID, channelID string) sdk.AccAddress {
	// a slash is used to create domain separation between port and channel identifiers to
	// prevent address collisions between escrow addresses created for different channels
	contents := fmt.Sprintf("%s/%s", portID, channelID)

	// ADR 028 AddressHash construction
	preImage := []byte(Version)
	preImage = append(preImage, 0)
	preImage = append(preImage, contents...)
	hash := sha256.Sum256(preImage)
	return hash[:20]

func (k Keeper) TransferOwner(
	ctx sdk.Context, denomID, tokenID string, srcOwner, dstOwner sdk.AccAddress,
) error {
	if !k.HasDenomID(ctx, denomID) {
		return sdkerrors.Wrapf(types.ErrInvalidDenom, "denom ID %s not exists", denomID)
	}

	nft, err := k.IsOwner(ctx, denomID, tokenID, srcOwner)
	if err != nil {
		return err
	}

	nft.Owner = dstOwner.String()

	k.setNFT(ctx, denomID, nft)
	k.swapOwner(ctx, denomID, tokenID, srcOwner, dstOwner)
	return nil
}

```go
func (nftpd NonFungibleTokenPacketData) ValidateBasic() error {
	if strings.TrimSpace(nftpd.ClassId) == "" {
		return newsdkerrors.Wrap(ErrInvalidClassID, "classId cannot be blank")
	}

	if len(nftpd.TokenIds) == 0 {
		return newsdkerrors.Wrap(ErrInvalidTokenID, "tokenId cannot be blank")
	}

	if len(nftpd.TokenIds) != len(nftpd.TokenUris) {
		return newsdkerrors.Wrap(ErrInvalidPacket, "tokenIds and tokenUris lengths do not match")
	}

	if strings.TrimSpace(nftpd.Sender) == "" {
		return newsdkerrors.Wrap(sdkerrors.ErrInvalidAddress, "sender address cannot be blank")
	}

	// decode the sender address
	if _, err := sdk.AccAddressFromBech32(nftpd.Sender); err != nil {
		return newsdkerrors.Wrap(sdkerrors.ErrInvalidAddress, "invalid sender address")
	}

	if strings.TrimSpace(nftpd.Receiver) == "" {
		return newsdkerrors.Wrap(sdkerrors.ErrInvalidAddress, "receiver address cannot be blank")
	}

	// decode the receiver address
	if _, err := sdk.AccAddressFromBech32(nftpd.Receiver); err != nil {
		return newsdkerrors.Wrap(sdkerrors.ErrInvalidAddress, "invalid receiver address")
	}
	return nil
}
