### Title
Missing Upper-Bound Validation on `protocol_reward_rate` Causes Deterministic Panic in `calculate_reward` at Every Epoch Boundary — (`File: core/chain-configs/src/genesis_validate.rs`)

---

### Summary

`genesis_validate.rs` validates that `online_max_threshold ≤ 1` and `gas_price_adjustment_rate < 1`, but performs **no upper-bound check** on `protocol_reward_rate`. If `protocol_reward_rate > 1` is present in `genesis.json`, the node passes validation, starts successfully, and then panics with an unwrap on `None` at every epoch boundary inside `calculate_reward`, halting the chain permanently.

---

### Finding Description

`GenesisConfig` carries a `protocol_reward_rate: Rational32` field that controls what fraction of the epoch's total inflation reward is sent to the protocol treasury. [1](#0-0) 

`genesis_validate.rs` enforces upper-bound invariants on several similar ratio fields:

- `online_max_threshold` must be `≤ 1`: [2](#0-1) 

- `gas_price_adjustment_rate` must be `< 1`: [3](#0-2) 

However, **no analogous check exists for `protocol_reward_rate`**. The entire `validate_processed_records` function ends at line 192 without ever testing `protocol_reward_rate`: [4](#0-3) 

At every epoch boundary, `RewardCalculator::calculate_reward` computes:

```rust
let epoch_protocol_treasury = Balance::from_yoctonear(
    (U256::from(epoch_total_reward.as_yoctonear())
        * U256::from(*protocol_reward_rate.numer() as u64)
        / U256::from(*protocol_reward_rate.denom() as u64))
    .as_u128(),
);
// ...
let epoch_validator_reward =
    epoch_total_reward.checked_sub(epoch_protocol_treasury).unwrap();  // ← PANIC
``` [5](#0-4) 

When `protocol_reward_rate > 1`, `epoch_protocol_treasury > epoch_total_reward`, so `checked_sub` returns `None` and `.unwrap()` panics. This panic fires inside `finalize_epoch`, which is called at every epoch boundary: [6](#0-5) 

The hardcoded bypass that substitutes `1/10` only applies to mainnet (where `genesis_protocol_version == PROD_GENESIS_PROTOCOL_VERSION == 29`). For every other chain — testnet, localnet, or any custom deployment — the configured value is used verbatim: [7](#0-6) 

---

### Impact Explanation

A `genesis.json` with `"protocol_reward_rate": [11, 10]` (i.e., 1.1) passes `validate_genesis` without error, the node starts, and then panics unconditionally at the first epoch boundary inside `finalize_epoch`. Because `finalize_epoch` is called deterministically by every node on the network, **all nodes crash simultaneously**, producing a permanent chain halt. The chain cannot recover without a genesis restart.

**Impact: High** — deterministic chain halt at first epoch boundary; no recovery path without operator intervention.

---

### Likelihood Explanation

The `protocol_reward_rate` field is operator-supplied via `genesis.json`. The validation framework is explicitly designed to catch such misconfigurations (it already guards `online_max_threshold` and `gas_price_adjustment_rate`). The missing check is an inconsistency in the validation layer, not a deliberate omission. Any operator creating a non-mainnet chain (localnet, testnet fork, custom appchain) who sets `protocol_reward_rate` to a value `> 1` — whether by typo, copy-paste error, or misunderstanding of the field semantics — will trigger the panic. The field is a `Rational32` serialized as a two-element JSON array `[numer, denom]`, making it easy to accidentally invert the fraction (e.g., `[10, 1]` instead of `[1, 10]`).

**Likelihood: Low** — requires a misconfigured genesis on a non-mainnet chain, but the validation framework's own inconsistency is the root cause.

---

### Recommendation

Add an upper-bound check for `protocol_reward_rate` in `validate_processed_records` inside `core/chain-configs/src/genesis_validate.rs`, mirroring the existing checks for `online_max_threshold` and `gas_price_adjustment_rate`:

```rust
if self.genesis_config.protocol_reward_rate > Rational32::from_integer(1) {
    let error_message = format!(
        "Protocol reward rate must be less or equal than 1, but current value is {}",
        self.genesis_config.protocol_reward_rate
    );
    self.validation_errors.push_genesis_semantics_error(error_message)
}
```

Additionally, consider adding overflow guards on the numerator and denominator of `protocol_reward_rate` analogous to those already present for `online_min_threshold` and `online_max_threshold`. [8](#0-7) 

---

### Proof of Concept

1. Create a `genesis.json` for a non-mainnet chain with `"protocol_reward_rate": [11, 10]` (value = 1.1 > 1).
2. Run `validate_genesis` — it returns `Ok(())` because no check exists for `protocol_reward_rate`.
3. Start the node. It initializes successfully.
4. Wait for the first epoch boundary. `EpochManager::finalize_epoch` calls `RewardCalculator::calculate_reward`.
5. Inside `calculate_reward`:
   - `epoch_total_reward` = some positive value `T`
   - `epoch_protocol_treasury` = `T * 11 / 10` > `T`
   - `epoch_total_reward.checked_sub(epoch_protocol_treasury)` returns `None`
   - `.unwrap()` panics: `called Option::unwrap() on a None value`
6. Every node on the network panics at the same block height. The chain halts permanently. [9](#0-8) [10](#0-9)

                    let error_message = format!("validator staking key is not valid");
                    self.validation_errors.push_genesis_semantics_error(error_message);
                }
                (account_info.account_id, account_info.amount)
            })
            .collect::<HashMap<_, _>>();

        if validators.len() != self.genesis_config.validators.len() {
            let error_message = format!(
                "Duplicate account in validators. The number of account_ids: {} does not match the number of validators: {}.",
                self.account_ids.len(),
                validators.len()
            );
            self.validation_errors.push_genesis_semantics_error(error_message)
        }

        if validators.is_empty() {
            let error_message = format!("No validators in genesis");
            self.validation_errors.push_genesis_semantics_error(error_message)

            );
            self.validation_errors.push_genesis_semantics_error(error_message)

        if validators != self.staked_accounts {
            let error_message = format!("Validator accounts do not match staked accounts.");
            self.validation_errors.push_genesis_semantics_error(error_message)
        }

        for account_id in &self.access_key_account_ids {
            if !self.account_ids.contains(account_id) {
                let error_message = format!("access key account {} does not exist", account_id);
                self.validation_errors.push_genesis_semantics_error(error_message)
            }
        }

        for account_id in &self.contract_account_ids {
            if !self.account_ids.contains(account_id) {
                let error_message = format!("contract account {} does not exist,", account_id);
                self.validation_errors.push_genesis_semantics_error(error_message)
            }
        }

        if self.genesis_config.online_max_threshold <= self.genesis_config.online_min_threshold {
            let error_message = format!(
                "Online max threshold {} smaller than min threshold {}",
                self.genesis_config.online_max_threshold, self.genesis_config.online_min_threshold
            );
            self.validation_errors.push_genesis_semantics_error(error_message)
        }

        if self.genesis_config.online_max_threshold > Rational32::from_integer(1) {
            let error_message = format!(
                "Online max threshold must be less or equal than 1, but current value is {}",

            );
            self.validation_errors.push_genesis_semantics_error(error_message)
        }

        if *self.genesis_config.online_max_threshold.numer() >= 10_000_000 {
            let error_message =
                format!("online_max_threshold's numerator is too large, may lead to overflow.");
            self.validation_errors.push_genesis_semantics_error(error_message)
        }

        if *self.genesis_config.online_min_threshold.numer() >= 10_000_000 {
            let error_message =
                format!("online_min_threshold's numerator is too large, may lead to overflow.");
            self.validation_errors.push_genesis_semantics_error(error_message)
        }

        if *self.genesis_config.online_max_threshold.denom() >= 10_000_000 {
            let error_message =
                format!("online_max_threshold's denominator is too large, may lead to overflow.");
            self.validation_errors.push_genesis_semantics_error(error_message)
        }

        if *self.genesis_config.online_min_threshold.denom() >= 10_000_000 {
            let error_message =
                format!("online_min_threshold's denominator is too large, may lead to overflow.");
            self.validation_errors.push_genesis_semantics_error(error_message)
        }

        if self.genesis_config.gas_price_adjustment_rate >= Rational32::from_integer(1) {
            let error_message = format!(

                self.genesis_config.gas_price_adjustment_rate

            self.validation_errors.push_genesis_semantics_error(error_message)

        if self.genesis_config.epoch_length == 0 {
            let error_message = format!("Epoch Length must be greater than 0");
            self.validation_errors.push_genesis_semantics_error(error_message)
