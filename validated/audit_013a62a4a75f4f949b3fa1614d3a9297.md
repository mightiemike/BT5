### Title
Deprecated `verify_transaction_inclusion` Remains Fully Callable On-Chain, Bypassing 64-Byte Merkle Proof Forgery Protection — (`contract/src/lib.rs`)

---

### Summary

`verify_transaction_inclusion` is marked `#[deprecated]` in Rust to signal that callers should use `verify_transaction_inclusion_v2` instead. However, the `#[deprecated]` attribute is a compile-time hint only — it has zero runtime enforcement. The function remains a fully exported NEAR contract method callable by any unprivileged NEAR account. Any caller can invoke v1 directly, bypassing the coinbase Merkle proof check that v2 adds to mitigate the 64-byte transaction Merkle proof forgery vulnerability.

---

### Finding Description

The contract documents a known vulnerability in `verify_transaction_inclusion` (v1): it may return `true` for an internal Merkle tree node hash rather than a real transaction hash (the 64-byte transaction attack). To fix this, `verify_transaction_inclusion_v2` was introduced, which first validates a coinbase Merkle proof anchoring the proof to a real transaction at index 0. [1](#0-0) 

The `#[deprecated]` attribute is a Rust compiler annotation. When the contract is compiled to WASM and deployed on NEAR, the function is still exported as a public contract method. The NEAR runtime has no concept of Rust deprecation — it simply sees a callable function named `verify_transaction_inclusion`. [2](#0-1) 

`verify_transaction_inclusion_v2` enforces the coinbase proof check before delegating to v1: [3](#0-2) 

But any NEAR caller can skip v2 entirely and call v1 directly. The only guard on v1 is `#[pause]` (contract-wide pause), which is not engaged under normal operation. There is no access-control role check, no `#[private]` annotation, and no runtime flag that enforces the deprecation. [4](#0-3) 

The contract's own documentation acknowledges the risk: [5](#0-4) 

---

### Impact Explanation

**Impact: Medium.** A downstream contract or off-chain verifier that calls `verify_transaction_inclusion` (v1) directly — or that an attacker tricks into doing so — receives no protection against the 64-byte transaction Merkle proof forgery. An attacker can craft a 64-byte transaction whose hash equals an internal Merkle tree node, then supply a valid Merkle path for that internal node. v1 will return `true`, falsely asserting that a non-existent transaction is included in a Bitcoin block. Any protocol that gates value transfer or state transitions on this return value (e.g., a bridge contract calling `verify_transaction_inclusion`) can be defrauded.

---

### Likelihood Explanation

**Likelihood: High.** The function is unconditionally reachable by any NEAR account with no staking, role, or deposit requirement beyond the standard NEAR gas fee. The attack path requires only knowledge of the Bitcoin Merkle tree structure and the ability to construct a 64-byte transaction. The deprecation notice provides no on-chain barrier whatsoever.

---

### Recommendation

Remove the public export of `verify_transaction_inclusion` or enforce the deprecation at runtime. Concretely:

1. **Preferred**: Add `#[private]` to `verify_transaction_inclusion` so it can only be called by the contract itself (i.e., only via `verify_transaction_inclusion_v2`'s internal `#[allow(deprecated)]` call). This preserves the internal delegation while closing the external entry point.
2. **Alternative**: Delete the public `verify_transaction_inclusion` method entirely and inline its logic into `verify_transaction_inclusion_v2`.

---

### Proof of Concept

Any NEAR account can call v1 directly, bypassing the coinbase proof check:

```bash
near call <contract_id> verify_transaction_inclusion \
  --args-borsh <ProofArgs with internal_node_hash as tx_id> \
  --accountId attacker.near
```

The call succeeds (returns `true`) for a crafted internal Merkle node hash, while the same `tx_id` would be rejected by `verify_transaction_inclusion_v2` because the attacker cannot produce a valid coinbase Merkle proof anchoring the proof to a real transaction. [6](#0-5) [7](#0-6)

    #[pause]
    pub fn verify_transaction_inclusion(&self, #[serializer(borsh)] args: ProofArgs) -> bool {
        require!(
            args.confirmations <= self.gc_threshold,
            "The required number of confirmations exceeds the number of blocks stored in memory"
        );

        let heaviest_block_header = self
            .headers_pool
            .get(&self.mainchain_tip_blockhash)
            .unwrap_or_else(|| env::panic_str(ERR_KEY_NOT_EXIST));
        let target_block_height = self
            .mainchain_header_to_height
            .get(&args.tx_block_blockhash)
            .unwrap_or_else(|| env::panic_str("block does not belong to the current main chain"));

        // Check requested confirmations. No need to compute proof if insufficient confirmations.
        require!(
            (heaviest_block_header.block_height).saturating_sub(target_block_height) + 1
                >= args.confirmations,
            "Not enough blocks confirmed"
        );

        let header = self
            .headers_pool
            .get(&args.tx_block_blockhash)
            .unwrap_or_else(|| env::panic_str("cannot find requested transaction block"));

        require!(!args.merkle_proof.is_empty(), "Merkle proof is empty");

        // compute merkle tree root and check if it matches block's original merkle tree root
        merkle_tools::compute_root_from_merkle_proof(
            args.tx_id,
            usize::try_from(args.tx_index).unwrap(),
            &args.merkle_proof,
        ) == header.block_header.merkle_root
    }

    #[pause]
    pub fn verify_transaction_inclusion_v2(&self, #[serializer(borsh)] args: ProofArgsV2) -> bool {
        require!(
            args.merkle_proof.len() == args.coinbase_merkle_proof.len(),
            "Coinbase merkle proof and transaction merkle proof should have the same length"
        );

        let header = self
            .headers_pool
            .get(&args.tx_block_blockhash)
            .unwrap_or_else(|| env::panic_str("cannot find requested transaction block"));

        require!(
            merkle_tools::compute_root_from_merkle_proof(
                args.coinbase_tx_id.clone(),
                0usize,
                &args.coinbase_merkle_proof,
            ) == header.block_header.merkle_root,
            "Incorrect coinbase merkle proof"
        );

        #[allow(deprecated)]
        self.verify_transaction_inclusion(args.into())
    }
