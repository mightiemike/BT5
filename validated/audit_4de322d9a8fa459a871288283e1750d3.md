### Title
`SwapAllowlistExtension` gates the router address instead of the actual user, allowing any unprivileged caller to bypass the swap allowlist via `MetricOmmSimpleRouter` — (`metric-periphery/contracts/extensions/SwapAllowlistExtension.sol`)

---

### Summary

`SwapAllowlistExtension.beforeSwap` checks the `sender` argument, which the pool sets to `msg.sender` of the `pool.swap()` call. When a user routes through `MetricOmmSimpleRouter`, that `msg.sender` is the router contract, not the actual user. A pool admin who allowlists the router to enable router-mediated swaps for legitimate users simultaneously opens the gate to every unprivileged caller, because the router is a public, permissionless contract.

---

### Finding Description

`MetricOmmPool.swap` passes its own `msg.sender` as the first argument to `_beforeSwap`: [1](#0-0) 

`ExtensionCalling._beforeSwap` forwards that value as the `sender` argument to every configured extension: [2](#0-1) 

`SwapAllowlistExtension.beforeSwap` then checks whether that `sender` is allowlisted: [3](#0-2) 

When a user calls `MetricOmmSimpleRouter.exactInputSingle`, the router calls `pool.swap(params.recipient, ...)` directly: [4](#0-3) 

At that point `msg.sender` inside the pool is the **router**, so the extension receives `sender = router` and evaluates `allowedSwapper[pool][router]`. The actual user who called the router is never inspected.

This creates an irresolvable dilemma for any pool admin who wants to gate individual users:

| Admin choice | Effect |
|---|---|
| Do **not** allowlist the router | Allowlisted users cannot use the router at all |
| Allowlist the router | Every unprivileged user can bypass the individual allowlist via the router |

Because `MetricOmmSimpleRouter` is a public, permissionless contract, allowlisting it is equivalent to setting `allowAllSwappers[pool] = true`.

---

### Impact Explanation

Any user who is explicitly **not** in the allowlist can execute swaps on a curated pool by routing through `MetricOmmSimpleRouter`, provided the pool admin has allowlisted the router (the expected configuration for pools that want to support the standard periphery). The allowlist guard is rendered completely ineffective for router-mediated swaps, which is the primary user-facing entry point. This is a direct policy bypass on curated pools and constitutes broken core pool functionality.

---

### Likelihood Explanation

The `MetricOmmSimpleRouter` is the standard, documented swap entry point for end users. Any pool admin who deploys a `SwapAllowlistExtension` and also wants users to be able to use the router must allowlist the router. This is the natural and expected configuration, making the bypass reachable by any unprivileged caller without any special setup.

---

### Recommendation

The extension must resolve the actual user identity rather than the immediate pool caller. Two approaches:

1. **Pass the original caller through `extensionData`**: The router encodes `msg.sender` into `extensionData`; the extension decodes and checks it. This requires a coordinated change to the router and extension.

2. **Check `sender` against the router and fall back to a user-identity field**: The extension recognises known router addresses and reads the true originator from a transient or calldata field set by the router before calling the pool.

The simplest correct fix is for `SwapAllowlistExtension` to be documented as incompatible with the router for per-user gating, and for a new extension variant to be introduced that reads the true originator from a router-supplied `extensionData` field.

---

### Proof of Concept

```solidity
// SPDX-License-Identifier: UNLICENSED
pragma solidity ^0.8.35;

import {Test} from "forge-std/Test.sol";
import {MetricOmmSimpleRouter} from "metric-periphery/contracts/MetricOmmSimpleRouter.sol";
import {SwapAllowlistExtension} from "metric-periphery/contracts/extensions/SwapAllowlistExtension.sol";

contract SwapAllowlistBypassTest is Test {
    // Assume pool, router, extension, factory are deployed and configured.
    // Pool has SwapAllowlistExtension with only `allowedUser` in the allowlist.
    // Admin also allowlists the router so that `allowedUser` can use it.

    address pool;
    MetricOmmSimpleRouter router;
    SwapAllowlistExtension extension;
    address allowedUser  = makeAddr("allowed");
    address attackerUser = makeAddr("attacker");

    function test_bypassAllowlistViaRouter() public {
        // Admin allowlists the router so allowedUser can swap through it.
        vm.prank(poolAdmin);
        extension.setAllowedToSwap(pool, address(router), true);

        // attacker (not individually allowlisted) swaps through the router.
        // The extension sees sender = router (allowlisted) and passes.
        vm.prank(attackerUser);
        router.exactInputSingle(
            IMetricOmmSimpleRouter.ExactInputSingleParams({
                pool: pool,
                recipient: attackerUser,
                zeroForOne: true,
                amountIn: 1000,
                amountOutMinimum: 0,
                priceLimitX64: 0,
                tokenIn: token0,
                deadline: block.timestamp + 1,
                extensionData: ""
            })
        );
        // Swap succeeds — attacker bypassed the per-user allowlist.
    }
}
```
