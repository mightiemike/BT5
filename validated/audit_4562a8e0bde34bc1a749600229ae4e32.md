### Title
SwapAllowlistExtension Checks Router Address Instead of Actual User, Allowing Any User to Bypass Swap Allowlist via MetricOmmSimpleRouter — (`metric-periphery/contracts/extensions/SwapAllowlistExtension.sol`)

---

### Summary

`SwapAllowlistExtension.beforeSwap` gates swaps by checking the `sender` argument it receives from the pool. When a user swaps through `MetricOmmSimpleRouter`, the pool's `msg.sender` is the **router**, not the actual user. The extension therefore checks whether the router is allowlisted, not the real economic actor. If the pool admin allowlists the router (the natural action to let legitimate users use the official periphery), every non-allowlisted user can bypass the restriction by routing through the router.

---

### Finding Description

**Pool → Extension call chain**

`MetricOmmPool.swap()` passes `msg.sender` as the `sender` argument to `_beforeSwap`: [1](#0-0) 

`ExtensionCalling._beforeSwap` forwards that value unchanged as the first positional argument to every configured extension: [2](#0-1) 

`SwapAllowlistExtension.beforeSwap` then checks `allowedSwapper[msg.sender][sender]`, where `msg.sender` is the pool and `sender` is whatever the pool received as its own `msg.sender`: [3](#0-2) 

**Router path**

`MetricOmmSimpleRouter.exactInputSingle` calls `pool.swap(params.recipient, ...)` directly. The router is `msg.sender` of that call; the actual user who called the router is never forwarded: [4](#0-3) 

The same applies to `exactInput` (intermediate hops use `address(this)` as payer), `exactOutputSingle`, and `exactOutput`: [5](#0-4) 

**Result**: the extension's allowlist lookup is keyed on the router's address, not the user's address.

---

### Impact Explanation

A pool admin who deploys a curated pool with `SwapAllowlistExtension` (e.g., for KYC-gated or institutional-only trading) must choose between two broken states:

| Admin action | Effect |
|---|---|
| Allowlist the router | Every non-allowlisted user bypasses the gate by calling the router |
| Do not allowlist the router | Allowlisted users cannot use the official periphery at all |

In the first case, any address can call `MetricOmmSimpleRouter.exactInputSingle` and trade on the curated pool, defeating the access control entirely. The pool receives tokens from and sends tokens to non-allowlisted counterparties, which is the direct fund-impacting consequence the allowlist was designed to prevent.

---

### Likelihood Explanation

`MetricOmmSimpleRouter` is the primary user-facing swap interface. Pool admins who want their allowlisted users to have a normal UX must allowlist the router. The bypass is therefore reachable by any address with no special privilege, no malicious setup, and no non-standard token behavior — just a call to the public router.

---

### Recommendation

The `sender` argument the extension receives must represent the **economic actor**, not the intermediary. Two viable fixes:

1. **Router forwards caller identity via `extensionData`**: The router encodes `msg.sender` into `extensionData` and the extension decodes and checks it. This requires a coordinated convention between router and extension.

2. **Check `recipient` instead of `sender`**: For swap allowlists the recipient is often the correct identity to gate (the party receiving value). This is a simpler change but may not match all use-cases.

3. **Enforce allowlist at the router level**: The router itself checks whether `msg.sender` is allowed before calling the pool, and the pool-level extension is removed or made redundant. This keeps the check at the layer that knows the real user.

---

### Proof of Concept

```
Setup:
  pool configured with SwapAllowlistExtension
  alice  → allowedSwapper[pool][alice] = true   (KYC'd user)
  bob    → allowedSwapper[pool][bob]   = false  (non-KYC'd)
  router → allowedSwapper[pool][router]= true   (admin allowlists router so alice can use it)

Direct call (works as intended):
  bob calls pool.swap() directly
  → pool._beforeSwap(sender=bob, ...)
  → extension checks allowedSwapper[pool][bob] = false → revert NotAllowedToSwap ✓

Router call (bypass):
  bob calls router.exactInputSingle({pool: pool, ...})
  → router calls pool.swap(recipient=bob, ...)   // router is msg.sender
  → pool._beforeSwap(sender=router, ...)
  → extension checks allowedSwapper[pool][router] = true → passes ✓ (bypass)
  → bob receives tokens from the curated pool
```

The root cause is in `SwapAllowlistExtension.beforeSwap` at line 37: `allowedSwapper[msg.sender][sender]` where `sender` is the router, not the user. [6](#0-5)
