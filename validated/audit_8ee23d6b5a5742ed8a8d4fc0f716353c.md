### Title
`MsgIssueDenom` Can Be Frontrun to Steal NFT Denom Creator Role - (File: `x/nft/keeper/msg_server.go`)

### Summary

`MsgIssueDenom` is open to any account with no whitelist or access control. The `sender` of the transaction is permanently recorded as the `creator` of the denom. Because the `denom_id` is a user-supplied string that is visible in the mempool before inclusion, a malicious actor can observe a legitimate user's pending `MsgIssueDenom` transaction, submit an identical one with the same `denom_id` but their own address as `sender`, and — if included first — permanently seize the creator role for that denom ID.

### Finding Description

`MsgIssueDenom` in `x/nft/keeper/msg_server.go` has no access control:

```go

The `creator` field is set from the transaction sender at denom construction time:

```go

There is no whitelist, no signature-over-sender check, and no mechanism binding the `denom_id` to the originating address. The `ValidateBasic` check on `MsgIssueDenom` only validates format, not authorization: [5](#0-4) 

### Impact Explanation

The corrupted value is `denom.Creator` — the address stored as the permanent creator of the NFT denom. Once stolen, the attacker holds exclusive minting rights over the denom. The legitimate user's `MsgIssueDenom` transaction fails with `"denomID X has already exists"`, permanently blocking them from that denom ID. The attacker can then mint arbitrary NFTs under the stolen denom ID, impersonating the legitimate collection and issuing NFTs to any recipient. [6](#0-5) 

### Likelihood Explanation

Cosmos SDK mempools are publicly observable. Any node operator or RPC user can subscribe to unconfirmed transactions. The `denom_id` is a plain string in the transaction body. A frontrunner needs only to copy the `denom_id` and `denom_name` fields, substitute their own address as `sender`, and submit with a higher gas price (or rely on validator ordering). This requires no privileged access, no leaked keys, and no social engineering — only a standard signed transaction from any funded account. [7](#0-6) 

### Recommendation

1. **Bind `denom_id` to the creator address**: Derive the denom ID deterministically from the sender's address (e.g., `hash(sender || sequence)`) so it cannot be predicted and replicated by a third party.
2. **Whitelist-based creation**: Restrict `MsgIssueDenom` to a governance-approved set of addresses.
3. **Signature commitment**: Require the sender to sign a commitment over the `denom_id` and their address, and verify it on-chain, analogous to the OpenZeppelin ECDSA approach referenced in the original report.

### Proof of Concept

1. Alice broadcasts `MsgIssueDenom{Id: "my-collection", Name: "My NFTs", Sender: alice}`.
2. Mallory observes Alice's transaction in the mempool via any public RPC node.
3. Mallory broadcasts `MsgIssueDenom{Id: "my-collection", Name: "My NFTs", Sender: mallory}` with higher gas, ensuring earlier inclusion.
4. `SetDenom` stores `mallory` as `creator` of `"my-collection"`.
5. Alice's transaction is processed next and fails: `"denomID my-collection has already exists"`.
6. Mallory now passes `IsDenomCreator` checks and can call `MintNFT` under `"my-collection"`, minting NFTs to any recipient under Alice's intended denom identity. [8](#0-7) [2](#0-1)

func (k Keeper) SetDenom(ctx sdk.Context, denom types.Denom) error {
	if k.HasDenomID(ctx, denom.Id) {
		return sdkerrors.Wrapf(types.ErrInvalidDenom, "denomID %s has already exists", denom.Id)
	}

	if k.HasDenomNm(ctx, denom.Name) {
		return sdkerrors.Wrapf(types.ErrInvalidDenom, "denomName %s has already exists", denom.Name)
	}

	store := ctx.KVStore(k.storeKey)
	bz := k.cdc.MustMarshal(&denom)
	store.Set(types.KeyDenomID(denom.Id), bz)
	store.Set(types.KeyDenomName(denom.Name), []byte(denom.Id))
	return nil

func (k Keeper) IsDenomCreator(ctx sdk.Context, denomID string, address sdk.AccAddress) (types.Denom, error) {
	denom, err := k.GetDenom(ctx, denomID)
	if err != nil {
		return types.Denom{}, err

	creator, err := sdk.AccAddressFromBech32(denom.Creator)

	return denom, nil
}
