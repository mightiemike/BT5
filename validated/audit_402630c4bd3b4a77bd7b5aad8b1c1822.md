## Verdict

### Title
`FeeEstimation.overall_fee` includes L2 gas cost that clients cannot reconstruct from returned fields ‚Äî (`crates/apollo_rpc_execution/src/objects.rs`)

### Summary
`tx_execution_output_to_fee_estimation` sets `overall_fee = receipt.fee`, which includes `l2_gas * (l2_gas_price + tip)`. However, `l2_gas_consumed` is not a field in `FeeEstimation`, so clients receive an `overall_fee` they cannot reconstruct from the other returned fields (`gas_consumed`, `l1_gas_price`, `data_gas_consumed`, `l1_data_gas_price`). The struct's own doc comment states the invariant `overall_fee = gas_consumed * gas_price + data_gas_consumed * data_gas_price`, which is concretely violated for any V3 transaction with `AllResourceBounds` under StarknetVersion ‚â• 0.14.0.

### Finding Description
In `tx_execution_output_to_fee_estimation`:

```rust
Ok(FeeEstimation {
    gas_consumed: gas_vector.l1_gas.0.into(),       // L1 gas only
    data_gas_consumed: gas_vector.l1_data_gas.0.into(), // L1 data gas only
    // l2_gas_consumed: MISSING
    overall_fee: tx_execution_output.execution_info.receipt.fee, // includes l2_gas cost
    ...
})
```

`receipt.fee` is computed as:
```
l1_gas * l1_gas_price + l1_data_gas * l1_data_gas_price + l2_gas * (l2_gas_price + tip)
```

For V3 transactions with `AllResourceBounds`, `GasVectorComputationMode::All` routes all computation into `l2_gas`. The `l2_gas` component is nonzero and its cost is included in `overall_fee`, but `l2_gas_consumed` is not returned. The `l2_gas_price` field is returned but is useless without the consumed amount.

### Impact Explanation
Any client (wallet, SDK, dApp) that calls `estimate_fee` or `simulate_transactions` and attempts to verify or reconstruct `overall_fee` from the other returned fields will compute a value that is lower than `overall_fee` by exactly `l2_gas * (l2_gas_price + tip)`. For production blocks at StarknetVersion 0.14.4, `l2_gas` values of ~76 million units at `l2_gas_price = 0x1dcd65000` (8 Gwei) produce fees in the range of hundreds of millions of Fri ‚Äî a material discrepancy. This meets the "High" impact criterion: RPC fee estimation returns an authoritative-looking value that clients cannot reconstruct from the other returned fields.

### Likelihood Explanation
This is triggered by any V3 `InvokeTransaction` with `AllResourceBounds` on a node running StarknetVersion ‚â• 0.14.0. Production data confirms this is the normal operating mode. No special privileges are required ‚Äî any user calling `starknet_estimateFee` or `starknet_simulateTransactions` is affected.

### Recommendation
Add `l2_gas_consumed: Felt` to `FeeEstimation` and populate it from `gas_vector.l2_gas.0.into()` in `tx_execution_output_to_fee_estimation`. Update the doc comment to reflect the correct invariant: `overall_fee = gas_consumed * l1_gas_price + data_gas_consumed * l1_data_gas_price + l2_gas_consumed * (l2_gas_price + tip)`. The existing TODO at line 104 already tracks this work.

### Proof of Concept
1. Submit a V3 `InvokeTransaction` with `AllResourceBounds` to `estimate_fee` on a node at StarknetVersion ‚â• 0.14.0.
2. Receive `FeeEstimation { gas_consumed, l1_gas_price, data_gas_consumed, l1_data_gas_price, l2_gas_price, overall_fee }`.
3. Compute `reconstructed = gas_consumed * l1_gas_price + data_gas_consumed * l1_data_gas_price`.
4. Assert `overall_fee > reconstructed`.
5. The difference equals `l2_gas * (l2_gas_price + tip)`, which is unrecoverable from the returned fields because `l2_gas_consumed` is absent. [8](#0-7) [9](#0-8) [10](#0-9)

pub(crate) fn tx_execution_output_to_fee_estimation(
    tx_execution_output: &TransactionExecutionOutput,
    block_context: &BlockContext,
) -> ExecutionResult<FeeEstimation> {
    let gas_prices = &block_context.block_info().gas_prices;
    let (l1_gas_price, l1_data_gas_price, l2_gas_price) = (
        gas_prices.l1_gas_price(&tx_execution_output.price_unit.into()).get(),
        gas_prices.l1_data_gas_price(&tx_execution_output.price_unit.into()).get(),
        gas_prices.l2_gas_price(&tx_execution_output.price_unit.into()).get(),
    );

        gas_consumed: gas_vector.l1_gas.0.into(),

        l1_data_gas_price,
        l2_gas_price,
        overall_fee: tx_execution_output.execution_info.receipt.fee,
        unit: tx_execution_output.price_unit,
    })
}

    /// Computes the cost (in fee token units) of the gas vector (panicking on overflow).
    pub fn cost(&self, gas_prices: &GasPriceVector, tip: Tip) -> Fee {
        let tipped_l2_gas_price =
            gas_prices.l2_gas_price.checked_add(tip.into()).unwrap_or_else(|| {
                panic!(
                    "Tip overflowed: addition of L2 gas price ({}) and tip ({}) resulted in \
                     overflow.",
                    gas_prices.l2_gas_price, tip
                )
            });

        let mut sum = Fee(0);
        for (gas, price, resource) in [
            (self.l1_gas, gas_prices.l1_gas_price, Resource::L1Gas),
            (self.l1_data_gas, gas_prices.l1_data_gas_price, Resource::L1DataGas),
            (self.l2_gas, tipped_l2_gas_price, Resource::L2Gas),
        ] {
            let cost = gas.checked_mul(price.get()).unwrap_or_else(|| {
                panic!(
                    "{resource} cost overflowed: multiplication of gas amount ({gas}) by price \
                     per unit ({price}) resulted in overflow."
                )
            });
            sum = sum.checked_add(cost).unwrap_or_else(|| {
                panic!(
                    "Total cost overflowed: addition of current sum ({sum}) and cost of \
                     {resource} ({cost}) resulted in overflow."
                )
            });
        }
        sum
    }
