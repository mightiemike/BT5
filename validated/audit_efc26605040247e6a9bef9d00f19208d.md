Audit Report

## Title
Router-Mediated Swaps Always Fail SwapAllowlistExtension Per-User Check â€” (`metric-periphery/contracts/extensions/SwapAllowlistExtension.sol`)

## Summary
`SwapAllowlistExtension.beforeSwap` checks `allowedSwapper[pool][sender]` where `sender` is the `msg.sender` of `pool.swap()`. When users route through `MetricOmmSimpleRouter`, the router contract is `msg.sender` to the pool, so the check evaluates the router's allowlist status rather than the originating user's. Pool admins cannot individually allowlist users who use the router; allowlisted users cannot swap through the supported periphery path.

## Finding Description
`MetricOmmPool.swap()` passes `msg.sender` verbatim as the `sender` argument to `_beforeSwap`: [1](#0-0) 

`_beforeSwap` forwards that value unchanged to the extension via `_callExtensionsInOrder`. `SwapAllowlistExtension.beforeSwap` then checks:

```solidity
if (!allowAllSwappers[msg.sender] && !allowedSwapper[msg.sender][sender])
``` [2](#0-1) 

Here `msg.sender` is the pool and `sender` is whoever called `pool.swap()`. Both `exactInputSingle` and `exactInput` in `MetricOmmSimpleRouter` call `pool.swap()` directly: [3](#0-2) [4](#0-3) 

This makes the router the `msg.sender` seen by the pool, so `sender` passed to the extension is the router address, not the originating EOA. No mechanism exists to thread the original caller's identity through the router to the extension. The check `allowedSwapper[pool][router]` fails unless the router itself is allowlisted, which grants access to all router users and defeats per-user curation.

## Impact Explanation
Any pool deploying `SwapAllowlistExtension` with individual user allowlisting is non-functional for the router path. Allowlisted users are permanently blocked from swapping via the supported periphery (`exactInputSingle`, `exactInput`, `exactOutputSingle`, `exactOutput`). The only workaround is calling `pool.swap()` directly, bypassing the router entirely. This constitutes broken core swap functionality for the intended beneficiaries of the allowlist â€” a user who has been explicitly granted swap rights cannot exercise them through the standard interface. [5](#0-4) 

## Likelihood Explanation
Triggered by normal, expected usage of the supported periphery. Any pool that configures `SwapAllowlistExtension` with per-user entries and expects users to use the router will exhibit this failure on every router-mediated swap attempt. No special attacker capability is required â€” it is reproducible by any allowlisted user calling the router. [6](#0-5) 

## Recommendation
The extension should verify the original initiator via a trusted channel. One approach: the router encodes `msg.sender` into `extensionData` for each hop, and the extension decodes and checks that address only when the direct caller (`sender`) is a known trusted router. Alternatively, the pool could expose a `swapWithOriginator` entry point that passes the original caller explicitly, or the allowlist documentation must clearly state that only direct `pool.swap()` callers can be individually allowlisted and the router cannot be used. [7](#0-6) 

## Proof of Concept
1. Deploy a pool with `SwapAllowlistExtension` configured in `beforeSwap`.
2. Call `swapExtension.setAllowedToSwap(pool, userEOA, true)` â€” do **not** allowlist the router.
3. `vm.prank(userEOA); router.exactInputSingle(...)` â†’ reverts with `NotAllowedToSwap` because `allowedSwapper[pool][router] == false`.
4. `vm.prank(userEOA); pool.swap(...)` directly â†’ succeeds because `allowedSwapper[pool][userEOA] == true`.

Step 3 proves the invariant is broken: an individually allowlisted user cannot trade through the supported router. [8](#0-7)

contract SwapAllowlistExtension is BaseMetricExtension, ISwapAllowlistExtension {
  mapping(address pool => mapping(address swapper => bool)) public allowedSwapper;
  mapping(address pool => bool) public allowAllSwappers;

  constructor(address factory_) BaseMetricExtension(factory_) {}

  function setAllowedToSwap(address pool_, address swapper, bool allowed) external onlyPoolAdmin(pool_) {
    allowedSwapper[pool_][swapper] = allowed;
    emit AllowedToSwapSet(pool_, swapper, allowed);
  }

  function setAllowAllSwappers(address pool_, bool allowed) external onlyPoolAdmin(pool_) {
    allowAllSwappers[pool_] = allowed;
    emit AllowAllSwappersSet(pool_, allowed);
  }

  function isAllowedToSwap(address pool_, address swapper) external view returns (bool) {
    return allowAllSwappers[pool_] || allowedSwapper[pool_][swapper];
  }

  function beforeSwap(address sender, address, bool, int128, uint128, uint256, uint128, uint128, bytes calldata)
    external
    view
    override
    returns (bytes4)
  {
    if (!allowAllSwappers[msg.sender] && !allowedSwapper[msg.sender][sender]) {
      revert IMetricOmmPoolActions.NotAllowedToSwap();
    }
    return IMetricOmmExtensions.beforeSwap.selector;
  }
```
