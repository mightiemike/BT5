### Title
`ValidResourceBounds` protobuf deserialization uses value-based proxy instead of type discriminator, producing wrong transaction hash for `AllResources` transactions with zero L2/data-gas bounds — (`File: crates/apollo_protobuf/src/converters/transaction.rs`)

### Summary

The `TryFrom<protobuf::ResourceBounds> for ValidResourceBounds` converter reconstructs the `ValidResourceBounds` enum variant by inspecting whether the deserialized `l2_gas` and `l1_data_gas` values are zero, rather than by reading an explicit type tag. This is the direct sequencer analog of the `addInterest` proxy-metric bug: just as `addInterest` gates an interest update on a rate-change proxy instead of the actual share-value change, this converter gates the `AllResources` vs `L1Gas` classification on a value proxy instead of the actual type. A V3 transaction submitted with `AllResources` and zero `l2_gas`/`l1_data_gas` bounds is silently reclassified as `L1Gas` after any protobuf round-trip, producing a different Poseidon transaction hash than the one computed at ingestion time.

### Finding Description

**Root cause — value-based proxy in `TryFrom<protobuf::ResourceBounds> for ValidResourceBounds`** [1](#0-0) 

The condition on line 431 (`l1_data_gas.is_zero() && l2_gas.is_zero()`) is a proxy for the original Rust enum discriminant. The protobuf wire format carries no explicit tag distinguishing `L1Gas` from `AllResources`; the converter infers the variant from the numeric values alone. When a modern V3 transaction is submitted with `AllResourceBounds { l1_gas: X, l2_gas: 0, l1_data_gas: 0 }` — a configuration the gateway explicitly accepts — the serialized protobuf is indistinguishable from a legacy `L1Gas` transaction, and the converter returns `ValidResourceBounds::L1Gas(X)`.

impl TryFrom<protobuf::ResourceBounds> for ValidResourceBounds {
    type Error = ProtobufConversionError;
    fn try_from(value: protobuf::ResourceBounds) -> Result<Self, Self::Error> {
        let Some(l1_gas) = value.l1_gas else {
            return Err(missing("ResourceBounds::l1_gas"));
        };
        let Some(l2_gas) = value.l2_gas else {
            return Err(missing("ResourceBounds::l2_gas"));
        };
        // TODO(Shahak): Assert data gas is not none once we remove support for 0.13.2.
        let l1_data_gas = value.l1_data_gas.unwrap_or_default();
        let l1_gas: ResourceBounds = l1_gas.try_into()?;
        let l2_gas: ResourceBounds = l2_gas.try_into()?;
        let l1_data_gas: ResourceBounds = l1_data_gas.try_into()?;
        Ok(if l1_data_gas.is_zero() && l2_gas.is_zero() {
            ValidResourceBounds::L1Gas(l1_gas)
        } else {
            ValidResourceBounds::AllResources(AllResourceBounds { l1_gas, l2_gas, l1_data_gas })
        })
    }

pub fn get_tip_resource_bounds_hash(
    resource_bounds: &ValidResourceBounds,
    tip: &Tip,
) -> Result<Felt, StarknetApiError> {
    let l1_resource_bounds = resource_bounds.get_l1_bounds();
    let l2_resource_bounds = resource_bounds.get_l2_bounds();

    // L1 and L2 gas bounds always exist.
    // Old V3 txs always have L2 gas bounds of zero, but they exist.
    let mut resource_felts = vec![
        get_concat_resource(&l1_resource_bounds, L1_GAS)?,
        get_concat_resource(&l2_resource_bounds, L2_GAS)?,
    ];

    // For new V3 txs, need to also hash the data gas bounds.
    resource_felts.extend(match resource_bounds {
        ValidResourceBounds::L1Gas(_) => vec![],
        ValidResourceBounds::AllResources(all_resources) => {
            vec![get_concat_resource(&all_resources.l1_data_gas, L1_DATA_GAS)?]
        }
    });

    Ok(HashChain::new().chain(&tip.0.into()).chain_iter(resource_felts.iter()).get_poseidon_hash())
