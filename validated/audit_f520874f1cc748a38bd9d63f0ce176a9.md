Audit Report

## Title
`SwapAllowlistExtension` gates on router address instead of end-user identity, allowing any user to bypass per-pool swap allowlist via `MetricOmmSimpleRouter` - (File: metric-periphery/contracts/extensions/SwapAllowlistExtension.sol)

## Summary

`MetricOmmPool.swap()` passes `msg.sender` — the direct caller — as the `sender` argument to `_beforeSwap`. When `MetricOmmSimpleRouter` is the caller, `sender` is the router's address, not the originating user. `SwapAllowlistExtension.beforeSwap` checks `allowedSwapper[pool][sender]`, so allowlisting the router grants every user behind it unrestricted swap access, defeating the per-pool access control entirely.

## Finding Description

**Pool passes `msg.sender` (the router) as `sender`:**

In `MetricOmmPool.swap()`, `_beforeSwap` is called with `msg.sender` as the first argument: [1](#0-0) 

**Extension checks the router address, not the end user:**

`SwapAllowlistExtension.beforeSwap` evaluates `allowedSwapper[msg.sender][sender]` where `msg.sender` is the pool and `sender` is the router: [2](#0-1) 

**Router calls pool without forwarding the original user:**

`MetricOmmSimpleRouter.exactInputSingle` calls `pool.swap(...)` directly; `msg.sender` to the pool is the router. The original user's address (`msg.sender` inside the router) is stored only in transient callback context for payment purposes and is never forwarded to the pool or extension: [3](#0-2) 

**Forced dilemma:** A pool admin who wants to support router-mediated swaps must call `setAllowedToSwap(pool, router, true)`. Once set, `allowedSwapper[pool][router] = true` for every call arriving through the router, regardless of which end user initiated it: [4](#0-3) 

The same bypass applies to `exactOutputSingle`, `exactInput`, and `exactOutput` — all router entry points call `pool.swap()` with the router as `msg.sender`. [5](#0-4) 

## Impact Explanation

The `SwapAllowlistExtension` is the sole on-chain mechanism for pool admins to restrict who may trade (e.g., KYC-gated, institutional-only, or whitelist-only pools). Once the router is allowlisted, any address can swap in a restricted pool without individual authorization. This is a broken core pool functionality / admin-boundary break: an unprivileged trader bypasses an explicit access-control gate, enabling unauthorized swaps against pool liquidity and violating the pool's intended access model. [6](#0-5) 

## Likelihood Explanation

Any pool admin who deploys a `SwapAllowlistExtension` and also wants users to access the pool through `MetricOmmSimpleRouter` — the primary documented user-facing entry point — will naturally allowlist the router. This is the expected operational pattern. The bypass is reachable on every such pool without any privileged action by the attacker; a non-allowlisted user simply calls the router normally. [7](#0-6) 

## Recommendation

The extension must gate on the actual end-user identity. Two viable approaches:

1. **Pass the original user through `extensionData`:** The router encodes `msg.sender` into `extensionData`; the extension decodes and checks it. This requires a coordinated convention between router and extension but requires no core interface change.

2. **Add a `swapper` field to `beforeSwap`:** Expose both `sender` (direct caller) and `swapper` (original initiator) in the `IMetricOmmExtensions.beforeSwap` signature so extensions can choose which identity to gate. This is a cleaner fix but is a breaking interface change.

Until fixed, pool admins must not simultaneously allowlist the router and restrict individual swappers — the two settings are mutually exclusive under the current design. [2](#0-1) 

## Proof of Concept

```
1. Pool admin deploys pool with SwapAllowlistExtension as beforeSwap hook.
2. Pool admin allowlists the router (required for any router-mediated swap):
       swapExtension.setAllowedToSwap(pool, address(router), true);
3. Pool admin does NOT allowlist Bob:
       allowedSwapper[pool][bob] == false
4. Bob calls router.exactInputSingle({pool: pool, ...}).
5. Router calls pool.swap(recipient, ...) — msg.sender to pool = router.
6. Pool calls _beforeSwap(router, ...) → extension.beforeSwap(router, ...).
7. Extension evaluates:
       allowAllSwappers[pool]        → false
       allowedSwapper[pool][router]  → true  ← router is allowlisted
   → check passes, swap executes.
8. Bob successfully swaps in a pool he was never individually permitted to access.
``` [1](#0-0) [8](#0-7)
