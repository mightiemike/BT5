### Title
Missing `isRedelegating` Guard in `TierUndelegate` Enables Redelegation Slash Escape — (`x/tieredrewards/keeper/msg_validate.go`, `msg_server.go`)

---

### Summary

`validateUndelegatePosition` does not check `isRedelegating`, while every other exit path (`validateExitTierWithDelegation`, `transferDelegationFromPosition`, `validateRedelegatePosition`) does. An attacker can exploit this gap to remove the position's delegation from the destination validator while a redelegation slash window is still open, leaving no delegation for the SDK's `SlashRedelegation` to slash.

---

### Finding Description

**Missing guard.** `validateUndelegatePosition` checks owner, `IsDelegated`, `HasTriggeredExit`, and `CompletedExitLockDuration`, but never calls `isRedelegating`: [1](#0-0) 

Compare with `validateExitTierWithDelegation`, which explicitly blocks the same scenario: [2](#0-1) 

And `transferDelegationFromPosition`, which has a second defensive check: [3](#0-2) 

**Reachable call sequence.**

`validateRedelegatePosition` blocks redelegation only when exit is triggered AND the lock duration has already elapsed: [4](#0-3) 

This means the following sequence is valid:

1. `MsgTriggerExitFromTier` — starts exit lock duration.
2. `MsgTierRedelegate(positionId, valB)` — allowed because lock duration has not yet elapsed; creates a staking redelegation entry from valA → valB and stores `unbondingID → positionId` in `RedelegationMappings`.
3. Advance time past exit lock duration.
4. `MsgTierUndelegate(positionId)` — passes all checks in `validateUndelegatePosition`; calls `stakingKeeper.Undelegate`, which removes the delegation from valB and creates an unbonding delegation entry.
5. valA is slashed for a past infraction (within the 21-day redelegation window).
6. SDK's `SlashRedelegation` fires `BeforeRedelegationSlashed` → `slashRedelegationPosition`. At this point `pos.IsDelegated()` is false (delegation was removed in step 4), so the hook logs an error and returns nil without applying any slash: [5](#0-4) 

7. The SDK's own `SlashRedelegation` also finds no delegation on valB for `posDelAddr` and skips the slash for that entry.
8. Unbonding period completes; `WithdrawFromTier` passes `validateWithdrawFromTier` (position not delegated, not unbonding): [6](#0-5) 

9. `WithdrawFromTier` sends the full unbonded balance to the owner: [7](#0-6) 

The unbonding delegation created in step 4 is a separate staking record from the redelegation entry. It is only subject to slashing if valB (the destination) is slashed, not valA (the source). The redelegation slash from valA therefore never reaches the attacker's tokens.

---

### Impact Explanation

The attacker recovers the full principal that should have been reduced by the redelegation slash fraction. For a double-sign slash (5%) on a large position, this is a direct, quantifiable fund recovery that violates the protocol's slash-accountability invariant. The `isRedelegating` guard exists precisely to prevent this; `TierUndelegate` bypasses it entirely.

---

### Likelihood Explanation

The attacker controls the timing: they trigger exit, redelegate, wait for the exit lock to elapse, then undelegate. The only external dependency is that valA is actually slashed within the 21-day window. An attacker who suspects or knows a validator is about to be slashed (e.g., observing double-sign evidence in the mempool or monitoring validator behavior) can execute this sequence deliberately. No privileged access is required.

---

### Recommendation

Add an `isRedelegating` check to `validateUndelegatePosition`, mirroring the pattern already used in `validateExitTierWithDelegation`:

```go
func (k Keeper) validateUndelegatePosition(ctx context.Context, pos types.PositionState, owner string) error {
    // ... existing checks ...

    isRedelegating, err := k.isRedelegating(ctx, pos.DelegatorAddress)
    if err != nil {
        return err
    }
    if isRedelegating {
        return errorsmod.Wrapf(types.ErrActiveRedelegation,
            "position %d has an active redelegation", pos.Id)
    }

    return nil
}
```

This ensures that `TierUndelegate` is blocked for the same reason `ExitTierWithDelegation` is blocked, closing the bypass.

```go
func (k Keeper) validateUndelegatePosition(ctx context.Context, pos types.PositionState, owner string) error {

}
```

```go
func (k Keeper) validateWithdrawFromTier(ctx context.Context, pos types.PositionState, owner string) error {
	if !pos.IsOwner(owner) {
		return types.ErrNotPositionOwner
	}

	if !pos.HasTriggeredExit() {
		return types.ErrExitNotTriggered
	}

	sdkCtx := sdk.UnwrapSDKContext(ctx)
	if !pos.CompletedExitLockDuration(sdkCtx.BlockTime()) {
		return types.ErrExitLockDurationNotReached
	}

	if pos.IsDelegated() {
		return types.ErrPositionDelegated
	}

	isUnbonding, err := k.isUnbonding(ctx, pos.DelegatorAddress)
	if err != nil {
		return err
	}
	if isUnbonding {
		return types.ErrPositionUnbonding
	}

	return nil
```
