### Title
Gas-key host-function exec fees use borsh wire length instead of trie-id length for ML-DSA-65 keys, breaking fee-parity invariant - (File: `runtime/near-vm-runner/src/logic/logic.rs`)

### Summary

The host-function implementations of `promise_batch_action_add_gas_key_with_full_access`, `promise_batch_action_add_gas_key_with_function_call`, and `promise_batch_action_transfer_to_gas_key` pass the raw caller-supplied `public_key_len` (borsh wire length) to `gas_key_add_key_exec_fee` / `gas_key_transfer_exec_fee`. The transaction-path equivalent (`permission_exec_fees` in `runtime/runtime/src/config.rs`) correctly passes `public_key.trie_id_len()`. For ML-DSA-65 keys these two values diverge by 1920 bytes (wire = 1953, trie-id = 33), producing a ~53× overcharge in exec gas whenever a contract adds or funds an ML-DSA-65 gas key via the host-function API.

### Finding Description

The protocol documentation and the `access_key_storage_usage` / `gas_key_storage_cost` functions in `runtime/runtime/src/access_keys.rs` establish a clear invariant: all fee and storage calculations that touch the trie must use `PublicKey::trie_id_len()`, not `PublicKey::len()`. For ML-DSA-65 the trie stores a 32-byte SHA3-256 hash (tag `3` + 32 bytes = 33 bytes total), while the borsh wire encoding is 1 + 1952 = 1953 bytes.

The transaction path respects this invariant. In `permission_exec_fees`:

```rust
let nonce_fee = gas_key_add_key_exec_fee(
    fees,
    account_id.len(),
    public_key.trie_id_len(),   // ← correct: 33 for ML-DSA-65
    gas_key_info.num_nonces,
);
```

The host-function path does not. In `promise_batch_action_add_gas_key_with_full_access` (and the `_with_function_call` and `transfer_to_gas_key` variants):

```rust
let exec_fee = gas_key_add_key_exec_fee(
    &self.fees_config,
    receiver_id.len(),
    public_key_len as usize,    // ← wrong: 1953 for ML-DSA-65
    num_nonces,
);
```

`gas_key_add_key_exec_fee` feeds `public_key_len` directly into `access_key_key_len`, which computes the trie-key byte count used to price nonce writes. With `public_key_len = 1953` instead of `33`, every per-nonce exec-fee byte count is inflated by ~1920 bytes, multiplied by `num_nonces`.

The identical substitution error appears in the wasmtime host-function shim at `runtime/near-vm-runner/src/wasmtime_runner/logic.rs` for all three affected functions.

The `test_gas_key_fee_parity` integration test only exercises ED25519 keys, for which `len()` and `trie_id_len()` are identical (both 33), so the discrepancy is invisible to the existing test suite.

### Impact Explanation

Any smart contract that calls `promise_batch_action_add_gas_key_with_full_access` or `promise_batch_action_add_gas_key_with_function_call` with an ML-DSA-65 public key after `ProtocolFeature::PostQuantumSignatures` activates will be charged ~53× the correct exec gas for the nonce-write component. For a key with `num_nonces = 16` and a 10-byte account ID, the nonce key length used in the fee is `(2 + 10 + 1953 + 2) × 16` instead of `(2 + 10 + 33 + 2) × 16`, an overcharge of ~30 000 byte-units per action. At non-trivial `gas_key_byte` pricing this can exhaust a contract's prepaid gas budget, causing unexpected out-of-gas failures. It also permanently breaks the fee-parity invariant between the transaction path and the host-function path for ML-DSA-65 gas keys, making the gas model non-deterministic from a contract developer's perspective.

### Likelihood Explanation

The bug is latent until `ProtocolFeature::PostQuantumSignatures` activates. Once active, any contract that programmatically manages ML-DSA-65 gas keys via the host-function API triggers the overcharge. Because the existing fee-parity test covers only ED25519, the discrepancy will not be caught by CI before activation.

### Recommendation

Replace `public_key_len as usize` with the decoded key's `trie_id_len()` in all three host-function call sites. After `get_public_key` the decoded key is available; its `trie_id_len()` should be used for exec-fee computation:

```rust
// Before (wrong for ML-DSA-65):
let exec_fee = gas_key_add_key_exec_fee(
    &self.fees_config,
    receiver_id.len(),
    public_key_len as usize,
    num_nonces,
);

// After (correct):
let decoded_pk = public_key.decode()?;
let exec_fee = gas_key_add_key_exec_fee(
    &self.fees_config,
    receiver_id.len(),
    decoded_pk.trie_id_len(),
    num_nonces,
);
```

Apply the same fix to `gas_key_transfer_exec_fee` call sites in the host-function path. Extend `test_gas_key_fee_parity` to cover ML-DSA-65 keys.

pub fn gas_key_add_key_exec_fee(
    cfg: &RuntimeFeesConfig,
    account_id_len: usize,
    public_key_len: usize,
    num_nonces: NonceIndex,
) -> GasKeyAddFee {
    let num_nonces = num_nonces as u64;
    let base =
        cfg.fee(ActionCosts::gas_key_nonce_write_base).exec_fee().checked_mul(num_nonces).unwrap();
    let nonce_key_len =
        access_key_key_len(account_id_len, public_key_len) + std::mem::size_of::<NonceIndex>();
    let per_byte = cfg
        .fee(ActionCosts::gas_key_byte)
        .exec_fee()
        .checked_mul((nonce_key_len + AccessKey::NONCE_VALUE_LEN) as u64)
        .unwrap()
        .checked_mul(num_nonces)
        .unwrap();
    GasKeyAddFee { base, per_byte }

    /// Length, in bytes, of the on-trie identifier for an access-key
    /// entry owned by this public key. For ed25519/secp256k1 this matches
    /// `len()`; for ML-DSA-65 the trie stores a SHA3-256 hash (33 bytes
    /// including the type tag), not the 1953-byte borsh-encoded pubkey.
    /// Used by storage-fee calculations on the runtime side; cheap to call
    /// (no hashing) - for ML-DSA-65 this returns the size of the digest
    /// form without actually hashing the pubkey.
    pub fn trie_id_len(&self) -> usize {
        match self {
            Self::ED25519(_) => 1 + ed25519_dalek::PUBLIC_KEY_LENGTH,
            Self::SECP256K1(_) => 1 + 64,
            Self::MLDSA65(_) => 1 + ML_DSA_65_HASH_LENGTH,
        }
    }

fn test_gas_key_fee_parity(mode: GasKeyKind) {
    let mut setup = setup_host_function_test();
    let account = setup.account.clone();

    let num_nonces: NonceIndex = 4;
    let gas_key_a_signer: Signer =
        InMemorySigner::from_seed(account.clone(), KeyType::ED25519, "gas_key_tx").into();
    let gas_key_b_signer: Signer =
        InMemorySigner::from_seed(account.clone(), KeyType::ED25519, "gas_key_host_fn").into();

    let public_key_b_base64 = near_primitives_core::serialize::to_base64(
        &borsh::to_vec(&gas_key_b_signer.public_key()).unwrap(),
    );

    // Add gas key A via transaction, B via host function
    let add_a_outcome = setup.run_actions(vec![Action::AddKey(Box::new(AddKeyAction {
        public_key: gas_key_a_signer.public_key(),
        access_key: mode.access_key(num_nonces, &account),
    }))]);
    let add_b_outcome = setup.run_call_promise(serde_json::json!([
        {"batch_create": {"account_id": account.as_str()}, "id": 0},
        mode.add_action_json(num_nonces, &account, &public_key_b_base64),
    ]));
    assert_eq!(add_a_outcome.gas_burnt, add_b_outcome.gas_burnt);
    assert_eq!(add_a_outcome.tokens_burnt, add_b_outcome.tokens_burnt);
