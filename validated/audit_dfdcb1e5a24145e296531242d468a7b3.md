### Title
Division-by-Zero in `PerpEngine.socializeSubaccount` When `openInterest == 0` Permanently Blocks Subaccount Finalization — (`core/contracts/PerpEngine.sol`)

---

### Summary

`PerpEngine.socializeSubaccount` unconditionally divides by `state.openInterest` when insurance is insufficient to cover a negative `vQuoteBalance`. If `state.openInterest == 0` at the time of finalization, the call reverts with `ERR_DIV_BY_ZERO`, permanently preventing `_finalizeSubaccount` from completing and blocking protocol recovery for the insolvent subaccount.

---

### Finding Description

In `PerpEngine.socializeSubaccount`, after attempting to cover a negative `vQuoteBalance` with insurance, if the balance is still negative the code socializes the loss across open-interest holders: [1](#0-0) 

```solidity
if (balance.vQuoteBalance < 0) {
    int128 fundingPerShare = -balance.vQuoteBalance.div(
        state.openInterest   // <-- no zero-guard
    );
```

`MathSD21x18.div` enforces `require(y != 0, ERR_DIV_BY_ZERO)`: [2](#0-1) 

There is no guard in `socializeSubaccount` checking whether `state.openInterest > 0` before performing the division.

`_finalizeSubaccount` calls `socializeSubaccount` unconditionally: [3](#0-2) 

Before reaching `socializeSubaccount`, `_finalizeSubaccount` requires that **all** perp `balance.amount == 0`: [4](#0-3) 

This means the insolvent subaccount must have no open position. However, `vQuoteBalance` is independent of `amount` — it accumulates realized PnL from trades and funding. A subaccount can have `amount == 0` (position closed) while retaining a negative `vQuoteBalance` from prior trading losses or funding payments.

`state.openInterest` is a **global** counter across all subaccounts for a product. If every other participant has also closed their positions, `openInterest` reaches zero. This is a valid, reachable market state (e.g., a low-liquidity or end-of-life product).

**Concrete reachable path:**

1. Subaccount A opens a long perp position, accumulates negative `vQuoteBalance` (e.g., via funding payments or adverse price moves).
2. Subaccount A closes the position: `balance.amount = 0`, `balance.vQuoteBalance < 0` persists.
3. All other participants close their positions: `state.openInterest = 0`.
4. Subaccount A's maintenance health drops below zero (e.g., spot liabilities exceed assets).
5. Liquidator submits `LiquidateSubaccount` with `productId = type(uint32).max` to trigger finalization.
6. `_finalizeSubaccount` passes the `balance.amount == 0` check, settles what it can, then calls `perpEngine.socializeSubaccount`.
7. Insurance is insufficient to fully cover `vQuoteBalance` (e.g., insurance fund is depleted or `lastLiquidationFees` deduction makes `v.insurance` small).
8. `socializeSubaccount` reaches `fundingPerShare = -balance.vQuoteBalance.div(0)` → **revert "DBZ"**.
9. The entire `liquidateSubaccountImpl` transaction reverts. The state is unchanged. Every subsequent finalization attempt reverts identically.

---

### Impact Explanation

The insolvent subaccount can never be finalized. `_finalizeSubaccount` is the sole protocol path to resolve an insolvent subaccount with zero perp positions. Because it always reverts in this state, the negative `vQuoteBalance` is never cleared or socialized, the insurance fund cannot be applied to cover the loss, and the protocol is left with an unresolvable insolvent subaccount. This constitutes a permanent lock of the insurance fund's ability to recover this insolvency and a permanent DoS on the finalization path — matching the Critical scope of "permanent lock of insurance funds / blocking protocol recovery."

---

### Likelihood Explanation

The precondition (`openInterest == 0` for a product while a subaccount holds a negative `vQuoteBalance` with `amount == 0`) is reachable in normal operation: any product that winds down, loses liquidity, or where all participants exit will reach `openInterest == 0`. A subaccount that traded and closed its position retains its `vQuoteBalance`. No attacker privileges are required — this can occur organically.

---

### Recommendation

Add a zero-guard before the division in `socializeSubaccount`. If `state.openInterest == 0`, there are no open-interest holders to socialize the loss against; the loss should either be absorbed entirely by the insurance fund (already attempted above), written off to protocol bad debt, or the function should skip socialization gracefully:

```solidity
if (balance.vQuoteBalance < 0) {
    if (state.openInterest == 0) {
        // No open interest to socialize against; treat as unrecoverable bad debt
        balance.vQuoteBalance = 0;
    } else {
        int128 fundingPerShare = -balance.vQuoteBalance.div(state.openInterest);
        state.cumulativeFundingLongX18 += fundingPerShare;
        state.cumulativeFundingShortX18 -= fundingPerShare;
        balance.vQuoteBalance = 0;
    }
}
```
