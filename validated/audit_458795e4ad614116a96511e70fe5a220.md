Audit Report

## Title
`SwapAllowlistExtension.beforeSwap` checks router address instead of end-user, allowing full allowlist bypass via `MetricOmmSimpleRouter` — (File: `metric-periphery/contracts/extensions/SwapAllowlistExtension.sol`)

## Summary
`SwapAllowlistExtension` is documented to gate swaps by swapper address per pool, but its `beforeSwap` hook receives `sender = msg.sender` of `MetricOmmPool.swap`, which is the router contract when a user routes through `MetricOmmSimpleRouter`. If the pool admin allowlists the router (required for any router-mediated swap to succeed), every address on the network can bypass the per-user allowlist by calling the public router. The intended access control is completely nullified.

## Finding Description
`MetricOmmPool.swap` passes `msg.sender` verbatim as the first argument to `_beforeSwap`: [1](#0-0) 

`SwapAllowlistExtension.beforeSwap` then checks `allowedSwapper[msg.sender][sender]`, where `msg.sender` inside the extension is the pool and `sender` is the value forwarded from `pool.swap`: [2](#0-1) 

When a user calls `MetricOmmSimpleRouter.exactInputSingle`, the router calls `pool.swap(params.recipient, ...)` directly. The original `msg.sender` (the end-user) is stored only in transient storage for the payment callback and is never forwarded to any extension hook: [3](#0-2) 

The same pattern applies to `exactInput` (line 103), `exactOutputSingle` (line 136), and `exactOutput` (line 165), where `pool.swap` is always called with the router as `msg.sender`: [4](#0-3) 

The extension has no mechanism to recover the original end-user. The check `allowedSwapper[pool][router] == true` passes for every caller of the router, not just allowlisted ones.

## Impact Explanation
A pool admin deploying `SwapAllowlistExtension` to restrict swapping to a curated set of addresses must also allowlist the router contract for those addresses to use the standard UX path. The moment the router is allowlisted, the allowlist is rendered inoperative: any non-allowlisted address can call `exactInputSingle` and have the extension see `sender = router`, which is allowlisted. Unauthorized traders can execute swaps on a pool intended to be private, extracting value from LPs who expected restricted access. This constitutes direct loss of LP principal and owed fees — a High-severity broken core pool functionality impact.

## Likelihood Explanation
`MetricOmmSimpleRouter` is the standard periphery swap entry point. Any pool admin who enables `SwapAllowlistExtension` and wants legitimate allowlisted users to use the router will inevitably allowlist the router, triggering the bypass. No special privilege is required: a single call to `exactInputSingle` with valid swap parameters suffices. The condition is self-inflicted by the expected operational pattern.

## Recommendation
The extension must verify the economic actor, not the direct caller of `pool.swap`. The preferred fix is to have the router encode `msg.sender` into `extensionData` and have `SwapAllowlistExtension.beforeSwap` decode and check it. Alternatively, redesign the `beforeSwap` hook signature to include an `originalSender` field populated by the pool from a trusted periphery context (e.g., a signed permit embedded in `extensionData`). At minimum, the `SwapAllowlistExtension` NatSpec must document that allowlisting any intermediary contract grants that contract's entire user base access, and that the extension only provides meaningful access control for direct `pool.swap` callers.

## Proof of Concept
```
Setup:
  - Deploy pool with SwapAllowlistExtension configured.
  - Admin calls setAllowedToSwap(pool, alice, true).
  - Admin calls setAllowedToSwap(pool, router, true)  // required for alice to use router UX.
  - bob is NOT allowlisted.

Attack:
  1. bob calls MetricOmmSimpleRouter.exactInputSingle({pool, ...}).
  2. Router calls pool.swap(params.recipient, ...) — msg.sender inside pool.swap is the router.
  3. pool._beforeSwap(sender=router, ...) is invoked.
  4. SwapAllowlistExtension.beforeSwap receives sender=router.
  5. allowedSwapper[pool][router] == true → check passes.
  6. bob's swap executes; allowlist is bypassed.

Result: bob, a non-allowlisted address, successfully swaps on a pool
        intended to be restricted, extracting value from LPs.
```
