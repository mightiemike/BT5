### Title
Deprecated `verify_transaction_inclusion` Remains Publicly Callable, Bypassing Coinbase Merkle Proof Validation — (`contract/src/lib.rs`)

### Summary

The deprecated `verify_transaction_inclusion` function is still a live, publicly callable NEAR entry point. It omits the coinbase Merkle proof check that `verify_transaction_inclusion_v2` was specifically introduced to enforce. Any unprivileged NEAR caller — including an external DApp consuming the light client — can invoke the deprecated path directly, bypassing the 64-byte transaction Merkle proof forgery protection and obtaining a `true` proof result for a transaction that does not exist on the Bitcoin chain.

### Finding Description

`verify_transaction_inclusion_v2` was introduced to close the 64-byte transaction Merkle proof forgery vulnerability (CVE-class: internal-node hash substitution). It does so by requiring the caller to also supply a coinbase Merkle proof, verifying that the coinbase transaction sits at position 0 in the same block, before delegating to the original proof logic. [1](#0-0) 

The original function, `verify_transaction_inclusion`, carries a `#[deprecated]` attribute and an explicit code comment warning that it may return `true` when `tx_id` is an internal Merkle tree node rather than a real transaction hash. [2](#0-1) 

However, `#[deprecated]` in Rust is a compiler lint, not an access restriction. The function retains its `#[pause]` attribute and remains a fully public NEAR method. Nothing in the contract prevents an unprivileged caller from invoking it directly, completely skipping the coinbase proof gate added in v2. [3](#0-2) 

The analog to the external report is exact: just as `_processIncreaseRedeemRequest` called `authTransferFrom` which silently skipped ERC-1404 restriction checks, `verify_transaction_inclusion` silently skips the coinbase Merkle proof check that is the sole mitigation for the 64-byte forgery attack.

### Impact Explanation

An attacker who controls a 64-byte internal Merkle tree node hash can supply it as `tx_id` to `verify_transaction_inclusion`. The function computes a Merkle root from that node hash and a crafted proof path, matches it against the stored block's `merkle_root`, and returns `true`. Any downstream NEAR DApp that calls this entry point (rather than v2) will accept the forged proof as valid transaction inclusion on Bitcoin. The corrupted proof result is: `verify_transaction_inclusion` returns `true` for a Bitcoin transaction that was never broadcast or mined. [4](#0-3) 

### Likelihood Explanation

The entry point is reachable by any unprivileged NEAR account with no staking, role, or deposit requirement beyond the standard NEAR gas fee. The function is documented in the public API. External DApps that integrated before v2 was introduced, or that call the function by name without reading the deprecation notice, will use this path. The 64-byte forgery technique is publicly documented and the attack inputs are straightforward to construct.

### Recommendation

Remove `verify_transaction_inclusion` from the public NEAR interface entirely, or gate it with an access-control role that prevents unprivileged callers from reaching it. The `#[deprecated]` lint alone provides no runtime protection. If backward compatibility is required, the function body should be replaced with a call that panics with an explicit error message directing callers to `verify_transaction_inclusion_v2`.

### Proof of Concept

1. Identify any Bitcoin block whose Merkle tree has more than one transaction (so internal nodes exist).
2. Compute the hash of an internal Merkle node `N` at depth `d` in that block's tree.
3. Construct a Merkle proof path from `N` to the block's `merkle_root` (trivially possible since `N` is already an intermediate node).
4. Call `verify_transaction_inclusion` on the NEAR contract with:
   - `tx_id` = hash of internal node `N`
   - `tx_block_blockhash` = the real block hash (already stored in the contract)
   - `tx_index` = the index consistent with the crafted proof path
   - `merkle_proof` = the proof path from `N` to `merkle_root`
   - `confirmations` = 1
5. The function computes `compute_root_from_merkle_proof(N, index, proof)`, which equals the real `merkle_root`, and returns `true` — asserting that a non-existent transaction was included in the Bitcoin block. [5](#0-4)

    #[pause]
    pub fn verify_transaction_inclusion(&self, #[serializer(borsh)] args: ProofArgs) -> bool {
        require!(
            args.confirmations <= self.gc_threshold,
            "The required number of confirmations exceeds the number of blocks stored in memory"
        );

        let heaviest_block_header = self
            .headers_pool
            .get(&self.mainchain_tip_blockhash)
            .unwrap_or_else(|| env::panic_str(ERR_KEY_NOT_EXIST));
        let target_block_height = self
            .mainchain_header_to_height
            .get(&args.tx_block_blockhash)
            .unwrap_or_else(|| env::panic_str("block does not belong to the current main chain"));

        // Check requested confirmations. No need to compute proof if insufficient confirmations.
        require!(
            (heaviest_block_header.block_height).saturating_sub(target_block_height) + 1
                >= args.confirmations,
            "Not enough blocks confirmed"
        );

        let header = self
            .headers_pool
            .get(&args.tx_block_blockhash)
            .unwrap_or_else(|| env::panic_str("cannot find requested transaction block"));

        require!(!args.merkle_proof.is_empty(), "Merkle proof is empty");

        // compute merkle tree root and check if it matches block's original merkle tree root
        merkle_tools::compute_root_from_merkle_proof(
            args.tx_id,
            usize::try_from(args.tx_index).unwrap(),
            &args.merkle_proof,
        ) == header.block_header.merkle_root
    }

pub fn compute_root_from_merkle_proof(
    transaction_hash: H256,
    transaction_position: usize,
    merkle_proof: &Vec<H256>,
) -> H256 {
    let mut current_hash = transaction_hash;
    let mut current_position = transaction_position;

    for proof_hash in merkle_proof {
        if current_position % 2 == 0 {
            current_hash = compute_hash(&current_hash, proof_hash);
        } else {
            current_hash = compute_hash(proof_hash, &current_hash);
        }
        current_position /= 2;
    }

    current_hash
}
