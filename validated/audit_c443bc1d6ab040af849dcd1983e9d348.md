Audit Report

## Title
`SwapAllowlistExtension` Checks Router Address Instead of Actual User, Allowing Any User to Bypass the Per-Pool Swap Allowlist — (`File: metric-periphery/contracts/extensions/SwapAllowlistExtension.sol`)

## Summary
`SwapAllowlistExtension.beforeSwap` gates swaps by checking the `sender` argument, which is `msg.sender` of `MetricOmmPool.swap()`. When a user routes through `MetricOmmSimpleRouter`, `msg.sender` to the pool is the router contract address, not the actual user. The extension therefore evaluates `allowedSwapper[pool][router]` rather than `allowedSwapper[pool][user]`, allowing any unprivileged user to bypass a per-user swap allowlist on a curated pool by routing through the public router.

## Finding Description
`MetricOmmPool.swap()` passes `msg.sender` as the `sender` argument to `_beforeSwap`: [1](#0-0) 

`ExtensionCalling._beforeSwap` encodes and dispatches that same `sender` verbatim to each configured extension: [2](#0-1) 

`SwapAllowlistExtension.beforeSwap` enforces the allowlist using `msg.sender` (the pool) as the pool key and the received `sender` argument as the identity to check: [3](#0-2) 

When a user calls `MetricOmmSimpleRouter.exactInputSingle` (or `exactInput`, `exactOutputSingle`, `exactOutput`), the router calls `pool.swap(...)` directly, making `msg.sender` to the pool the router contract address: [4](#0-3) 

The router does not encode the original caller's identity into `extensionData`; it only stores the payer in transient storage for the swap callback, which is inaccessible to extensions. The extension therefore evaluates `allowedSwapper[pool][router]` — not `allowedSwapper[pool][user]`. The actual user's identity is never checked.

The pool admin faces an impossible choice:
- **Do not allowlist the router** → all router-mediated swaps revert with `NotAllowedToSwap`, even for individually allowlisted users (DoS on the supported periphery path).
- **Allowlist the router** → every user on the network can bypass the per-user allowlist by routing through the public router.

## Impact Explanation
A pool configured with `SwapAllowlistExtension` is a curated pool — only specific addresses are intended to trade. Any unprivileged user can bypass this restriction entirely by calling `MetricOmmSimpleRouter` instead of `pool.swap()` directly. Unauthorized users can trade on restricted pools (e.g., institutional, KYC-gated, or partner-only pools) at oracle-derived prices, causing direct loss of LP principal. This is a direct allowlist bypass with fund-impacting consequences, meeting the Critical/High threshold for direct loss of user principal. [5](#0-4) 

## Likelihood Explanation
`MetricOmmSimpleRouter` is a public, permissionless contract — any user can call it. [6](#0-5) 

The bypass requires only a single transaction through the router; no setup, no flash loan, no admin cooperation. Pool admins who configure `SwapAllowlistExtension` and also allowlist the router (the natural operational choice to support the periphery) unknowingly open the pool to all users. The bypass is reachable through the standard supported periphery path.

## Recommendation
The extension must check the economically relevant actor — the end user — not the intermediary router. Two sound approaches:

1. **Pass the real user through `extensionData`**: The router encodes the original `msg.sender` into `extensionData`; the extension decodes and checks it. This requires a trusted router convention and the extension to verify the caller is a trusted router before trusting the decoded address.
2. **Introduce an `originator` field**: Have the pool pass a separate `originator` field that the router populates with the original caller, and have the extension check that field instead of `sender`.

The `DepositAllowlistExtension` has the same structural risk: it checks `owner` (passed by the caller) rather than `sender` (`msg.sender` of `addLiquidity`), and should be audited with the same lens. [7](#0-6) 

## Proof of Concept
```
Setup:
  pool P configured with SwapAllowlistExtension E
  pool admin allowlists the router R: allowedSwapper[P][R] = true
  user Alice (address A) is NOT in the allowlist: allowedSwapper[P][A] = false

Attack:
  Alice calls MetricOmmSimpleRouter.exactInputSingle({pool: P, recipient: Alice, ...})
    → router calls P.swap(recipient=Alice, ...)  [msg.sender to pool = router R]
    → pool calls _beforeSwap(sender=R, ...)
    → ExtensionCalling encodes sender=R and dispatches to E
    → E.beforeSwap checks: allowedSwapper[P][R] == true  ✓
    → swap executes; Alice receives tokens

Result:
  Alice, an unauthorized user, successfully swaps on a curated pool.
  The per-user allowlist guard is completely bypassed.
  LP funds are exposed to any public user through the supported router path.
``` [8](#0-7) [9](#0-8)
