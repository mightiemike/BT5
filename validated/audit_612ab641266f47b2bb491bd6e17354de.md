### Title
Duplicate Position IDs in `ClaimTierRewards` Enable Double Bonus Reward Claim and Premature Event Deletion ŌĆö (`x/tieredrewards/keeper/msg_server.go`)

---

### Summary

`ClaimTierRewards` loads all requested positions into an in-memory slice before processing any of them. Because there is no deduplication guard on `msg.PositionIds`, an attacker can include the same position ID multiple times. Each duplicate entry holds a stale snapshot of the position (with the original `LastEventSeq`), so `processEventsAndClaimBonus` re-processes the same validator events on every duplicate, paying out bonus rewards multiple times. As a side-effect, `decrementEventRefCount` is called extra times per event, prematurely deleting events that other positions still need, causing those positions to permanently lose their accrued bonus rewards.

---

### Finding Description

In `ClaimTierRewards` (`x/tieredrewards/keeper/msg_server.go`, lines 429ŌĆō468), the handler first iterates over `msg.PositionIds`, loading each position from the store and appending it to a local slice: [1](#0-0) 

`validateClaimRewards` only checks ownership: [2](#0-1) 

There is no deduplication check. If `msg.PositionIds = [P1, P1]`, two stale copies of the same position (both with the same `LastEventSeq`) are appended to `positions`.

`claimRewardsAndUpdatesPositions` then iterates over this slice: [3](#0-2) 

For each entry it calls `processEventsAndClaimBonus`, which reads validator events from the store using the stale `pos.LastEventSeq`: [4](#0-3) 

After the first iteration, `setPosition` writes the updated `LastEventSeq` to the store, but the second iteration's `pos` is the other stale in-memory copy ŌĆö it still carries the original `LastEventSeq`. So `getValidatorEventsSince` returns the same events again.

The critical enabler is `decrementEventRefCount`: [5](#0-4) 

An event is only deleted when its `ReferenceCount` reaches zero. If the validator has multiple positions (a common scenario), events are created with `ReferenceCount > 1`. After the first duplicate iteration decrements the count to `N-1`, the event still exists. The second duplicate iteration finds it, pays bonus again, and decrements the count to `N-2`. This continues for every duplicate in the list.

The bonus payout itself is a real fund transfer: [6](#0-5) 

---

### Impact Explanation

**Double bonus reward drain:** An attacker submitting `PositionIds = [P1, P1, ŌĆ”, P1]` (k duplicates) receives up to k├Ś the legitimate bonus for that position in a single transaction, draining the `RewardsPoolName` module account.

**Permanent loss of rewards for other positions:** Each extra decrement of `ReferenceCount` for a shared event brings it closer to zero. Once it hits zero the event is deleted. Any other position that has not yet processed that event can never claim the bonus it was owed ŌĆö the event is gone from the store.

The corrupted value is: the `RewardsPoolName` module account balance (over-drained) and the `ValidatorEvents` collection (events deleted before all referencing positions have processed them).

---

### Likelihood Explanation

- The entry path is a standard, unprivileged `MsgClaimTierRewards` transaction signed by any position owner.
- The only precondition is that the attacker's validator has at least one other position delegated to it (so events have `ReferenceCount Ōēź 2`). This is the normal operating state of any active validator.
- No special permissions, governance, or leaked keys are required.

---

### Recommendation

Deduplicate `msg.PositionIds` before loading positions. A simple approach is to reject the message if any ID appears more than once:

```go
seen := make(map[uint64]struct{}, len(msg.PositionIds))
for _, posId := range msg.PositionIds {
    if _, dup := seen[posId]; dup {
        return nil, errorsmod.Wrapf(sdkerrors.ErrInvalidRequest,
            "duplicate position id %d", posId)
    }
    seen[posId] = struct{}{}
}
```

This should be added at the top of `ClaimTierRewards` in `x/tieredrewards/keeper/msg_server.go`, before the position-loading loop.

### Proof of Concept

**Setup:**
- Validator `V` has two positions: `P1` (attacker-owned) and `P2` (victim-owned), both with `LastEventSeq = S`.
- A new validator event `E` is appended at sequence `S+1` with `ReferenceCount = 2`.

**Attack transaction:**
```
MsgClaimTierRewards { Owner: attacker, PositionIds: [P1, P1] }
```

**Execution trace in `claimRewardsAndUpdatesPositions`:**

| Iteration | In-memory `LastEventSeq` | `getValidatorEventsSince` result | Bonus paid | `ReferenceCount` of E after |
|-----------|--------------------------|----------------------------------|------------|-----------------------------|
| 0 (P1 copy 1) | S | finds E (RC=2) | bonus_E | 1 |
| 1 (P1 copy 2) | S (stale) | finds E (RC=1) | bonus_E again | 0 ŌåÆ **E deleted** |

**Outcome:**
- Attacker receives `2 ├Ś bonus_E` instead of `1 ├Ś bonus_E`.
- Event `E` is deleted with `ReferenceCount = 0`.
- When `P2`'s owner later calls `ClaimTierRewards`, `getValidatorEventsSince(V, S)` returns nothing for `E` (it no longer exists). `P2` permanently loses its bonus for event `E`. [7](#0-6) [8](#0-7) [9](#0-8)

func (k Keeper) claimRewardsAndUpdatesPositions(ctx context.Context, positions []types.PositionState) (sdk.Coins, sdk.Coins, error) {
	totalBase := sdk.NewCoins()
	totalBonus := sdk.NewCoins()

	for i := range positions {
		pos := &positions[i]

		if !pos.IsDelegated() {
			continue
		}

		base, err := k.claimBaseRewards(ctx, *pos)
		if err != nil {
			return nil, nil, err
		}
		totalBase = totalBase.Add(base...)

		bonus, err := k.processEventsAndClaimBonus(ctx, pos)
		if err != nil {
			return nil, nil, err
		}
		totalBonus = totalBonus.Add(bonus...)

		if err := k.setPosition(ctx, pos.Position, nil); err != nil {
			return nil, nil, err
		}
	}

	return totalBase, totalBonus, nil
}

func (k Keeper) getValidatorEventsSince(ctx context.Context, valAddr sdk.ValAddress, startSeq uint64) ([]EventEntry, error) {
	// Range from (valAddr, startSeq+1) to end of valAddr prefix.
	rng := collections.NewPrefixedPairRange[sdk.ValAddress, uint64](valAddr).
		StartExclusive(startSeq)

	var entries []EventEntry
	err := k.ValidatorEvents.Walk(ctx, rng, func(key collections.Pair[sdk.ValAddress, uint64], event types.ValidatorEvent) (bool, error) {
		entries = append(entries, EventEntry{Seq: key.K2(), Event: event})
		return false, nil
	})
	if err != nil {
		return nil, err
	}
	return entries, nil
}

// decrementEventRefCount decrements the reference count of a validator event.
// If the reference count reaches zero, the event is deleted.
func (k Keeper) decrementEventRefCount(ctx context.Context, valAddr sdk.ValAddress, seq uint64) error {
	key := collections.Join(valAddr, seq)
	event, err := k.ValidatorEvents.Get(ctx, key)
	if errors.Is(err, collections.ErrNotFound) {
		return nil // already cleaned up
	}
	if err != nil {
		return err
	}

	if event.ReferenceCount <= 1 {
		return k.ValidatorEvents.Remove(ctx, key)
	}

	event.ReferenceCount--
	return k.ValidatorEvents.Set(ctx, key, event)
}
