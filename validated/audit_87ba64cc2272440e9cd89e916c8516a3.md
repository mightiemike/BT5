### Title
Accrued Bonus Rewards Permanently Lost When Position Deleted After Redelegation Slash — (File: x/tieredrewards/keeper/msg_server.go)

---

### Summary

In the `x/tieredrewards` module, when a position's delegation is zeroed by a 100% redelegation slash (making it undelegated without going through `TierUndelegate`), any bonus rewards that accrued while the position was delegated are permanently lost. `WithdrawFromTier` does not call `claimRewards` before deleting the position, and `claimRewards` itself unconditionally returns zero for undelegated positions. No other message path can recover the accrued bonus for an undelegated position. The position's reward checkpoints (`LastBonusAccrual`, `LastEventSeq`, `LastKnownBonded`) are silently discarded on deletion.

---

### Finding Description

The normal exit lifecycle settles rewards at `TierUndelegate`:

```
TierUndelegate → claimRewards(pos) → undelegate → setPosition
``` [1](#0-0) 

However, when a position is undelegated by a **redelegation slash** (not by `TierUndelegate`), the position transitions to undelegated state without any reward settlement. The user then follows the path:

```
TriggerExitFromTier → (no claimRewards) → WithdrawFromTier → deletePosition
```

`WithdrawFromTier` never calls `claimRewards`: [2](#0-1) 

Even if it did, `claimRewards` and `processEventsAndClaimBonus` both guard on `pos.IsDelegated()` and return zero for undelegated positions: [3](#0-2) [4](#0-3) 

There is no alternative path to recover the accrued bonus:

- `ClaimTierRewards` → `claimRewardsAndUpdatesPositions` skips undelegated positions (line 113: `if !pos.IsDelegated() { continue }`).
- `TierUndelegate` rejects undelegated positions with `ErrPositionNotDelegated`.
- `ExitTierWithDelegation` requires `pos.IsDelegated()`.
- `ClearPosition` calls `claimRewards` but that returns zero for undelegated positions. [5](#0-4) 

The position's bonus reward checkpoints (`LastBonusAccrual`, `LastEventSeq`, `LastKnownBonded`) are stored in the `Position` proto and are permanently discarded when `deletePosition` removes the record: [6](#0-5) 

The existing test `TestMsgWithdrawFromTier_RedelegSlashedToZero` confirms this path succeeds but does not assert that accrued bonus rewards were paid out before deletion: [7](#0-6) 

---

### Impact Explanation

A delegator who held a position for a significant duration (accruing bonus APY rewards) and then suffered a 100% redelegation slash loses all bonus rewards that accrued during the bonded period. The `RewardsPoolName` module account retains those tokens permanently — they are neither paid to the user nor returned to any treasury. The corrupted value is the user's bonus reward entitlement encoded in `LastBonusAccrual` / `LastEventSeq` / `LastKnownBonded` fields of the deleted position.

**Impact: 3 / 5** — Direct loss of earned bonus rewards for affected users; magnitude scales with position size, tier APY, and time bonded before the slash.

---

### Likelihood Explanation

A 100% redelegation slash requires a validator to commit a double-sign infraction during the redelegation window. This is a realistic on-chain event (not a theoretical one), and the tiered rewards module explicitly handles this case in its test suite (`slashRedelegationCompletely`). Any user who redelegated a tier position to a validator that subsequently double-signed would be affected.

**Likelihood: 3 / 5** — Requires a specific slash event, but the module's own tests demonstrate the path is reachable and the code explicitly accommodates zero-amount undelegated positions.

---

### Recommendation

`WithdrawFromTier` should attempt to settle any remaining accrued bonus rewards before deleting the position. Because `claimRewards` is a no-op for undelegated positions, a dedicated settlement path is needed that processes the bonus accrual checkpoints even when the position has no active delegation — for example, by computing the bonus up to the point the position became undelegated (i.e., up to the last recorded validator event before undelegation) and paying it out before calling `deletePosition`.

Alternatively, `processEventsAndClaimBonus` could be extended to handle the undelegated case by replaying events up to the last known bonded segment end, using the stored `LastKnownBonded` and `LastBonusAccrual` checkpoints, without requiring an active delegation.

---

### Proof of Concept

1. User calls `MsgLockTier` and delegates to validator V. Position accrues bonus rewards for N months.
2. User calls `MsgTierRedelegate` to move delegation from V to V2. An active redelegation entry is created.
3. Validator V commits a double-sign infraction. The Cosmos SDK slashes the redelegating delegation 100%.
4. The position's delegation is cleared by the slash hook; `pos.IsDelegated()` returns `false`. No `claimRewards` is called.
5. User calls `MsgTriggerExitFromTier` (succeeds; no reward settlement).
6. User waits for `ExitUnlockAt`.
7. User calls `MsgWithdrawFromTier`. `validateWithdrawFromTier` passes (not delegated, not unbonding, exit elapsed). `SpendableCoins` on `delAddr` is zero (nothing to sweep). `deletePosition` removes the position record including all bonus checkpoints.
8. All bonus rewards accrued in step 1 are permanently lost. The `RewardsPoolName` balance is unaffected (tokens remain locked in the pool). [8](#0-7)

func (ms msgServer) WithdrawFromTier(ctx context.Context, msg *types.MsgWithdrawFromTier) (*types.MsgWithdrawFromTierResponse, error) {
	if err := msg.Validate(); err != nil {
		return nil, err
	}

	pos, err := ms.getPositionState(ctx, msg.PositionId)
	if err != nil {
		return nil, err
	}

	if err := ms.validateWithdrawFromTier(ctx, pos, msg.Owner); err != nil {
		return nil, err
	}

	ownerAddr, err := sdk.AccAddressFromBech32(pos.Owner)
	if err != nil {
		return nil, errorsmod.Wrap(sdkerrors.ErrInvalidAddress, "invalid owner address")
	}

	delAddr, err := sdk.AccAddressFromBech32(pos.DelegatorAddress)
	if err != nil {
		return nil, errorsmod.Wrap(sdkerrors.ErrInvalidAddress, "invalid delegator address")
	}

	balances := ms.bankKeeper.SpendableCoins(ctx, delAddr)
	if !balances.IsZero() {
		if err := ms.bankKeeper.SendCoins(ctx, delAddr, ownerAddr, balances); err != nil {
			return nil, err
		}
	}

	if err := ms.deletePosition(ctx, pos.Position, nil); err != nil {
		return nil, err
	}

	sdkCtx := sdk.UnwrapSDKContext(ctx)
	if err := sdkCtx.EventManager().EmitTypedEvent(&types.EventPositionWithdrawn{
		Position: pos.Position,
		Amount:   balances,
	}); err != nil {
		return nil, err
	}

	return &types.MsgWithdrawFromTierResponse{
		Amount: balances,
	}, nil
}

func (k Keeper) claimRewards(ctx context.Context, pos types.PositionState) (types.PositionState, sdk.Coins, sdk.Coins, error) {
	if !pos.IsDelegated() {
		return pos, sdk.NewCoins(), sdk.NewCoins(), nil
	}

	base, err := k.claimBaseRewards(ctx, pos)
	if err != nil {
		return types.PositionState{}, nil, nil, err
	}

	bonus, err := k.processEventsAndClaimBonus(ctx, &pos)
	if err != nil {
		return types.PositionState{}, nil, nil, err
	}

	return pos, base, bonus, nil
}

func (k Keeper) validateUndelegatePosition(ctx context.Context, pos types.PositionState, owner string) error {
	if !pos.IsOwner(owner) {
		return types.ErrNotPositionOwner
	}

	if !pos.IsDelegated() {
		return types.ErrPositionNotDelegated
	}

	if !pos.HasTriggeredExit() {
		return types.ErrExitNotTriggered
	}

	sdkCtx := sdk.UnwrapSDKContext(ctx)
	if !pos.CompletedExitLockDuration(sdkCtx.BlockTime()) {
		return types.ErrExitLockDurationNotReached
	}

	// skip check for zero amount as we want those positions to be able to close their position properly

	return nil
}

// deletePosition validates and removes a position and cleans up secondary indexes.
func (k Keeper) deletePosition(ctx context.Context, pos types.Position, update *ValidatorTransition) error {
	owner, err := sdk.AccAddressFromBech32(pos.Owner)
	if err != nil {
		return errorsmod.Wrap(sdkerrors.ErrInvalidAddress, "invalid owner address")
	}

	delAddr, err := sdk.AccAddressFromBech32(pos.DelegatorAddress)
	if err != nil {
		return errorsmod.Wrap(sdkerrors.ErrInvalidAddress, "invalid delegator address")
	}

	del, err := k.getDelegation(ctx, pos.DelegatorAddress)
	if err != nil {
		return err
	}
	if del != nil {
		return errorsmod.Wrapf(types.ErrPositionDelegated, "cannot delete position %d: still has active delegation to %s", pos.Id, del.ValidatorAddress)
	}

	if err := k.removeBaseRewardsRouting(ctx, delAddr, owner); err != nil {
		return err
	}

	// Defensive
	if err := k.deletePositionRedelegationMappings(ctx, pos.Id); err != nil {
		return err
	}

	if err := k.Positions.Remove(ctx, pos.Id); err != nil {
		return err
	}
	if err := k.PositionsByOwner.Remove(ctx, collections.Join(owner, pos.Id)); err != nil {
		return err
	}
	if err := k.PositionsByTier.Remove(ctx, collections.Join(pos.TierId, pos.Id)); err != nil {
		return err
	}
	if err := k.decreasePositionCountForTier(ctx, pos.TierId); err != nil {
		return err
	}

	if update == nil {
		return nil
	}
	return k.reindexPositionCountByValidator(ctx, update.PreviousAddress, "")
}

// TestMsgWithdrawFromTier_RedelegSlashedToZero verifies the full lifecycle of
// a position that was zeroed by a redelegation slash: trigger exit, advance
// past exit duration, then withdraw. The zero-amount withdrawal should succeed
// and delete the position.
func (s *KeeperSuite) TestMsgWithdrawFromTier_RedelegSlashedToZero() {
	pos := s.setupNewTierPosition(sdkmath.NewInt(1000), false)
	delAddr := sdk.MustAccAddressFromBech32(pos.Owner)
	msgServer := keeper.NewMsgServerImpl(s.keeper)

	// Simulate redelegation slash: clear delegation and zero amount.
	pos = s.slashRedelegationCompletely(pos)

	// Trigger exit and advance past exit duration.
	_, err := msgServer.TriggerExitFromTier(s.ctx, &types.MsgTriggerExitFromTier{
		Owner:      delAddr.String(),
		PositionId: pos.Id,
	})
	s.Require().NoError(err)
	s.advancePastExitDuration()

	// Withdraw — zero-amount position should be cleanly deleted.
	resp, err := msgServer.WithdrawFromTier(s.ctx, &types.MsgWithdrawFromTier{
		Owner:      delAddr.String(),
		PositionId: pos.Id,
	})
	s.Require().NoError(err)
	s.Require().True(resp.Amount.IsZero(),
		"withdrawn amount should be zero")

	_, err = s.keeper.GetPositionState(s.ctx, pos.Id)
	s.Require().Error(err, "position should be deleted after withdrawal")
}
