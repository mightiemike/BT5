### Title
Duplicate Position IDs in `ClaimTierRewards` Enable Double Bonus Reward Withdrawal - (File: x/tieredrewards/keeper/msg_server.go)

---

### Summary

`MsgClaimTierRewards` loads all requested positions from the store **before** processing any of them. If the same position ID appears more than once in `msg.PositionIds`, multiple stale in-memory copies are processed sequentially. Because bonus coins are sent to the owner inside `processEventsAndClaimBonus` **before** `setPosition` persists the updated checkpoints, the second (stale) copy re-processes the same validator events and pays out the same bonus a second time — draining the `RewardsPoolName` module account beyond what was legitimately accrued.

---

### Finding Description

In `ClaimTierRewards` (`x/tieredrewards/keeper/msg_server.go`, lines 429–468), all positions are loaded from the KV store in a first loop and appended to a slice:

```go

    pos, err := ms.getPositionState(ctx, posId)   // reads from store
    ...

}
totalBase, totalBonus, err := ms.claimRewardsAndUpdatesPositions(ctx, positions)

**Concrete attack trace** (position `P` tracks validator `V`; a second position `Q` also tracks `V`, so each validator event `E1` has ref-count ≥ 2):

| Step | Action | Store state |
|---|---|---|
| 1 | Load `positions[0]` = P (LastEventSeq=0) | unchanged |
| 2 | Load `positions[1]` = P (LastEventSeq=0, stale) | unchanged |
| 3 | Process `positions[0]`: `processEventsAndClaimBonus` finds E1, sends bonus, decrements E1 ref-count 2→1 | E1 still exists |
| 4 | `setPosition(positions[0])` persists LastEventSeq=E1.Seq | updated |
| 5 | Process `positions[1]` (stale, LastEventSeq=0): `processEventsAndClaimBonus` finds E1 again (ref-count=1>0), sends **same bonus again**, decrements E1 ref-count 1→0 | E1 removed |
| 6 | `setPosition(positions[1])` persists LastEventSeq=E1.Seq | same as step 4 |

The owner receives **2× the legitimate bonus**. The attack generalises: listing the same ID `N` times pays `N×` the bonus, bounded by the event ref-count (i.e., the number of positions tracking the same validator).

---

### Impact Explanation

The `RewardsPoolName` module account is drained by a multiple of the legitimately accrued bonus. Every duplicate entry in `PositionIds` causes one extra `SendCoinsFromModuleToAccount` call for the full bonus amount of that position. With a large enough ref-count (many positions on the same validator, which is the normal case on a public chain), an attacker can multiply their payout arbitrarily within a single transaction, stealing funds that belong to other stakers.

---

### Likelihood Explanation

The entry point is a standard, unprivileged user transaction (`MsgClaimTierRewards`). No special role, leaked key, or social engineering is required — any position owner can submit it. The precondition (ref-count ≥ 2, i.e., at least one other position delegated to the same validator) is satisfied by default on any live validator with multiple tiered-rewards participants, making this trivially reachable in production.

---

### Recommendation

Apply the Checks-Effects-Interactions pattern:

1. **Deduplicate `msg.PositionIds`** in `msg.Validate()` or at the top of `ClaimTierRewards` before loading positions, so the same position cannot appear twice in one message.
2. **Persist the updated position to the store before sending coins**: call `setPosition` (or at minimum advance the checkpoint in the KV store) inside `processEventsAndClaimBonus` before `SendCoinsFromModuleToAccount`, so a stale in-memory copy cannot re-process the same events.

---

### Proof of Concept

```
// Attacker owns position P (ID=42) delegated to validator V.
// One other position Q also delegates to V → event ref-count = 2.

MsgClaimTierRewards{
    Owner:       attacker,
    PositionIds: [42, 42],   // same ID twice
}

// Result:
//   Iteration 0: bonus B sent to attacker; E1 ref-count 2→1
//   Iteration 1: stale copy re-processes E1 (still accessible);
//                bonus B sent to attacker again; E1 ref-count 1→0
// Attacker receives 2B instead of B.
// Repeat with [42,42,42,...] up to the initial ref-count to multiply further.

func (ms msgServer) ClaimTierRewards(ctx context.Context, msg *types.MsgClaimTierRewards) (*types.MsgClaimTierRewardsResponse, error) {
	if err := msg.Validate(); err != nil {
		return nil, err
	}

	positions := make([]types.PositionState, 0, len(msg.PositionIds))
	for _, posId := range msg.PositionIds {
		pos, err := ms.getPositionState(ctx, posId)
		if err != nil {
			return nil, err
		}

		if err := ms.validateClaimRewards(pos.Position, msg.Owner); err != nil {
			return nil, err
		}

		positions = append(positions, pos)
	}

	totalBase, totalBonus, err := ms.claimRewardsAndUpdatesPositions(ctx, positions)
	if err != nil {
		return nil, err
	}

	sdkCtx := sdk.UnwrapSDKContext(ctx)
	if err := sdkCtx.EventManager().EmitTypedEvent(&types.EventTierRewardsClaimed{
		Owner:        msg.Owner,
		PositionIds:  msg.PositionIds,
		BaseRewards:  totalBase,
		BonusRewards: totalBonus,
	}); err != nil {
		return nil, err
	}

	return &types.MsgClaimTierRewardsResponse{
		BaseRewards:  totalBase,
		BonusRewards: totalBonus,
		PositionIds:  msg.PositionIds,
	}, nil
}

func (k Keeper) claimRewardsAndUpdatesPositions(ctx context.Context, positions []types.PositionState) (sdk.Coins, sdk.Coins, error) {
	totalBase := sdk.NewCoins()
	totalBonus := sdk.NewCoins()

	for i := range positions {
		pos := &positions[i]

		if !pos.IsDelegated() {
			continue
		}

		base, err := k.claimBaseRewards(ctx, *pos)
		if err != nil {
			return nil, nil, err
		}
		totalBase = totalBase.Add(base...)

		bonus, err := k.processEventsAndClaimBonus(ctx, pos)
		if err != nil {
			return nil, nil, err
		}
		totalBonus = totalBonus.Add(bonus...)

		if err := k.setPosition(ctx, pos.Position, nil); err != nil {
			return nil, nil, err
		}
	}

	return totalBase, totalBonus, nil
}

func (k Keeper) processEventsAndClaimBonus(ctx context.Context, pos *types.PositionState) (sdk.Coins, error) {
	// Rewards should have been claimed before undelegation
	if !pos.IsDelegated() {
		return sdk.NewCoins(), nil
	}

	valAddr, err := sdk.ValAddressFromBech32(pos.Delegation.ValidatorAddress)
	if err != nil {
		return nil, err
	}

	events, err := k.getValidatorEventsSince(ctx, valAddr, pos.LastEventSeq)
	if err != nil {
		return nil, err
	}

	sdkCtx := sdk.UnwrapSDKContext(ctx)
	blockTime := sdkCtx.BlockTime()

	totalBonus := math.ZeroInt()
	// Use the persisted bonded state from the last replay, not a hardcoded default.
	// This prevents overpaying bonus for unbonded gaps between claims.
	bonded := pos.LastKnownBonded
	segmentStart := pos.LastBonusAccrual

	tier, err := k.getTier(ctx, pos.TierId)
	if err != nil {
		return nil, err
	}

	for _, entry := range events {
		evt := entry.Event

		if bonded {
			// Compute bonus for the bonded segment [segmentStart, eventTime]
			// using the snapshot rate at the event.
			bonus := k.computeSegmentBonus(*pos, tier, segmentStart, evt.Timestamp, evt.TokensPerShare)
			totalBonus = totalBonus.Add(bonus)
		}

		// Update bonded state based on event type.
		switch evt.EventType {
		case types.ValidatorEventType_VALIDATOR_EVENT_TYPE_UNBOND:
			bonded = false
		case types.ValidatorEventType_VALIDATOR_EVENT_TYPE_BOND:
			bonded = true
		case types.ValidatorEventType_VALIDATOR_EVENT_TYPE_SLASH:
			// Slash doesn't change bonded state.
		}

		segmentStart = evt.Timestamp
		pos.UpdateLastEventSeq(entry.Seq)

		// Decrement reference count.
		if err := k.decrementEventRefCount(ctx, valAddr, entry.Seq); err != nil {
			return nil, err
		}
	}

	val, err := k.stakingKeeper.GetValidator(ctx, valAddr)
	if err != nil {
		return nil, err
	}
	// Defensive: validator bond status check
	if bonded && val.IsBonded() {
		currentRate, err := k.getTokensPerShare(ctx, valAddr)
		if err != nil {
			return nil, err
		}
		bonus := k.computeSegmentBonus(*pos, tier, segmentStart, blockTime, currentRate)
		totalBonus = totalBonus.Add(bonus)
	}

	applyBonusAccrualCheckpoint(&pos.Position, blockTime)
	// Persist the bonded state so the next replay starts correctly.
	pos.UpdateLastKnownBonded(bonded)

	if totalBonus.IsZero() {
		return sdk.NewCoins(), nil
	}

	bondDenom, err := k.stakingKeeper.BondDenom(ctx)
	if err != nil {
		return nil, err
	}

	bonusCoins := sdk.NewCoins(sdk.NewCoin(bondDenom, totalBonus))

	if err := k.sufficientBonusPoolBalance(ctx, bonusCoins); err != nil {
		return nil, err
	}

	ownerAddr, err := sdk.AccAddressFromBech32(pos.Owner)
	if err != nil {
		return nil, errorsmod.Wrap(sdkerrors.ErrInvalidAddress, "invalid owner address")
	}

	if err := k.bankKeeper.SendCoinsFromModuleToAccount(ctx, types.RewardsPoolName, ownerAddr, bonusCoins); err != nil {
		return nil, err
	}

	if err := sdkCtx.EventManager().EmitTypedEvent(&types.EventBonusRewardsClaimed{
		PositionId: pos.Id,
		Owner:      pos.Owner,
		Rewards:    bonusCoins,
	}); err != nil {
		return nil, err
	}

	return bonusCoins, nil
