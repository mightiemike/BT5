Audit Report

## Title
`SwapAllowlistExtension` checks router address instead of actual user, enabling allowlist bypass via `MetricOmmSimpleRouter` — (`metric-periphery/contracts/extensions/SwapAllowlistExtension.sol`)

## Summary
`SwapAllowlistExtension.beforeSwap` receives the `sender` argument from the pool, which is `msg.sender` at the pool level — the router contract, not the end user. When a pool admin allowlists the router to support router-mediated swaps, every user (including those intended to be blocked) can bypass the allowlist by routing through `MetricOmmSimpleRouter`. Conversely, individually allowlisted users cannot use the router at all because the router's address is not in the allowlist.

## Finding Description
The call chain is confirmed by the production code:

1. `MetricOmmPool.swap` passes `msg.sender` as `sender` to `_beforeSwap`: [1](#0-0) 

2. `ExtensionCalling._beforeSwap` forwards `sender` unchanged to the extension: [2](#0-1) 

3. `SwapAllowlistExtension.beforeSwap` checks `allowedSwapper[msg.sender][sender]` — where `msg.sender` is the pool and `sender` is whoever called `pool.swap()` (the router): [3](#0-2) 

4. `MetricOmmSimpleRouter.exactInputSingle` stores the real user only in transient storage for payment settlement via `_setNextCallbackContext`, and never passes the original `msg.sender` to `pool.swap()`: [4](#0-3) 

The real user's identity is permanently lost from the extension's perspective. No existing guard in the extension, pool, or router recovers or forwards the original caller to the allowlist check.

## Impact Explanation
**Bypass path (High):** A pool admin who calls `setAllowedToSwap(pool, router, true)` to enable router-mediated swaps grants every user — including those the admin intended to block — the ability to swap on the curated pool. The allowlist is completely ineffective for router-mediated swaps, violating the core invariant that a curated pool enforces its allowlist policy regardless of entrypoint.

**Broken functionality path (Medium):** A pool admin who allowlists specific individual users (e.g., `allowedSwapper[pool][alice] = true`) will find those users cannot use the router at all, since the router's address is not allowlisted. Allowlisted users are forced to call the pool directly, bypassing the standard user-facing interface.

Both outcomes constitute broken core pool functionality and an admin-boundary break: the pool admin's access control configuration is either rendered ineffective or forces users off the intended interface.

## Likelihood Explanation
The router is the primary user-facing interface for swaps. Any pool admin deploying a `SwapAllowlistExtension`-gated pool who wants to support router users will naturally call `setAllowedToSwap(pool, router, true)`. This single standard admin action triggers the bypass. Exploitation requires only an unprivileged user calling `exactInputSingle` or `exactInput` through the router — both are normal, public, permissionless operations.

## Recommendation
The `sender` identity forwarded to extensions must reflect the original user, not the intermediary router. Two approaches:

1. **Router forwards the real user:** Add a `sender` field to swap parameters that the router populates with `msg.sender` before calling the pool. The pool passes this field (validated against `msg.sender` for direct calls) to extensions.
2. **Extension reads transient context:** The router already stores the real payer in transient storage via `_setNextCallbackContext`. Expose this through a standardized interface so extensions can read the original caller when the pool's `msg.sender` is a known router.

## Proof of Concept
```solidity
// 1. Deploy pool with SwapAllowlistExtension
// 2. Admin allowlists the router (to enable router-mediated swaps)
extension.setAllowedToSwap(address(pool), address(router), true);

// 3. Bob is NOT individually allowlisted
assertFalse(extension.isAllowedToSwap(address(pool), bob));

// 4. Bob bypasses the allowlist by going through the router
vm.prank(bob);
router.exactInputSingle(IMetricOmmSimpleRouter.ExactInputSingleParams({
    pool: address(pool),
    tokenIn: address(token0),
    recipient: bob,
    zeroForOne: true,
    amountIn: 1000,
    amountOutMinimum: 0,
    priceLimitX64: 0,
    extensionData: "",
    deadline: block.timestamp + 1
}));
// Swap succeeds: extension checked allowedSwapper[pool][router] = true
// instead of allowedSwapper[pool][bob] = false
```
