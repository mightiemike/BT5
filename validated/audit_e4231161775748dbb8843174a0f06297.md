Audit Report

## Title
`SwapAllowlistExtension` Checks Router Address Instead of End-User, Enabling Full Allowlist Bypass via `MetricOmmSimpleRouter` — (`metric-periphery/contracts/extensions/SwapAllowlistExtension.sol`)

## Summary
`SwapAllowlistExtension.beforeSwap` gates swaps by checking the `sender` argument, which `MetricOmmPool.swap` always sets to its own `msg.sender`. When a swap is routed through `MetricOmmSimpleRouter`, `msg.sender` at the pool is the router contract, not the end-user. If the pool admin allowlists the router (the natural action to enable allowlisted users to access the pool via the standard periphery), every on-chain caller of the router bypasses the allowlist silently.

## Finding Description
**Step 1 — Pool passes its own `msg.sender` as `sender`.**
`MetricOmmPool.swap` calls `_beforeSwap(msg.sender, recipient, ...)` at lines 230–240. When the call originates from `MetricOmmSimpleRouter`, `msg.sender` is the router contract address.

**Step 2 — `ExtensionCalling._beforeSwap` forwards `sender` verbatim.**
`ExtensionCalling._beforeSwap` (lines 149–177) encodes `sender` directly into the `beforeSwap` call without modification.

**Step 3 — `SwapAllowlistExtension.beforeSwap` checks `allowedSwapper[msg.sender][sender]`.**
At line 37 of `SwapAllowlistExtension.sol`, `sender` is the router address. The extension has no visibility into the actual end-user.

**Step 4 — The router never encodes the real caller into `extensionData`.**
`MetricOmmSimpleRouter.exactInputSingle` (lines 67–86) passes `params.extensionData` opaquely and user-supplied; it never encodes `msg.sender` into it. `SwapAllowlistExtension` ignores `extensionData` entirely.

**Resulting invariant break:** A pool admin who wants allowlisted users to access the pool via the standard router must call `setAllowedToSwap(pool, router, true)`. From that moment, `allowedSwapper[pool][router] == true` satisfies the check for every caller of the router, regardless of their identity. The allowlist is fully bypassed.

**Contrast with `DepositAllowlistExtension`:** `beforeAddLiquidity` checks the `owner` argument (the position beneficiary, line 38), not `sender`. The asymmetry confirms the swap-side check is the defective one.

## Impact Explanation
Any unprivileged on-chain actor can swap against a pool whose admin intended to restrict access to a specific set of counterparties (e.g., KYC'd users, RFQ-style pools, compliance-gated pools) by simply calling `MetricOmmSimpleRouter.exactInputSingle` (or `exactInput`/`exactOutput`/`exactOutputSingle`). The pool executes at the live oracle bid/ask price, so unauthorized swappers can trade against LP funds at oracle-fair prices without the LP's consent. This constitutes a direct breach of the access-control invariant and exposes LP principal to unintended counterparty risk — a High-severity admin-boundary break with direct fund impact.

## Likelihood Explanation
`MetricOmmSimpleRouter` is the primary user-facing swap interface. Any pool admin who deploys a swap-allowlisted pool and wants allowlisted users to access it through the standard periphery will naturally allowlist the router. The bypass requires no special knowledge beyond knowing the router address and calling it normally. It is reachable by any unprivileged on-chain actor with no preconditions beyond the admin having allowlisted the router.

## Recommendation
Pass the actual end-user identity through the extension pipeline. Two options:

1. **Preferred — encode caller in `extensionData` at the router level.** The router encodes `msg.sender` into `extensionData` before forwarding to the pool; `SwapAllowlistExtension.beforeSwap` decodes and checks that address instead of (or in addition to) `sender`. Must be applied consistently across `exactInputSingle`, `exactInput`, `exactOutput`, and `exactOutputSingle`.

2. **Alternative — expose real sender to extensions.** The pool stores the original `msg.sender` in transient storage before any callback and exposes it to extensions, similar to how Uniswap v4 passes `msgSender` through hook calls.

## Proof of Concept
```
1. Deploy pool with SwapAllowlistExtension configured.
2. Admin calls setAllowedToSwap(pool, router, true)
   — intended to let allowlisted users swap via the router.
3. Attacker (not in allowlist) calls:
     router.exactInputSingle({pool: pool, ...})
4. Pool calls _beforeSwap(msg.sender=router, ...)
   [MetricOmmPool.sol L230-240]
5. ExtensionCalling._beforeSwap forwards sender=router verbatim
   [ExtensionCalling.sol L149-177]
6. SwapAllowlistExtension checks allowedSwapper[pool][router] == true → passes
   [SwapAllowlistExtension.sol L37]
7. Attacker's swap executes at oracle price against LP funds.
8. Allowlist invariant is broken; no end-user identity check was ever performed.
```
