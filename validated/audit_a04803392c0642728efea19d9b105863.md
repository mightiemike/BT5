### Title
Missing `payable receive()` in `ContractOwner` Permanently Breaks Native ETH Recovery from `DirectDepositV1` — (`File: core/contracts/ContractOwner.sol`)

---

### Summary

`ContractOwner.sol` lacks a `payable receive()` function. When `withdrawFromDirectDepositV1(subaccount, address(0))` is called to recover native ETH from a `DirectDepositV1` contract, `DirectDepositV1.withdrawNative()` attempts to push ETH to `ContractOwner` (its owner). Because `ContractOwner` cannot receive ETH, the transfer fails, the call reverts, and any native ETH held in a `DirectDepositV1` contract is permanently unrecoverable.

---

### Finding Description

`ContractOwner` is the deployer and owner of every `DirectDepositV1` (DDA) contract it creates via `createDirectDepositV1()`. [1](#0-0) 

`DirectDepositV1.withdrawNative()` is `onlyOwner`, meaning only `ContractOwner` can invoke it. The intended recovery path is `ContractOwner.withdrawFromDirectDepositV1(subaccount, address(0))`: [2](#0-1) 

This calls `DirectDepositV1.withdrawNative()`, which does: [3](#0-2) 

Here `msg.sender` is `ContractOwner`. Since `ContractOwner` defines no `receive()` or `fallback()` function and is not `payable`, the low-level `.call{value: balance}("")` returns `success = false`. The `require(success, "Failed to transfer native token to owner")` then reverts the entire call chain.

`ContractOwner` extends only `EIP712Upgradeable` and `OwnableUpgradeable`, neither of which provides a `receive()` function, and the contract body itself contains none: [4](#0-3) 

---

### Impact Explanation

Any native ETH that ends up in a `DirectDepositV1` contract is permanently locked. The only callable recovery function (`withdrawFromDirectDepositV1` with `token == address(0)`) always reverts. There is no alternative path: `withdrawNative()` is `onlyOwner`, and `ContractOwner` is the sole owner.

Concrete scenarios where ETH lands in a DDA:

1. **Pre-funded CREATE2 address + failed wrapping in constructor**: `createDirectDepositV1` uses `new DirectDepositV1{salt: bytes32(uint256(1))}(...)`. If the deterministic address is pre-funded with ETH and `wrappedNative.call` fails, the constructor emits `NativeTokenTransferFailed` and leaves ETH in the contract without reverting: [5](#0-4) 

2. **`selfdestruct` force-send**: ETH force-sent via `selfdestruct` bypasses the `receive()` function entirely, leaving ETH stranded with no recovery path.

---

### Likelihood Explanation

Medium. The `NativeTokenTransferFailed` event and the `withdrawFromDirectDepositV1(subaccount, address(0))` code path both exist precisely because the developers anticipated native ETH getting stuck. The recovery function is explicitly implemented but is unconditionally broken. Any operator who calls it to recover stuck ETH will always receive a revert.

---

### Recommendation

Add a `payable receive()` function to `ContractOwner.sol` so it can accept incoming ETH transfers from `DirectDepositV1.withdrawNative()`:

```solidity
receive() external payable {}
```

This mirrors the fix applied to `LidoV1_WETHGateway` in commit `8614e53` and is consistent with how `DirectDepositV1` itself handles incoming ETH. [6](#0-5) 

---

### Proof of Concept

1. Owner calls `ContractOwner.withdrawFromDirectDepositV1(subaccount, address(0))`.
2. `directDepositV1Address[subaccount]` resolves to a deployed DDA with `address(this).balance > 0`.
3. `DirectDepositV1(directDepositV1).withdrawNative()` is called; `msg.sender` = `ContractOwner`.
4. Inside `withdrawNative()`: `(bool success, ) = msg.sender.call{value: balance}("")` — `ContractOwner` has no `receive()`, so the EVM finds no matching function selector and no fallback; the call returns `success = false`.
5. `require(success, "Failed to transfer native token to owner")` reverts.
6. The entire `withdrawFromDirectDepositV1` call reverts. ETH remains locked in the DDA forever. [3](#0-2) [7](#0-6)

        external
        onlyOwner
    {
        address payable directDepositV1 = directDepositV1Address[subaccount];
        require(directDepositV1 != address(0), "no dda");
        if (token == address(0)) {
            uint256 preBalance = address(this).balance;
            DirectDepositV1(directDepositV1).withdrawNative();
            uint256 postBalance = address(this).balance;
            require(postBalance > preBalance, "empty");
            (bool success, ) = msg.sender.call{value: postBalance - preBalance}(
                ""
            );
            require(success, "xfer");

            DirectDepositV1(directDepositV1).withdraw(IIERC20Base(token));
            uint256 postBalance = IERC20Base(token).balanceOf(address(this));
            require(postBalance > preBalance, "empty");
            IERC20Base(token).safeTransfer(
                msg.sender,
                postBalance - preBalance
            );
        }
    }
