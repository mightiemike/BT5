### Title
Duplicate Position IDs in `MsgClaimTierRewards` Enable Repeated Bonus Reward Draining - (File: x/tieredrewards/keeper/msg_server.go)

### Summary
`ClaimTierRewards` loads all requested positions into an in-memory slice before processing them. There is no deduplication of `PositionIds`. When the same position ID appears multiple times, each duplicate entry holds a stale snapshot of the position's reward checkpoint (`LastBonusAccrual`, `LastEventSeq`). The first iteration claims and persists the updated checkpoint, but subsequent iterations re-process the stale snapshot, re-computing and re-paying the same bonus rewards from the `RewardsPoolName` module account.

### Finding Description

In `ClaimTierRewards`, all positions are loaded into a slice before any processing begins:

```go
positions := make([]types.PositionState, 0, len(msg.PositionIds))
for _, posId := range msg.PositionIds {
    pos, err := ms.getPositionState(ctx, posId)
    // ...
    positions = append(positions, pos)
}
totalBase, totalBonus, err := ms.claimRewardsAndUpdatesPositions(ctx, positions)
``` [1](#0-0) 

There is no deduplication of `msg.PositionIds`. If the same ID appears twice, two independent copies of the same `PositionState` are appended to `positions`, both carrying the same stale `LastBonusAccrual` and `LastEventSeq`.

`claimRewardsAndUpdatesPositions` then iterates over the slice, calling `processEventsAndClaimBonus` and `setPosition` for each entry:

```go
for i := range positions {
    pos := &positions[i]
    // ...
    bonus, err := k.processEventsAndClaimBonus(ctx, pos)
    // ...
    if err := k.setPosition(ctx, pos.Position, nil); err != nil { ... }
}
``` [2](#0-1) 

After the first iteration persists the updated checkpoint, the second iteration operates on the stale in-memory copy. Inside `processEventsAndClaimBonus`, the current-segment bonus is computed using `segmentStart = pos.LastBonusAccrual` (the old value from the stale copy):

```go
segmentStart := pos.LastBonusAccrual
// ...
if bonded && val.IsBonded() {
    bonus := k.computeSegmentBonus(*pos, tier, segmentStart, blockTime, currentRate)
    totalBonus = totalBonus.Add(bonus)
}
``` [3](#0-2) 

Because `segmentStart` is the same old timestamp in both iterations, the same bonus amount is computed and paid again via:

```go
if err := k.bankKeeper.SendCoinsFromModuleToAccount(ctx, types.RewardsPoolName, ownerAddr, bonusCoins); err != nil {
    return nil, err
}
``` [4](#0-3) 

The second `setPosition` call then overwrites the persisted checkpoint with the stale one, resetting `LastBonusAccrual` back to the old value and making the position eligible for the same bonus again in the next transaction. [5](#0-4) 

### Impact Explanation

An attacker who owns any tiered-rewards position can drain the `RewardsPoolName` module account. By submitting `MsgClaimTierRewards` with `PositionIds = [id, id, id, …, id]` (N repetitions), they receive N times the accrued bonus for that position in a single transaction. Because the final `setPosition` call writes back the stale checkpoint, the position's `LastBonusAccrual` is reset to the pre-claim value, making the same attack repeatable across multiple transactions. The corrupted value is the `RewardsPoolName` module account balance and the position's `LastBonusAccrual` / `LastEventSeq` checkpoint stored in `Positions`. [6](#0-5) 

### Likelihood Explanation

Any position owner can trigger this with a single standard `MsgClaimTierRewards` transaction. No privileged role, leaked key, or social engineering is required. The entry path is a normal, unprivileged Cosmos SDK message handler reachable from any CLI or RPC client. [7](#0-6) 

### Recommendation

Deduplicate `msg.PositionIds` before loading positions, or re-read each position's state from the store at the start of each iteration in `claimRewardsAndUpdatesPositions` rather than relying on the pre-loaded in-memory snapshot. The simplest fix is to add a duplicate-ID check in `MsgClaimTierRewards.Validate()` or at the top of the `ClaimTierRewards` handler. [8](#0-7) 

### Proof of Concept

1. Create a tiered-rewards position (via `MsgLockTier` or `MsgCommitDelegationToTier`) and wait for bonus rewards to accrue.
2. Submit `MsgClaimTierRewards` with `Owner = <attacker>` and `PositionIds = [posId, posId, posId, …]` (e.g., 100 repetitions).
3. Observe that `processEventsAndClaimBonus` is called 100 times on the same stale snapshot, each time sending the full accrued bonus from `RewardsPoolName` to the attacker.
4. Observe that the final `setPosition` writes back the stale `LastBonusAccrual`, so the same attack can be repeated in the next block. [9](#0-8)

func (ms msgServer) ClaimTierRewards(ctx context.Context, msg *types.MsgClaimTierRewards) (*types.MsgClaimTierRewardsResponse, error) {
	if err := msg.Validate(); err != nil {
		return nil, err
	}

	positions := make([]types.PositionState, 0, len(msg.PositionIds))
	for _, posId := range msg.PositionIds {
		pos, err := ms.getPositionState(ctx, posId)
		if err != nil {
			return nil, err
		}

		if err := ms.validateClaimRewards(pos.Position, msg.Owner); err != nil {
			return nil, err
		}

		positions = append(positions, pos)
	}

	totalBase, totalBonus, err := ms.claimRewardsAndUpdatesPositions(ctx, positions)
	if err != nil {
		return nil, err
	}

	sdkCtx := sdk.UnwrapSDKContext(ctx)
	if err := sdkCtx.EventManager().EmitTypedEvent(&types.EventTierRewardsClaimed{
		Owner:        msg.Owner,
		PositionIds:  msg.PositionIds,
		BaseRewards:  totalBase,
		BonusRewards: totalBonus,
	}); err != nil {
		return nil, err
	}

	return &types.MsgClaimTierRewardsResponse{
		BaseRewards:  totalBase,
		BonusRewards: totalBonus,
		PositionIds:  msg.PositionIds,
	}, nil
}

func (k Keeper) claimRewardsAndUpdatesPositions(ctx context.Context, positions []types.PositionState) (sdk.Coins, sdk.Coins, error) {
	totalBase := sdk.NewCoins()
	totalBonus := sdk.NewCoins()

	for i := range positions {
		pos := &positions[i]

		if !pos.IsDelegated() {
			continue
		}

		base, err := k.claimBaseRewards(ctx, *pos)
		if err != nil {
			return nil, nil, err
		}
		totalBase = totalBase.Add(base...)

		bonus, err := k.processEventsAndClaimBonus(ctx, pos)
		if err != nil {
			return nil, nil, err
		}
		totalBonus = totalBonus.Add(bonus...)

		if err := k.setPosition(ctx, pos.Position, nil); err != nil {
			return nil, nil, err
		}
	}

	return totalBase, totalBonus, nil
}
