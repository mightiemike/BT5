### Title
Bonus checkpoint advancement before pool balance check in `processEventsAndClaimBonus` causes permanent loss of accrued bonus when `slashRedelegationPosition` swallows `ErrInsufficientBonusPool` — (`x/tieredrewards/keeper/claim_rewards.go`, `slash.go`)

### Summary
`processEventsAndClaimBonus` mutates the position's `LastEventSeq`, `LastBonusAccrual`, and `LastKnownBonded` checkpoints and decrements validator event reference counts in the store **before** checking whether the rewards pool has sufficient balance. When called from `slashRedelegationPosition`, which deliberately swallows `ErrInsufficientBonusPool` to avoid a chain halt, the mutated position is then persisted via `setPositionWithState`. The result is that the position's bonus accrual window is permanently advanced with no bonus paid, and the consumed events cannot be replayed.

### Finding Description
Inside `processEventsAndClaimBonus`:

1. The event loop (lines 172–198) calls `pos.UpdateLastEventSeq(entry.Seq)` and `k.decrementEventRefCount(ctx, valAddr, entry.Seq)` for every pending event. These are **unconditional** — they execute regardless of whether the pool will later be found sufficient.
2. After the loop, `applyBonusAccrualCheckpoint` (line 215) and `pos.UpdateLastKnownBonded` (line 217) advance the remaining checkpoints.
3. Only at line 230 does `sufficientBonusPoolBalance` run. If the pool is short, the function returns `nil, ErrInsufficientBonusPool` — but `pos` is already fully mutated and the store ref counts are already decremented.

In `slashRedelegationPosition` (slash.go lines 54–64), the `ErrInsufficientBonusPool` branch is explicitly swallowed. Execution continues to line 71 or 77, where `setPositionWithState(ctx, pos, ...)` persists the mutated position. The accrued bonus for the entire period since `LastBonusAccrual` is silently discarded. [1](#0-0) [2](#0-1) [3](#0-2) [4](#0-3) 

### Impact Explanation
The position owner permanently loses all bonus rewards accrued since their last claim up to the moment of the redelegation slash. The checkpoints are advanced so a subsequent `ClaimTierRewards` call starts from the post-slash block time, yielding zero bonus for the lost window. The consumed event ref counts mean those events cannot be replayed. There is no recourse for the owner.

### Likelihood Explanation
The trigger is a validator redelegation slash — a real, externally reachable staking event. The pool being below the computed bonus is a realistic operational condition (pool can be drained by prior claims or simply underfunded). No privileged access is required; the attacker only needs to hold a position on a validator that undergoes a redelegation slash while the pool is insufficient.

### Recommendation
Move the `sufficientBonusPoolBalance` check **before** any checkpoint mutations and store writes. Compute `totalBonus` first, check pool sufficiency, and only then advance `LastEventSeq`, decrement ref counts, update `LastBonusAccrual`, and `LastKnownBonded`. Alternatively, in `slashRedelegationPosition`, when `ErrInsufficientBonusPool` is caught, do **not** call `setPositionWithState` with the mutated `pos` — instead reload the original position from store and persist only the slash-related share adjustment, leaving bonus checkpoints untouched.

### Proof of Concept
1. Set up a keeper test with a `RewardsPoolName` balance of 0 (or below the computed bonus).
2. Create a position on validator V with a non-zero accrued bonus window.
3. Trigger `BeforeRedelegationSlashed` (via `Hooks{}.BeforeRedelegationSlashed`) with a valid `unbondingId` mapped to the position.
4. Assert that `pos.LastBonusAccrual` advanced to the current block time and `pos.LastEventSeq` advanced past all pending events.
5. Assert that the owner's balance did not increase.
6. Call `ClaimTierRewards` for the same position and assert that bonus returned is zero — confirming the accrual window was permanently consumed without payment. [5](#0-4) [6](#0-5)

```go
func (k Keeper) slashRedelegationPosition(ctx context.Context, unbondingId uint64, sharesToUnbond math.LegacyDec) error {
	positionId, err := k.getRedelegationMapping(ctx, unbondingId)
	if errors.Is(err, collections.ErrNotFound) {
		return nil
	}
	if err != nil {
		return err
	}

	pos, err := k.getPositionState(ctx, positionId)
	if errors.Is(err, types.ErrPositionNotFound) {
		k.logger(ctx).Error("position not found during redelegation slash",
			"position_id", positionId,
			"unbonding_id", unbondingId,
			"error", err.Error(),
		)
		return nil
	}
	if err != nil {
		return err
	}

	if !pos.IsDelegated() {
		// Defensive
		k.logger(ctx).Error("delegation missing during BeforeRedelegationSlashed",
			"position_id", positionId,
			"unbonding_id", unbondingId,
			"shares_to_unbond", sharesToUnbond.String(),
		)
		return nil
	}

	dstValStr := pos.Delegation.ValidatorAddress

	// Settle bonus against PRE-slash shares.
	if _, err := k.processEventsAndClaimBonus(ctx, &pos); err != nil {
		// Deliberately forgo bonus rewards if pool is insufficient to prevent chain halt.
		if errors.Is(err, types.ErrInsufficientBonusPool) {
			k.logger(ctx).Error("insufficient bonus pool during redelegation slash",
				"position_id", pos.Id,
				"error", err.Error(),
			)
		} else {
			return err
		}
	}

	fullSlash := sharesToUnbond.GTE(pos.Delegation.Shares)

	if fullSlash {
		pos.Delegation = nil
		pos.ClearBonusCheckpoints()
		return k.setPositionWithState(ctx, pos, &ValidatorTransition{PreviousAddress: dstValStr})
	}
	// In-memory only: the persisted Position carries no share count, and the
	// live delegation will reflect the post-Unbond shares on the next read.
	// Update the local copy so any follow-up logic in this call sees consistent state.
	pos.Delegation.Shares = pos.Delegation.Shares.Sub(sharesToUnbond)
	return k.setPositionWithState(ctx, pos, nil)
```
