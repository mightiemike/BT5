### Title
Accumulated NEAR from GC-freed Storage Deposits Cannot Be Retrieved — (`contract/src/lib.rs`)

### Summary
`submit_blocks` collects NEAR storage deposits from relayers. `run_mainchain_gc` later frees that storage, returning the locked NEAR to the contract's own account balance. No function exists to withdraw this accumulated balance, permanently trapping NEAR tokens in the contract.

### Finding Description
`submit_blocks` is `#[payable]` and requires callers to cover the storage cost of newly inserted block headers. After processing, it refunds only the excess from the *current call*: [1](#0-0) 

The refund path (`Promise::new(env::predecessor_account_id()).transfer(refund)`) only handles the surplus from the immediate deposit. It does not address NEAR that accumulates later.

`run_mainchain_gc` is called automatically inside every `submit_blocks` invocation and removes old block headers from storage: [2](#0-1) [3](#0-2) 

In NEAR Protocol, when storage entries are deleted, the storage-staking deposit locked for those bytes is returned to the contract account's available balance. Every GC cycle therefore increases the contract's free balance by the storage cost of the removed headers — NEAR that was originally paid by relayers.

A search across all contract source files (`contract/src/*.rs`) finds exactly one `transfer` call (the per-call refund in `submit_blocks`) and no `rescue`, `withdraw`, or `account_balance` withdrawal path: [4](#0-3) 

There is no privileged or unprivileged function that can move this accumulated balance out of the contract.

### Impact Explanation
NEAR tokens paid by relayers for storage that is subsequently freed by GC are permanently locked in the contract with no recovery path. Over the operational lifetime of the contract — which is designed to run indefinitely with a rolling `gc_threshold` window — the trapped amount grows proportionally to the volume of submitted blocks. The default `gc_threshold` of 52,704 blocks means roughly one year of headers are retained; every block beyond that threshold is GC'd and its storage deposit is silently absorbed into the contract's inaccessible balance.

### Likelihood Explanation
GC runs on every `submit_blocks` call once the stored chain exceeds `gc_threshold`. The relayer service is designed to submit blocks continuously. This means the accumulation begins automatically in normal production operation without any adversarial action required. [5](#0-4) 

### Recommendation
Add a privileged withdrawal function (e.g., restricted to `Role::DAO`) that transfers the contract's available balance to a designated destination:

```rust
pub fn rescue_near(&mut self, destination: AccountId, amount: NearToken) {
    // restrict to DAO or owner role
    Promise::new(destination).transfer(amount)
}
```

Alternatively, refund the storage-freed NEAR back to the original depositor inside `run_mainchain_gc` by tracking per-relayer deposits, or implement a simpler sweep using `env::account_balance()` as the transfer amount.

### Proof of Concept

1. Relayer calls `submit_blocks` with 85 headers and attaches `85 × storage_byte_cost × header_size` NEAR. The contract stores the headers and refunds zero (deposit exactly covers storage).
2. After enough blocks accumulate, `run_mainchain_gc` removes the oldest 85 headers on the next `submit_blocks` call. NEAR Protocol returns the storage-staking deposit for those 85 headers to the contract's account balance.
3. The contract's available balance is now elevated by that freed deposit.
4. No function in `BtcLightClient` can transfer this balance out. The only outbound transfer is the per-call excess refund in `submit_blocks`, which is `0` when the deposit exactly matches the new storage cost.
5. Repeat indefinitely: every GC cycle traps more NEAR. [6](#0-5) [7](#0-6)

    pub fn submit_blocks(
        &mut self,
        #[serializer(borsh)] headers: Vec<BlockHeader>,
    ) -> PromiseOrValue<()> {
        let amount = env::attached_deposit();
        let initial_storage = env::storage_usage();
        let num_of_headers = headers.len().try_into().unwrap();

        for header in headers {
            self.submit_block_header(header, self.skip_pow_verification);
        }

        self.run_mainchain_gc(num_of_headers);
        let diff_storage_usage = env::storage_usage().saturating_sub(initial_storage);
        let required_deposit = env::storage_byte_cost().saturating_mul(diff_storage_usage.into());

        require!(
            amount >= required_deposit,
            format!("Required deposit {}", required_deposit)
        );

        let refund = amount.saturating_sub(required_deposit);
        if refund > NearToken::from_near(0) {
            Promise::new(env::predecessor_account_id())
                .transfer(refund)
                .into()
        } else {
            PromiseOrValue::Value(())
        }
    }

    pub fn run_mainchain_gc(&mut self, batch_size: u64) {
        let initial_blockheader = self
            .headers_pool
            .get(&self.mainchain_initial_blockhash)
            .unwrap_or_else(|| env::panic_str(ERR_KEY_NOT_EXIST));

        let tip_blockheader = self
            .headers_pool
            .get(&self.mainchain_tip_blockhash)
            .unwrap_or_else(|| env::panic_str(ERR_KEY_NOT_EXIST));

        let amount_of_headers_we_store =
            tip_blockheader.block_height - initial_blockheader.block_height + 1;

        if amount_of_headers_we_store > self.gc_threshold {
            let total_amount_to_remove = amount_of_headers_we_store - self.gc_threshold;
            let selected_amount_to_remove = std::cmp::min(total_amount_to_remove, batch_size);

            let start_removal_height = initial_blockheader.block_height;
            let end_removal_height = initial_blockheader.block_height + selected_amount_to_remove;
            env::log_str(&format!(
                "Num of blocks to remove {selected_amount_to_remove}"
            ));

            for height in start_removal_height..end_removal_height {
                let blockhash = &self
                    .mainchain_height_to_header
                    .get(&height)
                    .unwrap_or_else(|| env::panic_str(ERR_KEY_NOT_EXIST));

                self.remove_block_header(blockhash);
                self.mainchain_height_to_header.remove(&height);
            }

            self.mainchain_initial_blockhash = self
                .mainchain_height_to_header
                .get(&end_removal_height)
                .unwrap_or_else(|| env::panic_str(ERR_KEY_NOT_EXIST));
        }
    }
