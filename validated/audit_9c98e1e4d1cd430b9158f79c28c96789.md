### Title
Attacker Can Frontrun `MsgIssueDenom` to Steal NFT Denom Creator Rights - (File: x/nft/keeper/denom.go)

### Summary

The `x/nft` module accepts a fully user-chosen `denom_id` (and `name`) in `MsgIssueDenom` with no binding between the sender address and the requested identifier. `SetDenom` rejects any second registration of the same ID, so an attacker who observes a pending `MsgIssueDenom` in the mempool and rebroadcasts it with a higher gas price will have their transaction included first. The attacker permanently becomes the `creator` of the denom, gaining exclusive minting rights, while the legitimate user's transaction is rejected.

### Finding Description

`MsgIssueDenom` is an open, permissionless Cosmos SDK message. Its handler in `msgServer.IssueDenom` extracts `msg.Sender` and passes it as the `creator` to `k.IssueDenom`, which calls `SetDenom`:

```go
// x/nft/keeper/msg_server.go:25-34
func (m msgServer) IssueDenom(goCtx context.Context, msg *types.MsgIssueDenom) (*types.MsgIssueDenomResponse, error) {
    sender, err := sdk.AccAddressFromBech32(msg.Sender)
    ...
    if err := m.Keeper.IssueDenom(ctx, msg.Id, msg.Name, msg.Schema, msg.Uri, sender); err != nil {
        return nil, err
    }
```

`SetDenom` enforces uniqueness on both `denom_id` and `name` but performs no check that ties either identifier to the sender:

```go
// x/nft/keeper/denom.go:26-33
func (k Keeper) SetDenom(ctx sdk.Context, denom types.Denom) error {
    if k.HasDenomID(ctx, denom.Id) {
        return sdkerrors.Wrapf(types.ErrInvalidDenom, "denomID %s has already exists", denom.Id)
    }
    if k.HasDenomNm(ctx, denom.Name) {
        return sdkerrors.Wrapf(types.ErrInvalidDenom, "denomName %s has already exists", denom.Name)
    }
```

Because `denom_id` is a free-form lowercase alphanumeric string chosen entirely by the user (validated only by `ValidateDenomID`), any account can race to register any valid identifier. The first transaction committed to a block wins, and the winner's address is permanently stored as `creator`.

The `creator` field is the sole authorization gate for all subsequent privileged operations on the denom: `MintNFT`, `EditNFT`, and `BurnNFT` all call `IsDenomCreator` and reject any sender whose address does not match:

```go
// x/nft/keeper/keeper.go:72-81
func (k Keeper) MintNFT(...) error {
    _, err := k.IsDenomCreator(ctx, denomID, sender)
    if err != nil {
        return err
    }
    return k.MintNFTUnverified(...)
}
```

### Impact Explanation

The corrupted on-chain value is the `creator` field of the `Denom` object stored under `KeyDenomID(id)`. Once set, it cannot be changed; there is no `UpdateDenom` or ownership-transfer path in the module. The attacker who wins the race:

1. Permanently holds the creator role for the stolen denom ID.
2. Can mint arbitrary NFTs under that denom ID, impersonating the legitimate project's collection.
3. Can prevent the legitimate creator from ever using their intended denom ID or name on this chain.

The legitimate user's `MsgIssueDenom` is rejected with `"denomID X has already exists"`, and because both `denom_id` and `denom_name` must be globally unique, the user cannot recover by reusing either identifier.

This exceeds a pure denial-of-service: the attacker acquires a concrete, persistent capability (exclusive NFT minting rights under the stolen denom) that belongs to the legitimate creator.

### Likelihood Explanation

Cosmos SDK nodes broadcast transactions to the peer-to-peer gossip network before block inclusion. Any full node or validator can observe pending transactions in its local mempool. An attacker running a full node can detect an incoming `MsgIssueDenom`, construct an identical message signed by their own key, and submit it with a marginally higher gas price. With ~5-second block times, the race window is narrow but real, and the cost is only a transaction fee. The attack is especially attractive against high-profile NFT project launches where the target denom ID is publicly announced off-chain before the on-chain transaction is sent.

### Recommendation

Derive the `denom_id` deterministically from the sender's address (e.g., `sha256(sender || sequence)`) inside the message server, removing the user's ability to choose an arbitrary identifier. Alternatively, enforce that the `denom_id` must contain or be prefixed by the sender's bech32 address, so no two senders can collide on the same ID.

### Proof of Concept

```
1. Alice runs a full node and watches the mempool.
2. Bob broadcasts:
       MsgIssueDenom { id: "myproject", name: "My Project NFT", sender: bob }
3. Alice sees Bob's pending transaction and immediately broadcasts:
       MsgIssueDenom { id: "myproject", name: "My Project NFT", sender: alice }
   with a higher gas price so validators prefer it.
4. Alice's transaction is included first.
   State: Denom{Id:"myproject", Creator: alice, ...} stored on-chain.
5. Bob's transaction is processed next and fails:
       "denomID myproject has already exists: invalid denom"
6. Alice can now call MintNFT(denomId="myproject", ..., sender=alice) freely,
   minting NFTs that appear to belong to the "myproject" collection.
   Bob can never use the id "myproject" or the name "My Project NFT" on this chain.
```

func (k Keeper) SetDenom(ctx sdk.Context, denom types.Denom) error {
	if k.HasDenomID(ctx, denom.Id) {

	return nil

// IsDenomCreator checks if address is the creator of Denom
// Return the Denom if true, an error otherwise
func (k Keeper) IsDenomCreator(ctx sdk.Context, denomID string, address sdk.AccAddress) (types.Denom, error) {
	denom, err := k.GetDenom(ctx, denomID)
	if err != nil {
		return types.Denom{}, err
	}

	creator, err := sdk.AccAddressFromBech32(denom.Creator)
	if err != nil {
		panic(err)
	}

	if !creator.Equals(address) {
		return types.Denom{}, sdkerrors.Wrapf(types.ErrUnauthorized, "%s is not the creator of %s", address, denomID)
	}

	return denom, nil

func (m msgServer) IssueDenom(goCtx context.Context, msg *types.MsgIssueDenom) (*types.MsgIssueDenomResponse, error) {
	sender, err := sdk.AccAddressFromBech32(msg.Sender)
	if err != nil {
		return nil, err
	}

	ctx := sdk.UnwrapSDKContext(goCtx)
	if err := m.Keeper.IssueDenom(ctx, msg.Id, msg.Name, msg.Schema, msg.Uri, sender); err != nil {
		return nil, err
	}

	ctx.EventManager().EmitEvents(sdk.Events{
		sdk.NewEvent(
			types.EventTypeIssueDenom,
			sdk.NewAttribute(types.AttributeKeyDenomID, msg.Id),
			sdk.NewAttribute(types.AttributeKeyDenomName, msg.Name),
			sdk.NewAttribute(types.AttributeKeyCreator, msg.Sender),
		),
		sdk.NewEvent(
			sdk.EventTypeMessage,
			sdk.NewAttribute(sdk.AttributeKeyModule, types.AttributeValueCategory),
			sdk.NewAttribute(sdk.AttributeKeySender, msg.Sender),
		),
	})

	return &types.MsgIssueDenomResponse{}, nil
}

// ValidateDenomID verifies whether the parameters are legal.
func ValidateDenomID(denomID string) error {
	if len(denomID) < MinDenomLen || len(denomID) > MaxDenomLen {
		return sdkerrors.Wrapf(ErrInvalidDenom, "the length of denom(%s) only accepts value [%d, %d]", denomID, MinDenomLen, MaxDenomLen)
	}
	if !IsBeginWithAlpha(denomID) || !IsAlphaNumeric(denomID) {
		return sdkerrors.Wrapf(ErrInvalidDenom, "the denom(%s) only accepts lowercase alphanumeric characters, and begin with an english letter", denomID)
	}
	return nil
}
