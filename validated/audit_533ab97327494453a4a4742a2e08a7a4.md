Audit Report

## Title
`SwapAllowlistExtension` Checks Router Address Instead of Actual Swapper, Allowing Any User to Bypass the Per-User Swap Gate - (File: metric-periphery/contracts/extensions/SwapAllowlistExtension.sol)

## Summary

`SwapAllowlistExtension.beforeSwap` gates swaps by checking the `sender` argument, which `MetricOmmPool.swap` sets to `msg.sender` of the `pool.swap()` call—the router contract, not the end user, when routing through `MetricOmmSimpleRouter`. If the router is allowlisted for a pool (required for any pool that wants to support router-mediated swaps for its allowlisted users), every user—regardless of individual allowlist status—can execute swaps on that restricted pool by calling the router.

## Finding Description

`SwapAllowlistExtension.beforeSwap` performs its identity check as:

```solidity
if (!allowAllSwappers[msg.sender] && !allowedSwapper[msg.sender][sender]) {
    revert IMetricOmmPoolActions.NotAllowedToSwap();
}
```

`msg.sender` here is the pool, confirmed by the `onlyPool` modifier in `BaseMetricExtension` which validates `IMetricOmmPoolFactory(FACTORY).isPool(msg.sender)`. `sender` is the first argument, which `MetricOmmPool.swap` sets to `msg.sender` of the pool call:

```solidity
_beforeSwap(
    msg.sender,   // ← whoever called pool.swap()
    recipient, ...
);
```

`ExtensionCalling._beforeSwap` passes this `sender` directly into `abi.encodeCall(IMetricOmmExtensions.beforeSwap, (sender, ...))`, and `CallExtension.callExtension` uses a regular `call` (not `delegatecall`), so `msg.sender` in the extension is always the pool.

When a user calls `MetricOmmSimpleRouter.exactInputSingle`, the router calls `pool.swap()` directly:

```solidity
(int128 amount0Delta, int128 amount1Delta) = IMetricOmmPoolActions(params.pool)
    .swap(params.recipient, params.zeroForOne, ..., "", params.extensionData);
```

The pool receives `msg.sender = router`. It passes `router` as `sender` to the extension. The extension evaluates `allowedSwapper[pool][router]`—the router's allowlist status—not the actual user's. The `extensionData` parameter is unnamed and ignored in `beforeSwap`'s signature, so there is no secondary path through which the actual user identity reaches the guard.

## Impact Explanation

A pool configured with `SwapAllowlistExtension` to restrict swapping to a curated set of addresses (e.g., KYC'd counterparties, whitelisted market makers) loses that restriction entirely for any user who routes through `MetricOmmSimpleRouter`. The unauthorized swapper can execute swaps against pool liquidity at oracle-derived prices, extracting value from LPs who deposited under the assumption that only vetted counterparties could trade. This constitutes direct loss of LP principal and a broken core pool invariant (the access-controlled swap gate), matching the "Broken core pool functionality causing loss of funds" and "Admin-boundary break bypassed by an unprivileged path" impact categories.

## Likelihood Explanation

The bypass requires the router to be allowlisted for the target pool. This is a natural and expected configuration: any pool admin who wants allowlisted users to be able to use the standard periphery router must call `setAllowedToSwap(pool, router, true)`. The moment they do, the per-user gate collapses to a single shared entry. The attacker needs no special privileges—only the ability to call a public periphery function (`exactInputSingle`, `exactInput`, `exactOutputSingle`, or `exactOutput`).

## Recommendation

The extension must check the identity of the economic actor, not the immediate caller of the pool. The preferred fix is to have the router encode `msg.sender` into `extensionData` before calling `pool.swap()`, and have `SwapAllowlistExtension.beforeSwap` decode and check it when `sender` is a known router address. Alternatively, document and enforce that pools using `SwapAllowlistExtension` must never allowlist shared router contracts, and that allowlisted users must call `pool.swap()` directly—though this is a fragile operational constraint, not a code fix.

## Proof of Concept

```
Setup:
  - Deploy pool with SwapAllowlistExtension as beforeSwap hook.
  - setAllowedToSwap(pool, router, true)   // admin allowlists the router
  - setAllowedToSwap(pool, alice, false)   // alice is NOT allowlisted

Attack:
  1. alice calls MetricOmmSimpleRouter.exactInputSingle({pool: pool, ...})
  2. Router calls pool.swap(recipient, ...) — msg.sender to pool = router
  3. Pool calls _beforeSwap(router, recipient, ...)
  4. ExtensionCalling encodes (router, ...) and calls extension
  5. Extension evaluates allowedSwapper[pool][router] == true → passes
  6. Swap executes; alice receives output tokens.

Expected: revert NotAllowedToSwap (alice is not allowlisted)
Actual:   swap succeeds (router is allowlisted, alice's identity is never checked)
