### Title
Caller-Supplied `tx_id` Accepted as Internal Merkle Node Enables False Transaction Inclusion Proof - (`contract/src/lib.rs`)

### Summary

`verify_transaction_inclusion` accepts a caller-supplied `tx_id` that is never validated to be a leaf-level transaction hash. An unprivileged NEAR caller can supply an internal Merkle tree node hash as `tx_id` with a shortened proof path, causing the function to return `true` and falsely certify that a non-existent Bitcoin transaction was included in a real block.

### Finding Description

`verify_transaction_inclusion` is still a live, publicly callable NEAR contract method despite its `#[deprecated]` annotation. The `#[deprecated]` attribute in Rust is only a compiler hint — it imposes no runtime access restriction. The function accepts three caller-controlled inputs: `tx_id`, `tx_index`, and `merkle_proof`. [1](#0-0) 

The only check performed is whether `compute_root_from_merkle_proof(tx_id, tx_index, merkle_proof)` equals the stored `header.block_header.merkle_root`. [2](#0-1) 

`compute_root_from_merkle_proof` is a pure hash-chaining function with no constraint on what `transaction_hash` represents: [3](#0-2) 

Bitcoin's Merkle tree is a binary tree. Every internal node is itself a valid `double_sha256` digest. If an attacker supplies an internal node at depth `D-k` as `tx_id`, they only need `k` proof elements (not the full `D` elements for a leaf) to reach the root. The function accepts any proof length ≥ 1: [4](#0-3) 

The code's own warning acknowledges this broken invariant: [5](#0-4) 

The analog to the external report is direct: in the DCA contract, only `token_in` and `token_out` (the endpoints) are validated while the intermediate path token is unconstrained. Here, only the final Merkle root (the endpoint) is validated while the starting input `tx_id` (the "intermediate" in the proof chain) is unconstrained — an attacker controls the starting node of the hash chain and can make it an internal tree node rather than a real transaction leaf.

### Impact Explanation

Any downstream NEAR contract or bridge that calls `verify_transaction_inclusion` to gate a financial action (e.g., releasing wrapped BTC, confirming a cross-chain deposit) can be deceived into accepting a fabricated proof. The attacker does not need to mine any Bitcoin block — they only need to know the Merkle tree structure of any real, already-confirmed Bitcoin block (fully public on-chain data). They can then construct a proof that an internal node hash "is" a transaction, causing the contract to return `true` for a transaction that never existed.

### Likelihood Explanation

- The function is `pub` with no role guard, callable by any NEAR account.
- `#[pause]` is the only runtime gate, and the contract is not paused by default.
- All required inputs (block Merkle tree structure, internal node hashes) are freely available from any Bitcoin block explorer.
- No privileged access, leaked keys, or social engineering is required.

### Recommendation

Remove `verify_transaction_inclusion` entirely from the public ABI, or add an `#[private]` attribute to prevent external calls. All callers should be migrated to `verify_transaction_inclusion_v2`, which adds the coinbase proof length check that defeats the internal-node substitution attack. [6](#0-5) 

### Proof of Concept

1. Select any real, confirmed Bitcoin block whose Merkle tree has depth ≥ 2. Obtain its `merkle_root` and the full transaction list (public data).
2. Compute the internal node `N` at depth 1 (the hash of the left and right children of the root): `N = double_sha256(child_left || child_right)`.
3. Call `verify_transaction_inclusion` on the NEAR contract with:
   - `tx_block_blockhash` = the real block hash (already in the contract's `headers_pool`)
   - `tx_id` = `N` (the internal node, not a real transaction)
   - `tx_index` = `0` (even position)
   - `merkle_proof` = `[sibling_of_N]` (one element: the sibling internal node at depth 1)
   - `confirmations` = `1`
4. `compute_root_from_merkle_proof(N, 0, [sibling])` = `double_sha256(N || sibling)` = the real `merkle_root`.
5. The function returns `true`, falsely certifying that `N` (a non-existent transaction) is included in the block. [7](#0-6)

    pub fn verify_transaction_inclusion(&self, #[serializer(borsh)] args: ProofArgs) -> bool {
        require!(
            args.confirmations <= self.gc_threshold,
            "The required number of confirmations exceeds the number of blocks stored in memory"
        );

        let heaviest_block_header = self
            .headers_pool
            .get(&self.mainchain_tip_blockhash)
            .unwrap_or_else(|| env::panic_str(ERR_KEY_NOT_EXIST));
        let target_block_height = self
            .mainchain_header_to_height
            .get(&args.tx_block_blockhash)
            .unwrap_or_else(|| env::panic_str("block does not belong to the current main chain"));

        // Check requested confirmations. No need to compute proof if insufficient confirmations.
        require!(
            (heaviest_block_header.block_height).saturating_sub(target_block_height) + 1
                >= args.confirmations,
            "Not enough blocks confirmed"
        );

        let header = self
            .headers_pool
            .get(&args.tx_block_blockhash)
            .unwrap_or_else(|| env::panic_str("cannot find requested transaction block"));

        require!(!args.merkle_proof.is_empty(), "Merkle proof is empty");

        // compute merkle tree root and check if it matches block's original merkle tree root
        merkle_tools::compute_root_from_merkle_proof(
            args.tx_id,
            usize::try_from(args.tx_index).unwrap(),
            &args.merkle_proof,
        ) == header.block_header.merkle_root
    }

                args.coinbase_tx_id.clone(),
                0usize,

pub fn compute_root_from_merkle_proof(
    transaction_hash: H256,
    transaction_position: usize,
    merkle_proof: &Vec<H256>,
) -> H256 {
    let mut current_hash = transaction_hash;
    let mut current_position = transaction_position;

    for proof_hash in merkle_proof {
        if current_position % 2 == 0 {
            current_hash = compute_hash(&current_hash, proof_hash);
        } else {
            current_hash = compute_hash(proof_hash, &current_hash);
        }
        current_position /= 2;
    }

    current_hash
}
