### Title
Non-Bond-Denom Tokens Sent to `RewardsPool` Module Account Are Permanently Irrecoverable — (`x/tieredrewards/keeper/abci.go`, `x/tieredrewards/keeper/claim_rewards.go`)

---

### Summary

The `tieredrewards` module's `RewardsPool` module account is explicitly whitelisted to receive external bank sends from any unprivileged account. However, every code path that drains the pool operates exclusively on the **bond denom** (`basecro`). Any other token denomination sent to the pool address is permanently locked with no governance message, admin sweep, or any other recovery path.

---

### Finding Description

`app/app.go` explicitly whitelists `tieredrewardstypes.RewardsPoolName` in `moduleAccsAllowedToReceiveExternalFunds`, bypassing the Cosmos SDK's default block on `MsgSend` to module accounts: [1](#0-0) 

This means any user can send any token denomination to the `RewardsPool` address via a standard `MsgSend` transaction.

The two code paths that ever move tokens **out** of the pool are:

1. **`topUpBaseRewards` (BeginBlocker)** — reads only the bond-denom balance and sends only bond-denom coins to the distribution module: [2](#0-1) 

2. **`processEventsAndClaimBonus`** — constructs bonus coins exclusively as `sdk.NewCoin(bondDenom, totalBonus)` and sends only that: [3](#0-2) 

Neither path ever touches non-bond-denom balances. The full message set for the module (`MsgUpdateParams`, `MsgAddTier`, `MsgUpdateTier`, `MsgDeleteTier`, `MsgLockTier`, `MsgCommitDelegationToTier`, `MsgTierUndelegate`, `MsgTierRedelegate`, `MsgAddToTierPosition`, `MsgTriggerExitFromTier`, `MsgClearPosition`, `MsgClaimTierRewards`, `MsgWithdrawFromTier`, `MsgExitTierWithDelegation`) contains no sweep or recovery message for the pool account: [4](#0-3) 

The `sufficientBonusPoolBalance` helper calls `GetAllBalances` but only checks whether the pool holds enough of the bond denom; non-bond-denom balances are visible in the query response but are never consumed: [5](#0-4) 

---

### Impact Explanation

Any token denomination other than `basecro` (the bond denom) that reaches the `RewardsPool` module account is permanently irrecoverable. The corrupted value is the full balance of the non-bond-denom token at the `RewardsPool` address. There is no on-chain path — privileged or unprivileged — to move those tokens out.

---

### Likelihood Explanation

The `RewardsPool` address is deterministic and publicly derivable. The pool is documented as the target for funding staking rewards, making it a natural destination for operators or integrators who may accidentally send the wrong denomination (e.g., an IBC voucher, a wrapped asset, or a governance token). The entry path is a plain `MsgSend` requiring no special role.

---

### Recommendation

Add a governance-gated message (e.g., `MsgSweepRewardsPool`) that allows the `authority` (x/gov) to send a specified set of non-bond-denom coins from `RewardsPoolName` to a designated address. Alternatively, restrict the `moduleAccsAllowedToReceiveExternalFunds` whitelist to enforce that only the bond denom can be sent to the pool, rejecting other denominations at the `AnteHandler` or bank `SendRestriction` layer.

---

### Proof of Concept

1. Derive the `RewardsPool` module address (SHA-256 of `"rewards_pool"`, first 20 bytes, bech32-encoded).
2. Submit a standard `MsgSend` from any account sending `1000ibc/XXXXXXXX` (any IBC voucher or non-bond denom) to that address.
3. The transaction succeeds because `moduleAccsAllowedToReceiveExternalFunds[RewardsPoolName] == true`.
4. Query the pool balance via `QueryRewardsPoolBalances` — the IBC voucher appears in `balances`.
5. Wait for any number of blocks: `BeginBlocker` only drains `basecro`; `MsgClaimTierRewards` only pays out `basecro`. The IBC voucher balance never decreases.
6. No message in the module's transaction set can move those coins out. The tokens are permanently locked. [6](#0-5)

```go
func (k Keeper) sufficientBonusPoolBalance(ctx context.Context, bonus sdk.Coins) error {
	if bonus.IsZero() {
		return nil
	}

	poolAddr := k.accountKeeper.GetModuleAddress(types.RewardsPoolName)
	poolBalance := k.bankKeeper.GetAllBalances(ctx, poolAddr)
	if !poolBalance.IsAllGTE(bonus) {
		return errorsmod.Wrapf(types.ErrInsufficientBonusPool,
			"bonus: %s, pool balance: %s",
			bonus.String(), poolBalance.String())
	}

	return nil
```

```go
func (q queryServer) RewardsPoolBalances(ctx context.Context, _ *types.QueryRewardsPoolBalancesRequest) (*types.QueryRewardsPoolBalancesResponse, error) {
```
