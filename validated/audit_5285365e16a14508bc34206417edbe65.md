### Title
Non-idempotent `delete_all(DBCol::BlockHeader)` in `delete_old_block_headers` permanently bricks node if migration 48→49 crashes mid-step - (File: `nearcore/src/migrations.rs`)

### Summary

The `migrate_48_to_49` DB schema migration commits an irreversible `delete_all(DBCol::BlockHeader)` as the first step of `delete_old_block_headers`, before the compensating re-add of required headers is complete. If the process crashes (OOM, SIGKILL, power failure) between those two steps, the DB version stays at 48 and `DBCol::BlockHeader` is permanently empty. On every subsequent restart the migration re-runs, but `verify_block_headers` — which executes **before** `delete_old_block_headers` — reads from the now-empty `DBCol::BlockHeader` and returns `Err(DBNotFoundErr)`, causing the migration to fail again. The node is permanently unable to start.

### Finding Description

`migrate_48_to_49` sequences three sub-steps:

```
update_epoch_sync_proof(hot_store.clone(), transaction_validity_period)?;  // line 209
verify_block_headers(hot_store)?;                                           // line 210
delete_old_block_headers(hot_store)?;                                       // line 211
``` [1](#0-0) 

Inside `delete_old_block_headers`, the very first committed write is a full column wipe:

```rust
let mut store_update = store.store_update();
store_update.delete_all(DBCol::BlockHeader);
store_update.commit();          // ← durably committed, irreversible
``` [2](#0-1) 

Only after that commit does the function re-populate `DBCol::BlockHeader` from `DBCol::Block` in batches:

```rust
let mut store_update = chain_store.store_update();
for height in tail_height..(latest_known_height + 1) {
    for block_hash in chain_store.get_all_header_hashes_by_height(height) {
        if let Ok(block) = chain_store.get_block(&block_hash) {
            store_update.set_block_header_only(block.header());
        }
    }
    if height % BATCH_SIZE == 0 {
        store_update.commit()?;
        store_update = chain_store.store_update();
    }
}
store_update.commit()?;
``` [3](#0-2) 

If the process dies after the `delete_all` commit but before the re-add loop finishes, the DB version stays at 48 (the version bump only happens after `migrate` returns `Ok`):

```rust
migrator
    .migrate(&hot_store, cold_db.as_ref(), version, is_snapshot)
    .map_err(StoreOpenerError::MigrationError)?;

// Update versions in both stores
hot_store.set_db_version(version + 1);
``` [4](#0-3) 

On the next startup the migration re-runs from the top. `verify_block_headers` iterates over `DBCol::HeaderHashesByHeight` (untouched by the wipe) and for every block hash calls:

```rust
let header_from_store = chain_store.get_block_header(&block_hash)?;
``` [5](#0-4) 

Because `DBCol::BlockHeader` is now empty, `get_block_header` returns `Err(DBNotFoundErr)`. The `?` propagates it, `verify_block_headers` returns an error, the migration returns an error, the DB version stays at 48, and the node cannot start — on every subsequent restart.

### Impact Explanation

Any node operator upgrading to 2.13 (DB version 48 → 49) whose process is killed during the `delete_old_block_headers` step is permanently unable to restart the node. Recovery requires either restoring from the RocksDB checkpoint snapshot left on disk by `ensure_version` (not automatic; operator must know the path and procedure) or manually using `neard database set-version` to skip the migration, which leaves `DBCol::BlockHeader` in an inconsistent state. The migration is documented to take up to ~1 hour for SST file creation plus ~10 minutes for SST ingestion, making an intervening crash (OOM, SIGKILL, power failure) realistic in production. [6](#0-5) 

### Likelihood Explanation

Every non-archival node upgrading from any release before 2.13 to 2.13 runs this migration exactly once. The migration is long-running (hours on large nodes). Any OS-level kill signal, OOM event, or power interruption during the `delete_old_block_headers` phase triggers the stuck state. No special attacker capability is required; the trigger is ordinary operational conditions.

### Recommendation

Make `delete_old_block_headers` idempotent with respect to a mid-step crash. Two concrete options:

1. **Detect the partial state on retry**: at the start of `migrate_48_to_49` (or `verify_block_headers`), check whether `DBCol::BlockHeader` is empty while `DBCol::HeaderHashesByHeight` is non-empty. If so, skip `verify_block_headers` and jump directly to the re-add loop in `delete_old_block_headers`.

2. **Atomic delete-and-rewrite**: instead of committing `delete_all` separately, build the full replacement set of headers in memory and commit a single transaction that deletes the old entries and inserts the new ones. /// This migration does three things:
/// 1. Copy block headers from hot_store to cold_db (if cold_db is present)
/// 2. Generate and save the compressed epoch sync proof
/// 3. Clear the block headers from genesis to tail in hot_store
fn migrate_48_to_49(
    hot_store: &Store,
    cold_db: Option<&ColdDB>,
    transaction_validity_period: BlockHeightDelta,
    home_dir: &Path,
    cold_store_config: Option<&StoreConfig>,
    is_snapshot: bool,
) -> anyhow::Result<()> {
    tracing::info!(target: "migrations", "starting migration from DB version 48 to 49");

    // State snapshot DBs only contain flat storage columns and lack the
    // epoch/chain data that every step of this migration requires. Skip them.
    if is_snapshot {
        tracing::info!(target: "migrations", "state snapshot DB, skipping chain-dependent migration steps");
        return Ok(());
    }

    // Fresh nodes and forknet-initialized nodes have BlockMisc cleared, so
    // HEAD is absent; nodes that only produced blocks in the genesis epoch
    // have HEAD set but head.epoch_id == EpochId::default(). In both cases
    // there are no block headers to copy, no epoch sync proof to derive, and
    // nothing to verify or delete.
    match hot_store.chain_store().head() {
        Ok(head) if head.epoch_id == EpochId::default() => {
            tracing::info!(target: "migrations", "chain is in the genesis epoch, skipping chain-dependent migration steps");
            return Ok(());
        }
        Err(Error::DBNotFoundErr(_)) => {
            tracing::info!(target: "migrations", "chain head not set (fresh/forknet DB), skipping chain-dependent migration steps");
            return Ok(());
        }
        Ok(_) => {}
        Err(err) => return Err(err.into()),
    }

    if let Some(cold_db) = cold_db {
        let cold_store_config =
            cold_store_config.expect("cold_store config must be present when cold_db exists");
        copy_block_headers_to_cold_db(hot_store, cold_db, home_dir, cold_store_config)?;
    }

    update_epoch_sync_proof(hot_store.clone(), transaction_validity_period)?;
    verify_block_headers(hot_store)?;
    delete_old_block_headers(hot_store)?;
    Ok(())
}
