Audit Report

## Title
Pool Admin Bypasses Drawdown Timelock via `timelock=0` Initialization, Disabling Stop-Loss Guard ‚Äî (`metric-periphery/contracts/extensions/OracleValueStopLossExtension.sol`)

## Summary

`OracleValueStopLossExtension.initialize` accepts `timelock=0` with no minimum validation. When `timelock=0`, a pool admin can call `proposeOracleStopLossDrawdown(pool, 0)` and `executeOracleStopLossDrawdown(pool)` in the same transaction, setting `drawdownE6` to `0`. This causes `_afterSwapOracleStopLoss` to return immediately on the `if (drawdown == 0) return` guard, permanently disabling the stop-loss for all future swaps and giving LPs zero reaction time.

## Finding Description

**Step 1 ‚Äî Initialization allows `timelock=0`.**
`initialize` decodes `(drawdownE6, decayPerSecondE8, timelock)` and stores them. `_validateDrawdown` only rejects values `> E6`; `_validateDecay` only rejects values `> E8`. There is no validation on `timelock`, so `timelock=0` is silently accepted. [1](#0-0) 

**Step 2 ‚Äî `proposeOracleStopLossDrawdown(pool, 0)` with `timelock=0`.**
`_afterTimelock` computes `uint32(block.timestamp + oracleStopLossConfig[pool_].timelock)`. With `timelock=0`, this returns exactly `block.timestamp`. The pending schedule is stored with `pendingDrawdownExecuteAfter = block.timestamp`. [2](#0-1) 

`_validateDrawdown(0)` passes because `0 > E6` is false. [3](#0-2) 

**Step 3 ‚Äî `executeOracleStopLossDrawdown(pool)` in the same transaction.**
`_requireElapsed` checks `if (block.timestamp < executeAfter) revert`. Since `executeAfter == block.timestamp`, the condition `block.timestamp < block.timestamp` is `false` ‚Äî no revert. `drawdownE6` is written to `0`. [4](#0-3) [5](#0-4) 

**Step 4 ‚Äî Stop-loss guard is permanently disabled.**
Every subsequent swap calls `_afterSwapOracleStopLoss`, which reads `cfg.drawdownE6 = 0` and returns immediately. No watermark check, no `OracleStopLossTriggered` revert ‚Äî the guard is dead. [6](#0-5) 

## Impact Explanation

The stop-loss extension's stated invariant is: *"Drawdown and decay changes are timelocked so LPs can react."* With `timelock=0`, the pool admin can atomically disable the guard in a single transaction, giving LPs zero reaction time. All subsequent swaps proceed without any value-per-share floor check, allowing unlimited value extraction from the pool without triggering `OracleStopLossTriggered`. This is a direct admin-boundary break: the pool admin exceeds their authorized capability by bypassing the timelock protection that LPs rely on, matching the contest's explicit admin-boundary break impact category. [7](#0-6) 

## Likelihood Explanation

Any pool deployed with `timelock=0` in the initialization data is immediately exploitable by its admin. The factory performs no minimum-timelock enforcement. The pool admin role is semi-trusted (expected to configure, not steal), and the timelock is the only mechanism protecting LPs from instant parameter changes. The attack requires only two sequential admin calls in one transaction, with no external dependencies.

## Recommendation

1. Add a minimum timelock validation in `initialize` (e.g., `require timelock >= MIN_TIMELOCK` where `MIN_TIMELOCK` is a protocol constant such as 24 hours).
2. Add a minimum drawdown validation so that `drawdownE6=0` cannot be proposed or executed through the admin path once a pool is live (e.g., `require drawdownE6 > 0`).
3. Change `_requireElapsed` to use strict `<=` semantics (`block.timestamp <= executeAfter` ‚Üí revert) to prevent same-block execution even if `timelock=0` is somehow permitted.

## Proof of Concept

```solidity
function test_timelockZeroBypassDisablesStopLoss() public {
    // 1. Initialize extension with timelock=0, drawdown=500_000 (50%)
    bytes memory initData = abi.encode(uint32(500_000), uint32(0), uint32(0));
    vm.prank(address(factory));
    extension.initialize(pool, initData);

    // 2. Pool admin proposes drawdown=0 and executes in same block
    vm.startPrank(poolAdmin);
    extension.proposeOracleStopLossDrawdown(pool, 0);
    extension.executeOracleStopLossDrawdown(pool); // passes: block.timestamp < block.timestamp == false
    vm.stopPrank();

    // 3. Assert drawdownE6 is now 0
    (uint32 drawdown,,,) = extension.oracleStopLossConfig(pool);
    assertEq(drawdown, 0);

    // 4. Assert afterSwap with a price that would have triggered stop-loss now returns silently
    vm.prank(pool);
    bytes4 sel = extension.afterSwap(
        pool, trader, true, 0, 0,
        packedSlot0Initial, packedSlot0Final,
        bidPriceX64, askPriceX64,
        0, 0, 0, ""
    );
    assertEq(sel, IMetricOmmExtensions.afterSwap.selector); // returned without revert
}
```
