Audit Report

## Title
`SwapAllowlistExtension` checks the router address instead of the end user, allowing any caller to bypass the swap allowlist via `MetricOmmSimpleRouter` — (`metric-periphery/contracts/extensions/SwapAllowlistExtension.sol`)

## Summary

`SwapAllowlistExtension.beforeSwap` receives the `sender` argument from the pool, which the pool sets to its own `msg.sender`. When swaps are routed through `MetricOmmSimpleRouter`, the pool's `msg.sender` is the router contract, not the end user. Any pool admin who allowlists the router to enable router-mediated swaps inadvertently grants every user — including those the allowlist is meant to block — the ability to bypass the gate entirely.

## Finding Description

**Root cause — three confirmed code facts:**

1. `MetricOmmPool.swap` passes its own `msg.sender` as the first argument to `_beforeSwap`: [1](#0-0) 

2. `SwapAllowlistExtension.beforeSwap` checks `allowedSwapper[msg.sender][sender]`, where `msg.sender` is the pool and `sender` is whatever the pool received as its own caller: [2](#0-1) 

3. `MetricOmmSimpleRouter.exactInputSingle` calls `pool.swap(...)` directly, making the router the pool's `msg.sender`: [3](#0-2) 

The same applies to `exactInput`, `exactOutputSingle`, and `exactOutput` — all four entry points call `pool.swap` with the router as `msg.sender`. [4](#0-3) 

**Exploit flow:**

```
bob (not allowlisted) → MetricOmmSimpleRouter.exactInputSingle()
  → pool.swap(recipient, ...)          [msg.sender = router]
      → _beforeSwap(sender=router, ...)
          → allowedSwapper[pool][router] → true → PASS ✓ (bypass)

bob → pool.swap(...) directly          [msg.sender = bob]
      → _beforeSwap(sender=bob, ...)
          → allowedSwapper[pool][bob]  → false → REVERT ✓ (correct)
```

The extension has no mechanism to distinguish between an allowlisted user and a blocked user once both arrive through the same router address. There are no existing guards that recover the original caller identity.

## Impact Explanation

A pool configured with `SwapAllowlistExtension` and `allowAllSwappers[pool] = false` is intended to restrict swaps to a specific set of addresses (e.g., KYC-verified counterparties, whitelisted market makers). Once the router is allowlisted — the necessary step for any admin who wants users to access the pool through the standard periphery — the allowlist is completely neutralised. Any address can execute a swap that the pool was designed to reject. This is a direct, complete bypass of the access-control invariant the pool admin configured, exposing the pool's liquidity to unrestricted trading and constituting a broken core pool functionality / admin-boundary break.

## Likelihood Explanation

The router is the primary user-facing interface for the protocol. A pool admin who deploys `SwapAllowlistExtension` and then wants legitimate users to trade through the router has no choice but to allowlist the router address — there is no mechanism to forward the original caller's identity. The bypass is therefore a natural consequence of normal, well-intentioned admin configuration, not an exotic attack path. Any pool that (a) uses `SwapAllowlistExtension` with `allowAllSwappers = false` and (b) allowlists the router is fully exposed. The attack requires no special privileges and is reachable by any unprivileged user.

## Recommendation

The `sender` identity forwarded to extensions must reflect the economic actor, not the intermediary contract. Two complementary fixes:

1. **Router-side:** Have `MetricOmmSimpleRouter` encode the original `msg.sender` into `extensionData` before calling `pool.swap`, so extensions can decode the real user.
2. **Extension-side:** `SwapAllowlistExtension.beforeSwap` should, when `sender` is a known router, decode the real caller from `extensionData` and check that address against `allowedSwapper`.

Alternatively, the pool could expose a dedicated "on-behalf-of" field in the swap call that the router populates with the original caller and that extensions receive as a separate, authenticated argument.

## Proof of Concept

```solidity
// Setup
pool.setAllowAllSwappers(false);
pool.setAllowedToSwap(alice, true);   // only alice is allowed
pool.setAllowedToSwap(router, true);  // admin adds router to enable router swaps

// Attack: bob (not on allowlist) calls:
vm.prank(bob);
router.exactInputSingle(ExactInputSingleParams({pool: pool, ...}));
// Router calls pool.swap(...) with msg.sender = router
// Pool calls extension.beforeSwap(sender=router, ...)
// Extension checks allowedSwapper[pool][router] → true → PASS
// Bob executes a swap in a pool that should have rejected him ✓

// Comparison: direct call by bob
vm.prank(bob);
pool.swap(...);
// Extension checks allowedSwapper[pool][bob] → false → REVERT ✓
```

The bypass is reachable through all four public `MetricOmmSimpleRouter` entry points: `exactInputSingle`, `exactInput`, `exactOutputSingle`, and `exactOutput`. [3](#0-2) [5](#0-4)
