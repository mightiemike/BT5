### Title
Insufficient Bonus Pool Balance Permanently Blocks Position Exit, Undelegation, and Redelegation - (`x/tieredrewards/keeper/claim_rewards.go`)

---

### Summary

The `x/tieredrewards` module atomically bundles bonus reward claiming with every critical position-exit operation (`TierUndelegate`, `ExitTierWithDelegation`, `TierRedelegate`, `ClearPosition`). When the `RewardsPoolName` module account has insufficient balance to cover accrued bonus rewards, `processEventsAndClaimBonus` hard-reverts with `ErrInsufficientBonusPool`, permanently blocking all exit paths for any position that has accrued bonus rewards exceeding the pool balance. Users' staked tokens become irrecoverably locked inside the position.

---

### Finding Description

`processEventsAndClaimBonus` in `claim_rewards.go` computes the total bonus owed to a position and then calls `sufficientBonusPoolBalance` before transferring:

```go
if err := k.sufficientBonusPoolBalance(ctx, bonusCoins); err != nil {
    return nil, err
}
```

`sufficientBonusPoolBalance` returns a hard error if the pool balance is insufficient:

```go
if !poolBalance.IsAllGTE(bonus) {
    return errorsmod.Wrapf(types.ErrInsufficientBonusPool, ...)
}
```

This error propagates up through `claimRewards`, which is called unconditionally at the top of every exit-path message handler before the actual delegation transfer or undelegation occurs:

- `TierUndelegate` — calls `claimRewards` at line 166 before `undelegate`
- `ExitTierWithDelegation` — calls `claimRewards` at line 532 before `transferDelegationFromPosition`
- `TierRedelegate` — calls `claimRewards` at line 229 before `redelegate`
- `ClearPosition` — calls `claimRewards` at line 406 before `ClearExit`
- `ForceFullExitWithDelegation` — calls `claimRewards` at line 37 before `transferDelegationFromPosition`

There is no fallback path that skips bonus claiming and proceeds with the exit. If the pool is empty or underfunded, every one of these operations reverts for every affected position.

The `RewardsPoolName` module account is not automatically replenished. The `BeginBlocker` (`topUpBaseRewards`) only tops up base staking rewards routed to the distribution module; it does not refill the bonus pool. The bonus pool is funded externally (governance/inflation deposits). If it is depleted — whether by a large number of simultaneous claims, a governance failure to replenish, or simply by the pool running dry over time — all positions with accrued bonus are frozen.

---

### Impact Explanation

Any position that has accrued bonus rewards exceeding the current `RewardsPoolName` balance cannot:
- Undelegate its staked tokens (`TierUndelegate`)
- Exit the tier and recover delegation (`ExitTierWithDelegation`)
- Redelegate to a different validator (`TierRedelegate`)
- Clear a triggered exit (`ClearPosition`)

The user's staked tokens are permanently locked inside the position delegator account with no emergency exit path. The `ForceFullExitWithDelegation` migration path (force_exit.go) also calls `claimRewards` first and is equally blocked.

The corrupted invariant is: **a user's staked tokens held in a position delegator account become unrecoverable** — neither the owner nor any governance action can release them without first replenishing the bonus pool to cover every affected position's accrued bonus.

---

### Likelihood Explanation

The bonus pool is a finite module account funded by governance. As positions accumulate bonus over time, the total accrued-but-unclaimed bonus across all positions can exceed the pool balance. This is a realistic steady-state condition, not a theoretical edge case. A single large position with a high-APY tier and a long lock duration can accrue enough bonus to exceed the pool balance on its own. No attacker action is required — normal protocol operation causes this state.

---

### Recommendation

Decouple bonus reward claiming from exit operations. The exit path (`TierUndelegate`, `ExitTierWithDelegation`, `TierRedelegate`, `ClearPosition`) should proceed unconditionally with the delegation transfer/undelegation even if bonus claiming fails due to an insufficient pool. Options:

1. **Defer bonus on pool exhaustion**: If `sufficientBonusPoolBalance` fails, skip the `SendCoinsFromModuleToAccount` call, persist the owed bonus as a claimable debt on the position, and allow the exit to proceed. The user can claim the deferred bonus once the pool is replenished.
2. **Separate claim from exit**: Remove `claimRewards` from exit message handlers entirely. Require users to call `ClaimTierRewards` separately before exiting, but do not make exit contingent on claim success.

The analog in the external report is using `emergencyWithdraw` (which skips reward transfer) instead of `withdraw` (which bundles reward transfer with LP withdrawal).

---

### Proof of Concept

1. Alice creates a position in a high-APY tier, locking 1,000,000 basecro for 1 year.
2. Over time, Alice accrues 50,000 basecro in bonus rewards.
3. The `RewardsPoolName` module account balance drops to 10,000 basecro (pool depleted by other claimants or governance inaction).
4. Alice's lock period expires. She calls `MsgExitTierWithDelegation`.
5. `ExitTierWithDelegation` calls `claimRewards` → `processEventsAndClaimBonus` → `sufficientBonusPoolBalance`.
6. `sufficientBonusPoolBalance` checks: pool balance (10,000) < bonus owed (50,000) → returns `ErrInsufficientBonusPool`.
7. The entire transaction reverts. Alice's 1,000,000 basecro remain locked in the position delegator account.
8. Alice tries `TierUndelegate` — same revert. She tries `TierRedelegate` — same revert. She tries `ClearPosition` — same revert.
9. Alice's funds are permanently inaccessible until governance replenishes the pool with at least 40,000 basecro.

func (k Keeper) sufficientBonusPoolBalance(ctx context.Context, bonus sdk.Coins) error {
	if bonus.IsZero() {
		return nil
	}

	poolAddr := k.accountKeeper.GetModuleAddress(types.RewardsPoolName)
	poolBalance := k.bankKeeper.GetAllBalances(ctx, poolAddr)
	if !poolBalance.IsAllGTE(bonus) {
		return errorsmod.Wrapf(types.ErrInsufficientBonusPool,
			"bonus: %s, pool balance: %s",
			bonus.String(), poolBalance.String())
	}

	return nil
