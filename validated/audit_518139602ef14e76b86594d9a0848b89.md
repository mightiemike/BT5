Audit Report

## Title
SwapAllowlistExtension Gates the Router Address Instead of the Actual End User, Allowing Full Allowlist Bypass via Router - (File: metric-periphery/contracts/extensions/SwapAllowlistExtension.sol)

## Summary

`SwapAllowlistExtension.beforeSwap` checks the `sender` argument, which resolves to `msg.sender` of `MetricOmmPool.swap()`. When a user swaps through `MetricOmmSimpleRouter`, that `msg.sender` is the router contract, not the end user. Any pool admin who allowlists the router to support router-mediated swaps for permitted users simultaneously grants unrestricted swap access to every unpermitted user who routes through the same router.

## Finding Description

`MetricOmmPool.swap` passes `msg.sender` as the first argument to `_beforeSwap`:

```solidity
// metric-core/contracts/MetricOmmPool.sol L230-240
_beforeSwap(
  msg.sender,   // whoever called pool.swap()
  recipient, ...
);
```

`ExtensionCalling._beforeSwap` forwards this value unchanged as `sender` to every configured extension via `abi.encodeCall(IMetricOmmExtensions.beforeSwap, (sender, ...))`.

`SwapAllowlistExtension.beforeSwap` then evaluates:

```solidity
// metric-periphery/contracts/extensions/SwapAllowlistExtension.sol L37-39
if (!allowAllSwappers[msg.sender] && !allowedSwapper[msg.sender][sender]) {
    revert IMetricOmmPoolActions.NotAllowedToSwap();
}
```

Here `msg.sender` is the pool and `sender` is the router. The check resolves to `allowedSwapper[pool][router]`. The extension has no visibility into which end user initiated the call.

`MetricOmmSimpleRouter.exactInputSingle` stores the real user only in transient callback context for payment purposes and passes `params.extensionData` (user-controlled) to the pool — neither provides a trustworthy identity signal to the extension:

```solidity
// metric-periphery/contracts/MetricOmmSimpleRouter.sol L71-80
_setNextCallbackContext(params.pool, CALLBACK_MODE_JUST_PAY, msg.sender, params.tokenIn);
IMetricOmmPoolActions(params.pool).swap(
    params.recipient, params.zeroForOne, ..., "", params.extensionData
);
```

A pool admin who wants to allow only KYC-approved users to trade must choose between two broken states:

| Admin action | Effect |
|---|---|
| Do **not** allowlist the router | KYC-approved users cannot use the router at all |
| Allowlist the router | **Every** user, including non-KYC users, can bypass the allowlist via the router |

## Impact Explanation

Any unpermitted user can trade in a pool that is supposed to be access-controlled by routing through `MetricOmmSimpleRouter`. The allowlist guard — the only on-chain mechanism preventing unauthorised swaps — is silently bypassed. Pools relying on this extension for regulatory compliance (KYC gating), private-pool access control, or partner-only liquidity are fully open to any caller who uses the router. This is a direct admin-boundary break: an unprivileged path (the public router) defeats a configured access guard.

## Likelihood Explanation

`MetricOmmSimpleRouter` is the standard periphery entry point for swaps. Any user who discovers the allowlist can trivially route through the router instead of calling the pool directly. No special privileges, flash loans, or multi-step setup are required — a single `exactInputSingle` call suffices. The trigger is fully reachable by any unprivileged actor.

## Recommendation

The extension must verify the economic initiator of the swap, not the immediate caller of `pool.swap()`. The most robust fix: the router encodes its `msg.sender` (the real user) into `extensionData` before calling `pool.swap()`; the extension decodes and verifies it. This is acceptable because the router is a factory-validated contract. Alternatively, extend the allowlist mapping to support a trusted-router pattern where the extension checks both the router allowlist and a per-router user allowlist decoded from `extensionData`.

## Proof of Concept

```
1. Pool admin deploys a pool with SwapAllowlistExtension configured.
2. Admin calls setAllowedToSwap(pool, alice, true)   // alice is KYC-approved
3. Admin calls setAllowedToSwap(pool, router, true)  // enable router for alice
4. Bob (non-KYC) calls MetricOmmSimpleRouter.exactInputSingle({pool: pool, ...})
5. Router calls pool.swap(recipient=bob, ...)        // msg.sender inside pool = router
6. pool._beforeSwap(sender=router, ...)
7. extension.beforeSwap(sender=router, ...)
8. allowedSwapper[pool][router] == true  → guard passes
9. Bob's swap executes in the restricted pool — allowlist fully bypassed.
```
