### Title
Bonus Checkpoint Advancement Without Payment in `slashRedelegationPosition` Causes Permanent Pre-Slash Bonus Forfeiture — (`x/tieredrewards/keeper/slash.go`, `x/tieredrewards/keeper/claim_rewards.go`)

### Summary

In `processEventsAndClaimBonus`, the position's bonus checkpoints (`LastBonusAccrual`, `LastEventSeq`, `LastKnownBonded`) are advanced in-memory **before** the pool balance is checked. When `ErrInsufficientBonusPool` is returned, `slashRedelegationPosition` silently swallows the error and then calls `setPositionWithState`, persisting the advanced checkpoints without ever paying the bonus. The accrued pre-slash bonus is permanently and irrecoverably lost.

### Finding Description

The execution sequence in `processEventsAndClaimBonus` is:

1. **Event loop** (lines 172–199): `pos.UpdateLastEventSeq(entry.Seq)` advances `LastEventSeq` in-memory; `k.decrementEventRefCount` is **persisted to the store** for each event.
2. **Checkpoint advancement** (lines 215–217): `applyBonusAccrualCheckpoint(&pos.Position, blockTime)` sets `LastBonusAccrual = blockTime`; `pos.UpdateLastKnownBonded(bonded)` updates `LastKnownBonded` — both in-memory on the `*PositionState` pointer.
3. **Pool check** (line 230): `sufficientBonusPoolBalance` is called **after** the checkpoints are already advanced. If the pool is insufficient, `return nil, err` fires. [1](#0-0) 

Back in `slashRedelegationPosition`:

On a **partial slash** (the common case), `ClearBonusCheckpoints()` is never called, so the advanced `LastBonusAccrual = blockTime` is persisted. The next claim starts from `blockTime`, skipping all pre-slash accrual. On a **full slash**, `ClearBonusCheckpoints()` is called (line 70), but the position is also undelegated, so the bonus is equally unrecoverable. [3](#0-2) 

Additionally, event reference counts are decremented and persisted inside the loop (line 196) before the pool check, so the validator events are garbage-collected even though no bonus was paid — making replay on a future claim impossible. [4](#0-3) 

### Impact Explanation

Any position with a pending redelegation whose validator is slashed while the `rewards_pool` module account is empty will permanently lose all bonus rewards accrued since `LastBonusAccrual`. The position owner receives zero bonus, but the position's checkpoints are advanced as if payment succeeded. There is no retry path: the accrual window is gone.

The ADR-006 documents this as intentional ("chain-halt avoidance"), but the implementation conflates two distinct operations — checkpoint advancement and payment — that must be atomic. The correct behavior is: either pay and advance, or neither. [5](#0-4) 

### Likelihood Explanation

The `rewards_pool` can reach zero through:
- **Natural depletion**: the BeginBlocker top-up continuously draws from the pool for base rewards; a governance delay in replenishment leaves the pool empty.
- **Griefing**: an attacker with large positions claims all accrued bonus via `MsgClaimTierRewards` (which fails atomically on insufficient pool in user paths, but the attacker times the claim to drain the pool just before a known slash event — e.g., after observing a validator go offline for downtime).

The slash event itself is externally triggered (validator downtime or double-sign), but the timing window between a validator going offline and the actual slash firing (governed by `signed_blocks_window`) is observable on-chain, giving an attacker a predictable window to drain the pool. [6](#0-5) 

### Recommendation

Move `applyBonusAccrualCheckpoint` and `pos.UpdateLastKnownBonded` to **after** the successful `SendCoinsFromModuleToAccount` call, so checkpoints are only advanced when payment actually succeeds. Alternatively, if the forfeit-on-empty behavior must be preserved for chain-halt avoidance, explicitly advance the checkpoints only inside the `ErrInsufficientBonusPool` branch in `slashRedelegationPosition` (not inside `processEventsAndClaimBonus`), and do not decrement event reference counts until payment is confirmed.

```go
func (s *KeeperSuite) TestSlashRedelegation_EmptyPool_BonusPermanentlyLost() {
    lockAmount := sdkmath.NewInt(sdk.DefaultPowerReduction.Int64())
    _, bondDenom := s.getStakingData()
    // Do NOT fund the rewards pool.

    pos, _, unbondingID := s.setupRedelegatingPosition(lockAmount)
    owner := sdk.MustAccAddressFromBech32(pos.Owner)

    // Advance time so bonus accrues.
    s.ctx = s.ctx.WithBlockTime(s.ctx.BlockTime().Add(30 * 24 * time.Hour))

    balBefore := s.app.BankKeeper.GetBalance(s.ctx, owner, bondDenom)

    // Partial slash fires with empty pool.
    sharesToUnbond := pos.Delegation.Shares.Quo(sdkmath.LegacyNewDec(2))
    err := s.keeper.Hooks().BeforeRedelegationSlashed(s.ctx, unbondingID, sharesToUnbond)
    s.Require().NoError(err) // error swallowed — no panic, no chain halt

    balAfter := s.app.BankKeeper.GetBalance(s.ctx, owner, bondDenom)
    // Owner received zero bonus.
    s.Require().True(balAfter.Amount.Equal(balBefore.Amount), "owner should have received zero bonus")

    // Checkpoints are advanced — bonus window is permanently closed.
    updated, err := s.keeper.GetPositionState(s.ctx, pos.Id)
    s.Require().NoError(err)
    s.Require().Equal(s.ctx.BlockTime(), updated.LastBonusAccrual,
        "LastBonusAccrual advanced to blockTime despite zero payment — bonus permanently lost")

    // Fund pool and claim — yields zero because window already advanced.
    s.fundRewardsPool(sdkmath.NewInt(1_000_000_000), bondDenom)
    bonus, err := s.keeper.ProcessEventsAndClaimBonus(s.ctx, &updated)
    s.Require().NoError(err)
    s.Require().True(bonus.IsZero(), "pre-slash bonus is unrecoverable")
}

func (s *KeeperSuite) TestSlashRedelegationPosition_ClaimsBonusRewardsUpToSlash() {
	lockAmount := sdkmath.NewInt(sdk.DefaultPowerReduction.Int64())
	_, bondDenom := s.getStakingData()
	s.fundRewardsPool(sdkmath.NewInt(1_000_000_000), bondDenom)

	pos, _, unbondingID := s.setupRedelegatingPosition(lockAmount)
	owner := sdk.MustAccAddressFromBech32(pos.Owner)
	preAccrual := pos.LastBonusAccrual

	// Advance block time so bonus accrues on the destination validator.
	s.ctx = s.ctx.WithBlockHeight(s.ctx.BlockHeight() + 1)
	s.ctx = s.ctx.WithBlockTime(s.ctx.BlockTime().Add(30 * 24 * time.Hour))

	balBefore := s.app.BankKeeper.GetBalance(s.ctx, owner, bondDenom)

	// Partial slash — a small fraction of shares.
	sharesToUnbond := pos.Delegation.Shares.Quo(sdkmath.LegacyNewDec(10))
	err := s.keeper.Hooks().BeforeRedelegationSlashed(s.ctx, unbondingID, sharesToUnbond)
	s.Require().NoError(err)

	balAfter := s.app.BankKeeper.GetBalance(s.ctx, owner, bondDenom)
	s.Require().True(balAfter.Amount.GT(balBefore.Amount),
		"owner should have received bonus rewards accrued up to slash: before=%s after=%s",
		balBefore.Amount, balAfter.Amount)

	updated, err := s.keeper.GetPositionState(s.ctx, pos.Id)
	s.Require().NoError(err)
	s.Require().True(updated.LastBonusAccrual.After(preAccrual),
		"LastBonusAccrual should have advanced past the pre-slash checkpoint")
	s.Require().Equal(s.ctx.BlockTime(), updated.LastBonusAccrual,
		"LastBonusAccrual should advance to the slash block time")
	s.Require().True(updated.LastKnownBonded,
		"LastKnownBonded should remain true — destination validator is still bonded")
}
