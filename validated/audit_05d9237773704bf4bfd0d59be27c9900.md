### Title
Cross-contract gas key injection inflates victim storage and blocks key deletion via `GasKeyBalanceTooHigh` — (`runtime/runtime/src/access_keys.rs`)

### Summary

Any deployed contract can call `promise_batch_create(victim_account_id)` followed by `promise_batch_action_add_gas_key_with_full_access` (or the function-call variant) to inject a gas key with up to `MAX_NONCES_FOR_GAS_KEY = 1024` nonces into any victim account without the victim's consent. This inflates the victim's `storage_usage` by ~85 KB per injected key, raising their storage-staking requirement by ~0.85 NEAR. A second cross-contract call using `promise_batch_action_transfer_to_gas_key` can fund the injected key with more than `GasKeyInfo::MAX_BALANCE_TO_BURN = 1 NEAR`, after which `delete_gas_key` refuses deletion with `GasKeyBalanceTooHigh`. The victim is left with an unwanted key they cannot immediately remove, inflated storage, and potential `LackBalanceForState` failures on their own transactions.

### Finding Description

**Root cause 1 — unconstrained cross-account gas key addition**

`action_add_key` in `runtime/runtime/src/access_keys.rs` applies the `AddKeyAction` to whichever account is the receipt receiver. There is no check that the receipt predecessor is the account owner. Any contract can therefore create a receipt targeting an arbitrary account and append an `AddKey` action with a `GasKeyFullAccess` or `GasKeyFunctionCall` permission. [1](#0-0) 

The host function `promise_batch_action_add_gas_key_with_full_access` (and its function-call variant) is exposed to all contracts once the `GasKeys` protocol feature is active. [2](#0-1) 

**Root cause 2 — storage inflation proportional to `num_nonces`**

`add_gas_key` writes one trie entry per nonce and increments `account.storage_usage` by `gas_key_storage_cost(...)`. With `num_nonces = MAX_NONCES_FOR_GAS_KEY = 1024` (the validated ceiling), the per-key storage cost is approximately:

```
1024 × (trie_id_len(ED25519) + sizeof(NonceIndex) + NONCE_VALUE_LEN + num_extra_bytes_record)
≈ 1024 × (33 + 2 + 8 + 40) = 1024 × 83 ≈ 85 KB
``` [3](#0-2) [4](#0-3) 

The validation gate only enforces `1 ≤ num_nonces ≤ MAX_NONCES_FOR_GAS_KEY`; it does not restrict the receipt receiver to the signer. [5](#0-4) 

**Root cause 3 — funded key blocks deletion**

`delete_gas_key` refuses to remove a gas key whose balance exceeds `GasKeyInfo::MAX_BALANCE_TO_BURN = 1 NEAR`. A second cross-contract receipt carrying a `TransferToGasKey` action (host function `promise_batch_action_transfer_to_gas_key`) can fund the injected key with > 1 NEAR from the attacker's own balance, triggering this guard. [6](#0-5) [7](#0-6) 

**Root cause 4 — storage staking check fails for inflated accounts**

`check_storage_stake` computes `storage_amount_per_byte × storage_usage` and rejects transactions when the account balance falls below that threshold. An account whose storage has been inflated by ~85 KB now requires ~0.85 NEAR more in staked balance; if the victim's balance is insufficient, every outgoing transaction fails with `LackBalanceForState`. [8](#0-7) 

### Impact Explanation

- **Storage staking DoS**: Victim's `storage_usage` is inflated by ~85 KB per injected gas key. If the victim's balance does not cover the new storage requirement, all their transactions are rejected with `LackBalanceForState` until they delete the key.
- **Temporary key-deletion block**: If the attacker also funds the key with > 1 NEAR, `delete_gas_key` returns `GasKeyBalanceTooHigh`. The victim must first issue a `WithdrawFromGasKey` transaction (spending gas) to drain the balance below the threshold, then issue a `DeleteKey` transaction — two forced on-chain operations.
- **Repeatable**: The attacker can use a different public key each time (since `AddKeyAlreadyExists` only blocks duplicate keys). Each iteration costs the attacker one cross-contract call's gas plus optionally > 1 NEAR to fund the key; the victim bears the cleanup cost.
- **Zero-balance account breakage**: Accounts that rely on the zero-balance exemption (`ZERO_BALANCE_ACCOUNT_STORAGE_LIMIT ≈ 770 bytes`) are pushed far above the limit by a single 85 KB injection, losing their exemption and immediately failing storage-staking checks.

### Likelihood Explanation

The `GasKeys` protocol feature is active in the codebase and the host functions are registered in the import table. Any account that can deploy a contract (i.e., any funded account) can execute this attack. The attacker needs only enough NEAR to pay for gas (and optionally > 1 NEAR to fund the key). The victim requires no prior interaction with the attacker. The attack is automatable and can target multiple victims in a single block.

### Recommendation

1. **Restrict gas key addition to self-receipts**: In `action_add_key` (or in receipt validation), reject `AddKey` actions with `GasKey*` permission when the receipt predecessor differs from the receipt receiver. Regular access keys already follow this pattern in practice (contracts add keys to accounts they create, not to arbitrary existing accounts).
2. **Alternatively, cap per-account gas key storage**: Introduce a protocol parameter `max_gas_key_nonces_per_account` and enforce it in `action_add_key`, analogous to `MAX_ASSETS_PER_OWNER` in the Ammplify fix.
3. **Allow unconditional deletion of externally-added gas keys**: If the account owner did not sign the `AddKey` action that created the gas key, allow deletion regardless of balance (burning the balance up to `MAX_BALANCE_TO_BURN` and refunding the rest to the deleter).

### Proof of Concept

```rust
// Attacker contract (deployed at attacker.near)
pub fn attack_victim(victim: AccountId, attacker_pubkey: PublicKey) {
    // Step 1: inject gas key with 1024 nonces → +85 KB storage on victim
    let p = env::promise_batch_create(&victim);
    env::promise_batch_action_add_gas_key_with_full_access(
        p,
        &attacker_pubkey,
        1024, // MAX_NONCES_FOR_GAS_KEY
    );

    // Step 2: fund the injected key with 1.000001 NEAR → blocks DeleteKey
    let p2 = env::promise_batch_create(&victim);
    env::promise_batch_action_transfer_to_gas_key(
        p2,
        &attacker_pubkey,
        1_000_001_000_000_000_000_000_000u128, // > MAX_BALANCE_TO_BURN
    );
}
```

After execution:
- `victim.storage_usage` increases by ~85 KB.
- `check_storage_stake` fails for victim if `victim.amount < 0.85 NEAR` (approximate).
- `action_delete_key` on the injected key returns `GasKeyBalanceTooHigh`.
- Victim must call `WithdrawFromGasKey` then `DeleteKey` to recover, spending gas on both.

fn gas_key_storage_cost(
    fee_config: &RuntimeFeesConfig,
    public_key: &PublicKey,
    access_key: &AccessKey,
    num_nonces: NonceIndex,
) -> StorageUsage {
    let storage_config = &fee_config.storage_usage_config;
    let per_nonce_value_size = borsh::object_length(&(0 as Nonce)).unwrap() as u64;
    let per_nonce_key_size = public_key.trie_id_len() as u64 + size_of::<NonceIndex>() as u64;

    num_nonces as u64
        * (per_nonce_key_size + per_nonce_value_size + storage_config.num_extra_bytes_record)
        + access_key_storage_usage(fee_config, public_key, access_key)
}

fn delete_gas_key(
    config: &RuntimeConfig,
    state_update: &mut TrieUpdate,
    account: &mut Account,
    result: &mut ActionResult,
    account_id: &AccountId,
    public_key: &PublicKey,
    access_key: &AccessKey,
    gas_key_info: &GasKeyInfo,
) -> Result<(), RuntimeError> {
    if gas_key_info.balance > GasKeyInfo::MAX_BALANCE_TO_BURN {
        result.result = Err(ActionErrorKind::GasKeyBalanceTooHigh {
            account_id: account_id.clone(),
            public_key: Some(Box::new(public_key.clone())),
            balance: gas_key_info.balance,
        }
        .into());
        return Ok(());
    }
    result.tokens_burnt =
        result.tokens_burnt.checked_add(gas_key_info.balance).ok_or(IntegerOverflowError)?;
    let num_nonces = gas_key_info.num_nonces as usize;
    for i in 0..gas_key_info.num_nonces {
        remove_gas_key_nonce(state_update, account_id.clone(), public_key.clone(), i);
    }
    let nonce_key_len = gas_key_nonce_key_len(account_id, &public_key.into());
    let nonce_remove_compute = storage_removes_compute(
        &config.wasm_config.ext_costs,
        num_nonces,
        nonce_key_len * num_nonces,
        AccessKey::NONCE_VALUE_LEN * num_nonces,
    );
    result.compute_usage = safe_add_compute(result.compute_usage, nonce_remove_compute)?;
    remove_access_key(state_update, account_id.clone(), public_key.clone());
    account.set_storage_usage(account.storage_usage().saturating_sub(gas_key_storage_cost(
        &config.fees,
        public_key,
        access_key,
        gas_key_info.num_nonces,
    )));
    Ok(())
}

fn add_gas_key(
    fee_config: &RuntimeFeesConfig,
    state_update: &mut TrieUpdate,
    account: &mut Account,
    account_id: &AccountId,
    public_key: &PublicKey,
    access_key: &AccessKey,
    gas_key_info: &GasKeyInfo,
    block_height: BlockHeight,
) -> Result<(), StorageError> {
    // For gas keys, nonce stored on access key is not used and should always be zero
    let mut access_key = access_key.clone();
    access_key.nonce = 0;
    set_access_key(state_update, account_id.clone(), public_key.clone(), &access_key);

    // Set up nonces for gas key
    let num_nonces = gas_key_info.num_nonces;
    let nonce = initial_nonce_value(block_height);
    for i in 0..num_nonces {
        set_gas_key_nonce(state_update, account_id.clone(), public_key.clone(), i, nonce);
    }

    account.set_storage_usage(
        account
            .storage_usage()
            .checked_add(gas_key_storage_cost(fee_config, public_key, &access_key, num_nonces))
            .ok_or_else(|| {
                StorageError::StorageInconsistentState(format!(
                    "Storage usage integer overflow for account {}",
                    account_id
                ))
            })?,
    );
    Ok(())
}

fn validate_add_key_action(
    limit_config: &LimitConfig,
    action: &AddKeyAction,
    current_protocol_version: ProtocolVersion,
) -> Result<(), ActionsValidationError> {
    validate_access_key_permission(limit_config, &action.access_key.permission)?;

    // If this is a gas key, apply additional gas key validation
    if let Some(gas_key_info) = action.access_key.gas_key_info() {
        require_protocol_feature(ProtocolFeature::GasKeys, "GasKeys", current_protocol_version)?;

        // For gas keys with FunctionCallPermission, allowance must be None
        if let Some(fc) = action.access_key.permission.function_call_permission() {
            if fc.allowance.is_some() {
                return Err(ActionsValidationError::GasKeyFunctionCallAllowanceNotAllowed);
            }
        }

        if gas_key_info.num_nonces == 0
            || gas_key_info.num_nonces > AccessKeyPermission::MAX_NONCES_FOR_GAS_KEY
        {
            return Err(ActionsValidationError::GasKeyInvalidNumNonces {
                requested_nonces: gas_key_info.num_nonces,
                limit: AccessKeyPermission::MAX_NONCES_FOR_GAS_KEY,
            });
        }

        if gas_key_info.balance != Balance::ZERO {
            return Err(ActionsValidationError::AddGasKeyWithNonZeroBalance {
                balance: gas_key_info.balance,
            });
        }
    }

    Ok(())
}

```rust
pub fn check_storage_stake(
    account: &Account,
    account_balance: Balance,
    runtime_config: &RuntimeConfig,
) -> Result<(), StorageStakingError> {
    let billable_storage_bytes = account.storage_usage();
    let required_amount = runtime_config
        .storage_amount_per_byte()
        .checked_mul(u128::from(billable_storage_bytes))
        .ok_or_else(|| {
            format!(
                "Account's billable storage usage {} overflows multiplication",
                billable_storage_bytes
            )
        })
        .map_err(StorageStakingError::StorageError)?;
    let available_amount = account_balance
        .checked_add(account.locked())
        .ok_or_else(|| {
            format!(
                "Account's amount {} and locked {} overflow addition",
                account.amount(),
                account.locked(),
            )
        })
        .map_err(StorageStakingError::StorageError)?;
    if available_amount >= required_amount {
        Ok(())
    } else {
        if is_zero_balance_account(account) {
            return Ok(());
        }
        Err(StorageStakingError::LackBalanceForStorageStaking(
            required_amount.checked_sub(available_amount).unwrap(),
        ))
    }
}
