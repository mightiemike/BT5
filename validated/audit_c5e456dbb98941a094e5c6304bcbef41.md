Audit Report

## Title
Swap Allowlist Bypassed via Router `multicall`: Any User Can Swap Through an Allowlisted Router — (`metric-periphery/contracts/extensions/SwapAllowlistExtension.sol`)

## Summary
`SwapAllowlistExtension.beforeSwap` gates swaps by checking the `sender` argument passed by the pool, which is always `msg.sender` of the `pool.swap()` call — i.e., the router address. Because `MetricOmmSimpleRouter.multicall` uses `delegatecall` internally but issues `pool.swap()` as a regular external call, the pool always sees `msg.sender = router`. Any user who routes through an allowlisted router bypasses the per-user allowlist entirely.

## Finding Description
The full call path is confirmed by production code:

**Step 1 — `multicall` uses `delegatecall`**

`MetricOmmSimpleRouter.multicall` dispatches each payload via `Address.functionDelegateCall(address(this), data[i])`. Under `delegatecall`, `msg.sender` inside the delegated function is the original caller (attacker), not the router. [1](#0-0) 

**Step 2 — `exactInputSingle` stores attacker as payer, then calls `pool.swap()` as the router**

`msg.sender` (attacker) is stored as the callback payer via `_setNextCallbackContext`. But `pool.swap()` is a regular external call — the pool sees `msg.sender = router`. [2](#0-1) 

**Step 3 — Pool passes `msg.sender` (router) as `sender` to `_beforeSwap`**

`MetricOmmPool.swap` unconditionally passes `msg.sender` as the first argument to `_beforeSwap`. [3](#0-2) 

**Step 4 — `ExtensionCalling._beforeSwap` forwards `sender` (router) to the extension**

`ExtensionCalling._beforeSwap` encodes and forwards the `sender` argument (router) to every configured extension. [4](#0-3) 

**Step 5 — `SwapAllowlistExtension.beforeSwap` checks `allowedSwapper[pool][router]`**

The extension evaluates `allowedSwapper[msg.sender][sender]` where `msg.sender` is the pool and `sender` is the router. If `allowedSwapper[pool][router] = true`, the gate passes for every user who routes through that router, regardless of who the actual user is. [5](#0-4) 

## Impact Explanation
`SwapAllowlistExtension` is entirely ineffective for any pool that allowlists the router. A pool admin who intends to restrict swaps to a curated set of addresses (e.g., KYC'd users, whitelisted protocols) cannot enforce that restriction — any address can bypass it by calling `router.multicall`. This breaks the core pool curation guarantee: restricted pools can be accessed and drained by unauthorized actors. This constitutes broken core functionality with direct fund impact.

## Likelihood Explanation
The router is a public, permissionless contract. Any attacker can call `multicall` with an `exactInputSingle` payload. The only precondition is that the router itself is allowlisted on the target pool — which is the expected and documented deployment pattern, since pool admins must allowlist the router so legitimate users can swap through it. This precondition is met in every normal deployment.

## Recommendation
The extension must check the **original user identity**, not the intermediary router. The cleanest fix is to have the router encode the real user address (`msg.sender`, captured before `delegatecall` context) into `extensionData`, and have `SwapAllowlistExtension.beforeSwap` decode and check it. The router acts as a trusted forwarder. Alternatively, add an `initiator` field to the `beforeSwap` hook signature populated by the router as the pre-`delegatecall` `msg.sender`.

```solidity
// Foundry integration test
function test_multicall_bypasses_swap_allowlist() public {
    // Setup: pool with SwapAllowlistExtension, router is allowlisted, attacker is not
    swapExtension.setAllowedToSwap(address(pool), address(router), true);
    // attacker is NOT allowlisted

    // Direct swap from attacker reverts
    vm.prank(attacker);
    vm.expectRevert(IMetricOmmPoolActions.NotAllowedToSwap.selector);
    pool.swap(attacker, true, 1000, 0, "", "");

    // But multicall through router succeeds — allowlist bypassed
    vm.prank(attacker);
    bytes[] memory calls = new bytes[](1);
    calls[0] = abi.encodeCall(router.exactInputSingle, (
        IMetricOmmSimpleRouter.ExactInputSingleParams({
            pool: address(pool), tokenIn: token0, tokenOut: token1,
            zeroForOne: true, amountIn: 1000, amountOutMinimum: 0,
            recipient: attacker, deadline: block.timestamp + 1,
            priceLimitX64: 0, extensionData: ""
        })
    ));
    router.multicall(calls); // succeeds
}
```
