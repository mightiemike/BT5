### Title
Stale Pending-Data Fallback Silently Injects Zero L2/L1-Data Gas Prices into RPC Fee Estimation and Simulation — (`crates/apollo_rpc/src/v0_8/api/api_impl.rs`)

### Summary

When the in-memory pending block's parent hash does not match the latest committed block hash, `read_pending_data` constructs a synthetic `DeprecatedPendingBlock` fallback. Because `PendingBlockOrDeprecated::l2_gas_price()` and `l1_data_gas_price()` unconditionally return `GasPricePerToken::default()` (both fields = 0) for the `Deprecated` variant, those zeros propagate through `client_pending_data_to_execution_pending_data` into `create_block_context`, where they are silently clamped to `NonzeroGasPrice::MIN` (= 1). Every `starknet_estimateFee`, `starknet_simulateTransactions`, and `starknet_call` request that targets the pending block during this window receives fee estimates computed against a gas price of 1 instead of the real ~10–30 Gwei, producing authoritative-looking values that are billions of times too low.

### Finding Description

**Step 1 – Stale-data fallback creates a `DeprecatedPendingBlock`.**

`read_pending_data` in `api_impl.rs` checks whether the cached pending block's parent hash equals the latest committed block hash. When it does not (startup, post-commit gap, feed disruption), it returns a synthetic block:

```rust
Ok(PendingData {
    block: PendingBlockOrDeprecated::Deprecated(DeprecatedPendingBlock {
        parent_block_hash: latest_header.block_hash,
        eth_l1_gas_price: latest_header.block_header_without_hash.l1_gas_price.price_in_wei,
        strk_l1_gas_price: latest_header.block_header_without_hash.l1_gas_price.price_in_fri,
        ..Default::default()   // ← status, transactions, etc. all zeroed
    }),
    ...
})
``` [1](#0-0) 

**Step 2 – `l2_gas_price()` and `l1_data_gas_price()` return zero for the `Deprecated` variant.**

```rust
pub fn l2_gas_price(&self) -> GasPricePerToken {
    match self {
        // In older versions, L2 gas price was 0.
        PendingBlockOrDeprecated::Deprecated(_) => GasPricePerToken::default(),
        PendingBlockOrDeprecated::Current(block) => block.l2_gas_price,
    }
}
pub fn l1_data_gas_price(&self) -> GasPricePerToken {
    match self {
        // In older versions, data gas price was 0.
        PendingBlockOrDeprecated::Deprecated(_) => GasPricePerToken::default(),
        ...
    }
}
``` [2](#0-1) 

**Step 3 – Zero propagates through the conversion layer.**

`client_pending_data_to_execution_pending_data` copies the zero directly:

```rust
l2_gas_price: client_pending_data.block.l2_gas_price(),   // GasPricePerToken { 0, 0 }
l1_data_gas_price: client_pending_data.block.l1_data_gas_price(), // GasPricePerToken { 0, 0 }
``` [3](#0-2) 

**Step 4 – `create_block_context` silently clamps zero to `NonzeroGasPrice::MIN` (= 1).**

```rust
l2_gas_price: NonzeroGasPrice::new(l2_gas_price.price_in_wei)
    .unwrap_or(NonzeroGasPrice::MIN),   // 0 → 1
l1_data_gas_price: NonzeroGasPrice::new(l1_data_gas_price.price_in_wei)
    .unwrap_or(NonzeroGasPrice::MIN),   // 0 → 1
```

No error is returned; the `BlockContext` is built with gas price = 1 and handed to the blockifier for fee estimation and simulation. [4](#0-3) 

The open TODO at line 379 — `// TODO(yair): What to do about blocks pre 0.13.1 where the data gas price were 0?` — confirms this path is unresolved for the current network version. [5](#0-4) 

### Impact Explanation

Every call to `starknet_estimateFee`, `starknet_simulateTransactions`, or `starknet_call` that targets `block_id = "pending"` during the stale-data window receives fee estimates computed with `l2_gas_price = 1` and `l1_data_gas_price = 1` instead of the real values (~10–30 Gwei). The returned `FeeEstimation` object carries these wrong prices as authoritative fields (`l2_gas_price`, `l1_data_gas_price`, `overall_fee`). Wallets and tooling that rely on these estimates to set `max_price_per_unit` bounds will submit transactions with bounds far below the actual network price, causing them to be rejected by the gateway's `validate_tx_l2_gas_price_within_threshold` check or to fail execution. [6](#0-5) 

This matches the allowed impact: **High — RPC execution, fee estimation, tracing, simulation, or pending view returns an authoritative-looking wrong value.**

### Likelihood Explanation

The stale-data condition is not an edge case. It occurs:
1. At every node startup before the first pending block is received.
2. In the gap between each committed block and the arrival of the next pending block (typically seconds on mainnet).
3. Whenever the pending-data feed is disrupted.

Any unprivileged caller who issues a pending-block fee estimation during these windows — which is the normal wallet flow — receives the wrong value. No special privileges or malicious input are required.

### Recommendation

Replace the silent `unwrap_or(NonzeroGasPrice::MIN)` fallback in `create_block_context` with an explicit error when the pending data source is the synthetic `DeprecatedPendingBlock` and the network version is ≥ 0.13.1 (where L2 and data gas prices are mandatory). Concretely:

1. Add a version-aware check in `read_pending_data`: if the starknet version of the latest committed header is ≥ 0.13.1, return an RPC error (`BLOCK_NOT_FOUND` or a new `PENDING_DATA_UNAVAILABLE` code) instead of constructing a `DeprecatedPendingBlock` with zero gas prices.
2. Alternatively, populate the synthetic fallback block with the gas prices from the latest committed header (which are already available at the call site) rather than leaving them at zero.

### Proof of Concept

```
1. Start the node (or wait for a block to be committed).
2. Before the next pending block is received by the pending-data feed,
   issue: POST /rpc/v0_8
   { "method": "starknet_estimateFee",
     "params": [<any valid tx>, [], {"block_id": "pending"}] }

3. Observe: the response contains
     "l2_gas_price": "0x1"          (should be ~0x6FC23AC00 = 30 Gwei)
     "l1_data_gas_price": "0x1"     (should be ~actual blob price)
     "overall_fee": <near-zero>

4. Root cause trace:
   read_pending_data()                          → DeprecatedPendingBlock (parent hash mismatch)
   PendingBlockOrDeprecated::l2_gas_price()     → GasPricePerToken { 0, 0 }
   client_pending_data_to_execution_pending_data → l2_gas_price = { 0, 0 }
   create_block_context / NonzeroGasPrice::new(0).unwrap_or(MIN) → 1
   tx_execution_output_to_fee_estimation        → l2_gas_price = 1, overall_fee ≈ 0
```

```rust
pub(crate) fn tx_execution_output_to_fee_estimation(
    tx_execution_output: &TransactionExecutionOutput,
    block_context: &BlockContext,
) -> ExecutionResult<FeeEstimation> {
    let gas_prices = &block_context.block_info().gas_prices;
    let (l1_gas_price, l1_data_gas_price, l2_gas_price) = (
        gas_prices.l1_gas_price(&tx_execution_output.price_unit.into()).get(),
        gas_prices.l1_data_gas_price(&tx_execution_output.price_unit.into()).get(),
        gas_prices.l2_gas_price(&tx_execution_output.price_unit.into()).get(),
    );

    let gas_vector = tx_execution_output.execution_info.receipt.gas;

    Ok(FeeEstimation {
        gas_consumed: gas_vector.l1_gas.0.into(),
        l1_gas_price,
        data_gas_consumed: gas_vector.l1_data_gas.0.into(),
        l1_data_gas_price,
        l2_gas_price,
        overall_fee: tx_execution_output.execution_info.receipt.fee,
        unit: tx_execution_output.price_unit,
    })
}
