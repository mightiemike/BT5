Audit Report

## Title
`SwapAllowlistExtension` Checks Router Address Instead of Actual Swapper, Allowing Allowlist Bypass via `MetricOmmSimpleRouter` - (File: `metric-periphery/contracts/extensions/SwapAllowlistExtension.sol`)

## Summary
`SwapAllowlistExtension.beforeSwap` gates swaps by checking the `sender` argument, which resolves to `msg.sender` of the pool's `swap` call — the router contract address, not the originating user. When a pool admin allowlists the router to support router-mediated swaps for legitimate users, any non-allowlisted address can bypass the gate by calling `MetricOmmSimpleRouter.exactInputSingle`, `exactInput`, `exactOutputSingle`, or `exactOutput`. The extension's per-pool allowlist invariant is completely broken for any pool that uses both `SwapAllowlistExtension` and the router.

## Finding Description
`SwapAllowlistExtension.beforeSwap` checks `allowedSwapper[msg.sender][sender]`, where `msg.sender` is the calling pool and `sender` is the first argument forwarded by the pool:

```solidity
// metric-periphery/contracts/extensions/SwapAllowlistExtension.sol L37
if (!allowAllSwappers[msg.sender] && !allowedSwapper[msg.sender][sender]) {
    revert IMetricOmmPoolActions.NotAllowedToSwap();
}
```

`MetricOmmPool.swap` passes its own `msg.sender` as the `sender` argument to `_beforeSwap`:

```solidity
// metric-core/contracts/MetricOmmPool.sol L230-231
_beforeSwap(
    msg.sender,   // whoever called pool.swap()
    ...
```

`ExtensionCalling._beforeSwap` encodes this `sender` and dispatches it to the extension:

```solidity
// metric-core/contracts/ExtensionCalling.sol L160-176
_callExtensionsInOrder(
    BEFORE_SWAP_ORDER,
    abi.encodeCall(IMetricOmmExtensions.beforeSwap, (sender, recipient, ...))
);
```

When a user calls `MetricOmmSimpleRouter.exactInputSingle`, the router is the direct caller of `pool.swap`:

```solidity
// metric-periphery/contracts/MetricOmmSimpleRouter.sol L72-80
(int128 amount0Delta, int128 amount1Delta) = IMetricOmmPoolActions(params.pool)
    .swap(
        params.recipient,
        params.zeroForOne,
        MetricOmmSwapInputs.asAmountSpecifiedIn(params.amountIn),
        priceLimitX64,
        "",
        params.extensionData
    );
```

The pool therefore passes `address(router)` as `sender` to the extension. The extension checks `allowedSwapper[pool][router]`, not `allowedSwapper[pool][actual_user]`.

This creates an irresolvable dilemma for any pool admin who wants to use the allowlist with the router: not allowlisting the router blocks all router-mediated swaps for legitimate users; allowlisting the router opens the gate to every unprivileged address. The `DepositAllowlistExtension` does not share this flaw because it checks `owner` (the LP position owner), which is passed as an explicit parameter independent of `msg.sender`.

## Impact Explanation
A pool configured with `SwapAllowlistExtension` to restrict trading to a curated set of addresses (e.g., KYC'd counterparties, institutional LPs, or whitelisted market makers) has that restriction completely bypassed. Any non-allowlisted user calls `MetricOmmSimpleRouter.exactInputSingle` targeting the restricted pool; the extension sees `sender = router`; if the router is allowlisted, the swap executes. The non-allowlisted user receives pool output tokens at oracle-derived prices, draining LP assets without authorization. This is a direct loss of LP principal and a complete failure of the pool's access-control invariant — a broken core pool functionality causing loss of funds, meeting the required impact gate.

## Likelihood Explanation
`MetricOmmSimpleRouter` is the primary user-facing swap interface in the periphery. Any pool admin who deploys a swap-allowlisted pool and also wants to support the standard router path must allowlist the router — the protocol provides no other mechanism to do so. The bypass is therefore reachable on any production pool that uses both `SwapAllowlistExtension` and the router, which is the expected deployment pattern. No special privileges, flash loans, or non-standard tokens are required; a single public `exactInputSingle` call suffices.

## Recommendation
1. **Pass the originating user through `extensionData`**: Have `MetricOmmSimpleRouter` encode `msg.sender` into the `extensionData` bytes it forwards to the pool. Update `SwapAllowlistExtension.beforeSwap` to decode and check this address when `sender` is a known router, falling back to `sender` for direct calls.
2. **Add a `swapFor` / `originator` field to the pool's swap interface**: The pool could accept an explicit `originator` address (verified via a trusted-router registry) and pass it as `sender` to extensions, preserving the allowlist semantics regardless of routing path.
3. **At minimum, document the incompatibility**: If neither fix is applied, the extension's NatSpec must warn that allowlisting the router defeats the allowlist for all users, and that router-mediated swaps are unsupported on allowlisted pools.

## Proof of Concept
```
Setup:
  - Deploy pool with SwapAllowlistExtension
  - Pool admin calls swapExtension.setAllowedToSwap(pool, router, true)
    (to support router-mediated swaps for legitimate users)
  - Pool admin does NOT call setAllowedToSwap(pool, attacker, true)

Attack:
  1. attacker (not allowlisted) calls:
       router.exactInputSingle(ExactInputSingleParams({
           pool: restrictedPool,
           tokenIn: token0,
           ...
           recipient: attacker,
           extensionData: ""
       }))

  2. Router calls pool.swap(attacker_as_recipient, ...)
     with msg.sender = router

  3. Pool calls _beforeSwap(sender=router, ...)

  4. ExtensionCalling calls extension.beforeSwap(sender=router, ...)

  5. Extension checks allowedSwapper[pool][router] == true  ✓
     → does NOT revert

  6. Swap executes; attacker receives output tokens from the pool

Result:
  - attacker bypassed the swap allowlist
  - LP assets transferred to a non-allowlisted address
  - SwapAllowlistExtension invariant broken
```

Foundry test: deploy pool with `SwapAllowlistExtension`, call `setAllowedToSwap(pool, address(router), true)`, prank as an address not in the allowlist, call `router.exactInputSingle(...)` targeting the pool, assert the swap succeeds and the non-allowlisted address receives output tokens.
