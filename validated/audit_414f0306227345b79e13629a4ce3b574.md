### Title
Non-Creator NFT Owners Cannot Burn Transferred NFTs, Making Them Permanently Unburnable — (`x/nft/keeper/keeper.go`)

### Summary
`BurnNFT` enforces **both** `IsOwner` and `IsDenomCreator` on the same `owner` address. Once a denom creator transfers an NFT to any non-creator address, no one on-chain can ever burn that token: the new owner fails `IsDenomCreator`, and the original creator fails `IsOwner`. The NFT is permanently unburnable and the total-supply counter never decrements.

### Finding Description

`BurnNFT` in `x/nft/keeper/keeper.go` applies two sequential guards:

```go
// x/nft/keeper/keeper.go  lines 141-161
func (k Keeper) BurnNFT(ctx sdk.Context, denomID, tokenID string, owner sdk.AccAddress) error {
    ...
    nft, err := k.IsOwner(ctx, denomID, tokenID, owner)   // guard 1
    if err != nil {
        return err
    }
    _, err = k.IsDenomCreator(ctx, denomID, owner)         // guard 2
    if err != nil {
        return err
    }
    k.deleteNFT(ctx, denomID, nft)
    k.deleteOwner(ctx, denomID, tokenID, owner)
    k.decreaseSupply(ctx, denomID)
    return nil
}
``` [1](#0-0) 

`IsOwner` requires the caller to be the **current owner** of the token: [2](#0-1) 

`IsDenomCreator` requires the caller to be the **original creator** of the denom (a field that never changes after `IssueDenom`): [3](#0-2) 

`TransferOwner` (called by `MsgTransferNFT`) only checks `IsOwner` — no creator restriction — so any owner can freely transfer: [4](#0-3) 

After a transfer from creator A → non-creator B:
- **B calls `BurnNFT`**: `IsOwner` passes, `IsDenomCreator` fails → `ErrUnauthorized`.
- **A calls `BurnNFT`**: `IsOwner` fails (A is no longer owner) → `ErrUnauthorized`.

Neither party can burn. `decreaseSupply` is never reached, so `GetTotalSupply` is permanently inflated.

`BurnNFTUnverified` (which skips the creator check) is **not** reachable via any user-facing `MsgServer` path — it is only called internally for IBC escrow: [5](#0-4) 

### Impact Explanation
Any NFT transferred away from its denom creator becomes permanently unburnable by anyone. The token is stuck in the recipient's account with no on-chain mechanism to destroy it. `GetTotalSupply` never decrements for such tokens, corrupting the supply invariant. If the NFT has economic value (e.g., represents a redeemable asset), the owner cannot exercise their right to destroy it.

### Likelihood Explanation
This is triggered by the ordinary, fully supported `MsgTransferNFT` → `MsgBurnNFT` sequence. No special privileges, governance, or operator access are required. Any user who receives an NFT from a denom creator is immediately affected.

### Recommendation
`BurnNFT` should require **only** `IsOwner`. The `IsDenomCreator` check is appropriate for `MintNFT` and `EditNFT` (privileged operations), but burn is an owner right. Remove the `IsDenomCreator` call from `BurnNFT`:

```go
func (k Keeper) BurnNFT(ctx sdk.Context, denomID, tokenID string, owner sdk.AccAddress) error {
    if !k.HasDenomID(ctx, denomID) {
        return sdkerrors.Wrapf(types.ErrInvalidDenom, "denom ID %s not exists", denomID)
    }
    nft, err := k.IsOwner(ctx, denomID, tokenID, owner)
    if err != nil {
        return err
    }
    k.deleteNFT(ctx, denomID, nft)
    k.deleteOwner(ctx, denomID, tokenID, owner)
    k.decreaseSupply(ctx, denomID)
    return nil
}
```

This aligns `BurnNFT` with `BurnNFTUnverified`'s logic, which already correctly omits the creator check.

### Proof of Concept

1. A issues denom `d1` → A is `denom.Creator`.
2. A mints NFT `n1` in `d1` to A via `MintNFT` (creator check passes).
3. A transfers `n1` to B via `MsgTransferNFT` → `TransferOwner` succeeds (only `IsOwner` checked).
4. B sends `MsgBurnNFT{DenomId:"d1", Id:"n1", Sender:B}` → `IsOwner` passes, `IsDenomCreator` returns `ErrUnauthorized`. Burn fails.
5. A sends `MsgBurnNFT{DenomId:"d1", Id:"n1", Sender:A}` → `IsOwner` returns `ErrUnauthorized` (A is no longer owner). Burn fails.
6. `GetTotalSupply("d1")` still returns 1. NFT `n1` remains in state indefinitely. [1](#0-0)

func (k Keeper) IsOwner(ctx sdk.Context, denomID, tokenID string, owner sdk.AccAddress) (types.BaseNFT, error) {
	nft, err := k.GetNFT(ctx, denomID, tokenID)
	if err != nil {
		return types.BaseNFT{}, err
	}

	if !owner.Equals(nft.GetOwner()) {
		return types.BaseNFT{}, sdkerrors.Wrapf(types.ErrUnauthorized, "%s is not the owner of %s/%s", owner.String(), denomID, tokenID)
	}

	return nft.(types.BaseNFT), nil
}

func (k Keeper) IsDenomCreator(ctx sdk.Context, denomID string, address sdk.AccAddress) (types.Denom, error) {
	denom, err := k.GetDenom(ctx, denomID)
	if err != nil {
		return types.Denom{}, err
	}

	creator, err := sdk.AccAddressFromBech32(denom.Creator)
	if err != nil {
		panic(err)
	}

	if !creator.Equals(address) {
		return types.Denom{}, sdkerrors.Wrapf(types.ErrUnauthorized, "%s is not the creator of %s", address, denomID)
	}

	return denom, nil
