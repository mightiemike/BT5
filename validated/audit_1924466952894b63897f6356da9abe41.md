### Title
Stale Zero Fee Rates Applied to New Products After Global Tier Fee Update — (`core/contracts/OffchainExchange.sol`)

### Summary
`updateTierFeeRates` with `productId == QUOTE_PRODUCT_ID` sets the `nonDefaultFeeTierMask` bit for a tier after writing fee rates only for products that exist at call time. Any product added afterward has zero-initialized `feeRates[tier][newProductId]`. Because the mask bit is already set, `getTierFeeRateX18` returns those zero values instead of the hardcoded 2 bps default, causing every trade on the new product to incur 0 taker fees.

### Finding Description
`getTierFeeRateX18` uses a single `nonDefaultFeeTierMask` bit to decide whether to return stored fee rates or the hardcoded default:

```solidity
// OffchainExchange.sol L938-945
if (nonDefaultFeeTierMask & (1 << tier) != 0) {
    return feeRates[tier][productId];   // returns zero for any product not yet written
}
return FeeRates({
    makerRateX18: 0,
    takerRateX18: 200_000_000_000_000   // 2 bps hardcoded default
});
```

`updateTierFeeRates` populates `feeRates[tier][productId]` only for products that exist at the moment of the call, then unconditionally sets the mask bit:

```solidity
// OffchainExchange.sol L966-990
if (txn.productId == QUOTE_PRODUCT_ID) {
    // iterates spotEngine.getProductIds() and perpEngine.getProductIds() at call time
    feeRates[txn.tier][spotProductIds[i]] = FeeRates(...);
    feeRates[txn.tier][perpProductIds[i]] = FeeRates(...);
}
nonDefaultFeeTierMask |= uint128(1) << txn.tier;   // bit set regardless
```

After this call, any new product added to the exchange has `feeRates[tier][newProductId]` at its zero-initialized default (`makerRateX18 = 0`, `takerRateX18 = 0`). Because the mask bit is already set, `getTierFeeRateX18` returns those zero values — the hardcoded 2 bps fallback is permanently bypassed for that tier on every new product.

Tier 0 is the default tier for every address (`feeTiers[user]` defaults to 0), so a single `updateTierFeeRates(tier=0, productId=QUOTE_PRODUCT_ID)` call — a routine protocol operation — causes all subsequent new products to have 0 taker fees for all users.

### Impact Explanation
Every `matchOrders` call on a new product routes through `applyFee` → `getUserFeeRateWithBuilder` → `getTierFeeRateX18`. With `takerRateX18 = 0`, `keepRateX18 = ONE`, `orderInfo.fee = 0`, and `market.collectedFees` never increments for taker trades on that product. Protocol fee revenue on all new markets is permanently zeroed for the affected tier. For tier 0 (all users), this is a complete loss of taker fee income on every market listed after the global fee update.

### Likelihood Explanation
`updateTierFeeRates` with `QUOTE_PRODUCT_ID` is the standard mechanism for applying a uniform fee schedule across all markets. It is called as a normal administrative operation. New products are regularly listed. The combination is not adversarial — it is the expected operational sequence. Any trader on a new product automatically benefits from 0 fees with no special action required.

### Recommendation
When `txn.productId == QUOTE_PRODUCT_ID`, store the intended `FeeRates` as a per-tier default (e.g., `defaultFeeRates[tier]`) rather than iterating existing products. `getTierFeeRateX18` should fall back to `defaultFeeRates[tier]` when `feeRates[tier][productId]` has not been explicitly set for that specific product, instead of returning zero-initialized storage.

### Proof of Concept
1. Protocol calls `updateTierFeeRates(tier=0, productId=QUOTE_PRODUCT_ID, takerRateX18=200_000_000_000_000)`.
   - `feeRates[0][existingProduct]` is written for all current products.
   - `nonDefaultFeeTierMask |= 1` — bit 0 is now set.
2. Protocol lists a new product `newProductId` via `addProduct`.
   - `feeRates[0][newProductId]` is never written; it remains `{makerRateX18: 0, takerRateX18: 0}`.
3. Trader (any user, since all users are in tier 0 by default) submits a taker order on `newProductId` via `matchOrders`.
4. `getTierFeeRateX18(0, newProductId)`: mask bit is set → returns `feeRates[0][newProductId]` = `{0, 0}`.
5. `applyFee`: `keepRateX18 = ONE - 0 = ONE`, `orderInfo.fee = 0`. Trader pays zero taker fees.
6. `market.collectedFees` is never incremented; protocol receives no fee revenue from this market. [1](#0-0) [2](#0-1) [3](#0-2)

    function updateTierFeeRates(IEndpoint.UpdateTierFeeRates memory txn)
        external
        onlyEndpoint
    {
        if (txn.productId == QUOTE_PRODUCT_ID) {
            uint32[] memory spotProductIds = spotEngine.getProductIds();
            uint32[] memory perpProductIds = perpEngine.getProductIds();
            for (uint32 i = 0; i < spotProductIds.length; i++) {
                if (spotProductIds[i] == QUOTE_PRODUCT_ID) {
                    continue;
                }
                feeRates[txn.tier][spotProductIds[i]] = FeeRates(
                    txn.makerRateX18,
                    txn.takerRateX18
                );
            }
            for (uint32 i = 0; i < perpProductIds.length; i++) {
                feeRates[txn.tier][perpProductIds[i]] = FeeRates(
                    txn.makerRateX18,
                    txn.takerRateX18
                );
            }
        } else {
            feeRates[txn.tier][txn.productId] = FeeRates(
                txn.makerRateX18,
                txn.takerRateX18
            );
        }
        nonDefaultFeeTierMask |= uint128(1) << txn.tier;
    }
