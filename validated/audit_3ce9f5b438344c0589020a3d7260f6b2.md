### Title
`SwapAllowlistExtension` Checks the Router Address Instead of the End-User, Allowing Any User to Bypass a Curated Pool's Swap Allowlist â€” (`metric-periphery/contracts/extensions/SwapAllowlistExtension.sol`)

---

### Summary

`SwapAllowlistExtension.beforeSwap` gates swaps on the `sender` parameter, which the pool sets to `msg.sender` of `pool.swap()`. When a user routes through `MetricOmmSimpleRouter`, the router is the direct caller of the pool, so the extension checks whether the **router** is allowlisted rather than the **end-user**. A pool admin who allowlists the router to let their users access periphery features inadvertently opens the gate to every address on-chain.

---

### Finding Description

The pool's `swap` function passes its own `msg.sender` as the `sender` argument to every extension hook: [1](#0-0) 

`_beforeSwap` forwards that value unchanged to each configured extension: [2](#0-1) 

`SwapAllowlistExtension.beforeSwap` then uses `msg.sender` (the pool) as the mapping key and the forwarded `sender` (the direct caller of the pool) as the swapper identity: [3](#0-2) 

When a user calls `MetricOmmSimpleRouter`, the router calls `pool.swap(...)`. The pool therefore passes the **router address** as `sender` to the extension. The extension evaluates `allowedSwapper[pool][router]`, not `allowedSwapper[pool][user]`.

A pool admin who wants their allowlisted users to access multi-hop routing or native-ETH wrapping must allowlist the router address. The moment they do, the check degenerates to `allowedSwapper[pool][router] == true`, which is satisfied for **every** caller regardless of their individual allowlist status.

By contrast, `DepositAllowlistExtension.beforeAddLiquidity` correctly gates on the `owner` parameter (the position owner), not on `sender` (the caller): [4](#0-3) 

This inconsistency confirms that `SwapAllowlistExtension` binds authorization to the wrong actor.

---

### Impact Explanation

A curated pool that deploys `SwapAllowlistExtension` to restrict trading to KYC'd or otherwise vetted addresses is completely bypassed the moment the router is allowlisted. Any unprivileged address can call `MetricOmmSimpleRouter` and execute swaps against the pool, draining LP value or violating the pool's curation policy. This is a direct loss-of-curation-control impact with fund-impacting consequences for LPs who deposited under the assumption that only vetted counterparties could trade.

---

### Likelihood Explanation

The `MetricOmmSimpleRouter` is the canonical user-facing entry point for swaps. Pool admins who configure an allowlist and also want their users to access routing features will naturally allowlist the router. The bypass requires no special privileges, no malicious setup, and no non-standard tokens â€” only a call through the standard periphery router.

---

### Recommendation

Gate the allowlist on the **end-user identity** rather than the direct pool caller. Two approaches:

1. **Check `recipient` instead of `sender`** â€” `recipient` is the address that receives swap output and is set by the end-user even when routing through the router. This is the minimal change.
2. **Pass the end-user through `extensionData`** â€” require the router to encode the originating user in `extensionData` and have the extension decode and verify it. This is more robust for multi-hop paths.

Additionally, align `SwapAllowlistExtension` with `DepositAllowlistExtension`, which already correctly gates on the economic actor (`owner`) rather than the operational caller (`sender`).

---

### Proof of Concept

```
1. Pool admin deploys a pool with SwapAllowlistExtension configured.
2. Admin allowlists alice (a KYC'd user): setAllowedToSwap(pool, alice, true).
3. Admin also allowlists the router so alice can use multi-hop routing:
       setAllowedToSwap(pool, router, true).
4. Bob (not KYC'd, not individually allowlisted) calls:
       router.swap(..., pool, ...)
5. The router calls pool.swap(...). The pool passes msg.sender = router as `sender`
   to the extension.
6. The extension evaluates:
       allowedSwapper[pool][router] == true  â†’  passes
7. Bob's swap executes against the curated pool, bypassing the allowlist entirely.
```
