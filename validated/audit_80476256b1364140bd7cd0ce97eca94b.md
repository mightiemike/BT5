### Title
`tx_index` Aliasing in `compute_root_from_merkle_proof` Allows Wrong-Index Inclusion Proof to Return `true` — (`merkle-tools/src/lib.rs`)

---

### Summary

`compute_root_from_merkle_proof` consumes only the low `proof_length` bits of `tx_index` (via repeated `% 2` / `/= 2`). Any `tx_index` value that is a multiple of `2^proof_length` produces an identical traversal path to `tx_index = 0`. There is no guard that bounds `tx_index < 2^proof_length`. An unprivileged NEAR caller can therefore supply an out-of-range `tx_index` (e.g., `4` for a 2-level proof) paired with a valid proof for index `0`, and `verify_transaction_inclusion` returns `true` for a leaf position that does not exist in the block.

---

### Finding Description

`ProofArgs.tx_index` is declared as `u64`: [1](#0-0) 

It is cast to `usize` without any upper-bound check and forwarded directly to `compute_root_from_merkle_proof`: [2](#0-1) 

Inside `compute_root_from_merkle_proof`, the position is consumed by `% 2` (left/right decision) and `/= 2` (move up one level), iterated exactly `proof_length` times: [3](#0-2) 

After `proof_length` halvings, only the low `proof_length` bits of the original `tx_index` have been examined. Any two values that share those bits produce an identical hash chain. Concretely, for `proof_length = 2`:

| `tx_index` | Step 1 (`pos % 2`) | `pos /= 2` | Step 2 (`pos % 2`) | `pos /= 2` | Result |
|---|---|---|---|---|---|
| `0` (binary `00`) | 0 → left | 0 | 0 → left | 0 | `hash(hash(tx, p0), p1)` |
| `4` (binary `100`) | 0 → left | 2 | 0 → left | 1 | `hash(hash(tx, p0), p1)` |

Both produce the same root. Index `4` does not exist in a 4-leaf tree (valid range: 0–3), yet the proof verifies.

The same flaw is present in `verify_transaction_inclusion_v2`: its coinbase sub-check uses a hardcoded `0usize` and is unaffected, but it then delegates to `verify_transaction_inclusion` with the caller-supplied `tx_index`: [4](#0-3) 

---

### Impact Explanation

`verify_transaction_inclusion` (and `_v2`) return `true` for a `(tx_id, tx_index)` pair where `tx_index` names a leaf position that does not exist in the block. Any downstream protocol that treats the `(tx_id, tx_index, blockhash)` triple as a unique, position-verified inclusion claim — for example, to prevent double-spending proofs, to distinguish coinbase from non-coinbase, or to enforce per-position uniqueness — can be deceived. The contract's own documented guarantee ("True if `tx_id` is **at the claimed position**") is violated.

---

### Likelihood Explanation

The attack requires no privileges. `verify_transaction_inclusion` is a public NEAR view/call with no access-control gate beyond the `#[pause]` flag. The attacker only needs to know a valid proof for any real transaction at index `0` in any mainchain block (the coinbase proof, which is always publicly derivable from the block). They then replay it with `tx_index = N * 2^proof_length` for any integer `N > 0`.

---

### Recommendation

After converting `tx_index` to `usize`, assert it is strictly less than `2^proof_length` (i.e., `2_usize.pow(merkle_proof.len() as u32)`):

```rust
let position = usize::try_from(args.tx_index).unwrap();
require!(
    position < (1usize << args.merkle_proof.len()),
    "tx_index out of range for the given proof depth"
);
```

This check costs O(1) and eliminates the entire aliasing class.

### Proof of Concept

```rust
#[test]
fn test_tx_index_aliasing() {
    // 4-leaf tree: [A, B, C, D]
    let a = H256([1u8; 32]);
    let b = H256([2u8; 32]);
    let c = H256([3u8; 32]);
    let d = H256([4u8; 32]);

    // Build level-1 nodes
    let ab = compute_hash(&a, &b);
    let cd = compute_hash(&c, &d);
    // Root
    let root = compute_hash(&ab, &cd);

    // Valid proof for tx A at index 0: siblings are [B, CD]
    let proof = vec![b.clone(), cd.clone()];

    // Legitimate call: index 0 → should return root ✓
    let r0 = compute_root_from_merkle_proof(a.clone(), 0, &proof);
    assert_eq!(r0, root);

    // Attack: index 4 (does not exist in a 4-leaf tree) with the same proof
    let r4 = compute_root_from_merkle_proof(a.clone(), 4, &proof);
    assert_eq!(r4, root);  // passes — same root, wrong position

    // Both are equal: the aliasing is confirmed
    assert_eq!(r0, r4);
}
```

#[must_use]
pub fn compute_root_from_merkle_proof(
    transaction_hash: H256,
    transaction_position: usize,
    merkle_proof: &Vec<H256>,
) -> H256 {
    let mut current_hash = transaction_hash;
    let mut current_position = transaction_position;

    for proof_hash in merkle_proof {
        if current_position % 2 == 0 {
            current_hash = compute_hash(&current_hash, proof_hash);
        } else {
            current_hash = compute_hash(proof_hash, &current_hash);
        }
        current_position /= 2;
    }

    current_hash
}
