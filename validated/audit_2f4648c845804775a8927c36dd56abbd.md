Audit Report

## Title
`SwapAllowlistExtension` Allowlist Bypassed via Router Intermediary — (`metric-periphery/contracts/extensions/SwapAllowlistExtension.sol`)

## Summary
`SwapAllowlistExtension.beforeSwap` checks the `sender` argument against a per-pool allowlist, but `MetricOmmPool.swap` passes its own `msg.sender` as `sender`. When `MetricOmmSimpleRouter` calls `pool.swap`, the extension receives the router's address as `sender`, not the originating user. Allowlisting the router — the necessary step for any permitted user to use the standard periphery — opens the gate to all callers.

## Finding Description
In `SwapAllowlistExtension.beforeSwap` (L37), the check is:

```solidity
if (!allowAllSwappers[msg.sender] && !allowedSwapper[msg.sender][sender]) {
    revert IMetricOmmPoolActions.NotAllowedToSwap();
}
```

`msg.sender` here is the pool (the hook caller), and `sender` is the first argument forwarded by the pool. In `MetricOmmPool.swap` (L230–240), the pool passes its own `msg.sender` as `sender`:

```solidity
_beforeSwap(
    msg.sender,   // whoever called pool.swap()
    recipient,
    ...
);
```

In `MetricOmmSimpleRouter.exactInputSingle` (L72–80), the router calls `pool.swap` directly with no originating-user forwarding:

```solidity
(int128 amount0Delta, int128 amount1Delta) = IMetricOmmPoolActions(params.pool)
    .swap(
        params.recipient,
        params.zeroForOne,
        MetricOmmSwapInputs.asAmountSpecifiedIn(params.amountIn),
        priceLimitX64,
        "",
        params.extensionData
    );
```

The pool's `msg.sender` is the router, so the extension evaluates `allowedSwapper[pool][router]`. The actual caller's identity is never checked. The same pattern applies to `exactInput` (L104), `exactOutputSingle` (L136), and `exactOutput` (L165). [1](#0-0) [2](#0-1) [3](#0-2) 

## Impact Explanation
A pool deploying `SwapAllowlistExtension` intends to restrict swaps to a specific set of addresses. Once the router is allowlisted — required for any allowlisted user to use the standard periphery — the restriction is entirely bypassed. Any unpermissioned EOA can call `exactInputSingle` on the public router and swap against LP capital deposited under the assumption of a restricted counterparty set. This is unauthorized access to pool liquidity and a direct violation of the LP's risk model, constituting a high-severity loss of access-control integrity with direct fund exposure.

## Likelihood Explanation
Exploitation requires only that the pool admin allowlists the router, which is the expected operational step for any permissioned pool supporting the standard periphery. No special privilege, flash loan, or unusual token behavior is needed — any EOA can call `exactInputSingle` on the public router. The combination of a natural admin action and a trivial user action makes exploitation highly likely once a permissioned pool is live.

## Recommendation
The extension must gate the originating user, not the immediate `msg.sender` of the pool call. The cleanest fix is to have the router encode `msg.sender` into `extensionData` and have `SwapAllowlistExtension.beforeSwap` decode and verify that field when present. This requires coordinated changes to the router (populate a standard "originating caller" field in `extensionData`) and the extension (decode and check it). Alternatively, require direct pool calls for permissioned pools and prohibit router allowlisting, though this is operationally restrictive.

## Proof of Concept
```
Setup:
  pool configured with SwapAllowlistExtension
  allowedSwapper[pool][alice]   = true   (alice is permitted)
  allowedSwapper[pool][router]  = true   (router allowlisted so alice can use it)
  allowedSwapper[pool][mallory] = false  (mallory is NOT permitted)

Attack:
  mallory calls router.exactInputSingle({pool: pool, ...})
    → router calls pool.swap(recipient=mallory, ...)
    → pool calls extension.beforeSwap(sender=router, ...)
    → extension checks allowedSwapper[pool][router] == true  ✓
    → swap executes — mallory receives output tokens

Result:
  mallory swaps successfully against a pool she was explicitly excluded from.
  The allowlist invariant is broken; LP capital is exposed to an unauthorized counterparty.
```
