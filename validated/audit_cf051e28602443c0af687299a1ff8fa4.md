### Title
Insufficient Bonus Pool Permanently Blocks `TierUndelegate`, Trapping User Principal — (`x/tieredrewards/keeper/claim_rewards.go`, `x/tieredrewards/keeper/msg_server.go`)

---

### Summary

`MsgTierUndelegate` (and `MsgExitTierWithDelegation`) atomically settle accrued bonus rewards before undelegating. If the `RewardsPoolName` module account has insufficient balance to cover the accrued bonus, the entire transaction fails with `ErrInsufficientBonusPool`. Because `MsgWithdrawFromTier` requires the position to be fully undelegated, a permanently empty bonus pool traps the user's locked principal with no emergency exit path.

---

### Finding Description

`processEventsAndClaimBonus` in `x/tieredrewards/keeper/claim_rewards.go` computes the accrued bonus for a position and then calls `sufficientBonusPoolBalance` before transferring:

```go
if err := k.sufficientBonusPoolBalance(ctx, bonusCoins); err != nil {
    return nil, err   // ErrInsufficientBonusPool
}
if err := k.bankKeeper.SendCoinsFromModuleToAccount(...); err != nil {
    return nil, err
}
``` [1](#0-0) 

This function is called by `claimRewards`, which is called by every mutation handler that touches a delegated position — including `TierUndelegate` — as documented in the ADR and confirmed by the unit tests that explicitly fund the pool before calling `TierUndelegate`: [2](#0-1) 

`MsgWithdrawFromTier` itself does **not** claim rewards; it only sends `SpendableCoins` from the position's delegator address to the owner. However, it enforces that the position is not delegated: [3](#0-2) 

The only way to clear the `IsDelegated()` flag is through `TierUndelegate` or `ExitTierWithDelegation`, both of which call `claimRewards` first. There is no emergency undelegation path that skips reward settlement.

The full blocking chain is:

```
bonus pool empty
  → TierUndelegate calls claimRewards
    → processEventsAndClaimBonus returns ErrInsufficientBonusPool
      → TierUndelegate fails, position stays delegated
        → WithdrawFromTier returns ErrPositionDelegated
          → user's principal is permanently inaccessible
``` [4](#0-3) [5](#0-4) 

---

### Impact Explanation

The corrupted invariant is the user's locked principal (bond-denom tokens held at the position's deterministic delegator address). When the bonus pool is empty and stays empty, the user cannot call `TierUndelegate`, cannot satisfy `WithdrawFromTier`'s `!IsDelegated()` precondition, and cannot recover their principal. The position is deleted only inside `WithdrawFromTier` after the send succeeds; it is never deleted by the failed `TierUndelegate`. The exact balance at risk is `pos.Amount` (the full locked principal) for every position whose accrued bonus exceeds the pool balance.

The existing test `TestMsgClaimTierRewards_FailsWhenBonusPoolInsufficient` acknowledges this failure mode and notes "the user can retry later" — but that assumption breaks if the pool is never refunded. [6](#0-5) 

---

### Likelihood Explanation

The `RewardsPoolName` module account is funded by governance/community transfers and drained by two concurrent paths: `BeginBlocker`'s `topUpBaseRewards` (which drains the pool every block to top up the fee collector) and `processEventsAndClaimBonus` (which pays per-position bonus on every claim/mutation). If governance stops funding the pool — or if the pool is drained faster than it is refilled — every position with non-zero accrued bonus is blocked from undelegating. This is a realistic operational condition, not a theoretical one: the `BeginBlocker` drains the pool continuously even when no positions are claiming. [7](#0-6) 

---

### Recommendation

Add an `EmergencyTierUndelegate` message (or a flag on `MsgTierUndelegate`) that skips bonus reward settlement and proceeds directly to undelegation. The accrued-but-unpaid bonus checkpoint should be preserved so it can be claimed later when the pool is refunded, but the principal withdrawal path must not be gated on bonus pool solvency. This mirrors the M-02 recommendation of an `emergencyWithdraw` that ignores failed reward token transfers.

---

### Proof of Concept

1. User calls `MsgLockTier` and delegates principal.
2. Time passes; bonus accrues. The `RewardsPoolName` account is drained to zero (e.g., by `BeginBlocker` top-ups with no governance refill).
3. User calls `MsgTierUndelegate`. Inside `claimRewards` → `processEventsAndClaimBonus` → `sufficientBonusPoolBalance` returns `ErrInsufficientBonusPool`. Transaction reverts.
4. User calls `MsgWithdrawFromTier`. `validateWithdrawFromTier` returns `ErrPositionDelegated` because step 3 never cleared the delegation.
5. User's principal is permanently locked with no recovery path. [8](#0-7) [9](#0-8)

func (k Keeper) validateWithdrawFromTier(ctx context.Context, pos types.PositionState, owner string) error {
	if !pos.IsOwner(owner) {
		return types.ErrNotPositionOwner
	}

	if !pos.HasTriggeredExit() {
		return types.ErrExitNotTriggered
	}

	sdkCtx := sdk.UnwrapSDKContext(ctx)
	if !pos.CompletedExitLockDuration(sdkCtx.BlockTime()) {
		return types.ErrExitLockDurationNotReached
	}

	if pos.IsDelegated() {
		return types.ErrPositionDelegated
	}

	isUnbonding, err := k.isUnbonding(ctx, pos.DelegatorAddress)
	if err != nil {
		return err
	}
	if isUnbonding {
		return types.ErrPositionUnbonding
	}

	return nil
}

func (k Keeper) sufficientBonusPoolBalance(ctx context.Context, bonus sdk.Coins) error {
	if bonus.IsZero() {
		return nil
	}

	poolAddr := k.accountKeeper.GetModuleAddress(types.RewardsPoolName)
	poolBalance := k.bankKeeper.GetAllBalances(ctx, poolAddr)
	if !poolBalance.IsAllGTE(bonus) {
		return errorsmod.Wrapf(types.ErrInsufficientBonusPool,
			"bonus: %s, pool balance: %s",
			bonus.String(), poolBalance.String())
	}

	return nil

func (ms msgServer) WithdrawFromTier(ctx context.Context, msg *types.MsgWithdrawFromTier) (*types.MsgWithdrawFromTierResponse, error) {
	if err := msg.Validate(); err != nil {
		return nil, err
	}

	pos, err := ms.getPositionState(ctx, msg.PositionId)
	if err != nil {
		return nil, err
	}

	if err := ms.validateWithdrawFromTier(ctx, pos, msg.Owner); err != nil {
		return nil, err
	}

	ownerAddr, err := sdk.AccAddressFromBech32(pos.Owner)
	if err != nil {
		return nil, errorsmod.Wrap(sdkerrors.ErrInvalidAddress, "invalid owner address")
	}

	delAddr, err := sdk.AccAddressFromBech32(pos.DelegatorAddress)
	if err != nil {
		return nil, errorsmod.Wrap(sdkerrors.ErrInvalidAddress, "invalid delegator address")
	}

	balances := ms.bankKeeper.SpendableCoins(ctx, delAddr)
	if !balances.IsZero() {
		if err := ms.bankKeeper.SendCoins(ctx, delAddr, ownerAddr, balances); err != nil {
			return nil, err
		}
	}

// TestMsgClaimTierRewards_FailsWhenBonusPoolInsufficient verifies that ClaimTierRewards
// returns ErrInsufficientBonusPool when accrued bonus cannot be paid, so the tx rolls
// back and the user can retry later without losing base rewards to a partial claim.
func (s *KeeperSuite) TestMsgClaimTierRewards_FailsWhenBonusPoolInsufficient() {
	lockAmount := sdkmath.NewInt(sdk.DefaultPowerReduction.Int64())
	pos := s.setupNewTierPosition(lockAmount, false)
	delAddr := sdk.MustAccAddressFromBech32(pos.Owner)
	valAddr := sdk.MustValAddressFromBech32(pos.Delegation.ValidatorAddress)
	_, bondDenom := s.getStakingData()
	msgServer := keeper.NewMsgServerImpl(s.keeper)

	s.setValidatorCommission(valAddr, sdkmath.LegacyZeroDec())

	// Advance time and allocate base rewards, but intentionally leave bonus pool empty.
	s.ctx = s.ctx.WithBlockHeight(s.ctx.BlockHeight() + 1)
	s.ctx = s.ctx.WithBlockTime(s.ctx.BlockTime().Add(time.Hour * 24 * 365))
	s.allocateRewardsToValidator(valAddr, sdkmath.NewInt(100), bondDenom)

	// Bonus pool remains at 0 — bonus accrued but pool cannot cover it.
	balBefore := s.app.BankKeeper.GetBalance(s.ctx, delAddr, bondDenom)

	// Use a branched context so a failed message does not persist state (matches DeliverTx rollback).
	cacheCtx, _ := s.ctx.CacheContext()
	resp, err := msgServer.ClaimTierRewards(cacheCtx, &types.MsgClaimTierRewards{
		Owner:       delAddr.String(),
		PositionIds: []uint64{pos.Id},
	})
	s.Require().Error(err)
	s.Require().True(errors.Is(err, types.ErrInsufficientBonusPool))
	s.Require().Nil(resp)

	balAfter := s.app.BankKeeper.GetBalance(s.ctx, delAddr, bondDenom)
	s.Require().True(balAfter.Amount.Equal(balBefore.Amount), "failed claim must not transfer rewards")
}
