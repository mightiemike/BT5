Audit Report

## Title
SwapAllowlistExtension checks router address instead of real user, allowing full allowlist bypass via MetricOmmSimpleRouter - (File: metric-periphery/contracts/extensions/SwapAllowlistExtension.sol)

## Summary
`SwapAllowlistExtension.beforeSwap` identifies the swapper by the `sender` argument, which is `msg.sender` of the pool's `swap` call. When a user enters through `MetricOmmSimpleRouter`, the pool receives the router contract as `msg.sender`, so the extension evaluates `allowedSwapper[pool][router]` rather than `allowedSwapper[pool][realUser]`. Any pool admin who allowlists the router to support standard periphery swaps inadvertently grants every user on the network unrestricted access to the curated pool.

## Finding Description
`MetricOmmPool.swap` passes `msg.sender` directly as the `sender` argument to `_beforeSwap`: [1](#0-0) 

`ExtensionCalling._beforeSwap` forwards that value unchanged through `_callExtensionsInOrder`: [2](#0-1) 

`SwapAllowlistExtension.beforeSwap` uses `msg.sender` (the pool) as the pool key and the received `sender` as the swapper identity: [3](#0-2) 

`MetricOmmSimpleRouter.exactInputSingle` calls `pool.swap(...)` directly, making the router contract `msg.sender` of the pool. The real user's address is stored only in transient storage via `_setNextCallbackContext` for payment settlement and is never forwarded to the extension: [4](#0-3) 

This creates an irreconcilable dilemma: if the admin does not allowlist the router, all router-mediated swaps revert with `NotAllowedToSwap`; if the admin does allowlist the router, the check `allowedSwapper[pool][router] == true` passes for every user who routes through the public periphery, completely nullifying the allowlist. There is no configuration that simultaneously permits router-mediated swaps for specific users and blocks them for others.

## Impact Explanation
Any unprivileged user can bypass the swap allowlist on a curated pool by calling `MetricOmmSimpleRouter` instead of the pool directly. If the allowlist was deployed to restrict trading to specific market makers or counterparties — a standard use case for oracle-anchored pools — the bypass allows arbitrary actors to trade against LP positions at oracle prices. This constitutes a direct loss-of-LP-principal path reachable by any user with token approval, satisfying the admin-boundary break and direct loss-of-user-principal criteria.

## Likelihood Explanation
`MetricOmmSimpleRouter` is the standard, documented swap entry point. A pool admin who wants allowlisted users to trade through the normal UX will naturally allowlist the router. The bypass requires no special knowledge or privilege: any user who observes the pool has a swap allowlist simply calls the router instead of the pool directly. The attack is a single public transaction with no preconditions beyond a token approval.

## Recommendation
The extension must check the economically relevant actor, not the immediate pool caller. Two viable approaches:

1. **Pass the real user through `extensionData`:** Have the router encode `msg.sender` into `extensionData` and have `SwapAllowlistExtension` decode and verify it, enforced by checking that the pool's caller (`sender`) is a factory-registered trusted router.

2. **Trusted-router registry:** Maintain a factory-level registry of trusted routers; when `sender` is a trusted router, decode the real user from `extensionData`; otherwise use `sender` directly.

The same fix must be applied symmetrically to `DepositAllowlistExtension` if the identical `sender`-forwarding pattern is used for liquidity adder paths.

## Proof of Concept
```
Setup:
  pool = deploy MetricOmmPool with SwapAllowlistExtension as beforeSwap extension
  admin calls extension.setAllowedToSwap(pool, router, true)
    // admin intends to allow router-mediated swaps for allowlisted users only
  alice = allowlisted EOA
  bob   = non-allowlisted EOA

Attack:
  bob calls router.exactInputSingle({pool: pool, ...})
    // router calls pool.swap(...) — msg.sender of pool = router
    // pool calls extension.beforeSwap(sender=router, ...)
    // extension checks allowedSwapper[pool][router] == true  ✓
    // swap executes — bob bypassed the allowlist

Verification:
  bob calls pool.swap(...) directly
    // pool calls extension.beforeSwap(sender=bob, ...)
    // extension checks allowedSwapper[pool][bob] == false  → NotAllowedToSwap ✓
```

The direct call correctly blocks `bob`, but the router path succeeds because the extension sees the router's address. The allowlist invariant is broken for every user who routes through the public periphery.
