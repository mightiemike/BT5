### Title
`ValidResourceBounds` Protobuf Deserialization Silently Downgrades `AllResources` to `L1Gas`, Producing a Different Transaction Hash - (File: `crates/apollo_protobuf/src/converters/transaction.rs`)

### Summary

The `TryFrom<protobuf::ResourceBounds> for ValidResourceBounds` implementation silently converts an `AllResources` variant to `L1Gas` whenever `l2_gas == 0` and `l1_data_gas == 0`. Because `get_tip_resource_bounds_hash` appends the `L1_DATA_GAS` element to the hash chain only for `AllResources`, the two variants produce structurally different Poseidon hash inputs. A transaction whose `resource_bounds` is `AllResources` with zero L2/data-gas fields will therefore carry a different `tx_hash` after a protobuf round-trip than the one the signer committed to, breaking signature binding across the consensus P2P boundary.
