### Title
`min_gas_price_percentage` Typed as `u8` Limits Gateway Admission Threshold to 1% Granularity, Causing Incorrect Accept/Reject Decisions - (File: `crates/apollo_gateway_config/src/config.rs`)

### Summary

The `StatefulTransactionValidatorConfig::min_gas_price_percentage` field is declared as `u8`, restricting the configurable L2 gas price admission threshold to whole-integer percentages (0%, 1%, 2%, …, 255%). The gateway uses this value to compute the minimum acceptable L2 gas price for incoming transactions. Because sub-1% precision is impossible to express, any intended threshold that falls between two integers (e.g., 99.5%) forces the operator to round up or down, causing the gateway to either accept transactions it should reject or reject transactions it should accept.

### Finding Description

`StatefulTransactionValidatorConfig` declares the admission threshold as:

```rust
pub min_gas_price_percentage: u8, // E.g., 80 to require 80% of threshold.
``` [1](#0-0) 

The default value is `100` and the production deployment configs hard-code it to `100`: [2](#0-1) 

In `validate_tx_l2_gas_price_within_threshold`, the field is cast to `u128` and used as the numerator of a `Ratio` over `100`:

```rust
let gas_price_threshold_multiplier =
    Ratio::new(self.config.min_gas_price_percentage.into(), 100_u128);
let threshold = (gas_price_threshold_multiplier
    * previous_block_l2_gas_price.get().0)
    .to_integer();
if tx_l2_gas_price.0 < threshold { ... }
``` [3](#0-2) 

Because `u8` only holds integers, the ratio numerator is always a whole number. The finest expressible step is exactly 1 percentage point. Any desired threshold between two integers — for example, 99.5% — cannot be represented. The operator must choose either 99 or 100:

- Choosing **99** → the gateway accepts transactions whose L2 gas price is between 99% and 99.5% of the previous block price, i.e., transactions that should have been rejected.
- Choosing **100** → the gateway rejects transactions whose L2 gas price is between 99.5% and 100% of the previous block price, i.e., transactions that should have been accepted.

The same coarseness applies across the entire range: a threshold of 0.5% (half a percent below the block price) is completely inexpressible.

### Impact Explanation

This directly affects gateway/mempool admission. The gateway is the first line of defense for transaction acceptance. A coarse threshold means the sequencer either:

1. **Accepts transactions with an L2 gas price below the intended minimum** — these under-priced transactions enter the mempool and can be sequenced, undermining the economic floor the operator intended to enforce.
2. **Rejects valid transactions** — transactions whose gas price satisfies the intended (fractional) threshold are turned away, degrading liveness.

Both outcomes match the allowed impact: *"High. Mempool/gateway/RPC admission accepts invalid transactions or rejects valid transactions before sequencing."*

### Likelihood Explanation

The current default and deployment value is exactly `100`, so the issue is latent but not yet triggered. However, any future tuning of the threshold to a non-round value (e.g., 99.5% to allow slight underbidding, or 100.5% to require a small premium) will silently snap to the nearest integer, producing the wrong admission boundary. Likelihood is moderate (analogous to the external report's 5/10) because the parameter is explicitly designed to be operator-configurable.

### Recommendation

Replace `u8` with a type that supports sub-1% precision. Two practical options:

1. **Use basis points (`u16` or `u32`)**: express the threshold in hundredths of a percent (e.g., `10000` = 100.00%). Change the denominator in the ratio from `100` to `10_000`. This gives 0.01% granularity with no floating-point risk.

2. **Use a rational/decimal type**: store numerator and denominator separately, or use a `Ratio<u64>` directly in the config, serialized as a JSON fraction.

The config schema, default value, deployment JSON files, and the `validate_tx_l2_gas_price_within_threshold` computation must all be updated consistently. [4](#0-3) 

### Proof of Concept

1. Set `min_gas_price_percentage = 99` (the only integer below 100 available).
2. Previous block L2 gas price = 1000 STRK/gas.
3. Computed threshold = `floor((99/100) * 1000)` = `990`.
4. Submit a transaction with `max_price_per_unit = 995` (which is 99.5% of 1000 — above the intended 99.5% floor but below 100%).
5. The gateway accepts it (`995 >= 990`), even though the operator's intended threshold of 99.5% would have required `max_price_per_unit >= 995` exactly — and any value of 994 would also pass, despite being below 99.5%.

Conversely, set `min_gas_price_percentage = 100`:

1. Threshold = `floor((100/100) * 1000)` = `1000`.
2. Submit a transaction with `max_price_per_unit = 995` (99.5% of 1000).
3. The gateway rejects it (`995 < 1000`), even though the operator's intended 99.5% threshold would have accepted it. [5](#0-4) [6](#0-5)

    fn validate_tx_l2_gas_price_within_threshold(
        &self,
        tx_resource_bounds: ValidResourceBounds,
        previous_block_l2_gas_price: NonzeroGasPrice,
    ) -> StatefulTransactionValidatorResult<()> {
        match tx_resource_bounds {
            ValidResourceBounds::AllResources(tx_resource_bounds) => {
                let tx_l2_gas_price = tx_resource_bounds.l2_gas.max_price_per_unit;
                let gas_price_threshold_multiplier =
                    Ratio::new(self.config.min_gas_price_percentage.into(), 100_u128);
                let threshold = (gas_price_threshold_multiplier
                    * previous_block_l2_gas_price.get().0)
                    .to_integer();
                if tx_l2_gas_price.0 < threshold {
                    return Err(StarknetError {
                        // We didn't have this kind of an error.
                        code: StarknetErrorCode::UnknownErrorCode(
                            "StarknetErrorCode.GAS_PRICE_TOO_LOW".to_string(),
                        ),
                        message: format!(
                            "Transaction L2 gas price {tx_l2_gas_price} is below the required \
                             threshold {threshold}.",
                        ),
                    });
                }
            }
            ValidResourceBounds::L1Gas(_) => {
                // No validation required for legacy transactions.
            }
        }
        Ok(())
    }
