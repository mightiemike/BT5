### Title
NLP Locked Balance Queue Never Decremented on Burn Allows Bypassing Lock Period — (`core/contracts/Clearinghouse.sol` / `core/contracts/SpotEngineState.sol`)

---

### Summary

`burnNlp` in `Clearinghouse.sol` enforces the NLP lock period by checking `getNlpUnlockedBalance`, but the underlying `unlockedBalanceSum` counter in the `NlpLockedBalanceQueue` is **never decremented** after a burn. Only the main `BalanceNormalized` balance is reduced. This desynchronization between the two tracking mechanisms allows a user to repeatedly pass the unlocked-balance gate and burn NLP tokens that are still within their lock period.

---

### Finding Description

`burnNlp` performs two distinct accounting operations:

1. **Gate check** — reads `spotEngine.getNlpUnlockedBalance(txn.sender).amount` to verify the user has enough unlocked NLP.
2. **Balance update** — calls `spotEngine.updateBalance(NLP_PRODUCT_ID, txn.sender, -nlpAmount)` to reduce the main `BalanceNormalized` balance. [1](#0-0) 

The unlocked balance is maintained in `NlpLockedBalanceQueue.unlockedBalanceSum` inside `SpotEngineState`. `tryUnlockNlpBalance` only ever **adds** to `unlockedBalanceSum` as time-locked entries mature; it never subtracts from it. [2](#0-1) 

Because `burnNlp` calls `updateBalance` (which writes to `balances[NLP_PRODUCT_ID][subaccount]`) and **not** any function that decrements `unlockedBalanceSum`, the queue counter is permanently stale after the first burn. Every subsequent call to `getNlpUnlockedBalance` returns the same pre-burn value, so the gate check passes again even though the user has already consumed their unlocked allocation. [3](#0-2) 

The only backstop is the post-burn assertion:

```solidity
require(
    spotEngine.getBalance(NLP_PRODUCT_ID, txn.sender).amount >= 0,
    ERR_SUBACCT_HEALTH
);
``` [4](#0-3) 

This prevents the main balance from going negative, but it does **not** enforce the lock period. A user can drain their entire NLP balance — including the locked portion — as long as the main balance stays ≥ 0 on each individual burn call.

---

### Impact Explanation

A user who has both locked and unlocked NLP can burn all of their NLP (locked + unlocked) by making repeated `burnNlp` calls. Each call passes the `getNlpUnlockedBalance` gate because `unlockedBalanceSum` is never reduced. The user receives quote tokens (`quoteAmount = nlpAmount * oraclePriceX18 - burnFee`) for NLP that was supposed to remain locked, effectively withdrawing liquidity before the lock period expires. This corrupts the protocol's NLP supply accounting and undermines the economic guarantees of the lock mechanism. [5](#0-4) 

---

### Likelihood Explanation

Any user who has minted NLP and has at least some unlocked balance can trigger this. The entry path is the standard `burnNlp` transaction submitted through the Endpoint and processed by the sequencer — no special privileges are required. The condition (having both locked and unlocked NLP) is the normal state for any long-term NLP holder, making this straightforwardly reachable. [6](#0-5) 

---

### Recommendation

After the `updateBalance` call in `burnNlp`, decrement `unlockedBalanceSum` in the `NlpLockedBalanceQueue` by `nlpAmount`. Expose a dedicated function in `SpotEngine`/`SpotEngineState` (e.g., `consumeNlpUnlockedBalance(bytes32 subaccount, int128 amount)`) that atomically reduces `unlockedBalanceSum` and is called from `burnNlp`. Add an invariant assertion that `unlockedBalanceSum.amount` never exceeds `getBalance(NLP_PRODUCT_ID, subaccount).amount` after any burn. [7](#0-6) 

---

### Proof of Concept

**Setup:**
- Alice mints 10 NLP. All 10 are locked for 30 days. Main balance = 10, `unlockedBalanceSum` = 0.
- 15 days pass. 3 NLP unlock. `tryUnlockNlpBalance` runs → `unlockedBalanceSum` = 3. Main balance = 10.

**Exploit:**
1. Alice calls `burnNlp(3)`. Gate: `getNlpUnlockedBalance` = 3 ≥ 3 ✓. `updateBalance` → main balance = 7. `unlockedBalanceSum` **unchanged** = 3.
2. Alice calls `burnNlp(3)`. Gate: `getNlpUnlockedBalance` = 3 ≥ 3 ✓. `updateBalance` → main balance = 4. `unlockedBalanceSum` still = 3.
3. Alice calls `burnNlp(3)`. Gate: passes again. Main balance = 1.
4. Alice calls `burnNlp(1)`. Gate: `getNlpUnlockedBalance` = 3 ≥ 1 ✓. Main balance = 0.

**Result:** Alice burned all 10 NLP and received quote tokens for 10 NLP, despite only 3 being unlocked. The 7 locked NLP were extracted before their lock period expired, violating the protocol's liquidity lock invariant. [8](#0-7) [9](#0-8)

    function burnNlp(
        IEndpoint.BurnNlp calldata txn,
        int128 oraclePriceX18,
        IEndpoint.NlpPool[] calldata nlpPools,
        int128[] calldata nlpPoolRebalanceX18
    ) external onlyEndpoint {
        require(!RiskHelper.isIsolatedSubaccount(txn.sender), ERR_UNAUTHORIZED);

        ISpotEngine spotEngine = _spotEngine();
        spotEngine.updatePrice(NLP_PRODUCT_ID, oraclePriceX18);

        require(txn.nlpAmount <= INT128_MAX, ERR_CONVERSION_OVERFLOW);
        int128 nlpAmount = int128(txn.nlpAmount);
        require(
            spotEngine.getNlpUnlockedBalance(txn.sender).amount >= nlpAmount,
            ERR_UNLOCKED_NLP_INSUFFICIENT
        );
        int128 quoteAmount = nlpAmount.mul(oraclePriceX18);
        int128 burnFee = MathHelper.max(ONE, quoteAmount / 1000);
        quoteAmount = MathHelper.max(0, quoteAmount - burnFee);

        _validateNlpRebalance(nlpPools, nlpPoolRebalanceX18, -quoteAmount);
        for (uint128 i = 0; i < nlpPoolRebalanceX18.length; i++) {
            require(nlpPoolRebalanceX18[i] <= 0, ERR_INVALID_NLP_REBALANCE);
        }

        spotEngine.updateBalance(NLP_PRODUCT_ID, txn.sender, -nlpAmount);
        spotEngine.updateBalance(NLP_PRODUCT_ID, N_ACCOUNT, nlpAmount);

        if (quoteAmount > 0) {
            spotEngine.updateBalance(QUOTE_PRODUCT_ID, txn.sender, quoteAmount);
            _applyNlpRebalance(spotEngine, nlpPools, nlpPoolRebalanceX18);
        }

        require(
            spotEngine.getBalance(NLP_PRODUCT_ID, txn.sender).amount >= 0,
            ERR_SUBACCT_HEALTH
        );

    function tryUnlockNlpBalance(bytes32 subaccount)
        public
        returns (Balance memory)
    {
        NlpLockedBalanceQueue storage queue = nlpLockedBalanceQueues[
            subaccount
        ];
        while (
            queue.unlockedUpTo < queue.balanceCount &&
            queue.balances[queue.unlockedUpTo].unlockedAt <= getOracleTime()
        ) {
            // we can unlock this balance
            queue.unlockedBalanceSum.amount += queue
                .balances[queue.unlockedUpTo]
                .balance
                .amount;
            delete queue.balances[queue.unlockedUpTo];
            queue.unlockedUpTo++;
        }

        return queue.unlockedBalanceSum;
    }
