### Title
AuxPoW Validation Silently Bypassed When `aux_data` Is `None` for AuxPoW-Flagged Dogecoin Blocks - (`contract/src/dogecoin.rs`)

### Summary

The Dogecoin `submit_block_header` function accepts `Option<AuxData>` as the AuxPoW payload. When a relayer submits a block with the AuxPoW version flag set (`version & 0x100 != 0`) but provides `None` for `aux_data`, the entire AuxPoW validation path is silently skipped. The code falls back to checking the Dogecoin block's own hash against the target — which is the wrong PoW check for AuxPoW blocks. This allows a block that the real Dogecoin network would reject to be accepted by the light client, corrupting its canonical chain state.

### Finding Description

The `BlockHeader` type for the Dogecoin build is `(Header, Option<AuxData>)`. [1](#0-0) 

In `submit_block_header`, when `skip_pow_verification` is `false`, the code branches on whether `aux_data` is `Some` or `None`: [2](#0-1) 

When `aux_data` is `None`, the fallback path checks `block_header.block_hash_pow()` — the Dogecoin block's own hash — against the target. For AuxPoW blocks, this is the wrong hash: the PoW is performed on the *parent Bitcoin block's* hash, not the Dogecoin block's hash. The `check_aux` function, which enforces the AuxPoW flag, validates the parent block's PoW, verifies the coinbase merkle proof, and checks the chain merkle root, is never called. [3](#0-2) 

Critically, the check inside `check_aux` that enforces the AuxPoW flag is present only within that function and is never reached when `aux_data` is `None`: [4](#0-3) 

The `bits` field is validated by `check_target` → `check_pow` → `get_next_work_required`, so the attacker cannot lower the difficulty. However, nothing prevents a relayer from submitting a block whose version has the AuxPoW bit set while providing `None` for `aux_data`. [5](#0-4) 

### Impact Explanation

A relayer who mines (or obtains) a Dogecoin block header with the AuxPoW version flag set (`version & 0x100 != 0`) and whose own hash falls below the current network target can submit it to the contract with `aux_data = None`. The contract accepts it; the real Dogecoin network would reject it (AuxPoW-flagged blocks without valid AuxPoW data are invalid by Dogecoin consensus). This corrupts the light client's canonical chain mapping (`mainchain_height_to_header`, `mainchain_header_to_height`, `mainchain_tip_blockhash`), enabling false SPV proofs via `verify_transaction_inclusion` / `verify_transaction_inclusion_v2` for transactions that do not exist on the real Dogecoin chain.

### Likelihood Explanation

The `submit_blocks` entry point is gated by `#[trusted_relayer]`, but the staking mechanism is permissionless — any NEAR account can stake to become a relayer. [6](#0-5) 

The attacker must mine a Dogecoin block with the AuxPoW flag set and a hash below the real network target, which requires the same computational work as mining a legitimate Dogecoin block. This is a meaningful barrier, but it is within reach of any participant with mining hardware. The structural analog to the external report is exact: an optional parameter (`aux_data` / `prices`) being absent silently bypasses a critical validation step rather than causing a revert.

### Recommendation

In `submit_block_header` (Dogecoin build), after the `check_target` call, inspect the block version and enforce that AuxPoW-flagged blocks must supply `aux_data`:

```rust
if !skip_pow_verification {
    self.check_target(&block_header, &prev_block_header);

    const BLOCK_VERSION_AUXPOW: i32 = 0x100;
    if block_header.version & BLOCK_VERSION_AUXPOW != 0 {
        // AuxPoW flag is set: aux_data MUST be present
        let aux = aux_data.as_ref()
            .unwrap_or_else(|| env::panic_str(
                "AuxPoW flag set but no AuxData provided"
            ));
        self.check_aux(&block_header, aux);
    } else {
        // Regular PoW block
        require!(
            aux_data.is_none(),
            "AuxData provided for non-AuxPoW block"
        );
        let pow_hash = block_header.block_hash_pow();
        require!(
            U256::from_le_bytes(&pow_hash.0) <= target_from_bits(block_header.bits),
            "block should have correct pow"
        );
    }
}
```

This mirrors the short-term mitigation recommended in the external report: revert (panic) when the required data is absent rather than silently falling back to an incorrect validation path.

### Proof of Concept

1. Compile the contract with `--features dogecoin`.
2. Initialize the contract with a valid Dogecoin genesis block.
3. Mine a Dogecoin block header where `version & 0x100 != 0` (AuxPoW flag set) and whose `double_sha256(header_bytes)` is below `target_from_bits(bits)` at the current network difficulty.
4. Call `submit_blocks` with `[(crafted_header, None)]`.
5. Observe that the block is accepted and stored in `mainchain_height_to_header` / `headers_pool` despite having no valid AuxPoW data — a block the real Dogecoin network would reject.
6. Call `verify_transaction_inclusion` with a fabricated `tx_id` and a `merkle_proof` constructed against the accepted block's `merkle_root` (which the attacker controls, since they mined the block). The function returns `true` for a transaction that does not exist on the real Dogecoin chain. [5](#0-4) [7](#0-6)

    pub(crate) fn check_aux(&mut self, block_header: &Header, aux_data: &AuxData) {
        // The Dogecoin block must have the AuxPoW flag set (bit 8) when AuxPoW data is present.
        // https://github.com/dogecoin/dogecoin/blob/master/src/auxpow.h
        const BLOCK_VERSION_AUXPOW: i32 = 0x100;

        require!(
            aux_data.chain_merkle_proof.len() <= 30,
            "Aux POW chain merkle branch too long"
        );
        require!(
            block_header.version & BLOCK_VERSION_AUXPOW != 0,
            "Aux POW block does not have AuxPoW flag set in version"
        );

        let chain_id = self.get_config().aux_chain_id;

        require!(
            chain_id == block_header.get_chain_id(),
            format!(
                "block does not have our chain ID (got {}, expected {chain_id})",
                block_header.get_chain_id()
            )
        );

        require!(
            chain_id != aux_data.parent_block.get_chain_id(),
            "Aux POW parent has our chain ID"
        );

        let chain_root = merkle_tools::compute_root_from_merkle_proof(
            block_header.block_hash(),
            aux_data.chain_id,
            &aux_data.chain_merkle_proof,
        );

        let coinbase_tx = aux_data.get_coinbase_tx();
        let coinbase_tx_hash = coinbase_tx.compute_txid();

        require!(
            merkle_tools::compute_root_from_merkle_proof(
                H256::from(coinbase_tx_hash.to_raw_hash().to_byte_array()),
                0,
                &aux_data.merkle_proof,
            ) == aux_data.parent_block.merkle_root
        );

        let script_sig = coinbase_tx
            .input
            .first()
            .unwrap()
            .script_sig
            .to_hex_string();
        let pos_merged_mining_header = script_sig.find(MERGED_MINING_HEADER);
        let mut pos_chain_root = script_sig
            .find(&chain_root.to_string())
            .expect("Aux POW missing chain merkle root in parent coinbase");

        match pos_merged_mining_header {
            Some(pos_merged_mining_header) => {
                if script_sig[pos_merged_mining_header + MERGED_MINING_HEADER.len()..]
                    .contains(MERGED_MINING_HEADER)
                {
                    env::panic_str("Multiple merged mining headers in coinbase");
                }

                require!(
                    pos_merged_mining_header + MERGED_MINING_HEADER.len() == pos_chain_root,
                    "Merged mining header is not just before chain merkle root"
                );
            }
            None => {
                require!(pos_chain_root <= 40, "Aux POW chain merkle root must start in the first 20 bytes of the parent coinbase");
            }
        }

        pos_chain_root += chain_root.to_string().len();
        require!(
            script_sig.len() - pos_chain_root >= 16,
            "Aux POW missing chain merkle tree size and nonce in parent coinbase"
        );

        let bytes = hex::decode(&script_sig[pos_chain_root..pos_chain_root + 8]).unwrap();
        let n_size = u32::from_le_bytes([bytes[0], bytes[1], bytes[2], bytes[3]]);
        require!(
            n_size == (1u32 << aux_data.chain_merkle_proof.len()),
            "Aux POW merkle branch size does not match parent coinbase"
        );

        let bytes = hex::decode(&script_sig[pos_chain_root + 8..pos_chain_root + 16]).unwrap();
        let n_nonce = u32::from_le_bytes([bytes[0], bytes[1], bytes[2], bytes[3]]);

        let chain_id = block_header.get_chain_id();

        let expected_index =
            Self::get_expected_index(n_nonce, chain_id, aux_data.chain_merkle_proof.len());

        require!(
            u32::try_from(aux_data.chain_id).ok() == Some(expected_index),
            "Aux POW wrong index"
        );
        let pow_hash = aux_data.parent_block.block_hash_pow();
        require!(
            self.skip_pow_verification
                || U256::from_le_bytes(&pow_hash.0) <= target_from_bits(block_header.bits),
            format!("block should have correct pow")
        );
    }

    pub(crate) fn submit_block_header(
        &mut self,
        header: (Header, Option<AuxData>),
        skip_pow_verification: bool,
    ) {
        let (block_header, aux_data) = header;

        let prev_block_header = self.get_prev_header(&block_header);
        let current_block_hash = block_header.block_hash();

        if !skip_pow_verification {
            self.check_target(&block_header, &prev_block_header);

            if let Some(ref aux_data) = aux_data {
                self.check_aux(&block_header, aux_data);
            } else {
                let pow_hash = block_header.block_hash_pow();
                // Check if the block hash is less than or equal to the target
                require!(
                    U256::from_le_bytes(&pow_hash.0) <= target_from_bits(block_header.bits),
                    format!("block should have correct pow")
                );
            }
        }

        let (current_block_computed_chain_work, overflow) = prev_block_header
            .chain_work
            .overflowing_add(work_from_bits(block_header.bits));
        require!(!overflow, "Addition of U256 values overflowed");

        let current_header = ExtendedHeader {
            block_header: block_header.clone().into_light(),
            block_hash: current_block_hash,
            chain_work: current_block_computed_chain_work,
            block_height: 1 + prev_block_header.block_height,
        };

        self.submit_block_header_inner(current_header, &prev_block_header);
    }

    pub fn verify_transaction_inclusion(&self, #[serializer(borsh)] args: ProofArgs) -> bool {
        require!(
            args.confirmations <= self.gc_threshold,
            "The required number of confirmations exceeds the number of blocks stored in memory"
        );

        let heaviest_block_header = self
            .headers_pool
            .get(&self.mainchain_tip_blockhash)
            .unwrap_or_else(|| env::panic_str(ERR_KEY_NOT_EXIST));
        let target_block_height = self
            .mainchain_header_to_height
            .get(&args.tx_block_blockhash)
            .unwrap_or_else(|| env::panic_str("block does not belong to the current main chain"));

        // Check requested confirmations. No need to compute proof if insufficient confirmations.
        require!(
            (heaviest_block_header.block_height).saturating_sub(target_block_height) + 1
                >= args.confirmations,
            "Not enough blocks confirmed"
        );

        let header = self
            .headers_pool
            .get(&args.tx_block_blockhash)
            .unwrap_or_else(|| env::panic_str("cannot find requested transaction block"));

        require!(!args.merkle_proof.is_empty(), "Merkle proof is empty");

        // compute merkle tree root and check if it matches block's original merkle tree root
        merkle_tools::compute_root_from_merkle_proof(
            args.tx_id,
            usize::try_from(args.tx_index).unwrap(),
            &args.merkle_proof,
        ) == header.block_header.merkle_root
    }
