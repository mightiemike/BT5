Audit Report

## Title
SwapAllowlistExtension checks router address instead of actual swapper when accessed via MetricOmmSimpleRouter, enabling allowlist bypass — (`metric-periphery/contracts/extensions/SwapAllowlistExtension.sol`)

## Summary
`SwapAllowlistExtension.beforeSwap` gates swaps by checking `allowedSwapper[msg.sender][sender]`, where `msg.sender` is the pool and `sender` is the immediate caller of `pool.swap()`. When a user routes through `MetricOmmSimpleRouter`, the pool receives `msg.sender = router`, so the extension checks `allowedSwapper[pool][router]` rather than `allowedSwapper[pool][user]`. This creates an irreconcilable Catch-22: allowlisting the router to enable router-mediated swaps simultaneously grants any unprivileged user the ability to bypass per-user access control.

## Finding Description
**Root cause:** `MetricOmmPool.swap()` passes `msg.sender` as the `sender` argument to `_beforeSwap`: [1](#0-0) 

`ExtensionCalling._beforeSwap` encodes that `sender` value and forwards it to the extension: [2](#0-1) 

`SwapAllowlistExtension.beforeSwap` then checks `allowedSwapper[msg.sender][sender]` — i.e., `allowedSwapper[pool][immediate_pool_caller]`: [3](#0-2) 

When `MetricOmmSimpleRouter.exactInputSingle` calls `pool.swap(...)`, the pool's `msg.sender` is the router. The router does not encode the actual user's identity anywhere in the call — `extensionData` is passed through verbatim from the user-supplied `params.extensionData` with no router-enforced user binding: [4](#0-3) 

The same pattern applies to `exactInput`, `exactOutputSingle`, and `exactOutput`.

**Catch-22:**

| Router allowlist state | Effect |
|---|---|
| Router **not** allowlisted | All router swaps revert for everyone, including legitimately allowlisted users |
| Router **allowlisted** | Any user bypasses the allowlist by routing through the router |

No configuration of `SwapAllowlistExtension` simultaneously allows legitimate router-mediated swaps and enforces per-user access control.

**Existing guards insufficient:** The `allowAllSwappers[pool]` escape hatch only disables the check entirely. There is no mechanism in the extension or the router to bind the real user's address to the swap call in a trustworthy way.

## Impact Explanation
A pool configured with `SwapAllowlistExtension` to restrict trading to a curated set of addresses cannot enforce that restriction when the canonical public router is used. Any unprivileged user can call `exactInputSingle` (or any other router entry point) and trade on an allowlist-gated pool, provided the pool admin has allowlisted the router — which is the only way to enable router-mediated swaps at all. This directly breaks the curation invariant and constitutes broken core pool functionality: the allowlist extension's access control is nullified for all router-mediated swaps.

## Likelihood Explanation
`MetricOmmSimpleRouter` is the canonical supported periphery path. Pool admins who want to allow router-mediated swaps must allowlist the router, at which point the bypass is available to any unprivileged user with no special setup, no privileged role, and no unusual preconditions. The attacker only needs to call the public router with a valid swap path.

## Recommendation
The extension must receive the **original user's address**, not the immediate pool caller. Concrete options:

1. **Router-enforced `extensionData` binding:** The router encodes `msg.sender` (the actual user) into `extensionData` before calling the pool. The extension reads and verifies it, and the pool enforces that only trusted routers can set this field (e.g., via a registry of known routers checked inside the extension or pool).
2. **Router-aware allowlist:** The extension checks whether `sender` is a known trusted router; if so, it decodes the real user from `extensionData` and checks `allowedSwapper[pool][realUser]`.
3. **`tx.origin` fallback:** Simple but incompatible with smart-contract wallets and generally discouraged.

## Proof of Concept
```
Setup:
- Deploy pool with SwapAllowlistExtension
- Pool admin: setAllowedToSwap(pool, alice, true)   // alice is allowlisted
- Pool admin: setAllowedToSwap(pool, router, true)  // required to allow any router swaps

Attack:
- bob (not allowlisted) calls MetricOmmSimpleRouter.exactInputSingle({pool: pool, ...})
- Router calls pool.swap(...); pool's msg.sender = router
- _beforeSwap(router, ...) → extension receives sender = router
- allowedSwapper[pool][router] == true → check passes
- bob's swap executes successfully despite not being allowlisted

Assert: bob received output tokens from an allowlist-gated pool.
```
