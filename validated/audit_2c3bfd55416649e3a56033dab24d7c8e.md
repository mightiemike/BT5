### Title
IBC NFT Receiver Is Not Checked Against Blocked Module Accounts, Permanently Locking NFTs - (File: x/nft-transfer/keeper/packet.go)

---

### Summary

`processReceivedPacket` in the IBC NFT-transfer keeper mints or unescrows NFTs directly to the `receiver` address decoded from the packet without verifying whether that address is a blocked module account. Because module accounts have no private key, any NFT delivered to one is permanently irrecoverable. The same omission exists in the direct `MsgMintNFT` and `MsgTransferNFT` message handlers in `x/nft/keeper/msg_server.go`.

---

### Finding Description

When an IBC NFT packet arrives, `processReceivedPacket` decodes the receiver:

```go
receiver, err := sdk.AccAddressFromBech32(data.Receiver)
if err != nil {
    return err
}
```

It then unconditionally mints or transfers the NFT to that address:

```go
// away-from-origin path
k.nftKeeper.MintNFT(ctx, voucherClassID, tokenID, "", data.TokenUris[i], "", escrowAddress, receiver)

// back-to-origin path
k.nftKeeper.TransferOwner(ctx, voucherClassID, tokenID, escrowAddress, receiver)
```

`sdk.AccAddressFromBech32` succeeds for any well-formed bech32 string, including the addresses of module accounts (e.g., `distribution`, `bonded_tokens_pool`, `gov`, `nonfungibletokentransfer`). No call to `BlockedAddrs()` or any equivalent check is made anywhere in the IBC NFT-transfer keeper — the `AccountKeeper` interface exposed to the keeper does not even include that method.

The same gap exists in the direct NFT message handlers: `msgServer.MintNFT` and `msgServer.TransferNFT` in `x/nft/keeper/msg_server.go` accept any valid bech32 recipient without a blocked-address check.

The `MsgTransfer.ValidateBasic` explicitly notes that the receiver format is not validated because the destination chain format is unknown, so no rejection happens at the sending side either.

---

### Impact Explanation

A module account address is a valid bech32 address. Once an NFT's owner field is set to a module account, the NFT is permanently stuck: module accounts have no private key and cannot sign `MsgTransferNFT` or `MsgBurnNFT` transactions. The NFT record remains in state indefinitely, owned by an address that can never act on it. This is the exact Cosmos SDK analog of the ERC721 `_mint`-to-non-receiver-contract scenario: the asset is irrecoverably locked.

**Impact: High** — NFT is permanently lost to the owner and unrecoverable without a chain upgrade.

---

### Likelihood Explanation

**Likelihood: Low-Medium.**

- For the IBC path: any user on a counterparty chain can specify any bech32 string as `receiver`. A user who mistakenly (or deliberately) supplies a module account address on the destination chain will cause the NFT to be locked. The sender on chain A has no way to know which addresses on chain B are module accounts.
- For the direct `MsgMintNFT` path: the denom creator can mint to any address, including a module account.
- For `MsgTransferNFT`: any NFT owner can transfer to a module account address.

The IBC path is the most realistic trigger because the receiver string is free-form and crosses chain boundaries where address semantics differ.

---

### Recommendation

1. Add a `BlockedAddresses() map[string]bool` method to the `AccountKeeper` interface in `x/nft-transfer/types/expected_keepers.go` and check it in `processReceivedPacket` before minting or transferring.
2. Apply the same check in `msgServer.MintNFT` and `msgServer.TransferNFT` in `x/nft/keeper/msg_server.go`.
3. If the receiver is a blocked address, return an error acknowledgement (IBC path) or reject the message (direct path) so the sender is refunded.

---

### Proof of Concept

**IBC path:**

1. On chain A, user owns NFT `(denomClass, token1)`.
2. User submits `MsgTransfer` with `receiver = sdk.GetModuleAddress("distribution").String()` (a valid bech32 address on chain B).
3. `MsgTransfer.ValidateBasic` passes — receiver is non-empty.
4. IBC packet is relayed to chain B.
5. `OnRecvPacket` → `processReceivedPacket`:
   - `sdk.AccAddressFromBech32(data.Receiver)` succeeds.
   - `k.nftKeeper.MintNFT(..., receiver)` succeeds — no blocked-address check.
6. NFT is now owned by the `distribution` module account on chain B.
7. No private key exists for that address; `MsgTransferNFT` and `MsgBurnNFT` cannot be signed. NFT is permanently stuck.

**Direct path:**

1. Denom creator submits `MsgMintNFT` with `recipient = authtypes.NewModuleAddress("gov").String()`.
2. `MsgMintNFT.ValidateBasic` passes — recipient is a valid bech32 address.
3. `msgServer.MintNFT` → `k.Keeper.MintNFT` → `MintNFTUnverified` — no blocked-address check.

func (k Keeper) processReceivedPacket(ctx sdk.Context, packet channeltypes.Packet,
	data types.NonFungibleTokenPacketData,
) error {
	receiver, err := sdk.AccAddressFromBech32(data.Receiver)
	if err != nil {
		return err
	}

	isAwayFromOrigin := types.IsAwayFromOrigin(packet.GetSourcePort(), packet.GetSourceChannel(), data.ClassId)

	// create the escrow address for creating denom and minting nft
	escrowAddress := types.GetEscrowAddress(packet.GetDestPort(), packet.GetDestChannel())

	if isAwayFromOrigin {
		// since SendPacket did not prefix the classID, we must prefix classID here
		classPrefix := types.GetClassPrefix(packet.GetDestPort(), packet.GetDestChannel())
		// NOTE: sourcePrefix contains the trailing "/"
		prefixedClassID := classPrefix + data.ClassId

		// construct the class trace from the full raw classID
		classTrace := types.ParseClassTrace(prefixedClassID)
		if !k.HasClassTrace(ctx, classTrace.Hash()) {
			k.SetClassTrace(ctx, classTrace)
		}

		voucherClassID := classTrace.IBCClassID()

		if !k.nftKeeper.HasDenomID(ctx, voucherClassID) {
			if err := k.nftKeeper.IssueDenom(ctx, voucherClassID, voucherClassID, "", data.ClassUri, escrowAddress); err != nil {
				return err
			}
		}
		sdkCtx := sdk.UnwrapSDKContext(ctx)
		sdkCtx.EventManager().EmitEvent(
			sdk.NewEvent(
				types.EventTypeClassTrace,
				sdk.NewAttribute(types.AttributeKeyTraceHash, classTrace.Hash().String()),
				sdk.NewAttribute(types.AttributeKeyClassID, voucherClassID),
			),
		)

		for i, tokenID := range data.TokenIds {
			if err := k.nftKeeper.MintNFT(ctx, voucherClassID, tokenID, "", data.TokenUris[i], "", escrowAddress, receiver); err != nil {
				return err
			}
		}

// ValidateBasic performs a basic check of the MsgTransfer fields.
// NOTE: timeout height or timestamp values can be 0 to disable the timeout.
// NOTE: The recipient addresses format is not validated as the format defined by
// the chain is not known to IBC.
func (msg MsgTransfer) ValidateBasic() error {
	if err := host.PortIdentifierValidator(msg.SourcePort); err != nil {
		return newsdkerrors.Wrap(err, "invalid source port ID")
	}
	if msg.SourcePort != PortID {
		return newsdkerrors.Wrapf(ErrInvalidSourcePort, "source port must be %q", PortID)
	}
	if err := host.ChannelIdentifierValidator(msg.SourceChannel); err != nil {
		return newsdkerrors.Wrap(err, "invalid source channel ID")
	}

	if strings.TrimSpace(msg.ClassId) == "" {
		return newsdkerrors.Wrap(ErrInvalidClassID, "classId cannot be blank")
	}

	if len(msg.TokenIds) == 0 {
		return newsdkerrors.Wrap(ErrInvalidTokenID, "tokenId cannot be blank")
	}

	for _, tokenID := range msg.TokenIds {
		if strings.TrimSpace(tokenID) == "" {
			return newsdkerrors.Wrap(ErrInvalidTokenID, "tokenId cannot be blank")
		}
	}

	// NOTE: sender format must be validated as it is required by the GetSigners function.
	_, err := sdk.AccAddressFromBech32(msg.Sender)
	if err != nil {
		return newsdkerrors.Wrapf(sdkerrors.ErrInvalidAddress, "string could not be parsed as address: %v", err)
	}
	if strings.TrimSpace(msg.Receiver) == "" {
		return newsdkerrors.Wrap(sdkerrors.ErrInvalidAddress, "missing recipient address")
	}
	return nil

func (m msgServer) MintNFT(goCtx context.Context, msg *types.MsgMintNFT) (*types.MsgMintNFTResponse, error) {
	recipient, err := sdk.AccAddressFromBech32(msg.Recipient)
	if err != nil {
		return nil, err
	}

	sender, err := sdk.AccAddressFromBech32(msg.Sender)
	if err != nil {
		return nil, err
	}

	ctx := sdk.UnwrapSDKContext(goCtx)
	if err := m.Keeper.MintNFT(ctx, msg.DenomId, msg.Id,
		msg.Name,
		msg.URI,
		msg.Data,
		sender,
		recipient,
	); err != nil {
		return nil, err
	}

	ctx.EventManager().EmitEvents(sdk.Events{
		sdk.NewEvent(
			types.EventTypeMintNFT,
			sdk.NewAttribute(types.AttributeKeyTokenID, msg.Id),
			sdk.NewAttribute(types.AttributeKeyDenomID, msg.DenomId),
			sdk.NewAttribute(types.AttributeKeyTokenURI, msg.URI),
			sdk.NewAttribute(types.AttributeKeyRecipient, msg.Recipient),
		),
		sdk.NewEvent(
			sdk.EventTypeMessage,
			sdk.NewAttribute(sdk.AttributeKeyModule, types.AttributeValueCategory),
			sdk.NewAttribute(sdk.AttributeKeySender, msg.Sender),
		),
	})

	return &types.MsgMintNFTResponse{}, nil
}
