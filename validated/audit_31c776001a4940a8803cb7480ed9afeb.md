### Title
Storage-Deposit NEAR Permanently Locked After Garbage Collection — (`contract/src/lib.rs`)

### Summary
Every time `run_mainchain_gc` removes old block headers it frees on-chain storage, which causes NEAR Protocol to credit the freed storage-stake back to the contract's liquid balance. Because no withdrawal function exists anywhere in the contract, those NEAR tokens accumulate in the contract account and can never be recovered by any party.

### Finding Description

`submit_blocks` is `#[payable]` and requires callers to cover the net storage cost of the headers they add:

```
let initial_storage = env::storage_usage();          // snapshot before
for header in headers { self.submit_block_header(…); }
self.run_mainchain_gc(num_of_headers);               // may delete old headers
let diff_storage_usage = env::storage_usage()
    .saturating_sub(initial_storage);               // NET delta only
let required_deposit = env::storage_byte_cost()
    .saturating_mul(diff_storage_usage.into());
``` [1](#0-0) 

The `diff_storage_usage` calculation captures only the **net** change within the current call. When GC removes blocks that were paid for by **previous** relayer calls, the freed storage-stake (NEAR tokens) is returned to the contract's liquid balance by the NEAR runtime — but those tokens belong to prior depositors and are now inside the contract with no path out.

`run_mainchain_gc` is designed to run continuously: once `amount_of_headers_we_store > gc_threshold` it removes up to `batch_size` headers per call, deleting entries from `mainchain_height_to_header`, `mainchain_header_to_height`, and `headers_pool`. [2](#0-1) 

The contract exposes no `withdraw`, `transfer`, or treasury function. The full public API is: `init`, `submit_blocks`, five read-only getters, `verify_transaction_inclusion{,_v2}`, `run_mainchain_gc`, and `migrate`. None of them move the contract's liquid balance to any external account. [3](#0-2) 

### Impact Explanation

Every production deployment accumulates locked NEAR monotonically. With the recommended `gc_threshold = 52704` (≈ one year of Bitcoin blocks) and a storage cost of ~10 milliNEAR per block, a single year of operation frees roughly **527 NEAR** of previously-deposited storage stake into the contract's unreachable balance. The amount grows without bound as the chain continues. Relayers who funded storage for GC'd blocks suffer a permanent, unrecoverable financial loss.

### Likelihood Explanation

GC is not optional — it is called automatically inside every `submit_blocks` invocation once the stored chain exceeds `gc_threshold`. [4](#0-3) 

Every production deployment will trigger this path continuously. The locked-NEAR accumulation is therefore **certain** for any live deployment.

### Recommendation

Add a privileged withdrawal function (callable only by `Role::DAO`) that transfers the contract's surplus liquid balance to a designated treasury account:

```rust
#[access_control_any(roles(Role::DAO))]
pub fn withdraw_surplus(&mut self, recipient: AccountId, amount: NearToken) {
    Promise::new(recipient).transfer(amount);
}
```

Alternatively, track each relayer's cumulative storage contribution and refund their proportional share when GC frees the corresponding storage.

### Proof of Concept

1. Relayer A calls `submit_blocks([h1, h2, …, h85])` with a 500 mN deposit. Storage grows; 500 mN is consumed as storage stake.
2. Chain grows past `gc_threshold`. Relayer B calls `submit_blocks([h86])`. Inside the call, `run_mainchain_gc(1)` removes `h1`, freeing its storage. NEAR runtime credits the freed stake (~5.9 mN) to the contract's liquid balance.
3. Relayer B's `diff_storage_usage` reflects only the net change of the current call (one block added, one removed ≈ 0 net), so `required_deposit ≈ 0` and Relayer B's full deposit is refunded.
4. The ~5.9 mN freed from Relayer A's original deposit now sits in the contract's liquid balance. No function exists to move it. Repeat for every subsequent GC cycle. [5](#0-4)

        let initial_storage = env::storage_usage();
        let num_of_headers = headers.len().try_into().unwrap();

            self.submit_block_header(header, self.skip_pow_verification);
        }

        self.run_mainchain_gc(num_of_headers);
        let diff_storage_usage = env::storage_usage().saturating_sub(initial_storage);

        require!(
            amount >= required_deposit,
            format!("Required deposit {}", required_deposit)
        );

        let refund = amount.saturating_sub(required_deposit);
        if refund > NearToken::from_near(0) {
            Promise::new(env::predecessor_account_id())
                .transfer(refund)
                .into()
        } else {
            PromiseOrValue::Value(())
        }
