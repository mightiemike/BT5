### Title
`PapyrusStateReader::get_compiled_class_hash` reads stale v1 hash from declaration-block state diff, causing `should_migrate` to perpetually re-trigger for already-migrated classes in RPC fee estimation — (File: `crates/apollo_rpc_execution/src/state_reader.rs`)

---

### Summary

`PapyrusStateReader::get_compiled_class_hash` resolves the compiled class hash by looking up the state diff at the block where the class was **declared**, not the current chain state. After a class is migrated from v1 (Poseidon-based) to v2 (Blake-based) compiled class hash, the declaration-block state diff permanently retains the v1 hash. The v2 hash is stored only in the migration-block state diff and in the separate `stateless_compiled_class_hash_v2` table. Because `should_migrate` compares `get_compiled_class_hash` (→ stale v1) against `get_compiled_class_hash_v2` (→ current v2), it perpetually signals migration-needed for already-migrated classes. The RPC fee-estimation and simulation path uses this reader and calls `get_tx_weights` → `CasmHashMigrationData::from_state` → `should_migrate`, so every `starknet_estimateFee` / `starknet_simulateTransactions` call that touches a migrated class returns an inflated, authoritative-looking wrong fee.

---

### Finding Description

**Step 1 – Declaration stores v1 hash in the declaration-block state diff.**

When a Cairo 1 class is declared, `class_hash_to_compiled_class_hash` in the state diff at block `B_decl` is written with the Poseidon-based (v1) compiled class hash.

**Step 2 – Migration updates the live state but not the declaration-block state diff.**

`finalize_block` calls `set_compiled_class_hash_migration`, which calls `set_compiled_class_hash` on the `CachedState`. This writes the v2 hash into the *current* block's state diff (`B_mig`). The declaration-block state diff at `B_decl` is immutable and still holds the v1 hash. [1](#0-0) 

**Step 3 – `PapyrusStateReader::get_compiled_class_hash` reads from the declaration block.**

The function calls `get_class_definition_block_number` to find `B_decl`, then reads `state_diff.class_hash_to_compiled_class_hash` at that block. It never consults the migration-block state diff or the current state. [2](#0-1) 

**Step 4 – `get_compiled_class_hash_v2` reads the correct v2 hash.**

The sibling method reads from the class manager or the `stateless_compiled_class_hash_v2` storage table, always returning the Blake-based v2 hash. [3](#0-2) 

**Step 5 – `should_migrate` sees a permanent mismatch.**

`should_migrate` compares `get_compiled_class_hash` (v1, stale) with `get_compiled_class_hash_v2` (v2, current). They are never equal for a migrated class, so the function always returns `Some((class_hash, (v2, v1)))`. [4](#0-3) 

**Step 6 – `get_tx_weights` calls `should_migrate` for every executed class hash.**

`CasmHashMigrationData::from_state` iterates over `executed_class_hashes` and calls `should_migrate` for each one (when `enable_casm_hash_migration` is true). The migration gas delta is added to both `sierra_gas` and `proving_gas` bouncer weights. [5](#0-4) 

**Step 7 – The RPC simulation path uses `PapyrusStateReader` and calls `get_tx_weights`.**

`simulate_transactions` (called by `starknet_estimateFee` / `starknet_simulateTransactions`) passes a `StorageReader` into `execute_transactions`, which constructs a `PapyrusStateReader`-backed `CachedState`. The blockifier's `Bouncer::try_update` calls `get_tx_weights` on every executed transaction. [6](#0-5) [7](#0-6) 

---

### Impact Explanation

Every `starknet_estimateFee` or `starknet_simulateTransactions` call that executes a Cairo 1 class which has already been migrated from v1 to v2 compiled class hash will include spurious migration gas in the returned fee estimate. The returned value is authoritative-looking (it comes from the node's own RPC endpoint) but is wrong: it overstates the actual on-chain cost. Users who rely on the estimate to set resource bounds will over-pay; wallets or dApps that use the estimate as an upper bound may reject transactions that would actually succeed.

**Matching impact**: *High — RPC execution, fee estimation, tracing, simulation, or pending view returns an authoritative-looking wrong value.*

---

### Likelihood Explanation

The `enable_casm_hash_migration` versioned-constant flag is present in the blockifier's versioned-constants JSON files starting from `V0_13_6` and is set to `true` in the latest constants. [8](#0-7) 

Once any block is finalized with migration enabled, every previously-declared Cairo 1 class that was executed in that block is migrated. From that point on, every RPC fee estimate for a transaction touching those classes is inflated. The trigger is unprivileged: any user can submit a transaction that executes a migrated class and then call `starknet_estimateFee`.

---

### Recommendation

Fix `PapyrusStateReader::get_compiled_class_hash` to read the compiled class hash from the **current** state (i.e., the latest committed state diff that contains an entry for `class_hash`), not from the declaration-block state diff. The correct approach is to scan state diffs in reverse order from the current block, or to maintain a separate index of the latest compiled class hash per class hash (analogous to `get_class_definition_block_number` but tracking the most recent migration). Alternatively, mirror the approach used by `ApolloReader::get_compiled_class_hash`, which calls `get_compiled_class_hash_at(state_number, &class_hash)` against the current state number. [9](#0-8) 

---

### Proof of Concept

1. Declare a Cairo 1 contract at block `B_decl`; its v1 compiled class hash is stored in the state diff at `B_decl`.
2. Enable `enable_casm_hash_migration = true` (present in versioned constants ≥ V0_13_6).
3. Execute any transaction that calls the contract; `finalize_block` migrates the class, writing the v2 hash into the state diff at block `B_mig`.
4. Call `starknet_estimateFee` for a new transaction that invokes the same contract.
5. Internally, `PapyrusStateReader::get_compiled_class_hash` returns the v1 hash (from `B_decl`'s state diff); `get_compiled_class_hash_v2` returns the v2 hash. `should_migrate` returns `Some`, and `get_tx_weights` adds migration gas to the estimate.
6. The returned fee is inflated by the migration gas cost, even though the class was already migrated at step 3 and no migration will occur on-chain.

    fn get_compiled_class_hash(&self, class_hash: ClassHash) -> StateResult<CompiledClassHash> {
        if let Some(pending_data) = &self.maybe_pending_data {
            for DeclaredClassHashEntry { class_hash: other_class_hash, compiled_class_hash } in
                &pending_data.declared_classes
            {
                if class_hash == *other_class_hash {
                    return Ok(*compiled_class_hash);
                }
            }
        }

        let maybe_block_number = self
            .storage_reader
            .begin_ro_txn()
            .map_err(storage_err_to_state_err)?
            .get_state_reader()
            .map_err(storage_err_to_state_err)?
            .get_class_definition_block_number(&class_hash)
            .map_err(storage_err_to_state_err)?;

        // Cairo 0 classes (and undeclared classes) do not have a compiled class hash.
        // According to the trait, return the default value.
        let Some(block_number) = maybe_block_number else {
            return Ok(CompiledClassHash::default());
        };

        let state_diff = self
            .storage_reader
            .begin_ro_txn()
            .map_err(storage_err_to_state_err)?
            .get_state_diff(block_number)
            .map_err(storage_err_to_state_err)?
            .ok_or(StateError::StateReadError(format!(
                "Inner storage error. Missing state diff at block {block_number}."
            )))?;

        let compiled_class_hash = state_diff
            .class_hash_to_compiled_class_hash
            .get(&class_hash)
            .ok_or(StateError::StateReadError(format!(
                "Inner storage error. Missing class declaration at block {block_number}, class \
                 {class_hash}."
            )))?;

        Ok(*compiled_class_hash)
    }

pub fn should_migrate(
    state_reader: &impl StateReader,
    class_hash: ClassHash,
) -> StateResult<Option<(ClassHash, CompiledClassHashV2ToV1)>> {
    let state_compiled_class_hash = state_reader.get_compiled_class_hash(class_hash)?;
    match state_compiled_class_hash {
        // Class hash does not exist in the state, or is a Cairo 0 class.
        CompiledClassHash(hash) if hash == StarkHash::ZERO => Ok(None),
        state_compiled_class_hash => {
            let compiled_class_hash_v2 = state_reader.get_compiled_class_hash_v2(
                class_hash,
                &state_reader.get_compiled_class(class_hash)?,
            )?;
            // If the state compiled class hash is compiled class hash v2, the class should not
            // migrate.
            if state_compiled_class_hash == compiled_class_hash_v2 {
                return Ok(None);
            }
            Ok(Some((class_hash, (compiled_class_hash_v2, state_compiled_class_hash))))
        }
    }
