### Title
`SwapAllowlistExtension` Swap Guard Bypassed via `MetricOmmSimpleRouter` â€” Router Identity Replaces Real Swapper Identity - (File: `metric-periphery/contracts/extensions/SwapAllowlistExtension.sol`)

---

### Summary

`SwapAllowlistExtension.beforeSwap` gates swaps by checking the `sender` argument against a per-pool allowlist. The pool passes `msg.sender` (the immediate caller) as `sender`. When a user routes through `MetricOmmSimpleRouter`, the pool's `msg.sender` is the router contract, not the real user. The extension therefore checks whether the **router** is allowlisted, not the actual swapper. Any user can bypass a curated pool's swap allowlist by calling the public router.

---

### Finding Description

`SwapAllowlistExtension.beforeSwap` enforces:

```solidity
// SwapAllowlistExtension.sol line 37
if (!allowAllSwappers[msg.sender] && !allowedSwapper[msg.sender][sender]) {
    revert IMetricOmmPoolActions.NotAllowedToSwap();
}
```

Here `msg.sender` is the pool (correct â€” the pool calls the extension), and `sender` is the first argument forwarded by the pool.

`MetricOmmPool.swap` passes its own `msg.sender` as `sender`:

```solidity
// MetricOmmPool.sol line 230-240
_beforeSwap(
    msg.sender,   // <-- whoever called pool.swap()
    recipient,
    ...
);
```

`MetricOmmSimpleRouter.exactInputSingle` calls `pool.swap()` directly:

```solidity
// MetricOmmSimpleRouter.sol line 72-80
(int128 amount0Delta, int128 amount1Delta) = IMetricOmmPoolActions(params.pool)
    .swap(
        params.recipient,
        params.zeroForOne,
        ...
    );
```

The call chain is:

```
user â†’ MetricOmmSimpleRouter.exactInputSingle()
     â†’ pool.swap()          [msg.sender = router]
     â†’ _beforeSwap(sender = router, ...)
     â†’ SwapAllowlistExtension.beforeSwap(sender = router)
     â†’ checks allowedSwapper[pool][router]
```

The extension never sees the real user's address. It only sees the router. If the pool admin allowlists the router (required for any router-mediated swap to succeed on the curated pool), every user on the network can swap through the router and pass the guard. The allowlist is structurally inert for router-mediated paths.

The same bypass applies to `exactInput`, `exactOutputSingle`, and `exactOutput` â€” all router entry points call `pool.swap()` directly with `msg.sender = router`.

---

### Impact Explanation

A pool admin deploys a curated pool with `SwapAllowlistExtension` to restrict swapping to a specific set of addresses (e.g., whitelisted market makers or institutional counterparties). The admin allowlists those addresses and also allowlists the router so that the approved parties can use the standard periphery. Any unpermissioned user can then call `MetricOmmSimpleRouter.exactInputSingle` targeting the curated pool and execute swaps at oracle-derived prices, draining LP assets at will. The allowlist provides zero protection on the router path. This is a direct loss of LP principal and a complete curation failure.

---

### Likelihood Explanation

`MetricOmmSimpleRouter` is the standard, publicly deployed periphery entry point. Any pool that uses `SwapAllowlistExtension` and also needs to support router-mediated swaps for its approved parties must allowlist the router, which simultaneously opens the gate to all users. The bypass requires no special privileges, no flash loans, and no unusual token behavior â€” only a standard router call. Likelihood is high.

---

### Recommendation

**Short term**: Change `SwapAllowlistExtension.beforeSwap` to check the `recipient` field or require the real originator to be passed in `extensionData` and verified against a signature or trusted forwarder. Alternatively, remove router allowlisting from the design and require approved parties to call `pool.swap()` directly.

**Long term**: Redesign the `sender` binding in the extension interface so that the pool propagates the original EOA (e.g., via EIP-2771 trusted forwarder pattern or by having the router attest the real caller in `extensionData`). Document clearly that `sender` in `beforeSwap` is the immediate pool caller, not the economic originator, so extension authors do not build identity guards on it.
