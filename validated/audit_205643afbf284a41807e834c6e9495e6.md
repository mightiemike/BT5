Audit Report

## Title
`SwapAllowlistExtension` Checks Router Address Instead of Actual Swapper, Allowing Any User to Bypass Per-User Swap Allowlist via `MetricOmmSimpleRouter` - (File: `metric-periphery/contracts/extensions/SwapAllowlistExtension.sol`)

## Summary
`SwapAllowlistExtension.beforeSwap` gates swaps by checking the `sender` argument, which `MetricOmmPool.swap` sets to `msg.sender` — the immediate caller of `pool.swap()`. When `MetricOmmSimpleRouter` is used, `msg.sender` at the pool is the router contract, not the end user. If the pool admin allowlists the router (the only way to let individually allowlisted users trade through it), every unprivileged user can bypass the per-user allowlist by routing through the same router contract.

## Finding Description
`MetricOmmPool.swap` passes `msg.sender` as the first argument to `_beforeSwap`: [1](#0-0) 

`SwapAllowlistExtension.beforeSwap` then checks that argument (`sender`) against `allowedSwapper[msg.sender][sender]`, where `msg.sender` is the pool and `sender` is the immediate caller of `pool.swap()`: [2](#0-1) 

`MetricOmmSimpleRouter.exactInputSingle` calls `pool.swap()` directly without encoding or forwarding the original `msg.sender` into `extensionData`: [3](#0-2) 

The same pattern applies to `exactInput`, `exactOutputSingle`, and `exactOutput`. [4](#0-3) 

The root cause: the extension evaluates `allowedSwapper[pool][router]` instead of `allowedSwapper[pool][end_user]`. Once the admin allowlists the router (required for any allowlisted user to trade through it), the check passes for every caller of the router regardless of whether they are individually allowlisted.

## Impact Explanation
A pool configured with `SwapAllowlistExtension` to restrict trading to specific counterparties (e.g., a KYC-gated pool, a private institutional pool, or a curated market-maker set) loses that restriction entirely once the router is allowlisted. Any unprivileged user can call `MetricOmmSimpleRouter.exactInputSingle` and execute swaps against LP funds. In an oracle-anchored pool, unauthorized traders can exploit oracle latency or manipulation windows, causing direct loss of LP principal. This is a **High** severity direct loss of user/LP principal.

## Likelihood Explanation
The bypass requires the pool admin to allowlist the router address via `setAllowedToSwap(pool, router, true)`. This is a natural and expected configuration step: without it, even individually allowlisted users cannot trade through the router (their swaps revert because `allowedSwapper[pool][router]` is false). An admin who wants to support router-mediated swaps for their allowlisted users has no other option. The configuration that enables the intended use case simultaneously enables the bypass for all users. The bypass is repeatable by any unprivileged user at any time after the router is allowlisted.

## Recommendation
The extension must gate the actual end-user, not the immediate caller of `pool.swap()`. Two approaches:

1. **Extension-data forwarding**: Require the router to ABI-encode the original `msg.sender` into `extensionData`; have the extension decode and check that address. The pool already threads `extensionData` through to every hook call unchanged.
2. **Trusted-relay model**: Distinguish between "router is a trusted relay" and "all users are allowed". The extension would check that `sender` (the router) is a trusted relay and that the actual user (decoded from `extensionData`) is individually allowlisted.

## Proof of Concept
```
Setup:
  pool = deploy MetricOmmPool with SwapAllowlistExtension
  admin calls setAllowedToSwap(pool, alice, true)   // alice is allowlisted
  admin calls setAllowedToSwap(pool, router, true)  // router allowlisted so alice can use it

Attack:
  bob (not allowlisted) calls:
    router.exactInputSingle({pool: pool, zeroForOne: true, amountIn: X, ...})

  Execution path:
    router → pool.swap(recipient, ...)
    pool passes msg.sender=router as `sender` to _beforeSwap → extension
    extension checks: allowedSwapper[pool][router] == true  ✓
    swap executes — bob drains LP funds despite never being allowlisted

Direct pool call by bob correctly reverts:
  bob → pool.swap(...)
  extension checks: allowedSwapper[pool][bob] == false → NotAllowedToSwap ✓
```
