### Title
Gateway `StatelessTransactionValidatorConfig::min_gas_price` Is a Stale Copy Diverging from `apollo_versioned_constants::VersionedConstants::min_gas_price` — (`crates/apollo_gateway_config/src/config.rs`)

### Summary

The gateway's stateless admission check enforces a minimum L2 gas price using a **separate, independently-configured** field (`StatelessTransactionValidatorConfig::min_gas_price`, default 8 Gwei) that is never read from — and never updated by — the canonical `apollo_versioned_constants::VersionedConstants::min_gas_price` used by the fee market. When the two values diverge (e.g., after a protocol upgrade bumps the versioned-constants minimum), the gateway either admits transactions that will never be included in any block, or rejects transactions that the fee market would accept.

### Finding Description

`StatelessTransactionValidatorConfig` carries its own `min_gas_price: u128` field, hardcoded to `8_000_000_000` (8 Gwei) in both the `Default` impl and `config_schema.json`. The code itself acknowledges the problem with a TODO:

```rust
// TODO(AlonH): Remove the `min_gas_price` field from this struct and use the one from the
// versioned constants.
pub min_gas_price: u128,
``` [1](#0-0) 

The stateless validator enforces this value at admission time:

```rust
if resource_bounds.l2_gas.max_price_per_unit.0 < self.config.min_gas_price {
    return Err(StatelessTransactionValidatorError::MaxGasPriceTooLow { ... });
}
``` [2](#0-1) 

The fee market, however, reads its minimum from the **orchestrator's** `VersionedConstants`, which is a completely separate versioned struct in `crates/apollo_versioned_constants`:

```rust
let fallback_min_gas_price = VersionedConstants::latest_constants().min_gas_price;
``` [3](#0-2) 

```rust
let gas_target = VersionedConstants::latest_constants().gas_target;
``` [4](#0-3) 

The orchestrator's `VersionedConstants` is a distinct type from the blockifier's `VersionedConstants`, versioned from V0_14_0 through V0_14_4, with its own `min_gas_price: GasPrice` field: [5](#0-4) 

There is no code path that reads `apollo_versioned_constants::VersionedConstants::min_gas_price` and writes it into `StatelessTransactionValidatorConfig::min_gas_price`. The gateway's value is set once at node startup from the static config file and never reconciled with the versioned constants.

The config schema confirms the gateway's value is a plain literal, not a pointer to any shared constant:

```json
"gateway_config.static_config.stateless_tx_validator_config.min_gas_price": {
    "description": "Minimum gas price for transactions.",
    "privacy": "Public",
    "value": 8000000000
}
``` [6](#0-5) 

### Impact Explanation

**Divergence case 1 — VC minimum raised above gateway minimum (e.g., after a protocol upgrade):**
Transactions with `l2_gas.max_price_per_unit` in the range `[gateway_min, vc_min)` pass the gateway's stateless check and enter the mempool. The fee market will never produce a block gas price below `vc_min`, so these transactions can never be included. The gateway is admitting transactions that are permanently unexecutable under the current protocol, wasting mempool capacity and enabling a low-cost admission-layer DoS.

**Divergence case 2 — VC minimum lowered below gateway minimum:**
Transactions with `max_price_per_unit` in `[vc_min, gateway_min)` are rejected by the gateway even though the fee market would accept and include them. Valid user transactions are silently censored at the RPC/admission layer.

Both cases match the allowed impact: **High — Mempool/gateway/RPC admission accepts invalid transactions or rejects valid transactions before sequencing.**

### Likelihood Explanation

The divergence is latent today and becomes active whenever the orchestrator's versioned constants are updated to a new Starknet version that changes `min_gas_price`. The codebase already ships five versioned-constants files (V0_14_0 through V0_14_4), demonstrating that these values do change across protocol versions. The TODO comment confirms the developers are aware the field should be removed and replaced with the versioned-constants value, meaning the fix has not yet been applied. Any operator who upgrades the node binary (picking up a new versioned-constants version) without simultaneously updating the gateway static config will trigger the divergence.

### Recommendation

Remove `min_gas_price` from `StatelessTransactionValidatorConfig` as the TODO instructs. In `StatelessTransactionValidator::validate_resource_bounds`, replace `self.config.min_gas_price` with a live read of `apollo_versioned_constants::VersionedConstants::latest_constants().min_gas_price.0`. This ensures the gateway's admission threshold is always identical to the value the fee market enforces, eliminating the possibility of divergence across protocol upgrades.

### Proof of Concept

1. Deploy the node with the default config (`gateway min_gas_price = 8_000_000_000`).
2. Upgrade the orchestrator versioned constants so that `VersionedConstants::latest_constants().min_gas_price` becomes `GasPrice(10_000_000_000)` (as would happen in a protocol upgrade).
3. Submit an invoke V3 transaction with `l2_gas.max_price_per_unit = GasPrice(9_000_000_000)`.
4. **Observed**: The gateway's `validate_resource_bounds` passes (9 Gwei ≥ 8 Gwei), the transaction enters the mempool.
5. **Observed**: The fee market sets the block gas price at or above 10 Gwei; the transaction's max price (9 Gwei) is below the block price, so it is never selected for inclusion.
6. **Expected**: The gateway should have rejected the transaction at step 4 with `MaxGasPriceTooLow`, matching the fee market's actual minimum.

The inverse (gateway min raised above VC min) causes step 4 to reject a transaction that the fee market would include, demonstrating the censorship path.
