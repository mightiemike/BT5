### Title
Storage Deposit NEAR Permanently Locked After Garbage Collection — (`contract/src/lib.rs`)

### Summary
`submit_blocks` collects NEAR storage deposits from relayers to cover on-chain storage costs. When `run_mainchain_gc` later deletes those block headers, the freed storage staking is released back to the contract's own account balance, but no withdrawal function exists. The released NEAR is permanently locked in the contract with no recovery path for the relayers who paid it.

### Finding Description
`submit_blocks` is `#[payable]` and enforces that the caller attaches enough NEAR to cover `storage_byte_cost × diff_storage_usage` for the newly stored headers. Any excess above the exact required deposit is refunded immediately to `predecessor_account_id`. [1](#0-0) 

However, `run_mainchain_gc` is called automatically inside every `submit_blocks` invocation (and is also publicly callable by anyone), and it deletes old mainchain block headers from `headers_pool`, `mainchain_height_to_header`, and `mainchain_header_to_height`: [2](#0-1) 

In NEAR Protocol, deleting storage entries releases the previously staked NEAR back to the contract's own account balance. That released NEAR is now "free" balance sitting in the contract account. Searching the entire public API of `BtcLightClient` — `init`, `submit_blocks`, `run_mainchain_gc`, `verify_transaction_inclusion`, `verify_transaction_inclusion_v2`, `migrate`, and all view functions — reveals **no function that transfers NEAR out of the contract**. The released storage-staking NEAR accumulates in the contract's balance indefinitely with no withdrawal path. [3](#0-2) 

### Impact Explanation
Every relayer who calls `submit_blocks` with a deposit for blocks that are subsequently garbage-collected permanently loses that NEAR. With the recommended `gc_threshold = 52704` (~1 year of Bitcoin blocks), GC runs continuously in steady-state operation. The cumulative locked NEAR grows proportionally to the number of headers ever submitted and GC'd. There is no admin escape hatch — even the DAO role has no `withdraw` or `transfer` method available. [4](#0-3) 

### Likelihood Explanation
GC is triggered automatically on every `submit_blocks` call once the mainchain exceeds `gc_threshold`. In production, this is a continuous, normal-operation event. Any relayer submitting blocks after the threshold is reached will have their deposit partially or fully locked. The condition is not adversarial — it is the designed steady-state behavior of the contract. [5](#0-4) 

### Recommendation
Add a privileged withdrawal function (e.g., callable by `Role::DAO`) that transfers the contract's excess balance (i.e., `env::account_balance() - env::storage_byte_cost() * env::storage_usage()`) to a designated recipient. Alternatively, track per-relayer deposits and refund the proportional released NEAR to each relayer when GC removes their blocks.

### Proof of Concept
1. Relayer calls `submit_blocks(headers)` with deposit `D = storage_byte_cost × N_bytes` for `N` new headers. Excess is refunded; exactly `D` is retained by the contract.
2. The mainchain grows past `gc_threshold`. On the next `submit_blocks` call, `run_mainchain_gc(num_of_headers)` deletes the oldest `batch_size` headers, freeing `N_bytes` of storage.
3. NEAR Protocol releases the `D` yoctoNEAR of storage staking back to the contract's account balance.
4. No function in `BtcLightClient` can move that `D` out of the contract. The relayer's NEAR is permanently locked. [6](#0-5)

    /// # Panics
    /// Cannot parse headers len as u64
    #[payable]
    #[pause]
    #[trusted_relayer]
    pub fn submit_blocks(
        &mut self,
        #[serializer(borsh)] headers: Vec<BlockHeader>,
    ) -> PromiseOrValue<()> {
        let amount = env::attached_deposit();
        let initial_storage = env::storage_usage();
        let num_of_headers = headers.len().try_into().unwrap();

        for header in headers {
            self.submit_block_header(header, self.skip_pow_verification);
        }

        self.run_mainchain_gc(num_of_headers);
        let diff_storage_usage = env::storage_usage().saturating_sub(initial_storage);
        let required_deposit = env::storage_byte_cost().saturating_mul(diff_storage_usage.into());

        require!(
            amount >= required_deposit,
            format!("Required deposit {}", required_deposit)
        );

        let refund = amount.saturating_sub(required_deposit);
        if refund > NearToken::from_near(0) {
            Promise::new(env::predecessor_account_id())
                .transfer(refund)
                .into()
        } else {
            PromiseOrValue::Value(())
        }
    }

    pub fn run_mainchain_gc(&mut self, batch_size: u64) {
        let initial_blockheader = self
            .headers_pool
            .get(&self.mainchain_initial_blockhash)
            .unwrap_or_else(|| env::panic_str(ERR_KEY_NOT_EXIST));

        let tip_blockheader = self
            .headers_pool
            .get(&self.mainchain_tip_blockhash)
            .unwrap_or_else(|| env::panic_str(ERR_KEY_NOT_EXIST));

        let amount_of_headers_we_store =
            tip_blockheader.block_height - initial_blockheader.block_height + 1;

        if amount_of_headers_we_store > self.gc_threshold {
            let total_amount_to_remove = amount_of_headers_we_store - self.gc_threshold;
            let selected_amount_to_remove = std::cmp::min(total_amount_to_remove, batch_size);

            let start_removal_height = initial_blockheader.block_height;
            let end_removal_height = initial_blockheader.block_height + selected_amount_to_remove;
            env::log_str(&format!(
                "Num of blocks to remove {selected_amount_to_remove}"
            ));

            for height in start_removal_height..end_removal_height {
                let blockhash = &self
                    .mainchain_height_to_header
                    .get(&height)
                    .unwrap_or_else(|| env::panic_str(ERR_KEY_NOT_EXIST));

                self.remove_block_header(blockhash);
                self.mainchain_height_to_header.remove(&height);
            }

            self.mainchain_initial_blockhash = self
                .mainchain_height_to_header
                .get(&end_removal_height)
                .unwrap_or_else(|| env::panic_str(ERR_KEY_NOT_EXIST));
        }
    }
