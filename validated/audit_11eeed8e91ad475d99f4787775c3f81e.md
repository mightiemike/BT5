### Title
Stale `LastBonusAccrual` After `ClearPosition` Allows Bonus Extraction for Post-Exit Gap Period - (File: `x/tieredrewards/keeper/bonus_rewards.go`, `x/tieredrewards/keeper/msg_server.go`)

---

### Summary

When a position owner calls `MsgClearPosition` after the exit commitment has elapsed, `applyBonusAccrualCheckpoint` sets `LastBonusAccrual = ExitUnlockAt` (a past timestamp), then `ClearExit` zeroes out `ExitUnlockAt`. On the next `MsgClaimTierRewards`, the bonus engine computes a segment from the stale `LastBonusAccrual` (= old `ExitUnlockAt`) to the current block time with no accrual cap, paying bonus for the entire gap period during which no bonus should have accrued. The module invariant "no bonus after exit commitment elapses" is violated.

---

### Finding Description

**Root cause — `applyBonusAccrualCheckpoint` vs. `ClearExit` desynchronization**

`applyBonusAccrualCheckpoint` is called at the end of every `processEventsAndClaimBonus` invocation:

```go
// x/tieredrewards/keeper/bonus_rewards.go
func applyBonusAccrualCheckpoint(pos *types.Position, blockTime time.Time) {
    accrualEnd := blockTime
    if pos.CompletedExitLockDuration(blockTime) {
        accrualEnd = pos.ExitUnlockAt   // ← set to past timestamp, NOT blockTime
    }
    pos.UpdateLastBonusAccrual(accrualEnd)
}
```

When the exit commitment has elapsed (`CompletedExitLockDuration = true`), `LastBonusAccrual` is set to `ExitUnlockAt` — a timestamp in the past — rather than `blockTime`.

`MsgClearPosition` then calls `ClearExit(blockTime)`, which zeroes `ExitTriggeredAt` and `ExitUnlockAt` but does **not** advance `LastBonusAccrual` to `blockTime`:

```go
// x/tieredrewards/keeper/msg_server.go
pos, _, _, err = ms.claimRewards(ctx, pos)   // sets LastBonusAccrual = ExitUnlockAt
...
pos.ClearExit(sdkCtx.BlockTime())             // zeroes ExitUnlockAt; LastBonusAccrual unchanged
```

After `ClearExit`, the position state is:
- `LastBonusAccrual = old ExitUnlockAt` (past)
- `ExitUnlockAt = zero` (cleared)

On the next `ClaimTierRewards`, `computeSegmentBonus` is called with:

```go
// x/tieredrewards/keeper/bonus_rewards.go
func (k Keeper) computeSegmentBonus(..., segmentStart, segmentEnd time.Time, ...) math.Int {
    if !pos.ExitUnlockAt.IsZero() && segmentEnd.After(pos.ExitUnlockAt) {
        segmentEnd = pos.ExitUnlockAt   // cap NOT applied — ExitUnlockAt is zero
    }
    ...
}
```

- `segmentStart = LastBonusAccrual = old ExitUnlockAt`
- `segmentEnd = blockTime`
- No cap is applied because `ExitUnlockAt.IsZero() = true`

Bonus is paid for the entire gap `[ExitUnlockAt, blockTime]` — the period during which the exit had already elapsed and no bonus should have accrued.

ADR-006 explicitly states: *"Bonus accrual is capped at ExitUnlockAt when exiting. No bonus after exit commitment elapses."* and in the edge-case table: *"ClearPosition after exit elapsed … Resets LastBonusAccrual to block\_time if delegated."* The code does not implement this reset.

---

### Impact Explanation

An unprivileged position owner can extract bonus rewards from the `RewardsPoolName` module account for a period they are not entitled to. The attacker controls the gap duration: the longer they wait between exit elapsing and calling `ClearPosition`, the larger the unauthorized bonus. For a large position with a high `BonusApy`, waiting months before calling `ClearPosition` drains a proportionally large amount from the shared rewards pool, directly harming all other position holders who depend on that pool.

The corrupted value is the `RewardsPoolName` module account balance, reduced by `shares × tokensPerShare × bonusApy × gapSeconds / SecondsPerYear` per exploit.

---

### Likelihood Explanation

Any position owner can trigger this with two standard user-facing transactions (`MsgClearPosition` after exit elapses, then `MsgClaimTierRewards`). No privileged access, no leaked keys, and no coordination with other users is required. The only precondition is owning a tier position and having previously triggered exit. The exploit is deterministic and repeatable.

---

### Recommendation

After `ClearExit`, explicitly advance `LastBonusAccrual` to `blockTime` so the gap period is not claimable:

```go
// x/tieredrewards/keeper/msg_server.go — ClearPosition handler
pos, _, _, err = ms.claimRewards(ctx, pos)
if err != nil { return nil, err }

sdkCtx := sdk.UnwrapSDKContext(ctx)
pos.ClearExit(sdkCtx.BlockTime())
// Reset accrual anchor so the gap [ExitUnlockAt, now] is not claimable.
if pos.IsDelegated() {
    pos.UpdateLastBonusAccrual(sdkCtx.BlockTime())
}
```

This matches the ADR-006 specification: *"Resets LastBonusAccrual to block\_time if delegated."*

---

### Proof of Concept

1. Alice calls `MsgLockTier` — creates a delegated position in tier 1 (BonusApy = 4%, ExitDuration = 1 year).
2. Alice calls `MsgTriggerExitFromTier` at time T₀. `ExitUnlockAt = T₀ + 1 year`.
3. Time advances past `ExitUnlockAt`. No bonus accrues (correctly capped).
4. Alice waits an additional 6 months (gap = G = 6 months). She does **not** claim.
5. Alice calls `MsgClearPosition`:
   - `claimRewards` → `applyBonusAccrualCheckpoint` sets `LastBonusAccrual = ExitUnlockAt` (bonus correctly zero for the gap so far).
   - `ClearExit` zeroes `ExitUnlockAt`. `LastBonusAccrual` remains at `ExitUnlockAt`.
6. Alice immediately calls `MsgClaimTierRewards`:
   - `segmentStart = ExitUnlockAt`, `segmentEnd = blockTime`, no cap.
   - Bonus paid = `shares × tokensPerShare × 0.04 × (6 months in seconds) / SecondsPerYear`.
   - For a 1 000 000 token position at 1:1 rate: `1 000 000 × 1.0 × 0.04 × 15 778 800 / 31 557 600 ≈ 20 000 tokens` extracted from the pool for a period where zero bonus was owed. [1](#0-0) [2](#0-1) [3](#0-2) [4](#0-3) [5](#0-4) [6](#0-5)

func (k Keeper) computeSegmentBonus(pos types.PositionState, tier types.Tier, segmentStart, segmentEnd time.Time, tokensPerShare math.LegacyDec) math.Int {
	if !pos.ExitUnlockAt.IsZero() && segmentEnd.After(pos.ExitUnlockAt) {
		segmentEnd = pos.ExitUnlockAt
	}

	if !segmentEnd.After(segmentStart) {
		return math.ZeroInt()
	}

	durationSeconds := int64(segmentEnd.Sub(segmentStart) / time.Second)
	if durationSeconds <= 0 {
		return math.ZeroInt()
	}

	tokens := pos.Delegation.Shares.Mul(tokensPerShare)

	return tokens.
		Mul(tier.BonusApy).
		MulInt64(durationSeconds).
		QuoInt64(types.SecondsPerYear).
		TruncateInt()
}

func (ms msgServer) ClearPosition(ctx context.Context, msg *types.MsgClearPosition) (*types.MsgClearPositionResponse, error) {
	if err := msg.Validate(); err != nil {
		return nil, err
	}

	pos, err := ms.getPositionState(ctx, msg.PositionId)
	if err != nil {
		return nil, err
	}

	if err := ms.validateClearPosition(ctx, pos, msg.Owner); err != nil {
		return nil, err
	}

	if !pos.HasTriggeredExit() {
		return &types.MsgClearPositionResponse{PositionId: pos.Id}, nil
	}

	pos, _, _, err = ms.claimRewards(ctx, pos)
	if err != nil {
		return nil, err
	}

	sdkCtx := sdk.UnwrapSDKContext(ctx)
	pos.ClearExit(sdkCtx.BlockTime())

	if err := ms.setPosition(ctx, pos.Position, nil); err != nil {
		return nil, err
	}

	if err := sdkCtx.EventManager().EmitTypedEvent(&types.EventExitCleared{
		PositionId: pos.Id,
		TierId:     pos.TierId,
		Owner:      pos.Owner,
	}); err != nil {
		return nil, err
	}

	return &types.MsgClearPositionResponse{PositionId: pos.Id}, nil
}

func (k Keeper) processEventsAndClaimBonus(ctx context.Context, pos *types.PositionState) (sdk.Coins, error) {
	// Rewards should have been claimed before undelegation
	if !pos.IsDelegated() {
		return sdk.NewCoins(), nil
	}

	valAddr, err := sdk.ValAddressFromBech32(pos.Delegation.ValidatorAddress)
	if err != nil {
		return nil, err
	}

	events, err := k.getValidatorEventsSince(ctx, valAddr, pos.LastEventSeq)
	if err != nil {
		return nil, err
	}

	sdkCtx := sdk.UnwrapSDKContext(ctx)
	blockTime := sdkCtx.BlockTime()

	totalBonus := math.ZeroInt()
	// Use the persisted bonded state from the last replay, not a hardcoded default.
	// This prevents overpaying bonus for unbonded gaps between claims.
	bonded := pos.LastKnownBonded
	segmentStart := pos.LastBonusAccrual

	tier, err := k.getTier(ctx, pos.TierId)
	if err != nil {
		return nil, err
	}

	for _, entry := range events {
		evt := entry.Event

		if bonded {
			// Compute bonus for the bonded segment [segmentStart, eventTime]
			// using the snapshot rate at the event.
			bonus := k.computeSegmentBonus(*pos, tier, segmentStart, evt.Timestamp, evt.TokensPerShare)
			totalBonus = totalBonus.Add(bonus)
		}

		// Update bonded state based on event type.
		switch evt.EventType {
		case types.ValidatorEventType_VALIDATOR_EVENT_TYPE_UNBOND:
			bonded = false
		case types.ValidatorEventType_VALIDATOR_EVENT_TYPE_BOND:
			bonded = true
		case types.ValidatorEventType_VALIDATOR_EVENT_TYPE_SLASH:
			// Slash doesn't change bonded state.
		}

		segmentStart = evt.Timestamp
		pos.UpdateLastEventSeq(entry.Seq)

		// Decrement reference count.
		if err := k.decrementEventRefCount(ctx, valAddr, entry.Seq); err != nil {
			return nil, err
		}
	}

	val, err := k.stakingKeeper.GetValidator(ctx, valAddr)
	if err != nil {
		return nil, err
	}
	// Defensive: validator bond status check
	if bonded && val.IsBonded() {
		currentRate, err := k.getTokensPerShare(ctx, valAddr)
		if err != nil {
			return nil, err
		}
		bonus := k.computeSegmentBonus(*pos, tier, segmentStart, blockTime, currentRate)
		totalBonus = totalBonus.Add(bonus)
	}

	applyBonusAccrualCheckpoint(&pos.Position, blockTime)
	// Persist the bonded state so the next replay starts correctly.
	pos.UpdateLastKnownBonded(bonded)
