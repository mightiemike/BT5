### Title
Checkpoint-Advance Without Bonus Payment in `slashRedelegationPosition` When Bonus Pool Is Drained — (`x/tieredrewards/keeper/slash.go`)

### Summary

When `RewardsPoolName` is empty and a redelegation slash fires via `BeforeRedelegationSlashed`, `slashRedelegationPosition` silently advances a position's `LastBonusAccrual` and `LastEventSeq` checkpoints without transferring the corresponding bonus coins to the owner. The position is then persisted with the advanced checkpoints, permanently erasing the owner's entitlement to bonus rewards for that period.

---

### Finding Description

The bug is a sequencing error across two functions.

**In `processEventsAndClaimBonus`** (`claim_rewards.go`):

The function mutates `pos` in-place via pointer before it checks pool sufficiency:

- Line 193: `pos.UpdateLastEventSeq(entry.Seq)` — advances `LastEventSeq` inside the event loop
- Line 215: `applyBonusAccrualCheckpoint(&pos.Position, blockTime)` — advances `LastBonusAccrual` to current block time
- Line 217: `pos.UpdateLastKnownBonded(bonded)` — updates bonded state

Only *after* all three mutations does it check the pool:

- Line 230: `k.sufficientBonusPoolBalance(ctx, bonusCoins)` — returns `ErrInsufficientBonusPool` if pool is empty

So when the pool is drained, the function returns an error with `pos` already fully checkpoint-advanced but no coins transferred. [1](#0-0) [2](#0-1) [3](#0-2) 

**In `slashRedelegationPosition`** (`slash.go`):

The function passes `&pos` (pointer) to `processEventsAndClaimBonus`, so all in-place mutations are visible after the call returns. When `ErrInsufficientBonusPool` is returned, the error is swallowed with only a log message. Execution then falls through to `setPositionWithState`, which persists the checkpoint-advanced `pos` to the store: [4](#0-3) 

The full-slash branch at line 70 calls `pos.ClearBonusCheckpoints()` before persisting, which would reset the checkpoints — but the partial-slash branch at line 77 does not, so it persists the advanced-but-unpaid checkpoints directly. [5](#0-4) 

**Invariant violated:** `LastBonusAccrual` is advanced to `blockTime` (line 215) without the corresponding bonus coins being sent to the owner. Any subsequent call to `ClaimTierRewards` or `processEventsAndClaimBonus` will start from the advanced checkpoint, permanently skipping the drained period.

---

### Impact Explanation

The position owner permanently loses all bonus rewards accrued between the previous `LastBonusAccrual` and the slash block time. After the pool is refilled, calling `ClaimTierRewards` yields zero bonus for that period because the checkpoint has already been advanced past it. There is no recourse — the missed period is irrecoverable.

This is a concrete, measurable loss of funds (bonus reward tokens) for the position owner. The magnitude scales with: position size × `BonusApy` × duration since last claim.

---

### Likelihood Explanation

The pool can be drained to zero through normal operation (many positions claiming rewards simultaneously, or simply insufficient pool funding). A redelegation slash is a standard Cosmos SDK staking event triggered whenever a validator with an active redelegation entry is slashed. No special privileges are required — the attacker only needs to ensure the pool is empty at the time a slash fires. This is a realistic, externally reachable condition.

---

### Recommendation

Move `applyBonusAccrualCheckpoint` and `UpdateLastKnownBonded` to execute **after** the pool sufficiency check and the `SendCoinsFromModuleToAccount` call succeed. This ensures checkpoints are only advanced when the bonus transfer actually completes.

Alternatively, in `slashRedelegationPosition`, when `ErrInsufficientBonusPool` is caught, explicitly reset the in-place mutations on `pos` back to the pre-call checkpoint values before calling `setPositionWithState`, so the position retains its original `LastBonusAccrual` and `LastEventSeq` and can be replayed later.

---

### Proof of Concept

1. Create a tier position and redelegate it (so a `RedelegationMapping` entry exists).
2. Drain the `RewardsPoolName` module account to zero.
3. Advance block time by 30 days so bonus accrues.
4. Trigger `BeforeRedelegationSlashed` (partial slash) → `slashRedelegationPosition` fires.
5. `processEventsAndClaimBonus` advances `pos.LastBonusAccrual` to block time, then returns `ErrInsufficientBonusPool`.
6. The error is swallowed; `setPositionWithState` persists the advanced checkpoint.
7. Refill the pool.
8. Call `ClaimTierRewards` for the position owner.
9. Assert: owner receives **zero** bonus for the 30-day period — the checkpoint was already advanced past it.

The existing test `TestProcessEvents_InsufficientPool_Error` confirms `ErrInsufficientBonusPool` is returned after checkpoints are mutated, and `TestSlashRedelegationPosition_ClaimsBonusRewardsUpToSlash` confirms the slash path calls `setPositionWithState` with the post-`processEventsAndClaimBonus` state — but no test covers the drained-pool + slash combination. // TestProcessEvents_InsufficientPool_Error verifies that claiming without

func (s *KeeperSuite) TestProcessEvents_InsufficientPool_Error() {
	lockAmount := sdkmath.NewInt(sdk.DefaultPowerReduction.Int64())
	pos := s.setupNewTierPosition(lockAmount, false)

	// Advance time so bonus would be non-zero. Do NOT fund the pool.
	s.ctx = s.ctx.WithBlockTime(s.ctx.BlockTime().Add(30 * 24 * time.Hour))

	_, err := s.keeper.ProcessEventsAndClaimBonus(s.ctx, &pos)
	s.Require().Error(err, "should fail when bonus pool is insufficient")
	s.Require().ErrorContains(err, "insufficient bonus pool",
		"error should mention insufficient bonus pool")
}

func (s *KeeperSuite) TestSlashRedelegationPosition_ClaimsBonusRewardsUpToSlash() {
	lockAmount := sdkmath.NewInt(sdk.DefaultPowerReduction.Int64())
	_, bondDenom := s.getStakingData()
	s.fundRewardsPool(sdkmath.NewInt(1_000_000_000), bondDenom)

	pos, _, unbondingID := s.setupRedelegatingPosition(lockAmount)
	owner := sdk.MustAccAddressFromBech32(pos.Owner)
	preAccrual := pos.LastBonusAccrual

	// Advance block time so bonus accrues on the destination validator.
	s.ctx = s.ctx.WithBlockHeight(s.ctx.BlockHeight() + 1)
	s.ctx = s.ctx.WithBlockTime(s.ctx.BlockTime().Add(30 * 24 * time.Hour))

	balBefore := s.app.BankKeeper.GetBalance(s.ctx, owner, bondDenom)

	// Partial slash — a small fraction of shares.
	sharesToUnbond := pos.Delegation.Shares.Quo(sdkmath.LegacyNewDec(10))
	err := s.keeper.Hooks().BeforeRedelegationSlashed(s.ctx, unbondingID, sharesToUnbond)
	s.Require().NoError(err)

	balAfter := s.app.BankKeeper.GetBalance(s.ctx, owner, bondDenom)
	s.Require().True(balAfter.Amount.GT(balBefore.Amount),
		"owner should have received bonus rewards accrued up to slash: before=%s after=%s",
		balBefore.Amount, balAfter.Amount)

	updated, err := s.keeper.GetPositionState(s.ctx, pos.Id)
	s.Require().NoError(err)
	s.Require().True(updated.LastBonusAccrual.After(preAccrual),
		"LastBonusAccrual should have advanced past the pre-slash checkpoint")
	s.Require().Equal(s.ctx.BlockTime(), updated.LastBonusAccrual,
		"LastBonusAccrual should advance to the slash block time")
	s.Require().True(updated.LastKnownBonded,
		"LastKnownBonded should remain true — destination validator is still bonded")
}
