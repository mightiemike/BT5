### Title
SwapAllowlistExtension Checks Router Address Instead of End User, Allowing Any User to Bypass the Swap Allowlist via MetricOmmSimpleRouter — (`metric-periphery/contracts/extensions/SwapAllowlistExtension.sol`)

---

### Summary

`SwapAllowlistExtension.beforeSwap` gates swaps by checking the `sender` parameter, which the pool sets to `msg.sender` of the pool — i.e., the immediate caller. When a user routes through `MetricOmmSimpleRouter`, the pool's `msg.sender` is the router contract, not the end user. If the pool admin allowlists the router address (the only way to permit router-mediated swaps), every unprivileged user can bypass the allowlist by calling the router.

---

### Finding Description

`MetricOmmPool.swap` passes `msg.sender` as the `sender` argument to `_beforeSwap`: [1](#0-0) 

`ExtensionCalling._beforeSwap` forwards that value unchanged to the extension: [2](#0-1) 

`SwapAllowlistExtension.beforeSwap` then checks `allowedSwapper[msg.sender][sender]`, where `msg.sender` is the pool and `sender` is the first parameter — the router address when the user routes through `MetricOmmSimpleRouter`: [3](#0-2) 

`MetricOmmSimpleRouter.exactInputSingle` calls `pool.swap` directly with no mechanism to forward the original `msg.sender`: [4](#0-3) 

The pool admin has two choices, both broken:

1. **Allowlist individual user addresses** — those users can only swap by calling the pool directly; they cannot use the router (the router address is not allowlisted, so the check fails).
2. **Allowlist the router address** — every user on the network can now swap through the router, completely defeating the allowlist.

There is no third option. The extension has no way to recover the original end-user identity from a router-mediated call.

---

### Impact Explanation

A pool configured with `SwapAllowlistExtension` to restrict trading to a curated set of addresses (e.g., KYC'd counterparties, whitelisted market makers, or a closed beta) is fully bypassed by any unprivileged user who routes through the public `MetricOmmSimpleRouter`. The attacker receives the same swap output as an allowlisted user, draining pool liquidity and breaking the pool's access-control invariant. This is a direct loss of LP assets and a broken core pool functionality.

---

### Likelihood Explanation

`MetricOmmSimpleRouter` is a public, permissionless contract. Any user who observes that a pool has the router allowlisted (readable on-chain via `allowedSwapper`) can immediately exploit this. The pool admin is likely to allowlist the router because it is the canonical periphery entry point; the documentation and test suite (`FullMetricExtension.t.sol`) do not demonstrate or warn about this identity-collapse problem. [5](#0-4) 

---

### Recommendation

The `SwapAllowlistExtension` must gate on the **original end user**, not the immediate pool caller. Two sound approaches:

1. **Pass the original caller through the router.** Add an `originalSender` field to the `extensionData` payload that the router populates with `msg.sender`. The extension decodes and checks that address. This requires a convention between the router and the extension.

2. **Check `sender` against the allowlist but also accept the router as a transparent forwarder only when the router itself encodes the real user.** This is equivalent to approach 1 but formalised in the extension interface.

The `DepositAllowlistExtension` avoids this problem by checking `owner` (the position owner parameter, which the caller supplies explicitly) rather than `sender`. A symmetric fix for swaps would require the pool or router to thread the real user identity through the call.

---

### Proof of Concept

```
Setup:
  - Deploy pool with SwapAllowlistExtension
  - Pool admin calls setAllowedToSwap(pool, router, true)
    (to allow router-mediated swaps for their users)
  - Alice (address not in allowlist) calls:
      router.exactInputSingle({pool: pool, recipient: alice, ...})
  - Router calls pool.swap(alice, ...) with msg.sender = router
  - Pool calls _beforeSwap(sender=router, ...)
  - SwapAllowlistExtension checks allowedSwapper[pool][router] == true  ✓
  - Alice's swap succeeds despite never being allowlisted

Expected: revert NotAllowedToSwap()
Actual:   swap executes, Alice receives tokens from the restricted pool
``` [6](#0-5) [7](#0-6)
