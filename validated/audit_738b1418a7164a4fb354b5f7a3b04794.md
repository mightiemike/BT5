### Title
Division by Zero in `socializeSubaccount()` Permanently Blocks Liquidation Finalization — (`File: core/contracts/PerpEngine.sol`)

---

### Summary

`PerpEngine.socializeSubaccount()` divides by `state.openInterest` without a zero-guard. When a perp market has no open positions (`openInterest == 0`) but the subaccount being finalized still carries a negative `vQuoteBalance`, the call reverts with `"DBZ"`, permanently blocking the liquidation finalization path and leaving an insolvent subaccount unresolvable.

---

### Finding Description

In `PerpEngine.socializeSubaccount()`, when insurance is insufficient to cover a negative `vQuoteBalance`, the protocol attempts to spread the loss across all open-interest holders:

```solidity
// PerpEngine.sol lines 164–171
if (balance.vQuoteBalance < 0) {
    // socialize across all other participants
    int128 fundingPerShare = -balance.vQuoteBalance.div(
        state.openInterest   // <-- divisor, can be 0
    );
    state.cumulativeFundingLongX18 += fundingPerShare;
    state.cumulativeFundingShortX18 -= fundingPerShare;
    balance.vQuoteBalance = 0;
}
```

`MathSD21x18.div()` enforces `require(y != 0, ERR_DIV_BY_ZERO)` at line 64, so the call reverts when `state.openInterest == 0`. [1](#0-0) [2](#0-1) 

`state.openInterest` is the **global** sum of absolute position sizes across all participants in that product. It is decremented when positions are closed and incremented when opened, as tracked in `_updateBalance()`:

```solidity
// PerpEngineState.sol lines 30, 46, 49
state.openInterest -= balance.amount.abs();   // pre-update
...
state.openInterest += balance.amount;          // post-update (long)
state.openInterest -= balance.amount;          // post-update (short)
``` [3](#0-2) 

A subaccount with `amount == 0` contributes nothing to `openInterest`. Therefore, if the subaccount being socialized is the **last or only** participant in a perp market (all others have also closed their positions), `state.openInterest == 0` at the time of socialization.

Crucially, `vQuoteBalance` can remain negative even after `amount` reaches zero — for example, from accumulated funding payments or realized losses on a closed position. This is a normal protocol state.

The contrast with `updateStates()` in `PerpEngineState.sol` is telling: that function explicitly guards against this case:

```solidity
// PerpEngineState.sol lines 111–113
if (state.openInterest == 0) {
    continue;
}
``` [4](#0-3) 

No equivalent guard exists in `socializeSubaccount()`.

---

### Impact Explanation

`socializeSubaccount()` is called from `_finalizeSubaccount()` in `ClearinghouseLiq.sol`:

```solidity
// ClearinghouseLiq.sol lines 386–389
v.insurance = perpEngine.socializeSubaccount(
    txn.liquidatee,
    v.insurance
);
``` [5](#0-4) 

`_finalizeSubaccount()` is the only mechanism to clean up a fully insolvent subaccount (all perp positions closed, all spot assets gone, negative residual `vQuoteBalance`). If it reverts, the insolvent subaccount is permanently stuck:

- The subaccount cannot be finalized — it remains in the system with unresolved negative `vQuoteBalance`.
- The insurance fund accounting is never updated for this subaccount's losses.
- Loss socialization across other participants never occurs, corrupting the protocol's solvency accounting.
- No alternative code path exists to finalize the subaccount.

This is a **solvency/accounting corruption** impact: the protocol's invariant that insolvent subaccounts can always be resolved is broken, and the insurance fund state diverges from reality.

---

### Likelihood Explanation

The trigger requires a perp market where:
1. The subaccount being finalized has `amount == 0` for the affected product (required by `_finalizeSubaccount`'s own checks at line 319).
2. All other participants in that product also have `amount == 0` (so `openInterest == 0`).
3. The subaccount still has negative `vQuoteBalance` (from funding or closed-position losses). [6](#0-5) 

This is realistic in:
- New or thin perp markets with few participants.
- Markets where all other traders have exited their positions.
- A market that has been effectively drained of open interest but still has residual negative `vQuoteBalance` from historical funding.

The liquidation path is reachable by any unprivileged liquidator calling `liquidateSubaccount` on the Endpoint, which delegatecalls `liquidateSubaccountImpl` on `ClearinghouseLiq`. [7](#0-6) 

---

### Recommendation

Add a zero-guard for `state.openInterest` in `socializeSubaccount()`, mirroring the existing guard in `updateStates()`. If `openInterest == 0`, there are no other participants to socialize losses to; the loss should be absorbed entirely by the insurance fund or written off:

```solidity
if (balance.vQuoteBalance < 0) {
    if (state.openInterest == 0) {
        // No open interest to socialize against; absorb or write off
        balance.vQuoteBalance = 0;
    } else {
        int128 fundingPerShare = -balance.vQuoteBalance.div(
            state.openInterest
        );
        state.cumulativeFundingLongX18 += fundingPerShare;
        state.cumulativeFundingShortX18 -= fundingPerShare;
        balance.vQuoteBalance = 0;
    }
}
```

---

### Proof of Concept

1. Deploy the protocol with a single perp product (e.g., `productId = 2`).
2. Alice opens a long position, accumulates negative `vQuoteBalance` from funding payments, then closes her position (`amount = 0`, `vQuoteBalance < 0`). No other traders have open positions → `state.openInterest == 0`.
3. Alice's account becomes insolvent (maintenance health < 0, e.g., due to negative `vQuoteBalance` exceeding her quote balance).
4. A liquidator submits `LiquidateSubaccount` with `txn.productId = type(uint32).max` to trigger finalization.
5. `_finalizeSubaccount` passes all position checks (Alice's `amount == 0`).
6. `perpEngine.socializeSubaccount(alice, insurance)` is called.
7. For `productId = 2`: `balance.vQuoteBalance < 0`, insurance insufficient → reaches `fundingPerShare = -balance.vQuoteBalance.div(state.openInterest)`.
8. `state.openInterest == 0` → `MathSD21x18.div` reverts with `"DBZ"`.
9. The entire liquidation transaction reverts. Alice's insolvent subaccount is permanently unresolvable. [8](#0-7) [9](#0-8)

    function _finalizeSubaccount(
        IEndpoint.LiquidateSubaccount calldata txn,
        ISpotEngine spotEngine,
        IPerpEngine perpEngine
    ) internal returns (bool) {
        if (txn.productId != type(uint32).max) {
            return false;
        }
        // check whether the subaccount can be finalized:
        // - all perps positions have closed
        // - all spread positions have closed
        // - all spot assets have closed
        // - all positive pnls have been settled

        FinalizeVars memory v;

        v.spotIds = spotEngine.getProductIds();
        v.perpIds = perpEngine.getProductIds();

        require(v.spotIds[0] == QUOTE_PRODUCT_ID);

        // all spot assets (except USDC) must be closed out
        for (uint32 i = 1; i < v.spotIds.length; ++i) {
            uint32 spotId = v.spotIds[i];
            if (spotEngine.getRisk(spotId).longWeightInitialX18 == 0) {
                continue;
            }
            ISpotEngine.Balance memory balance = spotEngine.getBalance(
                spotId,
                txn.liquidatee
            );
            require(balance.amount <= 0, ERR_NOT_FINALIZABLE_SUBACCOUNT);
        }

        for (uint32 i = 0; i < v.perpIds.length; ++i) {
            uint32 perpId = v.perpIds[i];
            IPerpEngine.Balance memory balance = perpEngine.getBalance(
                perpId,
                txn.liquidatee
            );
            require(balance.amount == 0, ERR_NOT_FINALIZABLE_SUBACCOUNT);
        }

        // settle all positive pnl
        for (uint32 i = 0; i < v.perpIds.length; ++i) {
            uint32 perpId = v.perpIds[i];
            IPerpEngine.Balance memory balance = perpEngine.getBalance(
                perpId,
                txn.liquidatee
            );
            if (balance.vQuoteBalance > 0) {
                _settlePnlAgainstLiquidator(
                    txn,
                    perpId,
                    balance.vQuoteBalance,
                    spotEngine,
                    perpEngine
                );
            }
        }

        ISpotEngine.Balance memory quoteBalance = spotEngine.getBalance(
            QUOTE_PRODUCT_ID,
            txn.liquidatee
        );

        // settle all negative pnl until quote balance becomes 0
        for (uint32 i = 0; i < v.perpIds.length; ++i) {
            uint32 perpId = v.perpIds[i];
            IPerpEngine.Balance memory balance = perpEngine.getBalance(
                perpId,
                txn.liquidatee
            );
            if (balance.vQuoteBalance < 0 && quoteBalance.amount > 0) {
                int128 canSettle = MathHelper.max(
                    balance.vQuoteBalance,
                    -quoteBalance.amount
                );
                _settlePnlAgainstLiquidator(
                    txn,
                    perpId,
                    canSettle,
                    spotEngine,
                    perpEngine
                );
                quoteBalance.amount += canSettle;
            }
        }

        v.insurance = insurance;
        v.insurance -= lastLiquidationFees;
        v.canLiquidateMore = (quoteBalance.amount + v.insurance) > 0;

        if (v.canLiquidateMore) {
            for (uint32 i = 1; i < v.spotIds.length; ++i) {
                uint32 spotId = v.spotIds[i];
                ISpotEngine.Balance memory balance = spotEngine.getBalance(
                    spotId,
                    txn.liquidatee
                );
                if (spotEngine.getRisk(spotId).longWeightInitialX18 == 0) {
                    continue;
                }
                require(balance.amount == 0, ERR_NOT_FINALIZABLE_SUBACCOUNT);
            }
        }

        v.insurance = perpEngine.socializeSubaccount(
            txn.liquidatee,
            v.insurance
        );

        // we can assure that quoteBalance must be non positive, because if quoteBalance.amount > 0,
        // there must be 1) no negative pnl in perps, and 2) no liabilities in spot after above actions.
        // however, in this case the liquidatee must be healthy and cannot pass the health check at
        // the beginning.
        int128 insuranceCover = MathHelper.min(
            v.insurance,
            -quoteBalance.amount
        );
        if (insuranceCover > 0) {
            v.insurance -= insuranceCover;
            spotEngine.updateBalance(
                QUOTE_PRODUCT_ID,
                txn.liquidatee,
                insuranceCover
            );
        }
        if (v.insurance <= 0) {
            spotEngine.socializeSubaccount(txn.liquidatee);
        }
        v.insurance += lastLiquidationFees;
        insurance = v.insurance;
        return true;
    }
