### Title
Bonus Rewards Permanently Lost When Pool Is Insufficient During `BeforeRedelegationSlashed` Hook - (`File: x/tieredrewards/keeper/slash.go`)

### Summary

In `slashRedelegationPosition`, when `processEventsAndClaimBonus` fails with `ErrInsufficientBonusPool`, the error is deliberately swallowed to prevent a chain halt. However, `processEventsAndClaimBonus` advances the position's accrual checkpoints (`LastBonusAccrual`, `LastEventSeq`, `LastKnownBonded`) **before** it checks the pool balance. The modified position is then persisted with those advanced checkpoints. The accrued bonus is permanently unclaimable: the user loses it, and the corresponding coins remain stranded in `rewards_pool` with no mechanism to route them to the rightful owner.

### Finding Description

`processEventsAndClaimBonus` in `x/tieredrewards/keeper/claim_rewards.go` advances the position's checkpoints unconditionally before performing the pool-sufficiency check:

```
line 215: applyBonusAccrualCheckpoint(&pos.Position, blockTime)   // advances LastBonusAccrual
line 217: pos.UpdateLastKnownBonded(bonded)                        // advances LastKnownBonded
...
line 193: pos.UpdateLastEventSeq(entry.Seq)  (inside event loop)  // advances LastEventSeq
...
line 230: if err := k.sufficientBonusPoolBalance(ctx, bonusCoins); err != nil {
              return nil, err   // returns ErrInsufficientBonusPool
          }
``` [1](#0-0) 

When `ErrInsufficientBonusPool` is returned, the `pos` pointer already carries the advanced checkpoints. In `slashRedelegationPosition`, this error is caught and swallowed:

```go
if _, err := k.processEventsAndClaimBonus(ctx, &pos); err != nil {
    if errors.Is(err, types.ErrInsufficientBonusPool) {
        k.logger(ctx).Error("insufficient bonus pool during redelegation slash", ...)
    } else {
        return err
    }
}
``` [2](#0-1) 

Execution then falls through to `setPositionWithState`, which persists the position with the already-advanced checkpoints: [3](#0-2) 

On every subsequent `processEventsAndClaimBonus` call, the replay starts from the new `LastBonusAccrual` / `LastEventSeq`, so the pre-slash accrual window is permanently skipped. The bonus coins that were computed as owed to the user were never sent from `RewardsPoolName`, yet the user can never claim them.

### Impact Explanation

The corrupted invariant is: **for every accrual window consumed by advancing `LastBonusAccrual` and `LastEventSeq`, the corresponding bonus coins must either be sent to the owner or the checkpoints must not be advanced.** This invariant is broken in the `BeforeRedelegationSlashed` path.

Concrete impact:
- The position owner permanently loses all bonus rewards accrued between `LastBonusAccrual` and the slash block time.
- The `rewards_pool` module account retains those coins with no routing mechanism to the rightful owner.
- This is an exact analog to the external report: fees are "deducted" (accrual window consumed) but never forwarded to the designated recipient (the position owner), leaving value stranded.

### Likelihood Explanation

The trigger requires two concurrent conditions:
1. A tier position has an active redelegation in the unbonding period (user called `MsgTierRedelegate`).
2. The source validator is slashed while that redelegation is pending AND the `rewards_pool` balance is below the computed bonus at that moment.

Both conditions are realistic in production. The `rewards_pool` can be depleted by heavy bonus claims or simply not yet replenished. Validator slashing is a normal protocol event. The combination is not exotic.

### Recommendation

In `slashRedelegationPosition`, when `ErrInsufficientBonusPool` is caught, do **not** persist the position with the already-advanced checkpoints. Instead, restore the original checkpoints before saving, so the user can retry the claim once the pool is replenished:

```go
originalPos := pos  // snapshot before calling processEventsAndClaimBonus
if _, err := k.processEventsAndClaimBonus(ctx, &pos); err != nil {
    if errors.Is(err, types.ErrInsufficientBonusPool) {
        k.logger(ctx).Error(...)
        pos = originalPos  // restore checkpoints so bonus is not lost
    } else {
        return err
    }
}
```

Alternatively, restructure `processEventsAndClaimBonus` to advance checkpoints only after a successful `SendCoinsFromModuleToAccount`, so that a pool-insufficiency error leaves the position state unchanged.

### Proof of Concept

1. User calls `MsgLockTier` to create a position on validator A.
2. User calls `MsgTierRedelegate` to move the delegation to validator B. A `RedelegationMapping(unbondingId → positionId)` entry is created.
3. Time passes; bonus accrues on validator B.
4. The `rewards_pool` is drained (e.g., by other users claiming large bonuses).
5. Governance slashes validator A (the source of the redelegation). The SDK fires `BeforeRedelegationSlashed(unbondingId, sharesToUnbond)`.
6. `slashRedelegationPosition` is called. It calls `processEventsAndClaimBonus(&pos)`.
7. Inside `processEventsAndClaimBonus`: checkpoints are advanced to `blockTime` (lines 215, 217), then `sufficientBonusPoolBalance` returns `ErrInsufficientBonusPool` (line 230). The function returns the error with `pos` already mutated.
8. Back in `slashRedelegationPosition`: the error is swallowed (lines 56–63). `setPositionWithState(ctx, pos, ...)` is called, persisting the advanced `LastBonusAccrual` = `blockTime`.
9. User later calls `MsgClaimTierRewards`. `processEventsAndClaimBonus` starts from the new `LastBonusAccrual` = `blockTime`, so the pre-slash accrual window yields zero bonus. The user permanently loses the accrued bonus. [4](#0-3) [5](#0-4)

func (k Keeper) processEventsAndClaimBonus(ctx context.Context, pos *types.PositionState) (sdk.Coins, error) {
	// Rewards should have been claimed before undelegation
	if !pos.IsDelegated() {
		return sdk.NewCoins(), nil
	}

	valAddr, err := sdk.ValAddressFromBech32(pos.Delegation.ValidatorAddress)
	if err != nil {
		return nil, err
	}

	events, err := k.getValidatorEventsSince(ctx, valAddr, pos.LastEventSeq)
	if err != nil {
		return nil, err
	}

	sdkCtx := sdk.UnwrapSDKContext(ctx)
	blockTime := sdkCtx.BlockTime()

	totalBonus := math.ZeroInt()
	// Use the persisted bonded state from the last replay, not a hardcoded default.
	// This prevents overpaying bonus for unbonded gaps between claims.
	bonded := pos.LastKnownBonded
	segmentStart := pos.LastBonusAccrual

	tier, err := k.getTier(ctx, pos.TierId)
	if err != nil {
		return nil, err
	}

	for _, entry := range events {
		evt := entry.Event

		if bonded {
			// Compute bonus for the bonded segment [segmentStart, eventTime]
			// using the snapshot rate at the event.
			bonus := k.computeSegmentBonus(*pos, tier, segmentStart, evt.Timestamp, evt.TokensPerShare)
			totalBonus = totalBonus.Add(bonus)
		}

		// Update bonded state based on event type.
		switch evt.EventType {
		case types.ValidatorEventType_VALIDATOR_EVENT_TYPE_UNBOND:
			bonded = false
		case types.ValidatorEventType_VALIDATOR_EVENT_TYPE_BOND:
			bonded = true
		case types.ValidatorEventType_VALIDATOR_EVENT_TYPE_SLASH:
			// Slash doesn't change bonded state.
		}

		segmentStart = evt.Timestamp
		pos.UpdateLastEventSeq(entry.Seq)

		// Decrement reference count.
		if err := k.decrementEventRefCount(ctx, valAddr, entry.Seq); err != nil {
			return nil, err
		}
	}

	val, err := k.stakingKeeper.GetValidator(ctx, valAddr)
	if err != nil {
		return nil, err
	}
	// Defensive: validator bond status check
	if bonded && val.IsBonded() {
		currentRate, err := k.getTokensPerShare(ctx, valAddr)
		if err != nil {
			return nil, err
		}
		bonus := k.computeSegmentBonus(*pos, tier, segmentStart, blockTime, currentRate)
		totalBonus = totalBonus.Add(bonus)
	}

	applyBonusAccrualCheckpoint(&pos.Position, blockTime)
	// Persist the bonded state so the next replay starts correctly.
	pos.UpdateLastKnownBonded(bonded)

	if totalBonus.IsZero() {
		return sdk.NewCoins(), nil
	}

	bondDenom, err := k.stakingKeeper.BondDenom(ctx)
	if err != nil {
		return nil, err
	}

	bonusCoins := sdk.NewCoins(sdk.NewCoin(bondDenom, totalBonus))

	if err := k.sufficientBonusPoolBalance(ctx, bonusCoins); err != nil {
		return nil, err
	}

	ownerAddr, err := sdk.AccAddressFromBech32(pos.Owner)
	if err != nil {
		return nil, errorsmod.Wrap(sdkerrors.ErrInvalidAddress, "invalid owner address")
	}

	if err := k.bankKeeper.SendCoinsFromModuleToAccount(ctx, types.RewardsPoolName, ownerAddr, bonusCoins); err != nil {
		return nil, err
	}

	if err := sdkCtx.EventManager().EmitTypedEvent(&types.EventBonusRewardsClaimed{
		PositionId: pos.Id,
		Owner:      pos.Owner,
		Rewards:    bonusCoins,
	}); err != nil {
		return nil, err
	}

	return bonusCoins, nil
```

func (k Keeper) slashRedelegationPosition(ctx context.Context, unbondingId uint64, sharesToUnbond math.LegacyDec) error {
	positionId, err := k.getRedelegationMapping(ctx, unbondingId)
	if errors.Is(err, collections.ErrNotFound) {
		return nil
	}
	if err != nil {
		return err
	}

	pos, err := k.getPositionState(ctx, positionId)
	if errors.Is(err, types.ErrPositionNotFound) {
		k.logger(ctx).Error("position not found during redelegation slash",
			"position_id", positionId,
			"unbonding_id", unbondingId,
			"error", err.Error(),
		)
		return nil
	}
	if err != nil {
		return err
	}

	if !pos.IsDelegated() {
		// Defensive
		k.logger(ctx).Error("delegation missing during BeforeRedelegationSlashed",
			"position_id", positionId,
			"unbonding_id", unbondingId,
			"shares_to_unbond", sharesToUnbond.String(),
		)
		return nil
	}

	dstValStr := pos.Delegation.ValidatorAddress

	// Settle bonus against PRE-slash shares.
	if _, err := k.processEventsAndClaimBonus(ctx, &pos); err != nil {
		// Deliberately forgo bonus rewards if pool is insufficient to prevent chain halt.
		if errors.Is(err, types.ErrInsufficientBonusPool) {
			k.logger(ctx).Error("insufficient bonus pool during redelegation slash",
				"position_id", pos.Id,
				"error", err.Error(),
			)
		} else {
			return err
		}
	}

	fullSlash := sharesToUnbond.GTE(pos.Delegation.Shares)

	if fullSlash {
		pos.Delegation = nil
		pos.ClearBonusCheckpoints()
		return k.setPositionWithState(ctx, pos, &ValidatorTransition{PreviousAddress: dstValStr})
	}
	// In-memory only: the persisted Position carries no share count, and the
	// live delegation will reflect the post-Unbond shares on the next read.
	// Update the local copy so any follow-up logic in this call sees consistent state.
	pos.Delegation.Shares = pos.Delegation.Shares.Sub(sharesToUnbond)
	return k.setPositionWithState(ctx, pos, nil)
