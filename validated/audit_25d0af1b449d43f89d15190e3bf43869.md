### Title
Bonus Reward Checkpoints Advanced Before Pool Balance Check Causes Permanent Reward Loss on Slash - (File: x/tieredrewards/keeper/slash.go, x/tieredrewards/keeper/claim_rewards.go)

### Summary

In `processEventsAndClaimBonus`, the position's bonus accrual checkpoints (`LastBonusAccrual`, `LastEventSeq`, `LastKnownBonded`) are mutated on the in-memory `*PositionState` pointer **before** the pool balance is verified. When `slashRedelegationPosition` deliberately swallows `ErrInsufficientBonusPool` to prevent a chain halt, it then persists the already-mutated position to state. The result is that the position's checkpoints are permanently advanced as if the bonus was paid, but the owner receives nothing.

### Finding Description

`processEventsAndClaimBonus` takes a `*types.PositionState` pointer and applies two checkpoint mutations unconditionally before checking whether the rewards pool can cover the computed bonus:

```go
// claim_rewards.go lines 215-217
applyBonusAccrualCheckpoint(&pos.Position, blockTime)
pos.UpdateLastKnownBonded(bonded)

if totalBonus.IsZero() {
    return sdk.NewCoins(), nil
}
// ...
if err := k.sufficientBonusPoolBalance(ctx, bonusCoins); err != nil {
    return nil, err   // pos is already mutated; error propagates to caller
}
``` [1](#0-0) 

Because `pos` is a pointer, the mutations to `LastBonusAccrual`, `LastEventSeq`, and `LastKnownBonded` are visible to the caller even after the error return. In `slashRedelegationPosition`, `ErrInsufficientBonusPool` is explicitly swallowed:

```go
// slash.go lines 54-64
if _, err := k.processEventsAndClaimBonus(ctx, &pos); err != nil {
    if errors.Is(err, types.ErrInsufficientBonusPool) {
        k.logger(ctx).Error("insufficient bonus pool during redelegation slash", ...)
        // execution continues — pos checkpoints are already advanced
    } else {
        return err
    }
}
``` [2](#0-1) 

Execution then falls through to `setPositionWithState`, which persists the mutated `pos` — with its checkpoints advanced — to the on-chain store:

```go
// slash.go lines 68-77
if fullSlash {
    ...
    return k.setPositionWithState(ctx, pos, &ValidatorTransition{...})
}
pos.Delegation.Shares = pos.Delegation.Shares.Sub(sharesToUnbond)
return k.setPositionWithState(ctx, pos, nil)
``` [3](#0-2) 

The next time the owner calls `ClaimTierRewards` or any reward-settling handler, `processEventsAndClaimBonus` will start its event replay from the already-advanced `LastEventSeq` and `LastBonusAccrual`, so the bonus window that was computed but never paid is permanently skipped.

### Impact Explanation

The exact corrupted values are `pos.LastBonusAccrual` (timestamp), `pos.LastEventSeq` (event sequence), and `pos.LastKnownBonded` (bonded flag) stored in the `Position` record in the `Positions` collection. These checkpoints mark the "high-water mark" of bonus accrual. Advancing them without a corresponding coin transfer means the owner permanently forfeits all bonus rewards that accrued from `pos.LastBonusAccrual` up to `blockTime` at the time of the slash. The forfeited amount is proportional to position size, tier `BonusApy`, and the elapsed time since the last claim. [4](#0-3) 

### Likelihood Explanation

The trigger path requires three concurrent conditions:

1. A tier position exists whose validator is undergoing a redelegation that is being slashed (`BeforeRedelegationSlashed` hook fires → `slashRedelegationPosition`).
2. The `RewardsPoolName` module account has insufficient balance to cover the computed bonus at that moment.
3. `totalBonus` is non-zero (the position has accrued bonus since its last checkpoint).

Condition 1 is a normal protocol event (validator misbehaviour during an active redelegation). Condition 2 is realistic: the pool is funded by governance deposits and can be depleted by concurrent claims or simply not yet replenished. Condition 3 is satisfied for any position that has not claimed recently. No privileged access, leaked keys, or social engineering is required; the slash hook fires automatically as part of standard Cosmos SDK staking logic. [5](#0-4) 

### Recommendation

Move `applyBonusAccrualCheckpoint` and `pos.UpdateLastKnownBonded` to **after** the successful `bankKeeper.SendCoinsFromModuleToAccount` call, so checkpoints are only advanced when coins are actually transferred. Alternatively, if the early-return on `ErrInsufficientBonusPool` must be preserved for chain-halt safety, reset the in-memory checkpoint fields to their pre-call values before returning the error, so the caller's `pos` remains unmodified and `setPositionWithState` persists the original checkpoints. [6](#0-5) 

### Proof of Concept

1. Alice creates a tier position delegating to validator V.
2. Alice initiates a redelegation from V to V2 (`TierRedelegate`). An unbonding ID is recorded in `RedelegationMappings`.
3. V is slashed while the redelegation is in flight. The SDK fires `BeforeRedelegationSlashed` → `slashRedelegationPosition`.
4. `processEventsAndClaimBonus` is called with `&pos`. It computes a non-zero `totalBonus`, advances `pos.LastBonusAccrual` and `pos.LastEventSeq` in memory (lines 215–217), then calls `sufficientBonusPoolBalance` which returns `ErrInsufficientBonusPool` because the pool is empty.
5. Back in `slashRedelegationPosition`, the `ErrInsufficientBonusPool` branch logs the error and continues. `pos` now carries the advanced checkpoints.
6. `setPositionWithState(ctx, pos, nil)` (or the full-slash variant) writes the mutated position to the store.
7. Alice later calls `MsgClaimTierRewards`. `processEventsAndClaimBonus` starts from the already-advanced `LastEventSeq` / `LastBonusAccrual` and computes zero bonus for the window that was silently skipped. Alice receives no bonus for that period and has no recourse. [7](#0-6)

func (k Keeper) slashRedelegationPosition(ctx context.Context, unbondingId uint64, sharesToUnbond math.LegacyDec) error {
	positionId, err := k.getRedelegationMapping(ctx, unbondingId)
	if errors.Is(err, collections.ErrNotFound) {
		return nil
	}
	if err != nil {
		return err
	}

	pos, err := k.getPositionState(ctx, positionId)
	if errors.Is(err, types.ErrPositionNotFound) {
		k.logger(ctx).Error("position not found during redelegation slash",
			"position_id", positionId,
			"unbonding_id", unbondingId,
			"error", err.Error(),
		)
		return nil
	}
	if err != nil {
		return err
	}

	if !pos.IsDelegated() {
		// Defensive
		k.logger(ctx).Error("delegation missing during BeforeRedelegationSlashed",
			"position_id", positionId,
			"unbonding_id", unbondingId,
			"shares_to_unbond", sharesToUnbond.String(),
		)
		return nil
	}

	dstValStr := pos.Delegation.ValidatorAddress

	// Settle bonus against PRE-slash shares.
	if _, err := k.processEventsAndClaimBonus(ctx, &pos); err != nil {
		// Deliberately forgo bonus rewards if pool is insufficient to prevent chain halt.
		if errors.Is(err, types.ErrInsufficientBonusPool) {
			k.logger(ctx).Error("insufficient bonus pool during redelegation slash",
				"position_id", pos.Id,
				"error", err.Error(),
			)
		} else {
			return err
		}
	}

	fullSlash := sharesToUnbond.GTE(pos.Delegation.Shares)

	if fullSlash {
		pos.Delegation = nil
		pos.ClearBonusCheckpoints()
		return k.setPositionWithState(ctx, pos, &ValidatorTransition{PreviousAddress: dstValStr})
	}
	// In-memory only: the persisted Position carries no share count, and the
	// live delegation will reflect the post-Unbond shares on the next read.
	// Update the local copy so any follow-up logic in this call sees consistent state.
	pos.Delegation.Shares = pos.Delegation.Shares.Sub(sharesToUnbond)
	return k.setPositionWithState(ctx, pos, nil)
```
