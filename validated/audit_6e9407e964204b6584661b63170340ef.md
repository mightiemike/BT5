### Title
Deprecated `verify_transaction_inclusion` Accepts Internal Merkle Nodes as Valid Transaction IDs, Enabling Proof-Verification Forgery — (File: `contract/src/lib.rs`)

---

### Summary

The deprecated `verify_transaction_inclusion` function remains publicly callable by any unprivileged NEAR account. It does not validate that the caller-supplied `tx_id` is a leaf node (a real transaction hash) rather than an internal Merkle tree node. An attacker can supply an internal node hash as `tx_id`, construct a valid sibling-path proof for it, and cause the function to return `true` for a non-existent Bitcoin transaction. Downstream contracts that consume this result are deceived into treating a forged proof as a confirmed transaction inclusion.

---

### Finding Description

`verify_transaction_inclusion` is a public, unguarded NEAR method. Its only verification step is:

```rust
merkle_tools::compute_root_from_merkle_proof(
    args.tx_id,
    usize::try_from(args.tx_index).unwrap(),
    &args.merkle_proof,
) == header.block_header.merkle_root
``` [1](#0-0) 

`compute_root_from_merkle_proof` is a pure hash-chain computation that treats `tx_id` as an opaque 32-byte starting value and walks up the tree using the caller-supplied `merkle_proof` siblings: [2](#0-1) 

There is no check that `tx_id` is a leaf node. If an attacker picks any internal node `N` at depth `d` in a real block's Merkle tree, they can supply the `(tree_depth − d)` real sibling hashes above `N` as `merkle_proof`. The hash chain will correctly recompute the block's `merkle_root`, and the function returns `true`.

The code itself acknowledges this broken invariant in its own `# Warning` block:

> "This function may return `true` if the provided `tx_id` is a hash of an internal node in the Merkle tree rather than a valid transaction hash." [3](#0-2) 

The `#[deprecated]` Rust attribute is a compile-time lint only; it does not prevent runtime invocation. The function carries no `#[pause]` guard that would block it independently, and no role check restricts callers. [4](#0-3) 

`verify_transaction_inclusion_v2` was introduced to mitigate this via a coinbase-proof length check, but it calls the deprecated function internally after the coinbase guard: [5](#0-4) 

The v1 path remains directly reachable and unguarded.

---

### Impact Explanation

Any downstream NEAR contract that calls `verify_transaction_inclusion` and acts on a `true` result — for example, releasing bridged funds, minting wrapped BTC, or settling a payment — can be deceived into executing those actions for a Bitcoin transaction that never existed. The attacker impersonates a legitimate confirmed transaction by forging its inclusion proof, directly analogous to the external report's attacker impersonating a legitimate user account by forging their session token. The corrupted value is the **proof result** (`true` for a non-existent transaction), and the broken invariant is the authorization assumption that a `true` result guarantees a real leaf-level transaction.

**Impact: 4** — Confidentiality and integrity of downstream protocol state (bridged assets, settlement records) are at risk.

---

### Likelihood Explanation

The attack requires no privileged role, no leaked key, and no social engineering. Bitcoin block Merkle trees are fully public; any attacker can read internal node hashes from any block explorer. The only prerequisite is that the target block has already been submitted to the contract (which is the normal operating state). The function is callable by any NEAR account with no deposit or staking requirement.

**Likelihood: 3** — Straightforward to execute by any external caller once a block is in the headers pool.

---

### Recommendation

1. **Remove or gate the deprecated function.** Add `env::panic_str("use verify_transaction_inclusion_v2")` as the function body, or apply `#[private]` to prevent external calls entirely.
2. **Do not rely on `#[deprecated]` for security boundaries.** Rust's `deprecated` attribute is a compiler hint, not an access control mechanism.
3. **Audit all downstream contracts** that may still call the v1 function and migrate them to `verify_transaction_inclusion_v2`.

---

### Proof of Concept

1. Identify any Bitcoin block already stored in the contract's `headers_pool` (e.g., height 685452, which is present in the test data).
2. From a public block explorer, obtain the full transaction list for that block and reconstruct its Merkle tree.
3. Select any internal node `N` at depth 1: `N = double_sha256(tx[0] || tx[1])`.
4. The sibling proof from `N` to the root is the sequence of sibling hashes at levels 2, 3, … up to the root — all publicly available.
5. Call `verify_transaction_inclusion` with:
   - `tx_id = N` (the internal node hash)
   - `tx_block_blockhash = <real block hash>`
   - `tx_index = 0`
   - `merkle_proof = [sibling at level 2, sibling at level 3, …]`
   - `confirmations = 1`
6. `compute_root_from_merkle_proof(N, 0, proof)` recomputes the correct `merkle_root`; the function returns `true` for a transaction that does not exist. [2](#0-1) [6](#0-5)

    #[pause]
    pub fn verify_transaction_inclusion(&self, #[serializer(borsh)] args: ProofArgs) -> bool {
        require!(
            args.confirmations <= self.gc_threshold,
            "The required number of confirmations exceeds the number of blocks stored in memory"
        );

        let heaviest_block_header = self
            .headers_pool
            .get(&self.mainchain_tip_blockhash)
            .unwrap_or_else(|| env::panic_str(ERR_KEY_NOT_EXIST));
        let target_block_height = self
            .mainchain_header_to_height
            .get(&args.tx_block_blockhash)
            .unwrap_or_else(|| env::panic_str("block does not belong to the current main chain"));

        // Check requested confirmations. No need to compute proof if insufficient confirmations.
        require!(
            (heaviest_block_header.block_height).saturating_sub(target_block_height) + 1
                >= args.confirmations,
            "Not enough blocks confirmed"
        );

        let header = self
            .headers_pool
            .get(&args.tx_block_blockhash)
            .unwrap_or_else(|| env::panic_str("cannot find requested transaction block"));

        require!(!args.merkle_proof.is_empty(), "Merkle proof is empty");

        // compute merkle tree root and check if it matches block's original merkle tree root
        merkle_tools::compute_root_from_merkle_proof(
            args.tx_id,
            usize::try_from(args.tx_index).unwrap(),
            &args.merkle_proof,
        ) == header.block_header.merkle_root
    }

pub fn compute_root_from_merkle_proof(
    transaction_hash: H256,
    transaction_position: usize,
    merkle_proof: &Vec<H256>,
) -> H256 {
    let mut current_hash = transaction_hash;
    let mut current_position = transaction_position;

    for proof_hash in merkle_proof {
        if current_position % 2 == 0 {
            current_hash = compute_hash(&current_hash, proof_hash);
        } else {
            current_hash = compute_hash(proof_hash, &current_hash);
        }
        current_position /= 2;
    }

    current_hash
}
