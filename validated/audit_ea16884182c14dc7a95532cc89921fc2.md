### Title
`burnNlp()` Has No Minimum Output Protection â€” User Receives Less Quote Than Expected at Signing Time - (File: `core/contracts/Clearinghouse.sol`)

---

### Summary

The `burnNlp()` function in `Clearinghouse.sol` computes the quote amount returned to the user using an `oraclePriceX18` that is supplied by the sequencer at execution time, not committed to by the user's EIP-712 signature. Because the `BurnNlp` struct the user signs contains only `(sender, nlpAmount, nonce)`, there is no on-chain mechanism for the user to enforce a minimum acceptable quote output. If the NLP oracle price falls between the time the user signs and the time the sequencer executes, the user receives less quote than expected with no recourse.

---

### Finding Description

When a user wants to redeem NLP tokens for quote, they sign a `BurnNlp` struct:

```solidity
struct BurnNlp {
    bytes32 sender;
    uint128 nlpAmount;
    uint64 nonce;
}
```

The EIP-712 type string confirms exactly what is committed to:

```
"BurnNlp(bytes32 sender,uint128 nlpAmount,uint64 nonce)"
``` [1](#0-0) 

The `oraclePriceX18` is **not** part of the signed message. It lives in `SignedBurnNlp` alongside the signature but outside the signed `BurnNlp` payload:

```solidity
struct SignedBurnNlp {
    BurnNlp tx;
    bytes signature;
    int128 oraclePriceX18;       // sequencer-supplied, not signed by user
    int128[] nlpPoolRebalanceX18; // sequencer-supplied, not signed by user
}
``` [2](#0-1) 

In `EndpointTx.sol`, the sequencer decodes the transaction and passes its own `oraclePriceX18` directly to `clearinghouse.burnNlp()`: [3](#0-2) 

Inside `burnNlp()`, the quote the user receives is computed entirely from this sequencer-controlled price:

```solidity
int128 quoteAmount = nlpAmount.mul(oraclePriceX18);
int128 burnFee = MathHelper.max(ONE, quoteAmount / 1000);
quoteAmount = MathHelper.max(0, quoteAmount - burnFee);
``` [4](#0-3) 

There is no check of the form `require(quoteAmount >= minOutputAmount)`. The error constant `ERR_SLIPPAGE_TOO_HIGH` exists in `Errors.sol` but is never referenced in any execution path related to NLP burns. [5](#0-4) 

---

### Impact Explanation

**Impact: Medium.** A user burns a fixed `nlpAmount` and receives a `quoteAmount` determined solely by the sequencer-provided `oraclePriceX18`. If the price used at execution is lower than the price the user observed when signing, the user suffers a direct, unrecoverable loss of quote collateral with no on-chain protection. The NLP tokens are already debited (`spotEngine.updateBalance(NLP_PRODUCT_ID, txn.sender, -nlpAmount)`) before the quote is credited, so the user cannot avoid the unfavorable execution. [6](#0-5) 

---

### Likelihood Explanation

**Likelihood: Medium.** The NLP oracle price is updated per-transaction by the sequencer (`spotEngine.updatePrice(NLP_PRODUCT_ID, oraclePriceX18)`). Any latency between signing and sequencer inclusion â€” or any sequencer behavior that uses a stale or lower price â€” results in the user receiving less than expected. This is a realistic scenario during volatile market conditions or sequencer queue delays. [7](#0-6) 

---

### Recommendation

Add a `minOutputAmount` field to the `BurnNlp` struct so it is part of the user's EIP-712 commitment, and enforce it in `burnNlp()`:

```diff
 struct BurnNlp {
     bytes32 sender;
     uint128 nlpAmount;
     uint64 nonce;
+    uint128 minOutputAmount;
 }
```

```diff
 int128 quoteAmount = nlpAmount.mul(oraclePriceX18);
 int128 burnFee = MathHelper.max(ONE, quoteAmount / 1000);
 quoteAmount = MathHelper.max(0, quoteAmount - burnFee);
+require(quoteAmount >= int128(txn.minOutputAmount), ERR_SLIPPAGE_TOO_HIGH);
```

Update the EIP-712 type string in `Verifier.sol` accordingly:

```diff
-"BurnNlp(bytes32 sender,uint128 nlpAmount,uint64 nonce)"
+"BurnNlp(bytes32 sender,uint128 nlpAmount,uint64 nonce,uint128 minOutputAmount)"
```

### Proof of Concept

1. User observes NLP oracle price = 1.00 USDC and signs `BurnNlp { sender, nlpAmount: 1000e18, nonce }` expecting ~999 USDC (after 0.1% burn fee).
2. Before the sequencer includes the transaction, the NLP price drops to 0.90 USDC.
3. Sequencer submits `SignedBurnNlp` with `oraclePriceX18 = 0.90e18`.
4. `burnNlp()` computes `quoteAmount = 1000e18 * 0.90 = 900 USDC`, deducts fee â†’ user receives ~899.1 USDC.
5. User loses ~99.9 USDC (~10%) with no on-chain recourse, because `BurnNlp` contained no price bound and the contract enforces none. [8](#0-7) [9](#0-8)

    function burnNlp(
        IEndpoint.BurnNlp calldata txn,
        int128 oraclePriceX18,
        IEndpoint.NlpPool[] calldata nlpPools,
        int128[] calldata nlpPoolRebalanceX18
    ) external onlyEndpoint {
        require(!RiskHelper.isIsolatedSubaccount(txn.sender), ERR_UNAUTHORIZED);

        ISpotEngine spotEngine = _spotEngine();
        spotEngine.updatePrice(NLP_PRODUCT_ID, oraclePriceX18);

        require(txn.nlpAmount <= INT128_MAX, ERR_CONVERSION_OVERFLOW);
        int128 nlpAmount = int128(txn.nlpAmount);
        require(
            spotEngine.getNlpUnlockedBalance(txn.sender).amount >= nlpAmount,
            ERR_UNLOCKED_NLP_INSUFFICIENT
        );
        int128 quoteAmount = nlpAmount.mul(oraclePriceX18);
        int128 burnFee = MathHelper.max(ONE, quoteAmount / 1000);
        quoteAmount = MathHelper.max(0, quoteAmount - burnFee);

        _validateNlpRebalance(nlpPools, nlpPoolRebalanceX18, -quoteAmount);
        for (uint128 i = 0; i < nlpPoolRebalanceX18.length; i++) {
            require(nlpPoolRebalanceX18[i] <= 0, ERR_INVALID_NLP_REBALANCE);
        }

        spotEngine.updateBalance(NLP_PRODUCT_ID, txn.sender, -nlpAmount);
        spotEngine.updateBalance(NLP_PRODUCT_ID, N_ACCOUNT, nlpAmount);

        if (quoteAmount > 0) {
            spotEngine.updateBalance(QUOTE_PRODUCT_ID, txn.sender, quoteAmount);
            _applyNlpRebalance(spotEngine, nlpPools, nlpPoolRebalanceX18);
        }

        require(
            spotEngine.getBalance(NLP_PRODUCT_ID, txn.sender).amount >= 0,
            ERR_SUBACCT_HEALTH
        );
        // Burning NLP can decrease health if the burn fee exceeds the health improvement
        // from the withdrawal. This check prevents malicious actors from deliberately
        // creating unhealthy subaccounts through NLP burns.
        require(
            getHealth(txn.sender, IProductEngine.HealthType.MAINTENANCE) >= 0,
            ERR_SUBACCT_HEALTH
        );
    }
```
