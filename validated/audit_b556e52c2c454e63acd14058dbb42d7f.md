Audit Report

## Title
SwapAllowlistExtension Checks Router Address Instead of Actual Swapper, Enabling Full Allowlist Bypass via MetricOmmSimpleRouter — (File: metric-periphery/contracts/extensions/SwapAllowlistExtension.sol)

## Summary
`SwapAllowlistExtension.beforeSwap` gates swaps by checking the `sender` argument, which resolves to `msg.sender` of the pool's `swap` call. When a user routes through `MetricOmmSimpleRouter`, the pool's `msg.sender` is the router contract, not the originating user. If a pool admin allowlists the router address (the natural configuration for enabling router-mediated access), every unprivileged user who calls any `exact*` function on the router bypasses the curated-pool gate entirely.

## Finding Description
`SwapAllowlistExtension.beforeSwap` performs the check:
```solidity
if (!allowAllSwappers[msg.sender] && !allowedSwapper[msg.sender][sender]) {
    revert IMetricOmmPoolActions.NotAllowedToSwap();
}
```
Here `msg.sender` is the pool (correct key) and `sender` is the first argument forwarded by `ExtensionCalling._beforeSwap`, which receives it from `MetricOmmPool.swap` as `msg.sender` — i.e., whoever called `pool.swap()`. When a user calls `MetricOmmSimpleRouter.exactInputSingle`, the router calls `IMetricOmmPoolActions(params.pool).swap(...)` directly, making the pool's `msg.sender` the router address. `ExtensionCalling` encodes that value verbatim as the `sender` argument to the extension. The router passes `params.extensionData` through unchanged and provides no mechanism to embed the original caller's identity. The same wrong-actor binding applies to `exactInput`, `exactOutputSingle`, and `exactOutput` paths. No existing guard in the extension, pool, or router corrects this mismatch.

## Impact Explanation
**Bypass path (High):** A pool admin who allowlists the router address (`allowedSwapper[pool][router] = true`) to enable router-mediated swaps for curated users inadvertently opens the gate to all callers. The extension checks `sender = router → true → passes`, so every user who calls any `exact*` function on the router trades on the curated pool regardless of individual allowlist status. The curated-pool access control is completely defeated.

**Broken-functionality path (Medium):** A pool admin who allowlists specific EOAs but not the router causes those EOAs to be unable to use the router at all — the extension sees `sender = router` and reverts with `NotAllowedToSwap`. Allowlisted users are forced to call the pool directly, bypassing the supported periphery path and breaking core swap functionality for the intended user set.

Both consequences are reachable by any unprivileged user without any privileged setup beyond the pool admin's ordinary configuration of the allowlist.

## Likelihood Explanation
The router is the primary user-facing entry point. Pool admins who deploy a `SwapAllowlistExtension` and also want users to access the pool through the router will naturally allowlist the router address — the contract provides no warning that doing so opens the gate to all callers. The bypass is reachable through a straightforward, expected admin configuration with no special attacker capability required beyond calling a public router function.

## Recommendation
The extension must gate the economic actor, not the intermediary. Two viable approaches:

1. **Pass original caller through `extensionData`:** Have the router encode `msg.sender` into `extensionData` and have `SwapAllowlistExtension` decode and verify it. This requires a coordinated interface change between router and extension.

2. **Trusted-router registry with user identity in payload:** The extension maintains a registry of trusted routers and requires them to supply the real user address in `extensionData`, falling back to `sender` for direct callers.

3. **Document and restrict:** Restrict the allowlist to direct-pool callers only and document that router access requires `allowAllSwappers = true`. Least invasive, but requires clear documentation and a separate mechanism for router-mediated curated access.

SwapAllowlistExtension is fully bypassed.
Any user who routes through MetricOmmSimpleRouter trades on the
