Audit Report

## Title
SwapAllowlistExtension Checks Router Address Instead of Original User, Enabling Full Allowlist Bypass - (File: metric-periphery/contracts/extensions/SwapAllowlistExtension.sol)

## Summary
`SwapAllowlistExtension.beforeSwap` gates swaps by checking the `sender` parameter, which is always `msg.sender` of the pool's `swap()` call. When a user routes through `MetricOmmSimpleRouter`, the router becomes `msg.sender` to the pool, so the extension checks whether the router is allowlisted rather than the originating user. A pool admin who allowlists the router to support router-mediated swaps for curated users inadvertently grants universal swap access to every user on the network.

## Finding Description
`MetricOmmPool.swap` passes `msg.sender` as the `sender` argument to `_beforeSwap`: [1](#0-0) 

`ExtensionCalling._beforeSwap` forwards that value unchanged as the first argument to the extension: [2](#0-1) 

`SwapAllowlistExtension.beforeSwap` then checks `allowedSwapper[msg.sender][sender]`, where `msg.sender` is the pool and `sender` is whoever called `pool.swap()`: [3](#0-2) 

In `exactInputSingle`, the router itself calls `pool.swap()`, making the router the `sender` delivered to the extension: [4](#0-3) 

The same pattern applies to `exactInput`, `exactOutputSingle`, and `exactOutput`: [5](#0-4) 

Therefore `sender` delivered to the extension is always the router address, never the originating EOA. Once the router is allowlisted, `allowedSwapper[pool][router] == true` for every call arriving through the router, regardless of who initiated the transaction. The extension cannot distinguish between an allowlisted user routing through the router and a completely non-allowlisted user doing the same.

## Impact Explanation
A pool admin who wants their allowlisted users to use the standard periphery must call `setAllowedToSwap(pool, router, true)`. Once done, `allowedSwapper[pool][router] == true` for every call that arrives through the router. Any non-allowlisted user can call `exactInputSingle` or any other router entry point and the extension will pass, completely defeating the curated-pool policy. The pool's LP assets are exposed to unrestricted trading that the allowlist was meant to prevent. This constitutes broken core pool functionality (the allowlist access control) causing direct exposure of LP assets to unauthorized traders — matching the "Admin-boundary break bypassed by an unprivileged path" and "broken core pool functionality causing loss of funds" allowed impact categories.

## Likelihood Explanation
The scenario requires the pool admin to allowlist the router, which is the natural and expected action for any admin who wants their allowlisted users to use the documented, supported swap entry point. There is no warning in the extension or its interface that allowlisting the router grants universal access. The bypass is reachable on any curated pool that supports router-mediated swaps, and is repeatable by any unprivileged caller with no special permissions.

## Recommendation
The extension must recover the original transaction initiator rather than trusting the `sender` argument forwarded by the pool. Two sound approaches:

1. **Pass `tx.origin` or an `originator` field** in the extension interface so the pool can forward the true initiator alongside `msg.sender`. This requires an interface change to `IMetricOmmExtensions.beforeSwap`.
2. **Check `tx.origin` inside the extension when `sender` is a known router**: gate on `tx.origin` when `sender` is a recognized router contract, or require `sender == tx.origin` for non-router callers.
3. At minimum, **document clearly** that allowlisting the router grants access to all router users, and require pool admins to use `allowAllSwappers` instead if that is the intent.

The cleanest fix is to add an `originator` field to the `beforeSwap` hook arguments so the pool can pass `tx.origin` alongside `msg.sender`.

## Proof of Concept
```
Setup:
  pool = MetricOmmPool with SwapAllowlistExtension as beforeSwap hook
  allowedUser  = 0xAAAA  (KYC'd, allowlisted)
  attacker     = 0xBBBB  (not allowlisted)
  router       = MetricOmmSimpleRouter

Admin actions:
  swapExt.setAllowedToSwap(pool, allowedUser, true)   // allowlist the real user
  swapExt.setAllowedToSwap(pool, router,      true)   // allowlist router so allowedUser can use it

Attack:
  vm.prank(attacker);
  router.exactInputSingle(ExactInputSingleParams({
      pool:            pool,
      recipient:       attacker,
      zeroForOne:      true,
      amountIn:        1_000e18,
      amountOutMinimum: 0,
      priceLimitX64:   0,
      tokenIn:         token0,
      extensionData:   ""
  }));
  // router calls pool.swap() → msg.sender to pool = router
  // extension checks allowedSwapper[pool][router] == true → passes
  // attacker receives token1 output; allowlist is bypassed
```
