Audit Report

## Title
`SwapAllowlistExtension` Checks Router Address Instead of Originating User, Allowing Any User to Bypass the Swap Allowlist via `MetricOmmSimpleRouter` — (`metric-periphery/contracts/extensions/SwapAllowlistExtension.sol`)

## Summary
`SwapAllowlistExtension.beforeSwap` gates swaps by checking `allowedSwapper[pool][sender]`, where `sender` is the direct `msg.sender` of `pool.swap()`. When users route through `MetricOmmSimpleRouter`, the router is `msg.sender` to the pool, so the extension checks the router address rather than the actual user. Allowlisting the router — the only way to let any allowlisted user use the standard periphery — simultaneously grants unrestricted swap access to every user, including those explicitly excluded from the allowlist.

## Finding Description
`MetricOmmPool.swap()` passes `msg.sender` as the `sender` argument to `_beforeSwap`:

```solidity
// metric-core/contracts/MetricOmmPool.sol:230-240
_beforeSwap(
    msg.sender,   // direct caller of pool.swap()
    recipient, ...
);
```

`SwapAllowlistExtension.beforeSwap` then checks that `sender` is allowlisted for the calling pool:

```solidity
// metric-periphery/contracts/extensions/SwapAllowlistExtension.sol:37-39
if (!allowAllSwappers[msg.sender] && !allowedSwapper[msg.sender][sender]) {
    revert IMetricOmmPoolActions.NotAllowedToSwap();
}
```

When a user calls `MetricOmmSimpleRouter.exactInputSingle`, the router calls `pool.swap()` directly:

```solidity
// metric-periphery/contracts/MetricOmmSimpleRouter.sol:72-80
(int128 amount0Delta, int128 amount1Delta) = IMetricOmmPoolActions(params.pool)
    .swap(params.recipient, params.zeroForOne, ...);
```

The pool's `msg.sender` is the router, so `sender` in the extension is the router address. The extension checks `allowedSwapper[pool][router]`, not `allowedSwapper[pool][actual_user]`. There is no mechanism in the extension or the call path to recover the originating user identity. The router does store the real payer in transient storage via `_setNextCallbackContext` for callback purposes, but this is never exposed to or read by extensions.

The broken invariant: the allowlist is designed to gate individual users. Instead, it gates the router as a single entity. Allowlisting the router (required for any allowlisted user to use the standard UX) simultaneously grants access to every user, including those explicitly not on the allowlist.

## Impact Explanation
A pool configured with `SwapAllowlistExtension` intends to restrict swaps to a curated set of addresses (e.g., KYC'd users, specific market makers, or protocol-controlled bots). Once the router is allowlisted — which is required for any allowlisted user to use the standard periphery — the gate is fully open to all users. Unauthorized users can swap against LP positions at live oracle prices, interact with pools designed to be restricted (private pools, institutional pools), and undermine any protocol-level access control that relies on the swap allowlist. This constitutes a direct loss-of-access-control impact on LP principal and pool integrity, matching the "Admin-boundary break: factory/oracle role checks are bypassed by an unprivileged path" impact gate.

## Likelihood Explanation
The router is the standard UX path for all users. Any pool admin who wants allowlisted users to be able to swap via the router must allowlist the router — this is the expected operational setup, not an edge case misconfiguration. The bypass is triggered by normal, expected pool configuration. Any unprivileged user can exploit it by simply calling `MetricOmmSimpleRouter.exactInputSingle` or `exactInput` targeting the restricted pool. No special privileges, flash loans, or complex setup are required.

## Recommendation
The extension must receive and check the originating user identity, not the direct pool caller. The preferred approach is to extend the transient storage pattern already used by the router for callback context (`_setNextCallbackContext`) to also carry the originating user. Before calling `pool.swap()`, the router writes the real `msg.sender` to a known transient slot; the extension reads it during `beforeSwap`. This requires a trusted encoding convention between the router and extension. Alternatively, the router can encode `msg.sender` into `extensionData`, and the extension decodes and checks it — but this requires the pool admin to configure the extension to trust the router's encoding.

## Proof of Concept
```
Setup:
  - Deploy pool with SwapAllowlistExtension as beforeSwap hook.
  - allowedSwapper[pool][router] = true   (admin allowlists router so Alice can use it)
  - allowedSwapper[pool][Alice]  = true
  - allowedSwapper[pool][Bob]    = false  (Bob is explicitly NOT allowlisted)

Attack:
  1. Bob calls MetricOmmSimpleRouter.exactInputSingle({pool: pool, ...}).
  2. Router calls pool.swap(recipient=Bob, ...) — msg.sender to pool = router.
  3. Pool calls _beforeSwap(sender=router, ...).
  4. SwapAllowlistExtension checks allowedSwapper[pool][router] → true → no revert.
  5. Bob's swap executes against LP positions despite being explicitly excluded.

Result:
  Bob bypasses the swap allowlist entirely. Any user can do the same.
  If the router is NOT allowlisted, Alice cannot use the router either,
  forcing all allowlisted users to implement IMetricOmmSwapCallback themselves.
```
