### Title
Hardcoded `LastKnownBonded = true` in `TierRedelegate` Causes Bonus Overpayment When Destination Validator Is Unbonded - (File: x/tieredrewards/keeper/msg_server.go)

### Summary

`MsgTierRedelegate` unconditionally sets `LastKnownBonded = true` for the destination validator regardless of its actual bonded status. When a user redelegates to a currently-unbonded validator, the stale `true` value is persisted as the starting bonded state for the next bonus replay. When the validator later bonds, `processEventsAndClaimBonus` computes bonus for the entire period since the redelegate — including the unbonded gap — and pays it out from the rewards pool.

### Finding Description

In `msg_server.go`, after a successful redelegate, the position's bonus checkpoints are updated:

```go
pos.UpdateBonusCheckpoints(latestSeq, sdkCtx.BlockTime(), true)
```

The third argument (`true`) hardcodes `LastKnownBonded = true` unconditionally. [1](#0-0) 

`latestSeq` is set to the latest event sequence for the destination validator at redelegate time. If the destination validator is currently unbonded, its UNBOND event is at sequence ≤ `latestSeq` and is therefore skipped by the next replay. The position's `LastEventSeq` is advanced past the UNBOND event, but `LastKnownBonded` is set to `true` instead of `false`. [2](#0-1) 

In `processEventsAndClaimBonus`, the bonus loop starts with `bonded = pos.LastKnownBonded`:

```go
bonded := pos.LastKnownBonded
segmentStart := pos.LastBonusAccrual
...
for _, entry := range events {
    evt := entry.Event
    if bonded {
        bonus := k.computeSegmentBonus(*pos, tier, segmentStart, evt.Timestamp, evt.TokensPerShare)
        totalBonus = totalBonus.Add(bonus)
    }
    ...
}
``` [3](#0-2) 

When the validator subsequently bonds (BOND event at seq `latestSeq+1`), the loop processes it with `bonded = true` and computes bonus for the segment `[redelegate_time, BOND_event_time]` — a period during which the validator was actually unbonded. No bonus should accrue during this gap.

`validateRedelegatePosition` does not check whether the destination validator is bonded: [4](#0-3) 

The Cosmos SDK `BeginRedelegate` does not require the destination validator to be bonded, so the transaction succeeds.

### Impact Explanation

An unprivileged user with a tier position can drain the `RewardsPoolName` module account by:
1. Redelegating to an unbonded validator (sets `LastKnownBonded = true` incorrectly).
2. Waiting for the validator to bond.
3. Claiming rewards — receiving bonus for the entire unbonded gap.

The corrupted value is the `RewardsPoolName` module account balance: tokens are transferred out via `SendCoinsFromModuleToAccount` for a period during which no bonus should have accrued. [5](#0-4) 

### Likelihood Explanation

Any user with a tier position can execute this. Unbonded validators exist on any live network (jailed validators, validators that have been removed from the active set). The attacker only needs to redelegate to one and wait for it to rejoin. No privileged access is required. The entry point is the standard `MsgTierRedelegate` transaction.

### Recommendation

Replace the hardcoded `true` with the actual bonded state of the destination validator at redelegate time:

```go
dstVal, err := ms.stakingKeeper.GetValidator(ctx, dstValAddr)
if err != nil {
    return nil, err
}
pos.UpdateBonusCheckpoints(latestSeq, sdkCtx.BlockTime(), dstVal.IsBonded())
```

This mirrors the pattern already used in the final-segment guard of `processEventsAndClaimBonus` (`bonded && val.IsBonded()`). [6](#0-5) 

### Proof of Concept

1. User holds a tier position on bonded validator A.
2. Validator B is currently unbonded (UNBOND event at seq `N`).
3. User submits `MsgTierRedelegate` to validator B.
   - `latestSeq = N`, `LastBonusAccrual = blockTime_T0`, `LastKnownBonded = true` (wrong).
4. 30 days pass. Validator B bonds (BOND event at seq `N+1`, timestamp `T1`).
5. User submits `MsgClaimTierRewards`.
   - `bonded = true` (from `LastKnownBonded`).
   - Loop processes BOND event: computes `computeSegmentBonus(pos, tier, T0, T1, rate)` with `bonded = true`.
   - Bonus for 30 unbonded days is paid from the rewards pool.
   - Correct behavior: zero bonus for this segment.

The overpayment scales with position size, APY, and the length of the unbonded gap. [7](#0-6)

func (k Keeper) validateRedelegatePosition(ctx context.Context, pos types.PositionState, owner, dstValidator string) error {
	if !pos.IsOwner(owner) {
		return types.ErrNotPositionOwner
	}

	if !pos.IsDelegated() {
		return types.ErrPositionNotDelegated
	}

	if pos.Delegation.ValidatorAddress == dstValidator {
		return types.ErrRedelegationToSameValidator
	}

	sdkCtx := sdk.UnwrapSDKContext(ctx)
	if pos.HasTriggeredExit() && pos.CompletedExitLockDuration(sdkCtx.BlockTime()) {
		return types.ErrExitLockDurationElapsed
	}

	tier, err := k.getTier(ctx, pos.TierId)
	if err != nil {
		return err
	}

	if tier.IsCloseOnly() {
		return types.ErrTierIsCloseOnly
	}

	isRedelegating, err := k.isRedelegating(ctx, pos.DelegatorAddress)
	if err != nil {
		return err
	}
	if isRedelegating {
		return errorsmod.Wrapf(types.ErrActiveRedelegation, "position %d has an active redelegation", pos.Id)
	}

	return nil
}

func (k Keeper) computeSegmentBonus(pos types.PositionState, tier types.Tier, segmentStart, segmentEnd time.Time, tokensPerShare math.LegacyDec) math.Int {
	if !pos.ExitUnlockAt.IsZero() && segmentEnd.After(pos.ExitUnlockAt) {
		segmentEnd = pos.ExitUnlockAt
	}

	if !segmentEnd.After(segmentStart) {
		return math.ZeroInt()
	}

	durationSeconds := int64(segmentEnd.Sub(segmentStart) / time.Second)
	if durationSeconds <= 0 {
		return math.ZeroInt()
	}

	tokens := pos.Delegation.Shares.Mul(tokensPerShare)

	return tokens.
		Mul(tier.BonusApy).
		MulInt64(durationSeconds).
		QuoInt64(types.SecondsPerYear).
		TruncateInt()
}
