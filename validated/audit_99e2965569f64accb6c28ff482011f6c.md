### Title
`SwapAllowlistExtension` Gates the Router Address Instead of the End User, Allowing Any Caller to Bypass the Swap Allowlist via `MetricOmmSimpleRouter` — (`metric-periphery/contracts/extensions/SwapAllowlistExtension.sol`)

---

### Summary

`SwapAllowlistExtension.beforeSwap` checks the `sender` argument, which the pool sets to `msg.sender` of the `pool.swap()` call. When a user routes through `MetricOmmSimpleRouter`, `msg.sender` to the pool is the router contract, not the end user. If the pool admin allowlists the router (the natural step to enable router-based swaps), every unprivileged user can bypass the curated allowlist by routing through the router.

---

### Finding Description

`MetricOmmPool.swap` passes its own `msg.sender` as the `sender` argument to `_beforeSwap`: [1](#0-0) 

`ExtensionCalling._beforeSwap` forwards that value unchanged to the extension: [2](#0-1) 

`SwapAllowlistExtension.beforeSwap` then checks `allowedSwapper[msg.sender][sender]`, where `msg.sender` is the pool and `sender` is whoever called `pool.swap()`: [3](#0-2) 

When a user calls `MetricOmmSimpleRouter.exactInputSingle`, the router calls `pool.swap(...)` directly: [4](#0-3) 

At that point `msg.sender` to the pool is the router address, so `sender` delivered to the extension is the router, not the end user. The allowlist check becomes `allowedSwapper[pool][router]`. If the pool admin has allowlisted the router (the only way to permit any router-mediated swap), every caller of the router passes the check regardless of their own allowlist status.

---

### Impact Explanation

A pool admin deploys a curated pool with `SwapAllowlistExtension` to restrict trading to a set of known counterparties. To let those counterparties use the standard router, the admin adds the router to `allowedSwapper`. From that moment, any address in the world can call `router.exactInputSingle` and trade against the pool. The allowlist is completely nullified for the router path. Unauthorized traders can execute swaps at oracle-anchored prices, extracting value from LP positions or front-running curated participants, causing direct loss of LP principal and fee revenue.

---

### Likelihood Explanation

The router is the primary user-facing entry point for swaps. Any pool admin who wants their allowlisted users to trade via the router must allowlist the router address. This is the expected operational pattern. Once the router is allowlisted, the bypass requires no special knowledge: any user calls `exactInputSingle` with the target pool. The trigger is unprivileged and requires no setup beyond the admin's own legitimate configuration.

---

### Recommendation

The extension must check the original end-user identity, not the intermediate caller. Two options:

1. **Pass the original initiator through the router.** The router already stores the real payer in transient storage (`_getPayer()`). The pool could accept an explicit `originator` argument, or the router could pass it in `extensionData` for the extension to decode.

2. **Check `sender` against the real user in the extension.** If the pool's `sender` is a known periphery contract (router), the extension should look up the actual payer from transient storage or require the router to attest the real user in `extensionData`.

The simplest correct fix is to have `SwapAllowlistExtension.beforeSwap` check the real end-user address, which the router must supply in `extensionData` and attest with its own identity, rather than relying on the raw `sender` argument that collapses to the router address for all router-mediated swaps.

---

### Proof of Concept

```
Setup:
  pool configured with SwapAllowlistExtension
  allowedSwapper[pool][router] = true   // admin adds router so legitimate users can trade
  allowedSwapper[pool][alice] = true    // alice is a curated counterparty
  allowedSwapper[pool][attacker] = false // attacker is NOT on the allowlist

Attack:
  attacker calls router.exactInputSingle({pool: pool, ...})
    → router calls pool.swap(recipient, ...)
    → pool calls _beforeSwap(msg.sender=router, ...)
    → SwapAllowlistExtension.beforeSwap(sender=router, ...)
    → checks allowedSwapper[pool][router] == true  ✓
    → swap executes — attacker bypasses the allowlist entirely

Direct call (blocked correctly):
  attacker calls pool.swap(...) directly
    → _beforeSwap(msg.sender=attacker, ...)
    → checks allowedSwapper[pool][attacker] == false  ✗  → NotAllowedToSwap
```

The bypass is reachable on every `exactInputSingle`, `exactInput`, `exactOutputSingle`, and `exactOutput` call through `MetricOmmSimpleRouter` whenever the router address is in `allowedSwapper`. [5](#0-4) [6](#0-5) [1](#0-0)
