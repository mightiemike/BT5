### Title
`Treehasher.sha256_treehash` closures resolve `shatree_atom`/`shatree_pair` from module scope (always `CHIA_TREEHASHER`), silently ignoring custom prefixes — (`wheel/python/clvm_rs/tree_hash.py`)

### Summary

`Treehasher.sha256_treehash` defines two inner closures (`handle_obj`, `handle_pair`) that call `shatree_atom` and `shatree_pair` as free variables. Python's LEGB scoping resolves these names at the **module level**, where they are permanently bound to `CHIA_TREEHASHER`'s methods. Any `Treehasher` instance with non-standard `atom_prefix`/`pair_prefix` silently produces CHIA-prefixed hashes regardless of its configuration.

### Finding Description

At module initialization, lines 108–109 create module-level aliases:

```python
shatree_atom = CHIA_TREEHASHER.shatree_atom   # line 108
shatree_pair = CHIA_TREEHASHER.shatree_pair   # line 109
``` [1](#0-0) 

Inside `sha256_treehash`, the `handle_obj` closure calls the bare name `shatree_atom`:

```python
r = shatree_atom(obj.atom)   # line 63 — resolves to CHIA_TREEHASHER.shatree_atom
``` [2](#0-1) 

And `handle_pair` calls the bare name `shatree_pair`:

```python
r = shatree_pair(p0, p1)     # line 86 — resolves to CHIA_TREEHASHER.shatree_pair
``` [3](#0-2) 

Neither closure references `self`. Python's LEGB rule finds `shatree_atom` and `shatree_pair` in the **global (module) scope**, not in any enclosing scope of `sha256_treehash`. The `self.atom_prefix` and `self.pair_prefix` of the custom instance are never consulted during the iterative hash walk. [4](#0-3) 

### Impact Explanation

Any caller that instantiates `Treehasher(custom_atom_prefix, custom_pair_prefix)` and calls `sha256_treehash` receives hashes computed with `b'\x01'`/`b'\x02'` (CHIA prefixes), not their custom prefixes. The result is silently wrong — no exception is raised, no warning is emitted. For non-standard dialects that rely on a different prefix to compute puzzle hashes, every computed hash is incorrect, producing a different hash space than intended. This breaks API-equivalence for any non-CHIA dialect consumer of the public `Treehasher` API. [5](#0-4) 

### Likelihood Explanation

The `Treehasher` class is a public, documented API with explicit `atom_prefix`/`pair_prefix` constructor parameters, signaling intent to support non-CHIA configurations. The bug is triggered by the ordinary, intended use of the class. No special privileges or attacker access are required — any caller using a non-CHIA `Treehasher` hits this path. [6](#0-5) 

### Recommendation

Replace the bare `shatree_atom` and `shatree_pair` calls inside the closures with `self.shatree_atom` and `self.shatree_pair`. Since the closures are defined inside an instance method, `self` is available in the enclosing scope and will be captured correctly by the closure:

```python
# handle_obj, line 63:
r = self.shatree_atom(obj.atom)

# handle_pair, line 86:
r = self.shatree_pair(p0, p1)
``` [7](#0-6) 

### Proof of Concept

```python
from hashlib import sha256
from clvm_rs.tree_hash import Treehasher, CHIA_TREE_HASH_ATOM_PREFIX

class Leaf:
    atom = b"hello"
    pair = None

custom = Treehasher(b'\xAA', b'\xBB')
result = custom.sha256_treehash(Leaf())

# What sha256_treehash actually computes (CHIA prefix b'\x01'):
chia_hash = sha256(CHIA_TREE_HASH_ATOM_PREFIX + b"hello").digest()

# What it should compute (custom prefix b'\xAA'):
correct_hash = sha256(b'\xAA' + b"hello").digest()

assert result == chia_hash    # PASSES — bug confirmed
assert result == correct_hash # FAILS  — custom prefix silently ignored
``` [8](#0-7)

class Treehasher:
    """
    `Treehasher` performs the standard sha256tree hashing in a non-recursive
    way so that extremely large objects don't blow out the python stack.

    We also force a `_cached_sha256_treehash` into the hashed sub-objects
    whenever possible so that taking the hash of the same sub-tree is
    more efficient in future.
    """

    atom_prefix: bytes
    pair_prefix: bytes
    cache_hits: int

    def __init__(self, atom_prefix: bytes, pair_prefix: bytes):
        self.atom_prefix = atom_prefix
        self.pair_prefix = pair_prefix
        self.cache_hits = 0

    def shatree_atom(self, atom: bytes) -> bytes:
        s = sha256()
        s.update(self.atom_prefix)
        s.update(atom)
        return s.digest()
```

    def sha256_treehash(self, clvm_storage: CLVMStorage) -> bytes:
        def handle_obj(
            obj_stack: List[CLVMStorage],
            hash_stack: List[bytes],
            op_stack: List[OP_STACK_F],
        ) -> None:
            obj = obj_stack.pop()
            r = getattr(obj, "_cached_sha256_treehash", None)
            if r is not None:
                self.cache_hits += 1
                hash_stack.append(r)
                return
            elif obj.atom is not None:
                r = shatree_atom(obj.atom)
                hash_stack.append(r)
                try:
                    setattr(obj, "_cached_sha256_treehash", r)
                except AttributeError:
                    pass
            else:
                pair = cast(Tuple[CLVMStorage, CLVMStorage], obj.pair)
                p0, p1 = pair
                obj_stack.append(obj)
                obj_stack.append(p0)
                obj_stack.append(p1)
                op_stack.append(handle_pair)
                op_stack.append(handle_obj)
                op_stack.append(handle_obj)

        def handle_pair(
            obj_stack: List[CLVMStorage],
            hash_stack: List[bytes],
            op_stack: List[OP_STACK_F],
        ) -> None:
            p0 = hash_stack.pop()
            p1 = hash_stack.pop()
            r = shatree_pair(p0, p1)
            hash_stack.append(r)
            obj = obj_stack.pop()
            try:
                setattr(obj, "_cached_sha256_treehash", r)
            except AttributeError:
                pass

        obj_stack: List[CLVMStorage] = [clvm_storage]
        op_stack: List[OP_STACK_F] = [handle_obj]
        hash_stack: List[bytes] = []
        while len(op_stack) > 0:
            op: OP_STACK_F = op_stack.pop()
            op(obj_stack, hash_stack, op_stack)
