### Title
`run_mainchain_gc` Lacks Caller Access Control, Allowing Any Account to Prematurely Purge Canonical Chain Headers — (File: `contract/src/lib.rs`)

### Summary
`run_mainchain_gc` is a public, state-mutating NEAR contract method that carries only a `#[pause]` guard. There is no restriction on *who* may call it. Any unprivileged NEAR account can invoke it with an arbitrarily large `batch_size`, immediately purging the maximum number of mainchain block headers in a single transaction and permanently advancing `mainchain_initial_blockhash`. This corrupts the canonical chain's stored state and causes `verify_transaction_inclusion` / `verify_transaction_inclusion_v2` to reject proofs for blocks that should still be within the retention window.

### Finding Description
`submit_blocks` — the only intended write path — is protected by both `#[pause]` and `#[trusted_relayer]`, which restricts callers to staked, approved relayers. [1](#0-0) 

`run_mainchain_gc` is also called internally from `submit_blocks` with `batch_size = num_of_headers` (the count of headers in that submission batch), so GC is intentionally incremental. [2](#0-1) 

However, `run_mainchain_gc` itself is separately exposed as a `pub` method with only a `#[pause]` decorator: [3](#0-2) 

The `#[pause(except(roles(Role::UnrestrictedRunGC)))]` macro only governs whether the function executes when the contract is in a paused state; it places no restriction on the caller's identity when the contract is live. Any NEAR account can call it directly.

Inside the function, `selected_amount_to_remove = min(total_amount_to_remove, batch_size)`. Passing `u64::MAX` as `batch_size` causes the full `total_amount_to_remove = amount_of_headers_we_store - gc_threshold` headers to be deleted in one call: [4](#0-3) 

The loop removes each header from both `mainchain_height_to_header` and `headers_pool`, then advances `mainchain_initial_blockhash` to the new oldest block. These deletions are permanent — there is no way to recover purged headers.

### Impact Explanation
After a successful attack call:
- All mainchain block headers older than `gc_threshold` blocks from the current tip are immediately deleted from `headers_pool` and `mainchain_height_to_header`.
- `mainchain_initial_blockhash` is permanently advanced.
- Any subsequent call to `verify_transaction_inclusion` or `verify_transaction_inclusion_v2` for a transaction in a purged block panics with `"block does not belong to the current main chain"`, because the block hash is no longer present in `mainchain_header_to_height`. [5](#0-4) 

Downstream NEAR contracts that rely on SPV proofs for bridge operations, settlement, or custody will receive permanent proof failures for any transaction confirmed in a purged block, even if that block was well within the intended retention window at the time of the attack.

### Likelihood Explanation
The entry point is a public NEAR contract method requiring no stake, no role, and no deposit. Any NEAR account can call it in a single transaction. The contract is deployed on a public network and the method is visible in the ABI. The attacker needs only to know the contract address and the method name.

### Recommendation
Add a caller restriction to `run_mainchain_gc` so that it can only be invoked internally (i.e., from `submit_blocks`) or by an account holding a privileged role such as `Role::DAO` or a dedicated `GCManager` role. The simplest fix is to make the public method `#[private]` or to add a `require!(env::predecessor_account_id() == env::current_account_id(), ...)` guard, and expose a separate role-gated wrapper if external GC triggering is needed.

### Proof of Concept
1. Deploy the contract with `gc_threshold = 52704` and a synced chain of, say, 53000 headers.
2. From any unprivileged NEAR account, call:
   ```
   run_mainchain_gc(18446744073709551615)  // u64::MAX
   ```
3. The function computes `total_amount_to_remove = 53000 - 52704 = 296` and immediately removes 296 headers from `headers_pool` and `mainchain_height_to_header`, advancing `mainchain_initial_blockhash` by 296 blocks.
4. Any call to `verify_transaction_inclusion` for a transaction in one of those 296 now-deleted blocks panics — the block hash is gone from `mainchain_header_to_height`.
5. Under normal operation (via `submit_blocks`), those same 296 headers would have been removed only gradually, one batch at a time, as new headers arrived. The attacker collapses that entire window instantly. [6](#0-5)

```rust
    /// Public call to run GC on a mainchain.
    /// `batch_size` is how many block headers should be removed in the execution
    ///
    /// # Panics
    /// If initial blockheader or tip blockheader are not in a header pool
    #[pause(except(roles(Role::UnrestrictedRunGC)))]
    pub fn run_mainchain_gc(&mut self, batch_size: u64) {
        let initial_blockheader = self
            .headers_pool
            .get(&self.mainchain_initial_blockhash)
            .unwrap_or_else(|| env::panic_str(ERR_KEY_NOT_EXIST));

        let tip_blockheader = self
            .headers_pool
            .get(&self.mainchain_tip_blockhash)
            .unwrap_or_else(|| env::panic_str(ERR_KEY_NOT_EXIST));

        let amount_of_headers_we_store =
            tip_blockheader.block_height - initial_blockheader.block_height + 1;

        if amount_of_headers_we_store > self.gc_threshold {
            let total_amount_to_remove = amount_of_headers_we_store - self.gc_threshold;
            let selected_amount_to_remove = std::cmp::min(total_amount_to_remove, batch_size);

            let start_removal_height = initial_blockheader.block_height;
            let end_removal_height = initial_blockheader.block_height + selected_amount_to_remove;
            env::log_str(&format!(
                "Num of blocks to remove {selected_amount_to_remove}"
            ));

            for height in start_removal_height..end_removal_height {
                let blockhash = &self
                    .mainchain_height_to_header
                    .get(&height)
                    .unwrap_or_else(|| env::panic_str(ERR_KEY_NOT_EXIST));

                self.remove_block_header(blockhash);
                self.mainchain_height_to_header.remove(&height);
            }

            self.mainchain_initial_blockhash = self
                .mainchain_height_to_header
                .get(&end_removal_height)
                .unwrap_or_else(|| env::panic_str(ERR_KEY_NOT_EXIST));
        }
    }
```
