Audit Report

## Title
SwapAllowlistExtension gates the router address instead of the actual swapper, enabling full allowlist bypass via MetricOmmSimpleRouter — (`metric-periphery/contracts/extensions/SwapAllowlistExtension.sol`)

## Summary
`SwapAllowlistExtension.beforeSwap` checks the `sender` argument, which the pool sets to `msg.sender` of `pool.swap()`. When a user routes through `MetricOmmSimpleRouter`, `msg.sender` of `pool.swap()` is the router contract address, not the originating user. Because the pool admin must allowlist the router address to permit any router-mediated swap for legitimate users, every non-allowlisted user can bypass the allowlist entirely by routing through the router.

## Finding Description
In `MetricOmmPool.swap()`, `msg.sender` is forwarded as the `sender` argument to `_beforeSwap`:

```solidity
// metric-core/contracts/MetricOmmPool.sol L230-240
_beforeSwap(
    msg.sender,   // ← always the direct caller of pool.swap()
    recipient,
    ...
    extensionData
);
```

`ExtensionCalling._beforeSwap` re-encodes this value and dispatches it to every configured extension:

```solidity
// metric-core/contracts/ExtensionCalling.sol L160-176
_callExtensionsInOrder(
    BEFORE_SWAP_ORDER,
    abi.encodeCall(
        IMetricOmmExtensions.beforeSwap,
        (sender, ...)
    )
);
```

`SwapAllowlistExtension.beforeSwap` then gates on that `sender`, where `msg.sender` is the pool:

```solidity
// metric-periphery/contracts/extensions/SwapAllowlistExtension.sol L37-39
if (!allowAllSwappers[msg.sender] && !allowedSwapper[msg.sender][sender]) {
    revert IMetricOmmPoolActions.NotAllowedToSwap();
}
```

`MetricOmmSimpleRouter.exactInputSingle` (and all other swap entry points) calls `pool.swap()` directly, making the router the `msg.sender` of the pool call:

```solidity
// metric-periphery/contracts/MetricOmmSimpleRouter.sol L72-80
(int128 amount0Delta, int128 amount1Delta) = IMetricOmmPoolActions(params.pool)
    .swap(
        params.recipient,
        params.zeroForOne,
        MetricOmmSwapInputs.asAmountSpecifiedIn(params.amountIn),
        priceLimitX64,
        "",
        params.extensionData
    );
```

The router passes no information about the originating user to the pool. The allowlist check therefore evaluates `allowedSwapper[pool][router_address]` rather than `allowedSwapper[pool][user_address]`. The pool admin has no way to allowlist individual users for router-mediated swaps — the only option is to allowlist the router contract itself. Once the router is allowlisted, every address in the world can call the router and bypass the allowlist.

**Exploit path:**
1. Pool admin deploys a pool with `SwapAllowlistExtension` configured, intending to restrict swaps to a set of KYC'd addresses.
2. Admin allowlists user A and also allowlists `MetricOmmSimpleRouter` so user A can swap via the router.
3. Unprivileged user B (not allowlisted) calls `MetricOmmSimpleRouter.exactInputSingle()` targeting the restricted pool.
4. The router calls `pool.swap()` — `msg.sender` is the router, which is allowlisted.
5. `SwapAllowlistExtension.beforeSwap` passes: `allowedSwapper[pool][router] == true`.
6. User B's swap executes successfully, bypassing the allowlist entirely.

## Impact Explanation
The `SwapAllowlistExtension` is a core access-control extension whose sole purpose is to restrict which addresses may swap in a pool. The bypass renders this control completely ineffective for any pool that permits router-mediated swaps. This constitutes a broken core pool functionality (the allowlist gate) and an admin-boundary break: the pool admin's configured access restriction is defeated by any unprivileged user routing through `MetricOmmSimpleRouter`. Severity: Medium — no direct loss of principal, but the intended access control invariant is fully broken for any pool using this extension with the router.

## Likelihood Explanation
The condition is trivially reachable: any unprivileged user who knows the pool uses `SwapAllowlistExtension` can call `MetricOmmSimpleRouter` directly. No special privileges, flash loans, or oracle manipulation are required. The router is a public, permissionless contract. The bypass is repeatable on every block.

## Recommendation
The extension should verify the originating user rather than the direct pool caller. Two approaches:
1. **Pass `tx.origin` or a user-supplied address through `extensionData`**: The router encodes the originating user in `extensionData`, and the extension reads and verifies it. This requires a coordinated change to the router and extension.
2. **Check `tx.origin` in the extension**: Replace or supplement the `sender` check with `tx.origin` to capture the EOA initiating the transaction. This is simpler but incompatible with smart-contract wallets.
3. **Preferred**: Redesign the allowlist to gate on the `recipient` field (which the router sets to the user-supplied `params.recipient`) rather than `sender`, if the intent is to restrict who receives swap output. Alternatively, require the pool admin to explicitly allowlist the router and accept that router-mediated swaps are open to all, documenting this limitation clearly.

## Proof of Concept
```solidity
// Foundry test sketch
function test_allowlistBypassViaRouter() public {
    // Setup: pool with SwapAllowlistExtension, only `allowedUser` is allowlisted
    // Router is allowlisted so allowedUser can use it
    swapAllowlist.setAllowedToSwap(address(pool), address(router), true);

    // Attacker is NOT allowlisted
    address attacker = makeAddr("attacker");
    deal(token0, attacker, 1e18);

    vm.startPrank(attacker);
    IERC20(token0).approve(address(router), type(uint256).max);

    // This should revert with NotAllowedToSwap but succeeds
    router.exactInputSingle(IMetricOmmSimpleRouter.ExactInputSingleParams({
        pool: address(pool),
        tokenIn: token0,
        recipient: attacker,
        zeroForOne: true,
        amountIn: 1e18,
        amountOutMinimum: 0,
        priceLimitX64: 0,
        deadline: block.timestamp,
        extensionData: ""
    }));
    vm.stopPrank();
    // Attacker successfully swapped despite not being on the allowlist
}
```
