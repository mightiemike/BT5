### Title
`TierUndelegate` (and other lifecycle messages) revert when bonus pool is empty, permanently blocking user exits until third-party intervention - (`x/tieredrewards/keeper/msg_server.go`)

### Summary

`TierUndelegate`, `TierRedelegate`, `AddToTierPosition`, `ClearPosition`, and `ExitTierWithDelegation` all call `claimRewards()` as an embedded sub-operation before performing their primary action. `claimRewards()` calls `processEventsAndClaimBonus()`, which hard-fails with `ErrInsufficientBonusPool` if the `RewardsPoolName` module account cannot cover accrued bonus. Because the error propagates unconditionally, the entire lifecycle transaction reverts. A user who has completed their exit lock duration and has any accrued bonus cannot call `TierUndelegate` — the mandatory prerequisite for `WithdrawFromTier` — until a third party (governance) replenishes the pool.

### Finding Description

`TierUndelegate` in `x/tieredrewards/keeper/msg_server.go` calls `claimRewards()` at line 166 before it calls `undelegate()`:

```go
pos, _, _, err = ms.claimRewards(ctx, pos)
if err != nil {
    return nil, err   // entire tx reverts
}
// ... undelegate never reached
```

`claimRewards()` in `x/tieredrewards/keeper/claim_rewards.go` calls `processEventsAndClaimBonus()` at line 97. Inside `processEventsAndClaimBonus()`, at line 230, the pool balance is checked:

```go
if err := k.sufficientBonusPoolBalance(ctx, bonusCoins); err != nil {
    return nil, err   // propagates up through claimRewards → TierUndelegate
}
```

`sufficientBonusPoolBalance()` in `x/tieredrewards/keeper/bonus_rewards.go` returns `ErrInsufficientBonusPool` whenever the pool balance is below the accrued bonus:

```go
if !poolBalance.IsAllGTE(bonus) {
    return errorsmod.Wrapf(types.ErrInsufficientBonusPool, ...)
}
```

The same `claimRewards()` call is embedded in:
- `TierRedelegate` (msg_server.go line 229)
- `AddToTierPosition` (msg_server.go line 314)
- `ClearPosition` (msg_server.go line 406)
- `ExitTierWithDelegation` (msg_server.go line 532)

`WithdrawFromTier` does **not** call `claimRewards()`, but it requires the position to be undelegated first (`validateWithdrawFromTier` returns `ErrPositionDelegated` if `pos.IsDelegated()`). Therefore, a user who cannot call `TierUndelegate` also cannot call `WithdrawFromTier`, and their locked CRO tokens are inaccessible.

The existing test `TestMsgTierUndelegate_Basic` explicitly funds the pool before undelegating with the comment `// Fund the rewards pool so bonus claim doesn't fail`, confirming the revert path is real.

Notably, the `BeforeRedelegationSlashed` hook handles pool insufficiency **silently** (bonus forfeits, chain does not halt), per ADR-006. The user-facing message path has no equivalent graceful degradation.

### Impact Explanation

A user who has locked CRO in a tier position, triggered exit, and waited the full exit lock duration cannot undelegate their position if the bonus pool is empty. Their locked CRO tokens remain in the tier module's per-position delegator account and cannot be recovered via `WithdrawFromTier` until `TierUndelegate` succeeds. Recovery requires governance to replenish the `RewardsPoolName` module account — third-party intervention outside the user's control.

The corrupted invariant: a user who has satisfied all exit preconditions (exit triggered, lock duration elapsed, position delegated) must be able to undelegate. This invariant is violated whenever accrued bonus > pool balance.

### Likelihood Explanation

The bonus pool is funded by governance deposits and drained by every `ClaimTierRewards`, `TierUndelegate`, `TierRedelegate`, `AddToTierPosition`, `ClearPosition`, and `ExitTierWithDelegation` call. As the number of positions and time elapsed grow, the pool drains faster. Any period where governance has not topped up the pool — including routine operational gaps — will block all users who have accrued any non-zero bonus from completing their exit flow. No attacker action is required; normal protocol usage is sufficient to trigger the condition.

### Recommendation

Decouple the bonus reward claim from the lifecycle operations. When the pool is insufficient, either:

1. **Forfeit the bonus silently and proceed** (matching the `BeforeRedelegationSlashed` hook behavior): skip `processEventsAndClaimBonus()` if `sufficientBonusPoolBalance()` fails, advance the checkpoint, and allow `TierUndelegate` / `ExitTierWithDelegation` to complete. The forfeited bonus is lost but the user can exit.
2. **Defer the bonus claim**: record the owed bonus as a claimable debt and allow the lifecycle operation to proceed unconditionally. The user can claim the bonus later once the pool is replenished.

### Proof of Concept

1. User calls `LockTier` and locks 10,000 CRO in Tier 1 on a bonded validator.
2. 365 days pass; significant bonus accrues (e.g., 500 CRO at 5% APY).
3. User calls `TriggerExitFromTier`; waits for `ExitUnlockAt`.
4. The `RewardsPoolName` module account balance drops to 0 (drained by other users' claims).
5. User calls `TierUndelegate` — transaction reverts with `ErrInsufficientBonusPool` because `processEventsAndClaimBonus()` at `claim_rewards.go:230` fails.
6. User calls `ExitTierWithDelegation` as an alternative — also reverts at `msg_server.go:532` for the same reason.
7. User calls `WithdrawFromTier` — reverts with `ErrPositionDelegated` because the position was never undelegated.

func (k Keeper) claimRewards(ctx context.Context, pos types.PositionState) (types.PositionState, sdk.Coins, sdk.Coins, error) {
	if !pos.IsDelegated() {
		return pos, sdk.NewCoins(), sdk.NewCoins(), nil
	}

	base, err := k.claimBaseRewards(ctx, pos)
	if err != nil {
		return types.PositionState{}, nil, nil, err
	}

	bonus, err := k.processEventsAndClaimBonus(ctx, &pos)
	if err != nil {
		return types.PositionState{}, nil, nil, err
	}

	return pos, base, bonus, nil
}

func (k Keeper) sufficientBonusPoolBalance(ctx context.Context, bonus sdk.Coins) error {
	if bonus.IsZero() {
		return nil
	}

	poolAddr := k.accountKeeper.GetModuleAddress(types.RewardsPoolName)
	poolBalance := k.bankKeeper.GetAllBalances(ctx, poolAddr)
	if !poolBalance.IsAllGTE(bonus) {
		return errorsmod.Wrapf(types.ErrInsufficientBonusPool,
			"bonus: %s, pool balance: %s",
			bonus.String(), poolBalance.String())
	}

	return nil
}

func (k Keeper) validateWithdrawFromTier(ctx context.Context, pos types.PositionState, owner string) error {
	if !pos.IsOwner(owner) {
		return types.ErrNotPositionOwner
	}

	if !pos.HasTriggeredExit() {
		return types.ErrExitNotTriggered
	}

	sdkCtx := sdk.UnwrapSDKContext(ctx)
	if !pos.CompletedExitLockDuration(sdkCtx.BlockTime()) {
		return types.ErrExitLockDurationNotReached
	}

	if pos.IsDelegated() {
		return types.ErrPositionDelegated
	}

	isUnbonding, err := k.isUnbonding(ctx, pos.DelegatorAddress)
	if err != nil {
		return err
	}
	if isUnbonding {
		return types.ErrPositionUnbonding
	}

	return nil
}
