### Title
SwapAllowlistExtension Checks Router Address as `sender`, Enabling Full Allowlist Bypass via MetricOmmSimpleRouter — (`metric-periphery/contracts/extensions/SwapAllowlistExtension.sol`)

---

### Summary

`SwapAllowlistExtension.beforeSwap` gates swaps by checking the `sender` argument, which the pool sets to `msg.sender` of the `pool.swap()` call. When a user routes through `MetricOmmSimpleRouter`, `msg.sender` of `pool.swap()` is the **router contract**, not the user. A pool admin who allowlists the router to enable router-mediated swaps for their curated users inadvertently opens the pool to **all** users, completely defeating the allowlist.

---

### Finding Description

`MetricOmmPool.swap()` passes `msg.sender` as the `sender` argument to `_beforeSwap`: [1](#0-0) 

`ExtensionCalling._beforeSwap` forwards that value unchanged to every configured extension: [2](#0-1) 

`SwapAllowlistExtension.beforeSwap` then gates on that `sender` value: [3](#0-2) 

When a user calls `MetricOmmSimpleRouter.exactInputSingle`, the router calls `pool.swap(...)` directly: [4](#0-3) 

At that point `msg.sender` of `pool.swap()` is the **router address**, so `sender` delivered to `beforeSwap` is the router, not the originating user. The allowlist lookup `allowedSwapper[pool][router]` is evaluated instead of `allowedSwapper[pool][user]`.

This creates an inescapable dilemma for any pool admin running a curated pool:

- **If the router is NOT allowlisted**: every allowlisted user who tries to swap through the standard periphery path is blocked. Core swap functionality is broken for the intended audience.
- **If the router IS allowlisted** (the natural fix): `allowedSwapper[pool][router] = true` passes for every caller of the router regardless of their own allowlist status. Any non-allowlisted address can call `exactInputSingle` / `exactInput` / `exactOutputSingle` / `exactOutput` and trade freely on the curated pool.

The `DepositAllowlistExtension` does **not** share this flaw because it gates on `owner` (the position owner explicitly passed by the adder), not on `sender`: [5](#0-4) 

---

### Impact Explanation

A pool configured with `SwapAllowlistExtension` for KYC, compliance, or LP-protection purposes is fully bypassed once the router is allowlisted. Any unprivileged address can trade on the pool by routing through `MetricOmmSimpleRouter`. LP assets are exposed to adverse selection from counterparties the pool admin explicitly intended to exclude. This is a direct, fund-impacting admin-boundary break reachable by any unprivileged user through the supported public periphery path.

---

### Likelihood Explanation

The router is the canonical, documented swap entry point. A pool admin who wants their allowlisted users to be able to use the router has no correct option: they must either allowlist the router (opening the bypass) or leave it un-allowlisted (locking out their own users). The bypass path requires no special privileges, no unusual token behavior, and no multi-step setup beyond a single `exactInputSingle` call.

---

### Recommendation

Gate the allowlist on the **originating user** rather than the immediate pool caller. Two complementary approaches:

1. **Pass the payer/originator through `extensionData`**: The router encodes `msg.sender` into `extensionData`; the extension decodes and checks it. This requires a coordinated extension + router design.

2. **Check `recipient` or introduce an `originator` field**: Redesign `beforeSwap` to receive the economic actor (the address paying tokens in) rather than the immediate `msg.sender` of `pool.swap()`. The router already stores the true payer in transient storage; it could forward it as a verified field.

3. **Allowlist the originator via a trusted router**: Deploy a router variant that verifies the caller is allowlisted before forwarding to the pool, and allowlist only that router variant. The extension then trusts the router's pre-check.

---

### Proof of Concept

```
Setup:
  - Pool deployed with SwapAllowlistExtension as beforeSwap hook
  - Pool admin allowlists alice: allowedSwapper[pool][alice] = true
  - Pool admin also allowlists the router so alice can use it:
      allowedSwapper[pool][router] = true

Attack:
  - eve (not allowlisted) calls:
      router.exactInputSingle({pool: pool, ..., extensionData: ""})
  - Router calls pool.swap(...) with msg.sender = router
  - beforeSwap receives sender = router
  - Check: allowedSwapper[pool][router] == true  → PASSES
  - Eve's swap executes on the curated pool

Result:
  - Eve trades on a pool she was explicitly excluded from
  - LP assets are exposed to the excluded counterparty
  - The allowlist invariant is broken with zero privilege required
```
