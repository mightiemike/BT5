Audit Report

## Title
Swap Allowlist Bypass via Router: `sender` Identity Is Router Address, Not Original User - (File: metric-periphery/contracts/extensions/SwapAllowlistExtension.sol)

## Summary
`SwapAllowlistExtension.beforeSwap` gates swaps by checking `sender`, which is `msg.sender` of the pool's `swap()` call. When `MetricOmmSimpleRouter` intermediates, `msg.sender` at the pool is the router contract, not the original user. If the pool admin allowlists the router address (the natural action to permit router-mediated swaps for allowlisted users), every unprivileged user can bypass the allowlist by routing through the public router.

## Finding Description

**Root cause — wrong actor identity in the hook:**

`SwapAllowlistExtension.beforeSwap` receives `sender` as its first argument and checks it against the per-pool allowlist:

```solidity
// metric-periphery/contracts/extensions/SwapAllowlistExtension.sol L31-41
function beforeSwap(address sender, address, bool, int128, uint128, uint256, uint128, uint128, bytes calldata)
    external view override returns (bytes4)
{
    if (!allowAllSwappers[msg.sender] && !allowedSwapper[msg.sender][sender]) {
        revert IMetricOmmPoolActions.NotAllowedToSwap();
    }
    return IMetricOmmExtensions.beforeSwap.selector;
}
```

`msg.sender` here is the pool (enforced by `onlyPool` in `BaseMetricExtension`). `sender` is whatever the pool passes as the first argument to `_beforeSwap`.

**Pool passes `msg.sender` as `sender`:**

```solidity
// metric-core/contracts/MetricOmmPool.sol L230-240
_beforeSwap(
    msg.sender,   // <-- whoever called pool.swap()
    recipient,
    ...
);
```

**Router calls `pool.swap()` directly — original user identity is lost:**

```solidity
// metric-periphery/contracts/MetricOmmSimpleRouter.sol L71-80
_setNextCallbackContext(params.pool, CALLBACK_MODE_JUST_PAY, msg.sender, params.tokenIn);
(int128 amount0Delta, int128 amount1Delta) = IMetricOmmPoolActions(params.pool)
    .swap(
        params.recipient,
        params.zeroForOne,
        MetricOmmSwapInputs.asAmountSpecifiedIn(params.amountIn),
        priceLimitX64,
        "",
        params.extensionData
    );
```

When the router calls `pool.swap()`, `msg.sender` inside the pool is the **router address**. The extension therefore checks `allowedSwapper[pool][router]`, not `allowedSwapper[pool][originalUser]`.

**Exploit flow:**

1. Pool admin deploys a curated pool with `SwapAllowlistExtension`.
2. Admin allowlists specific users: `allowedSwapper[pool][alice] = true`.
3. Admin also allowlists the router so that allowlisted users can use it: `allowedSwapper[pool][router] = true`.
4. Bob (not allowlisted) calls `MetricOmmSimpleRouter.exactInputSingle(...)`.
5. Router calls `pool.swap(...)` → pool calls `_beforeSwap(msg.sender=router, ...)` → extension checks `allowedSwapper[pool][router]` = **true** → Bob's swap executes.

The same bypass applies to `exactInput`, `exactOutputSingle`, and `exactOutput` — all router entry points call `pool.swap()` with `msg.sender = router`.

**Existing guards are insufficient:** The `onlyPool` modifier in `BaseMetricExtension` only verifies the immediate caller is a registered pool; it does not verify that `sender` is the actual end user. There is no mechanism in the router to forward the original user's identity to the extension — `extensionData` is passed through as opaque bytes, and `SwapAllowlistExtension` ignores it entirely.

## Impact Explanation
A disallowed user can execute swaps on a curated pool that is supposed to restrict trading to an approved set of addresses. This is a direct allowlist bypass: the pool's curation policy is completely defeated for any user willing to route through the public `MetricOmmSimpleRouter`. The exact corrupted value is `allowedSwapper[pool][sender]` where `sender` resolves to the router address instead of the originating user, causing the gate to pass for all router-mediated swaps when the router is allowlisted. This matches the "broken core pool functionality causing loss of funds or unusable swap flows" and "admin-boundary break" impact categories.

## Likelihood Explanation
The condition is realistic and likely: pool admins who want allowlisted users to access the router (a standard periphery contract) must allowlist the router address, which simultaneously opens the gate to all users. The bypass requires no special privileges, no flash loans, and no multi-transaction setup — any unprivileged user can call `MetricOmmSimpleRouter.exactInputSingle` in a single transaction. It is repeatable indefinitely.

## Recommendation
The extension must check the economically relevant actor, not the immediate pool caller. Two sound approaches:

1. **Pass original user via `extensionData`:** Have the router encode `msg.sender` into `extensionData` and have `SwapAllowlistExtension.beforeSwap` decode and check it when `sender` is a known router. This requires a protocol-level convention.
2. **Check `sender` as the original user at the pool level:** Introduce a trusted-forwarder pattern where the router passes the original user as the `recipient`-equivalent identity, and the pool forwards it to extensions as a distinct `originator` argument.
3. **Simplest fix:** Document that the router must never be allowlisted and instead require allowlisted users to call `pool.swap()` directly. However, this breaks the intended UX and is not a code-level fix.

The cleanest fix is option 1: `MetricOmmSimpleRouter` encodes `msg.sender` in `extensionData` under a known prefix, and `SwapAllowlistExtension` decodes and checks it when `sender` matches a registered router address.

## Proof of Concept

```solidity
// Foundry test sketch
function test_swapAllowlistBypassViaRouter() public {
    // Setup: curated pool with SwapAllowlistExtension
    // Alice is allowlisted; Bob is not
    swapExtension.setAllowedToSwap(address(pool), alice, true);
    // Admin allowlists the router so Alice can use it
    swapExtension.setAllowedToSwap(address(pool), address(router), true);

    // Add liquidity (as alice, allowlisted depositor)
    vm.prank(alice);
    pool.addLiquidity(...);

    // Bob (NOT allowlisted) bypasses the allowlist via the router
    vm.prank(bob);
    router.exactInputSingle(ExactInputSingleParams({
        pool: address(pool),
        recipient: bob,
        zeroForOne: true,
        amountIn: 1000,
        amountOutMinimum: 0,
        priceLimitX64: 0,
        deadline: block.timestamp,
        extensionData: ""
    }));
    // Bob's swap succeeds — allowlist bypassed
}
```
