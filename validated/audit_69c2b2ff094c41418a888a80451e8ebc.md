### Title
Merkle Internal-Node Forgery: Proof-Length Mismatch Allows Sub-Tree Root to Pass as Leaf Transaction — (`merkle-tools/src/lib.rs`)

---

### Summary

`compute_root_from_merkle_proof` performs no validation that the supplied proof length matches the actual Merkle tree depth. An attacker can pass an internal node (a sub-tree root) as `tx_id` with a proof that is shorter than the real tree depth, and the function will still reconstruct the canonical root, causing `verify_transaction_inclusion` and `verify_transaction_inclusion_v2` to return `true` for a transaction that does not exist.

---

### Finding Description

`compute_root_from_merkle_proof` in `merkle-tools/src/lib.rs` iterates over whatever proof elements are supplied and hashes upward: [1](#0-0) 

There is no check that `merkle_proof.len()` equals `log2(num_txs)`, and no check that `transaction_hash` is a leaf-level value. The function is purely positional: it hashes the input upward by however many steps the caller provides.

For a block with 4 transactions `[tx0, tx1, tx2, tx3]`:

```
Level 1:  sub01 = hash(tx0, tx1)    sub23 = hash(tx2, tx3)
Level 2:  root  = hash(sub01, sub23)
```

Calling `compute_root_from_merkle_proof(sub01, 0, [sub23])` produces:

- `current_position = 0`, `0 % 2 == 0` → `hash(sub01, sub23)` = `root` ✓

The result equals the canonical `merkle_root` stored in the block header, so both verification entrypoints return `true`.

**Entrypoint 1 — `verify_transaction_inclusion` (deprecated, still callable):** [2](#0-1) 

The only guard is `!args.merkle_proof.is_empty()` (line 315). A 1-element proof passes. No access control (`#[pause]` only), no `#[private]`, no `trusted_relayer`. Any external caller can invoke this.

**Entrypoint 2 — `verify_transaction_inclusion_v2`:** [3](#0-2) 

The coinbase check at lines 358–365 was intended to enforce proof depth parity. However, it only verifies that `coinbase_tx_id` with `coinbase_merkle_proof` reconstructs the merkle root — it does **not** verify that `coinbase_tx_id` is the actual coinbase transaction. The attacker sets `coinbase_tx_id = sub01` and `coinbase_merkle_proof = [sub23]` (length 1), which also reconstructs the canonical root, bypassing the guard. The length-equality check (`merkle_proof.len() == coinbase_merkle_proof.len()`) is satisfied trivially since both are length 1. [4](#0-3) 

The `ProofArgsV2` struct accepts arbitrary `coinbase_tx_id` — there is no on-chain binding between `coinbase_tx_id` and the block's actual coinbase transaction.

---

### Impact Explanation

Any caller can submit `tx_id = hash(tx0, tx1)` (an internal Merkle node) with a 1-element proof `[hash(tx2, tx3)]` against a 4-transaction block and receive `true` from both `verify_transaction_inclusion` and `verify_transaction_inclusion_v2`. Downstream contracts or bridges that gate asset release on a `true` return value will release assets for a transaction that was never broadcast or confirmed on-chain.

---

### Likelihood Explanation

- Both entrypoints are public NEAR view/call methods with no privileged-role requirement.
- The attacker only needs knowledge of the block's transaction hashes, which are public on-chain data.
- No cryptographic hardness assumption is broken; the attack is pure arithmetic on the Merkle tree structure.
- The deprecated `verify_transaction_inclusion` is still deployed and callable.

---

### Recommendation

1. **Enforce proof-length equals tree depth.** Before iterating, compute `ceil(log2(num_txs))` from the block header (or pass it as a validated parameter) and require `merkle_proof.len() == expected_depth`.
2. **Bind `coinbase_tx_id` to the actual coinbase.** In `verify_transaction_inclusion_v2`, require that `coinbase_tx_id` equals the first transaction hash stored in the block, or derive it from the block header directly rather than accepting it as caller input.
3. **Remove or hard-disable `verify_transaction_inclusion`.** The deprecated function remains a live attack surface.

---

#[must_use]
pub fn compute_root_from_merkle_proof(
    transaction_hash: H256,
    transaction_position: usize,
    merkle_proof: &Vec<H256>,
) -> H256 {
    let mut current_hash = transaction_hash;
    let mut current_position = transaction_position;

    for proof_hash in merkle_proof {
        if current_position % 2 == 0 {
            current_hash = compute_hash(&current_hash, proof_hash);
        } else {
            current_hash = compute_hash(proof_hash, &current_hash);
        }
        current_position /= 2;
    }

    current_hash
}

    pub fn verify_transaction_inclusion(&self, #[serializer(borsh)] args: ProofArgs) -> bool {
        require!(
            args.confirmations <= self.gc_threshold,
            "The required number of confirmations exceeds the number of blocks stored in memory"
        );

        let heaviest_block_header = self
            .headers_pool
            .get(&self.mainchain_tip_blockhash)
            .unwrap_or_else(|| env::panic_str(ERR_KEY_NOT_EXIST));
        let target_block_height = self
            .mainchain_header_to_height
            .get(&args.tx_block_blockhash)
            .unwrap_or_else(|| env::panic_str("block does not belong to the current main chain"));

        // Check requested confirmations. No need to compute proof if insufficient confirmations.
        require!(
            (heaviest_block_header.block_height).saturating_sub(target_block_height) + 1
                >= args.confirmations,
            "Not enough blocks confirmed"
        );

        let header = self
            .headers_pool
            .get(&args.tx_block_blockhash)
            .unwrap_or_else(|| env::panic_str("cannot find requested transaction block"));

        require!(!args.merkle_proof.is_empty(), "Merkle proof is empty");

        // compute merkle tree root and check if it matches block's original merkle tree root
        merkle_tools::compute_root_from_merkle_proof(
            args.tx_id,
            usize::try_from(args.tx_index).unwrap(),
            &args.merkle_proof,
        ) == header.block_header.merkle_root
    }

    pub fn verify_transaction_inclusion_v2(&self, #[serializer(borsh)] args: ProofArgsV2) -> bool {
        require!(
            args.merkle_proof.len() == args.coinbase_merkle_proof.len(),
            "Coinbase merkle proof and transaction merkle proof should have the same length"
        );

        let header = self
            .headers_pool
            .get(&args.tx_block_blockhash)
            .unwrap_or_else(|| env::panic_str("cannot find requested transaction block"));

        require!(
            merkle_tools::compute_root_from_merkle_proof(
                args.coinbase_tx_id.clone(),
                0usize,
                &args.coinbase_merkle_proof,
            ) == header.block_header.merkle_root,
            "Incorrect coinbase merkle proof"
        );

        #[allow(deprecated)]
        self.verify_transaction_inclusion(args.into())
    }
