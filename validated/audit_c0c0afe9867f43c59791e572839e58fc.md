### Title
SwapAllowlistExtension Gates Router Address Instead of End-User, Enabling Any User to Bypass Per-User Swap Restrictions via the Router — (File: `metric-periphery/contracts/extensions/SwapAllowlistExtension.sol`)

---

### Summary

`SwapAllowlistExtension.beforeSwap` checks the `sender` argument, which the pool sets to `msg.sender` of `pool.swap()`. When a user routes through `MetricOmmSimpleRouter`, that `msg.sender` is the router contract, not the end user. If the pool admin allowlists the router (a necessary operational step to let any authorized user use the router), every user—including unauthorized ones—can bypass the per-user swap allowlist by routing through the router.

---

### Finding Description

**Step 1 — Pool passes `msg.sender` as `sender` to the extension.**

In `MetricOmmPool.swap`, the pool calls `_beforeSwap(msg.sender, ...)`: [1](#0-0) 

`_beforeSwap` then ABI-encodes that value as the first argument to `IMetricOmmExtensions.beforeSwap`: [2](#0-1) 

**Step 2 — The extension checks the `sender` argument, not the true end-user.**

```solidity
// SwapAllowlistExtension.sol line 31-41
function beforeSwap(address sender, address, bool, int128, uint128, uint256, uint128, uint128, bytes calldata)
    external view override returns (bytes4)
{
    if (!allowAllSwappers[msg.sender] && !allowedSwapper[msg.sender][sender]) {
        revert IMetricOmmPoolActions.NotAllowedToSwap();
    }
    return IMetricOmmExtensions.beforeSwap.selector;
}
``` [3](#0-2) 

Here `msg.sender` is the pool (the pool calls the extension), and `sender` is whatever `msg.sender` was when `pool.swap()` was called. When the call originates from `MetricOmmSimpleRouter`, `sender` = router address.

**Step 3 — The dilemma the admin faces.**

The allowlist is keyed `allowedSwapper[pool][sender]`. The admin has two choices:

| Admin action | Effect |
|---|---|
| Allowlist only individual users (A, B) | A and B can swap directly; **neither can use the router** (router not allowlisted → reverts) |
| Also allowlist the router | A and B can use the router; **every other user can also use the router** (bypass) |

There is no configuration that simultaneously allows authorized users to use the router and blocks unauthorized users from using the router. The extension cannot distinguish between authorized and unauthorized users once they are behind the router. [4](#0-3) 

---

### Impact Explanation

When the pool admin allowlists the router (a routine operational step so that authorized users can benefit from slippage protection, multi-hop routing, etc.), the swap allowlist is fully bypassed for all users. Any address can call `MetricOmmSimpleRouter.exactInput*`, which calls `pool.swap()` with `msg.sender = router`, and the extension approves the call because the router is allowlisted. The pool admin's intent to restrict trading to a specific set of counterparties is violated. This constitutes broken core pool functionality: the configured guard no longer enforces the intended access boundary.

---

### Likelihood Explanation

The bypass is reachable whenever the pool admin allowlists the router—a step that is operationally necessary the moment any authorized user wants to use the periphery router. The research target explicitly identifies this path:

> *"Because public users may enter through the router, the hook must gate the same actor the pool designers thought they were allowlisting."* [5](#0-4) 

The trigger is a valid, non-malicious admin action (allowlisting the router), and the exploit is by an unprivileged user. Likelihood is **Medium**.

---

### Recommendation

The extension must verify the true end-user identity, not the intermediary. Two viable approaches:

1. **Router passes original caller in `extensionData`**: The router encodes `msg.sender` into `extensionData` before calling `pool.swap`. The extension decodes and checks that address. The extension must also verify the encoding came from a trusted router (e.g., via a factory-registered router registry).

2. **Pool exposes an explicit `originator` parameter**: Add an `originator` field to `pool.swap` that the router sets to `tx.origin` or the user's address, and the extension checks that field. (Note: `tx.origin` has its own risks and should be used carefully.)

The `DepositAllowlistExtension` has an analogous but inverted exposure: it checks `owner` (position recipient) rather than `sender` (payer), meaning any address can fund a position for an allowlisted owner. This is lower severity because the unauthorized payer does not gain LP shares, but it is worth noting as a related identity-binding inconsistency. [6](#0-5) 

---

### Proof of Concept

```
1. Pool admin deploys pool with SwapAllowlistExtension configured.
2. Admin allowlists userA:
       extension.setAllowedToSwap(pool, userA, true)
3. userA wants to use the router → admin allowlists the router:
       extension.setAllowedToSwap(pool, router, true)
4. Unauthorized userC calls:
       router.exactInputSingle({pool: pool, ...})
   → router calls pool.swap(recipient, ...) with msg.sender = router
   → pool calls extension.beforeSwap(router, ...)
   → extension checks allowedSwapper[pool][router] → true → passes
5. userC's swap executes successfully despite never being allowlisted.
```

The root cause is in `SwapAllowlistExtension.beforeSwap` checking `sender` (the intermediary router) rather than the originating user, mirroring the RubiconRouter bug class where an intermediate call path causes the wrong value to be forwarded to the enforcing function. [7](#0-6) [1](#0-0)

contract SwapAllowlistExtension is BaseMetricExtension, ISwapAllowlistExtension {
  mapping(address pool => mapping(address swapper => bool)) public allowedSwapper;
  mapping(address pool => bool) public allowAllSwappers;

  constructor(address factory_) BaseMetricExtension(factory_) {}

  function setAllowedToSwap(address pool_, address swapper, bool allowed) external onlyPoolAdmin(pool_) {
    allowedSwapper[pool_][swapper] = allowed;
    emit AllowedToSwapSet(pool_, swapper, allowed);
  }

  function setAllowAllSwappers(address pool_, bool allowed) external onlyPoolAdmin(pool_) {
    allowAllSwappers[pool_] = allowed;
    emit AllowAllSwappersSet(pool_, allowed);
  }

  function isAllowedToSwap(address pool_, address swapper) external view returns (bool) {
    return allowAllSwappers[pool_] || allowedSwapper[pool_][swapper];
  }

  function beforeSwap(address sender, address, bool, int128, uint128, uint256, uint128, uint128, bytes calldata)
    external
    view
    override
    returns (bytes4)
  {
    if (!allowAllSwappers[msg.sender] && !allowedSwapper[msg.sender][sender]) {
      revert IMetricOmmPoolActions.NotAllowedToSwap();
    }
    return IMetricOmmExtensions.beforeSwap.selector;
  }
}
