### Title
`abi: null` vs `abi: []` in `BroadcastedDeclareV1Transaction` produces divergent `ClassInfo::abi_length` (0 vs 2), causing `estimateFee`/`simulateTransactions` to return a wrong fee value — (`crates/papyrus_common/src/deprecated_class_abi.rs`)

---

### Summary

`calculate_deprecated_class_abi_length` short-circuits to `0` for `abi: None` but serializes `Some(vec![])` to the 2-byte JSON string `[]`, producing a non-canonical `abi_length`. This value flows directly into `ClassInfo::code_size()` and from there into the gas/fee vector returned by `estimateFee` and `simulateTransactions`.

---

### Finding Description

**Step 1 — Divergence in `calculate_deprecated_class_abi_length`** [1](#0-0) 

- `abi: None` → early-returns `Ok(0)`.
- `abi: Some(vec![])` → serializes the empty slice with `PythonJsonFormatter`, producing the 2-byte string `[]`, so returns `Ok(2)`.

**Step 2 — Attacker-controlled `abi` field reaches the function unchanged**

The RPC input type `DeprecatedContractClass` in the writer objects carries `abi: Option<Vec<...>>`: [2](#0-1) 

`user_deprecated_contract_class_to_sn_api` passes `value.abi` verbatim — no normalization of `None` → `Some(vec![])` or vice versa: [3](#0-2) 

**Step 3 — `abi_length` is computed from the un-normalized value and stored** [4](#0-3) 

**Step 4 — `abi_length` enters `ClassInfo` and directly affects `code_size()`** [5](#0-4) 

`code_size = (bytecode_length + sierra_program_length) * FELT_WIDTH + abi_length`

**Step 5 — `code_size` drives the gas vector in fee estimation**

From the receipt test, the formula is explicit: [6](#0-5) 

`code_gas_cost = gas_per_code_byte * ((bytecode + sierra_program) * WORD_WIDTH + abi_length)`

So `abi_length = 0` vs `abi_length = 2` produces a fee difference of `gas_per_code_byte × 2`.

**Step 6 — `ClassInfo` is constructed with the divergent value in the execution path** [7](#0-6) 

---

### Impact Explanation

Any unprivileged caller can send two semantically equivalent `estimateFee` or `simulateTransactions` requests — one with `"abi": null` and one with `"abi": []` — and receive different fee estimates. The RPC endpoint returns an authoritative-looking wrong value. The magnitude of the divergence is `gas_per_code_byte × 2` (2 bytes), which is small but non-zero and concrete.

---

### Likelihood Explanation

The path is fully public (no authentication). The `abi` field is optional and both `null` and `[]` are valid JSON representations of "no ABI entries." Any client or fuzzer that alternates between the two representations will observe the divergence.

---

### Recommendation

Normalize the `abi` field in `user_deprecated_contract_class_to_sn_api` before passing it to `calculate_deprecated_class_abi_length`: treat `Some(vec![])` identically to `None` (or vice versa), so that `abi_length` is canonical for a given class content. Alternatively, fix `calculate_deprecated_class_abi_length` to return `Ok(0)` for both `None` and `Some(vec![])`.

---

### Proof of Concept

```rust
use papyrus_common::deprecated_class_abi::calculate_deprecated_class_abi_length;
use starknet_api::deprecated_contract_class::ContractClass;

let class_none = ContractClass { abi: None, ..Default::default() };
let class_empty = ContractClass { abi: Some(vec![]), ..Default::default() };

let len_none  = calculate_deprecated_class_abi_length(&class_none).unwrap();
let len_empty = calculate_deprecated_class_abi_length(&class_empty).unwrap();

// Fails: len_none == 0, len_empty == 2
assert_eq!(len_none, len_empty,
    "abi: None gives {len_none} but abi: Some(vec![]) gives {len_empty}");
```

```rust
pub fn calculate_deprecated_class_abi_length(
    deprecated_class: &starknet_api::deprecated_contract_class::ContractClass,
) -> Result<usize, serde_json::Error> {
    let Some(abi) = deprecated_class.abi.as_ref() else {
        return Ok(0);
    };
    let mut chars = vec![];
    abi.serialize(&mut serde_json::Serializer::with_formatter(&mut chars, PythonJsonFormatter))?;
    Ok(chars.len())
```
