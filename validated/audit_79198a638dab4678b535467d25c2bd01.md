### Title
`MsgUpdateTier` Increases `exit_duration` Without Grace Period for Existing Positions, Immediately Extending Lock-Up Commitment - (File: `x/tieredrewards/keeper/msg_server_auth.go`)

### Summary
`MsgUpdateTier` applies a new `exit_duration` immediately to the tier state with no grace period and no per-position snapshot of the duration at creation time. Any existing position that has not yet called `TriggerExitFromTier` will be subject to the new (potentially much longer) duration when it eventually triggers exit. This is the direct analog of the collateralization-ratio-change-without-grace-period class: a governance-controlled parameter change immediately and silently worsens the terms for all existing, not-yet-exiting positions.

### Finding Description

`UpdateTier` in `msg_server_auth.go` writes the new tier parameters atomically:

```go
func (ms msgServer) UpdateTier(ctx context.Context, msg *types.MsgUpdateTier) (*types.MsgUpdateTierResponse, error) {
    ...
    if !oldTier.BonusApy.Equal(msg.Tier.BonusApy) {
        if err := ms.claimRewardsAndUpdateTierPositions(ctx, msg.Tier.Id); err != nil { ... }
    }
    if err := ms.SetTier(ctx, msg.Tier); err != nil { ... }
    ...
}
``` [1](#0-0) 

The only special handling is for `BonusApy` changes (reward settlement). A change to `exit_duration` has **no corresponding protection for existing positions**. The new duration is stored in the tier record immediately.

When a user later calls `TriggerExitFromTier`, the handler reads the **current** tier's `ExitDuration` to compute `ExitUnlockAt`:

```go
tier, err := ms.getTier(ctx, pos.TierId)
...
pos.TriggerExit(sdkCtx.BlockTime(), tier.ExitDuration)
``` [2](#0-1) 

`TriggerExit` stores the computed unlock time directly on the position:

```go
func (p *Position) TriggerExit(blockTime time.Time, duration time.Duration) {
    p.ExitTriggeredAt = blockTime
    p.ExitUnlockAt = blockTime.Add(duration)
}
``` [3](#0-2) 

The `exit_duration` is **never snapshotted at position creation time**. The `Position` struct stores only `ExitTriggeredAt` and `ExitUnlockAt` (set at trigger time), not the duration that was in effect when the position was created. [4](#0-3) 

The `Tier` struct's `ExitDuration` field is the sole source of truth read at trigger time: [5](#0-4) 

Additionally, when a tier is simultaneously set to `close_only = true`, `MsgClearPosition` is blocked for all positions in that tier: [6](#0-5) 

This means a user who triggers exit after the duration increase cannot cancel it.

### Impact Explanation

A user who locked tokens under a tier with `exit_duration = 5s` (or any short duration) and has not yet triggered exit will, after a governance-approved `exit_duration` increase to e.g. 1 year, be forced to wait the full new duration when they eventually trigger exit. Their principal is locked for a period far exceeding what they agreed to at lock time. If the tier is simultaneously set to `close_only = true`, `ClearPosition` is blocked and the user cannot cancel the exit commitment. The user's tokens are inaccessible for the extended period with no recourse. The corrupted invariant is: **the exit commitment duration a position owner faces must not exceed the duration in effect when the position was created**.

### Likelihood Explanation

`MsgUpdateTier` is a standard governance message reachable by any token holder who can submit and pass a governance proposal. The governance voting period provides some advance notice, but there is no on-chain mechanism that prevents the change from applying retroactively to existing positions the moment the proposal executes. Users who do not actively monitor governance proposals will have no warning. The integration test suite explicitly demonstrates that `exit_duration` changes take effect immediately for new trigger calls: [7](#0-6) 

### Recommendation

1. **Snapshot `exit_duration` at position creation.** Store the `exit_duration` (or the computed `ExitUnlockAt` relative to a future trigger) on the `Position` record at creation time, so that `TriggerExitFromTier` uses the snapshotted value rather than the live tier value.
2. **Alternatively, add a grace period for existing positions.** When `exit_duration` is increased, record the old duration alongside the new one and allow positions created before the change to use the old duration for a configurable grace window.
3. **Mirror the `BonusApy` pattern.** Just as `UpdateTier` calls `claimRewardsAndUpdateTierPositions` when `BonusApy` changes, an `exit_duration` increase should trigger a comparable protection step (e.g., emit a warning event and enforce a minimum notice period before the new duration applies to existing positions).

### Proof of Concept

1. Governance creates Tier 1 with `exit_duration = 5s`, `min_lock_amount = 1_000_000`.
2. User A calls `MsgLockTier` and locks `1_000_000 basecro` into Tier 1. User A expects to be able to exit after 5 seconds.
3. Governance passes `MsgUpdateTier` setting `exit_duration = 1 year` for Tier 1 (no special handling in `UpdateTier` for this field change).
4. User A calls `MsgTriggerExitFromTier`. The handler reads `tier.ExitDuration = 1 year` and sets `ExitUnlockAt = now + 1 year`.
5. User A's `1_000_000 basecro` is now locked for 1 year. If the tier was also set to `close_only = true` in the same proposal, `MsgClearPosition` returns `ErrTierIsCloseOnly` and the user cannot cancel the exit.
6. The user has no on-chain recourse to recover their principal for the full new duration. [1](#0-0) [8](#0-7) [3](#0-2)

func (ms msgServer) UpdateTier(ctx context.Context, msg *types.MsgUpdateTier) (*types.MsgUpdateTierResponse, error) {
	if err := ms.requireAuthority(msg.Authority); err != nil {
		return nil, err
	}

	oldTier, err := ms.getTier(ctx, msg.Tier.Id)
	if err != nil {
		return nil, err
	}

	if !oldTier.BonusApy.Equal(msg.Tier.BonusApy) {
		if err := ms.claimRewardsAndUpdateTierPositions(ctx, msg.Tier.Id); err != nil {
			return nil, err
		}
	}

	if err := ms.SetTier(ctx, msg.Tier); err != nil {
		return nil, err
	}

	if err := ms.emitTierChangedEvent(ctx, types.TierChangeAction_TIER_CHANGE_ACTION_UPDATE, msg.Tier); err != nil {
		return nil, err
	}

	return &types.MsgUpdateTierResponse{}, nil
}

func (ms msgServer) TriggerExitFromTier(ctx context.Context, msg *types.MsgTriggerExitFromTier) (*types.MsgTriggerExitFromTierResponse, error) {
	if err := msg.Validate(); err != nil {
		return nil, err
	}

	pos, err := ms.getPositionState(ctx, msg.PositionId)
	if err != nil {
		return nil, err
	}

	if err := ms.validateTriggerExit(pos.Position, msg.Owner); err != nil {
		return nil, err
	}

	tier, err := ms.getTier(ctx, pos.TierId)
	if err != nil {
		return nil, err
	}

	sdkCtx := sdk.UnwrapSDKContext(ctx)
	pos.TriggerExit(sdkCtx.BlockTime(), tier.ExitDuration)

	if err := ms.setPosition(ctx, pos.Position, nil); err != nil {
		return nil, err
	}

	if err := sdkCtx.EventManager().EmitTypedEvent(&types.EventExitTriggered{
		PositionId:   pos.Id,
		TierId:       pos.TierId,
		Owner:        pos.Owner,
		ExitUnlockAt: pos.ExitUnlockAt,
	}); err != nil {
		return nil, err
	}

	return &types.MsgTriggerExitFromTierResponse{
		ExitUnlockAt: pos.ExitUnlockAt,
		PositionId:   pos.Id,
	}, nil
}

def test_update_tier_non_apy_no_claim(cluster):
    """Changing exit_duration without changing APY does not trigger rewards claiming.

    Verified by checking that the owner balance is unchanged.
    """
    owner = cluster.address("signer1")

    # Confirm the signer1 position on Tier 3 exists
    result = query_command(cluster, MODULE, "positions-by-owner", owner)
    tier3_pos = next(
        p for p in result.get("positions", []) if int(p["tier_id"]) == TIER_3_ID
    )
    assert tier3_pos is not None
    balance_before = cluster.balance(owner, DENOM)

    rsp = submit_gov_proposal(
        cluster,
        "community",
        MSG_UPDATE_TIER,
        {
            "tier": {
                "id": TIER_3_ID,
                "exit_duration": "10s",
                "bonus_apy": "0.030000000000000000",
                "min_lock_amount": "2000000",
                "close_only": False,
            }
        },
        title="Update Tier 3 exit_duration",
        summary="Only exit_duration changes, no APY change",
    )
    approve_proposal(cluster, rsp, msg=f",{MSG_UPDATE_TIER}")

    # No rewards should have been paid out to the owner.
    assert (
        cluster.balance(owner, DENOM) == balance_before
    ), "owner balance should not change when APY is unchanged"

    # Verify exit_duration is updated
    result = query_tiers(cluster)
    tier3 = next(
        (t for t in result.get("tiers", []) if int(t["id"]) == TIER_3_ID), None
    )
    assert tier3["exit_duration"] == "10s"

    # Verify the new exit_duration applies to new positions
    owner = cluster.address("signer2")
    validator = get_validator_addr(cluster, 0)

    before = before_ids(cluster, owner)
    rsp = lock_tier(cluster, owner, TIER_3_ID, 2_000_000, validator=validator)
    assert rsp["code"] == 0, rsp["raw_log"]
    pos_id = new_pos_id(cluster, owner, before)

    rsp = trigger_exit(cluster, owner, pos_id)
    assert rsp["code"] == 0, rsp["raw_log"]

    pos = query_position(cluster, pos_id)["position"]
    triggered_at = isoparse(pos["exit_triggered_at"])
    unlock_at = isoparse(pos["exit_unlock_at"])
    actual_duration = (unlock_at - triggered_at).total_seconds()

    assert actual_duration == 10, (
        f"new position should use updated exit_duration (10s), "
        f"got {actual_duration}s"
    )
