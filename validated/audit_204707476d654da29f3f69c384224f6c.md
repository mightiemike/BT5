### Title
`ValidResourceBounds::AllResources` Silently Coerced to `L1Gas` in Protobuf Deserialization Produces Wrong Transaction Hash Preimage - (`crates/apollo_protobuf/src/converters/transaction.rs`)

### Summary

The protobuf `TryFrom<protobuf::ResourceBounds> for ValidResourceBounds` silently coerces `AllResources { l1_gas: X, l2_gas: 0, l1_data_gas: 0 }` into `L1Gas(X)`. Because `get_tip_resource_bounds_hash` hashes **2 resource felts** for `L1Gas` but **3 resource felts** for `AllResources`, the two variants produce different Poseidon digests and therefore different transaction hashes. A transaction accepted and hashed by the sequencer under `AllResources` will be re-hashed under `L1Gas` by any node that deserializes it from protobuf, producing a divergent hash and wrong fee-computation mode.

### Finding Description

`ValidResourceBounds` has two variants with different hash preimages: [1](#0-0) 

- `L1Gas` → hashes `[tip, L1_GAS_packed, L2_GAS_packed]` (2 resource felts)
- `AllResources` → hashes `[tip, L1_GAS_packed, L2_GAS_packed, L1_DATA_GAS_packed]` (3 resource felts)

The protobuf deserializer in `transaction.rs` reconstructs the variant by inspecting the wire values: [2](#0-1) 

When `l2_gas` and `l1_data_gas` are both zero, the deserializer returns `ValidResourceBounds::L1Gas`, regardless of whether the sender originally used `AllResources`. The gateway's stateless validator explicitly accepts `AllResourceBounds` with only `l1_gas` non-zero: [3](#0-2) 

So a transaction with `AllResourceBounds { l1_gas: X, l2_gas: 0, l1_data_gas: 0 }` is valid and admitted. The sequencer computes its hash via `InternalRpcInvokeTransactionV3::resource_bounds()` which always returns `ValidResourceBounds::AllResources`: [4](#0-3) 

This produces hash **H1** (3-felt preimage). When the same transaction is later serialized to protobuf and deserialized by a syncing node, the `ValidResourceBounds` is reconstructed as `L1Gas`, and any hash recomputation produces hash **H2** (2-felt preimage). **H1 ≠ H2**.

The codebase itself acknowledges this invariant break in the consensus test: [5](#0-4) 

The test works around it by forcing non-zero `l2_gas`. No such guard exists in the production P2P block-sync path.

The `From<ValidResourceBounds> for protobuf::ResourceBounds` serializer emits a zero `l1_data_gas` field for `L1Gas`, making the wire format ambiguous: [6](#0-5) 

By contrast, the JSON/`StorageSerde` path correctly distinguishes the variants by the presence or absence of the `L1DataGas` map key: [7](#0-6) 

The protobuf path lacks this discriminator.

### Impact Explanation

**High — RPC execution, fee estimation, tracing, simulation, or pending view returns an authoritative-looking wrong value.**

A syncing node that receives a block via P2P, deserializes its transactions from protobuf, and then serves them via `starknet_simulateTransactions` or `starknet_estimateFee` will:

1. Use `GasVectorComputationMode::NoL2Gas` instead of `All`, skipping L2-gas and L1-data-gas bound checks.
2. Compute a different transaction hash than the one committed by the sequencer.

The fee-mode divergence means the RPC node returns authoritative-looking but wrong fee estimates for any V3 transaction whose `l2_gas` and `l1_data_gas` bounds are zero. Additionally, if the node re-executes the transaction for tracing, it applies the wrong resource-bound enforcement, potentially reporting a successful execution where the sequencer would have reverted (or vice versa).

### Likelihood Explanation

**Medium.** The gateway accepts `AllResourceBounds` with only `l1_gas` non-zero (confirmed by the stateless-validator test). Client-side proving transactions explicitly set `l2_gas.max_price_per_unit = 0` and `l1_data_gas = 0`, making the zero-value case reachable in production. The consensus test already documents the coercion and patches around it, confirming the path is exercised.

### Recommendation

Add a variant discriminator to the protobuf `ResourceBounds` message (e.g., a boolean `is_all_resources` flag or a separate `oneof` field), and update both the serializer and deserializer to preserve the original variant unconditionally. Alternatively, always serialize `AllResources` with a sentinel non-zero `l1_data_gas` field and document the convention. The JSON path's approach—presence of the `L1DataGas` key signals `AllResources`—should be replicated in the protobuf schema.

### Proof of Concept

```
1. Craft a V3 invoke transaction:
     resource_bounds = AllResourceBounds {
         l1_gas:      { max_amount: 1000, max_price_per_unit: 1 },
         l2_gas:      { max_amount: 0,    max_price_per_unit: 0 },
         l1_data_gas: { max_amount: 0,    max_price_per_unit: 0 },
     }

2. Gateway accepts it (stateless validator: valid_l1_gas case).

3. Sequencer computes H1 via get_invoke_transaction_v3_hash →
   get_tip_resource_bounds_hash(AllResources) →
   Poseidon([tip, L1_GAS_packed, L2_GAS_packed, L1_DATA_GAS_packed_zero])
   (3 resource felts)

4. Transaction is serialized to protobuf::ResourceBounds:
     l1_gas      = { max_amount: 1000, max_price_per_unit: 1 }
     l2_gas      = { max_amount: 0,    max_price_per_unit: 0 }
     l1_data_gas = { max_amount: 0,    max_price_per_unit: 0 }

5. Syncing node deserializes via TryFrom<protobuf::ResourceBounds>:
     l1_data_gas.is_zero() && l2_gas.is_zero() == true
     → ValidResourceBounds::L1Gas({ max_amount: 1000, max_price_per_unit: 1 })

6. Syncing node computes H2 via get_tip_resource_bounds_hash(L1Gas) →
   Poseidon([tip, L1_GAS_packed, L2_GAS_packed])
   (2 resource felts)

7. H1 ≠ H2.
   get_gas_vector_computation_mode() returns NoL2Gas instead of All.
   Fee estimation and simulation on the syncing node return wrong values.
```

// An implementation of the SNIP: https://github.com/EvyatarO/SNIPs/blob/snip-8/SNIPS/snip-8.md
pub fn get_tip_resource_bounds_hash(
    resource_bounds: &ValidResourceBounds,
    tip: &Tip,
) -> Result<Felt, StarknetApiError> {
    let l1_resource_bounds = resource_bounds.get_l1_bounds();
    let l2_resource_bounds = resource_bounds.get_l2_bounds();

    // L1 and L2 gas bounds always exist.
    // Old V3 txs always have L2 gas bounds of zero, but they exist.
    let mut resource_felts = vec![
        get_concat_resource(&l1_resource_bounds, L1_GAS)?,
        get_concat_resource(&l2_resource_bounds, L2_GAS)?,
    ];

    // For new V3 txs, need to also hash the data gas bounds.
    resource_felts.extend(match resource_bounds {
        ValidResourceBounds::L1Gas(_) => vec![],
        ValidResourceBounds::AllResources(all_resources) => {
            vec![get_concat_resource(&all_resources.l1_data_gas, L1_DATA_GAS)?]
        }
    });

    Ok(HashChain::new().chain(&tip.0.into()).chain_iter(resource_felts.iter()).get_poseidon_hash())
}

impl TryFrom<protobuf::ResourceBounds> for ValidResourceBounds {
    type Error = ProtobufConversionError;
    fn try_from(value: protobuf::ResourceBounds) -> Result<Self, Self::Error> {
        let Some(l1_gas) = value.l1_gas else {
            return Err(missing("ResourceBounds::l1_gas"));
        };
        let Some(l2_gas) = value.l2_gas else {
            return Err(missing("ResourceBounds::l2_gas"));
        };
        // TODO(Shahak): Assert data gas is not none once we remove support for 0.13.2.
        let l1_data_gas = value.l1_data_gas.unwrap_or_default();
        let l1_gas: ResourceBounds = l1_gas.try_into()?;
        let l2_gas: ResourceBounds = l2_gas.try_into()?;
        let l1_data_gas: ResourceBounds = l1_data_gas.try_into()?;
        Ok(if l1_data_gas.is_zero() && l2_gas.is_zero() {
            ValidResourceBounds::L1Gas(l1_gas)
        } else {
            ValidResourceBounds::AllResources(AllResourceBounds { l1_gas, l2_gas, l1_data_gas })
        })
    }
