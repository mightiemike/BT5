Audit Report

## Title
SwapAllowlistExtension Checks Router Address Instead of Actual User, Allowing Any User to Bypass the Swap Allowlist - (File: metric-periphery/contracts/extensions/SwapAllowlistExtension.sol)

## Summary

`SwapAllowlistExtension.beforeSwap` gates swaps by checking the `sender` argument, which the pool sets to its own `msg.sender`. When users route through `MetricOmmSimpleRouter`, the pool's `msg.sender` is the router contract, so `sender` delivered to the extension is the router address — not the originating user. Any pool that allowlists the router (required for router-mediated swaps to work) is fully open to all users, defeating the curated allowlist entirely.

## Finding Description

`MetricOmmPool.swap` passes its own `msg.sender` as `sender` to `_beforeSwap`: [1](#0-0) 

`ExtensionCalling._beforeSwap` forwards that `sender` argument directly to the extension: [2](#0-1) 

`SwapAllowlistExtension.beforeSwap` then checks `allowedSwapper[msg.sender][sender]` — where `msg.sender` is the pool and `sender` is whoever called `pool.swap`: [3](#0-2) 

`MetricOmmSimpleRouter.exactInputSingle` calls `pool.swap(...)` directly, making the router the pool's `msg.sender`. The real user (`msg.sender` of `exactInputSingle`) is stored only in transient callback context for payment settlement and is never forwarded to the pool or any extension: [4](#0-3) 

**Exploit path:**
1. Pool admin deploys a pool with `SwapAllowlistExtension` and allowlists only Alice: `allowedSwapper[pool][alice] = true`.
2. To allow Alice to swap via the router, admin must also allowlist the router: `allowedSwapper[pool][router] = true`.
3. Non-allowlisted Bob calls `router.exactInputSingle(...)`. The router calls `pool.swap(...)`, so `sender = router` reaches `beforeSwap`. The check `allowedSwapper[pool][router] == true` passes. Bob's swap executes.

There is no configuration that simultaneously permits router-mediated swaps for allowlisted users and blocks non-allowlisted users, because the extension cannot distinguish between them once the router is the intermediary.

## Impact Explanation

This is a direct admin-boundary break: the pool admin's access control policy is silently bypassed by any unprivileged user routing through the standard periphery. Curated pools designed to restrict counterparties (e.g., to exclude arbitrageurs, sanctioned addresses, or non-KYC'd users) are rendered fully open. LP assets in such pools are exposed to unauthorized swap counterparties, which can cause direct LP losses. This meets the "Admin-boundary break: pool admin exceeds caps, bypasses timelocks, or factory/oracle role checks are bypassed by an unprivileged path" allowed impact.

## Likelihood Explanation

High. The router is the standard, documented periphery entrypoint. Any pool admin who wants allowlisted users to swap via the router must add the router to the allowlist — at which point the bypass is immediately and unconditionally available to all users. No special attacker capability is required beyond calling the public router function.

## Recommendation

The extension must gate on the economically relevant actor — the originating user — not the direct caller of `pool.swap`. Two viable approaches:

1. **Pass the original user through the pool**: Modify the pool's `swap` interface to accept an explicit `swapper` address (verified against `msg.sender` or a trusted router registry), and forward it as `sender` to extensions. This is the cleanest fix.
2. **Router-level identity forwarding via `extensionData`**: Have the router encode the real user's address in `extensionData` and have `SwapAllowlistExtension.beforeSwap` decode and verify it — but this requires the extension to trust the router, which must be enforced (e.g., by checking that `msg.sender` pool was called by a known router, or by verifying a signature).

## Proof of Concept

```
Setup:
  - Deploy pool with SwapAllowlistExtension
  - Pool admin: allowedSwapper[pool][alice] = true
  - Pool admin: allowedSwapper[pool][router] = true  ← required for Alice to use router

Attack (Bob, not allowlisted):
  1. Bob calls router.exactInputSingle({pool: pool, ...})
  2. Router calls pool.swap(recipient, zeroForOne, amount, ...)
     → pool's msg.sender = router
  3. Pool calls _beforeSwap(sender=router, ...)
     [MetricOmmPool.sol L230-240, ExtensionCalling.sol L159-177]
  4. Extension checks: allowedSwapper[pool][router] == true → PASSES
     [SwapAllowlistExtension.sol L37]
  5. Bob's swap executes on the curated pool

Result: Bob bypasses the allowlist entirely by routing through the router.
```
