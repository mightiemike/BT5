Audit Report

## Title
`SwapAllowlistExtension` Checks Direct Caller Instead of End User, Enabling Allowlist Bypass via `MetricOmmSimpleRouter` — (`metric-periphery/contracts/extensions/SwapAllowlistExtension.sol`)

## Summary

`SwapAllowlistExtension.beforeSwap` gates swaps by checking the `sender` argument, which `MetricOmmPool.swap` sets to its own `msg.sender` — the direct caller of `pool.swap()`. When a user routes through `MetricOmmSimpleRouter`, the direct caller is the router contract, not the end user. A pool admin who allowlists the router to enable multi-hop routing for their allowlisted users inadvertently opens the pool to every unprivileged user, because any address can call the public router and have the router's allowlisted identity forwarded to the extension.

## Finding Description

`MetricOmmPool.swap` passes `msg.sender` verbatim as the `sender` argument to `_beforeSwap`:

```solidity
// MetricOmmPool.sol L230-240
_beforeSwap(
    msg.sender,   // ← whoever called pool.swap()
    recipient,
    ...
);
```

`SwapAllowlistExtension.beforeSwap` then checks whether that `sender` is allowlisted for the calling pool (`msg.sender` inside the extension = the pool):

```solidity
// SwapAllowlistExtension.sol L37
if (!allowAllSwappers[msg.sender] && !allowedSwapper[msg.sender][sender]) {
    revert IMetricOmmPoolActions.NotAllowedToSwap();
}
```

`MetricOmmSimpleRouter.exactInputSingle` calls `pool.swap()` directly:

```solidity
// MetricOmmSimpleRouter.sol L72-80
IMetricOmmPoolActions(params.pool).swap(
    params.recipient,
    params.zeroForOne,
    ...
    params.extensionData
);
```

At this point `pool.msg.sender = router`, so the extension receives `sender = router` and evaluates `allowedSwapper[pool][router]`. The end user's address is never visible to the extension. A pool admin who wants to enable router-based swaps for their allowlisted users must add the router to the allowlist. Once `allowedSwapper[pool][router] = true`, every user — including those explicitly excluded — can swap by routing through the public, permissionless router. The same bypass applies to `exactInput`, `exactOutputSingle`, and `exactOutput` entry points.

## Impact Explanation

A pool configured with `SwapAllowlistExtension` is a restricted venue (e.g., KYC-gated or institutional-only). Bypassing the allowlist allows non-allowlisted users to execute swaps that drain token reserves from the pool, causing direct loss of LP principal. The pool's core access-control invariant — "only allowlisted addresses may swap" — is violated for every router-mediated swap once the router is allowlisted. This constitutes broken core pool functionality causing loss of funds and meets Sherlock High/Critical thresholds.

## Likelihood Explanation

The pool admin has a legitimate, non-malicious reason to allowlist the router: without it, allowlisted users cannot use multi-hop routing or the standard periphery UX. `MetricOmmSimpleRouter` is a public, permissionless contract — any address can call it. The extension design gives no warning that allowlisting the router is equivalent to `setAllowAllSwappers(pool, true)`. Once the router is allowlisted via a single `setAllowedToSwap` call, the bypass is permanently open to all users with no further action required.

## Recommendation

The `SwapAllowlistExtension` must gate on the original end user, not the direct caller of `pool.swap()`. Two complementary fixes:

1. **Pass the original payer through `extensionData`**: The router encodes `msg.sender` into `extensionData` before calling `pool.swap()`; the extension decodes and checks that address instead of `sender`.
2. **Maintain a trusted-router registry in the extension**: When `sender` is a known router, require `extensionData` to carry a signed or encoded end-user identity and check that identity against the allowlist.

Alternatively, document explicitly that allowlisting the router is equivalent to opening the pool to all users, and provide a separate `RouterSwapAllowlistExtension` that enforces per-user checks via signed payloads forwarded through `extensionData`.

## Proof of Concept

```solidity
// Setup: pool admin allowlists alice (KYC'd) and the router (for multi-hop UX)
swapExtension.setAllowedToSwap(address(pool), alice, true);
swapExtension.setAllowedToSwap(address(pool), address(router), true);

// Attack: bob (not allowlisted) bypasses the allowlist via the router
vm.startPrank(bob);
token0.approve(address(router), type(uint256).max);

// Direct swap: reverts NotAllowedToSwap (sender = bob, not allowlisted)
// pool.swap(bob, false, 1000, type(uint128).max, "", "");  // ← blocked

// Router swap: succeeds (sender = router, which IS allowlisted)
router.exactInputSingle(
    IMetricOmmSimpleRouter.ExactInputSingleParams({
        pool:             address(pool),
        tokenIn:          address(token0),
        recipient:        bob,
        deadline:         block.timestamp + 1,
        amountIn:         1000,
        amountOutMinimum: 0,
        zeroForOne:       false,
        priceLimitX64:    0,
        extensionData:    ""
    })
);
// bob successfully swaps despite not being on the allowlist
vm.stopPrank();
```

The extension receives `sender = router`, finds `allowedSwapper[pool][router] = true`, and passes — bob's swap executes and LP assets are transferred out of the pool.
