### Title
Interest Rate Config Values Specified as APR but Compounded as APY in `SpotEngineState._updateState` — (File: `core/contracts/SpotEngineState.sol`)

---

### Summary

`SpotEngineState._updateState` labels its annualized rate parameters as APR (comment: `// annualized borrower rate`) and divides them by 31 536 000 to obtain a per-second rate, but then raises that per-second rate to the power of elapsed seconds using integer exponentiation — i.e., compound interest. The resulting APY is always higher than the configured APR, with the gap growing dramatically at high rates. The hardcoded initialization value `interestLargeCapX18 = ONE` (100 % APR) produces an actual APY of ≈ 171.8 %, a 71.8 percentage-point overcharge to every borrower at maximum utilization.

---

### Finding Description

In `SpotEngineState._updateState`, the borrow rate is computed as follows:

```
// annualized borrower rate          ← comment explicitly says APR
int128 borrowerRateX18 = config.interestFloorX18;
...
// convert to per second
borrowerRateX18 = borrowerRateX18.div(MathSD21x18.fromInt(31536000));
borrowRateMultiplierX18 = (ONE + borrowerRateX18).pow(int128(dt));
``` [1](#0-0) 

The `pow` function in `MathSD21x18` is integer exponentiation via repeated squaring:

```
function pow(int128 x, int128 y) internal pure returns (int128) {
    ...
    for (int128 i = 1; i <= y; i *= 2) { ... }
}
``` [2](#0-1) 

This computes `(1 + r_annual / 31_536_000)^dt`. Over one full year (`dt = 31_536_000` seconds) this equals `(1 + r/n)^n → e^r`, which is continuous compounding — not APR.

The same APR/APY confusion applies to `minDepositRateX18`:

```
int128 minDepositRatePerSecondX18 = minDepositRateX18.div(
    MathSD21x18.fromInt(31536000)
);
int128 minDepositRateMultiplierX18 = (ONE +
    minDepositRatePerSecondX18).pow(int128(dt));
``` [3](#0-2) 

The hardcoded initialization values for `QUOTE_PRODUCT_ID` in `SpotEngine.initialize` are annotated as APR fractions:

```
interestFloorX18:    1e16,  // .01   → intended 1 % APR
interestSmallCapX18: 4e16,  // .04   → intended 4 % APR
interestLargeCapX18: ONE,   // 1     → intended 100 % APR
``` [4](#0-3) 

Actual compounded APY for each:

| Config value | Intended APR | Actual APY |
|---|---|---|
| `interestFloorX18 = 1e16` | 1 % | ≈ 1.005 % |
| `interestSmallCapX18 = 4e16` | 4 % | ≈ 4.08 % |
| `interestLargeCapX18 = ONE` | 100 % | ≈ 171.8 % |

---

### Impact Explanation

Every borrower in the SpotEngine pays more interest than the labeled rate. At maximum utilization the borrow rate reaches `interestFloorX18 + interestSmallCapX18 + interestLargeCapX18` = 105 % APR, which compounds to ≈ 185 % APY — an 80 percentage-point overcharge. The excess interest is real: it accrues into `cumulativeBorrowsMultiplierX18`, inflating every borrower's outstanding debt beyond what the protocol's documented rate schedule implies. Depositors receive a corresponding windfall, and the protocol collects fees on the inflated interest via `INTEREST_FEE_FRACTION`. The corrupted state variable is `cumulativeBorrowsMultiplierX18`, which grows faster than the APR-labeled config values indicate. [5](#0-4) 

---

### Likelihood Explanation

This is a constant, always-on miscalibration. Every call to `updateStates` (triggered by the endpoint on every state-update cycle) applies the compounding multiplier. Any user who borrows spot assets — an unprivileged action available to all traders — is affected immediately and continuously. No special conditions, elevated privileges, or adversarial setup are required. [6](#0-5) 

---

### Recommendation

Specify all rate config values as APY-equivalent per-second rates, or correct the initialization constants to their APY-equivalent values. For a desired APY of `y`, the correct annualized input is `ln(1 + y)`. For example, to achieve a 100 % APY cap, set `interestLargeCapX18 = ln(2) * 1e18 ≈ 693_147_180_559_945_309`. Alternatively, document and enforce that all callers of `addOrUpdateProduct` must supply APY-equivalent values, and update the inline comments accordingly.

---

### Proof of Concept

1. `QUOTE_PRODUCT_ID` is initialized with `interestLargeCapX18 = ONE` (100 % APR). [7](#0-6) 
2. At 100 % utilization, `borrowerRateX18 = (1e16 + 4e16 + 1e18) / 31_536_000 ≈ 3.34e10` per second.
3. `_updateState` computes `borrowRateMultiplierX18 = (1 + 3.34e10 / 1e18)^31_536_000`. [8](#0-7) 
4. `(1 + 1.05/31_536_000)^31_536_000 ≈ e^1.05 ≈ 2.858`, i.e. ≈ 185.8 % APY vs. the labeled 105 % APR.
5. `cumulativeBorrowsMultiplierX18` is multiplied by this value each cycle, permanently inflating all borrower debts above the protocol's stated rate schedule. [5](#0-4)
