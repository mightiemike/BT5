### Title
Stale User-Supplied `oraclePriceX18` Accepted Without Freshness Validation in `MintNlp`/`BurnNlp` — (File: `core/contracts/EndpointTx.sol`, `core/contracts/Clearinghouse.sol`)

---

### Summary

`SignedMintNlp` and `SignedBurnNlp` each carry an `oraclePriceX18` field that is provided and signed by the user. Neither `EndpointTx.processTransactionImpl` nor `Clearinghouse.mintNlp`/`burnNlp` validates that this price is recent, within a reasonable deviation from the current stored price, or bounded by any staleness window. An attacker can sign and submit a transaction embedding a stale or manipulated price to over-mint NLP tokens (low price → more NLP per quote) or over-receive quote on burn (high price → more quote per NLP). The attacker-controlled entry path is `submitTransactionsCheckedWithGasLimit`, which lacks any `msg.sender == sequencer` guard.

---

```solidity

The user signs the full `transaction` bytes (type byte + ABI-encoded `SignedMintNlp`), so `oraclePriceX18` is covered by the EIP-712 digest. This means the user freely chooses any positive price and signs it — there is no protocol-enforced bound.

---

### Impact Explanation

**MintNlp attack (over-mint):** An attacker signs a `MintNlp` with `oraclePriceX18` set to a historically low (stale) price. `nlpAmount = quoteAmount / oraclePriceX18` yields a larger NLP balance than the current fair value warrants. The attacker then waits for the price to recover and burns at the correct price, extracting the spread as profit at the expense of other NLP holders (dilution of the pool).

**BurnNlp attack (over-redeem):** An attacker signs a `BurnNlp` with `oraclePriceX18` set to a historically high (stale) price. `quoteAmount = nlpAmount * oraclePriceX18` yields more quote than the NLP is currently worth, draining quote from the NLP pool.

In both cases the corrupted `oraclePriceX18` is also written to `priceX18[NLP_PRODUCT_ID]` and propagated to `spotEngine` via `updatePrice`, poisoning the stored price used in subsequent health calculations for all subaccounts holding NLP. [11](#0-10) [12](#0-11) 

---

### Likelihood Explanation

The entry path requires no special privilege: the attacker only needs to (1) hold a registered subaccount, (2) read the public `nSubmissions` counter, and (3) call `submitTransactionsCheckedWithGasLimit` with a self-signed `MintNlp`/`BurnNlp` embedding a stale price. Historical on-chain price data is freely available, making it straightforward to identify a favorable stale price. The attack is repeatable as long as the function remains unguarded.

---

### Recommendation

1. **Add a sequencer guard to `submitTransactionsCheckedWithGasLimit`** — add `require(msg.sender == sequencer, ERR_UNAUTHORIZED)` mirroring `submitTransactionsChecked`.
2. **Validate `oraclePriceX18` against the current stored price** — in `mintNlp`/`burnNlp`, compare the supplied price against `_getPriceX18(NLP_PRODUCT_ID)` and revert if the deviation exceeds a configurable threshold (e.g., ±5%).
3. **Alternatively, remove `oraclePriceX18` from the user-signed struct** — have the contract fetch the NLP price from the sequencer-controlled `priceX18[NLP_PRODUCT_ID]` mapping directly, eliminating the user-supplied price surface entirely.

---

### Proof of Concept

```
1. Attacker observes that NLP oracle price was 1.50 USDC six hours ago but is now 2.00 USDC.
2. Attacker reads nSubmissions (public) = N.
3. Attacker constructs SignedMintNlp {
       tx: MintNlp { sender: attacker, quoteAmount: 1000e18, nonce: currentNonce },
       signature: sign(fullTxBytes),   // signs the stale price into the digest
       oraclePriceX18: 1.50e18,        // stale low price
       nlpPoolRebalanceX18: [...]
   }
4. Attacker calls Endpoint.submitTransactionsCheckedWithGasLimit(N, [encodedMintNlp], gasLimit).
   - validateSubmissionIdx passes (idx == nSubmissions).
   - No sequencer check → proceeds.
   - processTransactionImpl → MintNlp branch:
       priceX18[NLP_PRODUCT_ID] = 1.50e18  (stale price stored)
       nlpAmount = 1000e18 / 1.50e18 = 666.67 NLP  (vs fair 500 NLP at 2.00)
5. Attacker now holds 666.67 NLP instead of the fair 500 NLP.
6. Attacker submits BurnNlp at the correct price (2.00e18), receiving 1333 USDC for 666.67 NLP.
   Net profit: 333 USDC extracted from the NLP pool at the expense of other LPs.
```

        uint64 idx,
        bytes[] calldata transactions,
        bytes32 e,
        bytes32 s,
        uint8 signerBitmask
    ) external {
        validateSubmissionIdx(idx);
        require(msg.sender == sequencer);
        // TODO: if one of these transactions fails this means the sequencer is in an error state
        // we should probably record this, and engage some sort of recovery mode

        bytes32 digest = keccak256(abi.encode(idx));
        for (uint256 i = 0; i < transactions.length; ++i) {
            digest = keccak256(abi.encodePacked(digest, transactions[i]));
        }
        verifier.requireValidSignature(digest, e, s, signerBitmask);

        for (uint256 i = 0; i < transactions.length; i++) {
            bytes calldata transaction = transactions[i];
            processTransaction(transaction);
            nSubmissions += 1;
        }
    }

    function submitTransactionsCheckedWithGasLimit(
        uint64 idx,
        bytes[] calldata transactions,
        uint256 gasLimit
    ) external {
        uint256 initialGas = gasleft();
        validateSubmissionIdx(idx);
        for (uint256 i = 0; i < transactions.length; i++) {
            bytes calldata transaction = transactions[i];
            processTransaction(transaction);
            uint256 gasUsed = initialGas - gasleft();
            if (gasUsed > gasLimit) {
                verifier.revertGasInfo(i, gasUsed);
            }
        }
        verifier.revertGasInfo(transactions.length, initialGas - gasleft());
    }
