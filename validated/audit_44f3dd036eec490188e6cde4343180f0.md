Audit Report

## Title
SwapAllowlistExtension Per-User Allowlist Bypassed via Router `sender` Mismatch — (File: metric-periphery/contracts/extensions/SwapAllowlistExtension.sol)

## Summary
`SwapAllowlistExtension.beforeSwap` gates swaps by checking `allowedSwapper[pool][sender]`, where `sender` is the `msg.sender` of `pool.swap()`. When users route through `MetricOmmSimpleRouter`, `sender` is the router address, not the actual end user. Allowlisting the router (required for any router-based swap) grants every non-allowlisted user the ability to bypass the per-user gate by calling the public router.

## Finding Description
`MetricOmmPool.swap()` passes `msg.sender` as the `sender` argument to `_beforeSwap`: [1](#0-0) 

`ExtensionCalling._beforeSwap` forwards that value unchanged to every configured extension: [2](#0-1) 

`SwapAllowlistExtension.beforeSwap` then checks:
```solidity
if (!allowAllSwappers[msg.sender] && !allowedSwapper[msg.sender][sender]) {
    revert IMetricOmmPoolActions.NotAllowedToSwap();
}
``` [3](#0-2) 

Here `msg.sender` is the pool (the extension is called by the pool via `CallExtension.callExtension`) and `sender` is whoever called `pool.swap()`. When the call originates from `MetricOmmSimpleRouter.exactInputSingle`, the router calls `pool.swap()` directly: [4](#0-3) 

So `sender = address(router)`, not the actual user. The pool admin's only options are: (a) do not allowlist the router — all router-based swaps revert, including for legitimately allowlisted users; or (b) allowlist the router — any user, allowlisted or not, bypasses the per-user gate. There is no configuration that simultaneously allows allowlisted users to swap through the router and blocks non-allowlisted users from doing the same.

The `setAllowedToSwap` setter accepts a `swapper` address, clearly signalling the intent to gate individual users: [5](#0-4) 

## Impact Explanation
Any user excluded from the allowlist can execute swaps in a pool the admin intended to restrict (e.g., KYC-gated, regulatory-restricted, or partner-only pools) simply by calling the public router. This is a direct admin-boundary break: an unprivileged path (`MetricOmmSimpleRouter`) nullifies an admin-configured access control, allowing unauthorized swap execution and the associated token flows. The corrupted value is the `allowedSwapper[pool][sender]` extension decision — it evaluates `true` for the router address rather than the actual end user, causing the access control check to pass for unauthorized actors.

## Likelihood Explanation
`MetricOmmSimpleRouter` is the primary user-facing entry point for swaps. Any pool that (a) deploys `SwapAllowlistExtension` with per-user restrictions and (b) needs to support router-based swaps is affected. The attacker needs no special privileges — only the ability to call the public router. The attack is repeatable and requires no setup beyond the pool admin having allowlisted the router.

## Recommendation
The extension must identify the actual end user, not the immediate caller of `pool.swap()`. Two viable approaches:

1. **`extensionData` forwarding**: Have the router encode `msg.sender` (the actual user) into `extensionData` and have `SwapAllowlistExtension` decode and check that address instead of (or in addition to) `sender`.
2. **Dual check**: Check both `sender` and a user address extracted from `extensionData`; require at least one to be allowlisted, and require the router itself to be a trusted intermediary before trusting `extensionData`.

## Proof of Concept
```
Setup:
  pool configured with SwapAllowlistExtension
  allowAllSwappers[pool] = false
  allowedSwapper[pool][alice] = true   // only alice is allowed
  allowedSwapper[pool][router] = true  // router allowlisted so alice can use it

Attack:
  bob (not allowlisted) calls:
    router.exactInputSingle({pool: pool, ..., recipient: bob})

  Execution path:
    router.exactInputSingle()
      → pool.swap(recipient=bob, ...) [msg.sender = router]
        → _beforeSwap(sender=router, ...)
          → SwapAllowlistExtension.beforeSwap(sender=router, ...)
            → allowedSwapper[pool][router] == true → PASSES
          → swap executes, bob receives tokens

Result: bob swaps successfully despite not being on the allowlist.
```
