Audit Report

## Title
`SwapAllowlistExtension.beforeSwap` Gates on Router Address Instead of End User, Enabling Full Allowlist Bypass — (`metric-periphery/contracts/extensions/SwapAllowlistExtension.sol`)

## Summary
`SwapAllowlistExtension.beforeSwap` checks the `sender` parameter, which the pool sets to its own `msg.sender` — the router contract — not the originating user. When a pool admin allowlists the router to permit router-mediated swaps, every unprivileged user can bypass the allowlist by calling through `MetricOmmSimpleRouter`. The access-control invariant of the permissioned pool is completely broken.

## Finding Description
`MetricOmmPool.swap` passes `msg.sender` as the first argument to `_beforeSwap`:

```solidity
// metric-core/contracts/MetricOmmPool.sol L230-240
_beforeSwap(
  msg.sender,   // ← this is the router when called via MetricOmmSimpleRouter
  recipient,
  ...
);
```

`_beforeSwap` forwards that value unchanged to the extension as `sender`. `SwapAllowlistExtension.beforeSwap` then gates exclusively on this value:

```solidity
// metric-periphery/contracts/extensions/SwapAllowlistExtension.sol L37
if (!allowAllSwappers[msg.sender] && !allowedSwapper[msg.sender][sender]) {
  revert IMetricOmmPoolActions.NotAllowedToSwap();
}
```

`MetricOmmSimpleRouter.exactInputSingle` calls `pool.swap` directly with no encoding of the original caller:

```solidity
// metric-periphery/contracts/MetricOmmSimpleRouter.sol L72-80
(int128 amount0Delta, int128 amount1Delta) = IMetricOmmPoolActions(params.pool)
  .swap(
    params.recipient,
    params.zeroForOne,
    MetricOmmSwapInputs.asAmountSpecifiedIn(params.amountIn),
    priceLimitX64,
    "",
    params.extensionData   // ← msg.sender (alice) is NOT encoded here
  );
```

From the pool's perspective, `msg.sender` = router address, so `sender` delivered to the extension = router address. The check resolves to `allowedSwapper[pool][router]`, not `allowedSwapper[pool][alice]`. Any pool admin who allowlists the router (the natural configuration to permit router-mediated swaps) renders the extension completely ineffective for all users. The `DepositAllowlistExtension` does not share this flaw because it gates on `owner`, which is the explicitly-supplied position owner address regardless of the calling intermediary.

## Impact Explanation
A pool configured with `SwapAllowlistExtension` to restrict swaps to KYC'd or institutional addresses is rendered completely open when the router is allowlisted. Any unprivileged EOA can call `MetricOmmSimpleRouter.exactInputSingle` (or `exactInput`, `exactOutputSingle`, `exactOutput`) and the extension passes because it sees the allowlisted router address. This breaks the core access-control invariant of the permissioned pool, allowing unauthorized principals to swap at oracle prices, drain pool liquidity, and extract fees — a broken core pool functionality and admin-boundary break with direct fund impact.

## Likelihood Explanation
The router is the standard, documented entry point for swaps. Any pool admin deploying a `SwapAllowlistExtension`-gated pool who also wants users to use the router will naturally allowlist the router address. The bypass requires no special privileges, no flash loans, and no unusual token behavior — a single `exactInputSingle` call from any EOA suffices. Likelihood is high.

## Recommendation
The extension must gate on the end user, not the intermediary. The most robust fix is to require the real caller to be encoded in `extensionData` by trusted routers, and have the extension decode and check that address when `sender` is a known router. Alternatively, document and enforce that `SwapAllowlistExtension` is incompatible with router-mediated swaps by reverting when `sender` is not an EOA (e.g., `sender.code.length > 0`). A router registry approach — where the extension checks `extensionData`-decoded user when `sender` is a registered router — provides the most complete solution.

## Proof of Concept
```
Setup:
  Pool P configured with SwapAllowlistExtension
  Admin allowlists router R: allowedSwapper[P][R] = true
  Alice (not allowlisted) wants to swap

Attack:
  Alice calls MetricOmmSimpleRouter.exactInputSingle({pool: P, ...})
  Router calls P.swap(recipient=alice, ...)          // msg.sender = router
  Pool calls _beforeSwap(sender=router, ...)
  SwapAllowlistExtension.beforeSwap(sender=router, ...)
    → allowedSwapper[P][router] == true  ✓ passes
  Swap executes — alice's swap goes through despite not being allowlisted

Result:
  SwapAllowlistExtension guard is completely bypassed.
  Any user can swap on the "permissioned" pool via the router.
```
