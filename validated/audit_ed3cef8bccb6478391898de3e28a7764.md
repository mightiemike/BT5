Audit Report

## Title
`SwapAllowlistExtension.beforeSwap` Checks Router Address Instead of Actual Caller, Enabling Full Allowlist Bypass via `MetricOmmSimpleRouter` — (File: `metric-periphery/contracts/extensions/SwapAllowlistExtension.sol`)

## Summary
`SwapAllowlistExtension.beforeSwap` gates swaps by checking the `sender` argument it receives, which is `msg.sender` of `MetricOmmPool.swap()`. When `MetricOmmSimpleRouter` calls `pool.swap()`, `sender` equals the router's address, not the actual user. If the pool admin allowlists the router — the only way to let approved users reach the pool through the standard periphery — every unprivileged address can bypass the allowlist by routing through `MetricOmmSimpleRouter`.

## Finding Description
`MetricOmmPool.swap()` passes `msg.sender` as the `sender` argument to `_beforeSwap`:

```solidity
// MetricOmmPool.sol L230-240
_beforeSwap(
    msg.sender,   // whoever called pool.swap()
    recipient, ...
);
```

`SwapAllowlistExtension.beforeSwap` receives that value and checks it against the per-pool allowlist:

```solidity
// SwapAllowlistExtension.sol L37
if (!allowAllSwappers[msg.sender] && !allowedSwapper[msg.sender][sender]) {
    revert IMetricOmmPoolActions.NotAllowedToSwap();
}
```

Here `msg.sender` is the pool and `sender` is whoever called `pool.swap()`. When a user routes through `MetricOmmSimpleRouter.exactInputSingle()`:

```solidity
// MetricOmmSimpleRouter.sol L72-80
IMetricOmmPoolActions(params.pool).swap(
    params.recipient,
    params.zeroForOne,
    ...,
    params.extensionData   // user-supplied, router does NOT inject msg.sender here
);
```

The pool sees `msg.sender` = router, so the extension receives `sender` = router. The check becomes `allowedSwapper[pool][router]`. The actual user's address is never consulted. The router passes `params.extensionData` directly from user input without encoding the real caller, and the extension does not decode any caller identity from `extensionData`. The same bypass applies to `exactInput`, `exactOutputSingle`, and `exactOutput`, all of which call `pool.swap()` directly from the router.

## Impact Explanation
The `SwapAllowlistExtension` is the protocol's primary mechanism for restricting swap access to a curated set of addresses. When the router is allowlisted, the guard is completely neutralised: any unprivileged address can execute swaps on a pool configured to be restricted. This constitutes an admin-boundary break — a pool admin-configured guard is bypassed by an unprivileged path (the public router) — with potential direct financial loss to LPs whose pool was intended to trade only with trusted, low-risk counterparties.

## Likelihood Explanation
The bypass requires the pool admin to have added the router to the allowlist. This is not exotic — it is the only way to let allowlisted users interact with the pool through the standard periphery. A pool admin who deploys a restricted pool and then wants their approved users to use `MetricOmmSimpleRouter` will naturally call `setAllowedToSwap(pool, router, true)`. At that moment the allowlist is silently voided for all users. The trigger is a routine, well-motivated admin action, not an adversarial setup.

## Recommendation
The extension must gate the original economic actor, not the intermediary. Two complementary fixes:

1. **Pass the real user through `extensionData`**: Have the router encode `msg.sender` into `extensionData` and have `SwapAllowlistExtension` decode and check that value when present. This requires a convention between router and extension but keeps the architecture flexible.

2. **Disallow the router from being added to the allowlist**: Document that router-mediated swaps are incompatible with `SwapAllowlistExtension`. Users on allowlisted pools must call `pool.swap()` directly.

Either way, the extension must never treat an allowlisted intermediary as a proxy grant for all downstream callers.

## Proof of Concept
**Setup:**
- Pool deployed with `SwapAllowlistExtension` on `BEFORE_SWAP_ORDER`.
- Pool admin calls `setAllowedToSwap(pool, alice, true)` — Alice is the approved user.
- Pool admin calls `setAllowedToSwap(pool, router, true)` — added so Alice can use the router.

**Attack (Bob, not allowlisted):**
1. Bob calls `MetricOmmSimpleRouter.exactInputSingle({pool: restrictedPool, ...})`.
2. Router calls `restrictedPool.swap(recipient, ...)` — `msg.sender` in pool = router.
3. Pool calls `_beforeSwap(router, ...)`.
4. `ExtensionCalling._callExtensionsInOrder` calls `SwapAllowlistExtension.beforeSwap(router, ...)`.
5. Extension evaluates `allowedSwapper[restrictedPool][router]` → `true`.
6. Hook returns the success selector; swap executes in full.
7. Bob receives output tokens from a pool he was never authorised to trade on.
