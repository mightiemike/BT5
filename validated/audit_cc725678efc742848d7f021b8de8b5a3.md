### Title
`ValidResourceBounds` Protobuf Deserialization Silently Downgrades Post-0.13.3 `AllResources` Transactions to `L1Gas` Variant, Producing a Divergent Transaction Hash - (`crates/apollo_protobuf/src/converters/transaction.rs`)

---

### Summary

The protobuf converter for `ValidResourceBounds` classifies any transaction as the pre-0.13.3 `L1Gas` variant whenever `l2_gas == 0` and `l1_data_gas == 0`, even when the transaction was originally submitted and hashed as a post-0.13.3 `AllResources` transaction. Because `get_tip_resource_bounds_hash` produces structurally different hash preimages for the two variants (the `AllResources` path appends an extra `L1_DATA_GAS` element), the transaction hash computed after protobuf round-trip diverges from the hash that was originally signed and stored. This breaks hash canonicalization across the public-to-internal conversion boundary.

---

### Finding Description

**Step 1 — Two variants produce different hash preimages.**

`get_tip_resource_bounds_hash` in `crates/starknet_api/src/transaction_hash.rs` conditionally appends the `L1_DATA_GAS` element only for the `AllResources` variant:

```rust
// L1 and L2 gas bounds always exist.
let mut resource_felts = vec![
    get_concat_resource(&l1_resource_bounds, L1_GAS)?,
    get_concat_resource(&l2_resource_bounds, L2_GAS)?,
];
// For new V3 txs, need to also hash the data gas bounds.
resource_felts.extend(match resource_bounds {
    ValidResourceBounds::L1Gas(_) => vec![],                          // 2 elements
    ValidResourceBounds::AllResources(all_resources) => {
        vec![get_concat_resource(&all_resources.l1_data_gas, L1_DATA_GAS)?]  // 3 elements
    }
});
```

For `L1Gas(X)`: `poseidon([tip, L1_GAS_packed(X), L2_GAS_packed(0)])` — **2 resource felts**.
For `AllResources(X, 0, 0)`: `poseidon([tip, L1_GAS_packed(X), L2_GAS_packed(0), L1_DATA_GAS_packed(0)])` — **3 resource felts**.

These are cryptographically distinct hashes even when all numeric values are identical.

**Step 2 — The gateway accepts `AllResourceBounds` with zero L2/L1DataGas.**

`RpcInvokeTransactionV3` uses `AllResourceBounds` (not `ValidResourceBounds`). The gateway stateless validator explicitly accepts transactions with only `l1_gas` non-zero and `l2_gas = l1_data_gas = 0`. The hash is computed via `InternalRpcInvokeTransactionV3::calculate_transaction_hash` → `get_invoke_transaction_v3_hash` → `get_tip_resource_bounds_hash` with `ValidResourceBounds::AllResources(...)`, producing **H_all** (3-element preimage).

**Step 3 — The protobuf converter silently downgrades to `L1Gas`.**

`TryFrom<protobuf::ResourceBounds> for ValidResourceBounds` in `crates/apollo_protobuf/src/converters/transaction.rs`:

```rust
Ok(if l1_data_gas.is_zero() && l2_gas.is_zero() {
    ValidResourceBounds::L1Gas(l1_gas)          // ← pre-0.13.3 variant
} else {
    ValidResourceBounds::AllResources(AllResourceBounds { l1_gas, l2_gas, l1_data_gas })
})
```

When a post-0.13.3 `AllResources(X, 0, 0)` transaction is serialized to protobuf and deserialized on a syncing node, the converter produces `L1Gas(X)`. The hash is then recomputed as **H_l1** (2-element preimage). **H_all ≠ H_l1**.

**Step 4 — The `AllResources` protobuf converter for RPC transactions does not have this bug.**

`TryFrom<protobuf::ResourceBounds> for AllResourceBounds` in `crates/apollo_protobuf/src/converters/rpc_transaction.rs` always produces `AllResourceBounds` unconditionally. The bug is isolated to the `ValidResourceBounds` converter used for `InvokeTransactionV3` (the storage/state-sync type).

**Step 5 — Hash divergence propagates to execution.**

`executable_transaction::InvokeTransaction::create` recomputes the hash from the `InvokeTransaction` body:

```rust
let tx_hash = invoke_tx.calculate_transaction_hash(chain_id, &invoke_tx.version())?;
```

If the body carries `L1Gas(X)` (from protobuf deserialization), the stored `tx_hash` is **H_l1**, which is then returned by the `get_tx_info` syscall to the executing contract — a different value than the signer committed to.

---

### Impact Explanation

**High — Transaction conversion or signature/hash logic binds the wrong hash to the executable payload.**

Any syncing node that receives a block containing a V3 invoke transaction with `AllResources(l1_gas, l2_gas=0, l1_data_gas=0)` via protobuf state sync will:
1. Deserialize the transaction as `L1Gas` variant.
2. Recompute a hash **H_l1 ≠ H_all**.
3. Either reject the block (if hash validation is performed) or store and execute the transaction with the wrong `tx_hash`, causing `get_tx_info` to return an incorrect transaction hash to the contract — a wrong execution result for accepted input.

---

### Likelihood Explanation

The trigger is a normal, gateway-accepted V3 invoke transaction with only `l1_gas` non-zero. The gateway test suite explicitly validates this as a passing case (`valid_l1_gas`). No special privileges are required. Any user submitting such a transaction causes the divergence on every node that syncs the block via protobuf.

---

### Recommendation

The `TryFrom<protobuf::ResourceBounds> for ValidResourceBounds` converter must not use the `L1Gas` variant for transactions that were submitted post-0.13.3. Options:

1. **Add a version discriminator to the protobuf message** so the deserializer knows whether the original transaction was pre- or post-0.13.3, and always produces `AllResources` for post-0.13.3 transactions.
2. **Always produce `AllResources`** in this converter and reserve `L1Gas` only for explicitly tagged legacy transactions (matching the approach used in `rpc_transaction.rs`).
3. **Remove the `L1Gas` shortcut** from the converter entirely, since all new transactions use `AllResources`, and handle legacy transactions through a separate, explicitly versioned path.

---

### Proof of Concept

```
1. User submits RpcInvokeTransactionV3 with:
       resource_bounds = AllResourceBounds {
           l1_gas:      { max_amount: 1000, max_price_per_unit: 1 },
           l2_gas:      { max_amount: 0,    max_price_per_unit: 0 },
           l1_data_gas: { max_amount: 0,    max_price_per_unit: 0 },
       }

2. Gateway computes H_all via get_tip_resource_bounds_hash(AllResources(...)):
       resource_felts = [L1_GAS_packed, L2_GAS_packed(0), L1_DATA_GAS_packed(0)]  // 3 elements
       H_all = poseidon([tip, L1_GAS_packed, L2_GAS_packed(0), L1_DATA_GAS_packed(0), ...])

3. Transaction is sequenced, stored in block with tx_hash = H_all.

4. Syncing node receives block via protobuf state sync.
   TryFrom<protobuf::ResourceBounds> for ValidResourceBounds:
       l1_data_gas.is_zero() && l2_gas.is_zero()  →  ValidResourceBounds::L1Gas(l1_gas)

5. Syncing node computes H_l1 via get_tip_resource_bounds_hash(L1Gas(...)):
       resource_felts = [L1_GAS_packed, L2_GAS_packed(0)]  // 2 elements — L1_DATA_GAS absent
       H_l1 = poseidon([tip, L1_GAS_packed, L2_GAS_packed(0), ...])

6. H_all ≠ H_l1.
   - If hash validation is performed: valid block rejected.
   - If hash is recomputed for execution: get_tx_info syscall returns H_l1 ≠ H_all,
     diverging from the value the signer committed to.
```

impl TryFrom<protobuf::ResourceBounds> for ValidResourceBounds {
    type Error = ProtobufConversionError;
    fn try_from(value: protobuf::ResourceBounds) -> Result<Self, Self::Error> {
        let Some(l1_gas) = value.l1_gas else {
            return Err(missing("ResourceBounds::l1_gas"));
        };
        let Some(l2_gas) = value.l2_gas else {
            return Err(missing("ResourceBounds::l2_gas"));
        };
        // TODO(Shahak): Assert data gas is not none once we remove support for 0.13.2.
        let l1_data_gas = value.l1_data_gas.unwrap_or_default();
        let l1_gas: ResourceBounds = l1_gas.try_into()?;
        let l2_gas: ResourceBounds = l2_gas.try_into()?;
        let l1_data_gas: ResourceBounds = l1_data_gas.try_into()?;

            ValidResourceBounds::L1Gas(l1_gas)
        } else {
            ValidResourceBounds::AllResources(AllResourceBounds { l1_gas, l2_gas, l1_data_gas })

pub fn get_tip_resource_bounds_hash(
    resource_bounds: &ValidResourceBounds,
    tip: &Tip,
) -> Result<Felt, StarknetApiError> {
    let l1_resource_bounds = resource_bounds.get_l1_bounds();
    let l2_resource_bounds = resource_bounds.get_l2_bounds();

    // L1 and L2 gas bounds always exist.
    // Old V3 txs always have L2 gas bounds of zero, but they exist.
    let mut resource_felts = vec![
        get_concat_resource(&l1_resource_bounds, L1_GAS)?,
        get_concat_resource(&l2_resource_bounds, L2_GAS)?,
    ];

    // For new V3 txs, need to also hash the data gas bounds.
    resource_felts.extend(match resource_bounds {
        ValidResourceBounds::L1Gas(_) => vec![],
        ValidResourceBounds::AllResources(all_resources) => {
            vec![get_concat_resource(&all_resources.l1_data_gas, L1_DATA_GAS)?]
        }
    });

    Ok(HashChain::new().chain(&tip.0.into()).chain_iter(resource_felts.iter()).get_poseidon_hash())
}

pub(crate) fn get_invoke_transaction_v3_hash<T: InvokeTransactionV3Trait>(
    transaction: &T,
    chain_id: &ChainId,
    transaction_version: &TransactionVersion,
) -> Result<TransactionHash, StarknetApiError> {
    let tip_resource_bounds_hash =
        get_tip_resource_bounds_hash(&transaction.resource_bounds(), transaction.tip())?;
    let paymaster_data_hash =
        HashChain::new().chain_iter(transaction.paymaster_data().0.iter()).get_poseidon_hash();
    let data_availability_mode = concat_data_availability_mode(
        transaction.nonce_data_availability_mode(),
        transaction.fee_data_availability_mode(),
    );
    let account_deployment_data_hash = HashChain::new()
        .chain_iter(transaction.account_deployment_data().0.iter())
        .get_poseidon_hash();
    let calldata_hash =
        HashChain::new().chain_iter(transaction.calldata().0.iter()).get_poseidon_hash();
    let mut hash_chain = HashChain::new()
        .chain(&INVOKE)
        .chain(&transaction_version.0)
        .chain(transaction.sender_address().0.key())
        .chain(&tip_resource_bounds_hash)
        .chain(&paymaster_data_hash)
        .chain(&Felt::try_from(chain_id)?)
        .chain(&transaction.nonce().0)
        .chain(&data_availability_mode)
        .chain(&account_deployment_data_hash)
        .chain(&calldata_hash);
    if !transaction.proof_facts().0.is_empty() {
        let proof_facts_hash =
            HashChain::new().chain_iter(transaction.proof_facts().0.iter()).get_poseidon_hash();
        hash_chain = hash_chain.chain(&proof_facts_hash);
    }
    Ok(TransactionHash(hash_chain.get_poseidon_hash()))
}
