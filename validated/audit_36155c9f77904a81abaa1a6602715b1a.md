### Title
`SwapAllowlistExtension` checks router address instead of actual swapper when swaps route through `MetricOmmSimpleRouter`, enabling complete allowlist bypass — (`metric-periphery/contracts/extensions/SwapAllowlistExtension.sol`)

---

### Summary

`SwapAllowlistExtension.beforeSwap` gates swaps by checking the `sender` argument it receives from the pool. The pool always sets `sender = msg.sender` of the `swap` call. When a user routes through `MetricOmmSimpleRouter`, the pool's `msg.sender` is the router contract, not the actual user. The extension therefore checks the router's allowlist entry, not the real swapper's. If a pool admin allowlists the router to restore router-mediated swap access for their curated pool, the gate is silently opened to every address on-chain.

---

### Finding Description

**Actor binding chain:** [1](#0-0) 

`MetricOmmPool.swap` passes `msg.sender` as `sender` to `_beforeSwap`: [2](#0-1) 

`ExtensionCalling._beforeSwap` encodes that same `sender` value and forwards it verbatim to every configured extension. [3](#0-2) 

`SwapAllowlistExtension.beforeSwap` checks `allowedSwapper[msg.sender][sender]` — where `msg.sender` is the pool and `sender` is whoever called `pool.swap`.

**Router path:** [4](#0-3) 

`MetricOmmSimpleRouter.exactInputSingle` calls `pool.swap(params.recipient, ...)` directly. The pool therefore sees `msg.sender = router`. The extension receives `sender = router` and checks `allowedSwapper[pool][router]` — the actual user's address is never consulted.

The same wrong-actor binding applies to `exactOutputSingle`, `exactInput`, and `exactOutput`. [5](#0-4) 

---

### Impact Explanation

Two fund-impacting failure modes arise from the same root cause:

**Mode A — Broken core swap flow for allowlisted users.**  
A pool admin allowlists specific users (e.g., KYC addresses) via `setAllowedToSwap(pool, alice, true)`. Alice calls `exactInputSingle` through the router. The extension sees `sender = router`, which is not allowlisted, and reverts with `NotAllowedToSwap`. Alice's swap is permanently blocked through the only supported periphery path, making the curated pool unusable for its intended participants.

**Mode B — Complete allowlist bypass by any unprivileged user.**  
To restore router access for Alice, the pool admin calls `setAllowedToSwap(pool, router, true)`. This is a natural remediation step. The extension now passes for any caller who routes through the router, because `allowedSwapper[pool][router] = true`. Bob — who was never allowlisted — calls `exactInputSingle` through the router and swaps freely on the curated pool. The allowlist is entirely nullified for all router-mediated swaps. [6](#0-5) 

---

### Likelihood Explanation

The trigger is a semi-trusted pool admin action that is both natural and well-motivated: allowlisting the router to restore router-mediated swap access after discovering Mode A. The admin has no on-chain signal that doing so opens the gate to all users. The `setAllowedToSwap` setter accepts any address and emits only a generic `AllowedToSwapSet` event with no warning. The router is a factory-registered, protocol-endorsed contract, making it a plausible allowlist entry. [7](#0-6) 

---

### Recommendation

The extension must gate the economically relevant actor — the human initiating the swap — not the intermediary contract. Two viable approaches:

1. **Extension-data forwarding:** The router encodes the original `msg.sender` into `extensionData` for each hop. The extension decodes and checks that address instead of `sender`. The router already forwards per-hop `extensionData` to the pool.

2. **Separate payer/swapper fields:** Extend the `beforeSwap` hook signature (or use a dedicated field in `extensionData`) so the router can attest the real initiator, and the extension verifies the attestation came from a trusted router address.

Until fixed, the `SwapAllowlistExtension` should be documented as incompatible with `MetricOmmSimpleRouter` for pools that require per-user swap gating.

---

### Proof of Concept

```
Setup:
  pool configured with SwapAllowlistExtension
  setAllowedToSwap(pool, alice, true)   // only Alice is allowed

Step 1 — Mode A (broken flow):
  alice calls router.exactInputSingle({pool: pool, ...})
  pool.swap(msg.sender=router) → _beforeSwap(sender=router)
  extension: allowedSwapper[pool][router] == false → revert NotAllowedToSwap
  Alice cannot use the router at all.

Step 2 — Admin remediation (natural but fatal):
  admin calls setAllowedToSwap(pool, router, true)

Step 3 — Mode B (bypass):
  bob (never allowlisted) calls router.exactInputSingle({pool: pool, ...})
  pool.swap(msg.sender=router) → _beforeSwap(sender=router)
  extension: allowedSwapper[pool][router] == true → passes
  Bob swaps freely on the curated pool.
