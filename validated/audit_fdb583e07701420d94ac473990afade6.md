Audit Report

## Title
Router-Mediated Swaps Pass Router Address as `sender` to `SwapAllowlistExtension::beforeSwap`, Breaking Allowlist Identity Invariant — (`metric-periphery/contracts/extensions/SwapAllowlistExtension.sol`)

## Summary
`SwapAllowlistExtension::beforeSwap` gates swaps by checking `allowedSwapper[pool][sender]`, where `sender` is the immediate caller of `MetricOmmPool::swap`. When a user routes through `MetricOmmSimpleRouter`, the pool receives `msg.sender = router`, so the hook checks whether the **router** is allowlisted rather than the actual user. This breaks the allowlist in both directions: allowlisted users are blocked from the router, and allowlisting the router to restore access opens the gate to every address.

## Finding Description
In `MetricOmmPool::swap`, `_beforeSwap` is called with `msg.sender` as the first argument:

```solidity
// metric-core/contracts/MetricOmmPool.sol L230–240
_beforeSwap(
  msg.sender,   // ← always the immediate pool caller
  recipient, zeroForOne, amountSpecified,
  priceLimitX64, packedSlot0Initial, bidPriceX64, askPriceX64, extensionData
);
```

`ExtensionCalling::_beforeSwap` forwards this value unchanged as `sender` to every configured extension (lines 149–177). `SwapAllowlistExtension::beforeSwap` then evaluates:

```solidity
// metric-periphery/contracts/extensions/SwapAllowlistExtension.sol L37
if (!allowAllSwappers[msg.sender] && !allowedSwapper[msg.sender][sender]) {
  revert IMetricOmmPoolActions.NotAllowedToSwap();
}
```

Here `msg.sender` is the pool and `sender` is whoever called `pool.swap`. When `MetricOmmSimpleRouter::exactInputSingle` is used, the router calls `pool.swap(...)` directly as itself (lines 71–80), so `sender = router`. The original user's address is stored only in transient callback context for payment purposes and is never passed into the swap call or the extension hook. The extension therefore checks `allowedSwapper[pool][router]`, not `allowedSwapper[pool][user]`.

## Impact Explanation
Two concrete broken outcomes on any live pool with `allowAllSwappers = false`:

1. **Allowlisted users cannot use the router.** A pool admin allowlists `userA`. `userA` calls `exactInputSingle`; the hook sees `sender = router`, which is not allowlisted, and reverts `NotAllowedToSwap`. The user is blocked from the protocol's primary public entry point despite being explicitly permitted. This constitutes broken core swap functionality.

2. **Allowlisting the router defeats per-user access control entirely.** If the admin adds the router to the allowlist to restore router access, every address can swap through the router because the hook no longer distinguishes between users. The allowlist becomes meaningless, constituting a complete bypass of the intended access gate.

Both outcomes satisfy the "Broken core pool functionality causing unusable swap flows" impact criterion.

## Likelihood Explanation
Any pool deploying `SwapAllowlistExtension` with `allowAllSwappers = false` is immediately affected the moment a user attempts to swap via `MetricOmmSimpleRouter`. No special attacker capability is required — a standard `exactInputSingle` call is sufficient. The router is the canonical periphery entry point; most non-technical users will use it exclusively. The condition is triggered on every router-mediated swap to an allowlisted pool.

## Recommendation
The extension must verify the originating user, not the immediate pool caller. Viable approaches:

- **Embed the real user address in `extensionData`** and have the extension verify it against a signature or trusted forwarder registry. The router already has `msg.sender` available at `exactInputSingle` entry and could encode it into `params.extensionData`.
- **Redesign the hook signature** to carry a verified `originSender` that the pool populates from a trusted source (e.g., a callback-verified payer address already stored in transient storage by the router, which the pool could read before invoking extensions).
- **Allowlist at the router level** with the extension trusting only a canonical router address, though this requires the extension to know the router and still requires a per-user check inside the router.
