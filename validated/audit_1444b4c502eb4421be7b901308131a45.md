### Title
SwapAllowlistExtension checks router address instead of actual swapper when accessed via MetricOmmSimpleRouter, enabling allowlist bypass — (`metric-periphery/contracts/extensions/SwapAllowlistExtension.sol`)

---

### Summary

The `SwapAllowlistExtension.beforeSwap` hook gates swaps by checking `allowedSwapper[msg.sender][sender]`, where `msg.sender` is the pool and `sender` is the immediate caller of the pool's `swap()`. When a user routes through `MetricOmmSimpleRouter`, the pool's `msg.sender` is the **router**, not the actual user. The hook therefore checks whether the router is allowlisted — not whether the real user is allowlisted. This creates an irreconcilable Catch-22: if the router is not allowlisted, legitimate allowlisted users cannot use the router at all; if the router is allowlisted to enable router-mediated swaps, any user can bypass the allowlist by routing through it.

---

### Finding Description

**Call chain — direct pool call (correct):**

1. User → `MetricOmmPool.swap()`: `msg.sender = user`
2. Pool calls `_beforeSwap(msg.sender, ...)` → `_beforeSwap(user, ...)` [1](#0-0) 

3. `ExtensionCalling._beforeSwap` encodes `sender = user` and calls the extension. [2](#0-1) 

4. `SwapAllowlistExtension.beforeSwap(sender = user, ...)`: checks `allowedSwapper[pool][user]` ✓ [3](#0-2) 

---

**Call chain — router call (broken):**

1. User → `MetricOmmSimpleRouter.exactInputSingle()`: `msg.sender = user`
2. Router calls `IMetricOmmPoolActions(params.pool).swap(...)`: pool's `msg.sender = router` [4](#0-3) 

3. Pool calls `_beforeSwap(msg.sender, ...)` → `_beforeSwap(router, ...)`
4. `SwapAllowlistExtension.beforeSwap(sender = router, ...)`: checks `allowedSwapper[pool][router]` ✗

The hook receives the router's address as `sender`, not the actual user's address. The router passes no user-identity information to the pool.

---

**The Catch-22:**

| Router allowlist state | Effect |
|---|---|
| Router **not** allowlisted | All router swaps revert for everyone, including legitimately allowlisted users — broken core functionality |
| Router **allowlisted** | Any user bypasses the allowlist by routing through the router — allowlist is nullified |

There is no configuration of `SwapAllowlistExtension` that simultaneously allows legitimate router-mediated swaps and enforces per-user access control.

---

### Impact Explanation

A pool configured with `SwapAllowlistExtension` to restrict trading to a curated set of addresses cannot enforce that restriction when the public router is used. Any user can call `exactInputSingle` / `exactInput` / `exactOutputSingle` / `exactOutput` on `MetricOmmSimpleRouter` and trade on an allowlisted pool, provided the pool admin has allowlisted the router (which is the only way to enable router-mediated swaps at all). This directly breaks the curation invariant and constitutes a broken core pool functionality finding.

---

### Likelihood Explanation

The `MetricOmmSimpleRouter` is the canonical supported periphery path. Pool admins who want to allow router-mediated swaps must allowlist the router, at which point the bypass is available to any unprivileged user with no special setup. The attacker only needs to call the public router.

---

### Recommendation

The `SwapAllowlistExtension` must receive the **original user's address**, not the immediate pool caller. Options:

1. **Pass the original sender via `extensionData`**: The router encodes `msg.sender` (the actual user) into `extensionData` before calling the pool. The extension reads and verifies it. This requires the extension to trust the encoding, which introduces its own risks unless the pool validates the caller is a known router.
2. **Check `tx.origin`**: Simple but generally discouraged and incompatible with smart-contract wallets.
3. **Router-aware allowlist**: The extension checks whether `sender` is a known router, and if so, decodes the real user from `extensionData` (with the pool enforcing that only trusted routers can set this field).

---

### Proof of Concept

```
Setup:
- Deploy pool with SwapAllowlistExtension
- Pool admin calls setAllowedToSwap(pool, alice, true)   // alice is allowlisted
- Pool admin calls setAllowedToSwap(pool, router, true)  // router must be allowlisted to allow any router swaps

Attack:
- bob (not allowlisted) calls MetricOmmSimpleRouter.exactInputSingle({pool: pool, ...})
- Router calls pool.swap(...) with msg.sender = router
- beforeSwap receives sender = router
- allowedSwapper[pool][router] == true  →  check passes
- bob's swap executes successfully despite not being allowlisted

Assert: bob received output tokens from an allowlist-gated pool.
``` [5](#0-4) [6](#0-5)
