### Title
Protobuf `ValidResourceBounds` Conversion Silently Downgrades `AllResources` to `L1Gas`, Producing a Different Transaction Hash for Invoke V3 Transactions with Zero L2/Data-Gas Bounds ŌĆö (`crates/apollo_protobuf/src/converters/transaction.rs`)

---

### Summary

The external bug's core invariant is: **wrong dispatch based on input type causes the wrong execution path to be selected**, silently producing incorrect output for a specific class of inputs. The Sequencer analog is: the protobuf `TryFrom<protobuf::ResourceBounds> for ValidResourceBounds` conversion silently maps an `AllResources` transaction (with zero `l2_gas` and `l1_data_gas`) to the `L1Gas` variant. Because `get_tip_resource_bounds_hash` branches on this variant to decide whether to include `L1_DATA_GAS` in the hash preimage, the recomputed hash diverges from the hash that was computed at the gateway ŌĆö breaking the canonicalization invariant across the P2P/sync boundary.

---

### Finding Description

**Step 1 ŌĆö Gateway computes hash with `AllResources`.**

`InternalRpcInvokeTransactionV3` always stores resource bounds as `AllResourceBounds` and its `InvokeTransactionV3Trait` impl always returns `ValidResourceBounds::AllResources(self.resource_bounds)`: [1](#0-0) 

`InternalRpcTransactionWithoutTxHash::calculate_transaction_hash` calls `get_invoke_transaction_v3_hash`, which calls `get_tip_resource_bounds_hash`: [2](#0-1) 

For `AllResources`, the hash preimage is `[tip, L1_GAS_concat, L2_GAS_concat, L1_DATA_GAS_concat]` ŌĆö **four elements**, even when `l2_gas` and `l1_data_gas` are zero: [3](#0-2) 

**Step 2 ŌĆö Gateway accepts `AllResources` with zero L2/data-gas.**

The stateless validator explicitly accepts `AllResourceBounds` where `l2_gas` and `l1_data_gas` are zero (e.g., `l1_gas`-only transactions submitted in the new format): [4](#0-3) 

The hash stored in `InternalRpcTransaction.tx_hash` is therefore computed over the four-element `AllResources` preimage.

**Step 3 ŌĆö Protobuf round-trip silently downgrades the variant.**

When the transaction is propagated via P2P and deserialized, `TryFrom<protobuf::ResourceBounds> for ValidResourceBounds` applies this logic: [5](#0-4) 

If `l1_data_gas` is absent (old-format peer) it defaults to zero. Then, because both `l1_data_gas.is_zero()` and `l2_gas.is_zero()`, the result is `ValidResourceBounds::L1Gas(l1_gas)` ŌĆö **not** `AllResources`.

**Step 4 ŌĆö Recomputed hash uses the wrong preimage.**

With `L1Gas`, `get_tip_resource_bounds_hash` produces a **three-element** preimage `[tip, L1_GAS_concat, L2_GAS_concat(zero)]` ŌĆö the `L1_DATA_GAS_concat(zero)` element is absent: [6](#0-5) 

The full invoke-V3 hash is then finalized over this shorter chain: [7](#0-6) 

The resulting hash `H_l1` ŌēĀ `H_all` (the gateway-computed hash), even though every field value is identical.

**Step 5 ŌĆö Hash validation fails for valid transactions.**

`validate_transaction_hash` is called during central sync to verify that the hash stored in a block matches the hash recomputed from the transaction fields: [8](#0-7) 

A syncing node that receives a block containing such a transaction via protobuf will recompute `H_l1`, compare it to the block-stored `H_all`, find a mismatch, and reject the block ŌĆö even though the block and transaction are fully valid.

---

### Impact Explanation

A valid `InvokeTransactionV3` with `AllResources` bounds where `l2_gas = 0` and `l1_data_gas = 0` is accepted by the gateway and included in a block with hash `H_all`. Any node that receives this block via P2P and recomputes the transaction hash from the protobuf-deserialized fields will compute `H_l1 ŌēĀ H_all` and reject the block. This is a **transaction conversion / hash-binding failure** that causes valid blocks to be rejected by syncing nodes, matching the "High ŌĆö Transaction conversion or signature/hash logic binds the wrong signer, hash, type, or executable payload" impact.

---

### Likelihood Explanation

The gateway explicitly accepts `AllResources` transactions with zero `l2_gas` and `l1_data_gas` (confirmed by test cases). Any user submitting such a transaction triggers the divergence. The protobuf conversion path is exercised by every P2P-syncing node. No special privilege is required.

---

### Recommendation

In `TryFrom<protobuf::ResourceBounds> for ValidResourceBounds`, do not downgrade to `L1Gas` based on zero values. Instead, use the presence or absence of the `l1_data_gas` field (i.e., `value.l1_data_gas.is_none()`) as the discriminant between old-format (`L1Gas`) and new-format (`AllResources`) transactions:

```rust
Ok(match value.l1_data_gas {
    None => ValidResourceBounds::L1Gas(l1_gas),
    Some(_) => ValidResourceBounds::AllResources(AllResourceBounds { l1_gas, l2_gas, l1_data_gas }),
})
```

This preserves the `AllResources` variant for new transactions regardless of whether the gas values happen to be zero, ensuring the hash preimage is identical on both sides of the protobuf boundary.

pub fn get_tip_resource_bounds_hash(
    resource_bounds: &ValidResourceBounds,
    tip: &Tip,
) -> Result<Felt, StarknetApiError> {
    let l1_resource_bounds = resource_bounds.get_l1_bounds();
    let l2_resource_bounds = resource_bounds.get_l2_bounds();

    // L1 and L2 gas bounds always exist.
    // Old V3 txs always have L2 gas bounds of zero, but they exist.
    let mut resource_felts = vec![
        get_concat_resource(&l1_resource_bounds, L1_GAS)?,
        get_concat_resource(&l2_resource_bounds, L2_GAS)?,
    ];

    // For new V3 txs, need to also hash the data gas bounds.
    resource_felts.extend(match resource_bounds {
        ValidResourceBounds::L1Gas(_) => vec![],
        ValidResourceBounds::AllResources(all_resources) => {
            vec![get_concat_resource(&all_resources.l1_data_gas, L1_DATA_GAS)?]
        }
    });

    Ok(HashChain::new().chain(&tip.0.into()).chain_iter(resource_felts.iter()).get_poseidon_hash())
}

impl TryFrom<protobuf::ResourceBounds> for ValidResourceBounds {
    type Error = ProtobufConversionError;
    fn try_from(value: protobuf::ResourceBounds) -> Result<Self, Self::Error> {
        let Some(l1_gas) = value.l1_gas else {
            return Err(missing("ResourceBounds::l1_gas"));
        };
        let Some(l2_gas) = value.l2_gas else {
            return Err(missing("ResourceBounds::l2_gas"));
        };
        // TODO(Shahak): Assert data gas is not none once we remove support for 0.13.2.
        let l1_data_gas = value.l1_data_gas.unwrap_or_default();
        let l1_gas: ResourceBounds = l1_gas.try_into()?;
        let l2_gas: ResourceBounds = l2_gas.try_into()?;
        let l1_data_gas: ResourceBounds = l1_data_gas.try_into()?;
        Ok(if l1_data_gas.is_zero() && l2_gas.is_zero() {
            ValidResourceBounds::L1Gas(l1_gas)
        } else {
            ValidResourceBounds::AllResources(AllResourceBounds { l1_gas, l2_gas, l1_data_gas })
        })
    }
