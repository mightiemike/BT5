Audit Report

## Title
`SwapAllowlistExtension.beforeSwap` checks router address instead of end-user, enabling full allowlist bypass — (`metric-periphery/contracts/extensions/SwapAllowlistExtension.sol`)

## Summary
`SwapAllowlistExtension.beforeSwap` gates swap access by checking `sender`, which is `msg.sender` of `pool.swap()`. When swaps are routed through `MetricOmmSimpleRouter`, `msg.sender` of `pool.swap()` is the router contract, not the end user. If a pool admin allowlists the router address to enable router-based swaps for permitted users, every user — including non-allowlisted ones — can bypass the restriction by routing through the router, because the extension only checks whether the router is allowlisted, not the individual caller.

## Finding Description
**Call chain:**
```
User → MetricOmmSimpleRouter.exactInputSingle()
     → pool.swap(recipient=user, ...)   // msg.sender = router
```

In `MetricOmmPool.swap`, `_beforeSwap` is called with `msg.sender` as `sender`:
```solidity
_beforeSwap(msg.sender, recipient, ...);  // msg.sender = router
```
`ExtensionCalling._beforeSwap` forwards this as the first argument to the extension:
```solidity
abi.encodeCall(IMetricOmmExtensions.beforeSwap, (sender, recipient, ...))
```
`SwapAllowlistExtension.beforeSwap` then checks:
```solidity
if (!allowAllSwappers[msg.sender] && !allowedSwapper[msg.sender][sender]) {
    revert IMetricOmmPoolActions.NotAllowedToSwap();
}
```
Here `sender` = router address. The extension checks `allowedSwapper[pool][router]`, not `allowedSwapper[pool][user]`.

**Exploit path:**
1. Admin deploys pool with `SwapAllowlistExtension` in `BEFORE_SWAP_ORDER`.
2. Admin calls `setAllowedToSwap(pool, alice, true)` — only Alice is permitted.
3. Admin calls `setAllowedToSwap(pool, router, true)` to allow router-based swaps for Alice.
4. Bob (not allowlisted) calls `MetricOmmSimpleRouter.exactInputSingle(pool, ...)`.
5. Router calls `pool.swap(recipient=bob, ...)` — `msg.sender` = router.
6. Extension checks `allowedSwapper[pool][router]` → `true` → swap succeeds for Bob.

The invariant "only allowlisted users may swap" is broken. Allowlisting the router grants unrestricted access to all users indiscriminately.

**Contrast with `DepositAllowlistExtension`:** `beforeAddLiquidity` correctly checks `owner` (the position owner, the economically relevant actor), not `sender` (the caller of `addLiquidity`). The swap extension lacks this correct binding.

## Impact Explanation
A curated pool using `SwapAllowlistExtension` to restrict trading to vetted counterparties loses that restriction entirely once the router is allowlisted. Any unprivileged user can execute swaps at oracle-anchored prices the pool admin intended to reserve for specific parties. If the pool's oracle pricing is favorable and the allowlist is the sole access control, LP principal is directly at risk of extraction. This matches the "broken core pool functionality causing loss of funds" and "allowlist bypass" impact categories.

## Likelihood Explanation
`MetricOmmSimpleRouter` is the standard, documented periphery entry point for swaps. Allowlisting the router is the natural operational step a pool admin takes to enable router-based swaps for permitted users. No special privileges, malicious setup, or non-standard tokens are required. The bypass is triggered by ordinary router usage and is repeatable by any user.

## Recommendation
In `SwapAllowlistExtension.beforeSwap`, check `recipient` (the second argument, the beneficiary of the swap output) instead of `sender` (the router intermediary):
```solidity
function beforeSwap(address, address recipient, bool, int128, uint128, uint256, uint128, uint128, bytes calldata)
    external view override returns (bytes4)
{
    if (!allowAllSwappers[msg.sender] && !allowedSwapper[msg.sender][recipient]) {
        revert IMetricOmmPoolActions.NotAllowedToSwap();
    }
    return IMetricOmmExtensions.beforeSwap.selector;
}
```
This mirrors how `DepositAllowlistExtension.beforeAddLiquidity` checks `owner` rather than `sender`.

## Proof of Concept
1. Deploy pool with `SwapAllowlistExtension` in `BEFORE_SWAP_ORDER`.
2. Call `setAllowedToSwap(pool, alice, true)`.
3. Call `setAllowedToSwap(pool, router, true)` (admin's natural step to enable router usage for Alice).
4. Bob (not allowlisted) calls `MetricOmmSimpleRouter.exactInputSingle({pool: pool, recipient: bob, ...})`.
5. Router calls `pool.swap(recipient=bob, ...)` with `msg.sender = router`.
6. `beforeSwap(sender=router, recipient=bob, ...)` checks `allowedSwapper[pool][router]` → `true` → swap succeeds.
7. Bob has bypassed the allowlist. Repeat for any number of non-allowlisted users.

Foundry test: set up pool with extension, allowlist only Alice and the router, assert Bob's swap via router succeeds (demonstrating bypass), and assert Bob's direct `pool.swap()` call reverts (confirming the allowlist is otherwise enforced).
