### Title
`ValidResourceBounds` Protobuf Round-Trip Silently Mutates Variant, Causing Transaction Hash Preimage Divergence - (File: crates/apollo_protobuf/src/converters/transaction.rs)

### Summary

The protobuf deserializer for `ValidResourceBounds` silently converts `AllResources { l2_gas: 0, l1_data_gas: 0 }` into `L1Gas`. Because `get_tip_resource_bounds_hash` branches on the variant to decide whether to include `l1_data_gas` in the Poseidon hash preimage, a transaction whose resource bounds are serialized as `AllResources` and deserialized as `L1Gas` produces a different transaction hash on the receiving side. This breaks the one-to-one invariant between a transaction body and its canonical hash across the protobuf serialization boundary.

### Finding Description

`RpcInvokeTransactionV3`, `RpcDeclareTransactionV3`, and `RpcDeployAccountTransactionV3` all store `resource_bounds` as `AllResourceBounds` (always the three-resource variant). When converted to the canonical `InvokeTransactionV3` / `DeclareTransactionV3` / `DeployAccountTransactionV3`, the field is wrapped unconditionally as `ValidResourceBounds::AllResources`:

```rust
// crates/starknet_api/src/rpc_transaction.rs:568-583
impl From<RpcInvokeTransactionV3> for InvokeTransactionV3 {
    fn from(tx: RpcInvokeTransactionV3) -> Self {
        Self {
            resource_bounds: ValidResourceBounds::AllResources(tx.resource_bounds),
            ...
        }
    }
}
```

The transaction hash is then computed by `get_tip_resource_bounds_hash`, which includes `l1_data_gas` in the Poseidon preimage **only** for the `AllResources` variant:

```rust
// crates/starknet_api/src/transaction_hash.rs:203-208
resource_felts.extend(match resource_bounds {
    ValidResourceBounds::L1Gas(_) => vec![],
    ValidResourceBounds::AllResources(all_resources) => {
        vec![get_concat_resource(&all_resources.l1_data_gas, L1_DATA_GAS)?]
    }
});
```

So a transaction with `AllResources { l1_gas: X, l2_gas: 0, l1_data_gas: 0 }` produces a hash that includes the `L1_DATA_GAS` zero-element in the preimage.

When this canonical transaction is serialized to protobuf for P2P block sync and then deserialized, the `ValidResourceBounds` converter applies the following logic:

```rust
// crates/apollo_protobuf/src/converters/transaction.rs:431-435
Ok(if l1_data_gas.is_zero() && l2_gas.is_zero() {
    ValidResourceBounds::L1Gas(l1_gas)
} else {
    ValidResourceBounds::AllResources(AllResourceBounds { l1_gas, l2_gas, l1_data_gas })
})
```

Because `l2_gas = 0` and `l1_data_gas = 0`, the deserialized value is `ValidResourceBounds::L1Gas(X)`. The hash recomputed from this deserialized transaction **omits** the `L1_DATA_GAS` element, producing a different hash than the one originally committed to the block.

The two hashes diverge:
- **Original** (at sequencer): `poseidon(tip, L1_GAS_X, L2_GAS_0, L1_DATA_GAS_0)`
- **After round-trip** (at syncing node): `poseidon(tip, L1_GAS_X, L2_GAS_0)`

### Impact Explanation

Any component that recomputes the transaction hash after protobuf deserialization — including block hash verification during P2P sync — will derive a hash that does not match the hash committed to the block. This causes syncing nodes to reject valid blocks containing such transactions. The block hash is derived from transaction hashes; a divergent transaction hash propagates to a divergent block hash, making the block unverifiable by any honest peer.

This matches: **High. Transaction conversion or signature/hash logic binds the wrong signer, hash, type, or executable payload.**

### Likelihood Explanation

The trigger is a user-submitted transaction with `AllResourceBounds { l1_gas: X, l2_gas: 0, l1_data_gas: 0 }`. The gateway stateless validator accepts transactions with only `l1_gas` non-zero (confirmed by test cases in `crates/apollo_gateway/src/stateless_transaction_validator_test.rs`). The transaction is accepted, included in a block, and the block is then rejected by all syncing peers. No privileged access is required; any unprivileged user can trigger this by submitting a standard invoke transaction with zero `l2_gas` and `l1_data_gas`.

### Recommendation

The `TryFrom<protobuf::ResourceBounds> for ValidResourceBounds` converter must not silently change the variant. Two options:

1. **Preserve the variant**: Require an explicit discriminator field in the protobuf message to distinguish `L1Gas` from `AllResources`, and reject messages that lack it.
2. **Canonicalize at hash time**: Change `get_tip_resource_bounds_hash` to always include `l1_data_gas` regardless of variant (i.e., treat `L1Gas` as `AllResources` with zero `l2_gas` and `l1_data_gas`), making the hash invariant to the variant.

Option 2 is the simpler fix and mirrors the remediation in the NFT analog: enforce the invariant at the point where it matters (the hash boundary), rather than relying on all serialization paths to preserve the variant.

### Proof of Concept

```rust
use starknet_api::transaction::fields::{
    AllResourceBounds, ResourceBounds, ValidResourceBounds,
};
use starknet_api::transaction_hash::get_tip_resource_bounds_hash;
use starknet_api::transaction::fields::Tip;

// Transaction as submitted via RPC: AllResources with zero l2/l1_data
let all_resources = ValidResourceBounds::AllResources(AllResourceBounds {
    l1_gas: ResourceBounds { max_amount: GasAmount(100), max_price_per_unit: GasPrice(1) },
    l2_gas: ResourceBounds::default(),   // zero
    l1_data_gas: ResourceBounds::default(), // zero
});

// Transaction after protobuf round-trip: silently becomes L1Gas
let l1_gas_only = ValidResourceBounds::L1Gas(
    ResourceBounds { max_amount: GasAmount(100), max_price_per_unit: GasPrice(1) }
);

let tip = Tip(0);
let hash_before = get_tip_resource_bounds_hash(&all_resources, &tip).unwrap();
let hash_after  = get_tip_resource_bounds_hash(&l1_gas_only,   &tip).unwrap();

// hash_before != hash_after — same transaction body, different hash preimage
assert_ne!(hash_before, hash_after);
```

// An implementation of the SNIP: https://github.com/EvyatarO/SNIPs/blob/snip-8/SNIPS/snip-8.md
pub fn get_tip_resource_bounds_hash(
    resource_bounds: &ValidResourceBounds,
    tip: &Tip,
) -> Result<Felt, StarknetApiError> {
    let l1_resource_bounds = resource_bounds.get_l1_bounds();
    let l2_resource_bounds = resource_bounds.get_l2_bounds();

    // L1 and L2 gas bounds always exist.
    // Old V3 txs always have L2 gas bounds of zero, but they exist.
    let mut resource_felts = vec![
        get_concat_resource(&l1_resource_bounds, L1_GAS)?,
        get_concat_resource(&l2_resource_bounds, L2_GAS)?,
    ];

    // For new V3 txs, need to also hash the data gas bounds.
    resource_felts.extend(match resource_bounds {
        ValidResourceBounds::L1Gas(_) => vec![],
        ValidResourceBounds::AllResources(all_resources) => {
            vec![get_concat_resource(&all_resources.l1_data_gas, L1_DATA_GAS)?]
        }
    });

    Ok(HashChain::new().chain(&tip.0.into()).chain_iter(resource_felts.iter()).get_poseidon_hash())
}
