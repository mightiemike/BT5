### Title
Unchecked `transferFrom` Return Value Enables usdcE Drain Without USDC Payment - (File: `core/contracts/ContractOwner.sol`)

### Summary
`ContractOwner.replaceUsdcEWithUsdc()` calls `IERC20Base(usdc).transferFrom(msg.sender, directDepositV1, balance)` without checking the return value. If the transfer silently fails (returns `false`), the function continues to withdraw usdcE from the DDA and send it to the caller, resulting in the caller receiving usdcE tokens without providing any USDC.

### Finding Description
In `ContractOwner.sol`, the function `replaceUsdcEWithUsdc` is designed to swap usdcE held in a `DirectDepositV1` (DDA) contract for USDC provided by the caller. The intended flow is:
1. Pull USDC from `msg.sender` into the DDA.
2. Withdraw usdcE from the DDA to `ContractOwner`.
3. Forward usdcE to `msg.sender`.

At line 616, the USDC pull uses a raw `.transferFrom()` call on `IERC20Base`:

```solidity
IERC20Base(usdc).transferFrom(msg.sender, directDepositV1, balance);
```

The return value (`bool`) is never checked. The contract uses `ERC20Helper.safeTransferFrom` (which validates the return value via a low-level `call` and `require`) everywhere else in the codebase, but this specific call bypasses that wrapper entirely.

If the USDC token returns `false` instead of reverting (e.g., due to insufficient allowance or a non-standard implementation), execution continues to lines 617–618, which unconditionally withdraw usdcE from the DDA and transfer it to `msg.sender`.

The function has no `onlyOwner` or `onlyDeployer` modifier — only a `block.chainid == 57073` guard — making it callable by any unprivileged user on that chain.

### Impact Explanation
An unprivileged caller with zero USDC allowance granted to `ContractOwner` can call `replaceUsdcEWithUsdc` for any subaccount whose DDA holds a usdcE balance. The USDC pull silently fails, but the usdcE withdrawal and forwarding proceed. The caller receives the full usdcE balance of the targeted DDA for free. This is a direct asset theft of usdcE tokens from user DDAs.

### Likelihood Explanation
The function is externally callable by any address on chain 57073 with no privilege requirement. The USDC token at the hardcoded address `0x2D270e6886d130D724215A266106e6832161EAEd` is a non-standard or bridged token whose failure behavior may differ from canonical USDC. Even if the current deployment's USDC reverts on failure, the unchecked pattern is a latent vulnerability that becomes exploitable if the token is upgraded or replaced. The trigger path is direct and requires no special setup beyond identifying a DDA with a nonzero usdcE balance.

### Recommendation
Replace the raw `.transferFrom()` call with the `safeTransferFrom` wrapper already used throughout the codebase:

```solidity
// Before (unsafe):
IERC20Base(usdc).transferFrom(msg.sender, directDepositV1, balance);

// After (safe):
IERC20Base(usdc).safeTransferFrom(msg.sender, directDepositV1, balance);
```

`ERC20Helper.safeTransferFrom` is already imported and in scope via `using ERC20Helper for IERC20Base` at line 24 of `ContractOwner.sol`.

### Proof of Concept
1. A DDA exists for some `subaccount` with `usdcE.balanceOf(directDepositV1) = X > 0`.
2. Attacker calls `replaceUsdcEWithUsdc(subaccount)` with zero USDC allowance granted to `ContractOwner`.
3. Line 616: `IERC20Base(usdc).transferFrom(attacker, directDepositV1, X)` returns `false` (no revert).
4. Line 617: `DirectDepositV1(directDepositV1).withdraw(usdcE)` transfers `X` usdcE from DDA to `ContractOwner`.
5. Line 618: `IERC20Base(usdcE).safeTransfer(attacker, X)` transfers `X` usdcE to attacker.
6. Attacker receives `X` usdcE; DDA loses its entire usdcE balance; no USDC was paid. [1](#0-0) [2](#0-1) [3](#0-2)

```text
    function replaceUsdcEWithUsdc(bytes32 subaccount) external {
        require(block.chainid == 57073, ERR_UNAUTHORIZED);
        address payable directDepositV1 = directDepositV1Address[subaccount];
        require(directDepositV1 != address(0), "no dda");
        address usdcE = 0xF1815bd50389c46847f0Bda824eC8da914045D14;
        address usdc = 0x2D270e6886d130D724215A266106e6832161EAEd;
        uint256 balance = IERC20Base(usdcE).balanceOf(directDepositV1);
        if (balance > 0) {
            IERC20Base(usdc).transferFrom(msg.sender, directDepositV1, balance);
            DirectDepositV1(directDepositV1).withdraw(IIERC20Base(usdcE));
            IERC20Base(usdcE).safeTransfer(msg.sender, balance);
        }
    }
```

```text
    function safeTransferFrom(
        IERC20Base self,
        address from,
        address to,
        uint256 amount
    ) internal {
        (bool success, bytes memory data) = address(self).call(
            abi.encodeWithSelector(
                IERC20Base.transferFrom.selector,
                from,
                to,
                amount
            )
        );

        require(
            success && (data.length == 0 || abi.decode(data, (bool))),
            ERR_TRANSFER_FAILED
        );
    }
```
