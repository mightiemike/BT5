### Title
`ValidResourceBounds` Variant Silently Mutated During Protobuf Conversion Produces Wrong Transaction Hash - (File: crates/apollo_protobuf/src/converters/transaction.rs)

### Summary

The protobuf deserializer for `ValidResourceBounds` silently converts an `AllResources` variant (with zero `l2_gas` and `l1_data_gas`) into the `L1Gas` variant. Because `get_tip_resource_bounds_hash` produces structurally different hash preimages for these two variants, the transaction hash computed after a P2P round-trip diverges from the hash computed at gateway admission. A transaction signed and admitted with `AllResources` bounds (even with zero L2/data-gas values) will carry a different hash on any node that received it via P2P sync.

### Finding Description

**Step 1 – The silent variant mutation in the protobuf converter.**

`TryFrom<protobuf::ResourceBounds> for ValidResourceBounds` applies the following rule:

```rust
Ok(if l1_data_gas.is_zero() && l2_gas.is_zero() {
    ValidResourceBounds::L1Gas(l1_gas)
} else {
    ValidResourceBounds::AllResources(AllResourceBounds { l1_gas, l2_gas, l1_data_gas })
})
``` [1](#0-0) 

Any `AllResources` transaction whose `l2_gas` and `l1_data_gas` fields are both `{max_amount:0, max_price_per_unit:0}` is silently re-tagged as `L1Gas` after deserialization.

**Step 2 – The hash preimage differs between the two variants.**

`get_tip_resource_bounds_hash` builds the hash input conditionally on the variant:

```rust
resource_felts.extend(match resource_bounds {
    ValidResourceBounds::L1Gas(_) => vec![],
    ValidResourceBounds::AllResources(all_resources) => {
        vec![get_concat_resource(&all_resources.l1_data_gas, L1_DATA_GAS)?]
    }
});
``` [2](#0-1) 

- `L1Gas` → hash input = `[tip, L1_GAS_concat, L2_GAS_concat(zero)]`
- `AllResources(l2=0, l1_data=0)` → hash input = `[tip, L1_GAS_concat, L2_GAS_concat(zero), L1_DATA_GAS_concat(zero)]`

These are distinct Poseidon inputs and produce distinct digests.

**Step 3 – The gateway always uses `AllResources`.**

`RpcInvokeTransactionV3` carries `AllResourceBounds` (not `ValidResourceBounds`), so every RPC-submitted invoke V3 transaction is converted to `ValidResourceBounds::AllResources` before hashing: [3](#0-2) 

The gateway computes and stores hash **H₁** (the `AllResources` hash). When the transaction is propagated over P2P and deserialized via protobuf, the variant becomes `L1Gas`, and any node that recomputes the hash produces **H₂ ≠ H₁**.

**Step 4 – The `InvokeTransactionV3` storage type and the `InvokeTransactionV3Trait` implementation both delegate to `get_invoke_transaction_v3_hash`, which calls `get_tip_resource_bounds_hash`.** [4](#0-3) 

There is no gate that canonicalises the variant before hashing; the divergence is structural.

### Impact Explanation

A valid transaction admitted by the gateway with hash H₁ will be assigned hash H₂ by any node that received it via P2P sync. Depending on whether the receiving node trusts the transmitted hash or recomputes it:

- **Recompute path**: the transaction is rejected as having an invalid hash → valid transactions are silently dropped from the P2P-synced mempool.
- **Trust-transmitted-hash path**: the transaction is stored with the correct hash H₁ but with the wrong `ValidResourceBounds` variant (`L1Gas` instead of `AllResources`), causing the wrong `GasVectorComputationMode` (`NoL2Gas` instead of `All`) to be used during fee enforcement and execution.

Both outcomes match the allowed impact scope: **High – Mempool/gateway/RPC admission accepts invalid transactions or rejects valid transactions before sequencing** and **High – Transaction conversion or signature/hash logic binds the wrong signer, hash, type, or executable payload**.

### Likelihood Explanation

The trigger condition is an `AllResources` transaction where both `l2_gas.max_amount == 0 && l2_gas.max_price_per_unit == 0` and `l1_data_gas.max_amount == 0 && l1_data_gas.max_price_per_unit == 0`. This is a reachable, unprivileged user-controlled input: any user who submits an invoke V3 transaction with zero L2 and data-gas bounds (a common pattern for transactions that only consume L1 gas) will trigger the conversion. The TODO comment in the converter (`// TODO(Shahak): Assert data gas is not none once we remove support for 0.13.2`) confirms this path is intentionally kept open. [5](#0-4) 

### Recommendation

Remove the `L1Gas` / `AllResources` heuristic from the protobuf deserializer. The variant should be determined by the presence of the `l1_data_gas` field in the wire message, not by whether the numeric values happen to be zero. Alternatively, canonicalise the variant **before** hashing by always treating a missing `l1_data_gas` as `L1Gas` and a present (even zero) `l1_data_gas` as `AllResources`, and enforce this invariant consistently across the gateway, protobuf converter, and storage serializer.

### Proof of Concept

```rust
use starknet_api::transaction::fields::{
    AllResourceBounds, GasAmount, GasPrice, ResourceBounds, ValidResourceBounds,
};
use starknet_api::transaction_hash::get_tip_resource_bounds_hash;
use starknet_api::transaction::fields::Tip;

let l1_bounds = ResourceBounds { max_amount: GasAmount(100), max_price_per_unit: GasPrice(1) };
let zero_bounds = ResourceBounds { max_amount: GasAmount(0), max_price_per_unit: GasPrice(0) };

// Hash as AllResources (what the gateway computes)
let all_resources = ValidResourceBounds::AllResources(AllResourceBounds {
    l1_gas: l1_bounds,
    l2_gas: zero_bounds,
    l1_data_gas: zero_bounds,
});
let h1 = get_tip_resource_bounds_hash(&all_resources, &Tip(0)).unwrap();

// Hash as L1Gas (what the protobuf deserializer produces after round-trip)
let l1_gas = ValidResourceBounds::L1Gas(l1_bounds);
let h2 = get_tip_resource_bounds_hash(&l1_gas, &Tip(0)).unwrap();

// h1 != h2 — the full transaction hash diverges between gateway and P2P-synced nodes
assert_ne!(h1, h2);
```

The `AllResources` variant feeds three elements into the Poseidon chain (L1_GAS, L2_GAS, L1_DATA_GAS), while `L1Gas` feeds only two (L1_GAS, L2_GAS), producing a structurally different digest for identical numeric resource values. [6](#0-5) [7](#0-6)

impl TryFrom<protobuf::ResourceBounds> for ValidResourceBounds {
    type Error = ProtobufConversionError;
    fn try_from(value: protobuf::ResourceBounds) -> Result<Self, Self::Error> {
        let Some(l1_gas) = value.l1_gas else {
            return Err(missing("ResourceBounds::l1_gas"));
        };
        let Some(l2_gas) = value.l2_gas else {
            return Err(missing("ResourceBounds::l2_gas"));
        };
        // TODO(Shahak): Assert data gas is not none once we remove support for 0.13.2.
        let l1_data_gas = value.l1_data_gas.unwrap_or_default();
        let l1_gas: ResourceBounds = l1_gas.try_into()?;
        let l2_gas: ResourceBounds = l2_gas.try_into()?;
        let l1_data_gas: ResourceBounds = l1_data_gas.try_into()?;
        Ok(if l1_data_gas.is_zero() && l2_gas.is_zero() {
            ValidResourceBounds::L1Gas(l1_gas)
        } else {
            ValidResourceBounds::AllResources(AllResourceBounds { l1_gas, l2_gas, l1_data_gas })
        })
    }

pub fn get_tip_resource_bounds_hash(
    resource_bounds: &ValidResourceBounds,
    tip: &Tip,
) -> Result<Felt, StarknetApiError> {
    let l1_resource_bounds = resource_bounds.get_l1_bounds();
    let l2_resource_bounds = resource_bounds.get_l2_bounds();

    // L1 and L2 gas bounds always exist.
    // Old V3 txs always have L2 gas bounds of zero, but they exist.
    let mut resource_felts = vec![
        get_concat_resource(&l1_resource_bounds, L1_GAS)?,
        get_concat_resource(&l2_resource_bounds, L2_GAS)?,
    ];

    // For new V3 txs, need to also hash the data gas bounds.
    resource_felts.extend(match resource_bounds {
        ValidResourceBounds::L1Gas(_) => vec![],
        ValidResourceBounds::AllResources(all_resources) => {
            vec![get_concat_resource(&all_resources.l1_data_gas, L1_DATA_GAS)?]
        }
    });

    Ok(HashChain::new().chain(&tip.0.into()).chain_iter(resource_felts.iter()).get_poseidon_hash())
}

pub(crate) fn get_invoke_transaction_v3_hash<T: InvokeTransactionV3Trait>(
    transaction: &T,
    chain_id: &ChainId,
    transaction_version: &TransactionVersion,
) -> Result<TransactionHash, StarknetApiError> {
    let tip_resource_bounds_hash =
        get_tip_resource_bounds_hash(&transaction.resource_bounds(), transaction.tip())?;
    let paymaster_data_hash =
        HashChain::new().chain_iter(transaction.paymaster_data().0.iter()).get_poseidon_hash();
    let data_availability_mode = concat_data_availability_mode(
        transaction.nonce_data_availability_mode(),
        transaction.fee_data_availability_mode(),
    );
    let account_deployment_data_hash = HashChain::new()
        .chain_iter(transaction.account_deployment_data().0.iter())
        .get_poseidon_hash();
    let calldata_hash =
        HashChain::new().chain_iter(transaction.calldata().0.iter()).get_poseidon_hash();
    let mut hash_chain = HashChain::new()
        .chain(&INVOKE)
        .chain(&transaction_version.0)
        .chain(transaction.sender_address().0.key())
        .chain(&tip_resource_bounds_hash)
        .chain(&paymaster_data_hash)
        .chain(&Felt::try_from(chain_id)?)
        .chain(&transaction.nonce().0)
        .chain(&data_availability_mode)
        .chain(&account_deployment_data_hash)
        .chain(&calldata_hash);
    if !transaction.proof_facts().0.is_empty() {
        let proof_facts_hash =
            HashChain::new().chain_iter(transaction.proof_facts().0.iter()).get_poseidon_hash();
        hash_chain = hash_chain.chain(&proof_facts_hash);
    }
    Ok(TransactionHash(hash_chain.get_poseidon_hash()))
}
