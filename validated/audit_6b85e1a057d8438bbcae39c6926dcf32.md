### Title
`gas_prices_to_hash` omits `l2_gas_consumed` and `next_l2_gas_price` from block hash for Starknet version ≥ 0.14.0 due to missing `BlockHashVersion` gate — (`File: crates/starknet_api/src/block_hash/block_hash_calculator.rs`)

---

### Summary

The `gas_prices_to_hash` function, which produces the gas-price sub-hash committed inside every block hash, does not include `l2_gas_consumed` or `next_l2_gas_price` for any Starknet version. A developer TODO explicitly marks these fields as required "after 0.14.0", the current production version is already 0.14.4, and the echonet OS-verification tool already passes both values to the block-hash CLI. Because `PartialBlockHashComponents` never carries these fields and `BlockHashVersion` has no `V0_14_0` gate, every block produced at version ≥ 0.14.0 carries a canonically wrong block hash.

---

### Finding Description

`gas_prices_to_hash` is the sole function that serialises gas-price data into the Poseidon block-hash preimage. For `BlockHashVersion::V0_13_4` (which covers all versions ≥ 0.13.4, including 0.14.x) it hashes exactly seven fields under the `STARKNET_GAS_PRICES0` domain tag:

```
Poseidon("STARKNET_GAS_PRICES0",
         l1_gas_price_wei, l1_gas_price_fri,
         l1_data_gas_price_wei, l1_data_gas_price_fri,
         l2_gas_price_wei, l2_gas_price_fri)
```

The inline TODO immediately above the function body reads:

> `// TODO(Ayelet): add l2_gas_consumed, next_l2_gas_price after 0.14.0.` [1](#0-0) 

The `BlockHashVersion` enum has only two variants (`V0_13_2`, `V0_13_4`); the `TryFrom<StarknetVersion>` conversion maps every version ≥ 0.13.4 — including 0.14.0, 0.14.3, 0.14.4 — to `V0_13_4`, so no new gate can ever fire. [2](#0-1) 

`PartialBlockHashComponents`, the only struct fed into `calculate_block_hash`, carries `l1_gas_price`, `l1_data_gas_price`, and `l2_gas_price` but has **no** `l2_gas_consumed` or `next_l2_gas_price` fields. [3](#0-2) 

By contrast, `StorageBlockHeader` — the authoritative on-disk representation — already stores both values: [4](#0-3) 

And the echonet OS-verification tool explicitly passes both to the block-hash CLI: [5](#0-4) 

The `FeeMarketInfo` struct that carries these values is already defined and serialised: [6](#0-5) 

The protobuf header converter already deserialises `l2_gas_consumed` and `next_l2_gas_price` from consensus messages: [7](#0-6) 

---

### Impact Explanation

Every block produced at Starknet version ≥ 0.14.0 will have a block hash that omits `l2_gas_consumed` and `next_l2_gas_price` from the gas-prices sub-hash. The echonet verifier and the Starknet OS prover (which already accept these fields) will compute a different hash for the same block header, causing a canonicalization mismatch. Concretely:

- The wrong hash is stored in `StorageBlockHeader.block_hash` and propagated as the `parent_hash` of every subsequent block, poisoning the entire chain of block hashes.
- The wrong hash is committed to L1 in the state-update, causing on-chain proof verification to fail.
- RPC endpoints that return `block_hash` serve an authoritative-looking but incorrect value.

This matches **Critical – Wrong state/storage value** and **Critical – Incorrect L1 gas price effect with economic impact** from the allowed scope.

---

### Likelihood Explanation

The condition is triggered unconditionally for every block at version ≥ 0.14.0. No attacker action is required; the sequencer itself produces the wrong hash on every proposal. The current production starknet version in the test fixtures is already `0.14.4`. [8](#0-7) 

---

### Recommendation

1. Add a `V0_14_0` variant to `BlockHashVersion` and extend `TryFrom<StarknetVersion>` to map versions ≥ 0.14.0 to it.
2. Add `l2_gas_consumed: GasAmount` and `next_l2_gas_price: GasPrice` to `PartialBlockHashComponents`.
3. Update `gas_prices_to_hash` to accept and hash these two fields under a new `STARKNET_GAS_PRICES1` domain tag when `block_hash_version >= V0_14_0`, matching the echonet CLI contract.
4. Update `PartialBlockHashComponents::new` to populate the new fields from the block info / fee-market output.

---

### Proof of Concept

```rust
// Demonstrates that two blocks differing only in l2_gas_consumed
// produce identical hashes under the current implementation.

let make_components = |l2_gas_consumed: u64| PartialBlockHashComponents {
    l2_gas_price: GasPricePerToken {
        price_in_fri: 1_u8.into(),
        price_in_wei: 1_u8.into(),
    },
    starknet_version: StarknetVersion::V0_14_0, // maps to V0_13_4
    // l2_gas_consumed is NOT a field – it is silently dropped
    ..Default::default()
};

// Both calls return the same hash regardless of l2_gas_consumed value,
// because gas_prices_to_hash never receives it.
let hash_a = calculate_block_hash(&make_components(0),   GlobalRoot::default(), BlockHash::default());
let hash_b = calculate_block_hash(&make_components(999), GlobalRoot::default(), BlockHash::default());
assert_eq!(hash_a, hash_b); // passes – wrong block hashes are identical
```

The echonet verifier, which passes the real `l2_gas_consumed` to the CLI, will compute a different value for `hash_b`, exposing the divergence. [9](#0-8) [10](#0-9)

#[allow(non_camel_case_types)]
#[derive(Clone, Debug, PartialEq, Eq, PartialOrd)]
pub enum BlockHashVersion {
    V0_13_2,
    V0_13_4,
}

impl From<BlockHashVersion> for StarknetVersion {
    fn from(value: BlockHashVersion) -> Self {
        match value {
            BlockHashVersion::V0_13_2 => StarknetVersion::V0_13_2,
            BlockHashVersion::V0_13_4 => StarknetVersion::V0_13_4,
        }
    }
}

impl TryFrom<StarknetVersion> for BlockHashVersion {
    type Error = StarknetApiError;

    fn try_from(value: StarknetVersion) -> StarknetApiResult<Self> {
        if value < Self::V0_13_2.into() {
            Err(StarknetApiError::BlockHashVersion { version: value.to_string() })
        } else if value < Self::V0_13_4.into() {
            // Starknet versions 0.13.2 and 0.13.3 both have the same block hash mechanism.
            Ok(Self::V0_13_2)
        } else {
            Ok(Self::V0_13_4)
        }
    }
}

pub fn calculate_block_hash(
    partial_block_hash_components: &PartialBlockHashComponents,
    state_root: GlobalRoot,
    previous_block_hash: BlockHash,
) -> StarknetApiResult<BlockHash> {
    let block_hash_version: BlockHashVersion =
        partial_block_hash_components.starknet_version.try_into()?;
    let block_commitments = &partial_block_hash_components.header_commitments;
    Ok(BlockHash(
        HashChain::new()
            .chain(&block_hash_version.clone().into())
            .chain(&partial_block_hash_components.block_number.0.into())
            .chain(&state_root.0)
            .chain(&partial_block_hash_components.sequencer.0)
            .chain(&partial_block_hash_components.timestamp.0.into())
            .chain(&block_commitments.concatenated_counts)
            .chain(&block_commitments.state_diff_commitment.0.0)
            .chain(&block_commitments.transaction_commitment.0)
            .chain(&block_commitments.event_commitment.0)
            .chain(&block_commitments.receipt_commitment.0)
            .chain_iter(
                gas_prices_to_hash(
                    &partial_block_hash_components.l1_gas_price,
                    &partial_block_hash_components.l1_data_gas_price,
                    &partial_block_hash_components.l2_gas_price,
                    &block_hash_version,
                )
                .iter(),
            )
            .chain(
                &Felt::try_from(&partial_block_hash_components.starknet_version)
                    .expect("Expect ASCII version"),
            )
            .chain(&Felt::ZERO)
            .chain(&previous_block_hash.0)
            .get_poseidon_hash(),
    ))
}
