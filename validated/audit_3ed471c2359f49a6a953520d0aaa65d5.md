Audit Report

## Title
SwapAllowlistExtension Checks Direct Caller Instead of Original User, Enabling Full Allowlist Bypass via Router — (File: metric-periphery/contracts/extensions/SwapAllowlistExtension.sol)

## Summary
`SwapAllowlistExtension.beforeSwap` gates swaps by checking `sender`, which is the direct `msg.sender` of the pool's `swap` call. When users route through `MetricOmmSimpleRouter`, `sender` equals the router's address rather than the original user. A pool admin who allowlists the router to enable router-mediated swaps for their curated users inadvertently opens the pool to every unprivileged caller of the same public router.

## Finding Description
In `MetricOmmPool.swap`, `msg.sender` is forwarded as `sender` to `_beforeSwap`: [1](#0-0) 

`SwapAllowlistExtension.beforeSwap` then checks `allowedSwapper[msg.sender][sender]`, where `msg.sender` is the pool and `sender` is the direct caller of the pool's `swap`: [2](#0-1) 

`MetricOmmSimpleRouter.exactInputSingle` calls `pool.swap(...)` directly, making the router the immediate `msg.sender` inside the pool: [3](#0-2) 

Because the router is the immediate caller of `pool.swap`, the `sender` forwarded to the extension is the router address, not the original user. The allowlist check becomes `allowedSwapper[pool][router_address]`. A pool admin who wants allowlisted users to be able to use the router must add the router to the allowlist. Once the router is allowlisted, the check `allowedSwapper[pool][router]` passes for every call that arrives through the router — regardless of who the original caller is. The same structural problem applies to `exactInput`, `exactOutputSingle`, and `exactOutput`: [4](#0-3) 

No existing guard in `SwapAllowlistExtension` checks the original economic actor. The `extensionData` field passed through the router is caller-supplied and not authenticated, so it cannot serve as a trustworthy identity channel without a coordinated convention.

## Impact Explanation
Any unprivileged user can bypass the swap allowlist on a curated pool by routing through `MetricOmmSimpleRouter`. Curated pools designed for specific users (KYC'd participants, institutional LPs, whitelisted protocols) become accessible to all callers. Unauthorized swaps drain LP inventory at oracle prices, causing direct loss of LP principal and breaking the core access-control invariant the extension was deployed to enforce. This constitutes a direct loss of user principal and broken core pool functionality, meeting the Critical/High threshold under the allowed impact gate.

## Likelihood Explanation
High. `MetricOmmSimpleRouter` is the canonical, publicly documented swap entry point. A pool admin who deploys a swap-allowlisted pool and wants their allowlisted users to use the router has no choice but to allowlist the router address. This is the natural and expected operational step. Once taken, the bypass is trivially reachable by any user with no special privileges, no flash loan, and no admin interaction.

## Recommendation
The extension must check the original economic actor, not the intermediary. The most robust fix is to have the router encode `msg.sender` into `extensionData` using a standardized convention, and have `SwapAllowlistExtension.beforeSwap` decode and check that value when present. Alternatively, document and enforce that `SwapAllowlistExtension` is incompatible with router-mediated flows and must only be used with direct pool calls, enforced at the factory level by rejecting pool configurations that pair this extension with a known router address in the allowlist.

## Proof of Concept
```
1. Deploy pool with SwapAllowlistExtension as beforeSwap hook.
2. Pool admin calls:
     swapExtension.setAllowedToSwap(pool, alice, true);
     swapExtension.setAllowedToSwap(pool, address(router), true);
   (second call is required so alice can use the router)
3. charlie (not allowlisted) calls:
     router.exactInputSingle({pool: pool, recipient: charlie, ...})
4. Router calls pool.swap(...) — msg.sender inside pool = router.
5. SwapAllowlistExtension.beforeSwap receives sender = router.
6. Check: allowedSwapper[pool][router] == true → passes.
7. Charlie's swap executes on the curated pool despite never being allowlisted.
```
The bypass requires zero privileged access from charlie. The only precondition — the router being allowlisted — is the natural consequence of the pool admin trying to make the allowlist work with the supported periphery. [5](#0-4) [6](#0-5)
