Audit Report

## Title
`addLiquidity` Lacks `whenNotPaused` Guard, Allowing LP Deposits Into a Paused Pool — (`metric-core/contracts/MetricOmmPool.sol`)

## Summary

`MetricOmmPool.addLiquidity` is missing the `whenNotPaused` modifier that `swap` correctly applies. When the pool is paused in response to an oracle incident, swaps are blocked but LP deposits remain open. An LP who deposits during the pause has their shares priced against the stale pre-pause bin cursor; on unpause, the live oracle price may have moved significantly, making the LP's position immediately arbitrageable at a loss to their principal.

## Finding Description

The `whenNotPaused` modifier is defined at L174–177 and delegates to `_checkNotPaused()` (L643–644), which reverts if `pauseLevel != 0`. The pause level is set exclusively by the factory via `setPause` (L455–461).

`swap` at L217–224 carries `whenNotPaused nonReentrant(PoolActions.SWAP)`, correctly blocking price-sensitive settlement during a pause.

`addLiquidity` at L182–196 carries only `nonReentrant(PoolActions.ADD_LIQUIDITY)` — `whenNotPaused` is absent. No other guard inside `addLiquidity`, `_beforeAddLiquidity`, or any configured extension (including `DepositAllowlistExtension`, which checks depositor identity only, L32–42) compensates for this omission.

The periphery path through `MetricOmmPoolLiquidityAdder._addLiquidity` (L183–207) calls `IMetricOmmPoolActions(pool).addLiquidity(...)` directly and adds no pause check of its own, so the bypass is reachable via both the direct pool call and the supported periphery path.

Exploit flow:
1. Pool is live with an oracle-anchored price provider.
2. Oracle feed becomes stale or manipulated; factory admin calls `setPause(1)`.
3. `swap` reverts with `PoolPaused`. Swaps are blocked.
4. Any LP calls `pool.addLiquidity(...)` directly or via `MetricOmmPoolLiquidityAdder.addLiquidityExactShares`. The call succeeds; tokens are transferred in and shares are minted at the current (stale) bin cursor.
5. Factory admin resolves the oracle issue and calls `setPause(0)`.
6. The live oracle price has moved. Arbitrageurs immediately swap against the LP's newly deposited position at the stale bin price, extracting value from the LP's principal.

## Impact Explanation

The LP suffers direct principal loss: tokens deposited during the pause are immediately mispriced relative to the resumed oracle price and are extracted by arbitrageurs. This is not theoretical impermanent loss — it is the same mechanism that makes pausing necessary in the first place, now applied to freshly deposited capital. This matches the Metric OMM Allowed Impact Gate: direct loss of user principal and broken core pool functionality causing loss of funds.

## Likelihood Explanation

Triggering the pause requires a factory admin action, which is semi-trusted and specifically designed for emergency use — making it a realistic scenario. Once paused, any unprivileged LP can call `addLiquidity` directly or via `MetricOmmPoolLiquidityAdder` with no special access. The LP may not know why the pool is paused or that depositing during a pause is unsafe. `DepositAllowlistExtension`, if configured, provides no protection because it gates on depositor identity, not pause state.

## Recommendation

Add `whenNotPaused` to `addLiquidity`, mirroring the protection on `swap`:

```solidity
function addLiquidity(
    address owner,
    uint80 salt,
    LiquidityDelta calldata deltas,
    bytes calldata callbackData,
    bytes calldata extensionData
) external whenNotPaused nonReentrant(PoolActions.ADD_LIQUIDITY) returns (uint256 amount0Added, uint256 amount1Added) {
```

`removeLiquidity` should remain unguarded so existing LPs can exit a paused pool — this is the correct asymmetry.

## Proof of Concept

1. Deploy pool with an oracle price provider.
2. Call `setPause(1)` from the factory to pause the pool.
3. Confirm `swap` reverts with `PoolPaused`.
4. Call `pool.addLiquidity(owner, salt, deltas, callbackData, extensionData)` — observe it succeeds and tokens are transferred in.
5. Alternatively, call `MetricOmmPoolLiquidityAdder.addLiquidityExactShares(pool, owner, salt, deltas, max0, max1, extensionData)` — also succeeds.
6. Call `setPause(0)` to unpause.
