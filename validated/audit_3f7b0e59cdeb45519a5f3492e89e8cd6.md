### Title
Wire-format public key length used where trie-identifier length is expected in gas-key host-function exec-fee calculation — (`core/parameters/src/cost.rs`, `runtime/near-vm-runner/src/logic/logic.rs`)

### Summary

The exec-fee functions `gas_key_transfer_exec_fee` and `gas_key_add_key_exec_fee` are documented to model the **trie key length** (what the receiver reads/writes in the trie). For ML-DSA-65 keys, the trie identifier is a 33-byte SHA3-256 hash (`trie_id_len() = 33`), not the 1953-byte borsh-encoded full pubkey. The host-function layer passes the raw **wire-format length** (`public_key_len as usize = 1953`) to both exec-fee functions, while the transaction-level path correctly passes `public_key.trie_id_len()`. This is a direct unit-domain mismatch: wire-format bytes used where trie-identifier bytes are expected, with no conversion.

### Finding Description

`gas_key_transfer_exec_fee` is defined as:

```rust
/// Exec fee for TransferToGasKey / WithdrawFromGasKey actions.
/// Based on the access key trie key length + estimated value length (what the
/// receiver needs to read/write in the trie).
pub fn gas_key_transfer_exec_fee(
    cfg: &RuntimeFeesConfig,
    account_id_len: usize,
    public_key_len: usize,   // <-- must be trie_id_len, not wire len
) -> GasKeyTransferFee {
    let trie_key_len = access_key_key_len(account_id_len, public_key_len);
    ...
}
``` [1](#0-0) 

The host-function layer in both the interpreter and Wasmtime paths passes the raw guest-memory length:

```rust
// logic.rs (interpreter path)
let exec = gas_key_transfer_exec_fee(
    &self.fees_config,
    receiver_id.len(),
    public_key_len as usize,   // wire length: 1953 for ML-DSA-65
);
``` [2](#0-1) 

```rust
// wasmtime path — identical mistake
let exec =
    gas_key_transfer_exec_fee(&ctx.fees_config, receiver_id.len(), public_key_len as usize);
``` [3](#0-2) 

The same mismatch exists for `gas_key_add_key_exec_fee` in both `promise_batch_action_add_gas_key_with_full_access` and `promise_batch_action_add_gas_key_with_function_call`: [4](#0-3) [5](#0-4) 

By contrast, the **transaction-level** path in `runtime/runtime/src/config.rs` correctly uses `trie_id_len()`:

```rust
// send fee — correctly uses trie_id_len
TransferToGasKey(action) => {
    gas_key_transfer_send_fee(fees, sender_is_receiver, action.public_key.trie_id_len())
        .total()
}
``` [6](#0-5) 

```rust
// exec fee for AddKey — correctly uses trie_id_len
let nonce_fee = gas_key_add_key_exec_fee(
    fees,
    account_id.len(),
    public_key.trie_id_len(),   // correct
    gas_key_info.num_nonces,
);
``` [7](#0-6) 

The trie-identifier lengths are:

| Key type  | `trie_id_len()` | borsh wire `len()` |
|-----------|----------------|--------------------|
| ED25519   | 33 bytes       | 33 bytes           |
| SECP256K1 | 65 bytes       | 65 bytes           |
| ML-DSA-65 | **33 bytes**   | **1953 bytes**     | [8](#0-7) 

For ML-DSA-65, `access_key_key_len(account_id_len, 1953)` is used instead of `access_key_key_len(account_id_len, 33)`. For a 10-byte account ID: `(10 + 2 + 1953) / (10 + 2 + 33) ≈ 43.7×` overcharge on the exec-fee per-byte component. [9](#0-8) 

### Impact Explanation

Any contract that calls `promise_batch_action_transfer_to_gas_key`, `promise_batch_action_add_gas_key_with_full_access`, or `promise_batch_action_add_gas_key_with_function_call` with an ML-DSA-65 public key will be charged ~43× the correct exec gas for those actions. This makes contract-initiated ML-DSA-65 gas-key operations practically unusable (out-of-gas failures at normal gas budgets), while the identical operation submitted as a signed transaction succeeds at the correct cost. The divergence is deterministic and protocol-wide: every node computes the same wrong fee, so it does not cause a consensus split, but it permanently misprices a class of operations and breaks the gas-key feature for ML-DSA-65 keys when invoked from contracts.

### Likelihood Explanation

The `PostQuantumSignatures` feature is stabilized at protocol version 85. [10](#0-9)  The `GasKeys` feature is gated behind `ProtocolFeature::GasKeys`. [11](#0-10)  Any contract running after both features are active that attempts to manage ML-DSA-65 gas keys via host functions will trigger the overcharge. The attacker surface is unprivileged: any deployed contract can call these host functions.

### Recommendation

In `promise_batch_action_transfer_to_gas_key`, `promise_batch_action_add_gas_key_with_full_access`, and `promise_batch_action_add_gas_key_with_function_call`, replace `public_key_len as usize` with `public_key.trie_id_len()` when calling the exec-fee functions, mirroring the transaction-level path:

```rust
// After decoding the public key:
let decoded_key = public_key.decode()?;
let exec = gas_key_transfer_exec_fee(
    &self.fees_config,
    receiver_id.len(),
    decoded_key.trie_id_len(),   // trie-identifier length, not wire length
);
```

The send-fee call may legitimately keep `public_key_len as usize` (it models the sender's wire-format cost), but the exec-fee call must use `trie_id_len()`.

### Proof of Concept

1. Deploy a contract that calls `promise_batch_action_transfer_to_gas_key` with an ML-DSA-65 public key (1953-byte borsh encoding, so `public_key_len = 1953`).
2. The host function computes `gas_key_transfer_exec_fee(fees, receiver_id.len(), 1953)`, yielding `trie_key_len = account_id_len + 2 + 1953`.
3. Submit the identical `TransferToGasKey` action as a signed transaction. The runtime computes `gas_key_transfer_exec_fee(fees, account_id.len(), 33)` (via `trie_id_len()`), yielding `trie_key_len = account_id_len + 2 + 33`.
4. Observe that the exec-fee per-byte component charged to the contract is `(account_id_len + 2 + 1953) / (account_id_len + 2 + 33) ≈ 43.7×` larger than the transaction path, causing the contract call to exhaust its gas budget while the transaction succeeds. [12](#0-11) [13](#0-12) [6](#0-5)

/// Exec fee for TransferToGasKey / WithdrawFromGasKey actions.
/// Based on the access key trie key length + estimated value length (what the
/// receiver needs to read/write in the trie).
pub fn gas_key_transfer_exec_fee(
    cfg: &RuntimeFeesConfig,
    account_id_len: usize,
    public_key_len: usize,
) -> GasKeyTransferFee {
    let base = cfg.fee(ActionCosts::gas_key_transfer_base).exec_fee();
    let trie_key_len = access_key_key_len(account_id_len, public_key_len);
    let estimated_value_len = AccessKey::min_gas_key_borsh_len();
    let per_byte = cfg
        .fee(ActionCosts::gas_key_byte)
        .exec_fee()
        .checked_mul((trie_key_len + estimated_value_len) as u64)
        .unwrap();
    GasKeyTransferFee { base, per_byte }
```
